# AI Execution Specs — Quick Reference Index

## What Is This?

This folder contains **AI Execution Specifications** — the non-code equivalent of PRDs designed for human-AI collaboration where autonomous execution is the goal.

**The Core Problem:** When working with AI CLI tools on non-code projects, what document gives both AI and human enough confidence to execute autonomously until completion?

**The Solution:** Documents with PRD-level rigor that remove judgment calls, specify "done" precisely, and embed decision rules for ambiguous situations.

## How to Use

### For New Projects
1. **Pick the closest spec** from the index above
2. **Copy it** to your project folder
3. **Fill in EVERY section** — incomplete specs = AI asking questions = broken flow
4. **Pay special attention to:**
   - Decision Rules (what AI does when uncertain)
   - Completion Standard (how to know it's done)
   - Source Materials (what has authority)

### For AI Execution
1. **Share the completed spec** with Claude/AI tool
2. **AI reads spec** and executes autonomously
3. **AI only stops** when escalation triggers hit
4. **Both parties confident** because ambiguity is pre-resolved

## Key Sections in Every Spec

| Section | Purpose |
|---------|---------|
| **Objective** | What this produces — ruthlessly specific |
| **Success Criteria** | Measurable outcomes — not vibes |
| **Deliverables** | Exact outputs with format specs |
| **Scope** | What's in, what's out |
| **Constraints** | Boundaries, tone, format rules |
| **Source Materials** | What to reference, hierarchy of authority |
| **Decision Rules** | When encountering X, do Y |
| **Completion Standard** | What "done" looks like precisely |

---

## The Mental Model

**A good spec answers these questions BEFORE work starts:**

- What exactly will exist when this is done?
- How do I know it's done vs. "good enough"?
- What do I do when I hit ambiguity?
- What sources have authority?
- What's explicitly NOT my job?

If the AI has to stop and ask "but what about..." — the spec failed.
