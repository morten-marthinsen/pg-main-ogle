# LOMS Curate

Organizes packaged resources into a browsable library that enables quick lesson prep.

## Configuration

Read library path from `~/.claude/skills/loms-config.json`:
```json
{"library_base": "~/Documents/LOMS-Library"}
```

## When to Use

- After `/loms-package` has created today's packages
- When `/loms-run` executes the full pipeline
- Manually to reorganize or update indexes

## What It Creates

### Harvest Library Structure
```
{library_base}/library/harvest/
├── index.md              # Master index
├── usage-log.json        # Track what's been used
└── {year}/
    └── week-{nn}/
        ├── weekly-summary.md
        └── days/
            └── {date}/
                └── index.md
```

## Input/Output

**Input:** `{library_base}/library/packages/`

**Output:** `{library_base}/library/harvest/`

## Execution Instructions

### Step 0: Read Config

Read `~/.claude/skills/loms-config.json` to get library_base.

### Step 1: Determine Week

Calculate ISO week number from today's date.
Create week folder if needed: `{library_base}/library/harvest/{year}/week-{nn}/`

### Step 2: Load Package Data

Read packages from `{library_base}/library/packages/index.md` or scan the folders.

### Step 3: Generate Daily Index

Create `{library_base}/library/harvest/{year}/week-{nn}/days/{date}/index.md`:

```markdown
# LOMS Harvest: {date}

**{count} packages** ready for lessons

---

## Skills ({count})

| Name | Description | Status |
|------|-------------|--------|
| [name](link) | desc | New/Updated |

## Documentation ({count})

| Name | Description | Status |
|------|-------------|--------|

## Patterns ({count})

| Name | Description | Status |
|------|-------------|--------|

---

## Selection

Mark items for today's lesson:
- [ ] Item 1
- [ ] Item 2

---
*Generated by LOMS - {date}*
```

### Step 4: Update Weekly Summary

Update `{library_base}/library/harvest/{year}/week-{nn}/weekly-summary.md`:

```markdown
# Week {nn} - {year}

## Overview

| Metric | Count |
|--------|-------|
| Total Packages | N |
| Skills | N |
| Documentation | N |
| Patterns | N |

## Highlights

{Top 3 items with descriptions}

## Daily Breakdown

| Day | Date | Items | Highlight |
|-----|------|-------|-----------|
| Mon | link | N | item |
| ... | ... | ... | ... |

---
*Generated by LOMS*
```

### Step 5: Update Usage Log

If first run, create `{library_base}/library/harvest/usage-log.json`:
```json
{
  "version": "1.0",
  "lastUpdated": "ISO timestamp",
  "selections": []
}
```

## Notes

**Cross-Platform:** This skill does NOT use symlinks (which fail on Windows).
Instead, it maintains an index.md at `{library_base}/library/harvest/index.md` that links to the current week.

## Edge Cases

| Scenario | Behavior |
|----------|----------|
| Config missing | Use default library path |
| No packages today | Create index noting "No new content" |
| First day of week | Create fresh weekly summary |
| 20+ items in day | Group by category |
