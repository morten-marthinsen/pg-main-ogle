# Mechanism Ideator Enhanced (v3 - Research Library Integrated)

> **Purpose:** Generate fully researched, copy-ready marketing mechanisms from any program content. This enhanced version draws on a 121-file research library of mechanism knowledge - 30 documented mechanisms with full breakdowns, 12 creation frameworks, 15 psychology foundations, 9 naming systems, 10 market playbooks, and 12 synthesis guides - to produce output that would otherwise require deep copywriting expertise.
>
> **What makes this different from the base package:** The base skill relies on the AI's general knowledge + web search. This version reads from a curated research library at every phase, giving it access to specific frameworks from Georgi, Brown, Schwartz, Deutsch, Makepeace, Halbert, Carlton, Albuquerque, Sugarman, Hopkins, Kennedy, Lampropoulos, and others.

## Phase 0: Market Context (REQUIRED - Do This First)

### Required Reading (FROM RESEARCH LIBRARY)

Read these files BEFORE establishing market context:

1. **`research-library/07-Synthesis/02 Market Sophistication Mapping.md`** - Schwartz's 5-stage model, headline formulas per stage
2. **`research-library/05-Markets/00 Market Patterns Index.md`** - Overview of all 10 market playbooks
3. **`research-library/05-Markets/[relevant market playbook].md`** - The SPECIFIC playbook for this product's market

**THEN read the market-specific playbook.** If the product is in business/IM, read `04 Business Opportunity Mechanism Playbook.md`. If health, read `01 Health Supplements Mechanism Playbook.md`. Etc.

Each market playbook contains:
- Buyer psychology (primary motivations + fears)
- Dominant mechanism types for that market
- Copy formulas
- Proof distribution patterns
- Common villains
- Headline formulas by sophistication stage

### Establish

**1. Market Domain** - What market? What mechanism language fits?

| Market | Language | Playbook |
|--------|----------|----------|
| Health/Supplements | Biological, scientific, clinical | `01 Health Supplements` |
| Financial/Investing | Economic, structural, mathematical | `02 Financial Investing` |
| Dating/Relationships | Psychological, demonstration | `03 Dating Relationships` |
| Business Opportunity / IM | Strategic, identity-shifting, urgent | `04 Business Opportunity` |
| Weight Loss | Hormonal, metabolic, paradigm | `05 Weight Loss` |
| Self-Improvement | Psychological, behavioral, identity | `06 Self-Improvement` |
| Survival/Prep | Threat-based, collapse | `07 Survival Preparedness` |
| Golf/Sports | Biomechanical, technique | `08 Golf Sports` |
| SaaS/Technology | ROI, efficiency, automation | `09 SaaS Technology` |
| ED | Blood flow, root cause | `10 Erectile Dysfunction` |

**RULE: Never use health/medical mechanism language for a non-health product.**

**RULE: For entrepreneurial/IM markets, mechanisms must account for:**
- Speed/urgency
- Confusion about the path (problem-aware but solution-confused)
- Identity-level shifts
- FOMO/acceleration
- What they've already tried (courses, coaching, tools, self-learning, hiring)

**2. What the prospect believes** the problem is

**3. What they've tried** that hasn't worked (be exhaustive)

**4. Villain Archetype Selection**

Read **`research-library/07-Synthesis/04 Villain Framework Library.md`** for the FULL villain system (not just the 10 archetypes - includes 7 attack patterns, origin story templates, naming guidelines).

| Archetype | Best When... |
|-----------|-------------|
| **Hidden Enemy** | Something they don't know exists |
| **Betrayer** | Something they trust is hurting them |
| **Blocker** | Something preventing success |
| **Saboteur** | Active undermining |
| **Conspiracy** | System designed against them |
| **Deficiency** | Missing critical element |
| **Outdated System** | Approach worked before but doesn't now |
| **Environmental Mismatch** | World changed but they didn't |
| **Silent Accumulator** | Building up unnoticed |
| **Invisible Tax** | Hidden ongoing cost |
| **Accelerating Threat** | Problem compounds with time |

Villains can be LAYERED (primary + secondary).

**5. Mechanism Approach Classification**

Read **`research-library/07-Synthesis/00 Complete Mechanism Taxonomy.md`** sections on "Mechanism Type Taxonomy (MECE)" for the full classification system.

| Type | Use When... |
|------|------------|
| **Scientific Discovery** | Research/data reveals hidden cause |
| **Paradigm Shift** | Everything they know is wrong |
| **Hidden Enemy** | Unknown villain revealed |
| **Missing Link** | One thing was missing all along |
| **Insider Knowledge** | Expert reveals what others won't |
| **Environmental Shift** | World changed and old playbook expired |

## Phase 2: Mechanism Mining

### Required Reading

1. **`research-library/01-Examples/00 Mechanism Patterns Summary.md`** - The 7 archetypes extracted from 30 documented mechanisms
2. **`research-library/01-Examples/00 Mechanism Examples Index.md`** - Full index of documented mechanisms
3. **Read 3-5 relevant individual examples** from `01-Examples/` that match the market/archetype you're working with. Each example has full UMP/UMS/UMD breakdown, copy analysis, and what made it work.
4. **`research-library/07-Synthesis/06 Component Anatomy Breakdown.md`** - Detailed UMP/UMS/UMD construction guide with templates

### The Five Mechanism Tests

Every candidate MUST pass ALL five. Any failure = cut or rework.

1. **Sales Letter Test:** "It's not [what they think]. It's [MECHANISM] - [explanation]." Sounds like a revelation?
2. **Domain Test:** Belongs in copy for THIS market?
3. **Blame Shift Test:** Shifts blame completely off the prospect?
4. **Demand Test:** Creates demand for this specific solution?
5. **Discovery Test:** Sounds discovered, not invented?

### Anti-Patterns

Read **`research-library/07-Synthesis/03 Concept Distinction Guide.md`** for the full distinction system. Watch for:

1. **Concept Naming** - Renaming teaching content
2. **Domain Mismatch** - Wrong market language
3. **Clever-But-Empty** - Catchy names without real root causes
4. **Borrowed Concepts** - Someone else's named concept
5. **Abstraction Without Specificity** - Vague names
6. **Lists Instead of Mechanisms** - Name-storming instead of mechanism-building

After mining the library, fill remaining gaps with web search agents:

**Agent 1: Academic Foundations**
Search for named researchers, academic papers, historical parallels, key citations.

**Agent 2: Modern Case Studies**
Search for 3-5 case studies per mechanism with SPECIFIC numbers.

**Agent 4: Statistics & Proof Points**
Search for at least 5 statistics per mechanism with sources.

**Agent 5-7: Topic-Specific** (adapt to mechanism content)

### Step 3: Synthesize

Create `Research-Package.md` per mechanism + `/Research/` folder with cross-mechanism files.

## Phase 8: Output Structure

### OUTPUT FORMAT RULES (NON-NEGOTIABLE)

1. **Every mechanism MUST include full narrative prose sections** — not compressed bullet summaries, not one-liners inside structured boxes. Each of the five narrative sections (Hidden Root Cause, Why They Don't Know About It, How It Blocks Them, The Blame Shift, The Solution Logic) must be a **standalone paragraph** that tells the story of why this mechanism matters.

2. **Maximum 5 mechanisms per Mechanisms.md.** If you have more than 5 viable mechanisms, rank them and include only the top 5. Additional mechanisms can go in a separate `Additional-Mechanisms.md` file. This prevents output compression.

3. **The "How These Layer in Copy" section must be narrative prose** — a multi-paragraph story arc explaining which mechanism opens, which deepens, how they hand off. Not a table. Not a numbered list.

4. **Never abbreviate the per-mechanism format.** If context is getting long, split into multiple files rather than compressing the format.

5. **The Five Tests table must include full draft sales letter text** — not a one-sentence summary. Each test row gets the actual copy you'd use.
