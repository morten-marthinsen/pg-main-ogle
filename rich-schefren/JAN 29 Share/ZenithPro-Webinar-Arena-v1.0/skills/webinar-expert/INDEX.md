## System Components

### 1. Webinar-Expert Meta-Skill
**Purpose:** Synthesize multiple webinar experts into cohesive, context-aware recommendations

**4 Operational Modes:**
1. **Creation Mode** - Build webinars using synthesized best practices
2. **Analysis Mode** - Evaluate webinars against all expert standards
3. **Optimization Mode** - Diagnose and fix specific webinar issues
4. **Expert-Specific Mode** - Get one expert's pure perspective

**Key Features:**
- Cross-expert synthesis
- Conflict resolution
- Context-based recommendations
- Best-of-breed combinations

### 2. Webinar-Critic Agent
**Purpose:** Comprehensive evaluation against ALL expert standards using isolated context

**5-Tier Evaluation System:**
1. **Structural Compliance** - Framework adherence
2. **Principle Alignment** - Expert psychology application
3. **Contextual Appropriateness** - Right approach for context
4. **Craft Quality** - Execution excellence
5. **Empirical Benchmarks** - Performance metrics

**Key Features:**
- Multi-expert evaluation
- Prioritized improvement roadmap
- Testing recommendations
- Performance diagnosis

### 3. Synthesis Map
**Purpose:** Document where experts agree, where they differ, and how to resolve conflicts

**Key Sections:**
- Universal Truths (10 principles where experts agree)
- Fladlien Unique Contributions (7 frameworks only he provides)
- Cage Unique Contributions (8 frameworks only he provides)
- Conflict Resolution Rules (5 major conflicts addressed)
- Best-of-Breed Combinations (6 synthesis patterns)

### 4. Decision Trees
**Purpose:** Navigate which expert's approach to use based on context

**Key Decision Trees:**
- Master: Which expert's approach?
- Opening sequence design
- Content structure
- Close architecture
- Objection handling strategy
- Title development
- Q&A session design
- Meta: When experts completely conflict

### 5. Benchmark Standards
**Purpose:** Empirical metrics and quality standards from all experts

**Key Benchmarks:**
- Time allocation by section and price point
- Content element requirements (yeses, stories, proof, objections, emotions)
- Performance metrics (registration, attendance, conversion, revenue)
- Quality standards (opening, content, close)
- Optimization guidelines

## Quick Start Workflows

### Creating a New Webinar
1. **Context Gathering:** Price, market sophistication, product type, goal
2. **Expert Selection:** Use decision trees to determine approach bias
3. **Creation:** Invoke webinar-expert (Creation Mode)
4. **Evaluation:** Submit to webinar-critic agent
5. **Iteration:** Fix issues until A-grade achieved
6. **Launch:** Deploy and measure

**File Path:** [SKILL.md](rich-schefren/JAN%2029%20Share/ZenithPro-Webinar-Arena-v1.0/skills/webinar-expert/SKILL.md) → Creation Mode

### Diagnosing Underperformance
1. **Data Collection:** Conversion rate, drop-off points, attendance
2. **Diagnostic Evaluation:** Submit to webinar-critic agent
3. **Root Cause Analysis:** Receive priority fix list
4. **Solution Implementation:** Use webinar-expert (Optimization Mode)
5. **Re-testing:** Measure improvement

**File Path:** [Webinar-Critic AGENT.md](../../agents/webinar-critic/AGENT.md)

### Resolving Expert Conflicts
1. **Identify Conflict:** Experts disagree on approach
2. **Check Context:** What's the price, sophistication, type, goal?
3. **Consult Decision Trees:** Navigate to relevant tree
4. **Apply Resolution Rule:** Context determines choice
5. **Test if Uncertain:** A/B test both approaches

**File Path:** [Decision Trees](references/meta/decision-trees.md)

### Learning Best Practices
1. **Study Synthesis Map:** Understand universal truths
2. **Review Benchmarks:** Know performance standards
3. **Analyze Great Webinars:** Submit to critic in Learning Mode
4. **Apply Frameworks:** Use webinar-expert to implement learnings

**File Path:** [Synthesis Map](references/meta/synthesis-map.md)
