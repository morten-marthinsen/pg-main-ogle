# The 7 Competition Phases

What happens from brief to winner.

---

## The Full Flow

```mermaid
flowchart TD
P1[Phase 1: Setup] --> P2[Phase 2: Drafting]
P2 --> P3[Phase 3: Critique]
P3 --> P4[Phase 4: Revision]
P4 --> P5[Phase 5: Judgment]
P5 --> P6[Phase 6: Learning]
P6 --> P7[Phase 7: Evolution]
```

---

## Phase 1: Setup

```mermaid
flowchart TD
BRIEF[Your Brief] --> PARSE[Arena Parses It]
PARSE --> FOLDER[Creates Project Folder]
FOLDER --> INIT[Initializes Ledger]
INIT --> READY[Ready to Compete]
```

The Arena understands your project and prepares the competition.

---

## Phase 2: Parallel Drafting

All 7 competitors work at the same time.

```mermaid
flowchart TD
START[Competition Begins] --> F[Fladlien Drafts]
F --> C[Cage Drafts]
C --> B[Brunson Drafts]
B --> K[Kern Drafts]
K --> J[Joon Drafts]
J --> KE[Kennedy Drafts]
KE --> S[Synthesis Drafts]
S --> DONE[7 Drafts Ready]
```

Each uses their own methodology on your brief.

---

## Phase 3: Parallel Critique

Each draft is evaluated by its methodology critic.

```mermaid
flowchart TD
DRAFTS[7 Drafts] --> FC[Fladlien Critic]
FC --> CC[Cage Critic]
CC --> BC[Brunson Critic]
BC --> KC[Kern Critic]
KC --> JC[Joon Critic]
JC --> KEC[Kennedy Critic]
KEC --> SC[Synthesis Critic]
SC --> FEED[7 Critiques Ready]
```

Critics check against their expert's specific frameworks.

---

## Phase 4: Parallel Revision

Each expert improves based on critique.

```mermaid
flowchart TD
CRITIQUE[Critic Feedback] --> FIX[Experts See Issues]
FIX --> IMPROVE[Each Revises]
IMPROVE --> BETTER[7 Improved Versions]
```

---

## Phase 5: Judgment

```mermaid
flowchart TD
SEVEN[7 Revised Versions] --> JUDGE[Marketplace Judge]
JUDGE --> SCORE[Scores Each]
SCORE --> RANK[Ranks All 7]
RANK --> WINNER[Declares Winner]
```

The Judge predicts which would win in the real marketplace.

---

## Phase 6: Learning

```mermaid
flowchart TD
WINNER[Winner Known] --> ANALYZE[Analyze Why]
ANALYZE --> LOSER[Losers Learn]
LOSER --> RECORD[Win Records Updated]
RECORD --> BRIEFS[Learning Briefs Saved]
```

Every competitor learns from the outcome.

---

## Phase 7: Evolution

```mermaid
flowchart TD
LEARN[Learning Complete] --> EVOLVE[Skill Evolver Checks]
EVOLVE --> PATTERN{Patterns Found?}
PATTERN -->|Yes| UPDATE[Update Skills]
PATTERN -->|No| DONE[Complete]
UPDATE --> DONE
```

```mermaid
flowchart TD
SYNTHWIN{Synthesis Won All?} -->|Yes| SPAWN[Create New Expert]
SYNTHWIN -->|No| END[End]
SPAWN --> END
```

The system improves for next time.

---

## Multiple Rounds

```mermaid
flowchart TD
R1[Round 1 Complete] --> MORE{More Rounds?}
MORE -->|Yes| BACK[Return to Phase 2]
BACK --> BETTER[With New Learning]
MORE -->|No| FINAL[Final Results]
```

Each round builds on previous learning.

---

## Phase Summary

| Phase | What Happens | Output |
|-------|--------------|--------|
| 1. Setup | Parse brief, create project | Ready to compete |
| 2. Drafting | All 7 create versions | 7 draft webinars |
| 3. Critique | Each critic evaluates | 7 critiques |
| 4. Revision | All improve | 7 revised versions |
| 5. Judgment | Judge picks winner | Ranked results |
| 6. Learning | Analyze what worked | Learning briefs |
| 7. Evolution | Update skills | System improves |
