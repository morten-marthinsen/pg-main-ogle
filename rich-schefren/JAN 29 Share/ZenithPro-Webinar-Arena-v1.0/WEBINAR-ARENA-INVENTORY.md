## How It All Works Together

### The 7-Phase Competition Flow

```
PHASE 1: SETUP
    │   Create project folder, parse brief, initialize ledger
    ▼
PHASE 2: PARALLEL DRAFTING
    │   All 6 experts + synthesist create drafts simultaneously
    ▼
PHASE 3: PARALLEL CRITIQUE
    │   Each critic evaluates its methodology's draft
    ▼
PHASE 4: PARALLEL REVISION
    │   All experts revise based on critique
    ▼
PHASE 5: JUDGMENT
    │   Marketplace judge evaluates all 7, declares winner
    │   User can override if they disagree
    ▼
PHASE 6: LEARNING INTEGRATION
    │   Win records updated, learning briefs saved
    │   More rounds? → Return to Phase 2
    ▼
PHASE 7: POST-ARENA PROCESSING
    │   Skill evolver proposes updates
    │   If synthesis won all rounds → Expert spawner creates new methodology
```

### What Happens When Synthesis Wins

If a synthesis combination wins ALL rounds of a competition:

1. **Synthesis-critic** assesses codification potential (needs B or higher)
2. **Expert-spawner** extracts the winning formula
3. User provides a name for the new methodology
4. Spawner creates:
   - New skill folder at `~/.claude/skills/webinar-[name]/`
   - New critic agent at `~/.claude/agents/webinar-[name]-critic/`
   - Win record initialization
   - Ledger entry with lineage
   - Synthesis registry entry

The new methodology becomes a first-class expert in future competitions.
