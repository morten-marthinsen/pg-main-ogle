# Webinar Expert Meta-Skill

## ⛔ NON-NEGOTIABLE RULES

**These rules are ABSOLUTE. No exceptions. No shortcuts.**

| Rule | Consequence of Violation |
|------|--------------------------|
| ALWAYS identify which expert a recommendation comes from | User cannot trace/verify advice |
| NEVER blend experts without explicitly stating the blend | Loses methodology integrity |
| When experts conflict, MUST present both views + context-based recommendation | User makes uninformed choice |
| Individual expert skills MUST be invoked via Skill tool, not summarized | Loses 50-265 frameworks depth |
| Critic evaluations MUST use Task tool with agent type | Critique lacks enforcement |

## Universal Truths (All 6 Experts Agree)

These principles appear across ALL experts and represent fundamental laws:

1. **Structure Before Content** - All experts build architecture first
2. **Commitment Precedes Conversion** - Build commitment before asking for sale
3. **Objections Must Be Handled Before Close** - Never leave objections for Q&A
4. **Authority Must Be Established** - Even warm audiences need re-establishment
5. **The Close Is The Whole Presentation** - Not just the pitch section
6. **Tell Them What NOT To Do** - Contrast creates clarity
7. **Stories Beat Logic** - Emotion drives purchase, logic justifies
8. **Repetition Is Required** - Key points need 7-21 touches
9. **Scarcity Must Be Believed** - Fake scarcity destroys trust
10. **Risk Reversal Is Math** - Calculate and offer appropriately

## Operational Modes

### Mode 1: Creation Mode

**Purpose:** Build webinars from scratch using synthesized best practices.

**Execution:**
1. Capture all context variables
2. Select primary expert based on decision tree
3. **Invoke that expert's Skill** (e.g., `/webinar-brunson`)
4. Get draft with mandatory compliance summary
5. **Invoke webinar-critic agent via Task tool**
6. Apply fixes
7. Present final output

**GATE CHECK:** Expert Skill invoked? Critic agent run? Compliance summary present?

### Mode 2: Analysis Mode

**Purpose:** Evaluate existing webinars against all expert standards.

**Execution:**
1. Receive webinar content
2. **Invoke webinar-critic agent via Task tool**
3. Present multi-expert scorecard
4. Identify which expert principles present/missing
5. Provide priority recommendations

**GATE CHECK:** Critic agent invoked? Scorecard complete?

### Mode 3: Optimization Mode

**Purpose:** Diagnose and fix specific webinar issues.

**Execution:**
1. Identify the problem
2. Use decision tree to select expert
3. **Invoke that expert's Skill** for recommendation
4. Apply fix
5. **Invoke critic agent** to verify

**GATE CHECK:** Problem identified? Expert Skill invoked? Fix verified?

### Mode 4: Expert-Specific Mode

**Purpose:** Get one expert's pure perspective.

**Execution:**
1. User specifies expert
2. **Invoke ONLY that expert's Skill**
3. Deliver pure methodology output
4. Do NOT blend with other experts

**GATE CHECK:** Single expert invoked? No blending?
