## The Complete VSL Workflow

### Step 1: Define the One Belief

Before writing a word, identify THE belief that must be established:

```
/evaldo

BRIEF:
Product: [What you're selling]
Audience: [Who they are]
What they currently believe: [The wrong belief]
What they need to believe: [The One Belief that makes purchase inevitable]

Help me articulate the One Belief for this VSL.
```

**Example One Belief:**
> "The reason nothing has worked isn't you—it's that every solution you've tried was based on the same flawed assumption. Once you understand the REAL mechanism, everything changes."

### Step 2: Build the 10 Questions Structure

Evaldo's VSL architecture follows 10 key questions the viewer subconsciously asks:

```
/evaldo-leads

Using the One Belief: [Your One Belief]

Create the VSL structure answering these 10 questions:
1. Why should I listen to you?
2. What's in it for me?
3. How do I know this is real?
4. Why is this different?
5. How does it work?
6. Will this work for me?
7. What do I get?
8. Is it worth the price?
9. What if it doesn't work?
10. Why should I act now?

Audience: [Who they are]
Product: [What you're selling]
Price: $[Amount]
```

### Step 3: Write the Opening (First 5 Minutes)

The opening must hook and create massive curiosity:

```
/evaldo-leads

Write the first 5 minutes of this VSL:

One Belief: [Your One Belief]
Hook angle: [The surprising/curious element]
Audience pain: [What they're struggling with]

The opening must:
- Stop them from clicking away
- Create an open loop they MUST close
- Establish credibility without bragging
- Promise something they haven't heard before
```

### Step 4: Build the Body (Belief Building)

The body systematically builds belief:

```
/evaldo-body

Continue the VSL body:

One Belief to establish: [Your One Belief]
Mechanism to reveal: [How the product works]
Proof available: [Testimonials, stats, credentials]

Build belief gradually:
- Story that demonstrates the mechanism
- Proof that validates the story
- Explanation that creates understanding
- Additional proof layers
```

### Step 5: Craft the Close

The close handles objections and drives action:

```
/evaldo-close

Write the VSL close:

Offer: [What they get]
Price: $[Amount]
Guarantee: [Your guarantee]
Bonuses: [List bonuses]
Urgency: [Why act now]

Handle these objections:
- [Objection 1]
- [Objection 2]
- [Objection 3]

Use price anchoring and stacking.
```

## Adding Power from Other Masters

Evaldo owns VSL structure, but you can enhance with:

### Carlton Hooks (Pattern Interrupts)

VSLs need pattern interrupts to maintain attention:

```
/carlton-hooks

This VSL needs attention resets at these points:
- After the opening hook (3 min mark)
- Before the mechanism reveal (8 min mark)
- Before the close (15 min mark)

Create pattern interrupts that re-engage viewers
who might be drifting.
```

### Clayton Emotional Beats

Add emotional intensity at key moments:

```
/clayton-psychology

At these moments in the VSL, I need to amplify emotion:
- The "rock bottom" moment in the story
- The "discovery" moment
- The "what if you don't act" moment

What emotional triggers should I use at each point?
```

### Deutsch Story Integration

For more sophisticated VSLs:

```
/deutsch-leads

Make this VSL opening more story-driven
while maintaining Evaldo's structure:

[Your current opening]

Add dual journey (logic + emotion) elements.
```

## The Hybrid VSL Formula

Best-in-class VSLs often combine:

```
Evaldo Structure (foundation)
├── Carlton Hook (opening pattern interrupt)
├── Evaldo Lead (curiosity/One Belief setup)
├── Deutsch Story (credibility narrative)
├── Evaldo Mechanism (belief building)
├── Clayton Proof (proof cascade)
├── Carlton Interrupt (re-engagement)
├── Evaldo Close (price anchoring)
└── Clayton Urgency (final push)
```

## VSL Brief Template

Use this when requesting a full VSL:

```
VSL BRIEF

PRODUCT:
Name: [Product name]
What it does: [Core function]
Mechanism: [How it works]

PRICE:
$[Amount]
Payment options: [If any]

AUDIENCE:
Who they are: [Demographics]
What they've tried: [Past failures]
Why it failed: [The flawed assumption]
What they believe now: [Current wrong belief]

ONE BELIEF:
[The belief that makes purchase inevitable]

PROOF AVAILABLE:
- Testimonials: [Y/N, count]
- Case studies: [Y/N, topics]
- Statistics: [List]
- Credentials: [Your authority]

OFFER STACK:
- Main product: [What they get]
- Bonus 1: [Description]
- Bonus 2: [Description]
- Bonus 3: [Description]

GUARANTEE:
[Your risk reversal]

URGENCY:
[Why act now - deadline, limited spots, etc.]

LENGTH TARGET:
[Minutes - typically 15-45 for VSLs]
```
