## **Why this matters**

Before this extension, getting Claude Code to interact with your browser required workarounds. You would screenshot things, drop them into Claude Code, and work with it that way. Or you would use separate MCPs and extra tools to bridge the gap.

Now you get direct browser control. Open Claude Code next to your browser, and it can see what you are seeing. No copy-pasting. No screenshotting. No juggling multiple tools.

**Your co-writer now has eyes.**

That means tasks that used to require you to be the middleman (pulling data, navigating pages, filling out forms) can now run in the background while you work on something else. It is not blazing fast, but it is reliable enough for straightforward tasks that are not worth your time anyway.

## **Here are some key concepts to know**

### **Tab groups and permissions**

When you open the Chrome extension, you will see tabs highlighted in orange. This is a tab group. Everything in that group is accessible to Claude. Everything outside it is not.

This gives you granular control. Drag a tab into the group to give Claude access. Leave it out to keep Claude away. Every new site will ask for permission before Claude can access it, and you can revoke access anytime.

[![](https://substackcdn.com/image/fetch/$s_!MHU4!,w_1456,c_limit,f_auto,q_auto:good,fl_progressive:steep/https%3A%2F%2Fsubstack-post-media.s3.amazonaws.com%2Fpublic%2Fimages%2F4ce6badc-02ef-4e68-aa80-b90174b2e5e1_796x278.png)](https://substackcdn.com/image/fetch/$s_!MHU4!,f_auto,q_auto:good,fl_progressive:steep/https%3A%2F%2Fsubstack-post-media.s3.amazonaws.com%2Fpublic%2Fimages%2F4ce6badc-02ef-4e68-aa80-b90174b2e5e1_796x278.png)

You also have notification settings (so Claude tells you when tasks finish) and microphone access for the workflow recording feature in the regular extension.

### **Background task mentality**

This is not a speed tool. Claude navigates by taking screenshots and processing what it sees. If you start scrolling or clicking while it is working, it loses its place.

The right way to use this: delegate a task, open another terminal, launch another Claude session, and keep working. Let the browser task run in the background.

Think of it as handing off something tedious (like data entry or form filling) to an assistant who is methodical but not fast. You would not hover over their shoulder. You would let them work and check the result when they are done.

### **API-less platforms**

One of the most practical uses: platforms that do not have APIs. Substack is the perfect example. You cannot pull your analytics programmatically. There is no endpoint.

But now Claude can navigate to your stats page, read the data, and enter it into Airtable or wherever you track things. Same idea applies to any platform where automation is normally impossible because there is no API to hook into.

### **Some quick tips:**

  * Always tell Claude to use the Chrome extension explicitly (it does not always know to use it automatically)

  * Drop the direct link to whatever page you want Claude to access

  * Do not touch the browser window while Claude is working on it

  * Use this for background tasks, not speed tasks
