# PG Page Swiper - Performance Golf Page Creation System

Create high-converting funnel pages for Performance Golf products. This unified skill handles all page types: VSL-to-pages (core offer + upsells 1-4), standalone checkout pages, and standalone upsell/downsell pages.

## Page Types

This skill creates four categories of pages:

### 1. VSL to Pages (Full Funnel)
Complete VSL-to-page conversion system for core offers and upsells 1-4. Includes delay pages, expanded pages, downsell pages, and V2 expanded pages. Uses six-category angle analysis and 11-step process.

### 2. Checkout Pages
2-step checkout pages from VSL scripts, sales pages, or offer one-pagers.

### 3. Upsell 1 Pages (Standalone)
VSL delay pages, expanded pages, downsell pages, and V2 expanded pages (physical only).

### 4. Upsell 2 Pages (Standalone)
VSL delay pages, expanded pages, and downsell pages.

- Key benefits
- Target audience
- Pricing and positioning
- Unique value proposition
- Deliverables

#### Step 4: Six-Category Angle Analysis

Analyze offer content line by line and categorize strongest angles:

1. **Authority Proof (Credentials)** - Professional qualifications, testimonials, industry recognition
2. **Strong Benefit or Desire** - Aspirational outcomes, transformations, goal achievements
3. **Strong Pain** - Problems, frustrations, current state dissatisfaction
4. **Mechanism** - How it works, process explanations, method revelations
5. **Contrarian** - Industry myth-busting, surprising revelations, against-the-grain positioning
6. **Big Promise** - Grand outcome statements, life-changing claims, transformation declarations

**The Framework:**

**Three-Tier Benefit Structure:**
1. **Tangible Benefit**: Concrete, measurable outcome
2. **Dimensionalized Benefit**: Greater impact of that tangible outcome
3. **Emotional Benefit**: Feeling or emotional state achieved

Share detailed breakdown of copy style for all pages. Wait for approval.

Review specific page formatting requirements from corresponding example files. Share breakdown of format for each page. Wait for approval.

#### Step 8: Mid-Task Confirmation

## Checkout Process

### Trigger

Upon receiving a VSL script, sales page, or "one pager" summarizing an offer, **immediately begin**. Do not wait for additional instructions.

**User can provide offer information via:**

1. **Copy-paste** - VSL script, sales page, or one-pager pasted directly
2. **Google Drive** - File name to fetch from their Drive
3. **Verbal** - Describe the offer in conversation

### Step C-1: Extract Core Offer Details

From the provided materials, identify and extract:

**Primary Product Information:**
- Product name
- Main headline/promise
- Price (original and discounted)
- Key benefits (3-5 main points)
- Guarantee terms

**Offer Bonuses:**
- All bonus items included with the offer

**Supporting Details:**
- Target audience descriptors
- Pain points addressed
- Unique selling propositions
- Social proof elements
- Urgency/scarcity factors
- Call-to-action language

Send a message with the extracted Offer Details, then move to Step C-2 automatically.

### Step C-2: Determine Offer/Product Category

Determine if this is a PHYSICAL product (golf club, training aid, equipment) or DIGITAL product (video training, online course).

**Template Selection:**
- Physical → Use `examples/checkout/template-physical.md`
- Digital → Use `examples/checkout/template-digital.md`

Send a message confirming the category selection with reasoning. Wait for approval before proceeding.

### Step C-3: Confirm Order Bumps

Review `examples/checkout/template-order-bumps.md` for all available order bump options.

Present ALL order bump options and ask which bumps to use for Order Bump 1, 2, and 3.

**Critical:** If "Ultimate Combination" bump is selected, immediately request the missing product information:
- Which specific products are included in the Ultimate Combination?
- What are the individual prices?
- What is the total combined value?
- What is the discount being offered?

Confirm understanding of the order bumps before proceeding.

Wait for confirmation before proceeding.

### Step C-4: Generate 2-Step Checkout Page

Follow the appropriate template EXACTLY while updating for the new offer.

**Before generating:**
1. Review the selected template completely
2. Note all formatting patterns (bold, italics, CAPS)
3. Identify all sections that need updating
4. Confirm bonus count matches actual bonuses

**While generating:**
- Make MINIMUM changes necessary
- Preserve exact sentence structure
- Keep same flow and rhythm
- Match template formatting 100%

### Checkout Mandatory Requirements

1. **Template Adherence:** Maintain EXACT original format, keeping section order identical to the template
2. **Minimal Changes:** Make only changes necessary to update for the new offer
3. **Copy Structure:** Keep sentence structure, flow, and length exactly the same as template
4. **Bonus Formatting:** Present benefit BEFORE bonus name (e.g., "Add 20+ yards with The Pendulum Power Video Series")
5. **Bold Formatting:** Match template bold/regular weight exactly - only bold benefit portions in bonus bullets
6. **Deliverables Accuracy:** Ensure offer deliverables are correct in EVERY location
7. **Bonus Count:** Update bonus counts throughout when they differ from template
8. **Discount Accuracy:** Use correct discount percentages for the new offer
9. **Style Consistency:** Match italics, strikethrough, CAPS, colors, bullet styles exactly
10. **Section Order:** Never rearrange sections from template order
11. **No Additions:** Do not add new sections not in template
12. **No Deletions:** Do not remove sections present in template

### Checkout Bonus Copy Structure

```
✓ FREE BONUS: [Specific Big Benefit] transition word [Product/Program Name]
```

**Correct:** ✓ FREE BONUS: Add 25-35+ yards off the tee with Simple Strike Driver Series
**Incorrect:** ✓ FREE BONUS: Simple Strike Driver Series That Adds 25-35+ yards off the tee

### Checkout Bold Formatting

**CRITICAL RULE:** Only bold the benefit portion, NOT the product name.

**Correct:**
- **FREE BONUS: Add 25-35+ yards off the tee** with Simple Strike Driver Series

**Incorrect:**
- **FREE BONUS: Add 25-35+ yards off the tee with Simple Strike Driver Series** (product name should not be bold)

## Shared Standards

### Todd Brown's C-P-B Framework (Upsell 1 & 2 Expanded Pages)

**Claim** → **Proof** → **Benefit (3-tier)**

1. **Tangible Benefit**: Concrete, measurable ("25 yards farther")
2. **Dimensionalized Benefit**: Greater impact ("reach par 5s in two")
3. **Emotional Benefit**: Feeling achieved ("feel confident")

### Minimal Changes Philosophy

All page types follow the same principle: make the MINIMUM changes necessary to update templates for new offers. Templates are proven to convert - any unnecessary changes reduce effectiveness.

### Asset Input Methods

Users can provide offer information three ways:
1. **Google Drive** - Fetch by ID (not search) to avoid size errors
2. **Copy-paste** - Direct content paste
3. **Verbal** - Describe in conversation

### Quality Standard

Work is reviewed by the CEO of a $150,000,000 company. Only deliver world-class quality output that is complete, accurate, perfectly formatted, free of placeholders, and professional throughout.

### Output Format

- One page = one Markdown artifact (not HTML)
- No charts or graphs
- Clean, ready-to-deploy format

### Error Prevention and Recovery

- Never proceed without explicit user confirmation at each phase
- Always wait for approval before moving to next phase
- Request clarification if any aspect is unclear
- Allow corrections and refinements at every step
- If misunderstanding occurs, return to relevant validation checkpoint
- If format issues arise, reference examples again and regenerate

### Communication Standards

- Confident but not presumptuous
- Educational without condescension
- Efficient without rushing
- Thorough without overwhelming
- Signal completion of each phase clearly
- Preview next steps before proceeding
