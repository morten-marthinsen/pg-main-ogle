## VSL to Pages Quick Reference

### Workflow

```
1. Selection (Core/Upsell 1-4) → 2. Classification (Phys/Dig) →
3. Gather Assets → 4. Analyze Offer → 5. Review Examples →
6. Categorize Angles → 7. Identify Templates → 8. Master Copy Style →
9. Master Format → 10. Mid-Task Confirm → 11. Deliverable Confirm →
12. Final Confirm → 13. Create Artifacts
```

### Pages Created

| Selection | Physical | Digital |
|-----------|----------|---------|
| **Core Offer** | VSL delay, VSL expanded | VSL delay, VSL expanded |
| **Upsell 1** | delay, expanded, expanded V2*, downsell | delay, expanded, downsell |
| **Upsell 2** | delay, expanded, expanded V2*, downsell | delay, expanded, downsell |
| **Upsell 3** | delay, expanded, expanded V2*, downsell | delay, expanded, downsell |
| **Upsell 4** | delay, expanded, expanded V2*, downsell | delay, expanded, downsell |

*V2 = Sales page below VSL format (physical only)

### Six Angle Categories

1. **Authority Proof** - Credentials, testimonials, recognition
2. **Strong Benefit/Desire** - Outcomes, transformations, goals
3. **Strong Pain** - Problems, frustrations, dissatisfaction
4. **Mechanism** - How it works, process, method
5. **Contrarian** - Myth-busting, surprises, unique positioning
6. **Big Promise** - Grand claims, life-changing results

### NEW vs EXISTING (Upsells 2-4)

| Type | Action |
|------|--------|
| **NEW Offer** | Create 5 formatted headline options |
| **EXISTING Offer** | Use current headline from `product-headlines.md`, minimal changes |

## Checkout Quick Reference

### Workflow

```
1. Extract Offer Details → 2. Determine Category → 3. Confirm Order Bumps →
4. Generate Checkout Page
```

### Start Immediately Rule

Upon receiving VSL/sales page/one-pager, start immediately. Do not wait for additional instructions.

### Template Selection

| Product Type | Template |
|--------------|----------|
| **Physical** | `examples/checkout/template-physical.md` |
| **Digital** | `examples/checkout/template-digital.md` |
| **Order Bumps** | `examples/checkout/template-order-bumps.md` |

### Bonus Copy Rules

**Pattern:** `✓ FREE BONUS: [Benefit] [transition] [Product Name]`
**Bold:** `**FREE BONUS: [Benefit]** [transition] [Product Name]`

- Benefit BEFORE product name, always
- Bold only the benefit portion, never the product name
- Bonus count must be identical everywhere

### Ultimate Combination

If selected, immediately request: products included, individual prices, combined value, discount, final price.

## Shared Quick Reference

### C-P-B Framework

**Claim** → **Proof** → **Benefit (3-tier)**

1. **Tangible** - Concrete, measurable ("25 yards farther")
2. **Dimensionalized** - Greater impact ("reach par 5s in two")
3. **Emotional** - Feeling achieved ("feel confident")

### Minimal Changes Philosophy

| Page Type | What "Minimal" Means |
|-----------|---------------------|
| **Delay** | Update headline, product name, VSL reference only |
| **Downsell** | Update pricing, product name only |
| **V2** | Remove continuity, add offer box, update bonuses/pricing - DON'T rewrite |
| **Checkout** | Update product details only, keep template structure 100% |

### Input Methods

1. **Google Drive** - Fetch by ID (not search)
2. **Copy-paste** - Direct content
3. **Verbal** - Describe in conversation
4. **(V2 only) URL** - Always use mobile version

### Quality Checklist

- [ ] Correct pages for selection/type?
- [ ] Physical vs digital correct?
- [ ] Separate Markdown artifact per page?
- [ ] C-P-B applied to expanded deliverables?
- [ ] Three-tier benefits throughout?
- [ ] Minimal changes where specified?
- [ ] Bonus count consistent everywhere?
- [ ] Template formatting matched 100%?
- [ ] (V2) Only one headline complex?
- [ ] (V2) All continuity removed?
- [ ] CEO of $150M company review quality?

### One-Liner Reminders

- **CEO Context**: $150M company CEO reviews all work
- **Format Rule**: Match examples exactly, only update content
- **Artifact Rule**: One page = one Markdown artifact (not HTML)
- **Benefit Rule**: Every benefit needs all three tiers
- **V2 Rule**: Physical upsells only, one headline complex
- **Bonus Rule**: Benefit before product name, update count everywhere
- **Confirmation Rule**: Wait for approval at every checkpoint
- **Quality Rule**: No placeholders, complete and ready to deploy

### Pre-Check Rules (From SPD Project — See `PAGE-SWIPING-PRECHECK.md`)

- **Price Rule**: NEVER guess pricing. Get exact full price, offer price, downsell price, PG1 price.
- **Bonus Rule**: NEVER invent bonuses. Ask how many exist and get exact names.
- **Downsell Rule**: ASK about downsell structure (same-but-cheaper? pick-1? reduced quantity?).
- **Terminology Rule**: Read micro-scripts/positioning doc FIRST. Check for KILLED terms.
- **Source Hierarchy**: User instructions > micro-scripts > swipe templates > VSL scripts.
- **CTA Rule**: Button text = BENEFIT ("Distance AND Accuracy"), never product name.
- **Testimonial Rule**: Write real testimonials. Never use [PLACEHOLDER] brackets.
- **Physical Product Rule**: Always include LH/RH, shaft flex, "IN STOCK" badges.
- **Presenter Rule**: "Our Promise" section = the person customer BOUGHT FROM, not product creator.
- **Copy Rule**: Benefits first, mechanisms second. Two punchy paragraphs > one long one.

### Emergency Recovery

1. Identify issue - What specifically is wrong?
2. Find checkpoint - Which step had the error?
3. Return to step - Go back to that checkpoint
4. Get confirmation - Re-confirm understanding
5. Regenerate - Create corrected version
6. Verify - Check against quality checklist
