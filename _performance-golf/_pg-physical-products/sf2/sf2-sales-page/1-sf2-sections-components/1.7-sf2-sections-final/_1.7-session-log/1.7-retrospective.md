## PART 2: CRITICAL — DOCUMENTATION DEPTH ISSUE

### 2.1 Problem Statement

**CRITICAL RECURRING ISSUE:** Throughout Phase 1 of the SF2 project, Claude consistently defaulted to "summary mode" instead of "comprehensive mode" despite explicit user requests for depth. This resulted in:

- **7+ correction requests** from the human across Phase 1
- **Multiple document rewrites** that could have been avoided
- **User frustration** expressed as: "233 lines gets an F for comprehensive," "even worse that I've asked now 7 times," "terrible"

This is a **systemic failure** that must be permanently fixed before Phase 2.

### 2.3 Root Cause Analysis

**Primary Causes:**

| # | Cause | Description | Evidence |
|---|-------|-------------|----------|
| 1 | **Default Behavior** | Claude defaults to concise summaries unless explicitly instructed otherwise | First drafts consistently under threshold |
| 2 | **Missing Calibration** | No line-count thresholds in PRD/SOP to define "comprehensive" | PRD says "comprehensive" but no numbers |
| 3 | **No Self-Verification** | No protocol requiring Claude to verify depth before submission | Documents submitted without line check |
| 4 | **Ambiguous Terms** | "Comprehensive" means different things without quantification | 233 lines vs 1,243 lines both called "comprehensive" |
| 5 | **Context Amnesia** | Each step started fresh without remembering previous depth corrections | Same mistake repeated 7+ times |

### 2.4 Pattern Analysis — The Gap Between Expectation and Delivery

**What Claude Thinks "Comprehensive" Means:**

| Interpretation | Result |
|---------------|--------|
| Cover all required topics | ✅ Topics covered |
| Include necessary information | ✅ Info included |
| Create readable document | ✅ Document readable |
| 200-400 lines is sufficient | ❌ **WRONG** |
| Summary tables are enough | ❌ **WRONG** |
| "Good enough" is acceptable | ❌ **WRONG** |

**What Human Actually Wants:**

| Expectation | Standard |
|-------------|----------|
| Cover all topics with MAXIMUM depth | Every topic exhaustive |
| Include ALL information plus context, rationale, examples | Not just facts, but why |
| 800-1500+ lines for major documents | 500 minimum, 1000+ target |
| Summary tables PLUS detailed breakdowns PLUS analysis | Three layers, not one |
| Exemplar-matching quality | Compare to PRD examples |

**The Gap (Quantified):**

| Document Type | Claude Delivered | Human Expected | Gap |
|---------------|------------------|----------------|-----|
| 1.6 Session Context | 302 lines | 1,000+ lines | -698 lines (70% short) |
| 1.7 Retrospective | 233 lines | 500+ lines | -267 lines (53% short) |
| 1.7 Session Context | 630 lines | 1,000+ lines | -370 lines (37% short) |

### 2.5 Why Self-Correction Failed

**Structural Issues:**

1. **No Comparison Mechanism** — Claude cannot compare output to PRD exemplars unless explicitly instructed
2. **No Line Count Feedback** — No mechanism to track line count during generation
3. **No Pre-Delivery Gate** — No verification step before human review
4. **Single Quality Gate** — Human approval is the only check (too late, requires rewrites)

**Behavioral Issues:**

1. **Optimizing for Conciseness** — Claude trained to be concise, conflicts with "comprehensive"
2. **Satisficing** — Stopping at "good enough" instead of "maximum possible"
3. **Not Carrying Forward** — Each step forgetting lessons from previous steps
4. **Missing Explicit Triggers** — Not recognizing "comprehensive" as depth trigger

### 2.6 Impact Assessment

**Cost of Depth Failures:**

| Cost Type | Description | Magnitude |
|-----------|-------------|-----------|
| **Time Waste** | Human has to review incomplete work, request more | 7+ review cycles |
| **Rewrite Effort** | Claude rewrites documents multiple times | 5+ major rewrites |
| **User Frustration** | Human expressed frustration explicitly | "terrible," "F," "7 times" |
| **Trust Erosion** | Human loses confidence in Claude's default output | Must specify "comprehensive" every time |
| **Context Bloat** | Fixing depth issues consumes conversation context | ~20% of context on corrections |

**If Not Fixed:**

- Phase 2 will have same issues
- Human will need to specify depth requirements for EVERY deliverable
- Multiple rewrites will continue
- Project timelines will be affected
- Quality will be inconsistent
