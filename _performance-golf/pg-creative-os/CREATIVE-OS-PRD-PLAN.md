## 1. Intent

The Creative OS PRD is the **organization-level vision document** for Performance Golf's creative team. It is not an agent PRD; it sits above the AI system and defines what success looks like for the creative organization over 30/60/90 days.

**Must be explicitly connected to:**

- **Performance Golf brand guidelines and strategy** — the PRD should reference and align with PG's brand standards and strategic direction.
  - **Path:** `/Users/johnhardesty/Documents/The Sauce Vault/_performance-golf/pg-skills/pg-brand-guidelines`

- **Two 2026 brand themes/threads** (used as the lens for all creative output and prioritization):

  1. **A smarter way to improve golf and love your game** — education, improvement, loving the game, connected system (quiz → PG1 → Swing Scan AI), helping golfers play better faster.

  2. **Stories of Innovation** (not just "product innovation") — showing the innovation story behind products: Chris McGinley's expertise, Joe King's experience, robot testing, CAD iterations, the science and process. Goal: Build credibility, trust, and enable premium pricing ($249 mid-tier vs discount brand). Visualize innovation through 3D animations, clubmaker content, influencer reviews, PR. Context: Whether for everyday golfers or pros (RS1), the story is that PG has real innovation and expert designers, not white-labeled Chinese products.

**Purpose:** Define success for the creative team (people, process, and how they use the Creative OS agents), align execution with research/data/strategy/campaign themes, and serve as the single reference for how creative work is governed.

**Critical function:** The PRD should **capture strategic conversations** (like the innovation discussion) so they don't need to be repeated with every person. It should help scale the vision — ensure everyone understands the 360 view without constant meetings. The PRD should challenge and push to remember these things, recommend ideas that connect dots, so when creative output happens, it's already 90-95% aligned rather than requiring rebuilds.

## 8. Recommendations (AI)

### 8.1 Sub-Agent PRD Integration (Token-Efficient)

**Recommendation:** Use an **optional load** at session start instead of always loading the full PRD.

- At the **top of each sub-agent’s CLAUDE.md** (or main instruction file), add a **single line** that points to the Creative OS PRD and instructs: *“At the start of a session, ask the user: ‘Do you want to load the Creative OS PRD for alignment check?’ If yes, read [PRD path] and run a short alignment check (current focus vs PRD priorities). If no, proceed without loading.”*
- This keeps PRD as **user choice** and avoids using tokens every session. When the user is doing PRD-critical work (e.g. campaign planning, GTM prep), they say yes; when doing small execution tasks, they say no.
- **Alternative:** Trigger PRD load only when the user’s message contains certain keywords (e.g. “campaign,” “launch,” “strategy,” “GTM”) or when the session is in “plan mode.” That way it’s automatic for high-stakes work but still not every session.

### 8.2 Challenger Automation (Token-Efficient)

**Recommendation:** Make the challenger **event-triggered**, not always-on.

- **Option A:** Challenger runs only when the user says “challenge this” or “align check,” or at the **end** of a session (e.g. “Before we wrap, should I run a quick PRD alignment check on what we did?”). That way it doesn’t run on every turn.
- **Option B:** Challenger runs only when the PRD is loaded (see 8.1). So: “Load PRD?” → Yes → load PRD + run one short challenger pass (e.g. “Does this session’s focus align with PRD priorities? Any drift?”). One pass per session when PRD is in context.
- **Option C:** Add a **short “PRD reminder” block** (3–5 bullets) in each sub-agent’s CLAUDE.md: the two brand threads, “one root angle per asset,” “script–visual congruence.” No full PRD read; just a minimal reminder so the agent can nudge alignment in conversation without loading the full doc. Challenger sub-agent still exists for deeper checks when the user requests or when PRD is loaded.
