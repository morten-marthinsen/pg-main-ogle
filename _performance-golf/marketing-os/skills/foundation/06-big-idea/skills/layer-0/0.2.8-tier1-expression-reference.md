## Loading Instructions

### When to Load

This microskill executes during Layer 0, AFTER 0.2.7 (persona voice loader) and BEFORE Layer 1 begins. It loads TIER1 extraction data that will be consumed by microskill 3.9 (Expression Anchoring Score) in Layer 3.

### What to Load

From each TIER1 extraction JSON file in the current project's vertical:

1. **Extract headline and lead fields:**
   - `components.headline` — the primary headline
   - `components.lead` — the lead section (opening copy)
   - `configuration.big_idea_type` — discovery, enemy, prediction, secret, mechanism, contrarian

2. **Extract RSF analysis fields (if present):**
   - `rsf_analysis.fssit_candidates` — "finally someone said it" statements
   - `rsf_analysis.schema_violations` — what audience expectations are violated
   - `rsf_analysis.narrative_reorganization` — how the Big Idea explains the audience's past
   - `schema_analysis` — schema distance metrics

3. **Extract creative wrapping fields:**
   - `configuration.element_references` — what elements anchor the Big Idea
   - `rsf_framework` — R/S/F architecture
   - `metadata.era`, `metadata.source_publisher` — context for when/who

4. **Filter by vertical:**
   - Load ONLY files from `final_merged/[vertical]/` matching the current project vertical
   - If project vertical has fewer than 10 files, supplement with files from `health/` (largest vertical, most Big Idea diversity)

## Selection Priority for This Skill

Big Idea expression patterns are most valuable from controls that:

1. **Have high schema distance (4-8)** — controls whose Big Ideas genuinely surprise the audience while remaining relevant
2. **Contain strong FSSIT moments** — controls where the audience would say "finally someone said it"
3. **Show effective narrative reorganization** — controls that explain the audience's entire past through a single reframe
4. **Match the project vertical** — Big Idea wrapping conventions differ by vertical
5. **Have strong headlines** — controls where the headline alone communicates the Big Idea power

### Per-Vertical Loading Notes

| Vertical | Priority Patterns | Why |
|----------|------------------|-----|
| **Health** | Discovery + Enemy types, conspiracy-style schema violations | Health audiences respond to "hidden truth" and "hidden enemy" Big Ideas |
| **Golf** | Mechanism + Contrarian types, false-belief schema violations | Golf audiences respond to "everything you've been told is wrong" |
| **Finance** | Prediction + Enemy types, urgency-based schema violations | Finance audiences respond to "what's coming that nobody sees" |
| **Personal-Dev** | Secret + Discovery types, identity-based schema violations | Personal-dev audiences respond to "the thing about yourself you never realized" |
| **Technology** | Discovery + Mechanism types, innovation-based schema violations | Tech audiences respond to "the breakthrough you didn't know was possible" |

## Output Requirements

**Output file:** `[project]/06-big-idea/layer-0-outputs/0.2.8-tier1-expression-reference.md`

The output file must contain:

1. **Execution Context** — skill, vertical, timestamp, files loaded
2. **Pattern Summary** — top Big Idea types, headline patterns, lead architectures, FSSIT patterns, schema violation types with frequencies
3. **Verbatim Examples** — at least 5 complete headlines with their Big Idea type and schema violation
4. **Pattern Index** — the full `tier1_expression_reference` YAML block for downstream consumption by microskill 3.9

**Minimum size:** 2KB
