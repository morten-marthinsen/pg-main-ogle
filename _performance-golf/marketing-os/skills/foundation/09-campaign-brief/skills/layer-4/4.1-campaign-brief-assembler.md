# Layer 4.1 — Campaign Brief Assembler

**Skill:** 09 — Campaign Brief
**Microskill:** 4.1 — Campaign Brief Assembler
**Layer:** 4
**Model:** sonnet
**Purpose:** Compile the full Campaign Brief from all Layer 1-3 outputs into the authoritative campaign-brief.json — the single reference document every downstream writing skill consults.

---

## Input

| Source | File | Required |
|--------|------|----------|
| 1.1-campaign-summary-generator | `1.1-campaign-summary.json` | Yes |
| 1.2-element-relationship-mapper | `1.2-element-relationships.json` | Yes |
| 1.3-campaign-thesis-validator | `1.3-thesis-validation.json` | Yes |
| 2.1-coherence-dimension-scorer | `2.1-coherence-scores.json` | Yes |
| 2.2-weakness-detector | `2.2-weakness-report.json` | Yes |
| 2.3-coherence-narrative-generator | `2.3-coherence-narrative.md` | Yes |
| 3.1-lead-type-recommender | `3.1-lead-type-recommendation.json` | Yes |
| 3.2-story-type-recommender | `3.2-story-type-recommendation.json` | Yes |
| 3.3-emotional-arc-designer | `3.3-emotional-arc.json` | Yes |
| 0.1-upstream-loader | `0.1-upstream-context.json` | Yes |

## Output

| File | Format | Min Size |
|------|--------|----------|
| `campaign-brief.json` | JSON | 10KB |
| `campaign-brief-summary.md` | Markdown | 5KB |

## Execution Steps

### Step 1: Assemble Metadata Section

```yaml
metadata:
  generation_date: string
  upstream_skills_consumed: ["01-research", "02-proof-inventory", "03-root-cause", "04-mechanism", "05-promise", "06-big-idea", "07-offer", "08-structure"]
  coherence_score: number  # from 2.1
  human_approved: false    # set to true only after human approves
  approval_date: null      # set when approved
  version: 1               # increments on revision
```

### Step 2: Assemble Campaign Summary Section

Pull from 1.1-campaign-summary.json:
```yaml
campaign_summary:
  one_sentence_thesis: string
  full_paragraph: string
  element_summaries:
    market_context: string
    avatar: string
    root_cause: string
    villain: string
    mechanism: string
    promise: string
    big_idea: string
    offer: string
    structure: string
```

### Step 3: Assemble Coherence Audit Section

Pull from 2.1, 2.2, and 2.3:
```yaml
coherence_audit:
  overall_score: number
  dimension_scores: [object]  # all 10 dimensions with scores and assessments
  critical_weaknesses: [string]
  strengths: [string]
  coherence_narrative: string  # from 2.3-coherence-narrative.md
```

### Step 4: Assemble Creative Direction Section

Pull from 3.1, 3.2, 3.3:
```yaml
creative_direction:
  lead_type:
    recommended: string
    rationale: string
    alternatives: [object]
    human_selected: null    # populated after human review
    human_rationale: null
  story_type:
    recommended: string
    rationale: string
    alternatives: [object]
    human_selected: null
    human_rationale: null
  emotional_arc: object     # full arc from 3.3
  tone_direction: object    # full tone from 3.3
```

### Step 5: Generate Section Briefs

Create one brief per downstream writing skill. Each brief must be SPECIFIC to this campaign:

```yaml
section_briefs:
  - skill_number: 11
    skill_name: "Lead"
    section_purpose: string   # What this section must accomplish for THIS campaign
    key_assets: [string]      # Which specific upstream assets to draw from
    emotional_target: string  # What specific emotion this section produces
    coherence_requirements: [string]  # What must align in this section
    constraints: [string]     # What to avoid
    connects_from: string     # What section feeds into this one
    connects_to: string       # What section this feeds into
  - skill_number: 12
    skill_name: "Story"
    # ... same structure
  - skill_number: 13
    skill_name: "Root Cause Narrative"
    # ... same structure
  - skill_number: 14
    skill_name: "Mechanism Narrative"
    # ... same structure
  - skill_number: 15
    skill_name: "Product Introduction"
    # ... same structure
  - skill_number: 16
    skill_name: "Offer Copy"
    # ... same structure
  - skill_number: 17
    skill_name: "Close"
    # ... same structure
```

**Critical rule:** Section briefs must be SPECIFIC to this campaign. "Write compelling copy about the root cause" = REJECTED. "Externalize blame from the golfer to [specific villain]. Use the [specific root cause] framing. Target emotion: vindication. Key proof asset: [specific]." = CORRECT.

### Step 6: Initialize Human Decisions Section

```yaml
human_decisions:
  choices_made: []          # populated during human review
  overrides: []             # populated if human overrides recommendations
  approval_status: "pending"
  revision_notes: null
  execution_notes: null
```

### Step 7: Generate campaign-brief-summary.md

Create a human-readable markdown version of the brief for quick reference during writing sessions. Include:
- Campaign summary
- Creative direction (lead type, story type, tone)
- Emotional arc
- Section briefs (abbreviated)
- Threading guide placeholder (populated by 4.2)

### Step 8: Validate Assembly

Before outputting, verify:
- All sections populated (no null required fields)
- Schema is valid JSON
- All upstream skill outputs included
- Section briefs generated for ALL downstream writing skills
- campaign-brief.json minimum 10KB
- campaign-brief-summary.md minimum 5KB

## Quality Gate

- [ ] All 6 major sections assembled: metadata, campaign_summary, coherence_audit, creative_direction, section_briefs, human_decisions
- [ ] Section briefs generated for ALL 7 downstream writing skills (11-17)
- [ ] Each section brief is SPECIFIC to this campaign (no generic instructions)
- [ ] human_approved = false (not yet reviewed)
- [ ] Schema valid JSON
- [ ] campaign-brief.json >= 10KB
- [ ] campaign-brief-summary.md >= 5KB generated
- [ ] Threading guide placeholder present (populated by 4.2)
- [ ] No anti-slop violations anywhere in the assembled brief

---

## Anti-Patterns

- **Generic section briefs:** "Write compelling copy about the mechanism" — REJECT. Must cite specific mechanism name, specific proof assets, specific emotional target.
- **Missing sections:** Any of the 7 downstream skill briefs missing — FAIL
- **Pre-approved brief:** Setting human_approved = true before human review — VIOLATION
- **Schema violations:** Invalid JSON or missing required fields — automatic rejection

---

## Handoff

Pass assembled `campaign-brief.json` and `campaign-brief-summary.md` to:
- **4.2-threading-guide-generator.md** (threading guide added to the brief)
- **4.3-human-review-interface.md** (complete brief presented for human review)
- On COMPLETE: All downstream writing skills (11-17) via campaign-brief-outputs/
