# Layer 0.4 — Input Validator

**Skill:** 09 — Campaign Brief
**Microskill:** 0.4 — Input Validator
**Layer:** 0
**Model:** haiku
**Purpose:** Validate all inputs from 0.1, 0.2, and 0.3 are present, well-formed, and above minimum thresholds before synthesis begins.

---

## Input

| Source | File | Required |
|--------|------|----------|
| 0.1-upstream-loader | `0.1-upstream-context.json` | Yes |
| 0.2-vault-intelligence-loader | `0.2-vault-intelligence.json` | Yes |
| 0.3-teachings-loader | `0.3-teaching-index.json` | Yes |
| Pipeline Handoff Registry | `pipeline-handoff-registry.md` (Handoffs 1-8) | Yes |

## Output

| File | Format | Min Size |
|------|--------|----------|
| `0.4-validation-report.json` | JSON | 1KB |

## Execution Steps

### Step 1: Validate 8 Upstream Packages Exist

Check each upstream package against the Pipeline Handoff Registry schemas:

```
CRITICAL (HALT if missing):
  - deep-research-outputs/ (market snapshot, avatar, competitive landscape)
  - proof-inventory-output.json
  - root-cause-package.yaml
  - mechanism-package.json
  - promise-output.json
  - big-idea-output.json
  - offer-package.json
  - structure-package.json

IF any CRITICAL package missing:
  LOG: "GATE_0 FAILED: [package] not found"
  ACTION: HALT
  MESSAGE: "Run Skill [N] before 09-campaign-brief"
```

### Step 2: Validate Critical Field Thresholds

Check each package for minimum quality:

```
CHECK 1: proof-inventory overall_strength >= 40
  IF FAIL: HALT — "Insufficient proof for brief construction"

CHECK 2: mechanism has a NAME (not just description)
  IF FAIL: WARN — "Mechanism lacks name — downstream coherence will be weaker"

CHECK 3: big-idea exists as concrete concept (not vague direction)
  IF FAIL: HALT — "Big Idea not crystallized — run Skill 06 to completion"

CHECK 4: offer stack is complete (product + guarantee + bonuses defined)
  IF FAIL: WARN — "Offer incomplete — creative direction may need revision after 07-offer"

CHECK 5: structure has section outline with >= 5 CPB chunks
  IF FAIL: HALT — "Structure package insufficient — run Skill 08 to completion"

CHECK 6: research includes awareness level classification
  IF FAIL: WARN — "No awareness level found — lead type recommendation will be less precise"
```

### Step 3: Validate Vault Intelligence Completeness

```
CHECK 7: vault_intelligence.loading_status != "empty"
  IF FAIL: WARN — "No vault intelligence available — recommendations will lack empirical grounding"

CHECK 8: teaching_index.loading_status = true
  IF FAIL: WARN — "Teaching frameworks not loaded — scoring criteria will lack foundational reference"
```

### Step 4: Cross-Reference Upstream Package Schemas

Validate upstream package fields against `pipeline-handoff-registry.md` (Handoffs 1-8):
- Verify required fields present in each package
- Verify field types match expected schemas
- Verify minimum size thresholds met
- Flag any schema violations for human review

### Step 5: Generate Validation Report

```yaml
validation_report:
  timestamp: string
  packages_checked: 8
  packages_found: integer
  packages_missing: [string]

  critical_checks:
    - check_id: string
      check_name: string
      status: enum[PASS, FAIL, WARN]
      detail: string
      remediation: string  # only if FAIL or WARN

  schema_validation:
    - package: string
      fields_expected: integer
      fields_found: integer
      schema_compliant: boolean
      violations: [string]

  overall_status: enum[PASS, FAIL]
  halt_reasons: [string]  # empty if PASS
  warnings: [string]
  proceed_to_layer_1: boolean
```

### Step 6: Gate Decision

```
IF any HALT condition triggered:
  overall_status = FAIL
  proceed_to_layer_1 = false
  LOG all halt_reasons

IF only WARN conditions:
  overall_status = PASS
  proceed_to_layer_1 = true
  LOG all warnings for human awareness

IF all checks pass:
  overall_status = PASS
  proceed_to_layer_1 = true
```

## Quality Gate

- [ ] All 8 upstream packages checked against Pipeline Handoff Registry
- [ ] Proof inventory overall_strength >= 40 confirmed
- [ ] Mechanism has a name (not just description) — HALT or WARN logged
- [ ] Big Idea is a concrete concept — HALT if vague
- [ ] Structure has >= 5 CPB chunks — HALT if insufficient
- [ ] Offer stack completeness verified — WARN if incomplete
- [ ] Vault intelligence loading status checked
- [ ] Teaching index loading status checked
- [ ] Schema cross-reference against pipeline-handoff-registry.md complete
- [ ] Validation report generated with clear PASS/FAIL and remediation instructions
- [ ] Gate 0 status logged: PASS or FAIL (no invented statuses)

## Handoff

Pass `validation_report` to:
- **Layer 1 microskills** (proceed only if `proceed_to_layer_1 = true`)
- **Human** (if FAIL, present halt_reasons with remediation steps)
