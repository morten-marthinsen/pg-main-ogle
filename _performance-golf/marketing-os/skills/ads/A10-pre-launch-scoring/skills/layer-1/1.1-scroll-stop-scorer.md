# 1.1: Scroll-Stop Scorer

**Version:** 1.0
**Layer:** 1 (Individual Variant Scoring)
**Type:** Judgment / Prediction
**Model:** opus
**Dependencies:** 0.1 (Variant Matrix), 0.2 (Competitive Intelligence), 0.3 (Arena Scores), 0.5 (Benchmarks)
**Output:** scroll_stop_scores.md

---

## Purpose

Score EVERY variant on Dimension 1: Scroll-Stop Probability. This is the most critical prediction dimension — an ad that does not stop the scroll has zero chance of converting. Every score must trace to specific evidence from A01 competitive intelligence, A06 Arena evaluation, AD-HOOK-TAXONOMY.md benchmarks, and structural analysis. Every score must include a confidence range and confidence level — point estimates without ranges are NOT acceptable.

---

```yaml
input:
```

## Process

### Step 1: Load All Evidence Sources

```
READ all 4 input files
FOR EACH variant, COLLECT available evidence:

EVIDENCE SOURCE A: Hook Type Saturation (from 0.2)
  LOOK UP variant's hook_type in hook_saturation_map
  DETERMINE: oversaturated / moderate / underused / absent
  SCORING IMPACT:
    - Oversaturated: -1 to -2 penalty (audience is tired of this approach)
    - Moderate: neutral
    - Underused: +1 boost (competitive whitespace)
    - Absent: +1 to +2 boost (completely novel for this market)

EVIDENCE SOURCE B: Arena Scroll-Stop Score (from 0.3)
  LOOK UP variant's parent_concept_id in arena_scores
  EXTRACT scroll_stop_power score (1-10)
  IF no Arena score available: note as LOW confidence data gap

EVIDENCE SOURCE C: Benchmark Hook Rate (from 0.5)
  LOOK UP variant's hook_type in hook_benchmarks
  EXTRACT benchmark_hook_rate for this hook type
  COMPARE to 30% target threshold

EVIDENCE SOURCE D: Winning Specimens (from 0.2)
  CHECK fatigue_reference for similar hook types
  DO similar hooks appear in TIER_WINNER specimens (90+ days)?
  IF YES: strong signal of proven scroll-stop power

EVIDENCE SOURCE E: Structural Analysis
  ANALYZE the variant's hook_text directly:
    - Pattern interrupt present? (unexpected statement, question, visual)
    - Specificity present? (numbers, names, timeframes — not vague)
    - Curiosity gap present? (open loop that demands resolution)
    - First frame/line compelling? (strongest element first)
  EACH structural element: present (+0.5) or absent (0)
```

### Step 2: Score Each Variant

```
FOR EACH variant in the inventory:

  1. START with the Arena scroll_stop_power score as baseline (if available)
     IF no Arena score: START with structural analysis as baseline

  2. ADJUST for competitive saturation:
     - Oversaturated hook type: -1 to -2
     - Underused hook type: +1 to +2

  3. ADJUST for benchmark alignment:
     - Hook type with >30% benchmark rate: +0.5
     - Hook type with <15% benchmark rate: -0.5

  4. ADJUST for winning specimen match:
     - Similar hooks in TIER_WINNER: +0.5
     - No similar hooks in competitive set: neutral (novel, uncertain)

  5. ADJUST for structural elements:
     - Pattern interrupt + specificity + curiosity gap all present: +0.5
     - Missing 2+ structural elements: -0.5

  6. CLAMP final score to 1-10 range

  7. DETERMINE confidence range:
     - Count evidence sources available (A through E)
     - 4-5 sources: range = ±1 (e.g., score 8 → range 7-9)
     - 2-3 sources: range = ±1.5 (e.g., score 7 → range 5.5-8.5)
     - 1 source: range = ±2 (e.g., score 6 → range 4-8)

  8. DETERMINE confidence level:
     - HIGH: 3+ evidence sources agree, competitive data available, Arena score exists
     - MEDIUM: 2 evidence sources agree, some data gaps
     - LOW: limited evidence, prediction based primarily on structural analysis

  9. BUILD evidence chain (MANDATORY — must cite specific data):
     "Hook type [X] has [Y]% saturation in competitive set (A01 Section 3),
      Arena scored scroll-stop at [Z]/10 (A06), benchmark hook rate for this
      type is [W]% (AD-HOOK-TAXONOMY.md), [structural elements found]."
```

### Step 3: Apply Scoring Scale

```
SCORING SCALE (from AGENT.md):

9-10: HIGH probability of scroll-stop (70-85%+ estimated hook rate)
      Evidence: hook type is underexploited in competitive set + high Arena
      score + strong benchmark data + structural pattern interrupt present

7-8:  MODERATE-HIGH probability (45-70% estimated hook rate)
      Evidence: solid Arena score + competitive space not oversaturated +
      reasonable benchmark data

5-6:  MODERATE probability (25-45% estimated hook rate)
      Evidence: adequate but not exceptional indicators across evidence sources

3-4:  LOW probability (10-25% estimated hook rate)
      Evidence: hook type oversaturated OR weak Arena score OR missing
      pattern interrupt

1-2:  VERY LOW probability (<10% estimated hook rate)
      Evidence: multiple negative signals — oversaturated type + weak
      structural hook + no competitive data supporting this approach
```

### Step 4: Verify Differentiation

```
AFTER scoring all variants:
  CALCULATE score distribution
  CHECK: Are scores meaningfully spread?

  IF all scores fall within a 1.5-point range (e.g., all between 6.5-8.0):
    → HALT: "Scroll-stop scoring is insufficiently differentiated."
    → RE-EXAMINE: Which variants have the strongest scroll-stop evidence?
    → FORCE wider spread by re-evaluating evidence more carefully

  IF any two variants have IDENTICAL scores:
    → ACCEPTABLE if different evidence chains and different confidence levels
    → NOT ACCEPTABLE if evidence chains are also similar — differentiate further
```

## Quality Checks

1. ALL variants scored (count must match total from 0.1)
2. Every score has a confidence range (not just a point estimate)
3. Every score has a confidence level (HIGH / MEDIUM / LOW)
4. Every score cites at least 2 evidence sources
5. Scores are meaningfully differentiated (not all in a 1.5-point range)
6. Estimated hook rate ranges are realistic (not all "70-85%")
7. Competitive saturation impact is explicitly documented per variant
8. Variants without Arena scores are marked LOW confidence
