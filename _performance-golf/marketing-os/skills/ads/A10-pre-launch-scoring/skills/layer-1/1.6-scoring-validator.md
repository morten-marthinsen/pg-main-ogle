# 1.6: Scoring Validator

**Version:** 1.0
**Layer:** 1 (Individual Variant Scoring)
**Type:** Validation
**Model:** opus
**Dependencies:** 1.1 (Scroll-Stop), 1.2 (Engagement), 1.3 (Conversion), 1.4 (Compliance), 1.5 (Fatigue)
**Output:** scoring_validation.md

---

## Purpose

Validate ALL Layer 1 scoring outputs for completeness, evidence quality, confidence range presence, compliance flag consistency, and score differentiation. This is the quality gate before Layer 2 (comparative ranking). If scoring is incomplete or undifferentiated, ranking will produce meaningless results. The validator catches: missing dimension scores, missing evidence citations, missing confidence ranges, unflagged compliance violations, and uniform scoring.

---

## Process

### Step 1: Completeness Check

```
FOR EACH of the 5 scoring dimension files:
  VERIFY file exists
  VERIFY file size meets minimum (3KB each)

LOAD total_variants from 0.1

FOR EACH dimension:
  COUNT variants scored in that dimension's output file
  COMPARE to total_variants

  IF scored_count < total_variants:
    RECORD: "[Dimension] scored [X] of [Y] variants — INCOMPLETE"
    STATUS: FAIL

  IF scored_count == total_variants:
    RECORD: "[Dimension] scored [X] of [X] variants — COMPLETE"
    STATUS: PASS

BUILD completeness table:
  | Dimension     | Total | Scored | % Complete | Status |
  |---------------|-------|--------|------------|--------|
  | Scroll-Stop   | [Y]   | [X]    | [X]%       | [P/F]  |
  | Engagement    | [Y]   | [X]    | [X]%       | [P/F]  |
  | Conversion    | [Y]   | [X]    | [X]%       | [P/F]  |
  | Compliance    | [Y]   | [X]    | [X]%       | [P/F]  |
  | Fatigue       | [Y]   | [X]    | [X]%       | [P/F]  |

REQUIREMENT: ALL dimensions must be 100% complete.
```

### Step 2: Evidence Quality Check

```
FOR EACH variant across ALL 5 dimensions:
  COUNT evidence sources cited per score

  MINIMUM: 2 evidence sources per score

  IF any score cites < 2 evidence sources:
    RECORD: "Variant [ID], Dimension [X]: only [N] evidence source(s)"
    STATUS: FAIL — add evidence citations

BUILD evidence quality table:
  | Dimension     | Variants | All Have 2+ Sources | % With Evidence |
  |---------------|----------|---------------------|-----------------|
  | Scroll-Stop   | [Y]      | [Y/N]               | [X]%            |
  | Engagement    | [Y]      | [Y/N]               | [X]%            |
  | Conversion    | [Y]      | [Y/N]               | [X]%            |
  | Compliance    | [Y]      | [Y/N]               | [X]%            |
  | Fatigue       | [Y]      | [Y/N]               | [X]%            |

REQUIREMENT: 100% of scores must cite 2+ evidence sources.
```

### Step 3: Confidence Range Check

```
FOR EACH variant across ALL 5 dimensions:
  VERIFY: confidence_range exists (e.g., "7-9")
  VERIFY: confidence_level exists (HIGH / MEDIUM / LOW)

  IF confidence_range is MISSING:
    RECORD: "Variant [ID], Dimension [X]: missing confidence range"
    STATUS: FAIL — add confidence range

  IF confidence_level is MISSING:
    RECORD: "Variant [ID], Dimension [X]: missing confidence level"
    STATUS: FAIL — add confidence level

  IF confidence_range is a POINT ESTIMATE (e.g., "8" instead of "7-9"):
    RECORD: "Variant [ID], Dimension [X]: point estimate instead of range"
    STATUS: FAIL — express as range

REQUIREMENT: 100% of scores must have confidence ranges AND confidence levels.
```

### Step 4: Compliance Flag Consistency Check

```
FROM 1.4 compliance scores:
  EXTRACT all variants with compliance score <= 3
  VERIFY each is tagged as compliance_flag with launch_blocked = true

  EXTRACT all variants with compliance score 4-5
  VERIFY each has a compliance_warning

  CHECK for inconsistencies:
    - Variant with forbidden claims scoring > 3 (should be impossible)
    - Variant with no issues scoring < 6 (over-flagging)
    - Variant flagged but no specific_issue documented
    - Variant flagged but no recommended_fix provided

  IF inconsistencies found:
    RECORD each inconsistency
    STATUS: FAIL — fix compliance scoring

RECORD:
  compliance_flags_count: integer (score <= 3)
  compliance_warnings_count: integer (score 4-5)
  all_flags_have_specific_issues: boolean
  all_flags_have_recommended_fixes: boolean
```

### Step 5: Differentiation Check

```
FOR EACH dimension separately:
  EXTRACT all scores for that dimension
  CALCULATE:
    - min_score
    - max_score
    - score_spread = max - min
    - standard_deviation

  CHECK: Is the spread meaningful?
    - Spread < 1.5 points: FAIL — "Insufficiently differentiated"
    - Spread 1.5-3.0: WARNING — "Narrow differentiation"
    - Spread > 3.0: PASS — "Meaningful differentiation"

FOR ALL 5 dimensions combined:
  CHECK: Are any two variants IDENTICAL across all 5 dimensions?
    FOR EACH pair of variants:
      IF all 5 dimension scores are identical (within 0.1):
        RECORD: "Variants [A] and [B] are undifferentiated across all dimensions"
        STATUS: FAIL — re-examine scoring for these variants

RECORD:
  undifferentiated_variant_pairs: [list of pairs]
  per_dimension_spread:
    scroll_stop: float
    engagement: float
    conversion: float
    compliance: float
    fatigue: float
  dimensions_below_1.5_spread: integer  # should be 0
```

### Step 6: Produce Scoring Completeness Check

```
BUILD the MANDATORY SCORING COMPLETENESS CHECK table:

┌─────────────────────────────────────────────────────────────────────────────┐
│  SCORING COMPLETENESS CHECK - [timestamp]                                    │
│                                                                              │
│  ┌────────────────────────────────────────────────────────────────────────┐ │
│  │ DIMENSION           │ VARIANTS │ SCORED │ % COMPLETE │ WITH EVIDENCE │ │
│  ├────────────────────────────────────────────────────────────────────────┤ │
│  │ Scroll-Stop         │ [X]      │ [X]    │ [X]%       │ [X]%          │ │
│  │ Engagement          │ [X]      │ [X]    │ [X]%       │ [X]%          │ │
│  │ Conversion          │ [X]      │ [X]    │ [X]%       │ [X]%          │ │
│  │ Compliance          │ [X]      │ [X]    │ [X]%       │ [X]%          │ │
│  │ Fatigue             │ [X]      │ [X]    │ [X]%       │ [X]%          │ │
│  └────────────────────────────────────────────────────────────────────────┘ │
│                                                                              │
│  ALL DIMENSIONS HAVE CONFIDENCE RANGES: [Y/N]                               │
│  ALL SCORES CITE 2+ EVIDENCE SOURCES: [Y/N]                                │
│  COMPLIANCE FLAGS (score <= 3): [count — these MUST be addressed]           │
│  UNDIFFERENTIATED VARIANTS: [count — identical scores across all 5]         │
│                                                                              │
│  REQUIRED: 100% on ALL dimensions with 100% evidence citation.              │
│  OVERALL: [PASS - proceed to Layer 2] or [FAIL - complete scoring]          │
└─────────────────────────────────────────────────────────────────────────────┘

OVERALL VERDICT:
  IF all checks pass: "PASS — proceed to Layer 2 (Comparative Analysis)"
  IF any check fails: "FAIL — [specific items to fix]"
```

## Quality Checks

1. All 5 dimension scoring files loaded successfully
2. Completeness check runs against every variant in inventory
3. Evidence quality check verifies 2+ sources per score
4. Confidence range check verifies ranges AND levels
5. Compliance flag consistency verified (no flags without issues/fixes)
6. Differentiation check catches uniform scoring
7. Undifferentiated variant pairs identified
8. Overall verdict is PASS or FAIL only (no intermediate states)
9. If FAIL: specific items_to_fix listed with actionable instructions
