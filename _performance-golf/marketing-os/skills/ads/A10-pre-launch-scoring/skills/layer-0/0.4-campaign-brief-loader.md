# 0.4: Campaign Brief Loader

**Version:** 1.0
**Layer:** 0 (Foundation)
**Type:** Data Loading
**Dependencies:** Campaign Brief (Skill 09), A03 Format Strategy
**Output:** campaign_brief_extract.md

---

## Purpose

Load the Campaign Brief (Skill 09) and A03 Format Strategy outputs. Extract all campaign numbers needed for testing strategy: weekly ad spend, AOV, campaign timeline, funnel stage targets, platform priorities, and compliance constraints. Calculate the testing volume formula (min creatives/week) and kill threshold (3x AOV). These numbers are NOT optional — testing strategy without budget math is not strategy.

---

## Inputs

```yaml
input:
  from_skill_09:
    campaign_brief_path: "[project]/09-campaign-brief/campaign-brief.json"
  from_A03:
    format_strategy_path: "[project]/A03-format-strategy/FORMAT-STRATEGY.md"
    # OR equivalent A03 output
```

---

## Process

### Step 1: Locate Campaign Brief

```
VERIFY: Campaign Brief output exists at specified path
IF NOT EXISTS:
  → HALT: "Campaign Brief not found. Execute Skill 09 first."
  → A10 requires campaign numbers for testing strategy.
```

### Step 2: Extract Campaign KPIs and Budget

```
READ campaign-brief.json
EXTRACT the following — ALL are REQUIRED:

BUDGET & ECONOMICS:
  - weekly_ad_spend: dollar amount (REQUIRED — no default)
  - average_order_value (AOV): dollar amount (REQUIRED — no default)
  - target_cpa: dollar amount (if specified)
  - target_roas: multiplier (if specified)
  - total_campaign_budget: dollar amount (if specified)
  - campaign_timeline_weeks: integer

FUNNEL & TARGETING:
  - funnel_stage_target: awareness / consideration / conversion
  - platform_priorities: [list of platforms in priority order]
  - target_audience: demographic + psychographic

COMPLIANCE:
  - vertical: string (determines compliance constraints)
  - vertical_compliance_constraints: from ad-verticals/ config
  - specific_claim_restrictions: any brief-specified restrictions
  - platform_specific_restrictions: from A03

IF weekly_ad_spend is MISSING:
  → HALT: "Weekly ad spend not found in Campaign Brief. Cannot calculate
    testing volume or budget allocation. Get this number from the client."

IF aov is MISSING:
  → HALT: "AOV not found in Campaign Brief. Cannot calculate kill threshold
    or testing volume. Get this number from the client."
```

### Step 3: Load A03 Format Strategy Compliance Notes

```
READ A03 Format Strategy output
EXTRACT:
  - platform_format_decisions: which platforms, which formats
  - compliance_notes: any compliance constraints identified during format strategy
  - format_specific_restrictions: restrictions by format type
  - word_count_limits: per format and platform
```

### Step 4: Calculate Testing Volume Formula

```
FORMULA: min_creatives_per_week = weekly_ad_spend / (3 × AOV)

CALCULATE:
  min_creatives_per_week = FLOOR(weekly_ad_spend / (3 * aov))

EXAMPLES for verification:
  $5,000 spend / (3 × $75 AOV) = 22 creatives/week
  $10,000 spend / (3 × $100 AOV) = 33 creatives/week
  $25,000 spend / (3 × $200 AOV) = 42 creatives/week

THIS NUMBER CAPS TIER 1 SIZE.
If 30 variants qualify for Tier 1 but min_creatives_per_week = 22,
only the top 22 become Tier 1.
```

### Step 5: Calculate Kill Threshold

```
FORMULA: kill_threshold = 3 × AOV

CALCULATE:
  kill_threshold = 3 * aov

EXAMPLES for verification:
  $50 AOV → $150 kill threshold
  $100 AOV → $300 kill threshold
  $200 AOV → $600 kill threshold

THIS IS EXACT. Not "around" 3x. Not "roughly" $300. The specific dollar amount.
A variant that has spent $[kill_threshold] with zero conversions is CUT.
```

### Step 6: Build Campaign Numbers Summary

```
COMPILE all extracted numbers into a structured summary:

CAMPAIGN MATH:
  - Weekly ad spend: $[X]
  - AOV: $[X]
  - Min creatives/week: [X]
  - Kill threshold: $[X]
  - Target CPA: $[X] (or "not specified")
  - Target ROAS: [X]x (or "not specified")
  - Campaign timeline: [X] weeks
  - Total testing cycles: FLOOR(campaign_timeline / 1.5) (each cycle ~1.5 weeks)

FUNNEL CONTEXT:
  - Primary funnel stage: [awareness / consideration / conversion]
  - Conversion expectation calibration:
    - TOFU (awareness): Low direct conversion expected, optimize for engagement
    - MOFU (consideration): Moderate conversion, optimize for click-through
    - BOFU (conversion): High conversion expected, optimize for CPA/ROAS

COMPLIANCE BASELINE:
  - Vertical: [X]
  - Compliance risk level: [LOW for golf / MODERATE for personal-dev /
    HIGH for health/finance]
  - Platform restrictions: [list]
  - Claim restrictions: [list]
```

7. Compliance baseline established with vertical-specific risk level
