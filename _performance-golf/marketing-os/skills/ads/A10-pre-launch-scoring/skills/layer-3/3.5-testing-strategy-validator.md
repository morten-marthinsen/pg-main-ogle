# 3.5: Testing Strategy Validator

**Version:** 1.0
**Layer:** 3 (Testing Strategy)
**Type:** Validation
**Model:** opus
**Dependencies:** 3.1 (Testing Sequence), 3.2 (Budget Allocation), 3.3 (Kill Criteria), 3.4 (Scale Criteria)
**Output:** testing_strategy_validation.md

---

## Purpose

Validate the complete testing strategy for internal consistency, mathematical correctness, and operational readiness. This is the quality gate before Layer 4 (Output Packaging). If the testing strategy has budget math errors, vague kill criteria, or missing scale triggers, the PRE-LAUNCH-SCORECARD will contain bad recommendations. Every element must be specific and measurable — no vague strategies.

## Process

### Step 1: Testing Sequence Validation

```
CHECK 1: All Tier 1 variants in Week 1
  VERIFY: Every Tier 1 variant from 2.3 appears in Phase 1 launch
  IF any Tier 1 variant missing from Week 1: FAIL

CHECK 2: Platform review timelines included
  VERIFY: Timeline accounts for 24-72 hour review delays
  IF review delays not factored: FAIL

CHECK 3: Timeline fits campaign
  VERIFY: Testing phases fit within campaign_timeline_weeks
  VERIFY: At least 1 complete testing cycle is possible
  IF timeline exceeds campaign: FAIL — adjust phases
```

### Step 2: Budget Math Validation

```
CHECK 4: Budget sums correctly
  total_allocated = SUM(all per-variant weekly spend) + reserve
  VERIFY: total_allocated == weekly_ad_spend
  TOLERANCE: $0 (EXACT — no rounding errors)
  IF budget doesn't sum: FAIL — fix allocation math

CHECK 5: Per-variant minimum spend met
  FOR EACH Tier 1 variant:
    DETERMINE minimum daily spend threshold:
      IF aov <= $50: $20/day minimum
      IF aov $50-$150: $30/day minimum
      IF aov > $150: $50/day minimum
    VERIFY: per_variant_daily >= minimum
  IF any variant below minimum: FAIL — reduce Tier 1 count or increase budget

CHECK 6: Reserve budget allocated
  VERIFY: reserve >= 20% of weekly_ad_spend
  IF reserve < 20%: FAIL — insufficient reserve for scaling
```

### Step 3: Kill Criteria Validation

```
CHECK 7: Kill criteria are specific
  FOR EACH Tier 1 variant:
    VERIFY: kill_trigger is defined with specific dollar amount
    VERIFY: kill_dollar_threshold == 3 × AOV (exact)
    VERIFY: secondary kill signals have specific metric thresholds
  IF any kill trigger is vague ("kill when underperforming"): FAIL

CHECK 8: Kill timeline enforced
  VERIFY: minimum data window is 72 hours (no premature kills)
  VERIFY: maximum decision window is 7 days (no indefinite runs)
  IF timeline constraints missing: FAIL

CHECK 9: Reallocation targets defined
  FOR EACH Tier 1 variant:
    VERIFY: reallocation_target is specified (what happens to budget when killed)
  IF any reallocation target missing: FAIL
```

### Step 4: Scale Criteria Validation

```
CHECK 10: Scale criteria are specific
  VERIFY: Scale trigger has all 4 conditions:
    1. CPA target (specific dollar amount)
    2. Minimum conversions (specific number)
    3. Minimum days (specific number)
    4. Stability requirement (specific metric)
  IF any condition missing or vague: FAIL

CHECK 11: Scale protocol defined
  VERIFY: Increment percentage specified (20-30%)
  VERIFY: Minimum wait between scales (48 hours)
  VERIFY: Budget source for scaling documented
  IF any protocol element missing: FAIL

CHECK 12: Watch metrics defined
  VERIFY: All 4 watch metrics have specific thresholds
  VERIFY: Each metric has a breach protocol (what to do when exceeded)
  IF any metric missing threshold or protocol: FAIL
```

### Step 5: Cross-Validation

```
CHECK 13: Kill timing aligns with budget
  FOR EACH Tier 1 variant:
    days_to_kill = kill_threshold / per_variant_daily_spend
    VERIFY: days_to_kill >= 3 (minimum data window)
    VERIFY: days_to_kill <= 14 (reasonable timeline)
  IF misalignment: WARNING (not FAIL, but document concern)

CHECK 14: Scale budget is available
  max_scale_spend = reserve_budget + (estimated killed variant budget per week)
  VERIFY: Sufficient budget exists to scale at least 1 winner by 20%
  IF insufficient: WARNING — scaling may be limited

CHECK 15: Strategy connects to campaign numbers
  VERIFY: Every dollar amount traces to campaign brief numbers
  VERIFY: Kill threshold = exactly 3 × AOV
  VERIFY: min_creatives_per_week calculation is correct
  IF any number doesn't trace: FAIL
```

## Quality Checks

1. All 15 validation checks executed
2. Budget math verified to exact dollar (zero tolerance)
3. Kill criteria verified as specific (dollar amounts, not vague)
4. Scale criteria verified as complete (all 4 conditions + protocol + watch metrics)
5. Timeline verified against campaign duration
6. Reserve budget verified at 20%+ of weekly spend
7. Platform review timelines verified as included
8. Cross-validation checks identify any logical inconsistencies
9. Overall verdict is PASS or FAIL only
10. If FAIL: specific items_to_fix listed with actionable instructions
