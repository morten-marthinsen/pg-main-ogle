# 3.2: Human Selection Capture

## Purpose
Capture human selection. BLOCKING GATE -- cannot proceed without human input. Capture: which concepts are selected for A07, which hybrids are preferred, any modifications or notes for copy production, any concepts explicitly rejected with reason. Maximum 2 revision cycles if human requests changes.

## Inputs
- 3.1-presentation-assembly output (all candidates presented for human review)
- Human input (selection, notes, modifications, rejections)

## Process
```
STEP 1: Present 3.1 presentation to human
  - VERIFY: 3.1-presentation-assembly.md exists and is complete
  - PRESENT: All concepts ranked with scoring summaries and recommendations
  - AWAIT: Human selection input

STEP 2: BLOCKING GATE -- Wait for human input
  - This step CANNOT be automated
  - This step CANNOT be bypassed
  - This step CANNOT be skipped
  - The agent MUST wait for human to provide selection

STEP 3: Capture human selection
  FOR EACH concept in presentation:
    A. RECORD selection status:
      - SELECTED: advancing to A07
      - REJECTED: not advancing (capture reason)
    B. IF selected, RECORD version preference:
      - pure_round3: use best Round 3 persona evaluation as-is
      - hybrid_H1 / hybrid_H2 / hybrid_H3: use specific hybrid version
      - modified: human has specific changes to incorporate
    C. RECORD human notes:
      - Any modifications requested
      - Any specific instructions for A07 (Copy Production)
      - Any preferences for variant generation

STEP 4: CHECK if human requests revision
  IF revision_requested:
    IF revision_cycle < 2:
      1. Capture human's revision direction
      2. Run targeted improvement round (The Architect only, focused on human's direction)
      3. Score improved concept against 7 criteria
      4. Present revised concept to human
      5. Increment revision_cycle
      6. RETURN to STEP 2 (await human re-selection)
    IF revision_cycle = 2:
      HALT -- maximum revision cycles reached
      Present to human: "Maximum 2 revision cycles reached. Please select from
      available concepts or provide manual direction for A07."

STEP 5: Create HUMAN_SELECTION.yaml
  - ONLY created when human selection is received
  - Status is ALWAYS "PASS" (gate does not exist until human provides input)
  - Record all selected concepts with versions and scores
  - Record all rejected concepts with reasons
  - Record downstream instructions for A07

STEP 6: Verify selection completeness
  - At least 1 concept selected (preferably 3+)
  - Every selected concept has version specified
  - Every rejected concept has reason documented
  - Downstream instructions captured (even if "none")
```
