# 2.1: Round 1 Persona Evaluation

## Purpose
All 7 ad-specific personas independently evaluate each concept. Each persona produces a complete evaluation per the persona_evaluation format from the AGENT.md. The Architect generates an integrated evaluation (not synthesis of others). Specimens MUST be referenced in every evaluation.

## Inputs
- 1.3-concept-summary-generator output (concept summaries for evaluation)
- 0.5-persona-specimen-validator output (loaded specimens per persona)
- 0.1-upstream-package-loader output (strategic context)
- 0.0.1-vertical-profile-loader output (vertical-specific scoring adjustments)
- Persona behavioral specifications from AGENT.md (all 7 personas)

## Process
```
STEP 1: Load concept summaries and specimens
  - READ: layer-1-outputs/1.3-concept-summary-generator.md
  - VERIFY: Specimens loaded in active context per persona (from 0.5 output)

STEP 2: FOR EACH concept (C-001, C-002, ...):
  FOR EACH persona (7 total):

    A. DEEP READ concept summary (effort: max)
      - Understand hook + script + visual as ONE integrated unit
      - DO NOT evaluate elements in isolation

    B. CROSS-REFERENCE loaded specimens
      - Find 1-2 specimens most similar to this concept
      - Use as comparison benchmark

    C. EVALUATE from persona lens:
      - DR Strategist: conversion architecture, persuasion sequence, CTA alignment
      - Scroll Stopper: first 3 seconds, pattern interrupt, curiosity gap
      - UGC Native: platform authenticity, creator voice, ad signal detection
      - Brand Builder: memorability, distinctiveness, emotional resonance
      - Data Scientist: variant potential, scalability, framework alignment
      - Visual Storyteller: visual-copy coherence, visual narrative, shot specificity
      - Architect: integration across ALL 7 criteria simultaneously

    D. SCORE against 7 ad-specific criteria:
      - Scroll-Stop Power (25%): [1-10] + evidence
      - Visual-Copy Coherence (15%): [1-10] + evidence
      - Mechanism Clarity (15%): [1-10] + evidence
      - Platform Nativeness (15%): [1-10] + evidence
      - Proof Integration (10%): [1-10] + evidence
      - CTA Strength (10%): [1-10] + evidence
      - Memorability (10%): [1-10] + evidence

    E. CALCULATE weighted total

    F. GENERATE improvement recommendation:
      - Strongest element
      - Weakest element
      - 3-5 specific actionable improvements
      - Revised concept sketch (brief)

    G. REFERENCE specimen: "[which loaded specimen informed this evaluation]"

STEP 3: Compile all 7 evaluations per concept

CRITICAL RULES:
  - Concepts MUST be evaluated as ATOMIC UNITS (hook + script + visual together)
  - Every score MUST include specific evidence from the concept
  - Every evaluation MUST reference at least one loaded specimen
  - Persona perspectives MUST remain distinct (no consensus drift)
  - The Architect evaluates independently (not by aggregating other personas)
```
