# 1.5: Layer 1 Validator

## Purpose
Verify: every ad concept has a framework assignment. Every assignment cites all 5 selection variables. No framework was assigned without rationale. No framework is length-incompatible. No framework is platform-incompatible.

## Inputs
- Ad Concept Inventory (from 1.1)
- Framework Assignments (from 1.3)
- Bumper Ad Design (from 1.4 if applicable)

## Process

### Step 1: Load Layer 1 Outputs
```
READ: layer-1-outputs/1.1-ad-concept-inventory.md
READ: layer-1-outputs/1.3-framework-assignments.md
READ: layer-1-outputs/1.4-bumper-ad-design.md (if exists)
```

### Step 2: Completeness Check
```
VERIFY: Every concept from 1.1 has an assignment in 1.3
CROSS-REFERENCE: concept_ids from 1.1 against assignments in 1.3
IF any concept missing assignment:
  FAIL: "Concept [ID] has no framework assignment"
  LIST all missing assignments
```

### Step 3: Rationale Quality Check
```
FOR EACH assignment in 1.3:
  VERIFY: selection_rationale is not empty
  VERIFY: five_variable_citation shows all 5 as "cited: true"
  IF any variable not cited:
    FAIL: "Assignment for [concept_id] missing [variable] citation in rationale"
```

### Step 4: Compatibility Checks
```
FOR EACH assignment:
  CHECK length-incompatibility:
    IF (framework == "Edutainment" OR framework == "Story Narrative") AND length < 60s:
      FAIL: "Length-incompatible: [framework] cannot execute at [length]"
    IF framework == "Fast-Paced Viral" AND length > 30s:
      FAIL: "Length-incompatible: Fast-Paced Viral not suitable for [length]"
  
  CHECK platform-incompatibility:
    IF framework == "Edutainment" AND platform == "TikTok" AND length < 120s:
      WARN: "Platform mismatch: Edutainment unusual for TikTok short-form"
    IF framework == "Story Narrative" AND platform == "TikTok" AND length < 60s:
      FAIL: "Platform-incompatible: Story Narrative requires 60s+ on TikTok"
```

### Step 5: Framework Distribution Analysis
```
COUNT assignments per framework
CALCULATE percentages
IF any single framework > 80% of all assignments:
  WARN: "Low diversity: [framework] assigned to [X]% of concepts. Verify analysis."
IF any framework assigned 0 times:
  NOTE: "[Framework] not used in this campaign. (Acceptable if not suitable.)"
```

### Step 6: Bumper Validation (if applicable)
```
IF bumpers exist:
  VERIFY: All 6s concepts are marked as "Bumper" framework in 1.3
  VERIFY: All bumper designs in 1.4 exist for bumper concepts
  VERIFY: All bumper word counts are 10-15 words
```

### Step 7: Generate Validation Report
```
CREATE validation_report:
  total_concepts: integer
  concepts_with_assignments: integer
  all_assignments_complete: boolean
  all_rationales_complete: boolean
  incompatible_assignments: [list or empty]
  framework_distribution: object
  diversity_status: HEALTHY | WARN
  bumper_validation: PASS | NOT_APPLICABLE
  overall_verdict: PASS | FAIL
  issues: [list or empty]
```

## Output Schema
```yaml
layer_1_validation:
  validation_timestamp: string
  overall_verdict: PASS | FAIL
  checks:
    completeness: PASS | FAIL
    rationale_quality: PASS | FAIL
    length_compatibility: PASS | FAIL
    platform_compatibility: PASS | WARN | FAIL
    framework_diversity: HEALTHY | WARN
    bumper_validation: PASS | NOT_APPLICABLE
  issues_found: [list or empty]
  warnings: [list or empty]
  framework_distribution: object
  ready_for_layer_2: boolean
```

## Output File
- **Path:** `layer-1-outputs/1.5-layer-1-validator.md`
- **Minimum size:** 1KB

## Quality Checks
1. Validation ran against all Layer 1 outputs
2. If overall_verdict == FAIL, specific issues are documented
3. If ready_for_layer_2 == false, the blocker is clearly stated
4. Framework distribution percentages sum to 100%
5. All FAIL conditions are justified with evidence

## Dependencies
- ALL previous Layer 1 microskills (1.1, 1.2, 1.3, 1.4) must have executed
- This microskill MUST run last in Layer 1

## Gate 1 Dependency
**This microskill's verdict determines Gate 1 status.** If `overall_verdict == FAIL`, Gate 1 is BLOCKED and Layer 2 cannot proceed. Issues must be resolved before advancing.
