# 1.4: Bumper Ad Design

## Purpose
CONDITIONAL: Only execute if any ad concepts are 6-second bumpers. Bumpers don't use standard frameworks. Design single-idea, single-message structures with 10-15 word budgets.

## Inputs
- Ad Concept Inventory (from 1.1)
- Framework Assignments (from 1.3 - to identify which concepts are bumpers)

## Process

### Step 1: Identify Bumper Concepts
```
READ: layer-1-outputs/1.3-framework-assignments.md
FILTER: concepts where target_length == "6s"
COUNT: bumper_count
IF bumper_count == 0:
  SKIP execution -- create empty output file noting "No bumper ads in campaign"
  EXIT
ELSE:
  PROCEED with bumper design
```

### Step 2: Design Bumper Structure Per Concept
```
FOR EACH bumper concept:
  DEFINE bumper structure:
    - Single idea (not a sequence)
    - Single message (one claim/benefit/promise)
    - 10-15 word budget TOTAL
    - Visual + audio must work together (not redundant)
  
  DESIGN components:
    visual_primary: "What dominates the screen" (product, result, contrast)
    audio_primary: "What is spoken or on-screen text" (10-15 words EXACT)
    one_idea: "The single takeaway" (one sentence summary)
    cta_method: "How action is prompted" (button text, voiceover end, URL display)
```

### Step 3: Bumper-Specific Word Count Enforcement
```
FOR EACH bumper:
  COUNT words in audio_primary
  VERIFY: word_count >= 10 AND word_count <= 15
  IF out of range:
    COMPRESS or EXPAND to meet 10-15 word requirement
    (15 words = 6 seconds at 2.5 words/second)
```

### Step 4: Bumper Hook Integration
```
FOR EACH bumper:
  REFERENCE: original hook from A02
  VERIFY: bumper one_idea aligns with hook's promise
  DESIGN: How the hook translates to 6-second constraint
    - Hook: "3 foods destroying your gut" → Bumper: "These 3 foods damage your gut. Discover them free."
    - Hook: "Golf pros use this secret" → Bumper: "The swing tip pros don't share. Free video inside."
```

### Step 5: Visual-Audio Pairing
```
FOR EACH bumper:
  DEFINE visual-audio relationship:
    OPTION A: Visual shows, audio tells (classic split)
    OPTION B: Visual + text overlay, minimal/no voiceover (sound-off optimization)
    OPTION C: Visual action, voiceover narrates (documentary style)
  
  SELECT optimal pairing for platform:
    TikTok/Reels: Prefer Option B (text overlay, sound-off compatible)
    YouTube pre-roll: Prefer Option A or C (sound-on assumed)
    Meta: Prefer Option B (majority sound-off viewing)
```

## Output Schema
```yaml
bumper_ad_design:
  bumper_count: integer
  bumper_designs:
    - concept_id: string
      hook_reference: string (original hook from A02)
      visual_primary: string
      audio_primary: string (10-15 words)
      word_count: integer
      one_idea: string
      cta_method: string
      visual_audio_pairing: string (A | B | C)
      platform_optimization: string
  execution_status:
    bumpers_designed: integer
    all_word_counts_valid: boolean
```

## Output File
- **Path:** `layer-1-outputs/1.4-bumper-ad-design.md`
- **Minimum size:** 1KB (or 0KB if no bumpers exist -- note empty status in file)

## Quality Checks
1. If bumpers exist, all bumper concepts have designs
2. Every bumper audio_primary is 10-15 words (exact count)
3. Every bumper has single, clear idea (not trying to communicate multiple things)
4. Visual-audio pairing specified and platform-appropriate
5. CTA method specified for every bumper

## Dependencies
- 1.1 (Ad Concept Inventory) must be complete
- 1.3 (Framework Assignments) must be complete to identify which concepts are bumpers

## Conditional Execution Note
**This microskill only executes if bumpers exist.** If no 6-second ads in the campaign, the output file notes "No bumper ads -- skill not applicable" and execution proceeds to 1.5. This is NOT a failure condition.
