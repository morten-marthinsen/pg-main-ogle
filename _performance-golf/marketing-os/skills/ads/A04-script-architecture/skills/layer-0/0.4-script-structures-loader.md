# 0.4: Script Structures Loader

## Purpose
Load the comprehensive AD-SCRIPT-STRUCTURES.md reference document. Extract all 8 frameworks with their complete structures, length-specific beat structures, vertical-specific script patterns, modular architecture reference, UGC templates, and Harmon Brothers formula. HOLD IN ACTIVE CONTEXT for Layer 1 framework selection.

## Inputs
- `./References/AD-SCRIPT-STRUCTURES.md` (master reference document)

## Process

### Step 1: Locate and Read Script Structures
```
LOCATE: ./References/AD-SCRIPT-STRUCTURES.md
READ entire file (comprehensive reference, likely 50-100KB)
VERIFY file contains all 8 framework definitions
```

### Step 2: Extract 8 Framework Definitions
```
FOR EACH of 8 frameworks:
  EXTRACT:
    framework_name: string
    framework_code: string (PAS | AIDA | BAB | Hook-Body-CTA | Story | Edutainment | UGC-DR | Fast-Paced-Viral)
    structure: [ordered list of beats/phases]
    best_length_range: string
    best_platform: [list]
    best_hook_types: [list of compatible hook type codes]
    best_awareness_level: [list]
    when_to_use: string (decision criteria)
    when_not_to_use: string (anti-criteria)
```

### Step 3: Extract Length-Specific Beat Structures
```
FOR EACH ad length (6s, 15s, 30s, 60s, 2-3min):
  EXTRACT:
    - module_presence_map (which modules appear at this length)
    - word_count_budgets (suggested distribution across modules)
    - beat_compression_guidance (how to adapt frameworks to shorter lengths)
```

### Step 4: Extract Vertical-Specific Patterns
```
FOR EACH vertical (health, golf, finance, personal-dev, technology):
  EXTRACT:
    - dominant_frameworks (which frameworks work best in this vertical)
    - named_patterns (e.g., "Doctor Reveals," "Pro Secret," "Old Way vs New Way")
    - pattern_structures (beat-by-beat breakdown)
    - compliance_notes (if applicable)
```

### Step 5: Extract Modular Architecture Reference
```
READ modular architecture section
EXTRACT:
  - 5 standard modules definition ([HOOK], [SETUP], [MECHANISM], [PROOF], [CTA])
  - module independence rules
  - clean seam design principles
  - transition types at module boundaries
  - module word count budgets by ad length
```

### Step 6: Extract UGC Templates and Harmon Brothers Formula
```
READ UGC-DR framework section
EXTRACT UGC templates:
  - Problem-Discovery-Recommendation pattern
  - Skeptic-to-Believer arc
  - Authentic discovery framing

READ Story Narrative section
EXTRACT Harmon Brothers formula:
  - Character-Problem-Guide-Plan-Action-Success pattern
  - When to use extended story (60s+)
```

### Step 7: HOLD IN ACTIVE CONTEXT
```
CRITICAL: This loaded content must remain in active context through Layer 1 execution.
Framework selection (Layer 1) requires direct reference to these structures.
DO NOT summarize. HOLD the verbatim framework definitions.
```

## Output Schema
```yaml
script_structures:
  frameworks:
    - framework_name: string
      framework_code: string
      structure: [list of beats]
      best_length: string
      best_platform: [list]

      best_awareness: [list]
      when_to_use: string
      when_not_to_use: string
  length_specific_beats:
    - ad_length: string
      module_presence: [list of which modules appear]
      word_count_distribution: object
  vertical_patterns:
    - vertical: string
      dominant_frameworks: [list]
      named_patterns: [list with structures]
  modular_architecture:
    standard_modules: [5 module definitions]
    independence_rules: [list]
    transition_principles: [list]
  ugc_templates: [list]
  harmon_brothers_formula: object
  loading_status:
    frameworks_loaded: integer (should be 8)
    verbatim_context_held: boolean (must be true)
    warnings: [list or empty]
```

- **Minimum size:** 1KB (summary of what was loaded)

2. All 8 frameworks extracted with complete definitions
3. Length-specific beat structures documented for all target lengths
4. Vertical patterns extracted for all 5 verticals
5. Modular architecture principles documented
6. CRITICAL: Verbatim content is HELD IN ACTIVE CONTEXT (not just summarized)

## Dependencies
- AD-SCRIPT-STRUCTURES.md must exist in References/ directory
- No upstream skill dependencies (reference document is static)

**This microskill's success is NOT measured by the output file size alone.** The critical requirement is that the VERBATIM framework structures are held in active context for Layer 1. The output file documents WHAT was loaded, but the loaded content itself must be available during framework selection.
