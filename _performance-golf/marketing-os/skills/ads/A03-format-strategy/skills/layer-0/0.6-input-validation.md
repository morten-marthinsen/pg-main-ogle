# 0.6: Input Validation

## Purpose

Validate that all required inputs for format strategy execution are loaded, contain expected data, and meet quality thresholds. This is the GATE 0 determination microskill. If any REQUIRED input is missing or empty, GATE 0 = FAIL and execution HALTS. This prevents proceeding with incomplete foundations.

## Inputs

All outputs from microskills 0.0.1 through 0.5:
- `layer-0-outputs/0.0.1-vertical-profile.md`
- `layer-0-outputs/0.1-campaign-brief.md`
- `layer-0-outputs/0.2-ad-intelligence.md`
- `layer-0-outputs/0.3-hook-angle-matrix.md`
- `layer-0-outputs/0.4-reference-loader.md`
- `layer-0-outputs/0.5-soul-md.md`

## Process

### Step 1: Verify File Existence

For each required input, verify the output file EXISTS:
- [ ] `0.0.1-vertical-profile.md` exists
- [ ] `0.1-campaign-brief.md` exists
- [ ] `0.2-ad-intelligence.md` exists
- [ ] `0.3-hook-angle-matrix.md` exists
- [ ] `0.4-reference-loader.md` exists
- [ ] `0.5-soul-md.md` exists (content may be warning — that's acceptable)

IF any file missing → GATE 0 = FAIL → HALT

### Step 2: Verify Campaign Brief Data

1. **CHECK budget status:**
   - Budget specified: PASS
   - Budget flagged for human input: PASS (flagged is acceptable, missing is not)
   - Budget field not addressed: FAIL

2. **CHECK target platforms:**
   - Platforms specified OR defaulted (Meta + TikTok): PASS
   - Platforms field empty: FAIL

3. **CHECK audience demographics:**
   - At least age range + awareness level present: PASS
   - All demographic fields empty: WARN but PASS (degraded quality, not blocking)

IF budget or platforms fail → GATE 0 = FAIL → HALT

### Step 3: Verify A01 Intelligence Data Quality

1. **CHECK format distribution:**
   - Percentages calculated (not estimated): PASS
   - Percentages sum to ~100% (within 95-105% range): PASS
   - Missing or "approximately X%": FAIL

2. **CHECK winning format pattern:**
   - Specific count provided (e.g., "14/20 ads are UGC"): PASS
   - Vague description without count: FAIL

3. **CHECK platform-specific intelligence:**
   - At least one target platform has extracted intelligence: PASS
   - All platforms missing intelligence: FAIL

IF format distribution or winning pattern fail → GATE 0 = FAIL → HALT

### Step 4: Verify Hook-Angle Matrix Data

1. **CHECK selected hook count:**
   - 8-10 hooks extracted: PASS
   - 6-7 hooks: WARN but PASS (low variant volume, not blocking)
   - Fewer than 6 hooks: FAIL

2. **CHECK hook data completeness:**
   - All hooks have hook text, type, scores: PASS
   - Any hook missing required fields: FAIL

IF hook count < 6 or data incomplete → GATE 0 = FAIL → HALT

### Step 5: Verify Reference Documents

1. **CHECK Framework Selection Matrix:**
   - All 8 frameworks present: PASS
   - Missing frameworks: FAIL

2. **CHECK Word Count Enforcement Table:**
   - All 5 length tiers present with exact word counts: PASS
   - Missing tiers or approximate counts: FAIL

3. **CHECK Platform-Specific Considerations:**
   - Sections extracted from AD-HOOK-TAXONOMY.md: PASS
   - Section missing: WARN but PASS (degraded quality)

IF Framework Matrix or Word Count Table fail → GATE 0 = FAIL → HALT

### Step 6: Determine Gate Status

```
IF all required verifications PASS:
  GATE 0 = PASS
  Proceed to Layer 1
ELSE:
  GATE 0 = FAIL
  HALT execution
  Document which validation(s) failed
  Return control to human
```
