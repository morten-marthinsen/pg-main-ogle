# 1.5: Base Copy Validator

## Purpose
Validate ALL base copies produced in Layer 1 (microskills 1.2, 1.3, 1.4, and any additional concept files). Verify: (a) Every module is filled -- no empty or placeholder modules. (b) Word count is within hard limits for target ad length. (c) AV format is correct -- visual column has specific shot descriptions. (d) Framework compliance -- copy follows the assigned framework structure. (e) Voice consistency -- copy matches loaded specimens in tone and register.

## Inputs
- All base copy output files (1.2, 1.3, 1.4, and any additional)
- Concept mini-briefs (from 1.1) for reference
- Word count limits table
- Persona voice specimens (for voice comparison)

## Process
```
STEP 1: READ each base copy output file (1.2, 1.3, 1.4, etc.).

STEP 2: For EACH base copy, run 5 validation checks:

CHECK A: MODULE COMPLETION
  - Does every module in the framework have content?
  - No empty modules, no "[placeholder]" text, no "TBD"
  - RESULT: [X/X] modules filled

CHECK B: WORD COUNT
  - Count words in AUDIO column ONLY
  - Compare against hard limit for target ad length:
    6s = 15 words, 15s = 40, 30s = 75, 60s = 160, 2-3min = 450
  - RESULT: [X] words vs [Y] limit -- PASS/FAIL
  - IF FAIL: flag for compression (return to 1.2/1.3/1.4)

CHECK C: AV FORMAT & VISUAL SPECIFICITY
  - Is the script in proper two-column AV format?
  - Does every visual column row specify:
    - Shot type (CU/MS/WS)
    - Subject (what's in frame)
    - Action (what's happening)
  - FAIL conditions:
    - "Show product" (too vague)
    - "B-roll of nature" (too vague)
    - Any row missing shot type, subject, or action
  - RESULT: PASS/FAIL with specific violations listed

CHECK D: FRAMEWORK COMPLIANCE
  - Does the script follow the assigned framework structure?
  - For PAS: Problem section present, Agitation builds, Solution resolves
  - For AIDA: Attention hook, Interest builds, Desire created, Action CTA
  - For BAB: Before state, After state, Bridge to product
  - [etc. for all 8 frameworks]
  - RESULT: PASS/FAIL with deviation notes

CHECK E: VOICE CONSISTENCY
  - Compare the copy's tone and register against loaded specimens
  - Does the copy sound like it belongs in the same campaign as the specimens?
  - Check: sentence length, vocabulary level, formality, energy, rhythm
  - RESULT: CONSISTENT / INCONSISTENT with notes

STEP 3: COMPILE validation results into a single report.

STEP 4: DETERMINE validator verdict.
  - ALL concepts must pass ALL 5 checks for PASS
  - Any FAIL = validator verdict is FAIL with specific remediation instructions
```
