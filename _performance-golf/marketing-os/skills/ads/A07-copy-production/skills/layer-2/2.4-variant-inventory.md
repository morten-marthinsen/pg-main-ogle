# 2.4: Variant Inventory

## Purpose
Count and catalog ALL variants produced in Layer 2 (hook swaps, CTA variants, platform adaptations) plus base copies from Layer 1. Produce a comprehensive variant inventory table with complete metadata. Verify minimum variant counts are met per concept and overall. This is the coordination document that Layer 2.5 and Layer 3 work from.

## Inputs
- All Layer 1 base copies (1.2, 1.3, 1.4)
- Hook swap generation output (2.1)
- CTA variant generation output (2.2)
- Platform adaptation output (2.3)
- Format Strategy (from 0.5 -- for platform assignment verification)

## Process
```
STEP 1: READ all Layer 1 and Layer 2 output files.

STEP 2: CATALOG every variant into the inventory table.
  For each variant, record:
    a. Variant ID (temporary -- formal IDs assigned in Layer 3)
    b. Concept ID (C-001, C-002, C-003)
    c. Variant type (base / hook_swap / cta_variant / platform_adaptation)
    d. Platform (Meta, TikTok, YouTube, Google, etc.)
    e. Target length (6s, 15s, 30s, 60s, 2-3min)
    f. Hook type (if hook swap -- taxonomy code)
    g. CTA lever (if CTA variant -- lever name)
    h. Word count (exact number from audio column)
    i. Word count status (PASS/FAIL vs limit)

STEP 3: COUNT variants per dimension.
  - Per concept: total variants
  - Per concept: hook swaps (must be >= 5)
  - Per concept: CTA variants (must be >= 2)
  - Per concept: platform adaptations (must match A03 assignments)
  - Overall: total variants (must be >= 30)

STEP 4: VERIFY minimum thresholds.
  - Hook swaps per concept: >= 5
  - CTA variants per concept: >= 2
  - Platform adaptations per concept: matches A03
  - Total variants: >= 30
  - IF ANY threshold not met: FLAG as GATE_2 blocker

STEP 5: PRODUCE the inventory table and summary statistics.
```
