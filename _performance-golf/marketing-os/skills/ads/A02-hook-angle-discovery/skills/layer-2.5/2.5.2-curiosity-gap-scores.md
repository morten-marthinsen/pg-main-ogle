# 2.5.2: Curiosity Gap

## Purpose
Score all hooks on Curiosity Gap dimension (1-10 scale). This is one of 6 scoring dimensions used to rank hooks before human curation.

## Inputs
- All generated hooks from Layer 2 (2.2-2.6 deduplicated)

## Process
1. Load all hooks
2. For each hook, score on Curiosity Gap (1-10)
3. Provide rationale for each score
4. Document scoring in structured format

## Output Schema
```markdown
# 2.5.2: Curiosity Gap
## Hook Scores

### Hook C-001
- **Text:** "[hook text]"
- **Curiosity Gap Score:** [1-10]
- **Rationale:** [why this score]

[Continue for all hooks...]

## Score Distribution
- 9-10: [count]
- 7-8: [count]
- 5-6: [count]
- 3-4: [count]
- 1-2: [count]

## Quality Metrics
- All hooks scored: [Y/N]
- All scores have rationale: [Y/N]
```

## Output File
- **Path:** `layer-2.5-outputs/2.5.2-curiosity-gap-scores.md`
- **Minimum size:** 3KB

## Quality Checks
- Every hook has score (1-10)
- Every score has rationale
- Scores use full range (not all 7-8)

## Dependencies
- 2.6-deduplicated-hooks.md
