# 3.3: File Manifest Builder

## Purpose

For EVERY variant in the matrix, build a concrete file manifest that maps the variant to its actual copy files and asset files on disk. Verify that EVERY referenced file EXISTS. A variant without a verified file manifest is a phantom variant -- it cannot be uploaded to any ad platform and must not appear in the final matrix.

## Inputs

- **VARIANT-ID-MAP.yaml:** From 3.1 -- all variant IDs with source file paths
- **COPY-CATALOG.yaml:** From 0.1 -- copy file paths
- **ASSET-CATALOG.yaml:** From 0.2 -- asset file paths with format specs
- **PLATFORM-SPECS.yaml:** From 0.3 -- platform format requirements
- **PLATFORM-MAPPING.yaml:** From 1.3 -- adaptation requirements for NEEDS_ADAPTATION combinations

## Process

```
STEP 1: LOAD ALL INPUTS
  - Read VARIANT-ID-MAP.yaml (all variants with component source paths)
  - Read COPY-CATALOG.yaml (verified copy file paths)
  - Read ASSET-CATALOG.yaml (verified asset file paths with format specs)
  - Read PLATFORM-SPECS.yaml (platform format requirements)
  - Read PLATFORM-MAPPING.yaml (adaptation notes)

STEP 2: FOR EACH VARIANT, BUILD FILE MANIFEST
  For each variant in VARIANT-ID-MAP.yaml:

  a. COPY FILE RESOLUTION
    - Locate the copy file for the hook component (from COPY-CATALOG.yaml)
    - Locate the copy file for the body component
    - Locate the copy file for the CTA component
    - Record exact file paths
    - IF any copy file has a platform-specific version for this variant's platform:
      use the platform-specific version

  b. ASSET FILE RESOLUTION
    - Locate the primary visual/video asset file (from ASSET-CATALOG.yaml)
    - Locate any secondary assets (audio, supplementary visuals)
    - Record exact file paths
    - Record format specs per asset:
      - Aspect ratio
      - Duration (seconds) or null for static
      - File type
      - File size (MB)
      - Resolution

  c. PLATFORM FORMAT SPECS
    - From PLATFORM-SPECS.yaml, record the format requirements for this variant's platform:
      - Required aspect ratio
      - Duration limit
      - File type requirement
      - Max file size
      - Sound handling
      - Text overlay requirements

  d. FORMAT COMPLIANCE CHECK
    - Compare asset format specs against platform format requirements
    - Aspect ratio: matches? needs crop? incompatible?
    - Duration: within limit? needs trim? exceeds?
    - File type: supported? needs conversion?
    - File size: within limit? needs compression?
    - Record compliance status per check

  e. ADAPTATION NOTES
    - From PLATFORM-MAPPING.yaml, attach any adaptation notes
    - What modifications are needed before deployment?

STEP 3: VERIFY ALL FILES EXIST ON DISK
  For EVERY file path in every manifest:
    - Check: does the file exist at the specified path?
    - IF YES: record verified = true, record file size
    - IF NO: record verified = false, mark variant as BLOCKED
    - Log: variant ID + missing file path + expected location

  CRITICAL: "Expected path" or "should be at" is NOT verification.
  The file either EXISTS or it DOES NOT. Verify each one.

STEP 4: ASSIGN ASSEMBLY STATUS
  For each variant:
    - READY: All files exist, all format specs match platform requirements
    - NEEDS_ADAPTATION: All files exist, but some format adaptations needed
      (record specific adaptations)
    - BLOCKED: One or more files missing, or irreconcilable format mismatch
      (record specific blocking issues)

STEP 5: COMPUTE MANIFEST STATISTICS
  - Total variants with manifests
  - Total READY (deploy without changes)
  - Total NEEDS_ADAPTATION (deploy after modifications)
  - Total BLOCKED (cannot deploy -- resolve missing files)
  - Total unique copy files referenced
  - Total unique asset files referenced
  - All files verified on disk: [Y/N]
  - Missing files: [count and list]

STEP 6: WRITE FILE-MANIFEST.yaml
```

## Output Schema

```yaml
# FILE-MANIFEST.yaml

metadata:
  timestamp: "[ISO 8601]"
  total_variants: "[integer]"
  total_ready: "[integer]"
  total_needs_adaptation: "[integer]"
  total_blocked: "[integer]"
  total_unique_copy_files: "[integer]"
  total_unique_asset_files: "[integer]"
  all_files_verified: "[true/false]"
  missing_files: ["[list or empty]"]

manifests:
  - variant_id: "[ID]"
    assembly_status: "[READY | NEEDS_ADAPTATION | BLOCKED]"
    copy_files:
      hook:
        path: "[exact file path]"
        verified: "[true/false]"
        file_size_kb: "[integer]"
        platform_specific: "[true/false]"
      body:
        path: "[exact file path]"
        verified: "[true/false]"
        file_size_kb: "[integer]"
        platform_specific: "[true/false]"
      cta:
        path: "[exact file path]"
        verified: "[true/false]"
        file_size_kb: "[integer]"
        platform_specific: "[true/false]"
    asset_files:
      primary:
        path: "[exact file path]"
        verified: "[true/false]"
        format:
          aspect_ratio: "[ratio]"
          duration_seconds: "[integer or null]"
          file_type: "[type]"
          file_size_mb: "[float]"
          resolution: "[widthxheight]"
      secondary:
        - path: "[exact file path]"
          verified: "[true/false]"
          type: "[audio | supplementary visual]"
          format: { ... }
    platform: "[PLATFORM]"
    platform_format_specs:
      required_aspect_ratio: "[ratio]"
      duration_limit: "[seconds]"
      file_type_required: "[type]"
      max_file_size_mb: "[integer]"
      sound_handling: "[sound_on | sound_off | both]"
      text_overlay: "[required | optional]"
    format_compliance:
      aspect_ratio: "[match | needs_crop | incompatible]"
      duration: "[within_limit | needs_trim | exceeds]"
      file_type: "[supported | needs_conversion | unsupported]"
      file_size: "[within_limit | needs_compression | exceeds]"
    adaptation_notes:
      - "[specific adaptation needed]"
    blocking_issues:
      - "[specific blocking issue]"

  # ... for EVERY variant
```

## Quality Checks

- [ ] Every variant has a complete file manifest (copy files + asset files)
- [ ] Every file path verified to exist on disk (not assumed)
- [ ] Copy file paths include hook, body, and CTA components
- [ ] Asset file paths include primary visual/video and any secondary assets
- [ ] Format specs recorded for every asset (aspect ratio, duration, file type, size)
- [ ] Platform format requirements recorded from PLATFORM-SPECS.yaml
- [ ] Format compliance check completed for every variant
- [ ] Assembly status assigned (READY / NEEDS_ADAPTATION / BLOCKED)
- [ ] Missing files logged with exact paths and variant IDs
- [ ] Adaptation notes attached from PLATFORM-MAPPING.yaml where applicable
- [ ] Zero phantom variants (no manifest references to non-existent files in READY variants)
