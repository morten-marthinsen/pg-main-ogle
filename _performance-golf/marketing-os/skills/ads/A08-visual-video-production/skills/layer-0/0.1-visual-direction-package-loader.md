# 0.1: Visual Direction Package Loader

## Purpose
Load A05 VISUAL-DIRECTION-PACKAGE.md containing shot lists, style specifications, mood boards, and reference ads per concept. This is the PRIMARY source of truth for what every asset should look like. All production decisions derive from this package.

## Inputs
- A05 VISUAL-DIRECTION-PACKAGE.md (from upstream A05 -- Visual Direction skill)
- Per-concept visual briefs (if structured as separate files per concept)

## Process
```
STEP 1: LOCATE VISUAL DIRECTION PACKAGE
  - Search project directory for VISUAL-DIRECTION-PACKAGE.md
  - IF not found: HALT -- cannot proceed without visual direction
  - Log file path and size

STEP 2: LOAD AND PARSE
  - READ the full visual direction package
  - FOR EACH winning concept:
    - EXTRACT shot list (every shot specified)
    - EXTRACT style specifications (photorealistic, cinematic, editorial, UGC, etc.)
    - EXTRACT mood/atmosphere direction
    - EXTRACT color palette
    - EXTRACT lighting direction
    - EXTRACT typography/text style specs
    - EXTRACT reference ads or mood boards
    - EXTRACT brand-specific visual guidelines
    - EXTRACT platform-specific visual direction (if different per platform)

STEP 3: VALIDATE COMPLETENESS
  - Every winning concept from A06 must have visual direction
  - Every required asset type must have specifications
  - Flag any concepts or asset types missing visual direction

STEP 4: WRITE OUTPUT
  - Create per-microskill output file with loaded content summary
  - Include file path, size, concept count, and key specifications
```
