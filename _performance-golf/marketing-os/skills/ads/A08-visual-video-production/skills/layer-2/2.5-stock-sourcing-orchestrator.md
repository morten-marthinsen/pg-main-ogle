# 2.5: Stock Sourcing Orchestrator

## Purpose
Search and source stock footage/images for supplementary assets. Only for assets where stock was explicitly assigned in Layer 1.2 tool assignment. Stock is the LAST resort -- it should only be used when AI generation was either unnecessary (simple supplementary B-roll) or failed after all attempts.

## Inputs
- 1.5 Production Schedule (which assets use stock sourcing)
- 1.2 Tool Assignment (assets assigned to stock sources)
- A05 Visual Direction (what the stock content must match)
- Stock library availability from 0.5 (Pexels, Unsplash, Envato, Artgrid)

## Process
```
FOR EACH asset assigned to stock sourcing in the production schedule:

STEP 1: COMPOSE SEARCH QUERY
  From A05 visual brief:
    - Subject/content description
    - Style/mood requirements
    - Color tone requirements
    - Duration requirements (video stock)
    - Technical specs (resolution, aspect ratio)

STEP 2: SEARCH STOCK LIBRARIES
  Priority order:
    1. Free sources (Pexels, Unsplash) -- no licensing cost
    2. Premium sources (Envato, Artgrid) -- if free sources insufficient
  Search with multiple query variations
  Collect 5-10 candidates per asset

STEP 3: EVALUATE CANDIDATES
  For each candidate:
    - Content match to brief: [1-10]
    - Style/mood match: [1-10]
    - Technical specs met (resolution, duration): [PASS/FAIL]
    - License compliance (commercial use allowed): [PASS/FAIL]
    - Generic-looking vs. specific (avoid cliche stock): [1-10]
  Select best candidate
  IF best score < 6.0: Try different search queries (up to 3 rounds)
  IF all rounds fail: Create human sourcing brief

STEP 4: DOWNLOAD AND LOG
  Download selected stock asset
  Rename per naming convention (mark as stock in manifest: source="stock")
  Log source URL, license type, search queries used
  Save license attribution data
```

## Output File
- **Path:** `layer-2-outputs/2.5-stock-sourcing.md`
- **Minimum size:** 5KB

## Quality Checks
- Stock sourcing was ONLY used for assets assigned to stock in 1.2 (not as a shortcut)
- Content matches the A05 brief (not just "close enough")
- All licenses verified for commercial use
- No generic/cliche stock imagery selected (rated on generic score)
- Source URLs and attribution data are logged
- Multiple search queries tried before flagging as failed

## Dependencies
- Layer 1 must be complete (GATE_1_COMPLETE.yaml exists)
- 1.5 Production Schedule must exist
- 1.2 Tool Assignment must show stock as assigned tool
- Stock library APIs must be available (from 0.5)
