# 3.4: Stock Footage Queries

## Purpose
For each B-roll shot that requires stock footage rather than AI generation: write specific search queries for stock footage platforms (Shutterstock, Artgrid, Pexels). Include: primary search term, secondary filters (orientation, duration, style), specific frame descriptions, usage rights requirements. Provide alternative query variations for each shot (stock search is inexact). Estimate footage needed per concept (total seconds of B-roll). These queries enable A08 to efficiently source supplementary footage.

## Inputs
- B-roll strategy from 2.5 (shot descriptions and stock queries drafted there)
- Shot lists from 2.1 (which shots need stock footage)
- Platform specs from 0.2 (aspect ratio / orientation requirements)
- Treatment types from 1.1 (affects footage style -- UGC stock vs polished stock)

## Process
1. READ B-roll strategy from 2.5
2. READ shot lists from 2.1
3. READ platform specs from 0.2
4. READ treatment types from 1.1
5. IDENTIFY all shots requiring stock footage:
   - B-roll shots that cannot be AI-generated (complex motion, specific environments)
   - Supplementary footage (lab footage, manufacturing, nature, lifestyle)
   - Any shot flagged as "source stock" in B-roll strategy
6. FOR EACH stock footage need:
   a. WRITE primary search query:
      - Main subject + action + style
      - Example: "close-up hands picking green tea leaves slow motion"
   b. SPECIFY secondary filters:
      - Orientation: vertical (9:16) / horizontal (16:9) / square (1:1)
      - Duration: minimum seconds needed
      - Style: realistic / cinematic / documentary / UGC-style
      - Color tone: warm / cool / neutral / specific palette match
      - Resolution: HD / 4K minimum
   c. WRITE 2-3 alternative queries:
      - Different keyword combinations for the same shot
      - Broader and narrower versions
      - Example alternatives:
        * "tea harvesting macro morning light"
        * "green tea plantation worker hands closeup"
        * "herbal ingredient sourcing authentic"
   d. DESCRIBE specific frame needed:
      - What the ideal footage looks like (beyond search terms)
      - What to AVOID in footage selection (wrong style, wrong mood)
   e. SPECIFY usage rights:
      - Commercial use required: YES
      - Exclusivity needed: [Y/N]
      - Platform restrictions: [any]
7. ESTIMATE total stock footage per concept:
   - Total seconds of stock B-roll needed
   - Number of unique clips needed
   - Budget consideration (free sources vs paid)
8. RECOMMEND sources:
   - Premium: Shutterstock, Artgrid, Storyblocks
   - Free: Pexels, Pixabay, Coverr
   - Specialized: [if niche-specific sources exist]
9. WRITE to output file

## Quality Checks
- Every stock footage need has primary + 2 alternative queries
- Queries are specific (not "happy people" or "healthy lifestyle")
- Filters include orientation matching platform aspect ratio
- "Ideal frame" description is specific enough to judge footage matches
- "Avoid" section prevents wrong-style footage selection
- Total footage estimates are realistic for ad lengths
