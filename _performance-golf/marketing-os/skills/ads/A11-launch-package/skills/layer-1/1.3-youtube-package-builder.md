# 1.3: YouTube Package Builder

## Purpose

Package all YouTube-targeted variants into the correct directory structure with correct file specifications, correct naming convention, and correct placement targeting. Organize files by ad type: Pre-roll (15-30s, 16:9), Bumper (6s max, 16:9), Skippable (any length, 16:9), and Shorts (9:16). Generate or verify thumbnail files (1280x720) exist for each video. Produce YOUTUBE-PACKAGE-MANIFEST.md listing every file with its complete specifications.

This is a MECHANICAL ASSEMBLY operation -- no creative decisions are made here.

## Inputs

- 0.2-variant-matrix-loader.md output (approved variants targeting YouTube, with component IDs)
- 0.3-production-assets-loader.md output (asset file paths and preliminary specs)
- 0.4-format-strategy-loader.md output (YouTube placements in scope: pre-roll, bumper, skippable, shorts)
- AGENT.md PLATFORM SPECIFICATION REFERENCE -- YouTube section (hard technical requirements)
- AGENT.md NAMING CONVENTION section (file naming and in-platform naming rules)

## Process

```
STEP 1: IDENTIFY YOUTUBE-TARGETED VARIANTS
  - Filter approved variant list for variants targeting YouTube
  - Group by ad type: pre-roll, bumper, skippable, shorts
  - IF no variants target YouTube: Output N/A file, skip remaining steps
  - Count: total variants, per-type breakdown

STEP 2: CREATE DIRECTORY STRUCTURE
  - Create LAUNCH-PACKAGE/youtube/pre-roll/
  - Create LAUNCH-PACKAGE/youtube/bumper/
  - Create LAUNCH-PACKAGE/youtube/skippable/
  - Create LAUNCH-PACKAGE/youtube/shorts/
  - Create LAUNCH-PACKAGE/youtube/thumbnails/

STEP 3: PROCESS EACH VARIANT
  FOR EACH YouTube-targeted approved variant:

    3a. LOCATE SOURCE ASSET
      - Find the production asset file(s) from A08 AD-ASSETS/
      - IF asset not found: FLAG as missing

    3b. VERIFY FILE SPECIFICATIONS
      VIDEO (Standard -- Pre-roll, Bumper, Skippable):
        - Codec: H.264 (recommended), VP9 also accepted
        - Container: MP4 recommended (MOV, AVI, WMV, FLV also accepted)
        - File size: Must be < 128GB (or 12 hours, whichever is less)
        - Aspect ratio: 16:9 (1920x1080)
        - Resolution: Min 1920x1080
        - Frame rate: 24-60fps
        - Audio: AAC, 48kHz
        - Duration constraints by type:
          - Pre-roll: 15-30 seconds
          - Bumper: 6 seconds maximum (hard limit)
          - Skippable: any length (5-second skip point)

      VIDEO (Shorts):
        - Same codec/container/audio requirements
        - Aspect ratio: 9:16 (1080x1920)
        - Duration: 60 seconds maximum

      THUMBNAIL:
        - Resolution: 1280x720
        - Format: JPG, PNG, GIF, or BMP
        - File size: Must be < 2MB
        - Aspect ratio: 16:9

      IF ANY SPEC FAILS: Flag file with specific failure reason
        -> DO NOT include non-compliant files in the package
        -> Return to A08/A09 for re-export at correct specs

    3c. VERIFY/FLAG THUMBNAILS
      - For each video variant: check if a corresponding thumbnail exists in A08 assets
      - IF thumbnail exists: verify specs (1280x720, <2MB, 16:9)
      - IF thumbnail does NOT exist: FLAG -- thumbnail needed before launch
        -> Thumbnails are REQUIRED for YouTube ads (they appear in search results,
           suggested videos, and as preview images)

    3d. APPLY FILE NAMING CONVENTION
      - Video: YOUTUBE-[CONCEPT]-[HOOK]-[BODY]-[CTA]-[VISUAL]-[TYPE].[ext]
      - Example: YOUTUBE-C1-HK01-BD01-CT03-VIS-A-PREROLL.mp4
      - Thumbnail: YOUTUBE-[CONCEPT]-[HOOK]-THUMB.[ext]
      - Example: YOUTUBE-C1-HK01-THUMB.jpg
      - Record mapping: original file name -> new file name

    3e. GENERATE IN-PLATFORM AD NAME
      - Format: [Date]-[Concept]-[HookType]-[BodyVersion]-[CTAType]-[Visual]-[Type]-[Notes]
      - Record mapping: file name -> in-platform name

    3f. COPY TO TARGET DIRECTORY
      - Pre-roll variants -> LAUNCH-PACKAGE/youtube/pre-roll/
      - Bumper variants -> LAUNCH-PACKAGE/youtube/bumper/
      - Skippable variants -> LAUNCH-PACKAGE/youtube/skippable/
      - Shorts variants -> LAUNCH-PACKAGE/youtube/shorts/
      - Thumbnails -> LAUNCH-PACKAGE/youtube/thumbnails/

    3g. RECORD PER-VARIANT SPEC SHEET
      - File name, original file name, format, codec, resolution,
        aspect ratio, duration, file size, audio format, ad type,
        thumbnail status, spec compliance status (PASS/FAIL per check)

STEP 4: GENERATE YOUTUBE-PACKAGE-MANIFEST.md
  - Create LAUNCH-PACKAGE/youtube/YOUTUBE-PACKAGE-MANIFEST.md
  - Include:
    - Total files packaged (video + thumbnails separately)
    - Per-type file counts (pre-roll, bumper, skippable, shorts)
    - Complete per-file spec sheet table
    - File naming cross-reference
    - In-platform naming cross-reference
    - Thumbnail status per variant (present/missing)
    - Duration compliance per type (bumper <= 6s, pre-roll 15-30s, etc.)
    - Any flagged files (spec failures, missing assets, missing thumbnails)

STEP 5: WRITE OUTPUT
  - Produce per-microskill output file at designated path
```
