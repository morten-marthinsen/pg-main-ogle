# 1.4: Google Display Package Builder

## Purpose

Package all Google Display Network-targeted variants into the correct directory structure with correct file specifications, correct naming convention, and correct size targeting. Organize files into responsive display assets (1200x628 landscape + 1200x1200 square + logos) and standard display sizes (728x90, 300x250, 300x600, 336x280, 160x600, 320x50, 320x480, 970x250). Verify file sizes meet strict limits (150KB standard, 5120KB responsive). Produce GOOGLE-PACKAGE-MANIFEST.md.

This is a MECHANICAL ASSEMBLY operation -- no creative decisions are made here.

## Inputs

- 0.2-variant-matrix-loader.md output (approved variants targeting Google Display, with component IDs)
- 0.3-production-assets-loader.md output (asset file paths and preliminary specs)
- 0.4-format-strategy-loader.md output (Google Display placements in scope)
- AGENT.md PLATFORM SPECIFICATION REFERENCE -- Google Display Network section
- AGENT.md NAMING CONVENTION section

## Process

```
STEP 1: IDENTIFY GOOGLE-DISPLAY-TARGETED VARIANTS
  - Filter approved variant list for variants targeting Google Display
  - Group by type: responsive display ads vs. standard display sizes
  - IF no variants target Google Display: Output N/A file, skip remaining steps
  - Count: total variants, per-type breakdown

STEP 2: CREATE DIRECTORY STRUCTURE
  - Create LAUNCH-PACKAGE/google/responsive/
  - Create LAUNCH-PACKAGE/google/standard/

STEP 3: PROCESS RESPONSIVE DISPLAY ADS
  FOR EACH variant designated for responsive display:

    3a. VERIFY REQUIRED RESPONSIVE ASSETS
      Required image set:
        - At least one 1200x628 landscape image
        - At least one 1200x1200 square image
      Required logo set:
        - 1200x1200 square logo
        - 1200x300 landscape logo (optional but recommended)
      Format: JPG or PNG (no animated GIF for responsive)
      Max file size: 5120KB per asset
      IF any required asset missing: FLAG

    3b. VERIFY FILE SPECIFICATIONS PER ASSET
      - Format check: JPG or PNG only
      - Resolution check: exact dimensions required
      - File size check: each asset < 5120KB
      IF ANY SPEC FAILS: Flag with specific reason

    3c. APPLY FILE NAMING CONVENTION
      - GOOGLE-[CONCEPT]-[HOOK]-[BODY]-[CTA]-[VISUAL]-RESP-[SIZE].[ext]
      - Example: GOOGLE-C3-HK12-BD01-CT01-VIS-C-RESP-1200x628.jpg
      - Record mapping: original -> new file name

    3d. COPY TO RESPONSIVE DIRECTORY
      - Place in LAUNCH-PACKAGE/google/responsive/

STEP 4: PROCESS STANDARD DISPLAY SIZES
  FOR EACH variant designated for standard display:

    4a. IDENTIFY TARGET SIZES
      Standard display sizes:
        - Leaderboard:       728x90
        - Medium Rectangle:  300x250
        - Half Page:         300x600
        - Large Rectangle:   336x280
        - Billboard:         970x250
        - Skyscraper:        160x600
        - Mobile Banner:     320x50
        - Mobile Interstitial: 320x480

    4b. VERIFY FILE SPECIFICATIONS PER ASSET
      - Format check: JPG, PNG, GIF (animated OK), or HTML5
      - Resolution check: exact dimensions for the target size
      - File size check: STRICT 150KB maximum per asset (standard display)
        -> This is the most commonly violated spec for Google Display
        -> 150KB is approximately 0.15MB -- very small
      IF file > 150KB: FLAG -- must be compressed or re-exported

    4c. APPLY FILE NAMING CONVENTION
      - GOOGLE-[CONCEPT]-[HOOK]-[BODY]-[CTA]-[VISUAL]-[SIZE].[ext]
      - Example: GOOGLE-C3-HK12-BD01-CT01-VIS-C-300x250.jpg
      - Record mapping: original -> new file name

    4d. COPY TO STANDARD DIRECTORY
      - Place in LAUNCH-PACKAGE/google/standard/

STEP 5: GENERATE GOOGLE-PACKAGE-MANIFEST.md
  - Create LAUNCH-PACKAGE/google/GOOGLE-PACKAGE-MANIFEST.md
  - Include:
    - Total files packaged (responsive + standard separately)
    - Responsive display asset inventory (per variant: landscape, square, logos)
    - Standard display size coverage (which sizes have assets, which are missing)
    - Complete per-file spec sheet table
    - File naming cross-reference
    - In-platform naming cross-reference
    - File size analysis (especially standard display -- % under 150KB)
    - Any flagged files (spec failures, oversized files, missing required assets)

STEP 6: WRITE OUTPUT
  - Produce per-microskill output file at designated path
```
