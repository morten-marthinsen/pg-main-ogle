# 0.3: Production Assets Loader

## Purpose

Load the A08 AD-ASSETS/ directory and inventory all production asset files. Verify existence of every asset referenced in the variant matrix, check file formats, check file sizes, check resolutions. Map each asset to its variant matrix entry. Flag missing or corrupted assets. This is the physical asset verification step -- ensuring that every approved variant actually has its production files ready before packaging begins.

## Inputs

- A08 AD-ASSETS/ directory (primary input -- contains all production asset files)
- 0.2-variant-matrix-loader.md output (provides the variant-to-asset mapping)

## Process

```
STEP 1: LOCATE AD-ASSETS DIRECTORY
  - Find A08 AD-ASSETS/ in the project outputs
  - IF directory does not exist: HALT -- cannot proceed without production assets
  - IF directory exists but is empty: HALT -- A08 must re-run

STEP 2: INVENTORY ALL FILES
  - Walk the entire AD-ASSETS/ directory tree
  - For EACH file found, record:
    - file_name: [string]
    - file_path: [full path]
    - file_format: [mp4, mov, jpg, png, gif, html5]
    - file_size_bytes: [integer]
    - file_size_human: [KB/MB/GB]
    - resolution: [WxH pixels] (if image or video)
    - aspect_ratio: [calculated from resolution -- e.g., 16:9, 9:16, 1:1, 4:5]
    - duration_seconds: [float, video only -- if determinable from metadata]
    - codec: [H.264, H.265, VP9, etc. -- video only, if determinable]
    - audio_format: [AAC, MP3, etc. -- video only, if determinable]

STEP 3: MAP ASSETS TO VARIANT MATRIX
  - Load variant-to-asset mapping from 0.2 output
  - For EACH approved variant:
    - Check: do all referenced asset files exist in AD-ASSETS/?
    - IF YES: Mark variant as "assets_complete"
    - IF NO: Mark variant as "assets_missing" with list of missing files
  - For EACH file in AD-ASSETS/:
    - Check: is this file referenced by at least one approved variant?
    - IF NO: Mark file as "orphaned" (exists but not used by any approved variant)

STEP 4: PRE-CHECK FILE SPECIFICATIONS
  - For each asset file, perform preliminary spec checks:
    - Video: Is it MP4 or MOV? Is file size reasonable (<4GB)?
    - Image: Is it JPG or PNG? Is file size reasonable (<30MB)?
    - Resolution: Is it at least 720px on shortest side?
  - These are PRELIMINARY checks -- full spec compliance happens in Layer 1 (1.6)
  - Flag obvious problems (e.g., 320x240 video, 50MB image, .avi file)

STEP 5: COMPILE ASSET INVENTORY
  - Total files: [count]
  - Files mapped to approved variants: [count]
  - Orphaned files: [count]
  - Missing files (referenced but not found): [count]
  - Files with obvious spec issues: [count]

STEP 6: DETERMINE CRITICAL MISSING ASSETS
  - critical_missing = files that are referenced by approved variants but do not exist
  - IF critical_missing > 0: This will BLOCK Gate 0
  - non_critical_missing = orphaned files or files for rejected variants
  - These do NOT block Gate 0

STEP 7: WRITE OUTPUT
  - Produce per-microskill output file at designated path
```

## Output File

- **Path:** `[project]/A11-launch-package/layer-0-outputs/0.3-production-assets-loader.md`

## Quality Checks

1. Every approved variant has its production assets verified (found or flagged missing)
2. Critical missing asset count is determined (blocks Gate 0 if > 0)
3. File inventory includes format, size, and resolution for every file
4. Preliminary spec concerns are flagged (not blocking, but informational for Layer 1)
5. Variant-to-asset mapping is complete with no unmatched references
6. Output file meets minimum 2KB size

## Dependencies

- A08 AD-ASSETS/ directory must exist
- 0.2-variant-matrix-loader.md must be complete (provides variant-to-asset mapping)
- Runs AFTER 0.2 (needs variant-to-asset mapping to verify assets)
