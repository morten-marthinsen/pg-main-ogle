# 2.7: Impression Scorer (Dual-Signal Performance)

**Version:** 1.0
**Layer:** 2 (Classification)
**Type:** Performance Scoring
**Dependencies:** Layer 1 complete (LAYER_1_COMPLETE.yaml exists), 2.5 (Run Duration Estimator)
**Model:** opus
**Mode:** Executes in Tool-Assisted Scan mode. Skipped when no impression data is available.

---

## Purpose

Compute the dual-signal performance score for ads that have impression data. This enriches the run-duration-only tier assignments from 2.5 with impression-weighted scoring. The dual-signal formula produces more accurate winner identification by combining run duration (40%) with impression volume (60%).

---

## Inputs

```yaml
input:
  from_2.5:
    run_duration_estimates: object  # All ads with tier assignments
  from_layer_1:
    all_scraped_ads: object  # Includes impression data for tool-assisted ads
  reference:
    scoring_protocol: "skills/ads/meta-ad-spy-integration.md"
      # Dual-signal formula: (norm_duration * 0.4) + (norm_impressions * 0.6)
```

---

## Process

### Step 1: Identify Impression-Validated Ads

```
FROM all scraped ads:
  PARTITION into:
    impression_validated: ads where impression_validated = true (from Meta Ad Spy)
    no_impression_data: ads where impression_validated = false (from Apify/Firecrawl)

LOG:
  "Impression-validated ads: [count] ([pct]% of total)"
  "Ads without impression data: [count] ([pct]% of total)"

IF impression_validated count = 0:
  LOG: "No impression data available. Skipping dual-signal scoring."
  OUTPUT: empty output file with skip reason
  EXIT
```

### Step 2: Normalize Signals (Per-Brand)

```
FOR EACH brand with impression-validated ads:

  CALCULATE per-brand normalization:
    run_duration:
      min_duration = min(run_duration_days) for this brand
      max_duration = max(run_duration_days) for this brand
      FOR EACH ad: normalized_duration = (duration - min) / (max - min)
        IF max = min: normalized_duration = 0.5 (all same duration)

    impressions:
      min_impressions = min(impressions) for this brand
      max_impressions = max(impressions) for this brand
      FOR EACH ad: normalized_impressions = (impressions - min) / (max - min)
        IF max = min: normalized_impressions = 0.5 (all same impressions)
```

### Step 3: Compute Dual-Signal Performance Score

```
FOR EACH impression-validated ad:

  performance_score = (normalized_duration * 0.4) + (normalized_impressions * 0.6)

  RECORD:
    ad_id: string
    brand_name: string
    run_duration_days: integer
    impressions: integer
    normalized_duration: float
    normalized_impressions: float
    performance_score: float  # 0.0 to 1.0
    original_tier: string  # from 2.5 (duration-only)
```

### Step 4: Reassign Tiers (Percentile-Based)

```
FOR impression-validated ads ONLY:

  SORT by performance_score descending

  ASSIGN new tiers:
    Top 20%: TIER_WINNER_DUAL
    21st-50th percentile: TIER_PERFORMER_DUAL
    51st-80th percentile: TIER_TESTING_DUAL
    Bottom 20%: TIER_NEW_DUAL

  COMPARE with original tiers (from 2.5):
    LOG tier changes: "Ad [id] moved from [original_tier] to [new_tier]"
    COUNT upgrades (moved to higher tier) and downgrades (moved to lower tier)

NOTE: Non-impression-validated ads RETAIN their original 2.5 tier assignments.
      Dual-signal tiers are additive, not replacement.
```

### Step 5: Generate Impression Benchmarks

```
CALCULATE vertical-level benchmarks:
  median_impressions: across all impression-validated ads
  p75_impressions: 75th percentile
  p90_impressions: 90th percentile
  median_performance_score: across all dual-signal scored ads
  p75_performance_score: 75th percentile
  p90_performance_score: 90th percentile

These benchmarks are passed to A10 (Pre-Launch Scoring) for competitive comparison.
```

## Quality Checks

1. **All impression-validated ads scored** — no ads with impression data left unscored.
2. **Per-brand normalization applied** — not global normalization (prevents spend-level bias).
3. **Formula applied correctly** — 0.4 x duration + 0.6 x impressions.
4. **Tier changes documented** — every tier upgrade/downgrade is logged.
5. **Non-impression ads untouched** — original 2.5 tiers retained for ads without impression data.
6. **Benchmarks computed** — median, p75, p90 for both impressions and performance scores.
7. **Edge cases handled** — single-ad brands (normalization = 0.5), zero-impression ads, same-value normalization.
