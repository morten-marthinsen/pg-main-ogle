# 2.2: Format Analyzer

**Version:** 1.0
**Layer:** 2 (Classification)
**Type:** Format Classification
**Dependencies:** Layer 1 complete (LAYER_1_COMPLETE.yaml exists)
**Model:** opus

---

## Purpose

Classify each scraped ad by format type (video, static image, carousel, stories/reels) with sub-classifications for video length. Calculate format distribution percentages per platform and overall. Identify format x platform patterns that reveal where specific formats dominate.

This data feeds Layer 3 format-platform intelligence (3.3) and opportunity gap analysis (3.5).

---

## Inputs

```yaml
input:
  from_layer_1:
    all_scraped_ads: object  # Combined output from 1.1, 1.2, 1.3, 1.4
    total_ad_count: integer  # From 1.5 validation
    platform_breakdown:
      meta_count: integer
      tiktok_count: integer
      google_count: integer
      other_count: integer
```

## Process

### Step 1: Define Format Taxonomy

```
PRIMARY FORMAT TYPES:
  - video (with sub-classification by length)
  - static_image (single image)
  - carousel (multi-image/video swipeable)
  - stories_reels (vertical, ephemeral format)
  - search_text (text-only search ads — Google primarily)
  - display_banner (standard display ad)

VIDEO LENGTH SUB-CLASSIFICATIONS:
  - 6s (bumper / ultra-short)
  - 15s (short-form)
  - 30s (standard short)
  - 60s (standard)
  - 2_3min (mid-length)
  - 3min_plus (long-form / VSL)

NOTE: Some ads may combine formats (e.g., carousel with video slides).
Classify by PRIMARY format — the dominant creative element.
```

### Step 2: Per-Ad Format Classification

```
FOR EACH scraped ad:

  1. DETERMINE primary format:
     - If format_type field exists from scraping: USE it (verify correctness)
     - If format_type is "unknown" or missing: INFER from available data
       - Has video_url → video
       - Has image_url only → static_image
       - Multiple images → carousel
       - Video length data → video with sub-classification

  2. FOR VIDEO ADS — determine length bucket:
     - If video_length_seconds available:
       → <= 6s: "6s"
       → 7-15s: "15s"
       → 16-30s: "30s"
       → 31-60s: "60s"
       → 61-180s: "2_3min"
       → > 180s: "3min_plus"
     - If video_length_seconds NOT available:
       → Classify as "video_length_unknown"

  3. RECORD:
     ad_format_classification:
       ad_id: string
       brand_name: string
       platform: string
       primary_format: string
       video_length_bucket: string  # null if not video
       video_length_seconds: integer  # null if not video or unknown
       format_confidence: string  # "confirmed" if from scraper, "inferred" if deduced

  4. INCREMENT format counters
```

### Step 3: Platform Distribution Calculation

```
FOR EACH platform (meta, tiktok, google, other):
  CALCULATE:
    - Count per format type
    - Percentage per format type
    - For video: length distribution within video ads

  PRODUCE platform_format_table:
    platform: string
    total_ads: integer
    format_distribution:
      - format: string
        count: integer
        pct: float
    video_length_distribution:  # only for video ads on this platform
      - bucket: string
        count: integer
        pct_of_video: float

CALCULATE overall_distribution (all platforms combined):
  Same structure as platform-specific, but aggregated
```

### Step 4: Pattern Identification

```
IDENTIFY format x platform patterns:
  - Which formats dominate on which platforms?
    e.g., "TikTok: 92% video, 8% carousel — virtually no static"
    e.g., "Meta: 55% video, 30% static, 15% carousel"
    e.g., "Google: 60% search_text, 25% display, 15% video"

  - Which video lengths dominate on which platforms?
    e.g., "TikTok: 85% under 30s, 15% 30-60s"
    e.g., "Meta: 40% under 30s, 35% 30-60s, 25% over 60s"

  - Which brands have distinctive format strategies?
    e.g., "Brand X uses only video (100%), Brand Y uses 50% carousel"

RECORD patterns as structured observations.
```

## Output Schema

```yaml
output:
  format_classification:
    total_ads_classified: integer  # MUST equal total scraped
    classification_completeness_pct: 100

    per_ad_classifications:
      - ad_id: string
        brand_name: string
        platform: string
        primary_format: string
        video_length_bucket: string
        video_length_seconds: integer
        format_confidence: string

    overall_distribution:
      - format: string
        count: integer
        pct: float

    platform_distributions:
      - platform: string
        total_ads: integer
        format_breakdown:
          - format: string
            count: integer
            pct: float
        video_length_breakdown:
          - bucket: string
            count: integer
            pct_of_video: float

    brand_format_profiles:
      - brand_name: string
        total_ads: integer
        dominant_format: string
        format_diversity_score: float  # 1.0 = all one format, 5.0 = evenly distributed

    patterns:
      - pattern_id: string
        observation: string  # e.g., "TikTok is 92% video under 30s"
        evidence: string
        implication: string  # what this means for our campaign
```

## Quality Checks

1. **100% classification coverage** -- every scraped ad has a format classification.
2. **All platforms have distribution tables** -- no platform skipped.
3. **Video length sub-classification complete** -- all video ads have length buckets (or explicit "unknown" with count).
4. **Platform patterns identified** -- at least 3 substantive pattern observations per platform.
5. **Brand profiles include diversity scores** -- brands using only 1 format vs. multiple formats are distinguished.
6. **Percentages sum to ~100%** -- rounding may cause slight variance, but should be within 100 +/- 1%.
7. **No "unknown" overuse** -- if more than 20% of ads are "unknown" format, investigate data quality.
