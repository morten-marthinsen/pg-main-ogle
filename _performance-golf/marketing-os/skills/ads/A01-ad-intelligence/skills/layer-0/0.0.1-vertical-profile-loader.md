# 0.0.1: Vertical Profile Loader

**Version:** 1.0
**Layer:** 0 (Foundation)
**Type:** Configuration Loading
**Dependencies:** None (first microskill in execution)
**Output:** vertical_profile.yaml

---

## Purpose

Load ad-specific vertical configuration from `ad-verticals/[vertical].md`. This microskill extracts platform priorities, hook patterns, compliance constraints, and anti-slop rules specific to the campaign's vertical (golf, health, finance, personal-dev, technology). This configuration shapes all downstream scraping, classification, and analysis decisions.

---

## Inputs

```yaml
input:
  from_campaign_brief:
    vertical: string  # One of: golf, health, finance, personal-dev, technology
  from_ad_verticals_directory:
    vertical_profile_path: "skills/ads/ad-verticals/[vertical].md"
```

---

## Process

### Step 1: Identify Vertical

```
READ Campaign Brief (Skill 09 output) for vertical identifier
IF vertical not specified:
  → DEFAULT to "universal" (no vertical-specific config)
  → LOG warning: "No vertical specified — using universal configuration"
```

### Step 2: Load Vertical Profile

```
IF vertical specified:
  CONSTRUCT path: skills/ads/ad-verticals/[vertical].md
  READ vertical profile file
  EXTRACT:
    - platform_priorities (which platforms to prioritize for this vertical)
    - hook_patterns (which of the 32 hook types perform best in this vertical)
    - compliance_constraints (regulatory/platform-specific restrictions)
    - anti_slop_rules (vertical-specific language to avoid)
    - format_preferences (video vs static vs carousel by platform)
    - visual_style_guidelines (UGC vs polished for this vertical)
```

### Step 3: Normalize and Structure

```
COMPILE vertical_profile:
  vertical_id: "[vertical identifier]"
  platform_priorities:
    primary: "[platform]"
    secondary: "[platform]"
    tertiary: "[platform]"
  hook_performance_data:
    high_performing_types: [list of hook types that work well]
    underperforming_types: [list of hook types that don't work well]
  compliance_constraints:
    forbidden_claims: [list of claims that violate platform/regulatory rules]
    required_disclaimers: [list if applicable]
  anti_slop:
    forbidden_phrases: [vertical-specific slop to avoid]
    forbidden_language_patterns: [e.g., health: "cure" "treat", finance: "guaranteed returns"]
  format_guidance:
    recommended_formats_by_platform: [map]
    video_length_preferences: [map]
  visual_style_guidance:
    recommended_styles: [list]
    avoid_styles: [list]
```

### Step 4: Set Downstream Configuration

```
CONFIGURE downstream microskills:
  - Layer 1 scrapers will prioritize platforms per platform_priorities
  - Layer 2 classification will flag compliance violations per compliance_constraints
  - Layer 3 opportunity gap analysis will reference hook_performance_data
  - Layer 4 handoff will include vertical-specific recommendations
```

## Quality Checks

1. If vertical specified, profile file was successfully loaded
2. If vertical not specified, universal fallback is documented
3. All required sections populated (platform_priorities, hook_performance_data, compliance_constraints, anti_slop, format_guidance, visual_style_guidance)
4. Platform priorities are specific (not "varies" or "TBD")
5. Hook performance data includes at least 3 high-performing types
6. Compliance constraints are actionable (specific forbidden claims, not vague warnings)
