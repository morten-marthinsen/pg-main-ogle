# 0.5: JSON Import Loader (Meta Ad Spy)

**Version:** 1.0
**Layer:** 0 (Foundation)
**Type:** Data Import
**Dependencies:** 0.1 (Campaign Brief Loader)
**Model:** haiku
**Mode:** Tool-Assisted Scan ONLY (skip in Initial Scan and Continuous Monitor)

---

## Purpose

Parse the Meta Ad Spy tool's JSON export into A01's internal ad data format. Validates JSON structure, maps tool fields to A01 fields, and produces a normalized dataset ready for brand matching (0.6) and ingestion (1.6). This microskill executes ONLY in Tool-Assisted Scan mode.

---

## Inputs

```yaml
input:
  from_tool:
    json_export_path: string  # Path to Meta Ad Spy JSON export file
    # OR
    json_data: object  # Direct JSON data if provided inline
  reference:
    import_schema: "skills/ads/meta-ad-spy-integration.md"
      # Canonical field mapping: tool fields → A01 fields
```

---

## Process

### Step 1: Load and Validate JSON

```
LOAD JSON from export_path or inline data

VALIDATE structure:
  - JSON is valid (parseable)
  - Contains array of ad objects
  - Each ad object has REQUIRED fields:
    - brand_name (string, non-empty)
    - ad_text (string, non-empty)
    - headline (string)
    - impressions (integer, >= 0)
    - first_seen (date string)
    - last_seen (date string)

LOG:
  "Total ads in JSON: [count]"
  "Ads with valid structure: [count]"
  "Ads with invalid/missing fields: [count] — [list field names]"

IF invalid ads > 10% of total:
  WARNING: "Over 10% of ads have invalid structure. Review JSON export."
```

### Step 2: Field Mapping

```
FOR EACH valid ad in JSON:

  MAP tool fields → A01 internal fields:
    brand_name → brand_name (direct)
    ad_text → full_body_copy (direct)
    headline → headline (direct)
    cta_text → cta_text (direct)
    asset_type → format_raw (direct — mapped to format_code in Layer 2)
    visual_format → visual_format_raw (direct — classified in Layer 2)
    messaging_angle → messaging_angle (direct — informational, not used for classification)
    hook_tactic → hook_tactic_raw (direct — mapped to hook_type_code in Layer 2)
    offer_type → offer_type (direct)
    impressions → impressions (direct — integer)
    first_seen → first_seen (direct — ISO 8601)
    last_seen → last_seen (direct — ISO 8601)

  COMPUTE derived fields:
    run_duration_days = last_seen - first_seen (in days)
    copy_length_bucket = categorize(word_count(ad_text)):
      micro: < 25 words
      short: 25-75 words
      medium: 76-150 words
      long: 150+ words
    data_source = "meta_ad_spy" (fixed)
    impression_validated = true (fixed)
    platform = "meta" (fixed — Meta Ad Spy only covers Meta)

  ASSIGN unique ad_id: MAS-[sequential 4-digit number]
```

### Step 3: Produce Normalized Dataset

```
OUTPUT normalized ad dataset:
  - All tool fields mapped to A01 internal schema
  - All derived fields computed
  - Ready for brand matching (0.6) and ingestion (1.6)

LOG:
  "Total ads normalized: [count]"
  "Brands represented: [count] — [list]"
  "Date range: [earliest first_seen] to [latest last_seen]"
  "Impression range: [min] to [max]"
```

## Quality Checks

1. **All ads parsed** — no silently dropped records. Invalid ads logged with specific reasons.
2. **All required fields mapped** — every tool field has a corresponding A01 field.
3. **Derived fields computed** — run_duration_days, copy_length_bucket, data_source all populated.
4. **Unique ad_ids assigned** — no duplicates, sequential MAS-NNNN format.
5. **Date fields are valid ISO 8601** — first_seen and last_seen parseable as dates.
6. **Impressions are non-negative integers** — no negative values, no nulls passed through.
