# 0.6: Brand Database Matcher (Meta Ad Spy)

**Version:** 1.0
**Layer:** 0 (Foundation)
**Type:** Brand Matching
**Dependencies:** 0.3 (Competitor List Builder), 0.5 (JSON Import Loader)
**Model:** haiku
**Mode:** Tool-Assisted Scan ONLY

---

## Purpose

Match brands in the Meta Ad Spy import against the competitor list (0.3) and the vertical config's `meta_ad_spy_brands` section. Determine which competitor brands have tool-assisted data available (skip Apify scrape for their Meta ads) and which need standard scraping. Produces a routing table that Layer 1 uses to decide scraping strategy per brand.

---

## Inputs

```yaml
input:
  from_0.5:
    normalized_ads: object  # JSON Import Loader output
    brands_in_import: [string]  # Distinct brand names from import
  from_0.3:
    competitor_list: object  # Competitor List Builder output
  from_0.0.1:
    vertical_config: object  # Includes meta_ad_spy_brands section
```

---

## Process

### Step 1: Match Imported Brands to Competitor List

```
FOR EACH brand in competitor_list:
  SEARCH for match in normalized_ads.brands_in_import:
    - Exact name match (case-insensitive)
    - Fuzzy match (handle "Gundry MD" vs "Dr. Gundry" etc.)

  IF match found:
    brand.tool_assisted = true
    brand.meta_ad_spy_ad_count = count of ads from this brand in import
    brand.meta_scrape_needed = false  # Skip Apify for Meta
    brand.other_platform_scrape_needed = true  # Still need TikTok/Google
  ELSE:
    brand.tool_assisted = false
    brand.meta_scrape_needed = true  # Standard Apify scraping
    brand.other_platform_scrape_needed = true

LOG:
  "Brands with tool-assisted data: [count] / [total competitors]"
  "Brands needing standard Meta scraping: [count]"
  "Tool-assisted ads available: [total from matched brands]"
```

### Step 2: Identify Unmatched Import Brands

```
FOR EACH brand in normalized_ads.brands_in_import:
  IF NOT in competitor_list:
    LOG: "Brand '[name]' in Meta Ad Spy import but not in competitor list"
    ASSESS: Is this brand relevant to the vertical?
    IF relevant:
      RECOMMEND: "Consider adding [name] to competitor list"
    ELSE:
      TAG: vertical = "unclassified"
```

### Step 3: Produce Routing Table

```
CREATE brand_routing_table:
  FOR EACH competitor:
    - brand_name
    - tool_assisted: boolean
    - meta_scraping_source: "meta_ad_spy" | "apify"
    - tiktok_scraping_source: "apify" | "firecrawl"  # always standard
    - google_scraping_source: "firecrawl"  # always standard
    - meta_ad_spy_ad_count: integer (0 if not tool-assisted)

This routing table is consumed by Layer 1 microskills to determine scraping strategy.
```

---

## Output Schema

```yaml
output:
  brand_matching:
    timestamp: string
    total_competitors: integer
    tool_assisted_count: integer
    standard_scraping_count: integer
    total_tool_assisted_ads: integer

    routing_table:
      - brand_name: string
        tool_assisted: boolean
        meta_scraping_source: enum[meta_ad_spy, apify]
        tiktok_scraping_source: enum[apify, firecrawl]
        google_scraping_source: enum[firecrawl]
        meta_ad_spy_ad_count: integer

    unmatched_import_brands:
      - brand_name: string
        relevant_to_vertical: boolean
        recommendation: string

    matching_quality:
      exact_matches: integer
      fuzzy_matches: integer
      no_matches: integer
```

---

## Quality Checks

1. **All competitors in routing table** — every brand from 0.3 has a routing entry.
2. **Tool-assisted brands have ad counts** — meta_ad_spy_ad_count > 0 for all tool_assisted = true brands.
3. **Unmatched brands logged** — import brands not in competitor list are documented.
4. **Non-Meta platforms always standard** — tiktok and google sources are never "meta_ad_spy".
5. **Fuzzy matches documented** — if fuzzy matching was used, the match pair is logged.
