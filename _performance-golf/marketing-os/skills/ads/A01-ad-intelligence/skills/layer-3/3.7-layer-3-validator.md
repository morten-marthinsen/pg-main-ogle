# 3.7: Layer 3 Validator

**Version:** 1.0
**Layer:** 3 (Intelligence Synthesis)
**Type:** Validation Gate
**Dependencies:** 3.1, 3.2, 3.3, 3.4, 3.5 complete; 3.6 complete if Continuous Monitor mode
**Model:** opus

---

## Purpose

Validate that ALL Layer 3 synthesis outputs are complete, substantive, and meet quality thresholds before allowing progression to Layer 4 (Output Packaging). This is the Gate 3 enforcement mechanism. It verifies competitive landscape coverage, hook distribution completeness, winning specimen quality, and opportunity gap actionability.

---

## Inputs

```yaml
input:
  mode: string  # "INITIAL_SCAN" or "CONTINUOUS_MONITOR"
  from_layer_3:
    competitive_landscape: "layer-3-outputs/3.1-competitive-landscape-map.md"
    hook_distribution: "layer-3-outputs/3.2-hook-type-distribution.md"
    format_platform_intel: "layer-3-outputs/3.3-format-platform-intelligence.md"
    winning_specimens: "layer-3-outputs/3.4-winning-ad-specimens.md"
    opportunity_gaps: "layer-3-outputs/3.5-opportunity-gap-analysis.md"
    trend_detection: "layer-3-outputs/3.6-trend-detection.md"  # may be stub if Initial Scan
  from_layer_0:
    competitor_list_count: integer  # from 0.3
```

## Process

### Step 1: Read All Layer 3 Outputs

```
READ each Layer 3 output file.
VERIFY file exists and is non-empty.
EXTRACT key metrics from each.
```

### Step 2: Competitive Landscape Validation (3.1)

```
CHECK:
  brands_profiled: integer  # from 3.1
  brands_required: integer  # from competitor list (10+)
  all_brands_covered: boolean  # brands_profiled >= brands_required

  top_3_leaders_identified: boolean
  market_position_for_all: boolean  # every brand has a position assignment
  competitive_intensity_calculated: boolean

  IF brands_profiled < 10:
    FAIL: "Competitive landscape covers [X] brands. Minimum: 10."
  IF any brand lacks position:
    FAIL: "Brand [X] has no market position assignment."
```

### Step 3: Hook Distribution Validation (3.2)

```
CHECK:
  types_in_distribution: integer  # should be 32
  all_32_types_covered: boolean
  saturation_assessments_present: boolean
  top_5_prevalent_identified: boolean
  bottom_10_underused_identified: boolean
  competitor_diversity_calculated: boolean

  IF types_in_distribution < 32:
    FAIL: "Hook distribution covers [X] types. All 32 required (even if 0%)."
  IF saturation_assessments missing:
    FAIL: "Missing saturation assessments for hook types."
```

### Step 4: Format-Platform Intelligence Validation (3.3)

```
CHECK:
  platforms_analyzed: integer
  all_platforms_have_format_breakdown: boolean
  winning_combinations_identified: boolean
  compliance_constraints_documented: boolean
  recommendations_per_platform: boolean

  IF any platform lacks format breakdown:
    FAIL: "Platform [X] missing format distribution analysis."
  IF winning combinations < 3 per platform:
    FAIL: "Fewer than 3 winning combinations identified for platform [X]."
```

### Step 5: Winning Specimens Validation (3.4)

```
CHECK:
  specimens_extracted: integer  # minimum 20
  all_have_verbatim_copy: boolean
  all_have_hook_classification: boolean
  all_have_analysis: boolean
  diversity_targets_met: boolean
  file_size_kb: float  # should be >= 30KB

  FOR EACH specimen:
    has_headline: boolean
    has_body_copy: boolean  # must be FULL text, not truncated
    has_cta: boolean
    has_hook_type: boolean
    has_why_winner: boolean

  IF specimens_extracted < 20:
    FAIL: "Only [X] winning specimens extracted. Minimum: 20."
  IF any specimen lacks verbatim copy:
    FAIL: "Specimen [WS-XXX] is missing verbatim copy transcription."
  IF file_size_kb < 30:
    FAIL: "Winning specimens file is [X]KB. Minimum: 30KB.
           20 full verbatim specimens should be 30-50KB."
```

### Step 6: Opportunity Gap Validation (3.5)

```
CHECK:
  total_gaps_identified: integer  # minimum 5
  all_gaps_have_evidence: boolean
  all_gaps_have_impact_rating: boolean
  all_gaps_have_recommended_action: boolean
  gaps_are_ranked: boolean
  gap_types_covered: [string]  # should include at least hook + format

  FOR EACH gap:
    has_specific_evidence: boolean  # not vague — cites data
    has_actionable_recommendation: boolean  # not "consider this" — specific action
    impact_rating_is_justified: boolean

  IF total_gaps_identified < 5:
    FAIL: "Only [X] opportunity gaps identified. Minimum: 5."
  IF any gap lacks evidence:
    FAIL: "Gap [OG-XXX] has no specific evidence."
  IF any gap lacks actionable recommendation:
    FAIL: "Gap [OG-XXX] recommendation is vague — must be specific and actionable."
```

### Step 7: Trend Detection Validation (3.6)

```
IF mode == "CONTINUOUS_MONITOR":
  CHECK:
    trend_detection_executed: boolean
    previous_baseline_loaded: boolean
    shift_analysis_complete: boolean
    emerging_patterns_identified: boolean

    IF trend_detection not executed:
      FAIL: "Trend detection is MANDATORY in Continuous Monitor mode."
    IF previous baseline not loaded:
      FAIL: "Trend detection did not load previous handoff for comparison."

IF mode == "INITIAL_SCAN":
  CHECK:
    trend_detection_stub_exists: boolean
    IF stub missing:
      WARN: "3.6 output file missing. Create stub documenting skip reason."
```

### Step 8: Gate Decision

```
IF ALL checks pass:
  GATE_3 = PASS
  CREATE LAYER_3_COMPLETE.yaml (see output schema)
  PROCEED to Layer 4

IF ANY check fails:
  GATE_3 = FAIL
  DOCUMENT all failures with specific remediation instructions
  DO NOT proceed to Layer 4
  RETURN to the failing microskill(s)
```

## Quality Checks

1. **All 6 synthesis outputs checked** -- 3.1 through 3.6 (3.6 may be stub in Initial Scan).
2. **Quantitative thresholds enforced** -- 10+ brands, 32 types, 20+ specimens, 5+ gaps.
3. **Quality checks go beyond existence** -- specimens verified for verbatim completeness, gaps verified for actionability.
4. **File size check on specimens** -- 30KB minimum enforced for 3.4 output.
5. **Gate decision is BINARY** -- PASS or FAIL. No partial pass.
6. **Specific remediation instructions** -- if FAIL, exactly which microskill to fix and what to address.
7. **Mode-aware validation** -- trend detection checked only in Continuous Monitor mode.

---

## Dependencies

- All Layer 3 microskills (3.1-3.6) have output files
- Competitor list count from Layer 0 (for brand coverage verification)
- Mode information (Initial Scan vs. Continuous Monitor)
