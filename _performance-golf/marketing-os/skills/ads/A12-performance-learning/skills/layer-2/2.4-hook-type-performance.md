# 2.4: Hook Type Performance

**Version:** 1.0
**Layer:** 2 (Prediction vs Reality Analysis)
**Type:** Analysis
**Model:** opus
**Dependencies:** Layer 1 outputs, 0.2 variant-matrix-loader, 0.4 benchmark-loader
**Output:** layer-2-outputs/2.4-hook-type-performance.md

---

## Purpose

Map actual performance data to hook types from the AD-HOOK-TAXONOMY.md. For each hook type used in the campaign: calculate average hook rate, average CPA, average ROAS, and winner/loser ratio. Compare against taxonomy benchmark data. Identify hook types that outperformed benchmarks, underperformed, or had insufficient data. Calculate performance variance by hook category (A-J). This feeds both Layer 3.1 (Hook Learnings) and Layer 3.5.2 (A02 Hook Scoring Update).

## Process

### Step 1: Map Variants to Hook Types

```
FOR EACH variant with sufficient data:
  LOOKUP hook_type from variant matrix (0.2 output)
  BUILD hook_type --> [variant_ids] mapping

  RESULT:
    hook_type_A2 (Curiosity Gap): [V-001, V-005, V-012, ...]
    hook_type_D3 (UGC Testimonial): [V-003, V-007, V-015, ...]
    hook_type_B1 (Pain Point Amplifier): [V-002, V-008, ...]
    ... etc

IF hook_type mapping unavailable for some variants:
  --> GROUP as "UNMAPPED" -- analyze separately but do not attribute to a hook type
```

### Step 2: Calculate Per-Hook-Type Performance

```
FOR EACH hook type with at least 1 sufficient-data variant:

  AGGREGATE:
    variant_count: count of variants with this hook type
    avg_hook_rate: mean(hook_rate of all variants with this type)
    avg_CPA: weighted_mean(CPA, weights=spend) -- spend-weighted average
    avg_ROAS: weighted_mean(ROAS, weights=spend)
    avg_CTR: mean(CTR of all variants)
    total_spend: sum of spend for all variants with this type
    total_conversions: sum of conversions

  CLASSIFICATION DISTRIBUTION:
    winners: count(WINNER variants with this hook type)
    performers: count(PERFORMER variants)
    losers: count(LOSER variants)
    win_rate: winners / (winners + performers + losers) * 100

  STATISTICAL NOTE:
    IF variant_count < 3: "Limited sample -- treat results as directional only"
    IF variant_count >= 5: "Sufficient sample for moderate confidence"
    IF variant_count >= 10: "Strong sample for high confidence conclusions"
```

### Step 3: Compare Against Taxonomy Benchmarks

```
FOR EACH hook type analyzed:
  LOOKUP taxonomy benchmark:
    benchmark_hook_rate: float (from AD-HOOK-TAXONOMY.md)
    benchmark_engagement: float (if available)

  CALCULATE:
    hook_rate_vs_benchmark = actual_avg_hook_rate / benchmark_hook_rate
    performance_vs_benchmark: enum[
      OUTPERFORMED (> 120% of benchmark),
      AT_BENCHMARK (80-120% of benchmark),
      UNDERPERFORMED (< 80% of benchmark)
    ]

  PRODUCE comparison record:
    hook_type: [type]
    actual_avg_hook_rate: [X]%
    taxonomy_benchmark: [Y]%
    ratio: [Z]x
    status: OUTPERFORMED / AT_BENCHMARK / UNDERPERFORMED
    hypothesis: [brief explanation of why -- feeds Layer 3.1]
```

### Step 4: Calculate Hook Category (A-J) Performance

```
AGGREGATE hook type results by CATEGORY (A-J from taxonomy):

  FOR EACH category:
    category_avg_hook_rate: mean of all hook types in this category
    category_avg_CPA: weighted average across all hook types in category
    category_win_rate: total_winners / total_classified in category
    category_variant_count: total variants across all types in category

  RANK categories by performance:
    By CPA: [best to worst category]
    By hook_rate: [best to worst category]
    By win_rate: [best to worst category]
```

### Step 5: Hook Type x Platform Cross-Tabulation

```
IF variants span multiple platforms:
  BUILD cross-tabulation:
    | Hook Type | Meta Hook Rate | Meta CPA | TikTok Hook Rate | TikTok CPA | Google CTR | Google CPA |
    |-----------|---------------|----------|-----------------|-----------|------------|-----------|
    | [type]    | [X]%          | $[X]     | [X]%            | $[X]      | [X]%       | $[X]      |

  IDENTIFY:
    - Hook types that perform well on ALL platforms (universal winners)
    - Hook types that perform well on one platform but not others (platform-specific)
    - Hook types that underperform everywhere (universal losers for this campaign)
```

### Step 6: Produce Hook Type Performance Table

```
PRODUCE:

| Hook Type | ID | Cat | Variants | Avg Hook | Avg CPA | Avg ROAS | Win Rate | vs Bench | Confidence |
|-----------|-----|-----|----------|----------|---------|----------|----------|----------|------------|
| [name]    | [X] | [X] | [X]     | [X]%     | $[X]    | [X]x     | [X]%     | [status] | [H/M/L]    |

SORT by avg_CPA ascending (best performing hook types first)
```

## Quality Checks

1. All hook types used in the campaign are analyzed (none skipped)
2. Metrics are spend-weighted where appropriate (CPA, ROAS)
3. Benchmark comparison uses AD-HOOK-TAXONOMY.md data
4. Confidence levels assigned based on sample size
5. Category-level aggregation produces meaningful ranking
6. Platform cross-tabulation completed (if multi-platform)
7. Top and bottom performers identified with clear rationale
8. Statistical limitations noted for small-sample hook types
