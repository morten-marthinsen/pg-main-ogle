# 3.5: Prediction Learnings

**Version:** 1.0
**Layer:** 3 (Learning Extraction)
**Type:** Learning Extraction
**Model:** opus
**Dependencies:** 2.1 prediction-comparison, 2.2 divergence-analysis, 2.3 scoring-model-audit
**Output:** layer-3-outputs/3.5-prediction-learnings.md

---

## Purpose

Extract actionable learnings about prediction accuracy. What systematic biases existed in A10's scoring? (e.g., consistently overestimating video vs static? Consistently underestimating UGC?) Which scoring criteria should have higher weights? Lower weights? What new predictive signals emerged from the data that A10 doesn't currently model? Generate specific scoring model adjustments with magnitude (e.g., "Increase UGC bonus from +0.5 to +1.2 based on 2.4x outperformance"). This directly feeds Layer 3.5.4 (A10 Prediction Model Update).

---

```yaml
input:
  from_2.1:

  from_2.3:

```

## Process

### Step 1: Synthesize Prediction Accuracy Findings

```
FROM 2.1, 2.2, 2.3 outputs:

  SUMMARIZE:
    overall_accuracy: [X]% of predictions were ACCURATE
    systematic_bias: OPTIMISTIC / PESSIMISTIC / BALANCED
    strongest_predictive_criterion: [name] (r=[X])
    weakest_predictive_criterion: [name] (r=[X])
    most_common_divergence_pattern: [pattern from 2.2]
    blind_spots_identified: [from 2.2]
```

### Step 2: Identify Systematic Scoring Biases

```
ANALYZE patterns across all divergent predictions:

  FORMAT BIAS:
    Does A10 systematically over/undervalue certain ad formats?
    e.g., "A10 scored video variants 0.8 points higher on average than static,
    but actual performance difference was only 0.2 points"
    BIAS MAGNITUDE: [X] points overestimation for video

  VISUAL STYLE BIAS:
    Does A10 systematically over/undervalue certain visual styles?
    e.g., "A10 scored polished variants 0.5 points higher than UGC,
    but UGC outperformed polished by 1.4x on CPA"
    BIAS MAGNITUDE: [X] points overestimation for polished

  HOOK TYPE BIAS:
    Does A10 systematically over/undervalue certain hook types?
    e.g., "A10 rated authority hooks highest but UGC testimonial hooks
    actually converted better"

  PLATFORM BIAS:
    Does A10 predict differently for different platforms?
    e.g., "A10 predictions were 65% accurate on Meta but only 35% on TikTok"

  FOR EACH BIAS IDENTIFIED:
    bias_type: [format/visual/hook/platform]
    direction: [overestimates/underestimates]
    magnitude: [quantified]
    evidence: [specific data]
    recommended_correction: [specific adjustment]
```

### Step 3: Identify Missing Predictive Signals

```
ANALYZE: What ACTUAL performance predictors exist in the data that
A10 doesn't currently model?

  POSSIBLE MISSING SIGNALS:
    1. Hook length (seconds to key claim) -- does this predict hook rate?
    2. Text overlay presence -- does this predict performance on sound-off platforms?
    3. Face presence in thumbnail -- does this predict hook rate?
    4. Specific timeframe claims -- does this predict conversion rate?
    5. Number of proof points in body -- does this predict conversion?
    6. CTA placement timing (early vs late) -- does this predict CTR?
    7. Music/sound design quality -- does this predict platform performance?

  FOR EACH potential signal:
    IF data available to test:
      Calculate correlation with performance metrics
      IF |r| > 0.2: "Potential new predictive signal identified"

  NEW SIGNALS RECOMMENDED:
    signal_name: [description]
    correlation: [r value with performance metric]
    proposed_weight: [starting weight for A10]
    confidence: [HIGH/MEDIUM based on data]
```

### Step 4: Generate Specific Scoring Model Adjustments

```
COMPILE all recommended changes to A10 scoring model:

  WEIGHT ADJUSTMENTS (from 2.3 scoring model audit):
    | Criterion | Current Weight | Recommended | Change | Data Basis |
    | [name]    | [X]           | [Y]         | [+/-Z] | [evidence] |

  BIAS CORRECTIONS:
    | Bias | Current Effect | Correction | Data Basis |
    | [type] | [description] | [specific fix] | [evidence] |

  NEW SIGNALS:
    | Signal | Proposed Weight | Correlation | Data Basis |

  PLATFORM-SPECIFIC ADJUSTMENTS:
    | Platform | Current Modifier | Recommended | Data Basis |

  EACH ADJUSTMENT MUST INCLUDE:
    - Specific magnitude (not "increase" but "increase from X to Y")
    - Data justification (variant count, performance delta, confidence)
    - Risk assessment (how confident are we in this change?)
```

### Step 5: Extract Prediction Learnings

```
FOR EACH actionable learning:

  FORMAT (MANDATORY):

  ### Learning PR-[ID]: [Title]

  **WHAT:** [Specific prediction accuracy observation with data]
  **WHY:** [Root cause of prediction error]
  **HOW:** [Specific A10 model adjustment]
  **CONFIDENCE:** [HIGH/MEDIUM/LOW]
  **SCOPE:** [Vertical-specific/Universal]
  **PROPAGATION TARGET:** [A10 scoring model, A06 Arena criteria, vertical profile]

MINIMUM: 2 prediction learnings. Target: 3-5.
```

## Quality Checks

1. All systematic biases from Layer 2 analysis documented
2. Missing predictive signals investigated
3. Scoring adjustments include specific magnitude (not vague "increase")
4. Every adjustment has data justification with confidence level
5. Minimum 2 prediction learnings in required format
6. If predictions unavailable (A10 not run), document as NOT_APPLICABLE
7. Platform-specific adjustments provided if multi-platform
8. Risk assessment included for each proposed change
