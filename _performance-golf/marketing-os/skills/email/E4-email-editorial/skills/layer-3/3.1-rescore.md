# Layer 3.1 -- Rescore

## Purpose

Rescore ALL emails in the revised sequence against email-quality-rubric.md using the same 9 individual criteria and 5 campaign-level criteria. Compare every score to the Layer 0 baseline. Verify that every email meets the minimum 7.5 threshold. This is the proof that the editorial process worked.

**CRITICAL:** Rescore ALL emails, not just the ones that were revised. Revisions can affect adjacent email flow and sequence-level dynamics.

---

```yaml
from_layer_2:
  revision_execution:
    arena_results: [object]  # which emails were revised and how
    direct_fix_results: [object]

  rubric: "./skills/email/shared/email-quality-rubric.md"
```

## Execution Protocol

### Step 1: Re-Read email-quality-rubric.md (MANDATORY)

```
READ the rubric file again. Even if it was read in Layer 0.
Context may have shifted. Do NOT score from memory.

IF rubric was not re-read in THIS step:
  HALT -- Read the file. Scoring without rubric = protocol violation.
```

### Step 2: Rescore Each Email (Individual Criteria)

For EVERY email in the revised sequence (including unchanged emails):

```
FOR email IN revised_sequence.emails:

  SCORE all 9 criteria (identical process to 0.2-quality-baseline):

  1. Opening Hook Strength (15%)
  2. Content-to-Pitch Ratio (15%)
  3. Transition/Bridge Smoothness (15%)
  4. Body Type Compliance (10%)
  5. Voice Consistency (10%)
  6. Engagement/Entertainment Value (10%)
  7. CTA Clarity and Strength (10%)
  8. Paragraph Brevity (5%)
  9. Niche Appropriateness (10%)

  CALCULATE weighted average:
    final_score = (hook * 0.15) + (ratio * 0.15) + (bridge * 0.15) +
                  (compliance * 0.10) + (voice * 0.10) + (engagement * 0.10) +
                  (cta * 0.10) + (brevity * 0.05) + (niche * 0.10)

  COMPARE to baseline:
    improvement = final_score - baseline_score
    per_criterion_improvement for each of 9 criteria

  CHECK threshold:
    IF final_score < 7.5:
      FLAG: "Email [position] scores [X] -- below 7.5 minimum"
      ACTION: Return to Layer 2 for additional revision
```

### Step 3: Rescore Campaign-Level Criteria (C1-C5)

```
C1: BODY TYPE VARIETY (re-verify)
  Revisions should not have changed body types, but verify:
  - No consecutive same type
  - In any 5-email window, >= 3 different types
  Result: PASS or FAIL

C2: EMOTIONAL ARC (re-verify)
  Verify the emotional progression is still intact after revisions:
  - Trust -> education -> desire -> urgency
  - No emotional regression (urgency email followed by trust email)
  Result: PASS or FAIL

C3: CROSS-EMAIL CONTINUITY (re-verify)
  Verify revisions didn't remove callbacks:
  - Search for cross-email references in emails 3+
  - If fewer callbacks than baseline: flag as regression
  Result: PASS or FAIL

C4: URGENCY ESCALATION (re-verify)
  Verify urgency pattern is intact:
  - First 2/3 low urgency
  - Final 1/3 escalating
  Result: PASS or FAIL

C5: SUBJECT LINE DIVERSITY (re-verify)
  Verify SL diversity (if any SLs were revised):
  - >= 3 different formula categories
  Result: PASS or FAIL
```

### Step 4: Calculate Improvement Metrics

```
AGGREGATE METRICS:

  average_baseline = mean(all baseline scores)
  average_final = mean(all final scores)
  average_improvement = average_final - average_baseline

  biggest_improvement:
    email_position: integer
    baseline: float
    final: float
    improvement: float

  smallest_improvement (or regression):
    email_position: integer
    baseline: float
    final: float
    improvement: float  # negative if regression

  criterion_improvements:
    FOR each of 9 criteria:
      average_baseline_for_criterion: float
      average_final_for_criterion: float
      improvement: float

  largest_criterion_gain:
    criterion: string
    improvement: float

  distribution_shift:
    baseline:
      excellent_8_5_plus: integer
      good_7_5_to_8_4: integer
      acceptable_7_0_to_7_4: integer
      below_standard_6_0_to_6_9: integer
      failure_below_6_0: integer
    final:
      excellent_8_5_plus: integer
      good_7_5_to_8_4: integer
      acceptable_7_0_to_7_4: integer
      below_standard_6_0_to_6_9: integer
      failure_below_6_0: integer
```

### Step 5: Threshold Verification

```
FINAL VERIFICATION:

  [ ] EVERY email scores >= 7.5 (no exceptions)
  [ ] Campaign-level C1 passes
  [ ] Campaign-level C2 passes
  [ ] Campaign-level C3 passes
  [ ] Campaign-level C4 passes
  [ ] Campaign-level C5 passes
  [ ] No email has REGRESSED from baseline (final < baseline)

IF any email < 7.5:
  HALT -- Return to Layer 2 for additional revision.
  Identify which criterion is weakest for that email.
  If the weakness was already addressed by Arena:
    Escalate to human: "Arena revision did not achieve 7.5. Manual edit may be needed."

IF any campaign-level criterion FAILS:
  HALT -- Address as P2 issue. May require Layer 2 re-entry.

IF any email regressed:
  WARN -- "Email [position] regressed from [baseline] to [final]."
  Investigate: Did a revision to adjacent email cause this?
  Fix the regression before proceeding.
```

## Failure Modes

| Failure | Remediation |
|---------|-------------|
| Email still below 7.5 after revision | Return to Layer 2. If Arena was already run, escalate to human. |
| Scoring inconsistency (same email scores differently from baseline without revision) | Re-read rubric. Apply scoring more carefully. Minor variance (0.2) is normal. Major variance (>0.5) suggests scoring error. |
| Campaign-level criterion regressed (was PASS, now FAIL) | A revision broke sequence-level quality. Investigate which revision caused it. Fix without breaking the revision's benefit. |
| All emails show regression | Scoring error likely. Re-read rubric, rescore from scratch. |
| Rubric file changed between baseline and rescore | Use the same rubric version. Note if rubric was updated during the session. |
