# Layer 0.3 — Input Validator

## Purpose

Validate loaded inputs from either 0.1 (Mode A upstream packages) or 0.2 (Mode B brief). Confirm minimum thresholds are met for campaign design. Set the mode flag. Determine context_richness score (1-10) that influences downstream campaign complexity decisions.

---

## Input Requirements

```yaml
inputs_to_validate:
  # One of these will be populated, the other null:
  upstream_context: object  # from 0.1-upstream-loader (Mode A)
  brief_context: object     # from 0.2-brief-parser (Mode B)
```

---

## Execution Protocol

### Step 1: Determine Operating Mode

```
IF upstream_context is populated AND upstream_context.loading_status.packages_loaded_count >= 1:
  → mode = "A"
  → Validate using Mode A thresholds

ELIF brief_context is populated AND brief_context.validation_status == "COMPLETE":
  → mode = "B"
  → Validate using Mode B thresholds

ELSE:
  → mode = "UNDETERMINED"
  → HALT — neither mode has sufficient input
```

### Step 2: Mode A Validation (Upstream Packages)

Check minimum package count:

```
REQUIRED: packages_loaded_count >= 5 of 8

CRITICAL PACKAGES (at least 3 of these 4 MUST be present):
  - offer_loaded = true      # Cannot design campaign without knowing what's sold
  - research_loaded = true   # Cannot profile audience without research
  - mechanism_loaded = true  # Cannot plan content without mechanism
  - promise_loaded = true    # Cannot plan CTAs without promise

IMPORTANT PACKAGES (enhance quality but not blocking):
  - proof_loaded
  - root_cause_loaded
  - big_idea_loaded
  - structure_loaded

IF packages_loaded_count < 5:
  → gate_0_recommendation = "FAIL"
  → List missing packages with upstream skill numbers

IF any critical package missing:
  → gate_0_recommendation = "FAIL"
  → Specify which critical package is missing and why it blocks design
```

Assess Mode A context richness:

| Score | Criteria |
|-------|----------|
| 8-10 | All 8 packages loaded, all with complete data |
| 6-7 | 6-7 packages loaded, critical packages complete |
| 5 | Exactly 5 packages loaded (minimum threshold) |
| 1-4 | Below minimum — FAIL |

### Step 3: Mode B Validation (Brief)

Check minimum fields:

```
REQUIRED FIELDS (all 3 MUST be present):
  - product description: non-empty string
  - target audience: non-empty string
  - campaign goal: non-empty string

IF any required field missing:
  → gate_0_recommendation = "FAIL"
  → Return brief_context.missing_field_questions to user
```

Inherit context_richness score from 0.2-brief-parser.md output. If not present, calculate:

| Score | Criteria |
|-------|----------|
| 8-10 | All required + 5+ optional fields with detail |
| 6-7 | All required + 3-4 optional fields |
| 4-5 | All required + 1-2 optional fields |
| 1-3 | Required fields only, minimal detail |

### Step 4: Map Context Richness to Campaign Complexity

Context richness constrains what campaign types are feasible:

```yaml
complexity_mapping:
  minimal:  # context_richness 1-3
    viable_types: [BP, DU]
    max_emails: 3
    body_type_variety: "limited — ST, LB only"

  standard:  # context_richness 4-5
    viable_types: [BP, DU, CP, AL]
    max_emails: 7
    body_type_variety: "moderate — ST, LB, QO, TM"

  full:  # context_richness 6-7
    viable_types: [BP, DU, CP, AL, AR, DR]
    max_emails: 15
    body_type_variety: "full — all 7 types"

  complex:  # context_richness 8-10
    viable_types: [BP, DU, CP, AL, AR, DR, PL]
    max_emails: 30
    body_type_variety: "full with deep variety"
```

### Step 5: Compile Validation Report

---

## Output

```yaml
validation_report:
  mode: enum[A, B, UNDETERMINED]
  gate_0_recommendation: enum[PASS, FAIL]

  mode_a_validation:  # null if Mode B
    packages_loaded: integer
    packages_required: 5
    critical_packages_present: integer
    critical_packages_required: 3
    missing_packages: [string]
    missing_critical: [string]

  mode_b_validation:  # null if Mode A
    required_fields_present: boolean
    missing_fields: [string]
    missing_field_questions: [string]

  context_richness:
    score: integer  # 1-10
    complexity_level: enum[minimal, standard, full, complex]
    viable_campaign_types: [string]
    max_recommended_emails: integer
    body_type_variety: string

  warnings: [string]
  blockers: [string]
```

---

## Failure Modes

| Failure | Remediation |
|---------|-------------|
| Mode A: fewer than 5 packages | List missing packages. Recommend running upstream skills first. |
| Mode A: critical package missing | Specify which critical package and why it blocks. "Cannot design email campaign without [offer/research/mechanism/promise]." |
| Mode B: required field missing | Return specific questions from brief parser. Block until user provides answers. |
| Both modes empty | "No upstream packages loaded and no brief provided. Provide either upstream skill outputs (Mode A) or a campaign brief (Mode B)." |
| Context richness too low for requested campaign type | Warn that requested type requires richer context. Recommend simpler type or additional brief detail. |
