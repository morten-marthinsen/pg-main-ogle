# 1.1 — Baseline Scorer

**Version:** 1.0
**Skill:** U5 — Upsell Editorial
**Layer:** 1
**Microskill:** 1.1 — Baseline Scorer
**Model:** opus
**Purpose:** Score every piece in the assembled upsell sequence across all 7 editorial lenses (1-10 each), score all 5 Sequence-Level Criteria (S1-S5), and calculate weighted averages. This establishes the BASELINE that all subsequent revision must beat.

---

## INPUT

- `layer-0/0.1-input-context.md` (piece inventory, congruence thread, pricing data)
- `layer-0/0.2-scoring-rubric.md` (7 lenses with rubrics, S1-S5 criteria, weighted average formula)
- The assembled upsell sequence (full text of every piece — from U4)

## EXECUTION STEPS

1. **Confirm prerequisites.** Verify that 0.1 and 0.2 output files exist and are complete. If either is missing, HALT — Layer 0 must complete before scoring begins.

2. **Load the scoring rubric.** Read the full per-lens rubric from `0.2-scoring-rubric.md`. Internalize the 1-10 scale definitions for every lens. Reference the rubric for EVERY score assignment — do not score from memory or gut feel.

3. **Score ORDER BUMP across all 7 lenses.** For each lens:
   - Read the bump copy in full
   - Apply the lens's key question
   - Assign a score (1-10) using the rubric tier definitions
   - Write specific rationale citing exact passages from the copy
   - Note: Bump copy is 50-150 words. It is SHORT. Still score all 7 lenses. The Congruence Auditor checks mechanism name presence. The Speed Enforcer checks word economy. The CTA Optimizer checks whether the yes/no is clean. Short copy is not exempt from any lens.

4. **Score UPSELL PAGE(S) across all 7 lenses.** For each lens:
   - Read the upsell page copy in full
   - Apply the lens's key question
   - Assign a score (1-10) using the rubric tier definitions
   - Write specific rationale citing exact passages
   - Pay special attention to: CAIRO structure compliance (is each section identifiable?), mechanism name count (2+ required), tone (post-purchase celebration, not selling), proof count (max 2 elements)

5. **Score DOWNSELL PAGE(S) across all 7 lenses.** For each lens:
   - Read the downsell page copy in full
   - Apply the lens's key question
   - Assign a score (1-10) using the rubric tier definitions
   - Write specific rationale citing exact passages
   - Pay special attention to: ARO structure compliance (not CAIRO), tone (acknowledgment, not guilt), reframe quality (genuine alternative, not "just cheaper"), proof (max 1 element)

6. **Calculate weighted average per piece.**
   - Formula: (Congruence * 0.25) + (Speed * 0.15) + (Tone * 0.20) + (Value * 0.15) + (Flow * 0.10) + (CTA * 0.10) + (Integrator * 0.05)
   - Record the weighted average for each piece
   - Flag any piece with weighted average < 7.5 (will need revision to pass GATE_3)
   - Flag any individual lens score < 5.0 (severe weakness regardless of average)

7. **Score S1-S5 Sequence-Level Criteria.** These are scored across the ENTIRE sequence, not per piece:
   - **S1 — Pricing Cascade:** Extract all prices. Calculate Bump/FE, Upsell/FE, Downsell/Upsell ratios. Compare to standard ranges. Score PASS or FAIL with exact ratios documented.
   - **S2 — Congruence Chain:** Check mechanism name across ALL pieces. Check root cause language across ALL pieces. Check promise extension across ALL pieces. Score PASS or FAIL with evidence per piece.
   - **S3 — Speed Compliance:** Sum word counts from 0.1 inventory. Calculate total reading time at 200 wpm. Score PASS (<4 min) or FAIL (>=4 min) with exact time.
   - **S4 — Voice Consistency:** Scan ALL pieces for tone breaks. Celebration expected on bump + upsell. Acknowledgment expected on downsell. Any pre-purchase selling language = violation. Score PASS or FAIL with evidence.
   - **S5 — Value Escalation:** At each step (FE -> Bump -> Upsell -> Downsell), verify descending commitment + ascending value-per-dollar. Score PASS or FAIL with per-step analysis.

8. **Compute overall baseline summary.** Average of all piece weighted averages. Count of S-criteria passing. Identify the weakest piece and weakest lens overall.

9. **Write output file** with complete scoring data.

## CONSTRAINTS

1. **Score BEFORE revising.** This is the baseline. It must reflect the AS-RECEIVED quality from U4. Any revision applied before scoring invalidates the baseline.
2. **Every piece, every lens.** No exceptions. The order bump is 100 words — it still gets scored on all 7 lenses. A short piece is not exempt.
3. **Rationale must cite exact passages.** "Tone is okay" is not rationale. "The Congratulate section opens with 'You've made an amazing choice...' which correctly uses post-purchase celebration framing" is rationale.
4. **Use the rubric tier definitions.** A score of 8 means you can justify it fits the 7-8 tier description AND does not fit the 9-10 tier. Scores are not arbitrary.
5. **S-criteria are sequence-level.** Do not score S2 (Congruence Chain) per piece — check it ACROSS pieces. The question is whether the chain holds from FE through Downsell.
6. **Weighted average formula is FIXED.** Do not adjust weights. The formula is defined in 0.2 and must be applied exactly.
7. **Flag all weaknesses explicitly.** Any piece below 7.5 or any lens below 5.0 gets explicitly flagged. These inform issue identification in 1.2.

## FAILURE MODES

| Failure | Detection | Response |
|---------|-----------|----------|
| Scoring without rubric | No reference to 0.2-scoring-rubric.md | HALT — load rubric first. Unbounded scoring produces inconsistent results. |
| Missing piece | Fewer pieces scored than inventoried in 0.1 | Score all pieces. If a piece exists in the inventory, it gets scored. |
| Vague rationale | Rationale does not cite specific passages | Rescore with passage-level evidence. Every score needs a quote. |
| Lens confusion | Scoring tone under Congruence Auditor or vice versa | Each lens has a distinct focus. If a passage is relevant to multiple lenses, score it under each with that lens's specific criteria. |
| S-criteria scored per-piece | S1-S5 entries reference individual pieces instead of the sequence | Re-scope. S-criteria evaluate the sequence as a whole. |
| Weighted average miscalculated | Sum of (score * weight) doesn't match reported average | Recalculate. Show the math. |
