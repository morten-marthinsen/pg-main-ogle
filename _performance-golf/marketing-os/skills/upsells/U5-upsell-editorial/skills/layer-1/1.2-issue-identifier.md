# 1.2 — Issue Identifier

**Version:** 1.0
**Skill:** U5 — Upsell Editorial
**Layer:** 1
**Microskill:** 1.2 — Issue Identifier
**Model:** opus
**Purpose:** Identify ALL issues across the assembled upsell sequence based on baseline scores, classify each by severity (P1-P4), and tag with affected piece, lens, exact passage, specific problem description, and target fix. Carry forward all U4 validation flags as pre-identified issues.

---

## INPUT

- `layer-1/1.1-baseline-scores.md` (per piece x per lens scores + S1-S5 results)
- `layer-0/0.1-input-context.md` (U4 validation flags carried forward, congruence thread)
- The assembled upsell sequence (full text — for passage citation)
- U5-UPSELL-EDITORIAL-AGENT.md (severity classification table)

## EXECUTION STEPS

1. **Carry forward U4 flags FIRST.** Before identifying new issues, ingest ALL flags from `0.1-input-context.md` (originally from U4's validation-report.md). For each U4 flag:
   - Assign severity (P1/P2/P3/P4) based on the severity classification table
   - Tag with affected piece and relevant lens
   - Mark as "U4-FLAGGED" so downstream processes know this was pre-identified
   - Do NOT re-analyze or downgrade U4 flags. If U4 flagged a congruence warning, it enters as at least P2 until baseline scoring confirms or escalates.

2. **Scan baseline scores for issues.** Systematically review every score in `1.1-baseline-scores.md`:
   - Any individual lens score <= 6 on any piece → issue. The score itself identifies the weakness; now identify the SPECIFIC passage and problem.
   - Any piece with weighted average < 7.5 → at least one issue must be identified that explains why
   - Any S-criterion that FAILED → issue at the sequence level
   - Any lens score that is the lowest across all pieces → even if above 6, flag as a potential issue if it drags the piece down

3. **For each identified issue, create a full tag set:**

   **(a) Issue ID:** Sequential number (I-001, I-002, ...)
   **(b) Severity:** P1, P2, P3, or P4 — using this classification:
   - **P1 Critical:** Fundamental structural or congruence failure. Examples: mechanism name missing/wrong, PAS structure in upsell, inverted pricing (downsell > upsell), CAIRO in downsell (should be ARO), complete tone reversal
   - **P2 Major:** Significant quality issue degrading conversion. Examples: tone shift to selling, missing CAIRO/ARO section, weak reframe in downsell ("just cheaper"), proof overload (>2 upsell / >1 downsell), guilt in CTA
   - **P3 Minor:** Quality issue reducing polish. Examples: word choice improvements, CTA phrasing refinement, minor price presentation gaps, awkward transitions, redundant sentences
   - **P4 Cosmetic:** Formatting/presentation. Examples: spacing, capitalization, formatting inconsistencies, typos, section header styling
   **(c) Piece affected:** Order Bump / Upsell Page [N] / Downsell Page [N] / Sequence-wide
   **(d) Lens(es):** Which of the 7 lenses detected this issue (may be multiple)
   **(e) Passage:** EXACT quote from the copy that contains the problem. Include enough surrounding context to locate it unambiguously.
   **(f) Problem:** What specifically is wrong. Not "tone is off" but "This sentence uses 'Don't miss this opportunity' which is pre-purchase urgency language in a post-purchase context."
   **(g) Target fix:** What "fixed" looks like. Not "improve the tone" but "Rewrite as post-purchase validation: 'You've already secured the foundation — this simply accelerates your results.'"

4. **Cross-reference for completeness.** After individual lens scans:
   - Check: does every lens score <= 6 have a corresponding issue? If not, identify the missing issue.
   - Check: does every S-criterion failure have a corresponding issue? If not, create one.
   - Check: are there cross-cutting issues that span multiple lenses or pieces? Tag these with all affected lenses/pieces.
   - Check: are there issues the lenses might miss? (e.g., factual errors, broken guarantees, missing product names). Identify these as lens-independent issues.

5. **Severity validation.** Review all assigned severities:
   - P1 should be RARE. If more than 2-3 P1 issues exist, the upstream pipeline has significant problems — flag for potential escalation to U2/U3 re-execution.
   - P2 should capture the bulk of meaningful issues.
   - P3 should be the most common category.
   - P4 should be mechanical only.
   - If all issues are P3/P4, the sequence is strong and editorial is primarily polish.

6. **Count and summarize.** Total issues by severity. Total issues by piece. Total issues by lens. Identify the most-flagged piece and most-flagged lens.

7. **Write output file** with complete issue list and summary statistics.

## CONSTRAINTS

1. **U4 flags are carried forward IN FULL.** Do not filter, reinterpret, or discard U4 flags. They enter the issue list with at least their U4-estimated severity. You can ESCALATE a U4 flag's severity after baseline scoring (e.g., U4 flagged as warning -> you confirm it's P1), but never downgrade without explicit evidence.
2. **Passages must be EXACT QUOTES.** Not paraphrases. Not summaries. The exact words from the copy, quoted, with enough context to locate the passage unambiguously.
3. **Target fixes must be SPECIFIC.** "Fix the tone" is not a target fix. "Rewrite the Amplify section opening from urgency framing to future-pacing: replace 'Don't let this slip away' with 'Picture yourself 30 days from now...'" is a target fix.
4. **Every score <= 6 must map to at least one issue.** If a lens scored 5 on a piece, there MUST be an issue explaining why. Scores and issues must be traceable.
5. **Severity classification follows the AGENT.md table.** Do not invent new severity levels. P1, P2, P3, P4 only.
6. **This microskill identifies issues — it does not fix them.** Do not rewrite copy here. Identify what's wrong and describe what fixed looks like. Actual fixing happens in Layer 2.
7. **Cross-cutting issues get tagged with ALL affected pieces and lenses.** A congruence break that spans the entire sequence is one issue tagged "Sequence-wide" — not three separate issues per piece.

## FAILURE MODES

| Failure | Detection | Response |
|---------|-----------|----------|
| U4 flags dropped | Issue list has zero U4-FLAGGED entries but 0.1 carried forward flags | Re-ingest U4 flags. Every flag must appear in the issue list. |
| Score-issue mismatch | Lens scored <= 6 but no issue maps to that lens + piece | Identify the missing issue. A low score without an explanation is incomplete analysis. |
| Vague problem description | Problem says "not great" or "needs work" without specifics | Rewrite with exact diagnosis: what word, phrase, or structural element is wrong and why. |
| Vague target fix | Target fix says "improve" or "make better" | Rewrite with specific alternative phrasing or structural change. |
| Over-classification as P1 | More than 3 P1 issues identified | Review each P1 — is it truly a fundamental structural/congruence failure? Downgrade to P2 if it degrades quality but doesn't break structure. |
| Zero issues found | No issues identified across any piece or S-criterion | Extremely unlikely. Re-examine with fresh eyes. At minimum, every sequence has P3/P4 polish opportunities. |
