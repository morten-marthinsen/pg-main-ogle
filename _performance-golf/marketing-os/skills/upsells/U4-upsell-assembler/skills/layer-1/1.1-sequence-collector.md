# 1.1 — Sequence Collector

**Skill:** U4 — Upsell Sequence Assembler
**Microskill:** 1.1 — Sequence Collector
**Layer:** 1
**Model:** sonnet
**Input:** `layer-0/0.1-input-context.md`, all loaded upstream copy pieces (U1, U2, U3)
**Output:** `layer-1/1.1-sequence-order.md`

---

## PURPOSE

Arrange all loaded upsell sequence pieces in correct funnel order, verify completeness against U0 strategy, calculate per-piece word counts, and flag any missing pieces. This produces the ordered reference that all other Layer 1 validators and the Layer 4 packager consume.

---

## EXECUTION STEPS

1. **Read `layer-0/0.1-input-context.md`** to determine operating mode (Full vs Partial) and which pieces are loaded.

2. **Establish the canonical funnel order:**
   ```
   ORDER FORM: [FE Product] — $[price]
     └── ORDER BUMP: [Bump Product] — $[price]
            ↓ [purchase completes]
         UPSELL 1: [Upsell Product] — $[price]
            ├── YES → Thank You (or Upsell 2 if applicable)
            └── NO → DOWNSELL 1: [Downsell Product] — $[price]
                       └── YES/NO → Thank You
   ```

3. **Map each loaded piece to its position in the funnel order.** Reference the U0 strategy to confirm position assignments. If U0 specified a different order than the standard above, use U0's order (U0 is the strategic authority).

4. **Verify completeness against U0 strategy:**
   - Compare the pieces loaded (from 0.1) against the planned sequence in U0
   - For each planned piece, mark: PRESENT or MISSING
   - If Mode A and any piece is MISSING: this should already be a GATE_0 fail — log it as a critical error
   - If Mode B: document which pieces are present and which are absent. Flag the partial nature.

5. **Calculate per-piece word counts:**
   - For each loaded piece, count total words in the copy body (exclude YAML metadata, section headers, implementation notes)
   - Record: piece name, position, word count
   - Calculate total word count across all loaded pieces

6. **Build the sequence map** — a visual representation of the funnel with actual product names and prices slotted in.

7. **Flag any structural anomalies:**
   - Downsell without a preceding upsell (structural impossibility)
   - Upsell 2 without Upsell 1 (gap in chain)
   - Order bump without FE context (no parent product)

8. **Write output file** with ordered sequence, completeness check, word counts, and flags.

## CONSTRAINTS

- The funnel order is fixed: Order Form → Bump → Upsell → Downsell → Thank You. U0 may specify variations (e.g., Upsell 2 before Downsell), but the fundamental hierarchy is always: purchase event → immediate post-purchase offers → exit.
- Word counts measure COPY BODY only — exclude YAML metadata headers, section labels (## CONGRATULATE, etc.), implementation notes, and downstream handoff data.
- Do not reorder pieces based on your own judgment. U0 strategy is the authority on sequence order. If U0's order seems unusual, document it but follow it.
- Missing pieces in Mode A at this stage indicate a GATE_0 failure that was not caught. Log as CRITICAL ERROR but continue collecting what is available — the gate enforcement is upstream.
- This microskill produces the ORDERED REFERENCE that 1.2-1.5 and 4.1 all consume. It must be accurate — any error here cascades to all validators.

---

## FAILURE MODES

| Failure | Detection | Response |
|---------|-----------|----------|
| Piece loaded but cannot be positioned in funnel | No matching position in U0 strategy for this piece | Flag as unpositioned. Likely a mismatch between U0 plan and actual output. Escalate for human clarification. |
| Word count severely exceeds U0 target | Delta > 50% of target | Flag with specific delta. Speed validator (1.5) will catch the reading time impact, but flag the strategic deviation here. |
| Duplicate pieces for same position | Two files both claim to be "Upsell 1" | HALT. Determine which is the Arena-selected version (check GATE_2.5 checkpoints). Only one piece per position. |
| U0 strategy specifies pieces not in standard funnel | e.g., Upsell 3 or second downsell | Document the extended funnel structure. Proceed with available pieces. Flag positions that have no loaded copy. |
