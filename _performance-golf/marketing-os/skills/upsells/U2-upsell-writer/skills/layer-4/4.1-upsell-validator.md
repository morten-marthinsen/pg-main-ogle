# 4.1 — Upsell Validator

**Version:** 1.0
**Skill:** U2 — 1-Click Upsell Writer
**Layer:** 4
**Microskill:** 4.1 — Upsell Validator
**Purpose:** Run 7 validation gates against the Arena-selected upsell page to confirm structural compliance, congruence integrity, tone correctness, price completeness, and binary choice presence before output packaging.

---

## INPUT

- Arena-selected upsell page (the winning output from Layer 2.5 — HUMAN_SELECT gate)
- `layer-1/1.1-congruence-map.md` (mechanism name, root cause language — for exact-match validation)
- `layer-0/0.1-input-context.md` (FE mechanism name, root cause language — canonical source)
- `layer-0/0.3-format-classification.md` (word count target, format type)

**CRITICAL:** Validate the ARENA-SELECTED output, NOT the Layer 2 reference draft. This is the most common validation error — checking the wrong file. Confirm the file being validated is the human-selected Arena winner.

## PROCESS

1. **Confirm validation target.** Identify the Arena-selected file. Check for GATE_2.5 (HUMAN_SELECT) checkpoint confirming human selection. If no selection checkpoint exists, HALT — cannot validate without Arena completion.

2. **Run Gate 1: Word Count (HARD FAIL)**
   - Count total words in the upsell page (excluding metadata, section headers, and stage directions if video)
   - PASS: 500-2000 words inclusive
   - FAIL: Under 500 or over 2000
   - This is a HARD FAIL — no exceptions, no "close enough." If the count is 499 or 2001, it fails.
   - Evidence: report exact word count

3. **Run Gate 2: CAIRO Structure Present**
   - Scan for all 5 CAIRO sections: Congratulate, Amplify, Intrigue, Reason, Offer
   - Each section must be identifiable — either by explicit header, by clear structural break, or by content function
   - PASS: All 5 sections present and distinguishable
   - FAIL: Any section missing, or two sections merged without distinct identity
   - Evidence: quote the opening line of each identified section

4. **Run Gate 3: Congruence — Mechanism Name**
   - Search for the EXACT mechanism name string from the congruence map
   - Must appear in the Congratulate section (MANDATORY)
   - Must appear in at least 1 additional section (MANDATORY — total 2+ appearances)
   - PASS: Mechanism name appears by exact name in 2+ sections
   - FAIL: Mechanism name missing, paraphrased, or only appears once
   - Evidence: quote each instance with section location
   - Common failure: "this system" or "this protocol" used instead of the actual mechanism name

5. **Run Gate 4: Congruence — Root Cause Language**
   - Search for the VERBATIM root cause language from the congruence map
   - Must appear at least once in the upsell page
   - PASS: Root cause language appears verbatim
   - FAIL: Root cause language absent or paraphrased (e.g., "hormone problems" instead of "hormonal disruption caused by environmental toxins")
   - Evidence: quote the instance with section location, or quote the closest paraphrase if failed

6. **Run Gate 5: Tone — Post-Purchase Warmth Check**
   - Scan the ENTIRE page for pre-purchase selling language:
     - **Flag as violations:** "imagine your problem," "are you tired of," "do you struggle with," "limited time," "act now," "don't miss out," "before it's too late," "you can't afford to," "what if I told you," "are you sick of"
     - **Flag as warnings (may be acceptable in context):** "urgent," "exclusive," "special offer," "one-time," "only available today"
   - PASS: Zero violation flags. Warning flags are acceptable if used in post-purchase context (e.g., "this special offer is only available on this page" is contextually appropriate).
   - FAIL: Any violation flag detected
   - Evidence: quote each flagged instance with section location and classification (violation vs warning)

7. **Run Gate 6: Price Presentation Complete**
   - Check for all 4 mandatory price presentation elements:
     - (a) Value anchor: a higher reference price is stated before the actual price
     - (b) Actual price: the upsell price is clearly stated with contrast language
     - (c) Per-day breakdown: price divided by usage period, presented as a daily amount
     - (d) Comparison anchor: price compared to a familiar daily expense
   - Also check: guarantee is present (any form)
   - PASS: All 4 elements present + guarantee present
   - FAIL: Any element missing
   - Evidence: quote each element found, note any missing

8. **Run Gate 7: Binary Choice Present**
   - Check for BOTH yes and no options:
     - YES option: contains product name + price + action verb ("Add")
     - NO option: clean decline with zero guilt language
   - Scan the NO option specifically for manipulation:
     - "No, I don't want to [desirable thing]" = FAIL
     - "No, I'm fine being [undesirable thing]" = FAIL
     - "No, I'll pay full price later" = FAIL
     - "No thanks, I'll pass on this special offer" = PASS
     - "No thanks, I'll continue with just the [FE product]" = PASS
   - PASS: Both options present, NO option is clean
   - FAIL: Either option missing OR NO option contains guilt/manipulation
   - Evidence: quote both YES and NO options verbatim

9. **Compile validation report.** Summary table + detailed evidence for each gate.

10. **Determine overall status:**
    - All 7 gates PASS → proceed to 4.2 (Congruence Score)
    - Any gate FAIL → document failures, return to Layer 2 for revision with specific failure flags
    - Gate 1 (word count) FAIL is the hardest fail — requires structural revision, not just editing

## CONSTRAINTS

1. **Validate the Arena-selected output, not the Layer 2 draft.** This is the #1 validation error. Confirm the validation target is the human-selected Arena winner before running any gate.
2. **Gates are PASS or FAIL.** No partial pass. No "PASS with notes." No "conditional pass." PASS or FAIL. Binary.
3. **Gate 1 (word count) is a HARD FAIL with zero tolerance.** 499 words = FAIL. 2001 words = FAIL. Do not rationalize "it's close enough."
4. **Gate 3 requires the EXACT mechanism name string.** Case-sensitive, full name, including articles. "The Metabolic Switch Protocol" is not matched by "metabolic switch" or "the Metabolic Switch" (missing "Protocol").
5. **Gate 5 flag list is not exhaustive.** If you encounter pre-purchase selling language not on the list, flag it. The list provides common examples, not a complete dictionary. Use judgment: does this sentence assume the buyer has NOT yet purchased? If yes, it is a violation.
6. **Gate 7 NO option analysis must be strict.** Any framing that makes the buyer feel bad, stupid, or irresponsible for declining is a FAIL. When in doubt, it fails. Clean means clean.
7. **Revision directives must be SPECIFIC.** Not "fix the tone" but "Intrigue section line 3 contains 'Are you tired of...' — rewrite as post-purchase framing. Suggested: 'You've already taken the first step by...'." The Layer 2 agent needs to know exactly what to change and where.
