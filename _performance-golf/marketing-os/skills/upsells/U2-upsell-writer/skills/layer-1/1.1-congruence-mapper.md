# 1.1 — Congruence Mapper

**Version:** 1.0
**Skill:** U2 — 1-Click Upsell Writer
**Layer:** 1
**Microskill:** 1.1 — Congruence Mapper
**Purpose:** Map the exact congruence thread from the front-end offer to the upsell, ensuring mechanism name, root cause language, and promise phrasing are preserved and extended — not replaced or paraphrased.

---

## INPUT

- `layer-0/0.1-input-context.md` (mechanism name, root cause language, FE promise, FE product name, upsell product name)
- `mechanism-package.json` (from Skill 04, if Mode A)
- `root-cause-package.yaml` (from Skill 03, if Mode A)
- `promise-output.json` (from Skill 05, if Mode A)

## PROCESS

1. **Extract the three congruence anchors** — these are the EXACT strings that must appear in the upsell:
   - **Mechanism name:** The branded name of the front-end mechanism (e.g., "The Metabolic Switch Protocol"). Record this as an exact string. No paraphrasing. No abbreviation.
   - **Root cause language:** The verbatim phrasing of the root cause from the front-end (e.g., "hormonal disruption caused by environmental toxins"). Record the exact phrase, not a summary.
   - **Promise phrasing:** The core promise from the front-end (e.g., "finally break through your weight loss plateau"). Record the exact wording.

2. **Define the extension logic for each anchor** — how the upsell EXTENDS each element without replacing it:
   - **Mechanism extension:** The upsell mechanism is a COMPLEMENT to the FE mechanism. Write the logical connection: "[Upsell product] works WITH [mechanism name] to [additional benefit]." The upsell does NOT introduce a new mechanism. It accelerates, deepens, or completes the existing one.
   - **Root cause extension:** The upsell addresses the SAME root cause, not a new one. Write the extension: "[FE product] addresses [root cause]. [Upsell product] ensures [root cause] stays addressed by [additional function]."
   - **Promise extension:** The upsell makes the FE promise BIGGER, FASTER, or MORE COMPLETE. Write the extension: "You'll get [FE promise]. [Upsell product] makes that happen [faster / more completely / more permanently]."

3. **Write the congruence bridge** — the single logical sentence that connects the purchase to the upsell. This sentence is the heart of the upsell's logic. Examples:
   - "You just secured [FE product] to [FE promise] using [mechanism name]. [Upsell product] makes [mechanism name] work [faster/deeper/more completely] by [specific function]."
   - Test: could this bridge sentence work for a DIFFERENT upsell product with a different mechanism? If yes, it is too generic. The bridge must be SPECIFIC to this mechanism + this upsell.

4. **Run the congruence stress test** — three checks:
   - **Generic substitution test:** Replace the mechanism name with "this system" in the bridge sentence. Does it still make sense? If YES → congruence is surface-level only. The mechanism name is decorative, not structural. FAIL — rewrite the bridge so the mechanism name is load-bearing.
   - **Product swap test:** Could you swap the upsell product for ANY other product and the bridge sentence would still work? If YES → the extension logic is too vague. FAIL — make the extension specific to this upsell product's actual function.
   - **New problem test:** Does the extension logic introduce a problem the FE didn't address? If YES → you've broken congruence by creating a new anxiety. FAIL — the upsell addresses the SAME problem more thoroughly, not a new problem.

5. **Map mechanism name placement** — identify where in the CAIRO structure the mechanism name will appear:
   - Congratulate section: MANDATORY (validate their purchase of this mechanism)
   - At least 1 additional section: MANDATORY (reinforce the thread)
   - Recommended: Mechanism name in C, A, and R sections (three touchpoints)

6. **Map root cause language placement** — identify where root cause language appears:
   - Amplify section: RECOMMENDED (future pacing the resolution of this root cause)
   - Reason section: RECOMMENDED (the "why" connects back to addressing the same root cause more completely)

7. **Write output file** with all three anchors, extension logic, bridge sentence, stress test results, and placement map.

## CONSTRAINTS

1. Mechanism name is an EXACT STRING. "The Metabolic Switch Protocol" is not "the metabolic switch" or "Metabolic Switching" or "this protocol." Case, articles, and full name must match.
2. Root cause language is VERBATIM. "Hormonal disruption caused by environmental toxins" is not "hormone problems" or "toxic hormone disruption." The exact phrasing from the front-end.
3. The congruence bridge must pass all three stress tests. If any test fails, rewrite the bridge before proceeding. Do not move to Layer 2 with a weak bridge.
4. Extension logic must be SPECIFIC to this upsell product. "Makes it work better" is not extension logic. "Primes your metabolism before meals so the Metabolic Switch activates 40% faster" is extension logic.
5. The upsell does NOT introduce a new problem. It addresses the same problem more completely. If the extension logic requires a new anxiety, the congruence is broken.
6. Mechanism name must appear in at minimum 2 CAIRO sections. One appearance is decorative. Two is structural. Three is ideal.
7. This microskill executes FIRST in Layer 1 because all subsequent Layer 1 microskills depend on the congruence map. Do not run 1.2 or 1.3 until 1.1 is complete.
