# 0.2 — Brief Parser

**Layer:** 0
**Model:** haiku
**Depends on:** 0.1 (only executes if Mode B determined)
**Produces:** `layer-0/0.2-brief-context.md`

---

## Purpose

Parse a standalone brief when upstream Main Pipeline packages are not available (Mode B). Extract the minimum required context for upsell strategy design.

## When This Runs

ONLY when 0.1 (Upstream Loader) determines Mode B — i.e., offer-package.json or campaign-brief.json is missing.

## Required Inputs (Mode B)

| Input | Required? | What's Needed |
|-------|-----------|---------------|
| Product/offer details | REQUIRED | What's being sold, at what price, what's included |
| Upsell product descriptions | REQUIRED | What products/services could be offered as upsells |
| Target audience | REQUIRED | Who's buying (demographics + psychographics, even informal) |
| Front-end mechanism/angle | RECOMMENDED | The core selling angle, even if not formally named |
| Guarantee | RECOMMENDED | What guarantee is offered on front-end |

## Execution Steps

1. Parse user-provided brief or conversation context
2. Extract and organize:
   - **Product:** name, price, what's included, key benefit
   - **Upsell candidates:** list of products/services available for upsell positions
   - **Audience:** who's buying, their situation, their desires
   - **Mechanism/angle:** the selling angle (may need to be inferred)
   - **Guarantee:** if mentioned
3. Flag any missing elements
4. Write organized context to output file
5. Add MODE B WARNING to output

## Output Format

```markdown
# U0 Brief Context (Mode B)

⚠️ **MODE B: Limited upstream context. Strategy will be less precise than Mode A.**

## Front-End Offer
- Product: [name]
- Price: $[price]
- Included: [list]
- Key benefit: [statement]

## Available Upsell Products
1. [product 1]: [description, potential price]
2. [product 2]: [description, potential price]
3. [product 3]: [description, potential price]

## Mechanism/Angle
- [description — may be informal]

## Audience
- [description]

## Guarantee
- [description or "Not specified"]

## Gaps
- [list of missing context that would improve strategy]
```

## Failure Conditions
- No product details at all → CANNOT PROCEED. Request from human.
- No upsell candidates → CANNOT PROCEED. Need at least 1 product to upsell.
- No audience info → CAN PROCEED with flag (strategy will be generic).
