# 0.1 — Upstream Loader

**Layer:** 0
**Model:** haiku
**Depends on:** Nothing (first microskill)
**Produces:** `layer-0/0.1-upstream-context.md`

---

## Purpose

Load upstream packages from the Main Pipeline to provide the Upsell Strategist with full context about the front-end offer, campaign, and mechanism.

## Inputs (Mode A)

| Package | Source Skill | Required? | Key Data Points |
|---------|-------------|-----------|-----------------|
| `offer-package.json` | Skill 07 | REQUIRED | Core product, price, bonuses, guarantee, value stack |
| `campaign-brief.json` | Skill 09 | REQUIRED | Audience profile, tone, constraints, campaign goals |
| `mechanism-package.json` | Skill 04 | RECOMMENDED | Mechanism name, explanation, proof elements |
| `root-cause-package.yaml` | Skill 03 | OPTIONAL | Root cause framing, language, three-part structure |
| `promise-output.json` | Skill 05 | OPTIONAL | Promise statement, calibration, proof ceiling |
| `Soul.md` | Project | OPTIONAL | Voice constraints, anti-voice, pacing |

## Execution Steps

1. Check project directory for each package file
2. Load each found file and extract key data points
3. Determine operating mode:
   - If offer-package.json AND campaign-brief.json exist → Mode A
   - If either missing → Flag, switch to Mode B (trigger 0.2 instead)
4. Extract and organize:
   - **From offer-package:** product name, base price, bonus stack, guarantee type, value equation
   - **From campaign-brief:** target audience demographics/psychographics, tone direction, any upsell notes
   - **From mechanism-package:** mechanism name (EXACT — this threads through entire upsell sequence), explanation summary, key proof elements
   - **From root-cause:** root cause language (for congruence mapping)
   - **From promise:** promise statement (for extension logic)
5. Write organized context to output file

## Output Format

```markdown
# U0 Upstream Context

## Operating Mode: A (downstream)

## Front-End Offer
- Product: [name]
- Price: $[price]
- Bonuses: [list]
- Guarantee: [type + terms]

## Mechanism
- Name: [exact name]
- Core explanation: [1-2 sentences]

## Root Cause
- Language: [exact phrasing]

## Promise
- Statement: [exact]

## Audience
- Demographics: [summary]
- Psychographics: [summary]

## Packages Loaded: [count]/6
## Packages Missing: [list]
```

## Failure Conditions
- offer-package.json not found → Switch to Mode B
- campaign-brief.json not found → Switch to Mode B
- Package file exists but is empty/corrupted → Flag + switch to Mode B for that data point
