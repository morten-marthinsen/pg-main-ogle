# 0.2 — Specimen Calibrator

**Version:** 1.0
**Skill:** U1 — Order Bump Writer
**Layer:** 0
**Microskill:** 0.2 — Specimen Calibrator
**Model:** haiku
**Depends on:** 0.1 (Input Loader — needs congruence type and product context)
**Produces:** `layer-0/0.2-specimen-calibration.md`
**Purpose:** Classify the bump template type and extract structural patterns from upsell specimens to calibrate generation. No persona voice loading.

---

## INPUT

- `layer-0/0.1-input-context.md` — operating mode, congruence type, product context
- `UPSELL-specimen-index.md` — at `skills/upsells/specimens/system-2/UPSELL-specimen-index.md`
- GATE_0 must be PASS before executing this microskill

---

## PROCESS

1. **Confirm GATE_0 PASS:**
   - Read 0.1 output. If GATE_0 = FAIL, do NOT execute. Halt.

2. **Classify bump template type:**
   - Read congruence_type from 0.1 output
   - Map to one of 4 order bump templates:

   | Template | Trigger Conditions | Core Frame | Specimen References |
   |----------|-------------------|------------|-------------------|
   | Completeness | Congruence type = Completeness. Product completes or extends the FE system. | "Add [X] and get the complete [mechanism] system" | US-14 (complete the set framing), US-10 (same tech, adjacent use) |
   | Accelerator | Congruence type = Accelerator. Product speeds up FE results. | "Get results faster with [X]" | US-08 (one tweak to accelerate), US-11 (approach + short game extension) |
   | Insurance | Congruence type = Insurance. Product protects the investment. | "Protect your investment with [X]" | US-20 (stopping reverses gains — consistency protection) |
   | Exclusive | Congruence type = Exclusive. Product only available at checkout. | "Only available at checkout — [X]" | US-04 (VIP access, checkout-only pricing), US-09 (page-only offer framing) |

3. **Extract structural patterns from relevant specimens:**

   **For ALL templates, extract from short-form specimens (best models for bump brevity):**
   - US-06 (Neuroctane quantity bump, ~1,200 words but contains per-day cost reframing + VIP framing patterns applicable to bump copy)
   - US-16 (SSTS downsell, ~400 words — the shortest specimen, closest to bump format in brevity)
   - US-19 (ProvaSlim quantity bump — template structure with $XX placeholders, supply chain scarcity)
   - US-20 (SwingSmooth Pro quantity bump — per-day cost reframing, consistency argument)

   **Extract these structural elements:**
   - Opening pattern (how the specimen begins — what frame does the first sentence set?)
   - Congruence bridge (exact language that connects upsell to FE)
   - Price presentation (anchor format, savings language, per-day reframing if present)
   - Proof element used (if any — maximum one for bump)
   - Close/CTA language (binary choice phrasing)

   **For template-specific specimens, extract additional patterns:**
   - Completeness: "complete your set" / "they go together" / "bread and butter" system positioning
   - Accelerator: "one tweak" / "get results 2x faster" simplicity framing
   - Insurance: "stopping reverses your gains" / consistency science framing
   - Exclusive: "only on this page" / "checkout-only" urgency framing

4. **Determine which emphasis angles are most natural for this template:**

   | Template | Primary Emphasis | Secondary Emphasis | Least Natural |
   |----------|-----------------|-------------------|---------------|
   | Completeness | Congruence (system completion) | Value (bundled savings) | Urgency (less relevant — it completes, not expires) |
   | Accelerator | Value (results per dollar) | Social proof (others who accelerated) | Exclusivity (accelerators aren't usually exclusive) |
   | Insurance | Urgency (protect now before risk) | Congruence (investment protection) | Social proof (insurance is personal, not social) |
   | Exclusive | Exclusivity (checkout-only) | Urgency (disappears after checkout) | Congruence (exclusivity overrides congruence framing) |

   This matrix guides 1.2 (Variant Generator) — which 5-7 emphasis angles to prioritize.

5. **Confirm: NO persona voice loading for U1.**
   - Soul.md anti-voice constraints apply (from 0.1, if loaded)
   - System 2 persona specimens are NOT loaded
   - Voice at this word count comes from: FE mechanism language + template structure + Soul.md constraints
   - This is not an omission — it is by design. 50-150 words is too short for meaningful voice differentiation.

6. **Write output file**
