# 4.1 — Downsell Validator

**Version:** 1.0
**Skill:** U3 — Downsell Writer
**Microskill:** 4.1 — Downsell Validator
**Layer:** 4
**Model:** sonnet
**Input:** Arena-selected downsell page (the winning output from Layer 2.5 HUMAN_SELECT gate), `layer-1/1.2-congruence-map.md` (mechanism name, root cause language — for exact-match validation), `layer-0/0.1-input-context.md` (FE mechanism name, root cause language — canonical source, declined upsell price)
**Output:** `layer-4/4.1-validation-results.md`

---

## PURPOSE

Run 8 validation gates against the Arena-selected downsell page to confirm structural compliance (word count, ARO sections), reframe quality (genuinely different angle), congruence integrity (mechanism + root cause), tone correctness (zero guilt in Acknowledge), pricing validity (lower than declined upsell), and binary choice presence (clean YES/NO). Gates are PASS or FAIL only — no partial pass, no conditional pass, no invented statuses.

---

## EXECUTION STEPS

1. **Confirm validation target.** Identify the Arena-selected file. Check for GATE_2.5 (HUMAN_SELECT) checkpoint confirming human selection. If no selection checkpoint exists, HALT — cannot validate without Arena completion.

   **CRITICAL:** Validate the ARENA-SELECTED output, NOT the Layer 2 reference draft. This is the most common validation error — checking the wrong file. Confirm the file being validated is the human-selected Arena winner.

2. **Run Gate 1: Word Count (HARD FAIL)**
   - Count total words in the downsell page (excluding metadata, section headers, and formatting marks)
   - PASS: 300-1000 words inclusive
   - FAIL: Under 300 or over 1000
   - This is a HARD FAIL — no exceptions, no "close enough." If the count is 299 or 1001, it fails.
   - Evidence: report exact word count and position-specific target range

3. **Run Gate 2: ARO 3 Sections Present**
   - Scan for all 3 ARO sections: Acknowledge, Reframe, Offer + Choice
   - Each section must be identifiable — either by explicit header, by clear structural break, or by content function
   - Verify ordering: Acknowledge FIRST, then Reframe, then Offer + Choice
   - PASS: All 3 sections present, distinguishable, and in correct order
   - FAIL: Any section missing, two sections merged without distinct identity, or sections out of order
   - Evidence: quote the opening line of each identified section

4. **Run Gate 3: Reframe Check — NOT "Same Thing Cheaper"**
   - Analyze the Reframe section's value proposition
   - Identify the reframe type being executed (Core Extract / Payment Plan / Lite Version / Different Format)
   - Compare the downsell's angle against the declined upsell's angle
   - PASS: Reframe presents a genuinely different angle, entry point, or value proposition. The reader encounters a different REASON to buy, not just a lower price for the same reason.
   - FAIL: Reframe is just the upsell shortened/cheapened with no genuine angle change. "Same product, lower price" without restructured commitment (Payment Plan) = FAIL.
   - Evidence: state the upsell's angle, state the downsell's angle, explain what is different
   - Note: Payment Plan is the borderline case. It PASSES only if the product is genuinely unchanged and the commitment restructuring is explicit (e.g., "3 payments of $X" vs flat price). Payment Plan with a reduced product = not a Payment Plan, it is an undifferentiated discount.

5. **Run Gate 4: Congruence — Mechanism Name**
   - Search for the EXACT mechanism name string from the congruence map
   - Case-sensitive, full name, including articles
   - PASS: Mechanism name appears by exact name in 2+ locations
   - FAIL: Mechanism name missing, paraphrased, or appears only once
   - Evidence: quote each instance with section location
   - Common failures: "this system" or "this protocol" or "the method" used instead of the actual mechanism name

6. **Run Gate 5: Congruence — Root Cause Language**
   - Search for the VERBATIM root cause language string from the congruence map
   - PASS: Root cause language appears verbatim at least once
   - FAIL: Root cause language absent or paraphrased
   - Evidence: quote the instance with section location, or quote the closest paraphrase if failed
   - Example: "hormonal disruption caused by environmental toxins" is not matched by "hormone problems" or "toxin exposure"

7. **Run Gate 6: Tone — Zero Guilt in Acknowledge**
   - Scan the Acknowledge section specifically for guilt, pressure, FOMO, or manipulation language
   - **Violation flags (instant FAIL):**
     - "Wait!" / "Hold on!" / "Before you go!" — pressure
     - "You're missing out" / "Most people regret" / "You'll wish you had" — guilt
     - "Are you sure?" / "Last chance" / "Don't miss" — FOMO
     - "You're leaving money on the table" — manipulation
     - Any sentence implying the buyer made a wrong decision
   - **Also scan Reframe and Offer sections** for pressure leakage (guilt that migrated out of Acknowledge into other sections)
   - PASS: Zero guilt/pressure/FOMO language in Acknowledge section AND no pressure leakage in other sections
   - FAIL: Any violation flag detected anywhere in the downsell
   - Evidence: quote each flagged instance with section location and classification

8. **Run Gate 7: Price — Lower Than Declined Upsell**
   - Extract the downsell price from the Offer section
   - Compare against the declined upsell price from `0.1-input-context.md`
   - PASS: Downsell price < declined upsell price
   - FAIL: Downsell price >= declined upsell price
   - Evidence: state both prices and the relationship
   - Note: If the downsell price is not clearly stated in the copy, that is also a FAIL (price must be explicit)

9. **Run Gate 8: Binary Choice — Clean YES/NO**
   - Check for BOTH yes and no options
   - YES option: contains downsell product name + price + action verb
   - NO option: clean decline with zero guilt language
   - **Scan the NO option specifically for manipulation:**
     - "No, I don't want to [desirable thing]" = FAIL
     - "No, I'm fine being [undesirable thing]" = FAIL
     - "No, I'll pay full price later" = FAIL
     - "No, I'd rather struggle with [problem]" = FAIL
     - "No thanks, I'll continue with just my original purchase" = PASS
     - "No thanks, I'm happy with just the [FE Product]" = PASS
   - PASS: Both options present, NO option is clean
   - FAIL: Either option missing OR NO option contains guilt/manipulation
   - Evidence: quote both YES and NO options verbatim

10. **Compile validation report.** Summary table + detailed evidence for each gate.

11. **Determine overall status:**
    - All 8 gates PASS → proceed to 4.2 (Output Packager)
    - Any gate FAIL → document failures with specific revision directives, return to appropriate layer
    - Multiple gates FAIL → prioritize structural failures (Gate 1, Gate 2) over content failures (Gates 3-8)

## CONSTRAINTS

- **Validate the Arena-selected output, not the Layer 2 draft.** This is the #1 validation error. Confirm the validation target is the human-selected Arena winner before running any gate.
- **Gates are PASS or FAIL.** No partial pass. No "PASS with notes." No "conditional pass." PASS or FAIL. Binary.
- **Gate 1 (word count) is a HARD FAIL with zero tolerance.** 299 words = FAIL. 1001 words = FAIL. Do not rationalize "it's close enough."
- **Gate 4 requires the EXACT mechanism name string.** Case-sensitive, full name, including articles. "The Metabolic Switch Protocol" is not matched by "metabolic switch" or "the Metabolic Switch" (missing "Protocol").
- **Gate 6 flag list is not exhaustive.** If you encounter guilt/pressure language not on the list, flag it. The list provides common examples, not a complete dictionary. Use judgment: does this sentence make the buyer feel bad about declining? If yes, it is a violation.
- **Gate 8 NO option analysis must be strict.** Any framing that makes the buyer feel bad, stupid, or irresponsible for declining is a FAIL. When in doubt, it fails. Clean means clean.
- **Revision directives must be SPECIFIC.** Not "fix the reframe" but "Reframe section presents the same product at a lower price without a different angle. The upsell offered [X] at $[Y]. The downsell offers [X] at $[Z]. This is an undifferentiated discount. Reframe must change the angle: isolate a component (Core Extract), restructure payments (Payment Plan), reduce scope (Lite Version), or change format (Different Format)." The revision agent needs to know exactly what is wrong and what "fixed" looks like.

## FAILURE MODES

| Failure | Detection | Response |
|---------|-----------|----------|
| Validating Layer 2 draft instead of Arena output | File path does not match Arena winner or GATE_2.5 checkpoint missing | HALT. Locate Arena-selected file. Cannot validate without Arena completion. |
| Gate status invented ("CONDITIONAL_PASS") | Status value is anything other than PASS or FAIL | Rewrite to PASS or FAIL. No invented statuses. Binary. |
| Revision directives are vague | Directives say "fix the tone" or "improve congruence" without specifics | Rewrite with: (a) exact location (section + line), (b) what is wrong (quote the problem), (c) what "fixed" looks like (provide suggestion). |
| Multiple gates fail but no prioritization | Report lists failures without triage | Prioritize: structural failures (Gates 1, 2) first, then congruence (Gates 4, 5), then quality (Gates 3, 6, 7, 8). |
