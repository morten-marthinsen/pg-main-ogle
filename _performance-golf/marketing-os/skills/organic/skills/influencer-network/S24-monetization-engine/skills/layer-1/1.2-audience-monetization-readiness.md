# 1.2 Audience Monetization Readiness

## Purpose
Assess when audience is ready for monetization. Define readiness criteria, evaluate current status, create timeline to readiness. Prevent premature monetization that kills audience trust.

## Model Assignment
**Model:** claude-opus-4-20250514

## Input Requirements
- Upstream context summary (audience performance data)
- Revenue model mapping (from 1.1)
- Teaching summary (audience readiness frameworks)

## Execution Protocol

### MUST
- Define 6 readiness criteria per persona: follower threshold (1K+ engaged), engagement rate (2%+), trust signals, content consistency (3+ months), value demonstration, audience requests
- Assess current status against each criterion
- Identify gaps (which criteria not yet met)
- Create timeline to readiness (when will criteria be met)
- Document early monetization risks (what happens if monetized too soon)
- Differentiate readiness by revenue model (some models need more trust than others)
- Create phased monetization plan (what to launch when)
- Include trust-building strategies to accelerate readiness

### NEVER
- Skip readiness assessment (assume audience is ready)
- Use generic thresholds without persona-specific adaptation
- Ignore engagement quality (not just follower count)
- Plan monetization before readiness criteria met
- Forget to document early monetization risks
- Create identical readiness criteria across all personas

## Output
**Filename:** `audience-monetization-readiness.md`

**Format:** Markdown readiness assessment

**Minimum Size:** 5KB

**Required Sections:**
1. Per-Persona Readiness Criteria (6 criteria with thresholds)
2. Current Status Assessment (where each persona stands on each criterion)
3. Gaps Identified (which criteria not yet met, by how much)
4. Timeline to Readiness (when will criteria be met per persona)
5. Early Monetization Risks (what happens if monetized too soon)
6. Model-Specific Readiness (different models need different readiness levels)
7. Phased Monetization Plan (what to launch when based on readiness)
8. Trust-Building Accelerators (strategies to reach readiness faster)

## Quality Gates
- [ ] 6 readiness criteria defined per persona
- [ ] Current status assessed against all criteria
- [ ] Gaps identified with specific metrics
- [ ] Timeline to readiness realistic (not overly optimistic)
- [ ] Early monetization risks documented
- [ ] Phased plan prevents premature monetization
- [ ] Trust-building strategies included

## Handoff
Feeds into: 1.3 Product Development Strategy (readiness informs what products to develop)
