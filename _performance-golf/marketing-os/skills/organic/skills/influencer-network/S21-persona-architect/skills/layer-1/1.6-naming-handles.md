# 1.6 Naming and Handles

## Purpose
Name each persona and generate social media handles. Test for availability, memorability, niche relevance. Minimum 3 name options per persona to provide choice.

## Model Assignment
**Model:** claude-opus-4-20250514

## Input Requirements
- Approved identity concepts (from 1.3, post-checkpoint)
- Backstory development (from 1.4)
- Personality framework (from 1.5)
- Niche ownership map (from 1.1)

## Execution Protocol

### MUST
- Generate minimum 3 full name options per persona
- Create 3-5 social media handle variations per name
- Test handle availability on major platforms (Twitter/X, Instagram, TikTok, LinkedIn)
- Assess memorability (easy to spell, pronounce, remember)
- Evaluate niche relevance (does name signal expertise area?)
- Consider SEO and discoverability
- Ensure names feel authentic for persona's background/age/culture
- Provide rationale for each name option

### NEVER
- Provide only one name option per persona
- Skip handle availability testing
- Create handles that are too long or complex
- Ignore cultural authenticity when naming
- Use names that reveal parent brand connection

## Output
**Filename:** `naming-handles.md`

**Format:** Markdown naming options document

**Minimum Size:** 5KB

**Required Sections:**
1. Per-Persona Naming Options (3+ names per persona with rationale)
2. Handle Variations (3-5 handle options per name with availability status)
3. Memorability Assessment (scoring each option on recall, spelling, pronunciation)
4. Niche Relevance Scoring (how well name signals expertise)
5. Recommended Selection (top choice per persona with justification)

## Quality Gates
- [ ] Minimum 3 name options per persona
- [ ] Handle availability tested on 4+ major platforms
- [ ] Memorability assessed for each option
- [ ] Niche relevance evaluated
- [ ] Cultural authenticity verified
