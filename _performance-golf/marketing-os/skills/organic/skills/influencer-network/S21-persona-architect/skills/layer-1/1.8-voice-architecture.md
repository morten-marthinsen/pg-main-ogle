# 1.8 Voice Architecture

## Purpose
Formalize voice for each persona: tone spectrum, vocabulary, phrases they'd use, phrases they'd NEVER use, energy level, formality spectrum. Builds per-persona Brand Voice Framework (BVF).

## Model Assignment
**Model:** claude-opus-4-20250514

## Input Requirements
- Personality framework (from 1.5)
- Approved names (from 1.6)
- Brand voice constraints (from upstream context)
- Archetype assignments (from 1.2)

## Execution Protocol

### MUST
- Define tone spectrum: where persona lands on warm/cool, formal/casual, serious/playful, direct/nuanced
- Specify vocabulary level: sophistication, jargon usage, slang, technical language
- Document signature phrases: 10+ phrases they frequently use
- List forbidden phrases: 10+ expressions they'd NEVER use
- Define energy level: high/medium/low, consistent or variable
- Specify sentence structure preferences: short/punchy vs long/flowing, simple vs complex
- Create voice differentiation from other personas
- Respect parent brand voice constraints while allowing persona distinctiveness

### NEVER
- Create identical voices across personas
- Define voice solely by tone (must include vocabulary, phrases, structure)
- Ignore brand voice constraints from BVF
- Skip "never use" list (critical for maintaining boundaries)
- Create voice that contradicts personality or backstory

## Output
**Filename:** `voice-architecture.md`

**Format:** Markdown voice specifications (per-persona BVF)

**Minimum Size:** 7KB

**Required Sections:**
1. Per-Persona Voice Profiles (tone, vocabulary, energy, structure)
2. Signature Phrases (10+ per persona)
3. Forbidden Phrases (10+ per persona)
4. Voice Differentiation Matrix (how personas differ in voice)
5. Brand Alignment Check (how each respects parent brand constraints)

## Quality Gates
- [ ] Each persona has complete voice profile across 6+ dimensions
- [ ] Signature phrases list includes minimum 10 expressions
- [ ] Forbidden phrases list includes minimum 10 expressions
- [ ] Voice differentiation across personas verified
- [ ] Brand voice constraints respected
