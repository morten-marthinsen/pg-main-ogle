# 1.5 Cross-Promotion Map

## Purpose
Plan how personas reference, share, and amplify each other without breaking character or revealing network connection. Must look organic, not coordinated.

## Model Assignment
**Model:** claude-opus-4-20250514

## Input Requirements
- Network Overview (relationship map, brand firewall rules)
- Platform selections (from 1.1)
- Content calendar frameworks (from 1.3)
- Persona Bibles (voice, personality)

## Execution Protocol

### MUST
- Design cross-promotion frequency limits (maximum 1-2 per persona per week to maintain authenticity)
- Create varied interaction types: share, comment, quote, respond, collaborate
- Define authentic value rationale: why would this persona genuinely engage with this content?
- Use persona-specific language when referencing others (never identical wording)
- Stagger timing across personas (no simultaneous engagement patterns)
- Maintain brand firewall (personas never reveal they're part of same network)
- Create content discovery narratives (how did persona "find" this content?)
- Design reciprocal but varied patterns (not always symmetric)

### NEVER
- Have all personas share same content simultaneously
- Use identical language across personas when cross-promoting
- Create obvious coordination patterns (e.g., every Tuesday Persona A shares Persona B)
- Make personas constantly tag/mention each other
- Reveal network connection in any way
- Force cross-promotion that doesn't fit persona's niche/voice

## Output
**Filename:** `cross-promotion-map.md`

**Format:** Markdown cross-promotion playbook

**Minimum Size:** 4KB

**Required Sections:**
1. Cross-Promotion Frequency Limits (per persona, per platform)
2. Interaction Type Variations (share, comment, quote, respond — when to use each)
3. Authentic Value Rationale (why personas would genuinely engage with each other's content)
4. Language Variation Guidelines (how to reference others without identical wording)
5. Timing Stagger Strategy (avoid simultaneous engagement patterns)
6. Brand Firewall Maintenance (how to cross-promote without revealing connection)
7. Discovery Narratives (how personas "find" each other's content naturally)

## Quality Gates
- [ ] Frequency limits defined (maximum 1-2 per persona per week)
- [ ] Multiple interaction types specified (not just shares)
- [ ] Authentic value rationale for cross-promotion documented
- [ ] Language variation guidelines prevent identical wording
- [ ] Timing stagger strategy prevents coordination patterns
- [ ] Brand firewall maintenance protocols clear
