# 1.7 Amplification Sequences

## Purpose
Design strategic engagement sequences where personas amplify each other at key moments. Create momentum through coordinated (but authentic-appearing) amplification while maintaining firewall and authenticity.

## Model Assignment
**Model:** claude-opus-4-20250514

## Input Requirements
- Engagement choreography (from 1.2)
- Network calendar sync (from 1.1)
- Brand firewall protocol (from 1.5)
- Cross-persona campaigns (from 1.4)

## Execution Protocol

### MUST
- Identify amplification opportunities: which content warrants network-level amplification
- Design amplification sequences: who amplifies when, in what order, with what timing
- Create variation in amplification types: share, comment, quote, respond, collaborate
- Ensure timing stagger: minimum 4-hour gaps, spread over 24-72 hours
- Maintain authenticity: language variation, realistic participation rates, missed opportunities
- Verify firewall integrity: amplification doesn't reveal coordination
- Include measurement: how to assess amplification impact
- Design amplification frequency limits: not every post gets amplified

### NEVER
- Amplify every post (reduces authenticity, signals coordination)
- Create simultaneous amplification (obvious coordination pattern)
- Use identical amplification language across personas
- Skip timing stagger (makes coordination obvious)
- Forget authenticity maintenance (participation rates, missed opportunities)
- Ignore firewall during amplification
- Design amplification without measurement plan

## Output
**Filename:** `amplification-sequences.md`

**Format:** Markdown amplification playbook

**Minimum Size:** 5KB

**Required Sections:**
1. Amplification Opportunity Criteria (what content warrants network-level amplification)
2. Sequence Design (who amplifies when, timing patterns, order variations)
3. Amplification Type Mix (share, comment, quote, respond — when to use each)
4. Timing Stagger Strategy (gaps, spread patterns, variation)
5. Authenticity Maintenance (language variation, participation rates, missed opportunities)
6. Firewall Verification (amplification doesn't reveal coordination)
7. Measurement Plan (how to assess amplification impact)
8. Frequency Limits (not every post amplified — selection criteria)

## Quality Gates
- [ ] Amplification criteria defined (not every post)
- [ ] Sequence design includes timing stagger (4+ hour gaps)
- [ ] Amplification types diverse (not just shares)
- [ ] Authenticity maintained (language variation, realistic participation)
- [ ] Firewall verified (no coordination reveals)
- [ ] Measurement plan operational
- [ ] Frequency limits enforced
