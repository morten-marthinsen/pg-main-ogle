# 4.2 Network Dashboard

## Purpose
Create real-time network coordination view showing all persona activity, coordination status, firewall integrity, and performance metrics. Operational dashboard for day-to-day network management.

## Model Assignment
**Model:** claude-3-5-sonnet-20241022

## Input Requirements
- Network Coordination Plan (from 4.1)
- Network Calendar Sync (from 1.1)
- All Layer 1 coordination outputs

## Execution Protocol

### MUST
- Create dashboard sections: Network Activity, Coordination Status, Firewall Integrity, Performance Metrics, Alerts
- Design Network Activity view: who's posting when, what content type, which platform
- Build Coordination Status tracker: active campaigns, narrative arcs in progress, amplification sequences
- Create Firewall Integrity monitor: cross-promotion frequency check, discovery narrative audit, breach detection
- Design Performance Metrics view: follower growth per persona, engagement rates, network reach
- Include Alert System: coordination conflicts, firewall breaches, performance drops, missed choreography
- Make dashboard updateable: how to refresh data daily/weekly
- Create visualization recommendations: what should be graphed/charted

### NEVER
- Create static dashboard that can't be updated
- Skip alert system (critical for proactive management)
- Forget firewall integrity monitoring
- Omit performance metrics per persona
- Create dashboard without visualization recommendations

## Output
**Filename:** `network-dashboard.md`

**Format:** Markdown operational dashboard specification

**Minimum Size:** 8KB

**Required Sections:**
1. Dashboard Overview (purpose, update frequency, key sections)
2. Network Activity View (who posts what when, across all personas/platforms)
3. Coordination Status Tracker (active campaigns, narrative arcs, amplification sequences)
4. Firewall Integrity Monitor (cross-promotion frequency, discovery narratives, breach detection)
5. Performance Metrics (follower growth, engagement, reach per persona and network-wide)
6. Alert System (conflicts, breaches, performance drops, missed choreography — with triggers)
7. Update Protocol (how to refresh dashboard daily/weekly)
8. Visualization Recommendations (what to graph/chart for quick insights)

## Quality Gates
- [ ] Dashboard minimum 8KB with all required sections
- [ ] Network activity view covers all personas and platforms
- [ ] Coordination status tracker includes active orchestration
- [ ] Firewall integrity monitor operational
- [ ] Performance metrics per persona and network-wide
- [ ] Alert system with specific triggers defined
- [ ] Update protocol clear and actionable
