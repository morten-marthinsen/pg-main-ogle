# 1.5 Learning Flag Generation

## Purpose
Generate structured learning flags for S20 Learning Capture. Categories: teaching_update, specimen_addition, skill_refinement, gate_adjustment. Each flag has evidence, confidence, and recommended action.

## Assigned Model
`claude-opus-4` (Layer 1 analysis)

## Input
- 1.4-pattern-detection.json
- 1.3-variance-analysis.json
- 1.2-prediction-comparison.json
- 0.2-teaching-context.json (learning flag taxonomy)

## Process

### MUST
1. **Generate teaching_update flags** — Where patterns suggest scoring models/teaching files need updates
   - Include: which teaching file, specific update recommended, evidence (pattern IDs), confidence
2. **Generate specimen_addition flags** — High-performing content worth adding to specimen library
   - Include: content_id, why exemplary, performance data, specimen category
3. **Generate skill_refinement flags** — Where skill execution (S06/S13) needs process improvements
   - Include: which skill, what refinement, evidence, expected impact
4. **Generate gate_adjustment flags** — Where pass/fail thresholds may need recalibration
   - Include: which gate, current threshold, recommended new threshold, justification
5. **Prioritize flags** — Rank by: impact (high/medium/low), confidence (high/medium/low), urgency
6. **Require evidence** — Every flag must cite specific content_ids, patterns, or variance data
7. **Assign confidence** — High (≥3 data points), medium (2 data points), low (1 data point/hypothesis)

### NEVER
- Generate flags without evidence
- Skip prioritization
- Flag single-instance findings as "teaching_update" (insufficient data)
- Generate vague recommendations ("make S06 better")

## Quality Gates
1. Every flag has evidence
2. Confidence assigned
3. Priority ranked
4. Recommended actions specific

## Handoff
