# 0.1 Input Validator

## Purpose
Validate that all required performance data exists, content IDs match published content, and minimum sample size thresholds are met before analysis begins.

## Assigned Model
`claude-haiku-4` (Layer 0 validation)

## Input
- Performance data file (CSV/JSON with platform metrics)
- Published content inventory from S14/S15
- Original predictions from S06/S13

## Process

### MUST
1. **Verify performance data exists** — Check file path, non-empty, parseable format
2. **Validate content_id alignment** — Every content_id in performance data matches a published piece from S14/S15
3. **Check minimum sample size** — Require minimum 5 published pieces with at least 7 days live before analysis
4. **Verify metric completeness** — Each content piece has: views, engagement_rate, reach, impressions. Missing data flagged.
5. **Validate prediction files** — Original virality predictions from S06/S13 exist for all analyzed content
6. **Check timestamp validity** — Performance data timestamps are AFTER publish dates
7. **Output validation result** — Binary PASS/FAIL with specific error messages if FAIL

### NEVER
- Proceed with missing content_id matches
- Accept performance data <7 days after publish (insufficient maturity)
- Skip metric completeness check
- Analyze content without original predictions

## Output
**File:** `0.1-validation-result.json`

```json
{
  "validation_status": "PASS|FAIL",
  "timestamp": "ISO8601",
  "performance_data_file": "path",
  "content_pieces_found": 12,
  "content_pieces_matched": 12,
  "minimum_sample_met": true,
  "minimum_days_live_met": true,
  "metric_completeness": "100%",
  "prediction_files_found": true,
  "errors": []
}
```

## Quality Gates
1. All content IDs matched
2. Minimum 5 pieces validated
3. All pieces live ≥7 days
4. Prediction files present

## Handoff
Passes to 0.2-teaching-loader.md if PASS, halts if FAIL.
