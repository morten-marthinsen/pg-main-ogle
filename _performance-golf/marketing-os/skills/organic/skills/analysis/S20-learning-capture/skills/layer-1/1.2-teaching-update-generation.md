# 1.2 Teaching Update Generation

## Purpose
For teaching_update learnings with High confidence, draft specific YAML changes to teaching files. Include before/after. Require minimum 3 data points to update a teaching.

## Assigned Model
`claude-opus-4` (Layer 1 intelligence)

## Input
- 1.1-learning-categories.json (teaching_update flags marked "act_now")
- 0.2-teaching-context.json (current teaching YAML files)
- S19 patterns and evidence

## Process

### MUST
1. **For each High-confidence teaching_update flag:**
   - Identify target teaching file and section
   - Draft specific YAML changes (add/modify/delete)
   - Show before/after snippets
   - Cite evidence (≥3 data points required)
   - Increment teaching file version number
2. **Follow YAML update patterns:**
   - For scoring updates: adjust baseline values, add sub-factors, modify weights
   - For hook type baselines: update expected performance ranges
   - For pattern detection: add new pattern types, adjust thresholds
   - For platform teachings: update normalization formulas
3. **Validate update logic:**
   - Does update align with evidence?
   - Will update improve prediction accuracy?
   - Are there unintended side effects?
4. **Write update to file** — ACTUALLY modify the teaching YAML file
5. **Log update** — Record what changed, why, evidence used
6. **Output teaching updates** — Summary of all updates made

### NEVER
- Update teaching based on <3 data points
- Make speculative updates without evidence
- Skip before/after documentation
- Forget to increment version number
- Draft update but not apply it (propagation required)

## Quality Gates
1. Minimum 3 data points per update
2. Before/after documented
3. Updates actually written to files
4. Version numbers incremented
5. Propagation status COMPLETE
