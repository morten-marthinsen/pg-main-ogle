# 4.1 Learning Log Assembler

## Purpose
Assemble learning log entry with all categorized learnings, updates made, propagation status. Output to learning-log/ directory.

## Assigned Model
`claude-sonnet-4` (Layer 4 packaging)

## Input
- 1.1-learning-categories.json
- 1.2-teaching-updates.yaml
- 1.3-specimen-promotions.json
- 1.4-prediction-calibration.json
- 1.5-gate-threshold-review.json
- 1.6-propagation-verification.json
- S19 PAR metadata

## Process

### MUST
1. **Assemble learning log entry structure:**
   - **Header:** learning_id, date, source campaign, PAR reference
   - **Summary:** learning count, category distribution, confidence distribution
   - **Detailed Learnings:** Each learning with:
     - Learning type, content, evidence (citations), confidence, impact
     - Action required, action taken, propagation status, target files
   - **Teaching Updates:** All teaching file modifications with before/after
   - **Specimen Additions:** All promoted specimens with performance data
   - **Gate Adjustments:** All threshold changes with justification
   - **Deferred Learnings:** Any learnings deferred with revisit triggers
   - **System State:** Updated specimen counts, teaching versions, gate thresholds
2. **Verify completeness:**
   - All learnings from 1.1 addressed
   - All propagation statuses resolved (no PENDING)
   - All deferred learnings have revisit triggers
3. **Check minimum size:** Learning log should be ≥3KB (indicates depth)
4. **Write to learning-log/ directory:** File named `learning-log-[campaign_id]-[date].md`
5. **Output both JSON and markdown** — JSON for machine readability, markdown for human review

### NEVER
- Omit any learning from S19
- Leave propagation status as PENDING
- Skip deferred learnings section
- Accept <3KB output (insufficient detail)
- Forget to write file to learning-log/ directory
