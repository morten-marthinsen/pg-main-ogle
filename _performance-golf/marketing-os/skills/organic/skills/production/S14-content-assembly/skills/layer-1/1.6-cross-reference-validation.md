# 1.6 Cross-Reference Validation

## Purpose
Final Layer 1 check — verify all CBF requirements are met, check content calendar coverage, validate objective alignment, and flag any gaps before proceeding to assembly.

---

## Input Requirements

From Layer 0:
- `0.4-cbf-context.yaml` (campaign requirements)

From Layer 1:
- `1.1-arena-verification.yaml` (Arena status)
- `1.2-content-inventory.yaml` (all content)
- `1.3-virality-scores.yaml` (quality scores)
- `1.4-platform-compliance.yaml` (platform status)
- `1.5-voice-consistency.yaml` (voice status)

---

## Execution Protocol

### Step 1: Validate All CBF Content Requirements Met

From CBF (via 0.4), check every required content piece has been produced:

```yaml
cbf_requirements_check:
  total_content_pieces_required: [Number from CBF]
  total_content_pieces_produced: [Number from inventory]
  all_requirements_met: [Yes/No]

  required_content:
    - content_id: "[ID from CBF]"
      content_type: "[Type]"
      platform: "[Platform]"
      produced: [Yes/No]
      status: [Ready/Issues/Missing]
      issues_summary: "[From Layer 1 checks]"

  missing_content:
    - content_id: "[ID from CBF]"
      content_type: "[Type]"
      platform: "[Platform]"
      reason_missing: "[Why not produced]"
      impact: "[High/Medium/Low]"

  orphaned_content:
    - content_id: "[ID not in CBF]"
      content_type: "[Type]"
      platform: "[Platform]"
      recommendation: "[Include or discard?]"
```

**If missing_content is non-empty:** Flag HIGH priority gap.

---

### Step 2: Validate Content Calendar Coverage

Check content calendar from CBF is fully covered:

```yaml
calendar_coverage_check:
  campaign_duration:
    start_date: "[From CBF]"
    end_date: "[From CBF]"
    total_weeks: [Number]

  planned_schedule:
    week_1:
      slots: [Number of publishing slots]
      content_assigned: [Number]
      coverage: [%]
      gaps: []

    week_2:
      # [... etc for each week]

  overall_coverage:
    total_slots: [Number]
    slots_filled: [Number]
    coverage_percentage: [%]
    gaps_count: [Number]

  scheduling_gaps:
    - week: [Number]
      day: "[Day]"
      time_slot: "[Time]"
      reason: "[Why no content assigned]"
      impact: "[High/Medium/Low]"
```

**If coverage < 80%:** Flag for human review (incomplete calendar).

---

### Step 3: Validate Objective Alignment

Check content aligns with campaign objectives:

```yaml
objective_alignment_check:
  primary_goal: "[From CBF]"
  secondary_goals: []

  content_supporting_primary_goal:
    - content_id: "[ID]"
      how_it_supports: "[Description]"
      alignment_strength: [Strong/Moderate/Weak]

  content_supporting_secondary_goals:
    [goal]:
      - content_id: "[ID]"
        how_it_supports: "[Description]"

  objectives_without_content_support: []

  orphaned_objectives:
    - objective: "[Objective from CBF]"
      content_count: 0
      impact: "[No content supports this objective]"
      recommendation: "[Add content or revise objectives]"
```

**If any objective has zero content supporting it:** Flag gap.

---

### Step 4: Validate Content Function Distribution

Check distribution matches CAF targets (via CBF):

```yaml
function_distribution_check:
  target_distribution:
    awareness: "[% from CAF]"
    engagement: "[% from CAF]"
    conversion: "[% from CAF]"
    community: "[% from CAF]"

  actual_distribution:
    awareness: "[% from inventory]"
    engagement: "[% from inventory]"
    conversion: "[% from inventory]"
    community: "[% from inventory]"

  distribution_gaps:
    - function: "[Function]"
      target: "[%]"
      actual: "[%]"
      gap: "[Difference]"
      severity: [HIGH if >15%, MEDIUM if 10-15%, LOW if <10%]

  distribution_acceptable: [Yes/No - if all gaps < 15%]
```

**If any gap > 15%:** Flag HIGH priority imbalance.

---

### Step 5: Validate Quality Gate Passage

Aggregate all Layer 1 quality checks:

```yaml
quality_gates_aggregate:
  gate_g08_arena_verification:
    status: [PASS/FAIL from 1.1]
    issues: []

  gate_g09_virality_threshold:
    status: [PASS/FAIL from 1.3]
    pieces_below_threshold: [Count]
    blocked_pieces: []

  platform_compliance:
    status: [PASS/FAIL from 1.4]
    non_compliant_pieces: [Count]
    critical_failures: []

  voice_consistency:
    status: [PASS/FAIL from 1.5]
    voice_violations: [Count]
    critical_violations: []

  all_gates_passed: [Yes/No]
  gates_failed: []
  blocking_issues_count: [Number]
```

**If any gate = FAIL:** Assembly CANNOT proceed.

---

### Step 6: Create Layer 1 Summary

```yaml
layer_1_summary:
  verification_complete: [Yes/No]

  content_status:
    total_pieces: [Number]
    ready_for_assembly: [Number]
    flagged_for_review: [Number]
    blocked: [Number]

  critical_issues:
    - issue_type: "[Type]"
      affected_content: []
      required_action: "[What must be fixed]"
      blocking: [Yes/No]

  warnings:
    - issue_type: "[Type]"
      affected_content: []
      recommendation: "[What should be considered]"
      blocking: No

  gaps:
    missing_content: [Count]
    calendar_gaps: [Count]
    objective_gaps: [Count]
    distribution_gaps: [Count]

  assembly_readiness:
    ready_to_proceed_to_layer_4: [Yes/No]
    blocking_issues_count: [Number]
    blocking_issues_detail: []

  next_steps: |
    [If ready: Proceed to Layer 4]
    [If not ready: List all required fixes before assembly can proceed]
```

**Minimum size:** 3KB

---

## Quality Gates

- [ ] All CBF content requirements verified
- [ ] Content calendar coverage checked
- [ ] Objective alignment validated
- [ ] Function distribution validated
- [ ] All Layer 1 quality gates aggregated
- [ ] Assembly readiness determined

**Ready for assembly IF:**
- All CBF requirements met (or gaps acknowledged and accepted)
- All quality gates passed (G08, G09, compliance, voice)
- No blocking issues

**NOT ready for assembly IF:**
- Missing required content
- Any quality gate failed
- Any blocking issues unresolved

---

## Handoff

Pass validation results to: **Layer 4 Assembly** (if ready) or **Human Review** (if blocked)

**Create checkpoint:**
```yaml
# checkpoints/LAYER_1_COMPLETE.yaml
skill: "S14-content-assembly"
layer: 1
status: [COMPLETE / BLOCKED]
timestamp: "[ISO 8601]"

verification_results:
  arena_verified: [Yes/No]
  virality_threshold_met: [Yes/No]
  platform_compliant: [Yes/No]
  voice_consistent: [Yes/No]
  cbf_requirements_met: [Yes/No]

ready_for_assembly: [Yes/No]
blocking_issues_count: [Number]
blocking_issues: []

next_action: "[Proceed to Layer 4 / Resolve blocking issues]"
```

**If ready_for_assembly = No:** DO NOT proceed to Layer 4. Display blocking issues and required actions to human.
