# 1.2 Content Inventory

## Purpose
Create a comprehensive inventory of all content pieces with their metadata, Arena status, platform assignments, and format types. This provides a master list for assembly and distribution.

---

## Input Requirements

From Layer 0:
- `0.3-production-outputs.yaml` (all loaded content)
- `0.4-cbf-context.yaml` (campaign requirements)

From Layer 1:
- `1.1-arena-verification.yaml` (verification status)

---

## Execution Protocol

### Step 1: List All Content Pieces

For each content piece in production outputs:

```yaml
content_pieces:
  - content_id: "[ID]"
    content_title: "[Title/working name]"
    content_type: "[Reel/Carousel/Thread/Long-form/etc]"
    platform: "[Primary platform]"
    secondary_platforms: []
    content_function: "[Awareness/Engagement/Conversion/Community]"
    pillar: "[From CAF via CBF]"
    series: "[If part of series]"

    status:
      arena_verified: [Yes/No from 1.1]
      production_complete: [Yes/No - all files present]
      ready_for_assembly: [Yes/No]

    files:
      script: "[Path or N/A]"
      caption: "[Path]"
      carousel_slides: "[Path or N/A]"
      thread_tweets: "[Path or N/A]"
      visual_direction: "[Path]"
      arena_synthesis: "[Path]"

    metadata:
      word_count: [Number]
      character_count: [Number]
      estimated_duration_seconds: [Number if video]
      slide_count: [Number if carousel]
      tweet_count: [Number if thread]
```

---

### Step 2: Categorize by Content Type

Group content by type for easier assembly:

```yaml
content_by_type:
  video_content:
    count: [Number]
    pieces:
      - content_id: "[ID]"
        platform: "[Platform]"
        format: "[Reel/TikTok/Short]"
        duration: [Seconds]

  carousel_content:
    count: [Number]
    pieces:
      - content_id: "[ID]"
        platform: "[Platform]"
        slide_count: [Number]

  thread_content:
    count: [Number]
    pieces:
      - content_id: "[ID]"
        platform: "[Platform]"
        tweet_count: [Number]

  static_post_content:
    count: [Number]
    pieces:
      - content_id: "[ID]"
        platform: "[Platform]"
        format: "[Single image/text post]"
```

---

### Step 3: Categorize by Platform

Group content by platform for distribution planning:

```yaml
content_by_platform:
  instagram:
    total_pieces: [Number]
    by_format:
      reels: [Number]
      carousels: [Number]
      static_posts: [Number]
    pieces:
      - content_id: "[ID]"
        format: "[Format]"

  tiktok:
    total_pieces: [Number]
    pieces:
      - content_id: "[ID]"
        format: "[Format]"

  youtube:
    total_pieces: [Number]
    by_format:
      shorts: [Number]
      long_form: [Number]
    pieces:
      - content_id: "[ID]"
        format: "[Format]"

  x_twitter:
    total_pieces: [Number]
    pieces:
      - content_id: "[ID]"
        format: "[Thread/Single tweet]"

  linkedin:
    total_pieces: [Number]
    pieces:
      - content_id: "[ID]"
        format: "[Format]"

  facebook:
    total_pieces: [Number]
    pieces:
      - content_id: "[ID]"
        format: "[Format]"
```

---

### Step 4: Categorize by Function

Group by content function (from CAF via CBF):

```yaml
content_by_function:
  awareness:
    count: [Number]
    percentage: [% of total]
    pieces: []

  engagement:
    count: [Number]
    percentage: [% of total]
    pieces: []

  conversion:
    count: [Number]
    percentage: [% of total]
    pieces: []

  community:
    count: [Number]
    percentage: [% of total]
    pieces: []
```

**Compare to CAF targets:**
```yaml
function_distribution_analysis:
  target_from_caf:
    awareness: [Target %]
    engagement: [Target %]
    conversion: [Target %]
    community: [Target %]

  actual_produced:
    awareness: [Actual %]
    engagement: [Actual %]
    conversion: [Actual %]
    community: [Actual %]

  gaps:
    - function: "[Function if gap]"
      target: [%]
      actual: [%]
      gap: [Difference]
```

**If gaps > 10%:** Flag for human review (distribution imbalance).

---

### Step 5: Categorize by Pillar

Group by content pillar (from CAF):

```yaml
content_by_pillar:
  [pillar_name]:
    count: [Number]
    pieces:
      - content_id: "[ID]"
        content_type: "[Type]"
        platform: "[Platform]"
```

**Verify pillar balance:**
```yaml
pillar_distribution:
  - pillar: "[Name]"
    count: [Number]
    percentage: [% of total]
    balanced: [Yes/No - roughly equal distribution expected]
```

---

### Step 6: Identify Gaps and Issues

Flag any problems:

```yaml
inventory_issues:
  incomplete_content:
    - content_id: "[ID]"
      missing_files: []
      impact: "[Cannot assemble without these]"

  arena_verification_failed:
    - content_id: "[ID]"
      issue: "[From 1.1]"
      impact: "[Cannot use this content]"

  unbalanced_distribution:
    - dimension: "[Platform/Function/Pillar]"
      issue: "[Description of imbalance]"
      recommendation: "[What to do]"

  orphaned_content:
    - content_id: "[ID]"
      issue: "[Not in CBF, why produced?]"
      recommendation: "[Include or discard?]"
```

---

### Step 7: Create Assembly Readiness Summary

```yaml
assembly_readiness:
  total_content_pieces: [Number]
  ready_for_assembly: [Number]
  blocked: [Number]
  issues_requiring_resolution: [Number]

  ready_pieces: []
  blocked_pieces:
    - content_id: "[ID]"
      reason: "[Why blocked]"
      required_action: "[What needs to happen]"

  overall_status: [READY / PARTIAL / BLOCKED]
```

**Overall status:**
- **READY:** All content verified and complete
- **PARTIAL:** Some content ready, some blocked
- **BLOCKED:** Critical issues prevent assembly

---

## Quality Gates

- [ ] All content pieces inventoried
- [ ] All content categorized by type, platform, function, pillar
- [ ] Distribution analysis complete
- [ ] Gaps identified and flagged
- [ ] Assembly readiness determined for each piece
- [ ] Overall status determined

**If overall_status = BLOCKED:** Proceed to Layer 1 checks to gather full diagnostic, but flag that assembly cannot complete without resolution.

---

## Handoff

Pass content inventory to: **1.3 Virality Score Check**

**Note:** If overall_status = BLOCKED, remaining Layer 1 checks should still run (to gather full diagnostic), but Layer 4 assembly CANNOT proceed until blocks resolved.
