# 2.5.3 Synthesis

## Purpose
Extract best elements from all personas, create hybrid combinations, and present options for human selection.

---

## Input Requirements

From Layer 2.5:
- `2.5.1-round1-submissions.yaml` (original submissions)
- `2.5.2-adversarial-critique.yaml` (all critiques + synthesis blueprint)

---

## Execution Protocol

### Step 1: Extract Winning Elements

Based on synthesis blueprint from critique:

```yaml
winning_elements:
  hook:
    selected: |
      [Exact hook tweet]
    source_persona: "[Name]"
    source_round: [1 or 2]
    selection_reasoning: |
      [Why this hook won — scores, critique consensus, standalone strength]

  structure:
    selected: "[Structure description]"
    source_persona: "[Name or 'Hybrid']"
    selection_reasoning: |
      [Why this structure works best]

  standout_tweets:
    - tweet_content: |
        [Specific tweet]
      source_persona: "[Name]"
      source_round: [1 or 2]
      placement: "[Where in final thread]"
      why_wins: "[Reasoning]"

    - tweet_content: |
        [Specific tweet]
      source_persona: "[Name]"
      source_round: [1 or 2]
      placement: "[Where in final thread]"
      why_wins: "[Reasoning]"

  cta:
    selected: |
      [Exact CTA tweet]
    source_persona: "[Name]"
    selection_reasoning: |
      [Why this CTA won]
```

### Step 2: Create Hybrid Threads

Generate 2-3 hybrid combinations:

**Hybrid A:**
```yaml
hybrid_a:
  name: "[Descriptive name]"

  synthesis_strategy: |
    [How elements were combined]

  element_sources:
    hook: "[Persona X]"
    tweets_2_3: "[Persona Y]"
    tweets_4_7: "[Persona Z or hybrid]"
    cta: "[Persona X]"

  full_thread: |
    Tweet 1:
    [Text]

    Tweet 2:
    [Text]

    # ... all tweets

  synthesis_notes: |
    [What makes this hybrid unique]

  predicted_virality: [Score]
```

**Hybrid B:**
```yaml
hybrid_b:
  name: "[Descriptive name]"

  synthesis_strategy: |
    [Different combination approach]

  element_sources: {...}

  full_thread: |

  synthesis_notes: |

  predicted_virality: [Score]
```

**Hybrid C (Optional):**
```yaml
hybrid_c:
  name: "[Descriptive name]"
  synthesis_strategy: |
  element_sources: {...}
  full_thread: |
  synthesis_notes: |
  predicted_virality: [Score]
```

### Step 3: Present for Human Selection

Create decision matrix:

```yaml
human_selection_options:
  - option: "Hybrid A — [Name]"
    thread: |
      [Full thread]
    strengths:
      - "[Strength 1]"
      - "[Strength 2]"
    weaknesses:
      - "[Weakness 1]"
    best_for: "[When to choose this]"

  - option: "Hybrid B — [Name]"
    thread: |
      [Full thread]
    strengths: []
    weaknesses: []
    best_for: "[When to choose this]"

  - option: "Hybrid C — [Name]"
    thread: |
      [Full thread]
    strengths: []
    weaknesses: []
    best_for: "[When to choose this]"

  - option: "Pure Persona: [Name]"
    thread: |
      [Best single-persona thread]
    strengths: []
    weaknesses: []
    best_for: "[When to choose this]"

recommendation:
  suggested_option: "[Option name]"
  reasoning: |
    [Why this is recommended]
```

### Step 4: Capture Human Selection

Document decision:

```yaml
human_selection:
  selected_option: "[Option chosen]"
  selection_timestamp: "[ISO 8601]"

  selection_rationale: |
    [Why human chose this option]

  modifications_requested:
    - modification: "[Any changes requested]"
      applied: [Yes/No]

  final_thread: |
    [Complete selected thread with any modifications]
```
