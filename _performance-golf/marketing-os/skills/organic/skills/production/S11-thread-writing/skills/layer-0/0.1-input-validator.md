# 0.1 Input Validator

## Purpose
Validate that all required inputs exist and are structurally valid before thread writing execution begins.

---

## Input Requirements

From upstream:
- Campaign Brief File (CBF) from S07
- Brand Voice File (BVF) from S03
- Thread brief (YAML or structured input)

---

## Execution Protocol

### Step 1: Validate CBF Existence

Check that CBF exists and contains:
```yaml
required_fields:
  - campaign_name
  - brand_name
  - content_architecture.pillars (≥1)
  - audience_intelligence.pain_expressions (≥3)
  - brand_voice.tone
  - brand_voice.anti_voice
```

**If CBF missing or invalid:** FAIL immediately. Thread writing requires campaign context.

### Step 2: Validate BVF Existence

Check that BVF exists and contains:
```yaml
required_fields:
  - voice_attributes (≥3)
  - energy_level
  - vocabulary.power_words (≥5)
  - vocabulary.banned_words (≥3)
  - tone_by_platform
```

**If BVF missing or invalid:** FAIL immediately. Voice constraints required.

### Step 3: Validate Thread Brief

Check that thread_brief contains:
```yaml
required_fields:
  - topic # What this thread teaches/shares
  - platform # X_TWITTER or LINKEDIN
  - content_function # Awareness/Engagement/Conversion/Community
  - pillar # From CAF
  - key_points # Array with ≥3 points
  - thread_goal # What we want (RT/Like/Reply/Click)
  - cta_goal # What action after consuming

optional_fields:
  - tone_override # If different from BVF default
  - tweet_count # 5-15, or null for auto-determine
  - include_visuals # Yes/No/Partial
```

**If required field missing:** FAIL with specific field name.

### Step 4: Platform Validation

Verify platform value:
```yaml
allowed_values:
  - X_TWITTER
  - LINKEDIN

disallowed_values:
  - "Twitter" # Use X_TWITTER
  - "X" # Use X_TWITTER
  - "LinkedIn Articles" # Use LINKEDIN
```

**If platform invalid:** FAIL and request correction.

### Step 5: Content Function Validation

Verify content_function matches CBF content architecture:
```yaml
allowed_values:
  - Awareness
  - Engagement
  - Conversion
  - Community
```

**If invalid:** FAIL with allowed values.

---

## Output

**File:** `layer-0/0.1-input-validation.yaml`

**Format:**
```yaml
timestamp: "[ISO 8601]"
validation_status: PASS

validated_inputs:
  cbf_file: "[Path]"
  cbf_valid: true

  bvf_file: "[Path]"
  bvf_valid: true

  thread_brief:
    topic: "[Topic]"
    platform: "[X_TWITTER or LINKEDIN]"
    content_function: "[Function]"
    pillar: "[Pillar name]"
    key_points_count: [N]
    thread_goal: "[Goal]"
    cta_goal: "[Action]"

validation_timestamp: "[ISO 8601]"
```

**Minimum size:** 300 bytes
