# 1.1 Platform Analysis

## Purpose
Analyze platform-specific requirements and algorithm behaviors for X/Twitter vs LinkedIn to inform thread architecture.

---

## Input Requirements

From Layer 0:
- `0.1-input-validation.yaml` (platform confirmed)
- `0.4-cbf-context.yaml` (platform strategy from CBF)

---

## Execution Protocol

### Step 1: Platform Specification Analysis

For the target platform, document specifications:

**If X/Twitter:**
```yaml
platform_specs:
  name: "X_TWITTER"
  character_limit: 280
  thread_display: "Connected thread UI"
  media_per_tweet:
    images: 4
    video: 1
    link_card: 1

  numbering_convention:
    recommended: true
    formats: ["1/15", "1.", "1)"]

  algorithm_factors:
    engagement_signals:
      - factor: "First tweet engagement"
        weight: "Critical"
        implication: "Hook determines thread reach"

      - factor: "Quote tweets"
        weight: "High"
        implication: "Amplify significantly vs simple RT"

      - factor: "Thread completion rate"
        weight: "Medium"
        implication: "Readers who finish signal quality"

      - factor: "Bookmarks"
        weight: "Medium"
        implication: "Save-worthy content gets distribution"

    distribution_mechanics:
      initial_boost: "First hour engagement determines initial reach"
      viral_threshold: "High engagement triggers broader distribution"
      decay_rate: "Fast — most engagement in first 24 hours"

  optimal_thread_characteristics:
    tweet_count_range: "7-12 tweets (sweet spot)"
    hook_strength_required: "9/10 minimum"
    visual_recommendation: "First tweet or key tweets"
    cta_placement: "Final tweet + optional mid-thread"
```

**If LinkedIn:**
```yaml
platform_specs:
  name: "LINKEDIN"
  character_limit: 3000
  thread_display: "Posts are separate, manual linking"
  approach_options:
    - "Single long-form post (1500-3000 chars)"
    - "Multi-post series (linked in comments)"

  algorithm_factors:
    engagement_signals:
      - factor: "Dwell time"
        weight: "Critical"
        implication: "Longer reading time = higher quality signal"

      - factor: "Comments (especially long)"
        weight: "Critical"
        implication: "Thoughtful discussion > quick likes"

      - factor: "Reposts with commentary"
        weight: "High"
        implication: "Adds original thought = amplified"

      - factor: "Native content preference"
        weight: "Medium"
        implication: "Text posts > external links"

    distribution_mechanics:
      initial_boost: "First 90 minutes critical"
      viral_threshold: "Slower build than X, longer tail"
      decay_rate: "Slow — can gain traction over days"

  optimal_thread_characteristics:
    format_recommendation: "Single long post for most topics"
    multi_post_when: "Multi-day series or separate concepts"
    hook_strength_required: "8/10 minimum"
    professional_tone_required: true
    visual_recommendation: "Document posts, carousels work well"
```

### Step 2: Thread Format Decision

Based on platform:

```yaml
format_decision:
  platform: "[X_TWITTER or LINKEDIN]"

  recommended_format:
    structure: "[Connected thread OR Single long post OR Multi-post series]"
    rationale: |
      [Why this format for this platform and content]

  character_budget:
    per_unit: [280 for X, 3000 for LinkedIn]
    total_available: [tweet_count × limit OR 3000]
    hook_allocation: "[Percent of first unit]"
    body_allocation: "[Percent per body unit]"
    cta_allocation: "[Percent of final unit]"

  platform_specific_requirements:
    - requirement: "[Specific to platform]"
      must_do: "[Action]"
    - requirement: "[Specific to platform]"
      must_avoid: "[Action]"
```

### Step 3: Algorithm Optimization Strategy

Define engagement tactics:

```yaml
algorithm_strategy:
  primary_signal: "[What algorithm values most]"

  optimization_tactics:
    hook_optimization:
      goal: "[Platform-specific goal]"
      tactics:
        - "[Tactic 1]"
        - "[Tactic 2]"

    engagement_triggers:
      - trigger: "[What prompts engagement]"
        placement: "[Where in thread]"

    completion_strategy:
      - "[How to keep readers through entire thread]"

  distribution_timing:
    optimal_post_time: "[From CBF platform strategy]"
    first_hour_strategy: "[What to do in first 60 minutes]"
```
