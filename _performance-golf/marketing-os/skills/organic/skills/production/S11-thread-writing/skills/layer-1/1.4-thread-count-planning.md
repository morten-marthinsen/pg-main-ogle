# 1.4 Thread Count Planning

## Purpose
Determine optimal thread length (tweet count) based on content density, platform, and audience attention.

---

## Input Requirements

From Layer 0:
- `0.3-specimen-library.yaml` (thread count patterns)
- Thread brief (key_points array)

From Layer 1:
- `1.1-platform-analysis.yaml` (platform sweet spot)
- `1.2-thread-type-selection.yaml` (estimated count)

---

## Execution Protocol

### Step 1: Content Density Analysis

Analyze available content:

```yaml
content_inventory:
  key_points_provided: [N]

  content_breakdown:
    - point: "[Point 1]"
      tweet_requirement: [1 or 2]
      reasoning: "[Why this needs N tweets]"

    - point: "[Point 2]"
      tweet_requirement: [1 or 2]
      reasoning: "[Why this needs N tweets]"

    # ... for all points

  total_content_tweets: [Sum of requirements]

  structural_tweets:
    hook: 1
    context_or_credibility: [0-1]
    transitions: [0-2]
    cta: 1
    total_structural: [Sum]

  total_tweet_need: [content + structural]
```

### Step 2: Platform Optimization

Apply platform constraints:

```yaml
platform_constraints:
  platform: "[X_TWITTER or LINKEDIN]"

  optimal_range:
    minimum: 5
    maximum: 15
    sweet_spot: "[7-12 for X, 8-15 for LinkedIn]"
    reasoning: |
      [Why this range for this platform]

  attention_span_factor:
    platform_attention: "[Short/Medium/Long]"
    implication: |
      [How this affects thread length]

  algorithm_preference:
    completion_rate_importance: "[High/Medium/Low]"
    implication: |
      [Shorter threads = higher completion = better distribution]
```

### Step 3: Thread Count Decision

Balance content need against platform optimization:

```yaml
thread_count_decision:
  content_need: [N tweets]
  platform_optimal: [N tweets]

  decision_logic:
    if_content_need_below_minimum:
      action: "Add context, examples, or combine with related topic"
      rationale: "5 tweets minimum for thread format"

    if_content_need_in_range:
      action: "Use content need"
      rationale: "Content fits platform sweet spot"

    if_content_need_above_maximum:
      action: "Compress or split into multiple threads"
      rationale: "15 tweets maximum for completion rate"

  final_tweet_count: [5-15]

  tweet_allocation:
    tweet_1: "Hook"
    tweet_2: "[Context OR Point 1]"
    tweets_3_to_N_minus_1: "[Body points]"
    tweet_N: "CTA"
```

### Step 4: No-Filler Enforcement

Ensure every tweet adds value:

```yaml
no_filler_protocol:
  each_tweet_must:
    - "Provide standalone insight or value"
    - "Advance the thread's main message"
    - "Be removable ONLY if thread becomes incomplete"

  filler_detection:
    warning_signs:
      - "Here's what I mean..."
      - "Let me explain..."
      - "This is important..."
      - "Stay with me here..."
      - Pure transitions without insight

  filler_prevention:
    if_tweet_is_transition_only:
      action: "Merge with adjacent tweet"

    if_tweet_is_context_only:
      action: "Add insight or example to make valuable"

    if_tweet_count_feels_forced:
      action: "Reduce count, increase density"
```
