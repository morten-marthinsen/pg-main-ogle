# 0.1 Input Validator

## Purpose
Validate carousel brief exists with all required fields and verify CBF is loaded before execution begins.

## Input Requirements
- `carousel-brief.yaml` in project directory
- Content Brief File (CBF) from upstream (S01 or S05)

## Execution Protocol

**MUST:**
- Check carousel_brief.yaml contains: topic, platform, content_function, key_points (array), slide_count (target 5-10), visual_style
- Verify platform is either "instagram" or "linkedin"
- Confirm content_function is one of: educational/promotional/engagement/thought_leadership
- Validate key_points array has 3-8 items
- Verify CBF file exists and is loadable
- Check CBF contains: platform_plan, voice_summary, content_pillars, hook_strategy sections

**NEVER:**
- Proceed with missing required fields
- Accept slide_count <5 or >15
- Skip CBF validation
- Accept undefined platform values

## Output
**Filename:** `validation-report.yaml`

**Format:**
```yaml
validation_status: PASS | FAIL
carousel_brief_present: true/false
cbf_present: true/false
required_fields_complete:
  - topic: true/false
  - platform: true/false
  - content_function: true/false
  - key_points: true/false
  - slide_count: true/false
  - visual_style: true/false
issues: []
timestamp: ISO8601
```

**Minimum size:** 400 bytes

## Quality Gates
- All required fields must be present
- Platform must be valid value
- Slide count within range
- CBF successfully loaded

## Handoff
Passes validation results to Layer 1. BLOCKS execution if status = FAIL.
