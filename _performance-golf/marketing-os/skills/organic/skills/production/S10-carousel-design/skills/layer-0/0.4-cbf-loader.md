# 0.4 CBF Loader

## Purpose
Extract relevant context from Content Brief File (CBF) to inform carousel design.

## Input Requirements
- CBF file path from validation report
- Confirmation validation status = PASS

## Execution Protocol

**MUST:**
- Load complete CBF file
- Extract platform_plan section (posting frequency, content mix, carousel role)
- Extract voice_summary (tone, language level, key phrases)
- Extract content_pillars (3-5 core themes)
- Extract hook_strategy (preferred hook types, proven patterns)
- Note any carousel-specific guidance
- Extract target audience characteristics
- Document brand constraints (visual, tonal, compliance)

**NEVER:**
- Proceed without successful CBF load
- Skip voice_summary extraction
- Ignore content_pillars
- Miss platform-specific constraints

## Output
**Filename:** `cbf-context.yaml`

**Format:**
```yaml
cbf_loaded: true
platform_plan:
  posting_frequency: string
  content_mix: {}
  carousel_role: string
voice_summary:
  tone: string
  language_level: string
  key_phrases: [list]
content_pillars: [list of 3-5]
hook_strategy:
  preferred_types: [list]
  proven_patterns: [list]
audience:
  demographics: {}
  psychographics: {}
  pain_points: [list]
brand_constraints:
  visual: [list]
  tonal: [list]
  compliance: [list]
timestamp: ISO8601
```

**Minimum size:** 2KB

## Quality Gates
- CBF successfully loaded
- All 4 required sections extracted (platform, voice, pillars, hooks)
- Minimum 3 content pillars
- Hook strategy documented

## Handoff
Provides brand/audience context to all Layer 1 and Layer 2 microskills.
