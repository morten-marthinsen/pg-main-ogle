# 2.2 Body Slide Writing

## Purpose
Write body slides that deliver educational value, pass screenshot test, and build toward CTA.

## Input Requirements
- Slide architecture (slide sequence, concepts per slide)
- Save optimization strategy
- Platform requirements (character limits)
- CBF voice summary
- Teaching context

## Execution Protocol

**MUST:**
- Write body slides (slides 2 through N-1) following architecture blueprint
- Apply "one concept per slide" rule strictly
- Structure each slide:
  - **Headline:** Big idea of slide (<100 char Instagram / <150 LinkedIn)
  - **Supporting lines:** 2-3 points that develop headline (<200 char Instagram / <300 LinkedIn)
- Pass "screenshot test" — each slide stands alone without prior context
- Vary formatting across slides:
  - Bullets (Slides 2, 5, 8)
  - Numbers (Slides 3, 6)
  - Short paragraphs (Slides 4, 7)
  - Pull quotes (where appropriate)
- Apply voice consistently from CBF
- Build narrative progression across slides (don't make slides random)
- Front-load practical value (STEPPS)
- Generate all body slides, label PRE-ARENA
- Include formatting notes per slide

**NEVER:**
- Exceed character limits per slide
- Cram multiple concepts into one slide
- Create slides requiring previous slides to understand
- Use same formatting for all slides (visual monotony)
- Skip standalone test
- Forget PRE-ARENA label

## Output
**Filename:** `body-slides-PRE-ARENA.yaml`

**Format:**
```yaml
body_slide_count: number
slides:
  - slide_number: 2
    concept: string
    headline: string (<limit)
    supporting_lines: [2-3 strings, each <limit]
    formatting_type: "bullets | numbers | paragraph | quote"
    screenshot_test: "PASS | needs_revision"
    standalone_score: 0-10
    practical_value: string
    narrative_function: string
  - slide_number: 3
    [...same structure]
formatting_variety_check:
  bullets_count: number
  numbers_count: number
  paragraph_count: number
  quote_count: number
narrative_arc_summary: string
voice_consistency: string
timestamp: ISO8601
```

**Minimum size:** 4KB (for 5-10 body slides)

## Quality Gates
- All body slides within character limits
- Minimum 3 different formatting types used
- All slides pass standalone test (score ≥7.0)
- One concept per slide verified
- Narrative progression described
- All labeled PRE-ARENA

## Handoff
Provides body slide content to 2.5.1 Arena Submission for full carousel evaluation.
