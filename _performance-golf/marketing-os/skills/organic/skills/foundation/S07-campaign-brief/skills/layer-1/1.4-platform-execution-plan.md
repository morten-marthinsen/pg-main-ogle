# 1.4 Platform Execution Plan

## Purpose
Define specific execution details for each platform: posting frequency, optimal times, formats, hashtag strategy, and engagement protocol.

---

## Input Requirements

From Layer 1:
- `1.1-synthesis.yaml` (PSF context)
- `1.3-calendar.yaml` (content plan)

---

## Execution Protocol

### Step 1: Primary Platform Execution

Detail the main platform strategy:

```yaml
primary_platform:
  name: "[Platform]"
  posting_frequency: "[X times per day/week]"
  optimal_times: [times from AIF + PSF]
  content_formats:
    - format: "[Type]"
      length: "[Duration/size]"
      frequency: "[How often]"
      technical_specs: "[Resolution, aspect ratio, etc.]"

  hashtag_strategy:
    approach: "[Branded/niche/trending mix]"
    count_per_post: [number]
    primary_tags: [5-10 core tags]
    rotating_tags: [10-20 niche tags]

  engagement_protocol:
    first_hour: |
      [What happens in first 60 minutes after posting]
      Example: "Reply to all comments within 15 min. Engage with 10-15
      accounts in niche using saved hashtag lists. Share to Story with
      poll/question to drive engagement."

    ongoing: |
      [Daily engagement routine]
      Example: "Check comments 3x daily (morning/lunch/evening). DM respond
      to saves/shares. Engage with 30 accounts daily in target audience."
```

---

### Step 2: Secondary Platform Plans

For each secondary platform:

```yaml
secondary_platforms:
  - name: "[Platform]"
    posting_frequency: "[X times per week]"
    adaptation_from_primary: |
      [How primary content adapts to this platform]
      Example: "Repurpose top-performing Instagram Reels (>5% engagement)
      to TikTok same day. Adjust caption for TikTok culture. Add trending
      hashtags specific to TikTok."

    unique_content_percentage: "[X% original vs Y% adapted]"
    optimal_times: [times]
    platform_specific_notes: |
      [Any unique requirements for this platform]
```

---

### Step 3: Cross-Platform Workflow

How content moves between platforms:

```yaml
cross_platform_workflow:
  primary_to_secondary:
    - step: "1. Publish to Instagram Reels at optimal time"
    - step: "2. Monitor first-hour performance"
    - step: "3. If engagement >5%, adapt to TikTok same day"
    - step: "4. If engagement >8%, create YouTube Short within 48hrs"

  content_lifecycle:
    day_1: "[Primary platform publish]"
    day_1_plus_4hrs: "[Evaluate performance]"
    day_2: "[Repurpose to secondary if criteria met]"
    day_7: "[Evaluate top performers for YouTube]"
    day_30: "[Evaluate for evergreen content bank]"
```

---

### Step 4: Production Requirements

What's needed to execute:

```yaml
production_requirements:
  equipment:
    - "[Item 1]"
    - "[Item 2]"

  time_per_piece:
    - content_type: "[Type]"
      production_time: "[X hours]"
      frequency: "[X per week]"
      weekly_time_commitment: "[Y hours]"

  team_capacity:
    creator: "[Responsibilities]"
    editor: "[Responsibilities if applicable]"
    community_manager: "[Engagement responsibilities]"

  approval_process: |
    [How content gets approved before publishing]
```
