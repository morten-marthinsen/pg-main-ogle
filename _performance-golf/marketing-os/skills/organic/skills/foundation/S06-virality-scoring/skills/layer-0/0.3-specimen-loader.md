# 0.3 Specimen Loader

## Purpose
Load viral content specimens from the target platform for calibration. These specimens provide the empirical data needed to set accurate benchmarks and validate the scoring framework.

---

## Input Requirements

From user or project files:
- Viral specimens from target platform (minimum 50 requested, minimum 10 required)
- Growth trajectory case studies
- High-performing competitor content

From 0.1 validation:
- Primary platform identified

---

## Execution Protocol

### Step 1: Specimen Collection Requirements

Define what constitutes a valid calibration specimen:

**Minimum criteria:**
- From the same platform as primary platform (PSF)
- Published within last 12 months (recency)
- Performance data available (views, engagement rate, shares)
- Similar niche or content function to campaign
- Clear virality signal (>3x account average OR >100K views if new account)

---

### Step 2: Specimen Data Structure

For each specimen, capture:

```yaml
specimen_id: "[unique_id]"
platform: "[platform]"
content_type: "[format]"
title_or_description: "[text]"
performance:
  views: [number]
  engagement_rate: [percentage]
  shares: [number]
  saves: [number]
  comments: [number]
virality_factor: "[what made it spread — will inform dimension scores]"
hook_excerpt: "[first 3 seconds / first line]"
url: "[if available]"
```

---

### Step 3: Categorize Specimens

Group specimens by performance tier:

- **Tier 1 (Viral):** >500K views OR >10% engagement rate
- **Tier 2 (Strong):** 100K-500K views OR 5-10% engagement rate
- **Tier 3 (Solid):** 50K-100K views OR 3-5% engagement rate
- **Tier 4 (Baseline):** Account average performance

**REQUIREMENT:** Must have specimens across at least 3 tiers for valid calibration.

---

### Step 4: Flag Missing Data

If user provides <10 specimens:
- Set `calibration_status: INSUFFICIENT`
- Request additional specimens OR
- Proceed with warning: "Framework will require recalibration after campaign launch"

If user provides 10-49 specimens:
- Set `calibration_status: MINIMUM`
- Proceed but flag for enhancement

If user provides 50+ specimens:
- Set `calibration_status: ROBUST`

---

## Output

**File:** `layer-0/0.3-specimen-inventory.yaml`

**Format:**
```yaml

  count: [number]
  platform: "[primary_platform]"
  tier_distribution:
    tier_1_viral: [count]
    tier_2_strong: [count]
    tier_3_solid: [count]
    tier_4_baseline: [count]

specimen_list:
  - id: "SPEC_001"
    platform: "instagram_reels"
    title: "[description]"
    performance:
      views: 1200000
      engagement_rate: 8.5
      shares: 15000
    virality_factor: "Unexpected transformation reveal + trending audio"

    tier: "viral"

  # [... additional specimens]

gaps:
  - "Need more Tier 3 (solid) specimens for mid-range calibration"
  - "No baseline performers — will use account average estimates"

warnings: []
```

---

## Quality Gates

- [ ] At least 10 specimens loaded
- [ ] All specimens from primary platform
- [ ] Performance data available for each
- [ ] Specimens span at least 3 performance tiers
- [ ] Virality factors documented

---

## Handoff

Pass specimen inventory to: **0.4 Upstream Loader**

If calibration_status is INSUFFICIENT, FLAG for human review before proceeding to Layer 1.
