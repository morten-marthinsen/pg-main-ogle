# 1.5 Quick-Score Builder

## Purpose
Create a rapid assessment checklist for content creators to evaluate content quality before full scoring. This is the practical tool used daily during production.

---

## Input Requirements

From Layer 1:
- `1.1-dimensions.yaml` (dimension criteria)
- `1.3-calibration.yaml` (what viral content has)
- `1.4-benchmarks.yaml` (what to avoid)

---

## Execution Protocol

### Step 1: Define Must-Haves

From calibration, what do ALL high-performing pieces have:

```yaml
must_haves:
  - "Hook stops scroll in <3 seconds"
  - "Clear emotional trigger identified"
  - "Specific value delivered (not vague advice)"
  - "Platform-optimized format (aspect ratio, length)"
  - "On-voice execution (matches BVF)"
```

**Minimum 5, derived from top-scoring dimensions in calibration.**

---

### Step 2: Define Red Flags

From calibration, what do failing pieces consistently have:

```yaml
red_flags:
  - "Slow/no hook (opens with setup)"
  - "Generic advice (could apply to anyone)"
  - "Off-voice elements (banned words, wrong tone)"
  - "Platform mismatch (wrong format for platform)"
  - "No clear CTA or takeaway"
```

**Minimum 5, derived from low-scoring specimens.**

---

### Step 3: Define Green Lights

Indicators of viral potential from calibration:

```yaml
green_lights:
  - '"I need to share this" feeling when watching'
  - "Specific and memorable (can repeat it back)"
  - "Creates conversation/debate potential"
  - "Teaches something new or reframes known thing"
  - "Makes audience feel something strong (awe/anger/excitement)"
```

**These are BONUS indicators — not required, but strong signals.**

---

### Step 4: Define Kill Criteria

When to abandon content without further work:

```yaml
kill_criteria:
  - "Score below 40 = fundamentally flawed, start over"
  - "Multiple red flags (3+) = structural issues"
  - "Off-voice = cannot publish as-is, brand integrity violation"
  - "Platform violation = won't pass review/algorithm"
```

---

### Step 5: Create Quick-Score Workflow

```
STEP 1: Check Must-Haves
├── All 5 present? → Proceed to Step 2
└── Any missing? → Fix before continuing

STEP 2: Scan for Red Flags
├── 0 flags? → Proceed to Step 3
├── 1-2 flags? → Address before publishing
└── 3+ flags? → Kill or major rework

STEP 3: Count Green Lights
├── 3+ green lights? → Likely 75+ score, prioritize
├── 1-2 green lights? → Likely 60-75 score, publishable
└── 0 green lights? → Check full score, may need boost

STEP 4: Apply Kill Criteria
└── If any criterion met → STOP, do not publish
```

## Output

**File:** `layer-1/1.5-quick-score.yaml`

**Format:**
```yaml
timestamp: "[ISO 8601]"

must_haves:
  - "Hook stops scroll in <3 seconds"
  - "Clear emotional trigger identified"
  - "Specific value delivered"
  - "Platform-optimized format"
  - "On-voice execution"

red_flags:
  - "Slow/no hook"
  - "Generic advice"
  - "Off-voice elements"
  - "Platform mismatch"
  - "No clear CTA or takeaway"

green_lights:
  - '"I need to share this" feeling'
  - "Specific and memorable"
  - "Creates conversation/debate"
  - "Teaches something new"
  - "Makes audience feel something strong"

kill_criteria:
  - "Score below 40"
  - "3+ red flags"
  - "Off-voice (brand violation)"
  - "Platform policy violation"

quick_score_workflow: |
  1. Check must-haves (all 5 required)
  2. Scan red flags (0-2 acceptable, 3+ = rework)
  3. Count green lights (3+ = strong, 0 = needs boost)
  4. Apply kill criteria (any met = do not publish)

usage_note: |
  This checklist is for rapid evaluation during production.
  Full dimension scoring still required for final publish decision.
```

**Minimum size:** 1KB
