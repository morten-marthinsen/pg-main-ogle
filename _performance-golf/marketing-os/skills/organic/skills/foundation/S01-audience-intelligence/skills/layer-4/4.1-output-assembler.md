# 4.1 Output Assembler

## Purpose
Assemble the complete AIF (Audience Intelligence File) from all Layer 1 outputs and validate against Gate G01 requirements.

---

## Input Requirements
All Layer 1 outputs:
- demographic-foundation.yaml (1.1)
- psychographic-depth.yaml (1.2)
- platform-behavior.yaml (1.3)
- language-mining.yaml (1.4)
- competitor-audience.yaml (1.5)
- pain-desire-mapping.yaml (1.6)

---

## Execution Protocol

### Step 1: Load All Layer 1 Outputs
```
VERIFY all 6 Layer 1 output files exist:
  - demographic-foundation.yaml
  - psychographic-depth.yaml
  - platform-behavior.yaml
  - language-mining.yaml
  - competitor-audience.yaml
  - pain-desire-mapping.yaml

IF any missing:
  HALT: "Cannot assemble AIF — Layer 1 incomplete"
  LOG: Missing file(s)
```

### Step 2: Assemble Complete AIF
```
MERGE all sections into single AIF.yaml following schema:

STRUCTURE:
  campaign_name: [from input]
  brand_name: [from input]
  date_created: [ISO 8601 timestamp]
  version: "1.0"

  demographics: [from 1.1]
  psychographics: [from 1.2]
  platform_behavior: [from 1.3]
  language_mining: [from 1.4]
  competitors_they_follow: [from 1.5]
  pain_mapping: [from 1.6]
  desire_mapping: [from 1.6]

  key_insights: [synthesize from all layers]
```

### Step 3: Synthesize Key Insights
```
EXTRACT from all Layer 1 outputs:

PRIMARY INSIGHT:
  - The single most important finding about this audience
  - Must be actionable for content strategy
  - Should be surprising or non-obvious

SECONDARY INSIGHTS (3-5):
  - Supporting insights from different sections
  - Cross-section connections (e.g., pain + platform behavior)

CONTENT IMPLICATIONS (5-7):
  - Specific content strategy recommendations
  - Format recommendations
  - Topic recommendations
  - Voice recommendations
  - Platform prioritization

EXAMPLE:
  primary_insight: "Audience values freedom over stability — anti-9-5, pro-flexibility content will resonate strongly"

  secondary_insights:
    - "Primary consumption is mobile during commute (6-8am) — favor short-form video"
    - "Root pain is fear of irrelevance, not lack of skills — content should address visibility/positioning"

  content_implications:
    - "Lead with freedom/flexibility angles in hooks"
    - "Create 60-sec videos for morning commute consumption"
    - "Address positioning/visibility over skill-building"
```

### Step 4: Gate G01 Validation
```
VALIDATE all 8 required fields:

REQUIRED:
  1. demographics.age_range.primary — NOT empty
  2. demographics.location.primary_markets — ≥1 item
  3. platform_behavior.primary_platforms — ≥2 items
  4. platform_behavior.best_times_active — NOT empty
  5. language_mining.pain_expressions.how_they_describe_problems — ≥3 items
  6. competitors_they_follow.accounts — ≥3 items
  7. pain_mapping.surface_pains — ≥3 items
  8. desire_mapping.stated_desires — ≥3 items

FOR EACH requirement:
  CHECK: Field exists and meets minimum

IF ANY fail:
  validation_status: FAIL
  WRITE: List of failed requirements
  HALT: "Cannot pass Gate G01 — requirements not met"

IF ALL pass:
  validation_status: PASS
  CREATE: checkpoints/GATE_G01_PASS.yaml
```

## Quality Gates
- [ ] All 6 Layer 1 outputs successfully loaded
- [ ] key_insights section synthesized (not empty)
- [ ] primary_insight is actionable and specific
- [ ] ≥3 secondary_insights
- [ ] ≥5 content_implications
- [ ] All 8 Gate G01 requirements validated
- [ ] validation_status = PASS
- [ ] AIF.yaml file created in outputs directory

**CRITICAL:** If gate_g01_validation.status = FAIL, the skill has FAILED. Cannot proceed to S02.
