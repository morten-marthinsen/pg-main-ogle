# 1.1 Content Atomization

## Purpose
Break down original content into atomic units — the smallest valuable pieces that can stand alone or be recombined into new formats.

---

## Input Requirements

From Layer 0:
- `0.2-upstream-content.yaml` (loaded content)

---

## Execution Protocol

### Step 1: Identify Atomic Units

Extract these unit types from original content:

**Hooks:**
- Opening statements that stop scrolls
- First 3-10 seconds / first 1-2 sentences
- Standalone potential: High

**Key Insights:**
- Single valuable ideas or revelations
- 3-7 distinct insights within content
- Each insight can be its own piece

**Memorable Quotes:**
- Statements worth sharing/saving
- Under 280 characters with punch
- Quote graphics, tweetable moments

**Story Beats:**
- Complete story arcs within content
- Beginning/middle/end structure
- Emotional impact moments

**Frameworks:**
- Processes, systems, mental models
- Step-by-step processes
- Highly shareable, carousel-ready

**Data Points:**
- Statistics, numbers, research citations
- Specific numbers with source
- Visualization potential

**Controversial Takes:**
- Opinions that generate discussion
- Statements that invite disagreement

**Visual Moments:**
- Particularly visual or demonstrative moments
- Moments that work without audio context

### Step 2: Score Standalone Strength

For each atomic unit, rate standalone strength (1-10):
- Can this stand alone without context?
- Does it deliver value independently?
- Is it shareable/saveable?

### Step 3: Map Variant Potential

For each atomic unit, list potential formats:
- TikTok clip, Instagram Reel, YouTube Short
- Carousel, Thread, Quote graphic
- LinkedIn post, Blog section

### MUST Complete

- Minimum 3 atomic units per type (where applicable)
- Standalone strength scored for each unit
- Variant potential mapped for each unit

### NEVER Skip

- Atomization worksheet documentation
- Standalone strength assessment
- Timestamp documentation (for video content)

---

## Output

**File:** `layer-1/1.1-atomization.yaml`

**Format:**
```yaml
timestamp: "[ISO 8601]"
content_id: "[ID]"

atomic_units:
  hooks:
    - text: |
      timestamp: "[If video]"
      standalone_strength: [1-10]
      variant_potential: ["TikTok clip", "Reel", "Quote graphic"]

  key_insights:
    - insight: |
      timestamp: "[If video]"
      single_idea: true
      standalone_strength: [1-10]
      variant_potential: []

  memorable_quotes:
    - quote: |
      character_count: [X]
      tweetable: true/false
      visual_potential: high/medium/low
      variant_potential: []

  story_beats:
    - story_summary: |
      timestamp: "[If video]"
      duration: "[If video]"
      standalone_complete: true/false
      variant_potential: []

  data_points:
    - data: |
      source: "[If available]"
      visualization_potential: high/medium/low
      variant_potential: []

  frameworks:
    - framework_name: |
      steps: [Count]
      carousel_potential: true/false
      thread_potential: true/false
      variant_potential: []

  controversial_takes:
    - take: |
      engagement_potential: high/medium/low
      risk_level: high/medium/low
      variant_potential: []

  visual_moments:
    - description: |
      timestamp: "[Start-End]"
      works_without_audio: true/false
      variant_potential: []

atomization_summary:
  total_atomic_units: [X]
  high_potential_units: [Count with strength >= 7]
  total_variant_possibilities: [X]
```

**Minimum size:** 3KB

---

## Quality Gates

- [ ] Minimum 3 atomic units per type (where content allows)
- [ ] All units have standalone strength scored
- [ ] All units have variant potential mapped
- [ ] Video content has timestamps
- [ ] Total variant possibilities >= 10
