# 1.4 Production Sequencing

## Purpose
Organize variant production into realistic timeline with proper sequencing, resource allocation, and coordination with S15 scheduling.

---

## Input Requirements

From Layer 1:
- `1.3-variant-plan.yaml` (all variants)

---

## Execution Protocol

### Step 1: Create Production Queue

Organize variants into timeline:

**Immediate (Day 0):**
- Variants that can be produced quickly
- Typically low-effort variants
- High-priority distribution targets

**Short-Term (Day 1-3):**
- Medium-effort variants
- Design-intensive variants
- Coordination-required variants

**Extended (Day 3+):**
- Long-form content adaptations
- High-effort variants
- Lower-priority or evergreen variants

### Step 2: Assess Dependencies

For each variant:
- Does it need other variants completed first?
- Does it require specific tools/resources?
- Are there team coordination needs?

### Step 3: Define Distribution Strategy

For each platform:
- Which variants go to this platform?
- What's the posting cadence?
- Scheduling notes for S15
- Cross-reference strategy

### Step 4: Create Tracking Structure

Define:
- Metrics per variant (what to track)
- Aggregate metrics (total reach multiplier)
- Success criteria per variant

### MUST Define

- Production timeline for all variants
- Platform distribution strategy
- Tracking metrics
- Coordination with S15 (scheduling)

### NEVER Skip

- Realistic timeline assessment
- Resource requirements
- Platform-specific scheduling notes

---

## Output

**File:** `layer-1/1.4-production-sequence.yaml`

**Format:**
```yaml
timestamp: "[ISO 8601]"
original_content_id: "[ID]"

production_queue:
  immediate:
    day_0:
      - variant_id: "[ID]"
        assignee: "[Team member or AI/human]"
        estimated_time: "[Hours]"
        tools_required: []
        status: "Queued"

  short_term:
    day_1_to_3:
      - variant_id: "[ID]"
        assignee: "[Team member]"
        estimated_time: "[Hours]"
        tools_required: []
        dependencies: "[Other variants needed first]"
        status: "Queued"

  extended:
    day_3_plus:
      - variant_id: "[ID]"
        assignee: "[Team member]"
        estimated_time: "[Hours]"
        tools_required: []
        status: "Queued"

distribution_strategy:
  per_platform:
    instagram:
      variants: ["[variant_id]", "[variant_id]"]
      posting_cadence: "1 per day, stagger feed vs reels"
      scheduling_notes: "Coordinate with S15 for optimal times"

    tiktok:
      variants: ["[variant_id]", "[variant_id]"]
      posting_cadence: "1-2 per day"
      scheduling_notes: "Post at different times to test algorithm"

    youtube:
      variants: ["[variant_id]"]
      posting_cadence: "Shorts 3x per week"
      scheduling_notes: "Reference full video in description"

    linkedin:
      variants: ["[variant_id]"]
      posting_cadence: "2x per week"
      scheduling_notes: "Morning posts before 8am or after 6pm"

    x_twitter:
      variants: ["[variant_id]"]
      posting_cadence: "Thread on best-performing day"
      scheduling_notes: "Coordinate with S15 optimal times"

cross_reference_strategy:
  internal_linking:
    - variant_id: "[ID]"
      references: "Original content"
      method: "Link in bio, caption mention"

  content_callbacks:
    - variant_id: "[ID]"
      callback_to: "[Original or other variant]"
      callback_method: "Comment, caption reference"

tracking:
  metrics_per_variant:
    - variant_id: "[ID]"
      metrics_to_track:
        - views
        - engagement_rate
        - compared_to_original
        - platform_performance_index

  aggregate_metrics:
    total_variants: [Count]
    total_reach_goal: "[X]x original reach"
    best_performer_criteria: "Highest engagement_rate"
    worst_performer_criteria: "Lowest view_index"

production_summary:
  total_variants: [Count]
  estimated_total_time: "[Hours]"
  timeline_span: "[Days]"
  coordination_with_s15: "Ready for scheduling handoff"
```

**Minimum size:** 3KB
