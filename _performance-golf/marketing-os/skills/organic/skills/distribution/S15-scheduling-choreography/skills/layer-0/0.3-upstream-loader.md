# 0.3 Upstream Package Loader

## Purpose
Load and parse all upstream files from S14, S02, and S07 into structured data for scheduling strategy.

---

## Input Requirements

From Layer 0:
- `0.1-validation-results.yaml` (must be PASS)
- `0.2-teachings-loaded.yaml`

---

## Execution Protocol

### Step 1: Load S14 Content Packages

Extract from `[campaign]-content-packages.yaml`:

```yaml
content_inventory:
  - content_id:
    title:
    platform:
    secondary_platforms: []
    content_type:
    function: [Awareness/Engagement/Conversion/Community]
    pillar:
    virality_score:
    production_status:
```

### Step 2: Load S02 Platform Strategy

Extract from `[campaign]-PSF.yaml`:

```yaml
platform_timing:
  [platform]:
    optimal_times: []
    posting_frequency:
    algorithm_velocity_windows:
    minimum_gaps:
```

### Step 3: Load S07 Campaign Brief

Extract from `[campaign]-CBF.yaml`:

```yaml
campaign_context:
  campaign_dates:
    start:
    end:
  content_calendar:
    posting_cadence:
    key_dates: []
  timezone:
```

### Step 4: Cross-Reference Check

- Verify all content platforms have timing data in PSF
- Verify campaign dates span all content
- Identify any mismatches

---

## Output

**File:** `layer-0/0.3-upstream-data.yaml`

**Format:**
```yaml
timestamp: "[ISO 8601]"

content_to_schedule:
  total_pieces: [count]
  by_platform:
    instagram: [count]
    tiktok: [count]
    # [etc]
  by_function:
    awareness: [count]
    engagement: [count]
    conversion: [count]
    community: [count]

platform_constraints:
  [platform]:
    optimal_windows: []
    posting_frequency:
    minimum_gap:

campaign_parameters:
  date_range: "[start] to [end]"
  timezone:
  key_dates: []
  posting_cadence:

cross_reference_status: [PASS/ISSUES]
issues: []
```

**Minimum size:** 1KB
