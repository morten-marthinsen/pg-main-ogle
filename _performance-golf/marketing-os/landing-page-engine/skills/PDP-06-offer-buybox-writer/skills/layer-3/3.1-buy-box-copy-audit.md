# 3.1 — Buy Box Copy Audit

**Skill:** PDP-06-offer-buybox-writer
**Position:** Layer 3, Microskill 1
**Purpose:** Audit all buy box copy against Baymard best practices. Verify every component has copy, savings are displayed correctly in both formats, no generic language exists, subscription tiles are not pre-selected, and ATC label is standard. This is the quality gate before package assembly.
**Effort Level:** medium
**Dependencies:** All Layer 1 outputs (pricing-copy.md, variant-copy.md, subscription-copy.md, quantity-copy.md, short-description-copy.md), All Layer 2 outputs (atc-copy.md, express-payment-copy.md, micro-trust-copy.md, sticky-atc-copy.md, mini-cart-copy.md), 0.3 blueprint-extraction.md (component inventory)
**Output:** `buy-box-copy-audit.md`

## Execution Logic

**Step 1:** Read `blueprint-extraction.md`. Get the full component inventory from PDP-03.

**Step 2:** Cross-check: for every component PDP-03 specified, verify a corresponding copy file exists with actual copy written:

```
Component Completeness Check:
  [ ] Pricing copy exists (pricing-copy.md)
  [ ] Variant selector copy exists (variant-copy.md) -- or N/A if no variants
  [ ] Subscription tile copy exists (subscription-copy.md) -- or N/A if no subscription
  [ ] Quantity stepper copy exists (quantity-copy.md)
  [ ] Short description copy exists (short-description-copy.md)
  [ ] ATC button copy exists (atc-copy.md)
  [ ] Express payment copy exists (express-payment-copy.md) -- or N/A if no express
  [ ] Micro-trust signal copy exists (micro-trust-copy.md)
  [ ] Sticky ATC bar copy exists (sticky-atc-copy.md)
  [ ] Mini-cart copy exists (mini-cart-copy.md)
```

**Step 3:** Audit pricing copy:
- Original price shown struck-through: Y/N
- Sale price shown: Y/N
- Savings in percentage format: Y/N
- Savings in dollar amount format: Y/N
- Per-unit pricing on multi-packs: Y/N
- Shipping visible near price: Y/N

**Step 4:** Audit subscription copy:
- Neither tile pre-selected: Y/N
- OTP tile has clear price: Y/N
- S&S tile has savings percentage in label: Y/N
- S&S tile has explicit dollar savings: Y/N
- "Cancel anytime" visible in S&S tile: Y/N
- Frequency selector present: Y/N

**Step 5:** Audit ATC copy:
- Default label = "Add to Cart": Y/N
- All 5 states defined: Y/N
- Post-click = slide-out mini-cart: Y/N

**Step 6:** Audit micro-trust copy:
- Signal count = 3-4: Y/N
- All signals are specific factual statements: Y/N
- No marketing language: Y/N

**Step 7:** Audit short description:
- 3-5 bullets: Y/N
- All bullets benefit-forward: Y/N
- No feature-forward bullets: Y/N

**Step 8:** Audit variant copy:
- All variants have labels: Y/N
- Price differentials shown inline: Y/N
- No abbreviations: Y/N

**Step 9:** Score the overall buy box copy:

```
Scoring (12-point checklist):
  1. All PDP-03 components have copy files: [0/1]
  2. Savings in BOTH % and $: [0/1]
  3. Subscription not pre-selected: [0/1]
  4. "Cancel anytime" in S&S tile: [0/1]
  5. ATC = "Add to Cart": [0/1]
  6. ATC has all 5 states: [0/1]
  7. Micro-trust = 3-4, all specific: [0/1]
  8. Short description benefit-forward: [0/1]
  9. Variant labels complete: [0/1]
  10. Post-ATC = mini-cart: [0/1]
  11. Sticky bar ATC matches main: [0/1]
  12. Zero generic AI language: [0/1]

  Score: [X]/12
  Threshold: 12/12 (all must pass)
```

**Step 10:** If any item scores 0, document the specific failure and what needs to be fixed.

**Step 11:** Write `buy-box-copy-audit.md`.

## Quality Criteria
- Every PDP-03 component is checked for copy completeness
- All 12 audit points are evaluated with clear PASS/FAIL
- Failures include specific fix instructions (not vague "needs improvement")
- Score is accurate (no rounding, no partial credit)
- If any item fails, overall status = FAIL
- File exists after execution

## Failure Modes
- One or more copy files missing -> Overall FAIL; list missing files; Layer 1 or 2 must be re-run
- Score < 12/12 -> Overall FAIL; specific failures documented for targeted fixes
- Audit finds generic AI language not caught by individual microskills -> Document specific instances and required rewrites
- N/A components (no variants, no subscription) -> Correctly mark as N/A, do not count against score
