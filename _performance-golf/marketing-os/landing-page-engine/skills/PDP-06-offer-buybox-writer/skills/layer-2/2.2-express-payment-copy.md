# 2.2 — Express Payment Copy

**Skill:** PDP-06-offer-buybox-writer
**Position:** Layer 2, Microskill 2
**Purpose:** Write the express payment section copy: the "OR" divider text between ATC and express checkout, payment method labels, and express checkout button labels. This section reduces friction for users who want to skip the cart entirely.
**Effort Level:** medium
**Dependencies:** 0.3 blueprint-extraction.md (express payment spec from PDP-03), 2.1 atc-copy.md (ATC placement context)
**Output:** `express-payment-copy.md`

---

## Input Schema

```yaml
input:
  blueprint_extraction: "[path to blueprint-extraction.md]"
  pdp_reference_load: "[path to pdp-reference-load.md]"
  atc_copy: "[path to atc-copy.md]"
```

---

## Execution Logic

**Step 1:** Read `blueprint-extraction.md`. Extract PDP-03's express payment layout spec:
- Which payment methods are included (Apple Pay, Google Pay, Shop Pay, PayPal, Amazon Pay, etc.)
- Layout direction (horizontal row vs stacked)
- Position relative to ATC button (below)

**Step 2:** Read `pdp-reference-load.md`. Confirm express payment copy standards.

**Step 3:** Write the OR divider copy:
```
Text: "— OR —"
Alternative: "or" (lowercase, with horizontal rules on each side)
```
- Position: centered between ATC button and express payment buttons
- Visually separates "Add to Cart" from express checkout options
- Keep minimal -- it's a visual separator, not a persuasion element

**Step 4:** Write express checkout button labels:
- Each express payment button uses the payment provider's branded label
- These are NOT custom copy -- they follow the provider's standard format:

```
Apple Pay:      [Apple Pay logo] "Buy with Apple Pay"
Google Pay:     [Google Pay logo] "Buy with Google Pay"
Shop Pay:       [Shop Pay logo] "Buy with Shop Pay"
PayPal:         [PayPal logo] "PayPal"
Amazon Pay:     [Amazon Pay logo] "Amazon Pay"
```

**Step 5:** Write the express checkout section header (if needed):
- Usually no header needed -- the OR divider + branded buttons are self-explanatory
- If a header is required: "Express checkout" (lowercase, small text)
- NOT "Quick Buy Options" or "Faster Ways to Pay"

**Step 6:** Write the payment security note (if applicable):
```
Text: "All transactions are secure and encrypted."
Position: small text below express payment buttons
```
- Only include if not already covered by micro-trust signals
- Do not duplicate trust messaging

**Step 7:** Write `express-payment-copy.md`.

## Quality Criteria
- OR divider is clean and minimal (not marketing language)
- Express payment buttons use provider-standard labels (not custom)
- Only payment methods specified in PDP-03 blueprint are included
- No duplicate trust messaging (check against micro-trust signals in 2.3)
- Section header is minimal or absent (buttons are self-explanatory)
- File exists after execution

## Failure Modes
- PDP-03 blueprint does not specify express payments -> Skip this microskill; document in execution log. Not all PDPs have express checkout.
- Payment method list not specified in brief or blueprint -> Use common defaults (Apple Pay, Google Pay, Shop Pay); flag for human confirmation
- OR divider copy gets creative ("OR CHECK OUT FASTER WITH:") -> Replace with minimal "— OR —"
- Express payment buttons get custom labels -> Replace with provider-standard labels. These are branded elements, not customizable copy.
