# 1.4 — Quantity Stepper Copy

**Skill:** PDP-06-offer-buybox-writer
**Position:** Layer 1, Microskill 4
**Purpose:** Write all copy associated with the quantity stepper: label, default value, maximum value display, and bundle savings callout that appears when quantity triggers a bundle threshold.
**Effort Level:** low
**Dependencies:** 0.1 input-verification.md (offer structure), 0.3 blueprint-extraction.md (quantity stepper spec from PDP-03), 1.1 pricing-copy.md (multi-pack pricing)
**Output:** `quantity-copy.md`

---

## Input Schema

```yaml
input:
  input_verification: "[path to input-verification.md]"
  blueprint_extraction: "[path to blueprint-extraction.md]"
  pricing_copy: "[path to pricing-copy.md]"
```

---

## Execution Logic

**Step 1:** Read `input-verification.md`. Extract:
- Maximum order quantity (if specified)
- Bundle thresholds (e.g., "buy 3, save 15%")
- Multi-pack options from offer structure

**Step 2:** Read `blueprint-extraction.md`. Extract PDP-03's quantity stepper spec:
- Stepper type (confirmed +/- stepper, not dropdown)
- Default value (typically 1)
- Min/max values
- Touch target size (44px+ -- UX spec, not copy)

**Step 3:** Read `pricing-copy.md`. Get multi-pack pricing details for bundle savings callouts.

**Step 4:** Write stepper label copy:
- Label: "Quantity" (standard, not "How many?" or "Select quantity")
- Position: above or beside the stepper

**Step 5:** Write stepper values:
- Default value: 1
- Minimum: 1
- Maximum: product-specific (e.g., 10) or platform default

**Step 6:** Write bundle savings callout copy (if applicable):
- Triggered when quantity reaches a bundle threshold
- Format: "Add [X] more to save [Y]%" or "You qualify for [X]% off!"
```
Examples:
  Qty 1: (no callout)
  Qty 2: "Add 1 more to save 15%!"
  Qty 3: "You're saving 15%!"
```
- Callout appears inline near stepper or price when threshold is reached or approaching

**Step 7:** Write max quantity notification (if applicable):
```
If user hits max: "Maximum [X] per order"
```

**Step 8:** Write `quantity-copy.md`.

---

| [X] | [threshold reached text] | [bundle triggered] |

- **Text:** ["Maximum [X] per order" | N/A]

- [If no bundle thresholds exist: "No bundle savings callouts. Stepper is label + value only."]

---

## Quality Criteria
- Label is exactly "Quantity" (not "How many?" or "Select amount")
- Default is 1
- Bundle savings callouts are specific with exact percentages/amounts
- Callout copy is encouraging, not pushy ("Add 1 more to save" not "You're missing out!")
- Max quantity notification is factual
- File exists after execution

## Failure Modes
- No bundle threshold data -> Skip bundle callouts; write label + default only
- PDP-03 blueprint does not include quantity stepper -> Skip this microskill; document in execution log
- Max quantity not specified -> Omit max notification; default to platform behavior
- Bundle savings percentages inconsistent with pricing-copy.md -> Reconcile with 1.1; 1.1 is source of truth
