# 1.3 — Review Histogram Spec

**Skill:** PDP-05-social-proof-review-system
**Position:** Layer 1, Microskill 3
**Purpose:** Design the star distribution histogram — tappable bars that double as filters, count labels, visual proportions, active/inactive states, and zero-count display. The histogram is the single most important trust signal in a review system — it lets users self-validate the aggregate rating before reading individual reviews.
**Effort Level:** medium
**Dependencies:** `input-verification.md` (0.1), `pdp-reference-load.md` (0.3), `review-system-architecture.md` (1.1)
**Output:** `review-histogram-spec.md`

## Execution Logic

**Step 1:** Check scope. If `scope = minimal` or `scope = generation_only`, SKIP histogram entirely. Document: "Histogram omitted per scope determination. Scope = [scope]." Output a stub file and exit.

**Step 2:** Determine star distribution source:
- `star_distribution` provided from client data -> use exact numbers
- `star_distribution` NOT provided but `aggregate_rating` + `total_review_count` available -> generate realistic distribution using the Realism Table (from AGENT.md)
- Neither available -> generate distribution using category defaults

**Step 3:** Star Distribution Realism Table (reference from AGENT.md):

| Rating Band | 5-Star % | 4-Star % | 3-Star % | 2-Star % | 1-Star % | Avg Rating |
|-------------|----------|----------|----------|----------|----------|------------|
| Excellent   | 55-65%   | 15-20%   | 5-10%    | 3-5%     | 5-10%    | 4.3-4.6    |
| Good        | 40-50%   | 20-25%   | 10-15%   | 5-10%    | 5-10%    | 4.0-4.3    |
| Average     | 25-35%   | 15-20%   | 15-20%   | 10-15%   | 10-15%   | 3.5-4.0    |

FORBIDDEN distributions:
- 95%+ five-star (unrealistic, triggers distrust)
- 0% one-star (obviously fake — real products always have SOME one-star reviews)
- Uniform distribution (20% each — statistically improbable)
- All reviews at same rating (e.g., 100% 4-star)

**Step 4:** Design the histogram layout:
- Position: directly below Section Header ("Customer Reviews" + aggregate + count)
- Orientation: 5 horizontal bars, 5-star at TOP, 1-star at BOTTOM
- Each bar has 3 elements:
  - LEFT: star label ("5 ★" or "5 stars")
  - MIDDLE: horizontal fill bar (proportional to count)
  - RIGHT: count label (exact number, e.g., "1,847")
- Fill bar width: proportional to the HIGHEST count bar (the bar with the most reviews = 100% width, all others proportional)
- Minimum bar width: even 0-count bars show a thin outline (never completely invisible)

**Step 5:** Design the tappable filter behavior:
- Each bar is a TAPPABLE element (this is the critical Baymard requirement — histograms are not just display, they are FILTERS)
- Tap on any bar -> filters the review list below to show ONLY reviews with that star rating
- Active state visual: filled bar with brand accent color, slightly enlarged, subtle shadow or border
- Inactive state visual: gray fill or outline only, muted color
- Deselect behavior: tap the active bar AGAIN to deselect, returning to "All Reviews"
- Multi-select: NOT allowed. Only one star filter active at a time. (Tapping a different bar switches the filter, does not add to it.)
- Filter integration: histogram filter works WITH keyword filter chips (combined filtering — e.g., "4-star reviews mentioning 'battery life'")

**Step 6:** Design zero-count bar behavior:
- If a star rating has 0 reviews: bar is still displayed
- Fill bar: empty (outline only, no fill)
- Count label: "0"
- Tappable: YES, but tapping shows "No [N]-star reviews" message in the review list area
- FORBIDDEN: hiding any star rating row. All 5 rows ALWAYS visible.

**Step 7:** Design the aggregate rating display (above or integrated with histogram):
- Large aggregate number: e.g., "4.3" (one decimal place, never two)
- Star visualization: 4.3 stars filled (4 full + 0.3 partial fill)
- Total review count: "(2,847 reviews)" or "Based on 2,847 reviews"
- Position: left of histogram or above histogram as a header element
- FORBIDDEN: rounding aggregate to whole number (4.3 displayed as "4 stars")
- FORBIDDEN: displaying aggregate without total count (the count is the credibility anchor)

**Step 8:** Design responsive behavior:
- Desktop: histogram and aggregate side-by-side (aggregate left, histogram right) or stacked
- Mobile: stacked (aggregate on top, histogram below), full-width bars
- Touch targets: bars must be at least 44px tall for reliable tap (mobile accessibility)
- Count labels: remain visible at all viewport widths (never truncated)

**Step 9:** Document the complete histogram specification.

## Quality Criteria
- All 5 star rating rows displayed (including zero-count)
- Bars are tappable filters (not static display)
- Star distribution passes realism check (no 95%+ five-star, no 0% one-star)
- Aggregate rating shown with one decimal + total count
- Active/inactive visual states clearly defined
- Deselect behavior documented (tap again to remove filter)
- Combined filtering with keyword chips specified
- Responsive behavior documented with 44px minimum touch target

## Failure Modes
- Star distribution has 0% one-star -> HALT. Adjust distribution to include minimum 3-5% one-star per Realism Table.
- Star distribution has 95%+ five-star -> HALT. Adjust to maximum 65% five-star per Realism Table.
- Histogram bars not specified as tappable -> HALT. Tappable filtering is mandatory Baymard requirement.
- Any star rating row hidden -> HALT. All 5 rows must display regardless of count.
- Aggregate rating displayed without review count -> HALT. Count is credibility anchor.
- Aggregate rating rounded to whole number -> HALT. Display one decimal place.
