# 3.2 — Specificity Check

**Skill:** PDP-05-social-proof-review-system
**Position:** Layer 3, Microskill 2
**Purpose:** Verify that every generated review contains measurable outcomes, concrete product details, and personal context — not vague praise or generic sentiment. A review that says "this product is amazing" fails. A review that says "my joint pain dropped from a 7 to a 3 within 2 weeks" passes. This is the qualitative health check — does every review sound like a REAL person describing a REAL experience?
**Effort Level:** high
**Dependencies:** `input-verification.md` (0.1), `featured-reviews.md` (2.1), `reviewer-personas.md` (2.3), `review-card-template.md` (2.5), `volume-audit.md` (3.1)
**Output:** `specificity-check.md`

## Execution Logic

**Step 1:** Collect all generated review content that needs specificity validation:
- All featured reviews from `featured-reviews.md` (2.1)
- Any general pool review samples specified in `review-system-architecture.md` (1.1)
- All Q&A answers (brand and community) from `qa-section-architecture.md` (2.4)

**Step 2:** Define the 5-dimension specificity framework. Every review is scored on each:

| Dimension | What It Means | Pass Example | Fail Example |
|-----------|--------------|--------------|--------------|
| **Measurable Outcome** | Contains numbers, timeframes, before/after, or quantifiable change | "Pain dropped from 7/10 to 3/10 in 2 weeks" | "It really helped with my pain" |
| **Product Detail** | References specific product attributes (ingredients, features, form factor, taste, texture) | "The vanilla flavor doesn't have that chalky aftertaste" | "It tastes good" |
| **Personal Context** | Includes reviewer's situation, why they bought, what they tried before | "After 3 knee surgeries and trying glucosamine with no luck" | "I decided to try this product" |
| **Temporal Specificity** | Mentions specific timeframes for usage, results, or ownership | "Started noticing changes around day 10, full effect by week 4" | "After using it for a while" |
| **Sensory/Experiential Detail** | Describes specific sensory experiences (how it felt, tasted, looked, worked in practice) | "The capsules are small enough to swallow without water" | "Easy to take" |

**Step 3:** Score each featured review on all 5 dimensions (1-3 scale per dimension):

| Score | Meaning |
|-------|---------|
| 3 | Strong — specific, concrete, measurable |
| 2 | Adequate — somewhat specific but could be sharper |
| 1 | Weak — vague, generic, or missing |

**Step 4:** Calculate per-review specificity score:
- Sum of all 5 dimensions = raw score (5-15 range)
- Specificity Score = raw score / 15 * 10 (normalized to 0-10 scale)
- Minimum passing: 7.0/10 per featured review
- Minimum passing: 6.0/10 per general pool review sample
- HALT if ANY featured review scores below 7.0

**Step 5:** Run the vague praise detector. Scan all review text for these patterns:

**FORBIDDEN PHRASES (immediate fail if found in any review):**
- "This product is amazing"
- "Highly recommend"
- "Best product ever"
- "Works great"
- "Love it"
- "Five stars"
- "Changed my life" (without specific explanation of HOW)
- "Can't live without it"
- "Worth every penny" (without specific value comparison)
- "Better than expected" (without saying what was expected)
- "Exceeded my expectations" (AI telltale — also caught by 3.4)

**YELLOW FLAG PHRASES (acceptable ONLY if followed by specific detail):**
- "Really helped" — OK if followed by "with my [specific condition]"
- "Great product" — OK if followed by "because [specific reason]"
- "Noticed a difference" — OK if followed by "in [specific timeframe] for [specific outcome]"
- "Would buy again" — OK if followed by "especially for [specific use case]"

**Step 6:** Run the outcome specificity test on each featured review:

| Test | Pass Criteria | Fail Criteria |
|------|--------------|---------------|
| Contains a number | At least 1 specific number (%, days, mg, hours, rating) | No numbers anywhere in review |
| Contains a timeframe | At least 1 time reference (days, weeks, months, "by day X") | No temporal markers |
| Contains a comparison | Before/after, vs competitor, vs previous product, vs expectation | No comparative reference |
| Contains product attribute | References specific ingredient, feature, design element, or form factor | Only mentions product by name |
| Contains personal detail | Age, condition, lifestyle, purchase reason, prior experience | Generic "I bought this" |

Minimum for featured reviews: 4 of 5 tests pass.
Minimum for general pool reviews: 3 of 5 tests pass.

**Step 7:** Check persona-review alignment:
- Does each featured review match its assigned reviewer persona (from 2.3)?
- Does the review voice match the persona's archetype (Enthusiast = excited but specific, Analytical = data-driven, Skeptic Convert = initially doubtful then convinced, etc.)?
- Does the review mention details consistent with the persona's demographics (age-appropriate concerns, lifestyle-appropriate use case)?
- FAIL if review content contradicts persona attributes (e.g., persona is 25-34 but review mentions "after decades of trying everything")

**Step 8:** Check headline specificity (from review card template 2.5):
- Every featured review headline MUST contain a specific outcome or product detail
- PASS: "Joint Pain Down 60% After 3 Weeks" or "Finally a Protein Powder My Kids Will Drink"
- FAIL: "Great Product!" or "Love It!" or "Five Stars"
- Headline specificity test: Does the headline alone tell you WHAT improved and by HOW MUCH or in WHAT WAY?

**Step 9:** Check pros/cons specificity:
- Every pro MUST be specific (not "good quality" but "capsule size is small, easy to swallow")
- Every con MUST be specific and realistic (not "none" or "can't think of any" but "took 3 weeks to notice anything" or "unflavored version tastes chalky")
- FAIL if any pro is a single generic word ("quality," "effective," "good")
- FAIL if any con is dismissive or non-specific

**Step 10:** Calculate overall specificity check result:

| Metric | Weight | Calculation |
|--------|--------|-------------|
| Featured review avg score | 40% | Average of all featured review specificity scores |
| Vague praise violations | 20% | 10 - (2 * count of forbidden phrases found) |
| Outcome specificity pass rate | 20% | (tests passed / total tests) * 10 |
| Headline specificity | 10% | All pass = 10, per fail -3.3 |
| Pros/cons specificity | 10% | All pass = 10, per fail -2.5 |

Overall specificity score = weighted average. Minimum passing: 7.5/10.

**Step 11:** Document all findings, per-review scores, flagged phrases, and the overall specificity result.

## Quality Criteria
- All featured reviews individually scored on 5-dimension specificity framework
- Every featured review scores >= 7.0/10 individually
- All review text scanned for forbidden vague praise phrases
- Outcome specificity tests run on every review (4/5 minimum for featured, 3/5 for general)
- Persona-review alignment verified (voice match + demographic match)
- Headlines checked for specific outcomes (not generic praise)
- Pros/cons checked for specificity (no generic words, no dismissive cons)
- Overall specificity score >= 7.5/10
- All FAIL items have specific remediation actions with rewrite guidance

## Failure Modes
- Any featured review scores below 7.0/10 specificity -> HALT. Rewrite with specific outcomes, numbers, and timeframes.
- Forbidden vague phrase found in featured review -> HALT. Remove phrase and replace with specific detail.
- Featured review fails 2+ outcome specificity tests -> HALT. Add missing specificity dimensions.
- Review voice contradicts assigned persona -> HALT. Rewrite to match persona archetype and demographics.
- Headline is generic praise without specific outcome -> HALT. Rewrite headline with measurable result.
- Con section says "None" or equivalent -> HALT. Add realistic, specific minor con.
- Overall specificity score below 7.5/10 -> HALT. Address all flagged reviews before proceeding to Layer 4.
- All featured reviews have identical specificity patterns (same dimensions strong, same weak) -> HALT. Vary which dimensions each review emphasizes.
