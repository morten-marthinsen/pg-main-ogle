# 2.5 — Review Card Template

**Skill:** PDP-05-social-proof-review-system
**Position:** Layer 2, Microskill 5
**Purpose:** Define the per-card layout specification for every review in the system — star display, bold headline, reviewer attributes, pros/cons structure, body text, badges (Verified Buyer + Time Owned), helpful votes, and merchant response insertion rules. This template is the atomic unit of the entire review system. Every review card follows this spec exactly.
**Effort Level:** medium
**Dependencies:** `input-verification.md` (0.1), `pdp-reference-load.md` (0.3), `review-system-architecture.md` (1.1), `featured-reviews.md` (2.1), `reviewer-personas.md` (2.3)
**Output:** `review-card-template.md`

## Execution Logic

**Step 1:** Define the card anatomy. Every review card has these sections in this EXACT order:

```
┌─────────────────────────────────────────────────────────┐
│ SECTION 1: HEADER ROW                                    │
│ [★★★★☆] [Bold Headline — max 80 chars]                  │
├─────────────────────────────────────────────────────────┤
│ SECTION 2: REVIEWER LINE                                 │
│ [Name] | [Verified Buyer ✓] | Owned for [duration] | [Location] │
├─────────────────────────────────────────────────────────┤
│ SECTION 3: REVIEWER ATTRIBUTES (product-specific)        │
│ [Age: 35-44] [Skin Type: Oily] [Goal: Anti-aging]       │
├─────────────────────────────────────────────────────────┤
│ SECTION 4: PROS / CONS                                   │
│ Pros:                                                    │
│ • [specific pro 1]                                       │
│ • [specific pro 2]                                       │
│ Cons:                                                    │
│ • [specific con — MANDATORY]                             │
├─────────────────────────────────────────────────────────┤
│ SECTION 5: BODY TEXT                                     │
│ [2-5 sentences with specific experience and outcomes]    │
├─────────────────────────────────────────────────────────┤
│ SECTION 6: ACTIONS ROW                                   │
│ [Helpful? Yes (47) | No (3)]     [Report] [Share]        │
├─────────────────────────────────────────────────────────┤
│ SECTION 7: MERCHANT RESPONSE (conditional — <= 3 stars)  │
│ [Brand Logo/Name] responded on [date]:                   │
│ "[Empathetic response with resolution offer]"            │
└─────────────────────────────────────────────────────────┘
```

**Step 2:** Define Section 1 — Header Row:
- Star display: 1-5 filled stars (filled = brand accent or gold, empty = gray outline)
- Star size: 16-18px
- Stars aligned LEFT
- Headline: bold, 16-18px, same line as stars or immediately below
- Headline max length: 80 characters
- Headline requirements: specific outcome (from 2.1 rules), NOT generic praise
- If no headline provided: display body text first line as fallback (truncated at 80 chars with ellipsis)

**Step 3:** Define Section 2 — Reviewer Line:
- Reviewer name: first name + last initial (e.g., "Sarah M.") or full name if provided
- Verified Buyer badge: green checkmark icon + "Verified Buyer" text
- Badge position: after name, before Time Owned
- Time Owned: "Owned for [N] months" or "Purchased [N] weeks ago"
- Location: optional — "City, State" or "State" (e.g., "Austin, TX" or "California")
- Separator: pipe character "|" or bullet "·" between elements
- Font: 13-14px, muted text color (not as prominent as headline)

**Step 4:** Define Section 3 — Reviewer Attributes:
- Product-specific attribute tags displayed as inline pills or plain text
- Attributes vary by product category:

| Product Category | Attribute Examples |
|-----------------|-------------------|
| Skincare | Skin Type, Age Range, Skin Concern, Climate |
| Supplement | Age Range, Health Goal, Activity Level, Diet Type |
| Electronics | Use Case, Tech Level, Prior Device, Environment |
| Apparel | Body Type, Height Range, Style Preference, Climate |
| Fitness Equipment | Fitness Level, Space (Apartment/House), Training Type |
| Food/Beverage | Dietary Restriction, Taste Preference, Frequency of Use |

- Display format: tag pills (rounded, light background, small text) or comma-separated text
- Max attributes shown: 3-4 (additional behind "Show more" or omitted)
- If no attributes available: omit section entirely (do not show empty row)

**Step 5:** Define Section 4 — Pros/Cons:
- Section header: "Pros" and "Cons" labels in bold or semi-bold
- Pros: green checkmark icon + bullet text, 1-3 items
- Cons: red/orange X icon or minus icon + bullet text, 1-2 items
- MANDATORY: every review card MUST have at least 1 con (even 5-star reviews)
- Font: 14-15px body text
- Pros always listed BEFORE cons
- Visual separation: slight indent or icon differentiation (green vs red/orange)
- If review has no structured pros/cons: extract from body text and structure retroactively
- FORBIDDEN: "None" or "Can't think of any" as a con entry (must be a real minor issue)
- FORBIDDEN: empty cons section on any card

**Step 6:** Define Section 5 — Body Text:
- Font: 14-16px body text, standard line height (1.5-1.6)
- Length: 2-5 sentences for featured reviews, 1-5 sentences for general reviews
- Truncation: if body exceeds 3 lines on desktop (or 4 on mobile), truncate with "Read more" expand link
- "Read more" expands inline (no modal, no page navigation)
- Body text requirements (from 2.1): specific measurable outcomes, personal context, natural voice
- FORBIDDEN: marketing language, AI-telltale phrases, uniform sentence length

**Step 7:** Define Section 6 — Actions Row:
- "Helpful?" prompt with Yes/No buttons + vote counts
- Yes button: thumbs up icon + "(N)" count
- No button: thumbs down icon + "(N)" count
- Interaction: tap to vote (one vote per user, toggleable)
- Visual feedback: filled icon after voting
- Report link: "Report" text link (far right, de-emphasized)
- Share: share icon (optional — some implementations omit)
- Font: 12-13px, muted color
- Position: bottom of card, full width, left-aligned helpful + right-aligned report/share

**Step 8:** Define Section 7 — Merchant Response (conditional):
- Trigger: ONLY on reviews rated 3 stars or below
- Format: indented block below the review card body, visually distinct
- Visual treatment: left border (brand color, 3-4px), light background tint, or slight indent
- Header: "[Brand Name] responded on [date]:" in bold or semi-bold
- Content: 2-4 sentences — acknowledge concern, offer resolution, provide contact
- Tone: empathetic and solution-oriented, NOT defensive, NOT corporate boilerplate
- FORBIDDEN: "We're sorry to hear about your experience" as sole content (must offer specific resolution)
- FORBIDDEN: merchant response on 4-star or 5-star reviews (clutters positive reviews)
- If 4-star review has a significant con: merchant response is OPTIONAL (not mandatory)

**Step 9:** Define card visual hierarchy:
- Background: white or very light gray card
- Border: subtle 1px border or shadow (card elevation)
- Padding: 16-20px on all sides
- Spacing between cards: 12-16px vertical gap
- Card width: full-width of review column (not fixed width)
- Featured review cards: slightly elevated treatment (subtle highlight border or "Featured Review" badge at top-right)

**Step 10:** Define featured vs general card differences:

| Element | Featured Card | General Card |
|---------|--------------|--------------|
| Position | Top of review list (before general) | Below featured section |
| Badge | "Featured Review" or "Staff Pick" | None |
| Border | Accent color left border or top highlight | Standard border |
| Attributes | Always shown (3-4) | Optional (may be empty) |
| Body length | Full display (no truncation) | Truncated at 3 lines with "Read more" |
| Headline | REQUIRED (mandatory specific outcome) | Optional (some reviews may lack headlines) |
| Pros/Cons | REQUIRED (structured) | Structured if available, extracted from body if not |

**Step 11:** Define responsive behavior:
- Desktop: cards in single column (full-width of review area)
- Tablet: same single column
- Mobile: same single column, full-width, increased padding (16px horizontal)
- Touch targets: all interactive elements (helpful buttons, report, read more) minimum 44px
- Star size: 14-16px on mobile (slightly smaller)
- Font sizes: body 14px minimum on mobile (never smaller)
- Truncation breakpoint: 4 lines on mobile (vs 3 on desktop) before "Read more"

**Step 12:** Define loading states:
- Skeleton card: gray placeholder bars matching card anatomy (star row, headline bar, text bars, action bar)
- Loading indicator: subtle pulse animation on skeleton
- Load transition: fade-in (200ms) when real card replaces skeleton

**Step 13:** Document the complete review card template specification.

## Quality Criteria
- All 7 card sections defined in exact order (header, reviewer, attributes, pros/cons, body, actions, merchant response)
- Star display specification complete (filled/empty, size, color)
- Headline requirements enforced (max 80 chars, specific outcome, not generic)
- Verified Buyer badge + Time Owned on every card (mandatory)
- Pros/Cons section requires at least 1 con on every card (even 5-star)
- Body text truncation specified with "Read more" expand
- Merchant response triggers only for <= 3 star reviews
- Featured vs general card differences documented
- Responsive behavior with 44px touch targets
- Loading/skeleton state specified
- Template validated against 2.1 featured reviews

## Failure Modes
- Card missing Verified Buyer badge -> HALT. Badge is mandatory on every card.
- Card missing Time Owned -> HALT. Time Owned is mandatory credibility signal.
- Cons section empty or "None" -> HALT. At least 1 realistic con per card.
- Merchant response on 5-star review -> HALT. Only <= 3 stars trigger merchant response.
- Headline exceeds 80 characters -> HALT. Truncate or rewrite.
- No truncation spec for body text -> HALT. Long reviews must truncate with "Read more."
- Card sections in wrong order -> HALT. Must follow exact 7-section sequence.
- Featured and general cards identical -> HALT. Featured cards need elevated treatment.
- Touch targets below 44px -> HALT. Accessibility requirement.
