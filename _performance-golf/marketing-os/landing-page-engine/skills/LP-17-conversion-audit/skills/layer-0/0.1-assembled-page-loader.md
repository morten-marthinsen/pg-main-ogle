# 0.1 — Assembled Page Loader

> **Skill:** LP-17-conversion-audit
> **Layer:** 0
> **Position:** 1 of 4
> **Purpose:** Load the assembled page package from LP-15 and extract all sections, elements, and copy for audit evaluation.
> **Effort Level:** haiku
> **Dependencies:** `assembled-page-package.json` from LP-15 Page Assembly
> **Output:** `assembled-page-load.md`

---

## INPUT

```yaml
input:
  assembled_page_path: "[path to assembled-page-package.json from LP-15]"
  project_folder: "[path to project output folder]"
```

---

## EXECUTION

1. Locate `assembled-page-package.json` in the project output folder. Search paths:
   - `./outputs/[project-name]/landing-page/lp-15-page-assembly/assembled-page-package.json`
   - `./outputs/[project-name]/lp-15-page-assembly/assembled-page-package.json`

2. Parse the assembled page package. Extract and catalog:
   - **Page metadata:** project name, page type, section count, total estimated word count
   - **Section inventory:** List every section in order with: section name, section position, section purpose, estimated word count
   - **Above-fold elements:** headline text, subheadline/deck text, hero image description, CTA text + position, trust signals, price display (if present)
   - **Copy blocks:** All headline copy, body copy blocks, CTA text, guarantee text, FAQ content
   - **Proof elements:** All testimonials, reviews, citations, endorsements, stats, before/after references
   - **Offer elements:** Pricing, bundles, subscription options, bonuses, guarantee, urgency elements
   - **Technical specs:** Image format directives, mobile specifications, load optimization notes

3. If `assembled-page-package.json` is not found, search for alternative assembly outputs:
   - `assembled-page-brief.md`
   - `ASSEMBLED-PAGE-SUMMARY.md`
   - Any file in the LP-15 output folder

4. Write the extracted inventory to `assembled-page-load.md`.

## QUALITY CRITERIA

- Assembled page package located and fully parsed
- All sections inventoried with position and purpose
- All copy blocks extracted verbatim (headlines, CTAs, guarantee, FAQ)
- All proof elements cataloged with type and content
- Technical specifications noted
- File exists after execution

---

## FAILURE MODES

- Assembled page package not found -> Write `assembled-page-load.md` with status `NOT_FOUND` and list all paths searched. Triggers GATE_0 HALT.
- Package found but malformed/unparseable -> Write with status `PARTIAL`, extract what is available, note unparseable sections. Triggers GATE_0 review.
- Package is extremely large (>50 sections) -> Extract in structured chunks, prioritize above-fold and proof sections.
