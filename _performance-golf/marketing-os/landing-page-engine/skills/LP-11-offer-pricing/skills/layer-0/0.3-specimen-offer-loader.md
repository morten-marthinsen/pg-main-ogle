# 0.3 — Specimen Offer Loader

**Skill:** LP-11-offer-pricing
**Position:** Layer 0, Microskill 3
**Purpose:** Load type-matched and vertical-matched pricing/offer specimens from the specimen library to serve as structural and tonal references for Layer 2 generation. Specimens ground the AI in real pricing copy patterns rather than generic templates.
**Effort Level:** low
**Dependencies:** `packages-loaded.md` (0.1), specimen library in `LandingPageEngine/specimens/`
**Output:** `specimens-loaded.md`

---

## Input Schema

```yaml
input:
  page_type: "[type_a | type_b | hybrid — from packages-loaded.md]"
  vertical: "[from packages-loaded.md]"
  offer_type: "[from offer-architecture-loaded.md]"
  specimen_directory: "LandingPageEngine/specimens/"
  swipe_vault_index: "LandingPageEngine/swipe-vault-index.md"
```

---

## Execution Logic

1. **Determine specimen needs based on page type:**

   **Type A specimens needed:**
   - 2-3 value stack examples (component descriptions with value anchors)
   - 1-2 "if all it did was" framework examples (any vertical — framework is universal)
   - 1-2 price reveal sequences (the copy moment when price is unveiled)
   - 1 bonus description section example

   **Type B specimens needed:**
   - 2-3 pricing tier displays (bundle/subscription pricing patterns)
   - 1-2 subscription savings examples
   - 1 savings callout/crossed-out price example
   - 1 bonus/free-gift display example (if applicable)

2. **Search specimens by vertical match first.** Prioritize specimens from the same vertical as the project. If insufficient specimens match the vertical, expand to adjacent verticals.

3. **Prioritized specimen sources for Type B pricing:**
   - `bioptimizers-magnesium-trial-raw.md` — 3-tier bundle pricing, crossed-out prices, subscription savings, free bonuses with first-month qualifier
   - `lmnt-electrolyte-pdp-raw.md` — DTC subscription savings, INSIDER bundle, per-stick pricing
   - `native-build-your-pack-raw.md` — Build-your-pack bundle mechanics
   - `meandmygolf-fix-your-slice-raw.md` — Trial-to-subscription, curriculum-based value display

4. **Prioritized specimen sources for Type A pricing:**
   - `dean-graziosi-ai-advantage-club-raw.md` — VSL-first with aggressive order bumps, authority-driven value proposition
   - CopywritingEngine TIER1 vault (if available) — value stack patterns from long-form sales letters
   - Cross-page pattern analysis for offer section patterns

5. **For each loaded specimen, extract:**
   - The pricing display pattern (how price is shown)
   - The value communication approach (how value is framed before/around price)
   - Savings callout format (how discounts are displayed)
   - Bonus presentation pattern (how bonuses are described)
   - The specific copy language used (verbatim key phrases)

6. **Load at least 2 specimens total.** If the specimen library has fewer than 2 relevant specimens, note the gap and reference the SWIPE-VAULT-INDEX for additional pattern guidance.

7. **Write `specimens-loaded.md`** with extracted patterns organized by specimen source.

## Quality Criteria
- At least 2 specimens loaded with extracted patterns
- Specimens are type-matched (Type A specimens for Type A pages, Type B for Type B)
- Key phrases are verbatim, not paraphrased — these serve as tonal anchors for generation
- Cross-specimen patterns identified — consistent patterns are higher confidence
- Any gaps are noted with mitigation sources

## Failure Modes
- Zero specimens loaded -> Flag as WARNING and reference SWIPE-VAULT-INDEX + CROSS-PAGE-PATTERN-ANALYSIS. Generation can proceed with framework templates from AGENT.md, but quality will be lower.
- All specimens from wrong vertical -> Acceptable if page type matches. Cross-vertical specimens are useful for structural patterns even if the product category differs.
- Specimens loaded but pricing patterns not extracted -> Re-read specimens with focus on the pricing/offer sections specifically. The full-page specimen may have the pricing content buried in later sections.
