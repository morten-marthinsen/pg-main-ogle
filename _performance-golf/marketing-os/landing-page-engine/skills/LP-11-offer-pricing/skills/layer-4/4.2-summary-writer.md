# 4.2 — Summary Writer

**Skill:** LP-11-offer-pricing
**Position:** Layer 4, Microskill 2
**Purpose:** Write the human-readable summary of the pricing section for quick review. This file gives the human reviewer (or any reviewer) a complete picture of the pricing section without requiring them to parse the full JSON package.
**Effort Level:** low
**Dependencies:** `pricing-section-package.json` (4.1), `price-value-audit.md` (3.2)
**Output:** `PRICING-SECTION-SUMMARY.md`

---

## Input Schema

```yaml
input:
  pricing_package: "[from pricing-section-package.json — compiled output]"
  price_value_audit: "[from price-value-audit.md — score and evidence]"
  page_type: "[type_a | type_b | hybrid]"
```

---

## Execution Logic

1. **Extract key data from pricing-section-package.json** for each section of the summary.

2. **Write the summary following the template defined in LP-11-AGENT.md.** Include all required sections:
   - Page type and pricing display strategy
   - Value stack table (Type A)
   - "If all it did was" benefit table (Type A)
   - Price point and value-to-price ratio
   - Pricing tiers table (Type B)
   - Subscription details (if applicable)
   - Bonuses table
   - Savings callouts list
   - Guarantee reference
   - Audit results
   - Downstream handoffs

3. **Keep the summary scannable.** Use tables, not paragraphs. Use bullet points, not prose. A reviewer should be able to assess the pricing section in under 2 minutes.

4. **Include the actual copy for high-impact elements:**
   - The price reveal sentence (Type A)
   - The "if all it did was" transition (Type A)
   - The guarantee reference sentence
   - The recommended tier copy (Type B)
   These are the highest-leverage copy elements and deserve to be visible in the summary.

5. **Flag any concerns or observations** that emerged during the build but were not blocking failures. These go in a "Notes" section at the bottom.

## Quality Criteria
- All required sections present (skip sections marked N/A for page type)
- Tables used for structured data (not paragraphs)
- High-impact copy elements quoted directly
- Summary is reviewable in under 2 minutes
- Audit results match price-value-audit.md exactly

## Failure Modes
- Summary missing a section -> Check against the template. Every section must be present (mark N/A if not applicable).
- Copy quoted in summary doesn't match package -> All quoted copy must be verbatim from pricing-section-package.json.
- Audit score in summary doesn't match audit file -> Copy the score exactly from price-value-audit.md.
