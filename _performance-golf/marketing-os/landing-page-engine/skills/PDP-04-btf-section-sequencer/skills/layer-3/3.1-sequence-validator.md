# 3.1 — Sequence Validator

**Skill:** PDP-04-btf-section-sequencer
**Position:** Layer 3, Microskill 1
**Purpose:** Run all 4 PDP-specific sequence logic tests AND the 10-point PDP sequence audit. This is the structural integrity check — it verifies that the section ordering, proof architecture, conversion architecture, and proportions all meet PDP requirements. All 4 logic tests must PASS and the sequence score must be >= 7.0/10 before the sequence can proceed to package assembly.
**Effort Level:** high (opus)
**Dependencies:** `section-ordering.md` (1.2), `optional-decisions.md` (1.4), `word-count-plan.md` (2.1), `proof-density-plan.md` (2.2), `visual-density-plan.md` (2.3), `cta-placement-plan.md` (2.4), `sticky-atc-plan.md` (2.5)
**Output:** `sequence-validation.md`

## Execution Logic

### PART A: 4-Test PDP Sequence Logic Validator

Run each test explicitly. Do NOT skip any test. Do NOT assume a test passes based on visual inspection.

**Test 1: Scannable Sections First**
- **Question:** Do the top 2-3 BTF sections serve hunt-and-peck visitors?
- **Check:** Are sections 1-3 from the set {product_highlights, ugc_video_carousel, the_expert, reviews_ratings}?
- **Pass condition:** At least 2 of the first 3 positions are scannable/social-proof sections
- **Fail condition:** Education or comparison sections appear in positions 1-3
- **Document:** List sections in positions 1, 2, 3 with their section_ids

**Test 2: Authority Before Education**
- **Question:** Does at least one authority signal appear before the first education section?
- **Check:** Find the position of the first education section (any of: the_problem, the_solution, why_it_works, whats_in_it, what_to_expect). Find the position of the first authority section (any of: the_expert, ugc_video_carousel, reviews_ratings).
- **Pass condition:** authority_position < first_education_position
- **Fail condition:** Education begins with no preceding trust/authority signal
- **Document:** Which authority section (position X) precedes which education section (position Y)

**Test 3: Value Before Comparison**
- **Question:** Is the product's value proposition established before the comparison chart?
- **Check:** If comparison_chart is in the sequence, find its position. Count how many of {whats_in_it, what_to_expect, why_it_works} appear before it.
- **Pass condition:** At least 2 of the 3 value sections appear before comparison_chart
- **Fail condition:** Comparison chart appears before product education, OR fewer than 2 value sections precede it
- **If comparison_chart not in sequence:** Test PASS (auto-pass — nothing to validate)
- **Document:** Which value sections (positions) appear before comparison chart (position)

**Test 4: FAQ Last (or Near-Last)**
- **Question:** Is FAQ in the final 3 BTF positions?
- **Check:** FAQ position >= (total_btf_sections - 2)
- **Pass condition:** FAQ is in position N, N-1, or N-2 (where N = total sections)
- **Fail condition:** FAQ is buried in the middle of the sequence
- **Document:** FAQ position X of N total sections

### PART B: 10-Point PDP Sequence Audit

Score each point 0 (fail) or 1 (pass):

**Module Independence (3 points)**
1. Every section has a documented persuasion function that is standalone (no "as mentioned above" language)
2. Each section has a UNIQUE persuasion function (no two sections do the same job)
3. A visitor arriving at any section gets immediate value without reading preceding sections

**Proof Architecture (3 points)**
4. At least one heavy proof section in the first 3 BTF positions
5. Reviews & Ratings section present with volume proof spec (25+ reviews specified in proof density plan)
6. At least 3 different proof types across all sections (check proof-density-plan.md)

**Conversion Architecture (2 points)**
7. Sticky ATC trigger point defined with scroll-depth rationale (check sticky-atc-plan.md exists and has trigger depth)
8. >= 2 mid-page CTA placements mapped (check cta-placement-plan.md, not counting sticky bar)

**Proportions (2 points)**
9. All section word counts within PDP benchmark ranges (check word-count-plan.md benchmark check)
10. No single section exceeds 25% of total BTF word count

### PART C: Calculate sequence score and determine result

```
sequence_score = sum of all 10 audit points
IF sequence_score >= 9: "Sequence ready for downstream writing"
IF sequence_score >= 7: "Minor adjustments possible, can proceed"
IF sequence_score >= 5: "Structural issues — revise before proceeding"
IF sequence_score < 5: "Sequence rebuild required"
```

## Quality Criteria
- All 4 logic tests explicitly run — not assumed or visually inspected
- Each test has documented evidence (section positions, counts, comparisons)
- All 10 audit points scored with evidence
- Sequence score calculated correctly (sum of points)
- Gate 3 eligibility clearly stated
- Any failures have specific resolution instructions

## Failure Modes
- Any logic test FAILS -> Document which test failed and what needs to change. Do NOT proceed past GATE_3.
- Sequence score < 7.0 -> Identify which audit points failed and provide resolution instructions.
- Module independence audit finds "as mentioned above" language -> FAIL point 1. All section descriptions must be rewritten as standalone.
- Proof density plan missing or vague -> Cannot score points 4-6. HALT and return to 2.2.
- CTA placement plan missing -> Cannot score points 7-8. HALT and return to 2.4/2.5.
- Score calculated incorrectly (e.g., 8/10 but only 7 points scored) -> Recount. Math must be exact.
