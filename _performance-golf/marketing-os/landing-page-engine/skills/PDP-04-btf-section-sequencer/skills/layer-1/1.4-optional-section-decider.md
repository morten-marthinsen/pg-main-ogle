# 1.4 — Optional Section Decider

**Skill:** PDP-04-btf-section-sequencer
**Position:** Layer 1, Microskill 4
**Purpose:** Make final include/exclude decisions for every optional section. Each optional section requires a strategic justification tied to the specific product, audience, and brief — not a generic "it could be useful" rationale. This microskill produces the final, locked section list that Layer 2 proportioning will use. No sections can be added or removed after this point without re-running Layer 1.
**Effort Level:** medium
**Dependencies:** `taxonomy-selection.md` (1.1), `section-ordering.md` (1.2), `mandatory-check.md` (1.3), `brief-verified.md` (0.1)
**Output:** `optional-decisions.md`

## Execution Logic

**Step 1:** Identify all optional sections from the taxonomy (those NOT mandatory or conditional-mandatory):
- `the_problem` — Agitation section
- `cost_savings` — Bundle pricing / affordability
- `how_its_made` — Quality/sourcing proof
- `identity_matching` — "Who Is This For?" self-selection
- `the_proof` — Clinical/survey data evidence

**Step 2:** For each optional section, apply the decision framework:

### the_problem (Agitation)
- **INCLUDE if:** Awareness stage 1-3 AND the product solves a problem the audience may not fully understand
- **EXCLUDE if:** Awareness stage 4-5 (audience already problem-aware) OR the_solution section alone sufficiently frames the problem
- **CONSOLIDATION option:** If included, consider consolidating with the_solution and why_it_works into a Problem->Solution->Why flow group
- **Dead weight test:** Does this section add a persuasion function NOT already served by other sections? If the_solution already agitates the problem, this section is dead weight.

### cost_savings (Bundle Pricing)
- **INCLUDE if:** Bundle offer exists AND price point justifies a savings visualization
- **EXCLUDE if:** Single SKU only OR no bundle pricing structure
- **Dead weight test:** If there is no bundle, this section serves no function.

### how_its_made (Quality/Sourcing)
- **INCLUDE if:** Premium positioning AND (manufacturing process is a differentiator OR certifications/sourcing are selling points OR quality-conscious audience)
- **EXCLUDE if:** Commoditized product OR manufacturing is standard/unremarkable
- **Dead weight test:** If the answer to "Would knowing how it's made change the purchase decision?" is no, this section is dead weight.

### identity_matching ("Who Is This For?")
- **INCLUDE if:** Broad audience that could segment into 3+ distinct personas AND self-selection helps conversion (reduces "is this for me?" friction)
- **EXCLUDE if:** Narrow audience that already self-selects by finding the page OR product type makes the audience obvious
- **Dead weight test:** If every visitor who reaches this PDP already knows the product is for them, this section adds nothing.

### the_proof (Clinical/Survey Data)
- **INCLUDE if:** Clinical studies, survey data, or formal research exists AND the audience values evidence (health/supplement/clinical products)
- **EXCLUDE if:** No formal data exists OR product is lifestyle/fashion where data doesn't drive purchase decisions
- **Dead weight test:** If data doesn't exist, do NOT include this section with "coming soon" or placeholder content.

**Step 3:** For each INCLUDE decision, assign a strategic justification (1-2 sentences explaining WHY this section serves a purchase-decision function for THIS specific product).

**Step 4:** For each EXCLUDE decision, confirm the dead weight test passes (the section truly adds no unique persuasion function).

**Step 5:** Finalize the complete section list: mandatory sections (from 1.3) + included optional sections. Verify the final count is 8-14.

**Step 6:** Update the section ordering from 1.2 with any newly included or excluded optional sections. Maintain ordering principles.

## Quality Criteria
- Every optional section has a clear INCLUDE/EXCLUDE decision — no "maybe" or "TBD"
- Every INCLUDE has a strategic justification tied to THIS product, not generic reasoning
- Every EXCLUDE passes the dead weight test — the section truly adds nothing unique
- Final section count is 8-14
- Final section list matches the ordering from 1.2 (with any updates from optional decisions)
- Section list explicitly marked as LOCKED

## Failure Modes
- Optional section included without strategic justification -> HALT. Every included section must serve a documented persuasion function.
- Optional section included but fails dead weight test -> Remove the section. Dead weight sections are forbidden on PDPs.
- Final section count below 8 -> WARNING. The PDP may be too sparse. Review if any conditional sections should be included.
- Final section count above 14 -> WARNING. Review optional inclusions — trim any section where the strategic justification is weak.
- "TBD" or "maybe" left as a decision -> HALT. All decisions must be finalized before GATE_1.
