# 1.1 — Section Taxonomy Selector

**Skill:** PDP-04-btf-section-sequencer
**Position:** Layer 1, Microskill 1
**Purpose:** Map the specific product against the NLS 16-Section PDP BTF Taxonomy to determine which of the 16 sections apply. For each section, produce a clear INCLUDE/EXCLUDE/CONDITIONAL decision based on product type, proof inventory, and brief data. This is the first structural decision — it determines the section set that all downstream microskills operate on.
**Effort Level:** medium
**Dependencies:** `brief-verified.md` (0.1), `pdp-reference-loaded.md` (0.3)
**Output:** `taxonomy-selection.md`

---

## Input Schema

```yaml
input:
  product_name: "[from brief-verified.md]"
  product_type: "[supplement | physical-product | beauty | food-bev | consumer-goods]"
  vertical: "[from brief-verified.md]"
  proof_inventory: "[from brief-verified.md — section eligibility signals]"
  taxonomy_reference: "[16-section taxonomy from pdp-reference-loaded.md]"
```

---

## Execution Logic

**Step 1:** Load the 16-section taxonomy from `pdp-reference-loaded.md`. For each section, apply the decision logic:

### Always Mandatory (include regardless):
- `product_highlights` — Every PDP needs a TLDR scannable section
- `what_to_expect` — Every PDP needs benefit timeline/expectation setting
- `the_solution` — Every PDP needs its USP/differentiation statement
- `why_it_works` — Every PDP needs logical mechanism proof
- `how_to_use` — Every PDP needs usage instructions (friction reduction)
- `faq` — Every PDP needs final objection handling

### Conditional Mandatory (include if condition met):
- `ugc_video_carousel` — INCLUDE if `ugc_video_count >= 3`. EXCLUDE if fewer than 3 UGC videos.
- `the_expert` — INCLUDE if expert endorser exists (`has_expert_endorsement = Y`). EXCLUDE if no expert.
- `whats_in_it` — INCLUDE if product is supplement or formulated product. EXCLUDE for non-formulated physical goods.
- `comparison_chart` — INCLUDE if competitors identified (`has_comparison_competitors = Y`). EXCLUDE if no competitors.
- `reviews_ratings` — INCLUDE if 25+ reviews exist (`testimonials_count >= 25`). EXCLUDE if insufficient reviews.

### Optional (include based on strategic fit):
- `the_problem` — INCLUDE if awareness stage 1-3 (need agitation). May CONSOLIDATE with the_solution and why_it_works into a Problem/Solution/Why combo section. EXCLUDE if awareness stage 4-5.
- `cost_savings` — INCLUDE if bundle exists (`has_bundle = Y`). EXCLUDE if single SKU only.
- `how_its_made` — INCLUDE if premium positioning, quality-conscious audience, or manufacturing is a differentiator. EXCLUDE if commoditized product.
- `identity_matching` — INCLUDE if broad audience that needs self-selection help. EXCLUDE if narrow, well-defined audience.
- `the_proof` — INCLUDE if clinical studies or survey data exist (`has_clinical_studies = Y`). EXCLUDE if no formal data.

**Step 2:** For sections with UNKNOWN eligibility (proof inventory data not available):
- If the section is conditional mandatory: flag as PENDING — include in working set, but note that proof verification is needed before finalizing
- If the section is optional: flag as EXCLUDED — do not include without evidence

**Step 3:** Count total included sections. PDP BTF should have 8-14 sections. If below 8, review exclusion decisions. If above 14, review optional inclusions for dead weight.

**Step 4:** Check for the Problem/Solution/Why consolidation opportunity:
- If all three are included AND total section count is high (12+), consider consolidating into a 2-section or 3-section group that flows Problem -> Solution -> Why It Works
- Document consolidation decision with rationale

**Step 5:** Write taxonomy selection to `taxonomy-selection.md`.

## Quality Criteria
- All 16 taxonomy sections evaluated — no sections skipped
- Every decision has a specific rationale tied to product data or proof inventory — not generic reasoning
- Mandatory sections (6) all marked INCLUDE
- Conditional mandatory sections resolved against proof inventory data
- Optional sections evaluated against strategic fit, not included by default
- PENDING sections clearly documented with resolution path
- Total section count within 8-14 range
- Consolidation opportunities evaluated

## Failure Modes
- Fewer than 6 mandatory sections marked INCLUDE -> HALT. Something is structurally wrong — the 6 always-mandatory sections cannot be excluded.
- Total included sections below 8 -> WARNING. Review exclusion decisions — the PDP may be too sparse.
- Total included sections above 14 -> WARNING. Review optional inclusions — look for dead weight or consolidation opportunities.
- Proof inventory entirely missing -> All conditional mandatory sections marked PENDING. Layer 1.3 will enforce resolution before GATE_1.
- Product type unknown -> HALT. Cannot determine What's in It eligibility without product type.
