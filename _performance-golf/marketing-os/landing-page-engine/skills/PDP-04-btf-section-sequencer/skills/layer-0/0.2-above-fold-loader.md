# 0.2 — Above-Fold Loader

**Skill:** PDP-04-btf-section-sequencer
**Position:** Layer 0, Microskill 2
**Purpose:** Load `pdp-above-fold-blueprint.json` from PDP-03. Extract the buy box specification, hero layout, and above-fold conversion elements. The BTF sequence must complement — not duplicate — what the above-fold already establishes. This loader also captures the buy box dimensions for sticky ATC trigger planning in Layer 2.
**Effort Level:** low
**Dependencies:** `pdp-above-fold-blueprint.json` (PDP-03 output)
**Output:** `above-fold-loaded.md`

---

## Input Schema

```yaml
input:
  above_fold_path: "[path to pdp-above-fold-blueprint.json from PDP-03]"
```

---

## Execution Logic

**Step 1:** Locate and load `pdp-above-fold-blueprint.json` from the project output directory. If file does not exist -> HALT with error: "PDP-03 output missing. Run PDP-03 Above-Fold Architecture first."

**Step 2:** Extract above-fold structure:
- Hero image/gallery specification (image count, carousel vs static, zoom capability)
- Buy box elements (product name display, variant selector, price display, ATC button, trust badges)
- Above-fold proof elements (star rating, review count, "X sold" badge, trust seals)
- Above-fold word count (headline + subheadline + bullet points + trust copy)

**Step 3:** Identify what the above-fold already establishes (to prevent BTF duplication):
- Proof already shown above fold (star rating, review count -> BTF Reviews section adds depth, not repeat)
- Trust signals already present (badges, seals -> BTF trust elements complement, not duplicate)
- Product claims already made (headline promise -> BTF sections substantiate, not restate)

**Step 4:** Extract buy box dimensions for sticky ATC planning:
- Buy box vertical position (distance from top of page)
- Buy box height (to calculate when it scrolls out of viewport)
- Mobile buy box position (often different from desktop due to image gallery)
- ATC button style and text (sticky bar must match for consistency)

**Step 5:** Write extracted data to `above-fold-loaded.md`.

## Quality Criteria
- PDP-03 output loaded successfully — all key structural elements extracted
- Buy box specification captured in enough detail for sticky ATC planning
- Duplication avoidance notes written — prevents BTF sections from restating above-fold content
- Above-fold proof inventory documented so Layer 2 proof density planning knows what's already covered

## Failure Modes
- `pdp-above-fold-blueprint.json` not found -> HALT. PDP-03 must run first.
- File exists but buy box specification is missing or empty -> WARNING. Proceed with standard PDP buy box assumptions but flag for downstream sticky ATC planning.
- File exists but hero layout is missing -> WARNING only. BTF sequence design does not strictly depend on hero layout, but it's useful context.
- Above-fold proof elements section empty -> Assume no proof above fold. BTF sequence will need to establish proof earlier (impacts Layer 1 ordering decisions).
