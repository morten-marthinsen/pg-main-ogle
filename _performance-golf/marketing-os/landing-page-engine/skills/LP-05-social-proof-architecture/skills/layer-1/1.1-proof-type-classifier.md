# 1.1 — Proof Type Classifier

**Skill:** LP-05-social-proof-architecture
**Position:** Layer 1, Microskill 1
**Purpose:** Classify all available proof assets from the inventory against the 6-level persuasiveness hierarchy. Rank every proof asset so that the highest-ranked types get placed in the highest-visibility slots during Layer 2.
**Effort Level:** medium
**Dependencies:** `proof-inventory.md` (0.2)
**Output:** `proof-classification.md`

---

## Input Schema

```yaml
input:
  proof_inventory: "[from 0.2 — all documented proof assets]"
  vertical: "[from page-brief.json]"
```

---

## The 6-Level Proof Hierarchy

| Rank | Type | Code | Persuasiveness | Notes |
|------|------|------|---------------|-------|
| 1 | Clinical studies | `clinical_study` | Highest (especially health) | Double-blind, peer-reviewed, institution-named |
| 2 | Expert endorsements | `expert_endorsement` | Very high | Named MD/PhD/expert with credentials and specific statement |
| 3 | Specific before/after results | `before_after` | High | Named person, specific outcome, timeframe, photo if available |
| 4 | Photo testimonials | `photo_testimonial` | Medium-high | Customer photo + name + location + specific result |
| 5 | Social proof volume | `volume_signal` | Medium | Review counts, customer counts, star ratings, stat bars |
| 6 | Text testimonials | `text_testimonial` | Medium-low | Written testimonial without photo — name + result |

**Additional types (not ranked but classified):**

| Type | Code | Classification Note |
|------|------|-------------------|
| Media mention | `media_mention` | Authority signal — ranks between 2 and 3 depending on publication prestige |
| Certification | `certification` | Trust signal — not persuasive alone but removes objections |
| Platform review | `platform_review` | Volume signal (rank 5) when aggregated; rank 4 individually if includes photo |
| Performance guarantee | `performance_guarantee` | Unique to B2B/high-ticket — substitutes for testimonials entirely |
| Video testimonial | `video_testimonial` | Rank 3.5 — between before/after and photo testimonial in persuasiveness |
| Stat bar | `stat_bar` | Volume signal (rank 5) — formatted as 3-number credibility strip |

---

## Execution Logic

**Step 1:** Load proof inventory from 0.2 output.

**Step 2:** For each proof asset in inventory, classify:
- Assign proof type code
- Assign rank (1-6 or note if additional type)
- Assess quality: Does it include specific outcomes/numbers? Named source? Photo?
- Flag quality deficiencies: "Testimonial from 'Sarah M.' — no last name, no location, no specific outcome. Quality: LOW."

**Step 3:** Sort classified proof by rank (highest persuasiveness first).

**Step 4:** Count totals per type. Identify which hierarchy levels are PRESENT and which are ABSENT.

**Step 5:** Determine the highest-ranked proof type available. This will anchor the above-fold and early-page proof strategy.

**Step 6:** Apply vertical-specific adjustments:
- **Health/Supplements:** Clinical studies move to rank 1 (they are the strongest proof type for this vertical)
- **Info Products/Coaching:** Expert endorsements and celebrity testimonials are most persuasive
- **B2B Services/SaaS:** Performance guarantees and case studies outrank testimonials
- **Ecomm/DTC:** Volume signals (review counts, star ratings) are the primary trust driver

## Quality Criteria
- Every proof asset from inventory is classified with a type code and rank
- Quality assessment is honest — low-quality proof flagged, not hidden
- Hierarchy coverage table shows gaps clearly
- Vertical-specific adjustments noted

## Failure Modes
- Zero proof assets in inventory -> Output shows all ranks at count 0. Add note: "No proof assets available. LP-05 will generate proof REQUIREMENTS. All elements marked requires_sourcing."
- Proof assets have no specific outcomes -> Flag as LOW quality. Add note: "Testimonials lack specific outcomes. LP-10 will need to request outcome-specific testimonials from client."
