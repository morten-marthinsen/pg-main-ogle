# 4.1: Intelligence Compiler

> **Skill:** LP-01-conversion-intelligence
> **Layer:** 4
> **Position:** 1 of 3
> **Purpose:** Compile all validated and audited outputs into the final conversion-intelligence.json file following the exact schema defined in LP-01-AGENT.md.
> **Effort Level:** haiku
> **Dependencies:** `benchmark-synthesis.md` (from 2.1), `specimen-analysis.md` (from 2.2), `filtered-patterns.md` (from 2.3), `strategic-context.md` (from 2.4), `element-impact-ranking.md` (from 1.4), `brief-extract.md` (from 0.1), `validation-report.md` (from 3.1), `completeness-audit.md` (from 3.2)
> **Output:** `conversion-intelligence.json`

---

## INPUT

```yaml
input:
  brief_extract: "brief-extract.md"
  benchmark_synthesis: "benchmark-synthesis.md"
  specimen_analysis: "specimen-analysis.md"
  filtered_patterns: "filtered-patterns.md"
  strategic_context: "strategic-context.md"
  element_impact_ranking: "element-impact-ranking.md"
  validation_report: "validation-report.md"
  completeness_audit: "completeness-audit.md"
```

---

## EXECUTION

1. **Verify pre-conditions:**
   - validation-report.md exists AND result = PASS
   - completeness-audit.md exists AND result = PASS
   - If either is missing or FAIL → HALT. Do not compile unvalidated data.

2. **Compile classification_context** from brief-extract.md:
   - page_type, vertical_primary, vertical_secondary, traffic_temperature, awareness_stage

3. **Compile vertical_benchmarks** from benchmark-synthesis.md:
   - conversion_rate_range, conversion_rate_top_performer, traffic_type
   - aov_lift_from_bundles, guarantee_claim_rate, review_rating_threshold
   - source_citations (list of exact citations from the synthesis)

4. **Compile page_type_benchmarks** from benchmark-synthesis.md:
   - target_conversion_rate, bounce_rate_target, page_speed_target, mobile_speed_target
   - section_count_range, cta_count_range
   - source_citations

5. **Compile element_level_impact** from benchmark-synthesis.md:
   - The 7 universal element-level impact stats from conversion-data-reference.md
   - Source attribution

6. **Compile matched_specimens** from specimen-analysis.md:
   - For each matched specimen: specimen_id, name, match_score, match_reason, page_type, vertical, proof_density, key_pattern, specimen_file, relevance_to_build

7. **Compile element_priority_ranking** from element-impact-ranking.md:
   - For each ranked element: rank, element name, category, frequency, impact_data, required_for_this_build, rationale

8. **Compile cross_page_patterns** from filtered-patterns.md:
   - section_sequence_pattern, proof_architecture_pattern, cta_architecture_pattern
   - headline_pattern, offer_structure_pattern
   - vertical_specific_sections array
   - Source attribution

9. **Compile strategic_context** from strategic-context.md:
   - lp_03_above_fold_guidance, lp_04_section_sequence_guidance
   - lp_05_social_proof_guidance, lp_06_offer_cta_guidance
   - lp_17_conversion_audit_targets
   - key_risks array, key_opportunities array, data_gaps array

10. **Compile data_confidence** from completeness-audit.md:
    - overall_score, benchmark_confidence, specimen_match_confidence
    - element_ranking_confidence, pattern_confidence, rationale

11. **Compile downstream_handoffs** (static text from AGENT.md schema).

12. **Write conversion-intelligence.json** with valid JSON formatting:
    - Use 2-space indentation
    - All string values in double quotes
    - Arrays use proper JSON syntax
    - Include schema_version "1.0", created timestamp, project_name, source_brief path

13. **Validate JSON syntax** — the file must be parseable JSON.

---

## OUTPUT SCHEMA

`conversion-intelligence.json` — follows the exact schema defined in LP-01-AGENT.md OUTPUT SCHEMA section. Do not deviate from the schema. Do not add fields. Do not remove fields. If a field has no data, use `"not_available"` with a string explaining why.

---

## QUALITY CRITERIA

- Valid JSON (parseable by any JSON parser)
- All schema sections present
- No field left as empty string `""` — use `"not_available"` with rationale
- All data matches the validated source files (no re-interpretation during compilation)
- schema_version set to "1.0"
- created field has ISO timestamp
- project_name matches brief-extract.md
- source_brief path is valid

---

## FAILURE MODES

- **Validation or completeness audit not passed:** HALT — do not compile. Return to Layer 3.
- **JSON syntax error:** Fix the syntax. Common issues: trailing commas in arrays, unescaped quotes in strings, missing closing brackets.
- **Data mismatch during compilation:** If a value in the source file doesn't match what was validated, HALT — something changed between Layer 3 and Layer 4. Re-run Layer 3.
- **File too large:** conversion-intelligence.json should be 5-25KB for a typical build. If significantly larger, check for duplicated data. If significantly smaller, check for missing sections.
