# 4.1 — Audit Report Compiler

> **Skill:** PDP-16-mobile-conversion-audit
> **Layer:** 4 (Package Assembly)
> **Position:** 1 of 3
> **Model:** haiku
> **Purpose:** Assemble the final `pdp-audit-report.json` from all Layer 2 and Layer 3 outputs. This is the structured, machine-readable audit output that LP-18 (A/B Test Plan) and downstream tools can consume. Every finding, score, and priority item must be present.

**Step 1:** Read all input files.

**Step 2:** Assemble the JSON structure:

```json

  "metadata": {

    "version": "1.0",
    "project": "[project name from assembled-page-load.md]",
    "page_type": "[type_b | hybrid]",
    "audit_date": "[YYYY-MM-DD]",
    "total_criteria": 42,
    "applicable_criteria": "[count after N/A]",
    "total_pass": "[count]",
    "total_fail": "[count]",
    "total_na": "[count]"
  },
  "composite_score": {
    "score": "[X.XX]",
    "classification": "[exceptional | good | needs_revision | structural_issues]",
    "recommendation": "[proceed | proceed_with_notes | flag_for_revision | halt_for_rework]"
  },
  "sections": [
    {
      "id": 1,
      "name": "Image Gallery / Carousel",
      "weight": "[X]%",
      "score": "[X.X]",
      "weighted_score": "[X.XX]",
      "criteria": [
        {
          "id": "1.1",
          "name": "Thumbnail Navigation",
          "verdict": "[PASS | FAIL | N/A]",
          "evidence": "[evidence text]",
          "baymard_reference": "[reference text]",
          "fix": "[fix recommendation or null]"
        }
      ]
    }
  ],
  "priority_ranking": [
    {
      "rank": 1,
      "criterion_id": "[ID]",
```

**Step 3:** Validate the JSON structure:
- All 8 sections present
- All criteria from pdp-audit-criteria.md accounted for
- All FAIL items from per-section-scores.md appear in priority_ranking
- Composite score matches composite-score.md
- Weights match weight-assignment.md

**Step 4:** Write `pdp-audit-report.json`.

## Quality Criteria

- Valid JSON (parseable)
- All 8 sections with all criteria included
- Every FAIL item in priority_ranking
- Metadata complete (project, date, counts)
- Composite score matches Layer 3 calculation
- Upstream impact map included
- File exists after execution

## Failure Modes

- Missing Layer 3 files -> HALT. Cannot assemble without scores and ranking.
- Invalid JSON -> Validate structure before writing. Common errors: trailing commas, unquoted keys, missing brackets.
- Criterion count mismatch -> Cross-reference pdp-audit-criteria.md. Every criterion must appear in the sections array.
- FAIL items missing from priority_ranking -> Cross-reference per-section-scores.md FAIL summary. Every FAIL must appear.
