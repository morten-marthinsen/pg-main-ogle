# 2.1 — Image Gallery Audit

> **Skill:** PDP-16-mobile-conversion-audit
> **Layer:** 2 (Audit Execution)
> **Position:** 1 of 8
> **Model:** sonnet
> **Purpose:** Audit the PDP image gallery / carousel against Baymard best practices. The carousel is the primary above-fold persuasion tool on a PDP. 50-80% of users miss additional images when only dots are used. 40% of mobile users fail at gesture controls without proper pinch-to-zoom support.
> **Dependencies:** `assembled-page-load.md`, `pdp-reference-load.md`, `pdp-audit-criteria.md`
> **Output:** `image-gallery-audit.md`

## Execution Logic

**POSITIONAL REINFORCEMENT:** You are auditing the image gallery / carousel of the assembled PDP page. You are NOT auditing the buy box, BTF sections, or any other component. Stay focused on Section 1 criteria only.

**Step 1:** From `assembled-page-load.md`, extract all carousel/gallery specifications:
- Number of slides
- Slide content descriptions
- Thumbnail vs dot navigation
- Zoom behavior specification
- Image types (product, lifestyle, in-scale, video)
- Overlay behavior
- Back-button handling

**Step 2:** Evaluate each criterion against the assembled page content:

### Criterion 1.1: Thumbnail Navigation
- **Baymard stat:** 50-80% of users miss content when only dots are used
- **PASS:** Assembled page specifies thumbnails visible below main image
- **FAIL:** Assembled page specifies dots, or does not specify navigation type
- **Evidence required:** Quote the assembled page's carousel navigation specification

### Criterion 1.2: Pinch-to-Zoom Support
- **Baymard stat:** 40% of mobile sites fail to support pinch-to-zoom; dynamically swap for high-res on zoom
- **PASS:** Assembled page specifies both double-tap AND pinch zoom, with high-res swap
- **FAIL:** Only one zoom method specified, no zoom specified, or no high-res swap on zoom
- **Evidence required:** Quote the assembled page's zoom specification

### Criterion 1.3: In-Scale Image
- **Baymard finding:** Users lose sense of scale on 6-inch screens
- **PASS:** At least one carousel slide shows product relative to everyday object or human hand/body
- **FAIL:** All slides are isolated product shots without scale reference
- **Evidence required:** Identify which slide (by number) provides scale reference, or note absence

### Criterion 1.4: Gallery Overlay
- **Baymard finding:** Carousel is for browsing; overlay is for inspecting
- **PASS:** Assembled page specifies tap-to-open full-screen dark overlay with left/right swiping and 2x-4x deep zoom
- **FAIL:** No overlay specified, or overlay lacks swipe/zoom
- **Evidence required:** Quote overlay specification or note absence

### Criterion 1.5: Back-Button Safety
- **Baymard finding:** Users hit browser back to close overlay and accidentally leave PDP
- **PASS:** Assembled page specifies back-button hijack to close overlay only
- **FAIL:** No back-button behavior specified in overlay context
- **Evidence required:** Quote back-button specification or note absence

### Criterion 1.6: Video Integration
- **Baymard finding:** Users expect to see product in motion
- **PASS:** At least one video in carousel with prominent play icon overlay on thumbnail
- **FAIL:** No video in carousel, or video exists but no play icon specified
- **Evidence required:** Identify which slide has video, or note absence

**Step 3:** Write finding for each criterion with:
- Verdict: PASS / FAIL / N/A
- Evidence: Specific quote or reference from assembled page
- Baymard reference: The statistic or finding this criterion is based on
- Fix recommendation (if FAIL): Specific action to resolve

**Step 4:** Write `image-gallery-audit.md`.

## Quality Criteria

- All 6 criteria evaluated (or marked N/A with justification)
- Each verdict has evidence from the assembled page (quoted or referenced)
- Each verdict has the correct Baymard reference
- UNSPECIFIED = FAIL (not PASS)
- Fix recommendations are specific and actionable
- Section score calculated correctly (passed / applicable, scaled to 10)
- File exists after execution

## Failure Modes

- Assembled page has no carousel specification -> Score all 6 criteria as FAIL with evidence "No carousel specification found in assembled page." This is a catastrophic PDP failure.
- assembled-page-load.md not found -> HALT, run 0.1 first.
- Carousel has fewer than 10 slides -> Note as additional finding (not a scored criterion in Section 1, but relevant context for the priority ranking).
