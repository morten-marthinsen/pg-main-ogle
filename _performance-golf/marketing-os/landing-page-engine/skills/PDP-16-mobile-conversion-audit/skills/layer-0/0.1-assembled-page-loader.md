# 0.1 — Assembled Page Loader

> **Skill:** PDP-16-mobile-conversion-audit
> **Layer:** 0 (Foundation Loading)
> **Position:** 1 of 3
> **Model:** haiku
> **Purpose:** Load LP-15 assembled page output and verify the PDP route was taken. If page type is `type_a`, HALT immediately — use LP-16 + LP-17 instead.
> **Dependencies:** LP-15 output (assembled page files in project output directory)
> **Output:** `assembled-page-load.md`

---

## Input Schema

```yaml
input:
  project_output_dir: "[path to ./outputs/[project-name]/landing-page/]"
  lp15_output: "[LP-15 assembled page files — typically assembled-page.md, page-assembly.json, or similar]"
```

---

## Execution Logic

**Step 1:** Locate LP-15 output in the project output directory. Look for:
- `lp-15-page-assembly/` directory
- `assembled-page.md` or `page-assembly.json` (primary assembled page)
- `PDP-ASSEMBLY-SUMMARY.md` (if PDP route was taken)
- Any `PDP_ROUTE` or `page_type` indicator files

**Step 2:** Verify page type. Extract the page type classification from LP-15 output or LP-00 brief:
- If `type_b` or `hybrid` -> PROCEED (PDP-16 is correct)
- If `type_a` -> HALT with message: "Page type is Type A. Use LP-16 + LP-17 instead of PDP-16."
- If page type is unclear -> Check for PDP-specific content (carousel architecture, buy box components, review system). If PDP content is present, proceed as `type_b`.

**Step 3:** Extract the assembled page structure. Identify all sections present in the assembled page:
- Above-fold sections (carousel, facts panel, buy box)
- BTF sections (list each by name/type)
- Review/rating section
- FAQ section
- CTA/sticky elements
- Navigation elements

**Step 4:** Inventory upstream PDP skill outputs referenced by LP-15:
- PDP-03 output (carousel architecture, buy box blueprint)
- PDP-04 output (BTF section sequence)
- PDP-05 output (social proof / review system)
- PDP-06 output (offer / buy box copy)
- PDP-07 output (BTF section copy)

**Step 5:** Write `assembled-page-load.md`.

## Quality Criteria

- Page type verified as `type_b` or `hybrid` (not `type_a`)
- All sections in the assembled page inventoried
- Upstream PDP skill outputs identified
- Missing components noted (these will be scored as FAIL in Layer 2)
- File exists after execution

## Failure Modes

- LP-15 output not found -> HALT. LP-15 must run before PDP-16.
- Page type = `type_a` -> HALT with redirect to LP-16 + LP-17.
- Assembled page has no PDP components (no carousel, no buy box) -> HALT. Either LP-15 used the wrong route or PDP skills did not run.
- Partial LP-15 output (assembly started but incomplete) -> HALT. LP-15 must complete before PDP-16 audit.
