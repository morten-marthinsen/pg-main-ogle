# 0.3 — Benchmark Loader

> **Skill:** PDP-16-mobile-conversion-audit
> **Layer:** 0 (Foundation Loading)
> **Position:** 3 of 3
> **Model:** haiku
> **Purpose:** Load conversion-data-reference.md for industry benchmarks. These benchmarks provide context for scoring — what conversion rates, mobile metrics, and UX standards the assembled PDP should be measured against.
> **Dependencies:** conversion-data-reference.md
> **Output:** `benchmark-load.md`

---

## Input Schema

```yaml
input:
  conversion_reference_path: "[path to conversion-data-reference.md]"
  page_vertical: "[supplement | beauty | food_bev | consumer_goods | other]"
```

---

## Execution Logic

**Step 1:** Read conversion-data-reference.md.

**Step 2:** Extract PDP-relevant benchmarks:
- Ecommerce conversion rates by vertical
- Mobile vs desktop conversion rate differences
- Page speed impact on conversion (% loss per second)
- Above-fold conversion influence benchmarks
- Social proof impact benchmarks (review presence, UGC impact)

**Step 3:** Extract mobile-specific benchmarks:
- Mobile traffic share by vertical (typically 65-80%)
- Mobile bounce rate benchmarks
- Mobile time-on-page benchmarks
- Touch target standards (44px minimum per Apple/Google HIG)
- Font size minimums (14px body per mobile readability research)

**Step 4:** Extract UX pattern benchmarks relevant to PDP audit:
- Sticky element conversion lift data (if available)
- Single-page checkout vs multi-page impact
- Express payment adoption rates
- Review section engagement rates

**Step 5:** Filter benchmarks by page vertical (from LP-00 brief). If vertical-specific data is available, include it alongside general benchmarks.

**Step 6:** Write `benchmark-load.md`.

## Quality Criteria

- Conversion benchmarks extracted with source attribution
- Mobile-specific benchmarks identified
- Vertical-specific data included when available
- Standards referenced with authoritative source
- File exists after execution

## Failure Modes

- conversion-data-reference.md not found -> Proceed with Baymard-only benchmarks from pdp-reference-load.md. Write benchmark-load.md noting "conversion-data-reference.md not available — using Baymard defaults only."
- No vertical-specific data available -> Use general benchmarks with note "no vertical-specific data."
- Benchmarks appear outdated -> Use available data with note on recency. Do not fabricate updated numbers.
