# 3.2 — Emotional Appeal Audit

> Version: 1.0
> Layer: 3
> Model: sonnet
> Input: CTA_SELECTION.yaml, cta-validation.md
> Output: emotional-appeal-audit.md

**Skill:** LP-14-cta-optimizer
**Position:** Layer 3, Microskill 2
**Purpose:** Deep audit of emotional appeal coverage and effectiveness across the entire CTA set. While 3.1 checks basic coverage (>= 5/6 types), this audit evaluates whether each appeal is EFFECTIVELY executed — not just present. A CTA tagged as "Social Proof" that says "Join now" without any proof number is tagged but not effective.
**Effort Level:** medium
**Dependencies:** `CTA_SELECTION.yaml`, `cta-validation.md` (3.1)
**Output:** `emotional-appeal-audit.md`

---

## Input Schema

```yaml
input:
  cta_selection: "CTA_SELECTION.yaml"
  cta_validation: "cta-validation.md"
  required_fields:
    - all_selected_ctas (primary + secondary + micro)
    - emotional_appeal_tags
    - appeal_coverage_from_validation
```

---

## Execution Logic

1. **List every CTA in the final selected set** (primary, all secondary positions, micro-CTAs if selected).

2. **For each CTA, verify the emotional appeal tag is ACCURATE:**
   - Read the button text and context text
   - Determine the ACTUAL emotional appeal the CTA activates (regardless of what it was tagged as)
   - If the actual appeal differs from the tag: flag as MISTAGGED and assign correct appeal

3. **For each emotional appeal type present, evaluate EFFECTIVENESS:**

   **Confidence/Desire (#1):** Does the CTA convey confident forward momentum? Does it assume the sale? Does it name a specific desired outcome? Rating: 1-10.

   **Consequence Awareness (#2):** Does the CTA name a specific consequence of inaction? Does it use the audience's language for their pain? Or is it vague ("Don't wait")? Rating: 1-10.

   **Urgency (#3):** Does the CTA contain a specific, justified deadline or quantity? Or is it vague urgency ("Limited time")? Only valid if LP-13 data exists. Rating: 1-10.

   **Social Proof (#4):** Does the CTA contain a specific number, community name, or peer reference? Or is it generic ("Join others")? Rating: 1-10.

   **Value (#5):** Does the CTA reference specific items in the value stack, the price, or a savings amount? Or is it vague ("Get everything")? Rating: 1-10.

   **Specificity (#6):** Does the CTA contain a measurable outcome, timeframe, or concrete result? Or is it abstract ("See results")? Rating: 1-10.

4. **Calculate appeal effectiveness scores:**
   - Average effectiveness across all represented appeals
   - Identify weakest appeal execution
   - If any appeal scores < 5.0 effectiveness: recommend revision

5. **Identify appeal gaps:**
   - Which of the 6 types are NOT represented in any CTA?
   - For each gap: is it acceptable (urgency blocked) or a coverage failure?

6. **Write `emotional-appeal-audit.md`** with full analysis.

## Quality Criteria
- Every CTA in the selected set audited (not just primary)
- Appeal tags verified against actual text (mistagging caught)
- Effectiveness scored with evidence (quoted text, not just a number)
- Weakest appeal identified with specific revision recommendation
- Coverage threshold met (>= 5/6, or >= 4/5 if urgency blocked)

## Failure Modes
- Tagged appeal accepted without verification -> "Social Proof" CTA that says "Get Started" has no social proof. Verify tags against actual text.
- Effectiveness scored without evidence -> Each score must cite the specific text that justifies it. A score without a quote is unverifiable.
- All appeals rated 10/10 -> Suspiciously generous. At least one appeal should have room for improvement. Re-evaluate honestly.
- Urgency counted as present when BLOCKED -> If LP-13 is unavailable, urgency cannot count. Adjust threshold.
