# 1.1 — CTA Type Planner

> Version: 1.0
> Layer: 1
> Model: sonnet
> Input: packages-loaded.md, cta-architecture-loaded.md, specimens-loaded.md
> Output: cta-type-plan.md

**Skill:** LP-14-cta-optimizer
**Position:** Layer 1, Microskill 1
**Purpose:** Map the 6 emotional appeal types to specific CTA page positions. Each position gets a primary appeal and optional secondary appeal. This plan governs what types of CTAs are generated for each position, ensuring coverage and strategic placement.
**Effort Level:** medium
**Dependencies:** `packages-loaded.md` (0.1), `cta-architecture-loaded.md` (0.2), `specimens-loaded.md` (0.3)
**Output:** `cta-type-plan.md`

## Execution Logic

1. **Load the CTA Position Map** from `cta-architecture-loaded.md`. List all CTA positions.

2. **Load LP-06's emotional appeal recommendations.** These are the starting point, not gospel. LP-14 may adjust based on page flow and awareness stage.

3. **For each CTA position, assign appeals using this decision matrix:**

   **Above-Fold CTA:**
   - Cold traffic (Stage 1-2): Primary = Specificity (#6). Secondary = Confidence (#1).
   - Warm traffic (Stage 3): Primary = Confidence (#1). Secondary = Value (#5).
   - Hot traffic (Stage 4-5): Primary = Confidence (#1). Secondary = Urgency (#3, if LP-13 permits).

   **Mid-Page CTA (after problem/mechanism section):**
   - Primary = Consequence Awareness (#2) — reader just learned about the problem.
   - Secondary = Specificity (#6) — "Start seeing [result] in [timeframe]."

   **Post-Proof CTA (after clinical data or authority section):**
   - Primary = Social Proof (#4) — "Join 83,502+ customers."
   - Secondary = Confidence (#1) — proof has built confidence to act.

   **Post-Testimonial CTA:**
   - Primary = Social Proof (#4) — testimonials prime belonging.
   - Secondary = Specificity (#6) — "Get [specific result] like [customer name]."

   **Post-Offer/Pricing CTA:**
   - Primary = Value (#5) — "Get [X] + [Y] + [Z] — Just [Price]."
   - Secondary = Confidence (#1).

   **Final/Terminal CTA:**
   - Primary = Urgency (#3, if LP-13 permits) or Confidence (#1, if no urgency).
   - Secondary = Consequence Awareness (#2) — "Don't let [problem] continue."

4. **Verify coverage:** Across all positions, ensure >= 5 of 6 appeal types are assigned as primary or secondary. If any type is missing, adjust secondary assignments to include it.

5. **If urgency is FORBIDDEN** (no LP-13 data or not justified), replace all urgency assignments with the next-best appeal for that position. Do not leave urgency assigned to any position.

6. **Write `cta-type-plan.md`** with the complete position-to-appeal mapping.

## Quality Criteria
- Every CTA position has a primary appeal assigned
- >= 5 of 6 appeal types represented across all positions
- Urgency assignments removed if LP-13 is FORBIDDEN — no exceptions
- Rationale provided for each assignment (not just a matrix lookup — consider page flow)
- LP-06 recommendations honored unless specific adjustment reason provided

## Failure Modes
- Urgency assigned to a position when LP-13 is FORBIDDEN -> Remove immediately. Fabricated urgency.
- All positions assigned Confidence (#1) -> Monotone CTAs. Diversify across appeal types.
- Coverage below 5/6 -> Adjust secondary assignments to fill gaps before proceeding.
- Assignments made without considering awareness stage -> Cold traffic with "Buy Now" CTAs or hot traffic with "Learn More" CTAs. Calibrate.
