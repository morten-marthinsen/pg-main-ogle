# 1.3 — Type Declaration

**Skill:** LP-00-brief-classifier
**Position:** Layer 1, Microskill 3
**Purpose:** Commit to a final page type classification based on all signal and hybrid evidence. This declaration is BINDING — all downstream skills use it.
**Effort Level:** low
**Dependencies:** `classification-signals.md` (1.1), `hybrid-test.md` (1.2)
**Output:** `type-declaration.md`

---

## Input Schema

```yaml
input:
  type_a_count: "[from 1.1]"
  type_b_count: "[from 1.1]"
  hybrid_result: "[hybrid_detected | not_hybrid — from 1.2]"
  hybrid_zone_split: "[from 1.2 if applicable]"
```

---

## Decision Logic

```
IF hybrid_result = hybrid_detected:
  → classification = hybrid
  → Use hybrid_zone_split from 1.2

ELSE IF type_a_count >= 4:
  → classification = type_a

ELSE IF type_b_count >= 4:
  → classification = type_b

ELSE IF type_a_count == type_b_count AND hybrid_result = not_hybrid:
  → classification = hybrid (default when genuinely ambiguous)
  → Note: "Ambiguous classification — defaulting to Hybrid. Human confirmation recommended."

ELSE:
  → classification = [whichever has more signals]
  → Note: "Low confidence classification — only [X]/7 signals confirmed. Human review recommended."
```

**If fewer than 4 signals confirmed for the declared type:**
- Still declare, but flag as low-confidence
- Add validation_flag: "Low classification confidence — only [X] signals. Recommend human review before Phase 3."

---

## Output Schema

`type-declaration.md`:

```markdown
# Page Type Declaration — [Product Name]

## Final Classification: [TYPE A | TYPE B | HYBRID]

## Evidence Summary
- Type A signals confirmed: [count] / 7
- Type B signals confirmed: [count] / 7
- Hybrid detection result: [triggered | not triggered]
- Confidence level: [high (5+) | medium (4) | low (3)]

## Rationale
[2–4 sentences explaining the specific combination of signals that led to this classification. Must reference at least 2 specific signals with their evidence.]

## Hybrid Zone Split (Hybrid only)
[Zone 1: Type B above-fold, Sections 1–X
Zone 2: Type A content, Sections X+1 onward
or: N/A]

## Downstream Impact
- LP-03 will use: [Type A or B or Hybrid above-fold pattern]
- LP-04 will load: [Type A or B or Hybrid base template]
- All Phase 3 writing skills will receive: [page type classification]

## Validation Flags
[Any low-confidence notes, or: "None — classification confident"]
```

---

## Quality Criteria
- Classification is one of three exact values: `type_a`, `type_b`, `hybrid`
- Rationale references at least 2 specific signals
- Confidence level stated
- Downstream impact noted (this primes the next agents)

## Failure Modes
- 0 signals for any type (completely unclear brief) → HALT with message: "Brief does not contain sufficient product/market information to classify. Please provide: price point, target audience, traffic source, and key claims."
