# 2.6 — PDP Trust Badge Writer

**Skill:** LP-08-trust-elements
**Position:** Layer 2, Microskill 6
**Purpose:** Write PDP-specific trust badge copy that integrates with the buy box architecture from PDP-03. These badges sit between the ATC button and express payment options, following Baymard-validated patterns (guarantee + shipping + quality + security). PDP badges are tighter than standard badges (3-5 words vs 3-7) because they share vertical space with the ATC button and must not compete with the primary conversion action.
**Effort Level:** high
**Dependencies:** `trust-classification.md` (1.1), `trust-placement-map.md` (1.3), `trust-inventory.md` (0.2), `badge-copy.md` (2.4 — must NOT duplicate), PDP-03 `pdp-above-fold-blueprint.json` (micro-trust signal specs + buy box architecture)
**Output:** `pdp-trust-badge-copy.md`

**Execution Condition:** Type B or Hybrid pages ONLY. Skip entirely for Type A — write `pdp-trust-badge-copy.md` with `status: skipped_type_a` and terminate.

## Execution Logic

### If Type A page:
Write `pdp-trust-badge-copy.md` with:
```
Status: skipped_type_a
Reason: PDP trust badges are Type B/Hybrid only. This microskill does not apply to Type A pages.
```
Terminate this microskill.

### If Type B or Hybrid:

**Step 1: Load PDP-03 buy box architecture.**
Read `pdp-above-fold-blueprint.json` and extract:
- Micro-trust signal slots (positions, sizing constraints)
- ATC button placement and surrounding whitespace
- Express payment row location (Apple Pay, Shop Pay, etc.)
- Any micro-trust signals PDP-03 already specified (these are OFF-LIMITS for LP-08 — no duplication)

**Step 2: Identify PDP-03 micro-trust signals already specified.**
List every trust signal PDP-03 already placed in the buy box. These are EXCLUDED from LP-08 badge generation. Common PDP-03 micro-trust signals include free shipping callouts, review stars, and stock indicators. If PDP-03 already specified "Free Shipping" as a micro-trust signal in the buy box, LP-08 does NOT write a separate "Free Shipping" badge.

**Step 3: Select badge set from the 4 Baymard badge categories.**
Every PDP should have representation from these 4 categories (if data exists in trust inventory):

| Category | Purpose | Example Badges |
|----------|---------|---------------|
| Guarantee | Risk removal — the #1 purchase barrier | "XX-Day Money Back", "Full Refund Guarantee", "XX-Day Free Returns" |
| Shipping | Delivery confidence | "Free Shipping Over $X", "Ships in 24 Hours", "Free 2-Day Shipping" |
| Quality | Manufacturing/formula credibility | "GMP Certified", "3rd Party Tested", "Non-GMO Verified" |
| Security | Payment/checkout safety | "Secure Checkout", "SSL Encrypted", "Secure Payment" |

Select 3-5 badges total. Do NOT exceed 5 — PDP badge strips with more than 5 badges create visual clutter that reduces trust (paradox of choice).

**Step 4: Write badge copy at 3-5 word constraint.**
PDP badges are SHORTER than standard badges (3-5 words, not 3-7). The buy box is the highest-value real estate on any ecommerce page. Every word competes with the ATC button for attention. Shorter = better.

Rules:
- 3-5 words per badge MAXIMUM (hard constraint — 6+ words FAIL)
- No articles ("a", "the") unless grammatically required
- No filler words ("our", "your", "with")
- Front-load the specific detail: "365-Day Money Back" not "Money Back for 365 Days"
- Each badge = 1 trust signal (same as 2.4, but stricter word budget)

**Step 5: Write icon direction for each badge.**
PDP badges are typically displayed with small inline icons (16-24px). Provide brief icon direction for the designer:
- Guarantee: shield icon, checkmark-in-circle, money-back arrow
- Shipping: truck icon, box icon, delivery arrow
- Quality: lab flask, certificate seal, leaf (for natural)
- Security: lock icon, credit card with shield, padlock

**Step 6: Cross-reference against 2.4 output to prevent duplication.**
Compare every PDP badge against the badge set from 2.4. If 2.4 already wrote a badge for the same trust signal, the PDP version REPLACES it for Type B pages (not duplicates it). Document which 2.4 badges are superseded.

**Step 7: Verify placement in buy box architecture.**
Confirm that the badge strip placement is:
- BELOW the ATC button
- ABOVE express payment options (or below them if PDP-03 specifies that layout)
- Inline / horizontal layout (not stacked vertically)
- Small sizing consistent with PDP-03 micro-trust signal specs

**Step 8: Write `pdp-trust-badge-copy.md`.**

## Quality Criteria
- Badge text is 3-5 words per badge (stricter than standard 3-7 — hard constraint)
- Each badge communicates exactly 1 trust signal
- 3-5 badges total (no fewer, no more)
- All 4 Baymard categories represented where data exists (guarantee, shipping, quality, security)
- Zero duplication with PDP-03 micro-trust signals already specified in buy box
- Zero unresolved duplication with 2.4 badge set (PDP badges supersede, not duplicate)
- Every badge traceable to trust inventory (no fabrication)
- Badge placement integrates with PDP-03 buy box architecture (below ATC, inline)
- Icon direction provided for every badge

## Failure Modes
- Running this microskill for Type A pages -> Type A has no buy box or ATC button. Skip entirely.
- Badge text over 5 words -> PDP badges must be 3-5 words. The buy box has no room for 6-7 word badges. Trim ruthlessly.
- Duplicating PDP-03 micro-trust signals -> PDP-03 may already specify "Free Shipping" or "In Stock" in the buy box. Check before writing.
- Duplicating 2.4 badge output without superseding -> If both 2.4 and 2.6 produce a "GMP Certified" badge, the PDP version supersedes. Document the replacement.
- Missing guarantee badge -> Guarantee is the #1 risk-removal signal. If guarantee data exists in trust inventory and no guarantee badge is written, this is a failure.
- Exceeding 5 badges -> Visual clutter in the buy box reduces trust. Maximum 5 badges, period.
- Placing badges above ATC button -> Badges below ATC reinforce the purchase decision. Badges above ATC compete with the purchase decision. Placement must be below ATC.
