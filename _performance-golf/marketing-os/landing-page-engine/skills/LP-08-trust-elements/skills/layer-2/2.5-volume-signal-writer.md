# 2.5 — Volume Signal Writer

**Skill:** LP-08-trust-elements
**Position:** Layer 2, Microskill 5
**Purpose:** Write customer count, review count, and unit-sold copy using SPECIFIC numbers. Volume signals communicate market validation through scale — "1,270,268 customers can't all be wrong." This microskill also writes authority signal framing and security signal copy, consolidating all remaining trust element types that are not trust bar, media mentions, certifications, or badges.
**Effort Level:** high (when applicable)
**Dependencies:** `trust-classification.md` (1.1), `trust-placement-map.md` (1.3), `trust-inventory.md` (0.2)
**Output:** `volume-signal-copy.md`

---

```yaml
input:
  volume_status: "[from trust-classification.md — included | not_applicable]"
```

## Execution Logic

### SECTION A: Volume Signals

**If volume `not_applicable`:**
Document in output: `Volume signals: not_applicable. No specific volume data in trust inventory.`

**If volume `included`:**

1. **For each available volume metric, write the copy:**

   **Customer Count:**
   - Format: "Join [SPECIFIC_NUMBER]+ [customers/members/subscribers] who [benefit verb]"
   - Examples:
     - "Join 1,270,268+ customers who trust [Brand] for daily health"
     - "Trusted by 1,270,268+ customers since 2019"
   - NEVER: "Over a million customers" or "Thousands trust us"
   - Write 2 variants per metric

   **Review Count:**
   - Format: "Rated [X.X]/5 from [SPECIFIC_NUMBER] verified reviews"
   - Examples:
     - "Rated 4.8/5 from 2,847 verified reviews"
     - "2,847 verified reviews — 4.8/5 average rating"
   - NEVER: "Thousands of 5-star reviews" or "Tons of great reviews"
   - Write 2 variants per metric

   **Units Sold:**
   - Format: "[SPECIFIC_NUMBER]+ [units/bottles/packages] sold [timeframe]"
   - Examples:
     - "2,847,000+ bottles sold since 2019"
     - "Over 2.8 million bottles delivered to customers in 47 countries"
   - Exception: For units sold, "million" is acceptable IF the specific decimal is included ("2.8 million" not "millions")
   - Write 2 variants per metric

   **Rating Strip Context (Type B):**
   - Pre-strip copy: Text that appears BEFORE the star rating widget
   - Post-strip copy: Text that appears AFTER the star rating widget
   - Examples:
     - Pre: "What our customers say:"
     - Post: "Based on [SPECIFIC_NUMBER] verified reviews"

2. **Verify every number matches the trust inventory exactly.** Do not round, estimate, or inflate.

### SECTION B: Authority Signals

**If authority `not_applicable`:**
Document in output: `Authority signals: not_applicable. No named experts in trust inventory.`

**If authority `included`:**

1. **For each named expert, write the authority framing copy:**

   **Full framing format:**
   ```
   "[Specific endorsement quote about the product]"
   — Dr. [Full Name], [Specific Credential], [Institution/Practice]
   ```

   **Introduction sentence (for context before the quote):**
   - Format: "[Expert name], [brief credential], [why they are relevant to this product]:"
   - Example: "Dr. James DiNicolantonio, a cardiovascular research scientist and author of The Mineral Fix, has studied magnesium supplementation for over a decade:"

2. **Apply authority signal rules:**
   - ALWAYS include: Name, Credential, Institution
   - Credential must be RELEVANT to the product claim
   - Quote must be SPECIFIC to this product (not a general category endorsement)
   - NEVER: "Doctors recommend" without naming the doctor
   - NEVER: "Backed by experts" without naming the experts

### SECTION C: Security Signals

**If security `not_applicable`:**
Document in output: `Security signals: not_applicable. No specific security data in trust inventory.`

**If security `included`:**

1. **Write security copy for each placement:**
   - Near checkout: "Your payment is protected by [SPECIFIC TECHNOLOGY]. We never store your credit card information."
   - Near ATC (Type B): "Secure checkout powered by [PROCESSOR NAME]"
   - Footer: "[BRAND] uses [ENCRYPTION LEVEL] encryption to protect your data."

2. **Rules:**
   - Name the SPECIFIC security technology or processor (Stripe, SSL, 256-bit)
   - 1 sentence maximum per placement
   - NEVER: "Your information is safe with us" without specifying HOW

3. **Write `volume-signal-copy.md`.**

## Quality Criteria
- Volume numbers are SPECIFIC and match inventory exactly (no rounding)
- Authority signals include Name + Credential + Institution (no anonymous authority)
- Security signals name specific technology (no vague "safe" claims)
- 2 variants written per volume metric
- All copy traceable to trust inventory
- Rating strip context written if Type B/Hybrid with reviews

## Failure Modes
- Rounding volume numbers ("Over a million" instead of "1,270,268+") → Specific numbers are more credible. Use exact numbers from inventory.
- Anonymous authority ("Doctors recommend") → Name the doctor. If no named doctor exists, mark as `not_applicable`.
- Vague security ("Your information is safe") → Name the encryption level or payment processor.
- Inflating numbers beyond inventory data → Fabrication. Use only numbers from trust inventory.
- Missing rating strip context for Type B → If reviews exist and page is Type B, rating strip context is required.
- Forgetting authority signal when in AUTHORITY_SUBSTITUTION proof mode → When proof is scarce, authority signals become MORE important, not less. Check proof strategy mode.
