# 0.3 — Specimen Trust Loader

**Skill:** LP-08-trust-elements
**Position:** Layer 0, Microskill 3
**Purpose:** Load trust element patterns from matching specimens by vertical and page type. These patterns serve as structural attractors during Layer 2 generation — showing what high-performing trust copy looks like in comparable real-world pages.
**Effort Level:** low
**Dependencies:** `packages-loaded.md` (0.1), specimen-index.md, element-taxonomy.md
**Output:** `specimen-trust-patterns.md`

---

## Input Schema

```yaml
input:
  page_type: "[from packages-loaded.md — type_a | type_b | hybrid]"
  vertical: "[from packages-loaded.md — health, golf, info_product, b2b, ecomm, etc.]"
  specimen_index_path: "LandingPageEngine/specimens/specimen-index.md"
  element_taxonomy_path: "LandingPageEngine/element-taxonomy.md"
```

---

## Execution Logic

1. **Read specimen-index.md** and identify specimens matching the project's vertical and page type.

2. **Read element-taxonomy.md** Trust Elements section — extract frequency data for each trust element type across all 11 specimens.

3. **Load matching specimens** (raw files from `specimens/raw/`) and extract trust-specific patterns:

   **For each matching specimen, extract:**
   - Trust bar: What signals? How many? What copy? What order?
   - Trust badges: What badges? Where placed? What text?
   - Certifications: Which ones listed? How explained?
   - Media mentions: How displayed? With quotes or just logos?
   - Authority signals: Named experts? How credentialed?
   - Volume signals: Customer count? Review count? How formatted?
   - Security signals: Where placed? What copy?
   - Rating strip: Format? Context copy?

4. **Priority loading order based on page type:**

   **Type A specimens (long-form):**
   - LP-SPEC-10 (BIOptimizers): GMP, Prop 65, COA certifications + 4 expert endorsements + 1M+ customers + 365-day guarantee + 1,561 reviews
   - LP-SPEC-01/02 (MAMG): Trustpilot widget + "1M golfers" + partner logos
   - LP-SPEC-06 (Richmond Dinh): Celebrity endorsement cards + "$30M+, 1200+, 500+" stat claim bar

   **Type B specimens (ecomm/PDP):**
   - LP-SPEC-09 (LMNT): 83,502 reviews + NQARP guarantee + trust badges above fold + rating strip
   - LP-SPEC-10 (BIOptimizers): Certification badges + "Best Seller" badge + subscription benefits
   - LP-SPEC-07 (Hyros): 35+ testimonials as trust architecture (extreme proof density model)

   **If no exact vertical match:** Load the closest page-type match and note the vertical difference.

5. **Extract 3–5 verbatim trust copy examples** from loaded specimens — these become reference points during Layer 2 generation.

6. **Write `specimen-trust-patterns.md`** with all extracted patterns organized by trust element type.

## Quality Criteria
- At least 2 specimens loaded and analyzed (even if not perfect vertical match)
- Trust patterns extracted by type (not lumped together)
- Verbatim copy examples captured for Layer 2 reference
- Patterns identified across specimens (what is common vs unique)
- Relevance to current project noted for each specimen

## Failure Modes
- Loading zero specimens → Must load at least 2. If no vertical match, load closest page-type match.
- Extracting only surface-level observations ("has trust badges") without the actual copy text → Verbatim copy examples are required.
- Treating all specimens as equally relevant → Note relevance level (HIGH/MEDIUM/LOW) based on vertical and page type match.
- Skipping the element-taxonomy.md frequency data → This data shows which trust elements are universal vs specialized.
