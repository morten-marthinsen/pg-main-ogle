# 0.1 — Brief Loader

**Skill:** LP-08-trust-elements
**Position:** Layer 0, Microskill 1
**Purpose:** Load and verify the three upstream package files that LP-08 depends on: `page-brief.json` (LP-00), `above-fold-blueprint.json` (LP-03), and `proof-architecture.json` (LP-05). Extract the key fields that govern trust element decisions.
**Effort Level:** low
**Dependencies:** LP-00 output (`page-brief.json`), LP-03 output (`above-fold-blueprint.json`), LP-05 output (`proof-architecture.json`)
**Output:** `packages-loaded.md`

---

## Input Schema

```yaml
input:
  project_directory: "[path to active project output directory]"
  expected_files:
    - "page-brief.json"
    - "above-fold-blueprint.json"
    - "proof-architecture.json"
```

---

## Execution Logic

1. **Locate each upstream file** in the project output directory. Check for existence of all three files.

2. **If any file is MISSING:** HALT immediately. Report which file(s) are absent. LP-08 cannot proceed without all three upstream packages.

3. **Load `page-brief.json`** and extract:
   - `page_type` (type_a | type_b | hybrid)
   - `vertical` (health, finance, info_product, personal_dev, technology, etc.)
   - `awareness_stage` (1-5)
   - `product_name`
   - `guarantee_details` (type, duration, conditions — feeds trust bar)
   - `shipping_policy` (free shipping threshold, delivery time — feeds trust bar)
   - `certifications_listed` (GMP, NSF, Prop 65, etc. — feeds certification writer)
   - `media_mentions` (publication names, quotes if available)
   - `expert_endorsements` (names, credentials, statements if available)
   - `customer_count` (specific number if available)
   - `review_count` (specific number if available)
   - `copywriting_engine_mode` (downstream | standalone)

4. **Load `above-fold-blueprint.json`** and extract:
   - `trust_bar_placement` (above_fold | below_hero | below_nav)
   - `trust_badge_slots` (count and position — Type B)
   - `rating_strip_included` (Y/N — Type B)
   - `above_fold_layout` (for understanding trust element spatial context)

5. **Load `proof-architecture.json`** and extract:
   - `proof_strategy.mode` (proof_saturation | authority_substitution)
   - `proof_inventory_summary` (certifications, media mentions, customer counts available)
   - `section_proof_map` — identify which sections have trust-type elements assigned
   - `proof_gaps` — identify trust-related gaps

6. **Cross-validate page type** across all three files. If page types conflict, flag and HALT.

7. **Write `packages-loaded.md`** with all extracted fields organized by source file.

## Quality Criteria
- All three files loaded and confirmed present
- Key fields extracted with no gaps (missing fields marked `not_available`, not silently omitted)
- Page type cross-validated across all three upstream packages
- Trust signal availability clearly summarized — Layer 1 can immediately see what is and is not available
- Output is self-contained — downstream microskills can read this file without loading the original JSONs

## Failure Modes
- One or more upstream files missing → HALT with specific file names listed. Do not proceed with partial data.
- Page type conflict between files → HALT and flag for human review. Do not assume either file is correct.
- Fields present but empty in upstream JSON → Extract as `not_available` and flag — do not invent values.
- Proof architecture does not exist → LP-05 has not run. HALT — LP-05 must complete before LP-08.
