# 3.1 — Trust Copy Validator

**Skill:** LP-08-trust-elements
**Position:** Layer 3, Microskill 1
**Purpose:** Score all trust copy against a 10-point validation audit AND run the anti-slop scan. This is the quality gate that ensures trust copy meets the specificity, accuracy, and credibility standards required before package assembly.
**Effort Level:** medium
**Dependencies:** All Layer 2 outputs (`trust-bar-copy.md`, `media-mention-copy.md`, `certification-copy.md`, `badge-copy.md`, `volume-signal-copy.md`)
**Output:** `trust-validation.md`

---

## Input Schema

```yaml
input:
  trust_bar_copy: "[from trust-bar-copy.md — both variant sets]"
  media_mention_copy: "[from media-mention-copy.md — or not_applicable]"
  certification_copy: "[from certification-copy.md — or not_applicable]"
  badge_copy: "[from badge-copy.md — or not_applicable]"
  volume_signal_copy: "[from volume-signal-copy.md — volume, authority, security sections]"
  trust_inventory: "[from trust-inventory.md — for factual verification]"
  page_type: "[type_a | type_b | hybrid]"
```

---

## Execution Logic

### Part 1: 10-Point Trust Audit

Score each point 0 (fail) or 1 (pass). Minimum 7.0/10 to proceed.

**Trust Bar Quality (3 points)**
1. Trust bar has correct signal count for page type (3–5 Type A, 4–6 Type B)
2. Every trust bar signal is 3–5 words (word count verified per signal)
3. Trust bar signals are ordered by impact (guarantee/strongest first)

**Copy Specificity (3 points)**
4. Zero generic trust signals — every signal contains a specific number, name, timeframe, or certification name
5. Certification explanations translate jargon into audience benefit (not just restate the certification name)
6. Volume signals use specific numbers (not rounded)

**Factual Accuracy (2 points)**
7. Every trust signal traces to the trust inventory (no fabrication)
8. No "FDA Approved" for supplements; no unverifiable "clinically proven" claims

**Copy Quality (2 points)**
9. Zero AI telltales from the anti-slop forbidden word list
10. All copy meets word count constraints (badges 3–7, trust bar 3–5, callouts 1 sentence, certifications 1–2 sentences)

**Scoring:**
- 10/10: Trust copy is ready for assembly
- 7–9/10: Minor issues — note and proceed
- 5–6/10: Structural issues — revise before proceeding
- Below 5: Significant rewrite required

### Part 2: Anti-Slop Scan

Scan ALL trust copy against the 5-category LP-08 forbidden word list:

**Category 1 — Generic Trust Signals:**
100% satisfaction guaranteed, premium quality, best seller (without source), trusted brand, doctor recommended (without named doctor), highest quality, top-rated (without source), world-class, industry-leading, superior quality

**Category 2 — AI Telltales:**
revolutionary, groundbreaking, cutting-edge, state-of-the-art, innovative, next-level, transformative, game-changing, unlock, harness

**Category 3 — Unverifiable Claims:**
clinically proven (without citation), backed by science (without study), doctor-approved (without named doctor), lab-tested (without specifying which lab), FDA-approved (for supplements)

**Category 4 — Vague Volume Claims:**
thousands of happy customers, millions served, countless reviews, tons of 5-star reviews, loved by many, trusted by thousands

**Category 5 — Empty Credibility:**
you can trust us, we guarantee quality, your satisfaction is our priority, we stand behind our product, quality is our passion, excellence in every bottle

**Scan each element independently:**
- Trust bar copy (both variant sets)
- Media mention copy (if applicable)
- Certification copy (if applicable)
- Badge copy (if applicable)
- Volume signal copy (if applicable)
- Authority signal copy (if applicable)
- Security signal copy (if applicable)

### Part 3: Factual Accuracy Cross-Check

For every trust signal in the output, verify it has a matching entry in trust-inventory.md.

## Quality Criteria
- All 10 audit points explicitly evaluated (no points skipped)
- Anti-slop scan covers ALL trust copy elements (not just trust bar)
- Factual accuracy cross-check covers every trust signal in output
- `not_applicable` elements get auto-PASS on relevant audit points (not penalized for absence)
- Failed elements have specific revision instructions (not just "fix this")

## Failure Modes
- Scoring a generic signal as specific ("Premium Quality" passes specificity) → "Premium Quality" is on the forbidden list. It fails both specificity AND anti-slop.
- Skipping anti-slop scan because "trust copy is short" → Short copy is MORE likely to contain generic filler. Scan every element.
- Not cross-checking factual accuracy → This is the fabrication safety net. Every signal must trace to inventory.
- Scoring 6/10 as "close enough to 7" → 6 is not 7. Revise until 7.0 is met.
