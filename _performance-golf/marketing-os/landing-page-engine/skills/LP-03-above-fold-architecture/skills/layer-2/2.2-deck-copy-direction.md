# 2.2 — Deck Copy Direction

**Skill:** LP-03-above-fold-architecture
**Position:** Layer 2, Microskill 2
**Purpose:** Specify the promise expansion strategy for deck copy (the subheadline/supporting text beneath the main headline) based on the selected headline type. Deck copy bridges the headline promise to the first scroll action.
**Effort Level:** medium
**Dependencies:** 2.1 headline-type.md, 1.2 awareness-map.md, 0.1 input-verification.md
**Output:** `deck-direction.md`

---

## Input Schema

```yaml
input:
  headline_type: "[path to headline-type.md]"
  awareness_map: "[path to awareness-map.md]"
  input_verification: "[path to input-verification.md]"
  page_type: "[Type A | Type B | Hybrid]"
```

---

## Execution Logic

**Step 1:** Read `headline-type.md`. Extract selected headline type (T1-T7) and the deck copy strategy from the AGENT.md mapping.

**Step 2:** Apply the Deck Copy Strategy table from AGENT.md:

| Headline Type | Deck Copy Direction |
|--------------|---------------------|
| T1 Direct Benefit | Mechanism hint + specificity addons ("...even if you've tried everything") |
| T2 Curiosity | Expand the curiosity gap, hint at the villain |
| T3 Problem-First | Validate the struggle, promise the revelation |
| T4 Warning | Agitate the danger, promise the solution |
| T5 Question | Answer the question PARTIALLY, promise full answer below |
| T6 How-To | Set up the story, establish the expert/Yoda |
| T7 Number-Specific | Expand the social proof, make it relevant to THIS reader |

**Step 3:** Adapt the direction for page type:
- **Type A:** Deck copy can be 2-4 lines. Room for narrative setup. Functions as the "lead-in" to the below-fold education.
- **Type B:** Deck copy must be 1-2 lines max. Functions as a product descriptor or benefit bullet. Must work alongside product image.
- **Hybrid:** 1-3 lines. Balances product context with narrative hook.

**Step 4:** Write 2-3 directional variants for the deck copy. Each variant is NOT the final copy — it is a strategy brief that LP-07 will execute. Include:
- The emotional lever being pulled (fear, curiosity, desire, urgency, belonging)
- The information being revealed or withheld
- The specific reader action implied (keep reading, watch video, scroll to learn more)

**Step 5:** Flag any CopywritingEngine downstream data that should inform deck copy:
- If promise-output.json exists → deck copy should echo the promise language
- If mechanism name exists → deck copy should hint at mechanism without naming it fully
- If research quotes exist → deck copy vocabulary should match audience language

**Step 6:** Write `deck-direction.md`.

## Quality Criteria
- Deck strategy directly maps to selected headline type per AGENT.md table
- Length constraint appropriate to page type (Type B never exceeds 2 lines)
- Minimum 2 variants provided, each with distinct emotional lever
- Variants are strategy briefs, NOT draft copy
- CopywritingEngine integration flags populated when in Downstream mode
- File exists after execution

## Failure Modes
- headline-type.md not found → HALT, run 2.1 first
- Deck strategy not applicable to page type (e.g., T6 story setup for Type B PDP) → Adapt strategy to fit Type B constraints, note the adaptation
- All variants use the same emotional lever → REJECT, diversify levers
- Variant reads as draft copy instead of direction → Rewrite as strategy brief
