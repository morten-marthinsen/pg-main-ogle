# 2.1 — Headline Type Selector

**Skill:** LP-03-above-fold-architecture
**Position:** Layer 2, Microskill 1
**Purpose:** Select the optimal headline type from the 7-type taxonomy (T1-T7), scored against awareness stage, traffic temperature, and vertical norms. This selection governs all downstream headline writing in LP-07.
**Effort Level:** high
**Dependencies:** 1.1 traffic-analysis.md, 1.2 awareness-map.md, 1.3 pattern-selection.md, 0.2 specimen-load.md
**Output:** `headline-type.md`

---

```yaml
input:
  traffic_analysis: "[path to traffic-analysis.md]"
  awareness_map: "[path to awareness-map.md]"
  pattern_selection: "[path to pattern-selection.md]"
  specimen_load: "[path to specimen-load.md]"
  page_type: "[Type A | Type B | Hybrid]"
  vertical: "[vertical]"
```

## Execution Logic

**Step 1:** Read all dependency files. Extract:
- Traffic temperature and headline constraints (from 1.1)
- Awareness stage and primary/avoid headline types (from 1.2)
- Selected above-fold pattern (from 1.3)
- Specimen headline types observed (from 0.2)

**Step 2:** Score each of the 7 headline types against the awareness stage validation table:

| Type | Unaware | Problem-Aware | Solution-Aware | Product-Aware | Most-Aware |
|------|---------|---------------|----------------|---------------|------------|
| T1: Direct Benefit | 0 | 1 | 2 | 3 | 3 |
| T2: Curiosity/Revelation | 3 | 2 | 2 | 1 | 0 |
| T3: Problem-First | 1 | 3 | 1 | 0 | 0 |
| T4: Warning/Alert | 3 | 2 | 0 | 0 | 0 |
| T5: Question | 1 | 2 | 2 | 1 | 0 |
| T6: How-To/Discovery | 2 | 2 | 2 | 1 | 0 |
| T7: Number-Specific | 0 | 1 | 2 | 3 | 2 |

**Step 3:** Apply traffic temperature modifier:
- Cold traffic: +1 to T2, T3, T4, T6. -1 to T1, T7.
- Warm traffic: +1 to T1, T5, T6. -1 to T4.
- Hot traffic: +1 to T1, T7. -1 to T2, T3, T4.

**Step 4:** Apply pattern compatibility modifier:
- Pattern 1 (VSL-First): +1 to T2, T4 (pattern interrupt needed to get play)
- Pattern 2 (Product Hero): +1 to T1, T7 (product context established by image)
- Pattern 3 (Headline+CTA+Trust): No modifier (pattern works with all types)
- Pattern 4 (Proof-First): +1 to T7, T6 (proof-led entry amplifies these)
- Pattern 5 (Brand Hero): +1 to T1, T5 (brand context enables direct approach)

**Step 5:** Apply specimen evidence modifier:
- If 3+ specimens in vertical use a type: +1 to that type
- If 0 specimens use a type: -1 (unvalidated in vertical)

**Step 6:** Rank all 7 types by final score. Select the top-scoring type as PRIMARY. Select the second-highest as SECONDARY (for LP-07 A/B test consideration).

**Step 7:** Validate the selection:
- If awareness stage is "Avoid" for the selected type → FLAG and reconsider
- If selected type contradicts traffic constraints from 1.1 → Document conflict and resolve
- If Type B page and selected type is T4 (Warning) → FLAG — Warning headlines rarely work on PDPs

**Step 8:** Write `headline-type.md`.

## Quality Criteria
- All 7 types scored with visible math (base + modifiers = total)
- Selected type is highest-scoring unless validation override applies
- Deck copy strategy extracted from AGENT.md, not invented
- Secondary type identified for A/B test consideration
- No "Avoid" types selected without explicit override justification
- File exists after execution

## Failure Modes
- Top two types tied → Select the one with higher awareness base score (core fit over modifiers)
- All types score below 2 → Flag anomaly, document why no type fits, default to T3 (Problem-First) as safest universal type
- Awareness stage missing → HALT, run 1.2 first
- Selected type is on "Avoid" list for awareness stage → Must provide explicit justification or select next-best type
