# 1.2 — Awareness Stage Mapper

**Skill:** LP-03-above-fold-architecture
**Position:** Layer 1, Microskill 2
**Purpose:** Map the audience's Schwartz awareness stage to determine headline approach and deck copy strategy. Awareness stage is the foundational constraint for all above-fold copy direction.
**Effort Level:** medium
**Dependencies:** 0.1 input-verification.md, 1.1 traffic-analysis.md
**Output:** `awareness-map.md`

---

## Input Schema

```yaml
input:
  input_verification: "[path to input-verification.md]"
  traffic_analysis: "[path to traffic-analysis.md]"
  lp00_awareness: "[awareness stage from page-brief.json, or 'not specified']"
  lp01_awareness: "[awareness stage from conversion-intelligence.json, or 'not available']"
```

---

## Execution Logic

**Step 1:** Check for explicit awareness stage from LP-00 brief or LP-01 intelligence. If provided by either source, use it as the starting classification.

**Step 2:** If no explicit awareness stage is available, infer from available signals:

```
UNAWARE: Doesn't know they have a problem.
  Signals: Cold traffic + broad targeting, no search intent, interruption media
  Example: "You're scrolling Facebook and see an ad about metabolism"

PROBLEM-AWARE: Knows the problem, not the solution category.
  Signals: Cold/warm traffic, symptom-based search, problem-oriented content
  Example: "I can't lose weight no matter what I try"

SOLUTION-AWARE: Knows solutions exist, hasn't chosen one.
  Signals: Warm traffic, category search ("best weight loss supplements"), comparison shopping
  Example: "I know supplements can help but which one?"

PRODUCT-AWARE: Knows THIS product, hasn't bought.
  Signals: Warm/hot traffic, branded search, retargeting, email list
  Example: "I've seen [Product] before, is it worth it?"

MOST-AWARE: Knows product, needs final push.
  Signals: Hot traffic, cart abandoner, repeat visitor, "buy [product]" search
  Example: "I'm ready to buy, just need the deal"
```

**Step 3:** Cross-validate awareness stage against traffic temperature from 1.1:
- Cold traffic + Most-Aware = UNLIKELY → re-examine classification
- Hot traffic + Unaware = IMPOSSIBLE → override to Product-Aware minimum
- Warm traffic should map to Problem-Aware through Product-Aware range

**Step 4:** Build the headline type mapping from awareness stage:

| Awareness Stage | Primary Headline Types | Secondary | Avoid |
|----------------|----------------------|-----------|-------|
| Unaware | T4 (Warning), T2 (Curiosity) | T6 (How-To) | T1 (Direct Benefit), T7 (Number) |
| Problem-Aware | T3 (Problem-First), T5 (Question), T6 (How-To) | T2 (Curiosity) | T7 (Number) |
| Solution-Aware | T1 (Direct Benefit), T2 (Curiosity), T6 (How-To) | T7 (Number) | T4 (Warning) |
| Product-Aware | T1 (Direct Benefit), T7 (Number) | T5 (Question) | T4 (Warning), T3 (Problem) |
| Most-Aware | T1 (Direct Benefit), T7 (Number) | — | T2-T6 (too much preamble) |

**Step 5:** Document the deck copy strategy implied by awareness stage:
- Unaware: Deck must EDUCATE about the problem before hinting at solution
- Problem-Aware: Deck must VALIDATE the struggle, then promise revelation
- Solution-Aware: Deck must DIFFERENTIATE from known alternatives
- Product-Aware: Deck must REINFORCE with proof and specifics
- Most-Aware: Deck must CLOSE with urgency and offer details

**Step 6:** Write `awareness-map.md`.

---

## Quality Criteria
- Awareness stage mapped to exactly one of five Schwartz levels
- Cross-validation performed against traffic temperature (no impossible combinations)
- Headline type mapping aligns with AGENT.md taxonomy table
- Deck copy strategy is specific to awareness stage (not generic)
- File exists after execution

## Failure Modes
- No awareness data from any source → Infer from traffic temperature: Cold=Problem-Aware (safe default), Warm=Solution-Aware, Hot=Product-Aware
- Traffic temperature and awareness stage conflict → Document conflict, use awareness stage if from explicit source, traffic temperature if both inferred
- input-verification.md or traffic-analysis.md not found → HALT, run prerequisite microskills
