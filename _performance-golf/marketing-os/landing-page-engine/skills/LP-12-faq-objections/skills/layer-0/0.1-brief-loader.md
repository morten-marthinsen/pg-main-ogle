## Execution Logic

1. **Locate each upstream file** in the project output directory. Check for existence of all three required files.

2. **If any required file is MISSING:** HALT immediately. Report which file(s) are absent. LP-12 cannot proceed without all three upstream packages.

3. **Load `page-brief.json`** and extract:
   - `page_type` (type_a | type_b | hybrid)
   - `vertical` (health, finance, info_product, personal_dev, technology, etc.)
   - `awareness_stage` (1-5)
   - `traffic_temperature` (cold | warm | hot)
   - `audience_profile` (primary pain, failed solutions, primary desire, demographics)
   - `voice_direction` (tone, register, anti-voice)
   - `offer_summary` (product name, price point, guarantee, bonuses)
   - `copywriting_engine_mode` (downstream | standalone)
   - `research_quotes` (if available â€” audience language about objections/concerns)

4. **Load `conversion-intelligence.json`** and extract:
   - `industry_benchmarks` (conversion rates, bounce rates for this vertical)
   - `objection_signals` (any objection data from competitive analysis or conversion data)
   - `price_sensitivity_indicators` (if available)
   - `trust_deficit_signals` (if available â€” new brand vs established, refund rate data)
   - `comparison_landscape` (key competitors mentioned, alternatives audience considers)

5. **Load `section-sequence.json`** and extract:
   - `faq_section_position` (where FAQ falls in the page sequence)
   - `section_before_faq` (what section precedes the FAQ â€” offer, CTA, urgency?)
   - `section_after_faq` (what section follows the FAQ â€” final CTA, guarantee, close?)
   - `total_sections` (count)
   - `proof_density_tier` (high/medium/low â€” indicates how much proof is elsewhere on page)

6. **Optionally load `hero-section-package.json`** (LP-07) if it exists:
   - `threading_anchors.promise_statement` (core promise the FAQ must be consistent with)
   - `hero_copy.headline.text` (the headline the reader saw at the top)
   - Voice/tone reference from the hero section

7. **Optionally load `offer-pricing-package.json`** (LP-11) if it exists:
   - `price_point` (exact price â€” needed for price/value FAQ answers)
   - `guarantee_details` (guarantee type, duration, conditions â€” needed for trust FAQ answers)
   - `bonuses` (bonus items â€” context for value questions)
   - `comparison_value` (the "value" number from the offer stack)

8. **Cross-validate page type** across all files. If `page-brief.json` says Type A but `section-sequence.json` was built for Type B, flag the conflict and HALT.

9. **Write `packages-loaded.md`** with all extracted fields organized by source file.

## Quality Criteria
- All three required files loaded and confirmed present
- Key fields extracted with no gaps (missing fields marked `not_available`, not silently omitted)
- Page type cross-validated across all upstream packages
- Optional files loaded if available (hero-section, offer-pricing) for threading and specificity
- Sequencing implication noted (page type determines FAQ sequence strategy)
- Output is self-contained â€” any downstream microskill can read this file without loading the original JSONs

## Failure Modes
- One or more required upstream files missing -> HALT with specific file names listed. Do not proceed with partial data.
- Page type conflict between files -> HALT and flag for human review. Do not assume either file is correct.
- Fields present but empty in upstream JSON -> Extract as `not_available` and flag â€” do not invent values.
- Optional files missing -> Note as `NOT AVAILABLE` and proceed. These are helpful but not blocking.
