# 2.4 — CTA Placement Planner

**Skill:** LP-04-section-sequence
**Position:** Layer 2, Microskill 4
**Purpose:** Map every CTA placement in the page sequence — which sections carry CTAs, what type each CTA is, what emotional appeal drives each placement, and how CTAs escalate from soft to hard through the page. Minimum 3 CTA placements. This plan is consumed by LP-06 (Offer/CTA Architecture) for detailed CTA design and LP-14 (CTA Copy Optimizer) for copy execution.
**Effort Level:** high
**Dependencies:** `word-count-plan.md` (2.1), `proof-density-plan.md` (2.2), `template-selection.md` (1.1), `section-1-integration.md` (1.4), `page-brief.json`
**Output:** `cta-placement-plan.md`

## CTA Types

| Type | Purpose | Typical Placement |
|------|---------|------------------|
| `primary_buy` | Main purchase CTA — "Buy Now" / "Add to Cart" / "Get Started" | After value stack/offer block, pre-close, close |
| `mid_page_conversion` | Re-engage scrollers — "Yes, I Want This" / "Reserve Mine" | After major proof block or mechanism reveal |
| `pre_close` | Urgency-driven — "Claim Your [X] Before [Deadline]" | Before or after guarantee block |
| `soft_cta` | Read-on or engagement — "See the Proof" / "Keep Reading" | Early page, above fold (Type A) |
| `sticky_mobile` | Always-visible mobile CTA bar | Persistent on scroll (Type B mandatory) |
| `exit_intent` | Last-chance capture — "Wait — Special Offer" | Triggered on exit behavior |

## Execution Logic

**Step 1:** Identify all potential CTA placement points in the sequence:
- After each proof block (belief established → ask)
- After mechanism reveal (understanding established → bridge to product)
- After value stack/offer block (value established → price)
- After guarantee block (risk removed → ask)
- Close section (final emotional push → ask)
- P.S. section (last chance → ask)

**Step 2:** Select minimum 3 CTA placements. Apply the escalation principle:
- **First CTA:** Softest ask. Appears after first major credibility moment. Type A: often after early proof block or mechanism. Type B: above-fold ATC.
- **Second CTA:** Medium commitment. Appears after peak proof density or offer block.
- **Third CTA:** Hardest ask. Appears in close section with full emotional urgency.

**Step 3:** For each CTA placement, specify:
- Section number and section name
- CTA type (from types table)
- Emotional appeal driving this CTA (what belief state the visitor is in at this point)
- Button copy direction (not final copy — that's LP-14 — but the general tone/direction)
- Visual treatment (button size, color emphasis, standalone vs. inline)

**Step 4:** Apply awareness-stage modifiers:
- Stage 1–2: First CTA appears LATER (after more education). Minimum 40% of page before first primary buy CTA.
- Stage 3: Standard CTA pacing.
- Stage 4–5: First CTA can appear EARLIER. Above-fold CTA acceptable even for Type A.

**Step 5:** Apply page-type rules:
- Type B: Sticky mobile CTA is MANDATORY. Above-fold ATC is standard.
- Type A: No above-fold buy CTA (unless Stage 4–5). First buy CTA after mechanism or proof.
- Hybrid: Above-fold ATC (Type B zone) + escalating CTAs in Type A education zone.

**Step 6:** Verify minimum 3 CTA placements mapped. If fewer than 3, add placements.

## Quality Criteria
- Minimum 3 CTA placements mapped — 3 is the absolute minimum, more common to have 4–6
- Each CTA has a specific emotional appeal, not just "buy now here"
- Escalation principle followed — soft to hard through the page
- Awareness-stage modifier applied — not default pacing
- Sticky mobile CTA present for Type B pages
- No primary buy CTA before trust/proof signal (validates Test 2 from 3.1)

## Failure Modes
- Fewer than 3 CTA placements → FAIL. Add placements before proceeding.
- Primary buy CTA placed before any proof or trust element → FAIL. Reposition after first proof block.
- Type B page missing sticky mobile CTA → FAIL. Add sticky mobile CTA.
- All CTAs clustered in bottom 20% of page → WARNING. Spread CTAs across page sections for scroll-dropoff capture.
- CTA emotional appeals are generic ("buy now because it's good") → FAIL anti-slop. Each CTA must reference a specific belief state the visitor holds at that point in the sequence.
