# 2.3 — Visual Density Planner

**Skill:** LP-04-section-sequence
**Position:** Layer 2, Microskill 3
**Purpose:** Specify the image count, image type, layout, and mobile collapse behavior for every section in the sequence. This is the visual specification that LP-15 (Assembly) and design teams execute against. No section should be a flat wall of text — every section gets a visual density spec even if the spec is "text_primary with 0 images."
**Effort Level:** medium
**Dependencies:** `word-count-plan.md` (2.1), `template-selection.md` (1.1), `page-brief.json` (product type, available imagery)
**Output:** `visual-density-plan.md`

---

## Input Schema

```yaml
input:
  sections_with_word_counts: "[from 2.1]"
  page_type: "[type_a | type_b | hybrid]"
  product_type: "[supplement | info-product | service | physical-product | software]"
  available_imagery: "[from page-brief.json — product photos, lifestyle shots, before-afters, headshots]"
```

## Visual Density Targets

**Type A — Long-Form:**
- Overall minimum: 1 image per 400 words
- Above fold: 1 hero image (large)
- Per proof block: 1–3 testimonial headshots
- Per ingredient section: 1 ingredient photo (real, not icon)
- Guarantee block: 1 badge/certificate visual
- Story section: 0–1 (optional founder/discoverer photo)

**Type B — Ecomm/PDP:**
- Overall minimum: 1 image per 200 words
- Above fold: 1 hero image (large, lifestyle) + product gallery (4–8 images)
- Per ingredient: 1 ingredient photo
- Before/after: 2+ comparison photos per result
- Review section: Photo reviews prioritized
- Trust bar: Badge/certification icons

**Image Types:**

| Code | Type | Use Case |
|------|------|----------|
| `product` | Product photography | Hero shots, ingredient sections, offer blocks |
| `lifestyle` | Lifestyle/in-use | Above fold, story sections, benefit illustrations |
| `ingredient` | Ingredient close-up | Ingredient deep-dive sections |
| `before_after` | Transformation | Proof blocks, before/after galleries |
| `infographic` | Data visualization | Mechanism narrative, how-it-works, comparison |
| `badge` | Trust badge/cert | Trust bar, guarantee block |
| `headshot` | Person photo | Testimonials, authority endorsements, founder story |
| `icon` | Illustrated icon | Benefits strip, how-it-works steps |

---

## Execution Logic

**Step 1:** Load the section list with word counts from `word-count-plan.md`.

**Step 2:** For each section, apply the visual density target based on page type:
- Calculate minimum images: section_word_count / density_ratio (400 for Type A, 200 for Type B)
- Round up to nearest whole number
- Minimum 0 images for very short sections (<100 words)

**Step 3:** For each section, assign specific image types based on the section's persuasion function:
- Proof blocks → headshots + before_after
- Mechanism/ingredient sections → ingredient + infographic
- Story sections → lifestyle + headshot
- Offer/pricing → product
- Guarantee → badge
- Trust bar → badge + icon

**Step 4:** Specify layout type for each section:
- `text_primary` — Text dominates, images support
- `image_text_alternating` — Image and text blocks alternate
- `image_grid` — Multiple images in grid (galleries, before/afters)
- `full_width_banner` — Full-width image with text overlay
- `card_grid` — Card layout (benefits, features, ingredients)

**Step 5:** Specify mobile collapse behavior for each section:
- How images restack on mobile
- Which images hide on mobile (if any)
- Touch target considerations for interactive elements

**Step 6:** Check total planned images against available imagery from the brief. Flag gaps.

## Quality Criteria
- Every section has a visual density spec — no section left without explicit image planning
- Image types are specific (not just "add images" — specify type, count, requirement)
- Layout types assigned per section
- Mobile collapse behavior specified for every section with images
- Total image count meets the page-type minimum ratio
- Imagery gaps flagged with specific requests

## Failure Modes
- Word count plan missing → Cannot calculate density ratios. HALT — run 2.1 first.
- Available imagery field in brief is empty → Plan assumes all images need to be created. Flag every planned image as "requires creation."
- Type B page planned with fewer than 1 image per 200 words → FAIL density check. Add images or justify exception.
