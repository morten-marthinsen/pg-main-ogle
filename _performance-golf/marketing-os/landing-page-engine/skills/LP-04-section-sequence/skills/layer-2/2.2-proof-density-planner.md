# 2.2 — Proof Density Planner

**Skill:** LP-04-section-sequence
**Position:** Layer 2, Microskill 2
**Purpose:** Assign specific proof requirements to every section — not "add testimonials" but "3 testimonials (named, with specific result and location) + 1 clinical study (institution name + specific finding)." This is the specification that LP-10 (Social Proof Writer) and LP-15 (Assembly) execute against.
**Effort Level:** medium
**Dependencies:** `word-count-plan.md` (2.1), `page-brief.json` (especially `proof_inventory`)
**Output:** `proof-density-plan.md`

## Proof Density Levels

| Level | Definition |
|-------|-----------|
| **None** | No proof elements in this section |
| **Light** | 1 signal — authority mention, credibility stat, or quick social validation |
| **Medium** | 2–3 elements — mix of types (e.g., 1 study mention + 1 testimonial) |
| **Heavy** | 4+ elements — multiple testimonials, studies, before/afters, volume proof |
| **Maximum** | 10+ elements — full review cascade, gallery, all proof types |

## Proof Types

| Code | Type | Description |
|------|------|-------------|
| `testimonial` | Customer testimonial | Named person, location, specific result, timeframe |
| `photo_testimonial` | Testimonial with image | Higher credibility than text-only |
| `review` | Platform review | Star rating + text, shown in bulk cascade |
| `study` | Clinical/scientific | Institution name + specific finding + statistic |
| `before_after` | Transformation | Side-by-side with specific outcome + timeframe |
| `authority` | Expert endorsement | Named expert + credentials + specific statement |
| `volume` | Social proof counter | "X customers" / "X reviews" / "X bottles sold" |
| `media` | Media mention | "As seen in [named publication]" |
| `certification` | Third-party cert | GMP, NSF, USP, etc. |
| `demonstration` | Result demonstration | Before/after lab test, quantified outcome |

---

## Execution Logic

**Step 1:** Load proof inventory from page-brief.json. Understand what proof actually EXISTS.

**Step 2:** Apply proof wave pattern:
- **Early zone** (above fold through story/root cause): Light proof only. Audience hasn't been convinced yet — heavy proof too early feels desperate.
- **Mechanism zone** (mechanism narrative + ingredient deep-dives): Medium proof — studies and demonstrations woven in.
- **Proof blocks** (dedicated proof sections): Heavy to Maximum — this is where proof accumulates.
- **Close zone** (guarantee, close, P.S.): Light to medium — transformation callbacks, confidence proof.

**Step 3:** For each section, assign density level AND specific proof elements:
- Specify COUNT per type (not just "testimonials")
- Specify QUALITY requirements (named, specific result, location — not generic "great product!")
- Flag if available proof in inventory is insufficient: "PROOF GAP: 3 testimonials required, only 1 in inventory — flag for client"

**Step 4:** Calculate total proof elements planned across all sections:
- Type A: Minimum 15 total proof touches (testimonials + studies + authority signals combined)
- Type B: Minimum 30 total proof touches (review count is the volume driver)

## Quality Criteria
- Every section has a density level and specific element specifications
- No section has "add testimonials" without a count and quality requirement
- Wave pattern is followed (no Maximum density sections in the lead/story zone)
- Proof gaps explicitly identified
- Total proof touches meet minimums

## Failure Modes
- Proof inventory shows `not_available` → Plan assumes minimum viable proof (e.g., 5 testimonials, 1 study). Flag all planned elements as "requires client confirmation."
- Proof gap identified but product has no testimonials at all → Flag as critical: "CRITICAL PROOF GAP — No testimonials available. Page launch should be delayed until minimum 5 customer testimonials are collected."
