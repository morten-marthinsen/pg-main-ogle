# 1.4 — Above-Fold Integration

**Skill:** LP-04-section-sequence
**Position:** Layer 1, Microskill 4
**Purpose:** Lock Section 1 of the sequence to the LP-03 above-fold blueprint. Section 1 is NOT designed by LP-04 — it is inherited from LP-03 as a fixed input. This microskill pulls the LP-03 spec, formats it as the Section 1 entry in the sequence, and marks it as content-source-locked so no downstream skill attempts to rewrite it.
**Effort Level:** low
**Dependencies:** `above-fold-loaded.md` (0.2), `template-selection.md` (1.1)
**Output:** `section-1-integration.md`

---

## Input Schema

```yaml
input:
  above_fold_spec: "[from above-fold-loaded.md — full Section 1 specification]"
  page_type: "[type_a | type_b | hybrid]"
  template_section_1: "[Section 1 row from template-selection.md]"
```

---

## Execution Logic

**Step 1:** Load the above-fold specification from `above-fold-loaded.md` (produced by 0.2).

**Step 2:** Format the LP-03 spec as a Section 1 entry using the section-sequence schema:
- `number`: 1
- `section_id`: "above_fold"
- `name`: "Above-Fold"
- `mandatory`: true
- `persuasion_function`: Extract from LP-03 — the specific function LP-03 designed for (interrupt, hook, promise, etc.)
- `content_source`: "LP-03 output (above-fold-blueprint.json) — do not re-write this section"
- `word_count`: From LP-03 spec
- `proof_density`: From above-fold-loaded.md proof elements
- `visual_density`: From above-fold-loaded.md hero image spec
- `cta_present`: From above-fold-loaded.md CTA field
- `cta_type`: From above-fold-loaded.md CTA type

**Step 3:** Apply the `CONTENT_SOURCE_LOCK` flag. This flag tells all downstream Phase 3 writing skills: "Do NOT generate content for this section. It is sourced from LP-03."

**Step 4:** Verify Section 1 in the template matches the LP-03 output. If the template's Section 1 assumptions conflict with LP-03's actual design, note the discrepancy (LP-03 is authoritative — it overrides the template default).

## Quality Criteria
- Section 1 is sourced exclusively from LP-03 — no LP-04 redesign of above-fold content
- CONTENT_SOURCE_LOCK flag explicitly set
- All proof and visual density elements from LP-03 captured for Layer 2 planners
- CTA presence and type captured for 2.4 CTA Placement Planner

## Failure Modes
- `above-fold-loaded.md` missing → HALT. Run 0.2 first.
- LP-03 output has no persuasion function specified → Use the page-type default: Type A = "Interrupt + hook + promise"; Type B = "Product + rating + price + ATC"
- LP-03 output missing word count → Estimate from page type defaults: Type A above-fold ~200–400w; Type B above-fold ~50–200w
