# 0.2 — Section Sequence Loader

**Skill:** PDP-07-btf-section-writer
**Position:** Layer 0, Microskill 2
**Purpose:** Load PDP-04's `pdp-section-sequence.json` to get the ordered section list with per-section word count targets, proof density specs, and visual density specs. This file is PDP-07's execution manifest — it determines WHICH sections to write and HOW LONG each should be.
**Effort Level:** low
**Dependencies:** PDP-04 (pdp-section-sequence.json)
**Output:** `section-sequence-loaded.md`

---

## Input Schema

```yaml
input:
  section_sequence_path: "[path to pdp-section-sequence.json — PDP-04 output]"
```

---

## Execution Logic

**Step 1:** Locate and load `pdp-section-sequence.json`. Verify schema version and structure.

**Step 2:** Extract the complete section list. For each section, capture:
- `section_id` (from PDP-04 taxonomy: product_highlights, the_expert, whats_in_it, etc.)
- `name` (human-readable)
- `number` (position in sequence)
- `word_count.target` (the specific number PDP-07 must hit +/-10%)
- `word_count.range` (min-max for reference)
- `proof_density.level` (none/light/medium/heavy/maximum)
- `proof_density.elements` (type + count per section)
- `visual_density` (image count + primary type + layout)
- `notes` (special instructions from PDP-04)

**Step 3:** Identify which sections PDP-07 writes vs which are handled elsewhere:
- **PDP-07 writes:** product_highlights, the_expert, whats_in_it, what_to_expect, the_problem, the_solution, why_it_works, comparison_chart, cost_savings, how_its_made, how_to_use, identity_matching, the_proof
- **PDP-05 writes:** reviews_ratings
- **LP-12 writes:** faq
- **Curated upstream:** ugc_video_carousel (PDP-07 writes labels/captions only if PDP-04 flagged)

**Step 4:** Build PDP-07's execution queue — only sections PDP-07 is responsible for, in sequence order.

**Step 5:** Calculate total word count target for PDP-07's sections (sum of all targets for PDP-07-owned sections).

**Step 6:** Write `section-sequence-loaded.md`.

## Quality Criteria
- pdp-section-sequence.json successfully loaded and parsed
- Every section in PDP-04's sequence accounted for (owned by PDP-07 or delegated to other skill)
- Word count targets extracted as specific numbers (not ranges)
- Proof density specs extracted with type + count
- PDP-07 execution queue built in correct sequence order
- Total word count target calculated

## Failure Modes
- pdp-section-sequence.json not found -> HALT. PDP-04 must run first.
- pdp-section-sequence.json has no sections array -> HALT. PDP-04 output is malformed.
- Word count targets missing for any section -> HALT. PDP-04 must re-run with complete proportioning.
- Proof density specs missing for any section -> WARN. Use PDP-04 defaults from AGENT.md proof density table.
- Section in sequence not recognized (not in 16-section taxonomy) -> HALT. Flag for PDP-04 review.
