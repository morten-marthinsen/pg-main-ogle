# 4.2 — Summary Writer

**Skill:** PDP-07-btf-section-writer
**Position:** Layer 4, Microskill 2
**Purpose:** Write `PDP-BTF-SECTIONS-SUMMARY.md` — a human-readable summary of all BTF sections written, with per-section word counts, key copy highlights, data gaps, and validation results. This is the document Anthony (or any human reviewer) reads to quickly assess PDP-07 output without opening the JSON.
**Effort Level:** low
**Dependencies:** `pdp-btf-sections.json` (4.1)
**Output:** `PDP-BTF-SECTIONS-SUMMARY.md`

---

## Input Schema

```yaml
input:
  btf_sections_json: "[path to pdp-btf-sections.json]"
  product_name: "[from brief]"
```

---

## Execution Logic

**Step 1:** Load `pdp-btf-sections.json`.

**Step 2:** For each section in the JSON, extract:
- Section name and ID
- Word count (target vs actual)
- Key copy highlight (the strongest line or element from the section)
- Validation results (scannability, independence, proof density)
- Data gaps (if any were flagged)

**Step 3:** Build summary table with all sections.

**Step 4:** Calculate aggregate statistics:
- Total word count (target vs actual)
- Number of sections written
- Number of sections passing all validations
- Number of data gaps flagged

**Step 5:** Write `PDP-BTF-SECTIONS-SUMMARY.md`.

---

## Quality Criteria
- Every section from pdp-btf-sections.json represented in summary
- Word counts shown as target vs actual per section
- Key copy highlight is genuinely the strongest line (not generic)
- Data gaps clearly listed
- Summary is scannable and useful for human review (not a data dump)

## Failure Modes
- pdp-btf-sections.json not found -> HALT. Layer 4.1 must run first.
- Section in JSON missing key fields -> Flag incomplete section in summary.
- Key copy highlight is generic ("Great section about ingredients") -> Extract an actual line from the copy.
