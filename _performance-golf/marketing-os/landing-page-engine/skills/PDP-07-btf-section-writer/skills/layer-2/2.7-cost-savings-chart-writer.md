# 2.7 — Cost Savings Chart Writer

**Skill:** PDP-07-btf-section-writer
**Position:** Layer 2, Microskill 7
**Purpose:** Write the Cost Savings Chart section — a side-by-side comparison of buying each ingredient/component individually versus the all-in-one product, dramatizing value with a "You Save $X" punchline. Reduces price objection by doing the math for the customer.
**Effort Level:** high
**Dependencies:** `per-section-briefs.md` (1.1 — cost_savings brief), `data-validation.md` (1.2 — GREEN or YELLOW), `pdp-reference-loaded.md` (0.3 — Section 5.10 template)
**Output:** `section-cost-savings.md`

---

## Input Schema

```yaml
input:
  section_brief: "[cost_savings brief from per-section-briefs.md]"
  word_count_target: "[from PDP-04]"
  proof_density: "[from PDP-04 — none: math, not proof]"
  pdp_reference: "[Section 5.10 template from pdp-reference-loaded.md]"
  product_price: "[from brief]"
  bundle_pricing: "[from brief — if bundles exist]"
  individual_component_prices: "[from brief — per-ingredient or per-component standalone pricing]"
```

---

## Execution Logic

**Step 1:** Read the Cost Savings Chart template from PDP-BEST-PRACTICES-REFERENCE.md Section 5.10. Key pattern: 3-column table (Ingredient | Product | Standalone Price), centered "You Save $X" punchline.

**Step 2:** Build the savings table. Two possible formats based on available data:

**Format A: Ingredient Breakdown (for supplements/formulated products)**
- Column 1: Ingredient/component name
- Column 2: Included in [Product Name] (checkmark + product price at bottom)
- Column 3: Buying separately (individual price per ingredient + total at bottom)

**Format B: Bundle Breakdown (for multi-pack or subscription)**
- Column 1: Package type
- Column 2: Price
- Column 3: Price per serving/unit
- Highlight best value option

**Step 3:** Calculate savings:
- Individual total: Sum of all component standalone prices
- Product price: [from brief]
- Savings: Individual total - Product price
- Savings percentage: (Savings / Individual total) x 100

**Step 4:** Write the "You Save $X" punchline:
- Format: "You Save $[X]" in prominent styled element (starburst, banner, or colored rectangle)
- Include both dollar amount and percentage: "You Save $47 (63% off buying separately)"
- Optional: Add price-per-serving or price-per-day math: "That's just $X.XX per day"

**Step 5:** Write section header that frames the value story:
- Not: "Our Pricing" (generic, sounds like a pricing page)
- Better: "The Real Cost of [Benefit Category]" or "What You'd Pay Separately"

**Step 6:** Add a brief context line above the chart:
- Example: "Getting all these ingredients separately would cost over $[X]/month. [Product Name] gives you everything in one formula."

**Step 7:** If bundle pricing exists, add a bundle comparison below the ingredient breakdown:
- 1-bottle | 3-bottle | 6-bottle with per-unit pricing
- Highlight best value bundle
- "You Save $X with the [N]-pack" per tier

**Step 8:** Verify word count against target (+/-10%).

**Step 9:** Write `section-cost-savings.md`.

## Quality Criteria
- All math is verified and accurate (no rounding errors)
- "You Save $X" punchline prominently displayed
- Savings shown in both dollar amount and percentage
- Individual component prices are realistic (not inflated to exaggerate savings)
- Per-serving or per-day math included for additional value framing
- Section is self-contained (no references to other sections)
- Word count within +/-10% of PDP-04 target

## Failure Modes
- Product price not in brief -> HALT. Cannot build savings chart without pricing data.
- Individual component prices not available -> Use market average prices with "based on average market pricing" disclaimer. Flag as data gap.
- Math error in savings calculation -> HALT. Double-check all arithmetic before writing file.
- Savings amount appears implausibly large -> Verify individual prices are realistic, not inflated.
- Missing "You Save" punchline -> HALT. This is the entire point of the section.
- Bundle pricing data not available but PDP-04 specified bundles -> Write ingredient breakdown only, flag bundle gap.
