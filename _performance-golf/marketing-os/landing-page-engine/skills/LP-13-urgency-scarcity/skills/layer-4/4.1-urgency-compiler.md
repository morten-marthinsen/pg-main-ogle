# 4.1 — Urgency Package Compiler

**Skill:** LP-13-urgency-scarcity
**Position:** Layer 4, Microskill 1
**Purpose:** Assemble all validated urgency/scarcity copy into a single structured package for downstream skills. LP-14 (CTA Optimizer) and LP-15 (Assembly) will load this file to integrate urgency elements into CTAs and the full page.
**Effort Level:** low
**Dependencies:** All Layer 2 and Layer 3 output files, `urgency-types-selected.md` (1.1), `justification-plan.md` (1.2), `placement-strategy.md` (1.3)
**Output:** `urgency-package.json`

---

## Input Schema

```yaml
input:
  urgency_types_selected: "[from 1.1]"
  justification_plan: "[from 1.2]"
  placement_strategy: "[from 1.3]"
  countdown_copy: "[from 2.1 if applicable]"
  stock_level_copy: "[from 2.2 if applicable]"
  price_deadline_copy: "[from 2.3 if applicable]"
  availability_copy: "[from 2.4 if applicable]"
  urgency_support_copy: "[from 2.5]"
  urgency_audit: "[from 3.1]"
  justification_audit: "[from 3.2]"
  page_type: "[type_a | type_b | hybrid]"
  project_name: "[from packages-loaded.md]"
```

---

## Pre-Assembly Verification

Before assembling the package, verify all gates pass:

```yaml
ASSEMBLY-GATE:
  urgency_audit_score: "[X.X -- must be >= 7.0]"
  justification_audit: "[PASS | FAIL -- must be PASS]"
  fabricated_claims: "[must be 0]"
  urgency_type_count: "[must be 0, 1, or 2]"
  anti_slop_scan: "[PASS | FAIL -- must be PASS]"
  all_gates_pass: "[Y/N]"

  IF all_gates_pass = N: HALT -- do not assemble. Return to failed validation.
```

---

## Execution Logic

1. **Verify all gates pass.** Run the assembly gate check above. If any gate fails, HALT and return to the failing validation step.

2. **Determine urgency presence.** If `urgency_types_selected.md` contains `no_justified_urgency: true`, assemble a package with `urgency_present: false` and skip copy assembly.

3. **Assemble Type A elements** (if page_type = type_a or hybrid):
   - Pull urgency section headline (from 2.1, 2.2, 2.3, or 2.4 — whichever type's copy includes a headline)
   - Pull urgency section body copy
   - Pull natural urgency callbacks (from 2.5 if Type 6 selected)
   - Pull P.S. urgency callback (from 2.5)
   - Pull close section urgency line (from 2.5)
   - Calculate total word count

4. **Assemble Type B elements** (if page_type = type_b or hybrid):
   - Pull countdown timer context (from 2.1 if applicable)
   - Pull stock level indicator (from 2.2 if applicable)
   - Pull price deadline badge (from 2.3 if applicable)
   - Pull availability window (from 2.4 if applicable)
   - Pull cart urgency (from relevant Layer 2 file)
   - Assign placements from placement-strategy.md

5. **Assemble urgency support copy:**
   - Pull master justification paragraph (from 2.5)
   - Pull consequence of delay statement (from 2.5)
   - Calculate word count

6. **Assemble validation block:**
   - Urgency audit score
   - Justification audit result
   - Fabricated claims count (must be 0)
   - Urgency type count
   - Anti-slop result

7. **Assemble downstream handoffs:**
   - LP-14: Urgency type + deadline/constraint for CTA framing
   - LP-15: Section placement map + callbacks + P.S. line + close line

8. **Write `urgency-package.json`** following the schema from LP-13-AGENT.md.

## Package Schema

Follow the `urgency-package.json` schema defined in LP-13-AGENT.md. Key sections:

```json
{
  "schema_version": "1.0",
  "skill": "LP-13-urgency-scarcity",
  "created": "[ISO timestamp]",
  "project_name": "[product name]",
  "page_type": "[type_a | type_b | hybrid]",
  "urgency_present": "[true | false]",
  "urgency_types_selected": [],
  "urgency_type_count": "[0 | 1 | 2]",
  "type_a_urgency": {},
  "type_b_urgency": {},
  "urgency_support_copy": {},
  "validation": {},
  "downstream_handoffs": {}
}
```

**For `urgency_present: false` packages:**
- `urgency_types_selected`: empty array
- `urgency_type_count`: 0
- `type_a_urgency`: all fields null
- `type_b_urgency`: all fields null
- `urgency_support_copy`: all fields null
- `validation.urgency_audit_score`: "N/A — no urgency"
- `validation.justification_audit`: "N/A — no urgency"
- `downstream_handoffs`: note that no urgency elements are available

**Important assembly rules:**
- Only include copy elements for selected urgency types. If Type 1 was not selected, `stock_level_indicator` fields should be null.
- All copy in the package must have passed the anti-slop scan and justification audit.
- Threading info for downstream skills must be explicit — LP-14 and LP-15 read this package without context about LP-13's execution.

## Quality Criteria
- Package contains ALL validated copy elements (nothing dropped during assembly)
- Only selected urgency types have populated fields (unselected types are null)
- Validation block accurately reflects audit scores (not inflated)
- Downstream handoff descriptions are specific enough for LP-14 and LP-15 to act on
- Valid JSON structure (parseable by downstream skills)
- `urgency_present: false` packages are properly structured (not just empty)

## Failure Modes
- Assembling package before all gates pass -> Run assembly gate check first. One failed gate = HALT.
- Dropping copy elements during assembly (support copy written but not included in package) -> Verify every Layer 2 output file's content appears in the package.
- Including copy from unselected urgency types -> If Type 1 was not selected, no stock level copy should appear in the package.
- Downstream handoffs that are vague ("urgency available for CTA") -> Handoffs must specify the urgency TYPE, the specific constraint, and the deadline/number so downstream skills can act without loading LP-13's execution files.
- Package showing `urgency_present: true` with no copy elements -> If urgency is present, at least one copy element must exist for the selected type(s).
