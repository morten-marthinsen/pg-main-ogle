# 2.1 — Product Title Rules

**Skill:** PDP-03-hero-carousel-buybox
**Position:** Layer 2, Microskill 1
**Purpose:** Specify the product title display rules for the buy box. Baymard research: users confirm they have the correct product variation by reading the full title. Truncation creates purchase anxiety. The title MUST display in full, even if it takes 3 lines.
**Effort Level:** low
**Dependencies:** 0.1 input-verification.md
**Output:** `title-rules.md`

---

## Input Schema

```yaml
input:
  input_verification: "[path to input-verification.md]"
  product_name: "[full product name from brief]"
  variant_types: "[size, flavor, color, etc.]"
```

---

## Execution Logic

**Step 1:** Extract the full product title from `input-verification.md`. The title typically includes:
- Brand name
- Product name
- Key descriptor (e.g., "Extra Strength", "Original Formula")
- Size/count (e.g., "60 Capsules", "30 Servings")
- Variant if pre-selected (e.g., "Chocolate Flavor")

**Step 2:** Construct the title hierarchy:
- **Line 1:** Brand name (if brand is not the product name)
- **Line 2:** Product name + descriptor
- **Line 3:** Size/count + variant (if applicable)
- If brand IS the product name, combine into 2 lines max

**Step 3:** Specify display rules:
- **NEVER truncate.** Full title displays regardless of length.
- **Max lines:** Allow up to 3 lines on mobile, 2 lines on desktop
- **Typography:** H1 tag, product title font (typically 20-24px mobile, 24-28px desktop)
- **Line height:** 1.2-1.3 for readability
- **Color:** Primary text color (dark, high contrast)
- **Variant update:** Title dynamically updates when user selects a different variant (e.g., flavor changes from "Chocolate" to "Vanilla")

**Step 4:** Specify what the title does NOT include:
- Rating stars (separate component — 2.2)
- Price (separate component — 2.4)
- Marketing taglines or benefit claims
- "Best Seller" or "New" badges (these go near title but are separate elements)

**Step 5:** Specify badge positioning (if applicable):
- "Best Seller" badge: Above or inline-left of title, small pill shape
- "New" badge: Same position options
- "Limited Edition" badge: Same position options
- Badges must not push title down or cause layout shift

**Step 6:** Write `title-rules.md`.

---

## Output Schema

`title-rules.md`:

```markdown
# Product Title Rules — PDP-03

## Full Title: [complete product title]

---

## Display Structure
- **Line 1:** [brand name or first line]
- **Line 2:** [product name + descriptor]
- **Line 3:** [size/count + variant, if applicable]

## Typography
- **Tag:** H1
- **Font Size:** [mobile: Xpx, desktop: Xpx]
- **Line Height:** [1.2-1.3]
- **Color:** [primary text color]
- **Weight:** [bold / semibold]

## Rules
- NEVER truncate — full title always visible
- Max 3 lines mobile, 2 lines desktop
- Dynamic variant update on selection change
- No marketing taglines in title element

## Badge Positioning
- **Applicable badges:** [list or "none"]
- **Position:** [above title / inline-left]
- **Style:** [small pill, muted color]
- **Must not cause layout shift**

## Forbidden
- Truncation with "..." or ellipsis
- Title in ALL CAPS (reduces readability)
- Marketing claims mixed into title
- Font size below 18px on mobile
```

---

## Quality Criteria
- Full product title extracted and displayed without truncation
- Typography specs are specific (exact px values, not "large")
- Dynamic variant update behavior specified
- Badge positioning documented (even if "none applicable")
- Mobile and desktop specifications both present
- Forbidden patterns listed
- File exists after execution

## Failure Modes
- Product title not in brief → HALT, cannot proceed without product identity
- Title exceeds 3 lines on mobile at specified font size → Reduce font size to minimum 18px, allow 4 lines if needed. NEVER truncate.
- Multiple variants create extremely long title → Show base title + variant indicator separately
- input-verification.md not found → HALT, run 0.1 first
