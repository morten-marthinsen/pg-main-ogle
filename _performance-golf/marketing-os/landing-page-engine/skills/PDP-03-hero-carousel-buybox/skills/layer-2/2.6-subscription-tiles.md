# 2.6 — Subscription Tile Design

**Skill:** PDP-03-hero-carousel-buybox
**Position:** Layer 2, Microskill 6
**Purpose:** Design the subscription vs one-time purchase selection using selectable tiles (NEVER pre-selected subscription). Baymard: sneaky subscriptions are the top cause of cart abandonment and chargebacks. Honest, explicit tiles with visible savings convert better long-term because they build trust.
**Effort Level:** medium
**Dependencies:** 0.1 input-verification.md, 2.4 price-display.md
**Output:** `subscription-tiles.md`

---

## Input Schema

```yaml
input:
  input_verification: "[path to input-verification.md]"
  price_display: "[path to price-display.md]"
  has_subscription: "[Y/N]"
  subscription_discount: "[percentage or dollar amount]"
  subscription_frequency: "[monthly, bi-monthly, quarterly, etc.]"
```

---

## Execution Logic

**Step 1:** Check if product offers subscription. If no subscription → write "N/A — product does not offer subscription" file and exit.

**Step 2:** Design two selectable tiles side-by-side or stacked:

### Tile 1: One-Time Purchase (OTP)
- **Label:** "One-Time Purchase"
- **Price:** $XX.XX
- **Subtext:** None (this is the straightforward option)
- **Radio button:** Unselected by default or selected by default (see Step 4)

### Tile 2: Subscribe & Save
- **Label:** "Subscribe & Save [XX%]"
- **Price:** $XX.XX/[frequency]
- **Subtext:** "Cancel anytime" + frequency selector if applicable
- **Savings callout:** "Save $X.XX per [delivery/month]" — specific dollar amount, not just percentage
- **Radio button:** NOT pre-selected (Step 4)

**Step 3:** Specify tile visual design:
- **Layout:** Side-by-side on desktop (50/50), stacked on mobile (full-width each)
- **Style:** Bordered cards with radio button indicator (left side or top-left)
- **Selected state:** Highlighted border (brand color, 2px), light background tint
- **Unselected state:** Light gray border (1px), white background
- **Touch target:** Entire tile is tappable (not just the radio button)
- **Size:** Minimum 48px height per tile

**Step 4:** Default selection rules:
- **DEFAULT = OTP.** One-time purchase is pre-selected.
- **NEVER pre-select subscription.** User must actively choose to subscribe.
- **Rationale:** Pre-selecting subscription is a dark pattern that causes chargebacks, support tickets, and trust destruction. Baymard classifies this as a deceptive pattern.

**Step 5:** Specify savings display within Subscribe tile:
- Show savings in BOTH formats: "$X.XX off" AND "XX% savings"
- Show per-delivery price: "$XX.XX every [frequency]"
- Show comparison to OTP: "vs $YY.YY one-time"
- "Cancel anytime" text: Below price, in muted smaller text. This reduces subscription anxiety.

**Step 6:** If frequency selector is needed (customer chooses delivery interval):
- **Options:** "Every 30 days", "Every 60 days", "Every 90 days"
- **Default:** Most popular frequency pre-selected
- **Style:** Small dropdown OR inline chips within the subscription tile
- **Price updates:** If frequency changes affect price, update immediately

**Step 7:** Specify interaction behavior:
- Selecting OTP tile → price display (2.4) updates to OTP price
- Selecting S&S tile → price display (2.4) updates to subscription price + savings
- Tile selection persists through quantity changes
- Tile selection persists through variant changes (subscription discount applies to all variants equally unless brief specifies otherwise)

**Step 8:** Write `subscription-tiles.md`.

## Quality Criteria
- Two tiles: OTP and Subscribe & Save
- OTP pre-selected by default (NEVER subscription)
- Savings shown in both dollar AND percentage within S&S tile
- "Cancel anytime" visible in S&S tile
- Entire tile is tappable (not just radio button)
- Mobile layout specified (stacked full-width)
- Frequency selector included if product offers multiple delivery intervals
- File exists after execution

## Failure Modes
- Subscription pre-selected → HALT. Redesign with OTP as default. Non-negotiable.
- No subscription offered → Write N/A file, exit cleanly
- Savings only shown as percentage → Add dollar amount calculation
- "Cancel anytime" missing from S&S tile → HALT. Add it. Users need this reassurance.
- Subscription tile made visually dominant to push subscriptions → Redesign. Both tiles must be visually equal.
- input-verification.md not found → HALT, run 0.1 first
