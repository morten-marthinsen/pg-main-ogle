# 2.7 — Quantity Stepper Spec

**Skill:** PDP-03-hero-carousel-buybox
**Position:** Layer 2, Microskill 7
**Purpose:** Specify the quantity selector using a stepper (+/-) control (NEVER a dropdown). Baymard: dropdown quantity selectors hide options and require 3+ taps. Steppers are faster (single tap to increment), more accessible, and reduce selection friction.
**Effort Level:** low
**Dependencies:** 0.1 input-verification.md
**Output:** `quantity-stepper.md`

---

## Input Schema

```yaml
input:
  input_verification: "[path to input-verification.md]"
  max_quantity: "[max purchasable quantity, or 'no limit']"
  bundle_savings: "[Y/N — if Y, savings at higher quantities]"
```

---

## Execution Logic

**Step 1:** Determine quantity constraints:
- **Default value:** 1
- **Minimum:** 1 (minus button disabled at 1)
- **Maximum:** From brief (typically 5-10 for consumables, higher for small items). If not specified, default to 10.

**Step 2:** Design stepper control:
- **Layout:** Horizontal: [–] [number] [+]
- **Button size:** 44x44px minimum (Baymard mobile touch target)
- **Number display:** Centered between buttons, 16-18px, bold
- **Button style:** Circle or rounded square, outlined border, "−" and "+" symbols centered
- **Container:** Optional light border around entire stepper group

**Step 3:** Define stepper states:
- **At minimum (1):** Minus button disabled (grayed out, not tappable)
- **At maximum:** Plus button disabled (grayed out, not tappable)
- **Normal:** Both buttons active, standard styling
- **Tap feedback:** Brief color change or scale pulse on tap (not just visual, haptic if native)

**Step 4:** Specify bundle/quantity savings callout (if applicable):
- If buying 3+ triggers a discount: Show savings callout below stepper
- **Format:** "Buy 3, Save 10%" or "Buy 3+ for $X.XX each"
- **Style:** Small text, accent color, appears when quantity ≥ threshold
- **Dynamic:** Savings text updates as quantity changes

**Step 5:** Specify label:
- **Text:** "Quantity" or "Qty"
- **Position:** Left of stepper (inline) or above stepper
- **Typography:** 14px, medium weight

**Step 6:** Specify interaction behavior:
- Single tap on +/- changes quantity by 1
- Long-press on +/- does NOT auto-increment (prevents accidental high quantities)
- Total price in price display (2.4) optionally updates to show quantity × unit price
- Quantity persists through variant changes (unless variant change resets to 1 — specify which)

**Step 7:** Write `quantity-stepper.md`.

## Quality Criteria
- Stepper used (NEVER dropdown) — non-negotiable
- 44x44px minimum touch targets on +/- buttons
- Default quantity = 1
- Minus disabled at 1, plus disabled at max
- Label present ("Quantity" or "Qty")
- No auto-increment on long-press
- Bundle savings callout specified if product has quantity discounts
- File exists after execution

## Failure Modes
- Dropdown specified → HALT. Redesign with stepper. Non-negotiable per Baymard.
- Touch targets below 44px → HALT. Increase to minimum.
- No max quantity specified → Default to 10.
- Bundle savings exist but not shown → Add quantity savings callout.
- input-verification.md not found → HALT, run 0.1 first
