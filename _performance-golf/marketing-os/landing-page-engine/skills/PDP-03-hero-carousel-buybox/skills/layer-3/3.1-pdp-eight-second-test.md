# 3.1 — PDP 8-Second Test

**Skill:** PDP-03-hero-carousel-buybox
**Position:** Layer 3, Microskill 1
**Purpose:** Simulate a first-time PDP visitor and verify the above-fold answers three critical questions within 8 seconds without any scrolling. If ANY question fails, the above-fold must be redesigned. This is the PDP equivalent of the LP-03 8-second test, adapted for PDP architecture (carousel + buy box, not headline + deck).
**Effort Level:** high
**Dependencies:** 1.2 carousel-architecture.md, 2.1-2.10 (all buy box component files)
**Output:** `eight-second-test.md`

## Execution Logic

**Step 1:** Set the simulation context:
- **User:** First-time visitor, arrived from a product ad or search result
- **Device:** Mobile (worst case — smallest viewport)
- **Time budget:** 8 seconds, no scrolling permitted
- **Viewport:** Only carousel first thumbnail + buy box visible (above the fold)

**Step 2:** Evaluate Question 1: "What is this product?"

Test: Can the user identify the product category, name, and primary function from:
- Carousel Slide 1 (Outcome) — product shot + outcome headline
- Product title (full, not truncated)
- Short description bullets (first 2-3 visible)

Scoring:
- **PASS:** Product identity clear within 3 seconds. User knows what category, what brand, and what it does.
- **FAIL:** Product name truncated, no outcome headline on Slide 1, short description is vague, or product shot is unclear.

**Step 3:** Evaluate Question 2: "Is this for me?"

Test: Can the user self-select (this solves MY problem) from:
- Carousel Slide 1 outcome headline (speaks to their desired result)
- Carousel Slide 2 thumbnail visible in thumbnail strip (problem signs — teases relevance)
- Short description bullets (specific benefits matching their needs)

Scoring:
- **PASS:** A visitor with the target problem can see themselves in the above-fold content. The outcome headline or benefit bullets contain language that matches their situation.
- **FAIL:** Outcome headline is generic ("Premium Supplement"), benefit bullets are feature-forward ("Contains Vitamin D"), or no self-selection signal visible.

**Step 4:** Evaluate Question 3: "How much does it cost and how do I buy it?"

Test: Can the user see the price and path to purchase from:
- Price display (current price + savings visible)
- ATC button (visible without scrolling)
- Variant selector (if variants exist, selector is visible)

Scoring:
- **PASS:** Price is visible with savings, ATC button is fully visible in viewport, and variant selector (if needed) is above the fold.
- **FAIL:** Price pushed below fold, ATC button requires scrolling, or variant selector is hidden.

**Step 5:** Calculate overall test result:
- **ALL 3 PASS:** Above-fold architecture is sound. Proceed to Layer 4.
- **ANY FAIL:** HALT. Document which question(s) failed and why. Redesign required before proceeding.

**Step 6:** Write `eight-second-test.md`.

## Quality Criteria
- All three questions evaluated with specific evidence
- Mobile viewport used as the test case (not desktop)
- Each question has a clear PASS/FAIL determination
- Failed questions include specific redesign recommendations
- Overall result is binary: ALL PASS or HALT
- No partial passes or conditional passes
- File exists after execution

## Failure Modes
- Question 1 fails (product unclear) → Revise Slide 1 headline and/or product title display
- Question 2 fails (no self-selection) → Revise Slide 1 outcome headline or short description bullets to include audience-specific language
- Question 3 fails (price/ATC below fold) → Reduce buy box component spacing, ensure price and ATC are visible on mobile without scrolling
- Multiple questions fail → Major above-fold redesign required — go back to Layer 1/2 and revise
- Component files missing → HALT, run Layer 2 first
