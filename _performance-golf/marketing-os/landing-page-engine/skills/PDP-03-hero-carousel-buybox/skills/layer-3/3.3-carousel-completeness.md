# 3.3 — Carousel Completeness Check

**Skill:** PDP-03-hero-carousel-buybox
**Position:** Layer 3, Microskill 3
**Purpose:** Verify all 10 NLS story positions are filled in the carousel architecture. Each position has a specific persuasion function. Missing positions = missing persuasion steps. This is a structural validation, not a quality judgment.
**Effort Level:** low
**Dependencies:** 1.2 carousel-architecture.md
**Output:** `carousel-check.md`

---

## Input Schema

```yaml
input:
  carousel_architecture: "[path to carousel-architecture.md]"
```

---

## Execution Logic

**Step 1:** Read `carousel-architecture.md`.

**Step 2:** Verify each of the 10 story positions exists:

| Position | Story Position | Check |
|----------|---------------|-------|
| 1 | Outcome | [Present Y/N] |
| 2 | Problem Signs | [Present Y/N] |
| 3 | Ingredients | [Present Y/N] |
| 4 | Enjoyment | [Present Y/N] |
| 5 | How It Works | [Present Y/N] |
| 6 | What to Expect | [Present Y/N] |
| 7 | How to Use | [Present Y/N] |
| 8 | Credibility | [Present Y/N] |
| 9 | Comparison | [Present Y/N] |
| 10 | Testimonials | [Present Y/N] |

**Step 3:** For each present slide, verify it has all three required elements:
- Purpose statement (not empty)
- Copy direction (specific, not vague)
- Visual direction (specific, not vague)

**Step 4:** Flag any slides with vague directions:
- "Add benefit image" → VAGUE
- "Show product" → VAGUE
- "Include testimonial" → VAGUE
- Specific directions include exact copy text, specific imagery descriptions, and layout details

**Step 5:** Check for position mismatches:
- Is the Outcome slide actually about outcomes (not ingredients)?
- Is the Comparison slide an actual comparison table (not just "we're better")?
- Is each slide doing the job of its assigned position?

**Step 6:** Calculate completeness:
- 10/10 positions filled with complete elements → PASS
- Any position missing or incomplete → FAIL with specific gaps listed

**Step 7:** Write `carousel-check.md`.

## Quality Criteria
- All 10 positions checked for presence
- Each position verified for purpose, copy direction, and visual direction
- Vague directions flagged with specific fix recommendations
- Position mismatches identified
- Result is binary: PASS (all 10 complete) or FAIL (any incomplete)
- File exists after execution

## Failure Modes
- Fewer than 10 slides → FAIL. List missing positions.
- Slide present but missing copy or visual direction → FAIL for that position.
- All slides present but directions are vague → FAIL. Vague = incomplete.
- carousel-architecture.md not found → HALT, run 1.2 first
