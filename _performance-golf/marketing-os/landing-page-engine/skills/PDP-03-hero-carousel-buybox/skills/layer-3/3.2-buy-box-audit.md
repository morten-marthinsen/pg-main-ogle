# 3.2 — Buy Box Audit

**Skill:** PDP-03-hero-carousel-buybox
**Position:** Layer 3, Microskill 2
**Purpose:** Score the buy box architecture against the 10-point Baymard-validated checklist from the AGENT.md. Each point is binary (pass/fail). Total score must be >= 7.0/10 to proceed to Layer 4. Below 7.0 requires revision.
**Effort Level:** medium
**Dependencies:** 2.1-2.10 (all buy box component files)
**Output:** `buy-box-audit.md`

## Execution Logic

**Step 1:** Read all 10 buy box component files.

**Step 2:** Score each of the 10 audit points:

### Title & Identity (2 points)

**Point 1: Full product title displayed (never truncated)?**
- Read `title-rules.md`
- PASS (1): Title explicitly specified as never-truncated, up to 3 lines allowed
- FAIL (0): Any truncation, ellipsis, or character limit specified

**Point 2: Rating strip is an anchor link to reviews section?**
- Read `rating-strip.md`
- PASS (1): Entire strip is clickable anchor to #reviews with smooth-scroll
- FAIL (0): Stars are static, only text is linked, or no link at all

### Price & Value (2 points)

**Point 3: Original price shown struck through with savings in both % and $ amount?**
- Read `price-display.md`
- PASS (1): Compare-at price struck through AND savings shown in both formats
- FAIL (0): Only one savings format, no compare-at, or no savings display

**Point 4: Shipping cost or "Free Shipping" visible near price?**
- Read `price-display.md`
- PASS (1): Shipping info directly below or adjacent to price display
- FAIL (0): Shipping hidden, in footer only, or not specified

### Selection UX (3 points)

**Point 5: Variant selector uses exposed chips (not dropdowns)?**
- Read `variant-selector.md`
- PASS (1): Chips explicitly specified, dropdown explicitly forbidden
- FAIL (0): Dropdown used, or selector type not specified

**Point 6: Subscription vs OTP uses selectable tiles with explicit savings?**
- Read `subscription-tiles.md`
- PASS (1): Two tiles with OTP default-selected, subscription NOT pre-selected, savings in $ and %
- FAIL (0): Pre-selected subscription, or savings not explicit, or no tile design
- N/A: If product has no subscription → auto-PASS (score 1)

**Point 7: Quantity selector uses stepper (not dropdown), 44px+ touch targets?**
- Read `quantity-stepper.md`
- PASS (1): Stepper (+/-) specified with 44px+ touch targets
- FAIL (0): Dropdown used, or touch targets below 44px

### Conversion (3 points)

**Point 8: ATC button is full-width with standard "Add to Cart" label?**
- Read `atc-button.md`
- PASS (1): Full-width (100%), standard "Add to Cart" label
- FAIL (0): Not full-width, or creative label used

**Point 9: Post-click behavior is slide-out mini-cart (not cart redirect)?**
- Read `atc-button.md`
- PASS (1): Slide-out mini-cart specified, cart redirect explicitly forbidden
- FAIL (0): Cart redirect, or post-click behavior not specified

**Point 10: Micro-trust signals (3-4) visible directly below ATC button?**
- Read `micro-trust.md`
- PASS (1): 3-4 specific, verified signals below ATC/express payments
- FAIL (0): Fewer than 3, more than 6, vague signals, or missing entirely

**Step 3:** Calculate total score:
- Sum all points (0 or 1 each)
- Maximum: 10/10
- Threshold: >= 7.0 to proceed

**Step 4:** Determine gate status:
- 10/10: Exceptional — proceed
- 8-9/10: Good — proceed with revision notes
- 7/10: Minimum acceptable — proceed with caution and notes
- 6/10: Needs revision — revise failing points before Layer 4
- Below 6: Structural rethink — multiple components need redesign

**Step 5:** Write `buy-box-audit.md`.

## Quality Criteria
- All 10 points evaluated with specific evidence from component files
- Each point is binary PASS (1) or FAIL (0) — no partial scores
- Total score calculated correctly
- Failed points have specific required fixes
- Gate decision follows the scoring interpretation (>= 7 proceed, < 7 revise/halt)
- File exists after execution

## Failure Modes
- Score below 7 → Do NOT proceed. Revise failing components and re-run audit.
- Component file missing → Score that point as FAIL, note "file not found"
- Score of 6 with "it's close enough" rationalization → HALT. 7 is the minimum. No exceptions.
- N/A subscription inflates score dishonestly → N/A counts as PASS (1 point) — this is correct behavior since the audit assumes subscription should either work right or not apply
