# 3.4 — Anti-Slop Check

**Skill:** PDP-03-hero-carousel-buybox
**Position:** Layer 3, Microskill 4
**Purpose:** Scan all copy directions in the carousel and buy box component files for generic AI language ("slop"). Copy directions must be specific enough for a designer/developer to execute. Generic AI telltales signal the AI synthesized rather than specified.
**Effort Level:** medium
**Dependencies:** 1.2 carousel-architecture.md, 2.1-2.10 (all buy box files)
**Output:** `anti-slop-check.md`

---

## Input Schema

```yaml
input:
  carousel_architecture: "[path to carousel-architecture.md]"
  all_layer_2_files: "[paths to all 2.x files]"
  short_description: "[path to short-description.md]"
```

---

## Execution Logic

**Step 1:** Define the forbidden word/phrase list:

### Category 1: AI Telltale Words
- revolutionary
- game-changing
- cutting-edge
- state-of-the-art
- transform/transformative
- unlock/unlocking
- journey
- leverage/leveraging
- seamless/seamlessly
- elevate/elevating
- empower/empowering
- harness/harnessing
- robust
- holistic

### Category 2: Vague Directions
- "add benefit text"
- "include relevant image"
- "show product"
- "add lifestyle imagery"
- "insert testimonial"
- "use appropriate colors"
- "compelling headline"
- "engaging content"
- "eye-catching design"

### Category 3: Meaningless Claims
- "high quality"
- "premium formula"
- "advanced technology"
- "natural ingredients" (without specifics)
- "scientifically formulated" (without citing studies)
- "trusted by thousands" (without specific number)
- "best in class"

### Category 4: Cliche Patterns
- "Say goodbye to [problem]"
- "Welcome to the future of"
- "Unlike anything you've tried before"
- "What if there was a better way"
- "It's time to take control"

**Step 2:** Scan `carousel-architecture.md` — check every copy direction and visual direction field in all 10 slides against all four forbidden categories.

**Step 3:** Scan `short-description.md` — check all benefit bullets against forbidden categories.

**Step 4:** Scan all Layer 2 buy box files — check any copy-related specifications (button labels, signal text, pricing copy, subscription tile labels).

**Step 5:** For each violation found:
- Record the file, location (which slide or component), the offending text, and the category
- Suggest a specific replacement that is product-relevant and concrete

**Step 6:** Calculate result:
- **0 violations:** PASS
- **Any violations:** FAIL — all violations must be fixed before proceeding

**Step 7:** Write `anti-slop-check.md`.

## Quality Criteria
- All carousel slides scanned (all 10)
- All buy box component files scanned
- Short description bullets scanned
- Each violation has a specific replacement suggestion (not just "fix this")
- Replacement suggestions are product-specific and concrete
- Zero violations required to pass (no threshold — it's 0 or fail)
- File exists after execution

## Failure Modes
- Violations found → FAIL. Fix all violations in source files, then re-run check.
- Source files not found → HALT. Run earlier layers first.
- Replacement suggestion is also sloppy → Rewrite the replacement. The fix must be genuinely specific.
- "It's only 1 violation, we can proceed" → No. Zero violations is the standard. Fix it.
