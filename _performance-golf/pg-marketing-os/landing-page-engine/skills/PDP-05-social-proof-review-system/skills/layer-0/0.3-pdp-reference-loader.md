# 0.3 — PDP Reference Loader

**Skill:** PDP-05-social-proof-review-system
**Position:** Layer 0, Microskill 3
**Purpose:** Load PDP-BEST-PRACTICES-REFERENCE.md sections 7-9 (Reviews, UGC, Q&A) to ground all Layer 1-2 architecture decisions in Baymard-validated patterns. This is the authoritative source for review system UX requirements.
**Effort Level:** low
**Dependencies:** PDP-BEST-PRACTICES-REFERENCE.md (landing page engine reference file)
**Output:** `pdp-reference-load.md`

---

## Input Schema

```yaml
input:
  reference_path: "[landing-page-engine root]/reference/PDP-BEST-PRACTICES-REFERENCE.md"
  target_sections: [7, 8, 9]
```

---

## Execution Logic

**Step 1:** Locate PDP-BEST-PRACTICES-REFERENCE.md in the landing page engine reference directory.

**Step 2:** Extract Section 7 — Review System Best Practices:
- Histogram requirements (tappable bars, star distribution display)
- Review card template (required fields, layout order)
- Loading mechanism (Load More specification)
- Sort options (Most Relevant default)
- Verified Buyer badge rules
- Time Owned display rules
- Merchant response requirements (3-star and below)
- "Write a Review" de-emphasis

**Step 3:** Extract Section 8 — UGC Best Practices:
- Photo strip position (above text reviews)
- Full-screen gallery overlay behavior
- Video carousel specifications (muted autoplay, avatar diversity)
- UGC diversity requirements
- Fallback behavior (omit if no UGC, never fake)

**Step 4:** Extract Section 9 — Q&A Best Practices:
- Search-first pattern (keyword field at top)
- Brand vs community answer distinction
- Zero-state handling (hide if empty)
- "Ask a Question" de-emphasis

**Step 5:** If reference file not found, load requirements from PDP-05-AGENT.md Baymard Requirements section as fallback (the AGENT.md contains all critical requirements).
