# 0.2 — Research Loader

**Skill:** PDP-05-social-proof-review-system
**Position:** Layer 0, Microskill 2
**Purpose:** Load verbatim audience quotes and keyword data from LP-01 conversion intelligence. These quotes seed the reviewer persona builder and ensure generated reviews use real audience language, not AI language.
**Effort Level:** low
**Dependencies:** LP-01 output (conversion-intelligence.json or equivalent), 0.1 output (input-verification.md)
**Output:** `research-load.md`

---

## Input Schema

```yaml
input:
  project_folder: "[path to project output folder]"
  intelligence_path: "[project_folder]/lp-01-conversion-intelligence/"
  scope: "[from 0.1 — full | standard | minimal | generation_only]"
```

---

## Execution Logic

**Step 1:** Locate LP-01 conversion intelligence outputs. Look for:
- `conversion-intelligence.json` (primary)
- Any research files with verbatim quotes (audience voice data)
- Keyword/topic frequency data from existing reviews (if LP-01 analyzed competitors)

**Step 2:** Extract verbatim audience quotes. Prioritize:
- Product-specific complaints ("The taste was awful" > "I didn't like it")
- Specific outcome statements ("Lost 12 lbs in 6 weeks" > "Helped me lose weight")
- Purchase driver language ("I bought this because my doctor recommended it")
- Objection language ("I was worried about the price but...")
- Competitor comparison language ("Better than [Brand X] because...")

**Step 3:** Extract keyword frequency data for filter chip seeding:
- What topics do reviewers mention most? (taste, results, shipping, texture, value, ingredients, side effects)
- What product attributes generate the most discussion?
- What concerns appear repeatedly?

**Step 4:** Extract audience demographic patterns for reviewer persona seeding:
- Age range distribution
- Gender split
- Geographic patterns
- Purchase motivation patterns (gift, self-use, replacement, first-time)
- Usage duration patterns (new user, 3-month, 6-month, 1-year+)

**Step 5:** If no LP-01 data available (standalone mode), document as empty and note that Layer 2 will generate from product data alone.

## Quality Criteria
- All available verbatim quotes extracted and categorized
- Keyword frequency data organized for filter chip seeding
- Demographic patterns documented for persona building
- Data availability honestly assessed (not inflated)
- Quotes preserved EXACTLY as found (no cleaning, no editing)

## Failure Modes
- LP-01 not run -> WARNING (not halt): "No conversion intelligence available. Set data availability to 'none.' Layer 2 will generate from product data and vertical defaults."
- Quotes available but no keyword data -> Proceed with quotes. Note keyword gap — Layer 2 keyword topic selector will derive from product attributes instead.
- Rich data available -> Flag as advantage: "Rich research data available. Generated reviews should incorporate verbatim language patterns."
