# 0.1 — Brief Loader

**Skill:** PDP-05-social-proof-review-system
**Position:** Layer 0, Microskill 1
**Purpose:** Load page-brief.json from LP-00. Confirm page type is type_b or hybrid. Extract product data, audience profile, proof inventory, and vertical classification needed for review system architecture.
**Effort Level:** low
**Dependencies:** LP-00 output (page-brief.json), PDP-03 output (pdp-above-fold-blueprint.json)
**Output:** `input-verification.md`

---

## Input Schema

```yaml
input:
  project_folder: "[path to project output folder]"
  page_brief_path: "[project_folder]/lp-00-brief-classifier/page-brief.json"
  pdp_above_fold_path: "[project_folder]/pdp-03-hero-carousel-buybox/pdp-above-fold-blueprint.json"
```

---

## Execution Logic

**Step 1:** Locate and load `page-brief.json`. Extract:
- `page_type.classification` (type_a | type_b | hybrid)
- `operating_mode` (downstream | standalone)
- `product.name`, `product.category`, `product.key_attributes[]`
- `audience.primary_segment`, `audience.pain_points[]`, `audience.purchase_drivers[]`
- `proof_inventory` block (testimonials_count, has_reviews, review_count, star_rating, has_ugc_photos, has_ugc_video)
- `traffic.temperature` (cold | warm | hot)
- `vertical.primary`

**Step 2:** Validate page type. If `type_a` -> HALT: "PDP-05 is for Type B / Hybrid pages only. Use LP-05 + LP-10 for Type A."

**Step 3:** Load `pdp-above-fold-blueprint.json` if available. Extract:
- `buy_box.rating_strip` (aggregate rating, review count — this anchors the review section)
- `buy_box.rating_strip.anchor_link_target` (the section ID the rating strip links to)

**Step 4:** Determine review system scope from proof inventory:
- If `review_count >= 100`: Full system (histogram, filters, UGC, Q&A)
- If `review_count 10-99`: Standard system (histogram, filters, featured reviews — UGC conditional)
- If `review_count < 10`: Minimal system (featured reviews, no histogram, no filters — content generation focus)
- If `review_count = 0` or `not_available`: Generation-only mode (generate sample reviews as content direction for client)

**Step 5:** Document all extracted data and scope determination.

## Quality Criteria
- page-brief.json loaded and page type confirmed as type_b or hybrid
- Product data extracted with sufficient detail for review system design
- Proof inventory summarized with review-specific fields
- Review system scope determined based on existing review count
- PDP-03 anchor link documented (connects buy box rating strip to review section)

## Failure Modes
- `page-brief.json` not found -> HALT: "PDP-05 requires LP-00 output. Run LP-00 first."
- Page type = type_a -> HALT: "PDP-05 is for Type B / Hybrid only. Use LP-05 + LP-10."
- `pdp-above-fold-blueprint.json` not found -> WARNING (not halt): "PDP-03 not yet run. Rating strip anchor link unknown. Proceed but note dependency."
- Proof inventory empty -> Set scope to `generation_only` and document.
