# 4.1 — Social Proof Package Compiler

**Skill:** PDP-05-social-proof-review-system
**Position:** Layer 4, Microskill 1
**Purpose:** Assemble all upstream outputs — review system architecture, UGC carousel spec, Q&A section, featured reviews, histogram, filter chips, card template, audit scores — into a single validated JSON file (`pdp-social-proof-system.json`) that downstream skills can consume without ambiguity. This is the structured handoff — every field populated, every score recorded, every Baymard requirement verified present. If the JSON is incomplete, downstream skills will build broken pages.
**Effort Level:** medium
**Dependencies:** ALL Layer 0-3 outputs. Specifically: `input-verification.md` (0.1), `research-loaded.md` (0.2), `pdp-reference-loaded.md` (0.3), `review-system-architecture.md` (1.1), `ugc-video-carousel-architecture.md` (1.2), `review-histogram-spec.md` (1.3), `keyword-filter-spec.md` (1.4), `featured-reviews.md` (2.1), `keyword-topic-selection.md` (2.2), `reviewer-personas.md` (2.3), `qa-section-architecture.md` (2.4), `review-card-template.md` (2.5), `volume-audit.md` (3.1), `specificity-check.md` (3.2), `badge-verification.md` (3.3), `anti-slop-check.md` (3.4)
**Output:** `pdp-social-proof-system.json`

## Execution Logic

**Step 1:** Verify GATE_3 status. All four Layer 3 audits must show PASS:
- Volume audit (3.1): score >= 7.0/10 — PASS
- Specificity check (3.2): score >= 7.5/10, all featured reviews >= 7.0 — PASS
- Badge verification (3.3): score >= 8.0/10 — PASS
- Anti-slop check (3.4): score >= 8.0/10, zero hard-fail telltales — PASS

If ANY Layer 3 audit shows FAIL or any score below threshold: HALT. Do not compile. Return to failed audit step.

## Quality Criteria
- GATE_3 verified PASS before compilation begins (all 4 Layer 3 audits passed)
- JSON contains all 7 top-level sections: review_system, ugc_carousel, qa_section, featured_reviews, reviewer_personas, social_proof_audit, baymard_compliance
- review_system.histogram.bars_tappable = true (Baymard requirement)
- review_system.loading.method = "load_more" (NOT pagination, NOT infinite scroll)
- review_system.card_template has all 7 sections (stars_and_headline, reviewer_line, reviewer_attributes, pros_cons, body_text, actions_row, merchant_response)
- review_system.card_template.required_fields includes: stars, headline, reviewer_name, verified_buyer, time_owned, pros, cons, body, helpful_votes
- Featured reviews use anti-slop corrected versions where corrections were applied in 3.4
- All audit scores present and above thresholds (volume >= 7.0, specificity >= 7.5, badge >= 8.0, anti-slop >= 8.0)
- baymard_compliance.all_10_requirements_met = true
- 13-point validation check passes before write (no empty strings, no nulls, no placeholders, counts match, scores present, JSON valid)
- File parses as valid JSON

## Failure Modes
- GATE_3 not verified before compilation -> HALT. Must confirm all 4 Layer 3 audits passed.
- Any empty string field in final JSON -> HALT. Trace to upstream file and populate.
- Any null field in final JSON -> HALT. Trace to upstream file and populate.
- Any placeholder/TBD value remaining -> HALT. Resolve before writing.
- loading.method is anything other than "load_more" -> HALT. Fix to "load_more".
- histogram.bars_tappable is false or missing -> HALT. Fix to true.
- card_template missing any of the 7 sections -> HALT. Add missing section.
- Featured review count outside 3-5 range -> HALT. Verify against 2.1 output.
- Star distribution percentages don't sum to 100% (±1%) -> HALT. Fix distribution.
- Composite audit score below 7.0 -> HALT. Return to failed audit step.
- baymard_compliance has any false value -> HALT. Identify and fix failed requirement.
- Featured review body text is the pre-correction version when anti-slop corrections exist -> HALT. Use corrected version from 3.4.
- JSON syntax error -> HALT. Fix syntax before writing.
