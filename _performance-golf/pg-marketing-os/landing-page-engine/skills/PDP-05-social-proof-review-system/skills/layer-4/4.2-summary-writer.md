# 4.2 — Summary Writer

**Skill:** PDP-05-social-proof-review-system
**Position:** Layer 4, Microskill 2
**Purpose:** Write a human-readable summary (`PDP-SOCIAL-PROOF-SUMMARY.md`) that documents the complete social proof review system architecture, featured review content, audit results, and downstream handoff information. This file is what a human reads to understand the review system without parsing JSON. Every number in this summary must match the compiled JSON from 4.1 exactly — any discrepancy means one of the two files is wrong.
**Effort Level:** low
**Dependencies:** `pdp-social-proof-system.json` (4.1), all Layer 0-3 outputs
**Output:** `PDP-SOCIAL-PROOF-SUMMARY.md`

---

## Input Schema

```yaml
input:
  compiled_json: "[from 4.1 — pdp-social-proof-system.json, fully validated]"
  input_verification: "[from 0.1 — product name, category, scope, page type]"

  badge_verification: "[from 3.3 — badge verification score]"
```

## Execution Logic

**Step 1:** Load the compiled JSON from `pdp-social-proof-system.json` (4.1). Verify the file exists and is valid JSON before proceeding.

**Step 2:** Write the summary following the EXACT template structure required by ANTI-DEGRADATION.md. Every section listed below is MANDATORY — omitting a section means the summary is incomplete.

**Step 3:** Verify that every number, count, score, and label in the summary matches the JSON source exactly:
- Featured review count matches JSON `featured_reviews` array length
- Star distribution percentages match JSON `review_system.histogram.star_distribution`
- Average rating matches JSON `review_system.average_rating`
- Filter chip labels match JSON `review_system.filter_chips.labels`
- All audit scores match JSON `social_proof_audit` section
- Q&A question count matches JSON `qa_section.total_questions`
- UGC counts match JSON `ugc_carousel` section

**Step 4:** Write the file to disk.

## Quality Criteria
- Every section in the template above is present — no section omitted
- Every number matches `pdp-social-proof-system.json` exactly (review counts, scores, percentages, chip labels)
- All 4 audit scores listed with thresholds and PASS/FAIL status
- All 10 Baymard compliance checks listed
- Featured review summaries include specificity score, slop score, and badge verification status from Layer 3 audits
- Time Owned distribution table populated
- Q&A answer type breakdown populated
- Downstream handoff table confirms which skills can consume this output
- Anti-slop corrections listed (or confirmed as "None")
- Summary reads as a complete, self-contained document — a human can understand the entire review system without reading any other file

## Failure Modes
- `pdp-social-proof-system.json` (4.1) does not exist -> HALT. Cannot write summary without compiled JSON.
- Any number in summary does not match JSON source -> HALT. Verify and correct discrepancy.
- Missing section in summary template -> HALT. All sections are mandatory.
- Audit score listed without threshold or status -> HALT. Every score needs its threshold and PASS/FAIL.
- Baymard checklist has unchecked items -> HALT. All 10 must be checked (if any failed, compilation should not have proceeded past 4.1).
- Featured review body preview uses pre-correction text when anti-slop corrections were applied -> HALT. Use corrected version.
- Downstream handoff table missing or incomplete -> HALT. All 4 downstream skills must be listed.
