# 1.1 — Traffic Source Analyzer

**Skill:** LP-03-above-fold-architecture
**Position:** Layer 1, Microskill 1
**Purpose:** Identify the primary traffic temperature (cold/warm/hot) to constrain headline type selection and above-fold pattern choice. Traffic temperature is the single strongest determinant of what the above-fold section must accomplish.
**Effort Level:** medium
**Dependencies:** 0.1 input-verification.md (LP-01 traffic data or user brief)
**Output:** `traffic-analysis.md`

---

## Input Schema

```yaml
input:
  input_verification: "[path to input-verification.md]"
  lp01_traffic_data: "[extracted traffic sources from LP-01, or 'not available']"
  user_brief_traffic: "[any traffic source info from LP-00 brief, or 'not specified']"
```

---

## Execution Logic

**Step 1:** Read `input-verification.md`. Check if LP-01 traffic source data is available.

**Step 2:** If LP-01 data IS available, extract:
- List of traffic sources (Facebook Ads, Google Search, Email, Organic, YouTube, etc.)
- Temperature classification per source
- Estimated volume split (which source drives majority of traffic)

**Step 3:** If LP-01 data is NOT available, infer from LP-00 brief:
- Page type signals: Type A long-form → likely cold traffic (education-first exists for a reason)
- Vertical signals: Supplement PDP → mixed (branded search = warm, social ads = cold)
- Operating mode: Downstream with CopywritingEngine = likely cold (DR funnel)
- If all signals are ambiguous, default to COLD (safest — over-educating warm traffic is a smaller error than under-educating cold traffic)

**Step 4:** Classify primary traffic temperature:

```
COLD: No prior relationship. Visitor has never heard of brand/product.
  → Sources: Facebook/Meta Ads, Display, Cold email, YouTube pre-roll
  → Above-fold job: Interrupt → Hook → Promise
  → Headline constraint: Must be self-contained (no assumed knowledge)

WARM: Some prior exposure. Visitor knows brand or problem.
  → Sources: Retargeting, Email list, Blog referral, Podcast mention
  → Above-fold job: Remind → Deepen → Convert
  → Headline constraint: Can reference shared context

HOT: Active buyer intent. Visitor is looking for THIS solution.
  → Sources: Google Search (branded), Direct, Affiliate referral, "Buy now" email
  → Above-fold job: Confirm → Reassure → Convert
  → Headline constraint: Can lead with offer/price/specifics
```

**Step 5:** If multiple traffic sources exist, identify the PRIMARY source (highest volume) and note SECONDARY sources. The above-fold must be designed for the primary source but not alienate secondary sources.

**Step 6:** Document headline approach constraints based on temperature:
- Cold → T2 (Curiosity), T3 (Problem-First), T4 (Warning), T6 (How-To) preferred
- Warm → T1 (Direct Benefit), T5 (Question), T6 (How-To) preferred
- Hot → T1 (Direct Benefit), T7 (Number-Specific) preferred

**Step 7:** Write `traffic-analysis.md`.

---

## Quality Criteria
- Temperature classified as exactly one of COLD, WARM, HOT
- Classification justified with evidence (LP-01 data or inference chain)
- Headline type constraints align with AGENT.md taxonomy
- Default-to-cold logic applied when data is ambiguous (not default-to-warm)
- File exists after execution

## Failure Modes
- No traffic data AND no brief signals → Default to COLD, document reasoning clearly
- Multiple sources with conflicting temperatures → Use highest-volume source as primary, note conflict
- input-verification.md not found → HALT, run Layer 0 first
- User specifies traffic source that contradicts LP-01 data → User input overrides LP-01
