# 4.2 — Summary Writer

**Skill:** LP-03-above-fold-architecture
**Position:** Layer 4, Microskill 2
**Purpose:** Write a human-readable summary of the above-fold architecture decisions for review. This file is what the human reviewer reads to approve/reject the blueprint before LP-07 executes. It must be scannable in under 2 minutes.
**Effort Level:** low
**Dependencies:** 4.1 above-fold-blueprint.json, 3.1 eight-second-test.md, 3.2 above-fold-audit.md
**Output:** `ABOVE-FOLD-SUMMARY.md`

---

## Input Schema

```yaml
input:
  blueprint: "[path to above-fold-blueprint.json]"
  eight_second_test: "[path to eight-second-test.md]"
  above_fold_audit: "[path to above-fold-audit.md]"
  pattern_selection: "[path to pattern-selection.md]"
```

---

## Execution Logic

**Step 1:** Read `above-fold-blueprint.json`. This contains all structured decisions.

**Step 2:** Read `eight-second-test.md` and `above-fold-audit.md` for quality scores and any remediation notes.

**Step 3:** Write the summary in scannable format. The summary must be readable in under 2 minutes. Use headers, bullet points, and bold for key decisions. No prose paragraphs.

**Step 4:** Structure the summary in this order:
1. One-line verdict (the most important decision)
2. Key decisions table (5-6 rows, 2 columns)
3. 8-second test results (3 lines)
4. Audit score
5. Risks or open questions
6. Next skill in pipeline

**Step 5:** Highlight any decisions that are non-obvious or require human judgment:
- Headline type selection that diverges from specimen norms
- Trust signals that need verification
- CTA direction that depends on unconfirmed offer details
- Price display with placeholder values

**Step 6:** Write `ABOVE-FOLD-SUMMARY.md`.

## Quality Criteria
- Summary is scannable in under 2 minutes (no dense paragraphs)
- Key decisions table captures the 5-6 most important choices
- 8-second test results visible at a glance
- Risks and open questions explicitly listed (not hidden)
- Human review checklist includes actionable approval items
- Next skill identified for pipeline continuity
- File exists after execution

## Failure Modes
- Summary is longer than 1 page → Trim to essential decisions only
- Key decisions table missing entries → Verify all Layer 2 outputs were compiled
- Risks section says "None" when placeholder values exist → Re-scan blueprint for [PLACEHOLDER] or unconfirmed data
- blueprint.json not found → HALT, run 4.1 first
