# 4.1 — Blueprint Compiler

**Skill:** LP-03-above-fold-architecture
**Position:** Layer 4, Microskill 1
**Purpose:** Compile all Layer 2 outputs into the structured `above-fold-blueprint.json` that downstream skills (LP-07 Hero Section Writer, LP-04 Section Sequence Planner) consume. This is the primary deliverable of LP-03.
**Effort Level:** low
**Dependencies:** All Layer 2 outputs (2.1-2.6), 3.2 above-fold-audit.md (score >= 7), 3.3 anti-slop-check.md (PASS)
**Output:** `above-fold-blueprint.json`

---

## Input Schema

```yaml
input:
  headline_type: "[path to headline-type.md]"
  deck_direction: "[path to deck-direction.md]"
  hero_image_brief: "[path to hero-image-brief.md]"
  trust_architecture: "[path to trust-architecture.md]"

  traffic_analysis: "[path to traffic-analysis.md]"

  input_verification: "[path to input-verification.md]"
```

---

## Execution Logic

**Step 1:** Verify gates before compiling:
- Read `above-fold-audit.md` — confirm score >= 7/8. If score < 7, HALT. Do not compile a below-threshold blueprint.
- Read `anti-slop-check.md` — confirm result = PASS. If FAIL, HALT. Do not compile a blueprint with AI slop in the direction.

**Step 2:** Read all Layer 2 output files. Extract the structured data from each:

From `headline-type.md`:
- selected_type (T1-T7)
- selected_type_name
- secondary_type
- deck_copy_strategy

From `deck-direction.md`:
- recommended_variant
- emotional_lever
- information_posture
- max_length
- copywriting_engine_flags

From `hero-image-brief.md`:
- image_category
- context, emotion, subject, setting, format
- pattern_specific_requirements

From `trust-architecture.md`:
- signal_count
- signals (ordered array with type, display_text, visual_treatment)
- placement

From `cta-direction.md`:
- cta_function
- primary_variant (button_text, micro_copy, emotional_appeal)
- all_variants
- secondary_cta (if applicable)

From `price-display.md` (Type B only):
- variant_price (original, sale, savings)
- multi_pack_options
- subscription_option
- savings_messaging

**Step 3:** Assemble the JSON structure. Use the schema below exactly. All fields are required unless marked optional.

**Step 4:** Validate the JSON is well-formed. Verify all required fields are populated (not null, not empty string).

**Step 5:** Write `above-fold-blueprint.json` to the outputs directory.

## Quality Criteria
- JSON is valid and well-formed (parseable by any JSON parser)
- All required fields populated with actual values (no empty strings, no "TBD")
- Data matches the source Layer 2 files exactly (no paraphrasing or re-interpretation)
- Gate scores included in audit section (audit >= 7, anti-slop = PASS)
- Price display section populated for Type B, set to applicable=false for Type A
- File exists after execution

## Failure Modes
- Audit score < 7 → HALT, do not compile blueprint. Return to Layer 2/3 for revision.
- Anti-slop check FAIL → HALT, do not compile. Fix slop in Layer 2 outputs first.
- Layer 2 output file missing → HALT, identify missing file, run that microskill
- JSON validation fails → Fix structure, re-validate before writing
- Data mismatch between blueprint and source files → Re-read source, correct the extraction
