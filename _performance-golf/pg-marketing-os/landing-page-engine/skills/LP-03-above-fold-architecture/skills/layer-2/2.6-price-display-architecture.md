# 2.6 — Price Display Architecture

**Skill:** LP-03-above-fold-architecture
**Position:** Layer 2, Microskill 6
**Purpose:** (Type B only) Architect the above-fold price block including anchor pricing, savings display, multi-pack options, and subscription tiers. Price display is one of the highest-impact conversion elements on ecomm pages.
**Effort Level:** medium
**Dependencies:** 0.1 input-verification.md (offer data), 1.3 pattern-selection.md, 2.4 trust-architecture.md
**Output:** `price-display.md`

---

## Input Schema

```yaml
input:
  input_verification: "[path to input-verification.md]"
  pattern_selection: "[path to pattern-selection.md]"
  trust_architecture: "[path to trust-architecture.md]"
  page_type: "[Type B | Hybrid — skip if Type A]"
  offer_data:
    original_price: "[original/compare-at price]"
    sale_price: "[current price]"
    multi_pack_options: "[list of pack options with pricing, or 'none']"
    subscription_available: "[Y/N]"
    subscription_discount: "[% or amount]"
    guarantee: "[guarantee details]"
```

## Execution Logic

**Step 1:** Confirm page type. If Type A, write a single-line file noting "N/A — Type A pages do not display price above the fold" and exit. Price display is ONLY for Type B and Hybrid pages.

**Step 2:** Read offer data from input-verification.md or CopywritingEngine offer-package.json. If no pricing data is available, write a template with [PLACEHOLDER] values and flag for human input.

**Step 3:** Apply the Price Block Element Order from AGENT.md:

```
ELEMENT 1: Variant Price (with context)
  → Original price struck through: ~~$79.99~~
  → Sale price in accent color/bold: $59.99
  → Savings badge: [SAVE $20 (25%)]
  → RULE: ALWAYS show both % AND dollar savings

ELEMENT 2: Multi-Pack Selector (if applicable)
  → Radio buttons or pill selectors
  → Per-unit price for each option
  → "Best Value" badge on highest-quantity option
  → RULE: Bundle options increase AOV 35-60% when showing per-unit savings

ELEMENT 3: Subscription Option (if applicable)
  → Checkbox or toggle
  → Subscription price with savings shown
  → "Subscribe & Save [X%]" language
  → RULE: Never auto-select subscription — let user opt in

ELEMENT 4: Savings Messaging
  → "You save $60 with the 3-pack"
  → Contextual based on selected option
  → RULE: Savings message updates dynamically with selection
```

**Step 4:** Apply Price Display Rules from AGENT.md:
1. ALWAYS show original price struck through (anchoring)
2. ALWAYS show savings as BOTH % and dollar amount
3. For Type B: Price CAN appear above fold (images/rating = value signal)
4. Bundle options with per-unit savings (AOV driver)
5. Never auto-select subscription

**Step 5:** Define visual treatment:
- Price font: Largest non-headline text on page
- Sale price color: Accent color (typically red, green, or brand accent)
- Original price: Gray, struck through, smaller than sale price
- Savings badge: High-contrast pill or inline text
- Multi-pack: Clear selected state, obvious per-unit math

**Step 6:** Document the price-to-trust flow:
- Price display → Trust badge row → CTA button
- The trust signals from 2.4 should appear immediately after price to reduce price shock
- Guarantee badge is the most critical trust signal adjacent to price

**Step 7:** Write `price-display.md`.

## Quality Criteria
- Type A pages correctly skipped with N/A notation
- All 4 price block elements addressed (even if "Not available" for some)
- Original price anchoring present (never show sale price alone)
- Savings shown in BOTH percentage and dollar amount
- Subscription never auto-selected
- Price-to-trust flow documented (price must lead to reassurance, not isolation)
- File exists after execution

## Failure Modes
- Type A page → Write N/A file and exit (not an error)
- No pricing data available → Write template with [PLACEHOLDER], flag for human input
- Only one price point (no compare-at) → Flag that anchoring is missing, recommend establishing original price
- Subscription auto-selected in design → REJECT, explicit opt-in only
- Multi-pack without per-unit pricing → REJECT, per-unit math is required for AOV lift
- input-verification.md not found → HALT, run 0.1 first
