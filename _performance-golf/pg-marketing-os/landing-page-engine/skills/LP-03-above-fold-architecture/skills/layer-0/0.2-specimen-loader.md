# 0.2 — Specimen Loader

**Skill:** LP-03-above-fold-architecture
**Position:** Layer 0, Microskill 2
**Purpose:** Load above-fold specimens from the Landing Page Engine specimen vault, filtered by page type and vertical, to provide real-world reference patterns for architecture decisions.
**Effort Level:** low
**Dependencies:** 0.1 input-verification.md (page_type, vertical)
**Output:** `specimen-load.md`

---

## Input Schema

```yaml
input:
  page_type: "[Type A | Type B | Hybrid]"
  vertical: "[vertical from input-verification.md]"
  specimen_directory: "landing-page-engine/specimens/"
  specimen_index: "landing-page-engine/specimens/specimen-index.md"
```

---

## Execution Logic

**Step 1:** Read `input-verification.md` from 0.1. Extract `page_type` and `vertical`.

**Step 2:** Read `specimen-index.md` from the specimens directory. Identify all specimens that match:
- Primary filter: `page_type` match (Type A specimens for Type A, Type B for Type B, both for Hybrid)
- Secondary filter: `vertical` match (same vertical preferred, adjacent vertical acceptable)

**Step 3:** For each matched specimen, extract above-fold metadata:
- Headline type used (map to T1-T7 taxonomy)
- Trust signals visible above the fold
- CTA placement and language
- Hero image approach (product, lifestyle, authority, etc.)
- Price display presence (Type B only)
- Rating strip presence (Type B only)

**Step 4:** If fewer than 2 specimens match both filters, expand:
- First expansion: Drop vertical filter, keep page_type filter
- Second expansion: Include Hybrid specimens regardless of page_type filter
- Note any expansions in output

**Step 5:** Rank specimens by relevance:
- Exact page_type + exact vertical = Tier 1
- Exact page_type + adjacent vertical = Tier 2
- Expanded match = Tier 3

**Step 6:** Write `specimen-load.md` with all matched specimens and their above-fold characteristics.

## Quality Criteria
- At least 2 specimens loaded (after expansion if necessary)
- Each specimen has above-fold metadata extracted (headline type, trust, CTA, hero)
- Specimens tiered by relevance with clear labeling
- Pattern summary synthesizes cross-specimen trends
- File exists after execution

## Failure Modes
- Specimen index not found → HALT, report missing landing-page-engine/specimens/specimen-index.md
- Zero specimens match even after full expansion → Write file noting zero specimens, proceed with pattern library only (0.3)
- Specimen file referenced in index but not on disk → Skip with WARNING, note in output
- input-verification.md not found → HALT, run 0.1 first
