### Step 2: Scan Assembled Sections

Read through all three assembly files and check off each element:

For each required element:
1. Search for it across above-fold, body, and conversion assemblies
2. If found: note the section where it appears and the source skill
3. If not found: flag as MISSING with the upstream skill responsible

### Step 3: Count and Classify

- **Present:** Element exists in the assembled page
- **Missing — Required:** Element is required for page type but not present. This is a completeness failure.
- **Missing — Recommended:** Element is recommended but not required. This is a note, not a failure.
- **N/A:** Element does not apply to this page type (e.g., P.S. for Type B)

### Step 4: Calculate Completeness Score

```
Completeness % = (Required elements present / Total required elements) x 100
```

- 100%: All required elements present. PASS.
- 90-99%: Minor gaps. Flag missing elements. May still PASS depending on what is missing.
- Below 90%: Significant gaps. FAIL. Assembly is incomplete.

## QUALITY CRITERIA

- Every required element for the page type checked
- Each element has a clear PRESENT or MISSING status
- Missing required elements have the upstream skill identified
- Completeness percentage calculated correctly
- PASS/FAIL status based on clear threshold (90%+ for PASS)
- File exists after execution

---

## FAILURE MODES

- Marking an element as PRESENT when it is actually a placeholder "[CONTENT NEEDED]" -> Placeholders are NOT present content. If the content is a placeholder, the element is MISSING.
- Applying Type A requirements to a Type B page -> Check page type from brief-load.md BEFORE applying the checklist. Type A and Type B have different required element sets.
- Counting recommended elements toward completeness percentage -> Only REQUIRED elements factor into the percentage. Recommended elements are tracked but do not affect the PASS/FAIL status.
- Inflating completeness by counting elements that are present but empty/stub -> An element must have actual content to be PRESENT. An empty section header with no body content = MISSING.
