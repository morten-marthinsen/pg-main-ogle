# 3.2 — Arena Copy Verification

> **Skill:** LP-15-page-assembly
> **Layer:** 3
> **Position:** 2 of 3
> **Purpose:** GATE — Verify that Arena-selected copy from LP-07 (hero headline) and LP-14 (CTA) appears VERBATIM in the assembled output. This is the final check against the #1 failure mode: using pre-Arena candidates instead of Arena-selected variants.
> **Effort Level:** sonnet
> **Dependencies:** `arena-verification-load.md`, `above-fold-assembly.md`, `conversion-section-assembly.md`
> **Output:** `arena-copy-verification.md`

---

## WHY THIS GATE EXISTS

In the CopywritingEngine, Skill 19 (Campaign Assembly) used a pre-Arena draft instead of the Arena-selected Hybrid A. The entire lead section was wrong. The error was not caught until human review.

LP-15 has a DEDICATED verification step — this microskill — that catches this error BEFORE the page package is compiled. If this gate fails, LP-15 HALTS. The assembled page does NOT proceed to LP-16, LP-17, or LP-18.

This is not paranoia. This is a structural prevention of a documented failure mode.

## EXECUTION

### Step 1: Extract Authoritative Text

From `arena-verification-load.md`, extract:
- **Authoritative headline text** — the EXACT text from HERO_SELECTION.yaml
- **Authoritative CTA text** — the EXACT text from CTA_SELECTION.yaml for each position

### Step 2: Extract Assembled Text

From `above-fold-assembly.md`, extract:
- **Assembled headline text** — the headline that was placed in the above-fold section

From `above-fold-assembly.md` and `conversion-section-assembly.md`, extract:
- **Assembled CTA text** — the CTA text at every placement position

### Step 3: Verbatim Comparison — Headline

Compare the authoritative headline to the assembled headline character-by-character:

```
Authoritative: "[exact text from HERO_SELECTION.yaml]"
Assembled:     "[exact text from above-fold-assembly.md]"
Match:         [VERBATIM MATCH / MISMATCH]
```

If MISMATCH, identify the specific differences:
- Words added: [list]
- Words removed: [list]
- Words changed: [list with before -> after]
- Punctuation differences: [list]
- Capitalization differences: [list — these are acceptable]

### Step 4: Verbatim Comparison — CTA

For each CTA placement, compare the authoritative CTA to the assembled CTA:

```
Position: [above-fold / after-pricing / pre-close / etc.]
Authoritative: "[exact text from CTA_SELECTION.yaml]"
Assembled:     "[exact text from assembly]"
Match:         [VERBATIM MATCH / MISMATCH]
```

If MISMATCH, identify differences (same analysis as headline).

### Step 5: Determine Gate Result

**PASS conditions:**
- Headline: VERBATIM MATCH (capitalization differences acceptable)
- Primary CTA (above-fold): VERBATIM MATCH
- All other CTA placements: VERBATIM MATCH for Arena-selected text

**FAIL conditions:**
- Headline has ANY word substitution, addition, removal, or reordering
- Primary CTA has ANY word change
- A CTA placement uses a non-Arena variant

**HALT on FAIL:** If this gate fails, LP-15 HALTS. The assembled page must be corrected before proceeding. Identify which assembled file contains the wrong text and what the correct text should be.

### Step 6: Write arena-copy-verification.md

## QUALITY CRITERIA

- Both headline and CTA verified against authoritative source text
- Comparison is character-by-character (not "looks close enough")
- Differences explicitly identified if any exist
- Gate result is binary: PASS or FAIL
- FAIL triggers HALT with specific correction instructions
- File exists after execution

---

## FAILURE MODES

- Calling a near-match a VERBATIM MATCH -> "Get Your Supply Today" is NOT verbatim with "Get My Supply Today." "Your" vs "My" is a word change. FAIL.
- Skipping CTA verification because headline passed -> Both must pass. Headline PASS + CTA FAIL = overall FAIL.
- Rationalizing the mismatch ("the assembled version is actually better") -> You do not judge quality. The Arena judged quality. Your job is to verify the Arena's selection was used. Period.
- Verification passes but a CTA placement was missed entirely -> If a CTA placement position from LP-06 architecture has NO CTA text at all, that is also a FAIL. The Arena-selected CTA should be at every specified position.
