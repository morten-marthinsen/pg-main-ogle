### Step 2: Flag Missing Packages

For every package with status NOT_FOUND:
- Identify the upstream skill that must produce it
- Classify the impact: BLOCKING (cannot proceed) vs. DEGRADED (can proceed but page will be incomplete)

**BLOCKING packages (assembly cannot proceed):**
- page-brief.json (LP-00) — no page type = no assembly
- section-sequence.json (LP-04) — no sequence = no assembly order
- hero-section-package.json (LP-07) — no headline = no page
- cta-copy-package.json (LP-14) — no CTA = no conversion
- HERO_SELECTION.yaml (LP-07) — no Arena verification = risk of #1 failure mode
- CTA_SELECTION.yaml (LP-14) — no Arena verification = risk of #1 failure mode

**DEGRADED packages (assembly can proceed with gaps):**
- above-fold-blueprint.json (LP-03) — use MASTER-BLUEPRINT.md defaults
- social-proof-architecture.json (LP-05) — use default proof wave pattern
- offer-cta-architecture.json (LP-06) — use content packages directly
- trust-elements.json (LP-08) — above-fold trust bar may be empty
- benefits-features.json (LP-09) — benefits section will be missing
- social-proof-copy.json (LP-10) — proof blocks will be empty
- pricing-section.json (LP-11) — pricing section will be missing
- faq-package.json (LP-12) — FAQ section will be missing
- urgency-package.json (LP-13) — urgency section will be missing

### Step 3: Determine Assembly Mode

- **Full mode:** All 14 packages present AND Arena verification VERIFIED -> Proceed with full assembly.
- **Partial mode:** 1-4 DEGRADED packages missing but all BLOCKING packages present -> Proceed with partial assembly. Note gaps in inventory.
- **HALT:** Any BLOCKING package missing -> Cannot proceed. List all missing BLOCKING packages.

### Step 4: Page Type Element Check

Cross-reference page type from `brief-load.md` against the available content:

**If Type A:**
- Required: headline, deck, hero image, story/lead content (from benefits/proof), root cause, mechanism, proof blocks, value stack, pricing, guarantee, urgency, FAQ, CTA placements, close, P.S.
- Flag any Type A required elements not covered by available packages

**If Type B:**
- Required: product title/H1, rating strip, hero product image, price block, ATC button, trust badges, key benefits, ingredient/feature section, review cascade, FAQ, guarantee
- Flag any Type B required elements not covered by available packages

**If Hybrid:**
- Required: Type B above-fold elements + Type A body elements
- Flag any Hybrid required elements not covered by available packages

## QUALITY CRITERIA

- All 14 packages accounted for (present or explicitly flagged)
- Assembly mode correctly determined based on which packages are present
- BLOCKING vs. DEGRADED classification accurate
- Page type element coverage checked
- Arena verification status carried forward from 0.4
- File exists after execution

---

## FAILURE MODES

- Inventory shows all packages present but arena-verification-load.md shows MISMATCH -> Assembly mode = HALT despite all packages being present. Arena integrity takes priority.
- Inventory shows partial packages but operator wants to proceed -> Assembly mode = PARTIAL is valid ONLY if all BLOCKING packages are present. Document the gaps clearly.
- Inventory double-counts a package -> Each package appears exactly once. Deduplicate.
