# 0.1 — Brief Loader

> **Skill:** LP-15-page-assembly
> **Layer:** 0
> **Position:** 1 of 4
> **Purpose:** Load the page brief from LP-00 and extract page type, audience, vertical, awareness stage, and any CopywritingEngine integration data.
> **Effort Level:** haiku
> **Dependencies:** `page-brief.json` from LP-00 Brief Classifier
> **Output:** `brief-load.md`

---

## INPUT

```yaml
input:
  page_brief_path: "[path to page-brief.json from LP-00]"
  project_folder: "[path to project output folder]"
```

---

## EXECUTION

1. Locate `page-brief.json` in the project output folder. Search paths:
   - `./outputs/[project-name]/landing-page/lp-00-brief-classifier/page-brief.json`
   - `./outputs/[project-name]/lp-00-brief-classifier/page-brief.json`

2. Parse the page brief. Extract and catalog:
   - **Page type:** type_a, type_b, or hybrid
   - **Classification signals:** which of the 7 signals triggered the classification
   - **Audience:** target audience description, awareness stage (1-5), sophistication level
   - **Vertical:** product vertical (health, golf, finance, personal-dev, technology, other)
   - **Product/offer:** product name, core promise, price point, guarantee
   - **CopywritingEngine mode:** downstream (has CE packages) or standalone
   - **CE packages available:** list any CopywritingEngine packages referenced in the brief

3. If `page-brief.json` is not found -> Write `brief-load.md` with status `NOT_FOUND`. This triggers GATE_0 HALT.

4. Write the extracted data to `brief-load.md`.

---

## OUTPUT SCHEMA

`brief-load.md`:

```markdown
# Brief Load — [Project Name]

## Load Status: [LOADED | NOT_FOUND]

## Source File
- Path: [full path to page-brief.json]

## Page Type
- Classification: [type_a | type_b | hybrid]
- Classification signals: [list the signals that triggered this classification]

## Audience
- Target: [audience description]
- Awareness stage: [1-5 with label: Unaware / Problem-Aware / Solution-Aware / Product-Aware / Most Aware]
- Sophistication level: [if available]

## Vertical
- Vertical: [health | golf | finance | personal-dev | technology | other]
- Sub-vertical: [if specified]

## Product/Offer
- Product name: [name]
- Core promise: [the main promise/benefit]
- Price point: [price or price range]
- Guarantee: [guarantee summary]

## CopywritingEngine Integration
- Mode: [downstream | standalone]
- CE packages available: [list or "none"]
```

---

## QUALITY CRITERIA

- Page brief located and fully parsed
- Page type extracted and confirmed as one of: type_a, type_b, hybrid
- Audience and awareness stage extracted
- Vertical identified
- CopywritingEngine mode determined
- File exists after execution

---

## FAILURE MODES

- Page brief not found -> Write `brief-load.md` with status `NOT_FOUND`. Triggers GATE_0 HALT: "LP-15 requires page-brief.json from LP-00. Run LP-00 first."
- Page brief found but page type missing or ambiguous -> Extract what is available. Flag "PAGE_TYPE_AMBIGUOUS" in the output. Triggers human review before proceeding.
- Page brief references CE packages that are not present in the project folder -> Note this as informational; LP-15 does not require CE packages, only LP-00 through LP-14 packages.
