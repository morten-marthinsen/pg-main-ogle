## EXECUTION

1. Locate each content package in the project output folder. Search paths per package:
   - `./outputs/[project-name]/landing-page/lp-XX-[skill-name]/[package-name].json`
   - `./outputs/[project-name]/lp-XX-[skill-name]/[package-name].json`

2. For each package found, extract the key content elements:

### From LP-07 (hero-section-package.json):
- **Headline variants** — list ALL variants, flag which is Arena-selected
- **Deck copy / subheadline** — the copy that expands the headline promise
- **Hero image brief** — image specification for the hero section
- **Pre-head** — micro-targeting hook if present
- **Early trust signals** — any trust elements specified for above-fold placement

### From LP-08 (trust-elements.json):
- Trust badges (with specifics: "GMP Certified", "Trustpilot 4.6/5")
- Media mentions ("As Seen On...")
- Certification badges
- Partner logos
- Customer count claims ("50,000+ customers")

### From LP-09 (benefits-features.json):
- Benefit bullets (benefit statements in audience voice)
- Feature deep-dives (ingredient/feature details with D-F-W-B-P structure)
- Benefit-before-feature ordering confirmation

### From LP-10 (social-proof-copy.json):
- Testimonials (full text, attribution, specific outcomes)
- Reviews (star rating, text, verified buyer status)
- Case studies (if any)
- Before/after content (if any)
- Social proof volume data (total count)

### From LP-11 (pricing-section.json):
- Value stack (itemized value with individual prices)
- Price display (sale price, original price, savings)
- Bundle options (multi-pack, subscription, single)
- Price anchoring technique
- Per-day calculation (if applicable)

### From LP-12 (faq-package.json):
- All FAQ Q&A pairs (question + answer, ordered by priority)
- Objection-handling framing
- Number of FAQs

### From LP-13 (urgency-package.json):
- Urgency elements (deadline, countdown, limited stock)
- Scarcity justification (why the scarcity is real)
- Urgency copy (the actual text)

### From LP-14 (cta-copy-package.json):
- **CTA variants** — list ALL variants, flag which is Arena-selected
- CTA text for each placement position (above-fold, mid-page, pre-close)
- CTA styling notes (color, size, emotional appeal type)

3. For each missing package, log as `NOT_FOUND` with the upstream skill that must be run.

4. Write the consolidated content data to `content-load.md`.

## FAILURE MODES

- LP-07 hero-section-package.json not found -> CRITICAL. The page has no headline. Write status as NOT_FOUND. Triggers GATE_0 HALT.
- LP-14 cta-copy-package.json not found -> CRITICAL. The page has no CTA copy. Write status as NOT_FOUND. Triggers GATE_0 HALT.
- Other packages not found -> Flag as missing. Assembly can potentially proceed in partial mode, but every missing package must be noted and will cause LP-17 audit failures.
- Package found but Arena selection not flagged in the data -> Note "ARENA_SELECTION_NOT_FLAGGED" — this means 0.4 Arena Verification Loader must use the YAML checkpoint files to determine the selection.
- Package extremely large (100+ testimonials, 20+ FAQ) -> Extract all. Do not truncate. The assembler needs the full inventory.
