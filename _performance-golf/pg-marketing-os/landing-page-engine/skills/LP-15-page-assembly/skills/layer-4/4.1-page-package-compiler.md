# 4.1 — Page Package Compiler

> **Skill:** LP-15-page-assembly
> **Layer:** 4
> **Position:** 1 of 3
> **Purpose:** Compile all assembly outputs into the final `assembled-page-package.json` — the single structured file that LP-16, LP-17, and LP-18 consume.
> **Effort Level:** haiku
> **Dependencies:** All Layer 2 outputs (above-fold-assembly.md, body-section-assembly.md, conversion-section-assembly.md), all Layer 3 outputs (completeness-audit.md, arena-copy-verification.md, assembly-audit.md), `threading-validation.md`, `brief-load.md`, `arena-verification-load.md`
> **Output:** `assembled-page-package.json`

---

## INPUT

```yaml
input:
  above_fold: "above-fold-assembly.md"
  body_sections: "body-section-assembly.md"
  conversion_sections: "conversion-section-assembly.md"
  completeness_audit: "completeness-audit.md"
  arena_verification: "arena-copy-verification.md"
  assembly_audit: "assembly-audit.md"
  threading_validation: "threading-validation.md"
  brief_load: "brief-load.md"
  arena_verification_load: "arena-verification-load.md"
```

---

## EXECUTION

### Step 1: Read All Source Files

Read all 9 input files and extract the data needed for the JSON schema defined in LP-15-AGENT.md.

### Step 2: Assemble JSON Structure

Build `assembled-page-package.json` following the schema from LP-15-AGENT.md OUTPUT SCHEMA:

1. **Header fields:**
   - `schema_version`: "1.0"
   - `created`: ISO timestamp
   - `project_name`: from brief-load.md
   - `page_type`: from brief-load.md
   - `vertical`: from brief-load.md

2. **Assembly metadata:**
   - `total_sections`: count from section assembly files
   - `total_estimated_words`: sum from all assembly files
   - `assembly_mode`: from package-inventory.md (full/partial)
   - `missing_packages`: list from package-inventory.md
   - `arena_verification`: hero_selection_verified, cta_selection_verified, variant IDs

3. **Above fold:** from above-fold-assembly.md
   - pre_head, headline (Arena-selected), subheadline_deck, hero_image, trust_bar, primary_cta (Arena-selected), rating_strip, price_display

4. **Sections array:** from body-section-assembly.md
   - Each section: section_number, section_name, section_purpose, source_skill, content (headline, body, proof_elements, cta, images), transition_in, transition_out, estimated_words

5. **Conversion block:** from conversion-section-assembly.md
   - value_stack, pricing (display, anchoring, bundles, subscription), guarantee (branded_name, duration, terms, full_text), urgency, faq array, close (benefit_summary, final_cta, ps)

6. **CTA placements array:** from conversion-section-assembly.md CTA Summary
   - Each placement: position, text, style, arena_selected flag

7. **Technical specs:** assembled from all sources
   - image_format, lazy_loading, mobile_sticky_cta, cta_touch_target, load_optimization_notes

8. **Threading summary:** from threading-validation.md
   - headline_promise, mechanism_connection, close_callback, threading_score

9. **Quality scores:** from assembly-audit.md
   - assembly_score, completeness_score, threading_score, arena_verification

### Step 3: Validate JSON Completeness

Before writing, verify:
- `above_fold.headline` is not null or empty
- `above_fold.primary_cta.text` is not null or empty
- `sections` array has at least 3 entries
- `conversion_block.guarantee.branded_name` is not null (if page type requires it)
- `cta_placements` array has at least 3 entries
- `quality_scores.arena_verification` is "PASS"
- Total word count is reasonable for page type (Type A: 4,000-18,000; Type B: 1,500-7,000)

### Step 4: Write assembled-page-package.json

Write the complete JSON file to the project output folder.

---

## OUTPUT SCHEMA

`assembled-page-package.json` — follows the complete schema defined in LP-15-AGENT.md OUTPUT SCHEMA section.

Key validation rules:
- `page_type` must be one of: type_a, type_b, hybrid
- `above_fold.headline` must be non-empty (this is the Arena-selected headline)
- `above_fold.primary_cta.text` must be non-empty (this is the Arena-selected CTA)
- `sections` array must be ordered by section_number
- Every section must have a `transition_in` (except the first body section, which gets transition from above-fold)
- `quality_scores.arena_verification` must be "PASS"
- `threading_summary.threading_score` must match the score in threading-validation.md

---

## QUALITY CRITERIA

- JSON is well-formed and parseable
- All sections from assembly files included in correct order
- Arena-selected headline and CTA appear in the correct fields
- Quality scores populated from Layer 3 audit
- Threading summary populated from threading validation
- Metadata complete (project name, page type, vertical)
- File exists after execution

---

## FAILURE MODES

- JSON malformed -> Re-check structure. Ensure all brackets, quotes, and commas are correct. Pay special attention to copy text that may contain quote characters — escape them.
- Missing data from source files -> Re-read the source file. If data is genuinely missing, include the field with a value of "MISSING — [source file] did not contain this field" rather than omitting it.
- Arena verification not PASS -> Do NOT write the file. Return to Layer 3 gate. LP-15 does not produce output with failed Arena verification.
- Word count outside expected range -> Include the count as-is but add a note in assembly_metadata: "word_count_note": "Total word count [X] is outside expected range for [page_type]. This may indicate missing sections or excessive content."
