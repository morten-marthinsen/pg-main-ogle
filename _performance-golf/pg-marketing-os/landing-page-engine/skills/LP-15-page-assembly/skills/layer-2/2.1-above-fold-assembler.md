# 2.1 — Above-Fold Assembler

> **Skill:** LP-15-page-assembly
> **Layer:** 2
> **Position:** 1 of 4
> **Purpose:** Assemble the above-fold section using Arena-selected headline from LP-07, deck copy, hero image brief, trust signals from LP-08, and primary CTA from LP-14 Arena selection. This is the first section readers see — it determines stay-or-leave in 5 seconds.
> **Effort Level:** sonnet
> **Dependencies:** `architecture-load.md` (LP-03 above-fold blueprint), `content-load.md` (LP-07, LP-08, LP-14 content), `arena-verification-load.md` (authoritative headline + CTA text), `section-order-plan.md`
> **Output:** `above-fold-assembly.md`

## EXECUTION

**POSITIONAL REINFORCEMENT:** You are assembling, not rewriting. The headline comes from LP-07 Arena selection (verbatim from arena-verification-load.md). The CTA comes from LP-14 Arena selection (verbatim). Trust signals come from LP-08. You arrange these elements according to LP-03's above-fold blueprint. You do NOT generate new copy.

### Step 1: Retrieve Authoritative Arena-Selected Copy

From `arena-verification-load.md`, extract:
- **Authoritative headline text** — this is the EXACT headline that goes into the assembled above-fold. No paraphrasing. No "improvements."
- **Authoritative CTA text** (above-fold position) — this is the EXACT primary CTA.

**CRITICAL CHECK:** If `arena-verification-load.md` status is not VERIFIED, STOP. Do not assemble with unverified copy.

### Step 2: Retrieve Above-Fold Blueprint

From `architecture-load.md` (LP-03 section), extract the above-fold element order and hierarchy. If LP-03 is not available, use the page-type default from MASTER-BLUEPRINT.md:

**Type A default above-fold order:**
1. Pre-head (micro-targeting hook)
2. Headline (primary promise)
3. Deck copy (expands promise)
4. Hero image (aspirational/product-in-context)
5. Trust bar (media logos, certifications)
6. [Optional: Early CTA for warm traffic]

**Type B default above-fold order:**
1. Breadcrumb navigation
2. Product title / H1
3. Rating strip (stars + review count)
4. Hero product image
5. Price block (sale price, savings)
6. ATC button (high contrast, action verb)
7. Short description (2-3 sentences)
8. Trust badges row

### Step 3: Retrieve Content Elements

From `content-load.md`, gather:
- **LP-07:** Deck copy, hero image brief, pre-head (if any)
- **LP-08:** Trust badges, media mentions, certification badges, partner logos, customer count claims
- **LP-14:** CTA styling notes (color, size, emotional appeal type)

### Step 4: Assemble Above-Fold Section

Arrange all elements according to the blueprint order. For each element:
1. Place the element in its blueprint position
2. Include the verbatim copy from the content package
3. Include any visual/design specifications
4. Note the element's purpose in the conversion funnel

**Headline placement rule:** The Arena-selected headline goes in the headline position. Period. Do not swap it for a "better" variant. Do not modify it.

**CTA placement rule:** The Arena-selected CTA goes in the primary CTA position. If Type B, this is the ATC button. If Type A, this is the early CTA (if the blueprint calls for one above-fold).

### Step 5: Write Transition Out

Write the transition FROM the above-fold TO the first body section. Reference the `transition-strategy.md` plan for this boundary.

For Type A: This is typically a bridge into the lead/hook section.
For Type B: This is typically a header-only transition ("Key Benefits" or similar).

### Step 6: Write above-fold-assembly.md
