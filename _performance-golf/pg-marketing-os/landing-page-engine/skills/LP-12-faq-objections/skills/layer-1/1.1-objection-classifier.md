# 1.1 — Objection Classifier

**Skill:** LP-12-faq-objections
**Position:** Layer 1, Microskill 1
**Purpose:** Classify each objection from the inventory into one of 6 categories, identify the hidden objection behind each surface question, and assign an objection strength score. This classification drives sequencing (1.2) and answer strategy (1.3).
**Effort Level:** medium
**Dependencies:** `objection-inventory.md` (0.2), `packages-loaded.md` (0.1)
**Output:** `objection-classification.md`

---

## Input Schema

```yaml
input:
  objection_inventory: "[all objections from objection-inventory.md]"
  page_type: "[type_a | type_b | hybrid]"
  vertical: "[vertical name]"
  audience_profile: "[from packages-loaded.md]"
  offer_summary: "[price, guarantee, bonuses from packages-loaded.md]"
```

## Execution Logic

1. **Load the 6 objection categories** from LP-12-AGENT.md:
   - Price/Value (MANDATORY)
   - Efficacy/Results (MANDATORY)
   - Trust/Guarantee (MANDATORY)
   - Timing
   - Comparison
   - Technical/Logistics

2. **For each objection in the inventory,** perform a 3-step classification:

   **Step A: Assign primary category.**
   Read the surface question and hidden objection. Assign to the single most accurate category. If an objection spans two categories, assign to the one that reflects the CORE concern:
   - "Is this worth $97?" -> Price/Value (not Timing, not Trust)
   - "What if it doesn't work for me?" -> This could be Efficacy OR Trust. If the answer requires proof of results, it is Efficacy. If the answer requires a refund policy, it is Trust. Assign based on what the ANSWER must deliver.
   - "How is this different from just taking a regular magnesium pill?" -> Comparison (not Efficacy)

   **Step B: Identify the hidden objection.**
   Every surface question has a deeper concern underneath it. The hidden objection is what the prospect ACTUALLY needs resolved to buy:
   - "How long does shipping take?" -> Hidden: "Will I have to wait an unreasonable amount of time?"
   - "What are the ingredients?" -> Hidden: "Is this safe? Are there things in it I don't want to put in my body?"
   - "Can I cancel anytime?" -> Hidden: "Am I going to get trapped in a subscription I can't escape?"
   - "Does this work for beginners?" -> Hidden: "Am I going to waste money on something too advanced for me?"

   **Step C: Assign final strength score (1-10).**
   Strength measures how likely this objection is to PREVENT a purchase if left unaddressed:
   - 9-10: Purchase will NOT happen without this being addressed (price objection on a $297 product, "does it work?" in a skeptical market)
   - 7-8: Strong barrier — many prospects will abandon without an answer
   - 5-6: Moderate — some prospects care, others do not
   - 3-4: Minor — only a small segment asks this
   - 1-2: Edge case — very few prospects have this concern

3. **Verify mandatory category coverage.** After classification, check:
   - Price/Value: >= 1 objection classified here
   - Efficacy/Results: >= 1 objection classified here
   - Trust/Guarantee: >= 1 objection classified here
   - If any mandatory category has 0: flag and recommend adding an objection from the expansion mining protocol in 0.2.

4. **Select the final 8-12 objections** for the FAQ:
   - Start with all mandatory-category objections (these are non-negotiable)
   - Add the highest-strength objections from other categories until 8-12 items reached
   - If more than 12 qualify, drop the lowest-strength items
   - If fewer than 8 qualify, return to 0.2 for expanded mining

5. **Flag the "hardest objection"** — the single objection with the highest strength score. This objection determines whether it goes first (Type A) or last (Type B).

6. **Write `objection-classification.md`** with complete classification.

## Quality Criteria
- Every objection classified into exactly 1 of 6 categories (no unclassified items)
- Hidden objection identified for every item (not just the surface question restated)
- Strength scores differentiated (not all clustered at 5-6)
- Mandatory categories all have >= 1 objection in the selected set
- Final selection is 8-12 items (not more, not fewer)
- Hardest objection clearly identified

## Failure Modes
- Classifying "What's the return policy?" as Technical/Logistics instead of Trust/Guarantee -> The hidden objection is about risk/trust, not logistics. Classify by what the ANSWER must address.
- All strength scores at 5-6 (no differentiation) -> Force-rank. If the audience's #1 barrier is price, that is a 9-10. If a dosing question is niche, that is a 3-4. Differentiate.
- Missing a mandatory category in the selected set -> Do not proceed. Return to 0.2 or add an objection from vertical norms.
- Selecting more than 12 items -> The FAQ becomes unwieldy. Cut the lowest-strength items. More items is not better.
