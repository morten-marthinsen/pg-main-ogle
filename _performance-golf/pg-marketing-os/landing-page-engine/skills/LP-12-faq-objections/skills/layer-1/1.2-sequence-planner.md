# 1.2 — Sequence Planner

**Skill:** LP-12-faq-objections
**Position:** Layer 1, Microskill 2
**Purpose:** Determine the optimal sequence order for FAQ items based on page type (Type A = hardest-first, Type B = hardest-last). Lock the sequence before any answers are written. Sequencing is a strategic decision — the order in which objections are addressed materially affects whether the reader finishes the FAQ section with confidence or doubt.
**Effort Level:** medium
**Dependencies:** `objection-classification.md` (1.1), `packages-loaded.md` (0.1)
**Output:** `sequence-plan.md`

---

## Execution Logic

1. **Determine sequencing strategy** from page type:

   | Page Type | Strategy | Rationale |
   |-----------|----------|-----------|
   | Type A | Hardest-First | Long-form readers who reach the FAQ are already invested. Leading with the hardest objection (price, efficacy) signals confidence and credibility. They did not scroll this far for softball questions. |
   | Type B | Hardest-Last | PDP/ecomm readers scan the FAQ quickly. Starting with easy logistics questions builds positive momentum ("this company gives clear answers") before addressing the harder concerns (price, efficacy). |
   | Hybrid | Hardest-Last | Hybrid FAQ sits after the Type B purchase zone. Use the Type B warm-up strategy. |

2. **Apply the sequencing template:**

   **Type A — Hardest-First Sequence:**
   ```
   Position 1-2: Highest-strength objections (strength 8-10)
                  Priority: price/value or efficacy — whichever is strongest
   Position 3-4: Second-tier hard objections (strength 7-8)
                  Priority: trust/guarantee, comparison
   Position 5-6: Moderate objections (strength 5-7)
                  Priority: timing, secondary efficacy/value questions
   Position 7-10+: Technical/logistics questions (strength 3-6)
                  Priority: product-specific practical questions
   Final position: A confidence-building close question
                  Often: guarantee reiteration or "What if I need help?"
   ```

   **Type B — Hardest-Last Sequence:**
   ```
   Position 1-2: Easy technical/logistics questions (strength 3-5)
                  Priority: "How do I take it?" "What's in it?" "How fast does it ship?"
   Position 3-4: Product-specific technical (strength 4-6)
                  Priority: dosing, ingredients, interactions, compatibility
   Position 5-6: Moderate objections (strength 5-7)
                  Priority: comparison, timing
   Position 7-8: Hard objections (strength 7-8)
                  Priority: efficacy/results
   Position 9-10: Hardest objections (strength 8-10)
                  Priority: trust/guarantee, then price/value
   Final position: The single hardest objection — addressed with maximum proof
   ```

3. **Assign each selected objection to a specific position.** Map every OBJ-XX from the classification to a numbered sequence position.

4. **Verify sequence integrity:**
   - Hard objections are in the correct zone (first 1/3 for Type A, last 1/3 for Type B)
   - No two adjacent questions address the same category (varies the rhythm)
   - The final question ends on a note that builds confidence toward the CTA that follows

5. **Consider the surrounding context:**
   - What section PRECEDES the FAQ? If the offer section precedes it, the reader's first objection is likely price/value.
   - What section FOLLOWS the FAQ? If the final CTA follows, end the FAQ on a trust/confidence note.
   - Adjust sequence if context demands it (e.g., if urgency section precedes FAQ, timing objections are already addressed — deprioritize them).

6. **Write `sequence-plan.md`** with locked sequence.

## Quality Criteria
- Sequence matches page type strategy (hardest-first for Type A, hardest-last for Type B)
- Every selected objection assigned to exactly 1 position
- No two adjacent questions are the same category
- Hard objections are in the correct zone (not buried in the middle for either page type)
- Final question sets up the CTA transition
- Surrounding section context considered and adjustments documented

## Failure Modes
- Hardest objection buried in the middle of the sequence -> Type A: must be in positions 1-3. Type B: must be in the final 3 positions. Middle placement is neither strategy.
- All questions from one category clustered together -> Distribute categories across the sequence. Clustering reads as repetitive.
- Final question is a logistics question for Type A -> Type A should close on a confidence-building note (guarantee, results expectation, support availability) — not "How fast does it ship?"
- Ignoring surrounding context -> If the offer section precedes the FAQ, the reader just saw the price. Adjust the sequence to address price in the optimal position (first for Type A, confirm in final zone for Type B).
