# 0.3 — Specimen FAQ Loader

**Skill:** LP-12-faq-objections
**Position:** Layer 0, Microskill 3
**Purpose:** Load type-matched FAQ specimens from the swipe vault to provide concrete reference examples for answer length, tone, question phrasing, and objection coverage. Specimens are NOT templates — they are reference points that calibrate quality expectations before generation begins.
**Effort Level:** low
**Dependencies:** `packages-loaded.md` (0.1) — specifically `page_type` and `vertical`
**Output:** `specimens-loaded.md`

---

## Input Schema

```yaml
input:
  page_type: "[type_a | type_b | hybrid — from packages-loaded.md]"
  vertical: "[health, finance, info_product, personal_dev, technology — from packages-loaded.md]"
  specimen_sources:
    type_b_primary:
      - "landing-page-engine/specimens/raw/bioptimizers-magnesium-trial-raw.md"
      - "landing-page-engine/specimens/raw/lmnt-electrolyte-pdp-raw.md"
    type_a_primary:
      - "landing-page-engine/specimens/raw/richmond-dinh-tiny-challenge-raw.md"
    fallback:
      - "landing-page-engine/swipe-vault-index.md"
```

## Execution Logic

1. **Determine which specimens to load** based on page type:

   **Type B (Ecomm/PDP):**
   - Primary: `bioptimizers-magnesium-trial-raw.md` — 10+ FAQ items covering safety, dosing, pregnancy, comparison, ingredients, sleep-specific usage
   - Primary: `lmnt-electrolyte-pdp-raw.md` — 6 FAQ items including radical transparency DIY recipe, shipping, diet compatibility
   - Both are health/supplement DTC pages with strong FAQ sections

   **Type A (Long-Form Sales Page):**
   - Primary: `richmond-dinh-tiny-challenge-raw.md` — 2 FAQ items (time commitment, content access)
   - Note: Type A specimens tend to have shorter FAQ sections. Supplement with swipe vault reference if available.

   **Hybrid:**
   - Load both Type A and Type B specimens. Hybrid FAQs blend logistics depth (Type B) with objection handling depth (Type A).

2. **Load each specimen file** and extract ONLY the FAQ section:
   - Locate the FAQ section within the raw specimen
   - Extract each Q&A pair verbatim
   - Note: total question count, category distribution, answer length range, tone/voice characteristics

3. **Analyze each specimen FAQ for patterns:**

   **Per specimen, note:**
   - Total FAQ items: [count]
   - Categories covered: [list]
   - Average answer length: [word count range]
   - Answer opener style: [direct answer / sycophantic / mixed]
   - Proof included in answers: [Y/N — which ones?]
   - Voice register: [conversational / clinical / corporate]
   - Hardest objection present: [which question?]
   - Weaknesses: [what is this FAQ missing?]

4. **Extract 3-5 reference patterns** across all loaded specimens:
   - What question types appear in ALL specimens? (These are universal)
   - What answer structure works best? (Direct answer + detail + confidence builder)
   - What is the typical answer length? (2-4 sentences in most good specimens)
   - What do the best answers do that the worst answers do not? (Specificity, proof, hidden-objection awareness)

5. **Note gaps in specimens** — what the specimens do NOT do well that LP-12 must improve upon:
   - Richmond Dinh has only 2 FAQ items — far too few
   - Some specimen answers may use sycophantic openers or vague language
   - Some specimens may lack price/value objections entirely

6. **Write `specimens-loaded.md`** with all extracted Q&A pairs, patterns, and gaps.

## Quality Criteria
- At least 2 specimens loaded (even if one is from a different vertical, the FAQ patterns transfer)
- All FAQ items extracted verbatim from each specimen (not paraphrased)
- Pattern analysis identifies both strengths and weaknesses of each specimen
- Cross-specimen patterns distilled into actionable calibration notes
- Improvement targets explicitly stated — LP-12 should produce BETTER FAQ sections than the specimens, not merely match them

## Failure Modes
- Loading zero specimens -> HALT. At least 2 must be loaded. If primary specimens are not relevant to this vertical, use swipe-vault-index.md to find alternatives.
- Only extracting 1-2 FAQ items from a specimen that has 10+ -> Extract ALL items. Partial extraction produces incomplete pattern analysis.
- Treating specimens as templates to copy -> Specimens are reference points for quality calibration. The actual FAQ must be written from scratch for this specific product.
- Ignoring specimen weaknesses -> If a specimen has sycophantic openers or generic answers, note these as anti-patterns to avoid, not patterns to follow.
