# 2.3 — Technical FAQ Writer

**Skill:** LP-12-faq-objections
**Position:** Layer 2, Microskill 3
**Purpose:** Write the Q&A pairs for product-specific technical questions. These are the practical, factual questions a prospect asks when they are close to buying but need specific details before committing. For supplements: dosing, ingredients, interactions, safety. For info products: access, format, time commitment, skill level. For physical products: materials, sizing, compatibility.
**Effort Level:** medium
**Dependencies:** `answer-strategies.md` (1.3), `sequence-plan.md` (1.2), `packages-loaded.md` (0.1)
**Output:** `technical-faq-answers.md`

## Execution Logic

1. **Identify the relevant technical question set** based on product type:

   **Supplement / Health Product:**
   - Ingredients/formulation: "What's in it?" "Are the ingredients natural?" "Is it non-GMO/gluten-free/vegan?"
   - Dosing: "How do I take it?" "How many per day?" "Morning or night?" "With food or without?"
   - Safety/interactions: "Is it safe with my medications?" "Safe for children?" "Safe during pregnancy/nursing?"
   - Comparison-technical: "Why all 7 forms instead of just 1?" "What is the difference between types?"
   - Side effects: "Are there any side effects?" "What if I experience X?"

   **Info Product / Course:**
   - Access: "How do I access the content?" "Is it online or downloadable?" "Is there an app?"
   - Time commitment: "How much time does this take?" "Can I go at my own pace?"
   - Skill level: "Is this for beginners?" "What if I'm already advanced?"
   - Format: "Is it video? Text? Live?" "Can I watch on my phone?"
   - Support: "Is there a community?" "Can I ask questions?" "Do I get 1-on-1 help?"

   **Physical Product (non-supplement):**
   - Materials/quality: "What is it made of?" "Where is it manufactured?"
   - Sizing/compatibility: "What size should I get?" "Is it compatible with X?"
   - Care/maintenance: "How do I clean/maintain it?" "How long does it last?"

   **Software / SaaS:**
   - Features: "Does it integrate with X?" "Can it do Y?"
   - Learning curve: "How long to set up?" "Is there a tutorial?"
   - Data/privacy: "Is my data secure?" "Can I export my data?"

2. **Write each technical Q&A pair** using the answer-strategies.md approach (almost always "Direct" strategy for technical questions):

   **Answer structure for technical questions:**
   - Sentence 1: Direct factual answer to the question
   - Sentence 2: Supporting detail that adds useful context
   - Sentence 3 (optional): Confidence builder or recommendation

   Technical answers CAN be shorter than core objection answers (2-3 sentences is typical). The reader wants facts, not persuasion.

3. **Ensure specificity:**
   - If ingredient amounts are known, state them ("1,000mg sodium, 200mg potassium, 60mg magnesium")
   - If dosing instructions exist, be specific ("2 capsules in the morning, 2 in the evening — with or without food")
   - If certifications exist, name them ("GMP certified, third-party tested, non-GMO verified")
   - NEVER use "high quality ingredients" or "carefully formulated" without specifics

4. **Handle "I don't know" scenarios:**
   - If specific product data is not available from upstream packages, note the gap and write a placeholder answer structure that the human can fill with real data
   - Mark these as NEEDS_HUMAN_INPUT
   - Do NOT invent product details that are not in the upstream data

5. **Special case: Radical Transparency FAQ** (LMNT pattern)
   - Some brands include FAQ items that are radically transparent (LMNT includes a DIY recipe for their own product)
   - If the product/brand lends itself to this approach, consider including 1 transparency FAQ item
   - This builds extraordinary trust ("they told me how to make it myself, so the product must be good")

6. **Per-answer quality gate** (same as 2.2):
   - First sentence is direct factual answer
   - No sycophantic openers
   - Product-specific (uses product name, specific ingredients/features)
   - 2-3 sentences (technical answers trend shorter)
   - Zero AI telltales

## Quality Criteria
- Every answer includes at least 1 specific product detail (ingredient name, dosing amount, feature name, certification)
- No generic answers ("We use high quality ingredients" without naming them)
- Safety-related answers include appropriate caveats ("Consult your healthcare provider") without making the caveat the entire answer
- Answers are factually grounded in upstream data — no invented product claims
- 2-3 sentences per answer (technical answers are concise)
- Items missing product data are clearly marked NEEDS_HUMAN_INPUT

## Failure Modes
- Answering "What's in it?" with "A proprietary blend of natural ingredients" -> Name the ingredients. If they are not in the upstream data, mark as NEEDS_HUMAN_INPUT.
- Safety answer that is ONLY a disclaimer -> "Consult your healthcare provider" is a legal note, not an answer. Lead with what IS known about safety (testing, certifications, known interactions), THEN add the disclaimer.
- Inventing dosing instructions that are not in the upstream data -> If dosing information is not available, mark as NEEDS_HUMAN_INPUT. Do not guess.
- Writing 5+ sentence technical answers -> Technical answers should be concise. 2-3 sentences. If the topic needs more explanation, it belongs in a dedicated product section, not the FAQ.
- Using "our advanced formula" or "carefully crafted" without specifics -> These are marketing filler phrases. Replace with actual formulation details.
