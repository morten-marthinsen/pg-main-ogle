# 2.5 — PDP Accordion FAQ Writer

**Skill:** LP-12-faq-objections
**Position:** Layer 2, Microskill 5
**Purpose:** Write the FAQ copy for PDP pages in accordion-ready format with full UX specification. PDP FAQs MUST use the accordion pattern (tap to expand, one open at a time) per Baymard best practices. This microskill produces both the FAQ copy AND the accordion UX spec in a single pass. It runs INSTEAD of 2.2 + 2.3 + 2.4 for Type B and Hybrid pages — all FAQ writing consolidated into one accordion-formatted output.
**Effort Level:** high
**Dependencies:** `pdp-faq-curation.md` (1.4), `answer-strategies.md` (1.3), `packages-loaded.md` (0.1), `specimens-loaded.md` (0.3)

## Pre-Writing Protocol

**Step 1: Verify this is a PDP page.** This microskill runs ONLY for Type B and Hybrid pages. If Type A, STOP — use 2.2 + 2.3 + 2.4 instead.

**Step 2: Load the Baymard FAQ rules into active context:**
- Curated top 5-7 questions ONLY
- Accordion pattern (tap to expand, one open at a time)
- Factual, direct, helpful — NOT persuasive
- If answerable in 1 sentence, answer in 1 sentence
- No marketing language in FAQ answers

**Step 3: State the anti-marketing commitment:**
Before writing, state: "PDP FAQ answers are factual and helpful, not persuasive. I will NOT use marketing language, benefit stacking, or emotional amplification in these answers. The reader is looking for facts, not a sales pitch."

**Step 4: Review anti-slop word list** from LP-12-AGENT.md. Commit to zero instances.

## Execution Logic

### Part A: FAQ Copy Writing

For each of the 5-7 curated items from `pdp-faq-curation.md`, write the Q&A pair:

1. **Answer structure for PDP accordion FAQ:**

   ```
   [Direct Answer] — 1 sentence. Answers the question. Period.
   [Supporting Detail] — 0-2 sentences. Only if needed for clarity.
   [Contextual Link] — Optional. If another page section answers this more fully, link to it.
   ```

   **Critical difference from Type A FAQ:** PDP answers are SHORTER and FACTUAL. No confidence builders. No reframes. No persuasion. The reader is scanning, not reading.

2. **The "1-sentence test":** For every question, ask: "Can this be answered completely in 1 sentence?" If yes, answer in 1 sentence. Do NOT pad to 2-3 sentences for the sake of length.

   | Question | 1-Sentence Answer? | Example |
   |----------|--------------------|---------|
   | "Is this gluten-free?" | YES | "Yes — Magnesium Breakthrough is gluten-free, soy-free, and non-GMO." |
   | "How do I take it?" | YES | "Take 2 capsules before bed with or without food." |
   | "What forms of magnesium are included?" | NO (needs list) | "Magnesium Breakthrough contains all 7 forms: bisglycinate, malate, citrate, taurate, orotate, sucrosomial, and chelate — each absorbed differently for full-spectrum coverage." |
   | "Does it actually work?" | NO (needs proof) | "In a survey of 2,100+ customers, 87% reported improved sleep quality within 14 days. The 7-form blend ensures your body can actually absorb and use the magnesium — unlike single-form supplements where most passes through unabsorbed." |

3. **No marketing language in answers.** Baymard research finds that marketing language in FAQ answers DECREASES trust. The FAQ section is the one place on the page where the reader expects unvarnished facts.

   | Forbidden in PDP FAQ Answers | Acceptable Alternative |
   |-----------------------------|----------------------|
   | "Our revolutionary formula..." | "The formula contains..." |
   | "You'll love the incredible results..." | "87% of customers reported..." |
   | "This game-changing blend..." | "This 7-form blend..." |
   | "Experience the power of..." | "[Product] delivers..." |
   | "Transform your sleep with..." | "Most customers notice improved sleep within 14 days." |
   | "Unlike anything else on the market..." | "Contains all 7 forms — most competitors include 1-2." |

4. **Contextual linking.** FAQ answers CAN link to relevant page sections when a fuller answer exists elsewhere on the page:
   - "See our full ingredient breakdown" -> anchor link to ingredients section
   - "Read what customers are saying" -> anchor link to reviews section
   - "View our shipping policy" -> anchor link or dedicated page link
   - Format: plain text link, not button. Example: "Learn more about our ingredients."

5. **Per-answer quality gate:**

   ```yaml
   PDP-ANSWER-CHECK:
     question_id: "[from pdp-faq-curation.md]"
     answer_is_factual: "[Y/N -- must be Y]"
     marketing_language_present: "[Y/N -- must be N]"
     could_be_shorter: "[Y/N -- if Y, shorten it]"
     first_sentence_is_direct_answer: "[Y/N -- must be Y]"
     product_specific: "[Y/N -- must be Y]"
     sentence_count: "[1-3 — if > 3, it is too long for PDP accordion]"
     contextual_link_appropriate: "[Y/N/N/A]"
     ai_telltales: "[Y/N -- must be N]"
     sycophantic_opener: "[Y/N -- must be N]"
   ```

### Part B: Accordion UX Specification

After all FAQ copy is written, produce the accordion UX spec:

1. **Accordion behavior rules:**

   | Rule | Specification |
   |------|--------------|
   | Default state | All items CLOSED (none expanded by default) |
   | Expand behavior | Tap/click question row -> answer expands below |
   | Multi-open | NO — opening a new item closes the previously open item |
   | Visual indicator | Chevron/arrow icon on the right side of each question row, rotates 180 degrees on expand |
   | Touch target | ENTIRE question row is tappable (not just the chevron) — minimum 44px height |
   | Animation | Smooth expand/collapse (200-300ms ease-in-out) |
   | Keyboard navigation | Enter or Space to toggle, Tab to move between items, Escape to close current |

2. **Typography specification:**

   | Element | Spec |
   |---------|------|
   | Question text | Bold, 16px minimum (18px recommended for mobile), dark color (not gray) |
   | Answer text | Regular weight, 14-16px, sufficient line-height (1.5-1.6), standard body color |
   | Chevron icon | 20-24px, same color as question text, rotates on toggle |
   | Item separator | 1px border-bottom between items, light gray (#E5E7EB or equivalent) |
   | Container padding | 16-24px horizontal, 12-16px vertical per item |

3. **Accessibility specification:**

   | Requirement | Implementation |
   |-------------|---------------|
   | `aria-expanded` | On each question button: `true` when open, `false` when closed |
   | `aria-controls` | Each question button references the ID of its answer panel |
   | `role="region"` | Each answer panel has `role="region"` with `aria-labelledby` referencing the question |
   | Keyboard | Enter/Space toggles, Tab navigates between questions |
   | Screen reader | Question reads as button with expanded/collapsed state |
   | Focus indicator | Visible focus ring on keyboard navigation (2px solid, brand color or #2563EB) |

4. **Search FAQ specification** (conditional — only if 7 items):

   If the curated FAQ has exactly 7 items, include a search bar specification:

   | Element | Spec |
   |---------|------|
   | Position | Above the accordion, below the section header |
   | Placeholder text | "Search FAQs..." |
   | Behavior | Real-time filter — hides non-matching items as user types |
   | Match logic | Search question text only (not answer text) — simple substring match |
   | No results | Show "No matching questions found. Contact us at [support email]." |
   | Minimum items to show search | 7 |

   If fewer than 7 items, omit the search bar — it adds unnecessary complexity for a short list.

5. **Positioning rule:** PDP FAQ sits AFTER the reviews section, before the final CTA. If no reviews section exists, it sits after the social proof section.

## Quality Criteria
- Exactly 5-7 FAQ items (Baymard curated rule — not 4, not 8)
- All 3 mandatory objection categories represented in the 5-7 items
- Every answer is factual and direct — zero marketing language
- Every 1-sentence-answerable question IS answered in 1 sentence (no padding)
- No answer exceeds 3 sentences
- Zero sycophantic openers, zero AI telltales
- Accordion UX spec is complete (behavior, typography, accessibility, conditional search)
- Contextual links are accurate (reference real page sections from section-sequence.json)
- Source attribution carried forward from pdp-faq-curation.md
- FAQ header is clean and PDP-appropriate (not Type A conversational headers)

## Failure Modes
- Writing persuasive/marketing answers instead of factual ones -> Baymard research: marketing language in FAQ answers DECREASES trust. Rewrite. "Our revolutionary 7-form blend" becomes "Contains all 7 forms of magnesium." The reader came to the FAQ for facts, not a sales pitch.
- Padding 1-sentence answers to 2-3 sentences -> If "Yes, it's gluten-free" is the complete answer, do not add "We take dietary restrictions seriously" and "Your health is our priority." Answer the question. Stop.
- Using open-by-default or inline text format -> PDP FAQ MUST use accordion. All items closed by default. One open at a time. No exceptions — this is a Baymard compliance requirement.
- Missing accessibility specification -> Accordion without aria-expanded, aria-controls, and keyboard navigation fails WCAG 2.1 AA. The UX spec is incomplete without the accessibility section.
- Omitting the search bar when there are 7 items -> 7 items triggers the search bar specification. If exactly 7, include it. If fewer than 7, omit it.
- Writing Type A-style answers (confidence builders, reframes, 4-sentence answers) -> PDP accordion answers are shorter, more factual, less persuasive than Type A answers. This is not the same skill as 2.2. Recalibrate to PDP voice.
- Using this microskill for Type A pages -> STOP. Type A pages use 2.2 + 2.3 + 2.4 with the full 10-12 item answer set. This microskill is for PDP pages only.
- Inventing contextual links to sections that do not exist -> Check section-sequence.json before adding any anchor link. If the section is not in the page architecture, do not link to it.
