# 2.5 — Problem/Solution/Logic Writer

**Skill:** PDP-07-btf-section-writer
**Position:** Layer 2, Microskill 5
**Purpose:** Write the consolidated Problem, Solution, and Logic sections — the education block of the PDP. PDP-04 may specify these as 1, 2, or 3 separate sections. When consolidated, the flow is: agitation (The Problem) -> paradigm shift (The Solution) -> mechanism diagram (Why It Works). When separate, each is a standalone module. This writer also handles The Proof (Data) section if included in the sequence.
**Effort Level:** high
**Dependencies:** `per-section-briefs.md` (1.1 — the_problem/the_solution/why_it_works/the_proof briefs), `data-validation.md` (1.2), `pdp-reference-loaded.md` (0.3 — Sections 5.6, 5.7, 5.8, 5.16)
**Output:** `section-problem-solution-logic.md`

## Execution Logic

**Step 1:** Determine consolidation mode from PDP-04's sequence:
- If PDP-04 lists `the_problem`, `the_solution`, AND `why_it_works` as separate sections -> write 3 standalone sections
- If PDP-04 lists only `the_solution` + `why_it_works` (problem omitted) -> write 2 sections
- If PDP-04 consolidated into fewer sections -> write as consolidated block
- If `the_proof` is also in the queue -> write as a fourth section or appended to Why It Works

**Step 2: THE PROBLEM (if included)**

Write the agitation section:
- **Pattern:** Problem amplification — make the reader feel the problem deeply
- **Structure:**
  - Bold problem headline that names the issue directly
  - 3-5 symptom/frustration bullets using audience pain language
  - 1-2 sentences on why the problem persists (conventional approach fails)
  - NO solution introduced here — pure agitation
- **Audience quotes:** Weave in pain quotes from research where available
- **Word count:** Match PDP-04 target (+/-10%)
- **Tone:** Empathetic, not dramatic. Understand the problem, don't exploit it.

**Step 3: THE SOLUTION (if included)**

Write the paradigm shift section:
- **Pattern:** 3-part structure from NLS:
  1. The conventional approach (what most people try — and why it falls short)
  2. The real cause (paradigm shift — the actual root of the problem)
  3. The new solution (how this product addresses the real cause)
- **Structure:**
  - Bold solution headline
  - 2-3 sentences on conventional approach failure
  - 1-2 sentences naming the real cause (paradigm shift moment)
  - 2-3 sentences on how this product uniquely addresses the real cause
- **Word count:** Match PDP-04 target (+/-10%)
- **Tone:** Confident, not aggressive. Educate, don't attack.

**Step 4: WHY IT WORKS (if included)**

Write the mechanism/logic section:
- **Pattern:** 4-step flow diagram: problem -> ingredient/component -> mechanism -> felt benefit
- **Structure:**
  - Bold mechanism headline
  - 4-step logical chain, each step as a labeled node:
    1. **The Problem:** [What goes wrong — e.g., "Cortisol spikes disrupt sleep cycles"]
    2. **The Ingredient:** [What addresses it — e.g., "Ashwagandha (300mg KSM-66)"]
    3. **The Mechanism:** [How it works — e.g., "Modulates HPA axis to normalize cortisol"]
    4. **The Benefit:** [What you feel — e.g., "Fall asleep faster, stay asleep longer"]
  - Optional supporting study reference for the mechanism
- **Visual direction:** Diagram, infographic, or 4-panel flow (not paragraph text)
- **Word count:** Match PDP-04 target (+/-10%)

**Step 5: THE PROOF (if included)**

Write the data/evidence section:
- **Pattern:** Study circles, survey data, graphs
- **Structure:**
  - Bold evidence headline
  - Per study/data point:
    - Study citation (journal, year, design)
    - Sample size
    - Key result (with specific number)
    - Visual direction (circle graphic, bar chart, etc.)
  - Minimum 1 data point, ideally 2-3
- **Word count:** Match PDP-04 target (+/-10%)

**Step 6:** If writing separate sections, ensure each is independently comprehensible:
- The Problem section must make sense without reading The Solution
- The Solution section must briefly name the problem before shifting to solution
- Why It Works must be self-contained (brief problem context + mechanism)

**Step 7:** Write `section-problem-solution-logic.md` with all included sections.

## Quality Criteria
- Only sections present in PDP-04's sequence are written
- Each section is self-contained (module independent)
- Problem section uses empathetic tone, not exploitative
- Solution section follows the 3-part paradigm shift pattern
- Why It Works uses 4-step logical chain, not paragraph prose
- The Proof uses specific data points (numbers, sample sizes, journals)
- Audience pain quotes woven into Problem section when available
- Word counts within +/-10% per section
- Zero generic AI language

## Failure Modes
- Writing The Problem with aggressive/fear-based tone -> Rewrite with empathetic framing. Understand, don't exploit.
- Solution section attacks competitors by name -> Remove brand names. Use "conventional approaches" or "typical solutions."
- Why It Works written as paragraph instead of 4-step flow -> Restructure as labeled steps/diagram spec.
- The Proof has no actual data points -> HALT. This section should not exist without data. Flag PDP-04 mismatch.
- Sections cross-reference each other -> Remove references. Add brief inline context instead.
- All 4 sections written when PDP-04 only included 2 -> Remove unauthorized sections. Follow PDP-04 sequence exactly.
