# 2.3 — Ingredients Section Writer

**Skill:** PDP-07-btf-section-writer
**Position:** Layer 2, Microskill 3
**Purpose:** Write the What's in It (Ingredients) section — a visual-first ingredient deep-dive using mandatory "X for Y" naming (e.g., "Ashwagandha for Stress Relief"). Each ingredient gets a visual card with compelling imagery, caveman benefit name, and a 1-2 sentence description connecting benefit to mechanism. Layout is vertical and expanded, NOT accordion-hidden.
**Effort Level:** high
**Dependencies:** `per-section-briefs.md` (1.1 — whats_in_it brief), `data-validation.md` (1.2 — GREEN or YELLOW), `pdp-reference-loaded.md` (0.3 — Section 5.4 template)
**Output:** `section-whats-in-it.md`

---

## Input Schema

```yaml
input:
  section_brief: "[whats_in_it brief from per-section-briefs.md]"
  word_count_target: "[from PDP-04 — note: per-ingredient target x ingredient_count]"
  proof_density: "[from PDP-04 — medium per ingredient: 1 study or 'clinically studied' badge]"
  pdp_reference: "[Section 5.4 template from pdp-reference-loaded.md]"
  ingredient_list: "[from brief — name, dosage, benefits, mechanism]"
  ingredient_count: "[number of ingredients]"
```

---

## Execution Logic

**Step 1:** Read the What's in It template from PDP-BEST-PRACTICES-REFERENCE.md Section 5.4. Key pattern: visual-first ingredient cards, "X for Y" naming, vertical expanded layout.

**Step 2:** For each ingredient in the list, create an ingredient card:

**Ingredient Card Structure:**
1. **"X for Y" Name:** `[Ingredient Name] for [Felt Benefit]`
   - X = ingredient name (common name, not chemical name unless well-known)
   - Y = felt benefit in 1-3 words (what the user notices)
   - Examples: "Ashwagandha for Stress Relief," "Vitamin D for Immune Support," "L-Theanine for Calm Focus"

2. **Dosage:** Amount per serving (e.g., "300mg per capsule")

3. **Microscript Benefit:** 1-2 sentences connecting the ingredient to the felt benefit via its mechanism.
   - Pattern: "[Ingredient] [mechanism verb] [biological process] to help [felt benefit]."
   - Example: "Ashwagandha reduces cortisol levels to help your body handle daily stress without the crash."
   - Never: "Ashwagandha is a powerful adaptogen." (vague, no mechanism, no felt benefit)

4. **Proof Badge:** Study reference or "clinically studied" badge per ingredient (if available from brief)
   - Specific: "Supported by 2019 Journal of Nutrition study (n=120)"
   - General: "Clinically Studied" badge (only if no specific study data exists)

5. **Visual Direction:** Compelling visual that brings ingredient to life
   - Options: botanical photography, molecular structure visualization, powder texture, whole food source
   - Must feel premium and intentional, not stock photo

**Step 3:** Order ingredients by importance:
- Lead ingredient (highest dose or most recognized) first
- Supporting ingredients in order of relevance
- Minor/trace ingredients last (or grouped as "supporting blend")

**Step 4:** Write section header that frames the ingredient story:
- Not: "Our Ingredients" (generic)
- Better: "What's Inside Every Serving" or "The Formula Behind [Product Name]"

**Step 5:** Ensure layout direction is vertical and expanded:
- All primary ingredients visible (NOT hidden behind accordion)
- If 8+ ingredients: show top 5-6 expanded + "Show All [X] Ingredients" toggle for the rest
- Baymard: vertical stacking > horizontal carousels for ingredients

**Step 6:** Confirm no carousel Slide 3 (Ingredients) content is repeated verbatim. Carousel showed ingredients orbiting product with 1-word benefits. BTF section EXTENDS with full cards, dosages, microscripts, and study references.

**Step 7:** Verify total word count against target (+/-10%). Note: PDP-04 set total target = per-ingredient target x ingredient_count.

**Step 8:** Write `section-whats-in-it.md`.

## Quality Criteria
- EVERY ingredient uses "X for Y" naming pattern — no exceptions
- Microscript benefits describe mechanism -> felt benefit (not vague claims)
- Dosages included per ingredient
- Proof badges present where study data exists in brief
- Layout is vertical expanded (not accordion-hidden for primary ingredients)
- No carousel Slide 3 content repeated verbatim
- Word count within +/-10% of PDP-04 target per ingredient
- Zero generic AI language ("powerful," "potent," "cutting-edge")

## Failure Modes
- Ingredient without "X for Y" naming -> HALT. Every ingredient must have a felt benefit pairing.
- Microscript says "powerful adaptogen" without mechanism -> Rewrite with specific mechanism: "reduces cortisol levels"
- Dosage missing for ingredient -> Use "proprietary blend" notation and flag as data gap
- No proof data for any ingredient -> Use "Clinically Studied" badge minimum; flag as proof gap
- Horizontal carousel layout specified -> HALT. Baymard: vertical stacking mandatory for ingredients.
- Word count exceeds target by >10% -> Condense microscripts. 1 sentence per ingredient if needed.
