# 2.1 — Product Highlights Writer

**Skill:** PDP-07-btf-section-writer
**Position:** Layer 2, Microskill 1
**Purpose:** Write the Product Highlights (TLDR) section — the first BTF section a PDP visitor sees after scrolling past the buy box. This section answers the top 3-5 product questions as expandable tiles, giving impatient visitors fast answers that build confidence without requiring them to read the entire page.
**Effort Level:** high
**Dependencies:** `per-section-briefs.md` (1.1 — product_highlights brief), `data-validation.md` (1.2 — GREEN or YELLOW), `pdp-reference-loaded.md` (0.3 — Section 5.1 template)
**Output:** `section-product-highlights.md`

---

```yaml
input:
  section_brief: "[product_highlights brief from per-section-briefs.md]"
  word_count_target: "[from PDP-04 via section-sequence-loaded.md]"
  proof_density: "[from PDP-04 — light: badge/icon proof only]"
  pdp_reference: "[Section 5.1 template from pdp-reference-loaded.md]"

```

## Execution Logic

**Step 1:** Read the Product Highlights template from PDP-BEST-PRACTICES-REFERENCE.md Section 5.1. Key pattern: digestible caveman bits of copy addressing top 3-5 questions/issues upfront.

**Step 2:** Select 3-5 tile topics from product data. Prioritize these categories (from NLS best practices):
1. **"What's in It"** — Show supplement/nutrition facts panel or key ingredient summary
2. **"How It Helps"** — 5-7 short benefit bullets with microscripts
3. **"Why It Works"** — 1-sentence mechanism + simple graphic reference
4. **"Shipping & Returns"** — Free S&H emphasis + expected delivery speed
5. **"Our Guarantee"** — Key benefit to experience within guarantee period

Adjust tiles based on product type. Not all 5 are mandatory — select the 3-5 most relevant for this specific product and audience.

**Step 3:** For each tile, write:
- **Tile Question/Header:** Short, clear question format ("What's in It?") or direct label ("Key Ingredients")
- **Expandable Answer:** 2-4 short sentences or a bullet list answering the question. Mix hard data (facts panel, ingredient counts) with good copy (personality, not boring facts).
- **Icon/Visual Direction:** Suggest icon or visual for the collapsed tile state

**Step 4:** Ensure the section:
- Answers the top questions that cause scroll-past visitors to bounce
- Mixes data (nutrition panel, ingredient counts) with personality (not boring labels)
- Does NOT repeat carousel slide text verbatim
- Is scannable even in collapsed tile state (headers alone convey value)

**Step 5:** Verify word count against target (+/-10%).

**Step 6:** Write `section-product-highlights.md`.

## Quality Criteria
- 3-5 tiles covering the most common visitor questions
- Each tile has: header, icon direction, collapsed preview, expanded content
- Expanded content is 2-4 sentences or a bullet list (not paragraphs)
- Mix of data (facts, numbers) and personality (good copy, not clinical)
- No carousel content repeated verbatim
- Headers alone convey value (scannable in collapsed state)
- Word count within +/-10% of PDP-04 target
- Zero generic AI language

## Failure Modes
- Fewer than 3 tiles -> HALT. Minimum is 3 tile topics. If brief lacks data for 3, escalate as data gap.
- Tiles read like marketing brochure copy -> Rewrite in plain, direct language. "What's in It?" not "Discover Our Premium Ingredients"
- Expanded content exceeds 4 sentences per tile -> Condense. Tiles are TLDR — keep it tight.
- Section header uses generic AI language ("Unlock the Power of...") -> Rewrite with plain language
- All tiles are benefit-focused with no data tile -> Add at least 1 data tile (ingredients, shipping, guarantee)
