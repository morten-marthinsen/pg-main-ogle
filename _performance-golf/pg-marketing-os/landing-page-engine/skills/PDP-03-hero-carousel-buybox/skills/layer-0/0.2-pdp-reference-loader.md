## Execution Logic

**Step 1:** Verify `input-verification.md` exists and page type is confirmed as type_b or hybrid. If not → HALT.

**Step 2:** Read `PDP-BEST-PRACTICES-REFERENCE.md`. Extract the following sections:

**Step 3:** Extract Section 2 — 10-Thumbnail Story Architecture:
- All 10 slide positions with story position name, persuasion function, and required elements
- Per-slide copy direction standards
- Per-slide visual direction standards
- Carousel UX requirements (thumbnails > dots, pinch-to-zoom, in-scale images, gallery overlay)

**Step 4:** Extract Section 3 — Facts Panel:
- Panel types (Supplement Facts, Nutrition Facts, Drug Facts)
- Quality/diet icons list and placement rules
- Panel positioning relative to carousel

**Step 5:** Extract Section 4 — Buy Box Architecture:
- All 10 buy box components with Baymard requirements
- Variant selector: exposed chips (not dropdowns)
- Subscription: selectable tiles (not pre-selected)
- Quantity: stepper (not dropdown)
- ATC: full-width, standard label, state change
- Post-click: slide-out mini-cart (not cart redirect)
- Micro-trust: 3-4 signals below ATC
- Shipping proximity rule
- Express payment layout

**Step 6:** Extract Baymard/NN-g statistics referenced in the document:
- Thumbnail dot miss rate (50-80%)
- Pinch-to-zoom fail rate (40%)
- Dropdown friction increase (60%)
- BTF overlooked rates (vertical 27% vs horizontal 8%)
- Any other specific conversion statistics

**Step 7:** If PDP-BEST-PRACTICES-REFERENCE.md is not found, fall back to AGENT.md tables (10-Thumbnail Story Architecture table and Buy Box 10-Point Audit). Document the fallback.

**Step 8:** Write `pdp-reference-load.md`.
