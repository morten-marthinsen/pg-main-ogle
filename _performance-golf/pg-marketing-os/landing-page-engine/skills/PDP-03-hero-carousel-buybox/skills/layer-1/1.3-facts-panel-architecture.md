# 1.3 — Facts Panel Architecture

**Skill:** PDP-03-hero-carousel-buybox
**Position:** Layer 1, Microskill 3
**Purpose:** Design the facts panel (Supplement Facts, Nutrition Facts, or Drug Facts) positioned below the carousel. Includes panel type selection, quality/diet icon specification, and layout direction.
**Effort Level:** medium
**Dependencies:** 0.1 input-verification.md, 0.2 pdp-reference-load.md
**Output:** `facts-panel.md`

---

## Input Schema

```yaml
input:
  input_verification: "[path to input-verification.md]"
  pdp_reference_load: "[path to pdp-reference-load.md]"
  product_category: "[extracted from input-verification.md]"
```

---

## Execution Logic

**Step 1:** Determine panel type based on product category:

| Product Category | Panel Type | Regulatory |
|-----------------|-----------|-----------|
| Dietary supplement | Supplement Facts | FDA 21 CFR 101.36 |
| Food / beverage | Nutrition Facts | FDA 21 CFR 101.9 |
| OTC drug | Drug Facts | FDA 21 CFR 201.66 |
| Skincare / cosmetic | Ingredient List | FDA 21 CFR 701.3 |
| Non-ingestible (electronics, apparel) | N/A — skip facts panel | N/A |

If product is non-ingestible → output `facts-panel.md` noting "Not applicable — product does not require facts panel" and specify what replaces it (spec sheet, materials list, etc.).

**Step 2:** Specify panel content structure:
- **Supplement Facts:** Serving size, servings per container, per-serving amounts, % Daily Value, other ingredients, allergen statement
- **Nutrition Facts:** Calories, fat, sodium, carbs, protein, vitamins/minerals
- **Drug Facts:** Active ingredient, purpose, uses, warnings, directions, other information, inactive ingredients

**Step 3:** Identify quality and diet icons applicable to this product:

| Icon Category | Examples | When to Include |
|--------------|---------|----------------|
| Diet compatibility | Gluten-Free, Vegan, Keto, Paleo, Non-GMO | Product meets certification |
| Allergen-free | Soy-Free, Dairy-Free, Nut-Free | Product tested/certified |
| Quality certification | GMP Certified, NSF Certified, USP Verified | Product holds certification |
| Sourcing | Made in USA, Organic, Wild-Caught | Product verified |
| Testing | Third-Party Tested, Lab Verified | Product has documentation |

**Step 4:** Select 4-8 applicable icons from the product brief data. Do NOT fabricate certifications — only include icons the product actually holds.

**Step 5:** Specify icon layout:
- Position: Below facts panel, single horizontal row
- Style: Small circular or shield icons, 24-32px, muted color (not competing with buy box)
- Spacing: Even distribution across container width
- Mobile: Wrap to second row if needed, maintain icon size

**Step 6:** Specify facts panel positioning:
- Desktop: Below carousel, left-aligned with carousel column
- Mobile: Full-width below carousel, collapsible/expandable (tap to view full panel)
- Default state on mobile: Collapsed with "View Supplement Facts" link

**Step 7:** Write `facts-panel.md`.

---

## Quality Criteria
- Panel type correctly matched to product category
- Only verified certifications included (no fabricated icons)
- 4-8 icons specified (not 0, not 15)
- Mobile behavior specified (collapsible with expand trigger)
- Positioning relative to carousel and buy box documented
- Allergen statement included if product is ingestible
- File exists after execution

## Failure Modes
- Product category unclear → Ask: is it ingestible? If yes, default to Supplement Facts for supplements.
- No certification data in brief → Include panel with "0 verified icons — pending human confirmation" note
- Non-ingestible product → Write N/A file with alternative (spec sheet, materials list)
- Facts panel data incomplete (missing serving size) → Flag specific gaps for human input
- input-verification.md not found → HALT, run 0.1 first
