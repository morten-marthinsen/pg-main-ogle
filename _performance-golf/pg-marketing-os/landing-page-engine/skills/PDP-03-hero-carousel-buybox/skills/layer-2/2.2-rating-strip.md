# 2.2 — Rating Strip Design

**Skill:** PDP-03-hero-carousel-buybox
**Position:** Layer 2, Microskill 2
**Purpose:** Specify the rating strip component (star display + review count + anchor link to reviews section). Baymard: the rating strip MUST be an anchor link that smooth-scrolls to the review section. Static stars without link miss a critical user behavior — shoppers tap stars to jump to reviews.
**Effort Level:** low
**Dependencies:** 0.1 input-verification.md
**Output:** `rating-strip.md`

---

## Input Schema

```yaml
input:
  input_verification: "[path to input-verification.md]"
  product_rating: "[average rating, or 'not yet available']"
  review_count: "[number of reviews, or 'not yet available']"
```

---

## Execution Logic

**Step 1:** Extract product rating and review count from brief data. If not available, specify placeholder behavior.

**Step 2:** Define star display:
- **Star count:** 5 stars, partially filled to match average rating
- **Fill method:** Precise fill (e.g., 4.3 stars = 4 full + 1 at 30% fill), NOT rounded to nearest half
- **Star size:** 16-18px
- **Star color:** Gold/amber (#FFC107 or brand-appropriate)
- **Empty star:** Light gray outline

**Step 3:** Define numeric display:
- **Format:** [X.X] + "(" + [count] + " reviews)" or "[count] ratings"
- **Example:** "4.6 (2,847 reviews)"
- **Rating number:** Display to one decimal place
- **Review count:** Display with comma separators for 1,000+
- **Font:** Smaller than title (14-16px), medium weight

**Step 4:** Define anchor link behavior:
- **Entire strip is clickable** (stars + text = one tap target)
- **Click action:** Smooth-scroll to reviews section (#reviews)
- **Scroll offset:** Account for sticky header height (don't scroll review section behind sticky nav)
- **Cursor:** Pointer on hover (desktop)
- **Underline:** Review count text underlined or shows underline on hover

**Step 5:** Define placeholder state (for products with no reviews yet):
- **0 reviews:** Show "Be the first to review" link (scrolls to Write a Review form)
- **Do NOT show 0 stars** — hide the star display entirely for 0-review products
- **Do NOT show "No reviews yet"** — negative framing. Use positive CTA instead.

**Step 6:** Define mobile-specific rules:
- Tap target: Minimum 44x44px for the entire strip
- Stars remain same size (16-18px)
- Text may wrap to second line if needed: Stars on line 1, "(X reviews)" on line 2

**Step 7:** Write `rating-strip.md`.

## Output Schema

`rating-strip.md`:

```markdown
# Rating Strip Design — PDP-03

## Product: [product name]
## Current Rating: [X.X or "not yet available"]
## Review Count: [N or "not yet available"]

---

## Star Display
- **Stars:** 5, partially filled to match average
- **Fill:** Precise percentage fill (not rounded to half)
- **Size:** [16-18px]
- **Color:** [gold/amber hex code]
- **Empty:** Light gray outline

## Numeric Display
- **Format:** "[X.X] ([N] reviews)"
- **Decimal:** One decimal place
- **Count format:** Comma-separated for 1,000+
- **Font size:** [14-16px]
- **Weight:** Medium

## Anchor Link Behavior
- **Clickable area:** Entire strip (stars + text)
- **Action:** Smooth-scroll to #reviews section
- **Scroll offset:** [sticky header height]px
- **Desktop:** Pointer cursor, underline on hover
- **Mobile:** Full 44px tap target

## Placeholder State (0 Reviews)
- **Stars:** Hidden (not shown)
- **Text:** "Be the first to review" (anchor to Write a Review)
- **FORBIDDEN:** "No reviews yet", empty stars, "0 reviews"

## Position
- **Below:** Product title (2.1)
- **Above:** Price display (2.4)
- **Spacing:** 4-8px below title, 8-12px above price
```

---

## Quality Criteria
- Star display uses precise fill (not rounded to nearest half)
- Entire strip is an anchor link (not just text)
- Smooth-scroll behavior specified with offset
- 0-review placeholder uses positive CTA (not negative "no reviews")
- Mobile tap target >= 44px
- Position relative to other buy box components documented
- File exists after execution

## Failure Modes
- Rating data not in brief → Use placeholder state, document gap
- Review section doesn't exist yet → Still specify anchor link; PDP-05 will create the target
- Rating is below 3.5 → Still display honestly. NEVER fabricate or round up ratings.
- input-verification.md not found → HALT, run 0.1 first
