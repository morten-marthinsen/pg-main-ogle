# 2.5 — Variant Selector Design

**Skill:** PDP-03-hero-carousel-buybox
**Position:** Layer 2, Microskill 5
**Purpose:** Design the variant selector using exposed chips (NEVER dropdowns). Baymard: dropdowns add 3+ taps, hide available options, and create 60% more friction than exposed chips. Chips let users see all options at a glance and select with a single tap.
**Effort Level:** medium
**Dependencies:** 0.1 input-verification.md, 0.2 pdp-reference-load.md
**Output:** `variant-selector.md`

---

```yaml

  input_verification: "[path to input-verification.md]"
  pdp_reference_load: "[path to pdp-reference-load.md]"
  variant_types: "[size, flavor, color, count, etc.]"
  sku_count: "[number of variants]"
```

## Execution Logic

**Step 1:** Extract variant types and SKU count from `input-verification.md`.

**Step 2:** Determine variant selector type based on variant dimension:

| Variant Type | Selector Style | Example |
|-------------|---------------|---------|
| Flavor | Text chips | "Chocolate", "Vanilla", "Berry" |
| Size/Count | Text chips with quantity | "30ct", "60ct", "90ct" |
| Color | Color swatch circles | [color circle] + label |
| Strength | Text chips | "Regular", "Extra Strength" |
| Format | Text chips | "Capsules", "Powder", "Gummies" |

**Step 3:** Design chip layout:
- **Style:** Rounded rectangle or pill shape, outlined border
- **Size:** Minimum 44x44px touch target (Baymard mobile requirement)
- **Spacing:** 8px gap between chips
- **Layout:** Horizontal row, wrap to next line if more than 4-5 per row
- **Mobile:** Same layout, chips must not require horizontal scrolling

**Step 4:** Define chip states:
- **Default (unselected):** Light border, white/light background, dark text
- **Selected:** Dark/filled background (brand color), white text, subtle shadow or bold border
- **Hover (desktop):** Slight background darken or border emphasis
- **Out of stock:** Grayed out with diagonal strikethrough line, non-tappable, tooltip on hover "Currently unavailable"
- **Low stock:** Normal appearance + small text below chip "Only X left"

**Step 5:** Specify per-chip content:
- **Text:** Variant name (short — "Chocolate" not "Rich Dark Chocolate Flavor")
- **Price differential:** If variant prices differ, show price change inline. Example: "60ct (+$10)" or show updated price when selected
- **Stock status:** Per-variant stock indicator (in stock, low stock, out of stock)

**Step 6:** Specify label above chips:
- **Format:** "Flavor:" or "Size:" or "Select [variant type]:"
- **Typography:** 14px, medium weight, above chip row
- **Required:** Always show label — chips without context confuse users

**Step 7:** Specify selection behavior:
- **Default selection:** Pre-select the default/most popular variant
- **Single-select:** Only one variant selected at a time
- **On selection:** Update product title (if variant-specific), update price, update carousel image (Slide 1), update stock status
- **No selection = no ATC:** If variants exist, one MUST be selected before ATC is active

**Step 8:** Handle edge cases:
- **1 variant only:** Hide the selector entirely (no point showing 1 chip)
- **2 variants:** Show both chips, pre-select default
- **10+ variants:** Show first 6-8, "Show All [N] Options" expandable link
- **Multiple variant dimensions** (e.g., flavor AND size): Stack vertically — one row per dimension, each with its own label

**Step 9:** Write `variant-selector.md`.

## Quality Criteria
- Chips used (NEVER dropdowns) — this is non-negotiable
- All chip states defined (default, selected, hover, OOS, low stock)
- Minimum 44x44px touch target per chip
- Label above chips identifies variant type
- Per-variant price differential shown if prices vary
- Out-of-stock handling specified (grayed + strikethrough, non-tappable)
- Selection behavior updates all related components (title, price, carousel)
- File exists after execution

## Failure Modes
- Dropdown specified → HALT. Redesign with chips. This is non-negotiable per Baymard.
- No variant data in brief → If product has only 1 SKU, write "N/A — single SKU, selector hidden"
- 20+ variants → Implement "Show All" expandable. Do NOT create horizontal scroll.
- Chips too small for mobile → Increase to minimum 44x44px. No exceptions.
- input-verification.md not found → HALT, run 0.1 first
