# 2.9 — Express Payment Layout

**Skill:** PDP-03-hero-carousel-buybox
**Position:** Layer 2, Microskill 9
**Purpose:** Specify the express payment section below the ATC button — Apple Pay, Google Pay, Shop Pay, PayPal, etc. positioned with an "OR" divider and displayed in a single row of payment logos. This is a trust and convenience signal, not a primary conversion path.
**Effort Level:** low
**Dependencies:** 0.1 input-verification.md, 2.8 atc-button.md
**Output:** `express-payments.md`

---

## Input Schema

```yaml
input:
  input_verification: "[path to input-verification.md]"
  atc_button: "[path to atc-button.md]"
  available_payment_methods: "[list from brief, or 'standard ecomm']"
```

---

## Execution Logic

**Step 1:** Identify available express payment methods from brief. If not specified, use standard ecommerce defaults:
- Apple Pay
- Google Pay
- Shop Pay (if Shopify)
- PayPal
- Amazon Pay (if applicable)

**Step 2:** Design "OR" divider:
- **Layout:** Horizontal line with "OR" text centered
- **Style:** Thin gray line (1px), "OR" in small caps or uppercase, gray text, 12-14px
- **Spacing:** 12-16px above and below the divider
- **Full width:** Same width as ATC button

**Step 3:** Design express payment button row:
- **Layout:** Single horizontal row of payment buttons/logos
- **Button style:** Each payment method's official button style (dark Apple Pay, yellow PayPal, etc.)
- **Button size:** Standard payment button height (40-44px), width auto or equal-split
- **Spacing:** 8px gap between buttons
- **Max buttons per row:** 3-4 (if more, wrap to second row)
- **Mobile:** Stack 2 per row if more than 3 methods

**Step 4:** Specify button behavior:
- Each button triggers its respective express checkout flow
- Express checkout bypasses cart → goes directly to payment method's checkout
- Product, variant, and quantity selections carry through to express checkout
- If no variant selected → express buttons disabled (same logic as ATC)

**Step 5:** Specify platform-specific rendering:
- Apple Pay only shows on Safari / iOS devices
- Google Pay only shows on Chrome / Android
- If no express payment methods are supported on user's device → hide entire section (no empty space)

**Step 6:** Write `express-payments.md`.

---

## Quality Criteria
- OR divider separates ATC from express payments
- Official payment button styles used (not custom designs)
- Device-conditional rendering specified (Apple Pay on iOS only, etc.)
- Single row layout (not stacked vertically unless 4+ methods)
- Express buttons carry product selections through to checkout
- Hidden when no methods supported (no empty space)
- File exists after execution

## Failure Modes
- No payment method data in brief → Use standard defaults (Apple Pay, Google Pay, Shop Pay, PayPal)
- Express payments placed above ATC → HALT. Must be below with OR divider.
- Custom button designs → Replace with official brand buttons
- Missing OR divider → HALT. Must visually separate primary ATC from express options.
- atc-button.md not found → HALT, run 2.8 first
