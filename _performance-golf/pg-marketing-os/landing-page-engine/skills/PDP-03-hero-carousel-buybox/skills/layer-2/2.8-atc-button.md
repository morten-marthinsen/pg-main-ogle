# 2.8 — ATC Button Spec

**Skill:** PDP-03-hero-carousel-buybox
**Position:** Layer 2, Microskill 8
**Purpose:** Specify the Add to Cart button — full-width, standard label, state changes, and post-click slide-out mini-cart behavior. Baymard: the ATC button is the final conversion element. Standard "Add to Cart" outperforms creative labels. Post-click must NOT redirect to cart page — slide-out mini-cart keeps user on PDP.
**Effort Level:** medium
**Dependencies:** 0.1 input-verification.md, 0.2 pdp-reference-load.md
**Output:** `atc-button.md`

## Execution Logic

**Step 1:** Define button label — standard text only:
- **Default:** "Add to Cart"
- **With subscription selected:** "Subscribe & Save" or "Add to Cart" (both acceptable, brief determines)
- **FORBIDDEN creative labels:** "Get Yours Now", "Buy Now & Save", "Unlock Your Discount", "Start Your Journey", "Claim Your Bottle"
- **Rationale:** Baymard testing shows standard "Add to Cart" reduces cognitive friction. Users expect this exact label.

**Step 2:** Define button visual design:
- **Width:** Full-width of buy box container (100%)
- **Height:** 48-56px (generous touch target)
- **Color:** Primary brand color (high contrast against background)
- **Text:** White, 16-18px, bold/semibold, centered
- **Shape:** Rounded corners (4-8px radius), not fully rounded pill on desktop
- **Shadow:** Subtle drop shadow for depth (optional)

**Step 3:** Define button states:

| State | Label | Visual | Duration |
|-------|-------|--------|----------|
| Default | "Add to Cart" | Brand color, active | — |
| Hover (desktop) | "Add to Cart" | Slightly darker, cursor:pointer | — |
| Pressed | "Add to Cart" | Darker + scale down 2% | 100ms |
| Loading | "Adding..." | Spinner icon, muted color | 500-1500ms |
| Success | "Added!" + checkmark | Green background | 2 seconds |
| Return to default | "Add to Cart" | Fade back to brand color | After 2s |
| Disabled | "Select a [variant type]" | Gray, non-tappable | Until variant selected |
| Out of stock | "Sold Out — Notify Me" | Gray/muted, different action | — |

**Step 4:** Define post-click behavior — SLIDE-OUT MINI-CART:
- **Action:** On success state, slide-out mini-cart panel appears from right edge
- **Mini-cart contains:**
  - Product thumbnail
  - Product name + selected variant
  - Quantity
  - Price (line item + total)
  - "Continue Shopping" link (closes mini-cart, stays on PDP)
  - "Checkout" button (proceeds to checkout)
  - Optional: Cross-sell suggestion (1 product, "Frequently bought together")
- **Backdrop:** Semi-transparent dark overlay behind mini-cart
- **Close:** X button in mini-cart header, click on backdrop, or "Continue Shopping" link
- **FORBIDDEN:** Redirecting to /cart page. This interrupts the shopping flow.

**Step 5:** Define disabled state behavior:
- If product has variants and none is selected → button shows "Select a [Flavor/Size/etc.]"
- If product is out of stock → button shows "Sold Out — Notify Me" and triggers email capture modal on click
- If both variant and quantity are required → button disabled until both are set

**Step 6:** Define accessibility requirements:
- `role="button"` with `aria-label="Add [product name] to cart"`
- Keyboard accessible: Enter/Space to activate
- Focus ring visible (not hidden by CSS)
- Loading state announced to screen readers

**Step 7:** Write `atc-button.md`.

## Quality Criteria
- Label is standard "Add to Cart" (not creative)
- Full-width of buy box container
- All 7 states defined (default, hover, pressed, loading, success, return, disabled/OOS)
- Post-click is slide-out mini-cart (NOT cart redirect)
- Mini-cart includes "Continue Shopping" and "Checkout"
- Disabled state shows what needs to be selected
- Out-of-stock triggers email capture, not dead end
- Accessibility requirements specified
- File exists after execution

## Failure Modes
- Creative button label → HALT. Change to "Add to Cart". Non-negotiable.
- Cart page redirect → HALT. Change to slide-out mini-cart. Non-negotiable.
- No loading/success states → HALT. Users need feedback that action occurred.
- Button not full-width → HALT. Redesign to 100% width.
- input-verification.md not found → HALT, run 0.1 first
