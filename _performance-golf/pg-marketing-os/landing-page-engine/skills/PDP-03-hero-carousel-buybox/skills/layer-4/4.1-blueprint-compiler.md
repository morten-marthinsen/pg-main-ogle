# 4.1 — Blueprint Compiler

**Skill:** PDP-03-hero-carousel-buybox
**Position:** Layer 4, Microskill 1
**Purpose:** Compile all Layer 1-3 outputs into the final `pdp-above-fold-blueprint.json` package. This is the primary downstream handoff file consumed by PDP-06 (Offer/Buy Box Writer), PDP-04 (BTF Section Sequencer for above-fold context), and LP-15 (Page Assembly).
**Effort Level:** medium
**Dependencies:** All Layer 1-3 output files, all Layer 3 audit pass results
**Output:** `pdp-above-fold-blueprint.json`

## Execution Logic

**Step 1:** Verify all Layer 3 gates passed:
- 8-second test: ALL 3 PASS
- Buy box audit: >= 7.0/10
- Carousel completeness: PASS (10/10)
- Anti-slop check: PASS (0 violations)

If ANY gate failed → HALT. Do not compile blueprint from failed audits.

**Step 2:** Read all Layer 1 and Layer 2 output files.

## Output Schema

The JSON structure defined in Step 3 above.

---

## Quality Criteria
- JSON is valid and parseable
- All 10 carousel slides present with all 4 required fields each
- All 10 buy box components present with their specifications
- Audit results included (not omitted)
- No empty/null fields
- Variant selector type = "chips" (never "dropdown")
- Subscription default = "otp" (never subscription pre-selected)
- ATC post_click = "slide-out-mini-cart" (never "cart-redirect")
- File exists after execution

## Failure Modes
- Any Layer 3 gate failed → HALT. Do not compile from failed audits.
- Missing Layer 1 or 2 output file → HALT. Run missing microskill first.
- JSON is malformed → Fix syntax. Must be valid JSON.
- Fields missing from compiled JSON → Add missing fields from source files.
