# 4.1 — Benefits Package Compiler

**Skill:** LP-09-benefits-features
**Position:** Layer 4, Microskill 1
**Purpose:** Assemble all validated benefit section copy into a single structured `benefits-section-package.json` for downstream skills. LP-15 (Assembly) and LP-17 (Conversion Audit) will load this file. Every benefit element, D-F-W-B-P chain, proof gap, and validation result is compiled here.
**Effort Level:** low
**Dependencies:** All Layer 2 outputs (validated), `benefit-audit.md` (3.1), `dfwbp-audit.md` (3.2), `anti-slop-scan.md` (3.3), `benefit-section-classification.md` (1.1), `packages-loaded.md` (0.1)
**Output:** `benefits-section-package.json`

---

## Input Schema

```yaml
input:
  layer_2_outputs:
    benefit_bullets: "[benefit-bullets.md]"
    ingredient_deep_dives: "[ingredient-deep-dives.md -- if generated]"
    feature_explanations: "[feature-explanations.md -- if generated]"
    how_it_works: "[how-it-works.md -- if generated]"
    comparison_benefits: "[comparison-benefits.md -- if generated]"
  validation_outputs:
    benefit_audit: "[benefit-audit.md -- score + result]"
    dfwbp_audit: "[dfwbp-audit.md -- PASS/FAIL + gap list]"
    anti_slop_scan: "[anti-slop-scan.md -- PASS/FAIL]"
  classification: "[benefit-section-classification.md -- section map]"
  threading: "[from packages-loaded.md -- hero anchor, promise, tone]"
  project_metadata:
    project_name: "[product name]"
    page_type: "[type_a | type_b | hybrid]"
    vertical: "[vertical name]"
```

## Pre-Assembly Verification

Before assembling the package, verify all gates:

```yaml
ASSEMBLY-GATE:
  benefit_audit_score: "[X.X -- must be >=7.5]"
  dfwbp_audit: "[PASS | FAIL -- must be PASS]"
  anti_slop_scan: "[PASS | FAIL -- must be PASS]"
  all_classified_sections_have_output: "[Y/N -- must be Y]"
  all_gates_pass: "[Y/N]"

  IF all_gates_pass = N: HALT -- do not assemble. Return to failed validation.
```

---

## Assembly Logic

1. **Verify all three validation gates PASS.** If any FAIL -> HALT. Do not assemble a package with failed validation.

2. **For each classified benefit section** (from 1.1), locate the corresponding Layer 2 output and compile:
   - Section ID and name (from section-sequence.json)
   - Content type (from classification)
   - Section heading and subheading (from Layer 2 output)
   - All elements within the section (each with full D-F-W-B-P chain documented)
   - Per-element copy text (the actual rendered copy)
   - Per-element word count
   - Section total word count
   - Section closing statement (if present)

3. **Compile threading information:**
   - Hero anchor reference (from hero-section-package.json)
   - Promise consistency check (how benefit language supports the hero promise)
   - Tone match confirmation (from benefit-audit.md point 6)

4. **Compile validation block:**
   - Benefit audit score
   - D-F-W-B-P audit result
   - Anti-slop scan result
   - Proof gaps list (from dfwbp-audit.md)
   - All gates passed status

5. **Compile downstream handoff notes:**
   - What LP-14 (CTA) can use from this package
   - What LP-15 (Assembly) needs to know about placement
   - What LP-17 (Conversion Audit) should check

6. **Write `benefits-section-package.json`** following the schema from LP-09-AGENT.md.

---

## Output Schema

Follow the `benefits-section-package.json` schema defined in LP-09-AGENT.md exactly. Key fields:

```json
{
  "schema_version": "1.0",
  "skill": "LP-09-benefits-features",
  "created": "[ISO timestamp]",
  "project_name": "[product name]",
  "page_type": "[type_a | type_b | hybrid]",
  "vertical": "[vertical name]",
  "sections": [...],
  "threading": {...},
  "validation": {...},
  "downstream_handoffs": {...}
}
```

Every section must have:
- Unique section_id matching section-sequence.json
- Content type from classification
- Complete copy with heading, elements, and optional closing
- Every element with full D-F-W-B-P chain and rendered copy_text

---

## Quality Criteria
- All gates verified PASS before assembly begins
- Every classified section has a corresponding entry in the package
- Every element has a complete D-F-W-B-P chain in the output
- Copy text fields contain the actual rendered copy (not placeholders or summaries)
- Word counts are accurate
- Threading block is populated with hero anchor reference
- Validation block accurately reflects the audit results
- Package is self-contained — downstream skills can work from this file without reading Layer 2 outputs

## Failure Modes
- Assembling with a FAIL gate -> HALT. Never assemble an invalid package.
- Copy text contains the D-F-W-B-P plan instead of the actual copy -> Copy text must be the rendered, polished copy as written in Layer 2. The plan is metadata, not output.
- Missing section in package that was classified in 1.1 -> Every classified section must appear. Cross-reference the classification map.
- Proof gaps not listed in validation block -> All PROOF_GAP flags from the dfwbp-audit.md must appear in the proof_gaps array.
- Threading block empty -> Hero anchor reference is mandatory. If hero-section-package.json was not loaded, document the fallback source.
