# 0.3 — Specimen Benefits Loader

**Skill:** LP-09-benefits-features
**Position:** Layer 0, Microskill 3
**Purpose:** Load benefit/feature section specimens matched by vertical and page type. These specimens provide structural patterns and quality benchmarks for Layer 2 generation. The model must see real-world benefit copy before writing any of its own.
**Effort Level:** low
**Dependencies:** `packages-loaded.md` (0.1), `benefit-sections-identified.md` (0.2), `specimens/specimen-index.md`
**Output:** `specimens-loaded.md`

---

## Input Schema

```yaml
input:
  page_type: "[from packages-loaded.md]"
  vertical: "[from packages-loaded.md]"
  content_types_needed: "[from benefit-sections-identified.md -- which content types were classified]"
  specimen_index_path: "LandingPageEngine/specimens/specimen-index.md"
  swipe_vault_index_path: "LandingPageEngine/swipe-vault-index.md"
```

---

## Execution Logic

1. **Read specimen-index.md** to identify specimens matching the project's vertical and page type.

2. **Priority matching** (in order):
   - First: Same vertical + same page type
   - Second: Same vertical + different page type
   - Third: Different vertical + same page type (for structural patterns)
   - Fourth: Highest-value specimens regardless of match (LP-SPEC-10 BIOpt, LP-SPEC-09 LMNT)

3. **For each matched specimen**, load the raw specimen file and extract:
   - Benefit section copy (verbatim)
   - Benefit formatting pattern (bullets, deep-dives, explanations, tables)
   - Proof integration pattern (how proof is woven into benefits)
   - Word count per benefit section
   - Bullet count (if bullet format)

4. **Content-type-specific loading:**

   **If `ingredient_deep_dive` needed:**
   - Prioritize LP-SPEC-10 (BIOpt) — 23 sections with ingredient breakdowns and mechanism explanations
   - Prioritize LP-SPEC-09 (LMNT) — ingredient transparency and dosing info
   - Extract: ingredient section structure, dose display, mechanism language, citation format

   **If `feature_explanation` needed:**
   - Prioritize LP-SPEC-03 (Graziosi) — 7 individual feature sections
   - Prioritize LP-SPEC-07 (Hyros) — feature-to-benefit framing for SaaS
   - Extract: feature heading format, explanation structure, proof integration

   **If `how_it_works` needed:**
   - Prioritize LP-SPEC-01/02 (MAMG) — 5-step curriculum preview
   - Prioritize LP-SPEC-06 (Richmond Dinh) — 4-lesson structure
   - Extract: step naming convention, step-to-benefit connection, final outcome statement

   **If `comparison` needed:**
   - Prioritize LP-SPEC-10 (BIOpt) — competitor comparison table
   - Extract: table structure, criteria selection, advantage display format

   **If `benefit_bullets` needed:**
   - Prioritize LP-SPEC-09 (LMNT) — scan-optimized bullet format
   - Prioritize LP-SPEC-01/02 (MAMG) — feature ticker/bullet format
   - Extract: bullet length, verb-first pattern, icon usage, ordering

5. **Load minimum 2 specimens.** If fewer than 2 match on vertical + page type, expand to adjacent matches.

6. **If swipe-vault-index.md exists**, cross-reference for additional pattern sources.

7. **Write `specimens-loaded.md`** with extracted patterns and verbatim excerpts.

## Quality Criteria
- Minimum 2 specimens loaded with verbatim excerpts
- Specimens matched to content types needed (not random selections)
- Verbatim excerpts are actual benefit/feature section copy (not headlines, not testimonials, not CTAs)
- Pattern extraction summary provides actionable norms for generation

## Failure Modes
- Zero specimens match vertical + page type -> Expand to adjacent matches. If still zero -> Load the 2 highest-value specimens (BIOpt + LMNT for supplement, Graziosi + Hyros for SaaS) as structural references.
- Specimen file referenced in index but not found on disk -> Skip and note. Load next best match.
- Loaded specimen has no dedicated benefit section (e.g., LP-SPEC-04 Closers which is VSL-only) -> Skip. Select a different specimen with actual benefit copy.
