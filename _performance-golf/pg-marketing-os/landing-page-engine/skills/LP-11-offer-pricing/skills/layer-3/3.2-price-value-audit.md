# 3.2 — Price-Value Audit

**Skill:** LP-11-offer-pricing
**Position:** Layer 3, Microskill 2
**Purpose:** Score the complete pricing section copy against a 10-point effectiveness rubric. This is the quality gate that determines whether the pricing section is ready for package assembly. Minimum score: 7.5/10 to proceed.
**Effort Level:** medium
**Dependencies:** All Layer 2 outputs, `offer-copy-validation.md` (3.1 — must be PASS)
**Output:** `price-value-audit.md`


## Pre-Audit Gate

Before scoring, verify `offer-copy-validation.md` shows:
- Math Accuracy: PASS
- Anti-Slop: PASS
- Architecture Consistency: PASS

**If offer-copy-validation.md does NOT show overall PASS -> HALT. Do not score copy that has known errors. Fix errors first, re-validate, then audit.**

## 10-Point Pricing Section Audit

Score each point 0 (fail) or 1 (pass). Apply Type A or Type B criteria as indicated.

### Value Communication (3 points)

**Point 1: Value Before Price**
- Type A: Perceived value is fully established (value stack + "if all it did was") BEFORE the price reveal appears. The reader has encountered $[total value] in anchors before seeing the actual price.
- Type B: Clear savings are visible (crossed-out price, savings percentage, or bundle savings) alongside or immediately adjacent to the price display.
- Score: [0 | 1]
- Evidence: [quote the specific copy that establishes value before price]

**Point 2: Math Integrity**
- Every savings claim is mathematically verifiable. No rounding tricks (percentages always round down). No inflated comparisons.
- Score: [0 | 1] — automatically 1 if 3.1 Math Accuracy = PASS
- Evidence: [reference 3.1 result]

**Point 3: Distinct Value Propositions**
- Type A: Each value stack component and each "if all it did was" benefit addresses a different outcome. No overlapping benefits.
- Type B: Each pricing tier has a distinct value proposition (not just "more of the same at a discount").
- Score: [0 | 1]
- Evidence: [list the distinct propositions]

### Copy Quality (3 points)

**Point 4: Benefit-First Copy**
- Type A: Bonus descriptions lead with benefits. Value stack components open with what they DO, not what they ARE.
- Type B: Product descriptions and free gift copy lead with outcomes, not ingredients/features.
- Score: [0 | 1]
- Evidence: [quote the opening of each bonus/component — does it lead with benefit?]

**Point 5: Value Framework Execution**
- Type A: "If all it did was" framework uses 3+ distinct benefits with individually believable values. The value justification phrases are varied (not repetitive).
- Type B: Subscription copy clearly communicates the ongoing savings with both promotional and regular pricing visible.
- Score: [0 | 1]
- Evidence: [list the benefits and their value phrases, or subscription copy review]

**Point 6: Zero AI Telltales**
- No forbidden words from any of the 5 anti-slop categories appear in any pricing copy element.
- Score: [0 | 1] — automatically 1 if 3.1 Anti-Slop = PASS
- Evidence: [reference 3.1 result]

### Structural Integrity (2 points)

**Point 7: One Offer Served**
- All pricing copy drives toward a single conversion action. No competing products. No split decision points. Every element in the pricing section serves ONE purchase decision.
- Score: [0 | 1]
- Evidence: [identify the single conversion goal — what is the reader being asked to buy?]

**Point 8: Guarantee Referenced**
- The branded guarantee name and duration appear in the pricing section copy. Not a full guarantee block — just a confidence-building reference (1-2 sentences).
- Score: [0 | 1]
- Evidence: [quote the guarantee reference copy]

### Effectiveness (2 points)

**Point 9: Deal Clarity**
- Type A: The value-to-price gap is ≥3x (total perceived value is at least 3 times the actual price). The reader can mentally calculate the deal quality.
- Type B: A reader can understand the complete deal in under 10 seconds. Test: read the pricing tier copy aloud. Does it take more than 10 seconds? If yes, it fails.
- Score: [0 | 1]
- Evidence: [state the value-to-price ratio (Type A) or the 10-second assessment (Type B)]

**Point 10: Price Confidence**
- The price is presented with confidence. No excessive "just $X" or "only $X" hedging (0-1 instances acceptable, 2+ = fail). No apologetic language. The copy signals that the price is fair and the product earns it.
- Score: [0 | 1]
- Evidence: [count instances of "just" or "only" before a price. List any apologetic language found.]

## Quality Criteria
- All 10 points scored with explicit evidence (not "seems fine")
- Score calculated correctly (sum of 0/1 points)
- Failed points include specific revision instructions
- Pre-audit gate enforced (no scoring of copy with known math errors)
- Result is binary: ≥7.5 = PASS, <7.5 = FAIL

## Failure Modes
- Scoring without reading the actual copy -> Every point must reference specific text from the Layer 2 outputs. Generic assessments like "copy is good" do not constitute evidence.
- Giving a PASS to copy with math errors -> The pre-audit gate catches this. If 3.1 validation is FAIL, this audit cannot run. Period.
- Score of 7.5 given by inflating borderline points -> Each point is binary (0 or 1). There is no partial credit. If the evidence for a point is weak, it scores 0.
- FAIL result without revision instructions -> Every failed point must include a specific, actionable fix. "Improve the value communication" is not a fix. "Rewrite component 3 description to lead with the time-saving benefit instead of the format description" IS a fix.
