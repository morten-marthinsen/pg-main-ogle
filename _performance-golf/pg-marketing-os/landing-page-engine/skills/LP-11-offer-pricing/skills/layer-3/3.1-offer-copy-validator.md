# 3.1 — Offer Copy Validator

**Skill:** LP-11-offer-pricing
**Position:** Layer 3, Microskill 1
**Purpose:** Validate all Layer 2 pricing copy across three dimensions: mathematical accuracy (every number checks out), anti-slop compliance (zero AI telltales), and architectural consistency (copy matches LP-06 offer architecture). A single math error or forbidden word = FAIL.
**Effort Level:** medium
**Dependencies:** All Layer 2 outputs (`pricing-copy.md`, `value-stack-copy.md`, `bonus-copy.md`, `savings-callouts.md`, `if-all-it-did-copy.md`), `offer-architecture-loaded.md` (0.2)
**Output:** `offer-copy-validation.md`

---

input:

## Execution Logic

### Check 1: Math Accuracy Verification

1. **Extract every number from all Layer 2 outputs:** prices, percentages, per-day amounts, value anchors, savings claims, bundle calculations.

2. **For each savings percentage claim:** Recalculate independently.
   ```
   claim: "SAVE 32%"
   original: $[X]
   offer: $[Y]
   calculated: (($X - $Y) / $X) × 100 = [Z]%
   claimed: 32%
   |Z - 32| ≤ 1.0? → PASS / FAIL
   claimed > calculated? → FAIL (inflated — never round UP)
   ```

3. **For each per-day claim:** Recalculate independently.
   ```
   claim: "Less than $2.23/day"
   price: $[X]
   timeframe: [Y] days
   calculated: $X / Y = $[Z]
   Z ≤ 2.23? → PASS (the "less than" claim is valid)
   Z > 2.23? → FAIL (the per-day claim is false)
   ```

4. **For each bundle per-unit price:** Recalculate.
   ```
   claim: "$27.20/bottle"
   bundle_total: $[X]
   bundle_qty: [Y]
   calculated: $X / Y = $[Z]
   |Z - 27.20| < $0.01? → PASS / FAIL
   ```

5. **For value stack totals (Type A):** Sum all component values. Verify the total matches the claimed total.

6. **For "if all it did was" value sum:** Sum all individual benefit values. Verify the transition copy uses the correct sum.

**Math Accuracy Result: PASS (all calculations verified) or FAIL (list every error).**

### Check 2: Anti-Slop Scan

1. **Load the 5-category forbidden word list** from LP-11-AGENT.md.

2. **Scan every copy element from Layer 2:**
   - pricing-copy.md (all sections)
   - value-stack-copy.md (all component descriptions)
   - bonus-copy.md (all bonus descriptions)
   - savings-callouts.md (all callout display copy)
   - if-all-it-did-copy.md (full framework copy)

3. **Check for forbidden words and phrases** in all 5 categories:
   - Category 1: AI Telltales
   - Category 2: Vague Value Language
   - Category 3: Manipulative Price Language
   - Category 4: Unearned Comparison Anchors
   - Category 5: Passive/Hedge Pricing

4. **Check for near-misses:**
   - "Unlock" (any usage)
   - "Empower" / "empowering"
   - "Journey" (metaphorical)
   - "Seamless" / "seamlessly"
   - "Robust"
   - "Elevate" / "elevated"

**Anti-Slop Result: PASS (zero forbidden words) or FAIL (list every instance with location).**

### Check 3: Architecture Consistency

1. **Verify pricing display matches LP-06 strategy.** If LP-06 says "value-first" and 2.1 reveals price before value stack -> FAIL.

2. **Verify value stack components match LP-06 list.** Every component in LP-06's architecture should appear in the value stack copy. No components silently dropped. No components added that LP-06 didn't include.

3. **Verify bonus list matches LP-06 list.** Same check — every bonus accounted for.

4. **Verify guarantee reference matches LP-06.** The branded guarantee name in 2.1 must match LP-06's branded name exactly.

5. **Verify pricing tiers match LP-06 (Type B).** Same number of tiers, same labels, same prices.

6. **Verify ONE-OFFER compliance.** All pricing copy drives toward a single conversion action. No competing products, no split CTAs.

**Architecture Consistency Result: PASS (all checks verified) or FAIL (list every discrepancy).**

## Quality Criteria
- Every number in the pricing copy independently verified
- Every savings claim has a matching calculation
- All 5 anti-slop categories explicitly checked (each listed in output even if clean)
- Architecture comparison is element-by-element (not summary-level)
- FAIL result includes specific revision instructions, not just identification
- Overall result is PASS only if ALL THREE checks pass

## Failure Modes
- Math error found but marked "close enough" -> No. Math is exact or it fails. "Close enough" is a forbidden rationalization.
- Anti-slop scan skips one Layer 2 output -> All five output files must be scanned. The most common hiding place for AI telltales is bonus descriptions and savings callouts.
- Architecture check misses a dropped component -> Compare LP-06 component list element-by-element against value-stack-copy.md. Count them.
- Overall PASS given when one check failed -> Overall PASS requires ALL THREE checks passing. No partial passes.
