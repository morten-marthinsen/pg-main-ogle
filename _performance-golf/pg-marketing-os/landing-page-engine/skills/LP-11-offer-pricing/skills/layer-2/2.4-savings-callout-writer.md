# 2.4 — Savings Callout Writer

**Skill:** LP-11-offer-pricing
**Position:** Layer 2, Microskill 4
**Purpose:** Write every savings callout in the pricing section — crossed-out prices, percentage savings, per-day/per-unit breakdowns, bundle savings comparisons, and subscription savings copy. Every number this microskill produces must be mathematically exact. Rounding errors destroy trust.
**Effort Level:** medium
**Dependencies:** `pricing-display-confirmed.md` (1.1), `offer-architecture-loaded.md` (0.2), `pricing-copy.md` (2.1)
**Output:** `savings-callouts.md`

---

## Input Schema

```yaml
input:
  pricing_display: "[from pricing-display-confirmed.md]"
  pricing_tiers: "[from offer-architecture-loaded.md — all tiers with prices]"
  actual_price: "$[base price]"

  subscription_price: "$[subscription price if applicable]"
  bundle_prices: "[list of bundle tier prices]"

  per_day_timeframe: "[number of days for per-day calculation — from pricing-display-confirmed.md]"
  product_name: "[from packages-loaded.md]"
```

## Execution Logic

### Step 1: Identify All Savings Claims Needed

Based on the offer structure, determine which savings callouts to write:

| Savings Type | When to Write | Example |
|-------------|--------------|---------|
| Crossed-out price | Original price exists and is higher than offer price | ~~$99~~ $67 |
| Percentage savings | Any discount from original/regular price | SAVE 32% |
| Dollar savings | Discount exceeds $10 | Save $32 |
| Per-day breakdown | Product price > $30 and has a usage timeframe | Less than $2.23/day |
| Per-unit bundle savings | Multiple pricing tiers exist | $27.20/bottle vs $35/bottle |
| Subscription savings | Subscribe-and-save option exists | Subscribe & Save 30% |
| Comparison anchor savings | Type A with consultant/competitor anchor | vs. $5,000 for private coaching |

### Step 2: Calculate Every Number

**For each savings claim, perform the exact calculation:**

**Crossed-out price + percentage:**
```
savings_amount = original_price - offer_price
savings_percentage = (savings_amount / original_price) × 100
Display: ~~$[original]~~ $[offer] — SAVE [floor(percentage)]%
```
Rule: Always round DOWN (floor) for percentage savings. Never round up. Claiming 32% when the math produces 31.7% is a trust violation.

**Per-day breakdown:**
```
per_day = actual_price / timeframe_days
Display: Less than $[per_day rounded to nearest cent]/day
```
Rule: Round UP to the nearest cent for per-day. "Less than $2.23/day" must mean the actual calculation is ≤$2.23.

**Per-unit bundle savings:**
```
per_unit_bundle = bundle_total / bundle_quantity
savings_vs_single = single_unit_price - per_unit_bundle
savings_pct = (savings_vs_single / single_unit_price) × 100
Display: $[per_unit_bundle]/[unit] — SAVE [floor(savings_pct)]%
```

**Subscription savings:**
```
savings_amount = one_time_price - subscription_price
savings_pct = (savings_amount / one_time_price) × 100
Display: Subscribe & Save [floor(savings_pct)]% — $[subscription_price]/[period]
```

### Step 3: Write Savings Callout Copy

For each calculated savings claim, write the display copy in the appropriate format:

**Format 1: Crossed-Out Price**
```
~~$[original]~~ $[offer price]
SAVE [XX]% ($[savings amount] OFF)
```

**Format 2: Per-Day Minimizer**
```
That's less than $[per-day]/day — [relatable comparison].
```
Relatable comparisons by price range:
- <$1/day: "less than a cup of coffee"
- $1-3/day: "less than a daily latte"
- $3-5/day: "less than a fast food lunch"
- $5-10/day: "less than a single [relevant professional service]"
- >$10/day: use the dollar amount only — no relatable comparison needed

**Format 3: Bundle Savings Table**
```
| Quantity | Per-Unit | Total | You Save |
|----------|----------|-------|----------|
| [qty 1] | $[per-unit] | $[total] | [XX]% |
| [qty 2] | $[per-unit] | $[total] | [XX]% |
| [qty 3] | $[per-unit] | $[total] | [XX]% |
```

**Format 4: Subscription Badge**
```
Subscribe & Save [XX]%
$[subscription price]/[period] (reg. $[regular price])
[Cancel/pause language]
```

**Format 5: First-Order Promotional Savings**
```
First [order/month]: [XX]% OFF — $[promo price]
Then $[ongoing price]/[period] — still [XX]% off regular price
```
Rule: BOTH the promotional and ongoing savings must be calculated and displayed. Never hide the ongoing price.

### Step 4: Compile All Callouts With Math Verification

Every callout includes its calculation inline for Layer 3 verification.

## Quality Criteria
- Every savings claim has inline math verification
- Percentage savings use floor rounding (never round up)
- Per-day calculations use ceiling rounding for "less than" claims
- All calculations are accurate to the penny
- Subscription savings show BOTH promotional and ongoing prices
- Bundle savings show per-unit price (not just total)
- Relatable comparison is appropriate for the price range
- Zero AI telltales
- No fabricated numbers

## Failure Modes
- Math produces a different percentage than displayed -> Recalculate and update the display copy. The number in the copy must match the math.
- Per-day "less than" claim is false (displayed amount < actual per-day) -> Round up, not down, for per-day claims.
- Subscription ongoing price hidden -> Add the ongoing price explicitly. Hiding it is deceptive and reduces long-term trust.
- Bundle savings calculated from wrong baseline -> Savings must be calculated from the single-unit price, not from a made-up "retail" price.
- Relatable comparison inappropriate for audience -> Match the comparison to the audience's context. "Less than a green juice" works for health buyers. "Less than a business lunch" works for B2B.
