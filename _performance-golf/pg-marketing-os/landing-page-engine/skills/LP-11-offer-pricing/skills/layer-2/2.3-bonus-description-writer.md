# 2.3 — Bonus Description Writer

**Skill:** LP-11-offer-pricing
**Position:** Layer 2, Microskill 3
**Purpose:** Write benefit-forward descriptions for each bonus item in the offer. Bonus copy must make the reader feel they are receiving genuine additional value that COMPLEMENTS the core product — not filler items included to inflate the value stack.
**Effort Level:** high
**Dependencies:** `bonus-strategy.md` (1.3), `value-stack-plan.md` (1.2), `specimens-loaded.md` (0.3)
**Output:** `bonus-copy.md`

---

## Execution Logic

### Type A: Full Bonus Descriptions

Write each bonus following this pattern:

```
FREE BONUS #[X]: [Bonus Name] — $[Value] Value

[Sentence 1: The benefit — what this bonus DOES for the buyer. Use the benefit angle
from bonus-strategy.md. Lead with the outcome, not the format.]

[Sentence 2: The specifics — what the bonus actually IS. Make the deliverable tangible.
Format, length, scope.]

[Sentence 3 (optional): The complementary hook — why this bonus matters specifically
for someone buying the core product. Connect it back to the main offer.]
```

**Example (Speed Angle):**
```
FREE BONUS #1: The 7-Day Quick-Start Blueprint — $97 Value

Get your first measurable results within one week — not one month.
This step-by-step daily action plan lays out exactly what to do each morning
for your first 7 days, so you skip the "where do I start?" paralysis entirely.
Designed specifically for people starting [Product Name] who want proof
it works before they're halfway through the main program.
```

**Example (Protection Angle):**
```
FREE BONUS #2: The Mistake Prevention Checklist — $47 Value

The 12 most common errors that derail results — and exactly how to avoid each one.
A printable one-page reference (created from analyzing 2,000+ customer support tickets)
that keeps you on the right track when things get confusing.
```

**Example (from BIOptimizers specimen — physical bonus):**
```
FREE BONUS #1: MassZymes (30 capsules) — $29 Value

Digest better and absorb more of every nutrient you take in — including your magnesium.
This full-spectrum enzyme blend helps break down proteins, fats, and carbs so your body
actually uses what you're putting into it. Awarded on your first month.
```

### Bonus Copy Rules

1. **"FREE" comes first.** Every bonus header starts with "FREE BONUS" — this is a gift, and the word "FREE" must be the first thing the reader sees.

2. **Benefit FIRST, format SECOND.** Never open with "This is a 47-page PDF..." Open with what the PDF DOES for the reader.

3. **One bonus, one primary benefit.** Each bonus description communicates ONE clear benefit. If a bonus has 3 benefits, choose the most compelling one and lead with it.

4. **Value anchors must be present and believable.** Every bonus header includes "$XX Value." The value must pass the believability check from bonus-strategy.md.

5. **Complementary language required.** At least one sentence in each bonus description must connect the bonus back to the core product. The reader should feel "this makes the main product EVEN BETTER."

6. **Time-limited bonuses need clear copy.** If a bonus is only available on the first order/month:
   ```
   [At the end of the description:]
   "Available with your first [order/month] only."
   ```
   State the limitation factually. Do not manufacture urgency.

7. **Bonus copy is shorter than value stack copy.** Each bonus: 1-3 sentences (30-75 words). Bonuses are gifts — they don't need the same selling effort as the core product.

### Type B: Bonus Callout Copy

For Type B pages with bonuses (e.g., BIOptimizers free MassZymes sample), write brief callout copy:

```
FREE with your order: [Bonus Name]
[1 sentence: What it does for you]
```

**Example:**
```
FREE with your order: MassZymes (30 caps)
Digest better, absorb more — supports full nutrient uptake from every supplement you take.
```

### Type B: No Bonuses

If the Type B offer has no bonuses, skip this microskill entirely. Write `bonus-copy.md` with a single line: "No bonuses in this offer — 2.3 skipped per pricing-display-confirmed.md."

## Quality Criteria
- Every bonus description opens with a benefit, never a feature or format
- "FREE BONUS" and "$XX Value" present in every header
- Each bonus includes at least one sentence connecting it to the core product
- Copy length is proportional: 1-3 sentences per bonus, none exceeding 75 words
- Time-limited bonuses have factual limitation copy (no manufactured urgency)
- Total bonus values match the plan from 1.3
- Zero AI telltales

## Failure Modes
- Bonus opens with "This is a..." or "Includes a..." -> Rewrite with benefit-first opening.
- Bonus value anchor missing from header -> Add it. Headers must show the perceived value.
- Bonus description is longer than the hero component description -> Cut. Bonuses are supporting cast.
- All bonuses use the same sentence pattern -> Vary the patterns. Read all bonus descriptions back-to-back; they should feel distinct.
- No complementary hook -> Add at least one sentence per bonus connecting it to the core product outcome.
- Bonus for Type B exceeds 1 sentence -> Cut to 1 sentence. Type B bonus callouts are scan-speed copy.
