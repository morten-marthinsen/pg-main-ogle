# 2.2 — Value Stack Writer

**Skill:** LP-11-offer-pricing
**Position:** Layer 2, Microskill 2
**Purpose:** Write individual component descriptions for the value stack (Type A) or condensed product descriptions per tier (Type B). Each component description must translate a feature into a benefit, anchor a perceived value, and make the reader feel that this ONE component alone justifies a significant portion of the price.
**Effort Level:** max
**Dependencies:** `value-stack-plan.md` (1.2), `offer-architecture-loaded.md` (0.2), `specimens-loaded.md` (0.3), `packages-loaded.md` (0.1)
**Output:** `value-stack-copy.md`

## Pre-Writing Protocol

Before writing any component description:

1. **Re-read the value-stack-plan.md** for each component's assigned copy approach, benefit angle, and target value.

2. **Re-read specimen value stack examples** from specimens-loaded.md. Hold verbatim phrases in context as tonal anchors.

3. **State the transformation.** For each component, write one sentence (for internal reference, not for output): "This component takes the buyer FROM [current state] TO [desired state]." This anchors the benefit angle.

4. **Check the anti-slop word list.** Commit to zero forbidden words before writing.

---

## Type A: Full Value Stack Copy

Write each component description following this pattern:

```
[Component Name] — [$XX Value]

[Sentence 1: The benefit — what this DOES for the buyer. Start with a benefit verb or a
problem-elimination statement. NOT "This includes..." or "You'll receive..."]

[Sentence 2: The specifics — what it IS, concretely. Format, scope, quantity, depth.
Make the deliverable tangible.]

[Sentence 3 (optional): The proof or comparison — why this value anchor is justified.
"The same [thing] that [credentialed person/company] charges $[amount] for."
Or: "Based on [X years/hours] of [experience/research]."]
```

### Component Description Rules

1. **Sentence 1 is ALWAYS the benefit.** Never open with what the component IS. Open with what it DOES.
   - BAD: "Module 3 is a 47-page workbook covering the implementation steps."
   - GOOD: "Cut your implementation time from weeks to days with a step-by-step action plan that tells you exactly what to do each morning."

2. **Value anchors must be present** — every component has "$XX Value" in its header.

3. **No two components should sound the same.** Vary the sentence structure:
   - Component 1: Benefit statement. Specifics. Proof.
   - Component 2: Problem elimination. Deliverable detail. Comparison.
   - Component 3: Outcome vision. How it works. Why it's included.

4. **The Hero Component** (identified in value-stack-plan.md) gets 3-5 sentences. All others get 2-3.

5. **Avoid "you'll also get" transitions** between components. Each component stands alone. The value stack is a list, not a narrative.

6. **Use specific numbers** wherever possible:
   - "47 done-for-you templates" not "a collection of templates"
   - "6 video modules totaling 4.5 hours" not "comprehensive video training"
   - "Based on 14 years of clinical practice" not "years of experience"

### Component Copy Length Targets

| Component Position | Sentence Count | Approximate Word Count |
|-------------------|---------------|----------------------|
| Hero component (rank 1) | 3-5 sentences | 50-100 words |
| Secondary components (rank 2-3) | 2-4 sentences | 30-70 words |
| Supporting components (rank 4+) | 2-3 sentences | 20-50 words |

## Type B: Condensed Value Stack Copy

For Type B pages, write a 1-2 sentence product description for each pricing tier that communicates value quickly.

**Per-tier description pattern:**
```
[1 sentence: What the product does — primary benefit claim]
[1 sentence: What makes this tier specifically valuable — quantity advantage, format preference, savings highlight]
```

**Example:**
```
Tier 1 (5-bottle bundle):
"Full-spectrum magnesium support for deeper sleep, less stress, and sustained energy.
Stock up and save — 5 bottles at our lowest per-bottle price."

Tier 2 (3-bottle bundle):
"Complete magnesium complex supporting 7 essential body functions.
Our most popular option — 3-month supply at 27% off."

Tier 3 (1 bottle):
"Try the 7-form magnesium blend trusted by over 1 million customers.
One bottle, 60 capsules — a full month of daily support."
```

## Quality Criteria
- Every component description leads with a benefit, not a feature
- Every component has a "$XX Value" anchor in its header
- Hero component receives the longest, most detailed description
- No two component descriptions use the same opening pattern
- Specific numbers used throughout (quantities, durations, counts)
- Total perceived value matches the planned target from 1.2
- Zero AI telltales from the forbidden word list
- Type B descriptions are each 1-2 sentences maximum

## Failure Modes
- Component description opens with "This includes..." or "You'll receive..." -> Rewrite. Benefit first, always.
- Value anchors missing from headers -> Add them. Every component must display its perceived value.
- All components use the same sentence pattern -> Vary deliberately. Read them back-to-back; they should feel distinct.
- Hero component same length as others -> Expand. The hero component earned its position by being the most valuable. Give it proportional copy weight.
- Type B descriptions exceed 2 sentences -> Cut. Type B is scan-optimized. Every extra word reduces scanning efficiency.
