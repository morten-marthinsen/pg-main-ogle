# 0.2 — Offer Architecture Loader

**Skill:** LP-11-offer-pricing
**Position:** Layer 0, Microskill 2
**Purpose:** Deep-extract the structural offer architecture from LP-06 output into a working reference that governs all Layer 2 generation. This goes beyond 0.1's summary load — it extracts the full value stack ordering, every bonus with its architect-assigned value, the complete price anchor strategy, and the guarantee reference needed for the pricing section.
**Effort Level:** low
**Dependencies:** `packages-loaded.md` (0.1), `offer-cta-architecture.json` (LP-06 output)
**Output:** `offer-architecture-loaded.md`

---

## Input Schema

```yaml
input:
  packages_loaded: "[from packages-loaded.md — confirms offer-cta-architecture.json is loaded]"
  offer_cta_architecture: "[full offer-cta-architecture.json from LP-06]"
```

---

## Execution Logic

1. **Confirm `offer-cta-architecture.json` is loaded** from 0.1 packages-loaded.md. If not loaded -> HALT.

2. **Extract the complete value stack** from LP-06 architecture:
   - Component name
   - Component description (architect's summary)
   - Architect-assigned perceived value ($XX)
   - Position in value stack (order matters — LP-06 determined the sequence)
   - Component type: core product | module/lesson | tool/template | access/community | bonus
   - Notes from architect (any specific framing instructions)

3. **Extract all bonus items** from LP-06 architecture:
   - Bonus name
   - Bonus description (architect's summary)
   - Architect-assigned perceived value ($XX)
   - Bonus type: digital (PDF/video/template) | physical (product/sample) | access (membership/community) | service (consultation/audit)
   - Delivery: immediate | delayed | conditional
   - Time-limited: [Y/N] — some bonuses expire (first X buyers, first month only, etc.)

4. **Extract the price anchor strategy** from LP-06:
   - Primary anchor type: value-sum comparison | competitor comparison | consultant-rate comparison | cost-of-problem comparison | cost-of-alternative comparison
   - Anchor values: $[amount 1], $[amount 2] (the "not prices" before the real price)
   - Per-day minimizer: [Y/N] — if Y, over what timeframe?
   - Savings display format: crossed-out | percentage | dollar-amount | per-unit | combination

5. **Extract pricing tiers** (Type B or multi-tier offers):
   - Tier name / label
   - Quantity
   - Price per unit
   - Total price
   - Savings vs. single-unit baseline
   - Badge/label: "Best Value" / "Most Popular" / "Starter" / none
   - Subscription option: [Y/N] — if Y, subscription price + savings %

6. **Extract guarantee reference**:
   - Branded guarantee name (from LP-06)
   - Duration
   - Type: unconditional / conditional / hybrid
   - Key terms (1-2 sentences maximum — this is the reference, not the full block)

7. **Extract the "if all it did was" framing direction** from LP-06 (if provided):
   - Suggested benefits for the framework
   - Suggested value anchors per benefit
   - If LP-06 did not provide this direction -> Note as "LP-11 to determine from product benefits"

8. **Write `offer-architecture-loaded.md`** with complete extraction organized for Layer 1 and 2 consumption.

## Quality Criteria
- Every value stack component extracted with perceived value — no components silently dropped
- Bonus items include delivery timing (matters for copy — "instant access" vs "ships separately")
- Price anchor strategy fully captured — Layer 2 writers need this to build the price reveal sequence
- Pricing tiers include per-unit calculations (not just total prices)
- Guarantee reference is concise (1-2 sentences) — not the full guarantee block

## Failure Modes
- Value stack empty in LP-06 output -> Flag for Type B (acceptable — Type B may not have a value stack) or HALT for Type A (value stack is mandatory)
- No guarantee in LP-06 architecture -> HALT — every offer needs a guarantee reference. Escalate to LP-06 for guarantee design.
- Price anchor values missing -> Flag as WARNING. LP-11 can generate anchor values, but LP-06's architect-level anchors are preferred.
- Bonus values unbelievable at extraction -> Flag but do not modify. LP-11 Layer 1 will evaluate and adjust in bonus-strategy.md.
