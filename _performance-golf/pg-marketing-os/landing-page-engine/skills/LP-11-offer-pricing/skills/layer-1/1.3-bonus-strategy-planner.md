# 1.3 — Bonus Strategy Planner

**Skill:** LP-11-offer-pricing
**Position:** Layer 1, Microskill 3
**Purpose:** Rank bonuses by perceived value, select the primary benefit angle for each bonus description, and assign individually believable value anchors. This planner ensures bonus copy will lead with benefits (not features) and complement rather than compete with the core offer.
**Effort Level:** medium
**Dependencies:** `offer-architecture-loaded.md` (0.2), `specimens-loaded.md` (0.3), `value-stack-plan.md` (1.2)
**Output:** `bonus-strategy.md`

## Execution Logic

1. **List all bonuses** from offer-architecture-loaded.md.

2. **For each bonus, determine the complementary angle.** A bonus should complement the core offer — it fills a gap, accelerates a result, removes a friction point, or extends the value. Ask: "After the buyer gets the main product, what does THIS bonus give them that the main product doesn't?"

3. **Rank bonuses by perceived value to the target audience** (not by dollar amount). The ranking considers:

   | Factor | Weight | How to Evaluate |
   |--------|--------|----------------|
   | Problem-pain alignment | 40% | Does this bonus directly address a secondary pain the audience has? |
   | Complementarity | 25% | Does this bonus enhance the core product's outcome? |
   | Perceived format value | 20% | Is this a format the audience values? (Video > PDF for most markets. Template > Theory.) |
   | Urgency/scarcity | 15% | Is this bonus time-limited? First X buyers only? These rank higher. |

4. **Assign value anchors to each bonus.** Rules:
   - Bonus values must be individually believable
   - Total bonus value should not exceed 50% of total value stack value (bonuses are gifts, not the main event)
   - Value anchors should reflect what the bonus would cost if sold separately, NOT what the outcome is worth
   - Digital bonuses (PDFs, checklists): $7-47 range unless they contain genuinely valuable frameworks/templates
   - Video training bonuses: $47-197 range
   - Physical product bonuses (samples, etc.): manufacturer cost × 2-3x is believable
   - Access bonuses (trial memberships, community access): monthly rate × duration

5. **Select the benefit angle for each bonus description:**

   | Angle Type | When to Use | Copy Opening Pattern |
   |-----------|-------------|---------------------|
   | Speed angle | Bonus accelerates the core product's result | "Get [result] even faster with..." |
   | Protection angle | Bonus prevents a common failure mode | "Make sure [common mistake] never derails your..." |
   | Extension angle | Bonus extends the core result to a new area | "Take your [core result] and apply it to..." |
   | Simplification angle | Bonus removes complexity from implementation | "Skip the trial-and-error. This [bonus] gives you..." |
   | Exclusive access angle | Bonus provides insider/premium access | "For the first time, get access to..." |

6. **Plan the bonus copy format:**
   - Type A: Each bonus gets a "FREE BONUS [#]" header with name, value anchor, and 1-3 sentence benefit description
   - Type B: Bonuses appear as brief callouts ("FREE with your order: [Bonus Name]") with 1-sentence benefit

7. **Handle time-limited bonuses** (e.g., "Free bonuses awarded ONLY on the first month" — BIOptimizers pattern):
   - Note the limitation clearly in the strategy
   - Plan copy that states the limitation without creating FOMO (state the fact, don't manufacture urgency)

8. **Write `bonus-strategy.md`.**

## Quality Criteria
- Every bonus from LP-06 architecture is ranked — no bonuses silently dropped
- Ranking reflects AUDIENCE-perceived value, not just dollar amount
- Each bonus has a specific benefit angle that complements (not repeats) the core product benefit
- Value anchors are individually believable (within format-appropriate ranges)
- Total bonus value ≤50% of total value stack value
- Time-limited bonuses have clear limitation copy direction

## Failure Modes
- No bonuses in LP-06 architecture -> Acceptable for some offers. Write `bonus-strategy.md` with "No bonuses — 2.3 will be SKIPPED." Layer 2.3 is skipped.
- All bonuses are digital downloads with inflated values -> Reduce all to $7-47 range. A stack of "$497 value" PDFs is the most common trust destroyer in value stacks.
- Bonus benefit angle identical to core product benefit -> Revise the angle. If Bonus 1 also says "lose weight faster," it competes with the core product instead of complementing it. Find the DIFFERENT value.
- Bonus exceeds core product in perceived value -> Reduce bonus value. If a bonus is more valuable than the main product, the reader wonders why it's not the main product. Bonuses should enhance, not overshadow.
