# 1.1 — Pricing Display Classifier

**Skill:** LP-11-offer-pricing
**Position:** Layer 1, Microskill 1
**Purpose:** Confirm the pricing display strategy from LP-06 (value-first vs price-visible) and declare the specific copy execution approach for this page. This is the structural decision that determines whether Layer 2 runs the full Type A sequence (value stack -> "if all it did was" -> price reveal) or the Type B sequence (pricing tiers -> savings callouts -> bundle incentives).
**Effort Level:** low
**Dependencies:** `packages-loaded.md` (0.1), `offer-architecture-loaded.md` (0.2)
**Output:** `pricing-display-confirmed.md`

---

## Input Schema

```yaml
input:
  page_type: "[from packages-loaded.md]"
  pricing_display_strategy: "[from offer-architecture-loaded.md — LP-06's declared strategy]"
  offer_type: "[from offer-architecture-loaded.md]"
  value_stack_exists: "[Y/N — does LP-06 include a value stack?]"
  pricing_tiers_exist: "[Y/N — does LP-06 include multiple pricing tiers?]"
  awareness_stage: "[from page-brief.json via packages-loaded.md]"
```

---

## Execution Logic

1. **Read LP-06's declared pricing display strategy.** This is the architect's decision. LP-11 confirms or flags a concern — it does not override without strong justification.

2. **Validate strategy matches page type:**

   | Page Type | Expected Strategy | If Mismatch |
   |-----------|------------------|-------------|
   | Type A | value-first | Flag if LP-06 says price-visible for a Type A page |
   | Type B | price-visible | Flag if LP-06 says value-first for a Type B page |
   | Hybrid | value-first (below fold) + price-visible (above fold) | Both strategies apply in different zones |

3. **Declare the Layer 2 execution plan** based on confirmed strategy:

   **Value-First Execution (Type A):**
   - 2.2: Value stack writer — EXECUTE (full version)
   - 2.5: "If all it did was" writer — EXECUTE
   - 2.1: Pricing section writer — EXECUTE (price reveal mode)
   - 2.3: Bonus description writer — EXECUTE
   - 2.4: Savings callout writer — EXECUTE (per-day minimizer, comparison anchor)

   **Price-Visible Execution (Type B):**
   - 2.1: Pricing section writer — EXECUTE (pricing tier mode)
   - 2.2: Value stack writer — EXECUTE (condensed — product description per tier)
   - 2.3: Bonus description writer — EXECUTE (if bonuses exist; skip if none)
   - 2.4: Savings callout writer — EXECUTE (crossed-out prices, bundle savings, subscription savings)
   - 2.5: "If all it did was" writer — SKIP

   **Hybrid Execution:**
   - All microskills EXECUTE — 2.2 and 2.5 for below-fold value section
   - 2.1 runs TWICE: above-fold pricing tier mode + below-fold price reveal mode

4. **Determine price reveal copy approach** (Type A only):
   - Will the price be revealed after value stack + "if all it did was"? (standard)
   - Will a teaser price appear earlier (e.g., "for less than $X/day")? (early hint approach)
   - What is the first "not-price" — the inflated comparison number? (from LP-06 anchor strategy)

5. **Write `pricing-display-confirmed.md`.**

## Quality Criteria
- LP-06 strategy confirmed or flagged with specific justification
- Layer 2 execution plan explicitly states EXECUTE or SKIP for every microskill
- Price reveal approach (Type A) includes the specific "not-price" sequence
- Pricing tier approach (Type B) includes tier count and labels
- No ambiguity — every downstream microskill knows whether it runs and in what mode

## Failure Modes
- LP-06 pricing strategy missing or unclear -> Default to page type convention (Type A = value-first, Type B = price-visible) and flag for human review.
- Strategy mismatch with page type -> Flag the mismatch but proceed with LP-06's strategy unless human overrides. LP-06 may have a justified reason (e.g., low-price Type A product that doesn't need value stacking).
- Hybrid page with unclear zone boundaries -> Determine which sections are above-fold (Type B treatment) and below-fold (Type A treatment) from section-sequence.json.
