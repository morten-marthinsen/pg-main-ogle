# 0.3 — Competitor URL Collector

> **Skill:** LP-02-competitive-audit
> **Layer:** 0
> **Position:** 3 of 3
> **Purpose:** Collect 3-5 competitor/reference page URLs from user input, LP-01 data, or Firecrawl search. Validate URLs are live and relevant to the target vertical/page type.
> **Effort Level:** haiku
> **Dependencies:** `brief-load.md`, `intelligence-load.md`, user input
> **Output:** `competitor-urls.md`

---

## INPUT

```yaml
input:
  brief_load: "brief-load.md"
  intelligence_load: "intelligence-load.md"
  user_provided_urls: "[list of URLs provided by user, if any]"
```

---

## EXECUTION

### Step 1: Gather URLs from All Sources

**Source 1 — User Input (highest priority):**
- If the user has provided competitor URLs, use those first.
- User-provided URLs take priority because the user knows their competitive landscape.

**Source 2 — LP-01 Intelligence:**
- Check `intelligence-load.md` for any competitor URLs identified during conversion intelligence.

**Source 3 — Firecrawl Search (if needed to reach minimum 3):**
- If Sources 1 and 2 do not provide at least 3 URLs, use `firecrawl_search` to discover competitors.
- Search query construction:
  - `[vertical] + [product category] + "landing page"` or `[vertical] + [product type] + "buy now"` or `[brand competitors] + [product category]`
  - Example: "golf training program landing page" or "magnesium supplement buy now" or "DTC supplement landing page"
- Filter search results for pages that appear to be actual landing pages or product pages (not blog posts, news articles, or directories).

### Step 2: Validate URLs

For each collected URL:
1. Confirm the URL is a valid, complete URL (starts with https://)
2. Confirm the URL appears to be a landing page or product page (not a homepage, blog post, or category page — unless a homepage IS the conversion page, like Sunday Red)
3. Confirm the URL is relevant to the target vertical from `brief-load.md`
4. Note any page type expectations (Type A, Type B, or unknown pending scrape)

### Step 3: Prioritize and Select

- Target: 3-5 URLs
- Prioritize by relevance to the target vertical and page type
- Include at least 1 direct competitor (same product category) if possible
- Include at least 1 aspirational reference (best-in-class in adjacent vertical) if available
- If more than 5 URLs are available, select the 5 most relevant

### Step 4: Write URL Collection

Write the collected URLs to `competitor-urls.md`.

## QUALITY CRITERIA

- At least 3 valid URLs collected
- Each URL has: brand name, source, expected page type, vertical match classification
- URLs are actual landing/product pages (not blog posts or directories)
- URLs are relevant to the target vertical
- File exists after execution

---

## FAILURE MODES

- User provides fewer than 3 URLs and LP-01 has none -> Use Firecrawl search to discover additional competitor pages. Construct search queries from the vertical + product category in `brief-load.md`.
- Firecrawl search returns no relevant landing pages -> Broaden search query. Try adjacent verticals. If still insufficient, HALT with message listing what was tried and asking user for competitor URLs.
- All collected URLs are for the same brand -> Flag: "3+ URLs but only 1 brand — insufficient competitive diversity." Collect URLs from at least 2 different brands.
- URLs appear to be gated/login-required -> Flag as "possibly gated" in notes. Attempt scrape anyway in Layer 1 — some gated pages have public landing page content before the gate.
