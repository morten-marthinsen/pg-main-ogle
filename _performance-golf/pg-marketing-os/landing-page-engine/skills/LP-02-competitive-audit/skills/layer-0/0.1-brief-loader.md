# 0.1 — Brief Loader

> **Skill:** LP-02-competitive-audit
> **Layer:** 0
> **Position:** 1 of 3
> **Purpose:** Load the page brief from LP-00 and extract vertical, page type, audience, product, and any competitor references.
> **Effort Level:** haiku
> **Dependencies:** `page-brief.json` from LP-00 Brief & Page Type Classifier
> **Output:** `brief-load.md`

---

## INPUT

```yaml
input:
  page_brief_path: "[path to page-brief.json from LP-00]"
  project_folder: "[path to project output folder]"
```

---

## EXECUTION

1. Locate `page-brief.json` in the project output folder. Search paths:
   - `./outputs/[project-name]/landing-page/lp-00-brief-classifier/page-brief.json`
   - `./outputs/[project-name]/lp-00-brief-classifier/page-brief.json`
   - User-provided path

2. Parse the page brief and extract the fields LP-02 needs:
   - **Project name** — identifies the project
   - **Vertical** — governs which competitor pages are relevant (golf, health, finance, info products, etc.)
   - **Target page type** — Type A / Type B / Hybrid — determines which competitor page types to prioritize
   - **Target audience** — who the page is for (awareness stage, demographics, psychographics)
   - **Product/offer** — what is being sold (name, category, price point)
   - **Competitor references** — any competitors already mentioned in the brief
   - **Brand/company** — the brand building this page

3. If `page-brief.json` is not found, check for alternative brief outputs:
   - `page-brief.md`
   - `PAGE-BRIEF-SUMMARY.md`
   - Any file in the LP-00 output folder

4. Write extracted data to `brief-load.md`.

---

## OUTPUT SCHEMA

`brief-load.md`:

```markdown
# Brief Load — [Project Name]

## Load Status: [LOADED | NOT_FOUND | PARTIAL]

## Source File
- Path: [full path]
- Page type: [type_a | type_b | hybrid]

## Extracted Fields for LP-02

### Project
- Project name: [name]
- Brand/Company: [name]

### Vertical & Audience
- Vertical: [golf | health | finance | info_products | personal_dev | technology | saas | ecomm | other]
- Target audience: [description]
- Awareness stage: [stage 1-5]

### Page Type
- Target page type: [type_a | type_b | hybrid]
- Page type rationale: [from LP-00 classification]

### Product/Offer
- Product name: [name]
- Product category: [category]
- Price point: [range or specific]

### Competitor References (from brief)
- Competitors mentioned: [list any competitor names, URLs, or brands mentioned in the brief]
- Competitor notes: [any context about competitive landscape from the brief]
```

---

## QUALITY CRITERIA

- Page brief located and parsed
- Vertical extracted (critical for competitor selection)
- Page type extracted (critical for type-matching competitors)
- All available fields populated
- File exists after execution

---

## FAILURE MODES

- Page brief not found -> Write `brief-load.md` with status `NOT_FOUND` and list all paths searched. Triggers GATE_0 HALT. LP-00 must be run first.
- Brief found but missing vertical -> Infer vertical from product/audience description if possible. Flag as inferred. If cannot infer, HALT — vertical is required for competitor selection.
- Brief found but missing page type -> Default to "hybrid" and flag for human confirmation.
