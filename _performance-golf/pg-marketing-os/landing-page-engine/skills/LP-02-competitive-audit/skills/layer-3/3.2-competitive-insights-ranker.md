# 3.2 — Competitive Insights Ranker

> **Skill:** LP-02-competitive-audit
> **Layer:** 3
> **Position:** 2 of 2
> **Purpose:** Rank all competitive insights by strategic value. Categorize as: differentiation opportunity (what we can do uniquely), pattern to replicate (what works and we should adopt), pattern to avoid (what is saturated or ineffective).
> **Effort Level:** sonnet
> **Dependencies:** `differentiation-synthesis.md`, `coverage-validation.md`
> **Output:** `insights-ranking.md`

---

## INPUT

```yaml
input:
  differentiation_synthesis: "differentiation-synthesis.md"
  coverage_validation: "coverage-validation.md"
  brief_load: "brief-load.md"
```

---

## EXECUTION

### Step 1: Collect All Insights

From `differentiation-synthesis.md`, collect:
- All universal patterns
- All unique differentiators
- All gaps and opportunities
- All recommended patterns
- All patterns to avoid

### Step 2: Categorize Each Insight

Assign each insight to one of three strategic categories:

**Category A — Differentiation Opportunity (Highest Value)**
Insights where our page can do something NO competitor does, or do something significantly BETTER. These create competitive advantage.

**Category B — Pattern to Replicate (Table Stakes)**
Insights where competitors universally do something that works. Not replicating these creates a gap. These are defensive — must-haves, not differentiators.

**Category C — Pattern to Avoid**
Insights about competitor approaches that are saturated (everyone does it the same way — diminishing returns), ineffective (appears to weaken the page), or inappropriate for our page type/vertical.

### Step 3: Rank Within Categories

Within each category, rank by strategic impact:

**For Category A (Differentiation):**
- Impact potential (how much competitive advantage?)
- Feasibility (can we execute this?)
- Uniqueness (how far from what competitors do?)

**For Category B (Replicate):**
- Universality (how many competitors use it? More = more essential)
- Execution risk (what happens if we skip it?)

**For Category C (Avoid):**
- Saturation level (how overused is this approach?)
- Risk of inclusion (what happens if we DO use it?)

### Step 4: Generate Top 5 Overall Insights

Across all three categories, identify the 5 highest-impact strategic insights — the findings that should most influence how LP-03 through LP-06 design the target page.

## QUALITY CRITERIA

- All insights from differentiation-synthesis.md categorized and ranked
- Top 5 insights are genuinely the highest-impact findings
- Each insight has clear evidence source (which competitors, which data)
- Rankings have explicit rationale (not arbitrary ordering)
- Downstream routing provides specific, actionable guidance per skill
- File exists after execution

---

## FAILURE MODES

- All insights land in Category B (replicate) with no differentiation opportunities -> Push harder on gaps analysis. If genuinely no gaps exist, note: "This is a mature competitive landscape with limited structural differentiation opportunities. Differentiation must come from copy quality, not architecture."
- Rankings are arbitrary (no clear rationale for ordering) -> Add a 1-sentence rationale for each ranking position explaining why it outranks the item below it.
- Downstream routing is vague ("consider the competition") -> Make it specific: "LP-05 should target EXTREME proof density because 3 of 4 competitors use HIGH or EXTREME proof and the one competitor with LOW proof compensates with celebrity authority we do not have."
