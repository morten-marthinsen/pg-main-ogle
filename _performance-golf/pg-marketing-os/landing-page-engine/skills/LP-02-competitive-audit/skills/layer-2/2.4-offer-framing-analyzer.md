# 2.4 — Offer Framing Analyzer

> **Skill:** LP-02-competitive-audit
> **Layer:** 2
> **Position:** 4 of 5
> **Purpose:** Analyze each competitor page's offer presentation: how they display pricing, stack value, frame the guarantee, create urgency/scarcity, and structure bundles or tiers.
> **Effort Level:** opus
> **Dependencies:** All valid `scrape-[N].md` files, `element-map.md`
> **Output:** `offer-framing-analysis.md`

---

## INPUT

```yaml
input:
  scrape_files: "All usable/marginal scrape-[N].md files"
  element_map: "element-map.md"
```

---

## EXECUTION

**POSITIONAL REINFORCEMENT:** You are analyzing HOW each competitor frames their offer — the psychology of their pricing, guarantee, and urgency strategies. You are not recommending what our page should do (that is 2.5's job). Map the strategy. Quote the evidence.

### For Each Competitor, Analyze:

#### A. Pricing Display Strategy

- **Where pricing appears:** above_fold / mid_page / late_page / hidden (demo/application model)
- **Pricing model:** one_time / subscription / trial_to_subscription / bundle_tiers / freemium / demo_required / free_registration
- **Exact pricing:** Quote any visible prices from scraped content
- **Price anchoring technique:**
  - Crossed/original price (was $X, now $Y)
  - Per-unit calculation ("just $1.13 per stick")
  - Value comparison ("$2,500 value for $37")
  - Alternative cost ("less than your daily coffee")
  - Savings display ("Save 25%")
  - None visible

#### B. Value Stack

- **Present:** Y/N
- **Items count:** How many items in the value stack?
- **Value stack structure:** List each item with its stated value and format (image + description, bullet list, cards)
- **Total stated value:** The "total value" claim if present
- **Value-to-price ratio:** Stated value / actual price

#### C. Guarantee

- **Present:** Y/N
- **Type:** money_back / performance / cancel_anytime / satisfaction / none
- **Duration:** 30 days, 60 days, 365 days, unlimited, etc.
- **Branded name:** Does the guarantee have a custom name? (e.g., "No Questions Asked Refund Policy," "Iron-Clad Performance Guarantee")
- **Specificity:** Generic ("money-back guarantee") vs. Specific ("If you don't get 40 qualified calls in 90 days, you don't pay a cent")
- **Position:** Where on the page is the guarantee?
- **Verbatim text:** Quote the guarantee copy

#### D. Urgency/Scarcity

- **Type:** countdown_timer / limited_stock / limited_time / deadline / seasonal / none
- **Mechanism:** What creates the urgency? (timer, stock count, "spots remaining," launch pricing, etc.)
- **Specificity:** Vague ("limited time") vs. Specific ("Only 23 spots remain" or "Price increases at midnight EST")
- **Position:** Where on the page?
- **Verbatim text:** Quote the urgency copy

#### E. Bundle/Tier Strategy

- **Structure:** single_option / good_better_best / quantity_discount / subscription_tiers / add_ons / none
- **Tier details:** Name and price of each tier/bundle
- **Recommended tier:** Is one tier highlighted as "most popular" or "best value"?
- **Upsell/cross-sell:** Order bumps, related products, "frequently bought together"?

#### F. Offer Framing Summary

Write a 2-3 sentence summary: What is the offer STRATEGY? High-value perception with anchoring? Low-barrier trial? Premium-price authority? Volume discount? Hidden-price qualification?

## QUALITY CRITERIA

---

## FAILURE MODES

- Pricing is hidden/not on page -> This IS the strategy. Classify as "demo_required" or "application_required." Analyze: What action must a visitor take to see pricing? What qualifies them?
- Guarantee text not found -> Check footer, FAQ, and fine print sections. If genuinely absent, classify as "none" and note which competitors DO vs. DO NOT offer guarantees.
- Value stack is ambiguous (is a feature list a value stack?) -> A value stack explicitly assigns dollar values to items. A feature list without dollar values is education, not offer framing.
- Urgency appears fabricated/evergreen -> Note it but map it anyway. "Limited time" with no deadline is weak urgency. A countdown timer that resets is evergreen urgency. Both are strategies worth documenting.
