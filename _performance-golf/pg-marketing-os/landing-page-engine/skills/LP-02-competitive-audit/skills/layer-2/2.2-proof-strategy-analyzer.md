# 2.2 — Proof Strategy Analyzer

> **Skill:** LP-02-competitive-audit
> **Layer:** 2
> **Position:** 2 of 5
> **Purpose:** Analyze each competitor page's proof architecture: what types of proof they use, where they place it, how dense it is, where proof first appears, and the overall proof-to-copy ratio.
> **Effort Level:** opus
> **Dependencies:** All valid `scrape-[N].md` files, `element-map.md`
> **Output:** `proof-strategy-analysis.md`

---

## INPUT

```yaml
input:
  scrape_files: "All usable/marginal scrape-[N].md files"
  element_map: "element-map.md"
```

---

## EXECUTION

**POSITIONAL REINFORCEMENT:** You are analyzing HOW each competitor uses proof — not whether they do it well. Map the strategy. Describe the architecture. Quantify the approach. Do not judge effectiveness.

### Step 1: For Each Competitor, Analyze Proof Architecture

#### A. Proof Types Used

Scan the full scraped content and `element-map.md` proof section. Classify each proof element into one of these types:

| Proof Type | What to Look For |
|-----------|-----------------|
| Customer reviews / star ratings | Star ratings, review text, "verified buyer" badges, review count |
| Video testimonials | Embedded videos with customer stories, video thumbnails with play buttons |
| Celebrity/influencer endorsements | Named public figures, headshots with quotes, "as recommended by" |
| Expert/professional authority | Doctors, researchers, industry experts with credentials |
| Stat claims | "1M+ customers," "$30M+ in sales," "83,502 reviews" |
| Trust badges | Trustpilot, BBB, industry certifications, security badges |
| Scientific citations | Numbered references, PubMed links, "clinically proven" with sources |
| Performance guarantees | "40 calls or you don't pay," "15% more or don't pay" |
| Case studies | Detailed customer stories with specific metrics |
| Third-party review widgets | Embedded Trustpilot, Google Reviews, Yelp widgets |
| Screenshot testimonials | Chat screenshots, email screenshots, social media screenshots |
| Before/after | Visual or narrative transformation evidence |

For each proof type found, record:
- **Count:** How many instances
- **Placement:** Where on the page (section numbers)
- **Specificity:** High (specific numbers, names, timeframes) / Medium (some specifics) / Low (vague praise)
- **Sample:** Representative example quoted from scraped content

#### B. Proof Density Classification

Based on the total proof inventory, classify the page's proof density:

| Density | Criteria |
|---------|---------|
| ZERO | No proof elements on page (brand authority or deferred proof model) |
| LOW | 1-5 proof elements, often only authority/endorsement-based |
| MEDIUM | 6-15 proof elements, mixed types |
| HIGH | 16-30 proof elements, multiple types, distributed throughout page |
| EXTREME | 30+ proof elements OR a single proof type at massive scale (83K reviews, 38 citations) |

#### C. First-Proof Position

Identify where proof FIRST appears on the page:
- Above fold (proof opens the conversation)
- Section 2-3 (proof follows first product impression)
- Mid-page (proof supports after education)
- Late page (proof as closing argument)
- None (no proof on page)

#### D. Proof-to-Copy Ratio

Estimate what percentage of the total page content is dedicated to proof vs. other content (education, offer, story, etc.). This is approximate — based on section count and word count allocation.

#### E. Proof Architecture Summary

Write a 2-3 sentence summary of the overall proof strategy: What is the proof DOING on this page? Is it creating inevitability through volume? Establishing authority through celebrity? Replacing sales copy with social evidence? Creating FOMO through scale claims?

## QUALITY CRITERIA

- Every competitor has complete proof analysis (density, first position, types, ratio, summary)
- Every proof type has count, placement, specificity, and sample
- Proof density classification uses the defined criteria (not arbitrary)
- First-proof position is specific (section number, not just "early")
- Cross-competitor patterns identified
- No claims without scraped content evidence
- File exists after execution

---

## FAILURE MODES

- Competitor page has zero proof -> Classify as ZERO density. This IS the analysis — analyze WHY: Is it a brand play? Deferred proof (proof on separate page)? Too-new product? Application model where proof comes after qualification?
- Proof elements are ambiguous (is a stat claim proof or marketing?) -> Classify by what it would mean to the visitor. "1M+ customers" functions as proof whether or not it is verifiable.
- Scrape did not capture embedded videos -> Note that video testimonials may exist but were not captured in markdown scrape. Count video thumbnails or "Watch Video" links as indicators.
- Proof is spread across multiple scrape sections -> Consolidate. Count total proof elements across the entire page, not per-section.
