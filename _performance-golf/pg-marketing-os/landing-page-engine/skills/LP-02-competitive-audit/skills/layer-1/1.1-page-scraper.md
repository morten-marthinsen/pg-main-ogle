# 1.1 — Page Scraper

> **Skill:** LP-02-competitive-audit
> **Layer:** 1
> **Position:** 1 of 2
> **Purpose:** Scrape each competitor URL via Firecrawl MCP. Two passes per URL: full markdown content extraction + structured data extraction. Handle errors with retry chain.
> **Effort Level:** sonnet
> **Dependencies:** `competitor-urls.md`
> **Output:** `scrape-1.md` through `scrape-[N].md` (one per competitor URL)

---

## INPUT

```yaml
input:
  competitor_urls: "competitor-urls.md"
```

---

## EXECUTION

For EACH competitor URL in `competitor-urls.md`, perform the following two-pass scrape:

### Pass 1: Full Markdown Content Extraction

Use `firecrawl_scrape` with these settings:

```
Tool: firecrawl_scrape
Parameters:
  url: "[competitor URL]"
  formats: ["markdown"]
  onlyMainContent: true
  waitFor: 5000
```

This returns the full page content in markdown format — headings, body copy, links, images (as alt text), tables, lists. This is the primary data source for all Layer 2 analysis.

### Pass 2: Structured Data Extraction

Use `firecrawl_scrape` with JSON extraction schema:

```
Tool: firecrawl_scrape
Parameters:
  url: "[competitor URL]"
  formats: ["extract"]
  extract:
    schema:
      type: "object"
      properties:
        page_type:
          type: "string"
          description: "Type of page: sales_page, product_page, opt_in, application, homepage, bundle_builder"
        headline:
          type: "string"
          description: "The main H1 headline of the page"
        subheadline:
          type: "string"
          description: "The subheadline or deck copy below the headline"
        section_count:
          type: "number"
          description: "Total number of distinct content sections on the page"
        sections:
          type: "array"
          items:
            type: "object"
            properties:
              name:
                type: "string"
              position:
                type: "number"
          description: "List of section names in order"
        cta_count:
          type: "number"
          description: "Total number of CTA buttons on the page"
        cta_texts:
          type: "array"
          items:
            type: "string"
          description: "Text of each CTA button found"
        testimonial_count:
          type: "number"
          description: "Number of testimonials or reviews visible"
        pricing_above_fold:
          type: "boolean"
          description: "Whether pricing is visible without scrolling"
        guarantee_text:
          type: "string"
          description: "The guarantee copy if present"
        trust_signals:
          type: "array"
          items:
            type: "string"
          description: "List of trust signals found (badges, certifications, endorsements)"
      required: ["headline", "section_count", "cta_count"]
```

### Error Handling Chain

For each URL, if the primary scrape fails or returns insufficient content:

**Retry 1:** Re-run `firecrawl_scrape` with extended wait time:
```
waitFor: 10000
```

**Retry 2:** Use `firecrawl_agent` with a descriptive prompt:
```
Tool: firecrawl_agent
Parameters:
  url: "[competitor URL]"
  prompt: "Navigate to this page and extract: 1) The main headline, 2) All section headings in order, 3) All CTA button text, 4) All testimonials or review content, 5) Pricing information, 6) Guarantee text, 7) Trust badges or signals. Return the full page content in markdown."
```

**Skip:** If all attempts fail, mark this URL as `scrape_failed` in the output file. Do NOT fabricate content.

### Rate Limiting

- Pause 2-3 seconds between scrape requests to avoid rate limiting
- If a rate limit error is returned, wait 10 seconds and retry
- Log all timing in the output file

### Write Output

For each competitor, write a `scrape-[N].md` file (e.g., `scrape-1.md`, `scrape-2.md`).

## QUALITY CRITERIA

- Every URL in `competitor-urls.md` has a corresponding `scrape-[N].md` file
- Each file clearly indicates scrape status (success/partial/failed)
- Full markdown content included for successful scrapes
- Structured extraction results included where available
- Scrape quality assessment completed with clear usable/unusable determination
- Error handling chain documented (which methods attempted, in which order)
- Timestamps recorded for scrape freshness verification

---

## FAILURE MODES

- URL returns empty content -> Follow full error handling chain (retry with 10s wait, firecrawl_agent, skip). Document each attempt.
- URL returns only nav/footer -> Classify as "minimal" content quality. Follow retry chain. If still minimal after retries, mark as "unusable."
- URL redirects to a different page -> Note the redirect in scrape metadata. Analyze the page you landed on (it may still be relevant).
- Rate limit error -> Wait 10 seconds, retry. If persistent, wait 30 seconds. Log all waits.
- Firecrawl service unavailable -> HALT with clear error message. Cannot proceed without scraping capability.
- Structured extraction returns empty but markdown is full -> This is acceptable. Use markdown content for Layer 2 analysis. Note that structured extraction failed.
