# 4.1 — Audit Compiler

> **Skill:** LP-02-competitive-audit
> **Layer:** 4
> **Position:** 1 of 3
> **Purpose:** Compile all analysis data from Layer 2 (five analysis dimensions) and Layer 3 (coverage validation, insights ranking) into the final `competitive-audit.json` output file.
> **Effort Level:** haiku
> **Dependencies:** All Layer 2 analysis files, `coverage-validation.md`, `insights-ranking.md`, `brief-load.md`
> **Output:** `competitive-audit.json`

---

## INPUT

```yaml
input:
  element_map: "element-map.md"
  proof_analysis: "proof-strategy-analysis.md"
  cta_analysis: "cta-architecture-analysis.md"
  offer_analysis: "offer-framing-analysis.md"
  differentiation_synthesis: "differentiation-synthesis.md"
  coverage_validation: "coverage-validation.md"
  insights_ranking: "insights-ranking.md"
  brief_load: "brief-load.md"
  scrape_validation: "scrape-validation.md"
```

---

## EXECUTION

### Step 1: Read All Source Files

Read all 9 input files and extract the data needed for the JSON schema defined in LP-02-AGENT.md.

### Step 2: Assemble JSON Structure

Build the `competitive-audit.json` following the schema:

1. **Header fields:** schema_version (1.0), skill_id (LP-02), created (ISO timestamp), project_name, target_page_type, target_vertical, competitors_analyzed (count)

2. **Competitors array** — one entry per analyzed competitor, each containing:
   - competitor_id, url, brand, page_type, platform_detected, scrape_quality
   - element_map (from `element-map.md`)
   - proof_strategy (from `proof-strategy-analysis.md`)
   - cta_architecture (from `cta-architecture-analysis.md`)
   - offer_framing (from `offer-framing-analysis.md`)

3. **Cross-page synthesis** (from `differentiation-synthesis.md` and `insights-ranking.md`):
   - universal_patterns array
   - unique_differentiators array
   - gaps_and_opportunities array
   - recommended_patterns array
   - patterns_to_avoid array

4. **Metadata:** page_brief_source, intelligence_source, taxonomy_source, pattern_analysis_source, scrape_tool, scrape_timestamps

### Step 3: Validate JSON Completeness

Before writing, verify:
- All analyzed competitors present in competitors array
- Each competitor has all 4 analysis objects (element_map, proof_strategy, cta_architecture, offer_framing)
- Cross-page synthesis has all 5 sub-arrays populated
- No empty required fields
- Competitor count matches `competitors_analyzed` header field
- All scrape timestamps present in metadata

### Step 4: Write competitive-audit.json

Write the complete JSON file to the project output folder.

---

## OUTPUT SCHEMA

`competitive-audit.json` — follows the complete schema defined in LP-02-AGENT.md OUTPUT SCHEMA section.

Key validation rules:
- `competitors_analyzed` must match the length of the `competitors` array
- Each competitor must have `competitor_id`, `url`, `brand`, `page_type`
- Each competitor must have non-empty `element_map`, `proof_strategy`, `cta_architecture`, `offer_framing` objects
- `cross_page_synthesis` must have all 5 arrays (may be empty arrays if no items, but must exist)
- `metadata.scrape_timestamps` must have an entry for each competitor_id

---

## QUALITY CRITERIA

- JSON is well-formed and parseable
- All competitors present with complete analysis objects
- Cross-page synthesis populated from differentiation analysis
- Metadata includes all source references
- Competitor count matches header
- File exists after execution

---

## FAILURE MODES

- JSON malformed -> Re-check structure. Ensure all brackets, quotes, commas correct. Pay special attention to nested arrays and objects.
- Missing data from source files -> Re-read the source file. If data is genuinely missing, include the field with a value describing the gap (e.g., "DATA_MISSING — scrape was partial for this competitor") rather than omitting the field.
- Competitor count mismatch -> Reconcile with `scrape-validation.md` to confirm which competitors were analyzed. Only include competitors that passed scrape validation.
