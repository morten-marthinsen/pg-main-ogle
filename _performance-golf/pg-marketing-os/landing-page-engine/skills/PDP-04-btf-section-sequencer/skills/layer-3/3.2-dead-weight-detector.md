# 3.2 — Dead Weight Detector

**Skill:** PDP-04-btf-section-sequencer
**Position:** Layer 3, Microskill 2
**Purpose:** Identify and remove any section in the BTF sequence that has no unique persuasion function or serves no purchase-decision purpose. PDP visitors are in buy mode — every section must directly serve the buy decision. Conventional ecomm sections like "Our Story," "Brand Values," or "Sustainability Statement" that have no conversion function are dead weight. This check also validates module independence for every section.
**Effort Level:** medium
**Dependencies:** `optional-decisions.md` (1.4 — locked section list), `sequence-validation.md` (3.1 — audit results)
**Output:** `dead-weight-check.md`

---

## Input Schema

```yaml
input:
  final_section_sequence: "[from optional-decisions.md — all sections with persuasion functions]"
  sequence_validation: "[from 3.1 — module independence scores]"
```

---

## Execution Logic

**Step 1:** For every section in the locked sequence, evaluate against 3 criteria:

### Criterion 1: Unique Persuasion Function
- Does this section serve a persuasion function that NO other section in the sequence already serves?
- If two sections serve the same function (e.g., both "build trust"), one is dead weight
- Consolidation is an acceptable resolution (merge, don't just delete)

### Criterion 2: Purchase-Decision Relevance
- Does this section directly influence the buy decision?
- "Directly" means: seeing this section would change a visitor's likelihood to add to cart
- Sections that are "nice to have" but don't move the needle are dead weight

### Criterion 3: Module Independence
- Can a visitor arriving at ONLY this section (e.g., from a search snippet, from a FAQ link) understand the section and be persuaded?
- Does the section contain any "as mentioned above," "as we discussed," or "see section X" language?
- Any cross-reference between sections = module independence FAIL

**Step 2:** For each section, produce a dead weight assessment:

```
Section [N]: [Section Name]
Persuasion function: [1 sentence — what this section UNIQUELY does]
Dead weight: [Y/N]
Module independence: [Y/N — standalone capability confirmed?]
Cross-references found: [list any inter-section dependencies, or "none"]
If dead weight or not independent: Action: [remove | rewrite to standalone | merge with X]
```

**Step 3:** Flag common dead weight patterns:
- "Our Story" or "About the Brand" — dead weight unless the brand story IS the mechanism
- "Brand Values" — dead weight on PDP (belongs on About page, not product page)
- "Sustainability Statement" — dead weight unless sustainability is the product's USP
- "Press / As Seen In" — potentially valid proof (check if it duplicates The Expert section)
- Any section that is essentially a restatement of another section's content

**Step 4:** Count dead weight sections:
- If 0 dead weight sections: PASS
- If 1+ dead weight sections: list each with resolution action
- Resolution actions: REMOVE (delete from sequence), MERGE (combine with another section), REWRITE (adjust to serve a unique function)

**Step 5:** Verify module independence for every section:
- Scan all section persuasion functions for inter-section language
- If any cross-references found, document them and require rewrite

**Step 6:** Write dead weight check results.

## Quality Criteria
- Every section evaluated against all 3 criteria — no sections skipped
- Each section's persuasion function stated in 1 sentence (forces clarity)
- Dead weight = 0 for PASS status
- Module independence confirmed for ALL sections — no "as mentioned above" language anywhere
- Common dead weight patterns explicitly checked
- Resolution actions are specific (not "review later")

## Failure Modes
- Dead weight section found -> Document and provide specific resolution action. Do NOT proceed to GATE_3 with dead weight sections in the sequence.
- Module independence violation found -> Rewrite the section description to be standalone. Remove all cross-references.
- Two sections with same persuasion function -> Merge into one or differentiate their functions. Two sections cannot do the same job.
- Section removal drops count below 8 -> WARNING. Review if the removed section should be kept but rewritten instead.
- Section description is vague ("adds context") -> This IS dead weight. A persuasion function must be specific and conversion-relevant.
- "As mentioned above" or "see section X" found -> HALT for that section. Rewrite description as standalone module.
