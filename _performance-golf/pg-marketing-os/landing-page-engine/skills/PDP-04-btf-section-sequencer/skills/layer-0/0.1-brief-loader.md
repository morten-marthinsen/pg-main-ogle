# 0.1 — Brief Loader

**Skill:** PDP-04-btf-section-sequencer
**Position:** Layer 0, Microskill 1
**Purpose:** Load and verify `page-brief.json` from LP-00. Confirm page type is `type_b` or `hybrid`. Extract the key fields PDP-04 needs: page type, vertical, product type, awareness stage, proof inventory, and offer structure. If page type is `type_a`, REDIRECT to LP-04 — PDP-04 does not handle long-form sales pages.
**Effort Level:** low
**Dependencies:** `page-brief.json` (LP-00 output)
**Output:** `brief-verified.md`

---

## Input Schema

```yaml
input:
  page_brief_path: "[path to page-brief.json from LP-00]"
```

---

## Execution Logic

**Step 1:** Locate and load `page-brief.json` from the project output directory. If file does not exist -> HALT with error: "LP-00 output missing. Run LP-00 Brief & Page Type Classifier first."

**Step 2:** Read `page_type.classification`. Apply routing logic:
- `type_b` -> PROCEED (this is a PDP — PDP-04 is the correct skill)
- `hybrid` -> PROCEED (PDP-04 handles the BTF architecture for hybrid pages)
- `type_a` -> REDIRECT to LP-04. Write redirect notice and HALT. PDP-04 does not handle Type A long-form pages.

**Step 3:** Verify required fields are present and non-empty:
- `page_type.classification` — REQUIRED (must be type_b or hybrid)
- `product.name` — REQUIRED
- `product.vertical` — REQUIRED
- `product.type` — REQUIRED (supplement | physical-product | beauty | food-bev | consumer-goods)
- `audience.awareness_stage` — REQUIRED (1-5)
- `offer.price_point` — REQUIRED
- `offer.has_bundle` — RECOMMENDED (determines Cost Savings section eligibility)
- `proof_inventory` — RECOMMENDED (drives mandatory/optional section decisions in Layer 1)
  - `testimonials_count` — how many reviews/testimonials available
  - `ugc_video_count` — how many UGC video testimonials available
  - `has_clinical_studies` — Y/N
  - `has_expert_endorsement` — Y/N + expert name/credential if Y
  - `has_before_afters` — Y/N + count
  - `has_comparison_competitors` — Y/N (determines Comparison Chart eligibility)
  - `ingredient_count` — number of key ingredients (determines What's in It section scope)

**Step 4:** Flag any missing RECOMMENDED fields as warnings. Do not halt — downstream microskills will use defaults, but the warning propagates to Layer 1 section decisions.

**Step 5:** Extract and write the verified brief summary to `brief-verified.md`.

## Quality Criteria
- Page type confirmed as type_b or hybrid — FAIL if type_a (redirect to LP-04)
- All REQUIRED fields present and non-empty — FAIL if any are missing
- Proof inventory extracted even if partial — downstream Layer 1 needs whatever is available
- Section eligibility signals pre-computed to accelerate Layer 1 taxonomy selection
- Warnings clearly separated from blocking failures

## Failure Modes
- `page-brief.json` not found -> HALT. LP-00 must run first.
- `page_type.classification` is `type_a` -> REDIRECT to LP-04. Write redirect notice to `brief-verified.md` and HALT.
- `page_type.classification` missing or empty -> HALT. Page type is required for PDP-04 to proceed.
- `page_type.classification` contains an invalid value (not type_a, type_b, or hybrid) -> HALT with error describing the invalid value.
- Product name or vertical missing -> HALT. These are required for section selection and downstream writing.
- Proof inventory entirely missing -> WARNING only — proceed with UNKNOWN eligibility flags. Layer 1 will treat UNKNOWN as "include if mandatory, exclude if optional."
