# 4.2 — Summary Writer

**Skill:** PDP-04-btf-section-sequencer
**Position:** Layer 4, Microskill 2
**Purpose:** Write the human-readable `PDP-SECTION-SEQUENCE-SUMMARY.md` that provides a clear overview of the entire BTF section sequence. This file is for human review — it must be scannable, complete, and match the JSON output exactly. The summary follows the required template defined in PDP-04-ANTI-DEGRADATION.md.
**Effort Level:** low
**Dependencies:** `pdp-section-sequence.json` (4.1 — the compiled JSON)
**Output:** `PDP-SECTION-SEQUENCE-SUMMARY.md`

---

## Input Schema

```yaml
input:
  compiled_json: "[pdp-section-sequence.json from 4.1]"
  dead_weight_results: "[from 3.2 — sections removed]"
  anti_slop_results: "[from 3.3 — corrections applied]"
```

---

## Execution Logic

**Step 1:** Load `pdp-section-sequence.json` and extract all data needed for the summary.

**Step 2:** Write the summary following the EXACT template from PDP-04-ANTI-DEGRADATION.md `PDP-SECTION-SEQUENCE-SUMMARY.MD REQUIRED SECTIONS`. Every section in the template MUST be present in the output.

**Step 3:** Verify the summary matches the JSON:
- Section count matches
- Section names and order match
- Word count totals match
- Logic test results match
- Sequence score matches

**Step 4:** Write to `PDP-SECTION-SEQUENCE-SUMMARY.md`.

## Quality Criteria
- Every section from the ANTI-DEGRADATION template is present — no sections omitted
- All data matches the compiled JSON exactly — no discrepancies
- Taxonomy decisions table has all 16 NLS sections (included AND excluded)
- Complete BTF Section List table has all included sections with all columns populated
- Sticky ATC bar specification is complete
- Logic test results match JSON
- Downstream handoff notes specify what each consuming skill receives

## Failure Modes
- Summary missing a required section from the template -> HALT. Add the missing section.
- Section count in summary differs from JSON -> Fix. Must match exactly.
- Word count or score in summary differs from JSON -> Fix. Must match exactly.
- Taxonomy decisions table missing sections (fewer than 16 rows) -> HALT. All 16 must appear.
- "TBD" or placeholder text in any field -> HALT. All fields must have specific values.
