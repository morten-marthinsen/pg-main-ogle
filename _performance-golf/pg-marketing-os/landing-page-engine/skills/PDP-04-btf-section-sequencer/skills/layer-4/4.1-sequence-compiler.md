# 4.1 — Sequence Compiler

**Skill:** PDP-04-btf-section-sequencer
**Position:** Layer 4, Microskill 1
**Purpose:** Assemble the final `pdp-section-sequence.json` from all validated Layer 1-3 outputs. This is the single canonical output file that all downstream PDP writing skills consume. The compiler does not make decisions — it compiles validated data into the schema defined in PDP-04-AGENT.md. Every field must be populated. No placeholders. No "TBD." No empty strings.
**Effort Level:** low
**Dependencies:** All Layer 1-3 outputs — specifically: `taxonomy-selection.md`, `section-ordering.md`, `mandatory-check.md`, `optional-decisions.md`, `word-count-plan.md`, `proof-density-plan.md`, `visual-density-plan.md`, `cta-placement-plan.md`, `sticky-atc-plan.md`, `sequence-validation.md`, `dead-weight-check.md`, `anti-slop-check.md`
**Output:** `pdp-section-sequence.json`

---

## Input Schema

```yaml
input:
  all_layer_outputs: "[all .md files from Layers 1-3]"
  sequence_score: "[from 3.1 — must be >= 7.0]"
  all_4_tests_pass: "[from 3.1 — must be Y]"
  dead_weight_count: "[from 3.2 — must be 0]"
  anti_slop_corrections: "[from 3.3 — replacements to apply]"
  gate_3_status: "[PASS — required to proceed]"
```

---

## Execution Logic

**Step 1:** Verify GATE_3 status is PASS. Check:
- Sequence score >= 7.0 (from 3.1)
- All 4 logic tests PASS (from 3.1)
- Dead weight sections = 0 (from 3.2)
- Anti-slop check PASS or PASS WITH CORRECTIONS (from 3.3)

If any condition fails -> HALT. Do not compile a sequence that failed validation.

**Step 2:** Build the JSON structure following the exact schema from PDP-04-AGENT.md `PDP-SECTION-SEQUENCE.JSON SCHEMA`:

**Header fields:**
- `schema_version`: "1.0"
- `created`: ISO timestamp
- `project_name`: from brief
- `page_type`: "type_b" or "hybrid"
- `total_btf_sections`: count from optional-decisions.md (minus any dead weight removals)
- `estimated_btf_word_count`: {minimum, target, maximum} from word-count-plan.md
- `sequence_score`: from sequence-validation.md

**pdp_section_taxonomy block:**
- One entry per all 16 NLS taxonomy sections
- Each with: `included` (true/false), `rationale` (from taxonomy-selection.md + optional-decisions.md)

**sections array:**
- One entry per included section, in sequence order
- Each with: number, section_id, name, mandatory, persuasion_function, word_count (target + range), proof_density (level + elements), visual_density (image_count + primary_type + layout), cta_present, cta_type, module_independence, notes

**sticky_atc_trigger_point:**
- mobile_trigger, desktop_trigger, bar_content, disappear_condition from sticky-atc-plan.md

**cta_placement_summary:**
- sticky_atc description, mid_page_ctas array, total_cta_touchpoints from cta-placement-plan.md

**proof_wave_summary:**
- early_zone, mid_zone, late_zone from proof-density-plan.md

**logic_validation:**
- test_1 through test_4 results + all_pass boolean from sequence-validation.md

**downstream_handoffs:**
- Skill-to-section mapping from PDP-04-AGENT.md

**Step 3:** For each section entry, pull data from the correct upstream file:
- Section order and IDs -> `optional-decisions.md` (final locked list)
- Persuasion functions -> Use anti-slop-corrected versions from `anti-slop-check.md` (if corrections exist, use corrections instead of originals)
- Word counts -> `word-count-plan.md`
- Proof density -> `proof-density-plan.md`
- Visual density -> `visual-density-plan.md`
- CTA data -> `cta-placement-plan.md`
- Module independence -> confirmed from `dead-weight-check.md`

**Step 4:** Validate JSON structure before writing:
- No empty fields (no "", no null, no "TBD", no "to be determined")
- All section numbers are sequential (1, 2, 3... N)
- Total section count matches sections array length
- Word count total (sum of section targets) matches estimated_btf_word_count.target
- All 16 taxonomy entries present in pdp_section_taxonomy (both included and excluded)
- logic_validation.all_pass matches individual test results

**Step 5:** Write `pdp-section-sequence.json` to the project output directory.

## Output Schema

`pdp-section-sequence.json` — follows the exact JSON schema defined in PDP-04-AGENT.md under `PDP-SECTION-SEQUENCE.JSON SCHEMA`.

Key structural requirements:
- `schema_version`: "1.0"
- `pdp_section_taxonomy`: 16 entries (all NLS sections with included/rationale)
- `sections[]`: Array of section objects, each fully populated
- `sticky_atc_trigger_point`: Full trigger specification
- `cta_placement_summary`: All CTA positions + sticky bar + total count
- `proof_wave_summary`: 3-zone summary
- `logic_validation`: 4 test results + all_pass boolean
- `downstream_handoffs`: 8-skill mapping (PDP-05, PDP-06, PDP-07, LP-08, LP-09, LP-12, LP-13, LP-15)

---

## Quality Criteria
- JSON is valid and parseable — no syntax errors
- Every field populated — zero empty strings, nulls, or placeholders
- Section numbers sequential with no gaps
- Word count totals are internally consistent (sum of targets = total target)
- Persuasion functions use anti-slop-verified descriptions (corrections applied)
- All 16 taxonomy sections present in pdp_section_taxonomy (included AND excluded)
- Sticky ATC trigger point fully defined
- Logic validation all_pass matches individual test results
- Module independence confirmed for every section

## Failure Modes
- GATE_3 not passed -> HALT. Cannot compile unvalidated sequence.
- Upstream file missing data for a required field -> Do NOT use placeholder. HALT and identify which upstream output is incomplete.
- JSON syntax error on write -> Validate JSON before finalizing. Re-format if needed.
- Section count mismatch (sections array length != total_btf_sections) -> Fix before writing. This is a compilation error.
- Word count sum mismatch (sum of section targets != estimated_btf_word_count.target) -> Recalculate. Either adjust total or adjust individual targets.
- Anti-slop corrections not applied -> HALT. Check 3.3 for corrections and apply them.
- Taxonomy block has fewer than 16 entries -> HALT. All 16 NLS sections must appear (included or excluded with rationale).
