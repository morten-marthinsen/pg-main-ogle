# 4.3 — Log Writer

**Skill:** LP-13-urgency-scarcity
**Position:** Layer 4, Microskill 3
**Purpose:** Write the execution log documenting LP-13's full processing chain — what was loaded, what was decided, what was generated, what was validated, and what was packaged. This file serves as the audit trail. If any downstream issue arises with urgency copy, the execution log shows exactly what happened and why.
**Effort Level:** low
**Dependencies:** All Layer 0-4 output files
**Output:** `execution-log.md`

---

## Input Schema

```yaml
input:
  all_layer_outputs:
    layer_0: "[packages-loaded.md, urgency-data.md, specimens-loaded.md]"
    layer_1: "[urgency-types-selected.md, justification-plan.md, placement-strategy.md]"
    layer_2: "[countdown-copy.md, stock-level-copy.md, price-deadline-copy.md, availability-copy.md, urgency-support-copy.md -- as applicable]"
    layer_3: "[urgency-audit.md, justification-audit.md]"
    layer_4: "[urgency-package.json, URGENCY-SUMMARY.md]"
  execution_metadata:
    start_time: "[ISO timestamp]"
    end_time: "[ISO timestamp]"
    model_used: "[model identifier]"
```

---

## Execution Logic

1. **Document each layer's execution** in chronological order.
2. **Record gate results** at each transition point.
3. **Note any deviations** from the standard flow (skipped microskills, no-urgency path, revision cycles).
4. **Record the final output status** — all three files present and valid.

## Quality Criteria
- Every microskill execution documented with status and output file name
- Every gate result documented with specific data points
- Revision history captured (if any revisions occurred)
- Deviations from standard flow explicitly noted
- Final output verification shows all three files present
- Justification-before-generation sequence verified
- Log is factual and complete — no fabrication of execution history

## Failure Modes
- Writing the log before execution is complete -> Log must be the LAST file written, documenting the full execution.
- Omitting revision cycles from the log ("the first pass was fine") -> If a revision occurred (anti-slop fail, justification audit fail), document it. The log must be honest.
- Not verifying justification preceded generation -> The sequence Layer 1.2 -> Layer 2 is critical. If Layer 2 ran without a justification plan, this is a protocol violation worth flagging even if the output quality is acceptable.
- Logging a COMPLETE status when output files are missing -> Verify all three files exist before writing COMPLETE.
