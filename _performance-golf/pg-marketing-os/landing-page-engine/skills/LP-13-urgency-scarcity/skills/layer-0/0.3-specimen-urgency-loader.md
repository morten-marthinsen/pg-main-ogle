# 0.3 — Specimen Urgency Loader

**Skill:** LP-13-urgency-scarcity
**Position:** Layer 0, Microskill 3
**Purpose:** Load real urgency and scarcity patterns from specimen files before any generation begins. Specimens function as structural attractors — they show what justified urgency copy looks like in practice, preventing the model from defaulting to generic "act now" language.
**Effort Level:** low
**Dependencies:** `packages-loaded.md` (0.1), `urgency-data.md` (0.2)
**Output:** `specimens-loaded.md`

**Step 2:** Load specimens prioritized by relevance to identified urgency types and page type. Use these known specimen files and their urgency patterns:

**Step 3:** For each loaded specimen, extract:
- The exact urgency/scarcity language used (verbatim)
- The urgency TYPE (from 6-type taxonomy)
- Whether the urgency is JUSTIFIED (explanation provided) or ASSERTED (no justification)
- The placement on page (where urgency appears relative to offer)
- Word count of urgency copy elements

**Step 4:** Load minimum quantities:
- At least 2 urgency pattern examples relevant to the selected page type
- At least 1 example matching a likely urgency type from `urgency-data.md`
- If no specimen matches the exact urgency type, load the closest match and note the gap

## FALLBACK SPECIMEN PATTERNS (Use When Raw Specimens Insufficient)

### Type A Urgency Patterns (Long-Form)

**Supply Scarcity Pattern:**
"We produce [Product] in small batches of [number] to maintain [quality reason]. Each batch takes [timeframe] to produce. When this batch is gone, the next one won't be available until [date/timeframe]."
- Technique: Manufacturing transparency + specific numbers + timeline

**Time-Based Pattern:**
"This [descriptor] pricing is only available during [window]. After [specific date], the investment increases to [$amount]. This isn't a marketing tactic — [real reason for the deadline]."
- Technique: Named deadline + explicit reason + anti-manipulation disclaimer

**Natural Urgency Pattern:**
"Every [time unit] you continue with [problem], [specific consequence with data]. Research from [source] shows that [specific finding about delay consequences]."
- Technique: Research-backed cost of delay + specific deterioration data

### Type B Urgency Patterns (Ecomm/PDP)

**Stock Level Pattern:**
"Only [number] left in stock" / "Low stock — [number] remaining"
- Technique: Specific number, no vague "selling fast" language

**Price Deadline Pattern:**
"Sale ends [specific date]" / "Launch price — ends [day]" / "Save [amount] — offer ends [date/time]"
- Technique: Specific end date, specific savings amount

**Limited Edition Pattern:**
"Limited Edition — [variant name]" / "Available through [date] only"
- Technique: Named variant + specific availability window

## Quality Criteria
- At least 2 urgency pattern specimens loaded before generation
- Each specimen has verbatim urgency language extracted (not paraphrased)
- Each specimen's urgency type classified against the 6-type taxonomy
- Each specimen evaluated for whether justification is provided or just asserted
- Key technique extracted per specimen -- what SPECIFICALLY makes this urgency copy work
- Gap analysis identifies what types are NOT covered by specimens

## Failure Modes
- Loading zero specimens and proceeding to generation -> HALT. At minimum, load fallback patterns if no raw specimens match.
- Paraphrasing specimen urgency language instead of extracting verbatim -> Use exact text. Paraphrasing loses the voice and specificity patterns.
- Loading specimens but not extracting the justification technique -> The justification is the most important part. "Only 47 left" is meaningless without understanding whether/how the specimen EXPLAINED the constraint.
- Loading only specimens with fabricated urgency as examples -> Specimens with unjustified urgency should be loaded as NEGATIVE examples (what NOT to do), explicitly labeled.
