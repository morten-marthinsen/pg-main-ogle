# 2.1 — Countdown Copy Writer

**Skill:** LP-13-urgency-scarcity
**Position:** Layer 2, Microskill 1
**Purpose:** Write time-based urgency copy — deadlines, enrollment windows, seasonal windows, and bonus expirations. Every piece of countdown copy must name a specific date/time AND explain what happens when the countdown ends. This microskill ONLY executes when Type 2 (Time-Based) or Type 5 (Bonus Deadline) is selected in Layer 1.
**Effort Level:** high
**Dependencies:** `justification-plan.md` (1.2), `placement-strategy.md` (1.3), `specimens-loaded.md` (0.3), `urgency-types-selected.md` (1.1)
**Output:** `countdown-copy.md`

## Pre-Generation Protocol

**Step 1: Read authorized claims.** Open justification-plan.md. List the specific claims Layer 2 is authorized to make for this type. Do NOT generate claims beyond this list.

**Step 2: Read specimens.** Hold relevant countdown/deadline specimens in context. Extract their technique: How did they make the deadline feel real and urgent without being pushy?

**Step 3: Read voice direction.** Urgency copy must match the page's voice. A calm, authority-driven page uses measured urgency ("This pricing is available through Friday, March 14th"). A more energetic page can use direct urgency ("After midnight Friday, this price is gone").

**Step 4: Confirm specifics.** Before writing, verify you have:
- Exact date and time for the deadline
- Exact consequence (price change amount, access removal, bonus removal)
- Exact reason the deadline exists

If ANY of these are missing, do NOT generate countdown copy. Write a placeholder framework and flag for client input.

---

## Execution Logic

### For Type A Pages

1. **Write the urgency section headline** (for the dedicated urgency section):
   - Generate 3 headline options
   - Each must reference the deadline OR the consequence -- never generic ("Act Now" is forbidden)
   - Example patterns:
     - "This [descriptor] pricing ends [specific date]"
     - "After [date], [specific consequence]"
     - "You have until [date] to [specific action at this price/terms]"

2. **Write the urgency section body copy** (50-200 words for Type A):
   - Paragraph 1: State the deadline clearly and specifically
   - Paragraph 2: Explain WHY the deadline exists (the real constraint)
   - Paragraph 3: State what happens after the deadline passes (specific consequence)
   - Optional: Include a brief "I want to be direct about this" framing to signal honesty

3. **Write the P.S. countdown callback** (1-2 sentences):
   - Restate the deadline and consequence in compressed form
   - Link to the urgency section's logic without repeating it verbatim

4. **Write the close section countdown line** (1 sentence):
   - Shortest possible restatement of the deadline for the final CTA area

### For Type B Pages

5. **Write the countdown timer label** (3-15 words):
   - This appears above or below a visual countdown timer
   - Must state what the timer counts down TO
   - Example: "Launch pricing ends in:" / "Sale ends at midnight EST:"
   - NEVER just "Hurry!" or "Don't miss out!" -- must be specific

6. **Write the countdown explanation sentence** (1 sentence):
   - Appears below the timer
   - Explains what happens when the timer reaches zero
   - Example: "After this timer reaches zero, the price returns to $97."

7. **Write the price deadline badge copy** (3-8 words):
   - For product card or price display
   - Example: "Launch Price -- Ends Sunday" / "Introductory Rate -- Save $30"

### For Type 5 (Bonus Deadline) Additions

8. **Write bonus expiration copy** (per bonus or as a group):
   - State which bonuses expire
   - State when they expire
   - State why they expire (partner agreement, licensing, seasonal)
   - Example: "The 3 bonus trainings above are available through Friday because our guest experts donated their time for launch week only. After Friday, the core program remains -- but these bonuses will not be included."

## Quality Criteria
- Specific date/time named in every countdown element (no "soon" or "limited time")
- Consequence of deadline passing stated explicitly (no "you'll miss out")
- Reason for deadline explained (not just asserted)
- All copy stays within authorized claims from justification-plan.md
- Voice direction matched -- urgency tone is consistent with rest of page
- Zero forbidden words from anti-slop list

## Failure Modes
- Writing "limited time offer" without a specific end date -> Every countdown must name the date/time. Period.
- Stating the deadline without explaining what happens after -> "Ends Friday" is incomplete. "Ends Friday -- price returns to $97" is complete.
- Writing countdown copy that is significantly more aggressive than page voice -> If the page is calm and authoritative, the urgency copy must be calm and authoritative. Tonal whiplash = distrust.
- Generating claims not in the authorized list -> Re-read justification-plan.md. Only write what is pre-justified.
- Using placeholder dates ("[DATE]") in final copy -> If dates are unknown, write the framework and flag for client input. Do not present as final copy.
