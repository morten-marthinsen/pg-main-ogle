## Placement Rules

### Universal Rules (All Page Types)

1. **Early trust anchor:** At least 1 proof element must appear at or before the first-25% section boundary. Type: authority signal, media mention, volume counter, or rating strip.

2. **No proof deserts:** No 3+ consecutive sections may have density = "none." If the wave pattern creates a desert, insert a light proof element (embedded credibility mention, quick stat) to break it.

3. **Highest-ranked proof in highest-visibility slots:** Clinical studies go in mechanism/ingredient sections. Expert endorsements go in lead/product intro. Testimonials go in dedicated proof blocks. Volume signals go above fold or in social proof strips.

4. **Transition framing:** Each proof block needs a transition-in phrase direction (e.g., "But don't just take my word for it...") and optionally a transition-out direction (e.g., "And that's just one story...").

### Type A Placement Rules

| Section Type | Proof Assignment |
|-------------|-----------------|
| Above fold | 1-2 authority signals (trust bar, media logos) |
| Lead / Hook | 1 embedded credibility mention (expert name, study reference) |
| Story | Embedded only (Yoda credentials, institutional names woven into narrative) |
| Root cause | 2-3 scientific supports woven into text (not standalone blocks) |
| Mechanism | 3-5 clinical supports / demonstrations |
| Proof Block 1 | DEDICATED: 2-4 testimonials + 1 study/data point |
| Product Introduction | 1 authority endorsement |
| Ingredient deep-dives | 1 study + 1 testimonial per ingredient (medium density per) |
| Proof Block 2 | DEDICATED: 5-8 testimonials (cascade — names, locations, outcomes) |
| Value stack / offer | 1 value testimonial ("Worth every penny" type) |
| Guarantee | Confidence proof (visual certificate, claim stat) |
| Proof Block 3 (pre-close) | DEDICATED: Transformation proof + callbacks to lead promise |
| FAQ | Embedded: proof woven into 2-3 answers |
| Close / P.S. | 1-2 benefit summary callbacks referencing earlier proof |

### Type B Placement Rules

| Section Type | Proof Assignment |
|-------------|-----------------|
| Above fold | Rating strip + review count (IMMEDIATE and prominent) |
| Key benefits | Badge icons only (GMP, etc.) |
| Variant/quantity | None |
| Trust bar | Trust badges (certifications, guarantee, shipping) |
| How it works | None (illustration section) |
| Ingredient deep-dives | 1 study reference per key ingredient |
| Social proof strip | Volume counter + stat bar ("50,000+ customers") |
| Before/after gallery | Full gallery (if applicable) |
| Review cascade | MAXIMUM: 25-50+ reviews with filtering |
| FAQ | Embedded: proof in 2-3 answers |
| Guarantee | Branded guarantee badge/certificate |
| Comparison table | Competitive proof (checkmarks vs. X marks) |

## Execution Logic

**Step 1:** List every section from section-sequence.json in order.

**Step 2:** For each section, check the wave pattern zone assignment (from 1.3). This determines the density range for this section.

**Step 3:** Assign proof placements using the placement rules tables above. Match section type to the appropriate proof assignment.

**Step 4:** For AUTHORITY_SUBSTITUTION mode, modify assignments:
- Replace testimonial blocks with expert endorsement blocks or performance guarantee blocks
- Reduce testimonial counts to available inventory
- Increase authority signal placements
- Add performance guarantee as standalone proof element near close

**Step 5:** Verify no proof deserts (3+ consecutive sections with "none").

**Step 6:** Verify early trust anchor (proof at or before first-25% section).

**Step 7:** Mark each proof element as `available` (asset exists in inventory) or `requires_sourcing` (gap — asset must be created/collected).

## Quality Criteria
- Every section has an explicit assignment (including "none")
- Assignments match the wave pattern zone (no heavy proof in the "none" zones of Type B benefits section)
- No proof deserts
- Early trust anchor present
- Each element marked as available or requires_sourcing
- AUTHORITY_SUBSTITUTION mode adjustments applied if applicable

## Failure Modes
- Proof desert detected -> Insert a light proof element (embedded credibility mention) to break it
- No proof available for early sections -> Flag: "EARLY TRUST GAP — No proof assets available for sections 1-[N]. Add media mention, certification badge, or social proof counter."
- All proof elements marked requires_sourcing (standalone mode) -> Acceptable, but note: "All proof requires sourcing. This architecture is a proof requirements document, not a proof placement of existing assets."
