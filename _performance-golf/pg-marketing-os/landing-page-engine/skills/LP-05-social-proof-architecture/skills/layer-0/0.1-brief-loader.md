# 0.1 — Brief & Sequence Loader

**Skill:** LP-05-social-proof-architecture
**Position:** Layer 0, Microskill 1
**Purpose:** Load page-brief.json and section-sequence.json from upstream skills. Confirm page type, operating mode, and section count before any proof work begins.
**Effort Level:** low
**Dependencies:** LP-00 output (page-brief.json), LP-04 output (section-sequence.json)
**Output:** `brief-and-sequence-loaded.md`

---

## Input Schema

```yaml
input:
  project_folder: "[path to project output folder]"
  page_brief_path: "[project_folder]/lp-00-brief-classifier/page-brief.json"
  section_sequence_path: "[project_folder]/lp-04-section-sequence/section-sequence.json"
```

---

## Execution Logic

**Step 1:** Locate and load `page-brief.json`. Extract:
- `page_type.classification` (type_a | type_b | hybrid)
- `operating_mode` (downstream | standalone)
- `proof_inventory` block (testimonials_count, has_clinical_studies, has_before_afters, has_expert_endorsements, proof_inventory_path)
- `traffic.temperature` (cold | warm | hot)
- `traffic.awareness_stage` (1-5)
- `vertical.primary`

**Step 2:** Locate and load `section-sequence.json`. Extract:
- `total_sections`
- `sections[]` array — section_id, section_name, proof_density (from LP-04 density plan)
- `proof_wave_summary` (early/mid/close zones from LP-04)
- `page_type`

**Step 3:** Cross-validate page type: `page-brief.json` page_type MUST match `section-sequence.json` page_type. If mismatch -> HALT with error.

**Step 4:** Calculate first-25% section boundary: `total_sections * 0.25` (rounded up). This is the cutoff for early trust anchoring verification.

**Step 5:** Confirm operating mode implications:
- Downstream: Proof inventory likely has real data. Expect testimonial counts, study references.
- Standalone: Proof inventory likely sparse or `not_available`. LP-05 will generate proof REQUIREMENTS rather than mapping real assets.

## Quality Criteria
- Both files located and loaded
- Page type confirmed from both sources and matches
- Section list complete with LP-04 density assignments
- First-25% boundary calculated
- Operating mode documented

## Failure Modes
- `page-brief.json` not found -> HALT: "LP-05 requires LP-00 output. Run LP-00 first."
- `section-sequence.json` not found -> HALT: "LP-05 requires LP-04 output. Run LP-04 first."
- Page type mismatch -> HALT: "Page type in page-brief.json ([X]) does not match section-sequence.json ([Y]). Resolve before proceeding."
