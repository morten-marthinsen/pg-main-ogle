**Skill:** LP-05-social-proof-architecture
**Position:** Layer 1, Microskill 3
**Purpose:** Select the correct proof wave pattern for this page type AND select the proof operating mode (PROOF_SATURATION vs. AUTHORITY_SUBSTITUTION). These two decisions govern every placement decision in Layer 2.

## Wave Pattern Definitions

### Type A Wave (`type_a_wave`)
Oscillating rhythm: light -> embedded -> medium -> dense -> HEAVY -> light -> medium -> HEAVY -> light -> confidence -> HEAVY -> embedded -> medium

Maps to persuasion arc: earn attention -> build credibility -> prove mechanism -> cascade belief -> establish value -> confirm confidence -> close with transformation.

### Type B Wave (`type_b_wave`)
Volume-first, immediate: HEAVY (above fold) -> light -> none -> medium -> HEAVY (social strip) -> HEAVY (before/after) -> MAXIMUM (review cascade) -> confidence

Maps to decision process: instant trust -> scan benefits -> understand product -> verify with volume -> make purchase decision.

### Hybrid Wave (`hybrid_wave`)
Zone 1 (Type B above fold): HEAVY immediate proof
Zone 2 (Type A education): embedded -> medium -> dense proof following persuasion arc
Zone 3 (Conversion): review cascade + testimonial blocks + guarantee proof

---

## Proof Mode Selection Logic

```
Mode selection decision tree:

1. Is there a performance guarantee available AND page is B2B/high-ticket?
   -> If YES: Consider AUTHORITY_SUBSTITUTION (performance guarantee IS the proof)

2. How many testimonials are available?
   -> >= 15: PROOF_SATURATION eligible
   -> 5-14: PROOF_SATURATION with PROOF_GAP flags
   -> < 5: AUTHORITY_SUBSTITUTION

3. Are clinical studies OR expert endorsements available?
   -> If YES and testimonials >= 5: PROOF_SATURATION (studies/experts anchor the strategy)
   -> If YES and testimonials < 5: AUTHORITY_SUBSTITUTION (authority-led)

4. Is there a strong brand/celebrity authority available?
   -> If YES and testimonials < 10: AUTHORITY_SUBSTITUTION (brand IS proof)

5. Default: If testimonials >= 5, default to PROOF_SATURATION with gaps flagged
```

---

## Awareness Stage Proof Timing Adjustments

| Stage | Wave Adjustment |
|-------|----------------|
| Stage 1 (Unaware) | Delay testimonials until after root cause. Authority proof early. |
| Stage 2 (Problem-Aware) | Authority proof early. Testimonials mid-page. Standard wave. |
| Stage 3 (Solution-Aware) | Testimonials can appear earlier. Differentiation proof prioritized. |
| Stage 4 (Product-Aware) | Volume proof early. Everyone-uses-this messaging front-loaded. |
| Stage 5 (Most Aware) | Heavy proof immediately. Price + proof above fold. |

---

## Execution Logic

**Step 1:** Determine wave pattern based on page type:
- `type_a` -> `type_a_wave`
- `type_b` -> `type_b_wave`
- `hybrid` -> `hybrid_wave`

This is NOT a creative decision. Wave pattern must match page type.

**Step 2:** Run proof mode selection logic using the decision tree above. Consider inventory, vertical, and specimen patterns.

**Step 3:** Apply awareness stage adjustment to determine when within the wave heavy proof should appear.

**Step 4:** Document the complete wave pattern with zone boundaries mapped to actual section numbers from section-sequence.json.

## Quality Criteria
- Wave pattern matches page type exactly (no conflation)
- Proof mode has a clear rationale tied to inventory data
- Awareness stage adjustment documented
- Zone boundaries mapped to actual section numbers
- First proof position confirmed within first 25% of page

## Failure Modes
- Wave pattern does not match page type -> HALT. Re-select.
- No proof available for early sections -> Flag as critical: "No proof assets available for early trust anchoring. Recommend adding media mentions, certifications, or social proof counter to inventory."
