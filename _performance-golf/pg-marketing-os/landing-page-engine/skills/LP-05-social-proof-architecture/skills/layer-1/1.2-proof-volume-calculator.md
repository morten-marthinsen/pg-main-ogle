# 1.2 — Proof Volume Calculator

**Skill:** LP-05-social-proof-architecture
**Position:** Layer 1, Microskill 2
**Purpose:** Calculate the total proof volume needed for this page type against conversion data thresholds. Compare required volume to available inventory. Determine if the project can hit thresholds or needs gap-filling.
**Effort Level:** medium
**Dependencies:** `proof-classification.md` (1.1), `brief-and-sequence-loaded.md` (0.1)
**Output:** `volume-requirements.md`

---

## Input Schema

```yaml
input:
  page_type: "[type_a | type_b | hybrid]"
  total_sections: "[from section-sequence.json]"
  proof_classification: "[from 1.1 — classified assets with counts]"
  traffic_temperature: "[cold | warm | hot]"
  awareness_stage: "[1-5]"
```

## Traffic Temperature Adjustments

| Temperature | Adjustment |
|-------------|-----------|
| **Cold** | +25% proof volume over base. Cold traffic has zero prior trust. More proof needed to establish credibility. |
| **Warm** | Base volume. Standard thresholds apply. |
| **Hot** | -15% proof volume acceptable. Visitor already trusts the brand. Proof confirms rather than convinces. |

---

## Execution Logic

**Step 1:** Load page type and classified proof totals from upstream.

**Step 2:** Look up minimum and recommended thresholds for this page type from the tables above.

**Step 3:** Apply traffic temperature adjustment to the minimum threshold.

**Step 4:** Compare available proof (from 1.1 classification totals) against adjusted threshold:
- `meets_minimum`: Available >= adjusted minimum for all categories
- `meets_recommended`: Available >= recommended for all categories
- `below_minimum`: Available < adjusted minimum for one or more categories
- `critical_deficit`: Available < 50% of adjusted minimum

**Step 5:** Calculate the volume gap per proof type:
- Gap = adjusted_minimum - available_count
- If gap > 0: flag as proof volume shortfall
- If gap <= 0: sufficient for this type

**Step 6:** Calculate total planned proof touches. This is the number LP-05 will distribute across sections in Layer 2.

## Quality Criteria
- Correct thresholds applied for page type
- Traffic temperature adjustment applied
- Every proof type compared against threshold
- Gaps calculated with severity
- Total touches meets page type minimum

## Failure Modes
- Total available proof < 50% of minimum -> Flag as `critical_deficit`. Recommend AUTHORITY_SUBSTITUTION mode.
- Page type is type_b but fewer than 25 reviews available -> Flag: "Minimum 50 reviews needed for Type B review cascade. Consider delaying launch until review volume achieved."
