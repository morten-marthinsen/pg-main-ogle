# 2.3 — Proof Test Generator

> **Skill:** LP-18-ab-test-plan
> **Layer:** 2
> **Position:** 3 of 5
> **Purpose:** Generate social proof A/B test hypotheses covering proof placement (above vs below fold), proof density, video vs text testimonials, review count display, and proof format.
> **Effort Level:** opus
> **Dependencies:** `prioritized-opportunities.md`, `assembled-page-load.md`, `benchmark-load.md`
> **Output:** `proof-tests.md`

---

## INPUT

```yaml
input:
  prioritized_opportunities: "prioritized-opportunities.md"
  assembled_page: "assembled-page-load.md"
  benchmarks: "benchmark-load.md"
  page_type: "[from assembled-page-load.md]"
```

---

## EXECUTION

**POSITIONAL REINFORCEMENT:** You are generating SPECIFIC proof test hypotheses. Every hypothesis must cite the exact current proof state and propose a specific, implementable change. "Add more social proof" is forbidden.

### Step 1: Check if Proof Tests Are Warranted

Read `prioritized-opportunities.md`. Only proceed if proof-category opportunities appear. If none, write `proof-tests.md` with: "No proof test opportunities prioritized for this page." and exit.

### Step 2: Extract Current Proof State

From `assembled-page-load.md`, extract:
- Total proof element count
- Proof types present (testimonials, reviews, citations, endorsements, stats, badges)
- First proof position (above fold / section 2-3 / deep)
- Proof density classification (ZERO / LOW / MEDIUM / HIGH / EXTREME)
- Proof specificity (specific outcomes with numbers vs vague endorsements)
- Before/after usage (present / absent)
- Video testimonials (present / absent)

### Step 3: Generate Proof Test Hypotheses

For each proof opportunity, generate 1-2 specific hypotheses from these test dimensions:

**Dimension A: Proof Placement**
From cross-page-pattern-analysis.md — first proof position varies dramatically:
| Current Position | Test Against | Reasoning |
|-----------------|-------------|-----------|
| Section 4-5 (deep) | Above fold or Section 2 | Earlier proof shortens time-to-trust. CXL: "early placement = significant lift." |
| Above fold | Section 3 (after education) | For complex products, education before proof may increase proof comprehension |
| Concentrated in one section | Distributed throughout page | Each proof element placed near the claim it supports |

**Dimension B: Proof Density**
From cross-page-pattern-analysis.md — bimodal distribution (extreme or zero):
| Current Density | Test Against | Reasoning |
|----------------|-------------|-----------|
| LOW (3-5 elements) | HIGH (15+ elements) | Volume creates inevitability. BIOptimizers: 1,561 reviews + 38 citations |
| MEDIUM (6-14 elements) | HIGH or EXTREME | The middle ground underperforms both extremes in the specimen data |
| HIGH | EXTREME (add citations, expert endorsements) | Layering proof types (reviews + expert + scientific) compounds credibility |

**Dimension C: Proof Format**
| Current Format | Test Against | Reasoning |
|---------------|-------------|-----------|
| Text-only testimonials | Video testimonials | Video proof is more credible and harder to fake. 4/11 specimens use video. |
| Individual testimonials | Aggregate display ("83,502 5-star reviews") | Volume number creates social proof through scale. LMNT pattern. |
| Generic testimonials ("Changed my life") | Specific-outcome testimonials ("Lost 30 lbs in 8 weeks") | Specific numbers are more credible and more persuasive. |
| No before/after | Before/after section | For applicable categories (supplements, fitness, skincare). Notable: 0/11 specimens used before/after — competitive gap. |

**Dimension D: Proof Type Addition**
| Current Types | Test Adding | Reasoning |
|--------------|------------|-----------|
| Testimonials only | Expert endorsements | Authority proof + peer proof = compound credibility. BIOptimizers uses both. |
| Reviews only | Scientific citations | Evidence-based proof for health/supplement pages. BIOptimizers: 38 citations. |
| No performance guarantee | Performance guarantee | "15% more or don't pay" (Hyros pattern). Functions as proof + risk reversal + differentiation. B2B-specific. |
| No celebrity/influencer | Celebrity endorsement | Borrowed authority — most effective when placed above fold (Hyros pattern). |

### Step 4: Write Each Hypothesis Using the Schema

```yaml
test_id: "T-[N]"
category: "proof"
hypothesis: "If we change [exact proof element] from [control state] to [variant state], conversion rate will increase by [X-Y%] because [reasoning]."
control:
  description: "[current proof state]"
  specific_text_or_element: "[exact current proof — count, type, position]"
variant:
  description: "[proposed change]"
  specific_text_or_element: "[exact proposed change — what to add/move/replace, where, how many]"
expected_lift: "[X-Y%]"
lift_source: "[citation]"
implementation_difficulty: "[low | medium | high]"
```

### Step 5: Write Proof Tests

Compile all proof test hypotheses into `proof-tests.md`.

---

## QUALITY CRITERIA

- Only generates tests if proof opportunities were prioritized
- Current proof state fully documented from assembled page
- Each hypothesis specifies exact control state (count, types, positions)
- Variant descriptions are specific (not "add more proof" — specify: what type, how many, where)
- Lift estimates cite CXL early placement data, specimen patterns, or benchmark data
- Implementation difficulty reflects reality (adding video testimonials = medium-high, moving text block = low)
- File exists after execution

---

## FAILURE MODES

- Proof inventory incomplete from assembled page -> Note gaps. Propose tests based on available data. Flag as INCOMPLETE CONTROL.
- Proposing proof additions that require content creation (new testimonials, new video) -> Mark implementation difficulty as "high" and note: "Requires sourcing new proof content."
- Proposing before/after for a category where it is inappropriate (SaaS, B2B services) -> Only propose before/after for: supplements, weight loss, skincare, fitness, personal transformation. Auto-skip for other categories.
