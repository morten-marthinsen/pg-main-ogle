# 2.1 — Headline Test Generator

> **Skill:** LP-18-ab-test-plan
> **Layer:** 2
> **Position:** 1 of 5
> **Purpose:** Generate headline A/B test hypotheses with specific control/variant pairs. Cover headline type, emotional appeal, length, and specificity as test dimensions.
> **Effort Level:** opus
> **Dependencies:** `prioritized-opportunities.md`, `assembled-page-load.md`, `benchmark-load.md`
> **Output:** `headline-tests.md`

---

## INPUT

```yaml
input:
  prioritized_opportunities: "prioritized-opportunities.md"
  assembled_page: "assembled-page-load.md"
  benchmarks: "benchmark-load.md"
  page_type: "[from assembled-page-load.md]"
  awareness_stage: "[from assembled-page-load.md page brief data]"
```

---

## EXECUTION

**POSITIONAL REINFORCEMENT:** You are generating SPECIFIC headline test hypotheses. Every hypothesis must have the exact current headline text as the control and a specific alternative as the variant. "Test a different headline" is forbidden. You are not writing final copy — you are specifying what TYPE of headline to test and providing enough detail that a copywriter can execute it.

### Step 1: Check if Headline Tests Are Warranted

Read `prioritized-opportunities.md`. Only proceed if headline-category opportunities appear in the selected list. If no headline opportunities were prioritized, write `headline-tests.md` with content: "No headline test opportunities prioritized for this page." and exit.

### Step 2: Extract Current Headline State

From `assembled-page-load.md`, extract:
- Primary headline: exact text
- Subheadline/deck: exact text (if present)
- Headline type classification: result/outcome, how-to/method, curiosity, bold claim, community, guarantee-lead, product name

### Step 3: Generate Headline Test Hypotheses

For each headline opportunity in the prioritized list, generate 1-2 specific test hypotheses. Draw from the following test dimensions:

**Dimension A: Headline Type Change**
Test the current headline type against a different type:
| Current Type | Test Against | Reasoning |
|-------------|-------------|-----------|
| Result/outcome | How-to/method | Method headlines activate curiosity about the process |
| How-to/method | Result/outcome | Result headlines cut to the transformation |
| Bold claim | Result/outcome | Specificity often outperforms boldness |
| Generic benefit | Specific mechanism | Naming the mechanism increases credibility |
| Product name | Benefit-first | Benefit before brand for cold traffic |

**Dimension B: Emotional Appeal Change**
Test the current emotional angle against an alternative:
- Fear/pain-agitation vs aspiration/desire
- Curiosity vs direct benefit
- Urgency vs authority
- Social proof headline ("Join 10,000+ who...") vs outcome headline

**Dimension C: Length/Format Change**
- Short headline (5-8 words) vs long headline (15-25 words)
- Headline only vs headline + subheadline deck
- Single-line vs two-line (broken with period or em-dash)

**Dimension D: Specificity Level Change**
- General audience vs specific audience named ("Golfers over 50:")
- General outcome vs specific outcome with number ("Lower Your Scores by 7 Strokes")
- Unnamed mechanism vs named mechanism ("The 3-Pivot Method")

### Step 4: Write Each Hypothesis Using the Schema

For each hypothesis, provide:

```yaml
test_id: "T-[N]"
category: "headline"
hypothesis: "If we change the headline from [exact current] to [specific alternative type/approach], conversion rate will increase by [X-Y%] because [reasoning based on awareness stage, page type, and benchmark data]."
control:
  description: "[Current headline type and approach]"
  specific_text_or_element: "[EXACT current headline text from assembled page]"
variant:
  description: "[Proposed headline type and approach]"
  specific_text_or_element: "[Specific direction for the alternative — e.g., 'Result-outcome headline naming the 3-Pivot Method and a specific score improvement number. Example direction: Find 7 More Fairways Per Round With The 3-Pivot Method']"
expected_lift: "[X-Y%]"
lift_source: "[conversion-data-reference.md — personalized headline +15-30%, headline variance 5-500%]"
implementation_difficulty: "low"
```

**CRITICAL:** The variant `specific_text_or_element` must contain enough direction to implement. Acceptable: "Curiosity-driven headline that names the mechanism without revealing it. Example direction: The Strange 3-Pivot Technique That Eliminated My Slice In One Round." Not acceptable: "A different headline."

### Step 5: Write Headline Tests

Compile all headline test hypotheses into `headline-tests.md`.

---

## OUTPUT SCHEMA

`headline-tests.md`:

```markdown
# Headline Test Hypotheses — [Project Name]

## Tests Generated: [N]

## Current Headline State
- **Primary headline:** "[exact text]"
- **Subheadline:** "[exact text or N/A]"
- **Current type:** [classification]
- **Page type:** [Type A / Type B / Hybrid]
- **Awareness stage:** [stage]

---

### T-[N]: [Test Name]

**Hypothesis:** If we change the headline from "[control text]" to [variant approach], conversion rate will increase by [X-Y%] because [reasoning].

**Control:**
- Description: [current approach]
- Text: "[exact current headline]"

**Variant:**
- Description: [proposed approach]
- Direction: "[specific enough to brief a copywriter — include example direction]"

**Expected Lift:** [X-Y%]
**Lift Source:** [citation]
**Implementation Difficulty:** low
**Test Dimension:** [type change | emotional appeal | length | specificity]

---

[...repeat for each hypothesis...]
```

---

## QUALITY CRITERIA

- Only generates tests if headline opportunities were prioritized
- Current headline extracted verbatim from assembled page
- Each hypothesis specifies exact control text
- Each hypothesis provides variant direction specific enough to implement
- Expected lift cites benchmark data
- Each test targets a single dimension (type, emotion, length, or specificity — not all at once)
- Implementation difficulty is consistently "low" for headline tests (copy-only change)
- File exists after execution

---

## FAILURE MODES

- Current headline text not available -> Flag as INCOMPLETE CONTROL. Provide the hypothesis with a note that the actual headline text must be extracted from the live page.
- Too many headline test ideas (4+) -> Limit to 2 best hypotheses per prioritized opportunity. Layer 3 will validate and potentially trim further.
- Variant description is vague -> Re-read the variant. If it could describe many different headlines, it is too vague. Add specifics: name the approach, give a directional example, specify the key difference from control.
