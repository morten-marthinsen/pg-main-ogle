# 0.1 — Assembled Page Loader

> **Skill:** LP-18-ab-test-plan
> **Layer:** 0
> **Position:** 1 of 3
> **Purpose:** Load the assembled page package from LP-15 and extract the current (control) state of every testable element — headlines, CTAs, proof blocks, offer elements, layout structure.
> **Effort Level:** haiku
> **Dependencies:** `assembled-page-package.json` from LP-15 Page Assembly
> **Output:** `assembled-page-load.md`

---

## INPUT

```yaml
input:
  assembled_page_path: "[path to assembled-page-package.json from LP-15]"
  project_folder: "[path to project output folder]"
```

---

## EXECUTION

1. Locate `assembled-page-package.json` in the project output folder. Search paths:
   - `./outputs/[project-name]/landing-page/lp-15-page-assembly/assembled-page-package.json`
   - `./outputs/[project-name]/lp-15-page-assembly/assembled-page-package.json`

2. Parse the assembled page package. Extract and catalog every testable element with its exact current state:

   **Headlines (all instances):**
   - Primary headline: exact text
   - Subheadlines: exact text for each section
   - Section headers: exact text

   **CTAs (all instances):**
   - CTA text: exact text for each instance
   - CTA position: section name and approximate fold position
   - CTA format: button, text link, sticky bar, etc.

   **Social Proof (all instances):**
   - Proof type: testimonial, review, citation, endorsement, stat, badge
   - Proof position: section and fold position
   - Proof content: summary of each proof element
   - Proof volume: total count by type

   **Offer/Pricing Elements:**
   - Price display: exact format and text
   - Anchoring method: current approach
   - Guarantee: exact text and placement
   - Urgency elements: exact text and type
   - Value stack: structure and order

   **Layout/Structure:**
   - Section inventory: ordered list with section names and purposes
   - Above-fold composition: exact elements present
   - Page length: estimated word count and section count
   - Mobile specifications: sticky bar, touch targets, image formats

3. If `assembled-page-package.json` is not found, search for alternative assembly outputs:
   - `assembled-page-brief.md`
   - `ASSEMBLED-PAGE-SUMMARY.md`
   - Any file in the LP-15 output folder

4. Write the extracted inventory to `assembled-page-load.md`.

## QUALITY CRITERIA

- Assembled page package located and fully parsed
- All testable elements extracted with exact current state (verbatim text where applicable)
- Element positions documented (section location, fold position)
- Layout structure captured (section order, above-fold composition)
- File exists after execution

---

## FAILURE MODES

- Assembled page package not found -> Write `assembled-page-load.md` with status `NOT_FOUND`. Triggers GATE_0 HALT.
- Package found but incomplete (some elements missing) -> Write with status `PARTIAL`, extract what is available, note gaps. Proceed but flag limited control state data.
- Package is very large -> Focus on extractable testable elements: headlines, CTAs, proof blocks, offer elements, layout order. Summarize rather than verbatim-extract body copy paragraphs.
