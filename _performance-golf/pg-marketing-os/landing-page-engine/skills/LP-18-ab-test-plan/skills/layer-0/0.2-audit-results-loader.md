# 0.2 — Audit Results Loader

> **Skill:** LP-18-ab-test-plan
> **Layer:** 0
> **Position:** 2 of 3
> **Purpose:** Load LP-17 conversion audit results and LP-16 mobile audit results. Extract all FAIL points, priority rankings, optimization recommendations, and mobile-specific issues as prioritized test targets.
> **Effort Level:** haiku
> **Dependencies:** `conversion-audit.json` (LP-17), `mobile-audit.json` (LP-16, optional)
> **Output:** `audit-results-load.md`

---

## INPUT

```yaml
input:
  conversion_audit_path: "[path to conversion-audit.json from LP-17]"
  mobile_audit_path: "[path to mobile-audit.json from LP-16 — optional]"
  project_folder: "[path to project output folder]"
```

---

## EXECUTION

1. Locate `conversion-audit.json` in the project output folder. Search paths:
   - `./outputs/[project-name]/landing-page/lp-17-conversion-audit/conversion-audit.json`
   - `./outputs/[project-name]/lp-17-conversion-audit/conversion-audit.json`

2. From `conversion-audit.json`, extract:

   **Audit Overview:**
   - Total score (X/20)
   - Verdict (LAUNCH_READY / MINOR_OPTIMIZATION / SIGNIFICANT_GAPS / STRUCTURAL_REBUILD)
   - Page type

   **All FAIL Points (primary test targets):**
   - Point ID and description
   - FAIL evidence (what is wrong)
   - Fix recommendation (what LP-17 suggested)
   - Expected lift (from LP-17's analysis)
   - Category

   **Priority Ranking (from LP-17):**
   - Full priority-ranked list of FAIL items
   - Priority score for each item

   **Optimization Recommendations:**
   - All specific recommendations from LP-17
   - Implementation steps
   - Skill routing

3. Locate `mobile-audit.json` (LP-16) if available. Search paths:
   - `./outputs/[project-name]/landing-page/lp-16-mobile-audit/mobile-audit.json`
   - `./outputs/[project-name]/lp-16-mobile-audit/mobile-audit.json`

4. From `mobile-audit.json` (if found), extract:
   - Mobile-specific FAIL points
   - Mobile optimization recommendations
   - Mobile performance scores

5. Merge LP-17 and LP-16 results into a unified test target list. LP-17 failures are primary targets. LP-16 failures add mobile-specific test opportunities.

6. Write to `audit-results-load.md`.

## QUALITY CRITERIA

- LP-17 conversion audit loaded and all FAIL points extracted
- Priority ranking preserved from LP-17
- Evidence, fix, and lift data extracted for each FAIL point
- LP-16 mobile audit loaded if available (graceful degradation if not)
- Unified test target list compiled
- File exists after execution

---

## FAILURE MODES

- LP-17 conversion-audit.json not found -> Write `audit-results-load.md` with LP-17 status `NOT_FOUND`. Triggers GATE_0 HALT. LP-18 cannot proceed without audit data.
- LP-16 mobile-audit.json not found -> Write with LP-16 status `NOT_AVAILABLE`. Proceed with LP-17 data only. Note: mobile-specific test opportunities will be limited.
- LP-17 audit has zero FAIL points (score 20/20) -> Load the data but note that all points passed. Layer 1 will need to identify optimization opportunities from PASS points and benchmark comparisons rather than audit failures.
- LP-17 verdict is SIGNIFICANT_GAPS or STRUCTURAL_REBUILD -> Load the data but add a warning: "LP-17 recommends fixes before testing. Consider running LP-18 only on the highest-priority items while structural fixes are implemented."
