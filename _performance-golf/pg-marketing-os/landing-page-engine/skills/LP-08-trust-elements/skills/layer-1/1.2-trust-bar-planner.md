# 1.2 — Trust Bar Planner

**Skill:** LP-08-trust-elements
**Position:** Layer 1, Microskill 2
**Purpose:** Select the 3–5 (Type A) or 4–6 (Type B) highest-impact trust signals for the trust bar and determine their display order. The trust bar is the single most visible trust element on the page — visitors scan it in 0.5–1.5 seconds. Signal selection and ordering directly affect perceived credibility.
**Effort Level:** medium
**Dependencies:** `trust-classification.md` (1.1), `packages-loaded.md` (0.1), `specimen-trust-patterns.md` (0.3)
**Output:** `trust-bar-plan.md`

---

## Input Schema

```yaml
input:
  trust_classification: "[from trust-classification.md — trust bar candidates]"
  page_type: "[type_a | type_b | hybrid]"
  trust_bar_placement: "[from packages-loaded.md — above_fold | below_hero | below_nav]"
  specimen_patterns: "[from specimen-trust-patterns.md — trust bar patterns from specimens]"
  guarantee_details: "[from trust inventory]"
  strongest_differentiator: "[identified from trust inventory — what sets this product apart]"
```

---

## Execution Logic

1. **Determine signal count target:**
   - Type A: 3–5 signals
   - Type B: 4–6 signals
   - Hybrid: 4–5 signals

2. **Rank all trust bar candidates** from trust-classification.md by impact hierarchy:

   | Rank | Signal Type | Impact Rationale |
   |------|-----------|-----------------|
   | 1 | Guarantee (with specific terms) | Risk removal is the #1 purchase barrier |
   | 2 | Certification (most relevant) | Manufacturing/quality credibility |
   | 3 | Volume signal (customer count OR review count) | Social proof via scale |
   | 4 | Shipping/delivery | Logistical trust (reduces "hidden cost" anxiety) |
   | 5 | Security | Checkout-specific trust |
   | 6 | Media mention (short form) | Third-party validation |

3. **Override ranking when appropriate:**
   - If the brand's strongest differentiator is volume (e.g., 83K reviews), move volume to position 1
   - If the brand has an exceptional certification (e.g., NSF Certified for Sport in athletics market), move certification to position 1
   - If media coverage is from an extremely high-authority publication (e.g., "As Seen on Dr. Oz"), move media to position 1

4. **Select signals** — fill positions from the ranked list until target count is reached. If fewer candidates than target, use all available (minimum 3 for any page type).

5. **Apply the left-to-right ordering logic:**
   - Position 1 (leftmost, most scanned): Highest-impact signal or strongest differentiator
   - Positions 2–3: Supporting credibility signals
   - Position 4–5/6 (rightmost): Logistical or secondary signals

6. **Reference specimen patterns** — compare selected signals and ordering against specimen trust bars to validate the approach.

7. **Write `trust-bar-plan.md`.**

## Quality Criteria
- Signal count within range for page type (3–5 Type A, 4–6 Type B)
- Every selected signal has data in the trust inventory (no fabrication)
- Ordering follows impact hierarchy or has documented override rationale
- Copy direction for each signal is 3–5 words (verifying the constraint before Layer 2)
- Signals not selected are documented with rationale (prevents second-guessing in Layer 2)

## Failure Modes
- Selecting more signals than the page type allows → Cognitive overload reduces trust bar effectiveness. Stay within range.
- Selecting fewer than 3 signals → Insufficient to establish the credibility strip pattern. If fewer than 3 candidates exist, flag as a critical gap.
- Including a signal with no data to back it → Every signal must trace to the trust inventory. "Third-Party Tested" requires actual third-party testing data.
- Defaulting to the generic ordering without considering the brand's differentiator → If the brand's strongest signal is volume, it should lead.
