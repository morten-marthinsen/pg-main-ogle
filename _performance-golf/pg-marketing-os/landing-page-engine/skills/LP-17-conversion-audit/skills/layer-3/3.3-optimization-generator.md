# 3.3 — Optimization Generator

> **Skill:** LP-17-conversion-audit
> **Layer:** 3
> **Position:** 3 of 3
> **Purpose:** Generate specific, actionable optimization recommendations for every FAIL item. Each recommendation includes: current state, recommended state, implementation steps, expected lift, and which upstream skill to revise.
> **Effort Level:** sonnet
> **Dependencies:** `priority-ranking.md`, `benchmark-load.md`
> **Output:** `optimization-recommendations.md`

---

## INPUT

```yaml
input:
  priority_ranking: "priority-ranking.md"
  benchmark_load: "benchmark-load.md"
  above_fold_audit: "above-fold-audit.md"
  copy_quality_audit: "copy-quality-audit.md"
  social_proof_audit: "social-proof-audit.md"
  offer_conversion_audit: "offer-conversion-audit.md"
  mobile_speed_audit: "mobile-speed-audit.md"
```

## EXECUTION

### Step 1: Read Priority Ranking
From `priority-ranking.md`, get the sorted list of FAIL items with their priority scores.

### Step 2: Generate Recommendation for Each FAIL Item
For each FAIL item, in priority order, create a complete optimization recommendation:

**Required fields for each recommendation:**
1. **Point ID and Title** — e.g., "Point 06: Eliminate AI Telltale Language"
2. **Priority Rank** — from the ranking
3. **Current State** — what the page has now (from the audit file evidence)
4. **Recommended State** — what it should have after the fix
5. **Implementation Steps** — numbered steps to execute the fix (3-7 steps)
6. **Expected Lift** — from conversion-data-reference.md
7. **Lift Source** — the specific source citation
8. **Skill to Revise** — which LP Engine skill should be re-run to implement this fix
9. **Effort Level** — easy/medium/hard (from priority ranking)

### Step 3: Ensure Specificity
Every recommendation must pass the specificity test:

**SPECIFIC (acceptable):**
- "Replace 'unlock your potential' in hero section with 'reduce joint pain in 7 days' — specific outcome replaces generic benefit"
- "Add 3 testimonials with specific weight loss numbers to Section 3, before the pricing section"
- "Create branded guarantee: 'The 60-Day Pain-Free Promise' — full refund if no improvement, call 1-800-XXX-XXXX"

**VAGUE (not acceptable):**
- "Improve the headline"
- "Add more social proof"
- "Make the guarantee better"

If a recommendation is too vague, rewrite it with specific details from the audit evidence.

### Step 4: Group by Implementation Phase
Organize recommendations into implementation phases:

- **Phase 1: Quick Wins** (priority score >= 6) — can be implemented immediately with minimal effort
- **Phase 2: Medium Effort** (priority score 3-5) — requires some restructuring or content creation
- **Phase 3: Major Effort** (priority score 1-2) — requires significant work, multi-skill revision

### Step 5: Write Optimization Recommendations
Compile into `optimization-recommendations.md`.

## QUALITY CRITERIA

- Every FAIL item has a complete optimization recommendation
- Every recommendation includes all 9 required fields
- Every recommendation passes the specificity test (no vague "improve X" language)
- Recommendations are organized by implementation phase
- Expected lifts cite conversion-data-reference.md
- Skills-to-revise consolidated so the user knows which upstream skills to re-run
- Implementation steps are actionable (a human or agent could follow them)

---

## FAILURE MODES

- Vague recommendations -> Rewrite with specific details. If the audit file evidence is vague, reference the audit file and add specifics from the configured-checklist.md criteria.
- Recommendations that contradict each other -> Check for conflicts (e.g., "add more sections" vs "reduce page load"). Resolve by noting the trade-off and prioritizing the higher-priority fix.
- Missing lift data -> Use "Significant (no specific benchmark)" rather than fabricating a number. This is acceptable but should be the exception.
- Recommendations without implementation steps -> Every recommendation must have 3-7 numbered steps. If unsure of implementation details, provide the conceptual steps and note that the specific skill's AGENT.md should be consulted.
