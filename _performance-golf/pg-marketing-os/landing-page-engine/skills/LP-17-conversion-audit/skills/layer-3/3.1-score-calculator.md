# 3.1 — Score Calculator

> **Skill:** LP-17-conversion-audit
> **Layer:** 3
> **Position:** 1 of 3
> **Purpose:** Sum all 20 individual point scores from the five category audits. Calculate the total score and assign the verdict (LAUNCH_READY, MINOR_OPTIMIZATION, SIGNIFICANT_GAPS, or STRUCTURAL_REBUILD).
> **Effort Level:** sonnet
> **Dependencies:** `above-fold-audit.md`, `copy-quality-audit.md`, `social-proof-audit.md`, `offer-conversion-audit.md`, `mobile-speed-audit.md`
> **Output:** `score-calculation.md`

---

## INPUT

```yaml
input:
  above_fold_audit: "above-fold-audit.md"
  copy_quality_audit: "copy-quality-audit.md"
  social_proof_audit: "social-proof-audit.md"
  offer_conversion_audit: "offer-conversion-audit.md"
  mobile_speed_audit: "mobile-speed-audit.md"
```

---

## EXECUTION

### Step 1: Extract Individual Scores
From each of the 5 audit files, extract:
- Category score (0-4)
- Individual point scores (PASS=1 or FAIL=0 for each of the 4 points in the category)

### Step 2: Build Score Table
Create a complete 20-point score table:

| Point | Description | Score | Status |
|-------|------------|-------|--------|
| 01 | Headline promise | [0/1] | [PASS/FAIL] |
| 02 | Hero image | [0/1] | [PASS/FAIL] |
| ... | ... | ... | ... |
| 20 | Touch targets | [0/1] | [PASS/FAIL] |

### Step 3: Calculate Total Score
- Sum all 20 individual point scores.
- Verify: total = sum of 5 category scores.
- If totals do not match, re-read audit files and resolve the discrepancy before proceeding.

### Step 4: Assign Verdict
Apply the scoring thresholds:

| Score Range | Verdict | Action |
|------------|---------|--------|
| 18-20 | LAUNCH_READY | Proceed to LP-18 (A/B Test Plan) |
| 15-17 | MINOR_OPTIMIZATION | Proceed to LP-18 with priority fixes flagged |
| 12-14 | SIGNIFICANT_GAPS | Return to relevant Phase 2/3 skills. Do NOT proceed to LP-18 |
| 0-11 | STRUCTURAL_REBUILD | Return to LP-03/LP-04. Fundamental architecture problems |

### Step 5: Identify Category Strengths and Weaknesses
- Strongest category: highest score (0-4)
- Weakest category: lowest score (0-4)
- Note any category scoring 0/4 (complete failure in that area)
- Note any category scoring 4/4 (perfect in that area)

### Step 6: Compile FAIL List
Extract every FAIL point with:
- Point ID and description
- Evidence summary
- Fix recommendation
- Expected lift
- Route-to skill

This list feeds directly into the Priority Ranker (3.2).

### Step 7: Write Score Calculation
Compile results into `score-calculation.md`.

## QUALITY CRITERIA

- All 20 points accounted for in score table
- Total score matches sum of category scores (verified)
- Verdict matches the score range thresholds exactly
- FAIL list is complete (every FAIL point included)
- Category analysis identifies strengths and weaknesses
- File exists after execution

---

## FAILURE MODES

- Score mismatch between categories and total -> Re-read all 5 audit files. Resolve discrepancy. Do NOT proceed with mismatched scores.
- Fewer than 20 points in score table -> Layer 2 is incomplete. Return to GATE_2.
- Verdict does not match score range -> Recalculate. The verdict is mechanical, not judgmental. 14/20 = SIGNIFICANT_GAPS regardless of how the auditor feels about the page.
- FAIL list missing entries -> Every FAIL in the score table must appear in the FAIL list with all required fields.
