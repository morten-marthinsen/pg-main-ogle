# 2.1 — Above-Fold Auditor

> **Skill:** LP-17-conversion-audit
> **Layer:** 2
> **Position:** 1 of 5
> **Purpose:** Score Points 01-04 of the 20-point conversion audit checklist. Evaluate headline quality, hero image, CTA visibility, and trust signals in the above-fold section of the assembled page.
> **Effort Level:** opus
> **Dependencies:** `assembled-page-load.md`, `configured-checklist.md`, `benchmark-load.md`
> **Output:** `above-fold-audit.md`

---

## INPUT

```yaml
input:
  assembled_page: "assembled-page-load.md"
  configured_checklist: "configured-checklist.md"
  benchmarks: "benchmark-load.md"
  page_type: "[from page-type-validation.md]"
  awareness_stage: "[from brief-load.md]"
```

## EXECUTION

**POSITIONAL REINFORCEMENT:** You are evaluating the above-fold section ONLY. Score 4 points. Provide evidence for every score. Cite benchmarks for every FAIL. Do not skip points. Do not inflate scores.

### Step 1: Extract Above-Fold Content
From `assembled-page-load.md`, extract everything specified for the above-fold section:
- Headline text (exact)
- Subheadline/deck text (exact)
- Hero image specification
- CTA text and position
- Trust signals (if any)
- Price display (if present)
- Rating strip (if present)

### Step 2: Score Point 01 — Headline Promise
Read the criterion from `configured-checklist.md` Point 01.

**Evaluation process:**
1. Quote the headline verbatim.
2. Identify the specific promise element. Does it name: a specific outcome? A specific audience? A mechanism? A transformation?
3. Check against awareness stage from brief. If Stage 1-2 (cold): headline should use curiosity, problem-agitation, or story-based approach. If Stage 3+: direct benefit is valid.
4. Test for generic language: Does the headline work if you swap the product name? If yes, it is too generic = FAIL.

**PASS example:** "Find More Fairways. Lower Your Scores." (specific outcome for golfers)
**FAIL example:** "Unlock Your Full Potential Today" (generic, works for any product)

### Step 3: Score Point 02 — Hero Image
Read the criterion from `configured-checklist.md` Point 02.

**Evaluation process:**
1. Describe the specified hero image: subject, setting, emotion, format.
2. For Type A: Is it an aspirational outcome or persona image? (NOT a product shot)
3. For Type B: Is it a product hero in context? (NOT generic stock)
4. Would a visitor recognize what the page is about from the image alone?

**PASS example:** "Product bottle photographed on a kitchen counter with morning sunlight, fresh ingredients in background" (product in context)
**FAIL example:** "Generic smiling person on white background" (stock photo, no context)

### Step 4: Score Point 03 — CTA Above Fold
Read the criterion from `configured-checklist.md` Point 03.

**Evaluation process:**
1. Identify the primary CTA text.
2. Confirm it is specified as above-fold on both desktop and mobile.
3. For Type B: verify price AND ATC button are both above fold.
4. Check CTA specificity: "Get My 60-Day Supply" > "Buy Now" > "Submit"

**PASS:** CTA is above fold, text is action-specific.
**FAIL:** CTA is below fold, or CTA is present but generic ("Learn More" on a sales page).

### Step 5: Score Point 04 — Trust Signals
Read the criterion from `configured-checklist.md` Point 04.

**Evaluation process:**
1. List every trust signal found above fold or immediately below fold.
2. Check specificity of each signal. Apply the specificity test:
   - "Satisfaction Guaranteed" = GENERIC = does not count
   - "4.6/5 Trustpilot — 544 Reviews" = SPECIFIC = counts
   - "As Seen on GMA, Fox News, Dr. Oz" = SPECIFIC = counts
   - "GMP Certified" = SPECIFIC = counts
   - "100% Natural" = GENERIC = does not count
3. For Type A: at least 1 specific signal required.
4. For Type B: rating strip + trust badges required.

### Step 6: Write Above-Fold Audit
Compile all four point evaluations into `above-fold-audit.md`.

## QUALITY CRITERIA

- All 4 points explicitly scored PASS or FAIL
- Every score has specific evidence from the assembled page (quoted text, described elements)
- Every FAIL has: specific fix recommendation, expected lift with source, routing to upstream skill
- Type-specific criteria applied correctly (Type A vs Type B vs Hybrid)
- Awareness stage checked for Point 01
- No generic evidence ("headline is good" is not evidence)

---

## FAILURE MODES

- Headline text not extractable from assembled page -> Score as FAIL with note: "Headline text not found in assembled page. LP-15 assembly may be incomplete."
- Above-fold section not clearly delineated -> Use section inventory from assembled-page-load.md to estimate fold position. Note the approximation.
- Inflating Point 01 for a generic headline -> Apply the swap test. If the headline works for any product, it fails regardless of how "good" it sounds.
- Counting generic trust signals as specific -> Re-read the specificity test. "Satisfaction Guaranteed" never counts. Ever.
