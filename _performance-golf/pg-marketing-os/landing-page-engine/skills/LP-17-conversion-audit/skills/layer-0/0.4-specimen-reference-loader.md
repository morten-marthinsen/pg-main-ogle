## EXECUTION

1. Read `cross-page-pattern-analysis.md` from the LandingPageEngine specimens directory.

2. Extract the following audit-relevant patterns:

   **CTA Patterns:**
   - CTA count distribution across 11 specimens (range: 2 to 100+)
   - CTA text consistency vs variation patterns
   - Risk reversal attached to CTA patterns

   **Proof Architecture Patterns:**
   - Proof density distribution (ZERO to EXTREME)
   - First proof position patterns (above fold, section 2-3, section 4-5, none)
   - Proof type frequency (reviews, video testimonials, celebrity, expert, stats, citations)
   - Bimodal proof distribution insight (all-in or authority-substitute)

   **Section Sequence Patterns:**
   - Type A universal skeleton (Hero -> Qualification -> Feature -> Proof -> Value -> FAQ -> Final CTA)
   - Type B universal skeleton (Hero/Product -> Benefits -> Proof -> Purchase -> Footer)
   - Section count ranges by vertical

   **Headline Patterns:**
   - Result/Outcome headlines: 4/11
   - Method/How-To: 1/11
   - Bold Claim: 1/11
   - Community: 1/11

   **Offer Structure Patterns:**
   - Pricing psychology techniques used across specimens
   - Guarantee spectrum (none to 365-day)

3. Write the structured reference to `specimen-reference-load.md`.
