# 4.1 — Audit Compiler

> **Skill:** LP-17-conversion-audit
> **Layer:** 4
> **Position:** 1 of 3
> **Purpose:** Compile all audit data from Layer 2 (five category audits) and Layer 3 (scoring, ranking, recommendations) into the final `conversion-audit.json` output file.
> **Effort Level:** haiku
> **Dependencies:** All Layer 2 audit files, `score-calculation.md`, `priority-ranking.md`, `optimization-recommendations.md`
> **Output:** `conversion-audit.json`

---

## INPUT

```yaml
input:
  above_fold_audit: "above-fold-audit.md"
  copy_quality_audit: "copy-quality-audit.md"
  social_proof_audit: "social-proof-audit.md"
  offer_conversion_audit: "offer-conversion-audit.md"
  mobile_speed_audit: "mobile-speed-audit.md"
  score_calculation: "score-calculation.md"
  priority_ranking: "priority-ranking.md"
  optimization_recommendations: "optimization-recommendations.md"
  page_type_validation: "page-type-validation.md"
  brief_load: "brief-load.md"
```

---

## EXECUTION

### Step 1: Read All Source Files
Read all 8 input files and extract the data needed for the JSON schema.

### Step 2: Assemble JSON Structure
Build the `conversion-audit.json` following the schema defined in LP-17-AGENT.md:

1. **Header fields:** schema_version, created (ISO timestamp), project_name, page_type
2. **Total score and verdict** from `score-calculation.md`
3. **Category scores** from each of the 5 audit files:
   - For each category: score (0-4) and all 4 points with: score, status, evidence, fix, expected_lift
4. **Priority ranking** from `priority-ranking.md`:
   - Each ranked item: rank, point_id, category, expected_lift, lift_source, ease_of_implementation, priority_score, fix_summary
5. **Optimization recommendations** from `optimization-recommendations.md`:
   - Each recommendation: point_id, title, current_state, recommended_state, implementation_steps, expected_lift, lift_source, skill_to_revise
6. **Metadata:** source file paths, benchmark data version, specimen data version

### Step 3: Validate JSON Completeness
Before writing, verify:
- All 20 points are present in category_scores (5 categories x 4 points each)
- Total score matches sum of individual scores
- Priority ranking count matches FAIL count
- Every FAIL has a corresponding optimization recommendation
- No empty required fields

### Step 4: Write conversion-audit.json
Write the complete JSON file to the project output folder.

---

## OUTPUT SCHEMA

`conversion-audit.json` — follows the complete schema defined in LP-17-AGENT.md OUTPUT SCHEMA section.

Key validation rules:
- `total_score` must be an integer 0-20
- `verdict` must be one of: LAUNCH_READY, MINOR_OPTIMIZATION, SIGNIFICANT_GAPS, STRUCTURAL_REBUILD
- Each point `score` must be 0 or 1
- Each point `status` must be PASS or FAIL
- PASS points: `fix` and `expected_lift` should be null
- FAIL points: `evidence`, `fix`, and `expected_lift` must all be non-null strings
- `priority_ranking` array must be sorted by `rank` ascending (rank 1 = highest priority)
- `optimization_recommendations` array must have one entry per FAIL point

---

## QUALITY CRITERIA

- JSON is well-formed and parseable
- All 20 points present with complete data
- Score verification passes (total = sum of parts)
- Every FAIL has evidence + fix + expected_lift (no nulls on FAIL items)
- Priority ranking is sorted and complete
- Optimization recommendations match FAIL count
- Metadata includes all source file paths
- File exists after execution

---

## FAILURE MODES

- JSON malformed -> Re-check structure. Ensure all brackets, quotes, and commas are correct.
- Missing data from source files -> Re-read the source file. If data is genuinely missing, include the field with a value of "DATA_MISSING — [source file] did not contain this field" rather than omitting it.
- Score verification fails -> Do NOT write the file. Return to 3.1 Score Calculator to resolve the discrepancy.
