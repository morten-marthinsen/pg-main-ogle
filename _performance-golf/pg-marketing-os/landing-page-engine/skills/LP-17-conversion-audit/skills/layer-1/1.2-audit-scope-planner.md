# 1.2 — Audit Scope Planner

> **Skill:** LP-17-conversion-audit
> **Layer:** 1
> **Position:** 2 of 3
> **Purpose:** Determine which of the 20 audit points apply to this page, identify type-specific thresholds, and flag any points that need special handling.
> **Effort Level:** sonnet
> **Dependencies:** `page-type-validation.md`
> **Output:** `audit-scope.md`

---

## INPUT

```yaml
input:
  page_type_validation: "page-type-validation.md"
  brief_load: "brief-load.md"
```

---

## EXECUTION

1. Read the operational page type from `page-type-validation.md`.

2. Read the vertical and audience context from `brief-load.md`.

3. For each of the 20 audit points, determine:
   - **Applicability:** Does this point apply to this page type? (All 20 always apply, but some have type-specific criteria)
   - **Type-specific threshold:** What does PASS mean for this page type?
   - **Special handling:** Does the vertical or audience context modify the criteria?

4. Identify type-specific threshold differences:

   **Points that differ by type:**
   | Point | Type A Threshold | Type B Threshold | Hybrid |
   |-------|-----------------|-----------------|--------|
   | 02 (Hero image) | Aspirational/persona image | Product in context | Type B above fold, Type A below |
   | 03 (CTA above fold) | CTA visible without scrolling (may be below deck) | ATC + price MUST be above fold | Type B above fold rules |
   | 04 (Trust signals) | At least 1 specific signal | Rating strip + trust badges required | Type B above fold rules |
   | 10 (Proof volume) | 15+ testimonials/proof elements | 50+ reviews or volume signal | Whichever applies to the zone |
   | 12 (Before/after) | Before/after stories | Before/after images | Category-dependent auto-PASS |
   | 13 (Value before price) | Value stack before pricing block | Benefits/proof before pricing section | Both zones evaluated |
   | 17 (Page load) | <3 seconds target | <2 seconds target | <2 seconds (Type B standard) |
   | 18 (Sticky ATC) | Recommended, auto-PASS if absent | Required, FAIL if absent | Required (Type B standard) |

5. Identify vertical-specific modifications:
   - **Supplements, weight loss, skincare, fitness:** Point 12 (before/after) = REQUIRED. Cannot auto-PASS.
   - **SaaS, info products, finance, coaching:** Point 12 (before/after) = auto-PASS (category does not warrant before/after).
   - **Health/supplements:** Point 10 proof volume expectations are highest.

6. Write the scope plan to `audit-scope.md`.

## QUALITY CRITERIA

- All 20 points have determined applicability and thresholds
- Type-specific differences explicitly noted for all 8 divergent points
- Vertical-specific modifications applied (especially Point 12)
- Auto-PASS points justified with clear rationale
- File exists after execution

---

## FAILURE MODES

- Operational page type unclear -> Use Hybrid thresholds (strictest) as default. Flag for human review.
- Vertical not identified -> Apply universal thresholds. Note that vertical-specific modifications could not be applied.
- Trying to mark points as "N/A" to reduce audit scope -> All 20 points always apply. Some may auto-PASS with justification, but they are never skipped.
