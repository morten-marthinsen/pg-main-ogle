# 2.6 — Voice & Tone Extractor

**Skill:** LP-00-brief-classifier
**Position:** Layer 2, Microskill 6
**Purpose:** Extract voice direction (register, energy) and build the anti-voice list (minimum 5 entries). Set reading level target. The anti-voice MUST include standard AI telltales as a non-negotiable baseline. This output governs tone for every Phase 3 writing skill.
**Effort Level:** medium
**Dependencies:** `brief-load.md` (Layer 0), `audience-profile.md` (2.1 — language profile dimension), `vertical-id.md` (2.3)
**Output:** `voice-direction.md`

---

## Input Schema

```yaml
input:
  brief_content: "[from brief-load.md — full brief text]"
  language_profile: "[from audience-profile.md — Dimension 5]"
  vertical: "[from vertical-id.md — primary vertical]"
  page_type: "[from type-declaration.md]"
  soul_md_path: "[path to Soul.md if it exists for this project, else: null]"
```

---

## Execution Logic

**Step 1: Check for existing Soul.md.**
If a Soul.md file exists for this project (CopywritingEngine's taste constraint file), load it and use it as the primary voice source. Soul.md overrides brief-inferred voice direction.

**Step 2: Extract voice register.**
Determine the voice register from brief instructions, Soul.md, or vertical defaults:
- **Conversational:** Like talking to a smart friend. Contractions, short sentences, direct.
- **Authoritative:** Expert speaking from experience. Credentialed but not academic.
- **Clinical:** Professional, measured, evidence-first. Appropriate for medical/financial.
- **Provocative:** Challenges assumptions, creates tension, breaks patterns.
- **Warm/Empathetic:** Understanding, relatable, "I've been there" energy.

If the brief specifies a register, use it. If not, select based on vertical + audience:
- `health-supplements` → Conversational or Authoritative
- `finance-investing` → Authoritative or Clinical
- `info-products` → Conversational or Provocative
- `coaching-services` → Warm/Empathetic or Authoritative
- `beauty-wellness` → Warm/Empathetic or Conversational
- `ecomm-dtc` → Conversational
- `personal-development` → Warm/Empathetic or Provocative

**Step 3: Extract voice energy.**
Determine the emotional energy:
- **Urgent:** Time-sensitive, stakes are high, act now
- **Hopeful:** Future is bright, solution exists, transformation possible
- **Confident:** Authority, certainty, no hedging
- **Calm:** Measured, reassuring, steady
- **Rebellious:** Against the establishment, contrarian, bold

**Step 4: Build the anti-voice list (MINIMUM 5 entries).**
The anti-voice is what this page must NEVER sound like. Start with the mandatory AI telltales baseline, then add project-specific entries.

**Mandatory AI telltales (always included):**
- "revolutionary"
- "game-changing"
- "transform" / "transformative"
- "unlock" / "unlocking"
- "harness" / "harnessing"
- "empower" / "empowering"
- "journey"
- "comprehensive"

**Then add project-specific anti-voice entries** based on:
- Brief instructions ("don't sound like...")
- Vertical norms to avoid (e.g., health supplements → avoid "miracle cure" language)
- Audience sophistication (high skepticism → avoid hype language)
- Soul.md anti-voice list (if available)

**Step 5: Set reading level target.**
Based on audience language profile (Dimension 5) and vertical norms:
- Most DR copy: 6th-8th grade reading level
- Finance/investing: 8th-10th grade
- Coaching/high-ticket: 7th-9th grade
- Ecomm/DTC: 5th-7th grade

**Step 6: Check for Soul.md availability.**
Record whether Soul.md exists and its path. Downstream writing skills will load Soul.md directly if available.

---

## Quality Criteria
- Register selected from the defined options — not a custom label
- Energy selected from the defined options — not a custom label
- Anti-voice list has minimum 5 project-specific entries BEYOND the mandatory AI telltales
- AI telltales are always included regardless of project type
- Reading level target set with rationale
- Soul.md status checked and recorded
- Voice summary is specific enough that a writing skill could use it directly

## Failure Modes
- Brief provides no voice direction → Infer from vertical + audience, flag as `vertical_default`
- Soul.md exists but conflicts with brief → Soul.md takes precedence, note the conflict
- Anti-voice list has fewer than 5 project-specific entries → Expand using vertical norms and audience sophistication patterns until minimum met
- Reading level target unrealistic for audience → Cross-check against audience demographics, adjust
