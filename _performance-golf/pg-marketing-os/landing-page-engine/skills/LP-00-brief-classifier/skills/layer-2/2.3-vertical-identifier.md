# 2.3 — Vertical Identifier

**Skill:** LP-00-brief-classifier
**Position:** Layer 2, Microskill 3
**Purpose:** Identify the project vertical from the 8-option list defined in the AGENT.md. Vertical governs specimen loading, competitive benchmarks, and section sequence defaults in all downstream skills. Assign a primary vertical and an optional secondary.
**Effort Level:** low
**Dependencies:** `brief-load.md` (Layer 0), `offer-summary.md` (2.2)
**Output:** `vertical-id.md`

---

## Input Schema

```yaml
input:
  brief_content: "[from brief-load.md — full brief text]"
  product_category: "[from offer-summary.md — product category field]"
  product_name: "[product name]"
```

---

## Execution Logic

**Step 1:** Review the product category, product name, and brief content against the 8-option vertical list:

| Vertical ID | Vertical | Key Indicators |
|-------------|---------|----------------|
| `health-supplements` | Health & Dietary Supplements | Vitamins, minerals, herbal, nootropics, weight loss, sleep, gut health |
| `health-devices` | Health Devices & Equipment | Wearables, fitness equipment, medical devices, TENS, red light |
| `beauty-wellness` | Beauty & Personal Care | Skincare, haircare, cosmetics, anti-aging topicals, personal wellness |
| `info-products` | Info Products & Courses | Digital courses, ebooks, masterclasses, challenges, membership sites |
| `coaching-services` | Coaching & High-Ticket Services | Coaching programs, done-for-you, consulting, high-ticket offers |
| `finance-investing` | Finance & Investing | Trading, investing, financial tools, newsletters, crypto |
| `personal-development` | Personal Development | Self-help, mindset, productivity, relationships, spirituality |
| `ecomm-dtc` | Ecommerce / DTC Consumer Goods | Non-health physical products, food/beverage, gear, home goods |

**Step 2:** Assign the primary vertical based on the strongest match. Use the product's core category, not its marketing angle. A supplement marketed as "personal development" is still `health-supplements`.

**Step 3:** Check for a secondary vertical. Assign only if the product genuinely straddles two categories. Examples:
- A coaching program that includes a course → Primary: `coaching-services`, Secondary: `info-products`
- A supplement brand selling via Shopify DTC → Primary: `health-supplements`, Secondary: `ecomm-dtc`

If no genuine secondary exists, set to `null`.

**Step 4:** Note any downstream implications of the vertical selection:
- Which specimen files are most relevant
- Which competitive benchmarks apply
- Any vertical-specific page patterns (e.g., health-supplements almost always requires ingredient sections)

---

## Output Schema

`vertical-id.md`:

```markdown
# Vertical Identification — [Product Name]

## Primary Vertical
- ID: [vertical_id]
- Name: [full vertical name]
- Evidence: [1-2 sentences citing specific brief indicators]

## Secondary Vertical
- ID: [vertical_id | null]
- Name: [full vertical name | N/A]
- Evidence: [1-2 sentences or "No secondary vertical — product fits cleanly in primary"]

## Downstream Implications
- Specimen relevance: [which specimen types are most applicable]
- Competitive benchmarks: [what category norms to reference]
- Vertical-specific patterns: [any must-have sections or elements for this vertical]

## Classification Confidence: [high | medium | low]
[If low: note what additional information would resolve ambiguity]
```

---

## Quality Criteria
- Primary vertical selected from the exact 8-option list — no custom verticals
- Evidence cites specific brief content, not general assumptions
- Secondary vertical assigned only when genuinely applicable (not forced)
- Downstream implications are actionable, not generic
- Classification confidence reflects actual certainty

## Failure Modes
- Product doesn't fit any of the 8 verticals cleanly → Select closest match, flag as `low` confidence, note what's unclear
- Brief provides insufficient information to determine vertical → Infer from product name and category, flag `needs_validation`
- Product spans 3+ verticals → Select top 2 only (primary + secondary), note the third in downstream implications
