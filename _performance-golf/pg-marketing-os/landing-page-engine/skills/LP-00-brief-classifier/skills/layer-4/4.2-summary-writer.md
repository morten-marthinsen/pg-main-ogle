# 4.2 — Summary Writer

**Skill:** LP-00-brief-classifier
**Position:** Layer 4, Microskill 2
**Purpose:** Write the human-readable `PAGE-BRIEF-SUMMARY.md` from the assembled `page-brief.json`. This is the document a human reviews to quickly understand the classification, audience, offer, and strategic direction before downstream skills execute. Must follow the exact template defined in ANTI-DEGRADATION.md.
**Effort Level:** low
**Dependencies:** `page-brief.json` (4.1 output)
**Output:** `PAGE-BRIEF-SUMMARY.md`

---

## Input Schema

```yaml
input:
  page_brief_json: "[from 4.1 — the assembled page-brief.json]"
  project_name: "[product/client name]"
```

---

## Execution Logic

**Step 1: Load `page-brief.json`.**
Parse the assembled JSON. Confirm it loaded successfully and contains all required sections.

**Step 2: Write the Classification section.**
- Page Type (Type A / Type B / Hybrid)
- Signals confirmed (X/7)
- Rationale (2-3 sentences from the JSON)
- Operating Mode (Downstream / Standalone)

**Step 3: Write the Audience Profile section.**
Summarize all 5 dimensions in bullet format. Focus on the most actionable insights from each dimension:
- Demographics: age, gender, income (1-2 bullets)
- Pain: primary pain + intensity (1-2 bullets)
- Desire: primary desire + dream outcome (1-2 bullets)
- Sophistication: awareness stage + skepticism level (1-2 bullets)
- Language: reading level + emotional tone (1-2 bullets)

Include 2-3 of the strongest audience quotes if available (from research-validated data).

**Step 4: Write the Offer Summary section.**
Bullet format covering:
- Product name and category
- Price point(s)
- Bonus count and highlights
- Guarantee details
- Urgency levers in use

**Step 5: Write the Traffic Profile section.**
- Temperature (Cold / Warm / Hot)
- Primary traffic source
- Awareness Stage (1-5 with name)

**Step 6: Write the Vertical section.**
- Primary vertical
- Secondary vertical (if applicable)

**Step 7: Write the Copy Assets Available section.**
In Downstream Mode: list what was extracted from CopywritingEngine packages (headline territory, mechanism name, root cause, promise, assembled draft status).
In Standalone Mode: "Standalone mode — no CopywritingEngine packages available."

**Step 8: Write the Validation Flags section.**
List any fields flagged as `needs_validation` with the reason. If no flags: "None — all fields confirmed."

**Step 9: Write the Downstream Handoff section.**
List which downstream LP skills should load this brief and what they'll use from it.

---

## Quality Criteria
- Follows the exact template from ANTI-DEGRADATION.md — all 8 sections present
- Every section populated — no empty sections
- Audience profile summary captures the most actionable points from each dimension
- Offer summary is accurate to the JSON (prices match, bonus count matches)
- Human-readable — a non-technical reader can understand the page strategy in 2 minutes
- Validation flags are honest — any `needs_validation` items from the JSON are surfaced

## Failure Modes
- `page-brief.json` didn't load → HALT — 4.1 must complete first
- JSON section is empty for a required summary field → Write "not_available" in the summary, flag it
- Summary is too long (>500 words) → Tighten bullets, remove redundancy — this is a summary, not a report
