# 4.1 — Brief Compiler

**Skill:** LP-00-brief-classifier
**Position:** Layer 4, Microskill 1
**Purpose:** Assemble the complete `page-brief.json` from all upstream microskill outputs. Must follow the exact JSON schema defined in the AGENT.md. Zero empty required fields — every field is either populated with real data or marked `not_available`. This is the single most important output of LP-00 — every downstream skill consumes this file.
**Effort Level:** low
**Dependencies:** All upstream outputs — `mode-declaration.md`, `brief-load.md`, `package-inventory.md` (if Downstream), `classification-signals.md`, `type-declaration.md`, `audience-profile.md`, `offer-summary.md`, `vertical-id.md`, `traffic-temp.md`, `awareness-map.md`, `voice-direction.md`, `research-extract.md` (if Downstream), `copy-extract.md` (if Downstream), `offer-extract.md` (if Downstream)
**Output:** `page-brief.json`

---

## Input Schema

```yaml
input:
  operating_mode: "[downstream | standalone]"
  upstream_outputs:
    layer_0:
      - mode-declaration.md
      - brief-load.md
      - package-inventory.md  # Downstream only
    layer_1:
      - classification-signals.md

    layer_2:

      - voice-direction.md
    layer_3:  # Downstream only
      - research-extract.md
      - copy-extract.md
      - offer-extract.md
```

## Execution Logic

**Step 1: Pre-assembly validation.**
Before assembling, verify all required upstream files exist:
- Layer 0: `mode-declaration.md` + `brief-load.md` (mandatory)
- Layer 1: `classification-signals.md` + `type-declaration.md` (mandatory)
- Layer 2: All 6 files (mandatory)
- Layer 3: All 3 files (mandatory in Downstream Mode, absent in Standalone)

If any mandatory file is missing → HALT. Do not assemble a partial brief.

**Step 2: Map each upstream output to the JSON schema.**
Follow the exact schema from AGENT.md. Field mapping:

```
schema_version         → "1.0"
created                → current ISO timestamp
project_name           → from brief-load.md

operating_mode         → from mode-declaration.md
copywriting_engine_packages → from package-inventory.md (or empty in Standalone)

page_type              → from type-declaration.md + classification-signals.md
  classification       → from type-declaration.md
  signals_confirmed    → from classification-signals.md tally
  signals_detail       → from classification-signals.md per-signal results
  rationale            → from type-declaration.md
  hybrid_zone_split    → from hybrid-test.md (if Hybrid)

audience               → from audience-profile.md (all 5 dimensions)
offer_summary          → from offer-extract.md (Downstream) OR offer-summary.md (Standalone)
vertical               → from vertical-id.md
traffic                → from traffic-temp.md + awareness-map.md

voice_direction        → from voice-direction.md
copy_assets            → from copy-extract.md (Downstream) OR all "not_available" (Standalone)
proof_inventory        → from research-extract.md or "not_available"
validation_flags       → collected from all upstream files
downstream_handoffs    → static text per schema
```

**Step 3: Empty field sweep.**
Scan the assembled JSON for any empty strings, null values, or missing keys. Every required field must have a value. Use `not_available` for genuinely missing data — never leave a field empty.

**Step 4: Validation flag collection.**
Aggregate all `needs_validation` flags from upstream outputs into the `validation_flags` array with:
- Field name
- Source microskill
- Reason for flag

**Step 5: Write `page-brief.json`.**
Write the complete JSON to the project output directory. Verify file exists and is valid JSON after writing.

## Output Schema

`page-brief.json` — follows the exact schema defined in AGENT.md (Section: PAGE-BRIEF.JSON SCHEMA). The complete schema includes:

```
- schema_version, created, project_name
- operating_mode, copywriting_engine_packages
- page_type (classification, signals_confirmed, signals_detail, rationale, hybrid_zone_split)
- audience (demographics, pain_profile, desire_profile, sophistication, language_profile)
- offer_summary (full offer schema)
- vertical (primary, secondary)
- traffic (temperature, primary_source, awareness_stage)
- voice_direction (register, energy, anti_voice, reading_level_target, soul_md)
- copy_assets (headline_territory, mechanism_name, root_cause, promise, big_idea, assembled_draft)
- proof_inventory
- validation_flags
- downstream_handoffs
```

Refer to AGENT.md for the complete field-by-field schema.

---

## Quality Criteria
- JSON is valid and parseable
- Zero empty required fields — `not_available` used correctly for missing data
- All upstream files loaded and mapped to correct JSON sections
- Validation flags accurately collected from all upstream outputs
- File written to correct output path
- Schema version is "1.0"
- In Downstream Mode: copy_assets populated from Layer 3 extractions
- In Standalone Mode: copy_assets all marked `not_available`

## Failure Modes
- Missing upstream file → HALT — do not assemble a partial brief. Identify which file is missing and which layer/microskill failed.
- Conflicting data between upstream files → Use the later-layer value (Layer 3 enrichment overwrites Layer 2 baseline). Note conflict in validation_flags.
- JSON writing fails → Retry. If persistent, write as formatted markdown with `.json` extension and flag the format issue.
- Output path doesn't exist → Create the directory structure, then write the file.
