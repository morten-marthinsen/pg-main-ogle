# 3.2 — Copy Package Distiller

**Skill:** LP-00-brief-classifier
**Position:** Layer 3, Microskill 2
**Purpose:** Extract the headline territory, mechanism name, root cause phrase, and promise statement from CopywritingEngine copy packages (Skills 03-06, 09). Each field is either populated with validated data or marked `not_available`. This microskill ONLY executes in Downstream Mode.
**Effort Level:** medium
**Dependencies:** `package-inventory.md` (0.3 — confirms which copy packages exist), CopywritingEngine output files
**Output:** `copy-extract.md`

---

## Input Schema

```yaml
input:
  operating_mode: "downstream"  # This microskill ONLY runs in Downstream Mode
  copy_package_paths:
    big_idea: "[path to 03-big-idea/outputs/ or: not_found]"
    headline: "[path to 04-headline/outputs/ or: not_found]"
    promise: "[path to 05-promise/outputs/ or: not_found]"
    mechanism: "[path to 06-root-cause-mechanism/outputs/ or: not_found]"
    campaign_brief: "[path to 09-campaign-brief/outputs/ or: not_found]"
    assembled_draft: "[path to 19-campaign-assembly/outputs/ or: not_found]"
```

---

## Execution Logic

**Step 1: Confirm Downstream Mode.**
If operating mode is `standalone`, skip this microskill entirely. Log: "3.2 skipped — Standalone Mode. No copy packages to distill."

**Step 2: Load available copy packages.**
For each package that exists (per package-inventory.md), load the primary output file. If a package is `not_found`, skip it and mark the corresponding field as `not_available`.

**Step 3: Extract headline territory.**
Source priority:
1. `04-headline/outputs/` — headline package (primary)
2. `09-campaign-brief/outputs/campaign-brief.json` — headline section
3. `19-campaign-assembly/outputs/assembled-draft.md` — extract from assembled headline

Extract:
- Winning headline (the Arena-selected or final headline)
- Headline type (curiosity, benefit-direct, problem-agitation, mechanism-reveal, news/announcement)
- Headline territory description (the conceptual space the headline operates in)

**Step 4: Extract mechanism name.**
Source priority:
1. `06-root-cause-mechanism/outputs/` — mechanism package (primary)
2. `09-campaign-brief/outputs/campaign-brief.json` — mechanism section
3. `03-big-idea/outputs/` — mechanism reference within big idea

Extract:
- Mechanism name (exact branded/named mechanism)
- Mechanism type (discovery, process, ingredient, system, protocol)
- One-sentence mechanism explanation

**Step 5: Extract root cause phrase.**
Source priority:
1. `06-root-cause-mechanism/outputs/` — root cause section
2. `09-campaign-brief/outputs/campaign-brief.json` — root cause section

Extract:
- Root cause anchor phrase (the named root cause the audience can blame)
- Root cause category (biological, environmental, systemic, behavioral)

**Step 6: Extract promise statement.**
Source priority:
1. `05-promise/outputs/` — promise package (primary)
2. `09-campaign-brief/outputs/campaign-brief.json` — promise section
3. `03-big-idea/outputs/` — big idea promise thread

Extract:
- Primary promise statement (single sentence)
- Promise timeline (how fast results are implied)
- Promise specificity (vague, moderate, highly specific)

**Step 7: Extract big idea statement (if available).**
Source: `03-big-idea/outputs/`
- Big idea one-liner
- Big idea type (secret, discovery, prediction, contrarian, system)

**Step 8: Note assembled draft availability.**
If `19-campaign-assembly/outputs/assembled-draft.md` exists, record its path. The full copy may already be drafted — LP-00 outputs a brief to guide page STRUCTURE and IMPLEMENTATION, not to replace existing copy.

## Quality Criteria
- Every field populated or explicitly marked `not_available` — zero empty fields
- Extracted content matches source files exactly (no paraphrasing mechanism names or headlines)
- Source priority order followed — primary sources checked before fallbacks
- GATE_3 minimum check performed (root cause + mechanism name required for pass)
- Assembled draft availability noted for downstream skill awareness

## Failure Modes
- Key packages (mechanism, headline) not found → Mark fields `not_available`, log which downstream decisions will be impacted
- Package files exist but are empty or malformed → Log parse error, mark fields `not_available`
- Multiple conflicting headlines across packages → Extract the Arena-selected headline (from Skill 04), note alternatives
- Campaign brief exists but other individual packages don't → Extract from campaign brief as fallback
