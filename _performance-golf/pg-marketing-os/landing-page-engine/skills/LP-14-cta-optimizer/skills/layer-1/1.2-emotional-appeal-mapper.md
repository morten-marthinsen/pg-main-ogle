# 1.2 — Emotional Appeal Mapper

> Version: 1.0
> Layer: 1
> Model: sonnet
> Input: cta-type-plan.md, specimens-loaded.md
> Output: emotional-appeal-map.md

**Skill:** LP-14-cta-optimizer
**Position:** Layer 1, Microskill 2
**Purpose:** Map the 3 lenses (Converter, Empath, Skeptic) to the 6 emotional appeal types. Determine which lens generates the strongest candidates for each appeal type. This ensures that during generation (Layer 2), each lens is directed toward the appeal types where it has the highest natural advantage.
**Effort Level:** medium
**Dependencies:** `cta-type-plan.md` (1.1), `specimens-loaded.md` (0.3)
**Output:** `emotional-appeal-map.md`

## Execution Logic

1. **Read `cta-type-plan.md`** to get the position-to-appeal mapping.

2. **Map lenses to appeal types using natural affinity:**

   | Appeal Type | Primary Lens | Secondary Lens | Rationale |
   |------------|-------------|----------------|-----------|
   | 1. Confidence/Desire | Converter | Empath | DR formulas excel at confident action verbs. Empath adds "my" possessive warmth. |
   | 2. Consequence Awareness | Empath | Skeptic | Empath mirrors the reader's fear. Skeptic makes the consequence specific and credible. |
   | 3. Urgency | Converter | Skeptic | DR formulas have proven urgency patterns. Skeptic adds justified specificity (numbers, dates). |
   | 4. Social Proof | Skeptic | Empath | Skeptic leads with proof numbers. Empath adds belonging language. |
   | 5. Value | Converter | Skeptic | DR formulas excel at value stacking. Skeptic makes the value comparison specific. |
   | 6. Specificity | Skeptic | Converter | Skeptic leads with concrete numbers/timeframes. Converter structures the claim for action. |

3. **Cross-reference with specimen patterns.** If specimens show that this vertical's best CTAs use a different lens for a given appeal (e.g., health supplements use Empath language for Confidence CTAs — "Yes, I Want My Energy Back"), adjust the mapping.

4. **Build the generation instruction for each lens:**
   - **Converter:** Generate candidates for Appeal Types 1, 3, 5 (Confidence, Urgency, Value). These are your strengths. Also generate 1 candidate each for Appeal Types 2, 4, 6 as secondary attempts.
   - **Empath:** Generate candidates for Appeal Types 1, 2, 4 (Confidence possessive variant, Consequence Awareness, Social Proof belonging variant). Also generate 1 candidate each for Appeal Types 3, 5, 6 as secondary attempts.
   - **Skeptic:** Generate candidates for Appeal Types 4, 5, 6 (Social Proof numbers, Value specifics, Specificity outcomes). Also generate 1 candidate each for Appeal Types 1, 2, 3 as secondary attempts.

5. **Write `emotional-appeal-map.md`** with the lens-to-appeal mapping and generation instructions.

## Quality Criteria
- All 6 appeal types mapped to primary and secondary lenses
- Appeal Type 3 (Urgency) blocked if LP-13 data is FORBIDDEN
- Specimen patterns consulted — not just default affinities
- Generation instructions are actionable — each lens knows exactly which appeal types to prioritize
- Coverage guarantee confirms >= 9 candidates will be generated

## Failure Modes
- All appeal types mapped to the same lens -> Candidate homogeneity. Each appeal type must have a primary lens.
- Urgency mapped despite being FORBIDDEN -> Check `cta-type-plan.md` urgency status before mapping.
- Specimen patterns ignored -> Default affinities are reasonable but not optimal. Specimens from the same vertical may reveal different patterns.
- Generation instructions too vague -> "Generate good CTAs" is not actionable. Each lens needs specific appeal types, voice notes, and specimen references.
