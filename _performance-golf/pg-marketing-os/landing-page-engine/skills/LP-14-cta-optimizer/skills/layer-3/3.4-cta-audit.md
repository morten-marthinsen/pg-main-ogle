## Quality Criteria
- All 10 points scored with evidence (no scores without supporting quotes/data)
- Pre-audit gate verified (anti-slop scan must PASS before audit runs)
- Type B adjustments clearly documented (N/A items scored 1 with explanation)
- Failed points have specific revision recommendations
- Total score calculated correctly

## Failure Modes
- Scoring points without evidence -> Every 0 or 1 must be supported by a specific quote, count, or data point. Unsupported scores are unreliable.
- Running audit with anti-slop FAIL -> Audit cannot pass with slop present. Fix slop first.
- Scoring P.S./close points as 0 for Type B pages that intentionally exclude them -> Type B exclusions score as 1 (not applicable).
- Total score below 7.5 but marked as PASS -> 7.5 is the minimum. 7.0 is not 7.5. Be precise.
- Revision recommendations too vague ("improve the CTA") -> Must be specific: "Point 2 failed — button text 'Get Started' has no benefit. Revise to 'Get [Specific Benefit] Now.'"
