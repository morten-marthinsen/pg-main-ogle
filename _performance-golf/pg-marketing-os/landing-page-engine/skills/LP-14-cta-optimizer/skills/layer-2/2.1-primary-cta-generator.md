# 2.1 — Primary CTA Generator (3-Lens System)

> Version: 1.0
> Layer: 2
> Model: opus
> Input: emotional-appeal-map.md, cta-type-plan.md, specimens-loaded.md, packages-loaded.md
> Output: primary-cta-candidates.md

**Skill:** LP-14-cta-optimizer
**Position:** Layer 2, Microskill 1
**Purpose:** Generate >= 9 primary CTA candidates using the 3-lens system. Each candidate includes button text (3-8 words), optional context text (1-2 sentences), and an emotional appeal tag. Score all candidates against the 6-criteria CTA rubric. This is the highest-leverage creative output in LP-14 — the button text that determines whether a convinced reader actually converts.
**Effort Level:** max
**Dependencies:** `emotional-appeal-map.md` (1.2), `cta-type-plan.md` (1.1), `specimens-loaded.md` (0.3), `packages-loaded.md` (0.1)

## Pre-Generation Protocol (Execute Before Writing Any CTA)

**Step 1: Load specimens into active context**
Hold all loaded CTA specimens verbatim. Before writing CTA 1, explicitly read each specimen and extract: What makes this button text work? How specific is it? Does it use "my" or "your"? What emotional trigger does it activate?

**Step 2: Read threading anchors**
Read LP-07's threading anchors: `primary_hook_phrase` and `promise_statement`. At least 2 of the 9+ candidates MUST incorporate these phrases or direct echoes of them.

**Step 3: Read guarantee terms**
Read the guarantee from LP-06. At least 1 candidate must embed the guarantee in its context text (e.g., "365-day money-back guarantee" below the button).

**Step 4: Review anti-slop word list**
Before writing: scan the forbidden words list from LP-14-AGENT.md. Commit to producing zero instances. Especially: no "Click Here," no "Learn More" as primary CTA, no "Act Now" without LP-13 permission.

**Step 5: State the single offer**
Before generating: write "The single offer is: [product name] at [price] with [guarantee]. The button press initiates: [checkout / trial / demo / enrollment]." This anchors all CTAs to the actual conversion event.

## Generation Execution

### LENS 1: The Converter (Direct Response Formula)

**Perspective:** Pure DR optimization. Proven CTA formulas applied with maximum specificity. Action verb + benefit + urgency (where justified).

**Generate 3+ candidates, prioritizing these appeal types:**

```
L1-A: Confidence/Desire (#1)
  Pattern: "Get [Specific Benefit] Now" / "Start [Desired Outcome] Today"
  Must contain: action verb + specific benefit reference
  Example direction: "Get My 7-Form Magnesium Complex" (not "Buy Now")

L1-B: Value (#5)
  Pattern: "Get [Product] + [Bonus] — Just [Price]" / "Add [Bundle] to Cart"
  Must contain: value stack hint in button or context
  Example direction: "Add the Complete Stack — Save 25%"

L1-C: Urgency (#3) — ONLY if LP-13 permits
  Pattern: "Claim Your [X] Before [Specific Deadline]"
  Must contain: specific deadline/quantity from LP-13 data
  If LP-13 FORBIDDEN: Generate Confidence variant instead
  Example direction: "Claim Your Bottle — Only 847 Left"
```

**After generating:** Score each against 6 criteria. Converter usually scores highest on Action Clarity and Specificity.

### LENS 2: The Empath (Voice of Customer)

**Perspective:** The button says what the reader is THINKING. Uses the audience's own language for the desired outcome. First-person possessive ("my") preferred.

**Generate 3+ candidates, prioritizing these appeal types:**

```
L2-A: Confidence/Desire (#1) — Possessive variant
  Pattern: "Yes, I Want [Customer's Language for Result]" / "Give Me [What They Actually Want]"
  Must contain: possessive "my" or first-person "I" + audience vocabulary
  Example direction: "Yes, Give Me Deep Sleep Tonight"

L2-B: Consequence Awareness (#2)
  Pattern: "Stop [Their Word for Problem]" / "End [Pain They Describe]"
  Must contain: audience's own language for the problem
  Example direction: "I'm Done Waking Up Exhausted"

L2-C: Social Proof (#4) — Belonging variant
  Pattern: "Join [Community/Tribe/Group]" / "I Want In"
  Must contain: belonging/community language
  Example direction: "Join 83,502 Better Sleepers"
```

**After generating:** Score each. Empath usually scores highest on Emotional Pull and Threading (audience language echoes the promise).

### LENS 3: The Skeptic (Proof-Anchored)

**Perspective:** The CTA addresses hesitation directly. Button text or context reduces risk, provides evidence, or makes a specific credibility claim.

**Generate 3+ candidates, prioritizing these appeal types:**

```
L3-A: Social Proof (#4) — Numbers variant
  Pattern: "Join [Exact Number]+ [Customers/Users]" / "See Why [Number] Chose This"
  Must contain: specific social proof number
  Example direction: "Join 83,502+ Customers — Risk Free"

L3-B: Specificity (#6)
  Pattern: "Start Seeing [Specific Result] in [Timeframe]" / "Get [Measurable Outcome]"
  Must contain: concrete number, timeframe, or measurable claim
  Example direction: "Start Sleeping Better in 7 Days"

L3-C: Risk Removal (Confidence + Guarantee)
  Pattern: "Try [X] Risk-Free for [Guarantee Duration]" / "Protected by [Guarantee]"
  Must contain: explicit risk-removal language from guarantee terms
  Example direction: "Try Risk-Free — 365-Day Guarantee"
```

**After generating:** Score each. Skeptic usually scores highest on Risk Reduction and Freshness.

---

## Scoring Rubric

Score every candidate on all 6 criteria. Calculate weighted score.

| Criterion | Weight | 1 (Poor) | 5 (Average) | 10 (Excellent) |
|-----------|--------|----------|-------------|----------------|
| **Specificity** | 25% | Generic ("Buy Now") | Partial benefit | Specific benefit/result in button text |
| **Action Clarity** | 20% | Unclear next step | Somewhat clear | Reader knows exactly what clicking does |
| **Emotional Pull** | 20% | No emotional trigger | Mild motivation | Strong emotional reason to click NOW |
| **Threading** | 15% | No page connection | Vague reference | Directly echoes LP-07 promise |
| **Risk Reduction** | 10% | No risk addressed | Slight reassurance | Guarantee/proof embedded in context |
| **Freshness** | 10% | Identical to competitors | Somewhat unique | Original for this market |

**Weighted score formula:**
`score = (S x 0.25) + (A x 0.20) + (E x 0.20) + (T x 0.15) + (R x 0.10) + (F x 0.10)`

**Disqualification rules:**
- Specificity < 6.0 -> DISQUALIFIED (generic)
- Button text > 8 words -> DISQUALIFIED (too long)
- Any forbidden word from anti-slop list -> DISQUALIFIED (revise and re-submit)

**Minimum to qualify for human presentation:**
- Weighted score >= 7.0
- Specificity >= 6.0
- Button text 3-8 words
- Zero forbidden words

---

## Refinement Round

After initial 9+ candidates are scored:

1. Identify the highest-scoring candidate from each lens (3 winners)
2. Each lens refines its winner by absorbing the best TECHNIQUE (not voice) from the other lenses:
   - Lens 1 winner: absorb Empath's audience language OR Skeptic's proof anchor
   - Lens 2 winner: absorb Converter's action clarity OR Skeptic's risk-reduction language
   - Lens 3 winner: absorb Converter's specificity technique OR Empath's emotional language
3. Generate 1 refined candidate per lens
4. Score refined candidates
5. Optional: Generate 1-2 hybrid candidates by combining best elements from multiple lenses

---

## Top Candidate Selection

From all scored candidates (9 initial + 3 refined + 1-2 hybrids = 13-14 total):
- Rank all qualifying candidates by weighted score
- Select top 6-10 qualifying candidates
- Ensure diversity: at least 4 different emotional appeal types represented in top candidates
- If top candidates are all from one lens -> force inclusion of highest-scorers from other lenses

## Quality Criteria
- >= 9 candidates generated in Round 1 (3 per lens minimum)
- All candidates scored on all 6 criteria
- All forbidden words checked and absent
- Top candidates include >= 4 different emotional appeal types
- At least 2 candidates reference LP-07 threading anchors
- At least 1 candidate includes guarantee in context text
- All button text 3-8 words (no exceptions)
- Each top candidate has a "why this works" explanation

## Failure Modes
- Fewer than 6 candidates qualify -> Generate additional candidates with more varied approaches
- All top candidates use the same emotional appeal -> Force diversity before presenting
- No candidates reference threading anchors -> At least 2 must echo LP-07's hook phrase or promise
- Button text exceeds 8 words -> Shorten before scoring. Long CTAs reduce click rates.
- "Buy Now" / "Order Now" submitted without benefit specificity -> These must include context text that adds specificity, or the button text itself must be more specific
