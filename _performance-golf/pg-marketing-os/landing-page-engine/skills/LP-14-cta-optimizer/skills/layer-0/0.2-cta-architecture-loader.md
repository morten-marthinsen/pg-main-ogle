**Purpose:** Transform the raw CTA architecture data from LP-06 into an actionable generation plan. Map each CTA position to its function, emotional appeal priority, and generation constraints. This file becomes the structural blueprint that Layer 1 planning and Layer 2 generation execute against.

## Execution Logic

1. **Read `packages-loaded.md`** — extract all CTA architecture fields from the LP-06 section and page context from LP-00 section.

2. **Build CTA Position Map:** For each CTA position defined in LP-06:
   - Assign a position number (CTA-1 through CTA-N)
   - Define position function (first impression, reinforcement, proof-adjacent, close)
   - Assign primary emotional appeal type (from LP-06's recommendations)
   - Assign secondary emotional appeal type (if applicable)
   - Set generation constraints:
     - Above-fold CTA: Must be simplest, lowest-friction. Cold traffic = softer. Hot traffic = direct.
     - Mid-page CTA: Appears after problem/mechanism education. Can be more specific.
     - Post-proof CTA: Follows social proof or clinical data. Skeptic lens strongest here.
     - Post-testimonial CTA: Social proof appeal most natural here.
     - Final/Terminal CTA: Highest urgency permitted (if LP-13 data exists). Full emotional pull.

3. **Build Guarantee Integration Map:** From the guarantee terms:
   - Extract guarantee type (money-back, free trial, conditional, unconditional)
   - Extract guarantee duration (30-day, 90-day, 365-day, lifetime)
   - Extract guarantee-specific language that can be embedded in CTA context copy
   - Determine which CTA positions should include guarantee references

4. **Build Urgency Permission Matrix:** Based on whether LP-13 data is available:
   - If LP-13 loaded + justified = Y: Urgency language PERMITTED in CTAs. List the specific urgency copy that can be referenced.
   - If LP-13 loaded + justified = N: Urgency language FORBIDDEN. LP-13 found the urgency was manufactured.
   - If LP-13 not available: Urgency language FORBIDDEN. No data = no urgency.

5. **Build Awareness-Stage CTA Calibration:**
   - Stage 1 (Unaware): Softest CTAs — "See How It Works" / "Watch the Video"
   - Stage 2 (Problem-Aware): Problem-forward CTAs — "Find Your Solution" / "See What's Causing [Problem]"
   - Stage 3 (Solution-Aware): Benefit CTAs — "Get [Specific Result]" / "Start [Benefit] Today"
   - Stage 4 (Product-Aware): Direct CTAs — "Add to Cart" / "Start My Subscription"
   - Stage 5 (Most Aware): Urgency + offer CTAs — "Claim Your Discount" / "Get Started — [Price]"

6. **Write `cta-architecture-loaded.md`** with the complete generation blueprint.

## Quality Criteria
- Every CTA position from LP-06 mapped with function, appeal, and constraints
- Guarantee terms translated into usable CTA context language
- Urgency permission clearly set — PERMITTED or FORBIDDEN with explicit reason
- Awareness-stage calibration specific to this project (not generic defaults)
- Generation blueprint summary counts all expected outputs

## Failure Modes
- CTA positions from LP-06 not fully mapped -> Layer 2 will miss page positions. Every position must appear.
- Urgency marked as PERMITTED without LP-13 data -> This produces fabricated urgency. Check source.
- Guarantee language not extracted -> Post-proof and final CTAs lose their strongest risk-reduction tool. Extract verbatim.
- Awareness stage ignored -> Cold traffic gets "Buy Now" (too aggressive) or hot traffic gets "Learn More" (too soft). Calibration is required.
