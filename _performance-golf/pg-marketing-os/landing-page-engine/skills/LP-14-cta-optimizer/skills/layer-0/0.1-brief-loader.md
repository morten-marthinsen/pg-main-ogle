# 0.1 — Brief Loader

> Version: 1.0
> Layer: 0
> Model: haiku
> Input: project directory path
> Output: packages-loaded.md

**Skill:** LP-14-cta-optimizer
**Position:** Layer 0, Microskill 1
**Purpose:** Load and verify the five upstream package files that LP-14 depends on: `page-brief.json` (LP-00), `above-fold-blueprint.json` (LP-03), `offer-cta-architecture.json` (LP-06), `hero-section-package.json` (LP-07), and `urgency-package.json` (LP-13). Extract the key fields that govern all CTA generation decisions.
**Effort Level:** low
**Dependencies:** LP-00 output, LP-03 output, LP-06 output, LP-07 output, LP-13 output
**Output:** `packages-loaded.md`

---

## Input Schema

```yaml
input:
  project_directory: "[path to active project output directory]"
  expected_files:
    - "page-brief.json"
    - "above-fold-blueprint.json"
    - "offer-cta-architecture.json"
    - "hero-section-package.json"
    - "urgency-package.json"
```

---

## Execution Logic

1. **Locate each upstream file** in the project output directory. Check for existence of all five files.

2. **If `page-brief.json` is MISSING:** HALT immediately. LP-14 cannot proceed without the brief.

3. **If `offer-cta-architecture.json` is MISSING:** HALT immediately. LP-14 cannot proceed without CTA architecture.

4. **If `hero-section-package.json` is MISSING:** HALT immediately. LP-14 cannot thread CTAs to the hero section without this.

5. **If `urgency-package.json` is MISSING:** Mark as `not_available`. LP-14 CAN proceed without urgency data — but urgency-type CTAs (Appeal Type 3) are FORBIDDEN without this file. Flag this clearly.

6. **If `above-fold-blueprint.json` is MISSING:** Mark as `not_available`. LP-14 can proceed with reduced context but flag it.

7. **Load `page-brief.json`** and extract:
   - `page_type` (type_a | type_b | hybrid)
   - `vertical`
   - `awareness_stage` (1-5)
   - `traffic_temperature` (cold | warm | hot)
   - `audience_profile` (primary pain, failed solutions, primary desire)
   - `voice_direction` (tone, register, anti-voice)
   - `offer_summary` (product name, price point, guarantee)

8. **Load `offer-cta-architecture.json`** and extract:
   - `cta_placement_count` (how many CTA positions on the page)
   - `cta_positions` (above-fold, mid-page, post-proof, post-testimonial, final, etc.)
   - `emotional_appeal_categories` (which of the 6 types LP-06 recommended)
   - `guarantee_terms` (type, duration, specific language)
   - `offer_structure` (single product, bundle, subscription, free trial)

9. **Load `hero-section-package.json`** and extract:
   - `threading_anchors.primary_hook_phrase`
   - `threading_anchors.promise_statement`
   - `threading_anchors.audience_qualifier`
   - `hero_copy.headline.text` (the selected headline)
   - `hero_copy.headline.type` (T1-T7)

10. **Load `urgency-package.json`** (if available) and extract:
    - `urgency_type` (deadline, quantity, access, seasonal)
    - `urgency_copy` (pre-written urgency text from LP-13)
    - `scarcity_data` (specific numbers, dates, quantities)
    - `justified` (Y/N — is the urgency real or manufactured)

11. **Load `above-fold-blueprint.json`** (if available) and extract:
    - `above_fold_cta_direction` (LP-03's CTA recommendation for above-fold position)

12. **Cross-validate page type** across loaded files. Flag any conflicts.

13. **Write `packages-loaded.md`** with all extracted fields organized by source file.

## Quality Criteria
- All critical files loaded (page-brief, offer-cta-architecture, hero-section-package)
- Key fields extracted with no gaps (missing fields marked `not_available`, not silently omitted)
- Threading anchors from LP-07 extracted verbatim — these are the most important inputs
- Urgency availability clearly flagged — controls whether Appeal Type 3 can be used
- Output is self-contained — downstream microskills can read this without loading original JSONs

## Failure Modes
- One or more critical files missing -> HALT with specific file names listed. Do not proceed with partial data.
- Threading anchors not extracted from hero-section-package.json -> HALT. CTAs without threading are disconnected from the page.
- Urgency data missing but not flagged -> This will lead to fabricated urgency in CTAs. Must be clearly marked.
- Fields present but empty in upstream JSON -> Extract as `not_available` and flag — do not invent values.
