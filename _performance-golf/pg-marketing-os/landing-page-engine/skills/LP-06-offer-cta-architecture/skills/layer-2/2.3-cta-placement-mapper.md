# 2.3 — CTA Placement Mapper

**Skill:** LP-06-offer-cta-architecture
**Position:** Layer 2, Microskill 3
**Purpose:** Map each CTA to its specific section in the page sequence, define the trigger context (what the visitor just read), conversion intent (what mental state the visitor is in), and the specific conversion action. This bridges the strategic CTA plan (1.3) with the language direction (2.5) by anchoring each CTA in its page context.
**Effort Level:** medium
**Dependencies:** `cta-strategy.md` (1.3), `brief-loaded.md` (0.1 — section numbers from LP-04), `pricing-display-strategy.md` (1.2)
**Output:** `cta-placement-map.md`

## CTA-Section Mapping Rules

| Page Zone | Section Examples (Type A) | Section Examples (Type B) | Conversion Intent |
|-----------|--------------------------|--------------------------|-------------------|
| Above-Fold | Section 1 (hero) | Section 1 (product hero + ATC) | Impulse — ready to buy NOW |
| Early | Section 2-3 (lead/story) | Section 2-3 (benefits/how-it-works) | Interest — wants to learn more |
| Mid-Page | Section 6-8 (proof blocks/product intro) | Section 5-7 (proof/reviews) | Educated — building conviction |
| Post-Proof | Section 10-12 (value stack/pricing) | Section 8-9 (guarantee/comparison) | Convinced — ready to commit |
| Close | Section 16-18 (close/P.S.) | Section 10+ (final CTA) | Final Decision — now or never |

## Execution Logic

**Step 1:** Load the section sequence from LP-04. Identify all sections already flagged for CTAs.

**Step 2:** For each CTA from the strategy (1.3), assign it to a SPECIFIC section number:
- Match the CTA's zone assignment to the corresponding section range
- If LP-04 already specified CTA sections, use those as primary placements
- If additional CTAs are needed (e.g., LP-04 specified 3, strategy calls for 5), add to appropriate sections

**Step 3:** For each placement, define the trigger context — what has the visitor just read?

Examples:
- CTA after proof block: "Visitor just read 5 testimonials with specific results. Belief is high. Trust is established."
- CTA after value stack: "Visitor just saw $397 total value reduced to $69. The deal frame is set."
- CTA at close: "Visitor has read the entire page. They either buy now or leave. Binary moment."

**Step 4:** For each placement, define conversion intent:
- **Impulse:** Ready to buy, no education needed (above-fold, Type B)
- **Educated:** Understands the mechanism, sees the value, needs final push
- **Convinced:** Proof has landed, guarantee has registered, price feels right
- **Final Capture:** Last chance before page end — consequence framing

**Step 5:** Note which CTAs sit near the guarantee (for inline risk-reversal) and which sit near the pricing block (for specificity/savings appeals).

**Step 6:** Define the sticky mobile CTA placement:
- Trigger: when does it appear (e.g., "after Section 1 ATC scrolls out of view")
- Elements: what it contains (product name, price, rating, button)
- Persistence: when does it disappear (e.g., "always visible after trigger" or "hides on checkout modal")

## Quality Criteria
- Every CTA mapped to a specific section number (not just a zone)
- Trigger context explains what the visitor experienced immediately before this CTA
- Conversion intent classified for each placement
- Sticky mobile CTA addressed
- Cross-reference table shows all sections with CTA flags
- Minimum 3 placements verified

## Failure Modes
- LP-04 section-sequence.json not available -> Use zone-based placement with estimated section ranges. Note: "Exact section numbers pending LP-04 output. Update when available."
- Strategy calls for above-fold CTA but awareness stage is 1-2 -> Flag conflict: "Above-fold buy CTA for unaware audience may underperform. Consider soft CTA (learn more) or delaying buy CTA to after mechanism section." Proceed with placement but document the risk.
- Only 2 suitable CTA sections in the page -> Add at least 1 more. Consider: after a proof block, near the guarantee, or at the page close. Every page must have >= 3 CTA positions.
