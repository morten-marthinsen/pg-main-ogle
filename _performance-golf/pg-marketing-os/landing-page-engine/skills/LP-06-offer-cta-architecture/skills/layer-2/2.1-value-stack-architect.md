## Value Stack Principles

1. **Value ALWAYS precedes price (Type A).** The value stack creates the psychological anchor. Without it, the price is evaluated in a vacuum.
2. **The "If All It Did Was" frame.** For the most important benefit, frame as: "If all [Product] did was [primary benefit], it would be worth $[high value]. But it does so much more..."
3. **Assigned values must be BELIEVABLE.** Inflated values destroy credibility. Use market-rate comparisons (e.g., "A personal consultation costs $200/session" for a coaching bonus).
4. **Order by impact, not cost.** Lead with the most compelling value item, not the most expensive.
5. **Bonus items should feel like genuine additions, not padding.** If a bonus is clearly low-value filler, exclude it.

---

## Value Stack Element Types

| Element Type | Description | Assigned Value Source |
|-------------|-------------|---------------------|
| **Core Product** | The primary product being sold | Market comparison (what comparable solutions cost) |
| **Bonus — Digital** | eBook, guide, report, checklist | Market rate for standalone purchase |
| **Bonus — Physical** | Additional physical product, tool | Retail value |
| **Bonus — Access** | Community, coaching call, live event | Price of comparable service |
| **Bonus — Service** | Consultation, custom plan, assessment | Professional service rate |
| **Bonus — Guarantee Enhancement** | Extended guarantee, priority support | Value of peace of mind (framed, not priced) |

## Execution Logic

**Step 1:** Determine if a full value stack is appropriate:
- **Type A:** Full value stack REQUIRED. List all items with assigned values. Total before price reveal.
- **Type B:** Abbreviated value stack. Key benefits listed. No formal "total value: $X" framing. Price is already visible.
- **Hybrid:** Full value stack in below-fold education zone. Abbreviated benefits near above-fold price.

**Step 2:** If Type A or Hybrid, order the value stack elements:

Recommended ordering:
1. Core product (always first — "Here's what you get")
2. Most compelling bonus (the one with highest perceived value)
3. Additional bonuses in descending perceived value
4. Access/community bonus (if applicable — social value)
5. Service bonus (if applicable — personalization value)
6. "If all it did was" frame on primary benefit

**Step 3:** Assign values to each element:
- Core product: Use comparison anchor from a more expensive alternative
- Bonuses: Use market rate for standalone purchase of similar item
- TOTAL: Sum all assigned values
- Verify total is 2-5x the actual price (less than 2x = weak anchor; more than 5x = unbelievable)

**Step 4:** Define the "If All It Did Was" frame:
- Select the primary benefit
- Frame: "If all [Product] did was [benefit], it would be worth $[credible high value]"
- This goes BEFORE the total value statement

**Step 5:** Define the transition from value stack to price reveal:
- Total value statement: "Total value: $[sum]"
- Comparison anchor: "[Comparable service] costs $[higher amount]"
- Price reveal: "Today, your investment is just $[actual price]"

## Quality Criteria
- All value stack elements ordered with assigned values
- Assigned values have credible market comparisons (not inflated)
- Value-to-price ratio between 2x and 5x
- "If all it did was" frame targets the primary benefit
- Per-day minimizer calculated with relatable comparison
- Type A has full reveal sequence. Type B has abbreviated format.

## Failure Modes
- No bonuses in the offer -> Value stack is core product only. Use comparison anchor and per-day minimizer as primary value framing. Note: "No bonuses available. Value stack relies on comparison anchor and per-day minimizer. Consider adding bonuses to strengthen offer."
- Assigned values seem inflated (total > 5x price) -> Reduce assigned values to credible range. Document adjustment.
- Price not yet confirmed -> Build the stack structure with placeholder prices. Flag: "Price placeholders — update when confirmed."
- Type B page but client wants a full value stack -> Note this is unusual for Type B. Can include below-fold value education section. Flag as potential hybrid treatment.
