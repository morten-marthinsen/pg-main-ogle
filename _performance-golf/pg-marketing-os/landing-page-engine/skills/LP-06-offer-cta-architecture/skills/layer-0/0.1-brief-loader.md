# 0.1 — Brief Loader

**Skill:** LP-06-offer-cta-architecture
**Position:** Layer 0, Microskill 1
**Purpose:** Load and verify page-brief.json from LP-00 output. Extract page type, audience profile, offer summary, awareness stage, and traffic temperature — all required for downstream offer/CTA decisions.
**Effort Level:** low
**Dependencies:** `page-brief.json` (LP-00 output)
**Output:** `brief-loaded.md`

---

## Input Schema

```yaml
input:
  project_folder: "[path to project output folder]"
  page_brief_path: "[path to page-brief.json]"
```

---

## Execution Logic

**Step 1:** Locate and load `page-brief.json` from the project output folder. Confirm the file exists and is parseable.

**Step 2:** Extract the following fields (all required for LP-06):

- `page_type.classification` — governs pricing display strategy (Type A = price-after-value, Type B = price-above-fold)
- `audience.sophistication.awareness_stage` — governs CTA emotional appeal selection and urgency appropriateness
- `audience.sophistication.skepticism_level` — governs guarantee duration and urgency trust threshold
- `traffic.temperature` — governs CTA placement aggressiveness (cold = later CTAs, hot = earlier CTAs)
- `offer_summary` — full offer details (pricing, bonuses, guarantee, urgency levers)
- `vertical.primary` — governs specimen selection for offer patterns

**Step 3:** Confirm `section-sequence.json` exists from LP-04. Extract CTA placement sections already defined by LP-04 (these are the sections LP-06 designs CTA details for).

**Step 4:** Note operating mode (downstream vs. standalone) — this determines whether Step 0.2 looks for a CopywritingEngine offer package or extracts from the brief.

## Quality Criteria
- page-brief.json loaded and parsed
- All required fields extracted or flagged as missing
- Page type confirmed — this is the single most important field for LP-06
- CTA sections from LP-04 identified for use in Layer 2 CTA mapping

## Failure Modes
- page-brief.json not found -> HALT. LP-06 requires page type classification.
- Offer summary entirely empty -> Flag as CRITICAL GAP. LP-06 can proceed but all offer fields marked `needs_validation`.
- section-sequence.json not found -> Warning (not halt). LP-06 uses default CTA placement zones instead of specific section numbers.
