# 0.3 — Specimen Offer Loader

**Skill:** LP-06-offer-cta-architecture
**Position:** Layer 0, Microskill 3
**Purpose:** Load offer, CTA, and guarantee patterns from Landing Page Engine specimens that match the current project's vertical and page type. These patterns inform Layer 1 classification and Layer 2 design decisions with real-world precedent.
**Effort Level:** low
**Dependencies:** `brief-loaded.md` (0.1 — page type + vertical)
**Output:** `specimen-offers-loaded.md`

---

## Input Schema

```yaml
input:
  page_type: "[type_a | type_b | hybrid — from 0.1]"
  vertical: "[vertical_id from page-brief.json]"
```

---

## Execution Logic

**Step 1:** Read `specimens/specimen-index.md` to identify specimens matching the project vertical and page type. If no exact match, select the closest vertical + page type combination.

**Step 2:** Read `specimens/cross-page-pattern-analysis.md` — extract from these sections:
- Section 3: CTA Architecture Patterns (CTA count, text consistency, risk reversal attachment)
- Section 4: Offer Structure Patterns (primary offer models, pricing psychology, guarantee spectrum)

**Step 3:** For each matching specimen, extract:
- Offer model (trial, subscription, bundle, no-pricing/demo, etc.)
- CTA count and placement pattern
- CTA text approach (consistent vs. varying)
- Guarantee type and duration
- Urgency/scarcity approach (if any)
- Risk reversal inline with CTA (Y/N + phrasing)

**Step 4:** Identify the MOST relevant specimen for this project based on vertical + page type + offer model similarity. This becomes the primary pattern reference.

## Quality Criteria
- At least 1 primary specimen identified matching vertical or page type
- Offer patterns extracted with specific CTA text examples (not summaries)
- Cross-page pattern insights synthesized from cross-page-pattern-analysis.md
- Implications for LP-06 are actionable (specific recommendations, not "consider options")

## Failure Modes
- No specimens match vertical + page type -> Select closest vertical match. Note limited specimen coverage.
- Specimen files not accessible -> Load from cross-page-pattern-analysis.md only. Note that raw specimen data was not reviewed.
- Zero offer/CTA data in specimens -> Flag as SPECIMEN GAP. Proceed with conversion-data-reference.md benchmarks only.
