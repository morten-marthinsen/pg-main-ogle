# 1.4 — Guarantee Architect

**Skill:** LP-06-offer-cta-architecture
**Position:** Layer 1, Microskill 4
**Purpose:** Design the guarantee with a branded name, specific duration, clear terms, claim process, and visual treatment direction. The guarantee must NEVER be generic ("money-back guarantee" or "satisfaction guaranteed"). This microskill creates the guarantee specification that LP-11 writes copy for and LP-15 integrates into the page.
**Effort Level:** medium
**Dependencies:** `offer-data-loaded.md` (0.2 — guarantee details), `specimen-offers-loaded.md` (0.3 — guarantee patterns), `offer-type-classification.md` (1.1)
**Output:** `guarantee-architecture.md`

## Guarantee Duration Selection Guide

| Duration | Return Rate | Best For | Vertical Norms |
|----------|-----------|---------|----------------|
| 30-day | ~10-15% | Low-ticket, low-risk products | Digital products, low-cost supplements |
| 60-day | ~8-12% | Standard products | Most supplements, info products |
| 90-day | ~7-10% | Products needing time to show results | Supplements, health programs, coaching |
| 365-day | ~5-8% | High-confidence, brand-building products | Premium supplements, signature programs |
| Performance | Varies | B2B, measurable outcomes | SaaS, B2B services, coaching |

**Counter-intuitive:** Longer guarantees produce FEWER returns and HIGHER conversion. Default to the longest duration the business can support.

## Guarantee Naming Framework

The branded guarantee name follows one of these patterns:

| Pattern | Formula | Example |
|---------|---------|---------|
| **Outcome Promise** | "The [Outcome] Promise" | "The Total Energy Promise" |
| **Confidence Guarantee** | "The [Duration] Complete Confidence Guarantee" | "The 90-Day Complete Confidence Guarantee" |
| **Product Identity** | "The [Product Name] [Promise Word]" | "The MagBreak Works-or-Free Guarantee" |
| **Transformation Guarantee** | "The [Duration] [Transformation] Guarantee" | "The 60-Day Better Sleep Guarantee" |
| **Zero-Risk Framing** | "[Product] — [Duration], Zero Risk" | "Try BioMax for 90 Days, Zero Risk" |

**Forbidden names:**
- "Money-back guarantee" (generic, invisible)
- "Satisfaction guaranteed" (generic, no specifics)
- "Full refund guarantee" (unbranded)
- "100% guarantee" (meaningless)

## Execution Logic

**Step 1:** Review guarantee details from `offer-data-loaded.md`. If the CopywritingEngine already produced a branded guarantee name, use it. If `not_specified`, create one.

**Step 2:** Select guarantee duration:
- If specified in offer data: use that duration
- If not specified: use vertical norm from specimen data
- Default recommendation: 60 days (optimal balance of conversion lift and return rate)
- For supplements: recommend 90 days (products need time to show results)
- For info products: recommend 30-60 days (consumption is immediate)
- For B2B/performance: recommend outcome-based with specific metric

**Step 3:** Create or confirm the branded guarantee name using the naming framework above. The name must:
- Include the product name OR the primary outcome
- Include the duration
- Sound like a NAMED PROMISE, not a policy

**Step 4:** Define specific terms:
- What triggers the guarantee (e.g., "If you don't feel more energized within 90 days")
- How to claim (e.g., "Email support@company.com or call 1-800-XXX-XXXX")
- What the customer gets (full refund, exchange, credit)
- Any conditions (e.g., "Even if you've used the entire bottle")

**Step 5:** Define visual treatment direction:
- Badge/seal design brief (shape, colors, text elements)
- Certificate treatment (formal document style)
- Where it appears on the page (near pricing AND at close, minimum)

**Step 6:** Determine guarantee type:
- **Money-back:** Standard refund within duration
- **Exchange:** Swap for different product/variant
- **Results-based:** Tied to specific outcome ("If you don't see X results")
- **Performance:** Measurable metric guarantee ("15% improvement or don't pay")

## Quality Criteria
- Guarantee has a branded name (NOT generic)
- Duration is specific (number of days, not "generous")
- Terms include trigger, claim process, and what customer receives
- Visual treatment direction is specific enough for a designer to execute
- Placement specifies at least 2 locations on the page
- Generic check confirms name passes

## Failure Modes
- CopywritingEngine provided a generic guarantee name -> Override with branded name. Note: "CopywritingEngine guarantee name was generic. Replaced with branded alternative."
- No guarantee details in offer data at all -> Create full guarantee specification. Flag as `needs_client_confirmation`. Recommend 60-day money-back as default.
- Client has explicitly stated "no guarantee" -> Document the conversion cost: "Guarantee absence reduces conversion rate significantly. Recommend at minimum a branded 30-day guarantee." Flag for human review.
