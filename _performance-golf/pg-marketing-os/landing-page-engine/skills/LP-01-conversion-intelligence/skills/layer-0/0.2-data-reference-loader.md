# 0.2: Data Reference Loader

> **Skill:** LP-01-conversion-intelligence
> **Layer:** 0
> **Position:** 2 of 3
> **Purpose:** Load conversion-data-reference.md into working memory and produce a structured inventory of all available data sections for Layer 1 matching.
> **Effort Level:** haiku
> **Dependencies:** conversion-data-reference.md exists at `LandingPageEngine/conversion-data-reference.md`
> **Output:** `data-reference-load.md`

---

## INPUT

```yaml
input:
  data_reference_path: "LandingPageEngine/conversion-data-reference.md"
```

---

## EXECUTION

1. **Read conversion-data-reference.md** in full. This is the sole source of truth for all benchmark data. No other source is valid.

2. **Inventory all data sections.** Document every section header and what type of data it contains:
   - Benchmark Data (Primary) — Unbounce Q4 2024 dataset
   - Industry-Specific Rates table
   - Element-Level Impact Data table
   - Bounce Rate Context table
   - The 7 Most Important Rules (with specific lift/penalty numbers)
   - Offer Hierarchy (3 elements + 7 offer types)
   - Page Speed Targets table
   - CTA Data (Placement + Copy Performance)
   - Social Proof Thresholds tables
   - Guarantee Data tables
   - Mobile-Specific Data tables
   - A/B Test Priority Data tables
   - Vertical-Specific Benchmarks (Health/Supplement + Info Product subsections)
   - Quick Reference: The 20 Most Important Stats

3. **For each section, note:**
   - Section name
   - Number of data rows/entries
   - Whether it contains vertical-specific or universal data
   - Key stat highlights

4. **Write data-reference-load.md** with the complete inventory.

## QUALITY CRITERIA

- Every section of conversion-data-reference.md inventoried
- Section count matches actual file content
- Key statistics noted accurately (not paraphrased or rounded)
- Data availability summary correctly identifies which verticals have dedicated benchmark sections
- File exists after execution

---

## FAILURE MODES

- **conversion-data-reference.md not found:** HALT — this file is required. It should exist at the LandingPageEngine root.
- **File appears empty or truncated:** HALT — note the issue, request file verification.
- **Section headers changed:** Proceed with best-effort inventory, flag any sections that don't match expected structure.
