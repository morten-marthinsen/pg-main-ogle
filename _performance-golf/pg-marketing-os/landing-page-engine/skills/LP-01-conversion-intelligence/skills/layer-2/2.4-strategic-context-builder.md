## EXECUTION

1. **Read all 4 input files** and brief-extract.md.

2. **Build LP-03 (Above-Fold Architecture) Guidance:**
   - What should be above the fold? (Based on page type + specimen analysis)
   - Hero element recommendation (image, VSL, product shot — from specimens)
   - CTA above fold? (Based on page type: Type B = yes, Type A = depends on traffic temp)
   - Trust signals above fold? (Based on element ranking + specimen patterns)
   - Price above fold? (Based on page type + specimen patterns)
   - Headline type recommendation (from filtered headline patterns)
   - Speed targets for above-fold load time

3. **Build LP-04 (Section Sequence Planner) Guidance:**
   - Recommended section count range (from benchmark synthesis + specimen analysis)
   - Universal skeleton to follow (from filtered patterns)
   - Vertical-specific required sections (from filtered patterns)
   - Section proportion guidance (proof-heavy vs. education-heavy vs. balanced — from specimen analysis)
   - Where proof blocks should appear in the sequence
   - Where CTA blocks should appear in the sequence
   - FAQ placement recommendation

4. **Build LP-05 (Social Proof Architecture) Guidance:**
   - Proof density target (from filtered patterns + benchmark synthesis)
   - Proof types to prioritize (from specimen analysis + element ranking)
   - First proof placement (from filtered patterns + awareness-based timing)
   - Proof volume targets (from benchmark synthesis)
   - The bimodal proof decision: proof-saturated vs. authority-substitution (from filtered patterns)

5. **Build LP-06 (Offer/CTA Architecture) Guidance:**
   - CTA count and placement (from benchmark synthesis)
   - CTA text pattern (consistent vs. varying — from specimen analysis)
   - Risk reversal at CTA recommendation (from filtered patterns)
   - Offer type recommendation (from benchmark synthesis)
   - Guarantee display guidance (from benchmark synthesis)
   - Pricing psychology techniques (from filtered patterns)
   - ONE OFFER PER PAGE enforcement reminder

6. **Build LP-17 (Conversion Audit) Targets:**
   - Target CVR range (floor, target, stretch)
   - Bounce rate targets
   - Speed targets
   - CTA count minimum
   - Proof volume minimum
   - Element completeness checklist (from element ranking)
   - Mobile optimization targets

7. **Identify key risks for this build:**
   - Data gaps (verticals/types with insufficient benchmark or specimen data)
   - Pattern gaps (areas where specimen data contradicts best practice)
   - Competitive risks (if vertical is saturated with similar page approaches)

8. **Identify key opportunities:**
   - Underutilized patterns (from cross-page-pattern-analysis.md gaps section)
   - High-impact elements that specimens use but are rare in this vertical
   - Differentiation angles from specimen contradictions

9. **Write strategic-context.md** with explicit per-skill guidance.

## QUALITY CRITERIA

- Every guidance section references specific data from the input files (not general advice)
- LP-03, LP-04, and LP-06 guidance sections ALL present (GATE_2 requirement)
- LP-05 and LP-17 guidance sections present for full coverage
- No guidance contradicts the benchmark data (e.g., recommending above-fold CTA for Type A cold traffic when specimens show this is rare)
- Risks and opportunities are specific to this build, not generic
- ONE OFFER PER PAGE rule explicitly included in LP-06 guidance
- CTA personalization (+202%) explicitly included in LP-06 guidance

---

## FAILURE MODES

- **Insufficient data for meaningful guidance:** Write the guidance section anyway with whatever data exists. Clearly note: "Low confidence — limited data available for this vertical/type. LP-[X] should supplement with defaults from MASTER-BLUEPRINT.md."
- **Contradictory inputs:** When benchmark-synthesis.md and specimen-analysis.md suggest different approaches, document both and recommend the approach with stronger data support. Note the contradiction for downstream skills to consider.
- **Generic advice creep:** If a guidance section reads like it could apply to any page type/vertical, it is too generic. Delete it and rewrite with specific references to the build parameters.
