# 2.2: Specimen Analysis Extractor

> **Skill:** LP-01-conversion-intelligence
> **Layer:** 2
> **Position:** 2 of 4
> **Purpose:** Extract actionable patterns from the matched specimens, synthesizing their specimen-index.md entries with relevant observations from cross-page-pattern-analysis.md to produce specimen-specific intelligence for this build.
> **Effort Level:** sonnet
> **Dependencies:** `matched-specimens.md` (from 1.3), specimen-index.md, cross-page-pattern-analysis.md
> **Output:** `specimen-analysis.md`

---

## INPUT

```yaml
input:
  matched_specimens: "matched-specimens.md"
  specimen_index: "LandingPageEngine/specimens/specimen-index.md"
  cross_page_analysis: "LandingPageEngine/specimens/cross-page-pattern-analysis.md"
  brief_extract: "brief-extract.md"
  page_type: "[type_a | type_b | hybrid]"
  primary_vertical: "[vertical_id]"
```

---

## EXECUTION

1. **Read matched-specimens.md** to get the ranked list of matched specimens.

2. **For each matched specimen (top 5 maximum):**

   a. **Re-read the specimen's full entry** in specimen-index.md to extract:
      - Section count
      - Proof density and type
      - Key pattern
      - Platform
      - Specimen value rating

   b. **Cross-reference with cross-page-pattern-analysis.md** to find:
      - Where this specimen appears in the section sequence analysis
      - Where this specimen appears in the proof architecture analysis
      - Where this specimen appears in the CTA architecture analysis
      - Where this specimen appears in the headline pattern analysis
      - Where this specimen appears in the offer structure analysis
      - Any strategic insights that reference this specimen

   c. **Extract actionable patterns** specific to this build:
      - Section sequence: What section order does this specimen use? How does it compare to the universal skeleton?
      - Proof strategy: What proof types does it use? Where does proof first appear?
      - CTA strategy: How many CTAs? Consistent or varying text? Risk reversal attached?
      - Offer structure: What pricing model? Trial? Subscription? Bundle?
      - Above-fold decisions: What's above the fold? Product image? VSL? Form?

   d. **Assess what's transferable** to this build vs. what's specimen-specific (e.g., Sunday Red's Tiger Woods authority is not transferable to a supplement page).

3. **Synthesize across all matched specimens** to identify:
   - Patterns that appear in 2+ matched specimens (high confidence transferable)
   - Patterns unique to 1 specimen (lower confidence, note as "observed in [specimen] only")
   - Contradictions between specimens (document both approaches)

4. **Write specimen-analysis.md** with per-specimen breakdowns and a cross-specimen synthesis.

## QUALITY CRITERIA

- Every pattern attributed to a specific specimen ID
- Cross-page analysis data accurately referenced (not invented)
- Transferability assessments are specific (not "this is generally useful")
- Contradictions between specimens documented rather than hidden
- Cross-specimen synthesis separates HIGH confidence (2+ specimens) from MEDIUM (1 specimen)
- At minimum 3 strategic takeaways produced

---

## FAILURE MODES

- **Only 1 specimen matched:** Analyze the single specimen thoroughly. All patterns are MEDIUM confidence (single source). Note in synthesis that cross-specimen validation was not possible.
- **Specimens from different page types:** This happens when secondary/tertiary matching was needed. Clearly label which patterns apply to above-fold (from Type B specimens) vs. below-fold (from Type A specimens).
- **cross-page-pattern-analysis.md doesn't mention a matched specimen:** This means the specimen wasn't included in the cross-page analysis. Extract patterns from specimen-index.md entry alone and note lower confidence.
