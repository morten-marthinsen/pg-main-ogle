# 3.1: Intelligence Validator

> **Skill:** LP-01-conversion-intelligence
> **Layer:** 3
> **Position:** 1 of 2
> **Purpose:** Verify that every benchmark, statistic, and data point in the synthesized outputs can be traced back to conversion-data-reference.md. Zero tolerance for hallucinated data.
> **Effort Level:** sonnet
> **Dependencies:** `benchmark-synthesis.md` (from 2.1), conversion-data-reference.md
> **Output:** `validation-report.md`

---

## INPUT

```yaml
input:
  benchmark_synthesis: "benchmark-synthesis.md"
  specimen_analysis: "specimen-analysis.md"
  filtered_patterns: "filtered-patterns.md"
  strategic_context: "strategic-context.md"
  conversion_data_reference: "landing-page-engine/conversion-data-reference.md"
  specimen_index: "landing-page-engine/specimens/specimen-index.md"
  cross_page_analysis: "landing-page-engine/specimens/cross-page-pattern-analysis.md"
```

---

## EXECUTION

1. **Read conversion-data-reference.md** fresh (do not rely on the Layer 0 load — re-read the original source).

2. **Extract every numeric claim** from benchmark-synthesis.md:
   - Every percentage (CVR ranges, lift numbers, penalties)
   - Every speed target (seconds)
   - Every volume threshold (review counts, CTA counts)
   - Every named statistic (e.g., "Unbounce Q4 2024," "41,000 pages")

3. **For each numeric claim, verify:**
   - Does this exact number appear in conversion-data-reference.md?
   - If yes: record the section, table, and row where it appears. Mark VERIFIED.
   - If no: is it a reasonable derivation from source data (e.g., a range synthesized from two data points)?
     - If reasonable derivation: mark DERIVED, document the derivation chain.
     - If not traceable: mark HALLUCINATED.

4. **Extract every specimen reference** from specimen-analysis.md:
   - Every specimen ID
   - Every claim about a specimen's properties (section count, proof density, etc.)

5. **For each specimen reference, verify:**
   - Does this specimen ID exist in specimen-index.md?
   - Do the claimed properties match the specimen-index.md entry?
   - Mark VERIFIED or MISMATCH.

6. **Extract every pattern claim** from filtered-patterns.md:
   - Every claim about frequency (e.g., "7/11 pages")
   - Every pattern assertion (e.g., "Type A pages use X skeleton")

7. **For each pattern claim, verify:**
   - Does this pattern appear in cross-page-pattern-analysis.md?
   - Does the frequency match?
   - Mark VERIFIED or MISMATCH.

8. **Compile the validation report:**
   - Total claims checked
   - VERIFIED count
   - DERIVED count (with derivation chain documented)
   - HALLUCINATED count (MUST be 0 to pass)
   - MISMATCH count (MUST be 0 to pass)

9. **If any HALLUCINATED or MISMATCH claims found:**
   - List each one with the exact text and why it failed verification
   - HALT — return to the layer that produced the hallucinated/mismatched data
   - Fix the claim (replace with sourced data or mark `not_available`)
   - Re-run validation

## QUALITY CRITERIA

- Every numeric claim in the synthesis outputs has been checked
- Every specimen ID has been verified against specimen-index.md
- Every pattern frequency has been verified against cross-page-pattern-analysis.md
- Zero hallucinated claims (PASS condition)
- Zero mismatched claims (PASS condition)
- Derived claims have documented derivation chains
- Validation report is comprehensive enough that a third party could reproduce the verification

---

## FAILURE MODES

- **Hallucinated benchmark found:** This is a severity-10 failure. HALT immediately. Identify the source layer (2.1 benchmark-synthesis.md most likely). Return to that layer, replace the hallucinated stat with sourced data or `not_available`. Re-run 3.1.
- **Specimen property mismatch:** Severity-7. The specimen was matched but its properties were mis-stated. Return to 2.2, correct the property. Re-run 3.1.
- **Pattern frequency mismatch:** Severity-5. The pattern exists but the frequency was mis-stated. Return to 2.3, correct the frequency. Re-run 3.1.
- **Source file changed since Layer 0:** If conversion-data-reference.md or specimen-index.md has been modified during execution, note the discrepancy and re-load from Layer 0.
