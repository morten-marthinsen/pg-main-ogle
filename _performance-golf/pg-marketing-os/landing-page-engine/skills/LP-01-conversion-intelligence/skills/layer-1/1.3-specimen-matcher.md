# 1.3: Specimen Matcher

> **Skill:** LP-01-conversion-intelligence
> **Layer:** 1
> **Position:** 3 of 4
> **Purpose:** Match specimens from specimen-index.md to this build using a hierarchical matching algorithm (page_type + vertical primary match, then fallbacks). Rank matches by quality.
> **Effort Level:** sonnet
> **Dependencies:** `brief-extract.md` (from 0.1), `specimen-index-load.md` (from 0.3)
> **Output:** `matched-specimens.md`

---

## INPUT

```yaml
input:
  brief_extract: "brief-extract.md"
  specimen_index_load: "specimen-index-load.md"
  specimen_index: "landing-page-engine/specimens/specimen-index.md"
  page_type: "[type_a | type_b | hybrid]"
  primary_vertical: "[vertical_id]"
  secondary_vertical: "[vertical_id | null]"
  traffic_temperature: "[cold | warm | hot]"
```

---

## EXECUTION

1. **Read the build parameters** from brief-extract.md: page_type, primary_vertical, secondary_vertical.

2. **Run Primary Match** (page_type AND vertical):
   - Scan specimen-index-load.md for specimens where BOTH conditions are true:
     - Specimen page_type matches build page_type (Type A matches Type A; Type B matches Type B; Hybrid matches either)
     - Specimen vertical matches build primary_vertical OR secondary_vertical
   - Record all primary matches.

3. **If <2 primary matches, run Secondary Match** (page_type only):
   - Scan for specimens matching page_type regardless of vertical.
   - Add these to the match list with lower match scores.

4. **If still <2 total, run Tertiary Match** (vertical only):
   - Scan for specimens matching vertical regardless of page_type.
   - Add these to the match list with even lower match scores.

5. **If still <2 total, run Fallback:**
   - Document the gap explicitly.
   - Include the highest-value specimens regardless of match criteria.
   - Set data confidence LOW for specimen matching.

6. **Score each matched specimen (1-10):**
   - Page type match: +4 points
   - Vertical match: +3 points
   - Proof density within 1 tier of what the build likely needs: +1 point
   - Offer model similarity (trial, subscription, one-time, free): +1 point
   - Specimen value rating HIGH: +1 point

7. **Rank specimens by match score** (highest first).

8. **For each matched specimen, document:**
   - Specimen ID, name, file path
   - Match score and scoring breakdown
   - Match tier (primary, secondary, tertiary, fallback)
   - Why this specimen is relevant to this specific build
   - Key pattern to study from this specimen
   - What aspects of this specimen are NOT applicable to this build

9. **Write matched-specimens.md** with all matches, ranked by score.

---

## QUALITY CRITERIA

- Every specimen ID traceable to specimen-index.md
- No invented specimen IDs
- Match scoring consistent and documented per specimen
- Match hierarchy correctly applied (primary before secondary before tertiary before fallback)
- Relevance explanations are specific to this build, not generic
- At minimum 2 specimens matched (or gap explicitly documented with confidence set to LOW)
- "Not Applicable" section populated to prevent downstream skills from copying irrelevant patterns

---

## FAILURE MODES

- **Zero specimens in index:** HALT — specimen-index.md may be empty or not loaded. Verify 0.3 completed.
- **All specimens score <3/10:** Proceed but set match confidence to LOW and document that all matches are weak. Add to data_gaps in strategic context.
- **Hybrid page type matching:** Match Hybrid builds against BOTH Type A and Type B specimens. Score Type A specimens for below-fold relevance and Type B specimens for above-fold relevance. Note the zone applicability.
- **Vertical not represented in specimen index:** This is expected for some verticals (finance, personal development currently have 0 specimens). Document the gap clearly — this is NOT a failure, it's a data limitation.
