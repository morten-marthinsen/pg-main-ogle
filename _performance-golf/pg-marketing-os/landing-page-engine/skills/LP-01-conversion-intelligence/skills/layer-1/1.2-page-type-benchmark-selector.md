# 1.2: Page-Type Benchmark Selector

> **Skill:** LP-01-conversion-intelligence
> **Layer:** 1
> **Position:** 2 of 4
> **Purpose:** Select all benchmarks from conversion-data-reference.md that apply to the classified page type (Type A, Type B, or Hybrid) and traffic temperature, plus universal benchmarks that apply regardless of vertical.
> **Effort Level:** sonnet
> **Dependencies:** `brief-extract.md` (from 0.1), `data-reference-load.md` (from 0.2)
> **Output:** `page-type-benchmarks.md`

---

## INPUT

```yaml
input:
  brief_extract: "brief-extract.md"
  data_reference_load: "data-reference-load.md"
  conversion_data_reference: "LandingPageEngine/conversion-data-reference.md"
  page_type: "[type_a | type_b | hybrid]"
  traffic_temperature: "[cold | warm | hot]"
  awareness_stage: "[1-5]"
```

---

## EXECUTION

1. **Read the page type and traffic temperature** from brief-extract.md.

2. **Select page-type-relevant benchmarks** from conversion-data-reference.md:

   **For Type A (Long-Form Sales Page):**
   - Long-form vs short-form leads: +220% (KlientBoost) — directly relevant
   - Page speed target: <3 seconds (from Page Speed Targets table)
   - CTA placement: after social proof block + after guarantee + end of page (from CTA Placement Data)
   - Bounce rate target: 40-60% good (from Bounce Rate Context)
   - Section count guidance: inferred from specimen data (cross-referenced in 2.3)

   **For Type B (Ecomm/PDP):**
   - Page speed target: <2 seconds (from Page Speed Targets table)
   - Mobile speed target: <2 seconds (from Page Speed Targets table)
   - Ecomm typical CVR: 2.5-3% (Shopify) — baseline
   - Sticky ATC bar: high impact (from Mobile-Specific Data)
   - CTA placement: above fold high impact (from CTA Placement Data)

   **For Hybrid:**
   - Combine relevant benchmarks from both Type A and Type B
   - Type B benchmarks for above-fold zone
   - Type A benchmarks for below-fold education zone

3. **Select traffic-temperature-relevant benchmarks:**
   - Cold: longer page, more education, later CTA, higher bounce rate acceptable
   - Warm: moderate page, mix of proof and education, mid-page CTA effective
   - Hot: shorter page acceptable, above-fold CTA critical, proof density over education

4. **Select universal benchmarks** (apply to all page types):
   - Personalized CTAs vs generic: +202% (HubSpot)
   - Video on landing pages: +86% (VWO)
   - Multiple offers penalty: -266% (KlientBoost)
   - Mobile-responsive vs static: +25.2% (VWO)
   - 1-second load delay: -7% (Google)
   - Simple copy (grade 5-7): +56% (VWO)
   - Single offer per page rule

5. **Select A/B test priority data** relevant to this page type from the A/B Test Priority table.

6. **For each benchmark, record:**
   - The exact stat
   - The exact source citation
   - Whether it applies to this page type specifically or universally
   - The traffic temperature it applies to (if temperature-specific)

## QUALITY CRITERIA

- Every benchmark has an explicit citation to conversion-data-reference.md
- Page-type-specific benchmarks correctly filtered (Type A benchmarks for Type A builds, etc.)
- Traffic temperature influences noted where applicable
- Universal benchmarks included for all builds
- No duplication with vertical-benchmarks.md (vertical-specific data stays in 1.1)
- Speed targets match the correct page type row in the Page Speed Targets table

---

## FAILURE MODES

- **Page type not recognized:** If page_type is not type_a, type_b, or hybrid, HALT — LP-00 classification is invalid.
- **Traffic temperature missing:** Default to cold (most conservative assumptions) and flag the default.
- **Hybrid treatment confusion:** For hybrid pages, document which benchmarks apply to the above-fold zone (Type B) and which apply to the below-fold zone (Type A). Do not conflate the two zones.
