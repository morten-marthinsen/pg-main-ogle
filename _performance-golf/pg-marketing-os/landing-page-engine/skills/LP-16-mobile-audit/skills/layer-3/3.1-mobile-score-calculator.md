# 3.1 — Mobile Score Calculator

> **Skill:** LP-16-mobile-audit
> **Layer:** 3 (Scoring & Fixes)
> **Model:** sonnet
> **Input:** `viewport-analysis.md`, `content-density-audit.md`, `cta-mobile-audit.md`, `media-performance-audit.md`, `speed-estimation.md`
> **Output:** `mobile-score.md`

---

## PURPOSE

Calculate the overall Mobile Readiness Score as a weighted average across all 7 audit dimensions. Collect dimension scores from the 5 Layer 2 audit files, apply dimension weights, compute the weighted average, and assign a verdict.

---

## PROCEDURE

### Step 1: Collect Dimension Scores

Read the dimension score from each Layer 2 audit file:

| Dimension | Source File | Score Field |
|-----------|-----------|-------------|
| Viewport Fit | `viewport-analysis.md` | Viewport Fit score |
| Content Density | `content-density-audit.md` | Content Density score |
| CTA Accessibility | `cta-mobile-audit.md` | CTA Accessibility score |
| Media Optimization | `media-performance-audit.md` | Media Optimization score |
| Page Speed | `speed-estimation.md` | Page Speed score |
| Typography | `viewport-analysis.md` | Typography (partial) score |
| Navigation | Combine from `viewport-analysis.md` (above-fold), `content-density-audit.md` (density impact), and `cta-mobile-audit.md` (CTA flow) | Synthesized Navigation score |

**Typography note:** The Typography dimension is primarily scored in 2.1 (Viewport Analysis) where font sizes and line heights are evaluated.

**Navigation note:** The Navigation dimension is distributed across multiple audit files. Synthesize the score from: above-fold navigation clarity (2.1), content density impact on navigation (2.2), and CTA flow/frequency (2.3).

### Step 3: Assign Verdict

| Score Range | Verdict | Action |
|------------|---------|--------|
| 8.0 - 10.0 | MOBILE_READY | Proceed to LP-17. Page is optimized for mobile. |
| 6.0 - 7.9 | MINOR_MOBILE_FIXES | Proceed to LP-17 with mobile fixes flagged. Fix before launch. |
| 4.0 - 5.9 | SIGNIFICANT_MOBILE_GAPS | Return to LP-15 for mobile revisions. Do NOT proceed to LP-17. |
| 0.0 - 3.9 | MOBILE_REBUILD | Return to LP-03/LP-04. Structural rebuild required. |

### Step 5: Cross-Check Score Plausibility

**Sanity checks:**
- If CTA Accessibility < 5 (missing sticky bar on Type B), total score SHOULD be below 8.0 (CTA carries 25% weight)
- If Page Speed > target by 2+ seconds, total score SHOULD reflect this
- If 4+ dimensions score below 7.0, verdict should be SIGNIFICANT_MOBILE_GAPS or worse

## CRITICAL RULES

1. **Weighted average only.** The Mobile Readiness Score is a weighted average — not a sum, not a simple average. Apply the correct weights.
2. **Round to 1 decimal.** Score is X.X, not X.XX or a whole number.
3. **The verdict thresholds are hard boundaries.** 7.9 = MINOR_MOBILE_FIXES. 8.0 = MOBILE_READY. No rounding up across verdict boundaries.
4. **Plausibility check is mandatory.** If the total score seems inconsistent with individual dimension scores (e.g., all dimensions below 6 but total is 7.5), recalculate.
5. **All 7 dimensions must have scores.** If any dimension score is missing from the Layer 2 files, HALT and return to the relevant auditor.
