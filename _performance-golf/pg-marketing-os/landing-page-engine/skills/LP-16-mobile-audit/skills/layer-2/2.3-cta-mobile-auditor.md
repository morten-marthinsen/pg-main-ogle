# 2.3 — CTA Mobile Auditor

> **Skill:** LP-16-mobile-audit
> **Layer:** 2 (Mobile Audit)
> **Model:** opus
> **Input:** `assembled-page-load.md`, `mobile-criteria.md`
> **Output:** `cta-mobile-audit.md`
> **Covers Dimensions:** CTA Accessibility (primary)

---

## PURPOSE

Audit every CTA element in the assembled page against mobile-specific requirements: sticky ATC/CTA bar presence, touch target sizing, thumb-zone accessibility, spacing from other tappable elements, and CTA frequency relative to scroll depth. CTA Accessibility carries the highest weight (25%) of all 7 audit dimensions because it directly determines whether visitors can convert on mobile.

**POSITIONAL REINFORCEMENT:** You are auditing CTA accessibility on a mobile device held in one hand. The thumb naturally reaches the bottom 60% of the screen. A CTA at the very top requires grip adjustment. A sticky bar at the bottom is always reachable. Every finding must reference specific mobile interaction constraints.

---

## PROCEDURE

### Step 1: Sticky Bar Assessment

Check `assembled-page-load.md` CTA Inventory for sticky bar specification:

| Page Type | Sticky Bar Status | Score Impact |
|-----------|------------------|-------------|
| Type B / Hybrid | Specified as present | No penalty |
| Type B / Hybrid | Not specified or absent | **Score ceiling: 4** (automatic FAIL territory) |
| Type A | Specified as present | Bonus (no penalty regardless) |
| Type A | Not specified or absent | **Score ceiling: 7** (no FAIL, but capped) |

**Evidence required:** Quote the sticky bar specification from the assembled page, or state "sticky bar: not specified in assembled page."

### Step 2: Touch Target Assessment

For EVERY CTA button in the CTA Inventory:

| Criterion | Minimum | Ideal | Evidence |
|----------|---------|-------|----------|
| Touch target height | 44px | 48-56px | Quote button size spec or note "unspecified" |
| Touch target width | 44px | Full-width or near-full-width on mobile | Quote button width spec or note "unspecified" |
| Padding around button | 8px from other tappable elements | 16px+ | Note proximity to other buttons, links, or tappable elements |

**"Unspecified" handling:** If the assembled page does not specify button sizes, this is a finding. Score as if touch target requirements are NOT met, because implementation without explicit specs defaults to unpredictable sizing.

### Step 3: Thumb Zone Analysis

Map each CTA's position relative to the thumb zone:

```
Mobile Screen (vertical):
┌─────────────────┐
│  STRETCH ZONE   │ Top 20% — requires grip change
│  (avoid for CTA)│
├─────────────────┤
│  NATURAL ZONE   │ Middle 20% — comfortable reach
│                 │
├─────────────────┤
│  THUMB ZONE     │ Bottom 60% — natural thumb rest
│  (optimal for   │
│   primary CTA)  │
└─────────────────┘
```

For each CTA, determine its zone placement:
- **Thumb zone (bottom 60%):** Optimal — no penalty
- **Natural zone (middle 20%):** Acceptable — minor note
- **Stretch zone (top 20%):** Flag — requires grip adjustment to tap

**Sticky bar exception:** A sticky bar fixed to the bottom of the viewport is ALWAYS in the thumb zone by definition.

### Step 4: CTA Spacing Assessment

Check for "fat finger" risk — CTAs too close to other tappable elements:

| Risk | Spacing | Example |
|------|---------|---------|
| High risk | < 8px between tappable elements | CTA button directly adjacent to a text link |
| Medium risk | 8-16px | Adequate but tight on mobile |
| Low risk | 16px+ | Comfortable spacing |

### Step 5: CTA Frequency vs. Scroll Depth

Calculate whether the user encounters CTAs at regular intervals while scrolling:

| Page Length | Minimum CTA Frequency | Ideal |
|-------------|----------------------|-------|
| 1-3 screen heights | 2 CTAs (hero + close) | 3 |
| 4-7 screen heights | 3 CTAs (hero + mid + close) | 4-5 |
| 8+ screen heights | 4+ CTAs (every ~2 screen heights) | 5+ or sticky bar |

**Scroll height estimation:** Total sections x ~1.5 screen heights per section (rough mobile average).

## CRITICAL RULES

1. **Type B without sticky bar = automatic score ceiling of 4.** This is non-negotiable. Sticky ATC bar is the single highest-impact mobile conversion element (conversion-data-reference.md). A Type B page without it has a fundamental mobile architecture failure.
2. **"Unspecified" touch targets = assume non-compliant.** If the assembled page does not specify button sizes, the auditor cannot assume they will meet 44px minimum. Score accordingly.
3. **Thumb zone matters.** A CTA that requires the user to shift grip is a friction point. On mobile, friction = abandonment. The thumb zone analysis is not optional.
4. **CTA frequency scales with page length.** A 3-section Type B page needs 2-3 CTAs. A 13-section Type A page needs 5+ or a sticky bar. Calculate based on actual section count from the assembled page.
5. **Sticky bar is not just "nice to have" for Type B.** The data is clear: always-visible CTA on mobile is the highest-impact single change for mobile conversion. This is why CTA Accessibility carries 25% weight — the heaviest of all 7 dimensions.
