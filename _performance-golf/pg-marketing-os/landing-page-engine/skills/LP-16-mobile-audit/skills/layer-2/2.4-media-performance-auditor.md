# 2.4 — Media Performance Auditor

> **Skill:** LP-16-mobile-audit
> **Layer:** 2 (Mobile Audit)
> **Model:** opus
> **Input:** `assembled-page-load.md`, `mobile-criteria.md`
> **Output:** `media-performance-audit.md`
> **Covers Dimensions:** Media Optimization (primary)

---

## PURPOSE

Audit every image, video, and media element in the assembled page for mobile optimization: image format (WebP vs PNG/JPEG), lazy loading directives, dimension appropriateness for mobile screens, video autoplay behavior, and hero image mobile-specific treatment. Media is the largest contributor to page weight and therefore to load time — optimized media is the foundation of mobile page speed.

**POSITIONAL REINFORCEMENT:** You are auditing media assets for delivery over a mobile network connection. Every unoptimized image costs ~220KB extra (300KB PNG vs 80KB WebP). Every missing lazy loading directive means an image loads immediately even if the user never scrolls to it. These are not theoretical concerns — they translate directly to load time and conversion loss.

---

## PROCEDURE

### Step 1: Image Format Audit

For EVERY image in the Image Inventory from `assembled-page-load.md`:

| Check | Pass | Fail |
|-------|------|------|
| Format specified as WebP | Explicit WebP directive | PNG, JPEG, or format unspecified |
| Fallback format specified | WebP with JPEG/PNG fallback for older browsers | WebP only (no fallback) or no format spec |

**Why WebP matters:** WebP provides 25-35% smaller file sizes than JPEG at equivalent quality, and 80%+ smaller than PNG. On mobile networks, this directly translates to faster load times.

### Step 2: Lazy Loading Audit

For EVERY image in the assembled page:

| Position | Lazy Loading Required? | Rationale |
|----------|----------------------|-----------|
| Above fold (Section 1) | NO — load immediately | Above-fold images must render instantly |
| Below fold (Section 2+) | YES — lazy load required | Below-fold images should only load when user approaches them |

**Check:** Does the assembled page specify lazy loading directives for below-fold images?

### Step 3: Image Dimension Audit

For EVERY image, check whether dimensions are specified and appropriate for mobile:

| Check | Mobile Appropriate | Desktop Oversized |
|-------|-------------------|-------------------|
| Max width | 750px (2x retina for 375px display) | > 750px source for mobile-primary image |
| Responsive srcset | Multiple sizes specified for different viewports | Single size for all viewports |
| Art direction | Mobile-specific crop/version for hero image | Same crop for desktop and mobile (may lose critical content) |

### Step 4: Video Audit

For EVERY video embed in the assembled page:

| Check | Pass | Fail |
|-------|------|------|
| Autoplay behavior | Disabled on mobile, OR muted autoplay with poster frame | Unmuted autoplay (impossible on mobile anyway, but indicates poor spec) |
| Poster frame | Specified (shows before video loads) | No poster frame (blank space or spinner) |
| Mobile format | Short-form or responsive player | Fixed desktop-width player |
| Preload strategy | `preload="none"` or `preload="metadata"` | `preload="auto"` (loads full video immediately) |

### Step 5: Hero Image Special Assessment

The hero image has unique mobile requirements because it is the first visual element the user sees:

| Check | Required | Why |
|-------|----------|-----|
| Mobile-specific crop | Recommended for Type A, Required for Type B | Desktop hero images often have landscape orientation that loses impact when compressed to portrait mobile |
| Compression priority | Hero image should be the MOST optimized image on the page | It loads first and sets LCP (Largest Contentful Paint) |
| Alt text for accessibility | Descriptive alt text specified | Screen readers and SEO |

## CRITICAL RULES

1. **"Unspecified" format = assume PNG at 300KB average.** This is the conservative estimation for calculating page weight impact. The assembled page must specify formats for a PASS.
2. **Image optimization is the #1 speed lever** (conversion-data-reference.md, Speed Optimization Priority). Every audit must calculate the weight impact of current vs. optimized media.
3. **Hero image gets special treatment.** It is the first image loaded, sets LCP, and has the largest visual impact. Hero image optimization is never optional.
4. **Lazy loading is binary per image.** Above fold = no lazy load. Below fold = lazy load. There is no middle ground.
5. **Video on mobile has lower completion rates** (cross-page-pattern-analysis.md context). If the assembled page specifies video, note that short-form or optional video is preferred for mobile.
