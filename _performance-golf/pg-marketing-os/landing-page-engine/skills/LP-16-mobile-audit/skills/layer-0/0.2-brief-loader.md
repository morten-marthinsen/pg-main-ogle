# 0.2 — Brief Loader

> **Skill:** LP-16-mobile-audit
> **Layer:** 0 (Input Loading)
> **Model:** haiku
> **Input:** `page-brief.json` path (from LP-00)
> **Output:** `brief-load.md`

---

## PURPOSE

Load the page brief from LP-00 and extract the fields critical for mobile audit configuration: page type classification, traffic source, vertical, target audience demographics, and any mobile-specific requirements the brief may contain.

---

## PROCEDURE

1. **Read** the `page-brief.json` file from the project output directory.
2. **Extract** the following into `brief-load.md`:

### Required Extractions

| Field | What to Extract | Why It Matters for Mobile Audit |
|-------|----------------|-------------------------------|
| Page Type | `type_a`, `type_b`, or `hybrid` | Governs ALL mobile thresholds (sticky bar required vs recommended, load time targets, CTA placement rules) |
| Traffic Source | Primary traffic source (Facebook ads, Google ads, email, organic, etc.) | Determines primary device context — Facebook/Instagram = heavily mobile; Google = mixed; email = mixed |
| Vertical | Product/service vertical | Some verticals skew more mobile (health/supplement DTC, personal care) than others (B2B SaaS) |
| Target Demographics | Age range, tech sophistication | Older demographics = larger touch targets, simpler navigation. Younger = more thumb-zone aware |
| Mobile Requirements | Any mobile-specific requirements stated in the brief | Client may have explicit mobile mandates (sticky bar, app-like experience, etc.) |
| Platform | Target implementation platform (Shopify, ClickFunnels, WordPress, etc.) | Platform determines mobile responsive capabilities and constraints |

3. **Write** the extracted data to `brief-load.md`.

---

## OUTPUT FORMAT

```markdown
# Brief Load — LP-16 Mobile Audit

## Page Type Classification
- **Page Type:** [type_a | type_b | hybrid]
- **Classification Confidence:** [from LP-00]

## Traffic Context
- **Primary Traffic Source:** [Facebook/Instagram ads | Google ads | email | organic | mixed | unspecified]
- **Expected Mobile Traffic %:** [estimated based on traffic source — Facebook/Instagram: 70-80%, Google: 50-60%, email: 40-60%, organic: 50-65%]

## Vertical Context
- **Vertical:** [health/supplement | info product | B2B | ecommerce | personal care | etc.]
- **Mobile Skew:** [high | medium | low — based on vertical norms]

## Target Demographics
- **Age Range:** [if specified]
- **Tech Sophistication:** [if specified]
- **Mobile Implications:** [larger touch targets for older demo, etc.]

## Mobile-Specific Requirements
[Any explicit mobile requirements from the brief, or "NONE SPECIFIED"]

## Platform
- **Target Platform:** [Shopify | ClickFunnels | WordPress | custom | unspecified]
- **Platform Mobile Capabilities:** [built-in responsive | requires custom | unknown]
```

---

## GATE CHECK

**This microskill is complete when:**
- `brief-load.md` exists
- Page type is identified (this is the single most critical field — it governs all downstream thresholds)
- Traffic source is identified or marked as "unspecified"
- All sections populated (with "UNSPECIFIED" where data is absent)

**If `page-brief.json` is missing or unparseable:** HALT. Output message: "LP-16 requires page-brief.json from LP-00. Run LP-00 first."
