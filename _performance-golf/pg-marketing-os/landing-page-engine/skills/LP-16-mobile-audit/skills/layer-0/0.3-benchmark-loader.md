# 0.3 — Benchmark Loader

> **Skill:** LP-16-mobile-audit
> **Layer:** 0 (Input Loading)
> **Model:** haiku
> **Input:** `conversion-data-reference.md` (LandingPageEngine root)
> **Output:** `benchmark-load.md`

---

## PURPOSE

Load mobile-specific benchmark data from conversion-data-reference.md. Extract only the data relevant to mobile/speed audit — this is a focused extraction, not a full file copy. The goal is to provide Layer 2 auditors with citation-ready impact data they can reference in findings and fix recommendations.

---

## PROCEDURE

1. **Read** `conversion-data-reference.md` from the LandingPageEngine root directory.
2. **Extract** the following mobile-relevant benchmarks into `benchmark-load.md`:

### Required Extractions

| Section | What to Extract |
|---------|----------------|
| Page Speed Targets | Type A target (<3s), Type B target (<2s), maximum acceptable, mobile target |
| Speed Optimization Priority | Ranked list of speed optimization actions |
| Mobile-Specific Data | Full Mobile vs. Desktop Behavior Differences table |
| Mobile Optimization Priorities | Ranked list of mobile-specific optimization actions |
| Element-Level Impact Data | Only mobile-relevant rows: mobile-responsive (+25.2%), 1-second load delay (-7%), CTA above fold, sticky mobile CTA, 10% page load improvement (+7%) |
| CTA Placement Data | Sticky mobile CTA impact data |
| Social Proof Thresholds | Minimum thresholds (relevant for content density assessment) |

3. **Write** the extracted data to `benchmark-load.md`.

---

## OUTPUT FORMAT

```markdown
# Benchmark Load — LP-16 Mobile Audit

## Page Speed Targets
| Page Type | Target | Maximum Acceptable |
|-----------|--------|--------------------|

| Type A Long-Form | < 3 seconds | 4 seconds |
| Mobile (all types) | < 2 seconds | 3 seconds |

## Speed Impact Data
| Change | Impact | Source |
|--------|--------|--------|
| 1-second load delay | -7% conversions | Google |
| 10% page load improvement | +7% sessions | Google |

## Mobile Impact Data
| Change | Impact | Source |
|--------|--------|--------|
| Mobile-responsive vs static | +25.2% mobile conversions | VWO |
| Sticky ATC bar (mobile) | High impact | Industry |
| CTA button above fold | High impact | Industry |

## Mobile vs. Desktop Behavior Differences
[Full table from conversion-data-reference.md]

## Mobile Optimization Priority Ranking
1. Sticky ATC bar — always-visible CTA on mobile is highest-impact single change
2. Tap target size — minimum 44x44px for all buttons
3. Text size — minimum 16px body copy (prevents iOS auto-zoom)
4. Image compression — mobile networks slower
5. Above-fold CTA — must be visible on 375px (iPhone SE) width
6. Eliminate pop-ups — Google penalizes intrusive interstitials on mobile

## Speed Optimization Priority Ranking
1. Image optimization (WebP format, lazy loading) — biggest impact
2. Remove third-party scripts that block render
3. CDN delivery for global audiences
4. Core Web Vitals compliance (LCP, FID, CLS)

## CTA Placement Impact
| Placement | Impact | Notes |
|-----------|--------|-------|
| Above fold | High | Captures ready-to-buy visitors |
| Sticky (mobile) | High | Always-visible increases mobile CVR |
```

## GATE CHECK

**This microskill is complete when:**
- `benchmark-load.md` exists
- Page speed targets extracted for both Type A and Type B
- Mobile impact data extracted with source citations
- Mobile optimization priority ranking included
- All data includes source attributions (needed for Layer 2 fix citations)

**If conversion-data-reference.md is missing:** Continue with embedded fallback data (speed targets and mobile impact data are also embedded in LP-16-AGENT.md). Note in benchmark-load.md: "Source file not found — using embedded reference data."
