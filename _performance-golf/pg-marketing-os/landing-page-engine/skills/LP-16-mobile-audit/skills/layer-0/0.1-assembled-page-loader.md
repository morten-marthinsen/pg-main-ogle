# 0.1 — Assembled Page Loader

> **Skill:** LP-16-mobile-audit
> **Layer:** 0 (Input Loading)
> **Model:** haiku
> **Input:** `assembled-page-package.json` path (from LP-15)
> **Output:** `assembled-page-load.md`

---

## PURPOSE

Load the assembled page package from LP-15 and extract all content relevant to mobile audit: section inventory, element types, image references, video embeds, CTA placements, script references, and any existing mobile/responsive specifications.

---

## PROCEDURE

1. **Read** the `assembled-page-package.json` file from the project output directory.
2. **Extract** the following into `assembled-page-load.md`:

### Required Extractions

| Field | What to Extract | Where It Goes |
|-------|----------------|---------------|
| Section Inventory | Complete list of all sections with names, types, and estimated word counts | `## Section Inventory` |
| Element Inventory | All elements by type (headlines, images, CTAs, videos, forms, badges, etc.) | `## Element Inventory` |
| Image References | Every image reference: filename, format (if specified), dimensions (if specified), position, lazy loading directive (if specified) | `## Image Inventory` |
| Video Embeds | Every video embed: type (hosted/YouTube/Vimeo), autoplay setting, poster frame, position | `## Video Inventory` |
| CTA Placements | Every CTA: text, position (section number), button size specification (if any), sticky bar specification (if any) | `## CTA Inventory` |
| Script References | All third-party scripts: analytics, chat widgets, pop-up tools, tracking pixels | `## Script Inventory` |
| Mobile Specifications | Any existing mobile-specific specs: responsive breakpoints, mobile-specific images, font sizes, touch target sizes | `## Mobile Specifications` |
| Total Page Metrics | Total sections, total estimated word count, total images, total videos, total CTAs, total scripts | `## Page Metrics Summary` |

3. **Write** the extracted data to `assembled-page-load.md`.

---

---

## GATE CHECK

**This microskill is complete when:**
- `assembled-page-load.md` exists
- All 8 extraction tables are populated (with "NONE" or "UNSPECIFIED" where data is absent — never skip a table)
- Page Metrics Summary is complete

**If `assembled-page-package.json` is missing or unparseable:** HALT. Output message: "LP-16 requires assembled-page-package.json from LP-15. Run LP-15 first."
