> **Skill:** LP-16-mobile-audit
> **Layer:** 4 (Package Assembly)
> **Model:** haiku
> **Input:** All outputs from all layers
> **Output:** `execution-log.md`

---

## PURPOSE

Write the execution log documenting every microskill executed, every spec file read, every checkpoint file created, and the final output status. This log enables verification that LP-16 ran completely and correctly — no skipped microskills, no missing checkpoint files, no phantom outputs.

---

## PROCEDURE

1. **Enumerate** every microskill from LP-16-AGENT.md.
2. **Record** execution status, spec file read, and output file for each.
3. **Record** all gate checks and their results.
4. **Record** the final three output files and their status.
5. **Write** `execution-log.md`.

## CRITICAL RULES

1. **Every microskill must be logged.** 15 microskills = 15 rows. No exceptions.
2. **"SKIPPED" is a valid status but requires explanation.** If a microskill was skipped, explain why in the Notes column.
3. **All 4 gates must be logged.** Each gate shows PASS or FAIL with notes.
4. **The log is the last file written.** It records the status of all other files, including the three final outputs.
5. **If LP-16 is incomplete, the log must say so.** Completion status at the bottom must accurately reflect whether all outputs were successfully produced.
