# 3.1 — Per-Section Scoring

> **Skill:** PDP-16-mobile-conversion-audit
> **Layer:** 3 (Scoring + Prioritization)
> **Position:** 1 of 3
> **Model:** sonnet
> **Purpose:** Calculate a 0-10 score for each of the 8 Baymard sections based on the percentage of criteria passed. Roll up per-criterion PASS/FAIL results from all 8 Layer 2 audit files into section-level scores. This is a mathematical aggregation step — no subjective judgment.
> **Dependencies:** All 8 Layer 2 audit files: `image-gallery-audit.md`, `buy-box-audit.md`, `product-info-audit.md`, `btf-section-audit.md`, `social-proof-audit.md`, `navigation-audit.md`, `mobile-performance-audit.md`, `conversion-flow-audit.md`
> **Output:** `per-section-scores.md`

## Execution Logic

**Step 1:** Read all 8 Layer 2 audit files. For each file, extract:
- Total criteria evaluated
- Criteria passed (PASS count)
- Criteria failed (FAIL count)
- Criteria N/A (excluded from scoring)
- Applicable criteria = Total - N/A

**Step 2:** Calculate per-section score using the formula:

```
Section Score = (Criteria Passed / Applicable Criteria) * 10
```

Example: If a section has 6 criteria, 1 is N/A, 4 pass, and 1 fails:
- Applicable = 5
- Score = (4/5) * 10 = 8.0

## Quality Criteria

- All 8 sections scored
- Math is correct: (passed / applicable) * 10, rounded to 1 decimal
- N/A criteria excluded from both numerator and denominator
- FAIL summary lists every single FAIL criterion across all 8 sections
- Below-7.0 sections identified
- File exists after execution

## Failure Modes

- Missing Layer 2 audit file -> HALT. All 8 files must exist before scoring.
- Score calculation error -> Double-check: passed + failed = applicable. Score = (passed/applicable) * 10.
- N/A counted in denominator -> Recalculate. N/A criteria must be excluded from both numerator and denominator.
- FAIL list incomplete -> Cross-reference every criterion from every section. Missing FAILs will cause priority ranking to be incomplete.
