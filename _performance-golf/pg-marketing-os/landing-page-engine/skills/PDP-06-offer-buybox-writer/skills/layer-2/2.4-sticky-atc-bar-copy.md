# 2.4 — Sticky ATC Bar Copy

**Skill:** PDP-06-offer-buybox-writer
**Position:** Layer 2, Microskill 4
**Purpose:** Write the compact copy for the sticky ATC bar that appears when the user scrolls past the main buy box. Contains: product name (may truncate on mobile), price, ATC button label (identical to main), and variant indicator if applicable. This bar keeps the purchase action always accessible.
**Effort Level:** medium
**Dependencies:** 0.1 input-verification.md (product name, price), 0.3 blueprint-extraction.md (sticky bar spec from PDP-03), 2.1 atc-copy.md (ATC label for consistency), 1.1 pricing-copy.md (price display)
**Output:** `sticky-atc-copy.md`

---

## Input Schema

```yaml
input:
  input_verification: "[path to input-verification.md]"
  blueprint_extraction: "[path to blueprint-extraction.md]"
  atc_copy: "[path to atc-copy.md]"
  pricing_copy: "[path to pricing-copy.md]"
  variant_copy: "[path to variant-copy.md or 'no variants']"
```

---

## Execution Logic

**Step 1:** Read `input-verification.md`. Extract:
- Full product name
- Current price (sale price if applicable)

**Step 2:** Read `blueprint-extraction.md`. Check if PDP-03 specified a sticky ATC bar:
- If yes -> proceed with sticky bar copy
- If no -> this microskill still writes the copy (sticky bars are best practice for PDPs); note that PDP-03 did not specify it

**Step 3:** Read `atc-copy.md`. Extract the main ATC button label:
- Sticky bar ATC label MUST be identical to main buy box ATC label
- This is non-negotiable for CTA consistency

**Step 4:** Read `pricing-copy.md`. Get the sale price display (compact version for sticky bar).

**Step 5:** Write the sticky bar product name:
- Desktop: Full product name (no truncation)
- Mobile: May truncate with ellipsis if name exceeds available space
- Write both versions:
```
Desktop: "[Full Product Name]"
Mobile:  "[Product Name...]" (truncation point specified)
```

**Step 6:** Write the sticky bar price display:
- Compact format: just the sale price (not the full anchor + savings)
```
Format: "$[XX.XX]"
```
- If subscription is selected: "$[XX.XX]/mo"
- The full savings display is in the main buy box; sticky bar is compact

**Step 7:** Write the sticky bar ATC label:
```
Label: "Add to Cart"
```
- MUST match the main buy box default state label exactly
- Not "Buy Now", not "Add", not "Quick Add"

**Step 8:** Write the variant indicator (if applicable):
```
Format: "[Selected Variant]" or "Chocolate Mint · 60 Count"
```
- Shows the currently selected variant so user knows what they're adding
- If no variants: omit this element

**Step 9:** Write the sticky bar visibility rule:
```
Appears: When main buy box ATC scrolls out of viewport
Disappears: When main buy box ATC scrolls back into viewport
Position: Fixed bottom (mobile) or fixed top (desktop)
```

**Step 10:** Write `sticky-atc-copy.md`.

---

## Output Schema

`sticky-atc-copy.md`:

```markdown
# Sticky ATC Bar Copy -- PDP-06

## Visibility
- **Appears:** When main buy box ATC button scrolls out of viewport
- **Position:** Fixed bottom (mobile) / Fixed top (desktop)

## Elements

### Product Name
- **Desktop:** "[Full Product Name]"
- **Mobile:** "[Truncated Product Name...]"
- **Truncation point:** [character count or word count where truncation occurs]

### Price
- **Compact display:** "$[XX.XX]"
- **Subscription display:** "$[XX.XX]/mo"
- **Note:** Compact price only -- full savings display is in main buy box

### ATC Button
- **Label:** "Add to Cart"
- **Matches main buy box label:** [Y/N -- MUST BE Y]

### Variant Indicator (if applicable)
- **Text:** "[Selected Variant Name]" or "[Variant 1] · [Variant 2]"
- **Updates dynamically:** When user changes variant selection in main buy box

## Full Bar Layout (left to right)
```
[Product Name] | [$XX.XX] | [Variant] | [Add to Cart]
```

## Consistency Check
- ATC label matches main buy box: [Y/N]
- Price matches pricing-copy.md: [Y/N]
- Product name matches brief: [Y/N]
```

---

## Quality Criteria
- ATC label is IDENTICAL to main buy box label ("Add to Cart")
- Price is accurate and matches pricing-copy.md
- Product name is full on desktop, gracefully truncated on mobile
- Variant indicator present if product has variants
- No marketing language or extra persuasion text in the sticky bar (it's a utility element)
- File exists after execution

## Failure Modes
- ATC label written as something other than "Add to Cart" -> Fix immediately to match main buy box
- Product name too long for even desktop sticky bar -> Do NOT truncate on desktop. Let it wrap or reduce font size (UX handles this, not copy).
- No variant data -> Omit variant indicator element; bar is name + price + ATC only
- Pricing-copy.md not yet written -> HALT, run 1.1 first
- PDP-03 did not specify sticky bar -> Write the copy anyway (sticky bars are PDP best practice); note gap in execution log
