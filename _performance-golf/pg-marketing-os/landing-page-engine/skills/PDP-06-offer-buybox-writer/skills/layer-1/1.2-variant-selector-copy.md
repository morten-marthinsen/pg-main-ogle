# 1.2 — Variant Selector Copy

**Skill:** PDP-06-offer-buybox-writer
**Position:** Layer 1, Microskill 2
**Purpose:** Write per-chip label copy for every exposed variant chip: flavor names, size labels, color swatches with text labels, stock status indicators ("Only 3 left"), and per-variant price differentials displayed inline on the chip.
**Effort Level:** medium
**Dependencies:** 0.1 input-verification.md (variant list), 0.3 blueprint-extraction.md (variant selector spec from PDP-03)
**Output:** `variant-copy.md`

---

## Input Schema

```yaml
input:
  input_verification: "[path to input-verification.md]"
  blueprint_extraction: "[path to blueprint-extraction.md]"
  pdp_reference_load: "[path to pdp-reference-load.md]"
```

---

## Execution Logic

**Step 1:** Read `input-verification.md`. Extract:
- Variant types (flavor, size, color, count, etc.)
- Full variant list with names
- Per-variant pricing (if prices differ)
- Stock status information (if available)

**Step 2:** Read `blueprint-extraction.md`. Extract PDP-03's variant selector spec:
- Selector type confirmation (must be exposed chips)
- Number of variants
- Any grouping or ordering PDP-03 specified

**Step 3:** Read `pdp-reference-load.md`. Confirm variant chip standards:
- Exposed chips (not dropdowns) -- already confirmed by PDP-03
- Price differential shown inline
- Stock status indicators
- Clear, descriptive labels

**Step 4:** Write chip label copy for each variant:
- Label must be the actual variant name, not an abbreviation
- If flavor: "Chocolate Mint", "Vanilla Bean", "Unflavored" (not "Choc", "Van", "Unf")
- If size: "30 Count", "60 Count", "90 Count" (not "S", "M", "L")
- If color: "Midnight Black", "Pearl White" (with color swatch indicator note)

**Step 5:** Write price differential copy for variants with different prices:
```
Format: "[Variant Name] (+$X.XX)" or "[Variant Name] (-$X.XX)"
Example: "90 Count (+$15.00)" or "30 Count" (base price, no differential shown)
```
- Base variant shows no differential
- Higher-priced variants show "+$X.XX"
- Lower-priced variants show "-$X.XX" (rare, but possible)

**Step 6:** Write stock status indicator copy:
- Low stock (< 10): "Only [X] left"
- Out of stock: "Out of Stock" (chip grayed out, not clickable)
- Back in stock: "Back in Stock!" (temporary highlight)
- Pre-order: "Pre-Order -- Ships [date]"
- If no stock data available: omit indicator (do not write "In Stock" -- that's noise)

**Step 7:** Write the variant section header label:
- Single variant type: "Flavor:" or "Size:" or "Color:"
- Multiple variant types: "Flavor:" above flavor chips, "Size:" above size chips
- NOT "Select Your Preferred Option" -- too verbose

**Step 8:** Write `variant-copy.md`.

---

## Quality Criteria
- Every variant from the brief has a chip with a label
- Labels are full names, not abbreviations
- Price differentials shown inline for variants with different prices
- Stock status only shown where meaningful (low stock, OOS) -- not "In Stock" noise
- Section header is concise (single word + colon)
- Default variant identified and justified
- File exists after execution

## Failure Modes
- No variant data in brief -> If PDP-03 specified variant selector, HALT and request variant list. If PDP-03 did not specify variant selector, skip this microskill.
- Price differential data unavailable -> Write labels without differentials, flag gap for human
- Stock data unavailable -> Omit stock indicators entirely (do not fabricate stock numbers)
- More than 2 variant types -> Write each type as a separate chip group with its own header
- Variant names too long for chip display -> Do NOT abbreviate. Let the chip wrap or truncate visually (UX handles this, copy stays full)
