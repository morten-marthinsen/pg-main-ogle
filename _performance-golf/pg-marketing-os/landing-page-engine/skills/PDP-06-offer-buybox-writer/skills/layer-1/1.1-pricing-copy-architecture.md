# 1.1 — Pricing Copy Architecture

**Skill:** PDP-06-offer-buybox-writer
**Position:** Layer 1, Microskill 1
**Purpose:** Write all price display copy strings: anchor price (struck-through), sale price (accent), savings in both % AND $ amount, per-unit pricing for multi-packs, shipping proximity messaging. Every number the user sees near the price block is written here.
**Effort Level:** medium
**Dependencies:** 0.1 input-verification.md (offer structure), 0.3 blueprint-extraction.md (PDP-03 price display spec)
**Output:** `pricing-copy.md`

---

## Input Schema

```yaml
input:
  input_verification: "[path to input-verification.md]"
  blueprint_extraction: "[path to blueprint-extraction.md]"
  pdp_reference_load: "[path to pdp-reference-load.md]"
```

---

## Execution Logic

**Step 1:** Read `input-verification.md`. Extract:
- OTP price per unit
- Subscription price and discount percentage
- Multi-pack options with per-unit pricing
- Free shipping threshold
- Guarantee duration (for context, not display here)

**Step 2:** Read `blueprint-extraction.md`. Extract PDP-03's price display architecture spec:
- What price tiers PDP-03 specified
- How PDP-03 structured the price block layout
- Any pricing constraints or directions from PDP-03

**Step 3:** Read `pdp-reference-load.md`. Confirm Baymard pricing display standards:
- Anchor price must be struck-through
- Savings must show BOTH % and $ amount
- Per-unit pricing for multi-packs
- Shipping visible near price

**Step 4:** Write the single-unit price display string:
```
Format: ~~$[original]~~ $[sale] (Save $[amount] -- [X]% off)
Example: ~~$79.99~~ $59.99 (Save $20.00 -- 25% off)
```
- If no discount exists (no sale), display clean price: "$59.99"
- If discount exists, ALWAYS show anchor + savings in both formats

**Step 5:** Write multi-pack price display strings (if applicable):
```
Format per tier:
  Best Value: Buy [X], Get [Y] Free -- $[per-unit]/bottle (Save $[total])
  Popular:    Buy [X] -- $[per-unit]/bottle (Save [X]%)
  Single:     Buy 1 -- $[price]/bottle
```
- Per-unit price on every tier
- Total savings on bundle tiers
- "Best Value" label on highest-savings tier

**Step 6:** Write shipping proximity copy:
- If free shipping on all orders: "Free Shipping"
- If threshold: "Free Shipping on Orders $[X]+"
- If paid shipping: "Shipping: $[X] (Free on orders $[threshold]+)"
- Position: directly below or beside price block

**Step 7:** Write per-unit breakdown copy (if subscription):
```
Format: "That's just $[X.XX]/day" or "$[X.XX]/serving"
```
- Only if it makes the price feel trivially small
- Skip if per-day/per-serving price is not compelling

**Step 8:** Validate all pricing copy against Baymard standards:
- Original price shown struck-through: Y/N
- Sale price shown in accent: Y/N
- Savings in % format: Y/N
- Savings in $ format: Y/N
- Per-unit pricing on multi-packs: Y/N
- Shipping visible near price: Y/N

**Step 9:** Write `pricing-copy.md`.

## Quality Criteria
- Every price point from the brief has a formatted display string
- Savings shown in BOTH percentage AND dollar amount (not just one)
- Multi-pack tiers have per-unit pricing
- Shipping proximity copy is specific (not "shipping available")
- No rounding errors in savings calculations
- Anchor price (struck-through) present whenever a discount exists
- File exists after execution

## Failure Modes
- No pricing data in input-verification.md -> HALT, cannot write pricing copy
- Savings calculation doesn't match (e.g., $79.99 - $59.99 != stated savings) -> Fix calculation
- PDP-03 blueprint specifies price tiers not in brief -> HALT, request missing price data
- Multi-pack pricing requested but only single-unit price available -> HALT, request multi-pack pricing
- Shipping info missing from brief -> Use "[Shipping calculated at checkout]" as fallback, flag gap
