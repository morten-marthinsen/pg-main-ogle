# 1.3 — Subscription Tile Copy

**Skill:** PDP-06-offer-buybox-writer
**Position:** Layer 1, Microskill 3
**Purpose:** Write all copy for the OTP (One-Time Purchase) tile and Subscribe & Save tile: tile labels, pricing inside each tile, explicit savings display, "Cancel anytime" assurance, and frequency selector copy. Neither tile is pre-selected.
**Effort Level:** medium
**Dependencies:** 0.1 input-verification.md (subscription pricing), 0.3 blueprint-extraction.md (subscription tile spec from PDP-03), 1.1 pricing-copy.md (price points)
**Output:** `subscription-copy.md`

---

## Input Schema

```yaml
input:
  input_verification: "[path to input-verification.md]"
  blueprint_extraction: "[path to blueprint-extraction.md]"
  pricing_copy: "[path to pricing-copy.md]"
  pdp_reference_load: "[path to pdp-reference-load.md]"
```

## Execution Logic

**Step 1:** Read `input-verification.md`. Extract:
- OTP price
- Subscription price
- Subscription discount percentage
- Subscription discount dollar amount
- Delivery frequency options (30/60/90 days)

**Step 2:** Read `blueprint-extraction.md`. Extract PDP-03's subscription tile spec:
- Tile layout (selectable tiles, not checkboxes)
- Tile positioning (OTP first or S&S first)
- Any specific Baymard constraints PDP-03 noted

**Step 3:** Read `pricing-copy.md`. Get the confirmed price points to ensure consistency.

**Step 4:** Read `pdp-reference-load.md`. Confirm subscription tile Baymard standards:
- Neither tile pre-selected
- Savings explicit inside S&S tile
- "Cancel anytime" visible
- Frequency selector present

**Step 5:** Write OTP tile copy:
```
OTP Tile:
  ○ One-time purchase
    $[XX.XX]
```
- Radio button or selectable tile indicator (unselected by default)
- Label: "One-time purchase" (clear, no marketing language)
- Price: exact dollar amount

**Step 6:** Write Subscribe & Save tile copy:
```
S&S Tile:
  ○ Subscribe & Save [X]%
    $[XX.XX]/shipment
    You save $[XX.XX] per shipment
    Cancel anytime
```
- Radio button or selectable tile indicator (unselected by default)
- Label: "Subscribe & Save [X]%" (savings percentage in the label)
- Price: subscription price with "/shipment" or "/mo" suffix
- Savings callout: explicit dollar savings per shipment
- Cancel assurance: "Cancel anytime" in smaller text inside tile

**Step 7:** Write frequency selector copy:
```
Deliver every: [30 days ▼]
Options: 30 days | 60 days | 90 days
```
- Label: "Deliver every:"
- Default: most common frequency (typically 30 days)
- Options: 30, 60, 90 days (or product-specific intervals)
- Only visible when S&S tile is selected

**Step 8:** Write the subscription section header (if separate from the tiles):
- "Purchase options:" or no header (tiles are self-explanatory)
- NOT "Choose Your Plan" or "Select Your Savings" (marketing language)

**Step 9:** Validate against Baymard standards:
- Neither tile pre-selected: Y/N
- Savings explicit inside S&S tile: Y/N
- "Cancel anytime" present: Y/N
- Frequency selector present: Y/N
- No marketing language in tile copy: Y/N

**Step 10:** If product does not offer subscription (OTP only), document this and skip S&S tile. Write only the single purchase option without tile selection UI.

**Step 11:** Write `subscription-copy.md`.

## Quality Criteria
- Neither tile is pre-selected (user must actively choose)
- OTP tile shows clear price
- S&S tile shows: savings percentage in label, dollar savings inside tile, "Cancel anytime"
- Frequency selector has clear label and sensible default
- Prices in subscription tiles match prices in pricing-copy.md
- No marketing language ("Unlock savings", "Best deal", "Smart choice")
- File exists after execution

## Failure Modes
- No subscription data in brief -> Skip S&S tile entirely; write "OTP only" note; no tile selection UI needed
- Subscription price missing but discount % available -> Calculate subscription price from OTP price and discount %; document calculation
- Frequency options not specified -> Default to 30/60/90 days; flag for human confirmation
- Savings calculation mismatch between 1.1 and 1.3 -> Reconcile with 1.1 pricing-copy.md; 1.1 is source of truth for numbers
- PDP-03 blueprint does not include subscription tiles -> Skip this microskill entirely; document in execution log
