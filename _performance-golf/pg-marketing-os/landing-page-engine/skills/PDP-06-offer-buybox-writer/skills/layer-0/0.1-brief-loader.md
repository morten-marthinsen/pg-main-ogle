# 0.1 — Brief Loader

**Skill:** PDP-06-offer-buybox-writer
**Position:** Layer 0, Microskill 1
**Purpose:** Load LP-00 page-brief.json and PDP-03 pdp-above-fold-blueprint.json. Validate page type is type_b or hybrid. Extract product data, pricing details, and offer structure needed for buy box copy writing.
**Effort Level:** low
**Dependencies:** LP-00 page-brief.json, PDP-03 pdp-above-fold-blueprint.json
**Output:** `input-verification.md`

---

## Input Schema

```yaml
input:
  page_brief: "[path to page-brief.json from LP-00]"
  pdp_blueprint: "[path to pdp-above-fold-blueprint.json from PDP-03]"
  project_directory: "[path to active project outputs directory]"
```

---

## Execution Logic

**Step 1:** Locate and read `page-brief.json` from LP-00 output directory. If file does not exist -> HALT. LP-00 must run first.

**Step 2:** Validate page type classification:
- `type_b` -> PROCEED (PDP route)
- `hybrid` -> PROCEED (PDP route with enhanced BTF)
- `type_a` -> HALT -- PDP-06 is not designed for long-form sales pages. Use LP-11 + LP-14 instead.

**Step 3:** Locate and read `pdp-above-fold-blueprint.json` from PDP-03 output directory. If file does not exist -> HALT. PDP-03 must run first. PDP-06 writes copy for the architecture PDP-03 designed.

**Step 4:** Extract product data from brief:
- Product name (full, never truncated)
- Product category / vertical
- SKU count and variant types (size, flavor, color, etc.)
- Price points (single, multi-pack, subscription if applicable)
- Key ingredients or features (raw list for short description)
- Guarantee / return policy (for micro-trust signals)
- Shipping information (for micro-trust signals)
- Manufacturing / testing certifications (for micro-trust signals)

**Step 5:** Extract offer structure from brief:
- OTP price per unit
- Subscription discount percentage and resulting price
- Multi-pack options with per-unit pricing
- Bundle savings amounts
- Free shipping threshold (if applicable)
- Guarantee type and duration

**Step 6:** Extract audience context from brief:
- Primary pain points (for benefit-forward short description)
- Purchase motivations (for benefit framing)
- Awareness stage (for copy specificity level)

**Step 7:** Write `input-verification.md`.

## Quality Criteria
- Page type is exactly `type_b` or `hybrid` (not `type_a`)
- PDP-03 blueprint exists and is loaded
- Product name extracted in full (never truncated)
- All pricing tiers identified with exact dollar amounts
- Variant types enumerated individually (not "various")
- Guarantee details specific (not "money-back" without duration)
- Shipping info specific (not "available" without threshold/cost)
- File exists after execution

## Failure Modes
- page-brief.json not found -> HALT, run LP-00 first
- pdp-above-fold-blueprint.json not found -> HALT, run PDP-03 first
- Page type = type_a -> HALT, use LP-11 + LP-14 instead
- No pricing data in brief -> HALT, cannot write pricing copy without prices
- No product name -> HALT, cannot proceed without product identity
- No variant data when PDP-03 blueprint specifies variant selector -> HALT, request variant details
