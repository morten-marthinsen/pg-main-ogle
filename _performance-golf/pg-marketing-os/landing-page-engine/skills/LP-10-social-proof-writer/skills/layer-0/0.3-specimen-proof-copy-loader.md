# 0.3 — Specimen Proof Copy Loader

**Skill:** LP-10-social-proof-writer
**Position:** Layer 0, Microskill 3
**Purpose:** Load proof copy patterns from matching specimens by vertical and page type. Extract the specific proof copywriting techniques used in real high-converting pages — intro phrasing, transition patterns, testimonial formatting, volume signal language.
**Effort Level:** low
**Dependencies:** `brief-loaded.md` (0.1 — for vertical + page type), specimen-index.md, raw specimen files
**Output:** `specimen-proof-copy.md`

---

## Input Schema

```yaml
input:
  page_type: "[from brief-loaded.md — type_a | type_b | hybrid]"
  vertical: "[from brief-loaded.md]"
  specimen_index_path: "landing-page-engine/specimens/specimen-index.md"
  specimen_raw_path: "landing-page-engine/specimens/raw/"
  proof_density_targets:
    - "HIGH or EXTREME density specimens preferred"
    - "Match page type first, then vertical"
```

---

## Execution Logic

1. **Read specimen-index.md** to identify specimens matching the current project:
   - Primary filter: page type match (Type A -> Type A specimens; Type B -> Type B specimens)
   - Secondary filter: vertical match (health -> health specimens, golf -> golf specimens)
   - Tertiary filter: proof density (prefer HIGH/EXTREME proof density specimens)

2. **Select 2-3 specimens** using this priority:
   - First choice: Same page type + same vertical + high proof density
   - Second choice: Same page type + different vertical + high proof density
   - Third choice: Any specimen with EXTREME proof density (for proof technique extraction regardless of vertical)
   - Always include at least one high-proof-density specimen (Hyros, LMNT, or BIOptimizers)

3. **For each selected specimen, load the raw file** and extract proof-specific copy patterns:

   **Testimonial Intro Patterns:**
   - How does the specimen introduce its proof sections?
   - What framing language does it use?
   - Does it address the reader directly or use third-person?

   **Testimonial Formatting Patterns:**
   - How are testimonials attributed? (Name, location, credential, photo indicator)
   - What text is bolded/highlighted within testimonials?
   - How long are individual testimonials?
   - Are pull-quotes used separately from full testimonials?

   **Transition Patterns:**
   - How does the specimen bridge FROM narrative INTO proof sections?
   - How does it bridge FROM proof sections BACK to narrative/CTA?
   - Are transitions explicit or implicit?

   **Volume Signal Patterns:**
   - How are customer/review counts formatted?
   - Where do they appear? (Above fold, inline, standalone)
   - What framing language surrounds the numbers?

   **Expert Endorsement Patterns:**
   - How are expert credentials presented?
   - How much context is given for why the expert matters?

4. **Write `specimen-proof-copy.md`** with extracted patterns organized by copy type.

## Quality Criteria
- 2-3 specimens loaded with at least 1 high-proof-density specimen
- Proof copy patterns extracted with verbatim examples (not paraphrased summaries)
- All 5 pattern categories populated (testimonial intros, formatting, transitions, volume signals, expert endorsements) — mark "Not present in specimen" where a category is absent
- Synthesis sections provide actionable guidance for Layer 2 generation
- Output is self-contained — Layer 2 microskills can reference this file without loading raw specimens

## Failure Modes
- Zero specimens match page type + vertical -> Load the 3 highest proof-density specimens regardless (Hyros, LMNT, BIOptimizers). Proof copy techniques transfer across verticals.
- Specimen raw file is missing or unreadable -> Skip that specimen, select an alternative. Flag in output.
- Specimen has zero proof copy to extract (Closers, Sunday Red) -> Do not select low/zero proof density specimens. Select alternatives with HIGH/EXTREME density.
