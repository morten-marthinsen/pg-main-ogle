# 4.1 — Proof Copy Compiler

**Skill:** LP-10-social-proof-writer
**Position:** Layer 4, Microskill 1
**Purpose:** Assemble all validated proof copy into a single structured `social-proof-copy-package.json` for downstream consumption by LP-15 (Assembly) and LP-17 (Conversion Audit). Every slot_id from proof-architecture.json must have a corresponding entry in this package.
**Effort Level:** low
**Dependencies:** All Layer 2 outputs (validated by Layer 3), `proof-copy-validation.md` (3.1), `specificity-audit.md` (3.2), `anti-slop-scan.md` (3.3)
**Output:** `social-proof-copy-package.json`

---

```yaml
input:

```

## Pre-Assembly Verification

Before assembling the package, verify ALL gates:

```yaml
ASSEMBLY-GATE:
  proof_copy_validation_score: "[X/8 — must be >= 7]"
  specificity_audit_result: "[PASS | FAIL — must be PASS]"
  anti_slop_scan_result: "[PASS | FAIL — must be PASS]"
  all_gates_pass: "[Y/N]"

  IF all_gates_pass = N: HALT — do not assemble. Return to failed validation.
```

---

## Assembly Logic

1. **Verify all 3 validation gates pass.** If any fails -> HALT.

2. **Build the `proof_copy_by_slot` array.** For each slot_id from proof-architecture.json:
   - Find the corresponding copy in the Layer 2 output files
   - Map to the correct copy_type
   - Include the full copy text
   - Include word count
   - Include specificity score (from specificity audit)
   - Include source status (written vs. requires_sourcing)
   - Include formatting notes

3. **Build the `transition_pairs` array.** For each proof block with transitions:
   - Include the IN transition copy
   - Include the OUT transition copy
   - Include the next_section_bridge context

4. **Build the `volume_signals` array.** For each volume signal:
   - Include placement information
   - Include the copy
   - Include number source status (exact / approximate / requires_verification)

5. **Build the `validation` block** with scores and pass/fail results.

6. **Build the `downstream_handoffs` block** with loading instructions for LP-15 and LP-17.

7. **Run final coverage check:** Count slot_ids in package vs. slot_ids in proof-architecture.json. Must be 100%.

## Quality Criteria
- All validation gates pass (proof copy score >= 7/8, specificity PASS, anti-slop PASS)
- 100% slot coverage (every slot_id from proof-architecture.json has an entry)
- Copy text accurately transferred from Layer 2 outputs (no truncation, no modification)
- Transition pairs complete for every proof block
- Volume signals include number source status
- Package is self-contained — LP-15 can load this file and place all proof copy without reading any other LP-10 output files
- JSON is valid and parseable

## Failure Modes
- Assembling with a failed validation gate -> HALT. Do not assemble. Return to Layer 3.
- Missing slot_ids in the package -> Run coverage check against proof-slots-loaded.md. Every slot must have an entry.
- Modifying copy text during assembly -> Copy text must be transferred EXACTLY as written and validated. Assembly is compilation, not revision.
- Invalid JSON -> Verify JSON structure before writing. Missing brackets/commas/quotes = unusable package.
- `requires_sourcing` slots listed in validation but not in the slot array -> Every slot must appear in proof_copy_by_slot, including placeholders.
