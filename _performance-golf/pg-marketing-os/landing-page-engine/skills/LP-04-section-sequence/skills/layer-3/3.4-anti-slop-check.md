# 3.4 — Anti-Slop Check

**Skill:** LP-04-section-sequence
**Position:** Layer 3, Microskill 4
**Purpose:** Scan all section `persuasion_function` descriptions and `notes` fields for vague, generic, or AI-slop language. Every section description must be specific enough that a Phase 3 writing skill can execute against it without guessing. "Build trust" is slop. "Establish authority via 3 named clinical studies supporting the mechanism" is specific. This check catches lazy descriptions before they propagate into 15+ downstream writing skills.
**Effort Level:** low
**Dependencies:** Complete section sequence with persuasion function descriptions (all Layer 2 outputs)
**Output:** `anti-slop-check.md`

---

## Input Schema

```yaml
input:
  section_sequence: "[complete ordered list with persuasion_function and notes for every section]"
```

---

## Slop Patterns to Detect

| Pattern | Example (SLOP) | Fix (SPECIFIC) |
|---------|---------------|----------------|
| **Vague trust claim** | "Build trust with the audience" | "Establish authority via Dr. [Name]'s 20-year credential + 3 clinical study citations" |
| **Generic benefit** | "Show the benefits of the product" | "Present 5 key benefits in scannable bullet format, each linked to a specific root cause symptom" |
| **Unquantified proof** | "Add testimonials" | "3 testimonials (named, with specific result quantified, location included)" |
| **Abstract emotional** | "Create an emotional connection" | "Discovery story: founder's personal struggle with [condition], failed solutions, breakthrough moment" |
| **Passive persuasion** | "Help the reader understand the value" | "Value stack: 4 bonuses with individual dollar values anchored to standalone price, total value $497" |
| **Filler transitions** | "Transition to the next section" | "Bridge from mechanism proof to product reveal: 'Now that you understand WHY [mechanism] works...'" |
| **Buzzword stacking** | "Leverage social proof to drive conversions" | "6 photo testimonials in 2-column grid, each with name + location + specific result + timeframe" |

---

## Execution Logic

**Step 1:** Extract every `persuasion_function` description from the section sequence.

**Step 2:** For each description, run the specificity test:
- Does it name specific TYPES of content? (testimonials, studies, story, comparison, etc.)
- Does it include QUANTITIES? (3 testimonials, 5 benefits, 1 hero image)
- Does it reference specific PROOF TYPES? (clinical study, named expert, before/after)
- Does it describe specific EMOTIONAL STATES or BELIEF SHIFTS? (not just "build trust")
- Could a writer execute against this description WITHOUT asking clarifying questions?

**Step 3:** Flag any description that fails the specificity test. Categorize the failure:
- `VAGUE` — Description is too broad to execute against
- `GENERIC` — Description could apply to any product/section
- `UNQUANTIFIED` — Missing specific counts or types
- `BUZZWORD` — Marketing jargon without substance

**Step 4:** For each flagged description, provide a rewritten version that passes the specificity test.

**Step 5:** Count total flags. If more than 20% of section descriptions are flagged, the sequence needs a description revision pass before proceeding.

## Quality Criteria
- Every section description explicitly tested — no section skipped
- Flagged descriptions have specific rewrites, not just "make it more specific"
- Rewrites include quantities, types, and execution-ready detail
- 20% threshold correctly applied

## Failure Modes
- Section descriptions not yet written (sequence is just section IDs) → FAIL all descriptions. Descriptions must be populated before this check can pass.
- Rewrites are themselves generic → The rewrite must pass the same specificity test. If you can't write a specific version, the section's purpose may be unclear — escalate to human.
- 0% flagged on first pass → Verify the check was thorough. Read each description carefully. A 0% flag rate is possible but uncommon for first-pass sequences.
