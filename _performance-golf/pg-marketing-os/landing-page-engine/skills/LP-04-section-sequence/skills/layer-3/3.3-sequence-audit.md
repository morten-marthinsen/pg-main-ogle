# 3.3 — 10-Point Sequence Audit

**Skill:** LP-04-section-sequence
**Position:** Layer 3, Microskill 3
**Purpose:** Score the complete section sequence against a 10-point validation checklist covering Structure Integrity (3), Proof Flow (3), Conversion Architecture (2), and Proportions (2). Minimum score 7.0/10 to proceed past GATE_3. This is the comprehensive quality gate — the sequence's final exam before it becomes the blueprint for all Phase 3 writing.
**Effort Level:** high
**Dependencies:** All Layer 1 and Layer 2 outputs, `logic-validation.md` (3.1), `dead-weight-check.md` (3.2)
**Output:** `sequence-audit.md`

---

## 10-Point Checklist

### Structure Integrity (3 points)

**Criterion 1: All mandatory sections present**
- Check: Every mandatory section for page type is in the sequence
- Source: `mandatory-check.md` (1.2) — confirmed earlier, but re-verify post-adjustments
- Score: 1 (all present) or 0 (any missing)

**Criterion 2: No sections before their dependencies**
- Check: Logical ordering — story before root cause, root cause before mechanism, education before product, value before price
- Source: Section sequence order
- Score: 1 (correct order) or 0 (any dependency violation)

**Criterion 3: No duplicate persuasion functions**
- Check: No two sections serve the identical persuasion function
- Source: `dead-weight-check.md` (3.2)
- Score: 1 (no duplicates) or 0 (duplicates found)

### Proof Flow (3 points)

**Criterion 4: Proof density follows wave pattern**
- Check: Early = light, mechanism zone = medium, proof blocks = heavy, close = medium
- Source: `proof-density-plan.md` (2.2)
- Score: 1 (wave pattern intact) or 0 (proof density flat or inverted)

**Criterion 5: Heavy proof block in top 40% of page**
- Check: At least one heavy proof block appears in the first 40% of sections
- Source: Section numbering + proof density
- Score: 1 (heavy proof early enough) or 0 (all heavy proof in bottom 60%)

**Criterion 6: Pre-close proof block present**
- Check: A proof block (medium or heavy) appears within 2 sections before the close/final CTA
- Source: Section sequence
- Score: 1 (pre-close proof present) or 0 (close section has no nearby proof)

### Conversion Architecture (2 points)

**Criterion 7: Three or more CTA placements**
- Check: At least 3 CTA placements mapped
- Source: `cta-placement-plan.md` (2.4)
- Score: 1 (≥3 CTAs) or 0 (<3 CTAs)

**Criterion 8: Urgency/scarcity justified or correctly absent**
- Check: If urgency section present, it has documented justification (real deadline, limited supply, etc.). If absent, no urgency justification exists.
- Source: Section sequence + brief
- Score: 1 (justified or correctly absent) or 0 (unjustified urgency present OR justified urgency missing)

### Proportions (2 points)

**Criterion 9: Word counts within benchmark ranges**
- Check: Every section's word count target falls within the AGENT.md benchmark table for the page type
- Source: `word-count-plan.md` (2.1) vs. AGENT.md benchmarks
- Score: 1 (all within range) or 0 (any section outside range without justification)

**Criterion 10: No section exceeds 30% of total page word count**
- Check: No single section's word count target exceeds 30% of total planned word count
- Source: `word-count-plan.md` (2.1)
- Score: 1 (no section >30%) or 0 (imbalance detected)

---

## Execution Logic

**Step 1:** Score each of the 10 criteria as 0 or 1. Document evidence for each score.

**Step 2:** Calculate total score (0–10).

**Step 3:** Apply scoring thresholds:
- **9–10:** Sequence ready for Phase 3. No revisions needed.
- **7–8:** Minor adjustments. Can proceed but note areas for improvement.
- **5–6:** Structural issues. Revise sequence before Phase 3.
- **Below 5:** Sequence rebuild required. Return to Layer 1.

**Step 4:** For any criterion scoring 0, write a specific fix recommendation.

**Step 5:** Determine GATE_3 status: Score ≥ 7.0 = PASS. Score < 7.0 = FAIL.

## Quality Criteria
- All 10 criteria explicitly scored with evidence — no silent passes
- Evidence references actual data from upstream outputs, not assumptions
- Fix recommendations are specific and actionable (not "improve proof flow")
- Score threshold correctly applied to determine GATE_3 status
- GATE_3 result is binary PASS or FAIL — no partial pass

## Failure Modes
- Upstream outputs missing (proof plan, CTA plan not yet produced) → Cannot score criteria that depend on missing data. Score those criteria as 0 with note: "Cannot evaluate — upstream output missing."
- Score is exactly 7 → This is a PASS. 7.0 is the minimum threshold.
- Multiple criteria fail but total is still ≥7 → PASS, but fix recommendations should still be documented for improvement.
- All criteria pass but the sequence "feels wrong" → This audit is objective. Subjective quality concerns should be noted in the audit but do not change the score.
