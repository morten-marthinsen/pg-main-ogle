# 3.2 — Dead Weight Detector

**Skill:** LP-04-section-sequence
**Position:** Layer 3, Microskill 2
**Purpose:** Identify and flag sections in the sequence that have no unique persuasion function — sections that either duplicate another section's job or exist as convention without moving the visitor toward purchase. Dead weight sections bloat page length, dilute proof density, and create scroll fatigue. Zero dead weight sections should survive into the final sequence.
**Effort Level:** medium
**Dependencies:** All Layer 2 outputs — `word-count-plan.md`, `proof-density-plan.md`, `visual-density-plan.md`, `cta-placement-plan.md`, `pacing-adjustment.md`
**Output:** `dead-weight-check.md`

## Dead Weight Patterns

| Pattern | Detection Signal | Why It's Dead Weight |
|---------|-----------------|---------------------|
| **Duplicate authority** | Two sections both establishing "who made this" | Audience gets authority once — repetition reads as insecurity |
| **Generic quality claim** | Section with phrases like "committed to excellence" but no proof elements | Claims without proof do nothing. Merge into guarantee or remove. |
| **Pure navigation/category** | Section that organizes content but adds no persuasion | Navigation is UX, not conversion architecture. Remove. |
| **Redundant proof blocks** | Back-to-back testimonial sections with no educational break | Proof fatigue without rhythm. Merge or space with educational content. |
| **Mission statement** | "Our mission is..." section that's brand-first, not customer-first | Visitor doesn't care about your mission. Reframe as customer benefit or remove. |
| **Filler education** | Educational section that repeats root cause or mechanism without adding new information | Redundant education creates boredom. Merge into parent section. |
| **Orphan section** | Section with no proof, no CTA, no emotional escalation | If it doesn't move the visitor toward purchase, it's dead weight. |

---

## Execution Logic

**Step 1:** For every section in the sequence, ask the litmus test question:

> "What does this section do to MOVE THE VISITOR TOWARD PURCHASE that no other section does?"

If the answer is unclear, vague, or duplicates another section's answer — flag it.

**Step 2:** Run pattern matching against the 7 dead weight patterns:
- Check for duplicate persuasion functions (two sections with effectively the same job)
- Check for sections with zero proof density AND zero CTA presence AND no educational function
- Check for back-to-back proof blocks without intervening educational/emotional content
- Check for brand-first content that doesn't serve the visitor's decision journey

**Step 3:** For each flagged section, determine the recommended action:
- **REMOVE** — Section has no salvageable persuasion function
- **MERGE** — Section's content can be absorbed into an adjacent section
- **REFRAME** — Section has potential but needs its persuasion function sharpened

**Step 4:** Verify zero dead weight sections remain after recommendations are applied. If any REMOVE/MERGE recommendations exist, the sequence must be updated before GATE_3.

## Quality Criteria
- Every section explicitly audited with the litmus test question
- Dead weight patterns checked systematically — not just gut feeling
- Flagged sections have specific, actionable recommendations (not just "remove this")
- Mandatory sections are NEVER flagged for removal — they can only be flagged for reframing
- Zero dead weight sections in the final "after" count

## Failure Modes
- Mandatory section flagged as dead weight → Do NOT recommend removal. Recommend REFRAME only. Mandatory sections exist for structural reasons even if their current persuasion function description is weak.
- All proof blocks flagged as redundant → This is over-aggressive detection. Proof blocks serve different wave positions (early, mid, pre-close). Check wave position before flagging.
- Zero sections flagged → This is acceptable. Not every sequence has dead weight. But verify the audit was thorough — did every section get the litmus test?
