# 0.1 — Brief Loader

**Skill:** LP-04-section-sequence
**Position:** Layer 0, Microskill 1
**Purpose:** Load and verify `page-brief.json` from LP-00. Confirm page type classification exists. Extract the key fields LP-04 needs: page type, vertical, awareness stage, proof inventory, offer structure, and product type. This is the foundation — if the brief is missing or malformed, nothing downstream can execute.
**Effort Level:** low
**Dependencies:** `page-brief.json` (LP-00 output)
**Output:** `brief-verified.md`

---

## Input Schema

```yaml
input:
  page_brief_path: "[path to page-brief.json from LP-00]"
```

---

## Execution Logic

**Step 1:** Locate and load `page-brief.json` from the project output directory. If file does not exist → HALT with error: "LP-00 output missing. Run LP-00 Brief & Page Type Classifier first."

**Step 2:** Verify required fields are present and non-empty:
- `page_type.classification` — REQUIRED (type_a | type_b | hybrid)
- `product.name` — REQUIRED
- `product.vertical` — REQUIRED
- `audience.awareness_stage` — REQUIRED (1–5)
- `offer.price_point` — REQUIRED
- `proof_inventory` — RECOMMENDED (testimonials_count, has_clinical_studies, has_before_afters, has_expert_endorsements)

**Step 3:** Flag any missing RECOMMENDED fields as warnings (do not halt — downstream microskills will use defaults, but the warning propagates).

**Step 4:** Extract and write the verified brief summary to `brief-verified.md`.

---

## Output Schema

`brief-verified.md`:

```markdown
# Brief Verification — [Product Name]

## Verification Status: [PASS | FAIL]

## Page Type
- Classification: [type_a | type_b | hybrid]
- Confidence: [from LP-00 if available]

## Key Fields Extracted
- Product name: [name]
- Vertical: [vertical]
- Awareness stage: [1–5] — [Unaware | Problem-Aware | Solution-Aware | Product-Aware | Most Aware]
- Price point: [price]
- Product type: [supplement | info-product | service | physical-product | software]

## Proof Inventory Summary
- Testimonials available: [count or "not_available"]
- Clinical studies: [Y/N or "not_available"]
- Before/afters: [Y/N + count or "not_available"]
- Expert endorsements: [Y/N or "not_available"]
- Media mentions: [Y/N or "not_available"]

## Warnings
[List any missing RECOMMENDED fields]
[or: "No warnings — all fields present"]

## Source File
- Path: [full path to page-brief.json]
- Last modified: [timestamp]
```

---

## Quality Criteria
- Page type classification confirmed present and valid (one of the three accepted values)
- All REQUIRED fields present — FAIL verification if any are missing
- Proof inventory extracted even if partial — downstream skills need whatever is available
- Warnings clearly separated from blocking failures

## Failure Modes
- `page-brief.json` not found → HALT. Do not proceed. LP-00 must run first.
- `page_type.classification` missing or empty → HALT. Page type is required for template selection in Layer 1.
- `page_type.classification` contains an invalid value (not type_a, type_b, or hybrid) → HALT with error describing the invalid value.
- Proof inventory entirely missing → WARNING only — proceed with defaults, but flag for all Layer 2 density planners.
