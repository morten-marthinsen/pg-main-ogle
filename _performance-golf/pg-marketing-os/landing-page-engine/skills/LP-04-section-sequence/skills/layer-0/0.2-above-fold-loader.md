# 0.2 — Above-Fold Loader

**Skill:** LP-04-section-sequence
**Position:** Layer 0, Microskill 2
**Purpose:** Load `above-fold-blueprint.json` from LP-03. Extract Section 1 spec — headline, subhead, hero image type, CTA presence, proof elements, and layout. Section 1 of the sequence is LOCKED to LP-03 output. LP-04 does not redesign the above-fold; it integrates it as the fixed first section.
**Effort Level:** low
**Dependencies:** `above-fold-blueprint.json` (LP-03 output), `brief-verified.md` (0.1)
**Output:** `above-fold-loaded.md`

---

## Input Schema

```yaml
input:
  above_fold_path: "[path to above-fold-blueprint.json from LP-03]"
  page_type: "[from brief-verified.md]"
```

---

## Execution Logic

**Step 1:** Locate and load `above-fold-blueprint.json` from the project output directory. If file does not exist → HALT with error: "LP-03 output missing. Run LP-03 Above-Fold Architecture first."

**Step 2:** Extract the Section 1 specification:
- Headline framework (pattern, word count, emotional register)
- Subhead framework (if present)
- Hero image specification (type, layout, product vs. lifestyle)
- CTA presence and type (Type B: ATC button; Type A: read-on or none)
- Proof elements present in above-fold (rating strip, volume counter, trust badges)
- Layout type (hero-left, hero-center, split-screen, etc.)

**Step 3:** Confirm page type alignment — LP-03 above-fold design should match the page type from LP-00. If mismatch detected (e.g., LP-03 produced a Type B above-fold but LP-00 classified as Type A), flag as WARNING.

**Step 4:** Write extracted Section 1 spec to `above-fold-loaded.md`.

## Quality Criteria
- Section 1 spec completely extracted — no fields left as "unknown" if LP-03 provided them
- Page type alignment checked between LP-00 and LP-03
- CTA presence accurately captured — this feeds into 2.4 CTA Placement Planner
- Proof elements in above-fold accurately captured — this feeds into 2.2 Proof Density Planner

## Failure Modes
- `above-fold-blueprint.json` not found → HALT. LP-03 must run before LP-04.
- LP-03 output is malformed or missing critical fields (no headline framework) → HALT with specific error describing which fields are missing.
- Page type mismatch between LP-00 and LP-03 → WARNING (do not halt). Use LP-00 classification as authoritative. Note the mismatch for human review.
