# 1.2 — Mandatory Section Validator

**Skill:** LP-04-section-sequence
**Position:** Layer 1, Microskill 2
**Purpose:** Confirm every mandatory section for the classified page type is present in the selected template. Cross-reference template [M] flags against the page type's non-negotiable list. Resolve conditional mandatory sections (e.g., "[M] if supplement") using product type from the brief. A missing mandatory section is a blocking failure — the sequence cannot proceed to Layer 2 with gaps.
**Effort Level:** medium
**Dependencies:** `template-selection.md` (1.1), `brief-verified.md` (0.1), `template-loaded.md` (0.3)
**Output:** `mandatory-check.md`

---

## Input Schema

```yaml
input:
  template_selection: "[from 1.1 — selected template with all sections listed]"
  page_type: "[type_a | type_b | hybrid]"
  product_type: "[supplement | info-product | service | physical-product | software — from brief]"
  vertical: "[from brief-verified.md]"
  has_founder_story: "[Y/N — from brief]"
  has_bonuses: "[Y/N — from brief]"
  has_transformation_claims: "[Y/N — from brief]"
  review_count: "[number — from brief]"
  customer_count: "[number — from brief]"
```

---

## Execution Logic

**Step 1:** Load the mandatory section list for the page type from AGENT.md:
- **Type A mandatory (unconditional):** above_fold, lead, root_cause_narrative, mechanism_narrative, proof_block_mid, product_introduction, pricing_block, guarantee_block, proof_block_preclose, faq, close, ps_section
- **Type A mandatory (conditional):** story [M if founder/discovery story], ingredient_deep_dives [M if supplement], value_stack [M if bonuses exist], urgency_scarcity [M if justified]
- **Type B mandatory (unconditional):** above_fold, key_benefits, trust_bar, how_it_works, faq, guarantee_block
- **Type B mandatory (conditional):** variant_quantity [M if variants exist], ingredient_deep_dives [M if supplement], social_proof_strip [M if 500+ customers], before_after [M if available], review_cascade [M if 25+ reviews], sticky_atc_mobile [M for mobile]

**Step 2:** Resolve all conditional mandatory sections using brief data:
- Check product_type against supplement conditionals
- Check has_founder_story against story conditional
- Check has_bonuses against value_stack conditional
- Check has_transformation_claims against before_after conditional
- Check review_count against review_cascade threshold
- Check customer_count against social_proof_strip threshold

**Step 3:** Compare resolved mandatory list against the sections present in template-selection.md. For each mandatory section:
- Present → PASS
- Missing → FAIL (blocking)

**Step 4:** Generate the validation report.

---

## Quality Criteria
- Every mandatory section for the page type explicitly checked — no silent assumptions
- Conditional mandatory sections resolved using actual brief data, not defaults
- Missing sections listed with clear action items
- FAIL status only if mandatory sections are actually missing — optional section absence is not a failure

## Failure Modes
- Template-selection.md not yet produced → HALT. Run 1.1 first.
- Brief data insufficient to resolve conditionals (e.g., product_type unknown) → Assume most restrictive interpretation (section IS mandatory) and flag for human confirmation.
- Hybrid template missing sections from either zone → Check both Type A and Type B mandatory lists for the applicable zones.
