# 4.1 — Sequence Compiler

**Skill:** LP-04-section-sequence
**Position:** Layer 4, Microskill 1
**Purpose:** Assemble the final `section-sequence.json` from all validated Layer 1–3 outputs. This is the single canonical output file that all downstream skills consume. The compiler does not make decisions — it compiles validated data into the schema defined in AGENT.md. Every field must be populated. No placeholders. No "TBD."
**Effort Level:** low
**Dependencies:** All Layer 1–3 outputs — specifically: `template-selection.md`, `mandatory-check.md`, `optional-decisions.md`, `section-1-integration.md`, `word-count-plan.md`, `proof-density-plan.md`, `visual-density-plan.md`, `cta-placement-plan.md`, `pacing-adjustment.md`, `logic-validation.md`, `dead-weight-check.md`, `sequence-audit.md`, `anti-slop-check.md`
**Output:** `section-sequence.json`

---

## Input Schema

```yaml
input:
  all_layer_outputs: "[all .md files from Layers 1-3]"
  sequence_audit_score: "[from 3.3 — must be >= 7.0]"
  gate_3_status: "[PASS — required to proceed]"
```

---

## Execution Logic

**Step 1:** Verify GATE_3 status is PASS. If FAIL → HALT. Do not compile a sequence that failed validation.

**Step 2:** Build the JSON structure following the exact schema from AGENT.md `SECTION-SEQUENCE.JSON SCHEMA`:

- **Header fields:** schema_version, created (ISO timestamp), project_name, page_type, total_sections, estimated_word_count (min/target/max), sequence_score
- **Sections array:** One entry per section, each with: number, section_id, name, mandatory, persuasion_function, content_source, word_count (target + range), proof_density (level + elements), visual_density (image_count + primary_type + layout), cta_present, cta_type, notes
- **Proof wave summary:** early_zone, mid_zone, close_zone
- **CTA placement summary:** All CTA positions with section numbers and types, sticky_mobile_cta flag
- **Logic validation:** All 4 test results + all_pass flag
- **Downstream handoffs:** Map of which skills consume which parts of this file

**Step 3:** For each section entry, pull data from the correct upstream file:
- Section order and IDs → `template-selection.md` + `optional-decisions.md`
- Persuasion functions → verified descriptions from `anti-slop-check.md` (use rewrites if originals were flagged)
- Word counts → `word-count-plan.md` adjusted by `pacing-adjustment.md`
- Proof density → `proof-density-plan.md`
- Visual density → `visual-density-plan.md`
- CTA data → `cta-placement-plan.md`
- Section 1 → `section-1-integration.md` (CONTENT_SOURCE_LOCK)

**Step 4:** Validate JSON structure. Confirm:
- No empty fields (no "", no null, no "TBD")
- All section numbers are sequential
- Total section count matches sections array length
- Word count total (sum of section targets) matches estimated_word_count.target

**Step 5:** Write `section-sequence.json` to the project output directory.

---

## Output Schema

`section-sequence.json` — follows the exact JSON schema defined in LP-04-AGENT.md under `SECTION-SEQUENCE.JSON SCHEMA`.

Key structural requirements:
- `schema_version`: "1.0"
- `sections[]`: Array of section objects, each fully populated
- `proof_wave_summary`: 3-zone summary
- `cta_placement_summary`: All CTA positions + sticky mobile
- `logic_validation`: 4 test results + all_pass boolean
- `downstream_handoffs`: Skill-to-section mapping

---

## Quality Criteria
- JSON is valid and parseable — no syntax errors
- Every field populated — zero empty strings, nulls, or placeholders
- Section numbers sequential with no gaps
- Word count totals are internally consistent
- Persuasion functions use anti-slop-verified descriptions
- Section 1 has CONTENT_SOURCE_LOCK flag
- Logic validation all_pass matches individual test results

## Failure Modes
- GATE_3 not passed → HALT. Cannot compile unvalidated sequence.
- Upstream file missing data for a required field → Do NOT use placeholder. HALT and identify which upstream output is incomplete.
- JSON syntax error on write → Validate JSON before finalizing. Re-format if needed.
- Section count mismatch (sections array length does not match total_sections) → Fix before writing. This is a compilation error.
