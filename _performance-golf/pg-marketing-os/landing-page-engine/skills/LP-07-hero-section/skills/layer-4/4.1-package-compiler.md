# 4.1 — Hero Section Package Compiler

**Skill:** LP-07-hero-section
**Position:** Layer 4, Microskill 1
**Purpose:** Assemble all validated hero section copy into a single structured package for downstream skills. Every subsequent Phase 3 skill and LP-15 Assembly will load this file.
**Effort Level:** low
**Dependencies:** All Layer 3 validation outputs, `HERO_SELECTION.yaml`, `hero-audit.md`
**Output:** `hero-section-package.json`

---

## Input Schema

```yaml
input:
  hero_selection: "[HERO_SELECTION.yaml — selected headline, deck, pre-head, lead direction]"
  lead_final: "[lead-final.md — revised lead]"
  typeb_abovefold: "[typeb-abovefold.md — if Type B or Hybrid]"
  loop_audit: "[loop-audit.md — loop count and status]"
  anti_slop_scan: "[anti-slop-scan.md — PASS confirmation]"
  hero_audit: "[hero-audit.md — final score]"
```

---

## Pre-Assembly Verification

Before assembling the package, verify:

```yaml
ASSEMBLY-GATE:
  hero_selection_yaml_exists: "[Y/N]"
  hero_audit_score: "[X.X — must be ≥7.5]"
  anti_slop_scan: "[PASS | FAIL — must be PASS]"
  loop_audit_type_a: "[PASS — X loops | N/A for Type B]"
  reading_level_check: "[PASS — Grade X | FAIL]"
  all_gates_pass: "[Y/N]"

  IF all_gates_pass = N: HALT — do not assemble. Return to failed validation.
```

## Quality Criteria
- All required fields populated (no nulls in required sections)
- threading_anchors all populated — these are critical for page coherence
- all_headline_candidates includes all candidates from generation (for reference)
- validation block shows all gates PASS
- Package readable by downstream skills without requiring context about LP-07's execution

## Failure Modes
- Selected headline text not matching any candidate in all_headline_candidates → Resolve discrepancy before assembly
- Threading anchor phrase not identifiable from selected headline → Extract it explicitly from the headline before assembly
