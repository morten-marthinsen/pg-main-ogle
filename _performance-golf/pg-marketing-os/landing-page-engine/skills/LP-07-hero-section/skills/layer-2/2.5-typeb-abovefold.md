## Product Title / H1 Generation

### Rules
1. MUST contain an explicit benefit — not just the product name
2. Can include the product name IF the name already carries brand recognition OR is mechanism-descriptive (e.g., "Magnesium Breakthrough" communicates something)
3. Length: 5–15 words
4. Reading level: 6th grade maximum
5. No forbidden words from anti-slop list
6. If supplement: include primary benefit category (sleep, energy, cognitive, immune, etc.)

### H1 Formula Options (generate 3, present all 3 for selection)

**Formula A:** `[Product Name] — [Primary Benefit] + [Secondary Benefit]`
Example: "SleepDeep Pro — Deeper Sleep, Less Stress & More Morning Energy"

**Formula B:** `[Primary Benefit Category] Support — [Brief Mechanism/Source Qualifier]`
Example: "Advanced Magnesium Support — 7 Bioavailable Forms for Total Absorption"

**Formula C:** `For [Audience Qualifier] — [Benefit Outcome Statement]`
Example: "For Women 40+ — Natural Hormone Balance & All-Day Energy Support"

**Formula D (if strong mechanism/brand name):** `[Named Product/Mechanism] — [Specific Benefit + Proof Signal]`
Example: "Magnesium Breakthrough — Clinically Studied for Deeper Sleep in 30 Days"

Generate one candidate per formula. Score on: specificity, benefit clarity, audience relevance. Present all for human selection.

---

## Short Description Rules

- Exactly 2–3 sentences
- Sentence 1: Core benefit claim — specific, credible (contains a number or mechanism reference if possible)
- Sentence 2: Differentiator — what makes this different from the 50 other options (mechanism hint, sourcing, formula innovation)
- Sentence 3 (optional): Risk removal — guarantee reference OR social proof signal ("Trusted by 47,000+ customers")

**Forbidden in short description:**
- Generic opener: "Introducing our revolutionary new..." → cut
- Vague claim: "Feel better every day" → make specific
- Feature-first: "Contains 500mg of ashwagandha..." → lead with benefit
- Passive voice: "Is designed to support..." → "Supports [benefit]"

**Generate 2 options:**
- Option A: Benefit-mechanism-proof structure
- Option B: Benefit-differentiator-social proof structure

---

## Key Benefits Bullets (4–6 items)

### Bullet Formula
`[Emoji/Icon] [Benefit verb] + [specific benefit] [optional: mechanism or timeframe hint]`

**Benefit verbs (rotate — don't start all bullets with "Supports"):**
Supports, Promotes, Helps restore, Delivers, Naturally boosts, Maintains, Enhances, Encourages, Provides, Improves

**Rules:**
- Start with benefit verb — NEVER the ingredient name
- Each bullet is a distinct benefit (no overlap)
- One bullet may include a specificity number: "Studied in [X] clinical trials" or "Works within [X] minutes"
- Mobile scan test: covers each bullet with thumb — can user understand the benefit in 3 seconds?
- Order: Best benefit first, second-best benefit last, middle bullets support

**Generate 6 bullets minimum. Cull to best 4–5 for presentation.**

## Trust Badge Copy

Generate specific copy for 3–5 trust badges based on LP-03 above-fold blueprint trust signal selection.

**Badge categories and copy templates:**

| Badge Type | Copy Options |
|-----------|-------------|
| Guarantee | "[X]-Day Money-Back Guarantee" / "[Brand name] Promise: [X] Days, No Questions" |
| Manufacturing | "GMP Certified Manufacturing" / "Made in FDA-Registered Facility" |
| Third-party testing | "Third-Party Tested for Purity" / "Independently Lab Verified" |
| Shipping | "Free Shipping on Orders $[X]+" / "Fast + Free Shipping $[X]+" |
| Security | "SSL Secured Checkout" / "256-Bit Secure Checkout" |
| Natural/Clean | "Non-GMO · Gluten-Free · No Artificial Fillers" |
| Clinical | "Clinically Studied Ingredients" / "Research-Backed Formula" |

**Rule:** Use ONLY badges that are TRUE for this product. Do not add GMP certified badge if product is not GMP certified. Verify against product brief.

## Rating Strip Context

The rating strip itself is data (★★★★★ 4.8, 2,847 reviews). The LP-07 task is to write any surrounding copy that contextualizes it:

**Before rating strip (optional):** "[X] people love [Product/Brand]" or "See why [X]+ customers rate us [X] stars"
**After rating strip (optional):** "See all [X] reviews ↓" or a 5-word summary: "Real results. Real customers."

## Quality Criteria
- H1 contains explicit benefit (not brand name alone)
- All 3 H1 options use different formulas
- Short description sentence 1 is specific (contains number, mechanism reference, or clinical qualifier)
- 6 benefit bullets generated, all starting with benefit verb
- Trust badges verified against actual product certifications
- Zero forbidden words in any element

## Failure Modes
- Product brief lacks enough detail to write specific short description → Write what's possible, mark unknown specifics as "[VERIFY: insert actual data here]"
- No certifications mentioned in brief → Use only generic badges (Secure Checkout, Free Shipping) that are always true
- Only product name available (no benefit info) → Flag: "Insufficient product information to write benefit-forward H1 — request primary benefit claim from client"
