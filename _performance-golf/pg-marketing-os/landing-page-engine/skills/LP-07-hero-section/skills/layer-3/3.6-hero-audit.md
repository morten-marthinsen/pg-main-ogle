# 3.6 — 10-Point Hero Section Audit

**Skill:** LP-07-hero-section
**Position:** Layer 3, Microskill 6
**Purpose:** Score the final hero section package against the 10-point audit defined in LP-07-AGENT.md. This is the quality gate that determines whether the hero section proceeds to Layer 4 packaging. Minimum score: 7.5/10. Each point is binary PASS (1) or FAIL (0) with evidence required.
**Effort Level:** high
**Dependencies:** `lead-final.md` (3.1), `HERO_SELECTION.yaml`, `loop-audit.md` (3.2), `self-selection-test.md` (3.3), `anti-slop-scan.md` (3.4), `reading-level.md` (3.5), `typeb-abovefold.md` (2.5 — if Type B)
**Output:** `hero-audit.md`

---

## Input Schema

```yaml
input:
  page_type: "[type_a | type_b | hybrid]"
  selected_headline: "[from HERO_SELECTION.yaml]"
  selected_deck: "[from HERO_SELECTION.yaml]"
  selected_pre_head: "[from HERO_SELECTION.yaml — or 'None']"
  lead_final: "[full text — Type A/Hybrid]"
  typeb_above_fold: "[product title, description, bullets, badges — Type B/Hybrid]"
  validation_results:
    loop_audit: "[PASS/FAIL + loop count — from 3.2]"
    self_selection_test: "[PASS/FAIL — from 3.3]"
    anti_slop_scan: "[PASS/FAIL — from 3.4]"
    reading_level: "[PASS/FAIL + grade — from 3.5]"
```

---

## Execution Logic

1. **Determine which audit criteria apply** based on page type:
   - Type A: Points 1-3 (Headline) + Points 4-6 (Lead — Type A) + Points 7-10 (Universal)
   - Type B: Points 1-3 (Headline) + Points 4-6 (Above-Fold — Type B) + Points 7-10 (Universal)
   - Hybrid: Audit both Type A lead AND Type B above-fold; use the lower score.

2. **Score each point** with evidence:

### Headline Quality (3 points)

**Point 1: Headline Specificity Score >= 7.0**
- Pull the specificity score from `headline-candidates.md` for the selected candidate
- Does it contain a specific number, name, timeframe, or claim?
- Evidence: Quote the specific element(s) in the headline

**Point 2: Headline Believability Score >= 7.0**
- Pull the believability score from `headline-candidates.md` for the selected candidate
- Would a skeptic refrain from immediately dismissing it?
- Evidence: State why a skeptic would find this credible

**Point 3: Headline Opens at Least One Curiosity Loop**
- Does the headline create a question the reader must scroll to answer?
- Evidence: State the specific question the headline creates

### Lead Quality — Type A (3 points) — Use for Type A and Hybrid

**Point 4: Lead Opens >= 3 Curiosity Loops by End of First 200 Words**
- Pull from `loop-audit.md` results
- Evidence: List the 3 loops by type

**Point 5: Lead Reading Level <= Grade 8**
- Pull from `reading-level.md` results
- Evidence: State the grade level

**Point 6: Lead Self-Selects Target Audience Within First 2 Sentences**
- Pull from `self-selection-test.md` results
- Evidence: Quote the self-selecting phrase(s)

### Above-Fold Quality — Type B (3 points) — Use for Type B and Hybrid

**Point 4B: Product Title is Benefit-Forward**
- Is the H1 more than just the brand/product name?
- Evidence: Quote the benefit stated in the title

**Point 5B: Short Description States a Specific Claim in First Sentence**
- Is the first sentence of the description specific (number, timeframe, named mechanism)?
- Evidence: Quote the specific claim

**Point 6B: Key Benefits Bullets are Outcome-Focused**
- Do bullets lead with outcomes, not ingredients or features?
- Evidence: Quote the verb that opens each bullet

### Universal (4 points)

**Point 7: Zero AI Telltales in Any Copy Element**
- Pull from `anti-slop-scan.md` results
- Evidence: State scan result

**Point 8: Hero Section Answers "What Is This?" in 5 Seconds**
- After scanning pre-head + headline + deck, does the reader know what the product/offer IS?
- Evidence: State what the reader would understand

**Point 9: Hero Section Answers "Who Is This For?" in 8 Seconds**
- After scanning the hero section elements, does the reader know if this is for THEM?
- Evidence: State the audience signal

**Point 10: Primary CTA Visible or Immediately Accessible After Hero Section**
- Is there a CTA button/link within or immediately following the hero copy?
- Note: LP-07 writes the copy, not the CTA itself — check if the copy structure supports immediate CTA placement
- Evidence: State where the CTA would appear relative to the hero copy

3. **Calculate total score.** Sum all PASS points.

4. **Determine gate result:**
   - Score >= 7.5 → GATE_3 PASS
   - Score 6.0-7.4 → GATE_3 FAIL — identify which points failed and provide specific fix guidance
   - Score < 6.0 → GATE_3 FAIL — major revision required, flag for human attention

## Quality Criteria
- All 10 points scored with evidence — no points scored without stated justification
- Evidence is specific (quotes, scores, test results) not general ("seems specific enough")
- Failed points include actionable fix guidance
- Upstream validation results incorporated (not re-evaluated — trust the dedicated checks)
- Gate result is binary: >= 7.5 PASS, < 7.5 FAIL

## Failure Modes
- Scoring a point PASS without evidence ("the headline is specific") → Provide the evidence. What SPECIFIC element makes it specific?
- Ignoring a failed upstream check (e.g., anti-slop FAIL but marking Point 7 as PASS) → Upstream FAIL = this point FAIL. No exceptions.
- Audit score of 7.4 rationalized as "close enough" → 7.4 < 7.5. FAIL. Revise the weakest point and re-audit.
- Hybrid page audited on only one type → Hybrid must pass BOTH Type A lead criteria AND Type B above-fold criteria. Use the lower subtotal.
