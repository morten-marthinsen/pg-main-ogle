# 3.3 — Self-Selection Test

**Skill:** LP-07-hero-section
**Position:** Layer 3, Microskill 3
**Purpose:** Verify that the right audience immediately recognizes "this is for me" within the first 2 sentences of the hero section, AND that the wrong audience recognizes "this isn't for me." Self-selection is the mechanism that converts browsers into readers. Without it, the headline attracts attention but doesn't convert it into readership.
**Effort Level:** medium
**Dependencies:** `lead-final.md` (3.1) OR `typeb-abovefold.md` (2.5), `HERO_SELECTION.yaml`, `packages-loaded.md` (audience profile)
**Output:** `self-selection-test.md`

---

## Input Schema

```yaml
input:
  page_type: "[type_a | type_b | hybrid]"
  selected_headline: "[from HERO_SELECTION.yaml]"
  selected_deck: "[from HERO_SELECTION.yaml]"
  selected_pre_head: "[from HERO_SELECTION.yaml — or 'None']"
  first_2_sentences:
    type_a: "[first 2 sentences of lead-final.md]"
    type_b: "[product title + first sentence of short description from typeb-abovefold.md]"
  audience_profile:
    target_demographics: "[age, gender, situation descriptors from packages-loaded.md]"
    primary_pain: "[the specific problem they have]"
    failed_solutions: "[what they've already tried]"
    primary_desire: "[the outcome they want]"
  anti_audience: "[who this is NOT for — extracted from audience profile or brief]"
```

---

## Execution Logic

1. **Define the "right reader" profile** in 2-3 sentences:
   - Who they are (demographics + situation)
   - What they're experiencing (primary pain)
   - What they've tried (failed solutions)
   - What they want (primary desire)

2. **Define the "wrong reader" profile** in 1-2 sentences:
   - Who would click but NOT be the target (e.g., younger demographic, different condition, professional researchers, competitors)

3. **Run Test A — Right Reader Recognition:**
   Read the pre-head (if present) + headline + deck + first 2 sentences of the lead (Type A) or product title + first sentence of description (Type B).

   **Ask:** Would the right reader, within 5 seconds of scanning these elements, think one of the following?
   - "This is exactly my situation"
   - "They're talking about me"
   - "Finally, someone gets it"
   - "This is what I've been looking for"

   **Evidence required:** Identify the SPECIFIC words or phrases that trigger self-selection. Quote them. Explain which aspect of the audience profile they match.

4. **Run Test B — Wrong Reader De-Selection:**
   Would the wrong reader, after scanning the same elements, recognize this is NOT for them?

   **Evidence required:** Identify which element(s) would cause the wrong reader to self-de-select. If no de-selection mechanism exists, flag it.

5. **Run Test C — Specificity Check:**
   Could a DIFFERENT product in the same vertical use these exact opening elements? If yes, the self-selection is too generic.

6. **Score the self-selection** on a 3-point scale:

   | Score | Meaning |
   |-------|---------|
   | **STRONG PASS** | Right reader recognized in 5 seconds. Wrong reader excluded. Product-specific. |
   | **PASS** | Right reader recognized. Wrong reader partially excluded. Mostly product-specific. |
   | **FAIL** | Right reader not clearly identified, OR wrong reader not excluded, OR elements are too generic. |

7. **If FAIL:** Provide specific fix recommendations — which element needs to change and what specific audience language should replace it.

## Quality Criteria
- Both right reader and wrong reader profiles explicitly defined before testing
- Self-selecting phrases identified by exact quote, not summary
- All three tests (recognition, de-selection, specificity) explicitly run
- FAIL result includes actionable fix recommendations, not just "needs improvement"
- Test is run on the SELECTED copy (post-HERO_SELECTION.yaml), not the draft candidates

## Failure Modes
- Test run on pre-selection draft copy instead of post-selection final copy → Re-run on lead-final.md / selected elements
- "Right reader would recognize this" stated without quoting the specific self-selecting phrase → Quote the phrase. Assertion without evidence is not a test.
- Wrong reader test skipped ("anyone interested would benefit") → The wrong reader test exists to catch overly broad targeting. Run it.
- FAIL issued but no fix guidance provided → Every FAIL must include at least one specific fix recommendation.
