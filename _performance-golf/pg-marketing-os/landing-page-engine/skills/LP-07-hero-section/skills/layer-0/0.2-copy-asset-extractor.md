# 0.2 — Copy Asset Extractor

**Skill:** LP-07-hero-section
**Position:** Layer 0, Microskill 2
**Purpose:** Pull headline territory, mechanism name, promise statement, FSSIT candidates, and audience pain quotes from LP-00 copy assets (CopywritingEngine downstream mode) or from the page brief (standalone mode). Every field is either populated with a real value or explicitly marked `not_available` — no silent gaps.
**Effort Level:** low
**Dependencies:** `packages-loaded.md` (0.1), LP-00 copy assets (if downstream mode)
**Output:** `copy-assets.md`

---

## Input Schema

```yaml
input:
  copywriting_engine_mode: "[downstream | standalone — from packages-loaded.md]"
  project_directory: "[path to active project output directory]"
  page_brief_fields:
    headline_territory: "[from page-brief.json if present]"
    mechanism_name: "[from page-brief.json if present]"
    promise_statement: "[from page-brief.json if present]"
    audience_pain_quotes: "[from page-brief.json if present]"
    fssit_candidates: "[from CopywritingEngine research if available]"
    root_cause_anchor: "[from CopywritingEngine if available]"
```

---

## Execution Logic

1. **Determine extraction mode** from `packages-loaded.md`:
   - **Downstream mode:** CopywritingEngine packages exist. Extract from CopywritingEngine output files (research package, Skill 03 big idea, Skill 04 mechanism, Skill 06 promise).
   - **Standalone mode:** No CopywritingEngine packages. Extract from `page-brief.json` fields only.

2. **Extract headline territory:**
   - Downstream: Pull from Skill 03 Big Idea output or LP-00 extracted territory.
   - Standalone: Pull from `page-brief.json` headline_direction field.
   - If neither exists: Mark `not_available`.

3. **Extract mechanism name:**
   - Downstream: Pull from Skill 04 Mechanism output (the named mechanism).
   - Standalone: Pull from `page-brief.json` if user provided mechanism name.
   - If neither exists: Mark `not_available`. Note: mechanism name is optional for headline generation but improves specificity.

4. **Extract promise statement:**
   - Downstream: Pull from Skill 06 Promise output.
   - Standalone: Pull from `page-brief.json` primary_benefit or core_promise field.
   - If neither exists: Mark `not_available`.

5. **Extract audience pain quotes:**
   - Downstream: Pull from CopywritingEngine Skill 01 research FINAL_HANDOFF.md — extract 5-10 verbatim quotes from target audience.
   - Standalone: Pull from `page-brief.json` audience_pain_language field.
   - If neither exists: Mark `not_available`. Flag: "No audience voice data — Lens 2 (Empath) generation will rely on inferred language."

6. **Extract FSSIT candidates:**
   - Downstream: Pull from CopywritingEngine Skill 05 FSSIT output if available.
   - Standalone: Mark `not_available`.
   - Note: FSSIT is a CopywritingEngine-specific framework. Absence does not block LP-07.

7. **Extract root cause anchor:**
   - Downstream: Pull from CopywritingEngine Skill 04 root cause output.
   - Standalone: Pull from `page-brief.json` root_cause field if present.
   - If neither exists: Mark `not_available`.

8. **Write `copy-assets.md`** with all fields populated or explicitly marked.

## Quality Criteria
- Every field explicitly populated or marked `not_available` — no silent omissions
- Downstream mode extracts from actual CopywritingEngine output files, not from memory
- Pain quotes are VERBATIM (not paraphrased) when sourced from research
- Asset coverage summary provides at-a-glance status for all 6 asset types
- Flags section alerts downstream microskills to any data gaps that affect generation quality

## Failure Modes
- Downstream mode selected but CopywritingEngine output files not found in project directory → Fall back to standalone extraction from page-brief.json; flag the discrepancy
- All assets marked `not_available` → Proceed but flag: "Generation will rely entirely on page-brief.json profile data. Quality ceiling may be lower."
- Pain quotes paraphrased instead of verbatim → Re-extract exact quotes. Paraphrasing defeats the purpose of Lens 2.
