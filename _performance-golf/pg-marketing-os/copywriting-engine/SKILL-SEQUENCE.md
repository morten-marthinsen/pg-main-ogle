# Copywriting Engine — Skill Sequence

**Version:** 2.0
**Purpose:** Define the sequential flow of copywriting skills from research to final copy

---

## Overview

The Copywriting Engine is organized into three phases that mirror the professional copywriting process:

1. **Phase I: Research & Strategy** — Gather intelligence, define proof, understand psychology, create mechanism, calibrate promise, develop Big Idea
2. **Phase II: Offer & Architecture** — Finalize the offer, outline the copy structure using CBP (Claim-Benefit-Proof) segments
3. **Phase III: Section Writing** — Write each section of the copy in narrative form

---

## Phase II: Offer & Architecture

After Phase I, you know: the market (research), the proof (inventory), the problem (root cause), the solution (mechanism), the promise, and the Big Idea. Now you finalize the offer and outline the copy.

```
┌─────────────────────────────────────────────────────────────────┐
│                          06-OFFER                               │
│  Finalize the offer stack — now you know:                       │
│  • Mechanism name → can name the core offer                     │
│  • Big Idea → can frame the offer angle                         │
│  • Supporting benefits → can design bonuses                     │
│  • Research → can set pricing, terms, guarantee                 │
│                                                                 │
│  Output: Core offer, bonus stack, pricing, guarantee,           │
│          scarcity/urgency elements, offer name                  │
└───────────────────────────────┬─────────────────────────────────┘
                                ↓
┌─────────────────────────────────────────────────────────────────┐
│                        07-STRUCTURE                             │
│  Outline the copy using CBP (Claim-Benefit-Proof) segments      │
│  from E5 methodology                                            │
│                                                                 │
│  Determines:                                                    │
│  • Where the lead goes and what type                            │
│  • Where story appears (origin, credibility)                    │
│  • Where root cause revelation happens                          │
│  • Where mechanism introduction occurs                          │
│  • Where product reveal happens                                 │
│  • Where offer section goes                                     │
│  • Where close/CTA happens                                      │
│                                                                 │
│  Output: Detailed outline with CBP segments mapped,             │
│          proof deployment plan, emotional arc                   │
└─────────────────────────────────────────────────────────────────┘
```

### Phase II Summary

| Skill | Input From | Primary Output | Key Decision |
|-------|-----------|----------------|--------------|
| 07-offer | Big Idea, Mechanism, Research | Complete offer stack | Pricing, bonuses, guarantee |
| 08-structure | All Phase I + Offer | CBP outline | Copy architecture |

### About CBP (Claim-Benefit-Proof)

From E5 methodology: Copy is built in CBP segments where:
- **Claim** — An assertion you're making
- **Benefit** — What it means for the prospect
- **Proof** — Evidence supporting the claim

The Structure skill maps out these segments and determines where each proof element gets deployed.

## Data Flow

### Proof Inventory → Downstream Skills

```yaml
proof_inventory_output:
  flows_to:
    05-promise:
      - promise_ceiling (what claims are supportable)
      - believability_constraints
      - specific_proof_for_claims

    06-big-idea:
      - knockout_proof (anchor for Big Idea)
      - proof_themes (patterns to exploit)

    08-structure:
      - position_ranked_proof (where to deploy each proof)
      - objection_proof_mapping (proof for each objection)
      - gradualization_sequence (proof ordering)

    10-13 (writing skills):
      - specific_proof_elements (for CBP segments)
```

### Mechanism → Downstream Skills

```yaml
mechanism_output:
  flows_to:
    05-promise:
      - mechanism_capabilities (what can be promised)
      - mechanism_name (for campaign thesis)

    06-big-idea:
      - unique_mechanism_angle
      - villain_defeat_action

    07-offer:
      - offer_name_candidate (from mechanism name)
      - core_deliverable_framing

    11-mechanism-narrative:
      - full_mechanism_package (for writing)
```

### Promise → Downstream Skills

```yaml
promise_output:
  flows_to:
    06-big-idea:
      - primary_promise (for synthesis)
      - campaign_thesis
      - promise_constraints

    08-structure:
      - promise_placement_recommendations

    08-lead:
      - promise_hooks (for attention)

    14-close:
      - promise_reinforcement
      - future_pacing_language
```

## Notes

- Skills do NOT need to be in numbered subfolders for phases. All skills live at the same level in `Skills/` folder.
- Phase designations are conceptual — they indicate the natural workflow, not folder structure.
- Some skills may run partially in parallel (e.g., proof inventory and root cause can be extracted from the same source pass).
- The Structure skill (07) is the critical bridge between strategy and writing.
