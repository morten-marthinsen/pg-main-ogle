# Skill 1.2: Saturation Mapper

**Version:** 1.0
**Layer:** 1 (Intelligence)
**Type:** Analysis
**Dependencies:** layer0-output.json (from 0.6)
**Output:** saturation_mapping.json

---

## Purpose

Map mechanism saturation across the competitive landscape — which mechanisms are overdone, which are safe, and which represent complete whitespace. Cross-references competitor usage (from Deep Research) against vault-proven performance (from swipes) to produce an actionable saturation map.

---

## The Question

**"Which mechanisms are too crowded to use, and which represent open territory with proven performance?"**

---

## Input Schema

```yaml
input:
  from_layer0_output:
    component_pools.mechanisms: [MechanismEntry]
    deep_research_summary.from_mechanism_analysis.competitor_mechanisms: [MechanismEntry]
    deep_research_summary.from_mechanism_analysis.mechanism_saturation: [object]
```

## Process Logic

### Step 1: Load Competitor Mechanisms

```
LOAD deep_research_summary.competitor_mechanisms

BUILD competitor_lookup:
  FOR each competitor_mechanism:
    KEY = normalized_mechanism_name (lowercase, strip articles)
    VALUE = competitor_count (integer)
```

### Step 2: Load Swipe Mechanisms

```
LOAD component_pools.mechanisms

BUILD swipe_lookup:
  FOR each mechanism:
    KEY = normalized_mechanism_name
    VALUE = {usage_count, average_mechanism_score, average_quality_score, swipe_sources}
```

### Step 3: Name Normalization & Matching

```
FOR each swipe_mechanism:
  ATTEMPT exact match against competitor_lookup
  IF no exact match:
    ATTEMPT fuzzy match:
      - Case-insensitive comparison
      - Strip articles (a, an, the)
      - Strip trailing modifiers (protocol, system, method, technique)
      - Compare root concept words

  RECORD match as:
    matched_competitor_name: string|null
    match_type: enum[exact, fuzzy, unmatched]
    competitor_count: integer (0 if unmatched)
```

### Step 4: Calculate Saturation Levels

```
FOR each mechanism (merged from both sources):

  IF competitor_count >= 10:
    saturation_level = "avoid"
    recommendation = "AVOID — [competitor_count] competitors already using this angle
                      despite [swipe_avg_score] quality score in [swipe_usage_count] swipes.
                      Market is oversaturated."

  ELSE IF competitor_count >= 5 AND < 10:
    saturation_level = "saturated"
    recommendation = "CAUTION — [competitor_count] competitors using this.
                      Would need strong differentiation despite [swipe_avg_score]
                      quality score in [swipe_usage_count] swipes."

  ELSE IF competitor_count >= 1 AND < 4:
    saturation_level = "safe"
    recommendation = "USE — Only [competitor_count] competitor(s), proven in
                      [swipe_usage_count] swipes at [swipe_avg_score] quality.
                      Room for execution."

  ELSE IF competitor_count == 0 AND swipe_usage_count >= 5:
    saturation_level = "whitespace"
    recommendation = "PRIME OPPORTUNITY — Zero competitors but proven in
                      [swipe_usage_count] swipes at [swipe_avg_score] quality.
                      Complete whitespace."

  ELSE IF competitor_count == 0 AND swipe_usage_count < 5:
    saturation_level = "safe"
    recommendation = "LOW DATA — Zero competitors but only [swipe_usage_count]
                      swipe examples. Promising but unproven volume."
```

### Step 5: Sort Within Categories

```
FOR each saturation category:
  SORT by swipe_avg_score DESC (highest quality first)
  THEN by swipe_usage_count DESC (most evidence second)
```

---

## Constraints

- NEVER use vague qualifiers (many, often, most, some, several, usually, typically, around, approximately)
- NEVER estimate competitor counts — use exact integers from Deep Research data
- NEVER classify a mechanism without checking both competitor AND swipe data
- ALWAYS cite exact competitor_count in every recommendation string
- ALWAYS cite exact swipe_usage_count and swipe_avg_score in recommendations
- ALWAYS include minimum 3 swipe_sources for "safe" and "whitespace" entries
- EVERY mechanism must be assigned to exactly one saturation level (no orphans)

---

## Quality Criteria

1. All competitor counts are exact integers from deep_research_summary
2. All swipe counts/scores are exact values from component_pools
3. Every mechanism categorized into exactly one saturation level
4. Whitespace opportunities cite 3+ swipe source IDs
5. Recommendations contain only quantified claims (zero vague language)
6. summary_counts match actual array lengths
7. No mechanisms dropped during name matching (all accounted for)
