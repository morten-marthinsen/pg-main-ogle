# 3.3 Proof Availability Score

## Purpose
Score the AVAILABILITY and STRENGTH of proof assets for the expressed root cause. A root cause without proof creates skepticism. This validation ensures we can back up our claims.

---

## The Proof Principle

**Every root cause claim must be PROVABLE.**

The expressed root cause needs proof that it:
1. Exists (the problem is real)
2. Affects the audience (they have it)
3. Causes their symptoms (connection is real)
4. Is fixable (mechanism works)

## Quality Gates

Before proceeding:

- [ ] Total proof score is 6.0+ (minimum for credible copy)
- [ ] At least 3 of 4 requirements have proof coverage
- [ ] No critical gaps without mitigation strategy
- [ ] At least 2 different proof types available

---

## Threshold Guidelines

| Score | Status | Action |
|-------|--------|--------|
| 8.0+ | Strong | Rich proof available—leverage fully |
| 6.0-7.9 | Adequate | Usable proof—be strategic |
| 4.0-5.9 | Weak | Gaps present—soften claims or find more proof |
| Below 4.0 | Insufficient | Root cause unprovable—revisit |

---

Pass Proof Availability Report to:

- **If insufficient**: Return to derivation or gather more proof assets
