# 1.2 Symptom Convergence Analysis

## Purpose
When multiple surface symptoms cluster around the same failed solutions, trace them back to a SINGLE root cause that explains all of them. This derivation method produces root causes that feel revelatory—"That's why EVERYTHING hasn't worked!"

---

## Input Requirements

From 1.1 Research Pattern Analysis:
- Complaint clusters with common threads
- Failed solution patterns with shared flaws
- Symptom groupings

---

## The Convergence Principle

**Multiple symptoms → One cause → Total explanation**

The power of symptom convergence: Instead of treating each problem separately, you reveal they're ALL manifestations of the SAME underlying issue.

```
BEFORE (Audience's View):
├── Problem A → Solution A (fails)
├── Problem B → Solution B (fails)
├── Problem C → Solution C (fails)
└── Conclusion: "I have multiple problems"

AFTER (Converged View):
├── Problem A ─┐
├── Problem B ─┼──→ ROOT CAUSE X → One Solution
├── Problem C ─┘
└── Conclusion: "I had ONE problem all along"
```

---

## Execution Protocol

### Step 1: List All Surface Symptoms

From research, enumerate every distinct problem the audience reports:

```
SYMPTOM INVENTORY:
1. [Symptom]: "[Exact complaint language]"
2. [Symptom]: "[Exact complaint language]"
3. [Symptom]: "[Exact complaint language]"
...
```

**Golf Example** (from FIXD vault data):
1. Slice it right
2. Pull hook it left
3. Miss irons fat
4. Miss irons thin
5. Can't repeat swing

**Health Example** (from Resurge vault data):
1. Weight gain
2. Low energy
3. Brain fog
4. Premature aging
5. Stubborn belly fat

---

### Step 2: Find the Common Denominator

Ask: **"What single dysfunction could produce ALL of these symptoms?"**

| Symptom | What Would Have to Be Wrong for This to Happen? |
|---------|-----------------------------------------------|
| [Symptom 1] | [Underlying dysfunction] |
| [Symptom 2] | [Underlying dysfunction] |
| [Symptom 3] | [Underlying dysfunction] |

**Convergence Test**: If the same underlying dysfunction appears across multiple symptoms, you've found a candidate root cause.

**FIXD Example**:
| Symptom | Underlying Dysfunction |
|---------|----------------------|
| Slice right | Timing thrown off in swing |
| Pull hook left | Over-compensating for timing |
| Fat/thin shots | Inconsistent contact point |
| Can't repeat | No reliable swing sequence |

**Convergence**: All trace back to "off-track first move in backswing"

---

### Step 3: Validate the Convergence

Test the candidate root cause against ALL symptoms:

```
CONVERGENCE VALIDATION:
Root Cause Candidate: "[The single cause]"

Symptom 1: [Name]
├── Does root cause explain this? YES/NO
├── How? [Explanation]
└── Alternative explanation needed? YES/NO

Symptom 2: [Name]
├── Does root cause explain this? YES/NO
├── How? [Explanation]
└── Alternative explanation needed? YES/NO

[Continue for all symptoms]

COVERAGE SCORE: X/Y symptoms explained (target: 80%+)
```

**Quality Signal**: A true root cause should explain 80%+ of symptoms WITHOUT requiring convoluted logic.

---

### Step 4: Identify the Mechanism of Action

Document HOW the root cause produces each symptom:

```
ROOT CAUSE → SYMPTOM PATHWAY:

[Root Cause]
    │
    ├──→ [Mechanism 1] ──→ Symptom A
    │
    ├──→ [Mechanism 2] ──→ Symptom B
    │
    └──→ [Mechanism 3] ──→ Symptom C
```

**SSTS Example**:
```
"Ball at bottom of swing" misconception
    │
    ├──→ Decelerate clubhead ──→ Poor contact (fat/thin)
    │
    ├──→ Can't find low point ──→ Inconsistent results
    │
    └──→ Try harder ──→ More tension ──→ Worse performance
```

---

### Step 5: Quantify the Impact

Estimate how much of the total problem the root cause explains:

```
IMPACT ASSESSMENT:
Root Cause: "[Name]"

Total symptom count: [X]
Symptoms fully explained: [Y]
Symptoms partially explained: [Z]
Unexplained symptoms: [W]

Coverage percentage: [(Y + 0.5*Z) / X * 100]%

Impact statement: "This ONE root cause accounts for [X]% of your [problem category]"
```

**Vault Benchmark**: Top-performing swipes typically claim 65-100% of problems trace to single root cause.

## Anti-Patterns

**Weak Convergence** (Avoid):
- Forcing symptoms that don't fit
- Vague root causes that could mean anything ("poor health habits")
- Requiring 5-step explanations for each symptom pathway

**Strong Convergence** (Target):
- Simple, direct pathways from root to symptom
- Root cause is specific and concrete
- Audience immediately sees how symptoms connect
