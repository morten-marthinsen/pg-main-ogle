# Layer 0.2.5 — Specimen Decomposer

**Version:** 1.0
**Layer:** 0 (Foundation)
**Type:** Specimen Analysis
**Domain:** Product Introduction
**Dependencies:** vault_product-introduction_intelligence (from 0.2-vault-intelligence-loader.md)
**Output:** specimen_decomposition (structural templates, technique catalog, power elements)

---

## Purpose

Decompose top-scoring product introduction specimens from the vault-intelligence into reusable structural templates, technique catalogs, and power elements. Product introduction specimens map how elite campaigns reveal the product — the bridge from mechanism to offer, the introduction method, positioning, and the full offer structure embedded in narrative. This skill extracts the introduction architecture.

**This skill bridges the gap between:** vault data (product introduction maps with bridge language and offer structures) and generation layers (what Claude needs to construct seamless mechanism-to-product transitions that feel like natural revelation, not a sales pitch).

## Decomposition Protocol

### Phase 1: Specimen Selection & Tiering

```
IF tier_metadata present (Phase B loader active):
  LOAD gold_specimens → full decomposition
  LOAD silver_specimens → template-only decomposition
ELSE (pre-upgrade fallback):
  SELECT top 5 by position as pseudo-Gold
  SELECT next 10 as pseudo-Silver
  LOG "Operating in pre-upgrade mode — tiering estimated from array position"

VERIFY each specimen has:
  - product_name (required)
  - mechanism_name (required)
  - introduction_method (required)
  - bridge_from_mechanism (required — skip if absent)
  - section_position (preferred)
  - offer_structure{} (preferred)
  - source_file (required for provenance)
  - niche (required for type-matching)
  - format (preferred)
```

### Phase 2: Structural Template Extraction

For each Gold specimen:

```
ANALYZE bridge_from_mechanism and offer_structure to extract:

  introduction_skeleton:
    - element_type: [mechanism_recap | bridge_statement | product_reveal |
                     unique_mechanism_claim | positioning_statement |
                     value_preview | credibility_reinforcement |
                     urgency_setup]
    - function: string
    - position: integer
    - proportion: float

  bridge_architecture:
    introduction_method: string (from specimen — system_reveal, tool_reveal,
                                  program_reveal, discovery_reveal,
                                  authority_reveal, etc.)
    bridge_strategy: string (how mechanism connects to product —
                             natural_extension, only_solution,
                             exclusive_access, logical_conclusion)
    positioning: string (from offer_structure — system, tool, program, etc.)
    section_position: string (where in campaign this appears)
    section_position_number: integer

  offer_integration:
    offer_flow: string (continuity, one_time, hybrid)
    unique_mechanism_claim: string
    value_stack_preview: string (how value is first introduced)
    guarantee_preview: string (if guarantee is previewed during introduction)

For Silver specimens:
  Extract bridge_architecture ONLY
  SKIP element-level skeleton and offer_integration detail
```

### Phase 3: Technique Catalog Compilation

Aggregate across ALL decomposed specimens:

```
introduction_methods:
  FOR each unique introduction_method:
    - method_name: string (system_reveal, tool_reveal, program_reveal,
                            discovery_reveal, authority_reveal, etc.)
    - count: integer
    - implementation_pattern: string (how to execute this introduction style)
    - gold_example: string (1-2 sentence bridge excerpt)
    - best_for: string (product type / niche combination)
    - specimens_using: [source_file references]

bridge_strategies:
  FOR each unique bridge construction approach:
    - strategy_name: string
    - count: integer
    - mechanism_to_product_logic: string (the logical connection)
    - implementation_pattern: string
    - gold_example: string

positioning_techniques:
  FOR each unique product positioning approach:
    - positioning_type: string (system, tool, program, protocol, formula, etc.)
    - count: integer
    - best_for: string (product type)
    - specimens_using: [source_file references]

section_placement:
  FOR each observed section_position pattern:
    - position: string (early_middle, late_middle, late, etc.)
    - count: integer
    - preceding_section_function: string (what comes before introduction)
    - following_section_function: string (what comes after)
    - format_association: string
```

### Phase 4: Power Element Extraction

From Gold specimens ONLY:

```
bridge_statements:
  FOR each Gold specimen:
    - verbatim_bridge: string (the bridge_from_mechanism text)
    - introduction_method: string
    - bridge_strategy: string
    - mechanism_referenced: string
    - product_revealed: string
    - source_file: string

unique_mechanism_claims:
  FOR each Gold specimen with unique_mechanism_claim:
    - verbatim_claim: string
    - positioning: string
    - source_file: string

value_preview_language:
  FOR each Gold specimen where value is previewed during introduction:
    - verbatim_preview: string
    - preview_technique: string (implied, stated, compared)
    - source_file: string
```

## Quality Gates

- [ ] Minimum 3 Gold specimens fully decomposed (skeleton + bridge + offer integration)
- [ ] Minimum 5 Silver specimens template-decomposed
- [ ] All introduction skeletons have complete element sequences
- [ ] Technique catalog has minimum 3 introduction methods cataloged
- [ ] Technique catalog has minimum 2 bridge strategies documented
- [ ] Power elements contain minimum 3 verbatim bridge statements from Gold specimens
- [ ] Every power element includes source provenance
- [ ] No raw bridge_from_mechanism text passed through undecomposed

---

## Anti-Patterns

| Anti-Pattern | Signal | Fix |
|---|---|---|
| **Bridge Without Strategy** | Bridge text listed without analyzing the logical connection | Every bridge needs bridge_strategy |
| **Method Without Pattern** | Introduction methods named without implementation guidance | Every method needs implementation_pattern |
| **Position Ignored** | Introduction analyzed without campaign position context | section_placement critical — timing of reveal matters |
| **Offer Structure Flat** | Offer structure passed through without integration analysis | Analyze how offer is woven into introduction narrative |
| **Mechanism Disconnected** | Product introduced without referencing mechanism | Bridge MUST connect mechanism → product |
