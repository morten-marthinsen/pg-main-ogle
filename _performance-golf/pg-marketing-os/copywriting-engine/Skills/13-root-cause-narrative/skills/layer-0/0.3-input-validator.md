# Skill 13.0.3: Input Validator

**Version:** 1.0
**Skill:** 10 — Root Cause Narrative
**Layer:** 0 (Foundation & Loading)
**Position:** Third microskill — validates all loaded inputs before human checkpoint
**Type:** Validation & Quality Gate
**Dependencies:** upstream_data.json (from 0.1), tier1_intelligence_index.json (from 0.2)
**Output:** validation_report.json

---

## PURPOSE

Validate all loaded inputs from the upstream loader (0.1) and TIER1 intelligence loader (0.2) against schema requirements and minimum quality thresholds. Determine whether the data is sufficient to generate a compelling root cause narrative, flag specific gaps that need human resolution, and set the default response for the human checkpoint (CONFIRM if all data is present, MODIFY if gaps exist).

---

## IDENTITY

**This microskill IS:**
- A schema enforcer that checks every required field against its expected type and content
- A quality gate that determines readiness for narrative generation
- A gap identifier that produces specific, actionable descriptions of what is missing
- A checkpoint default setter that informs the human checkpoint of data status

**This microskill is NOT:**
- A data loader — loading is complete before this microskill runs (0.1 and 0.2 handle loading)
- A data fixer — it identifies gaps but does not fill them
- A human-facing component — that is 0.4-human-checkpoint-curator
- A narrative quality judge — it validates INPUT quality, not OUTPUT quality

---

## INPUT CONTRACT

```yaml
input:
  from_0_1_upstream_loader:
    upstream_data:
      root_cause: object  # three-part structure, villain, countersells, etc.
      structure: object   # campaign thesis, CPB chunks, flow architecture
      proof: object       # proof elements, credential stack, evidence types
      niche_context: object  # niche, sub-niche, sophistication
      loading_status: object  # completeness percentage, gaps, warnings

  from_0_2_tier1_intelligence:
    tier1_intelligence_index:
      metadata: object
      index: object  # all 12 indexed sections
      loading_status: object  # sections found, gaps, warnings
```

## PROCESS

### Step 1: Validate Three-Part Structure Completeness

```
CHECK three_part_structure:

  VALIDATE what_they_think:
    - Field exists and is non-empty string: PASS/FAIL
    - Length >= 20 characters (must be substantive): PASS/FAIL
    - Contains prospect-facing language (not internal jargon): PASS/FAIL
    - Describes a BELIEF the prospect currently holds: PASS/FAIL
    SCORE: 0-4 checks passed

  VALIDATE what_it_really_is:
    - Field exists and is non-empty string: PASS/FAIL
    - Length >= 20 characters: PASS/FAIL
    - Is MORE SPECIFIC than what_they_think (Rule 3): PASS/FAIL
    - Is DIFFERENT from what_they_think (not a restatement): PASS/FAIL
    - Contains a specific cause, agent, or process: PASS/FAIL
    SCORE: 0-5 checks passed

  VALIDATE why_nothing_worked:
    - Field exists and is non-empty string: PASS/FAIL
    - Length >= 20 characters: PASS/FAIL
    - References at least one failed approach: PASS/FAIL
    - Connects failure to the false belief, not to reader incompetence: PASS/FAIL
    SCORE: 0-4 checks passed

  THREE_PART_SCORE = (what_they_think_score + what_it_really_is_score +
                      why_nothing_worked_score) / 13 * 100

  IF THREE_PART_SCORE < 70%:
    ADD to critical_gaps: "Three-part structure incomplete or weak"
    ADD specific failures to gap_details
  IF THREE_PART_SCORE >= 70% AND < 85%:
    ADD to warnings: "Three-part structure has minor gaps"
```

### Step 2: Validate Villain Identification

```
CHECK villain:

  VALIDATE villain.name:
    - Field exists and is non-empty: PASS/FAIL
    - Is specific (not just "the system" or "them"): PASS/FAIL
    - Names an identifiable entity, group, or process: PASS/FAIL
    SCORE: 0-3

  VALIDATE villain.type:
    - Field exists: PASS/FAIL
    - Matches known taxonomy: PASS/FAIL
      [pharma, medical_establishment, government, government_conspiracy,
       food_industry, misinformation, biological_process, industry]
    SCORE: 0-2

  VALIDATE villain.role:
    - Field exists: PASS/FAIL
    - Matches known taxonomy: PASS/FAIL
      [suppressor, profiteer, deceiver, blocker, betrayer]
    SCORE: 0-2

  VILLAIN_SCORE = (name_score + type_score + role_score) / 7 * 100

  IF VILLAIN_SCORE < 60%:
    ADD to critical_gaps: "Villain identification incomplete"
    NOTE: "Villain deep dive phase (Phase 6) cannot be drafted without villain data"
  IF villain.name is generic ("the system", "big pharma" without specifics):
    ADD to warnings: "Villain name is generic — specificity needed for compelling narrative"
```

### Step 3: Validate Expression Method and Reframe Technique

```
CHECK expression_method:
  - Field exists and is non-empty: PASS/FAIL
  - Value is descriptive enough to guide communication type classification: PASS/FAIL

  IF missing:
    ADD to gaps: "expression_method missing — communication type classifier will use
                  niche defaults from TIER1 cross-vertical patterns"

CHECK reframe_technique:
  - Field exists and is non-empty: PASS/FAIL
  - Matches known taxonomy: PASS/FAIL
    [not_your_fault, conspiracy, hidden_truth, opposite_of_truth,
     suppressed_science, conspiracy_expose]

  IF missing:
    ADD to critical_gaps: "reframe_technique missing — reframe stacking optimizer
                           cannot function without primary reframe"
  IF present but does not match taxonomy:
    ADD to warnings: "reframe_technique value does not match known taxonomy —
                      may need mapping at human checkpoint"
```

### Step 4: Validate Countersells

```
CHECK countersells:
  - Array exists and is non-empty: PASS/FAIL
  - At least 2 countersells present: PASS/FAIL (minimum requirement)
  - Each countersell has target, pattern, and argument: PASS/FAIL

  FOR each countersell:
    VALIDATE target: non-empty, names a specific competing solution
    VALIDATE pattern: matches one of [treats_symptoms, makes_worse,
                                       wrong_enemy, dangerous]
    VALIDATE argument: non-empty, connects back to root cause

  COUNTERSELL_SCORE = (basic_checks + detail_checks) / total_possible * 100

  IF countersells.length < 2:
    ADD to gaps: "Fewer than 2 countersells defined — minimum is 2"
  IF countersells.length == 0:
    ADD to critical_gaps: "No countersells defined — countersell bridge writer
                           will need to generate from scratch using root cause data"
```

### Step 5: Validate Copy Blocks

```
CHECK copy_blocks:

  FOR each block in [false_belief_statement, root_cause_reveal,
                     failure_explanation, villain_narrative]:
    - Field exists: PASS/FAIL
    - Content is non-empty: PASS/FAIL
    - Content length >= 30 characters: PASS/FAIL

  COPY_BLOCK_SCORE = blocks_present / 4 * 100

  IF COPY_BLOCK_SCORE < 50%:
    ADD to warnings: "Copy blocks sparse — Layer 2 writers will generate
                      more from strategy data"
  NOTE: Copy blocks are HELPFUL but not CRITICAL — Layer 2 writers can
        generate from the three-part structure and other data if blocks
        are missing. They serve as starting points, not final text.
```

### Step 6: Validate Structure Alignment

```
CHECK structure package data:

  VALIDATE campaign_thesis:
    - Present and non-empty: PASS/FAIL
    - Relates to the root cause three-part structure: PASS/FAIL (semantic check)

  VALIDATE relevant_cpb_chunks:
    - At least 1 chunk identified for root cause: PASS/FAIL
    - Chunks have clear content summaries: PASS/FAIL

  VALIDATE flow_architecture:
    - Section sequence available: PASS/FAIL
    - Root cause position identified in sequence: PASS/FAIL

  VALIDATE word_count_budget:
    - Budget available for root cause section: PASS/FAIL
    - Budget is realistic (500-5000 words depending on format): PASS/FAIL

  VALIDATE format_type:
    - Type identified: PASS/FAIL
    - Type matches known formats: PASS/FAIL

  STRUCTURE_SCORE = checks_passed / total_checks * 100

  IF structure package was not loaded (from 0.1 loading_status):
    SET STRUCTURE_SCORE = 0
    ADD to warnings: "Structure package not available — narrative will use
                      default sequence and word count allocations"
```

### Step 7: Validate Proof Availability

```
CHECK proof inventory data:

  VALIDATE proof_elements:
    - At least 1 HIGH relevance element: PASS/FAIL
    - At least 3 total elements: PASS/FAIL

  VALIDATE credential_stack:
    - Non-empty: PASS/FAIL
    - At least 2 credentials: PASS/FAIL

  VALIDATE available_evidence_types:
    - At least 2 evidence types available: PASS/FAIL
    - At least 1 of [person_as_proof, institutional_name_drop]: PASS/FAIL
      (needed for authority bridge)

  PROOF_SCORE = checks_passed / total_checks * 100

  IF proof inventory was not loaded:
    SET PROOF_SCORE = 0
    ADD to warnings: "Proof inventory not available — evidence integration
                      calibrator will be severely limited"
  IF credential_stack is empty:
    ADD to gaps: "No credentials available — authority bridge (Phase 3)
                  will be weak"
```

### Step 8: Validate TIER1 Intelligence Completeness

```
CHECK tier1_intelligence_index:

  VALIDATE sections_found:
    - All 12 sections indexed: PASS/FAIL
    - At least 10 sections indexed: PASS/FAIL (minimum for operation)

  VALIDATE critical sections present:
    - communication_types indexed: PASS/FAIL (needed for 1.1)
    - framing_patterns indexed: PASS/FAIL (needed for 1.2)
    - emotional_arcs indexed: PASS/FAIL (needed for 1.3)
    - universal_sequence indexed: PASS/FAIL (needed for 1.4)
    - evidence_patterns indexed: PASS/FAIL (needed for 3.1)
    - reframe_taxonomy indexed: PASS/FAIL (needed for 3.2)
    - transition_patterns indexed: PASS/FAIL (needed for 3.3)
    - critical_rules indexed: PASS/FAIL (needed for 4.1)

  TIER1_SCORE = checks_passed / total_checks * 100

  IF TIER1_SCORE < 80%:
    ADD to critical_gaps: "TIER1 intelligence incomplete — classification and
                           validation reliability compromised"
  IF any critical section missing:
    ADD to gaps: "TIER1 section [name] missing — [downstream microskill]
                  cannot execute properly"
```

### Step 9: Calculate Overall Completeness Score

```
CALCULATE overall_completeness:

  WEIGHTS:
    three_part_structure: 25%
    villain: 15%
    expression_reframe: 10%
    countersells: 10%
    copy_blocks: 5%
    structure: 10%
    proof: 10%
    tier1_intelligence: 15%

  overall_completeness =
    (THREE_PART_SCORE * 0.25) +
    (VILLAIN_SCORE * 0.15) +
    (expression_reframe_score * 0.10) +
    (COUNTERSELL_SCORE * 0.10) +
    (COPY_BLOCK_SCORE * 0.05) +
    (STRUCTURE_SCORE * 0.10) +
    (PROOF_SCORE * 0.10) +
    (TIER1_SCORE * 0.15)

  THRESHOLD: 85%

  IF overall_completeness >= 85%:
    SET checkpoint_default = "CONFIRM"
    SET validation_status = "PASS"
    LOG: "Input validation PASSED at [percentage]%. Default: CONFIRM."

  IF overall_completeness >= 60% AND < 85%:
    SET checkpoint_default = "MODIFY"
    SET validation_status = "PASS_WITH_GAPS"
    LOG: "Input validation PASSED WITH GAPS at [percentage]%. Default: MODIFY.
          Gaps: [list of gaps]"

  IF overall_completeness < 60%:
    SET checkpoint_default = "PROVIDE"
    SET validation_status = "FAIL"
    LOG: "Input validation FAILED at [percentage]%. Default: PROVIDE.
          Critical gaps require human data entry before proceeding."
```

### Step 10: Compile Validation Report

```
COMPILE validation_report:
  overall_completeness: float
  validation_status: enum[PASS, PASS_WITH_GAPS, FAIL]
  checkpoint_default: enum[CONFIRM, MODIFY, PROVIDE]

  section_scores:
    three_part_structure: float
    villain: float
    expression_reframe: float
    countersells: float
    copy_blocks: float
    structure: float
    proof: float
    tier1_intelligence: float

  critical_gaps: [string]  # must be resolved before proceeding
  gaps: [string]           # should be resolved, can work around
  warnings: [string]       # noted for downstream awareness

  specific_gap_details:
    - field: string
      issue: string
      impact: string  # which downstream microskill is affected
      resolution: string  # what data is needed

  readiness_assessment:
    layer_1_ready: boolean  # classification can proceed
    layer_2_ready: boolean  # draft generation has enough data
    layer_3_ready: boolean  # refinement tools have what they need
    layer_4_ready: boolean  # validation has all criteria

WRITE validation_report to: layer-0-outputs/validation_report.json
HANDOFF to: 0.4-human-checkpoint-curator
```

## QUALITY CRITERIA

1. **85% minimum threshold enforced** — completeness must reach 85% for CONFIRM default; below that, gaps must be surfaced to human operator
2. **Every critical gap actionable** — each gap must describe what is missing, which downstream skill is affected, and what data is needed to resolve
3. **Three-part structure validated as a unit** — all three parts must be present AND substantive (not just non-empty)
4. **Villain specificity checked** — generic villain names ("the system") must be flagged; Rule 5 requires specific identification
5. **Reframe technique validated against taxonomy** — must match known values or be flagged for mapping
6. **TIER1 intelligence completeness gates downstream** — if critical TIER1 sections are missing, the corresponding downstream microskills must be flagged as not ready
7. **Checkpoint default reflects reality** — CONFIRM only when data is genuinely sufficient; MODIFY when workable gaps exist; PROVIDE when data is critically insufficient

---

## FAILURE MODES

| Failure | Severity | Detection | Handling |
|---------|----------|-----------|----------|
| Three-part structure missing entirely | CRITICAL | All three parts empty/null | Set checkpoint default to PROVIDE; human must supply root cause data |
| Villain generic or missing | HIGH | Name is vague or fields empty | Flag in gaps; villain deep dive will be weak or impossible |
| Reframe technique not in taxonomy | MODERATE | Value present but unrecognized | Flag for mapping at human checkpoint; may be a custom technique |
| TIER1 intelligence severely incomplete | CRITICAL | Fewer than 10 of 12 sections | Flag critical warning; classification reliability compromised |
| No proof inventory available | MODERATE | Proof score = 0 | Continue; evidence integration will be limited; flag in warnings |
| No structure package available | MODERATE | Structure score = 0 | Continue with defaults; alignment will be approximate; flag in warnings |
| Overall completeness below 60% | CRITICAL | Calculation result | Set FAIL status; human must provide missing data before proceeding |
| False positive on specificity check | LOW | what_it_really_is appears specific but is actually vague | Caught at human checkpoint or Layer 4 rules check |
