## Execution Protocol

### Step 1: Scan Extraction Index

Load TIER1 extraction files prioritizing:
- `metadata.quality_score` ≥ 85
- `metadata.extraction_confidence` ≥ 85
- Diverse niche representation (health, wealth, personal development)

### Step 2: Extract Lead Type Taxonomy

From each qualifying extraction, classify the lead type:

| Lead Type | Signature | Opening Device | Best For |
|-----------|-----------|---------------|----------|
| **Statistical** | Performance data cascade, ascending numbers | shocking_stat | Wealth, investment, quantifiable results |
| **Story Vulnerability** | Personal shame/failure → recovery arc | pattern_interrupt | Any niche where protagonist mirrors prospect |
| **Authority-Flex** | Credential stacking before radical claim | authority_flex | Health, wealth — when credentials are exceptional |
| **Compressed Discovery** | Mechanism naming + social proof, very tight | doctor_shock | Health supplements, scientific mechanisms |
| **Conspiracy Expose** | Named villains + specific evidence immediately | threat_frame | Financial, political, health cover-ups |
| **Prophecy Warning** | Data deconstruction + urgency signaling | rhetorical_question | Financial crisis, health warnings |
| **Doctor Discovery** | Medical authority + shocking finding + mechanism | doctor_shock | Health, supplements, medical breakthroughs |
| **Problem-Solution** | Pain point cascade → promise → mechanism tease | problem_callout | Universal — works across niches |
| **Hybrid** | Combines 2+ types (e.g., authority + problem-solution) | varies | Sophisticated markets (Stage 4-5) |

### Step 3: Extract Hook Patterns

Catalog opening hook sentences from elite controls:
- Hook text
- Word count (typically 5-15 words)
- Device type
- Attention mechanism (curiosity gap, shock, identification, fear)
- Niche context

### Step 4: Extract Emotional Progression Patterns

Map the emotional arcs documented across leads:
- `curiosity → hope` (most common across all niches)
- `shame → dignity` (vulnerability-driven leads)
- `confusion → clarity` (discovery-driven leads)
- `concern → urgency` (prophecy/warning leads)
- `concern → certainty` (authority-driven leads)
- `frustration → possibility` (problem-solution leads)

### Step 5: Extract Lead-to-Body Transition Patterns

How elite controls move from lead to the next section:
- Attention lock phrases
- Transition sentence structures
- Where the lead ends and credibility/intro begins

### Step 7: Specimen Tiering

```
FOR each lead specimen in loaded data:

  CALCULATE lead_completeness_score:
    +2 IF full_text present AND length > 50 words
    +1 IF hook_sentence present AND length > 5 words
    +1 IF opening_device classified (shocking_stat, pattern_interrupt, authority_flex, etc.)
    +1 IF type classified (statistical, story_vulnerability, authority_flex, etc.)
    +1 IF emotional_progression documented
    +1 IF word_count tagged
    +1 IF complete lead architecture (full_text + hook + type + device all present)
    MAX_SCORE: 8

  SORT specimens by lead_completeness_score DESC
  ASSIGN tier:
    Gold = top 10% (lead_completeness_score >= 7)
    Silver = next 15% (lead_completeness_score >= 5)
    Standard = remainder

  TAG each specimen with:
    tier: enum[gold, silver, standard]
    lead_completeness_score: integer (0-8)

  OUTPUT tier_distribution:
    total_specimens: integer
    gold_count: integer
    silver_count: integer
    standard_count: integer
```

### Step 8: Type-Matched Retrieval

```
DEFINE match_fields:
  primary: type (statistical, story_vulnerability, authority_flex, compressed_discovery,
                 conspiracy_expose, prophecy_warning, doctor_discovery, problem_solution, hybrid)
  secondary: opening_device (shocking_stat, pattern_interrupt, authority_flex, doctor_shock,
                             threat_frame, rhetorical_question, problem_callout, curiosity_gap, future_pace)
  tertiary: niche (health, wealth, personal_development, relationships, other)

WHEN campaign_context is available:
  STRATEGY 1 — exact_match:
    FILTER specimens WHERE type == target AND opening_device == target AND niche == target_niche
    PRIORITIZE Gold tier first, then Silver

  STRATEGY 2 — type_device_match:
    FILTER specimens WHERE type == target AND opening_device == target (any niche)
    PRIORITIZE Gold tier, same-niche Silver, then cross-niche Silver

  STRATEGY 3 — cross_type:
    RETURN all Gold specimens across lead types
    INCLUDE Silver specimens matching target niche or opening_device

WHEN campaign_context is NOT available:
  RETURN full specimen set with tier metadata attached
  SORT by lead_completeness_score DESC

OUTPUT retrieval_result:
  matched_specimens: [specimen]
  match_strategy_used: enum[exact_match, type_device_match, cross_type, full_set]
  gold_specimens_available: integer
  silver_specimens_available: integer
```
