# Microskill 4.2: Placement Verification

**Layer:** 4 (Validation)
**Purpose:** Verify all proof placements match structure requirements

---

## VERIFICATION PROCESS

### Step 1: Load Structure Requirements

From structure-package.json:
```yaml
required_placements:
  - section: [section name]
    proof_type_required: [type]
    minimum_count: [number]
    placement_notes: [any specific requirements]
```

### Step 2: Match Drafted Placements

For each structure requirement:
```yaml
placement_match:
  requirement:
    section: [from structure]
    type_required: [from structure]
    minimum: [from structure]

  fulfilled_by:
    - block_id: [drafted block]
      type: [matches requirement?]
    # ... for each matching block

  status: MET | UNMET | EXCEEDED
```

## ORPHAN CHECK

Verify no proof blocks are orphaned (no clear placement):

```yaml
orphan_check:
  orphaned_blocks:
    - block_id: [id]
      reason: "[why no placement]"
      recommendation: [place in X | remove | callback use]

  orphan_count: [number]
  action_required: true | false
```

---

## COVERAGE GAP ANALYSIS

Identify any structure requirements not fulfilled:

```yaml
coverage_gaps:
  unfulfilled_requirements:
    - section: [section]
      required: [what's needed]
      shortfall: [what's missing]
      severity: CRITICAL | WARNING

  recommendations:
    - "[specific action to fill gap]"
```
