## Purpose

Load story-specific patterns from TIER1 extractions to provide empirical reference points for story type classification, beat mapping, character architecture, emotional arc design, and proof integration decisions. Build a comprehensive story type taxonomy from the vault, extract `story_architecture` fields from qualifying extractions, and catalog the emotional progressions, transition patterns, and proof-story data cascade patterns that elite controls use to make mechanisms FEEL real through narrative.

Unlike the lead vault loader (which catalogs lead types and hooks), this loader focuses on the NARRATIVE STRUCTURES that appear after the lead — the discovery stories, proof stories, origin stories, and transformation arcs that carry the persuasion function of the campaign. The vault intelligence here serves as the pattern library that story type classification and beat mapping draw from.

## Execution Protocol

### Step 1: Scan Extraction Index

Load TIER1 extraction files prioritizing:

### Step 2: Build Story Type Taxonomy

From each qualifying extraction, classify and catalog the story type. Build a comprehensive taxonomy of all story types represented in the vault:

| Story Type | Signature Pattern | Beat Structure | Persuasion Function | Best For |
|------------|------------------|----------------|---------------------|----------|
| **Standard Discovery** | Trip -> Yoda -> Secret -> Verification -> Root Cause | protagonist travels/encounters, finds expert, gets secret, researches, discovers root cause | Makes mechanism feel DISCOVERED (not invented) — organic, credible | Health supplements, natural remedies, weight loss |
| **Accidental Discovery** | Accident -> Yoda -> Secret -> Verification -> Root Cause | unplanned event triggers encounter, leading to discovery | Makes mechanism feel DESTINED — fate/providence brought it | Health, weight loss (when discovery feels serendipitous) |
| **Straight-to-Expert** | Expert Contact -> Shocking Info -> Secret -> Verification -> Root Cause | protagonist directly engages expert/authority who reveals mechanism | Makes mechanism feel AUTHORITATIVE — expert-validated, clinical | Health (ED, sleep), relationships (when expert is credible authority) |
| **Proof Story / Live Experiment** | Beta-test framing -> Data cascade -> Escalation -> Ordinariness proof -> Transparency | documented results from real people following the system | Makes mechanism feel PROVEN — empirical, documented, undeniable | Finance (trading), golf instruction, any system with tracked results |
| **Origin Story** | Problem -> Search -> Discovery -> Development -> Launch | creator's journey from problem to solution development | Makes mechanism feel PASSIONATE — creator cares deeply | Products with compelling founder stories |
| **Transformation Story** | Before state -> Encounter -> Application -> Struggle -> Breakthrough -> After state | protagonist's personal journey using the mechanism | Makes mechanism feel TRANSFORMATIVE — life-changing, personal | Any niche where personal transformation is the core promise |
| **Revelation Story** | Expert investigation -> Hidden truth -> Evidence -> Exposure -> Solution | expert uncovers a suppressed or hidden truth | Makes mechanism feel SUPPRESSED — powerful forces don't want you to know | Health conspiracies, financial cover-ups, industry exposures |
| **Warning Story** | Current danger -> Evidence mounting -> Consequences -> Escape route | narrative builds urgency around an approaching threat | Makes mechanism feel URGENT — you need this NOW or face consequences | Financial crisis, health warnings, market timing |
| **Underdog / Vulnerability** | Shame/failure -> Rock bottom -> Encounter -> Reluctant trial -> Surprising success | protagonist's shame-to-dignity arc mirrors prospect's experience | Makes mechanism feel RELATABLE — if it worked for them, it works for me | Any niche where prospect feels shame about their situation |
| **Hybrid** | Combines 2+ types (e.g., vulnerability + discovery, proof + transformation) | varies based on combination | Multiple persuasion functions layered | Sophisticated markets (Stage 4-5), complex products |

### Step 9: Specimen Tiering

```
FOR each story specimen in loaded data:

  CALCULATE story_richness_score:
    +2 IF full_story_text present AND length > 100 words
    +1 IF story_beats present with 3+ beats documented
    +1 IF emotional_arc documented (progression specified)
    +1 IF protagonist classified (archetype identified)
    +1 IF story_role/persuasion_function identified
    +1 IF story_type classified
    +1 IF complete story architecture (text + beats + arc + protagonist all present)
    MAX_SCORE: 8

  SORT specimens by story_richness_score DESC
  ASSIGN tier:
    Gold = top 10% (story_richness_score >= 7)
    Silver = next 15% (story_richness_score >= 5)
    Standard = remainder

  TAG each specimen with:
    tier: enum[gold, silver, standard]
    story_richness_score: integer (0-8)

  OUTPUT tier_distribution:
    total_specimens: integer
    gold_count: integer
    silver_count: integer
    standard_count: integer
```

### Step 10: Type-Matched Retrieval

```
DEFINE match_fields:
  primary: story_type (standard_discovery, accidental_discovery, straight_to_expert,
                       proof_story, origin_story, transformation_story, revelation_story,
                       warning_story, underdog_vulnerability, hybrid)
  secondary: emotional_arc (desperation_hope_conviction, shame_vulnerability_dignity,
                           curiosity_suspense_astonishment, skepticism_interest_belief_greed,
                           confusion_clarity_urgency, pain_encounter_relief_gratitude,
                           frustration_discovery_validation_excitement)
  tertiary: niche (health, wealth, relationships, self_improvement, sports_instruction)

WHEN campaign_context is available:
  STRATEGY 1 — exact_match:
    FILTER specimens WHERE story_type == target AND niche == target_niche
    PRIORITIZE Gold tier first, then Silver

  STRATEGY 2 — type_match:
    FILTER specimens WHERE story_type == target (any niche)
    PRIORITIZE Gold tier, same-niche Silver, then cross-niche Silver

  STRATEGY 3 — cross_type:
    RETURN all Gold specimens across story types
    INCLUDE Silver specimens matching target niche or emotional_arc

WHEN campaign_context is NOT available:
  RETURN full specimen set with tier metadata attached
  SORT by story_richness_score DESC

OUTPUT retrieval_result:
  matched_specimens: [specimen]
  match_strategy_used: enum[exact_match, type_match, cross_type, full_set]
  gold_specimens_available: integer
  silver_specimens_available: integer
```
