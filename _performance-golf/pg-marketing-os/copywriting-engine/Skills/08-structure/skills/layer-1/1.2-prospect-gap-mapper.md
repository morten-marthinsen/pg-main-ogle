# Layer 1.2 — Prospect Gap Question Mapper

## Purpose

Identify all belief gaps and doubts the prospect holds that prevent acceptance of the campaign thesis. Map each gap to the E5 framework's 9 core gap question types and identify additional niche-specific gaps. These gaps become the foundation for CPB chunk assignments.

**Primary Persona:** Sarah Chen — Competitive Intelligence Analyst
**Secondary Persona:** Dr. Richard Stern — Skeptical Academic

---

## Execution Protocol

### Step 1: Apply the Skeptical Academic Filter

Before mapping gaps, adopt the skeptic's mindset. Read the campaign thesis and ask:

> "If I heard this thesis for the first time — as a skeptical, tired, burned-out prospect who has tried everything — what would I need to believe before I'd accept this as true?"

Document every doubt, objection, and "yeah but..." that surfaces.

### Step 2: Map Against Core 9 Gap Questions

For each of the 9 E5 gap questions, assess whether the campaign thesis triggers that gap:

| # | Gap Question | Gap Type | Triggered? | Priority |
|---|-------------|----------|------------|----------|
| 1 | What kind of results can I expect? | results | yes/no | critical/important/supporting |
| 2 | How is this different from everything else? | differentiation | yes/no | critical/important/supporting |
| 3 | Why is this better than the common mechanisms? | superiority | yes/no | critical/important/supporting |
| 4 | How and why does this work? | mechanism | yes/no | critical/important/supporting |
| 5 | How easy is it? | ease | yes/no | critical/important/supporting |
| 6 | How fast does this work? | speed | yes/no | critical/important/supporting |
| 7 | Is it safe? Is it predictable? | safety | yes/no | critical/important/supporting |
| 8 | Will this work for me? | personalization | yes/no | critical/important/supporting |
| 9 | How can I get started using this? | access | yes/no | critical/important/supporting |

**Priority assignment rules:**
- **Critical:** The argument collapses without answering this question. The prospect will not move forward.
- **Important:** The argument is significantly weakened without this answer. Most prospects need it.
- **Supporting:** Adds depth and thoroughness. Some prospects need it; others are already satisfied.

### Step 3: Identify Niche-Specific Gap Questions

Beyond the core 9, each niche generates additional doubts:

**Health niches may add:**
- "Is this backed by real science?"
- "Will my doctor approve?"
- "Are there side effects?"
- "How is this different from what Big Pharma offers?"

**Wealth niches may add:**
- "What's the risk level?"
- "Do I need a lot of capital to start?"
- "Has this been tested in bear markets?"
- "Is this legal/regulated?"

**Sports instruction niches may add:**
- "Will this work for my skill level?"
- "Do I need special equipment?"
- "How long before I see results on the course/field?"
- "Is this proven at the professional level?"

**Self-improvement niches may add:**
- "Is this just another self-help system?"
- "What makes this different from [popular competitor]?"
- "Do I need to change my entire routine?"

Document any additional gap questions specific to the current niche and mechanism.

### Step 4: Cross-Reference with Proof Inventory

For each identified gap, check:
- Does the proof inventory contain evidence that addresses this gap?
- If yes: mark as `proof_available = true` with `proof_ids`
- If no: mark as `proof_available = false` — flag for potential feedback bus request

### Step 5: Rank and Prioritize Gaps

Order all gaps by:
1. **Priority** (critical first, then important, then supporting)
2. **Proof availability** (gaps with available proof are more immediately addressable)
3. **Thesis proximity** (gaps most directly threatening the thesis rank higher)

### Step 6: Validate Gap Coverage

Check coverage against minimum requirements:
- Minimum 5 total gaps identified
- Minimum 3 gap types represented
- At least 1 critical gap identified
- "Access" gap (How can I get started?) always included
- If mechanism is complex: "mechanism" gap must be critical or important
- If promise is bold: "results" gap must be critical

## Quality Gates

- [ ] Minimum 5 prospect gap questions identified
- [ ] Minimum 3 different gap types represented
- [ ] At least 1 gap classified as "critical"
- [ ] "Access" gap (How can I get started?) included
- [ ] Each gap has a documented thesis_threat_level
- [ ] Proof availability checked for every gap
- [ ] Priority distribution is not all "supporting" (must have substance)
- [ ] Niche-specific gaps considered (not just core 9)
- [ ] Any proof gaps flagged for feedback bus

---

## Anti-Patterns

- **Shallow mapping:** Only listing 3-4 generic gaps without niche-specific analysis
- **All-critical assignment:** Marking everything as "critical" dilutes the priority signal
- **Ignoring proof availability:** Mapping gaps without checking if proof exists to fill them
- **Skipping niche context:** The same gap type requires different CPB content in health vs. wealth vs. sports
- **Forgetting access:** The "How can I get started?" gap must always be present
