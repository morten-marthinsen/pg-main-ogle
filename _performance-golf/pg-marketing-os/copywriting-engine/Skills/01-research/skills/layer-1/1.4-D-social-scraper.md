# Skill 1.4-D: Social Media Scraper

**Version:** 2.0
**Layer:** 1 (Infrastructure)
**Type:** Content Extraction
**Dependencies:** validated_sources.json, market_config.yaml
**Output:** Appends to master_raw_content.json

---

## Purpose

Scrape social media platforms (Twitter/X, Facebook Groups, LinkedIn, Instagram) for customer voice data. Social media provides real-time sentiment and raw, unfiltered expressions.

---

## Target Platforms

```
SUPPORTED PLATFORMS:

1. Twitter/X - Short-form pain/hope expressions
2. Facebook Groups - Community discussions
3. LinkedIn - Professional perspective
4. Instagram - Visual + caption content
5. TikTok - Comments on relevant content
```

---

## Input Schema

```yaml
input:
  validated_sources: validated_sources.json
  market_config: market_config.yaml
  filter:
    content_type: "social"
    platforms: ["Twitter", "Facebook", "LinkedIn", "Instagram", "TikTok"]
```

mcp__apify__call-actor:

## Execution Logic

### Step 1: Platform Prioritization

```
PRIORITIZE platforms based on market_config:

FOR each platform in market_config.platforms:
  IF platform.type = "social":
    ADD to scrape_queue with priority

DEFAULT priority order:
1. Twitter/X - Highest volume of raw expression
2. Facebook Groups - Community depth
3. LinkedIn - Professional perspective
4. Instagram - Lifestyle/visual markets
5. TikTok - Younger demographic
```

### Step 2: Query Adaptation

```
ADAPT queries for each platform:

TWITTER:
- Shorter, hashtag-enhanced
- Add sentiment operators
- Example: "#trading frustrating" OR "trading ruined"

FACEBOOK:
- Natural language
- Question format
- Example: "struggling with trading"

LINKEDIN:
- Professional tone
- Industry terms
- Example: "day trading challenges professional"

INSTAGRAM/TIKTOK:
- Hashtag-centric
- Trend-aware
- Example: #tradingstruggle #traderfail
```

### Step 3: Scrape and Extract

```
FOR each platform:

  EXECUTE appropriate Apify actor

  WAIT for results (async if large scrape)

  FOR each post/item returned:

    CAPTURE:
    - Full text content
    - Author info
    - Engagement metrics
    - Date/time
    - Replies/comments

    PRE-EXTRACT quotes:
    - First-person statements
    - Emotional expressions
    - Problem descriptions
    - Question reveals

    APPEND to social_content
```

### Step 4: Engagement Weighting

```
WEIGHT content by engagement:

High value (prioritize in analysis):
- Posts with 100+ likes/reactions
- Threads with 20+ replies
- Viral content

Medium value:
- 10-100 engagement
- Active discussion

Low value (include but lower weight):
- < 10 engagement
- No replies

CALCULATE engagement_score for each post
```

## Quote Pre-Extraction

```
SOCIAL-SPECIFIC indicators:

HIGH VALUE:
- Raw emotional expression ("I'm so frustrated")
- Real-time pain ("Just lost $500 today")
- Questions to community ("Has anyone else...")
- Confessions ("I have to admit...")
- Celebrations/breakthroughs ("Finally!")

LOWER VALUE:
- Links/promotions
- Generic statements
- Bot-like content
- Copied/meme content

CAPTURE with platform context (audience differs)
```

---

## Tool Resilience

```
SOCIAL MEDIA TOOL CHAIN:

Twitter:
1. quacker/twitter-scraper
2. apidojo/tweet-scraper
3. Perplexity search (fallback)

Facebook:
1. apify/facebook-groups-scraper
2. Perplexity with site:facebook.com

LinkedIn:
1. curious_coder/linkedin-post-search-scraper
2. Perplexity with site:linkedin.com

Instagram:
1. apify/instagram-scraper
2. apify/instagram-hashtag-scraper

TikTok (Audit Fix FAIL-003):
1. clockworks/tiktok-scraper + clockworks/tiktok-comments-scraper
2. apify/tiktok-scraper (fallback)
3. Perplexity research query (fallback)
4. Manual search URLs (last resort)

IF tool fails:
- Log error with specific tool name
- AUTOMATICALLY try next in chain (do not wait)
- Accept partial data from successful tools
- Continue to next platform only after exhausting chain
```

---

## Privacy Considerations

```
HANDLE social data responsibly:

- Public posts only
- No private messages
- No contact scraping
- Aggregate for patterns
- Individual quotes anonymized in final output if needed
```

---

## Quality Criteria

1. **Recency** - Social data should be recent (last 12 months)
2. **Authenticity** - Real users, not bots/spam
3. **Engagement** - Prioritize content with interaction
4. **Diversity** - Multiple voices, not echo chambers
