## Purpose

The Handoff Packager creates the SINGLE FINAL DELIVERABLE of the entire Deep Research v3 pipeline. This document is the comprehensive research package that leaves the system and enters the hands of strategists, copywriters, media buyers, product teams, and executives.

It consolidates weeks of automated research — thousands of quotes, dozens of analyses, scored opportunities, risk assessments, action plans, and measurement frameworks — into one self-contained markdown file that requires NO additional context to be actionable.

The document must be:
- **Complete**: Every finding from every layer represented
- **Traceable**: Every claim linked to evidence via Quote IDs
- **Structured**: 14 sections in consistent format
- **Self-contained**: No external references required to understand
- **Actionable**: Teams can execute directly from this document

### Step 1.5: Compile Business Context (Section 0)

```
FROM research brief (brief.yaml or brief.md):
  IF brief contains context section:
    EXTRACT why_now
    EXTRACT decisions_pending (array)
    EXTRACT target_customer

  IF brief contains hypotheses section:
    EXTRACT all hypotheses with statements and rationales

  IF brief contains exploration_emphasis section:
    EXTRACT emphasis areas (max 3)

COMPILE Section 0: Business Context

  - What decisions it will inform

  - Questions to be answered (list only — answers in Section 15)

sections are compiled. All findings are presented objectively regardless
of hypotheses or pending decisions.

IF brief does not contain sections 3-6:
  Section 0 displays: "No business context provided in research brief.
  Research conducted using standard comprehensive methodology."
```

### Step 2: Calculate Research Statistics

```
FROM source_manifest.json:
  total_sources = count(sources)
  sources_by_platform = group_by(sources, platform)

FROM scored_quotes.json:
  total_quotes = count(quotes)
  bucket_distribution = group_count(quotes, bucket)
  priority_distribution = group_count(quotes, priority)

FROM authenticity_report.json (if exists):
  authenticity_rate = gate_decision.authenticity_rate

CALCULATE confidence_score:
  confidence = (
    (authenticity_rate * 0.30) +
    (quote_volume_score * 0.25) +
    (source_diversity_score * 0.20) +
    (pattern_consistency_score * 0.15) +
    (cross_validation_score * 0.10)
  )

  WHERE:
    quote_volume_score = min(total_quotes / 1000, 1.0)
    source_diversity_score = min(unique_platforms / 8, 1.0)
    pattern_consistency_score = theme_synthesis.consistency_score
    cross_validation_score = avg(opportunity.confidence for each opportunity)
```

### Step 16: Compile Hypothesis Validation (Section 16)

```
IF hypothesis_checklist exists (from Step 1.5):

  FOR EACH hypothesis in hypothesis_checklist:

    SEARCH all layer outputs for evidence:
      - Evidence FOR: Quotes and data supporting the hypothesis
      - Evidence AGAINST: Quotes and data refuting the hypothesis

    CALCULATE evidence weight:
      - Count quotes/data points FOR
      - Count quotes/data points AGAINST
      - Assess strength (high-emotion/specific quotes weight more)

    DETERMINE verdict:
      - VALIDATED: >70% of evidence supports, minimal contradicting evidence
      - INVALIDATED: >70% of evidence contradicts, minimal supporting evidence
      - INCONCLUSIVE: Mixed evidence OR insufficient evidence (<10 relevant data points)

    COMPILE validation entry:
      - Hypothesis: [verbatim from brief]
      - Original Rationale: [from brief]
      - Verdict: [VALIDATED / INVALIDATED / INCONCLUSIVE]
      - Confidence: [percentage based on evidence clarity]

      - Evidence For:
        * [Quote ID]: "[supporting quote]"
        * [Data point]

      - Evidence Against:
        * [Quote ID]: "[contradicting quote]"
        * [Data point]

      - Nuance: [Important caveats, partial validations, segment-specific findings]

    QUALITY RULES:
      - NEVER force a VALIDATED verdict to please stakeholders
      - INCONCLUSIVE is a valid and honest verdict
      - Include contradicting evidence even when verdict is VALIDATED
      - Nuance section is MANDATORY — no hypothesis is black/white
      - Confidence percentage must be justified by evidence weight

  IF no hypotheses in brief:
    SKIP Section 16 or include: "No hypotheses specified in research brief for validation."

ANTI-BIAS PROTOCOL:
  - Review for confirmation bias: Did you look equally hard for evidence against?
  - Review for anchoring: Did the hypothesis statement influence your search?
  - If uncertain, default to INCONCLUSIVE rather than forced verdict
```

## MANDATORY CHUNKED ASSEMBLY PROTOCOL (v1.2)

### ROOT CAUSE: OUTPUT TOKEN EXHAUSTION

```
CRITICAL FAILURE MODE IDENTIFIED:
The model has a maximum output limit of ~15,000-30,000 tokens per response.
This translates to approximately 30KB of text per turn.

A 300KB handoff document CANNOT be written in a single response.
Attempting to do so triggers automatic truncation, resulting in:
- Missing quote sections
- Continuation markers appearing
- File sizes of 30-80KB instead of 300KB+

THIS IS A TECHNICAL LIMITATION, NOT A QUALITY ISSUE.
```

### MANDATORY ASSEMBLY STRATEGY

```yaml
assembly_strategy: incremental_chunked

# NEVER attempt to write the entire handoff in one operation
# ALWAYS use multiple Edit tool calls to build the file incrementally

chunking_rules:
  max_content_per_write: 10KB  # ~5,000 tokens
  staging_approach: section_by_section
  size_validation: after_each_major_section

expected_tool_calls:
  minimum_for_1000_quotes: 10
  typical_for_1000_quotes: 12-15
  minimum_for_1500_quotes: 15
  typical_for_1500_quotes: 18-22
```
