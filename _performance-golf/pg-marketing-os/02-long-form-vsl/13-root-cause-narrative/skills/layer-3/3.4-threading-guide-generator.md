# Skill 13.3.4: Threading Guide Generator

**Version:** 1.0
**Skill:** 10 — Root Cause Narrative
**Layer:** 3 (Refinement & Polish)
**Position:** Fourth refinement microskill — produces the root cause threading guide for downstream skills
**Type:** Guide Generation
**Dependencies:** All Layer 2 and Layer 3 outputs, checkpoint_response.json (from 0.4), tier1_intelligence_index.json (from 0.2)
**Output:** threading_guide.json

---

## PURPOSE

Produce the root cause threading guide that tells every downstream skill how to weave root cause language throughout the entire promotion. The key finding from TIER1 analysis is that the root cause is NOT just a section — it is the structural foundation of the entire campaign. 80% of elite controls weave root cause language and references throughout lead, story, mechanism, product, offer, and close sections. This microskill creates the anchor phrase, maps repetition touchpoints, sets emphasis level, and generates specific usage notes for each downstream location.

---

## IDENTITY

**This microskill IS:**
- A threading architect that maps root cause language across the entire promotion, not just the root cause section
- An anchor phrase creator that distills the root cause into a memorable, repeatable phrase
- A downstream guide generator that gives specific instructions to Skills 11, 12, 13, 14 and body copy assembly
- A repetition pattern designer that determines HOW root cause language recurs (woven, restated, or single reveal)

**This microskill is NOT:**
- A root cause section writer — the narrative is already written in Layer 2; this creates the threading map
- A downstream executor — it generates the GUIDE; downstream skills execute it
- A single-use tool — the threading guide is one of the most-consumed outputs of Skill 10
- An optional output — Rule 7 requires root cause to be woven throughout; this guide ensures compliance

## PROCESS

### Step 1: Create the Anchor Phrase

```
THE ANCHOR PHRASE is the single most memorable, repeatable expression of the root cause.
It appears in the root cause section AND recurs throughout the promotion.

TIER1 ANCHOR PHRASE EXAMPLES:
  Gundry: "lectins" / "plant proteins that destroy your gut"
  Montezuma: "Death begins in the colon"
  Haney: "IT'S A LOT OF BULL" / "clubface-to-path"
  Stansberry: "the end of America as we know it"
  FloraSource: "Prescription Roulette"

ANCHOR PHRASE RULES:
  1. SPECIFIC — names the root cause directly or memorably
  2. REPEATABLE — can be said multiple times without feeling forced
  3. COUNTER-INTUITIVE — surprises or challenges (for maximum memorability)
  4. SHORT — 2-7 words (can be woven into any sentence)
  5. VISUAL — creates a mental image if possible
  6. EMOTIONAL — triggers the emotional response designed in the arc

CREATION PROCESS:

  EXTRACT from three_part_structure.what_it_really_is:
    The root cause in its most specific form

  ATTEMPT to create anchor phrase using these techniques:

  Technique A — Named Cause:
    Directly name the root cause in 2-4 words
    Example: "Lectins," "Clubface-to-path," "Telomere shortening"
    BEST WHEN: Root cause has a specific, nameable element

  Technique B — Provocative Statement:
    Distill the root cause into a shocking short statement
    Example: "Death begins in the colon," "IT'S A LOT OF BULL"
    BEST WHEN: Root cause benefits from emotional punch

  Technique C — Villain Reference:
    Name the villain's action as the anchor
    Example: "Prescription Roulette," "The end of America"
    BEST WHEN: Villain is the dominant frame; villain name is memorable

  Technique D — Inversion Label:
    Name the conventional wisdom being inverted
    Example: "The 'healthy food' that's killing you"
    BEST WHEN: opposite_of_truth reframe is primary

  GENERATE 3-5 anchor phrase candidates
  SELECT the one that is most: specific, repeatable, counter-intuitive, short, visual, emotional

  VALIDATE anchor phrase:
    □ Can it be used in a headline? (brevity test)
    □ Can it be woven into body copy mid-sentence? (flexibility test)
    □ Does it immediately evoke the root cause? (recognition test)
    □ Is it distinct from competitors? (differentiation test)
    □ Would the reader remember it tomorrow? (memorability test)
```

### Step 2: Determine Emphasis Level

```
SELECT emphasis level from 3 options:

LEVEL 1: foundation (most common — root cause IS the structural foundation)
  Description: Root cause language woven into EVERYTHING — lead, story, mechanism,
    product, offer, close. The root cause is the lens through which the entire
    promotion is viewed.
  When to use: When root cause is the primary persuasive engine
  TIER1 example: Gundry — "lectins" appear in lead, mechanism, product description,
    guarantee, and close
  Downstream impact: EVERY skill receives root cause threading instructions

LEVEL 2: heavy (prominent section + frequent references)
  Description: Dedicated root cause section PLUS references in lead, mechanism,
    and close. Not in every section but in most.
  When to use: When root cause is important but shares persuasive weight with mechanism
  TIER1 example: Haney — root cause section is strong, referenced in lead and close,
    but mechanism section is also independently powerful
  Downstream impact: Key skills (lead, mechanism, close) receive threading instructions

LEVEL 3: moderate (specific section with limited callbacks)
  Description: Root cause has its own section with 1-2 callbacks in other sections.
  When to use: When mechanism is the primary persuasive engine and root cause is supporting
  TIER1 example: Rare in TIER1 controls — most use foundation or heavy
  Downstream impact: Only mechanism and close receive threading instructions

SELECTION CRITERIA:
  IF root cause is the campaign's central argument: foundation
  IF root cause and mechanism share equal persuasive weight: heavy
  IF mechanism is dominant and root cause is supporting: moderate
  DEFAULT: heavy (safest choice for most campaigns)

FROM upstream analysis:
  CHECK emphasis_level in root cause package (if specified)
  CHECK campaign_thesis alignment (does thesis revolve around root cause?)
  DECIDE and LOG rationale
```

### Step 3: Map Repetition Touchpoints

```
DEFINE where root cause language should appear in downstream sections:

TOUCHPOINT 1: Lead (Skill 11)
  Usage: Root cause hint or foreshadowing WITHOUT revealing the root cause
  Pattern: "What if the real cause of [problem] was hiding in the one place
            nobody thought to look?"
  Emphasis level applicability: foundation, heavy
  Threading note: "Lead should hint at root cause without revealing it.
    Use the language of surprise/counter-intuition. Anchor phrase should
    NOT appear in lead (preserve reveal power for root cause section)."

TOUCHPOINT 2: Story/Credibility Section
  Usage: Authority's discovery of the root cause woven into personal/origin story
  Pattern: "When [authority] discovered [root cause anchor phrase], everything
            [he/she] thought about [topic] changed."
  Emphasis level applicability: foundation, heavy
  Threading note: "Story section should reference the root cause discovery as
    a pivotal moment. Use anchor phrase in the discovery narrative.
    Villain can be introduced here at a high level."

TOUCHPOINT 3: Mechanism Narrative (Skill 17)
  Usage: Mechanism explicitly addresses the root cause — this is the solution TO the root cause
  Pattern: "[Mechanism] works by directly targeting [root cause anchor phrase]"
  Emphasis level applicability: ALL levels (mechanism MUST reference root cause)
  Threading note: "Every mechanism step should reference root cause as what
    it's solving. The reader should never lose sight of WHY the mechanism
    works — because it addresses the root cause directly."

TOUCHPOINT 4: Product Section (Skill 15)
  Usage: Product positioned as the delivery vehicle for the root cause solution
  Pattern: "Specifically designed to address [root cause anchor phrase]"
  Emphasis level applicability: foundation, heavy
  Threading note: "Product should be framed as the practical application of
    the mechanism that addresses the root cause. Use anchor phrase in
    product description."

TOUCHPOINT 5: Offer Section (Skill 16)
  Usage: Value justified through root cause lens — "now that you know the real cause..."
  Pattern: "Now that you know [root cause], can you afford NOT to address it?"
  Emphasis level applicability: foundation
  Threading note: "Offer urgency should be root-cause-driven. The value
    proposition is: you now know the real cause, and this is the only
    solution that addresses it."

TOUCHPOINT 6: Close (Skill 17)
  Usage: Root cause callback + fear of return + villain reminder
  Pattern: "Don't let [villain] keep profiting from your [problem].
            You know the real cause is [root cause]. Address it now."
  Emphasis level applicability: ALL levels (close ALWAYS references root cause)
  Threading note: "Close should callback to the root cause with:
    1. Fear of return: 'Without addressing [root cause], nothing changes'
    2. Villain reminder: 'Don't let [villain] keep winning'
    3. Root cause anchor phrase: final use for memorability"

TOUCHPOINT 7: Guarantee/Risk Reversal
  Usage: Guarantee framed through root cause lens
  Pattern: "If addressing [root cause] doesn't produce [result]..."
  Emphasis level applicability: foundation
  Threading note: "Guarantee language should reference the root cause —
    reader's confidence comes from KNOWING the cause and believing the
    solution addresses it."

FOR each touchpoint:
  IF emphasis_level includes this touchpoint:
    GENERATE specific usage note with example language
  ELSE:
    MARK as "not applicable at [emphasis_level]"
```

### Step 4: Select Placement and Repetition Patterns

```
SELECT placement_pattern:
  throughout (most common — root cause language appears in every major section)
  lead_and_throughout (root cause hinted in lead, explicit everywhere else)
  mid_body (root cause section appears mid-campaign, callbacks before and after)
  early_body (root cause section appears early, woven through remainder)

  SELECTION: Based on where root cause section sits in campaign flow
  DEFAULT: throughout (if emphasis_level is "foundation")

SELECT repetition_pattern:
  woven_throughout (85% of TIER1 controls):
    Root cause language subtly woven into every section
    Not always explicit — sometimes as background assumption
    Example: "Because lectins damage your gut..." appears as a given, not a re-argument

  restated_multiple_times (10%):
    Root cause explicitly restated at key moments
    More overt than woven — each restatement reinforces the argument
    Example: "Remember: the real cause is [root cause]" at each section transition

  single_powerful_reveal (5%):
    Root cause revealed ONCE with massive impact
    Callbacks are minimal — the single reveal carries the entire campaign
    Example: Only used when root cause reveal is the Big Idea itself

  DEFAULT: woven_throughout (matches foundation emphasis level)
```

### Step 5: Generate Downstream Usage Notes

```
FOR each downstream skill that receives threading instructions:

GENERATE specific_usage_note:
  skill: string
  anchor_phrase: string
  how_to_use: string  # specific instruction for that skill
  example_language: string  # sample text showing root cause threading
  frequency: string  # how often to reference root cause in that section
  tone: string  # how the root cause should feel in that section
  warnings: string  # what NOT to do

EXAMPLE downstream notes:

  For Skill 11 (Mechanism Narrative):
    anchor_phrase: "[anchor phrase]"
    how_to_use: "Every mechanism step must reference [root cause] as what it solves.
                 The reader should understand that this mechanism works BECAUSE it
                 addresses [root cause], unlike everything else they've tried."
    frequency: "Every major mechanism section"
    tone: "Confident connection — root cause is established fact by now"
    warnings: "Do not re-argue the root cause — it's accepted. Reference it as given."

  For Skill 14 (Close):
    anchor_phrase: "[anchor phrase]"
    how_to_use: "Final callback to root cause: fear of inaction, villain reminder,
                 anchor phrase repetition. The close should make the reader feel that
                 NOT acting means accepting [root cause] damage continuing."
    root_cause_callback: "[specific callback text]"
    fear_of_return: "[specific fear text]"
    villain_reminder: "[specific villain callback text]"
```

### Step 6: Compile Threading Guide

```
COMPILE threading_guide:
  anchor_phrase: string
  anchor_phrase_technique: string  # which of 4 techniques used
  anchor_phrase_alternatives: [string]  # 2 backup phrases

  emphasis_level: enum[foundation, heavy, moderate]
  emphasis_rationale: string

  placement_pattern: enum[throughout, lead_and_throughout, mid_body, early_body]
  repetition_pattern: enum[woven_throughout, restated_multiple_times, single_powerful_reveal]

  repetition_points:
    - location: string  # e.g., "lead", "story", "mechanism", "product", "offer", "close"
      applicable: boolean  # based on emphasis level
      usage: string
      example_language: string
      frequency: string
      tone: string
      warnings: string

  downstream_notes:
    for_skill_11: object
    for_skill_12: object
    for_skill_13: object
    for_skill_14: object
    for_body_copy_assembly: object

  root_cause_language_bank:
    anchor_phrase: string
    formal_statement: string  # "The root cause is [what_it_really_is]"
    casual_reference: string  # "[anchor phrase] is still the problem"
    fear_callback: string  # "Without addressing [root cause], nothing changes"
    villain_callback: string  # "[Villain] is still profiting from..."
    action_prompt: string  # "Now that you know [root cause], here's what to do"

WRITE to: layer-3-outputs/threading_guide.json
HANDOFF to: Layer 4 validators, downstream skills
```

## QUALITY CRITERIA

1. **Anchor phrase is memorable** — must pass the "remember it tomorrow" test; must be specific, short, and counter-intuitive
2. **Anchor phrase is repeatable** — must work in headlines, body copy, and close equally well
3. **Repetition points are comprehensive** — every downstream section has specific instructions (at foundation level)
4. **Emphasis level justified** — clear rationale for chosen level based on campaign structure
5. **Downstream skills can use guide directly** — each skill receives specific, actionable threading instructions with example language
6. **Language bank is complete** — formal, casual, fear, villain, and action variants all provided
7. **Anchor phrase is distinct** — not a phrase already used by competitors or in the market

---

## FAILURE MODES

| Failure | Severity | Detection | Handling |
|---------|----------|-----------|----------|
| Anchor phrase is generic | HIGH | Phrase could describe any root cause ("the real problem") | Rewrite with specificity; must name or evoke the specific root cause |
| Anchor phrase too long | MODERATE | More than 7 words | Shorten; anchor phrases must be weavable mid-sentence |
| Repetition points incomplete | MODERATE | Missing instructions for key downstream skills | Add missing points; mechanism and close ALWAYS need instructions |
| Emphasis level inconsistent with campaign | LOW | Foundation selected but campaign is mechanism-dominant | Re-evaluate; adjust to heavy or moderate |
| Language bank missing variants | LOW | Only anchor phrase provided, no casual/fear/villain variants | Generate all variants; downstream skills need multiple forms |
| Anchor phrase matches competitor | MODERATE | Phrase already used in the market | Generate alternative; must be differentiated |
| No downstream notes for Skill 11 | HIGH | Mechanism narrative has no threading instructions | Add immediately; mechanism MUST reference root cause |

## EXAMPLES

### Example 1: Gundry — "Lectins" Threading Guide

```
THREADING GUIDE:

anchor_phrase: "Lectins"
anchor_phrase_technique: "Named Cause"
alternatives: ["gut-destroying plant proteins", "the lectin crisis"]

emphasis_level: "foundation"
emphasis_rationale: "Gundry's entire campaign revolves around lectins as root cause.
    Campaign thesis is lectin-driven. Every product feature addresses lectins.
    Root cause IS the Big Idea's foundation."

placement_pattern: "throughout"
repetition_pattern: "woven_throughout"

repetition_points:
  Lead: "Hint at lectins without naming them: 'A protein hiding in your
         favorite healthy foods is destroying your gut from the inside out'"
  Story: "Dr. Gundry's personal discovery of lectins as the turning point"
  Mechanism: "Every mechanism step addresses lectin damage directly"
  Product: "Formulated to neutralize lectin damage at the gut wall"
  Offer: "Now that you know lectins are the cause, this is your solution"
  Close: "Don't let lectins keep destroying your health. You know the truth now."

root_cause_language_bank:
  anchor_phrase: "Lectins"
  formal_statement: "Lectins — plant proteins found in wheat, tomatoes, and beans —
    are the root cause of gut damage, weight gain, and chronic inflammation"
  casual_reference: "It's the lectins"
  fear_callback: "Every meal without addressing lectins is another meal of gut damage"
  villain_callback: "The food industry is still pushing lectin-heavy foods for profit"
  action_prompt: "Now that you know it's lectins, here's exactly how to stop the damage"
```
