# Layer 0.2.5 — Specimen Decomposer

**Version:** 1.0
**Layer:** 0 (Foundation)
**Type:** Specimen Analysis
**Domain:** Root-Cause Narrative
**Dependencies:** vault_root-cause-narrative_intelligence (from 0.2-vault-intelligence-loader.md)
**Output:** specimen_decomposition (structural templates, technique catalog, power elements)

---

## Purpose

Decompose top-scoring root-cause narrative specimens from the vault-intelligence into reusable structural templates, technique catalogs, and power elements. Root-cause narrative specimens are organized by passage type — blame shift, countersell, paradigm disruption, key metaphors — each containing verbatim text with identified techniques. This skill extracts the rhetorical architecture that makes elite root-cause narratives persuasive.

**This skill bridges the gap between:** vault data (categorized verbatim passages with technique annotations) and generation layers (what Claude needs to construct original root-cause narratives with the same structural sophistication).

---

input:

## Decomposition Protocol

### Phase 1: Specimen Selection & Tiering

```
IF tier_metadata present (Phase B loader active):
  LOAD gold_specimens → full decomposition
  LOAD silver_specimens → template-only decomposition
ELSE (pre-upgrade fallback):
  FOR each passage type (blame_shift, countersell, paradigm_disruption, key_metaphors):
    SELECT top 3 passages by position as pseudo-Gold
    SELECT next 5 passages as pseudo-Silver
  LOG "Operating in pre-upgrade mode — tiering estimated from position within type"

VERIFY each passage specimen has:
  - verbatim_text (required — skip if absent)
  - technique (required)
  - source_file (required for provenance)
  - niche (required for type-matching)
  - passage_type category (required — blame_shift | countersell |
                           paradigm_disruption | key_metaphors)
```

### Phase 2: Structural Template Extraction

For each Gold specimen (per passage type):

```
BLAME SHIFT PASSAGES:
  ANALYZE verbatim_text and extract:
    rhetorical_skeleton:
      - element_type: [villain_identification | blame_placement |
                       absolution_of_reader | evidence_of_villain |
                       emotional_validation | redirect_to_real_cause]
      - function: string
      - position: integer
      - proportion: float
    blame_architecture:
      villain_type: string (industry, institution, specific entity, system)
      absolution_method: string (how the reader is freed from self-blame)
      evidence_type: string (logical, emotional, factual, conspiratorial)

COUNTERSELL PASSAGES:
  ANALYZE verbatim_text and extract:
    rhetorical_skeleton:
      - element_type: [false_solution_naming | why_it_fails |
                       reader_experience_validation | structural_limitation |
                       transition_to_real_solution]
      - function: string
      - position: integer
      - proportion: float
    countersell_architecture:
      false_solution_named: string
      failure_mechanism: string (WHY the false solution doesn't work)
      transition_technique: string (how it bridges to the real solution)

PARADIGM DISRUPTION PASSAGES:
  ANALYZE verbatim_text and extract:
    rhetorical_skeleton:
      - element_type: [old_paradigm_statement | disruption_evidence |
                       new_paradigm_introduction | proof_of_new_paradigm |
                       implications_for_reader]
      - function: string
      - position: integer
      - proportion: float
    disruption_architecture:
      old_belief: string
      new_truth: string
      disruption_mechanism: string (what evidence/logic breaks the old belief)

KEY METAPHORS:
  ANALYZE verbatim_text and extract:
    metaphor_skeleton:
      source_domain: string (what the metaphor draws from)
      target_domain: string (what it explains)
      mapping: string (what corresponds to what)
      emotional_payload: string (what feeling the metaphor activates)

For Silver specimens:
  Extract rhetorical_skeleton ONLY (per passage type)
  SKIP architecture detail and proportion analysis
```

### Phase 3: Technique Catalog Compilation

Aggregate across ALL decomposed specimens:

```
blame_shift_techniques:
  FOR each unique technique in blame_shift passages:
    - technique_name: string
    - count: integer
    - implementation_pattern: string (how to execute this technique)
    - gold_example: string (1-2 sentence excerpt from Gold specimen)
    - best_for_niche: string

countersell_techniques:
  FOR each unique technique in countersell passages:
    - technique_name: string
    - count: integer
    - implementation_pattern: string
    - gold_example: string
    - false_solutions_targeted: [string] (what kinds of false solutions this countersells)

disruption_techniques:
  FOR each unique technique in paradigm_disruption passages:
    - technique_name: string
    - count: integer
    - implementation_pattern: string
    - gold_example: string
    - paradigm_type: string (scientific, economic, behavioral, systemic)

metaphor_techniques:
  FOR each unique metaphor construction approach:
    - pattern_name: string
    - source_domains_used: [string]
    - effectiveness_notes: string
    - specimens_using: [source_file references]

passage_sequencing:
  FOR each observed ordering of passage types within full root-cause narratives:
    - sequence: [string] (ordered list of passage types)
    - frequency: integer
    - effectiveness_notes: string
```

### Phase 4: Power Element Extraction

From Gold specimens ONLY:

```
blame_shift_fragments:
  FOR each Gold blame_shift passage:
    - verbatim_fragment: string (strongest 1-3 sentences)
    - technique: string
    - villain_named: string
    - emotional_impact: string
    - source_file: string

countersell_fragments:
  FOR each Gold countersell passage:
    - verbatim_fragment: string (strongest 1-3 sentences)
    - technique: string
    - false_solution_targeted: string
    - source_file: string

disruption_fragments:
  FOR each Gold paradigm_disruption passage:
    - verbatim_fragment: string (strongest 1-3 sentences)
    - technique: string
    - old_belief_challenged: string
    - source_file: string

key_metaphor_verbatims:
  FOR each Gold key_metaphors passage:
    - verbatim_metaphor: string (the complete metaphor text)
    - source_domain: string
    - target_domain: string
    - source_file: string
```

## Quality Gates

- [ ] Minimum 2 Gold specimens per passage type fully decomposed
- [ ] Minimum 3 Silver specimens per passage type template-decomposed
- [ ] All rhetorical skeletons have complete element sequences
- [ ] Technique catalog has minimum 2 techniques per passage type
- [ ] Power elements contain minimum 2 verbatim fragments per passage type from Gold specimens
- [ ] Key metaphors have source_domain and target_domain mapped (not just text)
- [ ] Every power element includes source provenance (source_file)
- [ ] Context window budget respected
- [ ] No raw verbatim_text passed through undecomposed

---

## Anti-Patterns

| Anti-Pattern | Signal | Fix |
|---|---|---|
| **Raw Passthrough** | Full passage text dumped to downstream skills | Decompose into skeleton + technique + elements |
| **Type-Blind Decomposition** | All passage types decomposed identically | Each passage type has its own skeleton and architecture |
| **Metaphor Without Mapping** | Key metaphors listed as text without domain analysis | Every metaphor needs source_domain, target_domain, mapping |
| **Technique Without Implementation** | Techniques named without execution guidance | Every technique needs implementation_pattern |
| **Passage Type Isolation** | Each passage type analyzed independently without sequencing | Include passage_sequencing in technique catalog |
