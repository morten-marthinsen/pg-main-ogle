## Loading Protocol

### Step 1: Load V3 Vault Intelligence File

```
LOCATE: vault-intelligence/root-cause-narrative-vault-intelligence.json

VALIDATE:
  - File exists and is valid JSON
  - Contains "version": "v3"
  - Contains "specimens" section with non-empty content
  - Contains "patterns" section with frequency distributions

IF file missing or invalid:
  SET status = FAIL
  LOG "V3 vault-intelligence build must run first"
  HALT
```

### Step 2: Extract Specimens

```
FROM specimens section:
  LOAD all verbatim content specimens
  INDEX by category/type for downstream lookup
  VERIFY verbatim_quote/text fields are present (not summaries)
```

### Step 3: Extract Patterns

```
FROM patterns section:
  LOAD all frequency distributions
  IDENTIFY top-frequency categories for defaults
  CALCULATE niche-specific pattern weightings
```

### Step 4: Extract Statistics

```
FROM statistics section:
  LOAD aggregate metrics
  SET quality baselines for validation gates
```

### Step 5: Specimen Tiering

```
FOR each specimen in loaded data:

  CALCULATE passage_richness_score:
    +2 IF verbatim_text present AND length > 30 words
    +2 IF technique identified (named technique, not generic)
    +1 IF passage_type classified (blame_shift, countersell, paradigm_disruption)
    +1 IF key_metaphors present with at least 1 entry
    +1 IF complete passage arc (technique + verbatim + type all present)
    MAX_SCORE: 7

  APPLY per-passage-type tiering:
    FOR each passage type (blame_shift, countersell, paradigm_disruption):
      SORT specimens within type by passage_richness_score DESC
      ASSIGN tier:
        Gold = top 10% within type (passage_richness_score >= 6)
        Silver = next 15% within type (passage_richness_score >= 4)
        Standard = remainder

  TAG each specimen with:
    tier: enum[gold, silver, standard]
    passage_richness_score: integer (0-7)
    tier_passage_type: string (passage type used for tiering)

  OUTPUT tier_distribution:
    total_specimens: integer
    gold_count: integer
    silver_count: integer
    standard_count: integer
    per_type_distribution: {passage_type: {gold: int, silver: int, standard: int}}
```

### Step 6: Type-Matched Retrieval

```
DEFINE match_fields:
  primary: passage_type (blame_shift, countersell, paradigm_disruption)
  secondary: technique (e.g., blame_redirect, false_enemy, paradigm_inversion)
  tertiary: niche (e.g., health, finance, survival, self-improvement)

WHEN campaign_context is available:
  STRATEGY 1 — exact_match:
    FILTER specimens WHERE passage_type == target AND technique == target AND niche == target_niche
    PRIORITIZE Gold tier first, then Silver

  STRATEGY 2 — type_match:
    FILTER specimens WHERE passage_type == target (any technique, any niche)
    PRIORITIZE Gold tier, same-niche Silver, then cross-niche Silver

  STRATEGY 3 — cross_type:
    RETURN all Gold specimens across passage types
    INCLUDE Silver specimens from target niche

WHEN campaign_context is NOT available:
  RETURN full specimen set with tier metadata attached
  SORT by passage_richness_score DESC within each passage type

OUTPUT retrieval_result:
  matched_specimens: [specimen]
  match_strategy_used: enum[exact_match, type_match, cross_type, full_set]
  gold_specimens_available: integer
  silver_specimens_available: integer
```
