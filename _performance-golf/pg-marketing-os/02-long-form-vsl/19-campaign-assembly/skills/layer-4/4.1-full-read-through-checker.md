# Microskill 4.1: Full Read-Through Checker

**Layer:** 4 (Validation)
**Purpose:** Execute complete read-through to verify assembled copy works as unified whole

---

## CORE PRINCIPLE: THE SEAMLESS EXPERIENCE

After assembly, transitions, proof insertion, callbacks, and attention maintenance—the copy must read as if it were written in one sitting by a master copywriter. The Full Read-Through Checker evaluates whether the assembled copy achieves this seamless quality.

---

## READ-THROUGH PROCESS

### Step 1: Assemble Complete Copy

Combine all sections into single document for continuous reading:

```yaml
complete_copy:
  sections:
    - lead: "[full lead copy]"
    - story: "[full story copy]"
    - root_cause: "[full root cause copy]"
    - mechanism: "[full mechanism copy]"
    - proof: "[full proof section copy]"
    - product: "[full product copy]"
    - offer: "[full offer copy]"
    - close: "[full close copy]"
    - ps: "[P.S. elements]"

  total_word_count: "[count]"
  estimated_read_time: "[minutes]"
```

### Step 2: Read Without Stopping

Execute complete read-through WITHOUT making notes or stopping. Objective is to experience copy as reader would:

- Start at headline
- Read continuously to final P.S.
- Note overall impression only

### Step 3: Document First-Pass Impressions

```yaml
first_pass_impressions:
  flow_assessment:
    rating: 1-10
    description: "[overall flow quality]"

  momentum_assessment:
    rating: 1-10
    description: "[did momentum carry through?]"

  seam_visibility:
    rating: 1-10
    description: "[how visible were section transitions?]"

  engagement_consistency:
    rating: 1-10
    description: "[did engagement stay high throughout?]"

  emotional_journey:
    rating: 1-10
    description: "[did emotional arc work?]"

  overall_impression:
    feels_unified: true | false
    feels_crafted: true | false
    feels_persuasive: true | false
    notes: "[key observations]"
```

## SECOND PASS: ISSUE IDENTIFICATION

### Read-Through Again with Issue Spotting

| Issue Type | What to Flag |
|------------|--------------|
| Flow Break | Where reading momentum stopped |
| Confusion Point | Where meaning became unclear |
| Redundancy | Where same point repeated unnecessarily |
| Tone Shift | Where voice/register changed abruptly |
| Energy Drop | Where copy felt flat or boring |
| Jarring Transition | Where section change felt forced |
| Missing Link | Where logical connection was unclear |
| Overwritten | Where copy felt bloated or excessive |
| Underwritten | Where copy felt thin or rushed |

### Issue Logging Format

```yaml
issues_identified:
  - issue_id: "RI_001"
    type: "flow_break"
    location: "[section + approximate position]"
    severity: CRITICAL | MAJOR | MINOR
    description: "[what happened]"
    surrounding_context: "[copy before and after]"
    suggested_fix: "[how to address]"

  - issue_id: "RI_002"
    type: "tone_shift"
    location: "[section + position]"
    # ...
```

## QUALITY THRESHOLDS

| Metric | Pass Threshold | Fail Threshold |
|--------|---------------|----------------|
| Critical Issues | 0 | 1+ |
| Major Issues | ≤3 | >5 |
| Minor Issues | ≤10 | >15 |
| Average Section Score | ≥7 | <5 |
| Flow Rating | ≥7 | <5 |
| Unified Copy Score | ≥70 | <50 |
