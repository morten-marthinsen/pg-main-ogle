# Microskill 4.2: Attention Audit

**Layer:** 4 (Validation)
**Purpose:** Final audit of attention-maintenance throughout assembled copy

---

## CORE FUNCTION

The Attention Audit provides final verification that the assembled copy will maintain reader engagement from headline through final P.S. This goes beyond pacing verification to assess actual attention-holding power.

---

## ATTENTION AUDIT DIMENSIONS

### 1. Hook Strength Audit

Assess every attention hook for effectiveness:

```yaml
hook_audit:
  hooks_identified: "[count]"

  by_hook:
    - hook_id: "H_001"
      location: "[section + position]"
      hook_text: "[the hook copy]"
      hook_type: "[technique used]"

      strength_assessment:
        curiosity_factor: 1-10
        forward_pull: 1-10
        natural_integration: 1-10
        overall_strength: STRONG | MODERATE | WEAK

      verdict: EFFECTIVE | NEEDS_STRENGTHENING | REMOVE

  summary:
    strong_hooks: "[count]"
    moderate_hooks: "[count]"
    weak_hooks: "[count]"
    hooks_needing_work: "[count]"
```

### 2. Momentum Audit

Map momentum flow throughout copy:

```yaml
momentum_audit:
  momentum_map:
    - section: "lead"
      entry_momentum: N/A (start)
      peak_momentum: "[1-10]"
      exit_momentum: "[1-10]"
      momentum_trend: BUILDING | STEADY | DECLINING

    - section: "story"
      entry_momentum: "[carries from lead]"
      peak_momentum: "[1-10]"
      exit_momentum: "[1-10]"
      momentum_trend: BUILDING | STEADY | DECLINING

    # ... all sections

  momentum_valleys:
    - location: "[section + position]"
      momentum_level: "[1-10]"
      cause: "[why momentum dropped]"
      intervention_needed: true | false

  overall_momentum_pattern:
    starts_strong: true | false
    maintains_through_middle: true | false
    peaks_at_close: true | false
    pattern_assessment: OPTIMAL | ACCEPTABLE | PROBLEMATIC
```

### 3. Fatigue Point Audit

Identify where reader fatigue is likely:

```yaml
fatigue_audit:
  fatigue_risk_points:
    - location: "[section + position]"
      word_count_at_this_point: "[count]"
      fatigue_type: "cognitive_load | emotional_drain | monotony | length"
      risk_level: HIGH | MEDIUM | LOW
      mitigation_present: true | false
      mitigation_type: "[what's preventing fatigue]"

  high_risk_points: "[count]"
  mitigated_points: "[count]"
  unmitigated_points: "[count]"

  fatigue_assessment: LOW_RISK | MODERATE_RISK | HIGH_RISK
```

### 4. Curiosity Loop Audit

Verify open-close pattern of curiosity:

```yaml
curiosity_audit:
  loops_identified:
    - loop_id: "CL_001"
      opened_at: "[section + position]"
      opening_text: "[curiosity trigger]"
      closed_at: "[section + position]"
      closing_text: "[satisfaction text]"
      span: "[words between open and close]"
      loop_status: OPENED_AND_CLOSED | STILL_OPEN | CLOSED_TOO_QUICKLY

  loop_summary:
    total_loops: "[count]"
    properly_closed: "[count]"
    still_open_at_end: "[count]"
    closed_too_quickly: "[count]"

  major_curiosity_thread:
    main_mystery: "[the big question being answered]"
    opened_in: "[section]"
    resolved_in: "[section]"
    maintains_interest_throughout: true | false
```

### 5. Value Delivery Audit

Assess if reader feels rewarded for attention:

```yaml
value_audit:
  value_moments:
    - location: "[section + position]"
      value_type: "insight | revelation | proof | benefit | hope"
      value_description: "[what reader receives]"
      value_strength: HIGH | MEDIUM | LOW

  value_distribution:
    lead: "[count] value moments"
    story: "[count]"
    root_cause: "[count]"
    mechanism: "[count]"
    proof: "[count]"
    product: "[count]"
    offer: "[count]"
    close: "[count]"

  value_density:
    average_words_between_value: "[count]"
    max_words_without_value: "[count]"
    sections_below_value_threshold: [list]

  value_assessment: REWARDING | ADEQUATE | THIN
```

## ATTENTION DANGER SIGNALS

### Red Flags to Check

| Danger Signal | Threshold | Check |
|---------------|-----------|-------|
| Long exposition without break | >400 words | `[ ]` |
| Technical explanation without proof | >200 words | `[ ]` |
| List without narrative | >5 items | `[ ]` |
| Repetitive structure | Same sentence pattern 3x | `[ ]` |
| Monotonous rhythm | No variation in 300+ words | `[ ]` |
| Unearned ask | CTA without sufficient proof/value | `[ ]` |
| Emotional flatline | 500+ words at same intensity | `[ ]` |

### Danger Signal Scan

```yaml
danger_signal_scan:
  signals_detected:
    - signal: "long_exposition"
      location: "[section + position]"
      span: "[X] words"
      severity: CRITICAL | MAJOR | MINOR
      intervention_present: true | false

  total_signals: "[count]"
  critical_signals: "[count]"
  signals_without_intervention: "[count]"
```
