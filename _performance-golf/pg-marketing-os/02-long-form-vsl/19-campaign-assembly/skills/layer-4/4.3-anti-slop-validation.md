# Microskill 4.3: Anti-Slop Validation

**Layer:** 4 (Validation)
**Purpose:** Detect and eliminate weak, generic, or "slop" copy that degrades quality

---

## CORE PRINCIPLE: SLOP KILLS CREDIBILITY

"Slop" is copy that feels AI-generated, generic, or lazy. It includes:
- Filler phrases that add no value
- Generic claims that could apply to any product
- Weak verbs and passive constructions
- Clichés and overused expressions
- Hedging language that undermines confidence
- Purple prose that obscures meaning

**TIER1 Observation:** Elite copy is tight, specific, and confident. Every sentence earns its place.

---

## SLOP CATEGORIES

### Category 1: Filler Phrases

Phrases that add words but not meaning:

| Slop Phrase | Problem | Better Alternative |
|-------------|---------|-------------------|
| "In order to" | Wordy | "To" |
| "The fact that" | Empty | Delete or restructure |
| "It is important to note that" | Throat-clearing | Just state the note |
| "As a matter of fact" | Padding | Delete |
| "In today's world" | Generic | Be specific about context |
| "When it comes to" | Delay | Get to the point |
| "At the end of the day" | Cliché | Be specific |
| "The truth is" | Overused | Just state the truth |

### Category 2: Generic Claims

Claims so vague they could apply to anything:

| Generic Claim | Problem | Specific Alternative |
|---------------|---------|---------------------|
| "Revolutionary approach" | Meaningless | State what's revolutionary |
| "Cutting-edge technology" | Empty | Name the technology |
| "Proven results" | Unsubstantiated | Cite specific results |
| "Unlike anything you've seen" | Lazy | Describe the difference |
| "The secret that X don't want you to know" | Overused trope | State the actual insight |
| "Life-changing solution" | Hyperbole | Describe the change |

### Category 3: Weak Constructions

Language that undermines confidence:

| Weak Construction | Problem | Stronger Alternative |
|-------------------|---------|---------------------|
| "May help" | Hedging | "Helps" (if true) |
| "Could potentially" | Double hedge | "Can" |
| "It seems like" | Uncertain | State with confidence |
| "In my opinion" | Unnecessary | Opinion is implied |
| "I believe that" | Weakening | State directly |
| Passive voice | Distance | Active voice |

### Category 4: AI-Generated Tells

Patterns that reveal AI generation:

| AI Tell | Example | Fix |
|---------|---------|-----|
| List addiction | "1. First... 2. Second... 3. Third..." | Vary structure |
| Transition overload | "Furthermore... Moreover... Additionally..." | Natural flow |
| Perfect parallelism | Every sentence same structure | Vary rhythm |
| Emoji abuse | Emojis in serious copy | Remove |
| Excessive enthusiasm | "Absolutely amazing!!!" | Tone down |
| Robotic conclusions | "In conclusion..." | End naturally |

### Category 5: Clichés & Overused Phrases

Expressions so common they've lost impact:

| Cliché | Problem | Fresh Alternative |
|--------|---------|------------------|
| "Think outside the box" | Dead metaphor | Specific example |
| "Take it to the next level" | Meaningless | Describe the level |
| "Game-changer" | Overused | Specific impact |
| "Best-kept secret" | Contradictory | Just share the insight |
| "Unlock your potential" | Vague | Specific capability |
| "Transform your life" | Generic | Specific transformation |

## SLOP DETECTION PROCESS

### Step 1: Automated Pattern Scan

Scan for known slop patterns:

```yaml
pattern_scan:
  filler_phrases_found:
    - phrase: "[phrase]"
      location: "[section + position]"
      count: "[times found]"

  generic_claims_found:
    - claim: "[claim]"
      location: "[section + position]"

  weak_constructions_found:
    - construction: "[construction]"
      location: "[section + position]"

  ai_tells_found:
    - tell: "[pattern]"
      location: "[section + position]"

  cliches_found:
    - cliche: "[phrase]"
      location: "[section + position]"

  total_slop_instances: "[count]"
```

### Step 2: Specificity Audit

Check that claims are backed by specifics:

```yaml
specificity_audit:
  claims_made: "[count]"

  by_claim:
    - claim: "[claim statement]"
      location: "[section + position]"
      specific_support: true | false
      support_type: "number | example | proof | mechanism"
      specificity_score: 1-10

  claims_without_specifics: "[count]"
  average_specificity_score: "[1-10]"
```

### Step 3: Confidence Audit

Check for hedging and weak language:

```yaml
confidence_audit:
  hedges_found: "[count]"
  passive_constructions: "[count]"
  weakening_phrases: "[count]"

  hedges_list:
    - phrase: "[hedge]"
      location: "[section + position]"
      justified: true | false  # some hedges are legally necessary
      recommendation: REMOVE | REWRITE | KEEP

  overall_confidence_level: STRONG | MODERATE | WEAK
```

### Step 4: Originality Check

Assess if copy feels fresh vs. templated:

```yaml
originality_check:
  templated_sections: [list of sections that feel formulaic]
  fresh_sections: [list of sections that feel original]

  distinctive_phrases:
    - phrase: "[memorable phrase]"
      section: "[name]"
      # These are GOOD - signs of non-slop

  generic_sections:
    - section: "[name]"
      reason: "[why it feels generic]"

  originality_score: "[1-10]"
```

---

## SLOP SEVERITY LEVELS

| Severity | Criteria | Action |
|----------|----------|--------|
| CRITICAL | Slop in headline, lead hook, or CTA | MUST FIX before assembly complete |
| MAJOR | Slop in mechanism explanation or proof | SHOULD FIX |
| MINOR | Slop in transitional copy or supporting text | FIX IF TIME |
| STYLISTIC | Slight clichés that don't undermine credibility | OPTIONAL FIX |

---

## REMEDIATION PROTOCOL

For each slop instance:

```yaml
slop_fix:
  instance_id: "SLOP_001"
  location: "[section + position]"
  slop_type: "[category]"
  severity: "[level]"

  current_text: "[the slop]"
  issue: "[what's wrong]"
  fixed_text: "[rewritten version]"
  fix_applied: true | false
```

slop_score:

  category_breakdown:

    generic_claims: "[count]"
    weak_constructions: "[count]"
    ai_tells: "[count]"
    cliches: "[count]"

  specificity_score: "[1-10]"
  confidence_score: "[1-10]"
  originality_score: "[1-10]"

  composite_slop_score: "[1-100]"  # Higher = less slop = better
