# Microskill 3.2: Terminology Consistency Checker

**Layer:** 3 (Refinement & Audit)
**Purpose:** Ensure terminology remains consistent throughout assembled copy

---

## CORE PRINCIPLE: ONE NAME, ONE CONCEPT

Inconsistent terminology confuses readers and breaks persuasion momentum. When the mechanism is called "The Metabolic Reset" in one section and "metabolic reset protocol" in another, the reader subconsciously questions if they're the same thing.

**TIER1 Observation:** Elite controls establish terminology early and maintain it religiously. Variations exist but are clearly connected to the primary term.

---

## CONSISTENCY CHECK PROCESS

### Step 1: Build Terminology Registry

Load from upstream packages:

```yaml
terminology_registry:
  mechanism:
    primary: "The Metabolic Reset"
    capitalization: "Title Case"
    article: "The"
    variations_allowed:
      - "Metabolic Reset" (without article in mid-sentence)
      - "the Reset" (shorthand after establishment)
    variations_forbidden:
      - "metabolic reset" (wrong capitalization)
      - "Metabolic Reset Protocol" (name change)
      - "Reset system" (name alteration)

  product:
    primary: "MetaboFlex Formula"
    capitalization: "Title Case"
    variations_allowed:
      - "MetaboFlex"
      - "the Formula"
    variations_forbidden:
      - "Metabo Flex" (spacing)
      - "METABOFLEX" (all caps)
      - "metaboflex" (no caps)

  # ... for each critical term
```

### Step 2: Scan Assembled Copy

For each critical term:

1. Find all occurrences in assembled copy
2. Record exact form used
3. Flag any variations not in allowed list
4. Flag any forbidden variations

```yaml
terminology_scan:
  mechanism:
    occurrences:
      - section: "lead"
        position: "[word count reference]"
        exact_text: "The Metabolic Reset"
        status: CORRECT

      - section: "mechanism"
        position: "[reference]"
        exact_text: "metabolic reset"
        status: INCORRECT (capitalization)

      # ... all occurrences

  product:
    # ... same structure
```

### Step 3: Identify Inconsistencies

```yaml
inconsistencies:
  - term_category: "mechanism"
    primary: "The Metabolic Reset"
    incorrect_usage: "metabolic reset"
    section: "mechanism"
    location: "[reference]"
    issue: "Missing capitalization"
    fix: "Change to 'The Metabolic Reset' or 'the Reset'"

  - term_category: "product"
    primary: "MetaboFlex Formula"
    incorrect_usage: "Metabo Flex"
    section: "offer"
    location: "[reference]"
    issue: "Incorrect spacing"
    fix: "Remove space: 'MetaboFlex'"
```

## SPECIFIC CONSISTENCY RULES

### Capitalization

```
[ ] Mechanism name: Consistent case throughout
[ ] Product name: Consistent case throughout
[ ] Framework name: Consistent case throughout
[ ] Villain name: Consistent case (often Title Case or ALL CAPS)
```

### Article Usage

```
[ ] "The [Mechanism]" vs "[Mechanism]" - consistent pattern
[ ] "The [Product]" vs "[Product]" - consistent pattern
[ ] Articles used appropriately (first mention with "The", later optional)
```

### Spacing and Punctuation

```
[ ] Multi-word names: Consistent spacing
[ ] Hyphenation: Consistent throughout
[ ] No errant punctuation within names
```

### Abbreviation/Shorthand

```
[ ] Shorthand only used AFTER full term established
[ ] Shorthand is clearly connected to primary term
[ ] Shorthand is consistent (always same short form)
```

## ADDITIONAL CONSISTENCY CHECKS

### Number Formatting

```yaml
number_consistency:
  rule: "[chosen format]"
  examples:
    - "47,000" not "47000"
    - "$97" not "$97.00" (unless .97 or similar)
    - "3-step" not "three-step" (after first use)
```

### Time References

```yaml
time_consistency:
  rule: "[chosen format]"
  examples:
    - "21 days" not "three weeks" (unless switching deliberately)
    - "60 seconds" not "one minute" (if precision intended)
```

### Pronoun Consistency

```yaml
pronoun_consistency:
  narrator: "I" throughout (not switching to "we")
  reader: "you" throughout (not switching to "one")
  company: "we" if used (consistent "we" vs "I")
```

## REMEDIATION PROTOCOL

For each inconsistency:

1. Identify correct term from registry
2. Locate all incorrect usages
3. Edit to correct form
4. Verify no new inconsistencies created

```yaml
remediation_action:
  term: "mechanism"
  incorrect: "metabolic reset"
  correct: "The Metabolic Reset"
  locations:
    - section: "mechanism"
      line: "[reference]"
      before: "...using the metabolic reset to..."
      after: "...using The Metabolic Reset to..."
```
