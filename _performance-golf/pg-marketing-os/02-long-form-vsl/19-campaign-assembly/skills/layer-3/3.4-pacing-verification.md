# Microskill 3.4: Pacing Verification

**Layer:** 3 (Refinement & Audit)
**Purpose:** Verify pacing plan is properly executed throughout assembled copy

---

## CORE FUNCTION

Pacing Verification confirms that:
1. Intensity patterns match the pacing plan
2. Re-engagement hooks are placed at planned intervals
3. Danger zones have proper interventions
4. Progressive density toward close is achieved
5. No attention dead zones exist

---

## PACING VERIFICATION PROCESS

### Step 1: Load Pacing Plan

From 1.4-pacing-planner output:

```yaml
pacing_plan_load:
  format: "[VSL | Magalog | Sales Letter]"

  section_patterns:
    lead: "HOOK → BUILD → PEAK"
    story: "REST → BUILD → PEAK"
    root_cause: "BUILD → PEAK → REST"
    mechanism: "BUILD → PEAK → BUILD → PEAK"
    proof: "BUILD → PEAK → REST → BUILD → PEAK"
    product: "REST → BUILD → PEAK"
    offer: "BUILD → PEAK → BUILD → PEAK"
    close: "BUILD → PEAK → PEAK → PEAK"

  danger_zones: [list with planned interventions]
  re_engagement_points: [list with planned hooks]
```

### Step 2: Verify Intensity Patterns

For each section, analyze if actual copy matches planned intensity:

```yaml
intensity_verification:
  lead:
    planned_pattern: "HOOK → BUILD → PEAK"
    actual_pattern: "[analysis of copy]"
    match_assessment: MATCHES | PARTIAL | MISMATCH
    notes: "[specific observations]"

  story:
    planned_pattern: "REST → BUILD → PEAK"
    actual_pattern: "[analysis]"
    # ...

  # ... for each section
```

### Intensity Pattern Markers

| Marker | Indicators in Copy |
|--------|-------------------|
| HOOK | Opening question, bold claim, curiosity trigger |
| BUILD | Rising stakes, accumulating evidence, tension increase |
| PEAK | Major reveal, proof cascade, emotional climax, urgency spike |
| REST | Story moment, relatable aside, breathing room, summary |
| BRIDGE | Neutral transition, topic shift, connector |

### Step 3: Verify Re-Engagement Hooks

Check that planned attention hooks exist at specified locations:

```yaml
reengagement_verification:
  hooks_planned: "[count]"
  hooks_found: "[count]"

  by_location:
    - planned_location: "[section + position]"
      expected_hook: "[type or specific text]"
      found: true | false
      actual_text: "[what was found]"
      effectiveness: STRONG | MODERATE | WEAK | MISSING

  coverage:
    words_between_hooks_average: "[count]"
    max_gap_without_hook: "[count] words"
    sections_below_target_density: [list]
```

### Step 4: Verify Danger Zone Interventions

```yaml
danger_zone_verification:
  zones_planned: "[count]"
  zones_addressed: "[count]"

  by_zone:
    - zone: "After first story section"
      planned_intervention: "Curiosity loop to mechanism"
      intervention_found: true | false
      actual_intervention: "[what was found]"
      effectiveness: EFFECTIVE | PARTIAL | MISSING

    - zone: "Mid-mechanism explanation"
      planned_intervention: "Proof break"
      # ...
```

### Step 5: Verify Progressive Density

Attention technique density should increase toward close:

```yaml
progressive_density_verification:
  by_section:
    lead:
      word_count: "[count]"
      attention_techniques: "[count]"
      density: "1 per [X] words"

    story:
      # ...

    # ... all sections

    close:
      word_count: "[count]"
      attention_techniques: "[count]"
      density: "1 per [X] words"

  progression_check:
    early_sections_lighter: true | false
    mid_sections_moderate: true | false
    close_densest: true | false
    pattern_assessment: CORRECT | NEEDS_ADJUSTMENT
```

## ATTENTION DEAD ZONE DETECTION

Identify any stretches without engagement triggers:

```yaml
dead_zone_scan:
  threshold: 500  # words without attention technique

  dead_zones_found:
    - section: "[name]"
      start_position: "[word count]"
      end_position: "[word count]"
      span: "[X] words"
      severity: CRITICAL | WARNING
      suggested_intervention: "[technique type]"

  total_dead_zones: "[count]"
  longest_dead_zone: "[X] words in [section]"
```

---

## PACING QUALITY CHECKS

```
[ ] Each section follows its intensity pattern
[ ] PEAK moments feel earned (preceded by BUILD)
[ ] REST sections don't exceed 300 words without re-engagement
[ ] No more than 500 words between attention hooks
[ ] Danger zones have effective interventions
[ ] Close section has highest hook density
[ ] No attention dead zones >500 words
[ ] Progressive density from beginning to end
```

---

## REMEDIATION PROTOCOL

### For Pattern Mismatches:

1. Identify where actual pattern diverges from planned
2. Determine if copy needs intensity adjustment
3. Add BUILD elements before unearned PEAKs
4. Add HOOK or attention technique at REST overruns

### For Missing Re-Engagement Hooks:

1. Locate gaps exceeding 500 words
2. Identify natural insertion points
3. Draft appropriate attention technique
4. Insert and verify flow

### For Dead Zones:

1. Map the dead zone boundaries
2. Identify what's causing attention drain
3. Insert 1-2 attention techniques within zone
4. Consider if copy needs restructuring
