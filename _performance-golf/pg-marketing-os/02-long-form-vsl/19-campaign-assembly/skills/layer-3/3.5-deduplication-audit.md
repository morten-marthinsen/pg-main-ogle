# 3.5 Deduplication Audit

**Version:** 1.0
**Created:** 2026-02-06
**Purpose:** Detect and flag substantially similar content across assembled sections BEFORE final packaging
**Position:** Layer 3 (After threading/callbacks, before validation)
**Status:** ACTIVE — CRITICAL GATE

---

## WHY THIS MICROSKILL EXISTS

**Catastrophic Failure Pattern Discovered (Ultra Liver 2026-02-05):**

The Ultra Liver campaign passed through Assembly with an A- grade despite having:
- The 3-phase explanation repeated in STORY, ROOT CAUSE, and MECHANISM sections
- The car wash metaphor used TWICE (Story + Mechanism)
- "Sticky bile" explained FOUR separate times
- Tri-Phase Flow ingredients explained THREE times

This happened because:
1. Upstream skills (11-17) each draft their section independently
2. Each skill tries to be "complete" — explaining the mechanism, root cause, etc.
3. Assembly concatenated without detecting the overlap
4. Editorial evaluated in silos, missing the macro-level repetition

**Result:** Copy that reads like three separate documents stitched together, not one flowing narrative.

---

## PURPOSE

Scan the assembled draft for substantially similar content across sections. Flag duplicates for:
1. Consolidation (keep one, remove others)
2. Restructuring (move content to appropriate section)
3. Bridging (turn repetition into callbacks)
4. Human review (when consolidation choice is unclear)

---

## DEDUPLICATION DETECTION CRITERIA

### Tier 1: CRITICAL DUPLICATIONS (Must be resolved)

**Definition:** Same core concept explained in substantially similar ways in 2+ sections.

**Detection Signals:**
- Same subject + same explanation structure
- Same metaphor/analogy used in multiple sections
- Same numbered list of items (e.g., "Phase 1, Phase 2, Phase 3")
- Same proof point cited multiple times
- Same "why this matters" framing repeated

**Similarity Threshold:** >50% semantic overlap = CRITICAL duplication

### Tier 2: MODERATE DUPLICATIONS (Should be reviewed)

**Definition:** Same concept touched on in multiple sections but with different framing.

**Detection Signals:**
- Same subject but different emphasis
- Same mechanism mentioned but different depth
- Same problem stated with different symptom focus

**Similarity Threshold:** 30-50% semantic overlap = MODERATE duplication

### Tier 3: ACCEPTABLE THREADING (Not a problem)

**Definition:** Strategic repetition that reinforces the message.

**Acceptable Patterns:**
- Mechanism NAME mentioned across sections (this is threading, not duplication)
- Root cause ANCHOR mentioned across sections
- Promise ECHOED in close (this is callback, not duplication)
- Brief reminder before building on concept

**Key Distinction:** Threading = SAME PHRASE, different context. Duplication = SAME EXPLANATION repeated.

## COMMON DUPLICATION PATTERNS TO DETECT

### Pattern 1: Mechanism Explained Multiple Times

**Symptom:** Story explains mechanism, Root Cause explains mechanism, Mechanism section explains mechanism AGAIN.

**Detection:** Look for same numbered lists, same component names, same process descriptions.

**Fix:** Mechanism explained ONCE in Mechanism section. Story and Root Cause BUILD TOWARD it without explaining it.

### Pattern 2: Same Metaphor in Multiple Sections

**Symptom:** "Think of it like a car wash" appears in Story AND Mechanism.

**Detection:** Scan for metaphor/analogy language patterns.

**Fix:** ONE metaphor per concept. If a better metaphor exists in a later section, remove the earlier one.

### Pattern 3: Root Cause = Mechanism Preview

**Symptom:** Root Cause section basically explains the mechanism, then Mechanism section explains it again.

**Detection:** Compare root cause explanation with mechanism explanation for semantic overlap.

**Fix:** Root Cause identifies the PROBLEM. Mechanism provides the SOLUTION. They should not overlap significantly.

### Pattern 4: Product = Mechanism Repeat

**Symptom:** Product Introduction re-explains the mechanism before revealing the product.

**Detection:** Compare mechanism narrative with product introduction narrative.

**Fix:** Product Introduction ASSUMES reader understands mechanism (from previous section). It reveals the product as the DELIVERY VEHICLE for the mechanism, not re-explains the mechanism.
