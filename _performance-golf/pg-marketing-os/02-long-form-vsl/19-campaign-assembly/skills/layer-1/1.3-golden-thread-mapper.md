# Microskill 1.3: Golden Thread Mapper

**Layer:** 1 (Sequencing & Proportion)
**Purpose:** Map where threading elements MUST appear to maintain coherence

---

## CORE PRINCIPLE: THE GOLDEN THREAD

The "Golden Thread" is the core theme/big idea that must be woven throughout the copy. Without it, sections feel disconnected. With it, the copy feels like one unified persuasion journey.

---

## THREADING ELEMENTS TO TRACK

### Primary Threads (Must Track)

| Element | Source | Minimum Count | Distribution Required |
|---------|--------|---------------|----------------------|
| Mechanism Name | mechanism-package | 8+ | Every section except P.S. |
| Root Cause Anchor | root-cause-package | 5+ | Lead, Root Cause, Mechanism, Close |
| Framework/System | mechanism-package | 4+ | Mechanism, Product, Offer, Close |
| Primary Promise | lead-package/promise | 6+ | Lead, Story, Mechanism, Product, Offer, Close |

### Secondary Threads (Should Track)

| Element | Source | Minimum Count | Distribution Required |
|---------|--------|---------------|----------------------|
| Villain | root-cause-package | 3+ | Root Cause, Mechanism, Close |
| Product Name | product-package | 4+ | Product, Offer, Close |
| Transformation Image | story-package | 2+ | Story, Close |

## MAPPING PROCESS

### Step 1: Extract Thread Elements

```yaml
threads:
  mechanism_name:
    exact_term: "[from mechanism-package]"
    variations_allowed: [list of acceptable variations]

  root_cause_anchor:
    exact_phrase: "[from root-cause-package]"
    variations_allowed: [list]

  framework_name:
    exact_term: "[from mechanism-package]"
    variations_allowed: [list]

  primary_promise:
    core_promise: "[from lead/promise-package]"
    variations_allowed: [list]

  villain:
    exact_name: "[from root-cause-package]"
    references: [industry, companies, etc.]
```

### Step 2: Map Required Appearances

```yaml
thread_map:
  mechanism_name:
    - section: lead
      required: true
      placement_note: "Early mention to establish"

    - section: story
      required: true
      placement_note: "Protagonist discovers mechanism"

    - section: root_cause
      required: true
      placement_note: "Mechanism contrasts with root cause"

    - section: mechanism
      required: true
      count: 3+
      placement_note: "Central to section"

    - section: proof
      required: true
      placement_note: "Proof validates mechanism"

    - section: product
      required: true
      placement_note: "Product delivers mechanism"

    - section: offer
      required: true
      placement_note: "Value tied to mechanism"

    - section: close
      required: true
      placement_note: "Final mechanism callback"
```

### Step 3: Identify Threading Gaps

For each section, check current content for thread appearances:

```yaml
gap_analysis:
  lead:
    mechanism_name: [found | missing]
    root_cause_anchor: [found | missing]
    primary_promise: [found | missing]

  story:
    # ... check all required threads

  # ... for each section
```

## THREADING INSERTION POINTS

Where threads should naturally appear:

### Mechanism Name Insertion Points

| Section | Natural Insertion Point |
|---------|------------------------|
| Lead | After hook, before story setup |
| Story | When protagonist discovers solution |
| Root Cause | When contrasting failed approaches |
| Mechanism | Throughout explanation |
| Proof | When explaining why proof subjects succeeded |
| Product | When explaining what product delivers |
| Offer | When restating value |
| Close | In benefit summary, CTAs |

### Root Cause Anchor Insertion Points

| Section | Natural Insertion Point |
|---------|------------------------|
| Lead | Hint at real problem |
| Root Cause | Central to section |
| Mechanism | When explaining why mechanism works |
| Close | Callback to why nothing else worked |

### Promise Insertion Points

| Section | Natural Insertion Point |
|---------|------------------------|
| Lead | Opening promise |
| Story | What protagonist achieved |
| Mechanism | What mechanism delivers |
| Product | Product promise |
| Offer | Value restatement |
| Close | Final promise callback |
