## LOADING PROTOCOL

```
STEP 1: Verify all 11 packages exist
  FOR each package in required_list:
    CHECK: File exists in project outputs folder
    IF missing: LOG error, note which skill must run first

STEP 2: Load and validate each package
  FOR each package:
    LOAD: Parse JSON/YAML
    VALIDATE: Required fields present
    EXTRACT: Key elements for assembly

STEP 3: Create unified context
  MERGE: All extracted elements into assembly_context
  INDEX: Section copy by section name
  INDEX: Proof blocks by target section
  INDEX: Threading elements for audit
```

## VALIDATION RULES

### Critical Failures (Blocking)

```
IF any of the 11 packages is missing:
  → STOP: Cannot assemble without all sections
  → LIST: Missing packages
  → INSTRUCT: Which skills must run first

IF any package has empty/null required fields:
  → STOP: Incomplete upstream execution
  → IDENTIFY: Specific missing fields
  → INSTRUCT: Return to that skill
```

### Warnings (Non-Blocking)

```
IF word counts exceed structure targets by >20%:
  → WARN: Section over-length may require trimming
  → NOTE: Which sections are over

IF proof blocks fewer than structure required:
  → WARN: Proof density may be below target
  → NOTE: Which sections are light
```
