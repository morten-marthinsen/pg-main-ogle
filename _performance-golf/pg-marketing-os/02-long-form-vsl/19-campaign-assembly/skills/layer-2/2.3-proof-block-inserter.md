# Microskill 2.3: Proof Block Inserter

**Layer:** 2 (Assembly)
**Purpose:** Insert proof blocks from proof-weaving-package at designated placements

---

## CORE FUNCTION

The Proof Block Inserter takes the pre-drafted, pre-sequenced proof blocks from the proof-weaving-package and inserts them at the exact positions specified. This is INSERTION, not writing—the proof is already written.

---

## INPUT REQUIREMENTS

### From proof-weaving-package.json:

```json
{
  "proof_blocks": {
    "lead": [...],
    "story": [...],
    "root_cause": [...],
    "mechanism": [...],
    "proof_section": [...],
    "product": [...],
    "offer": [...],
    "close": [...]
  }
}
```

Each proof block contains:
- `id`: Unique identifier
- `content`: Full drafted proof copy
- `placement.position`: Where in section
- `placement.transition_in`: Lead-in copy
- `placement.transition_out`: Exit copy

---

## INSERTION PROCESS

### Step 1: Load Proof Blocks by Section

```yaml
proof_load:
  section: "[name]"
  blocks_assigned: "[count]"
  blocks:
    - id: "[PROOF_XXX_001]"
      position: "[specific placement]"
      word_count: "[count]"
```

### Step 2: Identify Insertion Points in Assembled Copy

For each section, mark exact insertion points:

```yaml
insertion_points:
  section: "lead"
  points:
    - marker: "After hook, before story setup"
      line_reference: "[approximate line or paragraph]"
      proof_block_id: "PROOF_LEAD_001"

    - marker: "[next position]"
      # ...
```

### Step 3: Execute Insertion

For each proof block:

1. Locate insertion point in assembled section
2. Insert `transition_in` copy
3. Insert proof block `content`
4. Insert `transition_out` copy
5. Verify flow reads naturally

### Step 4: Validate Insertion

```yaml
insertion_validation:
  proof_id: "[PROOF_XXX_001]"
  section: "[name]"
  position: "[where inserted]"

  checks:
    transition_in_present: true | false
    content_complete: true | false
    transition_out_present: true | false
    flow_reads_naturally: true | false
    no_orphaned_text: true | false
```

---

## INSERTION RULES

### Positioning Rules

| Position Marker | Interpretation |
|-----------------|----------------|
| "opening" | After section transition, before main content |
| "mid-section" | Approximately 40-60% through section |
| "after [element]" | Immediately following named element |
| "before [element]" | Immediately preceding named element |
| "closing" | After main content, before transition out |

### Flow Maintenance Rules

```
[ ] Transition_in connects to preceding copy
[ ] Proof block content stands complete
[ ] Transition_out connects to following copy
[ ] No awkward breaks or orphaned sentences
[ ] Proof doesn't interrupt mid-thought
[ ] Reading aloud sounds natural
```

### Spacing Rules

```
[ ] Appropriate paragraph breaks before/after proof
[ ] Proof visually distinguished if format requires
[ ] No double-spacing or inconsistent formatting
[ ] Proof density doesn't overwhelm section
```

---

## SECTION-SPECIFIC INSERTION NOTES

### Lead Proof Insertion

- Light proof only (authority, scale)
- Don't interrupt hook momentum
- Proof should AMPLIFY curiosity, not satisfy it

### Story Proof Insertion

- Embedded within narrative flow
- Protagonist's transformation proof
- Before/after moments

### Root Cause Proof Insertion

- Data supporting the problem
- Logic proof for root cause claim
- Position AFTER root cause revealed

### Mechanism Proof Insertion

- Study citations supporting mechanism
- Demonstration proof
- Multiple insertion points (mechanism is proof-heavy)

### Proof Section Insertion

- This IS the proof—load the full cascade
- Maintain sequencing from proof-weaving-package
- Testimonial parade if applicable

### Product Proof Insertion

- Formula/ingredient proof
- Component validation
- Expert endorsements

### Offer Proof Insertion

- Value testimonials
- Guarantee-related proof
- "Worth it" validation

### Close Proof Insertion

- Callback proof (references earlier proof)
- Final authority stamp
- Urgency-supporting proof
