# 2.4-format-adapter.md

> **Layer:** 2 (Generation)
> **Skill:** 10-headlines
> **Purpose:** Adapt headline candidates for specific formats (VSL, print, advertorial)
> **Inputs:** All candidates from 2.1-2.3, `campaign_context.format`
> **Outputs:** Format-adapted versions of top candidates

---

## FUNCTION

Take headline candidates from the three generators (formula, vault, schema) and adapt them for the specific campaign FORMAT. Different formats have different constraints, and a great print headline might fail as a VSL headline.

**Key Principle:** Format is not cosmetic — it's structural. A VSL headline must work above a video. A print headline can be longer. An advertorial must feel editorial.

---

## FORMAT SPECIFICATIONS

### VSL (Video Sales Letter) Headlines

```yaml
CONSTRAINTS:
  word_count: 8-15 words (shorter is better)
  position: Text overlay above video OR first spoken words
  must_work_without: Subheadline (often no room)
  reader_state: Watching video, limited patience
  attention_span: 3-5 seconds to hook

FAVORED_PATTERNS:
  - Questions (engage immediately)
  - Bold claims (stop the scroll)
  - First-person hooks ("I discovered...")
  - Curiosity gaps (must watch to resolve)

AVOID:
  - Long headlines (won't fit, won't read)
  - Complex structures (too much to process)
  - Headlines requiring subhead to work
  - Subtle or clever wordplay (lost in video format)

EXAMPLES:
  GOOD: "What If Everything You Know About Weight Loss Is Wrong?"
  GOOD: "The 2-Minute Morning Ritual That Melted 47 Pounds"
  BAD: "Discover the Revolutionary New Approach to Sustainable Weight Management That's Changing How Nutritionists Think About Metabolism" (too long)
```

### Print/Magalog Headlines

```yaml
CONSTRAINTS:
  word_count: 12-25 words (can be longer)
  position: Cover or above-fold, with subheadline expected
  must_work_with: Subheadline and deck copy as unit
  reader_state: Holding physical piece, can re-read
  attention_span: 5-10 seconds, eye tracks headline → sub → deck

FAVORED_PATTERNS:
  - Benefit statements with specificity
  - Authority reveals
  - Warning patterns (print feels more "official")
  - Multi-part promises

ALLOWED:
  - Longer headlines (reader can process)
  - Complex structures (reader can re-read)
  - Headlines that set up subheadline
  - Em dashes, ellipses, colons

EXAMPLES:
  GOOD: "Shield Your Wealth From Certain Annihilation And Grow Up To 10 Times Richer In The Coming SLAUGHTER of The Stock Market"
  GOOD: "IMMUTOL: How seven sick salmon, unconventional wisdom, serendipity, and a beer brewery saved an economy, healed the hopeless, and changed the course of modern Immunology"
```

### Advertorial Headlines

```yaml
CONSTRAINTS:
  word_count: 10-18 words
  position: Appears as article headline
  must_feel: Editorial, not promotional
  reader_state: Thinks they're reading news/content
  attention_span: Article-reader mode (more patient if engaged)

FAVORED_PATTERNS:
  - News/discovery framing
  - "Research reveals..." patterns
  - Third-person authority
  - Story hooks

FORBIDDEN:
  - Obvious promotional language
  - Exclamation points
  - Hyperbolic claims
  - "Buy now" or urgency signals

EXAMPLES:
  GOOD: "New Study Reveals Why Some People Never Get Sick"
  GOOD: "The Unusual Reason This Small Town Has Zero Heart Disease"
  BAD: "AMAZING New Supplement Will Transform Your Health!" (too promotional)
```

### Email Subject Lines

```yaml
CONSTRAINTS:
  word_count: 5-10 words (shorter = better)
  character_count: Under 50 characters ideally
  position: Inbox, competing with all other emails
  must_work: On mobile (truncation risk)
  reader_state: Scanning inbox quickly

FAVORED_PATTERNS:
  - Curiosity gaps (open loop)
  - Personalization signals
  - Urgency (legitimate)
  - Pattern interrupts

AVOID:
  - SPAM trigger words (free, act now, limited time)
  - All caps
  - Excessive punctuation
  - Generic benefit claims

EXAMPLES:
  GOOD: "the weird reason your diet failed"
  GOOD: "I was wrong about cholesterol"
  BAD: "FREE Report: The Secret to Weight Loss!!!"
```

## EXECUTION PROTOCOL

### Step 1: Identify Campaign Format

```
LOAD campaign_context from upstream

IDENTIFY primary format:
  - VSL
  - Magalog/Print
  - Advertorial
  - Email (subject line)
  - Facebook/Social Ad
  - Landing Page

LOAD format constraints from specifications above
```

### Step 2: Collect All Candidates

```
GATHER candidates from:
  - 2.1-formula-based-generator → formula_candidates
  - 2.2-vault-inspired-generator → vault_candidates
  - 2.3-schema-violation-generator → schema_violation_candidates

CREATE unified candidate pool
```

### Step 3: Format Compliance Check

```
FOR each candidate in pool:

  CHECK against format constraints:
    [ ] Word count within range
    [ ] Pattern type appropriate for format
    [ ] Tone matches format expectations
    [ ] Works standalone (if format requires)
    [ ] Works with support (if format allows)

  IF candidate FAILS format check:
    FLAG for adaptation OR elimination
```

### Step 4: Adaptation (If Needed)

```
FOR each flagged candidate:

  IF word_count too high:
    - Compress promise to essential elements
    - Remove subordinate clauses
    - Use stronger single words vs. phrases

  IF pattern inappropriate:
    - Reframe in appropriate pattern
    - Preserve core Big Idea expression
    - Maintain quality score

  IF tone wrong:
    - Adjust word choice (promotional → editorial)
    - Remove/add punctuation as needed
    - Shift from command → question or vice versa
```

### Step 5: Format Variants

```
FOR top 5 candidates after filtering:

  IF campaign includes multiple formats:
    CREATE format variants:
      - VSL version (compressed)
      - Print version (expanded with sub)
      - Email version (curiosity-focused)

  TRACK which version came from which original
```

## ADAPTATION EXAMPLES

### Example 1: Print → VSL Compression

**Original (Print, 24 words):**
```
IMMUTOL: How seven sick salmon, unconventional wisdom, serendipity, and
a beer brewery saved an economy, healed the hopeless, and changed the
course of modern Immunology.
```

**Adapted (VSL, 11 words):**
```
How Sick Fish From a Beer Brewery Changed Modern Medicine
```

**What changed:** Removed product name, compressed story elements, kept core intrigue

---

### Example 2: Statement → Advertorial

**Original (Promotional):**
```
The Amazing 17-Cent Circulation Miracle That Opens Clogged Arteries!
```

**Adapted (Advertorial):**
```
The 17-Cent Compound Cardiologists Are Now Recommending for Clogged Arteries
```

**What changed:** Removed "Amazing!", added authority frame, softened "Miracle" to "Compound"

---

## ANTI-PATTERNS (FORBIDDEN)

1. **Format-Blind Generation**
   - Never generate without knowing target format
   - A great headline in wrong format = failed headline

2. **Losing Big Idea in Adaptation**
   - Compression must preserve core promise
   - If Big Idea can't fit format, question the format choice

3. **Over-Adapting**
   - Don't change more than necessary
   - If headline fits format, don't touch it
