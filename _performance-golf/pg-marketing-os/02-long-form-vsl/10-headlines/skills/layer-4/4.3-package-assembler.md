# 4.3-package-assembler.md

> **Layer:** 4 (Selection & Packaging)
> **Skill:** 10-headlines
> **Purpose:** Assemble headline-package.json with selected headline and all metadata
> **Inputs:** Confirmed selection from 4.2, all Layer 1-3 data
> **Outputs:** Complete `headline-package.json`

---

## FUNCTION

Assemble the final output file (`headline-package.json`) that contains the selected headline, all supporting metadata, alternatives, and handoff information for the Lead skill (11-lead).

**Key Principle:** This package is the COMPLETE output of the Headline skill. It must contain everything downstream skills need without requiring them to re-query upstream.

## EXECUTION PROTOCOL

### Step 1: Gather All Data

```
COLLECT from each layer:

  FROM Layer 0:
    - project_id
    - Big Idea loaded
    - Specimens loaded

  FROM Layer 1:
    - pattern_type_selected
    - curiosity_architecture
    - schema_distance_target

  FROM Layer 2:
    - candidates_generated (count)
    - generator_sources

  FROM Layer 3:
    - refinements_applied
    - subheadlines_generated
    - lead_connections_mapped

  FROM Layer 4 (4.2):
    - selected_headline
    - selection_source
    - modifications

  FROM 3.5 Final Scorer:
    - quality_scores
    - alternatives_ranked
```

### Step 2: Assemble Core Selection

```
BUILD selected_headline block:

{
  "main_headline": [from 4.2 selection],
  "subheadline": [from 3.3 if exists],
  "deck_copy": [if applicable for format],
  "selection_source": [from 4.2],
  "modifications": [from 4.2 if any]
}

VERIFY all fields populated (no nulls except deck_copy)
```

### Step 3: Assemble Architecture Block

```
BUILD headline_architecture block:

{
  "pattern_type": [from 1.2],
  "format_pattern": [from 1.2],
  "specimen_source": [from 2.2 if vault-inspired],
  "formula_source": [from 2.1 if formula-based],
  "curiosity_gap": [from 1.3],
  "schema_distance": [from 1.4 + 2.3],
  "big_idea_expression": [from 1.1]
}
```

### Step 4: Assemble Lead Handoff

```
BUILD lead_handoff block:

{
  "recommended_lead_type": [from 3.4],
  "lead_type_rationale": [from 3.4],
  "gap_resolution_timing": [from 3.4],
  "emotional_trajectory": [from 3.4],
  "callback_opportunities": [from 3.4],
  "first_sentence_guidance": [derived from headline + lead type]
}

THIS IS CRITICAL: Lead skill depends on this handoff
```

### Step 5: Assemble Alternatives

```
BUILD alternatives array:

FOR ranks 2-5 from final_ranked_candidates:
  {
    "rank": [2-5],
    "headline": [headline text],
    "subheadline": [subheadline text],
    "score": [weighted_total],
    "rationale": [from scoring notes],
    "why_not_selected": [from 4.2 if available]
  }
```

### Step 6: Validate Package

```
BEFORE writing file:

  VALIDATE all required fields:
    [ ] selected_headline.main_headline EXISTS and not empty
    [ ] headline_architecture.pattern_type EXISTS
    [ ] lead_handoff.recommended_lead_type EXISTS
    [ ] quality_scores.overall_weighted EXISTS and >= 7.5
    [ ] alternatives array has 4 entries (ranks 2-5)

  IF any validation fails:
    DO NOT write incomplete package
    FLAG missing fields
    REQUIRE resolution before proceeding
```

### Step 7: Write Package

```
WRITE headline-package.json to:
  /[project_folder]/04-mechanism/outputs/headline-package.json

  OR

  /[project_folder]/outputs/headline-package.json

VERIFY file written successfully
LOG completion timestamp
```

---

## ANTI-PATTERNS (FORBIDDEN)

1. **Partial Package**
   - Every schema section must be populated
   - Don't write incomplete package "to be finished later"

2. **Missing Lead Handoff**
   - The entire point is to hand off to Lead skill
   - lead_handoff block is MANDATORY

3. **Placeholder Values**
   - No "[INSERT X]" or "TODO" entries
   - Real values or explicit null only

4. **Skipping Validation**
   - Validation is the GATE
   - Don't bypass to "finish faster"
