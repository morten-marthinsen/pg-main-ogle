# 4.4-lead-handoff-preparer.md

> **Layer:** 4 (Selection & Packaging)
> **Skill:** 10-headlines
> **Purpose:** Prepare lead-specific context based on selected headline
> **Inputs:** Confirmed selection, lead connection mapping
> **Outputs:** `HEADLINE-SUMMARY.md` + lead-ready context

---

## FUNCTION

Create the human-readable summary document and prepare the specific context the Lead skill (11-lead) will need. This skill bridges the Headline skill to the Lead skill, ensuring smooth handoff.

**Key Principle:** The Lead skill should be able to start immediately with the context this skill provides. No re-analysis needed, no hunting for information.

---

## OUTPUT FILES

### File 1: HEADLINE-SUMMARY.md
Human-readable summary of the headline process and selection.

### File 2: Lead-ready context embedded in headline-package.json
Machine-readable context for Lead skill consumption.

## LEAD SKILL EXPECTATIONS

When 11-lead begins, it should be able to immediately access:

1. **The Selected Headline** — Exact text to reference/callback
2. **Lead Type Recommendation** — Where to start (story, revelation, etc.)
3. **First Sentence Guidance** — What the opener must do
4. **Emotional Arc** — Where to take the reader emotionally
5. **Callback Opportunities** — Phrases to echo in lead
6. **Curiosity Resolution** — When to address headline's promise

All of this must be in the handoff. If Lead skill has to "figure it out," handoff failed.

---

## ANTI-PATTERNS (FORBIDDEN)

1. **Missing Summary File**
   - HEADLINE-SUMMARY.md is required
   - Human needs readable documentation

2. **Incomplete Lead Context**
   - Lead skill shouldn't have to guess
   - Every handoff field must be populated

3. **Vague Recommendations**
   - "Maybe try a story lead" is not guidance
   - "Use revelation lead because headline creates 'what enzyme?' gap that needs immediate resolution" is guidance

4. **Breaking the Chain**
   - Headline → Lead is a continuous experience
   - Handoff must preserve the thread
