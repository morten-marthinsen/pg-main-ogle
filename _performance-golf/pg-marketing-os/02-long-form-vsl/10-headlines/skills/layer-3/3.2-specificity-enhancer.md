# 3.2-specificity-enhancer.md

> **Layer:** 3 (Refinement)
> **Skill:** 10-headlines
> **Purpose:** Add concrete numbers, names, and timeframes to headlines
> **Inputs:** `top_5_candidates` from Layer 2
> **Outputs:** Specificity-enhanced versions of each candidate

---

## FUNCTION

Analyze the top 5 headline candidates and inject SPECIFIC ELEMENTS that increase credibility and memorability. Specificity transforms vague promises into concrete, believable claims.

**Key Principle:** Specificity is credibility. "Lose weight" is forgettable. "Lose 23 pounds in 21 days" is memorable and testable. The brain trusts specific claims more than round numbers.

---

## SPECIFICITY ELEMENT TAXONOMY

### Type 1: NUMBER Specificity

```
CATEGORIES:
  - Results: "47 pounds", "230 yards", "$127,000"
  - Time: "21 days", "7 minutes", "90 seconds"
  - Quantity: "7 foods", "3 mistakes", "1 weird trick"
  - Statistics: "94% of doctors", "8 out of 10 users"

RULES:
  - Odd numbers feel more specific than round (47 vs 50)
  - Precise numbers beat ranges (23 vs "20-30")
  - Numbers must be defensible in body copy
```

### Type 2: NAME Specificity

```
CATEGORIES:
  - Geographic: "Japanese", "Harvard", "Wall Street"
  - Personal: "Dr. Smith", "CEO John Williams"
  - Institutional: "Mayo Clinic", "FDA", "Nobel Prize"
  - Product/Mechanism: "Chlorella Growth Factor", "The Pendulum Method"

RULES:
  - Names create instant credibility anchors
  - Geographic names trigger cultural associations
  - Personal names make claims more testimonial
```

### Type 3: TIME Specificity

```
CATEGORIES:
  - Duration: "In 7 days", "Over 20 years", "Since 1981"
  - Speed: "In just 10 minutes", "Overnight", "By tomorrow"
  - Frequency: "Every day", "Once a week", "Every 3 hours"

RULES:
  - Compressed timeframes increase perceived value
  - Longer timeframes (20 years) build credibility
  - Specific times beat vague ("afternoon" vs "2:47 PM")
```

### Type 4: COMPARISON Specificity

```
CATEGORIES:
  - Price: "$300 injection vs. 17-cent cream"
  - Performance: "5x faster than", "50% more effective"
  - Competition: "Better than Botox", "Outperforms [competitor]"

RULES:
  - Comparisons need a known anchor (Botox, surgery, etc.)
  - Price comparisons make value tangible
  - Performance comparisons must be defensible
```

## EXECUTION PROTOCOL

### Step 1: Analyze Specificity Gaps

```
FOR each of top_5_candidates:

  SCAN for specificity elements:
    - Numbers present? [Y/N]
    - Names present? [Y/N]
    - Timeframes present? [Y/N]
    - Comparisons present? [Y/N]

  CALCULATE specificity_score from 2.5:
    IF score < 7: High priority for enhancement
    IF score 7-8: Moderate priority
    IF score 9-10: Low priority (already specific)

  IDENTIFY insertion opportunities:
    - Generic nouns that could be named
    - Vague quantities that could be numbered
    - Undefined timeframes that could be specified
```

### Step 2: Source Specificity Elements

```
PULL from upstream packages:

  FROM proof-inventory:
    - Specific study numbers
    - Testimonial results
    - Clinical percentages

  FROM mechanism-package:
    - Mechanism name
    - Discovery date/origin
    - Named researchers/institutions

  FROM promise-package:
    - Promise magnitude (defensible numbers)
    - Timeframe claims

  FROM campaign-brief:
    - Geographic specifics
    - Authority names
    - Price points
```

### Step 3: Injection Protocol

```
FOR each candidate needing enhancement:

  IDENTIFY the most impactful insertion point:

  PRIORITY 1: Result numbers
    "Lose weight" → "Lose 23 pounds"

  PRIORITY 2: Time compression
    "Quickly improve" → "Improve in 7 days"

  PRIORITY 3: Authority names
    "A doctor discovered" → "Harvard's Dr. Smith discovered"

  PRIORITY 4: Mechanism naming
    "The secret" → "The Pendulum Method"

  INSERT no more than 2 new specific elements per headline
  (Beyond 2 becomes cluttered)
```

### Step 4: Authenticity Check

```
FOR each enhanced headline:

  VERIFY all specifics are:
    [ ] TRUE (can be supported in body copy)
    [ ] SPECIFIC (not rounded or approximated)
    [ ] CREDIBLE (within believable range)
    [ ] NATURAL (doesn't feel forced)

  IF any specificity is unverifiable:
    - Remove or soften ("up to 23 pounds")
    - Replace with verifiable alternative
```

## SPECIFICITY DO's AND DON'Ts

### DO:
- Use odd numbers (47, 23, 17) — feel more researched
- Use precise timeframes (21 days vs "3 weeks")
- Name mechanisms even if unfamiliar (curiosity trigger)
- Tie numbers to proof in body copy

### DON'T:
- Use round numbers (50, 100) — feel made up
- Stack more than 3 specific elements (cluttered)
- Use unverifiable numbers
- Name authorities without permission/proof

---

## ANTI-PATTERNS (FORBIDDEN)

1. **Fake Specificity**
   - Don't invent numbers to sound specific
   - Every number must be sourceable

2. **Specificity Overload**
   - "27 pounds in 21 days for $17 with Dr. Smith's 3-step method"
   - Too much — headline becomes cluttered

3. **Round Number Laziness**
   - "Lose 20 pounds" feels generic
   - Find the actual average from proof

4. **Undefensible Claims**
   - If you can't prove it in body copy, don't promise it in headline
