# 0.4-input-validator.md

> **Layer:** 0 (Foundation)
> **Skill:** 10-headlines
> **Purpose:** Validate all inputs are present and meet minimum thresholds
> **Outputs:** `validation_status` with pass/fail and remediation guidance

---

## FUNCTION

Validate that all required inputs from Layer 0 loaders are present, properly formatted, and above minimum quality thresholds. Gate Layer 1 execution until validation passes.

---

## VALIDATION CHECKLIST

### Critical Requirements (Must Pass)

| Input | Source | Validation Rule | Failure Action |
|-------|--------|-----------------|----------------|
| Big Idea | upstream_context | `central_idea` non-null, > 10 words | FAIL — return to 06-big-idea |
| Campaign Brief | upstream_context | `creative_direction` present | FAIL — return to 09-campaign-brief |
| Avatar Pain | upstream_context | `primary_pain` non-null | FAIL — return to 01-research |
| Niche Identified | upstream_context | niche classification present | FAIL — cannot select patterns |

### High Priority Requirements (Should Pass)

| Input | Source | Validation Rule | Failure Action |
|-------|--------|-----------------|----------------|
| Mechanism Name | upstream_context | `name` non-null | WARN — proceed in degraded mode |
| Promise | upstream_context | `primary_promise` non-null | WARN — use Big Idea as promise |
| RSF Scores | upstream_context | schema_distance present | WARN — use default 6.5 |
| Vault Intelligence | vault_patterns | `available` = true | WARN — use fallback patterns |

### Quality Thresholds

| Metric | Minimum | Optimal |
|--------|---------|---------|
| Big Idea coherence score | 6.0 | 8.0+ |
| Promise clarity score | 6.0 | 8.0+ |
| Mechanism uniqueness score | 6.0 | 8.0+ |
| Gold specimen count | 5 | 20+ |

---

## VALIDATION PROTOCOL

```
EXECUTE 0.4-input-validator:

  CRITICAL_CHECKS:
    1. CHECK upstream_context.big_idea.central_idea
       IF null OR length < 10:
         FAIL: "Big Idea not defined. Return to 06-big-idea."

    2. CHECK upstream_context.creative_direction
       IF null:
         FAIL: "Campaign Brief not complete. Return to 09-campaign-brief."

    3. CHECK upstream_context.avatar.primary_pain
       IF null:
         FAIL: "Avatar pain not defined. Return to 01-research."

    4. CHECK niche_classification
       IF null:
         FAIL: "Niche not identified. Cannot select headline patterns."

  IF any CRITICAL_CHECK fails:
    SET validation_status.passed = false
    SET validation_status.blocker = true
    RETURN validation_status

  HIGH_PRIORITY_CHECKS:
    5. CHECK upstream_context.mechanism.name
       IF null:
         WARN: "Mechanism not named. Headline will not include mechanism."
         SET degraded_mode.mechanism = true

    6. CHECK upstream_context.promise.primary_promise
       IF null:
         WARN: "Promise not defined. Using Big Idea as promise."
         SET degraded_mode.promise = true

    7. CHECK upstream_context.big_idea.rsf_scores.schema_distance
       IF null:
         WARN: "RSF schema distance not available. Using default 6.5."
         SET defaults.schema_distance = 6.5

    8. CHECK vault_patterns.vault_status.available
       IF false:
         WARN: "Vault intelligence not available. Using fallback patterns."
         SET degraded_mode.vault = true

  QUALITY_CHECKS:
    9. CHECK Big Idea coherence score
       IF < 6.0:
         WARN: "Big Idea coherence below threshold. Headline quality may suffer."

    10. CHECK Gold specimen count
        IF < 5:
          WARN: "Insufficient Gold specimens. Pattern matching limited."

  ASSEMBLE validation_status
  RETURN validation_status
```

---

## GATE CONDITION

**Layer 1 may proceed when:**
- `validation_status.passed` = true
- `validation_status.blocker` = false
- All critical checks passed

**Layer 1 may NOT proceed when:**
- Any critical check failed
- `validation_status.blocker` = true
- Remediation required before continuing
