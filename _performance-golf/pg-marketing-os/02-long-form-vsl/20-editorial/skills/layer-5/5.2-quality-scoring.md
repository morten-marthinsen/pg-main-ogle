**Layer:** 5 (Validation & Output)
**Purpose:** Calculate final quality scores using the 7-criterion marketplace scoring system

## FINAL 7-CRITERION MARKETPLACE SCORING

Re-score the edited copy using the same criteria from Layer 1, now assessing the final version.

---

## CRITERION 1: STOPPING POWER (Weight: 20%)

```yaml
final_stopping_power:
  score: "[1-10]"
  evidence: "[Quote from opening that supports score]"
  notes: "[Explanation]"

  comparison:
    layer_1_score: "[original score]"
    final_score: "[new score]"
    delta: "[change]"

  scoring_guide:
    10: "Can't look away — magnetic"

    6_7: "Decent, would stop some people"

```

## CRITERION 2: BELIEVABILITY (Weight: 15%)

```yaml
  notes: "[Explanation]"

  comparison:
    layer_1_score: "[original score]"
    final_score: "[new score]"
    delta: "[change]"

  # PLACEHOLDER EXCLUSION RULE (Learning #48)
  # Acknowledged placeholders (e.g., "[Testimonial Section]", "[Client to provide]")
  # should NOT penalize the score. Score based on PRESENT copy only.
  # List placeholders separately as "awaiting client content" not "gaps"
  placeholder_handling:
    placeholders_detected: "[list any bracketed placeholder text]"
    excluded_from_scoring: true
    note: "Score reflects present copy only; placeholders await client content"

  scoring_guide:
    10: "Ironclad proof, impossible to doubt"
    8_9: "Strong proof, most would believe"
    6_7: "Some proof, skeptics would doubt"
    4_5: "Weak proof, many would doubt"

```

## PLACEHOLDER CONTENT HANDLING (MANDATORY)

Per Learning #48: Placeholders should NOT penalize scores.

```yaml
placeholder_audit:
  detected_placeholders:
    - location: "[section/line]"
      text: "[placeholder text, e.g., '[Testimonial Section]']"
      type: "[testimonial | proof | image | client_content]"

  scoring_treatment:
    excluded_from_believability: true
    excluded_from_proof_density: true
    listed_as: "Awaiting client content"
    NOT_listed_as: "Proof gap" # This would incorrectly penalize

  report_format:
    # In editorial report, list placeholders under "Pending Content" not "Weaknesses"
    section_heading: "Pending Client Content"
    items:
      - "[Placeholder 1] — awaiting client testimonials"
      - "[Placeholder 2] — awaiting client data"
```
