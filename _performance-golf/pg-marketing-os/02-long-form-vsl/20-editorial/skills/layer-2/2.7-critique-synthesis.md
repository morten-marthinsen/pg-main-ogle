# Microskill 2.7: Critique Synthesis

**Layer:** 2 (Multi-Expert Critique)
**Purpose:** Synthesize all 6 expert lens critiques into prioritized action plan

---

## SYNTHESIS PURPOSE

The 6 expert lenses each identify issues from their perspective. This microskill:
1. Consolidates all identified issues
2. Eliminates duplicates and conflicts
3. Prioritizes by impact and effort
4. Creates a unified fix roadmap

## ISSUE CONSOLIDATION

### Master Issue List

Compile ALL issues from all lenses:

```yaml
master_issue_list:
  - id: "ISSUE_001"
    description: "[issue description]"
    identified_by: ["[lens 1]", "[lens 2]"]  # Multiple if flagged by several
    severity: "[CRITICAL | MAJOR | MINOR]"
    location: "[section/paragraph]"
    quote: "[evidence]"
    lens_specific_fixes:
      - lens: "[lens name]"
        fix: "[their recommended fix]"

  - id: "ISSUE_002"
    # ... continue for all issues
```

### Duplicate Detection

```yaml
duplicate_analysis:
  merged_issues:
    - original_ids: ["MAKEPEACE_003", "HALBERT_001"]
      merged_into: "ISSUE_007"
      reason: "Both identify momentum loss in mechanism section"

    # ... continue for all merges
```

### Conflict Resolution

When experts disagree:

```yaml
conflict_resolution:
  conflicts_found:
    - conflict_id: "CONFLICT_001"
      issue: "[what they disagree about]"
      lens_a: "[lens name]"
      lens_a_position: "[their recommendation]"
      lens_b: "[lens name]"
      lens_b_position: "[their recommendation]"
      resolution: "[which approach to take and why]"
      reasoning: "[why this resolution]"

    # ... continue for all conflicts
```

**Conflict Resolution Hierarchy:**
1. **Kennedy** wins on offer/CTA issues (DR fundamentals)
2. **Schwartz** wins on market calibration issues (sophistication)
3. **Clemens** wins on mechanism architecture issues
4. **Ogilvy** wins on credibility/proof issues
5. **Makepeace** wins on flow/momentum issues
6. **Halbert** wins on voice/entertainment issues

## SEVERITY ASSESSMENT

### Final Severity Assignment

```yaml
severity_distribution:
  critical:
    count: "[number]"
    issues:
      - id: "[issue ID]"
        description: "[brief]"
        must_fix_reason: "[why critical]"

  major:
    count: "[number]"
    issues:
      - id: "[issue ID]"
        description: "[brief]"
        should_fix_reason: "[why major]"

  minor:
    count: "[number]"
    issues:
      - id: "[issue ID]"
        description: "[brief]"
```

**Severity Criteria:**
- **CRITICAL:** Would prevent purchase / breaks the copy / reader would stop
- **MAJOR:** Significantly weakens effectiveness / loses substantial conversions
- **MINOR:** Suboptimal but not damaging / polish-level improvement

## IMPACT-EFFORT MATRIX

### Categorize Each Fix

```yaml
impact_effort_matrix:
  high_impact_low_effort:  # DO FIRST
    - issue_id: "[ID]"
      description: "[brief]"
      fix: "[what to do]"
      expected_lift: "[what improves]"

  high_impact_high_effort:  # SCHEDULE
    - issue_id: "[ID]"
      description: "[brief]"
      fix: "[what to do]"
      expected_lift: "[what improves]"

  low_impact_low_effort:  # IF TIME PERMITS
    - issue_id: "[ID]"
      description: "[brief]"
      fix: "[what to do]"

  low_impact_high_effort:  # SKIP
    - issue_id: "[ID]"
      description: "[brief]"
      recommendation: "Skip unless time permits"
```
