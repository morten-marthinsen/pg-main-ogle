# 1.5 Full-Read Structural Coherence Audit

**Version:** 1.0
**Created:** 2026-02-06
**Purpose:** Read the ENTIRE draft as a reader would to detect macro-level structural issues the 6 expert lenses miss
**Position:** Layer 1 (After blind read capture, before multi-lens critique)
**Status:** ACTIVE — CRITICAL GATE

---

## WHY THIS MICROSKILL EXISTS

**Catastrophic Failure Pattern Discovered (Ultra Liver 2026-02-05):**

The Ultra Liver campaign passed Editorial with an A- (91%) grade despite having:
- The 3-phase liver detox explanation repeated THREE TIMES (Story, Root Cause, Mechanism)
- The car wash metaphor used TWICE
- "Sticky bile" concept explained FOUR TIMES
- Sections that felt like separate documents stitched together

**How this passed Editorial's 6 expert lenses:**
- Makepeace Lens: "Flow good" — evaluated transitions individually, not as whole narrative
- Halbert Lens: "Entertainment good" — personality consistent (because same content repeated)
- Schwartz Lens: "Sophistication calibrated" — irrelevant to repetition issue
- Ogilvy Lens: "Credibility strong" — each explanation was credible individually
- Clemens Lens: "Mechanism clear" — yes, explained THREE TIMES so very clear
- Kennedy Lens: "DR fundamentals solid" — offer/close were fine

**The problem:** Each lens evaluated its criteria in isolation. No lens asked: "If I read this straight through as a reader, would I feel like I'm being told the same thing multiple times?"

**Threading counts were INFLATED by repetition:**
- "Tri-Phase Flow" appeared 12 times → But 4+ were in repeated explanations
- "Phase 3" appeared 18 times → Inflated by the same content appearing 3x

---

## PURPOSE

Execute a FULL READ-THROUGH of the assembled draft as a READER would experience it, detecting:

1. **Content repetition** — Same concepts explained multiple times
2. **Narrative fragmentation** — Sections feel stitched, not unified
3. **Momentum killers** — Places where reader would think "didn't they just say this?"
4. **Threading inflation** — High counts from repetition, not strategic placement
5. **Section redundancy** — Sections that could be merged without losing value

---

## THE READER EXPERIENCE TEST

### Core Question

> "If someone reads this draft from beginning to end in one sitting, will they:
> - Feel like they're reading ONE continuous narrative, or multiple documents?
> - Feel information PROGRESSES, or REPEATS?
> - Want to keep reading, or feel like they've already heard this?"

### Detection Signals

| Signal | What It Means | Severity |
|--------|---------------|----------|
| Same numbered list appears 2+ times | Core concept repeated | CRITICAL |
| Same metaphor/analogy used 2+ times | Explanation duplicated | CRITICAL |
| Reader would think "didn't you just say this?" | Narrative breaks | CRITICAL |
| Section feels complete on its own | Not building toward next section | MODERATE |
| Later section re-explains earlier concept | Redundancy | MODERATE |
| High threading counts but low unique contexts | Inflated threading | MODERATE |

## EXECUTION PROCESS

### Step 1: Fresh Full Read

Read the ENTIRE assembled draft from beginning to end WITHOUT analysis.

**Capture during reading:**
- Any moment where you think "wait, didn't they say this already?"
- Any moment where the narrative feels like it restarts
- Any moment where the "voice" seems to shift
- Any moment where forward momentum stalls

```yaml
full_read_capture:
  total_reading_time: "[minutes]"

  repetition_moments:
    - location: "[section, paragraph, line]"
      reaction: "I've heard this before"
      refers_back_to: "[earlier location]"

  restart_moments:
    - location: "[section, paragraph]"
      reaction: "Feels like the pitch is starting over"

  momentum_stalls:
    - location: "[section, paragraph]"
      reaction: "[what caused the stall]"

  overall_feeling: "[one continuous narrative | multiple documents | somewhere between]"
```

### Step 2: Progressive Information Audit

For each major section, map what NEW information the reader learns:

```yaml
progressive_information_audit:

  lead:
    new_information:
      - "[What reader learns for the FIRST time]"
    builds_on: "N/A (first section)"

  story:
    new_information:
      - "[What reader learns for the FIRST time]"
    builds_on: "[What from lead]"
    REPEAT_FLAG: "[Any info already given in lead]"

  root_cause:
    new_information:
      - "[What reader learns for the FIRST time]"
    builds_on: "[What from story]"
    REPEAT_FLAG: "[Any info already given in story or lead]"

  mechanism:
    new_information:
      - "[What reader learns for the FIRST time]"
    builds_on: "[What from root cause]"
    REPEAT_FLAG: "[Any info already given earlier]"

  # ... continue for all sections
```

### Step 3: Repetition Heat Map

Create a visual map of where concepts appear:

```yaml
repetition_heat_map:

  concept_1:
    name: "[e.g., Three phases of liver detox]"
    appearances:
      - section: "Story"
        depth: "[FULL explanation | MENTION | REFERENCE]"
        word_count: [number]
      - section: "Root Cause"
        depth: "[FULL explanation | MENTION | REFERENCE]"
        word_count: [number]
      - section: "Mechanism"
        depth: "[FULL explanation | MENTION | REFERENCE]"
        word_count: [number]

    assessment:
      full_explanations: [count]
      should_be: 1
      verdict: [CRITICAL DUPLICATION | ACCEPTABLE THREADING | OK]

  concept_2:
    # ... etc
```

### Step 4: Threading Quality vs Quantity

Distinguish genuine threading from inflated counts:

```yaml
threading_quality_audit:

  mechanism_name:
    raw_count: [number]

    unique_contexts:
      - context: "[section + function]"
        quote: "[exact usage]"
      # ... list each UNIQUE usage

    unique_context_count: [number]

    repeated_context_count: "[raw - unique = inflation]"

    quality_score: "[unique / raw = percentage]"

    IF quality_score < 70%:
      FLAG: "Threading count inflated by repetition"
```

## QUALITY GATE

### Coherence Scoring

```yaml
coherence_score:

  progression_score:
    description: "Does information progress or repeat?"
    score: [1-10]
    minimum: 7.0
    evidence: "[specific observations]"

  unity_score:
    description: "Does it feel like one narrative or multiple documents?"
    score: [1-10]
    minimum: 7.0
    evidence: "[specific observations]"

  momentum_score:
    description: "Does reader want to keep reading or feel they've heard this?"
    score: [1-10]
    minimum: 7.0
    evidence: "[specific observations]"

  threading_quality_score:
    description: "Are threading counts genuine or inflated?"
    score: [1-10]
    minimum: 7.0
    evidence: "[specific observations]"

  overall_coherence_score:
    calculation: "(progression + unity + momentum + threading_quality) / 4"
    result: [number]
    minimum: 7.0
```

### PASS/FAIL Criteria

```yaml
coherence_gate:

  overall_coherence_score: [number]

  IF overall_coherence_score < 7.0:
    HALT — "Full-read coherence audit failed"

    diagnostic:
      progression_issues: "[list]"
      unity_issues: "[list]"
      momentum_issues: "[list]"
      threading_inflation: "[list]"

    recommended_action: "[Return to Assembly for restructure | Heavy editorial revision | Human checkpoint]"

  IF overall_coherence_score >= 7.0 AND < 8.0:
    PROCEED_WITH_CAUTION — "Coherence adequate but not strong"

    issues_to_address: "[list for editorial attention]"

  IF overall_coherence_score >= 8.0:
    PASS — "Strong narrative coherence"
```

## SWIPE FILE COMPARISON

### Quality Benchmark: Emma (Ryan Kowalski)

**Structure Analysis:**
- Problem (leaky gut) introduced ONCE in Negative Hook section
- Mechanism (archaea producing methane) explained ONCE
- Solution (berberine + 3 other ingredients) introduced ONCE
- Product (Emma) revealed as delivery system
- PROGRESSIVE BUILD throughout — each section adds new information

**No Repetition:**
- Leaky gut not re-explained in later sections
- Methane mechanism not re-explained
- Each section assumes reader absorbed previous section

### Quality Benchmark: Gundry Bio Complete 3

**Structure Analysis:**
- Problem (leaky gut from lectins) introduced ONCE
- Mechanism (butyrate repairs gut lining) explained ONCE
- Solution (prebiotics + probiotics + butyrate) introduced ONCE as "three-pronged defense"
- Each section BUILDS on previous — never re-explains

**Progression Pattern:**
```
Hook → "There's a problem"
Authority → "I discovered something"
Mechanism → "Here's how it works" (EXPLAINED ONCE)
Solution → "Here's what fixes it" (BUILDS ON mechanism, doesn't repeat it)
Product → "Here's my formula" (ASSUMES you understand mechanism)
Offer → "Here's how to get it"
Close → "Here's why to act now"
```

### Failure Example: Ultra Liver Original Draft

**Structure Problems:**
```
Lead → Introduces Phase 3, sticky bile
Story → RE-EXPLAINS Phase 3, sticky bile, car wash metaphor
Root Cause → RE-EXPLAINS Phase 3, sticky bile, laundry metaphor
Mechanism → RE-EXPLAINS Phase 3, car wash metaphor AGAIN
```

**What it should have looked like:**
```
Lead → Teases the problem (sticky bile)
Story → Dr. Levitt's discovery journey (personal, not technical)
Root Cause → WHAT'S WRONG (Phase 3 gap identified)
Mechanism → HOW TO FIX IT (Tri-Phase Flow explained ONCE)
```
