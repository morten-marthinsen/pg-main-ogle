## PURPOSE

Assembles the complete close-package.json — the final output of Skill 14 and the FINAL output of the entire copy execution pipeline. This is the definitive assembly point where all drafted, refined, and validated close sections are merged into a single package with continuous prose, validation scores, and campaign assembly handoffs.

The assembler performs three critical functions: (1) merges all close sections into a single continuous full_narrative_text string — the actual close copy prose in reading order; (2) computes weighted validation scores from 4.1 Makepeace elements check, 4.2 anti-slop, and 4.3 vault comparison; (3) packages campaign assembly handoffs so the close integrates seamlessly into the final campaign.

**This is the LAST microskill in the LAST copy execution skill.** After this, there are no more writing skills. The close-package.json is handed directly to campaign assembly. Every field must be populated. Every quality gate must pass. The close must be production-ready.

---

## IDENTITY

**This microskill IS:**
- The FINAL assembler for the close skill AND the entire copy pipeline
- A section merger that creates continuous close prose in correct sequence
- A validation score aggregator from 4.1, 4.2, and 4.3
- A campaign assembly handoff packager
- A completeness verifier — every required field must be populated
- A minimum quality enforcer — overall score must be >= 7.0
- The pipeline terminus — close_is_final: true

**This microskill is NOT:**
- A drafter or writer (all drafting happened in Layer 2-3)
- A validator (validation happened in 4.1-4.3)
- A remediation engine (if score < 7.0, it flags, doesn't fix)
- An offer writer (Skill 13 handled that)

## PROCESS

### Step 1: Merge Sections into Full Narrative Text

1. Assemble close sections in Makepeace Close Sequence order:

   1. **Opening transition** — seamless entry from offer section
   2. **Benefit summary** — "You get" format recapping everything
   3. **Guarantee close** — branded guarantee as confidence builder
   4. **Closing theme** — primary emotional close (crossroads, logical restatement, etc.)
   5. **CTA 1** — first ask for the sale (confidence-based)
   6. **Consequence amplification** — what happens if they DON'T act
   7. **CTA 2** — second ask (consequence avoidance-based)
   8. **Future pacing / crossroads** — dream vs. nightmare, decision ownership
   9. **CTA 3** — third ask (urgency-based)
   10. **Urgency/scarcity section** — justified urgency with staccato
   11. **CTA 4-6+** — additional asks distributed through remaining sections
   12. **Checkout / action instructions** — step-by-step with themes woven in
   13. **CTA (checkout)** — action ask within checkout flow
   14. **P.S. section(s)** — prime selling real estate with technique
   15. **CTA (P.S.)** — final ask within P.S.

2. Ensure smooth transitions between merged sections
3. Verify Cialdini commitment/consistency elements are woven throughout (from 3.4)
4. Produce one continuous string: full_narrative_text
5. Compute total word count

### Step 2: Compute Validation Scores

6. Aggregate dimension scores:

```yaml
scores:
  benefit_summary_impact: float  # from 4.3 vault comparison
  guarantee_confidence_building: float  # from 4.3 vault comparison
  cta_repetition_effectiveness: float  # from 4.3 vault comparison
  cta_variety: float  # from 4.3 vault comparison + 4.1 element 3
  closing_theme_execution: float  # from 4.3 vault comparison
  urgency_credibility: float  # from 4.3 vault comparison
  future_pacing_vividness: float  # from 4.3 vault comparison
  ps_impact: float  # from 4.3 vault comparison
  cialdini_integration: float  # from 4.3 vault comparison
  vault_pattern_comparison: float  # from 4.3 overall score
```

7. Compute overall weighted average:
   - Benefit summary impact: 10% weight
   - Guarantee confidence building: 10% weight
   - CTA repetition effectiveness: 15% weight (close relies heavily on repetition)
   - CTA variety: 10% weight
   - Closing theme execution: 10% weight
   - Urgency credibility: 10% weight
   - Future pacing vividness: 5% weight
   - P.S. impact: 10% weight
   - Cialdini integration: 5% weight
   - Vault pattern comparison: 15% weight

8. Flag if overall < 7.0 — remediation needed

### Step 3: Package Makepeace Elements Check

9. Include pass/fail for all 6 elements:

```yaml
makepeace_elements_check:
  element_1_benefit_summary: boolean
  element_2_guarantee: boolean
  element_3_ask_for_sale: boolean  # 6+ asks
  element_4_tell_what_to_do: boolean
  element_5_ps: boolean
  element_6_sidebars: boolean
  all_six_present: boolean
  additional_metrics:
    total_cta_count: integer
    guarantee_framing: string
    ps_technique: string
    cialdini_present: boolean
    future_pacing_present: boolean
    staccato_present: boolean
```

### Step 4: Package Anti-Slop Results

```yaml
  pass: boolean
```

## FAILURE MODES

### Failure Mode 1: Overall Score Below 7.0
- **Symptom:** Weighted average < 7.0
- **Cause:** One or more dimensions scored poorly
- **Prevention:** Layer 3 refinement should have caught quality issues
- **Recovery:** Identify lowest-scoring dimension, trigger targeted remediation in that layer

### Failure Mode 2: Missing Sections
- **Symptom:** full_narrative_text has gaps or missing close sections
- **Cause:** Layer 2 microskill didn't produce output for a section
- **Prevention:** Step 7 completeness check catches all missing fields
- **Recovery:** Identify missing section, re-run the specific Layer 2 microskill

### Failure Mode 3: CTA Count Below 6
- **Symptom:** Assembled close has fewer than 6 asks for the sale
- **Cause:** CTAs weren't distributed across all sections as planned in 1.3
- **Prevention:** Step 1 merge process distributes CTAs per 1.3 plan
- **Recovery:** Add CTAs to underrepresented sections (benefit summary, checkout, P.S.)

### Failure Mode 4: Incoherent Merge
- **Symptom:** Sections are present but don't flow together
- **Cause:** Sections written independently without transition awareness
- **Prevention:** Step 1 includes transition verification during merge
- **Recovery:** Add bridging sentences between sections that don't flow

### Failure Mode 5: Cialdini Not Distributed
- **Symptom:** Commitment/consistency appears in one section only
- **Cause:** 3.4 integration points not applied across merged text
- **Prevention:** Step 1 verifies Cialdini distribution from 3.4 output
- **Recovery:** Redistribute integration points across sections per 3.4 plan
