## PURPOSE

Compares the assembled close copy against elite close sections from the TIER1 vault. The vault contains 482+ extraction reports from proven controls, each with close-specific sections covering benefit summaries, guarantees, CTA sequences, urgency mechanics, P.S. sections, and closing themes. This microskill benchmarks the current close against these elite patterns — not to replicate them, but to ensure the close meets the persuasive density and structural completeness of TIER1 standards.

The close is the FINAL copy section in the pipeline. Its quality determines whether the entire campaign's persuasive architecture converts. This comparison ensures the close operates at the elite level established by controls that have generated millions in proven revenue.

The comparison serves two functions: (1) quality benchmarking — does this close reach the persuasive intensity of TIER1 close sections? (2) differentiation documentation — how does this close differ from vault patterns, and is that differentiation intentional and beneficial?

---

## IDENTITY

**This microskill IS:**
- A vault pattern benchmarking system for close copy quality
- A TIER1 elite close section comparison engine
- A differentiation documenter — how this close differs from vault patterns
- A quality scoring system based on competitive close benchmarks
- A pattern matcher across benefit summaries, guarantees, CTAs, urgency, P.S., and closing themes

**This microskill is NOT:**
- A Makepeace elements checker (that is 4.1 — structural compliance)
- An anti-slop validator (that is 4.2 — language quality)
- A copy generator (this compares, doesn't write)
- A template enforcer (vault patterns inform quality standards, not dictate structure)

## PROCESS

### Step 1: Select Comparison Cohort

1. From the TIER1 vault, select 5-8 comparison controls that match:
   - Same or similar niche (health, finance, self-improvement, golf, etc.)
   - Same or similar product format (supplement, digital, info, physical)
   - Controls with strong close sections (CTA density, guarantee quality, P.S. impact)

2. If exact niche matches unavailable, expand to adjacent niches with similar close architectures

3. Document the comparison cohort with rationale for each selection

### Step 2: Compare Benefit Summary Quality

4. Assess benefit summary vs. vault patterns:
   - How many "You get" bullets do vault controls use? (Typical: 5-12)
   - Do vault controls attach "reason why" to each benefit?
   - How vivid are the benefit descriptions compared to vault?
   - Do vault controls use the benefit summary to reignite desire?
   - How does this close's benefit summary density compare?

5. Score: Benefit summary impact (1-10)

### Step 3: Compare Guarantee Execution

6. Assess guarantee presentation vs. vault patterns:
   - Do vault controls use branded guarantee names? (Almost always yes)
   - What guarantee styles appear? (Contract, personal promise, challenge)
   - How do vault controls frame guarantee as confidence vs. refund?
   - What "rush" or speed language do vault guarantees use?
   - How does this close's guarantee compare?

7. Score: Guarantee confidence-building (1-10)

### Step 4: Compare CTA Architecture

8. Assess CTA sequence vs. vault patterns:
   - How many total asks do vault controls include? (Typical: 6-10+)
   - How varied are vault CTA emotions and phrases?
   - Where do vault controls place CTAs? (After benefit summary, after guarantee, after urgency, in P.S., etc.)
   - Do vault controls use action instructions with themes woven in?
   - How does this close's CTA count, variety, and placement compare?

9. Score: CTA repetition effectiveness (1-10)

10. Score: CTA variety (1-10)

### Step 5: Compare Closing Theme Execution

11. Assess closing theme vs. vault patterns:
    - Which Makepeace themes do vault controls use most frequently?
    - How do vault controls establish sincerity without fabrication?
    - What emotional arcs do vault close themes follow?
    - How does this close's theme execution compare?

12. Score: Closing theme execution (1-10)

### Step 6: Compare Urgency/Scarcity Mechanics

13. Assess urgency vs. vault patterns:
    - What urgency justifications do vault controls use?
    - Do vault controls use staccato tempo in urgency sections?
    - How specific are vault urgency claims?
    - Do vault controls pair urgency with consequence amplification?
    - How does this close's urgency credibility compare?

14. Score: Urgency credibility (1-10)

### Step 7: Compare Future Pacing / Crossroads

15. Assess future pacing vs. vault patterns:
    - Do vault controls use dream vs. nightmare contrast?
    - How vivid are vault future pacing scenarios?
    - Do vault controls use crossroads framing?
    - How does decision ownership manifest in vault controls?
    - How does this close's future pacing compare?

16. Score: Future pacing vividness (1-10)

### Step 8: Compare P.S. Impact

17. Assess P.S. section vs. vault patterns:
    - Which of 6 Makepeace P.S. techniques do vault controls use most?
    - How many P.S./P.P.S. do vault controls include? (Typical: 1-3)
    - Do vault P.S. sections contain their own CTAs?
    - Could vault P.S. sections standalone as compelling sells?
    - How does this close's P.S. impact compare?

18. Score: P.S. impact (1-10)

### Step 9: Compare Cialdini Integration

19. Assess Cialdini commitment/consistency vs. vault patterns:
    - Do vault controls use implicit agreement leveraging?
    - How subtle is the Cialdini integration in vault controls?
    - Do vault controls build on pre-existing commitments (reading, engaging, requesting info)?
    - How does this close's Cialdini integration compare?

20. Score: Cialdini integration (1-10)

### Step 10: Document Differentiation

21. For each comparison dimension, document:
    - How this close DIFFERS from vault patterns
    - Whether the differentiation is INTENTIONAL (serving a strategic purpose)
    - Whether the differentiation is BENEFICIAL (making the close stronger)
    - Whether the differentiation is CONCERNING (potential weakness vs. vault benchmarks)

## FAILURE MODES

### Failure Mode 1: No Comparable Close Sections
- **Symptom:** No vault controls have strong close sections for this niche
- **Cause:** Vault coverage gap for close patterns in this niche
- **Prevention:** Step 1 expands to adjacent niches with similar close architectures
- **Recovery:** Use adjacent niche close patterns with documented niche differences

### Failure Mode 2: Cherry-Picked Comparisons
- **Symptom:** Only compared to weakest vault close sections — artificially high score
- **Cause:** Selection bias toward favorable comparisons
- **Prevention:** Step 1 requires 5-8 controls including strong and moderate examples
- **Recovery:** Add strongest vault close sections to cohort, re-score

### Failure Mode 3: Scoring Without Close-Specific Evidence
- **Symptom:** Scores assigned based on general copy quality, not close-specific patterns
- **Cause:** Comparator used general vault data instead of close-section-specific patterns
- **Prevention:** Each score requires specific vault close section citation
- **Recovery:** Re-score using only close-section-specific vault patterns

### Failure Mode 4: CTA Count Mismatch Ignored
- **Symptom:** Close has 3 CTAs but vault average is 8 — scored as "adequate"
- **Cause:** Comparator failed to weight CTA count heavily enough
- **Prevention:** CTA effectiveness and variety are separate dimensions, both scored
- **Recovery:** Flag CTA count gap explicitly in recommendations

## EXAMPLES

### Example 1: Strong Close Comparison — Male Vitality Supplement

**Cohort:** Montezuma's Secret, Resurge, Total Restore, Testo Rise, Uri Varx (5 controls)
**Scores:**
- Benefit summary: 8.5 (8 "You get" bullets vs. vault avg 7)
- Guarantee: 9.0 ("Triple-Back Guarantee" — matches vault best practice)
- CTA effectiveness: 7.5 (9 asks — above vault avg of 7)
- CTA variety: 8.0 (5 emotions used — matches vault diversity)
- Closing theme: 8.0 (Crossroads theme — strong execution)
- Urgency: 8.5 (Production-justified, staccato present)
- Future pacing: 8.0 (Vivid dream/nightmare contrast)
- P.S.: 7.5 (Fast-reply bonus technique — effective but could be more romanced)
- Cialdini: 7.0 (Present but subtle — vault controls more aggressive)
- **Overall: 8.1**

**Differentiation:**
- Intentional: Uses crossroads over logical restatement (niche-appropriate emotional close)
- Beneficial: CTA count (9) exceeds vault average (7)
- Concerning: Cialdini integration lighter than vault controls — could strengthen

**Recommendation:** Strengthen Cialdini commitment/consistency by adding one more implicit agreement reference.

### Example 2: Below Threshold — Digital Product Close

**Cohort:** BBW, Superbrain, Elixir of Eros + 3 adjacent (6 controls)
**Scores:**
- Benefit summary: 6.5 (only 4 "You get" bullets vs. vault avg 7)
- Guarantee: 6.0 (generic name, weak confidence framing)
- CTA effectiveness: 5.5 (only 4 asks vs. vault avg 7)
- CTA variety: 6.0 (only 2 emotions used)
- Closing theme: 7.0 (adequate but not distinctive)
- Urgency: 7.5 (justified but lacks staccato)
- Future pacing: 7.0 (present but not vivid enough)
- P.S.: 5.0 (administrative — "Don't forget!")
- Cialdini: 6.5 (present but heavy-handed)
- **Overall: 6.3 — BELOW MINIMUM**

**Recommendations:**
1. Add 3+ more "You get" bullets to match vault density
2. Brand the guarantee with a unique name and confidence framing
3. Add 3+ more CTAs with different emotional appeals
4. Rewrite P.S. with Makepeace technique — romance a fast-reply bonus or add testimony
5. Add staccato tempo to urgency section
