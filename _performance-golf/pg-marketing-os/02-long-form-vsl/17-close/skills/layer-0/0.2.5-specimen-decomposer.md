# Layer 0.2.5 — Specimen Decomposer

**Version:** 1.0
**Layer:** 0 (Foundation)
**Type:** Specimen Analysis
**Domain:** Close
**Dependencies:** vault_close_intelligence (from 0.2-vault-intelligence-loader.md)
**Output:** specimen_decomposition (structural templates, technique catalog, power elements)

---

## Purpose

Decompose top-scoring close specimens from the vault-intelligence into reusable structural templates, technique catalogs, and power elements. Close specimens map the final persuasion sequence — close sequence elements, future pacing text, final push language, PS sections, and close type. This skill extracts the closing architecture that converts convinced prospects into buyers.

**This skill bridges the gap between:** vault data (close sequence maps with verbatim text) and generation layers (what Claude needs to construct original closing sequences with the same conversion power and emotional momentum).

## Decomposition Protocol

### Phase 1: Specimen Selection & Tiering

```
IF tier_metadata present (Phase B loader active):
  LOAD gold_specimens → full decomposition
  LOAD silver_specimens → template-only decomposition
ELSE (pre-upgrade fallback):
  SELECT top 5 by position as pseudo-Gold
  SELECT next 10 as pseudo-Silver
  LOG "Operating in pre-upgrade mode — tiering estimated from array position"

VERIFY each close specimen has:
  - close_sequence[] (required — at least 2 elements)
  - close_type (required)
  - cta_text (required)
  - source_file (required for provenance)
  - niche (required for type-matching)
  - future_pacing_text (preferred)
  - final_push (preferred)
  - ps_sections (preferred — may be null)
  - format (preferred)
```

### Phase 2: Structural Template Extraction

For each Gold specimen:

```
ANALYZE close_sequence[], future_pacing_text, final_push, and ps_sections to extract:

  close_skeleton:
    FOR each element in close_sequence[]:
      - element_name: string (from specimen — e.g., summarize_transformation,
                              opportunity_framing, risk_reversal,
                              pattern_established, inevitability,
                              elite_preparation, personal_stake, final_cta)
      - element_text: string (from specimen — the verbatim text)
      - function: string (persuasion function this element serves)
      - position: integer
      - proportion: float (approximate % of close text)
    sequence_length: integer
    close_type: string (from specimen — emotional, logical, urgency, etc.)

  close_architecture:
    emotional_strategy: string (how emotions build through the close)
    momentum_pattern: string (how urgency/desire escalates)
    final_move: string (what the last persuasion element does before CTA)
    cta_style: string (direct_command | benefit_wrapped | fear_of_loss |
                        assumptive | question_close)

  future_pacing_architecture:
    pacing_present: boolean
    pacing_technique: string (vivid_visualization | outcome_projection |
                               contrast_with_present | identity_shift)
    sensory_elements: [string] (which senses are engaged)

  ps_architecture:
    ps_present: boolean
    ps_count: integer
    ps_functions: [string] (per PS — urgency_reinforcement, bonus_reminder,
                             guarantee_restatement, new_proof, emotional_appeal)

For Silver specimens:
  Extract close_architecture and close_type ONLY
  SKIP element-level analysis and PS architecture
```

### Phase 3: Technique Catalog Compilation

Aggregate across ALL decomposed specimens:

```
close_types:
  FOR each unique close_type:
    - type_name: string (emotional, logical, urgency, fear_of_loss,
                          social_proof, authority, etc.)
    - count: integer
    - element_sequence: string (typical element order for this type)
    - implementation_pattern: string
    - gold_example: string
    - best_for: string (niche / product type)

close_elements:
  FOR each unique close_sequence element type:
    - element_name: string
    - count: integer (how many closes use this element)
    - function: string
    - typical_position: string
    - implementation_pattern: string
    - gold_example: string (verbatim text excerpt)

future_pacing_techniques:
  FOR each unique pacing technique:
    - technique_name: string
    - count: integer
    - implementation_pattern: string
    - gold_example: string
    - sensory_channels: [string]

final_push_techniques:
  FOR each unique final push approach:
    - technique_name: string
    - count: integer
    - implementation_pattern: string
    - gold_example: string
    - emotional_lever: string (what emotion it activates)

ps_techniques:
  FOR each unique PS function:
    - function_name: string
    - count: integer
    - implementation_pattern: string
    - gold_example: string
    - position: string (PS1, PS2, PPS)
```

### Phase 4: Power Element Extraction

From Gold specimens ONLY:

```
close_sequence_verbatims:
  FOR each Gold specimen:
    - verbatim_elements: [object] (each element with name + text from close_sequence)
    - close_type: string
    - sequence_length: integer
    - source_file: string
    - niche: string

future_pacing_verbatims:
  FOR each Gold specimen with future_pacing_text:
    - verbatim_pacing: string
    - pacing_technique: string
    - sensory_elements: [string]
    - source_file: string

final_push_verbatims:
  FOR each Gold specimen with final_push:
    - verbatim_push: string
    - push_technique: string
    - emotional_lever: string
    - source_file: string

ps_verbatims:
  FOR each Gold specimen with ps_sections (non-null):
    - verbatim_ps: [string] (each PS section)
    - ps_functions: [string]
    - source_file: string

cta_verbatims:
  FOR each Gold specimen:
    - verbatim_cta: string (from cta_text)
    - cta_style: string
    - source_file: string
```

## Quality Gates

- [ ] Minimum 3 Gold specimens fully decomposed (sequence + architecture + pacing + PS)
- [ ] Minimum 5 Silver specimens template-decomposed
- [ ] All close skeletons have complete element sequences with functions
- [ ] Technique catalog has minimum 2 close types cataloged
- [ ] Technique catalog has minimum 3 close elements documented
- [ ] Power elements contain minimum 3 verbatim close sequences from Gold specimens
- [ ] Future pacing extracted from all Gold specimens where present
- [ ] PS sections extracted from all Gold specimens where present
- [ ] Every power element includes source provenance
- [ ] No raw close_sequence text passed through undecomposed

---

## Anti-Patterns

| Anti-Pattern | Signal | Fix |
|---|---|---|
| **Sequence Without Function** | Close elements listed without persuasion function | Every element needs function annotation |
| **Future Pacing Ignored** | Close analyzed without future pacing extraction | Future pacing is high-value conversion element — always extract |
| **PS Dismissed** | PS sections treated as afterthought or skipped | PS sections are powerful conversion tools — analyze by function |
| **CTA Without Style** | CTA listed as text without style classification | Every CTA needs cta_style |
| **Momentum Invisible** | Close elements analyzed individually without momentum pattern | close_architecture must capture emotional escalation |
