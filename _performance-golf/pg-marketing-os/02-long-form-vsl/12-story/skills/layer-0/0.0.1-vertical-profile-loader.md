# Layer 0.0.1 — Vertical Profile Loader

**Version:** 1.0
**Created:** 2026-02-20
**Layer:** 0 (Foundation — FIRST microskill to execute)
**Type:** Configuration Loading
**Purpose:** Load the vertical profile that determines which persona panel, specimens, taste defaults, and anti-slop rules apply to this project

---

## Why This Exists

The CopywritingEngine serves 5 verticals: Golf, Health, Finance, Personal Development, and Technology. Each vertical has different optimal personas, different specimen libraries, different taste constraints, and different anti-slop rules. Loading the wrong specimens (e.g., health specimens for golf copy) causes cross-niche contamination that degrades output quality.

The Vertical Profile Loader is the FIRST thing that executes, before upstream packages, before specimens, before anything else — because everything downstream depends on knowing which vertical this project targets.

---

## Execution Protocol

```
0.0.1 VERTICAL PROFILE LOADER:

  STEP 1: IDENTIFY VERTICAL
    - CHECK: Does the project brief specify a vertical?
    - Valid verticals: golf | health | finance | personal-dev | technology
    - Source: project brief, Soul.md, or explicit human instruction

  STEP 2: LOAD PROFILE (if vertical specified)
    a. READ: ~system/verticals/[vertical].md
    b. EXTRACT: persona_panel (which 7 personas to use)
    c. EXTRACT: specimen_directories (system_1_override, system_2_override)
    d. EXTRACT: taste_defaults (merge into Soul.md if Soul.md exists)
    e. EXTRACT: vertical_anti_slop (add to anti-slop validator)
    f. EXTRACT: niche_matching_guide (specimen selection priorities)

  STEP 3: CONFIGURE DOWNSTREAM
    - 0.2.6 System 1 Loading: Use system_1_override path if vertical specimens exist for this skill
    - 0.2.7 System 2 Loading: Use persona_panel for which personas to load, system_2_override for specimens
    - Arena: Load persona specs from persona-registry/[name].md per panel config
    - Anti-Slop: Merge vertical_anti_slop into skill-specific anti-slop rules

  STEP 4: WRITE OUTPUT
    - Write verification file to layer-0-outputs/0.0.1-vertical-profile.md

  FALLBACK (no vertical specified):
    - DEFAULT to universal configuration (standard 6 personas + Architect)
    - WARN: "No vertical specified — using universal specimen library"
    - Anti-slop: standard rules only
    - Specimens: universal 0.2.6 and 0.2.7 files
```

## Gate 0 Enhancement

When 0.0.1 exists in a skill's Layer 0, Gate 0 validation MUST include:

```
GATE 0 REQUIREMENTS (with Vertical Profile):
[ ] Vertical profile loaded (0.0.1) OR universal fallback confirmed
[ ] Persona panel configuration extracted
[ ] Specimen directory overrides identified
[ ] Taste defaults merged (if applicable)
[ ] Anti-slop rules merged (if applicable)
[ ] 0.0.1 output file exists with verification section

IF vertical profile loading fails:
  → LOG WARNING: "Vertical profile failed to load — falling back to universal"
  → PROCEED with universal configuration (does not block execution)
  → This is NOT a HALT condition — it degrades optimization, not correctness
```

---

## Forbidden Behaviors

1. Executing ANY other Layer 0 microskill before 0.0.1 completes
2. Loading specimens without checking for vertical overrides
3. Loading persona specs from ARENA-PERSONA-PANEL.md instead of persona-registry/[name].md (when vertical is active)
4. Ignoring vertical anti-slop rules during generation
5. Overriding vertical taste_defaults without human approval
6. Claiming vertical is loaded without producing 0.0.1 output file
