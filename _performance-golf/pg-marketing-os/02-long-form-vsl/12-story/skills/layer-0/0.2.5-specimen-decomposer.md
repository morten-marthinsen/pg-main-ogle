# Layer 0.2.5 — Specimen Decomposer

**Version:** 1.0
**Layer:** 0 (Foundation)
**Type:** Specimen Analysis
**Domain:** Story
**Dependencies:** vault_story_intelligence (from 0.2-vault-intelligence-loader.md)
**Output:** specimen_decomposition (structural templates, technique catalog, power elements)

---

## Purpose

Decompose top-scoring story specimens from the vault-intelligence into reusable structural templates, technique catalogs, and power elements. Story specimens contain rich sequential data — story beats with emotional states, arc shapes, protagonist types, and story roles. This skill extracts the narrative architecture that makes elite direct response stories persuasive, not the specific stories.

**This skill bridges the gap between:** vault data (elite story specimens with beat-level annotation) and generation layers (what Claude needs to construct original stories with the same structural power).

## Decomposition Protocol

### Phase 1: Specimen Selection & Tiering

```
IF tier_metadata present (Phase B loader active):
  LOAD gold_specimens → full decomposition
  LOAD silver_specimens → template-only decomposition
ELSE (pre-upgrade fallback):
  SELECT top 5 specimens by position in vault-intelligence as pseudo-Gold
  SELECT next 10 specimens as pseudo-Silver
  LOG "Operating in pre-upgrade mode — tiering estimated from array position"

VERIFY each specimen has:
  - full_story_text (required — skip if absent)
  - story_beats[] (required — at least 3 beats)
  - emotional_arc (required)
  - protagonist (preferred)
  - story_role (preferred)
  - source_file (required for provenance)
  - niche (required for type-matching)
```

### Phase 2: Structural Template Extraction

For each Gold specimen:

```
ANALYZE story_beats[] and full_story_text to extract:

  beat_skeleton:
    FOR each beat in story_beats[]:
      - beat_type: string (from specimen — e.g., setup, complication, crisis,
                           discovery, transformation, proof, lesson)
      - function: string (what this beat accomplishes in the persuasion arc)
      - emotional_state: string (from specimen)
      - position: integer (from specimen)
      - proportion: float (approximate % of full_story_text)

  arc_architecture:
    arc_shape: string (from emotional_arc — e.g., "descent_then_rise",
                       "steady_escalation", "crisis_to_discovery")
    emotional_low_point: string (which beat contains the nadir)
    emotional_high_point: string (which beat contains the peak)
    pivot_mechanism: string (what drives the turn from low to high)

  protagonist_construction:
    protagonist_type: string (from protagonist field)
    identification_method: string (how the story makes reader identify with protagonist)
    credibility_markers: string (what makes this protagonist believable)

  story_role_architecture:
    role: string (from story_role — e.g., origin_story, proof_story,
                  identification_story, mechanism_story)
    persuasion_function: string (what this story type accomplishes in the campaign)
    position_in_campaign: string (where this story type typically appears)

For Silver specimens:
  Extract beat_skeleton and arc_architecture ONLY
  SKIP protagonist_construction detail
  SKIP proportion analysis
```

### Phase 3: Technique Catalog Compilation

Aggregate across ALL decomposed specimens:

```
beat_types:
  FOR each unique beat_type found:
    - beat_name: string
    - count: integer (how many stories use this beat)
    - function: string (what it accomplishes)
    - typical_position: string (early, middle, late, climax)
    - implementation_pattern: string (how to write this beat type)

arc_shapes:
  FOR each unique emotional_arc pattern:
    - arc_name: string
    - count: integer
    - beat_sequence: string (typical beat order for this arc)
    - best_for: string (which story roles this arc serves best)
    - specimens_using: [source_file references]

protagonist_types:
  FOR each unique protagonist construction:
    - type_name: string
    - identification_mechanism: string (why readers connect)
    - specimens_using: [source_file references]

story_roles:
  FOR each unique story_role:
    - role_name: string
    - count: integer
    - persuasion_function: string
    - typical_arc: string (which arc shape pairs best)
    - campaign_position: string (where in the copy this type appears)

transition_techniques:
  FOR each method of entering/exiting a story within copy:
    - technique: string
    - entry_example: string (how the story begins within the larger copy flow)
    - exit_example: string (how the story transitions back to sales argument)
    - source_file: string
```

### Phase 4: Power Element Extraction

From Gold specimens ONLY:

```
pivot_sentences:
  FOR each Gold specimen:
    - verbatim_pivot: string (the sentence(s) where the story turns)
    - beat_context: string (which beat contains the pivot)
    - emotional_shift: string ("from → to")
    - source_file: string

identification_hooks:
  FOR each Gold specimen:
    - verbatim_hook: string (the sentence(s) that create reader-protagonist identification)
    - identification_type: string (shared_pain, shared_aspiration, shared_frustration)
    - position_in_story: string
    - source_file: string

emotional_state_language:
  SCAN Gold specimen beat_text fields for:
    - word_or_phrase: string (specific language evoking emotional state)
    - emotional_state: string
    - beat_type: string (which beat type uses this language)

story_openers:
  FOR each Gold specimen:
    - verbatim_opener: string (first 1-2 sentences of full_story_text)
    - opening_technique: string (in_medias_res, scene_setting, shocking_fact, question)
    - source_file: string
```

## Quality Gates

- [ ] Minimum 3 Gold specimens fully decomposed (beat skeleton + arc + protagonist + power elements)
- [ ] Minimum 5 Silver specimens template-decomposed (beat skeleton + arc)
- [ ] All beat skeletons have minimum 3 beats with emotional states
- [ ] Technique catalog has minimum 4 beat types cataloged
- [ ] Technique catalog has minimum 2 arc shapes documented
- [ ] Power elements contain minimum 3 verbatim pivot sentences from Gold specimens
- [ ] Power elements contain minimum 3 story openers from Gold specimens
- [ ] Every power element includes source provenance (source_file)
- [ ] Context window budget respected (templates ~40%, techniques ~35%, elements ~25%)
- [ ] No raw full_story_text passed through undecomposed

---

## Anti-Patterns

| Anti-Pattern | Signal | Fix |
|---|---|---|
| **Raw Passthrough** | Full story text dumped to downstream skills | Decompose into beat skeleton + techniques + elements |
| **Beat Without Function** | Beats listed by type without persuasion function | Every beat needs function (what it accomplishes in the sales argument) |
| **Arc Without Pivot** | Emotional arc documented without identifying the turn point | Every arc needs pivot_mechanism explicitly identified |
| **Protagonist Without Identification** | Protagonist described without explaining reader connection | identification_method required for every protagonist |
| **Story Without Role** | Story decomposed without campaign context | story_role_architecture required — why does this story exist in the copy? |
