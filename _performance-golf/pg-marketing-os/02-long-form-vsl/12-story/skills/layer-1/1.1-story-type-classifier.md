# Layer 1.1 — Story Type Classifier

**Version:** 1.0
**Layer:** 1 (Story Architecture)
**Type:** Classification & Strategy
**Dependencies:** upstream_data (from 0.1), vault_story_intelligence (from 0.2), teaching_index (from 0.3), validation_report (from 0.4)
**Output:** story_type_classification.json

---

## Purpose

Classify the optimal story type based on niche context, mechanism type, available campaign assets, vault patterns, market sophistication, and proof inventory strength. The story type is the single most consequential architectural decision in the entire 12-story skill — it determines beat structure, character architecture, emotional arc shape, construction sequence, and every downstream microskill's behavior. A misclassified story type produces a narrative that feels forced, artificial, or misaligned with the campaign's persuasion goal.

The classifier must answer ONE question: **"What type of story will most effectively make the prospect FEEL that the mechanism is real?"**

This is NOT about what story the copywriter wants to tell. It is about what story architecture the PROSPECT needs to experience in order to move from skepticism/indifference to belief/desire.

---

**Primary Persona:** Sarah Chen — Competitive Intelligence Analyst
Brings pattern recognition across competitor landscapes and niche-specific story conventions. Identifies which story types dominate specific niches, which are saturated, and which represent fresh opportunities. ## Execution Protocol

### Step 1: Build Classification Parameter Set

```
EXTRACT from upstream_data:
  niche_context:
    niche = product_context.niche (or infer from mechanism + promise)
    sub_niche = product_context.sub_niche
    schwartz_stage = product_context.market_sophistication

  mechanism_profile:
    mechanism_name = mechanism.name
    mechanism_type = mechanism.type
    e5_type = mechanism.e5_type
    uniqueness_strength = mechanism.uniqueness_claim (assessed 1-10)
    discovery_narrative_available = boolean (is there a story of HOW it was found?)
    mechanism_origin = string (where/how was mechanism discovered?)

  proof_profile:
    overall_strength = proof.overall_strength
    knockout_proof_type = proof.knockout_proof.type
    has_authority_credentials = boolean (doctor, PhD, institution, etc.)
    has_statistical_proof = boolean (performance data, percentages, studies)
    has_experiment_data = boolean (beta test, live experiment, documented trial)
    has_personal_narrative = boolean (protagonist's story of discovery/transformation)
    has_vulnerability_arc = boolean (shame, failure, struggle in the narrative)
    has_villain_profile = boolean (root_cause.villain_profile exists)
    has_beta_test_results = boolean (documented group results with specific numbers)

  story_asset_inventory:
    CATALOG all available narrative assets:
      - discovery_location (exotic location, lab, clinic, etc.)
      - expert/yoda_figure (named authority who revealed the mechanism)
      - personal_struggle (protagonist's pain before discovery)
      - experiment_documentation (dates, numbers, percentages)
      - transformation_evidence (before/after, results timeline)
      - supporting_characters (spouse, skeptic, beta testers)

  lead_story_tease:
    what_was_teased = lead_package.discovery_story_teased
    open_loops_to_resolve = lead_package.open_loops
    # The story must EXPAND what the lead teased, not repeat it
```

### Step 2: Score Each Story Type for Fit

```
FOR each of 10 story types, calculate fit_score (0-100):

1. STANDARD_DISCOVERY (Trip -> Finding Yoda -> Cry for Help -> Yoda Reveals -> Verification -> Root Cause):
   niche_fit: +30 if health supplements, +20 if relationships, +15 if self_improvement, +10 otherwise
   asset_fit: +25 if discovery_location EXISTS AND expert/yoda_figure EXISTS
   mechanism_fit: +20 if mechanism was genuinely discovered through travel/encounter
   proof_fit: +15 if has_personal_narrative AND has_vulnerability_arc
   schwartz_fit: +10 if Stage 2-4 (discovery stories still fresh)
   BEST FOR: Health supplements (Leptitox, Erect on Demand), relationship products (Elixir of Eros)
   VAULT EXEMPLARS: leptitox (Malaysia trip), elixir_of_eros (Athens trip), erect_on_demand (Peru trip)

2. ACCIDENTAL_DISCOVERY (Accident -> Finding Yoda -> Yoda Reveals -> Verification -> Root Cause):
   niche_fit: +25 if health, +20 if self_improvement, +15 otherwise
   asset_fit: +30 if inciting_accident/event EXISTS AND it is genuinely unplanned
   mechanism_fit: +20 if mechanism discovery was authentically accidental
   proof_fit: +15 if has_personal_narrative
   schwartz_fit: +10 if Stage 2-4
   BEST FOR: Health products where discovery was serendipitous
   VAULT EXEMPLARS: fat_flusher (African plant accidental discovery)

3. STRAIGHT_TO_EXPERT (Finding Yoda -> Trip/New Reality -> Reveals -> Verification -> Root Cause):
   niche_fit: +25 if health with medical authority, +25 if self_improvement with guru, +15 otherwise
   asset_fit: +30 if expert_name EXISTS AND authority_credentials are strong
   mechanism_fit: +20 if mechanism comes from recognized expert/authority
   proof_fit: +15 if has_authority_credentials
   schwartz_fit: +10 if Stage 2-3 (authority still trusted)
   BEST FOR: Doctor-fronted health products, expert-fronted instruction
   VAULT EXEMPLARS: resurge (celebrity sleep specialist), montezumas_secret (doctor reveals), desire_system (expert teaches)

4. PROOF_STORY_LIVE_EXPERIMENT (Experiment Setup -> First Data -> Escalation -> Comprehensive Results -> Transparency -> Simplicity):
   niche_fit: +30 if wealth/investment, +25 if sports_instruction, +20 if health with clinical data
   asset_fit: +30 if has_beta_test_results AND has_experiment_data with specific numbers
   mechanism_fit: +15 if mechanism produces MEASURABLE, DOCUMENTABLE outputs
   proof_fit: +20 if has_statistical_proof AND overall_strength >= 80
   schwartz_fit: +5 if Stage 1-3 (data still impresses), +10 if Stage 4-5 (transparency differentiates)
   BEST FOR: Finance (documented trades), golf instruction (live experiment proving method), health (clinical trials)
   VAULT EXEMPLARS: makepeace_speed_profits (487 beta-testers, 23 documented trades)
   CRITICAL: NEVER classify as proof_story without documented experiment data with specific numbers

5. ORIGIN_STORY (Why -> Personal Mission -> Obstacle -> Breakthrough -> Mission Continues):
   niche_fit: +20 universal — works across all niches
   asset_fit: +25 if founder/creator has compelling personal WHY
   mechanism_fit: +20 if mechanism creation was driven by personal need
   proof_fit: +15 if personal transformation is documented
   schwartz_fit: +20 if Stage 4-5 (origin stories build trust in saturated markets)
   BEST FOR: Founder-fronted brands, products born from personal struggle

6. TRANSFORMATION_STORY (Before State -> Catalyst -> Discovery -> Application -> After State):
   niche_fit: +25 if health transformation, +25 if self_improvement, +20 otherwise
   asset_fit: +25 if clear before/after documentation EXISTS
   mechanism_fit: +20 if mechanism produces visible/feelable transformation
   proof_fit: +20 if transformation evidence is specific and verifiable
   schwartz_fit: +10 if Stage 2-4
   BEST FOR: Weight loss, fitness, personal development, skill acquisition

7. REVELATION_STORY (Expert Discovers -> Hidden Truth -> Evidence Cascade -> Implications):
   niche_fit: +30 if health with scientific mechanism, +25 if financial with hidden data
   asset_fit: +25 if expert_name EXISTS AND discovery is genuinely surprising
   mechanism_fit: +25 if mechanism contradicts conventional wisdom
   proof_fit: +15 if has_authority_credentials AND evidence supports revelation
   schwartz_fit: +5 if Stage 1-2, +15 if Stage 3-5 (revelations cut through noise)
   BEST FOR: Counter-intuitive mechanisms, scientific discoveries, hidden truths

8. WARNING_STORY (Threat Identified -> Evidence -> Escalation -> Consequence -> Escape Route):
   niche_fit: +30 if financial/political crisis, +25 if health threat/toxin
   asset_fit: +25 if villain_profile EXISTS AND threat is documented
   mechanism_fit: +20 if mechanism provides PROTECTION from identified threat
   proof_fit: +15 if has_statistical_proof supporting the warning
   schwartz_fit: +10 if Stage 3-5 (warning stories penetrate sophisticated defenses)
   BEST FOR: Financial newsletters, health conspiracies, political commentary

9. UNDERDOG_VULNERABILITY (Shame/Failure -> Rock Bottom -> Unlikely Discovery -> Tentative Try -> Triumph):
   niche_fit: +25 if health with personal struggle, +30 if relationships, +25 if self_improvement
   asset_fit: +30 if has_vulnerability_arc AND protagonist mirrors prospect's shame
   mechanism_fit: +15 if mechanism was found through personal desperation
   proof_fit: +20 if transformation from shame to dignity is documented
   schwartz_fit: +10 if Stage 3-5 (vulnerability stories cut through skepticism)
   BEST FOR: ED products, weight loss (shame-driven), relationship failures, financial recovery

10. HYBRID (combines elements of 2+ types):
    niche_fit: +15 for any sophisticated market
    asset_fit: +25 if multiple asset types available (narrative + data + credentials)
    mechanism_fit: +20 if mechanism supports multiple framing approaches
    proof_fit: +15 if multiple proof types exist
    schwartz_fit: +25 if Stage 4-5 (hybrid needed to penetrate sophisticated defenses)
    BEST FOR: Stage 4-5 markets where single types are saturated
    CRITICAL: Identify which 2 types are being combined and which is DOMINANT
```

### Step 3: Cross-Reference with Vault Patterns

```
LOAD vault_story_intelligence.story_type_taxonomy

FOR each story type scored in Step 2:
  SEARCH vault extractions for same story type + similar niche
  IF match found:
    vault_alignment_bonus = +10
    vault_reference = extraction source identifier
    vault_evidence_notes = effectiveness observations from extraction
    vault_beat_structure = documented beat sequence from extraction
  IF no match:
    vault_alignment_bonus = 0
    vault_reference = "no direct vault match"

ALSO CHECK:
  - Which story types are OVERREPRESENTED in vault for this niche
    (caution: may be saturated in the market)
  - Which story types are UNDERREPRESENTED
    (opportunity if assets support it, or poor fit indicator)
  - Which vault exemplar is CLOSEST to this campaign's asset profile
```

### Step 4: Apply Niche-Specific Decision Matrix

```
NICHE DECISION MATRIX:

  HEALTH (supplements, remedies, natural cures):
    PRIMARY recommendation: standard_discovery OR accidental_discovery
    WHY: Health supplement buyers respond to "exotic location + ancient wisdom +
         modern verification" narrative arc. The trip/yoda/secret pattern is
         proven across dozens of TIER1 controls in this niche.
    SECONDARY: straight_to_expert (if doctor-fronted with strong credentials)
    AVOID: proof_story_live_experiment (unless clinical trial data exists)
    VAULT VALIDATION: leptitox, erect_on_demand, fat_flusher all use discovery format

  WEALTH (investment, finance, trading):
    PRIMARY recommendation: proof_story_live_experiment
    WHY: Finance prospects demand documented results with specific dates, numbers,
         percentages. The Makepeace beta-test pattern with data cascade is the
         proven format. "Show me the money" is the core psychology.
    SECONDARY: warning_story (if market crash / threat angle)
    AVOID: standard_discovery (feels unserious for finance)
    VAULT VALIDATION: makepeace_speed_profits uses proof_story format

  RELATIONSHIPS (dating, attraction, intimacy):
    PRIMARY recommendation: standard_discovery OR underdog_vulnerability
    WHY: Relationship prospects need to see themselves in a protagonist who
         overcame the same shame/failure they experience. The vulnerability arc
         creates deep identification before the mechanism reveal.
    SECONDARY: transformation_story (before/after relational dynamics)
    AVOID: proof_story_live_experiment (feels clinical for relationships)
    VAULT VALIDATION: elixir_of_eros uses standard_discovery, desire_system uses straight_to_expert

  SELF_IMPROVEMENT (goals, productivity, mindset):
    PRIMARY recommendation: straight_to_expert OR transformation_story
    WHY: Self-improvement prospects seek authority figures who have achieved
         what they want. Expert-reveals-truth or documented transformation
         creates aspirational identification.
    SECONDARY: origin_story (founder's personal journey)
    AVOID: warning_story (too negative for aspirational market)

  SPORTS_INSTRUCTION (golf, fitness, athletic performance):
    PRIMARY recommendation: proof_story_live_experiment OR transformation_story
    WHY: Sports instruction prospects want to see the method WORKING on real
         people with documented performance improvements. "Live experiment"
         with measured results is the proven pattern.
    SECONDARY: straight_to_expert (if coach/instructor has elite credentials)
    AVOID: standard_discovery with exotic location (feels forced for sports)
```

### Step 5: Validate Asset-Type Alignment

```
CRITICAL VALIDATION: The selected story type must be SUPPORTABLE by available assets.

FOR the top-scoring story type:
  CHECK asset requirements:

  standard_discovery REQUIRES:
    - Discovery location or inciting trip event
    - Yoda/guide figure (named or nameable)
    - Personal struggle/cry for help moment
    - Mechanism revelation scene
    - Verification/research sequence
    IF any MISSING: FLAG "asset gap" and reduce score by 15

  accidental_discovery REQUIRES:
    - Genuine accidental event (not forced)
    - Guide figure encountered through the accident
    - Verification sequence
    IF any MISSING: FLAG "asset gap" and reduce score by 15

  straight_to_expert REQUIRES:
    - Named expert with verifiable credentials
    - Expert's revelation or teaching moment
    - Verification of expert's claims
    IF any MISSING: FLAG "asset gap" and reduce score by 15

  proof_story_live_experiment REQUIRES:
    - Documented experiment with specific numbers, dates, percentages
    - Beta-tester or participant group (named or described)
    - Multiple data points showing ascending results
    - At least one loss/limitation acknowledged (transparency)
    IF any MISSING: FLAG "CRITICAL asset gap — cannot classify as proof_story"
    and reduce score by 30

  origin_story REQUIRES:
    - Founder/creator personal narrative
    - Clear personal WHY driving the product/method
    IF any MISSING: FLAG "asset gap" and reduce score by 15

  transformation_story REQUIRES:
    - Documented before state
    - Documented after state
    - Specific results/changes
    IF any MISSING: FLAG "asset gap" and reduce score by 15

  revelation_story REQUIRES:
    - Expert with credentials to make the revelation credible
    - Evidence supporting the counter-intuitive claim
    IF any MISSING: FLAG "asset gap" and reduce score by 15

  warning_story REQUIRES:
    - Documented threat with evidence
    - Named or identifiable villain/cause
    - Escape route connected to the mechanism
    IF any MISSING: FLAG "asset gap" and reduce score by 15

  underdog_vulnerability REQUIRES:
    - Genuine vulnerability/shame moment
    - Rock bottom scene
    - Transformation evidence from shame to dignity
    IF any MISSING: FLAG "asset gap" and reduce score by 15

IF asset validation reduces top scorer below second scorer:
  PROMOTE second scorer to primary selection
  DOCUMENT reason: "Primary type lacked required assets; promoted runner-up"
```

### Step 6: Verify Lead-Story Integration

```
CHECK lead_package.discovery_story_teased:
  WHAT did the lead tease about the story?
  The selected story type MUST deliver on that tease.

  IF lead teased a discovery journey:
    story type must be standard_discovery, accidental_discovery, or straight_to_expert
  IF lead teased data/results:
    story type must be proof_story_live_experiment or transformation_story
  IF lead teased an expert's revelation:
    story type must be straight_to_expert or revelation_story
  IF lead teased a warning/threat:
    story type must be warning_story

  FLAG MISMATCH if selected type contradicts what the lead teased.
  Per Georgi framework: the lead TEASES the story, the story DELIVERS.
  Mismatch = broken promise to the reader.
```

### Step 7: Calculate Final Scores and Select

```
FOR each story type:
  final_fit_score = base_fit_score + vault_alignment_bonus - asset_gap_penalty

SORT by final_fit_score DESC

SELECT primary_type = highest scoring type
SELECT runner_up = second highest scoring type

IF top two scores are within 5 points:
  FLAG "close classification — consider hybrid approach"
  IF schwartz_stage >= 4:
    CONSIDER promoting to HYBRID combining elements of both top types

VALIDATE selection:
  - primary_type.final_fit_score >= 60 (minimum viable classification)
  - primary_type has at least one vault reference supporting it
  - primary_type aligns with niche context per decision matrix
  - primary_type is supportable by available campaign assets
  - primary_type delivers on what the lead teased

CONFIDENCE LEVEL:
  high = primary type leads by 15+ points
  medium = primary type leads by 5-15 points
  low = primary type leads by < 5 points
```

- [ ] All 10 story types scored with specific fit values (zero types skipped)
- [ ] Selected type has fit_score >= 60 (minimum viable classification)
- [ ] Vault reference provided supporting classification (not "none" or "no match")
- [ ] Classification rationale cites specific scores and selection logic (no vague justification)
- [ ] Runner-up identified with fit score and explanation for non-selection
- [ ] Niche decision matrix consulted and alignment documented
- [ ] Schwartz stage considered in scoring (not ignored)
- [ ] Available story assets inventoried and type-asset alignment validated
- [ ] No story type classified without supporting assets (especially proof_story)
- [ ] Lead-story integration validated — story type delivers on what the lead teased
- [ ] Confidence level accurately reflects score margin (high = 15+ lead, medium = 5-15, low = < 5)
- [ ] If hybrid selected, component types identified with dominant type specified
- [ ] Asset gaps flagged with mitigation strategy if any gaps exist

## Anti-Patterns

| Anti-Pattern | What It Looks Like | Why It Fails |
|---|---|---|
| **Default to Discovery** | Always selecting standard_discovery because it is "common" in the vault | Ignores niche-specific advantages; finance and sports instruction almost never use standard discovery |
| **Proof Story Without Data** | Selecting proof_story_live_experiment without documented experiment results | The Makepeace pattern REQUIRES specific dates, numbers, percentages, and documented participant results; without data, the story has no substance |
| **Ignoring Schwartz Stage** | Selecting standard_discovery for a Stage 5 health market | Stage 5 prospects have seen every discovery story variant; may need hybrid, revelation, or vulnerability to penetrate |
| **Vault Mimicry** | Copying an exact vault story type without adapting to this campaign's assets | Vault patterns are reference points, not templates; the story must fit THIS campaign's specific assets and mechanism |
| **Force-Fitting Vulnerability** | Selecting underdog_vulnerability when no genuine shame/struggle exists | Vulnerability stories require REAL emotional pain; manufactured vulnerability feels manipulative and destroys trust |
| **Ignoring Lead Tease** | Selecting a story type that contradicts what the lead teased | The lead promised the reader a certain type of story; delivering something different breaks trust and creates dissonance |
| **Single-Factor Classification** | Choosing based only on niche without considering mechanism, proof, assets, or Schwartz stage | All classification criteria must inform the decision; single-factor choices produce misaligned stories |
| **Asset-Blind Selection** | Selecting a type without verifying the required assets exist | A perfectly classified type with missing assets produces a story full of fabricated details that feel hollow |
