# Layer 4.4 — Vault Pattern Comparator (Story)

**Version:** 1.0
**Layer:** 4 (Validation & Assembly)
**Type:** Competitive Benchmarking & Differentiation Gate
**Dependencies:** full draft story text (from Layer 3 output), vault_story_intelligence (from 0.2), story_classification (from 1.1), story_beats (from 1.2), character_architecture (from 1.3), emotional_arc (from 1.4), sensory_detail_calibration (from 3.1), suspense_pacing (from 3.2), proof_integration (from 3.3)
**Output:** vault_comparison.json

---

## Purpose

Compare the engineered campaign story against elite story patterns from TIER1 vault extractions. Benchmark the story's quality dimensions against the best stories in the vault -- discovery stories from Leptitox, Elixir of Eros, and Erect on Demand; proof/data cascade stories from Makepeace's Speed Profits; transformation narratives from health and relationship niches. The story should be INFORMED by vault patterns but DIFFERENTIATED from them. Exact replication of a vault story's structure is derivative. Sharing no patterns with any vault story is uninformed. The goal is comparable QUALITY with unique EXECUTION. A story that matches vault quality on every dimension but executes with originality is the ideal output.

---

## Primary Persona

**Sarah Chen -- Competitive Intelligence Analyst**
Maps the story's structural DNA against the elite competitive landscape. Identifies which patterns were adopted from vault exemplars, which were adapted, and which are genuinely novel. Ensures the story would hold up as original work in a market where these elite controls already exist.

## Secondary Persona

**Marcus Webster -- Pattern Synthesis Analyst**
Provides systematic pattern comparison methodology. Ensures the comparison is structured and quantitative, not impressionistic. Identifies the specific dimensions where the story excels or falls short relative to vault benchmarks.

## Execution Protocol

### Step 1: Select Comparison Set

```
FROM vault_story_intelligence, select 3-5 most relevant vault stories for comparison.

SELECTION CRITERIA (ranked by priority):

  1. SAME STORY TYPE: Select vault stories that share the classified story type
     - If classified as "standard_discovery" → prioritize vault standard discovery stories
     - If classified as "proof_story_live_experiment" → prioritize Makepeace patterns
     - If classified as "hybrid" → select components that match hybrid elements

  VAULT EXEMPLAR MAP:
    standard_discovery:
      - Leptitox (Malaysia firefighter → plant discovery → weight loss mechanism)
      - Elixir of Eros (Athens trip → fanfiction discovery → desire mechanism)
      - Erect on Demand (Peru ceremony → ancient remedy → ED mechanism)
    accidental_discovery:
      - Fat Flusher (accidental event → African plant → weight loss mechanism)
    straight_to_expert:
      - Resurge (celebrity sleep specialist → sleep/weight mechanism)
      - Montezuma's Secret (doctor → ancient remedy → ED mechanism)
      - Desire System (expert → emotional contagion → relationship mechanism)
    proof_story_live_experiment:
      - Makepeace Speed Profits (487 beta-testers → 23 trades → currency mechanism)

  2. SAME NICHE: If vault stories exist in the same or adjacent niche, prioritize
     - Same niche = direct competitor comparison
     - Adjacent niche = structural pattern comparison

  3. HIGHEST QUALITY: Among matching stories, prioritize those with:
     - quality_score >= 85
     - extraction_confidence >= 85

  4. DIVERSITY: Ensure at least ONE comparison story is a DIFFERENT type
     - This validates the choice to use the selected type over alternatives

RECORD comparison_set:
  - vault_story_id: string
  - vault_story_name: string
  - story_type: string
  - niche: string
  - quality_score: float
  - selection_reason: string
```

### Step 2: Compare Story Type Match

```
FOR each story in comparison_set:

  EVALUATE: Does our story type match what elite controls use for this niche?

  COMPARE:
    - Our story type vs. vault story types used in this niche
    - If our type differs from vault preference → is the difference justified?
    - If our type matches vault preference → do we execute it differently?

  ASSESS type fitness:
    - Does this story type serve the campaign's needs?
    - Does the vault evidence support this type choice for this niche?
    - Would a different type have been more effective? (from comparison with different-type vault story)

  SCORE type_match (1-10):
    10: Our type choice is perfectly suited AND well-justified by vault patterns
    8-9: Strong type choice with solid vault support
    6-7: Reasonable type choice but vault evidence is mixed
    4-5: Type choice is adequate but vault evidence suggests a different type might be stronger
    2-3: Type choice conflicts with vault evidence for this niche
    1: Type is clearly wrong for this niche based on vault patterns

  RECORD:
    - type_match_score: integer[1-10]
    - vault_preferred_types: array (what types do vault stories use for this niche)
    - type_match_notes: string
```

### Step 3: Compare Beat Structure

```
FOR each same-type vault story in comparison_set:

  COMPARE beat sequences:
    - Our beat sequence vs. vault beat sequence
    - Which beats are shared?
    - Which beats are in our story but not in the vault story?
    - Which beats are in the vault story but not in ours?
    - Is our beat ORDER consistent with proven patterns?
    - Are there beats in our story that have no vault precedent?

  EVALUATE structural fidelity:
    - Does our beat structure follow the classified type's proven format?
    - Where do we diverge from the vault pattern?
    - Are divergences IMPROVEMENTS or DEGRADATIONS?

  SCORE beat_structure (1-10):
    10: Beat structure is as tight and effective as the best vault patterns --
        every beat serves a purpose, sequence creates maximum narrative impact
    8-9: Strong beat structure with deliberate, effective variations from vault patterns
    6-7: Adequate beat structure but vault stories have tighter or more effective sequences
    4-5: Beat structure has gaps or unnecessary beats compared to vault benchmarks
    2-3: Beat structure is significantly weaker than vault patterns
    1: Beat structure is disorganized or does not follow any proven pattern

  RECORD:
    - beat_structure_score: integer[1-10]
    - shared_beats: array
    - unique_to_our_story: array
    - missing_from_our_story: array
    - beat_structure_notes: string
```

### Step 4: Compare Protagonist Design

```
COMPARE our protagonist against vault protagonist archetypes:

  DIMENSIONS:
    - Role: Same archetype (everyman, authority, reluctant discoverer)?
    - Prospect mirroring: How effectively does our protagonist mirror the prospect
      compared to vault protagonists?
    - Vulnerability: Is our protagonist's vulnerability as compelling as vault exemplars?
    - Specificity: Is our protagonist as specifically drawn as vault exemplars?
    - Relatability: Would the target prospect relate to our protagonist as readily
      as to vault protagonists?

  VAULT PROTAGONIST BENCHMARKS:
    - Leptitox: Firefighter wife -- everywoman with specific health crisis,
                relatable vulnerability, clear stakes
    - Elixir of Eros: Man struggling with marriage -- specific relationship failure,
                       humiliation detail, desperate search
    - Makepeace: "Ordinary people just like you and me" -- explicit ordinariness proof
    - Erect on Demand: Man facing ED -- specific shame moment, relationship at risk
    - Resurge: Celebrity-adjacent expertise -- authority role with relatable problem

  SCORE protagonist_comparison (1-10):
    10: Our protagonist is as specifically drawn, relatable, and compelling as
        the best vault protagonists
    8-9: Strong protagonist that holds its own against vault benchmarks
    6-7: Adequate protagonist but vault exemplars are more specific or compelling
    4-5: Protagonist is below vault benchmark on specificity or relatability
    2-3: Protagonist is significantly less compelling than vault patterns
    1: Protagonist would not compete in a market where vault protagonists exist

  RECORD:
    - protagonist_comparison_score: integer[1-10]
    - closest_vault_protagonist: string
    - our_protagonist_advantages: string
    - our_protagonist_gaps: string
```

```

COMPARE our story's pacing against vault pacing patterns:
```

### Step 11: Assess Mimicry Risk

```
CRITICAL CHECK: Is our story too similar to any single vault story?

MIMICRY INDICATORS:
  - Same story type + same niche + same beat sequence + same protagonist archetype
    + same emotional arc shape = HIGH mimicry risk
  - More than 4 structural elements identical to a single vault story = FLAG
  - Setting or discovery location that mirrors a specific vault story = FLAG
  - Language patterns or dialogue style that echoes a specific vault story = FLAG
  - Protagonist situation nearly identical to a vault protagonist = FLAG

MIMICRY RISK ASSESSMENT:
  LOW: Story is clearly informed by vault patterns but original in execution --
       the reader would never mistake this for a rewrite of a vault story
  MODERATE: Story shares significant structural DNA with one vault story but
            has enough differentiation (setting, characters, mechanism) to stand alone
  HIGH: Story could be mistaken for a variant of a specific vault story --
        a reader familiar with both would notice the parallels
  CRITICAL: Story is essentially a template fill of a vault story with
            mechanism/character names swapped -- must be fundamentally revised

FOR each vault story in comparison set:
  - structural_similarity_percentage: float (0-100%)
  - language_similarity: enum[none, low, moderate, high]
  - setting_similarity: enum[none, low, moderate, high]
  - protagonist_similarity: enum[none, low, moderate, high]
  - overall_mimicry_risk: enum[LOW, MODERATE, HIGH, CRITICAL]

GATE CHECK:
  IF any single vault story has mimicry_risk = CRITICAL → FAIL
  IF any single vault story has mimicry_risk = HIGH → WARN
```

### Step 12: Compile Differentiation Notes

```
DOCUMENT what makes our story DIFFERENT from vault patterns:

  STRUCTURAL DIFFERENTIATORS:
    - What structural choices are unique to our story?
    - What beat variations distinguish our approach?
    - What sequence innovations are present?

  NARRATIVE DIFFERENTIATORS:
    - What setting, location, or context is original?
    - What character choices are novel?
    - What dialogue style or voice distinguishes our story?
    - What sensory approach is distinctive?

  MECHANISM-DRIVEN DIFFERENTIATORS:
    - How does our specific mechanism create unique story opportunities?
    - What about our mechanism requires a different narrative approach?
    - What proof elements are unique to our mechanism?

  EMOTIONAL DIFFERENTIATORS:
    - What emotional arc innovations distinguish our story?
    - What emotional beats are unique to our narrative?
    - What vulnerability or identification angles are original?

  NICHE-ADAPTATION DIFFERENTIATORS:
    - How has the story been adapted for this specific niche?
    - What niche-specific elements are present that vault stories don't have?
    - What audience expectations are uniquely addressed?

RECORD: differentiation_summary: string (comprehensive summary)
RECORD: differentiation_categories: object (per-category notes)
```

### Step 13: Calculate Overall Vault Comparison Score

```
overall_vault_score = WEIGHTED AVERAGE:
  type_match × 0.10
  beat_structure × 0.15
  protagonist_comparison × 0.10
  arc_comparison × 0.15
  sensory_density × 0.10
  pacing_comparison × 0.10
  proof_weave × 0.10
  revelation_comparison × 0.10
  word_count_comparison × 0.05
  (mimicry assessed separately as gate check)

NOTE: Weights emphasize narrative architecture (beat_structure 0.15, arc 0.15)
and content quality (proof_weave 0.10, revelation 0.10) over format metrics
(word_count 0.05, type_match 0.10).

GATE DECISION:
  IF mimicry_risk = CRITICAL for any vault story → FAIL
  IF overall_vault_score >= 7.0 AND mimicry_risk <= MODERATE → PASS
  IF overall_vault_score >= 5.0 AND < 7.0 → PASS_WITH_WARNINGS
  IF overall_vault_score < 5.0 → FAIL (story is below vault benchmark)

NOTE: This is a BENCHMARKING gate, not a pure pass/fail binary.
A score of 6.5 with excellent differentiation and LOW mimicry is more
valuable than a score of 8.5 with CRITICAL mimicry risk. Quality that
is derivative has limited market value.
```

## Quality Gates

- [ ] 3-5 vault stories selected for comparison with documented selection rationale
- [ ] At least 1 comparison story is the SAME type; at least 1 is a DIFFERENT type
- [ ] All 9 comparison dimensions scored independently (1-10) with evidence
- [ ] Mimicry risk assessed against EVERY story in the comparison set across 4 sub-dimensions
- [ ] CRITICAL mimicry risk triggers automatic FAIL
- [ ] Differentiation notes document structural, narrative, mechanism-driven, emotional, and niche-adaptation differentiators
- [ ] Overall vault score calculated with correct weighting (0.10/0.15/0.10/0.15/0.10/0.10/0.10/0.10/0.05)
- [ ] Protagonist comparison includes specificity, vulnerability, and relatability assessment
- [ ] Arc comparison includes shape, range, and peak placement analysis
- [ ] Revelation comparison benchmarks against specific vault exemplar moments
- [ ] Word count compared against vault averages and ranges for this story type
- [ ] Gate result distinguishes between quality failure (below benchmark) and originality failure (mimicry)
- [ ] Differentiation summary is comprehensive enough to explain HOW this story is unique

---

## Anti-Patterns

- **Vault worship:** Penalizing a story for differing from vault patterns when the difference is an intentional, well-reasoned creative choice -- the vault is a benchmark, not a template to copy
- **Mimicry blindness:** Failing to flag a story that shares protagonist situation, setting, discovery format, AND beat sequence with a single vault story just because the mechanism is different -- swapping the mechanism does not make a derivative story original
- **Apples-to-oranges comparison:** Comparing a discovery story's pacing against a proof story's pacing without accounting for type conventions -- discovery stories and data cascade stories have fundamentally different rhythm patterns
- **Score inflation through selection:** Choosing only vault stories that are weaker or from different niches to inflate comparison scores -- the comparison set must include the BEST relevant vault stories
- **Differentiation worship:** Praising novelty that makes the story worse -- being different from vault patterns is only valuable if the difference IMPROVES the story's persuasion function
- **Surface-level protagonist comparison:** Comparing protagonists only on archetype (both are "everyman") without comparing SPECIFICITY -- the archetype can match while the quality of execution differs dramatically
- **Revelation deflation:** Scoring revelation comparison low because our revelation uses a different device than vault stories -- the comparison is about IMPACT and QUALITY, not about using the same specific technique
- **Ignoring niche context:** Scoring story length against vault stories from a different niche where word count conventions differ dramatically
