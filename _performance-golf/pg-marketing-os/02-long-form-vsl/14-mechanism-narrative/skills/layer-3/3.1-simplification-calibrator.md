# Skill 14 — Microskill 3.1: Simplification Calibrator

**Skill:** 11 — Mechanism Narrative
**Microskill:** 3.1 — Simplification Calibrator
**Version:** 1.0
**Layer:** 3 (Refinement & Polish)
**Position:** First microskill in Layer 3
**Type:** Calibration & Quality Refinement
**Dependencies:** All Layer 2 outputs (phase drafts), 1.4-explanation-sequence-mapper (simplification assignments), 0.2-tier1-intelligence-loader (simplification_techniques), 0.4-checkpoint (e_level, scorecard)
**Output:** simplification-calibration-report.json + revised phase drafts

---

## PURPOSE

Calibrates all 6 simplification techniques across the full mechanism narrative to achieve the critical balance: simple enough for the target audience to grasp without effort, NOT so simple that it feels patronizing or loses scientific credibility. This microskill reviews every phase draft from Layer 2, assesses comprehension level, adds or removes simplification as needed, and verifies both the "12-year-old test" and scientific credibility are maintained. It is the quality gate for mechanism graspability.

---

## IDENTITY

**What this IS:**
- A calibration engine that tunes simplification across all narrative phases
- A balance optimizer that holds BOTH simplicity and credibility simultaneously
- A cross-phase consistency enforcer for simplification technique usage

**What this is NOT:**
- Not a rewriter — makes targeted adjustments, not full rewrites
- Not a dumbing-down tool — simplification means clarity, not condescension
- Not a content generator — refines existing Layer 2 drafts

## PROCESS

### Step 1: Audit Current Simplification Usage

```
FOR EACH phase draft, SCAN for simplification technique presence:

TECHNIQUE 1 — binary_reduction:
  SCAN for: either/or constructions, two-option frameworks, on/off language
  Mark each instance with location (phase, paragraph, sentence)
  Count: instances_found
  Expected in: Phase 1 (if paradigm_shift), Phase 3 (if paradigm_shift or data_driven)

TECHNIQUE 2 — single_metaphor_anchor:
  SCAN for: the primary metaphor from Phase 4 Part A
  Mark each instance
  Count: instances_found
  Expected in: Phase 3 (naming), Phase 4 (explanation), Phase 5 (callback)

TECHNIQUE 3 — layered_analogy_chain:
  SCAN for: progressive analogies that build on each other
  Mark each analogy in the chain
  Count: chain_length
  Expected in: Phase 4 (explanation)

TECHNIQUE 4 — just_think_shortcut:
  SCAN for: "Think about it" / "Put simply" / "Bottom line" / "In other words"
  Mark each instance
  Count: instances_found
  Expected in: Phase 4 (Part C), Phase 5 (after proof data)

TECHNIQUE 5 — relatable_body_knowledge:
  SCAN for: references to reader's existing body knowledge
  Mark each instance
  Count: instances_found
  Expected in: Phase 1 (problem), Phase 4 (explanation anchor)

TECHNIQUE 6 — numbered_system:
  SCAN for: numbered components, step counts, list structures
  Mark each instance
  Count: instances_found
  Expected in: Phase 3 (if proprietary), Phase 4 (if system mechanism)

COMPILE audit_results:
  FOR EACH technique:
    assigned_to_phases: [phases where it should appear]
    found_in_phases: [phases where it actually appears]
    instance_count: number
    missing_from: [phases where assigned but absent]
    extra_in: [phases where present but not assigned]
```

### Step 2: Assess Comprehension Level Per Phase

```
FOR EACH phase draft, APPLY comprehension assessment:

METRIC 1 — Jargon Density:
  Count scientific/technical terms per 100 words
  Threshold by E-level:
    E2: max 2 technical terms per 100 words
    E3: max 3 technical terms per 100 words
    E4: max 4 technical terms per 100 words
    E5: max 5 technical terms per 100 words
  Each term MUST be defined within 1 sentence of introduction
  Undefined jargon = FAIL

METRIC 2 — Sentence Complexity:
  Average sentence length (words per sentence)
  Threshold: 12-20 words average (varies by format)
    VSL: 10-15 words (shorter for spoken delivery)
    Magalog: 12-18 words
    Advertorial: 15-22 words
    Sales letter: 12-18 words
  Sentences > 30 words = FLAG for splitting

METRIC 3 — Cause-Effect Chain Length:
  Count steps between initial cause and reader-relevant effect
  Threshold: max 3 steps
  Example PASS: "AMPK activates → fat burning starts → belly fat reduces"
  Example FAIL: "AMPK activates → mTOR pathway inhibited → autophagy
    triggered → lysosomal degradation increases → cellular waste
    cleared → inflammation reduced → belly fat reduces" (6 steps)
  If chain > 3: Insert simplification bridge

METRIC 4 — Metaphor Concreteness:
  Assess whether metaphors use physical, visible, tangible objects
  CONCRETE: "on/off switch," "castle wall," "floodgates," "radio station"
  ABSTRACT: "a process," "a pathway," "a dynamic," "a cascade"
  Abstract metaphors = FLAG for replacement

METRIC 5 — Reader Anchor Points:
  Count moments where explanation connects to reader's existing knowledge
  Minimum per phase:
    Phase 1: 1+ anchor point
    Phase 4: 3+ anchor points (most critical)
    Phase 5: 1+ anchor point
  Anchor point = "You know how [familiar thing]... That's what [mechanism] does"
```

### Step 3: Identify Calibration Gaps

```
COMPARE audit_results to assignments and thresholds:

GAP TYPE A — Missing Technique:
  Technique assigned to phase but not found in draft
  Action: ADD the technique at appropriate location
  Priority: HIGH if primary technique missing, MEDIUM if secondary

GAP TYPE B — Excessive Jargon:
  Jargon density exceeds E-level threshold
  Action: Define undefined terms OR replace with accessible equivalents
  Priority: HIGH — undefined jargon breaks comprehension

GAP TYPE C — Sentence Overload:
  Average sentence length exceeds format threshold
  Action: Split long sentences, prefer active voice
  Priority: MEDIUM — affects readability but not comprehension

GAP TYPE D — Chain Too Long:
  Cause-effect chain exceeds 3 steps
  Action: Insert simplification bridge mid-chain
  Example: "AMPK activates → [bridges to] → belly fat reduces.
    Here's how: [then provide the intermediate steps]"
  Priority: HIGH — long chains lose readers

GAP TYPE E — Abstract Metaphor:
  Metaphor uses non-physical, non-visual language
  Action: Replace with concrete physical equivalent
  Priority: HIGH for Phase 4 Part A (anchor metaphor), MEDIUM elsewhere

GAP TYPE F — Missing Anchor Points:
  Fewer reader anchor points than minimum
  Action: Add "You know how..." connections
  Priority: MEDIUM — improves graspability

GAP TYPE G — Over-Simplification:
  Explanation lacks scientific credibility markers
  Signs: No institutions named, no specific numbers, no researcher names
  Action: ADD credibility markers without increasing complexity
  Priority: HIGH — over-simplification kills believability
```

### Step 4: Apply Calibration Corrections

```
FOR EACH identified gap:

  IF Gap Type A (Missing Technique):
    LOCATE optimal insertion point in phase draft
    INSERT technique application:
      binary_reduction: Add either/or framing sentence
      single_metaphor_anchor: Add metaphor reference sentence
      layered_analogy_chain: Add next analogy layer
      just_think_shortcut: Add "Think about it..." re-compression
      relatable_body_knowledge: Add "You know how..." anchor
      numbered_system: Add numbered breakdown

  IF Gap Type B (Excessive Jargon):
    FOR EACH undefined technical term:
      IF term is essential: ADD inline definition
        Pattern: "[term] — that's the [plain description]"
      IF term is non-essential: REPLACE with accessible equivalent
      IF term is mechanism name: KEEP (already defined in Phase 3)

  IF Gap Type C (Sentence Overload):
    FOR EACH sentence > threshold:
      SPLIT at natural break points (conjunctions, list items)
      Prefer 2 short sentences over 1 long one
      Maintain meaning and flow

  IF Gap Type D (Chain Too Long):
    INSERT simplification bridge at step 3
    Pattern: "In simple terms, [steps 1-3 → reader effect].
      Here's the mechanism: [detailed steps if needed]"

  IF Gap Type E (Abstract Metaphor):
    REPLACE abstract with concrete:
      "process" → "assembly line" / "chain reaction"
      "pathway" → "highway" / "pipeline"
      "cascade" → "domino effect" / "chain of falling dominoes"
      "dynamic" → "tug-of-war" / "balancing act"

  IF Gap Type F (Missing Anchor Points):
    ADD reader-relevant connection:
      "You know that feeling when [familiar experience]?
        That's [mechanism] at work."
      "Remember the last time [common experience]?
        That was because [mechanism connection]."

  IF Gap Type G (Over-Simplification):
    ADD credibility without complexity:
      Name 1 institution in Phase 4
      Add 1 specific number in Phase 5
      Reference 1 researcher name
      These additions should be woven in, not appended
```

### Step 5: Verify Visual Metaphor Correlation

```
FROM tier1_index.cross_cutting_patterns.visual_metaphor_correlation:

  FINDING: Higher visual metaphor scorecard scores predict better
  simplification quality. More metaphors → more complex mechanism
  explainable while maintaining simplicity.

APPLY to current narrative:

  CHECK scorecard.visual_metaphor score:
    IF >= 8: Mechanism has strong metaphor potential
      → Ensure multiple visual metaphors are present (3+)
      → Phase 4 can handle more detail layers (5+)
    IF 6-7: Moderate metaphor potential
      → 2 visual metaphors adequate
      → Phase 4 should have 3-4 detail layers
    IF < 6: Weak metaphor potential
      → Invest extra effort in metaphor development
      → Consider adding metaphors not in original mechanism package
      → Phase 4 should be shorter with 2-3 detail layers
      → Rely more on numbered_system or binary_reduction

  VERIFY: Metaphor count matches metaphor potential
  IF under-metaphored: Add visual metaphors from concrete imagery
  IF over-metaphored: Check for mixed metaphors (contradictory images)
```

### Step 6: Final 12-Year-Old Test + Credibility Check

```
APPLY dual test to calibrated narrative:

TEST 1 — 12-Year-Old Comprehension:
  Read each phase as if you are 12 years old with no science background
  FOR EACH paragraph:
    Can the core idea be understood? YES/NO
    Are there words that need looking up? LIST
    Does the cause-effect make intuitive sense? YES/NO
    Could this be explained to a friend? YES/NO
  PASS: All phases pass, no more than 2 words need definition across narrative

TEST 2 — Skeptical Adult Credibility:
  Read each phase as if you are a moderately skeptical adult
  FOR EACH phase:
    Are institutions/researchers named? YES/NO
    Are specific numbers present? YES/NO
    Does the explanation feel solid (not hand-wavy)? YES/NO
    Would you feel comfortable sharing this with a smart friend? YES/NO
  PASS: Phases 4 and 5 pass all criteria, Phases 1 and 3 pass 3/4

BALANCE CHECK:
  IF Test 1 passes but Test 2 fails: ADD credibility markers
  IF Test 2 passes but Test 1 fails: SIMPLIFY further
  IF both fail: Major calibration needed — flag for attention
  IF both pass: Calibration complete
```

## QUALITY CRITERIA

1. **12-year-old test passes** — every phase comprehensible without prior knowledge
2. **Skeptical adult test passes** — credibility maintained through institutions, numbers, specifics
3. **All assigned techniques present** — no assigned simplification missing from its phase
4. **No undefined jargon** — every technical term defined within 1 sentence
5. **Cause-effect chains <= 3 steps** — no runaway complexity chains
6. **Metaphors are concrete** — physical, visual, tangible images
7. **Not patronizing** — simplification achieves clarity, not condescension

---

## FAILURE MODES

| Failure | Symptom | Resolution |
|---------|---------|------------|
| 12-year-old test fails | Paragraphs require prior knowledge | Add definitions, simplify analogies, shorten chains |
| Credibility test fails | No institutions, no numbers, feels vague | Add named institutions, specific data, researcher names |
| Both tests fail | Major calibration problem | Review Layer 1 assignments, may need different simplification approach |
| Mixed metaphors | "The switch opens the floodgate of the castle wall" | Choose ONE metaphor per concept, maintain consistency |
| Over-calibrated | Too many simplification techniques competing | Reduce to 1 primary per phase, remove redundant techniques |
| Under-calibrated | Techniques assigned but weak application | Strengthen technique application — make it more explicit |
