# Layer 0.2.5 — Specimen Decomposer

**Version:** 1.0
**Layer:** 0 (Foundation)
**Type:** Specimen Analysis
**Domain:** Mechanism Narrative
**Dependencies:** vault_mechanism-narrative_intelligence (from 0.2-vault-intelligence-loader.md)
**Output:** specimen_decomposition (structural templates, technique catalog, power elements)

---

## Purpose

Decompose top-scoring mechanism narrative specimens from the vault-intelligence into reusable structural templates, technique catalogs, and power elements. Mechanism narrative specimens contain verbatim explanation passages with annotated simplification techniques and visual language cues. This skill extracts the narrative architecture that makes elite mechanism explanations clear, vivid, and credible.

**This skill bridges the gap between:** vault data (annotated explanation passages from 260+ campaigns) and generation layers (what Claude needs to write original mechanism explanations that are as clear and compelling as the best direct response copywriters).

## Decomposition Protocol

### Phase 1: Specimen Selection & Tiering

```
IF tier_metadata present (Phase B loader active):
  LOAD gold_specimens → full decomposition
  LOAD silver_specimens → template-only decomposition
ELSE (pre-upgrade fallback):
  SELECT top 5 by position in explanation_passages as pseudo-Gold
  SELECT next 10 as pseudo-Silver
  LOG "Operating in pre-upgrade mode — tiering estimated from array position"

VERIFY each specimen has:
  - mechanism_name (required)
  - verbatim_explanation (required — skip if absent)
  - simplification_technique (required)
  - visual_language[] (preferred)
  - source_file (required for provenance)
  - niche (required for type-matching)
```

### Phase 2: Structural Template Extraction

For each Gold specimen:

```
ANALYZE verbatim_explanation to extract:

  explanation_skeleton:
    DECOMPOSE the narrative explanation into ordered structural elements:
    - element_type: [problem_setup | mechanism_naming | step_by_step_sequence |
                     analogy_insertion | scientific_grounding | visual_description |
                     before_after_contrast | credibility_insertion |
                     experiential_description | emotional_payoff |
                     differentiation_statement]
    - function: string (what this element accomplishes)
    - proportion: float
    - position: integer

  narrative_architecture:
    simplification_technique: string (from specimen — step_by_step,
                                      analogy_based, contrast_based,
                                      layered_reveal, etc.)
    complexity_level: string (reader-facing — simple, moderate, technical)
    visual_density: integer (count of visual language elements)
    explanation_depth: string (surface, moderate, deep)
    word_count: integer (approximate from verbatim_explanation)

For Silver specimens:
  Extract narrative_architecture ONLY
  SKIP element-level skeleton analysis
```

### Phase 3: Technique Catalog Compilation

Aggregate across ALL decomposed specimens:

```
simplification_techniques:
  FOR each unique simplification_technique:
    - technique_name: string (step_by_step, analogy_based, contrast_based,
                              narrative_embedded, visual_walkthrough, etc.)
    - count: integer
    - implementation_pattern: string (how to write using this technique)
    - gold_example: string (2-3 sentence excerpt showing this technique)
    - best_for: string (mechanism complexity level)
    - specimens_using: [source_file references]

visual_language_patterns:
  FOR each unique visual language approach:
    - pattern_name: string (anatomical_imagery, process_animation,
                            scale_comparison, before_after_visual, etc.)
    - count: integer
    - implementation_pattern: string
    - gold_example: string
    - sensory_channels: [string] (visual, kinesthetic, auditory)

explanation_structures:
  FOR each unique element sequence pattern:
    - structure_name: string
    - element_sequence: [string]
    - frequency: integer
    - simplification_techniques_using: [string]
    - specimens_using: [source_file references]

credibility_weaving:
  FOR each method of inserting credibility into explanations:
    - technique: string
    - position_in_explanation: string
    - implementation_pattern: string
    - gold_example: string
```

### Phase 4: Power Element Extraction

From Gold specimens ONLY:

```
explanation_fragments:
  FOR each Gold specimen:
    - verbatim_fragment: string (strongest 3-5 sentences from verbatim_explanation)
    - fragment_function: string (what this fragment demonstrates)
    - simplification_technique: string
    - mechanism_name: string
    - source_file: string

visual_language_verbatims:
  FOR each Gold specimen with visual_language[]:
    - verbatim_descriptions: [string] (all visual language elements)
    - sensory_type: string (visual, kinesthetic, spatial)
    - source_file: string

analogy_fragments:
  FOR Gold specimens using analogy_based technique:
    - verbatim_analogy: string (the analogy from the explanation)
    - source_domain: string
    - target_domain: string
    - source_file: string

step_sequences:
  FOR Gold specimens using step_by_step technique:
    - verbatim_steps: string (the step sequence from the explanation)
    - step_count: integer
    - source_file: string
```

## Quality Gates

- [ ] Minimum 3 Gold specimens fully decomposed (skeleton + architecture + power elements)
- [ ] Minimum 5 Silver specimens template-decomposed
- [ ] All explanation skeletons have complete element sequences
- [ ] Technique catalog has minimum 3 simplification techniques cataloged
- [ ] Technique catalog has minimum 2 visual language patterns documented
- [ ] Power elements contain minimum 3 explanation fragments from Gold specimens
- [ ] Visual language verbatims extracted from all Gold specimens with visual_language[]
- [ ] Every power element includes source provenance
- [ ] No raw verbatim_explanation passed through undecomposed

---

## Anti-Patterns

| Anti-Pattern | Signal | Fix |
|---|---|---|
| **Raw Passthrough** | Full verbatim_explanation dumped to downstream | Decompose into skeleton + techniques + elements |
| **Technique Without Example** | Simplification techniques named without implementation | Every technique needs implementation_pattern + gold_example |
| **Visual Language Ignored** | visual_language[] data present but not extracted | Visual language is high-value power element — always extract |
| **Analogy Unmapped** | Analogies extracted as text without domain analysis | Map source_domain → target_domain |
| **Explanation Flat** | All explanations treated as same structure | Differentiate by simplification_technique and element sequence |
