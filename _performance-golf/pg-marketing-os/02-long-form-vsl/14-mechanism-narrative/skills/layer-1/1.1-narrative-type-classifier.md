# Skill 14 — Microskill 1.1: Narrative Type Classifier

**Skill:** 11 — Mechanism Narrative
**Microskill:** 1.1 — Narrative Type Classifier
**Version:** 1.0
**Layer:** 1 (Strategic Classification)
**Position:** First microskill in Layer 1
**Type:** Classification & Decision
**Dependencies:** 0.2-tier1-intelligence-loader (narrative_types index, e_level_matrix), 0.4-human-checkpoint-curator (confirmed_state)
**Output:** narrative-type-classification.json

---

## PURPOSE

Classifies the confirmed mechanism into one of 7 narrative types using the E-level decision matrix from TIER1 analysis. This classification determines the fundamental storytelling approach for the entire mechanism narrative — whether the mechanism is presented as a scientific discovery, an accidental finding, a paradigm shift, or another narrative frame. Every downstream microskill depends on this classification to calibrate tone, depth, pacing, and proof integration strategy.

---

## IDENTITY

**What this IS:**
- A strategic classifier that selects the optimal narrative type for the confirmed mechanism
- A decision matrix executor that maps mechanism characteristics to storytelling approach
- The foundation decision that shapes all narrative generation in Layers 2-3

**What this is NOT:**
- Not a writer — produces no prose, only classification decisions
- Not a creative brainstormer — follows the TIER1 decision matrix, not open-ended ideation
- Not reversible after Layer 1 completes — narrative type locks in after this point

## PROCESS

### Step 1: Load the 7 Narrative Types

```
LOAD from tier1_index.narrative_types:

TYPE 1: scientific_revelation
  Definition: Complex biochemistry reduced to graspable metaphor
  E-Level Affinity: E3, E4
  Foundation: Real science — peer-reviewed, institutional backing
  Key Marker: Genuine scientific complexity made to feel like personal discovery
  Best When: Mechanism is a real biological process with research support
  Worst When: Science is thin or fabricated, no institutional backing

TYPE 2: accidental_discovery
  Definition: Specific person stumbles upon mechanism through unexpected event
  E-Level Affinity: E3
  Foundation: Folk wisdom validated by science
  Key Marker: Story-driven — the narrative IS the mechanism reveal
  Best When: There is a genuine discovery story, a real person, a specific moment
  Worst When: No discovery story exists, mechanism is purely engineered

TYPE 3: historical_ancient_secret
  Definition: Ancient knowledge meets modern scientific validation
  E-Level Affinity: E3, E4
  Foundation: Dual credibility — wisdom of ages plus modern proof
  Key Marker: Time-tested with modern confirmation creates double authority
  Best When: Mechanism has historical roots AND modern studies
  Worst When: Historical connection is forced or invented

TYPE 4: paradigm_shift
  Definition: Replaces prospect's entire framework of understanding
  E-Level Affinity: E5
  Foundation: Binary naming essential, new worldview required
  Key Marker: Longest sections — must dismantle old framework before installing new
  Best When: Market is saturated, prospect has tried everything, needs new lens
  Worst When: Simple mechanism that doesn't warrant worldview change

TYPE 5: proprietary_technology
  Definition: Branded engineered system with multiple components
  E-Level Affinity: E4
  Foundation: Innovation emphasis — built, engineered, designed
  Key Marker: Named system with numberable components
  Best When: Mechanism is genuinely engineered or formulated with unique specs
  Worst When: "Proprietary" claim is hollow, mechanism is commodity ingredient

TYPE 6: identity_driven
  Definition: Personal transformation story IS the proof mechanism works
  E-Level Affinity: Any, with charismatic authority present
  Foundation: The person's story carries the mechanism
  Key Marker: Mechanism and guru are inseparable — you believe because of WHO
  Best When: Charismatic guru with dramatic personal transformation story
  Worst When: No guru, no personal story, or guru lacks credibility

TYPE 7: data_driven_debunking
  Definition: Destroy competing mechanism with data first, then present yours
  E-Level Affinity: E2, E3
  Foundation: Market has tried dominant solution and failed
  Key Marker: Must prove what DOESN'T work before showing what does
  Best When: Dominant competitor exists with provable failure data
  Worst When: No clear competitor to debunk, market is not solution-aware
```

### Step 2: Execute Primary Decision Matrix

```
LOAD decision_matrix from tier1_index

PRIMARY CLASSIFICATION (Market Sophistication x E-Level):

  IF market_sophistication == "low" AND e_level == "E2":
    primary_candidates: [scientific_revelation, data_driven_debunking]
    rationale: "Low sophistication market, new application — straightforward
      science or debunking works best"

  IF market_sophistication == "medium" AND e_level == "E3":
    primary_candidates: [accidental_discovery, scientific_revelation, historical_ancient_secret]
    rationale: "Medium sophistication, reframe needed — story-driven or
      recontextualized science"

  IF market_sophistication == "high" AND e_level == "E4":
    primary_candidates: [proprietary_technology, scientific_revelation]
    rationale: "High sophistication, proprietary claim needed — engineered
      differentiation or deep science"

  IF market_sophistication == "saturated" AND e_level == "E5":
    primary_candidates: [paradigm_shift]
    rationale: "Saturated market, paradigm shift is the ONLY option —
      must replace entire framework"

  IF market_sophistication == "medium" AND e_level == "E4":
    primary_candidates: [proprietary_technology, historical_ancient_secret]
    rationale: "Medium market with proprietary mechanism — branded tech
      or validated ancient knowledge"

  IF market_sophistication == "high" AND e_level == "E3":
    primary_candidates: [data_driven_debunking, accidental_discovery]
    rationale: "High sophistication with reframe — debunk dominant approach
      or surprise with discovery story"

  IF market_sophistication == "low" AND e_level == "E3":
    primary_candidates: [scientific_revelation, accidental_discovery]
    rationale: "Low sophistication with reframe — science or story carries
      the new perspective"

  DEFAULT (any combination not explicitly mapped):
    primary_candidates: [scientific_revelation]
    rationale: "Scientific revelation is the most versatile default"
```

### Step 3: Apply Secondary Filters

```
FOR EACH candidate IN primary_candidates:

  FILTER 1 — Mechanism Type Compatibility:
    IF mechanism.type == "biological" → BOOST: scientific_revelation, data_driven_debunking
    IF mechanism.type == "compound" → BOOST: scientific_revelation, historical_ancient_secret
    IF mechanism.type == "technology" → BOOST: proprietary_technology
    IF mechanism.type == "system" → BOOST: proprietary_technology, identity_driven
    IF mechanism.type == "behavioral" → BOOST: identity_driven, paradigm_shift
    IF mechanism.type == "frequency" → BOOST: paradigm_shift, scientific_revelation

  FILTER 2 — Evidence Availability:
    IF available_evidence includes "institutional" AND count >= 3:
      BOOST: scientific_revelation
    IF available_evidence includes "clinical" AND count >= 2:
      BOOST: data_driven_debunking, scientific_revelation
    IF available_evidence includes "testimonial" with guru story:
      BOOST: identity_driven, accidental_discovery
    IF available_evidence includes "historical" references:
      BOOST: historical_ancient_secret

  FILTER 3 — Guru Authority:
    IF guru_authority_present == true:
      BOOST: identity_driven, accidental_discovery, expert_revelation
    IF guru_authority_present == false:
      REDUCE: identity_driven (unless mechanism itself carries the story)

  FILTER 4 — Format Compatibility:
    IF format == "vsl":
      All types compatible (VSL supports any narrative length)
    IF format == "magalog":
      REDUCE: paradigm_shift (may be too long for print format)
    IF format == "advertorial":
      BOOST: scientific_revelation, data_driven_debunking (editorial feel)
    IF format == "email_series":
      BOOST: accidental_discovery (story unfolds across emails)

  SCORE each candidate:
    base_score = position in primary_candidates (first = 10, second = 8, etc.)
    + BOOST count * 2
    - REDUCE count * 3
```

### Step 4: Select Primary and Fallback Types

```
RANK candidates by final score (descending)

SET primary_narrative_type = highest scoring candidate
SET fallback_narrative_type = second highest scoring candidate

IF primary_narrative_type == fallback_narrative_type:
  SET fallback from next candidate in ranked list

VALIDATE selection:
  - primary_narrative_type is in the 7 valid types: PASS/FAIL
  - primary differs from fallback: PASS/FAIL
  - primary is compatible with confirmed e_level: PASS/WARN

LOG classification rationale:
  "Selected [primary] because: [rationale from matrix] + [filter boosts/reduces]"
```

## QUALITY CRITERIA

1. **Single primary type selected** — exactly one of 7 types, no ambiguity
2. **Fallback type differs** — backup option is genuinely different from primary
3. **Matrix logic followed** — classification traces to decision matrix, not guesswork
4. **E-level alignment verified** — selected type is compatible with mechanism's E-level
5. **Type parameters complete** — all 6 parameter fields populated
6. **Downstream signals issued** — every dependent microskill receives its signal
7. **Rationale is specific** — explains WHY this type, not just WHICH type

---

## FAILURE MODES

| Failure | Symptom | Resolution |
|---------|---------|------------|
| No primary candidates from matrix | Unusual sophistication x e_level combination | Use default: scientific_revelation |
| All candidates filtered out | Every type reduced to zero | Re-run with filters relaxed, accept highest-scoring regardless |
| Paradigm shift selected for E2/E3 | E-level mismatch | Override — paradigm_shift requires E5, fall back to next candidate |
| Identity_driven selected without guru | No charismatic authority | Verify — if mechanism itself has strong story, can proceed; otherwise fall back |
| Tie between candidates | Two types score identically | Prefer the type with more TIER1 examples in same niche |
| Confidence below 0.5 | Weak match to any type | Flag for human review, present top 3 candidates with scores |
