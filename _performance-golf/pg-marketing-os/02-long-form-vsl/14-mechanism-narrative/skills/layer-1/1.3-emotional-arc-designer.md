# Skill 14 — Microskill 1.3: Emotional Arc Designer

**Skill:** 11 — Mechanism Narrative
**Microskill:** 1.3 — Emotional Arc Designer
**Version:** 1.0
**Layer:** 1 (Strategic Classification)
**Position:** Third microskill in Layer 1
**Type:** Emotional Architecture Design
**Dependencies:** 0.2-tier1-intelligence-loader (emotional_arcs index), 1.1-narrative-type-classifier (primary_type), 1.2-framing-pattern-selector (emotional_effects), 0.4-human-checkpoint (emotional_state)
**Output:** emotional-arc-design.json

---

## PURPOSE

Designs the emotional progression for the mechanism narrative, critically CONTINUING FROM the emotional state established by Skill 10's root cause narrative. The mechanism narrative does not start emotionally from zero — the reader arrives carrying fear, shame, frustration, curiosity, or empathy from the root cause section. This microskill selects the appropriate arc type, maps emotional targets to each of the 5 explanation phases, and identifies the precise location of the "aha moment" — the peak emotional point where mechanism understanding clicks and belief locks in.

---

## IDENTITY

**What this IS:**
- An emotional architecture designer that creates a phase-by-phase emotional roadmap
- A continuity enforcer that ensures the root cause emotional handoff is honored
- The "aha moment" planner — identifies the peak emotional beat in the mechanism narrative

**What this is NOT:**
- Not a prose writer — produces emotional targets, not copy
- Not a standalone emotional journey — MUST continue from Skill 16's handoff state
- Not a mood board — produces specific, measurable emotional targets per phase
- Not optional — mechanism narrative without designed emotional arc feels flat and educational

### Step 2: Match Handoff State to Arc Type

```
LOAD emotional_state from checkpoint.confirmed_state
LOAD arc_to_handoff_state from tier1_index.cross_references

MATCH emotional_state to accepted handoff states:

  IF emotional_state MATCHES any state in Arc 1 (fear/anxiety/dread/worry):
    primary_arc_candidate: "fear_education_hope"

  IF emotional_state MATCHES any state in Arc 2 (shame/inadequacy/self_blame):
    primary_arc_candidate: "shame_vindication_dignity"

  IF emotional_state MATCHES any state in Arc 3 (frustration/exhaustion/resignation):
    primary_arc_candidate: "frustration_revelation_confidence"

  IF emotional_state MATCHES any state in Arc 4 (curiosity/intrigue/fascination):
    primary_arc_candidate: "curiosity_aha_urgency"

  IF emotional_state MATCHES any state in Arc 5 (empathy/connection/rapport):
    primary_arc_candidate: "empathy_belief_aspiration"

  IF emotional_state matches MULTIPLE arcs (e.g., "fear_mixed_with_curiosity"):
    EVALUATE both candidates
    SELECT the one that better aligns with narrative_type from 1.1
    SET other as secondary_arc_influence

  IF emotional_state matches NO arc:
    DEFAULT to "curiosity_aha_urgency" (most versatile)
    WARN: "Emotional handoff state not recognized — using default arc"
```

### Step 3: Validate Arc-Type Compatibility

```
CHECK: Does selected arc align with narrative type from 1.1?

COMPATIBILITY MATRIX:
  scientific_revelation:
    Strong: fear_education_hope, curiosity_aha_urgency
    Acceptable: frustration_revelation_confidence, empathy_belief_aspiration
    Weak: shame_vindication_dignity (science rarely vindication-focused)

  accidental_discovery:
    Strong: curiosity_aha_urgency, empathy_belief_aspiration
    Acceptable: shame_vindication_dignity
    Weak: frustration_revelation_confidence (discovery isn't about past failure)

  historical_ancient_secret:
    Strong: curiosity_aha_urgency
    Acceptable: fear_education_hope, empathy_belief_aspiration
    Weak: frustration_revelation_confidence

  paradigm_shift:
    Strong: shame_vindication_dignity, curiosity_aha_urgency
    Acceptable: fear_education_hope
    Weak: empathy_belief_aspiration (paradigm shift is conceptual, not personal)

  proprietary_technology:
    Strong: frustration_revelation_confidence
    Acceptable: curiosity_aha_urgency, fear_education_hope
    Weak: shame_vindication_dignity

  identity_driven:
    Strong: empathy_belief_aspiration, shame_vindication_dignity
    Acceptable: curiosity_aha_urgency
    Weak: frustration_revelation_confidence

  data_driven_debunking:
    Strong: frustration_revelation_confidence
    Acceptable: curiosity_aha_urgency, fear_education_hope
    Weak: empathy_belief_aspiration

IF selected arc is "Weak" for narrative type:
  CONSIDER switching to a "Strong" arc
  BUT: emotional continuity from Skill 16 takes PRIORITY over type alignment
  LOG: "Arc-type alignment is weak but emotional continuity requires this arc.
    Downstream writers should compensate with frame blending."
```

### Step 4: Map Emotional Targets to Each Phase

```
USING selected arc, GENERATE phase-by-phase emotional targets:

FOR EACH phase IN [1, 2, 3, 4, 5]:
  SET emotional_target:
    phase_number: integer
    phase_name: string
    entry_emotion: string              # How reader feels entering this phase
    target_emotion: string             # How reader should feel by end of phase
    emotional_shift: string            # The movement that happens during phase
    intensity: enum[low, medium, high, peak]
    techniques: [string]               # Specific emotional techniques to use
    danger_zone: string                # What to avoid emotionally in this phase

EXAMPLE for fear_education_hope:
  Phase 1:
    entry_emotion: "fear (from Skill 16 handoff)"
    target_emotion: "escalated fear — problem feels worse than they thought"
    emotional_shift: "fear intensifies"
    intensity: high
    techniques: ["specificity escalation", "personal relevance markers",
      "time urgency", "physical symptom naming"]
    danger_zone: "Do NOT introduce hope yet — premature hope kills urgency"

  Phase 2:
    entry_emotion: "peak fear"
    target_emotion: "fear + first understanding — 'oh, THAT'S why'"
    emotional_shift: "fear begins to transform into understanding"
    intensity: peak (the emotional transition begins)
    techniques: ["root cause connection", "blame removal", "causal clarity"]
    danger_zone: "Do NOT fully explain mechanism yet — only bridge to it"

  Phase 3:
    entry_emotion: "fear dissolving into curiosity"
    target_emotion: "fascination — mechanism name creates conceptual handle"
    emotional_shift: "fear → fascination (the flip)"
    intensity: high
    techniques: ["dramatic naming", "positioning statement", "contrast with
      what they believed"]
    danger_zone: "Do NOT underwhelm the naming — this is the emotional pivot"

  Phase 4:
    entry_emotion: "fascination + intellectual hunger"
    target_emotion: "comprehension → growing hope"
    emotional_shift: "education creates hope (understanding = control)"
    intensity: medium-to-high (sustained)
    techniques: ["metaphor anchoring", "progressive detail", "relatable
      simplification", "connection to their specific situation"]
    danger_zone: "Do NOT make this feel like a lecture — emotion must
      accompany every fact"

  Phase 5:
    entry_emotion: "hope + intellectual satisfaction"
    target_emotion: "confident hope — mechanism proven, path clear"
    emotional_shift: "hope solidifies into belief"
    intensity: high (building toward product bridge)
    techniques: ["proof stacking", "authority validation", "result preview",
      "possibility expansion"]
    danger_zone: "Do NOT pitch the product — proof builds hope for
      the mechanism, not the product"
```

### Step 5: Identify the Aha Moment

```
BASED ON selected arc type:

  phase: integer                       # Which phase contains the aha
  description: string                  # What the aha IS
  trigger: string                      # What causes it
  emotional_signature: string          # How it FEELS to the reader
  writing_requirements: string         # What the copy must do at this moment

EXAMPLES by arc type:

  fear_education_hope:
    phase: 4
    description: "The moment the mechanism explanation clicks — reader
      UNDERSTANDS what was happening and what can fix it"
    trigger: "Metaphor + explanation aligns with reader's experience"
    emotional_signature: "Relief + excitement — 'THAT'S what's been going on!'"
    writing_requirements: "The simplification must land perfectly.
      One sentence that crystallizes the entire mechanism into
      something the reader can SEE and FEEL."

  shame_vindication_dignity:
    phase: 3
    description: "The moment the mechanism is NAMED and the reader realizes
      their struggle was caused by something outside their control"
    trigger: "Mechanism naming proves it was never their fault"
    emotional_signature: "Vindication surge — 'It wasn't ME. It was THIS.'"
    writing_requirements: "The naming moment must carry maximum emotional
      weight. Not just what the mechanism IS, but what it MEANS for
      the reader's self-perception."

  frustration_revelation_confidence:
    phase: 4
    description: "The moment past failures suddenly MAKE SENSE in light
      of the mechanism explanation"
    trigger: "Explanation shows exactly why previous approaches failed"
    emotional_signature: "Illumination — 'Now I see why nothing worked.'"
    writing_requirements: "Must explicitly connect mechanism explanation
      to past failures. Each past attempt now has a clear reason for
      failing in the mechanism framework."

  curiosity_aha_urgency:
    phase: 3
    description: "The payoff moment — curiosity has been building and
      the mechanism name delivers the answer"
    trigger: "Mechanism naming satisfies the information hunger"
    emotional_signature: "Satisfaction + immediate 'I need this' surge"
    writing_requirements: "Maximum dramatic buildup before the name drop.
      The reveal must feel like a gift after sustained tension."

  empathy_belief_aspiration:
    phase: 5
    description: "The moment proof shows what the mechanism has accomplished

    emotional_signature: "Aspiration — 'I want what they have.'"
    writing_requirements: "Proof must show TRANSFORMATION, not just
```

### Step 6: Design Emotional Continuity Bridge

```
VERIFY continuity from Skill 16 to Skill 11:

  Skill 10 ends with: [emotional_state_at_handoff]
  Skill 11 Phase 1 begins with: [entry_emotion for Phase 1]

  CONTINUITY CHECK:
    IF Skill 10 handoff emotion == Phase 1 entry emotion:
      PASS — seamless transition
    IF Skill 10 handoff emotion is CLOSE to Phase 1 entry emotion:
      BRIDGE NEEDED — 1-2 sentences that shift from handoff to Phase 1 target
    IF Skill 10 handoff emotion is DISTANT from Phase 1 entry emotion:

    from_state: string
    to_state: string
    gap_size: enum[none, small, large]

  Skill 11 Phase 5 ends with: [target_emotion for Phase 5]
  Product bridge needs: "belief + readiness"

  IF Phase 5 target emotion includes belief/hope/confidence:
    PASS — natural bridge to product
  ELSE:
    NOTE: "Product bridge constructor (3.3) must engineer the emotional
    transition from [Phase 5 emotion] to product readiness"
```

## QUALITY CRITERIA

1. **Arc continues from Skill 16** — entry emotion explicitly traces to handoff state
2. **All 5 phases have targets** — no phase left without emotional guidance
3. **Aha moment identified** — specific phase, trigger, and signature defined
4. **No emotional gaps** — each phase's exit emotion matches next phase's entry emotion
5. **Danger zones specified** — each phase warns what to avoid (premature hope, flat education, etc.)
6. **Continuity verified** — bridge to Skill 10 assessed, bridge to Skill 12 assessed
7. **Arc-type compatibility noted** — if arc and narrative type are weakly aligned, it is flagged

---

## FAILURE MODES

| Failure | Symptom | Resolution |
|---------|---------|------------|
| Handoff state not recognized | emotional_state matches no arc | Default to curiosity_aha_urgency, flag for human review |
| Arc contradicts narrative type | Weak alignment per compatibility matrix | Proceed with arc (emotional continuity > type alignment), flag for Layer 2 writers |
| Aha moment falls in wrong phase | Arc says Phase 4 but explanation is too short | Adjust — aha can shift +/- 1 phase based on word budget |
| Emotional gap between phases | Phase 2 exit doesn't connect to Phase 3 entry | Insert transition guidance, note gap for 3.4 polisher |
| Multiple arcs equally valid | Handoff state matches 2+ arcs identically | Select based on narrative type compatibility, use other as secondary influence |
| Phase intensity all "high" | No emotional dynamics | Re-calibrate — at least one phase must be medium to create contrast |
