## PURPOSE

Assembles the complete product-introduction-package.json output file. This is the FINAL microskill in Skill 15. It combines all draft sections, validation results, scoring data, and downstream handoffs into a single output package that feeds Skill 14 (close), body copy assembly, and campaign assembly. The assembler also calculates the overall weighted validation score and determines whether the product introduction meets the minimum quality threshold (>= 7.0) for release.

This microskill does not write or edit copy. It ASSEMBLES, SCORES, and PACKAGES. If validation failures exist from 4.1, 4.2, or 4.3, it blocks assembly and routes for remediation. If all validations pass, it produces the final output.

---

## IDENTITY

**This microskill IS:**
- The final assembly engine that produces product-introduction-package.json
- A validation aggregator that combines results from 4.1, 4.2, and 4.3
- A weighted scoring calculator across 9+ quality dimensions
- A downstream handoff packager for Skill 14 (close) and campaign assembly
- The quality gate that determines pass/fail for the entire skill

**This microskill is NOT:**
- A copywriter or editor — it assembles existing copy
- A validator — that was handled by 4.1, 4.2, 4.3
- A decision-maker about strategy — all decisions were made in Layers 0-1
- A partial assembler — it produces the COMPLETE package or nothing

## PROCESS

### Step 1: Check Validation Prerequisites

1. Read master_principles_report.summary.all_pass
   - If FALSE: BLOCK assembly
   - Log: "Master principles check FAILED. Principles [list] violated."
   - Route to remediation: failing drafts must be revised to pass all 8 principles
   - Do NOT proceed until all 8 pass

2. Read anti_slop_report.verdict.pass
   - If FALSE: BLOCK assembly
   - Log: "Anti-slop validation FAILED. [N] violations detected."
   - Route to remediation: violations must be eliminated
   - Do NOT proceed until zero violations

3. Read vault_comparison_report.composite_pass
   - If FALSE: WARN but do not block
   - Log: "Vault comparison below minimum (score: [X]). Recommend improvement but proceeding."
   - Assembly proceeds with warning flag

4. If BOTH 4.1 and 4.2 pass: proceed to assembly
5. If EITHER 4.1 or 4.2 fail: HALT — route to remediation

### Step 2: Assemble Full Narrative Text

6. Concatenate all draft sections in the correct order:

10. Calculate overall_weighted_average:
    - Sum of (each score * its weight) / sum of weights
    - Minimum passing: 7.0

### Step 4: Determine Overall Pass/Fail

11. Overall pass requires ALL of:
    - master_principles: all 8 pass (from 4.1)
    - anti_slop: zero violations (from 4.2)
    - vault_comparison: composite >= 6.0 (from 4.3) — WARN if below, not block
    - overall_weighted_average >= 7.0

12. Set overall_status:
    - PASS: All conditions met
    - CONDITIONAL_PASS: Weighted average >= 7.0 but vault comparison below 6.0
    - FAIL: Weighted average < 7.0 OR principles failed OR anti-slop failed

## FAILURE MODES

### Failure Mode 1: Assembly Proceeds Despite Validation Failure
- **Symptom:** Output package contains principle violations or slop
- **Cause:** Validation prerequisites not enforced
- **Prevention:** Step 1 BLOCKS assembly if 4.1 or 4.2 fail
- **Recovery:** Route to remediation; re-validate; re-assemble

### Failure Mode 2: Sections Out of Order
- **Symptom:** Price appears before value, or scarcity before guarantee
- **Cause:** Assembly concatenation does not follow commercial_section_sequence
- **Prevention:** Step 2 follows explicit section order
- **Recovery:** Reorder per strategy map sequence

### Failure Mode 3: Downstream Handoff Incomplete
- **Symptom:** Skill 14 cannot execute close because fields are missing
- **Cause:** Handoff builder skips fields
- **Prevention:** Step 5 populates all documented handoff fields
- **Recovery:** Populate missing fields from confirmed data

### Failure Mode 4: Score Calculation Error
- **Symptom:** Weighted average doesn't match dimension scores
- **Cause:** Weight miscalculation or missing dimension
- **Prevention:** Step 3 uses documented weights; Step 4 verifies sum
- **Recovery:** Recalculate with corrected weights
