## PURPOSE

Maps all commercial architecture sub-elements for the product introduction: price reveal architecture, value stacking technique, bonus introduction pattern, guarantee approach, scarcity type, and product naming timing. This is the comprehensive strategic planning microskill for the "commercial section" — everything from bonuses through close setup. Each sub-element is selected from TIER1-proven patterns with specific rationale.

This microskill ensures that the commercial architecture is not a random collection of tactics but a strategically coherent system where each element reinforces the others. The price reveal architecture must work with the value stack, which must work with the bonus pattern, which must work with the guarantee and scarcity.

---

## IDENTITY

**This microskill IS:**
- A comprehensive commercial architecture mapper covering 6 sub-element categories
- A strategic coherence enforcer ensuring all commercial elements work together
- A TIER1-pattern selector that maps each sub-element to proven patterns
- The commercial blueprint that Layer 2 (value stack) and Layer 3 (price, guarantee, scarcity) execute against

**This microskill is NOT:**
- A copywriter for any of these elements (that is Layers 2-3)
- An offer architect (that is Skill 06 — this skill EXECUTES what 06 designed)
- A price setter or guarantee designer — those decisions are confirmed in 0.4
- A creative decision engine — selections are strategically grounded, not aesthetic

## PROCESS

### Step 1: Select Price Reveal Architecture

1. Score each of the 5 price reveal architectures:

**ARCHITECTURE 1: DESCENDING ANCHOR CASCADE**
```
Score inputs:
  + Anchor price provided and significantly higher than actual (+3)
  + Multiple anchor points available (4-6 ideal) (+3)
  + Price gap is dramatic (anchor 5x+ higher than actual) (+2)
  + Campaign format is VSL (room for extended cascade) (+1)
  + Niche expects price negotiation/discounting (+1)
  - No anchor price available (-4)
  - Price gap is small (anchor less than 2x actual) (-2)
  - Campaign is short-form (-2)
Controls: Montezuma ($400→$300→$200→$149.99→$99→$69), BBW ($10K→$5K→$1K→$100→$49→$39), Resurge ($599→$297→$49)
```

**ARCHITECTURE 2: COMPARISON ANCHOR**
```
Score inputs:
  + Competitor pricing data available (+3)
  + Prospect already spending on inferior alternatives (+3)
  + Product is clearly cheaper than alternatives per-unit (+2)
  + Niche has expensive existing solutions (+1)
  - No competitor pricing data (-4)
  - Product is more expensive than alternatives (-3)
  - No clear spending comparison possible (-2)
Controls: Montezuma ("$10-$60/pill for prescriptions"), Gundry ("cost of 16 ingredients individually"), 357 ("$399 public website")
```

**ARCHITECTURE 3: RHETORICAL VALUE QUESTION**
```
Score inputs:
  + Product promises transformational outcomes (+3)
  + Multiple distinct benefits can be questioned (+2)
  + Campaign format allows 6-8 consecutive questions (+2)
  + Emotional state at product section is high desire (+2)
  + Niche has high lifetime value of solving the problem (+1)
  - Benefits are incremental, not transformational (-3)
  - Short-form format (-2)
  - Prospects are price-sensitive, not aspiration-driven (-1)
Controls: Superbrain (8 "What would it be worth..." questions), Montezuma, Resurge ("$10K? $20K? $50K?")
```

**ARCHITECTURE 4: PRICE-PER-DAY MINIMIZER**
```
Score inputs:
  + Product has daily usage (supplements, daily training) (+3)
  + Per-day price is trivially small (<$3) (+3)
  + Coffee/meal comparison available (+1)
  + Product is subscription or multi-month supply (+1)
  - Product is one-time purchase with no daily usage (-3)
  - Per-day price is still high (>$5) (-2)
Controls: Gundry ("$1.27 per day"), Superbrain ("<$3/day"), Montezuma ("<$2/day")
```

**ARCHITECTURE 5: NON-MONETARY JUSTIFICATION**
```
Score inputs:
  + Presenter has personal mission or moral obligation (+3)
  + "Not doing this for money" is credible (presenter is wealthy/successful) (+3)
  + Story includes altruistic motivation (+2)
  + Introduction type is honored_introduction or manufacturing_quest (+1)
  - Presenter has no established credibility for altruism (-4)
  - "Not for money" claim would strain credulity (-3)
  - Campaign is advertorial (third-party, no personal mission) (-2)
Controls: BBW ("Honoring Dr. Summers' final request"), Resurge ("This isn't about money — it's about having lived through the pain")
```

2. Select PRIMARY price architecture (highest score, minimum 6.0)
3. Select SECONDARY price architecture if applicable (many controls layer 2 architectures)
4. Common layering: descending_anchor_cascade + price_per_day_minimizer (cascade to low price, then minimize further with per-day)

### Step 2: Select Value Stacking Technique

5. Score each of the 4 value stacking techniques:

**TECHNIQUE 1: DOLLAR VALUE PARADE**
```
Score: bonus_count >= 3 (+3), each bonus has stated value (+3), total exceeds price 3x+ (+2)
Controls: PSS ($744+ in bonuses), BBW ($397.50), Montezuma ($136.92)
```

**TECHNIQUE 2: CATEGORY DIVERSIFICATION**
```
Score: bonuses span 3+ categories (+3), bonus count >= 4 (+2), diverse format types (+2)
Controls: 357 (6 bonuses, 5 categories: video, protocol, masterclass, app, experience)
```

**TECHNIQUE 3: CONTINUITY TROJAN HORSE**
```
Score: one bonus is free trial → subscription (+4), recurring revenue model desired (+2)
Controls: 357 (App 14-day trial → $29/mo), SSS (Scratch Club → $19/mo), Superbrain (Mindvalley Membership)
```

**TECHNIQUE 4: WORTH ENTIRE COURSE ANCHOR**
```
Score: one bonus has value >= actual price (+3), that bonus is genuinely substantial (+2)
Controls: Superbrain (Overcome Procrastination), PSS ($1,000 in coupons)
```

6. Select PRIMARY technique (may select 2 if compatible — dollar_value_parade + worth_entire_course_anchor work together)

### Step 3: Select Bonus Introduction Pattern

7. Score each of the 4 bonus introduction patterns:

**PATTERN 1: STACKED REVEAL**
```
Best for: 3-6 bonuses, each with clear name and value
Controls: BBW ("Bonus #1... Bonus #2... Bonus #3... Bonus #4...")
```

**PATTERN 2: BUT-WAIT ESCALATION**
```
Best for: Bonuses revealed after pricing, building excitement
Controls: Montezuma (bonuses introduced after price, escalating enthusiasm)
```

**PATTERN 3: PRE-PRICE BONUS STACK**
```
Best for: Maximum value perception before price reveal
Controls: SSS, PSS (full $744+ bonus stack before price)
```

**PATTERN 4: BONUS-FREE AUTHORITY PLAY**
```
Best for: Strong authority or extreme guarantee carries offer alone
Condition: bonus_count == 0 AND (authority is exceptional OR guarantee is extreme)
Controls: Gundry (zero bonuses, Dr. Gundry authority), Resurge (zero traditional bonuses), Australian Dream (zero bonuses, Empty Jar Guarantee)
```

8. If bonus_count == 0: MUST select bonus_free_authority_play
9. If bonus_count >= 1: Select based on scoring
10. Key decision: Bonuses BEFORE or AFTER price? (pre_price vs. but_wait)
    - Pre-price builds maximum value perception → lower price shock
    - Post-price adds unexpected value → increases delight

### Step 4: Map Guarantee Approach

11. Confirm guarantee strength from confirmed_product_package:

| Strength | Characteristics | TIER1 Examples |
|----------|----------------|----------------|
| EXTREME | Triple-back, empty jar, 365-day, money-back PLUS payment | Montezuma (Triple-Back), Australian Dream (Empty Jar), 357 (Greens In Regulation), PSS, SSS |
| STRONG | 90-day, no questions, full refund | Gundry (90-day), BBW (90-Day Ironclad) |
| MODERATE | 60-day money-back | Resurge (60-day), no!no! PRO (60-day) |
| STANDARD | 15-day trial, cancel anytime | Superbrain (15-day) |

12. Check branded name status:
    - If branded_name provided by human or upstream: ACCEPT
    - If branded_name_source is "skill_generated": FLAG for Layer 3 to generate
    - If branded_name is generic (contains "money-back guarantee" without branding): REJECT
      - Must be named: "Triple-Back-Guarantee," "Empty Jar Guarantee," "Greens In Regulation Guarantee," "Flush It Or It's Free"
      - Log: "Guarantee needs branded naming — Principle 6 requirement"

13. Assess guarantee-promise alignment:
    - EXTREME guarantee for bold claims (health transformation, life change)
    - STRONG guarantee for significant claims (improvement, results)
    - MODERATE guarantee for moderate claims
    - If guarantee is WEAKER than promise level: Flag feedback to Skill 06 for guarantee strengthening

### Step 5: Map Scarcity Architecture

14. Confirm scarcity type from confirmed_product_package
15. If scarcity type provided, validate against 5 architectures:

**ARCHITECTURE 1: QUANTITY LIMITED**
```
Justification required: manufacturing capacity, ingredient supply, batch size
Example: Montezuma "first 500 orders" + "strict limit 6 per household"
```

**ARCHITECTURE 2: PRICE INCREASING**
```
Justification required: supplier cost increases, next batch pricing, introductory offer
Example: Resurge "cost of key nutrients going up substantially on next order"
```

**ARCHITECTURE 3: SUPPRESSION/TAKEDOWN**
```
Justification required: powerful forces, industry opposition, controversial mechanism
Example: BBW "I don't know how much longer this site will remain online"
```

**ARCHITECTURE 4: TIME-LIMITED WINDOW**
```
Justification required: first-time customer offer, presentation-specific deal, launch window
Example: Gundry "only available to first-time customers through this video, next 24 hours"
```

**ARCHITECTURE 5: REAL-WORLD SUPPLY CONSTRAINT**
```
Justification required: tariffs, manufacturing disruption, ingredient shortage, geopolitical
Example: 357 "tariffs, production times, and manufacturing availability have all been disrupted"
```

16. CRITICAL CHECK: Does the scarcity have a JUSTIFICATION?
    - If YES: Accept
    - If NO: Flag — "Scarcity lacks justification. Principle 7 requires a REASON."
    - Naked urgency without justification is NEVER acceptable

17. If no scarcity type confirmed: Flag for Layer 3 generation or recommend based on niche:
    - Supplement: quantity_limited or price_increasing
    - Digital: suppression_takedown or time_limited_window
    - Physical: real_world_supply_constraint or quantity_limited
    - Info: time_limited_window or price_increasing

### Step 6: Confirm Product Naming Timing

18. Cross-reference naming_timing from 1.1 with introduction type and campaign format
19. Final naming timing decision:
    - IMMEDIATE (0-10%): Product name already in headline or first section
    - LATE (60-75%): Product name withheld until bridge moment / reveal
20. If LATE: The product name appears for the FIRST TIME at the bridge moment (2.1) or product reveal (2.2)
21. If IMMEDIATE: Product name is already known; reveal is about POSITIONING, not naming

### Step 7: Assess Strategic Coherence

22. Check all 6 sub-elements work together:

**Coherence checks:**
- Price architecture + value stack: If descending cascade, value must be established BEFORE cascade begins
- Bonus pattern + price: Pre-price bonuses must come before price reveal section
- Guarantee + scarcity: Guarantee must neutralize risk BEFORE scarcity creates urgency
  - Correct order: Guarantee → Scarcity (safe first, then urgent)
  - Incorrect order: Scarcity → Guarantee (urgent without safety feels manipulative)
- Naming timing + introduction type: Late naming requires bridge before reveal; immediate naming skips bridge
- Value stack + bonus-free: If no bonuses, value stack relies on product intrinsic value + guarantee

23. If coherence issue detected:
    - Identify conflict
    - Recommend adjustment
    - Log rationale

## FAILURE MODES

### Failure Mode 1: Price Revealed Before Value Established
- **Symptom:** Prospect sees price without context — sticker shock
- **Cause:** Section order not enforced (value_stack must precede price_reveal)
- **Prevention:** Step 7 coherence check enforces Principle 5 (Value before Price)
- **Recovery:** Reorder sections: value stack → price reveal

### Failure Mode 2: Scarcity Without Guarantee
- **Symptom:** Urgency created before safety established — feels manipulative
- **Cause:** Scarcity section placed before guarantee
- **Prevention:** Step 7 enforces Guarantee → Scarcity order
- **Recovery:** Reorder: guarantee → scarcity

### Failure Mode 3: Generic Guarantee Accepted
- **Symptom:** "60-day money-back guarantee" in final copy — violates Principle 6
- **Cause:** Branded name check bypassed
- **Prevention:** Step 4.12 explicit check for generic language with REJECT
- **Recovery:** Flag for Layer 3 branded name generation

### Failure Mode 4: Bonus-Free Without Authority or Guarantee to Compensate
- **Symptom:** No bonuses AND weak authority AND moderate guarantee — offer feels thin
- **Cause:** Bonus-free pattern selected without checking supporting elements
- **Prevention:** Bonus-free requires authority_exceptional OR guarantee_extreme
- **Recovery:** Add bonuses or strengthen guarantee
