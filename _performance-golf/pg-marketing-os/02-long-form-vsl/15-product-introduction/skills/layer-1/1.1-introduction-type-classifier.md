# Microskill 1.1 — Introduction Type Classifier

**Skill:** 15-product-introduction
**Microskill:** 1.1-introduction-type-classifier
**Version:** 1.0
**Layer:** 1 — Strategic Classification
**Position:** First microskill in Layer 1
**Persona:** Sarah Chen — Competitive Intelligence (Primary), Marcus Webster — Pattern Synthesis (Secondary)

---

## PURPOSE

Classifies the product introduction into one or more of 7 introduction types derived from TIER1 control analysis. Most elite controls layer 2-3 types together — this microskill identifies the PRIMARY type and up to 2 SUPPORTING types. The classification drives all downstream creative decisions: which bridge architecture to use, how the product is revealed, and what emotional sequence to follow.

This is a strategic classification, not a creative decision. The classification is based on product format, campaign format, niche, mechanism narrative handoff state, and TIER1 pattern matching — not on aesthetic preference.

---

## IDENTITY

**This microskill IS:**
- A pattern-matching classifier that maps product attributes to introduction types
- A multi-type selector that identifies primary + supporting introduction types
- A TIER1-grounded decision engine — every classification links to proven control examples
- A strategic router that determines how the product introduction section is structured

**This microskill is NOT:**
- A copywriter (that happens in Layer 2)
- A bridge selector (that is 1.2 — this microskill informs bridge selection)
- A creative preference engine — classification is strategic, not aesthetic
- An inventor of new introduction types — it selects from 7 proven patterns

## PROCESS

### Step 1: Inventory Classification Inputs

1. Extract product_format from confirmed_product_package
2. Extract campaign_format from structure_data (default to "vsl" if not available)
3. Extract niche from structure_data (infer from mechanism_context if not available)
4. Extract mechanism narrative handoff state:
   - bridge_text content and style
   - emotional_state at handoff
   - belief_level at handoff
5. Extract product characteristics:
   - Has clinical trial data? (check proof inventory)
   - Has manufacturing/sourcing story? (check mechanism context)
   - Has authority partnership? (check mechanism context)
   - Has competitive elimination upstream? (check bridge_text)
   - Is product name already revealed? (check naming_timing)

### Step 2: Score Each Introduction Type

6. Score each of the 7 types on a 0-10 compatibility scale:

**TYPE 1: MANUFACTURING QUEST**
```
Score inputs:
  + Product is supplement or physical (+3)
  + Manufacturing/sourcing story available in mechanism context (+3)
  + Mechanism involves rare ingredients or complex formulation (+2)
  + Campaign format is VSL (+1)
  + Investment amount or manufacturing difficulty documented (+1)
  - Product is digital (-5)
  - No manufacturing narrative available (-4)
  - Campaign is advertorial (-2)
Best fit: Supplement VSLs with ingredient sourcing stories
Controls: Montezuma's Secret, Resurge, Gundry Total Restore
```

**TYPE 2: FORMULA DEVELOPMENT**
```
Score inputs:
  + Product is supplement (+3)
  + R&D journey or testing narrative available (+3)
  + Breakthrough ingredient/component documented (+2)
  + Clinical trial data available (+1)
  + Authority figure involved in development (+1)
  - Product is physical equipment (-3)
  - No development story available (-4)
  - No testing/validation narrative (-2)
Best fit: Supplements with clinical validation and named discoverer
Controls: Gundry Total Restore, Montezuma's Secret
```

**TYPE 3: HONORED INTRODUCTION**
```
Score inputs:
  + Personal connection to creator established upstream (+3)
  + Belief level at handoff is "locked" (+2)
  + Emotional state at handoff is high positive (+2)
  + Product has strong positioning statement (+1)
  + Campaign format is VSL (+1)
  + Authority figure is the presenter (+1)
  - Advertorial format (third-person) (-3)
  - Low belief level at handoff (-3)
  - No personal authority established (-2)
Best fit: VSLs where presenter is the authority and mechanism belief is locked
Controls: Resurge, Superbrain
```

**TYPE 4: ALTERNATIVE PIVOT**
```
Score inputs:
  + Competitive elimination exists in upstream narrative (+4)
  + Bridge text contains counterselling elements (+3)
  + Multiple alternatives have been dismissed upstream (+2)
  + Product creates new category (+1)
  - No counterselling in upstream narrative (-5)
  - Product is an iteration of existing category (-2)
  - Mechanism narrative ends with positive story, not elimination (-2)
Best fit: Products that create new categories after systematic alternative destruction
Controls: 357 Fairway Hybrid, Australian Dream
```

**TYPE 5: NEWS ANNOUNCEMENT**
```
Score inputs:
  + Campaign format is advertorial (+4)
  + Clinical trial data recently released (+3)
  + Product name revealed early/immediately (+2)
  + Third-person journalistic tone upstream (+2)
  - Campaign is story-driven VSL (-3)
  - Product name withheld late (-3)
  - First-person presenter voice (-2)
Best fit: Advertorials with clinical/news framing
Controls: Uri Varx, no!no! PRO
```

**TYPE 6: MECHANISM-TO-PRODUCT BRIDGE**
```
Score inputs:
  + Mechanism fully explained upstream (+3)
  + Product positioned as delivery vehicle for mechanism (+3)
  + Bridge text frames product as "how you access the mechanism" (+2)
  + Belief level at handoff is "locked" (+1)
  + Any product format (+1)
  - Mechanism not fully explained (-4)
  - Product positioned as independent innovation (-2)
Best fit: Any format where mechanism education was thorough and product = access to mechanism
Controls: Billionaire Brain Wave, Testo Rise
```

**TYPE 7: CHALLENGE INTRODUCTION**
```
Score inputs:
  + Guarantee is extreme (triple-back, money-back PLUS payment) (+3)
  + Product has contrarian or bold positioning (+2)
  + Campaign format is magalog or direct mail (+2)
  + Price is very low relative to claims (+2)
  + Dare/challenge framing fits brand voice (+1)
  - Standard or moderate guarantee (-3)
  - Conservative brand voice (-3)
  - High price point (-2)
Best fit: Bold magalogs or advertorials with extreme guarantees and low prices
Controls: Testo Rise
```

### Step 3: Rank and Select Types

7. Rank all 7 types by compatibility score (highest first)
8. Select PRIMARY type: highest score (must be >= 6.0)
9. Select SUPPORTING types: any additional types scoring >= 5.0 (maximum 2 supporting)
10. If no type scores >= 6.0:
    - Default to mechanism_to_product_bridge (Type 6) as safest universal option
    - Log WARNING: "No strong type match — defaulting to mechanism-to-product bridge"
11. Validate type combination compatibility:
    - Some combinations are natural: manufacturing_quest + honored_introduction
    - Some combinations conflict: news_announcement + manufacturing_quest (different tonal registers)
    - If conflicting combination, drop lower-scoring type

### Step 4: Determine Layering Strategy

12. Based on selected types, determine how they layer together:

**Common Layering Patterns from TIER1:**
```
Pattern A: Manufacturing Quest (PRIMARY) + Honored Introduction (SUPPORTING)
  Example: Resurge — manufacturing story builds to "I'm honored to introduce..."
  Layer: Quest narrative → honored reveal moment

Pattern B: Formula Development (PRIMARY) + Mechanism-to-Product Bridge (SUPPORTING)
  Example: Gundry — R&D journey → "Not everyone has time to read a book" bridge
  Layer: Development story → accessibility bridge → product as mechanism vehicle

Pattern C: Alternative Pivot (PRIMARY) + Partnership Bridge (SUPPORTING)
  Example: 357 — eliminate all alternatives → partnership with engineer
  Layer: Elimination cascade → "But there IS another option" → partnership reveal

Pattern D: News Announcement (PRIMARY) + Challenge Introduction (SUPPORTING)
  Example: Testo Rise — news framing with challenge overlay
  Layer: News credibility → challenge dare

Pattern E: Mechanism-to-Product Bridge (PRIMARY) + Honored Introduction (SUPPORTING)
  Example: BBW/Superbrain — mechanism education → "I'm honored" / "we partnered"
  Layer: Mechanism saturation → honored reveal
```

13. Map selected types to one of these layering patterns (or create custom layering if novel combination)

### Step 5: Determine Product Naming Timing

14. Based on selected introduction type(s), determine naming timing:

| Introduction Type | Default Naming Timing | Rationale |
|-------------------|-----------------------|-----------|
| manufacturing_quest | late (65-75%) | Build desire through quest before naming |
| formula_development | late (65-75%) | Reveal after development story climax |
| honored_introduction | late (60-70%) | Name dropped at "honored" moment |
| alternative_pivot | late (55-65%) | Name revealed after all alternatives eliminated |
| news_announcement | immediate (0-10%) | Name IS the news — lead with it |
| mechanism_to_product_bridge | late (65-75%) | Name after mechanism belief locked |
| challenge_introduction | immediate (0-10%) | Name needed for challenge to work |

15. If naming_timing was human-confirmed in 0.4, use human value (override default)
16. Log selected naming timing with rationale

### Step 6: Generate Classification Output

17. Compile introduction_type_classification:

```yaml
introduction_type_classification:
  primary_type:
    type: string
    score: float
    rationale: string
    tier1_controls: [string]
  supporting_types:
    - type: string
      score: float
      rationale: string
    - type: string  # Optional second supporting
      score: float
      rationale: string
  layering_strategy:
    pattern: string
    sequence: [string]  # Ordered list of how types layer
    description: string
  naming_timing:
    timing: enum[immediate, late]
    estimated_position: string  # e.g., "65-75% through copy"
    source: enum[type_default, human_override]
  classification_confidence:
    score: float  # 0.0-1.0
    rationale: string
  all_scores:
    manufacturing_quest: float
    formula_development: float
    honored_introduction: float
    alternative_pivot: float
    news_announcement: float
    mechanism_to_product_bridge: float
    challenge_introduction: float
```

## QUALITY CRITERIA

| Criterion | Pass Condition |
|-----------|---------------|
| Primary type selected | Exactly 1 primary type with score >= 6.0 (or default with warning) |
| Supporting types valid | 0-2 supporting types, each >= 5.0, non-conflicting with primary |
| Type-format alignment | Selected types are compatible with campaign format |
| Type-product alignment | Selected types are compatible with product format |
| Naming timing set | Timing determined from type or human override |
| TIER1 grounding | Every selected type links to specific control examples |
| Layering strategy defined | If 2+ types, layering order specified |
| Confidence assessed | Classification confidence score with rationale |
| No arbitrary selection | Every score justified by specific input signals |
| Mechanism handoff respected | Classification considers emotional state and belief level at handoff |

## FAILURE MODES

### Failure Mode 1: Single Type Forced When Layering Required
- **Symptom:** Product introduction feels flat — missing the multi-dimensional impact of elite controls
- **Cause:** Only primary type selected, supporting types ignored
- **Prevention:** Explicit supporting type check — most elite controls use 2-3 types
- **Recovery:** Re-score and add supporting types if any score >= 5.0

### Failure Mode 2: Conflicting Types Layered Together
- **Symptom:** Product introduction has jarring tonal shifts
- **Cause:** news_announcement (third-person, journalistic) layered with honored_introduction (first-person, ceremonial)
- **Prevention:** Conflict check in Step 3.11 with defined incompatible pairs
- **Recovery:** Drop lower-scoring conflicting type

### Failure Mode 3: Type Selected Without Format Compatibility
- **Symptom:** Advertorial uses "I'm honored to introduce" (first-person in third-person format)
- **Cause:** Format not considered in scoring
- **Prevention:** Format compatibility is a weighted factor in type scoring
- **Recovery:** Re-score with format penalty applied

### Failure Mode 4: Default Naming Timing Overrides Human Intent
- **Symptom:** Product name withheld when human wants it immediate (or vice versa)
- **Cause:** Human naming_timing preference not checked
- **Prevention:** Step 5.15 explicitly checks for human override
- **Recovery:** Use human value when provided

## EXAMPLES

### Example 1: Supplement VSL — Manufacturing Quest + Honored Introduction (Montezuma Pattern)

```
Inputs:
  product_format: supplement
  campaign_format: vsl
  mechanism_context: "Firming flavonoids / libido-override blend"
  bridge_text: "Now, I've talked a lot about these remarkable ingredients..."
  belief_level: locked
  emotional_state: high desire
  manufacturing story: available ($120K investment, 6-month partner search)
  clinical data: available (39 couples tested)

Scores:
  manufacturing_quest: 9.0 (supplement + manufacturing story + rare ingredients + VSL)
  formula_development: 7.0 (supplement + testing narrative + breakthrough)
  honored_introduction: 6.5 (personal connection + locked belief + high positive)
  alternative_pivot: 2.0 (no counterselling upstream)
  news_announcement: 1.0 (not advertorial, not news frame)
  mechanism_to_product_bridge: 7.0 (mechanism explained + delivery vehicle)
  challenge_introduction: 3.0 (no extreme guarantee framing)

Selection:
  PRIMARY: manufacturing_quest (9.0)
  SUPPORTING: [formula_development (7.0), mechanism_to_product_bridge (7.0)]
  Layering: Quest narrative → breakthrough discovery → product as mechanism delivery
  Naming timing: late (65% — after quest narrative climax)
  Confidence: 0.92
```
