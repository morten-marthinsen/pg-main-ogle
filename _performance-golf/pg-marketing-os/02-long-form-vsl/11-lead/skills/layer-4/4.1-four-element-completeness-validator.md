# Layer 4.1 — Four-Element Completeness Validator

**Version:** 1.0
**Layer:** 4 (Validation & Assembly)
**Type:** Structural Completeness Gate
**Dependencies:** four_element_map (from 1.3), mechanism_elevator_pitch (from 2.2), full draft lead text (from Layer 3 output), teaching_index.four_elements (from 0.3)
**Output:** four_element_validation.json

---

## Purpose

Verify that all four E5 lead elements — New and Different, Simple and Easy, Works Quickly, and Predictable and Reliable — are present in the assembled lead, that each element is effective at communicating its intended signal, and that each element is explicitly connected to the unique mechanism. A lead missing ANY of the four elements is structurally incomplete and cannot make the emotional sale. This is a hard gate — all four must be present or the lead fails.

## Execution Protocol

### Step 1: Locate Each Element in the Lead Text

```
FOR each of the four E5 elements:

  SCAN full_draft_lead.full_text for the mapped text from 1.3

  RECORD:
    - element_name: string
    - located_in_text: boolean
    - exact_text_found: string (the passage that delivers this element)
    - position_in_lead: string (beginning / middle / end)

  IF element text from 1.3 is NOT found in the lead:
    CHECK if the element was rephrased during Layer 2-3 refinement
    SEARCH for semantic equivalent in the lead text
    IF semantic equivalent found → mark located = true, note rephrasing
    IF no equivalent found → mark located = false → FLAG as MISSING
```

### Step 2: Evaluate Each Element — New and Different

```
ASSESS new_and_different:

  PRESENCE CHECK:
    - Does the lead explicitly communicate that the mechanism is NEW?
    - Does the lead communicate WHY or HOW it is DIFFERENT from alternatives?
    - "New and different is better than better" — does it avoid just saying "better"?

  EFFECTIVENESS SCORE (1-10):
    10: Prospect immediately understands this is unlike anything they've tried
    8-9: Clear differentiation communicated, mechanism feels novel
    6-7: Newness mentioned but not vivid or specific
    4-5: Vaguely suggests something different without conviction
    1-3: Generic or absent — could apply to any product

  MECHANISM CONNECTION SCORE (1-10):
    10: The newness IS the mechanism — inseparable
    8-9: Clearly tied to a specific mechanism attribute
    6-7: Connected but the link could be stronger
    4-5: Loosely associated — mechanism mentioned nearby but not causally linked
    1-3: No real connection — generic "new" claim floating without anchor
```

### Step 3: Evaluate Each Element — Simple and Easy

```
ASSESS simple_and_easy:

  PRESENCE CHECK:
    - Does the lead show at least ONE simple aspect of the mechanism?
    - Does it communicate ease without trivializing?
    - Does it follow the principle: "May showcase just one simple step of many"?

  EFFECTIVENESS SCORE (1-10):
    10: Prospect believes they could do this immediately, effortlessly
    8-9: Simplicity clearly communicated for at least one specific aspect
    6-7: Mentions ease but lacks concrete detail (e.g., "easy to use")
    4-5: Implies simplicity without showing it
    1-3: Absent or the mechanism sounds complicated/difficult

  MECHANISM CONNECTION SCORE (1-10):
    10: The simplicity IS a mechanism feature — "just flip this switch"
    8-9: Specific mechanism step shown to be simple
    6-7: Simplicity mentioned in mechanism context but not demonstrated
    4-5: Generic ease claim near mechanism mention
    1-3: No connection — simplicity claim floats independently
```

### Step 4: Evaluate Each Element — Works Quickly

```
ASSESS works_quickly:

  PRESENCE CHECK:
    - Does the lead communicate speed of INITIAL results (not full results)?
    - Does it follow the principle: "Not full results fast — just that it STARTS working fast"?
    - Is there a time-based signal (days, weeks, "right away," "immediately")?

  EFFECTIVENESS SCORE (1-10):
    10: Prospect feels this will start working almost immediately
    8-9: Clear speed signal with credible timeframe
    6-7: Speed mentioned but vague ("fast results")
    4-5: Implied speed without explicit timeframe
    1-3: No speed signal — sounds like a long process

  MECHANISM CONNECTION SCORE (1-10):
    10: The speed IS because of the mechanism — "because it targets X directly"
    8-9: Speed explicitly attributed to how the mechanism functions
    6-7: Speed mentioned alongside mechanism but causal link unclear
    4-5: Speed claim and mechanism coexist without connection
    1-3: Speed claim has no mechanism anchor

  CRITICAL CHECK:
    - Does the speed claim reference INITIAL results or FULL results?
    - IF full results → FLAG as teaching violation (overclaim risk)
    - Correct: "start seeing changes within days"
    - Incorrect: "completely transformed in 3 days"
```

### Step 5: Evaluate Each Element — Predictable and Reliable

```
ASSESS predictable_and_reliable:

  PRESENCE CHECK:
    - Does the lead communicate consistency / safety / reliability?
    - Does it signal "this works every time" or "thousands have used it"?
    - Does it follow the principle: "Brain categorizes safe vs. dangerous — reliability = safety"?

  EFFECTIVENESS SCORE (1-10):
    10: Prospect feels this is a safe, proven path — zero perceived risk
    8-9: Strong reliability signal — social proof, consistency language, safety framing
    6-7: Some reliability mentioned but not viscerally convincing
    4-5: Implied safety without concrete signal
    1-3: Absent — prospect has no reason to believe this is reliable

  MECHANISM CONNECTION SCORE (1-10):
    10: Reliability IS a mechanism property — "because it works with your body's natural process"
    8-9: Reliability explicitly linked to mechanism design
    6-7: Reliability mentioned in mechanism context
    4-5: Generic reliability claim near mechanism
    1-3: No connection between reliability and mechanism
```

### Step 6: Calculate Overall Scores

```
FOR each element:
  element_score = (effectiveness × 0.6) + (mechanism_connection × 0.4)

overall_score = WEIGHTED AVERAGE of all four element_scores:
  new_and_different: weight 0.30 (most important — "new and different > better")
  simple_and_easy: weight 0.20
  works_quickly: weight 0.25
  predictable_and_reliable: weight 0.25

all_four_present = (new_and_different.present AND simple_and_easy.present
                    AND works_quickly.present AND predictable_and_reliable.present)

CRITICAL GATE:
  IF all_four_present = false → FAIL (regardless of scores)
  IF all_four_present = true AND overall_score >= 7.0 → PASS
  IF all_four_present = true AND overall_score >= 5.0 AND < 7.0 → PASS_WITH_WARNINGS
  IF all_four_present = true AND overall_score < 5.0 → FAIL (elements present but ineffective)
```

### Step 7: Generate Remediation Notes (If Not PASS)

```
IF gate_result ≠ PASS:

  FOR each element scoring below 7.0:
    GENERATE specific remediation:
      - What is weak about this element
      - What the teaching framework says it should accomplish
      - Suggested revision direction (without writing the copy)
      - Which upstream microskill should re-execute (1.3, 2.2, or 3.x)

  FOR each missing element:
    GENERATE insertion guidance:
      - Where in the lead this element should appear
      - What mechanism asset should anchor it
      - Reference to teaching_index.four_elements for the principle
```

## Quality Gates

- [ ] All four E5 elements scanned for in full draft lead text
- [ ] Each element scored on both effectiveness (1-10) and mechanism_connection (1-10)
- [ ] `all_four_present` correctly computed — any single missing element = false
- [ ] `works_quickly` checked for initial vs. full results overclaim
- [ ] Overall score calculated with correct weighting (0.30 / 0.20 / 0.25 / 0.25)
- [ ] FAIL triggered if any element is missing regardless of other scores
- [ ] Remediation notes generated for every element scoring below 7.0
- [ ] No element rated "present" when it is merely implied or generic

---

## Anti-Patterns

- **Checkbox syndrome:** Marking an element "present" because a vaguely related word exists — "new" appearing in "new way to..." does NOT mean the New and Different element is effectively communicated
- **Generous scoring:** Rating a weak element 7+ because it technically exists — if a prospect would not FEEL the element, it is not effective
- **Disconnected elements:** Scoring mechanism_connection high when the element floats independently near the mechanism mention but is not causally linked
- **Full results speed claim:** Accepting "you'll be completely transformed in days" as Works Quickly — the teaching says INITIAL results, not full transformation
- **Safety through vagueness:** Accepting "it's totally safe" as Predictable and Reliable — reliability needs concrete signal (social proof, consistency evidence, mechanism-based safety)
