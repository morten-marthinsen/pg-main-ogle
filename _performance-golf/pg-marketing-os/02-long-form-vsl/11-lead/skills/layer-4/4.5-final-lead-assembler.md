# Layer 4.5 — Final Lead Assembler

**Version:** 1.0
**Layer:** 4 (Validation & Assembly)
**Type:** Package Assembly & Output Compiler
**Dependencies:** four_element_validation (from 4.1), emotional_sale_audit (from 4.2), anti_slop_validation (from 4.3), vault_comparison (from 4.4), lead_classification (from 1.1), hook (from 1.2), four_element_map (from 1.3), emotional_arc (from 1.4), problem_callout (from 2.1), mechanism_elevator_pitch (from 2.2), open_loops (from 2.3), credibility_insertions (from 2.4), clemens_calibration (from 3.1), georgi_compliance (from 3.2), conversational_flow (from 3.3), attention_lock (from 3.4), full draft lead text (from Layer 3 output), lead_context (from 0.1)
**Output:** lead-package.json

---

## Purpose

Assemble the complete `lead-package.json` from all upstream layer outputs. Verify that all four Layer 4 validators passed before assembly proceeds. Compile the full lead text, calculate the overall weighted average score, populate all metadata fields, and construct the downstream handoff objects that feed body copy, campaign argument, and credibility section skills. This is the terminal microskill of the 11-lead skill — its output IS the skill's output.

## Execution Protocol

### Step 1: Verify All Layer 4 Validators Passed

```
GATE CHECK — HARD BLOCK:

  CHECK 4.1 — Four-Element Completeness Validator:
    REQUIRED: gate_result = "PASS" OR "PASS_WITH_WARNINGS"
    REQUIRED: all_four_present = true
    IF FAIL → BLOCK ASSEMBLY
    LOG: "Assembly blocked: Four-element validation failed — [failure reason]"

  CHECK 4.2 — Emotional Sale Auditor:
    REQUIRED: gate_result = "PASS" OR "PASS_WITH_WARNINGS"
    REQUIRED: this_sounds_amazing_test = "PASS"
    REQUIRED: no_premature_proof.clean = true
    IF FAIL → BLOCK ASSEMBLY
    LOG: "Assembly blocked: Emotional sale audit failed — [failure reason]"

  CHECK 4.3 — Anti-Slop Validator:
    REQUIRED: status = "PASS" OR "PASS_WITH_WARNINGS"
    REQUIRED: all automatic_fail_triggers = false
    IF FAIL → BLOCK ASSEMBLY
    LOG: "Assembly blocked: Anti-slop validation failed — [failure reason]"

  CHECK 4.4 — Vault Pattern Comparator:
    REQUIRED: gate_result = "PASS" OR "PASS_WITH_WARNINGS"
    REQUIRED: mimicry highest_risk ≠ "CRITICAL"
    IF FAIL → BLOCK ASSEMBLY
    LOG: "Assembly blocked: Vault comparison failed — [failure reason]"

  IF ANY validator FAILED:
    HALT assembly
    RETURN blocker_report with:
      - which validators failed
      - specific failure reasons
      - remediation routes (which upstream skills to re-execute)
    DO NOT proceed to Step 2

  IF ALL validators PASSED or PASSED_WITH_WARNINGS:
    PROCEED to Step 2
    LOG warnings for any PASS_WITH_WARNINGS results
```

### Step 11: Calculate Validation Scores and Overall Weighted Average

```
COMPILE all validation scores:

  validation_scores:
    four_element_completeness: FROM four_element_validation.aggregate.overall_score (round to integer 1-10)
    emotional_sale_achievement: FROM emotional_sale_audit.overall_achievement (round to integer 1-10)
    hook_strength: FROM hook.attention_score
    open_loop_quality: [DERIVE from open_loops quality — average of loop effectiveness if scored, or from vault_comparison.loop_comparison]
    conversational_flow: FROM conversational_flow.flow_score
    georgi_compliance: FROM georgi_compliance.compliance_status
    clemens_calibration: FROM clemens_calibration.calibration_status
    anti_slop: FROM anti_slop_validation.status (map to pass/fail)
    anti_slop_violations: FROM anti_slop_validation.total_violations
    vault_comparison: FROM vault_comparison.overall_vault_score (round to integer 1-10)
    vault_differentiation_notes: FROM vault_comparison.differentiation_notes

  CALCULATE overall_weighted_average:

    numeric_scores = {
      four_element_completeness: validation_scores.four_element_completeness,
      emotional_sale_achievement: validation_scores.emotional_sale_achievement,
      hook_strength: validation_scores.hook_strength,
      open_loop_quality: validation_scores.open_loop_quality,
      conversational_flow: validation_scores.conversational_flow,
      vault_comparison: validation_scores.vault_comparison
    }

    # Georgi and Clemens are bonus modifiers, not scored 1-10
    georgi_bonus:
      IF georgi_compliance = "pass" → 10 (full bonus)
      IF georgi_compliance = "fail" → 0 (no bonus — should not reach here)

    clemens_bonus:
      IF clemens_calibration = "pass" → 10 (full bonus)
      IF clemens_calibration = "fail" → 0 (no bonus — should not reach here)

    overall_weighted_average = (
      (four_element_completeness × 0.20) +
      (emotional_sale_achievement × 0.20) +
      (hook_strength × 0.15) +
      (open_loop_quality × 0.10) +
      (conversational_flow × 0.15) +
      (vault_comparison × 0.05) +
      (georgi_bonus × 0.10) +
      (clemens_bonus × 0.05)
    )

    ROUND to 2 decimal places

  VALIDATE:
    - overall_weighted_average >= 7.0 → PASS
    - overall_weighted_average >= 5.0 AND < 7.0 → PASS_WITH_WARNINGS
    - overall_weighted_average < 5.0 → FAIL (should not reach assembly if validators passed)
```

### Step 12: Populate Downstream Handoffs

```
COMPILE handoff objects for downstream skills:

  downstream_handoffs:

    for_body_copy:
      open_loops_to_close:
        [FOR each open loop]:
          - loop_id: string
            loop_text: string
            loop_type: string
            suggested_closure_location: string  # FROM open_loops.closure_location
      emotional_state_at_handoff: FROM emotional_arc.ending_state
      mechanism_teases_to_expand:
        [EXTRACT mechanism references from lead that need expansion in body]:
          - string (each mechanism tease that body copy should develop)
      discovery_story_teased: boolean  # FROM georgi_compliance.do_checklist.tease_discovery_story

    for_campaign_argument:
      thesis_previewed: boolean
        # true if the lead hints at or sets up the campaign thesis
        # DERIVE from: does lead_context.structure.campaign_thesis appear (in preview form) in the lead?
      first_cpb_chunk_setup: string
        # What the lead has set up for the first claim-proof-benefit chunk
        # DERIVE from: what specific claim or belief does the lead create that the first CPB chunk should address?
      belief_gaps_surfaced:
        [IDENTIFY beliefs the prospect holds at end of lead that the argument must address]:
          - string (each belief gap — e.g., "prospect believes it works but needs proof it works for THEM")

    for_credibility_section:
      credibility_teased:
        [FOR each credibility insertion in the lead]:
          - string (what was teased — e.g., "Harvard physician credentials mentioned briefly")
      authority_to_expand: string
        # The primary authority/credibility element that was briefly inserted in the lead
        # and should be EXPANDED in the credibility section

  VALIDATE:
    - open_loops_to_close array is not empty (minimum 2 loops)
    - emotional_state_at_handoff is populated
    - for_campaign_argument has at least one belief_gap
    - for_credibility_section has at least one credibility_teased item
```

## Anti-Patterns

- **Premature assembly:** Assembling the package before all four validators have returned results — the assembler WAITS for all validators, it does not proceed optimistically
- **Silent field omission:** Skipping a field because the upstream data was not cleanly formatted — every field in the schema must be populated or the assembler must halt and report the gap
- **Score laundering:** Adjusting upstream scores during assembly to achieve a passing overall average — the assembler TRANSMITS scores, it does not modify them
- **Downstream assumption:** Leaving downstream handoff fields vague ("body copy will figure it out") — every handoff must contain specific, actionable data
- **Ghost assembly:** Outputting a lead-package.json where full_lead_text is empty or a placeholder — the full_lead_text IS the deliverable; everything else is metadata about it
- **Weight miscalculation:** Using equal weights instead of the specified formula — the weighted average must use the exact weights: 0.20 / 0.20 / 0.15 / 0.10 / 0.15 / 0.05 / 0.10 / 0.05
- **Validator bypass:** Proceeding with assembly when a validator returned FAIL because other validators passed — ALL four validators must pass for assembly to proceed
