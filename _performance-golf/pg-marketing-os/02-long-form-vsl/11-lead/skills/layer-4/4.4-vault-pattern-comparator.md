# Layer 4.4 — Vault Pattern Comparator

**Version:** 1.0
**Layer:** 4 (Validation & Assembly)
**Type:** Competitive Benchmarking & Differentiation Gate
**Dependencies:** full draft lead text (from Layer 3 output), vault_lead_intelligence (from 0.2), lead_classification (from 1.1), hook (from 1.2), emotional_arc (from 1.4), open_loops (from 2.3), four_element_map (from 1.3)
**Output:** vault_comparison.json

---

## Purpose

Compare the assembled lead against elite lead patterns from TIER1 extractions vault. Identify structural similarities to proven controls, validate the lead represents informed original work (not accidental mimicry of a single source), and benchmark quality dimensions against the best leads in the vault. The lead should be INFORMED by vault patterns but DIFFERENTIATED from them. A lead that is structurally identical to a single vault exemplar is derivative. A lead that shares no patterns with any vault exemplar is uninformed. The goal is the sweet spot: pattern-informed originality.

---

## Primary Persona

**Sarah Chen — Competitive Intelligence Analyst**
Maps the lead's structural DNA against the competitive landscape of elite controls. Identifies which patterns were adopted, which were adapted, and which are genuinely novel. Ensures the lead would hold up as original work in a market where these controls already exist.

## Secondary Persona

**Marcus Webster — Pattern Synthesis Analyst**
Provides systematic pattern comparison methodology. Ensures the comparison is structured and quantitative, not impressionistic. Identifies the specific dimensions where the lead excels or falls short relative to vault benchmarks.

## Execution Protocol

### Step 1: Select Comparison Set

```
FROM vault_lead_intelligence, select 3-5 most relevant vault leads for comparison.

SELECTION CRITERIA (ranked by priority):

  1. SAME LEAD TYPE: Select vault leads that share the classified lead type (from 1.1)
     - If classified as "problem_solution" → prioritize vault problem_solution leads
     - If classified as "hybrid" → select components that match hybrid elements

  2. SAME NICHE: If vault leads exist in the same or adjacent niche, prioritize those
     - Same niche = direct competitor comparison
     - Adjacent niche = structural pattern comparison

  3. HIGHEST QUALITY: Among matching leads, prioritize those with:
     - quality_score >= 85
     - extraction_confidence >= 85

  4. DIVERSITY: Ensure at least ONE comparison lead is a DIFFERENT type
     - This validates the choice to use the selected type over alternatives

RECORD comparison_set:
  - vault_lead_id: string
  - vault_lead_name: string
  - lead_type: string
  - niche: string
  - quality_score: float
  - selection_reason: string
```

### Step 2: Compare Lead Type Alignment

```
FOR each lead in comparison_set:

  COMPARE lead type structure:
    - Does our lead follow the same structural sequence?
    - Where does our lead diverge from the vault pattern?
    - Is the divergence intentional (adaptation) or accidental (drift)?

  SCORE type_alignment (1-10):
    10: Expertly adapted the type pattern — follows the blueprint with fresh execution
    8-9: Strong alignment with deliberate variations
    6-7: Generally aligned but some structural elements out of sequence
    4-5: Loosely follows the type — more deviation than alignment
    2-3: Minimal alignment — barely recognizable as the classified type
    1: No alignment — lead type classification may be incorrect

  RECORD:
    - vault_lead: string
    - structural_sequence_match: string (how our sequence maps to theirs)
    - divergence_points: array (where we differ and why)
    - type_alignment_score: integer[1-10]
```

### Step 3: Compare Hook Effectiveness

```
COMPARE our hook against vault hooks in the hook_catalog:

  DIMENSIONS:
    - Word count: How does our hook length compare? (vault range typically 5-15 words)
    - Device type: Are we using the same or different opening device?
    - Attention mechanism: Same attention trigger (curiosity, shock, identification, fear)?
    - Specificity: Is our hook more or less specific than vault hooks?
    - Originality: Does our hook read as original or as a variant of a vault hook?

  SCORE hook_comparison (1-10):
    10: Our hook matches or exceeds the best vault hooks in attention-grabbing power
    8-9: Strong hook that holds its own against vault benchmarks
    6-7: Competent hook but vault hooks are clearly stronger
    4-5: Hook is below vault benchmark — needs strengthening
    2-3: Hook is significantly weaker than vault patterns
    1: Hook would not compete in a market where vault controls exist

  RECORD:
    - closest_vault_hook: string (which vault hook is most similar)
    - our_hook_advantages: string
    - our_hook_disadvantages: string
    - hook_comparison_score: integer[1-10]
```

### Step 4: Compare Emotional Arc

```
COMPARE our emotional arc against vault emotional_progressions:

  DIMENSIONS:
    - Starting emotional state match
    - Ending emotional state match (vault leads that succeed end at emotional buy-in)
    - Arc shape comparison (ascending, spike, slow build, etc.)
    - Number of emotional stages
    - Peak moment placement (early, middle, late in the lead)

  IDENTIFY the vault leads with the most similar emotional arcs.
  IDENTIFY the vault leads with the most DIFFERENT emotional arcs.
  NOTE: Different is not bad — it may indicate a novel approach.

  SCORE arc_comparison (1-10):
    10: Our emotional arc is as compelling as the best vault arcs
    8-9: Strong emotional journey — comparable to vault benchmarks
    6-7: Functional arc but vault leads create more emotional intensity
    4-5: Arc is flat compared to vault patterns
    2-3: Arc is significantly weaker — prospect may disengage
    1: No discernible emotional arc

  RECORD:
    - most_similar_vault_arc: string
    - arc_advantages: string
    - arc_disadvantages: string
    - arc_comparison_score: integer[1-10]
```

### Step 5: Compare Word Count and Pacing

```
COMPARE our lead's word count against vault leads:

  COLLECT vault word counts for comparison set
  CALCULATE: vault average, vault range, our lead's position

  ASSESS pacing:
    - Does our lead arrive at the emotional sale faster or slower than vault leads?
    - Does our lead bury the mechanism tease? (vault leads typically tease early)
    - Does our lead have dead space where vault leads maintain momentum?

  SCORE pacing_comparison (1-10):
    10: Our pacing matches or exceeds vault efficiency — no wasted words
    8-9: Strong pacing — competitive with vault leads
    6-7: Adequate pacing but vault leads are tighter
    4-5: Some bloat compared to vault benchmarks
    2-3: Significantly longer or slower than vault patterns
    1: Pacing problems would cause prospect dropout

  RECORD:
    - our_word_count: integer
    - vault_average_word_count: integer
    - vault_word_count_range: string
    - pacing_comparison_score: integer[1-10]
    - pacing_notes: string
```

### Step 6: Compare Four-Element Delivery Method

```
COMPARE how our lead delivers the four E5 elements vs. vault patterns:

  FOR each E5 element (New/Different, Simple/Easy, Quick, Reliable):
    - HOW does our lead deliver this element?
    - HOW do vault leads deliver this element?
    - Is our delivery method common (many vault leads do it this way)?
    - Is our delivery method novel (no vault lead does it this way)?
    - Is our delivery EFFECTIVE compared to vault patterns?

  SCORE element_delivery_comparison (1-10):
    10: Our four-element delivery is as strong or stronger than vault patterns
    8-9: Effective delivery that competes well with vault approaches
    6-7: Adequate but vault leads deliver these elements more compellingly
    4-5: Delivery is weaker than vault benchmarks on multiple elements
    2-3: Vault leads significantly outperform our element delivery
    1: Four-element delivery does not compete

  RECORD:
    - element_delivery_notes: object (per-element comparison)
    - element_delivery_comparison_score: integer[1-10]
```

### Step 7: Compare Open Loop Placement and Type

```
COMPARE our open loops against vault transition patterns:

  DIMENSIONS:
    - Number of loops: More or fewer than vault leads?
    - Loop types: Same or different types?
    - Placement: Are loops in similar positions within the lead?
    - Strength: Do our loops create comparable pull-forward tension?

  SCORE loop_comparison (1-10):
    10: Our open loops create equal or greater forward momentum than vault leads
    8-9: Strong loops — competitive with vault patterns
    6-7: Adequate loops but vault leads create more suspense
    4-5: Loops are weaker than vault benchmarks
    2-3: Vault leads significantly outperform our loop engineering
    1: Open loops are ineffective compared to vault standards

  RECORD:
    - our_loop_count: integer
    - vault_average_loop_count: integer
    - loop_type_comparison: string
    - loop_comparison_score: integer[1-10]
```

### Step 8: Assess Mimicry Risk

```
CRITICAL CHECK: Is our lead too similar to any single vault lead?

MIMICRY INDICATORS:
  - Same lead type + same hook device + same emotional arc + same niche = HIGH mimicry risk
  - More than 3 structural elements identical to a single vault lead = FLAG
  - Language patterns that echo a specific vault lead's phrasing = FLAG
  - Identical opening device with similar subject matter = FLAG

MIMICRY RISK ASSESSMENT:
  LOW: Lead is clearly informed by vault patterns but original in execution
  MODERATE: Lead shares significant structural DNA with one vault lead but
            has enough differentiation to stand alone
  HIGH: Lead could be mistaken for a variant of a specific vault lead
  CRITICAL: Lead is essentially a template fill of a vault lead — must revise

FOR each vault lead in comparison set:
  - structural_similarity_percentage: float (0-100%)
  - language_similarity: enum[none, low, moderate, high]
  - overall_mimicry_risk: enum[LOW, MODERATE, HIGH, CRITICAL]

GATE CHECK:
  IF any single vault lead has mimicry_risk = CRITICAL → FAIL
  IF any single vault lead has mimicry_risk = HIGH → WARN
```

### Step 9: Compile Differentiation Notes

```
DOCUMENT what makes our lead DIFFERENT from vault patterns:

  STRUCTURAL DIFFERENTIATORS:
    - What structural choices are unique to our lead?
    - What sequence variations distinguish our approach?
    - What elements are present in our lead but absent from vault leads?

  CREATIVE DIFFERENTIATORS:
    - What language or imagery is original?
    - What hook approach is novel?
    - What emotional angle hasn't been used by vault leads?

  MECHANISM-DRIVEN DIFFERENTIATORS:
    - How does our specific mechanism create unique lead opportunities?
    - What about our mechanism allows a different lead approach?

  NICHE-ADAPTATION DIFFERENTIATORS:
    - How has the lead been adapted for this specific niche?
    - What niche-specific elements are present?

RECORD: differentiation_notes: string (comprehensive summary)
```

### Step 10: Calculate Overall Vault Comparison Score

```
overall_vault_score = WEIGHTED AVERAGE:
  type_alignment × 0.15
  hook_comparison × 0.25
  arc_comparison × 0.20
  pacing_comparison × 0.10
  element_delivery_comparison × 0.15
  loop_comparison × 0.15

GATE DECISION:
  IF mimicry_risk = CRITICAL for any vault lead → FAIL
  IF overall_vault_score >= 7.0 AND mimicry_risk <= MODERATE → PASS
  IF overall_vault_score >= 5.0 AND < 7.0 → PASS_WITH_WARNINGS
  IF overall_vault_score < 5.0 → FAIL (lead is below vault benchmark)

NOTE: This is a BENCHMARKING gate, not a pass/fail binary.
A score of 6.0 with good differentiation notes is more valuable
than a score of 9.0 with CRITICAL mimicry risk.
```

## Quality Gates

- [ ] 3-5 vault leads selected for comparison with documented selection rationale
- [ ] At least 1 comparison lead is the SAME type; at least 1 is a DIFFERENT type
- [ ] All 6 comparison dimensions scored independently (1-10)
- [ ] Mimicry risk assessed against EVERY lead in the comparison set
- [ ] CRITICAL mimicry risk triggers automatic FAIL
- [ ] Differentiation notes document structural, creative, mechanism-driven, and niche-adaptation differentiators
- [ ] Overall vault score calculated with correct weighting (0.15 / 0.25 / 0.20 / 0.10 / 0.15 / 0.15)
- [ ] Hook comparison includes word count, device type, and originality assessment
- [ ] Pacing comparison includes word count benchmarking against vault averages
- [ ] Gate result distinguishes between quality failure (below benchmark) and originality failure (mimicry)

---

## Anti-Patterns

- **Vault worship:** Penalizing a lead for being different from vault patterns when the difference is an intentional, well-reasoned adaptation — the vault is a benchmark, not a template
- **Mimicry blindness:** Failing to flag a lead that is structurally identical to a single vault exemplar just because the language is different — structural mimicry is still mimicry
- **Apples-to-oranges scoring:** Comparing a health supplement lead against a financial newsletter lead on hook effectiveness without accounting for niche conventions — comparisons must be contextualized
- **Score inflation through selection:** Choosing only vault leads that are weaker than our lead to inflate comparison scores — the comparison set must include the BEST relevant vault leads
- **Differentiation for its own sake:** Praising novelty that makes the lead worse — being different from vault patterns is only valuable if the difference serves the emotional sale
- **Ignoring niche context:** Scoring pacing against vault leads from a different niche where word count conventions differ (financial leads typically run longer than supplement leads)
