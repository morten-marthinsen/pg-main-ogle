# Layer 0.4 — Input Validator

## Purpose

Validate that all loaded upstream packages, vault intelligence, and teaching indexes meet minimum thresholds for lead engineering. Determine lead strategy viability and flag blockers before Layer 1 begins.

---

## Input Requirements

```yaml
inputs_to_validate:
  lead_context: object  # from 0.1
  vault_lead_intelligence: object  # from 0.2
  teaching_index: object  # from 0.3
```

---

## Execution Protocol

### Step 1: Validate Upstream Package Completeness

Required fields check:
- Mechanism: name, uniqueness_claim present
- Structure: campaign_thesis, lead_handoff_data present
- Promise: primary_statement present
- Root cause: three_part_structure present
- Headline: main_headline present

### Step 2: Validate Lead-Specific Prerequisites

- Mechanism has a **name** (not just a description) — the lead must NAME the mechanism
- Promise is **elevator-pitch compatible** (can be stated in 1-2 sentences)
- Root cause has counter-intuitive element (what_they_think ≠ what_it_really_is)
- Headline establishes a promise that the lead can continue

### Step 3: Validate Vault Intelligence

- At least 3 lead patterns loaded
- At least 2 lead types represented
- Hook catalog populated

### Step 4: Validate Teaching Index

- E5 four elements indexed
- Georgi DO/DON'T lists complete
- Clemens principle documented
- Open loop types cataloged

### Step 5: Compile Validation Report

---

## Output

```yaml
validation_report:
  overall_status: enum[pass, pass_with_warnings, fail]
  upstream_completeness:
    mechanism_complete: boolean
    structure_complete: boolean
    promise_complete: boolean
    root_cause_complete: boolean
    headline_present: boolean
    missing_fields: [object]
  lead_prerequisites:
    mechanism_named: boolean
    promise_pitch_compatible: boolean
    counter_intuitive_element: boolean
    headline_momentum: boolean
  vault_intelligence:
    status: string
    patterns_loaded: integer
  teaching_index:
    status: string
  warnings: [string]
  blockers: [string]
  gate_0_recommendation: enum[proceed, proceed_with_caution, halt]
```

---

## Quality Gates

**PASS if:**
- [ ] All required upstream fields present
- [ ] Mechanism has a specific name
- [ ] Promise is pitch-compatible
- [ ] Headline present
- [ ] Teaching index complete

**FAIL if:**
- [ ] Mechanism package missing or unnamed
- [ ] Headline absent
- [ ] Teaching index incomplete

---

## Handoff

Pass `validation_report` to:
