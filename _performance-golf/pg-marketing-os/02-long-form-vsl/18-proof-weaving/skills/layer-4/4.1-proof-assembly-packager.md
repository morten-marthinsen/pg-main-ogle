# Microskill 4.1: Proof Assembly Packager

**Layer:** 4 (Validation)
**Purpose:** Tag and package all proof blocks for downstream assembly (Skill 16)

---

## CORE PRINCIPLE

Proof blocks are not just drafted copy — they are ASSEMBLY-READY UNITS with metadata that tells Campaign Assembly exactly where they go, how they connect, and what they prove.

---

## PACKAGING STRUCTURE

### Each Proof Block Package

```yaml
proof_block:
  id: "PROOF_[section]_[number]"  # e.g., PROOF_LEAD_001

  content:
    full_text: "[complete drafted proof copy]"
    word_count: [number]

  classification:
    proof_type: [testimonial | before_after | study_citation | demonstration | authority | scale | data | logic]
    format: [specific format within type]
    emotional_register: [clinical | emotional | urgent | conversational | dramatic]

  placement:
    target_section: [section name]
    position_in_section: [beginning | middle | end | specific_after_X]
    placement_rationale: "[why here]"

  transitions:
    transition_in: "[copy for entering this block]"
    transition_out: "[copy for exiting this block]"
    requires_transition_in: true | false
    requires_transition_out: true | false

  connections:
    supports_claim: "[specific claim this proves]"
    threads_to: [list of related blocks/sections]
    callbacks_from: [if this is a callback, what it references]
    callbacks_to: [if something later references this]

  assembly_instructions:
    can_stand_alone: true | false
    requires_preceding: "[block_id or section]"
    required_by_following: "[block_id or section]"
    optional_variations: [list if any alternate placements]
```

## PACKAGING BY SECTION

### Lead Section Proof Package

```yaml
lead_proof_package:
  blocks:
    - id: PROOF_LEAD_001
      type: [type]
      placement: [specific location]
      # ... full block structure
    - id: PROOF_LEAD_002
      # ...

  total_blocks: [count]
  total_word_count: [sum]

  assembly_notes:
    sequence_required: true | false
    flexibility: "[how much rearrangement allowed]"
```

## CALLBACK REGISTRY

Track all callbacks for assembly verification:

```yaml
callback_registry:
  - callback_id: CALLBACK_001
    source_block: PROOF_LEAD_001
    source_content: "[specific phrase/reference]"
    target_block: PROOF_CLOSE_001
    target_content: "[how it's callbacked]"
    callback_type: [lead_to_close | proof_to_close | story_to_product | root_cause_to_mechanism]

  - callback_id: CALLBACK_002
    # ...
```

## THREADING MAP

Track how key elements are threaded:

```yaml
threading_map:
  mechanism_name:
    term: "[exact mechanism name]"
    appearances:
      - block_id: [id]
        section: [section]
        context: "[how used]"
      # ... for each appearance
    total_count: [number]
    distribution_adequate: true | false

  root_cause_anchor:
    phrase: "[anchor phrase]"
    appearances: [...]
    total_count: [number]
    distribution_adequate: true | false

  primary_promise:
    statement: "[promise]"
    appearances: [...]
    total_count: [number]
    distribution_adequate: true | false
```

## ASSEMBLY HANDOFF CHECKLIST

Before finalizing package:

```
[ ] Every proof block has unique ID
[ ] Every proof block has target section assigned
[ ] Every proof block has position specified
[ ] Every proof block has transitions drafted
[ ] All callbacks registered in callback registry
[ ] Threading map complete with adequate distribution
[ ] Density summary accurate
[ ] No orphaned blocks (all have clear placement)
[ ] Assembly instructions clear for each block
```
