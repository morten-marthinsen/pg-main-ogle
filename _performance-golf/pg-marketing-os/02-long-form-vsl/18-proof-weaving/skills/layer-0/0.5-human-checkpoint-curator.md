# Microskill 0.5: Human Checkpoint Curator

**Layer:** 0 (Foundation)
**Purpose:** Present proof approach to human for confirmation before drafting

---

## HUMAN CHECKPOINT IS BLOCKING

This checkpoint MUST receive human confirmation before proceeding to Layer 1.

**Why:** Proof weaving commits to a dramatization approach, placement strategy, and tone. These decisions should be validated before drafting begins.

---

## QUESTIONS FOR HUMAN

Present these decision points:

### Question 1: Proof Approach
> "Based on the inventory, I recommend a **[APPROACH]** approach because [rationale]. Does this align with your vision, or would you prefer a different approach?"

Options:
- Confirm recommended approach
- Switch to [Alternative 1]
- Switch to [Alternative 2]
- Custom direction

### Question 2: Lead Proof Element
> "The strongest proof element is [ELEMENT]. Should this be featured in the lead, or reserved for the proof section?"

Options:
- Feature in lead (Proof-First)
- Reserve for proof section
- Use partially in lead, fully later

### Question 3: Testimonial Style
> "For testimonials, which voice feels right for this campaign?"

Options:
- Conversational/folksy ("Let me tell you...")
- Professional/measured ("After using...")
- Emotional/dramatic ("I couldn't believe...")
- Clinical/factual ("Results showed...")

### Question 4: Proof Density Preference
> "Standard TIER1 density is 17+ proof elements distributed across sections. Would you like:"

Options:
- Standard density (17+)
- Heavy density (25+) — more proof-intensive
- Selective density (12-15) — quality over quantity

## FAILURE HANDLING

```
IF human does not respond within session:
  → PAUSE skill execution
  → Save state for continuation
  → "Waiting for human confirmation of proof approach before proceeding."

IF human rejects all recommendations:
  → Request specific direction
  → Update approach based on human input
  → Re-present for confirmation
```
