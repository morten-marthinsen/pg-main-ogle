# Microskill 1.2: Density Planner

**Layer:** 1 (Architecture)
**Purpose:** Set and validate proof density targets for each section

---

## CORE PRINCIPLE

Proof density is not uniform. Elite copy follows specific density patterns based on:
1. Market sophistication level
2. Proof inventory available
3. Copy format (VSL vs Magalog vs Sales Letter)
4. Proof approach selected (proof-first vs authority-driven vs data-heavy)

## DENSITY ADJUSTMENT RULES

### If Inventory < Pattern Target

```
IF section target > available elements:
  OPTION 1: Redistribute from lighter sections
  OPTION 2: Reuse elements with different framing
  OPTION 3: Create derivative proof (callback to earlier proof)

  WARN: "Proof density below optimal for [section]. Consider acquiring more proof."
```

### If Inventory > Pattern Target

```
IF surplus elements available:
  OPTION 1: Increase density in mechanism section (highest impact)
  OPTION 2: Add to proof section for cascade effect
  OPTION 3: Save for A/B testing variations

  NOTE: More proof is almost always better until testimonial fatigue
```

### Testimonial Fatigue Threshold

```
MAX consecutive testimonials without break: 5
MAX testimonials in single section: 8
MAX testimonial word count before mechanism return: 800

IF exceeding thresholds:
  → Insert mechanism reminder between testimonials
  → Insert "Think about it" simplification
  → Insert future pacing to break pattern
```
