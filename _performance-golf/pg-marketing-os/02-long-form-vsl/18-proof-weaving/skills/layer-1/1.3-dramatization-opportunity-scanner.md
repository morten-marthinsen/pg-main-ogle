# Microskill 1.3: Dramatization Opportunity Scanner

**Layer:** 1 (Architecture)
**Purpose:** Identify ALL opportunities to dramatize, accentuate, and emphasize proof

---

## CORE PRINCIPLE

Beyond standard proof placement, elite copy finds EVERY opportunity to dramatize proof. This microskill scans the entire copy structure for moments where proof can be accentuated, amplified, or reframed for maximum impact.

---

## DRAMATIZATION OPPORTUNITY CATEGORIES

### Category 1: Claim-Proof Proximity Gaps

**Scan for:** Claims without immediate proof support

```
FOR each claim in copy structure:
  CHECK: Is there proof within 3 sentences?

  IF NO:
    → FLAG as dramatization opportunity
    → IDENTIFY: What proof type would strengthen?
    → RATE: Impact of adding proof (High/Medium/Low)
```

**High-Impact Claim Types:**
- Promise claims ("You will experience...")
- Superiority claims ("Better than...")
- Speed claims ("In just X days...")
- Mechanism claims ("This works because...")
- Guarantee claims ("Risk-free because...")

---

### Category 2: Objection Anticipation Points

**Scan for:** Moments where reader skepticism peaks

| Objection Trigger | Dramatization Response |
|-------------------|------------------------|
| Bold opening promise | Immediate credibility proof |
| "Too good to be true" claim | Third-party validation |
| Price reveal | Value stack with proof |
| Mechanism complexity | Simplification with proof |
| "Will this work for me?" | Diverse testimonial proof |
| Urgency/scarcity claim | Justification proof |

```
FOR each section:
  IDENTIFY: Where does skepticism likely peak?
  MAP: What proof neutralizes that skepticism?
  RATE: Criticality (Must Have / Should Have / Nice to Have)
```

---

### Category 3: Emotional Peak Amplification

**Scan for:** High-emotion moments that can be reinforced with proof

| Emotional Peak | Proof Amplification |
|----------------|---------------------|
| Story crisis moment | Add suffering validation |
| Transformation reveal | Add before/after contrast |
| Mechanism "aha" moment | Add scientific validation |
| Product introduction | Add formula efficacy proof |
| Close decision point | Add risk reversal proof |

```
FOR each emotional peak:
  IDENTIFY: What emotion is at peak?
  MAP: What proof type amplifies that emotion?
  DESIGN: How should proof be dramatized?
```

---

### Category 4: Contrast Opportunities

**Scan for:** Moments where contrast can be sharpened

| Contrast Type | Dramatization Technique |
|---------------|------------------------|
| Before/After | Add specific measurable differences |
| Us vs. Them | Add competitive superiority data |
| Old Way vs. New Way | Add failure proof for old way |
| Without/With | Add consequence vs. benefit data |

```
FOR each comparison moment:
  CHECK: Is contrast maximized?
  IDENTIFY: What proof sharpens the contrast?
  DESIGN: How to dramatize the gap?
```

---

### Category 5: Scale Escalation Points

**Scan for:** Opportunities to escalate from individual to pattern to scale

| Escalation Stage | Example |
|------------------|---------|
| Individual | "Ken reversed his diabetes" |
| Pattern | "Thousands have done the same" |
| Scale | "40,000 patients prove the pattern" |

```
FOR each individual proof:
  CHECK: Can this escalate to pattern?
  CHECK: Can pattern escalate to scale?
  DESIGN: Where should escalation occur?
```

---

### Category 6: Authority Stacking Points

**Scan for:** Opportunities to stack multiple authority sources

| Authority Type | Stacking Pattern |
|----------------|------------------|
| Personal credentials | Add institutional backing |
| Single expert | Add peer validation |
| One study | Add corroborating studies |
| Track record | Add third-party recognition |

```
FOR each authority proof:
  CHECK: Can additional authority stack here?
  IDENTIFY: What complementary authority available?
  DESIGN: Stacking sequence
```

---

### Category 7: Callback Integration Points

**Scan for:** Opportunities to reference earlier proof for reinforcement

| Callback Type | Trigger Location |
|---------------|------------------|
| Lead-to-Close | Close section CTAs |
| Transformation-to-Product | Product introduction |
| Authority-to-Guarantee | Guarantee section |
| Story-to-Future-Pacing | Close visualization |

```
FOR each proof element used early:
  IDENTIFY: Where can this be callbacked?
  DESIGN: Callback language
  MAP: Placement in later section
```

---

## DRAMATIZATION INTENSITY LEVELS

Based on market sophistication (from brief):

### Stage 1-2: Standard Dramatization
- Basic claim-proof pairing
- Simple testimonials
- Straightforward data

### Stage 3: Enhanced Dramatization
- Contrast amplification required
- Specific numbers mandatory
- Multiple proof types per claim

### Stage 4-5: Maximum Dramatization
- Novel proof angles required
- Counter-intuitive proof positioning
- Meta-proof (proof of proof credibility)
- Competitive differentiation proof
