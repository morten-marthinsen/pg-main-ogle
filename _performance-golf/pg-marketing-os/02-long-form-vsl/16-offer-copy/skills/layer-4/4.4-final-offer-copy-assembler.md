## PURPOSE

Assembles the complete offer-copy-package.json — the final output of Skill 16. This is the definitive assembly point where all drafted, refined, and validated sections are merged into a single package with continuous prose, validation scores, and downstream handoffs for Skill 14 (Close) and campaign assembly.

The assembler performs three critical functions: (1) merges all sections into a single continuous full_narrative_text string — the actual offer copy prose in reading order; (2) computes weighted validation scores from 4.1 principles check, 4.2 anti-slop, and 4.3 vault comparison; (3) packages downstream handoffs so Skill 14 can continue seamlessly.

This microskill produces the final output. If the overall weighted average is below 7.0, the package is flagged for remediation.

## IDENTITY

**This microskill IS:**
- The FINAL assembler for the offer copy skill

- A validation score aggregator from 4.1, 4.2, and 4.3
- A downstream handoff packager for Skill 14 and campaign assembly
- A completeness verifier — every required field must be populated

- A drafter or writer (all drafting happened in Layer 2-3)
- A validator (validation happened in 4.1-4.3)
- A remediation engine (if score < 7.0, it flags, doesn't fix)
- A close writer (Skill 14 handles that)

## PROCESS

### Step 1: Merge Sections into Full Narrative Text

1. Assemble sections in SIN Offer Presentation order:
   1. Deliverable stack (from 2.1/3.x)
   2. Product summary
   3. Bonus transition + Bonus stack (from 2.2/3.x)
   4. Value totaling + Value demonstration (from 2.3/3.x)
   5. Price presentation (from 2.4/3.x)
   6. Stack review
   7. Savings statement
   8. Risk reversal segue
   9. Guarantee details
   10. CTA 1
   11. Consequence amplification
   12. CTA 2
   13. Urgency section
   14. CTA 3
   15. Optional contrast enhancement (if present)

2. Ensure smooth transitions between merged sections (transitions from 3.1)
3. Produce one continuous string: full_narrative_text
4. Compute total word count

### Step 2: Compute Validation Scores

5. Aggregate dimension scores:

```yaml
scores:
  dfwbp_completeness: float  # from 4.1 D-F-W-B-P audit
  value_demonstration_impact: float  # from 4.3 vault comparison
  price_psychology: float  # from 4.3 vault comparison
  guarantee_strength: float  # from 4.3 vault comparison
  cta_variety: float  # from 4.3 vault comparison + 4.1 CTA audit
  emotional_escalation: float  # from 3.3 verification
  transition_smoothness: float  # from 3.1 optimization
  promise_restatement_quality: float  # from 3.2 calibration
  vault_pattern_comparison: float  # from 4.3 overall score
```

6. Compute overall weighted average:
   - D-F-W-B-P completeness: 15% weight
   - Value demonstration: 10% weight
   - Price psychology: 10% weight
   - Guarantee strength: 10% weight
   - CTA variety: 10% weight
   - Emotional escalation: 10% weight
   - Transition smoothness: 10% weight
   - Promise restatement: 10% weight
   - Vault comparison: 15% weight

7. Flag if overall < 7.0 — remediation needed

### Step 3: Package Offer Principles Check

8. Include pass/fail for all 10 principles:

```yaml
offer_principles_check:
  dfwbp_sacred: boolean
  promise_restated_not_repeated: boolean
  value_before_price: boolean
  if_all_it_did_present: boolean
  guarantee_is_feature: boolean
  three_ctas_minimum: boolean
  consequence_amplification: boolean
  urgency_justified: boolean
  stack_review_before_price: boolean
  seamless_entry: boolean
  all_ten_pass: boolean
```

### Step 4: Package Anti-Slop Results

9. Include:
```yaml
anti_slop:
  violations: integer  # must be 0
  pass: boolean
```

### Step 5: Package Downstream Handoffs

10. For Skill 14 (Close):
```yaml
for_close:
  guarantee_reference: string  # branded name + key terms for close restatement
  cta_language_used: [string]  # phrases already used, avoid exact repetition
  urgency_reference: string  # urgency justification for close continuation
  emotional_state_at_handoff: string  # prospect's emotional state after offer section
  price_established: float  # actual price for reference in close
  value_established: float  # total value for reference in close
  primary_promise_latest_restatement: string  # most recent promise phrasing
```

11. For campaign assembly:
```yaml
for_campaign_assembly:
  full_offer_section: string  # complete offer copy for insertion
  deliverable_names: [string]
  bonus_names: [string]
  total_value: float
  actual_price: float
  guarantee_branded_name: string
```

### Step 6: Assemble Final Package

12. Produce the complete offer-copy-package.json with all fields from the output schema
13. Verify every required field is populated (no null/empty required fields)
14. Verify full_narrative_text is a single continuous string
15. Verify overall weighted average is computed and >= 7.0

### Step 7: Final Completeness Check

16. Verify:
    - [ ] metadata populated (skill, version, timestamp, niche, run_id)
    - [ ] offer_confirmed populated (product name, format, counts, checkpoint response)
    - [ ] pricing_confirmed populated (retail, actual, savings, terms)
    - [ ] deliverable_stack populated with full text and per-item detail
    - [ ] bonus_section populated with full text and per-item detail
    - [ ] value_demonstration populated with 3+ iterations
    - [ ] price_presentation populated with anchors and stack review
    - [ ] risk_reversal populated with branded guarantee
    - [ ] cta_sequence populated with 3+ CTAs and consequence amplification
    - [ ] full_narrative_text is non-empty string
    - [ ] validation scores computed
    - [ ] offer_principles_check completed (all 10)

17. If any check fails, flag the specific missing field for remediation

## FAILURE MODES

### Failure Mode 1: Overall Score Below 7.0
- **Symptom:** Weighted average < 7.0
- **Cause:** One or more dimensions scored poorly
- **Prevention:** Layer 3 refinement should have caught quality issues
- **Recovery:** Identify lowest-scoring dimension, trigger targeted remediation in that layer

### Failure Mode 2: Missing Sections
- **Symptom:** full_narrative_text has gaps or missing sections
- **Cause:** Layer 2 microskill didn't produce output for a section
- **Prevention:** Step 7 completeness check catches all missing fields
- **Recovery:** Identify missing section, re-run the specific Layer 2 microskill

### Failure Mode 3: Incoherent Merge
- **Symptom:** Sections are present but don't flow together
- **Cause:** Transitions from 3.1 not properly applied during merge
- **Prevention:** Step 1 includes transition verification during merge
- **Recovery:** Re-run 3.1 transition optimization with the merged text
