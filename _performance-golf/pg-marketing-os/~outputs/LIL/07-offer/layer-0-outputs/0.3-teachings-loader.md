## Teaching Frameworks Loaded

### Framework 1: Alex Hormozi — $100M Offers (Grand Slam Offer)

**Value Equation (Core Formula):**
```
Value = (Dream Outcome × Perceived Likelihood) / (Time Delay × Effort & Sacrifice)
```

**Grand Slam Construction Steps:**
1. Identify Dream Outcome — what the prospect ACTUALLY wants (their transformation)
2. List ALL Problems — before, during, and after using product
3. Create Solutions for Every Problem
4. Design Delivery Vehicles — attention level, client effort, delivery method, frequency
5. Trim & Stack — remove low-value/high-cost, keep high-value/low-cost, stack into bundle

**Enhancement Stack:**
- Scarcity (quantity-based) — limited supply, limited bonuses, never available again
- Urgency (time-based) — cohort, seasonal, pricing, exploding opportunity
- Bonuses — 12-point checklist: name with benefit, address issues, prove value, assign price, counter objections, anticipate needs, exceed core value
- Guarantees — unconditional, conditional, anti-guarantee, implied/stacked
- Naming — 5-part formula: magnetic reason + avatar + goal + time interval + container word

**LIL Adaptation Notes:**
- Scarcity: ONLY if real constraint exists (launch quantity, production run). No artificial scarcity.
- Urgency: MARKED N/A per upstream constraint. No urgency mechanics. Seasonal proximity to Father's Day is a natural asset, not an urgency mechanic.
- Bonuses: Must serve branded physical product context. Digital bonuses (PG1 app) have zero marginal cost.
- Guarantees: GAP-01 — architect options for Leadership. Must be branded, not generic.
- Naming: Adapt 5-part formula for physical product (no "challenge" / "blueprint" container words).

### Framework 2: Todd Brown — SIN Offer Presentation (D-F-W-B-P)

**Presentation Flow:**
1. Core Product Introduction — introduce by name
2. Deliverable Presentations — D-F-W-B-P for each component
3. Product Summary — restate primary promise
4. Transition to Bonuses
5. Bonus Presentations — D-F-W-B-P for each bonus
6. Total Value Statement
7. Value Demonstration — "If all it did was..." technique (3 rounds)
8. Price Presentation — anchor → discount → actual → terms → savings
9. Risk Reversal — guarantee presentation
10. CTA Sequence — 3 CTAs with consequence amplification

**D-F-W-B-P Formula:**
- D = Deliverable: "The [ordinal] thing you get is [name]..."
- F = Feature: What it IS (functional description)
- W = Why: Why this matters / why it's included
- B = Benefit(s): What it DOES for the prospect (outcome language)
- P = Proof: Evidence that this deliverable works

**LIL Adaptation Notes:**
- Presentation script adapted for e-comm product page, NOT VSL
- D-F-W-B-P applied to each physical component (3 clubs, Skills Mat, carry bag)
- "If all it did was..." adapted for e-comm value section — not spoken script
- CTA sequence simplified to single CTA for e-comm (Add to Cart)
- Consequence amplification replaced with future-pacing (hope market, not pain amplification)
- Risk reversal section holds space for guarantee (GAP-01 PENDING)
- No "But that's not all" transitions — violates Soul.md anti-voice
