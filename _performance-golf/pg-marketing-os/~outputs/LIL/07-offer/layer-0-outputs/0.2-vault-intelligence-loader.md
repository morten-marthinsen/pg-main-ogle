## Vault Pattern Extraction: Offer Structures

### Applicable TIER1 Offer Patterns for Physical Product E-Comm

**Note:** The vault contains 139+ swipe files primarily from info-product and DR contexts. For a branded physical product e-comm launch (Lil' Legends), the following patterns are adapted — not applied wholesale.

### Pattern 1: System Bundle (Physical Product)
- **Structure:** Core product + training component + accessory + digital bonus
- **Price psychology:** Position as "system" not "product" to justify premium over individual components
- **Value anchor:** Sum of individual component values vs. system price
- **Applicable:** YES — LIL is already a system (3 clubs + Skills Mat + carry bag)

### Pattern 2: Premium-Tier Physical Product with Digital Enhancement
- **Structure:** Physical product + app access / digital content as bonus
- **Price psychology:** Physical product anchors the value; digital content is "free" bonus that costs nothing to deliver but adds perceived value
- **Applicable:** YES — PG1 app (30-day free trial) is available as digital bonus

### Pattern 3: Developmental/Educational Product Offer
- **Structure:** Core product + guide/curriculum + community access
- **Price psychology:** Frame as developmental investment, not purchase. "Investment in their development" language (adapted for voice — "give them the game")
- **Value anchor:** Compare to lessons, tutoring, developmental programs
- **Applicable:** PARTIALLY — developmental framing fits, but must avoid clinical tone per Soul.md

### Pattern 4: Father's Day / Gift-Occasion Offer
- **Structure:** Product + gift-ready packaging + personal element
- **Price psychology:** Gift buyers have higher WTP and lower price sensitivity
- **Timing:** Launch May 27 = 3 weeks before Father's Day (June 15, 2026)
- **Applicable:** YES — seasonal timing is a natural asset per Soul.md

### Pattern 5: "Final Answer" Replacement Offer
- **Structure:** Position as the product that replaces the serial failure cycle
- **Price psychology:** Cost math — total spent on failures vs. one system that works
- **Value anchor:** "3 broken sets at $30 each = $90 wasted + frustrated kid. One system that works = $149."
- **Applicable:** YES — serial purchase pattern (2-4 failed sets) is strongly documented in research

## Offer Architecture Patterns (Adapted for Branded Physical Product)

### Value Stack Pattern (Physical Product Adaptation)
```
Core Product (physical)          → Stated value based on component quality
Included Accessory              → Would cost $X separately
Training Component              → Would cost $X if sold alone
Digital Bonus (app/guide)       → "Free" — zero marginal cost, high perceived value
```

### Price Anchoring Pattern (E-Comm Adaptation)
```
Anchor 1: Competitor comparison  → "Charlie Golf: $190-338 for clubs only"
Anchor 2: Component math         → "3 clubs + mat + bag individually = $X"
Anchor 3: Failure cycle math     → "3 broken cheap sets = $60-120 wasted"
Anchor 4: Lesson comparison      → "One 30-min private lesson for a toddler = $50-75"
Actual price                     → $149 (projected mid-range of $129-$179)
Savings frame                    → "Everything you need for less than what most parents waste on sets that break"
```

### Guarantee Pattern (Physical Product)
```
Standard:     30-day satisfaction guarantee (industry norm)
Enhanced:     60-90 day "give it a real try" guarantee
Conditional:  "Use the Skills Mat for 30 days. If your kid isn't asking to play..."
Branded:      Named guarantee with specific promise language
```

### Bonus Pattern (Physical Product with Digital)
```
Bonus 1: Digital content (zero cost to deliver, high perceived value)
Bonus 2: Extended access or subscription trial
Bonus 3: Physical accessory or upgrade
```

## Anti-Patterns (What NOT to Import from Info-Product Vault)

1. **Do NOT import countdown timers** — branded initiative, no urgency mechanics
2. **Do NOT import "order bumps"** — e-comm product page, not funnel
3. **Do NOT import "fast action bonuses"** — no artificial urgency
4. **Do NOT import "pay-what-you-want" pricing** — physical product with fixed costs
5. **Do NOT import "money-back PLUS keep the bonuses"** — physical product has return logistics
