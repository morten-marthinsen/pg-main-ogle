### Template 1: Physical Product System Mechanism (Most Relevant to LIL)
- **Source:** P90X / Invisalign / FrogTape composite pattern
- **Type:** system
- **Explanation skeleton:**
  1. **Problem setup** (~15%) — What you've been doing doesn't work, and here's why
  2. **Root cause reframe** (~20%) — The real reason is [principle], not [what you think]
  3. **Mechanism introduction** (~15%) — We built a system based on [principle]
  4. **How-it-works sequence** (~25%) — Component A does X, Component B does Y, Component C does Z
  5. **Proof demonstration** (~15%) — Watch what happens when [knockout demo]
  6. **Differentiation claim** (~10%) — No other [product] does this because [reason]
- **Naming construction:** Name the PRINCIPLE, not the components
  - P90X → "Muscle Confusion" (principle: varying exercises prevents plateau)
  - LIL analogue → name the principle that makes equipment teach without words
- **Proof integration:** Demonstration woven into explanation (not appended)

### Template 3: Behavioral/Experiential Mechanism (Tertiary — Golf Vernacular)
- **Source:** PG vault — "Simple Strike Sequence," "First Move Fix"
- **Type:** behavioral
- **Explanation skeleton:**

  6. **Close** (~10%) — Golf buddy, not golf coach
