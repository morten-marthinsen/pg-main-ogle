## Key Decisions Log

1. **DSI is product-level, not per-feature.** SwingScan Diagnosis is the DSI mechanism. Other features support it.
2. **'Diagnosis' language threads through everything.** The hero feature name, the CTA, the guarantee name (The Diagnosis Promise), and the bonuses all use diagnosis/flaw language. This creates linguistic cohesion across the entire page.
3. **Technical features carry proof weight.** F-006 (2M+ Swing Training Database) and F-007 (96% Diagnostic Accuracy) exist primarily as proof elements. Their feature names ARE their proof — the numbers communicate credibility.
4. **Bonuses are named as deliverables.** F-009 (The Root Flaw Report) and F-010 (The Flaw Chain Map) use definite articles ("The") to signal specific, tangible things you receive — not abstract features.
5. **No 'AI-powered' or 'personalized' in feature names.** Per campaign brief language rules (SC-01 staleness 9, SC-02 staleness 8), these saturated patterns are avoided in all feature naming. Adaptive Intelligence describes BEHAVIOR (adapts). Personalized Correction Plan specifies WHAT is personalized (a correction plan from diagnosis).
6. **Arena adapted for autonomous mode.** Feature names derived from mechanism package, campaign brief, and offer package rather than multi-persona competition. All names validated against generic kill list and standalone test. Human review recommended before EC-02.
