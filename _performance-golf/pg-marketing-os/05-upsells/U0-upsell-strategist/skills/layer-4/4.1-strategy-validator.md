# 4.1 — Strategy Validator

**Layer:** 4
**Model:** sonnet
**Depends on:** GATE_2 HUMAN_APPROVED
**Produces:** `layer-4/4.1-validation-results.md`

---

## Purpose

Validate the complete upsell strategy against all rules and constraints before final packaging. This is the last quality gate before output.

## Validation Checklist

### Pricing Cascade Validation

| Check | Rule | How to Validate |
|-------|------|-----------------|
| Bump price range | 10-30% of FE | Calculate: bump_price / fe_price × 100 |
| Upsell 1 price range | 50-150% of FE | Calculate: upsell_price / fe_price × 100 |
| Downsell 1 price range | 30-50% of upsell 1 | Calculate: downsell_price / upsell_price × 100 |
| Descending commitment | Each step asks less | Verify price pattern or complexity pattern |
| Ascending value/$ | Each step delivers more per dollar | Verify value proposition grows relative to price |

### Congruence Validation

| Check | Rule | How to Validate |
|-------|------|-----------------|
| Mechanism name | Exact match in every position | String match against front-end mechanism name |
| Root cause | Consistent language | Same framing, no new problems introduced |
| Promise | Extends, never replaces | Each position adds to base promise |
| Buyer validation | Present in every opener | Every position references the purchase decision |

### Structural Validation

| Check | Rule | How to Validate |
|-------|------|-----------------|
| Minimum positions | ≥ 2 (bump + upsell) | Count positions |
| No invented products | All products are real | Cross-reference against input |
| Word count targets | Within position ranges | Bump 50-150, Upsell 500-2000, Downsell 300-1000 |
| Handoff specs complete | All fields populated | Check 2.3 output for completeness |
| Downsell reframed | Not just "cheaper upsell" | Verify reframe_type and angle specified |

### Narrative Validation

| Check | Rule | How to Validate |
|-------|------|-----------------|
| Emotional arc | Distinct targets per beat | No repeated emotional targets |
| Transitions | Natural flow between beats | Read sequence in order |
| All-NO path respectful | Decline path isn't guilt-laden | Read decline texts and acknowledgments |
| All-YES path celebratory | Full buyer path feels rewarding | Read all-YES sequence |
