# 4.2 — Output Packager

**Layer:** 4
**Model:** sonnet
**Depends on:** 4.1 (GATE_3 PASS)
**Produces:** `upsell-strategy.yaml` + `UPSELL-STRATEGY-SUMMARY.md`

---

## Purpose

Package the validated strategy into final deliverables: a machine-readable YAML file for downstream skill consumption and a human-readable summary.

## Execution Steps

1. Read all Layer 1, 2, and 4.1 outputs
2. Assemble `upsell-strategy.yaml` following the schema in AGENT.md
3. Write `UPSELL-STRATEGY-SUMMARY.md` for human reference
4. Verify all required YAML fields are populated
5. Write GATE_3 checkpoint file

## Output 1: upsell-strategy.yaml

Follow the exact schema defined in U0-UPSELL-STRATEGIST-AGENT.md → OUTPUT SCHEMA section. All fields must be populated. No empty strings, no placeholders.
