# 0.3 — Input Validator

**Layer:** 0
**Model:** haiku
**Depends on:** 0.1 or 0.2
**Produces:** `layer-0/0.3-input-validation.md`

---

## Purpose

Validate that sufficient context has been loaded to design a credible upsell strategy. Determine GATE_0 status.

## Execution Steps

1. Read output from 0.1 (Mode A) or 0.2 (Mode B)
2. Validate completeness:

### Mode A Validation

| Check | Required | Validation |
|-------|----------|------------|
| Product name | YES | Non-empty string |
| Product price | YES | Numeric > 0 |
| Bonus stack | NO | If present, list of items |
| Guarantee | NO | If present, type + terms |
| Mechanism name | RECOMMENDED | If missing, flag but proceed |
| Audience profile | YES | Demographics OR psychographics present |
| Upsell candidates | YES | At least 1 product identified for upsell |

### Mode B Validation

| Check | Required | Validation |
|-------|----------|------------|
| Product details | YES | Name + price at minimum |
| Upsell candidates | YES | At least 1 product/service |
| Audience | YES | Any description |
| Mechanism | NO | If missing, strategy will infer |

3. Determine GATE_0:
   - PASS: All required checks satisfied
   - FAIL: Any required check missing

4. Write validation results + GATE_0 status

## Output Format

```markdown
# U0 Input Validation

## Mode: [A/B]

## Validation Results
| Check | Status | Notes |
|-------|--------|-------|
| [check] | PASS/FAIL | [details] |

## GATE_0: [PASS/FAIL]

## Flags
- [any warnings or recommendations]
```

## Gate File

If PASS, write `checkpoints/GATE_0_VERIFIED.yaml`:

```yaml
gate: GATE_0
status: PASS
mode: [A/B]
timestamp: [ISO 8601]
packages_loaded: [count]
flags: [list or empty]
```
