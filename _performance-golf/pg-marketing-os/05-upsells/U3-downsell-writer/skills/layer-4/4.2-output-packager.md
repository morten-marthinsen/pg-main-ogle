# 4.2 — Output Packager

**Version:** 1.0
**Skill:** U3 — Downsell Writer
**Microskill:** 4.2 — Output Packager
**Layer:** 4
**Model:** sonnet
**Input:** Arena-selected downsell page (validated — GATE_3 must pass first), `layer-4/4.1-validation-results.md` (confirmation all 8 gates passed), `layer-1/1.1-reframe-selection.md` (reframe type, angle), `layer-1/1.2-congruence-map.md` (mechanism name, root cause), `layer-0/0.1-input-context.md` (all product/pricing/position metadata)
**Output:** `downsell-page-draft.md` (in project root, NOT in layer-4)

---

## PURPOSE

Package the validated, Arena-selected downsell page into the final deliverable file (`downsell-page-draft.md`) with a complete YAML metadata header and a downstream handoff section for U4 (Upsell Sequence Assembler). This is the terminal output of U3 — the file that leaves the skill and enters the next stage of the pipeline.

---

## EXECUTION STEPS

1. **Verify prerequisites:**
   - GATE_3 passed (all 8 validation checks in 4.1). If GATE_3 has not passed, HALT. Cannot package unvalidated output.
   - Arena-selected file identified and validated
   - All metadata sources available (0.1-input-context.md, 1.1-reframe-selection.md, 1.2-congruence-map.md)

2. **Collect metadata for YAML header:**
   - `skill`: "U3 — Downsell Writer"
   - `version`: "1.0"
   - `project`: [project name from 0.1-input-context.md]
   - `position`: [downsell_1 | downsell_2] from 0.1-input-context.md
   - `product`: [downsell product name] from 0.1-input-context.md
   - `price`: [downsell price] from 0.1-input-context.md
   - `declined_upsell_product`: [upsell product name] from 0.1-input-context.md
   - `declined_upsell_price`: [upsell price] from 0.1-input-context.md
   - `reframe_type`: [core_extract | payment_plan | lite_version | different_format] from 1.1-reframe-selection.md
   - `fe_product`: [FE product name] from 0.1-input-context.md
   - `fe_price`: [FE price] from 0.1-input-context.md
   - `fe_mechanism`: [exact mechanism name] from 1.2-congruence-map.md
   - `word_count`: [actual count from validated output]
   - `arena_selection.persona`: [selected competitor or hybrid name]
   - `arena_selection.type`: [pure | hybrid]
   - `arena_selection.weighted_score`: [score from Arena synthesis]
   - `mode`: [A | B]
   - `created`: [current date YYYY-MM-DD]

3. **Extract the validated downsell copy.** Take the Arena-selected, validated copy and format it with clean ARO section headers:
   - `## ACKNOWLEDGE` with word count
   - `## REFRAME` with word count
   - `## OFFER + CHOICE` with word count
   - Ensure the 2.2 price presentation has been integrated into the Offer + Choice section (no remaining `[PRICE PRESENTATION — from 2.2]` placeholders)

4. **Build the downstream handoff section for U4:**
   - `downsell_product`: [product name]
   - `downsell_price`: [price]
   - `reframe_type_used`: [type]
   - `reframe_angle`: [one-sentence summary from 1.1-reframe-selection.md angle statement]
   - `position`: [downsell_1 | downsell_2]
   - `word_count`: [actual count]
   - `upsell_declined_context.upsell_product`: [what they said no to]
   - `upsell_declined_context.upsell_price`: [upsell price]
   - `upsell_declined_context.upsell_angle`: [upsell angle]
   - `mechanism_name`: [exact — from congruence map]
   - `root_cause_language`: [exact — from congruence map]
   - `guarantee_type`: [mirrored | downsell-specific]
   - `acknowledge_tone`: [summary — e.g., "validating, zero-pressure, affirms FE purchase"]

   U4 needs this handoff to:
   - Validate cross-page narrative threading (mechanism name consistent, root cause consistent)
   - Position the downsell correctly in the assembled sequence
   - Understand the reframe angle for narrative arc continuity

5. **Verify output completeness:**
   - YAML header has all required fields (no empty values, no placeholders)
   - Copy has all 3 ARO sections with section headers
   - Price presentation is integrated (no placeholders remaining)
   - Binary CTA is present (both YES and NO)
   - Downstream handoff section is complete
   - Word count is reported accurately

6. **Write the final output file** to the project root directory (NOT layer-4).

## CONSTRAINTS

- Output file goes to PROJECT ROOT, not layer-4. The path is `[project-dir]/U3-downsell-writer/downsell-page-draft.md`, not `[project-dir]/U3-downsell-writer/layer-4/downsell-page-draft.md`. Layer-4 is for validation results, not deliverables.
- GATE_3 must have passed before packaging. Do not package unvalidated output. If 4.1 has not run or reported any FAIL, HALT.
- No remaining placeholders in the output. If `[PRICE PRESENTATION — from 2.2]` or any other placeholder is still present, the integration is incomplete. Resolve before writing.
- YAML metadata must have ALL fields populated. No empty strings, no "TBD," no "N/A" for required fields. If a value is genuinely unknown, it should have been caught at GATE_0.
- Downstream handoff must be COMPLETE. U4 cannot assemble the sequence without knowing the downsell's position, reframe type, pricing, and congruence anchors. Missing handoff data = broken pipeline.
- Word count in metadata must match the ACTUAL count in the validated copy. Do not carry forward a word count from a pre-Arena version.
- Mode B flag: if operating in Mode B, include "Mode B — limited upstream context" in metadata and note it in the downstream handoff.

---

## FAILURE MODES

| Failure | Detection | Response |
|---------|-----------|----------|
| Packaging before validation | GATE_3 checkpoint does not exist or shows FAIL | HALT. Run 4.1 first. Cannot package unvalidated output. |
| Placeholder remaining in copy | Text search finds `[PRICE PRESENTATION` or similar bracket patterns | Resolve placeholder. If 2.2 has not run, run it. If 2.2 output exists, integrate it. |
| YAML metadata incomplete | Any metadata field is empty, "TBD," or "N/A" | Fill from source files (0.1, 1.1, 1.2, Arena synthesis). If source data is missing, it should have failed at GATE_0. |
| Downstream handoff missing fields | Any handoff field is empty or missing | Fill from source files. U4 cannot function without complete handoff data. |
| Output written to wrong directory | File appears in layer-4/ instead of project root | Move to project root. Deliverables go to root, intermediate outputs go to layer directories. |
| Word count mismatch | Metadata word count differs from actual validated copy | Recount from the packaged copy. Use the actual count, not a carried-forward number. |
