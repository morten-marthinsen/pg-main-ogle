# 4.2 — Sequence Criteria Validator

**Version:** 1.0
**Skill:** U5 — Upsell Editorial
**Layer:** 4
**Microskill:** 4.2 — Sequence Criteria Validator
**Model:** sonnet
**Purpose:** Validate all 5 Sequence-Level Criteria (S1-S5) against the REVISED upsell sequence. Each criterion is evaluated with concrete evidence and scored PASS or FAIL. All 5 must pass for Gate 3 to clear.

---

## INPUT

- The REVISED upsell sequence (post-Layer 2 revisions — the current state of all copy)
- `layer-0/0.1-input-context.md` (pricing data, congruence thread, piece inventory)
- `layer-0/0.2-scoring-rubric.md` (S1-S5 criteria definitions and pass conditions)
- `layer-1/1.1-baseline-scores.md` (baseline S1-S5 results — for comparison)
- `layer-4/4.1-rescores.md` (to confirm all pieces >= 7.5 before sequence validation)
- UPSELL-ENGINE.md (5 Laws, pricing cascade standards, word count limits)

## EXECUTION STEPS

1. **Confirm prerequisite.** Verify that 4.1 shows ALL pieces >= 7.5 weighted average. If any piece is below threshold, this microskill should not execute — return to Layer 2 first. Sequence-level criteria are meaningless if individual pieces are failing.

2. **Validate S1 — Pricing Cascade Compliance.**
   - Extract all prices from the REVISED sequence:
     - FE price: $[price] (from 0.1 or as referenced in sequence)
     - Order bump price: $[price]
     - Upsell price: $[price]
     - Downsell price: $[price]
   - Calculate ratios:
     - Bump/FE ratio: [bump price] / [FE price] = [ratio] (standard range: 20-50%)
     - Upsell/FE ratio: [upsell price] / [FE price] = [ratio] (standard range: 50-150%)
     - Downsell/Upsell ratio: [downsell price] / [upsell price] = [ratio] (standard range: 30-60%)
   - Check for LOSS_LEADER_MODE: was this documented in U0 strategy? If yes, wider ranges are acceptable.
   - Check: Downsell price < Upsell price (absolute rule — no exceptions)
   - Check: No inverted cascade (each step should descend or maintain, never increase without clear justification)
   - **PASS:** All ratios within standard ranges OR LOSS_LEADER_MODE documented with rationale
   - **FAIL:** Any ratio outside range without LOSS_LEADER_MODE documentation, OR downsell >= upsell price

3. **Validate S2 — Congruence Chain.**
   - Scan EVERY piece in the REVISED sequence for:
     - **Mechanism name:** Search for the EXACT mechanism name string. Record presence/absence and location per piece.
     - **Root cause language:** Search for the VERBATIM root cause phrasing. Record presence/absence and location per piece.
     - **Promise threading:** Check that the promise is EXTENDED at each step, not replaced. The downsell should still reference the same promise, offering a different path to it.
   - Compile cross-piece consistency report:
     - Mechanism name appears in: [list pieces]
     - Root cause language appears in: [list pieces]
     - Promise continuity: [extended at each step / broken at step X]
   - **PASS:** Same mechanism name on every page, root cause language present on every page (verbatim or with minor natural variation), promise extended (not replaced) across the sequence
   - **FAIL:** Mechanism name missing from any page, root cause language absent or replaced with different framing, promise replaced (new promise introduced mid-sequence)

4. **Validate S3 — Speed Compliance.**
   - Count words in EVERY piece of the REVISED sequence:
     - Order bump: [count] words
     - Upsell page: [count] words
     - Downsell page: [count] words
     - Total: [sum] words
   - Calculate reading time: [total words] / 200 wpm = [minutes]
   - Check per-piece limits:
     - Bump <= 150 words
     - Upsell <= 2000 words
     - Downsell <= 1000 words
   - **PASS:** Total reading time < 4 minutes AND all pieces within individual word count limits
   - **FAIL:** Total reading time >= 4 minutes OR any piece exceeds its word count limit

5. **Validate S4 — Voice Consistency.**
   - Scan ALL pieces for post-purchase psychology compliance:
     - **Bump + Upsell pages:** Expected tone = celebration, validation, logical extension
     - **Downsell page:** Expected tone = acknowledgment, respect, genuine alternative
   - Scan for violations (pre-purchase selling language):
     - Urgency language: "limited time," "act now," "don't miss," "before it's too late"
     - Fear language: "imagine your problem getting worse," "what if you don't"
     - Guilt language: "you owe it to yourself," "don't you deserve," "can you afford not to"
     - PAS fragments: problem-agitate-solve structure in any section
   - Scan downsell specifically for:
     - Guilt in Acknowledge section
     - "Just cheaper" framing instead of genuine reframe
     - Passive aggression in NO option
   - **PASS:** Zero violations. Post-purchase tone maintained throughout. Downsell acknowledges without guilting.
   - **FAIL:** Any selling language violation detected. Any guilt in downsell. Any PAS structure in upsell or downsell.

6. **Validate S5 — Value Escalation.**
   - At each step in the sequence, assess value-per-dollar:
     - **FE -> Bump:** Bump should add value at a fraction of FE price (high value-per-dollar)
     - **FE -> Upsell:** Upsell should offer premium value. If price is higher than FE, the value must clearly exceed the price increase proportionally.
     - **Upsell -> Downsell:** Downsell offers MOST value per dollar (lowest price point, still meaningful benefit)
   - Check the "descending commitment + ascending value" pattern:
     - Commitment (perceived risk/decision weight) should decrease at each step after the initial purchase
     - Value-per-dollar should increase or maintain at each step
   - **PASS:** Each step demonstrably offers competitive or increasing value per dollar. Descending commitment pattern maintained.
   - **FAIL:** Any step where value-per-dollar clearly decreases. Any step where commitment/risk increases (e.g., downsell requires more thought than upsell).

7. **Compare to baseline S1-S5.** Note any criteria that changed status (FAIL->PASS or PASS->FAIL) after revision. A criterion that was PASS at baseline and is now FAIL indicates a revision broke something — flag for immediate attention.

8. **Determine overall S-criteria status.** All 5 must PASS for Gate 3.

9. **Write output file** with complete validation results.

## CONSTRAINTS

1. **Validate the REVISED sequence.** Not the original. Not the baseline copy. The copy after all Layer 2 revisions.
2. **S-criteria are PASS or FAIL.** No partial pass. No "conditional." Binary.
3. **All 5 must pass.** One failing criterion = sequence fails Gate 3. Return to Layer 2 for targeted fix.
4. **Evidence is mandatory.** Every PASS and FAIL must include specific evidence — quoted passages, calculated numbers, exact prices. An unsupported verdict is not a verdict.
5. **Regression from baseline is critical.** A criterion that was PASS at baseline and is now FAIL means a revision BROKE something. This is higher priority than a criterion that was already failing.
6. **S3 word counts must be EXACT.** Count words, do not estimate. The 4-minute / 800-word total is not negotiable.
7. **S4 violation scanning must be thorough.** The violation list in the execution steps is not exhaustive. Use judgment: any language that assumes the reader has NOT yet purchased is a violation.

## FAILURE MODES

| Failure | Detection | Response |
|---------|-----------|----------|
| Validating original instead of revised | Evidence quotes match baseline, not revised copy | Re-read the revised sequence. Validate the current state. |
| Missing criterion | Fewer than 5 criteria validated | Validate all 5. No criterion is optional. |
| Evidence-free verdict | PASS/FAIL stated without supporting quotes or calculations | Add evidence. Unsupported verdicts are not credible for human review. |
| Regression not detected | Baseline was PASS, final is FAIL, but no regression flag | Add regression flag. This is the highest-priority finding. |
| Word count estimated | S3 uses approximate word counts | Recount exactly. Word count drives reading time calculation. |
| S2 checked per-piece only | Congruence checked within each piece but not ACROSS pieces | S2 is about the CHAIN. Check cross-piece consistency: does the mechanism name thread FROM bump THROUGH upsell TO downsell? |
