# 1.2 — Narrative Thread Validator

**Skill:** U4 — Upsell Sequence Assembler
**Microskill:** 1.2 — Narrative Thread Validator
**Layer:** 1
**Model:** sonnet
**Input:** `layer-0/0.1-input-context.md`, `layer-1/1.1-sequence-order.md`, all loaded copy pieces (U1, U2, U3)
**Output:** `layer-1/1.2-thread-validation.md`

---

## PURPOSE

Verify that 5 mandatory narrative thread elements are present and consistent across ALL pages in the upsell sequence. These threads are the connective tissue that makes the sequence feel like one continuous experience rather than disconnected sales pitches.

---

## EXECUTION STEPS

1. **Read `layer-0/0.1-input-context.md`** for the canonical mechanism name, root cause language, and promise. These are the reference strings for thread checking.

2. **Read `layer-1/1.1-sequence-order.md`** for the ordered list of pieces and their positions.

3. **Read each loaded copy piece in sequence order.** For each page, search for and document the following 5 threads:

4. **Thread 1: Mechanism Name** — Same EXACT name on every page.
   - Search each page for the mechanism name string from mechanism-package.json
   - Match must be EXACT — "The Metabolic Switch Protocol" is not the same as "Metabolic Switch" or "the metabolic switching method"
   - Per page: cite the exact quote containing the mechanism name, or flag MISSING
   - PASS: mechanism name appears on every loaded page with exact string match
   - FAIL: mechanism name missing from any page, or appears with different wording

5. **Thread 2: Root Cause Reference** — 3+ times across the entire sequence.
   - Search for "Because [root cause]..." constructions and root cause language variations
   - The root cause need not be verbatim on every mention, but the core concept must be recognizable
   - Count total appearances across all pages
   - Per appearance: cite exact quote, page, and location
   - PASS: 3 or more root cause references across the full sequence
   - FAIL: fewer than 3 total references

6. **Thread 3: Promise Extension** — Each step adds to the promise, not repeats it.
   - The bump should COMPLETE the FE promise (fill a gap)
   - The upsell should ACCELERATE or AMPLIFY the FE promise (make it faster/bigger)
   - The downsell should deliver the CORE of the promise (essential version)
   - Per page: identify the promise language used and classify how it extends vs the FE promise
   - PASS: each page extends the promise in a distinct way (complete/accelerate/core)
   - FAIL: any page merely repeats the FE promise without extension, or contradicts it

7. **Thread 4: Buyer Validation** — "Smart decision" / "right choice" or equivalent on every page opening.
   - Search the first 2-3 sentences of each page for buyer validation language
   - Acceptable variants: "You made the right choice," "Smart decision," "Congratulations on your purchase," "Great choice," or similar language that validates the prior purchase
   - Per page: cite the exact opening validation language, or flag MISSING
   - PASS: every page opens with buyer validation
   - FAIL: any page lacks opening validation (the buyer feels uncertain instead of affirmed)

8. **Thread 5: Price Anchoring** — On upsell AND downsell pages.
   - Search upsell and downsell pages for price anchor references (reference back to FE value, prior pricing, or "for less than..." comparisons)
   - The bump does NOT require price anchoring (it's too short and appears during checkout)
   - Per qualifying page: cite the exact anchoring language
   - PASS: both upsell and downsell pages contain price anchoring
   - FAIL: either qualifying page lacks price anchoring

9. **Compile the per-thread, per-page evidence matrix.** Every cell must contain either an exact quote or the word MISSING.

10. **Calculate overall thread score:**
    - 5 threads, each PASS or FAIL
    - Total PASS count / 5 = thread compliance percentage
    - Any thread FAIL with mechanism name (Thread 1) = automatic GATE_1 escalation (structural, not cosmetic)

## CONSTRAINTS

- Every cell in the evidence matrix must contain either an EXACT QUOTE from the copy or the word MISSING. No summaries, no "yes it's there," no paraphrases. The quote IS the evidence.
- Mechanism name matching is EXACT STRING MATCH. Partial matches, abbreviations, and paraphrases all count as MISSING. "The Metabolic Switch Protocol" present on one page and "Metabolic Switch" on another is a Thread 1 FAIL.
- Root cause count is across the ENTIRE sequence, not per page. Three mentions on the upsell page and zero elsewhere still passes the count threshold, but note the uneven distribution.
- Promise extension requires DISTINCT extension per page. If the bump and the upsell both use identical promise language with no extension, Thread 3 fails for the second page.
- Buyer validation checks the OPENING of each page only (first 2-3 sentences). Validation language buried in the middle of the page does not count for this thread.
- Thread 1 (mechanism name) failure is the most critical — it breaks the congruence chain. Flag as GATE_1 escalation if mechanism name is missing from any page.

---

## FAILURE MODES

| Failure | Detection | Response |
|---------|-----------|----------|
| Mechanism name missing from one page | Thread 1 evidence matrix shows MISSING for a page | Flag as GATE_1 escalation. This is structural — the page was generated without the mechanism thread. Recommend specific fix for U5: "Add mechanism name '[exact]' to [page] in [section]." |
| Root cause references < 3 total | Thread 2 count < 3 | Flag for U5. Recommend where to add: typically Amplify (U2) and Reason (U2/U3) sections are best candidates for root cause reinforcement. |
| Promise repetition instead of extension | Thread 3 shows identical language across pages | Flag for U5. Identify which page merely repeats and suggest a specific extension angle (e.g., "Bump should frame as completing the FE, not just restating the promise"). |
| Buyer validation missing from page opening | Thread 4 shows MISSING for a page | Flag for U5. This is a simple fix — add 1 validation sentence to the page opening. Recommend specific language. |
| Price anchoring missing from upsell or downsell | Thread 5 shows MISSING | Flag for U5. Recommend where in the CAIRO structure to add the anchor (typically in the Offer section before price reveal). |
