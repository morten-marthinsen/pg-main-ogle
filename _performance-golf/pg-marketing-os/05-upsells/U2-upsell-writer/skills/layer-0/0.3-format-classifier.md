# 0.3 — Format Classifier

**Version:** 1.0
**Skill:** U2 — 1-Click Upsell Writer
**Layer:** 0
**Microskill:** 0.3 — Format Classifier
**Purpose:** Classify the output format (video script vs text page), set the word count target within 500-2000, and define format-specific structural requirements for the CAIRO draft.

---

## INPUT

- `layer-0/0.1-input-context.md` (format from U0 handoff or standalone brief, product type, price point, position)
- `layer-0/0.2-specimen-calibration.md` (specimen format distribution data)

## PROCESS

1. **Check for explicit format specification:**
   - If U0 handoff spec specifies format (video_script or text_page) → use that. U0's format recommendation is authoritative.
   - If standalone brief specifies format → use that.
   - If neither specifies → proceed to step 2 for recommendation.
2. **Recommend format based on three signals** (when not specified):
   - **(a) Product type:**
     - Physical product (supplement, device, kit) → recommend video_script (visual demonstration strengthens believability)
     - Digital product (course, PDF, software, membership) → either format works (no strong signal)
     - Service or access (coaching, community, VIP) → recommend text_page (authority and specificity matter more than visual)
   - **(b) Price point:**
     - Upsell price > $97 → recommend video_script (higher commitment benefits from presenter trust, spoken delivery)
     - Upsell price <= $97 → either format works (low enough for text decision)
   - **(c) Vertical norms:**
     - Golf → video_script (Performance Golf specimens are overwhelmingly video)
     - Health/supplement → video_script (VSL is the dominant format in health DR)
     - Finance → text_page (authority-based, data-heavy — text reads more credible)
     - Personal-dev → either (depends on brand voice)
     - Technology → text_page (feature specificity benefits from scannable text)
   - If signals conflict, weight product type highest, then price point, then vertical.
3. **Set word count target** based on position and complexity:
   - Upsell 1 (standard) → target 750-1000 words
   - Upsell 1 (complex product or high price) → target 1200-1600 words
   - Upsell 2 → target 500-750 words (shorter — buyer patience is lower)
   - HARD BOUNDARIES: never below 500, never above 2000, regardless of any other factor
4. **Define format-specific structural requirements:**

   **If video_script:**
   - Opening visual direction: `[VIDEO OPENS — Presenter on camera, warm expression]`
   - Speaker attribution: `PRESENTER:` before all spoken lines
   - Visual cues: `[CUT TO: ...]`, `[B-ROLL: ...]`, `[CLOSE-UP: ...]` at transition points
   - Text overlays: `[TEXT ON SCREEN: "..."]` for key phrases, product name, price
   - Timing marks: estimated seconds per CAIRO section (based on ~150 words/minute spoken pace)
   - CTA visual: `[CTA SCREEN: Two buttons displayed — YES / NO THANKS]`
   - Tone direction notes: `[conspiratorial lean-in]`, `[smile, nod]`, `[direct to camera]`

   **If text_page:**
   - Headline hierarchy: H1 for page title, H2 for CAIRO section transitions, H3 for sub-points
   - CTA button spec: primary button (YES) above text link (NO)
   - Layout notes: suggested image/graphic placement points
   - Formatting cues: bold for key phrases, horizontal rules between major sections
   - Mobile-first consideration: short paragraphs (2-3 sentences max), scannable structure

5. **Calculate CAIRO section timing** (video only):
   - Congratulate: 20-40 seconds
   - Amplify: 40-120 seconds
   - Intrigue: 20-60 seconds
   - Reason: 40-160 seconds
   - Offer: 40-120 seconds
   - Total target: 2:40 - 8:20 (matching 500-2000 word range at spoken pace)
6. **Write output file.**

## CONSTRAINTS

1. U0's format recommendation overrides all heuristics. If U0 says video_script, it is video_script. Do not second-guess upstream strategy.
2. Word count HARD BOUNDARIES are non-negotiable: 500 minimum, 2000 maximum. No target range may extend outside these boundaries.
3. Upsell 2 word count should be visibly shorter than Upsell 1. If both are set to the same range, the position analysis is being ignored.
4. Video script format MUST include stage directions, speaker attribution, visual cues, text overlays, and timing marks. A "video script" that is just text with no visual directions is a text page mislabeled.
5. Text page format MUST include headline hierarchy, CTA button spec, and mobile-first paragraph guidance. A "text page" with no formatting structure is an unformatted draft.
6. If format cannot be determined (no U0 spec, no brief, conflicting signals), default to text_page. Text is the safer format — it can be adapted to video more easily than the reverse.
7. Timing calculations for video use 150 words/minute as the baseline speaking pace. Adjust if soul.md specifies a different pacing target.
