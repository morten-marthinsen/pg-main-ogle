# 1.2 — Variant Generator

**Version:** 1.0
**Skill:** U1 — Order Bump Writer
**Layer:** 1
**Microskill:** 1.2 — Variant Generator
**Model:** sonnet
**Depends on:** 1.1 (Initial Bump Versions)
**Produces:** `layer-1/1.2-bump-variants.md`
**Purpose:** Expand initial bump versions into 5-7 total variants with meaningfully different emphasis angles for split testing.

---

## INPUT

- `layer-1/1.1-initial-bumps.md` — 2-3 initial versions with word counts and angle labels
- `layer-0/0.2-specimen-calibration.md` — emphasis angle priority list
- `layer-0/0.1-input-context.md` — FE mechanism name, prices, congruence type

---

## PROCESS

1. **Assess initial versions from 1.1:**
   - Read all 2-3 versions
   - Note which emphasis angles have already been covered
   - Determine how many additional variants are needed (target: 5-7 total)

2. **Select emphasis angles for remaining variants:**
   Reference the emphasis angle priority from 0.2, then fill uncovered angles. The 5 core emphasis categories:

   | # | Emphasis | Lead Frame | WHY NOW Angle | Price Frame |
   |---|---------|------------|---------------|-------------|
   | 1 | **Urgency** | "This is only available right now, during checkout" | Time pressure — this deal disappears when they leave the page | "Only $X right now — not available later" |
   | 2 | **Value** | "For less than [relatable comparison], you get..." | Dollar-for-dollar savings, per-day cost | "$X/day — less than a [coffee/golf ball/etc.]" |
   | 3 | **Congruence** | "Complete your [mechanism] system" | System incompleteness — the bump makes the FE work fully | "Add [X] to make your [mechanism] complete — just $X" |
   | 4 | **Social Proof** | "[Number] of buyers add this to their order" | Others who made this choice — bandwagon, authority | "Join [X] others — just $X to add" |
   | 5 | **Exclusivity** | "Members-only / checkout-only / not sold separately" | Access and scarcity — can't get this elsewhere | "Exclusive to new [FE product] members — just $X" |

   **Additional emphasis angles** (use if 5 core categories don't produce enough differentiation or if template type suggests them):

   | # | Emphasis | When to Use |
   |---|---------|-------------|
   | 6 | **Simplicity** | Product simplifies or eliminates a step. "One-click add — no setup required." |
   | 7 | **Results** | Product has a concrete, quantifiable outcome. "Add [X] and [specific result]." |

3. **Generate each remaining variant:**
   For each new variant:
   - Open with the emphasis-specific lead frame
   - Apply the 3-element structure: WHAT + WHY NOW + PRICE
   - Include FE mechanism name verbatim
   - Include price anchor + discount price
   - End with binary choice language
   - COUNT WORDS — must be 50-150

   **Differentiation rules:**
   - The opening sentence MUST be different from all other variants
   - The WHY NOW reason MUST be different from all other variants
   - The price framing CAN be similar (limited ways to show price) but should vary where possible (percentage off, dollar savings, per-day cost, relative to FE value)
   - If two variants feel interchangeable after writing, delete one and write a new angle

4. **Carry forward initial versions:**
   - Versions A and B (and C if generated) from 1.1 carry forward as-is
   - They become the first 2-3 entries in the full variant set
   - They are NOT rewritten — they were already generated to spec
   - If a 1.1 version overlaps with a new variant's angle, note the overlap and ensure the new variant differentiates meaningfully

5. **Variant labeling:**
   - Each variant gets a clear label: "Urgency," "Value," "Congruence," "Social Proof," "Exclusivity," "Simplicity," or "Results"
   - Labels match the emphasis angle, not a marketing spin
   - Two variants cannot share the same label (if both are value-focused, one must differentiate further)

6. **Total variant count check:**
   - Minimum: 5 variants
   - Target: 6-7 variants
   - Maximum: 7 variants
   - If <5 after generation + carryforward, write additional variants
   - If >7, select the 7 most differentiated

7. **Word count sweep:**
   - Count every word in every variant (including carried-forward ones from 1.1)
   - Any variant outside 50-150: rewrite before outputting
   - Record exact count for each

8. **Write output file**

## CONSTRAINTS

- **Every variant must be meaningfully different.** The Differentiation Matrix must show distinct values in at least 2 of 3 columns (Opening Frame, WHY NOW, Price Frame) for every variant pair. If it doesn't, the variants are too similar.
- **Do NOT generate 7 variants that are all urgency-focused with slight word changes.** The 5 emphasis categories exist for a reason. Cover at least 4 of 5.
- **Do NOT write any variant over 150 words.** Count before outputting. This is the most common failure in the entire skill.
- **Do NOT skip the proof element check.** Each variant should have zero or one proof element. If you find yourself adding "studies show" AND "50,000 customers" in the same variant, you have a proof cascade.
- **Do NOT rewrite 1.1 versions.** Carry them forward as-is unless they have a word count violation (which should have been caught in 1.1).
- **The social proof variant may use a fabricated number for the template** (e.g., "[X]% of buyers add this"). Mark it as a placeholder: "[INSERT ACTUAL %]" — the final implementation team fills in the real number. Do not invent a specific false statistic.
