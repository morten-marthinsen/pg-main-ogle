# Skill 2.3: Language Extraction

**Version:** 1.0
**Layer:** 2 (Analysis)
**Type:** Language Mining
**Dependencies:** quotes_validated.json
**Output:** language_arsenal.md

---

## PURPOSE

Extract high-frequency phrases, emotional intensity language, and metaphors/analogies used by the target audience. These are copy ingredients — exact words and phrases to use in headlines, leads, and body copy. Preserve exact phrasing; these are not to be cleaned or improved.

---

## INPUT

```yaml
input:
  quotes_validated: quotes_validated.json
```

---

## PROCESS

### Step 1: Extract High-Frequency Phrases

```
SCAN all quotes for recurring phrases:

PHRASE TYPES to extract:

Problem phrases (how they describe the problem):
  - "can't seem to..."
  - "no matter what I try..."
  - "I've tried everything..."
  - "the same thing happens every time..."

Emotion phrases (how they express feelings):
  - "it's so frustrating when..."
  - "I'm sick of..."
  - "I just want to..."
  - "if only I could..."

Outcome phrases (how they describe desired outcomes):
  - "all I want is..."
  - "I'd give anything to..."
  - "that would change everything..."

Experience phrases (how they describe experiences):
  - "I bought X and..."
  - "I tried X but..."
  - "the worst part is..."
  - "what finally worked was..."

COUNT occurrences of each phrase
RANK by frequency
INCLUDE if appears 3+ times across 2+ sources
```

### Step 2: Extract Emotional Intensity Language

```
IDENTIFY high-intensity expressions:

INTENSITY MARKERS:
  - Superlatives: worst, best, most, never, always
  - Amplifiers: completely, absolutely, totally
  - Extreme emotions: hate, love, desperate, terrified
  - Physical metaphors: killing me, makes me sick, punched in the gut

FOR each high-intensity expression:
  - Extract exact phrase
  - Note context (what triggered the intensity)
  - Count occurrences
```

### Step 3: Extract Metaphors and Analogies

```
IDENTIFY figurative language:

METAPHOR TYPES:
  - Problem metaphors: "it's like hitting a brick wall"
  - Solution metaphors: "it was like a light bulb went on"
  - Experience metaphors: "felt like throwing money away"
  - Identity metaphors: "I felt like a fraud"

FOR each metaphor/analogy:
  - Extract exact phrasing
  - Note what it describes
  - Assess vividness (how visual/memorable)
```

### Step 4: Extract Objection Language

```
IDENTIFY how they express doubt/resistance:

OBJECTION PATTERNS:
  - "Yeah but what if..."
  - "That sounds good but..."
  - "I've heard that before..."
  - "How do I know..."
  - "What's the catch..."

These are direct inputs for objection handling in copy.
```

---

## CONSTRAINTS

1. NEVER clean up or "improve" extracted language. Verbatim only.
2. NEVER include phrases with < 3 occurrences (too rare to be a pattern).
3. ALWAYS include source context — where was this phrase used?
4. NEVER fabricate phrases — must exist in quotes_validated.json.
5. ALWAYS preserve grammar errors, slang, abbreviations — that's the authenticity.

---

## PHRASE SELECTION CRITERIA

```
INCLUDE if:
  - Appears 3+ times
  - From 2+ different sources
  - Emotionally resonant (not generic)
  - Specific to this market (not universal)

EXCLUDE if:
  - Generic phrase ("I think", "I believe")
  - Only appears once
  - Platform-specific syntax that won't translate to copy
```

---

## ERROR HANDLING

| Error | Response |
|-------|----------|
| No phrases meet 3+ threshold | Lower threshold to 2, flag "limited phrase data" |
| No metaphors found | Note "No figurative language detected" |
| All phrases are generic | Flag "Market uses generic language — may need creative copy approach" |

---

## ANTI-DEGRADATION RULE REFERENCES

- **Rule 7:** Real language, not clean language — preserve exact phrasing.
- **Rule 8:** Do not synthesize during assembly — extraction produces the phrases, assembly compiles them.
