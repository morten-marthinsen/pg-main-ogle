# Skill 3.1: Handoff Assembly

**Version:** 1.0
**Layer:** 3 (Delivery)
**Type:** Document Assembly
**Dependencies:** All Phase 2-3 artifacts
**Output:** RAPID-HANDOFF-[project-code].md

---

## PURPOSE

Compile all Phase 2-3 artifacts into a single structured output document following `rapid-output-template.md`. This is an ASSEMBLY operation — no new analysis, no new synthesis, no new conclusions. If it wasn't produced in Phases 2-3, it doesn't belong in the handoff.

---

## INPUT

```yaml
input:
  # Phase 2 artifacts
  gate_status: gate_status.yaml
  quotes_validated: quotes_validated.json

  # Phase 3 artifacts
  patterns: patterns.md
  hypothesis_verdicts: hypothesis_verdicts.md
  language_arsenal: language_arsenal.md
  opportunity_signals: opportunity_signals.md

  # Config files
  rapid_intake: rapid-intake.yaml
  market_config: market_config.yaml
```

---

## PROCESS

### Step 1: Load Output Template

```
READ rapid-output-template.md

VERIFY all 10 sections are present:
1. Header & Metadata
2. Executive Summary
3. Volume Summary
4. Pattern Analysis
5. Hypothesis Verdicts (or Open Scout Findings)
6. Language Arsenal
7. Opportunity Signals
8. Scope Notice
9. Recommended Next Steps
10. Appendix: Quote Samples
```

### Step 2: Populate Header & Metadata

```
FROM rapid_intake + gate_status:

POPULATE:
  project_name: from intake
  generated_date: current date
  research_mode: "rapid"
  total_quotes: from gate_status.final_counts.total
  sources_scraped: count from scrape_log
  hypotheses_tested: count or "open scout"
  gate_status: PASS | PASS_WITH_FLAGS
```

### Step 3: Generate Executive Summary

```
COMPILE from existing artifacts (DO NOT write new analysis):

Executive Summary structure:
  1. One-sentence research scope
  2. Key findings (pull from patterns.md top patterns)
  3. Hypothesis outcomes (pull from hypothesis_verdicts.md summary table)
  4. Top opportunity signals (pull from opportunity_signals.md top 2-3)
  5. Recommendation: proceed to copy | recommend full research

WORD LIMIT: 200 words max
```

### Step 4: Insert Volume Summary

```
FROM gate_status.yaml:

CREATE table:
| Bucket | Count | Minimum | Status |
|--------|-------|---------|--------|
| Pain | X | 60 | PASS/FLAG |
| Hope | X | 50 | PASS/FLAG |
| Root Cause | X | 30 | PASS/FLAG |
| Solutions Tried | X | 20 | PASS/FLAG |
| **Total** | **X** | **150** | **X** |

ADD notes:
  - If expansion was triggered, note it
  - If any bucket flagged, explain briefly
```

### Step 5: Insert Pattern Analysis

```
FROM patterns.md:

COPY entire content into Section 4.

DO NOT modify. DO NOT summarize further.
The pattern analysis was produced in skill 2.1 and is inserted verbatim.
```

### Step 6: Insert Hypothesis Verdicts

```
FROM hypothesis_verdicts.md:

COPY entire content into Section 5.

IF mode was open_scout, rename section to "Open Scout Findings"

DO NOT modify. DO NOT add new analysis.
```

### Step 7: Insert Language Arsenal

```
FROM language_arsenal.md:

COPY entire content into Section 6.

DO NOT modify. These are copy ingredients for downstream engines.
```

### Step 8: Insert Opportunity Signals

```
FROM opportunity_signals.md:

COPY entire content into Section 7.

DO NOT modify. DO NOT add new signals.
```

### Step 9: Generate Scope Notice

```
MANDATORY section — always include:

## Scope Notice

This rapid research probe has the following limitations:

**NOT COLLECTED:**
- Competitor Mechanism bucket (requires competitive deep dive)
- Villain bucket (requires blame-frame analysis)

**NOT PERFORMED:**
- Proof Inventory (required for promise ceiling calculation)
- Layer 2.5 Synthesis (transformation pairs, WEB analysis, resonance grid)
- RSF Intelligence (expectation schema, latent resonance mapping)
- Promise Ceiling calculation (required before claim generation)

**EVIDENCE DEPTH:**
- Total quotes: [X] (vs. 1,000+ for full deep research)
- Sources scraped: [X] (vs. 15-25 for full deep research)
- Expansion rounds: [0-1] (vs. 3+ for full deep research)

This output is suitable for:
- Hypothesis validation
- Opportunity scouting
- Lightweight campaign briefs (ads, emails, organic)
- Input to full deep research (as head start)

This output is NOT sufficient for:
- Full VSL or long-form sales page
- Promise ceiling calculations
- Comprehensive mechanism development
- Multi-engine campaign builds
```

### Step 10: Generate Recommended Next Steps

```
BASED ON gate_status and findings:

IF gate_status == PASS and findings are clear:
  - Recommend proceeding to target engine (ads, emails, etc.)
  - Note which findings are strongest inputs

IF gate_status == PASS_WITH_FLAGS:
  - Note the flags
  - Recommend either:
    a) Proceed with caution for lightweight engines
    b) Upgrade to full deep research for comprehensive campaign

IF evidence is thin (< 100 quotes or multiple INCONCLUSIVE):
  - Strongly recommend full deep research
  - Do not recommend proceeding to copy generation
```

### Step 11: Compile Appendix

```
SELECT 5 representative quotes per bucket:
  - Highest engagement
  - Clearest expression of pattern
  - Source diversity

FORMAT:
## Appendix: Quote Samples

### Pain Quotes (sample)
1. "[quote]" — [source]
2. "[quote]" — [source]
...

### Hope Quotes (sample)
...

NOTE: Full quote dataset available in quotes_validated.json
```

### Step 12: Save Output

```
SAVE to: ~outputs/[project-code]/rapid-research/RAPID-HANDOFF-[project-code].md

VERIFY:
  - All 10 sections present
  - No empty sections
  - File size 30KB+ (sanity check for completeness)
```

## OUTPUT

**File:** `RAPID-HANDOFF-[project-code].md`

**Location:** `~outputs/[project-code]/rapid-research/`

**Contents:**
- Complete 10-section document
- All Phase 2-3 artifacts compiled
- Scope notice and next steps

---

## CONSTRAINTS

1. ASSEMBLY ONLY. No new analysis. No new patterns. No new signals.
2. NEVER modify content from source artifacts — copy verbatim.
3. NEVER omit the Scope Notice — downstream consumers must know the limitations.
4. NEVER claim full research was performed — this is explicitly a rapid probe.
5. ALWAYS verify file size > 30KB before completing — smaller indicates missing sections.

---

## QUALITY CHECKLIST

```
PRE-SAVE VERIFICATION:

[ ] Header metadata complete
[ ] Executive summary ≤ 200 words
[ ] Volume summary table accurate (matches gate_status.yaml)
[ ] Pattern analysis section populated (from patterns.md)
[ ] Hypothesis verdicts section populated (from hypothesis_verdicts.md)
[ ] Language arsenal section populated (from language_arsenal.md)
[ ] Opportunity signals section populated (from opportunity_signals.md)
[ ] Scope notice present with all limitations listed
[ ] Recommended next steps based on actual findings
[ ] Appendix contains sample quotes from each bucket
[ ] No new analysis generated during assembly
[ ] File path correct
```

---

## ERROR HANDLING

| Error | Response |
|-------|----------|
| Missing source artifact | HALT — cannot assemble without complete inputs |
| Artifact file empty | HALT — indicates Phase 2-3 failure |
| File size < 30KB | WARNING — review for missing sections |
| Template sections mismatch | Verify against rapid-output-template.md |

---

## ANTI-DEGRADATION RULE REFERENCES

- **Rule 8:** Do not synthesize during assembly — this is the primary enforcement point.
- **Rule 9:** Flag what you didn't do — Scope Notice is mandatory.
- **Rule 10:** Recommend upgrade when evidence is thin — Next Steps section enforces this.
