# Skill 0.1: Tool Discovery

**Version:** 1.0
**Layer:** 0 (Pre-Processing)
**Type:** Infrastructure Setup
**Dependencies:** None
**Output:** tool_config.yaml

---

## PURPOSE

Discover available MCP scraping tools (Firecrawl, Apify) and establish the tool chain for Phase 2 scraping. This skill runs ONCE at rapid research start and produces a config used by all Layer 1 scraping operations.

---

## INPUT

None. This skill queries the MCP environment directly.

---

## PROCESS

### Step 1: Query Available Tools

```
EXECUTE in order:
1. Call ToolSearch("firecrawl") or list available MCP tools
2. Call ToolSearch("apify") or list available MCP tools
3. Record which tools are available
```

### Step 2: Test Tool Availability

```
FOR each discovered tool:
  - Attempt a minimal health check (e.g., Firecrawl map on a known URL)
  - Record: tool_name, available (true/false), response_time_ms

IF Firecrawl available:
  primary_scraper: firecrawl

IF Apify available:
  fallback_scraper: apify
  reddit_scraper: apify (Firecrawl is blocked on Reddit)

IF neither available:
  fallback_scraper: manual_fetch (WebFetch tool)
  FLAG: "Limited scraping capability — expect lower quote volume"
```

### Step 3: Build Tool Chain

```
DEFINE tool_chain priority order:

FOR general_web:
  1. Firecrawl (if available)
  2. Apify rag-web-browser (if available)
  3. WebFetch (always available)

FOR reddit:
  1. Apify reddit-scraper (Reddit blocks Firecrawl)
  2. WebFetch (degraded capability)

FOR youtube_comments:
  1. Apify youtube-scraper
  2. SKIP (comments not critical for rapid research)

FOR reviews (Amazon, Trustpilot):
  1. Apify platform-specific actor
  2. Firecrawl (may have limited access)
  3. WebFetch (degraded capability)
```

### Step 4: Output Configuration

```yaml
# tool_config.yaml

discovered_tools:
  firecrawl:
    available: boolean
    response_time_ms: number
  apify:
    available: boolean
    actors_available:
      - rag-web-browser
      - reddit-scraper
      - youtube-scraper

tool_chain:
  general_web:
    - firecrawl
    - apify/rag-web-browser
    - webfetch
  reddit:
    - apify/reddit-scraper
    - webfetch
  youtube:
    - apify/youtube-scraper
  reviews:
    - apify/platform-actor
    - firecrawl
    - webfetch

scraping_budget:
  estimated_cost_usd: 3-5
  max_sources: 7

flags:
  - string (any capability warnings)
```

---

## OUTPUT

**File:** `tool_config.yaml` saved to project folder

**Contents:**
- List of available tools with health status
- Tool chain priority for each source type
- Scraping budget estimate
- Capability flags/warnings

---

## CONSTRAINTS

1. NEVER proceed to Layer 1 without at least ONE working scraper.
2. ALWAYS test tool availability before reporting as available.
3. ALWAYS flag capability limitations if primary tools unavailable.
4. NEVER exceed 30 seconds on tool discovery (timeout and proceed with available tools).
5. ALWAYS include WebFetch as final fallback — it's always available.

---

## ERROR HANDLING

| Error | Response |
|-------|----------|
| Firecrawl unavailable | Use Apify as primary, flag in config |
| Apify unavailable | Use Firecrawl + WebFetch, flag reduced Reddit capability |
| Both unavailable | Use WebFetch only, flag "severely limited scraping" |
| Tool timeout | Mark as unavailable, continue with next tool |

---

## ANTI-DEGRADATION RULE REFERENCES

- **Rule 10:** Recommend upgrade when evidence is thin — if only WebFetch available, expect thin evidence.
