# Skill 1.2: Source Validation

**Version:** 1.0
**Layer:** 1 (Scraping Infrastructure)
**Type:** Source Filtering
**Dependencies:** sources_discovered.yaml, tool_config.yaml
**Output:** sources_validated.yaml

---

## PURPOSE

Score and filter discovered sources to produce a final scrape list of 3-5 high-value sources. Consolidates source scanning, scoring, and validation from full deep research into a single pass. Removes sources unlikely to yield sufficient quotes.

---

## INPUT

```yaml
input:
  sources_discovered: sources_discovered.yaml
  tool_config: tool_config.yaml
```

---

## PROCESS

### Step 1: Quick Scan Each Source

```
FOR each source in sources_discovered:

  EXECUTE light scrape (first page only, ~100 words sample):
  - Use assigned tool from tool_config
  - Capture: page title, visible comment/reply count, sample content

  EXTRACT signals:
  - actual_comment_count: number (if visible)
  - content_recency: recent (<30 days) | moderate (30-180 days) | stale (>180 days)
  - language_match: does sample contain market terminology? yes | partial | no
  - accessibility: full | partial | blocked
```

### Step 2: Score Each Source

```
SCORING CRITERIA (0-100 scale):

quote_density_score (0-40 points):
  - 100+ comments/reviews: 40 points
  - 50-99 comments/reviews: 30 points
  - 20-49 comments/reviews: 20 points
  - 10-19 comments/reviews: 10 points
  - <10 comments/reviews: 5 points

relevance_score (0-30 points):
  - language_match == yes: 30 points
  - language_match == partial: 15 points
  - language_match == no: 0 points

recency_score (0-20 points):
  - content_recency == recent: 20 points
  - content_recency == moderate: 10 points
  - content_recency == stale: 0 points

accessibility_score (0-10 points):
  - accessibility == full: 10 points
  - accessibility == partial: 5 points
  - accessibility == blocked: 0 points

TOTAL_SCORE = quote_density + relevance + recency + accessibility

PRIORITY BOOST:
  - known_url (operator-provided): +15 points
  - tier_1 platform: +10 points
```

### Step 3: Filter and Rank

```
APPLY filters:

DISCARD sources where:
  - accessibility == blocked
  - TOTAL_SCORE < 25
  - language_match == no AND priority != high

RANK remaining sources by TOTAL_SCORE descending

SELECT top 5 sources (or all if fewer than 5 remain)

IF fewer than 3 sources remain:
  - FLAG: "Insufficient validated sources"
  - Include next-best discarded sources until 3 reached
  - Note quality concerns in output
```

### Step 4: Compile Validated List

```yaml
# sources_validated.yaml

metadata:
  validation_date: date
  sources_evaluated: number
  sources_validated: number
  sources_discarded: number

validated_sources:
  - url: string
    platform: string
    type: string
    tool: string
    score: number
    expected_quotes: number (estimate based on comment count)
    scrape_priority: 1-5 (1 = highest)
    notes: string

discarded_sources:
  - url: string
    reason: string
    score: number

quality_flags:
  - string (any concerns about source quality)
```

---

## OUTPUT

**File:** `sources_validated.yaml` saved to project folder

**Contents:**
- Ordered list of 3-5 validated sources ready for scraping
- Discarded sources with reasons (for audit trail)
- Quality flags if source pool is thin

---

## CONSTRAINTS

1. NEVER validate more than 5 sources — rapid research stays lean.
2. NEVER discard operator-provided URLs unless completely inaccessible.
3. ALWAYS preserve scrape priority order — highest score = scraped first.
4. NEVER spend more than 5 minutes on validation — one quick scan per source.
5. ALWAYS flag if fewer than 3 sources validated — GATE 0 may need intervention.

---

## ERROR HANDLING

| Error | Response |
|-------|----------|
| Source returns 404 | Discard with reason "not found" |
| Source requires login | Discard with reason "authentication required" |
| Tool fails on source | Try fallback tool, discard if all fail |
| All sources score low | Proceed with best available, flag quality concern |

---

## ANTI-DEGRADATION RULE REFERENCES

- **Rule 10:** Recommend upgrade when evidence is thin — if fewer than 3 usable sources, flag for full research.
