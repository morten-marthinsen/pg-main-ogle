# Skill 1.4: Extraction

**Version:** 1.0
**Layer:** 1 (Scraping Infrastructure)
**Type:** Quote Extraction
**Dependencies:** raw_content.json, market_config.yaml
**Output:** quotes_raw.json

---

## PURPOSE

Extract verbatim quotes from raw content and tag each to one of 4 buckets: Pain, Hope, Root Cause, Solutions Tried. Consolidates the extraction pipeline from full deep research (1.5-A through 1.5-D) into a single streamlined skill. Preserves original language exactly.

---

## INPUT

```yaml
input:
  raw_content: raw_content.json
  market_config: market_config.yaml
```

---

## PROCESS

### Step 1: Scan Content for Quote-Worthy Passages

```
FOR each content item in raw_content:
  FOR each reply/comment:

    SCAN for quote indicators:

    FIRST-PERSON MARKERS:
    - "I [verb]..." "My [noun]..." "I've..." "I can't..." "I finally..."

    EMOTIONAL MARKERS:
    - Frustration: frustrated, annoyed, sick of, tired of, hate
    - Hope: finally, breakthrough, dream, imagine, wish
    - Fear: afraid, worried, scared
    - Relief: solved, fixed, works, finally

    EXPERIENCE MARKERS:
    - "I tried..." "I bought..." "I used..." "What worked..."

    CAUSAL MARKERS:
    - "because" "the reason" "the problem is" "the real issue"

    EXTRACT each passage that contains 1+ markers
```

### Step 2: Determine Quote Boundaries

```
QUOTE BOUNDARY RULES:

START boundary:
  - Capital letter after period/newline
  - First-person pronoun
  - Emotional word opening

END boundary:
  - Period followed by topic change
  - New paragraph
  - Shift from personal to general

LENGTH CONSTRAINTS:
  - Minimum: 10 words
  - Maximum: 100 words
  - If exceeds 100: split at natural break or take most impactful section
```

### Step 3: Classify into Buckets

```
FOR each extracted quote:

CLASSIFY using keyword + context analysis:

PAIN (bucket: "pain"):
  - Expresses frustration, struggle, failure
  - Describes problems, obstacles, negative experiences
  - Keywords: frustrated, struggling, can't, failed, losing, hate, annoying, exhausted
  - Example: "I've tried everything and nothing works"

HOPE (bucket: "hope"):
  - Expresses aspiration, desire, goals
  - Describes wanted outcomes, dreams
  - Keywords: want, hope, dream, finally, imagine, if only, wish
  - Example: "I just want to hit the ball straight consistently"

ROOT_CAUSE (bucket: "root_cause"):
  - Explains WHY problems occur
  - Attribution statements, causal analysis
  - Keywords: because, reason, cause, problem is, real issue, due to
  - Example: "The real problem is that most people grip too tight"

SOLUTIONS_TRIED (bucket: "solutions_tried"):
  - Describes attempts to solve the problem
  - Reviews products, methods, alternatives
  - Keywords: tried, used, bought, tested, switched to, doesn't work
  - Example: "I tried three different swing trainers and none of them helped"

IF quote could fit multiple buckets:
  - Assign to PRIMARY bucket (strongest signal)
  - Note secondary_bucket in metadata
```

### Step 4: Assign Source Attribution

```
FOR each quote:

CAPTURE attribution:
{
  source_url: from content item
  source_platform: from content item
  author: from reply/comment
  date: from reply/comment (or "unknown" if not available)
  engagement: likes/upvotes if available
}

QUOTES WITHOUT ATTRIBUTION ARE DISCARDED.
```

### Step 5: Compile Quotes

```yaml
# quotes_raw.json

metadata:
  extraction_date: date
  content_items_processed: number
  total_quotes_extracted: number
  bucket_counts:
    pain: number
    hope: number
    root_cause: number
    solutions_tried: number

quotes: [
  {
    id: string (unique identifier)
    text: string (verbatim quote)
    bucket: pain | hope | root_cause | solutions_tried
    secondary_bucket: string | null

    source:
      url: string
      platform: string
      author: string
      date: string

    engagement: number
    word_count: number
    emotional_intensity: low | medium | high

    extraction_markers: [string] (which markers triggered extraction)
  }
]
```

## OUTPUT

**File:** `quotes_raw.json` saved to project folder

**Contents:**
- All extracted quotes with bucket classification
- Source attribution for each quote
- Extraction metadata and bucket counts

---

## CONSTRAINTS

1. NEVER paraphrase, clean, or modify quote text. Verbatim only.
2. NEVER fabricate quotes. If markers found but text is unclear, skip it.
3. NEVER discard quotes for "low quality" at this stage — volume gate handles filtering.
4. ALWAYS preserve original grammar, spelling, abbreviations, slang.
5. ALWAYS include source attribution. No attribution = no quote.
6. NEVER extract passages under 10 words.

---

## EMOTIONAL INTENSITY SCORING

```
HIGH intensity markers:
  - ALL CAPS (partial or full)
  - Exclamation marks
  - Extreme words: "never", "always", "worst", "hate"
  - Specific numbers/amounts
  - Multiple emotional markers in one quote

MEDIUM intensity markers:
  - Clear first-person frustration/hope
  - Specific details or examples
  - One emotional marker

LOW intensity markers:
  - General statements
  - Third-person observations
  - No specific details
```

---

## ERROR HANDLING

| Error | Response |
|-------|----------|
| Content item has no extractable quotes | Log, continue to next item |
| Quote boundary ambiguous | Take conservative boundary, err on shorter |
| Bucket classification unclear | Assign to strongest signal, note uncertainty |
| Duplicate quote detected | Keep first occurrence only |

---

## ANTI-DEGRADATION RULE REFERENCES

- **Rule 1:** No fabricated quotes — every quote must trace to a real source.
- **Rule 5:** Source attribution is non-negotiable.
- **Rule 7:** Real language, not clean language — preserve original text exactly.
