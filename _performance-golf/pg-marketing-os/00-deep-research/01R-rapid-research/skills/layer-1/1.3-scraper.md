# Skill 1.3: Scraper

**Version:** 1.0
**Layer:** 1 (Scraping Infrastructure)
**Type:** Content Extraction
**Dependencies:** sources_validated.yaml, tool_config.yaml
**Output:** raw_content.json

---

## PURPOSE

Scrape validated sources to extract raw content for quote extraction. This is a single parameterized scraper that replaces the 8 platform-specific scrapers in full deep research. Handles forums, Reddit, reviews, and social sources with unified logic.

---

## INPUT

```yaml
input:
  sources_validated: sources_validated.yaml
  tool_config: tool_config.yaml
```

---

## PROCESS

### Step 1: Iterate Through Sources (Priority Order)

```
FOR each source in validated_sources (sorted by scrape_priority):

  SELECT tool from source.tool
  EXECUTE scrape using appropriate method for source.type

  ON success: append to raw_content
  ON failure: attempt fallback tool, log error if all fail
```

### Step 2: Platform-Specific Scraping Logic

```
IF source.type == "forum":
  - Scrape thread content including all replies
  - Capture: author, date, content, reply_count
  - Paginate if thread spans multiple pages (up to 5 pages max)
  - Tool: Firecrawl scrape with markdown output

IF source.type == "reddit":
  - Use Apify Reddit scraper
  - Capture: post content + all comments
  - Include nested replies (up to 2 levels deep)
  - Tool: apify/reddit-scraper or apify/rag-web-browser

IF source.type == "review":
  - Scrape product reviews
  - Capture: reviewer, rating, date, review_text
  - Get up to 100 most recent reviews
  - Tool: Firecrawl or Apify (platform-specific actor if available)

IF source.type == "social":
  - Scrape post + comments/replies
  - Capture: author, date, content, engagement_metrics
  - Tool: Apify (platform-specific actor) or WebFetch

IF source.type == "video":
  - Scrape video comments (not transcript — rapid mode)
  - Capture: commenter, date, comment_text, likes
  - Get up to 200 top comments
  - Tool: Apify youtube-scraper
```

### Step 3: Content Normalization

```
FOR each scraped item:

NORMALIZE to standard format:
{
  source_url: string,
  source_type: string,
  platform: string,
  scrape_date: date,

  main_content: string (original post/thread starter),
  replies: [
    {
      author: string,
      date: string,
      content: string,
      engagement: number (likes/upvotes)
    }
  ],

  metadata: {
    total_replies: number,
    pages_scraped: number,
    scrape_duration_ms: number
  }
}
```

### Step 4: Compile Raw Content

```yaml
# raw_content.json

metadata:
  scrape_date: date
  sources_attempted: number
  sources_successful: number
  sources_failed: number
  total_content_items: number
  total_replies_captured: number

content: [
  {
    source_url: string,
    source_type: string,
    platform: string,
    main_content: string,
    replies: [...],
    metadata: {...}
  }
]

scrape_log: [
  {
    url: string,
    status: success | failed,
    tool_used: string,
    duration_ms: number,
    items_captured: number,
    error: string (if failed)
  }
]
```

---

## OUTPUT

**File:** `raw_content.json` saved to project folder

**Contents:**
- Normalized content from all scraped sources
- Main content + all replies/comments
- Scrape log with success/failure status

---

## CONSTRAINTS

1. NEVER scrape more than 5 pages per source — rapid mode stays fast.
2. NEVER exceed 10 minutes total scraping time across all sources.
3. ALWAYS use the tool chain fallback if primary tool fails.
4. NEVER scrape content requiring authentication unless credentials provided.
5. ALWAYS preserve original text — no cleaning or modification during scrape.
6. NEVER exceed 200 replies per source — sufficient for rapid research.

---

## ERROR HANDLING

| Error | Response |
|-------|----------|
| Tool timeout | Retry once, then use fallback tool |
| Rate limited | Wait 30 seconds, retry, then skip if persists |
| Source blocked | Try fallback tool, skip if all fail |
| Partial content | Accept partial, log warning |
| All sources fail | HALT — cannot proceed without content |

---

## TOOL SELECTION

```
PRIMARY TOOL SELECTION:

Firecrawl:
  - Best for: Forums, general web pages, review sites
  - Use: firecrawl_scrape with formats: ["markdown"]
  - Options: onlyMainContent: true

Apify:
  - Best for: Reddit, YouTube, social platforms
  - Use: apify/rag-web-browser for general pages
  - Use: platform-specific actors when available

WebFetch:
  - Fallback for: Any source when primary tools fail
  - Limitations: No JavaScript rendering, may miss dynamic content
```

---

## ANTI-DEGRADATION RULE REFERENCES

- **Rule 5:** Source attribution is non-negotiable — every content item must have source_url.
- **Rule 7:** Real language, not clean language — preserve original text exactly.
