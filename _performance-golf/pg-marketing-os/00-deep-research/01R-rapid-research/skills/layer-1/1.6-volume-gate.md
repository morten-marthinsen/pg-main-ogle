# Skill 1.6: Volume Gate

**Version:** 1.0
**Layer:** 1 (Scraping Infrastructure)
**Type:** Quality Gate
**Dependencies:** quotes_validated.json, market_config.yaml
**Output:** gate_status.yaml, expansion_log.md (if expansion triggered)

---

## PURPOSE

Check quote counts against minimums (Pain 60, Hope 50, Root Cause 30, Solutions Tried 20, Total 150). If below minimum, generate expansion queries and trigger ONE expansion round. This is GATE 1 from the rapid research pipeline.

---

## INPUT

```yaml
input:
  quotes_validated: quotes_validated.json
  market_config: market_config.yaml
```

---

## PROCESS

### Step 1: Count Quotes by Bucket

```
FROM quotes_validated:

COUNT:
  pain_count = quotes where bucket == "pain"
  hope_count = quotes where bucket == "hope"
  root_cause_count = quotes where bucket == "root_cause"
  solutions_tried_count = quotes where bucket == "solutions_tried"
  total_count = all quotes

MINIMUMS:
  pain_minimum = 60
  hope_minimum = 50
  root_cause_minimum = 30
  solutions_tried_minimum = 20
  total_minimum = 150
```

### Step 2: Evaluate Gate Status

```
GATE 1 EVALUATION:

CHECK each bucket:
  pain_pass = pain_count >= 60
  hope_pass = hope_count >= 50
  root_cause_pass = root_cause_count >= 30
  solutions_tried_pass = solutions_tried_count >= 20
  total_pass = total_count >= 150

GATE_STATUS:
  IF all checks PASS:
    status = "PASS"
    action = "proceed_to_layer_2"

  ELSE:
    status = "FAIL"
    action = "expand"
    deficit_buckets = [list of failing buckets with counts vs minimums]
```

### Step 3: Generate Expansion Queries (If FAIL)

```
IF GATE_STATUS == "FAIL":

  FOR each deficit_bucket:

    GENERATE 2-3 targeted queries:

    IF deficit == "pain":
      - "[market] frustrating reddit"
      - "[market] struggling forum"
      - "[market] problems reviews"

    IF deficit == "hope":
      - "[market] success stories"
      - "[market] finally worked"
      - "[market] transformation"

    IF deficit == "root_cause":
      - "why [problem] happens"
      - "real cause of [problem]"
      - "[problem] because"

    IF deficit == "solutions_tried":
      - "[product category] review"
      - "I tried [alternative]"
      - "[market] what didn't work"

  OUTPUT expansion_queries targeting deficit buckets
```

### Step 4: Execute Expansion (ONE Round Only)

```
IF expansion triggered:

  1. Execute expansion queries (skill 1.1 logic)
  2. Discover 1-3 additional sources
  3. Validate sources (skill 1.2 logic)
  4. Scrape new sources (skill 1.3 logic)
  5. Extract quotes (skill 1.4 logic)
  6. Validate authenticity (skill 1.5 logic)
  7. Merge new quotes with existing quotes_validated.json
  8. Re-count buckets

  LOG all expansion activity to expansion_log.md
```

### Step 5: Final Gate Evaluation

```
AFTER expansion (or if no expansion needed):

RE-EVALUATE gate:

IF all minimums met:
  final_status = "PASS"

ELSE:
  final_status = "PASS_WITH_FLAGS"
  quality_flags = [
    "Bucket [X] below minimum: [actual]/[required]"
    for each still-failing bucket
  ]
  recommendation = "Full deep research recommended for comprehensive evidence"

DO NOT LOOP AGAIN. One expansion round is the maximum.
```

### Step 6: Output Gate Status

```yaml
# gate_status.yaml

gate: "GATE_1"
evaluation_date: date

initial_counts:
  pain: number
  hope: number
  root_cause: number
  solutions_tried: number
  total: number

initial_status: PASS | FAIL

expansion_triggered: boolean
expansion_queries: [array of queries if triggered]
expansion_sources_added: number
expansion_quotes_added: number

final_counts:
  pain: number
  hope: number
  root_cause: number
  solutions_tried: number
  total: number

final_status: PASS | PASS_WITH_FLAGS
quality_flags: [array of flag strings]
recommendation: string

proceed_to: "layer_2"
```

---

## OUTPUT

**Files:**
- `gate_status.yaml` — Gate evaluation results
- `expansion_log.md` — Detailed expansion activity (if triggered)
- Updated `quotes_validated.json` — Merged quotes (if expansion added new quotes)

---

## CONSTRAINTS

1. NEVER loop more than once. ONE expansion round maximum.
2. NEVER lower minimums. If not met after expansion, flag and proceed.
3. ALWAYS generate queries for ALL deficit buckets, not just the worst one.
4. NEVER skip the gate check — even if expansion not triggered, record status.
5. ALWAYS produce gate_status.yaml regardless of outcome.

---

## EXPANSION BUDGET

```
Expansion constraints:
  - Max new sources: 3
  - Max new queries: 6-9 (2-3 per deficit bucket)
  - Max time: 15 minutes
  - Max additional scraping cost: $2

If expansion would exceed budget, prioritize:
  1. Largest deficit bucket
  2. Highest-density expected sources
```

---

## ERROR HANDLING

| Error | Response |
|-------|----------|
| Expansion yields no new sources | Log, proceed with PASS_WITH_FLAGS |
| Expansion scraping fails | Log, proceed with existing quotes |
| Total < 50 after expansion | HALT — insufficient for even rapid research |
| Single bucket at 0 | Major flag, recommend full research |

---

## ANTI-DEGRADATION RULE REFERENCES

- **Rule 2:** No rationalized shortcuts — if minimums not met, flag explicitly.
- **Rule 3:** Bucket minimums are real — counts are stated clearly.
- **Rule 4:** One expansion round is mandatory if needed — this skill enforces it.
- **Rule 10:** Recommend upgrade when evidence is thin.
