# Skill 1.5: Authenticity Check

**Version:** 1.0
**Layer:** 1 (Scraping Infrastructure)
**Type:** Quality Validation
**Dependencies:** quotes_raw.json
**Output:** quotes_validated.json

---

## PURPOSE

Validate that extracted quotes are real (not fabricated, not over-cleaned, not hallucinated). This is a critical guard against the most common failure mode in AI-assisted research: generating plausible-sounding quotes that never existed. Adapted from full deep research's 0.3-authenticity-validator.

---

## INPUT

```yaml
input:
  quotes_raw: quotes_raw.json
```

---

## PROCESS

### Step 1: Apply Authenticity Filters

```
FOR each quote in quotes_raw:

  CHECK authenticity signals:

  PASS signals (quote is likely real):
  - Has source URL and author
  - Contains messy human language (typos, slang, abbreviations)
  - Includes specific details (numbers, names, timeframes)
  - Grammar is imperfect (run-ons, fragments)
  - Contains platform-specific markers (Reddit formatting, emoji)

  FAIL signals (quote may be fabricated):
  - Reads like polished marketing copy
  - Perfect grammar and structure
  - Generic without specific details
  - Contains unlikely statistical claims without context
  - Uses AI-typical phrasing ("comprehensive", "leverage", "optimize")
  - Too perfectly fits a bucket category
```

### Step 2: Flag Suspicious Quotes

```
SUSPICIOUS if 2+ FAIL signals present:

FLAG with:
{
  flagged: true,
  flag_reason: string (which signals triggered)
  confidence: low | medium
  action: review | discard
}

DISCARD if:
- No source URL
- 3+ FAIL signals
- Exact duplicate of another quote
- Less than 10 words

RETAIN but FLAG if:
- 2 FAIL signals (may still be real, just polished)
- Source exists but unverifiable
```

### Step 3: Check for Over-Cleaning

```
OVER-CLEANING indicators:

COMPARE quote characteristics to expected source patterns:

Reddit quotes SHOULD have:
  - Casual language, abbreviations
  - Possible profanity or slang
  - "lol", "tbh", "imo" style markers

Forum quotes SHOULD have:
  - Longer, more detailed passages
  - Technical terminology
  - Reference to other posts

Review quotes SHOULD have:
  - Star rating context
  - Product-specific mentions
  - Comparative language

IF quote is too clean for its source type:
  - Flag as "possibly_over_cleaned"
  - Do not discard — may still be real
```

### Step 4: Deduplicate

```
CHECK for exact and near duplicates:

EXACT duplicate: same text, any source
  → Keep first occurrence, discard rest

NEAR duplicate: >80% text overlap
  → Keep quote with better source attribution
  → Discard others
```

### Step 5: Compile Validated Quotes

```yaml
# quotes_validated.json

metadata:
  validation_date: date
  quotes_input: number
  quotes_validated: number
  quotes_flagged: number
  quotes_discarded: number

  discard_reasons:
    no_source: number
    suspected_fabrication: number
    too_short: number
    duplicate: number

validated_quotes: [
  {
    # All fields from quotes_raw, plus:
    authenticity:
      status: validated | flagged
      confidence: high | medium | low
      flags: [string] (empty if validated)
  }
]

flagged_quotes: [
  {
    # Quotes that passed but have concerns
    quote: {...}
    flag_reason: string
    recommendation: review | use_with_caution
  }
]

discarded_quotes: [
  {
    quote_text: string (first 50 chars)
    reason: string
  }
]
```

---

## OUTPUT

**File:** `quotes_validated.json` saved to project folder

**Contents:**
- Validated quotes with authenticity status
- Flagged quotes (use with caution)
- Discarded quotes with reasons

---

## CONSTRAINTS

1. NEVER pass a quote without source attribution — no exceptions.
2. NEVER "clean up" quotes during validation — preserve original text.
3. ALWAYS flag rather than discard when uncertain — err on inclusion.
4. NEVER add quotes that weren't in quotes_raw.json — validation is filtering, not creation.
5. ALWAYS preserve engagement scores and emotional intensity from extraction.

---

## AUTHENTICITY PATTERNS

```
REAL quote characteristics:
- "I dunno why but every time I try to..."
- "ive been doing this for 3 years and its still not working lol"
- "Tbh the [specific product] was a waste of $200"
- "I remember when my dad told me..."
- Fragments: "Worst purchase ever. Complete waste."

FABRICATED quote characteristics:
- "I have found that implementing a consistent routine has yielded positive results"
- "The comprehensive approach to addressing my challenges proved effective"
- Perfect subject-verb agreement, no contractions
- Unnaturally balanced sentence structure
```

---

## ERROR HANDLING

| Error | Response |
|-------|----------|
| All quotes flagged | Proceed with flags, recommend source review |
| >50% quotes discarded | FLAG major quality concern, check source validity |
| Duplicate source for different quotes | Normal, do not discard |

---

## ANTI-DEGRADATION RULE REFERENCES

- **Rule 1:** No fabricated quotes — this skill is the primary enforcement.
- **Rule 5:** Source attribution is non-negotiable.
- **Rule 7:** Real language, not clean language — flag over-cleaned quotes.
