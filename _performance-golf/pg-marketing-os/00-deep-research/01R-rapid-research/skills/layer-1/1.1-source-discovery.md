# Skill 1.1: Source Discovery

**Version:** 1.0
**Layer:** 1 (Scraping Infrastructure)
**Type:** Source Identification
**Dependencies:** market_config.yaml, tool_config.yaml
**Output:** sources_discovered.yaml

---

## PURPOSE

Identify 3-7 high-value sources for quote extraction. Combines platform identification and query generation from full deep research into a single streamlined skill. Sources are URLs or platform locations where the target audience discusses their problems, hopes, and experiences.

---

## INPUT

```yaml
input:
  market_config: market_config.yaml
  tool_config: tool_config.yaml
```

---

## PROCESS

### Step 1: Seed from Known URLs

```
FROM market_config.known_urls:

FOR each URL:
  - Add to sources_discovered
  - Mark as priority: high (operator-provided)
  - Set expected_density: unknown (will be validated in 1.2)
```

### Step 2: Search for Additional Sources

```
USING queries from market_config:

FOR each query in pain_queries + hope_queries (first 3 of each):
  - Execute web search (Firecrawl search or Apify rag-web-browser)
  - Collect top 3-5 results per query
  - Filter to platforms in market_config.platforms.tier_1

DEDUPLICATE results by domain/thread
```

### Step 3: Platform-Specific Discovery

```
FOR each platform in market_config.platforms.tier_1:

  IF platform.type == "forum":
    - Search for threads matching terminology
    - Look for "megathreads" or popular posts
    - Target: 2-3 high-engagement threads per forum

  IF platform.type == "reddit":
    - Search subreddits for relevant posts
    - Sort by: top (month) or hot
    - Target: 3-5 posts per subreddit

  IF platform.type == "review":
    - Find product category pages with reviews
    - Target competitor products or alternatives
    - Target: 1-2 product pages with 50+ reviews

  IF platform.type == "social":
    - Search hashtags or topics
    - Target: 1-2 threads with high engagement
```

### Step 4: Estimate Quote Density

```
FOR each discovered source:

  ESTIMATE expected_quote_density:

  HIGH (30+ expected quotes):
    - Forum thread with 50+ replies
    - Reddit post with 100+ comments
    - Product page with 100+ reviews
    - Facebook group thread with 20+ comments

  MEDIUM (10-30 expected quotes):
    - Forum thread with 10-50 replies
    - Reddit post with 20-100 comments
    - Product page with 20-100 reviews

  LOW (1-10 expected quotes):
    - Blog post with comments
    - Small forum thread
    - Single social media post
```

### Step 5: Compile Source List

```yaml
# sources_discovered.yaml

metadata:
  discovery_date: date
  queries_executed: number
  sources_found: number

sources:
  - url: string
    platform: string
    type: forum | reddit | review | social | video | other
    tool: firecrawl | apify | webfetch
    priority: high | medium | low
    expected_density: high | medium | low
    discovery_method: known_url | search | platform_browse
    notes: string (any relevant context)
```

---

## OUTPUT

**File:** `sources_discovered.yaml` saved to project folder

**Contents:**
- List of 5-10 candidate sources (will be filtered to 3-5 in skill 1.2)
- Platform and tool assignments
- Expected quote density estimates

---

## CONSTRAINTS

1. NEVER discover more than 10 sources — rapid research stays focused.
2. ALWAYS include all known_urls from intake — operator-provided sources are high priority.
3. NEVER spend more than 10 minutes on discovery — this is rapid mode.
4. ALWAYS assign a tool from tool_config.yaml to each source.
5. NEVER include sources that require login/authentication (unless operator provides credentials).

---

## ERROR HANDLING

| Error | Response |
|-------|----------|
| Search returns no results | Broaden query, try tier_2 platforms |
| Platform blocked by tool | Assign fallback tool from tool_chain |
| Fewer than 3 sources found | Flag, proceed with available, recommend more URLs |
| Known URL returns 404 | Remove from list, flag in notes |

---

## ANTI-DEGRADATION RULE REFERENCES

- **Rule 6:** Do not bias scraping toward hypotheses — discover broadly across all query types.
