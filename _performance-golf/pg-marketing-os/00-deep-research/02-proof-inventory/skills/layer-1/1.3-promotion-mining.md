# 1.3 Promotion Mining

## Purpose
Extract proof elements from existing sales copy (VSLs, sales letters, email sequences). Previous promotions are rich multi-category sources where proof has already been curated and positioned—mine this systematically.

---

## Input Requirements

From 1.1 Source Parsing:
- Parsed promotion segments
- Copy section markers identified
- Existing proof patterns flagged

---

## Promotion Proof Density Map

Different sections of sales copy contain different proof types:

```
COPY SECTION → EXPECTED PROOF

HEADLINE/SUBHEAD:
├── DATA (shocking statistics)
├── SOCIAL (user counts, bestseller status)
└── DEMONSTRATION (result claims)

LEAD SECTION:
├── SOCIAL (story testimonials)
├── DEMONSTRATION (before/after)
├── DATA (problem statistics)
└── AUTHORITY (endorsements opening)

PROBLEM AGITATION:
├── DATA (scope of problem)
├── AUTHORITY (expert quotes on problem)
└── SOCIAL (shared frustration)

SOLUTION REVEAL:
├── MECHANISM (how it works)
├── AUTHORITY (research backing)
└── DEMONSTRATION (initial proof)

MECHANISM SECTION:
├── MECHANISM (scientific explanation)
├── AUTHORITY (studies, credentials)
├── DATA (mechanism statistics)
└── DEMONSTRATION (mechanism visualization)

PROOF STACK:
├── SOCIAL (testimonial clusters)
├── DATA (aggregated results)
├── AUTHORITY (validation, certifications)
├── DEMONSTRATION (before/after gallery)
└── MECHANISM (mechanism proof)

OFFER SECTION:
├── RISK REVERSAL (guarantees)
├── DATA (value stacking numbers)
└── SOCIAL (urgency proof)

CLOSE:
├── SOCIAL (final testimonials)
├── DATA (success rates)
├── RISK REVERSAL (guarantee reminder)
└── AUTHORITY (final credibility)
```

---

## Execution Protocol

### Step 1: Section Identification

Parse promotion into copy sections:

```
SECTION MARKERS:

HEADLINE: First major text, largest font, often a claim
SUBHEAD: Supporting headline, usually under main headline
LEAD: Opening paragraphs, story beginning, hook
PROBLEM: "But here's the problem...", pain agitation
SOLUTION: "Introducing...", "The answer is...", reveal moment
MECHANISM: "Here's how it works...", "The secret is..."
PROOF: "Don't just take my word...", testimonial sections
OFFER: "Here's everything you get...", stack section
GUARANTEE: "100% money-back...", risk reversal section
CLOSE: Final CTA, "Click below...", urgency
```

### Step 2: Signal Word Scanning

Scan each section for proof signals:

```
SOCIAL SIGNALS:
├── Quoted text in quotation marks
├── Attribution patterns (- Name, from [Location])
├── "customers say", "users report"
├── Star ratings, review mentions
├── "Join X,000+ customers"
└── Testimonial headers

AUTHORITY SIGNALS:
├── "Study", "research", "published"
├── "Dr.", "PhD", credentials
├── "university", institution names
├── "As seen in", media mentions
├── "Certified", "approved", "endorsed"
└── Patent numbers

DEMONSTRATION SIGNALS:
├── "Watch", "see", "look at"
├── "Before/after", "side by side"
├── "Here's proof", "let me show you"
├── Image/video references
├── "[PHOTO]", "[VIDEO]" tags
└── Result showcases

MECHANISM SIGNALS:
├── "Here's how it works"
├── "The reason is", "The secret is"
├── Scientific term explanations
├── Process descriptions
├── "Because", "which causes", "leading to"
└── "The key is", "the difference is"

DATA SIGNALS:
├── Numbers, percentages
├── Dollar amounts
├── Timeframes (days, weeks, months)
├── "X out of Y", success rates
├── "Average", "typical results"
└── Statistics with sources

RISK REVERSAL SIGNALS:
├── "Guarantee", "money back"
├── "Risk-free", "no risk"
├── "X-day trial"
├── "If you're not satisfied"
├── "100%", "full refund"
└── "Keep the bonus"
```

### Step 3: Element Extraction by Signal

For each signal detected:

```
EXTRACTION PROCESS:
1. Capture surrounding context (2-3 sentences)
2. Identify the proof claim
3. Note position in copy (section, approximate %)
4. Classify primary category
5. Extract supporting details
6. Flag for verification
```

**Headline Example:**
```
Source: "How 23,847 Golfers Fixed Their Slice in 14 Days
        Without Changing Their Swing"

Extracted Elements:
  Element 1:
    category: SOCIAL
    sub_type: user_count
    content: "23,847 golfers"
    copy_position: headline

  Element 2:
    category: DATA
    sub_type: data_exact_statistic
    content: "14 days"
    copy_position: headline

  Element 3:
    category: DATA
    sub_type: data_time_figures
    content: "14 days"
    copy_position: headline
```

### Step 4: Testimonial Block Mining

Previous promotions often have multiple testimonials. Extract each separately:

```
TESTIMONIAL BLOCK PATTERN:
"[Quote text...]"
- Name, Location
[Result if stated separately]

EXTRACT:
1. Quote content
2. Attribution
3. Specific results (embedded or stated)
4. Position in copy (lead testimonial vs. proof stack)
5. Media indicators (video, audio, written)
```

### Step 5: Study/Research Citation Extraction

Promotions often cite research. Capture the citation details:

```
RESEARCH CITATION PATTERN:
"A study at [University] found that [finding]..."
"Published in [Journal], researchers discovered..."
"According to [Source], X% of..."

EXTRACT:
  citation:
    source: "[University/Journal/Institution]"
    finding: "[The claim]"
    statistic: "[Any numbers]"
    year: "[If stated]"

  NOTE: Many promotion citations are incomplete—flag for
  verification in Discovery layer
```

### Step 6: Position-Aware Extraction

Track WHERE each proof element appeared:

```yaml
position_metadata:
  section: "proof_stack"
  approximate_position: 65%  # of total copy
  context: "third testimonial in proof section"
  prominence: "high"  # featured vs. buried
```

This matters for ranking—knockout proof often appears in lead or close.

## Quality Gates

Before promotion mining complete:

- [ ] All major copy sections scanned
- [ ] Testimonials extracted individually (not merged)
- [ ] Citations flagged for verification
- [ ] Position metadata captured
- [ ] All six proof categories checked for
- [ ] Duplicate claims deduplicated

---

## Common Mining Errors

**Error: Missing Embedded Statistics**
```
BAD: Extract only the testimonial, miss the stat in headline
GOOD: Headlines often contain the strongest DATA proof
```

**Error: Incomplete Citations**
```
BAD: "A study showed..." (no source)
GOOD: Flag incomplete citations, attempt to find source
```

**Error: Ignoring Position**
```
BAD: Extract elements without noting where they appeared
GOOD: Position = ranking signal (lead proof is usually strongest)
```

## Handoff

Pass Extracted Promotion Elements to:
- **1.5 Category Classification** (for verification)
- **1.6 Sub-Type Matching** (for refinement)
- **1.7 Element Scoring** (for full scoring)

Pass Position Summary to Layer 4 for ranking context.

Pass Citation Verification List to Layer 3 for Discovery operation.
