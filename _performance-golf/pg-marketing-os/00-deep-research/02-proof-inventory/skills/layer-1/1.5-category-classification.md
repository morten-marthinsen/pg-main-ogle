# 1.5 Category Classification

## Purpose
Assign each extracted proof element to one of the 6 primary categories. This is the first-pass classification that determines which sub-type matching protocol to apply next.

---

## Input Requirements

From extraction microskills (1.2, 1.3, 1.4):
- Extracted proof elements
- Preliminary category guesses
- Raw text and context

---

## The 6 Categories

```
PROOF CATEGORIES:

1. SOCIAL       → Other people's experiences/validation
2. AUTHORITY    → Credentials, studies, expert backing
3. DEMONSTRATION → Visual/experiential proof showing results
4. MECHANISM    → Proof explaining WHY/HOW it works
5. DATA         → Numbers, statistics, quantified claims
6. RISK_REVERSAL → Guarantees, trials, risk reduction
```

---

## Primary Classification Question

For each element, ask ONE question:

**"What is this proof primarily DOING for the reader?"**

| If it's... | Category |
|-----------|----------|
| Showing what other people experienced | SOCIAL |
| Establishing credibility through expertise/research | AUTHORITY |
| Showing the product/result in action | DEMONSTRATION |
| Explaining WHY or HOW something works | MECHANISM |
| Providing numbers or statistics | DATA |
| Reducing purchase risk | RISK_REVERSAL |

---

## Execution Protocol

### Step 1: Apply Decision Tree

```
START: Read the proof element

Q1: Does it quote/describe another person's experience?
    YES → SOCIAL
    NO → Continue

Q2: Does it cite credentials, studies, or expert authority?
    YES → Likely AUTHORITY (but check Q3 first)
    NO → Continue

Q3: Does it explain WHY or HOW something works scientifically?
    YES → MECHANISM (even if study-based)
    NO → Continue

Q4: Does it SHOW results visually/experientially?
    YES → DEMONSTRATION
    NO → Continue

Q5: Is it primarily a number, statistic, or quantified claim?
    YES → DATA
    NO → Continue

Q6: Does it reduce purchase risk (guarantees, trials)?
    YES → RISK_REVERSAL
    NO → Re-evaluate or flag for review
```

### Step 2: Handle Boundary Cases

**AUTHORITY vs. MECHANISM:**
```
"A Harvard study found that the compound works by
 blocking alpha-glucosidase enzymes..."

PRIMARY FUNCTION:
├── If focus is "Harvard study found" → AUTHORITY
├── If focus is "works by blocking enzymes" → MECHANISM
└── SPLIT if both prominent → Create 2 elements

RULE: Studies proving THAT something works = AUTHORITY
      Studies proving HOW/WHY something works = MECHANISM
```

**SOCIAL vs. DATA:**
```
"97% of our customers report being satisfied"

PRIMARY FUNCTION:
├── If emphasis is on "customers report" → SOCIAL
├── If emphasis is on "97%" → DATA
└── Context matters: customer voice = SOCIAL, stat = DATA
```

**DEMONSTRATION vs. MECHANISM:**
```
"Watch as the formula dissolves fat cells in real time"

PRIMARY FUNCTION:
├── If showing THAT it works → DEMONSTRATION
├── If explaining HOW it dissolves cells → MECHANISM
└── Demonstrations SHOW, mechanisms EXPLAIN
```

**AUTHORITY vs. SOCIAL (Celebrity):**
```
"Dr. Oz recommends this product"

PRIMARY FUNCTION:
├── If Dr. Oz is lending MEDICAL expertise → AUTHORITY (endorsement_expert)
├── If Dr. Oz is lending FAME (like any celebrity) → SOCIAL (testimonial_celebrity)
└── Does the credential matter or just the name?
```

### Step 3: Confidence Scoring

Assign classification confidence:

```
CONFIDENCE LEVELS:

HIGH (0.9+): Clear single-category fit
├── Pure testimonial quote → SOCIAL
├── Pure statistic → DATA
├── Pure guarantee → RISK_REVERSAL

MEDIUM (0.7-0.89): Mostly clear with minor ambiguity
├── Study citation that also has statistics
├── Testimonial that includes specific numbers

LOW (0.5-0.69): Ambiguous, could fit multiple
├── Expert quote that's also explaining mechanism
├── Demonstration that's proving mechanism

VERY LOW (<0.5): Flag for review
├── Can't determine primary function
├── Multiple categories equally valid
```

### Step 4: Multi-Category Elements

Some elements legitimately span categories. Handle by:

```
OPTION 1: Assign Primary
└── Choose dominant function, note secondary

OPTION 2: Split into Multiple Elements
└── If both functions are prominent, create 2 elements
└── Each element gets its own category

OPTION 3: Flag for Review
└── If truly unclear, flag for human classification
```

**Example Split:**
```
Original: "A Harvard study (n=500) found 73% improvement
          because the compound inhibits cortisol production"

Split into:
  Element A:
    category: AUTHORITY
    content: "Harvard study (n=500) found 73% improvement"

  Element B:
    category: MECHANISM
    content: "The compound inhibits cortisol production"

  Element C:
    category: DATA
    content: "73% improvement"

NOTE: All three reference same source, share citation
```

## Category Quick Reference

| Category | Key Signals | NOT This If... |
|----------|-------------|----------------|
| SOCIAL | "I", "customer", "user", quote marks | It's explaining how something works |
| AUTHORITY | "study", "Dr.", "university", "certified" | Focus is on mechanism, not credential |
| DEMONSTRATION | "watch", "see", "before/after", visual | It's explaining why it works |
| MECHANISM | "because", "how it works", "the reason" | It's just showing results |
| DATA | Numbers, %, statistics, "$" | Number is inside testimonial |
| RISK_REVERSAL | "guarantee", "money back", "risk-free" | It's proof of results |

---

## Quality Gates

Before classification complete:

- [ ] Every element has primary category assigned
- [ ] Confidence score assigned (0-1)
- [ ] Boundary cases explicitly resolved
- [ ] Multi-category elements split or primary chosen
- [ ] Ambiguous elements flagged for review

---

## Common Classification Errors

**Error: Defaulting to SOCIAL**
```
BAD: Anything with a name → SOCIAL
GOOD: Expert quotes may be AUTHORITY or MECHANISM
```

**Error: Confusing AUTHORITY and MECHANISM**
```
BAD: Any study → AUTHORITY
GOOD: If study explains HOW/WHY → MECHANISM
```

**Error: Overlooking DATA in context**
```
BAD: "Lost 30 pounds" in testimonial → SOCIAL only
GOOD: Extract the "30 pounds" as separate DATA element
```
