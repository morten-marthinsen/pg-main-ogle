# 1.1 Source Parsing Protocol

## Purpose
Parse and prepare different source material types for proof extraction. Each source type has distinct structure and proof density patterns that require different parsing approaches.

---

## Input Requirements

From Layer 0 Validation:
- Validated source file list
- Source type classification
- File existence confirmation

---

## Source Type Identification

### Automatic Classification

```
FILE EXTENSION MAPPING:
├── .txt, .md → Text-based (testimonials, product materials)
├── .pdf → Document (studies, reports, previous promotions)
├── .json → Structured (extracted data, previous skill outputs)
├── .vtt, .srt → Transcript (video/audio transcripts)
└── .html → Web (sales pages, landing pages)

CONTENT PATTERN MAPPING:
├── Contains "testimonial", "review", customer names → TESTIMONIAL_FILE
├── Contains "abstract", "methods", "results", "conclusion" → STUDY_DOCUMENT
├── Contains "headline", "subhead", "call to action" → PREVIOUS_PROMOTION
├── Contains "ingredients", "specifications", "features" → PRODUCT_MATERIALS
└── Contains timestamps, speaker labels → VIDEO_TRANSCRIPT
```

---

## Execution Protocol

### Step 1: Load and Normalize

```
FOR each source_file IN validated_sources:
    1. Read file contents
    2. Detect encoding (UTF-8, UTF-16, ASCII)
    3. Normalize line endings
    4. Strip excessive whitespace while preserving structure
    5. Classify source type (auto + manual override)
```

### Step 2: Segment by Source Type

#### Testimonial Files

```
TESTIMONIAL PARSING:
├── Look for DELIMITERS:
│   ├── Quoted blocks ("...", '...', «...»)
│   ├── Attribution patterns (- Name, -- Name, Name says)
│   ├── Section headers ("What Our Customers Say")
│   └── Star ratings (★★★★★, 5/5, 4.8 stars)
│
├── EXTRACT per testimonial:
│   ├── quote_text: The testimonial content
│   ├── attribution: Name, location, credentials
│   ├── rating: If present
│   ├── date: If present
│   └── media_type: written | video | audio | screenshot
│
└── GROUP similar testimonials (same outcome, same problem)
```

#### Study Documents

```
STUDY PARSING:
├── IDENTIFY SECTIONS:
│   ├── Abstract/Summary → Key claims
│   ├── Methods → Sample size, methodology
│   ├── Results → Statistics, findings
│   ├── Discussion → Interpretations
│   └── References → Citation info
│
├── EXTRACT per study:
│   ├── title: Study title
│   ├── authors: With credentials
│   ├── publication: Journal, date
│   ├── methodology: Study type, sample size
│   ├── key_findings: List of results
│   └── statistics: All numbers with context
│
└── FLAG: Peer-reviewed vs. company-funded vs. independent
```

#### Previous Promotions

```
PROMOTION PARSING:
├── IDENTIFY COPY SECTIONS:
│   ├── Headlines → Often contain DATA claims
│   ├── Lead → SOCIAL, DEMONSTRATION proof
│   ├── Story → SOCIAL proof narratives
│   ├── Mechanism → MECHANISM, AUTHORITY proof
│   ├── Proof Stack → All categories
│   ├── Offer → RISK REVERSAL
│   └── Close → SOCIAL, DATA proof
│
├── EXTRACT by signal:
│   ├── Quoted text → Testimonials
│   ├── Numbers/percentages → DATA
│   ├── "Study shows", "Research proves" → AUTHORITY
│   ├── "Guarantee", "Money back" → RISK REVERSAL
│   └── "Watch", "See", "Before/After" → DEMONSTRATION
│
└── PRESERVE copy position metadata for ranking
```

#### Product Materials

```
PRODUCT PARSING:
├── IDENTIFY SECTIONS:
│   ├── Ingredient/Component lists
│   ├── Feature specifications
│   ├── Manufacturing/quality info
│   ├── Certifications/badges
│   └── Guarantee terms
│
├── EXTRACT:
│   ├── ingredient_research: Studies on components
│   ├── specifications: Quantifiable attributes
│   ├── certifications: GMP, ISO, FDA status
│   ├── guarantee_terms: Duration, conditions
│   └── manufacturing_claims: Quality indicators
│
└── CROSS-REFERENCE: Ingredient names against known research
```

#### Video Transcripts

```
TRANSCRIPT PARSING:
├── IDENTIFY SECTIONS:
│   ├── Testimonial segments (customer speaking)
│   ├── Expert segments (authority speaking)
│   ├── Demonstration segments (showing results)
│   └── Claim segments (promises, data)
│
├── EXTRACT:
│   ├── speaker: Who is talking
│   ├── timestamp: When in video
│   ├── content: What they say
│   ├── context: What section of video
│   └── visual_notes: If transcript includes descriptions
│
└── FLAG: Customer testimonial video vs. demo video vs. expert video
```

## Quality Gates

Before passing to extraction microskills:

- [ ] All source files successfully loaded
- [ ] Source types correctly classified
- [ ] Segments properly delimited (no merged testimonials)
- [ ] Attribution data preserved where present
- [ ] Position metadata captured for ranking use
- [ ] At least 10 segments identified across all sources

---

## Edge Cases

### Unstructured Sources

```
IF source has no clear structure:
    1. Parse by paragraph breaks
    2. Flag as "unstructured"
    3. Let extraction layer handle classification
    4. Lower confidence scores for extracted elements
```

### Mixed-Language Sources

```
IF source contains multiple languages:
    1. Parse as-is
    2. Tag segments with detected language
    3. Flag for potential translation
    4. Do not discard—may contain usable proof
```

### Corrupted or Partial Files

```
IF file is partially readable:
    1. Extract what's readable
    2. Log corruption details
    3. Warn user about incomplete extraction
    4. Proceed with available content
```

---

## Handoff

Pass Parsed Source Collection to:
- **1.2 Testimonial Extraction** (for testimonial_file sources)
- **1.3 Promotion Mining** (for previous_promotion sources)
- **1.4 Study Document Extraction** (for study_document sources)

Each extraction microskill receives pre-parsed, segmented content.
