# 1.6 Sub-Type Matching

## Purpose
Match each classified proof element to its specific sub-type from the 75-sub-type taxonomy. Sub-types enable precise inventory analysis, gap detection, and pattern matching across the vault.

---

## Input Requirements

From 1.5 Category Classification:
- Categorized proof elements
- Primary category assigned
- Classification confidence

---

## Sub-Type Taxonomy Reference

Full taxonomy in [PROOF-TAXONOMY.md](../PROOF-TAXONOMY.md)

**Summary:**
- SOCIAL: 15 sub-types (SOC-01 to SOC-15)
- AUTHORITY: 20 sub-types (AUT-01 to AUT-20)
- DEMONSTRATION: 16 sub-types (DEM-01 to DEM-16)
- MECHANISM: 10 sub-types (MEC-01 to MEC-10)
- DATA: 12 sub-types (DAT-01 to DAT-12)
- RISK_REVERSAL: 12 sub-types (RSK-01 to RSK-12)

---

## Execution Protocol

### Step 1: Category-Specific Sub-Type Selection

For each element, match to the MOST SPECIFIC applicable sub-type.

---

## Quality Gates

Before sub-typing complete:

- [ ] Sub-type ID assigned (SOC-01, AUT-14, etc.)
- [ ] Sub-type matches category
- [ ] Rationale documented
- [ ] Confidence adjusted if ambiguous

---

## Common Sub-Type Errors

**Error: Too Generic**
```
BAD: All testimonials → testimonial_written
GOOD: Check for specific result, transformation, video first
```

**Error: Study vs. Mechanism Study**
```
BAD: Any study about mechanism → study_peer_reviewed
GOOD: If focus is HOW it works → mechanism_study (MEC-01)
```

**Error: Overlapping Data Types**
```
BAD: "97% success rate" → data_percentage
GOOD: Success rate is specific → data_success_rate (DAT-03)
```
