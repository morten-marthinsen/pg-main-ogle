# 2.1 Composite Score Calculation

## Purpose
Calculate the composite score for each proof element by weighting the 5 dimension scores. The composite score enables quality comparison, ranking, and gap analysis.

---

## Input Requirements

From 1.7 Element Scoring:
- Elements with 5 dimension scores
- At least 4 dimensions scored (relevance may be pending)

---

## The Weighting Formula

### Standard Weights

```python
composite_score = (
    specificity * 0.25 +      # 25% - How precise?
    credibility * 0.25 +      # 25% - How believable?
    relevance * 0.20 +        # 20% - How connected to claim?
    novelty * 0.15 +          # 15% - How fresh?
    emotional_impact * 0.15   # 15% - How moving?
)
```

**Rationale:**
- **Believability (50%)**: Specificity + Credibility = Will they believe it?
- **Connection (20%)**: Relevance = Does it support the promise?
- **Differentiation (30%)**: Novelty + Emotion = Does it stand out?

---

## Execution Protocol

### Step 1: Validate Dimension Scores

Before calculation, verify:

```
VALIDATION CHECKS:
├── All scores are integers 1-10
├── No null values except allowed (relevance pending)
├── Scores have documented rationale
└── No obvious scoring errors (10s should be rare)
```

### Step 2: Handle Missing Relevance

If relevance is null (pending promise context):

```python
def calculate_composite(element, relevance_available=True):
    if relevance_available:
        # Full formula
        composite = (
            element.specificity * 0.25 +
            element.credibility * 0.25 +
            element.relevance * 0.20 +
            element.novelty * 0.15 +
            element.emotional_impact * 0.15
        )
    else:
        # Partial formula (relevance pending)
        # Redistribute relevance weight proportionally
        composite_partial = (
            element.specificity * 0.3125 +   # 0.25 / 0.80
            element.credibility * 0.3125 +   # 0.25 / 0.80
            element.novelty * 0.1875 +       # 0.15 / 0.80
            element.emotional_impact * 0.1875  # 0.15 / 0.80
        )
        # Flag as partial
        return {
            'composite': composite_partial,
            'is_partial': True,
            'awaiting': 'relevance'
        }

    return {
        'composite': composite,
        'is_partial': False
    }
```

### Step 3: Apply Rounding

```python
# Round to 2 decimal places
composite = round(composite, 2)

# Example: 7.347 → 7.35
```

### Step 4: Assign Quality Tier

```
QUALITY TIERS:

STRONG (7.0 - 10.0)
├── Use prominently in copy
├── Feature in lead or close
├── Knockout proof candidate
└── High-confidence claims supported

MEDIUM (4.0 - 6.9)
├── Supporting proof
├── Proof stack material
├── Body copy use
└── Moderate confidence

WEAK (1.0 - 3.9)
├── Consider improving or omitting
├── May need verification
├── Low-priority in rankings
└── Gap indicator if many weak
```
