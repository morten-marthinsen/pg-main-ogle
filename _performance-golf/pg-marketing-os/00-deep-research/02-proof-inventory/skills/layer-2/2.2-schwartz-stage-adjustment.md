# 2.2 Schwartz Stage Adjustment

## Purpose
Adjust proof element scores based on market sophistication stage. Different proof types have different impact at different stages—a testimonial matters more in Stage 5, mechanism proof matters more in Stage 3.

---

## Input Requirements

From 2.1 Composite Score Calculation:
- Elements with composite scores
- Quality tiers assigned

From Deep Research (or user input):
- Schwartz sophistication stage (1-5)

---

## Schwartz Stage Overview

```
MARKET SOPHISTICATION STAGES:

STAGE 1 - First in Market
├── Prospect hasn't heard promise before
├── Simple demonstration works
├── Proof role: "Show it works at all"

STAGE 2 - Second Wave
├── Others have made similar promise
├── Need bigger/better claims
├── Proof role: "Prove we're better"

STAGE 3 - Crowded Market
├── Many competitors, skepticism rising
├── Need unique mechanism
├── Proof role: "Prove WHY it works differently"

STAGE 4 - Sophisticated Market
├── All mechanisms seem similar
├── Need unique mechanism proof
├── Proof role: "Prove our mechanism is unique/better"

STAGE 5 - Exhausted Market
├── Prospect heard it all
├── Identity/belonging matters
├── Proof role: "Prove I belong with these people"
```

## Execution Protocol

### Step 1: Get Stage Context

```python
def get_schwartz_stage(context):
    # Priority order for stage determination
    if context.deep_research_output:
        return context.deep_research_output.schwartz_stage

    if context.user_specified_stage:
        return context.user_specified_stage

    # Default to Stage 3 if unknown (most common scenario)
    return 3
```

### Step 2: Apply Stage Multiplier

```python
def apply_stage_adjustment(element, stage):
    category = element.category.lower()
    base_score = element.composite.score
    multiplier = STAGE_MULTIPLIERS[stage][category]

    adjusted_score = base_score * multiplier

    # Cap at 10 (scores can't exceed 10)
    adjusted_score = min(adjusted_score, 10.0)

    return {
        'base_score': base_score,
        'stage': stage,
        'multiplier': multiplier,
        'adjusted_score': round(adjusted_score, 2)
    }
```

### Step 3: Recalculate Quality Tier

After adjustment, quality tier may change:

```python
def get_adjusted_tier(adjusted_score):
    if adjusted_score >= 7.0:
        return 'strong'
    elif adjusted_score >= 4.0:
        return 'medium'
    else:
        return 'weak'
```

---

## Common Adjustment Errors

**Error: Ignoring Stage Context**
```
BAD: Same scoring regardless of market
GOOD: Stage 3 mechanism proof should score higher than Stage 1
```

**Error: Over-Adjusting**
```
BAD: Doubling scores for "important" categories
GOOD: Multipliers are calibrated (max 1.3x)
```

**Error: Forgetting Cap**
```
BAD: Adjusted score of 11.7
GOOD: Cap at 10.0
```
