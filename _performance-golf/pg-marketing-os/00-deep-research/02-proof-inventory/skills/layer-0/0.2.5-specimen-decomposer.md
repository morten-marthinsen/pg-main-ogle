# Layer 0.2.5 — Specimen Decomposer

**Version:** 1.0
**Layer:** 0 (Foundation)
**Type:** Specimen Analysis
**Domain:** Proof Inventory
**Dependencies:** vault_proof_intelligence (from 0.2-vault-intelligence-loader.md)
**Output:** specimen_decomposition (structural templates, technique catalog, power elements)

---

## Purpose

Decompose top-scoring proof specimens from the vault-intelligence into reusable structural templates, technique catalogs, and power elements. Proof specimens are organized by category (44+ types including AUTHORITY, SCIENTIFIC, TESTIMONIAL, SOCIAL_PROOF, etc.) with verbatim quotes. This skill extracts the patterns of proof construction, integration, and layering that make elite campaigns credible.

**This skill bridges the gap between:** vault data (categorized verbatim proof quotes) and generation layers (what Claude needs to construct original proof architectures at the same density and persuasive power).

---

input:

## Decomposition Protocol

### Phase 1: Specimen Selection & Tiering

```
IF tier_metadata present (Phase B loader active):
  LOAD gold_specimens → full decomposition
  LOAD silver_specimens → template-only decomposition
ELSE (pre-upgrade fallback):
  FOR each proof category with specimens:
    SELECT top 3 by position as pseudo-Gold
    SELECT next 5 as pseudo-Silver
  LOG "Operating in pre-upgrade mode — tiering estimated from position within category"

VERIFY each proof specimen has:
  - verbatim_quote (required — skip if absent)
  - category (required)
  - source_file (required for provenance)
  - niche (required for type-matching)
```

### Phase 2: Structural Template Extraction

For each Gold specimen:

```
ANALYZE verbatim_quote and extract:

  proof_construction:
    - element_type: [claim | evidence | source_citation | quantification |
                     third_party_endorsement | experiential_detail |
                     institutional_reference | comparison_anchor |
                     time_frame | scope_qualifier]
    - function: string (what this element contributes to credibility)
    - position: integer
    - proportion: float

  proof_architecture:
    category: string (from specimen)
    construction_pattern: string (how the proof is built — e.g.,
                                  "claim_then_evidence", "evidence_first",
                                  "stacking", "narrative_embedded")
    specificity_level: string (vague | moderate | precise | hyper_precise)
    credibility_source: string (what makes this proof believable)

For Silver specimens:
  Extract proof_architecture ONLY
  SKIP element-level construction analysis
```

### Phase 3: Technique Catalog Compilation

Aggregate across ALL decomposed specimens:

```
proof_categories:
  FOR each category with specimens:
    - category_name: string
    - specimen_count: integer
    - construction_patterns: [string] (unique patterns found in this category)
    - best_niches: [string] (which niches use this category most)
    - typical_position_in_copy: string (where this proof type usually appears)

construction_techniques:
  FOR each unique construction_pattern:
    - technique_name: string
    - count: integer
    - implementation_pattern: string (how to build proof using this technique)
    - gold_example: string (1-2 sentence excerpt)
    - categories_using: [string]

layering_patterns:
  FOR each observed proof-layering approach (multiple proofs in sequence):
    - pattern_name: string (e.g., "authority_then_data", "testimonial_sandwich",
                            "escalating_evidence")
    - category_sequence: [string] (ordered list of proof categories in sequence)
    - frequency: integer
    - specimens_using: [source_file references]
    - effectiveness_notes: string

specificity_techniques:
  FOR each method of making proof specific:
    - technique_name: string (exact_numbers, named_institutions, date_stamps,
                              geographic_references, named_individuals)
    - count: integer
    - implementation_pattern: string
    - gold_example: string
```

### Phase 4: Power Element Extraction

From Gold specimens ONLY:

```
proof_quotes_by_category:
  FOR each proof category with Gold specimens:
    - category: string
    - verbatim_quotes: [
        - text: string (the full proof quote)
        - construction_pattern: string
        - specificity_level: string
        - source_file: string
        - niche: string
      ]

specificity_fragments:
  FOR Gold specimens with hyper_precise specificity:
    - verbatim_fragment: string (the specific number, name, or detail)
    - specificity_type: string (number, institution, date, person, geographic)
    - context: string (what claim it supports)
    - source_file: string

transition_into_proof:
  FOR Gold specimens where proof follows a claim:
    - verbatim_transition: string (1-2 sentences bridging claim → proof)
    - transition_type: string
    - source_file: string
```

## Quality Gates

- [ ] Minimum 5 proof categories with Gold specimens decomposed
- [ ] Minimum 10 categories with Silver specimens template-decomposed
- [ ] All structural templates have complete construction elements
- [ ] Technique catalog has minimum 3 construction techniques cataloged
- [ ] Technique catalog has minimum 2 layering patterns documented
- [ ] Power elements contain minimum 5 verbatim proof quotes across categories
- [ ] Specificity fragments extracted from hyper-precise specimens
- [ ] Every power element includes source provenance (source_file + category)
- [ ] No raw verbatim_quote passed through undecomposed

---

## Anti-Patterns

| Anti-Pattern | Signal | Fix |
|---|---|---|
| **Category Dump** | All proof quotes listed by category without construction analysis | Decompose each quote into construction elements |
| **Flat Catalog** | Proof categories listed without layering pattern analysis | Include layering_patterns showing proof sequencing |
| **Specificity Blind** | Proof elements extracted without specificity_level classification | Rate every proof as vague/moderate/precise/hyper_precise |
| **Niche Ignored** | Proof techniques cataloged without niche context | Include niche with every specimen reference |
| **Integration Missing** | Proof quotes isolated without transition context | Extract transition_into_proof showing how claims bridge to evidence |
