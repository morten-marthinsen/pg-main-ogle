# 2.7-A-story-elements-researcher.md

> **Layer:** 2 (Intelligence Analysis)
> **Skill:** 01-research
> **Purpose:** Research and compile story elements that supplement client-provided brief data
> **Inputs:** Brief Section 8 (Story Elements), Layer 1 quotes, web research capabilities
> **Outputs:** `story_elements_research.md` with tagged elements for 10-Story consumption

---

## FUNCTION

This skill researches and compiles story elements that can be used by the 10-Story skill to generate authentic, grounded stories. It SUPPLEMENTS client-provided elements from Brief Section 8 — it does NOT fabricate core factual elements.

**Key Principle:** Client-provided elements are [FACTUAL] and preserved exactly. Researched elements are [RESEARCHED] and tagged with source confidence. The AI can ENHANCE narrative quality but cannot FABRICATE discovery narratives, founders, or proof data.

---

## RESEARCH TARGETS

### Target 1: Discovery/Origin Narrative Research

**If Brief Section 8A is sparse, search for:**

```
SEARCH QUERIES:
- "[Brand/Product name] origin story"
- "[Brand/Product name] how it started"
- "[Brand/Product name] founder story"
- "[Brand/Product name] discovery"
- "[Founder name if known] interview"
- "[Founder name if known] background"

SOURCES TO CHECK:
- Company "About Us" page
- Press releases (company newsroom)
- Founder interviews (YouTube, podcasts)
- LinkedIn profiles (founder/key personnel)
- Crunchbase, AngelList (if applicable)
- Media coverage of product launch
- Patent filings (may reveal invention story)

CAPTURE:
- Founding date and circumstances
- Founder background and credentials
- Any documented discovery moment
- Geographic origins
- Institutional affiliations
- Named researchers or experts involved
```

### Target 2: Expert/Authority Figure Research

**If Brief Section 8D is sparse, search for:**

```
SEARCH QUERIES:
- "[Product ingredient/mechanism] research"
- "[Product ingredient/mechanism] scientist"
- "[Product ingredient/mechanism] study"
- "[Brand name] doctor" OR "[Brand name] researcher"
- "[Brand name] clinical trial"

SOURCES TO CHECK:
- Product website (scientific advisors, medical board)
- PubMed / Google Scholar (ingredient research)
- University press releases
- Expert directory websites
- Conference speaker lists
- Media interviews with experts

CAPTURE:
- Named experts associated with product/mechanism
- Their credentials and affiliations
- Published research they've done
- Quotes or endorsements they've given
- Contact information (for verification)
```

### Target 3: Testimonial Story Mining

**Mine Layer 1 quotes for story-worthy testimonials:**

```
SCAN LAYER 1 QUOTES FOR:
- Before/after narratives (not just outcomes)
- Emotional journey descriptions
- Specific transformation timelines
- Named individuals with detailed stories
- Quotes that include "I was..." + "Now I..."

ALSO SEARCH:
- Video testimonials on YouTube
- Case study pages on product website
- Long-form reviews (Amazon, Trustpilot)
- Social media posts tagged with brand
- Forum posts with detailed user experiences

CAPTURE:
- Full story arc (before → catalyst → after)
- Specific results mentioned
- Emotional language used
- Names (or anonymized identifiers)
- Timeframes mentioned
```

### Target 4: Proof/Data Documentation

**If Brief Section 8C is sparse, search for:**

```
SEARCH QUERIES:
- "[Product name] results"
- "[Product name] study"
- "[Product name] clinical trial"
- "[Mechanism/ingredient] research"
- "[Product name] success rate"

SOURCES TO CHECK:
- Product website (results page, science page)
- Clinical trial registries (clinicaltrials.gov)
- Published studies (PubMed, Google Scholar)
- Press releases announcing results
- Third-party reviews with data

CAPTURE:
- Participant counts
- Success rates / effectiveness percentages
- Timeframes for results
- Study methodology (if documented)
- Third-party validation sources
```

### Target 5: Warning/Threat Data (for warning stories)

**If market analysis suggests warning story is appropriate:**

```
SEARCH QUERIES:
- "[Problem/threat] statistics"
- "[Problem/threat] trend"
- "[Problem/threat] expert warning"
- "[Problem/threat] getting worse"
- "[Industry] crisis"

SOURCES TO CHECK:
- Government statistics (CDC, BLS, Fed data)
- Industry reports
- Expert interviews and op-eds
- Academic research
- News coverage of the problem

CAPTURE:
- Specific statistics with sources
- Expert quotes about the threat
- Historical parallels
- Projected consequences
- Timeline of escalation
```

### Target 6: Competitive Story Analysis

**Understand what story approaches competitors use:**

```
SEARCH:
- Competitor sales pages (story sections)
- Competitor video content (origin stories)
- Competitor About pages

CAPTURE:
- What story types competitors use
- Recurring narrative patterns in market
- Story elements that appear saturated
- Story angles that appear underutilized
- What the market "expects" from stories
```

## EXECUTION PROTOCOL

### Step 1: Ingest Brief Section 8

```
READ brief section 8 (Story Elements)

FOR each subsection (8A through 8F):
  IDENTIFY what client has provided
  TAG provided elements as [FACTUAL]
  IDENTIFY gaps (empty or minimal fields)
  CREATE research agenda for gaps
```

### Step 2: Execute Research Queries

```
FOR each identified gap:
  EXECUTE relevant search queries
  SCAN sources for story-relevant information
  EXTRACT specific elements with source attribution
  TAG extracted elements as [RESEARCHED]
  SCORE source confidence (VERIFIED | SINGLE_SOURCE | INFERRED)
```

### Step 3: Mine Layer 1 Quotes

```
SCAN all Layer 1 quotes for story elements:
  - Testimonial stories (not just soundbites)
  - Before/after narratives
  - Emotional journey descriptions
  - Problem severity descriptions
  - Discovery/revelation moments in customer language

EXTRACT story-worthy quotes with full context
TAG as [FROM_VOC] (Voice of Customer)
```

### Step 4: Synthesize and Score

```
COMBINE:
  - Client-provided elements [FACTUAL]
  - Researched elements [RESEARCHED]
  - Voice of Customer elements [FROM_VOC]

SCORE each element category (0-3):
  0 = No information available
  1 = Minimal/vague information
  2 = Moderate detail available
  3 = Rich, specific detail available

GENERATE story_type_recommendation based on available material
```

### Step 5: Flag Gaps and Creative License Zones

```
IDENTIFY elements that are:
  [FACTUAL] — Must be preserved exactly
  [ENHANCE] — Facts exist, narrative can be improved
  [CREATIVE] — Gap exists, reasonable fill allowed (flag for review)
  [MISSING] — No basis found, cannot fabricate

FLAG any elements where creative license would be required
NOTE: 10-Story skill should NOT fabricate [MISSING] elements
```

## ANTI-PATTERNS (FORBIDDEN)

1. **Fabricating Discovery Narratives**
   - If no discoverer is documented, DO NOT invent one
   - Mark as [MISSING], recommend story type that doesn't require it

2. **Inventing Experts**
   - If no expert validation exists, DO NOT create fictional experts
   - Mark as [MISSING] or use mechanism research instead

3. **Creating Fake Testimonials**
   - All testimonials must be [FACTUAL] or [FROM_VOC]
   - Never generate fictional customer stories

4. **Ignoring Client Input**
   - [FACTUAL] elements from brief take precedence over research
   - Never override client-provided story elements

5. **Recommending Impossible Story Types**
   - Don't recommend "proof story" if no proof data exists
   - Don't recommend "discovery story" if no discovery narrative exists
