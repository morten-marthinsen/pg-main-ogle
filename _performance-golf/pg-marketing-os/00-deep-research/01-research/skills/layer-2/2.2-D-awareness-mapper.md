**Version:** 2.0
**Layer:** 2 (Intelligence)
**Type:** E5 Method - Awareness Analysis
**Dependencies:** scored_quotes.json, belief_analysis.json
**Output:** awareness_map.json

---

## Purpose

Map the target WEB's position on the awareness spectrum (Schwartz's 5 levels). This determines messaging strategy - where to meet prospects and how to move them toward action.

---

## Awareness Framework (Schwartz)

```
5 LEVELS OF AWARENESS:

1. UNAWARE
   - Don't know they have a problem
   - Can't be sold to directly
   - Need education/awakening

2. PROBLEM-AWARE
   - Know they have a problem
   - Don't know solutions exist
   - Need solution introduction

3. SOLUTION-AWARE
   - Know solutions exist
   - Don't know your solution
   - Need differentiation

4. PRODUCT-AWARE
   - Know your solution exists
   - Haven't decided to buy
   - Need convincing

5. MOST AWARE
   - Know and want your solution
   - Just need the offer
   - Need deal/action trigger

Most marketing targets levels 2-4.
```

## Analysis Logic

### Step 1: Classify Quotes by Awareness

```
FOR each quote:

IDENTIFY awareness level:

UNAWARE indicators:
- No problem mentioned
- General curiosity
- "Just exploring"
- (Rare in active research data)

PROBLEM-AWARE indicators:
- "I have a problem with..."
- "I'm struggling with..."
- "Why does X happen?"
- No solution mentions
- Questions about problems

SOLUTION-AWARE indicators:
- "I'm looking for..."
- "Which is better, X or Y?"
- "Has anyone tried..."
- General solution categories mentioned
- Comparison shopping language

PRODUCT-AWARE indicators:
- Specific products named
- "I'm thinking about X"
- "Has anyone used [specific product]?"
- Brand comparisons
- Review seeking

MOST-AWARE indicators:
- "Where can I buy..."
- "Is there a discount..."
- Ready-to-act language
- (Rare in research data)

ASSIGN awareness_level to each quote
```

### Step 2: Calculate Distribution

```
AGGREGATE awareness counts:

FOR each level:
  count = quotes at this level
  percentage = count / total * 100

BUILD awareness_distribution object
```

### Step 3: Identify Primary Level

```
DETERMINE where most prospects are:

primary_awareness_level = level with highest %

IF close distribution (e.g., 35% problem, 40% solution):
  NOTE: "Spread between [level1] and [level2]"

CALCULATE confidence:
- 60%+ in one level: high confidence
- 40-60% in one level: medium confidence
- <40% in any level: spread/mixed

DERIVE implications:
- What messaging strategy fits?
- Where to meet them?
- How to move them?
```

### Step 4: Map Awareness Journey

```
ANALYZE movement patterns:

TYPICAL PATH:
- How do people move through levels?
- What sequence is common?

STICKING POINTS:
- Where do people get stuck?
- What prevents level advancement?

MOVEMENT TRIGGERS:
- What causes advancement?
- What information moves them?
- What experiences shift awareness?

EXAMPLES:
Sticking point: "Stuck in solution-aware, trying endless strategies"
Trigger: "Breakthrough moment / hitting 'the wall'"
```

### Step 5: Define Messaging Strategy

```
BASED on primary awareness level:

IF Problem-Aware:
  entry_level = "problem"
  target_level = "solution → product"
  tactics:
    - Agitate problem first
    - Introduce solution category
    - Differentiate our approach

IF Solution-Aware:
  entry_level = "solution"
  target_level = "product"
  tactics:
    - Acknowledge they've tried things
    - Show why this is different
    - Focus on unique mechanism

IF Product-Aware:
  entry_level = "product"
  target_level = "most aware"
  tactics:
    - Address specific objections
    - Provide proof and validation
    - Remove final barriers
```

### Step 6: Extract Level-Specific Language

```
FOR each awareness level:

CAPTURE indicators:
- Phrases that signal this level
- Questions they ask
- Language patterns

CAPTURE example_quotes:
- Representative quotes
- Clear level signals

BUILD reference for targeting
```

3. **Actionability** - Clear messaging implications
4. **Evidence** - Distribution backed by quotes
