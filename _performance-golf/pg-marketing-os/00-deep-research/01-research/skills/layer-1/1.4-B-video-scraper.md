# Skill 1.4-B: Video Scraper (YouTube Protocol)

**Version:** 2.0
**Layer:** 1 (Infrastructure)
**Type:** Content Extraction
**Dependencies:** validated_sources.json, market_config.yaml
**Output:** Appends to master_raw_content.json

---

## Purpose

Execute the mandatory 3-step YouTube mining protocol: (1) Search for videos, (2) Extract transcripts, (3) Scrape comments. This captures both instructor knowledge and audience voice-of-customer.

---

## The 3-Step YouTube Protocol

```
MANDATORY SEQUENCE:

Step 1: VIDEO DISCOVERY
  - Search across 5 query categories
  - Find minimum 100 relevant videos
  - Capture video metadata

Step 2: TRANSCRIPT EXTRACTION
  - Get transcripts for top 30+ videos
  - These contain expert frameworks and explanations
  - Rich source of mechanism language

Step 3: COMMENT MINING
  - Extract 500+ comments per video
  - Comments contain raw customer voice
  - Pain, hope, and experience data
```

## Execution Logic

### Step 1: Video Discovery

```
SEARCH CATEGORIES (from market_config.youtube):

1. Direct topic queries
2. Problem-focused queries
3. Symptom-specific queries
4. Emotional queries
5. Solution-seeking queries

FOR each category:
  FOR each query in category:

    EXECUTE YouTube search:
    mcp__apify__call-actor:
      actor: "streamers/youtube-scraper"
      input:
        searchKeywords: [query]
        maxResults: 50
        type: "VIDEO"

    OR alternative:
    mcp__apify__call-actor:
      actor: "bernardo/youtube-scraper"
      input:
        searchTerms: [query]
        maxVideos: 50

    FOR each result:
      IF not duplicate:
        ADD to videos[]
        LOG query_matched

VALIDATE:
- [ ] Total videos >= 100
- [ ] Each category has >= 15 videos
- [ ] Multiple channels represented
```

### Step 2: Transcript Extraction

```
PRIORITIZE videos for transcript extraction:

SORT by:
1. View count (higher = more reach)
2. Comment count (higher = more engagement)
3. Duration (longer = more content)
4. Relevance to core topics

SELECT top 30+ videos for transcripts

FOR each selected video:

  TRY transcript extraction:
  mcp__apify__call-actor:
    actor: "bernardo/youtube-transcript-scraper"
    input:
      videoUrls: [video_url]
      includeTimestamps: true

  IF transcript available:
    STORE full_text
    STORE segments with timestamps
    EXTRACT key passages:
      - Framework explanations
      - Problem descriptions
      - Mechanism discussions
      - Solution presentations

  IF transcript unavailable:
    LOG as "no_transcript"
    Use title + description only

VALIDATE:
- [ ] Transcripts obtained >= 30
- [ ] Total transcript words >= 100,000
```

### Step 3: Comment Mining

```
FOR each video (all 100+, prioritize by engagement):

  EXTRACT comments:
  mcp__apify__call-actor:
    actor: "alexey/youtube-comment-scraper"
    input:
      videoUrls: [video_url]
      maxComments: 500

  FOR each comment:
    CAPTURE:
    - Author name
    - Comment text
    - Like count
    - Reply count
    - Date posted
    - Is it a reply?

  IDENTIFY high-value comments:
    - Long-form responses (>100 chars)
    - High like counts
    - Personal stories ("I", "my")
    - Questions (reveal pain points)
    - Disagreements (reveal objections)

VALIDATE:
- [ ] Total comments >= 15,000
- [ ] Average comments per video >= 150
```

## Quote Pre-Extraction

### From Transcripts

```
EXTRACT from transcripts:

LOOK FOR:
- Problem definitions ("The real issue is...")
- Mechanism explanations ("This happens because...")
- Framework language ("The three steps...")
- Common mistakes ("Most people...")
- Transformation stories ("Before I learned this...")

CAPTURE as potential quotes with:
- Exact text
- Timestamp reference
- Speaker context
- Quote type (explanation, mechanism, story)
```

### From Comments

```
EXTRACT from comments:

LOOK FOR:
- Personal experiences ("I've been struggling...")
- Success stories ("This changed everything...")
- Failures ("I tried this and...")
- Emotional expressions (frustration, hope, fear)
- Questions (reveal knowledge gaps)
- Objections ("But what about...")

CAPTURE as potential quotes with:
- Exact text
- Author attribution
- Engagement level (likes)
- Quote type (pain, hope, objection, question)
```

## Tool Resilience

```
YOUTUBE TOOL FALLBACK CHAIN:

For search:
1. streamers/youtube-scraper
2. bernardo/youtube-scraper
3. apify/youtube-search

For transcripts:
1. bernardo/youtube-transcript-scraper
2. apify/youtube-transcript-extractor
3. Manual via Firecrawl on transcript sites

For comments:
1. alexey/youtube-comment-scraper
2. streamers/youtube-scraper (includes comments)
3. Partial extraction via Firecrawl

IF all tools fail for a video:
- Log failure
- Continue to next video
- Acceptable loss: < 10% of videos
```

---

## Rate Limiting

```
YOUTUBE API CONSIDERATIONS:

- Space requests 2-3 seconds apart
- Batch video IDs where possible
- Monitor for rate limit responses
- If limited, pause 60 seconds and retry
- Maximum 1000 API calls per session
```

---

## Quality Criteria

1. **Volume** - 100+ videos, 30+ transcripts, 15K+ comments
2. **Relevance** - Videos match research topics
3. **Diversity** - Multiple channels and perspectives
4. **Completeness** - Full transcripts where available
