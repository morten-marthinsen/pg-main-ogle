# Skill 1.2-A: Source Scanner

**Version:** 3.0
**Layer:** 1 (Infrastructure)
**Type:** Source Discovery
**Dependencies:** queries.json, platform_list.json
**Output:** raw_sources.json

---

## Purpose

Execute queries across platforms to discover potential sources for scraping. This is the discovery phase - find as many relevant sources as possible before scoring and filtering.

---

## CRITICAL: MODEL + PERSONA + VOLUME AWARENESS (v3.2 ADDITION)

**Designated model: `sonnet`** — Source discovery is search + evaluate, not deep analysis.
**Designated persona: `PERSONA_SOURCE_HUNTER`** — See MASTER-AGENT.md Subagent Persona Library.

**Sources must yield 1,000+ quotes across 6 buckets.** When evaluating source potential, estimate quote yield per source. A high-reply forum thread (200+ comments) may yield 30-50 quotes. A thin blog post may yield 0-2.

**Source volume math:** If average yield is ~20 quotes per source, you need 50+ high-yielding sources to hit 1,000. If average yield is ~10, you need 100+. Always err on the side of MORE sources — it's better to over-discover than under-discover.

**Bucket coverage:** Ensure discovered sources cover ALL 6 buckets (Pain, Hope, Root Cause, Solutions Tried, Competitor Mechanism, Villain). If your source list is heavy on Pain/Hope but light on Villain/Competitor Mechanism, generate additional queries targeting those gaps BEFORE moving to scoring.

## Execution Logic

### Step 1: Prioritize Query Execution

```
QUERY PRIORITIZATION:

1. Sort queries by priority (high → medium → low)
2. Group by platform for efficient execution
3. Deduplicate similar queries

Execution order:
1. High priority queries on critical platforms
2. High priority queries on high platforms
3. Medium priority queries
4. Low priority queries (if budget allows)
```

### Step 2: Execute Platform-Specific Discovery

```
FOR each platform in platform_list:

  SELECT appropriate tool:
  - Reddit → Apify search
  - YouTube → Apify YouTube search
  - Twitter/X → Apify tweet scraper (Firecrawl BLOCKED on x.com)
  - Forums → Firecrawl search
  - General → Firecrawl, Gemini, or Perplexity

  FOR each query targeting this platform:

    EXECUTE search:
    - Use platform-appropriate search syntax
    - Capture first 20-50 results per query
    - Extract: URL, title, preview snippet

    LOG discovery:
    - Query executed
    - Results count
    - Sources added (deduped)
```

### Step 3: Tool-Specific Execution

```
FIRECRAWL EXECUTION:
  mcp__firecrawl-mcp__firecrawl_search:
    query: [query string]
    limit: 20
    scrapeOptions:
      formats: ["markdown"]
      onlyMainContent: true

APIFY EXECUTION (Reddit):
  mcp__apify__call-actor:
    actor: "fatihtahta/reddit-scraper-search-fast"
    input:
      searchKeyword: [query]
      postLimit: 100
      subreddits: [from platform_list specific_targets]

APIFY EXECUTION (YouTube):
  mcp__apify__call-actor:
    actor: "grow_media/youtube-search-api"
    input:
      searchKeywords: [queries]
      maxResults: 50

APIFY EXECUTION (Twitter/X):
  > CRITICAL: Firecrawl is HARD-BLOCKED on x.com (policy block). Apify is the ONLY path.

  PRIMARY:
  mcp__apify__call-actor:
    actor: "apidojo/tweet-scraper"
    input:
      searchQueries: [queries from market_config twitter_queries]
      maxItems: 500
      includeReplies: true

  WHY PRIMARY: 34,980 users, 99.9% success rate, $0.40/1K tweets.
  Supports: keyword search, hashtag search, profile scraping, URL scraping, list scraping.

  FALLBACK (if apidojo fails):
  mcp__apify__call-actor:
    actor: "quacker/twitter-scraper"
    input:
      searchTerms: [queries]
      maxTweets: 500
      includeReplies: true

GEMINI EXECUTION (search/research):
  > Use for broad web search discovery and complex multi-step research tasks.
  > Complements Firecrawl (structured scraping) and Perplexity (research synthesis).

  SEARCH:
  mcp__gemini__gemini-search:
    query: [query string]

  DEEP RESEARCH (complex/multi-step):
  mcp__gemini__gemini-deep-research:
    query: [detailed research question]

  URL ANALYSIS (specific page content):
  mcp__gemini__gemini-analyze-url:
    url: [target URL]
    query: [what to extract]

PERPLEXITY EXECUTION (research fallback):
  mcp__perplexity__perplexity_search:
    query: [query]
    max_results: 20

  mcp__perplexity__perplexity_research (deep research fallback):
    messages:
      - role: "user"
        content: [detailed research question]
```

### Step 4: Deduplicate and Normalize

```
DEDUPLICATION:

1. Normalize URLs (remove tracking params)
2. Identify duplicate content across queries
3. Merge duplicate entries (keep first, note all matching queries)

NORMALIZATION:

For each source:
- Clean URL
- Extract domain/platform
- Estimate content type (thread, article, video, review)
- Estimate content volume (short, medium, long)
```

### Step 5: Validate Discovery Volume

```
VALIDATION:

- [ ] Total sources >= 100
- [ ] Each critical platform has >= 15 sources
- [ ] Each high platform has >= 10 sources
- [ ] All research topics have at least 3 sources
- [ ] Mix of content types (threads, videos, reviews)

IF validation fails:
  - Identify underperforming platforms/topics
  - Execute additional queries
  - Try alternative query phrasing
  - DO NOT proceed with < 100 sources
```

---

## Tool Resilience

```
IF primary tool fails:
  1. Log failure: "[Tool] failed for [platform]: [error]"
  2. Switch to fallback tool from platform_list
  3. Retry query
  4. If fallback fails, log and continue with other queries
  5. DO NOT halt on single tool failure

PLATFORM-SPECIFIC RESILIENCE:
  Twitter/X: Firecrawl HARD-BLOCKED → Apify apidojo/tweet-scraper → quacker/twitter-scraper → Gemini → Perplexity → Manual URLs
  Reddit: Firecrawl blocked → Apify fatihtahta/reddit-scraper-search-fast
  YouTube: Apify grow_media/youtube-search-api → Apify alternative actors
  Forums: Firecrawl → Gemini gemini-analyze-url → Perplexity
  General: Firecrawl → Gemini gemini-search → Perplexity → WebFetch
```

---

## Quality Criteria

1. **Volume** - 100+ sources minimum
2. **Diversity** - Multiple platforms represented
3. **Topic Coverage** - All topics have sources
4. **Content Mix** - Threads, videos, reviews all present
