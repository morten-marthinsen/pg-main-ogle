# Skill 1.2-B: Source Scorer

**Version:** 2.0
**Layer:** 1 (Infrastructure)
**Type:** Source Scoring
**Dependencies:** raw_sources.json, market_config.yaml
**Output:** scored_sources.json

---

## Purpose

Score each discovered source for relevance, expected quote volume, and priority. This enables intelligent allocation of scraping resources to highest-value sources first.

---

## Input Schema

```yaml
input:
  raw_sources: raw_sources.json
  market_config: market_config.yaml
```

## Scoring Algorithm

### Step 1: Relevance Score (1-10)

```
RELEVANCE SCORING:

10 points: Direct match to primary problem/topic
  - Title contains exact problem terms
  - Preview shows first-person experience
  - Emotional language present

8-9 points: Strong relevance
  - Related to primary problem
  - Contains customer terminology
  - Shows engagement/discussion

6-7 points: Moderate relevance
  - Adjacent topic
  - Industry-related
  - May contain relevant quotes

4-5 points: Weak relevance
  - Tangentially related
  - May require filtering

1-3 points: Low relevance
  - Generic content
  - Off-topic
  - Likely noise
```

### Step 2: Volume Score (1-10)

```
VOLUME SCORING:

Based on estimated_content_volume and content_type:

Threads (forum/Reddit):
  - 100+ comments → 10
  - 50-99 comments → 8
  - 20-49 comments → 6
  - 10-19 comments → 4
  - <10 comments → 2

Videos (YouTube):
  - Long-form (>20 min) → 9
  - Medium (10-20 min) → 7
  - Short (<10 min) → 4
  - Add +1 if high comment count

Reviews:
  - Product page with 50+ reviews → 9
  - 20-49 reviews → 7
  - 10-19 reviews → 5
  - <10 reviews → 3

Articles:
  - Long-form with comments → 7
  - Long-form no comments → 5
  - Short-form → 3
```

### Step 3: Freshness Score (1-10)

```
FRESHNESS SCORING:

Based on content date (if available):

- Last 30 days → 10
- 1-3 months → 9
- 3-6 months → 8
- 6-12 months → 7
- 1-2 years → 5
- 2-3 years → 3
- 3+ years → 2
- Unknown date → 5 (neutral)

Note: Evergreen pain/hope content may warrant
override - freshness less critical for emotional truth
```

### Step 4: Engagement Score (1-10)

```
ENGAGEMENT SCORING:

Signals of engagement (where visible):

High engagement (9-10):
  - High upvotes/likes
  - Many replies/comments
  - Cross-referenced in other sources
  - Viral indicators

Medium engagement (6-8):
  - Moderate interaction
  - Active discussion
  - Some shares

Low engagement (3-5):
  - Minimal interaction
  - Few responses
  - Low visibility

Unknown (5):
  - Default when signals not available
```

### Step 5: Composite Score Calculation

```
COMPOSITE FORMULA:

composite_score = (
  relevance_score * 0.40 +
  volume_score * 0.30 +
  freshness_score * 0.15 +
  engagement_score * 0.15
) * 10

Maximum: 100
Minimum: 10

PRIORITY TIERS:

- High: composite_score >= 70
- Medium: composite_score >= 45 AND < 70
- Low: composite_score < 45
```

---

## Execution Logic

### Step 1: Load Context

```
LOAD:
1. raw_sources.json - all discovered sources
2. market_config.yaml - for terminology matching

EXTRACT from market_config:
- customer.primary terms
- problem.primary terms
- emotional_context patterns
- aspect keywords
```

### Step 2: Score Each Source

```
FOR each source in raw_sources:

  1. CALCULATE relevance_score:
     - Check title against problem terms
     - Check preview for customer language
     - Check for emotional indicators
     - Apply scoring rubric

  2. CALCULATE volume_score:
     - Use estimated_content_volume
     - Apply content_type multipliers
     - Estimate quote potential

  3. CALCULATE freshness_score:
     - Extract date if available
     - Apply time-based scoring
     - Consider evergreen override

  4. CALCULATE engagement_score:
     - Use available signals
     - Default to 5 if unknown

  5. COMPUTE composite_score

  6. ASSIGN priority_tier

  7. GENERATE scoring_notes:
     - Brief rationale for score
     - Key factors considered
```

### Step 3: Validate Distribution

```
VALIDATION CHECKS:

- [ ] High priority sources >= 30
- [ ] Each critical platform has high-priority sources
- [ ] All topics have at least 2 high-priority sources
- [ ] Total estimated quotes from high+medium >= 1000

IF validation fails:
  - Flag underserved topics/platforms
  - Recommend additional discovery
  - DO NOT proceed if high-priority < 20
```

### Step 4: Generate Coverage Analysis

```
FOR each topic in research_topics:

  COUNT sources by priority tier
  CALCULATE coverage_status:
    - "strong" if high_priority >= 5
    - "adequate" if high_priority >= 2
    - "weak" if high_priority < 2

GENERATE topic_coverage report
```

---

## Quality Criteria

1. **Discrimination** - Clear separation between priority tiers
2. **Accuracy** - Scores reflect actual quote potential
3. **Coverage** - All topics have high-priority sources
4. **Efficiency** - Highest-value sources identified for first scrape
