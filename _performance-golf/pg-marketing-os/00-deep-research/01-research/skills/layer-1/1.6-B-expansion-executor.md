# Skill 1.6-B: Expansion Executor

**Version:** 2.0
**Layer:** 1 (Infrastructure)
**Type:** Remediation & Loop Control
**Dependencies:** gate_decision.json, remediation_plan
**Output:** Updated Layer 1 outputs, expansion_log.json

---

## Purpose

Execute the remediation plan when Layer 1 validation fails. This skill orchestrates the expansion loop - targeting specific gaps, executing additional scraping, and re-running the pipeline until requirements are met.

---

## Expansion Loop Philosophy

```
AUTONOMOUS EXPANSION:

When validation fails, do NOT ask the user.
Execute the remediation plan automatically.

LOOP STRUCTURE:
1. Identify specific gaps
2. Generate targeted queries
3. Execute focused scraping
4. Merge new content
5. Re-run classification and scoring
6. Re-validate
7. If still failing, repeat (max 3 iterations)

ONLY escalate to user if:
- 3 iterations complete without passing
- Tools are failing consistently
- Budget constraints reached
```

## Execution Logic

### Step 1: Parse Remediation Plan

```
FROM gate_decision and validation_report:

EXTRACT remediation_plan.actions:

FOR each action:
  UNDERSTAND:
  - What gap to fill
  - What target to reach
  - What approach to use

BUILD expansion_tasks[]
```

### Step 2: Generate Targeted Queries

```
FOR each gap:

IF gap_type = "bucket_shortage":

  GENERATE queries targeting that bucket:

  PAIN gaps:
  - "[market term] frustrated"
  - "[market term] struggling"
  - "can't [desired outcome]"
  - "[market term] ruining my"
  - "losing [money/time] on [market term]"
  - "[market term] making me crazy"

  HOPE gaps:
  - "finally [achieved outcome] [market term]"
  - "want to [desired outcome]"
  - "[market term] goals"
  - "dream of [outcome]"

  ROOT_CAUSE gaps:
  - "why [problem] happens"
  - "the real reason [problem]"
  - "[problem] because"
  - "figured out why [problem]"

  SOLUTIONS_TRIED gaps:
  - "tried [solution type] and"
  - "[product category] review"
  - "I used [product] and"
  - "[product] didn't work"

IF gap_type = "platform_shortage":
  FOCUS queries on underrepresented platform

IF gap_type = "topic_shortage":
  GENERATE topic-specific queries
```

### Step 3: Execute Targeted Scraping

```
FOR each expansion_task:

  SELECT appropriate scraper:
  - Reddit gaps → 1.4-C Reddit Scraper
  - Forum gaps → 1.4-A Forum Scraper
  - Video gaps → 1.4-B Video Scraper
  - Social gaps → 1.4-D Social Scraper

  EXECUTE scraper with targeted queries:
  - Limit scope to gap-specific content
  - Use higher result limits if needed
  - Track results carefully

  LOG action taken and results
```

### Step 4: Merge New Content

```
MERGE new scraped content:

1. Add to master_raw_content.json
   - Avoid duplicates (URL check)
   - Mark as expansion_round = [iteration]

2. Re-run Quote Extractor (1.5-B)
   - Only on new content
   - Append to existing quotes

3. Re-run Quote Classifier (1.5-C)
   - On all quotes (ensures consistency)

4. Re-run Quote Scorer (1.5-D)
   - On all quotes

UPDATE all downstream files
```

### Step 5: Re-Validate

```
RUN Layer 1 Validator (1.6-A) again

COMPARE results:
- Previous gap counts
- New gap counts
- Improvements made

IF all checks pass:
  expansion_status = "success"
  PROCEED to Layer 2

ELSE IF improvement made but gaps remain:
  IF iteration < 3:
    LOOP back to Step 2
  ELSE:
    expansion_status = "partial"
    ESCALATE to user

ELSE IF no improvement:
  expansion_status = "failed"
  DIAGNOSE problem
  ESCALATE to user
```

## Iteration Control

```
ITERATION LIMITS:

max_iterations = 3
current_iteration = 1

PER ITERATION LIMITS:
- Max new sources: 50
- Max new queries: 20
- Max time: 30 minutes

IF iteration limit reached:
  STOP expansion
  REPORT current state to user
  ASK: Continue with available data or investigate?
```

---

## Tool Failure Handling

```
IF scraper fails:

  TRY fallback tool:
  - Firecrawl → Apify → Perplexity

  IF all tools fail:
    LOG failure
    SKIP to next task
    NOTE partial execution

IF multiple tools failing:
  CHECK tool availability
  WAIT and retry (exponential backoff)
  IF persistent: escalate to user
```
