# Skill 1.4-G: Funnel Scraper

**Version:** 2.0
**Layer:** 1 (Infrastructure)
**Type:** Content Extraction
**Dependencies:** market_config.yaml (competitors)
**Output:** Appends to master_raw_content.json

---

## Purpose

Scrape competitor sales funnels: landing pages, opt-in pages, sales letters, video sales letters (VSLs), webinar registration pages, and checkout flows. Funnels reveal positioning, mechanisms, proof strategies, and conversion psychology.

---

## Target Content

```
FUNNEL ELEMENTS:

1. Landing Pages / Opt-in Pages
   - Headlines
   - Lead magnets
   - Email capture

2. Sales Pages / Sales Letters
   - Long-form copy
   - Mechanism explanations
   - Proof stacks
   - Objection handling

3. Video Sales Letters (VSLs)
   - Transcripts
   - Slide content
   - Call-to-action timing

4. Webinar Registration / Thank You Pages
   - Urgency elements
   - Promise stacking
   - Social proof

5. Checkout / Order Pages
   - Price anchoring
   - Guarantee language
   - Order bumps/upsells
```

## Execution Logic

### Step 1: Identify Funnel Entry Points

```
FROM market_config.competitors:

FOR each competitor:

  IDENTIFY entry points:
  - Homepage → find opt-in
  - Known funnel URLs
  - Ads (from ads library) → destination URLs
  - Search "[competitor] free training"
  - Search "[competitor] webinar"

  BUILD funnel_list[]
```

### Step 2: Scrape Landing/Opt-in Pages

```
TOOL:

mcp__firecrawl-mcp__firecrawl_scrape:
  url: [landing_page_url]
  formats: ["markdown"]
  onlyMainContent: true
  waitFor: 3000

EXTRACT:
- Main headline
- Subheadline
- Lead magnet description
- Bullet points/promises
- Form fields
- CTA button text
- Trust elements (logos, testimonials)
```

### Step 3: Scrape Sales Pages

```
TOOL:

mcp__firecrawl-mcp__firecrawl_scrape:
  url: [sales_page_url]
  formats: ["markdown"]
  onlyMainContent: false  # Get full page
  waitFor: 5000

EXTRACT:
- Full copy (preserve structure)
- All headlines (H1, H2, H3)
- Testimonial blocks
- Price/offer details
- Guarantee language
- PS sections
- Order button text
```

### Step 4: Handle VSL Pages

```
FOR VSL pages:

STEP 1: Scrape page content
  mcp__firecrawl-mcp__firecrawl_scrape:
    url: [vsl_page_url]
    formats: ["markdown"]

STEP 2: If video source identifiable
  - If YouTube/Vimeo hosted → use video scraper for transcript
  - If Wistia/custom → note, may require manual transcript

STEP 3: Scrape visible text elements
  - Headlines around video
  - Bullets/benefits shown
  - CTA elements
  - Trust badges

CAPTURE:
- Video duration
- Transcript (if available)
- Supporting page copy
- Buy button appearance timing (if detectable)
```

### Step 5: Map Funnel Flow

```
FOR each funnel:

  ATTEMPT to follow the flow:

  1. Entry point (ad/search)
  2. Opt-in page
  3. Thank you / video page
  4. Sales page or webinar
  5. Checkout page

  FOR each step:
    - Scrape page
    - Note purpose
    - Capture key action
    - Follow next step (if possible)

  BUILD funnel_flow map
```

### Step 6: Deep Copy Analysis

```
FOR each sales page:

EXTRACT mechanism_language:
  - Primary mechanism (the "how it works")
  - Supporting mechanisms
  - Proprietary terms/names
  - Framework language

EXTRACT proof_elements:
  - Testimonials (with specifics)
  - Statistics cited
  - Credentials/authority
  - Media mentions
  - Case study summaries

EXTRACT psychological_elements:
  - Urgency (deadlines, countdown)
  - Scarcity (limited spots, bonuses ending)
  - Objection handling ("You might be thinking...")
  - Risk reversal (guarantee language)
```

---

## Tool Resilience

```
FUNNEL SCRAPING FALLBACK:

Primary: Firecrawl
1. mcp__firecrawl-mcp__firecrawl_scrape
   - Best for full page content
   - Handles JavaScript rendering

Fallback: Apify RAG Browser
2. mcp__apify__apify-slash-rag-web-browser
   - Good for content extraction
   - Handles complex pages

Last Resort: Perplexity
3. mcp__perplexity__perplexity_search
   - Search for "[competitor] sales page [product]"
   - May find archived/cached versions

IF page blocks scraping:
- Try with mobile: true
- Try with proxy: "stealth"
- Note as "blocked, manual review needed"
```

---

## Special Handling

### Webinar Funnels

```
WEBINAR SPECIFIC:

Registration page:
- Event title/promise
- Speaker credentials
- Date/time elements
- Urgency language

Replay page (if accessible):
- Full presentation content
- Pitch timing
- Offer details
- Bonuses listed
```

### Product Launch Funnels

```
LAUNCH SPECIFIC:

Pre-launch content (PLC):
- Video series content
- Teaching + positioning
- Anticipation building

Cart open pages:
- Full offer stack
- Bonus structure
- Payment options
- Deadline elements
```

---

## Quality Criteria

1. **Completeness** - Capture full copy, not summaries
2. **Structure** - Preserve headline hierarchy
3. **Flow** - Map complete funnel journey
4. **Analysis** - Extract patterns, not just collect
