# Skill 1.3-A: Source Validator (Agent Critic)

**Version:** 2.0
**Layer:** 1 (Infrastructure)
**Type:** Validation & Quality Assurance
**Dependencies:** scored_sources.json, market_config.yaml, RESEARCH-PRD.md
**Output:** validated_sources.json, validation_report.md

---

## Purpose

Act as the Agent Critic for source discovery. Validate that the source list meets all PRD requirements before proceeding to scraping. Identify gaps, flag concerns, and either approve for scraping or mandate re-expansion.

---

## Agent Critic Role

```
CRITIC MINDSET:

You are the quality gatekeeper. Your job is to FIND PROBLEMS
before they cascade downstream. A source list that passes
your validation should yield 1,000+ high-quality quotes.

BE SKEPTICAL:
- Optimistic volume estimates often disappoint
- Low-priority sources rarely deliver
- Missing topics won't fix themselves
- Platform concentration = risk

APPROVE ONLY WHEN CONFIDENT.
```

## Validation Checks

### Check 1: Volume Thresholds

```
VOLUME VALIDATION:

Required (from PRD):
- Total sources: >= 100
- High priority sources: >= 30
- Medium priority sources: >= 40
- Estimated quotes from high+medium: >= 1,000

Current assessment:
- total_sources: [number]
- high_priority: [number]
- medium_priority: [number]
- estimated_quotes: [number]

Status: PASS / FAIL / WARNING
```

### Check 2: Platform Distribution

```
PLATFORM VALIDATION:

For each critical platform (from market_config):
- [ ] Has >= 10 high-priority sources
- [ ] Has >= 20 total sources
- [ ] Tool assigned and accessible

For each high platform:
- [ ] Has >= 5 high-priority sources
- [ ] Has >= 10 total sources

Platform concentration check:
- [ ] No single platform > 40% of sources
- [ ] At least 4 platforms with sources

Status: PASS / FAIL / WARNING
```

### Check 3: Topic Coverage

```
TOPIC VALIDATION:

For each research topic (from context expansion):
- [ ] Has >= 2 high-priority sources
- [ ] Has >= 5 total sources
- [ ] Has sources from multiple platforms

Bucket coverage (must have sources for):
- [ ] Pain/frustration topics
- [ ] Hope/aspiration topics
- [ ] Root cause topics
- [ ] Solution/attempt topics
- [ ] Competitor-related topics
- [ ] Emotional experience topics

Status: PASS / FAIL / WARNING
```

### Check 4: Content Type Diversity

```
CONTENT TYPE VALIDATION:

Required mix:
- [ ] Forum threads: >= 20 sources
- [ ] Video content: >= 15 sources
- [ ] Review content: >= 10 sources
- [ ] Discussion threads: >= 15 sources
- [ ] At least 4 content types represented

Status: PASS / FAIL / WARNING
```

### Check 5: Quote Bucket Projection

```
BUCKET PROJECTION:

Estimated yield by bucket:
- Pain quotes: [projected] / 300 required
- Hope quotes: [projected] / 250 required
- Root cause quotes: [projected] / 200 required
- Solutions tried quotes: [projected] / 150 required

Projection method:
- High-priority sources: 15 quotes/source average
- Medium-priority sources: 6 quotes/source average
- Low-priority sources: 2 quotes/source average

Apply topic-to-bucket mapping for projections.

Status: PASS / FAIL / WARNING
```

### Check 6: Tool Accessibility

```
TOOL ACCESSIBILITY VALIDATION:

For each platform in source list:
- [ ] Primary tool available
- [ ] Fallback tool identified
- [ ] No blocked platforms without alternatives

Blocked platform check:
- [ ] Reddit sources → Apify assigned (not Firecrawl)
- [ ] Rate-limited platforms → Appropriate pacing

Status: PASS / FAIL / WARNING
```

## Execution Logic

### Step 1: Load Requirements

```
LOAD:
1. scored_sources.json
2. market_config.yaml
3. RESEARCH-PRD.md (extract minimums)

BUILD validation matrix:
- All checks with pass/fail criteria
- Platform requirements from config
- Topic requirements from context
```

### Step 2: Execute All Checks

```
FOR each validation check:

  EVALUATE against criteria

  RECORD:
  - Status (pass/fail/warning)
  - Current value
  - Required value
  - Gap if any

  IF fail:
    ADD to gaps_identified
    SET severity based on magnitude
    GENERATE remediation suggestion
```

### Step 3: Calculate Confidence

```
CONFIDENCE CALCULATION:

High confidence (approved):
- All critical checks pass
- No more than 2 warnings
- Estimated quotes > 1,200

Medium confidence (approved with notes):
- All critical checks pass
- 3-4 warnings
- Estimated quotes 1,000-1,200

Low confidence (requires_expansion):
- 1+ critical checks fail
- 5+ warnings
- Estimated quotes < 1,000

Critical gaps:
- 3+ critical checks fail
- Multiple platforms missing
- Topic coverage < 50%
```

### Step 4: Generate Expansion Requirements

```
IF status != "approved":

  FOR each gap:

    IDENTIFY need_type:
    - "platform_sources" - need more from specific platform
    - "topic_sources" - need more for specific topic
    - "content_type" - need more of specific format
    - "bucket_coverage" - need more for specific bucket

    CALCULATE:
    - current: what we have
    - required: what PRD requires
    - delta: gap to fill

    GENERATE suggested_queries:
    - 3-5 new queries to fill gap
    - Platform-specific formatting
    - Variation on existing queries
```

### Step 5: Create Validation Report

```
GENERATE validation_report.md:

- Ready for scraping: [yes/no]

[For each gap...]

[If expansion needed...]

[Count by platform and priority...]

[Clear next step...]
```

---

## Decision Matrix

```
FINAL DECISION:

IF all critical pass AND warnings <= 2:
  status = "approved"
  action = "Proceed to scraping"

ELSE IF critical_failures == 1 AND easily_remediated:
  status = "requires_expansion"
  action = "Run additional discovery for [specific gaps]"

ELSE IF critical_failures >= 2:
  status = "critical_gaps"
  action = "Return to Query Generator, expand significantly"

NEVER proceed with status = "critical_gaps"
```

---

## Quality Criteria

1. **Thoroughness** - All validation checks executed
2. **Accuracy** - Projections based on realistic estimates
3. **Actionability** - Clear remediation for each gap
4. **Gatekeeper** - No approval without confidence
