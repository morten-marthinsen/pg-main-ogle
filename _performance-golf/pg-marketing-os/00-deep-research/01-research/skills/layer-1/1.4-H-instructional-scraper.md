# Skill 1.4-H: Instructional Content Scraper

**Version:** 2.0
**Layer:** 1 (Infrastructure)
**Type:** Content Extraction
**Dependencies:** validated_sources.json, market_config.yaml
**Output:** Appends to master_raw_content.json

---

## Purpose

Scrape instructional content: courses (public previews), tutorials, guides, how-to articles, and educational material. This content reveals market language, teaching frameworks, common mistakes, and solution approaches that resonate with customers.

---

## Target Content

```
INSTRUCTIONAL SOURCES:

1. Blog posts / How-to articles
   - Step-by-step guides
   - Tutorial content
   - Best practice articles

2. Course platforms (public content)
   - Udemy course descriptions + reviews
   - Skillshare previews
   - Course landing pages

3. Knowledge bases
   - FAQ sections
   - Help documentation
   - Troubleshooting guides

4. Podcasts / Audio content
   - Show notes
   - Episode transcripts
   - Guest interviews

5. Ebooks / Lead magnets (accessible)
   - Free PDF content
   - Sample chapters
   - Report summaries
```

---

## Input Schema

```yaml
input:
  validated_sources: validated_sources.json
  market_config: market_config.yaml
  filter:
    content_type: ["article", "tutorial", "course", "guide", "podcast"]
```

## Execution Logic

### Step 1: Identify Instructional Sources

```
FROM validated_sources.json:

SELECT sources WHERE:
- content_type IN ["article", "tutorial", "course", "guide"]
- NOT video (handled by video scraper)
- NOT review (handled by review scraper)

ADD additional sources:
- Top blog posts from market_config platforms
- Course platforms with market category
- FAQ/help pages from competitors
```

### Step 2: Scrape Articles and Guides

```
TOOL:

mcp__firecrawl-mcp__firecrawl_scrape:
  url: [article_url]
  formats: ["markdown"]
  onlyMainContent: true

EXTRACT:
- Full article text
- Section headings
- Lists and steps
- Author information
- Publication date

PRESERVE structure:
- Heading hierarchy
- Numbered lists (steps)
- Bullet points (tips)
```

### Step 3: Scrape Course Content

```
FOR course platforms (Udemy, Skillshare, etc.):

TOOL:

mcp__apify__call-actor:
  actor: "epctex/udemy-scraper"
  input:
    searchTerms: [market category terms]
    maxCourses: 50

OR Firecrawl:

mcp__firecrawl-mcp__firecrawl_scrape:
  url: [course_page_url]
  formats: ["markdown"]

EXTRACT:
- Course title
- Instructor name
- Rating and students
- Course description
- Curriculum outline
- Top reviews
- Promise language in headline
```

### Step 4: Scrape Podcast Show Notes

```
FOR podcast sources:

TOOL:

mcp__firecrawl-mcp__firecrawl_scrape:
  url: [podcast_episode_url]
  formats: ["markdown"]

EXTRACT:
- Episode title
- Guest information
- Show notes / summary
- Timestamps / topics covered
- Key takeaways

IF transcript available:
- Extract full transcript
- Identify key quotes
```

### Step 5: Teaching Element Extraction

```
FOR each piece of content:

IDENTIFY frameworks_presented:
- Named methodologies ("The 5-Step Process")
- Numbered approaches ("3 Keys to...")
- Proprietary systems
- Acronyms (SMART, etc.)

IDENTIFY steps_outlined:
- Sequential instructions
- Phase breakdowns
- Before/during/after structures

IDENTIFY common_mistakes:
- "Don't do this..."
- "The biggest mistake is..."
- "Most people fail because..."
- "Avoid..."

IDENTIFY tips_and_advice:
- "Pro tip:"
- "The key is..."
- "What works best..."
- "My recommendation..."
```

### Step 6: Language Pattern Analysis

```
ANALYZE language across content:

TERMINOLOGY:
- Industry-specific terms
- Technical language
- Commonly used phrases

METAPHORS:
- How concepts are explained
- Comparisons made
- Abstract → concrete translations

ANALOGIES:
- "It's like..."
- "Think of it as..."
- "Similar to..."

JARGON:
- Insider terms
- Assumed knowledge
- Acronyms explained
```

---

## Framework Inventory

```
ACROSS all instructional content:

COMPILE unique frameworks:

FOR each framework:
  RECORD:
  - Framework name
  - Source/creator
  - Brief description
  - Steps/components
  - How many times seen

CATEGORIZE:
- Mindset frameworks
- Technical frameworks
- Process frameworks
- Diagnostic frameworks

FLAG:
- Universal frameworks (used by many)
- Unique frameworks (differentiation)
```

---

## Tool Resilience

```
INSTRUCTIONAL SCRAPING FALLBACK:

Articles/Blogs:
1. Firecrawl scrape
2. Apify RAG browser
3. Perplexity search

Courses:
1. epctex/udemy-scraper (Udemy)
2. Firecrawl on course pages
3. Perplexity "[course name] review curriculum"

Podcasts:
1. Firecrawl on show notes
2. Perplexity search for transcripts
3. Apple/Spotify page scraping

IF content paywalled:
- Capture available preview
- Note limitation
- Use course reviews for insight
```

---

## Quality Criteria

1. **Depth** - Full content, not summaries
2. **Structure** - Preserve teaching organization
3. **Attribution** - Track sources and authors
4. **Pattern Recognition** - Identify repeated frameworks
