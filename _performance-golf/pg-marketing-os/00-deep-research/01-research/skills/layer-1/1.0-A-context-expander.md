# Skill 1.0-A: Context Expander

**Version:** 2.0
**Layer:** 1 (Infrastructure)
**Type:** Research Scope Generator
**Dependencies:** market_config.yaml, research brief
**Output:** context_expansion.json

---

## Purpose

Expand the product/market context into a comprehensive research scope BEFORE any scraping begins. This prevents narrow research that misses critical market psychology by ensuring we investigate all relevant topics, not just the obvious ones.

---

## Input Schema

```yaml
input:
  market_config: market_config.yaml
  brief:
    product:
      name: string
      category: string
      description: string
      known_benefits: array
    context:
      why_now: string
      known_gaps: array
```

---

## Output Schema

```yaml
output:
  context_expansion:
    metadata:
      product_name: string
      category: string
      context: string
      generated_date: string

    primary_research_topics:
      - topic: string
        description: string
        why_important: string
        expected_sources: array

    category_specific_topics:
      - topic: string
        description: string
        unique_to_category: boolean

    competitor_context_topics:
      - topic: string
        competitors_to_analyze: array
        angles: array

    emotional_psychological_topics:
      - topic: string
        emotion_type: string (fear/frustration/hope/identity)
        trigger_patterns: array

    source_type_calibration:
      - source_type: string
        priority: string (critical/high/moderate)
        expected_content: string
        tool: string

    validation:
      primary_topics_count: number
      category_topics_count: number
      competitor_topics_count: number
      emotional_topics_count: number
      source_types_count: number
      all_minimums_met: boolean
```

## Execution Logic

### Step 1: Understand the Product Context

```
CONTEXT UNDERSTANDING:

1. Read product description and benefits
2. Identify:
   - CATEGORY: What type of product is this?
   - CONTEXT: What broader domain does this sit in?
   - PROBLEM SPACE: What problems does the market face?
   - SOLUTION SPACE: What solutions exist?

Example:
  Product: "AI-powered email deliverability platform"
  CATEGORY: SaaS / Email Marketing
  CONTEXT: Email Deliverability / Sender Reputation
  PROBLEM SPACE: Emails landing in spam, low open rates, domain blacklisting
  SOLUTION SPACE: Warmup tools, authentication, list hygiene, content optimization
```

### Step 2: Generate Primary Research Topics (10+ Required)

```
PRIMARY TOPIC GENERATION:

**Ruminate** on all angles of the problem space:

1. THE CORE PROBLEM
   - What is the primary pain point?
   - How do people describe it?
   - What variations exist?

2. RELATED PROBLEMS
   - What problems co-occur?
   - What problems lead to this one?
   - What problems result from this one?

3. ATTEMPTED SOLUTIONS
   - What has the market tried?
   - Why did those fail?
   - What do they think will work?

4. ROOT CAUSES
   - What do people believe causes the problem?
   - Are there competing theories?
   - What misconceptions exist?

5. DESIRED OUTCOMES
   - What does success look like?
   - What milestones matter?
   - What would life be like if solved?

6. INSTRUCTIONAL CONTENT
   - What are people trying to learn?
   - What skills are they developing?
   - What concepts are they struggling with?

7. COMPARISON SHOPPING
   - How do people compare options?
   - What criteria matter most?
   - What tradeoffs do they consider?

8. TIMING TRIGGERS
   - When does this problem become urgent?
   - What events trigger buying?
   - Seasonal patterns?

9. SOCIAL DYNAMICS
   - Who influences decisions?
   - What do peers think?
   - Status implications?

10. FINANCIAL CONSIDERATIONS
    - What have they spent?
    - What's their budget?
    - ROI expectations?

For each topic, document:
- topic: Name
- description: What this covers
- why_important: Why we need to research this
- expected_sources: Where to find this content
```

### Step 3: Generate Category-Specific Topics (5+ Required)

```
CATEGORY-SPECIFIC TOPICS:

Based on market_config.aspects, generate topics unique to this category:

Example for Trading:
- Backtesting expectations and frustrations
- Broker/platform comparisons
- Indicator vs price action debates
- Prop firm experiences
- Tax implications and concerns

Example for Health/Supplements:
- Side effect experiences
- Stacking protocols
- Blood test discussions
- Long-term usage concerns
- Interaction worries

For each topic:
- topic: Name
- description: What this covers
- unique_to_category: true
```

### Step 4: Generate Competitor Context Topics (5+ Required)

```
COMPETITOR CONTEXT:

Using market_config.competitors, generate research angles:

1. DIRECT COMPETITOR ANALYSIS
   - How are competitors positioning?
   - What claims are they making?
   - What mechanisms do they advertise?

2. COMPETITOR COMPLAINTS
   - What do customers hate about competitors?
   - What problems remain after using competitors?
   - What's missing from competitor solutions?

3. COMPETITOR COMPARISONS
   - How do prospects compare options?
   - What are the decision criteria?
   - Who wins in head-to-head?

4. COMPETITOR MESSAGING
   - What promises are they making?
   - What root causes do they claim?
   - What proof are they offering?

5. MARKET SATURATION
   - Which claims are overused?
   - Which mechanisms are tired?
   - Where is fresh territory?

For each topic:
- topic: Name
- competitors_to_analyze: [list]
- angles: [specific questions to answer]
```

### Step 5: Generate Emotional/Psychological Topics (5+ Required)

```
EMOTIONAL TOPICS:

Using market_config.emotional_context, ensure we research:

FEAR TOPICS:
- What keeps them up at night?
- What worst-case scenarios do they imagine?
- What failures haunt them?

FRUSTRATION TOPICS:
- What makes them angry about the problem?
- What's unfair about their situation?
- What excuses do they reject?

HOPE TOPICS:
- What transformation do they dream of?
- What would life look like if solved?
- What stories inspire them?

IDENTITY TOPICS:
- How does this problem affect how they see themselves?
- What identity do they aspire to?
- What identity do they reject?

EMBARRASSMENT TOPICS:
- What are they ashamed of?
- What won't they admit publicly?
- What do they hide from others?

For each topic:
- topic: Name
- emotion_type: fear/frustration/hope/identity/embarrassment
- trigger_patterns: [phrases that reveal this emotion]
```

### Step 6: Calibrate Source Types (6+ Required)

```
SOURCE CALIBRATION:

Based on topics and market_config.platforms:

For each source type, define:
- source_type: Name (e.g., "Forums", "YouTube Comments")
- priority: critical/high/moderate
- expected_content: What we'll find here
- tool: firecrawl/apify/perplexity
- specific_targets: [URLs, subreddits, channels]

Required source types:
1. Forums (problem discussions, peer advice)
2. Video Comments (unfiltered reactions, emotional language)
3. Review Sites (product feedback, comparison language)
4. Competitor Pages (messaging, claims, mechanisms)
5. Social Media (real-time sentiment, identity signals)
6. Instructional Content (what they're trying to learn)

Optional but valuable:
7. Q&A Sites (specific questions, confusion points)
8. Ad Libraries (competitor messaging, saturation)
9. Podcast Comments (engaged audience language)
```

### Step 7: Validate Expansion

```
VALIDATION:

Count and verify:
- [ ] primary_research_topics >= 10
- [ ] category_specific_topics >= 5
- [ ] competitor_context_topics >= 5
- [ ] emotional_psychological_topics >= 5
- [ ] source_type_calibration >= 6

- [ ] Emotional topics include fear AND hope
- [ ] Competitor topics cover messaging AND complaints
- [ ] Source types include forums AND video AND reviews

IF ANY validation fails:
  → Expand the deficient category
  → DO NOT proceed without meeting minimums
  → Log: "Context expansion validation failed: [specific failures]"
```

---

## Quality Criteria

1. **Comprehensive** - No obvious angle missed
2. **Specific** - Topics are actionable, not vague
3. **Market-Appropriate** - Uses terminology from market_config
4. **Source-Mapped** - Each topic has clear sources
5. **Emotion-Inclusive** - Psychological dimensions covered

---

## Anti-Patterns to Avoid

```
BAD: Only researching the product category
GOOD: Researching the entire problem/solution ecosystem

BAD: Generic topics like "customer pain points"
GOOD: Specific topics like "frustration with [specific issue]"

BAD: Skipping emotional topics because they're "soft"
GOOD: Prioritizing emotional topics because they drive decisions

BAD: 3-4 topics because "that seems like enough"
GOOD: 25+ topics because markets are complex
```

---

## Error Handling

| Error | Response |
|-------|----------|
| market_config missing | Cannot proceed - request config generation |
| Fewer than 10 primary topics | Expand using rumination prompts |
| No emotional topics | Explicitly add fear/hope/frustration angles |
| Source types < 6 | Add missing source types from universal list |
