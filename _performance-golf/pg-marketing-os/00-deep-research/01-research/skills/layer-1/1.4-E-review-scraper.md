# Skill 1.4-E: Review Scraper

**Version:** 2.0
**Layer:** 1 (Infrastructure)
**Type:** Content Extraction
**Dependencies:** validated_sources.json, market_config.yaml
**Output:** Appends to master_raw_content.json

---

## Purpose

Scrape product and service reviews from Amazon, Google, Trustpilot, G2, Capterra, and industry-specific review sites. Reviews are gold mines for competitor analysis, solution attempts, and honest assessments.

---

## Target Platforms

```
REVIEW PLATFORMS:

General:
- Amazon product reviews
- Google reviews
- Trustpilot
- BBB complaints

Software/Services:
- G2
- Capterra
- TrustRadius
- Software Advice

Industry-Specific:
- Market-dependent (identify from market_config)
```

## Execution Logic

### Step 1: Identify Review Targets

```
BUILD review target list:

FROM validated_sources (content_type = "review"):
- Extract product/service URLs

FROM market_config.competitors:
- Search for their review pages
- Add to target list

FROM market_config (category-specific):
- Add category bestsellers
- Add "vs" comparison products

DEDUPLICATE by product identifier
```

### Step 2: Scrape by Platform

```
FOR each review target:

  IDENTIFY platform from URL

  SELECT appropriate tool

  EXECUTE scrape:
  - Get up to 500 reviews per product
  - Sort by most recent or most helpful
  - Include all rating levels

  CAPTURE:
  - Full review text
  - Rating
  - Metadata
```

### Step 3: Rating Analysis

```
FOR each product:

  CALCULATE rating_distribution:
  - Count reviews by star level
  - Calculate percentages
  - Identify rating trends

  FLAG anomalies:
  - Suspiciously high 5-star %
  - Recent rating drops
  - Response patterns
```

### Step 4: Quote Pre-Extraction

```
PRIORITIZE reviews for quote extraction:

HIGHEST VALUE:
- 1-star and 2-star reviews (pain, complaints)
- 5-star reviews (hope, transformation)
- Long-form reviews (detail, specifics)
- Reviews with many helpful votes

EXTRACT quotes by sentiment:

NEGATIVE (pain/complaint):
- "Doesn't work..."
- "Waste of money..."
- "Expected more..."
- "The problem is..."

POSITIVE (hope/transformation):
- "Finally found..."
- "Changed my..."
- "Best decision..."
- "Unlike others..."

MIXED (nuanced insight):
- "Good for X but not Y..."
- "Would be better if..."
- "Compared to..."
```

### Step 5: Competitor Analysis

```
FOR each competitor product:

  AGGREGATE:
  - Average rating
  - Rating distribution
  - Review volume

  IDENTIFY PATTERNS:

  Common complaints (from 1-3 star):
  - Technical issues
  - Unmet promises
  - Support problems
  - Value concerns

  Common praises (from 4-5 star):
  - Features appreciated
  - Results achieved
  - Unique value props

  Mechanisms mentioned:
  - How product is described
  - What it claims to do
  - Technical terms used
```

## Tool Resilience

```
REVIEW SCRAPING FALLBACK:

Amazon:
1. junglee/amazon-reviews-scraper
2. epctex/amazon-reviews-scraper
3. Firecrawl (limited)

Google:
1. compass/google-maps-reviews-scraper
2. crawlee/google-places-scraper

Trustpilot:
1. emastra/trustpilot-scraper
2. Firecrawl scrape

G2/Capterra:
1. Firecrawl scrape
2. Perplexity search

IF tool fails:
- Log error
- Try fallback
- Accept partial data
- Note limitation in output
```

---

## Quality Criteria

1. **Diversity** - Include all star ratings (not just negative/positive)
2. **Recency** - Prioritize reviews from last 24 months
3. **Depth** - Long-form reviews over short ratings
4. **Verification** - Note verified vs unverified purchases
