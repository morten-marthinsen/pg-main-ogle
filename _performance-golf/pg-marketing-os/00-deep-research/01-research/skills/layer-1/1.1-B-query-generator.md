# Skill 1.1-B: Query Generator

**Version:** 3.0
**Layer:** 1 (Infrastructure)
**Type:** Search Query Factory
**Dependencies:** market_config.yaml, context_expansion.json, platform_list.json, awareness_baseline.json
**Output:** queries.json

---

## Purpose

Generate comprehensive search queries that will surface relevant content across all research topics. Queries are the net we cast - wider nets catch more fish.

---

## CRITICAL: MODEL + PERSONA + VOLUME AWARENESS (v3.2 ADDITION)

**Designated model: `sonnet`** — Query generation requires structured output, not deep reasoning.
**Designated persona: `PERSONA_QUERY_BUILDER`** — See MASTER-AGENT.md Subagent Persona Library.

**Queries must be sufficient to yield 1,000+ verbatim quotes across 6 buckets.** The 50-query minimum is a FLOOR, not a target. In most markets, 50 queries will NOT yield 1,000 quotes. Generate queries aggressively.

**Bucket-aware query generation:** Ensure query distribution covers ALL 6 buckets:
- Pain (target 300 quotes) → Generate 15+ pain-focused queries
- Hope (target 250 quotes) → Generate 12+ hope/transformation queries
- Root Cause (target 200 quotes) → Generate 10+ "why" queries
- Solutions Tried (target 150 quotes) → Generate 8+ "tried X" / "reviewed X" queries
- Competitor Mechanism (target 100 quotes) → Generate 8+ competitor-specific queries
- Villain (target 75 quotes) → Generate 5+ "hate" / "scam" / "disappointed" queries

**If any bucket has fewer than 5 queries, that bucket WILL fall short.** The orchestrator cannot fix a query deficit with expansion — it can only expand what was started.

## Input Schema

```yaml
input:
  market_config: market_config.yaml (terminology, query_patterns)
  context_expansion: context_expansion.json (all research topics)
  platform_list: platform_list.json (platforms with specific targets)
  awareness_baseline: awareness_baseline.json (level distribution for weighting)
```

## Execution Logic

### Step 1: Load Query Patterns

```
LOAD FROM market_config.query_patterns:
- pain_queries templates
- hope_queries templates
- root_cause_queries templates
- solution_queries templates
- competitor_queries templates
- emotional_queries templates

LOAD TERMINOLOGY:
- customer terms (primary, plural)
- problem terms
- solution terms
- competitor names
```

### Step 2: Generate Queries Per Topic

```
FOR each topic in context_expansion:

  1. IDENTIFY topic type:
     - Problem-related → use pain_queries + root_cause_queries
     - Solution-related → use solution_queries + hope_queries
     - Competitor-related → use competitor_queries
     - Emotional-related → use emotional_queries

  2. APPLY templates with terminology:
     Replace placeholders:
     - {problem} → market_config.terminology.problem.primary
     - {customer_plural} → market_config.terminology.customer.plural
     - {solution_category} → market_config.terminology.product.solution
     - {competitor} → each known competitor name
     - {topic} → topic-specific terms

  3. GENERATE variations:
     - Base query
     - With "reddit" appended (for Reddit targeting)
     - With "forum" appended (for forum targeting)
     - With question format ("why...", "how...")

  4. MINIMUM: 5 queries per major topic
```

### Step 3: Generate Platform-Specific Queries

```
FOR each platform in platform_list:

  IF platform is Reddit:
    - Add "site:reddit.com" queries
    - Add subreddit-specific queries

  IF platform is YouTube:
    - Generate video search queries
    - Include "[topic] tutorial", "[topic] review"

  IF platform is Twitter/X:
    - Shorter phrases (tweet-length awareness)
    - Hashtag-enhanced versions of all queries
    - Add sentiment operators for emotional content
    - Include thought leader handles: "@[expert_handle] [topic]"
    - Include engagement filters where supported: "min_replies:5", "min_faves:10"
    - Example pain: "#trading frustrated" OR "trading killing my business"
    - Example hope: "#trading breakthrough" OR "finally figured out trading"
    - Example fear: "#AI replacing" OR "[topic] worried" OR "falling behind"
    - Example urgency: "last chance [topic]" OR "window closing" OR "now or never"
    - Example competitor: "@[competitor] review" OR "[competitor] vs"

  IF platform is forum:
    - Generate forum-appropriate phrasing
    - Include common forum terms ("advice", "help", "anyone else")
```

### Step 4: Weight by Awareness Level

```
AWARENESS WEIGHTING (from awareness_baseline):

Based on distribution, adjust query count:

IF Level 5 > 20%:
  - Increase competitor comparison queries
  - Add price/offer queries

IF Level 3-4 > 50%:
  - Increase solution exploration queries
  - Add mechanism/method queries

IF Level 2 > 20%:
  - Increase problem articulation queries
  - Add symptom/frustration queries

IF Level 1 > 10%:
  - Add identity/aspirational queries
  - Add symptom awareness queries
```

### Step 5: Generate Emotional Queries

```
EMOTIONAL QUERY TYPES:

Fear queries:
- "[topic] scared"
- "[topic] worried about"
- "afraid of [problem]"
- "[problem] anxiety"

Frustration queries:
- "[topic] frustrated"
- "tired of [problem]"
- "[problem] driving me crazy"
- "nothing works for [problem]"

Hope queries:
- "finally [desired outcome]"
- "[topic] success story"
- "breakthrough [topic]"
- "transformed my [domain]"

Embarrassment queries:
- "[topic] embarrassing"
- "ashamed of [problem]"
- "don't want anyone to know"

MINIMUM: 10 emotional queries across types
```

### Step 6: Generate Competitor Queries

```
FOR each competitor in market_config.competitors:

  Generate:
  - "[competitor] review"
  - "[competitor] honest review"
  - "[competitor] worth it"
  - "[competitor] scam" (reveals skepticism)
  - "[competitor] vs [other competitor]"
  - "[competitor] complaints"
  - "[competitor] refund"
  - "alternative to [competitor]"
  - "[competitor] results"
  - "after [competitor]" (post-purchase experience)

MINIMUM: 5 queries per major competitor
```

### Step 7: Validate Query Coverage

```
VALIDATION:

- [ ] Total queries >= 50
- [ ] Every primary research topic has >= 5 queries
- [ ] Every category-specific topic has >= 3 queries
- [ ] Every competitor has >= 5 queries
- [ ] All query types represented (pain, hope, root_cause, solution, competitor, emotional)
- [ ] Emotional queries >= 10
- [ ] Platform-specific queries exist for each critical platform

IF validation fails:
  - Identify gaps
  - Generate additional queries
  - DO NOT proceed with insufficient queries
```

---

## Quality Criteria

1. **Comprehensive** - No topic without queries
2. **Varied** - Multiple query types per topic
3. **Platform-Aware** - Queries tailored to platform conventions
4. **Emotionally Rich** - Strong emotional query representation
5. **Competitor-Complete** - All competitors covered

---

## Anti-Patterns to Avoid

```
BAD: 10 generic queries for entire research
GOOD: 50+ specific queries across all topics

BAD: Only "[topic] tips" style queries
GOOD: Pain, hope, frustration, comparison, and emotional queries

BAD: Forgetting competitor queries
GOOD: 5+ queries per major competitor

BAD: Ignoring emotional language
GOOD: Explicit fear, frustration, hope, embarrassment queries
```
