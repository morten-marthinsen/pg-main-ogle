# 3.2-B: Audience Agent Constructor

**Purpose:** Construct audience agent personas from research data for deployment as Arena feedback agents.
**Runs after:** 3.2-A (Handoff Packager) — requires all Layer 1, 2, 2.5, 2.8 outputs
**Produces:** `audience-agent-personas.json`
**Reference:** `~system/protocols/AUDIENCE-AGENT-PROTOCOL.md` for full schema and evaluation framework

---

## WHEN TO RUN

This microskill runs during Layer 3 (Handoff Assembly), after ALL analysis layers are complete. It requires:

- Audience segments from Layer 2 (2.2-A)
- Belief inventory from Layer 2 (2.2-B)
- Now-After grid from Layer 2 (2.2-C)
- Scored VOC quotes from Layer 1 (1.5-D)
- Voice of customer analysis from Layer 2.5 (2.5-E)
- FSSIT candidates from Layer 2.8 (2.8-B)
- Identity tensions from Layer 2.8 (2.8-B)
- Expectation schema from Layer 2.8 (2.8-A)
- Sophistication analysis from Layer 2 (2.55-A)
- Solutions tried data from Layer 1 quote corpus

If ANY required input is missing, HALT and report which input is absent.

## CONSTRUCTION PROCEDURE

### Step 1: Segment Selection

1. Read `audience_segments.json` (or equivalent segment output from 2.2-A)
2. Count total segments identified
3. Select segments for persona construction:
   - **Full tier:** ALL segments (typically 5-7)
   - **Standard tier:** Top 3 by strategic importance
4. For Standard tier, apply selection criteria:
   - Segment 1: **Primary buyer** — largest market share or highest purchase intent
   - Segment 2: **Hardest to convert** — most skeptical, highest resistance
   - Segment 3: **Most emotionally driven** — highest emotional intensity scores
5. Document selection rationale in output

### Step 2: Demographic + Psychographic Foundation

For EACH selected segment:

1. Extract from segment data:
   - `demographics`: age_range, gender, income_range, family_structure, location_type
   - `psychographics`: values (list), lifestyle_description, identity_markers (list), media_consumption (list)

2. Extract from emotional analysis:
   - `primary_fear`: description + intensity (1-10) + source_quote_id
   - `primary_frustration`: description + intensity + source_quote_id
   - `primary_hope`: description + intensity + source_quote_id
   - `secondary_emotions`: list of emotion + intensity pairs

3. Cross-validate: emotional state must be supported by VOC quotes from THIS segment. If the segment's demographic doesn't match any quotes, flag the gap.

### Step 3: Belief System Mapping

For EACH persona:

1. Read `belief_inventory.json` from Layer 2 (2.2-B)
2. Map beliefs to this persona's segment:
   - Which WHY beliefs does THIS segment hold most strongly?
   - Which WHAT beliefs reflect THIS segment's experience?
   - Which WHO beliefs match THIS segment's trust/distrust patterns?
   - Which HOW beliefs align with THIS segment's solution-seeking behavior?
3. Each belief entry includes:
   - `belief`: exact wording (use customer language, not analytical)
   - `alignable`: boolean — can copy build on this belief (true) or must it be reframed (false)?
   - `strength`: 1-10 — how strongly this persona holds this belief
4. Minimum: 3 beliefs per category (12 total per persona)
5. Prioritize beliefs that create the most TENSION with the product's mechanism

### Step 4: Voice Profile Construction

For EACH persona:

1. Read scored VOC quotes from Layer 1 corpus
2. Filter quotes relevant to this segment (by source demographics, topic alignment, language patterns)
3. Select 10-15 verbatim quotes:
   - Prioritize: highest `emotional_intensity` + `specificity` scores
   - Include at least 2 quotes from EACH bucket (pain, hope, root cause, solutions tried, villain)
   - DO NOT paraphrase — these must be VERBATIM customer quotes
4. Extract from `voice_of_customer_analysis.md`:
   - `metaphors_used`: metaphors and analogies this segment uses
   - `speech_patterns`: how they talk (formal vs. casual, direct vs. indirect)
   - `sophistication_markers`: terminology they know and use
   - `skepticism_triggers`: specific claims or patterns that trigger distrust
   - `trust_signals`: what makes them believe a claim

### Step 5: Purchase History

For EACH persona:

1. Extract from Solutions Tried bucket:
   - 2-5 specific solutions this segment has tried
   - For each: what it was, outcome (failed/partial/abandoned), emotional residue
2. Extract:
   - `spending_willingness`: what price range they're comfortable with
   - `purchase_triggers`: what would push them to buy
   - `purchase_barriers`: what stops them from buying (including social/relational barriers)

### Step 6: Identity Tensions

For EACH persona:

1. Read identity tension data from Layer 2.8 (2.8-B)
2. Map 1-3 identity tensions to this segment:
   - `current_identity`: who they are now (in their own words)
   - `desired_identity`: who they want to be (in their own words)
   - `strength`: 1-10 from research data
3. Use CUSTOMER LANGUAGE for identity descriptions, not analytical labels

### Step 7: FSSIT Conditions (Critical Step)

For EACH persona:

1. Read FSSIT candidates from Layer 2.8 (2.8-B)
2. Identify which FSSIT candidates are most relevant to this segment
3. Translate FSSIT candidates into EXPERIENTIAL CONDITIONS:
   - DO NOT write: "You carry a latent fear of irrelevance"
   - DO write: "You've spent thousands on golf instruction over 15 years and your handicap hasn't moved"
   - The condition CREATES the emotion without NAMING it
4. Map recognition triggers:
   - Situations where the latent emotion surfaces as behavior or deflection
   - "When someone describes exactly how the range swing disappears on the course"
5. 3-5 experiential conditions per persona
6. 2-4 recognition triggers per persona

### Step 8: First-Person Voice

For EACH persona:

1. Write a 2-3 sentence introduction IN THE PERSONA'S OWN VOICE
2. This introduction must:
   - Use vocabulary from the VOC quotes (not marketing language)
   - Establish their situation, frustration, and stance
   - Sound like a real person talking, not a character description
   - Reflect their sophistication level and emotional state
3. This becomes the `first_person_voice` field — the persona's self-description

### Step 9: Assembly + Validation

1. Assemble all fields into the `audience-agent-personas.json` schema (see AUDIENCE-AGENT-PROTOCOL.md)
2. Validate:
   - [ ] `persona_count` matches tier requirement (5-7 Full, 3 Standard)
   - [ ] Each persona has 10+ verbatim VOC quotes
   - [ ] Each persona has 12+ beliefs (3 per category minimum)
   - [ ] Each persona has at least 1 identity tension with strength score
   - [ ] Each persona has 3+ FSSIT experiential conditions
   - [ ] Each persona has `first_person_voice` written in customer vocabulary
   - [ ] No FSSIT conditions name emotions directly (only describe situations)
   - [ ] All source_quote_ids reference real quotes in the corpus
3. Write validated file to `~outputs/[project-code]/00-research/audience-agent-personas.json`

## ANTI-DEGRADATION

### Forbidden Shortcuts

- Generating persona data without reading the actual research outputs
- Using generic psychographic descriptions instead of research-specific data
- Paraphrasing VOC quotes instead of using verbatim text
- Naming latent emotions in FSSIT conditions
- Creating personas that all sound the same
- Skipping belief system mapping
- Using analytical language in `first_person_voice`

### Quality Gate

This microskill PASSES only when:
1. All validation checks in Step 9 pass
2. Each persona is demonstrably distinct from the others (different beliefs, different fears, different voice)
3. A spot-check of 3 random VOC quotes confirms they exist in the Layer 1 corpus
4. FSSIT conditions describe situations, not emotions
