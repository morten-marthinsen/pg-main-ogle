## BINARY GATE ENFORCEMENT

Gates are **PASS** or **FAIL**. No other values exist.

- "Conditional pass" = DOES NOT EXIST
- "Partial pass" = DOES NOT EXIST
- "Pass with caveats" = DOES NOT EXIST

**If a gate cannot be passed, the gate status is FAIL and the skill does not execute.**

A gate checkpoint file (`S[XX]_COMPLETE.yaml`) can only be created when the gate status is PASS. Any other value means the file should NOT exist.

| Skill | Failure Mode | Detection | Response |
