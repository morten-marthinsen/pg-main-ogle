# 1.1 Network Calendar Sync

## Purpose
Synchronize posting schedules across all personas to avoid conflicts, maximize coverage, and create momentum. Design coordination that looks organic, not managed.

## Model Assignment
**Model:** claude-opus-4-20250514

## Input Requirements
- Upstream context summary (existing posting schedules per persona)
- Account Plans (content calendars from S22)
- Teaching summary (coordination frameworks)

## Execution Protocol

### MUST
- Map all existing posting schedules on unified calendar view
- Identify posting conflicts (multiple personas posting simultaneously)
- Resolve conflicts by spreading posting times across day/week
- Ensure 24-hour coverage without obvious gaps
- Create posting momentum (strategic clustering for impact, then breathing room)
- Verify maximum 30% of daily posts occur in same 2-hour window
- Design variation patterns (not every Monday same persona posts)
- Consider timezone considerations if network has geographic spread

### NEVER
- Create simultaneous posting patterns (obvious coordination signal)
- Allow obvious gaps in coverage (reduces network presence)
- Design predictable patterns (e.g., Persona A always posts 9am Mon/Wed/Fri)
- Exceed 30% daily posts in same window
- Ignore timezone spread if network is geographically distributed

## Output
**Filename:** `network-calendar-sync.md`

**Format:** Markdown synchronized calendar

**Minimum Size:** 5KB

**Required Sections:**
1. Unified Network Calendar (all personas, all platforms, week view minimum)
2. Conflict Resolution (conflicts identified and how resolved)
3. Coverage Analysis (24-hour distribution, gap identification)
4. Momentum Design (strategic clustering patterns)
5. Variation Patterns (how schedules vary week-to-week)
6. Posting Concentration Check (% of posts in peak 2-hour windows)

## Quality Gates
- [ ] All posting schedules mapped on unified calendar
- [ ] No simultaneous posting within 30-minute windows
- [ ] 24-hour coverage with no >6-hour gaps
- [ ] Maximum 30% concentration in any 2-hour window
- [ ] Variation patterns prevent predictable schedules
- [ ] Coverage analysis confirms strategic momentum design

## Handoff
Feeds into: 1.2 Engagement Choreography (synchronized calendar enables strategic engagement)
