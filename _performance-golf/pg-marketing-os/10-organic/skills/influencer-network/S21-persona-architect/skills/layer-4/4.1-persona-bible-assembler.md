# 4.1 Persona Bible Assembler

## Purpose
Assemble complete Persona Bible for each persona. One file per persona containing all Phase A + Phase B content integrated into comprehensive reference document. Minimum 30KB per persona.

## Model Assignment
**Model:** claude-3-5-sonnet-20241022

## Input Requirements
- All Layer 1 outputs (1.1 through 1.9)
- Niche ownership, archetype, identity, backstory, personality, naming, visual, voice, content style
- Teaching summary (for structural framework)

## Execution Protocol

### MUST
- Create one Persona Bible file per persona
- Include 10 required sections per Bible: (1) Overview, (2) Niche Ownership, (3) Archetype, (4) Identity, (5) Backstory, (6) Personality, (7) Name & Handles, (8) Visual Identity, (9) Voice, (10) Content Style
- Integrate all Layer 1 content (not just copy-paste, but synthesize into cohesive narrative)
- Ensure each Bible is minimum 30KB (comprehensive reference)
- Create internal consistency (all sections align and reinforce each other)
- Include "Quick Reference" section at top with key facts
- Add "Do's and Don'ts" section for content creators

### NEVER
- Create Bibles under 30KB (indicates insufficient detail)
- Copy-paste Layer 1 files without integration
- Create structural inconsistencies across Bibles
- Skip Quick Reference or Do's/Don'ts sections
- Omit any of the 10 required sections

## Output
**Filename:** `persona-bible-[persona-name].md` (one file per persona)

**Format:** Markdown comprehensive persona reference

**Minimum Size:** 30KB per file

**Required Sections (10):**
1. Quick Reference (one-page summary)
2. Niche Ownership (what they cover, boundaries)
3. Archetype (primary type, secondary traits, implications)
4. Identity (age, background, worldview, communication style)
5. Backstory (origin, credentials, journey, perspective)
6. Personality (traits, values, quirks, opinions, pet peeves)
7. Name & Handles (selected name, rationale, social handles)
8. Visual Identity (photo direction, colors, typography, mood)
9. Voice (tone, vocabulary, signature phrases, forbidden phrases)
10. Content Style (formats, rhythm, signature types, series)
11. Do's and Don'ts (guardrails for content creators)

## Quality Gates
- [ ] One complete Bible per persona created
- [ ] Each Bible minimum 30KB
- [ ] All 10 required sections present
- [ ] Quick Reference and Do's/Don'ts included
- [ ] Internal consistency verified (sections align)

## Handoff
Feeds into: 4.2 Network Overview (Bibles provide input for network-level analysis)
