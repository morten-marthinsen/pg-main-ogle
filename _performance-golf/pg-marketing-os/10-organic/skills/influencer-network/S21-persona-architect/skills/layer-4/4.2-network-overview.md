# 4.2 Network Overview

## Purpose
Create network overview showing how personas work together as a system. Includes differentiation matrix (how personas differ), relationship map (how they reference each other), coverage map (which audience segments each owns).

## Model Assignment
**Model:** claude-3-5-sonnet-20241022

## Input Requirements
- All Persona Bibles (from 4.1)
- Niche ownership map (from 1.1)
- Upstream context (audience segments from AIF)

## Execution Protocol

### MUST
- Create differentiation matrix: table showing how personas differ across 10+ dimensions (niche, archetype, age, voice, visual, content style, etc.)
- Build relationship map: define how personas reference/acknowledge each other without breaking brand firewall
- Design coverage map: show which audience segments each persona serves, verify 100% coverage
- Document network dynamics: how personas complement (not compete), when they collaborate, how they amplify
- Identify potential conflicts and mitigation strategies
- Create network-level brand firewall rules (how to keep parent brand hidden)

### NEVER
- Create overview without clear differentiation verification
- Skip relationship mapping (critical for network coordination)
- Allow audience coverage gaps
- Ignore potential persona conflicts
- Forget brand firewall considerations

## Output
**Filename:** `network-overview.md`

**Format:** Markdown network analysis document

**Minimum Size:** 8KB

**Required Sections:**
1. Network Summary (overview of all personas and their roles)
2. Differentiation Matrix (table showing differences across dimensions)
3. Relationship Map (how personas reference/acknowledge each other)
4. Coverage Map (audience segment assignments, 100% coverage verification)
5. Network Dynamics (complementary strengths, collaboration opportunities)
6. Conflict Mitigation (potential conflicts and resolution strategies)
7. Brand Firewall Rules (how to maintain separation from parent brand)

## Quality Gates
- [ ] Differentiation matrix includes 10+ comparison dimensions
- [ ] Relationship map defines cross-persona reference protocols
- [ ] Coverage map shows 100% audience segment coverage
- [ ] Network dynamics identify complementary strengths
- [ ] Brand firewall rules explicitly documented

## Handoff
Feeds into: 4.3 Execution Log (final documentation step) and downstream to S22 Account Strategy
