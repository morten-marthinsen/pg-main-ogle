# 0.1 Input Validator

## Purpose
Validate that all required inputs exist and are complete before persona architecture begins. Ensures brand context, target audience data, niche selection parameters, and competitive landscape are provided.

## Model Assignment
**Model:** claude-3-5-haiku-20241022

## Input Requirements
- Brand information document or brief
- Target audience definition (demographics, psychographics, pain points)
- Niche selection inputs (industry vertical, sub-niches to cover)
- Competitive context (existing influencers, market gaps)
- Platform preferences (if specified)

## Execution Protocol

### MUST
- Verify brand information includes: name, mission, values, voice constraints
- Confirm target audience data includes at least 3 demographic dimensions and 5 psychographic traits
- Validate niche selection includes clear boundaries (what's IN scope, what's OUT)
- Check competitive context names at least 5 existing influencers or content creators
- Flag missing or incomplete inputs with specific requirements

### NEVER
- Proceed with missing critical inputs
- Accept vague audience definitions like "entrepreneurs" or "health-conscious people"
- Allow undefined niche boundaries
- Skip competitive context validation

## Output
**Filename:** `input-validation-report.md`

**Format:** Markdown validation report

**Minimum Size:** 2KB

**Required Sections:**
1. Brand Information Status (PASS/FAIL + gaps)
2. Audience Definition Status (PASS/FAIL + gaps)
3. Niche Selection Status (PASS/FAIL + gaps)
4. Competitive Context Status (PASS/FAIL + gaps)
5. Overall Readiness (READY / INCOMPLETE)

## Quality Gates
- [ ] All 4 input categories assessed
- [ ] Specific gaps identified for any FAIL status
- [ ] Overall readiness decision is binary (no conditional passes)
- [ ] If INCOMPLETE, execution MUST halt

## Handoff
Feeds into: 0.2 Teaching Loader (if READY) or escalates to human (if INCOMPLETE)
