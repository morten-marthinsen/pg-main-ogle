# 4.1 Monetization Plan Assembler

## Purpose
Assemble complete monetization strategy per persona integrating all Layer 1 outputs. Create comprehensive, actionable monetization plans. Minimum 12KB per persona.

## Model Assignment
**Model:** claude-3-5-sonnet-20241022

## Input Requirements
- All Layer 1 outputs (1.1 through 1.6)
- Revenue models, readiness, products, partnerships, pricing, projections
- Persona Bibles (for voice/authority context)

## Execution Protocol

### MUST
- Create one Monetization Plan file per persona
- Include 8 required sections per plan: (1) Overview, (2) Revenue Models, (3) Audience Readiness, (4) Product Strategy, (5) Partnership Framework, (6) Pricing, (7) Revenue Projections, (8) Implementation Roadmap
- Integrate all Layer 1 content (synthesize, don't just copy-paste)
- Ensure each plan is minimum 12KB (comprehensive and actionable)
- Create internal consistency (all sections align with persona authority and audience)
- Include quick-start checklist for immediate monetization actions
- Add risk mitigation strategies for each revenue stream
- Design feedback loops (how to optimize based on performance)

### NEVER
- Create plans under 12KB (indicates insufficient detail)
- Copy-paste Layer 1 files without synthesis
- Create structural inconsistencies across plans
- Skip implementation roadmap (critical for execution)
- Omit risk mitigation strategies
- Forget feedback loops for optimization

## Output
**Filename:** `monetization-plan-[persona-name].md` (one file per persona)

**Format:** Markdown comprehensive monetization strategy

**Minimum Size:** 12KB per file

**Required Sections (8):**
1. Monetization Overview (persona, revenue models, strategic approach)
2. Revenue Model Strategy (primary/secondary models, rationale, implementation)
3. Audience Readiness Status (current status, gaps, timeline to full monetization)
4. Product Development Strategy (products, roadmap, validation, production)
5. Partnership Framework (types, alignment criteria, rate cards, disclosure)
6. Pricing Architecture (prices, value analysis, tiers, payment plans)
7. Revenue Projections (3 scenarios, assumptions, breakeven, risks)
8. Implementation Roadmap (month-by-month action plan, quick-start checklist)
9. Risk Mitigation (risks per revenue stream, mitigation strategies)
10. Optimization Feedback Loops (what to track, when to pivot)

## Quality Gates
- [ ] One complete plan per persona created
- [ ] Each plan minimum 12KB
- [ ] All 8 required sections present and integrated
- [ ] Quick-start checklist and risk mitigation included
- [ ] Internal consistency verified (sections align)
- [ ] Implementation roadmap actionable
- [ ] Feedback loops defined for optimization

## Handoff
Feeds into: 4.2 Revenue Dashboard (plans provide input for network revenue tracking)
