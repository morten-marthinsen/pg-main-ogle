# 4.2 Revenue Dashboard

## Purpose
Create network-wide revenue view tracking monetization performance across all personas. Operational dashboard for revenue management, optimization, and forecasting.

## Model Assignment
**Model:** claude-3-5-sonnet-20241022

## Input Requirements
- All Monetization Plans (from 4.1)
- Revenue projections (from 1.6)
- Audience readiness assessments (from 1.2)

## Execution Protocol

### MUST
- Create dashboard sections: Revenue Performance, Monetization Readiness, Product Performance, Partnership Performance, Optimization Opportunities
- Design Revenue Performance view: actual vs. projected per persona, revenue by stream, network totals
- Build Monetization Readiness tracker: where each persona stands on readiness criteria, timeline to launch
- Create Product Performance monitor: sales by product, conversion rates, customer feedback
- Design Partnership Performance tracker: partnership revenue, brand alignment scores, audience response
- Include Optimization Opportunities: underperformers, quick wins, scaling opportunities
- Make dashboard updateable: how to refresh data monthly
- Create visualization recommendations: what should be graphed/charted

### NEVER
- Create static dashboard that can't be updated
- Skip monetization readiness tracking (critical for launch timing)
- Ignore product-level performance (need to know what's working)
- Forget optimization opportunities section
- Omit network-wide revenue aggregation
- Create dashboard without visualization recommendations

## Output
**Filename:** `revenue-dashboard.md`

**Format:** Markdown operational revenue dashboard

**Minimum Size:** 8KB

**Required Sections:**
1. Dashboard Overview (purpose, update frequency, key sections)
2. Revenue Performance (actual vs. projected per persona, by stream, network totals)
3. Monetization Readiness Tracker (criteria status per persona, timeline to launch)
4. Product Performance Monitor (sales, conversion rates, feedback per product)
5. Partnership Performance Tracker (revenue, brand alignment, audience response)
6. Optimization Opportunities (underperformers, quick wins, scaling opportunities)
7. Update Protocol (how to refresh dashboard monthly)
8. Visualization Recommendations (what to graph/chart for quick insights)

## Quality Gates
- [ ] Dashboard minimum 8KB with all required sections
- [ ] Revenue performance tracks actual vs. projected
- [ ] Monetization readiness shows launch timeline per persona
- [ ] Product and partnership performance detailed
- [ ] Optimization opportunities identified systematically
- [ ] Update protocol clear and actionable
- [ ] Visualization recommendations included
