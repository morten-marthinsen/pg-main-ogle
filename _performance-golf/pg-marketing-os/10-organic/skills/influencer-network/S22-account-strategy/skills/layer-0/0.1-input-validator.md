# 0.1 Input Validator

## Purpose
Validate all S21 Persona Bibles exist and are complete, PSF (Platform Selection Framework) loaded, and platform selection inputs available before account strategy development begins.

## Model Assignment
**Model:** claude-3-5-haiku-20241022

## Input Requirements
- S21 Persona Bibles (all persona-bible-[name].md files)
- S21 Network Overview (network-overview.md)
- S03 Platform Selection Framework (PSF)
- Platform priorities or preferences (if specified)

## Execution Protocol

### MUST
- Verify all Persona Bibles from S21 exist (one per persona)
- Check each Bible is minimum 30KB and includes all 10 required sections
- Confirm Network Overview exists with differentiation matrix and coverage map
- Validate PSF loaded with platform priorities and audience presence data
- Check for any platform constraints (e.g., brand already on certain platforms)
- Flag missing or incomplete inputs with specific requirements

### NEVER
- Proceed with missing Persona Bibles
- Accept Bibles under 30KB without investigation
- Skip PSF validation (critical for platform selection)
- Assume platform priorities without data
- Continue if Network Overview missing

## Output
**Filename:** `input-validation-report.md`

**Format:** Markdown validation report

**Minimum Size:** 2KB

**Required Sections:**
1. Persona Bible Status (list all, confirm presence and size)
2. Network Overview Status (PASS/FAIL + gaps)
3. PSF Status (platform priorities, audience data availability)
4. Platform Constraints (any existing accounts or restrictions)
5. Overall Readiness (READY / INCOMPLETE)

## Quality Gates
- [ ] All Persona Bibles accounted for and size-validated
- [ ] Network Overview complete with required sections
- [ ] PSF loaded with platform-specific data
- [ ] Overall readiness decision is binary
- [ ] If INCOMPLETE, execution MUST halt

## Handoff
Feeds into: 0.2 Teaching Loader (if READY) or escalates to human (if INCOMPLETE)
