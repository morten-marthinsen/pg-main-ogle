# 0.1 Input Validator

## Purpose
Verify all required upstream files exist and meet minimum quality standards before scheduling choreography begins.

---

## Input Requirements

From S14 Content Assembly:
- `[project]/S14-content-assembly/outputs/[campaign]-content-packages.yaml`

From S02 Platform Strategy:
- `[project]/S02-platform-strategy/outputs/[campaign]-PSF.yaml`

From S07 Campaign Brief:
- `[project]/S07-campaign-brief/outputs/[campaign]-CBF.yaml`

---

## Execution Protocol

### MUST
- **MUST** verify all 3 upstream files exist
- **MUST** check content packages have required fields (content_id, platform, content_type, function, virality_score)
- **MUST** verify PSF has optimal_times defined for all platforms
- **MUST** verify CBF has campaign dates and calendar
- **MUST** check all content IDs are unique
- **MUST** verify at least 1 content package exists

### NEVER
- **NEVER** proceed if any upstream file missing
- **NEVER** proceed if content packages lack platform data
- **NEVER** proceed if PSF lacks timing data
- **NEVER** skip validation checks

---

```yaml
  issues: []

  exists: [true/false]
  path: "[path]"

  calendar_events_count: [count]
  issues: []

```

**Minimum size:** 500 bytes

---

## Quality Gates

- [ ] All 3 upstream files exist
- [ ] Content packages valid (≥1 package with required fields)
- [ ] PSF has optimal_times for all platforms
- [ ] CBF has campaign dates
- [ ] No duplicate content IDs
