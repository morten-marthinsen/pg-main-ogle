# 4.1 RMP Assembler

## Purpose
Assemble complete Repurpose Multiplication Plan (RMP) from all Layer 1 outputs. Create the master YAML file that coordinates all variant production.

---

## Input Requirements

From Layer 1:
- `1.1-atomization.yaml`
- `1.2-platform-adaptation-matrix.yaml`
- `1.3-variant-plan.yaml`
- `1.4-production-sequence.yaml`

From Layer 0:
- `0.2-upstream-content.yaml` (metadata)

---

## Execution Protocol

### Step 1: Compile RMP Metadata

From upstream content:
- campaign_name
- original_content_id
- original_content_title
- original_platform
- original_format
- date_created

### Step 2: Integrate Atomization Section

From 1.1:
- All atomic units (hooks, insights, quotes, stories, frameworks, data, takes)
- Standalone strength scores
- Variant potential mappings

### Step 3: Integrate Variant Plan

From 1.3:
- Complete variant specifications (all 10+)
- Source element mappings
- Platform assignments
- Production needs
- Priority and timeline

### Step 4: Integrate Production Queue

From 1.4:
- Production timeline (Day 0, Day 1-3, Day 3+)
- Distribution strategy per platform
- Cross-reference strategy
- Tracking metrics

### Step 5: Validate Complete RMP

Check:
- All required sections present
- Minimum thresholds met (10 variants, 3 platforms)
- All variants have source elements
- All variants have production needs
- Platform-native rebuilding confirmed

### MUST Include

- Complete atomization inventory
- All variant specifications (10+)
- Production queue with timelines
- Distribution strategy
- Tracking structure

### NEVER Output

- Incomplete RMP (missing sections)
- RMP with <10 variants
- RMP with <3 platforms
- RMP with cross-posting language

---
