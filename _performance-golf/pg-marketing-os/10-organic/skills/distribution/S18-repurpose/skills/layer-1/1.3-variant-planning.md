# 1.3 Variant Planning

## Purpose
Create specific variant plans from atomic units + platform adaptations. Define 10+ platform-native variants with production requirements.

---

## Input Requirements

From Layer 1:
- `1.1-atomization.yaml` (atomic units)
- `1.2-platform-adaptation-matrix.yaml` (adaptation rules)

---

## Execution Protocol

### Step 1: Generate Variant Specifications

For each variant (minimum 10):

**Variant Metadata:**
- variant_id (format: [Original-ID]-V01, V02, etc.)
- variant_type (e.g., "TikTok Clip", "Instagram Carousel")
- source_element (which atomic unit)
- platform
- format

**Variant Specs:**
- length (duration or word/slide count)
- hook_transformation (specific changes from original)
- adaptation_requirements (list of changes needed)

**Production Needs:**
- editing (Yes/No/Minimal — be specific)
- new_recording (Yes/No — if yes, what)
- design (Yes/No — if yes, what design work)

**Priority & Timeline:**
- priority (High/Medium/Low)
- timeline (Day 0, Day 1-3, Day 3+)
- status (Planned)

### Step 2: Ensure Platform Distribution

Verify:
- Minimum 3 platforms represented
- Each platform has 2+ variants
- Mix of formats per platform

### Step 3: Validate Platform-Native Rebuilding

For EACH variant, verify:
- NOT same content with minor edits
- Hook is transformed (not copied)
- Adaptation requirements are specific
- Production needs are explicit

### MUST Create

- Minimum 10 variants
- Variants on 3+ platforms
- All variants have source element
- All variants have production needs

### NEVER Accept

- Cross-posting language
- "Same as X but on Y" descriptions
- Missing production needs
- Generic adaptation requirements

---

## Output

**File:** `layer-1/1.3-variant-plan.yaml`

**Format:**
```yaml
timestamp: "[ISO 8601]"
original_content_id: "[ID]"

variants:
  - variant_id: "[Original-ID]-V01"
    variant_type: "TikTok Clip"
    source_element:
      atomic_unit_id: "hook_1"
      atomic_unit_type: "hook"
      text: "[Hook text]"

    platform: "tiktok"
    format: "native_video"
    length: "45 seconds"

    transformation:
      hook: "Re-cut to 0.3 second impact, higher energy"
      pacing: "Faster cuts, 2-3 second scenes max"
      text_styling: "TikTok native large text overlays"
      audio: "Match trending sound if applicable"
      ending: "Loop-friendly or strong CTA"

    production_needs:
      editing: "Yes — extract segment, add text overlays, color grade"
      new_recording: "No"
      design: "Yes — text overlay templates"

    priority: "High"
    timeline: "Day 0"
    status: "Planned"

  - variant_id: "[Original-ID]-V02"
    variant_type: "Instagram Carousel"
    source_element:
      atomic_unit_id: "framework_1"
      atomic_unit_type: "framework"
      text: "[Framework description]"

    platform: "instagram"
    format: "carousel"
    slides: "7 slides"

    transformation:
      slide_1: "Hook slide with framework name"
      slides_2_to_7: "One step per slide with visual hierarchy"
      design: "Brand colors, consistent typography"
      caption: "Expand on framework, engagement question"

    production_needs:
      editing: "No"
      new_recording: "No"
      design: "Yes — 7 carousel slides with brand styling"

    priority: "High"
    timeline: "Day 1-3"
    status: "Planned"

  # [Continue for all variants — minimum 10]

variant_summary:
  total_variants: [Count >= 10]
  platforms_covered: [List >= 3]
  variant_breakdown:
    by_platform:
      instagram: [Count]
      tiktok: [Count]
      youtube: [Count]
      linkedin: [Count]
      x_twitter: [Count]
    by_priority:
      high: [Count]
      medium: [Count]
      low: [Count]
    by_timeline:
      day_0: [Count]
      day_1_to_3: [Count]
      day_3_plus: [Count]

platform_native_validation:
  all_variants_rebuilt: true
  no_cross_posting: true
  all_have_production_needs: true
  all_have_source_element: true
```

**Minimum size:** 4KB
