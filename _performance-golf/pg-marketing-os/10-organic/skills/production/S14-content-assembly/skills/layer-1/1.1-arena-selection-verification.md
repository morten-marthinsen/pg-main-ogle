# 1.1 Arena Selection Verification

## Purpose
Verify that Arena-selected content is used in assembly, NOT pre-Arena drafts. This is the HIGHEST PRIORITY check in S14 — prevents the Skill 19 failure pattern (entire lead section missing from assembled draft due to consuming pre-Arena content).

**This check is BLOCKING. It cannot be bypassed.**

---

## Input Requirements

From Layer 0:
- `0.2-arena-results.yaml` (Arena-selected content with hashes)
- `0.3-production-outputs.yaml` (Production content with hashes)

---

## Execution Protocol

### Step 1: Load Comparison Data

For each content piece:
1. Load Arena-selected content hash from 0.2
2. Load production output hash from 0.3
3. Load Arena-selected full content
4. Load production output full content

```yaml
comparison_data:
  - content_id: "[ID]"
    arena_hash: "[Hash from 0.2]"
    production_hash: "[Hash from 0.3]"
    arena_content_path: "[Path to Arena-selected file]"
    production_content_path: "[Path to production output file]"
```

---

### Step 2: Hash Comparison (Primary Check)

Compare hashes:

**IF arena_hash == production_hash:**
- Mark as VERIFIED
- Content matches Arena selection
- Proceed to Step 5 (logging)

**IF arena_hash != production_hash:**
- Mark as MISMATCH
- Proceed to Step 3 (deep comparison)

---

### Step 3: Deep Content Comparison (Mismatch Investigation)

If hashes don't match, perform line-by-line comparison:

1. Load both content pieces as text
2. Split into lines
3. Compare line by line
4. Identify differences

```yaml
mismatch_analysis:
  content_id: "[ID]"
  hash_match: false
  line_by_line_comparison:
    total_lines_arena: [Number]
    total_lines_production: [Number]
    lines_matching: [Number]
    lines_different: [Number]
    match_percentage: [Percentage]

  differences:
    - line_number: [Number]
      arena_content: "[Line from Arena selection]"
      production_content: "[Line from production output]"
      difference_type: "[Addition/Deletion/Modification]"

  severity: [MINOR / MODERATE / CRITICAL]
```

**Severity classification:**
- **MINOR:** <5% lines different, cosmetic changes only (spacing, punctuation)
- **MODERATE:** 5-20% lines different, or structural changes (section reordering)
- **CRITICAL:** >20% lines different, or missing entire sections (like Skill 19 failure)

---

### Step 4: Determine Response

**IF severity = MINOR:**
- Log warning
- Allow to proceed with flag
- Note differences in verification file

**IF severity = MODERATE:**
- Log error
- Flag for human review
- Display side-by-side comparison
- HALT assembly until reviewed

**IF severity = CRITICAL:**
- Log CRITICAL error
- HALT immediately
- Display error message with differences
- Require reload of Arena-selected content
- DO NOT PROCEED

**Example CRITICAL error message:**
```
CRITICAL: Arena Selection Verification FAILED

Content ID: script-01-instagram-reel
Issue: Production content does NOT match Arena-selected version

Arena-selected hybrid: Hybrid A
Arena file path: /path/to/arena/Hybrid-A.md

Production content has 78 lines
Arena content has 124 lines
46 lines MISSING from production (37% of Arena content)

This is the same failure pattern as Skill 19 (entire lead section missing).

HALTING ASSEMBLY.

ACTION REQUIRED:
1. Review production output at: [path]
2. Verify it matches Arena selection at: [path]
3. If production output is incorrect, reload Arena-selected content
4. Re-run S14 assembly after correction
```

---

### Step 5: Log Verification Results

For each content piece, log verification status:

```yaml
verification_log:
  - content_id: "[ID]"
    content_type: "[Type]"
    platform: "[Platform]"

    verification_status: [VERIFIED / MINOR_MISMATCH / MODERATE_MISMATCH / CRITICAL_MISMATCH]

    arena_info:
      selected_hybrid: "[Hybrid identifier]"
      hybrid_path: "[Path]"
      content_hash: "[Hash]"

    production_info:
      production_path: "[Path]"
      content_hash: "[Hash]"

    comparison_result:
      hashes_match: [Yes/No]
      deep_comparison_performed: [Yes/No]
      match_percentage: [Percentage, if deep comparison done]
      differences_found: [Number]
      severity: [MINOR/MODERATE/CRITICAL/N/A]

    decision: [PROCEED / FLAG_FOR_REVIEW / HALT]

    notes: |
      [Any relevant notes about the comparison]
```

---

### Step 6: Create Arena Verification Checkpoint

If ALL content pieces VERIFIED (or MINOR_MISMATCH only):

```yaml
# For internal tracking
arena_verification_checkpoint:
  timestamp: "[ISO 8601]"
  all_content_verified: true
  critical_mismatches: 0
  moderate_mismatches: 0
  minor_mismatches: [Number]
  ready_to_proceed: true
```

If ANY MODERATE or CRITICAL mismatches:
```yaml
arena_verification_checkpoint:
  timestamp: "[ISO 8601]"
  all_content_verified: false
  critical_mismatches: [Number]
  moderate_mismatches: [Number]
  ready_to_proceed: false
  blocked_content_pieces: []
```

**Minimum size:** 2KB

---

## Quality Gates (Gate G08)

- [ ] All content pieces have verification entries
- [ ] All Arena-selected content loaded and hashed
- [ ] All production content loaded and hashed
- [ ] Comparison performed for all content pieces
- [ ] NO CRITICAL mismatches
- [ ] NO MODERATE mismatches (or all reviewed and approved by human)

**Gate G08 Status:**
- **PASS:** All content VERIFIED or MINOR_MISMATCH only
- **FAIL:** Any MODERATE or CRITICAL mismatches

**If Gate G08 = FAIL:** HALT. Do NOT proceed to 1.2.

---

## Handoff

Pass verification results to: **1.2 Content Inventory**

**CRITICAL:** If any content piece has decision = HALT, the entire S14 execution must stop. Display error message, require correction, and restart S14 from Layer 0.

**This is the most important quality gate in S14. It prevents publishing the wrong version of content.**
