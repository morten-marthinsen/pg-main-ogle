# 0.1 Input Validator

## Purpose
Verify all required inputs exist and are valid before beginning assembly process. This microskill ensures no downstream failures due to missing or corrupted input files.

---

## Input Requirements

None (this is the entry point).

---

## Execution Protocol

### Step 1: Verify Project Infrastructure

Check these directories exist:
```
[project]/S14-content-assembly/
├── PROJECT-STATE.md
├── PROGRESS-LOG.md
├── checkpoints/
└── outputs/
```

**If ANY missing:** Create them now.

---

### Step 2: Verify Required Input Files Exist

**Strategic Foundation (Required):**
- [ ] `[project]/S07-campaign-brief/outputs/[campaign]-CBF.yaml` — Campaign Brief File
- [ ] `[project]/S03-brand-voice/outputs/[campaign]-BVF.yaml` — Brand Voice File
- [ ] `[project]/S02-platform-strategy/outputs/[campaign]-PSF.yaml` — Platform Strategy File
- [ ] `[project]/S06-virality-scoring/outputs/[campaign]-VSF.yaml` — Virality Scoring Framework

**Production Outputs (At least one content type required):**
- [ ] `[project]/S08-scripts/outputs/` — Script outputs (if video content)
- [ ] `[project]/S09-captions/outputs/` — Caption outputs (if social posts)
- [ ] `[project]/S10-carousels/outputs/` — Carousel outputs (if carousel content)
- [ ] `[project]/S11-threads/outputs/` — Thread outputs (if thread content)
- [ ] `[project]/S12-visual-direction/outputs/` — Visual direction outputs (required)

**Arena Checkpoint (Critical):**
- [ ] `[project]/S13-arena-generation/checkpoints/ARENA_COMPLETE.yaml` — Arena verification

---

### Step 3: Load and Validate CBF

Load CBF and extract:
```yaml
campaign_name: "[Name]"
brand_name: "[Brand]"
minimum_virality_score: [Number]  # CRITICAL for Gate G09
content_pieces:
  - content_id: "[ID]"
    platform: "[Platform]"
    content_type: "[Type]"
    # [... more pieces]
```

**Validate:**
- CBF has campaign_name
- CBF has brand_name
- CBF has minimum_virality_score (number between 40-100)
- CBF has at least 1 content piece defined

**If validation fails:** HALT. Log error. Require valid CBF.

---

### Step 4: Check Arena Checkpoint

Load Arena checkpoint:
```yaml
skill: "S13-arena-generation"
status: COMPLETE
arena_selection_verified: true  # MUST be present and true
```

**Validate:**
- File exists
- status = COMPLETE
- arena_selection_verified = true

**If ANY validation fails:** HALT. Arena verification is BLOCKING. Require S13 completion.

---

### Step 5: Inventory Production Outputs

For each content piece in CBF, verify corresponding outputs exist:

```yaml
content_inventory:
  - content_id: "[ID from CBF]"
    required_outputs:
      script: "[Path if video content]"
      caption: "[Path]"
      visual_direction: "[Path]"
      arena_synthesis: "[Path from S13]"
    status: [COMPLETE / MISSING]
    missing_files: []
```

**For each content piece:**
- If video content → require script from S08
- If carousel → require slides from S10
- If thread → require tweets from S11
- ALL content → require caption (S09) + visual direction (S12) + Arena synthesis (S13)

---

### Step 6: Validate File Accessibility

Attempt to read each required file.

**For each file:**
- Try to open and read first 100 bytes
- Verify file is not empty (size > 100 bytes)
- Verify file is readable (no permission errors)

**If ANY file fails:** Log error, mark as MISSING in inventory, HALT if critical file.

---

---

## Quality Gates

- [ ] All critical files exist
- [ ] Arena checkpoint valid with arena_selection_verified = true
- [ ] CBF has minimum_virality_score defined
- [ ] At least 1 content piece has all required outputs
- [ ] No file accessibility errors

**If ANY gate fails:** Status = FAIL. HALT execution.

---

## Handoff

Pass validation results to: **0.2 Arena Results Loader**

**CRITICAL:** If validation_status = FAIL, do NOT proceed to 0.2.
