# 2.1 Hook Drafting

## Purpose
Draft the hook section for each script following the hook strategy from Layer 1. The hook determines retention—get this wrong and nothing else matters.

---

## Input Requirements

From Layer 1:
- `1.3-hook-strategy.yaml` (hook types, requirements)
- `1.4-script-structure-plan.yaml` (timing, structure)

From Layer 0:
- `0.3-specimens-loaded.yaml` (hook patterns)

---

## Execution Protocol

### Step 1: Review Hook Strategy

For each script, load hook requirements:

```yaml
hook_requirements:
  script_id: "script_1"
  platform: "TikTok"
  hook_type: "Curiosity Hook"
  timing: "0-1 second"
  specimen_pattern: "Question hook"
  success_criteria:
    - "Stops scroll in 0.5s"
    - "Creates curiosity gap"
    - "Platform-native"
```

---

### Step 2: Draft Hook Options

Generate 3-5 hook variations per script:

```yaml
hook_drafts:
  script_id: "script_1"
  platform: "TikTok"

  hook_option_1:
    verbal: "Want to know the one thing viral creators never tell you?"
    visual_direction: |
      Close-up face shot, direct eye contact with camera.
      Immediate zoom-in on first word.
    text_overlay: "The ONE THING..."
    timing: "0:00-0:01"
    pattern: "Question hook"
    score:
      pattern_interrupt: 8/10
      curiosity_gap: 9/10
      platform_fit: 9/10
      specimen_alignment: 8/10

  hook_option_2:
    verbal: "Stop scrolling—this changes everything about [topic]"
    visual_direction: |
      Hand enters frame making stop gesture.
      Quick cut to engaging visual.
    text_overlay: "STOP ✋"
    timing: "0:00-0:01"
    pattern: "Command hook"
    score:
      pattern_interrupt: 9/10
      curiosity_gap: 7/10
      platform_fit: 8/10
      specimen_alignment: 7/10

  # ... 3-5 total options
```

---

### Step 3: Score Hook Options

Evaluate each hook against criteria:

```yaml
hook_scoring:
  script_id: "script_1"

  scoring_rubric:
    pattern_interrupt: "Does it stop the scroll immediately?"
    curiosity_gap: "Does it create a need to know more?"
    platform_fit: "Is it native to the platform?"
    specimen_alignment: "Does it match proven patterns?"

  scores:
    hook_option_1:
      total: 34/40 (85%)
      recommendation: "STRONG - use for Arena"

    hook_option_2:
      total: 31/40 (78%)
      recommendation: "GOOD - backup option"
```

---

### Step 4: Add Production Notes

For each hook option, specify production requirements:

```yaml
production_specifications:
  hook_option_1:
    camera_setup:
      - "Vertical 9:16 format"
      - "Eye-level close-up"
      - "Natural lighting"

    editing_notes:
      - "Zoom-in effect on first word"
      - "Text overlay appears at 0:00"
      - "No intro graphics—immediate hook"

    audio_notes:
      - "Clear vocal delivery"
      - "Optional trending sound layer"
      - "No background music during hook"

    first_frame:
      importance: "CRITICAL - this is thumbnail"
      requirements: "Expressive face, clear text overlay"
```

### Score

## Hook Option 2

[Repeat structure]

**Minimum size:** 1KB per script
