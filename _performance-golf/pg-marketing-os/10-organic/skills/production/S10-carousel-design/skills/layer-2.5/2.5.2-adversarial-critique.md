# 2.5.2 Adversarial Critique

## Purpose
Run 3 rounds of adversarial critique where each persona attacks others' carousels and defends their own.

## Input Requirements
- Arena Round 1 submissions (7 complete carousels)
- Checkpoint: ARENA_ROUND1_COMPLETE
- Scoring criteria (Hook 30%, Save 30%, Platform 20%, Engagement 20%)

## Execution Protocol

**MUST:**
- Execute 3 rounds of critique
- **Round 1 Critique Focus:** Cover slide power and first impression
  - Each persona critiques 2-3 others' cover slides
  - Attack: Hook weakness, curiosity gap failures, scroll-stop inadequacy
  - Defend: Explain rationale for own cover slide choices
- **Round 2 Critique Focus:** Body slide independence and screenshot-worthiness
  - Each persona critiques 2-3 others' body slides
  - Attack: Slides requiring context, poor formatting, low utility
  - Defend: Justify slide structure and formatting choices
- **Round 3 Critique Focus:** Save-worthiness and engagement mechanics
  - Each persona critiques 2-3 others' complete carousels
  - Attack: Low save incentive, weak CTAs, poor engagement hooks
  - Defend: Explain overall strategy and expected performance
- Each critique must be SPECIFIC (cite slide numbers, quote text, name weaknesses)
- Track emerging consensus and divergent views
- Document any carousel revisions made during critique
- Create checkpoint after each round: ARENA_ROUND[N]_CRITIQUE_COMPLETE

**NEVER:**
- Allow generic critiques ("This is weak")
- Skip rounds
- Let personas avoid defending their choices
- Ignore emerging patterns across critiques

## Output
**Filename:** `adversarial-critique.yaml`

**Format:**
```yaml
critique_rounds: 3
round_1_cover_critique:
  - attacking_persona: string
    target_persona: string
    specific_criticism: string
    weakness_identified: string
  [...all critiques]
  - defending_persona: string
    defense: string
round_2_body_critique:
  [...same structure]
round_3_complete_critique:
  [...same structure]
emerging_patterns:
  consensus_strengths: [list across submissions]
  consensus_weaknesses: [list across submissions]
  divergent_views: [list where personas disagree]
revisions_during_critique:
  - persona: string
    revision: string
    rationale: string
checkpoints:
  - "ARENA_ROUND1_CRITIQUE_COMPLETE"
  - "ARENA_ROUND2_CRITIQUE_COMPLETE"
  - "ARENA_ROUND3_CRITIQUE_COMPLETE"
timestamp: ISO8601
```

**Minimum size:** 6KB (detailed critiques across 3 rounds)

## Quality Gates
- All 3 rounds completed
- Minimum 14 specific critiques per round (2 per persona)
- All defenses documented
- Emerging patterns identified
- Checkpoints created per round

## Handoff
Provides critique insights and consensus patterns to 2.5.3 Synthesis for hybrid generation.
