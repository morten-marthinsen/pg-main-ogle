# 1.4 Save Optimization

## Purpose
Design carousel structure to maximize save rate through reference-worthy formatting and actionable content.

## Input Requirements
- Teaching context (STEPPS — Practical Value)
- Specimen analysis (screenshot techniques)
- Slide architecture
- Platform requirements

## Execution Protocol

**MUST:**
- Design for saves by optimizing 3 dimensions:
  1. **Screenshot-worthiness:** High contrast text, clean backgrounds, quotable formatting
  2. **Actionable takeaways:** Each body slide delivers implementable insight
  3. **Reference-worthy structure:** Framework/system user wants to return to
- Front-load save prompt on Slide 1 or Slide 2 ("Save this for later")
- Use formatting that enhances screenshot utility:
  - Bullets/numbers for scannability
  - Bold key phrases
  - High contrast ratios
  - Minimal clutter
- Structure body slides as standalone reference cards
- Build framework/system that has "keep value" (user wants to reference again)
- Design final slide with explicit save CTA
- Apply practical value density (STEPPS principle)
- Test each slide: "Would I screenshot this?"

**NEVER:**
- Bury save CTA at end only
- Create slides that look good but lack utility
- Use low-contrast design that screenshots poorly
- Make content time-sensitive (kills reference value)
- Skip screenshot test

## Output
**Filename:** `save-optimization.yaml`

**Format:**
```yaml
save_strategy:
  primary_save_driver: string
  save_cta_placement: [slide numbers]
  reference_value_proposition: string
screenshot_optimization:
  - slide_number: number
    screenshot_utility: string
    formatting_choices: [list]
    actionable_takeaway: string
practical_value_per_slide:
  - slide_number: number
    takeaway: string
    implementation_ease: "easy | medium | requires planning"
save_cta_copy:
  early_prompt: string (Slide 1-2)
  final_cta: string (last slide)
screenshot_test_results:
  slides_passing: [list]
  slides_needing_revision: [list]
framework_description: string (if applicable)
timestamp: ISO8601
```

**Minimum size:** 2KB

## Quality Gates
- Save strategy explicitly defined
- Minimum 2 save CTA placements (early + final)
- All body slides pass screenshot test
- Practical value documented per slide
- Reference value proposition clear

## Handoff
Provides save optimization strategy to 2.2 Body Slide Writing and 2.3 CTA Slide Writing.
