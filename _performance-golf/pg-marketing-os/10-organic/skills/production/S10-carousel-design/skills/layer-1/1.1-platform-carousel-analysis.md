# 1.1 Platform Carousel Analysis

## Purpose
Analyze target platform carousel requirements and technical specifications.

## Input Requirements
- Platform specification from carousel brief
- CBF context
- Specimen analysis

## Execution Protocol

**MUST:**
- Define platform-specific requirements:
  - **Instagram:** 1:1 or 4:5 aspect ratio, 5-10 slides optimal, <100 char headline/<200 char body per slide, high visual impact required
  - **LinkedIn:** Variable ratio (4:5 or 1:1), 8-15 slides optimal, PDF upload format, <150 char headline/<300 char body, professional aesthetic
- Document character limits per slide type (cover, body, CTA)
- Identify optimal slide count for topic complexity
- Note platform algorithm preferences (saves, shares, time on page)
- Extract engagement patterns from specimen analysis
- Define technical export specifications

**NEVER:**
- Use generic cross-platform specs
- Ignore character limits
- Skip algorithm preference analysis
- Apply wrong platform requirements

## Output
**Filename:** `platform-requirements.yaml`

**Format:**
```yaml
platform: instagram | linkedin
technical_specs:
  aspect_ratio: string
  optimal_slide_count: [min, max]
  max_slide_count: number
  file_format: string
character_limits:
  cover_headline: number
  cover_subhead: number
  body_headline: number
  body_text: number
  cta_text: number
algorithm_preferences:
  - preference: string
    optimization_strategy: string
engagement_patterns:
  high_save_triggers: [list]
  high_share_triggers: [list]
  high_comment_triggers: [list]
export_specs:
  dimensions: string
  file_type: string
  color_profile: string
timestamp: ISO8601
```

**Minimum size:** 1.5KB

## Quality Gates
- Platform requirements match target platform
- Character limits specified for all slide types
- Optimal slide count range defined
- Export specifications complete

## Handoff
Provides technical constraints to 1.2 Slide Architecture and all Layer 2 writing microskills.
