# 4.3 Final Validation + Execution Log

## Purpose
Run final validation across all carousel outputs. Confirm Arena selection integrity, virality thresholds, platform compliance, and design completeness. Generate execution log documenting all decisions.

---

## Input Requirements

From 4.1:
- `outputs/[campaign]-carousels-assembled.yaml`

From 4.2:
- `outputs/[carousel-id]-[platform].md` (all exports)
- `outputs/CAROUSEL-INDEX.md`

From checkpoints:
- `checkpoints/ARENA_COMPLETE.yaml`

---

## Execution Protocol

### Step 1: Arena Selection Integrity Check

For each carousel, verify:
1. Arena-selected hybrid name matches ARENA_COMPLETE.yaml
2. Final slide content matches Arena synthesis output (not pre-Arena draft)
3. No slides reverted to pre-Arena versions

```yaml
arena_integrity:
  carousel_id: "carousel_1"
  arena_hybrid: "Hybrid A"
  checkpoint_match: true
  content_match: true
  status: PASS
```

**If content_match = false → HALT. Carousel assembly consumed wrong version.**

### Step 2: Virality Threshold Check

```yaml
virality_check:
  carousel_id: "carousel_1"
  score: [total]
  threshold: 60
  status: PASS | FAIL

  cover_slide_hook_score: [1-10]  # Must be >= 8
  save_worthiness_score: [1-10]   # Must be >= 7
```

**MUST:** Cover slide hook score >= 8.0. If below → revision required.

### Step 3: Platform Compliance Check

For each carousel export file:
- Slide count within platform range
- Character counts within platform limits
- Aspect ratio specified correctly
- Design brief present and complete

### Step 4: Design Completeness Check

For each slide in each carousel:
- [ ] Headline present
- [ ] Visual direction present (layout, colors, typography, imagery)
- [ ] Character count within limit
- [ ] Slide type labeled (cover/body/cta)

## Quality Gates

- [ ] All Arena selections verified (integrity check passed)
- [ ] All virality scores >= 60
- [ ] All cover slide hook scores >= 8.0
- [ ] All platform compliance checks passed
- [ ] All slides have complete design specs
- [ ] Execution log documents all decisions
- [ ] Final status = COMPLETE

**If any gate fails → status = INCOMPLETE → remediate before closing**
