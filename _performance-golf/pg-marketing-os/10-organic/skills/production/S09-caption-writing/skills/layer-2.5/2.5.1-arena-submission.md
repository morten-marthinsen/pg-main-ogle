This creates diverse high-quality options for critique and synthesis.

## EXECUTION PROTOCOL

**MUST:**
1. Load **PRE-ARENA caption** from checkpoint file
2. Load **ORGANIC-ARENA-PROTOCOL.md** for persona definitions
3. For EACH of 7 personas, generate complete caption variant:
   - **Volume Machine:** Optimizes for speed/consistency, templates, batch-friendly
   - **Value Architect:** Dense actionable content, screenshot-worthy, reference material
   - **Virality Engineer:** Emotional contagion, shareability, social currency
   - **Community Builder:** Conversation starters, engagement loops, relationship building
   - **Brand Purist:** Voice consistency, brand reinforcement, long-term positioning
   - **Algorithm Hacker:** Platform optimization, reach maximization, engagement signals
   - **Storyteller:** Narrative arcs, character-driven hooks, emotional journey
4. Each persona MUST:
   - Generate full caption (hook + body + CTA + hashtags)
   - Stay within platform character limits
   - Maintain BVF core voice (while emphasizing persona lens)
   - Document their specific improvements vs PRE-ARENA
5. Format each persona variant with:
   - Persona name
   - Full caption text
   - Character count
   - Persona-specific scoring (see persona criteria)
   - Rationale for changes
6. Generate Arena submission document with all 7 variants

**NEVER:**
- Skip any of the 7 personas (all required)
- Generate variants that ignore platform constraints (char limits, truncation)
- Abandon BVF voice completely (persona lens enhances, doesn't replace)
- Copy PRE-ARENA verbatim (each persona must improve/adapt)
- Generate only minor tweaks (personas should show distinct approaches)
