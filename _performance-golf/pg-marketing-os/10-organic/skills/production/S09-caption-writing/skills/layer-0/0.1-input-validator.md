# 0.1 Input Validator

**Layer:** 0 (Validation)
**Model:** Claude Haiku
**Upstream:** S08 (Script Writing) or standalone caption request
**Downstream:** 0.2 Teaching Loader

---

## PURPOSE

Validate that all required inputs exist before caption generation begins. Prevent downstream failures from missing CBF data, undefined platforms, or incomplete caption_brief schema.

---

## INPUT REQUIREMENTS

```yaml
required_inputs:
  caption_brief:
    file: "caption-brief.yaml"
    schema: CAPTION-BRIEF-SCHEMA.md
    fields:
      - platform              # Instagram, TikTok, LinkedIn, X, YouTube
      - content_type          # Reel, Static Post, Story, Carousel, Video
      - content_function      # Educate, Entertain, Inspire, Convert, Nurture
      - hook_strategy         # From HLF framework
      - cta_type             # Soft, Medium, Hard
      - character_target      # Platform-specific target length
      - hashtag_count        # 3-10 recommended
      - brand_voice_file     # Path to BVF

  optional_inputs:
    content_pillars: []      # From PSF if available
    script_text: ""          # If caption accompanies video (from S08)
    visual_description: ""   # Static post or carousel visual notes
    campaign_context: ""     # Broader campaign messaging
```

---

## EXECUTION PROTOCOL

**MUST:**
1. Check `caption-brief.yaml` exists in project directory
2. Validate all required fields are present and non-empty
3. Verify `platform` is one of: Instagram, TikTok, LinkedIn, X, YouTube
4. Confirm `content_function` maps to valid CTA type (Soft/Medium/Hard)
5. Check `brand_voice_file` path is valid and file exists
6. Validate `character_target` is within platform limits:
   - Instagram: ≤2200 chars
   - TikTok: ≤4000 chars
   - LinkedIn: ≤3000 chars
   - X: ≤280 chars
   - YouTube: ≤5000 chars
7. If validation PASSES → generate `0.1-validation-result.json`
8. If validation FAILS → HALT and report missing/invalid inputs to human

**NEVER:**
- Proceed without platform specification
- Accept empty/null required fields
- Invent default values for missing data
- Skip character limit validation
- Generate placeholder caption briefs

---

**File:** `outputs/layer-0/0.1-validation-result.json`

**Format:**

```json
  "inputs_validated": {
    "caption_brief_exists": true,

    "content_type_valid": true,
    "content_function_valid": true,
    "bvf_path_valid": true,
    "character_target_within_limits": true
  },
  "platform": "Instagram",
  "character_limit": 2200,
  "character_target": 1800,
  "content_function": "Educate",
  "cta_type": "Medium",
  "errors": [],
  "warnings": [
    "content_pillars not provided (optional)"
  ],
  "ready_for_layer_1": true

```

**Minimum Size:** 400 chars (structured validation result)

---

## QUALITY GATES

**GATE 0.1: Input Validation**
- Pass: All required fields present, platform valid, character target within limits
- Fail: Missing required fields, invalid platform, character target exceeds limit → HALT execution, request corrections

---

## HANDOFF

**Downstream:** 0.2 Teaching Loader
**Handoff File:** `0.1-validation-result.json`
**Conditions:** validation_status = "PASS" AND ready_for_layer_1 = true
