# 4.1 Output Assembler

## Purpose
Assemble final thread output with virality scoring, quality validation, and Arena verification for handoff to S14.

---

## Input Requirements

From Layer 2.5:
- `2.5.3-arena-synthesis.md` (selected thread)
- `ARENA_COMPLETE.yaml` checkpoint

From Layer 0:
- `0.4-cbf-context.yaml` (campaign context)

---

## Execution Protocol

### Step 1: Extract Arena-Selected Thread

Pull final thread from synthesis:

```yaml
arena_selected_thread:
  source_file: "layer-2.5/2.5.3-arena-synthesis.md"

  selected_option: "[Option name from human selection]"

  full_thread: |
    [Complete thread text from Arena synthesis]

  element_sources:
    hook: "[Source persona]"
    body: "[Source persona(s)]"
    cta: "[Source persona]"

  total_tweets: [N]
```

### Step 2: Calculate Virality Scores

Score thread using VSF (Virality Scoring Framework):

```yaml
virality_scoring:
  emotional_activation:
    score: [1-10]
    rationale: |
      [Does hook/content trigger emotion? Surprise, anger, awe, anxiety?]

  social_currency:
    score: [1-10]
    rationale: |
      [Does sharing this make reader look smart/informed?]

  pattern_interrupt:
    score: [1-10]
    rationale: |
      [Does hook stop scroll? Contrarian angle?]

  platform_fit:
    score: [1-10]
    rationale: |
      [Platform-native format? Algorithm-friendly?]

  shareability:
    score: [1-10]
    rationale: |
      [RT/repost potential? Standalone value?]

  total_score: [Sum of 5 dimensions]
  minimum_required: 60
  status: "[PASS if ≥60, FAIL if <60]"
```

**If status = FAIL:** Thread must be revised or returned to Arena.

### Step 3: Quality Scoring

Score thread on organic-specific dimensions:

```yaml
quality_scores:
  hook_strength:
    score: [1-10]
    criteria:
      - "Works standalone"
      - "Creates curiosity"
      - "Clear value"
    passes: [Yes/No per criterion]

  value_density:
    score: [1-10]
    criteria:
      - "Each tweet standalone valuable"
      - "No filler tweets"
      - "Insight per tweet"
    passes: [Yes/No per criterion]

  flow_and_pacing:
    score: [1-10]
    criteria:
      - "Smooth transitions"
      - "Varied intensity"
      - "Maintains momentum"
    passes: [Yes/No per criterion]

  cta_clarity:
    score: [1-10]
    criteria:
      - "Specific action"
      - "Easy to execute"
      - "Matches function"
    passes: [Yes/No per criterion]

  overall_quality: [Average of 4 dimensions]
```

### Step 4: Compile Complete Output

Assemble final output schema:

```yaml
thread_output:
  thread_id: "[Generated ID]"
  title: "[Thread title/topic]"
  platform: "[X_TWITTER or LINKEDIN]"
  content_function: "[Awareness/Engagement/Conversion/Community]"
  thread_type: "[Listicle/Story/Breakdown/Framework/Prediction/Curation]"
  total_tweets: [N]

  tweets:
    - tweet_number: 1
      type: "hook"
      content: |
        [Tweet text]
      character_count: [N]
      media_description: "[If applicable]"

    - tweet_number: 2
      type: "[context/body]"
      content: |
        [Tweet text]
      character_count: [N]

    # ... all tweets

  thread_copy_formatted: |
    [Full thread with numbering, ready to post]

  virality_scores:
    emotional_activation: [1-10]
    social_currency: [1-10]
    pattern_interrupt: [1-10]
    platform_fit: [1-10]
    shareability: [1-10]
    total_score: [Sum]

  quality_scores:
    hook_strength: [1-10]
    value_density: [1-10]
    flow_and_pacing: [1-10]
    cta_clarity: [1-10]

  arena_selection_verified: true
  arena_file: "layer-2.5/2.5.3-arena-synthesis.md"

  campaign_context:
    campaign_name: "[From CBF]"
    brand_name: "[From CBF]"
    pillar: "[Content pillar]"

  execution_log: "layer-4/4.2-execution-log.md"

  created_timestamp: "[ISO 8601]"
```

### Step 5: Validation

Run final validation checks:

```yaml
validation_checklist:
  - check: "platform = X_TWITTER or LINKEDIN"
    status: [PASS/FAIL]

  - check: "total_tweets between 5-15"
    status: [PASS/FAIL]

  - check: "tweets[0].type = hook"
    status: [PASS/FAIL]

  - check: "All tweets under character limit"
    status: [PASS/FAIL]

  - check: "virality_scores.total_score ≥ 60"
    status: [PASS/FAIL]

  - check: "quality_scores.hook_strength ≥ 7"
    status: [PASS/FAIL]

  - check: "arena_selection_verified = true"
    status: [PASS/FAIL]

  - check: "Arena synthesis file exists"
    status: [PASS/FAIL]

  all_checks_passed: [Yes/No]
```

**If all_checks_passed = No:** Identify failures, remediate, re-validate.

**File:** `outputs/[thread-id]-thread-output.yaml`

**Format:**
```yaml
thread_id: "[ID]"
title: "[Title]"
platform: "[X_TWITTER or LINKEDIN]"
content_function: "[Function]"
thread_type: "[Type]"
total_tweets: [N]

tweets:
  - tweet_number: 1
    type: "hook"
    content: |
    character_count: [N]

  # ... all tweets

thread_copy_formatted: |

virality_scores:
  emotional_activation: [1-10]
  social_currency: [1-10]
  pattern_interrupt: [1-10]
  platform_fit: [1-10]
  shareability: [1-10]
  total_score: [Sum]

quality_scores:
  hook_strength: [1-10]
  value_density: [1-10]
  flow_and_pacing: [1-10]
  cta_clarity: [1-10]

arena_selection_verified: true
arena_file: "[Path]"

campaign_context:
  campaign_name: "[Name]"
  brand_name: "[Brand]"
  pillar: "[Pillar]"

execution_log: "[Path]"
created_timestamp: "[ISO 8601]"
```

**Minimum size:** 2KB (scales with thread)

---

## Quality Gates

- [ ] Arena-selected thread extracted
- [ ] Virality score calculated (≥60)
- [ ] Quality scores calculated
- [ ] All validation checks passed
- [ ] Output schema complete
- [ ] arena_selection_verified = true

## Handoff

Pass thread output to: **4.2 Execution Log** and then **S14 Content Assembly**
