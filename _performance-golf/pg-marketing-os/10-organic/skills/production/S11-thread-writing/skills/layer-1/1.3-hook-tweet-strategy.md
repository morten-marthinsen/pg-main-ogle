# 1.3 Hook Tweet Strategy

## Purpose
Design the strategy for tweet 1 (the hook) — the most critical tweet that determines thread success.

---

## Input Requirements

From Layer 0:
- `0.2-teaching-reference.yaml` (virality principles)
- `0.3-specimen-library.yaml` (hook patterns from specimens)
- `0.4-cbf-context.yaml` (voice, audience)

From Layer 1:
- `1.1-platform-analysis.yaml` (platform requirements)
- `1.2-thread-type-selection.yaml` (thread type, hook template)

---

## Execution Protocol

### Step 1: Define Hook Requirements

Based on platform and thread type:

```yaml
hook_requirements:
  must_achieve:
    - "Stop mid-scroll"
    - "Work standalone (no context needed)"
    - "Create curiosity or desire"
    - "Be specific over vague"
    - "Strong enough to RT on its own"

  character_budget:
    platform_limit: [280 for X, 3000 for LinkedIn]
    recommended_length: "[60-80% of limit for X, 200-400 for LinkedIn]"
    reasoning: |
      [Why this length]

  structural_requirements:
    hook_element: "[What creates the hook]"
    promise_element: "[What value is promised]"
    optional_elements: ["Credential", "Number", "Time frame"]
```

### Step 2: Apply Hook Templates from Type

Use templates from selected thread type:

```yaml
hook_templates:
  primary_template:
    pattern: "[Template from 1.2]"
    example: "[Example from specimen]"
    adaptation_for_topic: |
      [How to adapt this template to current topic]

  alternative_templates:
    - pattern: "[Alternative template]"
      reason_alternative: "[When to use this instead]"

    - pattern: "[Alternative template]"
      reason_alternative: "[When to use this instead]"
```

### Step 3: Hook Amplifiers

Apply amplification techniques:

```yaml
hook_amplifiers:
  specificity:
    technique: "Use specific numbers over vague"
    application: |
      [How to make hook more specific]
    examples:
      vague: "Made a lot of money"
      specific: "Made $47K in 30 days"

  time_frame:
    technique: "Add time constraint"
    application: |
      [How to add urgency]
    examples:
      without: "I built a business"
      with: "I built a $100K business in 90 days"

  credentials:
    technique: "Signal authority or experience"
    application: |
      [What credentials to surface]
    examples:
      without: "Here's what I learned"
      with: "After 10 years and 50+ failures, here's what I learned"

  contrarian_angle:
    technique: "Challenge conventional wisdom"
    application: |
      [What contrarian position to take]
    examples:
      conventional: "Growth hacking tips"
      contrarian: "Growth hacking is dead. Here's what actually works:"
```

### Step 4: Hook Validation Criteria

Define tests for hook quality:

```yaml
hook_tests:
  standalone_test:
    question: "Does this tweet make sense without reading tweet 2?"
    pass_criteria: "Yes — complete thought"
    fail_signal: "Requires context to understand"

  scroll_stop_test:
    question: "Would this stop someone mid-scroll?"
    pass_criteria: "Creates strong curiosity or desire"
    fail_signal: "Feels generic or obvious"

  curiosity_gap_test:
    question: "Does this create a gap that must be filled?"
    pass_criteria: "Reader MUST keep reading"
    fail_signal: "Hook reveals too much or too little"

  clarity_test:
    question: "Is the value proposition obvious?"
    pass_criteria: "Reader knows what they'll learn/get"
    fail_signal: "Unclear what thread delivers"

  retweet_test:
    question: "Would this get RT'd on its own merit?"
    pass_criteria: "Tweet has standalone value or intrigue"
    fail_signal: "Only makes sense as part of thread"

  minimum_passing_score: "5/5 tests"
```
