# Microskill 0.4: CBF Loader
Skill: S12 Visual Direction
Layer: 0
Model: claude-haiku-4
Version: 1.0

## Purpose

Extract brand guidelines from the Campaign Brief Foundation (CBF) to ensure visual direction aligns with brand identity. Off-brand visuals require expensive rework and damage brand consistency.

## Input

**Required gate:** `GATE_0_INPUT_VALID`

**Required file:** `campaign-brief-foundation.yaml`

## Process

### MUST
1. **Extract brand color palette**
   - Read `brand_guidelines.colors` section from CBF
   - Extract: primary color(s), secondary color(s), accent colors, background colors
   - Document hex codes and usage guidelines (e.g., "Primary for CTAs only")
   - If missing: BLOCK and escalate to human

2. **Extract brand typography**
   - Read `brand_guidelines.typography` section from CBF
   - Extract: headline fonts, body fonts, font weights, font sizes
   - Document hierarchy rules (e.g., "Headlines: Montserrat Bold, Body: Open Sans Regular")
   - If missing: BLOCK and escalate to human

3. **Extract brand visual style**
   - Read `brand_guidelines.visual_style` section from CBF
   - Extract: photography style, illustration style, graphic treatment, mood/tone
   - Document do's and don'ts (e.g., "DO: lifestyle photography. DON'T: stock corporate photos")

4. **Extract logo usage guidelines**
   - Read `brand_guidelines.logo` section from CBF
   - Extract: logo variations (full, icon, wordmark), minimum sizes, clear space, placement rules
   - Document any logo lockups or restrictions

5. **Extract brand voice-to-visual alignment**
   - Cross-reference `brand_voice` section with visual guidelines
   - Document visual tone that matches brand voice:
     - Professional/Authoritative voice → clean, structured visuals
     - Playful/Casual voice → dynamic, colorful visuals
     - Premium/Sophisticated voice → minimal, elegant visuals

6. **Document any visual constraints**
   - Extract: compliance requirements, restricted imagery, platform-specific rules
   - Flag any constraints that affect visual direction (e.g., "No before/after photos per FTC")

### NEVER
- Proceed without brand color palette
- Proceed without brand typography specs
- Invent brand guidelines if missing from CBF
- Assume brand voice translates to visuals without explicit alignment
- Skip constraints section (compliance violations are expensive)

**File:** `s12-0.4-brand-guidelines.yaml`

```yaml
      name: "Energy Orange"

      hex: "#F9FAFB"

      primary: "#111827"
      secondary: "#6B7280"

      - "Clean, structured layouts (matches 'professional')"
      - "Data visualization, proof elements (matches 'authoritative')"

    platform_specific:

      - "YouTube: thumbnail text must be readable at 320px width"

  missing_elements:
    # Document any missing brand guidelines for escalation
    - element: null  # or flag if anything is missing
      escalation_needed: false
```

## Quality Gates

- [ ] Brand color palette extracted (minimum 3 colors with hex codes)
- [ ] Brand typography extracted (headline + body fonts minimum)
- [ ] Visual style guidelines documented
- [ ] Voice-to-visual alignment documented
- [ ] Constraints flagged (if any)
- [ ] Missing elements documented (if any)
- [ ] File written: `s12-0.4-brand-guidelines.yaml`

## Handoff

**Downstream consumer:**
- Layer 1 (1.3 visual-style-guide, 1.4 color-typography-system) enforces these guidelines
- Layer 2 (all microskills) validates against these guidelines
- Layer 4 (4.1 output-assembler) includes brand consistency verification

**BLOCKING GATE:** If brand guidelines are incomplete (missing colors OR typography), create `GATE_0_BRAND_INCOMPLETE` and escalate to human. Do NOT proceed to Layer 1.

```yaml
# GATE_0_BRAND_INCOMPLETE (only if guidelines incomplete)
gate: FAIL
timestamp: 2026-03-05T14:42:00Z
validator: claude-haiku-4
missing_elements:
  - colors: false
  - typography: true  # MISSING
escalation_message: "CBF lacks typography specifications. Cannot produce on-brand visual direction. Request updated CBF with font families and hierarchy."
```

**If brand guidelines are complete:** Layer 0 is complete. Proceed to Layer 1 after creating `GATE_0_INPUT_VALID`.
