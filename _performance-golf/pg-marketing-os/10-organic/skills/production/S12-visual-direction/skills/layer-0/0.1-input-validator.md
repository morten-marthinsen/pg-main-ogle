# Microskill 0.1: Input Validator
Skill: S12 Visual Direction
Layer: 0
Model: claude-haiku-4
Version: 1.0

## Purpose

Validate that all required inputs are present and well-formed before S12 execution begins. This prevents downstream failures from missing or malformed input data.

## Input

Expected files in project directory:
- `campaign-brief-foundation.yaml` (CBF) — includes brand guidelines
- Content draft file(s) from S08-S11 (copy that needs visual support)
- `content-format.yaml` from S04 (specifies content type)
- `platform-strategy.yaml` from S05 (specifies target platform)

## Process

### MUST
1. **Verify CBF exists and contains brand section**
   - Check for `campaign-brief-foundation.yaml`
   - Verify `brand_guidelines` section exists with colors, fonts, style notes
   - If missing: BLOCK and escalate

2. **Verify content draft file exists**
   - Check for content file from S08-S11 (filename pattern: `s08-*`, `s09-*`, `s10-*`, `s11-*`)
   - Verify content is complete (not empty or partial)
   - Extract content type (video, carousel, static_post, thread)

3. **Verify platform specification exists**
   - Check `platform-strategy.yaml` for target platform
   - Verify platform is supported: [youtube, instagram, tiktok, linkedin, twitter]
   - If multiple platforms: validate each has platform-specific visual requirements

4. **Verify format specification exists**
   - Check `content-format.yaml` for format details
   - Extract: video length (if video), slide count (if carousel), post type (if static)

5. **Check for stale S12 artifacts**
   - Scan for existing `s12-*.yaml` or `visual_direction_brief.yaml` files
   - If found: flag as stale, recommend archiving before new execution

6. **Validate project infrastructure**
   - Verify CLAUDE.md, PROJECT-STATE.md, PROGRESS-LOG.md exist
   - If missing: create or escalate per Fix 1 protocol

### NEVER
- Proceed without CBF brand guidelines
- Assume platform from content (always verify explicit platform spec)
- Ignore stale artifacts (creates confusion about "which version")
- Skip infrastructure check
- Invent missing data

## Output

**File:** `s12-0.1-validation-report.yaml`

```yaml
validation_status: PASS | FAIL
timestamp: 2026-03-05T14:32:00Z
validator: claude-haiku-4

inputs_validated:
  cbf_exists: boolean
  cbf_has_brand_guidelines: boolean
  content_draft_exists: boolean
  content_type: string
  platform_specified: boolean
  platform: string
  format_details_exist: boolean

stale_artifacts:
  found: boolean
  files: [array of stale files if found]
  recommendation: "Archive to ~brain/YYYY-MM-DD-HH-MM/"

project_infrastructure:
  claude_md_exists: boolean
  project_state_exists: boolean
  progress_log_exists: boolean

blocking_issues: [array of issues preventing S12 execution]
warnings: [array of non-blocking concerns]

gate_eligible: boolean  # true only if blocking_issues is empty
```

## Quality Gates

- [ ] Validation report written
- [ ] All required inputs confirmed present
- [ ] No blocking issues
- [ ] Stale artifacts identified (if any)
- [ ] gate_eligible = true

## Handoff

**Downstream consumer:** Microskill 0.2 (teaching-loader)

**BLOCKING GATE:** Create `GATE_0_INPUT_VALID` only if validation passes.

```yaml
# GATE_0_INPUT_VALID
gate: PASS
timestamp: 2026-03-05T14:32:00Z
validator: claude-haiku-4
validation_file: s12-0.1-validation-report.yaml
```

**If validation fails:** Do NOT create gate file. Escalate blocking issues to human.
