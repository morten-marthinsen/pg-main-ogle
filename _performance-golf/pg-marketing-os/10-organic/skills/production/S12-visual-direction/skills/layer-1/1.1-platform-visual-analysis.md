# Microskill 1.1: Platform Visual Analysis
Skill: S12 Visual Direction
Layer: 1
Model: claude-sonnet-4.5
Version: 1.0

## Purpose

Analyze platform-specific visual requirements, conventions, and performance patterns. Each platform has unique dimensions, UI constraints, and visual trends that must be understood before creating visual direction.

## Input

**Required gate:** `GATE_0_INPUT_VALID`

**Required files:**
- `s12-0.1-validation-report.yaml` — platform and content type
- `s12-0.2-teaching-context.md` — platform visual conventions
- `s12-0.3-specimens.yaml` — platform-specific performance data

## Process

### MUST
1. **Identify exact platform specifications**
   - Cross-reference target platform against teaching context
   - Document exact dimensions for each asset type:
     - Thumbnails (if video platform)
     - Feed posts (if social platform)
     - Stories/Reels (if story-native platform)
     - Carousel slides (if carousel content)
   - Document aspect ratios and file format requirements

2. **Map platform UI constraints**
   - Identify platform UI elements that obscure visuals:
     - Profile pictures, timestamps, usernames (top-left on Instagram)
     - Like/comment buttons (bottom-center on Instagram)
     - Video player controls (bottom 80px on YouTube)
   - Calculate text-safe zones accounting for UI elements
   - Document platform-specific cropping behavior (e.g., Instagram feed preview crops to center square)

3. **Analyze platform visual trends**
   - Extract patterns from specimens for target platform:
     - Dominant color palettes on platform
     - Typography trends (bold vs. minimal text)
     - Composition styles (face-forward vs. abstract)
     - Visual density (clean/minimal vs. info-rich)
   - Note any platform algorithm preferences (e.g., Instagram penalizes text-heavy images)

4. **Identify platform audience behavior**
   - Document consumption context for platform:
     - YouTube: intentional viewing, desktop + mobile, sound ON
     - Instagram: casual scrolling, mobile-first, sound OFF initially
     - TikTok: rapid scrolling, mobile-only, sound ON
     - LinkedIn: professional context, desktop + mobile, sound OFF
   - Implications for visual direction (e.g., TikTok needs instant hook, no slow build)

5. **Cross-platform comparison (if multi-platform)**
   - If content targets multiple platforms, document differences
   - Flag elements that need platform-specific adaptation:
     - Dimensions (always different)
     - Text overlay (mobile-first platforms need larger text)
     - Visual density (LinkedIn tolerates text-heavy, TikTok does not)

6. **Document platform-specific constraints**
   - Algorithm considerations (Instagram text penalty, YouTube CTR focus)
   - Compliance requirements (political ads, financial disclaimers)
   - Technical limits (file size, video length affecting thumbnail strategy)

### NEVER
- Provide generic dimensions ("square" instead of "1080×1080")
- Assume same visuals work across platforms
- Ignore platform UI elements in safe-zone calculations
- Skip platform audience behavior analysis
- Recommend cross-posting without adaptation
