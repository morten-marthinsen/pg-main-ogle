# 0.4 Upstream Loader

## Purpose
Load and synthesize relevant context from all upstream foundation skills (S01-S05) to inform platform modifiers, dimension weighting, and benchmark setting.

---

## Input Requirements

From validated files:
- `outputs/[campaign]-AIF.yaml`
- `outputs/[campaign]-PSF.yaml`
- `outputs/[campaign]-BVF.yaml`
- `outputs/[campaign]-CAF.yaml`
- `outputs/[campaign]-HLF.yaml`

---

## Execution Protocol

### Step 1: Extract Platform Context (from PSF)

Pull platform-specific requirements:

```yaml
platform_context:
  primary_platform: "[platform]"
  optimal_formats: [list]
  length_targets:
    short_form: "[e.g., 15-30s]"
    mid_form: "[e.g., 60-90s]"
    long_form: "[e.g., 3-10min]"
  technical_specs:
    aspect_ratio: "[e.g., 9:16]"
    resolution: "[e.g., 1080x1920]"
  algorithm_priorities: [list from PSF]
  cultural_fit: "[platform culture notes]"
```

This context will inform **platform_fit** dimension criteria.

---

### Step 2: Extract Audience Psychology (from AIF)

Pull emotional triggers and language patterns:

```yaml
audience_context:
  primary_pain: "[pain]"
  primary_desire: "[desire]"
  emotional_triggers: [list]
  language_to_use: [phrases]
  language_to_avoid: [phrases]
  scroll_stoppers: "[what makes them pause]"
```

This context will inform **emotional_resonance** and **hook_strength** dimension criteria.

---

### Step 3: Extract Voice Requirements (from BVF)

Pull voice dimensions and constraints:

```yaml
voice_context:
  tone: "[e.g., conversational_authority]"
  energy: "[e.g., high_energy]"
  signature_phrases: [list]
  banned_words: [list]
  acceptable_range: "[how much variation is allowed]"
```

This context will inform **voice_alignment** dimension criteria.

---

### Step 4: Extract Content Strategy (from CAF)

Pull pillar and function mix:

```yaml
content_strategy_context:
  primary_pillars: [list]
  function_mix:
    awareness: [%]
    engagement: [%]
    conversion: [%]
    community: [%]
  content_types_prioritized: [list]
```

This context will inform dimension **weighting** (e.g., if 60% engagement, weight **emotional_resonance** higher).

---

### Step 5: Extract Hook Intelligence (from HLF)

Pull approved hook types and patterns:

```yaml
hook_context:
  primary_hook_types: [list of 3-5]
  platform_specific_hooks: [hooks for primary platform]
  hook_templates: [proven patterns]
  testing_insights: "[what's been validated]"
```

This context will inform **hook_strength** dimension criteria and quick-score checklist.

---

### Step 6: Synthesize Context for Framework

Combine all inputs into actionable guidance:

```yaml
framework_guidance:
  dimension_priorities:
    highest: "[based on function_mix and platform]"
    high: "[supporting dimensions]"
    moderate: "[baseline dimensions]"

  platform_emphasis:
    critical_for_algorithm: [dimensions that affect reach]
    critical_for_culture: [dimensions that affect sharing]

  audience_alignment:
    must_trigger: [emotional states from AIF]
    must_avoid: [banned language from BVF]
    scroll_stop_requirement: "[specific to this audience]"
```
