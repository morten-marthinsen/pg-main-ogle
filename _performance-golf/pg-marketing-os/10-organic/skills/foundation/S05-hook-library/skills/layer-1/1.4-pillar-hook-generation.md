# 1.4 Pillar Hook Generation

## Purpose
Generate campaign-specific hooks for each content pillar. This is Phase B — actual hook generation.

---

## Input Requirements
- Taxonomy application from 1.1
- Template library from 1.2
- Platform optimization from 1.3
- Strategic context from 0.4 (audience language, pillars)
- CONCEPT_APPROVED.yaml checkpoint (REQUIRED)

---

## Execution Protocol

### Step 1: Load Pillar Context
For each pillar from CAF:
- Pillar name
- Description
- Topic clusters
- Audience need it addresses (from AIF)
- Business connection

---

### Step 2: Generate Hooks for Each Pillar
For EACH pillar, generate ≥5 hooks across multiple categories:

```yaml
pillar_hooks:
  pillar_1:
    pillar_name: "[Name]"
    pillar_description: "[From CAF]"

    curiosity_hooks:
      - hook: "[Specific hook text]"
        template_used: "[Template ID from 1.2]"
        mechanism: "[Specific mechanism]"
        audience_language: "[Which AIF phrase used]"
        topic_cluster: "[Which cluster from pillar]"
        platform_optimized: true | false

      - hook: "[Another hook]"
        # Same structure

    fear_hooks:
      - hook: "[Specific hook text]"
        template_used: ""
        mechanism: ""
        audience_language: ""
        topic_cluster: ""
        platform_optimized: true | false

    desire_hooks:
      - hook: "[Specific hook text]"
        # Same structure

    identity_hooks:
      - hook: "[Specific hook text]"
        # Same structure

    social_proof_hooks:
      - hook: "[Specific hook text]"
        # Same structure

    # Generate across prioritized categories for this pillar
```

**Requirements:**
- ≥5 hooks per pillar (across multiple categories)
- Use audience language from AIF (verbatim phrases where possible)
- Apply platform optimization rules from 1.3
- Use templates from 1.2 as starting points (but customize)

---

### Step 3: Hook Quality Check
For each generated hook, verify:

**Specificity check:**
- [ ] Hook uses specific numbers/details (not generic)
- [ ] Hook references pillar topic clusters
- [ ] Hook uses audience language from AIF

**Platform check:**
- [ ] Hook length appropriate for primary platform
- [ ] Hook structure matches platform_optimization rules
- [ ] Visual/verbal balance correct for platform

**Voice check (preliminary):**
- [ ] Hook doesn't violate anti_voice patterns
- [ ] Hook tone matches voice_attributes

---

### Step 4: Category Distribution Tracking
Track hook distribution across categories:

```yaml
category_distribution:
  pillar_1:
    curiosity: 0
    fear: 0
    desire: 0
    social_proof: 0
    identity: 0
    controversy: 0
    story: 0
    total: 0  # Must be ≥5

  # Repeat for all pillars

overall_distribution:
  curiosity: 0
  fear: 0
  desire: 0
  social_proof: 0
  identity: 0
  controversy: 0
  story: 0
  total: 0  # Will be pillar_count × 5 minimum
```

**Verify distribution reflects campaign_category_priority from 1.1**

---

## Quality Gates
- [ ] ≥5 hooks per pillar generated
- [ ] Each hook has template_used, mechanism, audience_language, topic_cluster
- [ ] Category distribution reflects priority from 1.1
- [ ] All hooks platform-optimized for primary platform
- [ ] Total hooks ≥ (pillar_count × 5)
- [ ] hook_generation_metrics.target_met = true

---

Pass pillar hooks to: **1.5 Voice Calibration**
