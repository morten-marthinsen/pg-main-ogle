# 0.3 Specimen Loader

## Purpose
Load content architecture specimens from successful creators to inform structural decisions.

---

## Input Requirements
- Teaching summary from 0.2
- Access to specimen library

---

## Execution Protocol

### Step 1: Load Architecture Examples
**Source:** Specimen library (content architecture examples)

**Extract for each specimen:**
- Creator name and niche
- Number of pillars used
- Pillar themes and percentages
- Series formats employed
- Posting cadence structure
- Funnel integration approach

---

### Step 2: Load Series Format Specimens
**Source:** Specimen library (series formats that drive engagement)

**Extract:**
- Series name and concept
- Hook pattern used
- Episode structure
- Frequency and consistency
- Performance indicators (if available)

---

### Step 3: Load Funnel Integration Specimens
**Source:** Specimen library (funnel integration examples)

**Extract:**
- Content type to CTA mapping
- Hard vs. soft CTA distribution
- Lead magnet integration points
- Conversion content examples

---

### Step 4: Pattern Recognition
Identify patterns across specimens:
- Common pillar count (typically 3-5)
- Effective pillar themes by vertical
- Successful series structures
- High-performing content-to-conversion paths

---

- **File:** `0.3-specimen-analysis.yaml`


---

- [ ] At least 2 funnel specimens loaded
