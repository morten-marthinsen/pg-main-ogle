# 0.1 Input Validator

## Purpose
Validate that all upstream outputs exist and contain required fields before architecture design begins.

---

## Input Requirements

Required upstream files:
- `[project]/S01-audience-intelligence/outputs/[campaign]-AIF.yaml`
- `[project]/S02-platform-strategy/outputs/[campaign]-PSF.yaml`
- `[project]/S03-brand-voice/outputs/[campaign]-BVF.yaml`

Business context (user-provided):
```yaml
business_context:
  core_offerings: []
  content_goals: {}
  existing_content: {}
  creator_preferences: {}
```

---

## Execution Protocol

### Step 1: File Existence Check
Verify all required upstream files exist:
- AIF.yaml (audience intelligence)
- PSF.yaml (platform strategy)
- BVF.yaml (brand voice)

**If any file missing → HALT. Request file before proceeding.**

---

### Step 2: Required Field Validation

**From AIF, verify presence of:**
- `pain_points` (list with ≥3 entries)
- `desires` (list with ≥3 entries)
- `language_patterns` (customer voice data)

**From PSF, verify presence of:**
- `primary_platform` (single platform selection)
- `content_formats` (list of formats)
- `posting_cadence` (frequency guidance)

**From BVF, verify presence of:**
- `voice_attributes` (brand voice definition)
- `tone_range` (acceptable tones)
- `anti_voice` (what to avoid)

**If any required field missing → FAIL validation. Return to upstream skill.**

---

### Step 3: Business Context Validation

**From user-provided business_context, verify:**
- `core_offerings` has ≥1 entry
- `content_goals.primary` is defined
- `creator_preferences.formats_enjoy` is defined (can be empty list if truly none)

**If core fields missing → REQUEST from user before proceeding.**

---

## Output
- **File:** `0.1-input-validation-report.yaml`
- **Format:** YAML

```yaml
validation_status: PASS | FAIL
timestamp: "[ISO 8601]"

file_checks:
  AIF_exists: true | false
  PSF_exists: true | false
  BVF_exists: true | false

field_checks:
  AIF_pain_points: true | false
  AIF_desires: true | false
  PSF_primary_platform: true | false
  PSF_content_formats: true | false
  BVF_voice_attributes: true | false

business_context_checks:
  core_offerings_defined: true | false
  content_goals_defined: true | false
  creator_preferences_defined: true | false

missing_items: []
blockers: []
warnings: []
```

---

## Quality Gates
- [ ] All upstream files exist
- [ ] All required fields present in AIF, PSF, BVF
- [ ] Business context has core_offerings and content_goals
- [ ] validation_status = PASS

---

## Handoff
Pass validation report to: **0.2 Teaching Loader**

If validation_status = FAIL → HALT and report missing items to user.
