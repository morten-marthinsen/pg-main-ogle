# 1.5 Funnel Integration

## Purpose
Connect content architecture to revenue path through funnel mapping and CTA strategy.

---

## Input Requirements
- Pillar concepts from 1.1 (approved and named)
- Function mapping from 1.2
- Format matrix from 1.4
- Business context (core offerings)

---

## Execution Protocol

### Step 1: Map Content Funnel Stages
Define content role at each funnel stage:

```yaml
content_funnel:
  top_of_funnel:
    content_role: |
      Attract and introduce — problem awareness, value demonstration
    content_types: ["[Format]", "[Format]"]
    primary_pillars: ["[Pillar]", "[Pillar]"]
    call_to_action: "Follow, Subscribe, Engage"
    next_step: "Middle of funnel content"

  middle_of_funnel:
    content_role: |
      Educate and nurture — solution education, trust building
    content_types: ["[Format]", "[Format]"]
    primary_pillars: ["[Pillar]", "[Pillar]"]
    call_to_action: "Join email list, Download lead magnet"
    next_step: "Bottom of funnel content or direct offer"

  bottom_of_funnel:
    content_role: |
      Convert and activate — social proof, direct value demo
    content_types: ["[Format]", "[Format]"]
    primary_pillars: ["[Pillar]", "[Pillar]"]
    call_to_action: "Purchase, Book call, Apply"
    next_step: "Customer onboarding"
```

---

### Step 2: Design CTA Strategy
Define primary and soft CTAs:

```yaml
cta_strategy:
  primary_ctas:
    - cta: "[What we ask: 'Join my email list']"
      content_types: ["[Which formats]"]
      frequency: "[How often: '15-20% of content']"
      destination: "[Where it leads: URL/landing page]"
      funnel_stage: "MOFU"
      expected_conversion: "[%]"

    - cta: "[Second primary CTA]"
      content_types: []
      frequency: ""
      destination: ""
      funnel_stage: "BOFU"
      expected_conversion: ""

  soft_ctas:
    - cta: "Follow for more content like this"
      content_types: ["All awareness content"]
      frequency: "80-90% of content"
      funnel_stage: "TOFU"

    - cta: "Tag someone who needs to hear this"
      content_types: ["Engagement-focused content"]
      frequency: "40-50% of content"
      funnel_stage: "TOFU/MOFU"
```

**CRITICAL:** At least 1 primary CTA must be defined.

---

### Step 3: Define Lead Magnet Integration
If lead magnets exist or are planned:

```yaml
lead_magnets:
  - name: "[Lead magnet title]"
    description: "[What it is]"
    target_audience: "[Who it's for]"
    content_that_promotes: ["[Format]", "[Pillar]"]
    promotion_frequency: "[How often mentioned]"
    funnel_position: "TOFU → MOFU transition"
```

---

### Step 4: Map Offer Integration
For each core offering, define content strategy:

```yaml
offer_integration:
  - offer: "[Product/service name]"
    price_point: "[Range]"
    target_customer: "[Avatar]"

    content_types_that_support: ["[Format]", "[Format]"]
    primary_pillar: "[Which pillar]"

    mention_frequency: "[How often: 'Weekly']"
    approach: "Indirect (case studies, results) | Direct (sales content)"

    conversion_path: |
      [How content → offer: 'Educational content → lead magnet → email sequence → offer']
```

---

### Step 5: Calculate Conversion Expectations
Based on industry benchmarks and function mapping:

```yaml
conversion_metrics:
  awareness_to_follower: "[% conversion expectation]"
  follower_to_email: "[% conversion expectation]"
  email_to_customer: "[% conversion expectation]"

  content_volume_required:
    for_1000_followers: "[Posts needed]"
    for_100_email_subs: "[Posts needed]"
    for_10_customers: "[Posts needed]"
```

```yaml
  awareness_to_follower: ""
  follower_to_email: ""
  email_to_customer: ""
  content_volume_required: {}

  primary_cta_count: 0
  all_stages_mapped: true | false

```

---

## Quality Gates
- [ ] All 3 funnel stages defined (TOFU/MOFU/BOFU)
- [ ] At least 1 primary CTA defined
- [ ] Soft CTAs defined
- [ ] Each core offering has content support strategy
- [ ] Conversion path documented for primary offer
- [ ] funnel_verification.primary_cta_count ≥ 1
