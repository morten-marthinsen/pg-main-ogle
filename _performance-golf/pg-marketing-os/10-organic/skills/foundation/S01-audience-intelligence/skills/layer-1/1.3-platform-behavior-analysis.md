# 1.3 Platform Behavior Analysis

## Purpose
Understand HOW they consume content across platforms. This directly feeds S02 Platform Strategy and determines WHERE and WHEN to publish.

---

## Input Requirements
- demographic-foundation.yaml (from 1.1)
- psychographic-depth.yaml (from 1.2)

---

## Execution Protocol

### Step 1: Primary Platforms Identification
```
IDENTIFY ≥2 platforms where target audience actively engages:

FOR EACH platform:
  DOCUMENT:
    - Platform name
    - Time spent estimate (daily/weekly)
    - Usage pattern (active posting vs passive consumption)
    - Content preference (what they engage with most)

EVIDENCE from:
  - Competitor audience analysis
  - Demographics (age→platform correlation)
  - Category norms (B2B→LinkedIn, Gen Z→TikTok)

MUST have MINIMUM 2 platforms identified.
```

### Step 2: Consumption Pattern Analysis
```
DETERMINE:

PREFERRED FORMAT:
  - Video (short/long)
  - Text (threads/articles)
  - Audio (podcasts)
  - Image (infographics/carousels)

PREFERRED LENGTH:
  - Short (<60 sec)
  - Medium (2-10 min)
  - Long (>10 min)

CONSUMPTION TIME:
  - Morning (5-9am)
  - Midday (11am-2pm)
  - Evening (5-9pm)
  - Late night (9pm+)

CONSUMPTION CONTEXT:
  - Commute (mobile, quick)
  - Desk work (background, multitasking)
  - Evening wind-down (deep focus)
```

### Step 3: Engagement Pattern Mapping
```
WHAT THEY LIKE:
  - Content types that get likes/hearts
  - Format patterns
  - Topic patterns

WHAT THEY COMMENT ON:
  - Controversial takes
  - Questions/polls
  - Relatable stories
  - Educational content

WHAT THEY SHARE:
  - High utility (how-to)
  - Identity-reinforcing (this is so me)
  - Aspirational (goals)
  - Funny/entertaining

WHAT THEY SAVE:
  - Reference material
  - Step-by-step guides
  - Resource lists
  - Templates/frameworks
```

### Step 4: Active Time Windows
```
IDENTIFY best posting times:

WEEKDAY WINDOWS:
  - [Time range 1]: [Activity level + type]
  - [Time range 2]: [Activity level + type]

WEEKEND WINDOWS:
  - [Time range 1]: [Activity level + type]
  - [Time range 2]: [Activity level + type]

TIMEZONE REFERENCE:
  - Primary timezone for scheduling

EVIDENCE:
  - Platform analytics (if available)
  - Competitor engagement timing
  - Category norms
  - Demographic patterns (work schedule correlation)
```

---

## Output
- **File:** `platform-behavior.yaml`
- **Format:** YAML

```yaml
platform_behavior:
  primary_platforms:
    - platform: "[Platform name]"
      time_spent: "[Daily/weekly estimate]"
      usage_pattern: "[Active/passive]"
      content_preference: "[What they engage with]"
      evidence: "[Source]"
    # ≥2 platforms required

  consumption_patterns:
    preferred_format: "[Video/text/audio/image]"
    preferred_length: "[Short/medium/long]"
    consumption_time: "[When they consume]"
    consumption_context: "[How/where they consume]"

  engagement_patterns:
    what_they_like: "[Patterns]"
    what_they_comment_on: "[Triggers]"
    what_they_share: "[Share-worthy content]"
    what_they_save: "[Save-worthy content]"

  best_times_active:
    weekday_windows:
      - time: "[Hour range]"
        activity_level: high | medium | low
        activity_type: "[What they do]"
    weekend_windows:
      - time: "[Hour range]"
        activity_level: high | medium | low
        activity_type: "[What they do]"
    timezone: "[Reference timezone]"

insights:
  optimal_format_platform_combo: "[Recommendation]"
  highest_engagement_time: "[When]"
  platform_priority_ranking: [1, 2, 3]
```

---

## Quality Gates
- [ ] ≥2 primary platforms identified
- [ ] Consumption patterns fully documented (format, length, time, context)
- [ ] All 4 engagement pattern types populated (like, comment, share, save)
- [ ] ≥2 weekday time windows identified
- [ ] Timezone specified for scheduling reference
