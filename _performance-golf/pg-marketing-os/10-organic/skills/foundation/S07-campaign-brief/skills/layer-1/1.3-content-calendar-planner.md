# 1.3 Content Calendar Planner

## Purpose
Build a detailed 30-day (minimum) content calendar with specific daily assignments. Every entry must be actionable, not generic.

---

## Input Requirements

From Layer 1:
- `1.1-synthesis.yaml` (pillars, formats, hooks, function mix)
- `1.2-objectives.yaml` (timeline)

From Layer 0:
- `0.4-strategic-synthesis.yaml` (strategic priorities)

---

## Execution Protocol

### Step 1: Determine Calendar Structure

Based on posting cadence from PSF:

```yaml
calendar_parameters:
  posting_frequency: "[daily, 5x/week, 3x/week, etc.]"
  platforms: "[primary + secondary]"
  days_to_plan: 30  # Minimum, extend if campaign longer
  total_pieces: "[X pieces over 30 days]"
```

---

### Step 2: Distribute Content Functions

Apply CAF function mix across calendar:

```yaml
function_distribution:
  awareness: "[X pieces - Y%]"
  engagement: "[X pieces - Y%]"
  conversion: "[X pieces - Y%]"
  community: "[X pieces - Y%]"
```

---

### Step 3: Map Pillars to Calendar

Ensure each pillar gets appropriate coverage:

```yaml
pillar_distribution:
  pillar_1: "[X pieces - Y%]"
  pillar_2: "[X pieces - Y%]"
  pillar_3: "[X pieces - Y%]"
```

---

### Step 4: Plan Each Day

**CRITICAL:** Every entry MUST have:
- Day of week
- Content type (specific format)
- Pillar
- Hook type
- Function
- Working title (specific concept)

**FORBIDDEN:**
- "Educational content"
- "Engagement post"
- "TBD"

**REQUIRED SPECIFICITY:**

```yaml
# GOOD ENTRY:
- day: "Monday, Week 1"
  content_type: "Instagram Reel (15-30s)"
  pillar: "Performance Fundamentals"
  hook_type: "Pain Amplification"
  function: "Awareness"
  working_title: "Why Your Driver Swing Feels Harder Than Your Wedge"
  notes: "Use trending audio, text overlay for key points"

# BAD ENTRY (too vague):
- day: "Monday"
  content_type: "Video"
  pillar: "Golf"
  hook_type: "Hook"
  function: "Engagement"
  working_title: "Golf tips"
```

---

### Step 5: Plan Week by Week

**Week 1:** [Theme/focus]
- Monday: [specific entry]
- Tuesday: [specific entry]
- Wednesday: [specific entry]
- Thursday: [specific entry]
- Friday: [specific entry]
- Saturday: [specific entry]
- Sunday: [specific entry]

**Week 2:** [Theme/focus]
[Continue...]

**Week 3:** [Theme/focus]
[Continue...]

**Week 4:** [Theme/focus]
[Continue...]

**Week 5+:** [If campaign longer than 30 days]

---

### Step 6: Identify Hero Content

Flag 3-5 "hero" pieces that get extra production effort:

```yaml
hero_content:
  - title: "[Working title]"
    format: "[Type]"
    platform: "[Where]"
    production_date: "[When to produce]"
    publish_date: "[When to launch]"
    why_hero: "[Strategic importance]"
    quality_target: 80+  # VSF score
```

---

### Step 7: Plan Recurring Content

Any series that repeat:

```yaml
recurring_content:
  - series_name: "[Name]"
    frequency: "[Every Monday, etc.]"
    format: "[Type]"
    pillar: "[Which pillar]"
    episodes_in_calendar: [count]
```

---

## Output

**File:** `layer-1/1.3-calendar.yaml`

**Format:**
```yaml
timestamp: "[ISO 8601]"

calendar_parameters: {...}
function_distribution: {...}
pillar_distribution: {...}

content_calendar:
  week_1:
    theme: "[Optional week theme]"
    content:
      - day: "Monday"
        content_type: "[Specific format]"
        pillar: "[Pillar]"
        hook_type: "[Hook type]"
        function: "[Function]"
        working_title: "[Specific concept]"
        notes: "[Production notes]"
      # [... repeat for each posting day]

  week_2:
    theme: "[...]"
    content: [...]

  # [Continue for 30+ days]

hero_content: [...]
recurring_content: [...]

calendar_stats:
  total_pieces: [X]
  days_planned: [Y]
  pieces_per_week_avg: [Z]
  function_distribution_achieved: true
  pillar_distribution_achieved: true
```

**Minimum size:** 5KB (comprehensive calendar)

---

## Quality Gates

- [ ] At least 30 days planned
- [ ] Every entry has all required fields
- [ ] No generic/vague entries ("educational post")
- [ ] Function mix matches CAF percentages (±5%)
- [ ] Pillar distribution balanced
- [ ] 3-5 hero pieces identified
- [ ] Working titles are specific concepts
