# 4.1 CBF Assembler

## Purpose
Compile all Layer 1 outputs into the final Campaign Brief File (CBF) — THE MASTER KEY that unlocks all production skills.

---

## Input Requirements

From Layer 1:
- `1.1-synthesis.yaml`
- `1.2-objectives.yaml`
- `1.3-calendar.yaml`
- `1.4-platform-plan.yaml`
- `1.5-voice-hooks.yaml`
- `1.6-quality-distribution.yaml`

From Layer 0:
- `0.2-foundation-context.yaml` (source file paths)

---

## Execution Protocol

### Step 1: Assemble Header

```yaml
campaign_name: "[Descriptive campaign name]"
date_created: "YYYY-MM-DD"
owner: "[Owner name]"
version: "1.0"
mode: "[Mode A: Downstream or Mode B: Organic Native]"
```

---

### Step 2: Assemble Objective Section

Pull from 1.2-objectives.yaml:

```yaml
objective:
  primary_goal: |
    [Specific, measurable goal statement]

  success_metrics:
    - metric: "[name]"
      current: [baseline]
      target: [goal]
      measurement: "[how]"
    # [All metrics from 1.2]

  timeline:
    start_date: "YYYY-MM-DD"
    end_date: "YYYY-MM-DD"
    milestones: [...]
```

---

### Step 3: Assemble Audience Section

Pull from 1.1-synthesis.yaml (AIF):

```yaml
audience_summary:
  target_segment: |
    [2-3 sentence description]

  key_insight: |
    [Most important finding]

  language_to_use:
    - "[phrase 1]"
    - "[phrase 2]"
    # [5-7 phrases]

  primary_pain: "[pain statement]"
  primary_desire: "[desire statement]"
```

---

### Step 4: Assemble Platform Section

Pull from 1.4-platform-plan.yaml:

```yaml
platform_plan:
  primary_platform: "[platform]"
  formats: [list]
  posting_cadence: "[frequency]"
  optimal_times: [times]
  secondary_platforms: [list]

  engagement_protocol:
    first_hour: |
      [Actions in first 60 minutes]
    ongoing: |
      [Daily engagement routine]
```

---

### Step 5: Assemble Voice Section

Pull from 1.5-voice-hooks.yaml:

```yaml
voice_summary:
  tone_for_campaign: |
    [Tone description]
  key_phrases: [list]
  phrases_to_avoid: [list]
  energy_level: "[high/medium/low with context]"
```

---

### Step 6: Assemble Content Section

Pull from 1.3-calendar.yaml:

```yaml
content_plan:
  pillar_focus:
    - pillar: "[name]"
      description: "[1 sentence]"
      percentage: "[% of content]"

  content_calendar:
    week_1: {...}
    week_2: {...}
    week_3: {...}
    week_4: {...}
    # [All weeks from 1.3]

  hero_content:
    - title: "[Working title]"
      format: "[Type]"
      platform: "[Where]"
      publish_date: "[When]"

  recurring_content:
    - series_name: "[Name]"
      frequency: "[Cadence]"

  content_function_mix:
    awareness: [%]
    engagement: [%]
    conversion: [%]
    community: [%]
```

---

### Step 7: Assemble Hooks Section

Pull from 1.5-voice-hooks.yaml:

```yaml
hook_strategy:
  primary_hook_types:
    - "[type 1]"
    - "[type 2]"
    - "[type 3]"

  specific_hooks_approved:
    - hook: "[exact text]"
      for_content_type: "[where]"
    # [10-15 hooks]

  hook_testing_plan: |
    [Testing approach]
```

---

### Step 8: Assemble Quality Section

Pull from 1.6-quality-distribution.yaml:

```yaml
virality_targets:
  minimum_score: [X]
  target_score: [Y]
  dimension_priorities:
    1: "[dimension]"
    2: "[dimension]"
    3: "[dimension]"
```

---

### Step 9: Assemble Distribution Section

Pull from 1.6-quality-distribution.yaml:

```yaml
distribution_strategy:
  cross_platform_sequence: |
    [How content moves]

  amplification_tactics:
    - tactic: "[name]"
      trigger: "[when]"
      action: "[what]"

  engagement_protocol: |
    [From platform plan]

  repurpose_plan: |
    [How 1 piece becomes many]
```

---

### Step 10: Assemble Approval Section

```yaml
approval_chain:
  content_approval: "[Who approves content]"
  final_approval: "[Who gives final sign-off]"
```

---

### Step 11: Add Source Files

```yaml
source_files:
  AIF: "core-message/S01-audience-intelligence/outputs/[name]-AIF.yaml"
  PSF: "core-message/S02-platform-strategy/outputs/[name]-PSF.yaml"
  BVF: "core-message/S03-brand-voice/outputs/[name]-BVF.yaml"
  CAF: "core-message/S04-content-architecture/outputs/[name]-CAF.yaml"
  HLF: "core-message/S05-hook-library/outputs/[name]-HLF.yaml"
  VSF: "core-message/S06-virality-scoring/outputs/[name]-VSF.yaml"
```

---

## Output

**File:** `outputs/[campaign-name]-CBF.yaml`

**Format:** Complete CBF template as specified in original SKILL.md

**Required sections (ALL must be present):**
- campaign_name, date_created, owner, version
- objective (with metrics and timeline)
- audience_summary
- platform_plan
- voice_summary
- content_plan (with 30+ day calendar)
- hook_strategy
- virality_targets
- distribution_strategy
- approval_chain
- source_files

**Minimum size:** 5KB (comprehensive brief)

---

## Quality Gates

- [ ] All required sections present
- [ ] Objective is measurable
- [ ] Content calendar ≥30 days
- [ ] Platform cadence defined
- [ ] Hook strategy complete (≥3 types, ≥10 specific hooks)
- [ ] Virality targets set
- [ ] File size ≥5KB
- [ ] No placeholder/TBD values

---

## Handoff

Pass CBF to: **4.2 Execution Log**

The CBF file is now ready to unlock Gate G07 and enable ALL production skills (S08-S14).
