# 1.1 Data Collection

## Purpose
Collect platform metrics per content piece (views, engagement rate, saves, shares, comments, reach, impressions) and normalize across platforms for apples-to-apples comparison.

## Assigned Model
`claude-opus-4` (Layer 1 analysis)

## Input
- 0.3-aligned-dataset.json
- Performance data file (raw platform metrics)
- 0.2-teaching-context.json (normalization formulas)

## Process

### MUST
1. **Extract raw metrics per content piece** — Views, engagement_rate, saves, shares, comments, reach, impressions from performance data
2. **Normalize cross-platform** — Apply teaching formulas to make Instagram vs TikTok vs YouTube comparable
   - Normalize engagement rate: (likes + comments + shares + saves) / reach
   - Normalize view-through rate for videos
   - Adjust for platform size/algorithm differences
3. **Calculate derived metrics** — Save rate, share rate, comment rate, virality coefficient (shares/views)
4. **Handle missing data** — Flag incomplete metrics, use platform averages if appropriate, never fabricate
5. **Timestamp each metric** — Record when data was collected (performance evolves over time)
6. **Quality check** — Flag anomalies (e.g., engagement rate >100%, impossible share counts)
7. **Output normalized dataset** — Ready for comparison

### NEVER
- Compare raw metrics across platforms without normalization
- Fabricate missing data
- Skip quality checks for anomalies
- Ignore platform-specific context (e.g., YouTube Shorts vs long-form)
