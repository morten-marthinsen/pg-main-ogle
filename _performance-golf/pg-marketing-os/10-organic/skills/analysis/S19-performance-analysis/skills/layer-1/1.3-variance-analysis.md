# 1.3 Variance Analysis

## Purpose
For each content piece with >15% variance, analyze WHY the prediction was off. Determine root cause: was the prediction model wrong or was execution different than planned?

## Assigned Model
`claude-opus-4` (Layer 1 analysis)

## Input
- 1.2-prediction-comparison.json (flagged variances)
- 0.3-aligned-dataset.json (original content specs)
- Final published content (to check execution fidelity)
- 0.2-teaching-context.json (root cause taxonomy)

## Process

### MUST
1. **For each flagged variance (>15%):**
   - Compare predicted content to published content — was execution faithful to plan?
   - Identify execution drift: hook changed, platform changed, timing changed, format changed
   - Categorize root cause:
     - **Hook failure:** Hook didn't land as predicted (overestimated or underestimated)
     - **Platform misfit:** Platform algorithm/audience responded differently than model predicted
     - **Timing error:** Publish time/day/season affected performance
     - **Audience mismatch:** Target audience assumption was wrong
     - **Algorithm shift:** Platform algorithm changed during campaign
     - **Execution drift:** Published content deviated from plan
2. **Assign primary root cause** — One main driver per flagged variance
3. **Assess prediction model accuracy** — Was the prediction methodology sound or does it need refinement?
4. **Identify learning opportunity** — What can be learned to improve future predictions?
5. **Rate confidence** — High (clear evidence), medium (strong correlation), low (hypothesis)
6. **Document analysis** — Detailed explanation for each variance

### NEVER
- Blame "algorithm randomness" without evidence
- Skip execution fidelity check
- Assign multiple root causes without prioritizing
- Make excuses for prediction misses
