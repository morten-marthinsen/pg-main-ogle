# 1.2 Prediction Comparison

## Purpose
Compare actual performance against predicted virality scores and calculate variance per dimension (EA, SC, PI, PF, SH) to identify where predictions were accurate vs off.

## Assigned Model
`claude-opus-4` (Layer 1 analysis)

## Input
- 1.1-collected-metrics.json
- 0.3-aligned-dataset.json (original predictions)
- 0.2-teaching-context.json (variance threshold: 15%)

## Process

### MUST
1. **Map metrics to virality dimensions** — Translate platform metrics back to EA/SC/PI/PF/SH framework
   - EA (Emotional Arousal): engagement rate, comment sentiment
   - SC (Social Currency): share rate, save rate
   - PI (Practical Information): view-through rate, completion rate
   - PF (Platform Fit): reach vs impressions ratio
   - SH (Shareability): virality coefficient, share/save ratio
2. **Calculate variance per dimension** — (Actual - Predicted) / Predicted * 100
3. **Flag significant variance** — Any dimension with >15% variance gets flagged for deeper analysis
4. **Calculate composite variance** — Overall performance vs prediction
5. **Categorize outcomes** — Overperformed (>15% above), matched (±15%), underperformed (>15% below)
6. **Identify surprise winners/losers** — Content that vastly exceeded or missed predictions
7. **Output comparison dataset** — Ready for variance analysis

### NEVER
- Skip dimension-level comparison (composite only is insufficient)
- Ignore small sample effects (first 3 days can be volatile)
- Compare without accounting for platform differences
- Flag variance without context (some variance is random)
