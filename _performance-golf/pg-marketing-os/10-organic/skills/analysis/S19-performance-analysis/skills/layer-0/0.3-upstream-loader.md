# 0.3 Upstream Loader

## Purpose
Load published content data from S14/S15 and original virality predictions from S06/S13 to enable prediction vs actual comparison.

## Assigned Model
`claude-haiku-4` (Layer 0 loading)

## Input
- S14/S15 output packages (published content inventory)
- S06 output (video content + virality scores)
- S13 output (text content + virality scores)
- Content ID mapping

## Process

### MUST
1. **Load S14/S15 content inventory** — Published dates, platforms, content IDs, final copy
2. **Load S06 virality predictions** — For video content: EA, SC, PI, PF, SH scores + composite
3. **Load S13 virality predictions** — For text content: EA, SC, PI, PF, SH scores + composite
4. **Align content IDs** — Match prediction IDs to published content IDs (handle any ID transformations)
5. **Extract metadata** — Platform, content function, hook type, publish date/time
6. **Verify prediction completeness** — Every published piece has corresponding prediction scores
7. **Output aligned dataset** — Ready for Layer 1 comparison

### NEVER
- Analyze content without original predictions
- Mix up video/text prediction sources
- Proceed with mismatched content IDs
- Omit metadata needed for pattern detection

## Quality Gates
1. All published content has predictions
2. Content IDs aligned
3. Metadata complete

## Handoff
Passes aligned dataset to Layer 1 (1.1-data-collection.md)
