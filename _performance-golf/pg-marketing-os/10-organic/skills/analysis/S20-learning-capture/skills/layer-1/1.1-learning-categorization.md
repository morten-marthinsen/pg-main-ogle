# 1.1 Learning Categorization

## Purpose
Categorize each S19 learning flag into: teaching_update, specimen_addition, skill_refinement, gate_adjustment. Assign confidence and impact to prioritize actions.

## Assigned Model
`claude-opus-4` (Layer 1 intelligence)

## Input
- 0.3-upstream-package.json (learning flags, patterns, evidence)
- 0.2-teaching-context.json (current teaching files)
- S20-ANTI-DEGRADATION.md (statistical rigor requirements)

## Process

### MUST
1. **For each learning flag from S19:**
   - Confirm category (teaching_update, specimen_addition, skill_refinement, gate_adjustment)
   - Validate confidence level:
     - **High:** ≥3 data points + statistical significance
     - **Medium:** 2 data points + strong correlation
     - **Low:** 1 data point (hypothesis only)
   - Assess impact:
     - **High:** Affects prediction accuracy or output quality significantly
     - **Medium:** Incremental improvement
     - **Low:** Minor refinement
2. **Apply statistical rigor rules:**
   - REJECT any flag claiming "teaching_update" with <3 data points
   - DOWNGRADE confidence if evidence is weak
   - FLAG any single-instance learnings
3. **Determine actionability:**
   - **Act Now:** High confidence + High/Medium impact
   - **Defer with Trigger:** Medium/Low confidence + define 3-campaign revisit
   - **Reject:** Low confidence + insufficient evidence
4. **Link to hypotheses** — Connect each actionable learning to corresponding H# from S19
5. **Output categorized learnings** — Ready for Layer 1 action microskills

### NEVER
- Accept single-instance learnings as "High confidence"
- Skip statistical rigor check
- Categorize without assessing impact
- Proceed with learnings that lack evidence
