# 1.3 Specimen Promotion

## Purpose
Identify high-performing content worth promoting to specimen library. Include performance data, platform, and why it's exemplary.

## Assigned Model
`claude-opus-4` (Layer 1 intelligence)

## Input
- 1.1-learning-categories.json (specimen_addition flags marked "act_now")
- S19 PAR (performance data)
- S19 aligned dataset (content metadata)
- Current specimen library inventory

## Process

### MUST
1. **For each specimen_addition flag:**
   - Verify performance threshold met:
     - Virality composite ≥70 OR
     - ≥2 dimensions with score ≥8.0 OR
     - Overperformed prediction by ≥20%
   - Extract actual performance metrics (not predictions)
   - Identify what makes it exemplary (specific hook, pacing, structure)
   - Categorize specimen (hook type, content function, platform)
   - Check specimen library balance (avoid over-representation of one type)
2. **Create specimen entry:**
   - Content ID and final copy
   - Performance data (actual metrics)
   - Platform and publish date
   - Why exemplary (specific elements)
   - Teaching points (what future content can learn)
3. **Add to specimen library** — Write specimen file to library
4. **Update specimen index** — Add to categorized index
5. **Output specimen promotions** — Summary of additions

### NEVER
- Promote content without actual performance data
- Skip performance threshold check
- Ignore specimen library balance
- Add specimen without "why exemplary" explanation
- Forget to update specimen index

2. Actual metrics included
3. "Why exemplary" explained

6. Library balance checked
