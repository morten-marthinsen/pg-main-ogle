# 1.4 Prediction Calibration

## Purpose
Analyze if virality scoring thresholds need recalibration. Compare predicted vs actual distributions. Recommend threshold changes with statistical justification.

## Assigned Model
`claude-opus-4` (Layer 1 intelligence)

## Input
- S19 PAR (prediction comparison data)
- 1.1-learning-categories.json
- Current scoring model baselines
- Historical prediction accuracy data (if available)

## Process

### MUST
1. **Analyze prediction accuracy distribution:**
   - Calculate mean variance per dimension (EA, SC, PI, PF, SH)
   - Identify systematic bias (consistently over or under predicting)
   - Check if variance is random or directional
2. **Assess calibration needs:**
   - If mean variance >10% on any dimension → recalibration candidate
   - If 80%+ of predictions in one direction (all high or all low) → systematic bias
   - If variance reduced over time → model learning correctly
3. **For each dimension needing recalibration:**
   - Calculate recommended adjustment (e.g., "reduce SC baseline by 0.8 points")
   - Show distribution before/after adjustment
   - Estimate impact on prediction accuracy
   - Cite evidence (minimum 10 content pieces for dimension-level adjustment)
4. **Draft calibration updates:**
   - Which teaching file sections to modify
   - Specific baseline/weight changes
   - Expected accuracy improvement
5. **Statistical justification:**
   - Show variance distribution
   - Calculate statistical significance
   - Require minimum sample size (10 pieces)

### NEVER
- Recalibrate based on <10 content pieces
- Adjust without checking for systematic bias (random variance doesn't warrant change)
- Overcorrect (adjust by full variance amount)
- Skip impact estimation
