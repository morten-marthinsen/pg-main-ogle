# 0.2 Teaching Loader

## Purpose
Load current teaching YAML files that may need updates based on S19 learning flags. These are the system's knowledge base that evolves with each campaign.

## Assigned Model
`claude-haiku-4` (Layer 0 loading)

## Input
- Teaching file registry
- S19 learning flags (to determine which teachings need loading)
- Current teaching file versions

## Process

### MUST
1. **Load virality scoring teachings** — S06/S13 scoring formulas, dimension weights, hook type baselines
2. **Load pattern detection teachings** — Statistical thresholds, correlation rules, sample size requirements
3. **Load platform teachings** — Platform-specific normalization formulas, algorithm behaviors
4. **Load hook type teachings** — Performance baselines per hook type, known strengths/weaknesses
5. **Load gate threshold configs** — Current pass/fail thresholds for S06, S13, and other skills
6. **Check teaching versions** — Record version numbers and last update dates
7. **Identify stale teachings** — Flag any teaching files not updated in >90 days
8. **Output loaded context** — Summary of teachings ready for Layer 1 updates

### NEVER
- Load outdated teaching versions
- Skip version tracking
- Omit teachings referenced in S19 learning flags
- Load corrupt/unparseable YAML files

2. Versions recorded
3. Stale teachings flagged
4. YAML files parseable

## Handoff
