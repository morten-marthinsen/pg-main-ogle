# Layer 0.1 — Email Draft Loader

## Purpose

Load ALL individual email drafts from E1 (Email Writer) output into working memory. Verify that every email position specified in the blueprint has a corresponding draft. Extract body text, body type, word count, and CTA pattern from each draft.

---

## Input Requirements

```yaml
from_blueprint:
  total_emails: integer
  emails:
    - position: integer
      body_type: string
      function: string
      content_focus: string

from_e1_output:
  email_drafts_directory: "[project]/E1-email-writer/outputs/"
  expected_files:
    - "email-[position]-draft.md"  # or email-[position]-draft.yaml
```

---

## Execution Protocol

### Step 1: Identify Expected Drafts

Read the blueprint's `emails` array to determine how many drafts are expected and at which positions:

```
FOR each email in blueprint.emails:
  expected_drafts.append({
    position: email.position,
    body_type: email.body_type,
    function: email.function,
    expected_file: "email-{position}-draft.md"
  })
```

### Step 2: Load Each Draft

For each expected draft:

1. READ the draft file from E1 output directory
2. EXTRACT the following fields:
   - `body_text` — the complete email body (opening through sign-off)
   - `body_type` — the structural type (CT, QO, TM, QA, LB, ST, NR)
   - `word_count` — total word count of body text
   - `cta_pattern` — CTA pattern used (A-E)
   - `content_pitch_ratio` — content vs. pitch percentage
   - `arena_selected` — whether this was an Arena-selected draft
3. VERIFY body_type matches blueprint assignment for this position

### Step 3: Track Loading Status

For each position, record:
- File found: Y/N
- File loaded successfully: Y/N
- Body type matches blueprint: Y/N
- Word count within target range: Y/N

### Step 4: Build Loaded Drafts Index

Merge all loaded drafts into a single `drafts_index` object, ordered by position.

### Step 5: Validate Completeness

```
IF drafts_loaded_count < blueprint.total_emails:
  missing_positions = [positions not loaded]
  loading_complete = false
  → HALT at Gate 0
ELSE:
  loading_complete = true
```

## Output

```yaml
drafts_index:
  total_expected: [integer]
  total_loaded: [integer]
  loading_complete: [true|false]
  missing_positions: [integer]  # empty if complete

  drafts:
    - position: 1
      file_path: "[path to draft file]"
      body_type: "[CT|QO|TM|QA|LB|ST|NR]"
      body_type_matches_blueprint: [true|false]
      function: "[DR|DU|PL|AL|BP|CP|AR]"
      body_text: "[complete email body text]"
      word_count: [integer]
      cta_pattern: "[A|B|C|D|E]"
      content_pitch_ratio: "[e.g., 75/25]"
      arena_selected: [true|false]
      load_status: "SUCCESS"

    - position: 2
      # ... same structure

  body_type_verification:
    all_match_blueprint: [true|false]
    mismatches: []  # list of {position, expected, actual}

  warnings: []  # any non-blocking issues
```

---

## Failure Modes

| Failure | Remediation |
|---------|-------------|
| Draft file not found for a position | Log missing position. Set loading_complete = false. Gate 0 will FAIL. |
| Draft file found but empty/malformed | Log parse error. Mark position as not loaded. Gate 0 will FAIL. |
| Body type in draft doesn't match blueprint | Log mismatch as warning. Load the draft anyway (E4 can address). |
| Word count outside target range | Log as warning. Not a loading failure — E4 handles quality. |
| Draft file has no body_text field | Mark as malformed. Position not loaded. Gate 0 will FAIL. |
