# Layer 0.3 — Blueprint Loader

## Purpose

Load the campaign blueprint from E0 (Email Strategist) output. Extract the authoritative sequence order, body type assignments, emotional arc, urgency plan, campaign type, and timing parameters. The blueprint is the SINGLE SOURCE OF TRUTH for how the sequence should be assembled.

---

## Input Requirements

```yaml
from_e0_output:
  blueprint_path: "[project]/E0-email-strategist/campaign-blueprint.yaml"
```

---

## Execution Protocol

### Step 1: Load Blueprint File

READ `campaign-blueprint.yaml` from E0 output directory. If file does not exist, HALT immediately — assembly cannot proceed without a blueprint.

### Step 2: Extract Campaign Metadata

```yaml
extract:
  campaign_name: string
  campaign_type: string  # PL, AL, AR, DR, CP, BP, DU
  total_emails: integer
  duration_days: integer
  target_audience: string
  primary_goal: string
```

### Step 3: Extract Per-Email Plan

For each email in the blueprint:

```yaml
extract_per_email:
  - position: integer       # AUTHORITATIVE sequence position
    day: integer             # which day to send
    body_type: string        # CT, QO, TM, QA, LB, ST, NR
    function: string         # DR, DU, PL, AL, BP, CP, AR
    content_focus: string    # what this email is about
    content_source: string   # which upstream asset provides content
    urgency_level: integer   # 0-5
    target_word_count: integer
    emotional_target: string # trust, curiosity, desire, urgency, etc.
    cta_pattern: string      # A, B, C, D, E
    notes: string            # special instructions
```

### Step 4: Extract Emotional Arc

```yaml
extract_emotional_arc:
  summary: string
  phases:
    - phase: string          # e.g., "trust_building"
      emails: [integer]      # which positions
      emotional_range: string
```

### Step 5: Extract Urgency Plan

```yaml
extract_urgency_plan:
  mechanism: string          # production_deadline, sale_expiration, etc.
  escalation:
    - day: integer
      level: integer
      language: string
```

### Step 6: Extract Variety Verification

```yaml
extract_variety:
  no_consecutive_same_type: boolean
  min_types_per_5_emails: integer
  nr_count: integer
```

### Step 7: Validate Blueprint Completeness

Check that all required fields are present and populated:

```
required_fields = [
  campaign_type, total_emails, duration_days,
  emails (array with position + body_type + function for each),
  emotional_arc (summary + phases),
  urgency_plan (mechanism + escalation)
]

FOR each field in required_fields:
  IF field is missing or empty:
    missing_fields.append(field)

IF missing_fields is not empty:
  blueprint_complete = false
  → HALT at Gate 0
ELSE:
  blueprint_complete = true
```

## Failure Modes

| Failure | Remediation |
|---------|-------------|
| Blueprint file not found | HALT — Cannot assemble without blueprint. Request E0 execution. |
| Blueprint file malformed (YAML parse error) | HALT — Log parse error. Request E0 re-generation. |
| Missing required fields | HALT — Log missing fields. Request E0 to complete blueprint. |
| Email count in blueprint is 0 | HALT — Blueprint has no emails to assemble. |
| Email positions have gaps (e.g., 1, 2, 4 — missing 3) | HALT — Blueprint sequence must be continuous. Flag to human. |
