# Layer 3.2 — Output Packager

## Purpose

Package the fully assembled, validated email sequence into two final deliverables: `assembled-sequence.yaml` (structured data for E4 editorial) and `SEQUENCE-SUMMARY.md` (human-readable overview). This is the final microskill in E3 — when it completes, the sequence is ready for E4.

---

## Input Requirements

```yaml
from_all_layers:
  compiled_sequence:
    emails: [complete email objects with connectors and P.S. attached]

  timing_plan:
    schedule: [day/time assignments]

  variety_check:
    overall_variety_passes: boolean
    type_distribution: object

  escalation_map:
    urgency_progression: [object]
    emotional_progression: [object]
    escalation_summary: object

  connector_results:
    connectors: [object]
    connector_map: [string]

  ps_strategy:
    ps_assignments: [object]
    ps_frequency: string

  sequence_validation:
    criteria: [C1-C5 results]
    overall: object
```

---

## Execution Protocol

### Step 1: Verify Validation Passed

```
IF NOT sequence_validation.overall.all_pass:
  HALT — "Cannot package. {failing_count} criteria failed: {failing_names}"
  RETURN to 3.1 or escalate to human
```

### Step 2: Assemble Final Email Objects

For each email in the compiled sequence, construct the final output object with connectors and P.S. physically integrated:

```
FOR each email in compiled_sequence.emails:
  final_body = ""

  # Prepend connector if present
  IF email.connector_added:
    final_body += email.connector_text + "\n\n"

  # Add original body text (UNMODIFIED)
  final_body += email.body_text

  # Append P.S. if present
  IF email.ps_added:
    final_body += "\n\n" + email.ps_text

  final_email = {
    position: email.position,
    day: email.day,
    send_time: email.send_time,
    subject_line: email.subject_line,
    subject_line_formula: email.subject_line_formula,
    body_type: email.body_type,
    function: email.function,
    body_text: final_body,
    original_body_text: email.body_text,  # preserved for E4 reference
    connector_added: email.connector_added,
    connector_text: email.connector_text,
    connector_references_email: email.connector_references_email,
    ps_added: email.ps_added,
    ps_type: email.ps_type,
    ps_text: email.ps_text,
    urgency_level: email.urgency_level,
    emotional_target: email.emotional_target,
    word_count: len(final_body.split()),
    notes: email.notes
  }
```

## Failure Modes

| Failure | Remediation |
|---------|-------------|
| Validation did not pass (3.1 failed) | HALT — Cannot package. Return to address failing criteria. |
| Output file write fails | Retry write. Check directory exists. |
| YAML schema validation fails | Fix schema issues. Re-validate against expected structure. |
| SEQUENCE-SUMMARY.md missing required sections | Add missing sections. All sections listed in Step 4 are mandatory. |
| assembled-sequence.yaml missing email objects | Recompile from Step 2. All emails must be present. |
| Total word count is 0 or suspiciously low | HALT — Emails may have empty body text. Recheck loading. |
