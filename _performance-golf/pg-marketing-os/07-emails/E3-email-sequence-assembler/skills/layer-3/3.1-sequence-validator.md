# Layer 3.1 — Sequence Validator

## Purpose

Validate the complete assembled sequence against the 5 campaign-level quality criteria (C1-C5) from email-quality-rubric.md. This is the final quality gate before output packaging. Every criterion must PASS for the sequence to proceed to E4 editorial.

---

## Execution Protocol

### Step 1: Validate C1 — Body Type Variety

Pull directly from Layer 1 variety check:

```
c1_result = {
  criterion: "C1 — Body Type Variety",
  rule: "No same type twice in a row. 3+ types per 5-email window.",
  consecutive_check: variety_check.rule_1_consecutive.passes,
  window_check: variety_check.rule_2_window_diversity.passes,
  nr_check: variety_check.rule_3_nr_frequency.passes,
  passes: variety_check.overall_variety_passes,
  notes: variety_check.violation_summary (if any)
}
```

### Step 2: Validate C2 — Emotional Arc

Analyze the emotional progression across the sequence:

```
emotional_targets = [email.emotional_target for email in compiled_sequence.emails]

# Check for progression (not flat)
unique_targets = set(emotional_targets)
has_progression = len(unique_targets) >= 3  # At least 3 different emotional targets

# Check for arc shape (early = trust/curiosity, late = urgency/action)
first_half = emotional_targets[:len(emotional_targets)//2]
second_half = emotional_targets[len(emotional_targets)//2:]

trust_emotions = {"trust", "credibility", "relatability", "empathy", "curiosity"}
action_emotions = {"urgency", "desire", "consequence", "deadline", "fear_of_missing"}

early_trust_heavy = sum(1 for e in first_half if e in trust_emotions) / len(first_half) > 0.5
late_action_heavy = sum(1 for e in second_half if e in action_emotions) / len(second_half) > 0.3

# For non-launch sequences, action emotions may be absent — that's OK
IF campaign_type IN [AR, DR, CP]:
  late_action_heavy = true  # Not required for non-launch

c2_result = {
  criterion: "C2 — Emotional Arc",
  has_progression: has_progression,
  early_trust_weighted: early_trust_heavy,
  late_action_weighted: late_action_heavy,
  passes: has_progression AND early_trust_heavy,
  notes: escalation_map.emotional_arc_validation details
}
```

### Step 3: Validate C3 — Cross-Email Continuity

Check that later emails reference earlier content:

```
connectors_present = connector_results.total_connectors_written
minimum_required = calculate_connector_minimum(total_emails)
  # 5+ emails: 2 minimum
  # 10+ emails: 4 minimum
  # 1-4 emails: 0 minimum

# Also check for organic continuity beyond explicit connectors
# (e.g., shared terminology, narrative threads, proof callbacks)
organic_continuity_signals = 0
FOR i in range(1, len(emails)):
  IF emails[i].content_focus references emails[i-1].content_focus:
    organic_continuity_signals += 1

total_continuity = connectors_present + organic_continuity_signals

c3_result = {
  criterion: "C3 — Cross-Email Continuity",
  explicit_connectors: connectors_present,
  organic_signals: organic_continuity_signals,
  total_continuity: total_continuity,
  minimum_required: minimum_required,
  passes: connectors_present >= minimum_required,
  notes: "Connectors reference actual content: {connector_accuracy}"
}
```

### Step 4: Validate C4 — Urgency Escalation

For launch/affiliate sequences, verify urgency escalates:

```
IF campaign_type IN [PL, AL]:
  urgency_levels = [email.urgency_level for email in compiled_sequence.emails]
  max_urgency = max(urgency_levels)
  final_email_urgency = urgency_levels[-1]

  # Urgency should peak in final 1/3 of sequence
  final_third_start = len(urgency_levels) * 2 // 3
  final_third_urgency = urgency_levels[final_third_start:]
  final_third_avg = sum(final_third_urgency) / len(final_third_urgency)

  first_third_urgency = urgency_levels[:len(urgency_levels)//3]
  first_third_avg = sum(first_third_urgency) / len(first_third_urgency)

  escalation_present = final_third_avg > first_third_avg
  peak_at_end = max_urgency == final_email_urgency or max_urgency == urgency_levels[-2]

  c4_passes = escalation_present AND peak_at_end

ELSE:
  # Non-launch sequences: C4 passes by default (urgency escalation N/A)
  c4_passes = true

c4_result = {
  criterion: "C4 — Urgency Escalation",
  applicable: campaign_type IN [PL, AL],
  escalation_present: escalation_present (if applicable),
  peak_at_end: peak_at_end (if applicable),
  passes: c4_passes,
  notes: escalation_map.urgency_validation details
}
```

### Step 5: Validate C5 — Subject Line Diversity

Check that subject lines use multiple formula categories:

```
sl_categories = [email.subject_line_formula for email in compiled_sequence.emails]
unique_categories = set(sl_categories)
sl_diversity = len(unique_categories)

c5_result = {
  criterion: "C5 — Subject Line Diversity",
  unique_categories: sl_diversity,
  categories_used: list(unique_categories),
  minimum_required: 3,
  passes: sl_diversity >= 3,
  notes: "Categories used: {', '.join(unique_categories)}"
}

# Exception: single-email campaigns (BP, DU) pass by default
IF total_emails <= 2:
  c5_result.passes = true
  c5_result.notes = "Single/dual email campaign — diversity N/A"
```

### Step 6: Generate Validation Summary

```
all_pass = c1_result.passes AND c2_result.passes AND c3_result.passes AND c4_result.passes AND c5_result.passes

IF NOT all_pass:
  failing_criteria = [c for c in [c1, c2, c3, c4, c5] if not c.passes]
  action_required = "HALT — {len(failing_criteria)} criteria failed. Address before packaging."
ELSE:
  action_required = "PROCEED to 3.2-output-packager.md"
```

## Failure Modes

| Failure | Remediation |
|---------|-------------|
| C1 fails (variety violation) | HALT — This is a blueprint error. Report to human. |
| C2 fails (flat emotional arc) | Flag for human review. May be intentional for DR/CP campaigns. |
| C3 fails (insufficient continuity) | Return to 2.2 — write additional connectors. |
| C4 fails (no urgency escalation in launch) | HALT — Blueprint urgency plan may be wrong. Report to human. |
| C5 fails (subject line categories too narrow) | Flag for E2 revision — more diverse SL formulas needed. |
| Multiple criteria fail | HALT — Sequence has structural problems. Report all failures to human. |
