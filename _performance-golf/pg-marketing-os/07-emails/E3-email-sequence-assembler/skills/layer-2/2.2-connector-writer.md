# Layer 2.2 — Connector Writer

## Purpose

Write cross-email callbacks and continuity references where they strengthen the sequence. Connectors are brief (1-2 sentence) references placed at the opening of an email that reference specific content from a previous email, creating cohesion across the campaign.

**Critical Rule:** Connectors are the ONLY new text E3 writes. They are added to the OPENING of emails (first 1-3 lines) and must reference ACTUAL content from the cited email. No body text modification allowed.

---

## Execution Protocol

### Step 1: Identify Connector Opportunities

Scan the compiled sequence for positions where a connector would strengthen continuity:

```
connector_opportunities = []

# Type 1: Inflection points (phase transitions)
FOR each inflection in escalation_map.inflection_points:
  connector_opportunities.append({
    target_position: inflection.between_positions[1],  # receiving email
    references_position: inflection.between_positions[0],  # cited email
    type: "phase_transition",
    reason: "Transition from {from_phase} to {to_phase}"
  })

# Type 2: Story continuations (same content thread across emails)
FOR i in range(len(emails) - 1):
  IF emails[i].content_focus overlaps emails[i+1].content_focus:
    connector_opportunities.append({
      target_position: i + 2,
      references_position: i + 1,
      type: "story_continuation",
      reason: "Shared content focus: {content_focus}"
    })

# Type 3: Urgency escalations
FOR i in range(len(emails) - 1):
  IF emails[i+1].urgency_level > emails[i].urgency_level + 1:
    connector_opportunities.append({
      target_position: i + 2,
      references_position: i + 1,
      type: "urgency_escalation",
      reason: "Urgency jump from {current} to {next}"
    })

# Type 4: Proof stacking (testimonial/data mentioned earlier, more added now)
FOR i in range(1, len(emails)):
  IF emails[i].body_type IN [TM, LB] AND any previous email.body_type IN [TM, LB]:
    connector_opportunities.append({
      target_position: i + 1,
      references_position: find_previous_proof_email(i),
      type: "proof_stacking",
      reason: "Additional proof building on earlier evidence"
    })
```

### Step 2: Filter and Prioritize

Not every opportunity needs a connector. Apply these filters:

```
EXCLUDE:
  - First email in sequence (position 1 never gets a connector)
  - Emails explicitly marked as "standalone" in blueprint notes
  - Daily relationship (DR) emails that are unrelated to each other
  - More than 1 connector in any 3-email window (avoid over-connecting)

PRIORITIZE:
  1. Phase transitions (highest value — smooths emotional shifts)
  2. Urgency escalations (acknowledges building deadline)
  3. Story continuations (narrative cohesion)
  4. Proof stacking (evidence building)

MINIMUM:
  - Sequences of 5+ emails: at least 2 connectors
  - Sequences of 10+ emails: at least 4 connectors
  - Sequences of 1-4 emails: 0-1 connectors (optional)
```

### Step 3: Write Connectors

For each selected connector opportunity, write the actual connector text:

```
FOR each selected_opportunity:
  # Extract specific content from the referenced email
  referenced_email = compiled_sequence.emails[references_position]
  specific_detail = extract_memorable_detail(referenced_email.body_text)

  # Write connector based on type
  connector_text = write_connector(
    type: opportunity.type,
    specific_detail: specific_detail,
    target_body_type: target_email.body_type,
    tone: match_target_email_tone
  )
```

**Connector Templates by Type:**

```
phase_transition:
  "[Yesterday/Last time] I [shared/told you about] [specific detail].
   Today I want to [show/explain/reveal] something [different/deeper/more urgent]..."

story_continuation:
  "Remember [character/situation/finding] from [yesterday's/my last] email?
   [One sentence setup for what comes next]..."

urgency_escalation:
  "I mentioned [deadline/offer detail] [timeframe ago].
   [Update on how situation has changed]..."

proof_stacking:
  "[Name]'s story [from yesterday] was powerful.
   But [he/she] isn't alone — [transition to new proof]..."
```

### Step 4: Insert Connectors

For each written connector:

```
target_email = compiled_sequence.emails[target_position]
target_email.connector_added = true
target_email.connector_text = connector_text
target_email.connector_references_email = references_position

# The connector is placed BEFORE the existing body text
# Final assembly in output-packager will prepend it
```

### Step 5: Validate Connector Quality

For each connector written:

```
VERIFY:
  - Contains specific detail from referenced email (not vague)
  - Is 1-2 sentences maximum (not a recap)
  - Tone matches the target email's body type
  - Does not spoil/repeat the target email's main content
  - References are accurate (the cited content actually exists)
```

## Failure Modes

| Failure | Remediation |
|---------|-------------|
| Connector references content that doesn't exist in cited email | Rewrite connector with actual content from the cited email. |
| Connector exceeds 2 sentences | Trim to 1-2 sentences. Connectors are bridges, not recaps. |
| Too many connectors (over-connecting) | Remove lowest-priority connectors. Max 1 per 3-email window. |
| No connectors written for 5+ email sequence | Review opportunities again. At least 2 connectors should be added. |
| Connector tone clashes with target email | Rewrite to match target email's body type tone. |
