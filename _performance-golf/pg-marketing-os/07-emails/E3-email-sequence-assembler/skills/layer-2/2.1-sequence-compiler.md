# Layer 2.1 — Sequence Compiler

## Purpose

Compile all individual email drafts into a single ordered sequence with subject lines attached. This is the core assembly step — taking discrete E1 drafts and E2 subject lines and joining them into a cohesive campaign sequence in the exact order specified by the E0 blueprint.

**Critical Rule:** This microskill COMPILES. It does NOT modify email body text, subject lines, or sequence order. Assembly means joining existing pieces, not editing them.

---

## Input Requirements

```yaml
from_layer_0:
  drafts_index:
    drafts:
      - position: integer
        body_type: string

  sl_index:

      - position: integer

    email_plan:
      - position: integer
```

## Execution Protocol

### Step 1: Verify Gate 1 Passed

Before compiling, confirm all Layer 1 checks passed:

```
IF NOT variety_check.overall_variety_passes:
  HALT — "Variety check failed. Cannot compile until resolved."
IF NOT escalation_map.escalation_validated:
  LOG WARNING — "Escalation validation has warnings. Proceeding with compilation but flagging."
```

### Step 2: Build Compiled Sequence

For each position in blueprint order (position 1, 2, 3, ...):

```
compiled_sequence = []
FOR position in range(1, total_emails + 1):
  draft = drafts_index.drafts[position]
  sl = sl_index.subject_lines[position]
  blueprint_entry = blueprint_context.email_plan[position]
  timing_entry = find_timing(position, timing_plan)

  compiled_email = {
    position: position,
    day: blueprint_entry.day,
    send_time: timing_entry.send_time,
    send_order: timing_entry.send_order,

    # Subject Line (from E2)
    subject_line: sl.selected_subject_line,
    subject_line_formula: sl.formula_category,

    # Email Body (from E1 — UNMODIFIED)
    body_type: draft.body_type,
    function: blueprint_entry.function,
    body_text: draft.body_text,

    # Metadata
    content_focus: blueprint_entry.content_focus,
    urgency_level: blueprint_entry.urgency_level,
    emotional_target: blueprint_entry.emotional_target,
    cta_pattern: draft.cta_pattern,
    word_count: draft.word_count,

    # Assembly slots (populated by 2.2 and 2.3)
    connector_added: false,
    connector_text: null,
    connector_references_email: null,
    ps_added: false,
    ps_type: null,
    ps_text: null,

    # Notes
    notes: blueprint_entry.notes or ""
  }

  compiled_sequence.append(compiled_email)
```

### Step 3: Verify Completeness

```
IF len(compiled_sequence) != blueprint_context.total_emails:
  HALT — "Compiled {len(compiled_sequence)} emails but blueprint expects {total_emails}."

FOR each email in compiled_sequence:
  IF email.subject_line is null or empty:
    HALT — "Email at position {position} has no subject line."
  IF email.body_text is null or empty:
    HALT — "Email at position {position} has no body text."
```

### Step 4: Generate Compilation Summary

```
compilation_summary = {
  total_compiled: len(compiled_sequence),
  total_word_count: sum(email.word_count for email in compiled_sequence),
  avg_word_count: mean(email.word_count for email in compiled_sequence),
  body_types_used: unique(email.body_type for email in compiled_sequence),
  sl_formulas_used: unique(email.subject_line_formula for email in compiled_sequence),
  days_covered: max(email.day for email in compiled_sequence),
  compilation_status: "COMPLETE"
}
```

## Failure Modes

| Failure | Remediation |
|---------|-------------|
| Email count mismatch after compilation | HALT — Verify all positions are present. Recount. |
| Missing subject line for any position | HALT — Cannot compile without SL. Request E2 generation. |
| Missing body text for any position | HALT — Cannot compile without draft. Request E1 generation. |
| Duplicate positions in compiled sequence | HALT — Each position must appear exactly once. De-duplicate and verify. |
| Timing data missing for a position | Use blueprint day assignment. Log that timing was not available. |
