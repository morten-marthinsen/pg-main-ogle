# Layer 1.2 — Variety Checker

## Purpose

Verify that the assembled email sequence meets body type variety rules. This microskill does NOT fix violations — it detects and reports them. E0 already assigned body types; if a violation exists here, it means the blueprint had an error that must be escalated, not silently repaired.

**Critical Rule:** E3 VERIFIES variety. It does NOT ENFORCE variety. Variety enforcement happened at E0 design time. If violations are found here, HALT and report.

---

## Input Requirements

```yaml
from_layer_0:
  drafts_index:
    drafts:
      - position: integer
        body_type: string
  blueprint_context:
    email_plan:
      - position: integer
        body_type: string
    variety_rules:
      no_consecutive_same_type: boolean
      min_types_per_5_emails: integer
      nr_count: integer

from_layer_1:
  timing_plan:
    schedule:
      - day: integer
        email_assignments:
          - position: integer
```

## Execution Protocol

### Step 1: Build Body Type Sequence

Create an ordered list of body types matching the assembled sequence:

```
body_type_sequence = []
FOR i in range(1, total_emails + 1):
  body_type_sequence.append({
    position: i,
    body_type: drafts_index.drafts[i].body_type,
    blueprint_type: blueprint.email_plan[i].body_type,
    match: drafts_index.drafts[i].body_type == blueprint.email_plan[i].body_type
  })
```

### Step 2: Check Consecutive Same Type (Rule 1)

```
consecutive_violations = []
FOR i in range(len(body_type_sequence) - 1):
  current = body_type_sequence[i].body_type
  next = body_type_sequence[i + 1].body_type
  IF current == next:
    consecutive_violations.append({
      position_a: i + 1,
      position_b: i + 2,
      repeated_type: current
    })

consecutive_check_passes = len(consecutive_violations) == 0
```

### Step 3: Check 5-Email Window Diversity (Rule 2)

```
window_violations = []
FOR i in range(len(body_type_sequence) - 4):
  window = body_type_sequence[i:i+5]
  unique_types = set(bt.body_type for bt in window)
  IF len(unique_types) < 3:
    window_violations.append({
      window_start: i + 1,
      window_end: i + 5,
      unique_types_count: len(unique_types),
      types_in_window: list(unique_types)
    })

window_check_passes = len(window_violations) == 0
```

### Step 4: Check NR (Negative Response) Frequency (Rule 3)

```
nr_emails = [bt for bt in body_type_sequence if bt.body_type == "NR"]
nr_count = len(nr_emails)
max_nr_allowed = total_emails // 7  # 1 per 7 emails
nr_check_passes = nr_count <= max(1, max_nr_allowed)

IF NOT nr_check_passes:
  nr_violation = {
    nr_count: nr_count,
    max_allowed: max(1, max_nr_allowed),
    nr_positions: [email.position for email in nr_emails]
  }
```

### Step 5: Check Blueprint-Draft Body Type Alignment

Verify that E1 actually wrote what E0 assigned:

```
alignment_mismatches = []
FOR each entry in body_type_sequence:
  IF NOT entry.match:
    alignment_mismatches.append({
      position: entry.position,
      blueprint_assigned: entry.blueprint_type,
      draft_actual: entry.body_type
    })

alignment_passes = len(alignment_mismatches) == 0
```

### Step 6: Generate Body Type Distribution Summary

```
type_distribution = {}
FOR each entry in body_type_sequence:
  type_distribution[entry.body_type] = type_distribution.get(entry.body_type, 0) + 1
```

---

## Failure Modes

| Failure | Remediation |
|---------|-------------|
| Consecutive same type detected | HALT — Report violation to human with positions. Do NOT swap types. |
| Window diversity below 3 | HALT — Report window violation to human. Do NOT reassign. |
| NR frequency exceeded | HALT — Report NR overuse to human. Do NOT remove NR emails. |
| Blueprint-draft body type mismatch | Log as warning. E1 may have had a valid reason. Flag for E4. |
| Only 1-2 body types used across entire sequence | HALT — Serious variety failure. Report to human. |
