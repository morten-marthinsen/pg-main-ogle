# 3.2-output-packager.md

> **Layer:** 3 (Validation & Output)
> **Skill:** E1-email-writer
> **Purpose:** Package email draft for E2 (subject line context) and E3 (assembly). Output includes full email text, metadata, quality scores.
> **Inputs:** Human-selected email, `validation_report` from 3.1, `email_context` from 0.1, Arena results
> **Outputs:** `email-draft-package.yaml` (structured output), `EMAIL-DRAFT-SUMMARY.md` (human-readable), execution-log.md

---

## FUNCTION

Package the validated email into the three mandatory output files required by the CopywritingEngine output protocol. The primary output (`email-draft-package.yaml`) is consumed by E2 (subject line generation) and E3 (sequence assembly). The summary is for human review. The execution log is for audit trail.

**Key Principle:** The output package must contain EVERYTHING downstream skills need without requiring them to read upstream files. E2 needs the email content to generate subject lines. E3 needs the email content plus metadata to assemble the sequence. Both need quality scores to make decisions.

---

## THREE MANDATORY OUTPUTS

### Output 1: email-draft-package.yaml (PRIMARY -- Structured Data)

This is what E2 and E3 consume.

### Output 2: EMAIL-DRAFT-SUMMARY.md (Human-Readable Review)

This is for the human reviewer to review the email draft with context.

### Output 3: execution-log.md (Audit Trail)

This shows every microskill that executed, in order, with status.

---

## EXECUTION PROTOCOL

```
EXECUTE 3.2-output-packager:

  STEP 1: Verify Gate 3 Passed
    1. CHECK validation_report.result == "PASS"
    2. CHECK validation_report.gate_3_status == "PASS"
    3. IF either is not PASS:
       HALT -- "Cannot package email that did not pass validation."

  STEP 2: Assemble Primary Output (email-draft-package.yaml)
    1. COMPILE all structured data (see output structure below)
    2. INCLUDE full email text
    3. INCLUDE all metadata
    4. INCLUDE quality scores
    5. INCLUDE downstream handoff fields for E2 and E3
    6. VERIFY all required fields are populated (not null, not placeholder)

  STEP 3: Assemble Summary (EMAIL-DRAFT-SUMMARY.md)
    1. WRITE human-readable document with:
       - Email position and body type
       - Full email text
       - Quality score summary
       - Arena selection notes
       - Key decisions made during generation
       - Any flags or notes for E2/E3

  STEP 4: Assemble Execution Log (execution-log.md)
    1. LIST every microskill executed:
       - 0.1 through 3.2
       - For each: spec file read, output file created, status
    2. LIST Arena execution summary:
       - Rounds completed, competitors, human selection
    3. LIST all checkpoint files created

  STEP 5: Write Output Files
    1. WRITE email-draft-package.yaml to project outputs
    2. WRITE EMAIL-DRAFT-SUMMARY.md to project outputs
    3. WRITE execution-log.md to project outputs (or update existing)

  STEP 6: Update Project State
    1. UPDATE PROJECT-STATE.md with email completion
    2. UPDATE PROGRESS-LOG.md with completion entry
    3. CREATE LAYER_3_COMPLETE.yaml checkpoint

  STEP 7: Verify All Outputs
    [ ] email-draft-package.yaml EXISTS and all fields populated
    [ ] EMAIL-DRAFT-SUMMARY.md EXISTS with all sections
    [ ] execution-log.md EXISTS showing all microskills
    [ ] LAYER_3_COMPLETE.yaml EXISTS

```

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Gate 3 not passed | HALT -- Cannot package without validation pass |
| CRITICAL | Required fields empty in package | HALT -- Populate all fields |
| CRITICAL | Missing output file | HALT -- Write all 3 outputs |
| WARNING | Downstream handoff fields sparse | Fill with available information |

---

## ANTI-PATTERNS (FORBIDDEN)

1. **Packaging without validation pass** -- Gate 3 MUST be PASS before packaging.
2. **Empty handoff fields** -- E2 and E3 depend on these. Every field must have real data.
3. **Missing summary file** -- All 3 outputs are mandatory per ~system/SYSTEM-CORE.md protocol.
4. **Missing execution log** -- Audit trail is required for every skill execution.
5. **Placeholder values** -- "[TBD]" or "[fill in later]" are not acceptable in output package.
6. **Abbreviating email text** -- Full email text must appear in BOTH the YAML and the summary.
