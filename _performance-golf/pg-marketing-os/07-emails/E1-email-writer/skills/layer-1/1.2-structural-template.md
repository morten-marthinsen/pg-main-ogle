# 1.2-structural-template.md

> **Layer:** 1 (Type-Specific Architecture)
> **Skill:** E1-email-writer
> **Purpose:** Load the skeleton/template for the selected body type (opening, body, bridge, CTA structure)
> **Inputs:** `routing_decision` from 1.1, `structural_specimens` from 0.2
> **Outputs:** `structural_template` with section-by-section blueprint including word count allocations, specimen-derived patterns, and generation guides

---

## FUNCTION

Transform the routing decision into a concrete structural template that Layer 2 generators will follow section by section. This template defines the EXACT skeleton of the email: how many words per section, what structural pattern each section follows, and what specimens demonstrate the pattern. The template is derived from both the routing table (1.1) and the loaded specimens (0.2).

**Key Principle:** The structural template is a BLUEPRINT, not a cage. Layer 2 generators follow the template's section structure and word count allocations, but they bring voice, angle, and creativity within those constraints. Think of it as the architectural blueprint for a house -- the rooms must be there, but the interior design is creative.

---

## TEMPLATE CONSTRUCTION PROTOCOL

```
EXECUTE 1.2-structural-template:

  STEP 1: Load Routing Decision
    1. READ routing_decision from 1.1
    2. EXTRACT body_type, structural_requirements, word_count range, tone

  STEP 2: Analyze Specimens for Patterns
    FOR each loaded specimen from 0.2:
      1. MEASURE section word counts (opening, body, bridge, CTA)
      2. IDENTIFY paragraph patterns (average length, rhythm)
      3. EXTRACT bridge techniques used
      4. MAP content-to-pitch ratio
      5. NOTE sign-off pattern

    CALCULATE specimen averages:
      - avg_opening_words: [number]
      - avg_body_words: [number]
      - avg_bridge_words: [number]
      - avg_cta_words: [number]
      - avg_content_ratio: [percent]
      - avg_paragraph_length: [sentences]

  STEP 3: Build Section Allocations
    BASED ON body_type word_count_target and specimen analysis:

    opening_allocation:
      word_count: "[10-15% of total]"
      lines: "[1-3]"
      pattern: "[derived from specimens and routing table]"

    body_allocation:
      word_count: "[55-70% of total]"
      sections: "[from routing table section count]"
      pattern: "[derived from specimens and routing table]"

    bridge_allocation:
      word_count: "[10-15% of total]"
      sentences: "[2-4]"
      pattern: "[derived from specimens and routing table]"

    cta_allocation:
      word_count: "[5-10% of total]"
      sentences: "[1-3]"
      pattern: "[from blueprint CTA pattern assignment]"

  STEP 4: Create Generation Guides Per Section
    FOR each section (opening, body, bridge, CTA):
      1. WRITE specific generation instructions
      2. CITE specimen examples showing the pattern
      3. LIST constraints (word count, structure, forbidden patterns)
      4. DEFINE success criteria for this section

  STEP 5: Assemble structural_template
  STEP 6: Return structural_template
```

## BODY-TYPE-SPECIFIC TEMPLATE PATTERNS

### CT (Contrarian) Template

```yaml
template:
  opening:
    word_allocation: "20-40 words"
    pattern: "Bold provocative claim that most readers would disagree with"
    specimen_guide: "Note how specimens lead with the contrarian statement directly -- no preamble"
    constraints:
      - Max 3 lines
      - Must be a specific claim, not a vague provocation
      - No questions (CT opens with statements)

  body:
    word_allocation: "200-320 words"
    pattern: "Stack-and-dismiss (present conventional wisdom, then demolish)"
    structure:
      - section_1: "Present what most people believe (2-3 sentences)"
      - section_2: "First dismissal with evidence/logic (3-5 sentences)"
      - section_3: "Second dismissal (optional, 3-5 sentences)"
      - section_4: "Build contrarian case -- what's actually true (3-5 sentences)"
    constraints:
      - Each dismissal must have evidence, not just opinion
      - Build from weakest to strongest argument
      - Paragraphs max 3 sentences

  bridge:
    word_allocation: "30-50 words"
    pattern: "Contrarian insight naturally leads to product relevance"
    specimen_guide: "Note how specimens pivot from 'so what does this mean for you' to product"
    constraints:
      - 2-4 sentences maximum
      - Must feel like natural conclusion, not sales pitch
      - Load bridge phrases from transition-phrase-library.md

  cta:
    word_allocation: "15-30 words"
    pattern: "[Assigned CTA pattern from blueprint]"
    constraints:
      - Max 3 sentences (transition phrase + URL + sign-off)
      - Sign-off: name only
```

### QO (Quote-Opener) Template

```yaml
template:
  opening:
    word_allocation: "15-35 words"
    pattern: "Formatted quote block with attribution"
    format: |
      "[Quote text]"
      -- [Attribution], [Context/Title]
    constraints:
      - Quote visually separated from body
      - Attribution required
      - Quote must be genuine or reasonably attributed

  body:
    word_allocation: "230-370 words"
    pattern: "Riff on the quote -- analyze, apply, extract lesson"
    structure:
      - section_1: "Why this quote matters (2-4 sentences)"
      - section_2: "How it applies to the reader's situation (3-6 sentences)"
      - section_3: "Lesson extraction -- the specific takeaway (2-4 sentences)"
    constraints:
      - Must go DEEPER than the quote, not just restate it
      - Apply to reader's specific situation (niche-appropriate)
      - End body with clear extracted lesson

  bridge:
    word_allocation: "30-50 words"
    pattern: "Lesson connects to product relevance"
    constraints:
      - 2-4 sentences
      - "This lesson is exactly why..." type pivot

  cta:
    word_allocation: "15-30 words"
    pattern: "[Assigned CTA pattern]"
```

### TM (Testimonial) Template

```yaml
template:
  opening:
    word_allocation: "10-25 words"
    pattern: "Attribution line"
    format: "[Name], [Location/Context]:"
    constraints:
      - Short and direct
      - Sets up who is speaking

  body:
    word_allocation: "150-280 words"
    pattern: "Testimonial block"
    structure:
      - section_1: "Verbatim testimonial (specific result or transformation)"
      - section_2: "Before/after contrast (if available)"
    constraints:
      - Keep as close to verbatim as possible
      - Let the testimonial speak -- minimal editorial

  bridge:
    word_allocation: "15-30 words"
    pattern: "Brief commentary"
    constraints:
      - 1-2 sentences maximum
      - Reaction, not selling ("Pretty amazing, right?")

  cta:
    word_allocation: "15-25 words"
    pattern: "Direct, shortest CTA"
    constraints:
      - TM emails have shortest CTA -- testimonial IS the pitch
```
