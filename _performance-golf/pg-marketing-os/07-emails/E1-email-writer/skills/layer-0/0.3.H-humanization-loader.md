## PART 1: PATTERN PRIMING (~100 tokens)

Before generating ANY prose output for this skill, internalize these 5 structural constraints:

```
1. ONE idea per sentence. If a sentence contains a logical pivot (but, however, so, which means)
   AND exceeds 25 words — split it. Each idea breathes in its own sentence.

2. NO tricolon structures. "Not X. Not Y. Z." is banned. Replace with a single declarative
   or a collapsed clause with escalation. Maximum 1 tricolon per 2,000 words — and it must
   be earned, not reflexive.

3. STOP after the punchline. If a line is strong enough to stand alone — let it. Do not add
   explanation, restatement, or summary. The next paragraph must advance the argument, not
   restate the last one.

4. REPLACE vague referents. Every "it," "what," "things," and "that" must be replaced with
   the specific noun it refers to. If the noun doesn't exist yet, create it.

5. NEVER re-teach. If a concept was explained earlier, reference it in a phrase — never
   re-explain it. Trust the audience to have absorbed the argument.
```

## PART 2: ENGINE PACK LOADING

```
HOLD: VERBATIM in active context during all generation layers
PURPOSE: These are real human-written passages. Study their STRUCTURE — sentence length
         variation, emphasis placement, emotional pacing, connective tissue. Your output
         should share these structural qualities WITHOUT copying their content.
```

### How to Use the Specimens

1. **DO** study the sentence length variation — notice how short and long sentences alternate without a pattern
2. **DO** study the emphasis placement — notice where CAPS and italics fall and WHY
3. **DO** study the connective tissue — notice "because," "so," "and yet" linking ideas causally
4. **DO** study the emotional pacing — notice how beats get their own paragraph
5. **DO NOT** copy the specimens' content, metaphors, or vertical-specific language
6. **DO NOT** summarize the specimens — hold them verbatim as structural reference
