# Urgency Mechanism Library

**Version:** 1.0
**Created:** 2026-02-21
**Source:** Structural analysis of 2,684 Ben Settle emails (365 "goes to print" mentions)
**Purpose:** Justified urgency patterns for email campaigns

---

## CORE PRINCIPLE

**Urgency must be JUSTIFIED.** Ben Settle's urgency works because Email Players has a REAL print deadline — the newsletter literally goes to print on a specific date. Naked urgency ("Act now!") without a credible reason is forbidden.

---

## THE 5 URGENCY MECHANISMS

### Mechanism 1: Production Deadline (Physical Product)

The product has a real manufacturing/printing cutoff.

**Specimens:**
```
The [month] issue goes to print [day].
```

```
Goes to print next week.
```

```
Goes to print in [X] days.
```

```
Better hurry though, it goes to print soon.
```

```
The deadline for the [month] issue is [date].
```

**When to use:** Physical newsletter, printed materials, manufactured goods
**Adaptation:** For digital products, replace with "enrollment closes" or "access closes"

---

### Mechanism 2: Sale Expiration (Time-Limited Offer)

The price or availability has a hard end date.

**Specimens:**
```
This sale ends [day] at midnight.
```

```
After [day], the price goes back to [higher price].
```

```
The bonuses disappear after [day].
```

```
[X] hours left.
```

```
14 minutes left to get the historic [product] and pile of bonuses.
```

**When to use:** Product Launch (PL), Affiliate Launch (AL)
**Escalation pattern:** Days → Hours → Minutes as deadline approaches

---

### Mechanism 3: Inventory/Capacity Limit

Genuine scarcity based on supply constraints.

**Specimens:**
```
I'm only accepting [X] new members.
```

```
Once they're gone, they're gone.
```

```
There are only [X] copies left.
```

```
I'm limiting this to [X] people because [reason].
```

**When to use:** When actual capacity or inventory limits exist
**Rule:** MUST be truthful. Fabricated scarcity destroys trust permanently.

---

### Mechanism 4: Bonus Deadline (Value Removal)

Bonuses disappear at a specific time even if the product remains available.

**Specimens:**
```
The bonuses go away at midnight [day].
```

```
You can still get [product] after [day], but you won't get [bonus].
```

```
The [bonus name] alone is worth the price, and it disappears [day].
```

**When to use:** Launch sequences, especially mid-sequence when bonuses are revealed
**Rule:** Bonus removal must actually happen. Don't bluff.

---

### Mechanism 5: Price Increase (Value Preservation)

Current price is temporary; future price will be higher.

**Specimens:**
```
The price goes up to [amount] on [date].
```

```
This is the lowest price this will ever be offered at.
```

```
After the launch, the price will be [higher amount].
```

**When to use:** Launch pricing, introductory offers
**Rule:** Price MUST actually increase. Evergreen "price increase" is dishonest.

## URGENCY ESCALATION PATTERN (Launch Sequences)

```
DAY -1 (pre-launch):
  - NO urgency. Just announcement/heads-up.
  - "Starting tomorrow I'll be sending more emails about [product]."

DAY 1-2 (early launch):
  - MINIMAL urgency. Focus on content/story.
  - "This is available through [end date]."
  - Urgency Mechanism: None or gentle mention

DAY 3-4 (mid launch):
  - MODERATE urgency. Mention deadline in context.
  - "The sale runs through [day]."
  - Urgency Mechanism: Sale Expiration (soft)

DAY 5 (penultimate):
  - HIGH urgency. Deadline is the frame.
  - "Tomorrow is the last day."
  - Urgency Mechanism: Sale Expiration + Bonus Deadline

FINAL DAY (close day):
  - MAXIMUM urgency. Multiple emails, escalating.
  - Morning: "Last day. [X] hours left."
  - Afternoon: "[X] hours left."
  - Evening: "Last call. [X] minutes."
  - Final: "Closing in [X] minutes." (under 120 words)
  - Urgency Mechanism: ALL applicable mechanisms stacked
```

## "BLATANT PITCH" MONTHLY RITUAL

Ben Settle sends a "blatant pitch" email on the last day of every month, coinciding with the EP print deadline. The subject line is the creative element:

**Subject Line Specimens:**
```
Blatant pitch time
```

```
The blatant sales pitch is strong with this one
```

```
I love it when a blatant sales pitch comes together
```

```
Behold the most frighteningly blatant sales pitch ever
```

```
A shameless in-your-nose blatant sales pitch
```

```
An email pitch so blatant it's dang near obnoxious
```

```
15 blatantly self-serving reasons to do as I say
```

**Pattern:** The meta-awareness IS the entertainment. The reader knows it's a pitch. The writer knows they know. The creative challenge is making the "blatant" framing fresh each month.

## URGENCY BY CAMPAIGN FUNCTION

| Function | Urgency Level | Primary Mechanism |
|----------|--------------|-------------------|
| Daily Relationship (DR) | NONE | — |
| Deadline Urgency (DU) | MODERATE | Production Deadline |
| Product Launch (PL) | ESCALATING | Sale Expiration + Bonus Deadline |
| Affiliate Launch (AL) | ESCALATING | Sale Expiration |
| Blatant Pitch (BP) | PLAYFUL | Production Deadline (meta-framed) |
| Content/Podcast (CP) | NONE | — |
| Autoresponder (AR) | MINIMAL | General availability |

## FORBIDDEN BEHAVIORS

1. ❌ **Naked urgency without mechanism** — "Act now!" without a reason
2. ❌ **Fabricated scarcity** — Claiming limited inventory when there isn't
3. ❌ **Urgency in first email** of any sequence — Build desire first
4. ❌ **Urgency in DR emails** — Daily relationship emails are content-forward
5. ❌ **Evergreen "deadline" language** — Autoresponders must not have time-specific urgency
6. ❌ **Same urgency framing across sequence** — Vary the angle each email
