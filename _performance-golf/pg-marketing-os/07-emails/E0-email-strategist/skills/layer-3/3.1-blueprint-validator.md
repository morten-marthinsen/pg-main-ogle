# Layer 3.1 — Blueprint Validator

## Purpose

Validate the complete campaign blueprint against structural rules before human review. Run six validation checks: variety rules, urgency escalation monotonicity, content-to-pitch ratio, content source coverage, word count realism, and CTA pattern consistency. Output a pass/fail validation report with specific remediation for any failures.

---

## Input Requirements

```yaml
from_layer_2:
  sequence_blueprint:
    emails: [object]  # complete sequence with all fields
    total_emails: integer
    variety_check: object

  body_type_assignments:
    assignments: [object]
    variety_validation: object

  emotional_arc:
    per_email_emotions: [object]
    monotonic_check: object

  confirmed_campaign:
    campaign_type: string

from_layer_1:
  asset_inventory:
    total_usable_assets: integer
    per_type_summary: object
```

## Execution Protocol

### Check 1: Variety Rules

Verify that body type variety rules are satisfied across the entire sequence:

```
CHECK 1A: No Consecutive Same Body Type
  FOR i in range(total_emails - 1):
    IF emails[i].body_type == emails[i+1].body_type:
      → FAIL: "Emails at positions {i+1} and {i+2} both use {body_type}"
      → REMEDIATION: "Swap position {i+2} body type to {alternative_type}"

CHECK 1B: Minimum 3 Types Per 5-Email Window
  FOR window_start in range(total_emails - 4):
    window = emails[window_start : window_start + 5]
    unique_types = set(email.body_type for email in window)
    IF len(unique_types) < 3:
      → FAIL: "Window starting at position {window_start+1} has only
        {len(unique_types)} unique types: {unique_types}"
      → REMEDIATION: "Replace one repeated type with {missing_type}"

CHECK 1C: NR Frequency Limit
  nr_count = count(email.body_type == "NR" for email in emails)
  max_allowed = floor(total_emails / 7)
  IF nr_count > max_allowed:
    → FAIL: "NR count {nr_count} exceeds maximum {max_allowed}
      for {total_emails} emails"
    → REMEDIATION: "Remove {nr_count - max_allowed} NR assignments"
```

### Check 2: Urgency Escalation Monotonicity

Verify that urgency never decreases then re-increases:

```
CHECK 2A: Monotonic Escalation
  FOR i in range(total_emails - 1):
    IF emails[i].urgency_level > emails[i+1].urgency_level
       AND any(emails[j].urgency_level > emails[i+1].urgency_level
               for j in range(i+2, total_emails)):
      → FAIL: "Urgency drops from {emails[i].urgency_level} to
        {emails[i+1].urgency_level} at position {i+2}, then rises again"
      → REMEDIATION: "Raise position {i+2} urgency to at least
        {emails[i].urgency_level}"

CHECK 2B: Final Email Has Maximum Urgency
  IF campaign_type IN [PL, AL] AND emails[-1].urgency_level < 4:
    → WARN: "Final email of launch sequence has urgency {urgency_level}.
      Consider raising to 4-5 for stronger close."

CHECK 2C: No Urgency in Trust Phase
  trust_end = floor(total_emails / 3)
  FOR i in range(trust_end):
    IF emails[i].urgency_level > 0:
      → FAIL: "Email at position {i+1} (trust phase) has urgency
        {urgency_level}. Must be 0."
      → REMEDIATION: "Set urgency_level to 0 for all trust phase emails"
```

### Check 3: Content-to-Pitch Ratio

Verify that content-forward emails dominate the overall sequence:

```
CHECK 3A: Overall Ratio
  content_emails = count(email.function IN [CP, DR, AR] for email in emails)
  pitch_emails = count(email.function IN [BP, DU, PL, AL] for email in emails)
  ratio = content_emails / total_emails

  IF ratio < 0.50:
    → FAIL: "Content ratio is {ratio:.0%}. Minimum 50% of emails
      must be content-forward."
    → REMEDIATION: "Convert {needed} pitch emails to content emails"

CHECK 3B: Per-5-Email-Window Ratio
  FOR window_start in range(total_emails - 4):
    window = emails[window_start : window_start + 5]
    content_in_window = count(email.function IN [CP, DR, AR] for email in window)
    IF content_in_window < 3:
      → WARN: "Window starting at position {window_start+1} has only
        {content_in_window}/5 content emails. Recommend at least 3/5."
```

### Check 4: Content Source Coverage

Verify that all emails have mapped content sources:

```
CHECK 4: Source Mapping
  unmapped = [email for email in emails if email.content_source == "to_be_written"]
  unmapped_count = len(unmapped)
  unmapped_ratio = unmapped_count / total_emails

  IF unmapped_ratio > 0.30:
    → FAIL: "{unmapped_count}/{total_emails} emails ({unmapped_ratio:.0%})
      have no content source. Maximum 30% unmapped allowed."
    → REMEDIATION: "Provide additional assets or reduce sequence length"

  IF unmapped_ratio > 0 AND unmapped_ratio <= 0.30:
    → WARN: "{unmapped_count} emails will need E1 to generate content
      from scratch. Quality may be lower for these positions."
```

### Check 5: Word Count Realism

Verify that target word counts are within acceptable ranges:

```
CHECK 5: Word Count Ranges
  FOR email in emails:
    IF email.target_word_count < 150:
      → FAIL: "Position {position} word count {wc} below minimum 150"
      → REMEDIATION: "Raise to at least 200"

    IF email.target_word_count > 600:
      → FAIL: "Position {position} word count {wc} above maximum 600.
        Emails over 500 words have lower completion rates."
      → REMEDIATION: "Reduce to 400-500 range"

    IF email.body_type == "DU" AND email.target_word_count > 300:
      → WARN: "Deadline urgency email at position {position} is
        {wc} words. DU emails should be 150-250 for maximum impact."
```

### Check 6: CTA Pattern Consistency

Verify that CTA patterns match email functions:

```
CHECK 6: CTA-Function Alignment
  FOR email in emails:
    IF email.function IN [CP, AR] AND email.cta_pattern IN [C, D]:
      → WARN: "Content email at position {position} has direct/urgency
        CTA ({cta_pattern}). Content emails should use A or E patterns."

    IF email.function IN [BP, DU] AND email.cta_pattern IN [A, E]:
      → WARN: "Pitch email at position {position} has soft CTA
        ({cta_pattern}). Pitch emails should use C or D patterns."

    IF email.urgency_level >= 4 AND email.cta_pattern != "D":
      → WARN: "High urgency email (level {urgency}) at position
        {position} should use urgency CTA (pattern D)."
```

### Step 7: Compile Validation Report

Aggregate all checks into a single validation report with overall pass/fail.

```
OVERALL PASS CRITERIA:
  - Zero FAIL results across all 6 checks
  - WARNs are acceptable (flagged for human review but not blocking)

OVERALL FAIL:
  - Any single FAIL result → blueprint needs remediation before Gate 2
```

## Failure Modes

| Failure | Remediation |
|---------|-------------|
| Multiple variety rule violations | Auto-fix: perform body type swaps to satisfy all three variety rules simultaneously. Re-validate after swaps. |
| Urgency escalation broken | Auto-fix: flatten urgency levels to monotonic pattern. Re-validate. |
| Content ratio too low | Cannot auto-fix — requires removing pitch emails or converting pitch to content. Flag for human decision. |
| Too many unmapped positions | Cannot auto-fix — requires additional assets or sequence reduction. Flag for human decision. |
| CTA misalignments | Auto-fix: reassign CTA patterns to match function codes. Re-validate. |
