# Layer 3.2 — Output Packager

## Purpose

Package the approved and validated blueprint into two deliverable files: `campaign-blueprint.yaml` (structured data for downstream E1, E2, E3 skills) and `CAMPAIGN-BLUEPRINT-SUMMARY.md` (human-readable overview with email-by-email plan in table format). Set downstream handoff flags indicating which skills are ready to begin execution.

---

## Execution Protocol

### Step 1: Assemble campaign-blueprint.yaml

Construct the YAML file following the schema from EMAIL-STRATEGIST-AGENT.md:

```yaml
campaign_blueprint:
  version: "1.0"
  generated_at: "[ISO timestamp]"
  skill_id: "E0-email-strategist"
  mode: "[A|B]"

  campaign_meta:
    campaign_name: "[Project] [Campaign Type Name]"
    campaign_type: "[type code]"
    total_emails: [integer]
    duration_days: [integer]
    target_audience: "[audience profile summary]"
    primary_goal: "[detected goal]"

  emails:
    # One entry per email, ordered by position
    - position: [integer]
      day: [integer]
      body_type: "[CT|QO|TM|QA|LB|ST|NR]"
      function: "[DR|DU|PL|AL|BP|CP|AR]"
      content_focus: "[what this email is about]"
      content_source: "[asset_id or to_be_written]"
      urgency_level: [0-5]
      target_word_count: [200-500]
      emotional_target: "[emotion]"
      cta_pattern: "[A|B|C|D|E]"
      notes: "[special instructions for E1]"

  emotional_arc:
    summary: "[arc description]"
    phases:
      - phase: "trust_building"
        emails: [positions]
        emotional_range: "[emotions]"
      - phase: "desire_activation"
        emails: [positions]
        emotional_range: "[emotions]"
      - phase: "urgency_escalation"
        emails: [positions]
        emotional_range: "[emotions]"

  urgency_plan:
    mechanism: "[type of urgency]"
    escalation:
      - day: [integer]
        level: [0-5]
        language: "[description]"

  variety_verification:
    no_consecutive_same_type: [boolean]
    min_types_per_5_emails: [integer]
    nr_count: [integer]

  subject_line_strategy:
    formula_distribution: "[which SL formula categories to prioritize]"
    avoid_categories: "[which SL formulas to avoid for this audience]"

  downstream_handoff:
    e1_ready: [boolean]
    e2_ready: [boolean]
    e3_ready: [boolean]
```

### Step 2: Construct CAMPAIGN-BLUEPRINT-SUMMARY.md

Build a human-readable markdown document with:

**Section 1: Campaign Overview**
- Campaign type and name
- Mode (A or B)
- Total emails and duration
- Audience profile summary
- Primary goal
- Context richness score

**Section 2: Email-by-Email Plan (Table)**

```markdown
| # | Day | Type | Function | Content Focus | Urgency | Words | Emotion | CTA |
|---|-----|------|----------|---------------|---------|-------|---------|-----|
| 1 | 1   | ST   | CP       | Discovery story | 0     | 400   | curiosity | A |
| 2 | 2   | QO   | CP       | Expert authority | 0    | 350   | trust   | A |
| 3 | 3   | CT   | CP       | Myth-busting   | 0     | 400   | surprise | B |
...
```

**Section 3: Emotional Arc Summary**
- Phase descriptions with email ranges
- Urgency escalation timeline
- Key emotional transitions

**Section 4: Body Type Distribution**
- Count per body type
- Pie chart description (text-based)
- Variety rule compliance status

**Section 5: Asset Coverage**
- Mapped vs unmapped positions
- Assets used (with richness scores)
- Gaps flagged

**Section 6: Validation Summary**
- All 6 check results
- Any warnings
- Any remediations applied

**Section 7: Downstream Readiness**
- E1 ready flag + what E1 needs
- E2 ready flag + what E2 needs
- E3 ready flag + what E3 needs

### Step 3: Set Downstream Handoff Flags

```yaml
downstream_handoff:
  e1_ready: true
    # E1 (Email Writer) is ready when:
    # - All emails have body type assignments
    # - All emails have content focus defined
    # - Upstream context or brief is available

  e2_ready: true
    # E2 (Subject Line Generator) is ready when:
    # - All emails have content focus defined
    # - Audience profile is available
    # - Subject line strategy is defined

  e3_ready: true
    # E3 (Sequence Assembler) is ready when:
    # - Blueprint YAML is complete
    # - Email sequence is fully defined
    # - E1 and E2 outputs will slot in
```

Set any flag to `false` if its conditions are not met. Document the reason.

### Step 4: Write Files to Project Outputs

Write both files to the project outputs directory:

```
[OUTPUTS_ROOT]/[project-name]/E0-email-strategist/campaign-blueprint.yaml
[OUTPUTS_ROOT]/[project-name]/E0-email-strategist/CAMPAIGN-BLUEPRINT-SUMMARY.md
```

### Step 5: Final Verification

Before marking packaging complete:

```
VERIFICATION CHECKLIST:
  [ ] campaign-blueprint.yaml EXISTS and is valid YAML
  [ ] campaign-blueprint.yaml contains ALL schema sections from AGENT.md
  [ ] campaign-blueprint.yaml emails array has exactly total_emails entries
  [ ] campaign-blueprint.yaml each email has ALL required fields populated
  [ ] CAMPAIGN-BLUEPRINT-SUMMARY.md EXISTS
  [ ] CAMPAIGN-BLUEPRINT-SUMMARY.md contains all 7 sections
  [ ] CAMPAIGN-BLUEPRINT-SUMMARY.md table has exactly total_emails rows
  [ ] downstream_handoff flags are set with justification
  [ ] Validation report warnings are included in summary
```

## Failure Modes

| Failure | Remediation |
|---------|-------------|
| YAML schema validation fails | Re-check each section against AGENT.md schema. Fix missing fields. |
| Email array count mismatch | Recount emails from sequence_blueprint. Ensure no duplicates or missing positions. |
| Summary table incomplete | Regenerate table from YAML data. Every email must appear as a row. |
| Downstream skill not ready | Set flag to false. Document specific blocker. E1/E2/E3 should not start until their flag is true. |
| Output directory does not exist | Create directory structure before writing files. |
| File write fails | Retry once. If second failure, log error and request human intervention. |

## Integration Notes

- **Receives from:** All Layer 2 outputs (confirmed campaign, sequence, body types, emotional arc), all Layer 1 outputs (goal, audience, assets), Layer 3 validation report, Layer 0 validation report
- **Passes to:** E1 (Email Writer), E2 (Subject Line Generator), E3 (Sequence Assembler) — via the campaign-blueprint.yaml file. Human review — via the CAMPAIGN-BLUEPRINT-SUMMARY.md file.
- **Runs parallel with:** Nothing — 3.2 runs AFTER 3.1 (must be validated before packaging)
