# Layer 2.4 — Emotional Arc Planner

## Purpose

Plan emotional escalation across the entire email sequence. Divide the sequence into three phases (trust_building, desire_activation, urgency_escalation), define the emotional range and intensity for each phase, and map urgency levels per email. The emotional arc ensures the reader experiences a coherent psychological journey from curiosity to action.

## Execution Protocol

### Step 1: Define the Three Phases

Divide the email sequence into three roughly equal phases:

```yaml
phase_boundaries:
  trust_building:
    start: 1
    end: floor(total_emails / 3)
    purpose: "Establish credibility, build rapport, earn attention"

  desire_activation:
    start: floor(total_emails / 3) + 1
    end: floor(total_emails * 2 / 3)
    purpose: "Create desire, demonstrate mechanism, build wanting"

  urgency_escalation:
    start: floor(total_emails * 2 / 3) + 1
    end: total_emails
    purpose: "Create urgency, overcome final objections, drive action"
```

**Phase boundary adjustments by campaign type:**
- **PL (Product Launch):** Trust phase shorter (20%), desire longer (40%), urgency extended (40%)
- **AR (Autoresponder):** Trust phase longer (40%), desire moderate (35%), urgency light (25%)
- **CP (Content Series):** Trust phase dominant (50%), desire moderate (35%), urgency minimal (15%)
- **DR (Daily Relationship):** No fixed phases — cycle through trust/desire weekly, urgency only for promotions
- **BP/DU (Single email):** All three phases compressed into one email

### Step 2: Define Emotional Range Per Phase

| Phase | Primary Emotions | Intensity Level | Reader State |
|-------|-----------------|-----------------|--------------|
| **Trust Building** | Curiosity, empathy, recognition, surprise, validation | Low-Medium (1-3) | "This person understands me. I want to keep reading." |
| **Desire Activation** | Desire, hope, understanding, excitement, possibility | Medium-High (3-6) | "I want this. I can see how it works. This could be me." |
| **Urgency Escalation** | Urgency, consequence, fear of missing out, relief, resolution | High-Maximum (6-10) | "I need to act NOW. The cost of inaction is too high." |

### Step 3: Map Emotions to Body Types

Different body types naturally carry different emotional profiles:

| Body Type | Natural Emotional Register |
|-----------|---------------------------|
| ST (Story) | Empathy, recognition, surprise, hope — broad emotional range |
| TM (Testimonial) | Desire, social proof, possibility, validation |
| QO (Quote-Opener) | Authority, curiosity, surprise, credibility |
| LB (List-Based) | Understanding, clarity, data-driven confidence |
| QA (Q&A) | Resolution, reassurance, objection dissolving |
| CT (Contrarian) | Surprise, paradigm shift, righteous anger, awakening |
| NR (Negative Response) | Entertainment, personality, tribal belonging |

Verify that each email's body type (from 2.3) aligns with its phase's emotional range. Flag conflicts:

```
CONFLICT EXAMPLE: Story (ST) in urgency phase
  → Not ideal — stories slow momentum when urgency is needed
  → RESOLUTION: Short story (200 words max) with urgency framing,
    or swap with TM/LB

CONFLICT EXAMPLE: Contrarian (CT) in trust phase
  → Risky — challenging beliefs before trust is built can backfire
  → RESOLUTION: Use "gentle contrarian" angle (surprising, not aggressive)
```

### Step 4: Set Urgency Levels Per Email

Refine the urgency levels from 2.2 with the emotional arc in mind:

```yaml
urgency_mapping:
  trust_building_phase:
    urgency_range: [0, 0]
    rule: "ZERO urgency. Pure value. No mention of deadlines, scarcity, or offers."

  desire_activation_phase:
    urgency_range: [0, 2]
    rule: "Urgency appears gradually. First mention of opportunity.
           Soft deadline language only."
    pattern:
      first_email_in_phase: 0
      middle_emails: 1
      last_email_in_phase: 2

  urgency_escalation_phase:
    urgency_range: [2, 5]
    rule: "Escalating urgency. Each email more urgent than the last.
           Final email = maximum urgency (5)."
    pattern:
      first_email_in_phase: 2-3
      middle_emails: 3-4
      penultimate_email: 4
      final_email: 5
```

**Monotonic escalation enforcement:**
```
FOR EACH pair of adjacent emails (email_N, email_N+1):
  IF email_N.urgency_level > email_N+1.urgency_level:
    → VIOLATION — urgency went down
    → FIX: Raise email_N+1.urgency_level to at least email_N.urgency_level
  EXCEPTION: email_N can equal email_N+1 (plateau is acceptable)
```

### Step 5: Define Emotional Transitions Between Emails

For each consecutive pair, define the emotional transition:

```
Email 3 (curiosity) → Email 4 (understanding):
  Transition: "Curiosity satisfied by mechanism explanation.
  Reader moves from 'what is this?' to 'oh, that makes sense.'"

Email 7 (hope) → Email 8 (urgency):
  Transition: "Hope becomes urgent when deadline is introduced.
  Reader moves from 'I want this' to 'I need to act before it's too late.'"
```

### Step 6: Audience-Aware Arc Adjustments

Modify the emotional arc based on audience profile:

```
IF audience is Cold:
  → EXTEND trust phase by 1-2 emails
  → REDUCE urgency intensity ceiling to 4 (never full 5 for cold audiences)
  → Emotion focus: heavy empathy, recognition, validation before ANY desire

IF audience is Unaware:
  → ADD "awakening" emotions to trust phase (surprise, concern, revelation)
  → Desire phase starts with "understanding" before "wanting"

IF audience is Most-Aware:
  → COMPRESS trust phase to 1-2 emails
  → Start desire at higher intensity (3-4 instead of 1-2)
  → Urgency can begin earlier
```

## Failure Modes

| Failure | Remediation |
|---------|-------------|
| Urgency escalation is not monotonic | Auto-fix: raise any email that breaks the pattern to at least the previous level. |
| Body type conflicts with phase emotion | Flag conflict. Recommend swap if alternative body type available. If not, note as "acceptable with framing adjustment." |
| Emotional range too narrow (same emotion repeated) | Diversify: ensure at least 3 different emotions per phase. |
| Audience arc adjustments conflict with campaign type defaults | Audience adjustments take priority. Campaign type provides the structure; audience profile tunes the emotional content. |
| Single-email campaign (BP/DU) | Compress all three phases into one email: hook (trust) → body (desire) → close (urgency). |
