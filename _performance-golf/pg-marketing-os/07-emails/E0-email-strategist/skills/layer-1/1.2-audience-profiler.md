# Layer 1.2 — Audience Profiler

## Purpose

Profile the email audience across four dimensions: awareness level, sophistication level, relationship stage, and preferred content density. These dimensions directly influence body type selection, content depth, urgency calibration, and sequence pacing in Layer 2.

---

## Input Requirements

```yaml
from_layer_0:
  mode: enum[A, B]

  # Mode A:
  upstream_context:
    research:
      audience_intelligence: object
      market_language: object
      pain_quotes: [string]
      hope_quotes: [string]
    structure:
      argument_strategy: string

  # Mode B:
  brief_context:
    audience:
      description: string
      pain_points: [string]
      desires: [string]
      awareness_level: string
      relationship_stage: string

from_layer_1:
  goal_analysis:
    campaign_type: string
```

## Execution Protocol

### Step 1: Determine Awareness Level

Apply Schwartz's 5 Levels of Awareness to the email audience:

| Level | Definition | Email Implication |
|-------|-----------|-------------------|
| **Unaware** | Don't know they have a problem | Lead with story, education. NO pitch in early emails. |
| **Problem-Aware** | Know the problem, don't know solutions exist | Lead with root cause, villain. Introduce mechanism gradually. |
| **Solution-Aware** | Know solutions exist, don't know THIS product | Lead with mechanism differentiation. Compare to alternatives. |
| **Product-Aware** | Know this product, haven't bought yet | Lead with proof, testimonials, urgency. Address objections. |
| **Most-Aware** | Know this product, nearly ready to buy | Lead with offer, deadline, guarantee. Minimal education needed. |

**Mode A:** Derive from research audience_intelligence. Look for explicit awareness mapping. If research categorized awareness levels, use that data directly.

**Mode B:** Estimate from brief context. Default assumptions:
- New subscriber list → Problem-Aware (they opted in for a reason)
- Existing customer list → Product-Aware or Most-Aware
- Cold traffic → Unaware or Problem-Aware
- Brief explicitly states → Use stated level

### Step 2: Determine Sophistication Level

Apply Schwartz's 5 Stages of Market Sophistication:

| Stage | Market State | Email Implication |
|-------|-------------|-------------------|
| **Stage 1** | First to market — claims are new | Direct promises work. Simple CTAs. |
| **Stage 2** | Second wave — claims being copied | Bigger promises needed. Stack benefits. |
| **Stage 3** | Market crowded — unique mechanism needed | Lead with mechanism. Differentiate from competitors. |
| **Stage 4** | Skeptical market — proof-heavy required | Testimonial-heavy. Data-heavy. Social proof dominant. |
| **Stage 5** | Burned-out market — identity/story driven | Story-first. Identity play. Contrarian angles. |

**Mode A:** Derive from research sophistication analysis if available.
**Mode B:** Estimate from brief's competitor context. More competitors → higher sophistication.

### Step 3: Determine Relationship Stage

Classify the audience's relationship with the sender:

| Stage | Definition | Email Implication |
|-------|-----------|-------------------|
| **Cold** | Never heard of sender | Trust-building mandatory. Content Series recommended before promotion. |
| **Warm** | On list, some engagement | Can introduce offers after 2-3 value emails. |
| **Hot** | Engaged, opened/clicked recently | Can pitch earlier. Urgency effective. |
| **Customer** | Already purchased something | Upsell/cross-sell. Relationship deepening. Skip trust-building. |

**Mode A:** Infer from research audience intelligence and campaign context.
**Mode B:** Use brief's relationship_stage if provided. Default to Warm for existing lists, Cold for new lists.

### Step 4: Determine Preferred Content Density

Based on the three dimensions above, determine how dense each email's content should be:

```yaml
content_density_matrix:
  light:  # 200-300 words
    when: "Most-Aware + Hot/Customer + Stage 1-2"
    body_types: "Short ST, BP direct, DU"
    pitch_ratio: "50%+ pitch acceptable"

  medium:  # 300-400 words
    when: "Solution-Aware or Product-Aware + Warm + Stage 2-3"
    body_types: "TM, QO, QA, short LB"
    pitch_ratio: "30% pitch, 70% content"

  dense:  # 400-500+ words
    when: "Unaware or Problem-Aware + Cold + Stage 4-5"
    body_types: "Long ST, CT, detailed LB, educational QA"
    pitch_ratio: "20% pitch, 80% content"
```

### Step 5: Generate Audience Profile Summary

Synthesize all four dimensions into a coherent profile narrative that describes WHO this audience is and HOW they should be approached via email.

## Output

```yaml
audience_profile:
  awareness_level:
    level: enum[unaware, problem_aware, solution_aware, product_aware, most_aware]
    confidence: enum[high, medium, low]
    evidence: string  # what data supports this classification
    email_implication: string

  sophistication_level:
    stage: integer  # 1-5
    confidence: enum[high, medium, low]
    evidence: string
    email_implication: string

  relationship_stage:
    stage: enum[cold, warm, hot, customer]
    confidence: enum[high, medium, low]
    evidence: string
    email_implication: string

  content_density:
    preferred: enum[light, medium, dense]
    target_word_count: string  # range, e.g., "300-400"
    pitch_ratio: string  # e.g., "30% pitch, 70% content"
    recommended_body_types: [string]

  profile_summary: string  # 2-3 sentence narrative describing the audience

  campaign_type_alignment:
    selected_type: string  # from 1.1
    alignment: enum[strong, moderate, weak]
    adjustment_notes: string  # if campaign type should be reconsidered based on audience
```

---

## Failure Modes

| Failure | Remediation |
|---------|-------------|
| Cannot determine awareness level (no data) | Default to Problem-Aware (safest middle ground). Flag low confidence. |
| Cannot determine sophistication (no competitor data) | Default to Stage 3 (assume moderate competition). Flag low confidence. |
| Audience profile contradicts campaign type | Flag at Gate 1: "Audience is Cold/Unaware but campaign type is Blatant Pitch. Recommend Content Series instead." |
| Multiple audience segments detected | Profile the PRIMARY segment. Note secondary segments as potential future sequences. |
| Mode B brief has minimal audience info | Extract what exists, default remaining dimensions, flag all as low confidence. |
