# Layer 0.1 — Upstream Package Loader (Mode A)

## Purpose

Load and parse all available upstream skill packages (Skills 01-09) into working memory for use by downstream layers. Extract the specific fields needed for email campaign design — audience intelligence, proof elements, root cause, mechanism, promise, big idea, offer structure, and campaign structure.

**Mode:** A only. If operating in Mode B (standalone), this microskill is SKIPPED and 0.2-brief-parser.md runs instead.

---

## Input Requirements

```yaml
upstream_paths:
  research_handoff: "01-research/outputs/FINAL_HANDOFF.md"
  proof_inventory: "02-proof-inventory/outputs/FINAL_HANDOFF.md"
  root_cause_package: "03-root-cause/outputs/root-cause-package.yaml"
  mechanism_package: "04-mechanism/outputs/mechanism-package.json"

```

## Execution Protocol

### Step 1: Load Research Handoff (Skill 01)

Extract and index:
- `audience_intelligence` — demographics, psychographics, awareness levels
- `market_language` — verbatim quotes, phrases, vocabulary the audience uses
- `pain_quotes` — direct quotes expressing pain points (fuel for Story body type)
- `hope_quotes` — direct quotes expressing aspirations (fuel for Story body type)
- `villain_inventory` — identified villains (fuel for Contrarian body type)
- `competitor_mechanisms` — what competitors promise (fuel for differentiation emails)

### Step 2: Load Proof Inventory (Skill 02)

Extract and index:
- `testimonials` — customer success stories (fuel for Testimonial body type)
- `studies` — clinical/research data (fuel for List-Based body type)
- `data_points` — specific numbers and statistics (fuel for List-Based body type)
- `expert_quotes` — authority endorsements (fuel for Quote-Opener body type)
- `case_studies` — detailed transformation stories (fuel for Story body type)
- `overall_strength` — aggregate proof strength score

### Step 3: Load Root Cause Package (Skill 03)

Extract and index:
- `root_cause.expression` — the core root cause statement
- `root_cause.three_part_structure` — what they think, what it really is, why nothing worked
- `root_cause.villain_profile` — the villain construct (fuel for Contrarian body type)
- `root_cause.anchor_phrase` — repeatable root cause phrase

### Step 4: Load Mechanism Package (Skill 04)

Extract and index:
- `mechanism.name` — the named mechanism
- `mechanism.explanation_text` — how it works (fuel for educational emails)
- `mechanism.uniqueness_claim` — differentiation statement
- `mechanism.metaphor_anchor` — graspable image for simplification

### Step 5: Load Promise Package (Skill 05)

Extract and index:
- `primary_promise.statement` — the core promise
- `primary_promise.emotional_frame` — emotional context
- `supporting_promises` — secondary promises for email variety
- `promise_ceiling` — maximum defensible claim

### Step 6: Load Big Idea Package (Skill 06)

Extract and index:
- `big_idea.concept` — the central idea driving the campaign
- `big_idea.creative_wrapper` — how the idea is packaged
- `big_idea.headline` — selected headline (for subject line inspiration)

### Step 7: Load Offer Package (Skill 07)

Extract and index:
- `core_offer.product_name` — product/service name
- `core_offer.positioning` — how the offer is positioned
- `core_offer.price` — pricing information
- `enhancement_stack` — bonuses and enhancements
- `guarantee` — guarantee details and language

### Step 8: Load Structure Package (Skill 08)

Extract and index:
- `campaign_thesis` — the thesis the campaign proves
- `argument_strategy` — prosecution, revelation, education, demonstration
- `cpb_chunks` — claim-proof-benefit chunks (content blocks for emails)

### Step 9: Build Integrated upstream_context Object

Merge all extracted data into a single `upstream_context` object. Tag each package with loading status.

## Failure Modes

| Failure | Remediation |
|---------|-------------|
| Package file not found | Log missing package, continue loading others. Record in loading_status. |
| Package file malformed | Log parse error with file path. Mark that package as not loaded. |
| Fewer than 5 packages loaded | Set minimum_met = false. Gate 0 will FAIL. |
| Required fields missing within a loaded package | Log missing fields. Package counts as loaded but with warnings. |
