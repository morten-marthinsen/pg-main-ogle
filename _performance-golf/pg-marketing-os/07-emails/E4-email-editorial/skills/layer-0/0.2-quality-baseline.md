# Layer 0.2 -- Quality Baseline

## Purpose

Score every email in the loaded sequence against email-quality-rubric.md to establish the pre-revision baseline. This baseline is the ONLY way to prove the editorial process improved quality. Without it, revision is unverifiable. Also score the 5 campaign-level criteria (C1-C5).

**CRITICAL:** This microskill MUST complete before ANY revision is applied. Baselines cannot be retroactively established.

---

## Input Requirements

```yaml
from_0_1:
  loaded_sequence:
    emails: [object]  # all parsed emails
    campaign_meta: object
    reference_files:
      rubric_available: true  # MUST be true

required_reads:
  rubric: "./email/shared/email-quality-rubric.md"
  niche_guide: "./email/shared/niche-adaptation-guide.md"
```

---

## Execution Protocol

### Step 1: Read email-quality-rubric.md (MANDATORY)

```
READ the rubric file. Do NOT score from memory.
EXTRACT the 9 individual criteria with their weights:
  1. Opening Hook Strength (15%)
  2. Content-to-Pitch Ratio (15%)
  3. Transition/Bridge Smoothness (15%)
  4. Body Type Compliance (10%)
  5. Voice Consistency (10%)
  6. Engagement/Entertainment Value (10%)
  7. CTA Clarity and Strength (10%)
  8. Paragraph Brevity (5%)
  9. Niche Appropriateness (10%)

IF rubric was not read in THIS session:
  HALT -- Read the file. Do not score from memory.
```

### Step 2: Score Each Email (Individual Criteria)

For EVERY email in the sequence:

```
FOR email IN loaded_sequence.emails:

  1. OPENING HOOK STRENGTH (15%)
     Read the first 3 lines. Score 1-10 per rubric definition.
     Ask: "Would a reader keep reading or delete?"

  2. CONTENT-TO-PITCH RATIO (15%)
     Count words in content sections vs. pitch sections.
     Bridge sentence counts as pitch.
     Transition + URL + sign-off = pitch.
     Calculate percentage. Score 1-10 per rubric thresholds.

  3. TRANSITION/BRIDGE SMOOTHNESS (15%)
     Locate the content-to-pitch transition point.
     Score 1-10: Does the reader notice the shift?

  4. BODY TYPE COMPLIANCE (10%)
     Check against the assigned body type checklist:
       CT: Bold claim? Arguments stacked? Dismissals? Bridge?
       QO: Quote formatted? Riff? Lesson? Bridge?
       TM: Attribution? Testimonial blocked? Reaction? CTA?
       QA: Attribution? Question? Substantive answer? Bridge?
       LB: Setup? Numbered list? Wrap-up? CTA?
       ST: Story opener? Narrative? Lesson? Bridge?
       NR: Troll displayed? Demolition? Lesson? Bridge?
     Score 1-10 based on how many structural elements are present.

  5. VOICE CONSISTENCY (10%)
     Read the full email. Does the voice stay consistent?
     Score 1-10 per rubric definition.

  6. ENGAGEMENT/ENTERTAINMENT VALUE (10%)
     Would the reader enjoy this email even if they never click?
     Score 1-10 per rubric definition.

  7. CTA CLARITY AND STRENGTH (10%)
     Is it clear what the reader should do and why?
     Score 1-10 per rubric definition.

  8. PARAGRAPH BREVITY (5%)
     Count sentences per paragraph.
     Target: 1-2 sentences. Max: 3 sentences.
     Score 1-10 per rubric definition.

  9. NICHE APPROPRIATENESS (10%)
     Cross-reference with niche-adaptation-guide.md.
     Is tone, vocabulary, and cultural reference set appropriate?
     Score 1-10 per rubric definition.

  CALCULATE weighted average:
    score = (hook * 0.15) + (ratio * 0.15) + (bridge * 0.15) +
            (compliance * 0.10) + (voice * 0.10) + (engagement * 0.10) +
            (cta * 0.10) + (brevity * 0.05) + (niche * 0.10)
```

### Step 3: Score Campaign-Level Criteria (C1-C5)

```
C1: BODY TYPE VARIETY
  Extract body types by position: [ST, CT, QA, TM, LB, ...]
  Check: No consecutive same type.
  Check: In any 5-email window, >= 3 different types.
  Result: PASS or FAIL (with specific violations noted)

C2: EMOTIONAL ARC
  Extract emotional targets by position.
  Check: Clear progression (trust -> education -> desire -> urgency).
  Result: PASS or FAIL (with arc description)

C3: CROSS-EMAIL CONTINUITY
  Search emails 3+ for references to earlier emails.
  Look for: callbacks, "as I mentioned," "yesterday I told you," etc.
  Result: PASS or FAIL (with callback count)

C4: URGENCY ESCALATION
  Extract urgency levels by position.
  Check: First 2/3 of sequence has urgency <= 1.
  Check: Final 1/3 shows clear escalation.
  Result: PASS or FAIL (with urgency pattern)

C5: SUBJECT LINE DIVERSITY
  Classify each subject line by formula category.
  Count unique categories.
  Check: >= 3 different formula categories used.
  Result: PASS or FAIL (with category distribution)
```

### Step 4: Compile Baseline Summary

Calculate aggregate statistics:
- Average weighted score across all emails
- Lowest scoring email (position + score)
- Highest scoring email (position + score)
- Score distribution (how many in each threshold band)
- Campaign criteria pass/fail summary
- Weakest individual criterion across the sequence

## Failure Modes

| Failure | Remediation |
|---------|-------------|
| Rubric file not read in this session | HALT -- read email-quality-rubric.md before scoring |
| Email body too short to meaningfully score (< 50 words) | Score as-is but flag. Opening hook and bridge may not exist. |
| Body type assignment missing for an email | Score body type compliance as 1. Flag as P1. |
| Niche guide unavailable | Score niche appropriateness based on general principles. Note limitation. |
| All emails score below 6.0 | Baseline is valid but note: sequence may need significant rework, not just editorial. Flag for human. |
