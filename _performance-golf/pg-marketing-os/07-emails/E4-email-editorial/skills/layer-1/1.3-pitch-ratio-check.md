# Layer 1.3 -- Pitch Ratio Check

## Purpose

Verify the content-to-pitch ratio for every email in the sequence. The target is 70-80% content / 20-30% pitch. This is the defining structural principle of Ben Settle-style emails: the content IS the marketing. Emails that exceed 30% pitch become sales letters. Emails below 20% pitch waste the business purpose of sending.

---

## Input Requirements

```yaml
from_layer_0:
  loaded_sequence:
    emails: [object]  # all parsed emails with body text and word counts
  quality_baseline:
    per_email_scores: [object]  # baseline content_pitch_ratio scores (criterion 2)
```

---

## Execution Protocol

### Step 1: Define Content vs. Pitch Boundaries

Before measuring, establish clear definitions:

```
CONTENT (counts toward content percentage):
  - Opening hook / attention grabber
  - Story, narrative, anecdote
  - Educational points, tips, insights
  - Quote riffs, commentary, analysis
  - Testimonial display and reaction (the testimonial itself)
  - List items (the educational content within them)
  - Q&A (the question and the substantive answer)
  - Any text that provides VALUE independent of the sale

PITCH (counts toward pitch percentage):
  - Bridge/transition sentence(s) to the offer
  - Product/offer mention by name
  - Benefit claims tied to the product (not general education)
  - CTA lines (every "click here" / "go here" / URL reference)
  - P.S. lines that are purely promotional
  - Urgency/deadline language
  - Price mentions
  - Guarantee mentions
  - Sign-off that includes a pitch element
  - "Learn more at [URL]" type sentences

GRAY ZONE (count as 50/50):
  - Bridge sentences that contain both lesson and pitch
  - "The reason I mention this is because [product]..."
  - Testimonials that are primarily about the product (product testimonial = pitch)
  - Educational content that is ONLY relevant to product users
```

### Step 2: Measure Each Email

For EVERY email in the sequence:

```
FOR email IN loaded_sequence.emails:

  1. READ the complete email body
  2. SEGMENT into content vs. pitch sections
  3. COUNT words in each section:
     content_words = [total words in content sections]
     pitch_words = [total words in pitch sections]
     gray_words = [total words in gray zone sections]
     total_words = content_words + pitch_words + gray_words

  4. CALCULATE ratios:
     adjusted_content = content_words + (gray_words * 0.5)
     adjusted_pitch = pitch_words + (gray_words * 0.5)
     content_percentage = adjusted_content / total_words * 100
     pitch_percentage = adjusted_pitch / total_words * 100

  5. CLASSIFY result:
     IF content_percentage >= 70 AND content_percentage <= 80:
       → OPTIMAL (no issue)
     IF content_percentage > 80:
       → PITCH_LIGHT (P3 -- could pitch more, but not critical)
     IF content_percentage >= 60 AND content_percentage < 70:
       → PITCH_HEAVY (P3 -- edging toward sales letter)
     IF content_percentage < 60:
       → RATIO_VIOLATED (P1 -- this is a sales letter, not a Ben Settle email)
     IF pitch_percentage < 10:
       → NO_PITCH (P3 -- content-only, missing business purpose)

  6. IDENTIFY specific pitch elements:
     - Where does the bridge occur? (word position / percentage through email)
     - How many CTA instances?
     - Bridge smoothness (is the transition abrupt or natural?)

  7. RECORD findings
```

### Step 3: Check Ratio Patterns Across Sequence

Beyond individual emails, check if the sequence as a whole maintains good ratio patterns:

```
SEQUENCE-LEVEL CHECKS:

  1. RATIO DRIFT:
     Do later emails in the sequence get more pitch-heavy?
     (Common failure: pitch creeps up as urgency increases)
     IF last 3 emails ALL have content_percentage < 65:
       → P2 issue: "Sequence becomes a sales blitz at the end"

  2. 5-EMAIL WINDOW RULE:
     In any 5-email window, at least 3 should be content-forward (>= 70% content).
     CHECK all windows. Flag violations.

  3. BLATANT PITCH EMAILS:
     Emails with function = BP (Blatant Pitch) are EXEMPT from ratio rules.
     They are intentionally pitch-heavy. Do not flag.

  4. DEADLINE EMAILS:
     Emails with function = DU (Deadline Urgency) get a relaxed ratio:
     60-70% content / 30-40% pitch is acceptable.
     Do not flag DU emails unless pitch > 40%.
```

## Failure Modes

| Failure | Remediation |
|---------|-------------|
| Email is extremely short (<50 words) | Ratio measurement is unreliable at very low word counts. Score but flag as low-confidence. |
| No pitch section found at all | Classify as NO_PITCH (P3). The email serves no business purpose. |
| Entire email is pitch | Classify as RATIO_VIOLATED (P1). Unless function = BP (exempt). |
| Gray zone sections dominate (>40% of email) | Note that measurement has high uncertainty. Still calculate using 50/50 split. |
| Bridge is unclear / cannot identify transition point | Note as a separate finding for 1.1-structural-audit. For ratio purposes, estimate best split point. |
