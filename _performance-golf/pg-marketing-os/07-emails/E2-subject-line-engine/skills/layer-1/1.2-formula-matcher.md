# 1.2-formula-matcher.md

> **Layer:** 1 (Formula Selection)
> **Skill:** E2-subject-line-engine
> **Purpose:** Match SL formula categories to the email content. Select 5-8 best-fit formulas.
> **Outputs:** `formula_selection` object with selected categories, rationale, and specimen assignments

---

## FUNCTION

Using the emotional triggers identified in 1.1 and the email metadata from Layer 0, select 5-8 formula categories that best match this specific email's content, tone, and purpose. Each selected formula must have a clear rationale connecting it to the email content. This selection drives the volume generation in Layer 2 and provides formula assignments for Arena competitors.

---

## MATCHING PROTOCOL

```
EXECUTE 1.2-formula-matcher:
  INPUT: emotional_triggers (from 1.1), email_context (from Layer 0), formula_library (from 0.2)
  OUTPUT: formula_selection object

  PHASE 1: TRIGGER-BASED MATCHING
    1. LOAD recommended_formula_pool from 1.1
    2. FOR each recommended formula:
       a. CHECK: Does this formula's structural pattern fit the email content?
       b. CHECK: Is the formula's tonal guidance compatible with the email's register?
       c. CHECK: Are specimens available for this formula?
       d. SCORE fit (1-10)
       e. DOCUMENT rationale

  PHASE 2: CONTENT-BASED MATCHING
    3. ANALYZE email content for formula-specific opportunities:
       a. Does email mention a specific person? --> SL-NAM candidate
       b. Does email contain a list/numbers? --> SL-NUM candidate
       c. Does email tell a story? --> SL-THE or SL-CUR candidate
       d. Does email make a contrarian argument? --> SL-WHY or SL-PRV candidate
       e. Does email include testimonial/proof? --> SL-PRF candidate
       f. Does email reference pop culture? --> SL-POP candidate
       g. Does email contain a confession/admission? --> SL-CON candidate
       h. Does email have a deadline? --> SL-URG candidate
       i. Does email compare two things? --> SL-VS candidate
       j. Does email contain a paradox or contradiction? --> SL-PAR candidate

  PHASE 3: BODY TYPE COMPATIBILITY
    4. CHECK body type to formula compatibility:

       | Body Type | High Compatibility Formulas | Low Compatibility Formulas |
       |-----------|---------------------------|---------------------------|
       | CT (Contrarian) | SL-WHY, SL-PRV, SL-PAR, SL-CMD | SL-PRF, SL-URG |
       | QO (Quote-Opener) | SL-NAM, SL-THE, SL-HOW | SL-URG, SL-BPM |
       | TM (Testimonial) | SL-PRF, SL-HOW, SL-THE | SL-CON, SL-PRV |
       | QA (Q&A) | SL-HOW, SL-WHY, SL-CUR | SL-BPM, SL-URG |
       | LB (List-Based) | SL-NUM, SL-HOW, SL-THE | SL-CON, SL-SIM |
       | ST (Story) | SL-THE, SL-CUR, SL-CON, SL-POP | SL-NUM, SL-BPM |
       | NR (Negative Response) | SL-EDG, SL-PRV, SL-WHY | SL-PRF, SL-URG |

  PHASE 4: SELECTION AND RANKING
    5. COMBINE trigger-based, content-based, and body-type scores
    6. RANK all formula categories by combined fit score
    7. SELECT top 5-8 categories
       MINIMUM: 5 categories (HARD FLOOR)
       MAXIMUM: 8 categories (diminishing returns beyond 8)
    8. ENSURE at least 1 "high-volume" formula (SL-THE, SL-HOW, or SL-WHY)
       These have the most specimens and highest statistical grounding
    9. ENSURE at least 1 "low-volume" formula (any category < 3% corpus)
       These provide originality and surprise

  PHASE 5: SPECIMEN ASSIGNMENT
    10. FOR each selected formula:
        a. LOAD 5-10 specimen SLs from that category
        b. SELECT specimens most relevant to the email's niche/topic
        c. ASSIGN specimens to the formula for generation reference

  11. ASSEMBLE formula_selection object
  12. RETURN formula_selection
```

## SELECTION CONSTRAINTS

1. **Minimum 5 categories** -- Below 5 reduces diversity unacceptably
2. **Maximum 8 categories** -- Above 8 dilutes generation quality per category
3. **At least 1 high-volume formula** -- SL-THE, SL-HOW, or SL-WHY (proven patterns)
4. **At least 1 low-volume formula** -- Any <3% category (originality)
5. **SL-EDG only as overlay** -- Must be paired with a base formula
6. **SL-URG only if deadline exists** -- email_context.pitch_elements.deadline_present must be true
7. **SL-PRF only if proof exists** -- email_context.pitch_elements.testimonial_present must be true
8. **SL-BPM only if function warrants** -- Blatant pitch meta works only for promotional emails

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Fewer than 5 formulas match | Relax matching thresholds. Include medium-fit formulas. |
| CRITICAL | Zero high-volume formulas match | Force-include SL-THE (most versatile) |
| DEGRADED | All matches are same trigger type | WARN -- Expand to secondary triggers |
| DEGRADED | Body type has no high-compatibility matches | WARN -- Use medium-compatibility formulas |
