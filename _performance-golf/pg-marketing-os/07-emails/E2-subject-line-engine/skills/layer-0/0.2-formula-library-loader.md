# 0.2-formula-library-loader.md

> **Layer:** 0 (Loading)
> **Skill:** E2-subject-line-engine
> **Purpose:** Load formula specifications for all 18 SL categories with specimens
> **Outputs:** `formula_library` object containing all 18 categories with specs and specimens

---

## FUNCTION

Load the complete formula library for all 18 subject line categories derived from the Ben Settle corpus analysis of 2,684 emails. Each category includes: structural pattern, sub-formulas (if applicable), corpus frequency, tonal guidance, specimen SLs, and niche adaptation notes. All 18 categories must be available for Layer 1 formula selection.

## LOADING PROTOCOL

```
EXECUTE 0.2-formula-library-loader:
  INPUT: Formula specification directory, specimen directories
  OUTPUT: formula_library object

  1. LOAD formula specifications for all 18 categories
     FOR each category:
       a. Load structural pattern definition
       b. Load sub-formulas (if applicable)
       c. Load corpus frequency data
       d. Load tonal guidance
       e. Load niche adaptation notes
       f. Load usage constraints

  2. LOAD specimen SLs for each category
     a. READ specimens/subject-lines/subject-line-specimens.md (master specimen file — 90-120+ curated SLs organized by all 18 categories)
     b. FOR each category: extract 5-8 specimen SLs from the category section
     c. HOLD specimens VERBATIM (do not summarize)
     d. ALSO LOAD Top 10 Most Reused SLs section (highest-signal proven performers)
     e. ALSO LOAD 20 Master Formulas quick-reference section

  3. LOAD formula interaction rules
     a. SL-EDG is overlay-only (must combine with base formula)
     b. SL-BPM max 1 per 7 emails
     c. SL-URG only for deadline emails
     d. SL-PRF requires real proof data
     e. Same category max 2 consecutive in a sequence

  4. VERIFY all 18 categories loaded
     IF any category missing:
       WARN but continue (degrade gracefully)
       LOG: "Category [X] not loaded. Generation diversity reduced."

  5. ASSEMBLE formula_library object
  6. RETURN formula_library
```

## FORMULA SPECIFICATION FORMAT (Per Category)

```yaml
formula_spec:
  code: "SL-THE"
  name: "The [X]"
  corpus_percentage: 15.5
  structural_pattern: "The + [noun/noun phrase]"
  description: "Definite article creates authority, specificity, and mystery. Works because 'The' implies there is ONE specific thing worth knowing."

  sub_formulas:
    - pattern: "The [adjective] [noun]"
      example: "The lazy way to sell"
    - pattern: "The [number] [noun]"
      example: "The 3 email killers"
    - pattern: "The [noun] [metaphor]"
      example: "The copywriting samurai"

  tonal_guidance:
    register: "authoritative to mysterious"
    energy: "medium -- confidence, not hype"
    avoid: "Generic nouns without specificity"

  niche_adaptation:
    health: "Use body-specific or condition-specific nouns"
    finance: "Use money/market/wealth-specific nouns"
    golf: "Use swing/course/game-specific nouns"
    personal_dev: "Use identity/growth-specific nouns"
    technology: "Use product/innovation-specific nouns"

  constraints:
    word_count_typical: "3-6 words"
    works_best_with: "Mystery, authority, specificity"
    avoid_with: "Urgency (too indirect), blatant pitch (too subtle)"

  specimens: # Loaded from specimens/subject-lines/subject-line-specimens.md
    - "[5-8 curated specimens per category — see subject-line-specimens.md]"
```
