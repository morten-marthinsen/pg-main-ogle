# 0.1-email-context-loader.md

> **Layer:** 0 (Loading)
> **Skill:** E2-subject-line-engine
> **Purpose:** Load the WRITTEN email body from E1 output -- the primary input for SL generation
> **Outputs:** `email_context` object containing full email text and extracted metadata

---

## FUNCTION

Load the complete, Arena-selected email body from E1's output. Extract metadata (body type, content focus, emotional register, key themes, opening hook, bridge/CTA). This metadata drives formula selection in Layer 1 and alignment checking throughout.

**CRITICAL:** This loader MUST load the ACTUAL WRITTEN email, not a planned/hypothetical email. The email body MUST come from E1's output (post-Arena selection). If E1 has not completed for this email position, E2 CANNOT proceed.

---

## DEPENDENCY CHECK

```
BEFORE loading email:
  1. LOCATE E1 output for the target email position
     PATHS to check:
       - [project]/E1-email-writer/[email-id]/email-draft.md
       - [project]/E1-email-writer/[email-id]/ARENA_COMPLETE.yaml
       - [project]/E1-email-writer/outputs/[email-id]-draft.md

  2. VERIFY the email draft is Arena-selected (not a Layer 2 pre-Arena draft)
     CHECK: ARENA_COMPLETE.yaml exists OR human_selection field populated

  3. IF email draft NOT found:
     FAIL: "E1 has not completed email [email-id]. Cannot generate subject lines."
     HALT -- Return to E1

  4. IF email draft is pre-Arena:
     FAIL: "E1 Arena not complete for email [email-id]. SLs must match Arena-selected body."
     HALT -- Complete E1 Arena first
```

---

## LOADING PROTOCOL

```
EXECUTE 0.1-email-context-loader:
  INPUT: E1 output directory, target email identifier
  OUTPUT: email_context object

  1. LOCATE email draft file
     VERIFY: File exists and is non-empty
     VERIFY: Word count >= 100 (minimum viable email)

  2. LOAD email body VERBATIM
     DO NOT summarize, paraphrase, or truncate
     Store as email_context.body_text

  3. EXTRACT email metadata:
     a. body_type: [CT|QO|TM|QA|LB|ST|NR] from E0 blueprint or E1 output
     b. content_focus: What is this email about? (1-2 sentence summary)
     c. emotional_register: [confident|playful|urgent|analytical|combative|thoughtful|vulnerable]
     d. key_themes: 3-5 primary themes/topics in the email
     e. opening_hook: First 3 lines of the email (verbatim)
     f. bridge_summary: The content-to-pitch transition (1-2 sentences)
     g. cta_summary: What action the email asks the reader to take
     h. word_count: Exact word count of email body

  4. EXTRACT pitch elements:
     a. product_mentioned: [Y/N] + product name if yes
     b. deadline_present: [Y/N] + deadline details if yes
     c. testimonial_present: [Y/N] + testimonial summary if yes
     d. mechanism_mentioned: [Y/N] + mechanism name if yes

  5. ASSEMBLE email_context object
  6. RETURN email_context
```

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | E1 email draft not found | HALT -- Return to E1 for this email position |
| CRITICAL | E1 Arena not complete | HALT -- Complete E1 Arena before starting E2 |
| CRITICAL | Email body empty or < 100 words | HALT -- E1 output is incomplete |
| DEGRADED | Body type not identified | WARN -- Proceed but formula matching will be limited |
| DEGRADED | Metadata extraction incomplete | WARN -- Manual formula selection may be needed |
