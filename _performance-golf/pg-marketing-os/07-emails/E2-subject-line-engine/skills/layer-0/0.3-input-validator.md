# 0.3-input-validator.md

> **Layer:** 0 (Loading)
> **Skill:** E2-subject-line-engine
> **Purpose:** Validate all inputs are present and meet minimum thresholds before SL generation
> **Outputs:** `validation_status` with pass/fail and remediation guidance

---

## FUNCTION

Validate that all required inputs from Layer 0 loaders are present, properly formatted, and above minimum quality thresholds. Gate Layer 1 execution until validation passes. This is the final checkpoint before SL generation begins.

---

## VALIDATION CHECKLIST

### Critical Requirements (Must Pass)

| Input | Source | Validation Rule | Failure Action |
|-------|--------|-----------------|----------------|
| Email body text | email_context | Non-null, >= 100 words | FAIL -- Return to E1 |
| Email Arena-selected | email_context | arena_selected = true | FAIL -- Complete E1 Arena |
| Email body type | email_context | Valid code [CT/QO/TM/QA/LB/ST/NR] | FAIL -- Identify body type |
| Formula library | formula_library | >= 10 categories loaded | FAIL -- Load formula library |
| Key themes extracted | email_context | >= 3 themes identified | FAIL -- Re-extract metadata |

### High Priority Requirements (Should Pass)

| Input | Source | Validation Rule | Failure Action |
|-------|--------|-----------------|----------------|
| Emotional register | email_context | Register identified | WARN -- Proceed with default |
| Opening hook | email_context | First 3 lines captured | WARN -- Manual theme extraction |
| Bridge summary | email_context | Bridge identified | WARN -- SL alignment may be weaker |
| CTA summary | email_context | CTA identified | WARN -- Urgency SLs limited |
| All 18 categories | formula_library | all_categories_available = true | WARN -- Diversity reduced |

### Quality Thresholds

| Metric | Minimum | Optimal |
|--------|---------|---------|
| Email word count | 100 | 250-500 |
| Formula categories available | 10 | 18 |
| Specimen SLs per category | 3 | 5-10 |
| Key themes extracted | 3 | 5 |

---

## VALIDATION PROTOCOL

```
EXECUTE 0.3-input-validator:

  CRITICAL_CHECKS:
    1. CHECK email_context.body_text
       IF null OR word_count < 100:
         FAIL: "Email body not loaded or too short. Return to E1."

    2. CHECK email_context.arena_selected
       IF false:
         FAIL: "Email is pre-Arena draft. Complete E1 Arena first."

    3. CHECK email_context.metadata.body_type
       IF null OR not in [CT, QO, TM, QA, LB, ST, NR]:
         FAIL: "Email body type not identified. Cannot match formulas."

    4. CHECK formula_library.total_categories
       IF < 10:
         FAIL: "Fewer than 10 formula categories loaded. Cannot ensure diversity."

    5. CHECK email_context.metadata.key_themes
       IF null OR count < 3:
         FAIL: "Fewer than 3 key themes extracted. Re-extract metadata."

  IF any CRITICAL_CHECK fails:
    SET validation_status.passed = false
    SET validation_status.blocker = true
    RETURN validation_status

  HIGH_PRIORITY_CHECKS:
    6. CHECK email_context.metadata.emotional_register
       IF null:
         WARN: "Emotional register not identified. Using default 'confident'."
         SET defaults.emotional_register = "confident"

    7. CHECK email_context.metadata.opening_hook
       IF null:
         WARN: "Opening hook not captured. Manual theme extraction needed."
         SET degraded_mode.opening_hook = true

    8. CHECK email_context.metadata.bridge_summary
       IF null:
         WARN: "Bridge not identified. SL alignment checking will be weaker."
         SET degraded_mode.bridge = true

    9. CHECK formula_library.all_categories_available
       IF false:
         WARN: "Some formula categories missing. Generation diversity reduced."
         SET degraded_mode.formula_diversity = true

  QUALITY_CHECKS:
    10. CHECK email_context.metadata.word_count
        IF < 200:
          WARN: "Short email. SL options may be limited."

    11. CHECK formula_library.total_specimens_loaded
        IF < 50:
          WARN: "Low specimen count. Pattern grounding reduced."

  ASSEMBLE validation_status
  RETURN validation_status
```

---

## GATE CONDITION

**Layer 1 may proceed when:**
- `validation_status.passed` = true
- `validation_status.blocker` = false
- All critical checks passed

**Layer 1 may NOT proceed when:**
- Any critical check failed
- `validation_status.blocker` = true
- Remediation required before continuing

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Email body missing | HALT -- Return to E1 |
| CRITICAL | Email not Arena-selected | HALT -- Complete E1 Arena |
| CRITICAL | Body type not identified | HALT -- Extract body type |
| CRITICAL | Formula library insufficient | HALT -- Load formula library |
| DEGRADED | Metadata incomplete | WARN -- Proceed with reduced quality |
