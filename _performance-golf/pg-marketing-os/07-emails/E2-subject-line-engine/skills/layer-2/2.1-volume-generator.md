# 2.1-volume-generator.md

> **Layer:** 2 (Generation)
> **Skill:** E2-subject-line-engine
> **Purpose:** Generate 20-50 SL candidates across selected formula categories
> **Outputs:** `candidate_pool` object with all generated SLs, formula tags, and preliminary scores

---

## FUNCTION

Generate a high-volume pool of subject line candidates (minimum 20, target 30-50) across the 5-8 formula categories selected in Layer 1. Each candidate must be tagged with its formula category, word count, and a preliminary alignment assessment. Volume generation creates the raw material from which the best candidates emerge.

**CRITICAL:** Formula specimens MUST be loaded and held in active context during generation. Generation without specimens = protocol violation.

---

## SPECIMEN LOADING (MANDATORY BEFORE GENERATION)

```
BEFORE generating any SL candidates:
  1. LOAD specimens for each selected formula category
     FROM: specimens/subject-lines/[category-directory]/
  2. HOLD specimens VERBATIM in active context
  3. DO NOT summarize or paraphrase specimens
  4. USE specimens as statistical attractors -- generate FROM the distribution they create

  IF specimens not loaded:
    HALT -- "Cannot generate without specimen injection"
```

---

## GENERATION PROTOCOL

```
EXECUTE 2.1-volume-generator:
  INPUT: formula_selection (from 1.2), niche_calibration (from 1.3), email_context (from Layer 0)
  OUTPUT: candidate_pool object

  PHASE 1: FORMULA-DRIVEN GENERATION
    FOR each selected formula category (5-8 categories):
      1. LOAD formula structural pattern
      2. LOAD specimens for this category (held in context)
      3. LOAD niche calibration for this category
      4. LOAD email key themes and emotional triggers

      5. GENERATE 4-8 candidates using this formula:
         a. Apply structural pattern to email content
         b. Use specimens as reference for quality/style
         c. Apply niche vocabulary guidance
         d. Target 6-7 words per SL (sweet spot)
         e. Ensure each candidate takes a DIFFERENT angle on the email content

      6. FOR each generated candidate:
         a. TAG with formula code (SL-XXX)
         b. COUNT words
         c. FLAG if > 10 words
         d. PRELIMINARY alignment check (does this SL match the email?)
         e. PRELIMINARY originality check (is this a cliche?)

  PHASE 2: CROSS-FORMULA GENERATION
    7. GENERATE 3-5 additional candidates that BLEND formula elements:
       a. SL-THE + SL-CUR: "The [mystery element]" with curiosity
       b. SL-HOW + SL-EDG: "How to [benefit]" with edge
       c. SL-WHY + SL-PAR: "Why [paradoxical claim]"
       d. Tag these as "BLEND-[code1]-[code2]"

  PHASE 3: WILD CARD GENERATION
    8. GENERATE 2-3 candidates from categories NOT selected in Layer 1:
       a. Pick 2-3 categories that were close to selection threshold
       b. Generate 1 candidate per category
       c. Tag as "WILDCARD-[code]"
       d. These provide diversity insurance

  PHASE 4: QUALITY FILTER
    9. REVIEW all candidates:
       a. REJECT any with banned slop phrases
       b. REJECT any with obvious alignment failure
       c. REJECT any over 12 words (too long to even flag)
       d. FLAG any over 10 words for trimming

    10. COUNT total candidates
        IF < 20: Generate more from highest-fit categories
        IF > 50: Trim lowest-quality candidates to 50

  11. ASSEMBLE candidate_pool object
  12. RETURN candidate_pool
```

---

## GENERATION CONSTRAINTS

### Word Count Targets
- **Sweet spot:** 6-7 words (highest open rates from corpus analysis)
- **Acceptable:** 3-10 words
- **Flagged:** > 10 words (needs trimming or replacement)
- **Auto-reject:** > 12 words (too long for a subject line)

### Per-Category Volume Targets
- **High-fit formulas (score 8-10):** 6-8 candidates each
- **Medium-fit formulas (score 5-7):** 4-6 candidates each
- **Blend candidates:** 3-5 total
- **Wild card candidates:** 2-3 total

### Diversity Requirements
- **No more than 30%** of candidates from a single formula category
- **At least 5 formula categories** represented in the pool
- **At least 2 candidates** from each selected formula category
- **At least 1 blend** and **at least 1 wild card** candidate

## ANTI-SLOP ENFORCEMENT

**Auto-reject any candidate containing:**
- "you won't believe" / "this changes everything" / "mind-blowing"
- "unlock" / "discover the secret" / "transform"
- "amazing" / "incredible" / "revolutionary" / "game-changing"
- "one weird trick" / "doctors hate" / "shocking truth"
- "don't miss out" / "act now" (without real deadline)
- "important message" / "quick update" / "open this"
- Excessive punctuation (!!!, ???)
- ALL CAPS for entire SL
- Emoji

**Replacement required:** Every rejected candidate must be replaced to maintain volume.

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Specimens not loaded | HALT -- Load specimens before generating |
| CRITICAL | Fewer than 20 candidates after filtering | Generate more from highest-fit categories |
| CRITICAL | Fewer than 5 categories represented | Add candidates from unused selected categories |
| DEGRADED | Over 30% from single category | Redistribute generation effort |
| DEGRADED | All candidates flagged for length | Regenerate with word count emphasis |
