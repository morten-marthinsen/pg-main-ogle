## Execution Logic

1. **Locate each upstream file** in the project output directory. Check for existence of all three files.

2. **If any file is MISSING:** HALT immediately. Report which file(s) are absent. LP-07 cannot proceed without all three upstream packages.

3. **Load `page-brief.json`** and extract:
   - `page_type` (type_a | type_b | hybrid)
   - `vertical` (health, finance, info_product, personal_dev, technology, etc.)
   - `awareness_stage` (1-5)
   - `traffic_temperature` (cold | warm | hot)
   - `audience_profile` (primary pain, failed solutions, primary desire, demographics)
   - `voice_direction` (tone, register, anti-voice)
   - `offer_summary` (product name, price point, guarantee)
   - `copywriting_engine_mode` (downstream | standalone)

4. **Load `above-fold-blueprint.json`** and extract:
   - `headline_type_recommendation` (T1-T7 from LP-03)
   - `above_fold_layout` (hero arrangement, image position, CTA placement)
   - `lead_type_suggestion` (if LP-03 provided one)
   - `type_b_elements` (if Type B: product title approach, badge slots, bullet count)

5. **Load `section-sequence.json`** and extract:
   - `total_sections` (count)
   - `section_after_hero` (what follows the hero â€” mechanism, story, proof, etc.)
   - `proof_density_tier` (from LP-04)
   - `estimated_page_length` (short | standard | long)

6. **Cross-validate page type** across all three files. If `page-brief.json` says Type A but `above-fold-blueprint.json` was built for Type B, flag the conflict and HALT.

7. **Write `packages-loaded.md`** with all extracted fields organized by source file.
