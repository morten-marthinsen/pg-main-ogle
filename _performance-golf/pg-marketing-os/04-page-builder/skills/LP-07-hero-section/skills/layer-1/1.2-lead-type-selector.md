# 1.2 — Lead Type Selector (Type A Only)

**Skill:** LP-07-hero-section
**Position:** Layer 1, Microskill 2
**Purpose:** Select the optimal lead type from the 6-type taxonomy for the specific combination of audience awareness, traffic temperature, and offer characteristics. This decision governs the entire lead generation in Layer 2.4.
**Effort Level:** medium
**Dependencies:** `page-brief.json` (awareness stage, traffic temp, audience profile), `headline-type-confirmed.md` (1.1)
**Output:** `lead-type-selected.md`

---

## Input Schema

```yaml
input:
  page_type: "[type_a | hybrid — skip if type_b]"
  awareness_stage: "[1-5]"
  traffic_temperature: "[cold | warm | hot]"
  audience_pain_intensity: "[1-10]"
  audience_skepticism_level: "[low | medium | high | very_high]"

  has_research_discovery: "[Y/N — from brief]"

```

## Lead Type Selection Matrix

Use this matrix to identify the PRIMARY recommendation. Override with specific rationale if needed.

| Awareness | Traffic | Skepticism | Primary Lead Type | Alternative |
|-----------|---------|-----------|------------------|------------|
| 1–2 (Unaware) | Cold | Any | Story Lead (Type 1) | Revelation Lead (Type 4) |
| 2–3 (Problem-Aware) | Cold | High | Agitation Lead (Type 5) | Story Lead (Type 1) |
| 2–3 (Problem-Aware) | Cold | Medium | Probing Question Lead (Type 3) | Agitation Lead (Type 5) |
| 3 (Solution-Aware) | Cold | High | Revelation Lead (Type 4) | Story Lead (Type 1) |
| 3 (Solution-Aware) | Warm | Medium | Promise Lead (Type 2) | Probing Question (Type 3) |
| 4 (Product-Aware) | Warm | Any | Promise Lead (Type 2) | If-Then Lead (Type 6) |
| 4–5 (Product/Most Aware) | Hot | Low | If-Then Lead (Type 6) | Promise Lead (Type 2) |

### Override Rules

**Override to Story Lead (Type 1) when:**
- `has_strong_origin_story = Y` (named person, specific discovery, documented journey)
- `has_research_discovery = Y` (clinical finding, lab result, accidental mechanism discovery)
- Vertical is health/supplements AND audience has tried multiple solutions (high fail history)

**Override to Revelation Lead (Type 4) when:**
- Vertical is financial/investing (insider information angle is native)
- The root cause from the brief is genuinely counterintuitive and surprising
- The mechanism is novel and wouldn't be believed without the "secret reveal" frame

**Override to Probing Question Lead (Type 3) when:**
- Multiple specific yes-answers can be extracted from the pain profile in `page-brief.json`
- The audience is fragmented into sub-segments with different surface symptoms but same root cause

### Headline-Lead Alignment Rules

The selected lead type must be compatible with the headline type:

| Headline Type | Compatible Lead Types |
|--------------|----------------------|
| T1 Direct Benefit | Type 2 (Promise), Type 3 (Question), Type 6 (If-Then) |
| T2 Curiosity/Revelation | Type 1 (Story), Type 4 (Revelation), Type 5 (Agitation) |
| T3 Problem-First | Type 5 (Agitation), Type 3 (Question), Type 1 (Story) |
| T4 Warning/Alert | Type 4 (Revelation), Type 5 (Agitation), Type 1 (Story) |
| T5 Question | Type 3 (Question), Type 5 (Agitation), Type 2 (Promise) |
| T6 How-To/Discovery | Type 1 (Story), Type 4 (Revelation) |
| T7 Number-Specific | Type 2 (Promise), Type 6 (If-Then), Type 3 (Question) |

If the matrix recommendation conflicts with headline-lead alignment → use the alignment rule (headline must be consistent with lead approach).

## Lead Length Determination

Based on awareness stage and page length tier from LP-04:

| Awareness Stage | Short Page | Standard Page | Long Page |
|----------------|-----------|--------------|----------|
| Stage 1–2 (Unaware) | 400–500 words | 600–800 words | 800–1,200 words |
| Stage 3 (Solution-Aware) | 300–400 words | 500–700 words | 700–1,000 words |
| Stage 4–5 (Product/Most Aware) | 200–300 words | 300–500 words | 500–700 words |

## Quality Criteria
- Lead type selected with documented matrix reasoning
- Headline-lead alignment verified
- 3 planned loops identified (not written yet — just planned)
- Target length specified with rationale
- N/A output produced efficiently for Type B pages

## Failure Modes
- Story lead selected but `has_strong_origin_story = N` → Override to alternative type with note
- Matrix is ambiguous (multiple equal options) → Default to Story Lead for Stage 1-2, Promise Lead for Stage 3+, document choice
