# 2.3 — Deck Copy Generator

**Skill:** LP-07-hero-section
**Position:** Layer 2, Microskill 3
**Purpose:** Write 3 deck copy variants for each of the top-2 headline candidates (6 total deck options). The deck MUST add new information that the headline does not contain — a mechanism hint, specificity expansion, audience qualification, or benefit that complements the headline's promise. Deck copy that restates the headline in different words is a protocol violation.
**Effort Level:** high
**Dependencies:** `headline-candidates.md` (2.2 — top 5 scored, top 2 identified), `copy-assets.md` (0.2), `hook-strategy.md` (1.3)
**Output:** `deck-candidates.md`

## Execution Logic

1. **Read each top-2 headline** and explicitly identify what information it CONTAINS:
   - Does it name the mechanism? [Y/N]
   - Does it state a specific benefit? [Y/N]
   - Does it include a number/timeframe? [Y/N]
   - Does it qualify the audience? [Y/N]
   - Does it create a curiosity gap? [Y/N]

2. **For each headline, identify what the deck MUST add** — the deck fills the gaps:

   | Headline Contains | Deck Must Add |
   |------------------|---------------|
   | Benefit but no mechanism | Mechanism hint ("using a [approach/method/discovery]...") |
   | Mechanism but no benefit | Specific benefit expansion (the result this produces) |
   | Curiosity gap but no specificity | Credibility anchor (number, study, authority) |
   | Specificity but no audience qualifier | Audience self-selection ("For [specific people] who [specific situation]") |
   | Number but no emotional hook | Emotional resonance (what the number means to the reader's life) |

3. **Generate 3 deck variants per headline** using different information-adding strategies:

   **Variant A — Mechanism Hint Deck:**
   Introduces or expands on the mechanism. Names it if the headline didn't. Hints at how it works without explaining.

   **Variant B — Specificity Expansion Deck:**
   Adds concrete details the headline couldn't fit — a number, a timeframe, a named study, or a credential.

   **Variant C — Audience Qualifier Deck:**
   Narrows the audience. Creates self-selection. "If you're a [specific descriptor] who's tried [specific things]..."

4. **Apply the repetition test** to each deck variant:
   - Remove the deck. Read only the headline. What do you know?
   - Add the deck back. Read headline + deck together. What NEW information did you gain?
   - If the answer is "nothing new" → FAIL. Rewrite.

5. **Score each deck variant** on three criteria:

   | Criterion | Weight | Description |
   |-----------|--------|-------------|
   | **New Information** | 50% | Does the deck add information the headline doesn't contain? |
   | **Headline Amplification** | 30% | Does the deck make the headline MORE compelling (not just longer)? |
   | **Reading Flow** | 20% | Does headline → deck read as a natural 1-2 sequence? |

6. **Check anti-slop** on all 6 deck variants before output.

## Quality Criteria
- 6 total deck variants generated (3 per headline, 2 headlines)
- Every deck adds demonstrably NEW information verified by repetition test
- All 6 decks checked against anti-slop word list
- Deck length is 1-3 sentences (not a paragraph — decks are subheadlines, not leads)
- Each deck variant uses a different information-adding strategy

## Failure Modes
- Deck restates headline in different words → FAIL repetition test. Rewrite with explicit new-information requirement.
- All 3 variants for one headline add the same type of information → Regenerate with mandatory strategy diversity (mechanism, specificity, audience)
- Deck is longer than 3 sentences → Trim. Decks are bridge copy, not standalone sections.
- Deck contradicts or undermines the headline's promise → Revise to amplify, not redirect. The deck serves the headline.
