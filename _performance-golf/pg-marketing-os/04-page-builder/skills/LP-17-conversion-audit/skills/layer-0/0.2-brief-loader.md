# 0.2 — Brief Loader

> **Skill:** LP-17-conversion-audit
> **Layer:** 0
> **Position:** 2 of 4
> **Purpose:** Load the page brief from LP-00 to establish page type, audience, offer, and vertical context for the audit.
> **Effort Level:** haiku
> **Dependencies:** `page-brief.json` from LP-00 Brief & Page Type Classifier
> **Output:** `brief-load.md`

---

## INPUT

```yaml
input:
  page_brief_path: "[path to page-brief.json from LP-00]"
  project_folder: "[path to project output folder]"
```

---

## EXECUTION

1. Locate `page-brief.json` in the project output folder. Search paths:
   - `./~outputs/[project-name]/landing-page/lp-00-brief-classifier/page-brief.json`
   - `./~outputs/[project-name]/lp-00-brief-classifier/page-brief.json`

2. Parse and extract the following audit-critical fields:
   - **Page type classification:** type_a, type_b, or hybrid (with signals detail)
   - **Vertical:** primary and secondary
   - **Audience awareness stage:** 1-5 (governs headline type validation)
   - **Traffic temperature:** cold, warm, hot (governs proof placement expectations)
   - **Offer summary:** product name, pricing, guarantee, bonuses, urgency levers
   - **Proof inventory:** testimonial count, clinical studies, before/afters, expert endorsements
   - **Voice direction:** reading level target, anti-voice list

3. If `page-brief.json` not found, search for `PAGE-BRIEF-SUMMARY.md` as fallback.

4. Write extracted context to `brief-load.md`.

---

## QUALITY CRITERIA

- Page brief located and parsed
- Page type classification extracted (this determines the entire audit criteria set)
- Audience awareness stage extracted (needed for headline validation)
- Offer summary extracted (needed for guarantee and pricing audit)
- Proof inventory noted (provides baseline for social proof audit)
- File exists after execution

---

## FAILURE MODES

- Page brief not found -> Write `brief-load.md` with status `NOT_FOUND`. Triggers GATE_0 HALT.
- Page type missing from brief -> Flag as critical gap. Cannot proceed without page type.
- Awareness stage missing -> Note as gap; audit can proceed but Point 01 headline validation will be limited.
