> **Output:** `benchmark-load.md`

## INPUT

```yaml
input:
  benchmark_path: "04-page-builder/conversion-data-reference.md"
```

---

## EXECUTION

1. Read `conversion-data-reference.md` from the LandingPageEngine root directory.

2. Extract and structure the following data sets for audit reference:

   **Element-Level Impact Data** (used for expected lift estimates):
   - Personalized CTAs vs generic: +202% (HubSpot)
   - Video on landing pages: +86% (VWO)
   - Long-form vs short-form leads: +220% (KlientBoost)
   - Multiple offers vs single: -266% penalty (KlientBoost)
   - Mobile-responsive vs static: +25.2% mobile CVR (VWO)
   - 1-second load delay: -7% conversions (Google)
   - 5th-7th grade reading vs complex: +56% (VWO)
   - Social proof early placement: Significant (CXL)
   - Guarantee visibility: Significant (Multiple)
   - Personalized headline: +15-30% (Multiple)

   **Industry-Specific Benchmarks** (used for vertical context):
   - Health/Supplements cold traffic: 2-4% (top: 8-15%)
   - Health/Supplements warm: 5-8% (top: 15-25%)
   - Info Products: 3-6% (top: 10-20%)
   - Ecommerce DTC: 2-5% (top: 8-15%)

   **Page Speed Targets:**
   - Type B: <2 seconds
   - Type A: <3 seconds
   - Mobile all types: <2 seconds

   **Social Proof Thresholds:**
   - Star rating display minimum: 10+ reviews (recommended 50+)
   - Review section minimum: 25+ reviews (recommended 100+)
   - Social proof counter: 500+ (recommended 5,000+)
   - Before/after gallery: 5+ (recommended 20+)

   **CTA Placement Data:**
   - Above fold: High conversion impact
   - After social proof block: High
   - After guarantee: Medium-High
   - End of page: Required
   - Sticky mobile: High

   **Guarantee Data:**
   - Branded name: Required
   - Longer guarantee = fewer returns (counterintuitive)
   - Visual treatment: Badge/certificate, not just text

3. Write the structured benchmark reference to `benchmark-load.md`.

## FAILURE MODES

- conversion-data-reference.md not found -> Write `benchmark-load.md` with status `NOT_FOUND`. Audit can still proceed but all lift estimates will state "no benchmark data available" instead of citing specific numbers. This significantly degrades audit quality.
- Data format changed -> Extract what is available, flag gaps.
