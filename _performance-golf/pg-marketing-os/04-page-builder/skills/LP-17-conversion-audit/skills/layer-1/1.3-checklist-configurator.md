# 1.3 — Checklist Configurator

> **Skill:** LP-17-conversion-audit
> **Layer:** 1
> **Position:** 3 of 3
> **Purpose:** Set exact criteria, thresholds, evidence requirements, and expected lift references for each of the 20 audit points. This is the configured instrument the Layer 2 auditors execute against.
> **Effort Level:** sonnet
> **Dependencies:** `audit-scope.md`, `benchmark-load.md`
> **Output:** `configured-checklist.md`

---

## INPUT

```yaml
input:
  audit_scope: "audit-scope.md"
  benchmark_load: "benchmark-load.md"
```

---

## EXECUTION

1. Read the scope plan from `audit-scope.md` (thresholds, special handling, auto-PASS notes).

2. Read the benchmark data from `benchmark-load.md` (lift estimates, sources).

3. For each of the 20 audit points, configure:
   - **Exact criterion text** (what must be true for PASS)
   - **Evidence requirement** (what the auditor must cite from the assembled page)
   - **FAIL fix template** (what category of fix to recommend)
   - **Expected lift reference** (which benchmark data point to cite if FAIL)
   - **Auditor assignment** (which Layer 2 auditor handles this point: 2.1-2.5)

4. Map each point to its benchmark data:

   | Point | Primary Lift Reference | Source |
   |-------|----------------------|--------|
   | 01 | Personalized headline: +15-30% | Multiple |
   | 02 | — (no direct benchmark) | Significant (industry) |
   | 03 | CTA above fold: High impact | Industry |
   | 04 | Social proof early placement: Significant | CXL |
   | 05 | 5th-7th grade reading: +56% | VWO |
   | 06 | (contributes to reading level lift) | VWO |
   | 07 | Single offer vs multiple: -266% penalty avoided | KlientBoost |
   | 08 | — (no direct benchmark) | Significant (industry) |
   | 09 | Social proof early placement: Significant | CXL |
   | 10 | — (volume threshold, not lift) | Industry norms |
   | 11 | — (specificity, not lift) | Industry norms |
   | 12 | — (format, not lift) | Industry norms |
   | 13 | — (no direct benchmark) | Significant (industry) |
   | 14 | Guarantee visibility: Significant | Multiple |
   | 15 | Personalized CTAs vs generic: +202% | HubSpot |
   | 16 | — (anchoring, not lift) | Industry norms |
   | 17 | 1-second delay: -7% | Google |
   | 18 | Mobile-responsive: +25.2% | VWO |
   | 19 | (contributes to speed) | Google |
   | 20 | (contributes to mobile UX) | VWO |

5. Write the fully configured checklist to `configured-checklist.md`.

```markdown
## CATEGORY A: Above-Fold Audit (Auditor: 2.1)

### Point 01: Headline Promise
- **Criterion:** Headline communicates a specific, credible promise. Must name a specific outcome, audience, mechanism, or transformation. Generic "benefit" language = FAIL.
- **Evidence required:** Quote headline verbatim. Identify the specific promise element.
- **Type-specific:** [Type A: must hook cold traffic. Type B: must name product benefit clearly.]
- **If FAIL, fix direction:** Revise headline to include specific outcome/mechanism. Route to LP-07.
- **If FAIL, expected lift:** +15-30% (Multiple sources — personalized/specific headline data)

### Point 02: Hero Image
- **Criterion:** Hero image reflects aspirational outcome OR product in context. Not generic stock photography.
- **Evidence required:** Describe the specified hero image (subject, setting, emotion, format).
- **Type-specific:** [Type A: aspirational/persona image. Type B: product hero in context.]
- **If FAIL, fix direction:** Replace with lifestyle/outcome image (Type A) or product-in-use image (Type B). Route to LP-03.
- **If FAIL, expected lift:** Significant (no specific benchmark — hero image impact is high variance)

### Point 03: CTA Above Fold
- **Criterion:** Primary CTA visible without scrolling on both desktop and mobile.
- **Evidence required:** State CTA text and confirm fold position.
- **Type-specific:** [Type A: CTA visible without scroll. Type B: ATC + price MUST be above fold.]
- **If FAIL, fix direction:** Move CTA above fold. For Type B, add price + ATC to above-fold section. Route to LP-03/LP-06.
- **If FAIL, expected lift:** High (industry consensus — CTA above fold is table stakes)

### Point 04: Trust Signals Above Fold
- **Criterion:** At least one specific trust signal present above or immediately below fold. Each signal must be specific (not "Satisfaction Guaranteed").
- **Evidence required:** List each trust signal with exact text/description.
- **Type-specific:** [Type A: at least 1 specific signal. Type B: rating strip + trust badges required.]
- **If FAIL, fix direction:** Add specific trust signals (media mention, stat, certification, rating). Route to LP-08.
- **If FAIL, expected lift:** Significant (CXL — early trust placement)

---

## CATEGORY B: Copy Quality Audit (Auditor: 2.2)

### Point 05: Reading Level
- **Criterion:** Body copy at 5th-7th grade reading level (Hemingway score <=7). Assessed by scanning for multi-syllable words, complex sentence structures, and jargon.
- **Evidence required:** Assess grade level. Quote any complex passages found.
- **Type-specific:** Universal.
- **If FAIL, fix direction:** Simplify vocabulary, shorten sentences, replace jargon with plain language. Route to relevant writing skill.
- **If FAIL, expected lift:** +56% (VWO — simple copy vs complex)

### Point 06: AI Telltales
- **Criterion:** Zero instances of: revolutionary, game-changing, unlock, transform, harness, empower, journey, comprehensive, elevate, leverage, cutting-edge, groundbreaking, seamless, robust.
- **Evidence required:** Search full page copy. Quote any telltale words found with location.
- **Type-specific:** Universal.
- **If FAIL, fix direction:** Replace each telltale with specific, concrete language. Route to relevant writing skill.
- **If FAIL, expected lift:** +56% (VWO — contributes to natural, simple copy)

### Point 07: Single Conversion Goal
- **Criterion:** All CTAs point to the same action. ONE offer, ONE CTA direction. No competing products, no split paths.
- **Evidence required:** List all CTAs and confirm they lead to the same conversion action.
- **Type-specific:** Universal.
- **If FAIL, fix direction:** Remove competing offers/CTAs. Route to LP-06.
- **If FAIL, expected lift:** Up to +266% improvement by removing multiple offers (KlientBoost)

### Point 08: Benefits Lead, Features Follow
- **Criterion:** Benefits appear before features in the page flow. The first substantive content after the hero section should be benefit-oriented, not a feature dump.
- **Evidence required:** Identify first benefit mention (section + text) and first feature mention (section + text). Confirm order.
- **Type-specific:** Universal.
- **If FAIL, fix direction:** Reorder sections so benefits precede features. Route to LP-04/LP-09.
- **If FAIL, expected lift:** Significant (industry — benefits-first is conversion best practice)

---

## CATEGORY C: Social Proof Audit (Auditor: 2.3)

### Point 09: Proof Early
- **Criterion:** First proof element appears within first 25% of page content.
- **Evidence required:** Calculate approximate page length. Identify first proof element and its position. Express as percentage.
- **Type-specific:** Universal. Cold traffic pages especially critical.
- **If FAIL, fix direction:** Move proof block earlier. Add trust signal or authority anchor to above-fold or Section 2. Route to LP-05.
- **If FAIL, expected lift:** Significant (CXL — early social proof placement)

### Point 10: Proof Volume
- **Criterion:** Proof volume meets minimum for page type.
- **Evidence required:** Count all proof elements. State the count and compare to threshold.
- **Type-specific:** [Type A: 15+ testimonials/proof elements. Type B: 50+ reviews or volume signal (e.g., "83,502 reviews").]
- **If FAIL, fix direction:** Add more proof elements. Source from proof inventory or request from client. Route to LP-10.
- **If FAIL, expected lift:** Significant (industry — proof volume correlates with conversion)

### Point 11: Proof Specificity
- **Criterion:** Proof elements include specific outcomes, numbers, timeframes — not vague praise.
- **Evidence required:** Quote 3+ proof elements. Each must contain a specific result, number, or timeframe.
- **Type-specific:** Universal.
- **If FAIL, fix direction:** Replace vague testimonials with specific-outcome versions. Request specific results from client if not available. Route to LP-10.
- **If FAIL, expected lift:** Significant (industry — specific proof outperforms vague)

### Point 12: Before/After Format
- **Criterion:** Before/after format used where the product category warrants it.
- **Evidence required:** If category warrants B/A: confirm presence and describe format. If category does NOT warrant B/A: state auto-PASS with category justification.
- **Type-specific:** [Categories requiring B/A: supplements, weight loss, skincare, fitness, beauty. Categories auto-PASS: SaaS, info products, finance, coaching, B2B.]
- **If FAIL, fix direction:** Add before/after section with transformation evidence. Route to LP-10.
- **If FAIL, expected lift:** Significant (industry — B/A is highest-converting proof format for physical transformation products)

---

## CATEGORY D: Offer/Conversion Architecture Audit (Auditor: 2.4)

### Point 13: Value Before Price
- **Criterion:** Value is established before price is revealed. The reader should understand what they get and why it matters before seeing the cost.
- **Evidence required:** Identify where value is first established (section) and where price first appears (section). Confirm value precedes price.
- **Type-specific:** [Type A: value stack section before pricing block. Type B: benefits + proof before primary pricing section. For Type B with price above fold, mechanism/benefit content must still precede the main pricing section below fold.]
- **If FAIL, fix direction:** Restructure section sequence to place value/benefit content before pricing. Route to LP-04/LP-11.
- **If FAIL, expected lift:** Significant (industry — value-first is foundational offer architecture)

### Point 14: Guarantee Quality
- **Criterion:** Guarantee is visible, has a branded name (not "money-back guarantee"), has specific duration, and has clear terms.
- **Evidence required:** Quote guarantee text. Confirm: branded name present, duration stated, specifics provided.
- **Type-specific:** Universal.
- **If FAIL, fix direction:** Create branded guarantee with specific name, duration, and easy claim language. Route to LP-06/LP-11.
- **If FAIL, expected lift:** Significant (Multiple — guarantee visibility lifts conversion)

### Point 15: CTA Count
- **Criterion:** 3+ CTA placements on page: early (hero/above-fold), mid (after proof or benefits), pre-close (after guarantee/urgency).
- **Evidence required:** Count all CTA instances. List each by section position.
- **Type-specific:** Universal minimum of 3. Type B with sticky ATC counts as continuous CTA.
- **If FAIL, fix direction:** Add CTA placements at key decision points. Route to LP-06/LP-14.
- **If FAIL, expected lift:** +202% (HubSpot — personalized CTA data; more placements + better copy = highest lift)

### Point 16: Price Anchoring
- **Criterion:** Price is anchored against a reference point: retail/compare price, alternative cost, per-day calculation, or savings display.
- **Evidence required:** Identify the anchoring technique. Quote the anchor price or calculation.
- **Type-specific:** Universal.
- **If FAIL, fix direction:** Add price anchor (crossed-out retail price, per-day calculation, "if purchased separately" comparison). Route to LP-11.
- **If FAIL, expected lift:** Significant (industry — price anchoring is foundational pricing psychology)

---

## CATEGORY E: Mobile/Speed Audit (Auditor: 2.5)

### Point 17: Page Load Target
- **Criterion:** Page is designed to load within target time. Assessed by checking for load-impacting elements.
- **Evidence required:** List load-impacting elements found (unoptimized images, heavy video, excessive scripts). Assess against target.
- **Type-specific:** [Type A: <3 seconds. Type B: <2 seconds.]
- **If FAIL, fix direction:** Optimize images (WebP), remove unnecessary scripts, add lazy loading. Route to LP-16/LP-15.
- **If FAIL, expected lift:** -7% per second of delay (Google)

### Point 18: Sticky ATC Bar
- **Criterion:** Sticky CTA/ATC bar specified for mobile viewing.
- **Evidence required:** Confirm sticky bar is in the page specification.
- **Type-specific:** [Type A: recommended, auto-PASS with note if absent. Type B: required, FAIL if absent.]
- **If FAIL, fix direction:** Add sticky ATC bar to mobile specification. Route to LP-16/LP-15.
- **If FAIL, expected lift:** +25.2% mobile conversions (VWO — mobile-responsive optimization)

### Point 19: Image Optimization
- **Criterion:** All images specified in WebP format with lazy loading directives.
- **Evidence required:** Check image format specifications and lazy loading directives in assembled page.
- **Type-specific:** Universal.
- **If FAIL, fix direction:** Specify WebP format and lazy loading for all images. Route to LP-15.
- **If FAIL, expected lift:** Contributes to speed — -7% per second saved (Google)

### Point 20: CTA Touch Targets
- **Criterion:** CTA buttons specified at minimum 44px touch target for mobile.
- **Evidence required:** Check CTA button size specifications.
- **Type-specific:** Universal.
- **If FAIL, fix direction:** Specify minimum 44x44px touch target for all CTA buttons. Route to LP-15.
- **If FAIL, expected lift:** Contributes to mobile UX — +25.2% mobile conversion improvement (VWO)
```

---

## QUALITY CRITERIA

- All 20 points fully configured with: criterion, evidence requirement, type-specific notes, fix direction, expected lift reference
- Benchmark data mapped to relevant points
- Auto-PASS points identified with justification
- File exists after execution

---

## FAILURE MODES

- Benchmark data unavailable -> Configure all points but use "Significant (no specific benchmark)" for lift estimates. Note reduced audit quality.
- Scope plan incomplete -> Re-read audit-scope.md. If still incomplete, use universal thresholds for all points.
- Trying to configure fewer than 20 points -> All 20 must be configured. No exceptions.
