# 3.2 — Priority Ranker

> **Skill:** LP-17-conversion-audit
> **Layer:** 3
> **Position:** 2 of 3
> **Purpose:** Rank all FAIL items by priority, calculated as (expected conversion lift x ease of implementation). The #1 priority is the fix that delivers the most lift for the least effort.
> **Effort Level:** sonnet
> **Dependencies:** `score-calculation.md`, all 5 Layer 2 audit files
> **Output:** `priority-ranking.md`

---

## INPUT

```yaml
input:
  score_calculation: "score-calculation.md"
  above_fold_audit: "above-fold-audit.md"
  copy_quality_audit: "copy-quality-audit.md"
  social_proof_audit: "social-proof-audit.md"
  offer_conversion_audit: "offer-conversion-audit.md"
  mobile_speed_audit: "mobile-speed-audit.md"
```

---

## EXECUTION

### Step 1: Extract FAIL List
From `score-calculation.md`, extract the complete FAIL list with each item's:
- Point ID
- Category
- Expected lift (from audit files)
- Fix recommendation (from audit files)

### Step 2: Assign Lift Weight
For each FAIL item, assign a lift weight based on the expected conversion impact:

| Lift Category | Lift Weight | Criteria |
|--------------|-------------|----------|
| High | 3 | Specific benchmark data: +56% or greater (e.g., reading level, CTA personalization, single offer) |
| Medium | 2 | Specific benchmark data: +15-55% (e.g., personalized headline, mobile optimization) |
| Low | 1 | "Significant" lift without specific benchmark, or contributing factor (e.g., image optimization contributing to speed) |

**Mapping from conversion-data-reference.md:**
- Point 05 (reading level): +56% -> High (3)
- Point 06 (AI telltales): contributes to +56% -> Medium (2)
- Point 07 (single offer): -266% penalty avoided -> High (3)
- Point 15 (CTA count): +202% -> High (3)
- Point 01 (headline): +15-30% -> Medium (2)
- Point 17 (page load): -7% per second -> Medium (2)
- Point 18 (sticky ATC): +25.2% -> Medium (2)
- Points with "Significant" only: Low (1)

### Step 3: Assign Ease Weight
For each FAIL item, assign an ease of implementation weight:

| Ease Category | Ease Weight | Criteria |
|--------------|-------------|----------|
| Easy | 3 | Copy change only. No structural redesign. Single skill revision. Examples: fix AI telltale words, add price anchor text, rewrite CTA copy. |
| Medium | 2 | Section reorder or addition. May require 2 skills. Examples: add guarantee section, move proof block, add CTA placements. |
| Hard | 1 | Structural redesign, major content creation, or multi-skill revision. Examples: restructure above-fold, create 30+ new testimonials, redesign page architecture. |

### Step 4: Calculate Priority Score
For each FAIL item:
**Priority Score = Lift Weight x Ease Weight**

Maximum possible: 9 (High lift x Easy implementation)
Minimum possible: 1 (Low lift x Hard implementation)

### Step 5: Sort by Priority Score
Sort all FAIL items from highest priority score to lowest. Ties broken by:
1. Higher lift weight wins
2. If still tied, earlier point number wins (above-fold issues more impactful than late-page issues)

### Step 6: Write Priority Ranking
Compile the sorted ranking into `priority-ranking.md`.

---

## QUALITY CRITERIA

- Every FAIL item from score-calculation.md is ranked
- Lift weight assigned based on conversion-data-reference.md data (not guessed)
- Ease weight assigned based on implementation complexity (not arbitrary)
- Priority scores calculated correctly (lift x ease)
- Ranking sorted descending by priority score with tie-breaking rules applied
- Quick wins and major efforts clearly separated
- File exists after execution

---

## FAILURE MODES

- No FAIL items (perfect score) -> Write `priority-ranking.md` noting 20/20 score and zero items to rank. Congratulate and proceed to LP-18.
- Lift data unavailable for a point -> Assign Low (1) lift weight with note "no specific benchmark." Do not guess.
- All items have the same priority score -> Break ties by lift weight first, then point number.
- Ease assessment is subjective -> When uncertain, lean toward Medium (2). Reserve Easy (3) for true copy-only fixes. Reserve Hard (1) for genuine structural overhauls.
