## EXECUTION

**POSITIONAL REINFORCEMENT:** You are evaluating social proof architecture. Score 4 points. Count proof elements precisely. Compare against type-specific thresholds. Do not estimate — count.

### Step 1: Build Proof Inventory
Before scoring any points, build a complete inventory of all proof elements in the assembled page:

1. From `assembled-page-load.md`, catalog every proof element:
   - Customer testimonials (text) — count each individually
   - Customer video testimonials — count each
   - Star ratings / review widgets — note count and average
   - Celebrity/influencer endorsements — list names
   - Expert endorsements — list names and credentials
   - Scientific citations — count
   - Media mentions — list publications
   - Stat claims ("1M+ customers") — list each
   - Trust badges — list each
   - Case studies — count
   - Before/after content — count
   - Performance guarantees — list

2. For each proof element, note its POSITION in the page (section name/number).

3. Calculate total proof element count.

### Step 2: Score Point 09 — Proof Early
Read the criterion from `configured-checklist.md` Point 09.

**Evaluation process:**
1. Calculate approximate total page length (total sections or word count).
2. Calculate the 25% mark (section count / 4 or word count / 4).
3. Identify the FIRST proof element from the inventory.
4. Determine if it falls within the first 25%.

**PASS:** First proof element is in Section 2 of 12 (16.7% — within first 25%).
**FAIL:** First proof element is in Section 6 of 12 (50% — outside first 25%).

**Comparison reference:** From `specimen-reference-load.md`:
- Proof above fold: 4/11 specimen pages
- Proof in first 25%: 7/11 specimen pages

### Step 3: Score Point 10 — Proof Volume
Read the criterion from `configured-checklist.md` Point 10.

**Evaluation process:**
1. Count total proof elements from the inventory built in Step 1.
2. Compare against the type-specific threshold:
   - **Type A:** 15+ testimonials/proof elements
   - **Type B:** 50+ reviews or a volume signal (e.g., "83,502 reviews" displayed)
3. For Type B: a review count badge or widget showing 50+ reviews satisfies the threshold even if individual reviews are not listed inline.

**PASS (Type A):** "23 proof elements: 12 testimonials + 4 expert endorsements + 3 stat claims + 2 media mentions + 2 case studies."
**FAIL (Type B):** "8 reviews displayed, no review count widget. Type B threshold is 50+ reviews."

**Comparison reference:** From `specimen-reference-load.md`:
- LMNT: 83,502 reviews (EXTREME)
- BIOptimizers: 1,561 reviews + 38 citations (EXTREME)
- MAMG: Trustpilot badge + 4 video testimonials (HIGH)

### Step 4: Score Point 11 — Proof Specificity
Read the criterion from `configured-checklist.md` Point 11.

**Evaluation process:**
1. Select 3-5 proof elements from the inventory.
2. For each, check: Does it include a specific outcome, number, timeframe, or measurable result?
3. Count how many proof elements include specifics.

**Specific proof examples:**
- "Lost 23 pounds in 6 weeks" = SPECIFIC (number + timeframe)
- "My cholesterol dropped from 240 to 180" = SPECIFIC (before/after numbers)
- "Saved $4,200 on my first campaign" = SPECIFIC (dollar amount)
- "Dr. Sarah Chen, MD, Harvard Medical School" = SPECIFIC (named authority)

**Vague proof examples:**
- "This product changed my life" = VAGUE (no specifics)
- "I love it so much!" = VAGUE (no outcome)
- "Works great" = VAGUE (no details)
- "A doctor recommends this" = VAGUE (unnamed)

**PASS:** At least 3 proof elements contain specific outcomes/numbers.
**FAIL:** Fewer than 3 proof elements contain specifics — most are vague praise.

### Step 5: Score Point 12 — Before/After Format
Read the criterion from `configured-checklist.md` Point 12.

**Evaluation process:**
1. Check the vertical from `brief-load.md`.
2. Determine if before/after format is warranted:
   - **Required categories:** supplements, weight loss, skincare, fitness, beauty, dental, hair growth — any product with visible physical transformation
   - **Auto-PASS categories:** SaaS, info products, finance, coaching, B2B, personal development, general ecommerce
3. If required: check if the assembled page includes before/after format (images, stories, comparison data).
4. If auto-PASS: note the justification.

**PASS (required category):** "Weight loss supplement. Before/after section present: 8 transformation stories with starting weight, ending weight, and timeframe."
**PASS (auto-PASS):** "SaaS product (ad tracking). Before/after format not applicable to this product category. Auto-PASS."
**FAIL:** "Skincare supplement. No before/after content found despite product category warranting visible transformation evidence."

### Step 6: Write Social Proof Audit
Compile all evaluations into `social-proof-audit.md`.

## QUALITY CRITERIA

- Complete proof inventory built before scoring any points
- All 4 points explicitly scored with evidence
- Point 09 includes exact calculation of page position
- Point 10 includes exact count compared to type-specific threshold
- Point 11 includes 3+ proof elements assessed individually for specificity
- Point 12 includes vertical check and justified auto-PASS or required assessment
- Specimen comparison data included where available

---

## FAILURE MODES

- Proof elements not clearly cataloged in assembled page -> Count what is available. If the assembled page only describes proof sections without listing individual elements, note the limitation and score based on available information.
- Conflating trust signals with proof -> Trust badges are NOT testimonials. Rating widgets count for volume (Point 10) but not for specificity (Point 11) unless individual reviews are quoted.
- Auto-PASSing Point 12 for a category that warrants before/after -> Check the category list carefully. Supplements, weight loss, skincare, fitness, beauty = before/after required.
- Counting authority endorsements as "volume" proof -> Expert endorsements count toward the total but are a different proof TYPE than customer testimonials. For Type A's 15+ threshold, a mix of proof types is expected.
