## EXECUTION

**POSITIONAL REINFORCEMENT:** You are evaluating copy quality across the full page. Score 4 points. Read the actual copy from the assembled page. Do not assume quality — verify it.

### Step 1: Score Point 05 — Reading Level
Read the criterion from `configured-checklist.md` Point 05.

**Evaluation process:**
1. Read the body copy from `assembled-page-load.md` — headlines, deck, lead, body sections, CTA copy.
2. Assess reading level by scanning for:
   - Multi-syllable words (3+ syllables) that could be replaced with simpler alternatives
   - Complex sentence structures (compound-complex sentences, nested clauses)
   - Jargon or technical vocabulary used without plain-language alternatives
   - Passive voice constructions
3. Estimate the Flesch-Kincaid grade level based on sentence length and word complexity.
4. Quote 2-3 specific passages that are either exemplary (simple, clear) or problematic (complex, jargon-heavy).
5. Target: 5th-7th grade. Anything above 8th grade = FAIL.

**PASS example:** "This tiny nutrient fights inflammation at the source. Your joints feel it in days, not weeks." (5th grade)
**FAIL example:** "The proprietary bioavailable formulation leverages synergistic mechanisms to facilitate comprehensive inflammatory response modulation." (14th grade)

### Step 2: Score Point 06 — AI Telltales
Read the criterion from `configured-checklist.md` Point 06.

**Evaluation process:**
1. Search the FULL page copy for every instance of these forbidden words:
   - revolutionary, game-changing, unlock, transform, harness, empower
   - journey, comprehensive, elevate, leverage, cutting-edge, groundbreaking
   - seamless, robust, holistic, synergy, paradigm, optimize
2. Quote every instance found with its location (section name).
3. Count total instances.
4. Zero instances = PASS. Any instance = FAIL.

**PASS:** "Zero telltale words found across [X] sections of body copy."
**FAIL:** "Found 4 instances: 'revolutionary' (hero section), 'transform' (benefits section), 'unlock' (CTA copy), 'comprehensive' (FAQ)."

### Step 3: Score Point 07 — Single Conversion Goal
Read the criterion from `configured-checklist.md` Point 07.

**Evaluation process:**
1. List every CTA on the page from `assembled-page-load.md`.
2. Determine what action each CTA drives:
   - "Add to Cart" -> purchase
   - "Start My Free Trial" -> trial signup
   - "Schedule a Demo" -> demo booking
   - "Learn More" -> secondary page (competing goal)
   - "Subscribe to Newsletter" -> email capture (competing goal)
3. Verify ALL CTAs point to the SAME primary action.
4. Check for competing offers: multiple products, split paths, "also check out" sections.
5. One action = PASS. Any competing action = FAIL.

**PASS:** "All 4 CTAs point to the same action: 'Start My Free Trial' / 'Begin Your Trial' / 'Try Free for 7 Days' / 'Start Trial' — all drive trial signup."
**FAIL:** "3 CTAs drive purchase but 1 CTA in the sidebar drives newsletter signup — competing conversion goals."

### Step 4: Score Point 08 — Benefits Lead, Features Follow
Read the criterion from `configured-checklist.md` Point 08.

**Evaluation process:**
1. Scan the page flow from top to bottom using the section inventory.
2. Identify the FIRST benefit mention — content that describes what the product does for the buyer (outcome, transformation, result).
3. Identify the FIRST feature mention — content that describes what the product IS (ingredients, specs, components, technical details).
4. Confirm benefits appear before features in the page flow.
5. Quote the first benefit and first feature with their section positions.

**PASS:** "First benefit at Section 2 ('Reduce joint pain in as little as 7 days'). First feature at Section 5 ('Contains 500mg Turmeric Extract'). Benefits precede features by 3 sections."
**FAIL:** "First content after hero is an ingredient list (Section 2). Benefits don't appear until Section 4. Features precede benefits."

### Step 5: Write Copy Quality Audit
Compile all four evaluations into `copy-quality-audit.md`.

## QUALITY CRITERIA

- All 4 points explicitly scored PASS or FAIL
- Point 05 includes actual copy passages with grade-level assessments
- Point 06 includes a complete search of all forbidden words with zero/all instances listed
- Point 07 includes a complete CTA inventory with action analysis
- Point 08 includes specific first-benefit and first-feature quotes with section positions
- Every FAIL has specific fix, expected lift with source, and routing

---

## FAILURE MODES

- Copy not extractable from assembled page -> Note that assembled page lacks inline copy (may only contain section descriptions). Score based on available copy; flag that a full copy audit requires actual copy text.
- AI telltale search is incomplete -> Search the FULL forbidden word list. Do not stop at the first 3-4 words. The list has 18+ terms.
- Benefits and features are mixed in the same section -> If the section leads with benefits and then transitions to features within the same section, this counts as PASS. Only FAIL if features appear in a standalone section before any benefit content.
- Inflating reading level assessment -> When in doubt, lean toward FAIL. A passage at 8th grade level is not "close enough" to 7th grade.
