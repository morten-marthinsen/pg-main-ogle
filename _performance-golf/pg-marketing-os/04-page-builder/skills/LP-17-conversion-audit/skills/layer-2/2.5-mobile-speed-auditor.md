> **Skill:** LP-17-conversion-audit
> **Layer:** 2
> **Position:** 5 of 5
> **Purpose:** Score Points 17-20 of the 20-point conversion audit checklist. Evaluate page load target, sticky ATC bar, image optimization, and CTA touch targets.
> **Effort Level:** opus

## EXECUTION

**POSITIONAL REINFORCEMENT:** You are evaluating mobile readiness and speed optimization from the page specification. This is an audit of what the assembled page SPECIFIES, not a live performance test. Score 4 points based on the technical specifications present (or absent) in the assembled page package.

### Step 1: Extract Technical Specifications
From `assembled-page-load.md`, extract all mobile and speed-related specifications:
- Image format directives (WebP, JPEG, PNG?)
- Lazy loading directives
- Video embed specifications (autoplay, preload settings)
- Mobile-specific layout notes
- Sticky CTA/ATC bar specification
- Button size specifications
- Third-party script references (tracking, chat widgets, pop-ups)
- CDN or hosting notes
- Core Web Vitals targets

### Step 2: Score Point 17 — Page Load Target
Read the criterion from `configured-checklist.md` Point 17.

**Evaluation process:**
1. Identify all elements that impact page load time:
   - **Heavy images:** Are images specified in non-optimized formats (PNG, uncompressed JPEG)?
   - **Video embeds:** Auto-playing video without lazy load = significant load impact
   - **Third-party scripts:** Tracking pixels, chat widgets, pop-up tools, social widgets
   - **Large sections:** 23+ sections with heavy media = load risk
2. Assess against target:
   - Type A: <3 seconds
   - Type B: <2 seconds
3. If the assembled page specifies image optimization, lazy loading, and minimal script references -> likely meets target (PASS).
4. If the assembled page has unoptimized images, autoplay video, multiple third-party tools, and no load optimization notes -> likely misses target (FAIL).

**PASS:** "Page specifies WebP images, lazy loading for all below-fold images, and no auto-playing video. 8 sections total. Likely meets Type A <3s target."
**FAIL:** "Page includes 15 full-width images with no format specification, auto-playing hero video, and references to live chat widget + exit intent pop-up. No load optimization notes. Type B target of <2s unlikely to be met."

### Step 3: Score Point 18 — Sticky ATC Bar
Read the criterion from `configured-checklist.md` Point 18.

**Evaluation process:**
1. Check if the assembled page specifies a sticky CTA or ATC bar for mobile.
2. For Type B: this is REQUIRED. Absence = FAIL.
3. For Type A: this is RECOMMENDED. Absence = auto-PASS with note.
4. If present: note what the sticky bar contains (CTA text, price, product name).

**PASS (Type B):** "Sticky ATC bar specified for mobile: 'Add to Cart — $49.99' with product name and price visible."
**PASS (Type A, absent):** "Type A page. Sticky CTA bar not specified. Auto-PASS (recommended, not required for Type A). Note: adding sticky CTA would likely improve mobile conversion."
**FAIL (Type B, absent):** "Type B page. No sticky ATC bar specified for mobile. This is required for Type B pages."

**Reference:** conversion-data-reference.md: Sticky ATC bar is the highest-impact single change for mobile conversion.

### Step 4: Score Point 19 — Image Optimization
Read the criterion from `configured-checklist.md` Point 19.

**Evaluation process:**
1. Check image specifications in the assembled page:
   - Format: WebP specified? (PASS) PNG/JPEG only? (FAIL)
   - Lazy loading: Specified for below-fold images? (PASS) Not mentioned? (FAIL)
   - Compression: Any compression notes? (bonus, not required)
2. If the assembled page has any image references, they should specify optimization.
3. If the page has no images specified -> auto-PASS with note (uncommon).

**PASS:** "All 6 images specified as WebP format with lazy loading for below-fold images."
**FAIL:** "12 images referenced in assembled page with no format specification and no lazy loading directives."

### Step 5: Score Point 20 — CTA Touch Targets
Read the criterion from `configured-checklist.md` Point 20.

**Evaluation process:**
1. Check CTA button specifications in the assembled page:
   - Size: Is a minimum size specified? >= 44px touch target?
   - Mobile-specific: Any mobile-specific button sizing noted?
2. If button size is specified at >= 44px -> PASS.
3. If button size is not specified at all -> FAIL (specification should include mobile touch target requirements).
4. If specified below 44px -> FAIL.

**PASS:** "CTA buttons specified at 48px height minimum with 16px padding. Exceeds 44px touch target."
**FAIL:** "No CTA button size specified in assembled page. Mobile touch targets not addressed."

### Step 6: Write Mobile/Speed Audit
Compile all evaluations into `mobile-speed-audit.md`.

## QUALITY CRITERIA

- All 4 points explicitly scored with evidence
- Technical specifications extracted from assembled page
- Load assessment based on identified load-impacting elements (not assumptions)
- Type-specific criteria applied (Type B sticky ATC required, Type A recommended)
- Every FAIL has specific technical fix recommendations
- No assumptions about live performance — audit is based on specification

---

## FAILURE MODES

- Assembled page lacks technical specifications entirely -> Score Points 17, 19, 20 as FAIL with note: "Assembled page does not include technical specifications for mobile/speed optimization. LP-15 assembly should include these specifications." Point 18 can still be scored based on whether sticky bar is mentioned.
- Conflating specification with live performance -> This audit evaluates what the page SPECIFIES, not measured performance. A page can specify all optimizations and still load slowly due to implementation issues. Note this limitation.
- Auto-PASSing Point 18 for Type B -> Type B pages REQUIRE a sticky ATC bar. This is not optional. Only Type A gets an auto-PASS for this point.
- Assuming modern platforms handle optimization -> Do not assume. If the assembled page does not specify WebP/lazy loading, it fails regardless of what platform might do automatically.
