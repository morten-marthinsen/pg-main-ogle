# 4.2 — Summary Writer

> **Skill:** LP-17-conversion-audit
> **Layer:** 4
> **Position:** 2 of 3
> **Purpose:** Write the human-readable CONVERSION-AUDIT-SUMMARY.md from the conversion-audit.json data. This is the primary document a human reviewer will read to understand the audit results and what needs to be fixed.
> **Effort Level:** haiku
> **Dependencies:** `conversion-audit.json`
> **Output:** `CONVERSION-AUDIT-SUMMARY.md`

---

## INPUT

```yaml
input:
  conversion_audit: "conversion-audit.json"
```

---

## EXECUTION

### Step 1: Read conversion-audit.json
Extract all data needed for the summary.

### Step 2: Write Header
Include project name, verdict, total score, and date.

### Step 3: Write Score by Category
A 5-row table with category name, score, and a 1-sentence summary of that category's performance.

### Step 4: Write Top 3 Priority Fixes
From the priority ranking, extract the top 3 items. For each:
- Fix title (bold)
- Expected lift with source
- 1-sentence description of the fix

### Step 5: Write Full Point-by-Point Results
A 20-row table showing every point with:
- Point number and description
- PASS or FAIL
- 1-line evidence summary (not the full evidence — just enough for the human to understand)

### Step 6: Write Recommended Next Steps
Based on the verdict:

- **LAUNCH_READY (18-20):** "Page is ready for implementation. Proceed to LP-18 A/B Test Plan to identify optimization experiments for post-launch."
- **MINOR_OPTIMIZATION (15-17):** "Page is close to launch-ready. Fix the [X] items above (especially the top 3 priorities) and re-audit. Consider proceeding to LP-18 in parallel."
- **SIGNIFICANT_GAPS (12-14):** "Page has [X] failing points requiring revision before launch. Route fixes to the skills listed in the optimization recommendations. Re-audit after revisions. Do NOT proceed to LP-18."
- **STRUCTURAL_REBUILD (0-11):** "Page has fundamental structural problems ([X] failing points). Return to LP-03 (Above-Fold Architecture) and LP-04 (Section Sequence Planner) to address core issues. A partial fix will not be sufficient."

### Step 7: Write Summary File
Compile into `CONVERSION-AUDIT-SUMMARY.md`.

## QUALITY CRITERIA

- Verdict and score prominently displayed at top
- Category summary table complete with 1-sentence descriptions
- Top 3 priorities clearly stated with expected lifts
- All 20 points in the results table with PASS/FAIL and evidence
- Recommended next steps are specific to the verdict
- File is human-readable (a non-technical reviewer should understand it)
- File exists after execution

---

## FAILURE MODES

- Summary contradicts the JSON data -> Re-read conversion-audit.json. Summary must exactly match the JSON scores and verdict.
- Evidence lines too long or too technical -> Keep each evidence line to 1 sentence, plain language.
- Top 3 priorities missing lift data -> Include whatever lift data is available. If "Significant (no specific benchmark)," state that.
- Recommended next steps too vague -> Must reference specific LP skills and specific actions.
