## TEACHING SOURCES

### 1. Todd Brown — The E5 Method (Mechanism Naming)

**Core Principles:**
- A mechanism must be **Named** — it needs a proprietary label, not a generic descriptor
- A mechanism must be **Unique** — the name itself must signal differentiation
- A mechanism must be **Credible** — the name must feel grounded in science/reality, not invented hype
- A mechanism must be **Demonstrable** — the name should imply observable/testable results
- A mechanism must be **Ownable** — competitors cannot use the same name without referencing you

**Application to Feature Naming:**
- Every feature name is a micro-mechanism — apply E5 at the feature level
- "Dynamic Loft Control" passes E5: Named (yes), Unique (yes), Credible (loft is real), Demonstrable (you can see loft adjustment), Ownable (no one else uses this exact phrase)
- "Advanced Technology" fails E5: Named (generic), Unique (no), Credible (vague), Demonstrable (what do you demonstrate?), Ownable (every competitor says this)
- Test every name against all 5 E5 criteria before finalizing

**Cross-References:** `2.1-caveman-benefit-namer.md` (naming execution), `4.2-name-standalone-tester.md` (validation)

### 2. Eugene Schwartz — Mechanization (Name the Process)

**Core Principles:**
- "Give the benefit a name and a process" — transform the abstract into the concrete
- The prospect needs to SEE the mechanism working — the name should paint a picture
- Sophistication level determines naming approach:
  - Level 1-2 (unaware/problem-aware): Name can be mysterious, curiosity-driven
  - Level 3 (solution-aware): Name must signal HOW it works
  - Level 4-5 (product/most-aware): Name must differentiate from known alternatives
- A named mechanism is 10x more persuasive than an unnamed benefit

**Application to Feature Naming:**
- Feature names ARE mechanization — they name the process by which benefits are delivered
- Match naming approach to market sophistication level from PDP-01 strategy
- At Level 3+, feature names need HOW-IT-WORKS clarity (not just WHAT-IT-DOES)
- Compound names work for mechanization: [Process Word] + [Benefit Word] = "HyperSpeed Face"

**Cross-References:** `1.2-hierarchy-designer.md` (feature ranking), `2.1-caveman-benefit-namer.md`

### 3. Joe Sugarman — Psychological Triggers Applied to Naming

**Core Principles:**
- **Specificity:** Specific names outsell vague names — "Triple-Layer Cushioning" beats "Advanced Comfort"
- **Authority:** Names that reference credible sources/methods carry more weight — "Forged Titanium Frame" implies metallurgical precision
- **Curiosity:** Names that create an information gap earn the next sentence — "BioSync Recovery Matrix" makes you ask "what IS that?"
- **Sense of Urgency:** Names that imply scarcity or timeliness — "Limited Edition Carbon Core" adds urgency to the name itself

**Application to Feature Naming:**
- Every feature name should trigger at least 2 of 4 Sugarman triggers
- Hero feature name should trigger all 4 if possible
- Technical tier names prioritize Authority + Specificity
- Supporting tier names balance Curiosity + Specificity
- Test: "Does this name make you want to know more?" (curiosity) + "Does this name feel real?" (authority)

**Cross-References:** `2.1-caveman-benefit-namer.md`, `1.2-hierarchy-designer.md`

### 4. Claude Hopkins — Preemptive Claims (Name What Everyone Does)

**Core Principles:**
- The first company to NAME a common process OWNS that process in the prospect's mind
- Schlitz didn't invent steam-cleaning bottles — they NAMED it first
- The feature doesn't need to be exclusive — the NAME needs to be exclusive
- Preemptive claims work when the audience doesn't know the process already has a name

**Application to Feature Naming:**
- Audit product capabilities for common-but-unnamed processes
- If competitors use the same technology but haven't named it — NAME IT FIRST
- The act of naming creates perceived ownership
- "Cold-Forged Face" — all clubs are forged, but naming it creates differentiation
- Test: "Do competitors do this too? Have they named it?" If no → preemptive opportunity

**Cross-References:** `1.1-capability-mapper.md` (identify all capabilities), `4.1-generic-name-detector.md` (ensure name isn't already generic)

### 5. Portmanteau / Coined Compound Naming (Specimen-Derived Pattern)

**Core Principles:**
- Fuse a **benefit or scientific root** + a **target or function root** into a single coined word
- The resulting word should be **self-describing** — the reader can decode what it does from the name
- Portmanteaus carry the highest perceived proprietary value because they are literally un-copyable
- This is the dominant naming pattern in premium supplements (FORTIBONE, FORTIGEL, VERISOL, ApresFlex)
- PG already uses this pattern unconsciously: "RollStraight" (Roll + Straight), "AutoGlide" (Auto + Glide), "SliceFix" (Slice + Fix)

**Construction Rules:**
- ONE word, no spaces, no hyphens — "FORTIBONE" not "Forti-Bone"
- CamelCase or AllCaps for readability
- Should pass instant decode test: can a reader guess function from the name alone?
- The benefit root typically comes FIRST (FORTI-bone), but can reverse (TENDO-forte)
- Must be phonetically smooth — say it out loud; if it's awkward, it won't stick

**Application to Feature Naming:**
- Hero features benefit most from portmanteau naming (maximum memorability + ownability)
- Use when the feature is genuinely proprietary and needs to FEEL proprietary
- Audit existing PG compound names (Auto-Glide, Roll Straight, Slice-Fix) — consider whether fusing them into single coined words would increase perceived value
- For supplements: the manufacturer ingredient name (FORTIBONE) often IS the hero — the brand wraps it in a positioning frame (Total Body Formula)

**Cross-References:** `specimens/feature-naming/supplement-naming-specimens.md`, `2.1-caveman-benefit-namer.md`, `4.2-name-standalone-tester.md`

### 6. Feature Name Architecture Patterns

**Proven Naming Structures:**

| Pattern | Structure | Example | Best For |
|---------|-----------|---------|----------|
| Tech + Benefit | [Technology] + [Outcome] | "HyperSpeed Face" | Hero features |
| Ingredient + Purpose | [Ingredient] for [Benefit] | "L-Theanine for Tranquility" | Supplement features |
| Process + Result | [Process Word] + [Result] | "Dynamic Loft Control" | Technology features |
| Material + Form | [Specific Material] + [Form Factor] | "Forged Titanium Frame" | Physical product features |
| Quantity + Component | [Number] + [Component] | "Triple-Layer Cushioning" | Stacking/layering features |
| Proprietary Prefix | [Invented Prefix] + [Function] | "BioSync Recovery" | Premium positioning |
| Action + Target | [Action Verb] + [Target] | "SwingMatch Calibration" | Performance features |
| Portmanteau | [Benefit Root] + [Target Root] → ONE coined word | "FORTIBONE" (Fortify+Bone), "ApresFlex" (After+Flex) | Hero ingredients, proprietary mechanisms |
| Counter-Problem | [Anti-Problem] + [Mechanism] | "Counter-Slice Weighting", "Anti-Chunk Sole" | Problem-solving features |

**Pattern Selection by Tier:**
- Hero features: Tech + Benefit or Proprietary Prefix (maximum memorability)
- Supporting features: Process + Result or Quantity + Component (clarity-focused)
- Technical features: Material + Form or Ingredient + Purpose (credibility-focused)
- NEVER use the same pattern for hero and supporting features — vary structure
