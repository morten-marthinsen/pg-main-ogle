# 0.2-specimen-calibrator.md

> **Layer:** 0 (Foundation)
> **Skill:** PDP-02-feature-naming
> **Purpose:** Load PG feature naming examples (sf2 pattern)
> **Outputs:** `specimen_patterns` object with naming templates and examples

---

## FUNCTION

Load exemplar feature naming patterns from Performance Golf physical products (sf2 pattern) and other high-quality ecom references. These specimens calibrate the naming quality bar and provide structural templates for caveman-benefit naming.

---

## SPECIMEN LOADING PROTOCOL

### Step 1: Load PG Feature Naming Specimens

```
LOCATE specimen sources:
  PRIMARY: PG physical product feature naming patterns
    - 357 driver features
    - CLST putter features
    - SF2 fairway wood features
    - SPD speed system features

EXTRACT naming patterns:
  FOR each product:
    feature_name: [exact name used]
    naming_pattern: [structural pattern]
    benefit_clarity: [how benefit is communicated]
    word_count: [words in name]
    hierarchy_tier: [hero/supporting/technical]
```

### Step 2: Identify Pattern Types

```
CLASSIFY specimens into pattern types:

  Type A: [Branded Mechanism] + [Result Word]
    Example: "HyperSpeed Face"
    Pattern: Brand-coined name implies function

  Type B: [Descriptive] + [Component]
    Example: "Dynamic Loft Control"
    Pattern: Action word + what it controls

  Type C: [Material/Ingredient] + "for" + [Benefit]
    Example: "Forged Titanium for Distance"
    Pattern: Specific material + explicit benefit

  Type D: [Number/Quantity] + [Component] + [Function]
    Example: "Triple-Layer Cushioning System"
    Pattern: Quantified structure + purpose

  Type E: [Benefit-Forward Name]
    Example: "Distance Lock"
    Pattern: Benefit IS the name
```

### Step 3: Build Specimen Index

```
BUILD specimen_patterns:
  total_specimens: [count]
  by_pattern_type: { A: [list], B: [list], C: [list], D: [list], E: [list] }
  by_hierarchy_tier: { hero: [list], supporting: [list], technical: [list] }
  quality_benchmarks:
    best_hero_name: [specimen]
    best_supporting_name: [specimen]
    most_memorable: [specimen]
    clearest_benefit: [specimen]
```

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| DEGRADED | PG specimens not available | Use general ecom naming specimens |
| DEGRADED | Fewer than 5 specimens loaded | Proceed with limited calibration, flag quality risk |
| INFO | Product category different from specimens | Adapt patterns, note category differences |
