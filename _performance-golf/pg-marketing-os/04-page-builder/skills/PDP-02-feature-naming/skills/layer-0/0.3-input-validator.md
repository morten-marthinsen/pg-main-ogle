# 0.3-input-validator.md

> **Layer:** 0 (Foundation)
> **Skill:** PDP-02-feature-naming
> **Purpose:** Validate product capability data present and sufficient
> **Outputs:** `validation_result` object with PASS/FAIL

---

## FUNCTION

Validate that the merged capability list from the upstream loader contains sufficient data to produce meaningful feature names. Minimum threshold: 4 identifiable capabilities with enough detail to name in caveman-benefit format.

---

## VALIDATION PROTOCOL

### Step 1: Capability Count Check

```
COUNT merged_capability_list:
  IF count >= 6: STRONG -- "Good depth for naming"
  IF count 4-5: ADEQUATE -- "Minimum met, may need research"
  IF count < 4: FAIL -- "Insufficient capabilities for feature naming"
```

### Step 2: Capability Quality Assessment

```
FOR each capability:
  ASSESS:
    has_identifiable_component: [Y/N]
      (Can you name WHAT this is? Ingredient, technology, material?)
    has_identifiable_benefit: [Y/N]
      (Can you name WHY this matters? What outcome does it drive?)
    has_differentiator: [Y/N]
      (Is this unique vs. competitors or category-standard?)

  RATE capability_readiness:
    3/3 = READY -- Can be named immediately
    2/3 = NAMEABLE -- Can be named with inference
    1/3 = RESEARCH_NEEDED -- Needs more data before naming
    0/3 = UNUSABLE -- Not a real capability
```

### Step 3: Hero Candidate Validation

```
CHECK strategy_scope.hero_candidates:
  IF at least 1 candidate rated READY or NAMEABLE:
    hero_validation = PASS
  ELSE:
    hero_validation = FAIL
    "No hero feature candidate has sufficient data for naming"
```

### Step 4: Downstream Readiness

```
CALCULATE naming_readiness:
  capabilities_ready: [count rated READY]
  capabilities_nameable: [count rated NAMEABLE]
  capabilities_needing_research: [count rated RESEARCH_NEEDED]

  IF (ready + nameable) >= 4: PASS
  IF (ready + nameable) < 4: FAIL -- "Need more capability data"
```

---

## OUTPUT STRUCTURE

```json
{
  "validation_result": {
    "status": "PASS|FAIL",
    "total_capabilities": 8,
    "ready": 5,
    "nameable": 2,
    "research_needed": 1,
    "hero_validation": "PASS",
    "naming_readiness": "PASS",
    "research_gaps": ["Capability X needs sourcing information"],
    "flags": []
  }
}
```

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Fewer than 4 capabilities | Request additional product data |
| CRITICAL | No hero candidate ready | Request product research for top features |
| DEGRADED | 1-2 capabilities need research | Proceed, flag for human input on those features |
| INFO | All capabilities ready | Best case -- proceed without flags |
