# 1.3-crossover-loader.md

> **Layer:** 1 (Capability Analysis)
> **Skill:** PDP-02-feature-naming
> **Purpose:** Load Skill 15 (Product Introduction) patterns
> **Outputs:** `crossover_patterns` object with naming-relevant patterns from Skill 15

---

## FUNCTION

Load copywriting patterns from Skill 15 (Product Introduction) in the long-form VSL pipeline. Skill 15 handles how products are introduced, named, and positioned in long-form copy -- these patterns directly inform ecom feature naming by providing proven approaches for making features memorable and benefit-clear.

---

## LOADING PROTOCOL

### Step 1: Locate Skill 15 Resources

```
LOCATE: 02-long-form-vsl/15-product-intro/
  LOAD:
    - Product naming patterns
    - Feature introduction sequences
    - Ingredient/technology presentation frameworks
    - Benefit-linkage patterns (feature -> benefit -> proof)
```

### Step 2: Extract Naming-Relevant Patterns

```
FROM Skill 15, EXTRACT patterns relevant to feature naming:

  NAMING PATTERNS:
    - How products are named (branded names, descriptive names, hybrid)
    - How individual features are introduced within product sections
    - How mechanism names become feature names
    - Word economy patterns (how few words convey maximum meaning)

  BENEFIT-LINKAGE PATTERNS:
    - Feature -> Benefit linkage structures
    - "This means..." bridges
    - Proof anchoring per feature (what evidence supports the claim)

  HIERARCHY PATTERNS:
    - How Skill 15 orders feature importance
    - Lead feature vs. supporting feature presentation
    - Technical credibility sequencing
```

### Step 3: Adapt for E-Commerce

```
FOR each loaded pattern:
  ADAPT for ecom context:
    long_form_version: [original pattern]
    ecom_adaptation: [scan-optimized version]
    adaptation_notes: [what changes for ecom]

  KEY ADAPTATIONS:
    - Long-form narrative -> Ecom micro-script (1-2 sentences)
    - Extended benefit argument -> Single benefit statement
    - Multi-paragraph proof -> Single proof point reference
    - Sequential introduction -> Visual hierarchy (bold name + micro-script)
```
