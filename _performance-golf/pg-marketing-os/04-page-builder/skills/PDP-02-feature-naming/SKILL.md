# PDP-02 — Feature Naming & Scoping

**Pipeline Position:** Second skill in E-Commerce Copy Engine pipeline. Executes after PDP-01 (Strategist). Feeds PDP-03 (Hero & Value Prop), PDP-07 (Section Copy), PDP-08 (Micro-Scripts).

---

## PURPOSE

Deep research into product capabilities, then name every feature using the caveman-benefit format: "[Ingredient/Technology/Component] for [Plain-Language Benefit]." Prioritize features into a hierarchy (hero feature, supporting features, technical features) and write micro-script descriptions per feature.

**This is THE critical skill of the E-Commerce Engine.** Feature naming is 30% of an ecom page. A great feature name does more work than a paragraph of description.

**Success Criteria:**
- All product features identified and named using caveman-benefit format
- Feature hierarchy established (hero, supporting, technical)
- Micro-script (1-2 sentences) written per feature
- No generic names ("Advanced Technology," "Premium Materials")
- Feature names tested: does the name alone communicate what this does and why I should care?
- Feature package structured for consumption by PDP-03, PDP-07, PDP-08
