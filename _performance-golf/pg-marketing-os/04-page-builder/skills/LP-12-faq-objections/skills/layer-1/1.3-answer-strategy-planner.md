# 1.3 — Answer Strategy Planner

**Skill:** LP-12-faq-objections
**Position:** Layer 1, Microskill 3
**Purpose:** Assign a specific answer strategy to each FAQ item before any answers are written. This prevents the common failure mode of every answer using the same approach (all direct, all reframe, all proof-based). Different objections require different resolution techniques. The strategy plan ensures each answer is purpose-built for the objection it addresses.
**Effort Level:** medium
**Dependencies:** `objection-classification.md` (1.1), `sequence-plan.md` (1.2), `packages-loaded.md` (0.1), `specimens-loaded.md` (0.3)
**Output:** `answer-strategies.md`

## Execution Logic

1. **Load the 4 answer strategy types:**

   | Strategy | When to Use | Answer Structure |
   |----------|------------|-----------------|
   | **Proof-Based** | When data, studies, review counts, or specific credentials are available AND the objection requires evidence to resolve | Direct answer + specific proof/data + confidence close |
   | **Reframe** | When the question reveals a misconception or when the hidden objection is different from the surface question | Reframe the question + address the real concern + new perspective |
   | **Direct** | When the question has a simple factual answer and the hidden objection is straightforward | Direct answer + supporting detail + confidence builder |
   | **Social Proof** | When the best answer is other people's experience (review data, testimonial reference, "X customers have...") | Direct answer + social proof data + confidence close |

2. **For each objection in the locked sequence,** assign a strategy:

   **Strategy selection logic:**

   | Objection Category | Default Strategy | Override Conditions |
   |--------------------|-----------------|---------------------|
   | Price/Value | **Reframe** — reframe from "expensive" to "cost per result" or "cost vs alternatives" | If strong proof available (comparison data, per-unit math), use **Proof-Based** |
   | Efficacy/Results | **Proof-Based** — lead with clinical data, review data, or specific results | If no proof available, use **Social Proof** (review count, customer results). NEVER use Direct alone for efficacy. |
   | Trust/Guarantee | **Direct** — state the guarantee details clearly and specifically | If high trust deficit (new brand, no reviews), use **Social Proof** |
   | Timing | **Reframe** — reframe from "not yet" to "cost of waiting" or "now is uniquely right" | -- |
   | Comparison | **Proof-Based** — use specific differentiators, comparison data, ingredient quality | If no comparison data available, use **Reframe** (why comparison is wrong frame) |
   | Technical/Logistics | **Direct** — straightforward factual answer | -- |

3. **Plan the proof elements** for each Proof-Based and Social Proof answer:
   - What specific data point will anchor this answer?
   - Where does this data come from? (upstream packages, product data, research)
   - If no proof is available for a Proof-Based answer, switch strategy to Reframe or Social Proof

4. **Plan the reframe angle** for each Reframe answer:
   - What is the misconception the question reveals?
   - What is the correct frame?
   - How do you shift the reader's perspective in 1 sentence?

5. **Verify strategy diversity:**
   - All 4 strategies should be represented across 8-12 items
   - No more than 3 consecutive answers using the same strategy (monotonous)
   - At least 3 answers use Proof-Based or Social Proof (proof density requirement)

6. **Plan the voice calibration** for all answers:
   - Read voice direction from packages-loaded.md
   - Type A: conversational, direct, confident — like a trusted advisor answering in person
   - Type B: clean, scan-friendly, matter-of-fact — like a well-designed help page
   - All answers must match the page's established voice register

7. **Write `answer-strategies.md`** with complete strategy assignments.

## Quality Criteria
- Every item assigned exactly 1 strategy with rationale
- Efficacy/results objections NEVER use "Direct" strategy alone — must include proof or social proof
- Price/value objections use Reframe or Proof-Based (not just "Direct: it costs $X")
- At least 3 answers planned with proof/data elements
- Strategy diversity across all 4 types
- Hidden objection explicitly referenced in strategy plan (the answer is built to address IT, not just the surface question)

## Failure Modes
- Every answer assigned "Direct" strategy -> Monotonous and ineffective. Hard objections need proof or reframe. Vary the strategies.
- Efficacy objection planned with "Direct" only -> "Yes, it works" is not a sufficient answer. Proof or social proof is required for efficacy claims.
- Price objection answered with "Direct" strategy -> "It costs $97" does not resolve the price objection. Reframe the value (cost per serving, cost vs alternative, value of result) or prove the value (comparison data, ROI data).
- No proof elements planned for any answer -> At least 3 answers must include proof. If upstream data is insufficient, note the gap and use the strongest available proof (review count, guarantee, authority credentials).
- All answers planned at exactly 2 sentences -> Some questions need 3-4 sentences. Vary the length based on complexity.
