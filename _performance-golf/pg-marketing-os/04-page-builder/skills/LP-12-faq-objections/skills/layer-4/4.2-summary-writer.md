# 4.2 — Summary Writer

**Skill:** LP-12-faq-objections
**Position:** Layer 4, Microskill 2
**Purpose:** Write the FAQ-SUMMARY.md file — a human-readable summary of the complete FAQ section, its structure, coverage, audit results, and downstream handoffs. This file is for human review — it should be immediately understandable without reading the JSON package or execution log.
**Effort Level:** low
**Dependencies:** `faq-package.json` (4.1), `faq-audit.md` (3.1), `coverage-audit.md` (3.2), `sequence-plan.md` (1.2)
**Output:** `FAQ-SUMMARY.md`

---

## Input Schema

```yaml
input:
  faq_package: "[from faq-package.json -- all fields]"
  faq_audit: "[from faq-audit.md -- score + per-point results]"
  coverage_audit: "[from coverage-audit.md -- category distribution]"
  sequence_plan: "[from sequence-plan.md -- strategy + rationale]"
```

---

## Execution Logic

1. **Extract the key information** from the faq-package.json:
   - Product name
   - Page type
   - FAQ header (headline + subhead)
   - All FAQ items in sequence order
   - Coverage statistics
   - Sequencing strategy
   - Audit results

2. **Build the summary** following the required template from LP-12-AGENT.md. Every section must be populated. The summary should be readable in under 2 minutes.

3. **Present all FAQ items** in sequence order with a compact table format that shows: position, question, category, strategy, proof presence, and sentence count.

4. **Highlight key decisions:**
   - Why this sequencing strategy was chosen
   - What the hardest objection is and where it is positioned
   - Which answers include proof/data

5. **Include audit results** with the actual score and any revision notes.

6. **Include downstream handoff information** so the human knows what happens next.

## Quality Criteria
- All sections populated (no empty sections)
- FAQ items presented in sequence order (not alphabetical, not by category)
- Full Q&A copy included for human review (the human should not need to read the JSON)
- Audit results accurately reflect the actual audit scores
- NEEDS_HUMAN_INPUT items clearly flagged so the human knows what requires their attention
- Summary readable in under 2 minutes

## Failure Modes
- Omitting the full Q&A copy section -> The human needs to read the actual copy to approve it. A table of metadata is not sufficient.
- Items not in sequence order -> The sequence IS the strategy. Presenting items out of order misrepresents the FAQ section.
- Inflating audit results -> Report the actual scores. If the FAQ scored 7.5, write 7.5 — not "strong performance."
- Missing NEEDS_HUMAN_INPUT flags -> If any answer was written with placeholder data (technical or logistics items where product data was unavailable), the human MUST know this.
