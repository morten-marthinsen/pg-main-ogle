# 4.1 — FAQ Package Compiler

**Skill:** LP-12-faq-objections
**Position:** Layer 4, Microskill 1
**Purpose:** Assemble all validated FAQ copy into a single structured JSON package for downstream skills. LP-15 Assembly and LP-17 Conversion Audit will load this file. Every field must be populated — this is the single source of truth for the FAQ section.
**Effort Level:** low
**Dependencies:** All Layer 3 validation outputs (`faq-audit.md`, `coverage-audit.md`), all Layer 2 output files (`faq-header.md`, `core-objection-answers.md`, `technical-faq-answers.md`, `logistics-faq-answers.md`), `sequence-plan.md`
**Output:** `faq-package.json`

---

## Input Schema

```yaml
input:
  faq_header: "[from faq-header.md -- selected headline + subhead]"
  core_answers: "[from core-objection-answers.md -- all core Q&A pairs]"
  technical_answers: "[from technical-faq-answers.md -- all technical Q&A pairs]"
  logistics_answers: "[from logistics-faq-answers.md -- all logistics Q&A pairs]"
  sequence_plan: "[from sequence-plan.md -- locked positions]"
  faq_audit: "[from faq-audit.md -- score + PASS confirmation]"
  coverage_audit: "[from coverage-audit.md -- PASS confirmation]"
  page_type: "[from packages-loaded.md]"
  product_name: "[from packages-loaded.md]"
```

## Pre-Assembly Verification

Before assembling the package, verify:

```yaml
ASSEMBLY-GATE:
  faq_audit_score: "[X.X -- must be >= 7.5]"
  faq_audit_result: "[PASS | FAIL -- must be PASS]"
  coverage_audit_result: "[PASS | FAIL -- must be PASS]"
  total_faq_items: "[count -- must be 8-12]"
  mandatory_categories_present:
    price_value: "[Y/N -- must be Y]"
    efficacy_results: "[Y/N -- must be Y]"
    trust_guarantee: "[Y/N -- must be Y]"
  all_gates_pass: "[Y/N]"

  IF all_gates_pass = N: HALT -- do not assemble. Return to failed validation.
```

---

## Assembly Execution

1. **Merge all FAQ items** from 2.2, 2.3, and 2.4 into a single ordered list using the locked sequence from sequence-plan.md.

2. **For each FAQ item, compile:**
   - Position (sequence number)
   - Question text (exact)
   - Answer text (exact)
   - Category
   - Hidden objection
   - Answer strategy used
   - Whether proof was included
   - Word count of answer
   - Sentence count of answer

3. **Compile coverage summary:**
   - Total items
   - Categories covered (list)
   - Mandatory categories present (all 3 must be true)
   - Total categories (must be >= 4)

4. **Compile sequencing metadata:**
   - Strategy used (hardest_first or hardest_last)
   - Rationale
   - Hardest objection identified
   - Position of hardest objection

5. **Compile validation block:**
   - FAQ audit score
   - Coverage audit result
   - Sycophantic opener count (must be 0)
   - AI telltale count (must be 0)
   - All gates passed (must be true)

6. **Compile downstream handoffs:**
   - LP-15 Assembly: complete FAQ section copy in sequence order
   - LP-17 Conversion Audit: objection category coverage map

7. **Write the JSON package** following the schema defined in LP-12-AGENT.md.

## Quality Criteria
- JSON is valid and parseable
- All FAQ items present with all required fields (no nulls in required fields)
- Items are in the locked sequence order (not alphabetical, not random)
- Validation block accurately reflects the audit results
- Coverage block accurately reflects the coverage audit
- Downstream handoff descriptions are clear and actionable

## Failure Modes
- JSON syntax error -> Validate JSON before writing. Missing commas, unescaped quotes, and trailing commas are the most common issues.
- Items out of sequence order -> Double-check against sequence-plan.md. The position field must match the locked sequence.
- Missing fields in any FAQ item -> Every item needs all fields: position, question, answer, category, hidden_objection, answer_strategy, proof_included, word_count, sentence_count.
- Validation block shows PASS but audit actually showed FAIL -> Cross-reference the actual audit files. Do not fabricate passing results.
