# 2.1 — FAQ Header Writer

**Skill:** LP-12-faq-objections
**Position:** Layer 2, Microskill 1
**Purpose:** Write the FAQ section headline and optional subhead. The header sets the tone for the entire FAQ section — it signals whether this is a corporate afterthought or a genuine effort to address the reader's concerns. Type A headers are conversational and address skepticism. Type B headers are clean and scan-friendly.
**Effort Level:** low
**Dependencies:** `packages-loaded.md` (0.1), `sequence-plan.md` (1.2), `answer-strategies.md` (1.3)
**Output:** `faq-header.md`

---

## Input Schema

```yaml
input:
  page_type: "[type_a | type_b | hybrid]"
  product_name: "[from packages-loaded.md]"
  voice_direction: "[from answer-strategies.md voice calibration]"
  hardest_objection: "[from sequence-plan.md -- what the reader is most concerned about]"
  section_context: "[what section precedes the FAQ]"
```

---

## Execution Logic

1. **Determine header approach based on page type:**

   **Type A (Long-Form Sales Page):**
   The reader has scrolled through a long page. They are invested but have remaining doubts. The FAQ header should acknowledge this directly — speak to their skepticism without being defensive.

   Generate 3 header options:
   - Option A: Acknowledgment frame — "We know you still have questions" angle
   - Option B: Confidence frame — "Here's what [X] people asked before they [result]" angle
   - Option C: Direct frame — "Honest answers" or "Before you decide" angle

   **Type B (Ecomm/PDP):**
   The reader is scanning the product page. The FAQ section is a utility section. Keep the header clean and functional.

   Generate 3 header options:
   - Option A: Product-specific — "Common Questions About [Product Name]"
   - Option B: Utility — "Questions? Answers." or "Everything You Need to Know"
   - Option C: Simple — "FAQ" (acceptable if the page design provides visual hierarchy)

   **Hybrid:**
   Use Type B header style (the FAQ section in a hybrid page lives in the Type B zone).

2. **Write optional subhead** (recommended for Type A, optional for Type B):
   - The subhead should hint at what the FAQ covers without giving away the answers
   - Example: "Including the ones about price, guarantee, and whether this actually works."
   - The subhead can reference the guarantee or social proof to build pre-FAQ confidence

3. **Verify header quality:**
   - Does NOT use "Frequently Asked Questions" as the only option for Type A (too generic)
   - Does NOT use sycophantic language ("We love answering your questions!")
   - Matches the page voice (conversational for Type A, clean for Type B)
   - Does NOT contain AI telltales

4. **Select the recommended option** with rationale, but present all 3 for human review.

5. **Write `faq-header.md`** with all options and recommendation.

## Quality Criteria
- 3 distinct options generated (not minor variations of the same phrase)
- Type A options are conversational and acknowledge the reader's skepticism
- Type B options are clean and functional
- At least 1 option includes a subhead (even if the recommendation is "no subhead")
- No sycophantic language, no AI telltales
- Recommendation includes clear rationale

## Failure Modes
- Only generating "Frequently Asked Questions" or "FAQ" for Type A -> Too generic. Type A requires a header that earns readership. Generate at least 2 non-generic options.
- Header that is longer than 12 words -> Keep it concise. A FAQ header is not a headline — it is a section label with personality.
- Subhead that gives away the answers -> The subhead hints at coverage without resolving anything. "Including the ones about price" is fine. "Yes, it works and here's the guarantee" is not.
- Header that promises something the FAQ does not deliver -> If the header says "Everything you need to know," the FAQ must actually cover comprehensive ground.
