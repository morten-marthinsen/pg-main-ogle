# 2.1-section-copy-generator.md

> **Layer:** 2 (Copy Generation)
> **Skill:** PDP-07-section-copy
> **Purpose:** Generate copy per section (iterative)
> **Outputs:** `section_copies` object with copy for all active BTF sections

---

## FUNCTION

Generate copy for every active BTF section, iterating through the section map in priority order. Each section follows its assigned template from the routing map, respects its word budget, and is written as a standalone unit.

---

## GENERATION PROTOCOL

### Step 1: Iterate Through Sections

```
FOR each section in routing_map (ordered by priority):

  LOAD:
    template: assigned template from 1.1
    word_budget: from PDP-01 strategy
    proof_plan: assigned proof from 1.2
    design_plan: assigned design notes from 1.3
    crossover_pattern: if available
    applicable_features: from upstream context

  GENERATE section copy following template structure
```

### Step 2: Write Per Template

```
TEMPLATE EXECUTION:

  T1 (Problem-Solution):
    1. Problem statement: 2-3 sentences naming the specific problem
    2. Pain bullets: 3-5 bullet points agitating the pain
    3. Product answer: 2-3 sentences introducing the product as solution
    FORMAT: Scannable, bolded key phrases, short paragraphs

  T2 (How-It-Works):
    1. Feature name introduction: Bold name + 1-sentence setup
    2. Steps: 3-5 numbered steps explaining the mechanism
    3. Proof point: 1 supporting evidence element
    FORMAT: Numbered list, icon-ready, one sentence per step

  T3 (Feature Showcase — C.P.B. Framework):
    FOR each applicable feature:
      FRONT OF CARD:
        1. Feature name (bold): Exact PDP-02 name
        2. Micro-script: Exact micro-script from PDP-02 (italicized)
      BACK OF CARD (C.P.B. order — do NOT label these in the copy):
        1. CLAIM: Bold statement of what the feature does (NOT the micro-script — a direct claim)
        2. PROOF: Technical WHY — the spec, engineering, or mechanism that makes the claim true
        3. BENEFIT: Layered benefits in this order:
           - Functional benefit (what it does mechanically)
           - Dimensionalized benefit (what that means for the golfer's game)
           - Emotional benefit (how it feels)
      CRITICAL RULES:
        - Do NOT label "Claim:" "Proof:" "Benefit:" in the output copy — just write in C.P.B. order
        - Do NOT jump from claim to description. PROVE the claim first, then layer benefits.
        - Benefits must be REAL benefits to the customer. "Extreme forward weighting impossible with single-material construction" is NOT a benefit. "You feel the precision the moment you pick it up" IS a benefit.
        - "Visible proof" is NOT a benefit. State what it DOES for the golfer.
        - Bullets always start with the benefit, not the spec
    FORMAT: Flip cards — front/back. Card or grid layout, scannable

  T4 (Proof Block):
    1. Proof element(s): Reviews, quotes, stats, awards
    2. Context: Why this proof matters
    3. Credibility anchor: Source attribution
    FORMAT: Pull-quote style, visual emphasis on proof

  T5 (Offer/Value):
    1. Value statement: Why this is worth the price
    2. Comparison/anchor: Price relative to alternatives or value
    3. CTA or guarantee: Action text or risk reversal
    FORMAT: Price card, comparison table, or badge layout

  T6 (Informational):
    1. Question/context: What the prospect needs to know
    2. Answer/steps: Clear, concise answer
    3. Supporting detail: Additional context if within budget
    FORMAT: Accordion (FAQ), numbered steps, or tabbed content

  T7 (Comparison):
    1. Comparison setup: What's being compared
    2. Table: Feature-by-feature comparison
    3. Differentiator callout: What makes this product win
    FORMAT: Responsive table with bolded advantages
```

### Step 3: Section-Level Quality Check

```
FOR each generated section:
  VERIFY:
    [ ] Word count within budget (+/- 10%)
    [ ] Follows assigned template structure
    [ ] Contains no references to other sections
    [ ] Feature names match PDP-02 exactly
    [ ] Scan-optimized (bolded key phrases, short paragraphs, bullets)
    [ ] First sentence establishes context (no assumed prior reading)
    [ ] No slop words present

  IF any check fails: REVISE before moving to next section
```

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Section exceeds word budget by >10% | Compress -- remove lowest-value sentences |
| CRITICAL | Section references another section | Rewrite with built-in context |
| WARNING | Feature name paraphrased | Correct to exact PDP-02 name |
| WARNING | Section lacks scan formatting | Add bold key phrases, shorten paragraphs |
