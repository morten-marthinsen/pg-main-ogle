# 2.2-proof-embedding.md

> **Layer:** 2 (Copy Generation)
> **Skill:** PDP-07-section-copy
> **Purpose:** Embed proof elements into section copy
> **Outputs:** Updated `section_copies` with embedded proof

---

## FUNCTION

Embed the planned proof elements from 1.2 into each section's copy. Proof must feel integrated, not bolted on -- it should flow naturally within the section's copy structure while maintaining visual distinctiveness for scanning.

---

## EMBEDDING PROTOCOL

### Step 1: Load Proof Assignments

```
FROM proof_plan (1.2 output):
  FOR each section:
    proof_type: [customer_story|statistic|expert|review|ugc|award]
    proof_content: [specific proof text or placeholder]
    placement: [where in section copy the proof belongs]
```

### Step 2: Embed Proof per Section

```
FOR each section:
  BASED ON proof_type:

  CUSTOMER STORY:
    FORMAT: Indented quote block with attribution
    PLACEMENT: After pain point, before product answer
    EXAMPLE:
      > "I tried everything for my back pain. Nothing worked
      > until I found [Product]." -- Mike R., verified buyer

  STATISTIC:
    FORMAT: Bold callout number + context sentence
    PLACEMENT: After claim, as evidence
    EXAMPLE:
      **93%** of users reported improvement within 30 days
      (based on 2,847 customer survey responses)

  EXPERT ENDORSEMENT:
    FORMAT: Credential + quote block
    PLACEMENT: After mechanism/technology explanation
    EXAMPLE:
      Dr. James Miller, Sports Biomechanist:
      > "[Product feature] addresses the #1 cause of
      > distance loss in amateur golfers."

  REVIEW QUOTE:
    FORMAT: Star rating + pull-quote + attribution
    PLACEMENT: Standalone or after benefit claim
    EXAMPLE:
      ★★★★★ "Best club I've ever owned. Added 20 yards
      to my drive in the first round." -- Tom K.

  UGC REFERENCE:
    FORMAT: Image caption reference
    PLACEMENT: With visual element note
    EXAMPLE:
      [UGC Image: Customer on course with product]
      "My Sunday foursome can't believe the difference."

  AWARD/CERTIFICATION:
    FORMAT: Badge reference + citation
    PLACEMENT: End of section or near credibility claim
    EXAMPLE:
      [Award Badge] Golf Digest Hot List 2026
```

### Step 3: Verify Integration Quality

```
FOR each section with embedded proof:
  CHECK:
    [ ] Proof feels natural, not forced
    [ ] Proof supports the section's specific claim
    [ ] Proof is visually distinct (quote block, bold number, badge)
    [ ] Proof has attribution (not anonymous)
    [ ] Proof doesn't disrupt scan flow

  IF proof feels forced:
    REPOSITION within section or adjust framing
```

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Section has no proof after embedding | Add fallback proof (customer count, general stat) |
| WARNING | Proof feels forced/bolted on | Rewrite section to naturally integrate |
| WARNING | Proof is anonymous ("one customer said") | Add name/attribution |
| INFO | Placeholder proof noted | Flag for client to provide real proof |
