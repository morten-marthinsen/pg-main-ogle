# 2.6 — Comparison Chart Writer

**Skill:** PDP-07-btf-section-writer
**Position:** Layer 2, Microskill 6
**Purpose:** Write the Comparison Chart section — a 3-column table (Product vs Competitor 1 vs Competitor 2) with 5-7 rows that collapses the purchase decision into an obvious verdict. Uses primacy effect (strongest advantage first) and recency effect (clincher advantage last). NEVER names real competitor brands — uses category terms only.
**Effort Level:** high
**Dependencies:** `per-section-briefs.md` (1.1 — comparison_chart brief), `data-validation.md` (1.2 — GREEN or YELLOW), `pdp-reference-loaded.md` (0.3 — Section 5.9 template)
**Output:** `section-comparison-chart.md`

---

## Input Schema

```yaml
input:
  section_brief: "[comparison_chart brief from per-section-briefs.md]"
  word_count_target: "[from PDP-04]"
  proof_density: "[from PDP-04 — medium: checkmarks as implicit proof]"
  pdp_reference: "[Section 5.9 template from pdp-reference-loaded.md]"
  product_differentiators: "[from brief — 5-7 key advantages]"

  product_name: "[from brief]"
```

## Execution Logic

**Step 1:** Read the Comparison Chart template from PDP-BEST-PRACTICES-REFERENCE.md Section 5.9. Key pattern: 3-column table, checkmarks for product, Xs for competitors, primacy/recency effect.

**Step 2:** Select column headers:
- Column 1: Feature/Attribute label
- Column 2: Product name (highlighted column — colored background)
- Column 3: "Leading Brand" or "Typical Alternative" (NEVER a real brand name)
- Optional Column 4: Second generic competitor (only if market is highly commoditized)

**Step 3:** Select 5-7 comparison rows. Order using primacy/recency effect:

```
ROW ORDERING STRATEGY:
  Row 1 (PRIMACY): Strongest differentiator — the "this alone justifies choosing us" advantage
  Row 2: Strong advantage
  Row 3-4: Standard advantages (important but not unique)
  Row 5-6: Supporting advantages
  Row 7 (RECENCY): Clincher — the final advantage that tips the decision
```

**Step 4:** For each row, write:
- **Feature Label:** Clear, specific attribute (not vague)
  - Good: "No Added Sugar," "Third-Party Tested," "Free Shipping Over $50"
  - Bad: "Quality," "Innovation," "Customer Service"
- **Product Column:** Checkmark + optional brief qualifier (e.g., "0g" or "NSF Certified")
- **Competitor Column:** X mark + optional brief qualifier (e.g., "3-6g" or "Not Tested")

**Step 5:** Write section header that frames the comparison:
- Not: "How We Compare" (generic)
- Better: "Why [Product Name] vs. the Rest" or "[Product Name] vs. Standard Alternatives"

**Step 6:** Add a brief intro line above the chart (1 sentence setting context):
- Example: "Here's how [Product Name] stacks up against the typical alternatives in [category]."

**Step 7:** Verify comparison chart ethics:
- Zero real competitor brand names
- Claims are defensible (not misleading comparisons)
- Competitor column is fair (represents actual category weaknesses, not strawmen)

**Step 8:** Verify word count against target (+/-10%).

**Step 9:** Write `section-comparison-chart.md`.

## Quality Criteria
- 5-7 rows (not fewer, not more than 7)
- Primacy effect: Row 1 = strongest advantage
- Recency effect: Last row = clincher advantage
- Zero real competitor brand names (category terms only)
- Feature labels are specific and clear
- Product column highlighted visually
- Intro line provides context without marketing fluff
- Word count within +/-10% of PDP-04 target

## Failure Modes
- Real competitor brand name used -> HALT. Replace with "Leading Brand" or "Typical Alternative."
- Fewer than 5 rows -> HALT. Minimum 5 comparison points needed. Escalate as data gap if brief lacks differentiators.
- All rows favor product with no nuance -> Review for strawman bias. At least 1-2 rows should show parity or slight qualifier.
- Feature labels are vague ("Quality," "Value") -> Rewrite with specifics ("Third-Party Tested," "Price Per Serving")
- Strongest advantage buried in middle -> Reorder: Row 1 = strongest (primacy), last row = clincher (recency)
- Table has 4+ competitor columns -> Simplify to 3 columns maximum (1 product + 1-2 competitors). Too many columns = noise.
