# 4.1 — BTF Sections Compiler

**Skill:** PDP-07-btf-section-writer
**Position:** Layer 4, Microskill 1
**Purpose:** Assemble all Layer 2 section output files into a single `pdp-btf-sections.json` package that LP-15 (Page Assembly) will consume. This JSON file contains the complete copy for every BTF section PDP-07 wrote, along with metadata (word counts, proof elements, visual directions, validation results).
**Effort Level:** medium
**Dependencies:** All Layer 2 section files, `quality-audit.md` (3.1), `independence-check.md` (3.2), `proof-density-check.md` (3.3), `section-sequence-loaded.md` (0.2)
**Output:** `pdp-btf-sections.json`

---

## Input Schema

```yaml
input:
  section_files: "[list of all section-*.md files from Layer 2]"
  quality_audit: "[path to quality-audit.md]"
  independence_check: "[path to independence-check.md]"
  proof_density_check: "[path to proof-density-check.md]"
  section_sequence: "[path to section-sequence-loaded.md — for ordering + targets]"
  product_name: "[from brief]"
```

---

## Execution Logic

**Step 1:** Load the PDP-07 execution queue from `section-sequence-loaded.md` for correct ordering.

**Step 2:** Load all Layer 2 section output files.

**Step 3:** Load Layer 3 validation results from `quality-audit.md`, `independence-check.md`, and `proof-density-check.md`.

**Step 4:** Verify all sections that should have been written exist:
- Cross-check Layer 2 output files against PDP-07 execution queue
- If any queued section has no output file -> HALT. Return to Layer 2.

**Step 5:** Verify all Layer 3 validations passed:
- All sections PASS quality audit (or have been revised)
- All sections INDEPENDENT (or have been fixed)
- All sections MATCH proof density (or have been addressed)
- If any FAIL remains -> HALT. Return to Layer 3 for revision.

**Step 8:** Write `pdp-btf-sections.json`.

## Output Schema

See JSON structure in Step 6 above.

---

## Quality Criteria
- JSON is well-formed (valid JSON syntax)
- All sections from PDP-07 execution queue are present
- Each section has: copy, word count (target + actual), proof elements, visual direction, validation results
- Sections ordered by PDP-04 sequence position
- Sections not written by PDP-07 explicitly listed with reason and handler
- Validation summary reflects Layer 3 results
- Overall status accurately reflects completion state

## Failure Modes
- Missing section output file -> HALT. Cannot compile incomplete package. Return to Layer 2.
- Layer 3 validation failures unresolved -> HALT. Cannot compile package with known failures. Return to Layer 3.
- JSON syntax error -> Fix and re-write. Validate JSON before saving.
- Word count field missing for any section -> Calculate from copy text. Never leave blank.
- Validation summary says COMPLETE but sections are missing -> Fix summary. Data integrity is critical.
