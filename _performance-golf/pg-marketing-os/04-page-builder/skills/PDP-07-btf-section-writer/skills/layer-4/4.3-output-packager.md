# 4.3-output-packager.md

> **Layer:** 4 (Validation & Packaging)
> **Skill:** PDP-07-section-copy
> **Purpose:** Package section-copy-package.json + section-copy-assembled.md
> **Outputs:** Two files in project output directory

---

## FUNCTION

Package the validated section copy into two output files: `section-copy-package.json` (structured data for downstream processing) and `section-copy-assembled.md` (human-readable markdown for review and editorial).

---

## PACKAGING PROTOCOL

### Step 1: Verify All Validations Passed
```
CHECK:
  standalone_test_result: MUST be PASS
  consistency_result: MUST be PASS
  arena_selection: MUST have human selection

IF any fails: HALT
```

### Step 2: Build section-copy-package.json
```
ASSEMBLE JSON with:
  header: version, timestamp, skill_id
  sections: [array of all section copy with metadata]
    per section: number, name, priority, copy, word_count, proof, design_notes
  validation: standalone_status, consistency_status
  downstream_handoffs: ec_05_assembly, ec_06_editorial
```

### Step 3: Build section-copy-assembled.md
```
ASSEMBLE markdown document:
  # Section Copy -- [Product Name]

  FOR each section (in priority order):
    ## [Section Name] (Priority [N])
    _Word count: X | Template: [T-type] | Proof: [type]_

    [Full section copy]

    **Design Notes:**
    - Layout: [layout]
    - Mobile: [behavior]
    - Visuals: [elements]

    ---
```

### Step 4: Write and Verify
```
WRITE both files to ~outputs/[project-code]/e-comm/
VERIFY:
  [ ] Both files written
  [ ] JSON is valid and parseable
  [ ] Markdown is properly formatted
  [ ] All sections present in both files
  [ ] Feature names consistent
  [ ] No placeholder text
```

---

## OUTPUT STRUCTURE

```json
{
  "packaging_result": {
    "status": "complete",
    "files_written": ["section-copy-package.json", "section-copy-assembled.md"],
    "total_sections": 13,
    "total_word_count": 2870,
    "all_validations_passed": true,
    "packaged_at": "ISO timestamp"
  }
}
```

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Validation not passed | Return to validators, fix issues |
| CRITICAL | File write fails | Check directory and permissions |
| WARNING | Markdown formatting issues | Fix formatting before finalizing |
