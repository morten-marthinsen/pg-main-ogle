# 0.2-crossover-pattern-loader.md

> **Layer:** 0 (Foundation)
> **Skill:** PDP-07-section-copy
> **Purpose:** Load long-form patterns (Skills 13,14,15,16,18)
> **Outputs:** `crossover_patterns` object with ecom-adapted patterns

---

## FUNCTION

Load copywriting patterns from long-form VSL skills (13-Root Cause, 14-Mechanism, 15-Product Intro, 16-Offer, 18-Proof) and adapt them for ecom scan-optimized format. These patterns provide proven copy structures that are condensed for ecom sections.

---

## LOADING PROTOCOL

### Step 1: Load Patterns by Skill
```
FOR each crossover skill in PDP-01 crossover_map:

  SKILL 13 (Root Cause) -> Section 9 (Problem/Solution):
    - Enemy identification: Condense to 1-2 sentences
    - Pain agitation: Convert to 3-5 scan-friendly bullets
    - Root cause reframe: Single paragraph, not narrative arc
    - Ecom adaptation: Problem -> Product Answer format

  SKILL 14 (Mechanism) -> Section 10 (Why It Works):
    - Mechanism introduction: Feature name + 2-3 sentence explanation
    - How-it-works: 3-5 step visual sequence
    - Science backing: Single proof point, not extended argument
    - Ecom adaptation: Icon + short copy per step

  SKILL 15 (Product Intro) -> Sections 4, 7, 13:
    - Feature presentation: Name -> Benefit -> Proof triplet
    - Ingredient introduction: Ingredient + dosage + function
    - How-to-use: Numbered steps, 1 sentence each
    - Ecom adaptation: Card/grid layout with icons

  SKILL 16 (Offer) -> Sections 3, 12, 16, 17:
    - Value stack: Simplified item list with values
    - Price anchoring: "Comparable products cost X. Yours: Y."
    - Guarantee: Single clear statement, not paragraph
    - Ecom adaptation: Price comparison table or badge format

  SKILL 18 (Proof) -> Sections 5, 6, 14, 19:
    - Review integration: Pull-quote with star rating
    - UGC referencing: Image caption format
    - Expert citation: Credential + single endorsement line
    - Statistical proof: Single stat with visual callout
    - Ecom adaptation: Card carousel or grid format
```

### Step 2: Build Pattern Index
```
INDEX patterns by section_number for quick lookup during generation
EACH pattern entry includes:
  long_form_source: [skill + pattern name]
  ecom_adaptation: [how to condense]
  structure_template: [copy structure to follow]
  word_budget_guidance: [how many words this pattern typically needs]
```
