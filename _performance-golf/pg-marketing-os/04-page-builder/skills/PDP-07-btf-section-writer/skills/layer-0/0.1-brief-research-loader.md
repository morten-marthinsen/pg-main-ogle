# 0.1 — Brief + Research Loader

**Skill:** PDP-07-btf-section-writer
**Position:** Layer 0, Microskill 1
**Purpose:** Load LP-00 page-brief.json, LP-01 conversion intelligence, and any available research quotes. Extract product data, audience profile, proof inventory, and audience language that downstream section writers will use.
**Effort Level:** low
**Dependencies:** LP-00 (page-brief.json), LP-01 (conversion intelligence output)
**Output:** `input-verification.md`

---

## Input Schema

```yaml
input:
  page_brief_path: "[path to page-brief.json — LP-00 output]"
  lp01_intelligence_path: "[path to LP-01 conversion intelligence output, or 'not available']"
  research_quotes_path: "[path to research quotes file, or 'not available']"
```

---

## Execution Logic

**Step 1:** Locate and load `page-brief.json`. Extract:
- Product name and description
- Product type (supplement, DTC consumer good, etc.)
- Page type (must be `type_b` or `hybrid`)
- Audience profile (demographics, psychographics, awareness level)
- Offer details (pricing, bundles, subscription options)
- Ingredient list (if supplement/formulated product)
- Proof inventory (studies, credentials, certifications, testimonials)
- Competitor context (market category, positioning)

**Step 2:** If LP-01 intelligence is available, load and extract:
- Traffic temperature (cold/warm/hot)
- Conversion benchmarks for vertical
- Top visitor objections identified
- Key conversion drivers identified

**Step 3:** If research quotes file is available, load and extract:
- Audience voice quotes (pain language, desire language)
- Objection quotes
- Competitor perception quotes
- Sort by relevance to BTF section types

**Step 4:** Confirm page type is `type_b` or `hybrid`. If `type_a`, HALT with error: "PDP-07 requires type_b or hybrid page type. Use LP-07 through LP-14 for type_a."

**Step 5:** Write `input-verification.md` summarizing all loaded data and flagging any missing inputs.

---

## Quality Criteria
- page-brief.json successfully loaded and parsed
- Page type confirmed as type_b or hybrid
- Product name and type extracted
- Ingredient list extracted with count (if applicable)
- Proof inventory extracted with count
- Data gaps explicitly listed (not hidden)

## Failure Modes
- page-brief.json not found -> HALT. LP-00 must run first.
- Page type is type_a -> HALT. Use LP-07 through LP-14 instead.
- page-brief.json exists but is empty or malformed -> HALT. Flag for LP-00 re-run.
- LP-01 intelligence not available -> WARN (proceed with brief data only, note reduced context)
- Research quotes not available -> WARN (proceed without audience voice, note gap in section briefs)
