# 0.3 — PDP Reference Loader

**Skill:** PDP-07-btf-section-writer
**Position:** Layer 0, Microskill 3
**Purpose:** Load Section 5 (BTF Section Templates) from PDP-BEST-PRACTICES-REFERENCE.md. This provides the structural templates, best practices, and specimen references for every BTF section type PDP-07 will write.
**Effort Level:** low
**Dependencies:** PDP-BEST-PRACTICES-REFERENCE.md (reference file)
**Output:** `pdp-reference-loaded.md`

---

## Input Schema

```yaml
input:
  reference_file_path: "[path to PDP-BEST-PRACTICES-REFERENCE.md]"
  sections_to_load: "[list of section_ids from 0.2 execution queue]"
```

---

## Execution Logic

**Step 1:** Load PDP-BEST-PRACTICES-REFERENCE.md. Navigate to Section 5 (BTF Section Templates).

**Step 2:** For each section_id in PDP-07's execution queue, extract the corresponding template from Section 5:
- **What it shows** — the visual/structural pattern
- **Why it exists** — the persuasion function
- **Best practices** — the specific rules for that section type
- **Specimens** — reference URLs for visual/structural reference

**Step 3:** For each section, identify the unique structural format required:
- Product Highlights: expandable tiles (question + answer)
- The Expert: credential strip + video CTA
- What's in It: visual ingredient cards, vertical expanded
- What to Expect: vertical timeline with milestone circles
- The Problem: agitation text (may consolidate with Solution/Logic)
- The Solution: paradigm shift pattern (3 parts)
- Why It Works: 4-step mechanism diagram
- Comparison Chart: 3-column table with checks/Xs
- Cost Savings: pricing table with "You Save $X" punchline
- How It's Made: sourcing/manufacturing layout with cert badges
- How to Use: 3-step numbered instructions with icons
- Identity Matching: 3-column persona grid
- The Proof: study circles, data visualizations

**Step 4:** Extract any Baymard/NN-g findings from Section 6 that apply to BTF section layout (vertical stacking vs carousels, accordion patterns, snackable content rules).

**Step 5:** Write `pdp-reference-loaded.md` with per-section templates and structural requirements.

## Quality Criteria
- PDP-BEST-PRACTICES-REFERENCE.md loaded successfully
- Section 5 templates extracted for every section in PDP-07's execution queue
- Each template includes structural format, best practices, and specimens
- Baymard/NN-g layout rules captured for section writers
- Structural format for each section type is distinct (not generic)

## Failure Modes
- PDP-BEST-PRACTICES-REFERENCE.md not found -> HALT. Reference file must exist.
- Section 5 missing or empty -> HALT. Reference file is incomplete.
- Section in execution queue has no template in Section 5 -> WARN. Write from principles, flag gap.
- Template extracted but missing specimens -> WARN. Proceed without specimen reference, note in output.
