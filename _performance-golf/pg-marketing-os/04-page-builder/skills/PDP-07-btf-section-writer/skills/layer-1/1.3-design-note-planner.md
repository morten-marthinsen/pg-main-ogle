# 1.3-design-note-planner.md

> **Layer:** 1 (Section Architecture)
> **Skill:** PDP-07-section-copy
> **Purpose:** Plan UX/design notes per section
> **Outputs:** `design_plan` object with layout/visual specs per section

---

## FUNCTION

Plan the UX/design notes that accompany every section's copy. On ecom pages, copy is inseparable from layout -- the page builder needs layout direction, mobile behavior, and visual element specifications for every section.

---

## DESIGN PLANNING PROTOCOL

### Step 1: Assign Layout per Section Type

```
SECTION TYPE -> DEFAULT LAYOUT:

  Problem/Solution (9): Two-column (problem left, solution right) or stacked
  Why It Works (10): Icon + text steps (3-5 step visual)
  Comparison (11): Comparison table (responsive)
  Cost Savings (12): Calculator or value comparison cards
  How to Use (13): Numbered steps with icons/images
  Reviews (14): Review card carousel or grid
  FAQ (15): Accordion (collapsible Q&A)
  Offer/Pricing (16): Price card with value stack
  Guarantee (17): Badge + text block (centered)
  Use Cases (18): Tab or card layout (one per segment)
  Product Highlights (4): Icon grid or card row
  UGC (5): Image carousel with caption overlay
  Expert (6): Photo + bio + quote block
  Ingredients (7): List or grid with dosage callouts
  Timeline (8): Timeline graphic (vertical or horizontal)
  CTA Box (3): Full-width banner with button
  Awards/Press (19): Logo bar + citation text
```

### Step 2: Specify Mobile Behavior

```
FOR each section:
  ASSIGN mobile_behavior:
    "stack_vertically": Two-column becomes single column
    "carousel_swipe": Grid becomes swipeable carousel
    "collapsible": Content collapses under header (accordion)
    "full_width": Stretches to full mobile width
    "hidden_secondary": Secondary elements hidden behind "show more"
```

### Step 3: Specify Visual Elements

```
FOR each section:
  DEFINE visual_elements:
    images: [product shots, lifestyle images, before/after]
    icons: [feature icons, step numbers, checkmarks]
    charts: [comparison tables, pricing tables, timelines]
    badges: [trust badges, award logos, certification marks]
    video: [if micro-script from PDP-08 maps here]
```

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| WARNING | Layout not assigned to section | Use "stacked" as default |
| WARNING | Mobile behavior not assigned | Use "stack_vertically" as default |
| INFO | All sections have complete design notes | Proceed |
