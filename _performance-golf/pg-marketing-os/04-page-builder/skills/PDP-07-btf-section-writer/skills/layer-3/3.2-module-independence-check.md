# 3.2 — Module Independence Check

**Skill:** PDP-07-btf-section-writer
**Position:** Layer 3, Microskill 2
**Purpose:** Verify that every section written in Layer 2 works as a standalone module with zero cross-section dependencies. PDP visitors hunt-and-peck — they jump directly to sections they care about. If any section contains "as mentioned above," "see the ingredients section," or any reference to another section, it breaks the module independence model and must be revised.
**Effort Level:** medium
**Dependencies:** All Layer 2 section output files (`section-*.md`)
**Output:** `independence-check.md`

---

## Input Schema

```yaml
input:
  section_files: "[list of all section-*.md files from Layer 2]"
```

---

## Execution Logic

**Step 1:** Load all Layer 2 section output files.

**Step 2:** For each section, scan for cross-section references. Forbidden patterns:

```
CROSS-REFERENCE DETECTION PATTERNS:
  Explicit references:
    - "as mentioned above"
    - "as mentioned in the [section name] section"
    - "see the [section name] section"
    - "building on what we discussed"
    - "as we covered earlier"
    - "refer to the [section] for details"
    - "the [section] above explains"

  Implicit dependencies:
    - Assumes knowledge of ingredient names without brief context
      (OK in Ingredients section; NOT OK in Timeline section without re-introduction)
    - Assumes the problem was already established
      (OK if Problem section; NOT OK if Solution section starts with "This is the answer")
    - References "the comparison above" or "as shown in the chart"

  Structural dependencies:
    - Section opening that only makes sense after reading the previous section
    - Section conclusion that teases the next section ("Up next, see how it works...")
    - Section body that fills in gaps left by another section
```

**Step 3:** For each section, run the Standalone Comprehension Test:

```
STANDALONE COMPREHENSION TEST:
  Question: If a visitor scrolled directly to THIS section (skipping everything above),
  would they:
  1. Understand what this section is about? [Y/N]
  2. Get value from the content without prior context? [Y/N]
  3. Know what product is being discussed? [Y/N]
  4. Encounter no confusing references to content they haven't read? [Y/N]

  PASS: All 4 = Y
  FAIL: Any = N — flag specific dependency
```

**Step 4:** For each failure, prescribe the fix:
- Explicit reference -> Remove and add brief inline context (1 sentence max)
- Implicit dependency -> Add standalone context within the section
- Structural dependency -> Rewrite opening/conclusion to be self-contained

**Step 5:** Classify each section:
- **INDEPENDENT:** Zero cross-references, passes standalone test
- **MINOR FIX:** 1-2 removable references, quick edit
- **REWRITE NEEDED:** Structural dependency that requires section rewrite

**Step 6:** Write `independence-check.md`.

## Quality Criteria
- Every Layer 2 section checked for cross-references
- Both explicit and implicit dependencies detected
- Standalone Comprehension Test applied to each section
- Specific fix instructions provided for each failure
- No section is allowed to "tease" or "set up" another section

## Failure Modes
- Problem/Solution/Logic sections have inherent dependencies (Problem sets up Solution) -> In consolidated mode, this is acceptable within the combined section. In separate mode, each MUST be standalone — Solution must briefly restate the problem context.
- Ingredients section references "the mechanism explained below" -> Remove reference. If mechanism context is needed, add 1 sentence inline.
- Timeline section uses ingredient names without context -> Add brief "X for Y" identifier inline (e.g., "Ashwagandha, the stress-relief ingredient in [Product]").
- Multiple sections have the same dependency pattern -> Systemic issue in Layer 2 generation. All affected sections need revision.
