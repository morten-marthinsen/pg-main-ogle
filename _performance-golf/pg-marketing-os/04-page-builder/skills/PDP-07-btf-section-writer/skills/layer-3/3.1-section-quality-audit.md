# 3.1 — Section Quality Audit

**Skill:** PDP-07-btf-section-writer
**Position:** Layer 3, Microskill 1
**Purpose:** Audit every section written in Layer 2 against three quality dimensions: (1) 5-second scannability — can you get the point of the section at a glance; (2) anti-slop — zero generic AI language; (3) reading level — PDP copy should be Grade 6-8 reading level. Sections that fail any dimension are flagged for revision before Layer 4 assembly.
**Effort Level:** high
**Dependencies:** All Layer 2 section output files (`section-*.md`)
**Output:** `quality-audit.md`

---

## Input Schema

```yaml
input:
  section_files: "[list of all section-*.md files from Layer 2]"
  word_count_targets: "[per-section targets from section-sequence-loaded.md]"
```

---

## Execution Logic

**Step 1:** Load all Layer 2 section output files.

**Step 2:** For each section, run the 5-Second Scannability Test:

```
5-SECOND SCANNABILITY TEST:
  1. Can you identify the section's topic from the header alone? [Y/N]
  2. Can you get the key takeaway from headers + bold text without reading body? [Y/N]
  3. Are bullets/lists used instead of paragraphs? [Y/N]
  4. Is any single paragraph longer than 3 sentences? [Y/N — should be N]
  5. Does the section use its type-specific structure? [Y/N]
     - Tiles for Highlights, timeline for What to Expect, table for Comparison, etc.

  PASS: All 5 criteria met
  FAIL: Any criterion not met — flag for revision
```

**Step 3:** For each section, run the Anti-Slop Check:

```
ANTI-SLOP FORBIDDEN WORDS:
  - revolutionary, game-changing, transform, transformative
  - unlock, harness, leverage, cutting-edge
  - synergy, synergize, synergistic
  - groundbreaking, breakthrough (unless citing actual breakthrough study)
  - empower, empowering, empowerment
  - holistic (unless product is explicitly holistic)
  - journey (when used for product experience)
  - elevate, elevating
  - seamless, seamlessly
  - world-class, best-in-class
  - robust, comprehensive (when used as filler)

  PASS: Zero forbidden words found
  FAIL: Any forbidden word found — flag specific words and locations
```

**Step 4:** For each section, estimate reading level:
- Target: Grade 6-8 (Flesch-Kincaid)
- Check for: multi-syllable jargon without explanation, complex sentence structures, academic language
- Supplements: technical ingredient names are acceptable; mechanism language should be simplified

**Step 5:** For each section, verify word count:
- Target from PDP-04 via section-sequence-loaded.md
- Actual word count
- Within +/-10%: PASS
- Outside +/-10%: FAIL — flag over/under

**Step 6:** Score each section:
- PASS: All 4 checks pass (scannability, anti-slop, reading level, word count)
- REVISE: 1-2 checks fail (fixable issues)
- REWRITE: 3+ checks fail (structural problems)

**Step 7:** Write `quality-audit.md`.

## Quality Criteria
- Every Layer 2 section audited (none skipped)
- Scannability test applied per section type (not generic)
- Anti-slop check catches all forbidden words
- Reading level estimated and flagged if above Grade 8
- Word count verified against PDP-04 targets
- Specific revision instructions provided for each failure

## Failure Modes
- Layer 2 section files missing -> HALT. Cannot audit what doesn't exist. Return to Layer 2.
- All sections fail scannability -> Systemic issue — Layer 2 used paragraph format instead of structured format. All sections need rewrite.
- Multiple sections contain same forbidden word -> Systemic issue — pattern in generation. Flag and provide replacement language.
- Word counts systematically over target -> LLM over-writing pattern. Provide specific cut instructions per section.
