# 3.3 — Proof Density Verification

**Skill:** PDP-07-btf-section-writer
**Position:** Layer 3, Microskill 3
**Purpose:** Verify that every section meets the proof density target set by PDP-04. Each section has a specified proof level (none / light / medium / heavy / maximum) with specific proof types and counts. This check ensures sections that should contain proof actually do, and sections specified as "none" don't have proof shoehorned in.
**Effort Level:** medium
**Dependencies:** All Layer 2 section output files (`section-*.md`), `section-sequence-loaded.md` (0.2 — proof density specs from PDP-04)
**Output:** `proof-density-check.md`

---

## Input Schema

```yaml
input:
  section_files: "[list of all section-*.md files from Layer 2]"
  proof_density_specs: "[per-section proof density from section-sequence-loaded.md]"
```

---

## Execution Logic

**Step 1:** Load proof density specifications from `section-sequence-loaded.md` (sourced from PDP-04's pdp-section-sequence.json).

**Step 2:** Load all Layer 2 section output files.

**Step 3:** For each section, check actual proof against PDP-04 specification:

```
PROOF DENSITY LEVELS:

  NONE: Section should have zero proof elements (instructional or math sections)
    - how_to_use: instructional only
    - cost_savings: math, not proof
    - Check: No study references, no testimonials, no authority badges in copy

  LIGHT: Minimal proof signals (badges, icons, brief references)
    - product_highlights: badge/icon proof only
    - what_to_expect: timeline icons, mechanism references
    - the_solution: 1 differentiator claim supported
    - identity_matching: persona-matched testimonial snippets
    - Check: 1-3 proof signals present (badges, brief references)

  MEDIUM: Substantive proof elements
    - whats_in_it: 1 study or "clinically studied" badge per key ingredient
    - why_it_works: mechanism diagram + 1-2 study references
    - comparison_chart: checkmarks as implicit proof
    - how_its_made: certification badges, sourcing photos
    - Check: 2-5 specific proof elements (studies, certifications, badges)

  HEAVY: Proof-dense sections
    - the_expert: credential strip, institution logos, video
    - Check: 3+ proof elements (credentials, logos, video direction, publications)

  MAXIMUM: Section IS proof
    - the_proof: study circles, survey data, data visualizations
    - Check: Every claim backed by specific data (journal, year, sample size)
```

**Step 4:** For each section, classify actual proof density:
- Count proof elements (studies cited, badges mentioned, credentials listed, data points, testimonial snippets)
- Classify as none / light / medium / heavy / maximum based on count and specificity

**Step 5:** Compare actual vs specification:
- **MATCH:** Actual density matches or exceeds specification
- **UNDER:** Actual density below specification — missing proof elements
- **OVER:** Actual density significantly above specification for a "none" or "light" section (proof shoehorned in)

**Step 6:** For UNDER matches, identify specific missing elements:
- "Ingredients section missing study references for 3 of 6 ingredients"
- "Expert section missing institution name"
- "Proof section has only 1 data point, specification calls for 2+"

**Step 7:** Write `proof-density-check.md`.

## Quality Criteria
- Every section checked against PDP-04's proof density specification
- Actual proof elements counted and classified
- Gaps are specific ("missing study reference for Ashwagandha" not "needs more proof")
- Vague proof claims flagged separately from missing proof
- Brief data not yet used is identified (available proof not utilized)

## Failure Modes
- Proof density specs not available from PDP-04 -> Use PDP-04 AGENT.md defaults table. WARN that specs were not in upstream package.
- Section specified as "maximum" has only vague claims -> HALT. Maximum density sections require specific data. If data doesn't exist, escalate to human or recommend PDP-04 exclude the section.
- Section specified as "none" has proof shoehorned in -> Flag as OVER. Remove proof elements — they dilute the section's instructional/math purpose.
- Brief contains proof data that no section used -> Flag as opportunity. May indicate Layer 2 underutilized available data.
