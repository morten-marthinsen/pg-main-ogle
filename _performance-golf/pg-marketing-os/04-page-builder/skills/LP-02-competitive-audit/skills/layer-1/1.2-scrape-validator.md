# 1.2 — Scrape Validator

> **Skill:** LP-02-competitive-audit
> **Layer:** 1
> **Position:** 2 of 2
> **Purpose:** Validate scraped content quality across all competitor pages. Flag incomplete scrapes. Confirm minimum 3 pages have usable content for Layer 2 analysis.
> **Effort Level:** sonnet
> **Dependencies:** All `scrape-[N].md` files from 1.1
> **Output:** `scrape-validation.md`

---

## INPUT

```yaml
input:
  scrape_files: "scrape-1.md through scrape-[N].md (all available)"
```

---

## EXECUTION

### Step 1: Read All Scrape Files

Read every `scrape-[N].md` file produced by 1.1 Page Scraper.

### Step 2: Assess Each Scrape Against Usability Criteria

For each scrape file, verify ALL four usability requirements:

| Requirement | How to Check | PASS/FAIL |
|------------|-------------|-----------|
| Headline extractable | H1 or clear heading text present in markdown content | |
| 3+ identifiable sections | H2/H3 headings or clear content blocks with different topics | |
| At least 1 CTA | Button text, link text with action language ("Buy Now," "Get Started," "Subscribe," etc.) | |
| 500+ words of body content | Estimate word count of main content (excluding nav/footer) | |

**Usability classification:**
- All 4 requirements met = `usable`
- 3 of 4 requirements met = `marginal` (still counts toward minimum if the missing element is not critical to analysis)
- 2 or fewer requirements met = `unusable` (does NOT count toward minimum)

### Step 3: Count Usable Scrapes

- Count total `usable` + `marginal` scrapes
- If count >= 3: GATE_1 passes
- If count < 3: Document the gap and trigger re-scrape recommendations

### Step 4: Flag Quality Issues

For each scrape, note any quality issues that may affect Layer 2 analysis:
- Missing sections (e.g., footer/pricing section cut off)
- Images not captured (expected for markdown — note it)
- JavaScript-dependent content not rendered
- Partial page load (only above-fold content)
- Duplicate content with another scrape (same template)

### Step 5: Write Validation Report

---

## QUALITY CRITERIA

- Every scrape file assessed against all 4 usability requirements
- Clear usable/marginal/unusable classification for each
- Minimum threshold check documented
- Quality issues flagged per scrape
- List of competitors proceeding to Layer 2 is explicit
- File exists after execution

---

## FAILURE MODES

- Fewer than 3 usable scrapes -> Do NOT proceed to Layer 2. Write re-scrape recommendations and return to 1.1 for retries. If all retries exhausted, HALT with specific message about which URLs failed and why.
- All scrapes are "marginal" -> Acceptable to proceed if 3+ marginal scrapes exist, but note in validation that analysis quality may be limited. Flag specific analysis dimensions that may be affected.
- Scrape files missing (1.1 did not produce expected files) -> Note missing files. Check if 1.1 halted mid-execution. Re-trigger 1.1 if needed.
