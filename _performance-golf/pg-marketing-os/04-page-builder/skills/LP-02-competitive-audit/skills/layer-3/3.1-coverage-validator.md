# 3.1 — Coverage Validator

> **Skill:** LP-02-competitive-audit
> **Layer:** 3
> **Position:** 1 of 2
> **Purpose:** Validate that all 5 analysis dimensions (element map, proof strategy, CTA architecture, offer framing, differentiation synthesis) are complete for each competitor page. Flag any gaps before proceeding to package assembly.
> **Effort Level:** sonnet
> **Dependencies:** All Layer 2 output files
> **Output:** `coverage-validation.md`

---

## INPUT

```yaml
input:
  element_map: "element-map.md"
  proof_analysis: "proof-strategy-analysis.md"
  cta_analysis: "cta-architecture-analysis.md"
  offer_analysis: "offer-framing-analysis.md"
  differentiation_synthesis: "differentiation-synthesis.md"
  scrape_validation: "scrape-validation.md"
```

---

## EXECUTION

### Step 1: Build Coverage Matrix

For each competitor that passed scrape validation, check whether they have complete data in ALL 5 analysis dimensions:

| Dimension | What to Check | Where to Check |
|-----------|--------------|---------------|
| Element Map | Competitor has element inventory with position, function, word count | `element-map.md` |
| Proof Strategy | Competitor has density, first-proof position, types inventory, ratio, summary | `proof-strategy-analysis.md` |
| CTA Architecture | Competitor has CTA inventory, text consistency, sticky check, distribution, summary | `cta-architecture-analysis.md` |
| Offer Framing | Competitor has pricing, value stack, guarantee, urgency, bundle, summary | `offer-framing-analysis.md` |
| Differentiation | Competitor is referenced in universal patterns, differentiators, or gaps | `differentiation-synthesis.md` |

### Step 2: Flag Gaps

For each competitor x dimension combination:
- **COMPLETE:** All required data present
- **PARTIAL:** Some data present but key fields missing (specify which)
- **MISSING:** No data for this competitor in this dimension

### Step 3: Check Cross-Page Synthesis

Verify that `differentiation-synthesis.md` references ALL analyzed competitors (not just a subset).

### Step 4: Determine GATE_2 Status

- All competitors COMPLETE across all 5 dimensions = PASS
- Any PARTIAL = conditional PASS with flagged items (can proceed if gaps are due to scrape limitations, not analysis omission)
- Any MISSING = FAIL — return to the relevant Layer 2 microskill

---

## QUALITY CRITERIA

- Every competitor checked against every dimension
- Clear COMPLETE/PARTIAL/MISSING classification
- Gaps specifically identified (not vague)
- Cross-page synthesis verified to include all competitors
- GATE_2 status explicitly declared
- File exists after execution

---

## FAILURE MODES

- Dimension is MISSING for a competitor -> Return to the relevant Layer 2 microskill (2.1-2.4). Do not proceed to Layer 4.
- Dimension is PARTIAL due to scrape limitations (e.g., pricing hidden) -> Accept as CONDITIONAL_PASS. Note the limitation. The data gap is real and cannot be fixed by re-analysis.
- Differentiation synthesis only references some competitors -> Return to 2.5 Differentiation Synthesizer to include all competitors.
