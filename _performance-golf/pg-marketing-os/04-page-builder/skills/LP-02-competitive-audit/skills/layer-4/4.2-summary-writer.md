# 4.2 — Summary Writer

> **Skill:** LP-02-competitive-audit
> **Layer:** 4
> **Position:** 2 of 3
> **Purpose:** Write the human-readable COMPETITIVE-AUDIT-SUMMARY.md from the competitive-audit.json data and insights ranking. This is the file the human reviewer reads to understand the competitive landscape at a glance.
> **Effort Level:** sonnet
> **Dependencies:** `competitive-audit.json`, `insights-ranking.md`
> **Output:** `COMPETITIVE-AUDIT-SUMMARY.md`

---

## INPUT

```yaml
input:
  competitive_audit: "competitive-audit.json"
  insights_ranking: "insights-ranking.md"
```

---

## EXECUTION

### Step 1: Read Source Files

Read `competitive-audit.json` and `insights-ranking.md`.

### Step 2: Write Summary

Follow the COMPETITIVE-AUDIT-SUMMARY.md template from LP-02-AGENT.md:

1. **Overview:** Competitor count, target vertical, target page type
2. **Competitor Snapshot Table:** One row per competitor with brand, page type, platform, sections, proof density, CTAs, key differentiator
3. **Top 5 Strategic Insights:** From `insights-ranking.md` — the 5 findings that most influence page architecture
4. **Universal Patterns (Table Stakes):** Bulleted list of what every competitor does
5. **Differentiation Opportunities:** Bulleted list of gaps and opportunities
6. **Patterns to Avoid:** Bulleted list of competitive patterns to skip
7. **Recommended Architecture Inputs:** 1-2 sentence recommendation for each downstream skill (LP-03, LP-04, LP-05, LP-06)
8. **Competitor-by-Competitor Detail:** 1 paragraph per competitor covering element highlights, proof approach, CTA strategy, offer framing, overall assessment

### Writing Guidelines

- Write for a human reader (the human reviewer) — clear, direct, no jargon without explanation
- Lead with the most strategically important information
- Use specific competitor names and evidence (not "one competitor")
- Keep each section concise — the JSON has the full detail
- The summary should be readable in 5 minutes or less

---

## OUTPUT SCHEMA

`COMPETITIVE-AUDIT-SUMMARY.md` — follows the template in LP-02-AGENT.md.

---

## QUALITY CRITERIA

- All 8 sections present
- Competitor snapshot table is complete and accurate
- Top 5 insights match `insights-ranking.md`
- Architecture recommendations are specific (not generic)
- Competitor detail paragraphs reference actual scraped content
- Readable in 5 minutes or less
- File exists after execution

---

## FAILURE MODES

- Summary is too long (>2000 words) -> Edit for conciseness. The JSON has full detail — the summary is a strategic overview.
- Architecture recommendations are vague -> Make them specific. "LP-05 should use proof" = vague. "LP-05 should target EXTREME proof density with customer reviews as primary type, deploying first proof above fold via rating strip, because 3 of 4 competitors establish proof in first 2 sections" = specific.
- Missing competitor in snapshot table -> Cross-check against `competitive-audit.json` competitors array. Every analyzed competitor must appear.
