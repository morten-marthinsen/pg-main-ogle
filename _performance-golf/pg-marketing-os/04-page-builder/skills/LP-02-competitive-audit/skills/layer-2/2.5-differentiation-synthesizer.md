## EXECUTION

**POSITIONAL REINFORCEMENT:** This is the strategic synthesis step. You are now looking ACROSS competitors to find patterns, gaps, and opportunities. Every insight must be grounded in the per-competitor analysis from 2.1-2.4. No speculation. No fabrication. Patterns require evidence from 2+ competitors. Gaps require evidence from ALL competitors (none of them do X).

### Step 1: Identify Universal Patterns (Saturated / Table Stakes)

Scan across all competitors for elements, strategies, or approaches that appear in ALL or NEARLY ALL (N-1) competitors:

**Element-level:** Which taxonomy elements appear on every competitor page? (These are table stakes — our page must include them.)

**Proof-level:** What proof approach is universal? (e.g., "all 4 competitors use customer reviews as primary proof")

**CTA-level:** What CTA patterns are universal? (e.g., "all competitors place primary CTA above fold")

**Offer-level:** What offer strategies are universal? (e.g., "all competitors use subscription model with savings display")

For each universal pattern:
- Name the pattern
- List which competitors use it
- State the frequency (N of M)
- Describe the implication for our page (table stakes = must include)

### Step 2: Identify Unique Differentiators

Scan for elements, strategies, or approaches that appear in ONLY 1 competitor:

For each unique differentiator:
- Name it
- Identify which competitor uses it
- Assess apparent effectiveness (does it seem to serve a clear strategic purpose?)
- State the implication (opportunity to adopt? Best left alone? Only works for their brand?)

### Step 3: Identify Gaps and Opportunities

Scan for elements or strategies that NO competitor uses but that the target page COULD use based on:
- Elements in the taxonomy that no competitor deploys
- Proof types from cross-page-pattern-analysis.md that are absent in this competitive set
- CTA approaches not represented
- Offer framing techniques not used
- Page type advantages (if competitors are all Type B and target is Type A, what does Type A enable?)

For each gap/opportunity:
- Name the gap
- Describe the opportunity
- Assess confidence (high: clearly missing and valuable; medium: potentially valuable; low: missing for a reason)
- List relevant taxonomy elements

### Step 4: Generate Recommended Patterns

Based on Steps 1-3, identify the patterns the target page SHOULD consider adopting:

For each recommended pattern:
- Name the pattern
- Source (which competitor(s) demonstrate it)
- Rationale (why replicate this)
- Adaptation notes (how to make it ours — NOT a copy)

### Step 5: Generate Patterns to Avoid

Identify patterns the target page should NOT adopt:

For each pattern to avoid:
- Name the pattern
- Source (which competitor(s) use it)
- Rationale (why avoid: saturated, ineffective, inappropriate for our page type, etc.)

### Step 6: Type-Specific Recommendations

If any competitors are a different page type than the target:
- Note which competitors are a different type
- Flag which of their patterns are type-specific (not transferable)
- Flag which patterns transcend type (can be adapted)

## QUALITY CRITERIA

- Universal patterns backed by frequency data (N of M competitors)
- Unique differentiators traced to specific competitors with evidence
- Gaps are genuine absences (confirmed NONE of the competitors use the element/strategy)
- Recommended patterns include adaptation notes (no verbatim copying)
- Patterns to avoid include clear rationale
- Architecture skill recommendations are specific and actionable
- Every claim traces to data from 2.1-2.4 analysis files
- File exists after execution

---

## FAILURE MODES

- Finding "patterns" from only 2 competitors -> A pattern needs 2+ of 3 at minimum. With only 2, note it as a "common approach" not a "universal pattern."
- Recommending a pattern without adaptation notes -> Every recommendation MUST include how to make it original. "Do what C1 does" is not a recommendation. "Adopt C1's approach of performance guarantee with specific metrics, adapted to our product's measurable outcomes" IS.
- Missing gaps because all competitors are similar -> Broaden the gap search: look at the full taxonomy, look at other verticals' patterns from cross-page-pattern-analysis.md, look at page type advantages. Similar competitors = bigger opportunity to differentiate.
- Listing too many insights without prioritization -> The Insights Ranker (3.2) handles prioritization, but the synthesis should naturally emphasize the highest-impact patterns first.
