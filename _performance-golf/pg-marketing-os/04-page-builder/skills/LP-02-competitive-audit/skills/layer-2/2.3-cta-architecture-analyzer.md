# 2.3 — CTA Architecture Analyzer

> **Skill:** LP-02-competitive-audit
> **Layer:** 2
> **Position:** 3 of 5
> **Purpose:** Analyze each competitor page's CTA strategy: how many CTAs, where they are placed, what the button text says, what emotional appeal drives each CTA, what supporting copy surrounds it, and whether sticky/persistent CTAs are used.
> **Effort Level:** opus
> **Dependencies:** All valid `scrape-[N].md` files, `element-map.md`
> **Output:** `cta-architecture-analysis.md`

---

## INPUT

```yaml
input:
  scrape_files: "All usable/marginal scrape-[N].md files"
  element_map: "element-map.md"
```

---

## EXECUTION

**POSITIONAL REINFORCEMENT:** You are analyzing CTA ARCHITECTURE — the strategic design of how, where, and why CTAs are deployed on each page. You are not judging CTA quality. You are mapping the strategy.

### For Each Competitor, Analyze:

#### A. CTA Inventory

Find EVERY CTA on the page. A CTA is any element that asks the visitor to take a conversion action — buttons, links styled as buttons, inline text links with action language, forms with submit buttons.

For each CTA instance, record:
- **Position:** above_fold / section_[N] / sticky / footer
- **Button text:** Exact text (e.g., "Start My Free Trial," "Add to Cart," "Schedule Demo")
- **Emotional appeal:** What drives the click?
  - Action (direct: "Buy Now," "Add to Cart")
  - Urgency ("Claim Your Spot Before It's Gone")
  - Value ("Get $200 Worth for $37")
  - Curiosity ("See What's Inside")
  - Social ("Join 1M+ Members")
  - Fear/Loss ("Don't Miss Out")
- **Supporting copy:** Text immediately surrounding the CTA (guarantee reminder, social proof, urgency text). Quote 10-30 words.
- **Risk reversal attached:** Is there a guarantee mention, cancellation policy, or "no risk" language near this CTA?

#### B. CTA Text Consistency

- Do all CTAs use the same text? (consistent)
- Do CTAs vary as you scroll? (varying — note the progression)
- If varying: Is there a pattern? (urgency escalation, action differentiation, emotional modulation)

#### C. Sticky/Persistent CTA

- Does the page have a sticky CTA bar (desktop or mobile)?
- If yes: what text? Where does it appear from?
- If no: note the absence

#### D. CTA Count and Distribution

- Total CTA count
- Distribution by page position:
  - Above fold: [count]
  - Top 25%: [count]
  - Mid 25-75%: [count]
  - Bottom 25%: [count]
  - Sticky: [Y/N]

#### E. CTA Architecture Summary

Write a 2-3 sentence summary of the CTA strategy: Is it repetitive reinforcement (same CTA everywhere)? Emotional escalation (building urgency through the page)? Minimal friction (one clear action)? Multi-path (different CTAs for different actions)?

## QUALITY CRITERIA

- Every CTA on every competitor page inventoried
- Each CTA has all 5 fields: position, text, appeal, supporting copy, risk reversal
- Text consistency assessed
- Sticky CTA presence/absence noted
- CTA distribution across page positions documented
- Cross-competitor patterns identified
- No CTA claims without scraped content evidence
- File exists after execution

---

## FAILURE MODES

- CTAs not clearly distinguishable in markdown -> Look for action language ("Get," "Start," "Buy," "Join," "Subscribe," "Schedule," "Apply," "Register," "Add to Cart"). Links with these verbs are likely CTAs.
- Page has 100+ micro-CTAs (interactive builder like Native) -> Note the pattern. Count unique CTA texts and positions rather than every instance. Describe the architectural approach.
- Supporting copy not clearly associated with a specific CTA -> Quote the nearest text block above or below the CTA. Note the approximation.
- Sticky CTA not detectable in markdown -> Markdown scrapes cannot detect sticky behavior. Look for indicators: repeated CTA at bottom, "fixed" or "sticky" in scraped CSS classes, or a separate CTA bar section. Note if detection is uncertain.
