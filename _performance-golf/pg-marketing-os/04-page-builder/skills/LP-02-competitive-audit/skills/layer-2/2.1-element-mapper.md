# 2.1 — Element Mapper

> **Skill:** LP-02-competitive-audit
> **Layer:** 2
> **Position:** 1 of 5
> **Purpose:** Map each competitor page's elements against the 40-element taxonomy from element-taxonomy.md. Record which elements are present, their position, approximate word count, and the function they serve on THIS specific page.
> **Effort Level:** opus
> **Dependencies:** All valid `scrape-[N].md` files, element-taxonomy.md
> **Output:** `element-map.md`

---

## INPUT

```yaml
input:
  scrape_files: "All usable/marginal scrape-[N].md files (from scrape-validation.md)"
  taxonomy: "element-taxonomy.md"
  scrape_validation: "scrape-validation.md"
```

---

## EXECUTION

**POSITIONAL REINFORCEMENT:** You are mapping WHAT elements exist on each page and WHERE they are positioned. You are NOT judging quality. You are building an inventory. Every element must include position, function, approximate word count, and a verbatim sample. "Present" alone is not analysis.

### Step 1: Load the Taxonomy

Read `element-taxonomy.md` and internalize the 8 functional categories and their elements:

1. **Attention Elements** (above fold): Headline, Subheadline, Hero Image, Video/VSL, Countdown Timer, Urgency Banner, Review Count Badge, Price Above Fold
2. **Trust Elements:** Trust Badges, Partner Logos, Celebrity Endorsement Cards, Expert Endorsements, Certification Badges, "X+ Customers" Claim
3. **Proof Elements:** Customer Video Testimonials, Written Reviews, Celebrity Testimonials, Screenshot Testimonials, Stat Claims Bar, Case Studies, Rating Distribution Chart, Scientific Citations, Third-Party Review Widget, "X+ Testimonials" Claim
4. **Education Elements:** Step/Lesson Preview, Ingredient Breakdown, How-To/Dosing Guide, Mechanism Explanation, Comparison Table, Feature Deep-Dives, Feature Ticker
5. **Conversion Elements:** Primary CTA, Subscription Toggle, Bundle Tier Pricing, Application Form, Registration Form, Inline Checkout, Order Bumps, Currency Selector, Quantity Selector, Variant Selector, Free Gift Threshold
6. **Offer Elements:** Value Stack, Free Trial, Performance Guarantee, Money-Back Guarantee, Subscription Benefits, Free Bonuses, Charity Tie-In, Savings Display, "Best Seller" Badge, Loyalty Program
7. **Navigation Elements:** FAQ Section, Product Category Navigation, Business Model Selector, Qualification Checklist, Footer Navigation
8. **Brand/Story Elements:** Authority Letter, Founder Story, Company Differentiators, Customer Lifestyle Photos, New Arrivals Grid
9. **Capture Elements:** Email/SMS Popup, Contest/Giveaway Popup, Live Chat Widget

### Step 2: Map Each Competitor Page

For EACH competitor (C1 through C[N]), read the full markdown content from their `scrape-[N].md` file and systematically scan for every taxonomy element.

For each element detected, record:
- **Element name** (from taxonomy)
- **Taxonomy category** (attention/trust/proof/etc.)
- **Present:** Yes
- **Position:** above_fold / early (top 25%) / mid (25-75%) / late (75%+) / footer
- **Section number:** Which section of the page (1, 2, 3...)
- **Approximate word count:** How many words this element occupies
- **Function served:** What this element DOES on THIS page (not what it does in theory). E.g., "Establishes authority through Trustpilot rating before any copy is read" or "Anchors price against retail value to create perceived savings"
- **Verbatim sample:** Short quote from the scraped content (10-30 words) that represents this element

### Step 3: Classify Each Competitor Page

Based on the element map, classify each competitor:
- **Page type:** Type A / Type B / Hybrid (using the structural signals from the taxonomy)
- **Platform:** Shopify / ClickFunnels / WordPress / Framer / Custom / Unknown
- **Section count:** Total distinct sections
- **Total elements detected:** Count of taxonomy elements found

### Step 4: Write Element Map

## QUALITY CRITERIA

- Every competitor page mapped against the full taxonomy
- Every detected element has ALL 6 fields: position, section, word count, function, verbatim sample
- Page type and platform classified for each competitor
- Elements NOT present are explicitly listed (not just omitted)
- Cross-competitor frequency table shows element distribution
- No elements listed as merely "present" without detail
- File exists after execution

---

## FAILURE MODES

- Scraped content too thin to map elements -> Use whatever is available. Note which taxonomy categories cannot be assessed due to thin content. Downgrade the competitor's map quality.
- Element is ambiguous (could be trust OR proof) -> Classify by primary function on the page. Note the ambiguity.
- Taxonomy element not found but a variant exists -> Map it to the closest taxonomy element and note the variation. E.g., "Video testimonial detected — classified as Customer Video Testimonial though format is audio-only."
- Competitor page has elements not in the taxonomy -> Add them to an "Unlisted Elements" section at the bottom of that competitor's map. These represent potential taxonomy gaps.
