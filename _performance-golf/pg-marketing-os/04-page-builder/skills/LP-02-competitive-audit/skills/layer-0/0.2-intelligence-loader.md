# 0.2 — Intelligence Loader

> **Skill:** LP-02-competitive-audit
> **Layer:** 0
> **Position:** 2 of 3
> **Purpose:** Load conversion intelligence from LP-01 (if available) and extract any competitor URLs, benchmark data, or vertical insights that inform LP-02's competitor selection and analysis.
> **Effort Level:** haiku
> **Dependencies:** `conversion-intelligence.json` from LP-01 Conversion Intelligence Loader (OPTIONAL — LP-01 may not have been run)
> **Output:** `intelligence-load.md`

---

## INPUT

```yaml
input:
  intelligence_path: "[path to conversion-intelligence.json from LP-01, if available]"
  project_folder: "[path to project output folder]"
```

---

## EXECUTION

1. Attempt to locate `conversion-intelligence.json` in the project output folder. Search paths:
   - `./~outputs/[project-name]/landing-page/lp-01-conversion-intelligence/conversion-intelligence.json`
   - `./~outputs/[project-name]/lp-01-conversion-intelligence/conversion-intelligence.json`

2. **If found,** parse and extract:
   - **Competitor URLs** — any URLs identified during conversion intelligence gathering
   - **Vertical benchmarks** — conversion rates, bounce rates, time-on-page norms for this vertical
   - **Industry patterns** — any structural patterns noted about competitors in this vertical
   - **Audience insights** — behavioral data about the target audience relevant to competitor analysis
   - **Reference pages** — any pages cited as examples or benchmarks

3. **If not found,** this is acceptable — LP-01 is optional for LP-02. Write `intelligence-load.md` with status `NOT_AVAILABLE` and note that competitor URLs must come from user input or Firecrawl search.

4. Write extracted data to `intelligence-load.md`.

---

## QUALITY CRITERIA

- LP-01 output searched in all expected paths
- If found: all relevant fields extracted
- If not found: status clearly noted, downstream impact documented
- File exists after execution

---

## FAILURE MODES

- LP-01 output not found -> This is expected and acceptable. Write status `NOT_AVAILABLE`. LP-02 can proceed without LP-01 data — URLs will come from user input or Firecrawl search.
- LP-01 output found but contains no competitor URLs -> Extract whatever is available (benchmarks, insights). Note that URLs still need to be collected in 0.3.
- LP-01 output is malformed/unparseable -> Write status `PARTIAL`, extract what is readable, flag the parsing issue.
