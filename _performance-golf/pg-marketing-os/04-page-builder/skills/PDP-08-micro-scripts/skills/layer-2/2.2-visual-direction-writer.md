# 2.2-visual-direction-writer.md

> **Layer:** 2 (Script Generation)
> **Skill:** PDP-08-micro-scripts
> **Purpose:** Write visual direction notes per script
> **Outputs:** `visual_directions` object with timestamped visual cues per script

---

## FUNCTION

Write timestamped visual direction notes for each script that tell the video production team what to show at each moment. Visual direction must align with narration timing and be specific enough for production.

---

## VISUAL DIRECTION PROTOCOL

### Step 1: Create Timestamped Shot List

```
FOR each script:
  DIVIDE duration into segments (5-10 second blocks)
  FOR each segment:
    DEFINE:
      timestamp: "Xs-Ys"
      shot_type: [wide|medium|close-up|cutaway|text-overlay|b-roll]
      visual_content: [specific description of what to show]
      narration_alignment: [which narration lines play during this shot]
      motion: [static|slow-pan|zoom-in|rotation|transition]
```

### Step 2: Shot Types per Script Type

```
HERO VIDEO shots:
  - Product hero shot (rotating, clean background)
  - Feature close-up (mechanism detail)
  - In-use demonstration (lifestyle context)
  - Results visualization (before/after, numbers on screen)
  - Customer social proof (review text overlay or UGC clip)
  - CTA screen (product + price + button)

FEATURE EXPLAINER shots:
  - Feature close-up (the actual component/ingredient)
  - How-it-works animation (mechanism in action)
  - Benefit visualization (the result of the feature)
  - Proof overlay (stat, testimonial, certification)

UGC SCRIPT shots:
  - Customer talking head or lifestyle
  - Product in customer's hands/environment
  - Before/after if applicable
  - Product close-up with customer reaction

HOW-TO shots:
  - Product unboxing or setup
  - Step-by-step close-ups
  - Correct usage demonstration
  - Result/outcome shot

COMPARISON shots:
  - Side-by-side product comparison
  - Performance difference visualization
  - Data overlay (chart, numbers)
  - Winner highlight
```

### Step 3: Quality Check

```
FOR each script's visual direction:
  VERIFY:
    [ ] Every timestamp has a shot defined
    [ ] No gaps in timeline coverage
    [ ] Shots align with narration content
    [ ] At least 3 distinct shots per 30-second script
    [ ] Shot descriptions are specific (not "show the product")
    [ ] CTA has a dedicated visual (product + action)
```

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Timeline gaps (no shot defined for segment) | Fill gap with appropriate shot |
| WARNING | Shots too generic ("show the product") | Add specific detail |
| WARNING | Fewer than 3 shots per 30s | Add visual variety |
