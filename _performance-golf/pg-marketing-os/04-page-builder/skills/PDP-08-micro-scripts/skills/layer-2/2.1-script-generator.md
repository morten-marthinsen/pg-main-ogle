# 2.1-script-generator.md

> **Layer:** 2 (Script Generation)
> **Skill:** PDP-08-micro-scripts
> **Purpose:** Generate video narration scripts anchored to approved DSI + micro-scripts
> **Outputs:** `generated_scripts` object with narration text per script
> **Framework:** Bill Schley "The Micro-Script Rules" — scripts USE approved micro-scripts, they don't replace them
> **Upstream dependencies:** PDP-02 (feature names + micro-scripts + DSI), PDP-07 (section copy)

---

## FUNCTION

Write video narration/copy for each assigned script within its duration budget. Scripts are built AROUND the product's approved DSI and micro-scripts from PDP-02 — these short phrases (~5 words) serve as the narrative anchors that viewers remember. The narration fills in context, proof, and emotion between the micro-scripts.

**CRITICAL:** Video scripts do NOT generate new taglines, catchphrases, or repeatable phrases. The DSI and micro-scripts are already locked from PDP-02. This skill weaves them into spoken narration so they land naturally and stick.

---

## MICRO-SCRIPT INTEGRATION RULES

```
BEFORE generating any script:

  1. LOAD from PDP-02 outputs:
     - Product DSI (the single most important sentence)
     - Primary tagline micro-script
     - Supporting theme micro-script
     - Per-feature micro-scripts (one per feature)

  2. PLACEMENT RULES:
     - DSI appears in HERO VIDEO scripts (verbatim or near-verbatim)
     - Primary tagline is the LAST LINE the viewer hears in hero scripts
     - Feature micro-scripts appear when their feature is referenced
     - Micro-scripts are spoken as standalone beats — pause before and after
     - Do NOT bury a micro-script inside a longer sentence
     - Do NOT paraphrase a micro-script — use it verbatim

  3. REPETITION STRATEGY:
     - Primary tagline can repeat across multiple scripts (that's the point)
     - Feature micro-scripts appear only in their feature's scripts
     - DSI appears in hero and comparison scripts
     - Supporting theme can appear in any script as a closer or transition
```

---

## GENERATION PROTOCOL

### Step 1: Generate per Script Type

```
FOR each assigned script:

  HERO VIDEO (30-60s):
    0-5s: Hook — problem statement or vivid image (NOT the micro-script yet)
    5-15s: Product introduction + DSI (spoken naturally, verbatim or near-verbatim)
    15-35s: Top 2-3 benefits — weave in feature micro-scripts as standalone beats
    35-50s: Social proof element (customer count, review quote, expert credential)
    50-55s: CTA — where to buy, risk reversal hint
    55-60s: Primary tagline micro-script as the LAST LINE (standalone, pause before it)

  FEATURE EXPLAINER (15-30s):
    0-3s: Feature name (exact PDP-02 name)
    3-5s: Feature micro-script as the opening beat (standalone line)
    5-15s: What it does — mechanism/function (expand the micro-script's claim)
    15-25s: Why it matters — specific benefit to the golfer
    25-30s: Proof point or CTA

  UGC SCRIPT (15-30s):
    0-5s: Customer introduction (name, context)
    5-15s: Their experience/problem
    15-25s: Their result with the product
    25-30s: Recommendation — can naturally include primary tagline or supporting theme

  HOW-TO (30-60s):
    0-5s: "Here's how to use [Product]"
    5-40s: Step-by-step demonstration (3-5 steps)
    40-55s: Expected result / what to notice
    55-60s: CTA or primary tagline as closer

  COMPARISON (15-30s):
    0-5s: "Most [category] products do X"
    5-15s: "But [Product] does Y because of [Feature]" — include DSI claim
    15-25s: Visual proof of difference — feature micro-script as the beat
    25-30s: CTA

  ROBOT TESTING / PROOF VIDEO (30-60s):
    0-5s: Setup — what's being tested and why
    5-15s: Test methodology (equipment, conditions)
    15-40s: Results narration — what the data shows, reference feature micro-scripts
    40-50s: What this means for the golfer (benefit translation)
    50-60s: Supporting theme or primary tagline as closer
```

### Step 2: Word Count Enforcement

```
FOR each script:
  CHECK narration word count against word_budget
  IF over budget: COMPRESS — cut context/filler, NEVER cut micro-scripts
  IF significantly under: ADD proof detail or extend benefit layering
  TOLERANCE: +/- 10% of word budget

  COMPRESSION PRIORITY (what to cut first):
    1. Transition phrases and filler
    2. Redundant proof points
    3. Extended benefit statements
    NEVER CUT: DSI, micro-scripts, feature names, CTA
```

### Step 3: Feature Name + Micro-Script Enforcement

```
FOR each script:
  VERIFY:
    [ ] All feature names match PDP-02 exactly
    [ ] All micro-scripts used verbatim (not paraphrased)
    [ ] Micro-scripts are standalone beats (not buried in sentences)
    [ ] DSI appears where required (hero, comparison)
    [ ] Primary tagline appears as final beat in hero scripts

  IF any micro-script is paraphrased: CORRECT to exact PDP-02 version
  IF any micro-script is buried in a longer sentence: BREAK IT OUT as standalone beat
```

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Narration exceeds word budget by >10% | Compress narration — never cut micro-scripts |
| CRITICAL | Micro-script paraphrased or reworded | Correct to exact PDP-02 approved version |
| CRITICAL | New tagline/catchphrase invented instead of using approved micro-script | Delete and replace with PDP-02 micro-script |
| WARNING | Micro-script buried inside a longer sentence | Break out as standalone beat with pause |
| WARNING | Feature name paraphrased | Correct to exact PDP-02 name |
| WARNING | CTA missing from script | Add CTA to final 5 seconds |
| WARNING | DSI missing from hero or comparison script | Add DSI in the 5-15s window |
