# 3.1 — 4-Test Sequence Logic Validator

**Skill:** LP-04-section-sequence
**Position:** Layer 3, Microskill 1
**Purpose:** Run the 4 non-negotiable sequence logic tests. These tests catch conversion-killing structural errors before any writing begins. All 4 must PASS. A FAIL blocks GATE_3.
**Effort Level:** medium
**Dependencies:** `word-count-plan.md`, `proof-density-plan.md`, `cta-placement-plan.md` — all Layer 2 outputs
**Output:** `logic-validation.md`

---

## Input Schema

```yaml
input:
  section_sequence: "[ordered list of sections with numbers]"
  pricing_block_number: "[section number where price is revealed]"
  mechanism_narrative_number: "[section number of mechanism section]"
  first_proof_block_number: "[section number of first heavy proof block]"
  first_paid_cta_section: "[section number of first CTA that asks for money]"
  first_trust_signal_section: "[section number of first authority/proof signal]"
  guarantee_section_number: "[section number of guarantee block]"
  page_type: "[type_a | type_b | hybrid]"
```

---

## Test 1: Value Before Price

**What it tests:** Does value establishment (proof, mechanism explanation, ingredient education, testimonials) appear BEFORE the pricing block?

**Evaluation:**
```
required_before_price:
  - mechanism_narrative (section [X])
  - at_least_one_proof_block (section [X])

pricing_block at section [Y]

PASS: mechanism_number < pricing_block_number
      AND first_proof_block_number < pricing_block_number
FAIL: Either mechanism OR first proof block appears AFTER pricing
```

**Common failure scenario:** Pricing block placed early "to capture impulse buyers" before the mechanism or proof sections. This kills conversion for cold/warm traffic.

**Exception:** Type B pages can show price above fold (Section 1) because the rating strip + social proof volume visible in the above fold section constitutes value signaling. Test 1 is PASS for Type B if above fold rating strip shows 4.5+ stars and 50+ reviews.

---

## Test 2: Trust Before Ask

**What it tests:** Is there at least one authority signal or social proof element before the first CTA that asks for money?

**Evaluation:**
```
PASS: first_trust_signal_section < first_paid_cta_section
FAIL: first_paid_cta_section appears before any trust signal
```

**Note on Type B:** Rating strip above fold = trust signal. If ATC is also above fold, this still PASSES for Type B because the rating strip appears in the same above-fold section before or alongside the ATC.

---

## Test 3: Education Before Mechanism Reveal (Type A Only)

**What it tests:** Does the above-fold + lead section earn sufficient readership before the mechanism is named?

**Evaluation:**
```
[Type A and Hybrid only — skip for pure Type B]

PASS: lead_section_number < mechanism_narrative_number
      AND story_or_root_cause_section_number < mechanism_narrative_number
FAIL: Mechanism named in headline (Section 1) or lead (Section 2) before
      story/root cause have run
```

**Rationale:** Naming the mechanism before the prospect has been emotionally prepared means they have no context for why it matters. The story and root cause create the "problem frame" that makes the mechanism revelation meaningful.

**Exception:** Stage 4–5 awareness audiences can see mechanism named earlier. If awareness_stage >= 4, this test is set to CONDITIONAL_PASS with note.

---

## Test 4: Guarantee Proximate to Pricing

**What it tests:** Is the guarantee block within 2 sections of the pricing block?

**Evaluation:**
```
distance = |guarantee_section_number - pricing_section_number|

PASS: distance <= 2
FAIL: distance > 2

Acceptable configurations:
  - Guarantee immediately BEFORE pricing (distance = 1) — removes risk before price
  - Guarantee immediately AFTER pricing (distance = 1) — reassures after price shock
  - Guarantee 2 sections after pricing (distance = 2) — acceptable
  - Guarantee 3+ sections from pricing — FAIL
```

**Why this matters:** Price creates anxiety. Guarantee relieves anxiety. Separating them by multiple sections means the visitor carries unresolved anxiety through intervening content, often bouncing.

## Quality Criteria
- All 4 tests explicitly evaluated with evidence from actual section numbers
- Any FAIL has a specific fix recommendation (not "fix the sequence")
- Exceptions documented when applied
- Result is binary PASS or FAIL — no "partial pass"

## Failure Modes
- Section numbers unclear (sections not yet numbered in sequence) → Number all sections first using the Layer 1 output
- Guarantee block excluded as optional section → Test 4 FAIL automatically — guarantee is mandatory and must be added
