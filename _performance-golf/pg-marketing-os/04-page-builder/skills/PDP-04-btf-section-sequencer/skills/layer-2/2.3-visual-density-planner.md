# 2.3 — Visual Density Planner

**Skill:** PDP-04-btf-section-sequencer
**Position:** Layer 2, Microskill 3
**Purpose:** Specify the exact image count, type, and layout for every section in the BTF sequence. PDP pages are image-heavy by nature — minimum 1 image per 150 words (vs Type A's 1 per 400 words). Vague visual plans ("add images") are the #3 failure mode. Every section must have a specific visual density spec that downstream design and writing skills can consume as a production brief.
**Effort Level:** medium
**Dependencies:** `optional-decisions.md` (1.4 — locked section list), `word-count-plan.md` (2.1 — word count targets for ratio calculation), `pdp-reference-loaded.md` (0.3 — visual density defaults)
**Output:** `visual-density-plan.md`

## Execution Logic

**Step 1:** Load visual density defaults from PDP-04-AGENT.md reference:

| Section | Image Strategy |
|---------|---------------|
| Product Highlights | Icon tiles, expandable with product shots |
| UGC Video Carousel | 3-6 video thumbnails, auto-play capable |
| The Expert | Expert headshot/video still, institution logos |
| What's in It | Ingredient hero images, "X for Y" visual cards |
| What to Expect | Timeline graphic with milestone icons |
| The Problem / Solution / Why | Diagrams, infographics, mechanism illustrations |
| Comparison Chart | Branded table with checkmark/X visual treatment |
| Cost Savings | Bundle visualization, savings callout graphic |
| How It's Made | Sourcing/manufacturing photography, cert badges |
| How to Use | 3-step numbered icons with product-in-use photos |
| Reviews & Ratings | Review cards with customer photos, histogram graphic |
| FAQ | Accordion UI (minimal imagery — text-focused) |
| Identity Matching | 3-column persona cards with lifestyle photos |
| The Proof (Data) | Study circle graphics, bar charts, data viz |

**Step 2:** For each section, calculate the minimum image count:
```
minimum_images = CEILING(word_count_target / 150)
```
This is a FLOOR — sections may have MORE images, but not fewer than this ratio.

**Step 3:** For each section, assign:
- **image_count:** Specific number (at minimum the 1-per-150-words ratio)
- **primary_type:** The dominant image type for this section (product | lifestyle | ingredient | infographic | badge | headshot | video_thumb | data_viz | icon)
- **layout:** How images are arranged (card_grid | carousel | accordion | timeline | table | full_width | split_panel | icon_strip)
- **notes:** Any specific image requirements or constraints

**Step 4:** Handle image-light sections:
- FAQ: Intentionally minimal imagery (accordion UI). Visual ratio exception granted. Document why.
- The Problem: May use 1 agitation illustration only. Document minimal visual rationale.
- Cost Savings: 1 savings visualization graphic. Text-light, graphic-heavy.

**Step 5:** Sum total images across all sections. Calculate overall visual density:
```
overall_ratio = total_btf_word_count / total_images
```
Target: overall_ratio <= 150 (1 image per 150 words or better)

**Step 6:** Write visual density plan.

## Quality Criteria
- Every section has a specific image count — not "add images" or "several"
- Every section has a primary image type and layout format assigned
- No section (except documented exceptions) falls below the 1:150 word-to-image ratio
- Overall page visual density meets or exceeds the 1:150 standard
- Image types are diverse — not all product shots or all icons
- Layout formats are appropriate for section content type
- Visual-light exceptions are documented with rationale

## Failure Modes
- Any section says "add images" without count -> HALT. This is the #3 failure mode. Specify count + type + layout.
- Overall ratio exceeds 1:150 (e.g., 1 image per 200 words) -> Add images to underserved sections until ratio is met.
- All sections use the same image type -> WARNING. PDP pages need visual variety. Diversify image types.
- Layout not specified for a section -> HALT. Downstream design skills need layout format to produce section mockups.
- Image count is 0 for a section with word count > 0 (and not an approved exception) -> HALT. Add images.
