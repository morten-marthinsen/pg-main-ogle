# 2.5 — Sticky ATC Trigger Planner

**Skill:** PDP-04-btf-section-sequencer
**Position:** Layer 2, Microskill 5
**Purpose:** Define when and how the sticky Add-to-Cart bar appears, what it contains, and when it disappears. This is the primary mobile conversion mechanism for PDP pages. Every PDP must have a sticky bar plan — omitting it is the #4 failure mode. The sticky bar ensures that no matter how far a visitor scrolls, a one-tap purchase path is always visible.
**Effort Level:** medium (opus)
**Dependencies:** `above-fold-loaded.md` (0.2 — buy box dimensions), `cta-placement-plan.md` (2.4 — mid-page CTA positions), `brief-verified.md` (0.1 — product/variant info)
**Output:** `sticky-atc-plan.md`

## Execution Logic

**Step 1:** Determine mobile trigger point:
- **Default trigger:** When the above-fold buy box scrolls completely out of the viewport
- **Calculate approximate trigger depth:** Buy box bottom position (typically 400-600px on mobile, 300-500px on desktop)
- If buy box position data is available from above-fold-loaded.md, use it
- If not, use default: 500px on mobile, 400px on desktop

**Step 2:** Determine desktop behavior:
- **Option A: Mirror mobile** — Sticky bar appears on desktop at same scroll depth as mobile
- **Option B: Desktop disabled** — No sticky bar on desktop (desktop has more screen real estate; buy box may remain visible in side panel)
- **Option C: Hover intent trigger** — Sticky bar appears on desktop when exit-intent is detected
- **Default recommendation:** Mirror mobile (Option A) unless desktop layout has a persistent side-panel buy box

**Step 3:** Define sticky bar content:
- **Product name:** Truncated to fit bar (max ~30 characters on mobile)
- **Selected variant:** Show currently selected variant (or default variant if none selected)
- **Price:** Show current price (match above-fold price display format)
- **ATC button:** Match text and style from above-fold buy box for consistency
- **Optional elements:** Star rating (compact), stock indicator, discount badge

**Step 4:** Define sticky bar behavior:
- **Appear animation:** Slide up from bottom of screen
- **Position:** Fixed to bottom of viewport (mobile) or bottom of viewport (desktop)
- **Disappear condition:** When visitor scrolls back up and buy box re-enters viewport
- **Re-appear:** When buy box scrolls out of viewport again
- **Z-index:** Above all page content, below modals/overlays

**Step 5:** Define interaction with mid-page CTAs:
- Sticky bar is ALWAYS visible (once triggered) regardless of mid-page CTA positions
- Mid-page CTAs serve visitors who want to act at a specific persuasion arc conclusion
- Sticky bar serves visitors who could act at ANY moment during their browse
- Both systems complement — do not conflict

**Step 6:** Define variant selection behavior:
- **Single SKU:** No variant display in sticky bar — just product name + price + ATC
- **Multiple variants:** Show selected variant. Tapping variant in sticky bar should:
  - Option A: Scroll to above-fold buy box (let visitor change variant there)
  - Option B: Open a mini-variant selector in the sticky bar itself
  - Default recommendation: Option A (simpler, less friction)

**Step 7:** Write sticky ATC plan.

## Quality Criteria
- Mobile trigger defined with specific scroll depth (px or viewport condition)
- Desktop behavior explicitly chosen (mirror/disabled/hover) with rationale
- Bar content fully specified — product name, variant, price, ATC text all documented
- ATC button text matches above-fold buy box for consistency
- Disappear condition defined (not "just stays visible forever")
- Variant selection behavior defined for both single-SKU and multi-variant cases
- Interaction with mid-page CTAs documented

## Failure Modes
- Sticky ATC plan not written -> HALT. This is the #4 failure mode. Every PDP must have this plan.
- Trigger depth undefined ("it appears when you scroll") -> HALT. Specify px depth or viewport condition.
- Bar content missing ATC button text -> HALT. The button is the conversion element.
- Desktop behavior not addressed -> HALT. Must explicitly choose mirror, disabled, or hover.
- ATC button text differs from above-fold buy box -> WARNING. Inconsistency reduces trust. Match the above-fold text.
- No disappear condition defined -> WARNING. Sticky bar should disappear when buy box is visible again.
