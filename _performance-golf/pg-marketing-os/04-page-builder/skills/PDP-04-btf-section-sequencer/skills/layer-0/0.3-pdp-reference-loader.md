# 0.3 — PDP Reference Loader

**Skill:** PDP-04-btf-section-sequencer
**Position:** Layer 0, Microskill 3
**Purpose:** Load the NLS 16-Section PDP BTF Taxonomy, PDP Word Count Benchmark Table, Proof Density Defaults, Visual Density Targets, and Sticky ATC Bar Logic from PDP-04-AGENT.md. This microskill ensures the agent has all PDP-specific reference data loaded before any structural decisions are made. Without this data, Layer 1 taxonomy selection and Layer 2 proportioning would operate from memory (which degrades under context pressure).
**Effort Level:** low
**Dependencies:** PDP-04-AGENT.md (this file — the master agent spec)
**Output:** `pdp-reference-loaded.md`

---

## Input Schema

```yaml
input:
  agent_md_path: "[path to PDP-04-AGENT.md]"
```

---

## Execution Logic

**Step 1:** Load the following reference tables from PDP-04-AGENT.md:

1. **NLS 16-Section PDP BTF Taxonomy** — All 16 section IDs, names, persuasion functions, and default mandatory/optional status
2. **PDP Section Ordering — NLS Recommended Flow** — The default ordering logic and ordering principles
3. **PDP Word Count Benchmark Table** — Minimum/Standard/Extended ranges for all 16 sections + BTF totals
4. **Proof Density By Section — PDP Defaults** — Per-section density level and primary proof types
5. **Visual Density Targets — PDP** — Per-section image strategy
6. **Sticky ATC Bar Logic** — Trigger rules, scroll depth milestones, content specs
7. **4-Test PDP Sequence Logic Validator** — All 4 tests with pass/fail conditions
8. **10-Point PDP Sequence Audit** — Scoring criteria

**Step 2:** Verify each reference table was loaded completely:
- 16-Section Taxonomy: exactly 16 rows loaded
- Word Count Table: all 16 sections + BTF total row present
- Proof Density Defaults: all 16 sections have a density level assigned
- Visual Density Targets: all 16 sections have an image strategy

**Step 3:** Write the loaded reference summary to `pdp-reference-loaded.md`. This file confirms all reference data is available for Layer 1 and Layer 2.

## Quality Criteria
- All 8 reference tables loaded — not "most" or "some"
- 16-section taxonomy count verified at exactly 16
- Word count benchmarks include all 3 tiers (Minimum/Standard/Extended) for every section
- Proof density defaults present for every section — no gaps
- Visual density targets present for every section — no gaps
- Sticky ATC logic captured including mobile AND desktop trigger rules
- Validation references (4-test and 10-point) loaded for Layer 3

## Failure Modes
- PDP-04-AGENT.md not found or inaccessible -> HALT. This is the master spec. Without it, no PDP-specific decisions can be made.
- Taxonomy table has fewer than 16 sections -> HALT. The NLS taxonomy is fixed at 16. A partial load means the agent file may be corrupted or incomplete.
- Word count benchmark table missing entirely -> HALT. Layer 2 word count proportioner cannot function without benchmarks.
- Proof density defaults missing -> WARNING. Layer 2 will use generic density assignments, but this reduces PDP specificity.
- Visual density targets missing -> WARNING. Layer 2 will use the 1-image-per-150-words minimum, but section-specific strategies will be generic.
