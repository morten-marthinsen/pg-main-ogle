# 3.3 — Anti-Slop Check

**Skill:** PDP-04-btf-section-sequencer
**Position:** Layer 3, Microskill 3
**Purpose:** Verify that all section descriptions, persuasion functions, and downstream instructions use specific, actionable language — not vague AI-typical filler. "Slop" is language that sounds professional but communicates nothing actionable to downstream skills. This check catches vague density plans, generic persuasion functions, and placeholder instructions before they reach the final JSON output.
**Effort Level:** medium
**Dependencies:** `optional-decisions.md` (1.4 — section descriptions), `word-count-plan.md` (2.1), `proof-density-plan.md` (2.2), `visual-density-plan.md` (2.3), `cta-placement-plan.md` (2.4), `sticky-atc-plan.md` (2.5), `dead-weight-check.md` (3.2 — updated descriptions)
**Output:** `anti-slop-check.md`

---

## Input Schema

```yaml
input:
  all_section_data: "[all Layer 1-3 outputs — descriptions, density plans, CTA plans]"
  dead_weight_updates: "[any rewritten descriptions from 3.2]"
```

## Execution Logic

**Step 1:** Define slop patterns to scan for:

### Category 1: Vague Persuasion Functions
- "Builds trust with the reader" -> Slop. HOW does it build trust? With what specific proof?
- "Provides social proof" -> Slop. What KIND of social proof? How many? From whom?
- "Educates the visitor" -> Slop. About what specific topic? To what end?
- "Adds credibility" -> Slop. Through what mechanism? What credential or data point?
- "Creates urgency" -> Slop. What specific scarcity or deadline?
- "Addresses objections" -> Slop. Which specific objections?

### Category 2: Vague Density Specs
- "Add testimonials" -> Slop. How many? What format? Text/video/photo?
- "Include images" -> Slop. How many? What type? What layout?
- "Add proof here" -> Slop. What proof type? What count?
- "Use studies" -> Slop. Which studies? How many? Citation format?
- "Add relevant visuals" -> Slop. What visuals? How many? What size?

### Category 3: Generic Instructions
- "Write compelling copy" -> Slop. What makes it compelling? What tone? What structure?
- "Create an engaging section" -> Slop. Engaging how? What element creates engagement?
- "Optimize for conversion" -> Slop. What specific conversion lever? CTA placement? Proof element?
- "Make it scannable" -> Slop. How? Bullets? Headers? Icon tiles? Accordion?
- "TBD" or "to be determined" -> Slop. Nothing should remain undetermined.

### Category 4: Inter-Section Dependencies (Module Independence Violations)
- "As mentioned above" -> Slop + violation
- "As we discussed in the ingredients section" -> Slop + violation
- "Building on the previous section" -> Slop + violation
- "Continuing from..." -> Slop + violation
- "See section X for details" -> Slop + violation

**Step 2:** Scan ALL section data — persuasion functions, density specs, notes, and instructions — for slop patterns.

**Step 3:** For each slop instance found:
- Identify the exact text
- Identify which file it came from
- Classify the slop category (1-4)
- Write a specific replacement that meets the actionability standard

**Step 4:** The actionability standard:
- Every persuasion function must answer: "What specific thing does this section do to move the visitor toward purchase?"
- Every density spec must have: type + count (not just type, not just "some")
- Every instruction must be executable by a downstream skill without asking clarifying questions
- Every section must be self-contained (no references to other sections)

**Step 5:** Score the anti-slop check:
- 0 slop instances: PASS
- 1-3 slop instances: PASS WITH CORRECTIONS (rewrite provided)
- 4+ slop instances: FAIL — indicates systemic vagueness that needs Layer 2 rework

## Quality Criteria
- Every section's persuasion function checked — not just a sample
- Every density spec (proof + visual) checked for type AND count
- Every instruction checked for actionability
- Every cross-reference / module dependency checked
- Replacements are genuinely specific (not just rephrased vagueness)
- Slop count is accurate — not inflated or deflated

## Failure Modes
- 4+ slop instances found -> FAIL. Return to Layer 2 for rework. Systemic vagueness means the planning was rushed.
- Replacement text is also vague -> The anti-slop check itself has slopped. Rewrite the replacement with specific type + count + format.
- Module independence violations found that weren't caught in 3.2 -> Add to dead-weight-check.md corrections. These must be resolved.
- All persuasion functions use the same template ("This section [verb]s the visitor by [noun]") -> WARNING. Functions should be genuinely differentiated, not templated.
- Density spec has type but no count (e.g., "testimonials" without a number) -> Slop. Add specific count.
