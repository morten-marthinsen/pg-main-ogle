# 0.3 — Pattern Library Loader

**Skill:** LP-03-above-fold-architecture
**Position:** Layer 0, Microskill 3
**Purpose:** Load the Type A/B/Hybrid above-fold pattern library from the visual hierarchy reference, providing the 5 canonical above-fold patterns that Layer 1 will select from.
**Effort Level:** low
**Dependencies:** 0.1 input-verification.md (page_type)
**Output:** `pattern-library.md`

---

## Input Schema

```yaml
input:
  page_type: "[Type A | Type B | Hybrid]"
  visual_hierarchy_ref: "04-page-builder/reference/visual-hierarchy.md"
```

---

## Execution Logic

**Step 1:** Read `input-verification.md` from 0.1. Extract `page_type`.

**Step 2:** Read `04-page-builder/reference/visual-hierarchy.md`. Extract the 5 above-fold patterns:

```
Pattern 1: VSL-First
  → Video/VSL dominates, minimal text, play button CTA
  → Best for: Warm traffic, video-first funnels, Type A

Pattern 2: Product Hero
  → Large product image, title, rating, price, ATC
  → Best for: Type B (Ecomm/PDP), branded DTC

Pattern 3: Headline + CTA + Trust
  → Bold headline, subhead, CTA button, trust bar
  → Best for: Type A long-form, cold traffic, education-first

Pattern 4: Proof-First
  → Social proof or authority credential leads, headline secondary
  → Best for: Solution-aware audience, high-credibility products

Pattern 5: Brand Hero
  → Lifestyle image dominates, brand message overlay
  → Best for: Brand-aware audience, premium positioning, Hybrid
```

**Step 3:** Filter patterns by page_type applicability:
- Type A: Patterns 1, 3, 4 are primary. Pattern 5 is secondary.
- Type B: Patterns 2, 4, 5 are primary. Pattern 3 is secondary.
- Hybrid: All 5 patterns are eligible.

**Step 4:** For each eligible pattern, document:
- Pattern name and number
- Visual hierarchy flow (what the eye sees first → second → third)
- Required elements (what MUST be above the fold for this pattern)
- Optional elements (what CAN be added)
- Traffic/awareness fit (when this pattern works best)
- Type A/B/Hybrid applicability

**Step 5:** Write `pattern-library.md` with filtered and documented patterns.

## Quality Criteria
- All 5 patterns documented with visual hierarchy flow
- Patterns correctly filtered by page_type (primary vs. secondary)
- Required vs. optional elements clearly separated per pattern
- Selection guidance connects traffic/awareness to pattern recommendation
- File exists after execution

## Failure Modes
- visual-hierarchy.md not found → Write patterns from AGENT.md definitions (fallback); note missing reference file
- page_type not recognized → HALT, verify 0.1 completed successfully
- Pattern definitions in reference file differ from AGENT.md → Use AGENT.md as authority, note discrepancy
