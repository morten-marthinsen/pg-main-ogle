# 0.1 — Input Loader

**Skill:** LP-03-above-fold-architecture
**Position:** Layer 0, Microskill 1
**Purpose:** Load LP-00 brief (page type, operating mode, product context) and LP-01 conversion intelligence (benchmarks, audience data, competitive signals) to establish the foundation for above-fold architecture decisions.
**Effort Level:** low
**Dependencies:** LP-00 outputs (page-brief.json, mode-declaration.md), LP-01 outputs (conversion-intelligence.json)
**Output:** `input-verification.md`

---

## Input Schema

```yaml
input:
  project_folder: "[path to project output folder]"
  lp00_brief: "[path to page-brief.json]"
  lp00_mode: "[path to mode-declaration.md]"
  lp01_intelligence: "[path to conversion-intelligence.json]"
```

---

## Execution Logic

**Step 1:** Locate LP-00 outputs in the project folder:
```
[project_folder]/lp-00-brief-classifier/outputs/page-brief.json
[project_folder]/lp-00-brief-classifier/outputs/mode-declaration.md
```

If either file is missing, HALT and report. Both are mandatory upstream dependencies.

**Step 2:** Extract from `page-brief.json`:
- `page_type`: Type A (Long-Form) | Type B (Ecomm/PDP) | Hybrid
- `product_name`: Product or offer name
- `vertical`: Health, supplement, finance, personal-dev, golf, technology, etc.
- `operating_mode`: Downstream (CopywritingEngine packages present) | Standalone
- `offer_summary`: 1-sentence offer description (if available)

**Step 3:** Locate LP-01 outputs in the project folder:
```
[project_folder]/lp-01-conversion-intelligence/outputs/conversion-intelligence.json
```

If missing, flag as WARNING (not blocking) — LP-03 can proceed with defaults but traffic/awareness data will be estimated.

**Step 4:** Extract from `conversion-intelligence.json` (if present):
- `traffic_sources`: List of expected traffic sources with temperature (cold/warm/hot)
- `awareness_stage`: Schwartz awareness level (Unaware / Problem-Aware / Solution-Aware / Product-Aware / Most-Aware)
- `benchmark_conversion_rate`: Target CR from industry benchmarks
- `audience_segments`: Primary and secondary audience segments
- `competitive_signals`: Key competitive page patterns observed

**Step 5:** Validate that page_type is one of the three valid values. Validate that vertical is populated. If operating_mode is Downstream, confirm at least one CopywritingEngine package reference exists.

**Step 6:** Write `input-verification.md`.

---

## Quality Criteria
- page_type extracted and validated as one of three allowed values
- All found files listed with full paths
- Missing LP-01 flagged as WARNING, not treated as blocking
- CopywritingEngine package scan performed only in Downstream mode
- File exists after execution

## Failure Modes
- LP-00 outputs not found → HALT execution, report missing upstream dependency
- page_type not one of [Type A, Type B, Hybrid] → HALT, report classification error
- LP-01 missing → Continue with WARNING, note default values will be used in Layer 1
- Project folder path invalid → Request correct path from user
