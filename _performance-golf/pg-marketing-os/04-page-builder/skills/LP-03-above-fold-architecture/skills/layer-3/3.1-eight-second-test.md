# 3.1 — Eight-Second Test

**Skill:** LP-03-above-fold-architecture
**Position:** Layer 3, Microskill 1
**Purpose:** Simulate a first-time visitor arriving at the page and determine whether the above-fold architecture answers three critical questions within 8 seconds. This is the core validation of the entire LP-03 output — if the above-fold fails this test, nothing below the fold matters.
**Effort Level:** high
**Dependencies:** All Layer 2 outputs (2.1 through 2.5, 2.6 if Type B)
**Output:** `eight-second-test.md`

---

## Input Schema

```yaml
input:
  headline_type: "[path to headline-type.md]"
  deck_direction: "[path to deck-direction.md]"
  hero_image_brief: "[path to hero-image-brief.md]"
  trust_architecture: "[path to trust-architecture.md]"
  cta_direction: "[path to cta-direction.md]"
  price_display: "[path to price-display.md, or 'N/A']"
  pattern_selection: "[path to pattern-selection.md]"
  traffic_analysis: "[path to traffic-analysis.md]"
```

## Execution Logic

**Step 1:** Read all Layer 2 output files and the pattern selection. Mentally reconstruct the above-fold section as a visitor would experience it: visual hierarchy flow from the selected pattern, headline type, deck copy direction, hero image, trust signals, CTA, and price (if Type B).

**Step 2:** Simulate visitor arrival. Assume:
- Visitor is arriving from the PRIMARY traffic source (from 1.1)
- Visitor is at the classified awareness stage (from 1.2)
- Visitor has never seen this page before
- Visitor will decide to stay or leave within 5-8 seconds
- Visitor scans (does NOT read every word) — visual hierarchy matters

**Step 3:** Answer the Three Questions:

**Question 1: "What is this?"**
- Can the visitor identify the product/offer category within 3 seconds?
- Which element answers this? (headline, hero image, product title, deck copy)
- Is the answer clear or does it require reading body copy?
- Score: CLEAR (instantly obvious) | PARTIAL (requires 2+ elements) | FAIL (ambiguous)

**Question 2: "Who is this for?"**
- Can the visitor self-select within 5 seconds?
- Which element triggers self-selection? (headline, hero image demographics, problem statement)
- Would the WRONG audience also feel this is for them? (bad self-selection)
- Score: CLEAR | PARTIAL | FAIL

**Question 3: "Why should I care right now?"**
- Is there an emotional or logical hook that creates urgency/desire within 8 seconds?
- Which element provides the "care" trigger? (headline promise, trust signal, price/savings, CTA urgency)
- Is the "why now" implicit or explicit?
- Score: CLEAR | PARTIAL | FAIL

**Step 4:** For each question, document:
- Which specific above-fold element answers it
- How many seconds into the scan the answer becomes clear
- Whether the answer requires reading (bad) or is visually communicated (good)

**Step 5:** Determine overall test result:
- 3/3 CLEAR = PASS
- 2/3 CLEAR + 1 PARTIAL = CONDITIONAL PASS — document the gap
- Any FAIL = TEST FAILED — document which question failed and why

**Step 6:** If any question scores PARTIAL or FAIL, write specific remediation recommendations:
- Which Layer 2 output needs revision
- What specific change would fix the gap
- Priority level of the fix

**Step 7:** Write `eight-second-test.md`.

## Quality Criteria
- All 3 questions answered with specific element attribution (not vague)
- Time-to-answer estimates are realistic (not "0.5 seconds" for everything)
- FAIL results include actionable remediation tied to specific Layer 2 outputs
- Visual hierarchy walkthrough traces actual eye path for selected pattern
- Test scenario matches the classified traffic/awareness from Layer 1
- File exists after execution

## Failure Modes
- Layer 2 outputs incomplete → HALT, complete all required Layer 2 microskills first
- All questions scored CLEAR without critical analysis → Suspect insufficient rigor; re-examine with adversarial mindset
- Remediation recommendations are vague ("improve the headline") → REJECT, must specify what to change in which output file
- Visual hierarchy walkthrough skipped → REJECT, this is required for understanding the scan path
