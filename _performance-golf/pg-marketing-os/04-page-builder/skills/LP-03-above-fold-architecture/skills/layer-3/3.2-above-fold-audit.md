# 3.2 — Above-Fold Audit

**Skill:** LP-03-above-fold-architecture
**Position:** Layer 3, Microskill 2
**Purpose:** Score the above-fold architecture against the 8-point checklist from AGENT.md. This is a binary pass/fail gate — minimum 7.0/8 required to proceed to Layer 4 packaging. Scores below 7.0 require revision of specific Layer 2 outputs.
**Effort Level:** medium
**Dependencies:** All Layer 2 outputs, 3.1 eight-second-test.md
**Output:** `above-fold-audit.md`

---

```yaml
input:
  headline_type: "[path to headline-type.md]"
```

## Execution Logic

**Step 1:** Read all Layer 2 outputs and the 8-second test results. These are the inputs being audited.

**Step 2:** Score each of the 8 audit points. Each point is scored 0 (fail) or 1 (pass):

**IDENTITY & PROMISE (2 points)**

**Point 1: Is the core product/offer identifiable in under 3 seconds?**
- Check: headline-type.md — does the selected type communicate product/offer identity?
- Check: hero-image-brief.md — does the image reinforce product recognition?
- Check: eight-second-test.md Q1 result
- Score 1 if: Q1 scored CLEAR
- Score 0 if: Q1 scored PARTIAL or FAIL

**Point 2: Does the headline make a specific, credible promise (not generic)?**
- Check: headline-type.md — is the type selection connected to a specific benefit/revelation/proof?
- Check: deck-direction.md — does the deck expand a specific promise?
- Score 1 if: Headline type + deck direction reference specific outcomes, numbers, or mechanisms
- Score 0 if: Direction is vague ("improve your health," "change your life")

**AUDIENCE & SELF-SELECTION (2 points)**

**Point 3: Does the above-fold section self-select the right audience?**
- Check: headline-type.md — does the type signal "this is for YOU"?
- Check: hero-image-brief.md — does the image demographic match target audience?
- Check: eight-second-test.md Q2 result
- Score 1 if: Q2 scored CLEAR or PARTIAL with strong self-selection element
- Score 0 if: Q2 scored FAIL or above-fold is generic enough for anyone

**Point 4: Would the WRONG audience immediately recognize this isn't for them?**
- Check: eight-second-test.md Q2 wrong-audience filter assessment
- Score 1 if: Wrong-audience filter rated "Strong"
- Score 0 if: Filter rated "Weak" — page could attract unqualified traffic

**TRUST & CREDIBILITY (2 points)**

**Point 5: Is there at least ONE trust signal visible above the fold?**
- Check: trust-architecture.md — are signals selected and positioned above fold?
- Score 1 if: At least 1 trust signal is in the above-fold architecture
- Score 0 if: No trust signals above the fold (all below)

**Point 6: Is the trust signal specific (not generic)?**
- Check: trust-architecture.md — specificity check section
- Score 1 if: All above-fold trust signals include specific claims, numbers, or names
- Score 0 if: Any signal uses generic language ("High Quality," "Satisfaction Guaranteed")

**CONVERSION READY (2 points)**

**Point 7: Is the primary CTA visible on first screen?**
- Check: cta-direction.md — CTA designed for above-fold placement
- Check: pattern-selection.md — selected pattern includes CTA in visual hierarchy
- Score 1 if: CTA is part of the above-fold architecture (desktop minimum, mobile required)
- Score 0 if: CTA is below the fold or dependent on scrolling

**Point 8: Is the visual hierarchy clear — Headline -> Image/Trust -> CTA?**
- Check: eight-second-test.md visual hierarchy walkthrough
- Check: pattern-selection.md visual hierarchy flow
- Score 1 if: Clear 3-step scan path documented and logical
- Score 0 if: Competing elements, unclear scan path, or CTA buried

**Step 3:** Calculate total score (/8) and apply the scoring rubric:
- 8/8: Exceptional — proceed to Layer 4
- 7/8: Good — proceed to Layer 4 with note on missed point
- 6/8: Good — proceed to Layer 4, but flag the 2 missed points for LP-07 awareness
- 5/8: Needs revision — identify which Layer 2 outputs to revise before proceeding
- Below 5: Structural rethink required — return to Layer 1

**Step 4:** For any point scoring 0, write a specific remediation note identifying which Layer 2 output needs revision and what specific change would flip the score to 1.

**Step 5:** Write `above-fold-audit.md`.

## Quality Criteria
- All 8 points scored with specific evidence (not just "pass" or "fail")
- Evidence references actual content from Layer 2 output files
- Remediation notes are actionable and tied to specific files
- Scoring is honest — do not inflate to avoid revision
- Minimum 7/8 to proceed; enforce this gate strictly
- File exists after execution

## Failure Modes
- Score inflated to avoid revision work → Auditor must score honestly; if quality is below threshold, revision is cheaper than a failed page
- Remediation notes are vague → REJECT, must specify file + change
- 8-second test not completed before audit → HALT, run 3.1 first
- All 8 points score 1 without substantive evidence → Suspect rubber-stamping; verify each claim against actual output content
