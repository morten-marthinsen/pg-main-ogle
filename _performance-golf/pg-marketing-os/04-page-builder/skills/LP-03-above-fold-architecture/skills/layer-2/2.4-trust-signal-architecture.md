# 2.4 — Trust Signal Architecture

**Skill:** LP-03-above-fold-architecture
**Position:** Layer 2, Microskill 4
**Purpose:** Select, sequence, and brief the trust signals that appear above the fold. Trust signals are the fastest shortcut to credibility — the wrong selection (or generic signals) actively damages conversion.
**Effort Level:** high
**Dependencies:** 1.3 pattern-selection.md, 1.2 awareness-map.md, 0.1 input-verification.md, 0.2 specimen-load.md
**Output:** `trust-architecture.md`

## Execution Logic

**Step 1:** Read dependency files. Extract page type, vertical, awareness stage, selected pattern, and specimen trust signal patterns.

**Step 2:** Reference the Trust Signal Types ranked by persuasiveness from AGENT.md:

| Rank | Signal Type | Example |
|------|------------|---------|
| 1 | Clinical study citation | "In a double-blind study, 87% of participants..." |
| 2 | Medical authority | "Developed with Dr. [Name], MD, Harvard Medical School" |
| 3 | Named media coverage | "As featured in Forbes, CNN, Dr. Oz Show" |
| 4 | Third-party testing | "Third-Party Tested / GMP Certified / NSF Approved" |
| 5 | Volume social proof | "47,328 customers served" or "4.8 (2,847 reviews)" |
| 6 | Money-back guarantee badge | "[90-Day] [Branded Guarantee Name]" |
| 7 | Security/payment badges | "Secured by SSL / Powered by Stripe" |

**Step 3:** Apply page type selection rules:

**Type A (Long-Form):**
- Select 2-3 signals maximum for above-fold trust bar
- Prioritize: media coverage + certification + authority (Ranks 1-3)
- Position: Immediately below headline OR as horizontal bar above headline
- Avoid cluttering — Type A trust builds through copy below the fold

**Type B (Ecomm/PDP):**
- Rating strip: ALWAYS immediately below product title (Rank 5)
- Trust badge row: Below ATC button (Ranks 4, 6, 7)
- Select 3-4 specific badges
- Trust bar pattern: [Certification] [Guarantee] [Shipping] [Security]

**Hybrid:**
- Blend: Rating strip (if reviews exist) + 2 trust bar signals
- Position varies by pattern selection

**Step 4:** For each selected trust signal, write a specificity brief:
- GENERIC (FORBIDDEN): "100% Satisfaction Guaranteed"
- SPECIFIC (REQUIRED): "90-Day Full Refund Guarantee — No Questions, No Hassle"
- GENERIC (FORBIDDEN): "High Quality Ingredients"
- SPECIFIC (REQUIRED): "Third-Party Tested by Eurofins / GMP Certified Facility"

Every trust signal brief must include the specific claim, number, name, or credential. Zero generic signals allowed.

**Step 5:** Determine sequence order (visual reading order, left-to-right or top-to-bottom):
- Lead with the STRONGEST signal (highest rank available)
- Follow with supporting signals in descending persuasiveness
- End with utility signals (security, shipping) for Type B

**Step 6:** Flag any trust signals that require verification:
- Clinical study claims need source
- Doctor/authority names need confirmation
- Media coverage needs confirmation of actual coverage
- Review counts need to be real numbers

**Step 7:** Write `trust-architecture.md`.

## Quality Criteria
- Signal count matches page type rules (2-3 for Type A, 3-4 for Type B)
- EVERY signal includes specific text — zero generic trust signals
- Signals ranked and sequenced by persuasiveness
- Verification flags set for any claim that needs source confirmation
- Health/supplement pages MUST include at least one certification signal (Rank 4)
- File exists after execution

## Failure Modes
- All signals are generic ("Satisfaction Guaranteed," "Premium Quality") → REJECT entire output, rewrite with specifics
- More than 3 signals for Type A → Reduce to top 3 by rank
- Health product with zero certification signals → FLAG as critical gap, recommend adding
- No product data available for specific claims → Write signal templates with [PLACEHOLDER] brackets, flag for human input
- pattern-selection.md not found → HALT, run 1.3 first
