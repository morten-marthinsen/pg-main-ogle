# 2.3 — Hero Image Brief

**Skill:** LP-03-above-fold-architecture
**Position:** Layer 2, Microskill 3
**Purpose:** Write the creative brief for the hero visual (the dominant image or video thumbnail above the fold). The hero image is the single most visually dominant element and must reinforce the headline promise, not merely decorate.
**Effort Level:** medium
**Dependencies:** 1.3 pattern-selection.md, 2.1 headline-type.md, 0.1 input-verification.md, 0.2 specimen-load.md
**Output:** `hero-image-brief.md`

## Execution Logic

**Step 1:** Read dependency files. Extract selected pattern, headline type, product context, and specimen hero image approaches.

**Step 2:** Determine hero image category based on page type and pattern:

```
PRODUCT HERO: Product bottle/box/device as primary subject
  → When: Type B (always), Hybrid (usually), Type A (rarely)
  → Required elements: Product at correct angle, label readable, context props

LIFESTYLE: Person experiencing the benefit/transformation
  → When: Type A (common), Hybrid (sometimes)
  → Required: Demographic match to target audience, authentic setting

AUTHORITY: Expert/doctor/creator as credibility anchor
  → When: Type A with medical/expert positioning, T6 headline type
  → Required: Professional setting, credentials visible or implied

PROBLEM VISUALIZATION: The pain/problem the product solves
  → When: T3 or T4 headline type, problem-aware audience
  → Required: Relatable scenario, NOT stock-photo quality

VIDEO THUMBNAIL: Still frame designed to trigger play action
  → When: Pattern 1 (VSL-First), video-first funnel
  → Required: Play button overlay, curiosity-generating frame, NOT a blank player

BEFORE/AFTER: Transformation comparison
  → When: Health/fitness vertical, T1 or T7 headline type
  → Required: Authentic images, same person, clear visual difference
```

**Step 3:** Write the creative brief with 5 mandatory specifications:

1. **Context:** What environment/setting should the image depict?
2. **Emotion:** What feeling should the viewer experience? (confidence, curiosity, alarm, desire, trust)
3. **Subject:** What is the primary subject? (product, person, scene, comparison)
4. **Setting:** Where does this scene take place? (kitchen, gym, doctor's office, outdoor, studio)
5. **Format:** Image dimensions, orientation, any text overlay requirements

**Step 4:** Add pattern-specific requirements:
- Pattern 2 (Product Hero): Image MUST show product large enough to read label. White/clean background preferred for Type B. Lifestyle background for premium positioning.
- Pattern 3 (Headline+CTA+Trust): Image is SECONDARY to headline. Keep it supporting, not competing.
- Pattern 5 (Brand Hero): Image is PRIMARY — full-width, lifestyle-dominant, brand feel over product detail.

**Step 5:** Document what the hero image must NOT be:
- NOT a generic stock photo
- NOT "just a product photo" without context
- NOT a person who doesn't match the target demographic
- NOT an image that contradicts the headline promise

**Step 6:** Reference specimen hero approaches from 0.2 as benchmarks. Note which specimen images work and why.

**Step 7:** Write `hero-image-brief.md`.

---

## Quality Criteria
- All 5 creative brief elements (context, emotion, subject, setting, format) specified
- Image category justified by page type and pattern selection
- Brief is specific enough for a designer/photographer to execute without guessing
- "Do NOT" section prevents the most common hero image failures
- NEVER outputs "product photo" as the entire brief — minimum 5 specifications
- File exists after execution

## Failure Modes
- Brief is too vague ("show the product") → REJECT, add context/emotion/setting details
- Image category contradicts selected pattern → Resolve conflict, pattern wins
- No specimen references available → Note gap, provide the brief based on vertical best practices
- pattern-selection.md not found → HALT, run 1.3 first
