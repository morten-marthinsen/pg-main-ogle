# 1.3 — Above-Fold Pattern Selector

**Skill:** LP-03-above-fold-architecture
**Position:** Layer 1, Microskill 3
**Purpose:** Select the single best above-fold pattern from the 5-pattern library based on traffic temperature, awareness stage, page type, and specimen evidence. This selection governs the entire Layer 2 element design.
**Effort Level:** high
**Dependencies:** 0.2 specimen-load.md, 0.3 pattern-library.md, 1.1 traffic-analysis.md, 1.2 awareness-map.md
**Output:** `pattern-selection.md`

---

## Input Schema

```yaml
input:
  pattern_library: "[path to pattern-library.md]"
  specimen_load: "[path to specimen-load.md]"
  traffic_analysis: "[path to traffic-analysis.md]"
  awareness_map: "[path to awareness-map.md]"
  page_type: "[Type A | Type B | Hybrid]"
```

---

## Execution Logic

**Step 1:** Read all four dependency files. Extract:
- Eligible patterns (from 0.3) with primary/secondary classification
- Specimen pattern distribution (from 0.2) — what patterns do real pages in this vertical use?
- Traffic temperature (from 1.1) and headline constraints
- Awareness stage (from 1.2) and copy density requirement

**Step 2:** Score each eligible pattern against 5 criteria (each 0-2 points, max 10):

| Criterion | 0 | 1 | 2 |
|-----------|---|---|---|
| **Page type fit** | Pattern is secondary for this type | Pattern works but not ideal | Pattern is primary for this type |
| **Traffic temperature fit** | Pattern misaligns with traffic temp | Partial alignment | Direct alignment |
| **Awareness stage fit** | Pattern assumes wrong awareness level | Workable with adaptation | Natural fit for awareness level |
| **Specimen evidence** | No specimens use this pattern in vertical | Some specimens use it | Most specimens in vertical use it |
| **Asset availability** | Required assets unlikely to exist | Some assets may exist | Core assets likely available |

**Step 3:** Rank patterns by total score. The highest-scoring pattern is the PRIMARY recommendation. The second-highest is the ALTERNATIVE.

**Step 4:** Validate the selection:
- Type B pages MUST have product image + price + ATC above fold → Pattern 2 (Product Hero) is almost always primary
- Type A pages with cold traffic MUST prioritize headline → Pattern 3 (Headline+CTA+Trust) is default
- If VSL is the primary content asset → Pattern 1 (VSL-First) overrides other signals
- If the strongest proof is authority/celebrity → Pattern 4 (Proof-First) gets bonus

**Step 5:** Document which Layer 2 microskills are required vs. optional based on the selected pattern:
- Pattern 1 (VSL-First): 2.1 required, 2.2 optional, 2.3 required (thumbnail), 2.4 minimal, 2.5 required, 2.6 N/A
- Pattern 2 (Product Hero): All 2.1-2.6 required
- Pattern 3 (Headline+CTA+Trust): 2.1 required, 2.2 required, 2.3 required, 2.4 required, 2.5 required, 2.6 N/A (Type A)
- Pattern 4 (Proof-First): 2.1 required, 2.2 required, 2.3 secondary, 2.4 CRITICAL, 2.5 required, 2.6 conditional
- Pattern 5 (Brand Hero): 2.1 required, 2.2 optional, 2.3 CRITICAL, 2.4 minimal, 2.5 required, 2.6 conditional

**Step 6:** Write `pattern-selection.md`.

## Quality Criteria
- All eligible patterns scored against all 5 criteria with specific justification
- Selected pattern is highest-scoring unless an override condition applies
- Override conditions explicitly checked and documented
- Layer 2 requirements table populated based on selected pattern
- Alternative pattern identified for fallback
- File exists after execution

## Failure Modes
- All patterns score below 4/10 → Flag for human review, document why no pattern fits well
- Tie between two patterns → Select the one with higher specimen evidence score (real-world validation wins)
- Missing dependency files → HALT, run prerequisite microskills
- Override condition contradicts scoring → Override wins, document clearly why
