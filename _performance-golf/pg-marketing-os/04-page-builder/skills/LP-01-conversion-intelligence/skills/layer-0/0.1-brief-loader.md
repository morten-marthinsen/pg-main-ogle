# 0.1: Brief Loader

> **Skill:** LP-01-conversion-intelligence
> **Layer:** 0
> **Position:** 1 of 3
> **Purpose:** Load page-brief.json from LP-00 and extract the classification fields that govern all downstream matching and filtering.
> **Effort Level:** haiku
> **Dependencies:** page-brief.json must exist (LP-00 must have completed)
> **Output:** `brief-extract.md`

---

## INPUT

```yaml
input:
  page_brief_path: "[path to page-brief.json from LP-00 output]"
```

---

## EXECUTION

1. **Locate page-brief.json.** Search in the project's LP-00 output directory. If not found, HALT with error: "LP-01 requires page-brief.json from LP-00. Run LP-00 first."

2. **Load and parse the JSON file.** If the file is malformed or unreadable, HALT with error: "page-brief.json is malformed. Re-run LP-00."

3. **Extract these classification fields:**
   - `page_type.classification` (type_a, type_b, or hybrid)
   - `vertical.primary` (vertical_id)
   - `vertical.secondary` (vertical_id or null)
   - `traffic.temperature` (cold, warm, or hot)
   - `traffic.awareness_stage` (1-5)
   - `traffic.primary_source` (e.g., facebook_cold, email_list)
   - `operating_mode` (downstream or standalone)
   - `project_name`
   - `offer_summary.product_category` (if available)
   - `audience.sophistication.skepticism_level` (if available)
   - `proof_inventory` summary (if available)

4. **Validate extracted fields.** Each classification field must be non-empty. If `page_type.classification` or `vertical.primary` is empty or missing, HALT — these are required for all downstream matching.

5. **Write brief-extract.md** with all extracted fields in a structured format.

---

## OUTPUT SCHEMA

`brief-extract.md`:

```markdown
# Brief Extract — [Project Name]

## Source
- File: [path to page-brief.json]
- LP-00 Operating Mode: [downstream | standalone]

## Classification Fields (Required)
- Page Type: [type_a | type_b | hybrid]
- Primary Vertical: [vertical_id]
- Secondary Vertical: [vertical_id | null]
- Traffic Temperature: [cold | warm | hot]
- Awareness Stage: [1-5 — name]
- Traffic Source: [primary source]

## Supplementary Fields (If Available)
- Product Category: [category | not_available]
- Skepticism Level: [level | not_available]
- Proof Inventory Available: [Y/N]
- Proof Inventory Summary: [brief summary | not_available]

## Validation
- page_type extracted: [Y/N]
- vertical extracted: [Y/N]
- traffic_temperature extracted: [Y/N]
- ALL REQUIRED FIELDS PRESENT: [Y/N]
```

---

## QUALITY CRITERIA

- All three required classification fields extracted: page_type, vertical.primary, traffic.temperature
- Fields match expected enum values (not free-text or hallucinated categories)
- page-brief.json path documented for traceability
- File exists after execution

---

## FAILURE MODES

- **page-brief.json not found:** HALT — LP-00 must complete first. Do not proceed without the brief.
- **JSON parse error:** HALT — page-brief.json is malformed. Re-run LP-00.
- **Missing required fields:** HALT — page_type, vertical.primary, or traffic.temperature missing from the brief. LP-00 may have failed its GATE_2.
- **Vertical not in 8-option list:** Flag as warning but proceed — may indicate a new vertical not yet in the taxonomy.
