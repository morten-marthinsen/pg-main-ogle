# 3.2: Completeness Audit

> **Skill:** LP-01-conversion-intelligence
> **Layer:** 3
> **Position:** 2 of 2
> **Purpose:** Verify that all required fields in the conversion-intelligence.json schema are populated, calculate the data confidence score, and confirm the output package is ready for assembly.
> **Effort Level:** sonnet
> **Dependencies:** All Layer 2 outputs (`benchmark-synthesis.md`, `specimen-analysis.md`, `filtered-patterns.md`, `strategic-context.md`), `element-impact-ranking.md` (from 1.4), `validation-report.md` (from 3.1)
> **Output:** `completeness-audit.md`

---

## INPUT

```yaml
input:
  benchmark_synthesis: "benchmark-synthesis.md"
  specimen_analysis: "specimen-analysis.md"
  filtered_patterns: "filtered-patterns.md"
  strategic_context: "strategic-context.md"
  element_impact_ranking: "element-impact-ranking.md"

  brief_extract: "brief-extract.md"
```

---

## EXECUTION

1. **Verify validation report passed.** Read validation-report.md. If result is FAIL, HALT — validation must pass before completeness audit.

2. **Check each required section of the conversion-intelligence.json schema:**

   ```
   [ ] classification_context — all 5 fields populated
   [ ] vertical_benchmarks — conversion_rate_range populated with citation
   [ ] page_type_benchmarks — target_conversion_rate populated with citation
   [ ] element_level_impact — at minimum 5 impact entries with sources
   [ ] matched_specimens — at minimum 2 entries (or gap documented)
   [ ] element_priority_ranking — at minimum 8 elements ranked
   [ ] cross_page_patterns — section_sequence_pattern populated
   [ ] cross_page_patterns — proof_architecture_pattern populated
   [ ] cross_page_patterns — cta_architecture_pattern populated
   [ ] strategic_context — lp_03_above_fold_guidance populated
   [ ] strategic_context — lp_04_section_sequence_guidance populated
   [ ] strategic_context — lp_06_offer_cta_guidance populated
   [ ] data_confidence — overall_score calculated
   [ ] downstream_handoffs — all 5 handoffs documented
   ```

3. **For any missing field:**
   - Identify which Layer 1 or Layer 2 output should have provided it
   - Record the gap
   - If the field can be legitimately marked `not_available` (data genuinely doesn't exist), document why
   - If the field should have been populated but wasn't, mark as INCOMPLETE

4. **Calculate Data Confidence Scores (1-10 each):**

   **Benchmark Confidence:**
   - 9-10: Vertical-specific AND page-type-specific data with direct citations
   - 7-8: Vertical-specific OR page-type-specific data with direct citations
   - 5-6: Adjacent vertical/page-type data used
   - 3-4: Only universal benchmarks available
   - 1-2: Minimal benchmark data from any source

   **Specimen Match Confidence:**
   - 9-10: 3+ primary matches (page_type + vertical) with HIGH specimen value
   - 7-8: 2+ primary matches
   - 5-6: Secondary/tertiary matches needed
   - 3-4: Fallback matches only
   - 1-2: Zero or 1 weak match

   **Element Ranking Confidence:**
   - 9-10: 12+ elements ranked with quantified impact data
   - 7-8: 8-11 elements ranked with mix of quantified and observed impact
   - 5-6: 8+ elements ranked but limited impact data
   - 3-4: Fewer than 8 elements ranked
   - 1-2: Minimal element data available

   **Pattern Confidence:**
   - 9-10: 3+ specimens from this page type + vertical in cross-page analysis
   - 7-8: 2+ relevant specimens in cross-page analysis
   - 5-6: Patterns inferred from related page types/verticals
   - 3-4: Limited pattern data available
   - 1-2: Minimal pattern data

   **Overall Score:** Average of 4 component scores, rounded to nearest integer.

5. **Write completeness-audit.md** with all checks and confidence scores.

## QUALITY CRITERIA

- Every required schema field explicitly checked (not assumed)
- Confidence scores have rationale, not just numbers
- Incomplete fields traced to their source layer/microskill
- PASS/FAIL decision is binary and justified
- Overall confidence score is an honest average (not inflated)

---

## FAILURE MODES

- **Validation report says FAIL:** HALT immediately. Do not attempt completeness audit on unvalidated data.
- **Multiple fields INCOMPLETE:** If 3+ required fields are missing, this suggests a Layer 1 or Layer 2 failure. Return to the relevant layer rather than marking everything `not_available`.
- **All confidence scores below 5:** The data foundation is insufficient. Document this clearly but proceed — downstream skills will fall back to MASTER-BLUEPRINT.md defaults. The intelligence package is still valuable even at low confidence as it documents what data IS available and what gaps exist.
