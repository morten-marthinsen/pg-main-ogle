# 4.2: Summary Writer

> **Skill:** LP-01-conversion-intelligence
> **Layer:** 4
> **Position:** 2 of 3
> **Purpose:** Write a human-readable CONVERSION-INTELLIGENCE-SUMMARY.md from conversion-intelligence.json, following the required sections template defined in LP-01-ANTI-DEGRADATION.md.
> **Effort Level:** haiku
> **Dependencies:** `conversion-intelligence.json` (from 4.1)
> **Output:** `CONVERSION-INTELLIGENCE-SUMMARY.md`

---

## INPUT

```yaml
input:
  intelligence_json: "conversion-intelligence.json"
```

---

## EXECUTION

1. **Read conversion-intelligence.json.**

2. **Write CONVERSION-INTELLIGENCE-SUMMARY.md** using the exact template from LP-01-ANTI-DEGRADATION.md. Do not deviate from the required sections.

3. **Section-by-section compilation:**

   **Classification Context:**
   - Page Type, Vertical, Traffic, Awareness — plain English, one line each

   **Key Benchmarks:**
   - Target CVR range (floor / target / stretch) with source citation
   - Bounce rate target
   - Page speed target (desktop + mobile)
   - Key element-level impacts (top 3-5 by relevance)
   - All citations in parentheses after the stat

   **Matched Specimens:**
   - For each specimen: ID, name, match score, 1-line key pattern, 1-line relevance to this build
   - If fewer than 2 matched, note the gap

   **Element Priority Ranking (Top 10):**
   - Ranked table: rank, element name, required Y/N, impact data
   - Separate "vertical-specific requirements" if applicable

   **Cross-Page Patterns (Filtered):**
   - Section sequence pattern (the skeleton)
   - Proof architecture decision (proof-saturated vs. authority-substitution)
   - CTA architecture (count, placement, text pattern)
   - Headline type recommendation

   **Strategic Context (per downstream skill):**
   - LP-03: Above-fold guidance (2-4 bullet points)
   - LP-04: Section sequence guidance (2-4 bullet points)
   - LP-05: Social proof guidance (2-4 bullet points)
   - LP-06: Offer/CTA guidance (2-4 bullet points)

   **Data Confidence:**
   - Overall score with rationale
   - Key gaps listed

   **Downstream Handoff:**
   - Which skills load this file and what to prioritize

4. **Formatting rules:**
   - Use markdown headers (##, ###)
   - Use bullet points for lists
   - Use tables for ranked/structured data
   - Keep language concise — this is a summary, not a report
   - Total length target: 150-300 lines

## QUALITY CRITERIA

- All required sections from ANTI-DEGRADATION.md template present
- Every benchmark includes a parenthetical source citation
- Specimen table matches conversion-intelligence.json exactly
- Element ranking matches conversion-intelligence.json exactly
- Strategic context is actionable (not vague)
- Data confidence score and gaps match the JSON
- File is readable as a standalone document (someone who hasn't seen the JSON can understand it)

---

## FAILURE MODES

- **conversion-intelligence.json not found or invalid:** HALT — 4.1 must complete first.
- **Section missing from JSON:** Write the section anyway with "Data not available — [field] was not populated in the intelligence package."
- **Summary exceeds 400 lines:** Trim. This is a summary, not a full report. Downstream skills read the JSON for detail.
