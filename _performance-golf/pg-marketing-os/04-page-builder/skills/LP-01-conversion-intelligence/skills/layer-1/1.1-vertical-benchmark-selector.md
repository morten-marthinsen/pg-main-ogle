# 1.1: Vertical Benchmark Selector

> **Skill:** LP-01-conversion-intelligence
> **Layer:** 1
> **Position:** 1 of 4
> **Purpose:** Select all benchmarks from conversion-data-reference.md that apply to the primary vertical identified in the brief, filtering out data from other verticals.
> **Effort Level:** sonnet
> **Dependencies:** `brief-extract.md` (from 0.1), `data-reference-load.md` (from 0.2)
> **Output:** `vertical-benchmarks.md`

---

## INPUT

```yaml
input:
  brief_extract: "brief-extract.md"
  data_reference_load: "data-reference-load.md"
  conversion_data_reference: "04-page-builder/conversion-data-reference.md"
  primary_vertical: "[extracted from brief-extract.md]"
  secondary_vertical: "[extracted from brief-extract.md, may be null]"
```

---

## EXECUTION

1. **Read the primary vertical** from brief-extract.md. This is the anchor for all benchmark selection.

2. **Scan conversion-data-reference.md for vertical-specific data.** Check these locations:
   - **Industry-Specific Rates table** — look for a row matching the vertical
   - **Vertical-Specific Benchmarks section** — look for a dedicated sub-section (currently exists for: Health/Supplement, Info Products)
   - **Social Proof Thresholds** — awareness-based placement timing data
   - **Guarantee Data** — if vertical has specific return rate data
   - **CTA Data** — performance patterns applicable to this vertical

3. **Extract all matching benchmarks.** For each benchmark extracted, record:
   - The exact stat (number, range, or percentage)
   - The exact source citation (section name + table/rule reference in conversion-data-reference.md)
   - Whether this is a direct vertical match or an adjacent-vertical inference

4. **If primary vertical has NO dedicated section** in conversion-data-reference.md:
   - Check if secondary vertical has data
   - Use the Industry-Specific Rates table for the closest match
   - Clearly document that this is a "closest available" match, not a direct vertical match
   - Note the gap

5. **Compile all vertical-specific benchmarks** into a structured output. Minimum 2 benchmarks required; target 4-6.

6. **Do NOT include universal benchmarks** (those go in 1.2). This file contains ONLY vertical-filtered data.

---

## QUALITY CRITERIA

- Every benchmark has an explicit citation to a specific section in conversion-data-reference.md
- No benchmark invented from general knowledge
- Vertical match quality honestly assessed (direct, adjacent, or none)
- Data confidence reflects actual availability
- Minimum 2 benchmarks extracted (or gap explicitly documented)
- Universal/non-vertical data excluded (that's for 1.2)

---

## FAILURE MODES

- **No vertical data at all:** Document the complete gap. Set data confidence to LOW. Proceed — 1.2 will provide universal benchmarks. Note in gaps section for strategic-context.md.
- **Vertical ID doesn't match any category in reference:** Use the closest vertical category. Document the inference chain (e.g., "beauty-wellness" → closest to "health-supplements" for benchmark purposes).
- **Ambiguous data match:** When a benchmark could apply to this vertical but isn't explicitly scoped to it, include it but mark as "adjacent inference" not "direct match."
