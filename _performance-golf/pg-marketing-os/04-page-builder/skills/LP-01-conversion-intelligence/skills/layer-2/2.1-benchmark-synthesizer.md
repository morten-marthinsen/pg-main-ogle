# 2.1: Benchmark Synthesizer

> **Skill:** LP-01-conversion-intelligence
> **Layer:** 2
> **Position:** 1 of 4
> **Purpose:** Merge vertical-specific benchmarks and page-type benchmarks into a unified benchmark package with traffic-temperature-aware targets, resolving any conflicts between data sources.
> **Effort Level:** sonnet
> **Dependencies:** `vertical-benchmarks.md` (from 1.1), `page-type-benchmarks.md` (from 1.2), `brief-extract.md` (from 0.1)
> **Output:** `benchmark-synthesis.md`

---

## INPUT

```yaml
input:
  vertical_benchmarks: "vertical-benchmarks.md"
  page_type_benchmarks: "page-type-benchmarks.md"
  brief_extract: "brief-extract.md"
  page_type: "[type_a | type_b | hybrid]"
  primary_vertical: "[vertical_id]"
  traffic_temperature: "[cold | warm | hot]"
  awareness_stage: "[1-5]"
```

---

## EXECUTION

1. **Read both benchmark files** (vertical-benchmarks.md and page-type-benchmarks.md).

2. **Identify overlap and conflict resolution:**
   - When vertical-specific data exists alongside universal data, vertical-specific ALWAYS takes priority.
   - When page-type-specific data exists alongside universal data, page-type-specific takes priority.
   - When vertical and page-type data conflict (rare), use the narrower/more-specific data.
   - Document every conflict and how it was resolved.

3. **Synthesize the target conversion rate range:**
   - Start with vertical-specific CVR range (from 1.1) for the correct traffic temperature
   - Narrow with page-type-specific expectations
   - Produce a single target range with "floor" (minimum acceptable), "target" (good), and "stretch" (top performer)
   - Example: Health/Supplements cold traffic → Floor: 2%, Target: 4-8%, Stretch: 12%+

4. **Synthesize speed targets:**
   - Use page-type-specific speed targets from 1.2
   - Mobile always gets the stricter target
   - Add the -7% per second context

5. **Synthesize CTA targets:**
   - Merge CTA placement data with page-type expectations
   - Set a CTA count range based on page type + section count expectations
   - Include CTA copy pattern guidance (personalized = +202%)

6. **Synthesize proof targets:**
   - Merge social proof thresholds with awareness-stage-based placement timing
   - Set minimum proof volume targets based on page type
   - Include proof type recommendations based on vertical

7. **Synthesize offer/guarantee targets:**
   - Merge guarantee data with vertical-specific claim rates
   - Include offer type recommendations from the 7-type hierarchy

8. **Calculate a synthesized benchmark confidence:**
   - HIGH: Both vertical-specific AND page-type-specific data available
   - MEDIUM: One of vertical or page-type data is from adjacent/inferred sources
   - LOW: Both sources are adjacent/inferred

9. **Write benchmark-synthesis.md** with the unified package.

---

## QUALITY CRITERIA

- Every synthesized target traces back to at minimum one source benchmark
- Conflicts between data sources documented and resolved
- Traffic temperature influences the CVR target range (cold targets lower than warm)
- CTA personalization requirement (+202%) always included
- One Offer Per Page rule (-266%) always included
- Confidence level honestly reflects data availability
- No new statistics introduced that weren't in the input files

---

## FAILURE MODES

- **Conflicting data with no clear resolution:** Document both numbers, use the more conservative (lower CVR, stricter speed) as the target. Flag for human review.
- **No vertical-specific CVR data:** Use universal median (6.6%) as a reference point but clearly label it as non-vertical-specific. Set CVR confidence to LOW.
- **Hybrid page synthesis:** Produce two sets of targets where relevant — above-fold zone (Type B targets) and below-fold zone (Type A targets). Clearly label which targets apply to which zone.
