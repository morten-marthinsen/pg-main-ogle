# 2.5 — Mini-Cart Copy

**Skill:** PDP-06-offer-buybox-writer
**Position:** Layer 2, Microskill 5
**Purpose:** Write all copy for the slide-out mini-cart that appears after the user clicks "Add to Cart". Contains: product name, variant selected, quantity, price, subtotal, "Continue Shopping" link, "Checkout" button, and a single cross-sell suggestion line. The mini-cart keeps users on the PDP (not a cart page redirect).
**Effort Level:** medium
**Dependencies:** 0.1 input-verification.md (product name), 0.3 blueprint-extraction.md (mini-cart spec from PDP-03), 2.1 atc-copy.md (post-click behavior confirmation), 1.1 pricing-copy.md (price)
**Output:** `mini-cart-copy.md`

## Execution Logic

**Step 1:** Read `input-verification.md`. Extract:
- Full product name
- Product image reference (for mini-cart thumbnail)
- Offer structure (for cross-sell context)

**Step 2:** Read `blueprint-extraction.md`. Extract PDP-03's mini-cart spec:
- Slide-out direction (right side is standard)
- Cross-sell strategy (if specified)
- Mini-cart elements listed

**Step 3:** Confirm post-click behavior from `atc-copy.md`: slide-out mini-cart (not cart redirect).

**Step 4:** Write the mini-cart header:
```
Text: "Cart ([X])" or "Your Cart ([X])"
```
- [X] = number of items in cart
- Header appears at top of slide-out panel

**Step 5:** Write the cart item display:
```
[Product Thumbnail]
[Full Product Name]
[Variant: Selected Variant Name]
[Qty: [stepper] X]
[$XX.XX]
```
- Product name: full, never truncated
- Variant: only shown if product has variants
- Quantity: editable stepper (matches main buy box stepper pattern)
- Price: per-item price

**Step 6:** Write the cart subtotal:
```
Subtotal: $[XX.XX]
```
- If free shipping threshold not met: "Add $[XX.XX] more for free shipping"
- If free shipping threshold met: "Free Shipping" with checkmark

**Step 7:** Write the free shipping progress bar copy (if applicable):
```
If below threshold: "You're $[XX.XX] away from free shipping!"
If at/above threshold: "You've unlocked free shipping!"
```

**Step 8:** Write the action buttons:
```
Primary:   "Checkout"
Secondary: "Continue Shopping"
```
- "Checkout" = proceeds to checkout page (this is the one place we DO leave the PDP)
- "Continue Shopping" = closes the mini-cart, returns to PDP
- NOT "Proceed to Checkout" (too verbose), NOT "View Cart" (defeats the purpose of mini-cart)

**Step 9:** Write the cross-sell suggestion line:
```
Section header: "You might also like"
Format: [Product thumbnail] [Product name] [$XX.XX] [Add]
```
- Single cross-sell suggestion (not a grid of 4-6 products)
- "Add" button adds the cross-sell to cart without leaving mini-cart
- Cross-sell product should be complementary (specified by offer structure or PDP-03)

**Step 10:** Write the remove item text:
```
Text: "Remove"
```
- Small text or icon link below the item
- Clicking removes item and updates cart count/subtotal

**Step 11:** Write `mini-cart-copy.md`.

## Output Schema

`mini-cart-copy.md`:

```markdown
# Mini-Cart Copy -- PDP-06

## Trigger: Slide-out from right after "Add to Cart" click

---

## Header
- **Text:** "Cart ([X])"
- **Close button:** "X" or close icon (top right)

## Cart Item Display
```
[Thumbnail]
[Full Product Name]
Variant: [Selected Variant] (if applicable)
Qty: [stepper] [X]
$[XX.XX]
Remove
```

## Subtotal
- **Text:** "Subtotal: $[XX.XX]"

## Free Shipping Progress (if applicable)
- **Below threshold:** "You're $[XX.XX] away from free shipping!"
- **At/above threshold:** "You've unlocked free shipping!"
- **No threshold:** [omit this element]

## Action Buttons
- **Primary:** "Checkout"
- **Secondary:** "Continue Shopping"

## Cross-Sell Section
- **Header:** "You might also like"
- **Format:** [thumbnail] [name] [$XX.XX] [Add]
- **Max items:** 1 suggestion
- **Cross-sell product:** [product name or "to be determined by merchandising"]

## Remove Item
- **Text:** "Remove"
- **Behavior:** Removes item, updates cart count and subtotal

## Empty Cart State
- **Text:** "Your cart is empty"
- **Action:** "Continue Shopping"
```

---

## Quality Criteria
- Mini-cart triggers on ATC click (slide-out, not cart redirect)
- Product name in mini-cart is full (not truncated)
- Variant shown if product has variants
- "Checkout" and "Continue Shopping" are the exact action labels
- Cross-sell is a single suggestion (not a full product grid)
- Free shipping progress copy is encouraging, not pushy
- Remove option is present for every cart item
- File exists after execution

## Failure Modes
- ATC post-click behavior is cart redirect instead of mini-cart -> Override per Baymard; write mini-cart copy anyway
- Cross-sell product not specified -> Write the section structure with placeholder "[to be determined by merchandising]"
- Free shipping threshold unknown -> Omit free shipping progress bar; flag gap for human
- PDP-03 did not specify mini-cart -> Write the copy anyway (post-ATC mini-cart is Baymard best practice); note in execution log
- Multiple products in cart from previous adds -> Mini-cart copy should handle multi-item display; show each item as separate line item
