# 0.3 — Blueprint Extractor

**Skill:** PDP-06-offer-buybox-writer
**Position:** Layer 0, Microskill 3
**Purpose:** Extract buy box component specifications from PDP-03's pdp-above-fold-blueprint.json. Inventory every component PDP-03 architected so PDP-06 knows exactly what copy to write. This is the bridge between architecture and copywriting.
**Effort Level:** low
**Dependencies:** 0.1 input-verification.md (confirms blueprint exists), PDP-03 pdp-above-fold-blueprint.json
**Output:** `blueprint-extraction.md`

---

## Input Schema

```yaml
input:
  input_verification: "[path to input-verification.md]"
  pdp_blueprint: "[path to pdp-above-fold-blueprint.json from PDP-03]"
```

---

## Execution Logic

**Step 1:** Verify `input-verification.md` confirms PDP-03 blueprint is loaded. If not -> HALT.

**Step 2:** Read `pdp-above-fold-blueprint.json` in full. Navigate to the `buy_box` section.

**Step 3:** Extract the component inventory -- every buy box component PDP-03 specified:
- Product title spec (display rules)
- Rating strip spec (stars, count, anchor link)
- Short description spec (bullet count, content direction)
- Price display spec (tiers, anchor price, savings format)
- Variant selector spec (type: chips, variant list, stock status rules)
- Subscription tile spec (OTP vs S&S, savings, frequency options)
- Quantity stepper spec (min, max, default, bundle triggers)
- ATC button spec (width, label direction, state changes)
- Express payment spec (layout, payment methods)
- Micro-trust signal spec (count, signal types, placement)

**Step 4:** For each component, extract:
- **Exists in blueprint:** Y/N
- **PDP-03 structural spec:** What PDP-03 specified about the component's structure/UX
- **Copy direction from PDP-03:** Any copy hints or constraints PDP-03 provided
- **Copy gaps:** What copy PDP-06 must generate that PDP-03 left blank (all actual text)

**Step 5:** Extract any additional buy box elements PDP-03 specified beyond the standard 10:
- Sticky ATC bar (if specified)
- Mini-cart (if specified)
- Shipping proximity messaging (if specified)
- Stock urgency indicators (if specified)

**Step 6:** Cross-check against PDP-03's buy box audit score. Note any audit findings that affect copy requirements.

**Step 7:** Write `blueprint-extraction.md`.

## Quality Criteria
- Every component in PDP-03's blueprint is extracted and inventoried
- For each component, the copy gap is clearly identified (what PDP-06 must write)
- No components invented that PDP-03 did not specify
- PDP-03 audit findings documented for copy context
- Extended components (sticky bar, mini-cart) captured if PDP-03 specified them
- File exists after execution

## Failure Modes
- pdp-above-fold-blueprint.json has no buy_box section -> HALT, PDP-03 may have failed
- Blueprint specifies variant selector but brief has no variant data -> HALT, request variant details
- Blueprint audit score < 7.0 -> WARNING, PDP-03 should have caught this; proceed but note risk
- input-verification.md not found -> HALT, run 0.1 first
- Component in blueprint not recognized -> Document and flag for human review
