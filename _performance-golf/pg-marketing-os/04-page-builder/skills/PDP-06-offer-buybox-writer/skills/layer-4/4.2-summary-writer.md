# 4.2 — Summary Writer

**Skill:** PDP-06-offer-buybox-writer
**Position:** Layer 4, Microskill 2
**Purpose:** Write PDP-OFFER-BUYBOX-SUMMARY.md -- a human-readable summary of all buy box copy produced by PDP-06. This file is for human review and sign-off, not for downstream skill consumption (that's pdp-offer-buy-box.json).
**Effort Level:** low
**Dependencies:** 4.1 pdp-offer-buy-box.json (assembled package), 3.1 buy-box-copy-audit.md (audit results)
**Output:** `PDP-OFFER-BUYBOX-SUMMARY.md`

---

## Input Schema

```yaml
input:
  pdp_offer_buy_box: "[path to pdp-offer-buy-box.json]"
  buy_box_copy_audit: "[path to buy-box-copy-audit.md]"
  cta_consistency_check: "[path to cta-consistency-check.md]"
  anti_slop_check: "[path to anti-slop-check.md]"
```

---

## Execution Logic

**Step 1:** Read `pdp-offer-buy-box.json`. Extract all copy for human-readable presentation.

**Step 2:** Read audit results from Layer 3 files.

**Step 3:** Write the summary in this structure:
- Product identification
- Pricing summary (all tiers, all formats)
- Short description bullets
- Variant chip labels
- Subscription tile copy (if applicable)
- ATC button states
- Micro-trust signals
- Sticky bar copy
- Mini-cart copy
- Audit results
- Notes and flags for human attention

**Step 4:** Highlight any items that need human review:
- Cross-sell product suggestion (if placeholder)
- Stock status data (if estimated)
- Variant names (if inferred from brief rather than confirmed)
- Any data gaps flagged during execution

**Step 5:** Write `PDP-OFFER-BUYBOX-SUMMARY.md`.

## Quality Criteria
- All buy box components represented in summary
- Copy is presented in human-readable format (not raw JSON)
- Audit results clearly stated
- Items needing human review explicitly called out
- No copy is omitted or summarized -- all actual text is shown
- File exists after execution

## Failure Modes
- pdp-offer-buy-box.json not found -> HALT, run 4.1 first
- Summary omits a component -> Cross-check against pdp-offer-buy-box.json keys
- Human review items not flagged -> Review execution log for any data gaps or placeholders
