# 0.2 — Proof Inventory Loader

**Skill:** LP-05-social-proof-architecture
**Position:** Layer 0, Microskill 2
**Purpose:** Load and catalog all available proof assets. If Downstream Mode, pull from CopywritingEngine Skill 02 research output and any provided proof files. If Standalone Mode, document what's known from the brief and flag everything else as requires_sourcing.
**Effort Level:** medium
**Dependencies:** `brief-and-sequence-loaded.md` (0.1), proof_inventory_path from page-brief.json
**Output:** `proof-inventory.md`

---

## Input Schema

```yaml
input:
  operating_mode: "[downstream | standalone]"
  proof_inventory_path: "[path from page-brief.json or none]"
  page_brief_proof_block:
    testimonials_count: "[number or not_available]"
    has_clinical_studies: "[Y/N or not_available]"
    has_before_afters: "[Y/N or not_available]"
    has_expert_endorsements: "[Y/N or not_available]"
  project_folder: "[path]"
```

---

## Execution Logic

**Step 1: Downstream Mode** — Search for proof assets:

```
Priority 1: proof_inventory_path (if provided in page-brief.json)
Priority 2: [project_folder]/01-research/outputs/ (Skill 01 research output — may contain audience quotes, expert references)
Priority 3: [project_folder]/02-proof/outputs/ (Skill 02 proof output — may contain categorized proof)
Priority 4: User-provided proof document (product brief may include testimonial list)
```

For each found source, extract and catalog:
- Customer testimonials (count, names available, specific outcomes mentioned)
- Clinical/scientific studies (count, institution names, specific findings)
- Expert endorsements (count, expert names, credentials)
- Before/after results (count, specific outcomes, timeframes)
- Platform reviews (count, average rating, platform name)
- Media mentions (count, publication names)
- Certifications (list with names)
- Video testimonials (count, descriptions)

**Step 2: Standalone Mode** — Document what's available from the brief:

- Pull whatever proof data exists from page-brief.json proof_inventory block
- Flag all missing proof types as `not_available`
- Mark all planned proof as `requires_sourcing`
- Note in output: "Standalone Mode — proof inventory is from brief only. All proof elements in the architecture will be marked requires_sourcing."

**Step 3:** Count totals by proof type. Calculate inventory sufficiency:
- `sufficient`: All major proof types available with enough volume for page type thresholds
- `gaps_identified`: Some proof types available but gaps in specific categories
- `critical_gaps`: Fewer than 5 total proof assets available; AUTHORITY_SUBSTITUTION mode likely needed

## Quality Criteria
- All proof types checked and documented (even if count = 0)
- Source paths documented for every found asset
- Quality notes capture whether proof includes specific outcomes
- Sufficiency assessment is honest (not optimistic)
- Standalone mode clearly flags everything as requires_sourcing

## Failure Modes
- No proof assets found AND operating mode is downstream -> Flag as anomaly: "Downstream Mode but no proof assets found. Verify CopywritingEngine Skill 02 was run."
- proof_inventory_path provided but file not found -> Note missing file, continue with brief-only data
