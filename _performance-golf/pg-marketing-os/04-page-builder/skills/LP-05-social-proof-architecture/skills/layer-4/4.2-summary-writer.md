# 4.2 — Summary Writer

**Skill:** LP-05-social-proof-architecture
**Position:** Layer 4, Microskill 2
**Purpose:** Write the human-readable PROOF-ARCHITECTURE-SUMMARY.md that presents the proof architecture in a scannable format for human review. This is the document the human reviewer reads to understand and approve the proof strategy before LP-10 executes it.
**Effort Level:** low
**Dependencies:** `proof-architecture.json` (4.1)
**Output:** `PROOF-ARCHITECTURE-SUMMARY.md`

---

## Input Schema

```yaml
input:
  proof_architecture_json: "[from 4.1 — the complete compiled architecture]"
```

---

## Execution Logic

**Step 1:** Load proof-architecture.json.

**Step 2:** Write the summary following the template below. Every section is required.

**Step 3:** Keep the language direct and scannable:
- Tables over paragraphs
- Bullet points over prose
- Numbers over adjectives
- Specific proof types over "social proof"

## Quality Criteria
- All sections present (no omissions)
- Numbers match proof-architecture.json exactly
- Gaps highlighted with severity and action
- Wave pattern visualization present and readable
- Key decisions clearly flagged for human review
- Language is direct and scannable (no filler)

## Failure Modes
- proof-architecture.json not available -> HALT. Run 4.1 first.
- Summary numbers don't match JSON -> Reconcile. The JSON is the source of truth.
