# 2.3 — Proof Type Sequencer

**Skill:** LP-05-social-proof-architecture
**Position:** Layer 2, Microskill 3
**Purpose:** Within sections that have multiple proof elements (medium, heavy, maximum density), determine the ORDER in which proof types appear. Sequence matters — the first proof element sets the credibility anchor; subsequent elements build on it. This microskill optimizes the within-section proof flow for maximum persuasive impact.
**Effort Level:** medium
**Dependencies:** `proof-density-map.md` (2.2)
**Output:** `proof-type-sequence.md`

## Sequencing Principles

### Principle 1: Authority First, Volume Last
Within a multi-element section, lead with the highest-authority proof and close with volume/social proof. The authority element establishes credibility; volume elements confirm it.

**Optimal within-section sequence:**
```
1. Authority anchor (study, expert, media mention)
2. Specific result proof (before/after, named testimonial)
3. Volume confirmation (star rating, customer count, review count)
```

### Principle 2: Specific Before General
Named, specific proof always precedes unnamed, general proof. "Dr. Sarah Chen, Harvard Medical School, found..." before "clinical studies show..."

### Principle 3: Emotional Escalation in Testimonial Blocks
Within a dedicated proof block (4+ testimonials), sequence for emotional escalation:
```
1. Relatable skeptic ("I was doubtful, but...")
2. Dramatic transformation ("I went from X to Y in Z weeks")
3. Authority testimonial (expert or public figure)
4. Volume/social ("Join 50,000+ who experienced...")
```

### Principle 4: Type B Review Cascade Sequencing
In Type B review cascade sections (25-50+ reviews), the first 3-5 visible reviews are critical. Sequence them:
```
1. Highest-star review with specific outcome + photo
2. Review addressing most common objection
3. Review from relatable demographic
4. Review with before/after result
5. Review highlighting specific feature/ingredient
[remaining reviews: sorted by recency or helpfulness]
```

### Principle 5: Ingredient Section Proof Pairing
Each ingredient's proof should lead with the clinical/study reference and follow with a testimonial that makes the study result FEEL real:
```
1. Study: "In a 2021 study at [institution], participants who took [ingredient] experienced [specific result]"
2. Testimonial: "I noticed [specific result] after just [timeframe]" — [Name, Location]
```

---

## Sequencing by Section Type

### For Lead / Hook Sections (Light Density)
Single element — no sequencing needed. Place the one authority signal in the opening or embed naturally in the first 2 paragraphs.

### For Mechanism / Root Cause Sections (Medium Density)
```
Sequence:
  1. Study/clinical reference (woven into narrative — not a standalone block)
  2. Expert quote or credential mention (supports the mechanism claim)
  3. [If third element] Demonstration or data point (visual or quantified)
```

### For Dedicated Proof Blocks (Heavy Density — Type A)
```
Proof Block 1 (early — belief initiation):
  1. Lead testimonial (most relatable, addresses primary pain)
  2. Before/after or specific result (quantified transformation)
  3. Supporting testimonial (different demographic or pain point)
  4. [If study] Clinical data point (grounds the testimonials in science)

Proof Block 2 (mid-page — belief cascade):
  1. Video testimonial (highest engagement, if available)
  2. Photo testimonial (strong specific outcome)
  3-6. Text testimonials (variety of outcomes, demographics, timeframes)
  7-8. [If available] Before/after pairs (visual evidence)

Proof Block 3 (pre-close — transformation proof):
  1. Most dramatic transformation story (the "hero" testimonial)
  2. Before/after with longest timeframe (sustained results)
  3. Callback testimonial referencing the lead/hook promise
  4. Volume proof ("Join 50,000+ who...")
```

### For Review Cascade Sections (Maximum Density — Type B)
```
Top 5 (visible without scrolling):
  1. Photo review, 5 stars, specific outcome
  2. Review addressing #1 purchase objection
  3. Review from target demographic
  4. Review with quantified result
  5. Long-form review with before/after narrative

Remaining reviews: sorted by star rating (highest first), then recency
Filter options: By concern, by rating, by verified buyer
```

## Execution Logic

**Step 1:** Load the density map from 2.2. Identify all sections with 2+ elements (light sections with 1 element need no sequencing).

**Step 2:** For each multi-element section, apply the appropriate sequencing pattern from the section type rules above.

**Step 3:** Number each element within the section to create a defined display order.

**Step 4:** For testimony-heavy sections (proof blocks), apply the emotional escalation principle to order the testimonials.

**Step 5:** Document the complete within-section sequence for every multi-element section.

## Quality Criteria
- Every multi-element section has a defined element order
- Sequencing follows the principles (authority first, specific before general)
- Emotional escalation applied to testimonial blocks
- Type B review cascade has a defined top-5 sequence
- Each element's position has a stated rationale

## Failure Modes
- All elements are the same type -> Cannot apply "authority first" principle. Flag: "Proof type monotony in section [N] — all elements are [type]. Consider diversifying."
- No authority-type proof available for lead position -> Use highest-ranked available type as lead. Note deviation from optimal sequence.
