# 4.1 — Trust Package Compiler

**Skill:** LP-08-trust-elements
**Position:** Layer 4, Microskill 1
**Purpose:** Assemble all validated trust element copy into a single structured package (`trust-elements-package.json`) for downstream skills. LP-15 (Assembly) and LP-17 (Conversion Audit) will load this file.
**Effort Level:** low
**Dependencies:** All Layer 2 outputs (validated in Layer 3), `trust-validation.md` (3.1), `specificity-audit.md` (3.2)
**Output:** `trust-elements-package.json`

---

## Input Schema

```yaml
input:
  trust_bar_copy: "[from trust-bar-copy.md — both variant sets]"
  media_mention_copy: "[from media-mention-copy.md — or not_applicable]"
  certification_copy: "[from certification-copy.md — or not_applicable]"
  badge_copy: "[from badge-copy.md — or not_applicable]"
  volume_signal_copy: "[from volume-signal-copy.md — volume, authority, security sections]"
  trust_validation: "[from trust-validation.md — audit score, anti-slop result]"
  specificity_audit: "[from specificity-audit.md — audit result]"
  trust_placement_map: "[from trust-placement-map.md — placement for each element]"
  packages_loaded: "[from packages-loaded.md — page type, product name]"
```

---

## Pre-Assembly Verification

Before assembling the package, verify all gates passed:

```yaml
ASSEMBLY-GATE:
  trust_audit_score: "[X.X — must be >= 7.0]"
  anti_slop_scan: "[PASS | FAIL — must be PASS]"
  specificity_audit: "[PASS | FAIL — must be PASS]"
  all_gates_pass: "[Y/N]"

  IF all_gates_pass = N: HALT — do not assemble. Return to failed validation.
```

---

## Execution Logic

1. **Verify all three GATE_3 checks passed** (trust audit >= 7.0, anti-slop PASS, specificity PASS). If any check failed, HALT.

2. **Assemble `trust-elements-package.json`** following the schema defined in LP-08-AGENT.md. Populate each section:

   - **trust_bar:** Both variant sets with positions, types, copy, icon direction
   - **media_mentions:** Status + publications + callout copy (or `not_applicable`)
   - **certifications:** Status + badge labels + explanation copy + placement (or `not_applicable`)
   - **trust_badges:** Status + badge text + tooltip + placement (or `not_applicable`)
   - **security_signals:** Status + copy + placement (or `not_applicable`)
   - **authority_signals:** Status + expert data + framing copy + placement (or `not_applicable`)
   - **volume_signals:** Status + numbers + copy + placement (or `not_applicable`)
   - **rating_strip:** Status + star rating + review count + context copy (or `not_applicable`)
   - **validation:** All scores and audit results
   - **downstream_handoffs:** Instructions for LP-15 and LP-17

3. **Verify every required field is populated.** No `null` values in required fields. Optional fields may be `null`.

4. **Verify the JSON structure is valid** — downstream skills must be able to parse this file without error.

5. **Write `trust-elements-package.json`** to the project output directory.

---

## Output Schema

The output follows the `trust-elements-package.json` schema defined in LP-08-AGENT.md. See AGENT.md for the complete schema.

Key verification points:
- `schema_version`: "1.0"
- `skill`: "LP-08-trust-elements"
- `created`: ISO timestamp
- `page_type`: matches upstream classification
- `trust_bar.signal_count`: within range (3–6)
- `trust_bar.variant_set_a` and `variant_set_b`: both populated with correct positions
- All `status` fields accurately reflect whether the element was included or not_applicable
- `validation.trust_audit_score`: >= 7.0
- `validation.specificity_audit`: "PASS"
- `validation.anti_slop_scan`: "PASS"
- `validation.generic_signals_found`: 0
- `validation.all_gates_passed`: true

---

## Quality Criteria
- Pre-assembly gate verification passes (all three checks)
- Every Layer 2 output correctly mapped into the JSON structure
- No fabricated data introduced during assembly (pure compilation)
- JSON structure is parseable
- All placement information from trust-placement-map.md preserved
- Downstream handoff instructions are clear and actionable

## Failure Modes
- Assembling package despite a failed gate → HALT and return to Layer 2/3 for revision. Assembly with failed gates is a protocol violation.
- Introducing new copy during assembly → Layer 4 COMPILES, it does not GENERATE. No new trust copy should be written here.
- Dropping an element during assembly → Every element from Layer 2 must appear in the package (including `not_applicable` elements with their status).
- Invalid JSON structure → Test the structure before writing the file.
