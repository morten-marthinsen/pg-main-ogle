# 4.2 — Summary Writer

**Skill:** LP-08-trust-elements
**Position:** Layer 4, Microskill 2
**Purpose:** Write `TRUST-ELEMENTS-SUMMARY.md` — a human-readable summary of all trust elements generated, their placements, audit results, and any gaps or notes. This is the file a human reviews to understand what LP-08 produced without reading the JSON package.
**Effort Level:** low
**Dependencies:** `trust-elements-package.json` (4.1), all Layer 2 and 3 outputs
**Output:** `TRUST-ELEMENTS-SUMMARY.md`

---

## Input Schema

```yaml
input:
  trust_package: "[from trust-elements-package.json — compiled package]"
  trust_validation: "[from trust-validation.md — audit results]"
  specificity_audit: "[from specificity-audit.md — per-signal results]"
  trust_classification: "[from trust-classification.md — what was available vs not]"
  trust_placement_map: "[from trust-placement-map.md — where everything goes]"
```

---

## Execution Logic

1. **Read trust-elements-package.json** and extract all key data.

2. **Write the summary** following the template below. Include:
   - Page type and product name
   - Trust element types used (and which are not_applicable)
   - Trust bar copy (primary variant set — full bar string)
   - Trust bar copy (alternate variant set — full bar string)
   - Per-type element summaries with placement
   - Audit scores and specificity results
   - Gaps and notes
   - Downstream handoff summary

## Quality Criteria
- Summary is human-readable without requiring the JSON file
- Trust bar copy displayed as full bar strings (both variants)
- Every trust element type documented (including not_applicable with explanation)
- Audit results clearly displayed
- Downstream handoff instructions included
- Gaps and notes section present (even if empty)

## Failure Modes
- Summary that just says "see trust-elements-package.json" → The summary must be self-contained and readable independently.
- Missing trust bar copy strings → The trust bar is the most important element — its copy must be visible in the summary.
- Omitting not_applicable elements → All 7+ types should appear in the summary, with status noted.
