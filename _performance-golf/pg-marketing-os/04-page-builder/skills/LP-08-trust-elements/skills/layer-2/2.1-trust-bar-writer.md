# 2.1 — Trust Bar Writer

**Skill:** LP-08-trust-elements
**Position:** Layer 2, Microskill 1
**Purpose:** Write the actual trust bar copy — 3–5 words per signal, 2 complete variant sets. The trust bar is scanned in 0.5–1.5 seconds. Every word must earn its place. This is the highest-visibility trust element on the page.
**Effort Level:** max
**Dependencies:** `trust-bar-plan.md` (1.2), `trust-inventory.md` (0.2), `specimen-trust-patterns.md` (0.3)
**Output:** `trust-bar-copy.md`

---

## Input Schema

```yaml
input:
  trust_bar_plan: "[from trust-bar-plan.md — selected signals, positions, copy directions]"
  trust_inventory: "[from trust-inventory.md — specific data for each selected signal]"
  specimen_patterns: "[from specimen-trust-patterns.md — verbatim trust bar examples]"
  page_type: "[type_a | type_b | hybrid]"
  product_name: "[from packages-loaded.md]"
```

---

## Pre-Writing Protocol

**Step 1: Load specimen trust bar examples**
Hold the verbatim specimen trust bar examples from specimen-trust-patterns.md in active context. Before writing Signal 1, read each example and note: How many words? What makes it specific? What is the strongest word?

**Step 2: Review the forbidden word list**
Before writing: scan the LP-08 anti-slop forbidden list. Commit to producing zero generic signals. "100% Satisfaction Guaranteed" is the enemy. "365-Day Money-Back Guarantee" is the standard.

**Step 3: State the word count constraint**
Every signal: 3–5 words. Count every word. If a signal is 6+ words, it is too long. Trim it.

---

## Execution Logic

1. **For each signal position from trust-bar-plan.md**, write the trust bar copy.

2. **Generate Variant Set A** (primary):
   - Write each signal at 3–5 words
   - Maximize specificity: numbers > names > timeframes > generic terms
   - Use the trust inventory data to make each signal as concrete as possible

3. **Generate Variant Set B** (alternate):
   - Same signals, different phrasing
   - If Set A leads with the number, Set B leads with the benefit
   - If Set A uses the certification name, Set B uses the audience benefit

4. **Apply signal-type-specific writing rules:**

   **Guarantee signals:**
   - Always include duration: "365-Day", "90-Day", "60-Day"
   - Always include type: "Money-Back", "Full Refund", "No Questions"
   - NEVER: "Satisfaction Guaranteed" without terms
   - Good: "365-Day Money-Back Guarantee"
   - Good variant: "Full Refund — 365 Days"

   **Certification signals:**
   - Use the certification abbreviation: "GMP", "NSF", "cGMP"
   - Add one clarifying word: "GMP Certified Facility", "NSF Verified"
   - NEVER: "Premium Quality" or "Highest Standards"
   - Good: "GMP Certified Facility"
   - Good variant: "Pharmaceutical-Grade Manufacturing"

   **Volume signals:**
   - Use the SPECIFIC number from inventory: "1,270,268+"
   - Add context word: "Customers", "Bottles Sold", "Reviews"
   - NEVER: "Thousands of Customers" or "Millions Served"
   - Good: "1,270,268+ Customers Served"
   - Good variant: "Trusted by 1,270,268+"

   **Shipping signals:**
   - State the specific policy: "Free Shipping", "Free Shipping Over $50"
   - Add delivery detail if impressive: "Free 2-Day Shipping"
   - NEVER: "Fast Shipping" without specifying how fast

   **Security signals:**
   - Name the specific technology or processor
   - Good: "256-Bit Secure Checkout"
   - Good variant: "Stripe-Secured Payments"

5. **Count words per signal.** Any signal > 5 words → rewrite shorter.

6. **Assemble the full trust bar string** for both variant sets using `|` or ` · ` separator.

7. **Write `trust-bar-copy.md`.**

## Quality Criteria
- Every signal is 3–5 words (hard constraint — count each one)
- Zero generic signals (specificity check passes)
- 2 complete variant sets provided (not just 1)
- Every signal traces to trust inventory data
- Trust bar reads as a coherent credibility strip, not a random collection
- Variant sets are genuinely different phrasing, not trivial word swaps

## Failure Modes
- Signal over 5 words → HALT. Rewrite shorter. If it cannot be said in 5 words, the signal is too complex for a trust bar — consider moving it to a badge or certification explanation instead.
- Generic signal passes through ("Premium Quality", "Trusted Brand") → Caught by specificity check. Must contain a specific number, name, timeframe, or certification.
- Both variant sets are nearly identical → Variant Set B should differ meaningfully in structure or emphasis, not just swap one word.
- Signal copy not supported by inventory data → Fabrication. Remove and replace with a supported signal.
