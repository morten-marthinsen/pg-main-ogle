# 1.3 — Trust Placement Mapper

**Skill:** LP-08-trust-elements
**Position:** Layer 1, Microskill 3
**Purpose:** Map every classified trust element to its specific page section using LP-05's proof architecture and LP-03's above-fold layout. This creates the complete placement plan that Layer 2 writers use to know WHERE each element goes.
**Effort Level:** medium
**Dependencies:** `trust-classification.md` (1.1), `trust-bar-plan.md` (1.2), `packages-loaded.md` (0.1)
**Output:** `trust-placement-map.md`

---

## Input Schema

```yaml
input:
  trust_classification: "[from trust-classification.md — all classified trust elements with status]"
  trust_bar_plan: "[from trust-bar-plan.md — selected signals and placement]"
  page_type: "[from packages-loaded.md]"
  proof_architecture_sections: "[from packages-loaded.md — trust-related sections from LP-05]"
  above_fold_layout: "[from packages-loaded.md — trust bar and badge placement from LP-03]"
```

---

## Execution Logic

1. **Start with the trust bar** — placement already determined in trust-bar-plan.md. Confirm section placement from LP-03 above-fold layout.

2. **Map remaining trust elements to page sections** using the LP-05 proof architecture as the primary guide:

   **Type A placement patterns:**
   | Trust Element | Typical Section | Priority |
   |--------------|----------------|----------|
   | Trust bar | Above fold / below hero | MUST place |
   | Media mentions | First 25% of page (early credibility) or authority section | Place if available |
   | Certifications | Near ingredient/mechanism section | Place if available (health verticals) |
   | Authority signals | Story section, proof block, or mechanism section | Place if available |
   | Volume signals | Trust bar (short form) + proof block opener (long form) | Place if available |
   | Security signals | Near order form / checkout area | Place if checkout on page |
   | Trust badges | Guarantee section or near CTA | Place 2-3 |

   **Type B placement patterns:**
   | Trust Element | Typical Section | Priority |
   |--------------|----------------|----------|
   | Trust bar | Directly below ATC button | MUST place |
   | Trust badges | Below ATC, above fold | MUST place (3-5) |
   | Rating strip | Above fold, below product title | MUST place if reviews exist |
   | Security signals | Near ATC and checkout | MUST place |
   | Certifications | Trust bar (badge form) + ingredient section (explanation) | Place if available |
   | Volume signals | Rating strip + trust bar | Place if available |
   | Media mentions | Below fold, logo strip | Place if available |
   | Authority signals | Ingredient/education section | Place if available |

3. **Cross-reference LP-05 architecture** — if LP-05 assigned trust-type elements to specific sections, honor those assignments. LP-08 fills the WHAT (copy), LP-05 determined the WHERE (section).

4. **Check for placement conflicts** — no section should have more than 2 trust element types (cluttering reduces individual signal impact). If a section has 3+ assigned trust elements, redistribute the lowest-priority element.

5. **Document every trust element with its placement** — include elements marked `not_applicable` (with note) so the map is complete.

6. **Write `trust-placement-map.md`.**

## Quality Criteria
- Every classified trust element has a placement (including `not_applicable` elements documented)
- LP-05 architecture is honored — no contradictions with proof-architecture.json
- No section has 3+ trust element types (max 2 per section)
- Trust element present in first 25% of page (early credibility requirement)
- MUST-place elements all have sections assigned
- Type A vs Type B placement patterns correctly applied

## Failure Modes
- Placing all trust elements in one section → Distribute across the page per placement patterns.
- Ignoring LP-05 section assignments → LP-05 determined WHERE trust signals go. LP-08 fills the copy. If LP-05 placed a certification in the ingredient section, that is where it goes.
- No trust in the first 25% of the page → At minimum, the trust bar should appear above fold or below hero.
- Type B page without trust badges near ATC → MUST place trust badges below/near ATC for Type B. This is the highest-anxiety point on the page.
- Overloading a section with 3+ trust types → Redistribute. Trust elements compete for attention when clustered.
