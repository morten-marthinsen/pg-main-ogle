# 0.1-upstream-loader.md

> **Layer:** 0 (Foundation)
> **Skill:** PDP-03-hero-value-prop
> **Purpose:** Load strategy + feature package
> **Outputs:** `upstream_context` object with strategy and named features

---

## FUNCTION

Load the e-commerce strategy from PDP-01 and the feature package from PDP-02. Extract hero section strategy (word budgets, priority) and the hero feature(s) that anchor the above-fold experience.

---

## LOADING PROTOCOL

### Step 1: Load Strategy
```
LOCATE: ecomm-strategy.yaml
EXTRACT:
  hero_section: section_map[section_number == 1]
  hero_word_budget: word_budget for hero section
  page_type: pdp|collection|bundle
  product_context: product_name, product_category
```

### Step 2: Load Feature Package
```
LOCATE: feature-package.json from PDP-02
EXTRACT:
  hero_features: feature_hierarchy.hero[]
  hero_feature_name: hero[0].name
  hero_micro_script: hero[0].micro_script
  supporting_features: feature_hierarchy.supporting[]
  highlights_candidates: all features for TLDR selection
```

### Step 3: Assemble Context
```
BUILD upstream_context:
  strategy: { hero_section, hero_word_budget, page_type, product }
  features: { hero_feature, supporting_features, all_features }
  constraints: { word_budgets from strategy }
```

---

## OUTPUT STRUCTURE

```json
{
  "upstream_context": {
    "hero_feature": { "name": "HyperSpeed Face", "micro_script": "..." },
    "strategy": { "hero_word_budget": 150, "page_type": "pdp" },
    "all_features": [],
    "loading_status": { "strategy_loaded": true, "features_loaded": true }
  }
}
```

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | PDP-01 strategy not found | HALT -- execute PDP-01 first |
| CRITICAL | PDP-02 feature package not found | HALT -- execute PDP-02 first |
| CRITICAL | No hero feature identified | HALT -- return to PDP-02 for hierarchy |
| DEGRADED | Strategy missing hero section detail | Use default hero word budgets |
