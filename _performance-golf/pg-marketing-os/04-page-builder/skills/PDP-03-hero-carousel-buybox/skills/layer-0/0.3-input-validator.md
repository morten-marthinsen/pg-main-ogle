# 0.3-input-validator.md

> **Layer:** 0 (Foundation)
> **Skill:** PDP-03-hero-value-prop
> **Purpose:** Validate hero feature + section map present
> **Outputs:** `validation_result` object with PASS/FAIL

---

## FUNCTION

Validate that all required inputs for hero generation are present: hero feature from PDP-02, hero section strategy from PDP-01, and NLS patterns from specimen loading.

---

## VALIDATION PROTOCOL

### Step 1: Hero Feature Validation
```
CHECK upstream_context.hero_feature:
  [ ] hero_feature.name EXISTS and non-empty
  [ ] hero_feature.micro_script EXISTS
  [ ] hero_feature.tier == "hero"

  IF hero feature missing: FAIL -- "PDP-02 hero feature required"
```

### Step 2: Strategy Validation
```
CHECK upstream_context.strategy:
  [ ] hero_section EXISTS in section_map
  [ ] hero_word_budget defined (100-200 range)
  [ ] page_type defined
  [ ] product_name defined

  IF strategy incomplete: FAIL -- "PDP-01 strategy required"
```

### Step 3: Specimen Validation
```
CHECK specimen_patterns:
  [ ] At least 1 carousel template loaded
  [ ] At least 1 headline pattern loaded

  IF specimens missing: WARN -- proceed with defaults
```

### Step 4: Highlights Candidate Count
```
CHECK upstream_context.all_features:
  [ ] At least 3 features available for highlights selection

  IF fewer than 3: WARN -- "Highlights TLDR may be thin"
```

---

## OUTPUT STRUCTURE

```json
{
  "validation_result": {
    "status": "PASS|FAIL",
    "hero_feature_present": true,
    "strategy_present": true,
    "specimens_loaded": true,
    "highlights_candidates": 5,
    "flags": []
  }
}
```

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | No hero feature | HALT -- return to PDP-02 |
| CRITICAL | No strategy | HALT -- return to PDP-01 |
| DEGRADED | No specimens | Proceed with defaults |
| WARNING | Fewer than 3 highlights candidates | Proceed with available features |
