# 0.3 — Specimen Loader

**Skill:** PDP-03-hero-carousel-buybox
**Position:** Layer 0, Microskill 3
**Purpose:** Load PDP specimen URLs filtered by vertical from PDP-BEST-PRACTICES-REFERENCE.md specimen library. These specimens are real-world PDP examples that inform carousel story architecture and buy box design decisions.
**Effort Level:** low
**Dependencies:** 0.1 input-verification.md (for vertical), 0.2 pdp-reference-load.md
**Output:** `specimen-load.md`

---

## Input Schema

```yaml
input:
  input_verification: "[path to input-verification.md]"
  pdp_reference: "[path to reference/PDP-BEST-PRACTICES-REFERENCE.md]"
  product_vertical: "[extracted from input-verification.md]"
```

---

## Execution Logic

**Step 1:** Read `input-verification.md` to identify product vertical (supplement, skincare, electronics, food/bev, etc.).

**Step 2:** Read PDP-BEST-PRACTICES-REFERENCE.md Section 11 (PDP Specimen Library). Extract all specimen URLs.

**Step 3:** Filter specimens by relevance to current product vertical:
- **Primary matches:** Same vertical (e.g., supplement → supplement PDP specimens)
- **Secondary matches:** Adjacent verticals or shared patterns (e.g., supplement → health/wellness)
- **Universal references:** Specimens noted as best-in-class regardless of vertical

**Step 4:** For each relevant specimen, extract:
- URL
- Brand name
- Product type
- What to study (carousel approach, buy box pattern, specific component)
- Vertical match quality (primary / secondary / universal)

**Step 5:** Identify which specimens are most relevant to specific PDP-03 decisions:
- Carousel story architecture specimens (best carousel examples)
- Buy box design specimens (best buy box examples)
- Facts panel specimens (if applicable)
- Video integration specimens

**Step 6:** If specimen library is not available in the reference file, document the gap and note that the skill will proceed using AGENT.md specifications and Baymard requirements only (no specimen-informed decisions).

**Step 7:** Write `specimen-load.md`.

## Quality Criteria
- Specimens filtered by actual product vertical (not random selection)
- Each specimen has a specific "study focus" (not generic "good PDP")
- Decision-specific mapping connects specimens to PDP-03 components
- Gaps documented honestly (no fabricating specimen references)
- File exists after execution

## Failure Modes
- No specimens available for this vertical → PROCEED with Baymard requirements only, document gap
- PDP-BEST-PRACTICES-REFERENCE.md specimen section empty → PROCEED, document gap
- Vertical not identified in input-verification.md → Use product category as fallback
- Specimen URLs are dead/inaccessible → Note in gaps, do not remove from reference list
