# 0.2-specimen-calibrator.md

> **Layer:** 0 (Foundation)
> **Skill:** PDP-03-hero-value-prop
> **Purpose:** Load NLS hero carousel patterns
> **Outputs:** `specimen_patterns` object with hero and carousel templates

---

## FUNCTION

Load exemplar hero section patterns from the NLS PDP framework and high-performing ecom product pages. These specimens calibrate the quality bar for hero copy -- particularly the 10-thumbnail carousel narrative structure.

---

## SPECIMEN LOADING PROTOCOL

### Step 1: Load NLS Hero Patterns
```
LOCATE: ~brain/nls-pdp-best-practices.md
EXTRACT hero-specific guidance:
  carousel_structure: 10-thumbnail narrative architecture
  hero_copy_components: headline, subhead, value prop, CTA, highlights
  visual_hierarchy: how hero elements are prioritized visually
  mobile_behavior: how hero adapts to mobile screens
```

### Step 2: Load Carousel Narrative Templates
```
IDENTIFY 10-position narrative patterns:

  PATTERN A: Product Story Arc
    1: Hero product shot + headline
    2: Problem/pain point
    3: Solution reveal (hero feature)
    4: How it works (mechanism)
    5: Key benefit #1
    6: Key benefit #2
    7: Social proof (review/UGC)
    8: Use case / lifestyle
    9: What's included
    10: CTA + offer

  PATTERN B: Feature Showcase
    1: Hero product shot + headline
    2-4: Top 3 features with visual demos
    5: Comparison vs. alternatives
    6: Social proof
    7-8: Benefits / results
    9: Trust elements (guarantee, shipping)
    10: CTA + offer

  PATTERN C: Before/After Journey
    1: Before state (problem visual)
    2: The discovery (product intro)
    3-5: The transformation (feature benefits)
    6: After state (result visual)
    7-8: Proof (reviews, stats)
    9: Risk reversal (guarantee)
    10: CTA + offer
```

### Step 3: Load Hero Headline Patterns
```
EXTRACT headline patterns effective for ecom:
  benefit_specific: "[Specific Outcome] + [Mechanism]"
  product_name_led: "[Product] + [Core Promise]"
  problem_solution: "Stop [Problem]. Start [Solution]."
  social_proof_led: "[X] Customers + [Result]"
  feature_led: "[Hero Feature Name] + [What It Means For You]"
```
