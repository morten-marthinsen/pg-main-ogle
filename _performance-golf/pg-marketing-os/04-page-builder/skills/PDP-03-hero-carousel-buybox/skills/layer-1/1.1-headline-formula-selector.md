# 1.1-headline-formula-selector.md

> **Layer:** 1 (Hero Architecture)
> **Skill:** PDP-03-hero-value-prop
> **Purpose:** Select headline approach for product type
> **Outputs:** `headline_approach` object with selected formula and rationale

---

## FUNCTION

Select the headline approach best suited for this product type, page type, and hero feature. Ecom headlines are NOT long-form VSL headlines -- they must work in 5-12 words, above a product image, for a prospect who is scanning, not reading.

---

## ECOM HEADLINE APPROACHES

### Approach 1: Benefit-Specific
```
PATTERN: [Specific Outcome] + [Product/Mechanism]
EXAMPLE: "Add 30 Yards With One Club Change"
USE WHEN: Clear, quantifiable benefit exists
WORD BUDGET: 6-10 words
HERO FEATURE FIT: High -- feature enables the specific benefit
```

### Approach 2: Product-Name-Led
```
PATTERN: [Product Name] + [Core Promise]
EXAMPLE: "SF2 Fairway Wood: The 30-Yard Shortcut"
USE WHEN: Product name has brand recognition or is inherently compelling
WORD BUDGET: 5-8 words (excluding product name)
HERO FEATURE FIT: Medium -- product name may overshadow feature
```

### Approach 3: Problem-Solution
```
PATTERN: "Stop [Problem]. Start [Solution]."
EXAMPLE: "Stop Slicing. Start Striping."
USE WHEN: Clear, universal pain point exists
WORD BUDGET: 5-8 words
HERO FEATURE FIT: Medium -- feature is the solution mechanism
```

### Approach 4: Social-Proof-Led
```
PATTERN: [Proof Point] + [Result]
EXAMPLE: "127,000+ Golfers Fixed Their Slice"
USE WHEN: Strong social proof numbers available
WORD BUDGET: 6-10 words
HERO FEATURE FIT: Low -- proof dominates, feature supports
```

### Approach 5: Hero-Feature-Led
```
PATTERN: [Hero Feature Name] + [What It Means For You]
EXAMPLE: "HyperSpeed Face: 15 More Yards, Guaranteed"
USE WHEN: Feature name is strong enough to lead (rare -- requires excellent PDP-02 output)
WORD BUDGET: 5-10 words
HERO FEATURE FIT: Maximum -- feature IS the headline
```

---

## SELECTION PROTOCOL

### Step 1: Evaluate Product Fit
```
FOR each approach:
  ASSESS fit based on:
    product_type: [physical|supplement|digital]
    page_type: [pdp|collection|bundle]
    hero_feature_strength: [from PDP-02 scores]
    available_proof: [numbers, reviews, results]
    target_audience: [awareness level, sophistication]

  SCORE approach_fit: [1-10]
```

### Step 2: Select Primary + Secondary
```
SELECT:
  primary_approach: highest scoring approach
  secondary_approach: second highest (for variety in generation)

NOTE: Layer 2 will generate candidates using both approaches
```

## OUTPUT STRUCTURE

```json
{
  "headline_approach": {
    "primary": {
      "approach": "benefit_specific",
      "rationale": "Clear quantifiable benefit available, hero feature enables it",
      "fit_score": 9
    },
    "secondary": {
      "approach": "hero_feature_led",
      "rationale": "Hero feature name is strong enough to lead",
      "fit_score": 7
    },
    "rejected": [
      { "approach": "social_proof_led", "reason": "Insufficient proof numbers available" }
    ]
  }
}
```

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| WARNING | No approach scores > 7 | Present options to human for direction |
| WARNING | Multiple approaches tied | Select most specific approach |
| INFO | Clear winner | Proceed with top approach |
