# 4.1-scroll-stop-validator.md

> **Layer:** 4 (Validation)
> **Skill:** PDP-03-hero-value-prop
> **Purpose:** Validate hero stops the scroll
> **Outputs:** `scroll_stop_result` object with PASS/FAIL

---

## FUNCTION

Validate that the complete hero section -- headline, subhead, value prop, highlights, and carousel -- would stop a scrolling prospect and earn the scroll-down. This is a holistic assessment of the ATF experience.

---

## VALIDATION PROTOCOL

### Step 1: 3-Second Scan Test
```
SIMULATE a 3-second scan of the hero section:
  IN 3 SECONDS, does the user absorb:
    [ ] What the product IS (category/type)
    [ ] What the product DOES (core benefit)
    [ ] Why they should CARE (relevance to them)
    [ ] What to DO next (CTA visible)

  SCORING:
    4/4 = PASS -- hero fully communicates in scan time
    3/4 = MARGINAL PASS -- one element requires reading
    2/4 = FAIL -- hero requires reading to understand
    1/4 = CRITICAL FAIL -- hero is confusing at scan speed
```

### Step 2: Pattern Interrupt Assessment
```
EVALUATE headline for pattern interrupt:
  Does the headline BREAK expected patterns?
    Specificity interrupt: Unexpected number or detail
    Curiosity interrupt: Gap that demands resolution
    Contradiction interrupt: Challenges assumption
    Emotional interrupt: Triggers visceral response

  SCORE pattern_interrupt: [1-10]
  MINIMUM: 7 for hero headlines
```

### Step 3: Hierarchy Validation
```
CHECK visual hierarchy of copy elements:
  [ ] Headline is the MOST prominent text element
  [ ] Subhead is clearly secondary
  [ ] Value prop is supporting, not competing with headline
  [ ] CTA is distinct and actionable
  [ ] Highlights are scannable (not paragraph blocks)

  IF hierarchy unclear: FAIL with specific fix guidance
```

### Step 4: Completeness Check
```
VERIFY all hero components present:
  [ ] Headline (5-12 words)
  [ ] Subhead (10-20 words)
  [ ] Value proposition (15-30 words)
  [ ] CTA (3-5 words)
  [ ] Product highlights (3-5 items)
  [ ] Carousel copy (10 positions)

  IF any missing: FAIL -- complete before proceeding
```

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | 3-second scan score <= 2/4 | Rewrite hero for scan clarity |
| CRITICAL | Pattern interrupt < 7 | Strengthen headline with specificity or curiosity |
| WARNING | 3-second scan score 3/4 | Tighten the weak element |
| INFO | All checks pass | Proceed to packaging |
