# 2.4 — Price Display Architecture

**Skill:** PDP-03-hero-carousel-buybox
**Position:** Layer 2, Microskill 4
**Purpose:** Architect the price display component including anchor price, struck-through original, savings in both percentage AND dollar amount, per-unit pricing for multi-packs, and shipping proximity. Baymard: savings must be shown in both % and $ formats because users process one or the other depending on price point.
**Effort Level:** medium
**Dependencies:** 0.1 input-verification.md
**Output:** `price-display.md`

---

## Input Schema

```yaml
input:
  input_verification: "[path to input-verification.md]"
  otp_price: "[one-time purchase price]"
  subscription_price: "[subscription price, or 'N/A']"
  original_price: "[MSRP / compare-at price, or 'N/A']"
  multi_pack_options: "[bundle pricing, or 'N/A']"
  shipping_threshold: "[free shipping threshold, or 'always free' / 'flat rate']"
```

---

## Execution Logic

**Step 1:** Extract all pricing data from `input-verification.md`.

**Step 2:** Determine price display structure based on pricing model:

### Single Product (No Subscription)
```
[$XX.XX]  [$YY.YY struck-through]  [Save $Z.ZZ (XX%)]
[Shipping line]
```

### Single Product with Subscription
```
Default view shows OTP price:
[$XX.XX]  [$YY.YY struck-through]  [Save $Z.ZZ (XX%)]
[Shipping line]

When S&S tile selected, price updates:
[$AA.AA/month]  [$XX.XX struck-through]  [Save $B.BB (XX%)]
[Free shipping included]
```

### Multi-Pack Options
```
1 Bottle: [$XX.XX]  [$YY.YY struck-through]  [Save XX%]
3 Bottles: [$AA.AA]  [$BB.BB struck-through]  [Save XX% — Best Value]
6 Bottles: [$CC.CC]  [$DD.DD struck-through]  [Save XX% — Most Popular]

Per-unit: [$X.XX/bottle]
```

**Step 3:** Calculate savings in BOTH formats:
- Dollar savings: Original price - Sale price = $X.XX saved
- Percentage savings: ((Original - Sale) / Original) × 100 = XX% off
- BOTH must be displayed. One or the other is not sufficient.

**Step 4:** Specify per-unit pricing (if applicable):
- Multi-packs: Show price per unit ("$X.XX per bottle")
- Subscription: Show price per month/delivery
- Daily cost: Optional ("Less than $X.XX per day") — powerful for supplements

**Step 5:** Specify shipping proximity:
- **Position:** Directly below or adjacent to price
- **Content:** "Free Shipping" or "Free Shipping on orders over $XX" or "$X.XX Shipping"
- **Why here:** Baymard shows shipping cost visibility near price reduces cart abandonment
- **Color:** Muted (secondary text), green checkmark for "Free Shipping"

**Step 6:** Specify price typography:
- **Current price:** Large, bold, primary color (24-28px desktop, 20-24px mobile)
- **Original/compare-at:** Smaller, struck-through, muted gray (16-18px)
- **Savings:** Small, accent color (green or red), (14-16px)
- **Per-unit:** Small, muted, below main price (12-14px)
- **Shipping:** Small, muted, with icon if free (14px)

**Step 7:** Specify dynamic price update behavior:
- When variant changes → price updates immediately (no page reload)
- When subscription tile selected → price updates to subscription price
- When quantity changes → total price updates (if showing total)
- Savings recalculate for each combination

**Step 8:** Write `price-display.md`.

## Output Schema

`price-display.md`:

```markdown
# Price Display Architecture — PDP-03

## Product: [product name]
## Pricing Model: [single | single+subscription | multi-pack | multi-pack+subscription]

---

## Price Structure

### Default View (OTP)
- **Current Price:** $[XX.XX]
- **Compare-At Price:** $[YY.YY] (struck-through)
- **Savings (Dollar):** Save $[Z.ZZ]
- **Savings (Percent):** ([XX]% off)
- **Per-Unit:** $[X.XX] per [unit] (if multi-pack)
- **Daily Cost:** $[X.XX] per day (if applicable)

### Subscription View (if applicable)
- **Subscription Price:** $[AA.AA]/[frequency]
- **Compare-At:** $[XX.XX] (OTP price, struck-through)
- **Savings vs OTP:** Save $[B.BB] ([XX]%)
- **Includes:** Free shipping

### Multi-Pack Tiers (if applicable)
| Quantity | Price | Per-Unit | Savings | Badge |
|----------|-------|----------|---------|-------|
| 1 | $XX.XX | $XX.XX | Save XX% | — |
| 3 | $XX.XX | $XX.XX | Save XX% | Best Value |
| 6 | $XX.XX | $XX.XX | Save XX% | Most Popular |

## Shipping Proximity
- **Content:** [exact shipping text]
- **Position:** Below price, same visual block
- **Style:** [14px, muted, green checkmark if free]

## Typography
| Element | Size (Desktop) | Size (Mobile) | Weight | Color |
|---------|---------------|---------------|--------|-------|
| Current price | [24-28px] | [20-24px] | Bold | Primary |
| Compare-at | [16-18px] | [14-16px] | Regular | Gray, struck-through |
| Savings | [14-16px] | [14px] | Medium | Accent (green/red) |
| Per-unit | [12-14px] | [12px] | Regular | Muted |
| Shipping | [14px] | [14px] | Regular | Muted / Green |

## Dynamic Behavior
- Variant change → price updates immediately
- Subscription toggle → price recalculates
- Quantity change → total updates
- All savings recalculate per combination

## Position
- **Below:** Short description (2.3)
- **Above:** Variant selector (2.5)

## Forbidden
- Showing savings in only one format (must be BOTH % AND $)
- Hiding shipping cost (must be visible near price)
- Rounding savings ("about 20%" → exact "18%")
- Showing subscription price without comparing to OTP
```

---

## Quality Criteria
- Savings shown in BOTH dollar AND percentage formats
- Shipping information visible adjacent to/below price
- Per-unit pricing calculated for multi-packs
- Dynamic update behavior specified for all selector interactions
- Typography specs include exact sizes for both mobile and desktop
- All price points from brief are represented
- File exists after execution

## Failure Modes
- No compare-at/original price in brief → Show current price only, no savings display
- Subscription pricing not available → Omit subscription view, proceed with OTP only
- Shipping info not in brief → Flag gap, use placeholder "Shipping calculated at checkout"
- input-verification.md not found → HALT, run 0.1 first
- Price calculations don't match (% and $ savings inconsistent) → Recalculate and fix
