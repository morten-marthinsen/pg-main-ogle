# 2.10 — Micro-Trust Signals

**Skill:** PDP-03-hero-carousel-buybox
**Position:** Layer 2, Microskill 10
**Purpose:** Specify the 3-4 micro-trust signals displayed in small text below the ATC button and express payments. These address last-second purchase anxiety — the moment between "I want this" and "I'll buy this." Baymard documents specific anxieties (security, shipping, returns) that cost conversions when unaddressed.
**Effort Level:** medium
**Dependencies:** 0.1 input-verification.md, 2.8 atc-button.md, 2.9 express-payments.md
**Output:** `micro-trust.md`

---

## Execution Logic

**Step 1:** Identify the 4 primary trust anxiety categories from Baymard research:

| Anxiety | Question User Asks | Signal That Resolves It |
|---------|-------------------|----------------------|
| Security | "Is my payment info safe?" | "Secure Checkout" + lock icon |
| Shipping | "How much is shipping? How fast?" | "Free Shipping" or "Ships within 24 hours" |
| Returns | "What if I don't like it?" | "30-Day Money-Back Guarantee" |
| Quality | "Is this legit?" | "Made in USA" / "Third-Party Tested" / "GMP Certified" |

**Step 2:** Select 3-4 signals from the brief data. Each signal must be:
- **True:** Only claim what the product/company actually offers
- **Specific:** "30-Day Money-Back Guarantee" not "Satisfaction Guaranteed"
- **Concise:** 2-5 words per signal
- **Relevant:** Addresses an actual purchase anxiety for this product category

**Step 3:** Compose signal copy:
- Each signal: [small icon] + [short text]
- Maximum 3-4 signals (more = visual clutter = reduced effectiveness)
- Icons: Small (12-14px), line-art style, muted color

Common signal patterns:
| Signal Type | Copy Examples | Icon |
|------------|--------------|------|
| Security | "Secure Checkout" | Lock |
| Shipping | "Free Shipping" / "Free Shipping $50+" / "Ships Tomorrow" | Truck / Box |
| Returns | "30-Day Money-Back Guarantee" / "Easy Returns" | Refresh / Shield |
| Quality | "Made in USA" / "Third-Party Tested" / "GMP Certified" | Flag / Badge / Checkmark |
| Support | "24/7 Customer Support" | Headset |

**Step 4:** Specify visual treatment:
- **Position:** Below express payments (2.9), final element in buy box
- **Layout:** Single line if 3 signals (separated by • or |), or 2×2 grid if 4 signals
- **Typography:** 12-13px, muted gray, regular weight
- **Icons:** 12-14px, inline with text, same gray color
- **Mobile:** Same layout, may wrap to 2 lines for 4 signals

**Step 5:** Specify what micro-trust signals are NOT:
- NOT a badge wall (no logos, no large certification images)
- NOT a marketing message ("Join 50,000+ happy customers" goes elsewhere)
- NOT a selling point (benefits go in short description, not here)
- NOT clickable links (static text only — detail pages for guarantee/shipping are in FAQ)

**Step 6:** Write `micro-trust.md`.

## Quality Criteria
- Exactly 3-4 signals (not 1, not 8)
- Each signal addresses a specific purchase anxiety (security, shipping, returns, quality)
- Each signal is verified from brief data (not fabricated)
- Each signal is concise (2-5 words)
- Visual treatment is muted (not competing with ATC button for attention)
- No marketing claims disguised as trust signals
- No false regulatory claims (especially "FDA Approved" for supplements)
- File exists after execution

## Failure Modes
- No trust signal data in brief → Use generic safe defaults: "Secure Checkout", "Free Shipping" (if true), "Money-Back Guarantee" (if true). Flag gaps.
- Fewer than 3 verifiable signals → Include "Secure Checkout" (always true for HTTPS sites) as one signal. Flag gap for remaining.
- "FDA Approved" claimed for supplements → HALT. Supplements are NOT FDA approved. Use "FDA-Registered Facility" or "GMP Certified" if applicable.
- More than 4 signals specified → Reduce to top 4 by anxiety priority (Security > Shipping > Returns > Quality).
- input-verification.md not found → HALT, run 0.1 first
