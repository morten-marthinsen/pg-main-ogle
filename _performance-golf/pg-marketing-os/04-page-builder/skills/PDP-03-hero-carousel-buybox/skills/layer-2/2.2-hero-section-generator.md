# 2.2-hero-section-generator.md

> **Layer:** 2 (Hero Generation)
> **Skill:** PDP-03-hero-value-prop
> **Purpose:** Generate value prop + CTA + highlights + thumbnail copy
> **Outputs:** Complete hero section (everything except headline/subhead)

---

## FUNCTION

Generate the remaining hero section components: value proposition, CTA text, product highlights TLDR, and 10-thumbnail carousel copy. These components work WITH the selected headline/subhead pairs to create the complete above-fold experience.

---

## GENERATION PROTOCOL

### Step 1: Value Proposition (15-30 words)
```
WRITE value prop that answers "Why this product, why now?"

  STRUCTURE:
    "[Product/Feature] + [core mechanism/approach] + [specific outcome] + [urgency/relevance]"

  CONSTRAINTS:
    - 15-30 words
    - NOT a feature list
    - NOT a repeat of headline
    - IS a reason to care RIGHT NOW
    - MUST include at least one specific element

  EXAMPLES:
    GOOD: "Engineered with HyperSpeed Face technology to add 15+ yards
           of carry distance on every drive -- without changing your swing."
    BAD: "A revolutionary new golf club with advanced technology, premium
          materials, and innovative design." (feature list, generic, slop words)
```

### Step 2: CTA Text (3-5 words)
```
WRITE action-specific CTA:

  APPROVED CTA PATTERNS:
    "Shop [Product]" -- direct, product-specific
    "Get Yours Today" -- ownership + urgency
    "Try Risk-Free" -- lowers barrier
    "Add to Cart" -- standard ecom
    "See It In Action" -- for video-heavy products
    "Build Your [X]" -- for customizable products

  BANNED CTA PATTERNS:
    "Learn More" -- passive, no commitment
    "Click Here" -- outdated, unhelpful
    "Submit" -- clinical, uninviting
    "Discover" -- AI telltale
    "Explore" -- too vague for ecom
```

### Step 3: Product Highlights TLDR
```
FROM highlights_selection (1.3 output):
  FORMAT each highlight as a scan-friendly line:
    [Icon/Checkmark] [Feature Name] -- [10-20 word benefit statement]

  CONSTRAINTS:
    - 3-5 highlights total
    - Hero feature is ALWAYS highlight #1
    - 50-100 words total
    - Each highlight standalone-readable
    - Use exact feature names from PDP-02
```

### Step 4: 10-Thumbnail Carousel Copy
```
FROM carousel_plan (1.2 output):
  WRITE copy overlay for each of 10 positions:

  PER POSITION:
    - 5-15 words of overlay text
    - Text must work WITH the planned visual (not compete)
    - Each position adds NEW information
    - Text must be readable at thumbnail size (short, punchy)

  QUALITY CHECK:
    [ ] All 10 positions have copy
    [ ] No position repeats another
    [ ] Narrative progresses logically
    [ ] Hero feature appears by position 3
    [ ] Position 10 includes CTA
```

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Value prop reads as feature list | Rewrite as "why now" statement |
| CRITICAL | CTA is passive (Learn More) | Replace with action CTA |
| CRITICAL | Carousel has fewer than 10 positions | Complete all positions |
| WARNING | Highlights exceed 100 words | Compress individual highlights |
| WARNING | Carousel positions repeat content | Rewrite with unique content per position |
