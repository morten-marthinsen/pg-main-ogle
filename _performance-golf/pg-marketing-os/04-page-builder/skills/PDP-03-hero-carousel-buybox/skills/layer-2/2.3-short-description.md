# 2.3 — Short Description Architecture

**Skill:** PDP-03-hero-carousel-buybox
**Position:** Layer 2, Microskill 3
**Purpose:** Architect the short description / key benefits list that appears in the buy box between rating strip and price. This is a 3-5 bullet-point list of specific, benefit-forward claims that answer "why should I buy this?" in scan-readable format.
**Effort Level:** medium
**Dependencies:** 0.1 input-verification.md, 1.1 ksp-inventory.md
**Output:** `short-description.md`

---

## Input Schema

```yaml
input:
  input_verification: "[path to input-verification.md]"
  ksp_inventory: "[path to ksp-inventory.md]"
```

---

## Execution Logic

**Step 1:** Read `ksp-inventory.md` to get the ranked KSPs.

**Step 2:** Select 3-5 KSPs for the short description bullets. Prioritize:
- Highest-ranked KSPs by composite score
- KSPs that answer the most common purchase objections
- KSPs that differentiate from competitors
- AVOID duplicating what's already visible in the carousel (Slide 1 headline)

**Step 3:** Convert each selected KSP into a benefit bullet:
- **Format:** Icon/checkmark + benefit statement
- **Length:** 5-12 words per bullet
- **Structure:** Benefit-forward (what it does for YOU), not feature-forward (what's in it)
- **Specificity:** Include numbers, timeframes, or measurable claims where possible

Good examples:
- "Supports calm focus for 6+ hours"
- "Third-party tested for 150+ contaminants"
- "Clinically studied KSM-66 Ashwagandha"
- "Formulated by board-certified neurologist"

Bad examples:
- "High quality ingredients" (vague)
- "Natural formula" (meaningless)
- "Premium supplement" (says nothing)
- "Advanced nootropic blend" (jargon)

**Step 4:** Specify visual treatment:
- **Icon:** Small checkmark, bullet point, or category-specific icon (e.g., leaf for natural, shield for tested)
- **Typography:** 14-16px, regular weight, secondary text color
- **Spacing:** 4-6px between bullets
- **Mobile:** Same format, may reduce to 3 bullets to save vertical space

**Step 5:** Specify what the short description is NOT:
- NOT a paragraph of marketing copy
- NOT an ingredient list (that's in the facts panel)
- NOT a repeat of the product title
- NOT a wall of badges or certifications (those are micro-trust signals below ATC)

**Step 6:** Write `short-description.md`.

## Quality Criteria
- 3-5 bullets, each 5-12 words
- Every bullet is benefit-forward (not feature-forward)
- Every bullet includes a specific claim (number, timeframe, certification name)
- No vague/generic language ("high quality", "premium", "advanced")
- Each bullet maps to a ranked KSP
- Visual treatment specified with exact sizes and colors
- File exists after execution

## Failure Modes
- Fewer than 3 distinct KSPs available → Use 3 bullets minimum, flag gap
- All bullets sound the same → Vary benefit categories (efficacy, quality, ease, credibility)
- Bullets are feature-forward ("Contains 500mg X") → Rewrite as benefit-forward ("500mg X for [benefit]")
- ksp-inventory.md not found → HALT, run 1.1 first
- Bullets exceed 12 words → Trim to be more concise
