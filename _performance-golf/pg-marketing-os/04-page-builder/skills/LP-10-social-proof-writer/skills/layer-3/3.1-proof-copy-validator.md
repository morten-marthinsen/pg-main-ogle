# 3.1 — Proof Copy Validator

**Skill:** LP-10-social-proof-writer
**Position:** Layer 3, Microskill 1
**Purpose:** Score all proof copy against the 8-point proof copy checklist from LP-10-AGENT.md. This is the quality gate that determines whether the proof copy package is ready for assembly. Minimum 7.5/8 (i.e., 7/8 passes) to proceed.
**Effort Level:** medium
**Dependencies:** All Layer 2 outputs (testimonial-intros.md, formatted-testimonials.md, review-section-copy.md, before-after-captions.md, expert-endorsements.md, proof-transitions.md, volume-signals.md)
**Output:** `proof-copy-validation.md`

```yaml
input:
  layer_2_outputs:
```

## The 8-Point Proof Copy Checklist

Score each point 0 (fail) or 1 (pass).

### Point 1: Testimonial Specificity (Specificity Category)
**Check:** Does every formatted testimonial contain a specific outcome with a number, timeframe, or measurable result?
**How to verify:** Review each testimonial in `formatted-testimonials.md`. Check that the bolded sentence contains a specific metric. "I feel better" = FAIL. "I lost 23 lbs in 8 weeks" = PASS.
**Score:** [0 | 1]

### Point 2: Volume Signal Exactness (Specificity Category)
**Check:** Do all volume signals use exact numbers (not rounded generics)?
**How to verify:** Review each signal in `volume-signals.md`. Check that numbers are exact or properly flagged with "+" notation. "Over 1,500" = FAIL. "1,561" or "50,000+" = PASS.
**Score:** [0 | 1]

### Point 3: Voice Alignment (Voice Category)
**Check:** Does proof copy tone match the page-brief.json voice direction?
**How to verify:** Read all proof copy as a cohesive unit. Compare register, vocabulary level, and emotional tone against voice direction. Formal voice on a conversational page = FAIL.
**Score:** [0 | 1]

### Point 4: Intro Register Match (Voice Category)
**Check:** Do testimonial intros match the page's narrative register? (Conversational for Type A, scannable for Type B)
**How to verify:** Check intros in `testimonial-intros.md` against page type. Type A prose intros on a Type B page = FAIL. Type B scannable headers on a Type A page = FAIL.
**Score:** [0 | 1]

### Point 5: Transition Uniqueness (Transition Quality Category)
**Check:** Does every proof block have a unique IN transition? (No repeated phrases across blocks)
**How to verify:** List all IN transitions from `proof-transitions.md`. Compare phrasing and technique. Any repetition = FAIL.
**Score:** [0 | 1]

### Point 6: Transition Propulsion (Transition Quality Category)
**Check:** Do OUT transitions propel toward the next section? (Not dead-end the reader)
**How to verify:** For each OUT transition, verify it bridges to the specific next section's topic. Summaries without forward propulsion = FAIL.
**Score:** [0 | 1]

### Point 7: Slot Coverage (Completeness Category)
**Check:** Does every slot_id from proof-architecture.json have corresponding copy?
**How to verify:** Count slot_ids in proof-architecture.json. Count slot_ids addressed across all Layer 2 outputs. Must be 100% (real copy + placeholder with sourcing flag both count).
**Score:** [0 | 1]

### Point 8: Anti-Slop Clean (Completeness Category)
**Check:** Zero AI telltales in any proof copy element?
**How to verify:** Run mental scan of all Layer 2 copy against the 5-category forbidden list in LP-10-AGENT.md. Any instance = FAIL.
**Score:** [0 | 1]

---

## Execution Steps

1. **Load all Layer 2 outputs** that were generated (skip microskills that were legitimately skipped due to 0 assigned slots).

2. **Run each of the 8 checks** independently. For each check:
   - State the evidence (what was found)
   - Score 0 or 1
   - If 0: list every failing element with specific details

3. **Calculate total score.** Minimum 7/8 to pass (7.5/8 threshold = 7/8 rounded).

4. **If score < 7/8:** Identify which checks failed and write specific revision requirements for each failure.

5. **Write `proof-copy-validation.md`.**

## Quality Criteria
- All 8 points explicitly scored with evidence
- Failing points include specific element-level detail (which testimonial, which transition, which volume signal)
- Score calculation is correct (simple sum of 0/1 scores)
- Revision requirements are actionable (not just "fix the transitions" but "transition for Block 2 uses same technique as Block 1 — rewrite Block 2 IN using the Volume Reveal technique")

## Failure Modes
- Scoring PASS on voice alignment without actually comparing to page-brief voice direction -> Must reference specific voice direction characteristics from page-brief.json.
- Counting `requires_sourcing` placeholders as missing for slot coverage -> Placeholders WITH sourcing flags count as addressed. Only truly missing slots (no output at all) = FAIL.
- Passing anti-slop check without scanning all 5 categories -> Every category from the LP-10-AGENT.md forbidden list must be checked. Missing a category = incomplete scan.
- Scoring 6/8 and calling it "close enough" -> 6/8 = FAIL. 7/8 = PASS. The threshold is not negotiable.
