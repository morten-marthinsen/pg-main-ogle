# 2.7 — Volume Signal Writer

**Skill:** LP-10-social-proof-writer
**Position:** Layer 2, Microskill 7
**Purpose:** Write copy for customer count signals, stat bars, review volume displays, and "join X+ customers" messaging. Volume signals create trust through sheer scale — they answer the reader's unspoken question: "How many OTHER people trust this?" The numbers must be exact, the framing must be persuasive, and the placement must match the wave pattern.
**Effort Level:** medium
**Dependencies:** `proof-slot-analysis.md` (1.1), `proof-slots-loaded.md` (0.2), `brief-loaded.md` (0.1), `specimen-proof-copy.md` (0.3)
**Output:** `volume-signals.md`

## Volume Signal Types

### 1. Customer Count Signal
**What it is:** "Join 50,000+ customers who..."
**Rules:**
- Use exact number where available. If only approximate: use "+" notation (50,000+, not "over 50,000")
- Frame the number in terms of the reader's DECISION, not just the brand's vanity metric
- Connect the number to a relevant outcome or community

**Formulas:**
- Decision frame: "Join [exact count]+ [audience descriptor] who've already [specific action]."
- Community frame: "[Exact count] [audience descriptor] trust [product] with their [specific concern]."
- Momentum frame: "Every [timeframe], [count] more [audience descriptor] make the switch to [product]."
- Authority frame: "Trusted by [count] [credentialed descriptor] across [count] countries."

### 2. Stat Bar (3-Number Strip)
**What it is:** A horizontal bar with 3 key statistics, separated by dividers.
**Rules:**
- Exactly 3 statistics
- Each stat: [NUMBER] + [LABEL — 3-5 words maximum]
- Numbers must be impressive at a glance
- Labels must be instantly parseable (no complex phrases)
- At least one stat must be audience-scale (customers/reviews), one must be credibility (years/satisfaction), and one should be outcome-related

**Example:**
```
50,000+ Customers Served | 4.8/5 Average Rating | 99.3% Satisfaction Rate
```

**Alternative patterns:**
```
1,270,268 Bottles Sold | 38 Clinical Studies | 365-Day Guarantee
```
```
$30M+ Client Revenue | 1,200+ Coaches Trained | 500+ Video Testimonials
```

### 3. Social Proof Counter
**What it is:** A standalone number display, often with animation (counter) or real-time update.
**Rules:**
- Single number, large display
- Contextualizing label below/beside
- Use exact numbers — counters feel more credible when they end in non-round digits

**Example:**
```
1,270,268
customers have made the switch
```

### 4. Review Volume Signal
**What it is:** The review count as a proof element (separate from the review section header in 2.3).
**Rules:**
- Exact count
- Star rating included
- "Verified" signal where applicable
- Can include a trend signal ("200+ new reviews this month")

**Example:**
```
Rated 4.8/5 by 83,502 verified customers
```

---

## Execution Rules

1. **EXACT numbers only.** "1,561" not "over 1,500." "83,502" not "80,000+." If only approximate data exists, use the "+" notation with the precise floor ("50,000+").

2. **Never fabricate numbers.** If customer count, review count, or other metrics are unavailable, write placeholder copy with `requires_verification` flag.

3. **Frame numbers in terms of the READER's decision.** "50,000 customers" is a vanity metric. "50,000 people trust this with their health" is a decision signal.

4. **Match volume signal type to placement:**
   - Above fold: Rating strip or social proof counter
   - Social proof strip (mid-page): Stat bar
   - Within proof blocks: Customer count signal
   - Near CTA: Momentum frame

5. **Audience descriptor must match the target audience.** "50,000 customers" is generic. "50,000 golfers over 50" is targeted.

---

## Execution Steps

1. **For each volume signal slot,** determine the signal type based on format and placement.
2. **Load available numbers** from proof inventory.
3. **Write the signal copy** using the appropriate formula and formatting rules.
4. **Verify all numbers are exact** (or properly flagged as approximate with "+" notation).
5. **Verify audience descriptor matches** the target audience from page-brief.json.
6. **If data is unavailable:** Write placeholder with `requires_verification` flag and specify what data is needed.

## Quality Criteria
- All numbers are exact or properly flagged with "+" notation
- Zero fabricated metrics
- Audience descriptors match the target audience from page-brief.json
- Numbers framed as decision signals, not vanity metrics
- Stat bars have exactly 3 statistics with 3-5 word labels
- Signal types appropriate for their placement (above fold, mid-page, near CTA)
- All copy matches page voice direction

## Failure Modes
- Rounding numbers for "cleaner" copy -> 1,561 is more credible than "over 1,500." Use exact.
- Generic audience descriptors ("customers", "people") -> Use targeted descriptors from page-brief.json audience profile.
- Vanity metrics without reader framing ("We've sold 1M bottles") -> Frame as reader decision ("1M people trust this with their health").
- Stat bar with more or fewer than 3 statistics -> Exactly 3. More is cluttered, fewer lacks impact.
- Fabricating impressive numbers -> If data unavailable, write placeholder. Never fabricate.
