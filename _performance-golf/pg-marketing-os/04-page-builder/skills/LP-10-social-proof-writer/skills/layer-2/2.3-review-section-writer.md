# 2.3 — Review Section Writer

**Skill:** LP-10-social-proof-writer
**Position:** Layer 2, Microskill 3
**Purpose:** Write copy for review cascade sections — section headers with star ratings and review counts, filtering option labels, rating distribution context, and review highlight copy. This is primarily a Type B skill (ecomm/PDP) but also applies to Hybrid pages with review sections.
**Effort Level:** high
**Dependencies:** `proof-slot-analysis.md` (1.1), `proof-slots-loaded.md` (0.2), `brief-loaded.md` (0.1), `specimen-proof-copy.md` (0.3)
**Output:** `review-section-copy.md`

## Review Section Copy Types

### 1. Review Section Header
The primary header that opens the review cascade section.

**Rules:**
- Use EXACT review count — never round ("1,561 Reviews" not "Over 1,500 Reviews")
- Include star rating if available ("4.8 out of 5 stars")
- Include "Verified" signal where applicable
- Keep to 1-2 lines

**Formulas:**
- `[Star Rating] out of 5 — [Exact Count] Verified Reviews`
- `[Exact Count] [Audience Type] Have Rated [Product Name]`
- `Rated [Star Rating]/5 by [Exact Count] Real Customers`

### 2. Rating Distribution Context
Copy that accompanies the star rating breakdown (5-star: 72%, 4-star: 18%, etc.).

**Rules:**
- Only write this if rating distribution data is available
- Frame the distribution positively without being dishonest
- Acknowledge lower ratings exist (transparency builds trust)

**Example:** "5-star: 72% | 4-star: 18% | 3-star: 7% | 2-star: 2% | 1-star: 1% — 90% of reviewers rate [product] 4 stars or higher."

### 3. Filter Option Labels
Copy for review filtering buttons/dropdowns (if cascade supports filtering).

**Standard filter labels:**
- By rating: "5 Stars", "4 Stars", "3 Stars", "All Ratings"
- By concern: [Vertical-specific — e.g., "Sleep", "Energy", "Stress" for supplements]
- By buyer type: "Verified Purchase", "Repeat Buyer"
- By recency: "Most Recent", "Most Helpful", "Highest Rated"

**Rules:**
- Filter labels should use audience language for concerns (not medical/technical terms)
- Include 3-5 concern-based filters based on the product's primary benefits
- Always include "Most Helpful" and "Verified Purchase" options

### 4. Review Highlight Copy
Short labels that appear above featured/pinned reviews.

**Formulas:**
- "Most Helpful Review" / "Top Review" / "Editor's Pick"
- "[Concern/Benefit] Highlight" — e.g., "Sleep Improvement Highlight"
- "Recently Added" / "This Week's Top Review"

### 5. Rating Strip Copy (Above Fold)
Short copy that accompanies the rating strip element above the fold.

**Rules:**
- Maximum 1 line
- Include star rating + review count
- May include qualifier ("from [source]" or "Verified Buyers")

**Example:** "★★★★★ 4.8/5 from 1,561 verified reviews"

---

## Execution Steps

1. **For each review section slot,** determine which copy types are needed (header, distribution, filters, highlights, rating strip).

2. **Write the review section header** using exact numbers from review data. Choose the formula that best fits the page's voice and the number magnitude.

3. **Write rating distribution context** if data is available.

4. **Write filter labels** — standard options plus 3-5 vertical-specific concern labels derived from the product's benefit categories.

5. **Write review highlight labels** for any featured/pinned review positions.

6. **Write rating strip copy** for any above-fold rating strip elements.

7. **If review data is unavailable** (`requires_sourcing`): Write placeholder copy with sourcing flags indicating what data is needed.

## Quality Criteria
- Review count is EXACT — never rounded
- Star rating included where available
- Filter labels use audience language, not technical terminology
- Concern-based filters are vertical-specific and relevant to the product
- Rating strip copy fits in 1 line
- All copy matches page voice direction
- Zero generic review section headers

## Failure Modes
- Rounding review counts ("over 1,500" instead of "1,561") -> Always use exact numbers. Specificity = credibility.
- Using the same header for multiple review sections on the same page -> Each review section gets unique header copy.
- Filter labels using medical/technical terms -> Use the language the audience uses, not clinical terminology.
- Writing review section copy for Type A pages that don't have review cascades -> Check proof-slot-analysis.md. Only write review section copy for slots classified as review_section type.
- Missing review data but writing as if it exists -> If data is unavailable, write placeholder copy with clear sourcing flags. Do not fabricate review counts.
