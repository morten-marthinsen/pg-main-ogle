# 2.2 — Testimonial Formatter

**Skill:** LP-10-social-proof-writer
**Position:** Layer 2, Microskill 2
**Purpose:** Execute the testimonial selection plan from 1.2. For each assigned testimonial slot: format the testimonial with proper attribution, bold the key outcome sentence, structure for maximum readability, and handle video testimonial CTAs. This is the highest-volume writing task in LP-10.
**Effort Level:** high
**Dependencies:** `testimonial-plan.md` (1.2), `proof-slots-loaded.md` (0.2), `brief-loaded.md` (0.1)
**Output:** `formatted-testimonials.md`

## Formatting Rules

### Attribution Format

Every testimonial must include attribution. The format varies by proof type:

| Proof Format | Attribution Pattern | Example |
|-------------|-------------------|---------|
| `text_testimonial` | Name, Location | — Sarah M., Denver, CO |
| `photo_testimonial` | Name, Age, Location + [PHOTO] indicator | — Michael T., 52, Austin, TX [PHOTO] |
| `video_testimonial` | Name + Pull-Quote + Duration + CTA | [See Video Testimonial CTA section below] |
| `platform_review` | Name + Star Rating + "Verified Buyer" | — Jennifer K. ★★★★★ Verified Buyer |

**Attribution rules:**
- Always use first name + last initial (or full name if public figure)
- Include at least ONE qualifier beyond the name (location, age, title, "verified buyer")
- If testimonial has a photo, indicate with [PHOTO] tag for LP-15 assembly
- If no real name available, use "Verified Customer" with location

### Bold Strategy

In every testimonial, bold the ONE sentence that contains the specific outcome. This is what the scanning reader sees first.

**Bold the sentence that answers:** "What specific result did this person get?"

**Good bold:** **"I went from a 15 handicap to a 9 in four months."**
**Bad bold:** **"I love this product so much!"** (no specific result)

**If the testimonial has no single clear outcome sentence:** Bold the most specific claim. If nothing is specific enough, this testimonial should have been flagged in 1.2 and either improved or replaced.

### Length Treatment

| Page Type | Standard Testimonial | Featured Testimonial |
|-----------|---------------------|---------------------|
| Type A | 40-80 words | 80-150 words |
| Type B | 20-50 words | 50-80 words |

**For available testimonials that are too long:** Excerpt the most relevant portion. Mark with "[...]" where text was cut. Keep the specific outcome sentence intact.

**For available testimonials that are too short:** Use as-is if specific. Do NOT pad with filler language.

### Video Testimonial CTA Format

```
[VIDEO EMBED — Duration: X:XX]
Pull-quote: "[Most compelling sentence from the video]"
— [Name], [Qualifier]
CTA: Watch [Name]'s Full Story →
```

**Pull-quote rules:**
- Select the single most specific, most emotional sentence from the video
- The pull-quote should create enough curiosity to click play
- Keep under 20 words

### Placeholder Format (for `requires_sourcing` slots)

```
[TESTIMONIAL NEEDED — Slot: {slot_id}]
Sourcing spec: [From testimonial-plan.md sourcing specification]
Required: [outcome type] from [audience segment] with [specificity requirement]
Placeholder: "[Draft testimonial structure — DO NOT USE AS REAL TESTIMONIAL]"
```

---

## Execution Steps

1. **For each testimonial slot** in the testimonial plan:
   - Load the assigned testimonial content from proof inventory
   - Apply the attribution format per proof type
   - Identify and bold the key outcome sentence
   - Apply length treatment (excerpt if too long)
   - If video: format with pull-quote and CTA
   - If `requires_sourcing`: write placeholder with sourcing spec

2. **Verify specificity** of every formatted testimonial:
   - Does the bolded sentence contain a specific outcome?
   - Score specificity 1-10 (must be >= 6)
   - Flag any that fall below threshold

3. **Verify variety** across the full set:
   - Are different demographics represented?
   - Are different outcome types represented?
   - Is there a mix of short and featured-length testimonials?

## Quality Criteria
- Every testimonial slot has formatted output (real or placeholder)
- Every testimonial has proper attribution with at least one qualifier beyond name
- Every testimonial has a bolded outcome sentence with specific result
- Specificity score >= 6 for all real testimonials
- Video testimonials have pull-quotes and CTAs
- Placeholders clearly marked — no fabricated outcomes
- Length appropriate for page type (Type A vs Type B ranges)

## Failure Modes
- Bolding the wrong sentence (emotional reaction instead of specific outcome) -> The bolded sentence must answer "what result?" not "how do they feel?"
- Fabricating specific outcomes for placeholder testimonials -> Placeholders must be clearly marked. Fabrication is fraud.
- All testimonials same length -> Mix standard and featured lengths for visual variety.
- Attribution without any qualifier ("— Sarah") -> Always include location, age, title, or "Verified Buyer."
- Exceeding word count limits without excerpting -> Long testimonials must be excerpted. Keep the outcome sentence. Cut the filler.
