## Quality Criteria
- All required fields populated (no nulls in required sections — optional sections can be null)
- Primary CTA matches the selection in CTA_SELECTION.yaml exactly
- All secondary CTAs present for every position defined in the architecture
- P.S. copy matches the selected variants from CTA_SELECTION.yaml
- Final close matches the selected variant
- emotional_appeal_coverage accurately reflects the final set
- all_primary_candidates includes ALL candidates from generation (for reference)
- Validation block shows all gates PASS
- Package is self-contained — LP-15 can assemble the page using only this file

## Failure Modes
- Selected CTA text not matching CTA_SELECTION.yaml -> Exact match required. Human modifications must be applied.
- Missing secondary CTA positions -> Every position from the architecture must be present (even if null with explanation).
- P.S. text differing from selected variant -> Use exact text from CTA_SELECTION.yaml selection + modifications.
- Validation block shows PASS but audit score is below 7.5 -> Cross-check. Assembly gate should have caught this.
