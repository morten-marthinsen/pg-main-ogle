# 2.3 — P.S. Copy Writer

> Version: 1.0
> Layer: 2
> Model: opus
> Input: ps-strategy.md, packages-loaded.md, primary-cta-candidates.md
> Output: ps-copy-draft.md

**Skill:** LP-14-cta-optimizer
**Position:** Layer 2, Microskill 3
**Purpose:** Write 1-3 P.S. lines following the strategy defined in Layer 1. P.S. copy RESTATES — it never introduces new claims. Scanners read headlines, captions, and P.S. — this is the final chance to convert a reader who scrolled to the bottom without reading the body. Each P.S. line must stand alone as a mini-pitch.
**Effort Level:** high
**Dependencies:** `ps-strategy.md` (1.3), `packages-loaded.md` (0.1), `primary-cta-candidates.md` (2.1)
**Output:** `ps-copy-draft.md`

## Pre-Writing Protocol

**Step 1: Re-read P.S. strategy**
Load `ps-strategy.md`. For each P.S. line, confirm the exact benefit/claim to restate and its source on the page.

**Step 2: New Claim Prohibition Reminder**
Before writing ANY P.S. line, state: "I am restating [specific benefit/claim] that already appears in [specific section]. I am NOT introducing new information."

**Step 3: Load primary CTA candidates**
The P.S. must include a CTA reference — use the top primary CTA candidate's button text (or a close variant) as the action prompt within the P.S.

---

## Execution Logic

### P.S. #1: Benefit Restate + CTA

**Structure:**
- Sentence 1: Restate the single strongest benefit from the page (use the exact claim, not a rewrite)
- Sentence 2: Connect it to the reader's primary desire (audience language)
- Sentence 3: CTA prompt referencing the primary CTA button text
- Sentence 4 (optional): Risk removal (guarantee mention)

**Rules:**
- 2-4 sentences. Not 1. Not 5.
- The benefit must be traceable to ps-strategy.md's identified source section
- Use the audience's language for the desired outcome (Empath lens)
- Include the CTA button text or a direct reference to it

**Generate 2 variants of P.S. #1** for human selection.

### P.S. #2: Urgency/Scarcity Reminder (if planned)

**Prerequisite:** Only write if ps-strategy.md says "INCLUDE" and urgency is PERMITTED.

**Structure:**
- Sentence 1: Restate the urgency/scarcity from LP-13 (use the exact copy or close paraphrase)
- Sentence 2: Connect urgency to consequence of inaction
- Sentence 3 (optional): CTA prompt

**Rules:**
- 2-3 sentences maximum
- The urgency copy must come from LP-13 — do NOT invent deadlines, quantities, or scarcity
- If urgency is fabricated or vague ("hurry," "limited time"), the P.S. fails

**Generate 2 variants of P.S. #2** for human selection.

### P.S. #3: Final Proof Point (if planned)

**Prerequisite:** Only write if ps-strategy.md says "INCLUDE."

**Structure:**
- Sentence 1: Restate the proof point (exact stat, name, or finding from the page)
- Sentence 2: Connect it to the promise (this proof validates the core claim)
- Sentence 3 (optional): CTA prompt

**Rules:**
- 2-3 sentences maximum
- The proof point must be traceable to ps-strategy.md's identified source section
- No new studies, new stats, or new authority figures not already on the page

**Generate 2 variants of P.S. #3** for human selection.

## Quality Criteria
- P.S. count matches ps-strategy.md plan
- Each P.S. line has 2-4 sentences (not shorter, not longer)
- Every benefit/claim is traceable to a specific page section
- Zero new claims introduced — verified in New Claim Audit
- CTA reference included in at least P.S. #1
- 2 variants per P.S. line for human selection
- P.S. #2 only exists if urgency is PERMITTED

## Failure Modes
- P.S. introduces a new benefit not on the page -> Immediate rewrite. Check against source section.
- P.S. exceeds 4 sentences -> Cut to 4. P.S. is scannable — long P.S. defeats its purpose.
- P.S. #2 includes urgency language without LP-13 data -> Remove P.S. #2 entirely.
- No CTA reference in P.S. #1 -> P.S. without a CTA is a dead end. Add CTA reference.
- P.S. reads like a new pitch instead of a reminder -> It should feel like "remember what I told you" not "here's one more thing."
