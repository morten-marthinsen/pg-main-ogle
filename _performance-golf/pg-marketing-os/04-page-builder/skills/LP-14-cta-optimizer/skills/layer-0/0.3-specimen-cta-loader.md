# 0.3 — Specimen CTA Loader

> Version: 1.0
> Layer: 0
> Model: haiku
> Input: packages-loaded.md, swipe-vault-index.md, specimens/
> Output: specimens-loaded.md

**Skill:** LP-14-cta-optimizer
**Position:** Layer 0, Microskill 3
**Purpose:** Load type-matched CTA specimens from the swipe vault and specimen files. Extract verbatim CTA button text, surrounding context copy, P.S. examples, and final close paragraphs. These specimens serve as structural attractors during generation — they anchor the model to real-world patterns instead of AI-generated defaults.
**Effort Level:** low
**Dependencies:** `packages-loaded.md` (0.1)
**Output:** `specimens-loaded.md`

## Execution Logic

1. **Read `packages-loaded.md`** to determine page type, vertical, and offer structure.

2. **Load specimens by match priority:**
   - Priority 1: Same vertical + same page type (e.g., health supplement Type B CTAs)
   - Priority 2: Same vertical + different page type (e.g., health supplement Type A CTAs — still useful for language)
   - Priority 3: Different vertical + same page type (e.g., finance Type B CTAs — useful for structural patterns)
   - Priority 4: Reference CTA patterns from LP-14-AGENT.md specimen table

3. **From each loaded specimen, extract:**
   - **Button text:** The exact CTA button copy (verbatim)
   - **Context text:** Any text immediately surrounding the CTA button (guarantee reminder, social proof stat, etc.)
   - **Emotional appeal type:** Classify using the 6-type system
   - **Position on page:** Where this CTA appears (above-fold, mid-page, final, etc.)
   - **P.S. copy:** If the specimen page has P.S. lines, extract verbatim
   - **Final close paragraph:** If visible in the specimen, extract verbatim

4. **Load at least 3 specimen CTAs** (more is better). If specimen directory is sparse, supplement with the reference patterns from LP-14-AGENT.md:
   - LMNT: "Add 1 Lemonade Salt to cart"
   - MAMG: "start my free trial"
   - Graziosi: "Join The AI Advantage Club"
   - Closers.io: "Schedule Demo"
   - Hyros: "Get started below"
   - Richmond Dinh: "Join The Free Masterclass Now!"
   - Native: "ADD TO BUNDLE"

5. **Extract CTA structural patterns** from loaded specimens:
   - Word count range for button text
   - Use of possessive "my" vs "your" vs neither
   - Capitalization style (Title Case, ALL CAPS, lowercase, Sentence case)
   - Verb choices (Get, Start, Join, Claim, Try, Add, See)
   - Whether context text is present and what type

6. **Write `specimens-loaded.md`** with all extracted specimens and pattern analysis.

## Quality Criteria
- At least 3 CTA specimens loaded (from any priority level)
- Each specimen has verbatim button text extracted — no paraphrasing
- Emotional appeal type classified for each specimen
- Structural pattern analysis completed — word count, verb choices, possessive style
- P.S. specimens loaded if available (valuable but not blocking)

## Failure Modes
- Fewer than 3 specimens loaded -> Supplement with LP-14-AGENT.md reference patterns. Do not proceed with 0-2 specimens.
- Specimens paraphrased instead of extracted verbatim -> Specimens must be exact. Paraphrased specimens lose the structural signal.
- Emotional appeal classification skipped -> Without classification, the generation step cannot ensure coverage of all 6 types.
- Pattern analysis skipped -> Without patterns, generation defaults to AI-native CTA patterns instead of real-world patterns. This produces "Learn More" and "Get Started" — generic copy.
