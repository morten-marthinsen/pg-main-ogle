# 1.3-feature-naming-consistency-auditor.md

> **Layer:** 1 (Audit)
> **Skill:** PDP-17-ecomm-editorial
> **Purpose:** Audit feature name consistency across entire page against PDP-02 registry
> **Outputs:** Feature consistency audit report with exact-match verification

---

## FUNCTION

Verify that every feature name used anywhere in the assembled copy exactly matches the PDP-02 feature-package.json registry. Detect paraphrasing, shortening, genericization, capitalization drift, and invention of new feature names not in the registry.

This audit runs EARLY in the editorial pipeline because feature name drift compounds — a renamed feature in one section creates inconsistency with every other section that uses the correct name.

---

## AUDIT PROTOCOL

### Step 1: Build Feature Registry
```
LOAD feature-package.json from upstream loader (0.1)
BUILD registry:
  FOR each feature in feature_hierarchy (hero, supporting, technical):
    RECORD: exact_name, feature_id, tier, micro_script
  TOTAL features: [count]
```

### Step 2: Scan All Sections
```
FOR each section in assembled copy:
  SCAN for:
    1. EXACT MATCHES — feature name appears verbatim → PASS
    2. PARTIAL MATCHES — feature name partially appears (e.g., "HyperSpeed" without "Face") → FLAG
    3. PARAPHRASED — feature concept described but name not used → FLAG
    4. GENERICIZED — feature name replaced with generic descriptor → FLAG
    5. CASE DRIFT — correct words but wrong capitalization → FLAG
    6. INVENTED — new feature name not in registry → FLAG
```

### Step 3: Frequency Audit
```
CHECK feature usage frequency:
  Hero features: MUST appear in hero section + at least 2 BTF sections
  Supporting features: MUST appear in their primary section + at least 1 cross-reference
  Technical features: MUST appear in at least 1 relevant section

  IF hero feature missing from hero section: CRITICAL FLAG
  IF hero feature appears fewer than 3 times total: WARNING
  IF supporting feature appears 0 times: CRITICAL FLAG
```

### Step 4: Micro-Script Alignment
```
FOR each section where a feature is described:
  COMPARE description against PDP-02 micro_script:
    ALIGNED: description is consistent with micro-script meaning → PASS
    CONTRADICTED: description says something different from micro-script → FLAG
    EXPANDED: description adds detail beyond micro-script → PASS (acceptable)
```

## SEVERITY LEVELS

| Severity | Condition | Action |
|----------|-----------|--------|
| CRITICAL | Hero feature missing from hero section | MUST fix in revision |
| CRITICAL | Feature name invented (not in registry) | MUST remove or replace |
| HIGH | Feature name paraphrased or genericized | MUST fix in revision |
| MEDIUM | Case drift (capitalization wrong) | Fix in revision |
| LOW | Feature underused (below frequency minimum) | Recommend additional usage |
| INFO | Feature description expanded beyond micro-script | No action needed |
