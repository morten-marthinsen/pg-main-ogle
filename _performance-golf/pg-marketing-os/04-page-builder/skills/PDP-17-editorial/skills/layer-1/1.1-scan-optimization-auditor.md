# 1.1-scan-optimization-auditor.md

> **Layer:** 1 (Audit)
> **Skill:** PDP-17-ecomm-editorial
> **Purpose:** Audit every section for scan optimization (3-5 second test)
> **Outputs:** `scan_audit` object with per-section scores and specific issues

---

## FUNCTION

Audit every section of the assembled e-commerce copy for scan optimization. E-commerce product pages are scanned, not read. Every section must communicate its core message within 3-5 seconds of visual contact. This auditor identifies sections that fail the scan test and provides specific, actionable revision directives.

---

## AUDIT PROTOCOL

### Step 1: Per-Section Scan Test
```
FOR each section in assembled copy:

  PERFORM 3-second scan simulation:
    1. Read ONLY the header and first bolded phrase
    2. CAN you identify what this section is about? [Y/N]
    3. CAN you identify the key benefit or proof? [Y/N]
    4. IS the visual hierarchy clear (what to read first, second, third)? [Y/N]

  IF any answer is N: FLAG section for revision
```

### Step 2: Formatting Audit
```
FOR each section:
  CHECK against scan_patterns from 0.2:
    [ ] Header: benefit-oriented, 3-8 words, not clever
    [ ] Paragraphs: max 3 sentences, preferably 2
    [ ] Bold usage: key phrases bolded, not entire sentences
    [ ] Bullets: used for 3+ parallel items
    [ ] Whitespace: adequate separation between elements
    [ ] Hierarchy: clear visual flow (header -> key phrase -> detail -> proof)
```

### Step 3: Score Each Section
```
SCAN OPTIMIZATION SCORING (per section):
  9-10: Instant scan comprehension; perfect formatting; nothing to change
  7-8: Good scan; 1-2 minor formatting improvements possible
  5-6: Needs work; core message requires reading, not scanning
  3-4: Poor scan; paragraph blocks, no bolding, no hierarchy
  1-2: Not scan-optimized at all; reads like long-form
```

### Step 4: Generate Revision Directives
```
FOR each section scoring < 7.0:
  PRODUCE specific directives:
    issue: [what specifically fails]
    fix: [exact formatting change needed]
    example: [before/after snippet if helpful]

  DIRECTIVE TYPES:
    - "Break paragraph X into 2 shorter paragraphs"
    - "Bold the phrase '[specific phrase]' in sentence Y"
    - "Convert sentences 3-5 into a bullet list"
    - "Shorten header from 12 words to 5-7 words"
    - "Add a bolded lead sentence before the detail"
```

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Average scan score < 5.0 | Major formatting pass needed before editorial |
| WARNING | More than 50% sections flagged | Prioritize highest-priority sections first |
| INFO | All sections >= 7.0 | Light editorial pass, focus on design notes |
