# 1.6-mobile-first-auditor.md

> **Layer:** 1 (Audit)
> **Skill:** PDP-17-ecomm-editorial
> **Purpose:** Audit mobile viewport readability and phone-first design compliance
> **Outputs:** Mobile-first audit report with section-by-section assessment

---

## FUNCTION

Evaluate the assembled e-commerce copy for phone viewport readability. 60-80% of ecom traffic is mobile — if the page doesn't work on a phone, it doesn't work for most visitors. This audit checks line length, touch targets, carousel function, scroll depth economy, text hierarchy, and image-to-copy ratio.

---

## AUDIT PROTOCOL

### Check 1: Phone Viewport Readability
```
FOR each section:
  EVALUATE:
    [ ] No sentences that become 5+ lines on mobile viewport (375-414px)
    [ ] Paragraphs are 2-3 sentences maximum
    [ ] Key information is in the first 2 lines of each paragraph (scanners stop there)
    [ ] Headers are short enough to fit one line on mobile (< 40 characters)
    [ ] Subheads fit two lines maximum on mobile

  GUIDELINES (approximate at standard font sizes):
    - 375px viewport ≈ 35-40 characters per line
    - A 15-word sentence ≈ 2-3 lines on mobile
    - A 25-word sentence ≈ 4-5 lines on mobile (too long)

  IF sentence > 20 words: FLAG for mobile readability
  IF paragraph > 3 sentences: FLAG
  IF header > 40 characters: FLAG
```

### Check 2: Touch Target Adequacy
```
FOR each CTA or interactive element:
  CHECK (via design notes):
    [ ] CTA buttons imply sufficient size (minimum 44x44px equivalent)
    [ ] CTAs are not placed adjacent to other tappable elements (fat finger risk)
    [ ] Interactive elements (carousels, accordions) have clear tap/swipe indicators

  IF design notes don't specify mobile CTA size: FLAG
  IF multiple CTAs clustered: FLAG
```

### Check 3: Carousel Function
```
FOR each carousel section:
  CHECK:
    [ ] Mobile carousel shows enough of next slide to indicate swipability
    [ ] Each carousel slide has standalone copy (no "continued on next slide")
    [ ] Carousel doesn't exceed 7-10 slides on mobile (attention limit)
    [ ] Carousel slide copy fits one mobile screen (no internal scroll)

  IF carousel has > 10 slides: FLAG
  IF slide copy requires scrolling within the slide: FLAG
```

### Check 4: Scroll Depth Economy
```
CHECK page-level mobile scroll depth:
  [ ] Most important sections are in the first 3 scrolls (hero, key benefit, proof)
  [ ] Secondary sections are prioritized correctly for mobile
  [ ] No "dead zones" — sections that add scroll depth without adding conversion value
  [ ] Total mobile scroll depth is reasonable for the product type

  ESTIMATE scroll depth:
    Each section ≈ 1-3 mobile screens
    14 sections × 2 screens average = 28 screens
    Acceptable for PDP: 20-35 screens
    If > 40 screens: FLAG for scroll fatigue

  IF critical content (offer, proof, CTA) is below scroll 10: FLAG
```

### Check 5: Text Size Hierarchy
```
CHECK (implied by copy structure + design notes):
  [ ] Headlines are clearly differentiated from body copy
  [ ] Subheads create visual breaks between sections
  [ ] Micro-copy (captions, disclaimers) is identified as smaller text
  [ ] Bold text is used strategically (not everything bolded)
  [ ] No more than 3 text size levels visible per section

  IF all text appears to be same size (no hierarchy markers in design notes): FLAG
```

### Check 6: Image-to-Copy Ratio
```
FOR sections with images:
  CHECK:
    [ ] On mobile, full-width images don't push copy below the fold of that section
    [ ] Copy is not overwhelmed by image size on mobile
    [ ] If section is image-heavy, the copy is SHORT and impactful
    [ ] If section is copy-heavy, images are supplementary not dominant

  IF image takes > 70% of mobile section and copy is > 100 words: FLAG
  IF copy has no images and is > 200 words: FLAG (wall of text on mobile)
```

## MINIMUM THRESHOLDS

- Phone readability: >= 80% of sections pass
- Touch targets: 100% of CTAs must have adequate spec
- Scroll depth: critical content in top 30% of page
- Any section with 3+ mobile flags: revision mandatory
