# 0.2-specimen-calibrator.md

> **Layer:** 0 (Foundation)
> **Skill:** PDP-17-ecomm-editorial
> **Purpose:** Load scan-optimization reference patterns from specimens
> **Outputs:** `scan_patterns` object with editorial reference patterns

---

## FUNCTION

Load and analyze editorial reference specimens to establish scan-optimization patterns. These specimens demonstrate what excellent e-commerce scan optimization looks like: how top-performing product pages format copy for rapid scanning, bolding patterns, paragraph length, bullet usage, and visual hierarchy.

---

## CALIBRATION PROTOCOL

### Step 1: Load Specimens
```
LOCATE: E-commerce specimens from ~brain/ or project specimen folder
IDENTIFY: Product pages with strong scan optimization
TYPES:
  - High-converting PDP pages
  - Best-in-class DTC product pages
  - NLS-framework exemplars
```

### Step 2: Extract Scan Patterns
```
FOR each specimen:
  ANALYZE:
    header_style: [how headers are written -- length, format, action orientation]
    paragraph_length: [typical sentence count per paragraph]
    bold_pattern: [what types of phrases are bolded]
    bullet_usage: [when bullets are used vs prose]
    visual_hierarchy: [what reads first, second, third]
    proof_formatting: [how proof elements are visually emphasized]
    cta_formatting: [how CTAs are distinguished from surrounding copy]
    whitespace_usage: [spacing between elements]
```

### Step 3: Build Reference Pattern Set
```
SYNTHESIZE across all specimens:
  optimal_header_length: [word range]
  optimal_paragraph_length: [sentence count]
  bold_candidates: [types of phrases that should be bolded]
  bullet_triggers: [when to use bullets vs prose]
  scan_hierarchy_template: [ideal visual flow for a section]
```

---

## OUTPUT STRUCTURE

```json
{
  "scan_patterns": {
    "header_guidelines": {
      "optimal_length": "3-8 words",
      "format": "benefit-oriented, specific, not clever"
    },
    "paragraph_guidelines": {
      "max_sentences": 3,
      "optimal_sentences": 2,
      "max_words_per_paragraph": 45
    },
    "bold_guidelines": {
      "bold_these": ["feature names", "key benefits", "proof numbers", "differentiators"],
      "never_bold": ["entire sentences", "generic phrases", "conjunctions"]
    },
    "bullet_guidelines": {
      "use_when": "3+ parallel items, feature lists, step sequences",
      "avoid_when": "narrative flow needed, emotional build, single points"
    },
    "hierarchy_template": "Header -> Bold lead sentence -> Supporting detail -> Proof",
    "specimens_analyzed": 3,
    "calibration_status": "complete"
  }
}
```
