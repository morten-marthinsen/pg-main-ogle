# 4.3-output-packager.md

> **Layer:** 4 (Scoring & Packaging)
> **Skill:** PDP-17-ecomm-editorial
> **Purpose:** Package ecomm-copy-final.md for page builder handoff
> **Outputs:** `ecomm-copy-final.md` in project output directory

---

## FUNCTION

Package the editorially revised, quality-scored, consistency-verified e-commerce copy into the final output file: `ecomm-copy-final.md`. This file is the deliverable that goes to the page builder (04-page-builder / LP-00). It includes all section copy, design notes, quality scores, and editorial metadata.

---

## PACKAGING PROTOCOL

### Step 1: Verify All Gates Passed
```
CHECK:
  quality_score.overall_status: MUST be PASS
  consistency_result.overall_status: MUST be PASS
  arena_selection: MUST have human selection (if Arena was executed)

IF any fails: HALT -- "Cannot package until all quality gates pass"
```

### Step 2: Build ecomm-copy-final.md
```
ASSEMBLE markdown document:

  # [Product Name] -- E-Commerce Copy (Final)

  _Generated: [timestamp] | Skill: PDP-17 | Quality Score: [overall]/10_
  _Total words: [count] | Sections: [count] | Editorial revisions: [count]_

  ---

  ## Quality Summary
  | Dimension | Score | Status |
  |-----------|-------|--------|
  | Scan Optimization | [X] | PASS |
  | Proof Density | [X] | PASS |
  | Feature Consistency | [X] | PASS |
  | Design Note Quality | [X] | PASS |
  | Word Budget | [X] | PASS |
  | Slop Cleanliness | [X] | PASS |
  | **Overall** | **[X]** | **PASS** |

  ---

  FOR each section (in priority order):
    ## [Section Name]

    [Full final section copy]

    **Design Notes:**
    - **Layout:** [specific layout instruction]
    - **Mobile:** [specific mobile behavior]
    - **Visuals:** [specific visual elements]

    _Word count: [X] | Priority: [N] | Proof: [type]_

    ---

  ## Feature Name Registry
  [Complete feature name list from PDP-02 for page builder reference]

  ## Editorial Changelog
  [Summary of all editorial changes made, by section]
```

### Step 3: Write and Verify
```
WRITE ecomm-copy-final.md to ~outputs/[project-code]/e-comm/
VERIFY:
  [ ] File written successfully
  [ ] All sections present
  [ ] Quality summary included
  [ ] Design notes present for every section
  [ ] Feature registry appended
  [ ] No placeholder text remaining
  [ ] Markdown properly formatted
  [ ] Word counts accurate
```

### Step 4: Update Page Builder Handoff
```
IF page-builder-handoff.yaml exists from LP-15:
  UPDATE with editorial changes:
    - Revised word counts per section
    - Updated design notes
    - Quality scores
    - Editorial status: "FINAL"

WRITE updated page-builder-handoff.yaml
```

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Quality gates not passed | Return to scoring, fix issues |
| CRITICAL | File write fails | Check directory and permissions |
| WARNING | Placeholder text detected | Remove and replace with actual content |
