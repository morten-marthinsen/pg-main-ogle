# 3.1 — Urgency Validator

**Skill:** LP-13-urgency-scarcity
**Position:** Layer 3, Microskill 1
**Purpose:** Score all urgency/scarcity copy against the 8-point urgency audit. Verify that copy meets specificity, justification, voice, and structural requirements. Minimum score of 7.0/10 required to proceed to Layer 4. Also runs the anti-slop scan on all urgency copy elements.
**Effort Level:** medium
**Dependencies:** All Layer 2 output files (countdown-copy.md, stock-level-copy.md, price-deadline-copy.md, availability-copy.md as applicable, urgency-support-copy.md), `justification-plan.md` (1.2), `placement-strategy.md` (1.3)
**Output:** `urgency-audit.md`

---

## Input Schema

```yaml
input:
  all_urgency_copy:
    countdown_copy: "[from 2.1 if applicable]"
    stock_level_copy: "[from 2.2 if applicable]"
    price_deadline_copy: "[from 2.3 if applicable]"
    availability_copy: "[from 2.4 if applicable]"
    urgency_support_copy: "[from 2.5 -- always present]"
  urgency_types_selected: "[from 1.1]"
  justification_plan: "[from 1.2]"
  placement_strategy: "[from 1.3]"
  page_type: "[type_a | type_b | hybrid]"
  voice_direction: "[from packages-loaded.md]"
```

## Execution Logic

### Part 1: 8-Point Urgency Audit

Score each point 0 (fail) or 1 (pass):

**Justification Quality (3 points)**

1. **Every urgency claim has an explicit, documented justification.**
   - Scan all copy elements. List every urgency claim (deadline, stock number, price change, capacity limit, cost-of-delay assertion).
   - For each claim, verify it appears in justification-plan.md with a source.
   - Score: 1 if ALL claims are justified. 0 if ANY claim lacks justification.

2. **Each justification references a real constraint.**
   - For each justification, verify it names a real constraint type: manufacturing, capacity, schedule, cost, progressive condition.
   - "Marketing decision" or "we decided to limit it" is NOT a real constraint.
   - Score: 1 if ALL justifications reference real constraints. 0 if any are circular or vague.

3. **Justification copy would survive a skeptic asking "Why?" three times.**
   - For each primary urgency claim, mentally run the "Why?" x3 test.
   - If the third "Why?" has no substantive answer, score 0.
   - Score: 1 if ALL claims survive the test. 0 if any fail.

**Copy Quality (3 points)**

4. **Urgency copy is specific.**
   - Deadlines have exact dates/times
   - Stock numbers are specific (not "low stock" or "selling fast")
   - Price changes have exact current price, future price, and change date
   - Capacity limits have exact numbers
   - Consequences of delay are concrete (not "you'll miss out")
   - Score: 1 if ALL urgency elements are specific. 0 if any are vague.

5. **Zero AI telltales or generic urgency language.**
   - Scan all copy against the urgency-specific anti-slop word list from LP-13-AGENT.md (5 categories)
   - Check for near-misses: "unlock," "empower," "journey," "seamless," "elevate"
   - Score: 1 if ZERO forbidden words found. 0 if ANY found.

6. **Urgency copy matches the page's voice direction.**
   - Compare urgency copy tone against voice_direction from page-brief.json
   - Urgency should not be significantly more aggressive, casual, or formal than the rest of the page
   - Score: 1 if voice is consistent. 0 if tonal mismatch detected.

**Structural Quality (2 points)**

7. **Maximum 2 urgency types used.**
   - Count urgency types in final copy (not just selected -- verify copy didn't introduce additional types)
   - Score: 1 if count is 1 or 2. 0 if count is 3+.

8. **Urgency appears AFTER value is established.**
   - Verify from placement-strategy.md that urgency section comes after offer/pricing section
   - Exception: Natural urgency callbacks in narrative sections are allowed
   - Score: 1 if value-before-urgency confirmed. 0 if urgency precedes value.

### Part 2: Score Calculation

```
Raw score = points earned / 8
Scaled score = raw score * 10

8/8 = 10.0
7/8 = 8.75
6/8 = 7.5
5/8 = 6.25
4/8 = 5.0
```

**Threshold: >= 7.0 to proceed (minimum 6/8 points)**

### Part 3: Anti-Slop Scan

Scan every copy element against the 5-category forbidden word list:

**Category 1 -- Fabricated Urgency Signals:**
act now, don't miss out, hurry, last chance, limited time offer, order now before it's too late, this won't last, once it's gone it's gone, time is running out

**Category 2 -- Pressure Selling Language:**
you'd be crazy not to, you can't afford to wait, what are you waiting for, don't let this slip away, you'll regret it, this is your only chance, now or never

**Category 3 -- Unverifiable Claims:**
selling fast, almost sold out, thousands are buying right now, everyone is signing up, [number] people are viewing this right now

**Category 4 -- AI Telltales:**
unlock, leverage, game-changing, revolutionary, groundbreaking, transformative, unprecedented, cutting-edge, next-level, don't miss this incredible opportunity

**Category 5 -- Generic Deadline Language:**
for a limited time, while supplies last, offer expires soon, limited availability

For each element scanned:
- List any forbidden words/phrases found
- Provide specific replacement suggestions
- Determine PASS or FAIL

### Part 4: Revision Requirements

If score < 7.0 OR anti-slop scan FAIL:
- List every failing point with specific remediation instructions
- Identify which Layer 2 files need revision
- Specify what changes are required

## Quality Criteria
- All 8 points explicitly scored with evidence (not just Y/N -- show the work)
- Anti-slop scan covers EVERY urgency copy element (not just the main section)
- Score calculation is correct (no rounding errors, no favorable rounding)
- Revision requirements are specific enough for Layer 2 to act on
- Every failing point has a concrete remediation instruction

## Failure Modes
- Scoring a point as PASS when evidence shows otherwise ("Point 5: no telltales" when "act now" appears in P.S.) -> Read every element. One instance anywhere = 0 on Point 5.
- Not scanning P.S. and close integration copy -> These are urgency elements. They must be scanned.
- Favorable scoring ("the voice is close enough") -> Voice match is binary. If urgency copy is noticeably more aggressive than page voice = 0 on Point 6.
- Counting urgency types from Layer 1 selection but not verifying the copy -> The copy may have introduced additional urgency signals not in the Layer 1 selection. Count types from what's actually IN the copy.
- Scoring PASS at 5/8 because "most points passed" -> 5/8 = 6.25. Threshold is 7.0. This is FAIL. No exceptions.
