# 1.3 — Placement Strategy Planner

**Skill:** LP-13-urgency-scarcity
**Position:** Layer 1, Microskill 3
**Purpose:** Map exactly where urgency/scarcity elements appear on the page. Urgency placement is governed by a single rule: value must be established BEFORE urgency is introduced. This microskill maps urgency elements to specific section IDs from LP-04's section-sequence.json, ensuring urgency appears in the right locations and never before the offer.
**Effort Level:** medium
**Dependencies:** `urgency-types-selected.md` (1.1), `justification-plan.md` (1.2), `packages-loaded.md` (0.1 -- section-sequence data)
**Output:** `placement-strategy.md`

---

## Input Schema

```yaml
input:
  selected_types:
    primary: "[type name]"
    secondary: "[type name or null]"
  page_type: "[type_a | type_b | hybrid]"
  section_sequence:
    urgency_section_id: "[from packages-loaded.md]"
    urgency_section_number: "[number]"
    offer_section_number: "[number]"
    close_section_number: "[number]"
    ps_section_number: "[number or null]"
    root_cause_section_number: "[number or null]"
    mechanism_section_number: "[number or null]"
    story_section_number: "[number or null]"
    total_sections: "[count]"
  natural_urgency_selected: "[Y/N -- is Type 6 one of the selected types?]"
```

---

## Execution Logic

1. **Verify the value-before-urgency rule.**
   - Confirm `urgency_section_number` > `offer_section_number`
   - If NOT: Flag for restructuring. Urgency CANNOT precede offer/pricing.
   - Exception: Natural urgency callbacks (Type 6) can appear in root cause, mechanism, and story sections because they describe cost-of-delay, not purchase pressure.

2. **Map Type A placement** (if page_type = type_a or hybrid):

   **Primary urgency section (dedicated section):**
   - Section ID: [urgency_scarcity from section-sequence.json]
   - Section number: [from sequence]
   - Content: Full urgency section (headline + body + justification copy)
   - Word count target: 50-300 words
   - Position: After offer/pricing, before or within close section

   **Natural urgency callbacks (if Type 6 selected):**
   For each callback section, define:
   - Target section ID (from section-sequence.json)
   - Callback purpose (what cost-of-delay angle to weave in)
   - Word count: 1-2 sentences each (20-50 words)
   - Integration approach: Woven into existing narrative, not a separate block

   Typical callback sections:
   - Root cause narrative: "Every day this continues, [specific consequence]"
   - Mechanism section: "The longer you wait, [specific compounding effect]"
   - Story section: "[Character] wishes they had started sooner — [specific cost of delay]"

   **P.S. urgency callback:**
   - Section: P.S. (last section or close section)
   - Content: 1-2 sentences restating the core urgency (deadline, supply, or cost of delay)
   - Word count: 20-50 words

   **Close section urgency line:**
   - Section: Close section
   - Content: 1 sentence urgency restatement integrated into final push
   - Word count: 10-30 words

3. **Map Type B placement** (if page_type = type_b or hybrid):

   **Product card urgency:**
   - Element type: Badge or inline text
   - Position: Below price, above ATC button
   - Content: Stock level indicator OR price deadline badge (3-15 words)
   - Visibility: Always visible in above-fold zone

   **Above ATC urgency:**
   - Element type: Countdown timer context OR availability window
   - Position: Directly above the Add to Cart button
   - Content: Timer label + explanation sentence
   - Visibility: Visible without scrolling on desktop

   **Cart urgency:**
   - Element type: Inline text
   - Position: Cart page or cart drawer
   - Content: 1 sentence (e.g., "Items in your cart are not reserved")
   - Visibility: Visible during checkout flow

   **Banner urgency (optional):**
   - Element type: Top-of-page sticky banner
   - Position: Above navigation
   - Content: Sale end date or launch deadline (1 line)
   - Visibility: Persistent across scroll

4. **Validate placement against section sequence:**
   - Every placement section ID exists in section-sequence.json
   - No urgency elements placed before the offer section (except natural urgency callbacks)
   - Type B elements mapped to specific page zones (above-fold, cart, checkout)

5. **Write `placement-strategy.md`** with complete placement map.

## Quality Criteria
- Every urgency element mapped to a specific section ID or page zone
- Value-before-urgency rule verified (urgency section after offer section)
- Natural urgency callbacks only in appropriate narrative sections (not product/pricing sections)
- Type B elements assigned to specific positions (not vague "somewhere on the page")
- Total urgency touchpoints documented -- gives Layer 3 a check against urgency overload
- Placement validated against section-sequence.json (no references to nonexistent sections)

## Failure Modes
- Placing urgency before the offer section -> Violates value-before-urgency rule. Reposition.
- Mapping urgency elements to section IDs not in section-sequence.json -> Verify every section ID exists. Do not reference sections that are not in the sequence.
- Overloading Type B pages with too many urgency touchpoints -> Product card + above ATC + cart + banner = 4 touchpoints. Verify this doesn't create pressure-selling environment. 2-3 touchpoints recommended for Type B.
- Placing natural urgency callbacks in the pricing/offer section -> Natural urgency callbacks belong in narrative sections (root cause, mechanism, story). In the pricing section, urgency is structural, not narrative.
- Not defining P.S. callback -> The P.S. is one of the highest-read sections on a long-form page. An urgency callback here is critical. Do not skip it.
