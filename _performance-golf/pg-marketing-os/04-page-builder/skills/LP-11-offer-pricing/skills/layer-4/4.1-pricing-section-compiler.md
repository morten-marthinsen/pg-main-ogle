# 4.1 — Pricing Section Package Compiler

**Skill:** LP-11-offer-pricing
**Position:** Layer 4, Microskill 1
**Purpose:** Assemble all validated pricing section copy into a single structured package for downstream skills. LP-15 (Assembly) and LP-14 (CTA Optimizer) will load this file. Every copy element from Layer 2 that passed Layer 3 validation is included here in its final form.
**Effort Level:** low
**Dependencies:** All Layer 2 outputs (validated), `offer-copy-validation.md` (3.1 — PASS), `price-value-audit.md` (3.2 — score ≥7.5)
**Output:** `pricing-section-package.json`

## Execution Logic

1. **Run the assembly gate check.** All gates must pass before assembly begins.

2. **Populate every field in the schema.** Use null for fields that don't apply to the page type (e.g., value_stack = null for Type B, pricing_tiers = null for Type A).

3. **Copy text from Layer 2 outputs VERBATIM.** Do not edit, rewrite, or "improve" any copy during assembly. The copy was validated in Layer 3. Assembly is mechanical.

4. **Calculate key_metrics from the assembled data.** These summary metrics help downstream skills quickly understand the pricing section without reading the full package.

5. **Write downstream_handoffs with specific, actionable notes** for LP-14, LP-15, and LP-17.

---

## Quality Criteria
- All required fields populated (no nulls in required sections for the page type)
- Copy text matches Layer 2 outputs verbatim (no edits during assembly)
- key_metrics calculated correctly from assembled data
- validation block shows all gates PASS
- downstream_handoffs are specific (not generic "pricing section is ready")

## Failure Modes
- Assembly attempted with FAIL in Layer 3 -> Assembly gate catches this. Do not proceed.
- Copy modified during assembly -> Assembly is MECHANICAL. Compile, do not edit. Any modification during assembly voids the Layer 3 validation.
- Key metrics miscalculated -> Verify: total_perceived_value = sum of component values. value_to_price_ratio = total_perceived_value / actual_price. savings_claims_count = count of callouts.
