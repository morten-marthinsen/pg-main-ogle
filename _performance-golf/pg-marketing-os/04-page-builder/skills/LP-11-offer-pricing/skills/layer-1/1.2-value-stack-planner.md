# 1.2 — Value Stack Planner

**Skill:** LP-11-offer-pricing
**Position:** Layer 1, Microskill 2
**Purpose:** Map every value stack component to a specific copy approach, set perceived value targets per component, and plan the overall value-to-price gap strategy. This is the bridge between LP-06's structural architecture and LP-11's copy generation — it translates "what goes in the stack" into "how each component will be described."
**Effort Level:** medium
**Dependencies:** `offer-architecture-loaded.md` (0.2), `specimens-loaded.md` (0.3), `pricing-display-confirmed.md` (1.1)
**Output:** `value-stack-plan.md`

---

## Execution Logic

1. **List all value stack components** from offer-architecture-loaded.md in their LP-06-designated order.

2. **For each component, determine the copy approach:**

   | Component Type | Copy Approach |
   |---------------|--------------|
   | Core product | Lead with the primary transformation it delivers. Frame as the centerpiece. |
   | Module/lesson | Lead with the specific problem it solves. Frame as a discrete solution. |
   | Tool/template | Lead with time-saved or complexity-removed. Frame as a shortcut. |
   | Access/community | Lead with the value of connection and accountability. Frame as ongoing support. |
   | Bonus | Lead with the benefit that complements the core product. Frame as enhancement. |

3. **Evaluate LP-06's perceived value assignments.** For each component:
   - Is the value believable? (Would a skeptical reader accept it?)
   - Is the value justified by a comparable market price?
   - Is the value proportional to the component's substance?

   **Believability scale:**
   - Digital download (PDF, checklist): $7-97 believable range
   - Video training module: $47-497 believable range
   - Software/tool access: $27-197/month believable range
   - Community/membership access: $47-297/month believable range
   - 1-on-1 consultation/coaching: $197-997/session believable range
   - Physical product: market price ± 30% is believable

   If LP-06's value exceeds the believable range, flag it and suggest a revised value.

4. **Set the total perceived value target.** The total should be:
   - 3x-5x actual price: standard (safe, credible)
   - 5x-8x actual price: aggressive (needs strong component justification)
   - 8x-10x actual price: maximum (only for offers with genuinely high-value components)
   - >10x actual price: DANGER ZONE — triggers skepticism. Reduce or remove components.

5. **Plan the copy length per component:**
   - Primary/core component: 3-5 sentences
   - Secondary modules: 2-4 sentences each
   - Tools/templates: 2-3 sentences each
   - Bonuses: 1-3 sentences each (shorter — bonuses are gifts, not main events)

6. **For Type B (condensed value stack):** Plan 1-2 sentence product descriptions per tier instead of full component breakdowns. The "value stack" for Type B is the product description and benefit list within each pricing tier.

7. **Identify the "hero component"** — the single highest-value item that will receive the longest description and serve as the value stack anchor. This is typically the core product or the highest-priced module.

8. **Write `value-stack-plan.md`.**

## Quality Criteria
- Every LP-06 component mapped to a copy approach — no components silently dropped
- Value believability evaluated per component (not just the total)
- Total perceived value ratio in 3x-10x range (or flagged if outside)
- Copy length proportional to component importance (hero component longest)
- Each component has a specific benefit to lead with (not "describe the component")
- Type B has condensed plan if applicable

## Failure Modes
- LP-06 value stack empty for Type A -> HALT. Type A requires a value stack. Escalate to LP-06.
- Total perceived value <3x actual price -> Flag. Either add components (LP-06 decision) or increase individual values (within believable range).
- Multiple components with flagged believability -> Reduce values. A stack where half the values are unbelievable contaminates the entire stack's credibility.
- Hero component not identifiable -> Default to the core product. If no single component stands out, the value stack may need restructuring (LP-06 concern).
