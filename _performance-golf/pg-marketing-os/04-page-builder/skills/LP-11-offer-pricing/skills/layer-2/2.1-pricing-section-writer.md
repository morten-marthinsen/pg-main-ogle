# 2.1 — Pricing Section Writer

**Skill:** LP-11-offer-pricing
**Position:** Layer 2, Microskill 1
**Purpose:** Write the core pricing copy — either the price reveal sequence (Type A) or the pricing tier copy (Type B). This is the copy that puts numbers in front of the reader. Every number must be accurate, every comparison must be justified, and the price must be framed to feel like an investment, not a cost.
**Effort Level:** max
**Dependencies:** `pricing-display-confirmed.md` (1.1), `value-stack-plan.md` (1.2), `offer-architecture-loaded.md` (0.2), `specimens-loaded.md` (0.3)
**Output:** `pricing-copy.md`

## Execution Logic

### TYPE A: Price Reveal Mode

1. **Write the value sum transition.** This sentence bridges from the value stack (2.2) and "if all it did was" (2.5) to the price reveal. It establishes the total perceived value one final time.

   **Pattern options:**
   - "Everything you see above has a combined value of $[total perceived value]."
   - "Add it all up, and you're looking at $[total perceived value] worth of [result-oriented description]."
   - "That's $[total perceived value] in [tools/training/support] — all designed to [primary outcome]."

   **Rule:** Use the total perceived value from value-stack-plan.md. This number must match the sum of all component values.

2. **Write the "not-prices."** These are the comparison anchors from LP-06 that the reader sees BEFORE the real price. They establish "what this COULD cost" to make the actual price feel like a deal.

   **First not-price:** The highest anchor — what a comparable solution actually costs.
   ```
   If you hired a [comparable professional] to [deliver this outcome],
   you'd invest $[first not-price] — minimum.
   ```

   **Second not-price:** A more moderate but still-high number — closer to reasonable.
   ```
   Even a [less expensive alternative] would cost you $[second not-price].
   ```

   **Rules for not-prices:**
   - Must be real, verifiable comparison points (named competitor, named professional, known market rate)
   - Never fabricate comparison prices — if you can't name the source, don't use the anchor
   - First not-price should be 5-20x actual price
   - Second not-price should be 2-5x actual price
   - If LP-06 provided specific anchors, USE THEM

3. **Write the price reveal.** This is a single sentence. It carries weight. Do not dilute it.

   **Pattern:**
   ```
   Your [investment/total] today is $[actual price].
   ```

   **Rules:**
   - Use "investment" or "total" — not "cost" or "fee"
   - State the number cleanly — no exclamation marks, no "just," no "only"
   - Let the number speak for itself after the value stack has done its work
   - The price sentence should be on its own line, with white space above and below

4. **Write the per-day minimizer** (if applicable from 1.1).
   ```
   That's less than $[per-day amount]/day [over timeframe].
   ```
   Or:
   ```
   Less than the price of [relatable daily expense].
   ```

   **Per-day calculation must be mathematically exact.** Calculate: actual_price / days_in_timeframe. Round to the nearest cent.

5. **Write the guarantee reference.** One to two sentences maximum. Use the branded guarantee name from LP-06.
   ```
   And it's backed by our [Branded Guarantee Name] —
   [1-sentence terms summary]. [Duration].
   ```

### TYPE B: Pricing Tier Mode

1. **Write copy for each pricing tier** from offer-architecture-loaded.md.

   **Per-tier copy structure:**
   ```
   [TIER LABEL — "Best Value" / "Most Popular" / "Starter"]
   [Quantity] [Product Name]
   $[per-unit price]/[unit] — [savings vs single unit]
   [Crossed-out original total] → $[actual total]
   ```

2. **Write subscription vs. one-time toggle copy** (if subscription exists).
   ```
   One-Time Purchase: $[one-time price]
   Subscribe & Save [XX]%: $[subscription price]/[period]
   [Delivery frequency copy]
   [Cancel policy copy — e.g., "Skip, pause, or cancel anytime"]
   ```

   **Subscription savings percentage must be calculated from one-time price:**
   savings_pct = ((one_time_price - subscription_price) / one_time_price) * 100

3. **Write tier recommendation copy** — the brief text that steers the reader toward the recommended tier.
   ```
   [Optional: "Our customers' #1 choice" or "Best value per [unit]"]
   ```

4. **Write delivery/shipping copy** (if applicable).
   ```
   [Free shipping threshold or flat rate]
   [Estimated delivery timeframe]
   ```

### HYBRID: Both Modes

For Hybrid pages, run Type B for the above-fold pricing module and Type A for the below-fold value section. Write both sets of copy, clearly labeled by zone.

## Quality Criteria
- Type A price reveal sequence follows the correct order: value sum -> not-prices -> actual price -> minimizer -> guarantee
- Every not-price has a named, verifiable source
- Per-day calculation is mathematically correct to the penny
- Type B tier copy is scannable in under 5 seconds per tier
- Subscription savings percentage matches the actual math
- Guarantee reference is 1-2 sentences (not the full guarantee block)
- Zero AI telltales
- "Just" or "only" appears 0-1 times in the entire pricing copy (not hedging)

## Failure Modes
- Price revealed before value establishment (Type A) -> This microskill assumes 2.2 and 2.5 have already run. If they haven't, HALT and re-check execution order.
- Not-prices without verifiable sources -> Remove the anchor. An unverifiable comparison destroys more trust than it builds. Use cost-of-problem or cost-of-inaction anchors instead.
- Per-day calculation off by more than $0.01 -> Recalculate. Precision matters — readers do the math.
- Type B tiers require more than 5 seconds to parse -> Simplify. Remove unnecessary detail. Use the BIOptimizers pattern: Label / Quantity / Per-unit / Total / Savings — nothing more.
