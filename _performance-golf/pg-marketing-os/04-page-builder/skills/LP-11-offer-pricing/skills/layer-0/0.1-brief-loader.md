# 0.1 — Brief Loader

**Skill:** LP-11-offer-pricing
**Position:** Layer 0, Microskill 1
**Purpose:** Load and verify the four upstream package files that LP-11 depends on: `page-brief.json` (LP-00), `offer-cta-architecture.json` (LP-06), `section-sequence.json` (LP-04), and `hero-section-package.json` (LP-07). Extract the key fields that govern all downstream generation decisions.
**Effort Level:** low
**Dependencies:** LP-00 output (`page-brief.json`), LP-06 output (`offer-cta-architecture.json`), LP-04 output (`section-sequence.json`), LP-07 output (`hero-section-package.json`)
**Output:** `packages-loaded.md`

---

## Input Schema

```yaml
input:
  project_directory: "[path to active project output directory]"
  expected_files:
    - "page-brief.json"
    - "offer-cta-architecture.json"
    - "section-sequence.json"
    - "hero-section-package.json"
```

---

## Execution Logic

1. **Locate each upstream file** in the project output directory. Check for existence of all four files.

2. **If `page-brief.json` or `offer-cta-architecture.json` is MISSING:** HALT immediately. These are hard dependencies. LP-11 cannot proceed without both. Report which file(s) are absent.

3. **If `section-sequence.json` is MISSING:** Flag as WARNING. LP-11 can proceed but may not know the surrounding section context. Mark as `not_loaded`.

4. **If `hero-section-package.json` is MISSING:** Flag as WARNING. LP-11 can proceed but will lack the promise statement and threading anchor for consistency checking. Mark as `not_loaded`.

5. **Load `page-brief.json`** and extract:
   - `page_type` (type_a | type_b | hybrid)
   - `vertical` (health, finance, info_product, personal_dev, technology, etc.)
   - `product_name`
   - `price_point` (base price, bundle prices if multiple tiers)
   - `audience_profile` (primary pain, primary desire, demographics)
   - `voice_direction` (tone, register, anti-voice)
   - `offer_summary` (what's included, bonuses, guarantee mention)
   - `copywriting_engine_mode` (downstream | standalone)

6. **Load `offer-cta-architecture.json`** and extract:
   - `offer_type` (Hard, Soft, Free+S&H, Limited, Limited-Time, OTO, Combination)
   - `pricing_display_strategy` (value-first | price-visible)
   - `value_stack_order` (ordered list of components with architect-assigned values)
   - `price_anchor_strategy` (comparison anchors, per-day minimizer, total-value-sum)
   - `bonus_list` (bonus items with architect-assigned values and descriptions)
   - `guarantee_reference` (branded name, duration, terms)
   - `urgency_scarcity_approach` (for awareness in pricing — LP-13 handles copy)
   - `cta_emotional_appeals` (for consistency — LP-14 handles copy)

7. **Load `section-sequence.json`** (if available) and extract:
   - `pricing_section_position` (which section number in the sequence)
   - `section_before_pricing` (what content precedes the pricing section)
   - `section_after_pricing` (what follows — typically guarantee or CTA)

8. **Load `hero-section-package.json`** (if available) and extract:
   - `promise_statement` (the core promise from the hero section)
   - `primary_hook_phrase` (threading anchor)
   - `audience_qualifier` (how the hero identifies the target reader)

9. **Cross-validate page type** across loaded files. If `page-brief.json` says Type A but `offer-cta-architecture.json` uses price-visible strategy, flag the conflict.

10. **Write `packages-loaded.md`** with all extracted fields organized by source file.

## Quality Criteria
- Both critical files loaded and confirmed present (page-brief + offer-cta-architecture)
- Key fields extracted with no gaps (missing fields marked `not_available`, not silently omitted)
- Page type cross-validated across loaded packages
- Pricing display strategy extracted and confirmed consistent with page type
- Output is self-contained — any downstream microskill can read this file without loading the original JSONs

## Failure Modes
- `page-brief.json` missing -> HALT with specific message. Do not proceed with offer architecture alone.
- `offer-cta-architecture.json` missing -> HALT. LP-11 requires LP-06 architecture. Running without it produces unanchored pricing copy.
- Page type conflict between files -> Flag the conflict and note which file's classification governs (page-brief.json takes precedence).
- Offer summary empty in page-brief.json -> Extract as `not_available`. LP-06 architecture may supply sufficient detail. If both are empty -> HALT.
