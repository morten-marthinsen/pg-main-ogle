# 1.2 — Pricing Display Planner

**Skill:** LP-06-offer-cta-architecture
**Position:** Layer 1, Microskill 2
**Purpose:** Declare the pricing display strategy based on page type classification. Type A = price-after-value (the value stack creates the anchor frame BEFORE price reveal). Type B = price-above-fold (strikethrough anchor + savings display immediately visible). Hybrid = both. This decision is structural and governs the entire offer presentation downstream.
**Effort Level:** medium
**Dependencies:** `brief-loaded.md` (0.1 — page type), `offer-type-classification.md` (1.1), `offer-data-loaded.md` (0.2)
**Output:** `pricing-display-strategy.md`

---

## Input Schema

```yaml
input:
  page_type: "[type_a | type_b | hybrid]"
  offer_type: "[from 1.1 classification]"
  pricing_structure: "[from 0.2 — all price points, bundles, subscription]"
  awareness_stage: "[1-5 from brief]"
  traffic_temperature: "[cold | warm | hot]"
```

---

## Pricing Strategy Decision Tree

```
IF page_type = type_a:
  -> PRICE-AFTER-VALUE strategy
  -> Price NEVER above fold
  -> Value stack (Section 7 equivalent) MUST precede pricing block (Section 8 equivalent)
  -> Anchor = total value sum + comparison anchor + per-day minimizer

IF page_type = type_b:
  -> PRICE-ABOVE-FOLD strategy
  -> Price IS above fold (within first screen)
  -> Anchor = strikethrough original price + savings percentage
  -> Bundle tiers visible near or above fold
  -> Subscribe-and-save toggle if subscription available

IF page_type = hybrid:
  -> HYBRID-BOTH strategy
  -> Type B price above fold (captures impulse/ready buyers)
  -> Type A value stack below fold (educates hesitant buyers)
  -> Second pricing CTA after value stack section
```

---

## Execution Logic

**Step 1:** Confirm page type from brief. This is the primary decision driver.

**Step 2:** Apply the decision tree above. Select the strategy.

**Step 3:** For the selected strategy, document:
- Where price first appears (section number or zone)
- What anchoring method(s) will be used
- Whether bundle tiers exist and how they will be displayed
- Whether subscription vs. one-time toggle is needed
- How savings are communicated (percentage, dollar amount, or both)

**Step 4:** Cross-check against awareness stage:
- Stage 1-2 (Unaware/Problem-Aware): Even for Type B, consider delaying prominent price display until after mechanism education. Flag for potential hybrid treatment.
- Stage 4-5 (Product-Aware/Most-Aware): Price can appear earlier even in Type A. Note potential for above-fold price display.

**Step 5:** Document the rationale connecting page type + awareness stage + offer type to the chosen strategy.

---

## Quality Criteria
- Strategy matches page type (Type A = price-after-value, Type B = price-above-fold)
- All price points documented
- Anchoring method specified (not "anchor the price")
- Bundle tier display direction provided if tiers exist
- Awareness stage cross-check performed and documented

## Failure Modes
- Page type not confirmed -> HALT. Cannot select strategy without page type.
- Pricing data all `needs_validation` -> Proceed with strategy selection (strategy is structural, not price-dependent). Flag price points as "placeholder — update when confirmed."
- Type A page but client insists on price above fold -> Document the conflict. Recommend hybrid approach. Note risk: "Price above fold without value stack reduces anchoring effectiveness for cold/warm traffic."
