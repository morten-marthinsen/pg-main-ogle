# 0.2 — Offer Data Loader

**Skill:** LP-06-offer-cta-architecture
**Position:** Layer 0, Microskill 2
**Purpose:** Load complete offer details from CopywritingEngine Skill 07 output (Downstream Mode) or extract from the page brief offer summary (Standalone Mode). This is the authoritative source for pricing, bonuses, guarantee, and urgency levers that LP-06 designs around.
**Effort Level:** low
**Dependencies:** `brief-loaded.md` (0.1), CopywritingEngine `offer-package.json` (optional)
**Output:** `offer-data-loaded.md`

---

## Input Schema

```yaml
input:
  operating_mode: "[downstream | standalone — from 0.1]"
  offer_package_path: "[path to offer-package.json if downstream mode]"
  offer_summary_from_brief: "[offer_summary section from page-brief.json]"
```

---

## Execution Logic

### Downstream Mode

**Step 1:** Load `offer-package.json` from CopywritingEngine Skill 07 output. Expected location:
```
./outputs/[product-name]/07-offer/outputs/offer-package.json
```

**Step 2:** Extract and document:
- Product name and category
- Full pricing structure (single, multi-pack, subscription, trial)
- Bonus stack (names, descriptions, assigned values)
- Guarantee details (duration, type, terms)
- Urgency levers (any legitimately available)
- Upsell/downsell structure (if present — from CopywritingEngine Upsell Engine)

**Step 3:** Cross-reference with page-brief.json offer summary. Note any discrepancies. CopywritingEngine package takes precedence for conflicts.

### Standalone Mode

**Step 1:** Extract offer details from `page-brief.json` `offer_summary` section.

**Step 2:** For each required field:
- If populated: record as-is
- If `not_available` or `not_specified`: flag as `needs_validation` and note the gap

**Step 3:** Document which offer elements are confirmed vs. inferred vs. missing.

```markdown
## Product

- Category: [category]
- Primary Benefit: [core promise]
- Mechanism Name: [from CopywritingEngine if available, else: not_available]

## Pricing Structure
- Single Unit: $[price] | [needs_validation]
- Multi-Pack Available: [Y/N]
  - Options: [list with prices]

- Duration: [days] | [needs_validation]

## Urgency Levers Available
- Stock Scarcity: [Y/N + justification] | [needs_validation]
- Price Deadline: [Y/N + details] | [needs_validation]

- [or: "No legitimate urgency levers identified"]

## Upsell/Downsell Structure (if present)
[from Upsell Engine if available, else: "not_applicable"]

- [list each needs_validation field with what information is needed]

```

---

## Quality Criteria
- All offer fields documented — populated, `needs_validation`, or `not_applicable`
- Source clearly identified (CopywritingEngine package vs. brief)
- Confidence level assessed
- Gaps explicitly listed for human review
- In Downstream Mode: CopywritingEngine package fully extracted

## Failure Modes
- Downstream Mode but offer-package.json corrupted or malformed -> Fall back to brief offer summary. Note discrepancy in output.
- Standalone Mode with entirely empty offer summary -> Flag as CRITICAL OFFER GAP. LP-06 can design CTA architecture and guarantee framing but pricing display requires offer details. Request from user.
- Pricing fields all missing -> LP-06 cannot design price anchor strategy. Flag and proceed with CTA + guarantee architecture only. Note incomplete output.
