# 4.2 — Summary Writer

**Skill:** LP-06-offer-cta-architecture
**Position:** Layer 4, Microskill 2
**Purpose:** Write the human-readable `OFFER-CTA-ARCHITECTURE-SUMMARY.md` — a clear, scannable overview of the complete offer/CTA architecture for human review. This file must stand alone as a comprehensible document. the human reviewer should be able to read this file and understand every offer, CTA, guarantee, and urgency decision without opening the JSON.
**Effort Level:** low
**Dependencies:** `offer-cta-architecture.json` (4.1)
**Output:** `OFFER-CTA-ARCHITECTURE-SUMMARY.md`

---

## Input Schema

```yaml
input:
  offer_cta_json: "[from 4.1 — the complete JSON output]"
  project_name: "[product name]"
```

---

## Execution Logic

**Step 1:** Read `offer-cta-architecture.json` from 4.1.

**Step 2:** Write the summary in the required format (see below). Every section must be populated from the JSON. No section may be omitted.

**Step 3:** Ensure the summary is scannable:
- Use tables for CTA maps
- Use bullet points for specifications
- Bold key decisions
- Include the one-offer audit result prominently

---

## Quality Criteria
- Every section populated from the JSON
- One-offer audit result prominently displayed
- CTA map presented as a scannable table
- Value stack math visible and verifiable
- Guarantee name is clearly branded (human can verify at a glance)
- Downstream handoffs explicitly described

## Failure Modes
- JSON not available -> HALT. Cannot write summary without the compiled JSON.
- Section missing from JSON -> Flag the gap in the summary: "WARNING: [section] not available in compiled JSON. Review 4.1 compilation."
