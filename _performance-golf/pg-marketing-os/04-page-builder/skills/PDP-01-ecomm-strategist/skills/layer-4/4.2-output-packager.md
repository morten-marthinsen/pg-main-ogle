# 4.2-output-packager.md

> **Layer:** 4 (Validation & Packaging)
> **Skill:** PDP-01-ecomm-strategist
> **Purpose:** Package ecomm-strategy.yaml
> **Outputs:** `ecomm-strategy.yaml` file in project output directory

---

## FUNCTION

Package the validated strategy into the canonical output file `ecomm-strategy.yaml`. This file is the single source of truth that all downstream E-Commerce Copy Engine skills consume.

---

## PACKAGING PROTOCOL

### Step 1: Verify Validation Passed

```
CHECK validation_report.overall_status:
  IF PASS: proceed to packaging
  IF FAIL: HALT -- "Cannot package strategy that failed validation"
```

### Step 2: Assemble YAML Structure

```
BUILD ecomm-strategy.yaml:

  header:
    version: "1.0"
    generated_at: [ISO timestamp]
    skill_id: "PDP-01-ecomm-strategist"
    source_mode: [campaign_brief|standalone_brief]

  product_context:
    product_name: [from upstream]
    product_category: [from upstream]
    page_type: [from classifier]
    page_type_rationale: [from classifier]

  section_map:
    active_sections: [from assembler, ordered by priority]
    excluded_sections: [from section mapper, with rationale]

  feature_research_scope:
    hero_candidates: [from assembler]
    supporting_candidates: [from assembler]
    technical_features: [from assembler]
    research_gaps: [from assembler]

  crossover_map:
    [from crossover identifier]

  word_budget_summary:
    [from assembler]

  downstream_handoffs:
    [from assembler]

  validation:
    status: PASS
    validated_at: [ISO timestamp]
```

### Step 3: Write File

```
WRITE ecomm-strategy.yaml to:
  ~outputs/[project-code]/e-comm/ecomm-strategy.yaml

VERIFY:
  [ ] File written successfully
  [ ] File is valid YAML (parseable)
  [ ] All required sections present
  [ ] No placeholder text ("[TODO]", etc.)
  [ ] No copy content in strategy (zero prose)
  [ ] Timestamps are valid ISO format
```

### Step 4: Create Packaging Verification

```
BUILD packaging_verification:
  file_written: true
  file_path: "~outputs/[project-code]/e-comm/ecomm-strategy.yaml"
  file_size: [size in KB]
  schema_version: "1.0"
  validation_status: "PASS"
  sections_count: [number]
  word_budget_total: [number]
  packaged_at: [ISO timestamp]
```

---

## OUTPUT STRUCTURE

The output IS the `ecomm-strategy.yaml` file itself, plus a verification object:

```json
{
  "packaging_result": {
    "status": "complete",
    "file_written": "ecomm-strategy.yaml",
    "file_path": "~outputs/[project-code]/e-comm/ecomm-strategy.yaml",
    "schema_version": "1.0",
    "validation_passed": true,
    "active_sections": 14,
    "total_word_budget": 3200,
    "packaged_at": "ISO timestamp"
  }
}
```

---

## ANTI-PATTERNS (FORBIDDEN)

1. **Packaging unvalidated strategy** -- validation must PASS first
2. **Including copy in YAML** -- strategy is architecture only
3. **Placeholder values** -- no "[TODO]" or "[INSERT]" entries
4. **Incomplete downstream handoffs** -- every downstream skill must have data
5. **Wrong output path** -- must be ~outputs/[project-code]/e-comm/

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Validation not passed | Return to 4.1, fix failures |
| CRITICAL | File write fails | Check directory permissions and path |
| WARNING | File size < 1KB | Strategy may be too thin -- review content |
| WARNING | File size > 20KB | Strategy may contain copy -- review for prose contamination |
