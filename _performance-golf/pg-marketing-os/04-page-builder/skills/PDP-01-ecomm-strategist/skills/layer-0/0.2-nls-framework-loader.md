# 0.2-nls-framework-loader.md

> **Layer:** 0 (Foundation)
> **Skill:** PDP-01-ecomm-strategist
> **Purpose:** Load ~brain/nls-pdp-best-practices.md (19 section types)
> **Outputs:** `nls_framework` object with all 19 section definitions

---

## FUNCTION

Load the NLS PDP Best Practices framework document that defines the 19 canonical section types for product detail pages. This framework is the architectural backbone of the E-Commerce Copy Engine -- all section mapping references these definitions.

---

## LOADING PROTOCOL

### Step 1: Locate Framework File

```
LOCATE: ~brain/nls-pdp-best-practices.md
  IF file exists:
    LOAD full document
    PARSE section definitions
  IF file not found:
    FAIL -- "NLS PDP framework not found at ~brain/nls-pdp-best-practices.md. Cannot proceed with section mapping."
```

### Step 2: Parse 19 Section Types

```
FOR each section (1-19):
  EXTRACT:
    section_number: [1-19]
    section_name: [canonical name]
    copy_type: [what kind of copy this section contains]
    proof_requirement: [what proof element is expected]
    word_budget_range: [typical min-max words]
    mobile_behavior: [how section behaves on mobile]
    design_notes: [framework-level design guidance]
    page_type_applicability: [PDP|collection|bundle or all]
```

### Step 3: Build Section Reference Index

```
BUILD nls_framework:
  total_sections: 19
  sections: [array of 19 parsed section objects]
  page_type_filters:
    pdp_applicable: [list of section numbers]
    collection_applicable: [list of section numbers]
    bundle_applicable: [list of section numbers]
  required_sections: [sections that should always be active]
  optional_sections: [sections that depend on product/data availability]
```

### Step 4: Validate Framework Integrity

```
VERIFY:
  [ ] All 19 sections parsed successfully
  [ ] Each section has name, copy_type, proof_requirement
  [ ] Page type filters populated
  [ ] No duplicate section numbers

IF any verification fails:
  WARN -- "Framework parsing incomplete. Missing sections: [list]"
  LOG degraded fields for downstream awareness
```

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | ~brain/nls-pdp-best-practices.md not found | HALT -- file must exist for section mapping |
| CRITICAL | File exists but cannot be parsed | HALT -- check file format, may be corrupted |
| DEGRADED | Some sections missing definitions | WARN -- proceed with available sections, flag gaps |
| DEGRADED | Page type filters incomplete | WARN -- default all sections to all page types |
