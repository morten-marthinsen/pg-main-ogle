# 1.3-crossover-identifier.md

> **Layer:** 1 (Classification)
> **Skill:** PDP-01-ecomm-strategist
> **Purpose:** Identify long-form skill crossovers (Skills 13, 14, 15, 16, 18)
> **Outputs:** `crossover_map` object linking active sections to long-form patterns

---

## FUNCTION

Map each active section to the long-form VSL skill that contains the most applicable copywriting patterns. This crossover mapping enables PDP-07 (Section Copy) to load proven long-form patterns and adapt them for ecom scan-optimized format.

## MAPPING PROTOCOL

### Step 1: Load Active Sections

```
FROM section_map LOAD:
  active_sections[] with section_numbers

FOR each active_section:
  LOOKUP crossover_reference_table
  IF match found:
    ASSIGN crossover_skill with specific pattern type
  IF no match:
    MARK as "ecom-native" (no long-form crossover applicable)
```

### Step 2: Identify Pattern Types per Crossover

```
FOR each crossover_skill assignment:
  SPECIFY which patterns from the long-form skill apply:

  Skill 13 (Root Cause) patterns for ecom:
    - Enemy identification (condensed to 2-3 sentences)
    - Pain agitation (scan-optimized bullet format)
    - Root cause reframe (single paragraph, not narrative)

  Skill 14 (Mechanism) patterns for ecom:
    - Named mechanism introduction (feature name + 1-2 sentence explanation)
    - How-it-works sequence (3-5 step visual format)
    - Science/technology backing (single proof point, not extended argument)

  Skill 15 (Product Intro) patterns for ecom:
    - Feature naming conventions (caveman-benefit format)
    - Feature hierarchy (hero / supporting / technical)
    - Feature-benefit linkage (ingredient -> what it does)

  Skill 16 (Offer) patterns for ecom:
    - Value stack construction (simplified for ecom pricing section)
    - Price anchoring (condensed to comparison format)
    - Guarantee framing (single clear statement)

  Skill 18 (Proof) patterns for ecom:
    - Review integration (pull-quote format)
    - UGC referencing (image + caption format)
    - Statistical proof (single stat with context)
    - Authority citation (credential + endorsement)
```

### Step 3: Assess Crossover Strength

```
FOR each mapping:
  RATE crossover_strength:
    STRONG: Long-form pattern directly applicable with format adaptation
    MODERATE: Long-form pattern partially applicable, needs significant condensing
    WEAK: Long-form pattern concept useful but execution very different

  IF crossover_strength == WEAK:
    NOTE: "PDP-07 should reference pattern concept but write ecom-native"
```
