# 1.1-page-type-classifier.md

> **Layer:** 1 (Classification)
> **Skill:** PDP-01-ecomm-strategist
> **Purpose:** Determine PDP / collection / bundle page type
> **Outputs:** `page_type_classification` object

---

## FUNCTION

Classify the product into one of three page types based on product structure, SKU count, and purchase intent. This classification determines which NLS sections are applicable and how the section map is structured.

---

## PAGE TYPE DEFINITIONS

| Page Type | Characteristics | Section Impact |
|-----------|----------------|----------------|
| **PDP** (Product Detail Page) | Single product, single SKU (or variant SKUs), deep feature story | All 19 sections potentially applicable |
| **Collection** | Category page, multiple products, browsing intent | Subset of sections (hero, filters, category proof, FAQ) |
| **Bundle** | Multi-product package, value-stack positioning, comparison between included items | Bundle-specific sections (individual item features, bundle value calc, savings) |

---

## CLASSIFICATION PROTOCOL

### Step 1: Product Structure Analysis

```
FROM upstream_context EXTRACT:
  product_count: How many distinct products?
  sku_structure: Single SKU, variant SKUs, or multi-product?
  purchase_model: Individual purchase, subscription, or bundle?

CLASSIFY:
  IF product_count == 1 AND (single_sku OR variant_skus):
    candidate_type = "pdp"
    confidence = HIGH

  IF product_count > 1 AND same_category:
    candidate_type = "collection"
    confidence = HIGH

  IF product_count > 1 AND sold_together:
    candidate_type = "bundle"
    confidence = HIGH

  IF ambiguous:
    candidate_type = "requires_human_input"
    confidence = LOW
    PRESENT options with rationale
```

### Step 2: Purchase Intent Assessment

```
ASSESS primary purchase intent:
  direct_purchase: Customer knows what they want -> PDP
  browsing_comparison: Customer exploring category -> Collection
  value_seeking: Customer evaluating package deals -> Bundle

IF intent conflicts with structure:
  FLAG for human review
  EXAMPLE: Single product sold only in bundles -> classify as Bundle PDP hybrid
```

### Step 3: Confidence Scoring

```
CALCULATE classification_confidence:
  product_structure_clarity: 40%
  purchase_intent_clarity: 30%
  category_precedent: 20%
  data_completeness: 10%

  IF confidence >= 0.85: PROCEED without checkpoint
  IF confidence 0.65-0.84: RECOMMEND with flag
  IF confidence < 0.65: HALT -- human classification required
```

## OUTPUT STRUCTURE

```json
{
  "page_type_classification": {
    "page_type": "pdp|collection|bundle",
    "confidence": 0.92,
    "rationale": "Single product with 3 size variants, deep feature set, direct purchase intent",
    "section_filter_applied": "pdp_applicable",
    "human_override_needed": false,
    "alternative_considered": {
      "type": "bundle",
      "why_rejected": "Product variants are sizes, not distinct products"
    }
  }
}
```

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| AMBIGUOUS | Product structure unclear | CHECKPOINT -- present options to human |
| CRITICAL | No product data to classify | HALT -- return to Layer 0 |
| EDGE CASE | Hybrid page type needed | Document hybrid rationale, apply primary type's section filter |
