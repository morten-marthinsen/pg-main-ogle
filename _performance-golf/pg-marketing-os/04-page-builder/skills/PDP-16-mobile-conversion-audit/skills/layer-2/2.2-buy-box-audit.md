## Execution Logic

**POSITIONAL REINFORCEMENT:** You are auditing the buy box / purchase interface. This is the most critical section of the entire PDP audit. Every criterion here has a specific Baymard-documented conversion impact. Do NOT rush through this section.

**Step 1:** From `assembled-page-load.md`, extract all buy box specifications:
- Product title display rules
- Rating strip specification
- Price display (original, savings, shipping)
- Variant selector type and design
- Subscription vs OTP design
- Quantity selector type
- ATC button specification
- Post-click behavior
- Express payment layout
- Micro-trust signals

**Step 2:** Evaluate each criterion:

### Criterion 2.1: Full Product Title
- **Baymard finding:** Users confirm correct variation via title. Truncation creates anxiety.
- **PASS:** Title specified as full display, never truncated (3 lines OK)
- **FAIL:** Title truncated, or truncation behavior not specified

### Criterion 2.2: Rating Strip Anchor Link
- **Baymard finding:** Users see rating and immediately want to verify why not 5 stars
- **PASS:** Stars + "(X Reviews)" specified as anchor link that smooth-scrolls to reviews section
- **FAIL:** Rating not linked to reviews, or link behavior not specified

### Criterion 2.3: Price with Savings
- **Baymard finding:** Users don't do mental math on mobile
- **PASS:** Original price struck through + savings displayed as BOTH percentage AND dollar amount
- **FAIL:** Only one format, no savings display, or not specified

### Criterion 2.4: Shipping Proximity
- **Baymard finding:** Users combine price + shipping to determine value
- **PASS:** Shipping cost or "Free Shipping" visible near price or ATC
- **FAIL:** Shipping information not near price/ATC, or not specified

### Criterion 2.5: Variant Selector = Chips
- **Baymard stat:** Dropdowns require 3+ taps and hide options -> 60% more friction
- **PASS:** Exposed chip buttons specified (rounded rect, pill, or similar)
- **FAIL:** Dropdown specified, or selector type not specified
- **N/A:** Product has no variants (single SKU)
- **AUTOMATIC FAIL if dropdown is specified — this is non-negotiable per Baymard**

### Criterion 2.6: Subscription = Tiles
- **Baymard finding:** Sneaky subscriptions are the top cause of cart abandonment and chargebacks
- **PASS:** Selectable tiles for OTP vs S&S, explicit savings shown, "Cancel anytime" visible
- **FAIL:** Pre-selected subscription, no savings display, or not specified
- **N/A:** Product has no subscription option

### Criterion 2.7: Quantity = Stepper
- **Baymard stat:** 90% of users want quantity "1"; dropdowns are high interaction cost
- **PASS:** Stepper (+/-) UI specified, 44px+ touch targets, default = 1
- **FAIL:** Dropdown quantity selector, or not specified

### Criterion 2.8: ATC Full-Width
- **Baymard finding:** Standard "Add to Cart" label — not creative alternatives
- **PASS:** Full-width button, standard "Add to Cart" label, state change on tap
- **FAIL:** Not full-width, creative label (e.g., "Get Yours Now"), or not specified

### Criterion 2.9: Post-Click Mini-Cart
- **Baymard finding:** Cart redirect interrupts shopping flow
- **PASS:** Slide-out mini-cart specified (user stays on PDP)
- **FAIL:** Cart page redirect specified, or post-click behavior not specified
- **AUTOMATIC FAIL if cart redirect is specified**

### Criterion 2.10: Micro-Trust Signals
- **Baymard finding:** Users have last-second anxieties right before tapping ATC
- **PASS:** 3-4 signals specified in small gray text below ATC (security, shipping, returns, guarantee)
- **FAIL:** Fewer than 3 signals, signals not below ATC, or not specified

**Step 3:** Write finding for each criterion with verdict, evidence, Baymard reference, and fix recommendation.

**Step 4:** Write `buy-box-audit.md`.

## Quality Criteria

- All 10 criteria evaluated (or marked N/A with justification)
- Each verdict has evidence from assembled page
- Dropdown variant selector = automatic FAIL regardless of other factors
- Cart redirect = automatic FAIL regardless of other factors
- Pre-selected subscription = automatic FAIL
- UNSPECIFIED = FAIL
- Fix recommendations are specific
- Section score calculated correctly
- File exists after execution

## Failure Modes

- Assembled page has no buy box specification -> Score all 10 criteria as FAIL. This indicates LP-15 did not properly assemble PDP-03/PDP-06 output.
- Buy box score below 6/10 -> This is a red flag. Note in audit that buy box needs structural revision before LP-18.
- Criteria 2.5/2.6/2.7 all fail (dropdown for variant, pre-selected sub, dropdown quantity) -> Note as "Triple UX Violation" — these three together indicate the assembled page ignored Baymard best practices entirely.
