# 2.6 — Navigation Audit

> **Skill:** PDP-16-mobile-conversion-audit
> **Layer:** 2 (Audit Execution)
> **Position:** 6 of 8
> **Model:** sonnet
> **Purpose:** Audit navigation and sticky elements against Baymard best practices. The most critical element here is the sticky ATC bar: for pages 4-10x viewport length (standard for mobile PDPs), a sticky ATC is mandatory. Without it, users who scroll past the buy box have no purchase path without scrolling back up 10+ screens.
> **Dependencies:** `assembled-page-load.md`, `pdp-reference-load.md`, `pdp-audit-criteria.md`
> **Output:** `navigation-audit.md`

## Execution Logic

**POSITIONAL REINFORCEMENT:** You are auditing navigation and sticky elements — sticky ATC bar, header behavior, breadcrumbs, and back-button handling. You are NOT auditing the buy box ATC button itself (2.2 covers that) or the gallery overlay back-button (2.1 covers that). Focus on page-level navigation and persistent purchase access.

**Step 1:** From `assembled-page-load.md`, extract navigation specifications:
- Sticky ATC bar specification (presence, trigger, content)
- Header behavior on scroll (collapse, fixed, hidden)
- Breadcrumb specification
- Back-button behavior (page-level, not overlay-level)
- Promo bar (bonus, not scored)

**Step 2:** Evaluate each criterion:

### Criterion 6.1: Sticky ATC Trigger
- **Baymard stat (6.8):** Mobile pages are 4-10x longer than desktop. Don't make users scroll back up 10 screens to buy.
- **PASS:** Sticky footer bar specified that appears when main ATC button scrolls out of view. Bar contains: product title (truncated OK here) + price + "Add to Cart" button.
- **FAIL:** No sticky ATC specified, sticky ATC always visible (not triggered), or not specified.
- **Key detail:** The sticky ATC must be TRIGGERED (appears only when main ATC leaves viewport), not always-present. An always-present sticky bar wastes screen real estate when the main ATC is already visible.
- **Evidence required:** Quote sticky ATC specification or note absence.

### Criterion 6.2: Header Behavior
- **Mobile UX principle:** Fixed full-height headers consume precious viewport space on mobile.
- **PASS:** Header collapses on scroll (slim mode), auto-hides on scroll down / appears on scroll up, or uses minimal height. Header does not obscure content or compete with sticky ATC.
- **FAIL:** Header remains full-height fixed on scroll, header overlaps sticky ATC bar, or not specified.
- **Evidence required:** Note header behavior specification.

### Criterion 6.3: Breadcrumbs Present
- **UX principle:** Users need orientation — especially those arriving from search or category pages.
- **PASS:** Breadcrumbs specified above product title (e.g., Home > Category > Product).
- **FAIL:** No breadcrumbs specified, or breadcrumbs placed in non-standard location.
- **Evidence required:** Note breadcrumb specification or absence.

### Criterion 6.4: Back-Button Works Correctly
- **UX principle:** Users must never be "trapped" on a page. Every use of browser back should do something expected.
- **PASS:** Back-button behavior addressed at page level — no states where back button leaves the site unexpectedly. Overlay states (gallery, mini-cart) properly intercept back button.
- **FAIL:** Back-button behavior not addressed, or back button documented as problematic in any page state.
- **Evidence required:** Note back-button handling or absence of specification.

**Step 3:** Write finding for each criterion.

**Step 4:** Write `navigation-audit.md`.

## Quality Criteria

- All 4 criteria evaluated
- Sticky ATC evaluated for TRIGGER behavior (not just presence)
- Header evaluated for SCROLL behavior (not just presence)
- UNSPECIFIED = FAIL
- Missing sticky ATC flagged as critical issue
- File exists after execution

## Failure Modes

- No navigation specifications in assembled page -> Score all 4 as FAIL. Note that sticky ATC absence is a critical conversion gap.
- Sticky ATC specified as always-visible -> Evaluate as PARTIAL PASS if it contains right elements, but note inefficiency (wastes viewport when main ATC visible). For scoring, treat as PASS if elements are correct.
- Header and sticky ATC conflict -> FAIL on header behavior (6.2). Note the conflict.
