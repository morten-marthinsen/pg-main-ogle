# 3.2 — Composite Score Calculator

> **Skill:** PDP-16-mobile-conversion-audit
> **Layer:** 3 (Scoring + Prioritization)
> **Position:** 2 of 3
> **Model:** sonnet
> **Purpose:** Calculate the weighted composite score from all 8 section scores using the weights from Layer 1. The composite score is the single number that determines whether the PDP proceeds to LP-18 (A/B Test Plan) or gets flagged for revision.
> **Dependencies:** `per-section-scores.md`, `weight-assignment.md`
> **Output:** `composite-score.md`

---

## Input Schema

```yaml
input:
  per_section_scores: "per-section-scores.md"
  weight_assignment: "weight-assignment.md"
```

---

## Execution Logic

**Step 1:** Read `per-section-scores.md` to get all 8 section scores.

**Step 2:** Read `weight-assignment.md` to get the weight for each section.

**Step 3:** Verify weights sum to 100%. If they do not, HALT and fix weight-assignment.md.

**Step 4:** Calculate weighted composite score:

```
Composite = SUM(section_score_i * weight_i) for i = 1 to 8
```

**Step 5:** Show the math explicitly:

```
Composite = (S1 * W1) + (S2 * W2) + (S3 * W3) + (S4 * W4) + (S5 * W5) + (S6 * W6) + (S7 * W7) + (S8 * W8)
```

Where S = section score (0-10) and W = weight (decimal, e.g., 0.25 for 25%).

**Step 6:** Classify the composite score:

| Range | Classification | Action |
|-------|---------------|--------|
| 9.0 - 10.0 | Exceptional | Proceed to LP-18 |
| 7.0 - 8.9 | Good | Proceed to LP-18 with improvement notes |
| 5.0 - 6.9 | Needs Revision | FLAG — upstream skill rework recommended before LP-18 |
| Below 5.0 | Structural Issues | HALT — major rework required |

**Step 7:** Identify the top 3 contributing factors to the score:
- Which sections pulled the composite UP the most (high score * high weight)?
- Which sections pulled the composite DOWN the most (low score * high weight)?

**Step 8:** Write `composite-score.md`.

## Quality Criteria

- All 8 sections included in calculation
- Weights sum to exactly 100%
- Math shown explicitly (each section_score * weight)
- Composite rounded to 2 decimal places
- Classification matches threshold table
- Impact analysis identifies real contributors (not generic)
- Score sensitivity calculated for top improvement opportunities
- File exists after execution

## Failure Modes

- Weights don't sum to 100% -> HALT. Fix weight-assignment.md before calculating.
- Missing section score -> HALT. All 8 scores must exist.
- Arithmetic error -> Show all intermediate calculations. Verify sum manually.
- Composite classified incorrectly -> Double-check against threshold table.
