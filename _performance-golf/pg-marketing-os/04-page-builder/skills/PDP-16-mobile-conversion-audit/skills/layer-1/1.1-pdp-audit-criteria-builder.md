# 1.1 — PDP Audit Criteria Builder

> **Skill:** PDP-16-mobile-conversion-audit
> **Layer:** 1 (Audit Scope)
> **Position:** 1 of 2
> **Model:** sonnet
> **Purpose:** Build the PDP-specific audit checklist from the 8 Baymard sections. Each section gets enumerated criteria with specific, binary pass/fail definitions. No vague criteria — every criterion must be testable against the assembled page content.
> **Dependencies:** `pdp-reference-load.md`, `assembled-page-load.md`
> **Output:** `pdp-audit-criteria.md`

---

## Input Schema

```yaml
input:
  pdp_reference_load: "pdp-reference-load.md"
  assembled_page_load: "assembled-page-load.md"
```

---

## Execution Logic

**Step 1:** Read `pdp-reference-load.md` to extract all Baymard findings organized by section.

**Step 2:** Read `assembled-page-load.md` to understand what components are present in the assembled page. This determines whether certain criteria are applicable (e.g., if no subscription option exists, subscription criteria are N/A, not FAIL).

**Step 3:** For each of the 8 Baymard sections, build an enumerated criteria list. Each criterion must have:
- **ID:** Section number + criterion number (e.g., 1.1, 1.2, 2.1, 2.2, ...)
- **Criterion name:** Short, descriptive name
- **Pass condition:** Specific, testable condition that must be true in the assembled page
- **Fail condition:** What constitutes a failure (including UNSPECIFIED)
- **Baymard reference:** The specific finding/statistic this criterion is based on
- **Applicability:** Whether this criterion applies to this specific PDP (based on assembled-page-load.md)

**Step 4:** Build the 8 section criteria lists:

### Section 1: Image Gallery / Carousel (6 criteria)
- 1.1: Thumbnail navigation (not dots)
- 1.2: Pinch-to-zoom support specified
- 1.3: In-scale image included
- 1.4: Gallery overlay behavior specified
- 1.5: Back-button trap prevention specified
- 1.6: Video integration in carousel

### Section 2: Buy Box / Purchase Interface (10 criteria)
- 2.1: Full product title (not truncated)
- 2.2: Rating strip as anchor link to reviews
- 2.3: Price with savings in both % and $
- 2.4: Shipping cost/status visible near price
- 2.5: Variant selector uses chips (not dropdown)
- 2.6: Subscription uses tiles (not pre-selected)
- 2.7: Quantity uses stepper (not dropdown)
- 2.8: ATC button full-width with standard label
- 2.9: Post-click behavior is mini-cart (not redirect)
- 2.10: Micro-trust signals (3-4) below ATC

### Section 3: Product Information (4 criteria)
- 3.1: Title not truncated
- 3.2: Short description with specific KSPs (not generic)
- 3.3: Facts panel present (if applicable product type)
- 3.4: Ingredient naming uses "X for Y" format

### Section 4: BTF Section Quality (4 criteria)
- 4.1: Each section scannable at a glance
- 4.2: Module independence (each section works standalone)
- 4.3: Proof density meets PDP-04 spec
- 4.4: No dead weight sections (every section has conversion purpose)

### Section 5: Social Proof System (6 criteria)
- 5.1: Review histogram present and tappable
- 5.2: Keyword filter chips (5-8) above reviews
- 5.3: UGC photos/videos above text reviews
- 5.4: Load More button (not pagination)
- 5.5: Verified buyer badges on review cards
- 5.6: Review card template complete (stars + headline + attributes + body)

### Section 6: Navigation / Sticky Elements (4 criteria)
- 6.1: Sticky ATC triggers when main ATC leaves viewport
- 6.2: Header does not obscure content on scroll
- 6.3: Breadcrumbs present above title
- 6.4: Back-button works correctly (no traps)

### Section 7: Mobile Performance (4 criteria)
- 7.1: All interactive elements >= 44x44px touch targets
- 7.2: Font sizes readable (body >= 14px, headlines >= 20px)
- 7.3: Images specify responsive sizing / lazy loading for BTF
- 7.4: No horizontal scroll at 375px width

### Section 8: Conversion Flow (4 criteria)
- 8.1: ATC -> mini-cart -> checkout path specified
- 8.2: Express payments below ATC with OR divider
- 8.3: Error state handling specified (OOS variant, quantity limits)
- 8.4: No dead ends (every state has a clear next action)

**Step 5:** Mark N/A criteria. Based on `assembled-page-load.md`:
- If no subscription option: 2.6 = N/A
- If no variants: 2.5 = N/A
- If not a supplement/food product: 3.3 = N/A, 3.4 = N/A
- N/A criteria do not count toward section score (reduce denominator)

**Step 6:** Write `pdp-audit-criteria.md`.

## Quality Criteria

- All 8 sections have enumerated criteria
- Total criteria = 42 (6 + 10 + 4 + 4 + 6 + 4 + 4 + 4)
- Each criterion has a specific, testable pass condition (not vague)
- Each criterion references the Baymard finding it is based on
- N/A exclusions are justified with evidence from assembled-page-load.md
- Applicable criteria count is accurate (42 minus N/A count)
- File exists after execution

## Failure Modes

- pdp-reference-load.md not found -> HALT, run 0.2 first
- assembled-page-load.md not found -> HALT, run 0.1 first
- Criteria are vague (e.g., "carousel is good") -> Rewrite with specific testable conditions
- Baymard references missing -> Every criterion must trace to a specific finding. Add references.
- N/A criteria counted in denominator -> Remove N/A from applicable count. They must not inflate or deflate scores.
