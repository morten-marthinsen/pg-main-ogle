# 1.2 — Weight Assignment

> **Skill:** PDP-16-mobile-conversion-audit
> **Layer:** 1 (Audit Scope)
> **Position:** 2 of 2
> **Model:** sonnet
> **Purpose:** Assign conversion impact weights to each of the 8 audit sections based on Baymard conversion data. Weights determine how much each section contributes to the composite score. The buy box is weighted highest because 80% of the PDP purchase decision happens there.
> **Dependencies:** `pdp-audit-criteria.md`, `benchmark-load.md`
> **Output:** `weight-assignment.md`

---

## Input Schema

```yaml
input:
  pdp_audit_criteria: "pdp-audit-criteria.md"
  benchmark_load: "benchmark-load.md"
```

---

## Execution Logic

**Step 1:** Read `pdp-audit-criteria.md` to understand the applicable criteria count per section.

**Step 2:** Read `benchmark-load.md` for any vertical-specific conversion data that might adjust weights.

**Step 3:** Assign default weights based on Baymard conversion research:

| Section | Default Weight | Rationale |
|---------|---------------|-----------|
| 1. Image Gallery / Carousel | 12% | 50-80% of users miss content with dots. Carousel is primary PDP persuasion tool. Thumbnails vs dots is a documented conversion driver. |
| 2. Buy Box / Purchase Interface | 25% | 80% of purchase decision happens at buy box. Most criteria (10), highest documented friction points (dropdowns = 60% more friction, sneaky subscriptions = top abandonment cause). |
| 3. Product Information | 10% | Supporting role to buy box. Title truncation causes anxiety, short description aids quick assessment. |
| 4. BTF Section Quality | 10% | Hunt-and-peck reassurance content. Important for considered purchases but secondary to above-fold conversion. |
| 5. Social Proof System | 15% | Reviews are the #2 conversion driver after buy box. Histogram, filters, and UGC directly impact purchase confidence. |
| 6. Navigation / Sticky Elements | 10% | Sticky ATC is mandatory for long mobile pages (4-10x viewport). Without it, users who scroll past buy box have no purchase path. |
| 7. Mobile Performance | 10% | Touch targets and font sizes prevent basic usability failures. Foundation-level — if these fail, nothing else matters. |
| 8. Conversion Flow | 8% | Cart path and error handling. Critical but often platform-handled (Shopify, etc.). |

**Step 4:** Validate weights sum to exactly 100%.
- 12 + 25 + 10 + 10 + 15 + 10 + 10 + 8 = 100

**Step 5:** Check for vertical-specific adjustments:
- **Supplement PDPs:** Increase Section 5 (Social Proof) by +3% and decrease Section 8 (Conversion Flow) by -3%. Supplements are high-skepticism purchases where reviews are disproportionately important.
- **Beauty/Skincare PDPs:** Increase Section 1 (Image Gallery) by +3% and decrease Section 4 (BTF Sections) by -3%. Visual product assessment is critical.
- **General/Default:** Use default weights as-is.

**Step 6:** Document the final weights with justification and write `weight-assignment.md`.

## Quality Criteria

- All 8 sections have weights assigned
- Weights sum to exactly 100%
- Each weight has a rationale connected to Baymard research
- Vertical-specific adjustments documented (or noted as default)
- Scoring formula documented
- File exists after execution

## Failure Modes

- Weights do not sum to 100% -> Recalculate. This is non-negotiable.
- Buy Box weight below 20% -> This is almost certainly wrong. Buy box is 80% of the PDP decision. Minimum weight = 20%.
- All sections weighted equally (12.5% each) -> This ignores Baymard's documented conversion impact hierarchy. Use the default distribution or justify deviations.
- pdp-audit-criteria.md not found -> HALT, run 1.1 first.
