# 1.2 — Device Context Planner

> **Skill:** LP-16-mobile-audit
> **Layer:** 1 (Audit Planning)
> **Model:** sonnet
> **Input:** `brief-load.md`
> **Output:** `device-context.md`

---

## PURPOSE

Determine the primary device context for the page being audited: is the majority of traffic expected to arrive on mobile or desktop? This determination affects how aggressively mobile issues should be weighted in the final score and informs which viewport widths to prioritize in the audit.

This is NOT about choosing mobile vs. desktop — mobile is ALWAYS the primary audit lens (LP-16 is a mobile-first audit). This is about calibrating the expected traffic split so fixes can be prioritized by actual impact.

---

## PROCEDURE

1. **Read** `brief-load.md` for traffic source, vertical, and platform data.
2. **Determine** primary device context using the decision matrix below.
3. **Output** the determination with reasoning to `device-context.md`.

### Conflict Resolution

When signals conflict (e.g., B2B vertical but Facebook traffic source):
1. **Traffic source takes priority** over vertical — the ad platform determines the device, not the product category.
2. If traffic source is unknown, **vertical takes priority**.
3. If both are unknown, **default to mobile-first** (conservative assumption, since underestimating mobile = higher risk than overestimating).

### Primary Viewport Widths by Context

| Context | Primary Viewports | Why |
|---------|------------------|-----|
| Mobile-First | 375px (iPhone SE), 390px (iPhone 14/15), 414px (iPhone Plus) | Smallest common screens first |
| Mixed | 390px (iPhone 14/15), 414px (iPhone Plus), 768px (iPad) | Mid-range mobile + tablet |
| Desktop-Leaning | 390px (mobile check), 1024px (small laptop), 1440px (desktop) | Still check mobile, but desktop more critical |

---

## GATE CHECK

**This microskill is complete when:**
- `device-context.md` exists
- Primary device context determined (one of three categories)
- Reasoning traces back to specific signals from brief-load.md
- Priority viewports listed (minimum 3)
- Expected traffic split estimated

**Important:** Even in DESKTOP_LEANING context, mobile is still audited. The determination only affects scoring weight, not audit scope. LP-16 always evaluates all 7 mobile dimensions regardless of device context.
