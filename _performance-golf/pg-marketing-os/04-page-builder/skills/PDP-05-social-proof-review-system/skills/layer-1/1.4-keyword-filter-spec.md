# 1.4 — Keyword Filter Spec

**Skill:** PDP-05-social-proof-review-system
**Position:** Layer 1, Microskill 4
**Purpose:** Design the keyword filter chip system — 5-8 product-specific filter chips that let users find reviews about topics they care about. Chips are seeded from research data (0.2) and finalized in Layer 2 (2.2 Keyword Topic Selector). This microskill defines the interaction pattern, visual spec, and structural rules.
**Effort Level:** medium
**Dependencies:** `input-verification.md` (0.1), `research-load.md` (0.2), `pdp-reference-load.md` (0.3), `review-system-architecture.md` (1.1)
**Output:** `keyword-filter-spec.md`

## Execution Logic

**Step 1:** Check scope. If `scope = minimal` or `scope = generation_only`, SKIP keyword filters entirely. Document: "Keyword filters omitted per scope determination. Scope = [scope]." Output a stub file and exit.

**Step 2:** Determine chip count based on scope:
- `scope = full` -> 5-8 chips (target 7-8)
- `scope = standard` -> 5-6 chips

**Step 3:** Seed initial keyword candidates from research data (0.2):
- Extract top mentioned topics from keyword frequency data
- Extract top concern categories from verbatim quotes
- Extract product attribute categories (e.g., "taste," "texture," "effectiveness," "shipping," "value")
- If research data sparse: use product category defaults (Step 4)

**Step 4:** Product Category Default Keywords (fallback when research data insufficient):

| Category | Default Keywords |
|----------|-----------------|
| Supplement | Taste, Effectiveness, Side Effects, Value, Shipping, Ingredients |
| Skincare | Texture, Results, Sensitivity, Packaging, Scent, Value |
| Electronics | Battery Life, Build Quality, Sound Quality, Ease of Use, Value, Customer Service |
| Apparel | Fit, Material Quality, Sizing, Comfort, Durability, Value |
| Food/Beverage | Taste, Freshness, Packaging, Value, Portion Size, Ingredients |
| Fitness Equipment | Build Quality, Ease of Assembly, Durability, Value, Size/Space, Noise Level |
| Pet Products | Pet Acceptance, Effectiveness, Ingredients, Value, Durability, Smell |

These are STARTING POINTS. Layer 2 (2.2 Keyword Topic Selector) will refine and finalize based on deeper research analysis.

**Step 5:** Define chip selection criteria (for Layer 2 to use):
- RELEVANCE: keyword must appear in 5%+ of reviews (or research quotes) to warrant a chip
- SPECIFICITY: keywords must be product-specific, NOT generic praise words
- COVERAGE: chips should collectively cover 60%+ of review content themes
- BALANCE: mix of positive topics (effectiveness, taste) AND concern topics (side effects, shipping)
- NO OVERLAP: chips should represent distinct topics (don't have both "Quality" and "Build Quality")

FORBIDDEN chip labels:
- "Quality" (too vague — specify WHAT quality: build quality, material quality, sound quality)
- "Great" (praise, not a topic)
- "Love it" (sentiment, not a topic)
- "Amazing" (sentiment, not a topic)
- "Product" (meaningless filter)
- "Review" (meta-label, not a topic)
- Any single-word adjective that describes sentiment rather than a product attribute

**Step 6:** Design chip visual specification:
- Shape: rounded rectangle (pill shape)
- Size: auto-width based on text length + padding
- Padding: 8-12px horizontal, 6-8px vertical
- Font: body font, slightly smaller than review text (13-14px)
- Spacing: 8px gap between chips
- Wrapping: chips wrap to second row if needed (max 2 rows visible, rest hidden behind "More" expand)

**Step 7:** Design chip interaction states:
- **Inactive state:** outline border, no fill, muted text color
- **Active state:** filled background (brand accent color or dark gray), white/light text, checkmark icon preceding label
- **Hover state (desktop):** subtle background tint, cursor pointer
- **Toggle behavior:** tap to activate, tap again to deactivate
- **Multi-select:** YES — users can activate multiple chips simultaneously
- **Combined filtering:** active chips filter reviews to show ONLY reviews mentioning ALL active keywords (AND logic, not OR)
- **Combined with histogram:** if a star filter is active AND a keyword chip is active, show reviews matching BOTH (4-star reviews mentioning "battery life")
- **Clear all:** if 2+ chips active, show "Clear Filters" link to reset all

**Step 8:** Design chip count indicators:
- Each chip optionally shows count of matching reviews: e.g., "Battery Life (247)"
- Count updates dynamically when histogram filter is active
- If count = 0 for a chip: chip remains visible but grayed out (not hidden)
- Count display is OPTIONAL (some implementations omit for cleaner look — note both options)

**Step 9:** Design overflow behavior:
- If more than 6 chips: display first row (4-6 chips), remaining behind "Show More" or "+N" indicator
- "Show More" tapping reveals all chips in second row
- On mobile: horizontal scroll (single row) with partial chip peek as scroll indicator
- ALTERNATIVE: two visible rows on desktop, single scrollable row on mobile

**Step 10:** Document the complete keyword filter specification.

## Quality Criteria
- 5-8 keyword candidates identified (research-sourced or category defaults)
- All keywords are product-specific attributes (not generic praise)
- Chip interaction states fully specified (inactive, active, hover, disabled)
- Multi-select behavior documented with AND filter logic
- Combined filtering with histogram specified
- Overflow behavior documented for both desktop and mobile
- FORBIDDEN labels listed and enforced
- Clear Filters mechanism documented

## Failure Modes
- Generic chip labels like "Quality" or "Great" -> HALT. Replace with product-specific attributes.
- Fewer than 5 chips for full/standard scope -> HALT. Add category default keywords to reach minimum.
- Chips not specified as toggleable -> HALT. Toggle interaction is mandatory.
- OR filter logic instead of AND -> HALT. Combined filtering must use AND logic.
- Chips hidden when 0 matches -> HALT. Show grayed/disabled state, never hide.
- Scope = minimal but filters included -> HALT. Remove filters per scope rules.
