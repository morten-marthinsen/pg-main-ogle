# 2.4 — Q&A Section Architecture

**Skill:** PDP-05-social-proof-review-system
**Position:** Layer 2, Microskill 4
**Purpose:** Design the search-first Q&A section with brand vs community answer distinction, "Ask a Question" de-emphasis, zero-question handling (hide section entirely), and answer formatting rules. The Q&A section is the final trust layer — it resolves purchase-blocking doubts that reviews alone cannot address.
**Effort Level:** high
**Dependencies:** `input-verification.md` (0.1), `research-load.md` (0.2), `pdp-reference-load.md` (0.3), `review-system-architecture.md` (1.1)
**Output:** `qa-section-architecture.md`

---

## Execution Logic

**Step 1:** Check scope and existing Q&A count:
- `scope = minimal`: SKIP Q&A section entirely. Document: "Q&A section omitted per minimal scope." Output stub and exit.
- `existing_qa_count = 0` AND no research-based questions identifiable: specify HIDE behavior (section does not render). Document the hide rule and exit.
- All other scopes: proceed with architecture.

**Step 2:** Determine Q&A content source:
- `existing_qa_count >= 10`: use existing Q&A pairs (curate best, format to template)
- `existing_qa_count = 1-9`: use existing + supplement with generated Q&A pairs
- `existing_qa_count = 0` AND scope = `full | standard | generation_only`: generate 8-15 Q&A pairs from research data
- Generated Q&A MUST be grounded in actual purchase-blocking concerns from research (0.2)

**Step 3:** Design search-first pattern (MANDATORY — Baymard requirement):
- Search field at TOP of Q&A section (above all question listings)
- Placeholder text: "Search questions about [Product Name]..." or "Have a question? Search here..."
- Search behavior: live filtering as user types (filter existing Q&A by keyword match)
- Zero-result behavior: "No matching questions found. Ask your question below." with expandable "Ask a Question" form
- Search field styling: full-width input, subtle border, search icon left, clear-X right when text entered
- Mobile: search field remains full-width, 44px minimum height

**Step 4:** Design question listing layout:
- Display format: expandable accordion (question visible, answer hidden until tapped)
- Question text: bold, 16-18px, full product question (not truncated)
- Expand indicator: chevron or plus icon (right-aligned), rotates/transforms on expand
- Answer reveal: smooth expand animation (200-300ms), no page jump
- Question count: display total at top: "X Questions & Answers" or "X Questions about [Product Name]"
- Sort: most helpful first (by upvote count), with option to sort by newest

**Step 5:** Design answer types and badges:

| Answer Type | Badge | Visual | Priority |
|------------|-------|--------|----------|
| Brand Answer | "[Brand Name] Official" | Brand color highlight, brand icon/logo | Always shown first if present |
| Community Answer | "Customer Answer" | Neutral styling, user icon | Shown below brand answer |
| Expert Answer | "Expert" or "[Title]" | Subtle accent, credential display | Shown between brand and community |

Rules:
- If both brand and community answers exist for a question: show BOTH, brand first
- Brand answers MUST be clearly visually distinct from community answers (different background color or left border)
- Community answers show the answerer's name + "Verified Buyer" badge if applicable
- Multiple community answers: show up to 3, with "See X more answers" expand

**Step 6:** Design brand answer template:
```
BRAND ANSWER FORMAT:
- Badge: "[Brand Name] Official" (brand accent color)
- Content: 2-5 sentences, direct answer to the question
- Tone: helpful, specific, NOT corporate/evasive
- MUST include: direct answer to the question asked (not a redirect to FAQ page)
- MAY include: link to relevant resource IF the answer also addresses the question directly
- Timestamp: "Answered [X days/weeks/months] ago"
- FORBIDDEN: "Please contact customer service" as sole answer (must ALSO answer the question)
- FORBIDDEN: Generic non-answers ("We appreciate your interest in our product")
- FORBIDDEN: Marketing language in answers (keep factual and helpful)
```

**Step 7:** Design community answer template:
```
COMMUNITY ANSWER FORMAT:
- Answerer: "[Name]" + "Verified Buyer" badge (if applicable)
- Content: 1-4 sentences, personal experience
- Tone: authentic customer voice (not corporate)
- Helpful votes: "X people found this helpful"
- Timestamp: "Answered [X days/weeks/months] ago"
```

**Step 8:** Generate Q&A pairs (if needed). For each generated Q&A:

**8a. Question generation rules:**
- Questions MUST be grounded in research data (0.2) — real concerns people have
- Questions should be specific, not vague ("Does this help with joint pain in knees?" not "Does it work?")
- Mix of question types:
  - Product efficacy questions (40%): "How long before I see results?"
  - Usage/compatibility questions (25%): "Can I take this with blood pressure medication?"
  - Shipping/logistics questions (15%): "How long does shipping take?" "Is there a subscription option?"
  - Comparison questions (10%): "How does this compare to [Competitor]?"
  - Ingredient/material questions (10%): "Where are the ingredients sourced?"

**8b. Answer generation rules:**
- Brand answers: factual, specific, include numbers when possible ("Most customers see initial results within 2-3 weeks")
- Community answers: use reviewer persona voices (from 2.3 if available), personal experience, natural language
- FORBIDDEN: all Q&A pairs answered only by brand (include community answers for at least 30% of questions)
- FORBIDDEN: AI-telltale language in any answer (same forbidden list as 2.1)

**Step 9:** Design "Ask a Question" mechanism:
- Position: below search field (secondary) OR below question listings (tertiary)
- Format: small text link ("Ask a Question"), NOT a prominent button
- On tap: expands inline form with text area + submit button
- Form fields: question text (required), name (optional), email (optional — for notification of answer)
- Submit behavior: "Your question has been submitted. You'll be notified when it's answered."
- FORBIDDEN: "Ask a Question" as prominent CTA competing with purchase flow

**Step 10:** Design zero-question state:
- If Q&A section has 0 questions AND no generated Q&A: HIDE the entire section
- Do NOT show: empty Q&A section with "No questions yet — be the first to ask!"
- Do NOT show: Q&A header with empty content area
- Rationale: empty Q&A signals low product interest. Better to omit than display emptiness.

**Step 11:** Design pagination/loading:
- Default visible: 5-8 questions (collapsed/accordion)
- If more than 8 questions: "Show More Questions" button below
- Load more batch: 5-8 additional questions per click
- FORBIDDEN: pagination (numbered pages) — same principle as review loading
- FORBIDDEN: infinite scroll

**Step 12:** Design responsive behavior:
- Desktop: search field full-width, questions in single column, answers expand inline
- Mobile: same layout (single column works for both), 44px minimum touch target for expand/collapse
- Accordion toggle: entire question row is tappable (not just the chevron)
- Answer text: readable at mobile font sizes (14-16px minimum)

**Step 13:** Document the complete Q&A section architecture.

## Quality Criteria
- Search field positioned at TOP of Q&A section (search-first pattern)
- Brand vs community answer types clearly defined with visual distinction
- Brand answers are direct and specific (not redirects or non-answers)
- "Ask a Question" de-emphasized (small text link, not button)
- Zero-question handling = HIDE section entirely (no empty state)
- Generated Q&A pairs grounded in research data concerns (not generic)
- Question type distribution covers efficacy, usage, shipping, comparison, ingredients
- Accordion expand/collapse pattern specified
- Responsive behavior documented with 44px touch targets
- Load more mechanism (not pagination or infinite scroll)

## Failure Modes
- No search field at top of Q&A -> HALT. Search-first pattern is mandatory Baymard requirement.
- Brand and community answers visually identical -> HALT. Must have distinct visual treatment.
- "Ask a Question" as prominent button -> HALT. Must be de-emphasized text link.
- Empty Q&A section displayed -> HALT. Must HIDE entire section when 0 questions.
- Brand answer is "Please contact customer service" only -> HALT. Must directly answer the question.
- All Q&A pairs only brand-answered -> HALT. At least 30% need community answers.
- Generated questions not grounded in research -> HALT. Re-derive from 0.2 verbatim concerns.
- AI-telltale language in generated answers -> HALT. Run same anti-slop check as 2.1.
- Q&A pagination (numbered pages) -> HALT. Use "Show More" button.
