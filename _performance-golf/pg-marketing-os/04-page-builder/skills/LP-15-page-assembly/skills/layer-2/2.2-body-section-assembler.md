# 2.2 — Body Section Assembler

> **Skill:** LP-15-page-assembly
> **Layer:** 2
> **Position:** 2 of 4
> **Purpose:** Assemble all body sections in LP-04 sequence order. Write transitions between sections. This is where the bulk of the page is assembled — story, root cause, mechanism, benefits, ingredients, and proof blocks.
> **Effort Level:** opus
> **Dependencies:** `section-order-plan.md`, `content-load.md`, `transition-strategy.md`
> **Output:** `body-section-assembly.md`

## EXECUTION

**POSITIONAL REINFORCEMENT:** You assemble body sections and write transitions. You do NOT rewrite the upstream copy. The content from LP-09 (benefits/features), LP-10 (social proof), and other writing skills goes in as-is. Your creative contribution is TRANSITIONS ONLY — the connective tissue between sections.

### Step 1: Identify Body Sections

From `section-order-plan.md`, extract all body sections (everything between above-fold and conversion sections). Body sections typically include:

**Type A body sections:**
- Lead / Hook
- Story
- Root Cause Narrative
- Mechanism Narrative
- Proof Block 1
- Product Introduction
- Ingredient/Feature Deep-Dives
- Proof Block 2

**Type B body sections:**
- Key Benefits (below fold)
- How It Works
- Ingredient/Mechanism Deep-Dive
- Social Proof Strip
- Review Cascade

**Hybrid body sections:**
- Type B above-fold elements (already assembled in 2.1)
- Type A education sections below

### Step 2: Map Content to Sections

For each body section, identify the content package that provides its copy:

| Body Section | Content Package | Source Skill | Content Available? |
|-------------|----------------|-------------|-------------------|
| Benefits | benefits-features.json | LP-09 | [Y/N] |
| Features/Ingredients | benefits-features.json | LP-09 | [Y/N] |
| Social Proof / Testimonials | social-proof-copy.json | LP-10 | [Y/N] |
| Proof Blocks | social-proof-copy.json | LP-10 | [Y/N] |
| [Other sections] | [package] | LP-XX | [Y/N] |

### Step 3: Assemble Each Body Section

For each body section in LP-04 order:

1. **Read the transition-in plan** from `transition-strategy.md` for this section's incoming boundary
2. **Write the transition-in** — the connective text that bridges from the previous section to this one
3. **Place the section content** from the content package — verbatim, no rewrites
4. **Read the transition-out plan** from `transition-strategy.md` for this section's outgoing boundary
5. **Note the transition-out** — this will be written as the transition-in of the next section

**Transition writing rules:**
- Reference the previous section's key takeaway
- Create forward momentum toward the next section
- Reinforce the relevant threading anchor (promise / mechanism / proof)
- Match the page's tone (conversational for Type A, scannable for Type B)
- Type B: Most transitions are section headers. Write full transitions only where the strategy calls for them.

### Step 4: Handle Proof Block Placement

Proof blocks from LP-10 (social-proof-copy.json) are placed at the positions specified by LP-05 (social proof architecture, via architecture-load.md) and confirmed in `section-order-plan.md`.

For each proof block:
- Place the testimonials/reviews/proof elements from LP-10
- Write the transition-in ("Don't take my word for it..." or equivalent)
- Write the transition-out (bridge to next content section)
- Note the proof density (count of proof elements in this block)

### Step 5: Handle Missing Content

If a body section has no content package available (PARTIAL assembly mode):
- Insert a `[CONTENT NEEDED — LP-XX: Section Name]` placeholder
- Note the missing section in the assembly log
- Write transitions that work around the gap (bridge from the last available section to the next available one)

### Step 6: Write body-section-assembly.md

## QUALITY CRITERIA

- Every body section from `section-order-plan.md` is present (or has an explicit placeholder)
- Every section boundary has a written transition (Type A) or header transition (Type B)
- Transitions reference threading anchors per `transition-strategy.md`
- Content from upstream packages is placed verbatim — no rewrites
- Proof blocks placed at positions specified by LP-05
- Section order matches LP-04 (no unexplained reordering)
- File exists after execution

---

## FAILURE MODES

- Transition writes itself into a creative tangent -> Transitions should be 1-3 sentences (Type A) or a section header (Type B). If a transition is becoming a paragraph of new copy, you are overstepping. LP-15 writes connective tissue, not new sections.
- Content from LP-09/LP-10 gets paraphrased during assembly -> DO NOT paraphrase. Place verbatim. If the content quality is poor, LP-17 will flag it. LP-15 does not judge quality.
- Proof block positioned differently than LP-05 specified -> Check `section-order-plan.md`. The proof block position was confirmed in Layer 1. If you are moving it, document the deviation with justification.
- Body section order deviates from LP-04 -> This should not happen. If you are reordering, STOP. The order was confirmed in 1.2. Place sections in the confirmed order.
- Type B page getting long narrative transitions -> Check page type. Type B transitions are mostly headers. "How It Works" not "Now that you understand the problem, let me walk you through a remarkable discovery that changes everything..."
