# 2.5 — PDP Page Assembler

> **Skill:** LP-15-page-assembly
> **Layer:** 2
> **Position:** 5 of 5
> **Purpose:** Assemble a complete PDP (Product Detail Page) from PDP skill outputs. This is the core PDP assembly microskill — it builds the above-fold (carousel + buy box + facts panel), assembles all BTF sections in PDP-04's sequence order, integrates the review system, and specifies the sticky ATC bar. Runs INSTEAD of 2.1 (Above-Fold Assembler) + 2.2 (Body Section Assembler) + 2.3 (Conversion Section Assembler) for PDP pages.
> **Effort Level:** opus
> **Dependencies:** `pdp-packages-loaded.md` (from 0.5), `section-order-plan.md` (adapted for PDP), `transition-strategy.md` (adapted for PDP — section headers as transitions)
> **Output:** `pdp-page-assembly.md`

---

## INPUT

```yaml
input:
  pdp_packages: "pdp-packages-loaded.md"
  section_order_plan: "section-order-plan.md"
  transition_strategy: "transition-strategy.md"
  brief_load: "brief-load.md"
  page_type: "[type_b | hybrid]"
```

---

## WHEN TO RUN

This microskill runs ONLY when `page_type` is `type_b` or `hybrid`. For `type_a` pages, use 2.1 + 2.2 + 2.3 instead. **Never run both paths.**

---

## EXECUTION

**POSITIONAL REINFORCEMENT:** You are assembling PDP elements, not rewriting them. Copy comes from PDP skills (PDP-03 through PDP-07) and shared LP skills (LP-08, LP-09, LP-12, LP-13). You arrange elements according to PDP-03's above-fold blueprint and PDP-04's BTF section sequence. You do NOT generate new copy. You DO specify structural elements (sticky ATC bar, section headers, visual separators) that are PDP-specific assembly artifacts.

### Step 1: Assemble Above-Fold — Carousel

From `pdp-packages-loaded.md`, extract PDP-03's carousel specification and assemble:

1. **10 carousel slides** in order — each slide with:
   - Slide number (1-10)
   - Image brief / description
   - Alt text (accessibility)
   - Lazy-loading rule: Slide 1 = eager load (visible immediately), Slides 2-10 = lazy load
   - Thumbnail indicator (for carousel navigation dots/thumbnails)

2. **Carousel technical specs:**
   - Swipe gesture support (mobile)
   - Arrow navigation (desktop)
   - Thumbnail strip or dot indicators
   - Auto-play: OFF (user-controlled — PDP best practice)
   - Zoom capability on tap/hover

**If fewer than 10 slides available:** Assemble what exists. Flag the gap. Do NOT invent placeholder slides.

### Step 2: Assemble Above-Fold — Buy Box

From `pdp-packages-loaded.md`, combine PDP-03 architecture with PDP-06 copy for all 10 buy box components:

| # | Component | Architecture Source | Copy Source |
|---|-----------|-------------------|------------|
| 1 | Product title / H1 | PDP-03 | PDP-06 |
| 2 | Rating strip (stars + review count) | PDP-03 | PDP-05 (count + average) |
| 3 | Price block (sale, original, savings) | PDP-03 | PDP-06 |
| 4 | Variant selector (size/flavor/count) | PDP-03 | PDP-06 |
| 5 | Quantity selector | PDP-03 | PDP-03 (structural) |
| 6 | ATC button (primary CTA) | PDP-03 | PDP-06 |
| 7 | Shipping info ("Free shipping on orders $49+") | PDP-03 | PDP-06 |
| 8 | Payment badges (Visa, MC, Amex, PayPal, etc.) | PDP-03 | LP-08 |
| 9 | Short description (2-3 sentences) | PDP-03 | PDP-06 |
| 10 | Stock indicator ("Only 12 left" / "In Stock") | PDP-03 | LP-13 (urgency) |

**Assembly rules:**
- ATC button uses PDP-06 CTA copy VERBATIM. Do not modify.
- Price block displays sale price prominently, original price struck through, savings calculated.
- Variant selector shows all options with price adjustments if applicable.
- Stock indicator uses LP-13 urgency copy if available; otherwise uses PDP-03 default.

### Step 3: Assemble Above-Fold — Facts Panel

From `pdp-packages-loaded.md`, extract PDP-03's facts panel:
- Key product specs/facts (serving size, ingredient count, supply duration, etc.)
- Display format (icon + text pairs, typically)
- Placement relative to carousel and buy box

### Step 4: Specify Above-Fold Layout

Assemble the complete above-fold layout:

**Desktop layout:**
- Left: Carousel (50-60% width)
- Right: Buy box (40-50% width)
- Below carousel: Facts panel (or adjacent, per PDP-03 spec)

**Mobile layout (60%+ traffic — MOBILE FIRST):**
- Full-width carousel (swipeable)
- Buy box below carousel (full width)
- Facts panel below buy box
- Sticky ATC bar triggers when buy box scrolls out of viewport

### Step 5: Assemble BTF Sections

From `pdp-packages-loaded.md`, assemble each BTF section in PDP-04's sequence order:

For each section in the PDP-04 sequence:

1. **Section header** — serves as both title AND transition (Type B = scan-optimized, NOT story transitions)
2. **Section content** from PDP-07's BTF section copy:
   - Headline (H2 or H3)
   - Body copy
   - Bullet points (if applicable)
   - Image briefs (if applicable)
   - Icons/graphics specifications (if applicable)
3. **Visual separator** between sections (horizontal rule, background color change, or padding)
4. **Source tracking** — which PDP/LP skill provided the content

**Section-type assembly rules:**

| Section Type | Content Source | Assembly Notes |
|-------------|--------------|----------------|
| Key Benefits | LP-09 (PDP-formatted) | 4-6 benefit bullets with icons. Scannable. |
| How It Works | PDP-07 | 3-5 step visual flow. Numbered or icon-based. |
| Ingredients / What's Inside | PDP-07 | Ingredient cards or expandable panels. |
| Science / Clinical Studies | PDP-07 | Study citations, percentage claims, visual data. |
| Comparison Table | PDP-07 | "Us vs. Them" or "vs. Store Brands" table. |
| Before/After | PDP-07 + PDP-05 | Real results with disclaimers. |
| Brand Story / Values | PDP-07 | Short brand narrative. Mission statement. |
| Reviews | PDP-05 | Full review system (see Step 6). |
| FAQ | LP-12 | Accordion format (see Step 7). |
| Urgency / Limited Offer | LP-13 | Urgency bar or banner. |
| Trust / Certifications | LP-08 | Badge display, certifications, media mentions. |
| Guarantee | PDP-06 / LP-11 | Guarantee badge + copy. |

**CRITICAL:** Do NOT write story-style transitions between sections. PDP pages are scan-optimized. Section headers serve as transitions. Use visual separators (not narrative bridges) between sections.

### Step 6: Assemble Reviews Section

From PDP-05's social proof system, assemble the complete review section:

1. **Review summary bar:**
   - Average rating (X.X/5 stars)
   - Total review count
   - Star histogram (5-star: X%, 4-star: X%, etc.)

2. **Filter and sort controls:**
   - Sort by: Most Recent, Most Helpful, Highest Rated, Lowest Rated
   - Filter by: Star rating, Verified Buyer, With Photos

3. **Review display:**
   - Individual review cards: star rating, reviewer name, verified badge, date, review text
   - UGC integration: customer photos/videos inline with reviews
   - "Most Helpful" highlight: top 2-3 reviews pinned

4. **Q&A section** (if available from PDP-05):
   - Community questions with brand answers
   - "Ask a Question" prompt

### Step 7: Assemble FAQ Section (Accordion Format)

From LP-12's FAQ package, assemble as accordion:
- Each Q&A as an expandable/collapsible panel
- Questions visible (collapsed state)
- Answers revealed on click/tap
- All panels collapsed by default (except optionally the first)
- Schema markup specification for FAQ rich snippets (SEO)

### Step 8: Specify CTA Placements

PDP pages have 3 CTA placement types:

1. **Buy box ATC button** (above fold) — PDP-06 CTA copy, high-contrast, full buy-box width
2. **Mid-page CTA** (after benefits section) — secondary ATC or "Shop Now" button. Links back to buy box or triggers sticky bar.
3. **Final CTA** (page bottom) — full-width CTA section with product summary, price, and ATC button

### Step 9: Specify Sticky ATC Bar

**MANDATORY for all PDP pages.** The sticky ATC bar is a critical mobile conversion element.

```yaml
sticky_atc_bar:
  trigger: "Buy box scrolls out of viewport (scroll-based)"
  position: "Fixed bottom (mobile), fixed bottom or top (desktop)"
  contents:
    - product_name: "[abbreviated product name]"
    - price: "[sale price]"
    - atc_button: "[PDP-06 ATC button text — VERBATIM]"
    - variant_selector: "[compact variant dropdown if multiple variants]"
  style:
    background: "[high contrast — matches buy box ATC button color]"
    height: "56-64px (mobile), 48-56px (desktop)"
    z_index: "above all content, below modals"
    shadow: "top shadow for visual separation"
    animation: "slide up on trigger, slide down when buy box visible"
  mobile_touch_target: "minimum 44x44px for ATC button"
  accessibility: "aria-label, keyboard navigable"
```

## FAILURE MODES

- Writing story-style transitions between PDP sections -> FORBIDDEN. PDP pages are scan-optimized. Section headers ARE the transitions. Use visual separators between sections, not narrative bridges. If you catch yourself writing "Now that you understand..." or "But that's not all..." — STOP. That is Type A copy leaking into a Type B assembly.
- Assembling without sticky ATC bar specification -> FORBIDDEN. The sticky ATC bar is mandatory for all PDP pages. It is the #1 mobile conversion element. If it is missing, the assembly is incomplete.
- Using LP-03/LP-04/LP-05/LP-06 packages instead of PDP packages -> FORBIDDEN. PDP skills replace these. If you are reading from `above-fold-blueprint.json` (LP-03) instead of `pdp-above-fold-blueprint.json` (PDP-03), STOP. You are on the wrong assembly path.
- Modifying PDP-06 ATC button text -> DO NOT. The ATC button text from PDP-06 is placed verbatim. LP-15 assembles, not rewrites.
- Assembling fewer than 10 carousel slides without flagging the gap -> If PDP-03 provided fewer than 10 slides, assemble what exists and explicitly flag: "Carousel: X/10 slides assembled. Gap: Y slides missing from PDP-03."
- Assembling BTF sections in a different order than PDP-04 -> PDP-04 set the sequence with scan-path optimization in mind. If you deviate, document the deviation, reason, and expected impact.
- Including a P.S. section on a PDP page -> P.S. is a Type A long-form element. PDP pages do not have P.S. sections. If P.S. text exists in any package, it is NOT assembled for Type B pages.
- Omitting the reviews section -> If PDP-05 data is loaded, the reviews section MUST be assembled. Reviews are the #1 trust element on PDP pages.
