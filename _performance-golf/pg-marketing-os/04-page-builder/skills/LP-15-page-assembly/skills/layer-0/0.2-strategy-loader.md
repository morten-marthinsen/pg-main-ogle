# 0.2-strategy-loader.md

> **Layer:** 0 (Foundation)
> **Skill:** LP-15-ecomm-assembly
> **Purpose:** Load strategy for section order
> **Outputs:** `assembly_order` object with section priority sequence

---

## FUNCTION

Load the PDP-01 strategy to determine the exact assembly order. Sections must be assembled in priority order as defined in the strategy.

---

## LOADING PROTOCOL

```
LOCATE: ecomm-strategy.yaml
EXTRACT:
  section_priority_order: [ordered list of section numbers by priority]
  word_budget_summary: [total page word count]
  design_notes_seeds: [per-section design seeds]
  page_type: [pdp|collection|bundle]
```

---

## OUTPUT STRUCTURE

```json
{
  "assembly_order": {
    "sections_in_order": [1, 2, 3, 4, 9, 10, 7, 5, 14, 8, 11, 12, 16, 17],
    "total_sections": 14,
    "page_type": "pdp",
    "total_word_budget": 3200
  }
}
```

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Strategy not found | HALT -- execute PDP-01 |
| WARNING | Priority order has gaps | Fill gaps based on section number |
