# 0.5 — PDP Package Loader

> **Skill:** LP-15-page-assembly
> **Layer:** 0
> **Position:** 5 of 5
> **Purpose:** Load PDP-specific skill outputs when page type is type_b or hybrid (from LP-00). PDP skills (PDP-03 through PDP-07) replace the standard LP architecture and content packages for PDP pages. This microskill also loads shared LP skill outputs that PDP pages still consume (LP-08, LP-09, LP-12, LP-13). Runs INSTEAD of 0.2 (Architecture Loader) and 0.3 (Content Loader) for PDP pages.
> **Effort Level:** haiku
> **Dependencies:** `pdp-above-fold-blueprint.json` (PDP-03), `pdp-section-sequence.json` (PDP-04), `pdp-social-proof-system.json` (PDP-05), `pdp-offer-buy-box.json` (PDP-06), `pdp-btf-sections.json` (PDP-07), `trust-elements-package.json` (LP-08), `benefits-section-package.json` (LP-09), `faq-package.json` (LP-12), `urgency-package.json` (LP-13)
> **Output:** `pdp-packages-loaded.md`

---

```yaml
input:
  page_type: "[type_b | hybrid — from brief-load.md]"
  pdp_above_fold_path: "[path to pdp-above-fold-blueprint.json from PDP-03]"

  pdp_social_proof_path: "[path to pdp-social-proof-system.json from PDP-05]"
  pdp_offer_buy_box_path: "[path to pdp-offer-buy-box.json from PDP-06]"
  pdp_btf_sections_path: "[path to pdp-btf-sections.json from PDP-07]"
  trust_elements_path: "[path to trust-elements-package.json from LP-08]"
  benefits_section_path: "[path to benefits-section-package.json from LP-09]"
  faq_path: "[path to faq-package.json from LP-12]"
  urgency_path: "[path to urgency-package.json from LP-13]"
  project_folder: "[path to project output folder]"
```

---

## WHEN TO RUN

This microskill runs ONLY when `page_type` from `brief-load.md` (0.1) is `type_b` or `hybrid`. For `type_a` pages, use 0.2 (Architecture Loader) + 0.3 (Content Loader) instead. **Never run both paths.**

---

## EXECUTION

1. Verify page type from `brief-load.md` is `type_b` or `hybrid`. If `type_a`, STOP — this microskill should not run.

2. Locate each PDP package in the project output folder. Search paths per package:
   - `./~outputs/[project-name]/landing-page/pdp-0X-[skill-name]/[package-name].json`
   - `./~outputs/[project-name]/pdp-0X-[skill-name]/[package-name].json`

3. Locate each shared LP package in the project output folder. Search paths:
   - `./~outputs/[project-name]/landing-page/lp-XX-[skill-name]/[package-name].json`
   - `./~outputs/[project-name]/lp-XX-[skill-name]/[package-name].json`

4. For each PDP package found, extract:

### From PDP-03 (pdp-above-fold-blueprint.json):
- **Carousel specification** — 10 image slides (slide order, image briefs, alt text, lazy-loading rules)
- **Buy box architecture** — 10 components (product title, rating strip, price block, variant selector, quantity selector, ATC button, shipping info, payment badges, short description, stock indicator)
- **Facts panel** — key specs/facts displayed adjacent to or below carousel
- **Desktop vs. mobile layout** — carousel position, buy box position, mobile stack order
- **Above-fold element hierarchy**

### From PDP-04 (pdp-section-sequence.json):
- **CRITICAL:** The ordered list of BTF (below-the-fold) sections — this is the PDP assembly blueprint
- **15+ PDP-specific sections** — section names, positions, purposes, types
- Section dependencies and groupings
- Estimated word counts per section
- Proof block placement positions

### From PDP-05 (pdp-social-proof-system.json):
- **Review system architecture** — review display format, histogram (5-star distribution), filter/sort functionality
- **UGC integration** — user-generated content display (photos, videos)
- **Q&A section** — community questions and answers
- **Review count and average rating**
- **Verified buyer badges**
- **Review highlight extraction** — top reviews, most helpful reviews

### From PDP-06 (pdp-offer-buy-box.json):
- **Buy box copy** — product title copy, short description, key selling points
- **CTA copy** — ATC button text, urgency micro-copy
- **Subscription tiles** — subscribe-and-save options, pricing tiers, frequency options
- **Price display** — sale price, original price, savings, per-unit, per-day
- **Variant options** — size, flavor, count options with pricing
- **Trust micro-copy** — shipping info, money-back guarantee summary, payment badges

### From PDP-07 (pdp-btf-sections.json):
- **All BTF section copy** — complete written copy for each section defined in PDP-04's sequence
- **Section-level content** — headlines, body copy, bullet points, image briefs per section
- **Section types** — benefits, how-it-works, ingredients, comparison, science, story, brand values, etc.

5. For each shared LP package found, extract:

### From LP-08 (trust-elements-package.json):
- Trust badges, certification badges, media mentions, partner logos
- Customer count claims
- Placement recommendations

### From LP-09 (benefits-section-package.json):
- Benefit bullets (audience voice)
- Feature deep-dives (PDP-formatted: scannable, icon-friendly)
- Benefit-before-feature ordering

### From LP-12 (faq-package.json):
- All FAQ Q&A pairs — ordered by priority
- Accordion formatting specification (PDP pages use accordion, not inline)
- Objection-handling framing
- FAQ count

### From LP-13 (urgency-package.json):
- Urgency elements (limited stock, bundle expiry, subscription savings deadline)
- Scarcity justification
- Urgency copy text

6. For each missing package, log as `NOT_FOUND` with the upstream skill that must be run.

7. Map PDP package fields to LP-15's assembly schema:
   - PDP-03 carousel + buy box -> `above_fold` in assembled-page-package.json
   - PDP-04 section sequence -> `sections` ordering in assembled-page-package.json
   - PDP-05 review system -> reviews section in `sections`
   - PDP-06 buy box copy -> `above_fold.buy_box` + `cta_placements` in assembled-page-package.json
   - PDP-07 BTF sections -> `sections` content in assembled-page-package.json
   - LP-08/LP-09/LP-12/LP-13 -> mapped to corresponding sections per PDP-04 sequence

8. Write the consolidated PDP data to `pdp-packages-loaded.md`.

## PACKAGES NOT LOADED (PDP path does NOT use these)

The following LP packages are NOT loaded for PDP pages because PDP skills replace them:

| LP Package | Replaced By | Reason |
|-----------|------------|--------|
| `above-fold-blueprint.json` (LP-03) | `pdp-above-fold-blueprint.json` (PDP-03) | PDP above-fold is carousel + buy box, not hero image + headline |
| `section-sequence.json` (LP-04) | `pdp-section-sequence.json` (PDP-04) | PDP sections are scan-optimized, not story arc |
| `social-proof-architecture.json` (LP-05) | `pdp-social-proof-system.json` (PDP-05) | PDP uses review system with histogram/filters, not proof waves |
| `offer-cta-architecture.json` (LP-06) | `pdp-offer-buy-box.json` (PDP-06) | PDP uses buy box architecture, not conversion block |
| `hero-section-package.json` (LP-07) | PDP-03 above-fold blueprint | PDP has no hero section — carousel replaces it |
| `cta-copy-package.json` (LP-14) | PDP-06 buy box copy | PDP CTA is ATC button in buy box, not standalone CTA |

## FAILURE MODES

- PDP-03 pdp-above-fold-blueprint.json not found -> CRITICAL. The PDP page has no above-fold architecture (no carousel, no buy box layout). Write status as NOT_FOUND. Triggers GATE_0 HALT: "LP-15 requires pdp-above-fold-blueprint.json from PDP-03. Run PDP-03 first."
- PDP-04 pdp-section-sequence.json not found -> CRITICAL. Without PDP-04, LP-15 has no assembly blueprint for BTF sections. Write status as NOT_FOUND. Triggers GATE_0 HALT: "LP-15 requires pdp-section-sequence.json from PDP-04. Run PDP-04 first."
- PDP-06 pdp-offer-buy-box.json not found -> CRITICAL. The buy box has no copy (no ATC text, no price display, no subscription options). Write status as NOT_FOUND. Triggers GATE_0 HALT.
- PDP-07 pdp-btf-sections.json not found -> CRITICAL. No BTF section content exists to assemble. Write status as NOT_FOUND. Triggers GATE_0 HALT.
- PDP-05 not found -> Assembly can proceed with reduced social proof specificity. Flag as NOT_FOUND. Warn that reviews section will use LP-10 fallback if available.
- Shared LP packages not found -> Flag as NOT_FOUND per package. Assembly can proceed with reduced element coverage. LP-17 will catch the gaps.
- Loading LP-03/LP-04/LP-05/LP-06/LP-07/LP-14 packages for a PDP page -> FORBIDDEN. These are replaced by PDP skills. If detected, STOP and correct to PDP package path.
- PDP-03 carousel has fewer than 10 slides -> Extract what is available. Flag the gap. Note in schema mapping.
- PDP-03 buy box has fewer than 10 components -> Extract what is available. Flag missing components. Note in schema mapping.
