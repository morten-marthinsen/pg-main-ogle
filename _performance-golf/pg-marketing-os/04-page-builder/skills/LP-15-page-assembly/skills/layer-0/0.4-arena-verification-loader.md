# 0.4 — Arena Verification Loader

> **Skill:** LP-15-page-assembly
> **Layer:** 0
> **Position:** 4 of 4
> **Purpose:** CRITICAL — Load Arena selection checkpoint files (HERO_SELECTION.yaml from LP-07, CTA_SELECTION.yaml from LP-14), verify that the selected variants match the content packages, and establish the authoritative text that MUST appear verbatim in the assembled page.
> **Effort Level:** haiku
> **Dependencies:** `HERO_SELECTION.yaml` (LP-07), `CTA_SELECTION.yaml` (LP-14), `hero-section-package.json` (LP-07), `cta-copy-package.json` (LP-14)
> **Output:** `arena-verification-load.md`

---

## WHY THIS MICROSKILL EXISTS

In the CopywritingEngine, Skill 19 (Campaign Assembly) used a pre-Arena draft for the lead section instead of the Arena-selected Hybrid A. The entire lead section was wrong. This happened because the assembler loaded the content package but did not cross-reference the Arena selection checkpoint.

LP-15 will NOT repeat this mistake. This microskill loads the Arena checkpoint files, extracts the selected variant IDs and text, and cross-references them against the content packages. If there is ANY mismatch, assembly HALTS at GATE_0.

---

## INPUT

```yaml
input:
  hero_selection_path: "[path to HERO_SELECTION.yaml from LP-07 output folder]"
  cta_selection_path: "[path to CTA_SELECTION.yaml from LP-14 output folder]"
  hero_package: "content-load.md section LP-07 (or direct hero-section-package.json)"
  cta_package: "content-load.md section LP-14 (or direct cta-copy-package.json)"
  project_folder: "[path to project output folder]"
```

---

## EXECUTION

### Step 1: Locate Arena Checkpoint Files

Search for `HERO_SELECTION.yaml` in:
- `./~outputs/[project-name]/landing-page/lp-07-hero-section/HERO_SELECTION.yaml`
- `./~outputs/[project-name]/lp-07-hero-section/HERO_SELECTION.yaml`

Search for `CTA_SELECTION.yaml` in:
- `./~outputs/[project-name]/landing-page/lp-14-cta-optimizer/CTA_SELECTION.yaml`
- `./~outputs/[project-name]/lp-14-cta-optimizer/CTA_SELECTION.yaml`

### Step 2: Extract Arena Selections

From `HERO_SELECTION.yaml`, extract:
- Selected variant ID (e.g., "variant_3", "hybrid_a")
- Selected headline text (verbatim)
- Selected deck copy text (verbatim, if included in selection)
- Arena round info (which round, which lens selected it)
- Arena score (if available)

From `CTA_SELECTION.yaml`, extract:
- Selected variant ID
- Selected CTA text (verbatim) for each placement position
- Arena round info
- Arena score (if available)

### Step 3: Cross-Reference Against Content Packages

**Hero Cross-Reference:**
1. Read the LP-07 section of `content-load.md` (or read `hero-section-package.json` directly)
2. Find the variant matching the selected variant ID from HERO_SELECTION.yaml
3. Compare the headline text character-by-character
4. Result: MATCH or MISMATCH

**CTA Cross-Reference:**
1. Read the LP-14 section of `content-load.md` (or read `cta-copy-package.json` directly)
2. Find the variant matching the selected variant ID from CTA_SELECTION.yaml
3. Compare the CTA text character-by-character
4. Result: MATCH or MISMATCH

### Step 4: Establish Authoritative Text

If both cross-references return MATCH:
- Record the Arena-selected headline as the AUTHORITATIVE headline — this exact text goes into the assembled page
- Record the Arena-selected CTA text as the AUTHORITATIVE CTA — this exact text goes into the assembled page

If ANY cross-reference returns MISMATCH:
- Record the mismatch details
- Flag which file (checkpoint YAML or content package) appears to have the discrepancy
- Set verification status to MISMATCH

### Step 5: Write arena-verification-load.md

## QUALITY CRITERIA

- Both Arena checkpoint files located (or explicitly flagged as NOT_FOUND)

- File exists after execution

---

## FAILURE MODES

- HERO_SELECTION.yaml not found -> Set status to CHECKPOINT_NOT_FOUND. Triggers GATE_0 HALT: "LP-15 requires HERO_SELECTION.yaml from LP-07 Arena. Run LP-07 Arena or provide checkpoint."
- CTA_SELECTION.yaml not found -> Set status to CHECKPOINT_NOT_FOUND. Triggers GATE_0 HALT: "LP-15 requires CTA_SELECTION.yaml from LP-14 Arena. Run LP-14 Arena or provide checkpoint."
- Cross-reference MISMATCH detected -> Set status to MISMATCH. Triggers GATE_0 HALT: "Arena-selected variant does not match content package. LP-[07/14] must re-export with Arena selection or Arena must be re-run."
- Checkpoint file found but malformed (not parseable YAML) -> Set status to CHECKPOINT_NOT_FOUND with note "file exists but is not parseable YAML." Triggers GATE_0 HALT.
- Content package does not contain the variant ID named in the checkpoint -> Set status to MISMATCH with note "variant ID [X] from checkpoint not found in content package." This suggests the content package was regenerated after Arena selection without preserving the selected variant.
