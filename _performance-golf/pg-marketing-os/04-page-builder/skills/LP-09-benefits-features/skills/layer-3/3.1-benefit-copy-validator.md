# 3.1 — Benefit Copy Validator (10-Point Audit)

**Skill:** LP-09-benefits-features
**Position:** Layer 3, Microskill 1
**Purpose:** Audit all benefit section copy against the 10-point scoring rubric. This is the quality gate that determines whether benefit copy is ready for the output package. Minimum score: 7.5/10. Below that, revisions are required before assembly.
**Effort Level:** medium
**Dependencies:** All Layer 2 outputs (benefit-bullets.md, ingredient-deep-dives.md, feature-explanations.md, how-it-works.md, comparison-benefits.md — whichever were generated)
**Output:** `benefit-audit.md`

---

## Execution Logic

1. **Collect all Layer 2 outputs.** Verify each expected output exists based on the classification from 1.1. If any classified section is missing its output -> Flag as `MISSING_OUTPUT` and score item 8 as FAIL.

2. **Score each of the 10 audit points:**

### The 10-Point Rubric

**Point 1: D-F-W-B-P Completeness** (0 or 1)
- Read the D-F-W-B-P chain for every element across all outputs
- PASS: Every element has all 5 components populated (PROOF_GAP counts as populated if explicitly flagged)
- FAIL: Any element has a missing component (empty, "TBD," or silently omitted)

**Point 2: Proof Specificity** (0 or 1)
- Check every Proof component in every D-F-W-B-P chain
- PASS: Every proof names a specific source + includes a number/finding (or is explicitly flagged PROOF_GAP)
- FAIL: Any proof says "studies show," "research proves," "experts agree," or similar vague language

**Point 3: Benefit-First Language** (0 or 1)
- Read the first 5 words of every benefit bullet (Type A headline, Type B full bullet)
- PASS: Benefits lead in copy — the outcome/benefit is stated before the feature/ingredient
- FAIL: Features or ingredients lead (e.g., "500mg Ashwagandha supports..." instead of "Fall asleep faster with...")

**Point 4: Scan-Friendliness (Type B)** (0, 1, or N/A)
- Only applies to Type B and Hybrid pages
- PASS: Bullets are 8-12 words, start with benefit verb, no ingredient leads
- FAIL: Bullets are too long, start with ingredients, or are not scannable

**Point 5: Ingredient Accuracy (Supplement)** (0, 1, or N/A)
- Only applies to supplement/health verticals
- PASS: All doses match the brief, mechanisms are in plain language, no fabricated citations
- FAIL: Any dose mismatch, jargon-heavy mechanism language, or fabricated citation detected

**Point 6: Tone Consistency** (0 or 1)
- Compare the voice/register of benefit copy against the hero section headline and threading anchor
- PASS: Benefit copy feels like it belongs on the same page as the hero section
- FAIL: Noticeable tone shift (e.g., hero is conversational, benefits are clinical/formal)

**Point 7: Zero AI Telltales** (0 or 1)
- This point is satisfied by the anti-slop scan (3.3)
- Pre-check: quickly scan for obvious AI telltales before formal scan
- PASS: No obvious telltales on quick scan
- FAIL: Telltales detected — defers to formal anti-slop scan for complete list

**Point 8: Section Completeness** (0 or 1)
- PASS: Every section classified in 1.1 has corresponding copy output from Layer 2
- FAIL: Any classified section is missing its output

**Point 9: Comparison Fairness** (0, 1, or N/A)
- Only applies if comparison sections were generated
- PASS: All comparison claims are factually stated, no competitor bashing, claim verification table present
- FAIL: Unverified claims, superlative language without data, or competitor attacks detected

**Point 10: How-It-Works Clarity** (0, 1, or N/A)
- Only applies if how-it-works sections were generated
- PASS: 3-5 steps, each with benefit, sequential, final step connects to hero promise
- FAIL: >5 steps, steps without benefits, non-sequential, or disconnected from promise

3. **Calculate score:** Points earned / applicable points * 10 = score out of 10.
   - Example: 8 applicable points, 7 pass = 8.75/10 (PASS)
   - Example: 10 applicable points, 7 pass = 7.0/10 (FAIL — below 7.5)

4. **If FAIL:** Identify which points failed and provide specific revision instructions for each.

## Quality Criteria
- All 10 points explicitly scored (N/A for non-applicable)
- Failed points have specific revision instructions (not "make it better")
- Score calculation is correct (verified mathematically)
- Tone consistency check references the actual hero section text
- Assessment is actionable — the model can execute revisions based on the audit

## Failure Modes
- Scoring all 10 points as PASS without actually checking -> Every point must have evidence in the Notes column. "PASS" without justification is not acceptable.
- Marking non-applicable points as PASS (inflating score) -> N/A points are excluded from the denominator. Do not count them.
- Vague revision instructions ("improve the proof") -> Instructions must be specific. "Ingredient 3 (Zinc): Proof says 'studies suggest zinc helps immunity.' Replace with specific study name, year, and finding — or flag as PROOF_GAP."
- Not catching tone drift -> Re-read the hero headline text alongside the benefit copy. Read them aloud (mentally). Do they sound like the same writer? If not, the tone check fails.
