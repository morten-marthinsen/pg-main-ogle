## Pre-Generation Protocol

**Step 1: Load specimens into active context**
Hold loaded benefit bullet specimens verbatim. Before writing Bullet 1, explicitly study each specimen: What makes these bullets scannable? How do they integrate proof? What verb patterns do they use?

**Step 2: Read hero threading anchor**
Read the primary hook phrase and promise statement from LP-07. All bullet language must feel like it belongs on the same page as the hero section. If the hero is direct and number-specific, bullets must be direct and number-specific. If the hero is conversational and empathetic, bullets follow.

**Step 3: Review anti-slop word list**
Before writing: scan the forbidden words from LP-09-AGENT.md. Commit to producing zero instances.

**Step 4: Read the D-F-W-B-P plan for each assigned element**
Each element has a planned chain from 1.2. The plan provides the scaffold — generation brings it to life with specific language.

## Generation Execution

### Type A Format (Long-Form Benefit Bullets)

For each assigned section, write 4-8 benefit bullets using this structure:

```
**[Benefit Headline — bold, outcome-focused, 8-15 words]**
[1-3 sentences weaving Feature + Why + Proof together. The Deliverable is context.
The Benefit is the headline. Feature explains HOW. Why bridges to the reader.
Proof is specific — named source, year, number.]
```

**Writing rules for Type A:**
- Benefit headline is the LEAD — not the feature, not the ingredient
- The body sentences must complete the D-F-W-B-P chain
- Proof must be embedded naturally, not bolted on as an afterthought
- Each bullet addresses a DISTINCT benefit (no overlapping bullets)
- Order: Most compelling benefit first, second-most compelling last
- Total word count per section: 200-600 words (4-8 bullets)

### Type B Format (Scan-Optimized Bullets)

For each assigned section, write 4-6 benefit bullets using this structure:

```
[Icon placeholder] [Benefit in 8-12 words — present tense, outcome-focused]
```

**Writing rules for Type B:**
- Start with a benefit VERB (Supports, Promotes, Helps restore, Delivers, Naturally boosts)
- NEVER start with ingredient or feature name
- Each bullet is a distinct benefit (no overlap)
- Order: Most compelling first, second-most compelling last (middle bullets are skimmed)
- Total: 4-6 bullets per section
- D-F-W-B-P chain is compressed: the bullet IS the Benefit; Feature and Proof may appear below as a subline or in a separate tooltip/expandable area

**Type B subline option** (for pages that allow expanded bullets):
```
[Icon] [Benefit — 8-12 words]
[Feature + Proof — 1 sentence, smaller text]
```

### Before/After Framing (Embedded Variant)

If any section is classified as `before_after`, embed the framing within benefit bullets:

```
**Without [Product]:**
- [Pain point 1 in audience language]
- [Pain point 2]
- [Pain point 3]

**With [Product]:**
- [Benefit 1 — mirrors pain 1, with proof]
- [Benefit 2 — mirrors pain 2, with proof]
- [Benefit 3 — mirrors pain 3, with proof]
```

## Quality Criteria
- 4-8 bullets per section (Type A) or 4-6 per section (Type B)
- Every bullet has a complete D-F-W-B-P chain documented
- Type B bullets are 8-12 words, starting with benefit verb
- Type A bullets have benefit-first headlines with Feature+Why+Proof body
- Proof is specific (named, dated, numbered) or flagged as PROOF_GAP
- No two bullets address the same benefit dimension
- Tone matches hero section threading anchor
- Zero forbidden words from anti-slop list

## Failure Modes
- All bullets start with the same verb pattern -> Vary the opening verbs. 6 bullets starting with "Supports..." is monotonous.
- Bullets are benefit headlines without body text (Type A) -> Type A requires 1-3 sentence body completing the D-F-W-B-P chain. Headlines alone are Type B format.
- Type B bullets exceed 12 words -> Tighten. Scan-optimized means compressed. Every word earns its place.
- Proof is "research shows" or "studies prove" -> Replace with specific citation or flag as PROOF_GAP. Vague proof is worse than flagged absence.
- Bullet order is random -> Reorder: most compelling first, second-most compelling last. Middle bullets get less attention.
