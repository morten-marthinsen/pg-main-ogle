# 1.3 — Ingredient Strategy Planner (Type A Supplement Only)

**Skill:** LP-09-benefits-features
**Position:** Layer 1, Microskill 3
**Purpose:** For Type A health/supplement pages, plan the detailed ingredient deep-dive strategy: confirm dose from brief, identify mechanism of action, locate clinical citation, and map to felt benefit. This microskill only runs for supplement/health verticals. Non-supplement pages skip to Layer 2.
**Effort Level:** medium
**Dependencies:** `packages-loaded.md` (0.1), `dfwbp-structure-plan.md` (1.2)
**Output:** `ingredient-strategy.md`

---

input:

## Pre-Execution Gate

**This microskill runs ONLY when:**
- Vertical is health, supplement, fitness, or wellness
- Page type is Type A or Hybrid
- At least one ingredient is listed in the brief

**If conditions not met:** Skip this microskill. Write `ingredient-strategy.md` with content: "N/A — non-supplement page or Type B (condensed format handled in 1.2)."

---

## Execution Logic

1. **For each ingredient**, create a strategy card:

   **Dose Confirmation:**
   - Extract dose from page-brief.json
   - If dose is missing: flag as `DOSE_MISSING` — this blocks the ingredient deep-dive for this ingredient. Cannot write a deep-dive claiming specific dosing without confirmed data.
   - If dose is present: record dose + form (e.g., "500mg KSM-66 Ashwagandha root extract")

   **Mechanism of Action:**
   - Identify the primary mechanism (how this ingredient works in the body)
   - Write in plain language accessible to the target audience
   - If mechanism data available from brief or proof assets -> use it
   - If mechanism data not available -> flag as `MECHANISM_NEEDS_RESEARCH` and note what information is needed
   - Examples of good mechanism language: "KSM-66 reduces cortisol production by modulating the HPA axis — your body's stress response system" (accessible) vs. "Withanolides inhibit NF-kB signaling" (too technical)

   **Clinical Citation:**
   - Match ingredient to available clinical studies from proof-architecture.json
   - Required format: [Journal/Source name] + [Year] + [Key numerical finding]
   - If no study available: flag as `CITATION_GAP`
   - Acceptable alternatives: expert endorsement, usage data, historical/traditional use data (clearly labeled as such, not presented as clinical evidence)

   **Felt Benefit:**
   - Translate the clinical/mechanistic outcome into language the customer would use
   - Must be specific and experiential, not clinical
   - Must connect to the audience's primary pain or desire from the brief
   - Examples: "Fall asleep in under 20 minutes" (good) vs. "Improved sleep onset latency" (clinical — bad for copy)

   **Presentation Priority:**
   - Rank ingredients by relevance to the hero promise
   - Primary ingredients (directly support the core promise) -> full deep-dive
   - Supporting ingredients (enhance or protect) -> condensed deep-dive
   - Secondary ingredients (general support) -> brief mention or bullet only

2. **Plan the deep-dive section order:**
   - Lead with the hero ingredient (most directly connected to the headline promise)
   - Follow with supporting ingredients in descending order of relevance
   - Close with a "complete formula" summary that ties all ingredients to the single core benefit

3. **Identify data gaps** that block writing:
   - `DOSE_MISSING` -> Cannot write deep-dive until dose is confirmed
   - `MECHANISM_NEEDS_RESEARCH` -> Can write a simplified version but flag as incomplete
   - `CITATION_GAP` -> Can write the deep-dive with PROOF_GAP flag (from 1.2)

4. **Write `ingredient-strategy.md`** with the complete strategy.

## Quality Criteria
- Every ingredient has a strategy card with all fields addressed
- Doses confirmed from brief (not assumed or estimated)
- Mechanisms written in plain language (test: would a non-scientist understand?)
- Clinical citations include journal name + year + numerical finding
- Felt benefits are in customer language, not clinical terminology
- Presentation order leads with hero ingredient connected to headline promise
- All data gaps explicitly flagged with gap type and resolution path

## Failure Modes
- Dose listed in brief without unit -> Flag as `DOSE_MISSING`. "500 ashwagandha" is incomplete. Need "500mg KSM-66 Ashwagandha root extract."
- Mechanism written in technical jargon -> Rewrite in accessible language. The audience is the benchmark, not the scientist.
- Clinical citation is "a study showed..." without specifics -> Not a valid citation. Must have journal/source name, year, and number. Flag as `CITATION_GAP` instead.
- Hero ingredient not identified -> Default to the ingredient most directly connected to the primary pain from the audience profile. If truly ambiguous, flag for human selection.
- Ingredients list in brief is just names without doses or forms -> Flag all as `DOSE_MISSING`. Cannot write accurate deep-dives. Escalate to human for complete ingredient data.
