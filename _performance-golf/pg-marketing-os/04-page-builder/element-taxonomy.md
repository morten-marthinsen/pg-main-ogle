# Landing Page Engine — Element Taxonomy

**Version:** 1.1
**Date:** 2026-03-05
**Source:** Cross-analysis of 11 landing page specimens from high-impression Facebook ads
**Purpose:** Validates and extends the PRD's theoretical element list with real-world frequency data. Informs all 19 LP Engine skills about which elements to prioritize.

---

## Overview

The PRD defines two theoretical page type anatomies:

**Type A (Long-Form Sales Page):** Pre-head > Headline > Deck > Lead > Story > Root Cause > Mechanism > Proof Block 1 > Product Introduction > Ingredient Deep-Dives > Proof Block 2 > Value Stack > Pricing > Guarantee > Urgency > Proof Block 3 > FAQ > Close > P.S.

**Type B (Ecomm/PDP):** Breadcrumb > H1 > Rating Strip > Hero Image > Price/Savings > ATC Primary > Trust Badges > Short Description > Key Benefits > Variant/Quantity > Ingredient Deep-Dives > How It Works > Review Cascade > FAQ > Guarantee > Comparison Table > Upsell/Bundle > Sticky ATC

What follows is what we ACTUALLY observed across 11 specimens, categorized by function.

## PDP-SPECIFIC ELEMENTS (Type B / Hybrid — PDP Enhancement Layer)

**Source:** Baymard Institute, NN/g, NLS PDP Best Practices research (1,186 lines). These elements are specific to Product Detail Pages and are handled by PDP skills (PDP-03 through PDP-16).

### Image Carousel Elements

| Element | Baymard Finding | PDP Skill |
|---------|----------------|-----------|
| Thumbnail navigation (vertical strip) | 50-80% of users miss dot-only navigation. Thumbnails required. | PDP-03 |
| 10-Thumbnail NLS Story Architecture | Outcome > Problem > Ingredient > Enjoyment > How It Works > What to Expect > How to Use > Credibility > Comparison > Testimonial | PDP-03 |
| Left-column vertical thumbnails (desktop) | Preferred layout over horizontal below-image thumbnails | PDP-03 |
| Swipe gesture support (mobile) | Required for mobile carousel — no arrow-only navigation | PDP-16 |

### Buy Box Elements

| Element | Baymard Finding | PDP Skill |
|---------|----------------|-----------|
| Full product title (never truncated) | 46% of users misidentify product when title is truncated | PDP-03, PDP-06 |
| Anchor-link star rating | Rating must link to review section below | PDP-03 |
| Dual savings display (% + $) | Show both percentage AND dollar amount saved | PDP-06 |
| Exposed variant chips | 60% friction with dropdown selectors vs. exposed chips | PDP-06 |
| Stepper quantity selector (+/-) | NEVER dropdown for quantity — UX finding | PDP-06 |
| Full-width ATC button | Button spans full buy box width — highest click-through | PDP-06 |
| Subscription tile (never pre-selected) | Pre-selected subscription = dark pattern, destroys trust | PDP-06 |
| Slide-out mini-cart | NEVER redirect to cart page — slide-out maintains page context | PDP-06 |
| Micro-trust signals (below ATC) | 3-5 icon + 3-5 word phrases: guarantee, shipping, security | PDP-06, LP-08 |
| Facts panel (below buy box) | 6-8 scannable chips: dosage, certifications, key claims | PDP-03 |

### BTF Section Elements

| Element | Baymard Finding | PDP Skill |
|---------|----------------|-----------|
| Vertical benefit layout | 27% overlooked in horizontal vs. 8% in vertical layout | LP-09 (2.8) |
| "X for Y" ingredient naming | Ingredient name + specific benefit in 3-5 words | LP-09 (2.6) |
| Benefit microscripts | 2-5 word benefit phrases used across carousel, badges, ATC | LP-09 (2.7) |
| Accordion FAQ (5-7 items only) | Top 5-7 curated questions — NOT 10+ (reduces engagement) | LP-12 (1.4, 2.5) |
| Review histogram with keyword filters | Star distribution + topic-based filtering | PDP-05 |
| UGC video carousel | Customer video reviews in horizontal scroll | PDP-05 |
| Q&A section (community-style) | Question + verified-purchase answers | PDP-05 |
| Sticky ATC bar (mobile) | Highest-impact single mobile conversion change | PDP-06, PDP-16 |
