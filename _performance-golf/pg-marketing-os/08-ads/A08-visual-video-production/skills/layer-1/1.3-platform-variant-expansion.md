# 1.3: Platform Variant Expansion

## Purpose
Expand each asset list by platform -- determine which assets need 1:1, 4:5, 9:16, and 16:9 versions. Not all assets need all aspect ratios. Some assets are platform-native (generated specifically for that platform), while others need multi-platform renders. This microskill calculates the total render count.

## Inputs
- 1.1 Asset Decomposition (complete asset inventory)
- 0.4 Format Strategy (platform-concept matrix)
- Platform-Specific Technical Requirements table from AGENT.md

## Process
```
FOR EACH asset in the 1.1 decomposition:

  STEP 1: DETERMINE PLATFORM TARGETS
    - Which platforms does this concept target? (from 0.4 format strategy)
    - Which aspect ratios are needed? (from platform spec table)

  STEP 2: DETERMINE GENERATION STRATEGY
    Option A: NATIVE GENERATION (preferred)
      - Generate separate assets per aspect ratio at Layer 2
      - Each version is composed natively for its platform
      - Result: Best quality, proper framing per platform

    Option B: REFRAME FROM MASTER (acceptable)
      - Generate one master asset and reframe for other ratios
      - Use intelligent crop/recompose (not simple crop)
      - Result: Faster production, slight quality trade-off

    Selection criteria:
      - Hero images: Option A (framing matters too much)
      - B-roll video: Option B acceptable (scene content matters more than framing)
      - Talking head: Option A (different framing per platform)
      - Text overlays: MUST be repositioned per platform (never simply scaled)
      - Product shots: Option A preferred (framing affects product prominence)

  STEP 3: EXPAND ASSET LIST
    - For each asset, create platform-specific entries
    - Apply naming convention: [concept]-[variant]-[PLATFORM]-[asset-type]-v1
    - Record generation strategy (native vs. reframe)

  STEP 4: CALCULATE TOTALS
    - Total unique assets before expansion: [count from 1.1]
    - Total platform-specific renders after expansion: [new count]
    - Expansion factor: [ratio]
```
