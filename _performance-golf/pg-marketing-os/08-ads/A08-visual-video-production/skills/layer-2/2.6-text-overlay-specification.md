# 2.6: Text Overlay Specification

## Purpose
Generate text overlay specifications for all assets that include text: exact text content from A07 copy, font/size/color from A05 visual direction, and position coordinates per platform aspect ratio. These specs are used by Layer 3 assembly to composite text onto images and video.

## Inputs
- A07 AD-COPY-FINAL/ (exact text content -- headlines, CTAs, key phrases)
- A05 Visual Direction (typography specs: font, size, color, style)
- 1.3 Platform Variant Expansion (aspect ratios and safe zones per platform)
- Platform-Specific Technical Requirements (safe zones, text areas)

## Process
```
FOR EACH asset that includes text overlay:

STEP 1: EXTRACT TEXT CONTENT
  From A07 copy:
    - Headline text (exact -- no modifications)
    - Subheadline text (if applicable)
    - CTA text (exact button/banner text)
    - Key phrases for video overlays (with timing marks from script)
    - Social proof text (if applicable)
    - Lower third text (name, title)

STEP 2: EXTRACT TYPOGRAPHY SPECS
  From A05 visual direction:
    - Font family (primary, secondary)
    - Font size (for each text element)
    - Font weight (bold, regular, light)
    - Font color (hex value)
    - Text shadow/outline (if specified)
    - Text animation (for video: fade in, type on, slide, etc.)

STEP 3: CALCULATE POSITION PER PLATFORM
  Platform-specific text positioning:
    Meta Feed (1:1 / 4:5):
      - Headline: top third, centered
      - CTA: bottom quarter, centered
      - Safe zone: 40px margin from edges
    Meta Stories (9:16):
      - Headline: upper third (below 250px from top)
      - CTA: lower third (above 250px from bottom)
      - Safe zone: 250px top/bottom, 40px sides
    TikTok (9:16):
      - Headline: upper third
      - CTA: above bottom 150px (platform UI zone)
      - Safe zone: 150px bottom, 250px top
    YouTube Pre-Roll (16:9):
      - Headline: center or lower third
      - CTA: avoid skip button area (bottom-right)
      - Safe zone: 100px from edges

  Calculate exact (x, y) coordinates for each text element per platform

STEP 4: SPECIFY TEXT RENDERING INSTRUCTIONS
  For each text element:
    - Text content: [exact text]
    - Font: [family, size, weight, color]
    - Position: (x, y) per platform
    - Timing: [start_time, end_time] for video overlays
    - Animation: [type, duration] for video overlays
    - Background: [none / semi-transparent box / gradient]
    - Readability check: contrast ratio against background

STEP 5: WRITE SPECIFICATIONS
  Output complete text overlay spec per asset per platform
```

## Quality Checks
- CRITICAL: All text content is verbatim from A07 (zero modifications)
- Typography specs match A05 visual direction
- Text positions are within platform-specific safe zones
- Text positions are DIFFERENT per platform (not copy-pasted from one ratio to another)
- Video overlay timing matches script timing marks
- Contrast ratio is adequate for readability at mobile preview size
- CTA placement respects platform UI conventions
