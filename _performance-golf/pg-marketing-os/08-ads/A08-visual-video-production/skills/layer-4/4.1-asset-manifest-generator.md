# 4.1: Asset Manifest Generator

## Purpose
Generate the complete YAML manifest of every produced asset with full metadata: lineage tracing (concept, variant, platform), tool used, exact prompt, quality scores, platform specs, file paths, and review verdicts. This manifest is the single source of truth for what was produced and where it lives.

## Inputs
- All Layer 2 generation logs (2.1, 2.2, 2.3, 2.4, 2.5, 2.6)
- All Layer 2.5 quality audit results (2.5.1, 2.5.2, 2.5.3, 2.5.4)
- All Layer 3 assembly logs (3.1, 3.2, 3.3, 3.4)
- AD-ASSETS/ directory contents
- Asset Naming Convention from AGENT.md

## Process
```
STEP 1: COLLECT ALL ASSET METADATA
  FOR EACH produced asset (component + assembled):
    Collect:
      - asset_id (from naming convention)
      - concept (C01, C02, etc.)
      - variant (V01, V02, etc.)
      - platform (meta-feed, tiktok, etc.)
      - asset_type (hero-img, vo, talking-head, etc.)
      - version (v1, v2, v3)
      - file_path (relative to AD-ASSETS/)
      - source_brief (exact A05 section reference)
      - source_copy (exact A07 section reference)
      - tool_used (Midjourney, ElevenLabs, etc.)
      - prompt_used (exact text for AI-generated assets)
      - generation_params (model version, aspect ratio, style params)
      - quality_review (PASS/FAIL/FALLBACK)
      - quality_notes (specific review notes)
      - platform_specs:
          aspect_ratio, resolution, file_size, format, color_space (images)
          duration, codec, bitrate (video/audio)

STEP 2: ORGANIZE BY HIERARCHY
  - Group by concept -> variant -> platform -> asset type
  - Ensure complete lineage chain for every asset

STEP 3: VERIFY COMPLETENESS
  - Cross-reference manifest against AD-ASSETS/ directory
  - Every file in directory must have a manifest entry
  - Every manifest entry must have a corresponding file
  - Flag any discrepancies

STEP 4: GENERATE YAML MANIFEST
  Output structured YAML with all metadata per asset
```

- Quality review verdicts match Layer 2.5 audit results
