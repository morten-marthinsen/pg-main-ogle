# 2.3: Tier Assignment

**Version:** 1.0
**Layer:** 2 (Comparative Analysis & Tier Assignment)
**Type:** Classification / Prioritization
**Model:** opus
**Dependencies:** 2.2 (Forced Ranking), 0.4 (Campaign Brief for min_creatives_per_week)
**Output:** tier_assignment.md

---

## Purpose

Assign each variant to one of three testing tiers based on ranking position, composite score thresholds, and budget constraints. Tier 1 variants launch first, Tier 2 are backup, Tier 3 are revised or cut. The tier assignment must respect the testing volume formula — Tier 1 count is CAPPED by the minimum creatives per week that the budget can support. Every Tier 3 variant must have a documented reason for demotion and (for 3A) a recommended revision direction.

## Process

### Step 1: Load Tier Thresholds and Constraints

```
TIER THRESHOLDS (from AGENT.md — FIXED):
  Tier 1 — Launch First:
    - Confidence-adjusted composite >= 7.0
    - Compliance score >= 6
    - Maximum count = min_creatives_per_week (from testing volume formula)

  Tier 2 — Backup:
    - Confidence-adjusted composite >= 5.5
    - Compliance score >= 5
    - Not selected for Tier 1

  Tier 3 — Revise or Cut:
    - Confidence-adjusted composite < 5.5
    - OR compliance score <= 4
    - Sub-tiers:
      3A = Revise (specific issues, creative potential exists)
      3B = Cut (fundamental problems, not worth revision effort)

LOAD min_creatives_per_week from 0.4
LOAD compliance flags from 1.4
```

### Step 2: Assign Tier 1

```
FROM the forced-ranked list (top to bottom):

SELECT variants that meet ALL Tier 1 criteria:
  - confidence_adjusted_composite >= 7.0
  - compliance_score >= 6
  - No unresolved compliance flags

COUNT qualifying variants

IF qualifying_count <= min_creatives_per_week:
  All qualifying variants → Tier 1

IF qualifying_count > min_creatives_per_week:
  Only the top [min_creatives_per_week] qualifying variants → Tier 1
  Remaining qualifying variants → Tier 2 (score qualifies, budget doesn't)
  DOCUMENT: "Variants V-XXX through V-YYY qualified for Tier 1 by score
    but were placed in Tier 2 due to weekly creative budget cap of
    [min_creatives_per_week]."

IF qualifying_count == 0:
  WARNING: "No variants meet Tier 1 thresholds. Consider:
    (a) Revising variants in A07
    (b) Lowering thresholds (requires human approval)
    (c) Re-examining scoring methodology"
  PRESENT to human for decision

RECORD Tier 1 list with ranks and scores
```

### Step 3: Assign Tier 2

```
FROM remaining unassigned variants:

SELECT variants that meet Tier 2 criteria:
  - confidence_adjusted_composite >= 5.5
  - compliance_score >= 5
  - Not in Tier 1

ALL qualifying → Tier 2

FOR EACH Tier 2 variant:
  DOCUMENT: activation trigger
  "This variant activates when: [specific condition]"
  Typical triggers:
    - A Tier 1 variant is killed (spend reallocation)
    - Tier 1 testing data arrives showing performance gaps
    - Week 2-3 of campaign (standard promotion timing)

RECORD Tier 2 list with ranks, scores, and activation triggers
```

### Step 4: Assign Tier 3 (Revise or Cut)

```
FROM remaining unassigned variants (everything not in Tier 1 or Tier 2):

FOR EACH remaining variant:
  CLASSIFY as 3A (Revise) or 3B (Cut):

  TIER 3A — REVISE (creative potential exists):
    Criteria for 3A:
      - Composite score 4.0-5.5 (close to Tier 2 threshold)
      - Has at least 2 dimensions scoring >= 6 (some strengths exist)
      - Problems are addressable (compliance fix, hook revision, CTA strengthening)
    FOR EACH 3A variant:
      DOCUMENT:
        - specific_reason: why demoted (which dimensions are weak)
        - revision_direction: specific recommendation for improvement
          e.g., "Rewrite hook to reduce competitive saturation overlap.
                 Current hook type matches 8 competitors. Suggest switching
                 to [alternative hook type] which has 2% competitive share."
        - priority: HIGH (close to Tier 2) / MEDIUM / LOW
        - send_to: "A07 Copy Production for revision" or "A04 Script Architecture for restructure"

  TIER 3B — CUT (fundamental problems):
    Criteria for 3B:
      - Composite score < 4.0
      - OR critical compliance violation (score <= 2) with no easy fix
      - OR 3+ dimensions scoring <= 4 (systemic weakness)
      - OR concept-level problem (not fixable by revision)
    FOR EACH 3B variant:
      DOCUMENT:
        - specific_reason: why cut (be specific — not "generally weak")
          e.g., "Compliance score 1 (disease cure claims), scroll-stop
                 score 3 (oversaturated hook type), conversion score 2
                 (CTA misaligned with funnel stage). Fundamental concept
                 issues make revision impractical."
        - creative_elements_salvageable: boolean + list
          (even cut variants may have usable elements — a strong visual
           treatment or proof element that can be transplanted)

RECORD Tier 3A and 3B lists with full documentation
```

### Step 5: Validate Tier Distribution

```
COUNT variants per tier:
  Tier 1: [X]
  Tier 2: [Y]
  Tier 3A: [Z]
  Tier 3B: [W]
  Total: X + Y + Z + W = total_variants (MUST match)

DISTRIBUTION CHECKS:
  1. Tier 1 count <= min_creatives_per_week (budget cap respected)
  2. No compliance-flagged variants in Tier 1 or Tier 2
  3. Distribution is not pathological:
     - If 90%+ in Tier 1: scoring was insufficiently critical
     - If 90%+ in Tier 3: consider whether upstream skills need re-execution
     - If 0 in Tier 1: present to human (no variants ready for launch)
  4. All Tier 3 variants have documented reasons
  5. All Tier 3A variants have revision directions
  6. All Tier 3B variants have cut rationale
  7. Score thresholds are consistent (no 8.0 in Tier 2 while 6.5 in Tier 1)
```

## Output File

- **Path:** `layer-2-outputs/2.3-tier-assignment.md`
- **Minimum size:** 5KB (must include all tiers with full documentation per variant)

---

## Quality Checks

1. ALL variants assigned to exactly one tier (total matches inventory)
2. Tier 1 count does not exceed min_creatives_per_week
3. No compliance-flagged variants (score <= 3) in Tier 1 or Tier 2
4. Score thresholds consistently applied (no threshold violations)
5. Every Tier 3A variant has a specific revision direction (not generic)
6. Every Tier 3B variant has a specific cut rationale (not generic)
7. Distribution is not pathological (not 90%+ in any single tier)
8. Activation triggers documented for all Tier 2 variants
9. Budget overflow variants documented if Tier 1 was capped
10. All Tier 3 reasons reference specific dimension scores and evidence
