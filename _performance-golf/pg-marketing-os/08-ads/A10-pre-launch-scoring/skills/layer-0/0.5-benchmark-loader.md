# 0.5: Benchmark Loader

**Version:** 1.0
**Layer:** 0 (Foundation)
**Type:** Data Loading
**Dependencies:** AD-HOOK-TAXONOMY.md, AD-ENGINE.md, A12 Performance Learning (optional)
**Output:** benchmark_reference.md

---

## Purpose

Load industry benchmarks and historical performance data to provide scoring reference points for all 5 prediction dimensions. Sources include AD-HOOK-TAXONOMY.md (32 hook types with performance benchmarks), AD-ENGINE.md (testing reference data, creative lifespan, win rate expectations), and A12 PERFORMANCE-LEARNING.md if available from previous campaigns. Build a benchmark reference table that Layer 1 scorers use as evidence.

---

## Inputs

```yaml
input:
  from_references:
    hook_taxonomy_path: "./References/AD-HOOK-TAXONOMY.md"
    ad_engine_claude_path: "./ads/AD-ENGINE.md"
  from_A12_optional:
    performance_learning_path: "[project]/A12-performance-learning/PERFORMANCE-LEARNING.md"
    # OPTIONAL — only exists if previous campaigns have been run
```

---

## Process

### Step 1: Load AD-HOOK-TAXONOMY.md Benchmarks

```
READ AD-HOOK-TAXONOMY.md
EXTRACT for each of the 32 hook types:
  - hook_type_name: string
  - hook_type_id: string
  - benchmark_hook_rate: float (% of viewers who stop scrolling)
  - benchmark_engagement_rate: float (if available)
  - typical_format_pairings: [video, static, carousel, etc.]
  - strength_indicators: what makes this hook type work
  - weakness_indicators: when this hook type fails
  - fatigue_profile: fast / medium / slow

KEY BENCHMARK: Target hook rate is 30%+ (only 14% of hooks achieve this)
This is the reference point for Dimension 1 (Scroll-Stop) scoring.

BUILD hook_benchmark_table:
  | Hook Type | Benchmark Hook Rate | Fatigue Profile | Engagement Profile |
```

### Step 2: Load AD-ENGINE.md Testing Data

```
READ AD-ENGINE.md
EXTRACT:
  - win_rate_expectations: 6.6-30% of tested creatives will be winners
  - creative_lifespan_by_spend:
    - Low (<$5K/week): 4-8 weeks
    - Medium ($5-25K/week): 2-4 weeks
    - High (>$25K/week): 1-2 weeks
  - fatigue_benchmark: 37% performance drop after 7 days at high spend
  - testing_volume_formula: weekly spend / (3 × AOV)
  - kill_rule: 3x AOV spend without conversion
  - scale_protocol: +20-30% daily spend, max 1 event per 48 hours
  - platform_review_timelines:
    - Meta: 24-48 hours
    - TikTok: 24-72 hours
    - Google: 24-48 hours
  - minimum_data_window: 72 hours before kill decisions

These become the default benchmarks for Dimensions 2 (Engagement),
3 (Conversion), and 5 (Fatigue) scoring.
```

### Step 3: Check for A12 Historical Data

```
CHECK: Does A12 PERFORMANCE-LEARNING.md exist?

IF EXISTS:
  READ A12 output
  EXTRACT:
    - prediction_vs_actual_calibration:
      - Previous A10 scroll-stop predictions vs. actual hook rates
      - Previous A10 conversion predictions vs. actual CPA
      - Previous A10 fatigue predictions vs. actual creative lifespan
    - tier_performance_history:
      - What % of Tier 1 variants were actually winners?
      - What % of Tier 2 variants were promoted?
      - What % of Tier 3 variants were correctly classified?
    - dimension_accuracy:
      - Which dimensions were most accurate?
      - Which dimensions had the widest error margins?
    - calibration_adjustments:
      - If A10 was consistently optimistic on scroll-stop, note the bias
      - If A10 was consistently pessimistic on fatigue, note the bias

  SET a12_available = true

  BUILD calibration_table:
    | Dimension | Previous Prediction | Actual Result | Bias Direction | Adjustment |

IF NOT EXISTS:
  SET a12_available = false
  LOG: "No A12 historical data available. All predictions will use benchmarks
    only. Confidence levels will be MEDIUM or LOW."
  NOTE: This is normal for first-time campaigns. Not an error.
```

### Step 4: Build Benchmark Reference Table

```
COMPILE all benchmark data into a unified reference:

SCROLL-STOP BENCHMARKS:
  - Target hook rate: 30%+
  - Per hook type: [benchmark_hook_rate from Step 1]
  - Competitive saturation penalty: -1 to -2 score points for oversaturated types
  - Competitive whitespace boost: +1 to +2 score points for underused types

ENGAGEMENT BENCHMARKS:
  - Platform average watch-through rates
  - Script structure effectiveness ratings
  - Word count compliance thresholds

CONVERSION BENCHMARKS:
  - Win rate expectations: 6.6-30%
  - Vertical-specific conversion benchmarks (if available)
  - Funnel stage calibration:
    - TOFU: direct conversion unlikely, engagement = proxy
    - MOFU: 1-3% CTR typical
    - BOFU: 2-5%+ CTR achievable

FATIGUE BENCHMARKS:
  - Default: 37% drop after 7 days at high spend
  - By spend level: [lifespan table from Step 2]
  - By hook type: [fatigue profiles from Step 1]

HISTORICAL CALIBRATION (if A12 available):
  - Prediction accuracy by dimension
  - Bias adjustments to apply
```
