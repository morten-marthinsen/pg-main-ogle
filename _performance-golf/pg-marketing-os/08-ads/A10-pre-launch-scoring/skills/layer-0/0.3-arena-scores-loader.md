# 0.3: Arena Scores Loader

**Version:** 1.0
**Layer:** 0 (Foundation)
**Type:** Data Loading
**Dependencies:** A06 Ad Arena Results
**Output:** arena_scores_extract.md

---

## Purpose

Load the A06 AD-ARENA-RESULTS.md and extract per-concept Arena scores across the 7 ad-specific judging criteria. Map Arena concept scores to individual variant IDs from the A09 matrix — multiple variants may derive from a single Arena-scored concept. This provides the expert-panel quality assessment that feeds Dimensions 1 (Scroll-Stop), 2 (Engagement), and 3 (Conversion).

---

## Inputs

```yaml
input:
  from_A06:
    arena_results_path: "[project]/A06-ad-arena/AD-ARENA-RESULTS.md"
  from_0.1:
    variant_inventory: "layer-0-outputs/0.1-variant-matrix-loader.md"
    # Needed for variant-to-concept mapping
```

---

## Process

### Step 1: Locate A06 Arena Results

```
VERIFY: A06 AD-ARENA-RESULTS.md exists at specified path
IF NOT EXISTS:
  → HALT: "A06 Arena Results not found. Execute A06 (Ad Arena) first."
  → A10 CANNOT score without Arena quality assessments.
```

### Step 2: Extract Per-Concept Arena Scores

```
READ AD-ARENA-RESULTS.md
FOR EACH concept evaluated in the Arena:
  EXTRACT the 7 ad-specific criteria scores:
    1. scroll_stop_power: float (1-10)
    2. visual_copy_coherence: float (1-10)
    3. mechanism_clarity: float (1-10)
    4. platform_nativeness: float (1-10)
    5. proof_integration: float (1-10)
    6. cta_strength: float (1-10)
    7. memorability: float (1-10)

  ALSO EXTRACT:
    - concept_id: unique identifier
    - overall_weighted_score: float
    - winning_competitor: which persona won
    - arena_round: which round (should be Round 2 (FINAL) final)
    - key_strengths: from Arena evaluation
    - key_weaknesses: from Arena evaluation
```

### Step 3: Map Concepts to Variants

```
LOAD variant inventory from 0.1 output
FOR EACH variant in inventory:
  FIND parent_concept_id
  LOOK UP that concept in Arena scores

  IF concept found:
    MAP all 7 Arena scores to this variant
    SET arena_data_available = true

  IF concept NOT found in Arena:
    SET arena_data_available = false
    LOG: "Variant [ID] parent concept [concept_id] has NO Arena scores."
    NOTE: This variant will be scored with LOWER confidence on dimensions that
          rely on Arena data (scroll-stop, engagement, conversion)

BUILD concept_to_variant_mapping:
  concept_001 → [V-001, V-002, V-003]  # 3 variants from same concept
  concept_002 → [V-004, V-005]          # 2 variants from same concept
  concept_003 → [V-006]                 # 1 variant from this concept
```

### Step 4: Identify Coverage Gaps

```
CALCULATE:
  - total_concepts_scored: count of unique concepts with Arena scores
  - total_variants_with_arena: count of variants with mapped Arena scores
  - total_variants_without_arena: count of variants with no Arena data
  - coverage_percentage: variants_with_arena / total_variants * 100

IF coverage_percentage < 50%:
  → WARNING: "Less than half of variants have Arena scores. Scoring confidence
    will be LOW for most variants."
  → DO NOT HALT — proceed but document the limitation

IF coverage_percentage == 0%:
  → HALT: "No variants have Arena scores. A06 must be executed before A10 can
    produce meaningful predictions."
```

## Output Schema

```yaml
output:
  arena_scores:
    a06_file_loaded: string  # path
    total_concepts_scored: integer
    total_variants_with_arena: integer
    total_variants_without_arena: integer
    coverage_percentage: float

    concept_scores:
      - concept_id: string
        scroll_stop_power: float
        visual_copy_coherence: float
        mechanism_clarity: float
        platform_nativeness: float
        proof_integration: float
        cta_strength: float
        memorability: float
        overall_weighted_score: float
        winning_competitor: string
        arena_round: integer
        key_strengths: [string]
        key_weaknesses: [string]
        variants_mapped: [string]  # list of variant IDs

    variants_without_arena:
      - variant_id: string
        parent_concept_id: string
        reason: string  # e.g., "concept not evaluated in A06"

    scoring_implications:
      high_confidence_variants: [string]  # have Arena scores
      low_confidence_variants: [string]   # no Arena scores
```

## Quality Checks

1. A06 Arena Results file exists and was read successfully
2. At least 1 concept has Arena scores (coverage > 0%)
3. All 7 ad-specific criteria scores extracted per concept (no partial scorecards)
4. Concept-to-variant mapping complete (every variant checked)
5. Coverage gaps documented with specific variant IDs
6. Variants without Arena data flagged for low-confidence scoring
7. No concept IDs are duplicated in the extraction

- A06 (Ad Arena) must be complete — produces AD-ARENA-RESULTS.md
- 0.1 (Variant Matrix Loader) must be complete — provides variant inventory with parent_concept_id mapping
