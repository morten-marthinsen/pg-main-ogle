# 0.0.1: Vertical Profile Loader

**Version:** 1.0
**Layer:** 0 (Foundation)
**Type:** Configuration Loading
**Dependencies:** None (first microskill in execution)
**Output:** vertical_profile.yaml

---

## Purpose

Load ad-specific vertical configuration from `ad-verticals/[vertical].md`. This microskill extracts vertical-specific scoring benchmarks, compliance thresholds, platform performance baselines, and anti-slop rules that calibrate all 5 prediction dimensions. Without vertical context, scroll-stop benchmarks are generic, compliance scoring misses vertical-specific restrictions, and fatigue predictions ignore vertical-specific creative lifespans.

---

## Inputs

```yaml
input:
  from_campaign_brief:
    vertical: string  # One of: golf, health, finance, personal-dev, technology
  from_ad_verticals_directory:
    vertical_profile_path: "ads/ad-verticals/[vertical].md"
```

---

## Process

### Step 1: Identify Vertical

```
READ Campaign Brief (Skill 09 output) for vertical identifier
IF vertical not specified:
  → DEFAULT to "universal" (no vertical-specific config)
  → LOG warning: "No vertical specified — using universal configuration"
```

### Step 2: Load Vertical Profile

```
IF vertical specified:
  CONSTRUCT path: ads/ad-verticals/[vertical].md
  READ vertical profile file
  EXTRACT:
    - scoring_benchmarks (vertical-specific hook rate targets, CPA ranges, ROAS expectations)
    - compliance_thresholds (regulatory/platform-specific restrictions for this vertical)
    - platform_performance_baselines (expected CTR, CPC, CVR by platform for this vertical)
    - anti_slop_rules (vertical-specific language to avoid in scoring commentary)
    - fatigue_baselines (vertical-specific creative lifespan data)
    - hook_type_performance (which of the 32 hook types perform best in this vertical)
```

### Step 3: Normalize and Structure

```
COMPILE vertical_profile:
  vertical_id: "[vertical identifier]"
  scoring_benchmarks:
    target_hook_rate: "[percentage — vertical-specific target]"
    target_cpa_range: "[dollar range — vertical-specific]"
    target_roas: "[multiplier — vertical-specific]"
    win_rate_expectation: "[percentage range — vertical-specific]"
  compliance_thresholds:
    risk_level: "[low/medium/high — overall vertical compliance risk]"
    forbidden_claims: [list of claims that violate platform/regulatory rules]
    required_disclaimers: [list if applicable]
    platform_specific_restrictions:
      meta: [restrictions]
      tiktok: [restrictions]
      google: [restrictions]
  platform_performance_baselines:
    meta:
      avg_ctr: "[percentage]"
      avg_cpc: "[dollar amount]"
      avg_hook_rate: "[percentage]"
    tiktok:
      avg_ctr: "[percentage]"
      avg_cpc: "[dollar amount]"
      avg_hook_rate: "[percentage]"
    google:
      avg_ctr: "[percentage]"
      avg_cpc: "[dollar amount]"
  anti_slop:
    forbidden_phrases: [vertical-specific slop to avoid]
    forbidden_language_patterns: [e.g., health: "cure" "treat", finance: "guaranteed returns"]
  fatigue_baselines:
    avg_creative_lifespan_weeks: "[range at moderate spend]"
    fatigue_onset_frequency: "[typical frequency at fatigue onset]"
    refresh_cadence: "[recommended refresh interval]"
  hook_type_performance:
    high_performing_types: [list of hook types that work well in this vertical]
    underperforming_types: [list of hook types that don't work well in this vertical]
```

### Step 4: Set Downstream Configuration

```
CONFIGURE downstream microskills:
  - Layer 1 scroll-stop scorer (1.1) will use vertical hook rate benchmarks
  - Layer 1 conversion scorer (1.3) will use vertical CPA/ROAS benchmarks
  - Layer 1 compliance scorer (1.4) will use vertical compliance thresholds
  - Layer 1 fatigue scorer (1.5) will use vertical fatigue baselines
  - Layer 2 composite scoring will reference vertical performance baselines
  - Layer 3 testing strategy will use vertical-specific budget and timeline guidance
```

## Quality Checks

1. If vertical specified, profile file was successfully loaded
2. If vertical not specified, universal fallback is documented
3. All required sections populated (scoring_benchmarks, compliance_thresholds, platform_performance_baselines, anti_slop, fatigue_baselines, hook_type_performance)
4. Scoring benchmarks are specific (not "varies" or "TBD")
5. Compliance thresholds include at least 3 forbidden claims for health/finance verticals
6. Hook type performance data includes at least 3 high-performing types
7. Fatigue baselines are grounded in vertical data (not generic defaults)
