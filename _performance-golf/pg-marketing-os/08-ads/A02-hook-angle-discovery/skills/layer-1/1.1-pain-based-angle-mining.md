# 1.1: Pain-Based Angle Mining

## Purpose

Extract strategic angles from Research pain quotes and Root Cause output. These angles address the prospect's felt pain and create entry points for scroll-stopping hooks.

## Inputs

- Research pain quotes (from 0.1-upstream-packages.md — top 10-15)
- Root Cause package (from Skill 03)

## Process

1. **Extract top pain quotes from research**
   - Load the 10-15 highest intensity pain quotes
   - Source, quote text, intensity rating

2. **Group pain quotes by theme**
   - What specific suffering do they describe?
   - Are there pain clusters (multiple quotes about same pain)?
   - What is the EMOTIONAL core of each pain theme?

3. **For each pain theme, derive a strategic angle**
   - What is the prospect FEELING?
   - What would make them stop scrolling?
   - How does the root cause REFRAME this pain?
   - State the angle in PLAIN LANGUAGE (no creative expression yet)

4. **Source attribution**
   - Each angle must cite:
     - Specific research quote(s) it derives from
     - Pain theme it addresses
     - How root cause connects to this pain

5. **Quality filter**
   - Angle must be SPECIFIC (not "people are frustrated")
   - Angle must be EXTERNAL (not the reader's fault)
   - Angle must create CURIOSITY (invites exploration)
   - Angle must LEAD to big idea (not dead-end)
