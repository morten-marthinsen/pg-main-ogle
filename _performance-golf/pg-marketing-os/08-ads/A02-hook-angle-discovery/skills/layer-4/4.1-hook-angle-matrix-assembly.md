# 4.1: HOOK-ANGLE-MATRIX.md Assembly

## Purpose
Assemble all 7 required sections of the primary output file HOOK-ANGLE-MATRIX.md. This is the complete deliverable consumed by downstream skills A03, A04, and A07.

## Inputs
- All Layer 1 outputs (1.1-1.7) — strategic angles
- All Layer 2 outputs (2.1-2.7) — full hook bank
- All Layer 2.5 outputs (2.5.1-2.5.7) — scoring
- All Layer 3 outputs (3.1-3.2) — human selection

## Process
1. Assemble Section 1: Strategic Angles (from 1.6)
2. Assemble Section 2: Full Hook Bank (from 2.6)
3. Assemble Section 3: Top 15-20 Presentation (from 2.5.7 + 3.1)
4. Assemble Section 4: Human Selections (from 3.2)
5. Assemble Section 5: Hook-Angle Cross-Reference
6. Assemble Section 6: Platform Recommendations
7. Assemble Section 7: Variant Potential

## Output Schema (All 7 Sections Required)

### Section 1: Strategic Angles
- All 10-15 angles with plain language statements
- Source attribution (quotes, data, upstream skills)
- Composite ranking scores
- Big Idea connection statements

### Section 2: Full Hook Bank
- All 50-100+ hooks (pre-curation)
- Unique IDs, hook text, type classification
- Source angle, platform tags
- All 6 individual scores + composite

### Section 3: Top 15-20 Presentation
- Hooks that advanced to human curation
- Organized by angle group
- Full scoring breakdown with rationale

### Section 4: Human Selections
- 8-10 selected hooks
- Modifications (before/after)
- New hooks added
- Human notes (binding constraints)

### Section 5: Hook-Angle Cross-Reference
- Matrix: which hooks serve which angles
- Angle coverage analysis
- Angles with no surviving hooks (strategic gap flag)

### Section 6: Platform Recommendations
- Per-selected-hook platform mapping
- Sound-on vs sound-off considerations
- Vertical-specific platform notes

### Section 7: Variant Potential
- Estimated variants per selected hook
- Body compatibility groups (which hooks can share same ad body)
- Total estimated variant count

## Output File
- **Path:** `HOOK-ANGLE-MATRIX.md` (project root)
- **Minimum size:** 30KB

## Quality Checks
- All 7 sections populated
- No empty sections
- File size >= 30KB
- Human selections documented

## Dependencies
- All upstream layers complete
- HUMAN_SELECTION.yaml exists
