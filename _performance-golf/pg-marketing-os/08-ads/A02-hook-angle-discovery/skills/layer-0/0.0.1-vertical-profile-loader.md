# 0.0.1: Vertical Profile Loader

## Purpose

Load ad-specific vertical configuration from `ads/ad-verticals/` to configure hook taxonomy emphases, platform-specific patterns, and niche-specific anti-slop rules for the campaign.

## Inputs

- Campaign Brief (Skill 09) — niche/vertical identifier
- `ads/ad-verticals/[vertical].md` files

## Process

1. **Extract vertical from Campaign Brief**
   - Read campaign brief niche field
   - Map to one of 5 verticals: golf, health, finance, personal-dev, technology
   - If no mapping → default to "universal"

2. **Load vertical profile file**
   - Path: `ads/ad-verticals/[vertical].md`
   - If file doesn't exist → log warning, use universal configuration
   - If exists → extract all sections

3. **Extract configuration**
   - Hook taxonomy emphases (which of 32 types perform best in this vertical)
   - Platform-specific patterns (platform preferences for this vertical)
   - Niche-specific anti-slop rules (language patterns to avoid)
   - Vertical-specific specimens (if any)
   - Compliance notes (regulatory considerations for this vertical)

4. **Normalize and validate**
   - Ensure all required fields present
   - Validate hook type codes against 32-type taxonomy
   - Flag any missing or malformed configuration

## Output File

- **Path:** `layer-0-outputs/0.0.1-vertical-profile.md`
- **Minimum size:** 1KB

## Quality Checks

- Vertical profile file found OR universal configuration used
- All taxonomy emphases reference valid hook type codes (A1-J3)
- Platform list is non-empty
- Anti-slop rules specific to vertical (not generic)

## Dependencies

- Campaign Brief must exist
- AD-HOOK-TAXONOMY.md must be available for validation
