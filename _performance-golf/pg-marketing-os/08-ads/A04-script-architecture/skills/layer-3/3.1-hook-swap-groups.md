# 3.1: Hook Swap Groups

## Purpose
For each ad concept, identify which additional hooks can share the same body. Group hooks by body compatibility. Verify each swap hook's promise is fulfilled by the body (coherence check). Design clean seam. Target: 3-5 compatible hooks per body.

## Inputs
- Complete architectures from Layer 2 (or 2.5 if Arena ran)
- Full hook bank from A02 (not just selected 8-10)

## Process
For each concept body, test additional hooks from A02 for coherence compatibility. Group compatible hooks. Design universal transition sentence that works with ANY hook in the group.

## Output File
- **Path:** `layer-3-outputs/3.1-hook-swap-groups.md`
- **Minimum size:** 5KB

## Dependencies
- Layer 2 (or 2.5) complete
