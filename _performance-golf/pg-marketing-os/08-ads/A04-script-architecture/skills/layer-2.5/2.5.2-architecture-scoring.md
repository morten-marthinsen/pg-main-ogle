# 2.5.2: Architecture Scoring

## Purpose
Score each architecture variant on 5 criteria: (1) Hook-body coherence strength, (2) Mechanism integration quality, (3) Variant testability, (4) Platform nativeness, (5) Persuasion sequence logic. Weighted composite.

## Inputs
- Alternative architectures (from 2.5.1)

## Process
Apply 5 Arena criteria specific to A04. Calculate weighted composite scores. Rank architectures per concept.

- **Path:** `layer-2.5-outputs/2.5.2-architecture-scoring.md`
- **Minimum size:** 3KB
