# 2.4: Full Unit Coherence

## Purpose

For EVERY combination that passed ALL THREE individual coherence dimensions (2.1 Hook-Body, 2.2 Body-CTA, 2.3 Hook-Visual), evaluate the COMPLETE ad unit as a coherent whole. Individual dimension passes do not guarantee overall coherence -- a hook, body, CTA, and visual can each pair well individually but produce a disjointed experience as a complete unit. This microskill evaluates the end-to-end viewer experience.

## Inputs

- **HOOK-BODY-COHERENCE.yaml:** From 2.1 -- per-combination hook-body verdicts
- **BODY-CTA-COHERENCE.yaml:** From 2.2 -- per-combination body-CTA verdicts
- **HOOK-VISUAL-COHERENCE.yaml:** From 2.3 -- per-combination hook-visual verdicts
- **RAW-COMBINATIONS.yaml:** From 1.1 -- full combination details
- **COPY-CATALOG.yaml:** From 0.1 -- full copy content
- **ASSET-CATALOG.yaml:** From 0.2 -- full visual descriptions

## Process

```
STEP 1: BUILD EVALUATION SET
  - Identify all combinations that received PASS on ALL THREE dimensions:
    - Hook-Body: PASS
    - Body-CTA: PASS
    - Hook-Visual: PASS
  - Also include combinations where all three are PASS or FLAG (mixed PASS/FLAG)
  - EXCLUDE combinations with ANY FAIL in any dimension
  - Count total combinations for full-unit evaluation

STEP 2: FOR EACH QUALIFYING COMBINATION, EVALUATE COMPLETE AD UNIT
  Evaluate the end-to-end experience:

  QUESTION 1: UNIFIED MESSAGE
    Does the hook -> body -> CTA -> visual sequence deliver a SINGLE,
    unified message? Or does the viewer receive contradictory signals?
    - A hook about gut health + a body about energy + a CTA about weight loss
      = three different messages = FAIL even if each pair works individually

  QUESTION 2: TONAL CONSISTENCY
    Is the tone consistent from hook through CTA through visual?
    - Hook is dramatic/serious but CTA is casual/playful = tonal break
    - Visual is professional but copy is informal = register mismatch
    - Body is educational but visual is emotional = tone gap

  QUESTION 3: NARRATIVE ARC
    Does the complete unit tell a coherent micro-story?
    - Hook creates interest/tension
    - Body develops the premise
    - CTA resolves with action
    - Visual supports the narrative throughout
    - If any element breaks the arc, the unit is incoherent

  QUESTION 4: VIEWER JOURNEY
    Would a real viewer who encounters this ad experience a smooth journey?
    - No jarring transitions between elements
    - No promise-delivery gaps
    - No "wait, what just happened?" moments
    - The viewer arrives at the CTA in a natural, motivated state

  QUESTION 5: CROSS-ELEMENT REINFORCEMENT
    Do the elements reinforce each other or merely coexist?
    - PASS: Each element adds to the others (hook builds on visual,
      body deepens hook's promise, CTA capitalizes on body's momentum)
    - FLAG: Elements coexist without contradiction but don't actively reinforce
    - FAIL: Elements work against each other

STEP 3: ASSIGN COHERENCE SCORE (1-10)
  For each combination, assign a numerical coherence score:

  9-10: Exceptional coherence -- all elements reinforce each other,
        unified message, seamless viewer journey
  7-8:  Strong coherence -- clear unified message, minor room for improvement
  6-7:  Acceptable coherence -- message is clear but some elements don't
        actively reinforce (FLAG territory)
  4-5:  Weak coherence -- message is muddled, viewer may be confused
  1-3:  Incoherent -- contradictory elements, broken viewer journey

STEP 4: ASSIGN VERDICT BASED ON SCORE
  - PASS: Score > 7.0 -- coherent, testable ad unit
  - FLAG: Score 6.0 - 7.0 -- borderline, needs human review
  - FAIL: Score < 6.0 -- incoherent, should not be in the matrix

STEP 5: DO NOT BATCH-EVALUATE
  CRITICAL: Each combination gets its own coherence score with specific
  reasoning across all 5 questions. No batch scoring. No "these 30 all
  score 8.0." Each unit is unique and gets unique evaluation.

STEP 6: COMPUTE STATISTICS
  - Total evaluated
  - Total PASS (score > 7.0)
  - Total FLAG (score 6.0 - 7.0)
  - Total FAIL (score < 6.0)
  - Average coherence score
  - Score distribution (how many in each range)
  - Most common coherence weakness

STEP 7: WRITE FULL-UNIT-COHERENCE.yaml
```
