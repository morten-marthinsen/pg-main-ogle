# 2.5: Coherence Validator

## Purpose

Aggregate all coherence results from 2.1-2.4 into a comprehensive validation report. Count total PASS, FAIL, and FLAG across all four dimensions. Verify the minimum 30 combinations have PASS status across all dimensions. If fewer than 30 pass, analyze failure patterns and determine whether FLAGged combinations can be promoted to PASS with documented reasoning.

## Inputs

- **HOOK-BODY-COHERENCE.yaml:** From 2.1 -- hook-body verdicts
- **BODY-CTA-COHERENCE.yaml:** From 2.2 -- body-CTA verdicts
- **HOOK-VISUAL-COHERENCE.yaml:** From 2.3 -- hook-visual verdicts
- **FULL-UNIT-COHERENCE.yaml:** From 2.4 -- full-unit verdicts and coherence scores

## Process

```
STEP 1: LOAD ALL COHERENCE RESULTS
  - Read all four coherence output files
  - Build per-combination coherence profile:
    For each combination, record:
      - Hook-Body: PASS/FAIL/FLAG
      - Body-CTA: PASS/FAIL/FLAG
      - Hook-Visual: PASS/FAIL/FLAG
      - Full-Unit: PASS/FAIL/FLAG (with score)

STEP 2: CLASSIFY EACH COMBINATION
  - ALL_PASS: Hook-Body=PASS AND Body-CTA=PASS AND Hook-Visual=PASS AND Full-Unit=PASS
    -> These are testable variants for the matrix
  - ANY_FAIL: At least one dimension is FAIL
    -> These are excluded from the matrix
  - MIXED_FLAG: No FAIL but at least one FLAG
    -> These need human review or can be promoted with reasoning

STEP 3: COUNT AND VERIFY THRESHOLD
  - Count ALL_PASS combinations
  - Count ANY_FAIL combinations
  - Count MIXED_FLAG combinations
  - Check: ALL_PASS >= 30? PASS or FAIL

STEP 4: ANALYZE FAILURE PATTERNS (if ALL_PASS < 30 or for diagnostic purposes)
  - Which dimension causes the most failures?
    Rank: Hook-Body fail count, Body-CTA fail count, Hook-Visual fail count, Full-Unit fail count
  - What are the top 3 failure reasons per dimension?
  - Which hook types fail most often?
  - Which visual types fail most often?
  - Which CTA types fail most often?
  - Are failures concentrated in specific concepts or platforms?

STEP 5: FLAG PROMOTION ANALYSIS (if ALL_PASS < 30)
  For each MIXED_FLAG combination:
    - Which dimension(s) are flagged?
    - What was the specific concern?
    - Can the concern be addressed without modifying the elements?
    - Would a human reasonably approve this as testable?
    - Document promotion reasoning if promoting to PASS

  Calculate: ALL_PASS + promotable_FLAG >= 30?
    - If YES: Proceed with documented promotions (human reviews flagged)
    - If NO: ESCALATE TO HUMAN with full analysis

STEP 6: PRODUCE MANDATORY COHERENCE VALIDATION CHECK TABLE
  Output the formatted check table from AGENT.md:

  +------------------------------------------------------------------------+
  | DIMENSION               | TOTAL | PASS | FAIL | FLAG | % PASS         |
  +------------------------------------------------------------------------+
  | Hook-Body Coherence     | [X]   | [X]  | [X]  | [X]  | [X]%           |
  | Body-CTA Coherence      | [X]   | [X]  | [X]  | [X]  | [X]%           |
  | Hook-Visual Coherence   | [X]   | [X]  | [X]  | [X]  | [X]%           |
  | Full Unit Coherence     | [X]   | [X]  | [X]  | [X]  | [X]%           |
  +------------------------------------------------------------------------+

  AGGREGATE:
  | Total PASS (all 4 dimensions)    | [X]  | PASS if >= 30 / FAIL if < 30 |
  | Total FLAG (any dimension)       | [X]  | For human review              |
  | Total FAIL (any dimension)       | [X]  | Excluded from matrix          |

STEP 7: DETERMINE GATE 2 STATUS
  Gate 2 PASSES if:
    - Minimum 30 combinations have ALL_PASS status
    - OR minimum 30 combinations have ALL_PASS + documented FLAG promotions
  Gate 2 FAILS if:
    - Fewer than 30 combinations can achieve PASS status

STEP 8: WRITE COHERENCE-VALIDATION-REPORT.md
```
