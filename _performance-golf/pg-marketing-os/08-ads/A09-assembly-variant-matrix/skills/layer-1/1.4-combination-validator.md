# 1.4: Combination Validator

## Purpose

Validate the filtered and platform-mapped combination list. Count total valid combinations (after filtering and platform mapping). Analyze distribution by concept, platform, and hook type. Verify the minimum 30 valid combinations survive. If fewer than 30, perform bottleneck analysis to identify which input dimension is the constraint and recommend specific upstream additions.

## Inputs

- **INCOMPATIBILITY-LOG.yaml:** From 1.2 -- filter results
- **PLATFORM-MAPPING.yaml:** From 1.3 -- platform mapping results
- **RAW-COMBINATIONS.yaml:** From 1.1 -- original enumeration
- **COPY-CATALOG.yaml:** From 0.1 -- for bottleneck analysis
- **ASSET-CATALOG.yaml:** From 0.2 -- for bottleneck analysis

## Process

```
STEP 1: LOAD ALL LAYER 1 OUTPUTS
  - Read RAW-COMBINATIONS.yaml (total theoretical)
  - Read INCOMPATIBILITY-LOG.yaml (removed, flagged, surviving)
  - Read PLATFORM-MAPPING.yaml (ready, needs adaptation, blocked)

STEP 2: COMPUTE VALID COMBINATION COUNT
  - Valid for coherence validation = combinations that are:
    - NOT removed by incompatibility filter (INC rules)
    - NOT blocked by platform mapping
  - This includes: READY + NEEDS_ADAPTATION + FLAGGED (soft incompatibility)
  - FLAGGED combinations proceed to coherence validation -- they're decided there
  - NEEDS_ADAPTATION combinations proceed -- adaptation is a production task, not a coherence issue

STEP 3: ANALYZE DISTRIBUTION
  Count valid combinations by:
    a. Per concept (C1: X, C2: X, C3: X)
    b. Per platform (Meta: X, TikTok: X, YouTube: X, etc.)
    c. Per hook type category (A: X, B: X, C: X, ..., J: X)
    d. Per body variant (B1: X, B2: X, etc.)
    e. Per visual type (UGC: X, polished: X, motion_graphics: X, etc.)

  Flag imbalances:
    - Concept with < 5 valid combinations: WARNING
    - Platform with < 5 valid combinations: WARNING
    - Hook category with 0 valid combinations: WARNING

STEP 4: GATE 1 THRESHOLD CHECK
  Minimum thresholds:
    a. Total valid combinations >= 30: PASS or FAIL
    b. Concepts represented >= 3: PASS or FAIL
    c. Platforms represented >= 2: PASS or FAIL

  IF ALL pass: Gate 1 PASSES -> proceed to Layer 2
  IF ANY fail: Gate 1 FAILS -> perform bottleneck analysis

STEP 5: BOTTLENECK ANALYSIS (if Gate 1 FAILS)
  Identify which input dimension is the bottleneck:
    - Hooks per concept: need >= 5 (how many does each concept have?)
    - Bodies per concept: need >= 2
    - CTAs per concept: need >= 2
    - Visuals per concept: need >= 2
    - Platforms targeted: need >= 2

  For each dimension below minimum:
    - Calculate: how many additional elements would bring total valid to 30+
    - Recommend: specific upstream action (e.g., "Request 3 more hooks for C2 from A07")

  Also analyze:
    - Are incompatibility rules removing too many? Which rules?
    - Are platform format issues blocking too many? Which platform/format?
    - Would adding one more platform or one more visual double the count?

STEP 6: WRITE COMBINATION-VALIDATION-REPORT.md
  - Write complete validation report with:
    - Summary statistics (theoretical -> filtered -> valid)
    - Distribution analysis (by concept, platform, hook type)
    - Gate 1 determination (PASS or FAIL)
    - Bottleneck analysis (if FAIL)
    - Specific recommendations (if FAIL)

STEP 7: PRODUCE MANDATORY COMBINATION COUNT CHECK TABLE
  Output the formatted check table from AGENT.md:

  +------------------------------------------------------------------------+
  | METRIC                      | VALUE    | STATUS                        |
  +------------------------------------------------------------------------+
  | Theoretical combinations    | [X]      | (reference only)              |
  | Incompatible pairs removed  | [X]      | (reference only)              |
  | Platform-blocked removed    | [X]      | (reference only)              |
  | Valid combinations remaining| [X]      | PASS if >= 30 / FAIL if < 30  |
  | Combinations per Concept 1  | [X]      | (reference only)              |
  | Combinations per Concept 2  | [X]      | (reference only)              |
  | Combinations per Concept 3  | [X]      | (reference only)              |
  | Platforms represented       | [X]      | PASS if >= 2 / FAIL if < 2    |
  +------------------------------------------------------------------------+
```
