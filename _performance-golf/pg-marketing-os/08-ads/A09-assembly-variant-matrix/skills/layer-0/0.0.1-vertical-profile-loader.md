# 0.0.1: Vertical Profile Loader

## Purpose

Load the ad-specific vertical configuration from `ad-verticals/` to configure platform priorities, compliance constraints, and anti-slop rules for the assembly and variant matrix process. This is the FIRST microskill to execute -- everything downstream depends on the vertical configuration being loaded.

## Inputs

- **Vertical Profile File:** `./ads/ad-verticals/[vertical].md` (one of: golf, health, finance, personal-dev, technology)
- **Project Brief:** The project brief specifying which vertical this campaign targets
- **AD-ENGINE.md:** For fallback/universal configuration if no vertical is specified

## Process

```
STEP 1: IDENTIFY VERTICAL
  - Read project brief to determine campaign vertical
  - IF vertical specified: proceed to Step 2
  - IF no vertical specified:
    - Log warning: "No vertical specified -- using universal configuration"
    - Load universal/default configuration
    - This is NOT a HALT condition

STEP 2: LOAD VERTICAL PROFILE
  - Read: ad-verticals/[vertical].md
  - Extract the following sections:
    a. Platform priorities (ordered list with primary/secondary/tertiary)
    b. Compliance constraints (platform-specific and vertical-specific)
    c. Anti-slop rules (vertical-specific banned phrases and patterns)
    d. Platform-specific format specs:
       - Aspect ratios per platform
       - File types per platform
       - Maximum durations per platform
       - Maximum file sizes per platform
       - Sound-on/sound-off defaults
       - Text overlay requirements

STEP 3: EXTRACT PLATFORM-SPECIFIC FORMAT SPECS
  - For each target platform (Meta, TikTok, YouTube, Google, Pinterest, Snapchat):
    - Record aspect ratio requirements (16:9, 9:16, 1:1, 4:5)
    - Record duration limits
    - Record file type requirements (mp4, jpg, png, gif)
    - Record max file size
    - Record sound handling (sound-on default, sound-off default, both)
    - Record text overlay requirements (required for sound-off, optional, etc.)

STEP 4: VALIDATE COMPLETENESS
  - Verify all required sections are populated
  - Verify at least 2 target platforms identified
  - Verify compliance constraints are present for vertical
  - Log any missing or incomplete sections

STEP 5: WRITE OUTPUT
  - Write per-microskill output file with all extracted configuration
```

```yaml
vertical_source_file: "[path to vertical profile file]"

    max_duration_seconds: "[integer]"

    sound_default: "[sound_on | sound_off | both]"
    text_overlay: "[required | optional | recommended]"

    aspect_ratios: ["9:16"]

    max_file_size_mb: "[integer]"

    - "[constraint 1]"
    - "[constraint 2]"
  platform_specific:
    META: ["[constraint]"]

  banned_phrases: ["[phrase 1]", "[phrase 2]"]

  compliance_present: "[Y/N]"
  format_specs_complete: "[Y/N]"
  warnings: ["[any warnings]"]
```

- [ ] At least 2 target platforms identified with format specs

- [ ] Compliance constraints extracted for the vertical
- [ ] Anti-slop rules extracted and include cross-vertical contamination blocks
