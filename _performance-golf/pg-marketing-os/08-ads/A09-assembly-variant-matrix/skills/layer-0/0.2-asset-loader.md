# 0.2: Asset Loader

## Purpose

Load ALL production assets from the A08 Visual/Video Production `AD-ASSETS/` directory. Build a comprehensive asset catalog that enumerates every visual treatment, video asset, and audio asset organized by concept and platform with full format specifications. This catalog is the visual/video input to combinatorial expansion in Layer 1.

## Inputs

- **A08 Assets Directory:** `[project]/A08-visual-video-production/AD-ASSETS/` -- contains all produced visual and video assets from A08
- **A06 Arena Results:** For concept ID cross-referencing (winning concept IDs)

## Process

```
STEP 1: LOCATE A08 ASSETS DIRECTORY
  - Verify AD-ASSETS/ directory exists at expected path
  - IF directory does not exist: HALT -- A08 has not completed
  - IF directory exists: list ALL files and subdirectories

STEP 2: ENUMERATE ALL ASSETS
  For each file in AD-ASSETS/:
    a. Identify asset type: VISUAL (static image), VIDEO, or AUDIO
    b. Identify parent concept (C1, C2, C3, etc.)
    c. Record the A08 asset ID (original ID from A08 output)
    d. Record file path (exact, verified)
    e. Record file size (bytes and MB)
    f. Record format specifications:
       - For images: resolution (width x height), aspect ratio, file type (jpg/png/gif)
       - For videos: resolution, aspect ratio, duration (seconds), file type (mp4/mov), codec
       - For audio: duration (seconds), file type (mp3/wav/aac), sample rate
    g. Identify visual type: UGC, polished, motion_graphics, text_overlay,
       before_after, talking_head, product_demo, lifestyle, screen_recording,
       split_screen, chart_data
    h. Record platform-specific versions (if A08 produced platform variants)

STEP 3: BUILD ASSET CATALOG
  Organize all assets into structured catalog:
    Per Concept:
      - Concept ID (C1, C2, C3)
      - Visuals: [list with IDs, type, format specs, file path]
      - Videos: [list with IDs, type, format specs, duration, file path]
      - Audio: [list with IDs, format specs, duration, file path]
      - Platform-specific versions: [if any]

STEP 4: COUNT AND VERIFY
  - Count total visuals per concept
  - Count total videos per concept
  - Count total audio assets per concept
  - Verify minimum 2 visual/video treatments per concept (Gate 0 requirement)
  - Verify no missing files (all paths resolve to existing files)
  - Verify format specs recorded for every asset
  - Flag any concept with fewer than minimum assets

STEP 5: WRITE ASSET-CATALOG.yaml
  - Write structured catalog to output file
  - Include per-concept asset counts
  - Include file path and format verification status
```
