# 3.5: Matrix Validator

## Purpose

Validate the complete organized matrix before output packaging begins. Verify every variant has an ID, testing priority, file manifest with verified file paths, and platform assignment. Check for orphaned variants (in matrix but not in manifest, or vice versa). Verify distribution reasonableness. This is the Gate 3 gatekeeper.

## Inputs

- **VARIANT-ID-MAP.yaml:** From 3.1 -- all variant IDs
- **TESTING-PRIORITY.yaml:** From 3.2 -- tier assignments
- **FILE-MANIFEST.yaml:** From 3.3 -- file manifests
- **PLATFORM-GROUPS.yaml:** From 3.4 -- platform groupings
- **COHERENCE-VALIDATION-REPORT.md:** From 2.5 -- coherence results (for cross-reference)

## Process

```
STEP 1: LOAD ALL LAYER 3 OUTPUTS
  - Read all four Layer 3 output files
  - Read COHERENCE-VALIDATION-REPORT.md for cross-reference

STEP 2: VERIFY COMPLETENESS -- EVERY VARIANT HAS ALL ATTRIBUTES
  For each variant in VARIANT-ID-MAP.yaml:
    a. Has variant ID: [Y/N]
    b. Has testing priority (in TESTING-PRIORITY.yaml): [Y/N]
    c. Has file manifest (in FILE-MANIFEST.yaml): [Y/N]
    d. Has platform assignment (in PLATFORM-GROUPS.yaml): [Y/N]
    e. File manifest has all files verified on disk: [Y/N]

  If ANY attribute is missing for any variant: FAIL

STEP 3: CHECK FOR ORPHANED VARIANTS
  - Variants in VARIANT-ID-MAP but NOT in FILE-MANIFEST = orphaned (no manifest)
  - Variants in FILE-MANIFEST but NOT in VARIANT-ID-MAP = orphaned (no ID)
  - Variants in TESTING-PRIORITY but NOT in VARIANT-ID-MAP = orphaned (no ID)
  - Variants in PLATFORM-GROUPS but NOT in VARIANT-ID-MAP = orphaned (no ID)
  - Count: total orphaned variants (MUST be 0)

STEP 4: VERIFY DISTRIBUTION REASONABLENESS
  - No concept with fewer than 5 variants: [check]
  - No platform with fewer than 5 variants: [check]
  - Tier distribution roughly matches 20%/50%/30%: [check]
  - All concepts have at least one TIER_1 variant: [check]

STEP 5: CROSS-REFERENCE WITH COHERENCE RESULTS
  - Total PASS from coherence validation = total variants in matrix
  - If mismatch: variants were gained or lost between Layer 2 and Layer 3
  - Verify: no variant that was FAIL in coherence appears in the matrix
  - Verify: every variant that was PASS in coherence appears in the matrix

STEP 6: VERIFY FILE INTEGRITY
  - Count total unique files referenced across all manifests
  - Count files verified on disk vs total
  - Any missing files: FAIL
  - Verify no manifest references "expected" or "should be" paths

STEP 7: PRODUCE MANDATORY MATRIX COMPLETENESS CHECK TABLE
  Output the formatted check table from AGENT.md:

  +------------------------------------------------------------------------+
  | METRIC                         | VALUE  | STATUS                       |
  +------------------------------------------------------------------------+
  | Total variants in matrix       | [X]    | PASS if >= 30 / FAIL < 30    |
  | All have variant IDs           | [Y/N]  | MUST be Y                    |
  | All have testing priority      | [Y/N]  | MUST be Y                    |
  | All have file manifest         | [Y/N]  | MUST be Y                    |
  | All files verified on disk     | [Y/N]  | MUST be Y                    |
  | All have platform assignment   | [Y/N]  | MUST be Y                    |
  | Orphaned variants              | [X]    | MUST be 0                    |
  [... full table as in AGENT.md]

STEP 8: DETERMINE GATE 3 STATUS
  Gate 3 PASSES if and only if:
    - Total variants >= 30
    - All variants have IDs
    - All variants have testing priority
    - All variants have file manifests
    - All files verified on disk
    - All variants have platform assignments
    - Orphaned variants = 0
    - No concept with < 5 variants (unless justified)
    - Variant count matches coherence PASS count

  Gate 3 FAILS if ANY of the above is not met.

STEP 9: WRITE MATRIX-VALIDATION-REPORT.md
```
