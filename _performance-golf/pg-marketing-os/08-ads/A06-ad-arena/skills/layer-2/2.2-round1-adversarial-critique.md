# 2.2: Round 1 Adversarial Critique

## Purpose
The dedicated adversarial Critic evaluates ALL persona outputs for each concept. If a material weakness exists, identifies the SINGLE most impactful one per persona evaluation per concept. Maps weakness to a specific criterion from the 7 ad-specific judging criteria. Provides actionable fix direction. Must cite evidence from the evaluation. If no element materially underperforms, reports `no_material_weakness` with a `strength_note`.

## Inputs
- 2.1-round1-persona-evaluation output (all 7 persona evaluations per concept)
- 7 ad-specific judging criteria (from AGENT.md)

## Process
```
STEP 1: Load all Round 1 persona evaluations
  - READ: layer-2-outputs/2.1-round1-persona-evaluation.md

STEP 2: FOR EACH concept:
  FOR EACH persona evaluation (7 total):

    A. READ the persona evaluation BLIND
      - The Critic has NO generation context
      - The Critic evaluates the evaluation itself against the 7 criteria

    B. ASSESS whether a material weakness exists
      - NOT a laundry list of issues
      - If a material weakness exists, identify the SINGLE most impactful one
      - FORCE prioritization -- what single fix would improve this most?
      - If no element materially underperforms, report no_material_weakness

    C. MAP weakness to specific criterion
      - Which of the 7 ad-specific criteria does this weakness affect?
      - Scroll-Stop Power | Visual-Copy Coherence | Mechanism Clarity |
        Platform Nativeness | Proof Integration | CTA Strength | Memorability

    D. PROVIDE actionable fix direction
      - NOT "make it better" or "needs improvement"
      - SPECIFIC: "Replace the text overlay opening with a face-to-camera
        reaction shot to increase scroll-stop power from 6 to 8"
      - CITE evidence: Reference the specific element in the evaluation

    E. ASSESS severity (1-10)
      - How much is this weakness dragging down the weighted score?

CRITICAL RULES:
  - The Critic is DEDICATED and ADVERSARIAL -- not self-critique
  - Critic receives outputs BLIND (no evaluation context)
  - At most ONE weakness per evaluation (forces prioritization). If no element materially underperforms, report no_material_weakness.
  - Fix direction must be ACTIONABLE and SPECIFIC (when weakness found)
  - Consensus among personas does NOT eliminate the need for critique
  - Do NOT manufacture weaknesses to justify the Critic role -- honest calibrated assessment is the goal
```
