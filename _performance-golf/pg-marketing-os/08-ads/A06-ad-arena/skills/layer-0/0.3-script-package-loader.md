# 0.3: Script Package Loader

## Purpose
Load A04 SCRIPT-PACKAGE.md. For each selected hook from A02, load the corresponding script architecture: framework (PAS, AIDA, BAB, etc.), modular sections (hook, setup, mechanism, proof, CTA), word count, and platform constraints.

## Inputs
- A04 SCRIPT-PACKAGE.md: `[project]/A04-script-architecture/SCRIPT-PACKAGE.md`
- Selected hooks from 0.2-hook-matrix-loader output

## Process
```
STEP 1: Load SCRIPT-PACKAGE.md
  - READ full file
  - IDENTIFY all script architectures

STEP 2: For EACH selected hook (from 0.2 output):
  - MATCH hook to its corresponding script architecture
  - IF no script exists for a hook: Flag as INCOMPLETE
  - IF script exists: Extract full architecture

STEP 3: For EACH matched script, extract:
  - Script ID (S-001, S-002, etc.)
  - Associated Hook ID (H-XXX)
  - Framework type (PAS, AIDA, BAB, Hook-Body-CTA, Problem-Agitate-Mechanism-Solution, etc.)
  - Modular sections with descriptions:
    - Hook section (opening)
    - Setup section (problem/pain)
    - Mechanism section (how/why)
    - Proof section (evidence)
    - CTA section (call to action)
  - Target word count
  - Target ad duration (seconds)
  - Platform constraints (sound-on/off, text overlay limits, etc.)

STEP 4: Validate hook-to-script mapping completeness
  - Every selected hook should have a script
  - Flag any gaps

STEP 5: Compile script inventory for concept assembly
```
