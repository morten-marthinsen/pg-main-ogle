# 4.1: Arena Results Assembler

## Purpose
Assemble AD-ARENA-RESULTS.md with all required sections (12 mandatory). Include complete scoring data, persona evaluations for winning concepts, hybrid compositions, human selection record, and downstream instructions. This is the primary output file for the entire A06 Ad Arena skill. Minimum 50KB.

## Inputs
- 1.1-concept-bundler output (concept bundles)
- 1.3-concept-summary-generator output (concept summaries)
- 2.4-round1-scoring output (Round 1 scores)
- 2.5-round1-learning-brief output (Round 1 Analytical Brief)
- 2.9-round2-scoring output (Round 2 scores)
- 2.10-round2-cumulative-learning-brief output (Cumulative Analytical Brief)
- 2.11-round3-persona-evaluation output (FULL Round 2 (FINAL) evaluations)
- 2.14-round3-final-scoring output (FINAL scores)
- 2.15-round3-final-ranking output (Campaign Analytical Brief)
- 2.5.1-improvement-decomposition output (decomposed improvements)
- 2.5.2-hybrid-concept-generation output (hybrids)
- 2.5.3-hybrid-validation output (validated hybrids)
- 3.2-human-selection-capture output (HUMAN_SELECTION.yaml)
- execution-log.md (execution timeline)

## Process
```
STEP 1: Load ALL required inputs
  - READ all files listed above
  - VERIFY: All required data present
  - VERIFY: HUMAN_SELECTION.yaml exists (human selection received)

STEP 2: Assemble Section 1 -- Executive Summary
  - Concepts evaluated: [count from Layer 1]
  - Rounds completed: 3
  - Concepts advancing: [count from HUMAN_SELECTION.yaml]
  - Quality assessment: [strong/adequate/weak based on scores]
  - Key finding: [1-2 sentences from Campaign Analytical Brief]

STEP 3: Assemble Section 2 -- Concept Registry
  - FOR EACH concept: complete description (hook + script + visual)
  - Include: concept IDs, platforms, ad lengths, frameworks

STEP 4: Assemble Section 3 -- Round 1 Results
  - Per-concept: all 7 persona scores, winning persona
  - Analytical Brief key findings
  - Cross-concept ranking after Round 1

STEP 5: Assemble Section 4 -- Round 2 Results
  - Per-concept: all 7 persona scores
  - Improvement tracking from R1 (deltas)
  - Cumulative Analytical Brief key findings

STEP 6: Assemble Section 5 -- Round 2 (FINAL) Results (FINAL)
  - Per-concept: all 7 persona FINAL scores
  - FULL evaluation text for winning concepts (from 2.11 output)
  - Cross-concept final ranking
  - Progression tracking: R1 -> R2 -> R3

STEP 7: Assemble Section 6 -- Adversarial Critique Summary
  - Per-concept: weaknesses identified per round
  - Which weaknesses persisted across rounds
  - Most common critique themes across all concepts

STEP 8: Assemble Section 7 -- Synthesis & Hybrid Results
  - Per-concept: hybrid compositions from Layer 2.5
  - Hybrid scores and coherence validation results
  - Comparison: best pure vs best hybrid per concept

STEP 9: Assemble Section 8 -- Human Selection Record
  - Which concepts selected, which versions (pure/hybrid)
  - Human notes and instructions
  - Rejected concepts with reasons

STEP 10: Assemble Section 9 -- Scoring Data (Complete)
  - Full scoring matrix: every persona x every criterion x every concept x every round
  - Weighted totals, rankings, progression data
  - This section is DATA-HEAVY -- include all numbers

STEP 11: Assemble Section 10 -- Campaign Analytical Brief
  - What hook types scored highest in Arena evaluation
  - What visual treatments were most effective
  - What criteria were hardest to meet
  - Which persona lens provided most unique insights
  - Recommendations for future campaign creative

STEP 12: Assemble Section 11 -- Downstream Handoff Summary
  - Selected concepts packaged for A07
  - Improvement recommendations incorporated
  - Word count limits and platform constraints confirmed
  - Variant generation guidance

STEP 13: Assemble Section 12 -- Execution Log
  - Complete execution timeline
  - All gate statuses
  - Model assignments used
  - MC-CHECK records
  - Context zone tracking

STEP 14: VERIFY file size
  - Measure assembled file
  - IF < 50KB: File is a summary, not complete results -- ADD MORE DATA
  - Minimum 50KB ensures comprehensive coverage
  - Use chunked writing (15-20 writes) if needed -- LLMs have ~30KB output limits

STEP 15: Final verification
  - All 12 sections present
  - No placeholder text remaining
  - Scoring data complete (not summarized)
  - Human selection accurately reflected
```
