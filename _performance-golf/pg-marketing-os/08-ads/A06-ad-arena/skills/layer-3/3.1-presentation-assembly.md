# 3.1: Presentation Assembly

## Purpose
Assemble all candidates for human review. Organize by concept with: (a) concept summary (hook + script + visual), (b) best persona evaluation per concept with score, (c) hybrid improvements per concept with scores, (d) transparent scoring breakdown, (e) Arena recommendation with rationale. Present concepts ranked by best available score (pure or hybrid).

## Inputs
- 1.3-concept-summary-generator output (original concept summaries)
- 2.14-round3-final-scoring output (FINAL scores per persona per concept)
- 2.15-round3-final-ranking output (cross-concept ranking + Campaign Analytical Brief)
- 2.5.3-hybrid-validation output (validated hybrids with scores)
- HUMAN_SELECTION.yaml from Layer 3.2 (if revision cycle -- otherwise not yet created)

## Process
```
STEP 1: Load all required inputs
  - READ: layer-1-outputs/1.3-concept-summary-generator.md
  - READ: layer-2-outputs/2.14-round3-final-scoring.md
  - READ: layer-2-outputs/2.15-round3-final-ranking.md
  - READ: layer-2.5-outputs/2.5.3-hybrid-validation.md
  - VERIFY: All concepts have Round 2 (FINAL) scores and at least validated hybrids

STEP 2: FOR EACH concept (ranked by best available score):
  A. COMPILE concept summary
    - Hook: verbatim hook text
    - Script Framework: framework name from A04
    - Visual Treatment: treatment type from A05
    - Platform: target platform
    - Ad Length: seconds

  B. COMPILE scoring summary table
    - FOR EACH of 7 criteria:
      - Best Pure score (which persona) from Round 2 (FINAL)
      - Best Hybrid score (which hybrid) from Layer 2.5
      - Arena Recommendation: pure or hybrid (whichever scored higher)
    - Weighted Total: best pure vs best hybrid

  C. COMPILE key improvements identified
    - Top 3 most impactful improvements from Arena process
    - Source: which round/persona/hybrid identified them

  D. WRITE recommendation
    - 2-3 sentence recommendation with rationale
    - Whether to use pure Round 2 (FINAL) version or hybrid version
    - Any caveats or conditions

STEP 3: Compile cross-concept ranking table
  - Rank all concepts by best available score (pure or hybrid)
  - Include: concept ID, best score, recommended version, key strength

STEP 4: Write instructions header for human
  - Select concepts as-is
  - Select with hybrid improvements incorporated
  - Request modifications before advancing
  - Reject concepts with notes
  - Request revision (maximum 2 revision cycles)

STEP 5: Verify presentation completeness
  - Every concept present with all sections
  - Scoring tables have all 7 criteria
  - Recommendations present for all concepts
  - Cross-concept ranking complete
```
