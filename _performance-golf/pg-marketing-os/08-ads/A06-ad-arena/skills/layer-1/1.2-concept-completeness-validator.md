# 1.2: Concept Completeness Validator

## Purpose
Validate each concept bundle has all required elements: hook text (verbatim from A02), script architecture (framework, modular sections, word count, platform constraints), visual direction (treatment type, shot list, mood, duration). Flag any concept with missing elements as INCOMPLETE.

## Inputs
- 1.1-concept-bundler output (concept bundles)

## Process
```
STEP 1: Load concept bundles from 1.1 output
  - READ: layer-1-outputs/1.1-concept-bundler.md

STEP 2: For EACH concept bundle, validate HOOK element:
  - Hook text present and non-empty: [Y/N]
  - Hook text is verbatim (not summarized): [verify against 0.2 output]
  - Hook type classification present: [Y/N]
  - Hook angle attribution present: [Y/N]
  - IF any missing: Mark hook as INCOMPLETE

STEP 3: For EACH concept bundle, validate SCRIPT element:
  - Framework type present: [Y/N]
  - Modular sections present (hook, setup, mechanism, proof, CTA): [Y/N]
  - Word count target present: [Y/N]
  - Duration target present: [Y/N]
  - Platform constraints present: [Y/N]
  - IF any missing: Mark script as INCOMPLETE

STEP 4: For EACH concept bundle, validate VISUAL element:
  - Treatment type present: [Y/N]
  - Shot list present with specific shots: [Y/N]
  - Mood/visual tone present: [Y/N]
  - Aspect ratio present: [Y/N]
  - Production level present: [Y/N]
  - IF any missing: Mark visual as INCOMPLETE

STEP 5: For EACH concept, validate INTEGRATION:
  - Hook matches the script's opening section: [Y/N]
  - Visual treatment matches the script's format: [Y/N]
  - Platform consistency (all elements target same platform): [Y/N]

STEP 6: Compile validation results
  - COMPLETE concepts: [list]
  - INCOMPLETE concepts: [list with specific missing elements]
  - IF any concepts INCOMPLETE: Flag for resolution before Arena
```
