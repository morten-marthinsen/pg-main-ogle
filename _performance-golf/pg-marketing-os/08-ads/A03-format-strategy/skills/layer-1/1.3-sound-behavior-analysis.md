# 1.3: Sound Behavior Analysis

## Purpose

Classify every selected hook by sound dependency (AUDIO_DEPENDENT, VISUAL_FIRST, DUAL_MODE) and map each classification to platform sound profiles. This analysis determines which hooks need text overlay backup, which can rely on audio, and which platforms are optimal for each hook based on sound behavior.

## Inputs

- `layer-0-outputs/0.3-hook-angle-matrix.md` — 8-10 selected hooks
- Platform sound behavior matrix (from AGENT.md Section: Sound Behavior Matrix)

## Process

1. **FOR EACH selected hook, CLASSIFY sound dependency:**

   **AUDIO_DEPENDENT:**
   - Hook's power comes from spoken words, tone, or sound effects
   - Cannot work sound-off without significant adaptation
   - Examples: "Doctor begs: stop eating this...", voiceover hooks, storytelling hooks, question hooks that rely on conversational tone
   - Indicators: rhetorical questions, vocal inflection, dialogue, sound-based pattern interrupts

   **VISUAL_FIRST:**
   - Hook's power comes from visual impact
   - Works sound-off natively
   - Examples: Before/after, demonstration, surprising visual, text-on-screen, product reveal, pattern interrupt
   - Indicators: visual transformations, product demos, text-driven hooks, data visualizations

   **DUAL_MODE:**
   - Hook works both sound-on and sound-off with minor adaptation
   - Examples: Text overlay hooks with optional voiceover, UGC with subtitles, hooks that work visually but are enhanced by audio
   - Indicators: hooks with strong visual component AND audio component, neither fully dependent

2. **FOR EACH hook, DETERMINE sound-off adaptation strategy:**
   - If AUDIO_DEPENDENT: What's the text overlay plan?
   - What key phrases must be displayed on-screen?
   - Does adaptation weaken the hook? By how much?
   - Can the hook survive sound-off, or is it better to avoid sound-off platforms?

3. **MAP each hook to platform sound profiles:**
   ```
   PLATFORM SOUND BEHAVIOR:
   - Meta Feed: OFF (85%) → AUDIO_DEPENDENT hooks need text backup
   - Meta Stories/Reels: ON (60%+) → Audio hooks viable but design sound-off version
   - TikTok: ON (80%+) → Audio hooks are POWERFUL here
   - YouTube Pre-Roll: ON (95%+) → Full audio dependency acceptable
   - YouTube Shorts: ON (70%+) → Audio viable, optimize for visual-first
   - GDN: N/A (static) → Pure visual + text only
   ```

4. **FOR EACH hook, DOCUMENT platform implications:**
   - Which platforms reward this hook's sound profile?
   - Which platforms penalize or require adaptation?
   - What specific adaptations are needed per platform?

5. **IDENTIFY hook-platform sound conflicts:**
   - Example: Audio-dependent hook assigned to Meta feed = CONFLICT (requires significant text overlay work OR platform should be avoided)
   - Example: Visual-first hook across all platforms = NO CONFLICT (universally compatible)
