# 1.5: Layer 1 Validation

## Purpose

Validate that Layer 1 platform analysis is complete, all hooks have been assessed across all platforms, no hook is unworkable across all platforms, and all required outputs meet quality standards. This is the GATE 1 determination microskill.

## Inputs

All Layer 1 outputs:
- `layer-1-outputs/1.1-platform-priority.md`
- `layer-1-outputs/1.2-hook-platform-compatibility.md`
- `layer-1-outputs/1.3-sound-behavior-analysis.md`
- `layer-1-outputs/1.4-compliance-precheck.md`

## Process

### Step 1: Verify All Hooks Assessed

1. **CHECK hook coverage:**
   - All 8-10 selected hooks appear in ALL four Layer 1 outputs
   - No hook was skipped in any analysis
   - Hook IDs consistent across all outputs

IF any hook missing from any output → FAIL

### Step 2: Verify Platform Coverage

1. **CHECK platform coverage:**
   - All target platforms analyzed in 1.1 (platform priority)
   - All platforms appear in 1.2 compatibility matrix
   - All platforms appear in 1.3 sound mapping
   - All platforms appear in 1.4 compliance check

IF any platform missing from any output → FAIL

### Step 3: Verify Hook Viability

1. **FOR EACH hook, CHECK platform assignments:**
   - Count OPTIMAL assignments
   - Count VIABLE assignments
   - Count POOR assignments
   - Count BLOCKED assignments

2. **FLAG unworkable hooks:**
   - IF hook has zero OPTIMAL and zero VIABLE assignments across ALL platforms → CRITICAL FLAG
   - Hook cannot proceed to Layer 2
   - Escalate to human for A02 re-evaluation or hook replacement

3. **CALCULATE overall compatibility health:**
   - Total hook-platform pairs: [hooks × platforms]
   - Percentage OPTIMAL + VIABLE
   - Percentage POOR + BLOCKED
   - IF >30% POOR/BLOCKED → WARN (hooks may need adaptation)

### Step 4: Verify Compliance Status

1. **COUNT compliance flags:**
   - CLEAR hooks
   - CAUTION hooks (with guidance)
   - RESTRICTED hooks (with guidance)
   - BLOCKED hooks (with guidance or escalation)

2. **VERIFY guidance provided:**
   - All CAUTION/RESTRICTED/BLOCKED hooks have A04 guidance
   - All trigger phrases identified
   - All rephrase options or disclaimer requirements documented

### Step 5: Verify Data Quality

1. **CHECK platform priority (1.1):**
   - All platforms have numerical priority scores
   - All rationales cite specific A01 data (not generic)
   - Strategic implications documented

2. **CHECK compatibility matrix (1.2):**
   - Every cell has compatibility level AND rationale
   - Rationales are specific (not "works well")
   - Statistics calculated

3. **CHECK sound behavior (1.3):**
   - All hooks classified (AUDIO_DEPENDENT / VISUAL_FIRST / DUAL_MODE)
   - Sound-off adaptation strategies documented
   - Platform sound mapping complete

4. **CHECK compliance (1.4):**
   - All hook-platform pairs have compliance flags
   - Specific guidance provided for flagged hooks

### Step 6: Gate Determination

```
IF all verifications PASS:
  GATE 1 = PASS
  Proceed to Layer 2 (Format Mapping)
ELSE:
  GATE 1 = FAIL
  HALT execution
  Document failure reasons
  Return control to human
```
