# 2.5: Element Attribution

**Version:** 1.0
**Layer:** 2 (Prediction vs Reality Analysis)
**Type:** Analysis
**Model:** opus
**Dependencies:** 0.2 variant-matrix-loader, 1.2 per-variant-calculator, 1.4 classification-engine
**Output:** layer-2-outputs/2.5-element-attribution.md

---

## Purpose

Decompose variant performance into ELEMENT-LEVEL attribution. Using the variant matrix from A09, compare variants that share elements to isolate each element's contribution to performance. (a) HOOK ATTRIBUTION: Compare variants with same body+CTA+visual but different hooks. (b) VISUAL ATTRIBUTION: Compare variants with same hook+body+CTA but different visuals. (c) CTA ATTRIBUTION: Compare variants with same hook+body+visual but different CTAs. (d) BODY ATTRIBUTION: Compare variants with same hook+CTA+visual but different bodies. This produces a per-element performance ranking that tells future campaigns which SPECIFIC elements to prioritize.

## Process

### Step 1: Build Element Isolation Groups

```
FROM variant matrix, BUILD isolation groups:

  An ISOLATION GROUP is a set of variants that share ALL elements EXCEPT ONE.
  The differing element's performance can be isolated by comparing variants in the group.

  HOOK ISOLATION GROUPS:
    For each unique combination of (body_id, visual_id, CTA_id):
      Find all variants that share this combination but have different hook_ids
      IF group has >= 2 variants with sufficient data:
        --> This is a valid hook isolation group
        --> Different hooks, same everything else = hook attribution

  VISUAL ISOLATION GROUPS:
    For each unique combination of (hook_id, body_id, CTA_id):
      Find all variants sharing this combination with different visual_ids
      IF group has >= 2 variants with sufficient data:
        --> Valid visual isolation group

  CTA ISOLATION GROUPS:
    For each unique combination of (hook_id, body_id, visual_id):
      Find all variants sharing this combination with different CTA_ids
      IF group has >= 2 variants with sufficient data:
        --> Valid CTA isolation group

  BODY ISOLATION GROUPS:
    For each unique combination of (hook_id, visual_id, CTA_id):
      Find all variants sharing this combination with different body_ids
      IF group has >= 2 variants with sufficient data:
        --> Valid body isolation group

DOCUMENT:
  hook_isolation_groups_found: [count]
  visual_isolation_groups_found: [count]
  CTA_isolation_groups_found: [count]
  body_isolation_groups_found: [count]

NOTE: If the variant matrix was NOT designed as a full factorial experiment,
some isolation groups may not exist. This is expected. Document which
element attributions can and cannot be performed.
```

### Step 2: Hook Attribution Analysis

```
FOR EACH hook isolation group:
  COMPARE variant performance across different hooks (holding body, visual, CTA constant):

  PRODUCE:
    group_id: [body_X + visual_Y + CTA_Z]
    variants_in_group: [list of variant_ids]
    hook_comparison:
      | Hook ID | Hook Type | Hook Rate | CPA | ROAS | CTR | Classification |
      | [id]    | [type]    | [X]%      | $[X]| [X]x | [X]%| WINNER/LOSER   |

    best_hook: [hook_id with lowest CPA / highest ROAS]
    worst_hook: [hook_id with highest CPA / lowest ROAS]
    performance_delta: [best vs worst CPA difference]
    hook_contribution: "Hook change from [worst] to [best] improved CPA by [X]%"

AGGREGATE across all hook isolation groups:
  For each unique hook:
    avg_relative_performance: mean of (hook_CPA / group_avg_CPA) across all groups it appears in
    win_count: how many groups this hook was the best performer
    lose_count: how many groups this hook was the worst performer

  RANK hooks by relative performance (best to worst)
```

### Step 3: Visual Attribution Analysis

```
FOR EACH visual isolation group:
  [Same comparison logic as Step 2 but for visuals]

  PRODUCE:
    group_id: [hook_X + body_Y + CTA_Z]
    visual_comparison table
    best_visual, worst_visual, performance_delta

AGGREGATE across all visual isolation groups:
  Rank visuals by relative performance
  Identify visual style patterns:
    - Does UGC consistently outperform polished?
    - Does talking head outperform text overlay?
    - Are there visual-hook synergies (certain visuals amplify certain hooks)?
```

### Step 4: CTA Attribution Analysis

```
FOR EACH CTA isolation group:
  [Same comparison logic as Step 2 but for CTAs]

  FOCUS ON:
    - Conversion rate (CTAs most directly affect post-hook conversion)
    - CPA (ultimate effectiveness)
    - CTA language patterns (does urgency CTA outperform benefit CTA?)

AGGREGATE across all CTA isolation groups:
  Rank CTAs by conversion rate and CPA
```

### Step 5: Body Attribution Analysis

```
FOR EACH body isolation group:
  [Same comparison logic as Step 2 but for body scripts]

  FOCUS ON:
    - ThruPlay rate (body scripts affect whether people watch to end)
    - Conversion rate (body content affects persuasion)
    - CPA (ultimate effectiveness)

AGGREGATE across all body isolation groups:
  Rank bodies by relative performance
```

### Step 6: Identify Winning and Losing Element Combinations

```
BEYOND individual element attribution, analyze COMBINATIONS:

  TOP 5 ELEMENT COMBINATIONS (by CPA):
    | Rank | Hook | Visual | CTA | Body | CPA | ROAS | Classification |

  BOTTOM 5 ELEMENT COMBINATIONS (by CPA):
    | Rank | Hook | Visual | CTA | Body | CPA | ROAS | Classification |

  SYNERGY DETECTION:
    Are there element pairs that perform significantly better TOGETHER
    than their individual performance would predict?
    e.g., "UGC visual + testimonial hook = 2.3x better CPA than either element alone"

  CONFLICT DETECTION:
    Are there element pairs that perform significantly worse together?
    e.g., "Polished visual + UGC testimonial hook = worse than either element alone"
```

### Step 7: Attribution Limitation Documentation

```
DOCUMENT what attribution analysis could and could NOT be performed:

  POSSIBLE ATTRIBUTIONS:
    - [element type]: [X] isolation groups found, [Y] elements compared

  IMPOSSIBLE ATTRIBUTIONS:
    - [element type]: Insufficient isolation groups (fewer than 2 variants share all other elements)
    - Reason: Variant matrix did not include sufficient combinations for full factorial analysis

  RECOMMENDATIONS FOR NEXT CAMPAIGN:
    - If hook attribution was weak: "Design variant matrix with more hook-only swaps"
    - If visual attribution was weak: "Design variant matrix with more visual-only swaps"
    - General: "A09 should design variant matrix as near-factorial to enable A12 attribution"
```

## Quality Checks

1. All four element types (hook, visual, CTA, body) analyzed (or limitations documented)
2. Isolation groups correctly identified (variants share ALL elements except one)
3. Only sufficient-data variants used in comparisons
4. Per-element rankings produced with relative performance scores
5. Top 5 and bottom 5 element combinations identified
6. Synergies and conflicts detected and documented
7. Attribution limitations explicitly documented
8. Next campaign design recommendations provided for improving attribution
9. Minimum 8KB output (element attribution is the most detailed analysis)
