# 2.2: Divergence Analysis

**Version:** 1.0
**Layer:** 2 (Prediction vs Reality Analysis)
**Type:** Analysis
**Model:** opus
**Dependencies:** 2.1 prediction-comparison output
**Output:** layer-2-outputs/2.2-divergence-analysis.md

---

## Purpose

For every variant where prediction diverged from reality (OPTIMISTIC, PESSIMISTIC, or WILDLY_WRONG): generate a hypothesis for WHY the prediction was wrong. This is the causal reasoning layer -- it is not enough to know that predictions were wrong; the engine must understand WHY they were wrong so the predictive model can be calibrated. Common divergence patterns include hook type platform mismatch, landing page bottlenecks, visual style surprises, creative fatigue speed, platform algorithm preferences, and audience targeting effects on creative.

## Process

### Step 1: Filter Divergent Predictions

```
FROM 2.1 prediction comparison output:
  SELECT all variants WHERE prediction_classification IN (OPTIMISTIC, PESSIMISTIC, WILDLY_WRONG)

  GROUP BY:
    optimistic_variants = [list]  # predicted better than actual
    pessimistic_variants = [list]  # predicted worse than actual
    wildly_wrong_variants = [list]  # completely missed

IF no divergent predictions exist (all ACCURATE):
  --> Produce minimal output documenting this exceptional result
  --> Note: 100% accuracy is unlikely -- verify prediction comparison was thorough
```

### Step 2: Generate Divergence Hypotheses

```
FOR EACH divergent variant:

  ANALYZE USING COMMON DIVERGENCE PATTERNS:

  (a) HOOK TYPE PLATFORM MISMATCH:
    Check: Did the hook type perform differently on this platform than taxonomy benchmarks suggest?
    Evidence: Compare hook_type's actual hook_rate on this platform vs taxonomy benchmark
    Hypothesis format: "[hook_type] achieved [X]% hook rate on [platform], vs [Y]% taxonomy benchmark.
      Platform-specific behavior likely explains divergence."
    Applicability: Check if other variants with same hook type showed similar pattern

  (b) LANDING PAGE / OFFER BOTTLENECK:
    Check: Did variants with high hook rates but low conversion rates suggest the creative was
    effective at stopping the scroll but the landing page or offer didn't convert?
    Evidence: Hook rate vs conversion rate comparison. High hook + low conversion = post-click issue.
    Hypothesis format: "Variant had [X]% hook rate (above average) but $[Y] CPA (above target).
      Post-click conversion rate of [Z]% suggests landing page/offer bottleneck, not creative failure."
    Applicability: Compare conversion rates across variants to isolate creative vs funnel issues

  (c) VISUAL STYLE SURPRISE:
    Check: Did A10 predict polished production would outperform UGC (or vice versa) and the
    opposite happened?
    Evidence: Compare visual_style of predicted winners vs actual winners
    Hypothesis format: "A10 scored [visual_style] variants higher, but [other_style] variants
      achieved [X]% better CPA. [other_style] likely benefited from [reason: authenticity trust /
      pattern interrupt / platform algorithm preference for native content]."
    Applicability: Check if pattern holds across multiple variants

  (d) CREATIVE FATIGUE FASTER THAN PREDICTED:
    Check: Did the variant perform well initially but fatigue faster than expected?
    Evidence: Check time_series for inflection points and fatigue classification
    Hypothesis format: "Variant performed at $[X] CPA for first [Y] days but fatigued to $[Z] CPA.
      Overall performance was LOSER despite strong start. A10 scored based on creative quality
      without modeling fatigue speed."
    Applicability: Compare fatigue curves of predicted winners vs actual performance

  (e) PLATFORM ALGORITHM PREFERENCE:
    Check: Did the platform's algorithm preferentially distribute one format/style?
    Evidence: Compare CPM and impression distribution across variants on the same platform
    Hypothesis format: "Platform algorithm gave [variant] [X]% more impressions at [Y]% lower CPM,
      suggesting algorithmic preference for [format/style]. A10 doesn't model platform distribution bias."
    Applicability: Check CPM variance across variants

  (f) AUDIENCE TARGETING EFFECT:
    Check: Did audience targeting interact with creative in unexpected ways?
    Evidence: If multiple ad sets with different audiences ran same variants, compare performance
    Hypothesis format: "Same creative performed differently across ad sets: $[X] CPA in [audience_A]
      vs $[Y] CPA in [audience_B]. Audience-creative interaction explains divergence."
    Applicability: Only applicable if variant ran in multiple ad sets

  SELECT the 1-3 MOST LIKELY hypotheses for each divergent variant.
  RANK hypotheses by evidence strength (STRONG / MODERATE / WEAK).
```

### Step 3: Cross-Variant Pattern Detection

```
ACROSS all divergent variants:
  DETECT PATTERNS:
    Are OPTIMISTIC divergences clustered by:
      - hook_type? (certain hook types consistently overestimated)
      - visual_style? (certain visual styles consistently overestimated)
      - platform? (predictions less accurate on certain platforms)
      - format? (predictions less accurate for certain ad formats)

    Are PESSIMISTIC divergences clustered by:
      - Same dimensions as above

    Are WILDLY_WRONG predictions clustered?
      - If yes, this reveals a systematic blind spot in the scoring model

  DOCUMENT pattern clusters with counts:
    "[X] of [Y] optimistic divergences involved [hook_type/visual_style/platform]"
```

### Step 4: Produce Divergence Report

```
FOR EACH divergent variant, produce:
  variant_id: [id]
  prediction_classification: OPTIMISTIC / PESSIMISTIC / WILDLY_WRONG
  predicted_tier: [A10 tier]
  actual_tier: [actual classification]
  key_metrics_comparison:
    predicted_CPA: $[X] | actual_CPA: $[Y] | delta: [Z]%
    predicted_hook_rate: [X]% | actual_hook_rate: [Y]% | delta: [Z]%
  hypotheses:
    - hypothesis_1:
        pattern: [a-f label]
        explanation: [full hypothesis text]
        evidence: [specific data supporting this]
        evidence_strength: STRONG / MODERATE / WEAK
    - hypothesis_2: [if applicable]
  variant_elements:
    hook_type: [type]
    visual_style: [style]
    platform: [platform]
    format: [format]
```

## Quality Checks

1. Every divergent variant has at least one hypothesis generated
2. Hypotheses use specific data, not generic speculation
3. Evidence strength rated for each hypothesis
4. Cross-variant patterns detected and documented
5. Systematic blind spots identified (if applicable)
6. Hypotheses reference specific creative elements (hook type, visual style) not just "the ad"
7. At least 3 of the 6 common divergence patterns investigated per variant
8. If no divergent predictions, document and verify prediction comparison thoroughness
