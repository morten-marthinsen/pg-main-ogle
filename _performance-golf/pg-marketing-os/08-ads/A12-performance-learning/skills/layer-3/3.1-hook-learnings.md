# 3.1: Hook Learnings

**Version:** 1.0
**Layer:** 3 (Learning Extraction)
**Type:** Learning Extraction
**Model:** opus
**Dependencies:** Layer 2 outputs (GATE_2 must be PASS), 2.4 hook-type-performance
**Output:** layer-3-outputs/3.1-hook-learnings.md

---

## Purpose

Extract actionable learnings about hook performance. Which of the 32 hook types performed best in THIS campaign? How does this compare to taxonomy benchmarks? Which hook types should be prioritized in the next campaign? Which should be deprioritized? WHY did winning hooks win (specific hypotheses: timeframe specificity? emotional trigger? curiosity gap strength? authority signal?)? Generate specific hook recommendations for next campaign with confidence levels. Every learning must follow the WHAT/WHY/HOW/CONFIDENCE/SCOPE/PROPAGATION_TARGET format.

## Process

### Step 1: Verify Gate 2 Passed

```
READ checkpoints/LAYER_2_COMPLETE.yaml
IF NOT EXISTS:
  --> HALT: "GATE_2 not passed. Cannot begin Layer 3."
```

### Step 2: Identify Hook Performance Patterns

```
FROM 2.4 hook-type-performance output:

  TOP PERFORMING HOOK TYPES (by CPA):
    List hook types ranked by CPA performance
    For each: note sample size, confidence level, platform performance

  BOTTOM PERFORMING HOOK TYPES (by CPA):
    List worst-performing hook types
    For each: note sample size, WHY it failed

  BENCHMARK OUTPERFORMERS:
    Hook types that exceeded AD-HOOK-TAXONOMY.md benchmarks
    Quantify outperformance magnitude

  BENCHMARK UNDERPERFORMERS:
    Hook types that fell below taxonomy benchmarks
    Quantify underperformance magnitude

FROM 2.5 element-attribution output:
  Hook attribution from isolation groups (hooks that performed well
  even when controlling for other elements)
```

### Step 3: Generate Causal Hypotheses for Hook Performance

```
FOR EACH significant hook performance finding:

  ANALYZE WHY the hook type performed the way it did:

  Hook performance drivers to investigate:
    1. TIMEFRAME SPECIFICITY: Did hooks with specific timeframes ("in 3 weeks",
       "30-day") outperform vague timeframes ("quickly", "soon")?
    2. EMOTIONAL TRIGGER TYPE: Did fear-based hooks outperform curiosity-based?
       Did social proof hooks outperform authority hooks?
    3. CURIOSITY GAP STRENGTH: Did hooks with stronger curiosity gaps (more
       specific tease + higher information asymmetry) perform better?
    4. AUTHORITY SIGNAL: Did hooks from credentialed sources (Dr., expert)
       outperform non-authority hooks?
    5. UGC vs SCRIPTED: Did UGC-style delivery outperform scripted delivery
       for the same hook type?
    6. PLATFORM ALIGNMENT: Did hook types designed for specific platforms
       outperform generic hooks on those platforms?
    7. HOOK LENGTH: Did shorter hooks (< 3 seconds to key claim) outperform
       longer setups?
    8. CATEGORY PATTERNS: Did entire hook categories (A-J) show consistent
       patterns?

  FOR EACH hypothesis:
    - State the hypothesis clearly
    - Cite specific data supporting it
    - Rate evidence strength (STRONG / MODERATE / WEAK)
```

### Step 4: Extract Learnings in Required Format

```
FOR EACH actionable learning:

  FORMAT (MANDATORY -- all 6 fields required):

  ### Learning H-[ID]: [Title]

  **WHAT:** [Specific observation with data]
  Example: "UGC testimonial hooks (D3) achieved a 34% average hook rate vs 18%
  for curiosity hooks (A2) across all Meta placements. D3 also had 1.4x better
  CPA ($23 vs $32). N=12 D3 variants, N=15 A2 variants, all with 3x+ AOV spend."

  **WHY:** [Causal hypothesis]
  Example: "UGC testimonials bypass the 'ad filter' -- users process them as
  social content rather than advertising. The specific timeframe claims ('in 3
  weeks') in D3 hooks created concrete expectation setting that A2 curiosity
  hooks lacked. A2 hooks required more cognitive effort to process."

  **HOW:** [Actionable recommendation for next campaign]
  Example: "Next campaign: Generate 40% of hooks as D3 UGC testimonial type
  (up from 20%). Ensure all UGC hooks include specific timeframe claims. Test
  D3 hooks first in initial ad set before scaling other types. Reduce A2
  allocation from 30% to 15%."

  **CONFIDENCE:** [HIGH/MEDIUM/LOW with basis]
  Example: "HIGH -- Based on 12 D3 variants vs 15 A2 variants, each with 3x+
  AOV spend. Pattern consistent across all 3 Meta ad sets."

  **SCOPE:** [Vertical-specific/Universal] [Campaign-specific/Generalizable]
  Example: "Likely vertical-specific (health/supplement). UGC trust may be
  lower in finance or technology verticals. Generalizable within health vertical."

  **PROPAGATION TARGET:** [Which engine file(s) to update]
  Example: "A02 hook scoring weights (increase D3 priority), A10 UGC scoring
  bonus, health vertical profile (hook recommendations section)"

MINIMUM: 2 hook learnings per campaign. Target: 3-5.
```

### Step 5: Generate Next Campaign Hook Recommendations

```
SYNTHESIZE learnings into concrete recommendations:

  HOOK TYPE PRIORITY FOR NEXT CAMPAIGN:
    1. [hook_type] -- [allocation %] -- [rationale from learnings]
    2. [hook_type] -- [allocation %] -- [rationale]
    3. [hook_type] -- [allocation %] -- [rationale]
    ...

  HOOK TYPES TO REDUCE OR ELIMINATE:
    - [hook_type] -- reduce from [X]% to [Y]% -- [rationale]

  HOOK TYPE TESTS TO RUN:
    - [hook_type not tested in this campaign] -- "Test at [X]% allocation to
      establish baseline data"

  PLATFORM-SPECIFIC HOOK RECOMMENDATIONS:
    Meta: Prioritize [types] because [reason from data]
    TikTok: Prioritize [types] because [reason from data]
    [etc.]
```

## Quality Checks

1. Every learning follows WHAT/WHY/HOW/CONFIDENCE/SCOPE/PROPAGATION_TARGET format
2. Minimum 2 hook learnings extracted (target 3-5)
3. Causal hypotheses are specific, not generic ("video worked well" is FORBIDDEN)
4. Confidence levels based on actual sample sizes and statistical basis
5. Next campaign recommendations include specific allocation percentages
6. Platform-specific recommendations provided (if multi-platform campaign)
7. Hook types to test in next campaign identified (gaps in current data)
8. All learnings cite specific data points (hook rates, CPAs, variant counts)
