# 3.4: Fatigue Learnings

**Version:** 1.0
**Layer:** 3 (Learning Extraction)
**Type:** Learning Extraction
**Model:** opus
**Dependencies:** Layer 2 outputs (GATE_2 must be PASS), 1.3 time-series-builder
**Output:** layer-3-outputs/3.4-fatigue-learnings.md

---

## Purpose

Extract actionable learnings about creative fatigue. What was the average creative half-life? Did some variants fatigue faster than others? WHY? (Hypothesis: novelty-dependent hooks fatigue faster than educational hooks? UGC fatigues slower than polished because it looks different each time?) What was the frequency at fatigue onset? Generate creative refresh cadence recommendations. Map fatigue patterns to the AD-ENGINE.md Creative Lifespan reference data -- do the benchmarks hold for this vertical?

## Process

### Step 1: Summarize Fatigue Data Across Variants

```
FROM 1.3 time-series-builder output:

  COMPILE fatigue summary:
    variants_with_fatigue_data: integer (those with >= 7 days)
    avg_creative_half_life: float (days)
    median_creative_half_life: float (days)
    avg_seven_day_drop: float (percentage)
    avg_frequency_at_fatigue_onset: float

    fatigue_distribution:
      MINIMAL_FATIGUE: count (< 10% drop)
      MODERATE_FATIGUE: count (10-25% drop)
      SIGNIFICANT_FATIGUE: count (25-50% drop)
      SEVERE_FATIGUE: count (> 50% drop)

  IDENTIFY extremes:
    fastest_fatiguing_variant: {id, half_life, elements}
    most_durable_variant: {id, half_life, elements}
    largest_seven_day_drop: {id, drop_pct, elements}
    smallest_seven_day_drop: {id, drop_pct, elements}
```

### Step 2: Fatigue Pattern Analysis by Element

```
ANALYZE fatigue patterns by creative element:

  BY HOOK TYPE:
    For each hook type with sufficient fatigue data:
      avg_half_life: float
      avg_seven_day_drop: float
    HYPOTHESIS: Do novelty-dependent hooks (curiosity gaps, shock) fatigue faster
    than educational hooks (problem-awareness, how-to)?
    Evidence: Compare half-lives across hook categories

  BY VISUAL STYLE:
    For each visual style:
      avg_half_life: float
      avg_seven_day_drop: float
    HYPOTHESIS: Does UGC fatigue slower because each piece looks different
    (less pattern recognition) vs polished which creates identical visual memory?
    Evidence: Compare half-lives for UGC vs polished

  BY PLATFORM:
    For each platform:
      avg_half_life: float
      avg_seven_day_drop: float
    HYPOTHESIS: Does TikTok have faster fatigue due to higher content velocity?
    Does Meta have different fatigue patterns for Feed vs Stories vs Reels?

  BY AD FORMAT:
    For each format (short video, long video, static, carousel):
      avg_half_life: float
    HYPOTHESIS: Do shorter formats fatigue faster because less content diversity?
```

### Step 3: Compare Against Reference Benchmarks

```
FROM 0.4 benchmarks (AD-ENGINE.md Creative Lifespan data):

  COMPARE actual fatigue data to reference:
    | Metric | Actual Campaign | Reference Benchmark | Status |
    |--------|----------------|--------------------|---------|
    | Avg half-life | [X] days | [Y] days | FASTER/SLOWER/ALIGNED |
    | Avg 7-day drop | [X]% | 37% typical | ABOVE/BELOW/ALIGNED |
    | Freq at onset | [X] | 2.5-3.5 typical | WITHIN/OUTSIDE range |

  IF actual fatigue is FASTER than benchmark:
    --> Investigate: audience saturation? competitive density? creative similarity?
  IF actual fatigue is SLOWER than benchmark:
    --> Investigate: strong creative? low frequency? diverse audience?
    --> Note as positive finding

  UPDATE BENCHMARK RECOMMENDATION:
    "Based on this campaign's data, the [vertical] vertical benchmark for
    creative half-life should be updated from [X] to [Y] days."
```

### Step 4: Extract Fatigue Learnings

```
FOR EACH actionable learning:

  FORMAT (MANDATORY):

  ### Learning F-[ID]: [Title]

  **WHAT:** [Specific fatigue observation with data]
  Example: "UGC visual variants had an average creative half-life of 14 days vs
  8 days for polished variants. UGC seven-day drop averaged 18% vs 35% for polished."

  **WHY:** [Causal hypothesis]
  Example: "UGC content shows natural variation in each impression (slightly different
  framing, lighting, expressions), creating less pattern fatigue. Polished production
  creates identical visual memory traces that saturate faster."

  **HOW:** [Actionable recommendation]
  Example: "Allocate 60%+ of variant budget to UGC formats for durability. Plan polished
  refreshes every 7-10 days. Plan UGC refreshes every 14-18 days. Prepare 2x refresh
  creatives for polished vs UGC."

  **CONFIDENCE:** [HIGH/MEDIUM/LOW]
  **SCOPE:** [Vertical-specific/Universal]
  **PROPAGATION TARGET:** [Vertical profile (fatigue benchmarks), AD-ENGINE.md
    creative lifespan section, A10 (fatigue prediction model)]

MINIMUM: 2 fatigue learnings per campaign. Target: 3-4.
```

### Step 5: Creative Refresh Strategy Recommendations

```
SYNTHESIZE into creative refresh strategy:

  REFRESH CADENCE:
    - [visual_style] variants: refresh every [X] days (based on half-life data)
    - [visual_style] variants: refresh every [X] days

  REFRESH PRIORITY:
    1. [element type to refresh first] -- highest fatigue rate
    2. [element type to refresh second]
    3. [element type most durable] -- refresh last

  REFRESH VOLUME:
    - Produce [X] new variants per refresh cycle
    - Maintain [X] active variants at all times
    - Kill threshold: [X] days without improvement (from actual fatigue data)

  FATIGUE MONITORING PROTOCOL:
    - Monitor hook rate daily
    - Flag when rolling 3-day average drops > [X]% from peak
    - Alert when frequency reaches [X] (from actual frequency-at-onset data)
    - Trigger refresh when CPA rises [X]% above target for 3+ consecutive days
```

1. Fatigue data summarized across all variants with sufficient time-series data
2. Fatigue patterns analyzed by element type (hook, visual, platform, format)
3. Actual fatigue compared against AD-ENGINE.md reference benchmarks
4. Minimum 2 fatigue learnings in required format
5. Creative refresh strategy includes specific cadence and volume recommendations
6. Fatigue monitoring protocol defined with specific thresholds
7. Benchmark update recommendations provided
8. All learnings cite specific fatigue metrics (half-life, 7-day drop, frequency)
