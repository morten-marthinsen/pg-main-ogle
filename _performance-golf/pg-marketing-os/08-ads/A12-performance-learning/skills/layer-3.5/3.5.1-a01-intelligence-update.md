# 3.5.1: A01 Intelligence Update

**Version:** 1.0
**Layer:** 3.5 (Learning Propagation)
**Type:** Engine Propagation
**Model:** opus
**Dependencies:** Layer 3 outputs (GATE_3 must be PASS)
**Output:** layer-3.5-outputs/3.5.1-a01-intelligence-update.md

---

## Purpose

Update A01 competitive intelligence with campaign performance data. Add winning ad specimens from THIS campaign to the intelligence database. Update vertical-specific performance benchmarks with real data. If A01 Continuous Monitor is active, flag high-performing hook types and visual styles for monitoring priority. Write A01-INTELLIGENCE-PATCH.md documenting all proposed changes to A01 intelligence. This layer exists ONLY in POST-CAMPAIGN mode. Mid-Flight Analysis skips this entirely.

---

## Process

### Step 1: Verify Post-Campaign Mode and Gate 3

```
READ checkpoints/LAYER_3_COMPLETE.yaml
IF NOT EXISTS:
  --> HALT: "GATE_3 not passed. Cannot begin Layer 3.5."

VERIFY mode == POST_CAMPAIGN:
IF mode == MID_FLIGHT:
  --> SKIP this microskill entirely
  --> Produce minimal output documenting deferral:
    "A01 intelligence update DEFERRED -- Mid-Flight mode. Will execute in Post-Campaign."
```

### Step 2: Identify A01 Intelligence Updates

```
COMPILE proposed updates:

  (a) WINNING AD SPECIMENS:
    For each WINNER variant:
      - variant_id
      - hook_type, visual_style, CTA, body
      - key metrics (CPA, ROAS, hook_rate)
      - WHY it won (from learning extraction)
      PROPOSAL: "Add as winning specimen to A01 intelligence database"

  (b) VERTICAL BENCHMARK UPDATES:
    Compare campaign results to current A01 vertical benchmarks:
      Current vertical avg CPA: $[X] --> Actual campaign CPA: $[Y]
      Current vertical avg hook rate: [X]% --> Actual campaign avg: [Y]%
      Current vertical avg CTR: [X]% --> Actual campaign avg: [Y]%
    PROPOSAL: "Update [vertical] benchmarks with actual campaign data"
    NOTE: Single campaign data should be AVERAGED with existing benchmarks, not replace them

  (c) COMPETITIVE INTELLIGENCE UPDATES:
    Any learnings about how our creative performed vs competitors:
      - Hook types that beat competitive benchmarks
      - Visual styles that differentiated us
      - Platform performance vs competitor estimated performance
    PROPOSAL: "Update competitive positioning data"

  (d) CONTINUOUS MONITOR PRIORITIES:
    If A01 Continuous Monitor is active:
      - Flag winning hook types for monitoring (track if competitors adopt)
      - Flag winning visual styles for monitoring
      - Suggest new monitoring keywords based on campaign learnings
    PROPOSAL: "Update monitoring priority list"
```

### Step 3: Write A01-INTELLIGENCE-PATCH.md

```
PRODUCE patch file with the following structure:

  # A01-INTELLIGENCE-PATCH.md
  ## Patch Metadata
  - Source: A12 Performance Learning Loop -- [project name]
  - Campaign: [dates]
  - Mode: Post-Campaign Analysis
  - Status: PROPOSED (awaiting human approval)

  ## Proposed Changes

  ### 1. Winning Specimens to Add
  For each winner:
    - Variant: [id]
    - Elements: hook_type=[X], visual=[X], CTA=[X]
    - Performance: CPA=$[X], ROAS=[X]x, hook_rate=[X]%
    - Add to: A01 intelligence database

  ### 2. Vertical Benchmark Updates
  | Benchmark | Current | Proposed | Data Source | Confidence |
  | vertical_avg_CPA | $[X] | $[Y] | [calculation] | [H/M/L] |
  | vertical_avg_hook_rate | [X]% | [Y]% | [calculation] | [H/M/L] |

  ### 3. Competitive Intelligence Updates
  [Specific updates with data]

  ### 4. Monitoring Priority Updates
  [New monitoring priorities with rationale]

  ## Application Instructions
  For each change:
    - Target file: [path]
    - Target section: [section]
    - Current content: [what exists now]
    - Proposed content: [what it should change to]
    - Justification: [data from this campaign]

  ## Approval Required
  [ ] Human has reviewed all proposed changes
  [ ] Approved changes marked
  [ ] Rejected changes documented with reason
```
