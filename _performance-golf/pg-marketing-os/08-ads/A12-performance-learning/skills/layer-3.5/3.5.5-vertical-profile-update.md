# 3.5.5: Vertical Profile Update

**Version:** 1.0
**Layer:** 3.5 (Learning Propagation)
**Type:** Engine Propagation
**Model:** opus
**Dependencies:** Layer 3 outputs (GATE_3 must be PASS), all domain learnings
**Output:** layer-3.5-outputs/3.5.5-vertical-profile-update.md

---

## Purpose

Update the relevant vertical profile (`ad-verticals/[vertical].md`) with campaign-specific learnings. Update: vertical-specific hook type performance data, vertical-specific format recommendations, vertical-specific visual style patterns, vertical-specific fatigue benchmarks, vertical-specific compliance learnings. Write VERTICAL-PATCH.md documenting all proposed vertical profile changes. POST-CAMPAIGN mode only.

---

## Inputs

```yaml
input:
  from_3.1:
    hook_learnings: "layer-3-outputs/3.1-hook-learnings.md"
  from_3.2:
    visual_learnings: "layer-3-outputs/3.2-visual-learnings.md"
  from_3.3:
    platform_learnings: "layer-3-outputs/3.3-platform-learnings.md"
  from_3.4:
    fatigue_learnings: "layer-3-outputs/3.4-fatigue-learnings.md"
  from_3.6:
    validated_learnings: "layer-3-outputs/3.6-learning-validator.md"
    # Contains: all learnings with vertical profile propagation targets
  current_vertical:
    vertical_profile: "./ads/ad-verticals/[vertical].md"
```

---

## Process

### Step 1: Verify Post-Campaign Mode and Gate 3

```
IF mode == MID_FLIGHT:
  --> SKIP -- produce deferral stub
```

### Step 2: Identify Vertical-Specific Learnings

```
FROM all validated learnings (3.6):
  FILTER learnings where:
    scope_vertical == "VERTICAL_SPECIFIC" OR
    propagation_targets includes vertical profile

  ALSO INCLUDE learnings where:
    scope_vertical == "UNIVERSAL" AND
    the learning applies to vertical profile sections
    (e.g., universal fatigue finding still updates vertical benchmarks)
```

### Step 3: Compile Vertical Profile Updates

```
ORGANIZE updates by vertical profile section:

  (a) HOOK TYPE PERFORMANCE DATA:
    Update vertical-specific hook type performance table:
    | Hook Type | Previous Performance | Campaign Performance | Updated Benchmark |
    Each row: running average of previous + current campaign data

  (b) FORMAT RECOMMENDATIONS:
    Update recommended ad formats for this vertical:
    - Best performing format: [format] -- [CPA, ROAS data]
    - Recommended format mix: [X]% video, [Y]% static, [Z]% carousel
    - Platform-specific format notes: [from 3.3]

  (c) VISUAL STYLE PATTERNS:
    Update visual style recommendations:
    - UGC vs polished performance in this vertical: [data from 3.2]
    - Recommended visual mix: [X]% UGC, [Y]% polished, [Z]% mixed
    - Visual-hook synergies specific to this vertical

  (d) FATIGUE BENCHMARKS:
    Update creative fatigue reference data:
    - Vertical creative half-life: previous=[X], this campaign=[Y], updated=[Z]
    - Vertical frequency-at-fatigue: previous=[X], this campaign=[Y], updated=[Z]
    - Refresh cadence recommendation: [from 3.4]

  (e) PLATFORM STRATEGY:
    Update platform recommendations for this vertical:
    - Platform priority ranking: [from 3.3]
    - Per-platform creative strategy: [from 3.3]
    - Platform budget allocation guide: [from 3.3]

  (f) COMPLIANCE LEARNINGS:
    Any platform compliance issues encountered:
    - [Platform] flagged/rejected [type of content] -- add to compliance avoidance list
    - [Platform] approved [type of content] -- add to safe list
```

### Step 4: Write VERTICAL-PATCH.md

```
PRODUCE patch file:

  # VERTICAL-PATCH.md
  ## Patch Metadata
  - Source: A12 Performance Learning Loop -- [project name]
  - Vertical: [vertical name]
  - Campaign: [dates]
  - Status: PROPOSED

  ## 1. Hook Type Performance Updates
  | Hook Type | Previous Benchmark | Campaign Data | Updated Benchmark | Confidence |

  ## 2. Format Recommendation Updates
  - Current: [current recommendations]
  - Proposed: [updated recommendations]
  - Data basis: [campaign performance data]

  ## 3. Visual Style Pattern Updates
  - Current: [current patterns]
  - Proposed: [updated patterns]
  - Data basis: [from 3.2 visual learnings]

  ## 4. Fatigue Benchmark Updates
  | Parameter | Previous | This Campaign | Proposed Update | Calculation |

  ## 5. Platform Strategy Updates
  - Current priority: [list]
  - Proposed priority: [list]
  - Per-platform creative strategy: [updates]

  ## 6. Compliance Additions
  - New items for compliance avoidance list: [if any]
  - New items for compliance safe list: [if any]

  ## Application Instructions
  - Target file: [path to vertical profile]
  - For each section: [current content] --> [proposed content]

  ## Approval Required
  [ ] Human has reviewed all vertical profile changes
```

## Quality Checks

1. All vertical-specific learnings from 3.6 addressed
2. Benchmark updates use running average (not single-campaign replacement)
3. All six update categories addressed (hook, format, visual, fatigue, platform, compliance)
4. Patch includes specific current vs proposed values
5. Application instructions point to exact file path and section
6. PROPOSED status maintained
7. Compliance learnings captured if any platform issues encountered
