# 1.4: Classification Engine

**Version:** 1.0
**Layer:** 1 (Performance Data Ingestion & Metric Calculation)
**Type:** Data Processing / Classification
**Model:** sonnet
**Dependencies:** 1.2 per-variant-calculator output
**Output:** layer-1-outputs/1.4-classification-engine.md

---

## Purpose

Classify each variant into performance tiers using the Winner/Performer/Loser framework. Apply statistical significance checks before classification. Produce the VARIANT CLASSIFICATION TABLE with: variant_id, classification (WINNER/PERFORMER/LOSER/INSUFFICIENT_DATA), key_metric (CPA or ROAS), hook_rate, total_spend, total_conversions, statistical_basis (SUFFICIENT/INSUFFICIENT), confidence_interval. This is the definitive judgment of variant performance -- no variant gets classified without meeting statistical minimums.

## Process

### Step 1: Apply Statistical Significance Checks

```
FOR EACH variant:

  CHECK STATISTICAL BASIS:
    spend_sufficient = (total_spend >= 3 * AOV)
    impressions_sufficient = (total_impressions >= 5000)
    days_sufficient = (days_active >= 3)

    IF spend_sufficient AND impressions_sufficient AND days_sufficient:
      statistical_basis = "SUFFICIENT"
    ELSE:
      statistical_basis = "INSUFFICIENT"
      --> Classification = "INSUFFICIENT_DATA"
      --> Do NOT classify as winner, performer, or loser
      --> Record which thresholds were not met

  FORBIDDEN:
    - Declaring winners with < $50 spend
    - Declaring winners based on CTR alone (CTR != conversion)
    - Declaring winners based on a single day's data
    - "This variant looks promising" without statistical backing
```

### Step 2: Classify Sufficient-Data Variants

```
FOR EACH variant WHERE statistical_basis == "SUFFICIENT":

  WINNER (Green -- Scale):
    CPA <= target_CPA
    AND ROAS >= target_ROAS (if revenue tracked)
    AND hook_rate >= 30% (or vertical benchmark if different)
    --> Classification = "WINNER"
    --> Recommended Action: Scale spend 20-30% incrementally

  PERFORMER (Yellow -- Optimize):
    CPA > target_CPA but <= 1.5 * target_CPA
    AND hook_rate >= 20%
    --> Classification = "PERFORMER"
    --> Recommended Action: Test hook swaps, CTA swaps, or visual swaps to improve

  LOSER (Red -- Kill):
    CPA > 2 * target_CPA after 3x AOV spend
    OR hook_rate < 15%
    OR zero conversions after 3x AOV spend
    --> Classification = "LOSER"
    --> Recommended Action: Kill immediately, extract learning about WHY it failed

  EDGE CASES:
    IF variant has high hook rate (>30%) but high CPA (>2x target):
      --> Classification = "LOSER" (CPA is the decision metric)
      --> Note: "High engagement but low conversion -- likely funnel/offer issue not creative issue"

    IF variant has low hook rate (<20%) but good CPA (<=target):
      --> Classification = "WINNER" (CPA is the decision metric)
      --> Note: "Low engagement but high conversion -- likely well-targeted niche audience"

    IF ROAS data not available:
      --> Use CPA as primary classification metric
      --> Note: "ROAS not tracked -- classification based on CPA only"
```

### Step 3: Calculate Win Rate and Distribution Metrics

```
CALCULATE:
  win_rate = count(WINNER) / count(SUFFICIENT_DATA variants) * 100
  performer_rate = count(PERFORMER) / count(SUFFICIENT_DATA variants) * 100
  loser_rate = count(LOSER) / count(SUFFICIENT_DATA variants) * 100

  BENCHMARK COMPARISON:
    Win rate 6.6-10% = Below average (but normal for first campaign)
    Win rate 10-20% = Average
    Win rate 20-30% = Above average
    Win rate > 30% = Exceptional (verify data quality)

  SPEND DISTRIBUTION:
    winner_spend_pct = sum(winner_spend) / total_campaign_spend * 100
    performer_spend_pct = sum(performer_spend) / total_campaign_spend * 100
    loser_spend_pct = sum(loser_spend) / total_campaign_spend * 100
    insufficient_spend_pct = sum(insufficient_spend) / total_campaign_spend * 100

    INSIGHT: If loser_spend_pct > 40%, significant budget was wasted.
    This is a key learning for next campaign's kill threshold timing.
```

### Step 4: Produce Variant Classification Table

```
PRODUCE the MANDATORY VARIANT CLASSIFICATION TABLE:

+-----------------------------------------------------------------------------+
|  VARIANT CLASSIFICATION TABLE - [timestamp]                                  |
|                                                                              |
|  +-----------------------------------------------------------------------+  |
|  | VARIANT_ID | CLASS  | CPA    | ROAS  | HOOK  | SPEND  | CONV | STAT  |  |
|  |            |        |        |       | RATE  |        |      | BASIS |  |
|  +-----------------------------------------------------------------------+  |
|  | [id]       | WINNER | $[X]   | [X]x  | [X]%  | $[X]   | [X]  | SUFF  |  |
|  | [id]       | WINNER | $[X]   | [X]x  | [X]%  | $[X]   | [X]  | SUFF  |  |
|  | [id]       | PERF   | $[X]   | [X]x  | [X]%  | $[X]   | [X]  | SUFF  |  |
|  | [id]       | LOSER  | $[X]   | [X]x  | [X]%  | $[X]   | [X]  | SUFF  |  |
|  | [id]       | INSUF  | --     | --    | [X]%  | $[X]   | [X]  | INSUF |  |
|  +-----------------------------------------------------------------------+  |
|                                                                              |
|  SUMMARY:                                                                    |
|  Winners: [X] ([X]% of sufficient-data variants)                             |
|  Performers: [X] ([X]%)                                                      |
|  Losers: [X] ([X]%)                                                          |
|  Insufficient Data: [X] ([X]%)                                               |
|                                                                              |
|  Win Rate: [X]% (benchmark: 6.6-30%)                                         |
|  Overall Campaign CPA: $[X] (target: $[X])                                   |
|  Overall Campaign ROAS: [X]x (target: [X]x)                                  |
|                                                                              |
|  Spend Distribution:                                                         |
|  Winners: [X]% of budget | Performers: [X]% | Losers: [X]% | Insuf: [X]%   |
+-----------------------------------------------------------------------------+

SORT ORDER:
  1. WINNERS (sorted by CPA ascending -- best first)
  2. PERFORMERS (sorted by CPA ascending)
  3. LOSERS (sorted by CPA ascending)
  4. INSUFFICIENT_DATA (sorted by spend descending)
```

## Quality Checks

1. Every variant classified (no variants missing from table)
2. No variant classified as WINNER/PERFORMER/LOSER with INSUFFICIENT statistical basis
3. Classification thresholds applied consistently (CPA vs target, hook rate vs benchmark)
4. Win rate calculated and compared to benchmark range (6.6-30%)
5. Spend distribution calculated (reveals budget efficiency)
6. Edge cases documented with specific notes
7. Variant Classification Table produced in required format
8. Sort order correct (Winners first by CPA, then Performers, Losers, Insufficient)
