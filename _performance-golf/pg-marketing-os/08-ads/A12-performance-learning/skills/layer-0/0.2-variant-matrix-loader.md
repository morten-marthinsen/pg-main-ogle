# 0.2: Variant Matrix Loader

**Version:** 1.0
**Layer:** 0 (Foundation)
**Type:** Data Loading
**Model:** haiku
**Dependencies:** A09 AD-VARIANT-MATRIX/ output
**Output:** layer-0-outputs/0.2-variant-matrix-loader.md

---

## Purpose

Load the A09 AD-VARIANT-MATRIX/ to map each performance data row to its variant definition. Each variant must trace back to: hook_id (from A02), hook_type (from taxonomy), body_id (from A04), visual_treatment_id (from A05), CTA_id (from A07), platform, and format. Without this mapping, per-element analysis in Layer 2.5 (Element Attribution) is impossible.

---

## Inputs

```yaml
input:
  from_a09:
    variant_matrix_path: "[project]/A09-variant-matrix/AD-VARIANT-MATRIX/"
    # Contains: variant definitions with element composition
  from_a02:
    hook_angle_matrix_path: "[project]/A02-hook-angles/HOOK-ANGLE-MATRIX.md"
    # Contains: hook_id to hook_type mapping
  from_taxonomy:
    hook_taxonomy_path: "./References/AD-HOOK-TAXONOMY.md"
    # Contains: 32 hook types, 10 categories
```

## Process

### Step 1: Load Variant Matrix

```
READ A09 AD-VARIANT-MATRIX/ directory
EXTRACT for each variant:
  - variant_id: unique identifier
  - hook_id: which hook was used
  - body_id: which body script was used
  - visual_treatment_id: which visual treatment was used
  - CTA_id: which CTA was used
  - platform: target platform (meta, google, tiktok, youtube)
  - format: ad format (video_short, video_long, static, carousel, etc.)
  - ad_set: which ad set this variant belongs to
  - any additional element identifiers

IF variant matrix does not exist:
  --> HALT: "A09 Variant Matrix not found. Cannot map performance data to creative elements."
  --> Request A09 output
```

### Step 2: Load Hook Type Mapping

```
READ A02 HOOK-ANGLE-MATRIX.md
BUILD hook_id --> hook_type lookup table:
  For each hook in the matrix:
    - hook_id
    - hook_type (from AD-HOOK-TAXONOMY.md 32-type taxonomy)
    - hook_category (A-J from taxonomy)
    - hook_description (brief text)

IF A02 matrix does not exist:
  --> WARNING: "A02 Hook Angle Matrix not found. Hook type analysis will be limited."
  --> Proceed but flag that hook_type mapping is unavailable
```

### Step 3: Build Variant-to-Element Mapping Table

```
FOR EACH variant in the matrix:
  CREATE mapping record:
    variant_id: [id]
    elements:
      hook:
        hook_id: [id]
        hook_type: [type from taxonomy] (via A02 lookup)
        hook_category: [A-J]
      body:
        body_id: [id]
      visual:
        visual_treatment_id: [id]
        visual_style: [UGC/polished/mixed/animated/static]
      cta:
        CTA_id: [id]
    platform: [platform]
    format: [format]
    ad_set: [ad_set]
```

### Step 4: Cross-Reference with Performance Data

```
COMPARE variant_ids in matrix against variant_ids in performance data (from 0.1):

PRODUCE:
  - matched_variants: variants that exist in BOTH matrix and performance data
  - unmapped_performance_variants: variant_ids in performance data but NOT in matrix
  - untested_matrix_variants: variant_ids in matrix but NOT in performance data

IF unmapped_performance_variants > 0:
  --> CRITICAL: Performance data rows cannot be attributed to creative elements
  --> Attempt to resolve by name matching, partial ID matching
  --> If unresolvable: flag for human review

IF untested_matrix_variants > 0:
  --> INFORMATIONAL: These variants were planned but not launched or have no data
  --> Document for completeness
```

## Quality Checks

1. A09 variant matrix exists and was loaded successfully
2. Every variant has all element IDs populated (hook, body, visual, CTA)
3. Hook type mapping is available (via A02 --> taxonomy lookup)
4. Cross-reference with performance data produces zero unmapped variants (or unmapped variants are flagged)
5. Platform and format fields are valid enum values
6. Variant-to-element mapping table is complete and can support Layer 2.5 element attribution
