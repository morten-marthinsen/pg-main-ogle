# 0.0.1: Vertical Profile Loader

**Version:** 1.0
**Layer:** 0 (Foundation)
**Type:** Configuration Loading
**Dependencies:** None (first microskill in execution)
**Output:** vertical_profile.yaml

---

## Purpose

Load ad-specific vertical configuration from `ad-verticals/[vertical].md`. This microskill extracts vertical-specific performance patterns, learning baselines, and optimization references that calibrate all downstream analysis. Without vertical context, winner/loser classification uses generic thresholds, fatigue analysis misses vertical-specific creative lifespans, and learning extraction cannot produce vertical-specific hook type insights.

---

## Inputs

```yaml
input:
  from_campaign_brief:
    vertical: string  # One of: golf, health, finance, personal-dev, technology
  from_ad_verticals_directory:
    vertical_profile_path: "ads/ad-verticals/[vertical].md"
```

---

## Process

### Step 1: Identify Vertical

```
READ Campaign Brief (Skill 09 output) for vertical identifier
IF vertical not specified:
  → DEFAULT to "universal" (no vertical-specific config)
  → LOG warning: "No vertical specified — using universal configuration"
```

### Step 2: Load Vertical Profile

```
IF vertical specified:
  CONSTRUCT path: ads/ad-verticals/[vertical].md
  READ vertical profile file
  EXTRACT:
    - performance_baselines (vertical-specific CTR, CPA, ROAS, hook rate benchmarks)
    - learning_baselines (vertical-specific winner/loser thresholds, statistical significance requirements)
    - optimization_references (vertical-specific fatigue curves, refresh cadences, seasonal patterns)
    - compliance_constraints (regulatory/platform-specific restrictions for learning validation)
    - anti_slop_rules (vertical-specific language to avoid in learning reports)
    - hook_type_performance (which of the 32 hook types historically perform best in this vertical)
```

### Step 3: Normalize and Structure

```
COMPILE vertical_profile:
  vertical_id: "[vertical identifier]"
  performance_baselines:
    target_hook_rate: "[percentage — vertical-specific benchmark]"
    target_cpa: "[dollar amount — vertical-specific benchmark]"
    target_roas: "[multiplier — vertical-specific benchmark]"
    avg_ctr_by_platform:
      meta: "[percentage]"
      tiktok: "[percentage]"
      google: "[percentage]"
    avg_cpc_by_platform:
      meta: "[dollar amount]"
      tiktok: "[dollar amount]"
      google: "[dollar amount]"
  learning_baselines:
    winner_cpa_threshold: "[at or below target CPA]"
    loser_cpa_threshold: "[2x target CPA after 3x AOV spend]"
    winner_hook_rate_threshold: "[30%+ or vertical benchmark]"
    loser_hook_rate_threshold: "[below 15%]"
    minimum_spend_per_variant: "[3x AOV]"
    minimum_impressions_per_variant: "[5000]"
    minimum_days_for_significance: "[3]"
  optimization_references:
    avg_creative_lifespan_weeks: "[range at moderate spend]"
    fatigue_onset_frequency: "[typical frequency at fatigue onset]"
    refresh_cadence: "[recommended refresh interval]"
    seasonal_patterns: "[vertical-specific seasonal notes]"
  compliance_constraints:
    forbidden_claims: [list of claims that violate platform/regulatory rules]
    required_disclaimers: [list if applicable]
  anti_slop:
    forbidden_phrases: [vertical-specific slop to avoid]
    forbidden_language_patterns: [e.g., health: "cure" "treat", finance: "guaranteed returns"]
  hook_type_performance:
    high_performing_types: [list of hook types that historically perform well in this vertical]
    underperforming_types: [list of hook types that historically underperform in this vertical]
```

### Step 4: Set Downstream Configuration

```
CONFIGURE downstream microskills:
  - Layer 1 classification engine (1.4) will use vertical winner/loser thresholds
  - Layer 2 hook analyzer (2.2) will reference vertical hook type performance
  - Layer 2 fatigue detector (2.5) will use vertical fatigue baselines
  - Layer 3 prediction gap analyzer (3.1) will use vertical benchmarks for calibration
  - Layer 3 learning generator (3.2) will produce vertical-specific hook type insights
  - Layer 3.5 propagation will update vertical profile with new performance data
```

## Quality Checks

1. If vertical specified, profile file was successfully loaded
2. If vertical not specified, universal fallback is documented
3. All required sections populated (performance_baselines, learning_baselines, optimization_references, compliance_constraints, anti_slop, hook_type_performance)
4. Performance baselines are specific (not "varies" or "TBD")
5. Learning baselines include concrete thresholds tied to campaign AOV
6. Hook type performance data includes at least 3 high-performing types
7. Fatigue baselines are grounded in vertical data (not generic defaults)
