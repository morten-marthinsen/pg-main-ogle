# 0.6: Hook Matrix Loader

## Purpose
Load A02 HOOK-ANGLE-MATRIX.md. Extract human-selected hooks (the 8-10 winning hooks), hook type classifications, and angle attributions. These inform hook swap generation in Layer 2 -- swaps should span different hook types, not just rewrite the same type.

## Inputs
- A02 HOOK-ANGLE-MATRIX.md (from Hook-Angle Matrix skill output)
- AD-HOOK-TAXONOMY.md reference (32 hook types, 10 categories)

## Process
```
STEP 1: LOCATE A02 Hook-Angle Matrix.
  - Expected path: [project]/A02-hook-angle-matrix/HOOK-ANGLE-MATRIX.md
  - IF file does not exist: HALT -- hook swaps require the hook taxonomy context

STEP 2: READ HOOK-ANGLE-MATRIX.md completely.

STEP 3: EXTRACT:
  a. Human-selected hooks (the 8-10 winning hooks from A02)
  b. Hook type classification per hook (from AD-HOOK-TAXONOMY.md)
  c. Hook category per hook (A-J categories)
  d. Angle attributions (which strategic angle each hook serves)
  e. Performance indicators (any scoring or ranking data)

STEP 4: ANALYZE hook type distribution.
  - Which hook categories (A-J) are represented?
  - Which categories are MISSING? (These are opportunity gaps for Layer 2 swaps)
  - Which hook types performed best?

STEP 5: CREATE hook diversity guide for Layer 2 generation.
  - List which categories to prioritize for swaps (underrepresented categories)
  - List which types to avoid (overrepresented in winning set)
  - Map hooks to concepts (which hooks work for which concepts)

STEP 6: WRITE extracted data to output file.
```
