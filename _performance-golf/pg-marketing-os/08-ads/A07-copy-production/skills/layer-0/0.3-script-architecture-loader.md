# 0.3: Script Architecture Loader

## Purpose
Load A04 SCRIPT-PACKAGE.md. Extract script frameworks per concept, module definitions ([HOOK]+[SETUP]+[MECHANISM]+[PROOF]+[CTA]), swap points, target ad lengths per concept, and AV format templates. These are the BLUEPRINTS that A07 fills with words.

## Inputs
- A04 SCRIPT-PACKAGE.md (from Script Architecture skill output)
- AD-SCRIPT-STRUCTURES.md reference (8 frameworks, modular architecture, word counts)

## Process
```
STEP 1: LOCATE A04 Script Package.
  - Expected path: [project]/A04-script-architecture/SCRIPT-PACKAGE.md
  - IF file does not exist: HALT -- cannot fill blueprints that don't exist

STEP 2: READ SCRIPT-PACKAGE.md completely.

STEP 3: EXTRACT per concept:
  a. Assigned framework (PAS, AIDA, BAB, Hook-Body-CTA, Story-Bridge-CTA,
     Problem-Mechanism-Solution, Demo-Proof-CTA, Testimonial-Bridge-CTA)
  b. Module definitions -- which modules exist and their order
  c. Module-level word count budgets (derived from total length)
  d. Swap points -- which modules are designed for modular swapping
  e. Target ad lengths per concept (6s, 15s, 30s, 60s, 2-3min)
  f. AV format template (two-column VISUAL | AUDIO structure)

STEP 4: CROSS-REFERENCE with AD-SCRIPT-STRUCTURES.md.
  - Verify the assigned framework is one of the 8 supported frameworks
  - Verify word count budgets align with framework module allocations
  - Load the framework-specific module structure

STEP 5: MAP word count budgets per module for each concept-length combination.
  - 60s (160 words): HOOK ~15w, SETUP ~35w, MECHANISM ~45w, PROOF ~25w, CTA ~20w
  - 30s (75 words): HOOK ~10w, SETUP ~15w, MECHANISM ~25w, PROOF ~10w, CTA ~10w
  - 15s (40 words): HOOK ~8w, SETUP ~10w, MECHANISM ~12w, CTA ~8w (proof compressed)
  - 6s (15 words): HOOK ~8w, CTA ~7w (minimal structure)

STEP 6: WRITE extracted data to output file.
```
