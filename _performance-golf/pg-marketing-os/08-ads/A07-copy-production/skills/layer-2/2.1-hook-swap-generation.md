# 2.1: Hook Swap Generation

## Purpose
For each base copy, generate 5-10 alternate hooks. Each hook must be a different hook TYPE from AD-HOOK-TAXONOMY.md (not rewrites of the same type), lead coherently into the existing body, meet the word count budget, and be classified by taxonomy type. Hook swaps span at least 4 different hook categories (A-J). This is the highest-ROI test variable -- hooks account for the largest single performance variable in ad testing.

## Inputs
- Base copy scripts (from Layer 1: 1.2, 1.3, 1.4)
- AD-HOOK-TAXONOMY.md (32 hook types, 10 categories A-J)
- Hook diversity guide (from 0.6-hook-matrix-loader.md)
- Persona voice specimens (held in active context)
- Word count limits per target length

## Process
```
FOR EACH base copy (C-001, C-002, C-003):

STEP 1: IDENTIFY the base hook type.
  - What hook type is the base copy using? (from AD-HOOK-TAXONOMY.md)
  - What hook category (A-J)?
  - What does the body EXPECT from the hook?
    (What does the setup module assume has been established?)

STEP 2: IDENTIFY 5-10 target hook types for swaps.
  - MUST span at least 4 hook categories (A-J)
  - Prioritize types that:
    a. Performed well in A01 intelligence
    b. Are underused by competitors (from 0.6 opportunity gaps)
    c. Match the platform conventions
  - AVOID: all 5+ hooks being variations of the same type
  - Reference: 0.6-hook-matrix-loader.md diversity guide

STEP 3: For EACH target hook type, WRITE the hook.
  - Read the AD-HOOK-TAXONOMY.md entry for this type
  - Write the hook using the patterns specific to this type
  - Verify hook-body coherence: does this hook logically lead into
    the existing body?
  - Verify word count: hook module must fit within its word budget
    (typically 10-15 words for a 60s ad, 8 words for 15s)
  - Write the matching VISUAL column entry for the hook

STEP 4: CLASSIFY each hook swap.
  - Taxonomy type (e.g., A3 -- Surprising Fact)
  - Taxonomy category (e.g., A -- Curiosity & Information Gap)
  - Coherence assessment: HIGH / MEDIUM / LOW with existing body
  - Word count (exact number)

STEP 5: RUN diversity check.
  - Count distinct categories represented (must be >= 4)
  - Count distinct types represented (must be >= 5)
  - Verify all coherence ratings are MEDIUM or above
  - Verify all within word budget

STEP 6: IF diversity check fails, generate additional hooks
  from underrepresented categories until minimums met.
```

## Quality Checks
- Minimum 5 hook swaps per concept (target 10)
- At least 4 different hook categories (A-J) represented per concept
- Every hook classified with taxonomy type and category
- Every hook has a coherence rating (HIGH/MEDIUM/LOW)
- No LOW coherence hooks in final set (must be revised or replaced)
- All hooks within word count budget
- Visual column entries are specific (not vague)
