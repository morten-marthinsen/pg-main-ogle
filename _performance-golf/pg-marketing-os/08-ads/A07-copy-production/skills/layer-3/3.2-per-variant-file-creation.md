# 3.2: Per-Variant File Creation

## Purpose
Create individual copy files for every variant in the AD-COPY-FINAL/ directory, organized by platform. Each file contains the variant ID, full AV script, word count, classifications, quality check results, and downstream production notes. This is the physical deliverable that A08 and A09 consume.

## Inputs
- Master Variant ID Table (from 3.1)
- All variant scripts (from Layers 1-2, with any Layer 2.5 revisions)
- Quality check results (from Layer 2.5)
- Visual direction (from 0.4)

## Process
```
STEP 1: CREATE the AD-COPY-FINAL/ directory structure.

  AD-COPY-FINAL/
    META/
    TIKTOK/
    YOUTUBE/
    GOOGLE/
    [additional platforms as needed]

STEP 2: For EACH variant in the Master Variant ID Table:

  a. DETERMINE the target directory (by platform)
  b. CREATE the file using the variant ID as filename
     (e.g., C001-META-60s-HOOK03.md)
  c. POPULATE the file with the standard per-variant format:

  PER-VARIANT FILE FORMAT:
    - Variant ID (formal)
    - Metadata: concept, platform, length, type, hook type, CTA lever,
      base concept reference
    - Quality Check Results: word count, coherence, compliance, tone
    - Script (full AV format -- VISUAL | AUDIO)
    - Copy Text (Audio Only -- for easy extraction into ad platforms)
    - Notes for A08 (Visual Production): visual treatment, key moments,
      text overlay content

STEP 3: VERIFY all files were created.
  - Count files in AD-COPY-FINAL/
  - Compare against Master Variant ID Table
  - Every ID must have a corresponding file

STEP 4: LOG all created files with paths and sizes.
```
