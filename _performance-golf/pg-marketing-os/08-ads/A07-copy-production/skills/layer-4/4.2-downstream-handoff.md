# 4.2: Downstream Handoff

## Purpose
Prepare structured handoff documents for downstream skills. For A08 (Visual/Video Production): list all variant IDs that need visual assets, reference A05 visual directions per concept, note text overlay content per variant. For A09 (Assembly & Variant Matrix): provide the variant matrix structure, the combinatorial mapping, and the platform-specific file paths.

## Inputs
- Master Variant ID Table (from 3.1)
- Per-variant files in AD-COPY-FINAL/ (from 3.2)
- Visual direction (from 0.4)
- Variant matrix summary (from 3.3)

## Process
```
STEP 1: BUILD the A08 (Visual Production) handoff.

  For A08, compile:
  a. Complete list of variant IDs that need visual assets
     (every variant with a video component)
  b. Per-concept visual direction references
     (which A05 visual treatment applies to which variants)
  c. Per-variant text overlay content
     (exact text that appears on screen -- extracted from variant files)
  d. Shot-level visual requirements summary
     (aggregated from visual columns across variants)
  e. Platform-specific visual requirements
     (aspect ratios, overlay positioning, etc.)

STEP 2: BUILD the A09 (Assembly & Variant Matrix) handoff.

  For A09, compile:
  a. Variant matrix structure
     (the concept x hook x CTA grid from 3.3)
  b. Combinatorial mapping
     (which hooks go with which bodies with which CTAs)
  c. AD-COPY-FINAL/ directory path and file listing
  d. Platform-specific file organization
  e. Variant metadata for each file
     (so A09 knows what it's assembling)

STEP 3: VERIFY handoffs are actionable.
  - A08 team can produce visual assets using ONLY this handoff + A05
  - A09 can assemble the full matrix using ONLY this handoff + AD-COPY-FINAL/
  - No missing information that would require going back to A07

STEP 4: WRITE the handoff document.
```
