# 1.6: Meta Ad Spy Ingester

**Version:** 1.0
**Layer:** 1 (Scraping)
**Type:** Data Ingestion
**Dependencies:** Layer 0 complete (LAYER_0_COMPLETE.yaml exists), 0.5 (JSON Import Loader), 0.6 (Brand Database Matcher)
**Model:** sonnet
**Mode:** Tool-Assisted Scan ONLY (replaces 1.1 for tool-assisted brands)

---

## Purpose

Ingest pre-scraped Meta ads from the Meta Ad Spy tool for brands flagged as `tool_assisted = true` in the brand routing table. This microskill REPLACES microskill 1.1 (Meta Ad Library Scraper) for tool-assisted brands only. Brands not in the tool still go through 1.1 via Apify. The combined dataset (tool + Apify) feeds into the shared Layer 1 validator (1.5).

---

## Inputs

```yaml
input:
  from_0.5:
    normalized_ads: object  # All Meta Ad Spy ads, normalized
  from_0.6:
    routing_table: object  # Which brands are tool-assisted
  from_0.3:
    competitor_list: object  # Full competitor list
```

---

## Process

### Step 1: Filter to Tool-Assisted Brands

```
FROM normalized_ads (0.5 output):
  FILTER to only ads where brand_name matches a tool_assisted = true entry
  in the routing_table

LOG:
  "Tool-assisted brands: [count]"
  "Total Meta ads from tool: [count]"
  "Brands skipping Apify scrape: [list]"
```

### Step 2: Validate Completeness

```
FOR EACH tool-assisted brand:
  CHECK: ad_count >= 10 (minimum per brand for meaningful intelligence)

  IF ad_count < 10:
    WARNING: "Brand [name] has only [count] ads from Meta Ad Spy.
              Consider supplementing with Apify scrape."
    DECISION: Accept low count (tool data is higher quality) OR
              flag for Apify supplementation

OVERALL CHECK:
  Total Meta ads (tool-assisted) should contribute toward 200 Meta ad minimum.
  IF total < 200:
    LOG: "Tool-assisted Meta ads: [count]. Need [200 - count] more from Apify (1.1)."
    NOTE: Microskill 1.1 MUST still run for non-tool-assisted brands
          to reach the 200 Meta ad threshold.
```

### Step 3: Format for Layer 1 Pipeline

```
OUTPUT tool-assisted ads in SAME FORMAT as 1.1 (Meta Ad Library Scraper) output:
  This ensures downstream microskills (1.5 validator, Layer 2 classification)
  can process tool-assisted and Apify-scraped ads identically.

  FOR EACH ad:
    INCLUDE all fields from 0.5 normalized output
    ADD: ingestion_source = "meta_ad_spy"
    ADD: ingestion_timestamp = [current time]

  MERGE instructions for 1.5 (Raw Data Validator):
    "Combine 1.6 output (tool-assisted Meta ads) with 1.1 output
     (Apify Meta ads for non-tool-assisted brands) for total Meta count."
```

## Quality Checks

1. **All tool-assisted brand ads included** — no brands from routing table dropped.
2. **Output format matches 1.1** — downstream microskills can process both sources identically.
3. **Impression data preserved** — every ad retains its impressions field from the tool.
4. **Meta ad count tracked** — clear accounting of how many Meta ads come from tool vs. Apify.
5. **Low-count brands flagged** — brands with < 10 ads are documented for potential supplementation.
6. **Data source tagged** — every ad has data_source = "meta_ad_spy" for traceability.
