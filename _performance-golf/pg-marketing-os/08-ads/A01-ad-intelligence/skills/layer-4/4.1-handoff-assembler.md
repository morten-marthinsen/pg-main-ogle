# 4.1: Handoff Assembler

**Version:** 1.0
**Layer:** 4 (Output Packaging)
**Type:** Assembly
**Dependencies:** Layer 3 complete (LAYER_3_COMPLETE.yaml exists)
**Model:** sonnet

---

## Purpose

Assemble AD-INTELLIGENCE-HANDOFF.md from all Layer 1-3 outputs. This is the PRIMARY DELIVERABLE of the A01 skill -- the comprehensive intelligence document consumed by all downstream Ad Engine skills (A02-A05+). It includes all 10 required sections, minimum 100KB, assembled via chunked writing (8-12 write operations) because LLMs have a ~30KB output limit per response.

This is an ASSEMBLY operation. It combines pre-validated artifacts from Layers 1-3. It does NOT generate new analysis. Every section pulls from a specific Layer 3 output.

---

## Process

### Step 0: Pre-Assembly Verification

```
BEFORE assembling, verify ALL source files exist:

  REQUIRED files:
    [ ] layer-3-outputs/3.1-competitive-landscape-map.md
    [ ] layer-3-outputs/3.2-hook-type-distribution.md
    [ ] layer-3-outputs/3.3-format-platform-intelligence.md
    [ ] layer-3-outputs/3.4-winning-ad-specimens.md
    [ ] layer-3-outputs/3.5-opportunity-gap-analysis.md
    [ ] layer-3-outputs/3.6-trend-detection.md

  IF any file missing:
    HALT: "Cannot assemble handoff — missing [file]. Layer 3 is not complete."

  VERIFY LAYER_3_COMPLETE.yaml exists.
  IF NOT: HALT.
```

### Step 1: PHASE 1 — Skeleton + Executive Summary (>= 5KB)

```
WRITE the file with all 10 section headers + Metadata + Section 1.

Section 1 (Executive Summary) pulls from:
  - 3.1: Market overview, dominant players, competitive intensity
  - 3.2: Dominant hook patterns
  - 3.5: Key opportunity signals (top 3)
  - Campaign Brief: campaign context

Content:
  - Market overview: size, dominant players, competitive intensity
  - Dominant creative patterns: what most competitors are doing
  - Key opportunity signals: where the market is NOT looking
  - Top 3 recommended hook types for this campaign (with rationale)
  - 3-sentence intelligence summary for quick consumption

SIZE CHECKPOINT: Verify file >= 5KB after Phase 1.
IF < 5KB: Executive Summary is too thin — expand.
```

### Step 2: PHASE 2 — Competitive + Distribution Data (>= 30KB cumulative)

```
APPEND Sections 2-5 to the file in 2-4 write operations:

WRITE 2A: Section 2 (Competitor Landscape)
  Source: 3.1-competitive-landscape-map.md
  For each competitor brand (10+):
    Brand name, URL, ad volume, platforms, dominant hooks, formats, visual style,
    market position, creative strategy notes.
  Market leader vs. challenger analysis.
  Competitive intensity score.

WRITE 2B: Section 3 (Hook Type Distribution)
  Source: 3.2-hook-type-distribution.md
  Full 32-type distribution table.
  Top 5 prevalent, bottom 10 underused.
  Hook diversity per competitor.
  Benchmark cross-reference.

WRITE 2C: Section 4 (Format Distribution) + Section 5 (Visual Style Analysis)
  Source: 2.2 + 2.3 + 3.3
  Overall format distribution.
  Platform-specific format tables.
  Video length distribution.
  Visual style distribution per platform.
  Visual style x performance correlation.
  Production quality assessment.

SIZE CHECKPOINT: Verify file >= 30KB after Phase 2.
IF < 30KB: Sections are too thin — expand with more detail from Layer 2/3 outputs.
```

### Step 3: PHASE 3 — Specimens + Intelligence (>= 100KB cumulative)

```
APPEND Sections 6-10 in 4-6 write operations:

WRITE 3A: Section 6 (Winning Ad Specimens — Top 20)
  Source: 3.4-winning-ad-specimens.md
  THIS IS THE LARGEST SECTION (30-50KB expected).
  For each of 20 specimens:
    Specimen ID, brand, platform, format, verbatim copy (headline + body + CTA),
    hook type, visual style, run duration, why it's a winner, key patterns.
  DO NOT truncate. DO NOT summarize. Full verbatim copy for all 20.

  NOTE: This section may require 2-3 write operations by itself.

WRITE 3B: Section 7 (Emerging Trends)
  Source: 3.6-trend-detection.md
  Initial Scan: General observations from run duration tiers.
  Continuous Monitor: Specific changes (new ads, retired ads, shifts).

WRITE 3C: Section 8 (Opportunity Gaps)
  Source: 3.5-opportunity-gap-analysis.md
  All identified gaps with evidence, impact, recommended action.
  Cross-reference with Skill 01 research (if available).
  Opportunity ranking.

WRITE 3D: Section 9 (Platform-Specific Intelligence)
  Source: 3.3-format-platform-intelligence.md
  Per-platform: volume, winning patterns, compliance, recommendations.

WRITE 3E: Section 10 (Recommended Hooks for This Campaign)
  Source: 3.2 + 3.5 + campaign brief
  Top 10 hook types ranked by: performance + gap size + audience alignment.
  For each: taxonomy classification, rationale, winning specimen reference,
  platform recommendation, competition level.

SIZE CHECKPOINT: Verify file >= 100KB after Phase 3.
IF < 100KB: Sections are too thin. Most likely Section 6 (specimens) needs
            expansion — full verbatim copy not included.
```

### Step 4: Post-Assembly Verification

```
AFTER assembly complete:

  1. VERIFY file exists at expected path
  2. MEASURE file size in KB
  3. VERIFY all 10 sections present (search for each section header)
  4. VERIFY no empty sections (each section has content)
  5. VERIFY specimen section contains all 20 WS-IDs
  6. VERIFY hook distribution table covers 32 types

  IF any verification fails:
    DOCUMENT specific failure
    FIX before proceeding to 4.2

  LOG:
    "AD-INTELLIGENCE-HANDOFF.md assembled."
    "File size: [X]KB"
    "Sections verified: 10/10"
    "Specimens included: [X]/20"
```

## Output File

- **Path:** `AD-INTELLIGENCE-HANDOFF.md` (in the project's A01-ad-intelligence directory)
- **Minimum size:** 100KB
- **Assembly method:** Chunked — 8-12 write operations across 3 phases
- **Size checkpoints:** After Phase 1 (>= 5KB), After Phase 2 (>= 30KB), After Phase 3 (>= 100KB)

---

## Quality Checks

1. **File size >= 100KB** -- this is the FLOOR. A comprehensive intelligence document with 20 verbatim specimens, 10+ competitor profiles, 32-type distribution, and 5+ opportunity gaps requires at least 100KB.
2. **All 10 sections present and populated** -- no empty sections, no placeholder text.
3. **Chunked assembly used** -- file was NOT written in a single operation (impossible at 100KB+).
4. **Size checkpoints passed** -- Phase 1 >= 5KB, Phase 2 >= 30KB, Phase 3 >= 100KB.
5. **Specimens are verbatim** -- Section 6 contains full copy transcriptions, not summaries.
6. **Sources cited** -- each section references its Layer 3 source file.
7. **No new analysis generated** -- this is assembly, not synthesis. All content comes from existing Layer 1-3 outputs.
