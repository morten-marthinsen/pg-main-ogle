# 2.1: Hook Type Classifier

**Version:** 1.0
**Layer:** 2 (Classification)
**Type:** Pattern Classification
**Dependencies:** Layer 1 complete (LAYER_1_COMPLETE.yaml exists), AD-HOOK-TAXONOMY.md loaded
**Model:** opus

---

## Purpose

Classify every scraped ad's hook against the 32-type taxonomy from `AD-HOOK-TAXONOMY.md`. Every single ad must receive a hook type classification -- 100% coverage, zero "unclassified" bucket. Ads that do not fit a single type perfectly get classified as the CLOSEST type with a documented note explaining the ambiguity.

This microskill produces the foundational hook classification data that Layer 3 uses for distribution analysis, opportunity gap identification, and downstream hook recommendations for A02.

---

## Inputs

```yaml
input:
  from_layer_1:
    all_scraped_ads: object  # Combined output from 1.1, 1.2, 1.3, 1.4
    total_ad_count: integer  # From 1.5 validation
  reference:
    hook_taxonomy: "./References/AD-HOOK-TAXONOMY.md"
      # 32 hook types across 10 categories (A-J)
      # Each type has: definition, examples, performance benchmarks
```

## Process

### Step 1: Load Hook Taxonomy

```
READ AD-HOOK-TAXONOMY.md completely.
INTERNALIZE all 32 hook types across 10 categories (A-J).
For EACH type, note:
  - Type ID (e.g., A1, B2, C3)
  - Type name (e.g., "Authority Doctor", "Curiosity Gap", "Testimonial UGC")
  - Defining characteristics
  - Example hooks
  - Performance benchmark data (if available)

DO NOT proceed until taxonomy is fully loaded.
VERIFY: Can you name all 32 types? If not, re-read.
```

### Step 2: Prepare Classification Pipeline

```
LOAD all scraped ads from Layer 1 outputs:
  - layer-1-outputs/1.1-meta-ad-library-scraper.md (Meta ads)
  - layer-1-outputs/1.2-tiktok-creative-center-scraper.md (TikTok ads)
  - layer-1-outputs/1.3-google-ads-transparency-scraper.md (Google ads)
  - layer-1-outputs/1.4-additional-platform-scanner.md (Other platforms, if executed)

COUNT total ads to classify: [X]
This count MUST match the total from 1.5 validation.

ESTABLISH classification tracking:
  - classified_count: 0
  - target_count: [total from 1.5]
  - classification_log: []
```

### Step 3: Per-Ad Classification

```
FOR EACH scraped ad:

  1. EXTRACT hook text:
     - For video ads: first 3 seconds of script / text overlay
     - For static/carousel: headline / primary text
     - For search ads: headline + description line 1

  2. CLASSIFY against taxonomy:
     - READ the hook text carefully
     - COMPARE against all 32 type definitions
     - ASSIGN the BEST FIT type
     - If ambiguous between 2+ types:
       → Assign the CLOSEST type
       → Add classification_note: "Ambiguous between [Type A] and [Type B]. Classified as [A] because [reason]."

  3. RECORD classification:
     ad_classification:
       ad_id: "[from scraped data]"
       brand_name: "[from scraped data]"
       platform: "[meta/tiktok/google/other]"
       hook_type: "[taxonomy type ID, e.g., A1]"
       hook_type_name: "[taxonomy type name]"
       hook_category: "[A through J]"
       hook_text: "[verbatim first 3 seconds / first line]"
       confidence_score: [1-10]
       classification_note: "[null if clean fit, explanation if ambiguous]"

  4. INCREMENT classified_count

  CRITICAL: Each ad gets INDIVIDUAL classification with specific reasoning.
  FORBIDDEN: Batch-classifying ("these 50 are all curiosity hooks")
```

### Step 4: Classification Progress Check

```
AFTER every 100 ads classified:
  CALCULATE: classified_count / target_count * 100 = classification_pct
  LOG: "[classified_count] / [target_count] classified ([classification_pct]%)"

  IF classification_pct < 100% AND no more ads remain:
    ERROR: Count mismatch — investigate missing ads

  CONTINUE until classification_pct = 100%
```

### Step 5: Distribution Summary

```
CALCULATE hook type distribution:
  FOR EACH of 32 hook types:
    - count: number of ads classified as this type
    - percentage: count / total * 100
    - platforms: which platforms this type appears on
    - brands: which brands use this type
    - avg_confidence: average confidence score for this type

CALCULATE hook category distribution:
  FOR EACH of 10 categories (A-J):
    - count: sum of all types in this category
    - percentage: count / total * 100

IDENTIFY:
  - top_5_types: most prevalent hook types
  - bottom_10_types: least prevalent (or absent) hook types
  - absent_types: hook types with zero ads (0%)
```

## Quality Checks

1. **100% classification coverage** -- total_ads_classified MUST equal total ads from Layer 1 validation. Zero gaps.
2. **All 32 types referenced** -- distribution table covers all 32 types (even if count = 0 for some).
3. **No "unclassified" bucket** -- every ad has a type assignment. Ambiguous ads use closest type + note.
4. **Individual classification** -- no batch-classification evidence (all ads in a group getting identical types without individual reasoning).
5. **Hook text is verbatim** -- extracted text matches the actual ad copy, not a paraphrase.
6. **Confidence scores are calibrated** -- scores of 9-10 are clean fits; scores of 5-6 are ambiguous with notes; scores below 5 are flagged for review.
7. **AD-HOOK-TAXONOMY.md was loaded** -- execution context confirms file was read before classification began.
