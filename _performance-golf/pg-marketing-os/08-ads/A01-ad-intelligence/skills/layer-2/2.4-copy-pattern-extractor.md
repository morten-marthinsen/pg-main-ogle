# 2.4: Copy Pattern Extractor

**Version:** 1.0
**Layer:** 2 (Classification)
**Type:** Copy Analysis
**Dependencies:** Layer 1 complete (LAYER_1_COMPLETE.yaml exists)
**Model:** opus

---

## Purpose

Extract and analyze copy patterns from all scraped ads. Identify headline structures, body copy patterns, CTA language, and copy length distributions. Extract VERBATIM copy from the top 50 ads (longest-running as proxy for winning). This microskill produces the raw copy intelligence that feeds winning specimen extraction (3.4) and opportunity gap analysis (3.5).

---

## Inputs

```yaml
input:
  from_layer_1:
    all_scraped_ads: object  # Combined output from 1.1-1.4
    total_ad_count: integer
  from_2.5:
    run_duration_tiers: object  # Optional — if 2.5 runs in parallel, use run duration
                                 # if available; otherwise identify top 50 by other signals
```

## Process

### Step 1: Extract All Copy Elements

```
FOR EACH scraped ad that contains copy text:

  EXTRACT and categorize:
    headline:
      text: string  # verbatim headline text
      word_count: integer
      structure: string  # e.g., "question", "number_list", "statement", "command",
                          # "how_to", "warning", "story_opener", "quote"
    body_copy:
      text: string  # verbatim body copy
      word_count: integer
      sentence_count: integer
      avg_sentence_length: float
      paragraph_count: integer
      uses_bullets: boolean
      uses_numbers: boolean
      uses_emojis: boolean
    cta:
      text: string  # verbatim CTA text
      cta_type: string  # "learn_more", "shop_now", "watch_video", "get_offer",
                         # "sign_up", "download", "book_now", "custom"
      action_verb: string  # primary verb in CTA

  NOTE: Not all ads have all elements. Record what exists; mark missing as null.
```

### Step 2: Headline Pattern Analysis

```
ANALYZE headline patterns across all ads:

  1. HEADLINE STRUCTURE DISTRIBUTION:
     Count and percentage per structure type:
     - question: "Do you know the #1 cause of..."
     - number_list: "7 Signs Your..."
     - statement: "This ingredient is destroying..."
     - command: "Stop eating this food immediately"
     - how_to: "How to reverse..."
     - warning: "WARNING: Do not take this supplement..."
     - story_opener: "I was about to lose everything when..."
     - quote: "Dr. Gundry says..."
     - other: anything not fitting above

  2. HEADLINE LENGTH DISTRIBUTION:
     - 1-5 words (ultra-short)
     - 6-10 words (short)
     - 11-15 words (medium)
     - 16-20 words (long)
     - 21+ words (extended)

  3. COMMON HEADLINE WORDS:
     Top 20 most frequent words in headlines (excluding articles/prepositions)
     Sorted by frequency with count

  4. HEADLINE FORMULAS:
     Identify recurring structural formulas:
     e.g., "[Number] [noun] that [verb]..."
     e.g., "The #1 [noun] for [benefit]"
     e.g., "How [subject] [achievement] in [timeframe]"
     Document at least 10 recurring formulas with examples
```

### Step 3: Body Copy Pattern Analysis

```
ANALYZE body copy patterns across all ads:

  1. COPY LENGTH DISTRIBUTION:
     By platform:
       - Ultra-short (1-25 words)
       - Short (26-75 words)
       - Medium (76-150 words)
       - Long (151-300 words)
       - Extended (300+ words)

  2. SENTENCE STRUCTURE PATTERNS:
     - Average sentence length by platform
     - Short vs. long sentence ratio
     - Question frequency (how often body copy contains questions)
     - Exclamation frequency

  3. FORMATTING PATTERNS:
     - Percentage using bullet points
     - Percentage using numbered lists
     - Percentage using emojis
     - Percentage using ALL CAPS for emphasis
     - Percentage using line breaks for readability

  4. CONTENT STRUCTURE PATTERNS:
     Common body copy sequences:
     e.g., "Hook → Problem → Solution hint → CTA"
     e.g., "Testimonial → Result → Product mention → CTA"
     e.g., "Question → Surprising answer → Explanation → CTA"
     Document the top 5 most common sequences
```

### Step 4: CTA Pattern Analysis

```
ANALYZE CTA patterns:

  1. CTA TEXT DISTRIBUTION:
     Count and percentage per CTA type:
     - "Learn More" variants
     - "Shop Now" variants
     - "Watch Video" / "Watch Now"
     - "Get [X]" / "Get Your [X]"
     - "Sign Up" / "Subscribe"
     - "Download" variants
     - "Book Now" / "Schedule"
     - Custom CTAs (list verbatim)

  2. CTA ACTION VERBS:
     Top 10 most frequent action verbs in CTAs

  3. PLATFORM-SPECIFIC CTA PATTERNS:
     Which CTAs dominate on which platforms
```

### Step 5: Top 50 Verbatim Copy Extraction

```
IDENTIFY top 50 ads for verbatim extraction:
  PRIMARY SORT: Estimated run duration (longest = most likely winners)
  SECONDARY SORT: Platform diversity (ensure coverage across platforms)
  TERTIARY SORT: Brand diversity (ensure coverage across competitors)

FOR EACH of the top 50:
  EXTRACT complete copy — verbatim, character for character:

  verbatim_specimen:
    rank: integer  # 1-50
    ad_id: string
    brand_name: string
    platform: string
    estimated_run_days: integer
    headline: string  # EXACT TEXT
    body_copy: string  # EXACT TEXT — full text, no truncation
    cta: string  # EXACT TEXT
    total_word_count: integer
    format_type: string

CRITICAL: These are VERBATIM transcriptions.
  "The ad talks about gut health" is NOT verbatim.
  The EXACT words in the ad, character by character, IS verbatim.
```

## Quality Checks

1. **Top 50 verbatim copies extracted** -- minimum 50 ads with full copy transcription. If fewer than 50 ads have sufficient copy, extract all that do and document the shortfall.
2. **Verbatim means verbatim** -- exact text, not paraphrases or summaries. Spot-check 5 random specimens.
3. **All pattern categories analyzed** -- headline, body copy, CTA all have distribution tables.
4. **Platform-specific breakdowns present** -- copy patterns per platform, not just overall.
5. **Recurring formulas identified** -- at least 10 headline formulas documented with examples.
6. **Top 50 ranked by run duration** -- longest-running ads prioritized, not subjective "best" picks.
7. **Word counts calculated** -- every verbatim specimen has accurate word count.
