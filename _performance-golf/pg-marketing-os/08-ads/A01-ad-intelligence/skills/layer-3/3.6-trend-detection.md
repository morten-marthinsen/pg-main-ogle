# 3.6: Trend Detection

**Version:** 1.0
**Layer:** 3 (Intelligence Synthesis)
**Type:** Temporal Analysis
**Dependencies:** 3.2 and 3.3 complete; previous AD-INTELLIGENCE-HANDOFF.md (Continuous Monitor mode ONLY)
**Model:** opus

---

## Purpose

Compare the current scan to the previous scan to identify trends, shifts, and emerging patterns in the competitive ad landscape. This microskill is MANDATORY in Continuous Monitor mode and SKIPPED in Initial Scan mode.

Trend detection answers: What has CHANGED since last time? New ads launched, ads retired, format shifts, hook type shifts, new competitor entrants, competitors scaling up or down. These temporal signals reveal market direction and inform campaign adjustments.

---

input:

## Process

### Step 0: Mode Check

```
IF mode != "CONTINUOUS_MONITOR":
  SKIP this microskill entirely.
  CREATE minimal output file:
    "# 3.6: Trend Detection
     ## Mode: INITIAL_SCAN
     Trend detection is skipped in Initial Scan mode.
     This microskill executes only in Continuous Monitor mode
     when a previous AD-INTELLIGENCE-HANDOFF.md exists for comparison."
  EXIT

IF mode == "CONTINUOUS_MONITOR":
  VERIFY previous AD-INTELLIGENCE-HANDOFF.md exists and is loadable.
  IF NOT: HALT — "Cannot run trend detection without previous scan data."
  PROCEED with full execution.
```

### Step 1: Load Previous Baseline

```
READ previous AD-INTELLIGENCE-HANDOFF.md.

EXTRACT baseline data:
  baseline:
    scan_date: string
    total_ads: integer
    competitor_count: integer
    hook_type_distribution: object  # all 32 types with counts and %
    format_distribution: object  # per platform
    visual_style_distribution: object
    winning_specimens: [string]  # specimen IDs
    competitor_profiles: [object]  # brand names, ad counts, positions
```

### Step 2: New Ad Analysis

```
IDENTIFY all ads in current scan that were NOT in previous scan:

  new_ads:
    total_count: integer
    by_platform:
      - platform: string
        count: integer
    by_brand:
      - brand: string
        count: integer
        net_change: string  # "new entrant" or "+X ads since last scan"
    notable_new_ads: [object]
      # New ads that are noteworthy (high estimated quality, unusual format, etc.)
```

### Step 3: Retired Ad Analysis

```
IDENTIFY ads from previous scan that are NO LONGER running:

  retired_ads:
    total_count: integer
    by_platform:
      - platform: string
        count: integer
    by_brand:
      - brand: string
        count: integer
    notable_retirements: [object]
      # Former TIER_WINNER or TIER_PERFORMER ads that were pulled
      # This signals creative fatigue or strategy shift
```

### Step 4: Hook Type Shift Analysis

```
FOR EACH of 32 hook types:
  COMPARE current_pct vs. baseline_pct:
    delta_pct: current_pct - baseline_pct
    direction: "increasing" / "decreasing" / "stable" / "new" / "disappeared"

  FLAG significant shifts (delta > 3% or new/disappeared):
    hook_type_shifts:
      - type_id: string
        type_name: string
        previous_pct: float
        current_pct: float
        delta_pct: float
        direction: string
        significance: string  # "significant" (> 3%) / "minor" (1-3%) / "stable" (< 1%)
        interpretation: string
          # What does this shift mean?
          # e.g., "UGC testimonial hooks surging (+8%) — competitors shifting to social proof"

  IDENTIFY emerging types:
    Hook types that were absent or < 2% in previous scan but are now > 5%.
    These represent market INNOVATION — new creative approaches being tested.
```

### Step 5: Format Shift Analysis

```
FOR EACH format type, by platform:
  COMPARE current distribution vs. baseline:
    delta_pct: current - baseline

  FLAG significant format shifts:
    format_shifts:
      - platform: string
        format: string
        previous_pct: float
        current_pct: float
        delta_pct: float
        direction: string
        interpretation: string

  VIDEO LENGTH SHIFTS:
    Compare video length distributions.
    e.g., "Short-form video (< 15s) up 12% on TikTok — market moving shorter"
```

### Step 6: Competitor Activity Analysis

```
COMPARE competitor profiles between scans:

  NEW ENTRANTS:
    Brands in current scan not in previous scan.
    For each: estimated size, platforms, creative approach.

  SCALING BRANDS:
    Brands with significantly more ads (> 20% increase).
    Indicates increased ad spend / aggressive testing.

  DECLINING BRANDS:
    Brands with significantly fewer ads (> 20% decrease) or pulling winners.
    Indicates budget cut, pivot, or creative fatigue.

  STRATEGY SHIFTS:
    Brands that changed their dominant hook type or format.
    e.g., "Brand X shifted from polished video to UGC — major creative pivot"

  competitor_changes:
    new_entrants: [object]
    scaling: [object]
    declining: [object]
    strategy_shifts: [object]
```

### Step 7: Emerging Patterns Summary

```
SYNTHESIZE all shift data into emerging patterns:

  emerging_patterns:
    - pattern: string
      evidence: [string]  # specific data points
      confidence: string  # "strong" (multiple signals) / "moderate" / "suggestive"
      implication_for_campaign: string
        # What should we DO differently based on this trend?
      urgency: string  # "immediate" / "monitor" / "long-term"

  EXAMPLES of pattern synthesis:
    "UGC is replacing polished production across all platforms"
      evidence: [visual style shift +15%, format shift to selfie-style, new winner ads are UGC]
    "Short-form video is cannibalizing long-form"
      evidence: [< 15s up 12%, > 60s down 8%, new TIER_TESTING ads mostly under 30s]
    "Authority hooks are declining as testimonial hooks surge"
      evidence: [authority hooks -5%, testimonial hooks +8%, new winners are testimonial]
```

## Quality Checks

1. **Mode verified** -- only executes in Continuous Monitor mode. In Initial Scan mode, produces minimal stub.
2. **Previous baseline loaded** -- previous handoff data is actually read and extracted, not assumed.
3. **All shift dimensions analyzed** -- hook type, format, visual style, competitor activity.
4. **Significant thresholds defined** -- shifts > 3% are flagged, not arbitrary.
5. **Emerging patterns are synthesized** -- not just raw deltas but interpreted trends with implications.
6. **Recommended actions are specific** -- tied to specific trends with clear rationale.
7. **Days between scans documented** -- temporal context for interpreting magnitude of changes.
8. **New entrants and departures tracked** -- competitive landscape changes documented.
