# 0.3: Competitor List Builder

**Version:** 1.0
**Layer:** 0 (Foundation)
**Type:** List Generation
**Dependencies:** 0.1 (Campaign Brief Loader)
**Output:** competitor_list.yaml

---

## Purpose

Build a comprehensive list of 10-20 competitor brands to scan for ad intelligence. Sources include: (a) competitors named in Campaign Brief, (b) competitors identified in Skill 01 Research, (c) additional brands discovered via Apify/Firecrawl search. This list becomes the target set for all Layer 1 scraping operations.

---

## Inputs

```yaml
input:
  from_0.1:
    competitor_names: [string]  # from Campaign Brief
  from_skill_01:
    research_handoff_path: "[project]/01-research/FINAL_HANDOFF.md"
    # Contains competitor mentions in market analysis
  from_mcp_tools:
    apify_search: available
    firecrawl_search: available
```

---

## Process

### Step 1: Load Known Competitors

```
START with competitors from Campaign Brief (0.1 output)
ADD competitors from Skill 01 Research:
  - READ FINAL_HANDOFF.md Section 2 (Competitive Landscape)
  - EXTRACT competitor brand names mentioned
  - DEDUPLICATE with Campaign Brief list

CURRENT COUNT: [X] competitors
```

### Step 1.5: Meta Ad Spy Brand Database Lookup (Tool-Assisted Scan Only)

```
IF mode = TOOL_ASSISTED_SCAN:

  LOAD vertical config's meta_ad_spy_brands section
  FOR EACH brand in meta_ad_spy_brands:
    IF brand NOT already in competitor list:
      ASSESS: Is this brand a relevant competitor for this campaign?
      IF relevant: ADD to competitor list with discovery_source = "meta_ad_spy_database"
    IF brand already in competitor list:
      FLAG: tool_assisted = true
      NOTE: "Meta Ad Spy data available — skip Apify scrape for Meta ads"

  LOG: "Brands flagged as tool_assisted: [count]"

  This step enriches the competitor list AND flags brands for efficient scraping.
  Flagged brands get their Meta ads from Meta Ad Spy JSON import (0.5/0.6/1.6)
  instead of live Apify scraping (1.1).

ELSE (Initial Scan or Continuous Monitor):
  SKIP this step — no Meta Ad Spy data available
```

### Step 2: Discover Additional Competitors

```
IF current_count < 10:
  → EXPAND via search

SEARCH STRATEGIES:
  1. Google Ads Transparency Center search for product category
  2. Meta Ad Library search for product category keywords
  3. Apify actor search for industry-specific scrapers
  4. Firecrawl search of industry review sites / comparison pages

FOR EACH discovered brand:
  VERIFY:
    - Brand actively advertising (not just mentioned)
    - Brand in same product category
    - Brand targets same/similar audience

  ADD to list if verified
```

### Step 3: Competitor Profiling

```
FOR EACH competitor on list:
  COMPILE profile:
    - brand_name: [official brand name]
    - known_platforms: [where they advertise — if known from discovery]
    - estimated_market_position: enum[leader, challenger, niche]
    - product_category: [specific category]
    - website_url: [if available]
    - notes: [any relevant context]
```

### Step 4: List Validation

```
VERIFY:
  - Total count >= 10 (minimum)
  - No duplicate brands
  - All brands are actual competitors (not related industries)
  - Mix of market leaders AND challengers (not just top 3)

IF count < 10 after expansion:
  → LOG warning: "Only [X] competitors found. Market may have limited competition."
  → PROCEED with available list
  → Flag for human review
```

## Output Schema

```yaml
output:
  competitor_list:
    total_count: integer
    competitors:
      - brand_name: string
        known_platforms: [string]  # meta, tiktok, google, etc.
        estimated_market_position: enum[leader, challenger, niche, unknown]
        product_category: string
        website_url: string
        discovery_source: enum[campaign_brief, research_handoff, apify_search, firecrawl_search, meta_ad_spy_database, manual]
        tool_assisted: boolean  # true if Meta Ad Spy data available for this brand
        notes: string

    list_status:
      meets_minimum: boolean  # >= 10
      expansion_rounds_executed: integer
      discovery_methods_used: [string]
      tool_assisted_brands: integer  # count of brands with Meta Ad Spy data
```

## Output File

- **Path:** `layer-0-outputs/0.3-competitor-list-builder.yaml`

## Quality Checks

1. Competitor count >= 10 (GATE_0 requirement)
2. All competitors are in same/similar product category
3. No duplicate entries
4. At least 3 discovery sources used (not just Campaign Brief)
5. Market position estimated for each (helps with analysis later)
6. Each competitor has at least brand_name + product_category (minimum data)

---

## Dependencies

- 0.1 (Campaign Brief Loader) complete
- Skill 01 Research output available
- MCP tools available (validated by 0.2)

## Expansion Protocol

```
IF initial list < 10:

ROUND 1:
  - Search Meta Ad Library for product category
  - Search Google Ads Transparency for product category
  - Add verified brands
  - Re-count

ROUND 2 (if still < 10):
  - Firecrawl industry review sites
  - Firecrawl comparison/vs pages
  - Add verified brands
  - Re-count

ROUND 3 (if still < 10):
  - Broaden product category (e.g., "gut health" → "digestive supplements")
  - Search with broader terms
  - Add verified brands
  - Re-count

AFTER ROUND 3:
  IF still < 10:
    → LOG: "Expansion complete. Found [X] competitors."
    → PROCEED with available list
    → Human review recommended
```
