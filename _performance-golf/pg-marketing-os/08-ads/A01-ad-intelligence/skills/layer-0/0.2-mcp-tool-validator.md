# 0.2: MCP Tool Validator

**Version:** 1.0
**Layer:** 0 (Foundation)
**Type:** Tool Validation
**Dependencies:** Firecrawl MCP server, Apify MCP server
**Output:** mcp_tool_validation.yaml

---

## Purpose

Verify that both required MCP servers (Firecrawl and Apify) are available and responding. These tools are CRITICAL for Layer 1 ad scraping — without them, the skill cannot execute. This microskill tests both servers with simple validation requests and logs their status.

---

## Inputs

```yaml
input:
  mcp_servers:
    firecrawl: "mcp__firecrawl-mcp__*"
    apify: "mcp__apify__*"
```

---

## Process

### Step 1: Test Firecrawl MCP Server

```
ATTEMPT: mcp__firecrawl-mcp__firecrawl_scrape
  url: "https://example.com"
  formats: ["markdown"]

EXPECTED: Successful response with content

IF SUCCESS:
  → firecrawl_status: AVAILABLE
  → LOG: "Firecrawl MCP server verified: [version info]"

IF FAILURE:
  → firecrawl_status: UNAVAILABLE
  → LOG ERROR: "Firecrawl MCP server unavailable: [error message]"
  → GATE_0 BLOCKED until resolved
```

### Step 2: Test Apify MCP Server

```
ATTEMPT: mcp__apify__search-actors
  query: "facebook"

EXPECTED: List of actor results

IF SUCCESS:
  → apify_status: AVAILABLE
  → LOG: "Apify MCP server verified: [actor count returned]"

IF FAILURE:
  → apify_status: UNAVAILABLE
  → LOG ERROR: "Apify MCP server unavailable: [error message]"
  → GATE_0 BLOCKED until resolved
```

### Step 3: Record Capabilities

```
IF Firecrawl AVAILABLE:
  QUERY available Firecrawl tools:
    - firecrawl_scrape
    - firecrawl_map
    - firecrawl_crawl
    - firecrawl_search

  LOG which tools are accessible

IF Apify AVAILABLE:
  SEARCH for key actors:
    - "facebook ad library scraper"
    - "tiktok creative center"
    - "google ads transparency"

  LOG actor IDs for top results (will be used in Layer 1)
```

### Step 4: Gate Decision

```
IF both_servers_available:
  → GATE_0_TOOLS: PASS
  → Can proceed to Layer 1 scraping

IF one_or_both_unavailable:
  → GATE_0_TOOLS: FAIL
  → HALT: Cannot proceed without MCP tools
  → HUMAN ESCALATION: Provide setup instructions
```

## Quality Checks

1. Both Firecrawl and Apify were tested (not assumed available)
2. Test requests were actually sent (not simulated)
3. Error messages are specific (not "tool failed")
4. If tools unavailable, blocking_issues lists specific resolution steps
5. If tools available, actor IDs captured for Layer 1 use

---

## Dependencies

- Firecrawl MCP server configured and running
- Apify MCP server configured and running

---

## Error Handling

If BOTH tools fail:
  → Log comprehensive failure report
  → Provide setup instructions for both servers
  → HALT execution — cannot proceed without tools
  → Gate 0 CANNOT pass

If ONE tool fails:
  → Log which tool failed
  → Identify fallback strategies (e.g., Firecrawl can partially replace Apify for some tasks)
  → Human decision: proceed with reduced capability OR fix the failing tool
