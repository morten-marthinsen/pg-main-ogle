# 1.1: Meta Package Builder

## Purpose

Package all Meta-targeted (Facebook + Instagram) variants into the correct directory structure with correct file specifications, correct naming convention, and correct placement targeting. Organize files into Feed (1:1 + 4:5), Stories (9:16), and Reels (9:16) subdirectories. Verify every file meets Meta's exact technical specifications. Generate a per-variant spec sheet. Produce META-PACKAGE-MANIFEST.md listing every file with its complete specifications.

This is a MECHANICAL ASSEMBLY operation -- no creative decisions are made here. Files are organized, renamed, verified, and documented.

## Inputs

- 0.2-variant-matrix-loader.md output (approved variants targeting Meta, with component IDs)
- 0.3-production-assets-loader.md output (asset file paths and preliminary specs)
- 0.4-format-strategy-loader.md output (Meta placements in scope: feed, stories, reels)
- AGENT.md PLATFORM SPECIFICATION REFERENCE -- Meta section (hard technical requirements)
- AGENT.md NAMING CONVENTION section (file naming and in-platform naming rules)

## Process

```
STEP 1: IDENTIFY META-TARGETED VARIANTS
  - Filter approved variant list for variants targeting Meta
  - Group by placement: feed, stories, reels
  - IF no variants target Meta: Output N/A file, skip remaining steps
  - Count: total variants, per-placement breakdown

STEP 2: CREATE DIRECTORY STRUCTURE
  - Create LAUNCH-PACKAGE/meta/feed/
  - Create LAUNCH-PACKAGE/meta/stories/
  - Create LAUNCH-PACKAGE/meta/reels/

STEP 3: PROCESS EACH VARIANT
  FOR EACH Meta-targeted approved variant:

    3a. LOCATE SOURCE ASSET
      - Find the production asset file(s) from A08 AD-ASSETS/
      - IF asset not found: FLAG as missing (should have been caught in 0.3)

    3b. VERIFY FILE SPECIFICATIONS
      VIDEO:
        - Codec: Must be H.264
        - Container: Must be MP4
        - File size: Must be < 4GB
        - Duration: Feed = max 241 min, Stories/Reels = max 120s
        - Aspect ratio: Feed = 1:1 (1080x1080) or 4:5 (1080x1350),
                        Stories = 9:16 (1080x1920), Reels = 9:16 (1080x1920)
        - Resolution: Min 1080px on shortest side
        - Frame rate: 30fps recommended
        - Audio: AAC, 128kbps+ stereo

      STATIC IMAGE:
        - Format: Must be JPG or PNG
        - File size: Must be < 30MB
        - Aspect ratio: 1:1 (1080x1080), 4:5 (1080x1350), or 9:16 (1080x1920)
        - Text overlay: Note if > 20% of image area (affects distribution)

      CAROUSEL:
        - 2-10 cards
        - All cards same aspect ratio
        - Per-card specs same as static image

      IF ANY SPEC FAILS: Flag file with specific failure reason
        -> DO NOT include non-compliant files in the package
        -> Return to A08/A09 for re-export at correct specs

    3c. APPLY FILE NAMING CONVENTION
      - Rename file to: META-[CONCEPT]-[HOOK]-[BODY]-[CTA]-[VISUAL]-[PLACEMENT].[ext]
      - Example: META-C1-HK03-BD01-CT02-VIS-A-FEED.mp4
      - Record mapping: original file name -> new file name

    3d. GENERATE IN-PLATFORM AD NAME
      - Format: [Date]-[Concept]-[HookType]-[BodyVersion]-[CTAType]-[Visual]-[Notes]
      - Example: 0222-HealthMech-CuriosityGap-PAS-Urgency-UGC-Phase1
      - Record mapping: file name -> in-platform name

    3e. COPY TO TARGET DIRECTORY
      - Place renamed file in correct placement subdirectory:
        - Feed variants -> LAUNCH-PACKAGE/meta/feed/
        - Stories variants -> LAUNCH-PACKAGE/meta/stories/
        - Reels variants -> LAUNCH-PACKAGE/meta/reels/

    3f. RECORD PER-VARIANT SPEC SHEET
      - File name, original file name, format, codec, resolution,
        aspect ratio, duration, file size, audio format, placement,
        spec compliance status (PASS/FAIL per check)

STEP 4: GENERATE META-PACKAGE-MANIFEST.md
  - Create LAUNCH-PACKAGE/meta/META-PACKAGE-MANIFEST.md
  - Include:
    - Total files packaged
    - Per-placement file counts
    - Complete per-file spec sheet table
    - File naming cross-reference (new name -> original name -> variant ID)
    - In-platform naming cross-reference
    - Any flagged files (spec failures or missing assets)

STEP 5: WRITE OUTPUT
  - Produce per-microskill output file at designated path
```
