# 1.6: Spec Compliance Validator

## Purpose

Validate every packaged file across ALL platforms against the exact platform specifications from the AGENT.md PLATFORM SPECIFICATION REFERENCE. This is the FINAL spec check before Gate 1 closes. Every file must pass every applicable spec check. ANY file that fails is flagged and excluded from the launch package until fixed. 100% compliance is required -- there is no "close enough."

An ad that cannot upload to the platform literally does not exist. This microskill prevents wasted launches, rejected uploads, and campaign delays.

## Inputs

- 1.1-meta-package-builder.md output (Meta files with per-file specs)
- 1.2-tiktok-package-builder.md output (TikTok files with per-file specs)
- 1.3-youtube-package-builder.md output (YouTube files with per-file specs)
- 1.4-google-display-package-builder.md output (Google files with per-file specs)
- AGENT.md PLATFORM SPECIFICATION REFERENCE (authoritative spec requirements)
- All packaged files in LAUNCH-PACKAGE/ directories

## Process

```
STEP 1: COMPILE COMPLETE FILE LIST
  - Collect all files from all platform directories
  - Cross-reference with platform builder outputs
  - Verify: file count in directories matches file count in builder outputs
  - IF mismatch: HALT -- investigate missing or extra files

STEP 2: VALIDATE EACH FILE AGAINST PLATFORM SPECS
  FOR EACH FILE in the launch package:

    2a. IDENTIFY TARGET PLATFORM AND PLACEMENT
      - From file name: extract platform and placement
      - Look up spec requirements for that platform + placement combination

    2b. RUN SPEC CHECKS (per AGENT.md PLATFORM SPECIFICATION REFERENCE)

      IF VIDEO FILE:
        CHECK 1: File format matches platform requirement
          - Meta: MP4 only
          - TikTok: MP4 or MOV
          - YouTube: MP4 recommended (MOV, AVI, WMV, FLV accepted)
          -> PASS / FAIL with [expected] vs [actual]

        CHECK 2: Resolution matches target placement
          - Meta Feed: 1080x1080 (1:1) or 1080x1350 (4:5)
          - Meta Stories/Reels: 1080x1920 (9:16)
          - TikTok: 1080x1920 (9:16), min 720x1280
          - YouTube Standard: 1920x1080 (16:9)
          - YouTube Shorts: 1080x1920 (9:16)
          -> PASS / FAIL with [expected] vs [actual]

        CHECK 3: File size is under platform maximum
          - Meta: < 4GB
          - TikTok: < 500MB
          - YouTube: < 128GB
          -> PASS / FAIL with [max] vs [actual]

        CHECK 4: Duration is within platform limits
          - Meta Feed: max 241 min
          - Meta Stories/Reels: max 120s
          - TikTok: 5-60 seconds
          - YouTube Pre-roll: 15-30s
          - YouTube Bumper: max 6s (HARD LIMIT)
          - YouTube Skippable: any length
          - YouTube Shorts: max 60s
          -> PASS / FAIL with [limit] vs [actual]

        CHECK 5: Aspect ratio matches target placement
          - Calculate from resolution
          -> PASS / FAIL with [expected] vs [actual]

        CHECK 6: Codec matches platform requirement
          - Meta: H.264
          - TikTok: H.264 or H.265
          - YouTube: H.264 recommended, VP9 accepted
          -> PASS / FAIL with [expected] vs [actual]

        CHECK 7: Audio format and bitrate
          - Meta: AAC, 128kbps+ stereo
          - TikTok: AAC or MP3
          - YouTube: AAC, 48kHz
          -> PASS / FAIL with [expected] vs [actual]

      IF STATIC IMAGE FILE:
        CHECK 1: Format (JPG/PNG per platform)
        CHECK 2: Resolution (exact dimensions for placement)
        CHECK 3: File size (Meta <30MB, TikTok <100MB, Google standard <150KB, Google responsive <5120KB)
        CHECK 4: Aspect ratio
        -> PASS / FAIL per check

      IF THUMBNAIL (YouTube):
        CHECK 1: Resolution = 1280x720
        CHECK 2: Format = JPG, PNG, GIF, or BMP
        CHECK 3: File size < 2MB
        CHECK 4: Aspect ratio = 16:9
        -> PASS / FAIL per check

STEP 3: COMPILE SPEC COMPLIANCE REPORT
  - Per-platform summary: total files, pass count, fail count, % compliant
  - Per-file detail: every check result with expected vs. actual values
  - Overall verdict: PASS (100% compliant) or FAIL (any file non-compliant)

STEP 4: PRODUCE MANDATORY SPEC COMPLIANCE CHECK TABLE
  (From AGENT.md MANDATORY SPEC COMPLIANCE CHECK format)

  +------------------------------------------------------------------------+
  | PLATFORM   | TOTAL FILES | PASS | FAIL | % COMPLIANT | STATUS          |
  +------------+-------------+------+------+-------------+-----------------+
  | Meta       | [X]         | [X]  | [X]  | [X]%        | PASS/FAIL       |
  | TikTok     | [X]         | [X]  | [X]  | [X]%        | PASS/FAIL       |
  | YouTube    | [X]         | [X]  | [X]  | [X]%        | PASS/FAIL       |
  | Google     | [X]         | [X]  | [X]  | [X]%        | PASS/FAIL       |
  +------------------------------------------------------------------------+
  REQUIRED: 100% compliant on ALL platforms. Any file failure = overall FAIL.
  OVERALL: [PASS - proceed] or [FAIL - fix non-compliant files]

STEP 5: FLAG NON-COMPLIANT FILES
  FOR EACH file that FAILED any check:
    - Record: file name, platform, failed check(s), expected vs. actual, required fix
    - Mark file for exclusion from launch package
    - Determine: can this be fixed by re-export (A08) or requires re-creation?

STEP 6: WRITE SPEC-COMPLIANCE-REPORT.md (Deliverable)
  - Write to LAUNCH-PACKAGE/documentation/SPEC-COMPLIANCE-REPORT.md

STEP 7: WRITE PER-MICROSKILL OUTPUT
  - Produce per-microskill output file at designated path
```

## Quality Checks

1. Every file in the launch package is validated (zero files skipped)
2. All 7 video checks run for every video file (format, resolution, size, duration, aspect, codec, audio)
3. All 4 image checks run for every image file (format, resolution, size, aspect)
4. All 4 thumbnail checks run for every YouTube thumbnail
5. Google Display standard files checked against STRICT 150KB limit
6. Per-platform summary matches per-file detail (math is correct)
7. Non-compliant files have specific fix instructions
8. SPEC-COMPLIANCE-REPORT.md written to deliverable location
9. Overall verdict is binary: 100% = PASS, anything less = FAIL
10. Output file meets minimum 3KB size
