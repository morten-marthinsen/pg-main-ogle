# 2.5: Layer 2 Validator

## Purpose

Validate all campaign structure outputs from Layer 2 for completeness, consistency, and cross-referential integrity. Every platform must have a campaign hierarchy. Audience targeting must cover at least Broad + one other tier. Budget plan must be realistic and add up. Testing methodology must have a clear Phase 1 priority order. All variants must be assigned to campaigns/ad sets. Variant count in campaign structure must match variant count from Layer 1.

This is the final checkpoint before Gate 2 closes. It catches errors, omissions, and inconsistencies that individual microskills might miss.

## Inputs

- 2.1-campaign-hierarchy-architect.md output
- 2.2-audience-targeting-strategy.md output
- 2.3-budget-distribution-planner.md output
- 2.4-testing-methodology-planner.md output
- Layer 1 outputs (variant counts per platform for cross-reference)

## Process

```
STEP 1: VALIDATE CAMPAIGN HIERARCHY (2.1)
  - Check: every target platform has a defined campaign hierarchy
  - Check: hierarchy includes Campaign, Ad Set/Group, and Ad levels
  - Check: every approved variant is assigned to a specific campaign + ad set + ad position
  - Check: no orphaned variants (approved but not in any campaign)
  - Check: no orphaned ad positions (positions without assigned variants)
  - Check: variant count in campaign hierarchy matches variant count from Layer 1
  - IF mismatch: FLAG with specific discrepancy details

STEP 2: VALIDATE AUDIENCE TARGETING (2.2)
  - Check: at least 2 targeting tiers defined (minimum: Broad + one other)
  - Check: all target platforms have audience strategy
  - Check: interest stacks are specific (not just generic category names)
  - Check: client data requirements are identified (for Lookalike and Retargeting)
  - Check: exclusion recommendations are present
  - Check: targeting tiers map to ad sets in campaign hierarchy (2.1 cross-reference)
  - IF targeting tiers don't map to ad sets: FLAG inconsistency

STEP 3: VALIDATE BUDGET DISTRIBUTION (2.3)
  - Check: budget percentages sum to 100% (arithmetic verification)
  - Check: all target platforms have budget allocation (no $0 platforms)
  - Check: Phase 1 daily total = sum of per-platform daily amounts
  - Check: Phase 1 7-day total = daily x 7
  - Check: budget is expressed in absolute amounts (not just percentages)
  - Check: kill/scale criteria are defined with specific thresholds
  - Check: minimum spend per variant is defined
  - Check: budget is sufficient for minimum viable testing
    (minimum: each Phase 1 variant gets 3x AOV spend within 7 days)
  - IF math errors: FLAG with specific calculation problems

STEP 4: VALIDATE TESTING METHODOLOGY (2.4)
  - Check: Phase 1 priority order is clear (hooks first, then bodies, etc.)
  - Check: test groups properly isolate one variable at a time
  - Check: win criteria are defined (statistical or practical significance)
  - Check: test cadence is defined with specific checkpoints
  - Check: Phase 1 variants are identified (which go live first)
  - Check: test groups map to campaign structure (2.1 cross-reference)
  - IF test groups don't align with campaign structure: FLAG

STEP 5: CROSS-REFERENCE CHECKS
  - Variant count consistency:
    - Layer 1 total files: [X]
    - Campaign hierarchy total ad positions: [X]
    - Must match (accounting for platform-specific variants)
  - Platform consistency:
    - Layer 1 platforms: [list]
    - Campaign hierarchy platforms: [list]
    - Audience strategy platforms: [list]
    - Budget plan platforms: [list]
    - All must match
  - Budget-to-test alignment:
    - Phase 1 budget per variant >= 3x AOV (minimum viable)
    - If not: FLAG as insufficient for reliable testing

STEP 6: COMPILE VALIDATION VERDICT
  - IF all checks pass: VERDICT = PASS -> Gate 2 can be created
  - IF critical failures: VERDICT = FAIL with specific reasons
  - Critical failures:
    - Platform missing from any strategy document
    - Variant count mismatch
    - Budget math errors
    - No test priority defined
  - Non-critical warnings:
    - Budget below optimal (but meets minimum)
    - Only 2 targeting tiers (minimum met but more recommended)
    - Test groups don't perfectly isolate variables (flagged but not blocking)

STEP 7: WRITE CAMPAIGN-STRUCTURE-VALIDATION.md (Deliverable)
  - Write to LAUNCH-PACKAGE/documentation/ (or project outputs)

STEP 8: WRITE PER-MICROSKILL OUTPUT
  - Produce per-microskill output file at designated path
```
