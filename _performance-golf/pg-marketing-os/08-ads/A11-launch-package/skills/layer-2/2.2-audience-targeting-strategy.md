# 2.2: Audience Targeting Strategy

## Purpose

Define audience targeting suggestions per platform across four targeting tiers: Broad, Interest-based, Lookalike/Similar, and Retargeting. For each tier, define the audience parameters, estimated audience size, rationale for inclusion, and which ad set it maps to in the campaign hierarchy (from 2.1). Include exclusion recommendations and identify which tiers require client-provided data.

A11 RECOMMENDS targeting. The media buyer DECIDES. Every suggestion includes rationale.

## Inputs

- 2.1-campaign-hierarchy-architect.md output (ad set structure to map audiences to)
- 0.0.1-vertical-profile-loader.md output (vertical-specific audience insights)
- 0.4-format-strategy-loader.md output (target platforms)
- Campaign Brief (target audience, product, market, customer profile)
- A01/Research outputs (if available -- audience language, pain points, demographics)

## Process

```
STEP 1: DEFINE FOUR TARGETING TIERS

  TIER 1 -- BROAD (No Targeting)
    - Platform: [all platforms]
    - Parameters: No interest or demographic restrictions. Let the algorithm find the audience.
    - Rationale: Algorithms (especially Meta's) often outperform manual targeting when given
      enough data. Broad is the recommended starting point for new campaigns.
    - Estimated audience size: [maximum available for platform + geo]
    - Maps to: [ad set from 2.1 hierarchy]
    - Client data required: No

  TIER 2 -- INTEREST-BASED
    FOR EACH target platform:
      - Define interest stacks (vertical-specific):
        - Interest Stack A: [core vertical interests]
          - List specific interests available in the platform's targeting system
          - Example (health): health & wellness, supplements, natural remedies, specific conditions
          - Example (golf): golf, golf equipment, Golf Digest, PGA Tour, specific brands
          - Example (finance): investing, personal finance, stock market, specific publications
        - Interest Stack B: [adjacent/related interests]
          - Interests correlated with the target audience but not directly about the product
          - Example: age-related interests, lifestyle interests, complementary hobbies
      - Estimated audience size per stack
      - Rationale: why these specific interests for this vertical + product
      - Maps to: [ad sets from 2.1 hierarchy]
      - Client data required: No

  TIER 3 -- LOOKALIKE / SIMILAR AUDIENCES
    FOR EACH target platform:
      - Source audience options:
        - Lookalike from customer list (1%, 2%, 3% -- recommend starting at 1%)
        - Lookalike from purchasers (if pixel has purchase data)
        - Lookalike from email list
        - Similar audience from website visitors
      - Recommended source: [which source is best for this campaign and why]
      - Recommended percentage: [1-3% range with rationale]
      - Maps to: [ad set from 2.1 hierarchy]
      - Client data required: YES -- specify what data is needed
        -> Customer email list, purchase history, pixel with conversion data

  TIER 4 -- RETARGETING
    FOR EACH target platform:
      - Retargeting audience segments:
        - Website visitors (last 30/60/90/180 days)
        - Video viewers (25%, 50%, 75%, 95% view)
        - Page engagers (liked, commented, shared)
        - Cart abandoners (if e-commerce)
        - Email list (opened but not purchased)
      - Recommended windows and segments
      - Separate messaging strategy note (retargeting audience has different awareness level)
      - Maps to: [ad set from 2.1 hierarchy]
      - Client data required: YES -- pixel installed and firing, conversion events set up

STEP 2: DEFINE EXCLUSION RECOMMENDATIONS
  - Existing customers (exclude from prospecting campaigns to avoid wasted spend)
  - Recent purchasers (30-60 day window)
  - Age/geo restrictions (if product not available in certain regions or for certain ages)
  - Brand safety exclusions (content categories to avoid)
  - Custom exclusions (specific to this campaign)

STEP 3: RECOMMEND TESTING ORDER
  - Which tiers to test first (typically: Broad + Interest A simultaneously)
  - Which tiers to add later (Lookalike after pixel has data, Retargeting as audiences build)
  - Expected timeline for tier deployment

STEP 4: IDENTIFY CLIENT DATA REQUIREMENTS
  - Compile complete list of data needed from client:
    - Customer email list (for lookalike creation)
    - Pixel installation status (for retargeting and lookalike)
    - Conversion event setup (for optimization)
    - Attribution window preference
  - These become checklist items in the launch checklist (3.4)

STEP 5: WRITE AUDIENCE-STRATEGY.md (Deliverable)
  - Write to LAUNCH-PACKAGE/documentation/AUDIENCE-STRATEGY.md

STEP 6: WRITE PER-MICROSKILL OUTPUT
  - Produce per-microskill output file at designated path
```
