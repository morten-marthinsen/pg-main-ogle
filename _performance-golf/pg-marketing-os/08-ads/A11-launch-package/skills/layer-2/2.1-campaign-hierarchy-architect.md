# 2.1: Campaign Hierarchy Architect

## Purpose

Define the Campaign -> Ad Set -> Ad hierarchy for each target platform. This is the STRATEGIC ARCHITECTURE of how the media buyer will organize ads inside the platform's ad manager. The hierarchy determines how budgets flow, how audiences are tested, how creative variants are structured, and how the algorithm optimizes performance. Wrong hierarchy = wasted budget, confounded tests, and poor optimization.

A11 RECOMMENDS campaign structure. The media buyer DECIDES. Every recommendation includes rationale so the media buyer can accept, modify, or override with informed judgment.

## Inputs

- 0.1-scoring-report-loader.md output (approved variants, scores, test priority)
- 0.2-variant-matrix-loader.md output (variant matrix with component IDs)
- 0.4-format-strategy-loader.md output (target platforms and placements)
- 1.5-naming-key-generator.md output (variant-to-file mapping)
- Campaign Brief (product, funnel, objectives)
- AGENT.md CAMPAIGN STRUCTURE REFERENCE section (Meta, TikTok, YouTube, Google architectures)

## Process

```
STEP 1: DETERMINE CAMPAIGN OBJECTIVES PER PLATFORM
  - For each target platform, recommend campaign objective:
    - Meta: Conversions (if pixel has data) or Sales. If new pixel: Traffic first, then Conversions.
    - TikTok: Website Conversions or Complete Payment. If new pixel: Traffic first.
    - YouTube: Video Views (skippable, in-feed) or Video Reach (bumper, non-skippable).
    - Google Display: Display campaign or Performance Max (for automated).
  - Include rationale for objective selection
  - Note: if pixel/conversion data is insufficient, recommend starting with traffic objective

STEP 2: DESIGN META CAMPAIGN HIERARCHY
  IF Meta is a target platform:

  CAMPAIGN LEVEL:
    - Recommend CBO (Campaign Budget Optimization) vs. ABO (Ad Set Budget Optimization)
      - CBO recommended for testing phase (algorithm distributes budget to best performers)
      - ABO recommended when you need equal spend per audience test
    - Campaign grouping logic:
      - Option A: One campaign per concept (if testing concepts)
      - Option B: One campaign per objective (if testing audiences)
      - Option C: One campaign per funnel stage (awareness vs. conversion)
    - Recommend the best option with rationale

  AD SET LEVEL:
    - Grouping logic: by audience targeting tier
      - Ad Set 1: Broad (no targeting -- let algorithm optimize)
      - Ad Set 2: Interest Stack A (vertical-specific interests -- list specific interests)
      - Ad Set 3: Interest Stack B (related/adjacent interests -- list specific interests)
      - Ad Set 4: Lookalike 1-3% (if client has customer list or pixel data)
      - Ad Set 5: Retargeting (website visitors, video viewers, engagers)
    - Map which variants go into which ad sets

  AD LEVEL:
    - Each ad = one variant from the scored matrix
    - Recommend: Dynamic Creative (automated hook/body/CTA testing by Meta's algorithm)
      vs. individual ads (manual control)
    - Map specific variant IDs to specific ad positions

STEP 3: DESIGN TIKTOK CAMPAIGN HIERARCHY
  IF TikTok is a target platform:

  CAMPAIGN LEVEL:
    - Objective: Website Conversions or Complete Payment
    - Budget: Campaign-level (TikTok's equivalent to CBO)

  AD GROUP LEVEL:
    - Ad Group 1: Broad (no targeting)
    - Ad Group 2: Interest-based (list specific TikTok interests for this vertical)
    - Ad Group 3: Custom audience (if client data available)
    - Map variants to ad groups

  AD LEVEL:
    - Recommend Spark Ads format for UGC-style creative
    - Recommend standard format for polished creative
    - Map specific variants to ad positions

STEP 4: DESIGN YOUTUBE CAMPAIGN HIERARCHY
  IF YouTube is a target platform:

  CAMPAIGN LEVEL:
    - Types by ad format:
      - Video Reach campaign (bumper ads, non-skippable in-stream)
      - Video Views campaign (skippable in-stream, in-feed video)
    - Separate campaigns by ad type (they have different optimization goals)

  AD GROUP LEVEL:
    - Group by audience + format combination
    - Map targeting to ad groups

  AD LEVEL:
    - One ad per video asset
    - Include companion banner recommendation
    - Custom thumbnail required (from 1.3 package)
    - Map specific variants to ad positions

STEP 5: DESIGN GOOGLE DISPLAY CAMPAIGN HIERARCHY
  IF Google Display is a target platform:

  CAMPAIGN LEVEL:
    - Campaign type: Display (manual) or Performance Max (automated)
    - Recommend Performance Max for broader reach, Display for more control

  AD GROUP LEVEL:
    - Group by audience targeting
    - Responsive Display Ads recommended (Google assembles optimal combinations)

  AD LEVEL:
    - Responsive: upload all image assets, Google auto-assembles
    - Standard: individual banner sizes per ad group
    - Map assets to ad positions

STEP 6: PRODUCE CAMPAIGN-HIERARCHY.md
  - Write to LAUNCH-PACKAGE/documentation/CAMPAIGN-HIERARCHY.md
  - Include full hierarchy diagrams for each platform
  - Include rationale for every grouping decision
  - Include variant-to-ad mapping table
  - Include CBO vs. ABO recommendation with reasoning

STEP 7: WRITE PER-MICROSKILL OUTPUT
  - Produce per-microskill output file at designated path
```
