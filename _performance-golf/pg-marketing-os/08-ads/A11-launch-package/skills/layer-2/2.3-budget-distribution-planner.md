# 2.3: Budget Distribution Planner

## Purpose

Recommend budget allocation across platforms, campaigns, and test phases. Define Phase 1 (Testing) per-platform daily budget, duration, kill thresholds, and success thresholds. Define Phase 2 (Scaling) budget increase methodology. Define Phase 3 (Optimization) ongoing reallocation cadence. Include minimum spend per variant before making kill/scale decisions.

A11 RECOMMENDS budget distribution. The media buyer DECIDES. Every recommendation includes rationale.

## Inputs

- 2.1-campaign-hierarchy-architect.md output (campaign structure to allocate budgets to)
- 0.1-scoring-report-loader.md output (variant count, test priority)
- 0.4-format-strategy-loader.md output (target platforms, platform priorities)
- Campaign Brief (total budget if provided, product price/AOV, target CPA/ROAS)
- AGENT.md BUDGET DISTRIBUTION REFERENCE section (default splits, kill/scale framework)
- AD-ENGINE.md testing volume reference data (if available)

## Process

```
STEP 1: DETERMINE BUDGET CONTEXT
  - Check: has the client provided a total budget?
    - IF YES: work from that number
    - IF NO: recommend a starting budget based on variant count and platform count
      - Rule of thumb: minimum $50-100/day per platform for testing phase
      - With 4 platforms and 3-5 variants per platform: recommend $200-400/day minimum
  - Extract product price / AOV (Average Order Value) from brief
  - Extract target CPA or target ROAS from brief (if available)
  - IF no CPA/ROAS target: note as "to be determined from Phase 1 data"

STEP 2: ALLOCATE BUDGET ACROSS PLATFORMS
  - Use AGENT.md default budget split as starting point:
    - Meta: 40-60% (largest audience, best targeting, most mature optimization)
    - TikTok: 15-30% (growing fast, lower CPMs, requires native creative)
    - YouTube: 10-25% (higher CPCs but higher intent)
    - Google Display: 5-15% (best as supplement to social, not primary)
  - Adjust based on:
    - Vertical (health supplements: Meta-heavy; younger demo products: TikTok-heavy)
    - Platform priority from A03 FORMAT-STRATEGY.md
    - Number of variants available per platform
    - Client preferences/history
  - Express as both percentages AND absolute daily dollar amounts
  - ALL PERCENTAGES MUST ADD UP TO 100% (budget math enforcement)

STEP 3: DEFINE PHASE 1 -- TESTING BUDGET
  Duration: 7 days (recommended)
  FOR EACH platform:
    - Daily budget: $[X] per day
    - Budget allocation method: CBO (campaign level) or ABO (ad set level)
    - Per-variant minimum spend before kill/scale decision:
      - Minimum impressions: 1,000 (for hook rate evaluation)
      - Minimum link clicks: 50 (for CTR evaluation)
      - Minimum spend: 3x AOV (for CPA evaluation)
      - Minimum time: 72 hours (for algorithm learning)
    - Kill criteria (from AGENT.md Kill/Scale Decision Framework):
      - Spent 3x AOV with 0 conversions -> KILL
      - CPA > 2x target CPA after minimum sample -> KILL
      - Hook rate < 15% (below industry minimum) -> KILL
      - CTR < 0.5% (not generating interest) -> KILL
    - Success criteria:
      - CPA at or below target CPA after minimum sample -> SCALE candidate
      - ROAS at or above target ROAS -> SCALE candidate
      - Hook rate > 30% (strong attention) -> SCALE candidate
      - Consistent performance over 3+ days -> SCALE confirmation
  Total Phase 1 budget: $[X] over 7 days

STEP 4: DEFINE PHASE 2 -- SCALING BUDGET
  Duration: Days 7-14
  - Scale methodology (from AGENT.md):
    - Increase budget 20-30% per increment
    - Wait 24-48h between increases
    - If CPA rises after increase, hold for 48h before adjusting
    - Never more than double budget in a single week
  - Platform rebalancing:
    - Shift budget toward platforms with best Phase 1 results
    - Reduce budget on platforms with poor Phase 1 results
    - But maintain minimum viable spend on all active platforms
  - Scaling decision criteria:
    - Winner identified with 3+ days of consistent performance -> scale
    - Multiple winners: allocate proportionally to performance
  Total Phase 2 budget: $[X] over 7 days (or provide range)

STEP 5: DEFINE PHASE 3 -- OPTIMIZATION BUDGET
  Duration: Days 14-30+
  - Ongoing budget reallocation cadence: weekly review
  - Budget allocation shifts based on performance data
  - Creative refresh triggers:
    - Best variant CPA rises 30%+ from baseline -> creative fatigue, refresh needed
    - CTR decline of 20%+ over 7 days -> audience saturation
  - When to request new creative from A01-A10 pipeline:
    - All variants below target CPA
    - Creative fatigue across all platforms
    - New audience segments identified for expansion
  Total Phase 3 budget: [ongoing, based on Phase 2 learnings]

STEP 6: PRODUCE BUDGET SUMMARY TABLE
  | Platform | Phase 1 Daily | Phase 1 Total (7d) | Phase 2 Range | Phase 3 (Monthly) |
  |----------|--------------|-------------------|--------------|-----------------|
  - Verify: all platform totals add up to overall total
  - Verify: no platform has $0 if it's a target platform
  - Verify: minimum viable spend thresholds are met per platform

STEP 7: WRITE BUDGET-PLAN.md (Deliverable)
  - Write to LAUNCH-PACKAGE/documentation/BUDGET-PLAN.md

STEP 8: WRITE PER-MICROSKILL OUTPUT
  - Produce per-microskill output file at designated path
```
