# 0.4: Format Strategy Loader

## Purpose

Load the A03 FORMAT-STRATEGY.md and extract the target platforms, platform-specific placement requirements, and format assignments per concept. This determines which platforms the launch package must serve and what placements within each platform are in scope. Without this, the package builder does not know which platform directories to create or which spec requirements to enforce.

## Inputs

- A03 FORMAT-STRATEGY.md (primary input)
- Campaign Brief (may contain platform preferences or constraints)

## Process

```
STEP 1: LOCATE AND READ FORMAT STRATEGY
  - Find A03 FORMAT-STRATEGY.md in the project outputs
  - IF file does not exist: HALT -- cannot proceed without platform targets
  - IF file exists but has no platform assignments: HALT -- A03 must re-run

STEP 2: EXTRACT TARGET PLATFORMS
  - Parse for all platforms designated for this campaign
  - Valid platforms: Meta, TikTok, YouTube, Google Display
  - For EACH target platform, extract:
    - platform_name: [string]
    - priority_level: [primary / secondary / tertiary]
    - placements: [list of specific placements within the platform]
      - Meta: feed, stories, reels
      - TikTok: in-feed, spark_ads
      - YouTube: pre-roll, bumper, skippable, shorts, in-feed
      - Google: responsive, standard display sizes (specific sizes)
    - format_requirements: [video / static / carousel / mixed]
    - concept_assignments: [which concepts target which platform]

STEP 3: EXTRACT PLACEMENT-SPECIFIC REQUIREMENTS
  - For each placement, record the spec requirements that will be enforced:
    - aspect_ratio: [required ratio]
    - resolution: [minimum and recommended]
    - duration: [min/max for video]
    - file_format: [required formats]
    - file_size_limit: [maximum]
  - Cross-reference with AGENT.md PLATFORM SPECIFICATION REFERENCE
  - These feed directly into Layer 1 platform package builders

STEP 4: EXTRACT FORMAT ASSIGNMENTS PER CONCEPT
  - Map concepts to formats:
    - Concept C1 -> [video for Meta feed, static for Google display, etc.]
    - Concept C2 -> [video for TikTok, video for YouTube shorts, etc.]
  - This determines which variants need which format at which placement

STEP 5: COMPILE PLATFORM SCOPE
  - List of all target platforms (determines which Layer 1 microskills run)
  - IF Meta is not a target: 1.1-meta-package-builder is N/A
  - IF TikTok is not a target: 1.2-tiktok-package-builder is N/A
  - IF YouTube is not a target: 1.3-youtube-package-builder is N/A
  - IF Google Display is not a target: 1.4-google-display-package-builder is N/A
  - At least ONE platform must be targeted

STEP 6: WRITE OUTPUT
  - Produce per-microskill output file at designated path
```
