# 0.0.1: Vertical Profile Loader

## Purpose

Load the ad-specific vertical configuration from `ad-verticals/`. Extract compliance constraints specific to the vertical (health claims, financial disclaimers, etc.) that will govern the compliance review in Layer 3 and constrain the launch package documentation in Layer 4.

This is the FIRST microskill that runs -- everything downstream depends on knowing which vertical the campaign operates in, because compliance requirements, platform-specific restrictions, and documentation language all vary by vertical.

## Inputs

- Campaign brief (product name, vertical identification)
- `./ads/ad-verticals/` directory -- contains vertical-specific configuration files
- A03 FORMAT-STRATEGY.md (may contain vertical designation)

## Process

```
STEP 1: IDENTIFY VERTICAL
  - Read campaign brief for vertical designation
  - IF explicit vertical specified: USE that vertical
  - IF not specified: INFER from product category, target audience, compliance landscape
  - Valid verticals: golf, health, finance, personal-dev, technology
  - IF cannot determine: DEFAULT to universal config, LOG warning

STEP 2: LOAD VERTICAL PROFILE
  - Read ad-verticals/[vertical].md
  - Extract:
    - vertical_id: [string]
    - compliance_constraints: [list of vertical-specific compliance rules]
      - health: FDA disclaimer requirements, prohibited claims (cure/treat/prevent),
        before/after photo requirements, testimonial disclaimer requirements
      - finance: SEC/FINRA disclaimer requirements, performance disclaimer,
        "past performance" language, risk disclosure requirements
      - golf/sports: product claim substantiation, endorsement disclosures
      - personal-dev: income claim disclaimers, results-not-typical language
      - technology: accuracy claims, comparison claims, data privacy compliance
    - platform_restrictions: [per-platform restricted category flags]
      - Meta: restricted ad categories that require special authorization
      - TikTok: restricted categories, community guideline intersections
      - YouTube: restricted ad categories
      - Google: restricted content categories
    - anti-slop_patterns: [vertical-specific language to avoid]
    - documentation_language: [vertical-specific terminology for launch guide]

STEP 3: VALIDATE VERTICAL PROFILE
  - Verify compliance_constraints is non-empty
  - Verify platform_restrictions covers all target platforms from A03
  - IF any target platform missing from restrictions: LOG gap, use general restrictions

STEP 4: WRITE OUTPUT
  - Produce per-microskill output file at designated path
```

## Output File

- **Path:** `[project]/A11-launch-package/layer-0-outputs/0.0.1-vertical-profile-loader.md`
- **Minimum size:** 1KB

## Quality Checks

1. Vertical ID is one of the 5 valid verticals (or "universal" with logged warning)
2. Compliance constraints are non-empty and specific (not generic)
3. Platform restrictions cover ALL target platforms from A03 FORMAT-STRATEGY.md
4. Anti-slop patterns are present and vertical-specific
5. Output file meets minimum 1KB size

## Dependencies

- Campaign brief must be available (product name, vertical)
- `ad-verticals/` directory must exist with vertical profile files
- No upstream microskill dependencies (this runs first or in parallel with 0.1)
