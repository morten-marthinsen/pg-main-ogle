# 0.2: Variant Matrix Loader

## Purpose

Load the A09 AD-VARIANT-MATRIX/ and extract the complete variant matrix with all variant IDs, hook-body-CTA-visual combinations, and per-variant file paths to production assets. Cross-reference with the A10 approved variant list (from 0.1) to identify which matrix entries are approved for launch and which are not. This establishes the exact mapping between variant IDs and their component parts that the naming convention and launch package will use.

## Inputs

- A09 AD-VARIANT-MATRIX/ directory (primary input)
- 0.1-scoring-report-loader.md output (approved variant list for cross-referencing)

## Process

```
STEP 1: LOCATE AND READ VARIANT MATRIX
  - Find A09 AD-VARIANT-MATRIX/ in the project outputs
  - Read all matrix files (may be single file or multi-file structure)
  - IF directory does not exist: HALT -- cannot proceed without variant matrix
  - IF directory exists but is empty: HALT -- A09 must re-run

STEP 2: PARSE COMPLETE MATRIX
  - For EACH variant entry, extract:
    - variant_id: [unique identifier]
    - concept_id: [C1, C2, C3 -- maps to winning concept]
    - hook_id: [HK01-HK99 -- maps to A02 HOOK-ANGLE-MATRIX]
    - hook_type: [from AD-HOOK-TAXONOMY.md -- e.g., curiosity_gap, authority_reveal]
    - body_id: [BD01-BD99 -- maps to A04 SCRIPT-PACKAGE]
    - body_framework: [PAS, BAB, AIDA, HookBodyCTA, etc.]
    - cta_id: [CT01-CT99 -- maps to CTA version]
    - cta_type: [Urgency, RiskReversal, LowFriction, SocialProof, etc.]
    - visual_id: [VIS-A, VIS-B, VIS-C -- maps to A05/A08 visual treatment]
    - visual_type: [UGC, Polished, TextOverlay, Demo, Mixed]
    - asset_file_paths: [list of production asset file paths from A08]
    - target_platforms: [list of platforms this variant targets]
    - target_placements: [list of specific placements -- feed, stories, reels, etc.]
    - duration: [seconds, if video]
    - format: [video/static/carousel]

STEP 3: CROSS-REFERENCE WITH APPROVED LIST
  - Load approved variant IDs from 0.1 output
  - For each matrix entry, tag:
    - approved_for_launch: [true/false]
    - a10_score: [score from 0.1 if approved]
    - a10_test_priority: [rank from 0.1 if approved]
  - Verify: every approved variant ID from 0.1 exists in the matrix
  - IF any approved variant is NOT in the matrix: FLAG -- data inconsistency

STEP 4: COMPILE SUMMARY STATISTICS
  - Total matrix entries: [count]
  - Approved entries: [count]
  - Entries per concept: [breakdown]
  - Entries per platform: [breakdown]
  - Entries per placement: [breakdown]
  - Unique hooks used: [count]
  - Unique bodies used: [count]
  - Unique CTAs used: [count]
  - Unique visual treatments used: [count]

STEP 5: WRITE OUTPUT
  - Produce per-microskill output file at designated path
```

## Quality Checks

1. Every approved variant ID from 0.1 exists in the matrix (zero missing)
2. Every matrix entry has complete component IDs (concept, hook, body, CTA, visual)
3. Every matrix entry has at least one asset file path
4. Every matrix entry has target platform(s) and placement(s)
5. Summary statistics are accurate (counts match actual entries)
6. Output file meets minimum 2KB size

## Dependencies

- A09 AD-VARIANT-MATRIX/ must exist
- 0.1-scoring-report-loader.md must be complete (provides approved variant list)
- Runs AFTER 0.1 (needs A10 approved list to cross-reference)
