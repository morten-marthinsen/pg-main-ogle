# 0.5: Input Validator

## Purpose

Validate that ALL required inputs are present, complete, and internally consistent before any packaging work begins. This is the final checkpoint before Gate 0 -- it aggregates the results from all other Layer 0 loaders and makes the binary PASS/FAIL determination. If any critical input is missing or inconsistent, this microskill blocks progression to Layer 1.

## Inputs

- 0.0.1-vertical-profile-loader.md output
- 0.1-scoring-report-loader.md output
- 0.2-variant-matrix-loader.md output
- 0.3-production-assets-loader.md output
- 0.4-format-strategy-loader.md output
- Campaign Brief (existence check)

## Process

```
STEP 1: VERIFY ALL LOADER OUTPUTS EXIST
  - Check: 0.0.1 output file exists and meets minimum size (1KB)
  - Check: 0.1 output file exists and meets minimum size (2KB)
  - Check: 0.2 output file exists and meets minimum size (2KB)
  - Check: 0.3 output file exists and meets minimum size (2KB)
  - Check: 0.4 output file exists and meets minimum size (1KB)
  - IF ANY MISSING: HALT -- cannot validate without all loader outputs

STEP 2: VALIDATE A10 SCORING REPORT
  - From 0.1 output: approved_variants_count >= 1
  - IF 0 approved variants: FAIL -- nothing to package (return to A10)

STEP 3: VALIDATE VARIANT MATRIX
  - From 0.2 output: matrix is loaded with complete variant data
  - Cross-reference: every approved variant ID from 0.1 exists in the matrix
  - IF any approved variant missing from matrix: FAIL -- data inconsistency

STEP 4: VALIDATE PRODUCTION ASSETS
  - From 0.3 output: critical missing assets = 0
  - IF critical_missing > 0: FAIL -- return to A08 for missing assets
  - Note preliminary spec concerns (informational, not blocking)

STEP 5: VALIDATE FORMAT STRATEGY
  - From 0.4 output: target_platforms list is non-empty
  - IF target_platforms is empty: FAIL -- return to A03 for platform decisions
  - Cross-reference: approved variants have assets for their target platforms

STEP 6: VALIDATE CAMPAIGN BRIEF
  - Check: Campaign Brief exists and is accessible
  - IF not available: FAIL -- brief is required for campaign structure and documentation

STEP 7: VALIDATE VERTICAL PROFILE
  - From 0.0.1 output: vertical_id is set
  - From 0.0.1 output: compliance_constraints are non-empty
  - IF vertical not loaded: WARN (not blocking, but compliance may be incomplete)

STEP 8: CROSS-REFERENCE CONSISTENCY
  - Variant count in 0.1 (approved) matches variant count in 0.2 (approved entries)
  - Platform list in 0.4 matches platforms in 0.2 (approved variants' target platforms)
  - All approved variants have at least one production asset in 0.3
  - FLAG any inconsistencies

STEP 9: COMPILE VALIDATION VERDICT
  - IF all checks pass: VERDICT = PASS -> Gate 0 can be created
  - IF any critical check fails: VERDICT = FAIL with specific failure reasons
  - IF only warnings (non-critical): VERDICT = PASS with warnings documented

STEP 10: WRITE OUTPUT
  - Produce per-microskill output file at designated path
```
