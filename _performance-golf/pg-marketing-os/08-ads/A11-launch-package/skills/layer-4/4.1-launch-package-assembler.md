# 4.1: Launch Package Assembler

## Purpose

Assemble the final LAUNCH-PACKAGE/ directory with the complete folder structure specified in the AGENT.md OUTPUT SCHEMA. This is the physical assembly of all verified, validated artifacts from Layers 1-3 into the deliverable directory structure that the media buyer receives. Every platform subdirectory must contain its spec-compliant ad files and manifest. The documentation/ subdirectory must contain all 10 strategic and operational documents. The checkpoints/ subdirectory must contain all gate verification files.

This is an ASSEMBLY operation -- it combines pre-validated artifacts. It does NOT generate new analysis or modify existing content.

## Inputs

- All Layer 1 platform packages (meta/, tiktok/, youtube/, google/ with manifests)
- All Layer 1-3 documentation deliverables:
  - NAMING-KEY.md (from 1.5)
  - SPEC-COMPLIANCE-REPORT.md (from 1.6)
  - CAMPAIGN-HIERARCHY.md (from 2.1)
  - AUDIENCE-STRATEGY.md (from 2.2)
  - BUDGET-PLAN.md (from 2.3)
  - TEST-PLAN.md (from 2.4)
  - COMPLIANCE-REPORT.md (from 3.1)
  - UTM-TRACKING.md (from 3.2)
  - launch-sequence.md (from 3.3)
  - LAUNCH-CHECKLIST.md (from 3.4)
- All checkpoint YAML files (LAYER_0_COMPLETE through LAYER_3_COMPLETE)
- AGENT.md OUTPUT SCHEMA: LAUNCH-PACKAGE/ (target directory structure)

## Process

```
STEP 1: CREATE DIRECTORY STRUCTURE
  Create the complete LAUNCH-PACKAGE/ directory tree:

  LAUNCH-PACKAGE/
  |
  +-- meta/                              # IF Meta is target platform
  |   +-- feed/
  |   +-- stories/
  |   +-- reels/
  |   +-- META-PACKAGE-MANIFEST.md
  |
  +-- tiktok/                            # IF TikTok is target platform
  |   +-- TIKTOK-PACKAGE-MANIFEST.md
  |
  +-- youtube/                           # IF YouTube is target platform
  |   +-- pre-roll/
  |   +-- bumper/
  |   +-- skippable/
  |   +-- shorts/
  |   +-- thumbnails/
  |   +-- YOUTUBE-PACKAGE-MANIFEST.md
  |
  +-- google/                            # IF Google Display is target platform
  |   +-- responsive/
  |   +-- standard/
  |   +-- GOOGLE-PACKAGE-MANIFEST.md
  |
  +-- documentation/
  |
  +-- checkpoints/

  Only create platform directories for TARGET platforms (from 0.4).
  Non-target platforms do not get empty directories.

STEP 2: VERIFY AND PLACE PLATFORM AD FILES
  FOR EACH target platform:
    - Verify all ad files from Layer 1 exist at expected paths
    - Copy/move ad files to correct LAUNCH-PACKAGE/ subdirectories
    - Verify file count matches manifest count
    - Verify naming convention is applied
    - Place platform manifest in correct directory
    - IF any files missing: HALT -- investigate before proceeding

STEP 3: ASSEMBLE DOCUMENTATION DIRECTORY
  Place all 10 documentation files in LAUNCH-PACKAGE/documentation/:
    1. NAMING-KEY.md (from 1.5)
    2. CAMPAIGN-HIERARCHY.md (from 2.1)
    3. AUDIENCE-STRATEGY.md (from 2.2)
    4. BUDGET-PLAN.md (from 2.3)
    5. TEST-PLAN.md (from 2.4)
    6. launch-sequence.md (from 3.3)
    7. LAUNCH-CHECKLIST.md (from 3.4)
    8. UTM-TRACKING.md (from 3.2)
    9. COMPLIANCE-REPORT.md (from 3.1)
    10. SPEC-COMPLIANCE-REPORT.md (from 1.6)

  FOR EACH file:
    - Verify file exists
    - Verify file is non-empty
    - Record file size
    - IF any file missing: HALT -- generate missing documentation

STEP 4: ASSEMBLE CHECKPOINTS DIRECTORY
  Place all checkpoint YAML files in LAUNCH-PACKAGE/checkpoints/:
    - LAYER_0_COMPLETE.yaml
    - LAYER_1_COMPLETE.yaml
    - LAYER_2_COMPLETE.yaml
    - LAYER_3_COMPLETE.yaml
    (LAYER_4_COMPLETE.yaml will be created by 4.3 after assembly is verified)

STEP 5: VERIFY DIRECTORY STRUCTURE
  Walk the complete LAUNCH-PACKAGE/ directory:
    - Count total files (ad files + documentation + checkpoints + manifests)
    - Verify no empty platform directories (for target platforms)
    - Verify documentation/ has all 10 files
    - Verify checkpoints/ has Layers 0-3 (Layer 4 added later)
    - Compare actual directory tree to AGENT.md OUTPUT SCHEMA
    - Flag any discrepancies

STEP 6: PRODUCE DIRECTORY INVENTORY
  Complete tree listing with file sizes:
    - Total ad files: [count]
    - Total documentation files: [count]
    - Total checkpoint files: [count]
    - Total manifest files: [count]
    - Grand total files: [count]
    - Grand total size: [MB/GB]

STEP 7: WRITE PER-MICROSKILL OUTPUT
  - Produce per-microskill output file at designated path
```
