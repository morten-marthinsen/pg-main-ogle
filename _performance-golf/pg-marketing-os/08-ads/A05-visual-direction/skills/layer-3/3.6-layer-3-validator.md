# 3.6: Layer 3 Validator

## Purpose
Validate all Layer 3 production specifications for completeness and correctness. Confirm: every concept has tool-specific specs for all required production methods, Midjourney prompts are syntactically valid (correct parameter format), Arcads briefs cover all required fields, ElevenLabs specs are complete for all voiceover concepts, stock footage queries are specific enough to return relevant results, platform export specs match A03 format strategy requirements. This is the quality gate before output packaging.

## Inputs
- All Layer 3 outputs: 3.1 (Midjourney), 3.2 (Arcads), 3.3 (ElevenLabs), 3.4 (stock), 3.5 (platform)
- Script inventory from 0.1 (for completeness check)
- Format strategy from 0.2 (for platform compliance)

## Process
1. READ all Layer 3 outputs (3.1 through 3.5)
2. READ script inventory from 0.1
3. READ format strategy from 0.2
4. VALIDATE Midjourney prompts (from 3.1):
   a. All required images have prompts: [Y/N]
   b. Prompt syntax validation:

      - `--no` properly formatted: [Y/N]
      - No invalid parameters: [Y/N]
   c. 2-3 variants per image: [Y/N]
   d. Result: [PASS/FAIL]
5. VALIDATE Arcads briefs (from 3.2):
   a. All AI talent concepts have briefs: [Y/N]
   b. Required fields populated:
      - Actor demographics: [Y/N]
      - Emotion settings: [Y/N]
      - Script text (verbatim): [Y/N]
      - Delivery parameters: [Y/N]
      - Visual parameters: [Y/N]
   c. Result: [PASS/FAIL]
6. VALIDATE ElevenLabs specs (from 3.3):
   a. All voiceover concepts have specs: [Y/N]
   b. Voice character defined: [Y/N]
   c. Pronunciation notes for all brand/product terms: [Y/N]
   d. Emotional arc mapped: [Y/N]
   e. Parameters within valid ranges: [Y/N]
   f. Result: [PASS/FAIL]
7. VALIDATE stock footage queries (from 3.4):
   a. All stock footage needs have queries: [Y/N]
   b. 2+ query variants per clip: [Y/N]
   c. Queries are specific (not generic): [Y/N]

8. VALIDATE platform export specs (from 3.5):
   a. All target platforms covered: [Y/N]
   b. Resolution at pixel level: [Y/N]
   c. Safe zones at pixel level: [Y/N]

   - Tool specs reference same concepts as shot lists
10. DETERMINE gate status: PASS or FAIL
11. WRITE validation results to output file
