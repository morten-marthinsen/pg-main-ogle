# 2.1: Shot List Generator

## Purpose
For each script, generate a complete shot list. For EVERY beat in the script, specify: shot number, shot type (CU/MCU/MS/MWS/WS/ECU), subject description (specific -- not "person" but "woman, 35-45, natural hair, casual clothes, kitchen setting"), action (what happens in frame), duration (seconds), framing notes (rule of thirds placement, headroom, leading space), text overlay (exact words, position, style), transition to next shot (cut/dissolve/swipe/zoom). This is the FOUNDATIONAL Layer 2 microskill -- all other Layer 2 work references the shot list.

## Inputs
- Script inventory with beat structures from 0.1
- Treatment type assignments from 1.1 (treatment affects shot style)
- Platform constraints from 0.2 (safe zones, aspect ratios)
- Color palettes from 1.2 (color direction for shots)
- Typography system from 1.3 (text overlay specifications)

## Process
1. READ script inventory with beat structures from 0.1
2. READ treatment type assignments from 1.1
3. READ platform constraints from 0.2
4. READ color palette and typography from 1.2, 1.3
5. FOR EACH script:
   a. IDENTIFY treatment type (drives shot style)
   b. IDENTIFY platform (drives aspect ratio, safe zones)
   c. FOR EACH beat in the script:
      i. DETERMINE shot type:
         - ECU (Extreme Close-Up): Product detail, ingredient label, texture
         - CU (Close-Up): Face/emotion, product in hand
         - MCU (Medium Close-Up): Shoulders up, conversational
         - MS (Medium Shot): Waist up, action with context
         - MWS (Medium Wide Shot): Full body with environment
         - WS (Wide Shot): Establishing shot, full environment
      ii. DESCRIBE subject SPECIFICALLY:
         - NOT "person" -- specify age range, gender, appearance, wardrobe, setting
         - NOT "product" -- specify angle, surface, lighting, background
         - Enough detail that a production team or AI tool can execute without questions
      iii. DESCRIBE action:
         - What happens IN FRAME during this beat
         - Movement (talent movement, camera movement, object movement)
         - Emotional beat (what the viewer should feel)
      iv. ASSIGN duration in seconds
      v. SPECIFY framing:
         - Rule of thirds placement
         - Headroom (tight/standard/loose)
         - Leading space (for movement direction)
         - Platform safe zone compliance
      vi. SPECIFY text overlay (if any):
         - Exact words
         - Position (percentage from top/left or zone name)
         - Style (from typography system)
         - Animation (entrance/exit method and duration)
      vii. SPECIFY transition to next shot:
         - Cut (hard, clean)
         - Dissolve (gentle transition)
         - Swipe (directional, TikTok-native)
         - Zoom (push in/pull out)
         - Match cut (visual continuity between shots)
   d. RUN SHOT SPECIFICITY CHECK:
      - Every beat has: type, subject, action, duration, framing, text overlay (or "none"), transition
      - Zero vague descriptions (any shot under 15 words = flagged)
      - "Show product" / "happy customer" / "lifestyle footage" = FAIL
6. COMPILE all shot lists
7. WRITE to output file
