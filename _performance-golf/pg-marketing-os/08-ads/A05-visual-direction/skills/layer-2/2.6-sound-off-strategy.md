# 2.6: Sound-Off Strategy

## Purpose
For each concept, design the complete sound-off experience: visual hook (first 3 seconds must communicate hook WITHOUT audio), text overlay narrative arc (the COMPLETE story told through text alone), caption design (styled captions that serve as primary narrative track), visual proof elements (charts, before/after, product demos that communicate without explanation), emoji/icon usage (platform-native visual shorthand). The test: if ALL audio is removed, can a viewer understand the ad's core message? If not, the sound-off strategy is incomplete.

## Inputs
- Shot lists from 2.1 (visual flow to overlay text onto)
- Script beat structures from 0.1 (audio content that must be communicated visually)
- Platform constraints from 0.2 (sound-off priority by platform)
- Typography system from 1.3 (text overlay styles)
- Motion graphics from 2.4 (animated text capabilities)

## Process
1. READ shot lists from 2.1
2. READ script beat structures from 0.1
3. READ platform constraints from 0.2
4. READ typography system from 1.3
5. READ motion graphics design from 2.4
6. FOR EACH concept:
   a. ASSESS sound-off priority:
      - Meta: HIGH (85% mute viewing)
      - TikTok: HIGH (significant mute viewing)
      - YouTube: MEDIUM (more sound-on, but sound-off support still important)
   b. DESIGN visual hook (first 3 seconds):
      - What does the viewer SEE that communicates the hook?
      - Text overlay for hook (exact words, 3-6 words maximum)
      - Visual intrigue element (what makes them stop scrolling WITHOUT audio)
      - First frame analysis: would this frame work as a thumbnail?
   c. DESIGN text overlay narrative arc:
      - Map the COMPLETE audio story to a TEXT-ONLY narrative
      - For each major audio beat, define:
        * Text overlay: exact words (3-6 words per overlay)
        * Display duration: enough time to read at casual pace
        * Position: within platform safe zones
        * Style: from typography system
        * Timing: aligned with visual beats
      - The text overlays alone must tell a coherent story
   d. DESIGN caption strategy:
      - Auto-captions: ON (minimum baseline)
      - Styled captions: font, color, background, position
      - Caption animation: word-by-word, line-by-line, or block
      - Caption safe zones: not overlapping with platform UI or other text
   e. IDENTIFY visual proof elements (sound-off compatible):
      - Before/after photos (no audio explanation needed)
      - Charts/graphs (data speaks visually)
      - Product demonstrations (action speaks visually)
      - Social proof numbers (text overlay with stats)
   f. DEFINE emoji/icon usage (platform-native):
      - Which emojis/icons reinforce key messages
      - Placement within text overlays
      - Frequency (not excessive -- maximum 1 per overlay)
   g. PERFORM MUTE TEST:
      - Strip ALL audio mentally
      - Walk through the visual + text overlays only
      - Can a viewer understand: the problem, the solution, the offer, and the CTA?
      - If ANY of those 4 are unclear on mute: REVISE
7. WRITE to output file
