# 2.8: Layer 2 Validator

## Purpose
Validate ALL visual briefs for completeness and quality. Confirm: every script beat has a corresponding shot specification, every shot specifies all 7 required elements (type, subject, action, duration, framing, text overlay, transition), no vague descriptions exist, talent direction is specific enough for casting/AI generation, product staging is specific enough for photography/AI generation, B-roll descriptions are specific enough for sourcing/generation, sound-off strategy is complete (ad works on mute), and YouTube concepts have thumbnail specs. This is the quality gate before coherence validation in Layer 2.5.

## Inputs
- All Layer 2 outputs: 2.1 (shot lists), 2.2 (talent), 2.3 (product staging), 2.4 (motion graphics), 2.5 (B-roll), 2.6 (sound-off), 2.7 (thumbnails)
- Script inventory from 0.1 (for completeness cross-check)
- Platform constraints from 0.2 (for platform compliance check)

## Process
1. READ all Layer 2 outputs (2.1 through 2.7)
2. READ script inventory from 0.1
3. READ platform constraints from 0.2
4. VALIDATE shot list completeness (from 2.1):
   a. Every script has a shot list: [count / total]
   b. Every beat has all 7 elements:
      - Shot type: [present Y/N per shot]
      - Subject: [present + specific Y/N per shot]
      - Action: [present Y/N per shot]
      - Duration: [present Y/N per shot]
      - Framing: [present Y/N per shot]
      - Text overlay: [present or "none" per shot]
      - Transition: [present Y/N per shot]
   c. VAGUE DESCRIPTION SCAN:
      - Search for: "show product", "happy customer", "lifestyle", "nature footage", "healthy", "results", "product shot"
      - Count vague descriptions: [must be 0]
      - List specific violations
   d. Shot description word count check: any shot described in fewer than 15 words = flagged
5. VALIDATE talent direction (from 2.2):
   a. Every talent concept has complete brief
   b. Descriptions are casting-ready (specific enough to source talent)
   c. AI talent has Arcads-compatible parameters
6. VALIDATE product staging (from 2.3):
   a. Every product concept has staging spec
   b. Before/after consistency (identical lighting/angle) where applicable
7. VALIDATE motion graphics (from 2.4):
   a. Elements match typography system
   b. Platform-specific adaptations exist
8. VALIDATE B-roll (from 2.5):
   a. All descriptions 30+ words
   b. Specificity test: no campaign-swappable descriptions
   c. Stock queries and AI prompts present
9. VALIDATE sound-off (from 2.6):
   a. All concepts have sound-off strategy
   b. Mute test: problem, solution, offer, CTA all communicated visually
10. VALIDATE thumbnails (from 2.7):
    a. All YouTube concepts have 2+ variants
    b. Text is 2-4 words
    c. Readable at mobile preview size
11. DETERMINE gate status: PASS or FAIL
12. WRITE validation results to output file
