# 0.5: Soul.md Visual Loader

## Purpose
Load Soul.md if it exists. Extract visual tone constraints: energy signature (high energy vs contemplative), visual register (premium vs accessible), anti-voice visual equivalents (what the brand should NEVER look like). Soul.md provides the intangible "feel" constraints that prevent visual direction from being technically correct but tonally wrong.

## Inputs
- Soul.md from the project directory (RECOMMENDED -- not required, but quality degrades without it)

## Process
1. CHECK if `[project]/SOUL.md` exists
2. IF EXISTS:
   a. READ Soul.md
   b. CHECK status field (Seed, Expanded, Finalized)
   c. WARN if status is not "Finalized" (visual direction is best constrained by finalized Soul.md)
   d. EXTRACT visual-relevant constraints:
      - **Energy signature:** High energy vs contemplative vs balanced -- maps to visual pacing and cut frequency
      - **Visual register:** Premium vs accessible vs raw -- maps to treatment type preference
      - **Anti-voice visual equivalents:** What the brand should NEVER look like visually
        - E.g., "never look like a late-night infomercial" or "never look like a corporate explainer"
      - **Emotional floor/ceiling:** Minimum and maximum emotional intensity -- maps to visual intensity
      - **Pacing signature:** Fast/slow/variable -- maps to shot duration and cut rhythm
   e. TRANSLATE each constraint into visual direction language
   f. WRITE to output file
3. IF NOT EXISTS:
   a. LOG warning: "No Soul.md found. Visual direction will lack taste constraints."
   b. WRITE output file documenting absence
   c. Flag all downstream outputs as "generated without Soul.md visual constraints"
