# Layer 4.3 — Human Review Interface

**Skill:** 09 — Campaign Brief
**Microskill:** 4.3 — Human Review Interface
**Layer:** 4
**Model:** sonnet
**Purpose:** Present the complete Campaign Brief to the human with explicit decision points, capture selections and overrides, and manage the approval/revision cycle.

---

## Input

| Source | File | Required |
|--------|------|----------|
| 4.1-campaign-brief-assembler | `campaign-brief.json` (complete brief) | Yes |
| 4.2-threading-guide-generator | `4.2-threading-guide.json` (threading integrated) | Yes |
| 2.2-weakness-detector | `2.2-weakness-report.json` (weaknesses for human attention) | Yes |

## Output

| File | Format | Min Size |
|------|--------|----------|
| `4.3-human-review-decisions.json` | JSON | 2KB |
| Updated `campaign-brief.json` | JSON | 10KB (with human decisions populated) |
| Updated `campaign-brief-summary.md` | Markdown | 5KB (with human decisions populated) |

## Execution Steps

### Step 1: Present Decision Points

Present the Campaign Brief to the human with 8 explicit decision points. Each decision point must show:
- What is being decided
- The recommendation (if any)
- The alternatives
- What the human needs to evaluate

```
DECISION POINTS FOR HUMAN REVIEW:

1. CAMPAIGN SUMMARY — Does this accurately capture the strategic position?
   [Present: one_sentence_thesis + full_paragraph + element_summaries]
   [Options: Approve | Revise | Override with notes]

2. COHERENCE AUDIT — Do the flagged weaknesses need addressing before writing?
   [Present: overall_score + dimension_scores + critical_weaknesses + coherence_narrative]
   [Options: Accept scores | Flag weakness for upstream revision | Override with rationale]

3. LEAD TYPE — Which lead type for this campaign?
   [Present: primary recommendation + rationale + alternatives + anti-recommendations]
   [Options: Accept recommendation: {type} | Select alternative: {type} | Specify other]

4. STORY TYPE — Which story type for this campaign?
   [Present: primary recommendation + rationale + alternatives + anti-recommendations]
   [Options: Accept recommendation: {type} | Select alternative: {type} | Specify other]

5. EMOTIONAL ARC — Does this emotional journey feel right?
   [Present: full emotional arc + intensity curve]
   [Options: Approve | Adjust | Redesign]

6. TONE — Is this the right voice for this campaign?
   [Present: primary_tone + secondary_tone + avoid list + calibration_notes]
   [Options: Approve | Adjust | Specify different]

7. THREADING GUIDE — Do the planned connections make sense?
   [Present: open loops + recurring elements + transition rules]
   [Options: Approve | Adjust | Add loops/threads]

8. OVERALL APPROVAL — Lock this brief for execution?
   [Present: complete brief summary]
   [Options: Approve and lock | Request revision | Major revision needed — return to synthesis]
```

### Step 2: Capture Human Selections

For each decision point, record:

```yaml
human_decisions:
  choices_made:
    - decision_point: integer  # 1-8
      decision_name: string
      selection: string        # What the human chose
      rationale: string        # Why (if provided)
      timestamp: string
  overrides:
    - decision_point: integer
      recommendation_was: string
      human_selected: string
      override_rationale: string  # REQUIRED for overrides
  approval_status: enum[pending, approved, revision_requested]
  revision_notes: string       # If revision requested
  revision_scope: string       # Which layers need re-execution
  execution_notes: string      # Additional human guidance for writing skills
```

### Step 3: Handle Revision Requests

```
IF human requests revision on decisions 1-2 (summary/coherence):
  revision_scope = "Layer 1 — re-synthesize"
  Return to Layer 1 with human notes

IF human requests revision on decisions 3-7 (creative direction/threading):
  revision_scope = "Layer 3 — re-recommend"
  Return to Layer 3 with human notes

IF human approves (decision 8 = "Approve and lock"):
  Set human_approved = true
  Set approval_date = current timestamp
  Lock brief for downstream consumption
  Proceed to COMPLETE state

Maximum 2 revision cycles before escalating:
  IF revision_cycle_count > 2:
    ESCALATE: "Two revision cycles completed. Persistent issues may indicate
    upstream strategic problems. Recommend reviewing upstream skills before
    continuing brief revision."
```

### Step 4: Update Campaign Brief with Human Decisions

After human review:
1. Populate `creative_direction.lead_type.human_selected` and `human_rationale`
2. Populate `creative_direction.story_type.human_selected` and `human_rationale`
3. Apply any adjustments to emotional arc and tone direction
4. Apply any additions to threading guide
5. Update `human_decisions` section with all choices and overrides
6. Set `approval_status` appropriately
7. Increment version if revised

### Step 5: Lock and Output

If approved:
- Set `metadata.human_approved = true`
- Set `metadata.approval_date`
- Write final `campaign-brief.json` to output directory
- Write final `campaign-brief-summary.md` to output directory
- Log COMPLETE state

```yaml
output_verification:
  campaign_brief_json_exists: boolean
  campaign_brief_json_size_kb: number  # must be >= 10
  campaign_brief_summary_md_exists: boolean
  campaign_brief_summary_md_size_kb: number  # must be >= 5
  human_approved: boolean
  approval_date: string
  version: integer
```

## Quality Gate

- [ ] All 8 decision points presented to human with clear options
- [ ] Human has reviewed and responded to ALL decision points
- [ ] Lead type selected (by human) — either recommendation accepted or alternative chosen
- [ ] Story type selected (by human)
- [ ] Emotional arc approved or adjusted
- [ ] Tone direction approved or adjusted
- [ ] Threading guide approved or adjusted
- [ ] Overall approval captured
- [ ] All overrides have documented rationale (no unexplained overrides)
- [ ] campaign-brief.json updated with human decisions
- [ ] campaign-brief-summary.md updated with human decisions
- [ ] human_approved = true before proceeding to COMPLETE
- [ ] Maximum 2 revision cycles enforced

---

## Anti-Patterns

- **Auto-approval:** Setting human_approved = true without actual human review — VIOLATION (the whole point of this skill is human oversight)
- **"Does this look good?":** Asking a single vague question instead of 8 specific decision points — REJECT
- **Override without rationale:** Human overrides a recommendation but no reason captured — REJECT (rationale required)
- **Infinite revision loops:** More than 2 revision cycles without escalating — this indicates upstream issues, not brief issues

---

## Handoff

On COMPLETE (human_approved = true):
- `campaign-brief.json` available to all downstream writing skills (11-17) via `campaign-brief-outputs/`
- `campaign-brief-summary.md` available for quick reference during writing sessions
- State machine transitions to COMPLETE
