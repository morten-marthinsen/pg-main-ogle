# Layer 1.2 — Element Relationship Mapper

**Skill:** 09 — Campaign Brief
**Microskill:** 1.2 — Element Relationship Mapper
**Layer:** 1
**Model:** opus
**Purpose:** Map the logical connections between all 10 strategic element pairs, scoring connection strength and surfacing evidence from upstream packages.

---

## Input

| Source | File | Required |
|--------|------|----------|
| 1.1-campaign-summary-generator | `1.1-campaign-summary.json` | Yes |
| 0.1-upstream-loader | `0.1-upstream-context.json` | Yes |

## Output

| File | Format | Min Size |
|------|--------|----------|
| `1.2-element-relationships.json` | JSON | 4KB |

## Execution Steps

### Step 1: Map All 10 Required Element Pairs

For every pair of strategic elements, map the logical relationship:

```yaml
element_relationships:
  - element_a: string
    element_b: string
    relationship_type: string  # (causes, resolves, proves, frames, delivers, supports)
    connection_description: string  # 2-3 sentences explaining HOW they connect
    connection_strength: number  # 1-10
    evidence: string  # specific text from upstream packages supporting this connection
```

### Step 2: Evaluate the 10 Required Pairs

**Pair 1: Root Cause -> Mechanism**
- Question: Does the mechanism directly address the root cause?
- Relationship type: "resolves"
- Strong (8-10): The mechanism IS the solution to the root cause
- Weak (1-4): The mechanism addresses a symptom, not the root cause itself

**Pair 2: Mechanism -> Promise**
- Question: Does the mechanism credibly deliver the promise?
- Relationship type: "delivers"
- Strong (8-10): The promise is the natural outcome of the mechanism working
- Weak (1-4): The promise is aspirational but the mechanism doesn't logically produce it

**Pair 3: Promise -> Proof (strongest proof assets)**
- Question: Is the promise supported by available proof?
- Relationship type: "proves"
- Strong (8-10): Multiple proof types directly support the specific promise
- Weak (1-4): Promise exceeds what proof can credibly support

**Pair 4: Big Idea -> Mechanism**
- Question: Does the Big Idea frame the mechanism in a compelling, coherent way?
- Relationship type: "frames"
- Strong (8-10): The Big Idea IS the mechanism seen from the audience's perspective
- Weak (1-4): The Big Idea frames something different than what the mechanism does

**Pair 5: Big Idea -> Root Cause**
- Question: Does the Big Idea illuminate why the root cause exists?
- Relationship type: "frames"
- Strong (8-10): The Big Idea makes the root cause revelation feel inevitable
- Weak (1-4): The Big Idea and root cause feel like they belong to different campaigns

**Pair 6: Villain -> Root Cause**
- Question: Does the villain explain why the root cause persists?
- Relationship type: "causes"
- Strong (8-10): The villain IS the reason the root cause hasn't been solved
- Weak (1-4): The villain is emotionally satisfying but logically disconnected

**Pair 7: Offer -> Promise**
- Question: Does the offer deliver what the promise claims?
- Relationship type: "delivers"
- Strong (8-10): Every offer component maps to a specific promise dimension
- Weak (1-4): Offer includes components that don't connect to the promise

**Pair 8: Structure -> Big Idea**
- Question: Does the argument structure serve the Big Idea's narrative?
- Relationship type: "supports"
- Strong (8-10): The CPB sequence builds the case for the Big Idea inevitably
- Weak (1-4): The argument follows a generic pattern that doesn't leverage the Big Idea

**Pair 9: Avatar -> Root Cause**
- Question: Does the avatar actually experience this root cause?
- Relationship type: "causes" (root cause causes the avatar's situation)
- Strong (8-10): The root cause directly explains the avatar's current frustration
- Weak (1-4): The root cause is theoretical — the avatar doesn't recognize it

**Pair 10: Competitive Landscape -> Big Idea**
- Question: Is the Big Idea differentiated from competitive messaging?
- Relationship type: "supports" (differentiation supports the Big Idea's power)
- Strong (8-10): No competitor is using this angle — the Big Idea is genuinely novel
- Weak (1-4): Competitors are making similar claims with similar framing

### Step 3: Calculate Aggregate Statistics

```yaml
relationship_statistics:
  total_pairs_mapped: 10
  average_connection_strength: float
  pairs_below_5: integer
  pairs_below_7: integer
  weakest_pair: string
  strongest_pair: string
  critical_pairs_status:  # Pairs 1-4 (load-bearing)
    all_above_6: boolean
    average: float
```

### Step 4: Flag Disconnections

For any pair scoring below 5:
- Identify what specific aspect is disconnected
- Suggest which upstream skill would need revision to strengthen the connection
- Assess whether the disconnection is fixable at the brief level or requires upstream rework

## Quality Gate

- [ ] All 10 element pairs mapped with relationship type, description, strength, and evidence
- [ ] Evidence field contains SPECIFIC text from upstream packages (not vague references)
- [ ] No pair mapped without a connection_description of at least 2 sentences
- [ ] Critical pairs (1-4) all scored — these are load-bearing
- [ ] Aggregate statistics calculated
- [ ] Any pairs scoring below 5 flagged with specific disconnection analysis
- [ ] FAIL if more than 3 relationships score below 5/10
- [ ] WARN if any relationship scores below 7/10

---

## Anti-Patterns

- **Inflated scoring:** All pairs scored 8+ with no weaknesses — almost every campaign has at least one weaker connection. Challenge: "What's the WEAKEST connection? Even a 7 has a reason it's not a 9."
- **Vague evidence:** "The mechanism connects to the root cause" without citing specific upstream text — REJECT
- **Missing pairs:** Skipping any of the 10 required pairs — FAIL
