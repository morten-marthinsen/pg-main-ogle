# Layer 0.1 — Upstream Package Loader

**Skill:** 09 — Campaign Brief
**Microskill:** 0.1 — Upstream Package Loader
**Layer:** 0
**Model:** haiku
**Purpose:** Load and parse ALL 8 upstream skill packages (Skills 01-08) into working memory for coherence auditing and creative direction.

---

| Source | File | Required |

| File | Format | Min Size |
|------|--------|----------|

## Execution Steps

### Step 1: Load Research Package (Skill 01)

Extract and index:
- `market_snapshot` — market size, trends, competitive density
- `avatar` — target prospect profile, awareness level, sophistication stage
- `competitive_landscape` — key competitors, positioning, claim patterns
- `quotes_by_category` — verbatim audience language organized by theme
- `schwartz_awareness_level` — current awareness stage classification

### Step 2: Load Proof Inventory (Skill 02)

Extract and index:
- `summary.total_elements` — total proof count
- `summary.overall_strength` — aggregate strength score (must be >= 40)
- `summary.promise_ceiling` — what the proof can support
- `by_category` — proof organized by type (studies, credentials, demonstration, logic, social_proof, testimonials)
- `rankings.knockout_proof` — the single most powerful proof element
- `rankings.by_position` — proof ranked for different argument positions

### Step 3: Load Root Cause Package (Skill 03)

Extract and index:
- `root_cause.expression` — the expressed root cause statement
- `root_cause.three_part_structure.what_they_think` — conventional belief
- `root_cause.three_part_structure.what_it_really_is` — the hidden truth
- `root_cause.three_part_structure.why_nothing_worked` — failure explanation
- `root_cause.villain_profile` — the enemy/villain construct
- `root_cause.expression_type` — simple_reframe, named_syndrome, villain_mechanism, etc.

### Step 4: Load Mechanism Package (Skill 04)

Extract and index:
- `mechanism.name` — the unique mechanism name
- `mechanism.type` — physiological, financial, psychological, etc.
- `mechanism.explanation_text` — the core explanation of HOW the mechanism works
- `mechanism.e5_mechanism_type` — E1-E5 classification
- `mechanism.uniqueness_claim` — what makes this mechanism different
- `mechanism.proof_elements` — proof already associated with the mechanism

### Step 5: Load Promise Package (Skill 05)

Extract and index:
- `primary_promise.statement` — the core promise
- `primary_promise.emotional_frame` — the emotional context of the promise
- `primary_promise.specificity_level` — specific vs. general
- `supporting_promises` — secondary promises
- `promise_ceiling` — the maximum defensible promise
- `proof_pairings` — which proof elements already paired to promises

### Step 6: Load Big Idea Package (Skill 06)

Extract and index:
- `big_idea.concept` — the Big Idea as a concrete concept
- `big_idea.angle` — the framing angle (conspiracy, counterintuitive, authority, transformation, etc.)
- `big_idea.one_sentence` — one-sentence expression of the Big Idea
- `big_idea.emotional_core` — the emotion the Big Idea evokes
- `big_idea.differentiation` — how the Big Idea separates this campaign from competitors

### Step 7: Load Offer Package (Skill 07)

Extract and index:
- `core_offer.product_name` — product/service name
- `core_offer.positioning` — how the offer is positioned
- `offer_stack` — complete stack including bonuses, enhancements
- `guarantee` — guarantee type and language
- `pricing` — price point and value framing

### Step 8: Load Structure Package (Skill 08)

Extract and index:
- `campaign_thesis` — the single overarching claim
- `cpb_sequence` — the ordered Claim-Proof-Benefit chunk sequence
- `section_outline` — ordered list of sections with purpose
- `argument_strategy` — the argument approach (prosecutor, educator, storyteller)
- `chunk_count` — total CPB chunks (must be >= 5)

### Step 9: Build Integrated Upstream Context Object

Merge all extracted data into a single `upstream_context` object:

```yaml
upstream_context:
  research:
    market_snapshot: object
    avatar: object
    competitive_landscape: object
    awareness_level: string
    sophistication_stage: integer
  proof_inventory:
    total_elements: integer
    overall_strength: float
    promise_ceiling: string
    by_category: object
    knockout_proof: object
  root_cause:
    expression: string
    expression_type: string
    three_part_structure: object
    villain_profile: object
  mechanism:
    name: string
    type: string
    explanation_text: string
    e5_type: string
    uniqueness_claim: string
  promise:
    primary_statement: string
    emotional_frame: string
    supporting_promises: [object]
    promise_ceiling: string
  big_idea:
    concept: string
    angle: string
    one_sentence: string
    emotional_core: string
    differentiation: string
  offer:
    product_name: string
    positioning: string
    offer_stack: [object]
    guarantee: object
  structure:
    campaign_thesis: string
    cpb_sequence: [object]
    section_outline: [object]
    argument_strategy: string
    chunk_count: integer
  loading_status:
    packages_loaded: integer  # target: 8
    packages_missing: [string]
    all_critical_loaded: boolean
```

## Quality Gate

- [ ] All 8 upstream packages located and loaded
- [ ] Research outputs include market snapshot, avatar, and competitive landscape
- [ ] Proof inventory overall_strength value extracted and >= 40
- [ ] Root cause three_part_structure complete
- [ ] Mechanism has a NAME (not just description)
- [ ] Big Idea exists as concrete concept (not vague direction)
- [ ] Offer stack is complete (product + guarantee + bonuses defined)
- [ ] Structure has section outline with >= 5 CPB chunks
- [ ] Integrated upstream_context object assembled with all_critical_loaded = true
