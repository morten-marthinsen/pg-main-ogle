# Layer 2.2 — Weakness Detector

**Skill:** 09 — Campaign Brief
**Microskill:** 2.2 — Weakness Detector
**Layer:** 2
**Model:** opus
**Purpose:** Detect critical gaps, contradictions, and weakness patterns in campaign logic by analyzing coherence scores, element relationships, and thesis validation results.

---

## Input

| Source | File | Required |
|--------|------|----------|
| 2.1-coherence-dimension-scorer | `2.1-coherence-scores.json` | Yes |
| 1.2-element-relationship-mapper | `1.2-element-relationships.json` | Yes |
| 1.3-campaign-thesis-validator | `1.3-thesis-validation.json` | Yes |
| 0.1-upstream-loader | `0.1-upstream-context.json` | Yes |

## Output

| File | Format | Min Size |
|------|--------|----------|
| `2.2-weakness-report.json` | JSON | 3KB |

## Execution Steps

### Step 1: Identify Weak Dimensions

Flag any coherence dimension scoring below 6:

```yaml
weak_dimensions:
  - dimension_number: integer
    dimension_name: string
    score: integer
    severity: enum[critical, high, medium]
    impact: string  # What downstream consequence this weakness produces
    remediation_upstream: string  # Which upstream skill would fix this
    remediation_brief: string  # What the brief can do to mitigate
```

Severity classification:
- **Critical:** Dimensions 1-2 below 6 (STRUCTURAL FAILURE — load-bearing connections)
- **High:** Any other dimension below 4 OR 3+ dimensions below 6 (CRITICAL COHERENCE FAILURE)
- **Medium:** Any dimension scoring 4-5 (significant weakness but not structural)

### Step 2: Detect Contradiction Patterns

Analyze where two elements imply different things about the same topic:

```yaml
contradictions:
  - topic: string  # The subject both elements address
    element_a: string
    element_a_implies: string  # What element A says about the topic
    element_b: string
    element_b_implies: string  # What element B says about the topic
    contradiction_type: enum[direct, implicit, tonal]
    severity: enum[critical, moderate, minor]
    resolution_suggestion: string
```

Contradiction types:
- **Direct:** Element A and B make opposite claims about the same fact
- **Implicit:** Element A's framing undermines Element B's logic without directly contradicting
- **Tonal:** Elements are factually compatible but emotionally misaligned (e.g., Big Idea is conspiratorial but proof tone is clinical/neutral)

### Step 3: Cross-Reference with Element Relationships

For each weak relationship pair (strength < 7 from 1.2):
- Does the weakness in the relationship correspond to a weak coherence dimension?
- Are there cascading weaknesses? (e.g., weak Root Cause -> Mechanism weakens Mechanism -> Promise)
- Does the thesis validation reveal issues that amplify the relationship weakness?

```yaml
cascade_analysis:
  - primary_weakness: string  # The originating weak relationship
    cascading_effects: [string]  # What other dimensions/relationships are weakened
    cascade_severity: enum[contained, spreading, systemic]
    root_remediation: string  # Fix the root to fix the cascade
```

### Step 4: Assess Overall Weakness Profile

```yaml
weakness_profile:
  total_weak_dimensions: integer  # scoring below 6
  total_contradictions: integer
  total_cascades: integer

  structural_failure: boolean  # D1 or D2 below 6
  critical_coherence_failure: boolean  # 3+ dimensions below 6
  fundamental_misalignment: boolean  # overall coherence < 5.0

  top_3_weaknesses:
    - weakness: string
      severity: string
      impact: string
      fix: string
    # ... up to 3

  strengths:  # What IS working well
    - strength: string
      dimension: string
      leverage_opportunity: string  # How to build on this strength
    # ... top 3

  overall_assessment: string  # 3-5 sentence narrative of the weakness profile
  proceed_recommendation: enum[proceed, proceed_with_caution, revise_upstream, halt]
```

### Step 5: Generate Remediation Priority

If weaknesses are found, rank remediations by impact:

```yaml
remediation_priority:
  - priority: integer  # 1 = highest
    weakness: string
    fix_location: enum[upstream_skill, brief_level, human_decision]
    fix_description: string
    expected_impact: string  # What improving this weakness would change
```

## Quality Gate

- [ ] All dimensions below 6 flagged as weak with severity classification
- [ ] Contradiction patterns analyzed — direct, implicit, and tonal
- [ ] Cascade analysis completed — spreading weaknesses identified
- [ ] Top 3 weaknesses AND top 3 strengths identified
- [ ] Structural failure flagged if dimensions 1-2 below 6
- [ ] Critical coherence failure flagged if 3+ dimensions below 6
- [ ] Remediation priorities ordered by impact
- [ ] Overall assessment written in 3-5 specific sentences
- [ ] Proceed recommendation issued (not auto-approved)

---

## Anti-Patterns

- **No weaknesses found:** Every campaign has at least one weaker dimension. "No weaknesses" = insufficient analysis
- **Weakness without remediation:** Flagging a problem without suggesting a fix is incomplete
- **Suppressing contradictions:** Downplaying tonal or implicit contradictions because they're "minor" — these leak credibility
