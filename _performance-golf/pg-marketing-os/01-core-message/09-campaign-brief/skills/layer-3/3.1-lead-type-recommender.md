# Layer 3.1 — Lead Type Recommender

**Skill:** 09 — Campaign Brief
**Microskill:** 3.1 — Lead Type Recommender
**Layer:** 3
**Model:** opus
**Purpose:** Recommend lead type based on mechanism type, awareness level, Big Idea angle, and market psychology — with vault precedent, confidence scores, and alternatives for human selection.

## Execution Steps

### Step 1: Determine Primary Recommendation from Awareness Level

The awareness level is the primary driver:

```
AWARENESS LEVEL drives primary recommendation:
  Unaware        -> Story Lead, Curiosity Lead, Prediction Lead
  Problem Aware  -> Problem-Agitation Lead, Empathy Lead
  Solution Aware -> Story Lead, Discovery Lead, Scientific Lead (MOST COMMON in DR)
  Product Aware  -> Differentiation Lead, Mechanism-Focused Lead
  Most Aware     -> Direct Offer Lead, Social Proof Lead
```

Extract awareness_level from upstream research. If not classified, default to Solution Aware (most common in DR long-form) and flag as WARN.

### Step 2: Modify Based on Mechanism Type

The mechanism type narrows the recommendation:

```
MECHANISM TYPE modifies recommendation:
  Scientific/Technical    -> Scientific Lead or Discovery Lead preferred
  Behavioral/Psychological -> Story Lead or Empathy Lead preferred
  Physical/Tangible       -> Demonstration Lead or Results Lead preferred
  Systemic/Process        -> Secret Lead or Counter-Intuitive Lead preferred
```

Cross-reference mechanism.type and mechanism.e5_type from upstream context.

### Step 3: Refine Based on Big Idea Angle

The Big Idea angle further specifies:

```
BIG IDEA ANGLE further refines:
  Conspiracy/Hidden Truth   -> Conspiracy Expose Lead
  Counterintuitive          -> Counter-Intuitive Lead
  Authority/Expert          -> Doctor-Discovery Lead or Authority-Flex Lead
  Transformation            -> Story-Driven Vulnerability Lead
  Revelation/Discovery      -> Discovery Lead
  Prediction/Warning        -> Prediction Lead
```

### Step 4: Check Vault Precedent

Query vault_intelligence.lead_type_correlations:
- What lead types have been used in similar campaigns (same mechanism type + awareness level)?
- What lead types appear in the highest-quality extractions in the same niche?
- Does vault data confirm or challenge the logic-driven recommendation?

### Step 5: Assess Market Psychology

Factor in market psychology from research:
- Is the market currently fearful, frustrated, hopeful, skeptical?
- What emotional state is the prospect in RIGHT NOW?
- Does the lead type enter the conversation already happening in the prospect's mind?

### Step 6: Produce Recommendation

```yaml
lead_type_recommendation:
  primary:
    type: string
    rationale: string  # 3-5 sentences connecting mechanism, awareness, Big Idea
    vault_precedent: string  # example from vault intelligence if available
    confidence: number  # 1-10
    awareness_basis: string  # which awareness level drives this
    mechanism_modifier: string  # how mechanism type influenced the choice
    big_idea_refinement: string  # how Big Idea angle refined the choice
  alternatives:
    - type: string
      rationale: string
      trade_off: string  # what you gain and lose vs. primary
      confidence: number
    - type: string
      rationale: string
      trade_off: string
      confidence: number
  anti_recommendations:
    - type: string
      why_not: string  # why this lead type would be wrong for this campaign
    - type: string
      why_not: string
```

### Step 7: Validate Recommendation

- Does the primary recommendation match the mechanism type AND awareness level?
- Is the rationale grounded in upstream data (not abstract preference)?
- Is the confidence score justified by the evidence?
- Are the alternatives genuinely different options (not slight variations)?
- Are anti-recommendations specific to THIS campaign (not generic warnings)?

## Quality Gate

- [ ] Primary lead type recommendation produced with 3-5 sentence rationale
- [ ] Rationale connects mechanism type, awareness level, and Big Idea angle
- [ ] Vault precedent cited if available
- [ ] Confidence score is 1-10 and justified
- [ ] At least 2 alternatives provided with trade-offs
- [ ] At least 2 anti-recommendations with specific reasons
- [ ] Low-confidence recommendations (<5) marked as "exploratory"
- [ ] Recommendation grounded in upstream data — not abstract preference
- [ ] No anti-slop violations in rationale text

---

## Anti-Patterns

- **"Story Lead" without why:** Recommending a lead type without connecting it to this specific campaign's mechanism, awareness, and Big Idea — REJECT
- **No alternatives:** Only recommending one option — the human needs informed choices
- **Generic anti-recommendations:** "Don't use a direct lead because it's too aggressive" — must be specific to THIS campaign
- **Ignoring vault data:** Logic-only recommendation when vault data is available — incomplete
