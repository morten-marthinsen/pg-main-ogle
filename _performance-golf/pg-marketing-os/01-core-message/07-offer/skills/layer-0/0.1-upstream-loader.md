# Skill 0.1: Upstream Loader

**Version:** 1.0
**Layer:** 0 (Foundation)
**Type:** Data Ingestion
**Dependencies:** Upstream skill outputs (Skills 00-05)
**Output:** upstream_data.json

---

## Purpose

Load, parse, and stage all upstream skill outputs required for offer construction. This skill ensures every prior strategic decision — research, proof, root cause, mechanism, promise, big idea — is available as structured data before offer architecture begins.

No offer component should ever be invented from scratch. Every element traces back to upstream intelligence.

---

## The Principle

**"The offer is the commercial wrapper around everything upstream. If it's not in the upstream data, it doesn't belong in the offer."**

---

## Loading Logic

### Step 1: Locate All Upstream Files

```
FOR each skill (00 through 05):
  LOCATE output file at expected path
  IF file NOT found:
    LOG missing_file with skill number and expected path
    SET status = INCOMPLETE
  IF file found:
    VALIDATE file is not empty
    VALIDATE file is parseable (markdown or JSON)
    SET status = LOADED
```

### Step 2: Extract Required Fields

```
FOR each loaded file:
  FOR each required_field:
    EXTRACT field value from file content
    IF field is absent or empty:
      LOG missing_field with skill number and field name
      SET field_status = MISSING
    IF field is present:
      PARSE into structured format
      SET field_status = EXTRACTED

  COMPILE extraction report:
    total_fields_expected: count
    total_fields_extracted: count
    missing_fields: [list]
    extraction_completeness: percentage
```

### Step 3: Cross-Reference Integrity Check

```
VALIDATE cross-references between upstream outputs:

CHECK 1: mechanism_name (skill_03) appears in big_idea context (skill_05)
  IF disconnect: FLAG "mechanism-big-idea misalignment"

CHECK 2: primary_promise (skill_04) references mechanism capability (skill_03)
  IF disconnect: FLAG "promise-mechanism misalignment"

CHECK 3: proof_assets (skill_01) cover mechanism claims (skill_03)
  IF disconnect: FLAG "proof-mechanism gap"

CHECK 4: root_cause (skill_02) justifies mechanism approach (skill_03)
  IF disconnect: FLAG "root-cause-mechanism gap"

CHECK 5: customer_language (skill_01) aligns with promise language (skill_04)
  IF disconnect: FLAG "language-promise mismatch"

GENERATE integrity_report:
  alignments_checked: 5
  alignments_passed: count
  flags_raised: [list with descriptions]
```

---

## Quality Criteria

1. All 6 upstream files located and loaded
2. Extraction completeness ≥ 90% (allows for optional fields)
3. Zero CRITICAL missing fields (all required fields present)
4. Cross-reference integrity check completed with all flags documented
5. No empty or null values in core fields (market_psychology, mechanism_name, primary_promise, big_idea_statement)

---

## Failure Conditions

- Any upstream file missing entirely → GATE_0 FAIL (cannot build offer without inputs)
- Extraction completeness < 70% → GATE_0 FAIL (insufficient data)
- mechanism_name or primary_promise missing → GATE_0 FAIL (offer cannot exist without these)
