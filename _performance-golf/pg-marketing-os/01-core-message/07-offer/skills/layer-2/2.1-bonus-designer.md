## Purpose

Design the complete bonus package for the offer following Hormozi's 12-Point Bonus Checklist. Each bonus is an independent value component that addresses a specific objection, accelerates results, or deepens the transformation. Bonuses are where "good offer" becomes "irresistible offer."

---

## The Principle

**"A bonus isn't a leftover. It's a strategic weapon that eliminates a specific reason someone would say no."**

---

## Hormozi 12-Point Bonus Checklist

Every bonus MUST pass this checklist before inclusion:

```
□  1. Always offer bonuses (minimum 3)
□  2. Name each bonus with a BENEFIT (not a format)
□  3. Explain how each bonus addresses a specific customer issue
□  4. Provide proof that the bonus works/has value
□  5. Use vivid imagery for the experience of using the bonus
□  6. Assign and justify a specific price tag
□  7. Prefer tools and checklists over extensive training
□  8. Each bonus should counter a specific objection
□  9. Anticipate future customer needs (post-purchase problems)
□ 10. Total bonus value should exceed core offer value
□ 11. Add scarcity and urgency to individual bonuses where possible
□ 12. Leverage guarantees as bonuses ("if you don't get X, you keep the bonuses")
```

## Design Process

### Step 1: Identify Bonus Opportunities

```
THREE SOURCES of bonus ideas:

SOURCE 1: Unmapped Objections
  FROM problem_solution_map.json → before_purchase problems
  IDENTIFY objections that the core product doesn't fully address
  EACH remaining objection is a bonus opportunity

  Examples:
    Objection: "I don't have time to implement this"
    → Bonus: "Quick-Start Implementation Checklist (15-Minute Setup)"

    Objection: "What if I get stuck?"
    → Bonus: "Direct Access Email Support for 90 Days"

SOURCE 2: Acceleration Opportunities
  FROM value_equation_optimization → quick_wins, time optimization
  IDENTIFY ways to make results come FASTER
  EACH accelerator is a bonus opportunity

  Examples:
    Acceleration: "Speed up first results"
    → Bonus: "48-Hour Fast Track Guide — See Your First Result This Weekend"

SOURCE 3: Future Needs
  FROM problem_solution_map → after_initial_use problems
  IDENTIFY problems that arise AFTER the prospect succeeds with core product
  EACH future need is a bonus opportunity

  Examples:
    Future need: "How to maintain results long-term"
    → Bonus: "The Maintenance Protocol — Keep Your Results for Life"

SOURCE 4: Vault Patterns
  FROM vault_offer_patterns → exemplar bonus structures
  IDENTIFY bonus types that appear in top-performing offers
  ADAPT (don't copy) patterns that fit this offer

COMPILE: bonus_opportunity_inventory (minimum 8-10 raw ideas)
```

### Step 2: Design Each Bonus

```
FOR each bonus opportunity (select top 3-5 from inventory):

  DESIGN SPECIFICATION:

  bonus_name:
    Apply naming formula: "[Benefit] [Container Word]"
    GOOD: "The 48-Hour Fast Track Blueprint"
    BAD: "Bonus PDF #1"
    NOTE: Name should make someone want it WITHOUT knowing what's in it

  bonus_description:
    One paragraph describing what it IS and what it DOES
    Format: "This is [what it is]. It [what it does]. So you can [benefit]."

  bonus_type:
    CLASSIFY: report | video | tool | checklist | template | calculator |
              access | community | coaching | software | resource_library

    PREFERENCE HIERARCHY (per Hormozi):
      Tools/Checklists > Templates > Short Trainings > Long Trainings
      Actionable > Educational > Informational

  objection_addressed:
    SPECIFY: Which exact objection or concern does this bonus eliminate?
    Every bonus MUST address a specific objection
    If a bonus doesn't eliminate a concern, it doesn't belong

  benefit_statement:
    Single sentence: "So you can [specific outcome]"
    This is the "why you care" — the result of using the bonus

  proof_of_value:
    How do you prove this bonus is valuable?
    Options:
      - "This is the same [resource] I use with my $X,000 clients"
      - "Based on [study/data] that shows [result]"
      - "Created from [X years / X clients] of experience"
      - "Members who use this see [specific result]"

  vivid_imagery:
    Paint a picture of USING this bonus
    "Imagine opening this up, scanning the checklist, and within 15 minutes
     knowing EXACTLY what to do first, second, and third — no guessing,
     no overwhelm, just clarity."

  stated_value:
    Dollar amount this would cost if sold separately
    MUST be justifiable — not inflated beyond reason
    METHODOLOGY: "If I sold this as a standalone product, what would the market bear?"

  value_justification:
    WHY is this worth $[amount]?
    "Personal coaching at this level runs $[X]/hour. This gives you the same
     strategic output in a fraction of the time."

  scarcity_element: (if applicable)
    Can this bonus have its own scarcity?
    "Only available for the first 100 buyers"
    "Removed after [date]"
    NOTE: Not every bonus needs scarcity — use selectively

  delivery_method:
    How the prospect receives this bonus
    Instant digital access preferred (reduces time delay lever)
```

### Step 3: Validate Bonus Package

```
VALIDATE the complete bonus package:

CHECK 1: Minimum 3 bonuses (Hormozi requirement)
  COUNT designed bonuses
  IF < 3: Design additional bonuses from opportunity inventory

CHECK 2: Total bonus value exceeds core offer value
  COMPARE: total_bonus_value vs core_product_value (from value_stack)
  IF bonus value < core value: Increase bonus count or individual values

CHECK 3: Each bonus addresses a DIFFERENT objection
  LIST all objection_addressed fields
  IF duplicates: Consolidate or redesign

CHECK 4: Mix of bonus types
  CHECK: Not all same type (not all PDFs, not all videos)
  Tools/checklists should dominate but variety matters

CHECK 5: 12-Point Checklist Compliance
  FOR each bonus, run through all 12 points
  FLAG any points not met
  RESOLVE before finalizing

CHECK 6: Vault Pattern Comparison
  FROM vault_offer_patterns → bonus patterns
  COMPARE our bonus count, types, and value structures
  FLAG significant deviations from successful patterns
```

### Step 4: Sequence Bonuses for Presentation

```
ORDER bonuses for maximum impact in the offer presentation:

SEQUENCING PRINCIPLE:
  - First bonus: Addresses the BIGGEST objection (immediate relief)
  - Middle bonuses: Build perceived value (accumulation effect)
  - Last bonus: Creates "tip the scales" feeling (the one that pushes them over)

FOR each bonus:
  ASSIGN: presentation_position (1, 2, 3...)
  WRITE: transition_language
    Position 1: "But that's not all. When you [grab/enroll] today, you also get..."
    Position 2+: "And that's not all. You also get..."
    Final: "And to make this truly a no-brainer..."
```

## Quality Criteria

1. Minimum 3 bonuses designed with full specifications
2. Total bonus value exceeds core offer value
3. Each bonus addresses a unique objection (no duplicates)
4. Every bonus passes Hormozi 12-point checklist (≥ 80% compliance)
5. Bonus names are benefit-forward (no "Bonus #1" or "Extra PDF")
6. Tools/checklists prioritized over long training content
7. Vivid imagery written for each bonus (prospect can picture using it)
8. Presentation sequence is strategic (biggest objection first)
