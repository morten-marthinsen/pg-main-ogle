**Persona:** Legendary Copywriter + Dr. Richard Stern (Skeptic/Devil's Advocate)

---

## Purpose

Design the scarcity mechanism that creates quantity-based fear of missing out. Scarcity answers the question: "Why might I NOT be able to get this?" It's about LIMITED SUPPLY — not limited time (that's urgency, Skill 2.4).

Dr. Stern validates that the scarcity is believable. The Legendary Copywriter crafts how it's communicated. Together they prevent both fake scarcity (destroys trust) and absent scarcity (kills conversion).

---

## The Principle

**"Real scarcity converts. Fake scarcity backfires. The prospect must believe the limitation is genuine — because if they don't, scarcity becomes a credibility wound."**

---

## Scarcity Types (Hormozi Framework)

```
TYPE 1: LIMITED SUPPLY OF SEATS/SLOTS
  "Only [X] spots available."
  Mechanism: Physical or operational capacity constraint
  Best for: Services, coaching, events, anything with real capacity limits
  Believability: HIGH (services genuinely have capacity limits)

TYPE 2: LIMITED SUPPLY OF BONUSES
  "The first [X] buyers also get [bonus]."
  Mechanism: Specific bonuses expire after a quantity threshold
  Best for: Any offer — bonuses can always be limited
  Believability: MEDIUM-HIGH (must explain WHY it's limited)

TYPE 3: NEVER AVAILABLE AGAIN
  "This offer will not be repeated."
  Mechanism: One-time release, limited production, closing permanently
  Best for: Launch events, seasonal offers, legacy products
  Believability: MEDIUM (must have credible reason)

TYPE 4: LIMITED EDITION / VERSION
  "Version 1.0 at this price — future versions will cost more."
  Mechanism: Product evolution justifies current pricing
  Best for: Software, courses being actively developed
  Believability: HIGH (version upgrades are natural)

TYPE 5: CAPACITY-BASED
  "We can only handle [X] new clients per [timeframe]."
  Mechanism: Service delivery capacity
  Best for: Service businesses, coaching, done-for-you
  Believability: VERY HIGH (genuine business constraint)
```

## Design Process

### Step 1: Identify Genuine Scarcity Anchors

```
FROM core_offer_definition + business context:

EVALUATE real constraints:

  CAPACITY CONSTRAINTS:
    - Does the product/service have a real delivery limit?
    - How many new customers can be onboarded per period?
    - Is there a physical constraint (venue, instructor, bandwidth)?

  PRODUCTION CONSTRAINTS:
    - Is this a limited production run?
    - Are materials/resources limited?
    - Is creation cost per unit significant?

  STRATEGIC CONSTRAINTS:
    - Is there a strategic reason to limit enrollment?
    - Does quality degrade with more customers?
    - Is exclusivity part of the brand positioning?

  BONUS CONSTRAINTS:
    - Can any bonuses genuinely be limited?
    - Are bonuses tied to the seller's personal time?
    - Do any bonuses have a real cost per recipient?

  LIST all genuine constraint anchors
  FLAG any constraints that are fabricated (DO NOT USE fabricated scarcity)
```

### Step 2: Select Scarcity Type

```
MATCH genuine constraints to scarcity types:

IF real capacity constraint exists:
  → TYPE 1 (Limited Seats) or TYPE 5 (Capacity-Based)
  This is the STRONGEST scarcity because it's undeniably real

IF bonus involves personal time/custom work:
  → TYPE 2 (Limited Bonuses)
  "The first 50 buyers get a personal strategy review"

IF this is a launch or one-time event:
  → TYPE 3 (Never Available Again)
  Must have credible reason for non-repetition

IF product is actively evolving:
  → TYPE 4 (Limited Edition/Version)
  "Founding member pricing — will increase as new modules are added"

IF no genuine constraint exists:
  → TYPE 2 (Limited Bonuses) — create a bonus with real limitations
  Example: Record a personal video critique for first X buyers
  This creates genuine scarcity around a new element

NEVER:
  → Fabricate seat limits that don't exist
  → Claim "only 7 left" when there's infinite digital supply
  → Lie about production limitations

Dr. Stern CHECK: "Would a skeptical prospect believe this limitation is real?"
  IF NO: Redesign or use a different approach
```

### Step 3: Calibrate Scarcity Numbers

```
IF using quantity-based scarcity:

  CALIBRATE the number:

  TOO LOW: "Only 3 spots available"
    Risk: Feels fabricated unless the service is genuinely exclusive
    Test: Does anyone believe a digital product has only 3 slots?

  TOO HIGH: "Only 10,000 spots available"
    Risk: Doesn't feel scarce at all
    Test: Would the prospect think "that's not actually limited"?

  SWEET SPOT: Number feels real and triggers action
    For coaching/services: 10-50 (believable capacity)
    For bonuses: 50-500 (feels limited but not fake)
    For early pricing: "Founding 100 members" or "First 250 buyers"

  ANCHOR to something real:
    "We can only onboard 25 new clients per quarter because each gets
     a personal strategy session with [coach name]."

  MAKE THE REASON VISIBLE:
    The prospect should understand WHY the number is what it is
    Invisible limits feel fake. Visible limits feel real.
```

### Step 4: Design Scarcity Communication

```
CRAFT the scarcity messaging:

STRUCTURE:
  1. State the limitation
  2. Explain WHY (the credible reason)
  3. State the consequence of waiting
  4. Connect to action

EXAMPLE (Capacity-Based):
  "I can only take on 25 new members this quarter. That's not an
   artificial limit — each new member gets a personal onboarding call
   with me, and I physically can't do more than 25 in a quarter.
   Once those spots fill, the next opening won't be until April."

EXAMPLE (Limited Bonus):
  "The first 100 buyers get the Personal Strategy Audit — that's me
   personally reviewing your situation and giving custom recommendations.
   I've priced it at $297 because that's what I charge my private clients,
   but I'm including it free for the first 100 people who act today.
   After 100, this bonus is permanently removed from the offer."

WRITE:
  - Primary scarcity statement (for sales page)
  - Short scarcity statement (for email/ads)
  - Scarcity reminder (for follow-up / closing)
```

### Step 5: Validate Believability

```
Dr. Stern SKEPTIC TEST:

  READ the scarcity statement as a hostile buyer:

  TEST 1: "Is this limitation real?"
    Would I believe this if I were skeptical?
    Is the reason transparent and logical?
    → IF DOUBT: Strengthen the reasoning or change the type

  TEST 2: "Have I seen this exact scarcity claim before?"
    Does this sound like every other "limited spots" claim?
    → IF GENERIC: Make it specific to THIS offer's genuine constraints

  TEST 3: "What happens if I ignore the scarcity?"
    Is there a clear consequence of not acting?
    Is the consequence believable and specific?
    → IF UNCLEAR: Strengthen the consequence statement

  TEST 4: "Does this match the rest of the offer?"
    If the offer feels mass-market, does "limited to 25" make sense?
    If the product is digital, does "limited supply" make sense?
    → IF MISMATCH: Align scarcity type with offer type

  RATE believability: 1-10
  MINIMUM: 7/10 to proceed
  IF < 7: Redesign with a more credible scarcity anchor
```

## Quality Criteria

1. Scarcity is anchored to a genuine constraint (not fabricated)
2. Reason for limitation is transparent and specific
3. Numbers are calibrated (not too low to be fake, not too high to be meaningless)
4. Skeptic believability score ≥ 7/10
5. Messaging includes primary, short, and reminder versions
6. Consequence of inaction is clear and specific
7. Scarcity type matches offer type (service scarcity for services, bonus scarcity for digital)
