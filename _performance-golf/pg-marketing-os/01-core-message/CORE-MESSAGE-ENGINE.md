## THE 5 LAWS (Never Scroll Past This)

1. **Strategy is not copy.** These skills produce strategic frameworks, not prose. Root cause framing, mechanism development, promise calibration — these are intellectual exercises, not writing exercises. If your output reads like a sales page, you are doing it wrong.
2. **Each skill builds on the last.** Root Cause → Mechanism → Promise → Big Idea → Offer → Structure → Campaign Brief. Skipping a skill or reordering the sequence produces incoherent foundations.
3. **The Arena is mandatory for Big Idea.** Skill 06 runs a 3-round Arena with minimum 3 personas. Big Ideas generated without competitive pressure converge on the first "good enough" concept.
4. **Proof constrains promises.** The promise ceiling (from Skill 02) determines what Skill 05 can claim. Bold promises without proof backing are lies, not marketing.
5. **The Campaign Brief is the contract.** Skill 09 audits coherence across all foundation skills. If it detects contradictions, the contradiction is resolved BEFORE writing begins — never during.

## CRITICAL: READ THIS FIRST

This file exists because **the Core Message Engine produces the strategic DNA of every campaign.** When root cause framing is generic, the entire campaign is generic. When the mechanism is vague, no amount of good copy can make it compelling. When the Big Idea is derivative, the campaign is forgettable.

Skills 03-09 run on Opus 4.6 (strategy model, not copy model) because these decisions require deep reasoning, not fluent prose. The Campaign Brief (Skill 09) synthesizes all seven foundation outputs into a single executable document that unlocks all downstream writing.

**This file is the fix.** Before executing ANY Core Message skill, read the relevant sections below.

## THE CORE PROBLEM: FOUNDATION-SPECIFIC DEGRADATION PATTERNS

### Pattern 1: The Generic Root Cause
The model identifies surface symptoms instead of the hidden cause. "People are stressed" is a symptom. "The cortisol-inflammation feedback loop that conventional stress management ignores" is a root cause. **The fix:** Skill 03 requires a specific named villain and a mechanism-ready reframe that goes deeper than what the audience already knows.

### Pattern 2: The Abstract Mechanism
The model describes a mechanism in generic terms — "our proprietary blend" or "advanced technology." A mechanism must be named, ownable, and specific enough to visualize. **The fix:** Skill 04 uses the caveman test: can you explain this mechanism to a caveman using only nouns and verbs? If not, it is too abstract.

### Pattern 3: The Uncalibrated Promise
The model makes claims that sound impressive but cannot be backed by the proof inventory. Or the opposite — the promise is so timid it creates no desire. **The fix:** Skill 05 calibrates against the promise ceiling from Skill 02. Bold enough to compel, calibrated enough to believe.

### Pattern 4: The First-Draft Big Idea
The model generates one Big Idea and declares it done. Big Ideas require competitive pressure — multiple candidates, Arena scoring, and explicit evaluation against novelty and market saturation. **The fix:** Skill 06 mandates a 3-round Arena with minimum 3 personas. The first idea is rarely the best.

### Pattern 5: The Feature Dump Offer
The model lists product features and calls it an offer. An offer is a strategic construction — core product + bonuses + guarantee + value anchoring + urgency mechanism — designed to make saying yes the only rational choice. **The fix:** Skill 07 architects the offer as a persuasion system, not a feature list.

### Pattern 6: The Coherence Gap
Each skill output looks fine in isolation, but contradictions emerge when combined — the promise exceeds the proof ceiling, the mechanism doesn't address the root cause, the Big Idea doesn't connect to the offer. **The fix:** Skill 09 runs coherence auditing across all dimensions before any writing begins.

### Anti-Degradation Protocol (Foundation-Specific)

```
WHEN YOU NOTICE YOURSELF:
- Describing root causes with generic terms → STOP. Name the specific villain.
- Writing mechanism descriptions that could apply to any product → STOP. Make it named and ownable.
- Making promises without checking proof ceiling → STOP. Calibrate against Skill 02 output.
- Accepting the first Big Idea without Arena → STOP. Run competitive evaluation.
- Listing features instead of architecting an offer → STOP. Build a persuasion system.
- Moving to writing without coherence audit → STOP. Run Skill 09 first.

IF CONTEXT IS LARGE:
- This does NOT excuse generic root cause framing
- This does NOT excuse skipping the Arena for Big Idea
- This does NOT excuse uncalibrated promises
- Request continuation rather than compressing quality
```

## ARCHITECTURE OVERVIEW

The Core Message Engine is a 7-skill pipeline that transforms research evidence into strategic campaign foundations. It sits between Deep Research (Skills 00-02) and Long-Form VSL (Skills 10-20).

### Skill Pipeline

```
Skill 03: Root Cause Framing
  → Identify hidden cause behind surface symptoms
  → Name the villain, create the reframe
  → Output: root-cause-package.yaml

Skill 04: Mechanism Development
  → Create named, ownable HOW behind the solution
  → Differentiate from everything else the audience has tried
  → Output: mechanism-package.json

Skill 05: Promise Calibration
  → Craft bold-but-believable outcome claim
  → Calibrate against proof ceiling from Skill 02
  → Output: promise-output.json

Skill 06: Big Idea Generation
  → Generate worldview-shifting creative concept
  → 3-round Arena with minimum 3 personas
  → Output: big-idea-output.json

Skill 07: Offer Stack Design
  → Architect irresistible offer (product + bonuses + guarantee + anchoring)
  → Output: offer-package.json

Skill 08: Campaign Structure
  → Define format (VSL/sales letter/landing page), channels, element relationships
  → Output: structure-package.json

Skill 09: Campaign Brief (SYNTHESIS)
  → Audit coherence across all Skills 01-08, synthesize into single executable brief
  → Detect contradictions, recommend lead type, story type, emotional arc
  → 16 microskills across 5 layers
  → Output: Campaign Brief package → UNLOCKS Skills 10-20
```

### Dependency Chain

```
Research (01) + Proof (02)
  ──→ Skill 03 (Root Cause) ──→ Skill 04 (Mechanism) ──→ Skill 05 (Promise)
                                                       ──→ Skill 06 (Big Idea)
                                                       ──→ Skill 07 (Offer)
                                                       ──→ Skill 08 (Structure)
                                                       ──→ Skill 09 (Campaign Brief)
                                                                    ──→ Skills 10-20
```

### Integration Points

| From | To | What Passes |
|------|-----|------------|
| Skill 01 (Research) | Skill 03 | Research handoff (verbatim quotes, market snapshot, avatar) |
| Skill 02 (Proof Inventory) | Skill 05 | Promise ceiling (maximum credible claim level) |
| Skill 03 (Root Cause) | Skill 04 | Root cause package (villain, reframe, hidden cause) |
| Skill 04 (Mechanism) | Skill 05 | Mechanism package (named mechanism, explanation) |
| Skill 05 (Promise) | Skill 06 | Promise output (calibrated claim, specificity markers) |
| Skill 06 (Big Idea) | Skill 07 | Big Idea output (creative concept, angle architecture) |
| Skill 07 (Offer) | Skill 08 | Offer package (product, bonuses, guarantee, anchoring) |
| Skills 01-08 (all) | Skill 09 | All foundation outputs → coherence audit → Campaign Brief |
| Skill 09 (Campaign Brief) | Skills 10-20 | Single executable brief with creative direction |

## ROOT CAUSE ARCHITECTURE (Skill 03)

The root cause is the hidden cause behind the audience's surface-level problem. It bridges symptom and cause, names the villain, and creates the reframe that makes the mechanism feel necessary.

**Key outputs:** Core problem identification, villain framing, reframe narrative
**Layer architecture:** Layer 0 (vertical profile loading, persona voice loading), Layer 2 (expression anchoring score)

---

## MECHANISM ARCHITECTURE (Skill 04)

The mechanism is the named, ownable process or ingredient that differentiates the product. Without a mechanism, the product is a commodity making the same claims as competitors.

**Key outputs:** Named mechanism, explanation narrative, naming anchoring score
**Layer architecture:** Layer 0 (vertical profile, persona voice, Tier 1 expression reference), Layer 1 (naming anchoring score)

---

## PROMISE ARCHITECTURE (Skill 05)

The promise is the specific, believable outcome the product delivers. Bold enough to compel action, calibrated enough to maintain credibility. Constrained by the promise ceiling from Skill 02.

**Key outputs:** Core promise, promise ceiling validation, specificity markers, believability anchors

---

## BIG IDEA ARCHITECTURE (Skill 06)

The Big Idea is the worldview-shifting insight that hooks the audience. Not a tagline — a central creative concept that makes the audience see their problem differently. All campaign elements orbit the Big Idea.

**Key outputs:** Central concept, creative wrapper, headline seeds, angle architecture
**Arena:** 3-round with minimum 3 personas (mandatory)
**Layer architecture:** Layer 0 (vertical profile, persona voice, Tier 1 expression reference), Layer 3 (expression anchoring score)

---

## OFFER ARCHITECTURE (Skill 07)

The offer is a strategic construction that transforms the Big Idea into a purchasable package. Core product + bonus stack + guarantee + price anchoring + urgency mechanisms.

**Key outputs:** Core product framing, bonus stack, guarantee architecture, price anchoring, urgency mechanisms

---

## STRUCTURE ARCHITECTURE (Skill 08)

Campaign structure defines format (VSL, sales letter, landing page, webinar), channel strategy, and element relationships. Structure decisions affect pacing, tone, and length for every downstream writing skill.

**Key outputs:** Format decision, channel mapping, element relationship architecture

---

## CAMPAIGN BRIEF ARCHITECTURE (Skill 09)

The Campaign Brief is the most complex foundation skill — it synthesizes all upstream outputs into a single executable document. 16 microskills across 5 layers perform coherence auditing, weakness detection, and creative direction.

**Key outputs:** Coherence scores, creative direction (lead type, story type, emotional arc), threading guides
**Layer architecture:**
- Layer 0: Upstream loader, vault intelligence, teachings, input validation
- Layer 1: Campaign summary, element relationship mapper, thesis validation
- Layer 2: Coherence dimension scorer, weakness detector, coherence narrative
- Layer 3: Lead type recommender, story type recommender, emotional arc designer
- Layer 4: Brief assembler, threading guide generator, human review interface
