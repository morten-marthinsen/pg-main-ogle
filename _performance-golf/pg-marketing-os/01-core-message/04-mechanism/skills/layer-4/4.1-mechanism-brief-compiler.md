# Skill 4.1: Mechanism Brief Compiler

**Version:** 1.0
**Layer:** 4 (Package Assembly)
**Type:** Aggregation & Synthesis
**Dependencies:** mechanism_selection.json, scorecard_results.json, explanation_structures.json, naming_candidates.json, analogy_deployments.json, proof_maps.json, vault_comparison.json
**Output:** mechanism_brief.json

---

## Purpose

Assemble the complete mechanism brief — a single, comprehensive document containing EVERYTHING downstream skills need to deploy this mechanism in copy. This is the "source of truth" file that headlines, leads, VSL scripts, and proof sections will all reference. Nothing should be left ambiguous or require re-derivation.

---

## The Question

**"Does this brief contain every piece of information a copywriter would need to write about this mechanism — without asking a single follow-up question?"**

---

## Input Schema

```yaml
input:
  mechanism_selection: from 3.5 (winner + runner-up)
  scorecard_results: from 3.1 (winner's full scorecard)
  explanation_structures: from 1.3 (winner's explanations at all 3 levels)
  naming_candidates: from 1.2 (winner's name + alternatives considered)
  analogy_deployments: from 1.4 (winner's analogies at all deployment levels)
  proof_maps: from 1.5 (winner's proof architecture)
  vault_comparison: from 3.3 (winner's vault benchmarking)
  upstream_context: from 0.1 (big_idea, audience, root_cause, niche)
```

## Compilation Logic

### Step 1: Pull Winner Data

```
FROM mechanism_selection.winner:
  GET candidate_name
  GET archetype
  GET selection_rationale
  GET vault_confidence

USE candidate_name to PULL corresponding data from:
  - explanation_structures (match by name)
  - naming_candidates (match by name)
  - analogy_deployments (match by name)
  - proof_maps (match by name)
  - scorecard_results (match by name)
```

### Step 2: Synthesize Root Cause Connection

```
FROM upstream_context:
  EXTRACT root_cause identified in big idea research

FROM explanation_structures.level_2_paragraph:
  EXTRACT the causal logic (how mechanism → solution)

SYNTHESIZE:
  what_root_cause = [the problem statement]
  how_mechanism_solves = [the causal chain in plain language]
  why_nothing_else_works = [from differentiation score context]
  inevitability_statement = "[Product] works because [Mechanism Name]
    is the only approach that [specific differentiator]"

QUALITY CHECK: inevitability_statement must:
  - Name the product
  - Name the mechanism
  - State a specific differentiator
  - Sound natural spoken aloud
```

### Step 3: Compile Emotional Architecture

```
FROM scorecard dimensions:

  primary_emotion = DERIVED from visceral_response assessment
  doomsday_scenario = FROM 2.8 doomsday developer (consequence timeline)
  super_power_promise = FROM 2.11 super power articulator (superpower statement)
  visceral_trigger = FROM 2.12 visceral response (strongest trigger)

SYNTHESIZE before/after:
  before_state = audience current reality + root cause impact
  after_state = audience transformed reality + mechanism benefit

QUALITY CHECK: before/after must be:
  - Specific to THIS audience (not generic)
  - Emotionally resonant (not just logical)
  - Concrete (tangible details, not abstractions)
```

### Step 4: Derive Deployment Notes

```
BASED ON mechanism archetype and audience:

  DETERMINE best_introduced:
    IF archetype = "THE PROVOCATEUR" → introduce early (hook/headline)
    IF archetype = "THE SCIENTIST" → build curiosity first, reveal mid-copy
    IF archetype = "THE EXPLAINER" → introduce after problem agitation
    IF archetype = "THE ACTIVATOR" → introduce alongside the offer

  DETERMINE name_first_or_concept_first:
    IF name is immediately intriguing → name first
    IF name needs context → concept first, then name as label
    IF name is technical → concept first, name as "scientists call it..."

  DETERMINE proof_timing:
    IF audience is highly skeptical → proof BEFORE mechanism reveal
    IF audience is curiosity-driven → proof DURING mechanism explanation
    IF audience is believer-type → proof AFTER mechanism (confirmation)

  DETERMINE language_register:
    FROM audience sophistication level
    FROM niche conventions
    FROM anti-slop validation (what passed)
```

### Step 5: Completeness Verification

```
VERIFY every field in output schema is populated:

CRITICAL FIELDS (cannot be empty):
  - identity.name
  - the_mechanism_in_brief.one_sentence
  - the_mechanism_in_brief.one_paragraph
  - root_cause_connection.inevitability_statement
  - proof_architecture.primary_proof
  - emotional_architecture.before_state / after_state
  - scorecard_snapshot.average_score

IMPORTANT FIELDS (should be populated):
  - All analogy levels
  - Competitive positioning
  - Deployment notes

IF any CRITICAL field is empty:
  TRACE back to source skill
  IDENTIFY gap
  FLAG for remediation

THE COPYWRITER TEST:
  "Could a copywriter who has NEVER seen the research
   write compelling copy about this mechanism using
   ONLY this brief?"

  IF NO → the brief is incomplete. Identify what's missing.
```

---

## Quality Criteria

1. Every field in the output schema populated (no placeholders, no "TBD")
2. Inevitability statement connects product → mechanism → differentiation
3. Before/after states are specific to this audience, not generic
4. Deployment notes are actionable (not just "use judgment")
5. The brief passes the "copywriter test" — all info needed, no follow-ups required
6. Emotional architecture is derived from actual scorecard assessments, not invented
7. Proof architecture includes specific citations, not "studies show"
