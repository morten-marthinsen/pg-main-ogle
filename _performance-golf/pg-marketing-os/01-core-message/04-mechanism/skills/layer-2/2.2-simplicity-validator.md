# Skill 2.2: Simplicity Validator

**Version:** 1.0
**Layer:** 2 (Scorecard Optimization)
**Type:** Dimension Evaluation + Optimization
**Dimension:** 2 of 12 — SIMPLICITY
**Dependencies:** gate_1_report.json, explanation_structures.json
**Output:** simplicity_scores.json

---

## Purpose

Evaluate and optimize each mechanism candidate on SIMPLICITY: Can a 12-year-old understand the core concept? Complexity is the enemy of conversion. The prospect must grasp the mechanism quickly or they disengage.

---

## The Question

**"Can a 12-year-old understand how this mechanism works?"**

---

## Scoring Rubric

```
1-3 (FAILING): Requires specialized knowledge to understand.
     - Multiple technical terms unexplained
     - Causal chain has 4+ steps
     - Reader needs background education first

4-6 (STRUGGLING): Understandable with effort.
     - 1-2 technical terms that need context
     - Causal chain has 3 steps
     - Reader gets it on second reading

7-8 (CLEAR): Immediately graspable.
     - Zero unexplained technical terms
     - Causal chain is 1-2 steps
     - Reader understands on first pass

9-10 (ELEGANT): So simple it feels obvious in hindsight.
     - Could explain it to a child
     - "Oh, that makes sense!" response
     - Simple enough to repeat to someone else from memory
```

## Evaluation Logic

### Step 1: Complexity Audit

```
FOR each candidate:

  FROM explanation_structures.level_2_paragraph:

  COUNT technical terms:
    - Words requiring domain knowledge
    - Scientific terminology
    - Industry jargon
    - Acronyms without explanation

  COUNT unexplained concepts:
    - Ideas introduced without definition
    - Assumed background knowledge
    - Logical leaps

  MEASURE causal chain:
    - How many steps from mechanism → result?
    - Each "which causes" or "leading to" = +1 step
    - Ideal: 1-2 steps. Problem: 4+ steps.

  ASSESS cognitive load:
    LOW: 0 technical terms, 1-step chain
    MEDIUM: 1-2 technical terms, 2-step chain
    HIGH: 3+ technical terms, 3+ step chain
```

### Step 2: The 12-Year-Old Test

```
FOR each candidate:

  ATTEMPT to rewrite the mechanism explanation as if
  explaining to a 12-year-old:

  RULES:
    - No words a 12-year-old wouldn't know
    - Maximum 2 sentences
    - "Because" only once
    - No abstractions — only concrete things

  IF you can do this easily → score ≥ 8
  IF you struggle to simplify → score 5-7
  IF it's impossible without losing meaning → score 1-4

  RECORD the 12-year-old version regardless
```

### Step 3: Simplify If Below Threshold

```
IF current_score < 7:

  TECHNIQUE 1: Remove Jargon
    FOR each technical term:
      REPLACE with everyday equivalent
      "mitochondrial function" → "your cells' energy factories"
      "neural pathway" → "the road your brain uses"
      "cortisol response" → "your stress alarm"

  TECHNIQUE 2: Shorten Causal Chain
    IF chain is A → B → C → D:
      TRY: A → D (skip intermediaries)
      OR: "A does something special that gives you D"
      Keep only the ESSENTIAL steps

  TECHNIQUE 3: Use Familiar Framework
    Map mechanism to something EVERYONE knows:
      "It works like..." [thermostat/battery/light switch/muscle]
      This borrows simplicity from existing understanding

  TECHNIQUE 4: The "Because" Test
    Force into: "[Mechanism] works because [one reason],
    which means you get [one result]."
    If you can't → too complex
```

### Step 4: Repeatability Test

```
FOR each candidate (simplified if optimized):

  SIMULATE: If the prospect reads this once, then their spouse
  asks "what is that thing you're looking at?" —

  WHAT WOULD THEY SAY?

  GOOD: "It's this thing that [simple action] so you [simple result]"
  BAD: "Well, it's complicated, but basically..."

  can_repeat_from_memory = true if the what_they_would_say
  captures the mechanism's core in ≤ 15 words
```

---

## Quality Criteria

1. Every candidate has complexity diagnosis with specific counts
2. Every candidate has a 12-year-old test version written
3. Candidates scoring < 7 receive simplification
4. Simplified versions have 0 unexplained technical terms
5. Causal chain ≤ 2 steps after optimization
6. Repeatability test captures mechanism in ≤ 15 words
