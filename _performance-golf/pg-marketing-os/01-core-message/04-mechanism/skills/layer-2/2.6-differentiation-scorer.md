# Skill 2.6: Differentiation Scorer

**Version:** 1.0
**Layer:** 2 (Scorecard Optimization)
**Type:** Dimension Evaluation + Optimization
**Dimension:** 6 of 12 — DIFFERENTIATION
**Dependencies:** gate_1_report.json, upstream_context.json, vault_patterns.json
**Output:** differentiation_scores.json

---

## Purpose

Evaluate and optimize each mechanism candidate on DIFFERENTIATION: Is this clearly different from every competitor mechanism in the market? In sophisticated markets, "different" is the price of entry. If the prospect has heard something similar before, the mechanism loses power.

---

## The Question

**"Is this clearly DIFFERENT from every other solution they've seen?"**

---

## Scoring Rubric

```
1-3 (SAME): Sounds like repackaged existing solution.
     - Same mechanism with different name
     - Same type as dominant competitor
     - Reader thinks "I've heard this before"

4-6 (SOMEWHAT DIFFERENT): Different angle on familiar concept.
     - New name for familiar type of approach
     - Different framing but similar core idea
     - Reader thinks "this is kind of different..."

7-8 (CLEARLY DIFFERENT): Distinct from everything in market.
     - Different mechanism TYPE than competitors
     - Different causal explanation
     - Reader thinks "I haven't seen this approach before"

9-10 (CATEGORY-CREATING): So different it creates its own space.
     - Doesn't fit existing categories
     - Competitors can't easily copy the positioning
     - Reader thinks "this is a completely different thing"
```

---

## Evaluation Logic

### Step 1: Map Competitive Landscape

```
FROM upstream_context.market_context:

  LIST all known competitor mechanisms:
    - mechanism_name
    - mechanism_type
    - core_claim
    - key_language

  FROM vault_patterns.type_distribution:
    IDENTIFY most common mechanism types in this niche
    IDENTIFY saturated naming patterns
```

### Step 2: Compare Each Candidate

```
FOR each candidate:

  COMPARE against each competitor mechanism:

  NAME SIMILARITY:
    - Does the name use same words or patterns?
    - Could it be confused with an existing mechanism?
    - Score: 1 (identical feel) to 10 (completely different vocabulary)

  TYPE SIMILARITY:
    - Same mechanism type as competitors?
    - Same: score 1-4 (must differentiate on execution)
    - Different type: score 7-10 (structural differentiation)

  CLAIM SIMILARITY:
    - Does it make the same core promise?
    - Does it explain the same root cause?
    - Does it cite the same kind of evidence?
    - More overlap = lower differentiation

  CALCULATE differentiation_distance:
    Average of name + type + claim similarity scores
```

### Step 3: Assess Category Position

```
FOR each candidate:

  DETERMINE: Where does this mechanism sit relative to the market?

  WITHIN EXISTING CATEGORY:
    - It's a "better version" of something that exists
    - Must compete on execution quality
    - Inherently limited differentiation (score cap: 6)

  ADJACENT TO CATEGORY:
    - Related but distinctly different angle
    - Borrows credibility from category while being new
    - Strong differentiation (score range: 7-8)

  CREATES NEW CATEGORY:
    - Doesn't map to existing competitive landscape
    - Requires new vocabulary to describe
    - Competitors can't easily replicate positioning
    - Maximum differentiation (score range: 9-10)
```

### Step 4: The Competitor Copy Test

```
FOR each candidate:

  SIMULATE: If a competitor heard about this mechanism,
  how quickly could they copy it?

  INSTANT COPY (score 1-3):
    → Just change the name on their existing approach
    → No structural difference to replicate

  MODERATE COPY (score 4-6):
    → Would need to adjust their messaging
    → Could get close in 2-3 weeks

  DIFFICULT COPY (score 7-8):
    → Would need fundamentally different approach
    → Requires different proof, different audience frame

  NEAR-IMPOSSIBLE COPY (score 9-10):
    → Would have to abandon their current positioning
    → Copying this IS becoming us
    → Built on unique assets they don't have
```

### Step 5: Optimize If Below Threshold

```
IF current_score < 7:

  TECHNIQUE 1: Change Type
    IF using same type as competitors:
      EXPLORE: Can the same root cause be addressed with a DIFFERENT type?
      "scientific" in a market of "scientific" → try "discovery" or "natural"

  TECHNIQUE 2: Change Level
    IF explaining at same level as competitors:
      GO DEEPER: Explain the mechanism BEHIND their mechanism
      "They say 'mindset' — we explain the NEUROLOGICAL process behind mindset"

  TECHNIQUE 3: Change Frame
    IF using same framing as competitors:
      REFRAME: Same truth, different lens
      They say "learn to trade" → We say "train your execution"
      They say "improve your swing" → We say "remove the interference"

  TECHNIQUE 4: Add Unique Asset
    IF mechanism is differentiable but not defensible:
      ADD something they can't copy:
      - Proprietary terminology
      - Unique research citation
      - Specific origin story
      - Patentable process

  RE-SCORE after optimization
```

## Output Schema

```yaml
output:
  differentiation_scores:
    for_each_candidate:
      - candidate_name: string
        current_score: integer[1-10]
        competitive_comparison:
          closest_competitor: string
          differentiation_distance: float
          similarity_breakdown:
            name_similarity: integer[1-10]
            type_similarity: integer[1-10]
            claim_similarity: integer[1-10]
        category_position: enum[within, adjacent, creates_new]
        copy_difficulty: enum[instant, moderate, difficult, near_impossible]

        optimization_applied: boolean
        optimized_version:
          differentiation_technique: string
          enhanced_positioning: string
          new_score: integer[1-10]
```

---

## Quality Criteria

1. Every candidate compared against known competitor mechanisms
2. Category position explicitly determined
3. Copy difficulty assessed for defensibility
4. Candidates scoring < 7 receive differentiation optimization
5. No candidate proceeds if it's within same category as market leader AND uses same type
