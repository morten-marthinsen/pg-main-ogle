# Skill 2.11: Super Power Articulator

**Version:** 1.0
**Layer:** 2 (Scorecard Optimization)
**Type:** Dimension Evaluation + Optimization
**Dimension:** 11 of 12 — SUPER POWER
**Dependencies:** gate_1_report.json, explanation_structures.json, upstream_context.json
**Output:** super_power_scores.json

---

## Purpose

Evaluate and optimize each mechanism candidate on SUPER POWER: Does this mechanism give the prospect a NEW CAPABILITY they didn't have before? Not just improvement of what they can already do — a qualitatively NEW ability. The prospect should feel like they're getting a power they never had access to.

---

## The Question

**"Does this mechanism give the prospect a CAPABILITY they've never had before?"**

---

## Scoring Rubric

```
1-3 (NO NEW CAPABILITY): Mechanism improves existing ability slightly.
     - "Get a little better at what you already do"
     - Marginal improvement, not transformation
     - Reader thinks "I can already kind of do this"

4-6 (ENHANCED CAPABILITY): Mechanism significantly improves existing ability.
     - "Do what you already do, but much better/faster"
     - Same kind of output, just more of it
     - Reader thinks "That would be nice to have"

7-8 (NEW CAPABILITY): Mechanism enables something they couldn't do before.
     - "Now you CAN do [thing that was previously impossible]"
     - Qualitative shift, not just quantitative
     - Reader thinks "I've never been able to do that"

9-10 (UNFAIR ADVANTAGE): Mechanism gives capability that feels like cheating.
     - "You'll be able to [extraordinary thing] while others can't"
     - Creates asymmetric advantage
     - Reader thinks "That would be like having a superpower"
```

---

## Evaluation Logic

### Step 1: Identify Current vs. New Capabilities

```
FOR each candidate:

  FROM upstream_context.audience_context:

  MAP the prospect's CURRENT capabilities:
    - What can they already do (even if poorly)?
    - What have they achieved before (even inconsistently)?
    - What is within their current skill range?

  FROM explanation_structures:

  MAP what the mechanism ENABLES:
    - What does the mechanism allow them to do?
    - What outcome does it produce?
    - What does their life/work look like AFTER activation?

  CLASSIFY the gap:
    SAME THING, SLIGHTLY BETTER → score 1-3
    SAME THING, MUCH BETTER → score 4-6
    DIFFERENT THING ENTIRELY → score 7-8
    THING OTHERS CAN'T DO → score 9-10
```

### Step 2: The "Superpower Statement" Test

```
FOR each candidate:

  ATTEMPT to write a superpower statement:
    "With [mechanism], you'll be able to [extraordinary capability]
     that [most people / competitors / your past self] can't."

  IF this statement:
    - Sounds genuinely extraordinary → high score
    - Sounds impressive but achievable → moderate score
    - Sounds like ordinary improvement → low score
    - Can't be written convincingly → very low score

  THE BEST superpower statements create ENVY:
    "Imagine being able to [capability] while everyone else
     is still struggling with [common problem]..."
```

### Step 3: Assess Capability Uniqueness

```
FOR each candidate:

  IS this capability available through ANY other means?

  EXCLUSIVE (score 9-10):
    - Only this mechanism provides this capability
    - No alternative path to this outcome
    - "The ONLY way to [capability] is through [mechanism]"

  RARE (score 7-8):
    - Very few paths to this capability exist
    - Mechanism provides the most accessible path
    - "You could theoretically [alternative], but [mechanism] is the first
       way to [capability] without [extreme cost/time/difficulty]"

  COMMON (score 4-6):
    - Multiple paths to similar capability exist
    - Mechanism may be faster/easier but not unique
    - Need to differentiate on speed/ease, not on capability itself

  CLASSIFY and note implications for scoring
```

### Step 4: Optimize If Below Threshold

```
IF current_score < 7:

  TECHNIQUE 1: Elevate from "Better" to "New"
    REFRAME improvement as capability:
    "Burn fat faster" → "Your body burns fat while you sleep for the first time"
    "Trade more consistently" → "Execute trades without emotional interference"
    KEY: Add "for the first time" or "without [previous barrier]"

  TECHNIQUE 2: Add the Asymmetry
    FRAME capability as advantage over others:
    "While others are still [struggling], you'll be [new capability]"
    Creates the "unfair advantage" feeling

  TECHNIQUE 3: Name the Capability
    GIVE the new capability its own mini-brand:
    "With [mechanism], you develop what I call [Capability Name]"
    Example: "Neural Trading Reset" gives you "Execution Immunity"

  TECHNIQUE 4: Show the Before/After Gap
    CONTRAST current state with mechanism-enabled state:
    BEFORE: "Currently, you [limitation]"
    AFTER: "With [mechanism], you [extraordinary capability]"
    THE GAP between these = perceived super power

  RE-SCORE after optimization
```

## Output Schema

```yaml
output:
  super_power_scores:
    for_each_candidate:
      - candidate_name: string
        current_score: integer[1-10]
        capability_analysis:
          current_capability: string
          mechanism_enabled_capability: string
          gap_type: enum[same_slightly_better, same_much_better, qualitatively_new, unfair_advantage]
          superpower_statement: string
          capability_exclusivity: enum[exclusive, rare, common]

        optimization_applied: boolean
        optimized_version:
          elevated_capability: string
          asymmetry_statement: string
          named_capability: string
          new_score: integer[1-10]
```

---

## Quality Criteria

1. Clear distinction between current and mechanism-enabled capability
2. Gap type classified (quantitative improvement vs. qualitative new capability)
3. Superpower statement written for every candidate
4. Candidates scoring < 7 receive capability elevation
5. Top candidates have "unfair advantage" framing
