# 1.7 Naming Anchoring Score

**Version:** 1.0
**Layer:** 1B (Naming)
**Type:** Scoring + Generation (Data-Grounded Validation)
**Purpose:** Score mechanism name candidates against audience vocabulary, TIER1 naming patterns, and FSSIT candidates, and generate additional quote-first naming candidates
**Shared Protocol:** `~system/protocols/EXPRESSION-ANCHORING-PROTOCOL.md`
**Position:** After 1.6B-naming-gate, BEFORE Layer 2 scorecard

---

## Purpose

This microskill serves two functions:

1. **SCORING:** Take the 15+ mechanism name candidates from 1.2-naming-generator and score each against the three anchoring dimensions (adapted for mechanism naming)
2. **GENERATION:** Produce 5+ additional naming candidates derived bottom-up from audience vocabulary patterns (Quote-First Naming)

Both concept-down names (from 1.2) and quote-first names are scored using the same protocol. Top 5 anchored names proceed to Layer 2.

---

## When to Use

- **Always.** This microskill is mandatory for all mechanism naming.
- Executes AFTER naming-gate (1.6B) confirms ≥15 naming candidates and BEFORE Layer 2 scorecard evaluation.
- Requires research quotes and TIER1 expression reference to be loaded (Layer 0 outputs).

## Execution Protocol

### Step 1: Load Scoring Inputs

```
1. LOAD naming candidates from 1.2-naming-generator output
   - Must have ≥15 candidates, each with name, naming pattern, embedded benefits check

2. LOAD top 20 research quotes from Root Cause and Competitor Mechanism buckets
   - Select by relevance to mechanism language (how the audience talks about solutions, science, processes)
   - These quotes reveal the audience's VOCABULARY for mechanism-adjacent concepts

3. LOAD tier1_expression_reference.json from Layer 0 outputs
   - Contains naming patterns, simplification techniques, word patterns from TIER1 vault

4. LOAD FSSIT candidates
   - IF FSSIT-RANKING.md exists: load top 5 FSSIT candidates
   - IF NOT: load top 10 Hope + Root Cause quotes as proxy
   - NOTE which mode in output
```

### Step 2: Score Each Name Candidate

For each naming candidate from 1.2:

```
DIMENSION 1 — AUDIENCE VOCABULARY MATCH (40%):
  (Adapted from "Quote Penetration" — for mechanism names, we measure vocabulary recognition)
  FOR each of top 20 research quotes:
    Score whether the audience would RECOGNIZE this name (1-10)
    - Does the name use words the audience already uses?
    - Would the audience understand what this name refers to?
    - Does the name feel like something from their world (not purely clinical/technical)?
  TAKE top 5 recognition scores and average them
  RECORD the 5 quotes with scores as citations

  NOTE: This is NOT asking "is this name simple?" — it's asking "does this name use
  vocabulary the audience already has in their mental model?" A name like "AMPK Switch"
  scores high if audience quotes mention "metabolism," "energy switch," "on/off."

DIMENSION 2 — TIER1 NAMING PATTERN MATCH (30%):
  COMPARE name against TIER1 mechanism naming patterns:
    - Naming specificity level: Does it match a proven specificity level?
    - Word pattern: Does it follow successful word patterns (acronym+number, body+action, etc.)?
    - Simplification technique: Does it lend itself to a proven simplification approach?
    - Scientific grounding: Does the name imply credible scientific backing?
  SCORE overall pattern alignment (1-10)
  CITE specific TIER1 sources that use similar naming patterns

DIMENSION 3 — FSSIT ECHO (30%):
  FOR each of top 5 FSSIT candidates (or proxy quotes):
    Score whether this name would resonate with the audience's unspoken feelings (1-10)
    - Does hearing this name create a "that's what's been going on" reaction?
    - Does the name make the audience feel their problem is UNDERSTOOD and NAMEABLE?
  TAKE highest single FSSIT score
  CITE the FSSIT candidate that scored highest

COMPOSITE = (D1 × 0.40) + (D2 × 0.30) + (D3 × 0.30)
```

### Step 3: Quote-First Naming

```
1. EXTRACT mechanism-adjacent language from research quotes:
   - How does the audience describe what they think is happening in their body/market/situation?
   - What words do they use for processes, causes, and solutions?
   - What competitor mechanisms do they mention? What vocabulary surrounds those mentions?

2. FOR each language cluster:
   GENERATE 1-2 mechanism names that:
   - Build directly from audience vocabulary
   - Apply a naming pattern that matches TIER1 conventions
   - Sound like something a credible expert would name, using the audience's own conceptual framework

3. PRODUCE at least 5 quote-first naming candidates

4. SCORE all quote-first names using the same 3-dimension protocol
```

### Step 4: Consolidate and Select

```
1. MERGE concept-down names (from 1.2) with quote-first names
2. SORT all names by composite score (descending)
3. REMOVE names scoring below 5.0 (REJECTED)
4. FLAG names scoring 5.0-6.4 with revision guidance

SELECTION:
  - Top 5 names by composite score proceed to Layer 2
  - Must include at least 1 quote-first name in top 5 (if any scored 6.5+)
  - IF fewer than 5 names score 6.5+: include highest-scoring flagged names to fill top 5
```

## Quality Gates

Before considering 1.7 complete:

- [ ] All 15+ concept-down naming candidates scored across all 3 dimensions
- [ ] At least 5 quote-first naming candidates generated and scored
- [ ] Top 5 names selected by composite score
- [ ] All names below 5.0 removed from candidate pool
- [ ] Flagged names (5.0-6.4) have specific revision guidance
- [ ] Top 5 quote citations included for each scored name
- [ ] TIER1 source swipe_ids cited for pattern matches
- [ ] Output file meets 3KB minimum

---

## Handoff

Top 5 anchored mechanism names proceed to **Layer 2 (Optimization/Scorecard)**. Layer 2 receives:
- 5 data-grounded names (instead of the full 15+ from 1.2)
- Anchoring scores as additional scorecard context
- Quote-first vs concept-down source labels

---

## Anti-Patterns

| Weak | Strong |
|------|--------|
| Generating quote-first names that sound "dumbed down" | Quote-first names that use audience vocabulary elevated to expert-level naming |
| Scoring all names 6-7 with no differentiation | Genuine spread reflecting real differences in audience vocabulary match |
| Ignoring TIER1 naming word patterns | Deliberately checking if name follows proven word patterns (acronym+number, etc.) |
| Treating Audience Vocabulary Match as "simplicity" | Treating it as "vocabulary recognition" — does the audience's mental model contain these words? |
