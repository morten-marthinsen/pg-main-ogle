# Skill 1.4: Analogy Developer

**Version:** 1.0
**Layer:** 1 (Mechanism Ideation)
**Type:** Creative Generation
**Dependencies:** explanation_structures.json, upstream_context.json, vault_patterns.json
**Output:** mechanism_analogies.json

---

## Purpose

Create the simplifying analogy or visual metaphor for each mechanism candidate. The analogy is what makes a complex mechanism GRASPABLE — it translates the unfamiliar into the familiar. The best mechanisms have a single analogy that makes the entire concept click instantly.

---

## Analogy Framework

```
WHY ANALOGIES MATTER:

A mechanism without an analogy is a lecture.
A mechanism WITH an analogy is an "aha moment."

THE RULE: If you can't explain it with a picture or comparison
that a 12-year-old understands, the mechanism isn't simple enough.

ANALOGY TYPES:
1. VISUAL — Creates a mental picture (dominoes falling, switch flipping)
2. FAMILIAR PROCESS — Maps to something everyone knows (thermostat, traffic light)
3. BODY METAPHOR — Uses the body's own processes (muscle memory, immune response)
4. MECHANICAL — Uses machine/system metaphors (engine, circuit, filter)
5. NATURAL — Uses nature metaphors (roots, streams, seasons)
6. EVERYDAY OBJECT — Maps to a common item (key/lock, dimmer switch)
```

## Generation Logic

### Step 1: Extract Mechanism Core Action

```
FOR each candidate:

  FROM explanation_structures.core_logic:

  IDENTIFY the core ACTION of the mechanism:
    - What does it DO? (switches, resets, activates, blocks, unlocks, etc.)
    - What does it DO IT TO? (a process, a substance, a signal, etc.)
    - What's the RESULT? (something starts, stops, reverses, amplifies, etc.)

  REDUCE to simple verb + object:
    Example: "It switches off the cortisol response"
    Example: "It unlocks the neural pathway for execution"
    Example: "It resets the body's fat-burning signal"

  THIS is what the analogy must illustrate.
```

### Step 2: Generate Visual Analogies

```
FOR each candidate:

  BRAINSTORM visual metaphors for the core action:

  IF action is SWITCHING/FLIPPING:
    → Light switch, circuit breaker, toggle, on/off button
  IF action is BLOCKING/PROTECTING:
    → Shield, firewall, filter, dam, fortress wall
  IF action is UNLOCKING/RELEASING:
    → Key in lock, uncorking a bottle, opening a valve
  IF action is RESETTING/RESTORING:
    → Factory reset, clearing a cache, rebooting, defrag
  IF action is BUILDING/ACCUMULATING:
    → Compound interest, snowball, muscle building, stacking
  IF action is REVERSING/UNDOING:
    → Unwinding a clock, rewinding tape, tracing steps back
  IF action is ACTIVATING/TRIGGERING:
    → Ignition key, starting an engine, catalyst in chemistry

  GENERATE 3-5 analogy candidates per mechanism
```

### Step 3: Test Audience Familiarity

```
FOR each analogy candidate:

  ASSESS: Does the target audience ALREADY understand this concept?

  HIGH FAMILIARITY (score 8-10):
    - Light switches, thermostats, traffic lights
    - Keys and locks, on/off buttons
    - Body processes they've experienced

  MEDIUM FAMILIARITY (score 5-7):
    - Circuit breakers, factory resets
    - Compound interest, catalysts
    - Immune responses, muscle memory

  LOW FAMILIARITY (score 1-4):
    - Technical processes (defragmentation)
    - Scientific processes (synaptic pruning)
    - Specialized knowledge

  PREFER analogies scoring ≥ 7 on audience familiarity
```

### Step 4: Test Mechanism Accuracy

```
FOR each analogy candidate:

  ASSESS: How accurately does this analogy represent the mechanism?

  CHECK mapping accuracy:
    - Does the analogy's INPUT match the mechanism's input?
    - Does the analogy's PROCESS match the mechanism's process?
    - Does the analogy's OUTPUT match the mechanism's result?
    - Does the analogy BREAK if you push it too far?

  HIGH ACCURACY (score 8-10):
    - All three elements map correctly
    - Analogy holds under scrutiny
    - No misleading implications

  LOW ACCURACY (score 1-4):
    - Only surface similarity
    - Breaks under questioning
    - Creates wrong expectations

  REJECT analogies scoring < 6 on accuracy
```

### Step 5: Select Primary Analogy

```
FOR each mechanism candidate:

  RANK analogies by:
    composite = (familiarity × 0.45) + (accuracy × 0.40) + (emotional_impact × 0.15)

  SELECT primary analogy:
    - Highest composite score
    - Must score ≥ 7 on familiarity
    - Must score ≥ 6 on accuracy

  SELECT 1-2 alternative analogies:
    - Different analogy types from primary
    - For use in different copy contexts
```

### Step 6: Develop Deployment Versions

```
FOR each candidate's primary analogy:

  HEADLINE VERSION (≤ 10 words):
    - Stripped to essence
    - Example: "The 'Off Switch' for Cortisol"
    - Example: "Your Brain's Hidden Reset Button"

  LEAD VERSION (1-2 sentences):
    - Adds context and intrigue
    - Example: "Think of it like a light switch in your brain. Right now,
      it's stuck in the 'on' position, flooding your body with..."

  EXPLANATION VERSION (3-5 sentences):
    - Full analogy with mechanism mapped
    - Example: "Imagine your stress response has a dimmer switch. Most people's
      is stuck at maximum brightness — always on, always flooding cortisol.
      [Mechanism Name] works by gradually dialing that switch down to its
      natural resting position, where your body was designed to operate."
```

---

## Quality Criteria

1. Every candidate has a primary analogy that passes both familiarity (≥ 7) and accuracy (≥ 6)
2. Primary analogy creates an instant mental image (image_test.can_prospect_picture_it = true)
3. Three deployment versions written (headline, lead, explanation)
4. No analogy uses clichéd/overused metaphors from the same niche
5. Alternative analogies use different analogy types for variety
