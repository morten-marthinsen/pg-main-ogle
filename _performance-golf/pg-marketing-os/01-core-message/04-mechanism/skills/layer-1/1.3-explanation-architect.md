# Skill 1.3: Explanation Architect

**Version:** 1.0
**Layer:** 1 (Mechanism Ideation)
**Type:** Structural Generation
**Dependencies:** type_candidates.json, naming_candidates.json, upstream_context.json, teachings_framework.json
**Output:** explanation_structures.json

---

## Purpose

Develop the core explanatory logic for each top mechanism candidate. HOW does the mechanism work? WHAT does it do inside the body/mind/system? WHY does it produce the promised result? The explanation is the engine of believability — without it, a mechanism is just a name.

---

## E5 Explanation Framework

```
FROM TODD BROWN E5 METHOD:

Every mechanism explanation must answer:

1. WHAT is it? (The naming + categorization)
2. HOW does it work? (The process/sequence)
3. WHY does it work? (The scientific/logical basis)
4. WHAT does it DO to you? (The experiential description)
5. WHY is it DIFFERENT? (The uniqueness claim)

EXPLANATION DEPTH LEVELS:
- Level 1: One-sentence elevator (headline-ready)
- Level 2: Paragraph explanation (lead-ready)
- Level 3: Full walkthrough (body copy-ready)
```

## Construction Logic

### Step 1: Build Root Cause → Mechanism Bridge

```
FOR each top-5 naming candidate:

  START with root_cause_context:
    "The prospect believes [surface_problem]..."
    "But the REAL problem is [real_root_cause]..."
    "This mechanism addresses the real cause by..."

  CONSTRUCT bridge_logic:
    - How does this specific mechanism TYPE address this root cause?
    - What is the LOGICAL chain from root cause → mechanism → result?
    - Why does this mechanism succeed where other solutions failed?

  THIS IS THE FOUNDATION of all explanation levels.
```

### Step 2: Develop Core Logic (E5)

```
FOR each candidate:

  WHAT IT IS:
    - One sentence defining the mechanism category
    - What kind of thing is this? (scientific process? technical system? natural compound? behavioral action? sequence? blend?)

  HOW IT WORKS:
    - Step-by-step or cause-effect description
    - What happens when someone uses/activates this?
    - Use concrete, specific language (no vague abstractions)

  WHY IT WORKS:
    - The scientific, logical, or experiential basis
    - What makes this INEVITABLY produce the result?
    - Reference credibility anchors

  WHAT IT DOES TO YOU:
    - Experiential/sensory description
    - What does the person FEEL or NOTICE?
    - Make it tangible and physical

  WHY IT'S DIFFERENT:
    - Explicit contrast with known alternatives
    - What does this do that nothing else does?
    - The uniqueness claim
```

### Step 3: Write Level 1 — Elevator Sentence

```
FOR each candidate:

  CONSTRAINT: One sentence, maximum 25 words
  MUST INCLUDE: mechanism name + primary action/benefit
  TONE: Confident, specific, intriguing

  PATTERN:
    "[Mechanism Name] is the [type] that [primary action] by [how],
     giving you [primary benefit]."

  OR:
    "[Mechanism Name] works by [specific process], which [why it matters]."

  EVALUATE: Does this sentence alone make someone want to know more?
```

### Step 4: Write Level 2 — Paragraph Explanation

```
FOR each candidate:

  CONSTRAINT: 3-5 sentences, 50-100 words
  STRUCTURE:
    Sentence 1: What it is (category + name)
    Sentence 2: How it works (the process)
    Sentence 3: Why it works (the credibility anchor)
    Sentence 4: What it does to you (the experience)
    Sentence 5: Why it's different (the uniqueness)

  MUST INCLUDE:
    - At least 1 specific detail (number, name, process)
    - At least 1 credibility anchor reference
    - Zero vague qualifiers ("may help", "could potentially")
```

### Step 5: Write Level 3 — Full Walkthrough

```
FOR each candidate:

  CONSTRAINT: 5-8 steps or paragraphs, 150-300 words
  STRUCTURE:
    Step 1: The problem setup (root cause)
    Step 2: Why other solutions fail (failure narrative)
    Step 3: The discovery/revelation (mechanism introduction)
    Step 4: How it works (detailed process)
    Step 5: The evidence (credibility anchors)
    Step 6: What you experience (tangible results)
    Step 7: Why it's different (competitive differentiation)
    Step 8: The implication (if this works, then...)

  MUST INCLUDE:
    - Specific mechanism actions (not abstractions)
    - At least 2 credibility anchors
    - Clear cause-effect chain
    - Experiential language (what you feel/notice)
```

### Step 6: Identify Credibility Anchors

```
FOR each candidate:

  WHAT evidence could support this mechanism?

  TYPES:
    SCIENTIFIC: Studies, research, biological processes
    HISTORICAL: Ancient use, historical precedent, longevity
    LOGICAL: First-principles reasoning, mathematical certainty
    EXPERIENTIAL: Testimonials, case studies, before/after
    AUTHORITATIVE: Expert endorsement, institutional backing

  FOR each anchor:
    - State the claim it supports
    - Identify potential evidence source
    - Note if evidence is available or needs to be found
```

### Step 7: Score Explanation Quality

```
FOR each candidate:

  clarity_score (1-10):
    - Is the explanation free of jargon?
    - Could a non-expert follow the logic?
    - Are there any leaps in reasoning?

  believability_score (1-10):
    - Does the explanation feel plausible?
    - Are credibility anchors specific enough?
    - Would a skeptic find holes?

  specificity_score (1-10):
    - Are concrete details provided?
    - Are there numbers, names, or specifics?
    - Zero tolerance for vague language
```

---

## Quality Criteria

1. All top-5 candidates have all 3 explanation levels
2. Every explanation connects explicitly to the root cause
3. No explanation uses vague qualifiers (may, might, could, potentially)
4. Each candidate has ≥ 2 identified credibility anchors
5. Clarity score ≥ 7 for all candidates (if < 7, simplify)
6. Root cause → mechanism → result chain is traceable in every explanation
