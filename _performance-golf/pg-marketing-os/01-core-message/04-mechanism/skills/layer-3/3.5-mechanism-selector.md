# Skill 3.5: Mechanism Selector

**Version:** 1.0
**Layer:** 3 (Validation & Selection)
**Type:** Final Decision
**Dependencies:** scorecard_results.json, threshold_gate_report.json, vault_comparison.json, slop_validation_report.json
**Output:** mechanism_selection.json

---

## Purpose

Select the WINNING mechanism from all viable candidates. This is the final decision point — one mechanism will proceed to Package Assembly (Layer 4) and eventually become the core mechanism in the sales copy. The selection is based on weighted scoring, strategic fit, and holistic assessment — not just the highest average score.

---

## The Question

**"Which mechanism gives us the highest probability of converting a skeptical prospect who has seen everything?"**

---

```yaml
input:
  scorecard_results: from 3.1

  vault_comparison: from 3.3
```

## Selection Logic

### Step 1: Compile Decision Matrix

```
FOR each viable candidate (passed gate 3.2 AND slop validation 3.4):

  COMPILE decision inputs:

  FROM scorecard_results:
    - average_score
    - strength_cluster
    - weakness_cluster
    - archetype
    - dimensions_above_8 (count and which)

  FROM vault_comparison:
    - vault_confidence (high/medium/low)
    - percentile_estimate
    - above_vault_average (boolean)
    - novel_elements
    - risk_elements

  FROM slop_validation:
    - overall_slop_density
    - post_revision_density (if revised)

  FROM upstream_context:
    - big_idea_alignment (how well mechanism serves the big idea)
    - audience_sophistication (does complexity match audience?)
    - market_saturation (how crowded is this mechanism type?)
```

### Step 2: Apply Strategic Weighting

```
STRATEGIC WEIGHTS (not just raw score):

  RAW SCORECARD AVERAGE: 40% weight
    - The 12-dimension score matters but isn't everything

  STRATEGIC FIT: 25% weight
    - How well does this mechanism SERVE the big idea?
    - Does it naturally lead to the product?
    - Does it create inevitability toward the offer?

  AUDIENCE MATCH: 20% weight
    - Does complexity match audience sophistication?
    - Does the mechanism TYPE resonate with this audience?
    - Does the name land with this specific prospect?

  VAULT CONFIDENCE: 15% weight
    - Proven patterns lower risk
    - Innovation bonus only if scorecard supports it
    - High vault confidence = reliable
    - Low vault confidence + high score = potential breakthrough OR potential failure

CALCULATE weighted_final_score for each candidate
```

### Step 3: Apply Tiebreaker Rules

```
IF top two candidates are within 0.5 points of each other:

  TIEBREAKER 1: Visceral Response Score
    - The mechanism that FEELS more powerful wins
    - This is the "gut check" dimension — it captures what numbers miss

  TIEBREAKER 2: Simplicity Score
    - If still tied, the SIMPLER mechanism wins
    - Simplicity compounds — it aids memorability, word-of-mouth, copy deployment

  TIEBREAKER 3: Differentiation Score
    - If STILL tied, the more DIFFERENT mechanism wins
    - In a crowded market, different beats marginally better

  TIEBREAKER 4: Vault Confidence
    - If all else equal, take the proven pattern over the novel risk
    - Unless the novel mechanism scores 8+ on visceral response
      (visceral overrides vault confidence as a breakthrough indicator)
```

### Step 4: Validate Winner Against Big Idea

```
BEFORE confirming winner:

  CRITICAL CHECK — Does the winning mechanism SERVE the big idea?

  ALIGNMENT TEST:
    1. Can you state: "[Big Idea] works because of [Mechanism]"?
       IF NO → mechanism doesn't serve the idea. Reconsider.

    2. Does the mechanism CREATE INEVITABILITY toward the product?
       "[Mechanism] is the reason [Product] works when nothing else does"
       IF NO → mechanism is interesting but not strategically useful.

    3. Does the mechanism DIFFERENTIATE from competitor mechanisms?
       IF competitor uses same type → our version must be clearly superior
       IF NO → mechanism is "me too" regardless of score.

  IF alignment fails:
    DEMOTE winner to runner-up position
    RE-EVALUATE remaining candidates for alignment
    IF no candidate aligns → FLAG for human review
```

### Step 4.5: Skepticism-Credibility Alignment Check (v1.1 ADDITION)

```
IF market_skepticism = high AND skepticism_type = scientific_validity:

  CRITICAL CHECK — Does the winning mechanism NAME survive skepticism scrutiny?

  CREDIBILITY ALIGNMENT TEST:

    1. Does the name sound like existing biology/science?
       IF naming_pattern ∈ [numbered_benefit, experiential, metaphor]:
         → FLAG: "Winner may trigger skeptic red flags"
         → CHECK: Is runner-up more scientifically credible?

    2. Could a skeptic dismiss this name as "marketing speak"?
       Examples that FAIL this test:
         - "The 700% Response" (sounds like hype, not science)
         - "Stress Switch-Off" (sounds like marketing)
         - "EMF Freedom Method" (sounds invented)
       Examples that PASS this test:
         - "Cellular Resilience Switch" (sounds biological)
         - "Autonomic Regulation Response" (sounds medical)
         - "Neural Coherence Activation" (sounds scientific)

    3. Does the name work WITHOUT explaining the physics?
       In high-skepticism markets where mechanism physics is hard to explain,
       the NAME must carry credibility. If the name requires extensive
       explanation to sound credible, it fails the credibility bridge test.

  IF credibility alignment fails AND runner-up passes:
    RECOMMEND: Promote runner-up to winner status
    DOCUMENT: "Winner demoted due to credibility mismatch in high-skepticism market"

  IF all top candidates fail credibility check:
    FLAG: "No candidates meet credibility requirements — return to naming generation"
    RECOMMEND: Generate new candidates focused on scientific/clinical terminology
```

### Step 5: Assess Selection Confidence

```
DETERMINE confidence level:

DECISIVE (clear winner):
  - Score gap > 1.0 points from runner-up
  - Winner passes ALL alignment checks
  - Winner has vault_confidence = high
  - No tiebreakers needed

STRONG (solid winner with minor concerns):
  - Score gap 0.5–1.0 points
  - Winner passes alignment but has 1 weakness to acknowledge
  - Vault confidence = medium or higher
  - At most 1 tiebreaker needed

CLOSE_CALL (winner by narrow margin):
  - Score gap < 0.5 points
  - Multiple tiebreakers needed
  - OR vault_confidence = low for winner
  - OR alignment check passed but barely
  - → RECOMMEND human review before proceeding

GENERATE clear rationale for selection at whatever confidence level
```

### Step 6: Document Eliminated Candidates

```
FOR each candidate that didn't win:

  RUNNER-UP:
    - WHY it lost (specific — not just "lower score")
    - WHEN it might be better (audience segment, offer type, market condition)
    - KEEP as backup if winner fails in copy testing

  ELIMINATED:
    - WHY eliminated (specific failure point)
    - SALVAGEABLE elements:
      - A great name that could be reused
      - A strong analogy worth keeping
      - A proof structure applicable elsewhere
      - A naming pattern for future mechanisms

  NOTHING IS WASTED — elements from losing candidates
  can inform future mechanism development
```

---

## Quality Criteria

1. Winner selected on strategic weighting, not just raw average
2. Big Idea alignment verified before confirmation
3. Runner-up documented with clear "when to use instead" guidance
4. Selection confidence honestly assessed (close calls flagged)
5. Eliminated candidates mined for salvageable elements
6. Rationale is specific enough that a human can evaluate the decision
7. Human review recommended when confidence is "close_call"
