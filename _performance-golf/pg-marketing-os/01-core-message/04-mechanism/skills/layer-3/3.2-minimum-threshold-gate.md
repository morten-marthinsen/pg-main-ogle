# Skill 3.2: Minimum Threshold Gate

**Version:** 1.0
**Layer:** 3 (Validation & Selection)
**Type:** Quality Gate
**Dependencies:** scorecard_results.json
**Output:** threshold_gate_report.json + GATE_3 status

---

## Purpose

Binary PASS/FAIL checkpoint enforcing minimum scorecard standards. No mechanism may proceed to final selection unless it meets absolute minimums. This gate prevents "good enough" mechanisms from reaching production.

---

## Threshold Rules

```
ABSOLUTE MINIMUMS:

1. AVERAGE SCORE ≥ 7.0
   - A mechanism averaging below 7 is not competitive
   - This is not "good" — this is the FLOOR

2. NO DIMENSION BELOW 5
   - Any dimension below 5 is a fatal weakness
   - Competitors or skeptics will exploit it
   - Must be addressed before proceeding

3. AT LEAST 3 DIMENSIONS ≥ 8
   - Mechanisms need standout strengths, not just adequacy
   - Three strong dimensions create the mechanism's "identity"

4. VISCERAL RESPONSE ≥ 7 (special rule)
   - Dimension 12 is the holistic gut check
   - If the mechanism doesn't FEEL right, nothing else matters
   - This dimension cannot be below 7 regardless of average
```

---

## Gate Logic

### Step 1: Apply Threshold Checks

```
FOR each candidate:

  CHECK 1: average ≥ 7.0
    PASS or FAIL

  CHECK 2: all dimensions ≥ 5
    IF any dimension < 5:
      FAIL
      LOG: which dimensions and their scores

  CHECK 3: count(dimensions ≥ 8) ≥ 3
    PASS or FAIL

  CHECK 4: visceral_response ≥ 7
    PASS or FAIL (special override)

  OVERALL: ALL four checks must PASS
```

### Step 2: Categorize Results

```
FOR each candidate:

  IF all 4 checks PASS:
    status = "VIABLE"
    proceed = true

  IF 1 check fails:
    status = "NEAR_VIABLE"
    proceed = false
    remediation = specific (which check, how far from threshold)

  IF 2+ checks fail:
    status = "NOT_VIABLE"
    proceed = false
    recommendation = "Discard or fundamental redesign"
```

### Step 3: Gate Decision

```
COUNT viable candidates

IF viable_count ≥ 1:
  gate_status = "PASS"
  MESSAGE: "[N] candidates viable for final selection"

IF viable_count = 0 AND near_viable_count ≥ 2:
  gate_status = "CONDITIONAL_FAIL"
  MESSAGE: "No candidates pass. [N] near-viable.
            Return to Layer 2 for targeted optimization."
  SPECIFY: which dimensions need +1-2 points for each near-viable

IF viable_count = 0 AND near_viable_count < 2:
  gate_status = "HARD_FAIL"
  MESSAGE: "Insufficient candidates. Return to Layer 1 for
            new mechanism ideation with different approach."

  IF iteration < 3:
    RETURN to Layer 1 with constraint adjustments
  ELSE:
    ESCALATE to human checkpoint
```

---

## Quality Criteria

1. Gate decision is absolute — no exceptions to thresholds
2. Every failing candidate has specific remediation path
3. Iteration tracking prevents infinite loops
4. Human escalation at iteration 3
