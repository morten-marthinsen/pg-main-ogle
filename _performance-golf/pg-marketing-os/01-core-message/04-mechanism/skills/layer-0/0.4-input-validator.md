# Skill 0.4: Input Validator

**Version:** 1.0
**Layer:** 0 (Foundation)
**Type:** Validation Gate
**Dependencies:** upstream_context.json, vault_patterns.json, teachings_framework.json
**Output:** validation_report.json + GATE_0 status

---

## Purpose

Validate that all Layer 0 outputs are complete, schema-compliant, and logically coherent. This is the Gate 0 checkpoint — mechanism generation cannot begin until all inputs are verified.

---

## Input Schema

```yaml
input:
  upstream_context: from 0.1-upstream-loader
  vault_patterns: from 0.2-vault-intelligence-loader
  teachings_framework: from 0.3-teachings-loader
```

---

## Output Schema

```yaml
output:
  validation_report:
    gate_status: "PASS" | "FAIL"
    timestamp: string
    checks:
      - check_name: string
        status: "PASS" | "FAIL" | "WARN"
        detail: string
    critical_failures: [string]
    warnings: [string]
    coherence_assessment:
      root_cause_big_idea_alignment: boolean
      market_context_available: boolean
      vault_patterns_populated: boolean
      scorecard_complete: boolean
    ready_for_layer_1: boolean
```

---

## Validation Logic

### Check 1: Upstream Completeness

```
VERIFY upstream_context contains:
  ✓ market_context.niche (non-empty)
  ✓ market_context.saturated_mechanisms (array, may be empty)
  ✓ audience_context.primary_audience (non-empty)
  ✓ audience_context.key_frustrations (≥ 3 items)
  ✓ big_idea_context.concept (non-empty)
  ✓ big_idea_context.promise (non-empty, specific)
  ✓ root_cause_context.real_root_cause (non-empty)
  ✓ root_cause_context.reframe_technique (valid enum)

CRITICAL if missing:
  - big_idea_context.concept
  - root_cause_context.real_root_cause
  - audience_context.primary_audience

WARNING if missing:
  - market_context.saturated_mechanisms (limits differentiation scoring)
  - audience_context.language_patterns (limits naming optimization)
```

### Check 2: Vault Patterns Populated

```
VERIFY vault_patterns contains:
  ✓ naming_patterns has ≥ 5 examples per type
  ✓ type_distribution has non-zero values
  ✓ exemplars.top_25 has ≥ 10 entries
  ✓ language_models has ≥ 10 entries per category
  ✓ cross_niche_opportunities has ≥ 1 entry

CRITICAL if missing:
  - exemplars.top_25 (no reference for quality comparison)

WARNING if thin:
  - naming_patterns < 5 per type (limits diversity)
  - language_models < 10 per category (limits linguistic quality)
```

### Check 3: Teachings Framework Complete

```
VERIFY teachings_framework contains:
  ✓ scorecard_dimensions has exactly 12 entries
  ✓ Each dimension has scoring_rubric with all 4 levels
  ✓ Each dimension has optimization_principle
  ✓ naming_rules.principles has ≥ 3 entries
  ✓ naming_rules.anti_patterns has ≥ 3 entries

CRITICAL if missing:
  - scorecard_dimensions (cannot score mechanisms)
  - naming_rules (cannot evaluate names)
```

### Check 4: Logical Coherence

```
ASSESS cross-input coherence:

ROOT CAUSE ↔ BIG IDEA:
  - Does the root cause logically connect to the Big Idea concept?
  - Can a mechanism bridge them? (root cause → mechanism → delivers promise)
  - If NO connection visible → WARNING (may produce incoherent mechanism)

MARKET CONTEXT ↔ VAULT PATTERNS:
  - Is the niche represented in vault data?
  - Are cross-niche opportunities relevant to the target niche?

AUDIENCE ↔ TEACHINGS:
  - Is the audience's sophistication level compatible with mechanism types?
  - (Stage 5 audiences need different mechanisms than Stage 3)
```

### Check 5: Generate Gate Decision

```
IF any CRITICAL check = FAIL:
  gate_status = "FAIL"
  ready_for_layer_1 = false
  LOG critical_failures
  RECOMMEND: what needs to be resolved

ELSE IF > 3 WARNINGS:
  gate_status = "PASS" (with warnings)
  ready_for_layer_1 = true
  LOG warnings for awareness

ELSE:
  gate_status = "PASS"
  ready_for_layer_1 = true
```

---

## Quality Criteria

1. Binary PASS/FAIL — no partial passes on critical checks
2. Clear failure reasons with resolution paths
3. Coherence assessment catches logical disconnects early
4. Warnings are informational, not blocking
