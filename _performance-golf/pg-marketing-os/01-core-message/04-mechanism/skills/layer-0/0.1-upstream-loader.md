# Skill 0.1: Upstream Loader

**Version:** 1.0
**Layer:** 0 (Foundation)
**Type:** Data Loading
**Dependencies:** 01-research outputs, 06-big-idea output, 03-root-cause output
**Output:** upstream_context.json

---

## Purpose

Load and normalize all upstream skill outputs required for mechanism development. Ensures mechanism generation is grounded in market reality, aligned with Big Idea direction, and connected to the diagnosed root cause.

---

## Loading Logic

### Step 1: Load Deep Research Outputs

```
LOCATE project outputs from 01-research:
  - market_psychology_snapshot
  - competitive_analysis
  - customer_language_database

EXTRACT mechanism-relevant fields:
  - Competitor mechanisms currently in market
  - Saturated vs. untapped mechanism spaces
  - Customer language around "why" and "how"
  - Key frustrations and current beliefs

IF mechanism_inventory available from research:
  INCLUDE as additional context
```

### Step 2: Load Big Idea Brief

```
LOCATE output from 06-big-idea:
  - big_idea_brief

EXTRACT:
  - The concept the mechanism must support
  - The promise it must deliver
  - The angle it must align with
  - The emotional core it must resonate with

VALIDATE:
  - concept is non-empty
  - promise is specific (not vague)
  - big_idea_type is valid enum value
```

### Step 3: Load Root Cause Package

```
LOCATE output from 03-root-cause:
  - root_cause_package

EXTRACT:
  - What the prospect thinks the problem is
  - What the real root cause is (mechanism must solve THIS)
  - The reframe technique used
  - The failure narrative (why nothing else worked)
  - Emphasis level (determines mechanism weight)

VALIDATE:
  - real_root_cause is specific and distinct from surface_problem
  - reframe_technique is valid enum
  - failure_narrative explains why past solutions failed
```

### Step 4: Normalize and Compile

```
COMPILE upstream_context:
  - Normalize all field names
  - Flag any missing required fields
  - Generate warnings for weak inputs

SET loading_status:
  - all_required_present = true if no missing required fields
  - missing_fields = list of any absent required inputs
  - warnings = list of quality concerns
```

---

## Quality Criteria

1. All three upstream sources loaded (research, big idea, root cause)
2. No critical fields empty or null
3. Root cause and big idea are logically compatible (mechanism can connect them)
4. Competitive mechanism landscape captured for differentiation
