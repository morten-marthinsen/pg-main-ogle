## Loading Instructions

### When to Load

This microskill executes during Layer 0, AFTER 0.2.7 (persona voice loader) and BEFORE Layer 1A begins. It loads TIER1 extraction data that will be consumed by microskill 1.7 (Naming Anchoring Score) in Layer 1B.

### What to Load

From each TIER1 extraction JSON file in the current project's vertical:

1. **Extract `mechanism` fields:**
   - `mechanism_name` — the named mechanism as used in the control
   - `mechanism_type` — scientific, behavioral, natural, compound, sequence, system
   - `simplification_technique` — binary_reduction, single_metaphor_anchor, layered_analogy_chain, etc.
   - `metaphor_anchor` — the core analogy used to explain the mechanism
   - `scientific_grounding` — what research/data backs the mechanism

2. **Extract mechanism-adjacent fields:**
   - `components.narrative_flow` — how the mechanism is introduced and explained
   - `rsf_framework.S` (specific_solution) — how the solution is positioned
   - `configuration.element_references` — ingredients, components, or system elements

3. **Filter by vertical:**
   - Load ONLY files from `final_merged/[vertical]/` matching the current project vertical
   - If project vertical has fewer than 10 files, supplement with files from `health/` (largest vertical, most mechanism diversity)

## Selection Priority for This Skill

Mechanism naming patterns are most valuable from controls that:

1. **Have memorable, sticky mechanism names** — names that a prospect would remember and repeat (e.g., "AMPK Switch," "T-Factor," "HSL Valve") over generic descriptive names
2. **Use effective simplification techniques** — controls where complex science is made graspable through a single metaphor or analogy
3. **Match the project vertical** — mechanism naming conventions differ by vertical (health uses more scientific terms, golf uses more action-oriented terms)
4. **Show the naming moment** — controls where the mechanism name is dramatically revealed with anticipation buildup

### Per-Vertical Loading Notes

| Vertical | Priority Patterns | Why |
|----------|------------------|-----|
| **Health** | Branded compound names, scientific acronyms, body process metaphors | Health audiences respond to scientific-sounding names that feel credible |
| **Golf** | Action-oriented names, simple movement descriptors | Golf audiences respond to names they can visualize in their swing |
| **Finance** | System names, predictive framework names | Finance audiences respond to names that imply a repeatable system |
| **Personal-Dev** | Mindset/framework names, numbered systems | Personal-dev audiences respond to frameworks they can "apply" |
| **Technology** | Technical-but-accessible names, comparison names | Tech audiences respond to names that feel innovative but understandable |
