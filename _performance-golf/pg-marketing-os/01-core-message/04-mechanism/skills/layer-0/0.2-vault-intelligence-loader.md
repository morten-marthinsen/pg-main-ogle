## Loading Logic

### Step 1: Load Vault Intelligence File

```
LOCATE: vault-intelligence/mechanism-vault-intelligence.json

VALIDATE:
  - File exists and is valid JSON
  - Contains required top-level keys
  - Has non-zero exemplar count

IF file missing or invalid:
  SET status = FAIL
  LOG "DeepAnalysisProtocol must run for mechanisms domain first"
  HALT
```

### Step 2: Extract Naming Patterns

```
FROM mechanism_specific.naming_patterns:

COMPILE naming examples by pattern strategy:
  - clinical: extract all examples
  - branded_format: extract all examples
  - metaphor: extract all examples
  - origin: extract all examples
  - numbered: extract all examples
  - experiential: extract all examples
  - invented: extract all examples

CALCULATE frequency_by_type:
  - Count occurrences in top-25 exemplars
  - Note which types correlate with highest scores
```

### Step 3: Extract Scoring Patterns

```
FROM patterns.scoring_correlations:

IDENTIFY what correlates with high mechanism scores:
  - Which emotional drivers pair with 9-10 mechanisms?
  - Which narrative structures pair with 9-10 mechanisms?
  - What other elements (promise, demonstration) co-occur?

COMPILE high_scorers_common_traits:
  - List traits shared by top-10% mechanisms
```

### Step 4: Extract Cross-Niche Opportunities

```
FROM cross_niche_opportunities:

IDENTIFY patterns from OTHER niches that:
  - Score high in their origin niche
  - Are absent or rare in the target niche
  - Could be adapted without direct copying

COMPILE as innovation seeds for Layer 1
```

### Step 5: Extract Language Models

```
FROM language section:

COMPILE:
  - mechanism_intro_phrases (how top files introduce mechanisms)
  - transition_to_proof (how they bridge to evidence)
  - explanation_structures (patterns for explaining how it works)
  - power_words (words appearing in top-25 mechanism names)
```

### Step 6: Specimen Tiering

```
FOR each specimen in loaded data:

  USE EXPLICIT total_score (unique to mechanism domain):
    Mechanism specimens contain pre-computed total_score (0-10)
    and dimension_scores{} from the V3 extraction process.
    This is the ONLY domain with explicit quality scores — use them directly.

  ASSIGN tier based on total_score:
    Gold = total_score >= 9 (top ~10% — elite mechanisms)
    Silver = total_score >= 7 (top ~25% — strong mechanisms)
    Standard = total_score < 7 (remainder)

  TAG each specimen with:
    tier: enum[gold, silver, standard]
    total_score: integer (from specimen — NOT calculated)
    dimension_scores: object (from specimen — preserved for downstream analysis)

  OUTPUT tier_distribution:
    total_specimens: integer
    gold_count: integer
    silver_count: integer
    standard_count: integer
    score_distribution: {10: int, 9: int, 8: int, 7: int, ...}
```

### Step 7: Type-Matched Retrieval

```
DEFINE match_fields:
  primary: e_type (scientific, technical, natural, compound, behavioral, sequence, system)
  secondary: naming_technique (clinical, branded_format, metaphor, origin, numbered, experiential, invented)
  tertiary: niche (e.g., health, finance, survival, self-improvement)

WHEN campaign_context is available:
  STRATEGY 1 — exact_match:
    FILTER specimens WHERE e_type == target AND naming_technique == target AND niche == target_niche
    PRIORITIZE Gold tier first, then Silver

  STRATEGY 2 — type_technique_match:
    FILTER specimens WHERE e_type == target AND naming_technique == target (any niche)
    PRIORITIZE Gold tier, same-niche Silver, then cross-niche Silver

  STRATEGY 3 — cross_type:
    RETURN all Gold specimens across mechanism types
    INCLUDE Silver specimens matching target niche
    NOTE: Cross-niche Gold mechanisms are especially valuable for innovation seeding

WHEN campaign_context is NOT available:
  RETURN full specimen set with tier metadata attached
  SORT by total_score DESC

OUTPUT retrieval_result:
  matched_specimens: [specimen]
  match_strategy_used: enum[exact_match, type_technique_match, cross_type, full_set]
  gold_specimens_available: integer
  silver_specimens_available: integer
```

## Quality Criteria

1. All pattern categories populated (no empty arrays)
2. Top-25 exemplars loaded with annotations
3. Naming patterns have concrete examples (not just categories)
4. Cross-niche opportunities identified for innovation seeding
5. Language models contain at least 10 examples per category
6. Tiering applied: Gold specimens identified using total_score >= 9
7. Tiering applied: Silver specimens identified using total_score >= 7
8. Type-matched retrieval functional with campaign context
9. Fallback to full set works without campaign context
10. Tier distribution logged with score histogram
