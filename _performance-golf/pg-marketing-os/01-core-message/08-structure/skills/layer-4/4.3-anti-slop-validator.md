# Layer 4.3 — Anti-Slop Validator

## Purpose

Detect and flag generic language, opinion-based statements, cliches, marketing buzzwords, and "copywriter mode" writing that would undermine the argument's credibility and specificity. The anti-slop validator enforces the quality standard that every word in the campaign argument earns its place through specificity and evidence — not through hype or vagueness.

**Primary Persona:** Sarah A. Conco — Client Protection Specialist
**Secondary Persona:** Dr. Richard Stern — Skeptical Academic

## Execution Protocol

### Step 1: Define the Slop Categories

**Category 1: Opinion-Based Claims**
Statements that sound like someone's opinion rather than provable fact.

| SLOP (Reject) | CLEAN (Accept) |
|---------------|---------------|
| "The best system available" | "The only system that addresses [specific root cause]" |
| "Amazing results" | "87% of users reported measurable improvement within 14 days" |
| "Revolutionary breakthrough" | "A mechanism identified in 2019 research from [institution]" |
| "Incredibly effective" | "3x more effective than conventional approaches in clinical testing" |
| "World-class quality" | "Formulated by Dr. [Name] with 23 years of [field] experience" |

**Category 2: Marketing Buzzwords**
Words that signal "advertising" rather than "education/prosecution."

Flagged words/phrases:
- "Revolutionary", "Breakthrough", "Game-changing", "Cutting-edge"
- "Incredible", "Amazing", "Unbelievable", "Stunning"
- "Secret", "Hidden", "Ancient secret" (unless factually supported)
- "Guaranteed", "Proven" (unless genuinely guaranteed/proven with evidence)
- "Act now", "Limited time", "Don't miss out" (urgency language in the argument section)
- "Unlike anything you've ever seen" (vague superlative)

**Exception:** These words are acceptable IF immediately followed by specific evidence. "Revolutionary" alone = slop. "Revolutionary — the first approach to target [specific process] at the cellular level, as documented in [study]" = acceptable.

**Category 3: Copywriter Mode Voice**
Language that sounds like a marketer wrote it rather than an expert speaking.

| COPYWRITER MODE (Reject) | EXPERT VOICE (Accept) |
|--------------------------|----------------------|
| "Picture this incredible scene..." | "Here's what users consistently report..." |
| "You won't BELIEVE what happens next..." | "Here's what the research shows..." |
| "This is going to blow your mind..." | "This is where it gets interesting..." |
| "Imagine the possibilities..." | "What this means in practice is..." |

**Category 4: Vague Language**
Statements that lack specificity.

| VAGUE (Reject) | SPECIFIC (Accept) |
|---------------|-------------------|
| "See results fast" | "See measurable improvement within 14 days" |
| "Save money" | "Save an average of $2,300 per year" |
| "Lots of people use this" | "Over 14,000 users across 23 countries" |
| "Works for everyone" | "Works for beginners, intermediates, and advanced practitioners" |
| "A few simple steps" | "A 3-step process that takes 15 minutes per day" |

**Category 5: Cliches and Filler**
Worn-out phrases that add no value.

Flagged phrases:
- "At the end of the day"
- "It goes without saying"
- "When all is said and done"
- "The fact of the matter is" (unless used once strategically)
- "To be perfectly honest"
- "In this day and age"
- "Take it to the next level"
- "Think outside the box"

### Step 2: Scan All Text Components

Scan the following for slop across all five categories:
1. Simple segue text
2. Every CPB chunk: claims, proof summaries, benefit statements
3. All connector texts
4. All coherence marker texts
5. SIN segue text

### Step 3: Score Each Violation

For each detected violation:

```yaml
violation:
  location: string  # which component (CPB-003.claim, simple_segue, CONN-002, etc.)
  original_text: string  # the flagged text
  category: enum[opinion_claim, marketing_buzzword, copywriter_mode, vague_language, cliche_filler]
  severity: enum[critical, moderate, minor]
  remediation_suggestion: string  # how to fix it
```

**Severity classification:**
- **Critical:** Opinion-based CLAIM (the structural unit fails without this fix)
- **Moderate:** Marketing buzzword or copywriter mode in proof/benefit text
- **Minor:** Cliche/filler in connectors or markers (easy to fix, lower impact)

### Step 4: Calculate Anti-Slop Score

```python
def calculate_anti_slop_score(violations, total_text_components):
    critical_count = sum(1 for v in violations if v["severity"] == "critical")
    moderate_count = sum(1 for v in violations if v["severity"] == "moderate")
    minor_count = sum(1 for v in violations if v["severity"] == "minor")

    # Any critical violation = automatic FAIL
    if critical_count > 0:
        return "FAIL"

    # Moderate + minor violations degrade score
    weighted_violations = (moderate_count * 2) + (minor_count * 1)
    if weighted_violations == 0:
        return "PASS"
    elif weighted_violations <= 3:
        return "PASS_WITH_WARNINGS"
    else:
        return "FAIL"
```

### Step 5: Generate Remediation Report

For each violation, provide a specific remediation suggestion — not just "make it more specific" but exactly what to replace and how.

## Quality Gates

- [ ] All text components scanned (segue, chunks, connectors, markers, SIN segue)
- [ ] Zero critical violations (opinion-based claims)
- [ ] Total weighted violations ≤ 3 for PASS_WITH_WARNINGS
- [ ] Every violation has a specific remediation suggestion
- [ ] Remediation priority ordered (critical first)
- [ ] Status = PASS or PASS_WITH_WARNINGS for Gate 4 passage
- [ ] Status = FAIL blocks structure-package.json assembly

---

## Anti-Patterns (Meta)

- **False positives:** Flagging legitimate specific language as "buzzword" — the word "breakthrough" is fine if followed by specific evidence
- **Over-policing:** Rejecting all emotional language — emotion is required, just not as substitute for evidence
- **Under-policing:** Allowing "incredible results" because it's in the benefit section — benefits must be grounded too
- **Remediation without specifics:** "Make this more specific" is not helpful — provide the exact replacement
