# Layer 4.4 — Vault Pattern Comparator

## Purpose

Compare the assembled campaign argument structure against elite control patterns from the TIER1 extractions vault. Identify structural similarities, differentiation opportunities, and validate that the argument represents original architecture informed by — but not mimicking — proven patterns.

**Primary Persona:** Sarah Chen — Competitive Intelligence Analyst
**Secondary Persona:** Marcus Webster — Pattern Synthesis Analyst

## Execution Protocol

### Step 1: Select Comparison Set

From vault intelligence, select the 3-5 most relevant elite controls for comparison based on:
- **Niche alignment:** Same or similar niche
- **Strategy alignment:** Same argument strategy type
- **Thesis type alignment:** Same thesis type
- **Quality score:** Highest quality scores available

If no exact niche match exists, select cross-niche patterns with the same argument strategy type.

### Step 2: Structural Comparison Matrix

Compare the current argument against each selected control across structural dimensions:

| Dimension | Current Argument | Control A | Control B | Control C |
|-----------|-----------------|-----------|-----------|-----------|
| Chunk count | N | N | N | N |
| Thesis type | type | type | type | type |
| Opening structure | segue type | segue type | segue type | segue type |
| Proof density (early) | level | level | level | level |
| Proof density (mid) | level | level | level | level |
| Proof density (late) | level | level | level | level |
| Dominant proof type | type | type | type | type |
| Emotional arc | pattern | pattern | pattern | pattern |
| Close structure | SIN type | transition type | transition type | transition type |
| Gap types addressed | list | list | list | list |

### Step 3: Identify Similarities

Document where the current argument structurally resembles elite controls:
- Same sequencing pattern?
- Same proof escalation approach?
- Same emotional arc shape?
- Same gap priority ordering?

**Similarity is not inherently bad** — proven structures work for a reason. The concern is when similarity becomes structural mimicry (copying the blueprint without understanding or adapting it).

### Step 4: Identify Differentiators

Document where the current argument structurally DIFFERS from elite controls:
- Different sequencing (and why this is strategic, not accidental)
- Different proof density distribution (and what drives this choice)
- Different emotional arc (and what the strategy demands)
- Unique elements not found in the comparison set

**Differentiation is not inherently good** — being different for its own sake can mean ignoring what works. The goal is INFORMED differentiation based on the current campaign's unique needs.

### Step 5: Assess Structural Mimicry Risk

**Structural mimicry** = copying the vault pattern's blueprint without adaptation. This produces:
- Generic-feeling arguments that lack originality
- Structures that don't fit the current campaign's unique assets
- Missed opportunities to leverage unique proof or mechanism characteristics

**Check for mimicry:**
1. Is the chunk sequence identical to any vault pattern? If so — justify why, or re-sequence.
2. Are connectors copied from the vault transition catalog without customization? If so — customize.
3. Is the SIN segue structurally identical to a vault example? If so — differentiate.
4. Are proof density patterns identical without being driven by the current inventory? If so — recalibrate.

### Step 6: Score Vault Comparison

| Dimension | Weight | Scale | Description |
|-----------|--------|-------|-------------|
| Structural awareness | 0.25 | 1-10 | Does the argument show awareness of proven patterns? |
| Informed differentiation | 0.30 | 1-10 | Are differences strategic and justified? |
| Mimicry avoidance | 0.25 | 1-10 | Does the argument avoid copying without adaptation? |
| Opportunity capture | 0.20 | 1-10 | Does the argument leverage unique campaign assets that vault patterns couldn't? |

**Minimum threshold:** Weighted average ≥ 6.0/10 (lower than other validators because vault comparison is informational, not prescriptive)

## Quality Gates

- [ ] Minimum 3 vault controls in comparison set
- [ ] Structural comparison matrix completed across all dimensions
- [ ] Similarities documented with assessment (strategic vs. mimicry)
- [ ] Differentiators documented with strategic justification
- [ ] Mimicry risk assessed (must be "none", "low", or "moderate" — "high" blocks)
- [ ] Weighted average ≥ 6.0/10
- [ ] Differentiation notes written for structure-package
- [ ] If mimicry_risk = moderate or high, remediation documented

---

## Anti-Patterns

- **Blind copying:** Reproducing a vault pattern's structure without understanding why it worked for THAT campaign
- **Contrarian differentiation:** Being different just to be different — ignoring proven patterns out of novelty bias
- **No comparison:** Skipping vault comparison entirely — missing the opportunity to learn from elite controls
- **Surface comparison:** Only comparing chunk count — missing deeper structural patterns (proof escalation, emotional arc, gap sequencing)
