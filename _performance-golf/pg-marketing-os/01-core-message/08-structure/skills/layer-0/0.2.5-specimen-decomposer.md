# Layer 0.2.5 — Specimen Decomposer

**Version:** 1.0
**Layer:** 0 (Foundation)
**Type:** Specimen Analysis
**Domain:** Structure
**Dependencies:** vault_structure_intelligence (from 0.2-vault-intelligence-loader.md)
**Output:** specimen_decomposition (structural templates, technique catalog, power elements)

---

## Purpose

Decompose top-scoring structure specimens from the vault-intelligence into reusable structural templates, technique catalogs, and power elements. Structure specimens map the section-by-section flow of entire campaigns — section names, functions, positions, word counts, transition techniques, and pacing rhythms. This skill extracts the macro-architecture that makes elite campaigns flow and convert.

**This skill bridges the gap between:** vault data (section-level campaign maps) and generation layers (what Claude needs to construct original campaign architectures with the same strategic flow and pacing).

## Decomposition Protocol

### Phase 1: Specimen Selection & Tiering

```
IF tier_metadata present (Phase B loader active):
  LOAD gold_specimens → full decomposition
  LOAD silver_specimens → template-only decomposition
ELSE (pre-upgrade fallback):
  SELECT top 5 by position in section_sequences as pseudo-Gold
  SELECT next 10 as pseudo-Silver
  LOG "Operating in pre-upgrade mode — tiering estimated from array position"

VERIFY each structure specimen has:
  - sections[] (required — at least 3 sections)
  - total_sections (required)
  - total_word_count (preferred)
  - transition_techniques[] (preferred)
  - pacing_rhythm (preferred)
  - format (required)
  - source_file (required for provenance)
  - niche (required for type-matching)
```

### Phase 2: Structural Template Extraction

For each Gold specimen:

```
ANALYZE sections[] to extract:

  campaign_skeleton:
    FOR each section:
      - section_name: string (from specimen)
        section_function: string (from specimen)
        position: integer (from specimen)
        word_count: integer (from specimen)
        proportion: float (word_count / total_word_count)

  flow_architecture:
    total_sections: integer
    total_word_count: integer
    format: string
    section_function_sequence: [string] (ordered list of functions only —
                                         e.g., "hook → proof → credibility →
                                         mechanism → pain → offer → close")
    weight_distribution: string (front_loaded | balanced | back_loaded)
    longest_section: string (which function gets the most space)
    shortest_section: string (which function is most compressed)

  pacing_profile:
    pacing_rhythm: string (from specimen)
    section_length_variance: string (uniform | varied | extreme_varied)
    compression_points: [string] (sections significantly shorter than average)
    expansion_points: [string] (sections significantly longer than average)

For Silver specimens:
  Extract section_function_sequence and flow_architecture ONLY
  SKIP proportion analysis and pacing profile
```

### Phase 3: Technique Catalog Compilation

Aggregate across ALL decomposed specimens:

```
section_types:
  FOR each unique section_function found:
    - function_name: string (hook, proof, credibility, mechanism_reveal,
                             pain_agitation, offer, close, story, etc.)
    - count: integer (how many campaigns include this function)
    - typical_position: string (range of positions where this appears)
    - word_count_range: string
    - implementation_notes: string

ordering_patterns:
  FOR each unique section_function_sequence (or common subsequence):
    - pattern_name: string
    - section_sequence: [string]
    - frequency: integer
    - format_association: [string] (which formats use this pattern)
    - specimens_using: [source_file references]

transition_techniques:
  FOR each unique transition_technique:
    - technique_name: string (proof_transition, authority_transition,
                              sequential_marker, emotional_bridge,
                              cliffhanger, etc.)
    - count: integer
    - implementation_pattern: string
    - gold_example: string (1-2 sentences showing this transition)
    - bridges_from: string (section function before)
    - bridges_to: string (section function after)

pacing_strategies:
  FOR each unique pacing_rhythm pattern:
    - rhythm_name: string (rapid_fire, gradual_build, crescendo, etc.)
    - count: integer
    - section_length_pattern: string
    - best_for: string (format/niche combination)
    - specimens_using: [source_file references]

format_blueprints:
  FOR each unique format type:
    - format_name: string (magalog, VSL, sales_letter, advertorial, etc.)
    - typical_section_count: string (range)
    - typical_word_count: string (range)
    - common_section_sequence: [string]
    - specimens_using: [source_file references]
```

### Phase 4: Power Element Extraction

From Gold specimens ONLY:

```
section_sequences:
  FOR each Gold specimen:
    - complete_sequence: [object] (full sections[] array)
    - format: string
    - total_word_count: integer
    - source_file: string
    - niche: string

transition_fragments:
  FOR Gold specimens with identifiable transitions:
    - verbatim_transition: string (transition language between sections)
    - technique: string
    - bridges_from: string
    - bridges_to: string
    - source_file: string

section_proportion_maps:
  FOR each Gold specimen:
    - proportions: [object] (section_name + proportion for each section)
    - weight_distribution: string
    - source_file: string
```

## Quality Gates

- [ ] Minimum 3 Gold specimens fully decomposed (skeleton + flow + pacing)
- [ ] Minimum 5 Silver specimens template-decomposed
- [ ] All campaign skeletons have complete section sequences with functions
- [ ] Technique catalog has minimum 2 ordering patterns cataloged
- [ ] Technique catalog has minimum 3 transition techniques documented
- [ ] At least 2 format_blueprints generated
- [ ] Power elements contain minimum 3 complete section sequences from Gold specimens
- [ ] Every power element includes source provenance
- [ ] No raw section sequences passed through undecomposed

---

## Anti-Patterns

| Anti-Pattern | Signal | Fix |
|---|---|---|
| **Sequence Without Function** | Sections listed by name without function annotation | Every section needs section_function |
| **Flat Ordering** | Section order listed without pattern analysis | Identify ordering_patterns across specimens |
| **Transition Ignored** | Sections mapped but transitions between them skipped | transition_techniques are critical architecture |
| **Pacing Invisible** | All sections treated as equal weight | Include proportion analysis + pacing_profile |
| **Format Agnostic** | Structure analyzed without format context | format_blueprints show format-specific patterns |
