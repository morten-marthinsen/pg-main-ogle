# Layer 0.1 — Upstream Package Loader

## Purpose

Load and parse all required upstream skill packages (mechanism, proof inventory, root cause, promise, offer) into working memory for use by downstream layers. Extract the specific fields needed for campaign argument engineering.

---

## Input Requirements

```yaml
upstream_paths:
  mechanism_package: "04-mechanism/outputs/mechanism-package.json"
  proof_inventory: "02-proof-inventory/outputs/proof-inventory-output.json"
  root_cause_package: "03-root-cause/outputs/root-cause-package.json"
  promise_package: "05-promise/outputs/promise-package.json"
  offer_package: "07-offer/outputs/offer-package.json"
```

---

## Execution Protocol

### Step 1: Load Mechanism Package

Extract and index:
- `mechanism.name` — the unique mechanism name for thesis and segue construction
- `mechanism.type` — physiological, financial, psychological, etc.
- `mechanism.explanation_text` — the core explanation of HOW the mechanism works
- `mechanism.e5_mechanism_type` — E1-E5 classification for strategy calibration
- `mechanism.uniqueness_claim` — what makes this mechanism different
- `mechanism.proof_elements` — proof already associated with the mechanism
- `mechanism.thirteen_dimension_scores` — scorecard for strength assessment

### Step 2: Load Proof Inventory

Extract and index:
- `summary.total_elements` — total proof count for density planning
- `summary.overall_strength` — aggregate strength score (must be ≥40)
- `summary.promise_ceiling` — what the proof can support
- `by_category` — proof organized by type (studies, credentials, demonstration, logic, social_proof, testimonials)
- `rankings.knockout_proof` — the single most powerful proof element
- `rankings.by_position` — proof ranked for different argument positions
- `rankings.by_objection` — proof mapped to specific objections

Build a **proof lookup index** keyed by `proof_id` for rapid mapping in Layer 2.

### Step 3: Load Root Cause Package

Extract and index:
- `root_cause.expression` — the expressed root cause statement
- `root_cause.three_part_structure.what_they_think` — conventional belief
- `root_cause.three_part_structure.what_it_really_is` — the hidden truth
- `root_cause.three_part_structure.why_nothing_worked` — failure explanation
- `root_cause.villain_profile` — the enemy/villain construct (if present)
- `root_cause.expression_type` — simple_reframe, named_syndrome, villain_mechanism, etc.

### Step 4: Load Promise Package

Extract and index:
- `primary_promise.statement` — the core promise for thesis and segue construction
- `primary_promise.emotional_frame` — the emotional context of the promise
- `primary_promise.specificity_level` — specific vs. general
- `supporting_promises` — secondary promises for CPB chunk benefit writing
- `promise_ceiling` — the maximum defensible promise
- `proof_pairings` — which proof elements already paired to promises

### Step 5: Load Offer Package

Extract and index:
- `core_offer.product_name` — product/service name for SIN segue
- `core_offer.positioning` — how the offer is positioned (for transition framing)
- `enhancement_stack` (if present) — bonuses and enhancements for reference
- `guarantee` (if present) — guarantee language for close transition

### Step 6: Build Integrated Context Object

Merge all extracted data into a single `upstream_context` object that Layer 1+ microskills can reference without re-loading.
