## Execution Protocol

### Step 1: Scan Extraction Index

Load the available TIER1 extraction files. Prioritize extractions with:
- `metadata.quality_score` ≥ 85
- `metadata.extraction_confidence` ≥ 85
- Niche alignment with current project (when available)

### Step 2: Extract Narrative Flow Patterns

From each qualifying extraction, pull:
- **Section sequence** — the ordered list of sections with purpose, dominant element, and proof density
- **Pacing rhythm** — how the promotion accelerates/decelerates through sections
- **Transition techniques** — specific language patterns used between sections
- **Callbacks and references** — how later sections reference earlier claims
- **Open loops and closures** — how attention is maintained through incomplete patterns

Build a **structural pattern library** organized by:
- Niche (health, wealth, personal development)
- Big idea type (conspiracy, revelation, transformation, prophecy)
- Format (magalog, VSL, sales page, letter)

### Step 3: Extract Proof Density Distribution

From each qualifying extraction, calculate:
- **Proof density per section** — sparse, moderate, heavy
- **Proof type distribution** — which proof types appear in which sections
- **Proof escalation pattern** — how proof strength increases through the argument
- **Knockout proof placement** — where the strongest proof element appears

### Step 4: Extract CPB-Equivalent Structures

TIER1 extractions don't use the CPB label, but the pattern exists. Identify:
- **Claim patterns** — how claims are introduced in each section
- **Proof pairing** — how proof immediately follows claims
- **Benefit delivery** — how benefits are expressed after proof
- **Chunk boundaries** — where one CPB-equivalent unit ends and another begins

### Step 5: Extract Transition Catalogs

Compile a catalog of transition techniques from elite controls:
- **Section-to-section transitions** — how major sections flow into each other
- **Claim-to-proof transitions** — how claims introduce their evidence
- **Proof-to-benefit transitions** — how proof leads to "what this means for you"
- **Escalation transitions** — "And not just that...", "But it gets better...", etc.

### Step 6: Build Vault Intelligence Object

Assemble all extracted patterns into a structured intelligence object.

### Step 7: Specimen Tiering

```
FOR each structural pattern specimen in loaded data:

  CALCULATE structural_completeness_score:
    +2 IF section_sequence present with 3+ sections (each with name + function + position)
    +1 IF pacing_rhythm documented
    +1 IF transition_techniques present with 2+ entries
    +1 IF proof_density_distribution mapped across sections
    +1 IF cpb_equivalent_patterns identified
    +1 IF format classified (magalog, VSL, sales_page, letter)
    +1 IF quality_score >= 85 (from extraction metadata)
    MAX_SCORE: 8

  SORT specimens by structural_completeness_score DESC
  ASSIGN tier:
    Gold = top 10% (structural_completeness_score >= 7)
    Silver = next 15% (structural_completeness_score >= 5)
    Standard = remainder

  TAG each specimen with:
    tier: enum[gold, silver, standard]
    structural_completeness_score: integer (0-8)

  OUTPUT tier_distribution:
    total_specimens: integer
    gold_count: integer
    silver_count: integer
    standard_count: integer
```

### Step 8: Type-Matched Retrieval

```
DEFINE match_fields:
  primary: format (magalog, VSL, sales_page, letter)
  secondary: big_idea_type (conspiracy, revelation, transformation, prophecy, discovery)
  tertiary: niche (health, wealth, personal_development, other)

WHEN campaign_context is available:
  STRATEGY 1 — exact_match:
    FILTER specimens WHERE format == target AND big_idea_type == target AND niche == target_niche
    PRIORITIZE Gold tier first, then Silver

  STRATEGY 2 — format_type_match:
    FILTER specimens WHERE format == target AND big_idea_type == target (any niche)
    PRIORITIZE Gold tier, same-niche Silver, then cross-niche Silver

  STRATEGY 3 — cross_format:
    RETURN all Gold specimens across formats
    INCLUDE Silver specimens matching target big_idea_type or niche

WHEN campaign_context is NOT available:
  RETURN full specimen set with tier metadata attached
  SORT by structural_completeness_score DESC

OUTPUT retrieval_result:
  matched_specimens: [specimen]
  match_strategy_used: enum[exact_match, format_type_match, cross_format, full_set]
  gold_specimens_available: integer
  silver_specimens_available: integer
```
