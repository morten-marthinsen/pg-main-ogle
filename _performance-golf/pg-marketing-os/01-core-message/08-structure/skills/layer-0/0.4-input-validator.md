# Layer 0.4 — Input Validator

## Purpose

Validate that all loaded upstream packages, vault intelligence, and teaching indexes meet minimum thresholds required for campaign argument engineering. Determine method viability and flag blockers before Layer 1 begins.

---

## Input Requirements

```yaml
inputs_to_validate:
  upstream_context: object  # from 0.1-upstream-loader
  vault_intelligence: object  # from 0.2-vault-intelligence-loader
  teaching_index: object  # from 0.3-teachings-loader
```

---

## Execution Protocol

### Step 1: Validate Upstream Package Completeness

```python
def validate_upstream_completeness(upstream_context):
    """Check all required fields are present and populated."""

    required_checks = {
        "mechanism": [
            ("name", "Mechanism name required for thesis and segues"),
            ("type", "Mechanism type required for strategy selection"),
            ("explanation_text", "Mechanism explanation required for CPB chunks"),
            ("e5_type", "E5 classification required for strategy calibration"),
            ("uniqueness_claim", "Uniqueness claim required for differentiation chunks"),
        ],
        "proof_inventory": [
            ("total_elements", "Proof count required for density planning"),
            ("overall_strength", "Overall strength required for threshold check"),
            ("by_category", "Category breakdown required for proof mapping"),
        ],
        "root_cause": [
            ("expression", "Root cause expression required for thesis connection"),
            ("what_they_think", "Three-part structure required for gap mapping"),
            ("what_it_really_is", "Three-part structure required for gap mapping"),
            ("why_nothing_worked", "Three-part structure required for gap mapping"),
        ],
        "promise": [
            ("primary_statement", "Primary promise required for thesis and segues"),
            ("emotional_frame", "Emotional frame required for benefit writing"),
        ],
        "offer": [
            ("product_name", "Product name required for SIN segue"),
            ("positioning", "Positioning required for offer transition"),
        ],
    }

    missing = []
    for package, fields in required_checks.items():
        for field, reason in fields:
            if not upstream_context.get(package, {}).get(field):
                missing.append({"package": package, "field": field, "reason": reason})

    return missing
```

### Step 2: Validate Proof Strength Threshold

```python
def validate_proof_strength(upstream_context):
    """Proof inventory overall_strength must be >= 40."""
    strength = upstream_context["proof_inventory"]["overall_strength"]
    if strength < 40:
        return {
            "status": "fail",
            "score": strength,
            "minimum": 40,
            "reason": "Insufficient proof strength for defensible argument construction"
        }
    elif strength < 60:
        return {
            "status": "warning",
            "score": strength,
            "minimum": 40,
            "reason": "Proof strength is marginal — argument may lack density"
        }
    else:
        return {"status": "pass", "score": strength}
```

### Step 3: Validate Vault Intelligence Completeness

Check:
- `vault_intelligence.loading_status` is "complete" or "partial" (NOT "empty")
- At least 3 structural patterns loaded
- Transition catalog populated with minimum 5 transitions
- If status is "partial", log warning but do not block

### Step 4: Validate Teaching Index Completeness

Check:
- All 9 gap questions indexed
- All 8 proof types indexed
- All 3 claim rules documented
- Simple segue template present
- SIN segue template present
- Makepeace framework principles present

---

## Quality Gates

**PASS if:**
- [ ] All required upstream fields present (zero blockers)
- [ ] Proof inventory overall_strength ≥ 40
- [ ] Teaching index complete (9 gap questions, 8 proof types, templates present)
- [ ] At least 1 argument strategy viability = "high"
- [ ] Vault intelligence status ≠ "empty"

**PASS WITH WARNINGS if:**
- [ ] PASS criteria met AND:
- [ ] Proof strength 40-60 (marginal)
- [ ] Vault intelligence = "partial"
- [ ] Only 1 strategy at "high" viability

**FAIL if:**
- [ ] Any required upstream field missing (blocker)
- [ ] Proof strength < 40
- [ ] Teaching index incomplete (missing gap questions or proof types)
- [ ] Zero strategies at "high" or "medium" viability

---

## Trigger-Template Refusals

**Missing mechanism package:**
> "BLOCKER: mechanism_package not loaded. Cannot engineer campaign argument without mechanism. Run 04-mechanism first."

**Missing proof inventory:**
> "BLOCKER: proof_inventory not loaded. Cannot construct CPB chunks without proof. Run 02-proof-inventory first."

**Proof strength below threshold:**
> "BLOCKER: proof_inventory.overall_strength = [X], below minimum of 40. Insufficient evidence for defensible argument. Strengthen proof inventory before proceeding."

**Teaching index incomplete:**
> "BLOCKER: teaching_index missing [X]. Cannot proceed without complete framework reference. Re-run 0.3-teachings-loader."

---

## Handoff

Pass `validation_report` to:
- **STRUCTURE-AGENT.md** (for Gate 0 evaluation)
- If `gate_0_recommendation` = "proceed" or "proceed_with_caution" → advance to Layer 1
- If `gate_0_recommendation` = "halt" → trigger feedback bus or human checkpoint
