# Skill 3.3: Defensibility Validator

**Version:** 1.0
**Layer:** 3 (Validation)
**Type:** Validation Gate
**Dependencies:** layer2-output.json (from 2.6), layer1-output.json (from 1.6), layer0-output.json (from 0.6)
**Output:** defensibility_validation.json

---

## Purpose

Verify every recommendation traces back to valid swipe sources — IDs exist in the vault, evidence trails are complete for all candidate elements, strategy claims match Layer 1 intelligence data, and quote counts are accurate. No undefended claims allowed.

---

## The Question

**"Can every claim, count, and recommendation be traced back to a specific, verified source?"**

---

## Input Schema

```yaml
input:
  layer2_output: from 2.6 (all candidates with swipe_sources arrays)
  layer1_output: from 1.6 (intelligence data for cross-reference)
  from_layer0_output:
    component_pools: object (for swipe_id existence verification)
```

## Process Logic

### Check 1: Swipe Source IDs Exist

```
COLLECT all swipe_source IDs from layer2_output:
  FROM each candidate:
    root_cause_reframe.swipe_sources
    mechanism.swipe_sources
    proof_architecture.swipe_sources
    each headline.swipe_source
    each lead.swipe_source

DEDUPLICATE to get unique IDs

FOR each unique swipe_id:
  SEARCH layer0_output.component_pools for this ID:
    CHECK mechanisms[].swipe_sources
    CHECK headline_formulas[].swipe_sources
    CHECK proof_architectures[].swipe_sources
    CHECK emotional_combinations[].swipe_sources
    CHECK root_cause_techniques[].swipe_sources
    CHECK big_idea_types[].swipe_sources

  IF found in any pool: VALID
  ELSE: INVALID
    LOG: "Swipe source [id] referenced by candidate [candidate_id] [element]
          does not exist in vault component pools"
```

### Check 2: Evidence Trails Complete

```
FOR each candidate:
  COUNT sources per element type:
    big_idea_sources = candidate.swipe_evidence_count (or core_concept sources)
    mechanism_sources = mechanism.swipe_sources.length
    root_cause_sources = root_cause_reframe.swipe_sources.length
    headline_sources = COUNT of headlines with non-null swipe_source
    lead_sources = COUNT of leads with non-null swipe_source
    proof_sources = proof_architecture.swipe_sources.length

  VERIFY minimums:
    mechanism_sources >= 2
    root_cause_sources >= 3
    proof_sources >= 3
    headline_sources == total_headlines (all must have sources)
    lead_sources == total_leads (all must have sources)

  IF any element below minimum:
    complete = false
    ADD to missing_elements: "[element_type] has [count] (needs [minimum])"
```

### Check 3: Recommendations Match Intelligence

```
FOR each candidate:
  READ strategy_type (whitespace_opportunity, dominant_pattern, hybrid)

  IF strategy_type == "whitespace_opportunity":
    VERIFY candidate's gap_exploited exists in
      layer1_output.gap_identification.whitespace_opportunities
    MATCH by gap_description or big_idea_type

  IF strategy_type == "dominant_pattern":
    VERIFY candidate's big_idea_type exists in
      layer1_output.pattern_analysis.dominant_patterns
    MATCH by big_idea_type name

  IF strategy_type == "hybrid":
    VERIFY big_idea_type in dominant_patterns
    AND mechanism in whitespace_mechanisms or safe_mechanisms

  VERIFY emotional_combination:
    CHECK candidate's primary + secondary drivers exist as a pair in
      layer1_output.emotional_mapping.winning_combinations

  IF any verification fails:
    verified = false
    LOG: "Candidate [id] claims [strategy] but [specific mismatch with L1 data]"
```

### Check 4: Quote Counts Accurate

```
FOR each candidate:
  READ claimed counts:
    pain_quote_count from emotional_driver_combination
    hope_quote_count from emotional_driver_combination

  READ actual counts from layer1_output:
    FIND matching driver in emotional_mapping.primary_drivers
    actual_pain = primary_drivers[matching].pain_quote_count
    actual_hope = primary_drivers[matching].hope_quote_count

  COMPARE:
    IF claimed_pain == actual_pain AND claimed_hope == actual_hope:
      accurate = true
    ELSE:
      accurate = false
      LOG: "Candidate [id] claims [claimed_pain] pain quotes but L1 shows [actual_pain]"
```

### Final Determination

```
status = IF (sources_invalid == 0 AND
             candidates_incomplete == 0 AND
             strategies_unsupported == 0 AND
             counts_mismatched == 0)
         THEN "PASS"
         ELSE "FAIL"
```

---

## Constraints

- NEVER pass if ANY swipe source ID is invalid (zero tolerance)
- NEVER pass if ANY evidence trail is incomplete
- NEVER accept a strategy claim without verifying against L1 data
- NEVER accept quote counts without cross-referencing L1 emotional_mapping
- ALWAYS check ALL candidates (not just first or random sample)
- ALWAYS report the specific element and candidate for every failure
- ALWAYS distinguish between "ID not found" vs "ID found but wrong element"

---

## Quality Criteria

1. All unique swipe_source IDs verified against vault data
2. Evidence trail completeness checked per element type per candidate
3. Strategy claims matched to specific L1 intelligence sections
4. Quote counts compared to exact L1 values (not ranges)
5. Every failure includes candidate_id, element_type, and specific issue
6. Status is binary: any invalid source = FAIL
7. Issues array populated with actionable fix descriptions
