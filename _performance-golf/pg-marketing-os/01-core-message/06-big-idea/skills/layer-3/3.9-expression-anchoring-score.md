# 3.9 Expression Anchoring Score

**Version:** 1.0
**Layer:** 3 (Validation)
**Type:** Scoring (Data-Grounded Validation)
**Purpose:** Score wrapped Big Idea candidates against audience research quotes, TIER1 patterns, and FSSIT candidates to validate that creative wrapping preserved data grounding
**Shared Protocol:** `~system/protocols/EXPRESSION-ANCHORING-PROTOCOL.md`
**Position:** After 3.8-anchor-distance-ratio, BEFORE 3.6-validation-gate

---

## Purpose

This microskill VALIDATES that the creative wrapping (Phase B) preserved the FSSIT resonance and audience grounding established in Phase A. Unlike Skills 03 and 04 where anchoring scores GENERATE additional candidates, the Big Idea skill already has FSSIT-first generation (v5.0). This microskill confirms that the creative packaging didn't drift away from the data foundation.

**Key distinction:** Big Ideas deliberately violate audience expectations (schema distance 4-8). The anchoring score does NOT penalize high schema distance — instead, it checks that the RESONANCE anchor survived the creative wrapping process.

---

## When to Use

- **Always.** This microskill is mandatory for all Big Idea validation.
- Executes AFTER 3.8-anchor-distance-ratio and BEFORE 3.6-validation-gate.
- Requires research quotes, TIER1 expression reference, and FSSIT data to be loaded.

## Execution Protocol

### Step 1: Load Scoring Inputs

```
1. LOAD wrapped Big Idea candidates from Layer 2B output
   - Each candidate includes: creative wrapper, headlines (10+), leads (3+), Big Idea type

2. LOAD top 20 research quotes from Pain, Hope, and FSSIT-adjacent buckets
   - Select by emotional intensity and relevance to the Big Idea's strategic synthesis

3. LOAD tier1_expression_reference.json from Layer 0 outputs
   - Contains Big Idea type patterns, headline patterns, FSSIT patterns from TIER1 vault

4. LOAD FSSIT-RANKING.md (or latent_resonance_field.json)
   - Top 5 FSSIT candidates — these are REQUIRED for Big Idea (not optional)

5. LOAD schema distance scores from 3.7 output
   - Used for the interaction score (Dimension 4)
```

### Step 2: Score Each Wrapped Candidate

For each wrapped Big Idea candidate:

```
DIMENSION 1 — QUOTE PENETRATION (30%):
  (Weight reduced from 40% for Big Idea — creative wrapping intentionally
   departs from audience vocabulary to create surprise)
  FOR each of top 20 research quotes:
    Score whether the audience would FEEL this Big Idea speaks to their experience (1-10)
    - Not vocabulary match — EMOTIONAL RECOGNITION
    - Would the audience say "that's exactly my situation" even if the words are different?
  TAKE top 5 overlap scores and average them
  RECORD the 5 quotes with scores as citations

DIMENSION 2 — TIER1 PATTERN MATCH (20%):
  (Weight reduced from 30% — Big Ideas need to be novel, not pattern-following)
  COMPARE wrapped candidate against TIER1 Big Idea patterns:
    - Big Idea type: Does it follow a proven type with appropriate adaptation?
    - Headline pattern: Does the headline use a validated structural pattern?
    - Lead architecture: Does the lead follow a conversion-proven opening approach?
    - Schema violation type: Does the schema violation follow patterns that worked?
  SCORE overall pattern alignment (1-10)
  CITE specific TIER1 sources

DIMENSION 3 — FSSIT ECHO (30%):
  (Weight maintained — FSSIT preservation is the PRIMARY validation for Big Ideas)
  FOR each of top 5 FSSIT candidates:
    Score whether this wrapped Big Idea STILL activates the FSSIT (1-10)
    - Does the creative wrapper preserve the "finally someone said it" power?
    - Or did the wrapping process abstract away the FSSIT resonance?
  TAKE highest single FSSIT score
  CITE the FSSIT candidate that scored highest

  CRITICAL CHECK: Compare FSSIT Echo score with the FSSIT Echo of the
  pre-wrapping concept (from Phase A). If wrapping REDUCED FSSIT Echo by
  more than 2 points, FLAG: "Creative wrapping may have diluted FSSIT resonance."

DIMENSION 4 — SCHEMA DISTANCE × ANCHORING INTERACTION (20%):
  (Unique to Big Idea — measures the optimal tension between surprise and grounding)

  LOAD schema_distance from 3.7 output
  LOAD anchoring_average = mean(D1, D3)  # Average of resonance dimensions

  CALCULATE interaction_score:
    IF schema_distance 4-6 AND anchoring_average ≥ 7:
      interaction = 9-10 (sweet spot: moderate surprise + high grounding)
    IF schema_distance 7-8 AND anchoring_average ≥ 6:
      interaction = 8-9 (good: high surprise + solid grounding)
    IF schema_distance 7-8 AND anchoring_average < 6:
      interaction = 5-6 (WARNING: high surprise but lost grounding)
    IF schema_distance < 4 AND anchoring_average ≥ 7:
      interaction = 6-7 (safe but not surprising enough)
    IF schema_distance > 8:
      interaction = 4-5 (RISKY: may confuse regardless of anchoring)
    IF anchoring_average < 5:
      interaction = 3-4 (DANGER: lost audience connection)

COMPOSITE = (D1 × 0.30) + (D2 × 0.20) + (D3 × 0.30) + (D4 × 0.20)
```

### Step 3: Validate and Flag

```
FOR each candidate:
  IF composite ≥ 6.0: STATUS = "validated"
  IF composite 5.0-5.9: STATUS = "flagged" (add revision guidance)
  IF composite < 5.0: STATUS = "concern" (flag for human attention)

NOTE: Big Idea anchoring scores do NOT block progression.
The 3.6-validation-gate makes the final go/no-go decision.
Anchoring scores INFORM that decision by showing which candidates
maintained their data grounding through creative wrapping.

FSSIT DRIFT CHECK:
  IF any candidate's FSSIT Echo dropped >2 points from Phase A concept:
    ADD to revision_guidance: "FSSIT resonance may have been diluted during
    creative wrapping. Consider revising wrapper to preserve the original
    FSSIT connection: [cite original FSSIT candidate]"
```

---

## Handoff

Anchoring scores feed into **3.6-validation-gate** as additional validation input. The validation gate receives:
- Per-candidate anchoring scores and statuses
- FSSIT drift flags (candidates where creative wrapping may have diluted resonance)
- Schema-anchoring interaction zones (identifying the optimal surprise-grounding balance)

The validation gate does NOT need to require 6.0+ anchoring scores for passage — it uses them as one input among several (alongside volume, quantification, defensibility, actionability, anti-slop).

---

## Anti-Patterns

| Weak | Strong |
|------|--------|
| Penalizing high schema distance (Big Ideas SHOULD violate expectations) | Checking that high schema distance is paired with high FSSIT Echo |
| Treating TIER1 pattern match as the primary dimension | Treating FSSIT Echo preservation as the primary dimension for Big Ideas |
| Ignoring FSSIT drift (assuming wrapping preserved resonance) | Explicitly comparing pre-wrapping FSSIT with post-wrapping FSSIT |
| Blocking candidates based on anchoring alone | Using anchoring as informational input for the validation gate |
