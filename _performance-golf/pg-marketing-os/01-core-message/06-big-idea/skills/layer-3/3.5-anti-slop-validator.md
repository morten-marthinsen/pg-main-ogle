# Skill 3.5: Anti-Slop Validator

**Version:** 1.0
**Layer:** 3 (Validation)
**Type:** Validation Gate
**Dependencies:** layer2-output.json (from 2.6)
**Output:** anti_slop_validation.json

---

## Purpose

Enforce language quality standards beyond basic actionability — catch AI-generated telltale patterns, generic naming, weak copy constructions, and lazy prose markers that indicate the output was generated without sufficient specificity or craft. This is the "would a human copywriter be embarrassed by this?" gate.

---

## The Question

**"Does this output read like it was written by a skilled copywriter — or does it have the hallmarks of lazy AI generation?"**

---

## Input Schema

```yaml
input:
  layer2_output: from 2.6 (all candidate text — headlines, leads, architectures)
```

## Process Logic

### Check 1: AI Tell Patterns

```
SCAN all text fields across all candidates for:

CRITICAL (auto-FAIL if found in leads/hooks):
  /^Imagine\b/i              → imagine_openers
    (Opening leads/hooks starting with "Imagine")
  /^You see,/i               → you_see_openers
    (Paragraph openers with "You see,")

WARNING (FAIL if 3+ occurrences total):
  /Let'?s dive in/i          → lets_dive_in
  /In today'?s (world|fast-paced|modern)/i → in_todays_world
  /It'?s no secret/i         → its_no_secret
  /Here'?s the thing/i       → heres_the_thing
  /game.?chang/i             → game_changer
  /unlock (your|the) (potential|power|secret)/i → unlock_potential
  /\bjourney\b/i             → journey_language
  /\bempower(ing|ed|ment)?\b/i → empower_language

FOR each match:
  RECORD: pattern, candidate_id, section, surrounding text (50 chars)
```

### Check 2: Generic Naming

```
SCAN mechanism names, enemy names, and promise statements:

GENERIC MECHANISM patterns (CRITICAL):
  "The [Adjective] [Method|System|Way|Approach]"
    WHERE adjective is: Natural, Simple, Easy, New, Special, Unique, Revolutionary
  "The [Adjective] [Generic-Noun]"
    WHERE generic-noun is: Solution, Answer, Fix, Cure, Secret, Key, Formula
  ANY mechanism name < 3 words without a specific/technical term

UNNAMED ENEMY patterns (CRITICAL):
  "bad habits"
  "stress"
  "modern life"
  "busy lifestyle"
  "poor choices"
  "the wrong approach"
  (Any enemy that isn't a NAMED entity — industry, company, belief system, specific practice)

VAGUE PROMISE patterns (WARNING):
  "feel better"
  "improve your health"
  "get results"
  "achieve your goals"
  "live your best life"
  "transform your life"
  (Any promise without a specific, measurable outcome)

FOR each match:
  RECORD: type, candidate_id, current_text, why it fails
```

### Check 3: Weak Copy Patterns

```
SCAN lead text (opening_hook, root_cause_intro, mechanism_tease):

ADVERB OVERUSE (WARNING if 5+ per candidate):
  /\b(really|very|truly|absolutely|incredibly|extremely|literally|basically|actually|essentially)\b/i

WEASEL WORDS (CRITICAL if in mechanism claims):
  /\b(may help|could potentially|might be able|possibly|perhaps|it seems)\b/i
  (These undermine mechanism credibility)

FILLER PHRASES (WARNING):
  "the fact that"
  "in order to"
  "it is important to note"
  "it goes without saying"
  "needless to say"
  "at the end of the day"
  "when all is said and done"
  "the reality is"

PASSIVE VOICE (WARNING if >30% of sentences in a lead):
  Pattern: [to be verb] + [past participle]
  "is caused by", "was discovered", "has been shown", "can be achieved"
  COUNT passive sentences / total sentences per lead
  IF ratio > 0.30: FLAG

FOR each match:
  RECORD: pattern_type, candidate_id, section, text_snippet, fix_suggestion
  fix_suggestion examples:
    Adverb: "Remove 'really' — the claim should stand without intensifiers"
    Weasel: "Replace 'may help' with direct claim: 'helps' or 'fixes'"
    Filler: "Delete 'the fact that' — start with the actual claim"
    Passive: "Rewrite in active voice: '[subject] [verb] [object]'"
```

### Check 4: Repetition Check

```
CROSS-CANDIDATE DUPLICATES:
  COMPARE all headlines across all 5 candidates
  IF any headline_text matches another (exact or >80% similar): FLAG
  COMPARE all opening_hooks across candidates
  IF any hook is structurally identical (same pattern, different nouns): FLAG

WITHIN-CANDIDATE REPEATS:
  FOR each candidate:
    COMPARE headline structures (formula_type + emotional_driver)
    IF same formula+driver combo used 3+ times: FLAG
    COMPARE lead opening structures
    IF 2+ leads start with same structural pattern: FLAG

FOR each match:
  RECORD: type, candidate_ids, duplicate text or structural description
```

### Severity Classification

```
CLASSIFY each violation:

CRITICAL (any count = FAIL):
  - imagine_openers in lead hooks
  - you_see_openers in paragraphs
  - generic_mechanisms (unnamed/vague mechanism names)
  - unnamed_enemies (non-specific enemy)
  - weasel_words in mechanism claims

WARNING (FAIL if 5+ total):
  - Other AI tell patterns (3+ occurrences of any single pattern)
  - adverb_overuse (5+ per candidate)
  - filler_phrases (3+ per candidate)
  - passive_voice_excess (>30% per lead)
  - vague_promises

ADVISORY (noted, doesn't affect gate):
  - Occasional adverb (1-2 per candidate)
  - Single filler phrase
  - <30% passive voice
```

### Final Determination

```
status = IF (critical_violations == 0 AND warning_violations < 5)
         THEN "PASS"
         ELSE "FAIL"
```

---

## Constraints

- NEVER pass output with "Imagine..." opening leads (most common AI tell)
- NEVER pass output with "You see," paragraph openers (second most common)
- NEVER pass generic mechanism names that a competitor could also claim
- NEVER pass enemies that aren't specifically named entities
- NEVER pass weasel words in mechanism efficacy claims
- ALWAYS scan ALL text fields (not just leads — check architectures too)
- ALWAYS provide fix_suggestion for weak copy patterns
- ALWAYS classify severity (critical vs warning vs advisory)

---

## Quality Criteria

1. All 10 AI tell patterns scanned across all text
2. Generic naming check covers mechanisms, enemies, AND promises
3. Weak copy patterns include adverbs, weasels, fillers, AND passive voice
4. Repetition check covers both cross-candidate AND within-candidate
5. Severity classification follows exact rules (not subjective)
6. Every violation includes candidate_id, section, and text snippet
7. Fix suggestions are actionable (tell writer what to do instead)
8. Critical violations auto-FAIL regardless of count
