# Skill 1.4: Emotional Mapper

**Version:** 1.1
**Layer:** 1 (Intelligence)
**Type:** Analysis
**Dependencies:** layer0-output.json (from 0.6)
**Output:** emotional_mapping.json
**Reference:** References/CORE-WOUNDS-FRAMEWORK.md (for wound indicators)

---

## Purpose

Map pain/hope intensity to the 9 emotional drivers, then identify the highest-performing driver combinations from vault data. Produces a ranked emotional landscape showing which driver pairings have both audience evidence (quotes) AND proven copy performance (swipe scores).

**Additionally:** Identify dominant Core Wounds (2-3 from 11 wounds) from quote patterns for downstream copy strategy. Core Wounds are deeper psychological drivers that underlie emotional language. See References/CORE-WOUNDS-FRAMEWORK.md for the complete wound framework.

---

## The Question

**"Which emotional driver combinations have the strongest audience evidence AND highest proven copy performance?"**

---

## Process Logic

### Step 1: Map Quotes to Emotional Drivers

```
DEFINE driver mapping rules:

PAIN QUOTE → DRIVER MAPPING:
  FEAR_PROTECTION: Language contains "afraid", "terrified", "worried", "what if",
                   "scared", "anxious about", "fear of"
  FRUSTRATION_ANGER: Language contains "frustrated", "angry", "ridiculous",
                     "sick of", "tired of", "can't believe"
  SHAME_DIGNITY: Language contains "embarrassed", "ashamed", "judged",
                 "people look at me", "humiliated", "don't want anyone to see"
  REVENGE_VINDICATION: Language contains "prove them wrong", "show them",
                       "make them regret", "get back at"

HOPE QUOTE → DRIVER MAPPING:
  HOPE_POSSIBILITY: Language contains "want to", "imagine if", "dream of",
                    "one day", "what if I could"
  LIBERATION_FREEDOM: Language contains "free from", "no longer", "finally done with",
                      "break free", "escape from"
  GREED_OPPORTUNITY: Language contains "could finally", "think of what",
                     "opportunity to", "make more", "achieve"
  BELONGING_TRIBE: Language contains "be part of", "like others who",
                   "join", "community", "people like me"
  CURIOSITY: Language contains "wonder", "how does", "what's really",
             "the truth about", "secret"

FOR each of 10 pain quotes:
  ASSIGN to 1-2 drivers (primary mapping)
  INCREMENT driver.pain_quote_count

FOR each of 10 hope quotes:
  ASSIGN to 1-2 drivers (primary mapping)
  INCREMENT driver.hope_quote_count

ALSO MAP:
  top_enemies → FRUSTRATION_ANGER or REVENGE_VINDICATION
  top_wants → HOPE_POSSIBILITY or GREED_OPPORTUNITY
```

### Step 1.5: Derive Core Wounds from Quote Patterns

```
REFERENCE: References/CORE-WOUNDS-FRAMEWORK.md

DEFINE wound indicator patterns:

WOUND LANGUAGE INDICATORS (search in ALL quotes):
  ABANDONMENT: "don't leave", "alone", "isolated", "lost everyone", "nobody there"
  MISTRUST: "can't trust", "they lied", "scam", "taken advantage", "ripped off"
  DEPENDENCE: "tell me what to do", "need someone", "don't know how", "too complicated"
  VULNERABILITY: "terrified", "catastrophe", "any moment", "what if [worst case]"
  DEPRIVATION: "never get", "can't have", "others have", "deserve more", "cheated"
  FAILURE: "tried everything", "nothing works", "always fail", "inadequate"
  DEFECTIVENESS: "something wrong with me", "not good enough", "flawed", "ashamed"
  EXCLUSION: "don't belong", "outsider", "not like them", "left out", "different"
  SUBJUGATION: "always putting others first", "sacrifice", "no time for myself"
  PERFECTIONISM: "never enough", "not good enough yet", "need to be the best"
  ENTITLEMENT: "I deserve", "shouldn't have to", "rules don't apply", "special"

FOR each pain quote + hope quote:
  SCAN for wound indicator language
  TALLY indicator counts per wound

IDENTIFY dominant_wounds (top 2-3):
  RANK wounds by indicator_count
  SELECT top 2-3 with meaningful indicator presence (>= 2 quotes)

  IF top wounds include incompatible pair (Entitlement + Subjugation):
    FLAG incompatible_wounds
    NOTE in derivation_rationale

FOR each dominant wound:
  RECORD indicator_phrases (2-3 actual phrases from quotes)
  MAP to copy_application from Core Wounds Framework:
    ABANDONMENT → "I'll be with you every step" bonding
    MISTRUST → Exposé angles, transparency copy
    DEPENDENCE → Step-by-step systems, done-for-you
    VULNERABILITY → Fear of loss, protection positioning
    DEPRIVATION → Scarcity/urgency, abundance fantasy
    FAILURE → "When all else fails", new mechanism
    DEFECTIVENESS → Transformation promises, before/after
    EXCLUSION → Velvet rope, insider access
    SUBJUGATION → Guilt-based, "you've done so much for others"
    PERFECTIONISM → Premium positioning, "the best"
    ENTITLEMENT → Insider secrets, "game the system"

  MAP to driver_correlation from emotional driver mapping:
    ABANDONMENT → FEAR_PROTECTION
    MISTRUST → FRUSTRATION_ANGER
    VULNERABILITY → FEAR_PROTECTION
    DEPRIVATION → GREED_OPPORTUNITY
    FAILURE → SHAME_DIGNITY
    DEFECTIVENESS → SHAME_DIGNITY
    EXCLUSION → BELONGING_TRIBE (inverse)
    SUBJUGATION → LIBERATION_FREEDOM (inverse)
    ENTITLEMENT → GREED_OPPORTUNITY

BUILD wound_derivation_rationale:
  "[X] quotes showed [wound1] language (e.g., '[phrase]').
   [Y] quotes showed [wound2] language (e.g., '[phrase]').
   These correlate with the high-intensity [driver] driver(s)."
```

### Step 2: Load Swipe Emotional Scores

```
LOAD component_pools.emotional_combinations

FOR each emotional_combination entry:
  READ primary_driver, secondary_driver
  READ usage_count, average_combined_score
  READ swipe_sources

BUILD driver_performance lookup:
  FOR each unique driver appearing as primary:
    CALCULATE:
      total_swipe_usage (sum of usage_count where this is primary)
      average_score_as_primary (weighted average)
```

### Step 3: Calculate Intensity Levels

```
FOR each of the 9 drivers:

  total_quote_support = pain_quote_count + hope_quote_count
  swipe_performance = swipe_average_score (as primary driver)

  ASSIGN intensity_level:
    HIGH: total_quote_support >= 5 AND swipe_average_score >= 8.0
    MEDIUM: total_quote_support >= 3 OR swipe_average_score >= 7.0
    LOW: total_quote_support < 3 AND swipe_average_score < 7.0

  BUILD intensity_rationale:
    "[pain_count] pain quotes + [hope_count] hope quotes mapped to this driver.
     Average score of [score] across [usage_count] swipes as primary driver."
```

### Step 4: Identify Winning Combinations

```
FROM component_pools.emotional_combinations:

FILTER to combinations where:
  average_combined_score >= 15.0 (out of 20 possible)
  OR usage_count >= 8

SORT by average_combined_score DESC

FOR each qualifying combination:
  LOOK UP pain_quote_support for primary_driver
  LOOK UP hope_quote_support for secondary_driver

  BUILD recommendation:
    "[usage_count] swipes use this pairing at [avg_score] avg combined score.
     Supported by [pain_count] [primary]-mapped pain quotes and
     [hope_count] [secondary]-mapped hope quotes."

  COLLECT swipe_sources (minimum 3)
```

### Step 5: Rank and Validate

```
SORT primary_drivers by:
  1. intensity_level (high first)
  2. total_quote_support DESC
  3. swipe_average_score DESC

VERIFY:
  - All 9 drivers appear in primary_drivers (even if low intensity)
  - All winning_combinations have 3+ swipe_sources
  - No intensity_rationale contains vague qualifiers

SET driver_intensity_rankings:
  highest_intensity = driver with highest (quote_support * swipe_score)
  total_drivers_mapped = 9
  high/medium/low counts from actual assignments
```

---

## Constraints

- NEVER use vague qualifiers (many, often, most, some, several, usually, typically, around, approximately)
- NEVER estimate quote counts — count actual quotes mapped per driver
- NEVER assign intensity without citing both quote count AND swipe score
- ALWAYS map all 9 drivers (even if some have low/zero intensity)
- ALWAYS cite minimum 3 swipe_sources for winning combinations
- ALWAYS preserve original quote text in example_pain_quotes / example_hope_quotes
- ALL averages must be calculated from actual emotional_combinations scores
- ALWAYS identify 2-3 dominant Core Wounds (never zero, never more than 3)
- NEVER combine incompatible wounds (Entitlement + Subjugation)
- ALWAYS cite actual quote phrases as wound indicators (not invented examples)
- Core Wounds are INFORMATIONAL — they inform copy strategy but are not gating criteria

---

## Quality Criteria

1. All 9 emotional drivers mapped (none skipped)
2. All quote counts are exact integers (counted from actual quote-to-driver assignments)
3. All swipe scores are calculated averages from emotional_combinations pool
4. Intensity levels follow threshold rules exactly
5. Winning combinations cite 3+ swipe sources
6. Recommendations contain only quantified claims (zero vague qualifiers)
7. Example quotes are preserved verbatim from Deep Research (not paraphrased)
8. Core wound mapping includes 2-3 dominant wounds with indicator counts
9. Wound indicator phrases are actual quotes (not invented)
10. Wound derivation rationale cites specific counts and phrase examples
11. Incompatible wounds flagged if present in top results
