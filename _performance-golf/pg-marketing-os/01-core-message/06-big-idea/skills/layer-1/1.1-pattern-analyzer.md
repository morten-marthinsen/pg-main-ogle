# Skill 1.1: Pattern Analyzer

**Version:** 1.0
**Layer:** 1 (Intelligence)
**Type:** Analysis
**Dependencies:** layer0-output.json (from 0.6)
**Output:** pattern_analysis.json

---

## Purpose

Identify which Big Idea types are proven winners in this market by ranking them by usage frequency, quality tier, and competitor saturation. Categorizes every pattern as Dominant, Rare High-Performer, or Avoid — with exact counts and swipe evidence.

---

## The Question

**"Which Big Idea types are proven winners, and which represent untapped whitespace?"**

---

```yaml
input:
  from_layer0_output:
```

## Process Logic

### Step 1: Rank Big Idea Types

```
LOAD component_pools.big_idea_types

FOR each big_idea_type entry:
  READ usage_count (exact integer from pool)
  READ average_quality_score (exact float from pool)

  ASSIGN quality_tier:
    IF average_quality_score >= 85: quality_tier = "Elite"
    IF average_quality_score >= 75 AND < 85: quality_tier = "Strong"
    IF average_quality_score >= 70 AND < 75: quality_tier = "Good"
    IF average_quality_score < 70: quality_tier = "Weak"
```

### Step 2: Cross-Reference with Competitor Mechanisms

```
LOAD deep_research_summary.competitor_mechanisms
LOAD deep_research_summary.mechanism_saturation

FOR each big_idea_type:
  COUNT how many competitor mechanisms align with this type
  ASSIGN competitor_saturation:
    IF competitor_count >= 10: "high"
    IF competitor_count >= 5 AND < 10: "medium"
    IF competitor_count >= 1 AND < 5: "low"
    IF competitor_count == 0: "none"
```

### Step 3: Categorize Patterns

```
FOR each big_idea_type:

  IF usage_count >= 20 AND average_quality_score >= 75:
    ASSIGN to dominant_patterns
    recommendation = "use"
    rationale = "[usage_count] swipe examples at [score] avg quality.
                 [saturation] competitor saturation ([count] competitors)."

  ELSE IF usage_count >= 3 AND usage_count <= 5
          AND average_quality_score >= 85:
    ASSIGN to rare_high_performers
    recommendation = "test"
    whitespace_opportunity = "Only [usage_count] examples but [score] quality.
                              [count] competitor(s) = [prime/moderate] whitespace."

  ELSE IF average_quality_score < 70:
    ASSIGN to avoid_patterns
    recommendation = "avoid"
    rationale = "Below quality threshold ([score]) despite [usage_count] examples."

  ELSE:
    ASSIGN to dominant_patterns IF usage_count >= 10 AND quality_tier != "Weak"
    ASSIGN to avoid_patterns OTHERWISE

  COLLECT example_swipes (minimum 3 swipe_id values from swipe_sources)
```

### Step 4: Flag Compound Signals

```
FOR each categorized pattern:
  IF competitor_saturation == "high" AND category == "dominant":
    ADD flag: "Proven but crowded"
  IF competitor_saturation == "low" AND category == "rare_high_performer":
    ADD flag: "Prime whitespace"
  IF competitor_saturation == "none" AND average_quality_score >= 80:
    ADD flag: "Untested whitespace — high potential"
```

---

## Constraints

- NEVER use vague qualifiers (many, often, most, some, several, usually, typically, around, approximately)
- NEVER estimate counts — all numbers must be calculated from actual pool data
- NEVER produce a recommendation without citing the exact usage_count and quality_score
- ALWAYS cite minimum 3 swipe_id values for "use" or "test" recommendations
- ALWAYS show competitor_count as exact integer (not ranges)
- ALL rationale strings must reference specific numbers from the data

---

## Quality Criteria

1. All counts are exact integers calculated from actual pool arrays
2. All quality tiers match the threshold rules exactly
3. All patterns assigned to exactly one category (no overlap)
4. All "use" recommendations cite 3+ swipe sources
5. All competitor saturation levels backed by exact competitor_count
6. Rationale strings contain only quantified claims (zero vague qualifiers)
7. summary_counts match actual array lengths of categorized patterns
