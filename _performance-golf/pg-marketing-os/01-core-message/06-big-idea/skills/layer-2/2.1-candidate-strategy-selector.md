# Skill 2.1: Candidate Strategy Selector

**Version:** 1.0
**Layer:** 2 (Generation)
**Type:** Strategy Selection
**Dependencies:** layer1-output.json (from 1.6)
**Output:** candidate_strategies.json

---

## Purpose

Select the 5 candidate strategies that will be developed into full Big Idea candidates. Picks 2 whitespace opportunities, 2 dominant proven patterns, and 1 hybrid — each matched to specific intelligence data from Layer 1. This is the strategic allocation step: WHAT to build, not HOW to build it.

---

## The Question

**"Which 5 strategic positions give us the best mix of proven safety and whitespace differentiation?"**

---

```yaml
input:
  from_layer1_output:
```

## Process Logic

### Step 1: Select Whitespace Candidates (2)

```
LOAD gap_identification.whitespace_opportunities
SORT by opportunity_tier DESC, swipe_avg_quality DESC

SELECT top 2 HIGH-priority whitespace opportunities

FOR each selected whitespace opportunity:
  MATCH big_idea_type:
    SEARCH pattern_analysis.rare_high_performers for type matching gap
    IF no match: USE closest dominant_pattern that aligns

  MATCH mechanism:
    SEARCH saturation_mapping.whitespace_mechanisms for mechanism matching gap
    IF no whitespace mechanism matches: USE safe_mechanism with lowest competitor count

  MATCH emotional_combination:
    SEARCH emotional_mapping.winning_combinations
    SELECT combination with highest pain_quote_support for this gap's theme

  MATCH root_cause_technique:
    SEARCH mechanism_mapping.root_cause_inventory
    SELECT technique with schwartz_stage_match == "perfect"
    IF no perfect match: SELECT "good" match

  BUILD selection_rationale:
    "[gap_description]: [swipe_count] swipes prove this at [quality] quality.
     [competitor_count] competitor(s) in this space.
     [pain_count] pain quotes support the frustration this addresses.
     Big Idea type: [type] ([source] — [usage_count] swipes, [quality] quality).
     Mechanism: [name] ([saturation] — [competitor_count] competitors).
     Emotional: [primary] + [secondary] ([score] combined, [pain_count] pain quotes)."
```

### Step 2: Select Dominant Pattern Candidates (2)

```
LOAD pattern_analysis.dominant_patterns
FILTER to:
  usage_count >= 20
  average_quality_score >= 75
  competitor_saturation != "high"

SORT by average_quality_score DESC, usage_count DESC

SELECT top 2 dominant patterns (different big_idea_types)

FOR each selected dominant pattern:
  MATCH mechanism:
    SEARCH saturation_mapping.safe_mechanisms
    SELECT mechanism with highest swipe_avg_score AND competitor_count <= 4

  MATCH emotional_combination:
    SELECT from winning_combinations the pair with highest swipe_usage_count

  MATCH root_cause_technique:
    SELECT technique with schwartz_stage_match == "perfect"
    MATCH emphasis to Schwartz stage

  BUILD selection_rationale:
    "Proven dominant pattern: [type] with [usage_count] swipe examples at [quality] quality.
     [competitor_saturation] saturation ([count] competitors) — room for differentiation.
     Safe mechanism: [name] ([count] competitors, [swipe_count] proof at [quality]).
     Emotional: [primary] + [secondary] ([score] combined, [usage_count] swipe proof)."
```

### Step 3: Select Hybrid Candidate (1)

```
COMBINE:
  big_idea_type = SELECT from dominant_patterns (proven safe)
  mechanism = SELECT from whitespace_mechanisms (differentiation)
  emotional_combination = SELECT highest-scoring combination not yet used
  root_cause_technique = SELECT technique with most example_reframes

ENSURE hybrid doesn't duplicate any component pairing from candidates 1-4

BUILD selection_rationale:
  "Hybrid approach: Proven [type] pattern ([usage_count] swipes) combined with
   whitespace mechanism [name] ([competitor_count] competitors, [quality] quality).
   This gives proven structural safety with mechanism-level differentiation.
   Emotional: [primary] + [secondary] ([score] combined).
   Root Cause: [technique] ([match] match for Stage [stage])."
```

### Step 4: Validate Allocation

```
VERIFY:
  - Exactly 2 whitespace candidates
  - Exactly 2 dominant candidates
  - Exactly 1 hybrid candidate
  - No two candidates share the same big_idea_type + mechanism combination
  - All 5 have schwartz_match of "perfect" or "good" (never "poor")
  - All 5 have mechanism_saturation of "whitespace" or "safe" (never "saturated" or "avoid")

IF any validation fails:
  SWAP the failing candidate's weakest component for next-best alternative
  RE-VALIDATE
```

---

## Constraints

- NEVER select a mechanism from "avoid" or "saturated" categories
- NEVER select a big_idea_type from "avoid_patterns"
- NEVER produce fewer than 5 candidates
- NEVER duplicate the same big_idea_type + mechanism pairing across candidates
- ALWAYS source big_idea_types from Layer 1 pattern_analysis (not invented)
- ALWAYS source mechanisms from Layer 1 saturation_mapping (not invented)
- ALWAYS source emotional combinations from Layer 1 emotional_mapping (not invented)
- ALL selection_rationale strings must cite exact counts (zero vague qualifiers)

---

## Quality Criteria

1. Exactly 5 candidates produced (2 whitespace, 2 dominant, 1 hybrid)
2. All components traced to specific Layer 1 data entries
3. No mechanism from "avoid" or "saturated" tiers
4. No big_idea_type from "avoid_patterns"
5. All schwartz_matches are "perfect" or "good"
6. All selection_rationale strings contain exact integer counts
7. No two candidates share the same type + mechanism pairing
8. Evidence summaries cite actual swipe counts and quality scores from L1
