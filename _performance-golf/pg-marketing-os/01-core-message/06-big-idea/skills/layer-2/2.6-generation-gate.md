# Skill 2.6: Generation Gate

**Version:** 1.0
**Layer:** 2 (Generation)
**Type:** Quality Gate
**Dependencies:** candidate_strategies.json (from 2.1), candidate_architectures.json (from 2.2), candidate_headlines.json (from 2.3), candidate_leads.json (from 2.4), candidate_proofs.json (from 2.5)
**Output:** layer2-output.json

---

## Purpose

Final quality gate for Layer 2 — validates all 5 candidates are complete with required volume (50+ headlines, 15+ leads, 5 proof plans), enforces anti-template and anti-description rules, and compiles everything into layer2-output.json for Layer 3 validation. Binary PASS/FAIL. No partial passes.

---

## The Question

**"Are all candidates complete, all creative elements actual copy (not templates/descriptions), and all volume thresholds met?"**

---

## Input Schema

```yaml
input:
  candidate_strategies: from 2.1
  candidate_architectures: from 2.2
  candidate_headlines: from 2.3
  candidate_leads: from 2.4
  candidate_proofs: from 2.5
```

## Gate Logic

### Check 1: Candidate Completeness

```
FOR each of 5 candidates:
  VERIFY architecture (from 2.2):
    core_concept.one_paragraph_description != empty
    root_cause_reframe.what_they_think != empty
    root_cause_reframe.what_it_really_is != empty
    root_cause_reframe.why_nothing_worked != empty
    mechanism.mechanism_name != empty
    mechanism.steps_or_components.length >= 3
    promise_statement.core_promise != empty
    enemy_identified.enemy_name != empty
    emotional_driver_combination.primary_driver != empty

IF any field empty or null:
  candidate_completeness = false
  LOG: "Candidate [id] missing [field] — route to 2.2"
```

### Check 2: Volume Thresholds

```
COUNT total_headlines across all candidates
COUNT total_leads across all candidates
COUNT total_proof_plans

VERIFY:
  total_headlines >= 50
  total_leads >= 15
  total_proof_plans == 5

PER CANDIDATE:
  headlines >= 10
  formula_types >= 5
  leads >= 3
  proof_complete = true

IF any threshold not met:
  volume_thresholds_met = false
  LOG: "[candidate_id] has only [count] [element] (needs [minimum])"
```

### Check 3: No Bracket Templates

```
SCAN all headline_text values across all candidates

SEARCH for bracket patterns: /\[.*?\]/

FOR each match:
  LOG violation: {candidate_id, headline_text, bracket_content}

IF any brackets found:
  no_bracket_templates = false
  LOG: "[count] headlines contain bracket templates"
```

### Check 4: No Description Leads

```
SCAN all lead sections (opening_hook, root_cause_intro, mechanism_tease)

SEARCH for description patterns:
  - "Opens with" (case-insensitive)
  - "Introduces the"
  - "Tells a story about"
  - "Teases the mechanism"
  - "Transitions to"
  - "Builds to"
  - "Establishes"

FOR each match:
  LOG violation: {candidate_id, section, matching_text}

IF any description language found:
  no_description_leads = false
  LOG: "[count] lead sections contain description language instead of actual copy"
```

### Check 5: No Placeholder Steps

```
SCAN all mechanism.steps_or_components arrays

SEARCH for placeholder patterns:
  - "Step [N]: First step" (or Second/Third)
  - "Take the supplement"
  - "Wait for results"
  - "See the difference"
  - Any step without a specific verb + object

FOR each match:
  LOG violation: {candidate_id, step_text}

IF any placeholders found:
  no_placeholder_steps = false
  LOG: "[count] mechanism steps are generic placeholders"
```

### Check 6: Evidence Citations

```
COLLECT all swipe_sources arrays from:
  - candidate_architectures (root_cause, mechanism)
  - candidate_headlines (each headline)
  - candidate_leads (each lead)
  - candidate_proofs (each proof plan)

VERIFY:
  - Root cause reframe: minimum 3 swipe sources
  - Mechanism: minimum 2 swipe sources
  - Each proof plan: minimum 3 swipe sources
  - No swipe_source is empty string or null

IF any citation requirements not met:
  evidence_citations_valid = false
  LOG: "[candidate_id] [element] has only [count] citations (needs [minimum])"
```

### Check 7: Strategy Allocation

```
COUNT strategy_types across 5 candidates:
  whitespace_opportunity count
  dominant_pattern count
  hybrid count

VERIFY:
  whitespace_opportunity == 2
  dominant_pattern == 2
  hybrid == 1

IF allocation incorrect:
  strategy_allocation_correct = false
  LOG: "Allocation mismatch: [actual counts] vs required 2/2/1"
```

### Check 8: Anti-Slop Scan

```
SCAN all text across all outputs:

  imagine_openers: COUNT leads where opening_hook starts with "Imagine"
  you_see_openers: COUNT paragraphs starting with "You see,"
  generic_mechanisms: COUNT mechanism names matching "[Adjective] [Generic]" pattern
                     (e.g., "The Natural Method", "The Simple System")
  vague_enemies: COUNT enemy names that are not specific entities
                 (e.g., "bad habits", "modern life", "stress")

total_violations = SUM of all categories

IF total_violations > 0:
  ADD to failure_report (warning level — does not auto-fail gate)
  LOG: "[count] anti-slop violations detected — review recommended"
```

### Final Determination

```
gate_status = IF (candidate_completeness AND volume_thresholds_met AND
                  no_bracket_templates AND no_description_leads AND
                  no_placeholder_steps AND evidence_citations_valid AND
                  strategy_allocation_correct)
              THEN "PASS"
              ELSE "FAIL"

IF gate_status == "PASS":
  COMPILE layer2-output.json:
    MERGE all 5 upstream outputs into unified candidate objects
    ADD generation_metadata with counts
  LOG: "LAYER 2 COMPLETE — 5 candidates, [X] headlines, [Y] leads, 5 proof plans"

IF gate_status == "FAIL":
  POPULATE failure_report
  DETERMINE routing:
    IF candidate_completeness fails: route to 2.2
    IF volume fails: route to 2.3 (headlines) or 2.4 (leads)
    IF bracket templates: route to 2.3
    IF description leads: route to 2.4
    IF placeholder steps: route to 2.2
    IF citations: route to relevant skill
    IF allocation: route to 2.1
  STOP: Cannot proceed to Layer 3
```

## Quality Criteria

1. All 7 gate checks are binary (no partial passes)
2. Volume audit counts actual array lengths (not estimates)
3. Template scan uses regex to catch all bracket patterns
4. Description scan checks all 3 lead sections (hook, intro, tease)
5. Placeholder scan catches generic step patterns
6. Anti-slop scan covers AI-tell patterns (Imagine, You see, generic names)
7. Failure report routes to exact upstream skill for each failure type
8. Compiled output contains all data from skills 2.1-2.5 merged per candidate
9. No advancement to Layer 3 unless gate_status == PASS
