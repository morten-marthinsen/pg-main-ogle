# Skill 2.3: Headline Generator

**Version:** 1.0
**Layer:** 2 (Generation)
**Type:** Creative Generation
**Dependencies:** candidate_architectures.json (from 2.2), layer0-output.json (from 0.6)
**Output:** candidate_headlines.json

---

## Purpose

Generate 10+ complete headlines for each of the 5 candidates using proven formula types from the vault. Every headline is finished copy — no bracket templates, no placeholders, no "fill in the blank" patterns. Matches emotional drivers, cites formula sources, and uses minimum 5 different formula types per candidate.

---

## The Question

**"Are all 50+ headlines complete, executable copy — not templates — using proven formulas matched to each candidate's architecture?"**

## Formula Library

### 10 Proven Headline Formula Types

```
1. WARNING
   Structure: "Warning: [Enemy/Problem] [Negative Consequence]"
   Emotional driver: FEAR_PROTECTION
   Use when: Enemy-based candidates, fear-driven markets

2. QUESTION
   Structure: "Could [Problem] Be Caused By [Root Cause]?"
   Emotional driver: CURIOSITY
   Use when: Root cause reframe candidates, curiosity-driven markets

3. HOW-TO
   Structure: "How [Unexpected Method] [Delivers Benefit]"
   Emotional driver: HOPE_POSSIBILITY
   Use when: Mechanism-forward candidates, solution-seeking audiences

4. DISCOVERY
   Structure: "[Authority] Discovers [Mechanism] That [Benefit]"
   Emotional driver: CURIOSITY + HOPE_POSSIBILITY
   Use when: Scientific/discovery mechanisms, authority-driven markets

5. SECRET
   Structure: "The [Adjective] Secret To [Benefit] (It's Not [False Solution])"
   Emotional driver: CURIOSITY + FRUSTRATION_ANGER
   Use when: Root cause reframe + enemy combination

6. NUMBER
   Structure: "[Number] [Timeframe] [Mechanism] To [Benefit]"
   Emotional driver: HOPE_POSSIBILITY + GREED_OPPORTUNITY
   Use when: Step-based mechanisms, specificity-driven markets

7. REASON-WHY
   Structure: "Why [Common Belief] Is Wrong About [Problem]"
   Emotional driver: FRUSTRATION_ANGER + CURIOSITY
   Use when: Contrarian angles, belief-enemy types

8. THEY-DON'T-WANT
   Structure: "[Enemy] Doesn't Want You To Know About [Mechanism]"
   Emotional driver: REVENGE_VINDICATION + CURIOSITY
   Use when: Industry-enemy types, conspiracy-adjacent angles

9. WHAT-IF
   Structure: "What If [Problem] Had Nothing To Do With [False Solution]?"
   Emotional driver: CURIOSITY + HOPE_POSSIBILITY
   Use when: Strong root cause reframe, false solution exposure

10. EXPOSE
    Structure: "The [Problem] Lie: Why [False Solution] Makes It Worse"
    Emotional driver: FRUSTRATION_ANGER + FEAR_PROTECTION
    Use when: Enemy-based + root cause combination, anger-driven markets
```

## Process Logic

### Step 1: Match Formulas to Candidate Architecture

```
FOR each candidate:

  LOAD from candidate_architectures:
    - mechanism.mechanism_name
    - root_cause_reframe.what_they_think (false solution)
    - root_cause_reframe.what_it_really_is (root cause)
    - enemy_identified.enemy_name
    - promise_statement.core_promise
    - emotional_driver_combination.primary_driver
    - emotional_driver_combination.secondary_driver

  SELECT formula types that match:
    PRIMARY MATCH: formula's emotional_driver matches candidate's primary_driver
    SECONDARY MATCH: formula's emotional_driver matches candidate's secondary_driver
    STRUCTURAL MATCH: formula structure fits candidate type
      (Enemy-based → WARNING, EXPOSE, THEY-DON'T-WANT)
      (Root-cause → QUESTION, WHAT-IF, SECRET)
      (Mechanism-forward → HOW-TO, DISCOVERY, NUMBER)

  ENSURE minimum 5 different formula types selected
```

### Step 2: Load Formula Evidence from Vault

```
FOR each selected formula type:
  SEARCH component_pools.headline_formulas for matching formula_type
  RECORD:
    formula_source_count = usage_count from pool
    swipe_source = first swipe_id from swipe_sources array
    average_quality_score from pool

  ONLY USE formulas where:
    formula_source_count >= 3 (proven in vault)
    average_quality_score >= 70 (quality threshold)
```

### Step 3: Generate Headlines

```
FOR each candidate (5 total):
  FOR each selected formula type (5+ types):
    GENERATE 2+ headline variations:

      APPLY formula structure to candidate's specific content:
        - Replace [Enemy] with enemy_identified.enemy_name
        - Replace [Problem] with the symptom/pain
        - Replace [Root Cause] with root_cause_reframe.what_it_really_is
        - Replace [Mechanism] with mechanism.mechanism_name
        - Replace [Benefit] with promise_statement.core_promise
        - Replace [False Solution] with root_cause_reframe.what_they_think
        - Replace [Number] with mechanism step count or timeframe
        - Replace [Authority] with relevant credibility source

      VERIFY each headline:
        - Is complete copy (no remaining brackets [])
        - Is a complete sentence or compelling fragment
        - Contains the candidate's specific mechanism or enemy name
        - Triggers the assigned emotional driver
        - Would work as an actual email subject line or ad headline

  MINIMUM output: 10 headlines per candidate
```

### Step 4: Validate Emotional Coverage

```
FOR each candidate's headline set:
  COUNT fear_based headlines (FEAR_PROTECTION, FRUSTRATION_ANGER, SHAME_DIGNITY)
  COUNT hope_based headlines (HOPE_POSSIBILITY, LIBERATION_FREEDOM, GREED_OPPORTUNITY)
  COUNT curiosity_based headlines (CURIOSITY)

  VERIFY:
    - At least 3 fear-based headlines
    - At least 3 hope-based headlines
    - At least 2 curiosity-based headlines
    - Total >= 10

  IF emotional coverage fails:
    GENERATE additional headlines targeting underrepresented driver
```

## Anti-Exemplar: What FAILS This Skill

```
FAIL (Template/Placeholder):
  "Warning: [Problem] Is Causing [Negative Outcome]"
  "Could [Your Issue] Be Caused By [Something Hidden]?"
  "How [This Method] [Helps You]"

PASS (Complete Copy):
  "Warning: Low-Fat Diets Are Shutting Down Your Bile Flow (And Making Belly Fat Worse)"
  "Could Your Stubborn Belly Fat Be Caused By Sluggish Bile Flow Instead Of Calories?"
  "How A 3-Step Bile Protocol Eliminates Belly Fat Without Dieting"
```

## Constraints

- NEVER produce headlines with bracket placeholders ([...])
- NEVER produce headlines that are descriptions ("A headline about...")
- NEVER use fewer than 5 different formula types per candidate
- NEVER generate fewer than 10 headlines per candidate
- NEVER use a formula type with fewer than 3 swipe examples in vault
- ALWAYS cite swipe_source for each formula type used
- ALWAYS match headlines to candidate's emotional drivers
- ALWAYS produce minimum 50 total headlines across all 5 candidates
- ALL headlines must contain the candidate's specific mechanism name OR enemy name OR root cause

---

## Quality Criteria

1. Minimum 10 headlines per candidate (50+ total)
2. Minimum 5 different formula types per candidate
3. Zero bracket placeholders in any headline text
4. All headlines are complete, executable copy
5. All formula types cite swipe_source and formula_source_count
6. Emotional coverage: 3+ fear, 3+ hope, 2+ curiosity per candidate
7. All headlines contain candidate-specific content (not generic)
8. Anti-exemplar check: no headline matches the FAIL pattern
