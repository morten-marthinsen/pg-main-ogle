# Skill 0.3: Input Threshold Validator

**Version:** 1.0
**Layer:** 0 (Foundation)
**Type:** Validation Gate
**Dependencies:** deep_research_files.json (from 0.1), normalized_vault.json (from 0.2)
**Output:** validation_status.json

---

## Purpose

Run 6 binary validation checks against Deep Research content and normalized vault data. Every check produces PASS or FAIL with exact counts. Any FAIL stops the pipeline.

---

## The Question

**"Does all input data meet minimum thresholds for intelligence analysis?"**

---

## Input Schema

```yaml
input:
  deep_research_files: from 0.1
  normalized_vault: from 0.2
```

---

## Output Schema

```yaml
output:
  validation_status:
    overall: enum[PASS, FAIL]
    checks:
      - check_name: string
        threshold: integer|string
        actual: integer|string
        status: enum[PASS, FAIL]
        detail: string

    summary:
      checks_passed: integer
      checks_failed: integer
      total_checks: 6
      first_failure: string|null  # Name of first failing check

    deep_research_ready: boolean
    vault_ready: boolean
    timestamp: string
```

---

## Validation Checks

### Check 1: Quote Volume

```
SOURCE: deep_research_files.loaded_content.classified_quotes.total_quote_count
THRESHOLD: >= 500
PASS: total_quote_count >= 500
FAIL: total_quote_count < 500

FAIL MESSAGE:
  "Insufficient quote volume — Found [X] quotes, need 500+.
   Expand Deep Research to reach 500+ quote threshold."
```

### Check 2: Mechanism Count

```
SOURCE: deep_research_files.loaded_content.mechanism_analysis.mechanism_count
THRESHOLD: >= 15 (target: 30+)
PASS: mechanism_count >= 15
FAIL: mechanism_count < 15

FAIL MESSAGE:
  "Insufficient mechanism analysis — Found [X] mechanisms, need 15+.
   Expand competitor mechanism research to 15+ unique mechanisms."
```

### Check 3: WEB Analysis Completeness

```
SOURCE: deep_research_files.loaded_content.desire_language
THRESHOLD: wants >= 3 AND enemies >= 3 AND beliefs >= 3
PASS: All three arrays have 3+ entries each
FAIL: Any array < 3 entries

FAIL MESSAGE:
  "Incomplete WEB analysis — [category] has only [X] entries, need 3+.
   Complete WEB analysis with 3+ entries per category."
```

### Check 4: Schwartz Stage Defined

```
SOURCE: deep_research_files.loaded_content.market_config.schwartz_stage
THRESHOLD: Integer 1-5
PASS: schwartz_stage is defined as 1, 2, 3, 4, or 5
FAIL: schwartz_stage undefined, null, or not 1-5

FAIL MESSAGE:
  "Schwartz stage not defined — Found [value], need integer 1-5.
   Define market sophistication stage in market_config.yaml."
```

### Check 5: Vault Swipe Count

```
SOURCE: normalized_vault.filter_results.final_count
THRESHOLD: >= 100
PASS: final_count >= 100
WARN: final_count >= 50 AND < 100
FAIL: final_count < 50

FAIL MESSAGE:
  "Insufficient swipe examples for [NICHE] — Found [X], need 100+.
   Options: 1) Expand to related niches, 2) Lower quality threshold to 65,
   3) Verify vault has sufficient extractions for this niche."

WARN MESSAGE:
  "Swipe count below ideal — Found [X], threshold is 100.
   Proceeding with reduced confidence. Consider expanding to related niches."
```

### Check 6: Vault Quality Average

```
SOURCE: normalized_vault.filter_results.quality_average
THRESHOLD: >= 70
PASS: quality_average >= 70
FAIL: quality_average < 70

FAIL MESSAGE:
  "Vault quality below threshold — Average [X], need 70+.
   Re-run extraction with higher quality targets or expand vault."
```

---

## Aggregation Logic

```
checks_passed = count of checks with status == PASS
checks_failed = count of checks with status == FAIL

overall = IF checks_failed == 0 THEN "PASS" ELSE "FAIL"

deep_research_ready = checks 1-4 all PASS
vault_ready = checks 5-6 all PASS

first_failure = name of first check with status == FAIL (or null if all PASS)
```

---

## Stop Conditions

```
IF overall == "FAIL":
  REPORT all failures with exact counts and required actions
  STOP: Cannot proceed to component pool construction
  DO NOT proceed to skill 0.4
```

---

## Quality Criteria

1. All 6 checks produce binary PASS/FAIL (no partial passes)
2. All thresholds are exact integers (not ranges)
3. All actual values are exact integers from real data
4. Failure messages include specific counts and actionable resolution
5. No fabricated counts — all numbers from upstream outputs
