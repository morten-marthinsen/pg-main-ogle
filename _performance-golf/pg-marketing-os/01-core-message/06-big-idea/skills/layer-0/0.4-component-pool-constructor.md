# Skill 0.4: Component Pool Constructor

**Version:** 1.0
**Layer:** 0 (Foundation)
**Type:** Data Structuring
**Dependencies:** normalized_vault.json (from 0.2), validation_status.json must be PASS (from 0.3)
**Output:** component_pools.json

---

## Purpose

Build 6 component pools from normalized vault data — structured inventories of mechanisms, headlines, proof types, emotional drivers, root cause techniques, and Big Idea types. These pools are the raw material for Layer 1 intelligence analysis.

---

## The Question

**"Are all 6 component pools built with exact counts and swipe evidence trails?"**

## Pool Construction Logic

### Pool 1: Mechanism Inventory

```
FOR each normalized swipe:
  EXTRACT components.mechanism.name
  EXTRACT components.mechanism.root_cause.reframe_technique
  EXTRACT configuration.elements.MECHANISM (integer score)

GROUP by mechanism_name (normalize: case-insensitive, strip articles)

FOR each unique mechanism:
  COUNT usage_frequency
  CALCULATE average_mechanism_score
  COLLECT swipe_id values
  COLLECT root_cause reframe_techniques paired with it
  CALCULATE average_quality_score from metadata.quality_score

SORT by usage_count DESC, then average_quality_score DESC
```

**Entry Schema:**
```yaml
MechanismEntry:
  mechanism_name: string
  swipe_sources: [string]  # swipe_id values
  usage_count: integer
  average_mechanism_score: float
  average_quality_score: float
  root_cause_techniques: [string]
```

### Pool 2: Headline Formula Inventory

```
FOR each normalized swipe:
  EXTRACT components.headline.pattern_type
  EXTRACT components.headline.main_headline
  EXTRACT metadata.quality_score

GROUP by pattern_type

FOR each type:
  COUNT usage_frequency
  COLLECT example headlines (main_headline text)
  COLLECT swipe_id values
  CALCULATE average_quality_score

SORT by usage_count DESC, then average_quality_score DESC
```

**Entry Schema:**
```yaml
HeadlineEntry:
  formula_type: string
  examples: [string]  # actual headline text
  swipe_sources: [string]
  usage_count: integer
  average_quality_score: float
```

### Pool 3: Proof Architecture Inventory

```
FOR each normalized swipe:
  EXTRACT components.proof.types_used (array)
  EXTRACT configuration.elements.DEMONSTRATION (integer score)
  EXTRACT components.proof.strongest_proof_element

CATEGORIZE by proof type values:
  testimonials, studies, credentials, demonstrations, statistics,
  case_studies, before_after, celebrity, expert, social_proof

FOR each proof type:
  COUNT usage frequency across all swipes
  CALCULATE average DEMONSTRATION score
  COLLECT swipe_id examples
  CALCULATE average_quality_score

SORT by average_demonstration_score DESC
```

**Entry Schema:**
```yaml
ProofEntry:
  proof_type: string
  swipe_sources: [string]
  usage_count: integer
  average_demonstration_score: float
  average_quality_score: float
```

### Pool 4: Emotional Driver Combinations

```
FOR each normalized swipe:
  READ all 9 values from configuration.drivers:
    FEAR_PROTECTION, FRUSTRATION_ANGER, REVENGE_VINDICATION,
    SHAME_DIGNITY, HOPE_POSSIBILITY, BELONGING_TRIBE,
    LIBERATION_FREEDOM, GREED_OPPORTUNITY, CURIOSITY

  IDENTIFY top 2 drivers (highest integer scores)
  CREATE combination: primary (highest) + secondary (second highest)
  CALCULATE combined_score = primary_score + secondary_score

GROUP by driver combination pair

FOR each combination:
  COUNT usage frequency
  CALCULATE average combined_score
  COLLECT swipe_id examples
  CALCULATE average_quality_score

SORT by average_combined_score DESC
```

**Entry Schema:**
```yaml
EmotionalEntry:
  primary_driver: string
  secondary_driver: string
  swipe_sources: [string]
  usage_count: integer
  average_combined_score: float
  average_quality_score: float
```

### Pool 5: Root Cause Reframe Techniques

```
FOR each normalized swipe:
  EXTRACT components.mechanism.root_cause.reframe_technique
  EXTRACT components.mechanism.root_cause.emphasis_level
  EXTRACT components.mechanism.root_cause.what_they_think_problem_is
  EXTRACT components.mechanism.root_cause.what_real_problem_is

SKIP swipes where root_cause is null or reframe_technique is null

GROUP by reframe_technique

FOR each technique:
  COUNT usage by emphasis_level (heavy/medium/light)
  MATCH to Schwartz stages:
    Stage 5: Heavy emphasis (health, golf)
    Stage 3: Light emphasis (finance)
    Stages 1-2: Minimal emphasis
  COLLECT example reframes (what_they_think → what_real_problem_is)
  COLLECT swipe_id values

SORT by usage_count DESC
```

**Entry Schema:**
```yaml
RootCauseEntry:
  reframe_technique: string
  emphasis_level_distribution:
    heavy: integer
    medium: integer
    light: integer
  schwartz_stage_match:
    stage_5: enum[perfect, good, poor]
    stage_3: enum[perfect, good, poor]
  swipe_sources: [string]
  usage_count: integer
  example_reframes:
    - what_they_think: string
      what_it_really_is: string
      swipe_source: string
```

### Pool 6: Big Idea Type Patterns

```
FOR each normalized swipe:
  EXTRACT metadata.big_idea_type
  EXTRACT metadata.central_concept
  EXTRACT metadata.quality_score

GROUP by big_idea_type

FOR each type:
  COUNT usage frequency
  CALCULATE average_quality_score
  COLLECT swipe_id examples
  COLLECT central_concept strings for pattern analysis

SORT by usage_count DESC, then average_quality_score DESC
```

**Entry Schema:**
```yaml
BigIdeaEntry:
  type: string
  usage_count: integer
  average_quality_score: float
  swipe_sources: [string]
  example_concepts: [string]  # central_concept values
```

---

## Quality Criteria

1. All 6 pools built from actual normalized vault data (no fabrication)
2. All counts are exact integers calculated from real iteration
3. All averages calculated from actual scores (not estimated)
4. All swipe_sources arrays contain valid swipe_id strings from processed files
5. Sorts applied correctly per pool specification
6. No null/undefined values in required fields
7. Pool statistics counts match actual array lengths
