# Skill 0.6: Foundation Gate

**Version:** 1.0
**Layer:** 0 (Foundation)
**Type:** Quality Gate
**Dependencies:** validation_status.json (from 0.3), component_pools.json (from 0.4), deep_research_summary.json (from 0.5)
**Output:** layer0-output.json

---

## Purpose

Final quality gate for Layer 0 — validates all upstream outputs meet completeness thresholds, then compiles them into a single layer0-output.json ready for Layer 1 intelligence consumption. Binary PASS/FAIL. No partial passes.

---

## The Question

**"Is the data foundation complete and ready for intelligence analysis?"**

## Gate Logic

### Check 1: Validation Status

```
IF validation_status.overall == "PASS":
  validation_passed = true
ELSE:
  validation_passed = false
  FAIL: "Input threshold validation did not pass — fix upstream (0.3)"
```

### Check 2: Pool Population Thresholds

```
FOR each pool in component_pools:
  CHECK pool array length against minimum threshold:
    mechanisms: >= 10
    headline_formulas: >= 5
    proof_architectures: >= 5
    emotional_combinations: >= 10
    root_cause_techniques: >= 3
    big_idea_types: >= 5

  IF any pool below threshold:
    pools_populated = false
    LOG: "[pool_name] has [X] entries, needs [minimum]+"
```

### Check 3: Summary Completeness

```
CHECK deep_research_summary:
  top_pains has 10 entries
  top_hopes has 10 entries
  competitor_mechanisms has 15+ entries
  schwartz_stage is integer 1-5
  market_category is non-empty string

IF any check fails:
  summary_complete = false
  LOG specific failure
```

### Check 4: Count Consistency

```
CROSS-CHECK:
  deep_research_summary.total_mechanisms_found ==
    deep_research_summary.competitor_mechanisms.length

  component_pools.pool_statistics.total_swipes_processed ==
    normalized_vault.filter_results.final_count (from validation_status context)

IF mismatches found:
  counts_consistent = false
  LOG specific mismatch
```

### Final Determination

```
gate_status = IF (validation_passed AND pools_populated AND
                  summary_complete AND counts_consistent)
              THEN "PASS"
              ELSE "FAIL"

IF gate_status == "PASS":
  COMPILE layer0-output.json with all three upstream outputs combined
  LOG: "LAYER 0 COMPLETE — [X] swipes loaded, [Y] mechanisms, [Z] quotes"

IF gate_status == "FAIL":
  POPULATE failure_report with all failures
  STOP: Cannot proceed to Layer 1
```

## Quality Criteria

1. All 4 gate checks are binary (no partial passes)
2. Pool thresholds are exact integers verified against actual array lengths
3. Summary thresholds verified against actual extracted content
4. Count consistency cross-checks catch any data loss between skills
5. Failure report identifies exact upstream skill to fix
6. Compiled output contains all three upstream files (not just references)
7. No advancement to Layer 1 unless gate_status == PASS
