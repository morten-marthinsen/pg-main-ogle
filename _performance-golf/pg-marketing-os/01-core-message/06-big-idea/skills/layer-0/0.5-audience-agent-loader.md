# 0.5 — Audience Agent Loader

**Purpose:** Load audience agent personas for Arena audience evaluation.
**When:** Layer 0 (Foundation) of any Arena-enabled skill.
**Source:** `audience-agent-personas.json` from `~outputs/[project-code]/01-research/`
**Reference:** `~system/protocols/AUDIENCE-AGENT-PROTOCOL.md`

---

## LOADING PROCEDURE

1. **LOCATE** `audience-agent-personas.json` in `~outputs/[project-code]/01-research/`
2. **VALIDATE** file structure:
   - File exists and is valid JSON
   - `persona_count` >= 3 (minimum for any tier)
   - Each persona has required fields: id, name, demographics, psychographics, emotional_state, belief_system, voice_profile, purchase_history, identity_tensions, fssit_conditions, sophistication, first_person_voice
   - Each persona has >= 10 verbatim VOC quotes in voice_profile.vocabulary_samples
3. **SELECT** personas for current tier:
   - Full tier: ALL personas (typically 5-7)
   - Standard tier: Top 3 (primary buyer, hardest to convert, most emotionally driven)
4. **STORE** in `upstream_context.audience_agents`:
   - Full persona specifications for selected agents
   - Ready for deployment during Arena audience evaluation step
5. **VERIFY** tier-appropriate count loaded

## FAILURE MODES

- **File missing:** HALT — "Audience agent personas not constructed. Run Skill 01 Layer 3 microskill 3.2-B (Audience Agent Constructor) first."
- **Insufficient personas:** HALT — "Only [N] audience agent personas found, minimum is 3."
- **Missing required fields:** HALT — "Persona [id] missing required field: [field]."
- **No VOC quotes:** HALT — "Persona [id] has fewer than 10 VOC quotes. Audience agents require real customer voice data."

## CRITICAL RULES

- Audience agent personas are READ-ONLY during Arena execution — never modified
- Personas are constructed ONCE during research and used across ALL Arena skills
- If the project changes scope significantly after research, audience agents should be reconstructed
