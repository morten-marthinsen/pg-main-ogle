# Layer 0.2.5 — Specimen Decomposer

**Version:** 1.0
**Layer:** 0 (Foundation)
**Type:** Specimen Analysis
**Domain:** Big Ideas
**Dependencies:** vault_big-ideas_intelligence (from 0.2-vault-intelligence-loader.md)
**Output:** specimen_decomposition (structural templates, technique catalog, power elements)

---

## Purpose

Decompose top-scoring big idea specimens from the vault-intelligence into reusable structural templates, technique catalogs, and power elements. Big idea specimens contain typed concepts with headlines, subheadlines, power words, and specificity elements. This skill extracts the ideation architecture that makes elite big ideas compelling at the highest conceptual level — the "one thing" that makes a campaign irresistible.

**This skill bridges the gap between:** vault data (typed big idea specimens with headline artifacts) and generation layers (what Claude needs to construct original big ideas with the same conceptual power and market sophistication).

---

input:

## Decomposition Protocol

### Phase 1: Specimen Selection & Tiering

```
IF tier_metadata present (Phase B loader active):
  LOAD gold_specimens → full decomposition
  LOAD silver_specimens → template-only decomposition
ELSE (pre-upgrade fallback):
  SELECT top 5 by position in specimens as pseudo-Gold
  SELECT next 10 as pseudo-Silver
  LOG "Operating in pre-upgrade mode — tiering estimated from array position"

VERIFY each big idea specimen has:
  - big_idea_type (required)
  - central_concept (required — skip if absent)
  - main_headline (required)
  - subheadline (preferred)
  - power_words[] (preferred)
  - specificity_elements[] (preferred)
  - source_file (required for provenance)
  - niche (required for type-matching)
  - format (preferred)
```

### Phase 2: Structural Template Extraction

For each Gold specimen:

```
ANALYZE central_concept, main_headline, and subheadline to extract:

  idea_construction:
    concept_skeleton:
      - element_type: [core_insight | market_gap | mechanism_preview |
                       exclusivity_claim | proof_backbone | emotional_hook |
                       specificity_anchor | urgency_driver]
      - function: string
      - position: integer (order in concept development)

    concept_architecture:
      big_idea_type: string (from specimen — INSIDER_ACCESS, MECHANISM_REVEAL,
                             PROPHECY, etc.)
      sophistication_level: string (implied market sophistication stage)
      differentiation_strategy: string (how this idea separates from market)
      emotional_core: string (primary emotional driver)

  headline_construction:
    headline_skeleton:
      - element_type: [power_word | specificity_element | curiosity_gap |
                       authority_signal | outcome_claim | mechanism_hint |
                       exclusivity_marker]
      - function: string
      - position: integer
    headline_architecture:
      headline_type: string (promise, curiosity, authority, story, challenge)
      word_count: integer
      power_word_density: float (power_words per total words)

  subheadline_construction:
    subheadline_skeleton:
      - element_type: string
        function: string
        position: integer
    relationship_to_headline: string (expands, qualifies, adds proof,
                                      adds urgency, adds specificity)

For Silver specimens:
  Extract concept_architecture and headline_architecture ONLY
  SKIP element-level construction analysis
```

### Phase 3: Technique Catalog Compilation

Aggregate across ALL decomposed specimens:

```
big_idea_types:
  FOR each unique big_idea_type:
    - type_name: string (INSIDER_ACCESS, MECHANISM_REVEAL, PROPHECY,
                         CONTRARIAN_TRUTH, SUPPRESSED_DISCOVERY, etc.)
    - count: integer
    - construction_pattern: string (typical concept development for this type)
    - best_for_niche: [string]
    - gold_example_concept: string (1-2 sentence concept excerpt)
    - market_sophistication: string (which stages this type works for)

headline_patterns:
  FOR each unique headline construction approach:
    - pattern_name: string
    - element_sequence: [string] (ordered elements in headline)
    - word_count_range: string
    - power_word_usage: string
    - specimens_using: [source_file references]

power_word_techniques:
  FOR each category of power words observed:
    - category: string (authority, urgency, exclusivity, specificity, emotion)
    - words: [string] (all power words in this category across specimens)
    - usage_frequency: integer
    - position_pattern: string (where in headlines these tend to appear)

specificity_techniques:
  FOR each method of grounding big ideas in specifics:
    - technique_name: string
    - implementation_pattern: string
    - gold_example: string
    - specificity_type: string (numerical, temporal, institutional, personal)

differentiation_strategies:
  FOR each approach to separating the big idea from market noise:
    - strategy_name: string
    - implementation_pattern: string
    - big_idea_types_using: [string]
    - specimens_using: [source_file references]
```

### Phase 4: Power Element Extraction

From Gold specimens ONLY:

```
headlines:
  FOR each Gold specimen:
    - verbatim_headline: string (the main_headline)
    - headline_type: string
    - word_count: integer
    - power_words_used: [string]
    - specificity_elements_used: [string]
    - niche: string
    - source_file: string

subheadlines:
  FOR each Gold specimen with subheadline:
    - verbatim_subheadline: string
    - relationship_to_headline: string
    - source_file: string

central_concepts:
  FOR each Gold specimen:
    - verbatim_concept: string (full central_concept text)
    - big_idea_type: string
    - differentiation_strategy: string
    - source_file: string

power_word_clusters:
  FOR each Gold specimen:
    - words: [string] (from power_words[])
    - headline_context: string (the headline they appear in)
    - source_file: string
```

## Quality Gates

- [ ] Minimum 3 Gold specimens fully decomposed (concept + headline + architecture)
- [ ] Minimum 5 Silver specimens template-decomposed
- [ ] All structural templates have complete concept and headline skeletons
- [ ] Technique catalog has minimum 3 big idea types cataloged
- [ ] Technique catalog has minimum 2 headline patterns documented
- [ ] Power elements contain minimum 3 verbatim headlines from Gold specimens
- [ ] Power elements contain minimum 3 central concepts from Gold specimens
- [ ] Every power element includes source provenance (source_file + niche)
- [ ] No raw central_concept passed through undecomposed

---

## Anti-Patterns

| Anti-Pattern | Signal | Fix |
|---|---|---|
| **Concept Without Construction** | Big idea concepts listed as flat text | Decompose into concept_skeleton with elements and functions |
| **Headline Without Architecture** | Headlines listed without structural analysis | Every headline needs skeleton + architecture (type, power_word_density) |
| **Type Without Pattern** | Big idea types named without construction guidance | Every type needs construction_pattern + market_sophistication |
| **Power Words Without Context** | Words listed without headline context | Include headline_context showing where words were used |
| **Differentiation Missing** | Ideas decomposed without differentiation analysis | differentiation_strategy required — this is the competitive edge |
