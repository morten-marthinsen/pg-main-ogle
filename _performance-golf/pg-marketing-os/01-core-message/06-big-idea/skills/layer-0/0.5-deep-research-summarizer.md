# Skill 0.5: Deep Research Summarizer

**Version:** 1.0
**Layer:** 0 (Foundation)
**Type:** Extraction & Summarization
**Dependencies:** deep_research_files.json (from 0.1)
**Output:** deep_research_summary.json

---

## Purpose

Extract the key insights from all 5 Deep Research files into a structured summary optimized for Layer 1 intelligence analysis. This summary is what Layer 1 actually consumes — not the raw files.

---

## The Question

**"Are the most important insights from Deep Research extracted and structured for intelligence processing?"**

---

## Input Schema

```yaml
input:
  deep_research_files: from 0.1 (loaded_content section)
```

## Extraction Logic

### From classified_quotes.json

```
TOP PAINS (10):
  SORT pain_quotes by intensity/frequency indicators
  SELECT top 10
  FOR each: preserve original quote text + source

TOP HOPES (10):
  SORT hope_quotes by intensity/frequency indicators
  SELECT top 10
  FOR each: preserve original quote text + source

ROOT CAUSE INSIGHTS (5):
  FROM root_cause_quotes:
  IDENTIFY the 5 most common/powerful root cause realizations
  Extract the insight pattern (what they discovered was really wrong)

FAILED SOLUTIONS (5):
  FROM solutions_tried_quotes:
  IDENTIFY the 5 most commonly mentioned failed approaches
  Extract what they tried and why it didn't work
```

### From mechanism_analysis.json

```
COMPETITOR MECHANISMS:
  EXTRACT all unique competitor mechanism names
  FOR each: include competitor count (how many use it)
  SORT by competitor_count DESC

MECHANISM SATURATION:
  GROUP mechanisms by saturation level:
    High: 10+ competitors
    Medium: 5-9 competitors
    Low: 1-4 competitors
```

### From desire_language.json

```
TOP WANTS (5):
  RANK wants by frequency/intensity
  SELECT top 5
  Preserve exact language used by audience

TOP ENEMIES (5):
  RANK enemies by frequency/intensity
  SELECT top 5
  Preserve exact language used by audience

CORE BELIEFS (5):
  RANK beliefs by prevalence
  SELECT top 5
  Preserve exact belief statements
```

### From opportunity_synthesis.json

```
STRATEGIC GAPS:
  EXTRACT all identified positioning opportunities
  FOR each: include gap description + evidence basis

POSITIONING RECOMMENDATIONS:
  EXTRACT all recommendations
  FOR each: include rationale + competitive context
```

### From market_config.yaml

```
EXTRACT directly:
  schwartz_stage (integer 1-5)
  market_category (string)
  competitive_density (string)

DERIVE root_cause_emphasis:
  IF schwartz_stage == 5: root_cause_emphasis = "heavy"
  IF schwartz_stage == 4: root_cause_emphasis = "heavy"
  IF schwartz_stage == 3: root_cause_emphasis = "light"
  IF schwartz_stage <= 2: root_cause_emphasis = "minimal"
```

---

## Quality Criteria

1. Exactly 10 pain quotes and 10 hope quotes extracted (not fewer)
2. Exactly 5 entries per category (root cause insights, failed solutions, wants, enemies, beliefs)
3. All competitor mechanisms captured (none dropped)
4. Schwartz stage correctly mapped to root_cause_emphasis
5. No interpretation or analysis — pure extraction and ranking
6. Original language preserved in all quote/want/enemy/belief entries
7. extraction_metadata counts match actual data
