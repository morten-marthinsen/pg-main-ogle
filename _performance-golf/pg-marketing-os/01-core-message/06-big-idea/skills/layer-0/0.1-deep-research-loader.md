# Skill 0.1: Deep Research Loader

**Version:** 1.0
**Layer:** 0 (Foundation)
**Type:** Data Loading
**Dependencies:** Project folder path, Deep Research output files
**Output:** deep_research_files.json

---

## Purpose

Verify existence and load all 5 required Deep Research output files into a single structured container for downstream processing.

---

## The Question

**"Do all required Deep Research files exist and are they loadable?"**

---

## Input Schema

```yaml
input:
  project_folder: string  # Path to project workspace containing Deep Research outputs
  required_files:
    - classified_quotes.json
    - mechanism_analysis.json
    - desire_language.json
    - opportunity_synthesis.json
    - market_config.yaml
```

---

```yaml
output:
```

---

## Loading Logic

### Step 1: Verify File Existence

```
FOR each required_file in [classified_quotes, mechanism_analysis, desire_language, opportunity_synthesis, market_config]:

  CHECK if file exists at project_folder/[filename]

  IF exists:
    existence_check.[file] = true
  ELSE:
    existence_check.[file] = false
    files_missing.push(filename)

existence_check.all_present = (files_missing.length == 0)
```

### Step 2: Load Each File

```
IF existence_check.all_present == false:
  STOP with error report (see Error Handling below)

FOR each existing file:
  ATTEMPT to parse (JSON or YAML as appropriate)

  IF parse succeeds:
    loaded_content.[file] = parsed content
    files_loaded += 1
  ELSE:
    load_errors.push({
      file: filename,
      error: parse_error_message
    })
```

### Step 3: Extract Quick Counts

```
FROM loaded classified_quotes:
  total_quote_count = sum of all category array lengths
    (pain_quotes.length + hope_quotes.length + root_cause_quotes.length + solutions_tried_quotes.length)

FROM loaded mechanism_analysis:
  mechanism_count = competitor_mechanisms.length

FROM loaded desire_language:
  VERIFY wants.length >= 3
  VERIFY enemies.length >= 3
  VERIFY beliefs.length >= 3

FROM loaded market_config:
  VERIFY schwartz_stage is integer 1-5
```

---

## Error Handling

### Missing File Error
```
ERROR: Missing Deep Research input - [filename]
LOCATION: Expected at [project_folder/filename]
REQUIRED ACTION: Complete Deep Research before running Generative Engine
STOP: Cannot proceed — Layer 0 requires all 5 files
```

### Parse Error
```
ERROR: Cannot parse [filename]
DETAILS: [parse_error_message]
REQUIRED ACTION: Verify file is valid JSON/YAML
STOP: Cannot proceed with corrupted input files
```

---

## Quality Criteria

1. All 5 files verified for existence before any loading attempt
2. All files successfully parsed (JSON/YAML as appropriate)
3. Quick counts extracted without error
4. No fabricated counts — all integers come from actual array lengths
5. Clear error messages with exact file paths on failure
