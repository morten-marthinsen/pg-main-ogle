# 1.2 Specificity Enhancement

## Purpose
Transform vague blue sky promises into specific, concrete promises with measurable elements. Specificity sells — vague generalities don't.

**Key Principle (from E5):** "Lose a lot of weight fast" is vague and falls flat. "Burn 8 pounds of belly fat in the next 72 hours" is specific and creates a concrete picture in the prospect's mind.

---

## Input Requirements

From 1.1 Blue Sky Ideation:
- Blue sky promise candidates
- Transformation dimensions

From 02-proof-inventory:
- Data proof elements (numbers, timeframes)
- Testimonial specifics (results achieved)

From Product Context:
- Typical results customers achieve
- Reasonable timeframes

---

## The Specificity Formula

Every specific promise contains up to 3 elements:

```
SPECIFICITY = QUANTITY + QUALIFIER + TIMEFRAME

Examples:
- "12 pounds" + "of belly fat" + "in 30 days"
- "8 hours" + "of uninterrupted sleep" + "within a week"
- "$500" + "in passive income" + "in your first month"
```

## Execution Protocol

### Step 1: Identify Specificity Opportunities

```python
def identify_specificity_opportunities(blue_sky_promise):
    """
    Find where specifics can be added to a vague promise.
    """
    opportunities = {
        'quantity': None,
        'qualifier': None,
        'timeframe': None
    }

    # Analyze promise for vague terms
    vague_quantities = ['a lot', 'significant', 'major', 'weight', 'results']
    vague_timeframes = ['fast', 'quickly', 'soon', 'in no time']
    vague_qualifiers = ['fat', 'pain', 'energy', 'health', 'results']

    promise_lower = blue_sky_promise.lower()

    for vague in vague_quantities:
        if vague in promise_lower:
            opportunities['quantity'] = {
                'vague_term': vague,
                'needs_replacement': True
            }

    for vague in vague_timeframes:
        if vague in promise_lower:
            opportunities['timeframe'] = {
                'vague_term': vague,
                'needs_replacement': True
            }

    for vague in vague_qualifiers:
        if vague in promise_lower:
            opportunities['qualifier'] = {
                'vague_term': vague,
                'needs_replacement': True
            }

    return opportunities
```

### Step 2: Source Specific Numbers

Pull specifics from proof inventory:

```python
def source_specifics(proof_inventory, niche):
    """
    Extract specific numbers from available proof.
    """
    specifics = {
        'quantities': [],
        'timeframes': [],
        'qualifiers': []
    }

    # From data proof
    for data in proof_inventory.get('data_elements', []):
        if data.get('sub_type') in ['data_user_results', 'data_percentages']:
            specifics['quantities'].append({
                'number': data['extracted_number'],
                'context': data['context'],
                'proof_backed': True,
                'source': data['source']
            })

    # From testimonials
    for testimonial in proof_inventory.get('social_elements', []):
        results = extract_numbers_from_text(testimonial['content'])
        for result in results:
            specifics['quantities'].append({
                'number': result,
                'context': 'testimonial',
                'proof_backed': True,
                'source': testimonial.get('attribution')
            })

    # From typical results
    specifics['timeframes'] = get_typical_timeframes(niche)
    specifics['qualifiers'] = get_niche_qualifiers(niche)

    return specifics
```

### Step 4: Generate Specific Versions

```python
def make_specific(blue_sky_promise, specifics, patterns):
    """
    Transform vague promise into multiple specific versions.
    """
    specific_versions = []

    # Extract core outcome from blue sky
    core_outcome = extract_core_outcome(blue_sky_promise)

    # Generate quantity-enhanced version
    if specifics['quantities']:
        best_quantity = select_best_quantity(specifics['quantities'])
        specific_versions.append({
            'original': blue_sky_promise,
            'specific': f"{best_quantity['number']} {core_outcome}",
            'specificity_type': 'quantity',
            'proof_backed': best_quantity['proof_backed']
        })

    # Generate timeframe-enhanced version
    if specifics['timeframes']:
        for timeframe in specifics['timeframes'][:3]:
            specific_versions.append({
                'original': blue_sky_promise,
                'specific': f"{core_outcome} in {timeframe}",
                'specificity_type': 'timeframe',
                'proof_backed': timeframe.get('proof_backed', False)
            })

    # Generate fully specific version (all 3 elements)
    if all([specifics['quantities'], specifics['timeframes'], specifics['qualifiers']]):
        specific_versions.append({
            'original': blue_sky_promise,
            'specific': f"{specifics['quantities'][0]['number']} {specifics['qualifiers'][0]} {core_outcome} in {specifics['timeframes'][0]}",
            'specificity_type': 'full',
            'proof_backed': True
        })

    return specific_versions
```

## Specificity Scoring Rubric

```yaml
specificity_score:
  10: "Exact number + specific qualifier + exact timeframe"
      example: "Burn 8 pounds of stubborn belly fat in the next 72 hours"

  9: "Two of three elements highly specific"
     example: "Lose 12 pounds in 30 days"

  8: "All three elements present, one is a range"
     example: "Lose 5-10 pounds of belly fat in about a month"

  7: "Two specific elements"
     example: "Lose 12 pounds of belly fat"

  6: "One highly specific element + general others"
     example: "Lose weight in exactly 30 days"

  5: "One specific element"
     example: "Lose 12 pounds"

  4: "Ranges only, no exact numbers"
     example: "Lose significant weight in a few weeks"

  3: "General language with implied specificity"
     example: "Lose a lot of weight fast"

  2: "Vague but directional"
     example: "Improve your weight"

  1: "Completely vague"
     example: "Get results"
```
