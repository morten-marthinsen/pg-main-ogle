# Layer 0.2.5 — Specimen Decomposer

**Version:** 1.0
**Layer:** 0 (Foundation)
**Type:** Specimen Analysis
**Domain:** Promise
**Dependencies:** vault_promise_intelligence (from 0.2-vault-intelligence-loader.md)
**Output:** specimen_decomposition (structural templates, technique catalog, power elements)

---

## Purpose

Decompose top-scoring promise specimens from the vault-intelligence into reusable structural templates, technique catalogs, and power elements. Promise specimens contain structured data — statements, campaign theses, promise types, emotional frames, and specificity levels. This skill extracts the construction patterns that make elite promises irresistible and believable.

**This skill bridges the gap between:** vault data (typed promise specimens with emotional framing) and generation layers (what Claude needs to construct original promises with the same specificity and emotional force).

## Decomposition Protocol

### Phase 1: Specimen Selection & Tiering

```
IF tier_metadata present (Phase B loader active):
  LOAD gold_specimens → full decomposition
  LOAD silver_specimens → template-only decomposition
ELSE (pre-upgrade fallback):
  SELECT top 5 by position in primary_promises as pseudo-Gold
  SELECT next 10 as pseudo-Silver
  LOG "Operating in pre-upgrade mode — tiering estimated from array position"

VERIFY each promise specimen has:
  - statement (required — skip if absent)
  - campaign_thesis (required)
  - type (required)
  - emotional_frame (required)
  - specificity_level (preferred)
  - source_file (required for provenance)
  - niche (required for type-matching)
```

### Phase 2: Structural Template Extraction

For each Gold specimen:

```
ANALYZE statement and campaign_thesis to extract:

  promise_construction:
    statement_skeleton:
      - element_type: [outcome_claim | timeframe | specificity_anchor |
                       mechanism_hint | emotional_trigger | comparison_frame |
                       exclusivity_marker | proof_integration]
      - function: string
      - position: integer
    statement_architecture:
      promise_type: string (from specimen — transformation, speed, outcome,
                            certainty, protection_plus_profit, etc.)
      emotional_frame: string (from specimen — hope, belonging, vindication,
                               security, relief, etc.)
      specificity_level: string (from specimen)
      word_count: integer

  thesis_construction:
    thesis_skeleton:
      - element_type: [problem_context | mechanism_preview | promise_expansion |
                       proof_preview | differentiation | urgency_context]
      - function: string
      - position: integer
    thesis_architecture:
      narrative_strategy: string (how the thesis tells the campaign's story)
      mechanism_integration: string (how the mechanism is previewed)
      word_count: integer

For Silver specimens:
  Extract statement_architecture and thesis_architecture ONLY
  SKIP element-level construction analysis
```

### Phase 3: Technique Catalog Compilation

Aggregate across ALL decomposed specimens:

```
promise_types:
  FOR each unique type:
    - type_name: string (transformation, speed, outcome, certainty,
                         dual_protection_and_profit, protection_plus_profit, etc.)
    - count: integer
    - construction_pattern: string (typical element sequence for this type)
    - best_for_niche: [string]
    - gold_example: string (1-2 sentence excerpt)

emotional_frames:
  FOR each unique emotional_frame:
    - frame_name: string (hope, belonging, vindication, security, relief, etc.)
    - count: integer
    - implementation_pattern: string (how to activate this frame in a promise)
    - types_paired_with: [string] (which promise types use this frame)
    - gold_example: string

specificity_techniques:
  FOR each method of making promises specific:
    - technique_name: string (exact_numbers, timeframes, named_outcomes,
                              before_after_contrast, comparison_anchors)
    - count: integer
    - implementation_pattern: string
    - gold_example: string
    - specificity_level_produced: string

thesis_strategies:
  FOR each unique campaign_thesis construction approach:
    - strategy_name: string
    - elements_included: [string]
    - word_count_range: string
    - specimens_using: [source_file references]
```

### Phase 4: Power Element Extraction

From Gold specimens ONLY:

```
promise_statements:
  FOR each Gold specimen:
    - verbatim_statement: string (the full promise statement)
    - promise_type: string
    - emotional_frame: string
    - specificity_level: string
    - niche: string
    - source_file: string

campaign_theses:
  FOR each Gold specimen:
    - verbatim_thesis: string (the full campaign thesis)
    - narrative_strategy: string
    - mechanism_preview: string
    - source_file: string

specificity_anchors:
  FOR Gold specimens with hyper_specific specificity:
    - verbatim_anchor: string (the specific number, timeframe, or outcome claim)
    - anchor_type: string (number, timeframe, outcome, comparison)
    - context: string (what promise it supports)
    - source_file: string

emotional_triggers:
  FOR Gold specimens:
    - verbatim_phrase: string (specific language activating the emotional frame)
    - emotional_frame: string
    - position_in_promise: string
    - source_file: string
```

## Quality Gates

- [ ] Minimum 3 Gold specimens fully decomposed (statement + thesis + architecture)
- [ ] Minimum 5 Silver specimens template-decomposed
- [ ] All structural templates have complete statement skeletons
- [ ] Technique catalog has minimum 3 promise types cataloged
- [ ] Technique catalog has minimum 2 emotional frames documented
- [ ] Power elements contain minimum 3 verbatim promise statements from Gold specimens
- [ ] Specificity anchors extracted from hyper-specific Gold specimens
- [ ] Every power element includes source provenance (source_file + niche)
- [ ] No raw statement text passed through undecomposed

---

## Anti-Patterns

| Anti-Pattern | Signal | Fix |
|---|---|---|
| **Statement Without Construction** | Promise statements listed as flat text | Decompose into element_type sequence with functions |
| **Type Without Pattern** | Promise types named without construction guidance | Every type needs construction_pattern |
| **Vague Specificity** | Promises labeled "specific" without identifying actual anchors | Extract verbatim specificity_anchors |
| **Thesis Ignored** | Only statement decomposed, campaign_thesis skipped | Thesis is half the promise architecture — always decompose |
| **Frame Without Trigger** | Emotional frames named without language examples | Extract emotional_triggers with verbatim phrases |
