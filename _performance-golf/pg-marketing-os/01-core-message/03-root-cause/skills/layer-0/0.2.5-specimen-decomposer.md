# Layer 0.2.5 — Specimen Decomposer

**Version:** 1.0
**Layer:** 0 (Foundation)
**Type:** Specimen Analysis
**Domain:** Root Cause
**Dependencies:** vault_root-cause_intelligence (from 0.2-vault-intelligence-loader.md)
**Output:** specimen_decomposition (structural templates, technique catalog, power elements)

---

## Purpose

Decompose top-scoring root-cause specimens from the vault-intelligence into reusable structural templates, technique catalogs, and power elements. Root-cause specimens are three-part reframes (what_they_think → real_problem → why_nothing_worked) with annotated reframe techniques and emphasis levels. This skill extracts the reframe architecture that makes elite root-cause formulations persuasive.

**This skill bridges the gap between:** vault data (structured reframes with technique annotations) and generation layers (what Claude needs to construct original root-cause reframes with the same depth and reader-freeing power).

## Decomposition Protocol

### Phase 1: Specimen Selection & Tiering

```
IF tier_metadata present (Phase B loader active):
  LOAD gold_specimens → full decomposition
  LOAD silver_specimens → template-only decomposition
ELSE (pre-upgrade fallback):
  SELECT top 5 by position in complete_reframes as pseudo-Gold
  SELECT next 10 as pseudo-Silver
  LOG "Operating in pre-upgrade mode — tiering estimated from array position"

VERIFY each reframe specimen has:
  - what_they_think (required)
  - real_problem (required)
  - why_nothing_worked (required)
  - reframe_technique (required)
  - emphasis_level (preferred)
  - source_file (required for provenance)
  - niche (required for type-matching)
```

### Phase 2: Structural Template Extraction

For each Gold specimen:

```
ANALYZE the three-part reframe structure:

  reframe_skeleton:
    what_they_think:
      - element_type: [conventional_belief | common_behavior |
                       industry_assumption | emotional_default]
      - specificity_level: string (vague | targeted | mirror_exact)
      - word_count: integer
      - identification_technique: string (how it makes reader say "that's me")

    real_problem:
      - element_type: [hidden_cause | suppressed_truth | systemic_failure |
                       misattributed_cause | missing_ingredient]
      - revelation_method: string (how the real problem is revealed —
                                   evidence_chain, authority_citation,
                                   logical_reframe, conspiratorial_reveal)
      - word_count: integer
      - emotional_impact: string (what the reader feels upon learning this)

    why_nothing_worked:
      - element_type: [structural_impossibility | missing_component |
                       wrong_approach | rigged_system | incomplete_solution]
      - absolution_method: string (how the reader is freed from self-blame)
      - word_count: integer
      - bridge_to_solution: string (how this sets up the mechanism/product)

  reframe_architecture:
    reframe_technique: string (from specimen)
    emphasis_level: string (from specimen)
    emotional_arc: string ("from → to" — e.g., "frustration → vindication")
    ratio: string (approximate word count ratio across three parts)

For Silver specimens:
  Extract reframe_skeleton part names and reframe_architecture ONLY
  SKIP element-level analysis within each part
```

### Phase 3: Technique Catalog Compilation

Aggregate across ALL decomposed specimens:

```
reframe_techniques:
  FOR each unique reframe_technique:
    - technique_name: string (suppressed_access, not_your_fault, hidden_truth,
                              enemy_reveal, false_solution_expose, etc.)
    - count: integer
    - implementation_pattern: string (how to construct a reframe using this technique)
    - gold_example: string (1-2 sentence excerpt from Gold specimen)
    - best_for_niche: [string]
    - emotional_arc: string (typical emotional journey for this technique)

emphasis_levels:
  FOR each unique emphasis_level:
    - level_name: string (foundation, moderate, heavy)
    - count: integer
    - characteristics: string (what distinguishes this level)
    - techniques_associated: [string]
    - word_count_range: string

identification_techniques:
  FOR each method of making "what_they_think" resonate:
    - technique_name: string
    - implementation_pattern: string
    - gold_example: string
    - specimens_using: [source_file references]

absolution_techniques:
  FOR each method of freeing the reader from self-blame in "why_nothing_worked":
    - technique_name: string
    - implementation_pattern: string
    - gold_example: string
    - emotional_payoff: string (what the reader feels after absolution)
```

### Phase 4: Power Element Extraction

From Gold specimens ONLY:

```
what_they_think_verbatims:
  FOR each Gold specimen:
    - verbatim_text: string (the complete what_they_think section)
    - identification_technique: string
    - specificity_level: string
    - niche: string
    - source_file: string

real_problem_verbatims:
  FOR each Gold specimen:
    - verbatim_text: string (the complete real_problem section)
    - revelation_method: string
    - reframe_technique: string
    - source_file: string

why_nothing_worked_verbatims:
  FOR each Gold specimen:
    - verbatim_text: string (the complete why_nothing_worked section)
    - absolution_method: string
    - bridge_to_solution: string
    - source_file: string

reframe_pivot_sentences:
  FOR each Gold specimen:
    - verbatim_pivot: string (the sentence(s) where "what they think" becomes "real problem")
    - pivot_technique: string
    - source_file: string
```

## Quality Gates

- [ ] Minimum 3 Gold specimens fully decomposed (all three reframe parts + architecture)
- [ ] Minimum 5 Silver specimens template-decomposed
- [ ] All reframe skeletons have all three parts populated
- [ ] Technique catalog has minimum 3 reframe techniques cataloged
- [ ] Technique catalog has minimum 2 identification techniques documented
- [ ] Power elements contain minimum 3 verbatim reframe sets from Gold specimens
- [ ] Reframe pivot sentences extracted from all Gold specimens
- [ ] Every power element includes source provenance (source_file + niche)
- [ ] No raw reframe text passed through undecomposed

---

## Anti-Patterns

| Anti-Pattern | Signal | Fix |
|---|---|---|
| **Flat Three-Part Dump** | Reframe parts listed as text without structural analysis | Decompose each part into elements with types and functions |
| **Technique Without Pattern** | Reframe techniques named without implementation guidance | Every technique needs implementation_pattern |
| **Missing Pivot** | Reframe extracted without identifying the pivot moment | Extract reframe_pivot_sentences from every Gold specimen |
| **Absolution Absent** | "Why nothing worked" analyzed without absolution method | absolution_method required — this is the reader-freeing power |
| **Identification Weak** | "What they think" listed without specificity_level | Rate identification as vague/targeted/mirror_exact |
