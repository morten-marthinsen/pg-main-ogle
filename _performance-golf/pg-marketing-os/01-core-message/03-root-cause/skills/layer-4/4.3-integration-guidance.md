# 4.3 Integration Guidance

## Purpose
Provide guidance on HOW to integrate the root cause into different copy formats and at different points in the sales argument. Root cause can appear in lead, body, proof section, or close—this microskill defines best practices for each.

---

## The Integration Principle

**Root Cause placement affects persuasion flow.**

Where you place the root cause changes its function:
- In the **LEAD**: Creates immediate "I'm in the right place" recognition
- In the **BODY**: Explains why they've struggled (builds need)
- In **PROOF**: Validates the root cause claim
- In the **CLOSE**: Reminds them WHY they need to act

---

## Integration by Copy Format

### VSL Integration

```
VSL ROOT CAUSE PLACEMENT:

TYPICAL POSITION: After hook, before mechanism reveal
APPROXIMATE TIMING: Minutes 3-8 of a 30-minute VSL

STRUCTURE:
├── [HOOK] - Pattern interrupt, curiosity
├── [CREDIBILITY] - Why listen to me
├── [ROOT CAUSE] ← Full revelation here
│   ├── What they think
│   ├── What's really happening
│   └── Why nothing worked
├── [MECHANISM] - The solution to root cause
├── [PROOF] - Evidence mechanism works
├── [OFFER] - The product
└── [CLOSE] - Root cause callback

ROOT CAUSE FUNCTIONS IN VSL:
1. Creates need (they have this root cause)
2. Explains failure (why past solutions failed)
3. Sets up mechanism (makes mechanism feel inevitable)
```

---

### Sales Letter Integration

```
SALES LETTER ROOT CAUSE PLACEMENT:

TYPICAL POSITION: Lead or first body section
APPROXIMATE LOCATION: First 1-2 pages

STRUCTURE:
├── [HEADLINE] - May hint at root cause
├── [LEAD] ← Can reveal root cause early
│   └── OR tease root cause for body reveal
├── [BODY - PROBLEM SECTION] ← If not in lead
│   └── Full root cause revelation
├── [BODY - MECHANISM]
├── [BODY - PROOF]
├── [OFFER]
└── [CLOSE] - Root cause callback

ROOT CAUSE FUNCTIONS IN SALES LETTER:
1. Qualifies reader (this is for you if...)
2. Builds problem urgency
3. Transitions to solution
```

---

### Email Sequence Integration

```
EMAIL SEQUENCE ROOT CAUSE PLACEMENT:

OPTION A: Root cause reveal email
├── Email 1: Tease problem
├── Email 2: Root cause reveal ← Full revelation
├── Email 3: Mechanism introduction
├── Email 4: Proof
└── Email 5: Offer

OPTION B: Root cause throughout
├── Email 1: Root cause hint in hook
├── Email 2: Different angle on root cause
├── Email 3: Root cause → mechanism connection
├── Email 4: Proof of root cause
└── Email 5: Root cause reminder + offer

ROOT CAUSE FUNCTIONS IN EMAIL:
1. Creates open loops (curiosity about root cause)
2. Educates incrementally
3. Builds toward mechanism + offer
```

---

### Landing Page Integration

```
LANDING PAGE ROOT CAUSE PLACEMENT:

SHORT-FORM LANDING PAGE:
├── [HEADLINE] - Root cause hint
├── [SUBHEAD] - Root cause expansion
├── [BULLET POINTS] - Root cause bullets
├── [CTA]
└── (Root cause is compressed throughout)

LONG-FORM LANDING PAGE:
├── [HERO SECTION] - Headline + root cause tease
├── [PROBLEM SECTION] - Full root cause reveal
├── [SOLUTION SECTION] - Root cause → mechanism
├── [PROOF SECTION] - Root cause validation
├── [OFFER]
└── [CLOSE] - Root cause reminder

ROOT CAUSE FUNCTIONS ON LANDING PAGE:
1. Stops scroll (headline/hero)
2. Qualifies visitor
3. Creates "this is for me" feeling
```

## Integration by Root Cause Function

### Root Cause as PATTERN INTERRUPT

**When**: Lead/headline position
**Function**: Stop the scroll, create curiosity
**Format**: Compressed or headline version

```
PATTERN INTERRUPT INTEGRATION:
"[Surprising/contrarian root cause statement]"

Example:
"There's Only ONE Reason Your Golf Game Is So Inconsistent—
And It's Not Your Swing"
```

---

### Root Cause as PROBLEM AMPLIFICATION

**When**: Body, before mechanism
**Function**: Build urgency, explain suffering
**Format**: Full revelation block

```
PROBLEM AMPLIFICATION INTEGRATION:
[Setup - Acknowledge what they've tried]
[Root cause reveal - Why it hasn't worked]
[Implications - How bad this really is]
[Transition - But there's a solution]
```

---

### Root Cause as MECHANISM SETUP

**When**: Immediately before mechanism reveal
**Function**: Make mechanism feel like the obvious solution
**Format**: Compressed, focused on "what needs to be fixed"

```
MECHANISM SETUP INTEGRATION:
"So the real problem is [root cause].
Which means the solution has to [what mechanism must do].
That's exactly what [mechanism name] does..."
```

---

### Root Cause as PROOF ELEMENT

**When**: Proof section
**Function**: Validate that the root cause is real
**Format**: Expert citation, data, demonstration

```
PROOF ELEMENT INTEGRATION:
"[Expert] discovered that [root cause evidence]..."
"Studies show that [root cause data]..."
"Watch what happens when [root cause demonstration]..."
```

---

### Root Cause as CLOSE CALLBACK

**When**: Final close section
**Function**: Remind them why they need to act
**Format**: Ultra-compressed, emotional

```
CLOSE CALLBACK INTEGRATION:
"Remember—[root cause] is the reason nothing has worked.
[Mechanism] is the only thing that addresses it.
Don't spend another [time period] fighting [root cause symptom]."
```

## Integration Timing Chart

| Copy Section | Root Cause Version | Word Count | Function |
|-------------|-------------------|------------|----------|
| Headline | Ultra-compressed | 10-15 | Pattern interrupt |
| Subhead | Compressed | 15-25 | Expansion |
| Lead (early) | Compressed/Full | 75-200 | Qualification |
| Body (problem) | Full revelation | 200-300 | Problem amplification |
| Pre-mechanism | Compressed | 50-100 | Mechanism setup |
| Proof | As needed | Variable | Validation |
| Close | Ultra-compressed | 25-50 | Callback |
