# 3.2 Mechanism Alignment Score

## Purpose
Score how well the expressed root cause ALIGNS with the mechanism that will solve it. Perfect alignment = the mechanism directly addresses the exact root cause. Poor alignment = root cause doesn't lead naturally to the mechanism.

---

## The Alignment Principle

**Root Cause → Mechanism should feel INEVITABLE.**

When audience hears the root cause, they should think "so what would fix that?" and the mechanism should be the obvious answer.

---

## Execution Protocol

### Step 1: State the Pair

```
ROOT CAUSE → MECHANISM PAIR:
Root Cause: "[Expressed root cause]"
Mechanism: "[Product/method/technology name]"
Mechanism Description: "[What it does]"
```

---

### Step 2: Test Direct Address

Does the mechanism DIRECTLY solve the root cause?

```
DIRECT ADDRESS TEST:
Root Cause Element → Mechanism Solution

[Root cause component 1]: "[What's wrong]"
└── Mechanism addresses this by: "[How mechanism fixes it]"
    └── Direct? YES / PARTIAL / NO

[Root cause component 2]: "[What's wrong]"
└── Mechanism addresses this by: "[How mechanism fixes it]"
    └── Direct? YES / PARTIAL / NO

[Root cause component 3]: "[What's wrong]"
└── Mechanism addresses this by: "[How mechanism fixes it]"
    └── Direct? YES / PARTIAL / NO
```

---

### Step 3: Test Logical Flow

Is the mechanism the LOGICAL solution to the root cause?

```
LOGICAL FLOW TEST:
1. If [root cause] is the problem...
2. Then [mechanism principle] is the logical solution...
3. Because [reasoning]...

FLOW STRENGTH:
□ Obvious/Inevitable: Audience would think of this solution themselves
□ Clear: Audience sees the connection immediately when stated
□ Requires Explanation: Connection needs to be explained
□ Forced: Connection feels like a stretch
```

**Examples**:

| Root Cause | Mechanism | Flow |
|-----------|-----------|------|
| "Technique breaks down under pressure" | "Technology that does it automatically" | Obvious |
| "Low point misconception" | "Training that teaches low point" | Clear |
| "Hormones override diet" | "Hormone-regulating supplement" | Clear |
| "Bad posture" | "Weight loss supplement" | Forced |

---

### Step 4: Check for Gaps

Identify any gaps between root cause and mechanism:

```
GAP ANALYSIS:
Does root cause promise something mechanism doesn't deliver?
└── [List any gaps]

Does mechanism do things not explained by root cause?
└── [List any unexplained capabilities]

Are there aspects of the problem the mechanism doesn't address?
└── [List unaddressed elements]

GAP SEVERITY: [None | Minor | Significant | Critical]
```

---

### Step 5: Test Uniqueness Fit

Does this mechanism UNIQUELY solve this root cause (vs. any generic solution)?

```
UNIQUENESS TEST:
Root Cause: "[Statement]"

1. Could a generic solution solve this root cause?
   └── [Yes, equally well | Partially | No, mechanism is unique]

2. What makes THIS mechanism the specific answer to THIS root cause?
   └── "[Unique qualification]"

3. Would another mechanism solve it better?
   └── [No | Possibly | Yes—wrong mechanism]

UNIQUENESS SCORE: [High | Medium | Low]
```

**Good Alignment Example** (SF1):
- Root Cause: "Technique breaks down under pressure—you can't manually square the clubface when it matters"
- Mechanism: "Square Face Technology"
- Uniqueness: High—specifically engineered to square face automatically, unlike instruction or other drivers

---

### Step 6: Calculate Alignment Score

```
ALIGNMENT SCORE CALCULATION:

Direct Address (0-10):
├── All components directly addressed: 9-10
├── Most components addressed: 7-8
├── Some components addressed: 4-6
├── Few components addressed: 1-3
└── No components addressed: 0
Score: [X]

Logical Flow (0-10):
├── Obvious/Inevitable: 9-10
├── Clear connection: 7-8
├── Requires explanation: 4-6
├── Forced: 1-3
└── No connection: 0
Score: [X]

Gap Severity (0-10):
├── No gaps: 9-10
├── Minor gaps: 7-8
├── Moderate gaps: 4-6
├── Significant gaps: 1-3
└── Critical gaps: 0
Score: [X]

Uniqueness Fit (0-10):
├── High uniqueness: 9-10
├── Moderate uniqueness: 6-8
├── Low uniqueness: 3-5
└── Generic/replaceable: 0-2
Score: [X]

TOTAL ALIGNMENT SCORE: [Average] / 10
```

---

## Quality Gates

Before proceeding:

- [ ] Total alignment score is 7.0+ (minimum for effective copy)
- [ ] Logical flow is at least "clear" (not forced)
- [ ] No critical gaps between root cause and mechanism
- [ ] Mechanism is reasonably unique solution to this root cause

---

## Threshold Guidelines

| Score | Status | Action |
|-------|--------|--------|
| 8.5+ | Strong | Perfect alignment—proceed |
| 7.0-8.4 | Good | Minor gaps acceptable—proceed |
| 5.0-6.9 | Weak | Significant gaps—consider adjusting root cause expression |
| Below 5.0 | Misaligned | Alignment broken—revisit derivation |

---

## Handoff

Pass Mechanism Alignment Report to:
- **3.3 Proof Availability Score** (to continue validation)
- **If weak/misaligned**: Return to **Layer 1** or **Layer 2** for adjustment
