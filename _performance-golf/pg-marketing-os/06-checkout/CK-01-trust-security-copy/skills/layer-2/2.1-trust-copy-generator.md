# 2.1-trust-copy-generator.md

> **Layer:** 2 (Generation)
> **Skill:** CK-01-trust-security-copy
> **Purpose:** Generate all trust/security/guarantee copy across all 5 categories
> **Outputs:** `trust_copy_set` object with copy for every mapped signal

---

## FUNCTION

Generate the actual copy text for every trust signal in the signal map. This is precision copywriting -- each piece of trust copy must be factual, minimal, specific, and reassuring. No persuasion language. No promotional framing. Every word exists to make the buyer feel safe enough to complete their purchase.

---

## GENERATION PROTOCOL

```
EXECUTE CK-01/2.1-trust-copy-generator:
  INPUT: trust_signal_map (from 1.1), placement_plan (from 1.2),
         guarantee_details (from 0.1)
  OUTPUT: trust_copy_set object

  STEP 1: GENERATE SECURITY COPY
    - SSL badge text: short, factual, specific
      Example: "Secure Checkout" / "256-bit SSL Encryption"
    - Lock icon label: minimal
      Example: "Protected" / "Encrypted"
    - Security statement: factual encryption claim
      Example: "Your information is protected with 256-bit SSL encryption"
    - RULE: No vague claims. "We take security seriously" -> BANNED.
      Must cite specific technology or standard.

  STEP 2: GENERATE PAYMENT COPY
    - Accepted payment label (if needed): "We accept" or none (logos only)
    - Express checkout labels: "Pay with PayPal" / "Apple Pay"
    - RULE: Payment signals are primarily visual (logos). Text is minimal.

  STEP 3: GENERATE GUARANTEE COPY
    - Guarantee headline: [Duration]-[Type] Guarantee
      Example: "60-Day Money-Back Guarantee"
    - Guarantee body: risk reversal in 8-15 words
      Example: "Try it risk-free. If you are not satisfied, return for a full refund."
    - RULE: Terms MUST match offer-package.json exactly.
      Duration, type, and conditions cannot be invented or assumed.
    - Guarantee reminder (footer): shortened version
      Example: "60-Day Money-Back Guarantee"

  STEP 4: GENERATE SOCIAL PROOF COPY
    - Customer count: "Trusted by [X]+ customers" or "Join [X]+ customers"
    - Review rating: "[X] stars from [Y] reviews" or star visual
    - RULE: Numbers must be real (from client) or placeholder "[CLIENT: actual number]"
    - NEVER fabricate: "Trusted by 100,000+ customers" without source

  STEP 5: GENERATE CONTACT COPY
    - Phone: "Questions? Call [number]" with hours
    - Email: "Need help? Email [address]"
    - RULE: Contact info must be real or "[CLIENT TO PROVIDE]"
    - NEVER invent phone numbers or email addresses
    - Mobile: phone number should be tap-to-call link

  STEP 6: PERSUASION SCAN
    a. Scan ALL generated copy for persuasion language
    b. Check against anti-slop lexicon
    c. Flag and replace any violations
    d. Violations include: "incredible," "amazing," "don't miss,"
       "exclusive," "best deal," superlatives without proof

  STEP 7: WORD COUNT VALIDATION
    - Verify each signal is within word budget from 1.1
    - Total trust copy should be 40-70 words (very lean)
    - If over budget: cut words until within budget

  IF persuasion language detected:
    REMOVE and REPLACE with factual alternative before proceeding.
```

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Persuasion language in generated copy | REMOVE and REPLACE -- do not proceed until clean |
| CRITICAL | Guarantee terms do not match offer package | FIX -- match exactly to offer package |
| CRITICAL | Contact info fabricated | REPLACE with "[CLIENT TO PROVIDE]" placeholder |
| DEGRADED | Social proof numbers unavailable | Use placeholder structure, flag for client |
| DEGRADED | Word count over budget | Cut words -- trust copy must be minimal |
