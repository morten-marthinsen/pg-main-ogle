# 1.2-placement-optimizer.md

> **Layer:** 1 (Mapping)
> **Skill:** CK-01-trust-security-copy
> **Purpose:** Optimize placement for 3+ trust signals visible at all scroll positions
> **Outputs:** `placement_plan` object with per-viewport signal assignments

---

## FUNCTION

Take the mapped trust signals from 1.1 and the flow map from CK-00, then optimize signal placement to guarantee that at least 3 trust signals are visible at every scroll position on both desktop and mobile viewports. This is a structural optimization -- the density target is non-negotiable.

---

## PLACEMENT OPTIMIZATION PROTOCOL

```
EXECUTE CK-01/1.2-placement-optimizer:
  INPUT: trust_signal_map (from 1.1), flow_map (from CK-00 strategy)
  OUTPUT: placement_plan object

  STEP 1: MAP SCROLL POSITIONS
    - Divide checkout page into viewport-height segments
    - For desktop: estimate 4-6 viewport segments for single-page checkout
    - For mobile: estimate 6-10 viewport segments (more scrolling)
    - For multi-step: evaluate each step as its own viewport
    - Label each segment by the checkout section(s) visible

  STEP 2: ASSIGN PRIMARY SIGNALS
    - Place guarantee near payment/CTA (highest anxiety moment)
    - Place security badges near card entry fields
    - Place payment logos in payment section
    - Place social proof near order summary
    - Place contact info in header or footer (always accessible)

  STEP 3: CHECK DENSITY PER VIEWPORT
    a. For each viewport segment, count visible trust signals
    b. If any segment has fewer than 3: FLAG
    c. For flagged segments: add supplementary signals
       - Options: sticky trust bar, repeated guarantee text,
         inline security mention, footer trust row

  STEP 4: OPTIMIZE MOBILE DENSITY
    - Mobile viewports show less content per screen
    - Trust signals must be visible even on small screens
    - Consider: sticky trust badge (always visible)
    - Consider: inline trust signals within form sections
    - Consider: collapsible trust panel

  STEP 5: DEFINE PLACEMENT RULES
    - Header trust row: security badge + contact info (always visible on scroll)
    - Inline trust: security badge near card fields, payment logos in payment section
    - Pre-CTA trust: guarantee badge above submit button
    - Footer trust row: all logos + guarantee reminder + contact
    - Sticky mobile: security badge or guarantee mini-badge

  IF density < 3 at any position after optimization:
    ESCALATE -- Add more signals. Options:
    1. Add sticky trust element (visible at all positions)
    2. Repeat key signals at multiple positions
    3. Add new signal type (testimonial snippet, certification badge)
```

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Any viewport segment below 3 signals | ADD signals -- sticky element, repeated signals, or new signal type |
| CRITICAL | Mobile density plan missing | HALT -- mobile density is mandatory |
| DEGRADED | Density achieved only with sticky element | Document dependency -- if sticky element removed, density fails |
| DEGRADED | Trust signals cluster in one area only | Redistribute across checkout flow |
