# 4.1-trust-density-validator.md

> **Layer:** 4 (Validation)
> **Skill:** CK-01-trust-security-copy
> **Purpose:** Verify 3+ trust signals visible at every scroll position
> **Outputs:** `density_validation` object with per-viewport verification

---

## FUNCTION

Validate that the generated trust copy, when placed according to the placement plan, achieves the mandatory minimum of 3 trust signals visible at every scroll position on both desktop and mobile. This is the critical quality gate -- trust density is the single most important structural metric for checkout abandonment prevention.

---

## DENSITY VALIDATION PROTOCOL

```
EXECUTE CK-01/4.1-trust-density-validator:
  INPUT: trust_copy_set (from 2.1), placement_plan (from 1.2)
  OUTPUT: density_validation object

  STEP 1: SIMULATE DESKTOP VIEWPORTS
    - Walk through each viewport segment from placement plan
    - For each segment: count trust signals with generated copy
    - Verify count >= 3 at every position
    - Record minimum count across all positions

  STEP 2: SIMULATE MOBILE VIEWPORTS
    - Walk through each mobile viewport segment
    - Account for stacking, reflow, and smaller viewport
    - Verify count >= 3 at every position
    - Account for sticky elements (always visible)
    - Record minimum count across all positions

  STEP 3: IDENTIFY SPARSE ZONES
    - Any viewport segment with exactly 3 signals: flag as "at minimum"
    - Any viewport segment with fewer than 3: flag as "FAILING"
    - For failing zones: recommend specific additions

  STEP 4: VALIDATE COPY COMPLETENESS
    a. Every signal in placement plan has generated copy
    b. No signals are placement-only without copy
    c. Guarantee copy verified against offer package terms

  STEP 5: FINAL PERSUASION SCAN
    - One final check across all generated copy
    - Any persuasion language found at this stage: CRITICAL FAILURE
    - Must be resolved before packaging

  STEP 6: PRODUCE VALIDATION VERDICT
    - PASS: all viewports >= 3 signals, no persuasion, copy complete
    - FAIL: specific positions below target, remediation plan

  IF any viewport < 3 signals:
    FAIL -- "Viewport segment [X] has only [Y] trust signals.
    Add: [specific recommendation]."
```

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Any viewport below 3 signals | ADD signals at sparse position -- sticky element, repeated signal, or new signal |
| CRITICAL | Persuasion language found in final scan | REMOVE and REPLACE immediately |
| CRITICAL | Signal in plan has no generated copy | GENERATE copy for missing signal |
| DEGRADED | Multiple positions at exactly 3 (minimum) | WARN -- fragile density, consider adding buffer signals |
