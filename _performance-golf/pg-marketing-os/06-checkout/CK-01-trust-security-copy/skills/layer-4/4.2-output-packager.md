# 4.2-output-packager.md

> **Layer:** 4 (Validation & Packaging)
> **Skill:** CK-01-trust-security-copy
> **Purpose:** Package trust-copy-package.json as final output
> **Outputs:** `trust-copy-package.json` file

---

## FUNCTION

Assemble the validated trust copy into the final `trust-copy-package.json` output file. This is the canonical deliverable that CK-03 (Checkout Editorial) consumes for review and integration. The package must include all trust copy organized by category, placement specifications, density verification results, and any client placeholder flags.

---

## PACKAGING PROTOCOL

```
EXECUTE CK-01/4.2-output-packager:
  INPUT: trust_copy_set (from 2.1), density_validation (from 4.1),
         placement_plan (from 1.2)
  OUTPUT: trust-copy-package.json file

  STEP 1: VERIFY VALIDATION PASSED
    - Confirm density_validation.status == PASS
    - Confirm persuasion_scan.clean == true
    - If either FAIL: HALT -- cannot package unvalidated copy

  STEP 2: ASSEMBLE JSON STRUCTURE
    - Header: version, timestamp, skill_id
    - Trust copy by category (all 5 categories)
    - Placement specifications per signal
    - Density verification summary
    - Copy metadata (total word count, persuasion status, client placeholders)
    - Client action items (data needed from client)

  STEP 3: VALIDATE SCHEMA
    - All required keys present
    - No null values where copy expected
    - Placeholders marked with "[CLIENT TO PROVIDE]" pattern
    - Word counts accurate
    - Guarantee terms match source

  STEP 4: WRITE FILE
    - Write trust-copy-package.json to project outputs directory
    - Verify file written successfully
    - Log completion timestamp

  STEP 5: CREATE HANDOFF SUMMARY
    - Document what CK-03 receives from this package
    - List any client action items
    - Note any warnings or limitations

  IF schema validation fails:
    HALT -- Fix violations before writing.
```

---

## OUTPUT STRUCTURE

The output IS the `trust-copy-package.json` file containing the full schema from the CK-01-AGENT.md output schema section.

Additionally, create a verification object:

```json
{
  "package_assembly": {
    "status": "complete",
    "file_written": "trust-copy-package.json",
    "file_path": "/path/to/file",
    "schema_version": "1.0",
    "validation_passed": true,
    "density_verified": true,
    "persuasion_free": true,
    "client_action_items": [
      "Provide actual customer count for social proof",
      "Provide support phone number and hours"
    ],
    "written_at": "ISO timestamp"
  }
}
```

```

[ ] All 5 trust categories have copy

[ ] Guarantee terms match offer package

[ ] Client placeholders clearly marked

[ ] Mobile display specified for every signal
```

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Density validation failed | HALT -- return to 4.1, add signals |
| CRITICAL | Persuasion language in package | HALT -- remove before packaging |
| CRITICAL | Schema validation fails | Fix specific violations |
| DEGRADED | Client placeholders present | Package with placeholders, flag in handoff |
