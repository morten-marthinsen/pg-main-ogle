# 0.2-input-validator.md

> **Layer:** 0 (Foundation)
> **Skill:** CK-01-trust-security-copy
> **Purpose:** Validate trust architecture plan present and complete
> **Outputs:** `validation_result` object with pass/fail and gap report

---

## FUNCTION

Validate that the loaded upstream context contains a complete trust architecture plan with all 5 signal categories defined, guarantee details verified against the offer package, and density targets specified. This gate prevents trust copy generation from starting with incomplete or mismatched inputs.

---

## VALIDATION PROTOCOL

```
EXECUTE CK-01/0.2-input-validator:
  INPUT: upstream_context (from 0.1)
  OUTPUT: validation_result object

  STEP 1: VALIDATE TRUST ARCHITECTURE COMPLETENESS
    - Security category present with signals and placement
    - Payment category present with signals and placement
    - Guarantee category present with signals and placement
    - Social proof category present with signals and placement
    - Contact category present with signals and placement
    - All 5 categories: minimum_visible count defined

  STEP 2: VALIDATE GUARANTEE ACCURACY
    - Guarantee type matches offer package (money_back/satisfaction/performance)
    - Duration matches offer package
    - Conditions match offer package
    - No invented or assumed terms

  STEP 3: VALIDATE DENSITY TARGETS
    - Density target is explicitly stated (not assumed)
    - Mobile density plan exists
    - Desktop density plan exists

  STEP 4: PRODUCE VERDICT
    a. PASS: all 5 categories complete, guarantee verified, density targets defined
    b. FAIL: specific gaps listed with remediation

  IF any category missing:
    FAIL -- "Trust category [X] has no signals planned. Return to CK-00."

  IF guarantee terms not verified:
    FAIL -- "Guarantee terms must be verified against offer package before copy generation."
```

---

## OUTPUT STRUCTURE

```json
{
  "validation_result": {
    "status": "PASS|FAIL",
    "checks": {
      "security_category_complete": true,
      "payment_category_complete": true,
      "guarantee_category_complete": true,
      "social_proof_category_complete": true,
      "contact_category_complete": true,
      "guarantee_terms_verified": true,
      "density_targets_defined": true
    },
    "critical_gaps": [],
    "warnings": [],
    "proceed": true
  }
}
```

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Any trust category missing entirely | HALT -- return to CK-00 |
| CRITICAL | Guarantee terms unverified | HALT -- verify against offer package |
| CRITICAL | No density target defined | HALT -- must know target before copy generation |
| DEGRADED | Social proof data unavailable | WARN -- will use placeholder structure |
| DEGRADED | Contact info not yet provided | WARN -- will use "[CLIENT TO PROVIDE]" |
