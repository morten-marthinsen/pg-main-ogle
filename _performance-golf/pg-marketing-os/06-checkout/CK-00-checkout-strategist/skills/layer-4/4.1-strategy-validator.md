# 4.1-strategy-validator.md

> **Layer:** 4 (Validation)
> **Skill:** CK-00-checkout-strategist
> **Purpose:** Validate strategy completeness against downstream skill requirements
> **Outputs:** `validation_report` object with pass/fail and gap analysis

---

## FUNCTION

Validate that the assembled checkout strategy contains everything CK-01 (Trust & Security), CK-02 (Form & Micro-Copy), CK-03 (Checkout Editorial), and U1 (Order Bump) need to execute without ambiguity. This is the final gate before packaging -- no strategy ships with gaps that would block downstream skills.

---

## VALIDATION PROTOCOL

```
EXECUTE CK-00/4.1-strategy-validator:
  INPUT: checkout_strategy (from 2.1)
  OUTPUT: validation_report object

  STEP 1: VALIDATE CK-01 REQUIREMENTS
    - Trust architecture present with all 5 categories
    - Each category has: signals list, placement, minimum visible count
    - Density target defined (3+ per viewport)
    - Guarantee details present (type, duration)
    - Mobile trust plan exists

  STEP 2: VALIDATE CK-02 REQUIREMENTS
    - Checkout pattern defined (single-page or multi-step)
    - Flow map present with sections in order
    - Field requirements listed per section
    - Error handling needs identified
    - Mobile field behavior specified
    - Order bump placement defined

  STEP 3: VALIDATE CK-03 REQUIREMENTS
    - Mobile strategy present for every section
    - Friction mitigation plan assigns responsibilities
    - Trust density target measurable
    - Flow map has no gaps

  STEP 4: VALIDATE U1 REQUIREMENTS
    - Order bump spec present
    - Word budget defined (50-150 words)
    - Format specified (checkbox with copy block)
    - Placement location defined
    - Integration constraint documented

  STEP 5: VALIDATE STRATEGY INTEGRITY
    a. No copy in strategy output (structural only)
    b. All funnel type implications addressed
    c. Every friction point has mitigation assignment
    d. Trust density meets minimum (3+ per viewport)
    e. Mobile behavior specified for every section

  STEP 6: PRODUCE VALIDATION VERDICT
    - PASS: all requirements met, no gaps
    - FAIL: specific gaps identified with remediation instructions

  IF any CRITICAL gap found:
    FAIL -- Return to Layer 2 for remediation.
    Specify exactly which gap and what is needed.
```

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Trust architecture missing category | Return to 2.1 -- add missing category |
| CRITICAL | Flow map has section gaps | Return to 2.1 -- fill gaps |
| CRITICAL | No mobile strategy | Return to 2.1 -- add mobile for every section |
| CRITICAL | Copy found in strategy output | Remove copy, replace with structural description |
| DEGRADED | Minor gap in one downstream requirement | Flag with specific remediation, allow packaging with warning |
