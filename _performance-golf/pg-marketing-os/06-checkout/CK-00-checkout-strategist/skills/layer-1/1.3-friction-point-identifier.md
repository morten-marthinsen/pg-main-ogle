# 1.3-friction-point-identifier.md

> **Layer:** 1 (Classification)
> **Skill:** CK-00-checkout-strategist
> **Purpose:** Identify all potential friction points in the checkout flow
> **Outputs:** `friction_analysis` object with categorized friction points, severity ratings, and mitigation assignments

---

## FUNCTION

Systematically analyze the planned checkout flow for every potential point of friction -- any moment where the buyer might hesitate, feel confused, lose trust, or abandon. Every checkout has friction points. If analysis returns zero, it means analysis was incomplete, not that the checkout is frictionless. Minimum output: 3 friction points identified.

---

## FRICTION IDENTIFICATION PROTOCOL

```
EXECUTE CK-00/1.3-friction-point-identifier:
  INPUT: funnel_classification (from 1.1), checkout_pattern (from 1.2), upstream_context (from 0.1)
  OUTPUT: friction_analysis object

  STEP 1: SCAN FORM FRICTION
    - Field count: more fields = more friction
    - Required vs. optional fields: required fields the buyer may not expect
    - Field labels: any ambiguous or confusing labels
    - Field ordering: does the order feel natural?
    - Auto-fill compatibility: can browser auto-fill work?
    - Phone number field: biggest single-field friction source (is it necessary?)
    - Address validation: real-time or on-submit?

  STEP 2: SCAN PAYMENT FRICTION
    - Payment options: are all expected options available?
    - Card entry: number, expiry, CVV -- any non-standard formatting?
    - Payment security perception: do buyers feel safe entering card info?
    - Subscription terms: if recurring, are billing terms crystal clear?
    - Trial terms: if trial, is the future charge amount and date explicit?
    - PayPal/Apple Pay: are express checkout options available?

  STEP 3: SCAN TRUST FRICTION
    - Trust signal visibility: are 3+ signals visible at all times?
    - Guarantee prominence: is the guarantee visible near payment entry?
    - Security indicators: SSL badge, lock icon, encryption mention?
    - Social proof presence: customer count, reviews, ratings?
    - Contact availability: can buyer reach support if stuck?

  STEP 4: SCAN FLOW FRICTION
    - Page load speed: heavy images or scripts slowing checkout?
    - Progress clarity: does the buyer know where they are in the process?
    - Back navigation: can the buyer go back without losing data?
    - Error recovery: does an error wipe the form or preserve entries?
    - CTA clarity: is the button text specific ("Complete Order") not vague ("Submit")?

  STEP 5: SCAN MOBILE-SPECIFIC FRICTION
    - Tap target size: are buttons and fields 44px+ height?
    - Keyboard switching: does field type trigger correct mobile keyboard?
    - Viewport scroll: can buyer see trust signals without scrolling?
    - Sticky elements: is CTA button sticky on long mobile forms?
    - Orientation: does checkout work in both portrait and landscape?

  STEP 6: RATE AND ASSIGN
    a. Rate each identified friction point: HIGH / MEDIUM / LOW
    b. Assign mitigation responsibility: CK-01 (trust), CK-02 (form/micro-copy), CK-03 (editorial)
    c. Prioritize by severity and conversion impact

  IF friction_points_count < 3:
    HALT -- "Analysis incomplete. Every checkout has at least 3 friction points. Re-run analysis."
```

---

## FRICTION CATEGORY REFERENCE

| Category | Common Friction Points | Typical Severity |
|----------|----------------------|-----------------|
| Form Fields | Too many fields, confusing labels, no auto-fill | HIGH |
| Payment | Limited options, unclear subscription terms | HIGH |
| Trust | Missing badges, no guarantee visible, no contact info | HIGH |
| Flow | Slow loading, unclear progress, no back button | MEDIUM |
| Mobile | Small tap targets, wrong keyboard, no sticky CTA | MEDIUM |
| Error Handling | Blaming errors, form reset on error, no inline validation | HIGH |
| Price Surprise | Unexpected shipping costs, hidden fees, tax surprise | HIGH |

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Zero friction points identified | HALT -- analysis incomplete. Every checkout has friction. Re-run. |
| CRITICAL | Fewer than 3 friction points | HALT -- analysis shallow. Check all 5 scan categories. |
| DEGRADED | Cannot assess mobile friction (no layout info) | Flag mobile category as "requires CK-03 audit" |
| DEGRADED | Cannot assess payment friction (options unknown) | Flag payment category with assumptions used |
