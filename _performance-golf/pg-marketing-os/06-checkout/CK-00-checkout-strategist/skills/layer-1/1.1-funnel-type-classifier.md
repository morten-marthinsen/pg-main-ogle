# 1.1-funnel-type-classifier.md

> **Layer:** 1 (Classification)
> **Skill:** CK-00-checkout-strategist
> **Purpose:** Classify funnel type (direct/supplement/digital/free+ship/trial)
> **Outputs:** `funnel_classification` object with type, confidence, and rationale

---

## FUNCTION

Analyze the upstream context (product type, pricing structure, offer architecture, fulfillment model) to classify the funnel into one of five recognized checkout funnel types. This classification drives all downstream decisions: checkout pattern, field requirements, trust signal emphasis, and order summary structure.

---

## CLASSIFICATION PROTOCOL

```
EXECUTE CK-00/1.1-funnel-type-classifier:
  INPUT: upstream_context (from 0.1), checkout_specimens (from 0.2)
  OUTPUT: funnel_classification object

  STEP 1: EXTRACT CLASSIFICATION SIGNALS
    - Product type: physical or digital
    - Pricing: one-time, subscription, trial, free+shipping
    - Shipping required: yes/no
    - Fulfillment: physical delivery, digital access, membership
    - Price point: high (>$100), mid ($30-$100), low (<$30)

  STEP 2: MATCH TO FUNNEL TYPE
    a. DIRECT SALE (standard e-commerce)
       Signals: one-time purchase, standard pricing, physical or digital
       Checkout needs: standard fields, straightforward payment

    b. SUPPLEMENT / PHYSICAL HEALTH
       Signals: physical product, often subscription option, shipping required
       Checkout needs: subscription toggle, shipping fields, recurring billing terms

    c. DIGITAL PRODUCT
       Signals: digital delivery, no shipping, instant access
       Checkout needs: simplified form (no shipping), fewer fields, access delivery plan

    d. FREE + SHIPPING
       Signals: product free, customer pays S&H only, card capture for S&H
       Checkout needs: "FREE" price emphasis, S&H amount clear, trust heavy (card for "free" raises anxiety)

    e. TRIAL OFFER
       Signals: trial period, future billing, conditional purchase
       Checkout needs: trial terms prominent, billing schedule crystal clear, trust maximum

  STEP 3: CALCULATE CONFIDENCE
    - Count matching signals for top candidate
    - Count total classification signals available
    - Confidence = matching_signals / total_signals
    - If top 2 candidates within 0.10 of each other: flag as AMBIGUOUS

  STEP 4: GENERATE CLASSIFICATION REPORT
    - Primary classification with confidence
    - Alternative classification if within range
    - Specific signals that drove the decision
    - Any ambiguity flags

  IF confidence < 0.50:
    HALT -- "Funnel type cannot be determined. Request human classification."
    Present top 2 candidates with signal analysis.

  IF confidence 0.50-0.74:
    WARN -- "Moderate confidence. Presenting classification for human confirmation."
    Trigger optional checkpoint.
```

---

## FUNNEL TYPE REFERENCE

| Funnel Type | Key Signal | Field Impact | Trust Emphasis |
|-------------|-----------|--------------|----------------|
| Direct Sale | Standard purchase | Standard fields | Standard trust |
| Supplement | Physical + subscription | Shipping + subscription toggle | Recurring billing clarity |
| Digital | No shipping | Reduced fields (4-5) | Instant access assurance |
| Free + Ship | Free product, S&H charge | S&H price prominent | High trust (card for "free") |
| Trial | Trial period + future billing | Trial terms area | Maximum trust (future charges) |

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Confidence < 0.50 | HALT -- request human classification with signal analysis |
| DEGRADED | Confidence 0.50-0.74 | WARN -- present for human confirmation, continue if confirmed |
| DEGRADED | Multiple types equally likely | Present top 2 with trade-off analysis, request human selection |
| DEGRADED | Product type hybrid | Classify based on primary product, note hybrid elements |
