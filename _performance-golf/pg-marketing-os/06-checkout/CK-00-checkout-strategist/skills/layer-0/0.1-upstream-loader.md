# 0.1-upstream-loader.md

> **Layer:** 0 (Foundation)
> **Skill:** CK-00-checkout-strategist
> **Purpose:** Load campaign brief and offer package from upstream skills
> **Outputs:** `upstream_context` object for downstream microskills

---

## FUNCTION

Load and parse all upstream skill packages required for checkout strategy planning. The campaign brief provides audience, voice, and product context. The offer package provides pricing, guarantee terms, payment options, and value stack. Optional E-Comm outputs provide product-specific checkout context.

---

## REQUIRED UPSTREAM PACKAGES

| Package | Source Skill | Critical Fields |
|---------|--------------|-----------------|
| `campaign-brief-package.json` | 09-campaign-brief | `avatar`, `product`, `creative_direction`, `niche` |
| `offer-package.json` | 07-offer-package | `pricing`, `guarantee`, `payment_options`, `value_stack`, `subscription` |
| E-Comm outputs (optional) | 03-e-comm | `product_type`, `shipping_requirements`, `fulfillment_model` |

---

## LOADING PROTOCOL

```
EXECUTE CK-00/0.1-upstream-loader:
  INPUT: skill output directories
  OUTPUT: upstream_context object

  STEP 1: LOAD CAMPAIGN BRIEF
    - Locate campaign-brief-package.json in outputs directory
    - Parse JSON and validate structure
    - Extract: avatar, product details, creative direction, niche
    - Validate: product field is non-null

  STEP 2: LOAD OFFER PACKAGE
    - Locate offer-package.json in outputs directory
    - Parse JSON and validate structure
    - Extract: pricing (price points, tiers), guarantee (type, duration),
      payment options (cards, PayPal, etc.), value stack, subscription terms
    - Validate: pricing field is non-null (CRITICAL -- cannot plan checkout without price)

  STEP 3: LOAD E-COMM CONTEXT (OPTIONAL)
    - Check if E-Comm outputs exist
    - If present: extract product_type, shipping_requirements, fulfillment_model
    - If absent: flag as N/A, proceed without

  STEP 4: ASSEMBLE CONTEXT
    - Combine all loaded data into upstream_context object
    - Set loading_status flags
    - Document any missing or degraded fields

  IF offer-package missing:
    HALT -- "Cannot plan checkout without offer package. Ensure Skill 07 has completed."

  IF campaign-brief missing:
    HALT -- "Cannot plan checkout without campaign brief. Ensure Skill 09 has completed."
```

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | offer-package.json missing | Cannot proceed -- request Skill 07 completion |
| CRITICAL | campaign-brief-package.json missing | Cannot proceed -- request Skill 09 completion |
| CRITICAL | Pricing field null in offer package | Cannot proceed -- pricing is required for checkout |
| DEGRADED | Guarantee details absent | Proceed with placeholder, flag for CK-01 |
| DEGRADED | Payment options unspecified | Default to standard set, flag for confirmation |
| DEGRADED | E-Comm context absent | Proceed without -- not required |
