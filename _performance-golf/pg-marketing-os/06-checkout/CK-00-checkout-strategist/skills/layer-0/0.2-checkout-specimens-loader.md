# 0.2-checkout-specimens-loader.md

> **Layer:** 0 (Foundation)
> **Skill:** CK-00-checkout-strategist
> **Purpose:** Load checkout specimen patterns from CHECKOUT-ENGINE.md
> **Outputs:** `checkout_specimens` object with flow patterns and trust templates

---

## FUNCTION

Load the institutional checkout knowledge from CHECKOUT-ENGINE.md, including flow patterns (single-page, multi-step), trust signal categories, form micro-copy templates, and funnel-specific checkout variations. This provides the strategic reference library that Layer 1 classification and Layer 2 assembly draw from.

---

## SPECIMEN LOADING PROTOCOL

```
EXECUTE CK-00/0.2-checkout-specimens-loader:
  INPUT: CHECKOUT-ENGINE.md, checkout specimen files
  OUTPUT: checkout_specimens object

  STEP 1: LOAD ENGINE DOCUMENT
    - Read CHECKOUT-ENGINE.md from 06-checkout/ root
    - Parse all sections: 5 Laws, Architecture, Trust Architecture,
      Form Architecture, Flow Patterns
    - Index each section for Layer 1/2 reference

  STEP 2: EXTRACT FLOW PATTERNS
    - Single-page checkout layout and section order
    - Multi-step checkout progression and step labels
    - Funnel-specific variations (direct/supplement/digital/free+ship/trial)
    - For each pattern: sections, field requirements, trust signal placements

  STEP 3: EXTRACT TRUST TEMPLATES
    - Trust signal categories (security/payment/guarantee/social/contact)
    - Minimum density rules (3+ visible at all times)
    - Guarantee copy templates (money-back/satisfaction/performance)
    - Placement specifications per category

  STEP 4: EXTRACT ANTI-DEGRADATION PATTERNS
    - The 5 Laws (checkout-specific rules)
    - 4 degradation patterns (sales page, missing trust, hostile errors, desktop-first)
    - Anti-degradation protocol triggers

  STEP 5: LOAD EXTERNAL SPECIMENS (if available)
    - Check for checkout specimen files in specimens/ directory
    - Load any captured checkout flows from real funnels
    - Index by funnel type for pattern matching

  IF CHECKOUT-ENGINE.md missing:
    HALT -- "Cannot proceed without institutional checkout knowledge"
```
