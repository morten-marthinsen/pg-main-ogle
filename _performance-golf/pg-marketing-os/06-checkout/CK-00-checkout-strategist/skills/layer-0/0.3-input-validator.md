# 0.3-input-validator.md

> **Layer:** 0 (Foundation)
> **Skill:** CK-00-checkout-strategist
> **Purpose:** Validate offer architecture present and complete for checkout planning
> **Outputs:** `validation_result` object with pass/fail status and gap report

---

## FUNCTION

Validate that all required inputs from 0.1 (upstream loader) and 0.2 (specimens loader) contain sufficient information to proceed with checkout strategy planning. This is the gate between loading and classification -- no classification begins until validation passes.

---

## VALIDATION PROTOCOL

```
EXECUTE CK-00/0.3-input-validator:
  INPUT: upstream_context (from 0.1), checkout_specimens (from 0.2)
  OUTPUT: validation_result object

  STEP 1: VALIDATE OFFER ARCHITECTURE
    - Pricing present and non-null
    - At least one price point defined
    - Product type identifiable (physical/digital/subscription/hybrid)
    - Guarantee terms present (type + duration minimum)
    - Payment options listed (at least one)

  STEP 2: VALIDATE CAMPAIGN CONTEXT
    - Avatar defined with at least primary_pain or primary_desire
    - Product name present
    - Niche identifiable

  STEP 3: VALIDATE SPECIMENS LOADED
    - CHECKOUT-ENGINE.md parsed successfully
    - Flow patterns available
    - Trust templates available
    - 5 Laws indexed

  STEP 4: ASSESS COMPLETENESS
    a. Count required fields present vs. total required
    b. Classify any gaps as CRITICAL (blocks execution) or DEGRADED (proceed with warning)
    c. If all CRITICAL fields present: PASS
    d. If any CRITICAL field missing: FAIL with specific gap report

  STEP 5: GENERATE VALIDATION REPORT
    - List all checked fields with status
    - Flag any degraded fields with downstream impact
    - Produce clear PASS/FAIL verdict

  IF validation FAILS:
    HALT with specific remediation instructions:
    "Missing [field] from [source]. Cannot proceed until [action]."
```

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Pricing missing | HALT -- cannot plan checkout without price |
| CRITICAL | Product type unidentifiable | HALT -- need physical/digital determination for field count |
| CRITICAL | CHECKOUT-ENGINE.md not loaded | HALT -- need institutional patterns |
| DEGRADED | Guarantee terms absent | WARN -- proceed with placeholder, CK-01 will need this |
| DEGRADED | Payment options unspecified | WARN -- default to standard set |
| DEGRADED | Avatar minimal | WARN -- reduced friction analysis quality |
