# 2.1-strategy-assembler.md

> **Layer:** 2 (Strategy Assembly)
> **Skill:** CK-00-checkout-strategist
> **Purpose:** Assemble trust architecture + checkout flow map + order bump spec
> **Outputs:** `checkout_strategy` object -- the complete strategic blueprint

---

## FUNCTION

Combine all Layer 1 outputs (funnel classification, checkout pattern, friction analysis) with upstream context to produce the complete checkout strategy. This is the core assembly step -- it builds the trust architecture plan, sequences the checkout flow map, defines the order bump specification, and assigns friction mitigation strategies. The output must be complete enough for CK-01 and CK-02 to execute without ambiguity.

---

## ASSEMBLY PROTOCOL

```
EXECUTE CK-00/2.1-strategy-assembler:
  INPUT: funnel_classification (1.1), checkout_pattern (1.2),
         friction_analysis (1.3), upstream_context (0.1),
         checkout_specimens (0.2)
  OUTPUT: checkout_strategy object

  STEP 1: BUILD TRUST ARCHITECTURE
    - Define 5 signal categories: security, payment, guarantee, social_proof, contact
    - For each category:
      a. List specific signals to include
      b. Assign placement locations in checkout flow
      c. Set minimum visible count
    - Validate density target: 3+ signals visible per viewport
    - Plan mobile density: how signals stack/reflow on mobile
    - Reference guarantee details from offer package
    - Cross-reference trust signal placement with friction points from 1.3

  STEP 2: BUILD CHECKOUT FLOW MAP
    - Sequence all sections in order (informed by checkout pattern from 1.2)
    - For each section:
      a. Define purpose (what this section accomplishes)
      b. List copy elements needed (labels, helper text, etc.)
      c. Assign trust signals to appear in/near this section
      d. Specify mobile behavior (stacking, sizing, visibility)
    - Ensure flow map covers: order summary, form sections, order bump, CTA,
      trust footer
    - Verify no gaps between sections

  STEP 3: BUILD ORDER BUMP SPECIFICATION
    - Determine placement: within order summary, after line items (standard)
    - Define format: checkbox with copy block
    - Set word budget: 50-150 words (U1 constraint)
    - Specify integration point: bridges to U1 (Upsell Engine) for copy
    - Note: order bump is the ONLY persuasion moment on checkout
    - If no order bump planned: document decision with rationale

  STEP 4: BUILD FRICTION MITIGATION PLAN
    - For each friction point from 1.3:
      a. Assign specific mitigation strategy
      b. Assign responsible downstream skill (CK-01, CK-02, or CK-03)
      c. Set priority based on severity rating
    - Verify every HIGH severity point has a specific mitigation
    - Verify every friction point is assigned to a downstream skill

  STEP 5: BUILD MOBILE STRATEGY
    - For every section in flow map:
      a. Specify mobile layout behavior (single column, stacking order)
      b. Specify field sizing (44px+ tap targets)
      c. Specify trust signal visibility on mobile viewport
    - Plan sticky elements (CTA button, progress bar)
    - Plan keyboard behavior per field type

  STEP 6: BUILD DOWNSTREAM HANDOFF SPEC
    - Define what CK-01 receives: trust architecture, guarantee details, trust density targets
    - Define what CK-02 receives: checkout pattern, flow map, field requirements, error handling needs
    - Define what U1 receives: order bump placement, format, word budget, checkout flow context
    - Verify each downstream skill has everything it needs

  IF trust density < 3 per viewport:
    REMEDIATE -- Add signals until target met. Do not proceed with insufficient trust density.

  IF flow map has gaps:
    REMEDIATE -- Identify missing sections and add before proceeding.
```

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Trust density below 3 per viewport | Add signals until target met -- do not proceed |
| CRITICAL | Flow map missing sections | Identify and add missing sections |
| CRITICAL | Order bump spec absent | Add spec (even if decision is "no order bump") |
| DEGRADED | Mobile strategy incomplete | Flag sections missing mobile behavior, assign to CK-03 |
| DEGRADED | Guarantee details from placeholder | Flag for CK-01 to confirm with client |
| DEGRADED | Friction point without specific mitigation | Assign generic mitigation, flag for CK-03 review |
