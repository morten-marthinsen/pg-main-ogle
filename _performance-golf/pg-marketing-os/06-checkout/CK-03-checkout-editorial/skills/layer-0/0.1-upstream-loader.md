# 0.1-upstream-loader.md

> **Layer:** 0 (Foundation)
> **Skill:** CK-03-checkout-editorial
> **Purpose:** Load all checkout copy elements from CK-01 and CK-02
> **Outputs:** `checkout_copy_elements` object with all copy for editorial review

---

## FUNCTION

Load both upstream copy packages -- `trust-copy-package.json` from CK-01 and `checkout-microcopy-package.json` from CK-02 -- and combine into a unified copy inventory for editorial review. Every piece of copy that will appear on the checkout page must be loaded here so that the audit passes in Layer 1 and Layer 2 can evaluate the complete picture.

---

## LOADING PROTOCOL

```
EXECUTE CK-03/0.1-upstream-loader:
  INPUT: CK-01 output directory, CK-02 output directory
  OUTPUT: checkout_copy_elements object

  STEP 1: LOAD TRUST COPY PACKAGE
    - Locate trust-copy-package.json from CK-01 outputs
    - Parse JSON and validate structure
    - Extract all trust copy by category (security, payment, guarantee, social, contact)
    - Extract placement specifications
    - Extract density verification results
    - Count total trust copy word count

  STEP 2: LOAD MICRO-COPY PACKAGE
    - Locate checkout-microcopy-package.json from CK-02 outputs
    - Parse JSON and validate structure
    - Extract all field copy (labels, helpers, errors, placeholders)
    - Extract order summary copy
    - Extract progress indicators (if multi-step)
    - Extract button text
    - Extract success confirmation
    - Count total micro-copy word count

  STEP 3: COMBINE INTO UNIFIED INVENTORY
    - Merge trust and micro-copy into single copy inventory
    - Organize by checkout section (matching flow map order)
    - Calculate total checkout word count (trust + micro-copy)
    - Index every copy element for audit reference

  STEP 4: SET LOADING STATUS
    - Both packages loaded: ready for audit
    - One package missing: HALT
    - Total word count: baseline for word reduction audit

  IF trust-copy-package.json missing:
    HALT -- "Cannot review checkout without trust copy. Run CK-01 first."

  IF checkout-microcopy-package.json missing:
    HALT -- "Cannot review checkout without micro-copy. Run CK-02 first."
```

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | trust-copy-package.json missing | HALT -- run CK-01 first |
| CRITICAL | checkout-microcopy-package.json missing | HALT -- run CK-02 first |
| DEGRADED | One package has validation warnings | Load with warnings, flag for extra scrutiny in audit |
| DEGRADED | Packages inconsistent (different checkout patterns) | FLAG -- may indicate CK-00 strategy changed between CK-01 and CK-02 runs |
