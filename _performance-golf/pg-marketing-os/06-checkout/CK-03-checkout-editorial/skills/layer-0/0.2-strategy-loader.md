# 0.2-strategy-loader.md

> **Layer:** 0 (Foundation)
> **Skill:** CK-03-checkout-editorial
> **Purpose:** Load checkout strategy for constraint checking during editorial review
> **Outputs:** `strategy_reference` object with constraints and targets for audit

---

## FUNCTION

Load the checkout-strategy.yaml from CK-00 as a reference document for editorial review. The strategy contains the trust density targets, funnel type classification, checkout pattern decisions, and friction mitigation plan that the editorial audit must verify against. This is a reference load, not a primary input -- the primary inputs are the copy packages from CK-01 and CK-02.

---

## LOADING PROTOCOL

```
EXECUTE CK-03/0.2-strategy-loader:
  INPUT: CK-00 output directory
  OUTPUT: strategy_reference object

  STEP 1: LOAD CHECKOUT STRATEGY
    - Locate checkout-strategy.yaml from CK-00 outputs
    - Parse YAML and validate structure
    - Extract key constraints for audit:
      a. Trust density target (3+ per viewport)
      b. Funnel type (for context)
      c. Checkout pattern (single-page or multi-step)
      d. Flow map (section order for density checking)
      e. Friction mitigation assignments (which were CK-03's responsibility)
      f. Order bump spec (persuasion only in this area)

  STEP 2: EXTRACT AUDIT CONSTRAINTS
    - Trust density: minimum signals per viewport
    - Mobile requirements: what mobile behaviors were planned
    - Friction points assigned to CK-03: editorial review items
    - Order bump boundary: where persuasion language IS acceptable

  STEP 3: SET REFERENCE STATUS
    - Strategy loaded: audit has full constraint context
    - Strategy missing: WARN but do not HALT (can audit without, use defaults)

  IF checkout-strategy.yaml missing:
    WARN -- "Strategy not available for constraint checking.
    Using defaults: 3+ trust density, all positions mobile-specified,
    zero persuasion outside order bump."
```

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| DEGRADED | Strategy file missing | WARN -- use default constraints (3+ density, zero persuasion, full mobile) |
| DEGRADED | Strategy partially parsed | Extract available constraints, default the rest |
| DEGRADED | Flow map not in strategy | Use section order from CK-01/CK-02 packages instead |
