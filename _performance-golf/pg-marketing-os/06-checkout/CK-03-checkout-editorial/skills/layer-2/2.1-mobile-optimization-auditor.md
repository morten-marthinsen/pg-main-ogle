# 2.1-mobile-optimization-auditor.md

> **Layer:** 2 (Audit Pass 2)
> **Skill:** CK-03-checkout-editorial
> **Purpose:** Is mobile behavior specified for every element?
> **Outputs:** `mobile_audit` object with per-element verification and gap list

---

## FUNCTION

Audit every copy element on the checkout page for mobile behavior specification. 70%+ of checkouts happen on mobile. Every field, label, trust signal, button, progress indicator, and order summary element must have its mobile behavior defined -- how it stacks, what size it renders, what keyboard it triggers, how it fits in the mobile viewport. Elements without mobile specs are treated as audit failures.

---

## MOBILE AUDIT PROTOCOL

```
EXECUTE CK-03/2.1-mobile-optimization-auditor:
  INPUT: checkout_copy_elements (from 0.1), strategy_reference (from 0.2)
  OUTPUT: mobile_audit object

  STEP 1: AUDIT FORM FIELDS
    FOR each form field:
      CHECK:
        [ ] mobile_keyboard specified (email/numeric/tel/default)
        [ ] mobile_width specified (full/half)
        [ ] tap_target_height >= 44px (or specified)
        [ ] auto_fill attribute set (for mobile autofill)
        [ ] placeholder appropriate for mobile (not too long)

      IF any missing:
        FLAG as gap with specific missing attribute

  STEP 2: AUDIT TRUST SIGNALS
    FOR each trust signal:
      CHECK:
        [ ] mobile_display specified (inline/stacked/hidden/icon_only)
        [ ] placement accounts for mobile viewport
        [ ] icon sizing for mobile (not too small to see, not too large)
        [ ] sticky behavior defined (if applicable)

      IF any missing:
        FLAG as gap with recommendation

  STEP 3: AUDIT BUTTONS
    FOR each button:
      CHECK:
        [ ] mobile_text specified (may differ from desktop if shorter)
        [ ] tap_target sized for mobile (minimum 44px height, full width recommended)
        [ ] sticky behavior defined (for primary CTA on long forms)
        [ ] position in mobile flow specified

  STEP 4: AUDIT ORDER SUMMARY
    CHECK:
      [ ] collapsible behavior defined for mobile
      [ ] default state specified (collapsed/expanded)
      [ ] order bump visibility on mobile (always visible, never collapsed)
      [ ] total visibility on mobile (always visible)
      [ ] product image sizing for mobile (if applicable)

  STEP 5: AUDIT PROGRESS INDICATORS (if multi-step)
    CHECK:
      [ ] step labels fit mobile width (1-2 words)
      [ ] indicator is horizontally scrollable if needed
      [ ] current step visually distinct on mobile
      [ ] tap targets for step navigation

  STEP 6: COMPILE AUDIT RESULTS
    a. Count total elements checked
    b. Count elements with complete mobile spec
    c. Count elements with missing mobile spec
    d. For each gap: provide specific recommendation
    e. Calculate coverage percentage

  IF coverage < 100%:
    FAIL -- List specific gaps with remediation recommendations.
    ADD missing mobile specs where possible (annotations only, not new copy).
```

---

## MOBILE SPEC CHECKLIST REFERENCE

| Element Type | Required Mobile Specs |
|-------------|----------------------|
| Form field | keyboard, width, tap target, autofill |
| Trust signal | display mode, placement, icon size, sticky |
| Button | text, tap target, sticky, position |
| Order summary | collapsible, default state, bump visible, total visible |
| Progress indicator | label length, scrollable, current state |
| Error message | display position, dismissibility |

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Coverage below 80% | HALT -- too many gaps, escalate to CK-01/CK-02 |
| DEGRADED | Coverage 80-99% | ADD missing specs via annotation, proceed |
| DEGRADED | Sticky CTA not defined for mobile | ADD recommendation -- sticky CTA critical for mobile conversion |
| DEGRADED | Order summary not collapsible on mobile | ADD recommendation -- long summaries push form below fold |
