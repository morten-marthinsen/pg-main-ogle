# 4.2-output-packager.md

> **Layer:** 4 (Validation & Packaging)
> **Skill:** CK-03-checkout-editorial
> **Purpose:** Package checkout-copy-final.md and checkout-audit-report.md
> **Outputs:** Two files: `checkout-copy-final.md` + `checkout-audit-report.md`

---

## FUNCTION

Assemble the two final output files for the Checkout Engine. `checkout-copy-final.md` is the integration-ready checkout copy -- every word that appears on the checkout page, organized by section, with mobile specifications. `checkout-audit-report.md` is the audit trail -- every review decision, word cut, density verification, and persuasion scan result documented for reference.

---

## PACKAGING PROTOCOL

```
EXECUTE CK-03/4.2-output-packager:
  INPUT: editorial_validation (from 4.1), checkout_copy_elements (from 0.1),
         word_reduction_report (from 1.1), trust_density_review (from 1.2),
         mobile_audit (from 2.1), persuasion_scan_result (from 2.2)
  OUTPUT: checkout-copy-final.md + checkout-audit-report.md

  STEP 1: VERIFY VALIDATION PASSED
    - Confirm editorial_validation.status is APPROVED or APPROVED_WITH_ESCALATIONS
    - If REJECTED: HALT -- cannot package unvalidated copy

  STEP 2: ASSEMBLE CHECKOUT-COPY-FINAL.MD
    - Organize by checkout section (following flow map order):
      a. Order Summary section (product, price, savings, total)
      b. Order Bump integration point (placement + format, NOT U1 copy)
      c. Form Fields by section (Contact, Shipping, Payment)
         Each field: label, helper, error, placeholder, mobile spec
      d. Trust Copy by category (security, payment, guarantee, social, contact)
         Each signal: copy text, placement, mobile display
      e. Buttons (primary CTA, navigation, back)
      f. Progress Indicators (if multi-step)
      g. Success Confirmation
    - Apply all word reduction cuts from 1.1
    - Include mobile specifications inline with each element
    - Mark any client placeholders clearly

  STEP 3: ASSEMBLE CHECKOUT-AUDIT-REPORT.MD
    - Compile all audit results:
      a. Word reduction findings (cuts made, words saved)
      b. Trust density verification (per-position results)
      c. Mobile optimization results (coverage percentage, gaps)
      d. Persuasion language scan (violations found, remediation)
      e. Error message quality check
    - Include overall verdict and readiness assessment
    - List any escalation items (for CK-01 or CK-02)
    - List any client action items

  STEP 4: VALIDATE BOTH FILES
    - checkout-copy-final.md: all sections present, no empty sections,
      no placeholder text (except [CLIENT TO PROVIDE]), mobile specs inline
    - checkout-audit-report.md: all 4 audit dimensions documented,
      findings specific (not rubber-stamped), verdict present

  STEP 5: WRITE BOTH FILES
    - Write checkout-copy-final.md to project outputs directory
    - Write checkout-audit-report.md to project outputs directory
    - Verify both files written successfully
    - Log completion timestamp

  IF only one file produced:
    HALT -- Both files are mandatory output. Produce both.

  IF either file has empty sections:
    HALT -- Fill all sections before completing.
```
