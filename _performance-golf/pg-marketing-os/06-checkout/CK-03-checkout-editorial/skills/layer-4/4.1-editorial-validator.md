# 4.1-editorial-validator.md

> **Layer:** 4 (Validation)
> **Skill:** CK-03-checkout-editorial
> **Purpose:** Final pass: is checkout the shortest copy in the funnel? Do all audits pass?
> **Outputs:** `editorial_validation` object with final verdict

---

## FUNCTION

Perform the final validation pass that confirms: (1) all four audit dimensions have been completed with findings addressed, (2) checkout copy is the shortest copy in the entire marketing funnel, (3) word reduction cuts have been applied, (4) trust density is verified, (5) mobile specs are complete, (6) zero persuasion language outside order bump. This is the last gate before the checkout copy ships.

---

## VALIDATION PROTOCOL

```
EXECUTE CK-03/4.1-editorial-validator:
  INPUT: word_reduction_report (from 1.1), trust_density_review (from 1.2),
         mobile_audit (from 2.1), persuasion_scan_result (from 2.2),
         checkout_copy_elements (from 0.1)
  OUTPUT: editorial_validation object

  STEP 1: VERIFY ALL AUDITS COMPLETE
    - Word reduction audit: completed with specific findings
    - Trust density review: completed with per-position analysis
    - Mobile optimization audit: completed with coverage percentage
    - Persuasion language detector: completed with scan results
    - IF any audit incomplete: FAIL

  STEP 2: VERIFY AUDIT FINDINGS ADDRESSED
    a. Word cuts: have proposed cuts been applied?
    b. Trust density: does it pass (3+ at every position)?
       If not: has escalation to CK-01 been documented?
    c. Mobile: are gaps filled (100% coverage)?
       If not: have annotations been added?
    d. Persuasion: are violations removed (zero outside order bump)?
       If not: have replacements been specified?

  STEP 3: VERIFY SHORTEST COPY IN FUNNEL
    - Compare checkout total word count against other funnel stages
    - Checkout should be significantly shorter than:
      * VSL/sales letter (thousands of words)
      * Advertorial (hundreds of words)
      * Email sequences (hundreds of words)
      * Upsell pages (150-500 words)
    - If checkout is longer than upsell pages: CUT MORE
    - Typical checkout target: 100-200 words total (excluding order bump)

  STEP 4: VERIFY ERROR MESSAGE QUALITY
    - Confirm zero blaming error messages in final copy
    - Cross-check against CK-02's tone validation
    - If blaming message found: escalate to CK-02

  STEP 5: PRODUCE FINAL VERDICT
    - APPROVED: all audits pass, all findings addressed, checkout is shortest
    - APPROVED WITH ESCALATIONS: audits pass but CK-01/CK-02 need action
    - REJECTED: critical failures unresolved

  IF checkout is not shortest copy:
    FAIL -- "Checkout word count [X] exceeds target. Cut until shortest."
```

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Audit dimension incomplete | HALT -- complete missing audit |
| CRITICAL | Trust density fails | ESCALATE to CK-01 |
| CRITICAL | Persuasion violations unresolved | REMOVE violations |
| CRITICAL | Checkout not shortest in funnel | CUT more words |
| DEGRADED | Minor mobile gaps remain | ADD annotations, proceed |
| DEGRADED | Blaming error found | ESCALATE to CK-02 |
