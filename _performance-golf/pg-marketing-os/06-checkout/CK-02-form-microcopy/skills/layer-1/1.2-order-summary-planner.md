# 1.2-order-summary-planner.md

> **Layer:** 1 (Field Mapping)
> **Skill:** CK-02-form-microcopy
> **Purpose:** Plan order summary structure and order bump integration
> **Outputs:** `order_summary_plan` object with all elements and order bump spec

---

## FUNCTION

Define the complete structure of the order summary section -- what elements appear, in what order, with what copy needs. The order summary is the buyer's last look at what they are purchasing before entering payment details. It must be clear, reassuring, and minimal. The order bump integration point is defined here (placement, format, word budget) but the actual order bump copy is generated by U1 (Upsell Engine).

---

## ORDER SUMMARY PLANNING PROTOCOL

```
EXECUTE CK-02/1.2-order-summary-planner:
  INPUT: upstream_context (from 0.1), offer_details
  OUTPUT: order_summary_plan object

  STEP 1: DEFINE CORE ELEMENTS
    - Product name: exactly as named in offer package
    - Product description: brief reminder (5-10 words max)
    - Product image: reference (if available)
    - Quantity: default 1, adjustable or fixed
    - Price display: formatted with currency symbol
    - Compare-at price: if discounted, show original price with strikethrough
    - Savings display: "You save $XX" or "XX% off"
    - Shipping: cost or "FREE"
    - Tax: display or "Calculated at checkout"
    - Total: bold, prominent, final amount

  STEP 2: DEFINE ELEMENT ORDER
    - Product name + description + image (top)
    - Quantity (if adjustable)
    - Line item price
    - Savings callout (if applicable)
    - Shipping cost
    - Order bump (if included)
    - Subtotal or separator
    - Tax (if applicable)
    - Total (bottom, most prominent)

  STEP 3: PLAN ORDER BUMP INTEGRATION
    - Placement: after line items, before total (standard)
    - Format: checkbox with copy block
    - Visual treatment: highlighted box or border to distinguish from order items
    - Word budget: 50-150 words (U1 constraint)
    - Checkbox label structure: "Yes, add [product] to my order"
    - Price addition: show how total changes if checked
    - NOTE: CK-02 defines the CONTAINER. U1 generates the COPY.

  STEP 4: PLAN SUBSCRIPTION DISPLAY (if applicable)
    - Subscription toggle or selection
    - Billing frequency (monthly/quarterly/annual)
    - Per-unit price at each frequency
    - Savings for longer commitments
    - Cancel anytime language
    - Trial terms (if trial offer)

  STEP 5: DEFINE MOBILE SUMMARY BEHAVIOR
    - Collapsible on mobile (expand/collapse toggle)
    - Default state: collapsed with total visible, expandable for details
    - Tap to expand: shows full order summary
    - Order bump: always visible (not collapsed)
    - Total: always visible (sticky or at bottom of summary)

  IF product name missing:
    FLAG -- Use placeholder "[Product Name]" and mark for client input.

  IF pricing not available:
    HALT -- "Cannot plan order summary without pricing. Check offer package."
```

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Pricing not available | HALT -- need price for order summary |
| CRITICAL | Order bump spec missing from CK-00 | HALT -- need placement decision |
| DEGRADED | Product name unavailable | Use placeholder "[Product Name]", flag |
| DEGRADED | Subscription terms unclear | Flag for client clarification, plan basic structure |
| DEGRADED | Savings calculation not possible | Omit savings display, show price only |
