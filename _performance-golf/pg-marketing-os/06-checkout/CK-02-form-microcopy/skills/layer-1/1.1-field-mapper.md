# 1.1-field-mapper.md

> **Layer:** 1 (Field Mapping)
> **Skill:** CK-02-form-microcopy
> **Purpose:** Map all form fields needed for this checkout type
> **Outputs:** `field_map` object with every field, its type, validation rules, and mobile spec

---

## FUNCTION

Based on the funnel type and checkout pattern, map every form field the checkout requires. For each field, define: the input type, validation rules, whether it is required or optional, the mobile keyboard type to trigger, the auto-fill attribute, and the field width for mobile layout. This map becomes the specification for micro-copy generation in Layer 2.

---

## FIELD MAPPING PROTOCOL

```
EXECUTE CK-02/1.1-field-mapper:
  INPUT: upstream_context (from 0.1), validation_result (from 0.2)
  OUTPUT: field_map object

  STEP 1: DETERMINE FIELD SET BY FUNNEL TYPE
    a. Digital Product:
       - email (required)
       - full_name (required)
       - card_number (required)
       - expiry (required)
       - cvv (required)
       Total: 5 fields

    b. Physical Product / Supplement:
       - email (required)
       - full_name (required)
       - address_line_1 (required)
       - address_line_2 (optional)
       - city (required)
       - state (required -- dropdown)
       - zip (required)
       - card_number (required)
       - expiry (required)
       - cvv (required)
       Total: 10 fields (1 optional)

    c. Free + Shipping:
       - Same as physical but with S&H price emphasis
       Total: 10 fields

    d. Trial:
       - Same as digital or physical (depending on product)
       - Plus: trial_terms acknowledgment (checkbox)
       Total: 6-11 fields

  STEP 2: MAP FIELD ATTRIBUTES
    FOR each field:
      - field_id: unique identifier
      - field_type: text/email/tel/number/select/checkbox
      - required: true/false
      - validation_rule: specific format requirement
      - max_length: character limit (if applicable)
      - mobile_keyboard: default/email/tel/numeric
      - auto_fill: autocomplete attribute (email/name/address-line1/cc-number/etc.)
      - mobile_width: full/half (for side-by-side layout)
      - section: which checkout section this field belongs to

  STEP 3: EVALUATE FIELD NECESSITY
    - Flag any field that is not strictly necessary for fulfillment or payment
    - Common unnecessary fields: phone (unless for delivery), company name, how did you hear about us
    - For each flagged field: recommend removal or make optional
    - Reference friction analysis from CK-00

  STEP 4: DEFINE FIELD GROUPINGS
    - Group fields by section: Contact, Shipping, Payment
    - Define visual grouping for multi-step (which fields per step)
    - Define stacking order for mobile (vertical sequence)

  IF funnel type unknown:
    HALT -- "Cannot determine fields without funnel type."
```

## FIELD REFERENCE TABLE

| Field | Type | Keyboard | Auto-Fill | Width | Required |
|-------|------|----------|-----------|-------|----------|
| Email | email | email | email | full | Yes |
| Full Name | text | default | name | full | Yes |
| Address Line 1 | text | default | address-line1 | full | Yes (physical) |
| Address Line 2 | text | default | address-line2 | full | No |
| City | text | default | address-level2 | half | Yes (physical) |
| State | select | N/A | address-level1 | half | Yes (physical) |
| Zip Code | text | numeric | postal-code | half | Yes (physical) |
| Card Number | text | numeric | cc-number | full | Yes |
| Expiry | text | numeric | cc-exp | half | Yes |
| CVV | text | numeric | cc-csc | half | Yes |

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Funnel type unknown -- cannot determine field set | HALT -- return to CK-00 |
| CRITICAL | Payment fields not included | HALT -- checkout must have payment |
| DEGRADED | Unnecessary field included | FLAG for removal, proceed with field mapped |
| DEGRADED | Mobile keyboard type unknown for custom field | Default to "text" keyboard, flag for review |
