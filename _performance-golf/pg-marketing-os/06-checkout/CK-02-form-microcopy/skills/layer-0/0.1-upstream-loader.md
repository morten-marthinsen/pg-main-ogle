# 0.1-upstream-loader.md

> **Layer:** 0 (Foundation)
> **Skill:** CK-02-form-microcopy
> **Purpose:** Load checkout strategy and checkout flow map from CK-00
> **Outputs:** `upstream_context` object with checkout pattern, flow map, and field requirements

---

## FUNCTION

Load the checkout-strategy.yaml output from CK-00 and extract the checkout pattern (single-page or multi-step), flow map (section sequence), field requirements, and order bump specification. This provides the structural blueprint that field mapping and micro-copy generation execute against.

---

## LOADING PROTOCOL

```
EXECUTE CK-02/0.1-upstream-loader:
  INPUT: CK-00 output directory, offer-package output directory
  OUTPUT: upstream_context object

  STEP 1: LOAD CHECKOUT STRATEGY
    - Locate checkout-strategy.yaml from CK-00 outputs
    - Parse YAML and validate structure
    - Extract: checkout_pattern (single_page/multi_step, step count, steps)
    - Extract: flow_map (sections in order with copy elements)
    - Extract: funnel_classification (for field requirement determination)
    - Extract: order_bump_spec (placement, format, word budget)

  STEP 2: LOAD OFFER PACKAGE
    - Locate offer-package.json
    - Extract: product name, pricing, savings, shipping details
    - Extract: order bump product details (if applicable)
    - These inform order summary copy generation

  STEP 3: LOAD FRICTION ANALYSIS
    - From strategy: friction_mitigation_plan
    - Filter for CK-02-assigned friction points
    - These inform which fields to optimize or eliminate

  STEP 4: ASSEMBLE CONTEXT
    - Combine checkout pattern, flow map, field requirements,
      offer details, and CK-02 friction assignments
    - Set loading status

  IF checkout-strategy.yaml missing:
    HALT -- "Cannot generate micro-copy without checkout strategy. Run CK-00 first."
```

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | checkout-strategy.yaml missing | HALT -- run CK-00 first |
| CRITICAL | Checkout pattern undefined | HALT -- return to CK-00 |
| CRITICAL | Flow map absent | HALT -- return to CK-00 |
| DEGRADED | Offer package partially loaded | Proceed with available data, flag gaps |
| DEGRADED | Friction assignments missing | Proceed without optimization targets |
