# 0.2-input-validator.md

> **Layer:** 0 (Foundation)
> **Skill:** CK-02-form-microcopy
> **Purpose:** Validate checkout pattern and field requirements present
> **Outputs:** `validation_result` object with pass/fail and gap report

---

## FUNCTION

Validate that the loaded upstream context contains a defined checkout pattern (single-page or multi-step), a flow map with section sequence, and enough information to determine which form fields are needed. This gate prevents micro-copy generation from starting without structural clarity.

---

## VALIDATION PROTOCOL

```
EXECUTE CK-02/0.2-input-validator:
  INPUT: upstream_context (from 0.1)
  OUTPUT: validation_result object

  STEP 1: VALIDATE CHECKOUT PATTERN
    - Pattern is defined (single_page or multi_step)
    - If multi_step: step count and step labels defined
    - Step structure includes field assignments (or field requirements)

  STEP 2: VALIDATE FLOW MAP
    - Flow map present with sections in order
    - Each section has copy_elements listed
    - No gaps in section sequence

  STEP 3: VALIDATE FIELD DETERMINABILITY
    - Funnel type allows field list determination:
      - Digital: email, name, card, expiry, CVV
      - Physical: + address, city, state, zip
      - Subscription: + subscription terms
    - Product type + funnel type sufficient to determine required fields

  STEP 4: VALIDATE ORDER SUMMARY DATA
    - Product name available
    - Price available
    - Order bump spec present (even if "not included")

  STEP 5: PRODUCE VERDICT
    a. PASS: pattern defined, flow map present, fields determinable
    b. FAIL: specific gaps with remediation

  IF pattern undefined:
    FAIL -- "Checkout pattern must be defined (single_page or multi_step)."
```

---

## OUTPUT STRUCTURE

```json
{
  "validation_result": {
    "status": "PASS|FAIL",
    "checks": {
      "pattern_defined": true,
      "flow_map_present": true,
      "fields_determinable": true,
      "order_summary_data": true,
      "order_bump_spec": true
    },
    "critical_gaps": [],
    "warnings": [],
    "estimated_field_count": 7,
    "proceed": true
  }
}
```

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Checkout pattern undefined | HALT -- return to CK-00 |
| CRITICAL | Flow map missing | HALT -- return to CK-00 |
| CRITICAL | Cannot determine required fields | HALT -- need funnel type + product type |
| DEGRADED | Order bump spec missing | WARN -- proceed without, flag for CK-00 |
| DEGRADED | Product name not in offer package | WARN -- use placeholder |
