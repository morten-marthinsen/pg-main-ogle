# 2.1-microcopy-generator.md

> **Layer:** 2 (Generation)
> **Skill:** CK-02-form-microcopy
> **Purpose:** Generate labels + helper text + error messages + progress indicators + button text
> **Outputs:** `microcopy_set` object with complete copy for every form element

---

## FUNCTION

Generate all micro-copy for the checkout form. This is precision UX writing -- every label must be instantly understood, every error message must guide toward correction, every button must communicate the exact action. Micro-copy is the invisible infrastructure that determines whether a buyer completes checkout or abandons in frustration.

---

## GENERATION PROTOCOL

```
EXECUTE CK-02/2.1-microcopy-generator:
  INPUT: field_map (from 1.1), order_summary_plan (from 1.2),
         checkout_pattern (from upstream)
  OUTPUT: microcopy_set object

  STEP 1: GENERATE FIELD LABELS
    FOR each field in field_map:
      - Write label: 1-3 words, instantly clear
      - Examples:
        email -> "Email address"
        full_name -> "Full name"
        card_number -> "Card number"
        address_1 -> "Street address"
        cvv -> "CVV"
        zip -> "Zip code"
      - RULE: Label must be unambiguous without helper text

  STEP 2: GENERATE HELPER TEXT (selective)
    FOR each field in field_map:
      - EVALUATE: does this field need helper text?
      - YES if: field purpose is not obvious from label alone
      - NO if: label is self-explanatory (email, zip code)
      - Examples of needed helper text:
        full_name -> "As it appears on your card"
        cvv -> "3 digits on back of card"
        email -> "For order confirmation and receipt"
      - RULE: If in doubt, skip. Unnecessary helper text = friction.

  STEP 3: GENERATE ERROR MESSAGES
    FOR each required field in field_map:
      - Write error message that GUIDES, never BLAMES
      - Include: what is wrong + what correct format looks like
      - BANNED patterns: "Invalid [field]", "Wrong [field]", "Error"
      - REQUIRED pattern: "Please [action] [specific format]"
      - Examples:
        email -> "Please enter a valid email address"
        card_number -> "Please check your card number -- it should be 16 digits"
        expiry -> "Please enter a future date (MM/YY)"
        cvv -> "Please enter the 3-digit code on the back of your card"
        zip -> "Please enter your 5-digit zip code"
        full_name -> "Please enter your first and last name"
        address_1 -> "Please enter your street address"

  STEP 4: GENERATE PLACEHOLDER TEXT (selective)
    FOR each field:
      - EVALUATE: does a placeholder help or hurt?
      - YES if: shows expected format (email: "you@example.com")
      - NO if: label is sufficient and placeholder would be redundant
      - RULE: Placeholder disappears on focus -- never use it as the only label

  STEP 5: GENERATE PROGRESS INDICATORS (if multi-step)
    IF checkout_pattern == multi_step:
      - Generate step labels: 1-2 words each
      - Examples: "Contact" -> "Shipping" -> "Payment"
      - Generate aria labels for accessibility
      - Define visual states: upcoming/current/completed
      - Completed step indicator: checkmark

  STEP 6: GENERATE BUTTON TEXT
    - Primary CTA: specific action, not vague
      GOOD: "Complete Order", "Place Order", "Complete Purchase"
      BAD: "Submit", "Go", "Continue", "Click Here"
    - Secondary CTA (multi-step): contextual navigation
      GOOD: "Continue to Payment", "Continue to Shipping"
      BAD: "Next", "Continue"
    - Back button: simple
      GOOD: "Back"

  STEP 7: GENERATE ORDER SUMMARY COPY
    - Product name: from offer package (exact)
    - Product description: 5-10 word benefit reminder (informational, not persuasion)
    - Price labels: "Price", "Shipping", "Total"
    - Savings copy: "You save $XX" or "XX% off"
    - Order bump checkbox label: "Yes, add [product] to my order"
    - NOTE: Order bump persuasion copy comes from U1. Do not write it here.

  STEP 8: GENERATE SUCCESS CONFIRMATION
    - Heading: "Order Confirmed" or "Thank You"
    - Message: "Check your email for order details and next steps."
    - Next step: clear direction for what happens next

  STEP 9: ERROR MESSAGE TONE CHECK
    a. Scan ALL error messages for blaming language
    b. Check against banned patterns (Invalid, Wrong, Bad, Error alone)
    c. If any found: REWRITE immediately
    d. Do not proceed with blaming messages in output

  IF blaming error message detected:
    REWRITE before proceeding. Blaming messages are never acceptable.
```

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Blaming error message generated | REWRITE immediately -- never ship blaming copy |
| CRITICAL | Field missing error message | GENERATE -- every required field needs one |
| CRITICAL | Button text is "Submit" or "Go" | REWRITE -- use specific action text |
| DEGRADED | Helper text added to every field | REMOVE unnecessary helper text -- only where needed |
| DEGRADED | Placeholder used as only label | ADD proper label -- placeholder is supplementary only |
| DEGRADED | Persuasion language in order summary | REMOVE -- order summary is informational |
