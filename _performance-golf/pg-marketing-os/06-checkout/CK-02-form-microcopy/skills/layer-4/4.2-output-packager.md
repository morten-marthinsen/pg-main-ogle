# 4.2-output-packager.md

> **Layer:** 4 (Validation & Packaging)
> **Skill:** CK-02-form-microcopy
> **Purpose:** Package checkout-microcopy-package.json as final output
> **Outputs:** `checkout-microcopy-package.json` file

---

## FUNCTION

Assemble the validated micro-copy into the final `checkout-microcopy-package.json` output file. This is the canonical deliverable that CK-03 (Checkout Editorial) consumes for review and integration. The package must include all field copy, order summary copy, progress indicators, button text, success confirmation, and error messages -- all validated for tone and completeness.

---

## PACKAGING PROTOCOL

```
EXECUTE CK-02/4.2-output-packager:
  INPUT: microcopy_set (from 2.1), tone_check_result (from 4.1)
  OUTPUT: checkout-microcopy-package.json file

  STEP 1: VERIFY VALIDATION PASSED
    - Confirm tone_check_result.status == PASS
    - Confirm zero blaming messages
    - If FAIL: HALT -- cannot package micro-copy with blaming errors

  STEP 2: ASSEMBLE JSON STRUCTURE
    - Header: version, timestamp, skill_id
    - Form fields array (each with label, helper, error, placeholder)
    - Order summary section (all elements)
    - Progress indicators (if multi-step)
    - Buttons (primary CTA, navigation, back)
    - Success confirmation
    - Copy metadata (word counts, field counts, validation status)

  STEP 3: VALIDATE SCHEMA
    - All required fields have error messages
    - No empty labels
    - No "Submit" or "Go" button text
    - Mobile specifications present for every field
    - Order bump integration defined (not the copy itself)
    - No persuasion language in any copy element

  STEP 4: WRITE FILE
    - Write checkout-microcopy-package.json to project outputs directory
    - Verify file written successfully
    - Log completion timestamp

  STEP 5: CREATE HANDOFF SUMMARY
    - Document what CK-03 receives
    - List total fields, word count, and validation status
    - Flag any elements that need client input

  IF schema validation fails:
    HALT -- Fix violations before writing.
```

---

## OUTPUT STRUCTURE

The output IS the `checkout-microcopy-package.json` file containing the full schema from the CK-02-AGENT.md output schema section.

Additionally, create a verification object:

```json
{
  "package_assembly": {
    "status": "complete",
    "file_written": "checkout-microcopy-package.json",
    "file_path": "/path/to/file",
    "schema_version": "1.0",
    "validation_passed": true,
    "tone_check_passed": true,
    "blaming_messages": 0,
    "total_fields": 7,
    "total_word_count": 95,
    "written_at": "ISO timestamp"
  }
}
```

## VALIDATION CHECKLIST

```
[ ] checkout-microcopy-package.json EXISTS
[ ] File is valid JSON (parseable)
[ ] All form fields have: label + error message
[ ] Zero blaming error messages
[ ] Button text is specific (not "Submit")
[ ] Order summary complete
[ ] Order bump integration defined (placement + format)
[ ] Progress indicators present (if multi-step)
[ ] Mobile specs for every field
[ ] No persuasion language anywhere
[ ] Success confirmation present
```

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Tone check failed | HALT -- return to 4.1, fix messages |
| CRITICAL | Field missing error message | HALT -- add error message |
| CRITICAL | Schema validation fails | Fix specific violations |
| DEGRADED | Client placeholders present | Package with placeholders, flag in handoff |
