# 2.1-strategy-assembler.md

> **Layer:** 2 (Assembly)
> **Skill:** EC-00-ecomm-strategist
> **Purpose:** Build section map + priority ranking + word budgets
> **Outputs:** `assembled_strategy` object with complete section-level detail

---

## FUNCTION

Assemble the complete e-commerce strategy by combining page type classification, section map, and crossover map into a unified document with priority rankings, word budgets per section, and feature research scope for EC-01. This is the core assembly step that produces the strategic blueprint.

---

## ASSEMBLY PROTOCOL

### Step 1: Priority Ranking

```
FOR each active_section in section_map:
  RANK by conversion_impact (from section mapper):

  PRIORITY FRAMEWORK:
    Priority 1-3: "Must-convert" sections
      - Hero (always priority 1)
      - Primary CTA
      - Highest-impact proof section

    Priority 4-8: "Core story" sections
      - Product highlights / features
      - How it works / mechanism
      - Problem/solution
      - Ingredients/materials
      - Key proof (reviews, UGC)

    Priority 9-14: "Supporting" sections
      - FAQ
      - Comparison chart
      - Cost savings
      - How to use
      - What to expect
      - Use cases

    Priority 15+: "Supplementary" sections
      - Awards/press
      - Additional proof
      - Secondary CTAs

  ASSIGN priority number to each section (1 = highest)
  NO duplicate priority numbers
```

### Step 2: Word Budget Allocation

```
DETERMINE total_page_budget based on page_type:
  PDP: 2500-4000 words typical
  Collection: 1500-2500 words typical
  Bundle: 2000-3500 words typical

ALLOCATE word_budget per section:

  ALLOCATION RULES:
    Hero (ATF): 100-200 words (headline + subhead + value prop + highlights)
    CTA sections: 50-100 words (offer + urgency + button)
    Feature sections: 150-400 words (depends on feature count)
    Proof sections: 100-300 words (review quotes + context)
    Problem/Solution: 150-300 words (condensed pain + product answer)
    Mechanism/Technology: 150-350 words (how it works explanation)
    FAQ: 200-500 words (5-10 Q&A pairs, 20-50 words each)
    Other sections: 100-250 words

  CONSTRAINTS:
    No section under 50 words
    No section over 500 words
    Total must be within page_type budget range

  IF total exceeds budget:
    REDUCE lowest-priority sections first
    NEVER reduce Priority 1-3 sections

  IF total under budget:
    INCREASE highest-impact sections that would benefit from depth
```

### Step 3: Feature Research Scope

```
FROM upstream_context.product.features:

  CLASSIFY each feature into hierarchy tier:
    hero_feature_candidates:
      - Features that could lead the product story
      - Criteria: unique, differentiated, benefit-clear, customer-facing
      - Minimum 1, maximum 3 candidates

    supporting_feature_candidates:
      - Features that reinforce the hero
      - Criteria: meaningful benefit, adds depth, not redundant with hero
      - Minimum 2

    technical_features:
      - Specs, materials, dimensions, certifications
      - No minimum (may have none)

  IDENTIFY research_gaps:
    - Features mentioned but not described
    - Ingredients/materials without sourcing info
    - Technology claims without proof points
    - Competitive differentiators not yet identified

  BUILD feature_research_scope for EC-01 handoff
```

### Step 4: Design Notes Seeding

```
FOR each active_section:
  SEED design_notes:
    layout_suggestion: "Full-width" | "Two-column" | "Card grid" | "Accordion" | etc
    mobile_behavior: "Stack vertically" | "Carousel swipe" | "Collapsible" | etc
    visual_elements: "Product image" | "Icon set" | "Before/after" | "Chart" | etc
    spacing_priority: "Breathing room" | "Compact" | "Full bleed"

  NOTE: These are SEEDS, not final design specs. Page builder (04-page-builder) makes final design decisions.
```

### Step 5: Downstream Handoff Assembly

```
BUILD downstream_handoffs:
  ec_01: feature_research_scope (hero candidates, supporting, technical, research gaps)
  ec_02: section_map[hero] + feature_research_scope[hero_candidates] + word_budget[hero]
  ec_03: section_map[all_btf_sections] + crossover_map + word_budgets[btf]
  ec_04: section_map + feature_research_scope (for script mapping)
  ec_05: section_map[priority_order] + word_budget_summary + design_notes

VERIFY each handoff has sufficient data for the downstream skill
```

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Total word budget outside bounds | Adjust section budgets or section count |
| CRITICAL | No hero feature candidates identified | Review product features, expand scope |
| WARNING | Feature research gaps > 3 | Flag for human review -- may need product research |
| WARNING | Multiple sections at minimum (50w) budget | Consider consolidating or removing low-value sections |
| INFO | Design notes sparse | Acceptable -- page builder will elaborate |
