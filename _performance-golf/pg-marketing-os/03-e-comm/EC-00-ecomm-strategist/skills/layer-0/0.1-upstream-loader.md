# 0.1-upstream-loader.md

> **Layer:** 0 (Foundation)
> **Skill:** EC-00-ecomm-strategist
> **Purpose:** Load campaign brief (Mode A) or standalone brief (Mode B)
> **Outputs:** `upstream_context` object for downstream skills

---

## FUNCTION

Load and parse the upstream product brief that provides the raw material for e-commerce strategy. Supports two modes: Mode A loads from campaign-brief-package.json (downstream from Skills 01-09), Mode B loads from a standalone product brief when the full pipeline has not been executed.

---

## MODE DETECTION PROTOCOL

### Step 1: Determine Source Mode

```
CHECK for campaign-brief-package.json:
  IF exists AND contains product_details:
    SET source_mode = "campaign_brief" (Mode A)
    LOAD campaign-brief-package.json

  IF not_exists OR missing product_details:
    CHECK for standalone product brief
    IF standalone brief provided:
      SET source_mode = "standalone_brief" (Mode B)
      LOAD standalone brief
    ELSE:
      FAIL -- "No product brief available. Provide campaign brief (Skill 09) or standalone product brief."
```

### Step 2: Extract Product Data (Mode A)

```
FROM campaign-brief-package.json EXTRACT:
  product_name: campaign_brief.product.name
  product_category: campaign_brief.product.category
  product_features: campaign_brief.product.features[]
  target_audience: campaign_brief.avatar.primary
  mechanism: campaign_brief.mechanism.name
  promise: campaign_brief.promise.primary
  proof_assets: campaign_brief.proof[]
  competitive_context: campaign_brief.competitive_landscape
  brand_voice: campaign_brief.creative_direction.tone
```

### Step 3: Extract Product Data (Mode B)

```
FROM standalone brief EXTRACT:
  product_name: [required]
  product_category: [required]
  product_features: [required, minimum 3]
  target_audience: [required]
  mechanism: [optional, if known]
  promise: [optional, if known]
  proof_assets: [optional]
  competitive_context: [optional]
  brand_voice: [optional]
```

### Step 4: Assemble Upstream Context

```
BUILD upstream_context:
  product: { name, category, features[], audience }
  strategic: { mechanism, promise, competitive_context }
  creative: { brand_voice, proof_assets }
  metadata: { source_mode, completeness_score }
```

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | No brief available (neither Mode A nor Mode B) | HALT -- request Skill 09 execution or standalone brief |
| CRITICAL | Product name missing | HALT -- cannot proceed without product identity |
| CRITICAL | Fewer than 3 features identifiable | HALT -- insufficient data for section mapping |
| DEGRADED | Mechanism not defined | Proceed, flag for EC-01 to investigate |
| DEGRADED | Proof assets empty | Proceed, flag proof density risk for EC-03 |
| DEGRADED | Competitive context missing | Proceed without comparison section recommendation |
