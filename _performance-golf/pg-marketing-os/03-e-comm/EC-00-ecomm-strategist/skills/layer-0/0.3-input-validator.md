# 0.3-input-validator.md

> **Layer:** 0 (Foundation)
> **Skill:** EC-00-ecomm-strategist
> **Purpose:** Validate product details present and sufficient for strategy creation
> **Outputs:** `validation_result` object with PASS/FAIL and detail flags

---

## FUNCTION

Validate that the loaded upstream context contains sufficient product data to produce a meaningful e-commerce strategy. This is the gatekeeper for Layer 0 -- if validation fails, the pipeline halts before classification begins.

---

## VALIDATION PROTOCOL

### Step 1: Required Field Validation

```
CHECK upstream_context for REQUIRED fields:

  product.name:
    EXISTS: [Y/N]
    NON_EMPTY: [Y/N]
    IF missing: FAIL -- "Product name is required"

  product.category:
    EXISTS: [Y/N]
    RECOGNIZED_CATEGORY: [Y/N]
    IF missing: FAIL -- "Product category is required for page type classification"

  product.features:
    EXISTS: [Y/N]
    COUNT: [number]
    IF count < 3: FAIL -- "Minimum 3 identifiable features required for section mapping"

  product.audience:
    EXISTS: [Y/N]
    NON_EMPTY: [Y/N]
    IF missing: WARN -- "Audience missing. Strategy will lack targeting precision."
```

### Step 2: Data Quality Assessment

```
FOR each feature in product.features:
  ASSESS:
    has_name: [Y/N]
    has_description_or_function: [Y/N]
    is_differentiating: [Y/N]

  SCORE feature_data_quality:
    3/3 = HIGH -- feature fully defined
    2/3 = MODERATE -- feature partially defined
    1/3 = LOW -- feature barely defined
    0/3 = INVALID -- not a real feature

AGGREGATE feature_quality_score = average across all features
IF feature_quality_score < 1.5: WARN -- "Feature data quality low. EC-01 will need significant research."
```

### Step 3: Completeness Score

```
CALCULATE completeness_score:
  product_name: 15%
  product_category: 15%
  features_count: 20% (scale: 3=50%, 5=75%, 8+=100%)
  feature_quality: 15%
  audience: 10%
  mechanism: 10%
  proof_assets: 10%
  competitive_context: 5%

  total = weighted sum

  IF total >= 0.75: PASS -- "Sufficient data for strategy"
  IF total 0.50-0.74: PASS_WITH_FLAGS -- "Proceed with documented gaps"
  IF total < 0.50: FAIL -- "Insufficient data. Request more product information."
```

### Step 4: Downstream Impact Assessment

```
FOR each missing/weak field:
  ASSESS downstream impact:
    mechanism_missing -> EC-01 must investigate from scratch
    proof_assets_empty -> EC-03 proof density at risk
    audience_missing -> EC-02 hero targeting weakened
    competitive_context_missing -> Section 11 (Comparison) may be excluded

  LOG downstream_impacts for strategy assembly awareness
```

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Product name missing | HALT -- cannot create strategy without product identity |
| CRITICAL | Product category missing | HALT -- page type classification requires category |
| CRITICAL | Fewer than 3 features | HALT -- request more product data |
| DEGRADED | Completeness score 0.50-0.74 | Proceed with flags, document gaps |
| DEGRADED | Feature quality score < 1.5 | Proceed, flag EC-01 for deep feature research |
| INFO | Optional fields missing | Log downstream impact, continue |
