# 1.2-section-mapper.md

> **Layer:** 1 (Classification)
> **Skill:** EC-00-ecomm-strategist
> **Purpose:** Map which of 19 NLS sections apply to this product
> **Outputs:** `section_map` object with active and excluded sections

---

## FUNCTION

Using the page type classification and loaded NLS framework, determine which of the 19 NLS PDP sections should be active for this specific product. Evaluate each section against product data availability, audience needs, and conversion priority.

---

## MAPPING PROTOCOL

### Step 1: Apply Page Type Filter

```
LOAD page_type_filters from nls_framework
APPLY filter for classified page_type:

  IF pdp: all 19 sections are candidates
  IF collection: sections [1, 2, 3, 4, 5, 9, 11, 14, 15, 16, 18] are primary candidates
  IF bundle: sections [1, 2, 3, 4, 5, 7, 9, 11, 12, 14, 15, 16, 17] are primary candidates

RESULT: candidate_sections[] = filtered list
```

### Step 2: Evaluate Each Candidate Section

```
FOR each section in candidate_sections:
  EVALUATE against 4 criteria:

  1. data_available:
     Does the product brief contain data to populate this section?
     Examples:
       Section 6 (Expert Credibility) -> Is there an expert/authority figure?
       Section 7 (Ingredients) -> Are ingredients/materials documented?
       Section 19 (Awards/Press) -> Are there press mentions or awards?

  2. audience_relevance:
     Does the target audience need this section?
     Examples:
       Section 12 (Cost Savings) -> Is price sensitivity high?
       Section 8 (What to Expect) -> Is the product new/unfamiliar?

  3. conversion_impact:
     How much does this section contribute to purchase decision?
     HIGH: Directly addresses purchase objection or desire
     MEDIUM: Supports purchase decision
     LOW: Nice to have but not critical

  4. competitive_differentiation:
     Does this section help differentiate from competitors?

  DECISION:
    IF data_available AND (audience_relevance >= MEDIUM OR conversion_impact >= HIGH):
      STATUS = ACTIVE
    ELSE:
      STATUS = EXCLUDED with reason
```

### Step 3: Enforce Minimum Thresholds

```
COUNT active_sections

IF active_sections < 8:
  REVIEW excluded sections in order of conversion_impact
  RE-ACTIVATE highest-impact excluded sections until count >= 8
  FLAG re-activated sections: "Activated to meet minimum. May need additional data."

IF active_sections > 17:
  REVIEW lowest-impact active sections
  RECOMMEND exclusions: "Consider excluding Section X (low impact, limited data)"
  NOTE: Do not auto-exclude -- flag for human review

MANDATORY active sections (never exclude):
  Section 1: Hero Carousel
  Section 3 OR 16: At least one CTA section
  At least one proof section (5, 14, or 19)
```

### Step 4: Document Exclusion Rationale

```
FOR each excluded section:
  DOCUMENT:
    section_number: [N]
    section_name: [name]
    exclusion_reason: [specific reason]
    data_gap: [what data would be needed to include]
    reactivation_trigger: [condition that would change decision]
```

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Fewer than 8 sections activatable | Request more product data or re-evaluate exclusion criteria |
| CRITICAL | Hero section data missing | HALT -- hero section is mandatory, need product imagery context |
| WARNING | No proof sections activatable | WARN -- activate Section 14 (Reviews) with placeholder, flag data gap |
| WARNING | More than 17 sections active | Flag for human review -- risk of page bloat |
| EDGE CASE | Data supports section but audience doesn't need it | Exclude with documented rationale |
