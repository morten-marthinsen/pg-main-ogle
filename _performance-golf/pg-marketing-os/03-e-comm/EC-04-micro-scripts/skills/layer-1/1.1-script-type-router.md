# 1.1-script-type-router.md

> **Layer:** 1 (Script Architecture)
> **Skill:** EC-04-micro-scripts
> **Purpose:** Assign script type per section
> **Outputs:** `script_assignments` object with finalized type per section

---

## FUNCTION

Finalize the script type assignment for each eligible section, adjusting for product-specific needs. Some sections may need a different script type than the default, and some may not need a script at all.

---

## ROUTING PROTOCOL

### Step 1: Review Default Assignments
```
FROM script_mapping (0.2):
  FOR each assignment:
    EVALUATE:
      [ ] Is this the best script type for this product?
      [ ] Is there enough content to fill the script?
      [ ] Does the section warrant a video element?

    ADJUST if needed:
      IF section has insufficient data for script: MARK as "skip"
      IF different script type better fits: REASSIGN
```

### Step 2: Prioritize Scripts
```
IF total planned scripts > 6:
  PRIORITIZE by:
    1. Hero video (always included)
    2. Feature explainer for hero feature (always included)
    3. Remaining by section priority order
  CAP at 6-8 scripts for a single product page
```

### Step 3: Confirm Assignments
```
BUILD final script_assignments:
  Each entry: section, script_type, priority, content_source
```

---

## OUTPUT STRUCTURE

```json
{
  "script_assignments": [
    {
      "section_number": 1,
      "script_type": "hero_video",
      "priority": 1,
      "status": "assigned"
    }
  ],
  "total_assigned": 6,
  "skipped_sections": [15, 16, 17]
}
```

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| WARNING | Zero scripts assigned | At minimum, hero video should exist |
| INFO | All assignments confirmed | Proceed to duration budgeting |
