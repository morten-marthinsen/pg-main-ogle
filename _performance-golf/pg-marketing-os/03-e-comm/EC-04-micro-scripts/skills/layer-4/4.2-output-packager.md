# 4.2-output-packager.md

> **Layer:** 4 (Validation & Packaging)
> **Skill:** EC-04-micro-scripts
> **Purpose:** Package micro-scripts.json
> **Outputs:** `micro-scripts.json` file in project output directory

---

## FUNCTION

Package all validated scripts with narration, visual direction, and section mapping into `micro-scripts.json`.

---

## PACKAGING PROTOCOL

### Step 1: Verify Validation Passed
```
CHECK validation_result.overall_status: MUST be PASS
IF FAIL: HALT
```

### Step 2: Assemble Package
```
BUILD micro-scripts.json:
  header: version, timestamp, skill_id
  scripts: [array of all scripts with narration + visual direction]
  summary: total_scripts, total_duration, total_words
  section_mapping: [which scripts go where on the page]
```

### Step 3: Write and Verify
```
WRITE micro-scripts.json to:
  ~outputs/[project-code]/e-comm/micro-scripts.json

VERIFY:
  [ ] Valid JSON
  [ ] All scripts present
  [ ] Visual directions included
  [ ] Section mappings accurate
  [ ] Feature names consistent
```

---

## OUTPUT STRUCTURE

```json
{
  "packaging_result": {
    "status": "complete",
    "file_written": "micro-scripts.json",
    "total_scripts": 6,
    "total_duration_seconds": 195,
    "packaged_at": "ISO timestamp"
  }
}
```

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Validation not passed | Return to validator |
| CRITICAL | File write fails | Check permissions |
