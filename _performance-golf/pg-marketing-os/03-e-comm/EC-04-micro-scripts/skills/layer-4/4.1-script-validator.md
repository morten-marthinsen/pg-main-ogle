# 4.1-script-validator.md

> **Layer:** 4 (Validation)
> **Skill:** EC-04-micro-scripts
> **Purpose:** Validate duration + feature consistency
> **Outputs:** `validation_result` object with PASS/FAIL per script

---

## FUNCTION

Validate every script for duration compliance, feature name consistency, visual direction completeness, and section mapping accuracy.

---

## VALIDATION PROTOCOL

### Step 1: Duration Validation
```
FOR each script:
  CHECK word_count against duration budget:
    max_words = duration_seconds * 2.5 * 1.1
    IF word_count > max_words: FAIL -- "Script too long for [X]s duration"
    IF duration > 60: FAIL -- "Maximum 60 seconds"
    IF duration < 15: FAIL -- "Minimum 15 seconds"
```

### Step 2: Feature Consistency
```
FOR each script:
  CHECK all feature name references against EC-01 package:
    [ ] Exact name match (case-sensitive)
    [ ] No paraphrasing
    [ ] No invented feature names
  IF violation: FAIL with specific fix guidance
```

### Step 3: Visual Direction Completeness
```
FOR each script:
  CHECK visual_directions:
    [ ] Every timestamp covered
    [ ] No timeline gaps
    [ ] Shots align with narration
    [ ] CTA has dedicated visual
  IF incomplete: FAIL with missing elements
```

### Step 4: Section Mapping Accuracy
```
FOR each script:
  CHECK:
    [ ] Maps to valid section number
    [ ] Section exists in EC-00 strategy
    [ ] Script content matches section purpose
```

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Duration violation | Compress narration or adjust budget |
| CRITICAL | Feature name wrong | Correct to EC-01 name |
| WARNING | Visual direction thin | Add specific shot descriptions |
