# 0.1-upstream-loader.md

> **Layer:** 0 (Foundation)
> **Skill:** EC-04-micro-scripts
> **Purpose:** Load strategy + feature package + section copy
> **Outputs:** `upstream_context` object with all inputs for script writing

---

## FUNCTION

Load the strategy (section map), feature package (named features with micro-scripts), and section copy (written BTF sections) that provide context for micro-script generation.

---

## LOADING PROTOCOL

### Step 1: Load Strategy
```
LOCATE: ecomm-strategy.yaml
EXTRACT: active_sections, section priorities, page_type
```

### Step 2: Load Feature Package
```
LOCATE: feature-package.json
EXTRACT: all features with names and micro-scripts (script seeds)
```

### Step 3: Load Section Copy
```
LOCATE: section-copy-package.json
EXTRACT: section copy per section (context for script content)
```

### Step 4: Identify Script-Eligible Sections
```
NOT every section needs a micro-script.
ELIGIBLE sections: Hero (1), Feature sections (4, 7, 10), UGC (5),
  How to Use (13), Comparison (11), Problem/Solution (9)
NON-ELIGIBLE: FAQ (15), Guarantee (17), Offer/Pricing (16), Awards (19)
```

---

## OUTPUT STRUCTURE

```json
{
  "upstream_context": {
    "script_eligible_sections": [1, 4, 5, 7, 9, 10, 11, 13],
    "features": [],
    "section_copy_available": true,
    "loading_status": "complete"
  }
}
```

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Strategy or features missing | HALT -- execute EC-00/EC-01 |
| DEGRADED | Section copy not yet available | Proceed using feature micro-scripts as content seeds |
