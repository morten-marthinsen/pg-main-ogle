# 4.1-generic-name-detector.md

> **Layer:** 4 (Validation)
> **Skill:** EC-01-feature-naming
> **Purpose:** Reject generic names ("Advanced Technology")
> **Outputs:** `detection_result` object with PASS/FAIL per feature name

---

## FUNCTION

Final-pass detection of generic feature names that survived the Layer 2 kill list. This is a more thorough check using pattern matching, semantic analysis, and competitive comparison. Zero generic names may reach the output package.

---

## DETECTION PROTOCOL

### Step 1: Pattern-Based Detection

```
FOR each feature name:
  CHECK against expanded generic patterns:

  GENERIC PREFIXES:
    Advanced, Premium, Superior, Professional, Ultra, Pro, Next-Gen,
    State-of-the-Art, Best-in-Class, Cutting-Edge, High-Performance,
    Multi-, All-New, Innovative, Smart, Intelligent

  GENERIC SUFFIXES (when used alone without specific modifier):
    Technology, System, Solution, Platform, Complex, Formula, Blend,
    Design, Engineering, Architecture, Matrix, Protocol

  GENERIC STANDALONE:
    "The Edge", "X-Factor", "ProSystem", "Formula 7", "Matrix",
    "Genesis", "Apex", "Pinnacle", "Zenith", "Prime"

  IF match found: FLAG as GENERIC
```

### Step 2: Semantic Analysis

```
FOR each feature name:
  ASK: "Could this name apply to ANY product in ANY category?"

  TEST: Replace the product category:
    Original: "[Feature Name] in a golf club"
    Test: "[Feature Name] in a blender"
    Test: "[Feature Name] in a car"
    Test: "[Feature Name] in a supplement"

    IF name works equally well for 3+ categories:
      FLAG as TOO_GENERIC -- name lacks product specificity

  EXAMPLE:
    "Advanced Technology" -- works for ALL categories -> GENERIC
    "Dynamic Loft Control" -- only makes sense for golf -> SPECIFIC
    "Premium Materials" -- works for ALL categories -> GENERIC
    "L-Theanine for Tranquility" -- only makes sense for supplements -> SPECIFIC
```

### Step 3: Competitive Comparison

```
FOR each feature name:
  ASK: "Could a direct competitor use this exact name?"

  IF competitor could use the same name without modification:
    FLAG as UNDIFFERENTIATED

  EXAMPLE:
    "High-Performance Core" -- any golf club maker could use this -> UNDIFFERENTIATED
    "HyperSpeed Face" -- coined, proprietary-feeling -> DIFFERENTIATED
```

### Step 4: Generate Remediation

```
FOR each flagged name:
  PROVIDE specific remediation:
    generic_name: [the flagged name]
    detection_method: [pattern|semantic|competitive]
    why_generic: [specific explanation]
    suggested_replacement_approach: [how to fix]

  EXAMPLE:
    generic_name: "Advanced Cushioning Technology"
    detection_method: "pattern (Advanced + Technology)"
    why_generic: "Both prefix and suffix are on the generic kill list"
    suggested_replacement_approach: "Name the specific cushioning mechanism + the specific comfort outcome"
```

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Any generic name detected | RETURN to Layer 2 for renaming |
| CRITICAL | Multiple generic names (3+) | RETURN to Layer 2 for complete renaming pass |
| WARNING | Name is borderline (passes pattern but fails semantic) | Present to human for judgment |
| INFO | All names pass all checks | Proceed to packaging |
