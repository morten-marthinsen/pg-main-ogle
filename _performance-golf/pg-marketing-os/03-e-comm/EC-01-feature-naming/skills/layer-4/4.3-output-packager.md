# 4.3-output-packager.md

> **Layer:** 4 (Validation & Packaging)
> **Skill:** EC-01-feature-naming
> **Purpose:** Package feature-package.json
> **Outputs:** `feature-package.json` file in project output directory

---

## FUNCTION

Package all validated feature names, hierarchy, micro-scripts, and validation results into the canonical output file `feature-package.json`. This file is consumed by EC-02 (Hero), EC-03 (Section Copy), and EC-04 (Micro-Scripts).

---

## PACKAGING PROTOCOL

### Step 1: Verify All Validations Passed

```
CHECK:
  generic_name_detection: MUST be PASS (zero generics)
  standalone_test: MUST be PASS (all features meet threshold)
  arena_selection: MUST have human selection recorded

IF any check fails:
  HALT -- "Cannot package feature set that failed validation"
```

### Step 2: Assemble Package

```
BUILD feature-package.json:

  header:
    version: "1.0"
    generated_at: [ISO timestamp]
    skill_id: "EC-01-feature-naming"
    product_name: [from upstream]

  feature_hierarchy:
    hero: [1-2 features with names, micro-scripts, scores]
    supporting: [3-5 features with names, micro-scripts, scores]
    technical: [1-4 features with names, micro-scripts, scores]

  naming_validation:
    generic_names_detected: 0
    standalone_test_failures: 0
    all_names_caveman_benefit: true
    arena_selection_source: [generated|modified|human_provided]

  downstream_handoffs:
    ec_02_hero: hero features for headline/value prop
    ec_03_sections: all features for section copy
    ec_04_scripts: micro-scripts as script seeds
```

### Step 3: Write and Verify

```
WRITE feature-package.json to:
  ~outputs/[project-code]/e-comm/feature-package.json

VERIFY:
  [ ] File written successfully
  [ ] Valid JSON (parseable)
  [ ] All features have names + micro-scripts
  [ ] Zero placeholder text
  [ ] Hero features present
  [ ] Validation results included
  [ ] Downstream handoff references populated
```

```json
{

  }

```

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Validation not passed | Return to 4.1/4.2, fix failures |
| CRITICAL | File write fails | Check directory permissions and path |
| WARNING | Package missing micro-scripts | Return to 2.2 for completion |
| WARNING | Hero feature not clearly identified | Return to 1.2 for hierarchy validation |
