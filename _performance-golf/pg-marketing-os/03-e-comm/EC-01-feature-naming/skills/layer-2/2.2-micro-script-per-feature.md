# 2.2-micro-script-per-feature.md

> **Layer:** 2 (Feature Naming)
> **Skill:** EC-01-feature-naming
> **Purpose:** Create DSI + micro-scripts using Bill Schley framework
> **Outputs:** `dsi` object + `micro_scripts` object with scripts for all named features
> **Framework:** Bill Schley "The Micro-Script Rules"
> **Training doc:** `/pg-training/micro-scripts.md`

---

## FUNCTION

Create a Dominant Selling Idea (DSI) at the product level, then create short, repeatable micro-scripts for each feature. The DSI is the single most important sentence for the product. Micro-scripts are ~5 word phrases that express the DSI and feature benefits in forms people naturally repeat. These scripts feed EC-02 (hero copy), EC-03 (section copy), and EC-04 (video scripts).

**CRITICAL:** A micro-script is NOT a 25-50 word explanatory paragraph. It is a ~5 word phrase that passes the "bubba at the bar" repeat test. If someone wouldn't naturally say it to a friend, it's not a micro-script.

---

## DSI PROTOCOL (PRODUCT-LEVEL — DO FIRST)

### Step 1: Create the DSI

```
The DSI is ONE sentence for the entire product. NOT per-feature.

STRUCTURE:
  "The only [category] with [named mechanism] that [benefit 1] and [benefit 2]."

SCHLEY'S 5 INGREDIENTS (all required):
  1. SUPERLATIVE: "The only" / "The first" / "The #1"
  2. IMPORTANT: Solves a real problem the customer cares about
  3. BELIEVABLE: Named mechanism, real engineering, credentialed designer
  4. MEASURABLE: Can be tested, demonstrated, or felt
  5. OWNABLE: No competitor can make the same claim

EXAMPLE (RS1):
  "The only premium putter with Forward Axis Weighting that eliminates
   face drift and improves your stroke at the same time."

GATE: DSI must pass all 5 ingredients. If any fails, revise before proceeding.
```

### Step 2: Create Product-Level Micro-Scripts

```
Create 3-4 product-level micro-scripts that express the DSI:

TYPES (from Schley):
  PRIMARY TAGLINE: The one phrase everyone repeats (Stark Reminder template)
  SUPPORTING THEME: Secondary repeatable phrase
  PAGE ELEMENT: Used as design/visual copy element
  SECTION-SPECIFIC: Used in a specific page section

CONSTRAINTS:
  - Average ~5.2 words per micro-script
  - Must pass the "bubba at the bar" repeat test
  - 4 Schley templates available:
    1. A/B Equation: "[A] means [B]" or "[Less A], [more B]"
    2. Stark Reminder: A vivid phrase that sticks ("Pull it back, let it fall.")
    3. Unique Wordplay: Alliteration, rhyme, or juxtaposition
    4. Whole Micro Story: A complete mini-narrative in one phrase

EXAMPLE (RS1):
  PRIMARY: "Pull it back, let it fall." (Stark Reminder)
  SUPPORTING: "Do less. Make more." (Stark Reminder)
  PAGE ELEMENT: "Gravity-driven. Face-controlled." (Wordplay)
  SECTION: "Gravitational face control." (Wordplay)
```

## FEATURE-LEVEL MICRO-SCRIPT PROTOCOL

### Step 3: Evaluate Superlative per Feature

```
FOR each named feature:
  QUESTION: Does this feature have a genuine "only" claim?

  IF YES: Feature gets a superlative micro-script
  IF NO: Feature gets a benefit micro-script (no superlative language)
  IF PARTIAL: Note it — unique to this product but the category exists elsewhere

  NOT every feature needs a superlative. Only features with genuine
  "only" claims get superlative micro-scripts. The DSI carries the
  product-level superlative.
```

### Step 4: Write Micro-Script per Feature

```
FOR each feature:

  STRUCTURE:
    ONE micro-script phrase (~3-7 words) using a Schley template

  CONSTRAINTS:
    - Target ~5.2 words (Schley average)
    - Must use a Schley template (A/B, Stark Reminder, Wordplay, Whole Micro Story)
    - Must pass "bubba at the bar" repeat test
    - Must be SPECIFIC to this feature (not generic)
    - No slop words
    - Does NOT need to contain the feature name — the micro-script sits next to the name

  GOOD EXAMPLES:
    "Forward weight kills face drift." (A/B Equation — 5 words)
    "Less arc, less error." (A/B Equation — 4 words)
    "Ten pieces, one purpose." (A/B Equation — 4 words)
    "Quiet hands, loud results." (Wordplay — 4 words)
    "Line it up, lock it in." (Wordplay — alliteration — 7 words)
    "Sculpted for scoring." (Wordplay — alliteration — 3 words)

  BAD EXAMPLES (these are NOT micro-scripts):
    "HyperSpeed Face uses variable thickness technology across the
     striking surface to maximize ball speed." (25 words — this is a description)
    "The revolutionary face design improves performance." (7 words but generic + slop)
```

### Step 5: Quality Check per Script

```
FOR each micro_script:
  VERIFY:
    [ ] Word count 3-8 (target ~5)
    [ ] Uses a Schley template (A/B, Stark Reminder, Wordplay, Whole Micro Story)
    [ ] Passes "bubba at the bar" repeat test — would someone say this to a friend?
    [ ] Benefit is specific (not generic)
    [ ] No slop words present
    [ ] Does not duplicate another feature's micro-script
    [ ] Standalone — doesn't reference other features

  IF any check fails:
    REVISE script
    LOG: "Script for [feature] revised -- [reason]"
```

### Step 6: Cross-Feature Consistency Check

```
REVIEW all scripts together:
  [ ] No two scripts make the same benefit claim
  [ ] Each uses a different Schley template where possible (pattern diversity)
  [ ] Tone consistent across all scripts
  [ ] No contradictions between scripts
  [ ] Scripts don't reference each other (each standalone)
  [ ] DSI is the anchor — feature micro-scripts support, not compete
```

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | DSI fails any of Schley's 5 ingredients | Revise DSI before creating micro-scripts |
| CRITICAL | Micro-script exceeds 8 words | Compress — micro-scripts are ~5 words |
| CRITICAL | Micro-script is a paragraph/description, not a phrase | Rewrite as short repeatable phrase |
| WARNING | Micro-script doesn't pass repeat test | Revise — if you wouldn't say it at a bar, it's not a micro-script |
| WARNING | Multiple features share the same Schley template | Diversify — use different templates for variety |
| WARNING | Slop word detected | Replace with concrete language |
| INFO | Feature has no superlative | Fine — not every feature needs one. DSI carries the product superlative. |
