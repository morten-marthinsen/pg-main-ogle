# 2.2-micro-script-per-feature.md

> **Layer:** 2 (Feature Naming)
> **Skill:** EC-01-feature-naming
> **Purpose:** Write 1-2 sentence micro-script per feature
> **Outputs:** `micro_scripts` object with scripts for all named features

---

## FUNCTION

Write a 1-2 sentence micro-script for every named feature. The micro-script is the feature's elevator pitch -- it uses the feature name, explains what it does, and states why it matters. These scripts feed EC-02 (hero copy), EC-03 (section copy), and EC-04 (video scripts).

---

## MICRO-SCRIPT PROTOCOL

### Step 1: Load Named Features

```
FROM named_features (2.1 output):
  FOR each named feature:
    feature_name: [selected name]
    tier: [hero/supporting/technical]
    what_it_is: [from capability map]
    what_it_does: [from capability map]
    why_it_matters: [from capability map]
```

### Step 2: Write Micro-Script per Feature

```
FOR each feature:

  STRUCTURE:
    Sentence 1: [Feature Name] + what it does (action/function)
    Sentence 2: Why that matters to the customer (benefit/outcome)

  CONSTRAINTS:
    - MUST use the exact feature name (as named in 2.1)
    - 15-50 words total (target 25-35)
    - Maximum 2 sentences
    - Benefit must be SPECIFIC (not "improves performance")
    - No slop words (unlock, harness, leverage, revolutionary)

  TONE CALIBRATION:
    Hero features: Slightly more assertive, benefit-forward
    Supporting features: Informative, clear, benefit-present
    Technical features: Factual, credibility-building, precise

  EXAMPLE (hero, 32 words):
    "HyperSpeed Face uses variable thickness technology across the
     striking surface to maximize ball speed on every hit -- even
     mishits -- delivering up to 15 more yards of carry distance."

  EXAMPLE (supporting, 28 words):
    "Dynamic Loft Control automatically adjusts the club face angle
     during your swing, optimizing launch conditions for maximum
     carry distance on every shot."

  EXAMPLE (technical, 22 words):
    "Forged from 9-1-1 titanium alloy, the face maintains structural
     integrity across 50,000+ impacts while staying 15% lighter
     than cast alternatives."
```

### Step 3: Quality Check per Script

```
FOR each micro_script:
  VERIFY:
    [ ] Uses exact feature name from 2.1
    [ ] Word count 15-50
    [ ] Maximum 2 sentences
    [ ] Sentence 1 explains WHAT
    [ ] Sentence 2 explains WHY (or sentence 1 covers both)
    [ ] Benefit is specific (not generic)
    [ ] No slop words present
    [ ] Tone matches tier (hero/supporting/technical)

  IF any check fails:
    REVISE script
    LOG: "Script for [feature] revised -- [reason]"
```

### Step 4: Cross-Feature Consistency Check

```
REVIEW all scripts together:
  [ ] No two scripts make the same benefit claim
  [ ] Feature names used consistently (exact match, not paraphrased)
  [ ] Tone consistent across all scripts
  [ ] No contradictions between scripts
  [ ] Scripts don't reference each other (each standalone)
```

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Feature name not used in script | Revise -- name must appear in script |
| CRITICAL | Script exceeds 50 words | Compress -- split into essential WHAT + WHY |
| WARNING | Benefit is generic ("improves performance") | Revise with specific outcome |
| WARNING | Slop word detected | Replace with concrete language |
| INFO | Script under 15 words | May be too thin -- verify WHAT + WHY both covered |
