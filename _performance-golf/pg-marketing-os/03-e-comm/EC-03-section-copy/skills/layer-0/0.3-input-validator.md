# 0.3-input-validator.md

> **Layer:** 0 (Foundation)
> **Skill:** EC-03-section-copy
> **Purpose:** Validate section map + word budgets present
> **Outputs:** `validation_result` object with PASS/FAIL

---

## FUNCTION

Validate that all required inputs are present and sufficient for section copy generation: section map from EC-00 with word budgets, feature package from EC-01, and crossover patterns.

---

## VALIDATION PROTOCOL

### Step 1: Section Map Validation
```
CHECK:
  [ ] Active BTF sections exist (at least 7 BTF sections)
  [ ] Every active section has a word budget (50-500 range)
  [ ] Every active section has a proof type requirement
  [ ] Priority order is defined (no gaps, no duplicates)
  [ ] Section names match NLS framework
```

### Step 2: Feature Package Validation
```
CHECK:
  [ ] Feature package loaded
  [ ] All features have exact names (from EC-01)
  [ ] Micro-scripts available for reference
  [ ] Hero feature identified
```

### Step 3: Crossover Pattern Validation
```
CHECK:
  [ ] Crossover patterns loaded for applicable sections
  [ ] Patterns adapted for ecom format
  IF patterns missing: WARN -- will write ecom-native
```

### Step 4: Word Budget Totals
```
SUM all BTF section word budgets
CHECK total is within page-type bounds (1200-4500 for BTF)
IF total exceeds bounds: WARN -- "BTF total is [X]. Consider reducing."
```

---

## OUTPUT STRUCTURE

```json
{
  "validation_result": {
    "status": "PASS",
    "btf_section_count": 13,
    "total_btf_word_budget": 2950,
    "features_available": 8,
    "crossover_patterns_loaded": 5,
    "flags": []
  }
}
```

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | No BTF sections in map | HALT -- return to EC-00 |
| CRITICAL | No word budgets defined | HALT -- return to EC-00 |
| DEGRADED | Some crossover patterns missing | Proceed, write ecom-native |
