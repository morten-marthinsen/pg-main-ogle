# 0.1-upstream-loader.md

> **Layer:** 0 (Foundation)
> **Skill:** EC-03-section-copy
> **Purpose:** Load strategy map + feature package
> **Outputs:** `upstream_context` object with section map and features

---

## FUNCTION

Load the e-commerce strategy from EC-00 (section map with priorities and word budgets) and the feature package from EC-01 (named features with micro-scripts). These two packages provide the complete context for writing every BTF section.

---

## LOADING PROTOCOL

### Step 1: Load Strategy
```
LOCATE: ecomm-strategy.yaml
EXTRACT:
  active_sections: [all BTF sections with priority, word budget, proof type, crossover skill]
  excluded_sections: [with rationale]
  word_budget_summary: [total and per-section]
  crossover_map: [section-to-skill mappings]
```

### Step 2: Load Feature Package
```
LOCATE: feature-package.json from EC-01
EXTRACT:
  all_features: [with exact names and micro-scripts]
  feature_hierarchy: [hero, supporting, technical]
  naming_validation: [confirmation all names are final]
```

### Step 3: Build Section Writing Context
```
FOR each active BTF section:
  ASSEMBLE:
    section_number, section_name, priority, word_budget
    proof_type_required
    crossover_skill (if any)
    applicable_features (which EC-01 features belong in this section)
    design_notes_seed (from EC-00)
```

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | EC-00 strategy not found | HALT -- execute EC-00 |
| CRITICAL | EC-01 feature package not found | HALT -- execute EC-01 |
| DEGRADED | Crossover mappings incomplete | Proceed, write ecom-native |
