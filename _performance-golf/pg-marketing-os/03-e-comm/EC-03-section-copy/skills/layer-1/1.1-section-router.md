# 1.1-section-router.md

> **Layer:** 1 (Section Architecture)
> **Skill:** EC-03-section-copy
> **Purpose:** Route each section to appropriate copy template
> **Outputs:** `routing_map` object with template assignment per section

---

## FUNCTION

Assign each active BTF section to the appropriate copy template based on section type, crossover pattern availability, and product data. The template determines the structural format of the section copy.

---

## ROUTING PROTOCOL

### Step 1: Template Library

```
SECTION TEMPLATES:

  T1: Problem-Solution
    Sections: 9 (Problem/Solution)
    Structure: Problem statement (2-3 sentences) -> Bullet agitation (3-5 points) -> Product answer (2-3 sentences)
    Crossover: Skill 13

  T2: How-It-Works
    Sections: 10 (Why It Works)
    Structure: Feature name intro -> 3-5 numbered steps -> Proof point
    Crossover: Skill 14

  T3: Feature Showcase
    Sections: 4 (Product Highlights), 7 (Ingredients/Materials)
    Structure: Feature name (bold) -> 1-2 sentence description -> Benefit callout
    Crossover: Skill 15

  T4: Proof Block
    Sections: 5 (UGC), 6 (Expert Credibility), 14 (Reviews), 19 (Awards/Press)
    Structure: Proof element(s) -> Context statement -> Credibility anchor
    Crossover: Skill 18

  T5: Offer/Value
    Sections: 3 (CTA Box), 12 (Cost Savings), 16 (Offer/Pricing), 17 (Guarantee)
    Structure: Value statement -> Price/comparison -> CTA or guarantee text
    Crossover: Skill 16

  T6: Informational
    Sections: 8 (What to Expect), 13 (How to Use), 15 (FAQ), 18 (Use Cases)
    Structure: Question/context -> Answer/steps -> Supporting detail
    Crossover: None (ecom-native)

  T7: Comparison
    Sections: 11 (Comparison Chart)
    Structure: Comparison table -> Key differentiators (bolded) -> Conclusion
    Crossover: None (ecom-native)
```

### Step 2: Route Each Section

```
FOR each active BTF section:
  MATCH section_number to template:
  ASSIGN template_id
  LOAD crossover_pattern if available
  NOTE: If crossover pattern doesn't fit, use ecom-native template
```

---

## OUTPUT STRUCTURE

```json
{
  "routing_map": [
    {
      "section_number": 9,
      "section_name": "Problem/Solution",
      "template": "T1_problem_solution",
      "crossover_skill": "13-root-cause",
      "structure": "Problem -> Bullets -> Product answer",
      "word_budget": 250
    }
  ],
  "template_distribution": {
    "T1": 1, "T2": 1, "T3": 2, "T4": 3, "T5": 3, "T6": 3, "T7": 1
  }
}
```

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| WARNING | Section doesn't fit any template | Use T6 (Informational) as default |
| INFO | Multiple templates could work | Select based on crossover availability |
