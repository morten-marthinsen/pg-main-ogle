# 4.1-quality-scorer.md

> **Layer:** 4 (Scoring & Packaging)
> **Skill:** EC-06-ecomm-editorial
> **Purpose:** Score final copy against ecom editorial quality criteria
> **Outputs:** `quality_scores` object with per-dimension and overall scores

---

## FUNCTION

Score the editorially revised e-commerce copy against six quality dimensions. This is the objective quality gate that determines whether the copy is ready for page builder handoff. Every dimension has a minimum threshold. The overall score must meet 7.5/10 to pass.

---

## SCORING PROTOCOL

### Step 1: Score Each Dimension
```
FOR the complete revised page copy:

  DIMENSION 1: SCAN OPTIMIZATION (weight: 25%)
    Score each section on 3-5 second scan test
    Average across all sections
    MINIMUM: 7.0

  DIMENSION 2: PROOF DENSITY (weight: 20%)
    Count proof elements per section
    Classify proof strength (STRONG/MODERATE/WEAK)
    Verify minimum 1 proof per section
    MINIMUM: 7.0

  DIMENSION 3: FEATURE CONSISTENCY (weight: 20%)
    Compare every feature reference to EC-01 registry
    Count deviations (paraphrasing, shortening, invention)
    MINIMUM: 9.0 (feature names must be near-perfect)

  DIMENSION 4: DESIGN NOTE QUALITY (weight: 15%)
    Check layout + mobile + visuals per section
    Verify actionability (not vague)
    MINIMUM: 7.0

  DIMENSION 5: WORD BUDGET COMPLIANCE (weight: 10%)
    Compare actual word counts to EC-00 budgets
    Score based on adherence
    MINIMUM: 7.0

  DIMENSION 6: SLOP CLEANLINESS (weight: 10%)
    Scan for banned lexicon words
    Check for AI telltales introduced during revision
    Count remaining slop instances
    MINIMUM: 8.0
```

### Step 2: Calculate Weighted Overall Score
```
overall = (scan * 0.25) + (proof * 0.20) + (feature * 0.20) +
          (design * 0.15) + (budget * 0.10) + (slop * 0.10)

IF overall >= 7.5: PASS
IF overall < 7.5: FAIL -- return to Layer 2 for additional revision
```

### Step 3: Generate Score Report
```
BUILD score_report:
  per_dimension: [score, min_threshold, status, specific notes]
  overall: [weighted score, status]
  weakest_dimension: [which dimension scores lowest]
  improvement_notes: [if FAIL, what needs to improve]
```

---

## SCORING RUBRIC

**Scan Optimization (25%)**
- 9-10: Every section instant-scannable; perfect hierarchy
- 7-8: Most sections scan well; minor formatting gaps
- 5-6: Mixed; some sections require reading
- 3-4: Poor scan optimization across page
- 1-2: Page reads like long-form; no scan formatting

**Proof Density (20%)**
- 9-10: Every section has strong proof; multiple types used across page
- 7-8: Every section has proof; most are strong
- 5-6: Most sections have proof; some gaps
- 3-4: Multiple proof deserts
- 1-2: Page largely unsubstantiated

**Feature Consistency (20%)**
- 10: Perfect -- zero deviations from EC-01 registry
- 9: Near perfect -- 1 minor variation
- 7-8: Mostly consistent -- 2-3 variations
- 5-6: Inconsistent -- feature names modified
- 1-4: Feature names largely ignored

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Overall score < 7.5 | Return to Layer 2 for targeted revision |
| CRITICAL | Any dimension below its minimum | Fix that dimension specifically |
| WARNING | Overall 7.5-8.0 | Passes but note areas for improvement |
| INFO | Overall >= 8.5 | Strong editorial pass; proceed to packaging |
