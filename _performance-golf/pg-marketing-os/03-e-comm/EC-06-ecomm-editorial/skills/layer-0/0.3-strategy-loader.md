# 0.3-strategy-loader.md

> **Layer:** 0 (Foundation)
> **Skill:** EC-06-ecomm-editorial
> **Purpose:** Load strategy for constraint checking during editorial
> **Outputs:** `strategy_constraints` object with editorial-relevant strategy data

---

## FUNCTION

Load the e-commerce strategy from EC-00 to establish the constraints editorial must respect: word budgets per section, priority rankings, section types, and page-level parameters. Editorial cannot change strategy -- it can only revise copy within strategy constraints.

---

## LOADING PROTOCOL

### Step 1: Load Strategy
```
LOCATE: ecomm-strategy.yaml from EC-00
EXTRACT editorial-relevant constraints:
  section_word_budgets: [per-section word limits -- hard constraints]
  section_priorities: [priority rankings for revision attention allocation]
  page_type: [pdp, collection, bundle -- affects scan expectations]
  total_page_budget: [maximum total word count]
  section_types: [NLS PDP type per section]
```

### Step 2: Build Constraint Checklist
```
FOR each section:
  BUILD constraint:
    section_number: [N]
    section_name: [name]
    word_budget: [max words -- editorial cannot exceed]
    priority: [higher priority = more editorial attention]
    section_type: [determines scan optimization expectations]
    crossover_origin: [if section used long-form crossover, note pattern]
```

### Step 3: Derive Editorial Priorities
```
RANK sections for editorial attention:
  1. Highest priority sections first (priority 1-3)
  2. Within same priority, sections flagged in audit first
  3. Sections already scan-optimized get lighter editorial pass
  4. Design notes always checked regardless of priority
```
