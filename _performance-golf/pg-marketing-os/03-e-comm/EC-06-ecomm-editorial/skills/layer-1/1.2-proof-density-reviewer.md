# 1.2-proof-density-reviewer.md

> **Layer:** 1 (Audit)
> **Skill:** EC-06-ecomm-editorial
> **Purpose:** Review proof density (minimum 1 proof element per section)
> **Outputs:** `proof_review` object with per-section proof status and gaps

---

## FUNCTION

Review every section for proof density. Every e-commerce section must contain at least one specific proof element. Vague proof ("studies show") does not count. This reviewer identifies proof deserts, weak proofs, and opportunities to strengthen existing proof elements.

---

## REVIEW PROTOCOL

### Step 1: Inventory Proof Elements
```
FOR each section in assembled copy:
  SCAN for proof elements:
    statistics: [specific numbers with source]
    customer_quotes: [named or attributed testimonials]
    expert_citations: [named expert with credentials]
    clinical_data: [study results with specifics]
    awards_press: [named publications or awards]
    comparison_data: [vs competitor or previous version]
    user_count: [number of customers, users, sold units]
    certifications: [third-party certifications]
    before_after: [specific transformation data]

  COUNT proof elements per section
  CLASSIFY each proof as STRONG, MODERATE, or WEAK
```

### Step 2: Proof Strength Classification
```
STRONG proof:
  - Has specific numbers ("87% improvement")
  - Has attribution ("Dr. Smith, Stanford")
  - Has context ("in a 12-week double-blind study")

MODERATE proof:
  - Has numbers but no attribution ("87% improvement")
  - Has attribution but no specifics ("recommended by Dr. Smith")
  - Has context but vague numbers ("significant improvement")

WEAK proof (does NOT satisfy minimum requirement):
  - "Studies show..." (no study cited)
  - "Experts agree..." (no expert named)
  - "Clinically proven" (no clinical data)
  - "Thousands of happy customers" (no specific count)
```

### Step 3: Identify Gaps
```
FOR each section:
  IF proof_count == 0: FLAG as "proof desert" -- CRITICAL
  IF proof_count >= 1 but all WEAK: FLAG as "weak proof" -- WARNING
  IF proof_count >= 1 and at least 1 STRONG: STATUS "adequate"
  IF proof_count >= 2 with STRONG elements: STATUS "excellent"
```

### Step 4: Generate Strengthening Directives
```
FOR each flagged section:
  DIRECTIVE options:
    - "Add [specific proof type] to this section"
    - "Strengthen weak proof: replace 'studies show' with specific study"
    - "Add attribution to existing stat: who conducted the study?"
    - "Reposition proof element for scan visibility: move to bolded callout"

  NOTE: Directives suggest strengthening -- NOT removal
  NOTE: If client data needed, flag as "requires client input"
```

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | 3+ sections with zero proof | Flag for proof sourcing before editorial |
| WARNING | Proof deserts in high-priority sections | Prioritize for proof addition |
| INFO | All sections have adequate proof | Focus on strengthening and repositioning |
