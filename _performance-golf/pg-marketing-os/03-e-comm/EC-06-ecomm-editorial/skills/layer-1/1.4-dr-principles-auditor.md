# 1.4-dr-principles-auditor.md

> **Layer:** 1 (Audit)
> **Skill:** EC-06-ecomm-editorial
> **Purpose:** Audit direct-response conversion principles across the assembled page
> **Outputs:** DR principles audit report with per-principle scoring

---

## FUNCTION

Evaluate the assembled e-commerce copy against core direct-response conversion principles. E-comm pages that read as informational product sheets instead of direct-response sales pages have fundamentally lower conversion rates. This audit checks offer clarity, risk reversal, CTA quality, urgency credibility, value stacking, and objection handling.

---

## AUDIT PROTOCOL

### Principle 1: Offer Clarity
```
CHECK:
  [ ] Product deliverables are explicitly stated (what they GET)
  [ ] Quantity/size/duration is clear
  [ ] No ambiguity about what's included vs. separate purchase
  [ ] Offer is stated in ONE clear section (not scattered)

SCORING:
  9-10: Crystal clear — a stranger knows exactly what they're buying
  7-8: Mostly clear — minor detail missing
  5-6: Somewhat clear — needs a second read to understand the offer
  3-4: Unclear — significant ambiguity about what's included
  1-2: No clear offer statement on the page
```

### Principle 2: Risk Reversal
```
CHECK:
  [ ] Guarantee exists and is prominently placed (not footer-only)
  [ ] Guarantee is specific (duration, terms, process)
  [ ] Guarantee has a NAME (not just "money-back guarantee")
  [ ] Guarantee section has copy weight (not legal fine print)
  [ ] Return/refund process is frictionless (stated or implied)

SCORING:
  9-10: Named guarantee, prominently placed, specific terms, feels risk-free
  7-8: Guarantee present and clear, could be more prominent or named
  5-6: Basic "money-back guarantee" without specifics or prominence
  3-4: Guarantee buried in footer or vague
  1-2: No guarantee visible on the page
```

### Principle 3: CTA Quality
```
CHECK:
  [ ] Primary CTA is action-specific ("Get Yours Today", "Try Risk-Free")
  [ ] No passive CTAs ("Learn More", "Submit", "Click Here")
  [ ] CTA appears at strategic points (hero, mid-page, near offer, footer)
  [ ] CTA copy matches the offer (if guarantee exists, CTA can reference it)
  [ ] CTA is visually prominent (implied by design notes)

SCORING:
  9-10: Action-specific CTAs at 3+ strategic points; CTA copy reinforces offer
  7-8: Good CTAs at 2+ points; mostly action-specific
  5-6: Generic CTAs ("Shop Now" everywhere); adequate placement
  3-4: Passive CTAs or poor placement; CTA feels like an afterthought
  1-2: Missing CTAs or "Learn More" / "Submit" used
```

### Principle 4: Urgency Credibility
```
CHECK:
  [ ] IF urgency is used, it is REAL (limited stock, seasonal, launch, genuine deadline)
  [ ] No fake countdown timers implied
  [ ] No "Order in the next 10 minutes" language without real basis
  [ ] Urgency feels earned, not manufactured
  [ ] IF no urgency is appropriate, none is forced

SCORING:
  9-10: Real urgency with credible basis, or appropriately absent
  7-8: Mild urgency that feels natural; not overdone
  5-6: Generic urgency ("Don't wait!") without specific basis
  3-4: Fake-feeling urgency that undermines trust
  1-2: Blatant fake scarcity ("Only 3 left!" on an unlimited product)
```

### Principle 5: Value Stack
```
CHECK:
  [ ] Value is established BEFORE price is revealed
  [ ] Individual component values are stated (if applicable)
  [ ] Alternative costs are referenced (what they'd pay elsewhere)
  [ ] Price is framed favorably (per day, per use, vs. alternative)

SCORING:
  9-10: Complete value stack — components valued, alternatives compared, price framed
  7-8: Good value establishment; price contextualized
  5-6: Some value context; price appears without strong framing
  3-4: Price appears with minimal value context
  1-2: Price shown with no value framing
```

### Principle 6: Objection Handling
```
CHECK:
  [ ] Top 3-5 buying objections are addressed somewhere on the page
  [ ] FAQ section covers real objections (not softball questions)
  [ ] Guarantee handles the "what if it doesn't work?" objection
  [ ] Comparison section handles the "why not the alternative?" objection
  [ ] Social proof handles the "does it actually work?" objection

SCORING:
  9-10: All major objections addressed across sections; feels complete
  7-8: Most objections addressed; 1-2 gaps
  5-6: Some objections addressed; major gaps remain
  3-4: Few objections addressed; FAQ is weak or missing
  1-2: No objection handling visible
```
