# 1.4 Script Structure Planning

## Purpose
Plan the complete structure for each script including section timing, content flow, pattern interrupts, and CTA placement.

---

## Input Requirements

From Layer 1:
- `1.2-format-selection.yaml` (format specs)
- `1.3-hook-strategy.yaml` (hook plans)

From Layer 0:
- `0.1-input-validation.yaml` (script requirements)
- `0.4-cbf-context.yaml` (messaging, CTAs)

---

## Execution Protocol

### Step 1: Define Section Structure Per Script

For each script, map out complete structure:

```yaml
script_structures:
  - script_id: "script_1"
    platform: "TikTok"
    format: "TikTok-Short-15-30s"
    target_length: "25 seconds"

    section_plan:
      hook:
        timing: "0:00-0:01"
        purpose: "Stop scroll, create curiosity"
        word_count: "5-8 words"
        visual_notes: "Pattern interrupt visual"

      value_section_1:
        timing: "0:01-0:10"
        purpose: "Deliver core insight #1"
        word_count: "30-40 words"
        pattern_interrupt: "0:05 (visual cut)"

      value_section_2:
        timing: "0:10-0:22"
        purpose: "Deliver core insight #2 + application"
        word_count: "40-50 words"
        pattern_interrupt: "0:15 (text overlay)"

      cta:
        timing: "0:22-0:25"
        purpose: "Save/follow CTA"
        word_count: "8-10 words"
        visual_notes: "End card or loop"
```

---

### Step 2: Plan Pattern Interrupts

Define pattern interrupt strategy per script:

```yaml
pattern_interrupt_strategy:
  - script_id: "script_1"
    total_length: "25s"
    interrupt_frequency: "Every 5 seconds"

    planned_interrupts:
      - timestamp: "0:00"
        type: "Hook (visual + verbal)"
        method: "Unexpected visual"

      - timestamp: "0:05"
        type: "Visual cut"
        method: "Camera angle change"

      - timestamp: "0:10"
        type: "Text overlay"
        method: "Key point emphasized"

      - timestamp: "0:15"
        type: "Pacing change"
        method: "Speed up delivery"

      - timestamp: "0:22"
        type: "CTA transition"
        method: "Visual end card"
```

---

### Step 3: Plan Content Flow

Map logical progression of ideas:

```yaml
content_flow:
  - script_id: "script_1"
    narrative_arc:
      - section: "Hook"
        message: "[Curiosity-creating question]"
        emotional_beat: "Curiosity"

      - section: "Value 1"
        message: "[Answer setup + insight]"
        emotional_beat: "Aha moment"

      - section: "Value 2"
        message: "[Deeper insight + application]"
        emotional_beat: "Empowerment"

      - section: "CTA"
        message: "[Save/follow prompt]"
        emotional_beat: "Action"

    transitions:
      - from: "Hook"
        to: "Value 1"
        method: "Direct answer to hook question"

      - from: "Value 1"
        to: "Value 2"
        method: "'But here's what most people miss...'"
```

---

### Step 4: Plan CTA Integration

Define CTA strategy per script:

```yaml
cta_planning:
  - script_id: "script_1"
    content_function: "Awareness"
    cta_type: "Soft"
    cta_specific: "Save this for later"

    cta_placement:
      timing: "0:22-0:25"
      integration_style: "Natural conclusion"
      visual_component: "Save icon animation"

    cta_rationale: |
      Awareness function + educational content → save CTA drives algorithm.
      Specimen analysis shows save CTAs boost reach 3x.
```
