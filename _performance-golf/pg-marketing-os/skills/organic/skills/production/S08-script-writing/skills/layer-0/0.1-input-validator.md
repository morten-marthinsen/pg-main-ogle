# 0.1 Input Validator

## Purpose
Validate that all required input exists before proceeding with script writing. Prevent execution with incomplete or invalid requests.

---

## Input Requirements

From upstream:
- `[project]/S07-campaign-brief/outputs/[campaign]-CBF.yaml` (Campaign Brief File)

Expected in request:
```yaml
script_requests:
  - content_title: "[Working title]"
    platform: "[TikTok/Instagram/YouTube/etc]"
    format: "[Reel/Short/Long-form]"
    target_length: "[seconds/minutes]"
    content_pillar: "[From CAF]"
    content_function: "[Awareness/Engagement/Conversion/Community]"
    key_message: "[Core message]"
    desired_action: "[What viewer should do]"
```

---

## Execution Protocol

### Step 1: Verify CBF Exists

```yaml
cbf_check:
  - [ ] CBF file path provided
  - [ ] CBF file exists and readable
  - [ ] CBF has campaign_name field
  - [ ] CBF has content_pillars defined
  - [ ] CBF has target_platforms defined
  - [ ] CBF has funnel_integration section
```

**CRITICAL:** If CBF doesn't exist, script writing CANNOT proceed. This violates Law 1 (No strategy, no content).

---

### Step 2: Validate Script Requests

For EACH script request:

```yaml
request_validation:
  required_fields:
    - [ ] content_title present and not empty
    - [ ] platform specified (TikTok/Instagram/YouTube/LinkedIn/X)
    - [ ] format specified (Reel/TikTok/Short/Long-form)
    - [ ] target_length specified in seconds or minutes
    - [ ] content_function defined (Awareness/Engagement/Conversion/Community)
    - [ ] key_message present and substantive (not placeholder)

  platform_compatibility:
    - [ ] Platform is in CBF target_platforms list
    - [ ] Format matches platform (e.g., "Reel" requires Instagram)

  minimum_count:
    - [ ] At least 3 script requests provided (per campaign minimum)
```

**If any required field missing → FAIL validation → request clarification**

---

### Step 3: Cross-Reference with CBF

Verify alignment between script requests and campaign strategy:

```yaml
strategic_alignment:
  - [ ] Each script's content_pillar exists in CBF content_pillars
  - [ ] Each script's desired_action aligns with CBF funnel_integration
  - [ ] Platform mix matches CBF platform_strategy
  - [ ] Content functions distribute appropriately (not all conversion)
```

---

### Step 4: Check for Platform-Native Requirements

Ensure scripts are NOT cross-platform duplicates:

```yaml
platform_native_check:
  - [ ] No two scripts have identical content_title AND different platforms
  - [ ] Each platform has unique script specifications
  - [ ] No "adapt for all platforms" instructions (violates Law 5)
```

**CRITICAL:** If requests indicate cross-posting → FAIL → explain platform-native requirement

---

**Format:**

```yaml
  requests:
    - request_id: "script_1"
      content_title: "[title]"
      platform: "[platform]"
      format: "[format]"
      validation: PASS/FAIL
      issues: [] # if any

strategic_alignment:
  pillars_aligned: true/false
  platforms_aligned: true/false
  functions_distributed: true/false

platform_native_verified: true/false

blocking_issues: []
warnings: []

ready_to_proceed: true/false
```

**Minimum size:** 1KB

---

## Quality Gates

- [ ] CBF exists and validated
- [ ] All script requests have required fields
- [ ] Minimum 3 script requests
- [ ] Platform-native verified (no cross-posting)
- [ ] Strategic alignment confirmed
- [ ] No blocking issues

**If ANY gate fails → validation_status: FAIL**

---

## Handoff

Pass validation results to: **0.2 Teaching Loader**

**CRITICAL:** If validation_status is FAIL, HALT execution. Do not proceed to 0.2.
