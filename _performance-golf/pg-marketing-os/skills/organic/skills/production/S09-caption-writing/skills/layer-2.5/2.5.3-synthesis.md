# 2.5.3 Synthesis

**Layer:** 2.5 (Arena — Synthesis & Human Selection)
**Model:** Claude Opus
**Upstream:** 2.5.2 Adversarial Critique
**Downstream:** Layer 4 (Final Packaging)

---

## PURPOSE

Generate 3+ hybrid variants by synthesizing strongest elements from top-scoring Arena variants. Present hybrids to human for final selection. Create ARENA_COMPLETE checkpoint file with selected caption for downstream handoff.

---

## INPUT REQUIREMENTS

```yaml
required_inputs:
  adversarial_critique: "outputs/layer-2.5/2.5.2-adversarial-critique.md"
  top_3_variants:
    - Brand Purist (8.95)
    - Virality Engineer (8.90)
    - Storyteller (8.85)
  synthesis_recommendations: "3 hybrid approaches from critique"
```

---

## EXECUTION PROTOCOL

**MUST:**
1. Load **critique results** from 2.5.2 (top 3 variants + synthesis recommendations)
2. Generate **3 hybrid variants** pulling strongest elements:
   - **Hybrid A:** Recommended combination 1
   - **Hybrid B:** Recommended combination 2
   - **Hybrid C:** Recommended combination 3
3. Each hybrid MUST:
   - Synthesize elements from 2+ Arena variants
   - Maintain BVF voice (non-negotiable)
   - Stay within platform character limits
   - Document which elements came from which persona
4. Score each hybrid on same criteria (Hook/Voice/Platform/Engagement)
5. Present hybrids to human with:
   - Full caption text (copy-paste ready)
   - Synthesis rationale (what came from where)
   - Predicted strengths
   - Character count
6. **WAIT FOR HUMAN SELECTION** (blocking gate)
7. After human selects, create **ARENA_COMPLETE checkpoint** with:
   - Selected hybrid caption
   - Arena process summary
   - Final scores
   - Human selection rationale (if provided)

**NEVER:**
- Auto-select hybrid without human approval (HUMAN_SELECT gate is mandatory)
- Generate hybrids that abandon BVF voice
- Exceed platform character limits
- Skip synthesis rationale (must document element sources)
- Proceed to Layer 4 without human selection
