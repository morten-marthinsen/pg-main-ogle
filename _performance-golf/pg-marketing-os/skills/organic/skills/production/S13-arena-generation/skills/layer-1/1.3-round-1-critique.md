# Microskill 1.3: Round 1 Critique
Skill: S13 Arena Generation | Layer: 1 | Model: claude-opus-4.6 | Version: 1.0

## Purpose
The Critic challenges Round 1 evaluations, identifies convergence, surfaces blind spots, forces differentiation.

## Input
s13-1.2-round-1-scores.yaml

## Process
### MUST
1. Analyze Round 1 scores for convergence (scores within 2 points = convergence)
2. Identify blind spots in evaluations
3. Challenge weak rationales
4. Force personas to defend or revise
5. Set severity (0-10): convergence detected = 8-9/10 severity

### NEVER
- Skip Critic challenge
- Provide generic critique
- Set low severity when convergence detected
- Allow personas to proceed unchallenged

## Output
**File:** `s13-1.3-round-1-critique.yaml`

```yaml
critic_challenge_round_1:
  timestamp: 2026-03-05T15:15:00Z

  convergence_analysis:
    detected: true
    pattern: "All scores 7-8, tight clustering"
    severity: 9

  challenge:
    summary: "[Specific challenge statement]"
    blind_spots: [list of evaluation gaps]
    forced_questions:
      - "Volume Machine: sustainability at scale not addressed"
      - "Virality Engineer: share triggers not evaluated"
      # [... targeted challenges per persona ...]

  escalation: "FORCE RE-EVALUATION with differentiation requirement"
```

## Quality Gates
- [ ] Convergence analyzed
- [ ] Blind spots identified
- [ ] Specific challenges per persona
- [ ] Severity appropriate
- [ ] File written

## Handoff
Downstream: 1.4 round-2-competition responds to critique
