# 2.5.1 Arena Submission

## Purpose
Submit the pre-Arena thread to all 7 organic personas for competitive evaluation and improvement.

---

## Input Requirements

From Layer 0:
- `0.4-cbf-context.yaml` (campaign context)

From Layer 2:
- `2.4-formatted-thread.yaml` (complete formatted thread)

---

## Execution Protocol

### Step 1: Prepare Arena Brief

Create briefing document for all personas:

```yaml
arena_brief:
  content_type: "Thread"
  platform: "[X_TWITTER or LINKEDIN]"
  function: "[Awareness/Engagement/Conversion/Community]"
  campaign_reference: "[Campaign name]"

  audience_context:
    segment: "[From CBF]"
    key_pain: "[From CBF]"
    key_desire: "[From CBF]"
    language_notes: "[Key vocabulary]"

  voice_context:
    tone: "[From BVF]"
    energy: "[Level]"
    avoid: "[Anti-voice]"

  content_request:
    topic: "[Thread topic]"
    key_message: "[Core message]"
    desired_action: "[What we want]"
    constraints:
      tweet_count: [N]
      platform: "[Platform]"
      character_limit: [280 or 3000]
```

### Step 2: Round 1 — Independent Generation

Each of 7 personas generates their version:

**7 Organic Personas:**
1. Volume Machine
2. Value Architect
3. Virality Engineer
4. Community Builder
5. Brand Purist
6. Algorithm Hacker
7. Storyteller

**Per Persona Output (Round 1):**
```yaml
persona: "[Name]"
round: 1
arena_mode: "generative_full_draft"

strategic_rationale: |
  [2-3 sentences on why this approach from persona's philosophy]

hook_approach:
  text: |
    [Tweet 1]
  type: "[Hook type]"
  why_chosen: "[Reasoning]"

thread_structure:
  opening: "[How it starts]"
  body: "[How content flows]"
  close: "[How it ends]"

full_thread: |
  Tweet 1:
  [Text]

  Tweet 2:
  [Text]

  # ... all tweets

  Tweet N:
  [Text]

predicted_performance:
  virality_estimate: [0-100]
  primary_strength: "[What this does well]"
  known_weakness: "[Where this falls short]"
```

**Repeat for all 7 personas.**
