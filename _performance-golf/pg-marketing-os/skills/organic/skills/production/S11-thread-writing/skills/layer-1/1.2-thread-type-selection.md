# 1.2 Thread Type Selection

## Purpose
Select the optimal thread type (listicle, story, breakdown, framework, prediction, curation) based on topic, audience need, and content function.

---

## Input Requirements

From Layer 0:
- `0.3-specimen-library.yaml` (thread type patterns)
- `0.4-cbf-context.yaml` (audience pains, desires, content function)
- Thread brief (topic, key_points)

From Layer 1:
- `1.1-platform-analysis.yaml` (platform requirements)

---

## Execution Protocol

### Step 1: Analyze Thread Types Against Topic

Evaluate 6 thread types:

```yaml
thread_type_analysis:
  listicle_thread:
    description: "Numbered insights, tips, or lessons"
    best_for: ["Tips", "Lessons learned", "Mistakes to avoid"]
    fit_score: [1-10]
    fit_rationale: |
      [Why this fits or doesn't fit the topic]

  story_thread:
    description: "Narrative arc with lessons"
    best_for: ["Personal stories", "Case studies", "Transformations"]
    fit_score: [1-10]
    fit_rationale: |

  breakdown_thread:
    description: "Analysis of specific example"
    best_for: ["Case studies", "Strategy analysis", "Teardowns"]
    fit_score: [1-10]
    fit_rationale: |

  framework_thread:
    description: "Teaching a methodology"
    best_for: ["Systems", "Processes", "Methodologies"]
    fit_score: [1-10]
    fit_rationale: |

  prediction_thread:
    description: "Forward-looking analysis"
    best_for: ["Thought leadership", "Industry trends", "Hot takes"]
    fit_score: [1-10]
    fit_rationale: |

  curation_thread:
    description: "Best resources, tools, examples"
    best_for: ["Resources", "Tools", "Examples"]
    fit_score: [1-10]
    fit_rationale: |
```

### Step 2: Map Type to Audience Need

Connect thread type to pain/desire from CBF:

```yaml
need_mapping:
  primary_pain: "[Pain from CBF]"

  types_addressing_pain:
    - type: "[Thread type]"
      addresses_how: |
        [How this type solves the pain]

  primary_desire: "[Desire from CBF]"

  types_fulfilling_desire:
    - type: "[Thread type]"
      fulfills_how: |
        [How this type delivers the desire]
```

### Step 3: Select Thread Type

Choose based on highest combined score:

```yaml
thread_type_selection:
  selected_type: "[Thread type]"

  selection_rationale:
    topic_fit: "[Why this type for this topic]"
    audience_fit: "[Why this type for this audience]"
    platform_fit: "[Why this type for this platform]"
    content_function_fit: "[Why this type for Awareness/Engagement/Conversion/Community]"

  structure_outline:
    opening:
      tweets: 1-2
      purpose: "[What opening achieves]"
      content: "[Hook + context/credibility]"

    body:
      tweets: [N]
      purpose: "[What body achieves]"
      content: "[Individual points/story beats/analysis]"

    close:
      tweets: 1
      purpose: "[What close achieves]"
      content: "[Summary + CTA]"

  estimated_tweet_count: [5-15]
```

### Step 4: Define Type-Specific Requirements

Based on selected type:

```yaml
type_requirements:
  hook_template: "[Specific hook pattern for this type]"

  body_structure:
    - element: "[What each body tweet should contain]"
    - element: "[What each body tweet should contain]"

  transition_strategy: |
    [How tweets connect in this type]

  cta_alignment: |
    [What CTA makes sense for this type]
```
