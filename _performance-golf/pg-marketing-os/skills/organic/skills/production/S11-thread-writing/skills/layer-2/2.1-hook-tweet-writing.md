# 2.1 Hook Tweet Writing

## Purpose
Write tweet 1 (the hook) — the most critical tweet that stops scrolls and earns the thread read.

---

## Input Requirements

From Layer 1:
- `1.2-thread-type-selection.yaml` (thread type, hook template)
- `1.3-hook-tweet-strategy.yaml` (hook requirements, amplifiers, tests)
- `1.4-thread-count-planning.yaml` (final tweet count)

From Layer 0:
- `0.4-cbf-context.yaml` (voice constraints)

---

## Execution Protocol

### Step 1: Apply Hook Template

Use template from thread type:

```
Template: [From 1.2]
Topic: [Thread topic]

Draft Hook (Template Application):
[Apply template to specific topic]
```

### Step 2: Apply Amplifiers

Layer in amplification techniques:

```
Base Hook: [Template application]

+ Specificity: [Add specific numbers/details]
+ Time Frame: [Add temporal urgency if applicable]
+ Credentials: [Add authority signal if applicable]
+ Contrarian: [Add contrarian angle if applicable]

Amplified Hook:
[Hook with amplifiers]
```

### Step 3: Voice Alignment

Apply voice constraints from BVF:

```
Amplified Hook: [Version with amplifiers]

Voice Adjustments:
- Tone: [Apply tone from BVF]
- Energy: [Adjust energy level]
- Power Words: [Incorporate if natural]
- Anti-Voice Check: [Remove any banned patterns]

Voice-Aligned Hook:
[Final hook with voice applied]
```

### Step 4: 5-Test Validation

Run all validation tests:

```yaml
hook_validation:
  hook_text: |
    [Final hook tweet]

  character_count: [N/280 or N/3000]

  test_results:
    standalone_test:
      passes: [Yes/No]
      notes: "[Can this be understood without tweet 2?]"

    scroll_stop_test:
      passes: [Yes/No]
      notes: "[Does this create curiosity or desire?]"

    curiosity_gap_test:
      passes: [Yes/No]
      notes: "[Must reader continue to close the gap?]"

    clarity_test:
      passes: [Yes/No]
      notes: "[Is value proposition clear?]"

    retweet_test:
      passes: [Yes/No]
      notes: "[Standalone RT-worthy?]"

  tests_passed: [N/5]
  status: "[PASS if 5/5, FAIL otherwise]"
```

**If status = FAIL:** Regenerate hook, adjusting failing elements.

### Step 5: Alternative Hooks (Optional)

Generate 2-3 alternative hooks for Arena:

```yaml
hook_variations:
  primary_hook: |
    [Main hook]

  alternative_1: |
    [Different angle/approach]
  alternative_1_rationale: "[Why this might work better]"

  alternative_2: |
    [Different angle/approach]
  alternative_2_rationale: "[Why this might work better]"
```
