# 2.3 CTA Tweet Writing

## Purpose
Write the final tweet with clear call-to-action that matches content function and converts attention to action.

---

## Input Requirements

From Layer 0:
- `0.4-cbf-context.yaml` (content function, voice)
- Thread brief (cta_goal)

From Layer 2:
- `2.1-hook-tweet-draft.yaml` (hook reference for loop close)
- `2.2-body-tweets-draft.yaml` (content delivered)

---

## Execution Protocol

### Step 1: Map Content Function to CTA Type

Select appropriate CTA based on content function:

```yaml
cta_type_mapping:
  content_function: "[Awareness/Engagement/Conversion/Community]"

  cta_type_options:
    awareness:
      - "Share/RT the first tweet"
      - "Follow for more"
      - "Bookmark for later"

    engagement:
      - "Reply with [question]"
      - "Which point hit hardest?"
      - "Quote tweet with your take"

    conversion:
      - "DM me [keyword] for [resource]"
      - "Link in bio for [offer]"
      - "Book a call [link]"

    community:
      - "Tag someone who needs this"
      - "Join [community] for more"
      - "Follow for daily [topic] threads"

  selected_cta_type: "[Type]"
  rationale: "[Why this CTA for this function]"
```

### Step 2: CTA Structure

Build the final tweet:

```yaml
cta_structure:
  optional_summary:
    include: [Yes/No]
    format: "TLDR: [Key points]" or "Recap: [Main takeaway]"
    text: |
      [If included]

  primary_cta:
    action: "[Specific action]"
    text: |
      [CTA text]

  optional_loop_close:
    include: [Yes/No]
    reference: "[Callback to hook]"
    text: |
      [If included]

  combined_tweet: |
    [Full final tweet]
```

### Step 3: CTA Strength Analysis

Evaluate CTA quality:

```yaml
cta_evaluation:
  clarity:
    score: [1-10]
    assessment: "[Is action crystal clear?]"

  ease_of_action:
    score: [1-10]
    assessment: "[How easy is it to take action?]"

  value_justification:
    score: [1-10]
    assessment: "[Does thread value justify the ask?]"

  alignment:
    score: [1-10]
    assessment: "[Does CTA match content function from CBF?]"

  total_score: [Sum/40]
  minimum_acceptable: 28/40

  status: "[PASS if ≥28, FAIL otherwise]"
```

**If status = FAIL:** Revise CTA addressing low-scoring dimensions.

### Step 4: Voice Alignment

Apply voice constraints:

```yaml
voice_check:
  cta_text: |
    [CTA tweet]

  tone_match: "[Does tone match BVF?]"
  energy_match: "[Does energy match BVF?]"
  anti_voice_clear: "[No banned patterns?]"

  adjustments_needed: [None OR list]

  voice_aligned_cta: |
    [Final CTA with voice applied]
```

---

## Output

**File:** `layer-2/2.3-cta-tweet-draft.yaml`

**Format:**
```yaml
timestamp: "[ISO 8601]"

cta_type_mapping:
  content_function: "[Function]"
  selected_cta_type: "[Type]"
  rationale: |

cta_structure:
  optional_summary:
    include: [Yes/No]
    text: |

  primary_cta:
    action: "[Action]"
    text: |

  optional_loop_close:
    include: [Yes/No]
    text: |

final_cta_tweet:
  tweet_number: [N]
  content: |
    [Complete final tweet]

  character_count: [N/280 or N/3000]

cta_evaluation:
  clarity: [Score + assessment]
  ease_of_action: [Score + assessment]
  value_justification: [Score + assessment]
  alignment: [Score + assessment]
  total_score: [N/40]
  status: PASS

voice_check:
  tone_match: true
  energy_match: true
  anti_voice_clear: true
  adjustments_needed: []
```

**Minimum size:** 600 bytes

---

## Quality Gates

- [ ] CTA type matches content function
- [ ] Action is specific and clear
- [ ] Thread value justifies ask
- [ ] CTA evaluation score ≥28/40
- [ ] Voice aligned
- [ ] Character count under limit
