# 0.4 CBF Loader

## Purpose
Load and extract relevant context from Campaign Brief File (CBF) to inform thread content, voice, and strategy.

---

## Input Requirements

From Layer 0:
- `0.1-input-validation.yaml` (CBF path confirmed)
- Thread brief with pillar and content_function

---

## Execution Protocol

### Step 1: Load CBF

Read Campaign Brief File:
```yaml
cbf_path: "[project]/S07-campaign-brief/outputs/[campaign]-CBF.yaml"
```

### Step 2: Extract Relevant Sections

Pull thread-relevant context:

```yaml
cbf_context:
  campaign_identity:
    campaign_name: "[Name]"
    brand_name: "[Brand]"
    vertical: "[Vertical]"

  audience_intelligence:
    primary_segment: "[Segment]"
    key_pains:
      - "[Pain 1]"
      - "[Pain 2]"
      - "[Pain 3]"
    key_desires:
      - "[Desire 1]"
      - "[Desire 2]"
    language_they_use:
      - "[Phrase]"
      - "[Phrase]"

  brand_voice:
    tone: "[Tone]"
    energy_level: "[1-10]"
    power_words: []
    banned_words: []
    anti_voice: |
      [What to avoid]

  content_architecture:
    pillar: "[Relevant pillar]"
    function: "[Content function]"
    series: "[If part of series]"

  platform_strategy:
    target_platform: "[Platform]"
    optimal_posting_times: []
    engagement_strategy: |
```

### Step 3: Map Thread Brief to CBF Context

Connect thread brief to campaign strategy:

```yaml
thread_mapping:
  thread_topic: "[From brief]"
  maps_to_pillar: "[Pillar name from CAF]"
  maps_to_pain: "[Which pain this addresses]"
  maps_to_desire: "[Which desire this fulfills]"

  voice_constraints:
    tone: "[From BVF via CBF]"
    energy: "[Level]"
    must_use: ["[Power words]"]
    must_avoid: ["[Banned words]"]

  funnel_context:
    content_function: "[From brief]"
    expected_action: "[From thread_goal]"
    downstream_asset: "[What comes next]"
```

### Step 4: Extract Hook Library Options

If CBF contains Hook Library File (HLF) reference:

```yaml
hook_library:
  available_hooks:
    - hook_id: "HOOK_001"
      type: "[Hook type]"
      text: "[Hook text]"
      performance_prediction: [score]

    - hook_id: "HOOK_002"
      type: "[Hook type]"
      text: "[Hook text]"
      performance_prediction: [score]

  recommended_for_thread: ["HOOK_001", "HOOK_003"]
  rationale: "[Why these hooks fit thread format]"
```
