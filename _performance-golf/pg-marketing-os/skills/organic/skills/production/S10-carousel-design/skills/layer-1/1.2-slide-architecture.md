# 1.2 Slide Architecture

## Purpose
Design the structural sequence and narrative arc across carousel slides.

## Input Requirements
- Carousel brief (topic, key_points)
- Platform requirements
- Teaching context (SUCCESs, STEPPS)
- Specimen patterns

## Execution Protocol

**MUST:**
- Plan complete slide sequence: cover slide (1) + body slides (n-2) + CTA slide (1)
- Assign one concept per body slide (no multi-concept slides)
- Design narrative arc: hook → build → payoff → CTA
- Apply "screenshot test" — each slide must stand alone
- Select template type from 5 options:
  1. Educational Listicle (numbered tips/lessons)
  2. How-To Guide (step-by-step process)
  3. Myth-Busting (misconception → truth)
  4. Framework Breakdown (teach a system)
  5. Before/After (transformation story)
- Design information hierarchy per slide (headline → supporting points → visual)
- Plan layout variety across slides (avoid visual monotony)
- Ensure practical value density (STEPPS principle)

**NEVER:**
- Cram multiple concepts into one slide
- Create slides that require previous context to understand
- Use same layout for all slides
- Exceed platform optimal slide count
- Skip narrative arc design

## Output
**Filename:** `slide-architecture.yaml`

**Format:**
```yaml
template_type: string (one of 5)
total_slide_count: number
narrative_arc: string (description)
slide_sequence:
  - slide_number: 1
    type: "cover"
    concept: string
    function: "hook + curiosity gap"
    layout_type: string
  - slide_number: 2-N
    type: "body"
    concept: string
    function: string
    layout_type: string
  - slide_number: N
    type: "cta"
    concept: string
    function: "engagement prompt"
    layout_type: string
information_hierarchy_rules: [list]
screenshot_test_criteria: [list]
timestamp: ISO8601
```

**Minimum size:** 2KB

## Quality Gates
- Template type selected and justified
- Slide count within platform optimal range
- Each slide has single clear concept
- Layout variety planned (minimum 3 different layouts)
- Narrative arc described

## Handoff
Provides structural blueprint to all Layer 2 writing microskills (2.1, 2.2, 2.3).
