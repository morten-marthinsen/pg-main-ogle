# 0.3 Specimen Loader

## Purpose
Load high-performing carousel specimens from target platform to inform design decisions.

## Input Requirements
- Validation report with platform specification
- Access to specimen library for organic content

## Execution Protocol

**MUST:**
- Load minimum 5 high-performing carousels from target platform
- Prioritize specimens with documented save rate >10%
- Filter by platform match (Instagram vs LinkedIn)
- Extract structural patterns: slide count, slide progression, hook style, CTA approach
- Document visual direction patterns: layout variety, color usage, typography hierarchy
- Note screenshot-worthy formatting techniques
- Identify engagement-driving elements per specimen

**NEVER:**
- Load specimens from wrong platform
- Include carousels without performance data
- Skip structural pattern extraction
- Load fewer than 5 specimens

## Output
**Filename:** `specimen-analysis.yaml`

**Format:**
```yaml
specimens_loaded: 5+
platform: instagram | linkedin
patterns_extracted:
  slide_count_range: [min, max]
  common_templates: [list]
  hook_patterns: [list]
  visual_patterns: [list]
  cta_patterns: [list]
per_specimen:
  - specimen_id: string
    save_rate: number
    slide_count: number
    template_type: string
    key_elements: [list]
    why_it_works: string
screenshot_techniques: [list]
timestamp: ISO8601
```

**Minimum size:** 3KB

## Quality Gates
- Minimum 5 specimens loaded
- All specimens from correct platform
- Structural patterns extracted for each
- Save rate documented where available

## Handoff
Provides pattern library to 1.2 Slide Architecture and 2.4 Visual Direction.
