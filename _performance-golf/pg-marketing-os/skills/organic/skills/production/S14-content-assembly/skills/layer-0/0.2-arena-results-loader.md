# 0.2 Arena Results Loader

## Purpose
Load Arena results and selections from S13, extracting which content versions were selected and where they're located. This is critical for Arena selection verification in Layer 1.

---

## Input Requirements

From Layer 0:
- `0.1-input-validation.yaml` (validation must have passed)

---

## Execution Protocol

### Step 1: Load Arena Checkpoint

Load the Arena checkpoint file:
```
[project]/S13-arena-generation/checkpoints/ARENA_COMPLETE.yaml
```

**Extract:**
```yaml
skill: "S13-arena-generation"
status: COMPLETE
timestamp: "[When Arena completed]"

arena_results:
  - content_id: "[ID]"
    arena_selection_verified: true
    selected_hybrid: "[Hybrid identifier, e.g., 'Hybrid A']"
    hybrid_path: "[Full path to Arena-selected content]"
    selection_rationale: "[Why this hybrid was selected]"
    arena_round_count: [Should be 3]

human_selection:
  selected_by: "[Name]"
  selection_date: "[Date]"
  selection_notes: "[Any guidance]"
```

---

### Step 2: Validate Arena Checkpoint Completeness

For each content piece in CBF:
- Verify corresponding entry in arena_results
- Verify arena_selection_verified = true
- Verify selected_hybrid is defined (not null/empty)
- Verify hybrid_path exists and is valid file path

**If ANY content piece missing from arena_results:**
- Log error
- Mark as INCOMPLETE
- HALT (cannot proceed without Arena verification)

---

### Step 3: Load Arena-Selected Content Files

For each arena_results entry:
1. Read file at hybrid_path
2. Extract full content (hook, body, CTA for scripts; full text for captions/threads)
3. Calculate content hash (for comparison in Layer 1)
4. Store content and hash for verification

**File reading protocol:**
```yaml
- content_id: "[ID]"
  hybrid_identifier: "[Hybrid A/B/C/etc]"
  hybrid_path: "[Path]"
  file_accessible: [Yes/No]
  content_hash: "[SHA256 or similar]"
  content_preview: "[First 200 characters]"
  word_count: [Number]
```

**If ANY file inaccessible:**
- Log error with path
- Mark as MISSING
- HALT (this is CRITICAL failure)

---

### Step 4: Extract Arena Metadata

For each content piece, extract Arena competition details:

```yaml
arena_metadata:
  - content_id: "[ID]"
    rounds_completed: [Number, should be 3]
    personas_competed: [List of 7 personas]
    winning_elements:
      - persona: "[Persona name]"
        contribution: "[What this persona contributed to hybrid]"
    arena_synthesis_notes: "[Overall synthesis rationale]"
```

This metadata will be used for quality verification and documentation.

---

### Step 5: Cross-Reference with Production Outputs

Verify Arena-selected content corresponds to actual production output structure:

For video content:
- Arena selection should have: hook, body, CTA sections
- Length should match script requirements from PSF

For captions:
- Arena selection should match caption structure
- Character count should meet platform limits

For threads:
- Arena selection should have correct tweet/post count
- Each post should meet character limits

**Flag any structural mismatches for human review.**

---

## Output

**File:** `layer-0/0.2-arena-results.yaml`

**Format:**
```yaml
timestamp: "[ISO 8601]"
checkpoint_loaded: [Yes/No]
checkpoint_path: "[Path to ARENA_COMPLETE.yaml]"

arena_results:
  - content_id: "[ID]"
    arena_selection_verified: true
    selected_hybrid: "[Hybrid identifier]"
    hybrid_path: "[Path]"
    content_hash: "[Hash for verification]"
    content_preview: |
      [First 200 characters of Arena-selected content]
    word_count: [Number]
    character_count: [Number]
    arena_metadata:
      rounds_completed: [3]
      personas_competed: []
      winning_elements: []
      selection_rationale: |

human_selection:
  selected_by: "[Name]"
  selection_date: "[Date]"
  notes: |

validation:
  all_content_pieces_have_arena_results: [Yes/No]
  all_hybrid_files_accessible: [Yes/No]
  all_arena_selection_verified_flags_true: [Yes/No]
  structural_matches_production_requirements: [Yes/No]

errors: []
warnings: []

ready_for_verification: [Yes/No]
```

**Minimum size:** 2KB (more with multiple content pieces)

---

## Quality Gates

- [ ] Arena checkpoint exists and loads successfully
- [ ] All content pieces have arena_results entries
- [ ] All arena_selection_verified flags = true
- [ ] All hybrid files accessible and readable
- [ ] Content hashes generated for all Arena selections
- [ ] Structural validation passed

**If ANY gate fails:** ready_for_verification = No. HALT.

---

## Handoff

Pass Arena results to: **0.3 Production Outputs Loader** and **1.1 Arena Selection Verification** (Layer 1)

**CRITICAL:** This data is the source of truth for what should be in the final assembly. Layer 1 will compare production outputs against these Arena selections.
