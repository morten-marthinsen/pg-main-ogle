# 4.1 Content Package Assembler

## Purpose
Assemble all verified content into a master content package file. This is the final integration of all production elements, ready for distribution.

---

## Input Requirements

From Layer 0:
- `0.2-arena-results.yaml` (Arena-selected content)
- `0.3-production-outputs.yaml` (all content files)
- `0.4-cbf-context.yaml` (campaign context)

From Layer 1:
- `1.1-arena-verification.yaml` (verification status)
- `1.2-content-inventory.yaml` (content list)
- `1.3-virality-scores.yaml` (quality scores)
- `1.4-platform-compliance.yaml` (compliance status)
- `1.5-voice-consistency.yaml` (voice status)
- `1.6-cross-reference-validation.yaml` (assembly readiness)

**CRITICAL PREREQUISITE:** `1.6-cross-reference-validation.yaml` must have `ready_for_assembly: true`. If false, DO NOT execute this microskill.

---

## Execution Protocol

### Step 1: Verify Assembly Readiness

Check Layer 1 validation results:

```yaml
assembly_readiness_check:
  layer_1_complete: [Yes/No from checkpoint]
  ready_for_assembly: [Yes/No from 1.6]
  blocking_issues_count: [Number from 1.6]
```

**If ready_for_assembly = No:**
- Log error: "Assembly blocked by Layer 1 issues"
- List blocking issues
- HALT execution
- Return to Layer 1 to resolve issues

**If ready_for_assembly = Yes:**
- Proceed to Step 2

---

### Step 3: Include Arena Selection References

For transparency and audit trail, include Arena information:

```yaml
arena_selection_audit:
  - content_id: "[ID]"
    arena_date: "[When Arena ran]"
    selected_hybrid: "[Hybrid identifier]"
    hybrid_path: "[Full path to Arena-selected file]"
    selection_rationale: "[From Arena checkpoint]"
    human_selector: "[Name]"
    verification_hash: "[Content hash for integrity check]"
```

This allows future verification that correct versions were used.

## Output

**File:** `outputs/[campaign]-content-package.yaml`

**Format:**
Complete YAML file with all sections from Steps 2-5.

**Minimum size:** 10KB+ (depends on content count, aim for 5KB+ per content piece)

**CRITICAL:** This file is the SINGLE SOURCE OF TRUTH for what content exists, what's been verified, and what's ready for distribution.

---

## Quality Gates

- [ ] All content pieces from inventory included
- [ ] All Arena-selected versions used (verified via hash)
- [ ] All quality verification data included
- [ ] All distribution plans complete
- [ ] Arena selection audit trail included
- [ ] Quality gates summary included
- [ ] Distribution handoff metadata included
- [ ] File size >= 10KB (substantive content)

**If any content piece missing required sections:** Assembly incomplete. HALT.

---

## Handoff

Pass content package to: **4.2 Per-Platform File Creation**

**CRITICAL:** This master file is the authoritative record. Per-platform files (4.2) are derived from this master.
