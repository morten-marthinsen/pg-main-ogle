# 0.4 CBF Loader

## Purpose
Load Campaign Brief File (CBF) and extract all context needed for assembly verification, including virality thresholds, campaign objectives, content calendar, and success criteria.

---

## Input Requirements

From Layer 0:
- `0.1-input-validation.yaml` (CBF path verification)
- `0.3-production-outputs.yaml` (content inventory for mapping)

---

## Execution Protocol

### Step 1: Load CBF File

Load the Campaign Brief File:
```
[project]/S07-campaign-brief/outputs/[campaign]-CBF.yaml
```

**Full CBF structure expected:**
```yaml
campaign_name: "[Name]"
brand_name: "[Brand]"
date_created: "[ISO 8601]"
version: "[Version]"

campaign_objectives:
  primary_goal: "[Goal]"
  secondary_goals: []
  success_metrics:
    - metric: "[Name]"
      target: "[Number]"

minimum_virality_score: [Number]  # CRITICAL for Gate G09

content_requirements:
  content_pieces:
    - content_id: "[ID]"
      content_type: "[Type]"
      platform: "[Platform]"
      content_function: "[Awareness/Engagement/Conversion/Community]"
      pillar: "[From CAF]"
      target_publish_date: "[Date]"
      minimum_virality_score_override: [Optional, if different from campaign default]

  content_calendar:
    week_1:
      - content_id: "[ID]"
        day: "[Day of week]"
        time: "[Time]"
    # [... more weeks]

  platform_strategy:
    [platform]:
      posting_frequency: "[Frequency]"
      optimal_times: []
      format_preferences: []

brand_voice_requirements:
  tone_expectations: []
  banned_phrases: []
  voice_energy: "[Level]"

quality_thresholds:
  minimum_virality_score: [Number, duplicate of above for clarity]
  minimum_hook_strength: [0-10]
  minimum_platform_fit: [0-10]
  anti_slop_density_max: [Percentage, e.g., 5]
```

---

### Step 2: Extract Virality Thresholds

**Global threshold:**
```yaml
virality_thresholds:
  campaign_minimum: [Number from CBF]
  per_content_overrides:
    - content_id: "[ID]"
      override_score: [Number, if present]
      rationale: "[Why different]"
```

**This data is CRITICAL for Gate G09 (Virality Threshold).**

**If minimum_virality_score missing from CBF:**
- Use default: 60
- Log warning
- Recommend CBF update

---

### Step 3: Map Content Requirements

For each content piece in production outputs (from 0.3):
1. Find corresponding entry in CBF content_requirements
2. Extract objectives, function, pillar, schedule
3. Flag any content in production NOT in CBF (orphaned content)
4. Flag any content in CBF NOT in production (missing content)

```yaml
content_mapping:
  - content_id: "[ID]"
    in_cbf: [Yes/No]
    in_production: [Yes/No]
    objectives_defined: [Yes/No]
    schedule_defined: [Yes/No]
    pillar_assigned: [Yes/No]
    status: [COMPLETE / PARTIAL / MISSING]
```

**If orphaned content found:** Flag for human review (why was this produced?).
**If missing content found:** CRITICAL error — CBF expects content not produced.

---

### Step 4: Extract Platform Requirements

For each platform in production:
```yaml
platform_requirements:
  [platform]:
    posting_frequency: "[From CBF]"
    optimal_times: []
    format_preferences: []
    content_count_expected: [Number]
    content_count_produced: [Number]
    gap: [Difference]
```

**If content_count_produced < content_count_expected:**
- Flag as INCOMPLETE
- Log missing content pieces

---

### Step 5: Extract Voice Requirements

```yaml
voice_requirements:
  tone_expectations: []
  banned_phrases: []
  voice_energy: "[Level]"
  anti_voice_patterns: []
```

This will be used in Layer 1 (1.5 Voice Consistency Check).

---

### Step 6: Build Campaign Context

Assemble high-level campaign context for reference:

```yaml
campaign_context:
  campaign_name: "[Name]"
  brand_name: "[Brand]"

  strategic_focus:
    primary_goal: "[Goal]"
    content_functions_distribution:
      awareness: "[X%]"
      engagement: "[Y%]"
      conversion: "[Z%]"
      community: "[W%]"

  timeline:
    campaign_start: "[Date]"
    campaign_end: "[Date]"
    total_weeks: [Number]
    content_pieces_planned: [Number]
    content_pieces_produced: [Number]

  quality_standards:
    minimum_virality_score: [Number]
    minimum_hook_strength: [Number]
    minimum_platform_fit: [Number]
    anti_slop_max: [Percentage]
```

---

## Quality Gates

- [ ] CBF loads successfully
- [ ] Virality thresholds extracted (campaign minimum at minimum)
- [ ] All production content mapped to CBF entries
- [ ] No critical missing content (all CBF content produced)
- [ ] Platform requirements extracted
- [ ] Voice requirements extracted for 1.5 check

**If missing_content is non-empty:** ready_for_layer_1 = No. HALT.
**If orphaned_content is non-empty:** Log warning, proceed (can assemble but flag for review).

---

## Handoff

Pass CBF context to: **Layer 1 Verification** (all 6 microskills will reference this data)

**CRITICAL DATA FOR DOWNSTREAM:**
- `virality_thresholds.campaign_minimum` → used in 1.3 Virality Score Check
- `voice_requirements` → used in 1.5 Voice Consistency Check
- `platform_requirements` → used in 1.4 Platform Compliance Check
- `content_mapping` → used in 1.6 Cross-Reference Validation

**Create checkpoint:**
```yaml
# checkpoints/LAYER_0_COMPLETE.yaml
skill: "S14-content-assembly"
layer: 0
status: COMPLETE
timestamp: "[ISO 8601]"
all_inputs_loaded: true
all_validations_passed: true
ready_for_layer_1: true
```
