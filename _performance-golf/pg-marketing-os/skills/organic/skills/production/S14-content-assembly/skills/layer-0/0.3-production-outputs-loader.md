# 0.3 Production Outputs Loader

## Purpose
Load all production outputs from S08-S13 and organize them for assembly. This creates a complete inventory of what content exists and where it's located.

---

## Input Requirements

From Layer 0:
- `0.1-input-validation.yaml` (content inventory)
- `0.2-arena-results.yaml` (Arena-selected versions)

---

## Execution Protocol

### Step 1: Load Scripts (S08 Outputs)

For each video content piece:

```yaml
scripts:
  - content_id: "[ID]"
    script_path: "[Path to script file from S08]"
    script_type: "[Reel/TikTok/YouTube Short/etc]"
    content:
      hook: |
      body: |
      cta: |
    metadata:
      word_count: [Number]
      estimated_duration_seconds: [Number]
      platform: "[Platform]"
    status: [LOADED / MISSING / ERROR]
```

**If script expected but missing:** Log error, mark content as INCOMPLETE.

---

### Step 2: Load Captions (S09 Outputs)

For ALL content pieces (captions are universal):

```yaml
captions:
  - content_id: "[ID]"
    caption_path: "[Path to caption file from S09]"
    platform: "[Platform]"
    content:
      hook: |
      body: |
      cta: |
      hashtags: []
    metadata:
      character_count: [Number]
      platform_limit: [Number]
      within_limit: [Yes/No]
    status: [LOADED / MISSING / ERROR]
```

**If caption missing for ANY content piece:** CRITICAL error. All content needs captions.

---

### Step 3: Load Carousels (S10 Outputs)

For each carousel content piece:

```yaml
carousels:
  - content_id: "[ID]"
    carousel_path: "[Path to carousel file from S10]"
    platform: "[Platform]"
    slides:
      - slide_number: [1]
        headline: |
        body: |
        visual_direction: |
      - slide_number: [2]
        # [... etc]
    metadata:
      slide_count: [Number]
      total_word_count: [Number]
    status: [LOADED / MISSING / ERROR]
```

**If carousel expected but missing:** Log error, mark content as INCOMPLETE.

---

### Step 4: Load Threads (S11 Outputs)

For each thread content piece:

```yaml
threads:
  - content_id: "[ID]"
    thread_path: "[Path to thread file from S11]"
    platform: "[Platform - X/LinkedIn/etc]"
    tweets:
      - tweet_number: [1]
        content: |
        character_count: [Number]
        media_notes: |
      - tweet_number: [2]
        # [... etc]
    metadata:
      tweet_count: [Number]
      total_character_count: [Number]
    status: [LOADED / MISSING / ERROR]
```

**If thread expected but missing:** Log error, mark content as INCOMPLETE.

---

### Step 5: Load Visual Direction (S12 Outputs)

For ALL content pieces (visual direction is universal):

```yaml
visual_direction:
  - content_id: "[ID]"
    visual_path: "[Path to visual direction file from S12]"
    thumbnail:
      concept: |
      composition: |
      text_elements: |
      technical_specs:
        dimensions: "[WxH]"
        format: "[PNG/JPG/etc]"
        text_size: "[Size]"
    supporting_visuals:
      - asset_type: "[B-roll/graphic/etc]"
        direction: |
    status: [LOADED / MISSING / ERROR]
```

**If visual direction missing for ANY content piece:** CRITICAL error. All content needs visuals.

---

### Step 6: Cross-Reference with Arena Results

For each loaded production output:
1. Find corresponding Arena result from 0.2
2. Generate content hash for production output
3. Compare production hash to Arena hash
4. Flag any mismatches (pre-Arena content detection)

```yaml
cross_reference:
  - content_id: "[ID]"
    production_hash: "[Hash of loaded production output]"
    arena_hash: "[Hash from 0.2-arena-results.yaml]"
    hashes_match: [Yes/No]
    mismatch_details: "[If No, describe difference]"
```

**If hashes DON'T match:** FLAG for verification in Layer 1 (1.1 microskill). This is potential pre-Arena content usage.

---

### Step 7: Organize by Platform

Group all loaded outputs by platform for easier assembly:

```yaml
platform_groups:
  instagram:
    - content_id: "[ID]"
      content_type: "[Reel/Carousel/etc]"
      all_files_present: [Yes/No]
  tiktok:
    - content_id: "[ID]"
      content_type: "[Video]"
      all_files_present: [Yes/No]
  # [... etc for each platform]
```

---

## Quality Gates

- [ ] All expected files loaded successfully
- [ ] All captions present (universal requirement)
- [ ] All visual direction present (universal requirement)
- [ ] Content hashes generated and compared to Arena hashes
- [ ] Platform grouping complete
- [ ] No CRITICAL errors (missing universal requirements)

**If CRITICAL files missing:** ready_for_layer_1 = No. HALT.

**If non-critical files missing:** Log warnings, proceed with flags.

---

## Handoff

Pass production outputs to: **0.4 CBF Loader** and **Layer 1 Verification** (all 6 microskills will reference this data)

**CRITICAL:** Any hash mismatches flagged here MUST be investigated in 1.1 Arena Selection Verification.
