# 4.2 Per-Platform File Creation

## Purpose
Create individual, ready-to-use files for each content piece per platform. One file per platform per content piece. These are the files that will be used during distribution (S15-S18).

---

## Input Requirements

From Layer 4:
- `4.1-content-package.yaml` (master assembly)

---

## Execution Protocol

### Step 1: Load Master Content Package

Load the assembled content package from 4.1:
```
outputs/[campaign]-content-package.yaml
```

Extract all content pieces.

---

### Step 2: Create Per-Platform Files

For each content piece:
1. Identify primary platform
2. Identify secondary platforms (if any)
3. Create one file per platform

**File naming convention:**
```
outputs/[platform]-[content-type]-[content-id].md
```

**Examples:**
- `instagram-reel-001.md`
- `tiktok-video-002.md`
- `x-thread-003.md`
- `linkedin-post-004.md`

---

### Step 4: Platform-Specific Adaptations

If content piece has secondary platforms, create adapted versions:

**For each secondary platform:**
1. Copy primary platform file
2. Adjust for platform-specific requirements:
   - Caption length adjustments
   - Hashtag adjustments
   - Format notes adjustments
   - Visual specs adjustments
3. Add adaptation notes section:

```markdown
## PLATFORM ADAPTATION NOTES

**Adapted From:** [Primary platform]
**Adaptation Type:** [Repurpose/Resize/Reformat]

**Changes Made:**
- [List of changes from primary version]
- [Why these changes needed for this platform]

**Platform-Specific Considerations:**
- [Anything unique to this platform]
```

## Output

**Multiple Files:**
1. One `.md` file per platform per content piece (format above)
2. `file-inventory.yaml` (tracking document)
3. `CONTENT-INDEX.md` (human-readable index)

**Output location:**
```
outputs/
├── [campaign]-content-package.yaml (from 4.1)
├── instagram-reel-001.md
├── instagram-carousel-002.md
├── tiktok-video-003.md
├── x-thread-004.md
├── linkedin-post-005.md
├── [... more platform files]
├── file-inventory.yaml
└── CONTENT-INDEX.md
```

**Minimum files:** 1 file per content piece (more if multi-platform)

---

## Quality Gates

- [ ] One file created per platform per content piece
- [ ] All files follow naming convention
- [ ] All files include complete content (script/caption/visuals)
- [ ] All files include distribution protocol
- [ ] All files include quality verification data
- [ ] File inventory matches expected count
- [ ] CONTENT-INDEX.md created and accurate
- [ ] All files saved to correct output directory

**If file count mismatch:** HALT. Investigate missing files.
