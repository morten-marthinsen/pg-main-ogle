# Microskill 2.1: Thumbnail Design Brief
Skill: S12 Visual Direction
Layer: 2
Model: claude-opus-4.6
Version: 1.0

## Purpose

Create the complete, actionable thumbnail design brief that a designer can execute pixel-perfect without clarification. This is the most critical asset brief — the thumbnail determines whether content gets consumed.

## Input

**Required gate:** `GATE_1_STRATEGY_COMPLETE`

**Required files:**
- `s12-1.2-thumbnail-strategy.md` — selected thumbnail concept
- `s12-1.3-style-guide.yaml` — visual approach
- `s12-1.4-color-typography.yaml` — precise specs
- `s12-1.1-platform-analysis.yaml` — platform requirements
- Content draft (for headline text)

## Process

### MUST
1. **Extract primary concept from strategy**
   - Read selected concept from 1.2
   - Document concept approach, scroll-stop mechanism, composition

2. **Define complete composition**
   - Layout: precise positioning of all elements (face, text, graphics, logo)
   - Visual hierarchy: what viewers see first, second, third
   - Balance: weight distribution across frame
   - Safe zones: enforce platform UI constraints

3. **Specify all visual elements**
   - Primary element (face, product, graphic): exact description, positioning, size
   - Secondary elements: supporting visuals, their role
   - Background: color, gradient, image (if image: specific direction)
   - Logo: placement, size, treatment

4. **Define text overlay precisely**
   - Headline text: exact wording from content
   - Font, size, weight, color: from typography system
   - Treatment: outline/shadow/background box specs
   - Position: coordinates or relative positioning
   - Readability verification: contrast check

5. **Document color usage**
   - Background color(s): hex codes
   - Text colors: hex codes with contrast ratios
   - Accent colors: where and how used
   - Overall color strategy: how colors create scroll-stop

6. **Provide reference imagery**
   - Reference images: similar compositions, styles, or moods
   - Specimen examples: high-performing thumbnails with similar approach
   - Avoid: what NOT to do (stock photos, overused patterns)

7. **Create production notes**
   - Designer instructions: specific guidance for execution
   - Dimensions: exact pixel dimensions (1280×720 for YouTube)
   - File format: JPG or PNG, file size constraints
   - Approval criteria: what makes this brief successfully executed

### NEVER
- Provide vague direction ("make it pop")
- Skip text overlay specs (most common failure point)
- Forget safe zones (results in UI-obscured elements)
- Omit reference imagery (designer needs visual anchor)
- Provide direction that contradicts brand guidelines

## Output

**File:** `s12-2.1-thumbnail-brief.yaml`

```yaml
thumbnail_design_brief:
  timestamp: 2026-03-05T16:05:00Z
  model: claude-opus-4.6
  content_id: "retirement-40-3bucket-video"
  platform: youtube
  dimensions: "1280x720px"

  concept:
    name: "Authority + Mechanism Visualization"
    scroll_stop_mechanism: "Emotional trigger (confident face) + Curiosity gap (3 bucket visual)"
    composition_approach: "Face (left 60%) + buckets + text (right 40%)"

  layout:
    primary_element:
      type: "Face — professional authority"
      description: "30s-40s professional, looking directly at camera, slight confident smile, business casual attire"
      position: "Left side, occupies 0-768px (60% of width)"
      size: "Head/shoulders framing, face fills left side from top to bottom"
      notes: "Eye line at vertical center (360px). Direct eye contact for connection. Warm, approachable expression (not stiff)."

    secondary_element:
      type: "3 Bucket Graphics"
      description: "3 stylized bucket/container graphics, modern flat design with subtle depth"
      position: "Right side, 800-1180px horizontal, 200-520px vertical (upper-right quadrant)"
      size: "Each bucket: 100px wide × 140px tall, 20px spacing between"
      colors: "Bucket 1: #1E40AF (Trust Blue), Bucket 2: #10B981 (Growth Green), Bucket 3: #F59E0B (Energy Orange)"
      labels: "NOW / SOON / LATER (Montserrat Semibold 20px, centered under each bucket)"
      notes: "Stylized cylinders, not literal buckets. Clean, modern, data-visualization feel."

    text_overlay:
      headline: "THE 3-BUCKET SYSTEM"
      position: "Right side, 800-1180px horizontal, 560-640px vertical (lower-right)"
      font: "Montserrat Bold"
      size: "96px"
      weight: 700
      color: "#1E40AF (Trust Blue)"
      treatment: "8px white outline for contrast"
      alignment: "Left-aligned (with buckets above)"
      notes: "Must be readable at 320px width (mobile sidebar). Test before finalizing."

    logo:
      placement: "Bottom-right corner"
      position: "1160px horizontal (20px from right), 660px vertical (20px from bottom)"
      size: "80px width (maintain aspect ratio)"
      treatment: "Full color logo on white background (no outline needed)"

    safe_zones:
      top: "60px"
      bottom: "80px (avoid YouTube timestamp)"
      left: "60px"
      right: "100px"
      notes: "All critical elements (face, text) within safe zones"

  visual_elements:
    background:
      treatment: "Clean white (#FFFFFF)"
      rationale: "Maximum contrast for text/graphics, professional clean look"

    face_treatment:
      photography_style: "Natural lighting, warm tones, shallow depth of field (blurred background)"
      expression: "Confident slight smile, direct eye contact"
      attire: "Business casual (button-down or polo, no tie)"
      avoid: "Stock photo feel, overly staged, fake smile"

    bucket_graphics:
      style: "Flat design with subtle gradient (5% darker at bottom for depth)"
      treatment: "2px outline in same color as fill (slightly darker)"
      label_position: "Centered below each bucket, 10px spacing"

  color_usage:
    background: "#FFFFFF"
    primary_text: "#1E40AF (Trust Blue) — headline"
    bucket_colors: "#1E40AF, #10B981, #F59E0B (one per bucket)"
    outline: "#FFFFFF (8px on headline text)"
    logo: "Full brand colors per brand guidelines"

    contrast_verification:
      headline_on_white: "8.2:1 (WCAG AAA) — PASS"
      bucket_labels_on_white: "8.2:1 — PASS"

  reference_imagery:
    specimen_examples:
      - "YT-THUMB-047: Face + text combo, similar composition"
      - "YT-THUMB-083: Clean professional look, high contrast"
    mood_references:
      - "Professional LinkedIn headshot (for face style)"
      - "Clean infographic design (for bucket graphics)"
    avoid:
      - "Stock corporate photos (sterile, fake)"
      - "Literal bucket images (not stylized)"
      - "Cluttered compositions (keep clean)"

  production_notes:
    for_designer:
      - "Face is PRIMARY element — must convey authority and approachability"
      - "Buckets are SECONDARY — support mechanism curiosity, don't dominate"
      - "Text must be readable at 320px width (test at 25% size)"
      - "Ensure 8px white outline on headline text for maximum contrast"
      - "Logo small and subtle (bottom-right, don't distract)"

    technical_specs:
      dimensions: "1280x720px"
      file_format: "JPG (compressed, <2MB) or PNG (if transparency needed)"
      color_space: "sRGB"
      resolution: "72 PPI (web standard)"

    approval_criteria:
      - "Face conveys confident authority (not arrogant or stiff)"
      - "Buckets clearly represent 3-part system"
      - "Headline readable at thumbnail size (320px width)"
      - "All elements within safe zones"
      - "Brand colors used correctly"
      - "Overall feel: professional, clean, approachable"

  alternative_concepts:
    # These will go to Arena for evaluation
    - concept_b: "Result-focused lifestyle aspiration"
    - concept_c: "Data visualization pattern interrupt"
    notes: "Arena Layer 2.5 will evaluate all concepts. This brief (Concept A) is primary."
```

## Quality Gates

- [ ] Complete composition defined with precise positioning
- [ ] All visual elements specified (face, buckets, text, logo, background)
- [ ] Text overlay fully specified (font, size, color, treatment, position)
- [ ] Color usage documented with contrast verification
- [ ] Reference imagery provided
- [ ] Production notes clear and actionable
- [ ] Technical specs complete
- [ ] Approval criteria defined
- [ ] File written: `s12-2.1-thumbnail-brief.yaml`

## Handoff

**Downstream consumer:**
- Layer 2.5 (Arena) evaluates this brief against alternatives
- After Arena selection, designer executes selected brief
- S14 (assembly) includes final thumbnail in content package
