# Microskill 1.4: Color Typography System
Skill: S12 Visual Direction
Layer: 1
Model: claude-sonnet-4.5
Version: 1.0

## Purpose

Create detailed color and typography specifications that designers can execute pixel-perfect. This is the technical implementation layer of the visual style guide — specific hex codes, font sizes, spacing, contrast ratios.

## Input

**Required files:**
- `s12-0.4-brand-guidelines.yaml` — brand color/typography baseline
- `s12-1.3-style-guide.yaml` — visual approach for this content
- `s12-1.1-platform-analysis.yaml` — platform readability requirements

## Process

### MUST
1. **Create detailed color palette**
   - Start with colors from style guide
   - Add all necessary variations:
     - Base colors (primary, secondary, accent, background)
     - Text colors (dark, medium, light)
     - State colors (hover, active, disabled if needed for interactive elements)
     - Overlay colors (semi-transparent backgrounds for text)
   - Document each with: hex code, RGB values, usage context, WCAG contrast ratio
   - Verify all text/background combinations meet WCAG AA minimum (4.5:1 for normal text, 3:1 for large text)

2. **Define typography scale**
   - Create complete type scale for all asset types:
     - Thumbnail typography (optimized for 1280×720, readable at 320px)
     - Video frame typography (optimized for 1920×1080)
     - Carousel slide typography (optimized for 1080×1080)
   - For each level (H1, H2, body), specify:
     - Font family
     - Font weight
     - Font size (in px)
     - Line height
     - Letter spacing (if non-default)
     - Text color (with contrast ratio)
     - Text treatment (outline, shadow, background)

3. **Specify text overlay treatments**
   - For each text overlay scenario (text on image, text on video, text on gradient):
     - Define readability enhancement:
       - Outline: color, width
       - Shadow: color, offset, blur
       - Background box: color, opacity, padding, border-radius
     - Specify contrast requirements (must meet WCAG AA at minimum)
     - Provide fallback if primary treatment fails readability

4. **Create spacing and sizing system**
   - Define spacing scale:
     - Safe zone margins (top, bottom, left, right)
     - Element spacing (between headline and subhead, between text and graphics)
     - Padding (inside background boxes, around frames)
   - Define sizing guidelines:
     - Minimum text size (for platform readability)
     - Maximum text size (to prevent overwhelming visual)
     - Icon/graphic sizes (consistent across assets)

5. **Document responsive adaptations**
   - If multi-platform or multi-format, document how typography scales:
     - YouTube thumbnail (1280×720) vs. Instagram post (1080×1080)
     - Desktop view vs. mobile view
   - Provide scaling formulas or specific size breakpoints

6. **Create designer reference sheet**
   - Compile all specs into designer-friendly format:
     - Color swatches with hex codes
     - Typography hierarchy visualization
     - Text treatment examples
     - Spacing diagram
   - This sheet should enable pixel-perfect implementation

### NEVER
- Provide colors without hex codes
- Provide font sizes without specifying unit (px, pt, rem)
- Skip contrast ratio verification
- Define typography without specifying treatment (outline, shadow, etc.)
- Create specs that vary wildly between assets (maintain consistency)
