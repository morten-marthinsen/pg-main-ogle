# 1.1 Taxonomy Application

## Purpose
Internalize the 7-category hook taxonomy and apply it to the specific campaign context. This is Phase A of concept/naming separation.

---

## Input Requirements
- Strategic context from 0.4
- Hook taxonomy from 0.3
- Pillar context from CAF

---

## Execution Protocol

### Step 1: Load Complete Taxonomy
Internalize all 7 categories with mechanisms:

**1. Curiosity Hooks**
- Open loop (incomplete story)
- Unexpected juxtaposition
- Impossible/unlikely statement
- Hidden knowledge reveal

**2. Fear Hooks**
- Warning/danger signal
- Loss framing
- Mistake exposure
- Future threat

**3. Desire Hooks**
- Result promise
- Secret/shortcut
- Transformation preview
- Status/identity upgrade

**4. Social Proof Hooks**
- Numbers/statistics
- Authority endorsement
- Trend signaling
- Community reference

**5. Identity Hooks**
- "You" direct address
- Tribe identification
- Struggle recognition
- Aspiration alignment

**6. Controversy Hooks**
- Contrarian take
- Myth busting
- Hot take
- Challenge to authority

**7. Story Hooks**
- "I was wrong about..."
- Transformation beginning
- Conflict introduction
- Stakes establishment

---

### Step 2: Map Categories to Pillars
For each pillar from CAF, identify which hook categories are most appropriate:

```yaml
pillar_category_mapping:
  pillar_1:
    name: "[Pillar name]"
    primary_categories:
      - category: "[Most relevant category]"
        rationale: "[Why this category fits this pillar]"
        mechanism_priority: "[Which mechanisms within category]"

      - category: "[Second most relevant]"
        rationale: ""
        mechanism_priority: ""

    secondary_categories:
      - category: "[Can use occasionally]"
        rationale: ""

    avoid_categories:
      - category: "[Poor fit]"
        reason: "[Why to avoid]"
```

**Mapping criteria:**
- Content pillar topic alignment
- Audience sophistication level
- Business objective (awareness/engagement/conversion)
- Voice compatibility

---

### Step 3: Prioritize Categories for Campaign
Based on:
- Pillar distribution (which pillars get most content)
- Content function mix (awareness/engagement/conversion percentages)
- Platform patterns (which categories perform on primary platform)
- Competitor gap analysis (underused categories)

```yaml
campaign_category_priority:
  tier_1_categories:
    - category: "[Highest priority]"
      usage_target: "[% of hooks]"
      rationale: "[Why prioritize]"

  tier_2_categories:
    - category: "[Medium priority]"
      usage_target: "[% of hooks]"
      rationale: ""

  tier_3_categories:
    - category: "[Lower priority]"
      usage_target: "[% of hooks]"
      rationale: ""
```

**Distribution should reflect content function mix from CAF.**

---

### Step 4: Mechanism Selection
For each prioritized category, select which specific mechanisms to emphasize:

```yaml
mechanism_selection:
  curiosity_hooks:
    emphasize:
      - mechanism: "Unexpected juxtaposition"
        examples_from_specimens: []
        campaign_application: ""
    de-emphasize:
      - mechanism: "Hidden knowledge reveal"
        reason: "[e.g., 'Overused by competitors']"

  # Repeat for all categories
```
