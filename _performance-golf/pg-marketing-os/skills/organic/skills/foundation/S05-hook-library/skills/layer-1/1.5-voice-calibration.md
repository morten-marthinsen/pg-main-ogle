# 1.5 Voice Calibration

## Purpose
Ensure all generated hooks match BVF voice constraints. Fix violations, create voice-calibrated examples.

---

## Input Requirements
- Pillar hooks from 1.4
- BVF from 0.4 (voice_attributes, anti_voice, tone_range)
- CONCEPT_APPROVED.yaml checkpoint (REQUIRED)

---

## Execution Protocol

### Step 1: Load Voice Constraints
From BVF:
```yaml
voice_filter:
  voice_attributes: []  # What hooks should reflect
  anti_voice: []  # Forbidden patterns
  tone_range: ""  # Acceptable tones
  language_style: ""  # Formal/casual/technical/etc
```

---

### Step 2: Voice Violation Detection
For EVERY hook from 1.4, check against anti_voice:

```python
def detect_voice_violations(hook, anti_voice_patterns):
    violations = []

    for pattern in anti_voice_patterns:
        if pattern.lower() in hook.lower():
            violations.append({
                'hook': hook,
                'violation': pattern,
                'severity': 'HIGH',
                'action': 'REJECT and regenerate'
            })

    return violations
```

**If violations found:**
- Document violating hooks
- Regenerate compliant alternatives
- Replace violations in pillar_hooks

---

### Step 3: Voice Attribute Expression Check
For each hook, verify it expresses at least 1 voice_attribute:

```yaml
voice_expression_check:
  - hook: "[Hook text]"
    expresses_attributes: ["[Attribute 1]", "[Attribute 2]"]
    tone_alignment: "On tone | Off tone"
    language_style_match: true | false
```

**If hook doesn't express any voice attributes:**
- Categorize as "voice-neutral" (acceptable but not ideal)
- OR revise to add voice expression
- Aim for 70%+ of hooks expressing ≥1 attribute

---

### Step 4: Create Voice-Calibrated Examples
Generate 5-10 exemplar hooks that perfectly embody voice:

```yaml
voice_exemplars:
  on_voice_examples:
    - hook: "[Hook that nails campaign voice]"
      category: "[Hook category]"
      pillar: "[Which pillar]"
      why_on_voice: |
        [Explanation: which attributes it expresses, how tone is perfect, etc.]

    - hook: "[Another exemplar]"
      # Same structure

  off_voice_examples:
    - original_hook: "[Generic or voice-violating hook]"
      voice_problem: |
        [What's wrong: violates anti_voice, wrong tone, etc.]
      fixed_hook: "[Hook rewritten in campaign voice]"
      what_changed: |
        [Explanation of fixes]

    - original_hook: "[Another example]"
      # Same structure
```

**These serve as reference for future hook generation.**

---

### Step 5: Voice Compliance Scoring
Score all hooks on voice compliance:

```yaml
voice_compliance_report:
  total_hooks: 0

  compliance_breakdown:
    perfect_voice_fit: 0  # Expresses attributes, no violations
    acceptable: 0  # No violations, voice-neutral
    needs_revision: 0  # Minor issues
    rejected: 0  # Violations, regenerated

  voice_attribute_expression:
    hooks_expressing_1_plus_attributes: 0
    percentage: "[%]"

  anti_voice_violations:
    violations_found: 0
    violations_fixed: 0

  tone_alignment:
    on_tone: 0
    off_tone: 0
    percentage_on_tone: "[%]"
```

**Target:** ≥90% acceptable or better, 0 violations remaining

---

### Step 6: Apply Voice Filters to Library
Create filtered hook sets:

```yaml
voice_filtered_hooks:
  tier_1_voice_perfect:
    - hook: "[Perfect voice fit]"
      category: ""
      pillar: ""
      voice_attributes_expressed: []

  tier_2_voice_acceptable:
    - hook: "[Acceptable, voice-neutral]"
      category: ""
      pillar: ""

  voice_violations_removed:
    - original_hook: "[Violating hook]"
      violation: ""
      replacement_hook: "[Fixed version]"
```
