# 0.1 Input Validator

## Purpose
Validate that all upstream outputs exist and contain required fields before hook library generation begins.

---

## Input Requirements

Required upstream files:
- `[project]/S01-audience-intelligence/outputs/[campaign]-AIF.yaml`
- `[project]/S02-platform-strategy/outputs/[campaign]-PSF.yaml`
- `[project]/S03-brand-voice/outputs/[campaign]-BVF.yaml`
- `[project]/S04-content-architecture/outputs/[campaign]-CAF.yaml`

Hook context (user-provided, optional):
```yaml
hook_context:
  top_performing_content: []
  competitor_hooks: []
```

---

## Execution Protocol

### Step 1: File Existence Check
Verify all required upstream files exist:
- AIF.yaml (audience intelligence)
- PSF.yaml (platform strategy)
- BVF.yaml (brand voice)
- CAF.yaml (content architecture)

**If any file missing → HALT. Request file before proceeding.**

---

### Step 2: Required Field Validation

**From AIF, verify presence of:**
- `pain_points` (list with ≥3 entries)
- `desires` (list with ≥3 entries)
- `language_patterns` (customer voice data for hook calibration)

**From PSF, verify presence of:**
- `primary_platform` (single platform selection)
- `content_formats` (list of formats that need hooks)

**From BVF, verify presence of:**
- `voice_attributes` (brand voice definition)
- `anti_voice` (what to avoid in hooks)
- `tone_range` (acceptable tones)

**From CAF, verify presence of:**
- `content_pillars` (≥3 pillars that need hooks)
- `pillar.name` for each pillar
- `content_series` (≥1 series — series may have hook patterns)

**If any required field missing → FAIL validation. Return to upstream skill.**

---

### Step 3: Hook Context Validation (if provided)
If user provides hook_context:

**Validate top_performing_content:**
- Each entry has `title` and `hook_used`
- At least 1 entry recommended (not required)

**Validate competitor_hooks:**
- Each entry has `creator` and `hook`
- At least 1 entry recommended (not required)

---

### Step 4: Pillar-Hook Alignment Check
Extract pillar count from CAF:
- Count pillars in CAF
- Verify count ≥3

**This determines hook generation volume:** Each pillar needs ≥5 hooks.

---

---

## Quality Gates
- [ ] All 4 upstream files exist
- [ ] All required fields present in AIF, PSF, BVF, CAF
- [ ] Pillar count ≥3
- [ ] validation_status = PASS

---

Pass validation report to: **0.2 Teaching Loader**
