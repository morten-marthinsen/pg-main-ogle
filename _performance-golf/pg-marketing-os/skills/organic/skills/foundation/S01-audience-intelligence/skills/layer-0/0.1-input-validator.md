# 0.1 Input Validator

## Purpose
Validate all required inputs exist and are substantive before proceeding to audience research. Prevents execution with incomplete context.

---

## Input Requirements
User must provide:
```yaml
brand_info:
  name: [Brand/person name]
  category: [What space we operate in]
  existing_positioning: [How we currently position]
  key_offerings: [What we sell/promote]

existing_data:
  current_audience_size: [If applicable]
  current_engagement_rate: [If applicable]
  existing_personas: [Any prior research]
  analytics_access: [What data we have]

competitive_context:
  direct_competitors: [List]
  adjacent_competitors: [List]
  aspirational_comparisons: [Who we want to be like]

research_budget:
  time_available: [How deep can we go]
  tool_access: [SparkToro, surveys, etc.]
```

---

## Execution Protocol

### Step 1: Validate Brand Info
```
CHECK brand_info.name is NOT empty
CHECK brand_info.category is NOT empty
CHECK brand_info.key_offerings has ≥1 item

IF any MISSING:
  HALT with message: "brand_info incomplete — need name, category, and offerings"
```

### Step 2: Validate Competitive Context
```
CHECK competitive_context.direct_competitors has ≥1 item

IF empty:
  WARN: "No direct competitors listed — will limit competitor analysis"
  PROCEED: true (non-blocking warning)
```

### Step 3: Validate Research Budget
```
CHECK research_budget.time_available is defined
CHECK research_budget.tool_access is defined

IF both undefined:
  SET defaults:
    time_available: "standard"
    tool_access: "manual research only"
```

### Step 4: Check for Existing Data
```
IF existing_data.analytics_access exists:
  PRIORITY: Use existing data first
  LOG: "Analytics access available — will prioritize actual data"

IF existing_data.existing_personas exists:
  LOG: "Prior personas exist — will validate against fresh research"
```

---

## Output
- **File:** `input-validation.yaml`
- **Format:** YAML

```yaml
validation_status: PASS | FAIL
timestamp: [ISO 8601]

validated_inputs:
  brand_info:
    name: [value]
    category: [value]
    offerings_count: [number]

  competitive_context:
    direct_competitors_count: [number]
    adjacent_competitors_count: [number]

  research_budget:
    time_available: [value]
    tool_access: [value]

  data_availability:
    has_analytics: true | false
    has_existing_personas: true | false

warnings: []
blocking_errors: []
```

---

## Quality Gates
- [ ] brand_info.name exists and is not empty
- [ ] brand_info.category exists and is not empty
- [ ] brand_info.key_offerings has at least 1 item
- [ ] validation_status = PASS written to output file

---

## Handoff
Pass to: **0.2 Teaching Loader**

If validation_status = FAIL, HALT execution and return errors to user.
