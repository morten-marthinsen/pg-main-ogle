# 1.1 Demographic Foundation

## Purpose
Establish baseline demographics with evidence. This is the quantifiable foundation — age, location, income, professional context. Without this, all downstream analysis is guesswork.

---

## Input Requirements
- brand_info (from 0.1)
- specimens-loaded (from 0.3) for pattern reference
- Analytics data if available

---

## Execution Protocol

### Step 1: Age Range Determination
```
SOURCES (priority order):
  1. Existing analytics data (if available)
  2. SparkToro audience intelligence
  3. Competitor audience analysis
  4. Proxy data from similar brands in specimens

MUST document:
  - Primary age range (e.g., 25-34)
  - Secondary age range (e.g., 35-44)
  - Rationale with evidence source

NEVER assume without evidence.
```

### Step 2: Gender Split Analysis
```
DETERMINE:
  - Male %
  - Female %
  - Other/non-binary %
  - Rationale based on data source

IF no data available:
  - State "Unknown — requires analytics setup"
  - DO NOT guess percentages
```

### Step 3: Location Mapping
```
IDENTIFY:
  - Primary markets (Top 3 geographic areas)
  - Primary language(s)
  - Timezone clusters (for posting optimization)

PRIORITY data sources:
  - Analytics geo data
  - SparkToro location intelligence
  - Competitor follower analysis
```

### Step 4: Income Level Estimation
```
DETERMINE:
  - Income bracket range
  - Spending capacity relevant to offerings

EVIDENCE from:
  - Industry salary data for professional_context
  - Product/service price points they currently buy
  - Lifestyle indicators from platform behavior
```

### Step 5: Professional Context
```
MAP:
  - Industries where they work
  - Job levels/seniority
  - Business owners percentage (if relevant)

CONNECTION:
  - How professional context relates to brand category
  - How it affects content consumption patterns
```

---

## Output
- **File:** `demographic-foundation.yaml`
- **Format:** YAML

```yaml
demographics:
  age_range:
    primary: "25-34"
    secondary: "35-44"
    rationale: "[Evidence source + reasoning]"
    confidence: high | medium | low

  gender_split:
    male: [%]
    female: [%]
    other: [%]
    rationale: "[Data source]"
    confidence: high | medium | low

  location:
    primary_markets: ["Market 1", "Market 2", "Market 3"]
    language: ["Primary language"]
    timezone_clusters: ["EST", "PST", etc.]
    evidence: "[Source]"

  income_level:
    bracket: "$50k-$75k" (example)
    spending_capacity: "[Relevant to offerings]"
    evidence: "[Source]"

  professional_context:
    industries: ["Industry 1", "Industry 2"]
    job_levels: ["Level 1", "Level 2"]
    business_owners: "[% if relevant]"
    relevance_to_brand: "[How it connects]"

evidence_sources:
  - "[Source 1]"
  - "[Source 2]"

confidence_notes: "[Any gaps or assumptions]"
```

---

## Quality Gates
- [ ] age_range.primary is populated with rationale
- [ ] location.primary_markets has ≥1 market
- [ ] All fields have evidence_sources documented
- [ ] Confidence level assigned to each major section
- [ ] No fabricated data (if unknown, state "Unknown — requires data")
