# 1.4 Benchmark Mapping

## Purpose
Establish score-to-outcome mappings based on calibration data. Define minimum publish threshold, target quality, and viral potential thresholds.

---

## Input Requirements

From Layer 1:
- `1.3-calibration.yaml` (specimen scores and tiers)

---

## Execution Protocol

### Step 1: Define Score Ranges

Based on calibration data, map score ranges to expected outcomes:

```yaml
score_mapping:
  "0-40":
    label: "Do not publish"
    expected_outcome: "Will underperform, waste resources"
    action: "Rework or kill"

  "40-60":
    label: "Below average"
    expected_outcome: "Baseline performance, limited reach"
    action: "Needs improvement before publishing"

  "60-75":
    label: "Publishable"
    expected_outcome: "Acceptable performance, audience growth"
    action: "Can publish, monitor performance"

  "75-85":
    label: "Strong"
    expected_outcome: "Above-average performance, good reach"
    action: "Prioritize in content calendar"

  "85-100":
    label: "Potential viral"
    expected_outcome: "High probability of breakout performance"
    action: "Maximum distribution push"
```

---

### Step 2: Set Campaign Thresholds

Define hard thresholds for this specific campaign:

**Minimum Publish Score:** 60
- Rationale: Below this, content underperforms baseline
- Below 60 = do not publish

**Target Score:** 75
- Rationale: This is where "strong" performance begins
- All content should aim here

**Viral Threshold:** 85
- Rationale: Calibration shows this is where viral potential emerges
- Worthy of maximum amplification

**Hero Content Requirement:** 80+
- Rationale: Pillar content must exceed target
- If hero content scores <80, it's not hero-worthy

---

### Step 3: Validate Against Calibration

Cross-check thresholds against actual specimen performance:

```yaml
threshold_validation:
  minimum_publish_60:
    specimens_in_range: [SPEC_012, SPEC_018]
    actual_performance: "Met baseline expectations"
    validated: true

  target_75:
    specimens_in_range: [SPEC_003, SPEC_007]
    actual_performance: "Strong performance confirmed"
    validated: true

  viral_85:
    specimens_in_range: [SPEC_001, SPEC_005]
    actual_performance: "Viral confirmed"
    validated: true
```

**If validation fails:** Adjust thresholds based on data.

---

## Output

**File:** `layer-1/1.4-benchmarks.yaml`

**Format:**
```yaml
timestamp: "[ISO 8601]"

benchmarks:
  minimum_publish: 60
  target: 75
  viral_threshold: 85
  hero_content_minimum: 80

score_mapping:
  "0-40": "Do not publish — will underperform"
  "40-60": "Needs work — below acceptable quality"
  "60-75": "Publishable — acceptable performance expected"
  "75-85": "Strong — prioritize distribution"
  "85-100": "Viral potential — maximum push"

validation:
  thresholds_validated: true
  based_on_specimens: 45
  calibration_source: "layer-1/1.3-calibration.yaml"

decision_tree:
  if_score_below_60: "Kill or rework — do not publish"
  if_score_60_74: "Publish as planned"
  if_score_75_84: "Prioritize in calendar + boost budget"
  if_score_85_plus: "Launch with maximum amplification"
```

**Minimum size:** 500B
