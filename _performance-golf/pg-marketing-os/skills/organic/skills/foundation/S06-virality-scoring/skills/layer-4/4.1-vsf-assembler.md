# 4.1 VSF Assembler

## Purpose
Compile all Layer 1 outputs into the final Virality Scoring Framework (VSF) file — the complete, production-ready scoring system.

---

## Input Requirements

From Layer 1:
- `1.1-dimensions.yaml`
- `1.2-modifiers.yaml`
- `1.3-calibration.yaml`
- `1.4-benchmarks.yaml`
- `1.5-quick-score.yaml`

From Layer 0:
- `0.4-upstream-synthesis.yaml` (source file references)

---

## Execution Protocol

### Step 1: Assemble Dimensions Section

Pull from 1.1-dimensions.yaml:
- All 8 dimensions with weights, criteria, indicators
- Verify weights sum to 100

---

### Step 2: Assemble Platform Modifiers Section

Pull from 1.2-modifiers.yaml:
- Primary platform identification
- Dimension adjustments
- Bonus points
- Penalties

---

### Step 3: Assemble Calibration Section

Pull from 1.3-calibration.yaml:
- Calibration date
- Specimens analyzed (count)
- Key insights (3-5 most important)
- Correlation validation

---

### Step 4: Assemble Benchmarks Section

Pull from 1.4-benchmarks.yaml:
- Minimum publish threshold
- Target score
- Viral threshold
- Hero content minimum
- Score mapping (5 ranges)

---

### Step 5: Assemble Quick-Score Section

Pull from 1.5-quick-score.yaml:
- Must-haves checklist
- Red flags checklist
- Green lights checklist
- Kill criteria

---

### Step 6: Add Scoring Calculator

Document the scoring formula:

```yaml
scoring_process:
  step_1: "Rate each dimension 0-10"
  step_2: "Apply dimension weights"
  formula: |
    Base = (hook * 0.20) + (emotion * 0.15) + (value * 0.15) +
           (social * 0.15) + (story * 0.10) + (platform * 0.10) +
           (specific * 0.10) + (voice * 0.05)
  step_3: "Apply platform modifiers (dimension adjustments)"
  step_4: "Add bonus points / subtract penalties"
  step_5: "Map final score to benchmark range"
  output: "Publish decision (Kill/Revise/Publish/Prioritize/Maximum Push)"
```

---

### Step 7: Add Source Files Section

Reference all upstream inputs:

```yaml
source_files:
  AIF: "skills/foundation/S01-audience-intelligence/outputs/[name]-AIF.yaml"
  PSF: "skills/foundation/S02-platform-strategy/outputs/[name]-PSF.yaml"
  BVF: "skills/foundation/S03-brand-voice/outputs/[name]-BVF.yaml"
  CAF: "skills/foundation/S04-content-architecture/outputs/[name]-CAF.yaml"
  HLF: "skills/foundation/S05-hook-library/outputs/[name]-HLF.yaml"
```

---

## Output

**File:** `outputs/[campaign-name]-VSF.yaml`

**Format:** Complete VSF template as specified in original SKILL.md

**Required sections:**
- campaign_name, date_created, version
- dimensions (all 8)
- platform_modifiers
- benchmarks
- calibration
- quick_score
- scoring_process
- source_files

**Minimum size:** 8-10KB (comprehensive framework)

---

## Quality Gates

- [ ] All 8 dimensions present with complete criteria
- [ ] Weights sum to 100
- [ ] Platform modifiers defined
- [ ] Benchmarks established
- [ ] Calibration insights included
- [ ] Quick-score checklist complete
- [ ] Scoring formula documented
- [ ] Source files referenced

---

## Handoff

Pass VSF to: **4.2 Execution Log**

The VSF file is now ready for use in production skills (S08-S13).
