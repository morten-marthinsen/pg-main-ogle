# 1.3 Specimen Calibration

## Purpose
Score each loaded specimen using the defined framework to validate dimension criteria, calibrate benchmarks, and learn what actual viral content scores.

---

## Input Requirements

From Layer 0:
- `0.3-specimen-inventory.yaml` (specimens to score)

From Layer 1:
- `1.1-dimensions.yaml` (scoring criteria)
- `1.2-modifiers.yaml` (platform adjustments)

---

## Execution Protocol

### Step 1: Score Each Specimen

For EVERY specimen (minimum 10, target 50+):

```yaml
specimen_id: "SPEC_001"
content_title: "[title/description]"
actual_performance:
  views: 1200000
  engagement_rate: 8.5
  tier: "viral"

dimension_scores:
  hook_strength: [0-10]
  emotional_resonance: [0-10]
  practical_value: [0-10]
  social_currency: [0-10]
  story_structure: [0-10]
  platform_fit: [0-10]
  specificity: [0-10]
  voice_alignment: [N/A for competitor content]

base_score: [weighted average]
modifiers_applied:
  bonuses: [list]
  penalties: [list]
final_score: [base + bonuses - penalties]

calibration_notes: |
  [What this specimen teaches about the framework]
```

**CRITICAL:** Justify each dimension score with specific observations from the content.

---

### Step 2: Pattern Analysis

After scoring all specimens, analyze patterns:

**High Performers (Viral tier):**
- Average dimension scores
- Which dimensions consistently score highest
- Common bonus triggers
- Score range (e.g., 82-95)

**Mid Performers (Strong/Solid tier):**
- Average dimension scores
- Score range (e.g., 65-80)
- What separates them from viral tier

**Baseline Performers:**
- Average dimension scores
- Score range (e.g., 50-65)
- Common weaknesses

---

### Step 3: Validate Framework Predictiveness

Check correlation between scores and actual performance:

```yaml
correlation_check:
  viral_tier:
    average_score: [X]
    score_range: [min-max]
    performance_range: [views/engagement]

  strong_tier:
    average_score: [X]
    score_range: [min-max]

  baseline_tier:
    average_score: [X]
    score_range: [min-max]

predictiveness: high | medium | low
```

**If predictiveness is LOW:** Framework needs adjustment. Review dimension criteria.

---

### Step 4: Document Calibration Insights

Extract learnings:

```yaml
calibration_insights:
  - insight: "Viral content ALWAYS scores 9-10 on hook_strength"
    implication: "Hook is non-negotiable for virality"

  - insight: "Practical_value varies widely (4-10) in viral tier"
    implication: "Entertainment can overcome lack of utility"

  - insight: "Platform_fit scores below 7 never go viral"
    implication: "Platform optimization is a threshold, not a differentiator"
```

---

## Output

**File:** `layer-1/1.3-calibration.yaml`

**Format:**
```yaml
timestamp: "[ISO 8601]"
specimens_scored: [count]

calibration_data:
  - specimen_id: "SPEC_001"
    title: "[title]"
    actual_performance: {...}
    dimension_scores: {...}
    base_score: 78
    final_score: 83
    tier: "viral"
    notes: "[observations]"

  # [repeat for all specimens]

pattern_analysis:
  viral_tier:
    average_score: 85
    score_range: [82, 95]
    key_dimensions: ["hook_strength: 9.2", "emotional_resonance: 8.8"]

  strong_tier:
    average_score: 72
    score_range: [68, 78]

  baseline_tier:
    average_score: 58
    score_range: [52, 64]

correlation: high
predictiveness_validated: true

calibration_insights:
  - "[Insight 1]"
  - "[Insight 2]"
  - "[Insight 3]"
```

**Minimum size:** 3KB (more with >10 specimens)
