# 0.1 Input Validator

## Purpose
Validate that all required upstream outputs from S01-S05 exist and contain the necessary data for building a calibrated virality scoring framework.

---

## Input Requirements

Required files from upstream skills:
- `outputs/[campaign]-AIF.yaml` (S01: Audience Intelligence)
- `outputs/[campaign]-PSF.yaml` (S02: Platform Strategy)
- `outputs/[campaign]-BVF.yaml` (S03: Brand Voice)
- `outputs/[campaign]-CAF.yaml` (S04: Content Architecture)
- `outputs/[campaign]-HLF.yaml` (S05: Hook Library)

---

## Execution Protocol

### Step 1: File Existence Check

Verify all 5 upstream files exist:

```yaml
file_check:
  AIF: [exists: true/false]
  PSF: [exists: true/false]
  BVF: [exists: true/false]
  CAF: [exists: true/false]
  HLF: [exists: true/false]
```

**BLOCKER:** If ANY file missing, HALT and return to foundation skills.

---

### Step 2: Content Validation

Verify each file contains required fields:

**From AIF (required):**
- `target_segment` (not empty)
- `primary_pain` (defined)
- `emotional_triggers` (at least 3)

**From PSF (required):**
- `primary_platform` (one of: instagram, tiktok, youtube, linkedin, x_twitter)
- `content_formats` (at least 2)
- `posting_cadence` (defined)

**From BVF (required):**
- `voice_dimensions` (defined)
- `banned_words` (exists, can be empty)

**From CAF (required):**
- `content_pillars` (at least 1)
- `content_function_mix` (awareness/engagement/conversion/community percentages)

**From HLF (required):**
- `hook_types` (at least 3)
- `platform_specific_hooks` (for primary platform)

---

### Step 3: Platform Alignment Check

Extract primary platform from PSF and verify:
- Platform is one of the 5 supported platforms
- PSF contains platform-specific guidance for this platform
- HLF contains hooks optimized for this platform

**WARNING:** If primary platform is "multi-platform," flag for human decision on which to prioritize for scoring modifiers.

---

## Output

**File:** `layer-0/0.1-validation-report.yaml`

**Format:**
```yaml
validation_status: PASS | FAIL
timestamp: "[ISO 8601]"

file_presence:
  AIF: true
  PSF: true
  BVF: true
  CAF: true
  HLF: true

content_validation:
  AIF:
    required_fields_present: true
    target_segment_defined: true
  PSF:
    required_fields_present: true
    primary_platform: "[platform]"
  BVF:
    required_fields_present: true
  CAF:
    required_fields_present: true
  HLF:
    required_fields_present: true
    platform_alignment: true

blockers: []
warnings: []
```

---

## Quality Gates

- [ ] All 5 files exist
- [ ] All required fields present
- [ ] Primary platform is valid and supported
- [ ] No critical blockers

---

## Handoff

Pass validation report to: **0.2 Teaching Loader**

If validation fails, HALT execution and report blockers to user.
