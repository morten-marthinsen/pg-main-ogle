# 1.6 Calendar Framework

## Purpose
Build the structural calendar that defines weekly rhythm, posting cadence, and batching strategy.

---

## Input Requirements
- Pillar concepts from 1.1 (approved and named)
- Function mapping from 1.2
- Series concepts from 1.3 (approved)
- Format matrix from 1.4
- Funnel integration from 1.5
- Platform context from PSF

---

## Execution Protocol

### Step 1: Design Weekly Rhythm
Assign primary format, function, and pillar to each day:

```yaml
weekly_rhythm:
  monday:
    primary_format: "[Format from matrix]"
    function: "[Awareness/Engagement/Conversion/Community]"
    pillar_focus: "[Which pillar]"
    content_type_example: "[Specific type]"
    rationale: "[Why this format/function on Monday]"

  tuesday:
    primary_format: ""
    function: ""
    pillar_focus: ""
    content_type_example: ""
    rationale: ""

  # Repeat for Wed, Thu, Fri, Sat, Sun
```

**Design considerations:**
- Monday: Often high engagement day (use engagement-focused content)
- Mid-week: Education and value delivery
- Friday/Weekend: Lighter, community-focused content
- Align with platform algorithm patterns (if known from PSF)

**All 7 days MUST be populated.**

---

### Step 2: Define Posting Cadence by Platform
Specify daily posting frequency for each platform:

```yaml
posting_cadence:
  instagram:
    feed_posts: "[Per day: e.g., 1-2]"
    reels: "[Per day: e.g., 1]"
    stories: "[Per day: e.g., 3-5]"
    carousel: "[Per week: e.g., 2-3]"
    total_daily_output: "[Total pieces per day]"

  tiktok:
    daily_posts: "[1-3]"
    story_frequency: "[If applicable]"

  # Add platforms as needed
```

**At least 1 platform MUST be fully defined.**

---

### Step 3: Design Content Batching Structure
Define how content creation is batched:

```yaml
content_batch_structure:
  batch_size: "[How many pieces at once: e.g., '10-15 pieces']"
  batch_frequency: "[How often batching happens: 'Weekly' or 'Biweekly']"
  batch_day: "[When batching occurs: 'Sunday']"

  batch_composition:
    - format: "[Format]"
      quantity_per_batch: "[Number]"
    - format: "[Format]"
      quantity_per_batch: "[Number]"

  production_workflow: |
    [Description of batching process: 'Sunday: Film 10 short videos, design 3 carousels. Monday-Tuesday: Edit and schedule.']
```

---

### Step 4: Plan Seasonal Adjustments
Define how calendar adapts to special periods:

```yaml
seasonal_adjustments:
  holidays:
    approach: |
      [How content changes: 'Reduce posting frequency, focus on community content']
    affected_periods: ["[Holiday/period]"]

  launches:
    approach: |
      [How launches affect calendar: 'Increase conversion content to 30%, reduce awareness to 35%']
    launch_windows: "[Duration]"

  slow_periods:
    approach: |
      [How to handle: 'Maintain consistency, use pre-batched evergreen content']
    slow_months: ["[Month]"]
```

---

### Step 5: Calculate Total Weekly Output
Sum all content across platforms:

```yaml
weekly_output_summary:
  total_pieces_per_week: 0
  breakdown_by_format:
    short_video: 0
    carousel: 0
    thread: 0
    # etc.

  breakdown_by_function:
    awareness: 0
    engagement: 0
    conversion: 0
    community: 0

  breakdown_by_pillar:
    pillar_1: 0
    pillar_2: 0
    pillar_3: 0

  time_commitment:
    content_creation: "[Hours per week]"
    editing_scheduling: "[Hours per week]"
    community_management: "[Hours per week]"
    total: "[Hours per week]"
```

**Verify this matches creator's available time from business context.**
