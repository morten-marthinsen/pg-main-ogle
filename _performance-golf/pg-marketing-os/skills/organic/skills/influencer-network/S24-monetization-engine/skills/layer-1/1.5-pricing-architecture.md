# 1.5 Pricing Architecture

## Purpose
Price products and services per persona based on value analysis, competitive research, audience purchasing power, and perceived value. Not guesswork — value-based pricing with psychological considerations.

## Model Assignment
**Model:** claude-opus-4-20250514

## Input Requirements
- Product development strategy (from 1.3)
- Partnership framework (from 1.4)
- Upstream context (audience segments, purchasing power)
- Teaching summary (pricing strategies)

## Execution Protocol

### MUST
- Conduct value analysis per product: what specific value does it provide? What problem solved? What result achieved?
- Research competitive pricing: find 3-5 comparable products, analyze pricing and value propositions
- Assess audience purchasing power: what can target audience afford? What have they purchased before?
- Identify perceived value factors: what signals increase perceived value? (credentials, results, scarcity, etc.)
- Apply pricing psychology: anchor pricing, tiering strategy, payment plans, pricing endings
- Create tiered offerings where appropriate: basic/premium/VIP with clear value differentiation
- Design payment plans for higher-ticket items (reduces friction)
- Document pricing rationale per product (not arbitrary numbers)

### NEVER
- Set prices without value analysis
- Skip competitive research (need market context)
- Ignore audience purchasing power (prices they can't afford won't sell)
- Forget perceived value factors (impacts willingness to pay)
- Use "industry standard" pricing without analysis
- Create identical pricing across all personas regardless of authority/value differences

## Output
**Filename:** `pricing-architecture.md`

**Format:** Markdown pricing strategy

**Minimum Size:** 5KB

**Required Sections:**
1. Per-Product Value Analysis (value provided, problem solved, result achieved)
2. Competitive Pricing Research (3-5 comparables per product category)
3. Audience Purchasing Power Assessment (what audience can afford, purchase history)
4. Perceived Value Factors (credentials, results, scarcity, social proof)
5. Pricing Psychology Application (anchors, tiers, payment plans, endings)
6. Final Pricing Per Product/Persona (with tier options where appropriate)
7. Pricing Rationale Documentation (why these numbers, not arbitrary)
8. Payment Plan Structures (for higher-ticket items)
