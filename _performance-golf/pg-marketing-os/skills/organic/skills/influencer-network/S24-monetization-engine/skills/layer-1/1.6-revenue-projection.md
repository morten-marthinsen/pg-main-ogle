# 1.6 Revenue Projection

## Purpose
Project revenue per persona and network-wide across conservative/moderate/aggressive scenarios. Include realistic assumptions about follower growth, conversion rates, and average order value.

## Model Assignment
**Model:** claude-opus-4-20250514

## Input Requirements
- All Layer 1 outputs (1.1-1.5)
- Revenue models, readiness timelines, products, partnerships, pricing
- Audience performance data (current followers, engagement)

## Execution Protocol

### MUST
- Create 3 projection scenarios per persona: conservative, moderate, aggressive
- Define assumptions per scenario: follower growth rate, conversion rates, average order value, churn rates
- Project monthly/quarterly revenue per persona for 12 months
- Calculate network-wide revenue totals across scenarios
- Document key risks that would prevent each scenario
- Include breakeven analysis: when does persona become profitable?
- Consider seasonality and market conditions
- Cite industry benchmarks where possible for assumption validation

### NEVER
- Create single-scenario projections (need range of outcomes)
- Use optimistic assumptions without industry benchmark validation
- Skip risk documentation (critical for realistic planning)
- Ignore network-wide aggregation (miss network effects)
- Forget breakeven analysis
- Project without documenting assumptions

## Output
**Filename:** `revenue-projection.md`

**Format:** Markdown revenue forecasts

**Minimum Size:** 5KB

**Required Sections:**
1. Projection Scenarios Overview (conservative, moderate, aggressive defined)
2. Per-Persona Assumptions (follower growth, conversion rates, AOV, churn per scenario)
3. Monthly/Quarterly Projections (12-month revenue forecast per persona per scenario)
4. Network-Wide Totals (aggregated revenue across all personas per scenario)
5. Key Risks (what would prevent each scenario from materializing)
6. Breakeven Analysis (when does each persona become profitable)
7. Assumption Validation (industry benchmarks cited where available)
8. Seasonality Considerations (revenue fluctuations expected)

## Quality Gates
- [ ] 3 scenarios (conservative, moderate, aggressive) per persona
- [ ] Assumptions explicit and realistic per scenario
- [ ] 12-month projections calculated monthly or quarterly
- [ ] Network-wide totals aggregated
- [ ] Key risks documented per scenario
- [ ] Breakeven analysis included
- [ ] Industry benchmarks cited where possible
- [ ] Seasonality considered
