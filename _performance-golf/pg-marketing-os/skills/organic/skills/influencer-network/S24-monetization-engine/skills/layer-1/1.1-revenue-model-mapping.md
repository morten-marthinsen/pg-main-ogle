# 1.1 Revenue Model Mapping

## Purpose
Map revenue models to each persona based on archetype, audience, and niche. Not one-size-fits-all — differentiate monetization approaches across network.

## Model Assignment
**Model:** claude-opus-4-20250514

## Input Requirements
- Upstream context summary (persona profiles, archetypes, niches)
- Teaching summary (monetization models, archetype fit)
- Persona Bibles (authority areas, audience)

## Execution Protocol

### MUST
- Evaluate each persona against 7+ revenue models: affiliate, sponsorship, digital products, courses, coaching, membership, consulting
- Score model fit per persona across 5 dimensions: archetype alignment, authority match, audience fit, scalability, margin potential
- Assign primary revenue model per persona (best fit)
- Assign 1-2 secondary revenue models (diversification)
- Ensure network-level differentiation (not all personas using same model)
- Justify model selection with archetype and audience rationale
- Consider production capacity (can persona sustain this model?)
- Document why other models were rejected per persona

### NEVER
- Assign same revenue model to all personas
- Select models without archetype-audience fit analysis
- Ignore authority limitations (persona can't sell what they're not qualified for)
- Skip production capacity assessment
- Create 70%+ overlap in primary revenue models across network
- Forget to document rejection rationale

## Output
**Filename:** `revenue-model-mapping.md`

**Format:** Markdown revenue model assignments

**Minimum Size:** 5KB

**Required Sections:**
1. Per-Persona Revenue Model Assignments (primary + secondary with rationale)
2. Model Fit Scoring (how each persona scores across 5 dimensions per model)
3. Network Differentiation Analysis (distribution of models, overlap check)
4. Authority Alignment Verification (can persona credibly deliver this model?)
5. Production Capacity Assessment (can this be sustained?)
6. Rejection Rationale (why other models NOT selected per persona)

## Quality Gates
- [ ] Primary + secondary revenue models assigned per persona
- [ ] Model fit scored across 5 dimensions
- [ ] Network differentiation verified (<70% primary model overlap)
- [ ] Authority alignment checked per persona
- [ ] Production capacity assessed
- [ ] Rejection rationale documented
