# 1.3 Content Calendar Framework

## Purpose
Define posting frequency, content mix, day-of-week patterns, and timing strategy per persona. Create sustainable content calendar frameworks that align with platform algorithms and audience behavior.

## Model Assignment
**Model:** claude-opus-4-20250514

## Input Requirements
- Platform selections (from 1.1)
- Persona Bibles (content style preferences, signature formats)
- Teaching summary (content calendar methodologies, timing research)
- Production capacity assessment (from 1.1)

## Execution Protocol

### MUST
- Define posting frequency per persona × platform (specific number per week, not "regularly")
- Specify day-of-week patterns aligned with platform best practices and audience behavior
- Design content mix ratio: educational/entertaining/promotional (e.g., 60/30/10)
- Map content types to posting schedule (e.g., Monday how-to, Wednesday story, Friday engagement)
- Consider persona's signature content types from Bible
- Verify total network posting volume is sustainable given production capacity
- Include testing and optimization plan (how to refine calendar based on performance)

### NEVER
- Use vague frequency like "post regularly" or "several times per week"
- Create unsustainable posting schedules (daily posts across 5 platforms)
- Ignore platform-specific timing research (LinkedIn weekday vs Instagram evening)
- Design identical content calendars across personas
- Skip content mix ratios (leads to promotional oversaturation)
- Forget production capacity reality check

## Output
**Filename:** `content-calendar-frameworks.md`

**Format:** Markdown content calendar specifications

**Minimum Size:** 5KB

**Required Sections:**
1. Per-Persona × Platform Posting Frequency (specific numbers with day-of-week patterns)
2. Content Mix Ratios (educational/entertaining/promotional percentages per persona)
3. Content Type Mapping (which formats on which days)
4. Timing Strategy (optimal posting times per platform based on research)
5. Network Posting Volume Summary (total posts per week, capacity verification)
6. Testing and Optimization Plan (how to refine based on performance data)

## Quality Gates
- [ ] Posting frequency specific (not vague) per persona × platform
- [ ] Day-of-week patterns defined with platform/audience rationale
- [ ] Content mix ratios specified (not 100% promotional)
- [ ] Content type mapping aligns with persona signature formats
- [ ] Network posting volume verified against production capacity
- [ ] Testing plan included for calendar optimization
