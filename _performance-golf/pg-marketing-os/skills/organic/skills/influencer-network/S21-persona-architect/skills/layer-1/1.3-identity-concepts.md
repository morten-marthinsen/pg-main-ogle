# 1.3 Identity Concepts

## Purpose
Create persona identity WITHOUT names. Define age, background, expertise area, worldview, communication style using concept descriptions only. This is Phase A — naming happens later after concept approval.

## Model Assignment
**Model:** claude-opus-4-20250514

## Input Requirements
- Niche ownership map (from 1.1)
- Archetype assignments (from 1.2)
- Teaching summary (persona development framework)
- Brand voice constraints (from upstream)

## Execution Protocol

### MUST
- Define age range and generation (impacts cultural references, platform preferences)
- Specify professional background and expertise source (credentials, experience, journey)
- Articulate worldview and core beliefs relevant to niche
- Define communication style: formality spectrum, energy level, pacing, vocabulary sophistication
- Ensure identity aligns with archetype and niche
- Respect brand voice constraints while allowing persona distinctiveness
- Create identities that feel REAL, not stereotypical

### NEVER
- Include names or handles (that's Phase B)
- Create identical identities across personas
- Define identity solely by job title ("marketing consultant")
- Ignore cultural and generational authenticity
- Create personas that contradict brand voice framework

## Output
**Filename:** `identity-concepts.md`

**Format:** Markdown identity descriptions

**Minimum Size:** 5KB

**Required Sections:**
1. Per-Persona Identity Profiles (one section per persona with 8-10 identity dimensions)
2. Identity Differentiation Matrix (how personas differ across key dimensions)
3. Brand Voice Alignment Check (how each respects brand constraints while being distinct)
4. Authenticity Assessment (are these believable people?)
5. Archetype Alignment (how identity matches assigned archetype)

## Quality Gates
- [ ] Each persona has detailed identity across 8+ dimensions
- [ ] No names or handles included (concept only)
- [ ] Identities feel authentic and specific (not generic archetypes)
- [ ] Clear differentiation across personas verified
- [ ] Brand voice constraints respected

## Handoff
Feeds into: 1.4 Backstory Development (identity provides foundation for backstory)

**CONCEPT CHECKPOINT:** Human reviews identity concepts before Phase B naming begins.
