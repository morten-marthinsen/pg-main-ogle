# 1.5 Personality Framework

## Purpose
Define personality traits, values, quirks, opinions, pet peeves for each persona. Must be specific enough that two people could write "in character" and sound similar.

## Model Assignment
**Model:** claude-opus-4-20250514

## Input Requirements
- Identity concepts (from 1.3)
- Backstory development (from 1.4)
- Archetype assignments (from 1.2)

## Execution Protocol

### MUST
- Define 10+ personality traits per persona (specific, not generic)
- Articulate core values hierarchy (what they prioritize, what they reject)
- Document quirks and idiosyncrasies (makes them memorable and human)
- List opinions on niche-relevant topics (not all mainstream views)
- Define pet peeves and triggers (what annoys them, what fires them up)
- Include communication patterns (how they express agreement/disagreement, humor style)
- Make personality consistent with backstory and archetype
- Create enough detail that voice can be replicated

### NEVER
- Use vague traits like "friendly" or "professional"
- Create personalities without contradictions or complexity
- Make all personas equally agreeable/mainstream
- Skip quirks (critical for parasocial connection)
- Define personality divorced from backstory

## Output
**Filename:** `personality-framework.md`

**Format:** Markdown personality profiles

**Minimum Size:** 6KB

**Required Sections:**
1. Per-Persona Personality Profiles (traits, values, quirks, opinions, pet peeves)
2. Values Hierarchy (what each persona prioritizes in order)
3. Communication Patterns (how personality shows up in expression)
4. Personality-Backstory Alignment (how personality flows from journey)
5. Personality Differentiation (how personas differ in personality)

## Quality Gates
- [ ] Each persona has 10+ specific personality traits
- [ ] Values hierarchy defined (not just list, but priority order)
- [ ] Quirks and pet peeves included (humanizing details)
- [ ] Opinions include some contrarian/distinctive views
- [ ] Communication patterns specific enough for voice replication

## Handoff
Feeds into: CONCEPT CHECKPOINT (human approves all Phase A work before naming)

**CONCEPT CHECKPOINT:** This completes Phase A. Human reviews all concept work (niche, archetype, identity, backstory, personality) before Phase B naming begins. This prevents naming bias from influencing concept evaluation.
