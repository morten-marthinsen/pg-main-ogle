# 0.2 Upstream Loader

## Purpose
Load complete content from S14, platform requirements from S02, and voice constraints from S03. Prepare all context needed for repurpose strategy.

---

## Input Requirements

From Layer 0:
- `0.1-validation.yaml` (validation passed)

---

## Execution Protocol

### Step 1: Load S14 Content Package

Extract:
- Original content ID, title, platform
- Full script/caption/content
- Key quotes, insights, stories
- Visual elements
- Performance data (if available)

### Step 2: Load S02 Platform Strategy

Extract:
- All active platforms
- Per-platform format requirements
- Length requirements
- Aspect ratios
- Platform-specific best practices

### Step 3: Load S03 Brand Voice

Extract:
- Voice attributes
- Tone spectrum
- Power words
- Banned words
- Platform-specific tone adjustments

### MUST Load

- Complete S14 package (not summary)
- All platform specs from S02
- Voice constraints from S03

### NEVER Skip

- Performance data (if available)
- Platform format requirements
- Voice attributes

---

**File:** `layer-0/0.2-upstream-content.yaml`

**Format:**
```yaml
timestamp: "[ISO 8601]"

original_content:
  content_id: "[ID]"
  content_title: "[Title]"
  platform: "[Platform]"
  content_type: "[Format]"
  full_content: |
    [Complete content]

  extracted_elements:
    hooks: []
    key_insights: []
    quotes: []
    stories: []
    frameworks: []
    data_points: []

  performance_data:
    views: [X]
    engagement_rate: [X]
    best_performing_moment: "[If available]"

platform_requirements:
  active_platforms: []
  per_platform_specs: {}

brand_voice:
  voice_attributes: {}
  tone_spectrum: {}
```

**Minimum size:** 2KB
