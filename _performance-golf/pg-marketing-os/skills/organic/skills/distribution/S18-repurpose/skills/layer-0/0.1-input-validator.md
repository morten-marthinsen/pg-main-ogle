# 0.1 Input Validator

## Purpose
Validate that all required inputs exist before repurpose planning begins. Ensure S14 content package is complete and platform/voice context is available.

---

## Input Requirements

**Required Files:**
- S14 Content Assembly package
- S02 Platform Strategy File (PSF)
- S03 Brand Voice File (BVF)

---

## Execution Protocol

### MUST Validate

1. **S14 Content Package exists and contains:**
   - content_id (unique identifier)
   - content_title
   - platform (original platform)
   - content_type/format
   - full_script or content_elements
   - performance_data (if published)

2. **S02 PSF loaded with:**
   - platforms_active (list of platforms)
   - per_platform_specs (format requirements)

3. **S03 BVF loaded with:**
   - voice_attributes
   - tone_spectrum

### NEVER Accept

- Missing content_id
- Empty content_elements
- S14 package without original content file path
- Missing PSF or BVF (required for adaptation)

---

## Output

**File:** `layer-0/0.1-validation.yaml`

**Format:**
```yaml
timestamp: "[ISO 8601]"
validation_status: "PASS"

validated_inputs:
  s14_content_package:
    path: "[path]"
    content_id: "[ID]"
    exists: true

  s02_psf:
    path: "[path]"
    exists: true
    platforms_count: [X]

  s03_bvf:
    path: "[path]"
    exists: true

gate_decision: "PROCEED"
```

**Minimum size:** 1KB

---

## Quality Gates

- [ ] All 3 required files exist
- [ ] S14 package has content_id
- [ ] S14 package has content_elements
- [ ] S02 PSF has 1+ platforms
- [ ] S03 BVF loaded

---

## Handoff

Pass validation result to: **0.2 Upstream Loader**
