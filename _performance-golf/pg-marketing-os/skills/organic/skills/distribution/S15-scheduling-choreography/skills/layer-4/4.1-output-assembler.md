# 4.1 Output Assembler

## Purpose
Assemble all Layer 1 outputs into final Scheduling Choreography File (SCF).

---

## Input Requirements

From Layer 1:
- `1.1-platform-timing.yaml`
- `1.2-content-type-timing.yaml`
- `1.3-cascade-sequences.yaml`
- `1.4-momentum-rules.yaml`
- `1.5-calendar-integration.yaml`

---

## Execution Protocol

### Step 1: Assemble SCF Structure

Combine all Layer 1 outputs into complete SCF following schema from S15-AGENT.md.

### Step 2: Add Engagement Triggers for S16

For each scheduled post:

```yaml
engagement_triggers:
  - content_id:
    platform:
    post_time:
    engagement_window_start: "[post_time]"
    engagement_window_end: "[post_time + 90 minutes]"
    priority: [High/Medium/Low based on virality score]
```

### Step 3: Add Network Amplification Flags for S17

Flag high-priority content:

```yaml
network_amplification: [Yes/No]
# Yes if virality_score >= 75 or hero content
```

### Step 4: Validate SCF Completeness

- All sections populated
- All content from S14 scheduled
- All engagement windows defined
- All cascade sequences complete

---

## Output

**File:** `outputs/[campaign]-SCF.yaml`

**Format:** Full SCF schema from S15-AGENT.md

**Minimum size:** 5KB

---

## Quality Gates

- [ ] All Layer 1 outputs integrated
- [ ] All S14 content scheduled
- [ ] Engagement triggers complete
- [ ] Network amplification flags set
- [ ] SCF validates against schema

---

## Handoff

Pass SCF to: **4.2 Execution Log**
