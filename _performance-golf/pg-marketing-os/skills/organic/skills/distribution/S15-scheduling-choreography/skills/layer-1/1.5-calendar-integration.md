# 1.5 Calendar Integration

## Purpose
Integrate holidays, events, and cultural moments into the posting schedule. Adjust content timing around key dates.

---

## Input Requirements

From Layer 0:
- `0.3-upstream-data.yaml` (campaign calendar from CBF)

From Layer 1:
- `1.2-content-type-timing.yaml` (content schedule)

---

## Execution Protocol

### Step 1: Load Calendar Events

From CBF + 2026 calendar:

```yaml
major_holidays:
  - date:
    event:
    content_strategy: "[How to approach]"
    posting_adjustment: "[Before/after/minimal]"
    avoid: "[What not to do]"

cultural_moments:
  - date:
    event: "[Awareness days, industry events]"
    relevance: [High/Medium/Low]
    opportunity: "[Content angle]"

campaign_key_dates:
  - date:
    event: "[Product launch, webinar, etc.]"
    content_focus: "[How content supports]"
```

### Step 2: Adjust Content Schedule

For each calendar event:

1. **High-impact holidays** (Christmas, Thanksgiving, etc.):
   - Move promotional content away
   - Schedule gratitude/community content if relevant
   - Expect low engagement

2. **Awareness days** (relevant to niche):
   - Create tie-in content if authentic
   - Schedule 1-2 days before event
   - Use event hashtags

3. **Campaign key dates**:
   - Build anticipation content week before
   - Day-of content scheduled
   - Follow-up content 1-2 days after

### Step 3: Identify Blackout Dates

Dates where posting is not recommended:
- Major holidays (if not relevant)
- Crisis/sensitive dates
- Known low-engagement periods

Move content scheduled on blackout dates to alternative times.


**Minimum size:** 1.5KB
