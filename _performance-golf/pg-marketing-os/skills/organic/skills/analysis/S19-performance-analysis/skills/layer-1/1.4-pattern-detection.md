# 1.4 Pattern Detection

## Purpose
Across all analyzed content, detect patterns: which hook types overperform, which platforms underperform, which content functions exceed predictions, which time slots work best.

## Assigned Model
`claude-opus-4` (Layer 1 analysis)

## Input
- 1.3-variance-analysis.json
- 1.2-prediction-comparison.json
- 1.1-collected-metrics.json
- 0.2-teaching-context.json (minimum pattern sample: 3)

## Process

### MUST
1. **Hook type pattern analysis** — Group by hook type, calculate avg variance per type
   - Which hooks consistently overperform predictions?
   - Which hooks consistently underperform?
   - Minimum 3 instances to call it a pattern
2. **Platform pattern analysis** — Group by platform, identify platform-specific trends
   - Does Instagram Reels overperform vs TikTok?
   - Are predictions more accurate on certain platforms?
3. **Content function pattern analysis** — Education vs entertainment vs inspiration vs community
   - Which functions exceed virality predictions?
4. **Timing pattern analysis** — Day of week, time of day, season
   - Do certain publish times consistently beat predictions?
5. **Cross-dimension patterns** — Are SC and SH variance correlated? Does high EA predict high PI?
6. **Statistical significance check** — Require minimum sample size (3), check for correlation strength
7. **Rank patterns by strength** — Which patterns are most reliable/actionable?

### NEVER
- Call something a "pattern" with <3 data points
- Confuse correlation with causation
- Ignore confounding variables (e.g., all high-performing content was published on same day)
- Cherry-pick data to support a preferred narrative

2. Statistical significance assessed
3. Confounding variables checked
4. Actionable insights defined
