# 4.1 Output Assembler

## Purpose
Assemble complete PAR (Performance Analysis Report) with all variance analysis, patterns, learning flags, and hypotheses. Minimum 5KB per report to ensure depth.

## Assigned Model
`claude-sonnet-4` (Layer 4 packaging)

## Input
- 1.1-collected-metrics.json
- 1.2-prediction-comparison.json
- 1.3-variance-analysis.json
- 1.4-pattern-detection.json
- 1.5-learning-flags.json
- 1.6-hypotheses.json

## Process

### MUST
1. **Assemble PAR structure:**
   - **Executive Summary** — Key findings, overall prediction accuracy, top patterns (300-500 words)
   - **Methodology** — Data sources, normalization approach, analysis timeframe
   - **Performance Overview** — Content inventory, platform distribution, date range
   - **Prediction Accuracy** — Composite variance, dimension-level variance, outcome distribution
   - **Variance Deep Dive** — Root cause analysis for all flagged variances
   - **Pattern Analysis** — All detected patterns with evidence and confidence
   - **Learning Flags** — All generated flags with priority/confidence
   - **Testable Hypotheses** — All hypotheses with test designs
   - **Recommendations** — Prioritized actions for S20
2. **Verify completeness** — All Layer 1 outputs incorporated
3. **Check minimum size** — PAR must be ≥5KB (indicates sufficient depth)
4. **Add visualizations** — Tables/charts for variance distribution, pattern strength
5. **Include appendices** — Full data tables, content ID references
6. **Version control** — PAR version, analysis date, data collection date

### NEVER
- Summarize without detail (PAR is comprehensive, not executive summary only)
- Omit supporting data
- Skip recommendations section
- Accept <5KB output (insufficient depth)

## Quality Gates
1. All sections present
2. ≥5KB file size
3. All data incorporated
4. Recommendations prioritized

## Handoff
Passes PAR to 4.2-execution-log.md and downstream to S20 Learning Capture.
