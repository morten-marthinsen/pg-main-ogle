# 1.6 Hypothesis Generation

## Purpose
For each significant finding, generate testable hypotheses in structured format: "We believe [X] because [evidence]. We can test this by [method]. Success looks like [metric]."

## Assigned Model
`claude-opus-4` (Layer 1 analysis)

## Input
- 1.5-learning-flags.json
- 1.4-pattern-detection.json
- 1.3-variance-analysis.json

## Process

### MUST
1. **For each high/medium priority learning flag, generate testable hypothesis:**
   - **Belief statement** — Clear, specific claim (e.g., "Shocking_stat hooks with specific/memorable stats outperform generic shocking stats")
   - **Evidence** — What data supports this belief (pattern IDs, variance data, content examples)
   - **Test method** — How to test in next campaign (A/B test design, sample size, control variables)
   - **Success criteria** — Quantitative metric that proves/disproves hypothesis (e.g., "SC score improves by ≥1.0 point")
   - **Failure criteria** — What result would disprove hypothesis
2. **Prioritize tests** — Rank by: ease of testing, expected impact, confidence in hypothesis
3. **Design A/B tests** — For high-priority hypotheses, spec out A/B test structure
   - Test group vs control group
   - Sample size needed for statistical significance
   - How to isolate variables
4. **Estimate timeline** — How many campaigns needed to gather sufficient data
5. **Link to learning flags** — Each hypothesis traces back to specific learning flag

### NEVER
- Generate untestable hypotheses ("quality matters")
- Skip success/failure criteria
- Ignore statistical significance requirements
- Design tests that can't isolate variables
