# 1.5 Gate Threshold Review

## Purpose
Review if current gate pass thresholds (virality >= 60, etc.) are still appropriate based on accumulated data. Tighten or loosen with evidence.

## Assigned Model
`claude-opus-4` (Layer 1 intelligence)

## Input
- 1.1-learning-categories.json (gate_adjustment flags)
- S19 PAR (content near threshold performance)
- 0.2-teaching-context.json (current gate thresholds)
- Historical pass rate data

## Process

### MUST
1. **For each gate_adjustment flag:**
   - Identify which gate (S06, S13, or other)
   - Current threshold value
   - Content pieces near threshold (within ±5 points)
   - How near-threshold content performed vs prediction
2. **Analyze threshold appropriateness:**
   - **If lowering threshold:** ≥3 pieces scored just below threshold but overperformed actual → threshold too conservative
   - **If raising threshold:** ≥3 pieces scored just above threshold but underperformed actual → threshold too lenient
   - Calculate recommended new threshold
   - Estimate impact on pass rate
3. **Statistical justification:**
   - Show distribution of near-threshold content
   - Calculate how pass rate would change
   - Require minimum 10 pieces near threshold for adjustment
   - Check if adjustment aligns with teaching updates (e.g., if SC scoring improved, threshold can stay)
4. **Draft threshold update:**
   - Which gate config file to modify
   - New threshold value
   - Rationale and evidence
5. **Impact analysis:**
   - How many more/fewer pieces would pass per batch?
   - Quality risk assessment (lowering too much = quality degradation)

### NEVER
- Adjust threshold based on <3 near-threshold cases
- Change threshold by >5 points at once
- Lower threshold without checking quality impact
- Raise threshold without evidence of quality issues
- Adjust without considering teaching updates (may fix root cause)

## Quality Gates
1. Minimum 3 near-threshold cases for adjustment
2. Impact analysis complete
3. Quality risk assessed
4. Statistical confidence rated
5. Action determined (adjust or defer)

## Handoff
Passes gate threshold adjustments to 1.6-propagation-verification.md if updates made, or logs deferred adjustments for future review.
