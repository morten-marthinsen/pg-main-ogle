# 0.1 Input Validator

## Purpose
Validate that S19 PAR exists, learning flags are present, and minimum evidence thresholds are met before proceeding with learning capture.

## Assigned Model
`claude-haiku-4` (Layer 0 validation)

## Input
- S19 PAR file path
- Learning flags from S19 (1.5-learning-flags.json)
- Current system state

## Process

### MUST
1. **Verify S19 PAR exists** — Check file path, non-empty, parseable markdown
2. **Validate learning flags file** — 1.5-learning-flags.json exists, minimum 1 flag present
3. **Check evidence completeness** — Every flag has evidence field with ≥1 citation
4. **Verify flag structure** — Required fields: flag_id, category, priority, confidence, evidence, recommended_action
5. **Count High-confidence flags** — If >80% are Low confidence → warn (insufficient data for learning)
6. **Check PAR size** — Minimum 5KB (indicates sufficient depth from S19)
7. **Validate timestamps** — PAR analysis date is recent (within 30 days)
8. **Output validation result** — Binary PASS/FAIL

### NEVER
- Proceed with missing PAR
- Accept learning flags without evidence
- Skip confidence distribution check
- Validate without checking PAR size threshold

## Output
**File:** `0.1-validation-result.json`

```json
{
  "validation_status": "PASS|FAIL",
  "timestamp": "ISO8601",
  "par_file": "path/to/PAR.md",
  "par_size": "12KB",
  "par_date": "2024-02-15",
  "learning_flags_file": "path/to/1.5-learning-flags.json",
  "flags_count": 8,
  "confidence_distribution": {
    "high": 4,
    "medium": 3,
    "low": 1
  },
  "high_confidence_percentage": 50,
  "evidence_complete": true,
  "warnings": [],
  "errors": []
}
```

## Quality Gates
1. PAR exists and ≥5KB
2. Learning flags present
3. Evidence complete
4. Flag structure valid

## Handoff
Passes to 0.2-teaching-loader.md if PASS, halts if FAIL.
