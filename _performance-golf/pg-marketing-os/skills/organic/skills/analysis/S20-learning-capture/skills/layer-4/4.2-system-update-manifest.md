# 4.2 System Update Manifest

## Purpose
List all files that were updated/created by this learning capture run. Include before/after for each change. This is the audit trail.

## Process

### MUST
1. **For each file modified, create entry with:**
   - File path (absolute)
   - File type (teaching, specimen, gate_config, learning_log)
   - Modification type (created, modified, deleted)
   - Timestamp of modification
   - Version change (if applicable)
   - Before/after snippets (for modifications)
   - Modification rationale
   - Linked learning flag(s)
2. **Group by file type:**
   - Teaching files
   - Specimen library files
   - Gate config files
   - Learning log files
   - Index files
3. **Calculate totals:**
   - Files created, modified, deleted
   - Total changes made
   - Verification status (all COMPLETE)
4. **Include git-style diff** for teaching file changes (shows exact lines changed)
5. **Output manifest file** — Ready for version control, audit, rollback if needed

### NEVER
- Omit any file modification
- Skip before/after snippets
- Forget to link to learning flags
- Leave timestamps blank
- Skip verification status
