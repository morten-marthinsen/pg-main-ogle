## PURPOSE

The Organic Marketing Engine produces systematic, platform-native organic content campaigns. It operates as a sub-engine within marketing-os, following the same structural patterns as the Email Engine, Ad Engine, and Upsell Engine.

**Output:** Platform-native organic content across Instagram, TikTok, YouTube, LinkedIn, X/Twitter, and Facebook — scripts, captions, carousels, threads, visual direction, scheduling, and performance learning loops.

---

## THE 5 ORGANIC LAWS

### Law 1: NO STRATEGY, NO CONTENT
Content production (S08-S14) is **LOCKED** until Strategic Foundation (S01-S07) is complete. The Campaign Brief File (CBF) is the key that unlocks production.

**Enforcement:** Hard gate. Attempting production without CBF triggers automatic redirect to S01.

### Law 2: EVERY PIECE RUNS THE ARENA
All content produced by S08-S13 must pass through the 7-persona Arena system. No single perspective dominates. The synthesis of competing approaches produces what no single approach would create.

**Enforcement:** Production skill outputs are labeled "PRE-ARENA" until arena-protocol completes.

### Law 3: SPECIMENS FIRST, THEN GENERATE
When creating any content format, relevant specimens MUST be loaded first. Teachings MUST be consulted. You don't know what "great" looks like until you've studied great.

**Enforcement:** CLAUDE-SPECIMENS.md defines required specimens per skill. Missing specimens = degraded quality alert.

### Law 4: VIRALITY IS SCORED, NOT ASSUMED
All content must be scored using the Virality Scoring Framework (VSF) before final approval. Score below 60 requires revision. Score below 40 is blocked.

**Scoring Dimensions:** EA (Emotional Activation), SC (Social Currency), PI (Pattern Interrupt), PF (Platform Fit), SH (Shareability) — each 1-10.
**Formula:** `VIRALITY SCORE = (EA x SC x PI x PF x SH) / 10,000`

### Law 5: PLATFORM-NATIVE OR NOTHING
Cross-posted content is lazy content. Every platform has specific algorithm signals, format preferences, audience behavior patterns, and optimal lengths/timing. Content must be **rebuilt** for each platform, not resized.

**Enforcement:** Platform Strategy File (PSF) defines requirements. Content violating PSF is flagged.

## 7 DEGRADATION PATTERNS

| # | Pattern | What Happens | Prevention |
|---|---------|-------------|------------|
| 1 | **Premature Creator** | Content before strategy complete | Gate system, CBF requirement |
| 2 | **Mono-Voice** | All content sounds the same | Mandatory Arena for all production |
| 3 | **Slop Machine** | Generic AI content about nothing | Anti-slop scanning, specificity requirements |
| 4 | **Island Content** | Content with no funnel connection | Content Architecture File maps to revenue |
| 5 | **One-Hit Wonder** | Success without system to replicate | Mandatory S19-S20 learning capture |
| 6 | **Platform Blind** | Same content across all platforms | PSF enforcement, platform-specific specs |
| 7 | **Copycat** | Imitating specimens without understanding | Teachings must be referenced, not just specimens |

**WHEN YOU NOTICE any pattern above:** STOP. Identify which degradation pattern is active. Re-read the relevant Law. Fix before proceeding.

## PER-MICROSKILL OUTPUT PROTOCOL

**Every microskill produces its own dedicated output file.** This is the marketing-os universal rule.

### 7 Universal Rules

1. Each microskill spec file names its exact output file
2. Output files go to the project output directory, not the skill directory
3. File naming: `[skill-id]-[microskill-id]-[name].md` or `.yaml`
4. No combining multiple microskill outputs into one file
5. "Synthesis" without reading individual specs = forbidden
6. If a microskill ran, its output file MUST exist
7. Execution log tracks every microskill with timestamp and output path

### 8 Forbidden Behaviors

1. Reading AGENT.md and synthesizing without reading individual microskill specs
2. Combining multiple outputs into one file to "save time"
3. Holding outputs in conversation context without writing to files
4. Claiming completion without verifiable output files
5. Skipping microskills that "seem unnecessary"
6. Generating output that doesn't match the spec's required schema
7. Producing empty/stub output files
8. Writing to wrong output paths

## ARENA PROTOCOL

The Organic Engine uses a 7-persona Arena system specific to organic content marketing.

**Full Protocol:** `arena/arena-protocol.md`
**Persona Files:** `arena/personas/` (7 files)

### 7 Organic Personas

| # | Name | Philosophy |
|---|------|-----------|
| 1 | Volume Machine | Consistency and speed, more at-bats |
| 2 | Value Architect | Give away premium value, edu-tainment |
| 3 | Virality Engineer | Data-driven format optimization |
| 4 | Community Builder | 1,000 true fans > 1M passive followers |
| 5 | Brand Purist | Less but better, every post is premium |
| 6 | Algorithm Hacker | Platform mechanics, trend surfing |
| 7 | Storyteller | Everything wrapped in narrative |

**Arena Mode:** `generative_full_draft` for production skills (S08-S13)
**Rounds:** 3 mandatory (no exceptions)
**Synthesis:** Pull strongest elements from each persona's output

## CONCEPT SEPARATION PROTOCOL

Skills S04 (Content Architecture), S05 (Hook Library), and S21 (Persona Architect) use concept separation:

```
Phase A: Generate CONCEPTS (ideas, structures, strategies)
    ↓
CONCEPT CHECKPOINT: Human reviews concepts WITHOUT packaging
    ↓
Phase B: Generate NAMES/PACKAGING (series names, hook copy, persona names)
```

This prevents good names from propping up weak concepts.

## COPYWRITINGENGINE INTEGRATION

The Organic Engine can operate in two modes:

### Mode A: Downstream from CopywritingEngine
When a full campaign runs through Skills 01-09, the Campaign Brief Package (`campaign-brief-package.json`) flows to S07 Campaign Brief, which translates it into an organic-specific CBF.

```
CE Skill 09 → campaign-brief-package.json → Organic S07 → CBF → S08-S14
```

### Mode B: Standalone
Organic S01-S07 runs independently to create a CBF from scratch.

## FORBIDDEN BEHAVIORS (Complete List)

### Generation Violations
1. Generating content without CBF (violates Law 1)
2. Skipping Arena for production skills (violates Law 2)
3. Generating without loading specimens AND teachings (violates Law 3)
4. Publishing without virality score (violates Law 4)
5. Cross-posting without platform adaptation (violates Law 5)

### Structural Violations
6. Skipping foundation skills (S01-S07 are sequential, mandatory)
7. Running fewer than 3 Arena rounds
8. Skipping any of the 7 Arena personas
9. Proceeding without human selection after Arena
10. Claiming skill completion without all output files

### Quality Violations
11. Slop density > 5% (automatic rewrite)
12. Virality score < 40 (blocked from production)
13. Holding outputs in conversation context without writing to files
14. Combining multiple microskill outputs into one file
15. Creating checkpoint YAML without verifying all microskill outputs

### Rationalization Violations (IMMEDIATE HALT)

| Rationalization | Why Invalid |
|-----------------|-------------|
| "This content doesn't need Arena" | Law 2. ALL production content. No exceptions. |
| "The virality score is close enough to 60" | Numbers are exact. 59 = revision required. |
| "We can skip S03 — the voice is obvious" | Foundation is sequential. No skipping. |
| "One Arena round produced great results" | 3 rounds mandatory. Round 1 = baseline, not peak. |
| "Cross-posting with minor edits counts as platform-native" | Law 5. Rebuilt, not resized. |
