# LLM-ANTI-DEGRADATION-SYSTEM.md — STRUCTURAL FORCING FRAMEWORK
## Preventing Quality Decay in OrganicMarketingEngine
## Version 1.0 — March 2026

---

## PURPOSE

LLMs degrade over long sessions. They take shortcuts. They produce generic outputs. They skip steps. This system exists to FORCE quality through structural mechanisms, not just prompting.

**The Problem:**
- LLMs optimize for completion, not excellence
- Context fatigue leads to corner-cutting
- Generic patterns are easier than specific outputs
- Without structural forcing, every output trends toward mediocrity

**The Solution:**
- Gates that block progress without prerequisites
- MC-CHECK metacognitive checkpoints
- Anti-slop scanning
- Context load management
- Session continuity protocols

## LAYER 1: STRUCTURAL FORCING (GATES)

### What Gates Are
Gates are HARD BLOCKS that prevent skills from executing without prerequisite files.

### Gate Registry

|------|--------|-------|
| G01 | S02 | S01 AIF exists and validates |
| G02 | S03 | S02 PSF exists and validates |
| G03 | S04 | S03 BVF exists and validates |
| G04 | S05 | S04 CAF exists and validates |

| G06 | S07 | S06 VSF exists and validates |

| G10 | S15-S18 | S14 output exists |
| G11 | S20 | S19 analysis complete |

### Gate Validation Protocol

When a skill is invoked:
```
1. CHECK: Does prerequisite file exist?
   - File path: [expected path]
   - EXISTS: ✓ or ✗

2. VALIDATE: Do required fields exist and pass rules?
   - Required fields: [list from PIPELINE-HANDOFF-REGISTRY]
   - VALID: ✓ or ✗
   - Failures: [specific field failures]

3. DECISION:
   - All checks pass → PROCEED
   - Any check fails → BLOCK + explain why
```

## LAYER 3: ANTI-SLOP ENFORCEMENT

### What Slop Is
Slop is generic, AI-flavored content that could have been written about anything by anyone. It destroys trust, wastes distribution, and signals low effort.

### Banned Phrases Registry

**Category: Opening Clichés**
- "In today's fast-paced world..."
- "Have you ever wondered..."
- "Let me tell you a story..."
- "Picture this..."
- "Imagine if..."
- "What if I told you..."

**Category: Closing Clichés**
- "Remember, [platitude]"
- "At the end of the day..."
- "The bottom line is..."
- "In conclusion..."
- "So what are you waiting for?"
- "Take action today!"

**Category: Filler Phrases**
- "It's important to note that..."
- "It goes without saying..."
- "Needless to say..."
- "As we all know..."
- "Experts agree that..."
- "Studies show..." (without citation)

**Category: Generic Metaphors**
- "Journey" (unless literally about travel)
- "Unlock your potential"
- "Take it to the next level"
- "Game-changer"
- "Paradigm shift"
- "Think outside the box"

**Category: AI-Signature Patterns**
- Starting with "I"
- Using "certainly" or "absolutely"
- Triple adjective stacking ("powerful, transformative, life-changing")
- Excessive em-dashes
- "Here's the thing..."

### Slop Density Metric

**Formula:**
```
Slop Density = (Banned Phrases Count / Total Sentences) × 100
```

**Thresholds:**
- 0%: Excellent (target)
- 1-2%: Acceptable
- 3-5%: Warning - revision needed
- >5%: Blocked - rewrite required

### Anti-Slop Scan Protocol

Every content output runs through this scan:

```
1. TOKENIZE content into sentences
2. CHECK each sentence against banned phrases list
3. FLAG matches with specific phrase
4. CALCULATE slop density
5. DECISION:
   - Density ≤ 2%: PASS (may publish)
   - Density 3-5%: WARN (revise flagged sentences)
   - Density > 5%: BLOCK (full rewrite)
6. REPORT specific flagged phrases for manual review
```

## LAYER 4: CONTEXT MANAGEMENT

### Context Load Zones

| Zone | Usage | Behavior |
|------|-------|----------|
| GREEN | 0-40% | Full operations. Load all relevant context. |
| YELLOW | 40-70% | Selective loading. Prioritize active skill only. Summarize completed work. |
| RED | 70-90% | Minimal loading. Single skill focus. Prepare handoff. |
| CRITICAL | 90%+ | Immediate handoff. Save full state. New session required. |

### Zone Transition Actions

**GREEN → YELLOW:**
- Summarize completed foundation files into key points
- Compress specimen references to IDs only
- Keep active skill protocol at full detail

**YELLOW → RED:**
- Archive all but current skill context
- Create condensed state summary
- Begin drafting handoff document

**RED → CRITICAL:**
- Stop current generation
- Create full handoff document
- Save to `learning-log/session-continuity/`
- Instruct user to start new session with handoff doc

## QUALITY PRINCIPLES (The 9 Laws)

These principles are checked at every MC-CHECK:

1. **Specificity Over Generality**
   Content must be specific to this brand, this audience, this platform. If you could swap the brand name and it still works, it's too generic.

2. **Density Over Length**
   Every sentence must add value. Cut filler. Compress. More value per word.

3. **Native Over Cross-Posted**
   Content must feel born on its platform. No cross-posting without full adaptation.

4. **Emotion Over Information**
   Content that informs but doesn't move fails. Find the emotional core.

5. **Hooks Over Everything**
   The first 3 seconds determine everything. Never settle for a weak hook.

6. **Shareability Over Impressions**
   Content that gets saved and shared > content that gets viewed and forgotten.

7. **Consistent Over Perfect**
   A good piece shipped beats a perfect piece stuck. But never ship slop.

8. **Compounding Over One-Off**
   Every piece should build on what came before and set up what comes next.

9. **Learnings Over Launches**
   An executed campaign that generates learnings > a planned campaign that never ships.
