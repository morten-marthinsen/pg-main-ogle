# 4.2: Launch Guide Writer

## Purpose

Write LAUNCH-GUIDE.md -- the single comprehensive document the media buyer reads before launching the campaign. This is the PRIMARY DELIVERABLE of A11, placed at the root of LAUNCH-PACKAGE/. It must be 30KB+ minimum and cover all 12 required sections from the AGENT.md OUTPUT SCHEMA: LAUNCH-GUIDE.md specification.

The launch guide synthesizes all documentation from Layers 1-3 into one cohesive, actionable document. The media buyer should be able to set up and launch the entire campaign using ONLY this guide (with the documentation/ files as reference).

**CHUNKED ASSEMBLY REQUIRED:** Due to the 30KB minimum, this document must be written in 5-8 write operations with size checkpoints after each phase.

## Inputs

- 4.1-launch-package-assembler.md output (package inventory, directory structure)
- All documentation deliverables:
  - NAMING-KEY.md, CAMPAIGN-HIERARCHY.md, AUDIENCE-STRATEGY.md,
    BUDGET-PLAN.md, TEST-PLAN.md, launch-sequence.md,
    LAUNCH-CHECKLIST.md, UTM-TRACKING.md, COMPLIANCE-REPORT.md,
    SPEC-COMPLIANCE-REPORT.md
- Campaign Brief (product, audience, objectives)
- 0.1-scoring-report-loader.md output (variant summary)
- 0.4-format-strategy-loader.md output (platform summary)

## Process

```
PHASE 1: HEADER + EXECUTIVE SUMMARY (>= 3KB cumulative)

  Write the following sections:

  # LAUNCH-GUIDE.md
  [Campaign name, date, version]

  ## Table of Contents
  [Numbered list of all 12 sections with anchor links]

  ## 1. Executive Summary
  - Campaign overview: product name, vertical, target audience
  - Total variants in launch package: [count] (broken down by platform)
  - Recommended initial test budget: $[X]/day total
  - Estimated timeline:
    - Phase 1: Test (7 days)
    - Phase 2: Scale (7 days)
    - Phase 3: Optimize (ongoing)
  - Key platform priorities: [primary platform], [secondary], etc.
  - Quick-start: where to begin (reference Section 3 for platform setup)

  ## 2. Package Contents
  - Complete inventory of all files in LAUNCH-PACKAGE/
  - Directory structure diagram (from 4.1)
  - How to use NAMING-KEY.md:
    - To trace any in-platform ad back to the variant matrix
    - To find the A10 score for any variant
    - To identify which hook/body/CTA/visual a variant uses

  SIZE CHECKPOINT: File >= 3KB -> IF NOT: HALT, expand sections

PHASE 2: PLATFORM SETUP GUIDES (>= 12KB cumulative)

  Write platform-specific setup guides:

  ## 3. Meta (Facebook + Instagram) Setup Guide
  - Step-by-step campaign creation:
    1. Navigate to Ads Manager
    2. Create new campaign with [objective] objective
    3. Configure CBO/ABO per CAMPAIGN-HIERARCHY.md recommendation
    4. Create ad sets per hierarchy (Broad, Interest A, Interest B, etc.)
    5. Configure targeting per AUDIENCE-STRATEGY.md
    6. Upload ad files from meta/ directory
    7. Paste ad copy from [source]
    8. Select CTA button
    9. Set destination URLs with UTM parameters from UTM-TRACKING.md
    10. Review and publish
  - Campaign structure: [summarize from CAMPAIGN-HIERARCHY.md]
  - Platform-specific notes:
    - Text overlay guidelines (< 20% for best distribution)
    - Aggregated Event Measurement for iOS 14+ compliance
    - Dynamic Creative recommendations

  ## 4. TikTok Setup Guide
  [Same level of detail, platform-specific steps]
  - Spark Ads vs. standard format guidance
  - TikTok-specific optimization notes

  ## 5. YouTube Setup Guide
  [Same level of detail, by ad type]
  - Pre-roll, bumper, skippable, shorts -- each has different setup
  - Thumbnail upload instructions
  - Companion banner notes

  ## 6. Google Display Setup Guide
  [Same level of detail]
  - Responsive display ad setup (upload assets, Google assembles)
  - Standard display size deployment
  - Performance Max option

  SIZE CHECKPOINT: File >= 12KB -> IF NOT: HALT, expand platform guides

PHASE 3: STRATEGY SECTIONS (>= 22KB cumulative)

  Write strategy walkthrough sections:

  ## 7. Campaign Structure Walkthrough
  - Full hierarchy diagram for each platform
    (Campaign -> Ad Set -> Ad tree with names and variant assignments)
  - Rationale for CBO vs. ABO decision
  - How variant testing maps to campaign structure
  - When to modify the structure (Phase 2 changes, consolidation)

  ## 8. Audience Targeting Walkthrough
  - Tier 1: Broad -- parameters, setup instructions
  - Tier 2: Interest -- specific interest names to add, per platform
  - Tier 3: Lookalike -- setup requirements (client data needed)
  - Tier 4: Retargeting -- pixel requirements, audience creation
  - Exclusions: what to exclude and why
  - Testing order: which tiers to enable first

  ## 9. Budget & Testing Walkthrough
  - Phase 1 budget per platform (daily amounts)
  - Per-variant minimum spend before kill/scale
  - Kill criteria (specific thresholds)
  - Scale criteria (specific thresholds)
  - Testing priority: hooks first, then bodies, then visuals, then CTAs
  - Budget increase methodology (20-30% increments, 24-48h between)
  - Budget reallocation triggers

  SIZE CHECKPOINT: File >= 22KB -> IF NOT: HALT, expand strategy sections

PHASE 4: LAUNCH OPERATIONS (>= 30KB cumulative)

  Write operational sections:

  ## 10. Launch Sequence
  - Phase 1: Initial test variants (list with platforms)
  - Phase 2: Backup variants (list with deployment triggers)
  - Phase 3: Scale/kill decisions (criteria and timing)
  - Monitoring cadence (24h, 48h, 72h, 7d check-ins)
  - Creative refresh triggers (CPA +30%, CTR -20%)
  - Day-by-day launch timeline

  ## 11. Tracking & Compliance
  - UTM parameter structure (copy-paste ready per variant)
  - Pixel verification steps (per platform)
  - Conversion event verification
  - Attribution window recommendations
  - Compliance notes per platform (from COMPLIANCE-REPORT.md)
  - Flagged items and resolutions

  ## 12. Launch Checklist
  - Complete pre-launch checklist (all 7 categories)
  - Each item with verification method
  - Sign-off fields
  - GO/NO-GO decision matrix

  SIZE CHECKPOINT: File >= 30KB -> IF NOT: HALT, expand operational sections

STEP FINAL: VERIFY FINAL SIZE
  - Measure final file size
  - IF < 30KB: HALT -- expand sections with more detail, examples, or troubleshooting
  - IF >= 30KB: PASS
```
