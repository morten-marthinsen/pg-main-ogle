# 3.3: Launch Sequence Planner

## Purpose

Define the exact go-live sequence: what launches when, what are the backup variants, and when to make kill/scale decisions. Without a launch sequence, the media buyer either launches everything at once (overwhelming the algorithm and budget) or picks randomly (no strategic intent). The 3-phase sequence ensures methodical testing, data-driven decisions, and creative backup deployment.

## Inputs

- 3.1-compliance-reviewer.md output (which variants are cleared for launch -- no FAILs)
- 0.1-scoring-report-loader.md output (A10 scores and test priority ranks)
- 2.3-budget-distribution-planner.md output (budget phases, kill/scale criteria)
- 2.4-testing-methodology-planner.md output (test priority order, Phase 1 variants)
- 2.1-campaign-hierarchy-architect.md output (campaign structure for deployment)

## Process

```
STEP 1: FILTER TO COMPLIANCE-CLEARED VARIANTS
  - From 3.1: identify which variants are PASS (cleared for launch)
  - Remove any FAIL variants from the launch sequence entirely
  - FLAG variants: include only if human has resolved the FLAG
  - This is the final variant pool for launch sequencing

STEP 2: DEFINE PHASE 1 -- INITIAL TEST (Days 1-7)
  SELECT Phase 1 variants:
    - Highest-scored variants from A10 (top test priority)
    - Aligned with test plan priority (hook tests first)
    - Number of Phase 1 variants per platform:
      - Meta: 3-5 variants (manageable test with sufficient budget)
      - TikTok: 3-5 variants
      - YouTube: 2-4 variants (higher cost per impression)
      - Google Display: 2-4 variants
    - Total Phase 1 variants should be testable within Phase 1 budget
      (each variant must get 3x AOV minimum spend within 7 days)

  DEFINE Phase 1 deployment:
    - Day 1: Launch all Phase 1 variants across all platforms
    - Monitoring cadence:
      - 24 hours: Sanity check
        -> Are all ads delivering? Any upload errors? Any immediate rejections?
        -> Check: impressions > 0 for each variant
      - 48 hours: Early signal check
        -> Hook rate trends (video views / impressions)
        -> CTR trends (clicks / impressions)
        -> Any variants with zero engagement? Investigate.
      - 72 hours: First kill evaluation
        -> Apply kill criteria from 2.3:
          - Spent 3x AOV with 0 conversions -> KILL
          - CPA > 2x target CPA after minimum sample -> KILL
          - Hook rate < 15% -> KILL
          - CTR < 0.5% -> KILL
        -> Kill confirmed losers. Reallocate budget to survivors.
      - 7 days: Phase 1 final evaluation
        -> Identify winners (meeting or beating success criteria from 2.3)
        -> Rank Phase 1 variants by performance
        -> Compare actual performance to A10 predictions
        -> Decide: which winners to scale in Phase 2?

STEP 3: DEFINE PHASE 2 -- BACKUP DEPLOY (Days 7-14)
  SELECT Phase 2 backup variants:
    - Next-highest-scored variants from A10 (after Phase 1 selections)
    - Replace killed Phase 1 variants with fresh creative
    - Complement winners with variants testing the next priority variable
      (e.g., if Phase 1 tested hooks, Phase 2 tests bodies with winning hook)

  DEFINE Phase 2 deployment:
    - Day 7-8: Deploy backup variants alongside Phase 1 winners
    - Scale Phase 1 winners (20-30% budget increase)
    - Monitoring cadence: same checkpoints as Phase 1
    - Kill criteria: same as Phase 1
    - Day 14: Phase 2 evaluation
      -> Which backup variants performed well?
      -> Which Phase 1 winners maintained performance at higher budget?
      -> Identify top performing variant-audience combinations

STEP 4: DEFINE PHASE 3 -- SCALE/KILL (Days 14-30)
  SCALE DECISIONS:
    - Winners from Phase 1 + Phase 2 get budget increases
    - Scale methodology: 20-30% increments, wait 24-48h between increases
    - Never more than double budget in a single week
    - Monitor CPA/ROAS daily during scaling

  KILL DECISIONS:
    - Any variant with sustained CPA > target after 7+ days -> KILL
    - Any variant showing declining performance over 3+ consecutive days -> FLAG for review
    - Platform-level decisions: if entire platform underperforms, reduce allocation

  CREATIVE REFRESH TRIGGERS:
    - Best variant CPA rises 30%+ from Phase 1 baseline -> creative fatigue
    - CTR decline of 20%+ over 7 days -> audience saturation
    - WHEN triggered: request new creative from A01-A10 pipeline

  ONGOING OPTIMIZATION:
    - Weekly performance review cadence
    - Budget reallocation based on platform and variant performance
    - Audience expansion (add new Lookalike segments, broader interests)
    - Feed performance data to A12 for systematic learning

STEP 5: PRODUCE LAUNCH TIMELINE
  | Day | Action | Variants | Budget | Decision Criteria |
  |-----|--------|----------|--------|-------------------|
  | 1   | Phase 1 launch | [list] | [budget] | N/A |
  | 1   | Sanity check | - | - | Ads delivering? |
  | 2   | Early signal | - | - | Hook rate, CTR trends |
  | 3   | First kill eval | - | - | Kill criteria from 2.3 |
  | 7   | Phase 1 final | - | - | Winners identified |
  | 7-8 | Phase 2 launch | [backup list] | [increased] | Scale winners + deploy backups |
  | 14  | Phase 2 final | - | - | Combined evaluation |
  | 14+ | Phase 3 scaling | [winners] | [scaled] | Ongoing optimization |

STEP 6: WRITE launch-sequence.md (Deliverable)
  - Write to LAUNCH-PACKAGE/documentation/launch-sequence.md

STEP 7: WRITE PER-MICROSKILL OUTPUT
  - Produce per-microskill output file at designated path
```
