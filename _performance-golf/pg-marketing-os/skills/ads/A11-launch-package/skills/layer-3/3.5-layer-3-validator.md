# 3.5: Layer 3 Validator

## Purpose

Validate all launch preparation outputs from Layer 3 for completeness and correctness. Verify: compliance review covers 100% of ads, UTM structure is complete and consistent, launch sequence has all 3 phases defined, and launch checklist has all 7 categories populated. Flag any FAIL compliance verdicts that have not been addressed. This is the final checkpoint before Gate 3 closes.

## Inputs

- 3.1-compliance-reviewer.md output
- 3.2-utm-tracking-builder.md output
- 3.3-launch-sequence-planner.md output
- 3.4-launch-checklist-generator.md output
- Layer 1 outputs (total ad count for compliance coverage verification)

## Process

```
STEP 1: VALIDATE COMPLIANCE REVIEW (3.1)
  - Check: compliance review covers 100% of ads in the package
    - Compare total ads reviewed (from 3.1) to total ads packaged (from Layer 1)
    - IF coverage < 100%: FAIL -- unreviewed ads exist
  - Check: compliance FAIL count = 0
    - IF FAIL count > 0: FAIL -- ads with violations must be modified or excluded
    - A FAIL verdict CANNOT be overridden by A11
  - Check: compliance FLAG count resolved
    - FLAGs require HUMAN DECISION
    - IF unresolved FLAGs exist: FAIL -- escalate to human
  - Check: vertical-specific compliance was applied
    - IF vertical constraints were not applied: WARN

STEP 2: VALIDATE UTM TRACKING (3.2)
  - Check: UTM parameters defined for every variant
    - Compare UTM parameter count to variant count
    - IF any variant missing UTM: FAIL
  - Check: UTM formatting is consistent
    - All lowercase: Y/N
    - No spaces or special characters: Y/N
    - All 5 standard parameters present per URL: Y/N
  - Check: pixel/conversion tracking checklist is complete
    - All platform-specific items present: Y/N

STEP 3: VALIDATE LAUNCH SEQUENCE (3.3)
  - Check: all 3 phases defined (Phase 1, Phase 2, Phase 3)
    - IF any phase missing: FAIL
  - Check: Phase 1 variants are identified
    - IF no Phase 1 variants listed: FAIL
  - Check: Phase 2 backup variants are identified
    - IF no backup variants: WARN (not blocking but risky)
  - Check: kill/scale criteria are present
    - IF no criteria: FAIL -- media buyer needs decision rules
  - Check: monitoring cadence is defined
    - IF no cadence: FAIL

STEP 4: VALIDATE LAUNCH CHECKLIST (3.4)
  - Check: all 7 categories are populated
    1. Account Setup
    2. Tracking
    3. Creative
    4. Targeting
    5. Budget
    6. Compliance
    7. Approval
  - IF any category missing: FAIL
  - Check: checklist item count is reasonable (minimum ~25 items across all categories)
  - Check: verification methods are present for items
  - Check: sign-off fields are present

STEP 5: CROSS-REFERENCE CHECKS
  - Compliance-cleared variants match Phase 1 launch list
    (Phase 1 should only contain PASS variants, not FLAG/FAIL)
  - UTM parameters exist for all Phase 1 variants
  - Checklist references all documentation files

STEP 6: COMPILE VALIDATION VERDICT
  - IF all checks pass: VERDICT = PASS -> Gate 3 can be created
  - IF any critical check fails: VERDICT = FAIL
  - Critical failures (BLOCK Gate 3):
    - Compliance FAIL count > 0
    - Unresolved compliance FLAGs
    - Missing UTM parameters
    - Missing launch phases
    - Missing checklist categories
  - Non-critical warnings:
    - No backup variants (risky but not blocking)
    - Vertical compliance not applied (quality concern but not structural block)

STEP 7: WRITE LAUNCH-PREP-VALIDATION.md
  - Write validation results to project outputs

STEP 8: WRITE PER-MICROSKILL OUTPUT
  - Produce per-microskill output file at designated path
```
