# 3.2: UTM Tracking Builder

## Purpose

Define the complete UTM parameter structure for all campaign tracking. Every variant must have properly formatted, consistent UTM parameters so that performance data in analytics can be traced back to specific creative variables (hook type, body framework, CTA, visual treatment, platform, audience tier). This also feeds A12 (Performance Learning Loop) which needs to map analytics data to creative decisions.

Additionally, compile a pixel/conversion tracking verification checklist so the media buyer can confirm all tracking infrastructure is operational before launch.

## Inputs

- 1.5-naming-key-generator.md output (variant IDs and naming cross-reference)
- 2.1-campaign-hierarchy-architect.md output (campaign names for utm_campaign)
- 2.2-audience-targeting-strategy.md output (targeting tiers for utm_term)
- 0.2-variant-matrix-loader.md output (variant components for utm_content)
- 0.4-format-strategy-loader.md output (platforms for utm_source)

## Process

```
STEP 1: DEFINE UTM PARAMETER TAXONOMY
  Standard UTM parameters and their values for this campaign:

  utm_source: [platform identifier]
    Values: facebook, instagram, tiktok, youtube, google_display
    Rule: lowercase, no spaces, consistent across all URLs

  utm_medium: [traffic type]
    Values: paid_social (Meta, TikTok), paid_video (YouTube),
            display (Google Display), paid_search (if applicable)
    Rule: consistent naming for medium grouping in analytics

  utm_campaign: [campaign identifier]
    Values: Follow naming convention from 2.1 campaign hierarchy
    Format: [project]-[platform]-[campaign-type]-[date]
    Example: healthproduct-meta-conversions-0222
    Rule: lowercase, hyphens for spaces, includes date for time tracking

  utm_content: [variant identifier]
    Values: variant ID from naming key
    Format: [concept]-[hook]-[body]-[cta]-[visual]
    Example: c1-hk03-bd01-ct02-visa
    Rule: maps directly to variant matrix coordinates for A12 analysis

  utm_term: [audience/targeting identifier]
    Values: From audience targeting strategy
    Format: [tier]-[stack-name]
    Example: broad, interest-stack-a, lookalike-1pct, retarget-30d
    Rule: identifies which audience the click came from

STEP 2: GENERATE UTM PARAMETER MATRIX
  FOR EACH variant x platform x audience combination:
    - Construct complete UTM string:
      ?utm_source=[source]&utm_medium=[medium]&utm_campaign=[campaign]
      &utm_content=[content]&utm_term=[term]
    - Verify: no spaces in any parameter value
    - Verify: consistent case (all lowercase)
    - Verify: no special characters except hyphens and underscores
    - Verify: each parameter is populated (no empty values)

STEP 3: VALIDATE UTM FORMATTING
  FOR EACH UTM string:
    - Check: properly formatted URL parameter syntax
    - Check: no duplicate parameter names
    - Check: no URL-unsafe characters
    - Check: all 5 standard parameters present
    - Check: values follow defined taxonomy (no ad-hoc values)

STEP 4: COMPILE PIXEL/CONVERSION TRACKING CHECKLIST
  Tracking infrastructure that must be verified before launch:

  META PIXEL:
    - [ ] Pixel ID configured in Ads Manager
    - [ ] Pixel installed on website (verify with Facebook Pixel Helper)
    - [ ] Standard events firing: PageView, ViewContent, AddToCart, Purchase
    - [ ] Custom conversions defined (if applicable)
    - [ ] Attribution window set (recommend: 7-day click, 1-day view)
    - [ ] Aggregated Event Measurement configured (iOS 14+ compliance)

  TIKTOK PIXEL:
    - [ ] Pixel ID configured in TikTok Ads Manager
    - [ ] Pixel installed on website (verify with TikTok Pixel Helper)
    - [ ] Events firing: PageView, ViewContent, CompletePayment
    - [ ] Attribution window set

  YOUTUBE / GOOGLE:
    - [ ] Google Ads conversion tracking installed
    - [ ] Google Analytics 4 linked to Google Ads
    - [ ] Conversion actions defined (purchase, lead, etc.)
    - [ ] Enhanced conversions configured (recommended)
    - [ ] Attribution model selected (data-driven recommended)

  GENERAL:
    - [ ] UTM parameters in all destination URLs
    - [ ] Landing page loads correctly with UTM parameters
    - [ ] UTM parameters appear in Google Analytics reports
    - [ ] Server-side tracking configured (if applicable)
    - [ ] CAPI (Conversions API) configured for Meta (recommended for accuracy)

STEP 5: WRITE UTM-TRACKING.md (Deliverable)
  - Write to LAUNCH-PACKAGE/documentation/UTM-TRACKING.md
  - Include: complete parameter matrix (copy-paste ready per variant)
  - Include: pixel/conversion tracking checklist
  - Include: troubleshooting guide for common tracking issues

STEP 6: WRITE PER-MICROSKILL OUTPUT
  - Produce per-microskill output file at designated path
```
