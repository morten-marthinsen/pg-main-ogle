# 3.4: Launch Checklist Generator

## Purpose

Generate the complete pre-launch checklist with every item that must be verified before go-live. This is the final QA document -- the media buyer works through it item by item before clicking "Publish" on any campaign. The checklist covers 7 categories: Account Setup, Tracking, Creative, Targeting, Budget, Compliance, and Approval. Every item has a verification method so the media buyer knows HOW to check it, not just WHAT to check.

## Inputs

- 3.1-compliance-reviewer.md output (compliance status per ad)
- 3.2-utm-tracking-builder.md output (UTM parameters and tracking checklist)
- 3.3-launch-sequence-planner.md output (which variants launch first)
- 2.1-campaign-hierarchy-architect.md output (campaign structure to verify)
- 2.2-audience-targeting-strategy.md output (audiences to configure)
- 2.3-budget-distribution-planner.md output (budgets to set)
- Layer 1 outputs (files to upload, specs to verify)

## Process

```
STEP 1: COMPILE CATEGORY 1 -- ACCOUNT SETUP
  Pre-launch account verification:
  - [ ] Ad account is active and in good standing (per platform)
    Verification: Log into each platform's Ads Manager, check account status
  - [ ] Payment method is valid and has sufficient funds/credit
    Verification: Check billing settings in each platform
  - [ ] Team members have correct permissions (ad creation, campaign management)
    Verification: Check team/access settings in each platform
  - [ ] Business Manager / Business Center is properly configured (Meta/TikTok)
    Verification: Check BM settings, domain verification, page connections
  - [ ] Google Ads account linked to Google Analytics 4
    Verification: Check Admin > Google Ads Linking in GA4
  - [ ] All platform accounts are linked to the same analytics/attribution system
    Verification: Cross-check account IDs in analytics platform

STEP 2: COMPILE CATEGORY 2 -- TRACKING
  Tracking infrastructure verification (from 3.2):
  - [ ] Meta Pixel installed and firing on website
    Verification: Use Facebook Pixel Helper browser extension on landing page
  - [ ] Meta standard events configured (PageView, ViewContent, AddToCart, Purchase)
    Verification: Trigger each event on website, check Events Manager for receipt
  - [ ] TikTok Pixel installed and firing (if TikTok is target)
    Verification: Use TikTok Pixel Helper on landing page
  - [ ] Google Ads conversion tracking installed
    Verification: Use Google Tag Assistant on landing page
  - [ ] Google Analytics 4 receiving data
    Verification: Check GA4 Realtime report while browsing site
  - [ ] UTM parameters in all destination URLs
    Verification: Click through each ad preview, check URL bar for UTM parameters
  - [ ] Landing page loads correctly WITH UTM parameters (no redirect stripping)
    Verification: Navigate to landing page with full UTM string, verify parameters persist
  - [ ] Conversions API (CAPI) configured for Meta (recommended)
    Verification: Check Events Manager for server-side events
  - [ ] Attribution windows set per platform recommendations
    Verification: Check in campaign settings for each platform

STEP 3: COMPILE CATEGORY 3 -- CREATIVE
  Creative file verification:
  - [ ] All Phase 1 ad files uploaded to each platform
    Verification: Check creative library / ad preview for each variant
  - [ ] All files passed spec compliance (from 1.6 report)
    Verification: Reference SPEC-COMPLIANCE-REPORT.md -- all PASS
  - [ ] File names match naming convention
    Verification: Cross-reference NAMING-KEY.md with uploaded files
  - [ ] In-platform ad names applied correctly
    Verification: Check ad names in Ads Manager match NAMING-KEY.md
  - [ ] Ad copy pasted correctly (no truncation, no formatting errors)
    Verification: Preview each ad, compare to A07 AD-COPY-FINAL
  - [ ] CTA buttons configured correctly per variant
    Verification: Preview each ad, check CTA button text and destination URL
  - [ ] YouTube thumbnails uploaded (if YouTube is target)
    Verification: Check thumbnail in YouTube video/ad setup
  - [ ] Landing page URLs correct (no broken links)
    Verification: Click through each ad preview to landing page

STEP 4: COMPILE CATEGORY 4 -- TARGETING
  Audience configuration verification:
  - [ ] Broad targeting ad sets created with NO restrictions
    Verification: Check ad set targeting -- should show "Broad" with no interests
  - [ ] Interest-based targeting stacks configured
    Verification: Check ad set targeting against AUDIENCE-STRATEGY.md interest lists
  - [ ] Lookalike audiences created and assigned (if data available)
    Verification: Check custom audiences list, verify lookalike source and %
  - [ ] Retargeting audiences created and assigned (if data available)
    Verification: Check custom audiences list, verify website visitor windows
  - [ ] Exclusions applied (existing customers, recent purchasers)
    Verification: Check ad set exclusion lists
  - [ ] Geographic targeting set correctly
    Verification: Check ad set geo targeting matches campaign brief
  - [ ] Age and demographic targeting set correctly (if applicable)
    Verification: Check ad set demographic settings
  - [ ] Placement targeting configured (automatic vs. manual)
    Verification: Check ad set placements match CAMPAIGN-HIERARCHY.md

STEP 5: COMPILE CATEGORY 5 -- BUDGET
  Budget configuration verification:
  - [ ] Campaign daily budgets set per BUDGET-PLAN.md
    Verification: Check campaign budget settings against plan
  - [ ] CBO/ABO correctly configured per campaign
    Verification: Check "Campaign Budget Optimization" toggle matches 2.1 recommendation
  - [ ] Bid strategy selected (lowest cost, cost cap, bid cap, etc.)
    Verification: Check bid strategy in campaign settings
  - [ ] Per-platform budgets match BUDGET-PLAN.md allocations
    Verification: Sum all campaign budgets per platform, compare to plan
  - [ ] Total daily spend matches overall Phase 1 budget
    Verification: Sum ALL campaign budgets across ALL platforms
  - [ ] Spending limits set if needed (to prevent overspend)
    Verification: Check account-level or campaign-level spend limits

STEP 6: COMPILE CATEGORY 6 -- COMPLIANCE
  Compliance verification:
  - [ ] All ads passed compliance review (COMPLIANCE-REPORT.md)
    Verification: Reference COMPLIANCE-REPORT.md -- 0 FAIL, 0 unresolved FLAG
  - [ ] Required disclaimers added to ad copy or landing page
    Verification: Check for vertical-specific disclaimers (FDA, financial risk, etc.)
  - [ ] Special ad categories configured if required (Meta: credit, housing, employment)
    Verification: Check campaign settings for special category designation
  - [ ] Age-gating applied if required (alcohol, gambling)
    Verification: Check ad set age restrictions
  - [ ] Political ad disclosures added if applicable
    Verification: Check for "Paid for by" disclaimers
  - [ ] Landing page compliance verified (matches ad claims, no deceptive elements)
    Verification: Review landing page for consistency with ad creative

STEP 7: COMPILE CATEGORY 7 -- APPROVAL
  Final approvals before launch:
  - [ ] Client sign-off on creative (all ad variants reviewed and approved)
    Verification: Written confirmation from client (email, message, or document)
  - [ ] Client sign-off on budget (Phase 1 spend approved)
    Verification: Written confirmation of budget authorization
  - [ ] Client sign-off on target audiences
    Verification: Written confirmation of audience targeting approach
  - [ ] Internal QA complete (all above categories checked)
    Verification: This checklist completed with all items checked
  - [ ] Launch date confirmed with all stakeholders
    Verification: Calendar confirmation with media buyer, client, and team

STEP 8: ADD SIGN-OFF FIELDS
  - Completed by: _______________
  - Date completed: _______________
  - Items checked: _____ / _____ (total)
  - Categories complete: _____ / 7
  - LAUNCH APPROVED: [ ] YES [ ] NO
  - Notes: _______________

STEP 9: WRITE LAUNCH-CHECKLIST.md (Deliverable)
  - Write to LAUNCH-PACKAGE/documentation/LAUNCH-CHECKLIST.md

STEP 10: WRITE PER-MICROSKILL OUTPUT
  - Produce per-microskill output file at designated path
```

1. All 7 categories are present (Account Setup, Tracking, Creative, Targeting, Budget, Compliance, Approval)
2. Every checklist item has a verification method (HOW to check, not just WHAT)
3. Checklist references source documents for each category
4. Sign-off fields are present at the end
5. Items are platform-specific where needed (not one-size-fits-all)
6. Compliance category references actual compliance report results
7. LAUNCH-CHECKLIST.md deliverable is written with checkbox format
8. Output file meets minimum 3KB size
