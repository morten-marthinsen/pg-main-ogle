# 1.2: TikTok Package Builder

## Purpose

Package all TikTok-targeted variants into the correct directory structure with correct file specifications, correct naming convention, and correct placement targeting. TikTok is exclusively vertical video (9:16, 1080x1920). Verify every file meets TikTok's exact technical specifications. Generate a per-variant spec sheet. Produce TIKTOK-PACKAGE-MANIFEST.md listing every file with its complete specifications.

This is a MECHANICAL ASSEMBLY operation -- no creative decisions are made here.

## Inputs

- 0.2-variant-matrix-loader.md output (approved variants targeting TikTok, with component IDs)
- 0.3-production-assets-loader.md output (asset file paths and preliminary specs)
- 0.4-format-strategy-loader.md output (TikTok placements in scope)
- AGENT.md PLATFORM SPECIFICATION REFERENCE -- TikTok section (hard technical requirements)
- AGENT.md NAMING CONVENTION section (file naming and in-platform naming rules)

## Process

```
STEP 1: IDENTIFY TIKTOK-TARGETED VARIANTS
  - Filter approved variant list for variants targeting TikTok
  - IF no variants target TikTok: Output N/A file, skip remaining steps
  - Count: total variants

STEP 2: CREATE DIRECTORY STRUCTURE
  - Create LAUNCH-PACKAGE/tiktok/

STEP 3: PROCESS EACH VARIANT
  FOR EACH TikTok-targeted approved variant:

    3a. LOCATE SOURCE ASSET
      - Find the production asset file(s) from A08 AD-ASSETS/
      - IF asset not found: FLAG as missing

    3b. VERIFY FILE SPECIFICATIONS
      VIDEO:
        - Codec: Must be H.264 or H.265
        - Container: Must be MP4 or MOV
        - File size: Must be < 500MB
        - Duration: Must be 5-60 seconds (recommended 21-34s for best performance)
        - Aspect ratio: 9:16 (1080x1920)
        - Resolution: Min 720x1280
        - Frame rate: 24-60fps
        - Audio: AAC or MP3

      STATIC IMAGE (Spark Ads):
        - Format: Must be JPG or PNG
        - File size: Must be < 100MB
        - Aspect ratio: 9:16 (1080x1920)

      IF ANY SPEC FAILS: Flag file with specific failure reason
        -> DO NOT include non-compliant files in the package
        -> Return to A08/A09 for re-export at correct specs

    3c. APPLY FILE NAMING CONVENTION
      - Rename file to: TIKTOK-[CONCEPT]-[HOOK]-[BODY]-[CTA]-[VISUAL].[ext]
      - Example: TIKTOK-C2-HK07-BD02-CT01-VIS-B.mp4
      - Record mapping: original file name -> new file name

    3d. GENERATE IN-PLATFORM AD NAME
      - Format: [Date]-[Concept]-[HookType]-[BodyVersion]-[CTAType]-[Visual]-[Notes]
      - Record mapping: file name -> in-platform name

    3e. COPY TO TARGET DIRECTORY
      - Place renamed file in LAUNCH-PACKAGE/tiktok/

    3f. RECORD PER-VARIANT SPEC SHEET
      - File name, original file name, format, codec, resolution,
        aspect ratio, duration, file size, audio format,
        spec compliance status (PASS/FAIL per check)

    3g. NOTE PERFORMANCE RECOMMENDATIONS
      - Flag videos in the 21-34s sweet spot (recommended for best TikTok performance)
      - Flag videos under 21s or over 34s (may underperform)
      - Note Spark Ads eligibility for UGC-style creative

STEP 4: GENERATE TIKTOK-PACKAGE-MANIFEST.md
  - Create LAUNCH-PACKAGE/tiktok/TIKTOK-PACKAGE-MANIFEST.md
  - Include:
    - Total files packaged
    - Complete per-file spec sheet table
    - File naming cross-reference (new name -> original name -> variant ID)
    - In-platform naming cross-reference
    - Duration analysis (how many in sweet spot vs. outside)
    - Spark Ads recommendations (which variants are UGC-suitable)
    - Any flagged files (spec failures or missing assets)

STEP 5: WRITE OUTPUT
  - Produce per-microskill output file at designated path
```
