# 1.1: Treatment Type Selector

## Purpose
For each script from A04, select the primary visual treatment type (1-5) using the Visual Treatment Selection Matrix. This is the most consequential Layer 1 decision -- treatment type drives color palette, typography, talent direction, shot style, and production tool selection downstream. Every script MUST have a treatment type assigned before Layer 2 begins.

## Inputs
- Script inventory from 0.1 (concept IDs, script IDs, frameworks, platforms, hook types)
- Competitive visual landscape from 0.3 (treatment type distribution, whitespace)
- Soul.md visual constraints from 0.5 (visual register, energy signature)
- Visual Treatment Selection Matrix (from AGENT.md reference section)

## Process
1. READ script inventory from 0.1 output
2. READ competitive visual landscape from 0.3 output
3. READ Soul.md constraints from 0.5 output (if available)
4. FOR EACH script:
   a. IDENTIFY script framework (PAS, AIDA, BAB, Hook-Body-CTA, Story, Edutainment, UGC-DR, Fast-Paced Viral)
   b. IDENTIFY target platform (Meta, TikTok, YouTube)
   c. IDENTIFY hook type category (A-J)
   d. CONSULT Script Framework --> Treatment Selection Matrix:
      - Cross-reference framework + platform to get primary treatment recommendation
   e. CONSULT Hook Type --> Treatment Influence Matrix:
      - Cross-reference hook category to refine treatment selection
   f. CHECK against competitive landscape:
      - Is this treatment overrepresented in competitive landscape? Consider differentiation.
      - Is there whitespace opportunity with a different treatment?
   g. CHECK against Soul.md constraints:
      - Does the treatment match the visual register?
      - Does the treatment match the energy signature?
   h. ASSIGN primary treatment type (1-5) with documented rationale
   i. NOTE any strategic overrides of matrix recommendations (with justification)
5. BUILD treatment assignment table
6. CHECK treatment variety across campaign:
   - If ALL concepts share one treatment type, flag for review
   - Document strategic justification if monotony is intentional
7. WRITE to output file

## Output File
- **Path:** `layer-1-outputs/1.1-treatment-type-selector.md`
- **Minimum size:** 3KB

## Quality Checks
- Every script has a treatment type assigned (zero unassigned)
- Every assignment has documented rationale (not "default")
- Treatment selection uses both matrices (framework + hook type)
- Competitive differentiation considered (not just matrix default)
- Treatment variety checked across campaign
- No platform-treatment mismatches without documented strategic reason (e.g., Polished on TikTok)

## Dependencies
- Layer 0 must be complete (GATE_0 = PASS)
- Specifically requires: 0.1 (script inventory), 0.3 (competitive landscape), 0.5 (Soul.md constraints)
- This is the FIRST Layer 1 microskill -- all others depend on treatment type decisions
