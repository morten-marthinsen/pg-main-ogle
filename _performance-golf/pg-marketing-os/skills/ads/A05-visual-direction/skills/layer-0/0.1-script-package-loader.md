# 0.1: Script Package Loader

## Purpose
Load A04 SCRIPT-PACKAGE.md. Extract all script architectures with modular beat structures, two-column AV format, assigned frameworks, hook integrations, ad lengths, and target platforms. Build a script inventory table mapping every concept to its script, framework, length, platform, and hook type. This inventory drives all downstream visual direction decisions.

## Inputs
- A04 SCRIPT-PACKAGE.md (REQUIRED -- A05 cannot execute without this)

## Process
1. READ A04 SCRIPT-PACKAGE.md from the project outputs directory
2. FOR EACH script in the package:
   a. EXTRACT concept ID
   b. EXTRACT script ID
   c. EXTRACT script framework (PAS, AIDA, BAB, Hook-Body-CTA, Story, Edutainment, UGC-DR, Fast-Paced Viral)
   d. EXTRACT ad length (in seconds)
   e. EXTRACT target platform(s) (Meta, TikTok, YouTube, Google Display)
   f. EXTRACT hook type classification (Category A-J from AD-HOOK-TAXONOMY.md)
   g. EXTRACT beat structure (list of beats with audio content)
   h. EXTRACT two-column AV format (audio column content per beat)
3. BUILD script inventory table with all extracted data
4. COUNT total scripts -- this is the denominator for all "100% coverage" checks downstream
5. VALIDATE: every script has all required fields populated
6. FLAG any scripts with missing fields
7. WRITE inventory to output file
