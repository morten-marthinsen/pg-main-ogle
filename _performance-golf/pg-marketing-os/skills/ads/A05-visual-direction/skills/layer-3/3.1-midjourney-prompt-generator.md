# 3.1: Midjourney Prompt Generator

## Purpose
For each concept requiring static images or visual references: generate Midjourney v6 prompts. Include: subject description, style parameters (--style, --stylize), aspect ratio (--ar), lighting description, color direction, mood keywords, negative prompts (--no), quality/chaos settings. Generate 2-3 prompt variants per required image (for A/B selection). Group prompts by: hero images, product shots, lifestyle shots, B-roll references, thumbnail images. These prompts feed directly to A08 (Visual/Video Production) for execution.

## Inputs
- Shot lists from 2.1 (shots that need static images or visual references)
- Product staging from 2.3 (product shot specifications)
- B-roll strategy from 2.5 (B-roll images needed)
- Thumbnail designs from 2.7 (thumbnail images for YouTube)
- Color palettes from 1.2 (color direction for prompts)
- Treatment types from 1.1 (style direction)

## Process
1. READ shot lists from 2.1
2. READ product staging from 2.3
3. READ B-roll strategy from 2.5
4. READ thumbnail designs from 2.7
5. READ color palettes from 1.2
6. READ treatment types from 1.1
7. IDENTIFY all images that need Midjourney generation:
   a. Hero/key visual images
   b. Product shots (where real photography unavailable)
   c. Lifestyle/environment reference images
   d. B-roll reference images
   e. Thumbnail images (YouTube)
   f. Motion graphics background elements
8. FOR EACH required image:
   a. CONSTRUCT Midjourney prompt:
      - Subject description (detailed, specific -- from shot list / staging spec)
      - Style direction (photorealistic, illustration, cinematic, etc.)
      - Lighting description (natural, studio, golden hour, etc.)
      - Color direction (incorporating palette hex or color descriptions)
      - Mood/atmosphere keywords
      - Camera/composition direction (angle, framing, DOF)
   b. ADD Midjourney parameters:
      - `--ar [X:Y]` (aspect ratio matching platform requirements)
      - `--stylize [0-1000]` (lower for realistic, higher for artistic)
      - `--no [negative prompts]` (what to exclude)
      - `--quality [.25/.5/1/2]` (quality level)
      - `--chaos [0-100]` (variation level)
      - `--style raw` (if photorealistic needed)
   c. VALIDATE prompt syntax (correct parameter format)
   d. GENERATE 2-3 variants per image (different angles, moods, or compositions)
9. GROUP prompts by category
10. WRITE to output file
