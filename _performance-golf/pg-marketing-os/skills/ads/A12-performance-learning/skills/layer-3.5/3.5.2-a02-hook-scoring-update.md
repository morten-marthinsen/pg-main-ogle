# 3.5.2: A02 Hook Scoring Update

**Version:** 1.0
**Layer:** 3.5 (Learning Propagation)
**Type:** Engine Propagation
**Model:** opus
**Dependencies:** Layer 3 outputs (GATE_3 must be PASS), 2.4 hook-type-performance
**Output:** layer-3.5-outputs/3.5.2-a02-hook-scoring-update.md

---

## Purpose

Update A02 hook scoring weights based on actual hook type performance. For each hook type used in the campaign: adjust scoring weight UP if it outperformed predicted score, adjust DOWN if it underperformed. Adjust vertical-specific hook type recommendations in the relevant vertical profile. Write A02-SCORING-PATCH.md documenting all proposed weight changes with justification and data backing. POST-CAMPAIGN mode only.

## Process

### Step 1: Verify Post-Campaign Mode and Gate 3

```
IF mode == MID_FLIGHT:
  --> SKIP -- produce deferral stub
  --> "A02 hook scoring update DEFERRED -- Mid-Flight mode."
```

### Step 2: Build Hook Scoring Adjustment Table

```
FOR EACH hook type used in the campaign with sufficient data:

  COMPARE actual performance to current scoring weight:
    hook_type: [type_id]
    hook_type_name: [name]
    current_scoring_weight: [from A02 if available, else "baseline 1.0"]
    actual_performance_data:
      avg_hook_rate: [X]%
      avg_CPA: $[X]
      win_rate: [X]%
      vs_benchmark: OUTPERFORMED / AT_BENCHMARK / UNDERPERFORMED
      outperformance_magnitude: [X]x (if outperformed)

  CALCULATE adjustment:
    IF outperformed benchmark by > 20%:
      --> INCREASE weight
      --> Magnitude: proportional to outperformance (1.2x = +0.2 weight increase)
      --> Example: "Hook type D3 outperformed by 2.3x. Increase weight from 1.0 to 1.3"
    IF underperformed benchmark by > 20%:
      --> DECREASE weight
      --> Magnitude: proportional to underperformance
      --> Example: "Hook type A2 underperformed by 40%. Decrease weight from 1.0 to 0.7"
    IF at benchmark (within 20%):
      --> NO CHANGE -- weight is appropriately calibrated
      --> Document: "Hook type [X] performing as expected. No weight change needed."

  CONSTRAINTS:
    - No weight below 0.3 (always leave room for testing)
    - No weight above 2.0 (prevent over-optimization on single campaign)
    - Changes > 0.5 require HIGH confidence backing
    - Changes 0.2-0.5 require MEDIUM confidence minimum
```

### Step 3: Build Vertical Profile Hook Recommendation Updates

```
FROM hook performance data:

  UPDATE vertical profile hook recommendations:
    CURRENT recommended hook types for [vertical]: [list]
    PROPOSED recommended hook types based on campaign data:
      1. [type] -- primary recommendation -- [data: CPA, win rate]
      2. [type] -- secondary recommendation
      3. [type] -- tertiary recommendation

    HOOK TYPES TO ADD to vertical recommendations:
      [types that outperformed but weren't previously recommended]

    HOOK TYPES TO REMOVE from vertical recommendations:
      [types that underperformed despite being recommended]
      NOTE: Only remove if underperformance is consistent (HIGH confidence)
```

### Step 4: Write A02-SCORING-PATCH.md

```
PRODUCE patch file:

  # A02-SCORING-PATCH.md
  ## Patch Metadata
  - Source: A12 Performance Learning Loop -- [project name]
  - Campaign: [dates]
  - Vertical: [vertical]
  - Status: PROPOSED

  ## Hook Scoring Weight Adjustments
  | Hook Type | ID | Current Weight | Proposed Weight | Change | Data Basis | Confidence |
  |-----------|-----|---------------|-----------------|--------|------------|------------|
  | [name]    | [X] | [X]           | [Y]             | +/-[Z] | [evidence] | [H/M/L]    |

  ## Vertical Profile Hook Recommendation Updates
  ### Additions
  - [hook_type]: Add to [vertical] recommended hooks. Basis: [performance data]
  ### Removals
  - [hook_type]: Remove from [vertical] recommended hooks. Basis: [performance data]
  ### Priority Reordering
  - New priority order: 1.[type] 2.[type] 3.[type]. Basis: [ranking data]

  ## Application Instructions
  - A02 scoring file: [path], [section], [current] --> [proposed]
  - Vertical profile: [path], [section], [current] --> [proposed]

  ## Approval Required
  [ ] Human has reviewed all scoring weight changes
  [ ] Human has reviewed vertical recommendation changes
```
