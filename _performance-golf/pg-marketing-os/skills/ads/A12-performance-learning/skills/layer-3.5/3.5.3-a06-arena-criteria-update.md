**Version:** 1.0

## Purpose

Update A06 Ad Arena judging criteria weights based on which criteria best predicted actual performance. If "Scroll-Stop Power" correlated strongly with hook rate, its weight should increase. If "Memorability" didn't correlate with any performance metric, consider reducing its weight. Also update platform nativeness scoring based on actual platform-specific performance patterns. Write A06-CRITERIA-PATCH.md documenting all proposed criteria weight changes. POST-CAMPAIGN mode only.

## Process

### Step 1: Verify Post-Campaign Mode and Gate 3

```
IF mode == MID_FLIGHT:
  --> SKIP -- produce deferral stub
```

### Step 2: Map Scoring Criteria to Arena Criteria

```
A06 Ad Arena uses judging criteria to evaluate creative variants.
A10 uses scoring criteria to predict variant performance.

These may overlap but are not identical:
  A10 Scoring Criteria --> predict performance before launch
  A06 Arena Criteria --> judge creative quality during Arena competition

CORRELATION DATA (from 2.3 scoring model audit):
  For each A10 criterion, we know its correlation with actual performance.
  If an A06 Arena criterion maps to an A10 criterion, we can use the
  correlation data to inform Arena weight adjustments.

MAP:
  A10 "Scroll-Stop Power" --> A06 "Hook Power" / "Scroll-Stop"
  A10 "Message Clarity" --> A06 "Clarity" / "Message Strength"
  A10 "Emotional Resonance" --> A06 "Emotional Impact"
  A10 "Platform Nativeness" --> A06 "Platform Fit"
  A10 "Memorability" --> A06 "Memorability"
  A10 "CTA Strength" --> A06 "CTA Effectiveness"

NOTE: Exact mapping depends on how A06 and A10 criteria are defined in this project.
Adapt mapping based on actual criterion names.
```

### Step 3: Calculate Arena Criteria Weight Adjustments

```
FOR EACH A06 Arena criterion that maps to an A10 criterion:

  actual_predictive_power = correlation from 2.3 scoring model audit
  current_A06_weight = current weight in Arena judging

  ADJUSTMENT LOGIC:
    IF criterion has HIGH predictive power (|r| > 0.5) AND current weight is low:
      --> INCREASE Arena weight
      --> Rationale: "Criterion [X] strongly predicts actual performance (r=[Y])
          but is underweighted in Arena judging"

    IF criterion has LOW predictive power (|r| < 0.2) AND current weight is high:
      --> DECREASE Arena weight
      --> Rationale: "Criterion [X] does not predict actual performance (r=[Y])
          but is overweighted in Arena judging"

    IF criterion has MODERATE predictive power (0.2-0.5):
      --> MINOR adjustment or NO CHANGE
      --> Document current weight as reasonably calibrated

  PLATFORM NATIVENESS SPECIFIC:
    From 3.3 platform learnings:
      IF platform nativeness correlated with performance on some platforms but not others:
        --> Recommend platform-conditional nativeness scoring
        --> "Increase nativeness weight for TikTok (r=0.6) but reduce for YouTube (r=0.1)"
```
