# 0.3: Prediction Loader

**Version:** 1.0
**Layer:** 0 (Foundation)
**Type:** Data Loading
**Model:** haiku
**Dependencies:** A10 SCORING-REPORT.md
**Output:** layer-0-outputs/0.3-prediction-loader.md

---

## Purpose

Load A10 SCORING-REPORT.md to extract pre-launch predictions. For each variant: predicted_rank, predicted_performance_tier, predicted_hook_rate, predicted_CPA_range, key_strength, key_risk. These become the comparison baseline for Layer 2's Prediction vs Reality analysis. Without predictions loaded, the engine cannot calibrate its predictive model.

---

## Inputs

```yaml
input:
  from_a10:
    scoring_report_path: "[project]/A10-scoring/SCORING-REPORT.md"
    # Contains: per-variant scoring, rankings, predicted performance tiers
    scoring_data_path: "[project]/A10-scoring/scoring-data.yaml"
    # Contains: raw scoring criteria values per variant (if exists)
```

---

## Process

### Step 1: Locate A10 Scoring Report

```
VERIFY: A10 SCORING-REPORT.md exists at specified path
IF NOT EXISTS:
  --> WARNING: "A10 Scoring Report not found. Prediction comparison in Layer 2 will be LIMITED."
  --> Prediction vs Reality analysis cannot run without this
  --> CAN still proceed with Layer 1 (metric calculation) and Layer 3 (learning extraction)
  --> But Layer 2 (prediction analysis) will be degraded
  --> LOG: "A10 predictions not available -- Layer 2 will skip prediction comparison"
```

### Step 2: Extract Per-Variant Predictions

```
READ SCORING-REPORT.md
FOR EACH variant scored:
  EXTRACT:
    - variant_id: unique identifier (must match A09 matrix and performance data)
    - predicted_rank: integer (1 = highest scored, N = lowest)
    - predicted_performance_tier: enum[TOP_PERFORMER, STRONG, AVERAGE, WEAK, AT_RISK]
    - predicted_overall_score: float (composite score from A10 criteria)
    - predicted_hook_rate_range: {low: float, high: float}  # if available
    - predicted_CPA_range: {low: float, high: float}  # if available
    - key_strength: string (A10's identified primary strength)
    - key_risk: string (A10's identified primary risk)
    - platform_nativeness_score: float (if available)
    - scroll_stop_power_score: float (if available)
```

### Step 3: Extract Scoring Criteria Values (if available)

```
IF scoring-data.yaml exists:
  READ scoring criteria values per variant
  FOR EACH variant:
    EXTRACT per-criterion scores:
      - scroll_stop_power: float
      - message_clarity: float
      - emotional_resonance: float
      - platform_nativeness: float
      - memorability: float
      - CTA_strength: float
      - [any other criteria used in A10]

  This enables Layer 2.3 (Scoring Model Audit) to calculate
  per-criterion correlation with actual performance.

IF scoring-data.yaml does NOT exist:
  --> WARNING: "Raw scoring data not available. Scoring model audit (2.3) will use overall scores only."
  --> Proceed with overall scores from SCORING-REPORT.md
```

### Step 4: Build Prediction Baseline Table

```
COMPILE prediction_baseline_table:
  For each variant with predictions:
    variant_id | predicted_rank | predicted_tier | predicted_score |
    predicted_hook_rate | predicted_CPA | key_strength | key_risk

COUNT:
  - variants_with_predictions: integer
  - variants_without_predictions: integer (variants in performance data but not scored)
  - prediction_coverage: variants_with_predictions / total_variants_in_data
```

## Quality Checks

1. A10 SCORING-REPORT.md located (or absence documented)
2. Per-variant predictions extracted with all available fields
3. variant_ids cross-reference with A09 matrix and performance data
4. Prediction coverage calculated (what percentage of tested variants have predictions)
5. Scoring criteria values extracted if raw data available
6. Prediction baseline table is complete and ready for Layer 2 comparison
