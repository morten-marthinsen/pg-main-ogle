# 0.4: Benchmark Loader

**Version:** 1.0
**Layer:** 0 (Foundation)
**Type:** Data Loading
**Model:** haiku
**Dependencies:** A01 AD-INTELLIGENCE-HANDOFF.md, AD-HOOK-TAXONOMY.md, Vertical Profile
**Output:** layer-0-outputs/0.4-benchmark-loader.md

---

## Purpose

Load competitive benchmarks from A01 AD-INTELLIGENCE-HANDOFF.md (vertical average CTR, CPA, hook rate, platform-specific benchmarks), performance benchmarks from AD-HOOK-TAXONOMY.md for hook type comparison, and vertical-specific thresholds from the vertical profile. These benchmarks provide the external reference points for evaluating campaign performance -- not just "did we hit our targets" but "how do we compare to the market."

---

## Inputs

```yaml
input:
  from_a01:
    intelligence_handoff_path: "[project]/A01-ad-intelligence/AD-INTELLIGENCE-HANDOFF.md"
    # Contains: competitive intelligence, vertical benchmarks, competitor creative patterns
  from_taxonomy:
    hook_taxonomy_path: "./References/AD-HOOK-TAXONOMY.md"
    # Contains: 32 hook types, 10 categories, performance benchmark data per type
  from_vertical:
    vertical_profile_path: "./skills/ads/ad-verticals/[vertical].md"
    # Contains: vertical-specific performance thresholds and patterns
  from_ad_engine:
    ad_engine_claude_path: "./skills/ads/AD-ENGINE-CLAUDE.md"
    # Contains: Creative Lifespan reference data, Testing Volume Reference
```

## Process

### Step 1: Load Competitive Benchmarks from A01

```
READ A01 AD-INTELLIGENCE-HANDOFF.md
EXTRACT:
  vertical_benchmarks:
    - vertical_average_CTR: float
    - vertical_average_CPA: float
    - vertical_average_hook_rate: float (if available)
    - vertical_average_ROAS: float (if available)
  platform_benchmarks:
    For each platform (meta, google, tiktok, youtube):
      - platform_average_CTR: float
      - platform_average_CPM: float
      - platform_average_CPC: float
  competitor_patterns:
    - dominant_hook_types: [string] (what competitors use most)
    - dominant_visual_styles: [string]
    - dominant_ad_formats: [string]
    - estimated_competitor_spend: float (if available)

IF A01 handoff does not exist:
  --> WARNING: "A01 Intelligence Handoff not found. Competitive benchmarking will be limited."
  --> Proceed with taxonomy benchmarks and vertical profile
```

### Step 2: Load Hook Type Benchmarks from Taxonomy

```
READ AD-HOOK-TAXONOMY.md
FOR EACH of the 32 hook types:
  EXTRACT benchmark data:
    - hook_type_id: string
    - hook_type_name: string
    - hook_category: enum[A-J]
    - benchmark_hook_rate: float (expected hook rate range)
    - benchmark_engagement: float (if available)
    - platform_effectiveness: {platform: effectiveness_level}
    - best_verticals: [string] (verticals where this type excels)
    - notes: string (performance context)

BUILD hook_type_benchmark_table for comparison in Layer 2.4
```

### Step 3: Load Vertical Profile Thresholds

```
READ vertical profile for the campaign's vertical
EXTRACT:
  - target_CPA: float (vertical-specific target)
  - target_ROAS: float (vertical-specific target)
  - target_hook_rate: float (vertical-specific benchmark)
  - typical_win_rate: float (what % of variants typically win in this vertical)
  - creative_fatigue_benchmark: float (typical creative half-life for this vertical)
  - compliance_flags: [string] (vertical-specific compliance concerns)
  - preferred_hook_types: [string] (hook types that historically work in this vertical)

IF vertical profile does not exist:
  --> WARNING: "Vertical profile not found. Using universal benchmarks."
  --> Fall back to AD-ENGINE-CLAUDE.md general benchmarks
```

### Step 4: Load Creative Lifespan Reference Data

```
READ AD-ENGINE-CLAUDE.md Creative Lifespan and Testing Volume Reference sections
EXTRACT:
  - creative_half_life_benchmarks: {spend_level: half_life_days}
  - fatigue_indicators: {metric: threshold}
  - testing_volume_recommendations: {variant_count: budget_recommendation}
  - frequency_fatigue_threshold: float (typical frequency at fatigue onset)
```

### Step 5: Compile Unified Benchmark Package

```
ASSEMBLE all benchmarks into unified reference:
  - competitive_benchmarks (from A01)
  - hook_type_benchmarks (from taxonomy)
  - vertical_thresholds (from vertical profile)
  - creative_lifespan_benchmarks (from AD-ENGINE-CLAUDE.md)

FLAG any gaps:
  - Which benchmark sources were available vs unavailable
  - Which metrics have external benchmarks vs no comparison point
```
