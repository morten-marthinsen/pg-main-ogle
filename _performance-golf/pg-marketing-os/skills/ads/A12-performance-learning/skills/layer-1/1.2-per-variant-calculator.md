# 1.2: Per-Variant Calculator

**Version:** 1.0
**Layer:** 1 (Performance Data Ingestion & Metric Calculation)
**Type:** Data Processing
**Model:** sonnet
**Dependencies:** 1.1 platform-normalizer output
**Output:** layer-1-outputs/1.2-per-variant-calculator.md

---

## Purpose

Calculate per-variant performance metrics. For each variant: CPA, ROAS, CTR, hook_rate, ThruPlay_rate, engagement_rate, conversion_rate, CPM, CPC. Calculate using the full date range. Calculate confidence intervals based on sample size. Flag variants with insufficient data (below 3x AOV spend or below 5,000 impressions). This is the core metric calculation engine -- every number used in subsequent analysis originates here.

## Process

### Step 1: Aggregate Per-Variant Totals

```
FOR EACH unique variant_id in normalized data:
  SUM across all dates:
    total_impressions = SUM(impressions)
    total_reach = SUM(reach)  # may not be additive -- use max if available
    total_link_clicks = SUM(link_clicks)
    total_conversions = SUM(conversions)
    total_spend = SUM(spend)
    total_revenue = SUM(revenue)  # null if not tracked
    total_video_views_3s = SUM(video_views_3s)
    total_video_completions = SUM(video_completions)
    total_engagement_actions = SUM(engagement_actions)
    days_active = COUNT(DISTINCT dates with impressions > 0)
    date_range = MIN(date) to MAX(date)
```

### Step 2: Calculate Primary Metrics

```
FOR EACH variant:

  PRIMARY METRICS (Decision-Making):
    CPA = total_spend / total_conversions
      IF total_conversions == 0: CPA = "NO_CONVERSIONS" (not infinity)
    ROAS = total_revenue / total_spend
      IF total_revenue is null: ROAS = "REVENUE_NOT_TRACKED"
      IF total_spend == 0: ROAS = "NO_SPEND"
    hook_rate = total_video_views_3s / total_impressions
      IF video format: calculate normally
      IF non-video format: hook_rate = "N/A_STATIC"
    conversion_rate = total_conversions / total_link_clicks
      IF total_link_clicks == 0: conversion_rate = "NO_CLICKS"

  SECONDARY METRICS (Diagnostic):
    CTR = total_link_clicks / total_impressions
    ThruPlay_rate = total_video_completions / total_impressions
      IF non-video: "N/A_STATIC"
    engagement_rate = total_engagement_actions / total_impressions
    CPM = total_spend / (total_impressions / 1000)
    CPC = total_spend / total_link_clicks
      IF total_link_clicks == 0: CPC = "NO_CLICKS"
    frequency = total_impressions / total_reach
      IF reach unavailable: frequency = "REACH_NOT_AVAILABLE"

  ALL METRICS: Round to 2 decimal places for percentages, $0.01 for currency
```

### Step 3: Calculate Confidence Intervals

```
FOR EACH variant:

  CONVERSION RATE CONFIDENCE (Wilson Score Interval):
    n = total_link_clicks (sample size)
    p = conversion_rate
    z = 1.645 for 90% confidence, 1.96 for 95% confidence

    IF n >= 100:
      lower_bound = (p + z²/2n - z*sqrt(p(1-p)/n + z²/4n²)) / (1 + z²/n)
      upper_bound = (p + z²/2n + z*sqrt(p(1-p)/n + z²/4n²)) / (1 + z²/n)
      confidence_level = "95%"
    ELIF n >= 30:
      Use same formula but note "90% confidence"
      confidence_level = "90%"
    ELSE:
      confidence_level = "INSUFFICIENT_SAMPLE"

  CPA CONFIDENCE:
    IF total_spend >= 3 * AOV AND total_conversions >= 10:
      CPA_confidence = "HIGH"
    ELIF total_spend >= 2 * AOV AND total_conversions >= 5:
      CPA_confidence = "MEDIUM"
    ELSE:
      CPA_confidence = "LOW"

  HOOK RATE CONFIDENCE:
    IF total_impressions >= 10000:
      hook_rate_confidence = "HIGH"
    ELIF total_impressions >= 5000:
      hook_rate_confidence = "MEDIUM"
    ELSE:
      hook_rate_confidence = "LOW"
```

### Step 4: Flag Insufficient Data Variants

```
FOR EACH variant:

  APPLY SUFFICIENCY THRESHOLDS:
    threshold_spend = 3 * AOV
    threshold_impressions = 5000
    threshold_days = 3

  DATA_SUFFICIENCY:
    IF total_spend >= threshold_spend
       AND total_impressions >= threshold_impressions
       AND days_active >= threshold_days:
      --> data_status = "SUFFICIENT"
    ELSE:
      --> data_status = "INSUFFICIENT"
      --> insufficient_reasons = []
        IF total_spend < threshold_spend:
          append "Spend $[X] below 3x AOV ($[threshold])"
        IF total_impressions < threshold_impressions:
          append "Impressions [X] below 5,000 minimum"
        IF days_active < threshold_days:
          append "Only [X] days active, need minimum 3"
```

### Step 5: Produce Per-Variant Metric Table

```
COMPILE per_variant_metrics_table:
  For EVERY variant (sufficient AND insufficient):

  +-------+------+------+------+------+------+------+-------+------+------+---------+
  | VAR   | CPA  | ROAS | CTR  | HOOK | THRP | ENGM | CONV  | IMPR | SPND | STATUS  |
  | ID    |      |      |      | RATE | RATE | RATE | RATE  |      |      |         |
  +-------+------+------+------+------+------+------+-------+------+------+---------+
  | [id]  | $[X] | [X]x | [X]% | [X]% | [X]% | [X]% | [X]% | [X]K | $[X] | SUFF   |
  +-------+------+------+------+------+------+------+-------+------+------+---------+

  SORT by CPA (ascending) for sufficient variants, then insufficient at bottom

CALCULATE campaign-level aggregates for context (but NOT as substitute for per-variant):
  campaign_CPA = total_campaign_spend / total_campaign_conversions
  campaign_ROAS = total_campaign_revenue / total_campaign_spend
  campaign_CTR = total_campaign_clicks / total_campaign_impressions
  campaign_average_hook_rate = mean of all variant hook rates (sufficient only)
```

## Quality Checks

1. Every matched variant has metrics calculated (no variants skipped)
2. All primary metrics (CPA, ROAS, hook_rate, conversion_rate) calculated for sufficient variants
3. Confidence intervals calculated for all sufficient variants
4. Insufficient data variants flagged with specific reasons
5. Campaign-level aggregates calculated for context
6. No impossible metric values (negative CPA, CPC > CPA, etc.)
7. All metrics rounded consistently (2 decimal places for percentages, $0.01 for currency)
8. Per-variant metric table is sorted and complete
