# 1.3: Time-Series Builder

**Version:** 1.0
**Layer:** 1 (Performance Data Ingestion & Metric Calculation)
**Type:** Data Processing
**Model:** sonnet
**Dependencies:** 1.1 platform-normalizer output
**Output:** layer-1-outputs/1.3-time-series-builder.md

---

## Purpose

Build daily time-series for each variant. Calculate: daily hook_rate, daily CPA, daily CTR, daily spend, cumulative spend. Calculate rolling 3-day averages to smooth noise. Identify performance inflection points (days where metrics shift significantly). Calculate creative fatigue curve: day-over-day hook rate change, 7-day performance drop percentage, estimated creative half-life. This is the temporal analysis layer -- aggregate numbers hide fatigue curves, inflection points, and performance trajectories.

## Process

### Step 1: Build Daily Time-Series Per Variant

```
FOR EACH variant with sufficient data (>= 3 days):
  BUILD daily_metrics array:
    For each date in the variant's data range:
      date: ISO date
      impressions: integer
      link_clicks: integer
      conversions: integer
      spend: float
      video_views_3s: integer
      cumulative_spend: float (running total)
      cumulative_conversions: integer (running total)
      cumulative_impressions: integer (running total)

      daily_metrics:
        hook_rate: video_views_3s / impressions (null if non-video)
        CTR: link_clicks / impressions
        CPA: spend / conversions (null if 0 conversions that day)
        CPC: spend / link_clicks (null if 0 clicks)
        CPM: spend / (impressions / 1000)
        engagement_rate: engagement_actions / impressions
```

### Step 2: Calculate Rolling Averages

```
FOR EACH variant's daily_metrics array:
  CALCULATE rolling 3-day averages:
    For each date (starting from day 3):
      rolling_3d_hook_rate = mean(hook_rate[day-2], hook_rate[day-1], hook_rate[day])
      rolling_3d_CTR = mean(CTR[day-2], CTR[day-1], CTR[day])
      rolling_3d_CPA = total_spend_3d / total_conversions_3d
        (NOT mean of daily CPAs -- use actual totals)
      rolling_3d_CPM = mean(CPM[day-2], CPM[day-1], CPM[day])

  PURPOSE: Smooths daily noise while preserving real trends.
  Single-day metrics can swing wildly with low spend.
  3-day rolling averages reveal actual trajectory.
```

### Step 3: Identify Performance Inflection Points

```
FOR EACH variant:
  SCAN rolling averages for inflection points:

  INFLECTION POINT = day where rolling average changes direction by > 15%

  For hook_rate:
    IF rolling_3d_hook_rate drops > 15% from previous 3-day:
      FLAG inflection: {date: [date], metric: "hook_rate",
        direction: "DOWN", magnitude: [X]%, previous: [X]%, current: [X]%}

    IF rolling_3d_hook_rate rises > 15% from previous 3-day:
      FLAG inflection: {date: [date], metric: "hook_rate",
        direction: "UP", magnitude: [X]%, previous: [X]%, current: [X]%}

  For CPA:
    IF rolling_3d_CPA increases > 25% from previous 3-day:
      FLAG inflection: {date: [date], metric: "CPA",
        direction: "UP" (worsening), magnitude: [X]%}

    IF rolling_3d_CPA decreases > 25% from previous 3-day:
      FLAG inflection: {date: [date], metric: "CPA",
        direction: "DOWN" (improving), magnitude: [X]%}

  CLASSIFY inflection significance:
    MAJOR: > 30% change
    MODERATE: 15-30% change
    MINOR: 10-15% change (logged but not flagged)
```

### Step 4: Calculate Creative Fatigue Curves

```
FOR EACH variant with >= 7 days of data:

  FATIGUE METRICS:

    1. Day-over-day hook rate change:
      For each consecutive day pair:
        dod_change = (hook_rate_today - hook_rate_yesterday) / hook_rate_yesterday
      Average daily decline: mean(all dod_change values)

    2. 7-day performance drop:
      peak_period = mean(hook_rate for days 1-3)  # or peak 3-day window
      recent_period = mean(hook_rate for most recent 3 days)
      seven_day_drop = (peak_period - recent_period) / peak_period
      BENCHMARK: 37% drop is typical at high spend levels

    3. Creative half-life estimation:
      peak_hook_rate = max(rolling_3d_hook_rate across all days)
      half_life_target = peak_hook_rate * 0.5
      half_life_day = first day where rolling_3d_hook_rate <= half_life_target
      IF never reaches half-life: estimated_half_life = "NOT_REACHED (still above 50%)"

    4. Frequency at fatigue onset:
      IF frequency data available:
        fatigue_onset_day = first inflection point where hook_rate drops > 15%
        frequency_at_onset = frequency on fatigue_onset_day
        BENCHMARK: Typical fatigue onset at frequency 2.5-3.5 for cold audiences

  FATIGUE CLASSIFICATION:
    IF seven_day_drop < 10%: "MINIMAL_FATIGUE"
    IF seven_day_drop 10-25%: "MODERATE_FATIGUE"
    IF seven_day_drop 25-50%: "SIGNIFICANT_FATIGUE"
    IF seven_day_drop > 50%: "SEVERE_FATIGUE"
```

### Step 5: Identify Peak Performance Windows

```
FOR EACH variant:
  IDENTIFY:
    peak_hook_rate_window: 3-day window with highest average hook rate
    peak_CPA_window: 3-day window with lowest CPA (best performance)
    peak_CTR_window: 3-day window with highest CTR

    days_to_peak: number of days from launch to peak performance
    days_of_sustained_performance: number of consecutive days within 20% of peak

  This data feeds Layer 3.4 (Fatigue Learnings) with specific windows
  and sustain periods for each variant.
```

## Quality Checks

1. Time series built for every variant with >= 3 days of data
2. Rolling 3-day averages calculated correctly (not averaging daily metrics but using totals)
3. Inflection points identified using consistent threshold (15% for moderate, 30% for major)
4. Creative fatigue curves calculated for variants with >= 7 days of data
5. Creative half-life estimated where data permits
6. Frequency at fatigue onset captured where frequency data available
7. Peak performance windows identified for each variant
8. Campaign-level fatigue summary produced
