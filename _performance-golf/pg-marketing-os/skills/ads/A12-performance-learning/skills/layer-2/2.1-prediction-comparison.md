# 2.1: Prediction Comparison

**Version:** 1.0
**Layer:** 2 (Prediction vs Reality Analysis)
**Type:** Analysis
**Model:** opus
**Dependencies:** Layer 1 outputs (GATE_1 must be PASS), 0.3 prediction-loader output
**Output:** layer-2-outputs/2.1-prediction-comparison.md

---

## Purpose

For each variant, compare A10's pre-launch predictions against actual performance. Compare: predicted rank vs actual rank, predicted performance tier vs actual tier, predicted hook rate range vs actual hook rate, predicted CPA range vs actual CPA. Classify each prediction as ACCURATE, OPTIMISTIC, PESSIMISTIC, or WILDLY_WRONG. Calculate overall prediction accuracy rate. This is where the engine measures how well it predicted reality -- the foundation for calibrating the predictive model.

## Process

### Step 1: Verify Gate 1 and Prediction Availability

```
READ checkpoints/LAYER_1_COMPLETE.yaml
IF NOT EXISTS:
  --> HALT: "GATE_1 not passed. Cannot begin Layer 2."

CHECK 0.3-prediction-loader.md for prediction availability:
IF predictions.load_status == "NOT_AVAILABLE":
  --> WARNING: "A10 predictions not available. Prediction comparison CANNOT be performed."
  --> This microskill produces a STUB output documenting the gap
  --> Layer 2 continues with 2.3, 2.4, 2.5 (which don't require predictions)
  --> This is NOT a gate failure -- it is a documented limitation
```

### Step 2: Per-Variant Prediction vs Actual Comparison

```
FOR EACH variant that has BOTH a prediction AND sufficient actual data:

  RANK COMPARISON:
    predicted_rank: [from A10]
    actual_rank: [rank by CPA, 1 = best] (among sufficient-data variants only)
    rank_delta: predicted_rank - actual_rank
    rank_direction: "HIGHER_THAN_PREDICTED" (if actual rank < predicted)
                    "LOWER_THAN_PREDICTED" (if actual rank > predicted)
                    "ACCURATE" (if within ±2 ranks)

  TIER COMPARISON:
    predicted_tier: [from A10 -- TOP_PERFORMER, STRONG, AVERAGE, WEAK, AT_RISK]
    actual_tier: [from 1.4 -- WINNER, PERFORMER, LOSER]
    tier_mapping:
      TOP_PERFORMER/STRONG --> expect WINNER
      AVERAGE --> expect PERFORMER
      WEAK/AT_RISK --> expect LOSER

    tier_match: boolean (does predicted map to actual?)

  HOOK RATE COMPARISON (if predicted range available):
    predicted_hook_rate_range: {low: [X], high: [X]}
    actual_hook_rate: [X]
    within_predicted_range: boolean
    deviation: actual - midpoint(predicted_range)
    deviation_direction: ABOVE / BELOW / WITHIN

  CPA COMPARISON (if predicted range available):
    predicted_CPA_range: {low: [X], high: [X]}
    actual_CPA: [X]
    within_predicted_range: boolean
    deviation: actual - midpoint(predicted_range)
    deviation_direction: BETTER (actual < predicted low) / WITHIN / WORSE (actual > predicted high)
```

### Step 3: Classify Prediction Accuracy Per Variant

```
FOR EACH compared variant:

  CLASSIFICATION LOGIC:
    ACCURATE:
      Tier prediction maps to actual classification
      AND (hook rate within predicted range OR within 20% of midpoint)
      AND (CPA within predicted range OR within 30% of midpoint)

    OPTIMISTIC:
      Predicted better performance than actual
      e.g., predicted TOP_PERFORMER but actual is LOSER
      OR predicted low CPA range but actual CPA significantly higher
      OR predicted high hook rate but actual significantly lower

    PESSIMISTIC:
      Predicted worse performance than actual
      e.g., predicted WEAK but actual is WINNER
      OR predicted high CPA but actual CPA significantly lower
      OR predicted low hook rate but actual significantly higher

    WILDLY_WRONG:
      Predicted tier is 2+ tiers away from actual
      e.g., predicted TOP_PERFORMER but actual is LOSER
      OR predicted AT_RISK but actual is WINNER
      OR CPA deviation > 3x predicted range
```

### Step 4: Calculate Overall Prediction Accuracy

```
CALCULATE:
  total_compared: count of variants with both prediction and actual data
  accurate_count: count(ACCURATE)
  optimistic_count: count(OPTIMISTIC)
  pessimistic_count: count(PESSIMISTIC)
  wildly_wrong_count: count(WILDLY_WRONG)

  overall_accuracy_rate = accurate_count / total_compared * 100
  BENCHMARK: 50-70% accuracy is considered good for creative performance prediction

  RANK CORRELATION (Spearman):
    Calculate Spearman rank correlation between predicted_rank and actual_rank
    r > 0.5 = strong predictive power
    r > 0.3 = moderate predictive power
    r < 0.3 = weak predictive power (model needs significant recalibration)
    r < 0 = inverse prediction (model is wrong about what drives performance)

  TIER ACCURACY:
    tier_accuracy_rate = count(tier_match) / total_compared * 100
    BENCHMARK: 40-60% tier accuracy is considered acceptable

  SYSTEMATIC BIAS DETECTION:
    IF optimistic_count > 2 * pessimistic_count:
      --> BIAS: "Scoring model is systematically OPTIMISTIC"
      --> The engine overestimates creative performance
    IF pessimistic_count > 2 * optimistic_count:
      --> BIAS: "Scoring model is systematically PESSIMISTIC"
      --> The engine underestimates creative performance
```

```
PRODUCE the MANDATORY PREDICTION ACCURACY TABLE:

+-----------------------------------------------------------------------------+

|  +-----------------------------------------------------------------------+  |
|  | METRIC                      | VALUE    | BENCHMARK | STATUS            |  |

|  | ACCURATE predictions        | [X]/[Y]  | --        | --                |  |
|  | OPTIMISTIC predictions      | [X]/[Y]  | --        | --                |  |
|  | PESSIMISTIC predictions     | [X]/[Y]  | --        | --                |  |
|  | WILDLY_WRONG predictions    | [X]/[Y]  | --        | --                |  |
|  +-----------------------------------------------------------------------+  |
```

2. Prediction classification uses consistent logic (ACCURATE/OPTIMISTIC/PESSIMISTIC/WILDLY_WRONG)
3. Spearman rank correlation calculated correctly
4. Systematic bias detected and documented
5. Prediction Accuracy Table produced in required format
6. If predictions unavailable, documented as NOT_AVAILABLE (not fabricated)
7. Edge cases handled (variants with predictions but insufficient actual data excluded)
8. Benchmarks cited for accuracy rate and correlation interpretation
