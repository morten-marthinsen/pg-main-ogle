# 2.3: Scoring Model Audit

**Version:** 1.0
**Layer:** 2 (Prediction vs Reality Analysis)
**Type:** Analysis
**Model:** opus
**Dependencies:** Layer 1 outputs, 0.3 prediction-loader output
**Output:** layer-2-outputs/2.3-scoring-model-audit.md

---

## Purpose

Audit A10's scoring model: which scoring criteria correlated with actual performance, which did not. Calculate correlation between A10 scroll-stop power score and actual hook rate, correlation between A10 overall score and actual ROAS, correlation between A10 platform nativeness score and actual platform performance. Identify the strongest and weakest predictive criteria. Recommend scoring weight adjustments for A10. This directly feeds Layer 3.5.4 (A10 Prediction Model Update).

## Process

### Step 1: Build Scoring-Performance Paired Dataset

```
FOR EACH variant with BOTH A10 scores AND sufficient actual data:
  CREATE paired record:
    variant_id: [id]
    a10_scores:
      overall_score: float
      scroll_stop_power: float (if available)
      message_clarity: float (if available)
      emotional_resonance: float (if available)
      platform_nativeness: float (if available)
      memorability: float (if available)
      CTA_strength: float (if available)
      [other criteria as available]
    actual_performance:
      CPA: float
      ROAS: float
      hook_rate: float
      CTR: float
      conversion_rate: float
      classification: WINNER/PERFORMER/LOSER

IF per-criterion scores NOT available (only overall scores):
  --> WARNING: "Only overall scores available. Per-criterion correlation analysis limited."
  --> Proceed with overall score correlation only
  --> This is a data limitation, not a gate failure
```

### Step 2: Calculate Per-Criterion Correlations

```
FOR EACH available scoring criterion:
  CALCULATE Pearson correlation with each actual performance metric:

    scroll_stop_power vs actual_hook_rate:
      r = correlation(scroll_stop_power_scores, actual_hook_rates)
      INTERPRETATION: Should be the strongest correlation.
        If weak (r < 0.3): scroll-stop scoring doesn't predict scroll-stop behavior
        If strong (r > 0.5): scoring criterion is well-calibrated

    overall_score vs actual_CPA:
      r = correlation(overall_scores, 1/actual_CPAs)  # inverse because lower CPA = better
      INTERPRETATION: Should be moderate-strong.
        If weak: overall scoring doesn't predict business outcomes

    overall_score vs actual_ROAS:
      r = correlation(overall_scores, actual_ROAS)
      INTERPRETATION: Direct relationship expected.

    platform_nativeness vs actual_platform_performance:
      r = correlation(nativeness_scores, actual_CTR_on_that_platform)
      INTERPRETATION: Does platform nativeness scoring predict platform-specific CTR?

    emotional_resonance vs actual_engagement_rate:
      r = correlation(resonance_scores, actual_engagement_rates)
      INTERPRETATION: Does emotional scoring predict engagement?

    CTA_strength vs actual_conversion_rate:
      r = correlation(CTA_scores, actual_conversion_rates)
      INTERPRETATION: Does CTA scoring predict post-click conversion?

  RANK criteria by predictive power:
    strongest_criterion: criterion with highest |r| to CPA/ROAS
    weakest_criterion: criterion with lowest |r| to CPA/ROAS
```

### Step 3: Classification Prediction Analysis

```
ANALYZE: How well did A10 overall scores predict winner/loser classification?

  CALCULATE:
    avg_score_winners = mean(overall_score for actual WINNERs)
    avg_score_performers = mean(overall_score for actual PERFORMERs)
    avg_score_losers = mean(overall_score for actual LOSERs)

    score_separation:
      winner_vs_loser_gap = avg_score_winners - avg_score_losers
      IF gap > 1.0: "STRONG separation -- scoring distinguishes winners from losers"
      IF gap 0.5-1.0: "MODERATE separation -- scoring has predictive value"
      IF gap < 0.5: "WEAK separation -- scoring doesn't meaningfully distinguish quality"

    false_positives: count of high-scored variants (top quartile) that were LOSERs
    false_negatives: count of low-scored variants (bottom quartile) that were WINNERs

    IF false_positives > 30%: "Scoring model has significant false positive problem"
    IF false_negatives > 20%: "Scoring model misses potential winners"
```

### Step 4: Recommend Scoring Weight Adjustments

```
BASED ON correlation analysis and classification prediction:

FOR EACH criterion:
  current_weight: [from A10 scoring model]
  correlation_with_CPA: [r value]
  correlation_with_hook_rate: [r value]

  RECOMMENDATION:
    IF |r| > 0.5 AND current_weight is low:
      --> "INCREASE weight -- high predictive value not reflected in scoring"
      --> Proposed_adjustment: +[X]

    IF |r| < 0.2 AND current_weight is high:
      --> "DECREASE weight -- low predictive value currently overweighted"
      --> Proposed_adjustment: -[X]

    IF |r| < 0.1:
      --> "CONSIDER REMOVING or REPLACING -- no predictive value detected"
      --> Note: May need more data before removing a criterion

    IF criterion not in A10 but correlates with performance:
      --> "CONSIDER ADDING as new criterion -- [description] correlates at r=[X]"

  PRODUCE recommended_weight_table:
    | Criterion | Current Weight | Correlation | Recommended Weight | Change | Justification |
```

6. Weight adjustment recommendations include specific magnitude and justification

8. Recommended weight table produced with current, recommended, and change columns
