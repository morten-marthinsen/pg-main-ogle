# 2.5: Run Duration Estimator

**Version:** 1.1
**Layer:** 2 (Classification)
**Type:** Performance Estimation
**Dependencies:** Layer 1 complete (LAYER_1_COMPLETE.yaml exists)
**Model:** opus
**Updated:** 2026-02-27 (v1.1 — dual-signal scoring support)

---

## Purpose

Estimate how long each scraped ad has been running using available date signals (start date, last seen date, platform indicators). Assign each ad to a performance tier based on run duration. Long-running ads are the strongest proxy for winning ads in the absence of actual performance data. This tier classification drives specimen selection in Layer 3 (3.4) and informs opportunity gap analysis (3.5).

**v1.1 Note:** When impression data is available (Tool-Assisted Scan mode), this microskill produces the run-duration-only tier assignments as before. The dual-signal performance scoring (combining run duration + impressions) is handled by microskill 2.7 (Impression Scorer), which runs AFTER this microskill and ENRICHES the tier assignments. This ensures backward compatibility — 2.5 always produces the same output regardless of mode.

input:

      TIER_PERFORMER: "30-89 days running — established performers"

      TIER_NEW: "< 7 days running — new/unproven"

## Process

### Step 1: Extract Date Signals

```
FOR EACH scraped ad:

  EXTRACT available date data:
    start_date: string  # from scraping output (may be null)
    last_seen_date: string  # from scraping output (may be null)
    platform_recency_signal: string  # e.g., "active" / "30d" / "90d" markers
    scan_date: string  # date this scan was performed

  DETERMINE estimation method:
    METHOD_A: start_date AND last_seen_date available
      → estimated_run_days = last_seen_date - start_date
      → confidence: "high"

    METHOD_B: start_date available, last_seen_date NOT available
      → estimated_run_days = scan_date - start_date
      → confidence: "medium" (assumes ad is still running)

    METHOD_C: Neither date available, but platform recency signal exists
      → Use platform signal to estimate:
        "active" → TIER_TESTING (assume 14 days)
        "30d" → TIER_TESTING or TIER_PERFORMER (use 30 days)
        "90d" → TIER_PERFORMER or TIER_WINNER (use 90 days)
      → confidence: "low"

    METHOD_D: No date data at all
      → estimated_run_days = null
      → tier: "TIER_UNKNOWN"
      → confidence: "none"
```

### Step 2: Tier Assignment

```
FOR EACH ad with estimated_run_days:

  IF estimated_run_days >= 90:
    tier: "TIER_WINNER"
    tier_label: "Proven Winner (90+ days)"

  ELSE IF estimated_run_days >= 30:
    tier: "TIER_PERFORMER"
    tier_label: "Established Performer (30-89 days)"

  ELSE IF estimated_run_days >= 7:
    tier: "TIER_TESTING"
    tier_label: "Active Test (7-29 days)"

  ELSE IF estimated_run_days >= 0:
    tier: "TIER_NEW"
    tier_label: "New/Unproven (< 7 days)"

  ELSE:
    tier: "TIER_UNKNOWN"
    tier_label: "Duration Unknown"

  RECORD:
    ad_duration_classification:
      ad_id: string
      brand_name: string
      platform: string
      start_date: string
      last_seen_date: string
      estimated_run_days: integer
      tier: string
      tier_label: string
      estimation_method: string  # METHOD_A/B/C/D
      estimation_confidence: string  # high/medium/low/none
```

### Step 3: Distribution Calculation

```
CALCULATE tier distribution:

  OVERALL:
    - TIER_WINNER: count, percentage
    - TIER_PERFORMER: count, percentage
    - TIER_TESTING: count, percentage
    - TIER_NEW: count, percentage
    - TIER_UNKNOWN: count, percentage

  PER PLATFORM:
    FOR EACH platform:
      Same tier distribution breakdown

  PER COMPETITOR BRAND:
    FOR EACH brand:
      Same tier distribution breakdown
      PLUS:
        brand_longevity_score: float
          # Weighted score: (TIER_WINNER * 4 + TIER_PERFORMER * 3 +
          #   TIER_TESTING * 2 + TIER_NEW * 1) / total_ads
          # Higher = brand runs ads longer (more winners/performers)
```

### Step 4: Winner Identification

```
EXTRACT all TIER_WINNER ads:
  Sort by estimated_run_days (descending)
  These are the primary candidates for specimen extraction in 3.4

EXTRACT all TIER_PERFORMER ads:
  Sort by estimated_run_days (descending)
  These are secondary candidates if TIER_WINNER count < 20

CREATE winner_candidate_list:
  Prioritized list of ads for specimen extraction:
  1. All TIER_WINNER ads (sorted by run duration)
  2. Top TIER_PERFORMER ads (if needed to reach 20+ specimens)

LOG:
  "TIER_WINNER ads available: [count]"
  "TIER_PERFORMER ads available: [count]"
  "Total specimen candidates: [count]"

  IF TIER_WINNER + TIER_PERFORMER < 20:
    WARNING: "Fewer than 20 high-confidence winning ads. Layer 3.4 may need
              to include TIER_TESTING ads with justification."
```

### Step 5: Estimation Quality Assessment

```
CALCULATE estimation quality metrics:
  - high_confidence_pct: percentage classified with METHOD_A
  - medium_confidence_pct: percentage classified with METHOD_B
  - low_confidence_pct: percentage classified with METHOD_C
  - unknown_pct: percentage classified with METHOD_D

  IF unknown_pct > 30%:
    WARNING: "Over 30% of ads have unknown run duration. Tier-based
              analysis has significant uncertainty. Consider supplementing
              with other winning signals (engagement data, creative iteration count)."

  IF high_confidence_pct < 20%:
    WARNING: "Fewer than 20% of ads have high-confidence duration estimates.
              Results should be interpreted cautiously."
```

## Quality Checks

1. **100% of ads processed** -- every ad has a tier assignment (including TIER_UNKNOWN for those without date data).
2. **Estimation methods documented** -- every ad shows which method was used and confidence level.
3. **TIER_UNKNOWN count tracked** -- the percentage of ads without duration data is explicit.
4. **Winner candidate list produced** -- ranked list of top ads for specimen extraction in Layer 3.
5. **Brand longevity scores calculated** -- every brand has a longevity score for competitive comparison.
6. **Quality assessment honest** -- if most ads lack date data, the limitations are clearly stated.
7. **Tier thresholds applied correctly** -- 90+ = WINNER, 30-89 = PERFORMER, 7-29 = TESTING, <7 = NEW.
