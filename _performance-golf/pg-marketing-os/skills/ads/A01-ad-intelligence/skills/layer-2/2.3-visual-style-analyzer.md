# 2.3: Visual Style Analyzer

**Version:** 1.0
**Layer:** 2 (Classification)
**Type:** Visual Classification
**Dependencies:** Layer 1 complete (LAYER_1_COMPLETE.yaml exists)
**Model:** opus

---

## Purpose

Classify each scraped ad's visual style from a defined taxonomy of 9 visual styles. Identify dominant visual patterns per competitor brand and per platform. Calculate visual style distribution to reveal what creative approaches the market is using and where visual style gaps exist.

This data feeds Layer 3 format-platform intelligence (3.3) and opportunity gap analysis (3.5).

---

## Inputs

```yaml
input:
  from_layer_1:
    all_scraped_ads: object  # Combined output from 1.1, 1.2, 1.3, 1.4
    total_ad_count: integer  # From 1.5 validation
  reference:
    visual_style_taxonomy:
      - ugc (user-generated content feel — handheld, personal, informal)
      - polished_professional (studio-quality, brand-consistent, high production)
      - motion_graphics (animated text, graphics, data visualization)
      - text_overlay_heavy (text dominates the visual — listicle style, bold text)
      - product_demo (product shown in use, unboxing, ingredient showcase)
      - lifestyle (aspirational settings, people in context, emotional atmosphere)
      - before_after (split comparison, transformation visual)
      - split_screen (two simultaneous views, comparison format)
      - green_screen (person over background footage/images)
```

---

## Process

### Step 1: Visual Style Classification

```
FOR EACH scraped ad:

  1. ASSESS visual style based on available data:
     - Image URL description / alt text
     - Video format indicators
     - Ad copy style (UGC-style copy often pairs with UGC visuals)
     - Platform norms (TikTok skews UGC, Meta is mixed)
     - Brand production patterns

  2. ASSIGN primary visual style from the 9-type taxonomy:
     - ugc: Handheld camera, selfie-style, informal setting, personal testimonial feel
     - polished_professional: Studio lighting, brand colors, professional models/actors
     - motion_graphics: Animated text, chart/graph animations, kinetic typography
     - text_overlay_heavy: Bold text cards dominate, minimal live footage
     - product_demo: Product close-ups, ingredient lists, "what's inside" reveals
     - lifestyle: People in aspirational settings, emotional imagery
     - before_after: Transformation comparison, side-by-side results
     - split_screen: Dual-view format comparing scenarios or products
     - green_screen: Person filmed against replaced background

  3. NOTE: Some ads combine styles (e.g., UGC with text overlay).
     Assign the DOMINANT style. If truly equal, assign the more distinctive style
     and add a classification_note.

  4. RECORD:
     ad_visual_classification:
       ad_id: string
       brand_name: string
       platform: string
       visual_style: string  # primary style from taxonomy
       secondary_style: string  # if hybrid, null otherwise
       classification_note: string  # explanation if ambiguous
       visual_confidence: string  # "high" / "medium" / "low"
         # "low" if classification was inferred without seeing the actual creative
```

### Step 2: Platform Distribution

```
FOR EACH platform:
  CALCULATE visual style distribution:
    - Count and percentage per style
    - Dominant style (highest %)
    - Style diversity (how many styles represented)

  NOTE platform-specific patterns:
    e.g., "TikTok: 65% UGC, 20% text_overlay, 10% green_screen"
    e.g., "Meta: 35% polished, 25% UGC, 20% product_demo"
```

### Step 3: Brand Visual Profiles

```
FOR EACH competitor brand:
  CALCULATE:
    - Total ads for this brand
    - Visual style distribution
    - Dominant style (what defines their creative approach)
    - Style consistency score (1-5):
      1.0 = uses only 1 style (highly consistent/rigid)
      3.0 = uses 3-4 styles
      5.0 = uses all styles (diverse/experimental)

  CREATE brand visual profile:
    brand_name: string
    total_ads: integer
    dominant_style: string
    style_consistency: float
    style_breakdown:
      - style: string
        count: integer
        pct: float
    creative_strategy_note: string
      # e.g., "Brand X uses exclusively UGC — zero professional production"
```

### Step 4: Visual Style x Performance Cross-Reference

```
IF run duration data available from Layer 1:
  FOR EACH visual style:
    CALCULATE representation in TIER_WINNER ads (90+ day runners)
    vs. representation in all ads

    IDENTIFY styles that over-index in winners:
      e.g., "UGC is 30% of all ads but 55% of TIER_WINNER ads — outperforming"
      e.g., "Polished is 40% of all ads but only 15% of winners — underperforming"

  NOTE: This is correlation, not causation. Record as intelligence signal.
```

## Quality Checks

1. **100% classification coverage** -- every scraped ad has a visual style assignment.
2. **All 9 styles referenced** in distribution tables (even if count = 0 for some).
3. **Low confidence count tracked** -- ads where visual style was inferred without seeing the creative are flagged.
4. **Brand profiles are substantive** -- each brand has a creative strategy note, not just raw numbers.
5. **Performance correlation data present** if run duration data was available from Layer 1.
6. **Platform patterns are specific** -- "Meta is mixed" is insufficient; specific percentages required.
7. **No "unknown" overuse** -- if more than 30% are low-confidence, note the limitation.
