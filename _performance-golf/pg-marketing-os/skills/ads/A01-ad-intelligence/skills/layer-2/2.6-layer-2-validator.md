# 2.6: Layer 2 Validator

**Version:** 1.0
**Layer:** 2 (Classification)
**Type:** Validation Gate
**Dependencies:** 2.1, 2.2, 2.3, 2.4, 2.5 all complete
**Model:** opus

---

## Purpose

Validate that ALL classification dimensions are 100% complete across ALL scraped ads before allowing progression to Layer 3 (Intelligence Synthesis). This is the Gate 2 enforcement mechanism. It produces the CLASSIFICATION COMPLETENESS CHECK table and either creates LAYER_2_COMPLETE.yaml (PASS) or halts execution (FAIL) with specific remediation instructions.

---

## Inputs

```yaml
input:
  from_layer_1:
    total_ad_count: integer  # from 1.5 validation — the ground truth count
  from_layer_2:
    hook_classification: "layer-2-outputs/2.1-hook-type-classifier.md"
    format_classification: "layer-2-outputs/2.2-format-analyzer.md"
    visual_classification: "layer-2-outputs/2.3-visual-style-analyzer.md"
    copy_patterns: "layer-2-outputs/2.4-copy-pattern-extractor.md"
    run_duration: "layer-2-outputs/2.5-run-duration-estimator.md"
```

---

## Process

### Step 1: Read All Layer 2 Outputs

```
READ each Layer 2 output file.
EXTRACT the classification count from each:
  - 2.1: total_ads_classified (hook type)
  - 2.2: total_ads_classified (format)
  - 2.3: total_ads_classified (visual style)
  - 2.4: total_ads_analyzed (copy patterns) + verbatim_copies_extracted
  - 2.5: total_ads_estimated (run duration)

GET ground truth total from Layer 1: total_ad_count
```

### Step 2: Calculate Completeness

```
FOR EACH dimension:
  completeness_pct = (classified_count / total_ad_count) * 100

  dimension_check:
    dimension: string
    total_ads: integer  # from Layer 1
    classified: integer  # from Layer 2 output
    pct_complete: float
    status: string  # PASS if 100%, FAIL otherwise
    deficit: integer  # total_ads - classified (0 if PASS)
```

### Step 3: Produce CLASSIFICATION COMPLETENESS CHECK Table

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  CLASSIFICATION COMPLETENESS CHECK - [timestamp]                            │
│                                                                              │
│  ┌────────────────────────────────────────────────────────────────────────┐ │
│  │ DIMENSION         │ TOTAL ADS │ CLASSIFIED │ % COMPLETE │ STATUS      │ │
│  ├────────────────────────────────────────────────────────────────────────┤ │
│  │ Hook Type         │ [X]       │ [X]        │ [X]%       │ PASS/FAIL   │ │
│  │ Format            │ [X]       │ [X]        │ [X]%       │ PASS/FAIL   │ │
│  │ Visual Style      │ [X]       │ [X]        │ [X]%       │ PASS/FAIL   │ │
│  │ Copy Patterns     │ [X]       │ [X]        │ [X]%       │ PASS/FAIL   │ │
│  │ Run Duration      │ [X]       │ [X]        │ [X]%       │ PASS/FAIL   │ │
│  └────────────────────────────────────────────────────────────────────────┘ │
│                                                                              │
│  REQUIRED: 100% on ALL dimensions. Any row < 100% = FAIL.                  │
│  OVERALL: [PASS - proceed to Layer 3] or [FAIL - classify remaining]       │
└─────────────────────────────────────────────────────────────────────────────┘
```

### Step 4: Additional Quality Checks

```
BEYOND completeness, validate quality:

  1. HOOK TYPE CHECK:
     - All 32 types referenced in distribution (even if 0% for some)
     - No "unclassified" bucket exists
     - Ambiguous classifications have notes

  2. FORMAT CHECK:
     - Platform distribution tables complete for all platforms
     - Video length sub-classification present for video ads

  3. VISUAL STYLE CHECK:
     - All 9 styles referenced in distribution
     - Low-confidence percentage noted

  4. COPY PATTERN CHECK:
     - Top 50 verbatim copies extracted
     - Headline patterns analyzed
     - CTA patterns analyzed

  5. RUN DURATION CHECK:
     - TIER_WINNER count documented
     - Winner candidate list produced
     - Estimation quality assessment present

  6. CROSS-DIMENSIONAL CONSISTENCY:
     - Same ad_id appears in all 5 classification outputs
     - No orphaned ads (in one output but missing from another)
```

### Step 5: Gate Decision

```
IF ALL dimensions = 100% AND all quality checks pass:
  GATE_2 = PASS
  CREATE LAYER_2_COMPLETE.yaml
  PROCEED to Layer 3

IF ANY dimension < 100%:
  GATE_2 = FAIL
  DOCUMENT which dimensions failed and deficit counts
  RETURN to the failing microskill(s) to classify remaining ads
  DO NOT proceed to Layer 3

IF quality checks fail (even if completeness = 100%):
  GATE_2 = FAIL
  DOCUMENT which quality checks failed
  RETURN to the failing microskill(s) to address quality gaps
  DO NOT proceed to Layer 3
```

## Quality Checks

1. **Ground truth count used** -- the total from Layer 1 validation (1.5) is the baseline, not a recount.
2. **All 5 dimensions checked** -- hook type, format, visual style, copy patterns, run duration.
3. **Quality checks go beyond counts** -- distribution coverage, verbatim extraction, cross-dimensional consistency all validated.
4. **CLASSIFICATION COMPLETENESS CHECK table produced** -- the mandatory visual table from the AGENT.md.
5. **Gate decision is BINARY** -- PASS or FAIL. No "conditional pass" or "close enough."
6. **If FAIL: specific remediation instructions** -- which microskill to re-run, how many ads remain, what to fix.
