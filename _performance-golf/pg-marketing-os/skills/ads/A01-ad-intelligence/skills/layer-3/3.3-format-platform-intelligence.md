# 3.3: Format-Platform Intelligence

**Version:** 1.0
**Layer:** 3 (Intelligence Synthesis)
**Type:** Cross-Dimensional Analysis
**Dependencies:** Layer 2 complete (LAYER_2_COMPLETE.yaml exists)
**Model:** opus

---

## Purpose

Produce platform-specific intelligence by cross-tabulating format, hook type, and visual style data. Identify which combinations (format x hook type x visual style) appear in TIER_WINNER ads on each platform. This reveals the winning creative patterns per platform -- not just what is being used, but what is WORKING.

This data feeds opportunity gap analysis (3.5), platform-specific recommendations (Section 9 of the handoff), and downstream creative decisions in A02-A05.

## Process

### Step 1: Platform-Specific Format Distribution

```
FOR EACH platform (meta, tiktok, google, other):

  PRODUCE format distribution:
    - Format type breakdown (video / static / carousel / stories / search / display)
    - For video: length distribution (6s / 15s / 30s / 60s / 2-3min / 3min+)
    - Format x TIER comparison:
      Which formats have the highest TIER_WINNER representation on this platform?

  platform_format_intelligence:
    platform: string
    total_ads: integer
    format_distribution:
      - format: string
        count: integer
        pct: float
        tier_winner_count: integer
        tier_winner_pct: float
    video_length_distribution:
      - bucket: string
        count: integer
        pct_of_video: float
        tier_winner_count: integer
    winning_format_signal: string
      # e.g., "30s video dominates TIER_WINNER on TikTok (62% of winners)"
```

### Step 2: Platform-Specific Hook Type Distribution

```
FOR EACH platform:

  CALCULATE which hook types are used on this platform:
    - Hook type distribution (all 32 types, sorted by frequency)
    - Hook types unique to this platform (used here but not on other platforms)
    - Hook types with high TIER_WINNER representation on this platform

  platform_hook_intelligence:
    platform: string
    hook_distribution:
      - type_id: string
        type_name: string
        count: integer
        pct: float
        tier_winner_pct: float
    platform_unique_types: [string]  # types used ONLY on this platform
    top_performing_types: [string]  # highest winner ratio on this platform
```

### Step 3: Platform-Specific Visual Style Distribution

```
FOR EACH platform:

  CALCULATE visual style patterns:
    - Style distribution (all 9 styles, sorted by frequency)
    - Dominant production quality (UGC-heavy vs. polished-heavy)
    - Sound-on vs. sound-off patterns (Meta feed = often sound-off, TikTok = sound-on)
    - Vertical vs. horizontal format distribution

  platform_visual_intelligence:
    platform: string
    style_distribution:
      - style: string
        count: integer
        pct: float
        tier_winner_pct: float
    production_quality: string  # "ugc_dominant" / "polished_dominant" / "mixed"
    sound_pattern: string  # "primarily_sound_on" / "primarily_sound_off" / "mixed"
    orientation: string  # "vertical_dominant" / "horizontal_dominant" / "mixed"
```

### Step 4: Winning Combination Analysis

```
CROSS-TABULATE: format x hook_type x visual_style for TIER_WINNER ads only.

FOR EACH platform:
  IDENTIFY the top 5 winning combinations:

  winning_combination:
    rank: integer
    platform: string
    format: string
    hook_type: string
    visual_style: string
    count_in_winners: integer
    pct_of_platform_winners: float
    example_ad_ids: [string]  # specific ads that match this combination
    pattern_note: string
      # e.g., "15s UGC video + Testimonial hook + Before/After visuals
      #        = the dominant winning pattern on Meta (28% of winners)"

  NOTE: Some combinations will have small sample sizes.
  Flag combinations based on < 3 winners as "suggestive, not conclusive."
```

### Step 5: Platform Compliance Constraints

```
FOR EACH platform, document compliance constraints:

  platform_compliance:
    platform: string
    character_limits:
      headline: integer
      body_copy: integer
      cta: string  # available CTAs
    format_restrictions: [string]
      # e.g., "TikTok: video only, max 3 minutes, vertical required"
    content_restrictions: [string]
      # e.g., "Meta: no before/after imagery in health vertical"
      # e.g., "Google: character limits on search text ads"
    vertical_specific_restrictions: [string]
      # From vertical profile: compliance rules for this niche
```

### Step 6: Platform Creative Recommendations

```
FOR EACH platform, produce actionable recommendations:

  platform_recommendations:
    platform: string
    recommended_formats:
      - format: string
        rationale: string  # why this format on this platform
    recommended_hook_types:
      - type_id: string
        type_name: string
        rationale: string  # why this hook type on this platform
    recommended_visual_style: string
    recommended_video_length: string  # if video
    creative_direction_note: string
      # 2-3 sentence creative direction for this platform
    watch_out: string
      # What to avoid on this platform based on data
```

## Quality Checks

1. **All platforms analyzed** -- every platform with scraped data has a complete intelligence profile.
2. **Winning combinations identified** -- at least top 5 format x hook x visual combos for each platform.
3. **Cross-tabulation uses TIER_WINNER data** -- not just overall frequency but performance-weighted.
4. **Compliance constraints documented** -- platform-specific and vertical-specific restrictions noted.
5. **Recommendations are platform-specific** -- different recommendations per platform, not generic.
6. **Small sample sizes flagged** -- combinations based on < 3 winners are explicitly noted as suggestive.
7. **Sound/orientation patterns documented** for video-heavy platforms (TikTok, Meta).
