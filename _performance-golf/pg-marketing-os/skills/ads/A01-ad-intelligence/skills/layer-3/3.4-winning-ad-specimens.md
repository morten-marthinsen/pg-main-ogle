# 3.4: Winning Ad Specimens

**Version:** 1.1
**Layer:** 3 (Intelligence Synthesis)
**Type:** Specimen Extraction
**Dependencies:** Layer 2 complete (LAYER_2_COMPLETE.yaml exists), run duration tiers from 2.5, dual-signal tiers from 2.7 (when available)
**Model:** opus
**Updated:** 2026-02-27 (v1.1 — impression-validated specimen support)

---

## Purpose

Extract the top 20 winning ads (longest-running / highest-performing) as full specimens with VERBATIM copy transcription. These specimens become the primary reference material for downstream ad generation (A02-A05). Each specimen includes the exact headline, full body copy, CTA, hook type classification, format notes, and analysis of what makes it a winner.

**v1.1 Note:** When dual-signal performance scores are available (Tool-Assisted Scan mode), specimen selection uses `performance_score` from 2.7 as the PRIMARY ranking signal instead of run duration alone. Specimens from impression-validated ads are flagged as `impression_validated: true`, giving downstream skills a confidence indicator. When dual-signal scores are NOT available, selection falls back to run-duration ranking (v1.0 behavior).

This is the LARGEST and most critical output in Layer 3. 20 full verbatim transcriptions with analysis should produce 30-50KB of content. Summaries are NOT specimens. Paraphrases are NOT specimens. The exact words, character by character, are required.

## Process

### Step 1: Select Top 20 Specimens

```
DETERMINE ranking signal:

IF impression_scores available (2.7 output exists):
  → USE performance_score as PRIMARY ranking signal
  → Primary candidates: TIER_WINNER_DUAL ads (top 20% by performance_score)
  → Secondary candidates: TIER_PERFORMER_DUAL ads (21st-50th percentile)
  → FLAG all impression-validated ads with impression_validated = true
  → LOG: "Using dual-signal ranking (40% duration + 60% impressions)"

ELSE (no impression data):
  → USE estimated_run_days as PRIMARY ranking signal (v1.0 behavior)
  → Primary candidates: TIER_WINNER ads (90+ days)
  → Secondary candidates: TIER_PERFORMER ads (30-89 days)
  → All ads get impression_validated = false
  → LOG: "Using run-duration-only ranking (no impression data available)"

SELECTION CRITERIA (in priority order):
  1. Performance score (dual-signal) OR run duration (fallback) — highest = strongest winner signal
  2. Platform diversity (aim for specimens from multiple platforms)
  3. Brand diversity (avoid 15 specimens from the same brand)
  4. Hook type diversity (avoid 15 specimens of the same hook type)
  5. Format diversity (include both video and static if available)

SELECT 20 specimens ensuring:
  - At least 2 platforms represented
  - At least 5 different brands represented
  - At least 5 different hook types represented
  - Prioritize TIER_WINNER (or TIER_WINNER_DUAL), then TIER_PERFORMER if < 20

IF dual-signal available:
  PREFER impression-validated specimens over non-validated ones at equal rank
  LOG: "Impression-validated specimens: [count] / 20"

IF TIER_WINNER + TIER_PERFORMER < 20:
  Include top TIER_TESTING ads with documented justification
  NOTE: "Only [X] TIER_WINNER and [Y] TIER_PERFORMER ads available.
         Included [Z] TIER_TESTING specimens to reach 20 minimum."
```

### Step 2: Per-Specimen Extraction

```
FOR EACH of the 20 selected specimens:

  ASSIGN specimen_id: WS-001 through WS-020

  EXTRACT from scraped data (Layer 1 outputs):

  specimen:
    specimen_id: string  # WS-001 through WS-020
    competitor_brand: string
    platform: string
    format: string  # video / static / carousel / etc.
    video_length: string  # if video, duration
    estimated_run_days: integer
    tier: string  # TIER_WINNER / TIER_PERFORMER / TIER_TESTING (or _DUAL variants)
    performance_score: float  # dual-signal score (0.0-1.0) — null if no impression data
    impressions: integer  # from Meta Ad Spy — null if no impression data
    impression_validated: boolean  # true if this specimen has impression data backing
    start_date: string  # if available
    last_seen_date: string  # if available

    verbatim_copy:
      headline: string
        # EXACT headline text. Character for character.
        # NOT "a headline about gut health"
        # YES "Board-Certified Cardiologist Reveals the #1 Food Destroying Your Gut"
      hook_text: string
        # First 3 seconds of video script OR first line of static
        # VERBATIM — exact words
      full_body_copy: string
        # COMPLETE body text / video script. Every word.
        # DO NOT truncate. DO NOT summarize. Full text.
      cta_text: string
        # EXACT CTA. "Learn More" or "Watch the Free Presentation" etc.

    classifications:
      hook_type: string  # from 2.1 classification
      hook_type_name: string
      hook_category: string  # A-J
      visual_style: string  # from 2.3
      format_type: string  # from 2.2

    analysis:
      why_winner: string
        # Specific analysis of what makes this a winning ad.
        # NOT "it's a good ad" — specific structural, copy, and creative observations.
        # e.g., "Uses authority hook (board-certified) + specific body part (gut) +
        #        numbered claim (#1 food). 15s UGC format matches TikTok norms.
        #        Run duration of 142 days suggests strong performance."
      key_patterns: [string]
        # 2-3 specific patterns other ads can learn from this specimen
        # e.g., "Authority credential in first 3 words",
        #        "Numbered claim creates specificity",
        #        "Body copy uses question-answer rhythm"
      copy_techniques: [string]
        # Specific copywriting techniques identified
        # e.g., "Pattern interrupt in headline",
        #        "Social proof stacking in body",
        #        "Urgency without scarcity in CTA"
```

### Step 3: Verbatim Quality Verification

```
FOR EACH specimen:

  VERIFY verbatim quality:
    headline_is_exact: boolean
      # Did we transcribe the EXACT headline, or is it a paraphrase?
    body_copy_is_complete: boolean
      # Is the FULL body copy / script included, or is it truncated?
    cta_is_exact: boolean
      # Is the CTA the exact text from the ad?

  IF any verification = false:
    GO BACK to Layer 1 raw data and re-extract
    DO NOT proceed with paraphrased specimens

  CALCULATE word_count for each specimen:
    headline_word_count: integer
    body_copy_word_count: integer
    total_word_count: integer

  EXPECTED: Average specimen should be 100-500 words of copy.
  If average < 50 words, the specimens may be under-extracted.
```

### Step 4: Specimen Diversity Check

```
VERIFY diversity targets:

  platform_distribution:
    - platform: string
      count: integer
  → At least 2 platforms represented? [Y/N]

  brand_distribution:
    - brand: string
      count: integer
  → At least 5 brands represented? [Y/N]

  hook_type_distribution:
    - type: string
      count: integer
  → At least 5 hook types represented? [Y/N]

  format_distribution:
    - format: string
      count: integer
  → Both video and non-video represented? [Y/N]

IF any diversity target not met:
  SWAP lowest-priority specimens for ones that improve diversity
  DOCUMENT swaps and rationale
```

## Quality Checks

1. **Minimum 20 specimens** -- fewer than 20 is a FAIL. Must expand or include TIER_PERFORMER/TIER_TESTING.
2. **All specimens have VERBATIM copy** -- headline, body, and CTA are exact text, not paraphrases.
3. **Body copy is COMPLETE** -- not truncated, not "continues..." -- the FULL text.
4. **Run duration ranking** -- specimens selected primarily by longest run duration, not subjective picks.
5. **Diversity targets met** -- at least 2 platforms, 5 brands, 5 hook types represented.
6. **Analysis is specific** -- "why_winner" cites specific structural and copy observations, not "it's a good ad."
7. **File size >= 30KB** -- 20 verbatim specimens with analysis requires substantial content. Under 30KB means specimens are truncated or missing body copy.
8. **Specimen IDs consistent** -- WS-001 through WS-020, referenced consistently in downstream outputs.
