# 1.5: Raw Data Validator

**Version:** 1.0
**Layer:** 1 (Scraping)
**Type:** Validation & Volume Check
**Dependencies:** 1.1, 1.2, 1.3, 1.4 complete
**Output:** scraping_validation_report.yaml

---

## Purpose

Validate that total scraped ads >= 500, competitor coverage >= 10 brands with ads found, and platform coverage >= 2 platforms. Produce the mandatory AD VOLUME CHECK table. If requirements not met, trigger expansion protocol (up to 3 rounds) before escalating to human.

---

## Inputs

```yaml
input:
  from_1.1: meta_ad_count: integer
  from_1.2: tiktok_ad_count: integer
  from_1.3: google_ad_count: integer
  from_1.4: additional_platform_ad_count: integer
  from_layer_0: competitor_list_count: integer
```

---

## Process

### Step 1: Aggregate Counts

```
CALCULATE totals:
  total_ads = meta + tiktok + google + additional
  platforms_with_ads = count(platforms where ads > 0)
  competitors_with_ads = count(unique brand_names across all platforms)
```

### Step 2: Generate AD VOLUME CHECK Table

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  AD VOLUME CHECK - [timestamp]                                              │
│                                                                              │
│  ┌────────────────────────────────────────────────────────────────────────┐ │
│  │ METRIC              │ REQUIRED │ ACTUAL │ STATUS     │ DEFICIT          │ │
│  ├────────────────────────────────────────────────────────────────────────┤ │
│  │ TOTAL ADS           │ 500      │ [X]    │ PASS/FAIL  │ [500-X if fail]  │ │
│  │ Meta Ads            │ 200      │ [X]    │ PASS/FAIL  │ [deficit]        │ │
│  │ TikTok Ads          │ 150      │ [X]    │ PASS/FAIL  │ [deficit]        │ │
│  │ Google Ads          │ 100      │ [X]    │ PASS/FAIL  │ [deficit]        │ │
│  │ Other Platform Ads  │ 50*      │ [X]    │ PASS/FAIL  │ [deficit]        │ │
│  │ Competitor Brands   │ 10       │ [X]    │ PASS/FAIL  │ [deficit]        │ │
│  │ Platforms Covered   │ 2        │ [X]    │ PASS/FAIL  │ [deficit]        │ │
│  └────────────────────────────────────────────────────────────────────────┘ │
│                                                                              │
│  * Other Platform Ads required ONLY if vertical profile includes them.      │
│  OVERALL: [PASS - proceed] or [FAIL - expand before proceeding]            │
└─────────────────────────────────────────────────────────────────────────────┘
```

### Step 3: Gate Decision

```
IF total_ads >= 500 AND competitors_with_ads >= 10 AND platforms_covered >= 2:
  → OVERALL: PASS
  → Can proceed to Layer 2
  → Create LAYER_1_COMPLETE.yaml

ELSE:
  → OVERALL: FAIL
  → BEGIN expansion protocol
```

### Step 4: Expansion Protocol (if FAIL)

```
EXPANSION ROUND 1:
  FOR EACH platform with deficit:
    - Add 5+ new competitor search queries
    - Retry scraping with broader keywords
    - Re-aggregate counts
  
  IF now PASS: proceed
  ELSE: ROUND 2

EXPANSION ROUND 2:
  - Try different Apify actors (search for alternatives)
  - Adjust search parameters (broader date ranges, different regions)
  - Re-aggregate counts

  IF now PASS: proceed
  ELSE: ROUND 3

EXPANSION ROUND 3:
  - Use ALL available fallback tools
  - Manual discovery via web search if needed
  - Re-aggregate counts

  IF now PASS: proceed
  ELSE: HUMAN ESCALATION

HUMAN ESCALATION:
  Present:
    - Exact metrics vs targets
    - What was tried (3 expansion rounds)
    - Why platform/market may lack data
  
  Options:
    (a) Approve reduced threshold
    (b) Provide additional competitor names
    (c) Suggest alternative sources
    (d) Adjust vertical scope
```
