# 0.1: Campaign Brief Loader

**Version:** 1.0
**Layer:** 0 (Foundation)
**Type:** Data Loading
**Dependencies:** Skill 09 (Campaign Brief) output
**Output:** campaign_brief_extract.yaml

---

## Purpose

Load Skill 09 Campaign Brief output and extract ad-specific intelligence requirements: product category, target audience, competitor names, market positioning, and awareness level. This provides the strategic foundation for all ad intelligence gathering.

---

## Inputs

```yaml
input:
  from_skill_09:
    campaign_brief_path: "[project]/09-campaign-brief/campaign-brief.json"
```

---

## Process

### Step 1: Locate Campaign Brief

```
VERIFY: Campaign Brief output exists at specified path
IF NOT EXISTS:
  → HALT: "Campaign Brief not found. Execute Skill 09 first."
  → Cannot proceed without strategic foundation
```

### Step 2: Extract Required Fields

```
READ campaign-brief.json
EXTRACT:
  - product_category (supplement, info product, software, service, etc.)
  - target_audience (demographic + psychographic profile)
  - competitor_names (explicitly named competitors)
  - market_positioning (how product is positioned vs competitors)
  - awareness_level (unaware, problem-aware, solution-aware, product-aware, most-aware)
  - vertical (if specified — feeds to 0.0.1)
  - unique_selling_proposition
  - primary_benefit_claims
```

### Step 3: Competitor Name Extraction

```
COMPILE competitor list from Campaign Brief:
  - Directly named competitors in brief
  - Competitors mentioned in positioning section
  - Competitors referenced in "vs" comparisons
  - Alternative solutions mentioned

MINIMUM: 3 competitor names extracted
IF < 3: Flag for expansion in Layer 0.3 (Competitor List Builder)
```

### Step 4: Awareness Level Validation

```
VERIFY awareness_level is one of:
  - unaware (don't know problem exists)
  - problem_aware (know problem, don't know solution category)
  - solution_aware (know solution category, evaluating options)
  - product_aware (know our product, considering purchase)
  - most_aware (ready to buy, need final nudge)

IF missing or unclear:
  → DEFAULT: "solution_aware"
  → LOG: "Awareness level not specified — defaulting to solution-aware"
```

## Output Schema

```yaml
output:
  campaign_brief_extract:
    product_category: string
    target_audience:
      demographic: string
      psychographic: string
      pain_points: [string]
      desires: [string]
    competitor_names: [string]  # minimum 3
    market_positioning: string
    awareness_level: enum[unaware, problem_aware, solution_aware, product_aware, most_aware]
    vertical: string
    unique_selling_proposition: string
    primary_benefit_claims: [string]

    extraction_status:
      all_required_fields_present: boolean
      missing_fields: [string]
      competitor_count: integer
```

## Quality Checks

1. Campaign Brief file exists and was read successfully
2. All required fields extracted (no missing critical fields)
3. Competitor count >= 3 (or flagged for expansion)
4. Awareness level is valid enum value
5. Target audience includes pain points AND desires
6. Product category is specific (not "various" or "multiple")
