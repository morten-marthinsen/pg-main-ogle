# 2.5.1: Word Count Verification

## Purpose
Count words in the AUDIO column of every single variant (base copies, hook swaps, CTA variants, platform adaptations). Produce a WORD COUNT COMPLIANCE TABLE. Any variant exceeding the hard limit for its target length is flagged FAIL and returned to Layer 2 for compression. No "close enough." Word count limits are physics -- they represent the maximum words that can be spoken in the target ad duration.

## Inputs
- Complete variant inventory (from 2.4)
- All variant scripts (from Layer 1 and Layer 2 outputs)
- Word count limits table:
  - 6s = 15 words
  - 15s = 40 words
  - 30s = 75 words
  - 60s = 160 words
  - 2-3min = 450 words

## Process
```
STEP 1: READ the variant inventory (2.4) to get the complete list of variants.

STEP 2: For EACH variant in the inventory:
  a. LOCATE the variant's script (in Layer 1 or Layer 2 output files)
  b. EXTRACT the AUDIO column text only (visual column doesn't count)
  c. COUNT words precisely
     - Hyphenated words count as one word
     - Numbers count as one word each
     - Contractions count as one word
  d. LOOK UP the word count limit for this variant's target length
  e. COMPARE actual word count vs limit
  f. ASSIGN status: PASS (within limit) or FAIL (over limit)
     - 161 words for a 160-word limit = FAIL. No exceptions.
     - There is no "close enough" -- the limit IS the limit.

STEP 3: PRODUCE the Word Count Compliance Table.

STEP 4: CALCULATE summary statistics:
  - Total variants checked
  - Total PASS
  - Total FAIL
  - Compliance rate (must be 100% for GATE_2.5)

STEP 5: IF any FAIL:
  - List each failing variant with: ID, actual count, limit, overage
  - Specify required action: return to Layer 2 for compression
  - GATE_2.5 CANNOT pass until all variants are within limits
```
