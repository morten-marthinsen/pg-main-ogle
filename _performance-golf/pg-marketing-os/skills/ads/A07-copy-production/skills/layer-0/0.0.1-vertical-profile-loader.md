# 0.0.1: Vertical Profile Loader

## Purpose
Load the ad-specific vertical configuration from `ad-verticals/`. Extract platform priorities, hook patterns, compliance constraints, anti-slop rules, and vertical-specific banned patterns. This runs FIRST in Layer 0 because everything downstream depends on vertical context.

## Inputs
- Vertical profile file from `ad-verticals/[vertical].md`
- Project brief (to identify which vertical applies)

## Process
```
STEP 1: IDENTIFY the vertical from the project brief or campaign brief.
  - Match to one of: golf, health, finance, personal-dev, technology
  - If no vertical specified, log WARNING and use universal defaults

STEP 2: READ the vertical profile file.
  - Path: ./skills/ads/ad-verticals/[vertical].md
  - IF file does not exist: HALT -- vertical profile required

STEP 3: EXTRACT the following sections:
  a. Platform priorities (which platforms matter most for this vertical)
  b. Hook patterns (which hook types perform best in this vertical)
  c. Compliance constraints (vertical-specific claim restrictions)
  d. Anti-slop rules (vertical-specific banned phrases and patterns)
  e. Tone register defaults (how this vertical "sounds")

STEP 4: WRITE extracted data to output file.
```

## Output File
- **Path:** `layer-0-outputs/0.0.1-vertical-profile-loader.md`
- **Minimum size:** 1KB

## Quality Checks
- Vertical ID matches a valid vertical
- All 5 extraction sections populated (platform priorities, hook patterns, compliance, anti-slop, tone)
- Compliance constraints are specific (not generic "follow the rules")
- Anti-slop rules contain actual banned phrases (not vague guidance)

## Dependencies
- Project brief or campaign brief (to identify vertical)
- Vertical profile file must exist in `ad-verticals/`
