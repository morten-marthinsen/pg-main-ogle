# 1.1: Concept Brief Extraction

## Purpose
For each Arena-approved concept, extract and compile a per-concept mini-brief that the copy generation subagent receives as context. This distills all upstream data into a focused, concept-specific package that enables efficient base copy generation.

## Inputs
- Layer 0 outputs (all 8 loader files)
- A06 Arena-approved concepts (from 0.2)
- A04 script architectures (from 0.3)
- A05 visual directions (from 0.4)
- A03 platform assignments (from 0.5)

## Process
```
STEP 1: For EACH Arena-approved concept from 0.2-arena-results-loader.md:

STEP 2: COMPILE the concept mini-brief by cross-referencing:
  a. From 0.2 (Arena): concept ID, name, strategic angle, Arena scores,
     human selection notes, human edits, strengths, watch areas
  b. From 0.3 (Script Architecture): assigned framework, module definitions,
     swap points, word count budgets per target length
  c. From 0.4 (Visual Direction): visual treatment type, shot list summary,
     text overlay requirements, copy-visual complementarity notes
  d. From 0.5 (Format Strategy): platform assignments, target ad lengths,
     platform-specific constraints
  e. From 0.1 (Campaign Brief): product name, audience, awareness level,
     price point, emotional drivers
  f. From 0.7 (Persona Voice): applicable voice specimens, Soul.md constraints

STEP 3: VERIFY each mini-brief is self-contained.
  - A copy generation subagent receiving ONLY this mini-brief
    should have everything needed to write the base copy
  - No missing fields that would require going back to upstream files

STEP 4: CREATE the per-concept briefs as a single organized output file.
```
