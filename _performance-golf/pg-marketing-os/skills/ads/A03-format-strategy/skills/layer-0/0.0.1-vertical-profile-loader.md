# 0.0.1: Vertical Profile Loader

## Purpose

Load the vertical-specific configuration that determines format patterns, compliance constraints, and anti-slop rules for this campaign. Every vertical (golf, health, finance, personal-dev, technology) has different optimal formats, platform performance patterns, and compliance requirements. This microskill loads that configuration first before any other Layer 0 work begins.

## Inputs

- Vertical identifier from Campaign Brief (golf / health / finance / personal-dev / technology)
- `skills/ads/ad-verticals/[vertical].md` — vertical profile file

## Process

1. **IDENTIFY vertical** from Campaign Brief (Skill 09 output)
   - Extract vertical designation from brief metadata
   - If not specified: DEFAULT to "health" with WARNING logged

2. **READ vertical profile file** at `skills/ads/ad-verticals/[vertical].md`
   - Full read required — do NOT summarize

3. **EXTRACT vertical-specific format patterns:**
   - Dominant hook types for this vertical (from A01 analysis)
   - Primary format distribution (video %, static %, carousel %)
   - Platform preference order (which platforms perform best in this vertical)
   - Length patterns (does this vertical favor short-form or long-form?)
   - Creative treatment patterns (UGC vs polished performance in this vertical)
   - Key format differences vs other verticals

4. **EXTRACT compliance constraints:**
   - Hard constraints for this vertical (platform policies, claim restrictions)
   - Required disclaimers
   - Forbidden claim types
   - Platform-specific restrictions for this vertical (e.g., Meta health claims, Google financial disclaimers)

5. **EXTRACT anti-slop rules:**
   - Banned patterns specific to this vertical
   - Language to avoid (generic supplement language in health, finance guru speak in finance, etc.)
   - Cross-vertical contamination prevention

6. **HOLD in context** for Layer 1-3 consumption
   - All downstream microskills in this skill will reference vertical config
   - Compliance pre-check (1.4) relies heavily on vertical compliance constraints
   - Format assignment (2.1) uses vertical format patterns as baseline

- [ ] Vertical profile file successfully read
- [ ] All three sections extracted (format patterns, compliance, anti-slop)
- [ ] Format patterns include quantified data (percentages, rankings)
- [ ] Compliance constraints are specific (not generic "follow platform rules")
- [ ] Anti-slop rules are actionable (specific patterns to avoid)
