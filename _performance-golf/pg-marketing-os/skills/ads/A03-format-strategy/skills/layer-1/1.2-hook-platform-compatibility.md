# 1.2: Hook-Platform Compatibility Matrix

## Purpose

Assess every selected hook against every target platform to determine compatibility levels (OPTIMAL, VIABLE, POOR, BLOCKED). This matrix drives format assignment decisions in Layer 2 by identifying which hook-platform combinations should be prioritized, which need adaptation, and which should be avoided.

## Inputs

- `layer-0-outputs/0.3-hook-angle-matrix.md` — 8-10 selected hooks with types
- `layer-0-outputs/0.1-campaign-brief.md` — target platforms
- `layer-0-outputs/0.2-ad-intelligence.md` — platform-specific format performance
- `layer-0-outputs/0.4-reference-loader.md` — platform-specific hook considerations from AD-HOOK-TAXONOMY.md
- Platform specification reference (from AGENT.md) — aspect ratios, length constraints, sound behavior

## Process

1. **BUILD compatibility matrix structure:**
   - Rows: 8-10 selected hooks
   - Columns: All target platforms (and sub-placements if relevant: Meta Feed, Meta Reels, etc.)
   - Cells: Compatibility assessment + rationale

2. **FOR EACH hook-platform pair, ASSESS using four criteria:**

   **Criterion 1: Length Fit**
   - Does the hook's natural length match the platform's sweet spot?
   - Examples:
     - 6s hook + YouTube pre-roll = POOR (too short for pre-roll format)
     - 60s story hook + TikTok = POOR (TikTok sweet spot is 15-30s)
     - 15s UGC hook + TikTok = OPTIMAL

   **Criterion 2: Sound Dependency**
   - Does the hook require audio on a sound-off platform?
   - Cross-reference hook type with platform sound behavior matrix
   - Examples:
     - Audio-dependent hook + Meta feed = POOR (85% sound-off)
     - Visual-first hook + Meta feed = OPTIMAL
     - Audio hook + TikTok = OPTIMAL (80%+ sound-on)

   **Criterion 3: Format Nativeness**
   - Does the hook feel native to the platform's aesthetic?
   - Examples:
     - UGC-style hook + TikTok = OPTIMAL (platform-native aesthetic)
     - Polished production hook + TikTok = POOR (penalized by algorithm)
     - Authority/expert hook + YouTube = OPTIMAL (educational content thrives)

   **Criterion 4: A01 Data Validation**
   - Does A01 intelligence support this combination?
   - If A01 shows 0 winning ads of this hook type on this platform = FLAG
   - If A01 shows this hook type dominates winners on this platform = BOOST

3. **ASSIGN compatibility level per pair:**
   ```
   COMPATIBILITY LEVELS:
     OPTIMAL   — Hook type and format align perfectly with platform strengths
     VIABLE    — Hook can work on this platform with adaptation
     POOR      — Hook type conflicts with platform characteristics
     BLOCKED   — Compliance or technical constraint prevents this hook on this platform
   ```

4. **DOCUMENT rationale for EVERY cell:**
   - "OPTIMAL" without explanation is not acceptable
   - Each cell must cite which criteria drove the assessment
   - Example: "OPTIMAL — 15s length fits TikTok sweet spot, UGC aesthetic is platform-native (F2 hook type), A01 shows 8/12 TikTok winners use F2 hooks"

5. **PRODUCE full compatibility matrix table**

6. **ANALYZE matrix patterns:**
   - Which hooks are platform-versatile (OPTIMAL across multiple platforms)?
   - Which hooks are platform-specific (OPTIMAL on one, POOR on others)?
   - Are any hooks POOR/BLOCKED across all platforms? (RED FLAG — escalate to A02)
   - What percentage of total hook-platform pairs are OPTIMAL vs VIABLE vs POOR?
