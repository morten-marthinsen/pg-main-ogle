# 1.4: Compliance Pre-Check

## Purpose

Flag compliance risks for every selected hook on every target platform BEFORE script generation begins. This pre-check identifies hooks that may trigger platform review, require disclaimer modifications, or are outright blocked. Early compliance flagging prevents wasted creative production on ads that can't run.

## Inputs

- `layer-0-outputs/0.3-hook-angle-matrix.md` — 8-10 selected hooks with full text
- `layer-0-outputs/0.0.1-vertical-profile.md` — vertical-specific compliance constraints
- `layer-0-outputs/0.1-campaign-brief.md` — target platforms
- Platform-specific compliance policies (knowledge base + vertical profile)

## Process

1. **LOAD vertical compliance constraints** from 0.0.1:
   - Hard constraints for this vertical (e.g., health: no disease claims, finance: no guaranteed returns)
   - Required disclaimers
   - Forbidden claim types

2. **FOR EACH hook, CHECK against platform-specific policies:**

   **Meta (Facebook + Instagram):**
   - Health claims restrictions: Before/after weight loss images restricted, performance claims need disclaimers
   - Financial disclaimers: Investment/income claims require risk disclosures
   - Personal attributes targeting: Cannot target based on sensitive categories
   - Specific text scan: Check for trigger words (cure, treat, FDA, guaranteed, etc.)

   **TikTok:**
   - Community guidelines: No misleading health claims, no get-rich-quick schemes
   - Branded content policies: Must disclose if creator partnership
   - Specific text scan: Check for sensational language that triggers review

   **YouTube:**
   - Content suitability guidelines: Advertiser-friendly content requirements
   - Ad policy: Misleading claims, unsubstantiated health claims, financial scams
   - Specific text scan: Check for policy trigger words

   **Google Display Network:**
   - Misleading content policies: Claims must be substantiated
   - Healthcare restrictions: Prescription drug claims, unproven medical device claims
   - Financial services policies: Predatory lending, guaranteed returns
   - Specific text scan: Check for restricted terms

3. **CHECK hook text against vertical compliance:**
   - **Health vertical:**
     - No disease claims ("cures diabetes", "treats cancer")
     - "Results may vary" required for transformation claims
     - No FDA-unapproved claims
     - Before/after images need disclaimers
   - **Finance vertical:**
     - Risk disclosures for investment claims
     - No "guaranteed returns" or "risk-free"
     - Income claims need "not typical" disclaimers
   - **Golf vertical:**
     - Distance claims should cite conditions
     - Performance claims should be qualified

4. **ASSIGN compliance flag level per hook-platform pair:**
   ```
   COMPLIANCE FLAGS:
     CLEAR     — No compliance issues identified on this platform
     CAUTION   — May trigger review; rephrase recommended before A04 scripting
     RESTRICTED — Blocked on this specific platform; cannot use without modification
     BLOCKED   — Cannot run this hook as-is on ANY platform (needs rework at A02 level)
   ```

5. **FOR CAUTION and RESTRICTED hooks, PROVIDE specific guidance for A04:**
   - What claim needs softening?
   - What disclaimer needs adding?
   - Which platform(s) are affected?
   - Suggested rephrase options

6. **CROSS-REFERENCE with A02 compliance scores:**
   - If A02 flagged compliance risk, does platform-specific check confirm?
   - If A02 scored high compliance safety, but platform check finds issues, escalate
