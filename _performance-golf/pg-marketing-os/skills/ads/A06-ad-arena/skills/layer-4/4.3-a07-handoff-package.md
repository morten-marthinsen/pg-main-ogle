# 4.3: A07 Handoff Package

## Purpose
Generate A07-HANDOFF.md containing: selected concepts with all elements (hook, script architecture, visual direction), improvement recommendations from Arena, human notes and instructions, word count limits, platform constraints, variant generation guidance. This is what A07 (Copy Production) consumes to produce final ad copy.

## Inputs
- 1.1-concept-bundler output (original concept bundles with all elements)
- 1.3-concept-summary-generator output (concept summaries)
- 2.11-round3-persona-evaluation output (winning evaluations for selected concepts)
- 2.14-round3-final-scoring output (FINAL scores)
- 2.5.2-hybrid-concept-generation output (hybrid details if hybrid selected)
- 3.2-human-selection-capture output (HUMAN_SELECTION.yaml)
- Upstream packages from Layer 0 (Campaign Brief, Research, Big Idea, Mechanism, etc.)

## Process
```
STEP 1: Load human selection and concept data
  - READ: checkpoints/HUMAN_SELECTION.yaml
  - READ: layer-1-outputs/1.1-concept-bundler.md
  - READ: layer-1-outputs/1.3-concept-summary-generator.md
  - READ: layer-2-outputs/2.11-round3-persona-evaluation.md (for selected concepts only)
  - READ: layer-2-outputs/2.14-round3-final-scoring.md
  - IF hybrid selected: READ: layer-2.5-outputs/2.5.2-hybrid-concept-generation.md

STEP 2: FOR EACH selected concept, compile complete handoff:

  A. CONCEPT IDENTITY
    - Concept ID
    - Hook text (verbatim)
    - Script architecture (from A04)
    - Visual direction (from A05)
    - Platform target
    - Ad length (seconds)
    - Selected version: [pure_round3 / hybrid_H1 / hybrid_H2 / modified]

  B. HOOK DETAILS
    - Verbatim hook text
    - Hook type classification
    - First 3 seconds description (what viewer sees/hears)
    - Scroll-stop mechanism (what makes them stop)
    - Text overlay copy (if applicable)
    - Audio hook (if applicable)

  C. SCRIPT ARCHITECTURE
    - Framework name
    - Beat sequence with timing
    - Key script moments (hook -> problem -> mechanism -> proof -> CTA)
    - Word count target per beat
    - Total word count target
    - Pacing notes

  D. VISUAL DIRECTION
    - Treatment type
    - Shot list / visual sequence
    - On-screen text/graphics plan
    - Color/mood direction
    - Talent/casting notes (if applicable)
    - B-roll requirements

  E. ARENA IMPROVEMENTS
    - Top improvements from Arena process (from winning evaluation)
    - Specific changes recommended by best persona
    - IF hybrid: composition details (which elements from which sources)
    - Weaknesses to avoid (from Critic analysis)

  F. HUMAN INSTRUCTIONS
    - All notes from HUMAN_SELECTION.yaml
    - Specific modifications requested
    - Preferences stated during selection

  G. PRODUCTION CONSTRAINTS
    - Platform specs (dimensions, format, length limits)
    - Word count limits
    - Compliance requirements (if any from vertical profile)
    - Anti-slop rules specific to this campaign

  H. VARIANT GENERATION GUIDANCE
    - Which elements can vary (hook text, visual treatment, CTA)
    - Which elements must stay fixed (mechanism, core message, brand)
    - Recommended variant axes (e.g., "test 3 hook variations")
    - A/B testing priorities

STEP 3: Compile campaign context for A07
  - Big Idea summary (from Skill 06)
  - Mechanism summary (from Skill 04)
  - Target audience profile (from Campaign Brief)
  - Awareness level (from Campaign Brief)
  - Key proof elements (from Research)
  - Soul.md voice constraints (if exists)

STEP 4: Compile downstream readiness checklist
  - Verify all selected concepts have complete handoff data
  - Verify no placeholder text remaining
  - Verify platform constraints are specific (not generic)
  - Verify word count targets are realistic for ad lengths

STEP 5: Write A07-HANDOFF.md
  - Organized by concept
  - Each concept self-contained (A07 can work on one concept at a time)
  - Clear section headers for each element type
  - No ambiguity in instructions
```

## Quality Checks
- Every selected concept has complete handoff (sections A through H)
- Hook text is VERBATIM (not paraphrased)
- Script architecture includes beat-by-beat timing and word count targets
- Visual direction includes shot sequence (not just "visual treatment type")
- Arena improvements are specific with persona/hybrid source attribution
- Human instructions incorporated verbatim from HUMAN_SELECTION.yaml
- Production constraints are specific to the platform (not generic)
- Variant generation guidance distinguishes variable from fixed elements
- Campaign context gives A07 sufficient upstream context to produce copy
- No placeholder text remaining in any field
- Effort level: high (handoff must be comprehensive and unambiguous)
