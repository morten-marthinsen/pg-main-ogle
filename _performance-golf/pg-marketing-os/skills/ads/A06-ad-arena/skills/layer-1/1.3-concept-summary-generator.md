# 1.3: Concept Summary Generator

## Purpose
For each complete concept, generate a 1-page summary that a persona evaluator can read and evaluate as a complete ad concept. The summary is the primary document each of the 7 personas evaluates during Arena rounds. It must present the concept as an ATOMIC UNIT -- hook + script + visual integrated together, not as separate elements.

## Inputs
- 1.1-concept-bundler output (concept bundles)
- 1.2-concept-completeness-validator output (only COMPLETE concepts)
- 0.1-upstream-package-loader output (strategic context)

## Process
```
STEP 1: Load complete concepts from 1.2 validation
  - FILTER: Only concepts marked COMPLETE proceed to summary generation
  - IF zero complete concepts: HALT -- resolve incomplete concepts first

STEP 2: For EACH complete concept, generate 1-page evaluation summary:

  SECTION A: Concept Overview
    - Concept ID: C-XXX
    - Platform: [target platform]
    - Ad length: [seconds]
    - Format: [Talking Head / B-Roll+VO / Text-on-Screen / etc.]

  SECTION B: The Hook (First 3 Seconds)
    - Hook text (VERBATIM): "[exact hook text]"
    - Hook type: [classification]
    - Opening visual: [what the viewer sees in the first frame]
    - Pattern interrupt technique: [how this stops the scroll]

  SECTION C: Script Architecture
    - Framework: [PAS/AIDA/BAB/etc.]
    - Story arc: [opening -> setup -> mechanism -> proof -> CTA]
    - Key beats:
      - [Beat 1]: [description + timing]
      - [Beat 2]: [description + timing]
      - [Beat 3]: [description + timing]
    - Mechanism moment: [where/how the mechanism is revealed]
    - Proof element: [what proof is included and where]
    - CTA: [call to action text and placement]

  SECTION D: Visual Direction
    - Treatment: [type]
    - Key visual moments:
      - Opening frame: [description]
      - Mechanism reveal visual: [description]
      - CTA visual: [description]
    - Visual-copy coherence: [how visual and copy work together]
    - Sound design: [sound-on / sound-off strategy]

  SECTION E: Strategic Alignment
    - Big idea alignment: [how this concept expresses the big idea]
    - Target audience: [who this concept is designed for]
    - Awareness level: [Schwartz stage this targets]
    - Mechanism clarity: [how the mechanism is communicated]

STEP 3: Verify each summary can stand alone
  - A persona evaluator reading ONLY this summary should understand
    the complete concept as an integrated unit
  - No element should feel detached from the others
```

## Output File
- **Path:** `layer-1-outputs/1.3-concept-summary-generator.md`
- **Minimum size:** 5KB

## Quality Checks
- Only COMPLETE concepts get summaries (no incomplete concepts)
- Summaries present concept as integrated unit (not three separate elements)
- Hook text is VERBATIM
- Script beats include timing
- Visual direction includes specific shot descriptions
- Strategic alignment section connects concept to big idea, audience, awareness
- Each summary is standalone-readable by a persona evaluator
