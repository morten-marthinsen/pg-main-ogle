# 2.5.1: Improvement Decomposition

## Purpose
For each concept, decompose all 7 Round 3 evaluation outputs into discrete improvement suggestions. Tag each suggestion by: which criterion it improves, how significant the improvement is (1-10), whether it conflicts with other suggestions. Build an improvement matrix per concept.

## Inputs
- 2.11-round3-persona-evaluation output (ALL 7 FULL Round 3 evaluations per concept)
- 2.13-round3-targeted-revision output (final revised versions)
- 2.14-round3-final-scoring output (scores per persona per concept)

## Process
```
STEP 1: Load all 7 Round 3 evaluations per concept (FULL text)
  - READ: layer-2-outputs/2.11-round3-persona-evaluation.md
  - VERIFY: All 7 evaluations present in full per concept

STEP 2: FOR EACH concept:
  FOR EACH persona evaluation (7 total):

    A. EXTRACT discrete improvement suggestions
      - Each persona's "improvement_recommendation" section
      - Each persona's "specific_changes" list
      - Each persona's "revised_concept_sketch"
      - Break complex suggestions into atomic improvements

    B. TAG each suggestion:
      - criterion_improved: [which of 7 criteria]
      - significance: [1-10 -- how much would this improve the concept?]
      - persona_source: [which persona generated this]
      - element_targeted: [hook / script / visual / integration]
      - specificity: [specific / general]

    C. CHECK for conflicts between suggestions
      - Does Suggestion A from DR Strategist conflict with Suggestion B from UGC Native?
      - Mark conflicting pairs
      - Assess which suggestion is stronger / has more evidence

STEP 3: BUILD improvement matrix per concept
  - Rows: aspects of concept (hook, script structure, visual treatment, mechanism clarity, etc.)
  - Columns: 7 personas
  - Cells: Specific improvement suggestion + significance score + conflict flags

STEP 4: IDENTIFY best suggestion per aspect
  - For each aspect, which persona had the BEST improvement suggestion?
  - Account for conflicts -- can't use two conflicting suggestions

STEP 5: Compile non-conflicting best-suggestion set per concept
  - This is the raw material for hybrid concept generation (2.5.2)
```
