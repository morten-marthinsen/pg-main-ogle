# 1.5: Fatigue Scorer

**Version:** 1.0
**Layer:** 1 (Individual Variant Scoring)
**Type:** Judgment / Prediction
**Model:** opus
**Dependencies:** 0.1 (Variant Matrix), 0.2 (Competitive Intelligence), 0.4 (Campaign Brief), 0.5 (Benchmarks)
**Output:** fatigue_scores.md

---

## Purpose

Score EVERY variant on Dimension 5: Creative Fatigue Risk. This predicts how quickly a variant's performance will DEGRADE once launched at scale. Creative fatigue is the #1 operational challenge in paid advertising — even winning ads eventually lose effectiveness. Fatigue scoring must reference A01 data on competitor ad run durations, hook type fatigue profiles, competitive density, planned spend level, and format fatigue characteristics. Scoring is INVERTED — higher score = LOWER fatigue risk / LONGER predicted lifespan.

## Process

### Step 1: Load All Evidence Sources

```
READ all input files
FOR EACH variant, COLLECT available evidence:

EVIDENCE SOURCE A: A01 Run Duration Data (from 0.2)
  LOOK UP variant's hook_type in fatigue_reference table
  EXTRACT:
    - longest_run_duration_days: from TIER_WINNER specimens (90+ days = low fatigue)
    - average_run_duration_days: across specimens of this hook type
    - fatigue_signal: low / medium / high
  IF no A01 data for this hook type: note as data gap

EVIDENCE SOURCE B: Hook Type Fatigue Profile (from 0.5 benchmarks)
  CLASSIFY hook type fatigue speed:
    - Story/narrative hooks: SLOW fatigue (higher rewatch value)
    - Demonstration hooks: SLOW fatigue (educational value)
    - UGC hooks: MEDIUM fatigue (dependent on actor saturation)
    - Benefit-led hooks: MEDIUM fatigue (standard)
    - Shock/controversy hooks: FAST fatigue (novelty-dependent)
    - Trend-riding hooks: FAST fatigue (trend expiration)
    - Meme-based hooks: VERY FAST fatigue (viral cycle exhaustion)

EVIDENCE SOURCE C: Competitive Density (from 0.2)
  LOOK UP hook_type in hook_saturation_map
  DETERMINE competitive density:
    - 5+ competitors using same hook type: ACCELERATED fatigue
      (audience sees this approach across multiple ads → faster burnout)
    - 2-4 competitors: MODERATE acceleration
    - 0-1 competitors: MINIMAL acceleration (unique approach)

EVIDENCE SOURCE D: Spend Level (from 0.4)
  DETERMINE client's spend level category:
    - Low (<$5K/week): typical lifespan 4-8 weeks
    - Medium ($5-25K/week): typical lifespan 2-4 weeks
    - High (>$25K/week): typical lifespan 1-2 weeks
  Higher spend = more impressions = faster audience saturation = faster fatigue

EVIDENCE SOURCE E: Format Fatigue Profile
  CLASSIFY format fatigue speed:
    - Video (long-form 60s+): SLOW (multiple rewatch angles)
    - Video (short-form <30s): MEDIUM (quick consumption = quick exhaustion)
    - Static image: MEDIUM-FAST (single impression, quickly processed)
    - Carousel: MEDIUM (multiple frames add variety)
    - UGC-style: MEDIUM (authenticity provides some protection)
```

### Step 2: Score Each Variant

```
FOR EACH variant:

  1. DETERMINE baseline from hook type fatigue profile:
     - Slow fatigue types: baseline 7-8
     - Medium fatigue types: baseline 5-6
     - Fast fatigue types: baseline 3-4

  2. ADJUST for A01 run duration data:
     - TIER_WINNER specimens (90+ days) with same hook type: +1 to +2
     - Average run > 30 days: +1
     - Average run < 14 days: -1
     - No A01 data: 0 (but note LOW confidence)

  3. ADJUST for competitive density:
     - 5+ competitors with same hook: -1 (accelerated fatigue)
     - 2-4 competitors: -0.5
     - 0-1 competitors: +0.5 (unique approach resists fatigue longer)

  4. ADJUST for spend level:
     - Low spend: +1 (slower impression accumulation)
     - Medium spend: 0
     - High spend: -1 (rapid impression accumulation)

  5. ADJUST for format:
     - Long-form video: +0.5
     - Short-form video: 0
     - Static: -0.5
     - Carousel: 0

  6. CLAMP to 1-10

  7. CALCULATE predicted effective lifespan range:
     Score 9-10 → 6+ weeks at planned spend
     Score 7-8 → 3-6 weeks
     Score 6 → 2-3 weeks (TYPICAL)
     Score 4-5 → 1-2 weeks
     Score 2-3 → < 1 week
     Score 1 → days

     ADJUST range for actual spend level:
     Multiply by 1.5x for low spend, 1.0x for medium, 0.7x for high

  8. DETERMINE confidence range and level:
     - HIGH: A01 data available + multiple evidence sources agree
     - MEDIUM: some A01 data + benchmark alignment
     - LOW: no A01 data for this hook type, prediction based on
       type classification and benchmarks only

  9. BUILD evidence chain citing A01 data specifically:
     "Hook type [X] ran for [Y] days in A01 specimens [IDs] (TIER_WINNER),
      competitive density is [Z] with [N] competitors using similar approach,
      at client's $[W]/week spend level, predicted lifespan is [range]."
```

### Step 3: Apply Scoring Scale

```
SCORING SCALE (INVERTED — higher = LOWER fatigue risk):

9-10: LOW fatigue risk — predicted effective lifespan 6+ weeks at moderate spend
      Evidence: unique hook type in competitive set + story/demo format +
      low competitive density

7-8:  MODERATE-LOW fatigue risk — predicted 3-6 weeks effective
      Evidence: solid hook type + moderate competitive density +
      some A01 longevity data

6:    MODERATE fatigue risk — predicted 2-3 weeks effective (TYPICAL)
      Evidence: standard hook type + average competitive density

4-5:  ELEVATED fatigue risk — predicted 1-2 weeks effective
      Evidence: common hook type + high competitive density OR
      novelty-dependent hook + high spend

2-3:  HIGH fatigue risk — predicted < 1 week effective at high spend
      Evidence: oversaturated hook type + high competitive density +
      novelty-dependent format

1:    EXTREME fatigue risk — novelty-dependent, likely exhausted within days
      Evidence: meme/trend-riding + very high competitive density +
      high spend level

NOTE: Fatigue prediction interacts with spend level. All predictions
must specify the client's PLANNED spend level, not a generic benchmark.
```

## Output Schema

```yaml

  fatigue_scores:
    dimension: "Creative Fatigue Risk"

    scoring_note: "INVERTED scale — higher = lower fatigue risk / longer lifespan"

      - variant_id: string

        confidence_range: string

        predicted_lifespan_range: string  # e.g., "3-6 weeks at $12K/week"
        fatigue_profile: enum[slow, medium, fast, very_fast]
        evidence_chain:
          a01_run_duration:
            source: "A01 Section 6"
            longest_run_days: integer  # or "N/A"
            average_run_days: integer  # or "N/A"
            tier_winner_match: boolean
            impact: string
          hook_type_profile:
            source: "benchmark classification"
            fatigue_speed: enum[slow, medium, fast, very_fast]
            rationale: string
            impact: string
          competitive_density:
            source: "A01 Section 3"
            competitors_with_same_hook: integer
            saturation_level: string
            impact: string
          spend_level:
            source: "Campaign Brief"
            weekly_spend: float
            spend_category: enum[low, medium, high]
            impact: string
          format_profile:
            format: string
            fatigue_speed: string
            impact: string
        refresh_strategy: string  # e.g., "Hook swap at week 3, concept refresh at week 5"
```

## Quality Checks

1. ALL variants scored (count must match total from 0.1)
2. Every score has a confidence range and confidence level
3. Every score includes predicted lifespan range at client's planned spend
4. A01 run duration data cited where available
5. Competitive density factored into every score
6. Client spend level explicitly documented and factored in
7. Format fatigue profile assessed for every variant
8. Refresh strategy suggested for each variant
9. Scores are meaningfully differentiated
10. "Fatigue prediction is too speculative" rationalization NOT present
