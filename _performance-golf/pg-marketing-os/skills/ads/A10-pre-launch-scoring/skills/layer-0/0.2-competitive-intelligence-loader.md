# 0.2: Competitive Intelligence Loader

**Version:** 1.0
**Layer:** 0 (Foundation)
**Type:** Data Loading
**Dependencies:** A01 Ad Intelligence Handoff
**Output:** competitive_intelligence_extract.md

---

## Purpose

Load the A01 AD-INTELLIGENCE-HANDOFF.md and extract all competitive context data required for scoring. This provides the market reality check for every prediction dimension: hook type saturation for scroll-stop scoring, winning ad specimens for fatigue prediction, competitor claim analysis for compliance scoring, and opportunity gaps for differentiation assessment.

---

## Inputs

```yaml
input:
  from_A01:
    handoff_path: "[project]/A01-ad-intelligence/AD-INTELLIGENCE-HANDOFF.md"
    # Minimum 100KB to be genuine A01 output (not a stub)
```

---

## Process

### Step 1: Locate A01 Handoff

```
VERIFY: A01 AD-INTELLIGENCE-HANDOFF.md exists at specified path
VERIFY: File size >= 100KB (genuine A01 output is substantial)

IF NOT EXISTS:
  → HALT: "A01 Ad Intelligence Handoff not found. Execute A01 first."
  → A10 CANNOT score without competitive context.

IF size < 100KB:
  → WARNING: "A01 output is only [X]KB — may be incomplete."
  → LOG warning but proceed (some competitive data > no competitive data)
```

### Step 2: Extract Hook Type Distribution (Section 3)

```
READ Section 3: Hook Type Distribution from A01 Handoff
EXTRACT:
  - Complete hook type frequency table
  - Per hook type: count, percentage of competitive set, trend direction
  - Oversaturated hook types (>20% of competitive set)
  - Underused hook types (<5% of competitive set)
  - Hook types with highest scroll-stop benchmarks

BUILD hook_type_saturation_map:
  For each hook type found in A01:
    - saturation_level: oversaturated / moderate / underused / absent
    - competitor_count: how many competitors use this type
    - benchmark_hook_rate: if available from AD-HOOK-TAXONOMY.md
    - scroll_stop_implication: penalty / neutral / boost

This feeds Dimension 1 (Scroll-Stop Probability) scoring in Layer 1.
```

### Step 3: Extract Format Distribution (Section 4)

```
READ Section 4: Format Distribution from A01 Handoff
EXTRACT:
  - Format frequency (video / static / carousel / UGC)
  - Platform-specific format preferences
  - Dominant format type in competitive set
  - Format gaps (formats competitors are NOT using)

This feeds Dimension 2 (Engagement Prediction) for platform nativeness assessment.
```

### Step 4: Extract Winning Ad Specimens (Section 6)

```
READ Section 6: Winning Ad Specimens from A01 Handoff
EXTRACT:
  - TIER_WINNER specimens (90+ days running — highest confidence)
  - TIER_STRONG specimens (30-89 days running)
  - Per specimen:
    - Hook type used
    - Format type
    - Estimated run duration
    - Platform(s)
    - Key creative elements

BUILD fatigue_reference_table:
  For each hook type in winning specimens:
    - longest_run_duration: from TIER_WINNER data
    - average_run_duration: across all specimens of this type
    - fatigue_signal: low (long runs) / medium / high (short runs only)

This feeds Dimension 5 (Creative Fatigue Risk) scoring in Layer 1.
```

### Step 5: Extract Opportunity Gaps (Section 8)

```
READ Section 8: Opportunity Gaps from A01 Handoff
EXTRACT:
  - Underexploited hook types (competitive whitespace)
  - Format gaps (underused formats)
  - Messaging gaps (angles competitors miss)
  - Platform gaps (platforms competitors neglect)

BUILD opportunity_map:
  For each gap identified:
    - gap_type: hook / format / message / platform
    - confidence: HIGH / MEDIUM / LOW
    - scoring_implication: variants exploiting this gap get a boost

This informs scroll-stop and engagement scoring boost for variants that exploit gaps.
```

### Step 6: Extract Platform Intelligence (Section 9)

```
READ Section 9: Platform Intelligence from A01 Handoff
EXTRACT:
  - Platform-specific compliance notes
  - What claims competitors are successfully running (platform tolerance proxy)
  - Platform ad policies affecting this vertical
  - Ad review rejection patterns observed

This feeds Dimension 4 (Compliance Risk) scoring in Layer 1.
```

### Step 7: Extract Recommended Hooks (Section 10)

```
READ Section 10: Recommended Hooks from A01 Handoff
EXTRACT:
  - A01's recommended hook types with rationale
  - Specific hook formulations suggested
  - Confidence levels on recommendations

This provides a cross-reference for scoring — variants aligned with A01 recommendations
get a confidence boost; variants diverging from recommendations need stronger evidence.
```

### Step 8: Build Competitive Context Summary

```
COMPILE all extractions into a unified competitive context document:

1. HOOK SATURATION MAP — which hook types are oversaturated vs. underused
2. FORMAT LANDSCAPE — dominant and missing formats
3. FATIGUE REFERENCE TABLE — how long similar hooks have run
4. OPPORTUNITY MAP — competitive whitespace
5. COMPLIANCE CONTEXT — what platforms are allowing
6. RECOMMENDED HOOKS — A01's prioritized suggestions
```

## Output Schema

```yaml
output:
  competitive_context:
    a01_file_loaded: string  # path
    a01_file_size_kb: integer

    hook_saturation_map:
      - hook_type: string
        saturation_level: enum[oversaturated, moderate, underused, absent]
        competitor_count: integer
        percentage_of_competitive_set: float
        benchmark_hook_rate: float  # if available
        scroll_stop_implication: enum[penalty, neutral, boost]

    format_landscape:
      dominant_format: string
      format_gaps: [string]
      platform_format_preferences:
        meta: [string]
        tiktok: [string]
        google: [string]

    fatigue_reference:
      - hook_type: string
        longest_run_duration_days: integer
        average_run_duration_days: integer
        specimen_count: integer
        fatigue_signal: enum[low, medium, high]

    opportunity_map:
      - gap_type: enum[hook, format, message, platform]
        description: string
        confidence: enum[HIGH, MEDIUM, LOW]
        scoring_implication: string

    compliance_context:
      platform_tolerance_notes: [string]
      claims_competitors_running: [string]
      rejection_patterns: [string]
      vertical_specific_warnings: [string]

    recommended_hooks:
      - hook_type: string
        rationale: string
        confidence: enum[HIGH, MEDIUM, LOW]

    extraction_completeness:
      sections_extracted: integer  # target: 6 (sections 3,4,6,8,9,10)
      sections_missing: [string]
      data_quality_notes: [string]
```
