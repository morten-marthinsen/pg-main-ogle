# 0.1: Variant Matrix Loader

**Version:** 1.0
**Layer:** 0 (Foundation)
**Type:** Data Loading
**Dependencies:** A09 Assembled Variant Matrix
**Output:** variant_matrix_extract.md

---

## Purpose

Load the A09 Assembled Variant Matrix (VARIANT-MATRIX.yaml or equivalent) and extract the complete list of variants to be scored. Build a variant inventory table with all fields required for downstream scoring dimensions. Count total variants. This establishes the scoring universe — every variant in this inventory MUST receive a complete 5-dimension scorecard in Layer 1.

---

## Inputs

```yaml
input:
  from_A09:
    variant_matrix_path: "[project]/A09-variant-assembly/VARIANT-MATRIX.yaml"
    # OR equivalent assembled variant output from A09
```

---

## Process

### Step 1: Locate Variant Matrix

```
VERIFY: A09 Variant Matrix output exists at specified path
IF NOT EXISTS:
  → HALT: "Variant Matrix not found. Execute A09 (Variant Assembly) first."
  → A10 cannot score variants that do not exist.
  → Gate 0 will FAIL if this file is missing.
```

### Step 2: Extract All Variant Records

```
READ VARIANT-MATRIX.yaml (or equivalent)
FOR EACH variant in the matrix:
  EXTRACT:
    - variant_id: unique identifier (e.g., V-001, V-002)
    - hook_text: first 3 seconds / first line (the scroll-stop element)
    - body_text: main copy / script body
    - cta_text: call-to-action text
    - visual_treatment: visual direction or description
    - platform_target: Meta / TikTok / Google / YouTube / multi-platform
    - format_type: video / static / carousel / UGC / native
    - ad_length: target duration for video, word count for text
    - parent_concept_id: link back to A06 Arena concept (for score mapping)
    - hook_type: from A02 classification (if available)

  IF any critical field is EMPTY or MISSING:
    LOG WARNING: "Variant [ID] has missing field: [field_name]"
    DO NOT skip the variant — include it but flag the gap
```

### Step 3: Count and Validate

```
total_variants = COUNT(all extracted variants)

VALIDATE:
  - total_variants > 0 (cannot score an empty matrix)
  - Every variant has a variant_id (no duplicates)
  - Every variant has hook_text (cannot score scroll-stop without it)
  - Every variant has platform_target (needed for compliance scoring)

IF total_variants == 0:
  → HALT: "Variant matrix is empty. A09 must produce at least 1 variant."

IF duplicate variant_ids detected:
  → HALT: "Duplicate variant IDs found: [list]. A09 output is malformed."
```

### Step 4: Build Variant Inventory Table

```
CONSTRUCT inventory table:

| Variant ID | Hook (first 20 words) | Platform | Format | Length | Parent Concept | Hook Type | Flags |
|------------|----------------------|----------|--------|--------|----------------|-----------|-------|

POPULATE:
  - One row per variant
  - Hook text truncated to first 20 words for readability
  - Flags column: note any missing fields, format issues, or warnings

OUTPUT:
  - Complete inventory table
  - Total variant count
  - Platform distribution summary (how many per platform)
  - Format distribution summary (how many per format type)
  - Parent concept mapping (which variants derive from which A06 concepts)
```

## Output Schema

```yaml
output:
  variant_inventory:
    total_variants: integer  # Total count of variants to score
    variants:
      - variant_id: string
        hook_text: string
        body_text: string
        cta_text: string
        visual_treatment: string
        platform_target: string
        format_type: string
        ad_length: string
        parent_concept_id: string
        hook_type: string  # from A02 classification
        flags: [string]  # any warnings or missing fields

    distributions:
      by_platform:
        meta: integer
        tiktok: integer
        google: integer
        youtube: integer
        other: integer
      by_format:
        video: integer
        static: integer
        carousel: integer
        ugc: integer
        native: integer
      by_parent_concept:
        "[concept_id]": integer  # count of variants per concept

    validation:
      all_variants_have_ids: boolean
      all_variants_have_hooks: boolean
      all_variants_have_platforms: boolean
      duplicate_ids: [string]  # should be empty
      missing_field_warnings: [string]
```

## Output File

---

## Quality Checks

1. A09 Variant Matrix file exists and was read successfully
2. Total variant count is > 0
3. No duplicate variant IDs
4. Every variant has hook_text populated (critical for scroll-stop scoring)
5. Every variant has platform_target populated (critical for compliance scoring)
6. Every variant has parent_concept_id mapped (critical for Arena score lookup)
7. Platform and format distribution summaries are complete
8. All missing field warnings are logged (variants are NOT silently skipped)

---

## Dependencies

- A09 (Variant Assembly) must be complete — produces the VARIANT-MATRIX.yaml
- A02 (Hook Strategy) hook type classifications should be present (but variants can still be scored with structural analysis if hook_type is missing)
