# 3.2: Budget Allocation

**Version:** 1.0
**Layer:** 3 (Testing Strategy)
**Type:** Calculation / Strategy
**Model:** opus
**Persona:** PERSONA_PERFORMANCE_STRATEGIST
**Dependencies:** 3.1 (Testing Sequence), 2.3 (Tier Assignment), 0.4 (Campaign Brief)
**Output:** budget_allocation.md

---

## Purpose

Distribute the testing budget across variants with mathematical precision. Calculate per-variant daily spend, weekly spend, reserve budget, and verify that budget sums correctly. The allocation must ensure each Tier 1 variant receives sufficient spend for statistical significance (minimum $20-50/day depending on AOV), while maintaining a 20-30% reserve for scaling winners and activating Tier 2 variants.

## Process

### Step 1: Establish Budget Parameters

```
LOAD from 0.4:
  weekly_ad_spend: $[X]
  aov: $[X]
  kill_threshold: $[X] (= 3 × AOV)

LOAD from 2.3:
  tier_1_count: [N]
```

### Step 2: Calculate Reserve Budget

```
RESERVE BUDGET:
  reserve_percentage: 25%  (default — range is 20-30%)
  reserve_amount = weekly_ad_spend × 0.25

  RATIONALE: Reserve budget is held for:
    1. Scaling winners (increase daily spend on performing variants)
    2. Activating Tier 2 variants (when Tier 1 losers are killed)
    3. Unexpected opportunities (competitor exits, platform changes)

  ADJUST reserve percentage:
    IF tier_2_count > tier_1_count: increase to 30% (more variants to activate)
    IF tier_2_count == 0: decrease to 20% (no backup variants)
    IF campaign is short (<3 weeks): decrease to 20% (less time for scaling)

ACTIVE TESTING BUDGET:
  active_budget = weekly_ad_spend - reserve_amount
```

### Step 3: Calculate Per-Variant Allocation

```
TIER 1 ALLOCATION:
  Approach: Equal split across all Tier 1 variants (no variant gets > 25% of active budget)

  per_variant_weekly = active_budget / tier_1_count
  per_variant_daily = per_variant_weekly / 7

  VERIFY minimum spend for statistical significance:
    IF aov <= $50: minimum $20/day per variant
    IF aov $50-$150: minimum $30/day per variant
    IF aov > $150: minimum $50/day per variant

  IF per_variant_daily < minimum:
    OPTIONS:
      a. Reduce Tier 1 count (remove lowest-ranked Tier 1 → Tier 2)
      b. Increase weekly budget (human decision)
      c. Accept lower statistical confidence (document risk)

    DOCUMENT: "Per-variant daily spend of $[X] is below minimum $[Y] for
      $[AOV] AOV. Recommendation: [a/b/c with specifics]."

  VERIFY maximum concentration:
    No single variant > 25% of total active budget
    (This prevents over-investing in one variant before data arrives)
```

### Step 4: Build Budget Allocation Table

```
CONSTRUCT complete allocation table:

| Variant ID | Tier | Platform | Daily Spend | Weekly Spend | % of Active | % of Total |
|------------|------|----------|-------------|-------------|-------------|------------|
| V-007      | 1    | Meta     | $[X]        | $[X]         | [X]%        | [X]%       |
| V-003      | 1    | TikTok   | $[X]        | $[X]         | [X]%        | [X]%       |
| V-005      | 1    | Meta     | $[X]        | $[X]         | [X]%        | [X]%       |
| ...        | ...  | ...      | ...         | ...          | ...         | ...        |
| RESERVE    | —    | —        | —           | $[X]         | —           | [X]%       |
| TOTAL      | —    | —        | —           | $[total]     | —           | 100%       |

VERIFY: Active variant totals + reserve = weekly_ad_spend
  Tolerance: $0 (must be EXACT, not "approximately")
```

### Step 5: Calculate Spend-to-Kill Ratios

```
FOR EACH Tier 1 variant:
  days_to_kill = kill_threshold / per_variant_daily_spend
  weeks_to_kill = days_to_kill / 7

  EXAMPLE:
    Kill threshold: $300 (3 × $100 AOV)
    Daily spend: $42
    Days to kill: $300 / $42 = ~7.1 days
    Weeks to kill: ~1 week

  THIS TELLS US: How long each variant will run before kill decision

  IF days_to_kill < 3:
    WARNING: "Variant will hit kill threshold before minimum data window (72 hours).
      Either daily spend is too high or AOV is too low."
    RECOMMENDATION: Reduce daily spend for this variant or adjust kill timing

  IF days_to_kill > 14:
    WARNING: "Variant will take 2+ weeks to reach kill threshold. This delays
      Tier 2 activation and reduces testing velocity."
    RECOMMENDATION: Consider increasing daily spend or lowering the kill threshold
      (requires human approval — default is 3x AOV)
```

### Step 6: Plan Budget Reallocation

```
WHEN a Tier 1 variant is killed:
  Released budget = per_variant_daily_spend × remaining_days_in_week

  REALLOCATION PRIORITY:
    1. Scale a winning Tier 1 variant (+20-30% daily spend increase)
    2. Activate the next Tier 2 variant
    3. Add to reserve for future scaling

  DOCUMENT reallocation protocol:
    "When V-[X] is killed, reallocate $[daily]/day to:
     Option A: Scale V-[winner] from $[X]/day to $[X+20%]/day
     Option B: Activate V-[next Tier 2] at $[X]/day"

WHEN scaling a winner:
  Source: Reserve budget + killed variant reallocation
  Maximum scale: +20-30% per scaling event
  Frequency: Max 1 scale event per 48 hours per variant
```

## Quality Checks

1. Budget sums EXACTLY to weekly ad spend (active + reserve = total, zero tolerance)
2. No single variant exceeds 25% of active budget
3. Per-variant daily spend meets minimum threshold for AOV
4. Reserve budget is 20-30% of total
5. Days-to-kill calculated for every Tier 1 variant
6. Reallocation protocol documented (what happens when a variant is killed)
7. Scale protocol documented (how winners get more budget)
8. All warnings documented for sub-minimum spend or extreme kill timelines
9. Budget allocation table is complete with all columns
