# 2.2: Forced Ranking

**Version:** 1.0
**Layer:** 2 (Comparative Analysis & Tier Assignment)
**Type:** Ranking / Prioritization
**Model:** opus
**Dependencies:** 2.1 (Composite Score Calculator)
**Output:** forced_ranking.md

---

## Purpose

Force-rank ALL variants by confidence-adjusted composite score from #1 (highest) to #N (lowest). NO TIES at the tier boundary level — if two variants have identical composite scores, ties must be broken using the defined tiebreaker hierarchy. The forced ranking is the foundation for tier assignment. Every tie-breaking decision must be documented with rationale.

---

## Inputs

```yaml
input:
  from_2.1:
    composite_scores: "layer-2-outputs/2.1-composite-score-calculator.md"
    # Per-variant: confidence_adjusted_composite, all 5 dimension scores
```

---

## Process

### Step 1: Initial Sort

```
LOAD all variants with confidence_adjusted_composite from 2.1
SORT descending by confidence_adjusted_composite

RESULT: Initial ranked list from highest to lowest

CHECK for ties: any two variants with identical confidence_adjusted_composite
(within 0.01 tolerance — scores rounded to 2 decimals)
```

### Step 2: Tiebreaker Resolution

```
IF ties exist, resolve using this hierarchy (in order):

TIEBREAKER 1 (Primary): Scroll-Stop Score
  Higher scroll-stop score wins.
  RATIONALE: Scroll-stop is the most critical dimension — if everything
  else is equal, the variant more likely to stop the scroll launches first.

TIEBREAKER 2 (Secondary): Compliance Score
  Higher compliance score wins (lower risk).
  RATIONALE: Lower compliance risk means less chance of rejection, which
  means more reliable performance prediction.

TIEBREAKER 3 (Tertiary): Fatigue Score
  Higher fatigue score wins (longer predicted lifespan).
  RATIONALE: Longer lifespan means more value per testing dollar.

IF STILL TIED after all 3 tiebreakers (extremely rare):
  TIEBREAKER 4: Engagement Score
  TIEBREAKER 5: Conversion Score
  TIEBREAKER 6: Earlier variant ID (lower number — arbitrary but deterministic)

DOCUMENT every tie-breaking decision:
  "Tie between V-005 (adjusted 6.33) and V-012 (adjusted 6.33).
   Tiebreaker 1 (Scroll-Stop): V-005 = 8.0, V-012 = 7.5.
   V-005 ranked higher."
```

### Step 3: Assign Rank Positions

```
FOR EACH variant in the sorted, tie-broken list:
  ASSIGN rank: #1, #2, #3, ... #N

  VERIFY: No two variants share the same rank
  VERIFY: Rank numbers are consecutive (no gaps)
  VERIFY: Total ranks = total variants
```

### Step 4: Build Forced-Ranked List

```
CONSTRUCT complete ranked table:

| Rank | Variant ID | Adjusted Composite | SS   | ENG  | CVR  | CMP  | FAT  | Confidence | Tie-Broken? |
|------|------------|-------------------|------|------|------|------|------|------------|-------------|
| #1   | V-007      | 7.82              | 9.0  | 8.0  | 7.0  | 9.0  | 8.0  | HIGH       | No          |
| #2   | V-003      | 7.45              | 8.5  | 7.5  | 7.0  | 8.0  | 7.0  | HIGH       | No          |
| #3   | V-005      | 6.33              | 8.0  | 7.0  | 6.0  | 7.0  | 6.5  | MEDIUM     | Yes (TB1)   |
| #4   | V-012      | 6.33              | 7.5  | 7.0  | 6.5  | 7.0  | 6.0  | MEDIUM     | Yes (TB1)   |
| ...  | ...        | ...               | ...  | ...  | ...  | ...  | ...  | ...        | ...         |
| #N   | V-019      | 3.40              | 5.0  | 4.0  | 3.0  | 2.0  | 4.0  | LOW        | No          |

TB1 = Tiebreaker 1 (Scroll-Stop), TB2 = Tiebreaker 2 (Compliance), etc.
```

### Step 5: Document Score Distribution

```
ANALYZE the ranked list:

SCORE CLUSTERS:
  - Identify natural break points in the scores
  - Where are the biggest score gaps? (These often align with tier boundaries)
  - Are scores evenly distributed or clustered?

DISTRIBUTION METRICS:
  - Top quartile range: [X] to [Y]
  - Middle 50% range: [X] to [Y]
  - Bottom quartile range: [X] to [Y]
  - Largest gap between consecutive ranks: [X] points between rank #[A] and #[B]
  - Number of tie-breaking decisions required: [N]

NATURAL TIER BOUNDARY INDICATORS:
  - If there's a 1.0+ point gap at rank #[X], this is a natural tier boundary
  - Document these for 2.3 (Tier Assignment) to consider
```

## Output Schema

```yaml
output:
  forced_ranking:
    total_variants_ranked: integer
    tie_breaking_decisions: integer

    ranked_list:
      - rank: integer  # 1 to N
        variant_id: string
        confidence_adjusted_composite: float
        dimension_scores:
          scroll_stop: float
          engagement: float
          conversion: float
          compliance: float
          fatigue: float
        overall_confidence: enum[HIGH, MEDIUM, LOW]
        tie_broken: boolean
        tiebreaker_used: string  # e.g., "TB1: Scroll-Stop" or "N/A"
        tiebreaker_detail: string  # specific comparison data

    tie_breaking_log:
      - tied_variants: [string]  # variant IDs
        tied_composite: float
        tiebreaker_applied: string
        deciding_scores: string  # e.g., "V-005 SS=8.0 vs V-012 SS=7.5"
        winner: string

    distribution:
      top_quartile_range: { min: float, max: float }
      middle_50_range: { min: float, max: float }
      bottom_quartile_range: { min: float, max: float }
      largest_gap:
        between_ranks: [integer, integer]
        gap_size: float
      natural_tier_boundaries: [{ after_rank: int, gap_size: float }]

    verification:
      all_variants_ranked: boolean
      no_duplicate_ranks: boolean
      ranks_consecutive: boolean
      total_matches_inventory: boolean
```

## Quality Checks

1. ALL variants ranked (count matches total from Layer 0)
2. No ties remain at any position (all resolved)
3. Rank numbers are consecutive with no gaps (1, 2, 3, ... N)
4. No two variants share the same rank
5. Every tie-breaking decision is documented with specific comparison values
6. Tiebreaker hierarchy followed correctly (scroll-stop first, compliance second, fatigue third)
7. Distribution analysis complete with natural tier boundaries identified
8. Ranked list sorted correctly (highest adjusted composite = rank #1)
