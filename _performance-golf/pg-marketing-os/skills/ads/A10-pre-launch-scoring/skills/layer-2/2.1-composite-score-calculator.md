# 2.1: Composite Score Calculator

**Version:** 1.0
**Layer:** 2 (Comparative Analysis & Tier Assignment)
**Type:** Calculation / Analysis
**Model:** opus
**Dependencies:** All Layer 1 outputs (1.1-1.6)
**Output:** composite_scores.md

---

## Purpose

Calculate the weighted composite score for each variant by combining all 5 dimension scores with the A10 weighting formula. Then calculate the confidence-adjusted composite score that accounts for prediction uncertainty. The composite score is the primary ranking metric — it determines which variants launch first, which are backup, and which are revised or cut. The compliance gate override ensures that no variant with critical compliance issues gets a high composite regardless of creative quality.

## Process

### Step 1: Load All Dimension Scores

```
FOR EACH variant:
  LOAD:
    - scroll_stop_score (from 1.1)
    - engagement_score (from 1.2)
    - conversion_score (from 1.3)
    - compliance_score (from 1.4)
    - fatigue_score (from 1.5)
    - confidence_levels for each dimension

  VERIFY: All 5 scores exist for this variant
  IF any score missing: HALT — return to Layer 1 to complete scoring
```

### Step 2: Calculate Weighted Composite Score

```
WEIGHTS (from AGENT.md — FIXED, not adjustable without human approval):
  Scroll-Stop:   0.30  (30%)
  Engagement:    0.20  (20%)
  Conversion:    0.25  (25%)
  Compliance:    0.15  (15%)
  Fatigue:       0.10  (10%)

FORMULA:
  weighted_composite = (scroll_stop × 0.30) +
                       (engagement × 0.20) +
                       (conversion × 0.25) +
                       (compliance × 0.15) +
                       (fatigue × 0.10)

FOR EACH variant:
  CALCULATE weighted_composite
  ROUND to 2 decimal places

EXAMPLE:
  Scroll-Stop: 8.0, Engagement: 7.5, Conversion: 6.0, Compliance: 9.0, Fatigue: 7.0
  Composite = (8.0 × 0.30) + (7.5 × 0.20) + (6.0 × 0.25) + (9.0 × 0.15) + (7.0 × 0.10)
            = 2.40 + 1.50 + 1.50 + 1.35 + 0.70
            = 7.45
```

### Step 3: Apply Compliance Gate Override

```
CRITICAL RULE:
  IF compliance_score <= 3:
    weighted_composite = MIN(weighted_composite, 4.0)

  RATIONALE: A brilliant ad that gets rejected by platform review has ZERO
  actual performance. Capping the composite at 4.0 ensures compliance-failed
  variants cannot rank above Tier 2 threshold regardless of creative quality.

FOR EACH variant with compliance <= 3:
  RECORD original composite (before cap)
  APPLY cap: composite = MIN(composite, 4.0)
  FLAG: "Compliance gate override — composite capped from [original] to 4.0"
```

### Step 4: Calculate Confidence-Adjusted Composite

```
CONFIDENCE MULTIPLIERS:
  HIGH:   1.0   (full confidence — score stands as-is)
  MEDIUM: 0.85  (moderate uncertainty — conservative adjustment)
  LOW:    0.70  (significant uncertainty — substantial discount)

DETERMINE overall confidence level for each variant:
  COUNT confidence levels across 5 dimensions:
    - If 3+ dimensions are HIGH: overall = HIGH
    - If 3+ dimensions are MEDIUM: overall = MEDIUM
    - If 3+ dimensions are LOW: overall = LOW
    - Mixed: use the mode (most common level)

FORMULA:
  confidence_adjusted_composite = weighted_composite × confidence_multiplier

FOR EACH variant:
  CALCULATE confidence_adjusted_composite
  ROUND to 2 decimal places

EXAMPLE (continuing from Step 2):
  Composite: 7.45, Overall confidence: MEDIUM
  Adjusted = 7.45 × 0.85 = 6.33
```

### Step 5: Build Composite Score Table

```
CONSTRUCT complete table:

| Variant ID | SS | ENG | CVR | CMP | FAT | Weighted | Compliance Override | Confidence | Multiplier | Adjusted |
|------------|-----|------|------|------|------|----------|--------------------|-----------:|-----------|----------|
| V-001      | 8.0 | 7.5  | 6.0  | 9.0  | 7.0  | 7.45     | No                 | MEDIUM     | 0.85      | 6.33     |
| V-002      | 7.0 | 8.0  | 7.5  | 2.0  | 6.0  | 6.60     | YES → 4.00         | MEDIUM     | 0.85      | 3.40     |

SS = Scroll-Stop, ENG = Engagement, CVR = Conversion, CMP = Compliance, FAT = Fatigue

SORT by confidence_adjusted_composite descending (highest first)
```
