# 3.4: Platform Grouper

## Purpose

Group all variants by deployment platform for operational efficiency. Each platform should have a self-contained view of all its variants with format verification. Identify platform-level gaps (platforms with too few variants for meaningful testing). This output directly drives the per-platform directory structure created in Layer 4.

## Inputs

- **VARIANT-ID-MAP.yaml:** From 3.1 -- all variant IDs with platform assignments
- **FILE-MANIFEST.yaml:** From 3.3 -- file manifests with format compliance status
- **PLATFORM-SPECS.yaml:** From 0.3 -- platform format requirements
- **TESTING-PRIORITY.yaml:** From 3.2 -- tier assignments per variant

## Process

```
STEP 1: LOAD ALL INPUTS
  - Read VARIANT-ID-MAP.yaml (variants with platform assignments)
  - Read FILE-MANIFEST.yaml (manifests with format compliance)
  - Read PLATFORM-SPECS.yaml (platform format requirements)
  - Read TESTING-PRIORITY.yaml (tier assignments)

STEP 2: GROUP VARIANTS BY PLATFORM
  For each target platform (META, TIKTOK, YT, GOOG, PIN, SNAP):
    - Collect all variants assigned to this platform
    - Count total variants
    - Count by tier (TIER_1, TIER_2, TIER_3)
    - Count by concept (C1, C2, C3)
    - Count by assembly status (READY, NEEDS_ADAPTATION, BLOCKED)

STEP 3: VERIFY PLATFORM-SPECIFIC FORMAT COMPLIANCE
  For each platform group:
    - Verify all variants meet the platform's format requirements
    - Check aspect ratios
    - Check duration limits
    - Check file types
    - Check file sizes
    - Record any variants needing adaptation for this specific platform

STEP 4: IDENTIFY PLATFORM-LEVEL GAPS
  For each platform:
    - IF total variants < 5: WARNING -- below recommended minimum for testing
    - IF total variants < 15: NOTE -- below recommended count for robust testing
    - IF no TIER_1 variants: WARNING -- no high-priority variants for this platform
    - IF all variants are one concept: WARNING -- no concept diversity
    - Record recommended minimum and gap analysis

STEP 5: CREATE PLATFORM-LEVEL SUMMARY
  For each platform:
    - Total variants
    - Variants by tier
    - Variants by concept
    - Variants ready for deployment vs needing adaptation
    - Platform-specific notes (format requirements, constraints)
    - Recommended testing cadence for this platform

STEP 6: COMPUTE CROSS-PLATFORM STATISTICS
  - Total variants across all platforms
  - Platform with most variants
  - Platform with fewest variants
  - Platform with highest TIER_1 concentration
  - Cross-platform variant distribution chart

STEP 7: WRITE PLATFORM-GROUPS.yaml
```

## Quality Checks

- [ ] Every variant assigned to exactly one platform group
- [ ] Total variants across all groups equals total variants in matrix
- [ ] Per-platform tier distribution computed
- [ ] Per-platform concept distribution computed
- [ ] Per-platform assembly status computed
- [ ] Platform-level gaps identified (< 5 variants = warning)
- [ ] Format requirements documented per platform
- [ ] Variant ID lists complete for each platform group
- [ ] Cross-platform statistics computed
- [ ] No variant appears in multiple platform groups (would indicate duplication)
