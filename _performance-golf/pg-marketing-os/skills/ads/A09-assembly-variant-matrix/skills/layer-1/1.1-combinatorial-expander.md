# 1.1: Combinatorial Expander

## Purpose

Compute the full Cartesian product of all copy elements and visual assets across all concepts and platforms. This produces the COMPLETE list of theoretical combinations before any filtering. Every possible hook x body x CTA x visual x platform combination must be enumerated -- no estimation, no sampling, no "representative set."

## Inputs

- **COPY-CATALOG.yaml:** From 0.1 -- all hooks, bodies, CTAs per concept
- **ASSET-CATALOG.yaml:** From 0.2 -- all visuals/videos per concept
- **PLATFORM-SPECS.yaml:** From 0.3 -- target platforms list
- **ARENA-SCORES.yaml:** From 0.4 -- concept IDs for numbering

## Process

```
STEP 1: LOAD ALL INPUT CATALOGS
  - Read COPY-CATALOG.yaml
  - Read ASSET-CATALOG.yaml
  - Read PLATFORM-SPECS.yaml
  - Read ARENA-SCORES.yaml
  - Extract: concepts list, per-concept hooks, bodies, CTAs, visuals, target platforms

STEP 2: COMPUTE THEORETICAL COMBINATION COUNT
  For each concept C:
    theoretical_C = hooks_C * bodies_C * ctas_C * visuals_C * platforms
  Total theoretical = SUM(theoretical_C for all concepts)

  Example calculation:
    C1: 5 hooks x 2 bodies x 3 CTAs x 2 visuals x 2 platforms = 120
    C2: 5 hooks x 2 bodies x 3 CTAs x 2 visuals x 2 platforms = 120
    C3: 5 hooks x 2 bodies x 3 CTAs x 2 visuals x 2 platforms = 120
    Total: 360 theoretical combinations

STEP 3: ENUMERATE EVERY COMBINATION
  For each concept C:
    For each hook H in C:
      For each body B in C:
        For each CTA in C:
          For each visual V in C:
            For each platform P in target platforms:
              Create combination record:
                - Provisional variant ID: C[n]-H[nn]-B[n]-CTA[n]-V[nn]-[PLATFORM]
                - Concept: C[n] (with name and Arena score)
                - Hook: H[nn] (with text, type, category)
                - Body: B[n] (with summary, word count, framework)
                - CTA: CTA[n] (with text, type)
                - Visual: V[nn] (with description, type, format specs)
                - Platform: [PLATFORM]
                - Status: ENUMERATED (pre-filtering)

  CRITICAL: Do NOT skip any combination. Do NOT estimate. Every single
  combination in the Cartesian product must appear as a row in the output.

STEP 4: ASSIGN PROVISIONAL VARIANT IDS
  - Use the binding naming convention: [CONCEPT]-[HOOK]-[BODY]-[CTA]-[VISUAL]-[PLATFORM]
  - Zero-pad hook and visual numbers to 2 digits (H01, V03)
  - Platform tag is LAST
  - Verify all IDs are unique
  - Verify all IDs follow the format exactly

STEP 5: VERIFY ENUMERATION COMPLETENESS
  - Count total enumerated combinations
  - Verify count matches the theoretical calculation from Step 2
  - IF count != theoretical: HALT -- missing combinations in enumeration
  - Record per-concept counts
  - Record per-platform counts

STEP 6: WRITE RAW-COMBINATIONS.yaml
  - Write every enumerated combination to output file
  - Include theoretical count for verification
  - Include per-concept and per-platform breakdowns
```
