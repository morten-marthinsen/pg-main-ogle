# 0.1: Copy Variant Loader

## Purpose

Load ALL copy variants from the A07 Copy Production `AD-COPY-FINAL/` directory. Build a comprehensive copy catalog that enumerates every hook, body variant, and CTA variant organized by concept and platform. This catalog is the copy input to combinatorial expansion in Layer 1.

## Inputs

- **A07 Copy Final Directory:** `[project]/A07-copy-production/AD-COPY-FINAL/` -- contains all final copy variants from A07
- **A06 Arena Results:** For concept ID cross-referencing (winning concept IDs)

## Process

```
STEP 1: LOCATE A07 COPY DIRECTORY
  - Verify AD-COPY-FINAL/ directory exists at expected path
  - IF directory does not exist: HALT -- A07 has not completed
  - IF directory exists: list ALL files and subdirectories

STEP 2: ENUMERATE ALL COPY ELEMENTS
  For each file in AD-COPY-FINAL/:
    a. Identify element type: HOOK, BODY, or CTA
    b. Identify parent concept (C1, C2, C3, etc.)
    c. Identify platform-specific version (if A07 produced platform variants)
    d. Record the A07 variant ID (original ID from A07 output)
    e. Record file path (exact, verified)
    f. Record file size
    g. For hooks: extract hook text, identify hook type (from 32-type taxonomy), identify hook category (A-J)
    h. For bodies: record word count, identify script framework (PAS/AIDA/BAB/etc.), create 1-2 sentence summary
    i. For CTAs: extract CTA text, identify CTA type (urgency/risk_reversal/low_friction/etc.)

STEP 3: BUILD COPY CATALOG
  Organize all elements into structured catalog:
    Per Concept:
      - Concept ID (C1, C2, C3)
      - Hooks: [list with IDs, text, type, category, file path]
      - Bodies: [list with IDs, summary, word count, framework, file path]
      - CTAs: [list with IDs, text, type, file path]
      - Platform-specific versions: [if any]

STEP 4: COUNT AND VERIFY
  - Count total hooks per concept
  - Count total bodies per concept
  - Count total CTAs per concept
  - Verify minimum 5 hooks per concept (Gate 0 requirement)
  - Verify minimum 2 bodies per concept
  - Verify minimum 2 CTAs per concept
  - Verify no missing files (all paths resolve to existing files)
  - Flag any concept with fewer than minimum elements

STEP 5: WRITE COPY-CATALOG.yaml
  - Write structured catalog to output file
  - Include per-concept element counts
  - Include file path verification status
```
