# 0.3: Format Strategy Loader

## Purpose

Load the FORMAT-STRATEGY.md from A03 (Format Strategy) and extract all platform-specific requirements, constraints, and specifications. This data drives platform-specific filtering in Layer 1 and platform assignment in Layer 3.

## Inputs

- **A03 Format Strategy:** `[project]/A03-format-strategy/FORMAT-STRATEGY.md` -- the format strategy output from A03
- **0.0.1 Vertical Profile:** For cross-referencing platform priorities

## Process

```
STEP 1: LOCATE FORMAT STRATEGY FILE
  - Verify FORMAT-STRATEGY.md exists at expected path
  - IF file does not exist: HALT -- A03 has not completed
  - IF file exists: read entire file

STEP 2: EXTRACT PLATFORM LIST WITH PRIORITY
  - Identify all target platforms (Meta, TikTok, YouTube, Google, Pinterest, Snapchat)
  - Record priority ranking (primary, secondary, tertiary)
  - Record which platforms are REQUIRED vs OPTIONAL for the campaign

STEP 3: EXTRACT FORMAT REQUIREMENTS PER PLATFORM
  For each target platform:
    a. Aspect ratios supported (16:9, 9:16, 1:1, 4:5)
    b. Preferred aspect ratio
    c. Duration limits by placement:
       - Feed: [max seconds]
       - Stories/Reels: [max seconds]
       - In-stream/pre-roll: [max seconds]
    d. File type requirements (mp4, jpg, png, gif)
    e. Maximum file size (MB)
    f. Resolution requirements (minimum and recommended)

STEP 4: EXTRACT PLATFORM-SPECIFIC CONSTRAINTS
  For each target platform:
    a. Meta: sound-off requirements (85% feed is sound-off),
       text overlay rules (20% text rule for images),
       caption/primary text character limits
    b. TikTok: vertical-first (9:16 native),
       sound-on default,
       native editing style expectations,
       duration sweet spots (15s, 30s, 60s)
    c. YouTube: skip button timing (5s non-skippable intro),
       aspect ratio (16:9 primary),
       bumper ad specs (6s),
       in-stream vs discovery formats
    d. Google: responsive display specs,
       headline/description character limits
    e. Pinterest: pin format specs,
       vertical image emphasis (2:3 preferred)
    f. Snapchat: full-screen vertical (9:16),
       duration limits,
       swipe-up mechanics

STEP 5: BUILD PLATFORM-SPECS.yaml
  - Write structured specs to output file
  - Include priority ranking
  - Include all format requirements and constraints
  - Include placement-specific duration limits

STEP 6: VALIDATE COMPLETENESS
  - Verify at least 2 platforms have complete specs
  - Verify format requirements are populated (not placeholder)
  - Cross-reference with 0.0.1 vertical profile platform priorities
```
