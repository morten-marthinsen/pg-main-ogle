# 0.5: Input Validator

## Purpose

Validate ALL inputs are present, complete, and properly cross-referenced before combinatorial expansion begins. This is the Gate 0 gatekeeper -- it verifies that A07 copy, A08 assets, A03 format strategy, and A06 Arena results all align on concept IDs, platform coverage, and minimum element counts.

## Inputs

- **COPY-CATALOG.yaml:** Output from 0.1 (copy variant loader)
- **ASSET-CATALOG.yaml:** Output from 0.2 (asset loader)
- **PLATFORM-SPECS.yaml:** Output from 0.3 (format strategy loader)
- **ARENA-SCORES.yaml:** Output from 0.4 (arena results loader)
- **0.0.1 Vertical Profile Output:** From 0.0.1 (vertical profile loader)

## Process

```
STEP 1: VERIFY ALL PREREQUISITE FILES EXIST
  - Check: COPY-CATALOG.yaml exists -> if not, HALT
  - Check: ASSET-CATALOG.yaml exists -> if not, HALT
  - Check: PLATFORM-SPECS.yaml exists -> if not, HALT
  - Check: ARENA-SCORES.yaml exists -> if not, HALT
  - Check: 0.0.1 vertical profile output exists -> if not, HALT
  - Record file paths and sizes for all inputs

STEP 2: CROSS-REFERENCE CONCEPT IDS
  - Extract concept IDs from Arena results (C1, C2, C3)
  - Verify EVERY concept in Arena results has copy variants in COPY-CATALOG.yaml
  - Verify EVERY concept in Arena results has assets in ASSET-CATALOG.yaml
  - IF any concept is missing from copy catalog: flag as BLOCKING gap
  - IF any concept is missing from asset catalog: flag as BLOCKING gap
  - IF concept numbering is inconsistent across catalogs: flag as BLOCKING gap

STEP 3: VERIFY MINIMUM ELEMENT COUNTS
  Per concept:
    a. Hooks >= 5 (if fewer, flag which concept and by how many)
    b. Bodies >= 2 (if fewer, flag)
    c. CTAs >= 2 (if fewer, flag)
    d. Visual/video treatments >= 2 (if fewer, flag)

  Aggregate:
    a. Total winning concepts >= 3
    b. Total hooks across all concepts >= 15
    c. Total visual treatments across all concepts >= 6

STEP 4: VERIFY PLATFORM COVERAGE
  - Extract platform list from PLATFORM-SPECS.yaml
  - Verify platforms in format strategy match platforms represented in copy catalog
  - Verify platforms in format strategy match platforms represented in asset catalog
  - Minimum 2 platforms must be covered
  - IF platform mismatch: flag which platforms are missing where

STEP 5: VERIFY FILE INTEGRITY
  - Check that all file paths in COPY-CATALOG.yaml reference existing files
  - Check that all file paths in ASSET-CATALOG.yaml reference existing files
  - IF any referenced file does not exist: flag as BLOCKING gap with exact path

STEP 6: COMPILE VALIDATION REPORT
  - Aggregate all checks into INPUT-VALIDATION-REPORT.md
  - For each check: PASS / FAIL with specific details
  - If ALL checks pass: Gate 0 can be PASSED
  - If ANY check fails: Gate 0 FAILS with specific resolution actions

STEP 7: DETERMINE GATE 0 STATUS
  Gate 0 PASSES if and only if:
    - All 5 prerequisite files exist
    - All concepts in Arena results have copy and assets
    - Concept numbering is consistent
    - Minimum 3 winning concepts
    - Minimum 5 hooks per concept
    - Minimum 2 visual treatments per concept
    - Copy and asset catalogs reference same concept IDs
    - Platform coverage >= 2 platforms
    - All referenced files exist on disk
```
