# 4.3: Summary Assembler

## Purpose

Assemble AD-VARIANT-MATRIX-SUMMARY.md -- the human-readable overview of the complete variant matrix. This document must contain ALL 7 required sections and meet the 20KB minimum size requirement. It is the primary document the human reviewer and team members read to understand what was produced, what to test first, and where everything is.

## Inputs

- **VARIANT-MATRIX.yaml:** From 4.1 -- complete variant database
- **AD-VARIANT-MATRIX/ directories:** From 4.2 -- per-platform directories
- **COHERENCE-VALIDATION-REPORT.md:** From 2.5 -- coherence results
- **TESTING-PRIORITY.yaml:** From 3.2 -- tier assignments and rationales
- **PLATFORM-GROUPS.yaml:** From 3.4 -- platform groupings
- **FILE-MANIFEST.yaml:** From 3.3 -- file verification status
- **COMBINATION-VALIDATION-REPORT.md:** From 1.4 -- combination statistics

## Process

```
STEP 1: LOAD ALL INPUTS
  - Read VARIANT-MATRIX.yaml (primary data source)
  - Read all supporting Layer 1-3 outputs for statistics
  - Aggregate all data needed for 7 required sections

STEP 2: WRITE SECTION 1 -- EXECUTIVE SUMMARY
  Contents:
    - Total testable variants in matrix (exact count)
    - Distribution by concept (C1: X, C2: X, C3: X)
    - Distribution by platform (Meta: X, TikTok: X, YouTube: X)
    - Distribution by testing priority (TIER_1: X, TIER_2: X, TIER_3: X)
    - Key coherence metrics:
      - Overall pass rate
      - Most common failure patterns
      - Average coherence score
    - Recommendation for first testing wave:
      - Which TIER_1 variants to launch first (5-10 specific variants)
      - Which platform to prioritize
      - Expected testing timeline

  Target length: 1-2 pages

STEP 3: WRITE SECTION 2 -- VARIANT MATRIX OVERVIEW
  Contents:
    - Full tabular display of ALL variants with columns:
      | Variant ID | Concept | Hook Type | Body Framework | CTA Type | Visual Type | Platform | Priority | Coherence |
    - Sorted by testing priority (TIER_1 first)
    - Every variant in the matrix must appear in this table
    - No abbreviation or "[continues...]" -- COMPLETE table

  CRITICAL: This section will be LARGE for 90-variant matrices.
  Use chunked writing if needed. Do NOT abbreviate.

STEP 4: WRITE SECTION 3 -- COHERENCE RESULTS SUMMARY
  Contents:
    - Overall coherence pass rate
    - Per-dimension pass rates:
      - Hook-Body: X%
      - Body-CTA: X%
      - Hook-Visual: X%
      - Full-Unit: X%
    - Most common failure patterns with counts (top 5)
    - Flagged variants requiring human review (if any):
      - List each with variant ID, flagged dimension, concern
    - Recommendations for improving coherence in future campaigns

  Source: COHERENCE-VALIDATION-REPORT.md

STEP 5: WRITE SECTION 4 -- TESTING PRIORITY RECOMMENDATIONS
  Contents:
    - TIER_1 variants listed with rationale for EACH:
      - Arena score for parent concept
      - Coherence score
      - Hook type performance data from A01
      - Platform priority from A03
      - Why this specific combination is prioritized
    - Recommended first-wave testing plan:
      - 5-10 variants to launch first
      - Platform and ad set recommendations
      - Budget allocation guidance
    - Recommended testing cadence:
      - How many variants per week
      - When to promote TIER_2 variants
      - Kill criteria for underperforming variants

  Source: TESTING-PRIORITY.yaml

STEP 6: WRITE SECTION 5 -- PLATFORM DEPLOYMENT GUIDE
  Contents:
    - For EACH platform:
      - Total variants assigned
      - Format requirements summary (aspect ratio, duration, file type, sound)
      - Variants ready for deployment (count)
      - Variants needing adaptation (count + what adaptations)
      - Platform-specific assembly notes
      - Directory location: AD-VARIANT-MATRIX/[PLATFORM]/
    - Cross-platform variant overlap analysis (same hook on different platforms)

  Source: PLATFORM-GROUPS.yaml, AD-VARIANT-MATRIX/ directories

STEP 7: WRITE SECTION 6 -- FILE MANIFEST SUMMARY
  Contents:
    - Total unique copy files referenced
    - Total unique asset files referenced
    - All files verified on disk: [Y/N]
    - Any missing files: [list or "none"]
    - File size summary (total size of all referenced files)
    - File organization notes

  Source: FILE-MANIFEST.yaml

STEP 8: WRITE SECTION 7 -- FLAGGED VARIANTS FOR HUMAN REVIEW
  Contents:
    - List of ALL FLAG variants (if any) with:
      - Variant ID
      - Which coherence dimension was flagged
      - Specific concern
      - Recommendation:
        (a) Promote to PASS (with reasoning)
        (b) Demote to FAIL (with reasoning)
        (c) Needs copy adjustment (specify what)
        (d) Needs visual swap (specify what)
    - If no flagged variants: "No flagged variants -- all variants in matrix passed full coherence validation."

  Source: FULL-UNIT-COHERENCE.yaml, COHERENCE-VALIDATION-REPORT.md

STEP 9: ADD METADATA HEADER
  At the top of the document:
    - Project name
    - Created timestamp
    - Total variants
    - Concepts
    - Platforms

STEP 10: VERIFY SIZE AND COMPLETENESS
  - Check file size >= 20KB
  - Verify all 7 sections are present and substantive
  - Verify no section is empty or placeholder
  - IF size < 20KB: expand sections with more detail (not filler)
```

```markdown
- Total Variants: [integer]
- Concepts: [integer]

## Section 1: Executive Summary
[1-2 pages of comprehensive summary with first-wave recommendations]

[COMPLETE table of ALL variants -- no abbreviation]

[pass rates, failure patterns, flagged variants, recommendations]

## Section 4: Testing Priority Recommendations

```
