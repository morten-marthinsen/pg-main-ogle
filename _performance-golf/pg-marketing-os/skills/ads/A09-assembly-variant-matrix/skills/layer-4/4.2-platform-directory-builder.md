# 4.2: Platform Directory Builder

## Purpose

Create per-platform directories under AD-VARIANT-MATRIX/ with platform-specific variant indexes and assembly instructions. Each platform directory should be self-contained: a user looking at AD-VARIANT-MATRIX/META/ should see everything needed for Meta deployment without consulting any other directory.

## Inputs

- **VARIANT-MATRIX.yaml:** From 4.1 -- complete variant database
- **PLATFORM-GROUPS.yaml:** From 3.4 -- platform groupings
- **FILE-MANIFEST.yaml:** From 3.3 -- file manifests with format compliance
- **PLATFORM-SPECS.yaml:** From 0.3 -- platform format requirements

## Process

```
STEP 1: CREATE DIRECTORY STRUCTURE
  Create: AD-VARIANT-MATRIX/ (root directory if not exists)
  For each target platform:
    Create: AD-VARIANT-MATRIX/[PLATFORM]/

  Example:
    AD-VARIANT-MATRIX/
      META/
      TIKTOK/
      YT/
      (additional platforms as needed)

STEP 2: FOR EACH PLATFORM DIRECTORY, CREATE INDEX FILE
  Create: AD-VARIANT-MATRIX/[PLATFORM]/VARIANT-INDEX.md

  Contents:
    - Platform name and deployment details
    - Total variants for this platform
    - Format requirements summary (aspect ratio, duration, file type, sound)
    - Variant table:
      | Variant ID | Concept | Hook Type | Priority | Status | Copy File | Asset File |
    - Sorted by testing priority (TIER_1 first)

STEP 3: FOR EACH PLATFORM DIRECTORY, CREATE ASSEMBLY INSTRUCTIONS
  Create: AD-VARIANT-MATRIX/[PLATFORM]/ASSEMBLY-INSTRUCTIONS.md

  Contents:
    - Platform-specific assembly process:
      a. How to combine copy + asset for this platform
      b. Sound handling instructions (add captions? text overlay?)
      c. Format adaptation steps (crop? resize? convert?)
      d. Upload specifications (file naming for ad platform, character limits)
      e. A/B testing setup notes for this platform's ad manager

    - Per-variant assembly notes:
      For each variant assigned to this platform:
        - Variant ID
        - Assembly status (READY / NEEDS_ADAPTATION)
        - If NEEDS_ADAPTATION: specific steps to adapt
        - Copy file location
        - Asset file location
        - Any platform-specific modifications needed

STEP 4: FOR EACH PLATFORM, CREATE FORMAT CHECKLIST
  Create: AD-VARIANT-MATRIX/[PLATFORM]/FORMAT-CHECKLIST.md

  Contents:
    - Pre-upload verification checklist:
      [ ] Aspect ratio correct for this platform
      [ ] Duration within platform limits
      [ ] File type supported
      [ ] File size under platform maximum
      [ ] Sound handling appropriate (captions/text overlay if needed)
      [ ] Text overlay within platform rules (e.g., Meta 20% rule)
      [ ] Copy fits within character limits
      [ ] CTA button type matches CTA text

STEP 5: VERIFY DIRECTORY COMPLETENESS
  For each platform directory:
    - VARIANT-INDEX.md exists and lists all platform variants
    - ASSEMBLY-INSTRUCTIONS.md exists with per-variant instructions
    - FORMAT-CHECKLIST.md exists
    - Variant count in index matches variant count in PLATFORM-GROUPS.yaml
```

## Output Schema

```
AD-VARIANT-MATRIX/
  META/
    VARIANT-INDEX.md         # All Meta variants with details
    ASSEMBLY-INSTRUCTIONS.md # How to assemble and upload for Meta
    FORMAT-CHECKLIST.md      # Pre-upload verification checklist
  TIKTOK/
    VARIANT-INDEX.md
    ASSEMBLY-INSTRUCTIONS.md
    FORMAT-CHECKLIST.md
  YT/
    VARIANT-INDEX.md
    ASSEMBLY-INSTRUCTIONS.md
    FORMAT-CHECKLIST.md
  [additional platforms as needed]

Each VARIANT-INDEX.md contains:
  # [Platform] Variant Index
  ## Platform Specifications
  - Aspect ratios: [list]
  - Duration limits: [per placement]
  - Sound handling: [description]
  - File types: [list]

  ## Variants ([X] total)
  ### TIER 1 ([X] variants)
  | Variant ID | Concept | Hook Type | Visual Type | Status | Copy File | Asset File |
  |-----------|---------|-----------|-------------|--------|-----------|------------|
  [... all TIER_1 variants for this platform]

  ### TIER 2 ([X] variants)
  [... all TIER_2 variants]

  ### TIER 3 ([X] variants)
  [... all TIER_3 variants]

Each ASSEMBLY-INSTRUCTIONS.md contains:
  # [Platform] Assembly Instructions
  ## General Assembly Process
  [platform-specific steps]

  ## Sound Handling
  [sound-on/sound-off instructions]

  ## Format Adaptations
  [crop/resize/convert instructions]

  ## Per-Variant Instructions
  ### [Variant ID]
  - Status: [READY / NEEDS_ADAPTATION]
  - Copy: [file path]
  - Asset: [file path]
  - Adaptations needed: [list or "none"]
  [... for each variant]
```

## Output File

- **Path:** `[project]/A09-assembly-variant-matrix/layer-4-outputs/4.2-platform-directory-builder.md`
- **Minimum Size:** 2KB per platform
- **Primary outputs:** `[project]/A09-assembly-variant-matrix/AD-VARIANT-MATRIX/[PLATFORM]/` directories with index, instructions, and checklist files

## Quality Checks

- [ ] Directory created for every target platform
- [ ] Each directory has VARIANT-INDEX.md, ASSEMBLY-INSTRUCTIONS.md, FORMAT-CHECKLIST.md
- [ ] Variant count in each index matches platform group from 3.4
- [ ] Variants sorted by testing priority in each index (TIER_1 first)
- [ ] Assembly instructions are platform-specific (not generic)
- [ ] Sound handling instructions match platform defaults
- [ ] Format adaptation steps are specific to each variant needing adaptation
- [ ] Pre-upload checklist covers all platform-specific requirements
- [ ] Each directory is self-contained (usable without consulting other directories)
- [ ] No variant missing from any platform directory
