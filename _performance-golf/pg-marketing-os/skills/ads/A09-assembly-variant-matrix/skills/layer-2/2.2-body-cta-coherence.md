# 2.2: Body-CTA Coherence

## Purpose

For EVERY valid combination, evaluate whether the CTA logically follows from the body. A body that educates should CTA to "learn more," not "buy now." A body that presents proof should CTA to "try it," not "read more." The CTA must be the natural next action the viewer would want to take after consuming the body content.

## Inputs

- **RAW-COMBINATIONS.yaml:** From 1.1 -- full combination details (body summary, CTA text)
- **INCOMPATIBILITY-LOG.yaml:** From 1.2 -- to identify which combinations are VALID
- **PLATFORM-MAPPING.yaml:** From 1.3 -- to exclude BLOCKED combinations
- **COPY-CATALOG.yaml:** From 0.1 -- full body content and CTA text for evaluation

## Process

```
STEP 1: BUILD EVALUATION SET
  - Same as 2.1: include all VALID, FLAGGED, and NEEDS_ADAPTATION combinations
  - Load full body content and CTA text from COPY-CATALOG.yaml
  - Count total combinations to evaluate

STEP 2: FOR EACH COMBINATION, EVALUATE BODY-CTA COHERENCE
  Apply the body type to CTA compatibility rules:

  EDUCATIONAL / VALUE-FIRST BODY:
    - Compatible CTAs: "Learn more", "Watch free video", "Get the guide",
      "See the research", "Download the report"
    - Incompatible CTAs: "Buy now", "Order today" (too aggressive for educational content)
    - Reasoning: Educational bodies build curiosity and understanding,
      not buying desire. The next step is more learning, not purchasing.

  PROBLEM-AGITATION BODY:
    - Compatible CTAs: "Discover the solution", "See if you qualify",
      "Find out if you're at risk", "Take the quiz"
    - Incompatible CTAs: "Subscribe" (no solution implied),
      "Read more" (too passive for agitated state)
    - Reasoning: Problem-agitation creates urgency to solve. CTA must
      offer a path to solution.

  PROOF / TESTIMONIAL BODY:
    - Compatible CTAs: "Try it risk-free", "Join X customers",
      "Get the same results", "Start your transformation"
    - Incompatible CTAs: "Learn more" (they've already seen the proof -- next step is action)
    - Reasoning: Proof bodies build conviction. The viewer is past the
      education stage. CTA should invite action.

  OFFER / PRICE-FOCUSED BODY:
    - Compatible CTAs: "Claim your discount", "Order now", "Get started today",
      "Add to cart", "Save X%"
    - Incompatible CTAs: "Read more" (price demands action, not more reading),
      "Watch video" (price is already revealed)
    - Reasoning: Offer bodies present value and price. The viewer is at
      decision point. CTA must capture the decision.

  MECHANISM EXPLANATION BODY:
    - Compatible CTAs: "See how it works", "Watch the presentation",
      "Discover the method", "Learn the [mechanism name]"
    - Incompatible CTAs: "Limited time offer" (mechanism explanation != urgency),
      "Buy now" (mechanism creates understanding, not buying impulse)
    - Reasoning: Mechanism bodies create "aha" understanding. CTA should
      deepen that understanding or bridge to the full presentation.

  URGENCY / SCARCITY BODY:
    - Compatible CTAs: "Claim before midnight", "Only X left",
      "Reserve your spot", "Lock in your price"
    - Incompatible CTAs: "Learn more" (urgency demands immediate action,
      not more contemplation)
    - Reasoning: Urgency bodies create time pressure. CTA must capture
      the urgency with an immediate action.

STEP 3: ASSIGN VERDICT
  For each combination:
    - PASS: CTA is the natural next action after consuming the body content.
      The viewer would instinctively want to take this action.
    - FAIL: CTA is disconnected from the body's intent. The viewer would
      be confused by this ask after consuming the body content.
      Record: what body type is, what CTA asks for, why the disconnect.
    - FLAG: CTA is somewhat related but not the optimal match.
      Record: concern and what CTA would be better aligned.

  Each verdict MUST include:
    - The body type classification
    - What the body establishes (specific text/summary)
    - What the CTA asks the viewer to do
    - Why the CTA does/doesn't follow naturally from the body

STEP 4: DO NOT BATCH-EVALUATE
  CRITICAL: Each combination gets INDIVIDUAL evaluation.
  No batch verdicts. No "these all have matching CTAs."

STEP 5: COMPUTE COHERENCE STATISTICS
  - Total evaluated
  - Total PASS / FAIL / FLAG
  - Pass rate (%)
  - Most common failure pattern (which body type + CTA type combinations fail most)
  - CTA type distribution across verdicts

STEP 6: WRITE BODY-CTA-COHERENCE.yaml
```
