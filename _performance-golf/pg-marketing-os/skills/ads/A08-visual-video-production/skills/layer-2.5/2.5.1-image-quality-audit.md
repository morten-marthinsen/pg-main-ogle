# 2.5.1: Image Quality Audit

## Purpose
Review every generated image against its A05 brief section. Evaluate on three dimensions: Brief Fidelity (40%), Technical Compliance (30%), and Production Quality (30%). Make a binary PASS/FAIL determination per image. Failed images enter the regeneration queue.

## Inputs
- All generated images from Layer 2.1
- A05 Visual Direction Package (the source of truth for what each image should look like)
- Platform-Specific Technical Requirements (resolution, aspect ratio, file format, file size)
- Quality Review Criteria from AGENT.md

## Process
```
FOR EACH generated image:

DIMENSION 1: BRIEF FIDELITY (Weight: 40%)
  Does the image match what the visual brief specified?
  - Subject/content match: [1-10]
    Does the image contain the correct subject? Correct action? Correct context?
  - Style/aesthetic match: [1-10]
    Does the style match? (photorealistic vs. cinematic vs. editorial vs. UGC)
  - Mood/atmosphere match: [1-10]
    Does the mood match? (warm vs. cool, inviting vs. urgent, calm vs. dramatic)
  Minimum: 7.0 average across three sub-dimensions

DIMENSION 2: TECHNICAL COMPLIANCE (Weight: 30%)
  Does the image meet platform specifications?
  - Resolution: [PASS/FAIL -- binary, must meet minimum]
  - Aspect ratio: [PASS/FAIL -- binary, must be exact]
  - File format: [PASS/FAIL -- binary]
  - File size: [PASS/FAIL -- under platform maximum]
  Minimum: ALL PASS (any FAIL = overall FAIL)

DIMENSION 3: PRODUCTION QUALITY (Weight: 30%)
  Is the image production-ready?
  - No AI artifacts (extra fingers, blurred text, warped objects): [PASS/FAIL]
  - Visual coherence (no random elements, consistent scene): [1-10]
  - Natural appearance (doesn't scream "AI-generated"): [1-10]
  Minimum: 6.0 average, AND no FAIL on artifacts check

OVERALL VERDICT:
  ALL three dimensions pass their minimums = PASS
  ANY dimension fails = FAIL -> regeneration queue
```

- **Path:** `layer-2.5-outputs/2.5.1-image-quality-audit.md`

## Quality Checks
- EVERY generated image was reviewed (none skipped)
- Scoring is honest (not inflated to reduce regeneration)
- Beautiful images that don't match the brief = FAIL
- Adequate images that match the brief = PASS
- Brief fidelity is measured against A05 (not against "what looks good")
- Technical compliance is binary (no "close enough")
- AI artifacts are caught (extra fingers, warped text, etc.)
- Every FAIL has specific, actionable regeneration guidance

## Dependencies
- Layer 2.1 (Image Generation) must be complete
- A05 Visual Direction must be loaded
- Platform specs must be referenced
