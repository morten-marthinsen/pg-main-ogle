# 2.5.3: Video Quality Audit

## Purpose
Review every generated video against its A05 visual brief. Evaluate talking head videos on lip sync, emotional delivery, and brief match. Evaluate B-roll on scene fidelity, motion quality, and consistency. Make a binary PASS/FAIL determination per video.

## Inputs
- All generated videos from Layer 2.4
- A05 Visual Direction Package (shot lists, actor descriptions, scene descriptions)
- A07 AD-COPY-FINAL/ (scripts for talking head verification)
- Platform-Specific Technical Requirements (resolution, duration, format)

## Process
```
FOR EACH generated video:

DETERMINE VIDEO TYPE AND APPLY TYPE-SPECIFIC CRITERIA:

TYPE A: TALKING HEAD (UGC/Arcads)
  BRIEF FIDELITY (40%):
    - Actor match (age, gender, appearance): [1-10]
    - Background match: [1-10]
    - Camera angle/framing match: [1-10]
  TYPE-SPECIFIC (added to production quality):
    - Lip sync accuracy: [1-10]
    - Emotional delivery: [1-10]
    - Natural gestures: [1-10]

TYPE B: B-ROLL / SCENE
  BRIEF FIDELITY (40%):
    - Scene content match: [1-10]
    - Camera movement match: [1-10]
    - Style match (cinematic, documentary, raw): [1-10]
  TYPE-SPECIFIC (added to production quality):
    - Motion quality (no jitter, no morphing): [1-10]
    - Visual consistency (no random elements appearing/disappearing): [1-10]

TYPE C: PRODUCT DEMO
  BRIEF FIDELITY (40%):
    - Product accuracy: [1-10]
    - Demo flow match to brief: [1-10]
    - Style/branding match: [1-10]

UNIVERSAL DIMENSIONS:
  TECHNICAL COMPLIANCE (30%):
    - Resolution: [PASS/FAIL]
    - Aspect ratio: [PASS/FAIL]
    - Duration match (+/- 5%): [PASS/FAIL]
    - File format: [PASS/FAIL]
    - File size: [PASS/FAIL]

  PRODUCTION QUALITY (30%):
    - No AI artifacts (morphing faces, flickering, temporal inconsistency): [PASS/FAIL]
    - Visual coherence: [1-10]
    - Natural appearance: [1-10]

OVERALL VERDICT:
  ALL dimensions pass = PASS
  ANY dimension fails = FAIL -> regeneration queue
```
