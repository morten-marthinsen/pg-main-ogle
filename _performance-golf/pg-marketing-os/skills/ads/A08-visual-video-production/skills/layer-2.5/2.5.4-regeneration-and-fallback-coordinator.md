# 2.5.4: Regeneration and Fallback Coordinator

## Purpose
For any asset marked FAIL in quality audits (2.5.1, 2.5.2, 2.5.3): orchestrate regeneration with refined prompts targeting the specific failure. After 3 total attempts per asset, produce a comprehensive human production brief. Track all regeneration attempts and maintain the acceptance rate threshold.

## Inputs
- 2.5.1 Image Quality Audit (failed images with regeneration guidance)
- 2.5.2 Voiceover Quality Audit (failed voiceovers with regeneration guidance)
- 2.5.3 Video Quality Audit (failed videos with regeneration guidance)
- Original prompts/parameters from Layer 2 generation logs
- Tool availability from 0.5 (for fallback tool selection)
- 1.2 Tool Assignment (fallback tool hierarchy)

## Process
```
FOR EACH asset marked FAIL in quality audits:

CHECK ATTEMPT COUNT:
  Attempt 1 was the original generation (Layer 2)
  Current attempt is 2 or 3

ATTEMPT 2:
  1. ANALYZE failure reason from quality audit:
     - Which dimension failed? (brief fidelity, technical, production quality)
     - Which specific sub-score was low?
  2. REFINE prompt/parameters targeting the failure:
     - Brief fidelity failure -> more specific prompt language, add details
     - Technical compliance failure -> adjust generation parameters
     - Production quality failure -> add negative prompts, adjust quality settings
  3. REGENERATE with refined approach using SAME tool
  4. RE-REVIEW against same quality criteria as original audit
  5. Log refined prompt, parameters, and results

ATTEMPT 3:
  1. If still failing, try DIFFERENT TOOL (fallback from Layer 1.2 hierarchy)
  2. If no fallback tool available, try fundamentally different prompt approach
  3. REGENERATE
  4. RE-REVIEW
  5. Log results

AFTER 3 TOTAL ATTEMPTS (all failed):
  1. CREATE Human Production Brief:
     - Asset ID and type
     - Visual brief requirements (from A05)
     - Copy requirements (from A07, if applicable)
     - Platform specifications
     - All 3 prompt attempts with exact text
     - Quality scores per attempt
     - Specific failure points per attempt
     - Recommendation for human production approach
     - Reference images/examples from A05
  2. SAVE to AD-ASSETS/fallback-briefs/[asset-id]-human-brief.md
  3. MARK asset as FALLBACK in manifest
  4. CONTINUE with other assets -- DO NOT block pipeline

TRACK OVERALL ACCEPTANCE RATE:
  acceptance_rate = (total PASS) / (total attempted) * 100
  IF acceptance_rate < 70%:
    -> HALT. Escalate to human.
    -> Tool landscape may be insufficient for this campaign.
```

## Output File
- **Path:** `layer-2.5-outputs/2.5.4-regeneration-log.md`

## Quality Checks
- Every FAIL asset had regeneration attempted (none silently dropped)
- Maximum 3 total attempts per asset (not infinite loops)
- Attempt 2 targets the specific failure dimension (not generic retry)
- Attempt 3 uses a different tool or fundamentally different approach
- Fallback briefs are comprehensive (all specs, all attempts, recommendations)
- Acceptance rate is accurately calculated
- If < 70%, HALT is triggered (not pushed through)

## Dependencies
- 2.5.1 Image Quality Audit must be complete
- 2.5.2 Voiceover Quality Audit must be complete
- 2.5.3 Video Quality Audit must be complete
- Layer 2 generation outputs must exist
- Tool availability from 0.5 (for fallback tool options)
