# 2.4: Video Generation Orchestrator

## Purpose
Compose video prompts and call video tools (ElevenLabs Platform, Arcads, Creatify, Runway) for all video assets. Handles three distinct video types: talking head (UGC), B-roll/scene, and product demo. Generate per production schedule Phases 1-2.

## Inputs
- 1.5 Production Schedule (which videos to generate, in dependency order)
- 1.2 Tool Assignment (which tool per video asset)
- 1.4 Dependency Sequence (voiceover duration for timing, lip-sync requirements)
- A05 Visual Direction (shot lists, camera direction, actor descriptions, settings)
- A07 AD-COPY-FINAL/ (scripts for talking head lip-sync)
- Phase 1 voiceover outputs (for timing reference)

## Process
```
FOR EACH video asset in production schedule:

DETERMINE VIDEO TYPE:
  A. TALKING HEAD (AI UGC Actor) -- use Arcads
  B. B-ROLL / SCENE -- use ElevenLabs Platform or Runway
  C. PRODUCT DEMO -- use Creatify or manual composition

TYPE A: TALKING HEAD (Arcads)
  STEP 1: Extract from brief:
    - Actor description (age, gender, ethnicity, style from A05)
    - Script text (from A07 -- verbatim)
    - Emotional direction (enthusiasm level, tone shifts)
    - Background setting (from A05 visual brief)
    - Wardrobe direction (from A05)
    - Camera angle and framing (from A05 shot list)
  STEP 2: Call Arcads API:
    - Select actor matching description
    - Input script with emotion markers
    - Set background, camera angle
    - Generate video
  STEP 3: Evaluate:
    - Lip sync accuracy: [1-10]
    - Emotional delivery: [1-10]
    - Natural gestures: [1-10]
    - Brief match (actor, background, framing): [1-10]
    - Technical quality: [1-10]

TYPE B: B-ROLL / SCENE
  STEP 1: Extract from brief:
    - Scene description (from A05 shot list -- specific shot)
    - Camera movement (static, pan, dolly, tracking)
    - Duration (from script timing)
    - Style (cinematic, documentary, raw)
  STEP 2: Compose video prompt:
    - Lead with scene description
    - Specify camera movement, duration, style
    - Include negative prompts (avoid morphing artifacts)
  STEP 3: Call tool:
    - ElevenLabs Platform: Select best model for scene type
    - Runway Gen-4: For scenes requiring fine control
    - Generate 2-4 variations
  STEP 4: Evaluate per variation:
    - Scene match to brief: [1-10]
    - Motion quality (no jitter, no morphing): [1-10]
    - Visual consistency: [1-10]
    - Duration match: [PASS/FAIL]
    - Technical quality: [1-10]

TYPE C: PRODUCT DEMO (Creatify)
  STEP 1: Input product URL + brief requirements
  STEP 2: Generate demo creative
  STEP 3: Evaluate against brief
  STEP 4: Select best or flag for human production

REGENERATION PROTOCOL (ALL TYPES):
  IF best score < 6.0: Regenerate with refined prompt/parameters
  Maximum 3 regeneration attempts per asset
  IF 3 attempts all < 6.0:
    -> Create HUMAN PRODUCTION BRIEF
    -> Include: original brief specs, failed prompt history, quality notes
    -> Save to fallback-briefs/ directory
    -> DO NOT hold up production of other assets
```

## Quality Checks
- Video type correctly identified per asset (talking head vs. B-roll vs. demo)
- Correct tool used per video type and 1.2 assignment
- Talking head actors match A05 demographic description
- Script text for talking head is verbatim from A07
- B-roll scenes match A05 shot list (not "inspired by")
- Duration matches voiceover timing (dependency from Phase 1)
- No AI morphing/jitter artifacts in selected variations
- Regeneration protocol followed for failures (max 3, then human brief)

## Dependencies
- Layer 1 must be complete (GATE_1_COMPLETE.yaml exists)
- 1.5 Production Schedule must exist
- 1.4 Dependency Sequence must exist (for timing dependencies)
- Phase 1 voiceover outputs must exist (for lip-sync and timing)
- Video generation tools must be available (from 0.5)
