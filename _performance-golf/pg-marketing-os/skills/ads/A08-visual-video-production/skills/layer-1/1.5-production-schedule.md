# 1.5: Production Schedule

## Purpose
Create the final production execution order: which assets are generated first (no dependencies), which second (depends on first batch), etc. Group parallel-safe assets for batch generation. This is the actionable production plan that Layer 2 follows sequentially.

## Inputs
- 1.1 Asset Decomposition (asset inventory)
- 1.2 Tool Assignment (which tool produces each asset)
- 1.3 Platform Variant Expansion (expanded asset list)
- 1.4 Dependency Sequencing (dependency graph with phases)

## Process
```
STEP 1: ORGANIZE BY PHASE
  - Group all Phase 1 assets (independent -- can run in parallel)
  - Group all Phase 2 assets (first-order dependencies)
  - Group all Phase 3 assets (second-order dependencies)

STEP 2: WITHIN EACH PHASE, BATCH BY TOOL
  - Group assets by assigned tool (all Midjourney images together, all ElevenLabs VOs together)
  - This enables batch generation (call the tool once with multiple prompts)
  - Mark batch boundaries

STEP 3: ORDER WITHIN BATCHES
  - Within each tool batch, order by concept (C01 first, then C02, etc.)
  - This keeps related assets together for quality consistency

STEP 4: CREATE PRODUCTION SCHEDULE
  Phase 1, Batch 1: [tool] -- [list of assets]
  Phase 1, Batch 2: [tool] -- [list of assets]
  ...
  Phase 2, Batch 1: [tool] -- [list of assets]
  ...
  Phase 3, Batch 1: [tool] -- [list of assets]
  ...

STEP 5: ESTIMATE TIMELINE
  - Total Phase 1 time (with parallelization)
  - Total Phase 2 time (sequential after Phase 1)
  - Total Phase 3 time (sequential after Phase 2)
  - Total estimated production time

STEP 6: IDENTIFY BOTTLENECKS
  - Which tool has the most assets? (potential bottleneck)
  - Which phase has the longest critical path?
  - Any single-tool dependencies (only one tool can produce this asset type)?
```
