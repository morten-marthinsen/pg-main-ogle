# 0.0.1: Vertical Profile Loader

## Purpose
Load ad-specific vertical configuration from `skills/ads/ad-verticals/` to establish production constraints, anti-slop rules, and any vertical-specific tool preferences or brand guidelines that affect asset production.

## Inputs
- Project brief with vertical identifier (health/golf/finance/personal-dev/technology)
- Vertical profile file from `skills/ads/ad-verticals/[vertical].md`

## Process
```
STEP 1: IDENTIFY VERTICAL
  - Extract vertical identifier from project brief or upstream packages
  - IF no vertical specified: log warning "No vertical specified -- using universal configuration"

STEP 2: LOAD VERTICAL PROFILE
  - READ vertical profile file: skills/ads/ad-verticals/[vertical].md
  - EXTRACT:
    - Production constraints (e.g., health: no before/after images that violate FTC guidelines)
    - Anti-slop rules specific to this vertical
    - Brand/compliance constraints
    - Preferred visual styles for this vertical
    - Any tool-specific overrides

STEP 3: VALIDATE LOAD
  - Confirm vertical file was actually read (not synthesized from memory)
  - Quote specific lines from loaded file as proof
  - Log any vertical-specific constraints that will affect Layer 1-3 execution

STEP 4: WRITE OUTPUT
  - Create per-microskill output file with loaded configuration
```
