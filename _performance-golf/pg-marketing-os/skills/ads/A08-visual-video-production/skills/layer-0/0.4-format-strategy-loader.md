# 0.4: Format Strategy Loader

## Purpose
Load A03 FORMAT-STRATEGY.md containing platform targets, aspect ratios, length constraints, and format decisions per concept. This determines which platforms each concept targets and the technical specifications every asset must meet.

## Inputs
- A03 FORMAT-STRATEGY.md (from upstream A03 -- Format Strategy skill)

## Process
```
STEP 1: LOCATE FORMAT STRATEGY
  - Search project directory for FORMAT-STRATEGY.md
  - IF not found: HALT -- cannot proceed without platform/format specifications
  - Log file path and size

STEP 2: LOAD AND PARSE
  - READ the full format strategy document
  - EXTRACT:
    - Target platforms per concept (Meta feed, Meta Stories, TikTok, YouTube, Google)
    - Aspect ratios per platform (1:1, 4:5, 9:16, 16:9)
    - Video duration constraints per platform
    - Ad format types per concept (static, video, UGC, carousel)
    - Any platform-specific constraints (TikTok bottom safe zone, etc.)
    - Budget allocation hints (which platforms are primary vs. secondary)

STEP 3: BUILD PLATFORM-CONCEPT MATRIX
  - Create a matrix showing which concept targets which platform
  - Identify total number of platform renders needed
  - Flag any platform-specific technical requirements

STEP 4: CROSS-REFERENCE WITH PLATFORM SPEC TABLE
  - Verify A03 format decisions align with current platform specs
  - Reference the Platform-Specific Technical Requirements table in AGENT.md
  - Log any discrepancies

STEP 5: WRITE OUTPUT
  - Create per-microskill output file with format strategy summary
```
