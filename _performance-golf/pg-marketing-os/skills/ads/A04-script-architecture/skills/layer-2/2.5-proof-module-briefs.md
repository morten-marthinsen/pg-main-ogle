# 2.5: Proof Module Briefs

## Purpose
Design the [PROOF] module content brief for each ad concept. Select proof type (testimonial, data stat, demonstration, authority citation, social proof number). Specify which proof element from Proof Inventory to use. Design for proof type that best serves hook's angle.

## Inputs
- Module Sequence Design (from 2.1)
- Hook types (from 0.2 - to match proof to hook)
- Proof Inventory (Skill 02 output if available)

## Process
Match proof type to hook type and framework. Select specific proof element. Design integration approach per module word budget.

- **Path:** `layer-2-outputs/2.5-proof-module-briefs.md`
- **Minimum size:** 5KB
