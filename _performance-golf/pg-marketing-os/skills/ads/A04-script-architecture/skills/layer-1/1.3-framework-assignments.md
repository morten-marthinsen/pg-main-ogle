# 1.3: Framework Assignments

## Purpose
Select the winning framework per ad concept based on the analysis matrix. Document: selected framework, rationale (citing all 5 variables), alternative frameworks considered, why alternatives were rejected.

## Inputs
- Framework Analysis Matrix (from 1.2)

## Process

### Step 1: Load Scored Framework Matrix
```
READ: layer-1-outputs/1.2-framework-analysis-matrix.md
FOR EACH concept: EXTRACT ranked_frameworks list
```

### Step 2: Select Winning Framework Per Concept
```
FOR EACH concept in analysis:
  DEFAULT: Select highest-scoring framework (rank 1)
  
  EXCEPTION CHECKS:
    IF highest score < 6.0:
      FLAG: "No strong framework match -- manual review recommended"
      SELECT: Second-highest if >= 6.0, else default to Hook-Body-CTA
    
    IF top 2 frameworks tied within 0.5 points:
      TIEBREAKER: Prefer framework more common in this vertical
    
    IF ad_length == 6s:
      OVERRIDE: Use Bumper structure (not a full framework)
      DOCUMENT: "6-second bumper -- single-idea structure"
  
  DOCUMENT selection:
    selected_framework: string
    selection_rationale: string (MUST cite all 5 variables)
    composite_score: float
    alternatives_considered: [list of next 2-3 highest scoring]
    why_alternatives_rejected: string
```

### Step 3: Verify Framework-to-5-Variable Alignment
```
FOR EACH assignment:
  VERIFY rationale explicitly cites:
    ✓ Hook type compatibility
    ✓ Platform compatibility
    ✓ Length compatibility
    ✓ Vertical preference
    ✓ Awareness level match
  
  IF any variable not cited in rationale:
    FLAG: "Incomplete rationale -- missing [variable] justification"
```

### Step 4: Check for Framework Diversity
```
AGGREGATE framework assignments across all concepts
COUNT assignments per framework
IF any single framework assigned to 80%+ of concepts:
  FLAG: "Low framework diversity -- verify 5-variable analysis was performed"
  RECOMMEND: Re-analyze concepts with same framework assignment
```

### Step 5: Incompatibility Detection
```
FOR EACH assignment:
  CHECK length-incompatibility:
    IF framework == "Edutainment" AND ad_length < 60s: REJECT
    IF framework == "Story Narrative" AND ad_length < 60s: REJECT
    IF framework == "Fast-Paced Viral" AND ad_length > 30s: REJECT
  
  CHECK platform-incompatibility:
    IF framework == "Story Narrative" AND platform == "TikTok" AND length < 60s: REJECT
    IF framework == "Edutainment" AND platform == "TikTok": FLAG (unusual but not impossible)
  
  IF REJECT triggered:
    REASSIGN to next-highest compatible framework
    DOCUMENT: "Original selection rejected due to [incompatibility]"
```

## Output Schema
```yaml
framework_assignments:
  total_concepts: integer
  assignments:
    - concept_id: string
      selected_framework: string
      selection_rationale: string
      composite_score: float
      five_variable_citation:
        hook_type: boolean (cited?)
        platform: boolean
        length: boolean
        vertical: boolean
        awareness: boolean
      alternatives_considered: [list]
      alternatives_rejected_because: string
      incompatibility_check: PASS | REASSIGNED
  framework_distribution:
    PAS: integer
    AIDA: integer
    BAB: integer
    Hook_Body_CTA: integer
    Story_Narrative: integer
    Edutainment: integer
    UGC_DR: integer
    Fast_Paced_Viral: integer
    Bumper: integer
  diversity_check: PASS | WARN
  quality_flags: [list or empty]
```

## Output File
- **Path:** `layer-1-outputs/1.3-framework-assignments.md`
- **Minimum size:** 3KB

## Quality Checks
1. Every concept has a framework assignment
2. Every assignment cites all 5 selection variables in rationale
3. No length-incompatible assignments (Edutainment for 15s)
4. No platform-incompatible assignments
5. Framework distribution shows reasonable diversity (no single framework > 80%)

## Dependencies
- 1.2 (Framework Analysis Matrix) must be complete
