# 1.1: Ad Concept Inventory

## Purpose
Build the complete inventory of ad concepts requiring script architectures. Each concept = selected hook + assigned format + assigned platform + target length. Cross-reference A02 hooks with A03 format assignments.

## Inputs
- Hook-Angle Matrix (from 0.2)
- Format Strategy (from 0.3)

## Process

### Step 1: Load Hook Inventory
```
READ: layer-0-outputs/0.2-hook-angle-matrix-loader.md
EXTRACT: all selected hooks with hook_ids
COUNT: total_selected_hooks
```

### Step 2: Load Format Assignments
```
READ: layer-0-outputs/0.3-format-strategy-loader.md
EXTRACT: all ad_concepts with format/platform/length assignments
COUNT: total_ad_concepts
```

### Step 3: Build Comprehensive Ad Concept Inventory
```
FOR EACH ad_concept in format_strategy:
  CREATE concept_record:
    concept_id: "[hook_id]-[platform]-[length]"
    hook_id: string (reference to A02 hook)
    hook_text: string (verbatim hook copy)
    hook_type: string (type code from taxonomy)
    angle_id: string (which angle)
    format_type: string (video | static | carousel)
    platform_primary: string
    target_length: string (6s | 15s | 30s | 60s | 90s | 2min | 3min)
    aspect_ratio: string
    sound_assumption: string
    framework_assignment: null (to be populated in 1.3)
```

### Step 4: Organize by Concept Groups
```
GROUP concepts by angle_id (concepts sharing same angle)
SORT groups by hook_count (largest angle groups first)
IDENTIFY:
  - Primary angle group (most concepts)
  - Secondary angle groups
  - Single-hook angles (if any)
```

### Step 5: Generate Inventory Summary
```
CALCULATE:
  - Total concepts
  - Concepts per platform
  - Concepts per length
  - Concepts per angle
  - Average concepts per hook
```

## Output Schema
```yaml
ad_concept_inventory:
  total_concepts: integer
  concepts:
    - concept_id: string
      hook_id: string
      hook_text: string
      hook_type: string
      angle_id: string
      format_type: string
      platform_primary: string
      target_length: string
      aspect_ratio: string
      sound_assumption: string
      framework_assignment: null
  concept_groups:
    - angle_id: string
      concept_count: integer
      concept_ids: [list]
  summary:
    concepts_by_platform: object
    concepts_by_length: object
    concepts_by_angle: object
```

## Output File
- **Path:** `layer-1-outputs/1.1-ad-concept-inventory.md`
- **Minimum size:** 3KB

## Quality Checks
1. Every selected hook from A02 has at least one ad concept
2. Every ad concept has all required fields populated (no nulls except framework_assignment)
3. Concept_ids are unique (no duplicates)
4. Summary totals match detail counts
5. All hook_ids cross-reference successfully to A02 hooks

## Dependencies
- 0.2 (Hook-Angle Matrix Loader) must be complete
- 0.3 (Format Strategy Loader) must be complete
