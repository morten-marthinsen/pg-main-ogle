# 0.0.1: Vertical Profile Loader

## Purpose
Load ad-specific vertical configuration from `ad-verticals/` directory. Extract vertical-specific script structures, dominant formats, compliance constraints, and anti-slop rules that will inform all framework selection and module design decisions.

## Inputs
- Project brief or campaign configuration indicating the vertical
- `./skills/ad-verticals/[vertical].md` (one of: golf, health, finance, personal-dev, technology)

## Process

### Step 1: Identify Vertical
```
READ project brief or campaign configuration
EXTRACT vertical field
VERIFY vertical is one of: golf | health | finance | personal-dev | technology
```

### Step 2: Load Vertical Profile
```
LOCATE file: ./skills/ad-verticals/[vertical].md
READ entire file
EXTRACT:
  - Dominant frameworks for this vertical
  - Script structure patterns
  - Compliance constraints (if any)
  - Platform preferences
  - Ad length recommendations
  - Anti-slop patterns (vertical-specific words/phrases to avoid)
  - UGC usage patterns
  - Mechanism integration requirements
```

### Step 3: Extract Key Directives
```
IDENTIFY:
  - Which frameworks to prioritize (e.g., health prioritizes AIDA, UGC-DR)
  - Which frameworks to avoid (e.g., finance avoids urgency-heavy patterns)
  - Compliance language requirements
  - Visual treatment defaults
  - CTA style constraints
```

### Step 4: Compile Vertical Context
```
CREATE vertical_context object:
  vertical_id: [vertical name]
  dominant_frameworks: [list with percentages]
  recommended_patterns: [named pattern structures]
  compliance_notes: [list of requirements or "none"]
  anti_slop_additions: [vertical-specific forbidden phrases]
  mechanism_importance: [critical | important | optional]
  typical_lengths: [list of recommended ad lengths]
  platform_affinity: [which platforms work best]
```

## Output Schema
```yaml
vertical_profile:
  vertical_id: string
  dominant_frameworks:
    - framework: string
      usage_frequency: float
      recommended_for: [list]
  script_patterns:
    - pattern_name: string
      structure: string
      when_to_use: string
  compliance_constraints:
    - type: string
      requirement: string
      platforms_affected: [list]
  anti_slop_additions:
    - phrase: string
      why_forbidden: string
  mechanism_integration:
    importance: enum[critical, important, optional]
    minimum_length: string
    integration_notes: string
  platform_recommendations:
    - platform: string
      affinity: enum[high, medium, low]
      notes: string
  loading_status:
    profile_loaded: boolean
    warnings: [list or empty]
```

## Output File
- **Path:** `layer-0-outputs/0.0.1-vertical-profile-loader.md`
- **Minimum size:** 1KB

## Quality Checks
1. Vertical profile file exists and was successfully read
2. All required fields extracted and populated
3. Dominant frameworks list contains at least 2 framework types
4. If vertical has compliance constraints, they are documented with platform scope
5. Anti-slop additions are specific to this vertical (not generic AI-slop)

## Dependencies
- Project brief exists with vertical field populated
- Vertical profile file exists in `skills/ad-verticals/`
- No upstream skill dependencies (this runs first in Layer 0)
