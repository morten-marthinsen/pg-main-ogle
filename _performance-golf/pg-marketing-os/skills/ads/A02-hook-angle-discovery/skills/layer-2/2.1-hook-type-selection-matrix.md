# 2.1: Hook Type Selection Matrix

## Purpose

For each approved angle from Layer 1, identify which of the 32 hook types are natural fits. Create a matrix mapping angles to compatible hook types to guide targeted hook generation across all angles.

## Inputs

- 1.6-ranked-angles.md (10-15 approved angles)
- 0.3-hook-taxonomy.md (all 32 types loaded)
- 0.0.1-vertical-profile.md (vertical-specific type preferences)
- Campaign Brief (target platforms, awareness level)

## Process

1. **For each angle, determine compatible hook types**
   Consider:
   - **Angle category:** Pain angles → Category C types; Mechanism → A/B; Identity → D/E; etc.
   - **Vertical patterns:** Health niche → "weird food" works (A6); Golf → distance promise works; etc.
   - **Target platform:** TikTok → F2/F3/G types; YouTube → A/B types; Meta → all types
   - **Awareness level:** Unaware → pattern interrupt/curiosity; Problem-aware → pain/solution; etc.

2. **Map each angle to 4-6 compatible types**
   - Each angle should map to at least 4 types
   - Types should span multiple categories (not all Category A)

3. **Ensure minimum type diversity across ALL angles**
   - At least 8 of 32 types should be targeted across the full matrix
   - No monoculture (all hooks being variations of 2-3 types)

4. **Create matrix visualization**
   - Rows = Angles
   - Columns = Hook type categories (A-J)
   - Cells = Specific types selected

5. **Document rationale for type selections**
   - Why does Angle X map to types A2, C1, F1?
   - What makes these types natural fits?
