# 3.2: Human Selection Capture (BLOCKING GATE)

## Purpose
Capture human hook selections, modifications, and notes. This microskill CANNOT execute without human input — it creates the BLOCKING gate before Layer 4.

## Inputs
- 3.1-presentation.md
- HUMAN INPUT (selections)

## Process
1. Present hooks to human (via 3.1 presentation)
2. Wait for selections
3. Capture selections in HUMAN_SELECTION.yaml
4. Document modifications
5. Note any new hooks added
6. Record rejection reasons

## Output Schema
HUMAN_SELECTION.yaml format:
```yaml
human_selection:
  skill: "A02-hook-angle-discovery"
  timestamp: "[ISO]"
  total_presented: [count]
  total_selected: [count] # must be >= 8

  selected_hooks:
    - hook_id: "[original ID]"
      original_text: "[original]"
      modified_text: "[if modified]"
      was_modified: [true/false]
      human_notes: "[notes]"
      angle: "[source angle]"
      type: "[hook type]"

  new_hooks_added:
    - text: "[new hook]"
      angle: "[angle or 'new']"
      type: "[type or 'unclassified']"
      notes: "[context]"

  rejected_hooks:
    - hook_id: "[ID]"
      reason: "[why rejected]"

  general_notes: "[overarching direction]"
```

## Output File
- **Path:** `layer-3-outputs/3.2-human-selection.md` (markdown documentation)
- **YAML Path:** `HUMAN_SELECTION.yaml` (structured data)
- **Minimum size:** 2KB

## Quality Checks
- Minimum 8 hooks selected
- HUMAN_SELECTION.yaml created
- All modifications documented

## Dependencies
- 3.1-presentation.md
- **HUMAN INPUT (BLOCKING)**

## Gate 3 Blocking
**THIS IS A BLOCKING GATE. Layer 4 cannot proceed without HUMAN_SELECTION.yaml.**
