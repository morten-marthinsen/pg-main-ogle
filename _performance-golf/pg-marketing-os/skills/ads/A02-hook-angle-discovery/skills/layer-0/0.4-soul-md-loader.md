# 0.4: Soul.md Loader

## Purpose

Load project Soul.md (if exists) to extract voice constraints for hook language. Hooks must match the campaign's voice register, energy signature, and avoid anti-voice patterns.

## Inputs

- `[project]/SOUL.md` (optional but recommended)

## Process

1. **Check for Soul.md existence**
   - Path: `[project]/SOUL.md`
   - If doesn't exist → log warning, proceed without voice constraints
   - If exists → load and extract

2. **Extract voice constraints**
   - Voice register: [conversational/authoritative/scientific/playful/etc.]
   - Energy signature: [urgent/calm/excited/measured/etc.]
   - Emotional ceiling: [maximum emotional intensity allowed]
   - Emotional floor: [minimum emotional intensity required]
   - Anti-voice patterns: [language to AVOID]
   - Pacing signature: [short punchy / flowing / varied / etc.]

3. **Extract specimens (if finalized)**
   - Positive specimens: [examples of voice done right]
   - Anti-specimens: [examples of voice done wrong]

4. **Translate to hook constraints**
   - Which hook types naturally fit this voice?
   - Which hook types would violate voice constraints?
   - What word choices align with voice register?
   - What word choices violate anti-voice patterns?

## Output Schema

```markdown
# 0.4: Soul.md Loader
## Execution Context
- Soul.md status: [exists / missing]
- Voice constraints loaded: [Y/N]
- Loaded: [timestamp]

## Voice Constraints for Hooks

### Voice Register
[Description of voice register — how hooks should sound]

### Energy Signature
[Energy level hooks should carry]

### Emotional Range
- Ceiling: [maximum intensity]
- Floor: [minimum intensity]
- Sweet spot: [target range]

### Anti-Voice Patterns (FORBIDDEN)
[Specific language patterns hooks must AVOID]

### Pacing Signature
[How hook text should flow — punchy vs. flowing]

## Hook Type Fit

### Natural Fit Types
[Hook types that align with this voice — prioritize these]

### Challenging Fit Types
[Hook types that may conflict with voice — use carefully or avoid]

## Word Choice Guidance
- Aligned vocabulary: [words/phrases that fit]
- Forbidden vocabulary: [words/phrases to avoid]

## Quality Metrics
- Soul.md loaded: [Y/N]
- Voice constraints extracted: [Y/N]
- Hook type fit assessed: [Y/N]
```

## Output File

- **Path:** `layer-0-outputs/0.4-soul-md.md`
- **Minimum size:** 1KB

## Quality Checks

- If Soul.md exists, all voice fields extracted
- Anti-voice patterns are specific (not "avoid generic language")
- Hook type fit is reasoned (not arbitrary)
- If Soul.md missing, output acknowledges lack of voice constraints

## Dependencies

- None (Soul.md is optional)
- If missing, hook generation proceeds without voice constraints (quality degradation but not blocking)
