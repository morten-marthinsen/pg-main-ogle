# Microskill 1.4: Pacing Planner

**Layer:** 1 (Sequencing & Proportion)
**Purpose:** Plan attention-maintenance pacing throughout the assembled copy

---

## CORE PRINCIPLE: THE ENGAGEMENT WAVE

Long-form copy must maintain reader attention through strategic pacing. Attention naturally ebbs and flows. The Pacing Planner designs deliberate peaks and valleys that PULL the reader forward rather than letting attention randomly decay.

**TIER1 Observation:** Elite controls don't maintain constant intensity. They alternate between:
- High-intensity moments (proof cascades, dramatic revelations)
- Rest moments (story interludes, relatable asides)
- Acceleration zones (urgency builds, stakes escalate)
- Payoff moments (big reveals, proof delivery)

---

## PACING ELEMENTS

### Intensity Markers

| Marker | Description | Function |
|--------|-------------|----------|
| PEAK | Maximum emotional/logical intensity | Major reveals, proof cascades, urgency climax |
| BUILD | Rising intensity toward peak | Stakes escalation, tension accumulation |
| REST | Intentional deceleration | Story moments, relatable interludes, breathing room |
| BRIDGE | Neutral transition | Section connections, topic shifts |
| HOOK | Re-engagement trigger | Pattern interrupts, curiosity loops, open questions |

### Pacing Patterns by Format

#### VSL Pacing (12-18 min)

```
[0-2min] HOOK → BUILD → PEAK (curiosity)
[2-5min] REST (story) → BUILD → PEAK (problem agitation)
[5-8min] BUILD → PEAK (root cause reveal) → REST
[8-12min] BUILD → PEAK → BUILD → PEAK (mechanism proof cascade)
[12-15min] REST (product intro) → BUILD → PEAK (offer)
[15-18min] BUILD → PEAK → PEAK → PEAK (close urgency cascade)
```

#### Magalog Pacing (24-40 pages)

```
[pp 1-4] HOOK → BUILD → PEAK (headline payoff)
[pp 5-10] REST (story) → BUILD → PEAK (problem establishment)
[pp 11-16] BUILD → BUILD → PEAK (root cause) → REST
[pp 17-24] HOOK → BUILD → PEAK → REST → BUILD → PEAK (mechanism)
[pp 25-32] BUILD → PEAK → REST → BUILD → PEAK (proof cascade)
[pp 33-38] BUILD → PEAK (offer) → BUILD → PEAK (urgency)
[pp 39-40] PEAK → PEAK → PEAK (close)
```

#### Sales Letter Pacing (4,000-8,000 words)

```
[0-500w] HOOK → BUILD → PEAK
[500-1500w] REST → BUILD → PEAK → REST
[1500-3000w] BUILD → PEAK → BUILD → PEAK
[3000-5000w] HOOK → BUILD → PEAK → REST → BUILD → PEAK
[5000-7000w] BUILD → PEAK → BUILD → PEAK (offer + urgency)
[7000-8000w] PEAK → PEAK → PEAK
```

## ATTENTION MAINTENANCE TECHNIQUES

### Re-Engagement Hooks

Deploy at natural attention decay points:

| Technique | Example | When to Use |
|-----------|---------|-------------|
| Pattern Interrupt | "But here's what nobody tells you..." | After REST sections |
| Open Question | "So what does this mean for YOUR..." | Before BUILD sections |
| Curiosity Loop | "In just a moment, I'll reveal..." | Mid-section to pull forward |
| Stakes Escalation | "And it gets even worse..." | Before PEAK sections |
| Contrast Pivot | "While everyone else... you'll..." | Before differentiation points |

### Rest Techniques

REST sections must maintain engagement while reducing intensity:

| Technique | Function |
|-----------|----------|
| Story Continuation | Narrative thread reader wants to complete |
| Relatable Moment | "You've probably felt this too..." |
| Quick Win | Small insight reader can apply immediately |
| Humanizing Aside | Personal/vulnerable moment from narrator |
| Summary Anchor | Brief recap that reinforces key points |

### Peak Techniques

PEAK sections must deliver maximum impact:

| Technique | Function |
|-----------|----------|
| Proof Cascade | Multiple proof elements in rapid succession |
| Revelation Stack | Layered reveals building to climax |
| Urgency Accumulation | Mounting reasons to act NOW |
| Transformation Vision | Vivid future-state painting |
| Stakes Pinnacle | Maximum consequence articulation |

## PACING MAP CREATION

### Step 1: Mark Section Intensity Defaults

Based on section type:

```yaml
section_intensity_defaults:
  lead: HOOK → BUILD → PEAK
  story: REST → BUILD → PEAK
  root_cause: BUILD → PEAK → REST
  mechanism: BUILD → PEAK → BUILD → PEAK
  proof_section: BUILD → PEAK → REST → BUILD → PEAK
  product: REST → BUILD → PEAK
  offer: BUILD → PEAK → BUILD → PEAK
  close: BUILD → PEAK → PEAK → PEAK
  ps: HOOK → PEAK
```

### Step 2: Insert Re-Engagement Points

Mark every location where:
- Section exceeds 500 words without intensity shift
- REST section exceeds 200 words
- BUILD section needs acceleration trigger

### Step 3: Identify Danger Zones

Flag attention-risk areas:

```yaml
danger_zones:
  - location: "After first story section"
    risk: "Reader may feel story is complete"
    intervention: "Open loop to mechanism"

  - location: "Mid-mechanism explanation"
    risk: "Technical fatigue"
    intervention: "Proof break or relatable example"

  - location: "Between proof section and product"
    risk: "Transition feels abrupt"
    intervention: "Bridge with transformation callback"

  - location: "Mid-offer section"
    risk: "Reader may feel sold-to fatigue"
    intervention: "Value reframe or objection anticipation"
```

### Step 4: Plan Peak Placement

Ensure peaks are:
- Spaced appropriately (not back-to-back exhaustion)
- Building in intensity toward close
- Earning their impact through proper BUILD sections

## PACING VALIDATION RULES

```
[ ] No section exceeds 600 words without intensity shift
[ ] REST sections don't exceed 300 words without re-engagement hook
[ ] Every PEAK is preceded by BUILD (no unearned peaks)
[ ] Close section has highest peak density
[ ] At least one re-engagement hook per 500 words
[ ] Danger zones have planned interventions
[ ] Pacing supports rather than fights section content
```
