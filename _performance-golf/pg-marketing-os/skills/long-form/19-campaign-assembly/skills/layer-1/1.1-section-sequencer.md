# Microskill 1.1: Section Sequencer

**Layer:** 1 (Sequencing & Proportion)
**Purpose:** Validate and finalize section sequence for assembly

---

## DEFAULT SECTION SEQUENCE

Standard RMBC-based sequence:

```
1. Pre-head (if applicable)
2. Headline
3. Subheadline (if applicable)
4. Lead
5. Story
6. Root Cause Narrative
7. Mechanism Narrative
8. Proof Section
9. Product Introduction
10. Offer
11. Close
12. P.S. Section(s)
```

---

## SEQUENCE VARIATIONS

### Proof-First Sequence (Whitaker Style)

When lead proof is dramatic enough to open:

```
1. Headline (proof-focused)
2. Lead (with dramatic proof)
3. Scale Proof ("Not alone")
4. Authority Introduction
5. Root Cause
6. Mechanism
7. Additional Proof
8. Product
9. Offer
10. Close
```

### Authority-First Sequence (Sinatra Style)

When authority establishes credibility first:

```
1. Headline (authority-focused)
2. Lead (authority establishment)
3. Problem Setup
4. Root Cause
5. Mechanism (with credentials woven)
6. Proof
7. Product
8. Offer
9. Close
```

### Data-Heavy Sequence (Stansberry Style)

When data cascade drives persuasion:

```
1. Headline (crisis-focused)
2. Lead (question setup)
3. Authority
4. Crisis Reveal (single devastating stat)
5. Data Cascade
6. Visual Proof
7. Historical Precedent
8. Expert Validation
9. Blueprint/Solution
10. Offer
11. Close
```

## SEQUENCE VALIDATION

### Check Structure Alignment

```
FROM structure-package.json:
  defined_sequence: [list]

VALIDATE: Does proposed sequence match structure?
  IF mismatch: Note deviation and rationale
```

### Check Section Dependencies

| Section | Depends On | Cannot Precede |
|---------|------------|----------------|
| Story | Lead | Root Cause |
| Root Cause | Story (or Lead) | Mechanism |
| Mechanism | Root Cause | Product |
| Proof Section | Mechanism (usually) | Product |
| Product | Mechanism | Offer |
| Offer | Product | Close |
| Close | Offer | — |

### Check Proof Integration Points

```
FROM proof-weaving-package.json:
  proof_blocks_by_section: [map]

VALIDATE: Does sequence allow all proof insertions?
  Each proof block must have a target section in sequence
```
