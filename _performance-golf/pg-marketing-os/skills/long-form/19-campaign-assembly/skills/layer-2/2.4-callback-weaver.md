# Microskill 2.4: Callback Weaver

**Layer:** 2 (Assembly)
**Purpose:** Weave callbacks throughout copy to create internal coherence

---

## CORE PRINCIPLE: THE ECHO EFFECT

Callbacks create the sensation that the copy "remembers" itself. When a phrase, image, or concept established early is echoed later, it:
- Validates the reader's attention investment
- Creates subconscious coherence
- Amplifies emotional impact through recognition
- Makes the copy feel crafted rather than assembled

**TIER1 Observation:** Elite controls establish 4-8 callback loops, with the close section "paying off" at least 3 earlier elements.

---

## 4 CALLBACK PATTERNS

### 1. Lead-to-Close Callback

The opening hook/promise echoes in the final close.

**Establish (Lead):**
> "What if you could wake up tomorrow and finally see the number you've been chasing for years?"

**Callback (Close):**
> "Tomorrow morning, when you wake up and step on that scale, you could be starting the journey to finally see that number—the one you've been chasing for years."

### 2. Proof-to-Close Callback

Specific proof elements are referenced in the close as accumulated evidence.

**Establish (Proof Section):**
> "47,000 people have already used this system..."

**Callback (Close):**
> "Remember: 47,000 people have already proven this works. The only question is whether you'll be number 47,001..."

### 3. Story-to-Product Callback

Story transformation connects to product delivery.

**Establish (Story):**
> "That was the moment I realized I'd been doing everything wrong..."

**Callback (Product):**
> "This is the exact system I wish I'd had in that moment—so you don't have to spend years doing everything wrong like I did."

### 4. Root-Cause-to-Mechanism Callback

The problem established in root cause is solved by mechanism.

**Establish (Root Cause):**
> "The real reason nothing has worked is because your metabolism has been 'hijacked' by..."

**Callback (Mechanism):**
> "That's why this approach works—it doesn't fight your hijacked metabolism. It liberates it."

## CALLBACK SOURCES

### From Upstream Packages

| Package | Callback Elements |
|---------|-------------------|
| lead-package | Opening hook, primary promise, curiosity element |
| story-package | Transformation moment, protagonist's struggle, discovery moment |
| root-cause-package | Villain/enemy name, anchor phrase, "real reason" statement |
| mechanism-package | Mechanism name, framework name, key differentiator |
| proof-weaving-package | callback_registry entries |
| promise-package | Primary promise, secondary promises |

### From Proof-Weaving Package

The `callback_registry` should contain pre-identified callback pairs:

```json
{
  "callback_registry": [
    {
      "callback_id": "CALLBACK_001",
      "source_block": "PROOF_LEAD_001",
      "source_content": "[phrase]",
      "target_block": "PROOF_CLOSE_001",
      "target_content": "[callback phrase]"
    }
  ]
}
```

## CALLBACK WEAVING PROCESS

### Step 1: Inventory Callback Opportunities

List all elements that COULD be called back:

```yaml
callback_inventory:
  opening_elements:
    - element: "[opening hook phrase]"
      section: "lead"
      location: "[where in lead]"
      callback_potential: HIGH | MEDIUM

    - element: "[primary promise]"
      section: "lead"
      # ...

  story_elements:
    - element: "[transformation moment]"
      section: "story"
      # ...

  root_cause_elements:
    - element: "[villain name]"
      # ...

  mechanism_elements:
    - element: "[mechanism name]"
      # ...
```

### Step 2: Identify Callback Destinations

Mark where callbacks should land:

```yaml
callback_destinations:
  close_section:
    - destination_point: "Before final CTA"
      suitable_for: [lead_callback, proof_callback]

    - destination_point: "Urgency section"
      suitable_for: [root_cause_callback]

    - destination_point: "Benefit summary"
      suitable_for: [story_callback, promise_callback]

  product_section:
    - destination_point: "Product introduction"
      suitable_for: [story_callback, mechanism_callback]
```

### Step 3: Draft Callbacks

For each selected callback:

```yaml
callback_draft:
  callback_id: "CB_001"
  type: "lead_to_close"

  source:
    element: "[exact phrase/concept]"
    section: "lead"
    location: "[where]"

  destination:
    section: "close"
    location: "[where inserting callback]"

  callback_copy: "[the callback phrase/sentence]"

  integration_method:
    - weave_into_existing: "[how it connects to surrounding copy]"
    - standalone_callback: "[if it's a distinct callback moment]"
```

### Step 4: Insert Callbacks

Insert drafted callbacks at designated destinations, ensuring:
- Callback feels natural, not forced
- Surrounding copy supports the callback
- Reader can recognize the echo without explicit "Remember when..."

---

## CALLBACK QUALITY RULES

### Effective Callbacks:

```
[ ] Echo without being verbatim (variation, not repetition)
[ ] Arrive at emotionally appropriate moments
[ ] Amplify rather than merely repeat
[ ] Feel earned by earlier establishment
[ ] Support the section's purpose
```

### Callback Anti-Patterns:

| Anti-Pattern | Example | Fix |
|--------------|---------|-----|
| Too explicit | "Remember when I said..." | Weave naturally |
| Verbatim repetition | Exact same phrase | Vary while maintaining recognition |
| Forced placement | Callback interrupts flow | Find natural insertion point |
| Too many | 10+ callbacks | Select 4-8 highest impact |
| Missing establishment | Callback to unstated element | Ensure original is present |

---

## MINIMUM CALLBACK REQUIREMENTS

```yaml
required_callbacks:
  lead_to_close:
    minimum: 2
    elements: [opening_hook, primary_promise]

  story_to_close:
    minimum: 1
    elements: [transformation_moment]

  proof_to_close:
    minimum: 2
    elements: [headline_proof, cascade_proof]

  root_cause_to_mechanism:
    minimum: 1
    elements: [villain_callback, problem_callback]
```
