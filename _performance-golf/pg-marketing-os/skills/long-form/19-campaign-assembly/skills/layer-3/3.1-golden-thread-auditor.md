# Microskill 3.1: Golden Thread Auditor

**Layer:** 3 (Refinement & Audit)
**Purpose:** Audit threading element counts against minimum requirements

---

## CORE FUNCTION

The Golden Thread Auditor verifies that all threading elements (mechanism name, root cause anchor, framework, promise) appear the required number of times throughout the assembled copy. This is COUNTING and VERIFICATION, not writing.

---

## THREADING REQUIREMENTS MATRIX

From 1.3-golden-thread-mapper:

| Thread Element | Minimum Count | Required Distribution |
|----------------|---------------|----------------------|
| Mechanism Name | 8+ | Every section except P.S. |
| Root Cause Anchor | 5+ | Lead, Root Cause, Mechanism, Close |
| Framework/System Name | 4+ | Mechanism, Product, Offer, Close |
| Primary Promise | 6+ | Lead, Story, Mechanism, Product, Offer, Close |
| Villain/Enemy | 3+ | Root Cause, Mechanism, Close |
| Product Name | 4+ | Product, Offer, Close, P.S. |

---

## AUDIT PROCESS

### Step 1: Extract Thread Elements

Load from upstream packages:

```yaml
thread_elements:
  mechanism_name:
    exact: "[from mechanism-package]"
    variations: ["[list]", "[acceptable]", "[variations]"]

  root_cause_anchor:
    exact: "[from root-cause-package]"
    variations: ["[list]"]

  framework_name:
    exact: "[from mechanism-package]"
    variations: ["[list]"]

  primary_promise:
    core: "[from promise-package]"
    variations: ["[list]"]

  villain:
    exact: "[from root-cause-package]"
    variations: ["[list]"]

  product_name:
    exact: "[from product-package]"
    variations: ["[list]"]
```

### Step 2: Count Occurrences

For each thread element, count occurrences in assembled copy:

```yaml
thread_count:
  mechanism_name:
    total: "[count]"
    by_section:
      lead: "[count]"
      story: "[count]"
      root_cause: "[count]"
      mechanism: "[count]"
      proof: "[count]"
      product: "[count]"
      offer: "[count]"
      close: "[count]"

  root_cause_anchor:
    total: "[count]"
    by_section:
      # ...

  # ... for each thread element
```

### Step 3: Compare Against Minimums

```yaml
thread_validation:
  mechanism_name:
    required: 8
    actual: "[count]"
    status: PASS | FAIL
    gap: "[X] additional needed"

  root_cause_anchor:
    required: 5
    actual: "[count]"
    status: PASS | FAIL
    gap: "[X] additional needed"

  # ... for each thread element
```

### Step 4: Check Distribution

Verify thread elements appear in required sections:

```yaml
distribution_check:
  mechanism_name:
    required_sections: [lead, story, root_cause, mechanism, proof, product, offer, close]
    present_in: [list of sections where found]
    missing_from: [list of sections where absent]
    distribution_status: PASS | FAIL

  # ... for each thread element
```

---

## GAP IDENTIFICATION

If any thread element fails:

```yaml
threading_gaps:
  - element: "mechanism_name"
    shortfall: "[X] additional mentions needed"
    missing_sections: [list]
    suggested_insertions:
      - section: "[name]"
        location: "[where in section]"
        insertion_context: "[surrounding copy hint]"

  - element: "primary_promise"
    shortfall: "[X] additional mentions needed"
    # ...
```

---

## THREADING QUALITY CHECK

Beyond quantity, check quality:

```
[ ] Thread elements are woven naturally (not forced insertions)
[ ] Variations maintain recognition while avoiding repetition
[ ] Thread elements appear at high-impact moments
[ ] No section is oversaturated with thread mentions
[ ] Threading supports persuasion flow (not interrupting it)
```

### Thread Saturation Limits

| Section | Maximum Thread Density |
|---------|----------------------|
| Lead | 1 per 100 words |
| Story | 1 per 150 words |
| Root Cause | 1 per 100 words |
| Mechanism | 1 per 80 words |
| Proof | 1 per 120 words |
| Product | 1 per 80 words |
| Offer | 1 per 100 words |
| Close | 1 per 60 words |

---

## REMEDIATION PROTOCOL

If threading audit fails:

### For Count Failures:

1. Identify sections with lowest thread density
2. Find natural insertion points in those sections
3. Draft thread insertions that flow naturally
4. Insert and recount

### For Distribution Failures:

1. Identify required sections missing thread
2. Find natural insertion points in each missing section
3. Draft thread insertions
4. Insert and verify distribution
