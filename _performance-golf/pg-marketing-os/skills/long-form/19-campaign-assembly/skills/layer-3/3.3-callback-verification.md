# Microskill 3.3: Callback Verification

**Layer:** 3 (Refinement & Audit)
**Purpose:** Verify all planned callbacks are properly established and echoed

---

## CORE FUNCTION

Callback Verification confirms that:
1. Every callback TARGET in the close has a corresponding ESTABLISHMENT earlier
2. Every ESTABLISHED callback element has its ECHO in the appropriate section
3. Callbacks create recognition without being redundant

---

## CALLBACK VERIFICATION PROCESS

### Step 1: Load Callback Registry

From 2.4-callback-weaver output:

```yaml
callback_registry_load:
  callbacks_planned: "[count]"
  callbacks:
    - id: "CB_001"
      type: "lead_to_close"
      source_section: "lead"
      source_content: "[exact phrase/concept]"
      target_section: "close"
      target_content: "[callback copy]"

    # ... all callbacks
```

### Step 2: Verify Establishments Exist

For each callback, confirm the source content exists in source section:

```yaml
establishment_verification:
  - callback_id: "CB_001"
    source_section: "lead"
    source_content: "[expected]"
    found_in_copy: true | false
    exact_location: "[word count or paragraph reference]"
    actual_text: "[what was found]"
    status: PRESENT | MISSING | MODIFIED
```

### Step 3: Verify Echoes Exist

For each callback, confirm the target content exists in target section:

```yaml
echo_verification:
  - callback_id: "CB_001"
    target_section: "close"
    target_content: "[expected callback]"
    found_in_copy: true | false
    exact_location: "[reference]"
    actual_text: "[what was found]"
    status: PRESENT | MISSING | MODIFIED
```

### Step 4: Verify Recognition Connection

The echo must be recognizable as connected to the establishment:

```yaml
recognition_check:
  - callback_id: "CB_001"
    establishment: "[source phrase]"
    echo: "[target phrase]"
    connection_type: EXPLICIT | IMPLICIT | THEMATIC
    recognition_strength: STRONG | MODERATE | WEAK
    assessment: EFFECTIVE | NEEDS_STRENGTHENING | TOO_SUBTLE | TOO_OBVIOUS
```

---

## CALLBACK QUALITY CRITERIA

### Effective Callbacks:

| Criterion | Description | Check |
|-----------|-------------|-------|
| Established First | Source appears before target | `[ ]` |
| Recognizable | Reader can connect echo to source | `[ ]` |
| Not Verbatim | Echo varies from source | `[ ]` |
| Emotionally Resonant | Echo amplifies original impact | `[ ]` |
| Naturally Placed | Echo fits context naturally | `[ ]` |
| Properly Timed | Echo arrives at high-impact moment | `[ ]` |

### Callback Anti-Patterns to Flag:

| Anti-Pattern | Example | Issue |
|--------------|---------|-------|
| Orphaned Echo | Echo exists but establishment missing | Reader won't recognize |
| Orphaned Establishment | Established but never called back | Wasted setup |
| Verbatim Repeat | Exact same phrase | Feels repetitive, not callback |
| Too Subtle | Vague thematic connection only | Reader won't notice |
| Too Explicit | "Remember when I said..." | Patronizing |
| Wrong Section | Echo in unexpected location | Breaks pattern |

## REMEDIATION PROTOCOL

### For Missing Establishments:

1. Identify what element was supposed to be established
2. Find appropriate location in source section
3. Draft and insert establishment
4. Reverify echo still connects

### For Missing Echoes:

1. Locate the establishment
2. Find appropriate location in target section
3. Draft callback that connects to establishment
4. Insert and verify recognition

### For Weak Connections:

1. Assess why connection is weak
2. Either strengthen echo to be more recognizable
3. Or strengthen establishment to be more memorable
4. Reverify connection strength
