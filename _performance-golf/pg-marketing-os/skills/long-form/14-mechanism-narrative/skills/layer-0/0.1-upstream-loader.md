# Skill 14 — Microskill 0.1: Upstream Loader

**Skill:** 11 — Mechanism Narrative
**Microskill:** 0.1 — Upstream Loader
**Version:** 1.0
**Layer:** 0 (Foundation & Loading)
**Position:** First microskill in pipeline
**Type:** Data Loading & Normalization
**Dependencies:** Skill 03 (mechanism-package.json), Skill 10 (root-cause-narrative-package.json), Skill 07 (structure-package.json), Skill 01 (proof-inventory-output.json), Skill 09 (campaign-brief-package.json), Skill 10 prose
**Output:** upstream-data-cache.json

---

## PURPOSE

Loads and normalizes all upstream skill outputs required for mechanism narrative generation. This microskill collects the strategic mechanism (from Skill 11), the root cause narrative handoff (from Skill 16), the structural architecture (from Skill 11), and proof elements (from Skill 11) into a unified data cache that downstream microskills can consume without re-loading. It is the single point of truth for all external data entering the Skill 11 pipeline.

---

## IDENTITY

**What this IS:**
- A data aggregation and normalization layer
- The only microskill that reads external skill outputs directly
- A schema-aware loader that understands the exact shape of each upstream package

**What this is NOT:**
- Not a validator (that is 0.3)
- Not a decision-maker — it does not interpret data, only collects and normalizes it
- Not a content generator — produces no prose, only structured data

## PROCESS

### Step 1: Locate Source Files

```
FOR EACH upstream_source IN [skill_03, skill_10, skill_07, skill_01]:
  1. Construct absolute file path from project root
  2. Verify file exists at expected location
  3. IF file not found:
     a. Check alternate locations (versioned outputs, dated folders)
     b. Log warning with expected path and searched alternates
     c. Mark source as MISSING in loading_manifest
  4. IF file found:
     a. Log file path, size, last modified timestamp
     b. Mark source as LOCATED in loading_manifest
```

### Step 2: Load Mechanism Package (Skill 11)

```
OPEN mechanism-package.json
PARSE JSON structure

EXTRACT winner object:
  - name: The mechanism's identifying name
  - type: Classification of mechanism kind
  - explanation: Full explanation object with all four sub-fields
    - core_logic: The fundamental causal chain
    - analogy: Primary comprehension metaphor
    - step_by_step: Sequential breakdown
    - scientific_backing: Research foundation
  - proof_elements: Array of proof items mapped to this mechanism
  - e5_strategy: E-level classification and strategy
  - copy_integration_map: How mechanism threads through copy sections
  - scorecard: All nine dimension scores plus average

VALIDATE structure:
  - winner.name is non-empty string
  - winner.type matches valid enum
  - explanation contains at minimum core_logic and analogy
  - e5_strategy.e_level is valid enum value
  - scorecard.average is numeric and between 1-10
  - proof_elements array has length >= 1

NORMALIZE:
  - Trim whitespace from all string fields
  - Ensure consistent enum casing (lowercase)
  - Convert any null sub-fields to empty strings with WARNING flag
```

### Step 3: Load Root Cause Narrative Package (Skill 16)

```
OPEN root-cause-narrative-package.json
PARSE JSON structure

EXTRACT downstream_handoffs.for_mechanism_narrative:
  - bridge_text: The prose that transitions from root cause to mechanism
  - root_cause_context: Summary of what root cause was established
  - villain_established: The villain entity (name, type, characteristics)
  - emotional_state_at_handoff: Reader's emotional state at transition
  - belief_state: What reader now accepts as true
  - questions_opened: Unanswered questions that mechanism must answer

VALIDATE structure:
  - bridge_text is non-empty (minimum 50 characters)
  - villain_established has at minimum a name and type
  - emotional_state_at_handoff is non-empty
  - questions_opened has at least 1 entry

NORMALIZE:
  - Ensure villain_established has consistent structure
  - Map emotional_state to standardized emotion taxonomy
  - Flag if bridge_text ends mid-thought (no period/question mark)
```

### Step 4: Load Structure Package (Skill 11)

```
OPEN structure-package.json
PARSE JSON structure

EXTRACT mechanism-relevant sections:
  - campaign_thesis: Full thesis statement
  - cpb_chunks: Filter to mechanism_chunks only
  - flow_architecture: Full architecture object

VALIDATE:
  - campaign_thesis is non-empty
  - At least 1 CPB chunk allocated to mechanism
  - flow_architecture.format is valid enum
  - word_budget allocates words to mechanism section

NORMALIZE:
  - Calculate total mechanism word budget from chunks
  - Map chunk_order to sequential integers
  - Standardize format enum to lowercase
```

### Step 5: Load Proof Inventory (Skill 11)

```
OPEN proof-inventory-output.json
PARSE JSON structure

EXTRACT proof elements relevant to mechanism narrative:
  - Filter by tag: "mechanism", "mechanism_support", "scientific", "clinical"
  - Categorize into: institutional, clinical, testimonial, expert, statistical
  - Count elements per category

VALIDATE:
  - Total mechanism-relevant proof elements >= 3
  - At least 2 different proof categories represented
  - No proof element has empty source_citation

NORMALIZE:
  - Assign unique identifiers to each proof element
  - Sort by strength/quality score if available
  - Create quick-lookup index by category
```

### Step 6: Load Campaign Brief

```
LOCATE campaign-brief-package.json:
  PATH: 09-campaign-brief/outputs/campaign-brief-package.json

  IF file found:
    EXTRACT:
      - campaign_thesis (the overarching argument)
      - tone_direction (clinical/emotional/urgent/conversational)
      - avatar_summary (who we're writing for)
      - market_sophistication (Schwartz stage 1-5)
      - proof_approach (how proof should be deployed)
      - promise_ceiling (maximum defensible promise level)
    SET campaign_brief_status = "LOADED"

  IF file not found:
    SET campaign_brief_status = "MISSING"
    LOG: "WARNING: Campaign brief not found. Mechanism narrative may drift from campaign positioning."
    CONTINUE — non-blocking
```

### Step 7: Load Root Cause Narrative Assembled Prose (for continuity)

```
LOCATE root-cause-narrative-assembled-prose.md:
  PATH: 13-root-cause-narrative/outputs/root-cause-narrative-assembled-prose.md

  IF file found:
    LOAD full text of root cause narrative prose
    EXTRACT for continuity:
      - Voice/tone established in root cause section
      - Metaphors and analogies already used (to AVOID repeating in mechanism)
      - Proof elements already cited (to AVOID re-citing)
      - How the villain was characterized (to build on, not repeat)
      - The last paragraph/bridge text (for seamless transition)
      - What the reader was told about the ROOT CAUSE (to NOT re-explain in mechanism)
    SET root_cause_prose_status = "LOADED"

  IF file not found:
    SET root_cause_prose_status = "NOT_AVAILABLE"
    LOG: "WARNING: Root cause narrative prose not available. Mechanism narrative will rely on bridge_text only. Risk of content repetition between root cause and mechanism sections."
    CONTINUE — non-blocking but degraded
```

**Critical Insight:** This is where the Ultra Liver failure originated. The mechanism narrative re-explained the root cause because it only had bridge_text, not the full root cause prose. Loading the full prose prevents the mechanism from duplicating content already covered.

### Step 8: Load Context Reservoir (if available)

```
LOCATE context-reservoir.md:
  PATH: outputs/[project-code]/context-reservoir.md

  IF file found:
    LOAD relevant sections:
      - voc_quotes (for authentic mechanism language)
      - expression_anchoring (for phrases that penetrate)
      - arena_intelligence.mechanism_selection (why THIS mechanism won — what to emphasize)
      - counter_intuitive_core (the truth the mechanism explains)
    SET context_reservoir_status = "LOADED"

  IF file not found:
    SET context_reservoir_status = "NOT_AVAILABLE"
    CONTINUE — non-blocking
```

### Step 9: Compile Upstream Data Cache

```
CONSTRUCT upstream_data_cache:
  metadata:
    loaded_at: timestamp
    skill: "14-mechanism-narrative"
    microskill: "0.1-upstream-loader"
    version: "1.0"
    sources_loaded: count of successfully loaded sources
    sources_expected: 4

  mechanism: [normalized Skill 03 data]
  root_cause_handoff: [normalized Skill 10 data]
  structure: [normalized Skill 07 data]
  proof_inventory: [normalized Skill 01 data]

  loading_manifest:
    skill_06: { status, path, fields_loaded, warnings }
    skill_10: { status, path, fields_loaded, warnings }
    skill_07: { status, path, fields_loaded, warnings }
    skill_06: { status, path, fields_loaded, warnings }

WRITE upstream_data_cache to working memory
PASS to 0.2-tier1-intelligence-loader
```

### Step 7: Generate Loading Report

```
COMPILE loading_report:
  - Total sources attempted: 4
  - Total sources loaded successfully: N
  - Total fields extracted: N
  - Total warnings generated: N
  - Critical gaps: [list any missing required fields]
  - Recommendation: PROCEED / HALT / PROCEED_WITH_CAUTION

IF any source completely missing:
  SET recommendation = HALT
  LOG "Cannot proceed without [missing source]"

ELIF critical fields missing > 3:
  SET recommendation = PROCEED_WITH_CAUTION
  LOG "Multiple gaps detected — quality may be compromised"

ELSE:
  SET recommendation = PROCEED
```

## QUALITY CRITERIA

1. **All four sources loaded** — mechanism-package, root-cause-narrative-package, structure-package, proof-inventory-output all present and parsed
2. **No silent failures** — every missing field is logged, not ignored
3. **Normalization complete** — all enums lowercase, all strings trimmed, all nulls flagged
4. **Mechanism winner fully loaded** — name, type, explanation (with all four sub-fields), e5_strategy, scorecard
5. **Root cause handoff captured** — bridge_text, villain, emotional_state all present
6. **Proof inventory indexed** — proof elements categorized and counted, minimum 3 mechanism-relevant
7. **Loading manifest accurate** — reflects actual state of each source

---

## FAILURE MODES

| Failure | Symptom | Resolution |
|---------|---------|------------|
| Mechanism package missing | skill_03 status = MISSING | HALT — cannot generate narrative without mechanism |
| Root cause handoff missing | skill_10 status = MISSING | HALT — cannot continue emotional arc without handoff |
| Structure package missing | skill_07 status = MISSING | PROCEED_WITH_CAUTION — use default word budgets and generic pacing |
| Proof inventory missing | skill_01 status = MISSING | PROCEED_WITH_CAUTION — narrative will lack proof integration |
| Mechanism winner has no name | winner.name is empty | HALT — mechanism must be named |
| E-level not classified | e5_strategy.e_level is null | PROCEED_WITH_CAUTION — downstream skills will use E3 default |
| Bridge text too short | bridge_text < 50 chars | Flag for human review in 0.4 checkpoint |
| Villain not established | villain_established is null | Flag — mechanism narrative may lack contrast |
| Zero proof elements | proof_inventory.total_count = 0 | PROCEED_WITH_CAUTION — proof integration phase will be thin |
