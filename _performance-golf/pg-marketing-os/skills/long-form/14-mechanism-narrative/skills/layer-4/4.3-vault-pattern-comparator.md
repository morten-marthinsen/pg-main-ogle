# Skill 14 — Microskill 4.3: Vault Pattern Comparator

**Skill:** 11 — Mechanism Narrative
**Microskill:** 4.3 — Vault Pattern Comparator
**Version:** 1.0
**Layer:** 4 (Validation & Assembly)
**Position:** Third microskill in Layer 4
**Type:** Comparative Validation & Differentiation Check
**Dependencies:** 3.4-narrative-flow-polisher (polished narrative), 0.2-tier1-intelligence-loader (TIER1 patterns), 4.1-e-level-alignment-checker (e_level confirmed)
**Output:** vault-comparison-report.json

---

## PURPOSE

Compares the generated mechanism narrative against elite mechanism narratives from TIER1 controls at the same E-level. This comparator serves two goals: (1) verifying the narrative achieves the QUALITY standard of elite controls, and (2) ensuring the narrative is DIFFERENTIATED — it should learn from elite patterns but not copy them. A mechanism narrative that reads like a rewrite of an existing TIER1 control fails on originality. A narrative that shares no patterns with elite controls fails on quality.

---

## IDENTITY

**What this IS:**
- A quality benchmark that measures against proven elite controls
- A differentiation enforcer that requires originality alongside pattern adherence
- A pattern comparison tool that maps generated narrative to TIER1 examples

**What this is NOT:**
- Not a plagiarism checker — checks pattern similarity, not text duplication
- Not a scoring rubric alone — also checks for differentiation
- Not a creative evaluator — measures structural and strategic quality, not "how good does it sound"
- Not a rewrite tool — identifies gaps, does not generate content

## PROCESS

### Step 1: Select Comparison Controls

```
IDENTIFY 3-5 TIER1 controls for comparison:

PRIMARY SELECTION CRITERIA:
  1. Same E-level as generated narrative
  2. Same or similar niche (health → health, wealth → wealth, etc.)
  3. Same or similar mechanism type
  4. Same narrative type if possible

COMPARISON POOL BY E-LEVEL:

  E2 Controls:
    - Garcinia (compound, health, E2, scientific_revelation)
    - TestoRise (compound, health, E2, scientific_revelation)

  E3 Controls:
    - Montezuma's Secret (compound, health/male, E3, accidental_discovery)
    - i-flex (compound, health/joint, E3, data_driven_debunking)
    - DIM (compound, health/hormones, E3, scientific_revelation)

  E4 Controls:
    - LA-3/AMPK (biological, health/aging, E4, scientific_revelation)
    - Total Restore (compound, health/gut, E4, scientific_revelation)
    - SF1 (technology, golf, E4, proprietary_technology)

  E5 Controls:
    - Billionaire Brain Wave (frequency, wealth, E5, paradigm_shift)
    - Superbrain (system, education, E5/identity, identity_driven)

SELECT controls:
  BEST MATCH: Same E-level + same niche → primary comparison
  GOOD MATCH: Same E-level + different niche → secondary comparison
  USEFUL MATCH: Adjacent E-level + same niche → context comparison

  SELECT 3-5 controls from pool, prioritizing best matches
  LOG: "Comparing against: [control_1], [control_2], [control_3]..."
```

### Step 2: Compare Narrative Type Match

```
FOR EACH comparison control:

  DIMENSION: Narrative Type
  Generated narrative type: [from 1.1]
  Control narrative type: [from TIER1 index]

  IF same type:
    QUALITY SIGNAL: Generated narrative follows the proven pattern for
      this type at this E-level
    DIFFERENTIATION RISK: Higher — must ensure originality within type

  IF different type:
    NOTE: "Generated narrative uses [type] while control uses [control_type].
      This may indicate creative choice or strategic deviation."
    QUALITY SIGNAL: Cross-check that type choice is justified by 1.1 rationale
    DIFFERENTIATION SIGNAL: Natural differentiation through type choice

  SCORE narrative_type_match: 1-10
    10: Same type, well-executed
    8: Different but justified type
    5: Different type without clear justification
    3: Type that doesn't match E-level requirements
```

### Step 3: Compare Explanation Depth

```
FOR EACH comparison control:

  DIMENSION: Explanation Depth
  COMPARE:
    - Detail layer count: generated vs. control
    - Scientific terminology density: generated vs. control
    - Metaphor count: generated vs. control
    - Cause-effect chain complexity: generated vs. control

  SCORING:
    IF generated depth is within 20% of control range → MATCHED (8-10)
    IF generated depth is significantly less → SHALLOW (4-6)
    IF generated depth is significantly more → DEEP (7-9 if justified, 4-6 if unnecessary)

  QUALITATIVE COMPARISON:
    Does the generated explanation achieve the same level of UNDERSTANDING
    that the control achieves?
    - Control reader: "I now understand how AMPK works"
    - Generated reader: "I now understand how [mechanism] works"
    - If same comprehension level achieved through different means: HIGH SCORE
    - If comprehension level is lower: NEEDS IMPROVEMENT

  SCORE explanation_depth_match: 1-10
```

### Step 4: Compare Simplification Quality

```
FOR EACH comparison control:

  DIMENSION: Simplification Quality
  COMPARE:
    - Metaphor concreteness: generated vs. control
      Control (LA-3): "on/off switch" — highly concrete
      Generated: [assess concreteness]
    - Re-simplification presence: generated vs. control
      Control (BBW): "Just think POOR. Just think MONEY." — present
      Generated: [assess presence]
    - 12-year-old test: both should pass
    - Analogy chain depth: generated vs. control
      Control (Gundry): 10+ layers — very deep chain
      Generated: [count layers]

  SCORING:
    IF generated simplification matches control quality: 8-10
    IF generated simplification is less effective: 4-7
    IF generated simplification is better (rarer but possible): 9-10

  KEY COMPARISON POINTS:
    - Does the generated metaphor create the same instant understanding?
    - Is the "think about it" moment as effective?
    - Can the simplified version be repeated to a friend?

  SCORE simplification_quality: 1-10
```

### Step 5: Compare Proof Integration Density

```
FOR EACH comparison control:

  DIMENSION: Proof Integration
  COMPARE:
    - Number of proof elements: generated vs. control
    - Proof categories used: generated vs. control
    - Integration strategy: generated vs. control
    - Organic weaving quality: generated vs. control

  CONTROL BENCHMARKS:
    LA-3: 5+ institutional (Yale, Salk, Nature, NIH) — rapid stacking
    BBW: 3 institutional + self-proving logic — hybrid approach
    i-flex: 4 elements (2 competitor failure + 2 mechanism success) — debunking
    Montezuma: Escalating ladder (village → doctor → study → clinical trial)
    SF1: Visual demonstration (robot testing) + engineering proof

  SCORING:
    IF generated proof integration matches control density and quality: 8-10
    IF fewer proof elements but well-integrated: 6-8
    IF more proof elements but poorly integrated (feels like a list): 4-6
    IF proof is missing or token: 2-4

  SCORE proof_integration_density: 1-10
```

### Step 6: Compare Naming Memorability

```
FOR EACH comparison control:

  DIMENSION: Naming Memorability
  COMPARE mechanism names:

  CONTROL NAMES (benchmarks):
    - "AMPK" / "The Master Switch" — metaphorical rename, highly memorable
    - "Billionaire Brain Wave" / "Theta" — villain contrast, emotionally loaded
    - "Square Face Technology" — proprietary branded, immediately visual
    - "Firming Flavonoids" — proprietary branded, alliterative
    - "Danish Rosehips" — historical origin, geographic specificity
    - "The Boy With The Broken Brain" — identity naming (not mechanism per se)

  COMPARISON CRITERIA:
    Syllable economy: Generated name vs. control name (fewer = more memorable)
    Visual imagery: Does generated name create a picture? (control names do)
    Emotional loading: Does generated name carry emotion? (control names do)
    Repeatability: Can someone say the generated name naturally?
    Uniqueness: Does the generated name sound different from control names?

  SCORING:
    IF generated name matches control memorability: 8-10
    IF generated name is functional but forgettable: 5-7
    IF generated name is generic or confusing: 2-4

  SCORE naming_memorability: 1-10
```

### Step 7: Compare Emotional Arc Quality

```
FOR EACH comparison control:

  DIMENSION: Emotional Arc
  COMPARE:
    - Arc type: generated vs. control
    - Emotional range: generated vs. control (how far does the emotion travel?)
    - Aha moment impact: generated vs. control
    - Entry-to-exit emotional distance: generated vs. control

  CONTROL BENCHMARKS:
    LA-3: fear → education → hope (wide range, aha at Phase 4)
    BBW: shame → vindication → dignity (maximum emotional range)
    i-flex: frustration → revelation → confidence (focused range)
    Montezuma: shame → vindication → dignity (similar to BBW)
    Superbrain: empathy → belief → aspiration (inspiration-focused)

  SCORING:
    IF emotional range is comparable to control: 8-10
    IF narrower but appropriate for mechanism: 6-8
    IF flat or disconnected: 2-5

  SCORE emotional_arc_quality: 1-10
```

### Step 8: Compare Bridge Smoothness

```
FOR EACH comparison control:

  DIMENSION: Product Bridge
  COMPARE:
    - Bridge pattern used: generated vs. control
    - Naturalness of transition: generated vs. control
    - "Pitch feeling" (lower is better): generated vs. control

  CONTROL BENCHMARKS:
    LA-3: formula_activates_mechanism — smooth, research-driven
    BBW: mechanism_is_product — direct, clear
    i-flex: proof_first_bridge — clinical results lead naturally
    Superbrain: information_teaches_mechanism — story-driven
    SF1: mechanism_is_product — engineering demonstration leads in

  SCORING:
    IF bridge feels as natural as control: 8-10
    IF bridge is present but slightly pitchy: 5-7
    IF bridge feels like a sales pivot: 2-4

  SCORE bridge_smoothness: 1-10
```

### Step 9: Differentiation Check

```
CRITICAL: Generated narrative must be DIFFERENTIATED from controls.

DIFFERENTIATION DIMENSIONS:
  1. Mechanism is different (obviously — different product)
  2. Metaphors are original (not the same metaphors as controls)
  3. Proof elements are unique (different institutions, different data)
  4. Emotional arc has unique moments (not a copy of control arc beats)
  5. Naming is original (not a formula copy of control naming)
  6. Structure has variations (not an identical structural template)

SCAN for excessive similarity:
  FOR EACH comparison control:
    Count structural similarities vs. differences

  DIFFERENTIATION SCORE:
    7+ differences out of 10 structural elements: WELL DIFFERENTIATED (8-10)
    5-6 differences: ADEQUATELY DIFFERENTIATED (6-7)
    3-4 differences: INSUFFICIENTLY DIFFERENTIATED (3-5)
    < 3 differences: COPY — FAIL (1-2)

  SPECIFIC COPY INDICATORS:
    - Same metaphor as control (e.g., "on/off switch" for non-AMPK mechanism)
    - Same naming architecture for same niche (e.g., "[Location]'s Secret"
      for another male health mechanism)
    - Same proof stacking pattern for same E-level without unique elements
    - Same emotional arc beats in same order without variation

  IF COPY indicators found:
    FLAG: "Generated narrative too similar to [control name].
      Differentiate [specific dimension]."

SCORE differentiation: 1-10
```

### Step 10: Calculate Composite Score

```
COMPILE dimension scores across all comparison controls:

  narrative_type_match: average across controls
  explanation_depth: average across controls
  simplification_quality: average across controls
  proof_integration_density: average across controls
  naming_memorability: average across controls
  emotional_arc_quality: average across controls
  bridge_smoothness: average across controls
  differentiation: single score (against all controls)

WEIGHTED COMPOSITE:
  narrative_type_match: 10% weight
  explanation_depth: 15% weight
  simplification_quality: 20% weight (most critical)
  proof_integration_density: 15% weight
  naming_memorability: 10% weight
  emotional_arc_quality: 15% weight
  bridge_smoothness: 5% weight
  differentiation: 10% weight

  composite_score = weighted average

PASS THRESHOLD: >= 6.0
  6.0-6.9: PASS — meets minimum quality standard
  7.0-7.9: GOOD — solid mechanism narrative
  8.0-8.9: EXCELLENT — approaches elite control quality
  9.0-10.0: EXCEPTIONAL — matches or exceeds elite controls
```

## QUALITY CRITERIA

1. **Composite score >= 6.0** — minimum quality threshold met
2. **Differentiation documented** — narrative is clearly distinct from comparison controls
3. **No copy warnings** — narrative does not replicate control patterns wholesale
4. **All 8 dimensions scored** — no dimension skipped
5. **3-5 controls compared** — adequate comparison pool
6. **Gaps identified are actionable** — each gap specifies what to improve
7. **Matches documented** — what the narrative does well is recognized

---

## FAILURE MODES

| Failure | Symptom | Resolution |
|---------|---------|------------|
| Composite score < 6.0 | Below minimum quality | Identify weakest dimensions, return to Layer 3 for targeted improvement |
| Differentiation score < 5 | Too similar to a control | Change metaphors, naming, or structural approach for originality |
| Copy warning triggered | Specific element matches a control too closely | Replace the copied element with original variation |
| No controls available for comparison | Unusual E-level + niche combination | Use closest available controls, note comparison limitations |
| Simplification quality low | Generated metaphors weaker than controls | Strengthen metaphor anchor, add analogy chain, improve re-simplification |
| All dimensions high but differentiation low | High quality but unoriginal | Creative restructuring needed — same quality, different expression |
