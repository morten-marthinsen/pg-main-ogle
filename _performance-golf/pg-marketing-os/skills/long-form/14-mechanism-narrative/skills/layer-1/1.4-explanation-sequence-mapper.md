**Output:** explanation-sequence-map.json

---

## PURPOSE

Maps the 5-phase explanation sequence with precise word allocations, simplification technique assignments, and format-specific variations. This is the structural blueprint that Layer 2 writers follow — it tells each writer exactly how many words they have, which simplification technique to deploy, how their phase connects to adjacent phases, and how the format (VSL, magalog, advertorial) affects their approach. This microskill synthesizes all Layer 1 outputs into an actionable production plan.

---

## IDENTITY

**What this IS:**
- The structural blueprint for the mechanism narrative — word budgets, phase boundaries, simplification assignments
- The synthesis point for all Layer 1 decisions (type, frame, arc, and now sequence)
- The direct instruction set that Layer 2 writers execute against

**What this is NOT:**
- Not a writer — produces plans, not prose
- Not a creative decider — synthesizes previous decisions into structure
- Not flexible after Layer 2 begins — sequence locks in as the contract Layer 2 writes against

## PROCESS

### Step 1: Load the 5-Phase Explanation Sequence

```
LOAD from tier1_index.explanation_sequence:

PHASE 1: Problem Amplification
  Position: Before mechanism introduction
  Purpose: Escalate the problem beyond what root cause established
  Role in Narrative: Creates maximum need for the mechanism
  Typical Proportion: 15-25% of total mechanism narrative

PHASE 2: Root Cause Revelation Bridge
  Position: Transition from Skill 16 handoff
  Purpose: Connect root cause narrative to mechanism introduction
  Role in Narrative: The hinge that turns "here's WHY you suffer"
    into "here's WHAT can fix it"
  Typical Proportion: 5-15% of total mechanism narrative
  NOTE: Uses bridge_text from Skill 16 as foundation

PHASE 3: Mechanism Naming and Introduction
  Position: The reveal moment
  Purpose: Name the mechanism with maximum dramatic impact
  Role in Narrative: The most memorable single moment in the narrative
  Typical Proportion: 5-15% of total mechanism narrative

PHASE 4: Mechanism Explanation
  Position: Education section
  Purpose: Make mechanism understandable while maintaining credibility
  Role in Narrative: The substance — where real understanding is built
  Typical Proportion: 30-45% of total mechanism narrative

PHASE 5: Mechanism Proof Integration
  Position: Validation section
  Purpose: Prove mechanism works through woven evidence
  Role in Narrative: Converts understanding into belief
  Typical Proportion: 15-25% of total mechanism narrative
```

### Step 2: Calculate Total Word Budget

```
DETERMINE total mechanism word budget:

  SOURCE 1: structure.word_budget (from Skill 11)
    IF specific mechanism allocation exists: USE IT
    IF only percentage allocation: CALCULATE from total campaign word count

  SOURCE 2: type_parameters.typical_word_range (from 1.1)
    USE as validation range — budget should fall within this range

  SOURCE 3: E-level word scaling
    E2: base range * 0.8 (shorter mechanism sections)
    E3: base range * 1.0 (standard length)
    E4: base range * 1.2 (deeper explanation needed)
    E5: base range * 1.5 (paradigm shift requires longest sections)

  RECONCILE:
    IF structure budget is within type range: USE structure budget
    IF structure budget is below type range minimum:
      WARN: "Structure allocates fewer words than narrative type requires.
        Consider requesting structure adjustment."
      USE type range minimum
    IF structure budget is above type range maximum:
      ACCEPTABLE — more words available, distribute to Phase 4 primarily

  SET total_word_budget = reconciled value
  LOG: "Total mechanism narrative budget: [N] words"
```

### Step 3: Allocate Words Per Phase

```
BASED ON narrative type AND e_level, ALLOCATE words:

DEFAULT PROPORTIONS (adjusted by narrative type):

  scientific_revelation:
    Phase 1: 15% | Phase 2: 10% | Phase 3: 10% | Phase 4: 40% | Phase 5: 25%
    Rationale: Heavy Phase 4 for deep science explanation, strong Phase 5 for proof

  accidental_discovery:
    Phase 1: 15% | Phase 2: 10% | Phase 3: 15% | Phase 4: 35% | Phase 5: 25%
    Rationale: Phase 3 gets more — the discovery story IS the naming reveal

  historical_ancient_secret:
    Phase 1: 15% | Phase 2: 10% | Phase 3: 15% | Phase 4: 35% | Phase 5: 25%
    Rationale: Phase 3 needs space for historical context around naming

  paradigm_shift:
    Phase 1: 20% | Phase 2: 10% | Phase 3: 10% | Phase 4: 40% | Phase 5: 20%
    Rationale: Phase 1 needs extra for framework dismantling, Phase 4 for new worldview

  proprietary_technology:
    Phase 1: 15% | Phase 2: 8% | Phase 3: 12% | Phase 4: 40% | Phase 5: 25%
    Rationale: Phase 4 dominant for technical explanation, Phase 5 for demo/test proof

  identity_driven:
    Phase 1: 20% | Phase 2: 10% | Phase 3: 10% | Phase 4: 30% | Phase 5: 30%
    Rationale: Phase 1 expanded for identity story, Phase 5 expanded for transformation proof

  data_driven_debunking:
    Phase 1: 25% | Phase 2: 10% | Phase 3: 10% | Phase 4: 30% | Phase 5: 25%
    Rationale: Phase 1 biggest — debunking competitors IS the problem amplification

CALCULATE word_count per phase:
  phase_N_words = total_word_budget * phase_N_percentage

VALIDATE:
  - Sum of all phase words == total_word_budget (within 5% tolerance)
  - No phase has fewer than 50 words
  - Phase 4 has at minimum 200 words (mechanism must be explained)
  - Phase 3 has at minimum 50 words (naming cannot be skipped)
```

### Step 4: Assign Simplification Techniques Per Phase

```
LOAD 6 simplification techniques from tier1_index:

TECHNIQUE 1: binary_reduction
  Pattern: Complex biology → simple either/or
  Cognitive Load: Minimal
  Best Phases: 3 (naming), 1 (problem framing)
  Best Types: paradigm_shift, data_driven_debunking

TECHNIQUE 2: single_metaphor_anchor
  Pattern: One metaphor carries entire conceptual weight
  Cognitive Load: Low
  Best Phases: 4 (explanation — THE metaphor), 3 (naming)
  Best Types: scientific_revelation, biological mechanisms

TECHNIQUE 3: layered_analogy_chain
  Pattern: Chain of increasingly specific analogies
  Cognitive Load: Medium (but each layer builds on previous)
  Best Phases: 4 (explanation — progressive deepening)
  Best Types: scientific_revelation (complex mechanisms needing layered understanding)

TECHNIQUE 4: just_think_shortcut
  Pattern: Re-simplification after detail layer
  Cognitive Load: Reset — compresses detail back to simple
  Best Phases: 4 (after detail escalation), 5 (proof summary)
  Best Types: ALL — universal technique

TECHNIQUE 5: relatable_body_knowledge
  Pattern: Connects to something prospect already knows about body
  Cognitive Load: Low (leverages existing knowledge)
  Best Phases: 1 (problem setup), 4 (explanation anchor point)
  Best Types: biological, compound mechanisms

TECHNIQUE 6: numbered_system
  Pattern: Breaks mechanism into numbered components
  Cognitive Load: Structured
  Best Phases: 4 (explanation), 3 (naming with number)
  Best Types: proprietary_technology, system mechanisms

ASSIGN primary and secondary technique per phase:

ASSIGNMENT LOGIC:
  Phase 1 (Problem Amplification):
    IF paradigm_shift: primary = binary_reduction
    IF data_driven_debunking: primary = binary_reduction (works/doesn't work)
    IF scientific_revelation: primary = relatable_body_knowledge
    DEFAULT: primary = relatable_body_knowledge
    Secondary: NONE (Phase 1 is emotional, not educational)

  Phase 2 (Root Cause Bridge):
    ALL TYPES: No simplification technique assigned
    Rationale: Bridge is transitional prose, not explanation
    NOTE: Bridge text comes from Skill 16, technique not needed

  Phase 3 (Naming Reveal):
    IF paradigm_shift: primary = binary_reduction (Theta vs Beta)
    IF proprietary_technology: primary = numbered_system (3 components)
    IF scientific_revelation: primary = single_metaphor_anchor (The Master Switch)
    IF accidental_discovery: primary = single_metaphor_anchor (embedded in story)
    IF historical_ancient_secret: primary = single_metaphor_anchor
    IF identity_driven: primary = single_metaphor_anchor (guru's language)
    IF data_driven_debunking: primary = binary_reduction (old vs new)

  Phase 4 (Explanation):
    ALL TYPES: primary = single_metaphor_anchor (the core image)
    IF explanation_depth == "deep" or "maximum":
      secondary = layered_analogy_chain (progressive deepening)
      tertiary = just_think_shortcut (re-compress after detail)
    IF explanation_depth == "moderate":
      secondary = relatable_body_knowledge
      tertiary = just_think_shortcut
    IF explanation_depth == "embedded in story":
      secondary = relatable_body_knowledge
      tertiary = NONE

  Phase 5 (Proof Integration):
    ALL TYPES: primary = just_think_shortcut (simplify after proof data)
    IF heavy institutional proof: secondary = numbered_system (study 1, study 2...)
    IF personal transformation proof: secondary = relatable_body_knowledge
    DEFAULT: secondary = single_metaphor_anchor (callback to Phase 4 metaphor)
```

### Step 5: Apply Format Variations

```
ADJUST sequence based on output format:

VSL (Video Sales Letter):
  Pacing: Fast cuts between phases, transitions marked by visual/audio shifts
  Phase 1 adjustment: Can be punchier — visual reinforcement carries emotion
  Phase 3 adjustment: Naming reveal can use visual pause (dramatic moment)
  Phase 4 adjustment: Can use animation/visual aids — less text needed
  Phase 5 adjustment: Can show visual proof (brain scans, before/after)
  Special: Text-on-screen moments for key mechanism phrases
  Word budget modifier: 0.85x (visuals carry some explanatory weight)

MAGALOG (Magazine-format direct mail):
  Pacing: Longer sections, reader sets own pace, sidebars available
  Phase 1 adjustment: Can use pull-quotes and callout boxes for emphasis
  Phase 3 adjustment: Naming can get a dedicated callout/headline treatment
  Phase 4 adjustment: Longest — reader will spend time here, use sidebars
    for detailed science, main text for narrative flow
  Phase 5 adjustment: Can use charts, tables, institutional logos
  Special: Sidebars for deep science, main flow stays narrative
  Word budget modifier: 1.1x (print format supports longer reading)

ADVERTORIAL (Editorial-style advertisement):
  Pacing: Must feel like journalism, not sales copy
  Phase 1 adjustment: Problem stated as news/investigation, not fear-mongering
  Phase 3 adjustment: Naming is "journalistic discovery" — reporter finding
  Phase 4 adjustment: Explanation presented as investigative findings
  Phase 5 adjustment: Proof presented as corroborating sources
  Special: Must maintain editorial voice throughout — no hype language
  Word budget modifier: 1.0x (standard length)

SALES LETTER:
  Pacing: Letter-style intimacy, personal address
  Phase 1 adjustment: Problem felt personally by writer ("I know what it's like...")
  Phase 3 adjustment: Naming feels like sharing a personal discovery
  Phase 4 adjustment: Explanation in conversational tone
  Phase 5 adjustment: Proof as "what convinced ME"
  Word budget modifier: 0.95x (slightly tighter than standard)

EMAIL SERIES:
  Pacing: Phases may span multiple emails
  Phase 1: Email 1 — problem + curiosity hook
  Phase 2-3: Email 2 — bridge + naming reveal
  Phase 4: Email 3 — explanation
  Phase 5: Email 4 — proof + bridge to product
  Word budget modifier: 1.2x (repetition needed across emails)
  Special: Each email must standalone AND advance the sequence
```

### Step 6: Align with CPB Chunks

```
LOAD mechanism_chunks from upstream.structure

MAP phases to CPB chunks:
  FOR EACH mechanism_chunk:
    DETERMINE which phase(s) it covers
    ASSIGN chunk_id to phase
    VERIFY word_budget alignment (chunk word count ~= phase word count)

  IF chunks don't align to phases:
    ADJUST phase boundaries to match chunk structure
    LOG: "Phase boundaries adjusted to align with CPB chunk structure"

  IF chunks allocate more words than phase needs:
    DISTRIBUTE excess to Phase 4 (explanation) — always the most flexible phase

  IF chunks allocate fewer words than phase needs:
    COMPRESS Phase 1 or Phase 2 first (least critical for mechanism delivery)
    WARN if Phase 4 is compressed below minimum
```

## QUALITY CRITERIA

1. **Word budgets sum correctly** — all 5 phases add up to total (within 5% tolerance)
2. **No phase below minimum** — Phase 3 >= 50 words, Phase 4 >= 200 words
3. **Simplification assigned to every writing phase** — Phases 1, 3, 4, 5 all have primary technique
4. **Format variations applied** — word modifier and format notes reflect chosen format
5. **CPB chunk alignment verified** — phases map to structural chunks
6. **Aha moment positioned** — approximate word position and percentage calculated
7. **All Layer 1 decisions synthesized** — type, frame, arc, and sequence all coherent

---

## FAILURE MODES

| Failure | Symptom | Resolution |
|---------|---------|------------|
| Total word budget too low | Below type minimum range | Warn, use type minimum, compress Phases 1 and 2 |
| Phase 4 under 200 words | Insufficient explanation space | Redistribute from Phase 1 or Phase 5 |
| Simplification conflict | Phase 4 primary = binary_reduction for scientific_revelation | Override — scientific_revelation always uses single_metaphor_anchor in Phase 4 |
| Format not recognized | format enum invalid | Default to "sales_letter" format variation |
| CPB chunks don't cover all phases | Gaps in structural allocation | Create virtual chunks for uncovered phases, warn structure team |
| Phase proportions don't match type | e.g., Phase 1 at 40% for scientific_revelation | Re-apply type-specific proportions, override structure if needed |
