# Microskill 0.1: Upstream Loader

**Layer:** 0 (Foundation)
**Purpose:** Load all required upstream packages for proof weaving

---

## REQUIRED UPSTREAM PACKAGES

### Primary Inputs

| Package | Source Skill | Required Fields |
|---------|--------------|-----------------|
| `proof-inventory-output.json` | 02-proof-inventory | `elements[]`, `by_category`, `rankings`, `gaps` |
| `structure-package.json` | 08-structure | `sections[]`, `proof_placements`, `section_sequence` |
| `campaign-brief-package.json` | 09-campaign-brief | `tone`, `avatar`, `market_sophistication`, `proof_approach` |

### Secondary Inputs (for correlation)

| Package | Source Skill | Required Fields |
|---------|--------------|-----------------|
| `root-cause-package.yaml` | 03-root-cause | `anchor_phrase`, `villain`, `three_part_structure` |
| `mechanism-package.json` | 04-mechanism | `mechanism_name`, `simplification_metaphor` |
| `promise-package.json` | 05-promise | `primary_promise`, `proof_ceiling` |
| `story-package.json` | 12-story | `protagonist`, `transformation`, `yoda_figure` |

---

## LOADING PROTOCOL

```
STEP 1: Load proof-inventory-output.json
  - EXTRACT: All proof elements with scores
  - EXTRACT: Category distribution (testimonial, demonstration, study, etc.)
  - EXTRACT: Top-ranked elements by stage_adjusted_score
  - EXTRACT: Identified gaps

STEP 2: Load structure-package.json
  - EXTRACT: Section sequence (what order sections appear)
  - EXTRACT: Proof placement markers (where proof is designated)
  - EXTRACT: Section word count targets

STEP 3: Load campaign-brief-package.json
  - EXTRACT: Tone direction (clinical, emotional, urgent)
  - EXTRACT: Avatar details (who we're writing for)
  - EXTRACT: Market sophistication level (1-5)
  - EXTRACT: Proof approach preferences

STEP 4: Load secondary packages for correlation
  - Root cause anchor phrase (for threading)
  - Mechanism name (for threading)
  - Primary promise (for validation proof)
  - Story protagonist (for transformation callbacks)
```

---

## VALIDATION RULES

1. **proof-inventory-output.json MUST contain:**
   - At least 10 proof elements
   - `elements[]` array with full details
   - Scores for each element

2. **structure-package.json MUST contain:**
   - Section sequence
   - At least 6 sections
   - Proof placement indicators

3. **campaign-brief-package.json MUST contain:**
   - Tone direction
   - Avatar reference
   - Sophistication level

**IF ANY PRIMARY PACKAGE IS MISSING → FAIL GATE_0**

---

## FAILURE HANDLING

If packages are missing or incomplete:

```
IF proof-inventory missing:
  → STOP: "Cannot weave proof without proof inventory. Run 01-Proof-Inventory first."

IF structure missing:
  → STOP: "Cannot weave proof without structure. Run 07-Structure first."

IF brief missing:
  → STOP: "Cannot weave proof without campaign brief. Run 09-Campaign-Brief first."

IF secondary packages missing:
  → WARN: "Secondary packages missing. Correlation and threading will be limited."
  → CONTINUE with limitations noted
```
