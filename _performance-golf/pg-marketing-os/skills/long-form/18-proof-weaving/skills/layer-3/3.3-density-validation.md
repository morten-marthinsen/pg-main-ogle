# Microskill 3.3: Density Validation

**Layer:** 3 (Refinement)
**Purpose:** Verify all proof density targets are met across sections

---

## VALIDATION PROCESS

### Step 1: Collect Planned Density (From 1.2)

```yaml
planned_density:
  lead: [target count]
  story: [target count]
  root_cause: [target count]
  mechanism: [target count]
  proof_section: [target count]
  product: [target count]
  offer: [target count]
  close: [target count]
  total: [sum]
```

### Step 2: Count Actual Density

For each section, count drafted proof blocks:

```yaml
actual_density:
  lead:
    planned: [X]
    actual: [Y]
    status: MET | UNDER | OVER
    blocks:
      - [block_id_1]
      - [block_id_2]

  story:
    planned: [X]
    actual: [Y]
    status: MET | UNDER | OVER
    blocks: [...]

  # ... repeat for each section
```

### Step 3: Identify Gaps

```yaml
density_gaps:
  under_target:
    - section: [name]
      shortfall: [number]
      severity: CRITICAL | WARNING | MINOR

  over_target:
    - section: [name]
      excess: [number]
      recommendation: TRIM | KEEP
```

## TESTIMONIAL FATIGUE CHECK

```
IF consecutive_testimonials > 5:
  → FLAG: "Testimonial fatigue risk"
  → RECOMMENDATION: Insert mechanism break

IF testimonials_in_single_section > 8:
  → FLAG: "Section overload"
  → RECOMMENDATION: Distribute or trim

IF testimonial_word_count_without_break > 800:
  → FLAG: "Extended testimonial block"
  → RECOMMENDATION: Add "Think about it" moment
```
