# Layer 2.5: Arena Persona Generation — Proof Weaving Skill

**Version:** 2.1
**Parent Skill:** 18-proof-weaving
**Position:** After Layer 2 (Drafting), Before Layer 3 (Refinement)
**Type:** Multi-Perspective Creative Generation with Human Selection Gate
**Reference:** `skills/ARENA-PERSONA-PANEL.md` (Master Specification v1.3)

> **Arena Mode:** `generative_full_draft` — Competitors write COMPLETE pieces from scratch using upstream packages. Layer 2 draft = reference material, not template. See `ARENA-CORE-PROTOCOL.md` for 3-round execution protocol.

---

## PURPOSE

Generate multiple complete proof block packages through 7 competitors (6 legendary copywriter personas + The Architect), each bringing their distinctive approach to proof presentation, dramatization, and integration. Proof copy must FEEL credible while being maximally persuasive — the 6 personas each interpret "proof that sells" differently. This layer ensures proof blocks pass through diverse proof-presentation philosophies before human selection.

**Why Arena for Proof Weaving:**
- Proof presentation has massive variance in effectiveness — same testimonial can feel hollow or viscerally compelling depending on framing
- The 6 personas each approach proof differently (data-first vs. story-first, institutional vs. street-smart, dramatic vs. understated)
- Testimonial cascades require rhythm and flow that varies by persona voice
- Study citations can be academic (death) or weaponized (conversion) depending on approach
- Before/after contrasts can be factual or emotional depending on persona lens
- Human selection at Gate 2.5 ensures proof blocks match the campaign's established voice and emotional register

### Generation Requirements

Each persona MUST generate complete proof blocks for ALL sections:

1. **Lead Proof Blocks** — Authority anchors, credibility mentions (density: 1-2)
2. **Mechanism Narrative Proof** — Study citations, demonstrations (density: 3-4)
3. **Proof Section Blocks** — Testimonial cascades, before/after progressions (density: 6-8)
4. **Product Introduction Proof** — Single testimonial, results summary (density: 2-3)
5. **Offer Copy Proof** — Value testimonials, guarantee proof (density: 1-2)
6. **Close Proof Blocks** — Transformation reminders, scale callbacks (density: 2-3)
7. **Transition Copy** — Into proof, out of proof, between proof sections

**Target:** Complete proof-weaving-package with all blocks drafted

## 7 JUDGING CRITERIA

All candidates scored against these weighted criteria totaling 100%.

### Criterion 1: Testimonial Cascade Quality (20%) — CRITICAL

**What It Measures:** Whether testimonial sequences flow with rhythm, variety, and cumulative impact using the 8-beat pattern.

**Scoring Rubric:**

| Score | Description |
|-------|-------------|
| 9-10 | Testimonial cascade flows like a conversation. Each testimonial distinct (demographics, outcome, emotional note). 8-beat pattern present. Names/locations specific. Cumulative effect: "All these people can't be wrong." |
| 7-8 | Good flow between testimonials. Most are distinct. Cascade builds impact. Minor rhythm issues in 1-2 transitions. |
| 5-6 | Testimonials present but feel like a list rather than a cascade. Limited variety. Adequate but not compelling. |
| 3-4 | Testimonials feel copy-pasted. Same structure, similar outcomes. No flow rhythm. |
| 1-2 | Generic testimonials without specifics. No cascade structure. Feels manufactured. |

**8-Beat Flow Pattern (Reference):**
1. Hook testimonial (most dramatic)
2. Demographic variety testimonial
3. Skeptic-converted testimonial
4. Timeframe testimonial (speed of results)
5. Unexpected benefit testimonial
6. Long-term testimonial
7. Authority-adjacent testimonial
8. "Just like you" testimonial

**Critical Threshold:** Score < 7 triggers REJECT for proof section testimonials.

---

### Criterion 2: Before/After Contrast Impact (15%)

**What It Measures:** Whether before/after progressions create maximum emotional contrast.

**Scoring Rubric:**

| Score | Description |
|-------|-------------|
| 9-10 | Before state is viscerally painful — reader FEELS the struggle. After state is vividly specific — reader WANTS that outcome. Contrast is maximum. Timeframe adds credibility. "That could be me" reaction triggered. |
| 7-8 | Strong contrast between states. Most details are specific. Good emotional resonance. Minor opportunities for more vivid language. |
| 5-6 | Contrast present but somewhat generic. "Before: struggled. After: better." Adequate but not compelling. |
| 3-4 | Weak contrast. Before and after feel similar in tone. Vague outcomes. No emotional punch. |
| 1-2 | No real contrast. Generic "life changed" without specifics. Fails to create desire. |

**Maximum Contrast Formula:**
```
BEFORE: [Specific pain] + [Emotional consequence] + [Failed attempts]
AFTER: [Specific outcome] + [Emotional liberation] + [New capability]
CONTRAST: [Timeframe] + [Same conditions] + [Only variable was mechanism]
```

---

### Criterion 3: Study Citation Persuasiveness (15%)

**What It Measures:** Whether study citations are persuasively framed (not academic) using the 6-step structure.

**Scoring Rubric:**

| Score | Description |
|-------|-------------|
| 9-10 | Study citation reads like a revelation, not a footnote. Institution name-dropped strategically. Specific finding highlighted. Framing creates "proof mountain" effect. Reader thinks: "This is scientific fact." |
| 7-8 | Study well-framed with institution and finding. Persuasive but not masterful. Minor academic language creeping in. |
| 5-6 | Study mentioned but framing is neutral. Facts stated without persuasive shaping. Adequate credibility, weak persuasion. |
| 3-4 | Academic language dominates. "Suggests," "may indicate," hedge words. Study feels like disclaimer, not proof. |
| 1-2 | Vague study reference ("studies show"). No institution, no year, no finding. Zero credibility. |

**6-Step Citation Structure:**
1. Authority anchor (institution name)
2. Scale indicator (study size, duration)
3. Specific finding (exact number)
4. Relevance bridge (why this matters to reader)
5. Mechanism connection (ties to product)
6. Implication (what this means for reader's outcome)

**Banned Phrases (Auto-Reject):**
- "Studies show" (without citation)
- "Research suggests"
- "May indicate"
- "Could potentially"
- "Appears to"

---

### Criterion 4: Transition Smoothness (15%)

**What It Measures:** Whether transitions into, out of, and between proof sections feel natural rather than jarring.

**Scoring Rubric:**

| Score | Description |
|-------|-------------|
| 9-10 | Transitions feel inevitable, not manufactured. "But don't take my word for it..." flows naturally. Out of proof section creates momentum toward next section. Between-proof transitions vary and maintain interest. |
| 7-8 | Transitions work well. Most feel natural. One or two slightly mechanical moments. |
| 5-6 | Transitions functional but noticeable. Reader feels the "section break" even if not jarring. |
| 3-4 | Transitions feel forced. "Here are some testimonials" type language. Breaks flow. |
| 1-2 | No transitions OR transitions damage credibility. Proof sections feel bolted-on. |

**Approved Transition Patterns:**
- Into proof: "But don't take my word for it..."
- Into proof: "Skeptical? Good. Let me show you the evidence..."
- Out of proof: "And you could be next..."
- Out of proof: "Imagine yourself [timeframe] from now..."
- Between: "[Name] isn't alone. Not by a long shot..."

---

### Criterion 5: Density Target Achievement (15%)

**What It Measures:** Whether proof density targets are met for each section per TIER1 benchmarks.

**Scoring Rubric:**

| Score | Description |
|-------|-------------|
| 9-10 | All density targets met exactly. Distribution feels balanced — not too heavy in any section, not too light anywhere. Proof types varied appropriately per section. |
| 7-8 | 5/6 sections meet targets. One section slightly under or over. Overall balance maintained. |
| 5-6 | 4/6 sections meet targets. Some sections feel proof-light or proof-heavy. |
| 3-4 | 3/6 sections meet targets. Proof distribution unbalanced. Some sections starving, others bloated. |
| 1-2 | Most targets missed. Proof distribution chaotic or absent. |

**TIER1 Density Benchmarks (Reference):**

| Section | Target | Proof Types |
|---------|--------|-------------|
| Lead | 1-2 | Authority anchor, credibility mention |
| Mechanism Narrative | 3-4 | Study citation, demonstration |
| Proof Section | 6-8 | Testimonial cascade, before/after progression |
| Product Introduction | 2-3 | Single testimonial, results summary |
| Offer Copy | 1-2 | Value testimonial, guarantee proof |
| Close | 2-3 | Transformation reminder, scale callback |

---

### Criterion 6: Avatar Resonance (10%)

**What It Measures:** Whether proof subjects match target avatar demographics and struggles.

**Scoring Rubric:**

| Score | Description |
|-------|-------------|
| 9-10 | Proof subjects are mirror images of target avatar. Same age range, similar struggles, relatable circumstances. Reader thinks: "That person is EXACTLY like me." |
| 7-8 | Most proof subjects match avatar. One or two demographic mismatches but overall relatable. |
| 5-6 | Some avatar match. Proof subjects feel generic enough to relate but not specifically targeted. |
| 3-4 | Weak avatar match. Proof subjects feel like different demographic. Reader struggles to relate. |
| 1-2 | No avatar match. Proof subjects are clearly different from target reader. Damages credibility. |

**Resonance Dimensions:**
- Age match (within 10 years)
- Problem match (same primary struggle)
- Lifestyle match (similar circumstances)
- Skepticism match (addressed similar doubts)
- Outcome match (achieved what avatar wants)

---

### Criterion 7: Emotional Register Calibration (10%)

**What It Measures:** Whether proof tone matches campaign-brief emotional direction.

**Scoring Rubric:**

| Score | Description |
|-------|-------------|
| 9-10 | Proof emotional register perfectly matches brief direction (triumphant, relieved, amazed, etc.). Tone consistent across all proof blocks. Feels like same voice wrote everything. |
| 7-8 | Emotional register mostly aligned. One or two proof blocks slightly off-tone. |
| 5-6 | Mixed emotional register. Some proof triumphant, some subdued. Inconsistent feel. |
| 3-4 | Register mismatch. Proof feels emotionally disconnected from rest of campaign. |
| 1-2 | Wrong emotional register entirely. Proof undermines campaign emotional arc. |

**Register Types:**
- **Triumphant:** Bold claims, victory language, celebration tone
- **Relieved:** Finally solved, weight lifted, peace restored
- **Amazed:** Unexpected results, "I couldn't believe it," wonder
- **Validated:** "I knew it would work," confirmation, vindication
- **Grateful:** Changed my life, deeply thankful, emotional

## ARENA EXECUTION PROTOCOL

**See `ARENA-CORE-PROTOCOL.md` for the complete 3-round execution protocol.**

> **Effort Level:** All generation phases use `effort: max`. Critique uses `effort: high`. See CLAUDE.md v3.1 Effort Protocol.
> **Agent Team Mode:** When Agent Teams enabled, each persona runs as a separate teammate agent with full-draft generation in its own 200K context. See `ARENA-CORE-PROTOCOL.md` v2.0 Agent Team Execution Mode.

This skill uses `arena_mode: generative_full_draft`:
- **Competitors write COMPLETE pieces from scratch** — NOT variations of a Layer 2 draft
- Layer 2 draft output = reference material and structural guide, NOT a template
- Upstream packages (root cause, mechanism, promise, big idea, structure) are the primary input
- Competitors are NOT constrained to follow the Layer 2 draft's specific approach
- **7 competitors** (6 personas + The Architect) generating independently
- **Adversarial critique** before scoring (The Critic identifies ONE weakest element per output)
- **Targeted revision** (each competitor fixes their identified weakness)
- **3 rounds** of competition with learning briefs between rounds
- **Post-arena synthesis** (Layer 2.6) creating 2-3 phrase-level hybrids
- **Human selection** from 9-10 candidates (7 pure + 2-3 hybrids)

### Full-Draft Mode Specifics
- Each competitor generates their OWN complete version from the upstream strategic packages
- The Layer 2 draft is available as ONE reference among many, not THE template
- Competitors may take radically different approaches (different structure, different emphasis, different tone)
- This produces TRUE creative diversity, not minor variations

### What Stays Skill-Specific (Below)
- 7 judging criteria with weights (used by both The Critic and the Judge)
- Persona generation instructions for this skill
- Critique-specific guidance for this skill
- Quality thresholds
- Anti-slop enforcement
- Input/output requirements

### Critique-Specific Guidance

**What The Critic should particularly target in Proof Weaving Arena:**
- Testimonial cascades without proper 8-beat flow rhythm
- Before/afters without maximum contrast structure
- Study citations presented as dry academic references (not persuasively framed)
- Proof sections without smooth transitions in/out
- Density targets unmet for any section

## GATE 2.5 SPECIFICATION

**Gate Type:** HUMAN_SELECT (BLOCKING)

**Entry Condition:** Gate 2 passes (all proof blocks drafted)

**Exit Condition:** Human explicitly selects a candidate

**Constraints:**
- No default selection permitted
- No timeout bypass permitted
- Human MUST actively choose
- Selection logged with rationale

**Failure Handling:**
- If all candidates < 8.5 threshold: Present all with warning, allow selection or regeneration
- If human requests regeneration: Execute with specific direction, return to judging
- If human requests hybrid: Validate element compatibility (register, transitions) before assembly

## ANTI-SLOP ENFORCEMENT (PROOF-SPECIFIC)

### Banned Proof Phrases (All Candidates)

**Generic Testimonial Intros:**
- "Here's what people are saying..."
- "Let me share some testimonials..."
- "See what others say..."
- "Customer reviews show..."
- "Many have experienced..."

**Vague Outcomes:**
- "Felt better"
- "Life changed"
- "It works"
- "Amazing results"
- "Incredible transformation"
- "Everything improved"
- "Totally different now"

**Weak Study Language:**
- "Studies show" (without citation)
- "Research suggests"
- "May indicate"
- "Could potentially"
- "Appears to"
- "Tends to"
- "Some evidence shows"

**Generic Credibility:**
- "Experts agree"
- "Doctors recommend" (without name)
- "Research proves" (without source)
- "Science confirms" (without citation)

**AI Telltales:**
- "Journey"
- "Unlock"
- "Transform your life"
- "Discover the secret"
- "Breakthrough"
- "Game-changing"
- "Revolutionary"

### Validation Check

```
FOR each candidate:
    SCAN all proof blocks for banned phrases
    IF found:
        FLAG phrase location
        REQUIRE replacement before scoring
        DO NOT score candidate with poison phrases present
```

## PROOF TYPE COMPLETENESS AUDIT

Before any candidate advances past Arena, verify ALL proof types are properly drafted:

| Proof Type | Minimum Requirement | Audit Check |
|------------|---------------------|-------------|
| Testimonial Cascade | 8-beat pattern present | Count beats, verify variety |
| Before/After | Maximum contrast structure | Check for specific pain + specific outcome |
| Study Citation | 6-step structure | Verify institution, finding, relevance bridge |
| Demonstration | Quantified outcomes | Verify specific numbers |
| Authority | Personal stake established | Check for "living proof" elements |
| Scale | Individual → pattern → scale | Verify progression structure |
| Transformation Reminder | Callbacks to earlier proof | Verify cross-references |

**Audit Result:**
- All types complete and valid → Candidate eligible
- Any type incomplete → FLAG and require completion before selection
