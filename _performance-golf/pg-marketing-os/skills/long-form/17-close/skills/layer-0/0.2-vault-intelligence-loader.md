## Loading Protocol

### Step 1: Load V3 Vault Intelligence File

```
LOCATE: vault-intelligence/close-vault-intelligence.json

VALIDATE:
  - File exists and is valid JSON
  - Contains "version": "v3"
  - Contains "specimens" section with non-empty content
  - Contains "patterns" section with frequency distributions

IF file missing or invalid:
  SET status = FAIL
  LOG "V3 vault-intelligence build must run first"
  HALT
```

### Step 2: Extract Specimens

```
FROM specimens section:
  LOAD all verbatim content specimens
  INDEX by category/type for downstream lookup
  VERIFY verbatim_quote/text fields are present (not summaries)
```

### Step 3: Extract Patterns

```
FROM patterns section:
  LOAD all frequency distributions
  IDENTIFY top-frequency categories for defaults
  CALCULATE niche-specific pattern weightings
```

### Step 4: Extract Statistics

```
FROM statistics section:
  LOAD aggregate metrics
  SET quality baselines for validation gates
```

## Step 5: Specimen Tiering (Phase B)

After loading specimens, classify by quality tier for downstream decomposition:

```
TIERING_PROTOCOL:
  domain: close

  quality_indicator: close_completeness_score
    CALCULATE per specimen:
      - close_sequence[] has >= 3 elements: +2
      - close_type classified: +1
      - future_pacing_text present: +1
      - final_push present: +1
      - ps_sections present (non-null): +1
      - cta_text present: +1
      - source_file present: +1
    MAX_SCORE: 8

  gold_threshold: "Top 10% by close_completeness_score"
  silver_threshold: "Top 25% by close_completeness_score"

  SORT specimens by close_completeness_score DESC
  TOP 10%: SET tier = "Gold"
  NEXT 15%: SET tier = "Silver"
  REMAINDER: SET tier = "Standard"

  OUTPUT tier_metadata:
    gold_specimens: [list]
    silver_specimens: [list]
    gold_count: integer
    silver_count: integer
```

### Step 6: Type-Matched Retrieval

When campaign context is available from upstream (0.1-upstream-loader), filter specimens for relevance:

```
TYPE_MATCHING:
  match_fields:
    - close_type: "Prioritize matching close type (emotional, logical, urgency, etc.)"
    - niche: "Prioritize same-niche specimens"

  MATCH_STRATEGY:
    exact_match: [close_type + niche]
    type_match: [same close_type, any niche]
    cross_niche: [other niche specimens for close strategy transfer]

  RETRIEVAL_OUTPUT:
    matched_gold: [Gold specimens matching campaign context]
    matched_silver: [Silver specimens matching]
    cross_niche_gold: [Gold specimens from other niches]

  IF campaign_context NOT available:
    PASS all Gold + Silver specimens without filtering
```
