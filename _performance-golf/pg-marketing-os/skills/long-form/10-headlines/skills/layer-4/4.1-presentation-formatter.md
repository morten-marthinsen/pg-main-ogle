# 4.1-presentation-formatter.md

> **Layer:** 4 (Selection & Packaging)
> **Skill:** 10-headlines
> **Purpose:** Format final candidates for human review with scores and rationale
> **Inputs:** `final_ranked_candidates` from 3.5
> **Outputs:** Human-readable presentation of options

---

## FUNCTION

Transform the final ranked candidates into a clear, decision-ready presentation for human selection. The human will choose one headline (or provide modifications/alternatives). This skill ensures they have all the information needed to make a good decision.

**Key Principle:** Present OPTIONS, not recommendations. Give the human enough information to choose wisely, but don't over-advocate for any single option.

---

## PRESENTATION FORMAT

### Structure Overview

```
1. CONTEXT SUMMARY (Brief reminder of campaign/Big Idea)
2. TOP 5 CANDIDATES (Ranked by score)
3. COMPARISON TABLE (Quick side-by-side)
4. SELECTION GUIDANCE (What to consider)
5. MODIFICATION INVITATION (Human can edit/create)
```

---

## EXECUTION PROTOCOL

### Step 1: Context Summary

```
WRITE brief context block:

  CAMPAIGN: [Project name]
  BIG IDEA: [One-sentence Big Idea expression]
  FORMAT: [VSL/Print/Advertorial]
  TARGET: [Brief avatar description]
  THRESHOLD: [Minimum score achieved]
```

### Step 2: Format Each Candidate

```
FOR each of top 5 candidates (ranked):

  FORMAT as:
  ┌──────────────────────────────────────────────────────────┐
  │ OPTION [RANK]: [CANDIDATE_ID]                            │
  │ Score: [WEIGHTED_TOTAL]/10                               │
  ├──────────────────────────────────────────────────────────┤
  │ HEADLINE:                                                │
  │ "[Full headline text]"                                   │
  │                                                          │
  │ SUBHEADLINE:                                             │
  │ "[Full subheadline text]"                                │
  ├──────────────────────────────────────────────────────────┤
  │ SCORES:                                                  │
  │ Clarity: X | Curiosity: X | Specificity: X               │
  │ Emotion: X | Schema: X | Lead Connection: X              │
  ├──────────────────────────────────────────────────────────┤
  │ STRENGTHS:                                               │
  │ • [Strength 1]                                           │
  │ • [Strength 2]                                           │
  │                                                          │
  │ POTENTIAL CONCERNS:                                      │
  │ • [Concern 1]                                            │
  ├──────────────────────────────────────────────────────────┤
  │ RECOMMENDED LEAD: [Lead type] because [reason]           │
  └──────────────────────────────────────────────────────────┘
```

### Step 3: Create Comparison Table

```
FORMAT quick comparison:

| Rank | Score | Type | Key Strength | Key Risk |
|------|-------|------|--------------|----------|
| 1    | 8.65  | Curiosity | High emotional impact | Schema distance medium |
| 2    | 8.40  | Benefit | Clear promise | Less surprising |
| 3    | 8.25  | Story | Strong narrative | Needs story lead |
| 4    | 8.10  | Warning | High urgency | May feel aggressive |
| 5    | 7.80  | Proof | Credibility-first | Lower curiosity |
```

### Step 4: Selection Guidance

```
PROVIDE guidance without over-directing:

  CONSIDER selecting based on:

  • AUDIENCE SOPHISTICATION
    - More sophisticated? Higher schema distance may work better
    - Less sophisticated? Clarity over cleverness

  • PERSONAL BRAND VOICE
    - Does this fit how you/client normally communicate?
    - Would the authority figure "say" this headline?

  • FORMAT FIT
    - VSL: Shorter, punchier often wins
    - Print: Can handle more complexity
    - Email: Curiosity gaps drive opens

  • GUT CHECK
    - Which one would make YOU stop scrolling?
    - Which one feels most like THIS campaign?
```

### Step 5: Modification Invitation

```
CLOSE with invitation:

  YOUR OPTIONS:

  □ Select one of the 5 options as-is
  □ Select one and request modifications
  □ Provide your own headline for scoring
  □ Request additional options (different direction)

  If modifying or creating, I will score your version
  using the same rubric for comparison.
```

## ANTI-PATTERNS (FORBIDDEN)

1. **Over-Advocacy**
   - Don't push one option as "the clear winner"
   - Human choice respects human judgment

2. **Information Overload**
   - Keep each option to ~100 words of explanation
   - More detail on request, not by default

3. **Missing Context**
   - Human needs Big Idea reminder to judge fit
   - Always include context summary

4. **No Modification Path**
   - Always invite human to modify or create
   - This isn't a multiple choice test
