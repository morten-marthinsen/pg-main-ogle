# 3.5-final-scorer.md

> **Layer:** 3 (Refinement)
> **Skill:** 10-headlines
> **Purpose:** Re-score all refined candidates and validate top performer ≥ 7.5/10
> **Inputs:** Refined candidates with enhancements, subheadlines, and lead connections
> **Outputs:** Final scored candidates, top 5 ranked for human selection

---

## FUNCTION

Apply the SAME 6-dimension scoring rubric from 2.5 to the REFINED candidates. This validates that refinement improved (not degraded) quality, and confirms at least one candidate achieves the ≥ 7.5 threshold for Layer 4.

**Key Principle:** Refinement should IMPROVE scores. If a candidate's score dropped after enhancement, the enhancement was wrong. This skill catches over-enhancement before human review.

---

## SCORING RUBRIC (Same as 2.5)

### Dimension Weights (Total = 100%)

| Dimension | Weight | What It Measures |
|-----------|--------|------------------|
| Clarity | 15% | How clearly the promise is communicated |
| Curiosity | 25% | How strongly it creates need to know more |
| Specificity | 20% | How concrete and specific the elements are |
| Emotional Impact | 20% | How strongly it triggers emotional response |
| Schema Distance | 10% | How much it violates expectations (6-8 optimal) |
| Lead Connection | 10% | How well it sets up the lead |

---

## EXECUTION PROTOCOL

### Step 1: Collect Refined Candidates

```
GATHER all refined candidates from Layer 3:
  - Power-word enhanced versions (from 3.1)
  - Specificity enhanced versions (from 3.2)
  - Complete headline units with subheadlines (from 3.3)
  - Lead connection mappings (from 3.4)

CREATE final scoring pool (top 5 refined candidates)
```

### Step 2: Re-Score Each Candidate

```
FOR each refined candidate:

  SCORE headline + subheadline as UNIT:

    clarity_score = [1-10]
      - Does the UNIT clearly communicate the promise?
      - Consider headline AND subheadline together

    curiosity_score = [1-10]
      - Does the UNIT create irresistible curiosity?
      - Has refinement added or reduced curiosity?

    specificity_score = [1-10]
      - Are there specific numbers, names, timeframes?
      - Did 3.2 enhancement improve this?

    emotional_impact_score = [1-10]
      - Does the UNIT trigger strong emotion?
      - Did 3.1 power words improve this?

    schema_distance_score = [1-10]
      - Is the violation level optimal (6-8)?
      - Not too expected, not too unbelievable

    lead_connection_score = [1-10]
      - Is there a clear path to the lead?
      - Did 3.4 mapping reveal strong connection?

  CALCULATE weighted_total:
    weighted_total = (
      clarity_score * 0.15 +
      curiosity_score * 0.25 +
      specificity_score * 0.20 +
      emotional_impact_score * 0.20 +
      schema_distance_score * 0.10 +
      lead_connection_score * 0.10
    )
```

### Step 3: Compare to Pre-Refinement Scores

```
FOR each candidate:

  COMPARE scores:
    pre_refinement_total (from 2.5)
    post_refinement_total (from this scoring)

  CALCULATE delta = post - pre

  FLAG any candidate where delta < 0:
    - Refinement made it WORSE
    - Review what enhancement caused degradation
    - Consider reverting to pre-refinement version

  EXPECTED: Most deltas should be +0.5 to +1.5
```

### Step 4: Validate Threshold

```
REQUIREMENT: At least ONE candidate must score ≥ 7.5

IF top candidate < 7.5:
  FAIL GATE
  OPTIONS:
    - Return to Layer 2 for more candidates
    - Review refinement approach
    - Lower threshold with justification (not recommended)

IF top candidate ≥ 7.5:
  PASS GATE
  Proceed to Layer 4
```

### Step 5: Final Ranking

```
RANK all candidates by weighted_total (descending)

ENSURE diversity in top 5:
  - Not all from same generator source
  - Not all using same pattern type
  - Variety of approaches for human choice

CREATE final_ranked_candidates for Layer 4
```

## QUALITY GATES

### Gate 1: Threshold Achievement
```
REQUIRE: top_candidate.weighted_total >= 7.5
IF FAIL: Cannot proceed to Layer 4
```

### Gate 2: No Degradation
```
REQUIRE: No candidate has negative improvement_delta
IF FAIL: Review and revert problematic refinements
```

### Gate 3: Scoring Consistency
```
REQUIRE: All 6 dimensions scored for all candidates
IF FAIL: Complete missing scores before ranking
```


**DELTA:** -1.7 (degradation — too many power words killed credibility)

**ACTION:** Revert to pre-refinement, try lighter enhancement

---

## ANTI-PATTERNS (FORBIDDEN)

1. **Score Inflation**
   - Don't inflate scores to pass threshold
   - If headline isn't good enough, return to generation

2. **Ignoring Degradation**
   - Negative deltas are signals, not noise
   - Always investigate what went wrong

3. **Threshold Lowering**
   - 7.5 is the minimum for human review
   - Lowering it produces weak headlines

4. **Inconsistent Rubric**
   - Same rubric as 2.5, same standards
   - Don't get easier/harder between scorings
