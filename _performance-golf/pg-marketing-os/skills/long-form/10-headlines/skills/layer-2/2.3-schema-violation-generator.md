# 2.3-schema-violation-generator.md

> **Layer:** 2 (Generation)
> **Skill:** 10-headlines
> **Purpose:** Generate headline candidates optimized for schema distance (expectation violation)
> **Inputs:** `big_idea_distilled`, `schema_distance_target`, `identified_violation`
> **Outputs:** 2-3 high-schema-distance headline candidates

---

## FUNCTION

Generate headline candidates that VIOLATE reader expectations in a calculated way. Schema distance measures how much a headline deviates from what the reader expects — the sweet spot is 6-8 (surprising but not confusing).

**Key Principle:** The best headlines make you do a double-take. They say something that shouldn't be true, but might be. This skill engineers that cognitive dissonance.

---

## SCHEMA DISTANCE FRAMEWORK

### The RSF Scale (1-10)

| Score | Description | Reader Response | Example |
|-------|-------------|-----------------|---------|
| 1-3 | Expected | "Yeah, I knew that" | "Eat Healthy, Live Longer" |
| 4-5 | Mildly interesting | "Hm, tell me more" | "The Superfood Doctors Recommend" |
| 6-7 | Surprising | "Wait, really?" | "Why Your Healthy Diet Is Making You Sick" |
| 8-9 | Shocking | "That can't be right... can it?" | "I HAVEN'T HAD A GLASS OF WATER IN 20 YEARS" |
| 10 | Unbelievable | "That's impossible/crazy" | Often crosses into incredibility |

### Target Zone: 6-8

Headlines in this zone:
- Stop the scroll
- Create cognitive dissonance
- Demand resolution (reading continues)
- Remain plausible enough to engage

## VIOLATION TYPES

### Type 1: Factual Violation
Challenge what they believe to be TRUE.

```
PATTERN: State the opposite of accepted fact
EXAMPLE: "Why Drinking 8 Glasses of Water is Killing You"
MECHANISM: Reader believes X is healthy → You claim X is harmful
RESOLUTION: Provide new mechanism that explains the flip
```

### Type 2: Magnitude Violation
Promise MORE than they think possible.

```
PATTERN: Claim results beyond expected range
EXAMPLE: "Add 50 Yards to Your Drive in 10 Minutes"
MECHANISM: Reader expects 10 yards in weeks → You promise 50 in minutes
RESOLUTION: Reveal mechanism that makes magnitude possible
```

### Type 3: Effort Violation
Promise LESS effort than they expect.

```
PATTERN: Eliminate the "hard part" they assume is required
EXAMPLE: "How I Lost 40 Pounds Without Diet or Exercise"
MECHANISM: Reader believes weight loss = diet + exercise → You remove both
RESOLUTION: Reveal alternative mechanism (sleep, hormones, etc.)
```

### Type 4: Identity Violation
Challenge WHO they think succeeds.

```
PATTERN: Show unexpected person achieving result
EXAMPLE: "How a 69-Year-Old Doctor Has the Body of a 30-Year-Old"
MECHANISM: Reader believes age = decline → You show age reversal
RESOLUTION: Reveal what this person does differently
```

### Type 5: Temporal Violation
Challenge WHEN results happen.

```
PATTERN: Compress expected timeframe dramatically
EXAMPLE: "Speak Fluent Spanish in 10 Days"
MECHANISM: Reader believes fluency = years → You promise days
RESOLUTION: Reveal accelerated learning method
```

### Type 6: Source Violation
Challenge WHERE the solution comes from.

```
PATTERN: Reveal unexpected origin
EXAMPLE: "A Cream for Stretch Marks May Be Better Than Botox"
MECHANISM: Reader expects wrinkle cream from wrinkle research → You show accidental discovery
RESOLUTION: Explain how the mechanism transfers
```

## EXECUTION PROTOCOL

### Step 1: Identify Violation Opportunity

```
LOAD schema_distance_calibrator_output from Layer 1

EXTRACT:
  - expectation_to_violate
  - violation_type
  - target_schema_distance

IF not provided, DERIVE from Big Idea:
  ASK: "What does the reader currently believe that this Big Idea contradicts?"
  ASK: "What 'impossible' thing does this mechanism make possible?"
  ASK: "What expected effort/time does this eliminate?"
```

### Step 2: Craft Violation Statement

```
FOR identified expectation:

  INVERT the expectation into a headline claim:

  EXPECTATION: "You need to exercise to lose weight"
  VIOLATION: "Lose weight without exercise"

  EXPECTATION: "Learning takes time"
  VIOLATION: "Master [skill] in days, not years"

  EXPECTATION: "Doctors know best"
  VIOLATION: "What your doctor doesn't know about [condition]"

  CALIBRATE to target schema distance:
    IF target = 6-7: Moderate violation, clearly plausible
    IF target = 8-9: Strong violation, requires proof support
```

### Step 3: Add Credibility Anchor

```
CRITICAL: High schema distance REQUIRES credibility support

FOR each violation headline:

  ADD one of:
    - Authority anchor: "Harvard Doctor Reveals..."
    - Proof anchor: "Backed by 47 Studies..."
    - Social anchor: "Why 2.3 Million Americans..."
    - Mechanism anchor: "The [Named Mechanism] That..."
    - Specificity anchor: Exact numbers, names, dates

  The more extreme the violation, the stronger the anchor needed.
```

### Step 4: Test Plausibility Threshold

```
FOR each candidate:

  ASK: "Could a skeptical reader believe this MIGHT be true?"

  IF answer is NO:
    - Reduce schema distance
    - Add stronger credibility anchor
    - Soften the violation claim

  IF answer is "intriguing but doubtful":
    - PERFECT. This is the target zone.

  IF answer is "obviously true":
    - Increase schema distance
    - Make violation more dramatic
```

## CALIBRATION EXAMPLES

### Too Low (Schema Distance 3-4)
```
"Discover the Health Benefits of Chlorella"
PROBLEM: No violation. Reader expected health claims.
FIX: "Why the 'Healthiest' Foods Are Destroying Your Gut"
```

### Sweet Spot (Schema Distance 7)
```
"I HAVEN'T HAD A GLASS OF WATER IN 20 YEARS"
WHY IT WORKS: Violates universal advice, but specific timeframe makes it feel real
CREDIBILITY: First-person testimony, specific years
```

### Too High (Schema Distance 10)
```
"Cure Cancer With This One Weird Trick"
PROBLEM: Crosses into unbelievable, triggers scam detection
FIX: "The Compound Found in [Food] That Starves Cancer Cells"
```

---

## ANTI-PATTERNS (FORBIDDEN)

1. **Clickbait Without Substance**
   - Schema violation must be RESOLVABLE in the copy
   - Don't promise what you can't deliver

2. **Violation Without Anchor**
   - High schema distance + no credibility = scam signal
   - Always pair violation with believability element

3. **Over-Promising**
   - If your mechanism can't support the claim, reduce it
   - Better to under-promise than over-promise

4. **Arbitrary Violation**
   - The violation must connect to the Big Idea
   - Don't violate random expectations for shock value
