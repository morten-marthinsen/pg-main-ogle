## FUNCTION

Generate headline candidates by loading TYPE-INDEXED SPECIMENS from the vault and using them as statistical attractors. This is NOT formula application — it's pattern absorption and creative riffing.

**Key Principle:** Specimens reshape probability distributions. When you load elite headlines into context, your generations inherit their structural DNA without copying them.

## EXECUTION PROTOCOL

### Step 1: Specimen Loading

```
FOR each selected_specimen_type:

  READ verbatim headline from 0.2.6-curated-gold-specimens.md
  READ pattern DNA analysis
  READ "When to Use" guidance

  HOLD in active context as STATISTICAL ATTRACTOR

  DO NOT:
    - Summarize the specimen
    - Paraphrase the specimen
    - Skip loading "because you know the pattern"

  The VERBATIM text must be in context during generation.
```

### Step 2: Pattern DNA Absorption

```
FOR each loaded specimen:

  ANALYZE structural elements:
    - Opening device (question, statement, command, etc.)
    - Word count and rhythm
    - Specificity placement (numbers, names, timeframes)
    - Emotional trajectory
    - Curiosity gap technique
    - Authority integration method

  IDENTIFY what makes this specimen ELITE:
    - Why did this headline work?
    - What expectation does it violate?
    - What promise does it compress?
    - What emotion does it trigger?
```

### Step 3: Big Idea Integration

```
LOAD big_idea_distilled from Layer 1

FOR each specimen pattern:

  ASK: "How would THIS Big Idea be expressed using THIS pattern's DNA?"

  GENERATE candidate by:
    1. Taking the STRUCTURAL PATTERN from specimen
    2. Injecting THIS campaign's Big Idea content
    3. Preserving the specimen's rhythm and flow
    4. Adapting specificity to available proof points

  VERIFY: Result is NEW headline, not copied specimen
```

### Step 4: Creative Riffing (CRITICAL)

```
AFTER generating "faithful" adaptations:

  PUSH further with creative riffs:
    - What if we COMBINED patterns from two specimens?
    - What if we INVERTED the structure?
    - What if we made it SHORTER/LONGER?
    - What if we changed the EMOTIONAL ANGLE?

  GENERATE 1-2 "wild card" candidates that:
    - Draw from specimen DNA
    - But take creative liberties
    - Push schema distance higher
```

## QUALITY GATES

### Gate 1: Specimen Loading Verification
```
BEFORE generation begins:
  [ ] At least 3 specimens loaded into active context
  [ ] Verbatim text confirmed (not summarized)
  [ ] Pattern DNA analyzed for each
```

### Gate 2: Differentiation Check
```
FOR each candidate:
  [ ] Candidate is NOT identical to specimen
  [ ] Candidate expresses THIS Big Idea (not generic)
  [ ] Candidate could NOT be swapped to different product
```

### Gate 3: Pattern Fidelity Check
```
FOR each candidate:
  [ ] Structural DNA from specimen is visible
  [ ] Elite qualities preserved (specificity, rhythm, hook)
  [ ] Not just template fill-in
```

## ANTI-PATTERNS (FORBIDDEN)

1. **Specimen Copying** — Using specimen verbatim with word swaps
   - The specimen is an ATTRACTOR, not a template
   - Generate NEW headlines inspired by the pattern

2. **Specimen Skipping** — "I know what that pattern looks like"
   - NO. Load the VERBATIM text every time
   - Statistical attraction requires actual exposure

3. **Generic Adaptation** — Headlines that could fit any product
   - Every candidate must EXPRESS this specific Big Idea
   - Specimen pattern + Generic content = Failure

4. **Pattern Flattening** — Losing what made specimen elite
   - If specimen had 4 specificity elements, yours should too
   - If specimen had dramatic rhythm, preserve it
