# 1.3-curiosity-architect.md

> **Layer:** 1 (Architecture)
> **Skill:** 10-headlines
> **Purpose:** Design the curiosity gap and resolution timing
> **Outputs:** `curiosity_architecture` with gap design and resolution plan

---

## FUNCTION

Engineer the specific curiosity gap the headline creates — what information the reader now needs to know — and design when/how that gap gets resolved in the lead and body.

---

## CURIOSITY GAP TYPES

### Primary Gap Types

| Gap Type | What Reader Needs to Know | Tension Level | Resolution Location |
|----------|---------------------------|---------------|---------------------|
| `mechanism` | "What IS this thing that works?" | High | Lead/Body reveal |
| `result` | "What results did they achieve?" | Medium-High | Lead/Early body |
| `identity` | "Who is this person/expert?" | Medium | Lead (authority build) |
| `enemy` | "What's been sabotaging me?" | High | Root cause section |
| `secret` | "What information am I missing?" | Very High | Throughout, full reveal late |
| `story` | "What happened next?" | Medium | Story section |
| `number` | "What are the [X] things?" | Medium | Body enumeration |
| `how` | "How does this work?" | High | Mechanism section |

### Gap Combinations

Powerful headlines often combine multiple gaps:

```
"Better Than Botox?"
├── Mechanism gap: What IS this alternative?
├── Result gap: How much better?
└── How gap: How can a cream beat an injection?

"The 3-Second Swing Fix Tiger's Coach Won't Share"
├── Mechanism gap: What's the fix?
├── Identity gap: Who is this coach?
├── Secret gap: Why won't he share it?
└── How gap: How can it be 3 seconds?
```

## GAP ENGINEERING PROTOCOL

### Step 1: Identify Primary Gap

```
ANALYZE distilled_big_idea:
  IF mechanism is novel/proprietary:
    PRIMARY_GAP = mechanism ("What is it?")

  IF promise is extraordinary:
    PRIMARY_GAP = result ("How is that possible?")

  IF authority is key differentiator:
    PRIMARY_GAP = identity ("Who discovered this?")

  IF enemy/villain is central:
    PRIMARY_GAP = enemy ("What's been holding me back?")

  IF contrarian angle:
    PRIMARY_GAP = secret ("What don't I know?")
```

### Step 2: Layer Secondary Gaps

```
FOR each secondary_gap_opportunity:
  CHECK compatibility with primary_gap
  CHECK if adds tension without confusion
  CHECK if resolvable within content structure

ADD compatible secondary gaps (max 2)
```

### Step 3: Calibrate Tension Level

```
TENSION_FACTORS:
  - Relevance to avatar pain: +2 if direct hit
  - Specificity of tease: +1 if concrete
  - Schema distance: +1 if surprising
  - Promise magnitude: +1 if extraordinary
  - Time pressure: +1 if urgency present

IDEAL_TENSION: 7-8 (compelling but not overwhelming)
```

### Step 4: Design Resolution Timing

| Timing | Effect | Best For |
|--------|--------|----------|
| `instant` | Immediate payoff, builds trust | Simple mechanisms, direct benefits |
| `within_lead` | Quick partial resolution, opens new gaps | Most VSLs, story leads |
| `delayed` | Sustained tension, deeper engagement | Complex mechanisms, long-form |
| `gradual` | Progressive revelation, builds anticipation | Sophisticated audiences |
| `never_fully` | Maintains mystery, drives action | Premium offers, scarcity plays |

## FAILURE MODES

| Failure | Diagnosis | Remediation |
|---------|-----------|-------------|
| Gap too weak | Tension < 5 | Increase specificity or stakes |
| Gap too confusing | Multiple unrelated gaps | Focus on primary, cut secondary |
| Resolution too fast | No sustained tension | Delay full reveal |
| Resolution never comes | Reader frustration | Ensure clear payoff path |
| Gap not relevant | Doesn't connect to avatar | Tie to primary pain/desire |
