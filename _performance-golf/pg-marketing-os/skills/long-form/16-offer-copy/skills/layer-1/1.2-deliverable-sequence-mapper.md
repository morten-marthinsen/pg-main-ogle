## PURPOSE

The Deliverable Sequence Mapper takes the classified offer format and maps the exact D-F-W-B-P sequence for every deliverable and every bonus. For each item, it determines: what the Feature is (factual description), why it matters (the Why), what benefit the prospect gains (the Benefit), and what evidence supports it (the Proof element). It also determines the ORDER in which deliverables and bonuses should be presented — which item leads, which follows, and why.

Ordering is strategic, not random. The first deliverable establishes the foundation of what the prospect is getting. Subsequent deliverables build on that foundation. The last deliverable before the product summary should be the one that ties everything together. Bonuses are ordered by value contribution — the highest-value bonus first to maximize perceived generosity, or the most emotionally compelling bonus first to build excitement.

Without this mapper, the writer microskills (2.1 and 2.2) would present deliverables in whatever order they were loaded, with whatever Feature/Why/Benefit/Proof elements they could improvise. This produces inconsistent depth, illogical ordering, and missed proof opportunities. The mapper ensures every item is thoroughly prepared for writing.

**This microskill IS:**
- The D-F-W-B-P element identifier for every deliverable and bonus in the offer
- A strategic ordering engine for deliverable presentation sequence
- A strategic ordering engine for bonus presentation sequence
- A Feature extractor — determining the factual description for each item
- A Why generator — determining the reason each item matters to the prospect

- A value demonstration designer — that is 1.3

## PROCESS

### Step 1: Map D-F-W-B-P Elements for Each Deliverable

1. For each deliverable, extract or generate the five D-F-W-B-P elements:

**D (Deliverable) — Name and Introduction:**
- Use the confirmed deliverable name exactly as provided
- Determine the ordinal position: "The first thing you get is..." / "The second thing..."
- Format: "[Product Name] / [Deliverable Name]"

**F (Feature) — What It Is:**
- Extract from the deliverable description and features list
- Focus on CONCRETE SPECIFICS: quantities, components, formats, durations
- Examples: "a comprehensive 12-module video training," "30-day supply of 8 clinically studied ingredients," "a step-by-step blueprint covering all 7 phases"
- Anti-pattern: Do NOT use vague features like "everything you need" or "a powerful system"

**W (Why) — Why It Matters:**
- Connect the feature to the prospect's situation or problem
- Answer: "Why should the prospect care about this specific feature?"
- Pattern: "This matters because..." / "This is critical because..." / "Most [prospects] [problem], and this..."
- Must reference the prospect's current struggle or knowledge gap

**B (Benefit) — What They Gain:**
- Translate the feature into a personal outcome for the prospect
- Answer: "What changes in the prospect's life because of this?"
- Pattern: "Which means you..." / "So you can..." / "Giving you..."
- Must be specific and measurable when possible

**P (Proof) — Evidence:**
- Select from proof inventory based on proof_type_assigned (from 1.1)
- Match the most relevant proof asset to THIS specific deliverable
- Types: clinical data reference, testimonial excerpt, authority endorsement, statistical evidence, case study reference
- If no deliverable-specific proof exists: use general product proof or mechanism proof

2. Rate D-F-W-B-P completeness for each deliverable:
   - 5/5 elements identified: READY
   - 4/5 elements (missing proof): READY with generic proof note
   - 3/5 or fewer: FLAG — deliverable needs more information

### Step 2: Map D-F-W-B-P Elements for Each Bonus

3. Apply the same D-F-W-B-P mapping to each bonus, with bonus-specific considerations:
   - Bonuses need dollar values stated ("a $97 value")
   - Bonuses should feel like GIFTS, not just more product
   - Why element should emphasize "above and beyond" positioning
   - Proof can be lighter for bonuses (brief rather than full citations)

4. For each bonus, also determine:
   - Stated dollar value (from confirmed data)
   - Value justification: why is this bonus worth $[amount]?
   - Standalone value: could the prospect buy this separately?

### Step 3: Determine Deliverable Presentation Order

5. Order deliverables based on these strategic principles:

**Principle 1: Lead with the Core Product**
- The primary deliverable (the main thing they are buying) goes FIRST
- For supplements: the supplement itself
- For digital: the core training/system
- For physical: the physical product
- For info: the main course/book

**Principle 2: Build Logical Progression**
- Each subsequent deliverable should build on the previous
- Foundation items before advanced items
- Setup before execution
- Theory before application (for info products)

**Principle 3: End with the Connector**
- The last deliverable before the product summary should be the one that TIES everything together
- It should bridge to the product summary line: "With everything you're getting today..."
- Often this is a support/maintenance/ongoing access item

6. Record the final deliverable order with position numbers and ordering rationale

### Step 4: Determine Bonus Presentation Order

7. Order bonuses using these strategies:

**Strategy A: Value-First (recommended for high-value bonuses)**
- Highest dollar-value bonus presented first
- Creates immediate perception of generosity
- Each subsequent bonus builds the stack higher

**Strategy B: Emotional-First (recommended for transformation products)**
- Most emotionally compelling bonus first
- The bonus that most directly addresses the prospect's deepest desire or fear
- Dollar value is secondary to emotional impact

**Strategy C: Logical-First (recommended for information products)**
- Most logically connected bonus to the core product first
- "You'll need this right away" bonus leads
- Creates a sense of completeness

8. Select the ordering strategy based on:
   - Product format (from 1.1 classification)
   - Bonus count (3+ bonuses favors value-first)
   - Bonus value range (wide range favors value-first, narrow range favors emotional-first)
9. Record the final bonus order with position numbers and ordering rationale

### Step 5: Map Connection Points Between Items

10. For each transition between deliverables, note the connection:
    - How does deliverable N lead into deliverable N+1?
    - What linking concept connects them?
    - What SIN transition language pattern fits?

11. For the deliverable-to-summary transition:
    - How does the last deliverable lead into "With everything you're getting today..."?
    - What totaling language is appropriate?

12. For the summary-to-bonus transition:
    - What transition language bridges from product summary to first bonus?
    - What emotional shift occurs (from "you have everything" to "but there's MORE")?

13. For each transition between bonuses:
    - Vary the transition language (avoid repeating "and you also get...")
    - Build momentum with each successive bonus

### Step 6: Validate Sequence Completeness

14. Verify:
    - Every deliverable has all 5 D-F-W-B-P elements (or is flagged)
    - Every bonus has all 5 D-F-W-B-P elements plus stated value
    - Deliverable order is strategic (not default/load order)
    - Bonus order is strategic (not default/load order)
    - Connection points identified for all transitions
    - Ordering rationale documented for both deliverables and bonuses
    - Total items match confirmed counts from 0.4

15. Calculate sequence readiness score:
    - Count items with complete D-F-W-B-P (5/5): full_ready
    - Count items with near-complete D-F-W-B-P (4/5): partial_ready
    - Count items below 4/5: flagged
    - Score = (full_ready + 0.75 * partial_ready) / total_items

## FAILURE MODES

### Failure Mode 1: Feature-Benefit Conflation
- **Symptom:** Feature and Benefit say the same thing in different words
- **Cause:** Mapper did not distinguish between what the deliverable IS and what the prospect GAINS
- **Prevention:** Feature answers "What is this?" — Benefit answers "How does my life change?" These MUST be different
- **Recovery:** Rewrite: make Feature factual/concrete (quantities, formats, components) and Benefit personal/emotional (outcomes, feelings, capabilities)

### Failure Mode 2: Generic Why Elements
- **Symptom:** Why element says "This is important because it helps you succeed"
- **Cause:** Mapper used generic importance language instead of connecting to specific prospect pain/gap
- **Prevention:** Every Why must reference a specific problem, frustration, or knowledge gap the prospect has
- **Recovery:** Rewrite Why elements to start with the prospect's specific situation: "Most [prospects] struggle with [specific thing], and this..."

### Failure Mode 3: All Items Get Full Depth
- **Symptom:** Minor bonuses have full D-F-W-B-P depth, making the offer section excessively long
- **Cause:** Mapper did not respect depth assignments from 1.1
- **Prevention:** Check depth assignment for each item and scale D-F-W-B-P detail accordingly
- **Recovery:** Trim low-priority items to medium or brief depth; combine W and B for brief items

### Failure Mode 4: No Proof Available for Key Deliverables
- **Symptom:** Primary deliverable flagged with no proof element
- **Cause:** Proof inventory does not contain deliverable-specific evidence
- **Prevention:** Step 1 item P allows fallback to general product or mechanism proof
- **Recovery:** Use mechanism proof ("backed by the same [mechanism] research that..."), authority proof ("developed by [authority]"), or social proof ("already used by [N] people")

### Failure Mode 5: Illogical Deliverable Ordering
- **Symptom:** Advanced module listed before foundational module; bonus references deliverable not yet introduced
- **Cause:** Ordering did not consider logical dependency between items
- **Prevention:** Step 3, Principle 2 — check for logical prerequisites between deliverables
- **Recovery:** Re-order to ensure each item only references concepts already introduced by preceding items

## EXAMPLES

### Example 1: Supplement Stack — 2 Deliverables, 3 Bonuses

**Product:** Resurge (anti-aging sleep supplement)

**Deliverable Sequence:**
```yaml
deliverables:
  - position: 1
    name: "Resurge — 30-Day Supply"
    depth: full
    dfwbp:
      deliverable: "The first thing you get is a full 30-day supply of Resurge"
      feature: "Each bottle contains 60 vegetarian capsules with all eight clinically studied sleep and metabolic support ingredients in their exact research-backed dosages"
      why: "This matters because most sleep supplements use proprietary blends that hide the actual amounts — you never know if you're getting enough of each ingredient to make a difference. With Resurge, every single dosage matches the clinical studies"
      benefit: "Which means from the very first night, your body has the precise nutritional support it needs to shift from shallow, aging sleep into deep, restorative sleep — the kind that activates your body's natural metabolic regeneration process"
      proof: "In a peer-reviewed study involving 120 participants, the core ingredients in Resurge improved deep sleep duration by 41% within the first two weeks"
    completeness: ready
    ordering_rationale: "Primary deliverable — the core product that defines the offer"

  - position: 2
    name: "Deep Sleep Optimization Guide"
    depth: medium
    dfwbp:
      deliverable: "The second thing you get is the Deep Sleep Optimization Guide"
      feature: "A step-by-step companion guide covering sleep environment setup, evening routines, and timing protocols"
      why: "Because the supplement works best when your sleep habits support it"
      benefit: "So you maximize the deep sleep benefits from day one without guesswork"
      proof: "Based on the same sleep hygiene protocols used in the clinical research"
    completeness: ready
    ordering_rationale: "Supports the primary product — logical next step after the supplement"
```

**Bonus Sequence (value-first ordering):**
```yaml
bonuses:
  - position: 1
    name: "Quick Start Guide"
    stated_value: 97
    depth: medium
    dfwbp:
      deliverable: "The first bonus is the Resurge Quick Start Guide"
      feature: "A concise guide that maps out exactly what to do on days 1 through 7"
      why: "Because starting a new health protocol can feel overwhelming"
      benefit: "So you know exactly what to take, when to take it, and what to expect each day"
      proof: "The same onboarding sequence our most successful customers followed"
    ordering_rationale: "Highest value bonus — addresses the immediate 'what do I do first?' question"
```
