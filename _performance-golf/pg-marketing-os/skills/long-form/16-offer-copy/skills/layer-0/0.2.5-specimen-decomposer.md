# Layer 0.2.5 — Specimen Decomposer

**Version:** 1.0
**Layer:** 0 (Foundation)
**Type:** Specimen Analysis
**Domain:** Offer Copy
**Dependencies:** vault_offer-copy_intelligence (from 0.2-vault-intelligence-loader.md)
**Output:** specimen_decomposition (structural templates, technique catalog, power elements)

---

## Purpose

Decompose top-scoring offer copy specimens from the vault-intelligence into reusable structural templates, technique catalogs, and power elements. Offer copy specimens contain verbatim text organized by offer section type — value stack copy, guarantee copy, urgency copy, risk reversal copy, and CTA copy. This skill extracts the copywriting architecture that makes elite offer sections convert.

**This skill bridges the gap between:** vault data (verbatim offer copy segments by type) and generation layers (what Claude needs to write original offer copy with the same persuasive density and conversion architecture).

## Decomposition Protocol

### Phase 1: Specimen Selection & Tiering

```
IF tier_metadata present (Phase B loader active):
  LOAD gold_specimens → full decomposition
  LOAD silver_specimens → template-only decomposition
ELSE (pre-upgrade fallback):
  FOR each copy type (value_stack, guarantee, urgency, risk_reversal, cta):
    SELECT top 3 by position as pseudo-Gold
    SELECT next 5 as pseudo-Silver
  LOG "Operating in pre-upgrade mode — tiering estimated from position within type"

VERIFY each specimen has:
  - verbatim_text (required — skip if absent)
  - copy_type category (required)
  - source_file (required for provenance)
  - niche (required for type-matching)
```

### Phase 2: Structural Template Extraction

For each Gold specimen, by copy type:

```
VALUE STACK COPY:
  ANALYZE verbatim_text to extract:
    copy_skeleton:
      - element_type: [item_name | item_description | value_statement |
                       benefit_bridge | bonus_introduction | value_total |
                       savings_calculation]
      - function: string
      - position: integer
      - proportion: float
    stack_copy_architecture:
      items_count: integer
      total_stated_value: string
      value_presentation: string (cumulative | per_item | comparison)
      benefit_framing: string (feature_to_benefit | benefit_first | outcome_first)

GUARANTEE COPY:
  ANALYZE verbatim_text to extract:
    copy_skeleton:
      - element_type: [guarantee_statement | duration | conditions |
                       risk_reversal_logic | confidence_signal | reassurance]
      - function: string
      - position: integer
    guarantee_copy_architecture:
      guarantee_type: string
      risk_reversal_strength: string
      tone: string (confident | empathetic | bold | casual)

URGENCY COPY:
  ANALYZE verbatim_text to extract:
    copy_skeleton:
      - element_type: [deadline_statement | scarcity_claim | reason_for_limit |
                       consequence_of_delay | proof_of_scarcity]
      - function: string
      - position: integer
    urgency_copy_architecture:
      urgency_type: string
      believability_strategy: string

RISK REVERSAL COPY:
  ANALYZE verbatim_text to extract:
    copy_skeleton:
      - element_type: [risk_acknowledgment | reversal_statement |
                       confidence_logic | outcome_guarantee | safety_net]
      - function: string
      - position: integer
    risk_reversal_architecture:
      reversal_strategy: string
      confidence_source: string

CTA COPY:
  ANALYZE verbatim_text to extract:
    copy_skeleton:
      - element_type: [action_command | benefit_reminder | urgency_reinforcement |
                       simplicity_signal | format_instruction]
      - function: string
      - position: integer
    cta_architecture:
      cta_style: string (direct_command | benefit_wrapped | question_close |
                          assumptive | fear_of_loss)

For Silver specimens:
  Extract architecture classification ONLY (type-level, not element-level)
```

### Phase 3: Technique Catalog Compilation

Aggregate across ALL decomposed specimens:

```
value_stack_copy_techniques:
  FOR each unique value presentation approach:
    - technique_name: string
    - count: integer
    - implementation_pattern: string
    - gold_example: string (1-3 sentences)
    - items_count_range: string

guarantee_copy_techniques:
  FOR each unique guarantee writing approach:
    - technique_name: string
    - count: integer
    - implementation_pattern: string
    - gold_example: string
    - tone: string
    - best_for: string (price point / product type)

urgency_copy_techniques:
  FOR each unique urgency writing approach:
    - technique_name: string
    - count: integer
    - implementation_pattern: string
    - gold_example: string
    - believability_factor: string

cta_styles:
  FOR each unique CTA construction approach:
    - style_name: string
    - count: integer

offer_section_sequencing:
  FOR each observed ordering of offer copy sections:
    - sequence: [string] (ordered copy types)
    - frequency: integer
    - specimens_using: [source_file references]
```

## Quality Gates

- [ ] Minimum 2 Gold specimens per copy type fully decomposed
- [ ] Minimum 3 Silver specimens per copy type template-decomposed
- [ ] All copy skeletons have complete element sequences
- [ ] Technique catalog has minimum 2 techniques per copy type
- [ ] Power elements contain minimum 2 verbatim examples per copy type from Gold specimens
- [ ] Offer section sequencing documented from at least 3 specimens
- [ ] Every power element includes source provenance
- [ ] No raw verbatim_text passed through undecomposed

---

## Anti-Patterns

| Anti-Pattern | Signal | Fix |
|---|---|---|
| **Type-Blind Analysis** | All copy types analyzed with same skeleton | Each copy type has unique element types and architecture |
| **Copy Without Architecture** | Verbatim text listed without structural analysis | Every specimen needs skeleton + architecture |
| **Guarantee Without Strength** | Guarantee copy analyzed without risk_reversal_strength | Always rate reversal strength |
| **CTA Without Style** | CTAs listed without style classification | Every CTA needs cta_style |
| **Sequencing Ignored** | Individual copy types analyzed without section ordering | offer_section_sequencing captures how types flow together |
