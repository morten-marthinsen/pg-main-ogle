**Application:** Offer presentation structure

Present components and bonuses one at a time, then RESTACK the ENTIRE offer after each addition. By the end, you've recapped the full offer 6+ times so they remember everything.

1. Core offer (main modules) presented first with total value

## ANTI-PATTERN WARNINGS

### What These Specimens Are NOT

**NOT templates to fill in:**
These specimens demonstrate PATTERNS, not fill-in-the-blank structures. Adapt principles to specific offer.

**NOT permission to copy pricing structures:**
Price anchors and discount percentages must reflect actual offer economics. Never fabricate value claims.

**NOT replacements for human checkpoint data:**
The actual deliverables, bonuses, pricing, and guarantee terms come from upstream. Specimens inform HOW to present, not WHAT to present.

### Common Generation Failures

| Failure Mode | Specimen Pattern Violated | Correction |
|--------------|---------------------------|------------|
| Feature dump without benefits | All specimens connect features to WHY and BENEFIT | Add explicit "why this matters" and "what this means for you" |
| Generic guarantee | All specimens use SPECIFIC, BRANDED guarantees | Name the guarantee, add specific terms, include contact details |
| Single CTA | All specimens have 3+ CTAs with different emotions | Add confidence, urgency, and final push CTAs |
| Price without anchor | All specimens establish HIGH anchor before reveal | Add retail value, alternative cost, or "if all it did" justification |
| Missing value demonstration | All specimens use "if all it did was..." pattern | Add 3 iterations before price reveal |
| Identical CTAs | All specimens vary emotion and language per CTA | Change supporting reason and emotional appeal for each |
| Urgency without justification | TIER1 patterns use inevitability or specific reasons | Never generic "limited time" without credible rationale |
