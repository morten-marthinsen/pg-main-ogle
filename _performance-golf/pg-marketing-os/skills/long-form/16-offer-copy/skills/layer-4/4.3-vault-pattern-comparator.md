## PURPOSE

Compares the assembled offer copy against elite offer presentations from the TIER1 vault. The vault contains 482+ extraction reports from proven controls, each with `offer_architecture`, `guarantee`, `price_psychology`, and `CTA` sections. This microskill identifies how the current offer copy compares to elite patterns — not to copy them, but to ensure the copy meets the quality bar and to document differentiation.

The comparison serves two functions: (1) quality benchmarking — does this offer copy reach the persuasive density and structural completeness of TIER1 controls? (2) differentiation documentation — how does this offer copy differ from vault patterns, and is that differentiation intentional and beneficial?

---

## IDENTITY

**This microskill IS:**
- A vault pattern benchmarking system for offer copy quality
- A TIER1 elite offer comparison engine
- A differentiation documenter — how this copy differs from vault patterns
- A quality scoring system based on competitive benchmarks
- A pattern matching system across D-F-W-B-P, value demos, price reveals, and guarantees

**This microskill is NOT:**
- A principle checker (that is 4.1 — structural compliance)
- An anti-slop validator (that is 4.2 — language quality)
- A copy generator (this compares, doesn't write)
- A template enforcer (vault patterns inform quality, not dictate structure)

## PROCESS

### Step 1: Select Comparison Cohort

1. From the TIER1 vault, select 5-8 comparison controls that match:
   - Same or similar niche (health, finance, self-improvement, golf, etc.)
   - Same or similar product format (supplement, digital, info, physical)
   - Same or similar offer complexity (comparable deliverable/bonus count)

2. If exact matches unavailable, expand to adjacent niches with similar offer structures

### Step 2: Compare D-F-W-B-P Execution

3. Assess deliverable presentation quality vs. vault patterns:
   - Do vault controls use full D-F-W-B-P for each deliverable?
   - How does this copy's D-F-W-B-P depth compare?
   - Which vault controls have the strongest proof elements per deliverable?
   - Does this copy's proof quality match the vault benchmark?

4. Score: D-F-W-B-P execution quality (1-10)

### Step 3: Compare Value Demonstration

5. Assess value demonstration vs. vault patterns:
   - How many value demo iterations do vault controls use? (Typical: 3-5)
   - How compelling are the "why it would be worth it" reasons compared to vault?
   - Do vault controls use the "if all it did was..." pattern explicitly?
   - What additional value demonstration techniques do vault controls use?

6. Score: Value demonstration impact (1-10)

### Step 4: Compare Price Architecture

7. Assess price presentation vs. vault patterns:
   - What anchor ratios do vault controls use? (Typical: 3:1 to 10:1)
   - How do vault controls structure the descending cascade?
   - Do vault controls use per-day or per-unit minimizers?
   - How does this copy's price presentation compare?

8. Score: Price psychology effectiveness (1-10)

### Step 5: Compare Guarantee Execution

9. Assess guarantee vs. vault patterns:
   - Do vault controls use branded guarantee names? (Almost always yes)
   - What guarantee strength levels are typical for this niche?
   - How does the guarantee framing compare? (Contract/promise vs. refund policy)
   - Does the guarantee create confidence or just relieve risk?

10. Score: Guarantee strength (1-10)

### Step 6: Compare CTA Architecture

11. Assess CTA sequence vs. vault patterns:
    - How many CTAs do vault controls use? (Typical: 3-8 in offer section)
    - How varied are vault CTA emotions and phrases?
    - Do vault controls use consequence amplification between CTAs?
    - How does this copy's CTA density and variety compare?

12. Score: CTA effectiveness (1-10)

### Step 7: Document Differentiation

13. For each comparison dimension, document:
    - How this copy DIFFERS from vault patterns
    - Whether the differentiation is INTENTIONAL (serving a strategic purpose)
    - Whether the differentiation is BENEFICIAL (making the copy stronger)
    - Whether the differentiation is CONCERNING (potential weakness)

### Step 8: Compile Comparison Report

14. Assemble:

## QUALITY CRITERIA

| Criterion | Pass Condition |
|-----------|---------------|
| Cohort selected | 5-8 vault controls compared with niche/format match |
| All dimensions scored | D-F-W-B-P, value demo, price, guarantee, CTA all scored |
| Differentiation documented | Every significant difference from vault noted with assessment |
| Benchmarks established | Vault typical ranges established for each dimension |
| Overall score computed | Weighted average across all dimensions |
| Recommendations specific | Any score < 7.0 accompanied by specific improvement direction |
| Comparison fair | Vault controls selected are genuinely comparable, not cherry-picked |

---

## FAILURE MODES

### Failure Mode 1: No Comparable Controls
- **Symptom:** No vault controls match the niche/format
- **Cause:** Vault coverage gap for this niche
- **Prevention:** Step 1 expands to adjacent niches if exact matches unavailable
- **Recovery:** Use adjacent niche controls with documented niche differences

### Failure Mode 2: Cherry-Picked Comparisons
- **Symptom:** Only compared to weakest vault controls — artificially high score
- **Cause:** Selection bias toward favorable comparisons
- **Prevention:** Step 1 requires 5-8 controls including both strong and moderate examples
- **Recovery:** Add strongest vault controls to cohort, re-score

### Failure Mode 3: Scoring Without Evidence
- **Symptom:** Scores assigned but no textual evidence cited
- **Cause:** Subjective scoring without anchoring to specific vault patterns
- **Prevention:** Each score requires specific vault pattern citation
- **Recovery:** Re-score with specific vault examples cited for each dimension

## EXAMPLES

### Example 1: Supplement Offer Comparison

**Cohort:** Resurge, Montezuma's Secret, Total Restore, Testo Rise, Uri Varx (5 controls)
**Scores:** D-F-W-B-P: 8.0 | Value Demo: 7.5 | Price Psychology: 8.5 | Guarantee: 9.0 | CTA: 7.0
**Overall:** 8.0
**Differentiation:** Uses 3 value demo iterations (vault typical: 3-5) — within range. Uses branded "Triple-Back Guarantee" — matches vault best practice. CTA count (3) is below vault average (5) — potential improvement area.
**Recommendation:** Add 2 more CTAs to offer section to match vault CTA density.

### Example 2: Digital Product Comparison

**Cohort:** BBW, Superbrain, Elixir of Eros + 3 adjacent (6 controls)
**Scores:** D-F-W-B-P: 6.5 | Value Demo: 8.0 | Price Psychology: 7.0 | Guarantee: 6.0 | CTA: 8.0
**Overall:** 7.1
**Differentiation:** D-F-W-B-P weaker because digital products have fewer tangible proof elements. Guarantee generic — vault controls almost always brand their guarantee.
**Recommendation:** Strengthen D-F-W-B-P proof elements with testimonials. Brand the guarantee.
