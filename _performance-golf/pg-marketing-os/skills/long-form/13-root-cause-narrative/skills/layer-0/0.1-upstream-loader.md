# Skill 13.0.1: Upstream Loader

**Version:** 1.0
**Skill:** 10 — Root Cause Narrative
**Layer:** 0 (Foundation & Loading)
**Position:** First microskill — loads all upstream data before any processing
**Type:** Data Loading & Extraction
**Dependencies:** 03-root-cause outputs, 08-structure outputs, 02-proof-inventory outputs, 09-campaign-brief outputs, 12-story outputs
**Output:** upstream_data.json

---

## PURPOSE

Load all upstream skill packages required for root cause narrative generation — the root cause package (from Skill 11), the structure package (from Skill 11), and the proof inventory (from Skill 11). Extract the specific fields each downstream microskill needs, normalize field names, and validate schema completeness. This is the data foundation upon which the entire narrative pipeline operates.

---

## IDENTITY

**This microskill IS:**
- A data loading and extraction utility that gathers all upstream inputs into a single normalized context
- A schema-aware loader that knows exactly which fields each downstream microskill requires
- A completeness checker that flags missing fields before processing begins

**This microskill is NOT:**
- A validator of data quality (that is 0.3-input-validator)
- A TIER1 intelligence loader (that is 0.2-tier1-intelligence-loader)
- A data transformer or enricher — it loads what exists, it does not create what is missing
- A decision-maker about which root cause to use — it loads what Skill 02 produced

### Step 2: Load Root Cause Package (CRITICAL)

```
PARSE root-cause-package.yaml:

EXTRACT three_part_structure:
  what_they_think = root_cause.three_part_structure.what_they_think
  what_it_really_is = root_cause.three_part_structure.what_it_really_is
  why_nothing_worked = root_cause.three_part_structure.why_nothing_worked

  VALIDATE each field is non-empty string
  IF any field is empty or null:
    ADD to gaps: "three_part_structure.[field_name] is empty"

EXTRACT expression_method:
  expression_method = root_cause.expression_method
  VALIDATE is non-empty string
  IF empty: ADD to gaps: "expression_method is empty"

EXTRACT reframe_technique:
  reframe_technique = root_cause.reframe_technique
  VALIDATE is non-empty string and matches known taxonomy:
    [not_your_fault, conspiracy, hidden_truth, opposite_of_truth,
     suppressed_science, conspiracy_expose]
  IF invalid: ADD to gaps: "reframe_technique invalid or missing"

EXTRACT villain:
  villain.name = root_cause.villain.name
  villain.type = root_cause.villain.type
  villain.role = root_cause.villain.role
  VALIDATE all three fields present
  IF any missing: ADD to gaps: "villain.[field_name] is missing"

EXTRACT countersells:
  FOR each countersell in root_cause.countersells:
    VALIDATE has: target, pattern, argument
    ADD to countersells_array
  IF countersells_array.length < 1:
    ADD to gaps: "no countersells defined"

EXTRACT copy_blocks:
  FOR each block in [false_belief_statement, root_cause_reveal,
                     failure_explanation, villain_narrative]:
    IF present and non-empty: ADD to copy_blocks object
    IF missing: ADD to gaps: "copy_block.[block_name] missing"
```

### Step 3: Load Structure Package

```
PARSE structure-package.json:

EXTRACT campaign_thesis:
  campaign_thesis = structure.campaign_thesis
  IF empty: ADD to gaps: "campaign_thesis not available"

EXTRACT cpb_chunks relevant to root cause:
  FOR each chunk in structure.cpb_chunks:
    IF chunk.assigned_skill == "13-root-cause-narrative"
       OR chunk.chunk_type IN ["root_cause", "problem_agitation",
                                "failure_explanation", "villain"]:
      ADD to relevant_cpb_chunks

  IF relevant_cpb_chunks.length == 0:
    ADD to warnings: "No CPB chunks explicitly assigned to root cause narrative"
    ATTEMPT: Identify chunks by content_summary keywords

EXTRACT flow_architecture:
  section_sequence = structure.flow_architecture.section_sequence
  transition_points = structure.flow_architecture.transition_points

  IDENTIFY root_cause_position in section_sequence
  IDENTIFY what_comes_before and what_comes_after

  IF root_cause not found in sequence:
    ADD to warnings: "Root cause position not explicit in flow architecture"

EXTRACT downstream_handoffs.for_body_copy:
  placement_guide = structure.downstream_handoffs.for_body_copy.placement_guide
  word_count_budget = structure.downstream_handoffs.for_body_copy.word_count_budget

  IF word_count_budget available:
    EXTRACT root_cause_word_budget from word_count_budget
  ELSE:
    SET root_cause_word_budget = null
    ADD to warnings: "No explicit word count budget for root cause section"

EXTRACT format_type (optional):
  IF structure.format_type present:
    format_type = structure.format_type
  ELSE:
    format_type = "unknown"
    ADD to warnings: "Campaign format type not specified in structure package"
```

### Step 4: Load Proof Inventory

```
PARSE proof-inventory-output.json:

EXTRACT proof_elements:
  FOR each element in proof_inventory.proof_elements:
    CLASSIFY element by relevance to root cause narrative:
      HIGH relevance: evidence supporting the root cause claim,
                      villain evidence, failure evidence,
                      credibility/authority evidence
      MEDIUM relevance: general scientific evidence,
                         historical evidence, statistical data
      LOW relevance: product-specific proof,
                     testimonials about results (not root cause)

    TAG element with relevance_level
    ADD to proof_elements_array

EXTRACT credential_stack:
  credential_stack = proof_inventory.credential_stack
  IF empty or null:
    ADD to warnings: "No credential stack available for authority bridge"

EXTRACT evidence by type for root cause narrative:
  statistical_evidence = proof_inventory.statistical_evidence OR []
  institutional_citations = proof_inventory.institutional_citations OR []
  visual_proof = proof_inventory.visual_proof_available OR []

  COMPILE available_evidence_types:
    IF statistical_evidence.length > 0: ADD "specific_number" to available_types
    IF institutional_citations.length > 0: ADD "institutional_name_drop" to available_types
    IF visual_proof.length > 0: ADD "visual_demonstration" to available_types
    IF credential_stack.length > 0: ADD "person_as_proof" to available_types

    CHECK for villain's_own_data patterns in statistical_evidence
    CHECK for paradox_proof patterns in credential_stack or testimonials
    CHECK for geographic_historical_proof in historical evidence
```

**Why this matters:** Without reading the actual story prose, the root cause narrative may repeat metaphors, re-explain concepts, or break voice consistency. This is the #1 cause of the "stitched together" feeling in assembled drafts.

### Step 9: Calculate Completeness

```
CALCULATE completeness_percentage:

  CRITICAL fields (weight 3):
    - three_part_structure.what_they_think
    - three_part_structure.what_it_really_is
    - three_part_structure.why_nothing_worked
    - expression_method
    - reframe_technique
    - villain.name
    - villain.type

  IMPORTANT fields (weight 2):
    - villain.role
    - countersells (at least 1)
    - copy_blocks.root_cause_reveal
    - copy_blocks.false_belief_statement
    - campaign_thesis
    - credential_stack (non-empty)
    - proof_elements (at least 1 HIGH relevance)

  HELPFUL fields (weight 1):
    - copy_blocks.failure_explanation
    - copy_blocks.villain_narrative
    - relevant_cpb_chunks (non-empty)
    - word_count_budget
    - format_type
    - available_evidence_types (at least 2)
    - alternative_root_causes

  total_weight = sum of all field weights
  present_weight = sum of weights for present fields
  completeness_percentage = (present_weight / total_weight) * 100

  LOG: "Upstream loading completeness: [percentage]%"
  IF completeness_percentage < 60%:
    LOG: "CRITICAL: Upstream data severely incomplete. Narrative quality will be compromised."
  ELIF completeness_percentage < 85%:
    LOG: "WARNING: Upstream data has gaps. Some narrative phases may be weaker."
  ELSE:
    LOG: "Upstream data sufficient for full narrative generation."
```

## QUALITY CRITERIA

1. **All three upstream files attempted** — loader must attempt to load all three source files, even if some are missing
2. **Root cause package is CRITICAL** — if root-cause-package.yaml is missing, the loader must flag this as a blocking failure; narrative cannot proceed without it
3. **Three-part structure fully extracted** — all three parts (what_they_think, what_it_really_is, why_nothing_worked) must be individually extracted and available
4. **Villain extracted with all fields** — name, type, and role must be individually accessible
5. **Proof elements classified by relevance** — every proof element must be tagged HIGH, MEDIUM, or LOW for root cause narrative relevance
6. **Completeness percentage calculated accurately** — weighted calculation must reflect true data availability
7. **All gaps and warnings logged** — no silent failures; every missing or questionable field must appear in gaps or warnings
8. **Evidence types cataloged** — available_evidence_types must list which of the 7 TIER1 evidence patterns can be populated from the proof inventory

---

## FAILURE MODES

| Failure | Severity | Detection | Handling |
|---------|----------|-----------|----------|
| Root cause package missing | CRITICAL | File not found at primary or fallback path | HALT — send feedback request to Skill 02 for re-generation |
| Three-part structure incomplete | CRITICAL | One or more parts empty/null | Flag in gaps, set completeness below threshold, force MODIFY at human checkpoint |
| Structure package missing | WARNING | File not found | Continue with reduced alignment capability; flag in warnings |
| Proof inventory missing | WARNING | File not found | Continue with no evidence integration; flag in warnings |
| Villain not identified | HIGH | villain fields empty | Flag in gaps; narrative will lack villain deep dive phase |
| No countersells defined | MODERATE | countersells array empty | Flag in gaps; countersell bridge writer will need to generate from scratch |
| Format type unknown | LOW | format_type not in structure package | Default to vsl_sequence; flag in warnings |
| No CPB chunks for root cause | MODERATE | No chunks assigned to skill 10 | Attempt keyword-based identification; flag in warnings |
