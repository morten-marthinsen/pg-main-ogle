# Skill 13.1.1: Communication Type Classifier

**Version:** 1.0
**Skill:** 10 — Root Cause Narrative
**Layer:** 1 (Strategic Classification)
**Position:** First classification microskill — determines HOW the root cause is communicated
**Type:** Pattern Classification
**Dependencies:** checkpoint_response.json (from 0.4), tier1_intelligence_index.json (from 0.2)
**Output:** communication_type_classification.json

---

## PURPOSE

Classify the root cause narrative into one of 10 communication types based on the confirmed root cause data, niche characteristics, audience sophistication level, and available proof elements. The communication type determines the TONE, EVIDENCE STYLE, and NARRATIVE VOICE of the entire root cause section. This classification drives every downstream writing decision in Layer 2.

---

## IDENTITY

**This microskill IS:**
- A pattern classifier that maps campaign characteristics to one of 10 established communication types
- A TIER1-informed decision engine that uses cross-vertical patterns to select the most effective type for the specific niche and audience
- A rationale generator that documents WHY a specific type was selected, with vault references

**This microskill is NOT:**
- A writer — it classifies, not generates prose
- An inventor of new communication types — it selects from the 10 types found in TIER1 analysis
- A framing pattern selector (that is 1.2) — communication type and framing pattern are different axes
- A subjective preference system — selection is data-driven from niche patterns and audience characteristics

## PROCESS

### Step 1: Load the 10 Communication Types

```
FROM tier1_intelligence_index.communication_types, LOAD all 10 types:

TYPE 1: conspiracy_reveal
  Description: Powerful entities are actively suppressing the truth from the reader
  Optimal conditions:
    Niche: health, financial, political
    Sophistication: Stage 3-4
    Audience: frustrated with establishment, open to alternative narratives
  Evidence requirements: documented actions, institutional evidence, motive proof
  Tone: revelatory, insider knowledge, "what they don't want you to know"
  TIER1 example: Gundry — food industry pushing lectin-heavy foods despite knowing harm

TYPE 2: scientific_explanation
  Description: Clinical, precise language explains a misunderstood biological/physical process
  Optimal conditions:
    Niche: health/supplement, educated audiences
    Sophistication: Stage 2-4
    Audience: values credentials, responds to data and specifics
  Evidence requirements: research citations, specific numbers, institutional name-drops
  Tone: authoritative, precise, educational but accessible
  TIER1 example: Gundry — Citrate Lyase enzyme explanation, lectin biology

TYPE 3: direct_confrontation
  Description: Aggressively declares everything the reader has been doing is wrong
  Optimal conditions:
    Niche: golf, fitness, male-audience markets
    Sophistication: Stage 3-4
    Audience: male, ego-driven, responds to blunt challenge
  Evidence requirements: contrast evidence (wrong vs. right approach), demonstration proof
  Tone: confrontational, blunt, "everything you've been told is wrong"
  TIER1 example: Haney — "It's A LOT OF BULL" — directly attacks grip/stance mythology

TYPE 4: metaphor_driven
  Description: Vivid visual metaphors make an invisible or complex process tangible
  Optimal conditions:
    Niche: any with complex mechanism, VSL format
    Sophistication: Stage 2-5
    Audience: visual thinkers, lay audiences, complex root causes needing simplification
  Evidence requirements: vivid imagery, analogy accuracy, before/after contrast
  Tone: vivid, narrative, story-like, makes invisible visible
  TIER1 example: Montezuma — "Death begins in the colon" metaphorical framework

TYPE 5: data_presentation
  Description: Cascading statistics and data points systematically dismantle the false narrative
  Optimal conditions:
    Niche: financial, investment, analytical markets
    Sophistication: Stage 4-5
    Audience: data-literate, skeptical, values evidence over emotion
  Evidence requirements: specific numbers, charts/data references, statistical trends
  Tone: measured, data-heavy, factual cascade, "the numbers don't lie"
  TIER1 example: Stansberry — Federal Reserve data, currency devaluation percentages

TYPE 6: conspiracy_expose
  Description: Names specific conspirators with documented actions and traceable evidence
  Optimal conditions:
    Niche: financial, political, health (high sophistication)
    Sophistication: Stage 4-5
    Audience: skeptical of institutions, wants names and dates
  Evidence requirements: named individuals/entities, specific dates, documented actions
  Tone: investigative journalism, naming names, documented trail
  TIER1 example: Stansberry — names Federal Reserve officials, specific policy dates

TYPE 7: opposite_of_truth
  Description: Conventional wisdom is the exact OPPOSITE of reality
  Optimal conditions:
    Niche: any market with strong conventional belief
    Sophistication: Stage 4+
    Audience: frustrated by repeated failure despite following conventional wisdom
  Evidence requirements: strong counter-evidence, paradox proof, historical contrast
  Tone: shocking inversion, "the opposite of what you think"
  TIER1 example: Gundry — "healthy" whole grains are the CAUSE of disease

TYPE 8: hidden_truth
  Description: Information exists but has been overlooked, not actively suppressed
  Optimal conditions:
    Niche: any market at lower sophistication
    Sophistication: Stage 2-3
    Audience: lower skepticism, open to "new discovery" framing
  Evidence requirements: legitimate research, obscure studies, emerging science
  Tone: discovery, "finally found," academic-meets-accessible
  TIER1 example: Sears — lesser-known telomere science reframed for mainstream audience

TYPE 9: not_your_fault
  Description: Root cause simultaneously explains the problem AND removes reader's self-blame
  Optimal conditions:
    Niche: any market with repeated personal failure
    Sophistication: Any stage
    Audience: anyone who has tried and failed repeatedly, carries shame or frustration
  Evidence requirements: externalization evidence (biology, system, deception), failure catalog
  Tone: empathetic first, then explanatory, relief-giving
  TIER1 example: Gundry — "It's not willpower. It's lectins." Montezuma — "Your body isn't broken."

TYPE 10: suppressed_science
  Description: Real peer-reviewed science exists but has been deliberately buried by interests
  Optimal conditions:
    Niche: health with need for scientific credibility
    Sophistication: Stage 3-4
    Audience: health-conscious, values science, skeptical of alternatives
  Evidence requirements: actual studies, publication names, suppression evidence
  Tone: "the science is clear, but they buried it," righteous revelation
  TIER1 example: FloraSource — published microbiome research that pharmaceutical industry ignores
```

### Step 2: Score Each Type Against Campaign Characteristics

```
FOR each communication_type:

  CALCULATE niche_fit_score (0-10):
    QUERY: cross_vertical_patterns.by_niche[niche].dominant_communication_types
    IF type is in dominant list: score = 8-10 (based on rank)
    IF type is in secondary list: score = 5-7
    IF type is not in niche patterns: score = 1-4
    SPECIAL: If niche not found in cross-vertical patterns, use type's
             general optimal_conditions

  CALCULATE sophistication_match_score (0-10):
    IF market_sophistication_stage is within type's optimal range: score = 8-10
    IF within 1 stage of optimal range: score = 5-7
    IF outside optimal range by 2+ stages: score = 1-4

  CALCULATE expression_method_alignment (0-10):
    COMPARE expression_method from root cause package with type's tone/approach:
    IF direct match: score = 9-10
    IF compatible: score = 6-8
    IF misaligned: score = 1-5

  CALCULATE evidence_availability (0-10):
    COMPARE type's evidence_requirements against available_evidence_types:
    IF all requirements available: score = 9-10
    IF most requirements available: score = 6-8
    IF few requirements available: score = 3-5
    IF none available: score = 1-2

  CALCULATE villain_type_alignment (0-10):
    COMPARE villain.type with type's expected villain patterns:
    conspiracy_reveal → expects institutional villain: pharma, government, food_industry
    scientific_explanation → expects biological_process or misinformation
    direct_confrontation → expects industry or misinformation
    data_presentation → expects government, government_conspiracy
    conspiracy_expose → expects government_conspiracy, pharma
    not_your_fault → any villain type compatible
    suppressed_science → expects pharma, medical_establishment
    [etc.]

  COMPOSITE_SCORE = (niche_fit * 0.30) + (sophistication_match * 0.25) +
                    (expression_alignment * 0.20) + (evidence_availability * 0.15) +
                    (villain_alignment * 0.10)
```

### Step 3: Rank and Select Primary Type

```
RANK all 10 types by COMPOSITE_SCORE descending

SELECT primary_type:
  - Highest scoring type
  - Must score >= 6.0 composite
  - If top two types within 0.5 of each other, prefer the one with
    higher niche_fit_score (niche patterns are the strongest signal)

SELECT secondary_type (optional):
  - Second highest scoring type IF score >= 5.5
  - Must be compatible with primary (some types conflict)
  - Used only for hybrid approaches in high-sophistication markets

DOCUMENT selection_rationale:
  - Why primary type was selected (strongest scoring factors)
  - Why alternatives were rejected (weakest scoring factors)
  - If secondary type selected, why hybrid is appropriate
```

### Step 4: Identify Vault Reference

```
FROM tier1_intelligence_index.communication_types.by_name[selected_type]:

  IDENTIFY vault_reference:
    - Which TIER1 control uses this type in the same or similar niche
    - Extract the specific example of how the control deployed this type
    - Document the key technique that made it effective

  IF exact niche match found:
    vault_reference = {
      control_name: string,
      niche: string,
      technique: string,
      effectiveness_note: string
    }
  ELIF similar niche match found:
    vault_reference = {
      control_name: string,
      niche: string,
      similarity: string,
      adaptation_note: string
    }
  ELSE:
    vault_reference = {
      control_name: "Best available",
      note: "No exact niche match — using cross-vertical pattern"
    }
```

### Step 5: Define Type-Specific Writing Parameters

```
BASED ON selected_type, DEFINE writing_parameters:

  vocabulary_register:
    IF scientific_explanation: clinical, precise, technical-made-accessible
    IF conspiracy_reveal: insider, revelatory, exposing
    IF direct_confrontation: blunt, aggressive, challenging
    IF data_presentation: measured, statistical, factual
    IF metaphor_driven: vivid, visual, narrative
    [etc. for all 10 types]

  evidence_integration_style:
    IF scientific_explanation: inline citations, specific numbers woven into prose
    IF conspiracy_reveal: revelation structure, progressive disclosure
    IF data_presentation: cascading statistics, data-first arguments
    IF not_your_fault: empathy-first, evidence-as-relief
    [etc.]

  emotional_temperature:
    IF direct_confrontation: HIGH — aggressive, provocative
    IF not_your_fault: MEDIUM-HIGH — empathetic then assertive
    IF scientific_explanation: MEDIUM — authoritative calm
    IF data_presentation: MEDIUM-LOW — measured, analytical
    [etc.]

  three_part_delivery_style:
    IF conspiracy_reveal: build suspense → dramatic reveal → expose failure chain
    IF scientific_explanation: name misconception → present evidence → explain mechanism gap
    IF direct_confrontation: challenge belief → state truth bluntly → demolish old approach
    [etc.]
```

```
COMPILE communication_type_classification:
```

---

## QUALITY CRITERIA

1. **Selected type scores >= 6.0** — below 6.0 indicates poor fit; must re-evaluate or request upstream clarification
2. **Niche fit is the dominant signal** — niche patterns from TIER1 cross-vertical analysis carry the most weight because they reflect what actually works in the market
3. **Vault reference present** — every classification must cite at least one TIER1 control that uses this type in a relevant context
4. **Writing parameters defined** — downstream writers cannot operate without vocabulary, evidence style, emotional temperature, and delivery style guidance
5. **Rationale documented** — selection must explain WHY this type and WHY NOT the alternatives (for human review and learning log)
6. **Secondary type justified** — if selected, must explain why hybrid is needed and how the two types interact
7. **Scoring transparent** — all 10 types scored with breakdown visible for audit

---

## FAILURE MODES

| Failure | Severity | Detection | Handling |
|---------|----------|-----------|----------|
| No type scores above 6.0 | HIGH | All composite scores < 6.0 | Select highest scorer with caveat; flag for human review; may indicate upstream data misfit |
| Niche not in cross-vertical patterns | MODERATE | Niche lookup returns empty | Use general type conditions; score on other factors; flag reduced confidence |
| Expression method contradicts best type | MODERATE | High niche fit but low expression alignment | Flag conflict; human may need to adjust expression method or accept type override |
| Top two types tied | LOW | Scores within 0.1 of each other | Use niche_fit as tiebreaker; document both as viable |
| Villain type incompatible with best type | MODERATE | High score overall but villain alignment < 3 | Flag; villain may need adjustment at narrative level |
| Evidence requirements unmet | MODERATE | Best type needs evidence not available | Select type with lower evidence requirements; or flag for proof inventory enrichment |
