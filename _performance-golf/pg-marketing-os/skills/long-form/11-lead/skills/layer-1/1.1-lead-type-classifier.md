# Layer 1.1 — Lead Type Classifier

**Version:** 1.0
**Layer:** 1 (Lead Architecture)
**Type:** Classification & Strategy
**Dependencies:** lead_context (from 0.1), vault_lead_intelligence (from 0.2), teaching_index (from 0.3), validation_report (from 0.4)
**Output:** lead_type_classification.json

---

## Purpose

Classify the optimal lead type based on niche, mechanism assets, market sophistication, proof inventory strength, and vault patterns. The lead type determines the entire downstream architecture — hook device, emotional arc shape, construction sequence, and element weighting. A misclassified lead type cascades errors through every subsequent layer.

---

## Persona Deployment

**Primary Persona:** Sarah Chen — Competitive Intelligence Analyst
Brings pattern recognition across competitor landscapes. Identifies which lead types dominate specific niches and spots classification signals from market data.

**Secondary Persona:** Marcus Webster — Pattern Synthesis Analyst
Cross-references vault extraction patterns against current project parameters. Validates classification through empirical swipe evidence, not theoretical preference.

## Execution Protocol

### Step 1: Build Classification Parameter Set

```
EXTRACT from lead_context:
  niche_context:
    niche = product_context.niche (or infer from mechanism + promise)
    sub_niche = product_context.sub_niche
    schwartz_stage = product_context.market_sophistication

  mechanism_profile:
    mechanism_name = mechanism.name
    mechanism_type = mechanism.type
    e5_type = mechanism.e5_type
    uniqueness_strength = mechanism.uniqueness_claim (assessed 1-10)

  proof_profile:
    overall_strength = proof.overall_strength
    knockout_proof_type = proof.knockout_proof.type
    credential_assets = proof.top_credibility_elements
    has_authority_credentials = boolean (doctor, PhD, institution, etc.)
    has_statistical_proof = boolean (performance data, percentages, studies)
    has_story_elements = boolean (personal narrative, vulnerability arc)
    has_villain_profile = boolean (root_cause.villain_profile exists)

  promise_profile:
    primary_promise = promise.primary_statement
    emotional_frame = promise.emotional_frame
    counter_intuitive = root_cause.what_they_think != root_cause.what_it_really_is
```

### Step 2: Score Each Lead Type for Fit

```
FOR each of 9 lead types, calculate fit_score (0-100):

1. STATISTICAL (performance data cascade):
   niche_fit: +30 if wealth/investment, +20 if health with quantifiable data, +10 otherwise
   proof_fit: +30 if has_statistical_proof AND overall_strength >= 80
   mechanism_fit: +20 if mechanism produces measurable outputs
   schwartz_fit: +20 if Stage 1-3 (numbers still impress)
   BEST FOR: quantifiable results, wealth niches, investment performance

2. STORY_VULNERABILITY (shame-to-dignity arc):
   niche_fit: +25 for any niche where protagonist mirrors prospect
   proof_fit: +25 if has_story_elements AND story has vulnerability/shame moment
   mechanism_fit: +25 if mechanism discovered through personal struggle
   schwartz_fit: +25 if Stage 3-5 (stories cut through sophistication)
   BEST FOR: health transformations, personal struggles, any niche with relatable protagonist

3. AUTHORITY_FLEX (credential stacking):
   niche_fit: +25 if health or wealth with institutional backdrop
   proof_fit: +35 if has_authority_credentials AND credentials are exceptional
   mechanism_fit: +20 if mechanism is validated by credentialed authority
   schwartz_fit: +20 if Stage 2-4 (authority still carries weight)
   BEST FOR: when credentials are genuinely exceptional (Harvard, MD, White House)

4. COMPRESSED_DISCOVERY (mechanism naming + social proof, very tight):
   niche_fit: +25 if health supplements or scientific mechanism
   proof_fit: +25 if has_authority_credentials AND has social proof
   mechanism_fit: +30 if mechanism has a strong, nameable concept
   schwartz_fit: +20 if Stage 3-4 (mechanism novelty matters)
   BEST FOR: scientific mechanisms, supplement launches, named discoveries

5. CONSPIRACY_EXPOSE (named villains + evidence):
   niche_fit: +30 if financial, political, or health cover-up angle
   proof_fit: +25 if evidence of suppression/cover-up exists
   mechanism_fit: +20 if mechanism was suppressed or hidden
   schwartz_fit: +25 if Stage 4-5 (conspiracy cuts through noise)
   BEST FOR: financial newsletters, political commentary, health conspiracies

6. PROPHECY_WARNING (data deconstruction + urgency):
   niche_fit: +30 if crisis, financial warning, or health threat
   proof_fit: +25 if has_statistical_proof supporting warning thesis
   mechanism_fit: +20 if mechanism provides protection/escape
   schwartz_fit: +25 if Stage 3-5 (urgency transcends sophistication)
   BEST FOR: market crash warnings, health crises, political urgency

7. DOCTOR_DISCOVERY (medical authority + shocking finding):
   niche_fit: +35 if health, supplements, or medical breakthroughs
   proof_fit: +30 if has_authority_credentials of medical type (MD, researcher)
   mechanism_fit: +20 if mechanism has medical/scientific basis
   schwartz_fit: +15 if Stage 2-4
   BEST FOR: health supplements, medical breakthroughs, doctor-fronted campaigns

8. PROBLEM_SOLUTION (pain cascade -> promise -> mechanism tease):
   niche_fit: +20 universal — works across all niches
   proof_fit: +20 if proof supports promise chain
   mechanism_fit: +25 if mechanism directly solves identified problem
   schwartz_fit: +20 if Stage 1-3 (straightforward approach)
   UNIVERSAL: default safe option when no specialized type scores higher
   BEST FOR: broad audiences, Stage 1-3 markets, clear problem-solution narratives

9. HYBRID (combines 2+ types):
   niche_fit: +20 for any sophisticated market
   proof_fit: +25 if multiple proof types available (stats + authority + story)
   mechanism_fit: +25 if mechanism supports multiple framing approaches
   schwartz_fit: +30 if Stage 4-5 (hybrid needed to penetrate sophisticated defenses)
   BEST FOR: Stage 4-5 markets where single types are saturated
```

### Step 3: Cross-Reference with Vault Patterns

```
LOAD vault_lead_intelligence.lead_type_taxonomy

FOR each lead type scored in Step 2:
  SEARCH vault extractions for same lead type + similar niche
  IF match found:
    vault_alignment_bonus = +10
    vault_reference = extraction source identifier
    vault_evidence_notes = effectiveness observations from extraction
  IF no match:
    vault_alignment_bonus = 0
    vault_reference = "no direct vault match"

ALSO CHECK:
  - Which lead types are OVERREPRESENTED in vault for this niche (caution: may be saturated)
  - Which lead types are UNDERREPRESENTED (opportunity or poor fit)
```

### Step 4: Calculate Final Fit Scores and Select

```
FOR each lead type:
  final_fit_score = base_fit_score + vault_alignment_bonus

SORT by final_fit_score DESC

SELECT primary_type = highest scoring type
SELECT runner_up = second highest scoring type

IF top two scores are within 5 points:
  FLAG "close classification — consider hybrid approach"
  IF schwartz_stage >= 4:
    CONSIDER promoting to HYBRID combining elements of both top types

VALIDATE selection:
  - primary_type.final_fit_score >= 60 (minimum viable classification)
  - primary_type has at least one vault reference supporting it
  - primary_type aligns with niche context
```

### Step 5: Determine Opening Device

```
BASED ON selected lead type, assign default opening device:

  statistical → shocking_stat
  story_vulnerability → pattern_interrupt
  authority_flex → authority_flex
  compressed_discovery → doctor_shock OR curiosity_gap
  conspiracy_expose → threat_frame
  prophecy_warning → rhetorical_question
  doctor_discovery → doctor_shock
  problem_solution → problem_callout
  hybrid → varies (select from top 2 type devices)

CROSS-REFERENCE with vault_lead_intelligence.hook_catalog:
  IF vault shows a different device working for this type + niche:
    CONSIDER vault-evidenced device as alternative
```

### Step 6: Compile Classification Output

```
BUILD classification object with:
  - selected lead type
  - opening device recommendation
  - vault reference supporting classification
  - full classification rationale citing scores
  - runner-up type with score
  - fit scores for all 9 types
  - confidence level (high/medium/low based on score margin)
```

- [ ] All 9 lead types scored with specific fit values (zero skipped)
- [ ] Selected type has fit_score >= 60
- [ ] Vault reference provided supporting classification (not "none")
- [ ] Classification rationale cites specific scores and logic (no vague justification)
- [ ] Runner-up identified with fit score and explanation for non-selection
- [ ] Opening device assigned and aligned with selected lead type

- [ ] Confidence level accurately reflects score margin (high = 15+ point lead, medium = 5-15, low = < 5)
- [ ] If hybrid selected, component types identified

## Anti-Patterns

| Anti-Pattern | What It Looks Like | Why It Fails |
|---|---|---|
| **Default to Problem-Solution** | Always selecting problem_solution because it is "safe" | Ignores niche-specific advantages; misses opportunities for higher-performing specialized types |
| **Credential Inflation** | Selecting authority_flex when credentials are average | Authority-flex only works when credentials are genuinely exceptional; average credentials weaken the lead |
| **Ignoring Schwartz Stage** | Selecting statistical for a Stage 5 market | Stage 4-5 prospects have seen every stat cascade; need conspiracy, hybrid, or story to penetrate |
| **Vault Mimicry** | Copying an exact vault pattern without adaptation | Vault patterns are reference points, not templates; the lead must fit THIS campaign's assets |
| **Force-Fitting Conspiracy** | Selecting conspiracy_expose without genuine villain/cover-up evidence | Conspiracy leads without real evidence feel manufactured and destroy credibility |
| **Ignoring Proof Inventory** | Selecting doctor_discovery without medical credentials | The type requires genuine medical authority; forcing it without credentials creates a gap the lead cannot fill |
| **Single-Factor Classification** | Choosing based only on niche without considering mechanism, proof, or Schwartz stage | All six criteria must inform classification; single-factor decisions produce misaligned leads |
