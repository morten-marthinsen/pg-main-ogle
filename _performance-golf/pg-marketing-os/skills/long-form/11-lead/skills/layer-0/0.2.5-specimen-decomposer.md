# Layer 0.2.5 — Specimen Decomposer

**Version:** 1.0
**Layer:** 0 (Foundation)
**Type:** Specimen Analysis
**Domain:** Lead
**Dependencies:** vault_lead_intelligence (from 0.2-vault-intelligence-loader.md)
**Output:** specimen_decomposition (structural templates, technique catalog, power elements)

---

## Purpose

Decompose top-scoring lead specimens from the vault-intelligence into reusable structural templates, technique catalogs, and power elements. Raw specimens are the wrong unit of delivery for generation — they are too large, too specific, and too easy to surface-mimic. This skill extracts the *architecture* that makes elite leads work, not the content itself.

**This skill bridges the gap between:** vault data (what elite copywriters produced) and generation layers (what Claude needs to learn from them).

## Decomposition Protocol

### Phase 1: Specimen Selection & Tiering

```
IF tier_metadata present (Phase B loader active):
  LOAD gold_specimens → full decomposition
  LOAD silver_specimens → template-only decomposition
ELSE (pre-upgrade fallback):
  SELECT top 5 specimens by position in vault-intelligence as pseudo-Gold
  SELECT next 10 specimens as pseudo-Silver
  LOG "Operating in pre-upgrade mode — tiering estimated from array position"

VERIFY each specimen has:
  - full_text (required — skip specimen if absent)
  - hook_sentence (required)
  - type (required)
  - opening_device (preferred)
  - emotional_progression (preferred)
  - word_count (preferred)
  - source_file (required for provenance)
  - niche (required for type-matching)
```

### Phase 2: Structural Template Extraction

For each Gold specimen:

```
ANALYZE full_text and extract:

  skeleton:
    DECOMPOSE the lead text into ordered structural elements:
    - element_type: [hook | credential_establishment | mechanism_tease |
                     data_cascade | story_beat | problem_identification |
                     social_proof_insertion | emotional_shift |
                     curiosity_amplifier | transition_to_body]
    - function: [what this element accomplishes in the lead]
    - proportion: [approximate % of total word count]
    - position: [ordinal position in sequence]

  structural_signature:
    opening_move: [first structural element and its function]
    build_pattern: [how elements escalate — linear, spiral, cascade, nested]
    turn_point: [where the lead shifts emotional direction, if applicable]
    exit_move: [how the lead transitions to the next section]

For Silver specimens:
  Extract skeleton and structural_signature ONLY
  SKIP detailed element proportion analysis
  SKIP function annotation beyond primary category
```

### Phase 3: Technique Catalog Compilation

Aggregate across ALL decomposed specimens:

```
opening_devices:
  FOR each unique opening_device type found in specimens:
    - device_name: string
    - count: integer
    - implementation_pattern: string (1-2 sentence description of HOW to execute)
    - gold_example: string (verbatim hook from a Gold specimen, if available)

hook_techniques:
  FOR each unique hook construction technique observed:
    - technique_name: string
    - psychological_mechanism: string (what makes it work)
    - word_count_range: string
    - specimens_using: [source_file references]

emotional_progressions:
  FOR each unique emotional_progression pattern:
    - progression: string ("from → to")
    - lead_types_using: [list]
    - frequency: integer
    - specimens_using: [source_file references]

transition_techniques:
  FOR each method of transitioning from lead to next section:
    - technique: string
    - example_fragment: string (2-3 sentences from Gold specimen)
    - function: string (what it accomplishes)
```

### Phase 4: Power Element Extraction

From Gold specimens ONLY:

```
hook_sentences:
  FOR each Gold specimen:
    - verbatim_hook: string (the actual hook sentence)
    - device_type: string
    - word_count: integer
    - power_source: string (what makes this hook specifically powerful)
    - source_file: string

transition_fragments:
  FOR each Gold specimen:
    - verbatim_fragment: string (2-3 sentences showing lead-to-body transition)
    - transition_type: string
    - what_it_bridges: string
    - source_file: string

emotional_markers:
  SCAN Gold specimen full_text for:
    - word_or_phrase: string (specific language triggering emotional response)
    - emotional_state: string (which state it activates)
    - usage_context: string (position in lead structure)
```

## Quality Gates

- [ ] Minimum 3 Gold specimens fully decomposed (skeleton + techniques + power elements)
- [ ] Minimum 5 Silver specimens template-decomposed (skeleton + signature)
- [ ] All structural templates have complete element sequences (no gaps)
- [ ] Technique catalog has minimum 3 opening devices cataloged
- [ ] Technique catalog has minimum 2 emotional progressions documented
- [ ] Power elements contain minimum 3 verbatim hooks from Gold specimens
- [ ] Every power element includes source provenance (source_file)
- [ ] Context window budget respected (templates ~40%, techniques ~35%, elements ~25%)
- [ ] No raw specimen full_text passed through undecomposed

---

## Anti-Patterns

| Anti-Pattern | Signal | Fix |
|---|---|---|
| **Raw Passthrough** | Full specimen text dumped into downstream skills without decomposition | Decompose into template + techniques + elements |
| **Surface Mimicry Bait** | Power elements are full paragraphs instead of targeted fragments | Extract sentences and phrases, not blocks |
| **Template Without Function** | Skeleton lists elements but not their function/purpose | Every element needs element_type + function |
| **Technique Without Implementation** | Catalog names techniques but doesn't explain how to use them | Every technique needs implementation_pattern |
| **Unprovenanced Elements** | Power elements lack source attribution | Every extraction includes source_file + tier |
