# Layer 0.1 — Upstream Package Loader

## Purpose

Load and parse all required upstream skill packages (mechanism, structure, promise, root cause, proof inventory, headline) into working memory for lead engineering. Extract the specific fields needed for the lead — the elevator pitch elements, the campaign thesis, the emotional frame, and the credibility assets.

---

## Input Requirements

```yaml
upstream_paths:
  mechanism_package: "04-mechanism/outputs/mechanism-package.json"
  structure_package: "08-structure/outputs/structure-package.json"
  promise_package: "05-promise/outputs/promise-package.json"
  root_cause_package: "03-root-cause/outputs/root-cause-package.json"
  proof_inventory: "02-proof-inventory/outputs/proof-inventory-output.json"
  headline: "user_provided OR upstream headline skill output"
```

---

## Execution Protocol

### Step 1: Load Mechanism Package

Extract for lead use:
- `mechanism.name` — the named mechanism for the elevator pitch
- `mechanism.type` — determines which E5 elements are easiest to demonstrate
- `mechanism.explanation_text` — reference only (NOT for the lead — this goes in the argument)
- `mechanism.e5_mechanism_type` — E1-E5 classification for novelty framing
- `mechanism.uniqueness_claim` — the "new and different" element
- `mechanism.proof_elements` — credibility anchors for brief insertion

### Step 2: Load Structure Package

Extract for lead setup:
- `campaign_thesis.statement` — what the campaign argument will prove (lead previews this)
- `argument_strategy.type` — prosecution/revelation/education/demonstration (affects lead tone)
- `flow_architecture.simple_segue` — what the lead must transition INTO
- `downstream_handoffs.for_lead_writing` — specific data the structure skill prepared for the lead
  - `thesis_statement`
  - `primary_promise_echo`
  - `emotional_entry_point`
  - `first_chunk_preview`

### Step 3: Load Promise Package

Extract for elevator pitch:
- `primary_promise.statement` — the BIG promise for the lead
- `primary_promise.emotional_frame` — the emotional context
- `supporting_promises` — additional benefits for the elevator pitch
- `promise_ceiling` — maximum defensible promise level

### Step 4: Load Root Cause Package

Extract for counter-intuitive information:
- `root_cause.expression` — the root cause statement
- `root_cause.three_part_structure.what_they_think` — for counter-intuitive teasing
- `root_cause.three_part_structure.what_it_really_is` — for "surprising" info hint
- `root_cause.villain_profile` — for problem callout framing (if villain exists)

### Step 5: Load Proof Inventory

Extract for credibility insertion:
- `summary.overall_strength` — determines credibility confidence level
- `rankings.knockout_proof` — potential credibility anchor for brief insertion
- Top authority/credential proof elements — for lead credibility insertion

### Step 6: Load Headline

Extract:
- `main_headline` — the lead must continue the headline's emotional momentum
- `subheadline` (if present) — additional context
- Headline's implied promise and mechanism reference

### Step 7: Build Lead Context Object

Merge all extracted data into a single `lead_context` object.
