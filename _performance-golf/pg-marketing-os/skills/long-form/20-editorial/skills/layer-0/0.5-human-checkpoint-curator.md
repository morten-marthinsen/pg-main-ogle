# Microskill 0.5: Human Checkpoint Curator

**Layer:** 0 (Foundation)
**Purpose:** Present editorial approach to human for confirmation before proceeding

---

## CHECKPOINT PRESENTATION

Present the following to the human for confirmation:

### 1. Copy Summary

```yaml
copy_overview:
  total_word_count: "[count]"
  sections_present: "[list]"
  format_detected: "[VSL | Magalog | Sales Letter]"
  estimated_read_time: "[minutes]"

  from_assembly:
    assembly_quality_score: "[if available]"
    threading_status: "[if available]"
    callback_status: "[if available]"
```

### 2. Editorial Approach

```yaml
editorial_approach:
  lenses_to_apply:
    - Makepeace (Flow & Momentum)
    - Halbert (Entertainment & Engagement)
    - Schwartz (Sophistication & Awareness)
    - Ogilvy (Credibility & Clarity)
    - Clemens (Mechanism Architecture)
    - Kennedy (DR Fundamentals)

  evaluation_method:
    - Blind read first (capture gut reactions)
    - 6-lens critique (parallel evaluation)
    - 5-tier scoring (structure, principles, context, craft, benchmarks)
    - Priority fix application (highest impact first)
    - Anti-slop final pass
    - Polish pass

  output_format:
    - Silent critique (you see only polished output)
    - Final edited-draft.md
    - Quality grade (targeting A- or higher)
```

### 3. Context Confirmation

Confirm or adjust these context elements:

```yaml
context_to_confirm:
  sophistication_level:
    detected: "[1-5]"
    confirm_or_adjust: "[ ]"

  format:
    detected: "[VSL | Magalog | Sales Letter]"
    confirm_or_adjust: "[ ]"

  tone:
    from_brief: "[tone]"
    confirm_or_adjust: "[ ]"

  price_point:
    from_offer: "[$X]"
    confirm_or_adjust: "[ ]"

  primary_focus:
    options:
      - "Balanced edit (all lenses equal weight)"
      - "Flow-focused (Makepeace priority)"
      - "Credibility-focused (Ogilvy priority)"
      - "Mechanism-focused (Clemens priority)"
      - "Conversion-focused (Kennedy priority)"
    selected: "[ ]"
```

### 4. Priority Areas (Optional)

Ask if there are specific areas of concern:

```yaml
priority_areas:
  question: "Any specific areas you want extra attention on?"

  common_options:
    - "Opening/hook needs work"
    - "Transitions feel choppy"
    - "Proof isn't compelling"
    - "Close isn't strong enough"
    - "Overall flow issues"
    - "None - balanced edit is fine"

  user_selection: "[ ]"
```
