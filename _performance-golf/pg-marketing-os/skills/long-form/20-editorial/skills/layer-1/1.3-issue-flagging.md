# Microskill 1.3: Issue Flagging

**Layer:** 1 (Blind Read Assessment)
**Purpose:** Flag obvious problems for priority attention based on first-read impressions

---

## ISSUE SEVERITY LEVELS

| Severity | Definition | Action |
|----------|------------|--------|
| **CRITICAL** | Would prevent purchase / breaks the copy | MUST FIX before delivery |
| **MAJOR** | Significantly weakens effectiveness | SHOULD FIX in this pass |
| **MINOR** | Suboptimal but not damaging | FIX IF TIME permits |
| **STYLISTIC** | Preference/polish level | OPTIONAL improvement |

---

## OBVIOUS PROBLEM SCAN

Based on first-read impressions, flag obvious issues:

### Opening Issues

```yaml
opening_issues:
  - issue_id: "OPEN_001"
    severity: "[CRITICAL | MAJOR | MINOR]"
    description: "[What's wrong]"
    location: "[First X words / paragraph]"
    symptom: "[What you experienced as reader]"
    example_quote: "[Problematic text if identifiable]"
```

**Common Opening Problems:**
- Hook doesn't grab (MAJOR)
- Promise unclear (MAJOR)
- Too long before getting to point (MINOR)
- Sounds like every other ad (MAJOR)
- Generic/boring first line (MAJOR)

### Flow Issues

```yaml
flow_issues:
  - issue_id: "FLOW_001"
    severity: "[CRITICAL | MAJOR | MINOR]"
    description: "[What's wrong]"
    location: "[Section or transition point]"
    symptom: "[Where momentum died / jarring shift]"
```

**Common Flow Problems:**
- Jarring section transition (MAJOR)
- Momentum dies mid-section (MAJOR)
- Energy drops before close (CRITICAL)
- Sections feel disconnected (MAJOR)
- Repetitive rhythm/monotony (MINOR)

### Believability Issues

```yaml
believability_issues:
  - issue_id: "BELIEVE_001"
    severity: "[CRITICAL | MAJOR | MINOR]"
    description: "[What's wrong]"
    location: "[Where credibility faltered]"
    symptom: "[What made you doubt]"
```

**Common Believability Problems:**
- Unsubstantiated claims (CRITICAL)
- "Too good to be true" feeling (MAJOR)
- Missing proof where needed (MAJOR)
- Generic credentials (MINOR)
- Exaggerated language (MINOR)

### Clarity Issues

```yaml
clarity_issues:
  - issue_id: "CLARITY_001"
    severity: "[CRITICAL | MAJOR | MINOR]"
    description: "[What's confusing]"
    location: "[Where confusion occurred]"
    symptom: "[What you didn't understand]"
```

**Common Clarity Problems:**
- Mechanism not understood (CRITICAL)
- Offer unclear (CRITICAL)
- Price confusing (CRITICAL)
- Jargon without explanation (MAJOR)
- Convoluted sentences (MINOR)

### Emotional Issues

```yaml
emotional_issues:
  - issue_id: "EMOTION_001"
    severity: "[CRITICAL | MAJOR | MINOR]"
    description: "[What's missing emotionally]"
    location: "[Section or moment]"
    symptom: "[Lack of feeling/connection]"
```

**Common Emotional Problems:**
- Flat emotional register throughout (MAJOR)
- No desire activation (CRITICAL)
- Story not engaging (MAJOR)
- Close lacks urgency (CRITICAL)
- Generic, impersonal voice (MINOR)

### Close Issues

```yaml
close_issues:
  - issue_id: "CLOSE_001"
    severity: "[CRITICAL | MAJOR | MINOR]"
    description: "[What's weak in the close]"
    location: "[Close section]"
    symptom: "[Why you wouldn't buy]"
```

**Common Close Problems:**
- Weak CTA (CRITICAL)
- Fake/manufactured urgency (MAJOR)
- Risk not reversed (MAJOR)
- Unclear next steps (CRITICAL)
- Value not justified (MAJOR)
