## PURPOSE

Calibrates the 13-step emotional architecture across the full product introduction section. The TIER1 analysis revealed that all 12 elite controls follow a universal emotional sequence — a carefully orchestrated journey from negative (problem) through neutral-positive (mechanism) to peak positive (product reveal) to escalating (value) to relief (price) to security (guarantee) to urgency (scarcity) to peak desire (future pacing) to decisive (binary choice).

This microskill does NOT rewrite copy. It CALIBRATES — verifying that each section hits its intended emotional target, identifying any emotional dips or plateaus in the wrong places, and flagging the 3 critical emotional shifts that must occur. Steps 1-5 (pre-reveal) were handled by Skills 10-11. This skill is responsible for steps 6-13 (the reveal through close setup).

---

## IDENTITY

**This microskill IS:**
- An emotional architecture auditor that verifies each section hits its emotional target
- A sequence integrity checker that ensures no emotional dips or plateaus at wrong points
- A critical shift verifier for the 3 transformational emotional moments
- A calibration engine that identifies where emotional intensity needs adjustment
- A refinement tool — it adjusts existing drafts, not writes new ones

**This microskill is NOT:**
- A copywriter — it calibrates existing drafts from Layer 2 and Layer 3
- An emotional manipulation system — it ensures NATURAL emotional flow, not artificial spikes
- A pre-reveal calibrator (steps 1-5 are Skill 10-11's responsibility)
- A structural decision-maker (structure was set in Layer 1)

## PROCESS

### Step 1: Map Current Emotional State from Upstream

1. Read the emotional_state at handoff from Skill 17:
   - This tells us WHERE the reader is emotionally when entering the product introduction
   - Expected state: HIGH POSITIVE (mechanism understood, desire for solution, proof seen)
   - Expected belief level: HIGH or LOCKED

2. This microskill calibrates steps 6-13:
   ```
   Step 6: Product Introduction (relief + anticipation) — PEAK POSITIVE
   Step 7: Component/Ingredient Reveal (confidence) — SUSTAINED POSITIVE
   Step 8: Value Stacking (greed, excitement) — ESCALATING
   Step 9: Price Reveal (surprise, delight at low price) — RELIEF
   Step 10: Guarantee (safety, zero-risk feeling) — SECURITY
   Step 11: Scarcity/Urgency (fear of loss, FOMO) — URGENCY
   Step 12: Future Pacing (desire, visualization) — PEAK DESIRE
   Step 13: Binary Choice Close (fear of inaction) — DECISIVE
   ```

### Step 2: Calibrate Step 6 — Product Introduction (Bridge + Reveal)

3. Target emotional state: PEAK POSITIVE (relief + anticipation)
4. Read bridge_text (2.1) and reveal_text (2.2) together
5. Emotional checklist:
   - Does the bridge create RELIEF? ("Finally, there IS a solution")
   - Does the bridge create ANTICIPATION? ("I'm about to learn what it is")
   - Does the reveal deliver SATISFACTION? ("This sounds exactly right")
   - Does the reveal create DESIRE TO POSSESS? ("I want this")
   - Is there any DOUBT introduced? (should be zero at this point)
6. If emotional target NOT met:
   - Flag: "Step 6 emotional target missed — [specific issue]"
   - Recommend: [specific adjustment to bridge or reveal text]
   - Common fixes: Strengthen the relief language in bridge, add more drama to name reveal

### Step 3: Calibrate Step 7 — Component Reveal (Confidence)

7. Target emotional state: SUSTAINED POSITIVE (confidence in the product)
8. Read component_reveal_text (2.3)
9. Emotional checklist:
   - Does each component BUILD confidence? ("This is really well-researched")
   - Does the proof parade create CUMULATIVE conviction? ("Evidence keeps stacking up")
   - Is there any CONFUSION? (components too complex → loss of engagement)
   - Does the section maintain ENERGY? (no emotional dip in the middle)
   - Is there MOMENTUM toward the value section?
10. If emotional target NOT met:
    - Flag: "Step 7 emotional target missed — [specific issue]"
    - Common fixes: Simplify complex components, add stronger proof to weak components, vary proof types to maintain interest

### Step 4: Calibrate Step 8 — Value Stacking (Escalating Greed)

11. Target emotional state: ESCALATING (greed, excitement — "I'm getting so much")
12. Read value_stack_text (2.4)
13. Emotional checklist:
    - Does each bonus increase EXCITEMENT? ("And ALSO this?")
    - Does the running value total create ESCALATING GREED? ("This is worth way more than I expected")
    - Is there a "wow" moment? (one bonus that creates "that alone is worth it")
    - Does the total value statement create DELIGHT? ("All of this?")
    - If no bonuses (authority play): Does the product's intrinsic value feel SUFFICIENT?
14. If emotional target NOT met:
    - Flag: "Step 8 emotional target missed — [specific issue]"
    - Common fixes: Strengthen bonus descriptions, add worth-entire-course anchor, increase stated values

### Step 5: Calibrate Step 9 — Price Reveal (Relief/Delight)

15. Target emotional state: RELIEF (surprise and delight at low price)
16. Read price_reveal_text (3.1)
17. Emotional checklist:
    - Does the anchoring create EXPECTATION of high price? ("I'm expecting to pay $hundreds or $thousands")
    - Does the actual price create SURPRISE? ("Wait, that's IT?")
    - Does the per-day reframe create DELIGHT? ("That's less than coffee")
    - Is there any STICKER SHOCK? (price feels higher than expected → FAIL)
    - Does the reader feel they are SAVING money? (not spending money)
18. If emotional target NOT met:
    - Flag: "Step 9 emotional target missed — [specific issue]"
    - Common fixes: Add more anchor points, strengthen comparison, add per-day reframe

### Step 6: Calibrate Step 10 — Guarantee (Security)

19. Target emotional state: SECURITY (safety, zero-risk feeling)
20. Read guarantee_text (3.2)
21. Emotional checklist:
    - Does the guarantee create SAFETY? ("I can't lose")
    - Does the branded name create CONFIDENCE? ("They're so sure it works")
    - Does the amplification create ZERO RISK feeling? ("There is literally no downside")
    - Is there any RESIDUAL DOUBT? (guarantee conditions too complex → doubt persists)
    - Does the reader feel PROTECTED? (ready to act without fear)
22. If emotional target NOT met:
    - Flag: "Step 10 emotional target missed — [specific issue]"
    - Common fixes: Simplify guarantee terms, strengthen confidence amplification, add risk reversal

### Step 7: Calibrate Step 11 — Scarcity (Urgency)

23. Target emotional state: URGENCY (fear of loss, FOMO)
24. Read scarcity_text (3.2)
25. Emotional checklist:
    - Does the scarcity create FEAR OF MISSING OUT? ("I might not be able to get this later")
    - Is the justification BELIEVABLE? ("That's a real reason, not a tactic")
    - Does the scarcity feel JUSTIFIED, not manipulative?
    - Is there a shift from "should I buy?" to "can I still get this?"
    - Is the urgency PROPORTIONAL? (not over-the-top for a moderate claim)
26. If emotional target NOT met or scarcity absent:
    - If absent: Note that scarcity is not present — close must rely on future pacing alone
    - If present but weak: Flag and recommend stronger justification

### Step 8: Verify the 3 Critical Emotional Shifts

27. The 3 critical emotional shifts from TIER1 analysis must ALL occur:

**SHIFT 1: "I want this" → "I can have this"**
- Location: Bridge moment → Product reveal (Steps 6)
- Mechanism: Abstract desire becomes concrete possibility through product naming
- Check: Read bridge + reveal. Does the reader move from wanting to knowing they CAN have?
- If missing: Bridge doesn't deliver the product as accessible → strengthen accessibility language

**SHIFT 2: "This costs money" → "This is a steal"**
- Location: Value stack → Price reveal (Steps 8-9)
- Mechanism: Total value perception makes actual price feel trivially low
- Check: Read value stack + price reveal together. Does the reader feel they are SAVING?
- If missing: Value not dramatic enough or anchoring insufficient → strengthen value or anchoring

**SHIFT 3: "What if it doesn't work?" → "What if I miss out?"**
- Location: Guarantee → Scarcity (Steps 10-11)
- Mechanism: Guarantee eliminates downside; scarcity introduces FOMO
- Check: Read guarantee + scarcity together. Does the calculus shift from risk to regret?
- If missing: Guarantee doesn't feel safe enough or scarcity doesn't feel real → strengthen both

28. ALL 3 shifts must be present. If any is missing, flag as CRITICAL calibration issue.

### Step 9: Check for Emotional Dips and Plateaus

29. Map the emotional trajectory across all sections:

```
Section          | Expected Emotion    | Expected Intensity
---------------------------------------------------------
Bridge           | relief + anticipation | RISING (8→9)
Product Reveal   | satisfaction + desire | PEAK (9→10)
Component Reveal | confidence           | SUSTAINED (8-9)
Value Stack      | greed + excitement   | ESCALATING (8→9→10)
Price Reveal     | surprise + delight   | RELIEF DIP then RISE (7→9)
Guarantee        | safety + security    | SECURITY (8)
Scarcity         | urgency + FOMO       | URGENCY SPIKE (9)
Future Pacing    | desire + visualization| PEAK DESIRE (10)
Binary Choice    | fear + decisiveness  | DECISIVE (9→ACTION)
```

30. Identify any EMOTIONAL DIPS (intensity drops by 3+ points unexpectedly):
    - Allowed dip: Price reveal can briefly dip (money topic) before rising on deal feeling
    - Disallowed dip: Component reveal dropping to low engagement (boring ingredient list)
    - Disallowed dip: Value stack losing energy (generic bonuses)

31. Identify any EMOTIONAL PLATEAUS (same intensity for 3+ sections):
    - Allowed plateau: Component reveal at sustained 8-9 (confidence building)
    - Disallowed plateau: Everything at 7 (no peaks, no escalation — flat)

## FAILURE MODES

### Failure Mode 1: Flat Emotional Trajectory
- **Symptom:** Everything reads at the same emotional level — no peaks, no valleys, no escalation
- **Cause:** Copy is competent but lacks emotional dynamics
- **Prevention:** Step 9 plateau detection with 3+ section flag
- **Recovery:** Identify which sections need intensity boost; recommend specific amplification

### Failure Mode 2: Missing Critical Shift
- **Symptom:** Reader goes from "I want this" directly to "how much?" without feeling "I CAN have this"
- **Cause:** Bridge/reveal doesn't execute Shift 1 (desire → possession possibility)
- **Prevention:** Step 8 explicit verification of all 3 shifts
- **Recovery:** Flag specific missing shift; recommend where to insert transitional language

### Failure Mode 3: Sticker Shock at Price
- **Symptom:** Emotional trajectory crashes at price reveal — reader feels cost, not deal
- **Cause:** Shift 2 (costs → steal) not executed — insufficient anchoring or value
- **Prevention:** Step 5 price calibration checks for surprise/delight vs. shock
- **Recovery:** Strengthen anchoring, add anchor points, strengthen per-day reframe

### Failure Mode 4: Urgency Feels Manipulative
- **Symptom:** Reader distrust spikes at scarcity — feels "salesy" for first time
- **Cause:** Scarcity section too aggressive or justification weak
- **Prevention:** Step 7 checks scarcity proportionality and justification believability
- **Recovery:** Soften urgency language, strengthen justification

## EXAMPLES

### Example 1: Well-Calibrated Architecture (Montezuma Pattern)

```
Step 6 (Bridge + Reveal): PEAK POSITIVE (10/10)
  - Manufacturing quest builds anticipation → "$120K invested" creates earned feeling
  - "I'm finally able to share this" → relief + possession
  - Shift 1 STRONG: "I want this" → "It's called Montezuma's Secret" → "I can have this"

Step 7 (Components): SUSTAINED POSITIVE (9/10)
  - Each ingredient has authority proof (Harvard, Examine.com, Pfizer patent)
  - Cumulative confidence builds — parade of proof feeling achieved
  - No dips — each ingredient is interesting and different

Step 8 (Value Stack): ESCALATING (8/10)
  - 5 bonuses with values → running total builds greed
  - "But wait" escalation pattern maintains momentum
  - Shift 2 SETUP: Total value $136.92 + product value ($400+) = dramatic gap

Step 9 (Price Reveal): RELIEF (9/10)
  - Cascade $400 → $300 → $200 → $149.99 → $99 → $69
  - Per-day: "<$2/day"
  - Shift 2 STRONG: "$400 to produce" → "$69 today" → "this is a steal"

Step 10 (Guarantee): SECURITY (9/10)
  - "Triple-Back-Guarantee" — branded, extreme, memorable
  - "I'll give you TRIPLE your money back" → zero risk
  - Shift 3 BEGINS: "What if it doesn't work?" answered definitively

Step 11 (Scarcity): URGENCY (8/10)
  - "First 500 orders" + "strict limit 6" — quantity limited
  - Justified by rare ingredient sourcing
  - Shift 3 COMPLETE: "What if I miss out?" replaces doubt

All 3 shifts: PRESENT AND STRONG
Dips: 0
Plateaus: 0
Calibration score: 9.2
```

### Example 2: Calibration Issue — Missing Shift 2 (Hypothetical)

```
Step 8 (Value Stack): MODERATE (6/10)
  - 3 bonuses but values feel inflated, not specific
  - No "worth entire course" anchor
  - Total value: $285 vs. $97 price (ratio only 2.9:1)

Step 9 (Price Reveal): STICKER SHOCK (4/10)
  - Only 2 anchor points ($199 → $97)
  - No per-day reframe
  - No comparison anchor
  - Reader thinks "$97 is a lot" instead of "$97 is nothing"

ISSUE: Shift 2 ABSENT
  - Reader does NOT feel "this is a steal"
  - Anchoring insufficient + value stack not dramatic enough

RECOMMENDATION:
  - Add 2-3 more anchor points in price cascade
  - Add per-day minimizer ("$3.23 per day")
  - Add comparison anchor ("less than a single golf lesson")
  - Strengthen bonus values or add worth-entire-course anchor
```
