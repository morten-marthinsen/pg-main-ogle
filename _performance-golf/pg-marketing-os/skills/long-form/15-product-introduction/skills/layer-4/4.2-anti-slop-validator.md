**Position:** Second microskill in Layer 4
**Persona:** Sarah A. Conco — Client Protection (Primary), Dr. Richard Stern — Skeptical Academic (Secondary)

---

## PURPOSE

Performs a zero-tolerance scan of the entire product introduction narrative for generic, cliched, or undifferentiated commercial language. "Slop" is any language that could appear in any product introduction for any product in any niche — language that carries no specificity, no proof, no mechanism connection, and no emotional precision. Elite controls never use generic language. Every phrase earns its place through specificity.

Zero violations required. If the validator finds even one instance of banned language, the draft FAILS and must be remediated before assembly. This is the quality gate that prevents the product introduction from sounding like every other sales page on the internet.

---

## IDENTITY

**This microskill IS:**
- A zero-tolerance generic language scanner
- A banned-phrase detector across the full product introduction narrative
- A banned-pattern detector for structural slop (not just phrases)
- A specificity enforcer — every element must be concrete, not generic
- The final quality filter before assembly

**This microskill is NOT:**
- A style guide or tone checker
- A grammar or spelling checker
- A principle validator (that is 4.1)
- A creative editor — it detects slop, it does not write replacements (though it suggests them)
- A partial-credit system — zero violations required

## PROCESS

### Step 1: Compile Full Narrative Text

1. Concatenate all draft sections in sequence:
   - bridge_text + reveal_text + component_text + value_stack_text + price_reveal_text + guarantee_text + scarcity_text + future_pacing_text + binary_choice_text

2. This is the FULL product introduction narrative that will be scanned

### Step 2: Scan for Banned Phrases

3. Scan the full narrative for each of these BANNED PHRASES:

**Generic Urgency Phrases:**
- "Don't miss out"
- "Act now"
- "Limited time offer" (without specific justification IN the same sentence)
- "Once in a lifetime"
- "This won't last"
- "Hurry"
- "Before it's too late" (without specific consequence)
- "Time is running out"
- "Don't wait"
- "Grab yours now"
- "Claim yours before they're gone"
- "Supplies are running out" (without production justification)

**Generic Self-Worth Phrases:**
- "You deserve"
- "Treat yourself"
- "Invest in yourself"
- "You owe it to yourself"
- "Isn't it time you..."
- "You're worth it"
- "Give yourself the gift of..."
- "Take control of your life"
- "Be the best version of yourself"
- "Unlock your potential"
- "Live your best life"

**Generic Sales Phrases:**
- "But wait, there's more" (if used without genuine escalation context)
- "For a limited time only"
- "Exclusive offer"
- "Special deal"
- "Risk-free" (without specific guarantee details in same passage)
- "Satisfaction guaranteed" (without branded guarantee name)
- "No obligation"
- "What are you waiting for?"
- "Don't let this pass you by"
- "Take advantage of"
- "Click here now"
- "Order today and receive"

**Generic Aspirational Phrases:**
- "Imagine a better life"
- "Transform your life"
- "Change your life forever"
- "Reach your full potential"
- "The results will amaze you"
- "See results fast"
- "Works like magic"
- "Miracle solution"
- "Game-changer" (without specific game-change described)
- "Revolutionary" (without specific revolution described)
- "Breakthrough" (acceptable ONLY if tied to specific discovery)

4. For each phrase found:
   - Log: exact phrase, exact location in text, section where found
   - Flag as VIOLATION

### Step 3: Scan for Banned Patterns

5. Beyond individual phrases, scan for these STRUCTURAL slop patterns:

**Pattern A: Undifferentiated Bonus Descriptions**
- Check each bonus description in the value stack
- Does the description tell you WHAT the bonus does specifically?
- Or does it use generic language like "valuable resource," "comprehensive guide," "powerful tool"?
- VIOLATION if: Bonus description could apply to any bonus in any niche
- PASS if: Bonus description includes specific mechanism connection, specific outcome, or specific content preview

**Pattern B: Price Reveals Without Anchoring**
- Does the price appear without any prior anchor point?
- Is there a comparison, cascade, or value context before the number?
- VIOLATION if: Price number appears without anchoring context
- PASS if: At least one anchor point precedes the actual price

**Pattern C: Generic Guarantee Language**
- Does the guarantee use only standard refund language?
- Is there a branded name? Is the guarantee framed as a feature?
- VIOLATION if: "Money-back guarantee" without branded name or feature framing
- PASS if: Branded name + feature framing + confidence amplification

**Pattern D: Urgency Without Justification**
- Does any urgency/scarcity claim lack a stated reason?
- VIOLATION if: "Limited availability" without WHY it's limited
- PASS if: Every urgency claim has an adjacent justification

**Pattern E: Feature Lists Without Proof**
- Does any section list product features without attaching proof?
- VIOLATION if: "Contains [ingredient]" without proof package
- PASS if: Every feature/ingredient/component has proof or mechanism connection

**Pattern F: Motivational Close Without Specific Futures**
- Does the binary choice section use generic motivation?
- VIOLATION if: "Make the right choice" / "Choose a better future" without specifics
- PASS if: Both futures are vivid, specific, sensory, and tied to mechanism/villain

### Step 4: Scan for Filler and Empty Calories

6. Scan for sentences that add NO value:

**Empty Calorie Phrases:**
- "This is truly amazing"
- "I'm so excited to share this with you"
- "This is unlike anything you've ever seen"
- "Trust me on this"
- "I know what you're thinking"
- "You might be skeptical, and that's okay"
- "Let me be honest with you"
- "Here's the truth"
- "The fact is"
- "Believe it or not"
- "I'm not going to lie"

7. Note: Some of these phrases CAN be used if they precede SPECIFIC content. The violation is when they stand alone or precede more generic language.
   - "Trust me on this" → VIOLATION (empty)
   - "Trust me on this — I've treated over ten thousand patients with this exact protocol" → ACCEPTABLE (specific evidence follows)

8. Rule: If a filler phrase is followed by specific, provable content within the same sentence or next sentence, it is ACCEPTABLE. If it stands alone or is followed by more generic language, it is a VIOLATION.

### Step 5: Count and Classify Violations

9. Classify each violation by severity:

| Severity | Type | Examples |
|----------|------|---------|
| CRITICAL | Generic urgency without justification | "Act now — limited time!" |
| CRITICAL | Generic guarantee (no branding) | "100% satisfaction guaranteed" |
| CRITICAL | Feature list without proof | "Contains vitamin D, zinc, and ashwagandha" |
| HIGH | Generic aspirational language | "Transform your life forever" |
| HIGH | Undifferentiated bonus description | "A valuable bonus resource" |
| MODERATE | Empty calorie filler | "I'm so excited to share this" (alone) |
| MODERATE | Generic self-worth phrase | "You deserve this" |

10. Count violations by severity:
    - CRITICAL: Any count = FAIL
    - HIGH: Count > 2 = FAIL
    - MODERATE: Count > 5 = FAIL
    - But the STANDARD is ZERO violations across ALL severities

### Step 6: Generate Replacement Suggestions

11. For each violation, suggest a specific replacement:

**Example replacements:**
```
VIOLATION: "Don't miss out on this amazing offer"
REPLACEMENT: "Remember — the first 500 orders receive this pricing. After that, the cost returns to $149.99."
REASON: Generic urgency → specific, justified scarcity

VIOLATION: "You deserve to feel confident again"
REPLACEMENT: "In 33 of 39 couples tested, men reported renewed confidence within the first week."
REASON: Generic self-worth → specific proof-based statement

VIOLATION: "This powerful supplement contains..."
REPLACEMENT: "Each capsule delivers 213mg of L-glutamine — the exact dosage shown in a 258-person trial to restore gut barrier integrity."
REASON: Generic feature → specific proof with dosage and clinical data

VIOLATION: "Imagine a better life"
REPLACEMENT: "Imagine stepping on the scale 30 days from now and seeing a number you haven't seen since your twenties."
REASON: Generic aspiration → specific, sensory, time-framed visualization

VIOLATION: "Satisfaction guaranteed"
REPLACEMENT: "You're protected by the Triple-Back-Guarantee — if it doesn't work, I'll refund triple your investment."
REASON: Generic guarantee → branded, extreme, specific
```

### Step 7: Generate Final Verdict

12. Compile anti_slop_report:

```yaml
anti_slop_report:
  full_narrative_scanned: boolean
  total_violations: integer  # Must be 0 to pass
  violations_by_severity:
    critical: integer
    high: integer
    moderate: integer
  violations_list:
    - phrase: string
      location: string  # Section + approximate position
      severity: enum[critical, high, moderate]
      category: string  # urgency, self-worth, sales, aspirational, filler, structural
      suggested_replacement: string
      replacement_reason: string
  banned_patterns_detected:
    undifferentiated_bonuses: boolean
    price_without_anchor: boolean
    generic_guarantee: boolean
    urgency_without_justification: boolean
    features_without_proof: boolean
    motivational_close_without_specifics: boolean
  verdict:
    pass: boolean  # true only if total_violations == 0
    total_violations: integer
    sections_with_violations: [string]
    remediation_required: boolean
```

## FAILURE MODES

### Failure Mode 1: False Positive on Contextual Phrase
- **Symptom:** "Breakthrough" flagged even when describing a specific discovery (Pepsin GI)
- **Cause:** Scanner checks phrase without checking context
- **Prevention:** Step 4 context rule: filler/generic phrases acceptable when followed by specifics
- **Recovery:** Review flagged phrase in context; dismiss if specific content follows

### Failure Mode 2: Structural Slop Missed
- **Symptom:** Bonus descriptions are generic but individual phrase scan finds no violations
- **Cause:** Only phrase-level scanning, not pattern-level scanning
- **Prevention:** Step 3 banned patterns check for structural issues
- **Recovery:** Re-scan with pattern-level analysis

### Failure Mode 3: Zero-Tolerance Not Enforced
- **Symptom:** "Only 2 moderate violations — close enough to pass"
- **Cause:** Validator allows partial credit
- **Prevention:** Step 5 explicitly: total_violations must equal 0 for PASS
- **Recovery:** Enforce zero tolerance; route all violations for remediation

### Failure Mode 4: Replacement Suggestions Are Also Generic
- **Symptom:** Replacement for "Don't miss out" is "Don't let this opportunity pass"
- **Cause:** Suggestions generated without product-specific data
- **Prevention:** Replacements must reference specific product data (dosages, names, proof, numbers)
- **Recovery:** Re-generate replacements using confirmed product package data
