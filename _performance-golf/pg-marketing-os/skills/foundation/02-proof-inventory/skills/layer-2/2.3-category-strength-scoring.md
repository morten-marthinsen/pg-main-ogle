# 2.3 Category Strength Scoring

## Purpose
Calculate the overall strength score for each of the 6 proof categories. Category strength scores reveal where proof is robust vs. weak, enabling gap analysis and strategic recommendations.

---

## Input Requirements

From 2.2 Schwartz Stage Adjustment:
- All elements with stage-adjusted scores
- Quality tiers assigned
- Elements grouped by category

---

## Category Strength Formula

Category strength is NOT a simple average—it weights strong elements higher because one excellent proof is worth more than several mediocre ones.

```python
def calculate_category_strength(elements):
    if len(elements) == 0:
        return {
            'count': 0,
            'strength': 0,
            'status': 'missing',
            'quality_distribution': {'strong': 0, 'medium': 0, 'weak': 0}
        }

    # Quality distribution
    strong = len([e for e in elements if e.adjusted_score >= 7.0])
    medium = len([e for e in elements if 4.0 <= e.adjusted_score < 7.0])
    weak = len([e for e in elements if e.adjusted_score < 4.0])

    # Weighted strength calculation
    # Strong elements contribute more per element
    weighted_sum = (strong * 10) + (medium * 6) + (weak * 2)
    weighted_count = (strong * 3) + (medium * 2) + (weak * 1)

    strength = weighted_sum / weighted_count if weighted_count > 0 else 0

    # Determine status
    status = determine_status(strong, medium, weak, len(elements))

    return {
        'count': len(elements),
        'strength': round(strength, 1),
        'status': status,
        'quality_distribution': {
            'strong': strong,
            'medium': medium,
            'weak': weak
        }
    }
```

## Execution Protocol

### Step 1: Group Elements by Category

```python
def group_by_category(elements):
    categories = {
        'social': [],
        'authority': [],
        'demonstration': [],
        'mechanism': [],
        'data': [],
        'risk_reversal': []
    }

    for element in elements:
        category = element.category.lower()
        categories[category].append(element)

    return categories
```

### Step 2: Calculate Quality Distribution

For each category, count elements by tier:

```yaml
quality_distribution:
  strong: 5   # Score >= 7.0
  medium: 8   # Score 4.0 - 6.9
  weak: 3     # Score < 4.0
```

### Step 3: Calculate Weighted Strength

The weighting ensures strong elements have outsized impact:

```
WEIGHT RATIONALE:

Strong elements (7+):
├── Contribution: 10 points per element
├── Weight: 3x
├── Why: One great proof is worth three weak ones

Medium elements (4-6.9):
├── Contribution: 6 points per element
├── Weight: 2x
├── Why: Supporting value but not headline-worthy

Weak elements (<4):
├── Contribution: 2 points per element
├── Weight: 1x
├── Why: Minimal value, may hurt credibility

FORMULA:
strength = (strong*10 + medium*6 + weak*2) / (strong*3 + medium*2 + weak*1)
```

### Step 4: Determine Category Status

```python
def determine_status(strong, medium, weak, total):
    if total == 0:
        return 'missing'

    if strong >= 3:
        return 'robust'  # Excellent coverage

    if strong >= 1 and medium >= 2:
        return 'adequate'  # Sufficient for use

    if strong >= 1 or medium >= 3:
        return 'moderate'  # Has potential but needs work

    if total > 0:
        return 'weak'  # Present but low quality

    return 'missing'
```

**Status Definitions:**

| Status | Criteria | Recommendation |
|--------|----------|----------------|
| **robust** | 3+ strong elements | Use prominently, strong foundation |
| **adequate** | 1+ strong, 2+ medium | Sufficient, may want to strengthen |
| **moderate** | 1 strong OR 3+ medium | Needs improvement, use carefully |
| **weak** | Only weak elements | Consider discovery/generation |
| **missing** | 0 elements | Critical gap, prioritize filling |

## Category-Specific Considerations

### SOCIAL Strength

```
SOCIAL CONSIDERATIONS:
├── Diversity matters: Mix of testimonial types is stronger
├── Attribution quality: Named > anonymous
├── Cluster value: Multiple similar testimonials = confidence
└── Video premium: Video testimonials weight higher

IDEAL SOCIAL PROFILE:
├── 1+ transformation testimonial (strong)
├── 3+ specific result testimonials (strong/medium)
├── User count or review volume (supporting)
└── At least 1 video or photo testimonial
```

### AUTHORITY Strength

```
AUTHORITY CONSIDERATIONS:
├── Study tier matters: Peer-reviewed > company study
├── Credential relevance: Field-specific > general
├── Third-party premium: Independent validation weights higher
└── Recency factor: Recent research > dated

IDEAL AUTHORITY PROFILE:
├── 1+ peer-reviewed study (strong)
├── Expert endorsement with credentials (strong)
├── Certifications/validations (supporting)
└── Media coverage if available
```

### MECHANISM Strength

```
MECHANISM CONSIDERATIONS:
├── Scientific backing: Study-backed > logical chain
├── Uniqueness premium: Proprietary mechanism scores higher
├── Explainability: Clear explanation > complex
└── Stage 3+ critical: Mechanism proof essential in crowded markets

IDEAL MECHANISM PROFILE:
├── 1+ mechanism study (strong)
├── Expert explanation (strong/medium)
├── Logical chain or analogy (supporting)
└── Clear, memorable mechanism name
```
