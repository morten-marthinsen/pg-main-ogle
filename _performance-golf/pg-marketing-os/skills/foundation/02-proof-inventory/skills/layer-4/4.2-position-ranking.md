# 4.2 Position Ranking

## Purpose
Map and rank proof elements to optimal copy positions (lead, mechanism, body, close). Different proof types perform differently in different positions. This microskill ensures the right proof goes in the right place.

---

## Input Requirements

From Layer 1-2:
- All scored proof elements
- Category and sub-type classifications

From 4.1 Knockout Selection:
- Knockout proof identified
- Runner-up proof

From Context:
- Schwartz stage
- Lead type (if known)
- Copy format (VSL, sales letter, email)

---

## Copy Position Definitions

```
COPY POSITIONS:

LEAD (Opening)
├── Purpose: Hook attention, establish credibility, create curiosity
├── Length: First 10-20% of copy
├── Proof role: Immediate validation, pattern interrupt
└── Best proof types: Shocking stats, transformation stories, demonstrations

MECHANISM SECTION
├── Purpose: Explain WHY it works, build belief in the HOW
├── Length: 20-30% of copy
├── Proof role: Validate the mechanism, scientific backing
└── Best proof types: Studies, expert quotes, mechanism explanations

BODY/PROOF STACK
├── Purpose: Build cumulative belief, overcome objections
├── Length: 30-40% of copy
├── Proof role: Multiple proof points, testimonial variety
└── Best proof types: Testimonial collection, varied proof types

CLOSE
├── Purpose: Final push, overcome last-minute resistance
├── Length: Final 10-15% of copy
├── Proof role: Strongest emotional proof + risk reversal
└── Best proof types: Transformation stories, guarantees, scarcity proof
```

### Step 2: Position-Specific Ranking

```python
def rank_for_position(elements, position, schwartz_stage, knockout_id=None):
    """
    Rank all elements for a specific copy position.
    """

    ranked = []

    for element in elements:
        affinity = calculate_position_affinity(element, position)
        quality = element.stage_adjusted_score

        # Combined position score
        position_score = (affinity * 0.6) + (quality * 0.4)

        # Knockout bonus for lead/close
        if element.id == knockout_id and position in ['lead', 'close']:
            position_score *= 1.3

        # Stage adjustments
        if schwartz_stage >= 3 and position == 'mechanism':
            if element.category == 'mechanism':
                position_score *= 1.2
            if element.category == 'authority':
                position_score *= 1.15

        ranked.append({
            'element': element,
            'position': position,
            'affinity': affinity,
            'quality': quality,
            'position_score': position_score
        })

    # Sort by position score
    ranked.sort(key=lambda x: x['position_score'], reverse=True)

    return ranked


def generate_position_rankings(elements, schwartz_stage, knockout):
    """
    Generate rankings for all four positions.
    """

    knockout_id = knockout['element'].id if knockout else None

    rankings = {
        'lead': rank_for_position(elements, 'lead', schwartz_stage, knockout_id)[:5],
        'mechanism': rank_for_position(elements, 'mechanism', schwartz_stage, knockout_id)[:5],
        'body': rank_for_position(elements, 'body', schwartz_stage, knockout_id)[:10],
        'close': rank_for_position(elements, 'close', schwartz_stage, knockout_id)[:5]
    }

    return rankings
```

### Step 3: Lead Type Optimization

```python
def optimize_for_lead_type(rankings, lead_type):
    """
    If lead type is known, further optimize lead position ranking.
    """

    LEAD_TYPE_PREFERENCES = {
        'story_lead': {
            'boost': ['testimonial_transformation', 'case_study'],
            'reduce': ['data_exact_statistic', 'study_headline_finding']
        },
        'stat_lead': {
            'boost': ['data_exact_statistic', 'data_shocking_stat', 'study_headline_finding'],
            'reduce': ['testimonial_general', 'testimonial_transformation']
        },
        'proof_lead': {
            'boost': ['demo_before_after_photo', 'study_clinical_trial', 'endorsement_celebrity'],
            'reduce': ['testimonial_general']
        },
        'news_lead': {
            'boost': ['study_headline_finding', 'data_shocking_stat', 'mechanism_study'],
            'reduce': ['testimonial_general', 'guarantee_unconditional']
        },
        'question_lead': {
            'boost': ['data_exact_statistic', 'study_headline_finding'],
            'reduce': ['testimonial_transformation']
        }
    }

    if lead_type not in LEAD_TYPE_PREFERENCES:
        return rankings

    prefs = LEAD_TYPE_PREFERENCES[lead_type]

    for item in rankings['lead']:
        sub_type = item['element'].sub_type
        if sub_type in prefs['boost']:
            item['position_score'] *= 1.25
        elif sub_type in prefs['reduce']:
            item['position_score'] *= 0.8

    # Re-sort
    rankings['lead'].sort(key=lambda x: x['position_score'], reverse=True)

    return rankings
```

### Step 4: Generate Position Guidance

```python
def generate_position_guidance(rankings):
    """
    Generate actionable guidance for each position.
    """

    guidance = {}

    for position, ranked_elements in rankings.items():
        top_element = ranked_elements[0] if ranked_elements else None

        guidance[position] = {
            'recommended': top_element,
            'alternatives': ranked_elements[1:3] if len(ranked_elements) > 1 else [],
            'integration_notes': get_integration_notes(position, top_element),
            'avoid': get_avoid_notes(position)
        }

    return guidance


def get_integration_notes(position, element):
    """
    Specific notes on how to integrate proof at this position.
    """

    if not element:
        return "No strong proof for this position"

    notes = {
        'lead': {
            'testimonial_transformation': '''
Start with "Meet [Name]..." or "Take [Name]'s story..."
Present the before/after arc quickly
Connect to reader: "If [Name] could do it, so can you"
''',
            'data_exact_statistic': '''
Open with the stat, framed as news/discovery
"[Stat] according to [source]"
Immediately connect to reader's problem
''',
            'demo_before_after_photo': '''
Visual first, then story
"Look at these results..."
Let the image do the heavy lifting
'''
        },
        'mechanism': {
            'study_peer_reviewed': '''
Introduce the research: "Researchers at [institution] discovered..."
Present key finding with number highlighted
Explain what this means: "In other words, [mechanism] works because..."
''',
            'mechanism_expert_explanation': '''
Lead with expert credentials
Present quote or explanation
Connect to your specific product/mechanism
'''
        },
        'body': {
            'testimonial_specific_result': '''
Use as proof stack element
Keep tight—focus on the specific result
Vary the proof types around it
''',
            'case_study': '''
More detailed than testimonial
Use subheads to break up
Include specific numbers throughout
'''
        },
        'close': {
            'testimonial_transformation': '''
Bring back transformation story
Emphasize the emotional payoff
"Your story could be next"
''',
            'guarantee_unconditional': '''
Present guarantee with confidence
Remove all perceived risk
Make it feel generous, not defensive
'''
        }
    }

    sub_type = element['element'].sub_type
    position_notes = notes.get(position, {})
    return position_notes.get(sub_type, f"Use this {sub_type} to strengthen {position}")
```
