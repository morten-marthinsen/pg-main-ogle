# 4.6 Big Idea Handoff Packaging

## Purpose
Package proof inventory findings specifically for the 05-Big-Ideas Skill. Big Ideas must be calibrated to available proof—a brilliant concept without proof support fails. This microskill creates a tailored handoff that enables Big Idea concept development and calibration.

---

## Input Requirements

From Layer 2:
- Overall strength score
- Category strength breakdown

From Layer 4:
- Knockout proof
- Proof themes/clusters
- Gradualization sequence

From Context:
- Schwartz stage
- Market sophistication factors

---

## Big Idea Skill Needs

```
WHAT THE BIG IDEA SKILL NEEDS TO KNOW:

1. OVERALL PROOF STRENGTH
   └── How bold can the Big Idea be?

2. KNOCKOUT PROOF
   └── What's the single most powerful proof for concept anchoring?

3. PROOF THEMES
   └── What clusters of proof tell a story for Big Idea development?

4. PROOF CONSTRAINTS
   └── What Big Idea angles are NOT supported by proof?

5. NOVELTY ASSESSMENT
   └── What proof is fresh/unique enough to build Big Idea around?

6. MECHANISM PROOF STATUS
   └── Is the mechanism provable enough for Stage 3+ Big Ideas?
```

---

## Execution Protocol

### Step 1: Overall Strength Packaging

```python
def package_overall_strength(inventory_summary, schwartz_stage):
    """
    Package overall proof strength for Big Idea calibration.
    """

    strength_package = {
        'overall_score': inventory_summary['overall_strength_score'],
        'strength_tier': None,
        'big_idea_implications': None,
        'calibration_notes': []
    }

    score = inventory_summary['overall_strength_score']

    # Determine tier
    if score >= 80:
        strength_package['strength_tier'] = 'exceptional'
        strength_package['big_idea_implications'] = '''
Big Idea can be BOLD and AMBITIOUS.
Proof is strong enough to support dramatic claims.
Can build Big Idea around transformation/breakthrough concepts.
'''
    elif score >= 60:
        strength_package['strength_tier'] = 'strong'
        strength_package['big_idea_implications'] = '''
Big Idea should be CONFIDENT but GROUNDED.
Proof supports significant claims but not hyperbole.
Build Big Idea around proven mechanism or validated results.
'''
    elif score >= 40:
        strength_package['strength_tier'] = 'moderate'
        strength_package['big_idea_implications'] = '''
Big Idea should be INTERESTING but CAREFUL.
Focus on novelty of approach rather than dramatic results.
Build Big Idea around unique angle, not outcome magnitude.
'''
    else:
        strength_package['strength_tier'] = 'limited'
        strength_package['big_idea_implications'] = '''
Big Idea must be CREATIVE and INDIRECT.
Cannot build on proof—focus on story, enemy, or identification.
Consider building proof before major promotion.
'''

    # Stage-specific calibration
    if schwartz_stage >= 4 and score < 60:
        strength_package['calibration_notes'].append(
            'WARNING: Stage 4+ requires strong proof for Big Idea credibility. '
            'Current proof may be insufficient for skeptical market.'
        )

    if schwartz_stage >= 3 and inventory_summary.get('mechanism_strength', 0) < 6:
        strength_package['calibration_notes'].append(
            'WARNING: Stage 3+ Big Ideas often depend on mechanism. '
            'Mechanism proof is weak—consider non-mechanism Big Idea angle.'
        )

    return strength_package
```

### Step 2: Knockout Proof Packaging

```python
def package_knockout_for_big_idea(knockout):
    """
    Package knockout proof for potential Big Idea anchoring.
    """

    if not knockout or not knockout.get('knockout_selected'):
        return {
            'knockout_available': False,
            'recommendation': 'No knockout proof—Big Idea cannot anchor on single proof element'
        }

    element = knockout['element']

    knockout_package = {
        'knockout_available': True,
        'element_id': element.id,
        'category': element.category,
        'sub_type': element.sub_type,
        'raw_text': element.raw_text,
        'knockout_potential': knockout['knockout_potential'],

        'big_idea_anchor_potential': None,
        'anchor_angles': [],
        'usage_recommendations': []
    }

    # Assess Big Idea anchor potential
    if element.sub_type == 'testimonial_transformation':
        knockout_package['big_idea_anchor_potential'] = 'high'
        knockout_package['anchor_angles'] = [
            'Story-based Big Idea: Build around this transformation story',
            'Identity Big Idea: "For people like Sarah who..."',
            'Discovery Big Idea: "What Sarah discovered after 20 years..."'
        ]

    elif element.sub_type in ['study_peer_reviewed', 'study_clinical_trial']:
        knockout_package['big_idea_anchor_potential'] = 'high'
        knockout_package['anchor_angles'] = [
            'Discovery Big Idea: "Researchers found..."',
            'News Big Idea: "New study reveals..."',
            'Mechanism Big Idea: "The science behind..."'
        ]

    elif element.sub_type in ['data_exact_statistic', 'data_shocking_stat']:
        knockout_package['big_idea_anchor_potential'] = 'medium'
        knockout_package['anchor_angles'] = [
            'Problem-based Big Idea: "78% of Americans..."',
            'Contrarian Big Idea: "What they don\'t tell you about..."'
        ]

    else:
        knockout_package['big_idea_anchor_potential'] = 'low'
        knockout_package['anchor_angles'] = [
            'Supporting role—not primary anchor material'
        ]

    knockout_package['usage_recommendations'] = [
        'Use knockout proof to SUPPORT Big Idea, even if not anchor',
        'Reference in Big Idea proof demonstration',
        'Build credibility bridge between Big Idea and proof'
    ]

    return knockout_package
```

### Step 3: Proof Theme Identification

```python
def identify_proof_themes(elements, mechanism):
    """
    Identify clusters of proof that tell a story—potential Big Idea themes.
    """

    themes = []

    # Theme 1: Real People, Real Results
    transformation_elements = [e for e in elements
                              if e.sub_type in ['testimonial_transformation', 'testimonial_specific_result']]
    if len(transformation_elements) >= 3:
        themes.append({
            'theme_name': 'Real People, Real Results',
            'theme_type': 'social_proof',
            'supporting_elements': [e.id for e in transformation_elements[:5]],
            'big_idea_angle': 'Story-driven Big Idea featuring testimonial narratives',
            'strength': calculate_theme_strength(transformation_elements)
        })

    # Theme 2: Backed by Science
    science_elements = [e for e in elements
                       if e.category in ['authority', 'mechanism']
                       and e.sub_type in ['study_peer_reviewed', 'study_clinical_trial',
                                         'mechanism_study', 'endorsement_expert']]
    if len(science_elements) >= 2:
        themes.append({
            'theme_name': 'Backed by Science',
            'theme_type': 'authority_proof',
            'supporting_elements': [e.id for e in science_elements[:5]],
            'big_idea_angle': 'Discovery/research-based Big Idea',
            'strength': calculate_theme_strength(science_elements)
        })

    # Theme 3: Proven Mechanism
    mechanism_elements = [e for e in elements if e.category == 'mechanism']
    if len(mechanism_elements) >= 2:
        themes.append({
            'theme_name': 'Proven Mechanism',
            'theme_type': 'mechanism_proof',
            'supporting_elements': [e.id for e in mechanism_elements[:5]],
            'big_idea_angle': 'Mechanism-based Big Idea (Stage 3+ ideal)',
            'mechanism_name': mechanism.get('name', ''),
            'strength': calculate_theme_strength(mechanism_elements)
        })

    # Theme 4: Risk-Free Opportunity
    risk_elements = [e for e in elements if e.category == 'risk_reversal']
    if risk_elements:
        themes.append({
            'theme_name': 'Risk-Free Opportunity',
            'theme_type': 'risk_reversal',
            'supporting_elements': [e.id for e in risk_elements],
            'big_idea_angle': 'Irresistible offer angle',
            'strength': calculate_theme_strength(risk_elements)
        })

    # Theme 5: Visual Proof
    visual_elements = [e for e in elements
                      if e.sub_type in ['demo_before_after_photo', 'demo_video']]
    if visual_elements:
        themes.append({
            'theme_name': 'See the Results',
            'theme_type': 'demonstration',
            'supporting_elements': [e.id for e in visual_elements],
            'big_idea_angle': 'Proof-first Big Idea (show before tell)',
            'strength': calculate_theme_strength(visual_elements)
        })

    # Sort themes by strength
    themes.sort(key=lambda t: t['strength'], reverse=True)

    return themes


def calculate_theme_strength(elements):
    """Calculate aggregate strength of a proof theme."""
    if not elements:
        return 0
    scores = [e.stage_adjusted_score for e in elements]
    return (sum(scores) / len(scores)) * (min(len(elements), 5) / 5)  # Quantity bonus
```

### Step 4: Novelty Assessment

```python
def assess_novelty_for_big_idea(elements, competitor_patterns):
    """
    Identify proof with novelty potential for Big Idea differentiation.
    """

    novelty_assessment = {
        'high_novelty_elements': [],
        'unique_angles': [],
        'saturated_angles': [],
        'differentiation_opportunities': []
    }

    # Find high-novelty elements
    for element in elements:
        if element.scores.novelty >= 8:
            novelty_assessment['high_novelty_elements'].append({
                'element_id': element.id,
                'novelty_score': element.scores.novelty,
                'what_makes_novel': analyze_novelty_source(element),
                'big_idea_potential': 'High—fresh angle for market'
            })

    # Identify saturated angles (to avoid)
    if competitor_patterns:
        saturated = competitor_patterns.get('common_claims', [])
        novelty_assessment['saturated_angles'] = saturated

    # Suggest differentiation opportunities
    if novelty_assessment['high_novelty_elements']:
        novelty_assessment['differentiation_opportunities'].append(
            'Build Big Idea around unique proof elements for market differentiation'
        )

    if not any(e for e in elements if 'testimonial_transformation' in e.sub_type
               and e.scores.novelty >= 7):
        novelty_assessment['differentiation_opportunities'].append(
            'Transformation testimonials appear common—seek unique story angle'
        )

    return novelty_assessment
```

### Step 5: Big Idea Constraints

```python
def package_big_idea_constraints(gaps, ceiling, schwartz_stage):
    """
    Document what Big Idea angles are NOT supported by proof.
    """

    constraints = []

    # Constraint from ceiling
    if ceiling['level'] < 4:
        constraints.append({
            'constraint': 'No "transformation" language in Big Idea',
            'reason': 'Proof ceiling below transformation level',
            'alternative': 'Focus on "improvement" or "discovery" angles'
        })

    # Constraint from mechanism gaps
    mechanism_gaps = [g for g in gaps if g.get('category') == 'mechanism'
                     and g['severity_tier'] in ['critical', 'high']]
    if mechanism_gaps and schwartz_stage >= 3:
        constraints.append({
            'constraint': 'Mechanism cannot be Big Idea centerpiece',
            'reason': 'Mechanism proof insufficient for Stage 3+ market',
            'alternative': 'Use story, enemy, or identification as Big Idea driver'
        })

    # Constraint from authority gaps
    authority_gaps = [g for g in gaps if g.get('category') == 'authority'
                     and g['severity_tier'] in ['critical', 'high']]
    if authority_gaps:
        constraints.append({
            'constraint': 'Cannot make "scientifically proven" type Big Idea',
            'reason': 'Authority proof insufficient',
            'alternative': 'Focus on testimonial/social proof anchored Big Idea'
        })

    # Constraint from data gaps
    data_gaps = [g for g in gaps if g.get('category') == 'data'
                and g['severity_tier'] in ['critical', 'high']]
    if data_gaps:
        constraints.append({
            'constraint': 'Cannot lead with statistics in Big Idea',
            'reason': 'Data proof weak or missing',
            'alternative': 'Lead with story, follow with available proof'
        })

    return constraints
```
