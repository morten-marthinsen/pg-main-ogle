# Skill 3.2: Quantification Validator

**Version:** 1.0
**Layer:** 3 (Validation)
**Type:** Validation Gate
**Dependencies:** layer2-output.json (from 2.6), layer1-output.json (from 1.6)
**Output:** quantification_validation.json

---

## Purpose

Ensure zero vague qualifiers exist in any output — all claims are backed by exact numbers. Scans all text fields across Layer 1 and Layer 2 outputs for 17 forbidden terms, reports exact violation locations with context strings, and enforces the "every claim must be quantified" standard.

---

## The Question

**"Do ALL claims, recommendations, and rationale strings use exact numbers instead of vague qualifiers?"**

---

## Input Schema

```yaml
input:
  layer2_output: from 2.6 (all candidate text fields)
  layer1_output: from 1.6 (all analysis text fields)
```

## Process Logic

### Step 1: Identify All Scannable Text Fields

```
FROM layer2_output, COLLECT all text strings from:
  - candidate.core_concept.one_paragraph_description
  - candidate.core_concept.strategy_rationale
  - candidate.root_cause_reframe.what_they_think
  - candidate.root_cause_reframe.what_it_really_is
  - candidate.root_cause_reframe.why_nothing_worked
  - candidate.mechanism.uniqueness_claim
  - candidate.emotional_driver_combination.intensity_rationale
  - candidate.proof_architecture (all string fields)

FROM layer1_output, COLLECT all text strings from:
  - pattern_analysis (all rationale/recommendation strings)
  - saturation_mapping (all recommendation strings)
  - gap_identification (all recommendation/strategic_alignment strings)
  - emotional_mapping (all intensity_rationale strings)
  - mechanism_mapping (all recommendation strings)

RECORD total_text_fields_scanned = count of all collected strings
```

### Step 2: Scan Each Field for Forbidden Terms

```
FOR each collected text string:
  FOR each of 17 forbidden terms:
    SEARCH for term as whole word (case-insensitive):
      Use word boundary matching: \b[term]\b
      EXCLUDE terms within proper nouns or quoted text

    IF found:
      RECORD violation:
        term: the forbidden term
        file: which output file
        candidate_id: which candidate (if applicable)
        field_path: which specific field
        context: 20 characters before + [TERM] + 20 characters after
        required_fix: "Replace '[term]' with [replacement type] — e.g., '[term] swipes' → '[X] swipes'"
```

### Step 3: Verify Quantified Claims

```
SCAN all text fields for numeric patterns:
  - Integer counts: \d+
  - Percentages: \d+%
  - Fraction format: \d+ of \d+
  - Score format: \d+\.\d+

COUNT specific_counts_verified = total numeric claims found

VERIFY each percentage shows numerator/denominator:
  Pattern: \d+% \(\d+ of \d+\)
  IF percentage without breakdown found:
    ADD advisory (not violation): "Percentage at [location] should show (X of Y)"
```

### Step 4: Compile Results

```
total_violations = SUM of all term occurrences

status = IF total_violations == 0 THEN "PASS" ELSE "FAIL"

IF FAIL:
  SORT violations by candidate_id, then field_path
  INCLUDE required_fix for each specific instance
```

---

## Constraints

- NEVER skip any text field in the scan (complete coverage required)
- NEVER mark as PASS if any violation exists (zero tolerance)
- NEVER report a false positive from proper nouns (e.g., "Rare Breed" as a brand name)
- ALWAYS use word-boundary matching (not substring matching)
- ALWAYS include context string showing surrounding text
- ALWAYS provide specific required_fix for each violation
- ALWAYS report ALL violations (not just first found)

---

## Quality Criteria

1. All 17 forbidden terms checked against all text fields
2. Word-boundary matching prevents false positives from substrings
3. Context strings show enough surrounding text to understand the violation
4. Required_fix provides specific actionable guidance per instance
5. Scan coverage confirms all 5 field categories were checked
6. Total violations count matches sum of all term occurrence counts
7. Status is zero-tolerance: 1+ violations = FAIL
