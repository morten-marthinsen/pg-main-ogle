# Skill 3.8: Anchor-to-Distance Ratio Calculator

**Version:** 1.0
**Layer:** 3 (Validation)
**Type:** Scoring Engine
**Dependencies:** schema_distance_scores.json (from 3.7), candidate_architectures.json (from 2.2), proof_architectures.json (from 2.5), layer0-output.json (from 0.6)
**Output:** anchor_distance_ratios.json

---

## Purpose

Calculate the Anchor-to-Distance Ratio (ADR) for each candidate — the relationship between schema distance and the anchoring elements required to prevent cognitive rejection. Higher schema distance claims require proportionally more anchoring. This skill quantifies that relationship and determines whether each candidate has sufficient anchoring to support its schema distance level, directly informing the Resolution Accessibility (RA) dimension of the RSF.

---

## Theoretical Foundation

### The Anchoring Hypothesis

When encountering a high-schema-distance claim, the human mind performs rapid credibility assessment:
1. **Source anchoring** — "Who is making this claim? Do I trust them?"
2. **Evidence anchoring** — "What proof supports this claim?"
3. **Linguistic anchoring** — "Does this sound like legitimate expertise?"
4. **Social anchoring** — "Do others I respect believe this?"
5. **Incremental anchoring** — "Were there stepping stones that led me here?"

**The Critical Insight:** Each unit of schema distance requires proportionally more anchoring. A claim in the "elevated" zone (SDS 5.7-7.0) requires 2-3x the anchoring of an "optimal" zone claim (SDS 3.6-5.6). A claim in the "too_high" zone (SDS 7.1+) may require institutional-level anchoring or schema distance reduction.

### The ADR Formula

```
ADR = Total Anchoring Score / Calibrated Schema Distance

Optimal ADR Range: 1.2 - 2.0

ADR < 1.2 = UNDER-ANCHORED (rejection risk)
ADR 1.2 - 2.0 = PROPERLY ANCHORED (optimal credibility)
ADR > 2.0 = OVER-ANCHORED (may feel defensive, reduces impact)
```

### Anchoring Deficit vs. Surplus

| ADR | Interpretation | Prospect Response | Action |
|-----|----------------|-------------------|--------|
| < 0.8 | Severely under-anchored | "This sounds like BS" | Reduce SDS or dramatically increase anchoring |
| 0.8 - 1.1 | Under-anchored | "Interesting, but I'm skeptical" | Increase anchoring |
| 1.2 - 2.0 | Optimal | "This is surprising but credible" | Proceed |
| 2.1 - 2.5 | Slightly over-anchored | "They're really trying to convince me" | Acceptable but suboptimal |
| > 2.5 | Severely over-anchored | "Methinks they doth protest too much" | Reduce anchoring or increase SDS |

## The Five Anchoring Dimensions

### Dimension 1: Authority Anchoring

**Definition:** The credibility weight of the spokesperson, expert, or source making the claim.

```yaml
authority_anchoring:
  measurement_basis: "Credibility weight of claim source"

  scoring_rubric:
    score_0_2:
      description: "No authority"
      criteria: "Anonymous source or no credentials cited"
      examples:
        - "Scientists say..."
        - "Research shows..."
        - "Experts agree..."

    score_3_4:
      description: "Generic authority"
      criteria: "Named person with basic credentials"
      examples:
        - "Dr. Smith, a nutritionist..."
        - "According to health expert John..."

    score_5_6:
      description: "Specific authority"
      criteria: "Named expert with relevant impressive credentials"
      examples:
        - "Dr. Steven Gundry, former head of Cardiothoracic Surgery at Loma Linda..."
        - "Dr. Kellyann Petrucci, who has treated over 10,000 patients..."

    score_7_8:
      description: "Elite authority"
      criteria: "Top-tier credentials with quantified accomplishments"
      examples:
        - "The doctor who performed 10,000+ heart surgeries at one of America's top hospitals..."
        - "The researcher behind 300+ peer-reviewed studies..."

    score_9_10:
      description: "Institutional authority"
      criteria: "Backed by major institution or multiple elite authorities"
      examples:
        - "Harvard Medical School research team..."
        - "Nobel Prize-winning scientist..."
        - "Published in the New England Journal of Medicine..."

  calculation_inputs:
    from_candidate_architecture:
      - spokesperson_credentials
      - institutional_affiliations
      - quantified_accomplishments
```

### Dimension 2: Evidence Anchoring

**Definition:** The quantity and quality of proof elements supporting the claim.

```yaml
evidence_anchoring:
  measurement_basis: "Proof inventory strength"

  scoring_rubric:
    score_0_2:
      description: "No evidence"
      criteria: "Claim made without support"
      proof_count: "0 elements"

    score_3_4:
      description: "Minimal evidence"
      criteria: "1-2 proof elements, generic quality"
      proof_count: "1-2 elements"
      examples:
        - "Studies show..."
        - "Research indicates..."

    score_5_6:
      description: "Adequate evidence"
      criteria: "3-4 proof elements with specificity"
      proof_count: "3-4 elements"
      examples:
        - "In a 12-week study of 200 participants..."
        - "According to research published in..."

    score_7_8:
      description: "Strong evidence"
      criteria: "5-7 proof elements with institutional backing"
      proof_count: "5-7 elements"
      examples:
        - "Published in 3 peer-reviewed journals..."
        - "Replicated across 5 independent studies..."

    score_9_10:
      description: "Overwhelming evidence"
      criteria: "8+ proof elements with elite institutional backing"
      proof_count: "8+ elements"
      examples:
        - "Backed by Mayo Clinic research, Harvard studies, and NIH funding..."
        - "147 clinical trials across 12 countries..."

  calculation_inputs:
    from_proof_architectures:
      - proof_element_count
      - proof_quality_scores
      - institutional_sources
```

### Dimension 3: Linguistic Anchoring

**Definition:** Use of familiar language and concepts that ground novel claims.

```yaml
linguistic_anchoring:
  measurement_basis: "Familiar vocabulary surrounding novel claims"

  scoring_rubric:
    score_0_2:
      description: "All novel language"
      criteria: "Claim uses entirely unfamiliar terminology"
      pattern: "[Novel term] causes [novel term] through [novel mechanism]"

    score_3_4:
      description: "Mostly novel"
      criteria: "Some familiar terms but mostly new vocabulary"
      pattern: "[Novel term] affects your [familiar concept]"

    score_5_6:
      description: "Balanced"
      criteria: "Novel claims wrapped in familiar language"
      pattern: "Your [familiar body part] has a [novel mechanism] that controls [familiar outcome]"

    score_7_8:
      description: "Mostly familiar"
      criteria: "Familiar vocabulary frames novel claims"
      pattern: "[Familiar concept] is actually controlled by [novel but understandable mechanism]"

    score_9_10:
      description: "Maximally accessible"
      criteria: "Novel claims expressed in everyday language"
      pattern: "Think of it like [common analogy]—your [body part] works the same way"

  evaluation_criteria:
    vocabulary_familiarity:
      question: "What percentage of terms would a layperson recognize?"
      ideal: "70%+ familiar terms"

    analogy_presence:
      question: "Are novel concepts grounded with familiar analogies?"
      ideal: "At least one grounding analogy for each novel concept"

    jargon_density:
      question: "How much unexplained technical language?"
      ideal: "No unexplained jargon"
```

### Dimension 4: Social Anchoring

**Definition:** Evidence that others have accepted and benefited from the claim.

```yaml
social_anchoring:
  measurement_basis: "Social proof quantity and quality"

  scoring_rubric:
    score_0_2:
      description: "No social proof"
      criteria: "No testimonials or social validation"

    score_3_4:
      description: "Minimal social proof"
      criteria: "1-2 generic testimonials"
      examples:
        - "It worked for me!"
        - "I love this product."

    score_5_6:
      description: "Adequate social proof"
      criteria: "3-5 specific testimonials with details"
      examples:
        - "After 6 weeks, I lost 23 pounds..."
        - "My doctor was shocked when my blood work came back..."

    score_7_8:
      description: "Strong social proof"
      criteria: "Multiple detailed testimonials + scale indicators"
      examples:
        - "Over 47,000 people have tried this..."
        - "4.8 stars from 2,300 verified reviews..."

    score_9_10:
      description: "Overwhelming social proof"
      criteria: "Mass adoption + media validation + viral indicators"
      examples:
        - "Featured on Dr. Oz, Good Morning America, and the New York Times..."
        - "Over 2 million bottles sold..."

  calculation_inputs:
    from_proof_architectures:
      - testimonial_count
      - testimonial_specificity_scores
      - scale_indicators
      - media_mentions
```

### Dimension 5: Incremental Anchoring

**Definition:** The presence of stepping-stone claims that lead the reader to the main claim.

```yaml
incremental_anchoring:
  measurement_basis: "Logical stepping stones to high-distance claim"

  scoring_rubric:
    score_0_2:
      description: "No incremental path"
      criteria: "Main claim presented without buildup"
      pattern: "Here's the truth: [High-SDS claim]"

    score_3_4:
      description: "Minimal incremental path"
      criteria: "1 stepping stone before main claim"
      pattern: "[Setup claim] → [High-SDS claim]"

    score_5_6:
      description: "Adequate incremental path"
      criteria: "2-3 stepping stones building to main claim"
      pattern: "[Familiar truth] → [Mild surprise] → [High-SDS claim]"

    score_7_8:
      description: "Strong incremental path"
      criteria: "4-5 stepping stones with clear logic"
      pattern: "[Familiar] → [Slightly surprising] → [Moderately surprising] → [High-SDS claim]"

    score_9_10:
      description: "Masterful incremental path"
      criteria: "Complete logical journey where high-SDS claim feels inevitable"
      pattern: "By the time you reach the main claim, it feels like the only possible conclusion"

  evaluation_criteria:
    step_count:
      question: "How many logical steps lead to the main claim?"
      ideal: "3-5 steps for elevated-zone claims"

    step_acceptance:
      question: "Would reader accept each step before proceeding?"
      ideal: "Each step should be easy to accept given previous steps"

    logical_necessity:
      question: "Does the conclusion feel inevitable?"
      ideal: "Reader should feel 'of course!' not 'wait, what?'"
```

## ADR Calculation and Interpretation

```yaml
adr_calculation:
  formula: "ADR = Total_Anchoring_Score / Calibrated_SDS"

  example_calculations:
    example_1:
      anchoring_scores:
        authority: 7
        evidence: 6
        linguistic: 5
        social: 6
        incremental: 5
      total_anchoring: 29
      calibrated_sds: 5.2
      adr: "29 / 5.2 = 5.58"
      interpretation: "OVER-ANCHORED — may feel defensive"

    example_2:
      anchoring_scores:
        authority: 5
        evidence: 4
        linguistic: 4
        social: 3
        incremental: 3
      total_anchoring: 19
      calibrated_sds: 6.8
      adr: "19 / 6.8 = 2.79"
      interpretation: "UNDER-ANCHORED — rejection risk for this SDS level"

    example_3:
      anchoring_scores:
        authority: 6
        evidence: 5
        linguistic: 5
        social: 5
        incremental: 4
      total_anchoring: 25
      calibrated_sds: 5.0
      adr: "25 / 5.0 = 5.0"
      interpretation: "OPTIMAL — surprising but credible"

  scaling_note: |
    The raw ADR values in examples above use Total/SDS directly.
    To get ADR into the 0.8-2.5 interpretive range, normalize:
    Normalized_ADR = (Total_Anchoring / 10) / Calibrated_SDS

    This gives:
    - Example 1: (29/10) / 5.2 = 0.56 → Under-anchored
    - Example 2: (19/10) / 6.8 = 0.28 → Severely under-anchored
    - Example 3: (25/10) / 5.0 = 0.50 → Under-anchored

    For proper interpretation, use: Normalized_ADR = Total_Anchoring / (Calibrated_SDS × 10)
```

### Normalized ADR Formula

```
Normalized_ADR = Total_Anchoring_Score / (Calibrated_SDS × 10)

Interpretation:
- < 0.8: Severely under-anchored
- 0.8 - 1.1: Under-anchored
- 1.2 - 2.0: Optimal
- 2.1 - 2.5: Slightly over-anchored
- > 2.5: Severely over-anchored
```

## Process Logic

### Step 1: Score Authority Anchoring

```
FOR each candidate:

  LOAD spokesperson from candidate_architecture
  LOAD credentials from candidate_architecture

  EVALUATE authority_level:
    IF no credentials: score = 0-2
    IF generic credentials ("Dr.", "expert"): score = 3-4
    IF specific impressive credentials: score = 5-6
    IF elite credentials with quantified accomplishments: score = 7-8
    IF institutional backing: score = 9-10

  RECORD score with rationale
```

### Step 2: Score Evidence Anchoring

```
FOR each candidate:

  LOAD proof_elements from proof_architectures
  COUNT proof_element_count
  IDENTIFY institutional_sources

  EVALUATE evidence_level:
    IF 0 elements: score = 0-2
    IF 1-2 generic elements: score = 3-4
    IF 3-4 specific elements: score = 5-6
    IF 5-7 elements with institutions: score = 7-8
    IF 8+ elements with elite institutions: score = 9-10

  RECORD score with rationale
```

### Step 3: Score Linguistic Anchoring

```
FOR each candidate:

  ANALYZE candidate claims for:
    - Familiar vs. novel vocabulary
    - Presence of grounding analogies
    - Unexplained jargon

  CALCULATE familiar_term_percentage
  COUNT analogies_present
  CHECK jargon_explained

  EVALUATE linguistic_level:
    IF all novel language: score = 0-2
    IF mostly novel: score = 3-4
    IF balanced: score = 5-6
    IF mostly familiar: score = 7-8
    IF maximally accessible: score = 9-10

  RECORD score with rationale
```

### Step 4: Score Social Anchoring

```
FOR each candidate:

  LOAD testimonials from deep_research and proof_architectures
  COUNT testimonial_count
  IDENTIFY scale_indicators
  IDENTIFY media_mentions

  EVALUATE social_level:
    IF no testimonials: score = 0-2
    IF 1-2 generic: score = 3-4
    IF 3-5 specific: score = 5-6
    IF detailed + scale: score = 7-8
    IF mass adoption + media: score = 9-10

  RECORD score with rationale
```

### Step 5: Score Incremental Anchoring

```
FOR each candidate:

  ANALYZE candidate architecture for:
    - Logical stepping stones to main claim
    - Claim progression structure
    - Inevitability of conclusion

  COUNT stepping_stone_count
  MAP logical_path
  RATE inevitability (0-10)

  EVALUATE incremental_level:
    IF no buildup: score = 0-2
    IF 1 step: score = 3-4
    IF 2-3 steps: score = 5-6
    IF 4-5 steps with logic: score = 7-8
    IF masterful journey: score = 9-10

  RECORD score with rationale
```

### Step 6: Calculate ADR and Interpret

```
FOR each candidate:

  CALCULATE total_anchoring_score = SUM(all 5 dimension scores)

  LOAD calibrated_sds from schema_distance_scores
  LOAD sds_zone from schema_distance_scores

  CALCULATE normalized_adr = total_anchoring_score / (calibrated_sds × 10)

  DETERMINE adr_interpretation:
    IF normalized_adr < 0.8: "severely_under"
    IF normalized_adr >= 0.8 AND < 1.2: "under"
    IF normalized_adr >= 1.2 AND <= 2.0: "optimal"
    IF normalized_adr > 2.0 AND <= 2.5: "over"
    IF normalized_adr > 2.5: "severely_over"

  CHECK zone_requirements:
    LOAD requirements for sds_zone
    VERIFY each required dimension meets minimum
    FLAG any gaps

  CALCULATE resolution_accessibility_score:
    IF adr_interpretation == "optimal": RA = 7-9
    IF adr_interpretation == "under": RA = 4-6
    IF adr_interpretation == "severely_under": RA = 1-3
    IF adr_interpretation == "over": RA = 6-8
    IF adr_interpretation == "severely_over": RA = 5-7

  GENERATE recommendations based on interpretation
```
