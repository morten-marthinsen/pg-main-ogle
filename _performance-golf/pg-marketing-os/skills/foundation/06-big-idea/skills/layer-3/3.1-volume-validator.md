# Skill 3.1: Volume Validator

**Version:** 1.0
**Layer:** 3 (Validation)
**Type:** Validation Gate
**Dependencies:** layer2-output.json (from 2.6), layer0-output.json (from 0.6)
**Output:** volume_validation.json

---

## Purpose

Verify all candidates are backed by sufficient swipe evidence — total analysis volume, per-candidate Big Idea examples, mechanism sources, headline formula proof, lead technique evidence, and proof architecture citations. Binary PASS/FAIL per check. No recommendations without evidence.

---

## The Question

**"Does every element of every candidate trace back to sufficient swipe evidence (not just 1-2 examples)?"**

---

## Input Schema

```yaml
input:
  layer2_output: from 2.6 (5 candidates with headlines, leads, proofs)
  from_layer0_output:
    validation_status.swipe_count: integer
```

## Process Logic

### Check 1: Total Swipe Analysis Volume

```
READ layer0_output.validation_status.swipe_count

IF swipe_count >= 100: PASS
ELSE: FAIL
  LOG: "Total swipes analyzed: [count] (threshold: 100). Insufficient volume."
```

### Check 2: Big Idea Type Examples

```
FOR each candidate in layer2_output:
  READ swipe_evidence_count from candidate

  IF swipe_evidence_count >= 3: PASS
  ELSE: FAIL
    LOG: "Candidate [id] Big Idea type [type] has [count] examples (needs 3+)"
```

### Check 3: Mechanism Pattern Examples

```
FOR each candidate:
  READ mechanism.swipe_sources array
  COUNT = swipe_sources.length

  IF COUNT >= 5: PASS
  IF COUNT >= 3 AND < 5: WARN (pass with advisory)
  IF COUNT < 3: FAIL
    LOG: "Candidate [id] mechanism [name] has only [count] sources (needs 3+, prefers 5+)"
```

### Check 4: Headline Formula Examples

```
FOR each candidate:
  FOR each headline:
    READ formula_source_count

    IF formula_source_count >= 3: PASS
    ELSE: FAIL
      LOG: "Candidate [id] headline [index] formula [type] has only [count] examples (needs 3+)"
```

### Check 5: Lead Technique Examples

```
FOR each candidate:
  FOR each lead_variation:
    READ swipe_source
    READ swipe_quality_score

    IF swipe_source EXISTS AND swipe_quality_score >= 80: PASS
    IF swipe_source EXISTS AND swipe_quality_score < 80: FAIL
      LOG: "Candidate [id] lead [index] cites [source] at [score] quality (needs 80+)"
    IF swipe_source IS NULL: FAIL
      LOG: "Candidate [id] lead [index] has no swipe source citation"
```

### Check 6: Proof Architecture Examples

```
FOR each candidate:
  READ proof_architecture.swipe_sources array
  COUNT = swipe_sources.length

  IF COUNT >= 3: PASS
  ELSE: FAIL
    LOG: "Candidate [id] proof architecture has only [count] sources (needs 3+)"
```

### Final Determination

```
status = IF (all checks PASS — no FAIL in any category)
         THEN "PASS"
         ELSE "FAIL"

IF FAIL:
  POPULATE issues array with all failure details
  EACH issue includes: candidate_id, element_type, current_count, required_count, required_fix
```

---

## Constraints

- NEVER pass a check where count is below threshold (no exceptions)
- NEVER estimate counts — read actual array lengths
- NEVER skip any candidate or element (check ALL)
- ALWAYS report exact counts (passed: X, failed: Y, total: Z)
- ALWAYS cite specific candidate_id for every failure
- ALWAYS include the specific swipe_source or formula_source_count in results

---

## Quality Criteria

1. All counts are exact integers from actual array lengths
2. Every candidate checked for every check type (no skips)
3. Every failure includes candidate_id, element type, actual count, and threshold
4. Status is binary PASS/FAIL (no partial passes)
5. Issues array is populated for ALL failures (not just first failure found)
6. Warm advisory for mechanism sources 3-4 (pass but flagged)
