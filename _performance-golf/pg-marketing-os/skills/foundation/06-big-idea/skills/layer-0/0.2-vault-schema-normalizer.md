## Normalization Logic

### Step 1: Scan All Vault Files

```
READ all .json files from vault_path
total_files_scanned = file_count
```

### Step 2: Detect Schema Variant Per File

```
FOR each JSON file:

  IF file has "configuration.elements" object with 7 dimension keys:
    → Schema B (Canonical) — no conversion needed
    schema_b_canonical += 1

  ELSE IF file has "extraction.elements" object:
    → Schema A (Legacy) — apply Schema A mapping
    schema_a_converted += 1

  ELSE IF file has "copy_components" object:
    → Schema C (Transitional) — apply Schema C mapping
    schema_c_converted += 1

  ELSE:
    → Unknown schema — skip file
    unprocessable_skipped += 1
    skipped_files.push(filename)
    CONTINUE to next file
```

### Step 3: Apply Schema Mappings

**Schema A Mapping (Legacy → Canonical):**
```
extraction.elements.[DIM]         → configuration.elements.[DIM]
extraction.drivers.[DIM]          → configuration.drivers.[DIM]
extraction.structures.[DIM]       → configuration.structures.[DIM]
extraction.lead_element           → configuration.lead_element
extraction.lead_driver            → configuration.lead_driver
extraction.lead_structure         → configuration.lead_structure
copy_analysis.headline            → components.headline.main_headline
copy_analysis.headline_type       → components.headline.pattern_type
copy_analysis.lead_text           → components.lead.full_text
copy_analysis.lead_type           → components.lead.type
copy_analysis.mechanism_name      → components.mechanism.name
copy_analysis.proof_elements      → components.proof.types_used
source_info.collection            → metadata.source_collection
source_info.quality_score         → metadata.quality_score
source_info.niche                 → metadata.niche
source_info.format                → metadata.format
source_info.big_idea_type         → metadata.big_idea_type
```

**Schema C Mapping (Transitional → Canonical):**
```
copy_components.headline.text     → components.headline.main_headline
copy_components.headline.type     → components.headline.pattern_type
copy_components.lead.text         → components.lead.full_text
copy_components.lead.type         → components.lead.type
copy_components.mechanism.name    → components.mechanism.name
copy_components.proof.types       → components.proof.types_used
scoring.elements.[DIM]            → configuration.elements.[DIM]
scoring.drivers.[DIM]             → configuration.drivers.[DIM]
scoring.structures.[DIM]          → configuration.structures.[DIM]
meta.quality_score                → metadata.quality_score
meta.niche                        → metadata.niche
meta.big_idea_type                → metadata.big_idea_type
```

### Step 4: Filter by Niche and Quality

```
FROM all successfully normalized files:

  FILTER: metadata.niche matches target_niche
    niche_matched = count after filter

  FILTER: metadata.quality_score >= quality_threshold
    quality_passed = count after filter

  SORT by metadata.quality_score descending

  TAKE top max_results entries
    final_count = min(quality_passed, max_results)

  CALCULATE:
    quality_average = mean(quality_scores)
    top_quality_score = max(quality_scores)
    lowest_quality_score = min(quality_scores)
```

### Step 5: Validate Normalization Integrity

```
VERIFY: schema_b_canonical + schema_a_converted + schema_c_converted + unprocessable_skipped == total_files_scanned

IF mismatch:
  FLAG: "Normalization count mismatch — files lost in processing"

IF unprocessable_skipped > (total_files_scanned * 0.20):
  WARN: "20%+ files unprocessable — vault quality concern"
```

### Step 6: Specimen Tiering

```
FOR each normalized specimen in filtered data:

  CALCULATE big_idea_strength_score:
    +2 IF big_idea_type classified (revelation, conspiracy, transformation, prophecy, etc.)
    +2 IF central_concept articulated AND non-generic
    +1 IF main_headline present AND contains specificity element
    +1 IF subheadline present AND extends headline
    +1 IF power_words present with 2+ entries
    +1 IF specificity_elements present with 2+ entries
    MAX_SCORE: 8

  SORT specimens by big_idea_strength_score DESC
  ASSIGN tier:
    Gold = top 10% (big_idea_strength_score >= 7)
    Silver = next 15% (big_idea_strength_score >= 5)
    Standard = remainder

  TAG each specimen with:
    tier: enum[gold, silver, standard]
    big_idea_strength_score: integer (0-8)

  OUTPUT tier_distribution:
    total_specimens: integer
    gold_count: integer
    silver_count: integer
    standard_count: integer
```

### Step 7: Type-Matched Retrieval

```
DEFINE match_fields:
  primary: big_idea_type (revelation, conspiracy, transformation, prophecy, discovery, warning)
  secondary: niche (e.g., health, finance, survival, self-improvement)
  tertiary: format (magalog, VSL, sales_page, letter)

WHEN campaign_context is available:
  STRATEGY 1 — exact_match:
    FILTER specimens WHERE big_idea_type == target AND niche == target_niche
    PRIORITIZE Gold tier first, then Silver

  STRATEGY 2 — type_match:
    FILTER specimens WHERE big_idea_type == target (any niche)
    PRIORITIZE Gold tier, same-niche Silver, then cross-niche Silver

  STRATEGY 3 — cross_type:
    RETURN all Gold specimens across big idea types
    INCLUDE Silver specimens from target niche

WHEN campaign_context is NOT available:
  RETURN full specimen set with tier metadata attached
  SORT by big_idea_strength_score DESC

OUTPUT retrieval_result:
  matched_specimens: [specimen]
  match_strategy_used: enum[exact_match, type_match, cross_type, full_set]
  gold_specimens_available: integer
  silver_specimens_available: integer
```

## Quality Criteria

1. Every vault file classified into exactly one schema variant
2. All Schema A/C files correctly mapped to canonical field paths
3. Normalization count adds up (canonical + converted + skipped = total scanned)
4. Niche filter applied using exact match from market_config
5. Quality filter applied at threshold (default 70)
6. No fabricated quality scores or counts
7. Skipped files logged with filenames
8. Tiering applied: Gold specimens identified (top 10%)
9. Tiering applied: Silver specimens identified (top 25%)
10. Type-matched retrieval functional with campaign context
11. Fallback to full set works without campaign context
12. Tier distribution logged
