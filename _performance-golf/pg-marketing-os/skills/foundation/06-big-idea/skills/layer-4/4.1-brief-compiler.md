**Version:** 1.0
**Layer:** 4 (Package)
**Type:** Assembly
**Dependencies:** layer3-output.json (from 3.6), layer1-output.json (from 1.6), BIG-IDEA-BRIEF-TEMPLATE.md

---

## Purpose

Assemble the final BIG-IDEA-BRIEF.md from all validated outputs — market intelligence summary, 5 complete candidate profiles with full creative execution, validation report, evidence base summary, and recommended testing sequence. This is the human-readable deliverable. A copywriter receiving this document should be able to start writing immediately.

---

## The Question

**"Is the final brief assembled with all sections populated, all data pulled from validated sources, and all placeholder brackets replaced with actual values?"**

## Assembly Logic

### Step 1: Populate Header

```
FROM layer3_output.compiled_output.generation_metadata:
  project = [project-name from session context]
  market = [market category from deep research]
  schwartz_stage = layer1_output.mechanism_mapping.schwartz_stage
  generated = generation_metadata.validation_date
  swipes_analyzed = generation_metadata.swipes_analyzed
  validation_status = "PASSED (5/5 gates)"
```

### Step 2: Build Executive Summary

```
FROM generation_metadata:
  candidates_count = candidates_generated
  swipes_analyzed = swipes_analyzed
  total_headlines = total_headlines
  total_leads = total_leads

FROM layer1_output:
  competitor_mechanisms = COUNT of saturation_mapping.all_mechanisms
  customer_quotes = SUM of emotional_mapping.primary_drivers[].pain_quote_count +
                         emotional_mapping.primary_drivers[].hope_quote_count

  strategic_context:
    market_sophistication = "Stage [X] - [emphasis level] Root Cause emphasis"
    competitive_density = DERIVE from saturation_mapping zone distribution
    primary_whitespace = gap_identification.whitespace_opportunities[0].gap_description
    dominant_emotional_driver = emotional_mapping.winning_combinations[0].primary_driver
```

### Step 3: Compile Market Intelligence Summary

```
FROM layer1_output.pattern_analysis:
  POPULATE dominant_proven_patterns (top 3 by usage)
    FOR each: type, swipe_count, avg_quality, saturation, recommendation, evidence IDs

  POPULATE rare_high_performers (top 3 by quality)
    FOR each: type, swipe_count, quality_score, competitor_count, opportunity_size

FROM layer1_output.saturation_mapping:
  BUILD avoid_table (Red Zone mechanisms)
  BUILD safe_table (Green Zone mechanisms)
  BUILD whitespace_table (Blue Zone mechanisms)
  FORMAT as markdown tables with: Name, Competitor Count, Swipe Usage, Quality, Recommendation

FROM layer1_output.gap_identification:
  LIST whitespace_opportunities in priority order
  FOR each: description, swipe_count, quality, competitors, strategic_alignment, evidence

FROM layer1_output.emotional_mapping:
  LIST primary_drivers (top 3 by intensity)
    FOR each: name, pain_count, hope_count, swipe_score, intensity_level, rationale
  BUILD winning_combinations_table
    FOR each: primary + secondary, swipe_usage, combined_score, quote_support, recommendation
```

### Step 4: Assemble Candidate Profiles (×5)

```
FOR each candidate in layer3_output.compiled_output.big_idea_candidates:
  POPULATE complete candidate section:

  HEADER:
    candidate_number = position (01-05)
    name = candidate.mechanism.mechanism_name (or core_concept identifier)
    type = candidate.core_concept.big_idea_type
    strategy = candidate.core_concept.strategy_type
    swipe_evidence = "[X] examples at [Y] avg quality score"

  CORE CONCEPT:
    one_paragraph = candidate.core_concept.one_paragraph_description
    strategy_rationale = candidate.core_concept.strategy_rationale

  ROOT CAUSE REFRAME:
    what_they_think = candidate.root_cause_reframe.what_they_think
    what_it_really_is = candidate.root_cause_reframe.what_it_really_is
    why_nothing_worked = candidate.root_cause_reframe.why_nothing_worked
    technique = candidate.root_cause_reframe.technique_name
    emphasis_level = candidate.root_cause_reframe.emphasis_level
    evidence = candidate.root_cause_reframe.swipe_sources

  MECHANISM:
    name = candidate.mechanism.mechanism_name
    naming_pattern = candidate.mechanism.naming_pattern_used
    uniqueness_claim = candidate.mechanism.uniqueness_claim
    steps = candidate.mechanism.steps_or_components
    evidence = candidate.mechanism.swipe_sources

  PROMISE:
    core_promise = candidate.promise.statement
    specificity = candidate.promise.specificity_markers
    evidence = candidate.promise.swipe_sources

  ENEMY:
    name = candidate.enemy.enemy_name
    type = candidate.enemy.enemy_type
    role = candidate.enemy.how_caused_problem
    evidence = candidate.enemy.swipe_sources

  EMOTIONAL STRATEGY:
    primary = candidate.emotional_driver_combination.primary_driver
    secondary = candidate.emotional_driver_combination.secondary_driver
    combined_score = candidate.emotional_driver_combination.combined_score
    intensity_rationale = candidate.emotional_driver_combination.intensity_rationale

  HEADLINES:
    GROUP by emotional category (fear, hope, curiosity, authority)
    FOR each headline:
      text, formula_type, emotional_driver, swipe_source, formula_usage_count

  LEAD VARIATIONS:
    FOR each lead (3+):
      technique, narrative_structure, swipe_source
      opening_hook (actual copy)
      root_cause_intro (actual copy)
      mechanism_tease (actual copy)

  PROOF ARCHITECTURE:
    primary_type, secondary_type, swipe_sources
    scientific_elements (if applicable)
    transformation_elements
    demonstration_plan
    authority_elements
    social_proof_elements
    proof_priority_order (numbered list with rationale)
```

### Step 5: Build Validation Report

```
FROM layer3_output.gate_results + summary_stats:

  Gate 1 (Volume):
    swipes_analyzed, per-candidate example counts, mechanism sources,
    headline formula sources, lead quality, proof sources

  Gate 2 (Quantification):
    vague_qualifiers_found (0), specific_counts_verified, forbidden_terms (none)

  Gate 3 (Defensibility):
    sources validated (X/X), evidence trails (5/5), recommendations supported,
    quote counts accurate

  Gate 4 (Actionability):
    headlines ready (X/X), leads with copy (X/X), mechanisms with steps (5/5),
    proof plans specified (5/5)

  Gate 5 (Anti-Slop):
    ai_tell_patterns (0 critical), generic_naming (0), weak_copy (advisory count),
    repetition (0 duplicates)

  Overall: "PASS (5/5 gates)"
```

### Step 6: Generate Testing Sequence

```
DERIVE from candidate strategy types:
  Phase 1: Candidates with strategy == "whitespace_opportunity" (test first)
    ORDER by: opportunity_size (from gap_identification)
    RATIONALE: "Maximum differentiation potential"

  Phase 2: Candidates with strategy == "dominant_proven_pattern" (test if Phase 1 needs refinement)
    ORDER by: swipe_quality_avg
    RATIONALE: "Safer alternatives with proven track record"

  Phase 3: Candidate with strategy == "hybrid" (test as combination)
    RATIONALE: "Combines best-performing elements from Phases 1-2"

  Headline testing guidance:
    "Start with highest emotional intensity headlines (fear-based typically)
     then test hope-based variations"
```

### Step 7: Build Evidence Base Summary

```
COMPILE:
  premium_swipe_vault: swipes_analyzed, avg_quality, scoring_framework
  deep_research: quotes_analyzed, competitors_inventoried, WEB_analysis, schwartz_stage
  quality_standards: list all met standards with checkmarks
  traceability_statement: standard defensibility paragraph
```

### Step 8: Build Appendix (Swipe Source Index)

```
COLLECT all unique swipe_source IDs from all candidates
DEDUPLICATE
FOR each unique ID:
  LOOKUP quality_score from vault data
  LOOKUP market category
  LOOKUP big_idea_type
  LIST which candidates/elements reference this ID

FORMAT as markdown table:
  | Swipe ID | Quality Score | Market | Big Idea Type | Used For |
```

## Format Requirements

- Output as well-structured Markdown (.md)
- Use ## for major sections, ### for subsections, #### for element groups
- Tables for structured comparisons (saturation mapping, emotional combinations)
- Bold for section labels within candidate profiles
- Horizontal rules (---) between candidates and major sections
- All numeric values populated (no [X] placeholders remain)
- All swipe IDs included (no [IDs] placeholders remain)
- Candidate numbering: 01-05 (zero-padded)

---

## Constraints

- NEVER leave template placeholders in final output (zero bracket patterns: /\[.*?\]/)
- NEVER omit any candidate section (all 5 candidates must be complete)
- NEVER abbreviate lead copy (full opening_hook, root_cause_intro, mechanism_tease text)
- NEVER summarize headlines (include all 10+ per candidate with full metadata)
- NEVER skip the validation report (all 5 gates must be reported)
- ALWAYS pull data from validated layer3_output (not raw layer2 output)
- ALWAYS include swipe source IDs for every evidence claim
- ALWAYS format consistently across all 5 candidates (same structure)
- ALWAYS include the appendix swipe index (even if large)

---

## Quality Criteria

1. Zero template placeholders remain (scan for /\[.*?\]/ patterns)
2. All 5 candidates include every section (core concept through proof architecture)
3. All headlines include: text, formula, driver, swipe_source, usage_count
4. All leads include: full copy (not descriptions), technique, source
5. Market intelligence summary matches layer1_output values exactly
6. Validation report matches layer3_output gate_results exactly
7. Testing sequence correctly maps strategy types to phases
8. Evidence base totals match actual data counts
9. Appendix includes every unique swipe ID referenced anywhere in the brief
