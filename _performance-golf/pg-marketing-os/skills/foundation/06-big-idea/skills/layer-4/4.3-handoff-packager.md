# Skill 4.3: Handoff Packager

**Version:** 1.0
**Layer:** 4 (Package)
**Type:** Terminal Output
**Dependencies:** BIG-IDEA-BRIEF.md (from 4.1), downstream_map.json (from 4.2), layer3-output.json (from 3.6)
**Output:** big_idea_package.json

---

## Purpose

Package the complete self-contained output for downstream skill consumption. This is the terminal artifact — every downstream skill in the CopywritingEngine consumes from this single package. Verifies no external references remain, no unresolved placeholders exist, and generates the extraction instructions each downstream skill needs. After this skill completes, the Big Idea pipeline is done.

---

## The Question

**"Can every downstream skill execute using ONLY this package — without accessing any upstream files, vault data, or session context?"**

---

input:

## Assembly Logic

### Step 1: Build Package Metadata

```
package_metadata:
  version = "1.0"
  generated = current ISO timestamp
  project = [from session context]
  market = [from session context]
  schwartz_stage = [from layer1 data]
  pipeline_status = "COMPLETE"
  validation_status = "5/5 PASS"
  self_containment_verified = [set after Step 6]
```

### Step 2: Copy Candidate Data

```
FOR each candidate in layer3_output.compiled_output.big_idea_candidates:
  DEEP COPY all fields into package.candidates[]
  ENSURE all nested objects are fully expanded (no references)
  ENSURE all string fields contain actual content (not pointers)
  VERIFY:
    - headlines[].headline_text is actual copy (not formula name)
    - lead_variations[].opening_hook is actual copy (not description)
    - mechanism.steps_or_components[] are actual steps (not placeholders)
    - proof_architecture contains specific elements (not vague categories)
```

### Step 3: Condense Market Intelligence

```
FROM layer3_output (which contains layer1 intelligence):
  EXTRACT and condense:
    dominant_patterns: top 5 by usage (type, count, quality)
    saturation_zones: mechanism names per zone (avoid, safe, whitespace)
    whitespace_opportunities: top 5 by opportunity tier
    emotional_landscape: primary_drivers + winning_combinations
    mechanism_inventory: root_cause_techniques + naming_patterns

  PURPOSE: Provide context without requiring full layer1_output access
  CONSTRAINT: Include enough data for any downstream skill to understand
              the strategic context of each candidate's choices
```

### Step 4: Transform Downstream Instructions

```
FROM downstream_map.json:
  FOR each skill_map:
    TRANSFORM into extraction instructions:
      skill_id = skill_map.skill_id
      skill_name = skill_map.skill_name
      what_to_extract = CONVERT field paths to package.candidates[] paths
        (e.g., "candidate.mechanism.mechanism_name" →
               "package.candidates[N].mechanism.mechanism_name")
      how_to_use = skill_map.consumption_notes
      candidate_scope = "all" (unless user has selected specific candidate)

  VERIFY all field paths resolve to actual data in this package
```

### Step 5: Build Evidence Index

```
COLLECT all swipe_source IDs from:
  - Each candidate's root_cause_reframe.swipe_sources
  - Each candidate's mechanism.swipe_sources
  - Each candidate's promise.swipe_sources
  - Each candidate's enemy.swipe_sources
  - Each headline's swipe_source
  - Each lead's swipe_source
  - Each candidate's proof_architecture.swipe_sources

DEDUPLICATE to unique set
COUNT total_unique_sources

FOR each unique source:
  RECORD: swipe_id, quality_score, market
  COLLECT: which candidate_ids reference this source
  STORE in sources[]
```

### Step 6: Run Integrity Checks

```
CHECK 1: Placeholder Scan
  SCAN all string fields in package for:
    /\[.*?\]/           (bracket placeholders)
    /\{.*?\}/           (brace placeholders)
    /TODO/i             (todo markers)
    /TBD/i              (to-be-determined markers)
    /PLACEHOLDER/i      (explicit placeholder text)
  placeholder_scan = (matches == 0)

CHECK 2: External Reference Scan
  SCAN all fields for:
    "see layer[0-3]"    (references to upstream layers)
    "from 1.6"          (references to upstream skills)
    "layer[0-3]-output" (references to upstream files)
    File paths          (references to local files)
    "vault/"            (references to vault locations)
  external_reference_scan = (matches == 0)

CHECK 3: Completeness Scan
  FOR each candidate:
    VERIFY: core_concept.one_paragraph_description.length > 0
    VERIFY: root_cause_reframe (all 3 fields non-empty)
    VERIFY: mechanism.mechanism_name.length > 0
    VERIFY: mechanism.steps_or_components.length >= 3
    VERIFY: promise.statement.length > 0
    VERIFY: enemy.enemy_name.length > 0
    VERIFY: headlines.length >= 10
    VERIFY: lead_variations.length >= 3
    VERIFY: lead_variations[].opening_hook.length > 0 (all)
    VERIFY: lead_variations[].root_cause_intro.length >= 200 (all)
    VERIFY: proof_architecture.primary_type.length > 0
  completeness_scan = (all verifications pass)

CHECK 4: Consistency Scan
  VERIFY: generation_summary.candidates_generated == candidates.length
  VERIFY: generation_summary.total_headlines == SUM(candidates[].headlines.length)
  VERIFY: generation_summary.total_leads == SUM(candidates[].lead_variations.length)
  VERIFY: evidence_index.total_unique_sources matches actual deduplicated count
  VERIFY: testing_sequence candidate IDs exist in candidates[]
  VERIFY: downstream_instructions field paths resolve to real data
  consistency_scan = (all verifications pass)

self_containment_verified = (placeholder_scan AND external_reference_scan AND
                             completeness_scan AND consistency_scan)

IF any check fails:
  issues_found = [list of specific failures]
  STOP: Package is not self-contained. Fix issues before outputting.
```

## Self-Containment Test

The definitive test for this package:

```
FOR each downstream skill in downstream_instructions:
  SIMULATE extraction:
    1. Read what_to_extract field paths
    2. Resolve each path against this package's data
    3. VERIFY: all paths resolve to non-empty values
    4. VERIFY: no path requires external file access

  IF all paths resolve:
    PASS — this skill can execute from package alone
  ELSE:
    FAIL — list unresolvable paths as issues
```

## Constraints

- NEVER output a package where self_containment_verified == false
- NEVER include file path references to upstream outputs
- NEVER include partial candidate data (all 5 must be complete)
- NEVER use references/pointers where actual data should be
- NEVER truncate lead copy, headlines, or proof elements
- ALWAYS deep-copy all nested objects (no lazy references)
- ALWAYS run all 4 integrity checks before marking complete
- ALWAYS include downstream_instructions for all 7 downstream skills
- ALWAYS include the full evidence_index with quality scores
- ALWAYS verify testing_sequence candidate IDs are valid

---

## Quality Criteria

1. Package passes all 4 integrity checks (placeholder, external ref, completeness, consistency)
2. All 5 candidates include every field fully populated
3. All headlines contain actual copy (not formula names or templates)
4. All leads contain full written copy (not descriptions or summaries)
5. Market intelligence condensed but sufficient for downstream context
6. Downstream instructions use package-internal paths (not upstream file paths)
7. Evidence index includes every unique swipe ID with quality and referencing candidates
8. Self-containment test passes for all 7 downstream skills
9. No downstream skill needs to access any file outside this package
