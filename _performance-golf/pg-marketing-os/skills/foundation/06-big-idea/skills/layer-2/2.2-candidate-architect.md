# Skill 2.2: Candidate Architect

**Version:** 1.0
**Layer:** 2 (Generation)
**Type:** Architecture Generation
**Dependencies:** candidate_strategies.json (from 2.1), layer1-output.json (from 1.6), layer0-output.json (from 0.6)
**Output:** candidate_architectures.json

---

## Purpose

Build the complete Big Idea architecture for each of the 5 candidates — core concept, Root Cause reframe, named mechanism with steps, promise statement, enemy identification, and emotional driver combination. This is the structural skeleton that headlines, leads, and proof plans will attach to.

---

## The Question

**"Does each candidate have a complete, specific, actionable architecture — not generic descriptions or placeholder slots?"**

## Process Logic

### Step 1: Build Core Concept

```
FOR each candidate from candidate_strategies:

  LOAD selected_components (big_idea_type, mechanism_angle, gap_exploited, etc.)

  SYNTHESIZE one_paragraph_description:
    Sentence 1: The contrarian insight or enemy reveal
    Sentence 2: The root cause diagnosis
    Sentence 3: The mechanism solution
    Sentence 4: The specific promise/transformation
    Sentence 5: The evidence base (swipe proof + competitor context)

  QUALITY CHECK: one_paragraph_description must:
    - Name the specific mechanism (not "a new approach")
    - Name the specific enemy or root cause (not "hidden problem")
    - Include at least one number (quality score, swipe count, competitor count)
    - Be immediately understandable without reading other fields
```

### Step 2: Build Root Cause Reframe

```
FOR each candidate:

  LOAD root_cause_technique from selected_components
  LOAD matching examples from mechanism_mapping.root_cause_inventory
  LOAD schwartz_stage from deep_research_summary

  DETERMINE emphasis_level:
    IF schwartz_stage >= 4: emphasis_level = "heavy"
    IF schwartz_stage == 3: emphasis_level = "light"
    IF schwartz_stage <= 2: emphasis_level = "minimal" (but still include)

  BUILD what_they_think:
    - Must be a SPECIFIC false belief (not "they need to diet")
    - Source from failed_solutions in deep_research_summary
    - Example: "I need to eat fewer calories and exercise more to lose belly fat"

  BUILD what_it_really_is:
    - Must name a SPECIFIC root cause (not "hidden problem in their body")
    - Source from root_cause_insights in deep_research_summary
    - Connect to the mechanism angle
    - Example: "Sluggish bile flow is preventing fat breakdown regardless of calorie intake"

  BUILD why_nothing_worked:
    - Must explain WHY the false belief failed (causal reasoning)
    - Connect the enemy to the root cause
    - Example: "Low-fat diets reduced bile production, making fat storage worse"

  QUALITY CHECK: Root Cause must:
    - Be specific enough to teach the reader something new
    - Create an "aha moment" (the reframe genuinely shifts understanding)
    - Connect logically to the mechanism (mechanism fixes this root cause)
    - NOT be generic ("there's a problem you don't know about")

  COLLECT swipe_sources (minimum 3 from root_cause_inventory examples)
```

### Step 3: Build Mechanism

```
FOR each candidate:

  LOAD mechanism_angle from selected_components
  LOAD mechanism_naming_patterns from layer1 mechanism_mapping

  SELECT naming_pattern:
    MATCH to mechanism_angle characteristics:
      - Scientific discovery → "[Authority] [Scientific Term]" or "[Discovery] [Source]"
      - System/protocol → "[Number]-Step [Benefit] [System]"
      - Compound/ingredient → "[Adjective] [Scientific Term]"
      - Method/technique → "[Number]-[Timeframe] [Benefit]"

  BUILD mechanism_name:
    APPLY selected pattern to mechanism_angle content
    VERIFY name is:
      - Memorable (under 6 words)
      - Specific (not "The Weight Loss Method")
      - Intriguing (creates curiosity)
      - Pronounceable (can be spoken naturally)

  BUILD uniqueness_claim:
    - Must state ONE specific differentiator (not "it's different")
    - Source from saturation_mapping context (what competitors DON'T do)
    - Example: "First system to target bile flow specifically, not generic digestion"

  BUILD steps_or_components (3-5):
    - Each step must be actionable (verb + specific object)
    - Steps must be logical sequence (not random list)
    - Steps must connect mechanism to root cause fix
    - FORBIDDEN: "Step 1: First step" or any placeholder language
    - Example: "Step 1: Reactivate bile production with bitter compound"

  COLLECT swipe_sources (minimum 2)
```

### Step 4: Build Promise & Enemy

```
FOR each candidate:

  BUILD promise_statement:
    core_promise:
      - Must state the specific TRANSFORMATION (not just "feel better")
      - Must connect to root cause fix ("eliminate X by fixing Y")
      - Source specificity from swipe evidence (numbers/timeframes proven in vault)
    specificity_level:
      - Include numbers where swipes prove them
      - Include timeframes where swipes prove them
      - Mark as "directional" if no specific number proven

  BUILD enemy_identified:
    enemy_name:
      - Must be a NAMED entity (not "bad habits")
      - Source from deep_research_summary.top_enemies or root_cause logic
      - Examples: "The Low-Fat Diet Industry", "Big Pharma", "The 'Calories In Calories Out' Myth"
    enemy_type:
      ASSIGN: industry | belief | condition | person | habit
    how_enemy_causes_problem:
      - Causal chain from enemy → root cause → symptom
      - Must be logically sound (not conspiracy-level)
```

### Step 5: Assign Emotional Architecture

```
FOR each candidate:

  LOAD emotional_combination from selected_components
  LOAD matching data from emotional_mapping

  RECORD:
    primary_driver (from winning_combinations)
    secondary_driver (from winning_combinations)
    combined_score (exact float from L1)

  BUILD intensity_rationale:
    "[pain_count] pain quotes mapped to [primary] driver.
     [hope_count] hope quotes mapped to [secondary] driver.
     [usage_count] swipes use this combination at [score] avg combined score."

  VERIFY emotional alignment:
    - Enemy should trigger primary_driver emotion
    - Promise should trigger secondary_driver emotion
    - Root Cause should bridge both (frustration → hope pattern)
```

## Anti-Exemplar: What FAILS This Skill

```
WEAK (FAIL):
  core_concept: "A health supplement that uses natural ingredients to help people"
  what_they_think: "They need to diet and exercise"
  what_it_really_is: "There's a hidden problem in their body"
  mechanism_name: "The Natural Weight Loss Method"
  steps: ["Step 1: Take the supplement", "Step 2: Wait for results"]
  enemy_name: "Bad habits"

STRONG (PASS):
  core_concept: "A contrarian Big Idea exposing bile flow as the root cause of
                 stubborn belly fat. The 3-Step Bile Flow Optimization Protocol
                 reactivates fat breakdown. Enemy: Low-Fat Diet Industry.
                 Proven in 5 swipes at 86.1 quality with zero competitors."
  what_they_think: "I need to eat fewer calories and exercise more to lose belly fat"
  what_it_really_is: "Sluggish bile flow prevents fat breakdown regardless of calories"
  mechanism_name: "3-Step Bile Flow Optimization Protocol"
  steps: ["Reactivate bile production with bitter compound",
          "Clear bile ducts with specific enzyme blend",
          "Support ongoing bile flow with daily optimizer"]
  enemy_name: "The Low-Fat Diet Industry"
```

## Constraints

- NEVER produce generic descriptions ("a new approach", "hidden problem", "natural method")
- NEVER use placeholder steps ("Step 1: First step", "Take the supplement")
- NEVER leave mechanism unnamed or generically named ("The Method")
- NEVER produce a root cause reframe that doesn't teach something specific
- NEVER produce an enemy that isn't a named entity
- ALWAYS source all components from Layer 1 intelligence data
- ALWAYS cite minimum 3 swipe sources for root_cause_reframe
- ALWAYS produce exactly 5 complete candidate architectures
- ALL strategy_rationale must cite exact counts (zero vague qualifiers)

---

## Quality Criteria

1. All 5 candidates have complete architecture (no empty/null fields)
2. All mechanism names are specific, memorable, and pronounceable
3. All root cause reframes teach something genuinely new (not generic)
4. All steps are actionable verbs + specific objects (not placeholders)
5. All enemies are named entities with causal connections
6. All emphasis_levels match the Schwartz stage exactly
7. All one_paragraph_descriptions are self-contained (understandable standalone)
8. Anti-exemplar check: no candidate resembles the WEAK example pattern
