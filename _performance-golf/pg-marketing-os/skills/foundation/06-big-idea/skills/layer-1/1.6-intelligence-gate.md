# Skill 1.6: Intelligence Gate

**Version:** 1.0
**Layer:** 1 (Intelligence)
**Type:** Quality Gate
**Dependencies:** pattern_analysis.json (from 1.1), saturation_mapping.json (from 1.2), gap_identification.json (from 1.3), emotional_mapping.json (from 1.4), mechanism_mapping.json (from 1.5)

## Purpose

Final quality gate for Layer 1 — validates all 5 intelligence analyses are complete, enforces the Zero Vague Qualifiers Rule and Minimum Citation Rule across all outputs, then compiles them into a single layer1-output.json ready for Layer 2 generation. Binary PASS/FAIL. No partial passes.

---

## The Question

**"Do all 5 intelligence analyses meet evidence standards, and is the compiled intelligence ready for generation?"**

## Gate Logic

### Check 1: All Analyses Complete

```
FOR each of the 5 analysis files:
  VERIFY file exists and is non-empty
  VERIFY required top-level keys are present:
    pattern_analysis: dominant_patterns, rare_high_performers, avoid_patterns
    saturation_mapping: avoid_mechanisms, saturated_mechanisms, safe_mechanisms, whitespace_mechanisms
    gap_identification: whitespace_opportunities, competitive_gaps
    emotional_mapping: primary_drivers, winning_combinations
    mechanism_mapping: root_cause_inventory, mechanism_naming_patterns, proof_demonstration_types

  COUNT entries in each section
  VERIFY minimums:
    pattern_analysis: at least 1 dominant pattern OR 1 rare high-performer
    saturation_mapping: total mechanisms >= 5
    gap_identification: at least 1 gap identified
    emotional_mapping: exactly 9 drivers mapped, at least 1 winning combination
    mechanism_mapping: at least 1 reframe technique, 1 naming pattern, 1 proof type

IF any analysis missing or below minimum:
  all_analyses_complete = false
  LOG: "[analysis_name] incomplete — [specific issue]"
```

### Check 2: Zero Vague Qualifiers

```
DEFINE forbidden_terms = [
  "many", "often", "most", "some", "several",
  "usually", "typically", "around", "approximately"
]

SCAN all 5 analysis output files for forbidden terms:
  FOR each term in forbidden_terms:
    SEARCH all recommendation strings, rationale strings, and description strings
    COUNT occurrences
    IF occurrences > 0:
      LOG violation: {term, file, field, surrounding_text}

total_violations = SUM of all occurrences

IF total_violations > 0:
  zero_vague_qualifiers = false
  LOG: "[total_violations] vague qualifier violations found"
ELSE:
  zero_vague_qualifiers = true
```

### Check 3: Minimum Citations Met

```
COLLECT all recommendation/rationale strings from all 5 analyses

FOR each recommendation:
  COUNT swipe_source references (swipe_id patterns)

  IF recommendation type is "use" or "test":
    REQUIRE >= 3 swipe_sources
    IF < 3: LOG citation failure

  IF recommendation discusses saturation:
    REQUIRE exact competitor_count (integer present)
    IF missing: LOG citation failure

  IF recommendation discusses intensity:
    REQUIRE exact quote_count (integer present)
    IF missing: LOG citation failure

IF any citation failures:
  minimum_citations_met = false
```

### Check 4: Counts Are Integers

```
SCAN all numeric fields across 5 analyses:
  usage_count, competitor_count, pain_quote_count, hope_quote_count,
  swipe_usage_count, proven_in_swipes, total_* counts

FOR each numeric field:
  VERIFY value is exact integer (not float, not range, not null)
  IF any non-integer found:
    counts_are_integers = false
    LOG: "[field] in [file] is not an integer: [value]"
```

### Check 5: Evidence Trails Valid

```
COLLECT all swipe_sources arrays from all 5 analyses

FOR each swipe_id in collected sources:
  VERIFY format matches expected pattern (e.g., collection_NNN)
  FLAG any that appear fabricated or malformed

IF fabrication concerns:
  evidence_trails_valid = false
  LOG: "Suspicious swipe_id values found: [list]"
```

### Final Determination

```
gate_status = IF (all_analyses_complete AND zero_vague_qualifiers AND
                  minimum_citations_met AND counts_are_integers AND
                  evidence_trails_valid)
              THEN "PASS"
              ELSE "FAIL"

IF gate_status == "PASS":
  COMPILE layer1-output.json with all 5 analysis outputs combined
  LOG: "LAYER 1 COMPLETE — [X] patterns, [Y] mechanisms mapped,
        [Z] gaps identified, [W] driver combinations ranked"

IF gate_status == "FAIL":
  POPULATE failure_report with all failures
  IDENTIFY first_failure (earliest in check sequence)
  DETERMINE routing:
    IF vague qualifiers: "Re-run all 5 analyses with stricter language enforcement"
    IF missing citations: route to specific analysis skill (1.1-1.5)
    IF incomplete analysis: route to specific missing skill
  STOP: Cannot proceed to Layer 2
```

## Quality Criteria

1. All 5 gate checks are binary (no partial passes)
2. Vague qualifier scan checks all 9 forbidden terms across all text fields
3. Citation audit verifies minimum 3 sources for all "use"/"test" recommendations
4. Integer validation catches floats, ranges, nulls, and string numbers
5. Evidence trail validation catches fabricated or malformed swipe_ids
6. Failure report identifies exact upstream skill to fix
7. Compiled output contains all 5 analysis files in full (not references)
8. No advancement to Layer 2 unless gate_status == PASS
