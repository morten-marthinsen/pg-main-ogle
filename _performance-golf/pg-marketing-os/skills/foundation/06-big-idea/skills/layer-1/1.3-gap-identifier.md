# Skill 1.3: Gap Identifier

**Version:** 1.0
**Layer:** 1 (Intelligence)
**Type:** Analysis
**Dependencies:** layer0-output.json (from 0.6)
**Output:** gap_identification.json

---

## Purpose

Identify strategic whitespace — proven vault patterns NOT exploited by competitors. Cross-references swipe evidence with competitor intelligence and Deep Research opportunity signals to rank gaps by exploitability. This is where Layer 2 finds its most defensible candidate strategies.

---

## The Question

**"What proven patterns are competitors ignoring, and how strong is the evidence for each gap?"**

---

input:

## Process Logic

### Step 1: Build Proven-Pattern Inventory

```
COMPILE all patterns proven in swipes:

FROM component_pools.mechanisms:
  LIST all mechanism_name values with usage_count and avg_quality_score

FROM component_pools.big_idea_types:
  LIST all type values with usage_count and avg_quality_score

FROM component_pools.root_cause_techniques:
  LIST all reframe_technique values with usage_count
```

### Step 2: Cross-Reference Against Competitor Usage

```
FOR each proven pattern:
  CHECK against deep_research_summary.competitor_mechanisms:
    competitor_count = number of competitors using this pattern

  CATEGORIZE:
    IF competitor_count == 0: WHITESPACE (complete gap)
    IF competitor_count == 1-2: COMPETITIVE GAP (underexploited)
    IF competitor_count >= 3: NOT A GAP (skip)
```

### Step 3: Validate Against Deep Research Signals

```
FOR each identified gap:
  SEARCH deep_research_summary.strategic_gaps:
    Does a strategic gap align with this pattern?
    IF yes: strategic_alignment = [specific gap description + evidence basis]

  SEARCH deep_research_summary.top_pains:
    How many pain quotes relate to this gap?
    pain_quote_support = [exact count]

  SEARCH deep_research_summary.top_wants:
    Do any top wants align with this gap?
    IF yes: ADD to strategic_alignment evidence
```

### Step 4: Score Opportunity Tier

```
FOR each gap, evaluate 4 criteria:

  CRITERION 1: Proven in swipes
    PASS if swipe_avg_quality >= 80
    PARTIAL if swipe_avg_quality >= 70 AND < 80

  CRITERION 2: Low/zero competitor usage
    PASS if used_by_competitors <= 2
    PARTIAL if used_by_competitors == 3-4

  CRITERION 3: Strategic alignment
    PASS if matches a strategic_gap from Deep Research
    PARTIAL if matches a pain quote or want (indirect)

  CRITERION 4: Volume support
    PASS if proven_in_swipes >= 3
    PARTIAL if proven_in_swipes == 2

ASSIGN opportunity_tier:
  HIGH: All 4 criteria PASS
  MEDIUM: 3 criteria PASS (or 2 PASS + 2 PARTIAL)
  LOW: 2 criteria PASS
  DISCARD: Fewer than 2 PASS
```

### Step 5: Rank and Output

```
SORT whitespace_opportunities by:
  1. opportunity_tier (HIGH first)
  2. swipe_avg_quality DESC
  3. pain_quote_support DESC

SORT competitive_gaps by:
  1. opportunity_tier (HIGH first)
  2. used_by_competitors ASC (fewer = better)
  3. swipe_avg_quality DESC

BUILD recommendation strings:
  "[TIER] PRIORITY — [competitor_count] competitor(s) using [pattern].
   Proven in [swipe_count] swipes at [quality] quality.
   [pain_count] customer pain quotes support this angle.
   [Strategic alignment detail]."
```

---

## Constraints

- NEVER use vague qualifiers (many, often, most, some, several, usually, typically, around, approximately)
- NEVER identify a gap without cross-referencing BOTH competitor data AND swipe evidence
- NEVER assign HIGH priority without all 4 criteria passing
- ALWAYS cite minimum 3 swipe_id values for each gap
- ALWAYS reference specific Deep Research data in strategic_alignment (not "research shows")
- ALWAYS count pain_quote_support as exact integer (not "several quotes")
- EVERY gap must specify gap_type (mechanism, big_idea_type, root_cause, emotional, positioning)

---

## Quality Criteria

1. All swipe counts are exact integers from actual pool data
2. All competitor counts are exact integers from Deep Research
3. All gaps cite 3+ swipe evidence sources
4. Strategic alignment references specific Deep Research entries (not generic statements)
5. Opportunity tier assignment follows the 4-criteria scoring exactly
6. Recommendation strings contain only quantified claims
7. gap_rankings.total matches actual whitespace + competitive gap array lengths combined
