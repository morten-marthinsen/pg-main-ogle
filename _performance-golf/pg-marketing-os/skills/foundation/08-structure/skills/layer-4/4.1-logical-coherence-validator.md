# Layer 4.1 — Logical Coherence Validator

## Purpose

Validate that the assembled campaign argument holds together as a logically coherent whole. Check for logical fallacies, non-sequiturs, unsupported claims, circular reasoning, and gaps in the argumentative chain. The argument must withstand scrutiny from a skeptical reader.

**Primary Persona:** Dr. Richard Stern — Skeptical Academic
**Secondary Persona:** Dr. James Liu — Research Director

---

## Execution Protocol

### Step 1: Apply the Skeptical Academic Review

Adopt the mindset of Dr. Richard Stern — a skeptical academic who:
- Questions every claim
- Demands evidence for every assertion
- Looks for logical fallacies
- Tests whether conclusions follow from premises
- Identifies gaps in reasoning

Read through the complete argument (thesis → simple segue → CPB chunks in sequence → SIN segue) and flag every point where logic falters.

### Step 2: Test the Thesis-to-Conclusion Chain

The campaign argument must create a logical chain from thesis to conclusion:

```
THESIS: [Campaign thesis statement]
   │
   ├── CPB-001 proves: [claim supported by proof]
   ├── CPB-002 proves: [claim supported by proof]
   ├── CPB-003 proves: [claim supported by proof]
   │   ...
   └── CPB-N proves: [claim supported by proof]
   │
   ▼
CONCLUSION: The thesis is proven → The mechanism works → The prospect should act
```

For each CPB chunk, verify:
- Does the claim logically support the thesis?
- Does the proof logically support the claim?
- Does the benefit logically follow from the proven claim?

### Step 3: Scan for Logical Fallacies

Check the full argument for common logical fallacies:

| Fallacy | Description | Example in Copy |
|---------|-------------|-----------------|
| **Ad Hominem** | Attacking the opponent rather than the argument | "Doctors don't know what they're talking about" without evidence |
| **Appeal to Authority** (false) | Citing authority without relevant expertise | Using a celebrity endorsement as scientific proof |
| **Straw Man** | Misrepresenting opposing views | "Conventional approaches just throw pills at the problem" (oversimplification) |
| **False Dichotomy** | Presenting only two options when more exist | "Either you use this system or you'll never succeed" |
| **Post Hoc** | Assuming correlation is causation | "She took the supplement and got better, so the supplement cured her" |
| **Hasty Generalization** | Generalizing from too few examples | "One person got results, so everyone will" |
| **Circular Reasoning** | Conclusion is used as a premise | "It works because it's the best system, and it's the best system because it works" |
| **Red Herring** | Introducing irrelevant evidence | Discussing the founder's personal story when proving mechanism effectiveness |
| **Appeal to Emotion** (unsupported) | Using emotion as substitute for evidence | "Imagine how you'll feel" without proof to back the claim |
| **Non Sequitur** | Conclusion doesn't follow from premises | "The mechanism is natural, therefore it's safe" |

**Critical distinction:** The argument CAN use emotional appeal — but emotion must be SUPPORTED by evidence, not used as a substitute for it. Per Makepeace: "People buy on emotion BUT rationalize with logic — you need BOTH."

### Step 4: Test Proof-Claim Alignment

For each CPB chunk, verify:
- Does the specific proof offered actually support the specific claim made?
- Is the proof relevant (not just impressive but unrelated)?
- Is the proof sufficient (a single weak example doesn't prove a bold claim)?
- Is there a logical connection between the evidence and the assertion?

Flag misaligned pairings with severity:
- **Critical misalignment:** Proof doesn't support the claim at all
- **Moderate misalignment:** Proof partially supports the claim but has gaps
- **Minor misalignment:** Proof supports the claim but could be stronger

### Step 5: Test Sequence Logic

Verify that the argument builds logically across the sequence:
- Does each chunk build on what came before?
- Are there logical leaps between chunks?
- Does the argument reach a natural conclusion before the SIN segue?
- Would a skeptical reader agree that the thesis has been proven by the end?

### Step 6: Score Logical Coherence

| Dimension | Weight | Scale | Description |
|-----------|--------|-------|-------------|
| Thesis-conclusion chain | 0.25 | 1-10 | Does the argument logically prove the thesis? |
| Claim-proof alignment | 0.25 | 1-10 | Does every proof support its specific claim? |
| Fallacy freedom | 0.20 | 1-10 | Are there zero logical fallacies? |
| Sequence logic | 0.15 | 1-10 | Does the argument build logically across chunks? |
| Conclusion strength | 0.15 | 1-10 | Is the conclusion (SIN segue) earned by the argument? |

**Minimum threshold:** Weighted average ≥ 7.0/10, zero critical fallacies

## Output

```yaml
logical_coherence_validation:
  overall_score: integer[1-10]

  thesis_chain:
    thesis_statement: string
    chunks_supporting_thesis: integer
    chunks_not_supporting_thesis: integer
    chain_complete: boolean
    chain_gaps: [string]  # where the chain breaks

  fallacy_scan:
    total_fallacies_detected: integer
    critical_fallacies: integer
    moderate_fallacies: integer
    minor_fallacies: integer
    fallacy_details:
      - chunk_id: string
        fallacy_type: string
        description: string
        severity: enum[critical, moderate, minor]
        remediation: string

  proof_claim_alignment:
    total_pairings_checked: integer
    aligned: integer
    misaligned: integer
    misalignment_details:
      - chunk_id: string
        claim: string
        proof_id: string
        misalignment_type: enum[critical, moderate, minor]
        description: string

  sequence_logic:
    logical_flow_verified: boolean
    logical_leaps_detected: integer
    leap_details: [string]

  conclusion_strength:
    earned: boolean
    thesis_proven: boolean
    remaining_doubts: [string]

  dimension_scores:
    thesis_conclusion_chain: integer[1-10]
    claim_proof_alignment: integer[1-10]
    fallacy_freedom: integer[1-10]
    sequence_logic: integer[1-10]
    conclusion_strength: integer[1-10]
    weighted_average: float

  pass: boolean  # true if weighted_average >= 7.0 AND critical_fallacies == 0
  remediation_required: [string]  # specific fixes needed if not passing
```

---

## Quality Gates

- [ ] Zero critical logical fallacies
- [ ] Thesis-to-conclusion chain is complete (no gaps)
- [ ] Every claim-proof pairing is aligned (no critical misalignments)
- [ ] No logical leaps between chunks in sequence
- [ ] Conclusion (SIN segue) is earned by the argument
- [ ] Weighted average ≥ 7.0/10
- [ ] Any detected issues have documented remediations
