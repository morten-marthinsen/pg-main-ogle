## Purpose

Determine the optimal ordering of assembled CPB chunks to create maximum persuasive flow. The sequence must respect dependency constraints, strategic emphasis, proof escalation patterns, and the hard rule that the "access" chunk (How can I get started?) comes last.

## Execution Protocol

### Step 1: Identify Hard Constraints

Before optimizing, lock in non-negotiable positions:

| Constraint | Rule | Source |
|-----------|------|--------|
| Access chunk LAST | "How can I get started?" always final position | E5 Framework (M5D) |
| Dependencies respected | Chunk B cannot precede Chunk A if B depends on A | Layer 1.3 dependency map |
| Mechanism before superiority | "How does it work?" must precede "Why is it better?" | Logical necessity |

### Step 2: Apply Strategy-Specific Sequencing Heuristics

Different argument strategies favor different sequences:

**Prosecution Strategy:**
```
1. Strongest claim first (establish credibility immediately)
2. Evidence cascade (build momentum through accumulation)
3. Mechanism explanation (how it works — backed by proof)
4. Results evidence (what outcomes to expect)
5. Ease/speed/safety (reduce friction)
6. Personalization (will this work for ME?)
7. Access (how to get started) — LAST
```

**Revelation Strategy:**
```
1. Root cause revelation (what you don't know)
2. Why nothing else worked (root cause explains failures)
3. Mechanism discovery (what addresses the root cause)
4. Differentiation (why this is different)
5. Results evidence (proof it works)
6. Safety/predictability (it's reliable)
7. Personalization + Access — LAST
```

**Education Strategy:**
```
1. Mechanism explanation (how it works — teach first)
2. Why it works (reason why — deepen understanding)
3. Why it's different (differentiation through education)
4. Results evidence (demonstrate outcomes)
5. Ease (it's simpler than you think)
6. Speed (results come faster than expected)
7. Personalization + Access — LAST
```

**Demonstration Strategy:**
```
1. Results evidence FIRST (show the outcome immediately)
2. Before/after contrast (transformation evidence)
3. Mechanism explanation (how these results are produced)
4. Differentiation (why other approaches can't match this)
5. Ease/speed (it's accessible and fast)
6. Safety (it's reliable and predictable)
7. Personalization + Access — LAST
```

### Step 3: Map Chunk Priority to Position

Cross-reference gap priorities with position:
- **Critical gaps** → earlier positions (address the biggest doubts first)
- **Important gaps** → middle positions (build depth)
- **Supporting gaps** → later positions (add completeness before access)

Exception: If the argument strategy explicitly calls for a different ordering (e.g., demonstration leads with results), strategy overrides priority positioning.

### Step 4: Apply Proof Escalation Pattern

Optimal proof flow across the argument:

```
EARLY CHUNKS                    MID CHUNKS                     LATE CHUNKS
┌──────────────────┐     ┌──────────────────┐     ┌──────────────────┐
│ Establish logic   │     │ Build evidence   │     │ Deliver knockout │
│ Reason why        │ ──► │ Studies, stats   │ ──► │ Demo, before/    │
│ Logic flow        │     │ Authority        │     │ after, social    │
│ (Foundation)      │     │ (Accumulation)   │     │ (Conviction)     │
└──────────────────┘     └──────────────────┘     └──────────────────┘
```

- **Early:** Logical proof (reason why, logic flow) — establishes framework
- **Middle:** Empirical proof (studies, statistics, authority) — builds evidence
- **Late:** Experiential proof (demonstration, before/after) — delivers conviction
- **Knockout proof placement:** The single strongest proof element should appear in the final third (before access chunk)

### Step 5: Validate Sequence Coherence

After determining sequence, read through the chunk order and check:

1. Does each chunk build naturally on the previous one?
2. Are dependencies respected (no forward references)?
3. Does proof density escalate appropriately?
4. Is the access chunk in final position?
5. Does the emotional intensity build toward a peak before the SIN segue?

### Step 6: Document Sequence Logic

For the final sequence, provide explicit justification for each position:

```yaml
sequence_justification:
  - position: 1
    chunk_id: string
    reason: string
  - position: 2
    chunk_id: string
    reason: string
  # ... for all positions
```
