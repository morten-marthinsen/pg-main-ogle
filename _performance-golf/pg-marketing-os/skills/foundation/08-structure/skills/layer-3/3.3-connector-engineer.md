# Layer 3.3 — Connector Engineer

## Purpose

Build natural transitions (connectors) between CPB chunks to create conversational flow. Connectors prevent the argument from feeling like a list of disconnected points. They are used where needed for flow — NOT forced between every pair of chunks.

**Primary Persona:** The Legendary Copywriter — Master Copy Composite
**Secondary Persona:** Marcus Webster — Pattern Synthesis Analyst

---

## Execution Protocol

### Step 1: Assess Connection Needs

Read through the chunk sequence and identify where connectors are needed:

**Connector NEEDED when:**
- Two chunks address very different gap types (e.g., mechanism → ease)
- Emotional intensity shifts significantly between chunks
- The proof type changes dramatically (e.g., study → demonstration)
- The argument pivots direction (e.g., from "why it works" to "how easy it is")

**Connector NOT needed when:**
- Two chunks naturally flow into each other (e.g., "How does it work?" → "Why is it better?")
- The second chunk's claim begins with a natural continuation
- Forcing a connector would feel artificial

**Rule from E5 framework:** Use connectors where desired, NOT between every chunk. Overuse makes the argument feel formulaic.

### Step 2: Select Connector Types

Match connector type to the transition need:

| Connector Type | Use When | Examples |
|---------------|----------|----------|
| **Escalation** | Building intensity | "And not just that...", "And it gets better...", "But here's what really sets this apart..." |
| **Elaboration** | Deepening current thread | "Here's why that matters...", "Let me show you what this means...", "Here's the part most people miss..." |
| **Pivot** | Changing direction | "Now, here's something else you should know...", "But that's only part of the story...", "There's another piece to this..." |
| **Revelation** | Introducing surprise | "Here's the amazing part...", "What makes this truly different...", "But here's what nobody tells you..." |
| **Contrast** | Opposing previous point | "But unlike everything else...", "Here's where it gets interesting...", "Now, you might think..." |

### Step 3: Draft Connectors for Each Needed Transition

For each identified transition point, draft 2 connector options and select the one that:
- Sounds most natural when read aloud
- Matches the prosecutorial stance of the argument strategy
- Doesn't feel forced or formulaic
- Creates genuine anticipation for the next chunk

### Step 4: Read-Aloud Simulation

Per the E5 framework and Makepeace methodology, the argument must sound conversational. For each connector:

> Read the last sentence of Chunk N, then the connector, then the first sentence of Chunk N+1. Does it flow like a person talking, or does it feel like an ad?

If it feels like an ad or written marketing copy → revise or remove the connector.

### Step 5: Prevent Connector Overuse

**Maximum connector density:** Connectors between no more than 70% of chunk transitions. At least 30% of transitions should flow naturally without an explicit connector.

**Variety check:** No connector phrase should be used more than once. Each connector should use different language.

```yaml
        from_chunk: string
        to_chunk: string
      reason_no_connector: string

  variety_check:
    unique_connector_phrases: integer
    duplicates_detected: boolean

```

---

## Quality Gates

- [ ] Connectors placed only where needed (not between every pair)
- [ ] Connector density ≤ 70% of total transitions
- [ ] No duplicate connector phrases (all unique language)
- [ ] All connectors pass read-aloud check (no "forced" ratings)
- [ ] Connector types varied (not all escalation or all revelation)
- [ ] Natural transitions documented with rationale
- [ ] Read-aloud score ≥ 7.0/10
- [ ] Connectors match argument strategy tone

---

## Anti-Patterns

- **Every-pair connectors:** Placing connectors between all chunks — makes the argument feel formulaic and artificial
- **Same connector recycled:** "And not just that..." five times — vary the language
- **Connector as claim:** "And the most amazing part is..." — this sounds like it's making a claim; connectors transition, they don't assert
- **Forced enthusiasm:** "This is going to blow your mind!" — this is hype, not a connector
- **Written connectors:** "Furthermore," "Moreover," "Additionally" — these are essay connectors, not spoken transitions
- **Missing connectors:** No connectors at all between very different chunk types — creates a jarring, list-like feel
