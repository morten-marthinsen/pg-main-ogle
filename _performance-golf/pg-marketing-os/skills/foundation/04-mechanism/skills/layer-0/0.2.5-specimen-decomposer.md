# Layer 0.2.5 — Specimen Decomposer

**Version:** 1.0
**Layer:** 0 (Foundation)
**Type:** Specimen Analysis
**Domain:** Mechanism
**Dependencies:** vault_mechanism_intelligence (from 0.2-vault-intelligence-loader.md)
**Output:** specimen_decomposition (structural templates, technique catalog, power elements)

---

## Purpose

Decompose top-scoring mechanism specimens from the vault-intelligence into reusable structural templates, technique catalogs, and power elements. Mechanism specimens contain rich structured data — names, explanation types, analogies, credibility elements, and dimensional scores. This skill extracts the architectural patterns that make elite mechanisms persuasive, not the specific content.

**This skill bridges the gap between:** vault data (scored mechanism exemplars) and generation layers (what Claude needs to learn to create original mechanisms at the same quality level).

---

```yaml
input:
  vault_mechanism_intelligence:

    untiered_fallback: "If tier metadata absent, sort by total_score DESC, top 10% = Gold, next 15% = Silver"
```

## Decomposition Protocol

### Phase 1: Specimen Selection & Tiering

```
IF tier_metadata present (Phase B loader active):
  LOAD gold_specimens → full decomposition
  LOAD silver_specimens → template-only decomposition
ELSE (pre-upgrade fallback):
  SORT specimens by total_score DESC
  SELECT top 10% as pseudo-Gold
  SELECT next 15% as pseudo-Silver
  LOG "Operating in pre-upgrade mode — tiering from total_score"

VERIFY each specimen has:
  - mechanism_name (required)
  - explanation_text (required — skip if absent)
  - e_type (required)
  - total_score (required for tiering)
  - naming_technique (preferred)
  - reframe_technique (preferred)
  - key_analogies[] (preferred)
  - credibility_elements[] (preferred)
  - dimension_scores{} (preferred)
  - source_file (required for provenance)
  - niche (required for type-matching)
```

### Phase 2: Structural Template Extraction

For each Gold specimen:

```
ANALYZE explanation_text and extract:

  explanation_skeleton:
    DECOMPOSE the explanation into ordered structural elements:
    - element_type: [problem_setup | reframe_pivot | mechanism_introduction |
                     how_it_works_sequence | why_it_works_rationale |
                     analogy_insertion | credibility_proof |
                     differentiation_claim | experiential_description |
                     scientific_grounding]
    - function: [what this element accomplishes]
    - proportion: [approximate % of explanation]
    - position: [ordinal position]

  naming_construction:
    naming_technique: [from specimen field]
    name_structure: [pattern of the name — e.g., "Adjective + Scientific Noun",
                     "Number + Category", "Branded Compound"]
    power_source: [what makes this name work — specificity, authority, novelty]

  mechanism_architecture:
    e_type: [from specimen]
    explanation_approach: [step_by_step | analogy_based | contrast_based |
                          evidence_chain | narrative_embedded]
    proof_integration: [how credibility elements are woven into explanation]
    reframe_technique: [from specimen]
    complexity_level: [simple_singular | moderate_compound | complex_system]

For Silver specimens:
  Extract explanation_skeleton and mechanism_architecture ONLY
  SKIP naming_construction detail
  SKIP element proportion analysis
```

### Phase 3: Technique Catalog Compilation

Aggregate across ALL decomposed specimens:

```
naming_techniques:
  FOR each unique naming_technique found:
    - technique_name: string (clinical, branded_format, metaphor, origin,
                              numbered, experiential, invented)
    - count: integer
    - implementation_pattern: string (how to construct a name using this technique)
    - gold_examples: [string] (verbatim mechanism_name from Gold specimens)
    - score_correlation: string (how this technique correlates with total_score)

explanation_types:
  FOR each unique e_type / explanation_approach:
    - type_name: string
    - count: integer
    - structure_pattern: string (typical element sequence for this type)
    - best_for: string (which niches/product types this works best for)
    - specimens_using: [source_file references]

reframe_techniques:
  FOR each unique reframe_technique:
    - technique_name: string (blame_removal, enemy_reveal, false_solution_expose,
                              scientific_discovery, conspiracy_reveal, contrarian_truth)
    - count: integer
    - implementation_pattern: string (how to execute this reframe)
    - gold_example: string (1-2 sentence excerpt showing this reframe in action)

analogy_patterns:
  FOR each unique analogy construction approach:
    - pattern_name: string
    - source_domain: string (what the analogy draws from)
    - target_domain: string (what it explains)
    - specimens_using: [source_file references]
    - effectiveness_notes: string

credibility_integration:
  FOR each method of weaving proof into mechanism explanations:
    - technique: string
    - position_in_explanation: string (early, middle, throughout, climax)
    - example_fragment: string (2-3 sentences from Gold specimen)
```

### Phase 4: Power Element Extraction

From Gold specimens ONLY:

```
mechanism_names:
  FOR each Gold specimen:
    - verbatim_name: string
    - naming_technique: string
    - name_structure: string
    - niche: string
    - total_score: integer
    - source_file: string

key_analogies:
  FOR each Gold specimen with key_analogies[]:
    - verbatim_analogy: string
    - source_domain: string
    - what_it_explains: string
    - source_file: string

credibility_elements:
  FOR each Gold specimen with credibility_elements[]:
    - verbatim_element: string
    - credibility_type: string (study, endorsement, institutional, experiential)
    - source_file: string

explanation_fragments:
  FOR each Gold specimen:
    - verbatim_fragment: string (2-3 strongest sentences from explanation_text)
    - fragment_function: string (what this fragment accomplishes)
    - technique_demonstrated: string (which technique it exemplifies)
    - source_file: string
```

## Quality Gates

- [ ] Minimum 3 Gold specimens fully decomposed (skeleton + naming + architecture + power elements)
- [ ] Minimum 5 Silver specimens template-decomposed (skeleton + architecture)
- [ ] All structural templates have complete explanation skeletons (no gaps)
- [ ] Technique catalog has minimum 4 naming techniques cataloged with examples
- [ ] Technique catalog has minimum 2 reframe techniques documented
- [ ] Power elements contain minimum 3 verbatim mechanism names from Gold specimens
- [ ] Power elements contain minimum 3 verbatim analogies from Gold specimens
- [ ] Every power element includes source provenance (source_file + total_score)
- [ ] Context window budget respected (templates ~35%, techniques ~40%, elements ~25%)
- [ ] No raw explanation_text passed through undecomposed

---

## Anti-Patterns

| Anti-Pattern | Signal | Fix |
|---|---|---|
| **Raw Passthrough** | Full explanation_text dumped to downstream skills | Decompose into skeleton + techniques + elements |
| **Name Without Construction** | Mechanism names listed without naming_technique analysis | Every name needs technique + structure + power_source |
| **Analogy Without Mapping** | Key analogies listed as strings without source/target domain analysis | Map source_domain → target_domain for each analogy |
| **Score-Blind Selection** | Specimens selected without regard to total_score or dimension_scores | Always sort by total_score; Gold = top 10% |
| **Niche-Blind Catalog** | Techniques cataloged without niche context | Include niche with every specimen reference |
