# Skill 0.3: Teachings Loader

**Version:** 1.0
**Layer:** 0 (Foundation)
**Type:** Data Loading
**Dependencies:** source-teachings/ directory
**Output:** teachings_framework.json

---

## Purpose

Load domain expert teachings and extract the evaluative frameworks, naming principles, and development methods that will guide mechanism creation. Teachings provide the WHY — principles and rules — while vault intelligence provides the WHAT — proven patterns.

---

## Input Schema

```yaml
input:
  source_teachings:
    mechanism_scorecard: source-teachings/mechanism-scorecard.md
    e5_method: source-teachings/e5-method.md
    naming_principles: source-teachings/naming-principles.md
```

---

## Loading Logic

### Step 1: Load Mechanism Scorecard

```
FROM source-teachings/mechanism-scorecard.md:

EXTRACT 12 dimensions:
  1. Image Strength — "Can the prospect PICTURE this mechanism working?"
  2. Simplicity — "Can a 12-year-old understand the core concept?"
  3. Proof — "Is there believable evidence this is real?"
  4. Virality — "Would someone want to TELL someone else about this?"
  5. Ease of Use — "Does the product seem EASY to use through this mechanism?"
  6. Differentiation — "Is this clearly DIFFERENT from every other solution?"
  7. Embedded Benefits — "Are the BENEFITS baked into the mechanism name itself?"
  8. Doomsday Scenario — "What TERRIBLE thing happens if they DON'T use this?"
  9. Current Belief Compatibility — "Does this FIT what they already believe?"
  10. Central Thesis Cohesion — "Does this SUPPORT and align with the Big Idea?"
  11. Super Power — "Does this give the prospect a NEW capability?"
  12. "Can Ya Feel It??" — "Does this produce a GUT-LEVEL emotional response?"

FOR each dimension:
  COMPILE scoring_rubric from teaching examples
  EXTRACT optimization_principle (how to improve if scoring low)
```

### Step 2: Load E5 Method

```
FROM source-teachings/e5-method.md:

EXTRACT explanation framework:
  - How to structure mechanism explanations
  - Levels of detail (overview → deep)
  - Credibility anchor types and placement
  - Uniqueness claim construction

COMPILE into structured e5_framework object
```

### Step 3: Load Naming Principles

```
FROM source-teachings/naming-principles.md:

EXTRACT:
  - Core naming principles (memorable, specific, ownable)
  - Pattern effectiveness rankings
  - When to use each pattern type
  - Anti-patterns (what makes a bad mechanism name)
  - Naming tests (how to evaluate a candidate name)

COMPILE into naming_rules object
```

### Step 4: Extract Development Sequence

```
FROM all teachings combined:

RECONSTRUCT the recommended development sequence:
  1. Start with root cause → mechanism type
  2. Generate naming candidates
  3. Develop explanation structure
  4. Integrate proof elements
  5. Score against scorecard
  6. Optimize weak dimensions
  7. Validate against Big Idea

MAP each step to principle source
```

---

## Quality Criteria

1. All 12 scorecard dimensions loaded with rubrics
2. Each dimension has clear optimization principle
3. E5 framework has concrete explanation structures (not just principles)
4. Naming rules include both positive patterns AND anti-patterns
5. Development sequence is logically ordered
