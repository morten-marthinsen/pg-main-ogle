# Skill 2.3: Proof Integrator

**Version:** 1.0
**Layer:** 2 (Scorecard Optimization)
**Type:** Dimension Evaluation + Optimization
**Dimension:** 3 of 12 — PROOF
**Dependencies:** gate_1_report.json, proof_maps.json, vault_patterns.json
**Output:** proof_integration_scores.json

---

## Purpose

Evaluate and optimize each mechanism candidate on PROOF: Is there believable evidence that this mechanism is REAL and WORKS? This dimension goes beyond the proof MAP (Layer 1) — it evaluates how INTEGRATED and CONVINCING the proof feels within the mechanism presentation.

---

## The Question

**"Is there believable evidence that this is REAL — not just a story?"**

---

## Scoring Rubric

```
1-3 (UNPROVEN): Mechanism is pure assertion. No evidence cited.
     - "This ancient method works" — says who?
     - No studies, no experts, no results, no logic
     - Reader thinks "prove it"

4-6 (PARTIALLY PROVEN): Some evidence but gaps remain.
     - One type of proof only (just testimonials, or just one study)
     - Evidence is vague ("studies show...")
     - Proof doesn't directly map to the mechanism's specific claims

7-8 (WELL-PROVEN): Multiple proof types, specific and relevant.
     - 2-3 proof types supporting different claims
     - Specific citations (names, numbers, institutions)
     - Proof directly addresses mechanism's claims

9-10 (UNDENIABLE): Proof is so strong it would convince a skeptic.
     - 3+ proof types creating a layered case
     - Specific, verifiable citations
     - Proof architecture builds progressively
     - Includes preemptive objection handling
```

## Evaluation Logic

### Step 1: Assess Proof Specificity

```
FOR each candidate:

  FROM proof_maps.proof_assigned:

  EVALUATE specificity of each proof element:

  VAGUE: "Studies show..." / "Experts agree..." / "Research proves..."
    → No names, no dates, no institutions, no numbers
    → Score: -3 penalty

  MODERATE: "A 2019 study found..." / "Dr. Smith recommends..."
    → Some specifics but incomplete
    → Score: neutral

  SPECIFIC: "A 2019 Stanford study of 342 participants found that..."
    → Names, numbers, institution, methodology hint
    → Score: +2 bonus

  PINPOINT: "In a double-blind study published in the Journal of
    Clinical Nutrition (March 2021), researchers at Stanford
    found that participants using [method] showed 47% improvement..."
    → Full citation with verifiable details
    → Score: +3 bonus
```

### Step 2: Assess Claim Coverage

```
FOR each candidate:

  FROM proof_maps.claims_identified:

  CHECK: Does every CRITICAL claim have specific proof?

  UNCOVERED CRITICAL CLAIM = major vulnerability
    → Skeptic will attack here
    → Competitor will differentiate here

  MAP coverage:
    claim_coverage = (critical_claims_with_specific_proof / total_critical_claims)

  MINIMUM ACCEPTABLE: 0.80 (80% of critical claims proved)
```

### Step 3: Assess Integration Quality

```
FOR each candidate:

  EVALUATE how proof FEELS in the mechanism presentation:

  NATURAL INTEGRATION:
    ✓ Proof flows as part of the story
    ✓ Evidence appears at the moment of claim
    ✓ Reader doesn't feel "sold to"
    Example: "When researchers at Harvard tested this on traders,
    they found something remarkable..."

  FORCED INTEGRATION:
    ✗ Proof bolted on after the explanation
    ✗ Evidence in a separate "proof section"
    ✗ Feels like a courtroom defense
    Example: "Here's the proof: Study 1... Study 2... Study 3..."

  SCORE integration quality separately from proof strength
```

### Step 4: Identify Skeptic Vulnerabilities

```
FOR each candidate:

  SIMULATE a SKEPTIC reading the mechanism:

  What would they challenge?
    - "That study was too small"
    - "That's correlation, not causation"
    - "Those testimonials could be fake"
    - "That expert isn't qualified in this field"
    - "That's cherry-picked data"

  IDENTIFY the #1 skeptic vulnerability
  LOG for objection preemption in optimization
```

### Step 5: Optimize If Below Threshold

```
IF current_score < 7:

  TECHNIQUE 1: Upgrade Specificity
    FOR each vague proof element:
      ADD names, numbers, institutions, dates
      IF real citations unavailable:
        UPGRADE to logical proof or demonstration

  TECHNIQUE 2: Add Missing Proof Types
    IF only testimonials: ADD logical proof or expert endorsement
    IF only studies: ADD experiential demonstration or case study
    IF only logical: ADD data point or social proof

  TECHNIQUE 3: Preempt Objections
    FOR each skeptic vulnerability:
      ADD preemptive language:
        "You might think [objection], but [counter]..."
        "Before you dismiss this, consider [strong evidence]..."

  TECHNIQUE 4: Improve Integration
    MOVE proof INTO the explanation narrative
    ELIMINATE separate "proof dumps"
    WEAVE evidence naturally into the mechanism story

  RE-SCORE after optimization
```

---

## Quality Criteria

1. Every candidate has specificity assessment (no "vague" proof in top scorers)
2. Claim coverage ≥ 0.80 for all candidates proceeding
3. At least 2 proof types per candidate
4. Skeptic vulnerability identified and addressed
5. Proof feels integrated, not bolted on
6. No "studies show" without specifics in optimized versions
