# Skill 2.10: Thesis Cohesion Aligner

**Version:** 1.0
**Layer:** 2 (Scorecard Optimization)
**Type:** Dimension Evaluation + Optimization
**Dimension:** 10 of 12 — CENTRAL THESIS COHESION
**Dependencies:** gate_1_report.json, upstream_context.json, explanation_structures.json
**Output:** thesis_cohesion_scores.json

---

## Purpose

Evaluate and optimize each mechanism candidate on CENTRAL THESIS COHESION: Does this mechanism SUPPORT and strengthen the Big Idea? The mechanism must be the LOGICAL PROOF of the Big Idea — the "how" behind the "what if." A disconnected mechanism weakens both itself and the Big Idea.

---

## The Question

**"Does this mechanism PROVE the Big Idea is true and achievable?"**

---

## Scoring Rubric

```
1-3 (DISCONNECTED): Mechanism has no logical relationship to Big Idea.
     - Big Idea is about X, mechanism explains Y
     - Reader can't see how they connect
     - Feels like two different promotions stitched together

4-6 (LOOSELY CONNECTED): Mechanism is related but not essential.
     - Big Idea could exist without this mechanism
     - Connection requires explanation to see
     - "Oh, I guess that relates..."

7-8 (STRONGLY CONNECTED): Mechanism directly supports Big Idea.
     - Big Idea makes a promise, mechanism explains HOW to deliver it
     - Reader sees: "Ah, that's how it works"
     - Mechanism makes the Big Idea more believable

9-10 (INSEPARABLE): Mechanism IS the proof of the Big Idea.
     - Remove the mechanism and the Big Idea collapses
     - The mechanism is WHY the Big Idea is true
     - "The reason [Big Idea promise] is possible is because [mechanism]"
     - They are two sides of the same coin
```

---

## Evaluation Logic

### Step 1: Articulate the Connection Chain

```
FOR each candidate:

  FROM upstream_context.big_idea_context:
    BIG IDEA CONCEPT: [what it is]
    BIG IDEA PROMISE: [what it promises]
    BIG IDEA ANGLE: [how it approaches the market]

  FROM explanation_structures:
    MECHANISM CLAIM: [what it says]
    MECHANISM ACTION: [what it does]
    MECHANISM RESULT: [what it produces]

  BUILD CONNECTION CHAIN:
    Big Idea says: "[promise]"
    Mechanism explains: "[how that promise is delivered]"
    Therefore: "[mechanism] is WHY [Big Idea] is true"

  IF this chain flows naturally → high score
  IF this chain requires logical leaps → low score
  IF this chain can't be constructed → very low score
```

### Step 2: Test Necessity

```
FOR each candidate:

  THE REMOVAL TEST:
    "If I remove this mechanism, does the Big Idea still work?"

  IF YES (Big Idea stands alone):
    → Mechanism is supplementary, not essential
    → Score capped at 6 (loosely connected)
    → Consider: Is the mechanism even needed?

  IF PARTIALLY (Big Idea weakens):
    → Mechanism supports but doesn't prove
    → Score range: 5-7
    → Mechanism adds credibility but isn't the foundation

  IF NO (Big Idea collapses):
    → Mechanism is the proof of the Big Idea
    → Score range: 8-10
    → This is the target relationship
```

### Step 3: Test Coherence Under Scrutiny

```
FOR each candidate:

  SIMULATE a reader asking:
    "Wait, you say [Big Idea]. But HOW is that possible?"
    Does the mechanism answer this question COMPLETELY?

  THEN:
    "Okay, so [mechanism] does [action]. But does that actually
     mean [Big Idea promise] is achievable?"
    Does the logic hold?

  THEN:
    "If [mechanism] works, does anything else need to be true
     for [Big Idea] to deliver on its promise?"
    Are there missing links?

  GAPS in this chain = lower score
  TIGHT chain = higher score
```

### Step 4: Optimize If Below Threshold

```
IF current_score < 7:

  TECHNIQUE 1: Strengthen the "Because" Bridge
    REWRITE mechanism explanation to explicitly connect:
    "[Big Idea promise] is now possible BECAUSE [mechanism]
     does [specific action] that [directly produces result]"

  TECHNIQUE 2: Make Big Idea DEPENDENT on Mechanism
    ADJUST mechanism framing so Big Idea REQUIRES it:
    "The only way [Big Idea promise] can work is if [mechanism]
     is activated — and until now, nobody knew how to activate it"

  TECHNIQUE 3: Align Vocabulary
    USE the same core words in both Big Idea and Mechanism
    If Big Idea is about "liberation": mechanism "liberates" something
    If Big Idea is about "speed": mechanism "accelerates" something
    Linguistic cohesion reinforces logical cohesion

  TECHNIQUE 4: Show Exclusivity
    "Without [mechanism], [Big Idea promise] is just a nice idea.
     WITH [mechanism], it becomes inevitable."
    → Makes mechanism the KEY that unlocks the Big Idea

  RE-SCORE after optimization
```

## Output Schema

```yaml
output:
  thesis_cohesion_scores:
    for_each_candidate:
      - candidate_name: string
        current_score: integer[1-10]
        cohesion_analysis:

          mechanism_claim: string
          connection_chain: string
          necessity_level: enum[supplementary, supporting, essential]

        optimized_version:

          dependency_statement: string
```

## Quality Criteria

1. Connection chain explicitly articulated for every candidate
2. Necessity test determines if mechanism is essential to Big Idea
3. Logical gaps identified and addressed
4. Top candidates make Big Idea DEPENDENT on mechanism
5. Vocabulary alignment reinforces logical connection
