# Skill 3.4: Anti-Slop Validator

**Version:** 1.0
**Layer:** 3 (Validation & Selection)
**Type:** Language Quality Gate
**Dependencies:** threshold_gate_report.json (viable candidates), explanation_structures.json, naming_candidates.json, anti-slop-standards.md
**Output:** slop_validation_report.json

---

## Purpose

Enforce language quality across all mechanism text outputs. A mechanism can score perfectly on the 12-point scorecard and still FAIL if its language reads like AI-generated slop. This skill runs the anti-slop standards against every text element of each viable candidate — names, explanations, analogies, proof language — and flags or rewrites violations.

---

## The Question

**"Does this mechanism's language pass the 'read aloud' test — or does it sound like a robot wrote it?"**

---

## Input Schema

```yaml
input:
  viable_candidates: from 3.2 (only those that passed threshold gate)
  explanation_structures: from 1.3 (all text content)
  naming_candidates: from 1.2 (selected names)

```

## Validation Logic

### Step 1: Extract All Text Elements

```
FOR each viable candidate:

  COLLECT all text elements that will appear in final output:

  1. MECHANISM NAME — the selected name itself
  2. EXPLANATION L1 — one-sentence version
  3. EXPLANATION L2 — paragraph version
  4. EXPLANATION L3 — full walkthrough version
  5. ANALOGY — all deployed versions
  6. PROOF LANGUAGE — how proof is stated/integrated

  Each element is validated independently
  Total slop density is also calculated across ALL elements combined
```

### Step 2: Run Banned Phrase Scan

```
FOR each text element:

  SCAN for banned phrases (exact match or close variants):

  FILLER OPENERS:
    - "It's important to note that..."
    - "In today's fast-paced world..."
    - "At the end of the day..."
    - "It goes without saying..."
    - "As we all know..."
    - "In the realm of..."

  HEDGE WORDS (unless genuinely uncertain):
    - "perhaps"
    - "it seems"
    - "one might argue"
    - "arguably"
    - "it could be said"
    - "in some ways"

  EMPTY INTENSIFIERS:
    - "very" (replace with specific word)
    - "really" (delete or dimensionalize)
    - "extremely" (quantify instead)
    - "incredibly" (show, don't tell)
    - "absolutely" (let the fact speak)

  AI TELLTALES:
    - "Let's dive in..."
    - "Without further ado..."
    - "In conclusion..."
    - "Moving forward..."
    - "It's worth mentioning..."
    - "Needless to say..."
    - "harness" / "unlock" / "leverage" (when used generically)
    - "game-changer" / "revolutionary" (unsupported superlatives)
    - "seamlessly" / "effortlessly" (magic words hiding complexity)

  LOG each violation with:
    - phrase found
    - location (which element, which sentence)
    - suggested replacement or deletion
```

### Step 3: Structural Quality Checks

```
FOR each text element:

  CHECK 1: Active Voice Ratio
    COUNT sentences in active voice vs. passive
    THRESHOLD: ≥ 80% active voice
    FLAG each passive construction with active rewrite

  CHECK 2: Abstract Noun Density
    COUNT abstract nouns (concepts without physical referent)
    Examples: "optimization", "implementation", "functionality", "paradigm"
    THRESHOLD: ≤ 1 abstract noun per sentence on average
    FLAG each abstract noun with concrete alternative

  CHECK 3: Sentence Necessity
    FOR each sentence:
      APPLY deletion test: "If removed, is meaning lost?"
      IF meaning survives deletion → FLAG as filler

  CHECK 4: Specificity Audit
    FOR each paragraph-length element:
      REQUIRE at least one specific/concrete detail
      "37% improvement" beats "significant improvement"
      A name, number, or tangible object beats abstractions

  CHECK 5: Forward Momentum
    FOR each sentence pair:
      DOES sentence N make the reader want sentence N+1?
      FLAG any sentence that creates a dead end or circular reference
```

### Step 4: Calculate Slop Density

```
FOR each text element:

  slop_density = total_violations / (word_count / 500)

  RATING:
    0.0–1.0 → EXCELLENT (ship as-is)
    1.1–2.0 → ACCEPTABLE (minor revision)
    2.1–3.0 → WEAK (full rewrite of flagged sections)
    3.1+    → FAILING (restart element from scratch)

FOR aggregate (all elements combined):

  overall_slop_density = total_all_violations / (total_word_count / 500)

  GATE DECISION:
    overall ≤ 2.0 AND no single element > 3.0 → PASS
    overall ≤ 2.0 BUT one element > 3.0 → REVISE (fix that element)
    overall > 2.0 → REVISE (systematic cleanup)
    overall > 3.0 → FAIL (fundamental language problem)
```

### Step 5: Revise Violations (If REVISE)

```
IF gate_result = "REVISE":

  FOR each flagged violation:

    APPLY revision techniques:

    BANNED PHRASES → Delete or replace:
      "It's important to note that X" → "X"
      "In today's fast-paced world" → [DELETE — start with the actual point]
      "perhaps this could" → "this does" (if true) or [DELETE] (if uncertain)

    EMPTY INTENSIFIERS → Dimensionalize:
      "very effective" → "47% more effective" or "effective within 3 days"
      "incredibly powerful" → "powerful enough to [specific capability]"
      "really works" → "works for [specific evidence]"

    PASSIVE VOICE → Activate:
      "The toxins are flushed by the process" → "The process flushes toxins"
      "Results were achieved" → "Users achieved results"

    ABSTRACT NOUNS → Concretize:
      "optimization of functionality" → "making the switches faster"
      "implementation of the protocol" → "using the 3-step method"

    AI TELLTALES → Humanize:
      "This game-changing approach" → "This approach" [let evidence show impact]
      "seamlessly integrates" → "fits into your routine without extra steps"
      "harness the power of" → "use" [simple verb]

  RE-CALCULATE slop_density after revisions
  RE-ASSESS gate_result
```

### Step 6: The Read-Aloud Test

```
FOR each viable candidate (post-revision if revised):

  SIMULATE reading all text elements aloud:

  LISTEN FOR:
    - Phrases that feel awkward spoken (flag)
    - Sentences that require re-reading (flag)
    - Transitions that feel robotic (flag)
    - Language that sounds "written" not "spoken" (flag)
    - Any moment where a human would say "what?" (flag)

  THE HUMAN BAR:
    "Could a skilled copywriter have written this,
     or does it betray its AI origin?"

  IF betrays AI origin → additional revision needed
  IF sounds like natural expert speech → PASS
```

---

## Quality Criteria

1. Every text element scanned for all 4 banned phrase categories
2. Slop density calculated per-element AND aggregate
3. Active voice ratio verified (≥ 80%)
4. No element ships with density > 2.0
5. Post-revision density must be ≤ 2.0 for PASS
6. The "read aloud" test applied as final check
7. Zero tolerance for AI telltale phrases in final output
