# Skill 3.5: Anti-Slop Validator

**Version:** 1.0
**Layer:** 3 (Validation & Selection)
**Type:** Quality Enforcement
**Dependencies:** All Layer 1 + Layer 2 outputs (complete offer)
**Output:** anti_slop_report.json
**Persona:** Dr. Richard Stern (Skeptic/Devil's Advocate) + Sarah A. Conco (Compliance/Ethics)

---

## Purpose

Final quality sweep across the entire offer for generic language, unsupported claims, framework clichés, template artifacts, and anything that would make a sophisticated prospect think "this is just another guru offer." Zero tolerance. The anti-slop validator is the last line of defense before the offer is packaged for downstream skills.

This skill does not evaluate strategy or architecture — those were validated by Gates 1 and 2 and scored by Skills 3.1-3.4. This skill evaluates OUTPUT QUALITY: is the actual language, specificity, and originality at the level this system demands?

---

## The Principle

**"Slop is the natural decay state of AI-generated marketing. Without active resistance, every output gravitates toward template language, inflated claims, and hollow enthusiasm. This skill exists to fight that gravity."**

## Slop Categories

### Category 1: Framework Clichés

```
SCAN all text outputs for Hormozi/marketing framework language
that was meant as INSTRUCTION, not output:

BANNED PHRASES (direct framework language):
  ✗ "So good they'd feel stupid saying no"
  ✗ "Grand Slam Offer"
  ✗ "Value equation"
  ✗ "Dream outcome"
  ✗ "Perceived likelihood of achievement"
  ✗ "Trim and stack"
  ✗ "Enhancement stack"
  ✗ "Delivery vehicle"

These are INTERNAL terms. They should never appear in prospect-facing outputs.

ALSO SCAN for Todd Brown template artifacts:
  ✗ "[insert deliverable name here]"
  ✗ "D-F-W-B-P"
  ✗ "[your product name]"
  ✗ Any bracketed placeholder text
  ✗ Template instructions appearing as content

FINDING: Each instance → FLAG as FRAMEWORK_CLICHE
TOLERANCE: ZERO — every instance must be replaced with prospect-facing language
```

### Category 2: Generic Marketing Language

```
SCAN for language that could appear in ANY offer, regardless of product:

GENERIC PHRASES:
  ✗ "Revolutionary new system"
  ✗ "Transform your life"
  ✗ "Limited time offer" (without specific deadline)
  ✗ "Act now"
  ✗ "Don't miss out"
  ✗ "You deserve this"
  ✗ "Imagine a world where..."
  ✗ "What if I told you..."
  ✗ "Here's the thing..."
  ✗ "The truth is..."
  ✗ "But wait, there's more"
  ✗ "Game-changing"
  ✗ "Life-changing"
  ✗ "One-of-a-kind"
  ✗ "State of the art"

TEST: Replace our product name with any other product.
  Does the sentence still work?
  IF YES: It's too generic. It needs specificity.

FINDING: Each instance → FLAG as GENERIC_LANGUAGE
TOLERANCE: Maximum 3 instances in entire offer (some generic phrases are functional)
```

### Category 3: Unsupported Claims

```
SCAN for claims that lack evidence:

PATTERN 1: Numbers without sources
  ✗ "97% of people who..."  — source?
  ✗ "Studies show..."       — which studies?
  ✗ "Research proves..."    — what research?
  ✗ "Experts agree..."      — which experts?

PATTERN 2: Superlatives without proof
  ✗ "The most effective..."  — compared to what? Says who?
  ✗ "The fastest way to..."  — measured how?
  ✗ "The only system that..." — verified how?
  ✗ "The best [X] ever..."  — by what standard?

PATTERN 3: Results claims without testimonial/data support
  ✗ "You'll see results in 24 hours"  — who has?
  ✗ "Members routinely achieve..."    — specific members?
  ✗ "Guaranteed to work"              — backed by what proof?

FOR each unsupported claim:
  CHECK proof_inventory: Is there evidence available?
    IF YES: Flag to add the evidence
    IF NO: Flag as unsupportable — must be removed or softened

FINDING: Each instance → FLAG as UNSUPPORTED_CLAIM
TOLERANCE: ZERO for Pattern 1 and 2. Maximum 2 for Pattern 3 (with softening language)
```

### Category 4: AI-Generated Tells

```
SCAN for patterns that reveal AI generation:

STRUCTURAL TELLS:
  ✗ Perfect parallel sentence construction across multiple items
    (AI loves matching structures — real copy has natural variation)
  ✗ Every bullet starting with the same part of speech
  ✗ Overly balanced lists (exactly 3 of everything)
  ✗ Synonyms used in obvious rotation to avoid repetition

TONE TELLS:
  ✗ Excessive enthusiasm markers ("incredible!", "amazing!", "unbelievable!")
  ✗ Hedging language ("could potentially help you possibly...")
  ✗ Over-qualification ("it's important to note that...")
  ✗ Unnecessary summarization ("in summary, to conclude, in other words")

VOCABULARY TELLS:
  ✗ "Leverage" (used as a verb for anything)
  ✗ "Unlock" (unlock your potential, unlock results)
  ✗ "Harness" (harness the power of)
  ✗ "Elevate" (elevate your game, elevate your life)
  ✗ "Empower" (empower yourself)
  ✗ "Seamlessly" / "Effortlessly"
  ✗ "Cutting-edge" / "Groundbreaking"
  ✗ "Robust" / "Comprehensive"
  ✗ "Delve" / "Navigate" / "Landscape"

FINDING: Each instance → FLAG as AI_TELL
TOLERANCE: Maximum 5 instances across entire offer
```

### Category 5: Ethical/Compliance Issues

```
Sarah A. Conco COMPLIANCE SCAN:

HEALTH CLAIMS:
  ✗ Diagnosing, treating, or curing medical conditions
  ✗ Specific health outcomes without FDA-compliant language
  ✗ Before/after claims that imply medical results

INCOME CLAIMS:
  ✗ Specific income figures without "typical results" disclaimer
  ✗ Implying guaranteed income
  ✗ Lifestyle claims tied to income ("buy your dream house")

TESTIMONIAL CLAIMS:
  ✗ Testimonials presented as typical without disclaimer
  ✗ Fabricated or unverifiable testimonials
  ✗ Results that exceed what the product typically delivers

GUARANTEE CLAIMS:
  ✗ Guarantee language that doesn't match actual terms
  ✗ Implied guarantees stronger than what's offered
  ✗ Risk-free claims with hidden conditions

FTC COMPLIANCE:
  ✗ Material connections not disclosed (if applicable)
  ✗ Deceptive pricing (inflated "retail" values without basis)
  ✗ Fake scarcity or urgency

FINDING: Each instance → FLAG as COMPLIANCE_ISSUE
TOLERANCE: ZERO — all compliance issues must be resolved
```

## Validation Process

### Step 1: Scan All Text Outputs

```
COLLECT all prospect-facing text from:
  - dream_outcome.json → all text fields
  - value_stack.json → all descriptions and benefit statements
  - bonus_designs.json → all names, descriptions, imagery
  - guarantee_architecture.json → all language versions
  - scarcity_strategy.json → all messaging
  - urgency_strategy.json → all messaging
  - offer_names.json → primary and alternate names
  - price_architecture.json → all presentation scripts
  - value_demonstration_script.json → full script
  - core_offer_definition.json → offer thesis, tagline

TOTAL SCAN SURFACE: All prospect-facing language
```

### Step 2: Run All 5 Category Scans

```
FOR each category (1-5):
  SCAN entire text surface
  FLAG every instance with:
    - Category
    - Specific text flagged
    - Location (which output file, which field)
    - Severity (CRITICAL / WARNING / MINOR)
    - Recommended fix
```

### Step 3: Calculate Anti-Slop Score

```
SCORING:
  Start at 10.0 (perfect)

  DEDUCTIONS:
    Framework Cliché: -0.5 per instance
    Generic Language: -0.2 per instance (first 3 free)
    Unsupported Claim: -0.5 per instance
    AI Tell: -0.1 per instance (first 5 free)
    Compliance Issue: -1.0 per instance

  anti_slop_score = 10.0 - total_deductions (floor: 0.0)

PASS/FAIL DETERMINATION:
  PASS: anti_slop_score ≥ 8.0 AND zero compliance issues AND zero framework clichés
  FAIL: anti_slop_score < 8.0 OR any compliance issue OR any framework cliché
```

## Quality Criteria

1. All 5 category scans completed across entire text surface
2. Zero framework clichés in prospect-facing language
3. Zero compliance issues
4. Unsupported claims either evidenced or softened
5. AI tells minimized (≤ 5 across entire offer)
6. Anti-slop score ≥ 8.0/10
7. Every flag has a specific recommended fix
8. Language reads as written by a knowledgeable human, not generated by AI

---

## Anti-Patterns

- Do NOT rubber-stamp the output — this validator exists specifically to catch what other skills missed
- Do NOT flag technical accuracy as "slop" (specific claims with evidence are fine)
- Do NOT over-correct into bland, generic language trying to avoid AI tells
- Do NOT ignore compliance issues because "it sounds good" — compliance is non-negotiable
- Do NOT accept "studies show" without the study — ever
