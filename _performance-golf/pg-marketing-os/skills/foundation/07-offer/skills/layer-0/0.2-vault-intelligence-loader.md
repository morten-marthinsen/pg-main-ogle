## Loading Logic

### Step 1: Scan Vault Directory

```
SCAN TIER1_EXTRACTIONS/ for all .md files
COUNT total files available
LOG: "Found [N] processed swipe files for offer pattern mining"

FOR each file:
  CHECK file is non-empty
  CHECK file contains structured extraction fields
  IF valid: ADD to processing queue
  IF invalid: LOG as skipped with reason
```

### Step 2: Extract Offer-Relevant Fields

```
FOR each valid swipe file:

  EXTRACT offer_data:
    source_name: filename / campaign identifier
    niche: extracted niche/market

    # Core Offer Structure
    main_product: what is being sold
    product_type: info_product | supplement | software | service | physical | hybrid
    positioning: how the product is framed (tool/system/secret/method)

    # Value Architecture
    value_stack:
      components: [list of value stack items]
      sequence: order of presentation
      total_stated_value: aggregate dollar value claimed
      component_count: number of items in stack

    # Price Architecture
    price_psychology:
      anchor_price: stated retail or comparison value
      actual_price: what buyer pays
      discount_frame: how savings are presented
      payment_structure: single / installment / trial
      price_justification: narrative for why price is fair
      comparison_anchors: what price is compared to

    # Enhancement Elements
    bonuses:
      count: number of bonuses
      bonus_list: [name, description, stated_value, type]
      naming_patterns: how bonuses are named
      value_exceeds_core: boolean (do bonuses outvalue core?)

    urgency_scarcity:
      urgency_type: deadline / limited_time / seasonal / cohort
      scarcity_type: limited_supply / never_again / bonus_expiring
      mechanism: how urgency/scarcity is enforced
      language: exact phrases used

    guarantee:
      type: unconditional / conditional / anti / stacked / implied
      duration: timeframe
      language: exact guarantee statement
      conditions: what buyer must do (if conditional)
      risk_reversal_strength: 1-10

    # Presentation Flow
    offer_flow:
      sequence: ordered list of presentation chunks
      transition_language: how sections connect
      cta_count: number of calls to action
      cta_positions: where CTAs appear in the flow

    # Emotional Architecture
    emotional_drivers:
      primary_pain: dominant pain point leveraged
      primary_desire: dominant desire leveraged
      consequence_amplification: how current pain is amplified
      after_state_painting: how desired outcome is visualized

  IF extraction yields < 3 offer-relevant fields:
    SKIP file (not offer-rich enough)
  ELSE:
    ADD to compiled_patterns
```

### Step 3: Compile Pattern Intelligence

```
ACROSS all extracted files, COMPILE:

FREQUENCY ANALYSIS:
  product_types:
    [type]: count, percentage
  positioning_approaches:
    [approach]: count, percentage
  bonus_counts:
    average: number
    median: number
    range: min-max
  guarantee_types:
    [type]: count, percentage
  urgency_types:
    [type]: count, percentage
  payment_structures:
    [structure]: count, percentage
  cta_counts:
    average: number
    most_common: number

VALUE ARCHITECTURE PATTERNS:
  average_stated_value: dollar amount
  average_actual_price: dollar amount
  average_discount_percentage: percentage
  value_to_price_ratio: average multiple
  most_common_stack_size: number of components
  bonus_value_vs_core_ratio: percentage

NAMING PATTERNS:
  product_name_structures: [categorized patterns]
  bonus_naming_conventions: [categorized patterns]
  power_words_in_names: [frequency-ranked list]

GUARANTEE PATTERNS:
  most_common_type: type
  most_common_duration: timeframe
  conditional_vs_unconditional_ratio: percentage
  stacked_guarantee_frequency: percentage

PRESENTATION FLOW PATTERNS:
  most_common_sequence: ordered list
  average_deliverable_count: number
  transition_language_library: [common phrases]
  cta_placement_patterns: [positions]

LANGUAGE PATTERNS:
  value_demonstration_phrases: [examples]
  price_transition_phrases: [examples]
  risk_reversal_phrases: [examples]
  urgency_phrases: [examples]
  consequence_amplification_phrases: [examples]
```

### Step 4: Identify Exemplars

```
FROM compiled_patterns:

SELECT top 10 offer exemplars based on:
  - Completeness (most offer fields populated)
  - Sophistication (most complex value architecture)
  - Diversity (represent different niches/approaches)

FOR each exemplar:
  COMPILE full offer profile:
    source: campaign name
    niche: market
    complete_offer_summary: structured overview
    what_makes_it_notable: specific strengths
    techniques_worth_emulating: actionable patterns

ALSO SELECT bottom 5 anti-exemplars:
  - Sparse offer architecture
  - Missing critical components
  - Generic/commodity positioning

FOR each anti-exemplar:
  COMPILE avoidance profile:
    source: campaign name
    what_went_wrong: specific weaknesses
    what_to_avoid: actionable warnings
```

### Step 5: Specimen Tiering

```
FOR each specimen in compiled_patterns:

  CALCULATE offer_completeness_score:
    +1 IF value_stack present with 2+ components
    +1 IF price_psychology present with anchor_price AND actual_price
    +1 IF guarantee present with type AND language
    +1 IF bonuses present with 1+ entries
    +1 IF urgency_scarcity present with mechanism identified
    +1 IF offer_flow present with sequence
    +1 IF emotional_drivers present (primary_pain AND primary_desire)
    +1 IF complete offer architecture (5+ of above present)
    MAX_SCORE: 8

  SORT specimens by offer_completeness_score DESC
  ASSIGN tier:
    Gold = top 10% (offer_completeness_score >= 7)
    Silver = next 15% (offer_completeness_score >= 5)
    Standard = remainder

  TAG each specimen with:
    tier: enum[gold, silver, standard]
    offer_completeness_score: integer (0-8)

  OUTPUT tier_distribution:
    total_specimens: integer
    gold_count: integer
    silver_count: integer
    standard_count: integer
```

### Step 6: Type-Matched Retrieval

```
DEFINE match_fields:
  primary: product_type (info_product, supplement, software, service, physical, hybrid)
  secondary: niche (e.g., health, finance, survival, self-improvement)
  tertiary: guarantee.type (unconditional, conditional, anti, stacked, implied)

WHEN campaign_context is available:
  STRATEGY 1 — exact_match:
    FILTER specimens WHERE product_type == target AND niche == target_niche
    PRIORITIZE Gold tier first, then Silver

  STRATEGY 2 — type_niche_match:
    FILTER specimens WHERE product_type == target (any niche)
    PRIORITIZE Gold tier, same-niche Silver, then cross-niche Silver

  STRATEGY 3 — cross_type:
    RETURN all Gold specimens across product types
    INCLUDE Silver specimens from target niche

WHEN campaign_context is NOT available:
  RETURN full specimen set with tier metadata attached
  SORT by offer_completeness_score DESC

OUTPUT retrieval_result:
  matched_specimens: [specimen]
  match_strategy_used: enum[exact_match, type_niche_match, cross_type, full_set]
  gold_specimens_available: integer
  silver_specimens_available: integer
```

## Quality Criteria

1. Minimum 100 files scanned from vault (of 139+ available)
2. Minimum 50 files yield extractable offer data
3. All frequency analyses populated (no empty categories)
4. Minimum 10 exemplars identified with full profiles
5. Language library contains minimum 5 examples per category
6. Pattern data is quantified (counts and percentages, not just lists)
7. Tiering applied: Gold specimens identified (top 10%)
8. Tiering applied: Silver specimens identified (top 25%)
9. Type-matched retrieval functional with campaign context
10. Fallback to full set works without campaign context
11. Tier distribution logged

---

## Failure Conditions

- TIER1_EXTRACTIONS directory not found → GATE_0 FAIL
- Fewer than 30 files yield offer data → GATE_0 WARNING (proceed with reduced confidence)
- Zero exemplars identified → GATE_0 FAIL (no empirical reference for offer construction)
