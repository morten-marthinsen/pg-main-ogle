# Layer 0.2.5 — Specimen Decomposer

**Version:** 1.0
**Layer:** 0 (Foundation)
**Type:** Specimen Analysis
**Domain:** Offer
**Dependencies:** vault_offer_intelligence (from 0.2-vault-intelligence-loader.md)
**Output:** specimen_decomposition (structural templates, technique catalog, power elements)

---

## Purpose

Decompose top-scoring offer specimens from the vault-intelligence into reusable structural templates, technique catalogs, and power elements. Offer specimens are highly structured — value stacks, price psychology, guarantees, bonuses, urgency devices, and risk reversal. This skill extracts the offer architecture that makes elite direct response offers irresistible and conversion-maximized.

**This skill bridges the gap between:** vault data (structured offer components across 261 campaigns) and generation layers (what Claude needs to construct original offer architectures with the same strategic depth).

## Decomposition Protocol

### Phase 1: Specimen Selection & Tiering

```
IF tier_metadata present (Phase B loader active):
  LOAD gold_specimens → full decomposition
  LOAD silver_specimens → template-only decomposition
ELSE (pre-upgrade fallback):
  SELECT top 5 by position as pseudo-Gold
  SELECT next 10 as pseudo-Silver
  LOG "Operating in pre-upgrade mode — tiering estimated from array position"

VERIFY each offer specimen has:
  - value_stack[] (required — at least 1 item)
  - price_psychology{} (required)
  - guarantee{} (required)
  - source_file (required for provenance)
  - niche (required for type-matching)
  - bonuses[] (preferred)
  - urgency_devices[] (preferred)
  - risk_reversal{} (preferred)
```

### Phase 2: Structural Template Extraction

For each Gold specimen:

```
ANALYZE offer structure to extract:

  value_stack_skeleton:
    - position: integer (order in stack)
      item_type: string (core_product | bonus | special_report |
                         tool_access | community | consultation |
                         continuity | one_time_deliverable)
      value_framing: string (how value is communicated — stated_dollar,
                             implied_value, priceless_positioning,
                             comparison_anchor)
      function: string (what this item adds to the offer's persuasion)
    stack_size: integer
    total_stated_value: string
    value_to_price_ratio: string (if calculable)

  pricing_architecture:
    anchor_strategy: string (how the anchor price is established)
    discount_frame: string (how the actual price is positioned vs anchor)
    payment_structure: string (one_time | subscription | split_pay)
    justification_method: string (what logic supports the price)
    comparison_anchors: [string] (what the price is compared to)

  guarantee_architecture:
    guarantee_type: string (money_back | conditional | unconditional |
                            keep_everything | double_your_money)
    duration: string
    risk_reversal_strength: string (weak | moderate | strong | bulletproof)
    language_strategy: string (how the guarantee is framed)

  urgency_architecture:
    urgency_type: string (deadline | scarcity | bonus_expiration |
                          price_increase | capacity_limit | none)
    scarcity_proof: string (what makes the urgency believable)

For Silver specimens:
  Extract value_stack_skeleton.stack_size + pricing_architecture + guarantee_architecture ONLY
  SKIP element-level stack analysis and urgency detail
```

### Phase 3: Technique Catalog Compilation

Aggregate across ALL decomposed specimens:

```
value_stacking_techniques:
  FOR each unique stacking approach:
    - technique_name: string (ascending_value, bonus_cascade,
                              core_plus_support, tool_plus_guidance, etc.)
    - count: integer
    - implementation_pattern: string
    - optimal_stack_size: string
    - gold_example: string
    - best_for_niche: [string]

pricing_techniques:
  FOR each unique pricing/anchoring approach:
    - technique_name: string
    - count: integer
    - implementation_pattern: string
    - gold_example: string
    - comparison_types_used: [string]

guarantee_types:
  FOR each unique guarantee construction:
    - type_name: string
    - count: integer
    - risk_reversal_strength: string
    - implementation_pattern: string
    - gold_example: string
    - best_for: string (product type / price point)

urgency_techniques:
  FOR each unique urgency/scarcity device:
    - technique_name: string
    - count: integer
    - believability_factor: string (what makes it credible)
    - implementation_pattern: string
    - gold_example: string

bonus_strategies:
  FOR each unique bonus positioning approach:
    - strategy_name: string
    - bonus_count_range: string
    - relationship_to_core: string (complementary, amplifying, addressing_objection)
    - specimens_using: [source_file references]
```

### Phase 4: Power Element Extraction

From Gold specimens ONLY:

```
value_stack_items:
  FOR each Gold specimen:
    - verbatim_items: [string] (each item with its description from value_stack)
    - stack_size: integer
    - total_stated_value: string
    - niche: string
    - source_file: string

price_justification_language:
  FOR each Gold specimen with explicit price_justification:
    - verbatim_justification: string
    - comparison_anchors: [string]
    - source_file: string

guarantee_language:
  FOR each Gold specimen with guarantee language:
    - verbatim_guarantee: string
    - guarantee_type: string
    - risk_reversal_strength: string
    - source_file: string

urgency_language:
  FOR each Gold specimen with urgency devices:
    - verbatim_urgency: string
    - urgency_type: string
    - scarcity_proof: string
    - source_file: string
```

## Quality Gates

- [ ] Minimum 3 Gold specimens fully decomposed (stack + pricing + guarantee + urgency)
- [ ] Minimum 5 Silver specimens template-decomposed
- [ ] All value stacks decomposed with item_type and value_framing per item
- [ ] Technique catalog has minimum 2 value stacking techniques cataloged
- [ ] Technique catalog has minimum 2 guarantee types documented
- [ ] Power elements contain minimum 3 verbatim value stack sets from Gold specimens
- [ ] Every power element includes source provenance
- [ ] No raw offer structures passed through undecomposed

---

## Anti-Patterns

| Anti-Pattern | Signal | Fix |
|---|---|---|
| **Stack Without Architecture** | Value stack items listed without item_type or function | Every item needs type + framing + function |
| **Price Without Context** | Price listed without anchoring or justification analysis | Include anchor_strategy + comparison_anchors |
| **Guarantee Without Strength** | Guarantee listed without risk_reversal_strength rating | Rate every guarantee weak/moderate/strong/bulletproof |
| **Urgency Without Proof** | Urgency devices listed without believability analysis | scarcity_proof required for every urgency element |
| **Bonuses Ignored** | Only core product analyzed, bonuses skipped | Bonus strategy is part of offer architecture |
