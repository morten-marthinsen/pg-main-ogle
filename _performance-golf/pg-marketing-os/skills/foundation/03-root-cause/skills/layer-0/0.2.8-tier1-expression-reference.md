## Loading Instructions

### When to Load

This microskill executes during Layer 0, AFTER 0.2.7 (persona voice loader) and BEFORE Layer 1 begins. It loads TIER1 extraction data that will be consumed by microskill 2.8 (Expression Anchoring Score) in Layer 2.

### What to Load

From each TIER1 extraction JSON file in the current project's vertical:

1. **Extract `root_cause_architecture` fields:**
   - `what_audience_thinks` — surface-level false belief
   - `real_root_cause` — the actual root cause as framed in the control
   - `why_nothing_worked` — past failure explanation
   - `reframe_technique` — how the shift is achieved (blame_removal, enemy_reveal, false_solution_expose, etc.)
   - `villain` — who/what is blamed

2. **Extract root-cause-adjacent fields:**
   - `components.lead` — how the control opens (often contains root cause foreshadowing)
   - `rsf_framework.R` (relatable_problem) — how the relatable problem is framed
   - `configuration.drivers` — emotional drivers used

3. **Filter by vertical:**
   - Load ONLY files from `final_merged/[vertical]/` matching the current project vertical
   - If project vertical has fewer than 10 files, supplement with files from `health/` (largest vertical, most transferable patterns)

### How to Index

Create a queryable reference organized by:

```yaml
tier1_expression_reference:
  skill: "03-root-cause"
  vertical: "[project vertical]"
  files_loaded: [count]
  timestamp: "[ISO 8601]"

  patterns:
    externalization_methods:
      - pattern: "[description]"
        frequency: [count across loaded files]
        examples:
          - swipe_id: "[id]"
            expression: "[verbatim root cause framing]"
            reframe_technique: "[technique]"

    villain_types:
      - type: "[villain category]"
        frequency: [count]
        examples:
          - swipe_id: "[id]"
            villain: "[verbatim villain description]"

    three_part_structures:
      - swipe_id: "[id]"
        what_they_think: "[verbatim]"
        real_root_cause: "[verbatim]"
        why_nothing_worked: "[verbatim]"

    naming_patterns:
      - pattern: "[how root causes are named — syndrome, metaphor, simple reframe, etc.]"
        frequency: [count]
        examples:
          - swipe_id: "[id]"
            name: "[the root cause name/label used]"
```

## Selection Priority for This Skill

Root cause expression patterns are most valuable from controls that:

1. **Use named root causes** — controls that give the root cause a memorable name (e.g., "Shallow Sleep Syndrome," "Leaky Gut") over controls that describe it without naming
2. **Show strong externalization** — controls that masterfully remove self-blame and redirect to an external villain
3. **Match the project vertical** — golf controls for golf projects, health for health, etc.
4. **Have high emotional load in the root cause section** — controls where the root cause revelation is the emotional climax, not just a logical step

### Per-Vertical Loading Notes

| Vertical | Priority Patterns | Why |
|----------|------------------|-----|
| **Health** | Named syndromes, villain personification, conspiracy reveal | Health audiences respond to named conditions and hidden enemies |
| **Golf** | Simple reframes, false belief destruction | Golf audiences respond to "you've been doing it wrong" reveals |
| **Finance** | Villain + expose, conspiracy reveal | Finance audiences respond to hidden agendas and insider knowledge |
| **Personal-Dev** | Contrarian truth, false solution expose | Personal-dev audiences respond to paradigm shifts |
| **Technology** | Scientific discovery, false belief destruction | Tech audiences respond to data-driven reveals |

## Output Requirements

**Output file:** `[project]/03-root-cause/layer-0-outputs/0.2.8-tier1-expression-reference.md`

The output file must contain:

1. **Execution Context** — skill, vertical, timestamp, files loaded
2. **Pattern Summary** — top externalization methods, villain types, naming patterns with frequencies
3. **Verbatim Examples** — at least 5 complete three-part structures quoted directly from TIER1 files
4. **Pattern Index** — the full `tier1_expression_reference` YAML block for downstream consumption by microskill 2.8

**Minimum size:** 2KB
