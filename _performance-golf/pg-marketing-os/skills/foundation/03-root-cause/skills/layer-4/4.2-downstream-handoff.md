# 4.2 Downstream Handoff

## Purpose
Package the root cause output for handoff to DOWNSTREAM SKILLS in the copywriting sequence. Root cause feeds directly into Mechanism Skill and Promise Skill—this microskill ensures seamless integration.

---

## The Handoff Principle

**Root Cause → Mechanism → Promise → Big Idea**

Root cause sits early in the skill sequence. Its output constrains and informs:
- **Mechanism Skill**: Must SOLVE the root cause
- **Promise Skill**: Must be achievable once root cause is addressed
- **Big Idea Skill**: Synthesizes all upstream work

---

## Execution Protocol

### Step 1: Package for Mechanism Skill

The Mechanism Skill needs to know:
- What root cause must be solved
- What constraints the root cause creates
- What the mechanism must "do" to address it

```
MECHANISM SKILL HANDOFF:

ROOT CAUSE TO SOLVE:
"[Core root cause statement]"

MECHANISM CONSTRAINTS:
├── Must Address: "[Specific problem mechanism must fix]"
├── Must Explain: "[How mechanism solves root cause]"
├── Must NOT: "[What would contradict root cause]"
└── Match Type: [direct_solve | compensate | enable]

PROOF IMPLICATIONS:
"[What proof strategy from root cause carries forward]"

THREE-PART STRUCTURE (for mechanism to address):
├── They Think: "[Mechanism must NOT reinforce this false belief]"
├── Real Problem: "[Mechanism must directly address this]"
└── Why Failed: "[Mechanism must explain why IT works when others didn't]"
```

---

### Step 2: Package for Promise Skill

The Promise Skill needs to know:
- What becomes possible once root cause is addressed
- What constraints on promise the root cause creates
- What transformation arc the root cause enables

```
PROMISE SKILL HANDOFF:

ROOT CAUSE ADDRESSED:
"[Core root cause statement]"

PROMISE IMPLICATIONS:
├── What Becomes Possible: "[Once root cause fixed, audience can...]"
├── What's Now Achievable: "[Specific outcomes enabled]"
├── Transformation Arc: "[From X (root cause active) to Y (root cause solved)]"
└── Timeline Constraints: "[What root cause says about speed/effort]"

PROMISE CONSTRAINTS:
├── Must Be Achievable If: "[Root cause is addressed]"
├── Must NOT Promise: "[Things root cause doesn't enable]"
└── Realistic Scope: "[What root cause allows us to claim]"

EMOTIONAL HANDOFF:
├── From Root Cause: [Relief | Vindication | Hope]
└── To Promise: [Desire | Possibility | Aspiration]
```

---

### Step 3: Package for Big Idea Skill

The Big Idea Skill will synthesize root cause with mechanism and promise:

```
BIG IDEA SKILL HANDOFF:

ROOT CAUSE CONTRIBUTION:
"[Core root cause statement]"

BIG IDEA ELEMENTS FROM ROOT CAUSE:
├── Disruption Potential: "[How root cause challenges conventional wisdom]"
├── Enemy/Villain: "[If identified, carries to Big Idea]"
├── Unique Mechanism Setup: "[Root cause creates demand for unique solution]"
└── Emotional Foundation: "[Core emotion root cause establishes]"

REFRAME TECHNIQUE:
"[From schema: blame_removal | enemy_reveal | false_solution_expose | etc.]"

NICHE CALIBRATION:
├── Niche: "[Niche name]"
├── Expression Style: "[How root cause was expressed]"
└── Big Idea Compatibility: "[How root cause style affects Big Idea approach]"
```

---

### Step 4: Package Constraint Documentation

Document what root cause CONSTRAINS in downstream work:

```
CONSTRAINT DOCUMENTATION:

FOR MECHANISM:
├── MUST: "[Requirement the mechanism cannot violate]"
├── MUST: "[Another requirement]"
├── MUST NOT: "[What would contradict root cause]"
└── SHOULD: "[Recommended but not required]"

FOR PROMISE:
├── MUST: "[Promise constraint from root cause]"
├── MUST: "[Another constraint]"
├── MUST NOT: "[What root cause doesn't allow]"
└── SHOULD: "[Recommended scope]"

FOR BIG IDEA:
├── MUST: "[How root cause must appear in Big Idea]"
├── MUST: "[Consistency requirement]"
└── SHOULD: "[Recommended integration]"
```

---

### Step 5: Create Dependency Checklist

For downstream skills to verify alignment:

```
DEPENDENCY VERIFICATION CHECKLIST:

FOR MECHANISM SKILL:
□ Mechanism directly addresses root cause: "[Root cause statement]"
□ Mechanism explains why previous solutions failed
□ Mechanism doesn't contradict root cause expression
□ Proof strategy is compatible with root cause proof

FOR PROMISE SKILL:
□ Promise is achievable if root cause is solved
□ Promise doesn't claim more than root cause enables
□ Transformation arc follows from root cause resolution
□ Timeline is consistent with root cause implications

FOR BIG IDEA SKILL:
□ Big Idea incorporates root cause disruption
□ Reframe technique is consistent
□ Emotional through-line is maintained
□ Niche expression style is matched
```
