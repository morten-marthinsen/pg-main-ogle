# 2.5 Dual Problem Framing Expression

## Purpose
Express the derived root cause as TWO interconnected problems that COMPOUND each other. This expression method works when the root cause has multiple dimensions that are both necessary to address—making the solution feel more comprehensive and the problem feel more "complete."

---

## When to Use

**Ideal for**:
- Products that solve multiple issues at once
- Root causes with internal AND external components
- Situations where "partial solutions" have failed
- When mechanism addresses multiple failure points

**Examples from vault**:
- ONE Wedge: "Player's Mental Equation" + "Wedge Design Problem"
- Can work for health: "Hormone imbalance" + "Gut damage"
- Can work for skills: "Mental block" + "Technical flaw"

---

## The Dual Problem Pattern

```
STRUCTURE:
1. Reveal Problem #1 (what the audience knows or suspects)
2. Reveal Problem #2 (what they didn't know compounds #1)
3. Explain how they INTERACT/COMPOUND each other
4. Show why solving only ONE doesn't work
5. Position the solution as addressing BOTH

FORMAT:
"There are actually TWO problems here, and they compound each other.
Problem #1: [first issue]. Problem #2: [second issue].
Here's why this matters: [how they interact].
That's why fixing just [one] never works—you have to address BOTH."
```

---

## Execution Protocol

### Step 1: Identify Two Distinct Problem Dimensions

From derivation, find two separable aspects:

```
PROBLEM DIMENSION ANALYSIS:
Dimension 1: [Problem Type] - [Description]
└── Category: internal | external | mental | physical | technical | systemic

Dimension 2: [Problem Type] - [Description]
└── Category: internal | external | mental | physical | technical | systemic

Interaction: [How they compound each other]
```

**Common Dual Problem Patterns**:

| Pattern | Problem 1 | Problem 2 | Example |
|---------|-----------|-----------|---------|
| **Internal + External** | Your psychology | The system/tools | ONE Wedge |
| **Knowledge + Execution** | What you know | What you can do | Many skills |
| **Root + Symptom Loop** | Underlying cause | Surface issue that worsens cause | Health |
| **Past + Present** | Historical damage | Ongoing exposure | Health/aging |

---

### Step 2: Name Each Problem Distinctly

Give each problem a memorable name:

```
PROBLEM NAMING:
Problem 1: "[NAME]"
├── Type: [equation | syndrome | flaw | trap | etc.]
├── Description: "[What it is in 1 sentence]"
└── Evidence: "[How audience experiences it]"

Problem 2: "[NAME]"
├── Type: [problem | design | paradox | etc.]
├── Description: "[What it is in 1 sentence]"
└── Evidence: "[How audience experiences it]"
```

**ONE Wedge Example**:
```
Problem 1: "The Player's Mental Equation"
├── Type: equation
├── Description: "From 70 yards, your brain tells you to swing soft, so you decelerate"
└── Evidence: "You get nervous, feel tentative, and chunk it"

Problem 2: "The Wedge Design Problem"
├── Type: design problem
├── Description: "Modern wedge grinds require perfect technique you don't have"
└── Evidence: "Even when you try hard, the club doesn't help you"
```

---

### Step 3: Explain the Compounding Effect

Show how the two problems make each other WORSE:

```
COMPOUNDING FORMULA:
"Problem 1 causes [effect 1].
Problem 2 causes [effect 2].
But when you have BOTH:
[Effect 1] + [Effect 2] = [Much worse combined outcome]"

OR:

"You could handle Problem 1 if Problem 2 didn't exist.
You could handle Problem 2 if Problem 1 didn't exist.
But together, they create a situation where [impossible outcome]."
```

**ONE Wedge Example**:
```
"The mental equation makes you decelerate.
The wedge design demands perfect technique.

But when you have BOTH:
You're decelerating INTO a club that requires acceleration...
AND using a grind that punishes your inevitable technique flaws.

You never had a chance. The problems compound each other into
guaranteed inconsistency."
```

---

### Step 4: Show Why Partial Solutions Fail

Explain why addressing only ONE problem doesn't work:

```
PARTIAL SOLUTION FAILURE:
"If you only fix Problem 1: [Why Problem 2 still defeats you]"
"If you only fix Problem 2: [Why Problem 1 still defeats you]"
"That's why [previous partial attempts] didn't work."
```

**Example**:
```
"If you just work on your mental game, you're still fighting
a wedge designed for better players.

If you just get a 'forgiving' wedge, your brain still tells you
to decelerate, and no wedge can fix that.

That's why the tips, lessons, and new clubs haven't solved your
wedge problems—they only addressed HALF the issue."
```

---

### Step 5: Position the Complete Solution

Show the solution addresses BOTH:

```
COMPLETE SOLUTION FRAMING:
"[Solution] is the ONLY approach that addresses BOTH problems:
- It solves Problem 1 by [mechanism]
- It solves Problem 2 by [mechanism]
That's why it works when nothing else has."
```

## Quality Gates

Before considering expression complete:

- [ ] Two problems are genuinely distinct (not same problem restated)
- [ ] Compounding effect is clear and believable
- [ ] Partial solution failure is explained
- [ ] Solution genuinely addresses both
- [ ] Neither problem alone seems "complete enough"

## Anti-Patterns

**Weak Dual Problem** (Avoid):
- Two problems are really the same thing
- Compounding effect not explained
- Solution only addresses one
- Feels like artificial complexity

**Strong Dual Problem** (Target):
- Clearly distinct dimensions
- "Oh, I have BOTH of those" recognition
- Compounding feels inevitable
- Solution is uniquely complete
