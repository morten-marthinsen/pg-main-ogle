# 2.8 Expression Anchoring Score

**Version:** 1.0
**Layer:** 2 (Expression)
**Type:** Scoring + Generation (Data-Grounded Validation)
**Purpose:** Score expression variants against audience research quotes and TIER1 patterns, and generate additional quote-first expression candidates
**Shared Protocol:** `skills/protocols/EXPRESSION-ANCHORING-PROTOCOL.md`
**Position:** After 2.7-expression-synthesis, BEFORE Arena (Layer 2.5)

---

## Purpose

This microskill serves two functions:

1. **SCORING:** Take the 3+ expression variants from 2.7-expression-synthesis and score each against the three anchoring dimensions (Quote Penetration, TIER1 Pattern Match, FSSIT Echo)
2. **GENERATION:** Produce 3+ additional expression variants derived bottom-up from audience quotes (Quote-First Generation)

Both concept-down variants (from 2.1-2.7) and quote-first variants are scored using the same protocol. Only variants scoring 6.5+ proceed to Arena.

---

## When to Use

- **Always.** This microskill is mandatory for all root cause expression generation.
- Executes AFTER all expression methods have produced their variants (2.1-2.7) and BEFORE Arena entry.
- Requires research quotes and TIER1 expression reference to be loaded (Layer 0 outputs).

## Execution Protocol

### Step 1: Load Scoring Inputs

```
1. LOAD expression variants from 2.7-expression-synthesis output
   - Must have ≥3 variants, each with expression text and method type

2. LOAD top 20 research quotes from Pain and Solutions-Tried buckets
   - Select by emotional intensity (strongest frustration, most vivid language)
   - These are the quotes that will be scored against expressions

3. LOAD tier1_expression_reference.json from Layer 0 outputs
   - Contains externalization patterns, villain types, naming patterns from TIER1 vault

4. LOAD FSSIT candidates
   - IF FSSIT-RANKING.md exists: load top 5 FSSIT candidates
   - IF NOT: load top 10 Pain + Hope quotes as proxy
   - NOTE which mode in output
```

### Step 2: Score Each Expression Variant

For each expression variant from 2.7:

```
DIMENSION 1 — QUOTE PENETRATION (40%):
  FOR each of top 20 research quotes:
    Score semantic overlap with this expression (1-10)
    - Does the expression use the audience's vocabulary?
    - Does it capture their framing of the problem?
    - Would the audience recognize their experience in this expression?
  TAKE top 5 overlap scores and average them
  RECORD the 5 quotes with scores as citations

DIMENSION 2 — TIER1 PATTERN MATCH (30%):
  COMPARE expression against TIER1 root cause patterns:
    - Externalization method: Does it match a proven pattern?
    - Villain type: Does it use a validated villain framing?
    - Naming approach: Does it follow a successful naming convention?
    - Three-part structure: Does it mirror proven three-part framings?
  SCORE overall pattern alignment (1-10)
  CITE specific TIER1 sources that match

DIMENSION 3 — FSSIT ECHO (30%):
  FOR each of top 5 FSSIT candidates (or proxy quotes):
    Score whether this expression would trigger "finally someone said it" (1-10)
    - Does it crystallize an unspoken audience feeling?
    - Does it name something they've felt but couldn't articulate?
  TAKE highest single FSSIT score
  CITE the FSSIT candidate that scored highest

COMPOSITE = (D1 × 0.40) + (D2 × 0.30) + (D3 × 0.30)
```

### Step 3: Quote-First Generation

```
1. EXTRACT the 10 most emotionally loaded phrases from Pain and Solutions-Tried quotes
   - Focus on phrases carrying frustration, anger, helplessness, or revelation
   - Focus on phrases where the audience is already NAMING their problem

2. FOR each phrase cluster:
   GENERATE 1 root cause expression that:
   - Uses the audience's actual vocabulary (or close variants)
   - Reorganizes their language into a three-part root cause framing
   - Applies the expression method best suited to this language pattern

3. PRODUCE at least 3 quote-first expression variants

4. SCORE all quote-first variants using the same 3-dimension protocol
```

### Step 4: Consolidate and Gate

```
1. MERGE concept-down variants (from 2.1-2.7) with quote-first variants
2. SORT all variants by composite score (descending)
3. REMOVE variants scoring below 5.0 (REJECTED)
4. FLAG variants scoring 5.0-6.4 with revision guidance

GATE CHECK:
  - At least 3 variants must score 6.5+ to proceed to Arena
  - IF fewer than 3 score 6.5+:
    - Check if any flagged variants can be revised
    - Generate 2 additional quote-first candidates targeting weak dimensions
    - RE-SCORE
    - IF still fewer than 3 at 6.5+: FLAG for human review before Arena
```

## Quality Gates

Before considering 2.8 complete:

- [ ] All concept-down variants from 2.7 scored across all 3 dimensions
- [ ] At least 3 quote-first variants generated and scored
- [ ] At least 3 total variants score 6.5+ (composite)
- [ ] All variants below 5.0 removed from candidate pool
- [ ] Flagged variants (5.0-6.4) have specific revision guidance
- [ ] Top 5 quote citations included for each scored variant
- [ ] TIER1 source swipe_ids cited for pattern matches
- [ ] Output file meets 3KB minimum

---

## Handoff

Scored expression variants proceed to **Arena Layer (2.5)**. The Arena receives:
- All variants scoring 6.5+ (as Arena competitors' starting material)
- Anchoring scores as context for Arena judging (adds "Research Grounding" as awareness criterion)
- Quote-first variants labeled as such (Arena can evaluate concept-down vs quote-first quality)

---

## Anti-Patterns

| Weak | Strong |
|------|--------|
| Scoring from memory without loading actual quotes | Loading verbatim quotes from research outputs and scoring against each |
| Rating all variants 7+ without differentiation | Genuine scoring with score spread reflecting real differences |
| Generating quote-first variants that are just reworded concept-down variants | Quote-first variants that start from audience language and work up |
| Citing TIER1 patterns without checking actual TIER1 files | Citing specific swipe_ids from loaded TIER1 reference |
