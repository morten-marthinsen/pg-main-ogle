# 1.7 Derivation Synthesis

## Purpose
Synthesize all derivation work into a single, validated ROOT CAUSE ready for expression. This is the final Layer 1 output—the raw root cause that will be shaped into copy in Layer 2.

---

## Input Requirements

From all previous Layer 1 microskills:
- 1.1 Research Pattern Analysis → Research Pattern Report
- 1.2 Symptom Convergence → Convergence Analysis Report (if applicable)
- 1.3 False Belief Identification → False Belief Analysis Report (if applicable)
- 1.4 Hidden Layer Discovery → Hidden Layer Discovery Report (if applicable)
- 1.5 Mechanism Constraint Check → Mechanism Constraint Report
- 1.6 Proof Constraint Check → Proof Constraint Report

---

## Execution Protocol

List every root cause candidate from derivation:


---

### Step 2: Score Candidates on Four Dimensions

Rate each candidate:

| Dimension | Weight | Description |
|-----------|--------|-------------|
| **Truth** | 30% | Is this actually the real root cause? |
| **Revelation** | 25% | Will audience find this surprising/new? |
| **Mechanism Fit** | 25% | Can our mechanism solve it? |
| **Provability** | 20% | Can we demonstrate it credibly? |

```
SCORING MATRIX:
                    Truth  Revelation  Mech Fit  Provable  WEIGHTED
Convergence RC:     [1-10]    [1-10]    [1-10]    [1-10]   [Calc]
False Belief RC:    [1-10]    [1-10]    [1-10]    [1-10]   [Calc]
Hidden Layer RC:    [1-10]    [1-10]    [1-10]    [1-10]   [Calc]
```

**Calculation**: (Truth × 0.3) + (Revelation × 0.25) + (Mech Fit × 0.25) + (Provable × 0.2)

---

### Step 3: Select Primary Root Cause

Choose the highest-scoring candidate as PRIMARY. Consider combining if appropriate:

**Selection Criteria**:
1. Highest weighted score wins (if clear leader)
2. If tied, prefer the one with highest Truth score
3. If multiple root causes are compatible, consider combining (like ONE Wedge's "dual problem")

```
PRIMARY ROOT CAUSE SELECTION:
Winner: "[Root cause statement]"
Score: [X.X]
Why Selected: "[Explanation]"

Runner-up: "[Root cause statement]"
Score: [X.X]
Why Not Primary: "[Explanation]"

Combination Considered?: YES/NO
If yes, combined version: "[Combined root cause]"
```

---

### Step 4: Formulate the Three-Part Root Cause Structure

Every derived root cause should have three components:

```
THE THREE-PART ROOT CAUSE:

1. WHAT THEY THINK THE PROBLEM IS:
   "[The surface problem / false attribution]"

2. WHAT THE REAL PROBLEM IS:
   "[The derived root cause]"

3. WHY NOTHING HAS WORKED:
   "[How root cause explains all failures]"
```

**Vault Examples**:

| Swipe | Think | Real | Why Nothing Worked |
|-------|-------|------|-------------------|
| pg_ssts_vsl_001 | "Need to hit ball first, work on mechanics" | "Misconception that ball is at bottom of swing" | "Trying to hit ball first without understanding low point mechanics" |
| pg_fixd_vsl_001 | "Multiple swing flaws (grip, face, path, extension)" | "One root flaw: off-track first move in backswing" | "Fixing symptoms—as soon as you fix one, another appears" |
| pg_sf1_vsl_001 | "Need more practice, better swing" | "Technique breaks down under pressure on course" | "Practice doesn't transfer—you fall back to bad habits when it matters" |

---

### Step 5: Validate the Final Root Cause

Run final validation checks:

```
FINAL VALIDATION:
Root Cause: "[Statement]"

□ RESEARCH GROUNDED
  └── Directly derived from audience research? YES/NO

□ MECHANISM MATCHED
  └── Product/method can solve this? YES/NO

□ PROOF AVAILABLE
  └── Can demonstrate credibly? YES/NO

□ REVELATION POTENTIAL
  └── Audience will find this new? YES/NO

□ SIMPLE TO EXPLAIN
  └── Can state in 1-2 sentences? YES/NO

□ BLAME REMOVED
  └── Doesn't blame the audience? YES/NO

□ HOPE CREATED
  └── Points to solution? YES/NO

VALIDATION STATUS: APPROVED / NEEDS REVISION
```

---

### Step 6: Document Supporting Elements

Capture context needed for Layer 2 expression:

```
SUPPORTING ELEMENTS:

Strongest Proof:
└── "[The most compelling evidence for this root cause]"

Key Analogy Opportunity:
└── "[Natural analogy that could explain this]"

Enemy/Villain Potential:
└── "[If applicable, what/who can be blamed]"

Niche Calibration:
└── Niche: [health | golf | finance | etc.]
└── Expression tendency: [syndrome_naming | simple_reframe | villain | etc.]

Emotional Core:
└── "[The feeling this root cause should evoke]"
└── Primary: [relief | vindication | hope | anger | curiosity]
```

- [ ] Supporting elements documented for expression layer
- [ ] Confidence score is 7.0+ (if below, revisit derivation)
