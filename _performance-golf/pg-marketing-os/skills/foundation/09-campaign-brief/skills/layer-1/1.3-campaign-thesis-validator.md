# Layer 1.3 — Campaign Thesis Validator

**Skill:** 09 — Campaign Brief
**Microskill:** 1.3 — Campaign Thesis Validator
**Layer:** 1
**Model:** opus
**Purpose:** Validate that the campaign thesis (from 08-structure) is supported by all upstream elements — provable, logically sound, differentiated, and not contradicted by any strategic element.

---

## Input

| Source | File | Required |
|--------|------|----------|
| 1.1-campaign-summary-generator | `1.1-campaign-summary.json` (thesis + element summaries) | Yes |
| 1.2-element-relationship-mapper | `1.2-element-relationships.json` | Yes |
| 0.1-upstream-loader | `0.1-upstream-context.json` (structure.campaign_thesis) | Yes |
| 0.3-teachings-loader | `0.3-teaching-index.json` | Yes |

## Output

| File | Format | Min Size |
|------|--------|----------|
| `1.3-thesis-validation.json` | JSON | 2KB |

## Execution Steps

### Step 1: Extract the Campaign Thesis

Pull the campaign thesis from structure-package.json (via upstream context):
- `structure.campaign_thesis` — the single overarching claim
- Cross-reference with `1.1-campaign-summary.json` `one_sentence_thesis` for consistency

If the two versions differ, flag the discrepancy and use the structure-package version as authoritative.

### Step 2: Validate Thesis Against 4 Criteria

**Criterion 1: Every strategic element SUPPORTS the thesis (nothing contradicts)**

Walk through each upstream element and verify:
- Root cause: Does the root cause discovery make the thesis necessary?
- Mechanism: Does the mechanism make the thesis achievable?
- Promise: Is the promise the thesis delivered?
- Big Idea: Does the Big Idea frame the thesis compellingly?
- Villain: Does the villain explain why the thesis wasn't realized before?
- Offer: Does the offer enable the prospect to act on the thesis?
- Proof: Does the proof inventory substantiate the thesis?
- Structure: Does the argument structure prove the thesis?

For each element, produce:
```yaml
element_alignment:
  element: string
  supports_thesis: boolean
  alignment_description: string  # HOW it supports (or contradicts)
  contradiction_flag: boolean
  contradiction_detail: string  # if contradiction found
```

**Criterion 2: The thesis is PROVABLE given the proof inventory**

Assess:
- Can the thesis be proven through evidence and education (not mere assertion)?
- Does the proof inventory contain sufficient proof types and strength to support the thesis?
- Is there a gap between what the thesis claims and what the proof can demonstrate?
- Provability score: 1-10

**Criterion 3: The thesis FOLLOWS from the root cause + mechanism logic**

Verify the logical chain:
- Root cause establishes the problem
- Mechanism addresses the root cause
- Thesis articulates why this matters / what this means for the prospect
- The chain is logically sound — no leaps, no gaps, no non-sequiturs
- Logical soundness score: 1-10

**Criterion 4: The thesis is DIFFERENTIATED given the competitive landscape**

Check:
- No major competitor is making the same claim with the same mechanism
- The thesis carves out a unique position in the prospect's mind
- The Big Idea angle reinforces the differentiation
- Differentiation score: 1-10

### Step 3: Calculate Overall Thesis Validity

```yaml
thesis_validation:
  thesis_statement: string
  element_alignments: [object]  # from criterion 1
  contradiction_count: integer
  contradicting_elements: [string]

  provability:
    score: integer  # 1-10
    assessment: string
    proof_gaps: [string]

  logical_soundness:
    score: integer  # 1-10
    assessment: string
    logical_gaps: [string]

  differentiation:
    score: integer  # 1-10
    assessment: string
    competitive_overlap: [string]

  overall_status: enum[PASS, FAIL]
  overall_score: float  # average of 3 scores
  failure_reasons: [string]  # empty if PASS
  improvement_suggestions: [string]
```

### Step 4: Gate Decision

```
IF contradiction_count > 0:
  overall_status = FAIL
  LOG: "Thesis contradicted by [elements] — cannot proceed with incoherent thesis"

IF provability.score < 5:
  overall_status = FAIL
  LOG: "Thesis not provable — proof inventory insufficient for this claim"

IF logical_soundness.score < 5:
  overall_status = FAIL
  LOG: "Thesis has logical gaps — root cause -> mechanism -> thesis chain broken"

IF overall_score < 6.0:
  overall_status = FAIL

ELSE:
  overall_status = PASS
```

## Quality Gate

- [ ] Campaign thesis extracted from structure-package.json
- [ ] All 8 upstream elements checked for alignment with thesis
- [ ] Any contradictions flagged with specific detail
- [ ] Provability scored 1-10 with assessment and proof gap identification
- [ ] Logical soundness scored 1-10 with assessment and gap identification
- [ ] Differentiation scored 1-10 with competitive overlap analysis
- [ ] Overall status is PASS or FAIL only (no invented statuses)
- [ ] FAIL if any contradictions found
- [ ] FAIL if overall score < 6.0
- [ ] Improvement suggestions provided for any score below 7

---

## Anti-Patterns

- **Rubber-stamp validation:** Marking everything as "supports" without evidence — REJECT
- **Ignoring contradictions:** Downplaying a genuine element-thesis misalignment — the whole point of this microskill is to catch these
- **Overriding thesis:** Modifying the thesis to fit the elements — this skill validates, it does NOT modify upstream strategy
