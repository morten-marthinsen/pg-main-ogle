# Skill 1.5-C: Quote Classifier

**Version:** 4.0 (RSF-Enhanced)
**Layer:** 1 (Infrastructure)
**Type:** Data Classification
**Dependencies:** raw_quotes.json, market_config.yaml
**Output:** classified_quotes.json

---

## RSF ENHANCEMENT (v4.0)

This version adds temporal recency tagging to support downstream RSF analysis. Temporal data enables emotional momentum tracking in Skill 2.8-B (Latent Resonance Identifier) and source diversity metrics in Layer 1 Validator.

**New capabilities:**
- Temporal recency classification for each quote
- Source diversity tracking in metadata
- Enhanced output schema with RSF-ready fields

---

## Purpose

Classify each raw quote into the 6 primary buckets defined in PRD v3.0 (Pain, Hope, Root Cause, Solutions Tried, Competitor Mechanism, Villains) and secondary dimensions (aspect, sub-topic, emotional intensity). This creates the structured quote database for Layer 2 analysis.

**PRD Reference:** Section 3.7 - Top 25 Requirement for ALL 6 Buckets

---

## Batch Processing Configuration

```
CHUNKING STRATEGY (Audit Fix FAIL-001):

Large dataset classification requires batching to prevent context exhaustion.

BATCH PARAMETERS:
  batch_size: 100  # Quotes per classification batch
  max_context_quotes: 500  # Maximum quotes in active context
  checkpoint_frequency: 100  # Save progress every N quotes

PROCESSING FLOW:
  1. Load raw_quotes.json metadata (get total count)
  2. Process quotes in batches of 100
  3. For each batch:
     - Classify all quotes in batch
     - Write classified batch to intermediate file
     - Update running bucket counts
     - Log checkpoint: "Batch N complete: X quotes classified"
  4. Merge all batch files into final classified_quotes.json

STREAMING APPROACH:
  - Read quotes in chunks
  - Write results as processed
  - Maintain running statistics only

RESUME CAPABILITY:
  - Save classification_checkpoint.json after each batch
  - Track last_processed_quote_id
  - On resume, continue from checkpoint
```

## Classification Framework

```
PRIMARY BUCKETS (mutually exclusive) - PRD v3.0 Section 3.7:

1. PAIN (P-XXX) - Current problems, frustrations, struggles
   Minimum: 300 quotes | Top 25 required

2. HOPE (H-XXX) - Desires, aspirations, wished-for outcomes
   Minimum: 250 quotes | Top 25 required

3. ROOT_CAUSE (RC-XXX) - Why problems happen, underlying issues
   Minimum: 200 quotes | Top 25 required

4. SOLUTIONS_TRIED (ST-XXX) - Experiences with products/methods
   Minimum: 150 quotes | Top 25 required

5. COMPETITOR_MECHANISM (CM-XXX) - How competitors claim their solutions work
   Minimum: 100 quotes | Top 25 required

6. VILLAINS (V-XXX) - What prospects hate, blame, or resent
   Minimum: 75 quotes | Top 25 required

TOTAL MINIMUM: 1,075 quotes across all buckets

SECONDARY DIMENSIONS (multiple per quote):

- Aspect (from market_config.aspects)
- Sub-topic (specific issue within aspect)
- Emotional intensity (1-10)
- Specificity level (1-5)
- Awareness level (problem/solution/product aware)
```

## Classification Logic

### Step 1: Primary Bucket Classification

```
FOR each quote:

  ANALYZE text content:

  PAIN INDICATORS (weight: high):
  - Negative emotions: frustrated, angry, disappointed, struggling
  - Problem language: can't, won't, doesn't work, failing
  - Temporal struggle: "for years", "keep trying", "still"
  - Loss language: lost, wasted, down

  HOPE INDICATORS (weight: high):
  - Desire language: want, wish, hope, dream
  - Aspiration: finally, one day, goal, if only
  - Positive future: imagine, picture, would love
  - Relief seeking: just want to, need to

  ROOT_CAUSE INDICATORS (weight: high):
  - Explanation: because, reason, caused by
  - Attribution: "the problem is", "it's because"
  - Understanding: realized, figured out, understood
  - Pattern recognition: every time, whenever

  SOLUTIONS_TRIED INDICATORS (weight: high):
  - Past experience: tried, used, bought, tested
  - Product mentions: specific names, brands
  - Comparison: better than, worse than, switched
  - Review language: works, doesn't work, helped, didn't help

  COMPETITOR_MECHANISM INDICATORS (weight: high):
  - How competitor products claim to work
  - Explanation of competitor methodology
  - "Their approach is...", "It works by..."
  - Technical descriptions of competitor solutions
  - Marketing claims about HOW something works
  - Keywords: method, system, approach, technique, process, proprietary

  VILLAINS INDICATORS (weight: high):
  - Strong negative sentiment toward entities
  - Blame attribution: "It's [entity]'s fault"
  - Resentment language: hate, despise, can't stand
  - Market villains: gurus, scammers, industry practices
  - Systemic villains: algorithms, market makers, institutions
  - Self as villain: self-blame, "I'm my own worst enemy"
  - Keywords: hate, blame, fault, scam, rip-off, predatory, manipulative

  CALCULATE bucket_confidence:
  - Strong single-bucket match: 0.9+
  - Clear match with minor overlap: 0.7-0.9
  - Ambiguous (could be multiple): 0.5-0.7

  ASSIGN primary bucket (highest score)
```

### Step 2: Handle Ambiguous Quotes

```
IF bucket_confidence < 0.7:

  APPLY tiebreaker rules:

  1. First-person present tense → PAIN (current state)
  2. First-person past tense with lesson → ROOT_CAUSE
  3. "I tried X" without outcome → SOLUTIONS_TRIED
  4. "I tried X and it [worked/didn't]" → SOLUTIONS_TRIED
  5. "I want/wish/hope" → HOPE
  6. "I realized/understood" → ROOT_CAUSE
  7. "I'm struggling with" → PAIN
  8. "Their method/system works by..." → COMPETITOR_MECHANISM
  9. Describes HOW a competitor solution functions → COMPETITOR_MECHANISM
  10. "I hate/blame/can't stand [entity]" → VILLAINS
  11. Strong resentment toward market players → VILLAINS
  12. Self-blame patterns → VILLAINS (subcategory: self)

  IF still ambiguous:
  - Assign most likely bucket
  - Flag for human review
  - Lower confidence score
```

### Step 3: Aspect Classification

```
FROM market_config.aspects:

FOR each quote:

  MATCH to aspect based on:
  - Keyword presence (from aspect.example_terms)
  - Topic alignment (from aspect.definition)
  - Context indicators

  EXAMPLE (Trading market):
  - Mentions strategy, setup, entries → TECHNICAL
  - Mentions emotions, mindset, psychology → PSYCHOLOGICAL
  - Mentions money, losses, profits → FINANCIAL
  - Mentions time, lifestyle, balance → LIFESTYLE
  - Mentions comparison, embarrassment → SOCIAL

  ASSIGN primary aspect
  Note secondary aspects if applicable
```

### Step 4: Sub-Topic Assignment

```
WITHIN each aspect:

IDENTIFY specific sub-topic:

Example sub-topics (Trading/PSYCHOLOGICAL):
- Revenge trading
- Fear of missing out
- Overconfidence
- Analysis paralysis
- Emotional control
- Discipline issues

ASSIGN most specific applicable sub-topic
```

### Step 5: Awareness Level

```
CLASSIFY awareness level:

UNAWARE:
- Doesn't recognize they have a problem
- Rare in self-reported content
- Usually observed, not stated

PROBLEM-AWARE:
- Knows they have a problem
- Doesn't know the solution exists
- Keywords: "I'm struggling", "don't know what to do"

SOLUTION-AWARE:
- Knows solutions exist
- Evaluating options
- Keywords: "looking for", "which is better", "comparing"

PRODUCT-AWARE:
- Knows specific products
- Mentions by name
- Keywords: specific product names, "I tried [product]"
```

### Step 6: Intensity Metrics

```
CALCULATE emotional_intensity (1-10):

Factors:
- Extreme language (+2): never, always, completely
- Amplifiers (+1): very, really, so
- Exclamation marks (+1)
- Personal stakes (+2): money lost, time wasted
- Repetition (+1)
- ALL CAPS (+1)

Base: 5, adjust up/down

CALCULATE specificity_level (1-5):

1 - Vague: "It's frustrating"
2 - Somewhat specific: "I keep losing money"
3 - Specific: "I've lost $5000"
4 - Very specific: "I lost $5000 in 3 months on options"
5 - Highly specific: "I lost $5,247 between March and May trading SPY options, mostly on Fridays"

CALCULATE actionability (1-5):

1 - Pure emotion, no action
2 - Problem stated, no action
3 - Problem + attempted action
4 - Problem + specific action + result
5 - Clear insight that implies solution
```

## RSF v4.0: Temporal Recency Classification

```
PURPOSE:
Temporal tagging enables downstream RSF skills to:
1. Weight recent quotes more heavily in emotional analysis
2. Track emotional momentum (increasing/decreasing/stable patterns)
3. Identify temporally relevant vs. evergreen emotional territories
4. Flag stale data that may not reflect current audience sentiment

TEMPORAL CLASSIFICATION LOGIC:

FOR each quote:

  STEP 1: Determine date_posted (original content date)

  SOURCES for date determination:
  - Reddit: post/comment timestamp from API/scrape
  - YouTube: video upload date, comment date
  - Forums: post date, thread date
  - Reviews: review date
  - Social media: post timestamp
  - Ads/funnels: Capture date (assume current)

  IF date_posted determinable:
    RECORD date_posted in ISO format (YYYY-MM-DD)
  ELSE:
    SET date_posted = null
    FLAG as "date_unknown"

  STEP 2: Calculate temporal_recency

  days_old = current_date - date_posted

  IF days_old <= 30:
    temporal_recency = "current"
  ELSE IF days_old <= 90:
    temporal_recency = "recent"
  ELSE IF days_old <= 180:
    temporal_recency = "aging"
  ELSE IF days_old > 180:
    temporal_recency = "old"
  ELSE:
    temporal_recency = "unknown"

  STEP 3: Record date_captured

  date_captured = date when quote was scraped/collected
  (This is always known from the research execution date)

TEMPORAL WEIGHTING GUIDANCE (for downstream skills):
- current (0-30 days): Weight 1.5x in emotional analysis
- recent (31-90 days): Weight 1.25x
- aging (91-180 days): Weight 1.0x (baseline)
- old (180+ days): Weight 0.75x
- unknown: Weight 0.75x (conservative assumption)

NOTE: Weighting is applied in Layer 2.8-B, not in this skill.
This skill only TAGS; it does not weight.
```

## RSF v4.0: Source Diversity Tracking

```
PURPOSE:
Source diversity metrics enable:
1. Identification of over-reliance on single sources
2. Validation that temporal distribution is representative
3. Flagging of potential bias from homogeneous data

DIVERSITY CALCULATION:

AFTER all quotes classified:

  STEP 1: Count unique sources
  unique_sources = COUNT DISTINCT source_url

  STEP 2: Calculate source distribution
  FOR each unique source:
    count = quotes from this source
    percentage = count / total_quotes * 100

  STEP 3: Calculate temporal distribution
  FOR each temporal_recency category:
    count = quotes with this temporal_recency
    percentage = count / total_quotes * 100

  STEP 4: Generate diversity flags

  FLAGS (add to metadata.source_diversity.flags):

  IF any single source > 40% of total:
    FLAG: "High concentration: [source] provides [X]% of quotes — may skew results"

  IF unique_sources < 5:
    FLAG: "Low source diversity: Only [N] unique sources — recommend expanding"

  IF temporal_distribution.current < 20%:
    FLAG: "Low recent data: Only [X]% quotes from last 30 days — temporal trends may be incomplete"

  IF temporal_distribution.unknown > 30%:
    FLAG: "High date uncertainty: [X]% quotes have unknown dates — temporal analysis limited"

  IF any bucket has < 3 unique sources:
    FLAG: "[Bucket] bucket sourced from only [N] sources — validate representativeness"

VALIDATION THRESHOLDS (soft warnings, not hard blocks):
- Minimum 5 unique sources (WARNING if fewer)
- No single source > 40% of total (WARNING if exceeded)
- Minimum 20% quotes in "current" bucket (WARNING if fewer)
- Maximum 30% quotes in "unknown" temporal bucket (WARNING if exceeded)
```
