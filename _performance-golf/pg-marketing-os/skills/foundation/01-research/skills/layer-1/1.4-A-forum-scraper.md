# Skill 1.4-A: Forum Scraper

**Version:** 2.0
**Layer:** 1 (Infrastructure)
**Type:** Content Extraction
**Dependencies:** validated_sources.json, market_config.yaml
**Output:** Appends to master_raw_content.json

---

## Purpose

Scrape forum-based sources (excluding Reddit, which has its own scraper). Extract threads, replies, and discussions from specialized forums, communities, and Q&A sites.

---

## Target Platforms

```
FORUM TYPES:

1. Specialized forums (e.g., Elite Trader, StackExchange, niche hobbyist forums)
2. Community sites (e.g., Discourse-based communities)
3. Q&A platforms (e.g., Quora, specialized Q&A)
4. Discussion boards (e.g., vBulletin, phpBB forums)
5. Professional communities (e.g., industry-specific)
```

---

## Input Schema

```yaml
input:
  validated_sources: validated_sources.json
  market_config: market_config.yaml
  filter:
    content_type: "forum"
    exclude_platforms: ["Reddit"]  # Handled by Reddit scraper
```

## Execution Logic

### Step 1: Filter Sources

```
FROM validated_sources.json:

SELECT sources WHERE:
- content_type = "forum" OR "thread" OR "discussion"
- platform NOT IN ["Reddit", "YouTube"]
- priority_tier IN ["high", "medium"]

SORT by priority_tier DESC, composite_score DESC

RESULT: forum_sources[]
```

### Step 2: Tool Selection

```
PRIMARY TOOL: Firecrawl

mcp__firecrawl-mcp__firecrawl_scrape:
  url: [source_url]
  formats: ["markdown"]
  onlyMainContent: true
  waitFor: 2000  # Allow JS rendering

FALLBACK: Perplexity (for blocked sites)

mcp__perplexity__perplexity_search:
  query: "site:[domain] [thread_title]"
  max_results: 5
```

### Step 3: Scrape Each Source

```
FOR each source in forum_sources:

  TRY primary tool (Firecrawl):
    EXECUTE scrape
    EXTRACT:
      - Thread title and OP
      - All replies
      - Author names
      - Dates
      - Engagement metrics

    IF success:
      PARSE content into structured format
      EXTRACT preliminary quotes
      LOG success

  IF primary fails:
    LOG failure reason
    TRY fallback tool
    IF fallback succeeds:
      EXTRACT available content
      LOG partial success
    ELSE:
      LOG complete failure
      CONTINUE to next source

  APPEND to forum_content
```

### Step 4: Quote Pre-Extraction

```
FOR each scraped thread:

  SCAN content for quote-worthy passages:

  INDICATORS:
  - First-person statements ("I", "my", "we")
  - Emotional language (frustration, hope, fear)
  - Problem descriptions
  - Solution attempts
  - Specific experiences
  - Numbers and specifics

  FOR each potential quote:
    EXTRACT:
    - Raw text (preserve exact wording)
    - Author attribution
    - Context (what they're responding to)
    - Emotional indicators detected

  ADD to extracted_quotes[]
```

### Step 5: Pagination Handling

```
IF forum has pagination:

  DETECT pagination pattern:
  - ?page=N
  - /page/N
  - Load more button (requires actions)

  FOR pages 1 through min(5, total_pages):
    SCRAPE page
    APPEND content

  LOG pages_scraped count
```

---

## Tool Resilience

```
RESILIENCE PROTOCOL:

1. Primary: Firecrawl scrape
   - Timeout: 30 seconds
   - Retry: 1 time on timeout

2. If blocked/403:
   - Try with mobile: true
   - Try with proxy: "stealth"

3. If still failing:
   - Switch to Perplexity search
   - Accept partial content

4. If complete failure:
   - Log with full error details
   - Mark source as "failed"
   - DO NOT halt - continue to next source

5. Acceptable failure rate: < 20%
   - If > 20% failing, pause and assess
```

---

## Content Cleaning

```
CLEAN extracted content:

1. Remove navigation elements
2. Remove ads and sidebars
3. Remove duplicate posts (quotes of quotes)
4. Preserve quote formatting
5. Maintain attribution
6. Keep timestamps

OUTPUT: Clean, structured thread content
```

---

## Quality Criteria

1. **Completeness** - Full threads with all replies
2. **Attribution** - Author preserved for each post
3. **Structure** - Clear thread → reply hierarchy
4. **Cleanliness** - No navigation/ad noise

---

## Rate Limiting

```
RATE LIMITS:

- Pause 2-3 seconds between requests to same domain
- Maximum 10 concurrent scrapes across domains
- Respect robots.txt delays where specified
- If rate limited, exponential backoff (5s, 10s, 20s)
```
