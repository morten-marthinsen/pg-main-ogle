# Skill 1.1-A: Platform Identifier

**Version:** 2.0
**Layer:** 1 (Infrastructure)
**Type:** Source Discovery
**Dependencies:** market_config.yaml, context_expansion.json, awareness_baseline.json
**Output:** platform_list.json

---

## Purpose

Finalize the list of platforms to scrape based on context expansion topics and awareness distribution. Validates platform accessibility and assigns appropriate tools.

---

## Input Schema

```yaml
input:
  market_config: market_config.yaml (platforms section)
  context_expansion: context_expansion.json (source_type_calibration)
  awareness_baseline: awareness_baseline.json (source_priorities)
```

---

## Output Schema

```yaml
output:
  platform_list:
    platforms:
      - name: string
        type: string
        url: string
        tool: string (firecrawl/apify/perplexity)
        priority: string (critical/high/moderate)
        topics_served: array
        awareness_levels: array
        estimated_quote_volume: string
        specific_targets: array
        fallback_tool: string
        notes: string

    tool_distribution:
      firecrawl: array
      apify: array
      perplexity: array

    coverage_analysis:
      topics_covered: object
      awareness_levels_covered: object
      gaps_identified: array

    validation:
      total_platforms: number
      critical_count: number
      high_count: number
      all_topics_covered: boolean
```

---

## Execution Logic

### Step 1: Compile Platform Sources

```
COMPILATION:

1. Pull tier_1, tier_2, tier_3 from market_config.platforms
2. Pull source_type_calibration from context_expansion
3. Pull source_priorities from awareness_baseline
4. Merge into unified platform list
5. Remove duplicates
```

### Step 2: Validate Platform Accessibility

```
FOR each platform:
  1. Verify URL is valid
  2. Check if platform is in blocked list
  3. If blocked, ensure alternative tool is specified
  4. Note any access concerns
```

### Step 3: Assign Tools

```
TOOL ASSIGNMENT RULES:

Reddit → Apify ONLY (fatihtahta/reddit-scraper-search-fast)
YouTube → Apify (3-step protocol)
Instagram/TikTok/Facebook → Apify ONLY
Forums → Firecrawl primary, Apify fallback
Review Sites → Firecrawl primary
Competitor Pages → Firecrawl primary
Q&A Sites → Firecrawl primary

For each platform:
- primary_tool: Best tool for this platform
- fallback_tool: Backup if primary fails
```

### Step 4: Map Topics to Platforms

```
FOR each research topic from context_expansion:
  - Identify which platforms likely contain this content
  - Assign topic to platform(s)
  - Ensure no topic is orphaned (no platform assigned)
```

### Step 5: Map Awareness Levels to Platforms

```
FOR each awareness level from awareness_baseline:
  - Identify where this level congregates
  - Assign awareness_levels to platforms

Example:
  - Level 5: Review sites, comparison pages
  - Level 4: Forums, YouTube reviews
  - Level 3: Educational content, beginner forums
  - Level 2: Support communities, frustration threads
  - Level 1: General communities, aspirational content
```

### Step 6: Analyze Coverage

```
COVERAGE ANALYSIS:

1. Count topics per platform
2. Count awareness levels per platform
3. Identify any gaps:
   - Topics with no platform
   - Awareness levels with weak coverage

IF gaps found:
  - Add additional platforms to fill gaps
  - Document why gaps exist if unfillable
```

### Step 7: Validate and Output

```
VALIDATION:

- [ ] Total platforms >= 8
- [ ] Critical platforms >= 3
- [ ] All research topics have at least 1 platform
- [ ] All awareness levels have coverage
- [ ] No platform without tool assignment
- [ ] Blocked platforms have alternatives

Save to: platform_list.json
```

---

## Quality Criteria

1. **Comprehensive** - All topics and levels covered
2. **Tool-Appropriate** - Right tool for right platform
3. **Fallback-Ready** - All critical platforms have fallbacks
4. **Priority-Clear** - Critical vs high vs moderate is justified
