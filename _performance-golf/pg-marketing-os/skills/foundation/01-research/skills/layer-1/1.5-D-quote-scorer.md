# Skill 1.5-D: Quote Scorer

**Version:** 3.0
**Layer:** 1 (Infrastructure)
**Type:** Quality Scoring
**Dependencies:** classified_quotes.json, market_config.yaml
**Output:** scored_quotes.json

---

## Purpose

Score each classified quote for quality, usability, and impact potential. This creates a ranked quote database where the best quotes surface to the top for Layer 2 analysis.

**PRD Reference:** Section 3.7 - Top 25 Requirement for ALL 6 Buckets

**MANDATORY:** Extract Top 25 quotes for EACH of the 6 buckets:
- Pain (P-XXX)
- Hope (H-XXX)
- Root Cause (RC-XXX)
- Solutions Tried (ST-XXX)
- Competitor Mechanism (CM-XXX)
- Villains (V-XXX)

---

## Batch Processing Configuration

```
CHUNKING STRATEGY (Audit Fix FAIL-001):

Large dataset scoring requires batching to prevent context exhaustion.

BATCH PARAMETERS:
  batch_size: 100  # Quotes per scoring batch
  max_context_quotes: 500  # Maximum quotes in active context
  checkpoint_frequency: 100  # Save progress every N quotes

PROCESSING FLOW:
  1. Load classified_quotes.json metadata (get total count)
  2. Process quotes in batches of 100
  3. For each batch:
     - Score all quotes in batch
     - Track score distribution statistics
     - Write scored batch to intermediate file
     - Log checkpoint: "Batch N complete: X quotes scored"
  4. Merge all batch files
  5. Calculate final statistics across all batches
  6. Extract top quotes by bucket (requires second pass)

TOP QUOTE EXTRACTION (PRD v3.0 Section 3.7):
  - First pass: score all quotes, track top 50 per bucket (ALL 6 BUCKETS)
  - Second pass: Compare tracked tops, extract final top 25 per bucket
  - MANDATORY: Extract Top 25 for Pain, Hope, Root Cause, Solutions Tried, Competitor Mechanism, Villains

RESUME CAPABILITY:
  - Save scoring_checkpoint.json after each batch
  - Track last_processed_quote_id
  - On resume, continue from checkpoint
```

## Scoring Logic

### Authenticity Score (0-25)

```
CALCULATE authenticity:

FIRST-PERSON (0-8 points):
- Uses I/my/me throughout: +8
- Mixed first/third person: +4
- Third person only: +0

PLATFORM CREDIBILITY (0-7 points):
- Reddit/forums (raw, unmoderated): +7
- YouTube comments: +6
- Reviews (verified): +6
- Social media: +5
- Instructional content: +3
- Promotional sources: +0

ENGAGEMENT VALIDATION (0-5 points):
- engagement_score >= 80: +5
- engagement_score >= 60: +4
- engagement_score >= 40: +3
- engagement_score >= 20: +2
- engagement_score < 20: +1

VERIFICATION SIGNALS (0-5 points):
- Verified purchase/user: +3
- Long account history: +1
- Community recognition: +1
```

### Emotional Power Score (0-25)

```
CALCULATE emotional_power:

INTENSITY (0-10 points):
- emotional_intensity >= 9: +10
- emotional_intensity >= 7: +8
- emotional_intensity >= 5: +6
- emotional_intensity >= 3: +4
- emotional_intensity < 3: +2

VIVIDNESS (0-8 points):
- Metaphor/analogy present: +3
- Physical sensation described: +3
- Action/behavior described: +2

UNIVERSALITY (0-7 points):
- Describes common experience: +4
- Relatable emotions: +3
```

### Specificity Score (0-25)

```
CALCULATE specificity:

NUMBERS/QUANTITIES (0-10 points):
- Multiple specific numbers: +10
- Single specific number: +7
- Approximate numbers: +4
- No numbers: +0

NAMED ENTITIES (0-8 points):
- Specific products named: +4
- Specific methods named: +2
- Specific people referenced: +2

TEMPORAL DETAIL (0-7 points):
- Specific dates/timeframes: +4
- Duration mentioned: +3
- No temporal context: +0
```

### Usability Score (0-25)

```
CALCULATE usability:

LENGTH APPROPRIATENESS (0-8 points):
- 25-60 words: +8 (ideal)
- 15-24 words: +6 (usable)
- 61-80 words: +5 (slightly long)
- 81-100 words: +3 (edit needed)
- <15 or >100 words: +1 (problematic)

STANDALONE CLARITY (0-9 points):
- Requires no context to understand: +9
- Minor context helpful: +6
- Requires significant context: +3
- Confusing without context: +1

GRAMMATICAL QUALITY (0-8 points):
- Clean, natural language: +8
- Minor issues: +5
- Significant issues: +2
- Broken/unclear: +0
```

## Score Aggregation

```
CALCULATE total_score:

total_score = authenticity_score + emotional_power_score + specificity_score + usability_score

ASSIGN quality_tier:
- 80-100: "excellent" - Top quotes, ready for prime placement
- 60-79: "good" - Solid quotes, minor improvements possible
- 40-59: "acceptable" - Usable with context/editing
- 0-39: "low" - Volume only, not for primary use
```

## Execution Flow

### Step 1: Load and Validate

```
LOAD classified_quotes.json

VALIDATE:
- All required fields present
- Bucket assignments complete
- Aspect classifications complete
```

### Step 2: Score Each Quote

```
FOR each quote:

  CALCULATE authenticity_score:
    - Assess first-person usage
    - Check platform type
    - Apply engagement score
    - Note verification signals

  CALCULATE emotional_power_score:
    - Use emotional_intensity from classification
    - Scan for vivid language
    - Assess universality

  CALCULATE specificity_score:
    - Count numbers/quantities
    - Identify named entities
    - Check temporal detail

  CALCULATE usability_score:
    - Measure word count
    - Assess standalone clarity
    - Check grammatical quality

  SUM to total_score
  ASSIGN quality_tier
  GENERATE scoring_notes
```

### Step 3: Rank and Sort

```
SORT quotes by total_score DESC within each bucket

EXTRACT top_quotes_by_bucket (PRD v3.0 Section 3.7 - ALL 6 BUCKETS MANDATORY):
- Top 25 pain quotes (P-001 through P-025)
- Top 25 hope quotes (H-001 through H-025)
- Top 25 root_cause quotes (RC-001 through RC-025)
- Top 25 solutions_tried quotes (ST-001 through ST-025)
- Top 25 competitor_mechanism quotes (CM-001 through CM-025)
- Top 25 villains quotes (V-001 through V-025)

TOTAL: 150 top quotes across all 6 buckets
```

### Step 4: Flag for Review

```
FLAG quotes when:
- Score seems inconsistent with content
- High engagement but low score
- Low engagement but appears high quality
- Unusual patterns

ADD to flagged_for_review with reason
```

## Quality Validation

```
VALIDATE score distribution:

EXPECTED DISTRIBUTION:
- Excellent (80+): 10-20%
- Good (60-79): 25-35%
- Acceptable (40-59): 30-40%
- Low (<40): 15-25%

IF distribution significantly off:
  - Review scoring parameters
  - Check for calibration issues
  - LOG anomaly
```
