Scrape social media platforms (Twitter/X, Facebook Groups, LinkedIn, Instagram) for customer voice data. Social media provides real-time sentiment and raw, unfiltered expressions.

Firecrawl is HARD-BLOCKED on x.com — Apify is the ONLY primary path.

#### Twitter/X Query Adaptation

```
ADAPT queries for Twitter/X:

FORMAT RULES:
- Shorter phrases (tweet-length awareness)
- Hashtag-enhanced versions of all queries
- Sentiment operators for emotional content
- "min_replies:5" or "min_faves:10" for engagement filtering (if actor supports)

QUERY PATTERNS:
- Pain:      "#[topic] frustrated" OR "[problem] killing my business"
- Hope:      "#[topic] breakthrough" OR "finally figured out [solution]"
- Fear:      "#AI replacing" OR "[topic] worried" OR "falling behind"
- Urgency:   "last chance [topic]" OR "window closing" OR "now or never"
- Discourse: "[thought_leader] [topic]" OR "@[expert_handle] [topic]"
- Competitor: "@[competitor] review" OR "[competitor] vs"

HIGH-VALUE TWITTER INDICATORS:
- Threads (multi-tweet analysis) — often contain deep expertise
- Quote tweets with commentary — reveal discourse dynamics
- High-reply tweets — indicate controversy or resonance
- Verified/notable accounts — authority signals
```

## Execution Logic

### Step 1: Platform Prioritization

```
PRIORITIZE platforms based on market_config:

FOR each platform in market_config.platforms:
  IF platform.type = "social":
    ADD to scrape_queue with priority

DEFAULT priority order:
1. Twitter/X - Highest volume of raw expression
2. Facebook Groups - Community depth
3. LinkedIn - Professional perspective
4. Instagram - Lifestyle/visual markets
5. TikTok - Younger demographic
```

### Step 2: Query Adaptation

```
ADAPT queries for each platform:

TWITTER:
- Shorter, hashtag-enhanced
- Add sentiment operators
- Example: "#trading frustrating" OR "trading ruined"

FACEBOOK:
- Natural language
- Question format
- Example: "struggling with trading"

LINKEDIN:
- Professional tone
- Industry terms
- Example: "day trading challenges professional"

INSTAGRAM/TIKTOK:
- Hashtag-centric
- Trend-aware
- Example: #tradingstruggle #traderfail
```

### Step 3: Scrape and Extract

```
FOR each platform:

  EXECUTE appropriate Apify actor

  WAIT for results (async if large scrape)

  FOR each post/item returned:

    CAPTURE:
    - Full text content
    - Author info
    - Engagement metrics
    - Date/time
    - Replies/comments

    PRE-EXTRACT quotes:
    - First-person statements
    - Emotional expressions
    - Problem descriptions
    - Question reveals

    APPEND to social_content
```

### Step 4: Engagement Weighting

```
WEIGHT content by engagement:

High value (prioritize in analysis):
- Posts with 100+ likes/reactions
- Threads with 20+ replies
- Viral content

Medium value:
- 10-100 engagement
- Active discussion

Low value (include but lower weight):
- < 10 engagement
- No replies

CALCULATE engagement_score for each post
```
