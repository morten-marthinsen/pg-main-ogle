# Skill 2.5-B: Proof Inventory Builder

**Version:** 2.0
**Layer:** 2 (Intelligence)
**Type:** Evidence Cataloging
**Dependencies:** scored_quotes.json, competitor_analysis.json, theme_synthesis.json
**Output:** proof_inventory.json

---

## Purpose

Catalog all available proof elements from customer quotes, competitor claims, and market data that can be used to support positioning, mechanisms, and claims in copy. Strong proof turns claims into credibility.

---

## Proof Categories

```
5 PROOF TYPES:

1. SOCIAL PROOF
   - Customer testimonials
   - Success stories
   - Community size
   - Influencer endorsements

2. STATISTICAL PROOF
   - Numbers and data
   - Research citations
   - Performance metrics
   - Market statistics

3. DEMONSTRATION PROOF
   - Before/after examples
   - Case studies
   - Process reveals
   - Live examples

4. AUTHORITY PROOF
   - Credentials
   - Experience
   - Media mentions
   - Institutional backing

5. SPECIFICITY PROOF
   - Detailed descriptions
   - Named methodologies
   - Exact processes
   - Concrete examples
```

## Extraction Logic

### Step 1: Extract Social Proof from Quotes

```
FROM scored_quotes:

IDENTIFY transformation quotes:
- "I used to... now I..."
- "Before X... after X..."
- "Everything changed when..."

FOR each:
  EXTRACT before/after states
  CAPTURE timeline if mentioned
  ASSESS authenticity and power
  NOTE what claim it supports

IDENTIFY endorsement quotes:
- Recommendations
- Praise
- Referrals

IDENTIFY community signals:
- Size mentions
- Activity levels
- Engagement patterns
```

### Step 2: Compile Statistical Proof

```
FROM competitor_analysis + market_narrative:

GATHER market statistics:
- Industry size
- Failure rates
- Growth trends
- Benchmark data

FOR each stat:
  VERIFY source
  NOTE citation
  IDENTIFY claim it supports

IDENTIFY performance metrics:
- Success rates
- Time to result
- Improvement percentages

RESEARCH potential citations:
- Academic studies
- Industry reports
- Survey data
```

### Step 3: Build Demonstration Proof

```
FROM scored_quotes + competitor_analysis:

IDENTIFY before/after patterns:
- Specific transformations
- Measurable changes
- Visible differences

EXTRACT process reveals:
- Steps described
- Methods explained
- Systems outlined

COMPILE case study candidates:
- Complete stories
- Documented journeys
- Verifiable results
```

### Step 4: Catalog Authority Proof

```
FROM competitor_analysis + market data:

IDENTIFY relevant credentials:
- Industry certifications
- Educational background
- Professional experience

IDENTIFY experience markers:
- Years in field
- Number of clients
- Specific achievements

IDENTIFY endorsements:
- Expert quotes
- Media mentions
- Institutional affiliations
```

### Step 5: Extract Specificity Proof

```
FROM all sources:

IDENTIFY named methodologies:
- Proprietary frameworks
- Specific systems
- Unique approaches

EXTRACT exact processes:
- Step-by-step descriptions
- Specific timelines
- Detailed procedures

CAPTURE concrete examples:
- Specific scenarios
- Real situations
- Detailed illustrations
```

### Step 6: Gap Analysis

```
FOR each major claim that will need proof:

ASSESS proof strength:
- What proof exists?
- How strong is it?
- What's missing?

IDENTIFY proof gaps:
- Claims without proof
- Weak proof areas
- Development needs

GENERATE recommendations:
- What proof to gather
- How to strengthen
- What to document
```

---

## Proof Strength Scoring

```
SCORE each proof element:

AUTHENTICITY (1-10):
- Real vs. manufactured
- Verifiable vs. not
- Specific vs. generic

RELEVANCE (1-10):
- Directly supports claim
- Adjacent support
- Tangential

POWER (1-10):
- Emotional impact
- Logical persuasion
- Memorability

USABILITY (1-10):
- Ready to use
- Needs editing
- Needs permission

OVERALL SCORE = average of all factors
```

---

## Quality Criteria

```
STRONG PROOF INVENTORY HAS:

1. BALANCE
   - Multiple proof types
   - Proof for all major claims
   - Mix of emotional and logical

2. SPECIFICITY
   - Exact numbers
   - Real names (where permitted)
   - Verifiable details

3. AUTHENTICITY
   - Real customer language
   - Documented sources
   - Traceable claims

4. USABILITY
   - Ready-to-use quotes
   - Clear citations
   - Proper permissions
```
