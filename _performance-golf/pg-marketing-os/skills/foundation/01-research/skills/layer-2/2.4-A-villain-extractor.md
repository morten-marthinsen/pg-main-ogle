# Skill 2.4-A: Villain Extractor

**Version:** 2.0
**Layer:** 2 (Intelligence)
**Type:** Narrative Intelligence
**Dependencies:** belief_analysis.json, theme_synthesis.json, competitor_analysis.json
**Output:** villain_analysis.json

---

## Purpose

Identify and develop the "villains" in the market narrative - the forces, entities, beliefs, or systems that the prospect blames or can blame for their problems. Villains create an external enemy and position the solution as the hero.

---

## Villain Framework

```
WHAT IS A VILLAIN?

A villain is the antagonist in the prospect's story.
It's what they blame (or can blame) for their failures.

VILLAIN TYPES:

1. ENTITY VILLAINS
   - Specific groups or organizations
   - "The brokers", "Wall Street", "Gurus"

2. SYSTEM VILLAINS
   - Structures or systems
   - "The market", "The economy", "The industry"

3. IDEOLOGY VILLAINS
   - Wrong beliefs or conventional wisdom
   - "What they taught you", "Common advice"

4. INVISIBLE VILLAINS
   - Hidden forces they don't see
   - Their own brain, biases, conditioning

5. PAST VILLAINS
   - Historical causes
   - Bad education, wrong mentors, early mistakes
```

## Extraction Logic

### Step 1: Identify Explicit Villains

```
FROM scored_quotes:

SEARCH for blame language:
- "Because of [entity]..."
- "[Entity] is the problem"
- "[Entity] caused this"
- "[Entity] doesn't want you to..."
- "[Entity] is rigged/broken/corrupt"

FOR each identified target of blame:
  CAPTURE:
  - What/who is blamed
  - How often
  - How emotionally charged
  - Example quotes

CATEGORIZE by villain_type
```

### Step 2: Identify Implicit Villains

```
FROM belief_analysis:

EXTRACT implicit villains from:
- Frustration targets
- Systemic complaints
- "It's not fair" statements
- Inequality observations

FROM theme_synthesis:
- What forces appear in pain patterns?
- What explanations emerge for failures?
- What external attributions exist?

ADD implicit villains to inventory
```

### Step 3: Assess Villain Viability

```
FOR each villain:

CALCULATE blame_frequency:
- Direct mentions
- Indirect references
- Implied blame

CALCULATE emotional_charge (1-10):
- Intensity of blame
- Anger level
- Victimization depth

ASSESS marketing_viability:
- High: Widely blamed, emotionally charged, defensible
- Medium: Some blame, moderate emotion, some risk
- Low: Rarely blamed, weak emotion, or risky

NOTE caution_notes:
- Legal concerns
- Alienation risks
- Credibility issues
```

### Step 4: Build Villain Hierarchy

```
RANK villains by impact:

PRIMARY VILLAINS:
- Most frequently blamed
- Highest emotional charge
- Safest to use in marketing
- Most useful for positioning

SECONDARY VILLAINS:
- Moderate frequency
- Useful for variety
- Support primary narrative

POTENTIAL VILLAINS:
- Low frequency but valid
- Could be developed
- Might resonate if introduced
```

### Step 5: Develop Villain Narratives

```
FOR each viable villain:

CREATE narrative_arc:
- Who/what is the villain?
- What did they do?
- How did it hurt the prospect?
- What are they hiding?
- Why do they persist?

EXAMPLE:
Villain: "Conventional Trading Education"
Did: Taught strategy without psychology
Hurt: Left traders unprepared for real execution
Hiding: That information alone doesn't create results
Persists: Because it's easier to sell "the secret strategy"
```

### Step 6: Strategic Recommendations

```
SYNTHESIZE strategic recommendations:

RECOMMENDED PRIMARY:
- Best villain for main narrative
- Why it works
- How to position against it

RECOMMENDED SECONDARY:
- Supporting villains
- When to use each
- How they reinforce primary

VILLAIN COMBINATIONS:
- Which villains work together?
- Narrative flow suggestions
- Escalation patterns

POSITIONING IMPLICATIONS:
- How villains position the solution
- What hero role emerges
- Transformation enabled
```

### Step 7: Proof Requirements

```
FOR each recommended villain:

IDENTIFY proof_needed:
- What evidence makes villain real?
- What credibility supports the claim?
- What would the prospect need to see?

This ensures villain claims can be backed up
```

---

## Quality Criteria

1. **Validity** - Villains based on actual blame patterns
2. **Safety** - Legal/PR risks assessed
3. **Usefulness** - Clear marketing applications
4. **Evidence** - Proof requirements identified
