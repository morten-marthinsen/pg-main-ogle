# Skill 2.6-B: Layer 2 Expansion Executor

**Version:** 2.0
**Layer:** 2 (Intelligence)
**Type:** Remediation & Expansion
**Dependencies:** layer2_validation_report.md, gate_decision.json
**Output:** Updated Layer 2 outputs, expansion_log_L2.json

---

## Purpose

Execute remediation when Layer 2 validation fails. This skill receives the failed validation report and systematically addresses each gap to achieve passing status before proceeding to Layer 3.

---

## Execution Trigger

```
EXECUTE WHEN:
gate_decision.status = "fail" AND
gate_decision.can_proceed_to_layer3 = false

DO NOT EXECUTE WHEN:
gate_decision.status IN ["pass", "conditional_pass"]
```

## Expansion Logic

### Step 1: Parse Remediation Plan

```
FROM layer2_validation_report.remediation_plan:

EXTRACT all required actions:
FOR each action in remediation_plan.actions:
  LOG:
  - action_type
  - target (which analysis/skill)
  - issue (what's wrong)
  - fix (what to do)
  - skill_to_rerun (if applicable)

PRIORITIZE actions:
1. Critical checks first (E5 completeness)
2. Foundation analyses before synthesis
3. Independent fixes before dependent ones

BUILD execution_queue
```

### Step 2: Execute E5 Gap Fixes

```
IF e5_completeness.status = "fail":

FOR each incomplete E5 analysis:

  IF web_analysis incomplete:
    RERUN Skill 2.2-A-web-analysis
    WITH enhanced focus on missing elements
    VALIDATE primary_web defined

  IF belief_analysis incomplete:
    RERUN Skill 2.2-B-belief-excavator
    WITH deeper quote mining
    VALIDATE belief_inventory populated

  IF now_after_grid incomplete:
    RERUN Skill 2.2-C-now-after-grid
    ENSURE all 6 dimensions covered

  IF awareness_map incomplete:
    RERUN Skill 2.2-D-awareness-mapper
    ENSURE distribution calculated

SAVE updated outputs
LOG actions taken
```

### Step 3: Execute Pattern Quality Fixes

```
IF pattern_quality.status = "fail":

DIAGNOSE specific issues:

IF themes_identified < 5:
  RERUN Skill 2.1-A-pattern-analyzer
  WITH lower thresholds
  WITH forced clustering for minimum themes

IF patterns_per_theme_avg < 3:
  RERUN pattern analysis
  WITH expanded pattern recognition
  WITH deeper quote mining

IF quote_coverage < 70%:
  REVIEW unclassified quotes
  ADD to existing themes or create new ones
  EXPAND theme definitions if needed

RERUN Skill 2.1-B-theme-synthesizer
WITH updated patterns

SAVE updated theme_synthesis.json
LOG improvements
```

### Step 4: Execute Mechanism Readiness Fixes

```
IF mechanism_readiness.status = "fail":

DIAGNOSE specific issues:

IF mechanisms_cataloged < 15:
  RERUN Skill 2.3-B-mechanism-extractor
  WITH expanded competitor search
  WITH ad library deep dive
  WITH funnel content review

IF implied_mechanisms < 5:
  RERUN mechanism extraction
  WITH focus on customer language
  MINE quotes for mechanism language
  EXTRACT latent mechanism concepts

IF development_opportunities < 3:
  ANALYZE mechanism gaps more deeply
  CROSS-REFERENCE with customer needs
  IDENTIFY unmet mechanism needs

IF saturation_analysis missing:
  COMPLETE saturation analysis
  SCORE each mechanism for market frequency

SAVE updated mechanism_inventory.json
LOG improvements
```

### Step 5: Execute Competitive Intelligence Fixes

```
IF competitive_intelligence.status = "fail":

DIAGNOSE specific issues:

IF competitors_analyzed < 5:
  EXPAND competitor list
  RERUN Skill 2.3-A-competitor-analyzer
  WITH additional sources
  WITH broader definition

IF positioning_mapped = false:
  COMPLETE positioning landscape
  MAP all competitors on grid
  IDENTIFY position clusters

IF white_space_identified = false:
  ANALYZE positioning gaps
  IDENTIFY unoccupied territory
  DOCUMENT opportunities

IF reception_analysis incomplete:
  GATHER more competitor reception data
  MINE reviews and comments
  COMPLETE sentiment analysis

SAVE updated competitor_analysis.json
LOG improvements
```

### Step 6: Execute Synthesis Coherence Fixes

```
IF synthesis_coherence.status = "fail":

DIAGNOSE specific issues:

IF market_narrative_clear = false:
  RERUN Skill 2.3-C-market-narrative-builder
  ENSURE all components connected
  WRITE clear one-paragraph summary

IF transformation_defined = false:
  ENHANCE now_after_grid
  WRITE explicit transformation_summary
  CONNECT to market narrative

IF strategic_direction_clear = false:
  SYNTHESIZE from all analyses
  DEFINE clear positioning direction
  DOCUMENT recommended approach

IF contradictions found:
  IDENTIFY specific contradictions
  RESOLVE with authoritative source
  UPDATE conflicting documents
  ENSURE alignment

SAVE updated synthesis documents
LOG resolution steps
```

### Step 7: Cross-Check Dependencies

```
AFTER all fixes applied:

VERIFY dependencies are satisfied:
- Theme synthesis matches updated patterns
- Market narrative reflects all E5 analyses
- Opportunity gaps reflect competitor analysis
- Proof inventory reflects available quotes

IF dependency misalignment found:
  RERUN downstream analysis
  UPDATE to incorporate upstream changes

ENSURE all outputs are internally consistent
```

### Step 8: Revalidation

```
AFTER all fixes complete:

RERUN Skill 2.6-A-layer2-validator

COMPARE new validation results:
- Which checks now pass?
- Which issues remain?
- Is gate now passable?

IF gate_decision.status = "pass":
  LOG success
  COMPILE expansion_log
  PROCEED to Layer 3

IF gate_decision.status = "conditional_pass":
  LOG partial success
  NOTE remaining warnings
  PROCEED to Layer 3 with caution

IF gate_decision.status = "fail":
  INCREMENT iteration count
  IF iteration <= 3:
    RETURN to Step 1 with new remediation plan
  ELSE:
    ESCALATE to human review
    LOG persistent issues
    HALT expansion loop
```

---

## Iteration Control

```
MAXIMUM ITERATIONS: 3

ITERATION ESCALATION:
- Iteration 1: Standard remediation
- Iteration 2: Deep remediation with expanded sources
- Iteration 3: Full re-analysis with manual review flags
- Iteration 4+: Human intervention required

PREVENT infinite loops by:
- Tracking fixed vs. recurring issues
- Limiting same-fix attempts
- Escalating persistent problems
```

## Success Criteria

```
EXPANSION SUCCESSFUL WHEN:

1. All critical checks pass
2. Gate status = "pass" or "conditional_pass"
3. No critical issues remain
4. All outputs are internally consistent
5. Layer 3 prerequisites met

EXPANSION ESCALATION WHEN:

1. 3 iterations without pass
2. Same issue persists after targeted fix
3. Fundamental data gap identified
4. Human judgment required
```
