# 4.1 — Rescore

**Version:** 1.0
**Skill:** U5 — Upsell Editorial
**Layer:** 4
**Microskill:** 4.1 — Rescore
**Model:** opus
**Purpose:** Re-score every piece in the REVISED upsell sequence across all 7 editorial lenses, calculate final weighted averages, compute improvement deltas against baseline scores, and flag any regressions (scores that went DOWN after revision).

---

## INPUT

- `layer-1/1.1-baseline-scores.md` (baseline per piece x per lens scores — the "before")
- `layer-2/2.1-revisions.md` (revision record — what changed)
- The REVISED upsell sequence (the copy as it exists AFTER all Layer 2 revisions are applied)
- `layer-0/0.2-scoring-rubric.md` (same rubric used for baseline — consistency is critical)

## EXECUTION STEPS

1. **Confirm the scoring target.** You are scoring the REVISED sequence — the copy AFTER all Layer 2 fixes have been applied. Not the original. Not the baseline. The CURRENT state. Verify that the revision record in 2.1 shows all clusters executed and all fixes applied.

2. **Load the same scoring rubric.** Use IDENTICAL rubric from `0.2-scoring-rubric.md`. Same lens definitions, same tier descriptions, same weights. The delta is meaningless if the rubric changed between baseline and rescore.

3. **Score ORDER BUMP across all 7 lenses.** Same protocol as 1.1:
   - Read the revised bump copy in full
   - Per lens: score 1-10 using rubric tier definitions
   - Write specific rationale citing exact passages from the REVISED copy
   - Note improvements from baseline where relevant

4. **Score UPSELL PAGE(S) across all 7 lenses.** Same protocol:
   - Read the revised upsell page copy in full
   - Per lens: score 1-10 with passage-level rationale
   - Pay attention to revised passages — did the fixes actually improve the lens scores?

5. **Score DOWNSELL PAGE(S) across all 7 lenses.** Same protocol:
   - Read the revised downsell page copy in full
   - Per lens: score 1-10 with passage-level rationale
   - Verify ARO structure maintained after revisions

6. **Calculate weighted averages per piece.** Same formula: (L1 * 0.25) + (L2 * 0.15) + (L3 * 0.20) + (L4 * 0.15) + (L5 * 0.10) + (L6 * 0.10) + (L7 * 0.05)

7. **Compute deltas.** For each piece x each lens:
   - Delta = Final Score - Baseline Score
   - Positive delta = improvement
   - Zero delta = no change
   - Negative delta = REGRESSION (flag immediately)

8. **Flag regressions.** Any lens score that went DOWN after revision requires:
   - Identification of which revision caused the regression
   - Assessment: is the regression acceptable? (e.g., Speed score dropped from 9 to 8 because a necessary congruence fix added 20 words — net positive)
   - If regression is NOT acceptable: flag for reversion or additional fix in the specific area
   - A fix that drops a different lens by more than it raises the target lens is a net-negative fix

9. **Check 7.5 threshold.** Every piece must achieve >= 7.5 weighted average.
   - If ALL pieces >= 7.5: proceed to 4.2
   - If ANY piece < 7.5: identify which lens(es) are pulling it below threshold. This piece needs additional revision — return to Layer 2 with specific targets.
   - If a piece was < 7.5 at baseline and is STILL < 7.5 after revision: the fixes did not sufficiently address the issues. Flag for human decision: more revision or accept with documented weaknesses.

10. **Compute overall improvement statistics.** Average improvement across all lenses, across all pieces. Identify the most-improved lens and least-improved lens.

11. **Write output file** with complete rescore data, deltas, and regression flags.

## CONSTRAINTS

1. **Score the REVISED sequence.** Not the original. Not the baseline copy. The copy as it exists after all Layer 2 revisions. If you score the original, the delta is zero and the rescore is worthless.
2. **Use the SAME rubric.** Identical lens definitions, identical tier descriptions, identical weights. Changing the rubric between baseline and rescore invalidates the delta.
3. **Every piece, every lens.** Same completeness requirement as baseline. No skipping lenses because "it wasn't affected by revisions." Revisions can have unintended cross-lens effects.
4. **Regressions must be flagged and assessed.** A negative delta is not automatically a problem (trade-offs exist) but it MUST be acknowledged and assessed. Silent regressions are the #1 editorial failure mode.
5. **7.5 is the minimum threshold.** Not 7.4. Not "close enough." Binary: >= 7.5 passes, < 7.5 fails.
6. **Delta must be computed per lens per piece.** Not just weighted averages. A piece can have an improved average while one specific lens regressed — that regression still matters.
7. **If any piece fails the 7.5 threshold after revision, it returns to Layer 2.** This is not optional. Do not proceed to 4.2 with failing pieces and hope they pass at Gate 3.

## FAILURE MODES

| Failure | Detection | Response |
|---------|-----------|----------|
| Scoring original instead of revised | Rescore passages match baseline passages exactly | Confirm the scoring target is the post-revision copy. Re-read 2.1-revisions.md to verify what changed. |
| Different rubric used | Lens weights or tier definitions differ from 0.2 | Rescore using the original rubric. Consistency is mandatory. |
| Regression not flagged | Final score < baseline score but no regression flag | Add the regression flag with causal analysis and acceptability assessment. |
| Piece below 7.5 accepted | Weighted average < 7.5 but output says PASS | Correct to FAIL. Return to Layer 2 for additional revision. |
| Delta miscalculated | Final - Baseline doesn't match reported delta | Recalculate. Show the arithmetic. |
| Incomplete rescore | Fewer lenses or pieces scored than in baseline | Score all missing lenses/pieces. The rescore must be complete and parallel to the baseline. |
