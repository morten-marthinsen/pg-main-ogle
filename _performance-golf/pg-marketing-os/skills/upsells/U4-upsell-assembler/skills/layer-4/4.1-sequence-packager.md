# 4.1 — Sequence Packager

**Skill:** U4 — Upsell Sequence Assembler
**Microskill:** 4.1 — Sequence Packager
**Layer:** 4
**Model:** sonnet
**Input:** `layer-0/0.1-input-context.md`, `layer-1/1.1-sequence-order.md`, `layer-1/1.2-thread-validation.md`, `layer-1/1.3-congruence-chain.md`, `layer-1/1.4-pricing-cascade.md`, `layer-1/1.5-speed-validation.md`, all loaded copy pieces (U1, U2, U3)
**Output:** `upsell-sequence-assembled.md` + `validation-report.md`

---

## PURPOSE

Assemble ALL upsell sequence pieces verbatim into a single cohesive document with YAML metadata and sequence map, then compile all 5 validator results plus a drift analysis into a comprehensive validation report. This produces the two primary deliverables of U4.

---

## EXECUTION STEPS

### Part A: Assemble the Sequence Document

1. **Read `layer-0/0.1-input-context.md`** for all metadata (prices, products, word counts, mode, mechanism name).

2. **Read `layer-1/1.1-sequence-order.md`** for the canonical funnel order and verified word counts.

3. **Build the YAML metadata header.** Populate every field from upstream sources:
   - skill, version, project, mode
   - FE product, FE price, FE mechanism
   - Per sequence piece: product, price, word count, price ratio to FE (or to upsell for downsell)
   - Upsell format (text_page / video_script)
   - Downsell reframe type
   - Total word count, total reading time
   - Drift score (calculated in Part B — insert after drift calculation)
   - Validation summary (PASS/FAIL per dimension — insert after Part B)
   - Created date

4. **Build the sequence map diagram.** Use the tree format from the AGENT.md output schema:
   ```
   ORDER FORM: [FE Product] — $[price]
     └── ORDER BUMP: [Bump Product] — $[price] ([X%] of FE)
            ↓ [purchase completes]
         UPSELL 1: [Upsell Product] — $[price] ([X%] of FE)
            ├── YES → Thank You (or Upsell 2 if applicable)
            └── NO → DOWNSELL 1: [Downsell Product] — $[price] ([X%] of Upsell)
                       └── YES/NO → Thank You
   ```
   Populate with actual product names, prices, and calculated ratios.

5. **Assemble the full text of each piece in funnel order.**
   - For EACH piece, insert a clear page boundary header:
     ```
     ---
     ## ORDER BUMP — [Product Name]
     ```
   - Insert the COMPLETE copy body — verbatim, word-for-word, from the source file
   - **CRITICAL: Use the ACTUAL Arena-selected copy.** Do NOT summarize, excerpt, paraphrase, or "clean up." The copy goes in exactly as it was selected from the Arena (for U2/U3) or as the selected variant (for U1).
   - Do NOT strip CAIRO section headers from U2/U3 copy — they are part of the structure
   - Do NOT strip the 3-element structure from U1 copy
   - Strip only: YAML metadata headers from individual files, downstream handoff data, implementation notes, Arena scoring annotations. The COPY BODY transfers verbatim.

6. **Verify the assembled text matches source files.** After assembly, spot-check: compare the first and last sentence of each assembled piece against the source file. Any discrepancy = rewriting, which is forbidden.

### Part B: Generate the Validation Report + Drift Analysis

7. **Read all 5 validator output files:**
   - `layer-1/1.2-thread-validation.md`
   - `layer-1/1.3-congruence-chain.md`
   - `layer-1/1.4-pricing-cascade.md`
   - `layer-1/1.5-speed-validation.md`
   - (1.1 is structural, not a pass/fail validator — its data is already incorporated in the assembly)

8. **Compile the validation report.** For each validator, include:
   - Overall status (PASS / FAIL)
   - Summary of findings
   - Any specific failures with evidence
   - Fix recommendations (passed through to U5)

9. **Calculate drift from U0 strategy.** Compare the assembled sequence against the original U0 plan across 4 dimensions:

   | Drift Dimension | What to Compare | Measurement |
   |----------------|-----------------|-------------|
   | Mechanism Name Consistency | U0 mechanism name vs names used in assembled copy | EXACT MATCH = 0% drift. Any variation = flag + estimate severity. |
   | Pricing Cascade Compliance | U0 target prices vs actual prices in assembled copy | Per-position: abs(actual - target) / target x 100. Average across positions. |
   | Word Count Targets | U0 word count targets vs actual word counts | Per-position: abs(actual - target) / target x 100. Average across positions. |
   | Reframe Type Match | U0 planned downsell reframe type vs actual reframe type used | MATCH = 0% drift. MISMATCH = 25% drift for this dimension. |

   **Overall drift score:** Average of the 4 dimension scores.
   - < 15% → PASS
   - >= 15% → FAIL — escalate to human for review

10. **Insert drift score and validation summary into the YAML metadata header** of the assembled document (step 3).

11. **Write both output files.**

## CONSTRAINTS

- **VERBATIM assembly is non-negotiable.** The copy in the assembled document must be EXACTLY as it appears in the source files. Do not edit, improve, clean up, or rewrite a single word. U4 assembles. U5 edits. If you catch a typo, document it in the validation report for U5 — do not fix it yourself.
- **ACTUAL Arena-selected copy only.** Cross-reference against GATE_2.5 checkpoints verified in 0.1. If there is any doubt about which version is the Arena winner, HALT and clarify rather than assembling the wrong draft. This exact error cost an entire session with Skill 19.
- **Every metadata field must be populated.** No "TBD," no placeholders, no "see above." If a field cannot be populated from upstream data, something went wrong — document the gap but populate what you can.
- **Drift threshold is 15%.** At or above 15% requires human escalation. The drift score goes in the YAML metadata header so it is visible immediately when the document is opened.
- **Both output files are required.** The assembled sequence and the validation report are separate files. Do not combine them. The assembled sequence is the DELIVERABLE (what U5 works on). The validation report is the AUDIT TRAIL (what guides U5's editorial pass).
- **Strip only non-copy elements from source files.** YAML metadata, downstream handoff sections, implementation notes, Arena scoring annotations — these are removed. Everything else (CAIRO headers, copy body, guarantees, CTAs, bonus stacks) transfers verbatim.

---

## FAILURE MODES

| Failure | Detection | Response |
|---------|-----------|----------|
| Copy rewritten during assembly | Compare assembled text against source — any change beyond whitespace/formatting | Revert to source text. U4 does NOT edit. Document the attempted change for transparency. |
| Drift score >= 15% | Drift calculation shows overall average >= 15% | GATE_2 FAIL. Document exact deviations in validation report. Escalate to human: accept the drift (override) or request upstream revision. |
| Validation report missing a dimension | Fewer than 5 sections in report | Check which validator output file is missing. Request re-run of the missing validator before packaging. |
| Assembled document missing a piece | Piece count in assembled doc < piece count in 1.1 sequence order | Check which piece was dropped during assembly. Re-assemble with all pieces. |
| YAML metadata has placeholder values | Any field contains "TBD" or "[placeholder]" | Trace the field back to its source. Populate from upstream data. If upstream data is genuinely missing, document the gap explicitly rather than using a placeholder. |
