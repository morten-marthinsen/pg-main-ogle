# 0.1 — Input Loader

**Skill:** U4 — Upsell Sequence Assembler
**Microskill:** 0.1 — Input Loader
**Layer:** 0
**Model:** haiku
**Input:** U0 upsell-strategy.yaml, U1 order-bump-copy.md (selected variant), U2 upsell-page-draft.md (Arena-selected), U3 downsell-page-draft.md (Arena-selected), mechanism-package.json
**Output:** `layer-0/0.1-input-context.md`

---

## PATH RESOLUTION

All upstream packages use the standardized output folder:

```
PRIMARY:  outputs/[project-code]/upsell/[filename]
FALLBACK: outputs/[project-code]/[filename]
```

Where `[project-code]` is the 2-4 letter campaign identifier (e.g., RS1, SPD, ULV). Foundation packages (mechanism-package.json) are at the project-code root; upsell-specific outputs (upsell-strategy.yaml, drafts) are in the `upsell/` subdirectory.

## PURPOSE

Load all upstream outputs from U0-U3 and the mechanism package, verify Arena-selected versions are used for U2/U3, determine operating mode (Full vs Partial), and extract key fields for downstream validators.

---

## EXECUTION STEPS

1. **Check project directory** for all 5 upstream files:
   - `upsell-strategy.yaml` (from U0) — REQUIRED in both modes
   - `order-bump-copy.md` (from U1) — selected variant
   - `upsell-page-draft.md` (from U2) — Arena-selected
   - `downsell-page-draft.md` (from U3) — Arena-selected
   - `mechanism-package.json` (from Skill 04)

2. **CRITICAL: Verify Arena-selected versions for U2 and U3.**
   - Check U2 project directory for `GATE_2.5_HUMAN_SELECTED.yaml`. The file referenced in that checkpoint is the Arena-selected version. Load THAT file, not any other draft.
   - Check U3 project directory for `GATE_2.5_HUMAN_SELECTED.yaml`. Same verification.
   - If GATE_2.5 checkpoint files do not exist, HALT. The upstream skill has not completed Arena. Do NOT proceed with a pre-Arena draft.
   - **This exact failure happened with Skill 19** — pre-Arena section-2B-draft.md was assembled instead of Arena-selected Hybrid A. The entire lead was missing from the assembled draft.

3. **Determine operating mode:**
   - **Mode A (Full Sequence):** All 4 copy pieces present (U0 strategy + U1 bump + U2 upsell + U3 downsell). This is the standard path.
   - **Mode B (Partial Sequence):** U0 strategy present + at least 1 copy piece. Assembly proceeds with available pieces only. All output carries "PARTIAL SEQUENCE" flag.
   - **GATE_0 FAIL conditions:**
     - Mode A: any of the 4 required upstream outputs missing
     - Mode B: U0 strategy missing, OR zero copy pieces present

4. **Extract key fields from each loaded piece:**

   From **upsell-strategy.yaml** (U0):
   - Planned sequence (which pieces, what order)
   - Pricing targets per position
   - Reframe type for downsell
   - Word count targets per piece
   - Mechanism name (as strategic anchor)
   - Product names for each position

   From **order-bump-copy.md** (U1):
   - Bump product name
   - Bump price
   - Bump word count (actual)
   - Bump template type used (Completeness/Accelerator/Insurance/Exclusive)

   From **upsell-page-draft.md** (U2):
   - Upsell product name
   - Upsell price
   - Upsell word count (actual)
   - Upsell format (text_page / video_script)
   - Mechanism name as used in copy (EXACT string)
   - Root cause language as used in copy (VERBATIM)
   - Promise as extended in copy
   - Congruence score (from 4.2)
   - Arena selection data (persona, weighted score)

   From **downsell-page-draft.md** (U3):
   - Downsell product name
   - Downsell price
   - Downsell word count (actual)
   - Reframe type used
   - Mechanism name as used in copy (EXACT string — must match U2)
   - Arena selection data

   From **mechanism-package.json** (Skill 04):
   - Mechanism name (EXACT — this is the canonical version)
   - Root cause language (canonical)
   - Core promise

5. **Cross-reference mechanism names.** The mechanism name should appear identically in U0 strategy, U2 copy, U3 copy, and mechanism-package.json. Log any discrepancies — 1.2 will validate, but flag here if obvious.

6. **Calculate FE context:**
   - FE product name (from U0 strategy)
   - FE price (from U0 strategy)
   - These are needed for pricing cascade ratios in 1.4

7. **Write output file** with all extracted data organized by source.

## CONSTRAINTS

- Mode determination happens FIRST — before any package loading. Do not load Mode A packages and then "fall back" to Mode B mid-process.
- Arena verification for U2 and U3 is NON-NEGOTIABLE. If GATE_2.5 checkpoint files are not found, HALT. Do not proceed with any draft that might be pre-Arena.
- Mechanism name must be extracted as an EXACT string from each source, not paraphrased or summarized. If the mechanism is "The Metabolic Switch Protocol," record exactly that — not "metabolic switching."
- Root cause language must be VERBATIM from the copy as written.
- Never fabricate data for missing fields. If a field cannot be extracted, write "NOT FOUND" and flag as a gap.
- In Mode B, every downstream output file must carry the "PARTIAL SEQUENCE" flag.

---

## FAILURE MODES

| Failure | Detection | Response |
|---------|-----------|----------|
| Pre-Arena draft loaded instead of Arena-selected version | GATE_2.5 checkpoint file missing or not cross-referenced | HALT. Request Arena completion for U2/U3 before proceeding. Do NOT assemble with unselected copy. |
| Mode A declared but upstream piece missing | Piece count < 4 after loading | Either switch to Mode B (with human approval) or HALT and request the missing upstream output. |
| Mechanism name discrepancy between sources | Cross-reference check shows different strings | Log exact discrepancy. Do not resolve — pass to 1.2 for formal validation. Flag as potential GATE_1 issue. |
| U0 strategy YAML malformed or incomplete | Missing required fields (pricing targets, planned sequence) | GATE_0 FAIL. U0 must be re-run before U4 can proceed. |
