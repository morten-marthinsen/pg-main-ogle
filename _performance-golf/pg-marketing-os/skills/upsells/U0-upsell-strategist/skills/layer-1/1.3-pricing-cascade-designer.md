# 1.3 — Pricing Cascade Designer

**Layer:** 1
**Model:** opus
**Depends on:** 1.1 (can run parallel with 1.2)
**Produces:** `layer-1/1.3-pricing-cascade.md`

---

## Purpose

Design the pricing cascade for the upsell sequence, ensuring descending commitment and ascending value per dollar at each step.

## Pricing Rules (NON-NEGOTIABLE)

| Position | Price Range (% of Front-End) | Rationale |
|----------|------------------------------|-----------|
| Order Bump | 10-30% of FE price | Must feel like "nothing" added to existing purchase |
| Upsell 1 | 50-150% of FE price | Significant but anchored against FE value |
| Downsell 1 | 30-50% of Upsell 1 price | Clear reduction, but not insulting |
| Upsell 2 (if any) | 30-100% of FE price | Lower commitment than Upsell 1 |
| Downsell 2 (if any) | 30-50% of Upsell 2 price | Same pattern |

**Example cascade for $47 front-end:**
- Bump: $7-14
- Upsell 1: $24-71
- Downsell 1: $7-36 (30-50% of upsell 1)

## Execution Steps

1. Read front-end price from Layer 0 context
2. Read product assignments from 1.2 (or run in parallel using price ranges)
3. Calculate cascade ranges for each position
4. Select specific prices within ranges based on:
   - Product perceived value (what would buyer expect to pay?)
   - Common price points ($9.95, $19.95, $27, $37, $47, $67, $97)
   - Psychology: charm pricing for impulse (X.95), round for premium ($97 vs $96.95)
5. Calculate total sequence economics:
   - Maximum cart value (all YES)
   - Minimum cart value (all NO except front-end)
   - Estimated average cart value (using industry conversion rates)
6. Validate: does each step feel like descending commitment + ascending value?

## Estimated Conversion Rates (Industry Benchmarks)

| Position | Typical Conversion | Range |
|----------|-------------------|-------|
| Order Bump | 50-70% | High if congruent |
| Upsell 1 | 15-30% | Depends on price and congruence |
| Downsell 1 | 30-50% of those who declined upsell | Reframe quality matters |
| Upsell 2 | 10-20% | Fatigue factor |

```markdown

## Front-End Anchor: $[price]

## Cascade Design

| Position | Product | Price | % of FE | Value/$ Signal |
|----------|---------|-------|---------|---------------|
| Front-End | [product] | $[X] | 100% | Baseline |
| Order Bump | [product] | $[X] | [X]% | [signal] |
| Upsell 1 | [product] | $[X] | [X]% | [signal] |
| Downsell 1 | [product] | $[X] | [X]% of U1 | [signal] |

## Sequence Economics
- Max cart value (all YES): $[total]
- Min cart value (FE only): $[FE price]
- Estimated average cart value: $[weighted]
- Revenue uplift per buyer: [X]%

## Descending Commitment Check
- [ ] Each price point feels effortless relative to previous YES
- [ ] No position asks MORE than the one before it
- [ ] Downsells are clearly less than their paired upsell

## Ascending Value Check
- [ ] Each position delivers more value per dollar than the last
- [ ] Bundle pricing makes each addition feel like a steal
- [ ] The "all YES" buyer gets disproportionate value
```

## Failure Conditions
- Any position outside prescribed range → FAIL (return and re-price)
- Downsell priced higher than its paired upsell → FAIL
- Total sequence feels like ascending commitment → FAIL (invert the pattern)
