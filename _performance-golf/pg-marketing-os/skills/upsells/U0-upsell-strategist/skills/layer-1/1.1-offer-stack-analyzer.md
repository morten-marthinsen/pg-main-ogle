# 1.1 — Offer Stack Analyzer

**Layer:** 1
**Model:** opus
**Depends on:** GATE_0 PASS
**Produces:** `layer-1/1.1-offer-analysis.md`

---

## Purpose

Decompose the front-end offer into its component parts to identify upsell opportunities. Understand what the buyer is getting, what gaps exist, and what natural extensions the offer creates.

## Execution Steps

1. Read the validated context from Layer 0
2. Decompose the front-end offer:

### Offer Decomposition Framework

| Component | Extract | Upsell Implication |
|-----------|---------|-------------------|
| **Core product** | Primary deliverable | What completes it? What accelerates it? |
| **Bonuses** | What's included free | What's conspicuously absent? |
| **Guarantee** | Risk reversal type + terms | Can it be extended/enhanced? |
| **Price point** | Base price | Sets the cascade ceiling |
| **Value equation** | Dream outcome × likelihood / time × effort | What upsell shifts each variable? |
| **Mechanism** | How it works (from Skill 04) | What enhances the mechanism? |
| **Format** | Physical/digital/service/hybrid | Format switches as upsell opportunity |

3. Identify **natural extension points** — where does the buyer's journey naturally need "more"?
4. Identify **gap opportunities** — what's conspicuously NOT in the offer?
5. Identify **acceleration opportunities** — what makes the core product work faster/better?
6. Map the value equation levers (Hormozi framework):
   - Increase dream outcome → bigger promise upsell
   - Increase perceived likelihood → proof/guarantee upsell
   - Decrease time delay → accelerator upsell
   - Decrease effort/sacrifice → done-for-you upsell
