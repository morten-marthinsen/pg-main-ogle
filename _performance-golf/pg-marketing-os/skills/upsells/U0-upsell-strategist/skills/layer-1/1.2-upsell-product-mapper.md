# 1.2 — Upsell Product Mapper

**Layer:** 1
**Model:** opus
**Depends on:** 1.1 (Offer Stack Analysis)
**Produces:** `layer-1/1.2-product-mapping.md`

---

## Purpose

Map available products/services to upsell positions based on congruence with the front-end offer and strategic value.

## Execution Steps

1. Read 1.1 output (extension points, gap opportunities, acceleration opportunities)
2. Read available upsell products from context (Mode A: may be in offer-package or campaign-brief; Mode B: from brief)
3. For each available product, score against position fit:

### Position-Product Fit Matrix

| Position | Ideal Product Type | Congruence Requirement | Copy Length |
|----------|-------------------|----------------------|------------|
| Order Bump | Completeness add-on, quick-start guide, tool | HIGHEST — must feel like "of course I need this" | 50-150w |
| Upsell 1 | Program extension, accelerator, premium tier | HIGH — extends the mechanism directly | 500-2000w |
| Downsell 1 | Core extract, payment plan, lite version | MODERATE — maintains mechanism thread | 300-1000w |
| Upsell 2 | Complementary product, community access | MODERATE — related but can be more distant | 500-1500w |
| Downsell 2 | Minimum viable version, trial | LOWER — but still connected | 300-800w |

4. Score each product for each position:

### Scoring Criteria

| Criterion | Weight | Scoring |
|-----------|--------|---------|
| Congruence (mechanism continuity) | 40% | 1-10: How naturally does this extend the mechanism? |
| Value perception | 25% | 1-10: How much value does this add per dollar? |
| Price fit | 20% | 1-10: Does it fit the cascade for this position? |
| Buyer momentum match | 15% | 1-10: Does the decision feel effortless at this point? |

5. Assign products to positions (highest-scoring product per position)
6. Document rationale for each assignment
7. Flag positions with no suitable product

## Output Format

```markdown
# U0 Layer 1.2 — Product Mapping

## Available Products
| Product | Description | Suggested Price | Natural Position |
|---------|-------------|----------------|-----------------|
| [product 1] | [desc] | $[X] | [position] |
| [product 2] | [desc] | $[X] | [position] |

## Position Assignments
### Order Bump → [product name]
- Congruence score: [X/10]
- Rationale: [why this product at this position]
- Congruence type: completeness / accelerator / insurance / exclusive

### Upsell 1 → [product name]
- Congruence score: [X/10]
- Rationale: [why this product at this position]
- Extension logic: [how it extends the mechanism]

### Downsell 1 → [product name OR reframe of upsell 1]
- Congruence score: [X/10]
- Reframe type: core_extract / payment_plan / lite_version / different_format
- Rationale: [why this reframe, not just cheaper]

## Unmapped Positions
- [any positions with no suitable product — flag gap]

## Total Sequence Length: [N positions]
```

## Constraints
- NEVER invent products. Only map products that actually exist.
- Minimum 2 positions must be mapped (order bump + 1 upsell)
- Every mapping must have explicit congruence rationale
