# 2.3 — Handoff Spec Generator

**Layer:** 2
**Model:** opus
**Depends on:** 2.1 + 2.2
**Produces:** `layer-2/2.3-handoff-specs.md`

---

## Purpose

Generate concrete, self-contained handoff specifications for U1 (Order Bump Writer), U2 (1-Click Upsell Writer), and U3 (Downsell Writer). Each spec must contain everything the downstream skill needs to write copy WITHOUT access to U0's full context.

## Why This Matters

U1, U2, and U3 may execute in different sessions, with different context windows. The handoff spec IS the brief. If it's vague, the copy will be vague. If it's missing congruence details, the copy will break congruence.

## Handoff Spec Template (Per Position)

### Required Fields

| Field | Description | Example |
|-------|-------------|---------|
| `position` | Where in sequence | "order_bump" / "upsell_1" / "downsell_1" |
| `product_name` | What's being offered | "Meal Plan Companion" |
| `product_description` | What it is in 2-3 sentences | "A 30-day meal plan..." |
| `price` | Exact price | "$9.95" |
| `word_count_target` | Copy length range | "50-150 words" |
| `front_end_product` | What they just bought | "Metabolism Reset Protocol" |
| `front_end_price` | What they just paid | "$47" |
| `front_end_mechanism` | Exact mechanism name | "Metabolic Switch" |
| `congruence_type` | How this connects | "completeness" |
| `mechanism_reference` | How to reference the mechanism | "Activates your Metabolic Switch through food" |
| `root_cause_callback` | How to reference root cause | "Because your metabolism has been..." |
| `promise_extension` | What this adds to the promise | "Get results in half the time" |
| `buyer_validation` | Opening reference to purchase | "You just secured the Metabolic Switch system" |
| `emotional_target` | What buyer should feel | "Effortless completion" |
| `tone_direction` | Voice/energy guidance | "Casual, celebratory, zero pressure" |
| `proof_elements` | What proof to include (if any) | ["87% of users who add this..."] |
| `guarantee` | Guarantee for this position | "Same 60-day guarantee as your main order" |
| `cta_text` | Suggested CTA | "Yes, add the Meal Plan for just $9.95" |
| `decline_text` | Suggested decline | "No thanks, I'll figure out meals on my own" |

### Additional Fields for Downsell

| Field | Description |
|-------|-------------|
| `reframe_type` | core_extract / payment_plan / lite_version / different_format |
| `reframe_angle` | How this is different from "just cheaper" |
| `acknowledge_text` | Suggested acknowledgment opening |

## Execution Steps

1. Read congruence map (2.1) and narrative arc (2.2)
2. Read product mapping (1.2) and pricing (1.3)
3. For each position, generate a complete handoff spec using the template
4. Validate: could a copywriter execute from this spec ALONE?
5. Write all specs to a single handoff document (organized by position)

```markdown
| Field | Value |
|-------|-------|

## Completeness Check
- [ ] Every position has all required fields populated
- [ ] Mechanism name is identical across all specs
- [ ] Each spec is self-contained (no references to "see above")
- [ ] Word count targets match position requirements
- [ ] Price anchoring is explicit in each spec
```
