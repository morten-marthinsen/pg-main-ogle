# 2.2 — Bonus Stack Builder

**Version:** 1.0
**Skill:** U2 — 1-Click Upsell Writer
**Layer:** 2
**Microskill:** 2.2 — Bonus Stack Builder
**Purpose:** Build the bonus stack for the Offer section — maximum 3 bonuses, each congruent with the upsell product, with perceived value exceeding the upsell price.

---

## INPUT

- `layer-0/0.1-input-context.md` (bonus inventory from U0 handoff or brief, upsell product, upsell price, vertical)
- `layer-1/1.1-congruence-map.md` (mechanism name, extension logic — bonuses must be congruent)
- `layer-1/1.4-structure-selection.md` (Offer section word allocation — bonuses must fit within it)
- `layer-0/0.2-specimen-calibration.md` (specimen bonus stack patterns)

## PROCESS

1. **Check for pre-defined bonuses.** Three possible states:
   - **U0 handoff includes bonuses** → Use those. U0 has already validated congruence and value. Skip to step 5 (formatting).
   - **U0 handoff or brief mentions bonus inventory but not specific selections** → Select from available inventory per steps 2-4.
   - **No bonus information provided** → Flag "NO BONUSES IN HANDOFF." Do NOT fabricate bonuses. Write output with "Bonus stack: none provided — flag for human decision." Proceed to step 7.

2. **Select bonuses from available inventory.** Rules:
   - **Maximum 3 bonuses.** Law 3 (Speed kills) — every additional bonus adds words to the Offer section and complexity to the decision. 3 is the ceiling. 1-2 is perfectly acceptable.
   - **Every bonus must be CONGRUENT** with the upsell product AND the FE mechanism. Test: does this bonus make the upsell product work better or more easily? If the bonus could be bundled with ANY product, it is generic filler.
   - **Each bonus must be a distinct value-add.** No two bonuses should serve the same function. A "Quick Start Guide" and a "Getting Started Checklist" are the same bonus with different names.

3. **Apply the bonus selection hierarchy** (from specimen analysis):
   - **Tier 1 — Accelerators:** Bonuses that make the upsell product work FASTER or EASIER. These are the strongest because they address the buyer's implicit question: "Will this be easy to use?" Examples: Quick Start Guide, Cheat Sheet, Video Walkthrough, Done-For-You Template.
   - **Tier 2 — Extenders:** Bonuses that add VALUE to the upsell product by extending its application. Examples: Advanced Techniques Guide, Additional Routines, Supplementary Materials, Companion Tracker.
   - **Tier 3 — Access:** Bonuses that provide COMMUNITY or SUPPORT access. Examples: Private Community Access, Monthly Q&A, Email Support, VIP Status. These are less tangible but create perceived ongoing value.
   - Select from the highest available tier first. Ideal: 1 Accelerator + 1 Extender, or 1 Accelerator + 1 Extender + 1 Access.

4. **Assign perceived dollar values.** For each bonus:
   - The perceived value must be BELIEVABLE. "$997 value" for a PDF checklist is absurd. "$29 value" for the same checklist is credible.
   - Pricing heuristics from specimens:
     - PDF guide / checklist: $19-$47
     - Video training: $47-$97
     - Template / done-for-you tool: $29-$67
     - Community access (monthly): $47-$97/month, presented as "$47/month value — yours FREE"
     - Tracker / journal: $19-$29
   - **Total perceived bonus value must exceed the upsell price.** This is deal framing — the bonuses alone are "worth more" than what they're paying. Target 2-4x the upsell price in total bonus value.
   - Specimen benchmarks: golf training upsells bundle $197-$593 in bonus value; supplement upsells use quantity savings + VIP access as bonuses with $100-$300 perceived value.

5. **Format each bonus for the Offer section.** Each bonus gets exactly:
   - Bonus name (branded if possible — "The 7-Minute Morning Activation Sequence" not "a morning routine guide")
   - One-sentence value proposition (what it does for the buyer, not what it is)
   - Perceived dollar value
   - Format in the output:
     ```
     - [Bonus Name] ([$format: video / PDF / template / access]) — $[value] value
       [One-sentence value proposition]
     ```

6. **Calculate the value stack total:**
   - Sum all bonus perceived values
   - Add the upsell product's anchor value (from price presentation)
   - Present as: "Total value: $[sum]. Your price today: $[upsell price]."
   - Value ratio should be 3-5x minimum (e.g., $197 total value for $47 price = 4.2x ratio)

7. **Write output file.** If no bonuses were provided or available, the output file states this explicitly and flags for human decision.

## CONSTRAINTS

1. **Maximum 3 bonuses.** This is a HARD CAP per Law 3 (Speed kills). The bonus stack exists to create value perception, not to overwhelm. A 6-bonus upsell stack reads like a front-end offer, not a quick post-purchase add-on. More than 3 = delete the weakest.
2. **NEVER invent bonuses that don't exist.** If the U0 handoff doesn't specify bonuses and no inventory is available, the upsell ships without a bonus stack. Write "none provided — flag for human." Do not fabricate "The Ultimate Guide to [Topic]" out of thin air.
3. **Every bonus must be congruent** with both the upsell product AND the FE mechanism. A "Stress Management Audio" bonus for a golf swing upsell is not congruent. A "Pre-Round Warm-Up Routine" bonus for a golf swing upsell is congruent.
4. **Perceived values must be believable.** If a buyer would laugh at the stated value, it undermines the entire value stack. A PDF is not worth $497. A video course might be worth $97. Calibrate to what the buyer would find credible.
5. **Total bonus perceived value must exceed the upsell price.** If it doesn't, the deal framing is inverted — the buyer is "paying more than it's worth." Minimum 2x ratio. Target 3-5x.
6. **One-sentence value propositions describe BENEFIT, not format.** "A 47-page PDF guide" is a format description. "The exact morning sequence our fastest responders use to double their results in the first week" is a benefit statement.
7. **This microskill runs in parallel with 2.3** (Price Presentation Builder). Its output is integrated into the CAIRO draft after both 2.2 and 2.3 are complete.
