# 0.1 — Input Loader

**Version:** 1.0
**Skill:** U2 — 1-Click Upsell Writer
**Layer:** 0
**Microskill:** 0.1 — Input Loader
**Purpose:** Load all upstream context, determine operating mode (A vs B), and extract the complete set of inputs required for CAIRO draft generation.

---

## PATH RESOLUTION

All upstream packages use the standardized output folder:

```
PRIMARY:  outputs/[project-code]/[filename]
FALLBACK: [skill-dir]/outputs/[filename]
```

Where `[project-code]` is the 2-4 letter campaign identifier (e.g., RS1, SPD, ULV).

## INPUT

- U0 handoff spec (`layer-2/2.3-handoff-specs.md` → U2 section) — REQUIRED for Mode A
- `mechanism-package.json` (from Skill 04) — REQUIRED for Mode A
- `campaign-brief-package.json` (from Skill 09) — RECOMMENDED — provides audience profile, tone, constraints
- `context-reservoir.md` (human-curated) — RECOMMENDED — contains VOC quotes, proof elements, customer language patterns, FSSIT intelligence, and arena reasoning that make upsell copy feel congruent with the front-end voice
- `soul.md` (project-level voice constraints) — RECOMMENDED
- `story-elements` (from Skill 12) — OPTIONAL
- `root-cause-package.yaml` (from Skill 03) — OPTIONAL
- `promise-package.json` (from Skill 05) — OPTIONAL
- Standalone brief (user-provided) — REQUIRED for Mode B

If operating in Mode A, the U0 handoff spec is the primary authority. If operating in Mode B, the standalone brief must contain all REQUIRED fields or GATE_0 fails.

## PROCESS

1. **Check project directory** for the U0 handoff spec file (`upsell-strategy.yaml` AND `layer-2/2.3-handoff-specs.md`).
2. **Determine operating mode:**
   - If U0 handoff spec EXISTS and contains a U2 section → Mode A
   - If U0 handoff spec is MISSING or has no U2 section → Mode B
   - Do NOT mix modes. One or the other. Log the decision.
3. **Mode A extraction** — from U0 handoff spec, extract:
   - Upsell product name and description
   - Upsell price (and payment plan options if specified)
   - Position in sequence (Upsell 1 or Upsell 2)
   - Format recommendation (video script / text page / unspecified)
   - Bonus inventory (if provided by U0)
   - Guarantee terms (mirror FE or upsell-specific)
   - Narrative arc notes (if provided)
   - Congruence bridge suggestion (if provided)
4. **Mode A extraction** — from mechanism-package.json, extract:
   - Mechanism name (EXACT — this is the congruence anchor)
   - Mechanism explanation summary (1-2 sentences)
   - Key proof elements tied to mechanism
5. **Mode A extraction** — from additional packages:
   - root-cause-package.yaml → root cause language (VERBATIM phrasing)
   - promise-output.json → promise statement (exact wording)
   - soul.md → voice register, anti-voice, pacing constraints
   - story-elements → narrative callbacks available for Reason section
6. **Mode A extraction** — from front-end context:
   - FE product name
   - FE price
   - FE promise (from promise-package or U0 handoff)
   - FE guarantee type and terms
7. **Mode B extraction** — from standalone brief, extract all of:
   - FE product name — REQUIRED
   - FE mechanism name — REQUIRED
   - FE root cause language — REQUIRED
   - FE promise — REQUIRED
   - FE price — REQUIRED
   - Upsell product name and description — REQUIRED
   - Upsell price — REQUIRED
   - Target audience description — REQUIRED
   - Format preference — OPTIONAL (default to text page)
   - Bonus inventory — OPTIONAL
   - Guarantee terms — OPTIONAL (default to mirror FE)
8. **Validate completeness:**
   - Mode A: U0 handoff spec loaded + mechanism name extracted = minimum viable
   - Mode B: All 8 REQUIRED fields present = minimum viable
   - If minimum not met → GATE_0 FAIL with specific missing fields listed
9. **Flag gaps** — for any OPTIONAL field not found, log it as a gap (do not fabricate data).
10. **Write output file** with all extracted data organized by category.

## CONSTRAINTS

1. Mode determination happens FIRST — before any package loading. Do not load Mode A packages and then "fall back" to Mode B mid-process.
2. Mechanism name must be extracted as an EXACT string, not summarized. If the mechanism is "The Metabolic Switch Protocol," record "The Metabolic Switch Protocol" — not "metabolic switching" or "the metabolic method."
3. Root cause language must be VERBATIM. If the FE says "hormonal disruption caused by environmental toxins," record that exact phrase.
4. Never fabricate data for missing fields. If bonus inventory is not in the handoff, write "none provided — flag for human." Do not invent bonuses.
5. Mode B outputs must carry the flag "Mode B — limited upstream context" in every downstream file.
6. If soul.md is not found, log the gap but do not fail. Voice constraints will default to specimen calibration.
