# 1.1 — Bump Copy Generator

**Version:** 1.0
**Skill:** U1 — Order Bump Writer
**Layer:** 1
**Microskill:** 1.1 — Bump Copy Generator
**Model:** sonnet
**Depends on:** 0.1 (Input Context), 0.2 (Specimen Calibration), GATE_0 PASS
**Produces:** `layer-1/1.1-initial-bumps.md`
**Purpose:** Generate 2-3 initial order bump versions using the classified template type. These are the raw candidates that 1.2 will expand into 5-7 variants.

---

## INPUT

- `layer-0/0.1-input-context.md` — product name, FE mechanism name, prices, congruence type, price anchor strategy
- `layer-0/0.2-specimen-calibration.md` — template type, structural patterns, opening pattern, congruence bridge, price presentation format, proof element recommendation

---

## PROCESS

1. **Confirm prerequisites:**
   - GATE_0 = PASS
   - Both Layer 0 output files exist and are complete
   - FE mechanism name is available as exact string

2. **Internalize constraints before writing a single word:**
   - 50-150 words. Count them. Every variant.
   - 3-element structure: WHAT + WHY NOW + PRICE. All three. Every variant.
   - FE mechanism name must appear. Verbatim. Every variant.
   - No story. No PAS. No proof cascade. No mini sales page.
   - Post-purchase tone. They already bought. You are extending, not selling.

3. **Generate Initial Version A — Template-Aligned:**
   - Use the template type from 0.2 as the primary frame:
     - **Completeness:** Lead with system completion. "You just secured [FE product]. Complete your [mechanism] system with [bump product]..."
     - **Accelerator:** Lead with speed/results. "You just secured [FE product]. Get results [X]x faster with [bump product]..."
     - **Insurance:** Lead with protection. "You just secured [FE product]. Protect your [mechanism] investment with [bump product]..."
     - **Exclusive:** Lead with scarcity. "Only available right now at checkout: [bump product]..."
   - Apply the 3-element structure:
     - **WHAT** (first 1-3 sentences): Product name + one-sentence value proposition. What does this add?
     - **WHY NOW** (next 1-2 sentences): Why add it at checkout? Momentum, special pricing, completeness, timing.
     - **PRICE** (final 1-2 sentences): Anchor price + discount price. Make it feel like a no-brainer.
   - Include FE mechanism name naturally within the copy
   - Include one proof element maximum (or zero — zero is fine for bumps)
   - End with binary choice: "Add to My Order" / "No thanks"
   - COUNT WORDS. If >150, cut. If <50, expand slightly. Target: 80-120 words (sweet spot).

4. **Generate Initial Version B — Alternate Angle:**
   - Use a DIFFERENT emphasis than Version A
   - If A led with congruence (system completion), B should lead with value (savings)
   - If A led with speed, B should lead with exclusivity (checkout-only)
   - Same 3-element structure
   - Same mechanism name inclusion
   - Same word count constraint
   - Different opening sentence, different WHY NOW angle, different price framing

5. **Generate Initial Version C (optional — if natural):**
   - Only if a third distinct angle presents itself naturally
   - Do NOT force a third version that's just a reword of A or B
   - If not generating C, note: "2 initial versions produced — 3rd angle deferred to 1.2 variant generation"

6. **Word count verification:**
   - Count every word in every version
   - Words = space-delimited tokens (hyphenated words = 1 word, "$9.95" = 1 word)
   - Record exact count
   - If ANY version is outside 50-150: rewrite it BEFORE outputting
   - Do not output a version that violates the word count and mark it "needs trimming" — that is a failure mode

7. **3-element verification:**
   - For each version, mark where WHAT, WHY NOW, and PRICE appear
   - If any element is missing, rewrite before outputting
   - The elements can be interwoven (they don't need to be in strict order) but all three must be present

8. **Write output file**

## CONSTRAINTS

- **Do NOT exceed 150 words on any version.** This is the hardest constraint in the entire skill. Most failures happen here. If you find yourself at 160 words, you are writing a sales page, not a bump. Start over with a tighter frame.
- **Do NOT skip the word count.** Count BEFORE writing the output file. If you didn't count, the output is invalid.
- **Do NOT use PAS structure.** If your Version A opens with "Are you frustrated by..." or "Most people struggle with...", delete it and start over. The buyer already bought. They accepted the problem.
- **Do NOT include more than ONE proof element per version.** A stat, a number, an authority name — pick one or zero. Not two. Not three.
- **Do NOT write the same version twice with different words.** Version A and Version B must have genuinely different emphasis angles, not cosmetic rewording.
- **Target the 80-120 word sweet spot.** Versions at 50-60 words often feel sparse. Versions at 140-150 are often bloated. The best bumps land between 80-120.
