# 0.1 — Input Loader

**Version:** 1.0
**Skill:** U1 — Order Bump Writer
**Layer:** 0
**Microskill:** 0.1 — Input Loader
**Model:** haiku
**Depends on:** Nothing (first microskill)
**Produces:** `layer-0/0.1-input-context.md`
**Purpose:** Load and validate all required inputs for order bump generation. Determine operating mode (A or B).

---

## PATH RESOLUTION

All upstream packages use the standardized output folder:

```
PRIMARY:  outputs/[project-code]/[filename]
FALLBACK: [skill-dir]/outputs/[filename]
```

Where `[project-code]` is the 2-4 letter campaign identifier (e.g., RS1, SPD, ULV).

## INPUT

### Context Reservoir (RECOMMENDED for all copy-generating skills)
- `context-reservoir.md` from `outputs/[project-code]/` — the human-curated intelligence file containing best VOC quotes, proof elements with full detail, and customer language patterns. Even for 50-150 word bump copy, the reservoir provides the exact customer language that makes bump copy feel congruent with the front-end.

### Mode A (Downstream from U0)
- U0 handoff spec: `layer-2/2.3-handoff-specs.md` from the U0 execution directory
- The handoff spec contains a section for the order_bump position with:
  - Product name
  - Product description (one sentence)
  - Price
  - Price ratio to FE
  - Congruence type (Completeness / Accelerator / Insurance / Exclusive)
  - FE mechanism name (EXACT string)
  - Copy angle suggestion
- Additionally from U0's output:
  - `upsell-strategy.yaml` — for FE product name, FE price, mechanism name confirmation
- Optional: `Soul.md` from project directory — anti-voice constraints

### Mode B (Standalone Brief)
- Human-provided brief containing:
  - Product name (the bump product)
  - Product description
  - Bump price
  - FE product name
  - FE price (for anchoring)
  - FE mechanism name
  - Target audience (1-2 sentences)
  - Congruence angle (how bump relates to FE)
- Optional: `Soul.md`

## PROCESS

1. **Determine operating mode:**
   - Check for U0 handoff spec at expected path
   - If handoff spec exists AND contains order_bump position section --> Mode A
   - If handoff spec missing OR no order_bump position --> Mode B
   - Record mode determination with rationale

2. **Mode A — Extract from U0 handoff:**
   - Open handoff spec, locate the order_bump position section
   - Extract each field into structured format:
     - `product_name`: exact string
     - `product_description`: one sentence
     - `bump_price`: numeric with currency
     - `fe_product_name`: from upsell-strategy.yaml
     - `fe_price`: from upsell-strategy.yaml
     - `fe_mechanism_name`: EXACT string — this is the congruence anchor
     - `price_ratio`: decimal (should be 0.10-0.30 per cascade rules)
     - `congruence_type`: one of {Completeness, Accelerator, Insurance, Exclusive}
     - `copy_angle`: U0's suggested angle (advisory, not binding)
   - Cross-reference mechanism name between handoff spec and upsell-strategy.yaml — they MUST match
   - Load Soul.md if present — extract anti-voice constraints only

3. **Mode B — Parse standalone brief:**
   - Extract same field set from human-provided brief
   - Flag any missing fields
   - Calculate price_ratio (bump_price / fe_price) — flag if outside 0.10-0.30 range
   - Infer congruence_type if not explicitly stated:
     - "complete" / "add-on" / "companion" language --> Completeness
     - "faster" / "accelerate" / "speed up" language --> Accelerator
     - "protect" / "insurance" / "guarantee" / "backup" language --> Insurance
     - "only now" / "exclusive" / "checkout only" language --> Exclusive
   - If cannot infer, default to Completeness (most common template)
   - Add flag: "Mode B — congruence not pre-validated by U0"

4. **Validate required fields:**
   - REQUIRED (both modes): product_name, bump_price, fe_product_name, fe_price, fe_mechanism_name
   - RECOMMENDED: product_description, congruence_type
   - OPTIONAL: copy_angle, Soul.md anti-voice
   - If any REQUIRED field is missing --> GATE_0 = FAIL with specific missing field(s)

5. **Calculate price anchor spread:**
   - Determine the "value" or "retail" price for anchor display
   - If U0 provides a value price, use it
   - If not, calculate: `anchor_price = bump_price * 3-7x` (standard anchor range)
   - Or use FE price as anchor context: "Just $X more to complete your [mechanism]"
   - Record anchor strategy for Layer 1

6. **Write output file**
