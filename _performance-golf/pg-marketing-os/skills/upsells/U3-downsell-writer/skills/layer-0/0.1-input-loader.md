# 0.1 — Input Loader

**Version:** 1.0
**Skill:** U3 — Downsell Writer
**Microskill:** 0.1 — Input Loader
**Layer:** 0
**Model:** haiku
**Input:** U0 handoff spec (`upsell-strategy.yaml`), U2 downstream handoff (from `upsell-page-draft.md`), `mechanism-package.json` (from Skill 04), `context-reservoir.md` (human-curated), `soul.md` (project-level)
**Output:** `layer-0/0.1-input-context.md`

---

## PATH RESOLUTION

All upstream packages use the standardized output folder:

```
PRIMARY:  outputs/[project-code]/[filename]
FALLBACK: [skill-dir]/outputs/[filename]
```

Where `[project-code]` is the 2-4 letter campaign identifier (e.g., RS1, SPD, ULV).

## PURPOSE

Load all upstream context required for downsell generation, determine operating mode (A vs B), and extract the complete set of inputs needed for ARO draft generation. Mode A (full upstream from U0 + U2) produces stronger output because it has the declined upsell's full context and U0's reframe_type recommendation. Mode B (standalone brief) is inherently less rich.

---

## EXECUTION STEPS

1. **Check project directory** for the U0 handoff spec file (`upsell-strategy.yaml`) and the U2 output file (`upsell-page-draft.md` with its DOWNSTREAM HANDOFF section).

2. **Determine operating mode FIRST — before any package loading:**
   - If U0 handoff spec EXISTS and contains a U3/downsell section, AND U2 output EXISTS with a downstream handoff section → **Mode A**
   - If either U0 handoff spec is MISSING or has no U3 section, OR U2 downstream handoff is MISSING → **Mode B**
   - Do NOT mix modes. One or the other. Log the decision with rationale.
   - Do NOT load Mode A packages and then "fall back" to Mode B mid-process.

3. **Mode A extraction — from U0 handoff spec:**
   - Downsell product name and description
   - Downsell price
   - Position in sequence (downsell_1 after upsell_1 decline, or downsell_2 after upsell_2 decline)
   - `reframe_type` recommendation from U0 (Core Extract / Payment Plan / Lite Version / Different Format)
   - Front-end product name, price, mechanism name, root cause language, promise
   - Pricing cascade context (FE price → upsell price → downsell price)

4. **Mode A extraction — from U2 downstream handoff:**
   - Declined upsell product name
   - Declined upsell price
   - Declined upsell angle (what value proposition was the upsell framed on)
   - Declined upsell bonuses (what was included)
   - Declined upsell guarantee type
   - Mechanism name (cross-check against U0 — must match)
   - Root cause language (VERBATIM — cross-check against U0)
   - Promise extension (how the upsell extended the FE promise)
   - `reframe_suggestions` from U2 (specific suggestions for how to reframe)

5. **Mode A extraction — from mechanism-package.json:**
   - Mechanism name (EXACT string — this is the congruence anchor)
   - Mechanism explanation summary (1-2 sentences)
   - Cross-check: mechanism name from mechanism-package MUST match U0 handoff and U2 downstream handoff. If mismatch, log and use mechanism-package as canonical source.

6. **Mode A extraction — from context-reservoir.md (RECOMMENDED):**
   - Best VOC quotes (for authentic customer language in the reframe)
   - Proof elements with full detail (for lightweight proof in the downsell)
   - Customer language patterns (for voice congruence with front-end)
   - FSSIT intelligence (for understanding the emotional state of a buyer who just declined)
   - If not found: log gap, proceed without. The downsell will rely on compressed package data only.

7. **Mode A extraction — from soul.md:**
   - Anti-voice constraints (what the copy must NOT sound like)
   - Pacing constraints
   - Note: persona voice loading is NOT used for U3 (300-1000w is too short for voice differentiation). Soul.md provides anti-voice guardrails only.

8. **Mode B extraction — from standalone brief, extract ALL of:**
   - FE product name — REQUIRED
   - FE mechanism name (EXACT) — REQUIRED
   - FE root cause language (VERBATIM) — REQUIRED
   - FE promise — REQUIRED
   - FE price — REQUIRED
   - Declined upsell product name — REQUIRED
   - Declined upsell price — REQUIRED
   - Declined upsell angle — REQUIRED
   - Downsell product name and description — REQUIRED
   - Downsell price — REQUIRED
   - Reframe type (Core Extract / Payment Plan / Lite Version / Different Format) — REQUIRED
   - Position (downsell_1 / downsell_2) — OPTIONAL (default to downsell_1)

9. **Validate completeness:**
   - Mode A minimum viable: U0 handoff spec loaded + U2 downstream handoff loaded + mechanism name extracted
   - Mode B minimum viable: All 11 REQUIRED fields present
   - If minimum not met → **GATE_0 FAIL** with specific missing fields listed

10. **Cross-validate pricing cascade:**
   - FE price < upsell price (or equal, in some funnels)
   - Downsell price < declined upsell price — MANDATORY. If downsell price >= upsell price, flag IMMEDIATELY. Pricing cascade is broken. Cannot proceed.

11. **Flag gaps** — for any field not found, log it as a gap. Do NOT fabricate data. If reframe_suggestions are not in U2 handoff, write "not provided — Layer 1 will determine reframe independently."

12. **Write output file** with all extracted data organized by category.

## CONSTRAINTS

- Mode determination happens FIRST — before any package loading. Do not load Mode A packages and then "fall back" to Mode B mid-process.
- Mechanism name must be extracted as an EXACT string, not summarized. If the mechanism is "The Metabolic Switch Protocol," record "The Metabolic Switch Protocol" — not "metabolic switching" or "the metabolic method."
- Root cause language must be VERBATIM. If the FE says "hormonal disruption caused by environmental toxins," record that exact phrase. No paraphrasing.
- Never fabricate data for missing fields. If reframe_suggestions are not in the U2 handoff, write "not provided." Do not invent suggestions.
- Mode B outputs must carry the flag "Mode B — limited upstream context" in every downstream file.
- If soul.md is not found, log the gap but do not fail. Anti-voice constraints will be relaxed.
- Cross-validate pricing cascade. If downsell price >= upsell price, flag immediately. Do not proceed with broken pricing.

---

## FAILURE MODES

| Failure | Detection | Response |
|---------|-----------|----------|
| Missing U0 handoff spec (Mode A) | File not found in project directory | Switch to Mode B OR request handoff data from human |
| Missing U2 downstream handoff (Mode A) | U2 output file has no DOWNSTREAM HANDOFF section | Switch to Mode B OR request handoff data from human. Cannot write downsell without knowing what was declined. |
| Mechanism name mismatch between sources | mechanism-package, U0, and U2 report different names | Use mechanism-package as canonical. Log discrepancy. Flag for human review. |
| Pricing cascade broken (downsell >= upsell) | Price comparison in step 9 | HALT. Flag to human. Cannot fix pricing locally — this is a U0 strategy issue. |
| Mode B missing REQUIRED fields | Completeness check in step 8 fails | GATE_0 FAIL. List specific missing fields. Request from human. |
| Root cause language paraphrased | Comparison between sources shows different wording | Use the earliest canonical source (mechanism-package or U0). Flag discrepancy. |
