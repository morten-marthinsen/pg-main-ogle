**Output:** `layer-0/0.2-specimen-calibration.md`

---

## PURPOSE

Load and analyze available downsell specimen data to calibrate ARO proportions, tone patterns, reframe patterns, and price presentation patterns for this specific downsell. This microskill operates under a KNOWN LIMITATION: only 1 downsell specimen (US-16) exists in the vault, and its FILE IS MISSING. Calibration therefore relies on index metadata, adapted U2 patterns, and the ARO structure defined in UPSELL-ENGINE-CLAUDE.md.

---

## EXECUTION STEPS

1. **Load the specimen index** from `UPSELL-specimen-index.md`. Locate US-16 (the only downsell specimen). Extract all available metadata: product name, vertical, position, word count, structure notes, reframe type used, any section proportions recorded.

2. **Acknowledge the specimen gap explicitly.** US-16 is the only downsell specimen and its FILE IS MISSING. This means:
   - No verbatim copy to analyze for tone, phrasing, or transition patterns
   - No full ARO structure to reverse-engineer section proportions from
   - Calibration must be derived, not observed
   - This gap must be flagged in the output file

3. **Load ARO structure definition** from `UPSELL-ENGINE-CLAUDE.md` Section "DOWNSELL ARCHITECTURE (U3)":
   - Acknowledge: 50-100 words
   - Reframe: 150-500 words
   - Offer + Choice: 100-200 words
   - Total: 300-1000 words (position-dependent)

4. **Load ARO section proportions** from `U3-DOWNSELL-WRITER-AGENT.md` Section "ARO SECTION PROPORTIONS (By Position)":
   - Downsell 1: A (50-100w) + R (250-500w) + O (100-200w) = 400-800w
   - Downsell 2: A (50-75w) + R (150-300w) + O (75-150w) = 275-525w

5. **Adapt U2 specimen patterns for ARO.** Load relevant U2 calibration insights and adapt:
   - **Tone adaptation:** U2 Congratulate is celebratory/warm → U3 Acknowledge is validating/empathetic. Shift from "You made a great decision!" to "I completely understand your hesitation."
   - **Structure adaptation:** U2 CAIRO has 5 sections → U3 ARO has 3 sections. The Reframe section carries the weight that U2 distributes across Amplify + Intrigue + Reason. Reframe needs the narrative density of those 3 sections compressed into 1.
   - **Proof adaptation:** U2 allows 2 proof elements → U3 allows 1 MAX. If U2 specimens show proof-heavy patterns, note that downsell must strip to single strongest proof.
   - **CTA adaptation:** U2 binary CTA → U3 binary CTA but CLEANER. U3 NO option is the cleanest in the funnel. Adapt U2 CTA patterns with even less friction.
   - **Price adaptation:** U2 anchors against FE price → U3 anchors against DECLINED UPSELL price. Different anchor, same technique.

6. **Extract reframe patterns** from available data:
   - From UPSELL-ENGINE-CLAUDE.md: 4 reframe types with descriptions and "works when" conditions
   - From U3-DOWNSELL-WRITER-AGENT.md: Teaching Foundations section on 4 Reframe Types with full examples
   - From US-16 index metadata: what reframe type was used (if recorded)
   - Compile: which reframe types have the strongest structural guidance, which have the least

7. **Extract Acknowledge section patterns.** This is the most trust-sensitive copy in the funnel. From the AGENT.md Teaching Foundations section:
   - Good pattern: "I completely understand. The [Upsell Product] isn't for everyone, and there's absolutely nothing wrong with starting with just the [FE Product]. It's a powerful [mechanism] on its own."
   - Bad pattern: "Wait! Before you go — you're about to miss out on the biggest upgrade we've ever offered."
   - Core rules: validate hesitation, name what they declined, affirm their autonomy, zero guilt, zero pressure, zero FOMO

8. **Synthesize calibration targets:**
   - ARO proportion targets (by position) with sources
   - Tone register: warm, understanding, zero-pressure (adapted from U2 celebratory → U3 empathetic)
   - Energy trajectory: calm opening → building alternative angle → clean close (NOT excitement → urgency)
   - Reframe language patterns: "I completely understand," "there's nothing wrong with," "but before you go, I want to make sure you know about," "a different way to get started"
   - Price presentation: declined upsell as anchor, downsell with contrast, per-day if applicable
   - CTA: cleanest binary in funnel, NO option affirms original purchase

9. **Write output file** with all calibration data and the specimen gap flag.

## CONSTRAINTS

- The specimen gap must be EXPLICITLY flagged in the output. Do not present derived calibration as if it came from observed specimens.
- U2 patterns are ADAPTED, not directly applied. CAIRO is not ARO. Congratulate is not Acknowledge. The psychological context is fundamentally different (post-purchase vs post-decline).
- ARO proportion targets from the AGENT.md are the authoritative source. Do not override them with U2 CAIRO proportions.
- When downsell specimens become available in the vault, this microskill spec should be updated to include direct specimen loading and analysis (similar to U2's 0.2).
- Do not fabricate specimen data. If US-16 metadata is sparse, report what exists and flag what is missing.
- Specimen calibration informs generation — it does not replace the AGENT.md structure. Where calibration conflicts with AGENT.md constraints, AGENT.md wins.

---

## FAILURE MODES

| Failure | Detection | Response |
|---------|-----------|----------|
| US-16 index entry not found | UPSELL-specimen-index.md has no US-16 | Proceed with zero specimen data. Calibrate entirely from ARO structure + U2 adaptation. Flag severity as HIGH. |
| UPSELL-specimen-index.md not found | File missing from specimens directory | HALT. Cannot calibrate without index. Request file location from human. |
| U2 specimen calibration not available | No prior U2 run in project | Calibrate from ARO structure definition only. Reduce adaptation step to ARO-only sources. Flag gap. |
| Calibration presented as observed (not derived) | Output does not flag specimen gap | Re-write output with explicit SPECIMEN GAP section at top. This is the primary integrity constraint. |
