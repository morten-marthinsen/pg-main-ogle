# Layer 2.3 — P.S. Strategy

## Purpose

Add P.S. lines to emails strategically. P.S. lines serve as secondary CTAs, urgency reminders, testimonial reinforcements, cross-sell mentions, or curiosity teasers for the next email. Not every email gets one — maximum 60% of emails in the sequence.

**Critical Rule:** P.S. lines are appended AFTER the sign-off. They are the second and final type of new text E3 writes (along with connectors). No email body modification.

## Execution Protocol

### Step 1: Calculate P.S. Budget

```
total_emails = blueprint_context.total_emails
max_ps_count = floor(total_emails * 0.60)

# For common sequence lengths:
# 5 emails → max 3 P.S.
# 7 emails → max 4 P.S.
# 10 emails → max 6 P.S.
# 15 emails → max 9 P.S.
# 1 email → max 0 P.S. (see exclusion rules)
```

### Step 2: Apply Exclusion Rules

Certain emails NEVER get P.S. lines:

```
ALWAYS EXCLUDE:
  - Position 1 (first email — let it stand alone)
  - NR (Negative Response) body type emails (tone clash)

NEVER EXCLUDE:
  - Last 2 emails of PL/AL sequences (urgency P.S. mandatory)
```

### Step 3: Assign P.S. Types by Position

For each eligible email, determine the best P.S. type:

```
P.S. TYPE SELECTION MATRIX:

IF email is in last 2 positions AND campaign_type IN [PL, AL]:
  → ps_type = "urgency_reminder"
  → PRIORITY: MANDATORY (counts against budget)

IF email.body_type == "TM":
  → ps_type = "testimonial_snippet" (reinforce with another testimonial)
  → PRIORITY: HIGH

IF email.urgency_level >= 3:
  → ps_type = "urgency_reminder"
  → PRIORITY: HIGH

IF next_email exists AND next_email has a compelling tease:
  → ps_type = "curiosity_tease"
  → PRIORITY: MEDIUM

IF offer has bonuses not yet mentioned:
  → ps_type = "bonus_mention"
  → PRIORITY: MEDIUM

IF product has related cross-sell:
  → ps_type = "cross_sell"
  → PRIORITY: LOW

DEFAULT (if no strong match):
  → Skip P.S. for this email
```

### Step 4: Prioritize and Budget

```
# Sort all P.S. candidates by priority
candidates = sort(ps_candidates, key=priority, reverse=true)

# Take top candidates up to budget
selected_ps = candidates[:max_ps_count]

# Verify exclusion rules respected
FOR each selected in selected_ps:
  IF selected.position == 1: REMOVE
  IF emails[selected.position].body_type == "NR": REMOVE
```

### Step 5: Write P.S. Text

For each selected P.S., write the actual text:

```
TEMPLATES BY TYPE:

bonus_mention:
  "P.S. I almost forgot — when you [action] today, you also get [bonus name].
   [One sentence about what the bonus does and why it matters]."

urgency_reminder:
  "P.S. [Deadline mechanism] [timeframe]. After that, [specific consequence].
   [Link/CTA text]."

testimonial_snippet:
  "P.S. [Name] from [location] just told me: '[Brief quote — 1-2 sentences max].'
   [Optional: one-sentence reaction]."

cross_sell:
  "P.S. If you already have [related product], you should know about [this product].
   [One sentence about the connection/synergy]."

curiosity_tease:
  "P.S. Tomorrow I'm going to share [specific tease about next email's content].
   [Optional: why it matters — one sentence]."
```

### Step 6: Validate P.S. Frequency

```
ps_count = len(selected_ps)
ps_frequency = ps_count / total_emails

IF ps_frequency > 0.60:
  REMOVE lowest-priority P.S. lines until frequency <= 0.60
  LOG: "P.S. frequency exceeded cap. Removed {n} P.S. lines."

# Final validation
ASSERT ps_frequency <= 0.60
```

### Step 7: Insert P.S. Lines

```
FOR each selected_ps:
  target_email = compiled_sequence.emails[selected_ps.position]
  target_email.ps_added = true
  target_email.ps_type = selected_ps.ps_type
  target_email.ps_text = selected_ps.text

  # P.S. is appended AFTER the sign-off in the email body
  # Final assembly in output-packager handles the physical placement
```

---

## Failure Modes

| Failure | Remediation |
|---------|-------------|
| P.S. frequency exceeds 60% | Remove lowest-priority P.S. lines until within cap. |
| P.S. added to first email | Remove. First email never gets P.S. |
| P.S. added to NR body type email | Remove. Tone clash with NR format. |
| Last 2 emails of launch sequence lack urgency P.S. | Add urgency P.S. — this is mandatory for PL/AL. |
| P.S. text is longer than 3 sentences | Trim to 2 sentences max. P.S. is supplementary, not a second email. |
| All P.S. lines are the same type | Diversify — use at least 2 different P.S. types in any 5+ email sequence. |
| Curiosity tease references content not in next email | Rewrite to tease actual next-email content, or change to different P.S. type. |
