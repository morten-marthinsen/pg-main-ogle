# Layer 1.3 — Escalation Mapper

## Purpose

Map the urgency and emotional escalation across the assembled sequence. Verify that the sequence follows the blueprint's emotional arc and urgency plan. Produce an escalation map that shows how intensity progresses from email 1 through the final email.

---

## Input Requirements

```yaml
from_layer_0:
  blueprint_context:
    email_plan:
      - position: integer
        urgency_level: integer  # 0-5
        emotional_target: string
        function: string
    emotional_arc:
      summary: string
      phases:
        - phase: string
          emails: [integer]
          emotional_range: string
    urgency_plan:
      mechanism: string
      escalation:
        - day: integer
          level: integer
          language: string

  drafts_index:
    drafts:
      - position: integer
        body_type: string
        function: string

from_layer_1:
  timing_plan:
    schedule:
      - day: integer
        email_assignments:
          - position: integer
```

---

## Execution Protocol

### Step 1: Build Urgency Progression

Map urgency levels across the sequence using blueprint assignments:

```
urgency_progression = []
FOR each email in blueprint.email_plan:
  urgency_progression.append({
    position: email.position,
    day: email.day (from timing_plan),
    urgency_level: email.urgency_level,
    function: email.function
  })
```

### Step 2: Validate Urgency Escalation Pattern

Check that urgency follows the expected pattern for the campaign type:

```
IF campaign_type IN [PL, AL]:
  # Launch sequences MUST escalate
  first_third = urgency_progression[:total_emails // 3]
  last_third = urgency_progression[-(total_emails // 3):]

  first_third_avg = avg(u.urgency_level for u in first_third)
  last_third_avg = avg(u.urgency_level for u in last_third)

  urgency_escalates = last_third_avg > first_third_avg
  IF NOT urgency_escalates:
    escalation_warning = "Launch sequence does not escalate urgency. First third avg: {first}, last third avg: {last}."

IF campaign_type IN [AR, DR, CP]:
  # Non-launch sequences should be relatively flat or gently ascending
  max_urgency = max(u.urgency_level for u in urgency_progression)
  IF max_urgency > 3:
    escalation_warning = "Non-launch sequence has urgency level {max_urgency}. Expected max 3 for {campaign_type}."

IF campaign_type IN [BP, DU]:
  # Single emails — no escalation check needed
  escalation_warning = null
```

### Step 3: Build Emotional Progression

Map emotional targets across the sequence:

```
emotional_progression = []
FOR each email in blueprint.email_plan:
  emotional_progression.append({
    position: email.position,
    emotional_target: email.emotional_target,
    phase: determine_phase(email.position, blueprint.emotional_arc.phases)
  })
```

### Step 4: Validate Emotional Arc

Check that the emotional arc follows the blueprint phases:

```
FOR each phase in blueprint.emotional_arc.phases:
  phase_emails = [e for e in emotional_progression if e.position in phase.emails]
  FOR each email in phase_emails:
    IF email.emotional_target NOT IN expected_emotions_for_phase(phase.phase):
      arc_mismatch = {
        position: email.position,
        expected_phase: phase.phase,
        actual_target: email.emotional_target,
        expected_range: phase.emotional_range
      }
```

Expected emotions by phase:
```
trust_building: [trust, credibility, relatability, empathy]
education: [curiosity, understanding, revelation, insight]
desire_activation: [desire, aspiration, possibility, hope]
urgency_escalation: [urgency, fear_of_missing, consequence, deadline]
```

### Step 5: Identify Escalation Inflection Points

Find where the sequence transitions between phases:

```
inflection_points = []
FOR i in range(len(emotional_progression) - 1):
  current_phase = emotional_progression[i].phase
  next_phase = emotional_progression[i + 1].phase
  IF current_phase != next_phase:
    inflection_points.append({
      between_positions: [i + 1, i + 2],
      from_phase: current_phase,
      to_phase: next_phase,
      connector_recommended: true
    })
```

### Step 6: Generate Escalation Summary

```
escalation_summary = {
  urgency_pattern: describe_urgency_shape(urgency_progression),
  emotional_pattern: describe_emotional_shape(emotional_progression),
  inflection_count: len(inflection_points),
  peak_urgency_position: position_of_max_urgency,
  peak_urgency_day: day_of_max_urgency
}
```

---

## Failure Modes

| Failure | Remediation |
|---------|-------------|
| Launch sequence does not escalate urgency | Flag as warning for human review. Not an automatic HALT — blueprint may have a valid reason. |
| Non-launch sequence has high urgency early | Flag as warning. Urgency in first 2/3 violates E0 design rules. |
| Emotional arc has no progression (all same target) | Flag as warning. Flat emotional arcs reduce campaign effectiveness. |
| Phase mismatches detected | Log each mismatch. Not a HALT unless majority of emails mismatch. |
| No inflection points in a 7+ email sequence | Flag as warning. May indicate flat, monotone campaign. |
