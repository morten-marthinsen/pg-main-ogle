> **Inputs:** `email_context.assignment.body_type` from 0.1
> **Outputs:** `structural_specimens` object with verbatim specimen text held in active context

---

## FUNCTION

Load Ben Settle structural specimens that match the assigned body type for this email. These specimens serve as "statistical attractors" -- exact token sequences held in active context during generation that reshape probability distributions toward elite email patterns. Specimens are loaded VERBATIM, never summarized or paraphrased.

**Key Principle:** The probability shift toward elite email patterns requires EXACT token sequences. Summarized or paraphrased specimens do not produce the same quality uplift. Every specimen must be loaded word-for-word.

## LOADING PROTOCOL

```
EXECUTE 0.2-specimen-loader:

  STEP 1: Read Specimen Index
    1. READ specimens/specimen-index.md
    2. IDENTIFY available specimens for assigned body type
    3. IF specimen-index.md missing:
       FAIL: "Specimen index not found. Cannot load specimens."

  STEP 2: Select Specimens
    1. IDENTIFY assigned body type from email_context.assignment.body_type
    2. LOCATE specimen directory for that body type
    3. SELECT 3-5 specimens (prioritize variety in angle, tone, niche)
       - For types with 5+ specimens: prioritize author variety (mix Settle + legendary copywriters)
       - For NR (4 available): load all 4
    4. IF fewer than 3 specimens available:
       WARN: "Only [N] specimens available for [body_type]. Loading all."

  STEP 3: Load Verbatim
    FOR each selected specimen:
      1. READ full specimen file
      2. STORE complete text in active context -- DO NOT SUMMARIZE
      3. EXTRACT structural metadata:
         - Opening technique (how the first 3 lines work)
         - Body structure (how content is organized)
         - Bridge technique (how content transitions to pitch)
         - CTA pattern (how the call-to-action is structured)
         - Word count
         - Paragraph structure (average sentences per paragraph)
      4. VERIFY specimen is body-type-matched (not misclassified)

  STEP 4: Validate Loading
    1. CONFIRM 3+ specimens loaded
    2. CONFIRM all specimens are for correct body type
    3. CONFIRM specimens are in active context (not just metadata)
    4. SET structural_specimens.loaded = true

  STEP 5: Return structural_specimens
```

## WHAT TO EXTRACT FROM EACH SPECIMEN

For each loaded specimen, capture these structural patterns while holding the full text:

```yaml
specimen_analysis:
  specimen_id: "[filename]"
  body_type: "[confirmed type]"
  word_count: [number]

  structural_patterns:
    opening:
      technique: "[bold claim | quote block | attribution | question | setup | story hook | troll display]"
      line_count: [number]
      hook_mechanism: "[what makes the opening work]"

    body:
      structure: "[stack-and-dismiss | riff | testimonial | answer | list | narrative | demolition]"
      paragraph_avg_sentences: [number]
      content_sections: [number]

    bridge:
      technique: "[insight pivot | lesson connect | commentary | synthesis | lesson extraction]"
      pivot_sentence: "[the exact sentence where content becomes pitch]"
      smoothness: "[how natural the transition feels]"

    cta:
      pattern: "[direct | soft | urgent | curious | challenge]"
      sentence_count: [number]
      sign_off_style: "[name only | name + title | informal]"

    ratio:
      estimated_content_percent: [number]
      estimated_pitch_percent: [number]
```

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Specimen directory empty | HALT -- Cannot generate without specimens |
| CRITICAL | Specimen index missing | HALT -- Verify specimen infrastructure exists |
| CRITICAL | Zero specimens for assigned body type | HALT -- Body type requires specimens |
| DEGRADED | Fewer than 3 specimens available | WARN -- Proceed with available specimens |
| PROTOCOL VIOLATION | Specimens summarized instead of verbatim | HALT -- Reload VERBATIM |

---

## ANTI-PATTERNS (FORBIDDEN)

1. **Summarizing specimens** -- VERBATIM text only. Summaries lose the statistical attractor effect.
2. **Loading wrong body type** -- CT specimens for a QO email is a structural violation.
3. **Extracting metadata only** -- Metadata is secondary. FULL TEXT must be in active context.
4. **Loading fewer than 3 without warning** -- If under 3, this must be logged and flagged.
5. **Loading specimens after generation starts** -- Specimens must be loaded BEFORE any email generation begins.
