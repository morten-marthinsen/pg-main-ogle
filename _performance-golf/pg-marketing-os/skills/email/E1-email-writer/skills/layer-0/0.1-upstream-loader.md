# 0.1-upstream-loader.md

> **Layer:** 0 (Loading)
> **Skill:** E1-email-writer
> **Purpose:** Load campaign blueprint from E0, identify current email position, load upstream packages (Mode A) or brief context (Mode B)
> **Inputs:** E0 campaign-blueprint.yaml, upstream skill packages (Mode A) or brief (Mode B)
> **Outputs:** `email_context` object with blueprint data, email position, body type assignment, and available content

---

## FUNCTION

Load the campaign blueprint from E0-email-strategist. Identify which email in the sequence is being written (the current position). Extract the body type assignment, word count target, CTA pattern, and content direction for this specific email. In Mode A (full pipeline), also load upstream packages from Skills 01-09. In Mode B (brief-only), load brief context directly.

---

## REQUIRED INPUTS

### Campaign Blueprint (CRITICAL -- must exist)

| Field | Source | Purpose |
|-------|--------|---------|
| `campaign_blueprint.yaml` | E0-email-strategist | Full campaign sequence with per-email assignments |
| `current_position` | User instruction or sequential execution | Which email to write (e.g., "email-03") |

### Upstream Packages (Mode A -- Full Pipeline)

| Package | Source Skill | Critical Fields |
|---------|--------------|-----------------|
| `campaign-brief.json` | 09-campaign-brief | `big_idea`, `creative_direction`, `avatar` |
| `big-idea-output.json` | 06-big-idea | `central_idea`, `big_idea_type`, `mechanism_angle` |
| `mechanism-package.json` | 04-mechanism | `name`, `uniqueness_claim` |
| `promise-output.json` | 05-promise | `primary_promise`, `promise_ceiling` |
| `root-cause-package.yaml` | 03-root-cause | `three_part_structure`, `villain` |
| `research FINAL_HANDOFF.md` | 01-research | Quotes, pain points, audience voice |

### Brief Context (Mode B -- Brief-Only)

| Input | Source | Critical Fields |
|-------|--------|-----------------|
| Project brief | User-provided | Product, audience, positioning, pain points, desired outcomes |

---

## LOADING PROTOCOL

```
EXECUTE 0.1-upstream-loader:

  STEP 1: Load Campaign Blueprint
    1. LOCATE campaign-blueprint.yaml from E0 outputs
    2. PARSE full campaign structure
    3. IF blueprint missing:
       FAIL: "Campaign blueprint not found. Execute E0-email-strategist first."

  STEP 2: Identify Current Email Position
    1. READ current_position (from user instruction or sequential execution)
    2. LOCATE this position in blueprint.email_sequence[]
    3. EXTRACT:
       - body_type: "[CT|QO|TM|QA|LB|ST|NR]"
       - word_count_target: [number]
       - cta_pattern: "[A|B|C|D|E]"
       - content_direction: "[brief description of what this email should cover]"
       - sequence_context: "[what came before, what comes next]"
    4. IF position not found in blueprint:
       FAIL: "Email position [X] not found in campaign blueprint."

  STEP 3: Determine Mode
    1. CHECK: Do upstream packages from Skills 01-09 exist?
    2. IF YES: Mode A -- load upstream packages
    3. IF NO: Mode B -- load brief context only

  STEP 4: Load Content (Mode-Dependent)
    IF Mode A:
      1. Load campaign-brief.json
         EXTRACT: big_idea, creative_direction, avatar, voice_direction
      2. Load big-idea-output.json
         EXTRACT: central_idea, big_idea_type, mechanism_angle
      3. Load mechanism-package.json
         EXTRACT: name, uniqueness_claim
      4. Load promise-output.json
         EXTRACT: primary_promise, promise_ceiling
      5. Load root-cause-package.yaml
         EXTRACT: three_part_structure, villain
      6. Load research FINAL_HANDOFF.md (if available)
         EXTRACT: Key quotes, audience voice samples, pain language

    IF Mode B:
      1. Load project brief
         EXTRACT: product_info, audience_profile, positioning, pain_points, desired_outcomes
      2. SET degraded_mode = true
      3. LOG: "Mode B active -- no upstream packages. Email quality ceiling reduced."

  STEP 5: Assemble email_context
  STEP 6: Return email_context
```

## OUTPUT STRUCTURE

```yaml
email_context:
  mode: "A|B"
  blueprint:
    campaign_name: "[name]"
    total_emails: [number]
    current_position: "[email-XX]"
    position_index: [number]  # 1-based

  assignment:
    body_type: "[CT|QO|TM|QA|LB|ST|NR]"
    body_type_full: "[Contrarian|Quote-Opener|Testimonial|Q&A|List-Based|Story|Negative Response]"
    word_count_target: [number]
    cta_pattern: "[A|B|C|D|E]"
    content_direction: "[what this email should cover]"

  sequence_context:
    previous_email: "[body type and topic of previous email, or 'none' if first]"
    next_email: "[body type and topic of next email, or 'none' if last]"
    campaign_phase: "[awareness|engagement|conversion|retention]"

  upstream:  # Mode A only
    big_idea: "[central idea]"
    mechanism: "[mechanism name]"
    promise: "[primary promise]"
    root_cause: "[three-part structure summary]"
    avatar_pain: "[primary pain]"
    avatar_desire: "[primary desire]"
    voice_direction: "[tone/style guidance]"

  brief:  # Mode B only
    product: "[product info]"
    audience: "[audience profile]"
    positioning: "[positioning statement]"
    pain_points: ["pain1", "pain2"]
    desired_outcomes: ["outcome1", "outcome2"]

  loading_status:
    blueprint_loaded: true
    position_identified: true
    body_type_assigned: true
    mode: "A|B"
    degraded: [true/false]
    warnings: []
```

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Campaign blueprint missing | HALT -- Execute E0 first |
| CRITICAL | Email position not in blueprint | HALT -- Verify position exists |
| CRITICAL | Body type not assigned for position | HALT -- Check blueprint completeness |
| DEGRADED | Mode B (no upstream packages) | Proceed but flag quality ceiling |
| DEGRADED | Research FINAL_HANDOFF missing | Proceed without audience voice data |
