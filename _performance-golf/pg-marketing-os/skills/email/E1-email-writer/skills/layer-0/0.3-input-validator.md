# 0.3-input-validator.md

> **Layer:** 0 (Loading)
> **Skill:** E1-email-writer
> **Purpose:** Validate all inputs are present and meet minimum thresholds before proceeding to Layer 1
> **Inputs:** `email_context` from 0.1, `structural_specimens` from 0.2, persona voice loading status from 0.2.7
> **Outputs:** `validation_status` with pass/fail and remediation guidance

---

## FUNCTION

Validate that all required inputs from Layer 0 loaders are present, properly formatted, and above minimum quality thresholds. Gate Layer 1 execution until validation passes. This is the final quality check before email architecture begins.

---

## VALIDATION CHECKLIST

### Critical Requirements (Must Pass -- HALT if any fails)

| Input | Source | Validation Rule | Failure Action |
|-------|--------|-----------------|----------------|
| Campaign blueprint | 0.1 email_context | `blueprint` section populated | FAIL -- Execute E0 first |
| Current email position | 0.1 email_context | `current_position` identified | FAIL -- Specify email position |
| Body type assigned | 0.1 email_context | `body_type` is one of [CT, QO, TM, QA, LB, ST, NR] | FAIL -- Check blueprint |
| Word count target | 0.1 email_context | `word_count_target` between 200-500 | FAIL -- Verify blueprint assignment |
| CTA pattern assigned | 0.1 email_context | `cta_pattern` is one of [A, B, C, D, E] | FAIL -- Check blueprint |
| Structural specimens loaded | 0.2 structural_specimens | `loaded` = true AND `specimen_count` >= 3 | FAIL -- Return to 0.2 |
| Specimens are verbatim | 0.2 structural_specimens | `all_verbatim` = true | FAIL -- Reload verbatim |
| Specimens type-matched | 0.2 structural_specimens | `type_matched` = true | FAIL -- Load correct body type |

### High Priority Requirements (Warn but continue)

| Input | Source | Validation Rule | Failure Action |
|-------|--------|-----------------|----------------|
| Persona voice specimens | 0.2.7 | 6 personas have 2+ specimens each | WARN -- Proceed with degraded Arena |
| Upstream packages (Mode A) | 0.1 email_context | Big idea, mechanism, promise loaded | WARN -- Mode B quality ceiling |
| Content direction | 0.1 email_context | `content_direction` non-null | WARN -- Use general topic from blueprint |
| Sequence context | 0.1 email_context | Previous/next email info present | WARN -- Cannot optimize for sequence flow |

### Quality Thresholds

| Metric | Minimum | Optimal |
|--------|---------|---------|
| Structural specimens | 3 | 5 |
| Persona voice specimens per persona | 2 | 3 |
| Content direction length | 10 words | 50+ words |
| Blueprint completeness | Body type + word count + CTA | All fields populated |

---

## VALIDATION PROTOCOL

```
EXECUTE 0.3-input-validator:

  CRITICAL_CHECKS:
    1. CHECK email_context.blueprint
       IF null OR missing:
         FAIL: "Campaign blueprint not loaded. Execute E0 first."

    2. CHECK email_context.assignment.body_type
       IF null OR not in [CT, QO, TM, QA, LB, ST, NR]:
         FAIL: "Body type not assigned or invalid. Check blueprint."

    3. CHECK email_context.assignment.word_count_target
       IF null OR < 200 OR > 500:
         FAIL: "Word count target invalid. Must be 200-500."

    4. CHECK email_context.assignment.cta_pattern
       IF null OR not in [A, B, C, D, E]:
         FAIL: "CTA pattern not assigned. Check blueprint."

    5. CHECK structural_specimens.loaded
       IF false:
         FAIL: "Structural specimens not loaded. Return to 0.2."

    6. CHECK structural_specimens.specimen_count
       IF < 3:
         FAIL: "Fewer than 3 specimens loaded. Need minimum 3."

    7. CHECK structural_specimens.all_verbatim
       IF false:
         FAIL: "Specimens not loaded verbatim. Reload without summarizing."

    8. CHECK structural_specimens.type_matched
       IF false:
         FAIL: "Specimens don't match assigned body type. Reload correct type."

  IF any CRITICAL_CHECK fails:
    SET validation_status.passed = false
    SET validation_status.blocker = true
    RETURN validation_status

  HIGH_PRIORITY_CHECKS:
    9. CHECK persona voice loading (0.2.7)
       IF incomplete:
         WARN: "Persona voice specimens incomplete. Arena quality reduced."
         SET degraded_mode.persona_voices = true

    10. CHECK email_context.mode
        IF "B":
          WARN: "Mode B -- no upstream packages. Email quality ceiling reduced."
          SET degraded_mode.upstream = true

    11. CHECK email_context.assignment.content_direction
        IF null OR length < 10:
          WARN: "Content direction sparse. Using general blueprint topic."
          SET degraded_mode.content_direction = true

    12. CHECK email_context.sequence_context
        IF previous_email = null AND position_index > 1:
          WARN: "Previous email context missing. Cannot optimize flow."
          SET degraded_mode.sequence_context = true

  ASSEMBLE validation_status
  RETURN validation_status
```

---

## GATE CONDITION

**Layer 1 may proceed when:**
- `validation_status.passed` = true
- `validation_status.blocker` = false
- All 8 critical checks passed

**Layer 1 may NOT proceed when:**
- Any critical check failed
- `validation_status.blocker` = true
- Remediation required before continuing

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Blueprint missing | HALT -- Execute E0 |
| CRITICAL | Body type not assigned | HALT -- Check blueprint |
| CRITICAL | Specimens not loaded | HALT -- Return to 0.2 |
| CRITICAL | Specimens not verbatim | HALT -- Reload verbatim |
| DEGRADED | Mode B (no upstream) | Proceed with quality flag |
| DEGRADED | Persona voices incomplete | Proceed with Arena quality warning |
