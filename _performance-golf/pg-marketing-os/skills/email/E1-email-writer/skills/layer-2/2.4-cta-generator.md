# 2.4-cta-generator.md

> **Layer:** 2 (Generation)
> **Skill:** E1-email-writer
> **Purpose:** Generate CTA + sign-off following assigned pattern (A-E from cta-pattern-library.md)
> **Inputs:** `bridge_output` from 2.3, `email_context.assignment.cta_pattern` from 0.1, cta-pattern-library.md
> **Outputs:** CTA text (1-3 sentences) + sign-off, completing the email draft

---

## FUNCTION

Generate the Call-to-Action and sign-off that closes the email. The CTA follows directly from the bridge and is the SHORTEST section of the email (5-10% of total word count). It must be clear, emotionally compelling, and include a specific action path. The CTA pattern is ASSIGNED by the E0 blueprint and cannot be changed.

**Key Principle:** The CTA is NOT a sales pitch. The body and bridge have already done the work. The CTA is simply the CLEAR NEXT STEP -- "Here's where to go, here's why." Most email CTAs should be 1-3 sentences: a transition phrase + URL + sign-off. Brevity is power.

---

## CTA PATTERNS (Assigned by Blueprint)

### Pattern A: Direct Link

```
STRUCTURE: Statement + URL + sign-off
TONE: Confident, simple, clear
ENERGY: "Here it is. Go."

TEMPLATE:
  [Transition phrase that flows from bridge]
  [URL on its own line]
  [Sign-off: Name]

EXAMPLE PATTERN:
  "Get the full details right here:"
  [LINK]
  -- [Name]

BEST FOR: All body types. Default pattern. Works everywhere.
WORD COUNT: 10-20 words
```

### Pattern B: Curiosity CTA

```
STRUCTURE: Curiosity tease + URL + sign-off
TONE: Intriguing, incomplete
ENERGY: "You'll want to see this."

TEMPLATE:
  [Curiosity tease that creates a gap the link resolves]
  [URL on its own line]
  [Sign-off: Name]

EXAMPLE PATTERN:
  "See which [specific element] made the biggest difference (hint: it's not what you'd expect):"
  [LINK]
  -- [Name]

BEST FOR: CT, QO, ST. Body types that build curiosity.
WORD COUNT: 15-30 words
```

### Pattern C: Challenge CTA

```
STRUCTURE: Challenge/dare + URL + sign-off
TONE: Provocative, confident
ENERGY: "I dare you."

TEMPLATE:
  [Challenge that dares the reader to take action]
  [URL on its own line]
  [Sign-off: Name]

EXAMPLE PATTERN:
  "Think I'm wrong? See the proof for yourself:"
  [LINK]
  -- [Name]

BEST FOR: CT, NR. Body types with combative energy.
WORD COUNT: 10-25 words
```

### Pattern D: Social Proof CTA

```
STRUCTURE: Social proof reference + URL + sign-off
TONE: Confident through others' results
ENERGY: "They did it. You can too."

TEMPLATE:
  [Reference to others who have taken action + their results]
  [URL on its own line]
  [Sign-off: Name]

EXAMPLE PATTERN:
  "Join the [number] [people] who already [result]:"
  [LINK]
  -- [Name]

BEST FOR: TM, QA, LB. Body types that build on proof and community.
WORD COUNT: 15-30 words
```

### Pattern E: Soft CTA

```
STRUCTURE: Permission-based suggestion + URL + sign-off
TONE: Low-pressure, generous
ENERGY: "If you're ready, here's where to go."

TEMPLATE:
  [Permission-based suggestion that doesn't push]
  [URL on its own line]
  [Sign-off: Name]

EXAMPLE PATTERN:
  "If any of this resonated, you'll probably find this interesting:"
  [LINK]
  -- [Name]

BEST FOR: QO, ST, early-sequence emails. Relationship-building positions.
WORD COUNT: 15-25 words
```

## EXECUTION PROTOCOL

```
EXECUTE 2.4-cta-generator:

  STEP 1: Load Inputs
    1. READ bridge_output from 2.3 (CTA must follow from bridge)
    2. READ email_context.assignment.cta_pattern (assigned by blueprint)
    3. LOAD cta-pattern-library.md (if available)
    4. HOLD structural_specimens in active context

  STEP 2: Study Specimen CTAs
    FOR each loaded specimen:
      1. ISOLATE the CTA section
      2. IDENTIFY: transition phrase, URL placement, sign-off style
      3. NOTE word count and sentence count
      4. ASSESS: Does it feel like a natural close or a hard sell?
    INTERNALIZE: brevity and natural closure

  STEP 3: Generate CTA
    1. SELECT assigned CTA pattern (A-E from blueprint)
    2. WRITE transition phrase that flows from bridge
    3. PLACE URL on its own line (placeholder: [LINK])
    4. ADD sign-off (name only, unless brand voice specifies differently)
    5. VERIFY total is 1-3 sentences

  STEP 4: CTA Quality Check
    [ ] CTA follows assigned pattern (A, B, C, D, or E)
    [ ] CTA flows naturally from bridge (no jarring shift)
    [ ] CTA is 1-3 sentences maximum
    [ ] URL is on its own line
    [ ] Sign-off is present (name only default)
    [ ] CTA is emotionally compelling (not just "click here")
    [ ] CTA is specific (reader knows what they're clicking to)
    [ ] Zero slop words
    [ ] CTA doesn't repeat bridge content

  STEP 5: Full Email Assembly Check
    1. READ opening + body + bridge + CTA as one continuous piece
    2. VERIFY total word count is within target range (200-500, +/-20%)
    3. CALCULATE content-to-pitch ratio:
       - Content = opening + body word count
       - Pitch = bridge + CTA word count
       - Ratio must be 70-80% content / 20-30% pitch
    4. IF ratio violated: FLAG for revision (usually means bridge/CTA too long)

  STEP 6: Output
    WRITE CTA text + sign-off to output file
    INCLUDE full email assembly (opening + body + bridge + CTA) for review
```

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | CTA exceeds 3 sentences | Compress -- brevity is power |
| CRITICAL | CTA doesn't match assigned pattern | Regenerate with correct pattern |
| CRITICAL | Content-to-pitch ratio violated | Revise bridge/CTA to reduce pitch proportion |
| CRITICAL | Total word count outside range | Edit full email to fit |
| WARNING | CTA is generic ("click here") | Add emotional specificity |
| WARNING | CTA repeats bridge language | Differentiate CTA from bridge |

---

## ANTI-PATTERNS (FORBIDDEN)

1. **"Click here"** -- This is not a CTA, it's an instruction. CTA must have emotional motivation.
2. **CTA longer than 3 sentences** -- If CTA is long, it's become a pitch section. Compress.
3. **Wrong CTA pattern** -- Pattern is ASSIGNED by blueprint. Cannot substitute.
4. **CTA that introduces new content** -- CTA is for action, not new information.
5. **Generic urgency** -- "Act now!" without justification is banned.
6. **Missing URL placement** -- URL must be on its own line (placeholder: [LINK]).
7. **Missing sign-off** -- Every email ends with a sign-off.
8. **CTA that repeats body content** -- CTA advances, it doesn't recap.
