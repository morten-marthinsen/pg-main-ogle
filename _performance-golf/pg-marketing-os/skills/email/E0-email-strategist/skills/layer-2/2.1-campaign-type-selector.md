# Layer 2.1 — Campaign Type Selector

## Purpose

Finalize the campaign type selection by cross-referencing goal analysis (1.1) with audience profile (1.2) and asset inventory (1.3). Load the matching campaign template from the `templates/` directory. Apply audience-driven overrides where the audience profile suggests a different type than the goal alone would indicate.

---

## Input Requirements

```yaml
from_layer_1:
  goal_analysis:
    campaign_type: string
    template_file: string
    context_richness_check:
      passes: boolean
      downgrade_recommendation: string

  audience_profile:
    awareness_level:
      level: string
    sophistication_level:
      stage: integer
    relationship_stage:
      stage: string
    content_density:
      preferred: string
    campaign_type_alignment:
      alignment: string
      adjustment_notes: string

  asset_inventory:
    total_usable_assets: integer
    body_type_gaps: [string]
    gate_1_eligible: boolean
```

---

## Execution Protocol

### Step 1: Review Goal-Based Selection

Start with the campaign type selected in 1.1-goal-analyzer.md. This is the default unless audience or asset data forces an override.

### Step 2: Apply Audience-Driven Overrides

Cross-reference audience profile against the selected campaign type. Apply these override rules:

```
OVERRIDE RULE 1: Cold Audience Override
  IF audience.relationship_stage == "cold" AND campaign_type IN [PL, AL, BP]:
    → RECOMMEND override to CP (Content Series)
    → Rationale: Cold audiences need trust-building before promotion.
      Launching to a cold list wastes the launch sequence.
    → Flag: "AUDIENCE OVERRIDE — cold audience detected. Recommend
      Content Series (CP) before [selected type]."

OVERRIDE RULE 2: Unaware Audience Override
  IF audience.awareness_level == "unaware" AND campaign_type IN [PL, BP, DU]:
    → RECOMMEND override to CP or AR
    → Rationale: Unaware audiences need education before promotion.
    → Flag: "AUDIENCE OVERRIDE — unaware audience detected. Recommend
      education-first sequence."

OVERRIDE RULE 3: Most-Aware Shortcut
  IF audience.awareness_level == "most_aware" AND audience.relationship_stage == "hot":
    → campaign_type BP or DU is APPROPRIATE even if goal suggested longer sequence
    → Flag: "AUDIENCE SHORTCUT — most-aware hot audience. Short sequence viable."

OVERRIDE RULE 4: High Sophistication Override
  IF audience.sophistication_level.stage >= 4 AND campaign_type == PL:
    → MODIFY: Add 2-3 extra content emails before launch sequence begins
    → Rationale: Sophisticated audiences need proof-heavy pre-launch education.
    → Flag: "SOPHISTICATION ADJUSTMENT — Stage 4+ audience. Adding
      pre-launch content buffer."

OVERRIDE RULE 5: Asset Scarcity Override
  IF asset_inventory.total_usable_assets < 5:
    → RECOMMEND override to BP or CP (shorter sequences need fewer assets)
    → Rationale: Cannot sustain long sequence variety with thin asset inventory.
    → Flag: "ASSET OVERRIDE — insufficient assets for [selected type].
      Recommend shorter sequence."
```

### Step 3: Resolve Conflicts

If multiple override rules fire, apply priority:
1. Cold audience override (highest priority — never launch to cold)
2. Asset scarcity override (cannot design what we can't fill)
3. Unaware audience override
4. High sophistication adjustment (modification, not override)
5. Most-aware shortcut (lowest priority — optimization, not correction)

### Step 4: Load Campaign Template

Load the template file for the confirmed campaign type:

```
READ template from: ./skills/email/templates/[template_file]

EXTRACT from template:
  - default_email_count
  - default_duration_days
  - recommended_sequence_structure (email positions with function codes)
  - body_type_suggestions (per position)
  - urgency_escalation_pattern
  - volume_escalation_pattern (launch sequences only)
```

If template file does not exist (DU type or missing template):
- Use inline defaults from EMAIL-STRATEGIST-AGENT.md
- Log: "No template file loaded. Using inline defaults."

### Step 5: Apply Modifications to Template Defaults

Based on audience profile and asset inventory, modify template defaults:

```
MODIFICATION: Sequence Length
  IF audience is cold or unaware → ADD 2-3 trust-building emails to start
  IF assets are limited → REDUCE email count to match available content
  IF audience is most-aware and hot → REDUCE to minimum viable sequence

MODIFICATION: Body Type Distribution
  IF asset gaps exist → Remove gapped types from template suggestions
  IF audience prefers dense content → Weight toward ST, CT, LB
  IF audience prefers light content → Weight toward QO, TM, short LB

MODIFICATION: Urgency Timing
  IF audience is cold → DELAY urgency to final 20% of sequence (not final third)
  IF audience is hot → Urgency can start at 50% mark
```

## Output

```yaml
confirmed_campaign:
  campaign_type: string  # final confirmed type
  campaign_type_name: string
  original_type: string  # from 1.1 (before any overrides)
  overrides_applied: [string]  # list of override rules that fired
  override_justifications: [string]

  template:
    file_loaded: string  # path to template file, or "inline_defaults"
    default_email_count: integer
    default_duration_days: integer
    recommended_sequence: [object]  # email positions with function codes
    body_type_suggestions: [string]  # per position
    urgency_pattern: string
    volume_pattern: string  # null unless PL

  modifications:
    sequence_length_adjustment: string  # e.g., "+3 trust-building emails"
    body_type_adjustments: [string]
    urgency_timing_adjustment: string
    rationale: string

  final_parameters:
    email_count: integer  # after modifications
    duration_days: integer  # after modifications
    urgency_start_position: integer  # which email number urgency begins
```

---

## Failure Modes

| Failure | Remediation |
|---------|-------------|
| Template file not found | Fall back to inline defaults from AGENT.md. Log warning. |
| All override rules fire simultaneously | Apply priority resolution (Step 3). Present the single confirmed type with full justification. |
| Audience profile and goal analysis strongly contradict | Present both options to human at Gate 2 with trade-offs. Do not auto-resolve major contradictions. |
| Template defaults don't match available assets | Modify defaults to match reality. Never plan emails that cannot be filled with content. |

- **Passes to:** 2.2-sequence-blueprint.md (confirmed campaign + loaded template + modifications)
- **Runs parallel with:** Nothing — 2.1 runs FIRST in Layer 2 (loads template that 2.2-2.4 depend on)
