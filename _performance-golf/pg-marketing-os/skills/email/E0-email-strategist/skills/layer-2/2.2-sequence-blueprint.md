# Layer 2.2 — Sequence Blueprint

## Purpose

Design the email-by-email sequence for the campaign. For each email position: assign day, determine function code, set content focus, set urgency level (0-5), set target word count (200-500), set emotional target, and set CTA pattern. Follow the template structure but customize based on available assets and audience profile. Enforce variety rules throughout.

## Execution Protocol

### Step 1: Establish the Day-by-Day Grid

Map emails to days based on campaign type:

```yaml
day_mapping_by_type:
  PL:  # Product Launch — volume escalation on final day
    day_-1: [1 email]
    day_1: [1-2 emails]
    day_2: [2 emails]
    day_3: [2 emails]
    day_4: [2-3 emails]
    day_5: [5-8 emails]  # close day blitz

  AL:  # Affiliate Launch
    day_1: [1 email]
    day_2: [1 email]
    day_3: [1-2 emails]
    day_4: [1-2 emails]
    day_5: [2-3 emails]  # close day

  AR:  # Autoresponder — 1 per day, evenly spaced
    day_1_through_N: [1 email per day]

  DR:  # Daily Relationship — 1 per day, ongoing
    every_day: [1 email]

  CP:  # Content Series — 1 per day
    day_1_through_7: [1 email per day]

  BP:  # Blatant Pitch — single email
    day_1: [1 email]

  DU:  # Deadline Urgency — single email
    deadline_day: [1 email]
```

### Step 2: Assign Function Codes

Each email gets a function code that describes its PRIMARY purpose:

| Code | Function | Description |
|------|----------|-------------|
| DR | Daily Relationship | Value-forward content, relationship building |
| DU | Deadline Urgency | Deadline/scarcity-driven, action-focused |
| PL | Product Launch | New product introduction/anticipation |
| AL | Affiliate Launch | Partner product promotion |
| BP | Blatant Pitch | Direct offer with clear CTA |
| CP | Content/Education | Pure education, trust building |
| AR | Autoresponder | Evergreen nurture content |

**Assignment rules:**
- First 1/3 of any sequence: primarily CP or DR (content-forward)
- Middle 1/3: mix of CP, DR, and moderate PL/AL
- Final 1/3: primarily BP, DU, PL/AL (action-forward)
- Single-email types (BP, DU): entire email is that function

### Step 3: Set Content Focus Per Email

For each email, define what specific content it delivers:

```
Email 1: "[Root cause revelation — why the problem persists]"
Email 2: "[Mechanism education — how the solution works]"
Email 3: "[Testimonial showcase — social proof from real users]"
Email 4: "[Myth-busting — contrarian angle on common belief]"
...
```

Content focus must draw from available assets. Do NOT plan content that has no supporting asset in the inventory.

### Step 4: Set Urgency Level Per Email (0-5)

```yaml
urgency_scale:
  0: "No urgency. Pure content/value."
  1: "Subtle mention of opportunity. No deadline language."
  2: "Mention deadline exists. Soft reminder."
  3: "Deadline approaching. Clear consequence language."
  4: "Deadline imminent. Strong urgency. Last chance framing."
  5: "Final call. Cart closing. Maximum urgency."

urgency_rules:
  - Emails before urgency_start_position: urgency = 0
  - Urgency MUST escalate monotonically once started (never go down then back up)
  - Maximum urgency (5) reserved for FINAL email only
  - Never start above 2 in a single jump (gradual escalation)
```

### Step 5: Set Target Word Count Per Email

Base word count on content density preference from audience profile:

| Content Density | Base Range | Adjustments |
|----------------|-----------|-------------|
| Light | 200-300 | DU/BP emails: -50. Content emails: base. |
| Medium | 300-400 | Standard range. Most emails here. |
| Dense | 400-500 | Educational emails: +50. DU/BP: base only. |

**Adjustment rules:**
- Urgency level 4-5 emails: word count DECREASES (shorter = more urgent)
- Story (ST) body type: word count INCREASES by 50 (stories need room)
- DU emails: always on the lower end (urgency is brief)

### Step 6: Set Emotional Target Per Email

Assign the dominant emotion each email should evoke:

| Phase | Emotional Targets |
|-------|-------------------|
| First third (trust) | curiosity, empathy, recognition, surprise |
| Middle third (desire) | desire, hope, understanding, excitement |
| Final third (urgency) | urgency, consequence, relief, resolution |

### Step 7: Set CTA Pattern Per Email

Assign CTA pattern using the 5 standard patterns:

| Pattern | Description | When to Use |
|---------|-------------|-------------|
| A | Soft CTA — "Learn more" / "Read this" | Content-forward emails (first third) |
| B | Medium CTA — "Check this out" / "See the details" | Middle emails, educational with product mention |
| C | Direct CTA — "Click here to order" / "Get yours now" | Promotion emails (middle to final third) |
| D | Urgency CTA — "Don't miss out" / "Last chance" | Deadline/urgency emails (final third) |
| E | P.S. CTA — CTA in the P.S. only | When main body is content-heavy, CTA feels intrusive |

### Step 8: Enforce Variety Rules (NON-NEGOTIABLE)

Before finalizing the sequence, validate:

```
RULE 1: Never same body type twice in a row
  → Scan sequence. If consecutive same type detected, swap one.

RULE 2: In any 5-email window, at least 3 different body types
  → Slide a 5-email window across the sequence. Count unique types.
  → If fewer than 3 unique in any window, redistribute.

RULE 3: NR (Negative Response) maximum 1 per 7 emails
  → Count NR instances. If ratio exceeded, remove or replace one.
```

## Failure Modes

| Failure | Remediation |
|---------|-------------|
| Not enough assets to fill all email positions | Reduce email count. Mark unfilled positions as "to_be_written" (requires E1 to generate from scratch). |
| Variety rules cannot be satisfied with available types | Reduce email count until variety rules pass. Or mark some emails for "generic" body type that E1 adapts. |
| Urgency escalation breaks monotonic rule | Reorder urgency levels. Urgency can only go up or stay flat — never decrease then increase. |
| Template recommends more emails than assets support | Trim template to match asset reality. Quality over quantity. |
| Day mapping exceeds duration | Compress schedule: add more emails per day in final days rather than extending duration. |
