# Layer 1.5 -- Reader Focus + Deliverability Audit

## Purpose

Audit every email in the sequence for customer-centricity and deliverability risk. Reader-focus failures turn personal emails into company newsletters. Deliverability failures mean the email never reaches the inbox. Both are invisible killers -- the writer rarely notices them, but the reader (and spam filter) always does. Every email must be checked -- no sampling, no skipping.

## Execution Protocol

### Step 1: Reader Focus Ratio (You/Your vs. We/I)

For EVERY email in the sequence, count and classify pronouns:

```
PRONOUN CLASSIFICATION:

  READER-FOCUSED (positive):
    you, your, you're, you've, you'll, yourself
    Counts toward reader-focus ratio.

  SENDER-FOCUSED (negative):
    I, me, my, mine, myself, we, our, us, ourselves
    Counts against reader-focus ratio.

  NEUTRAL (excluded from ratio):
    he, she, they, them, it, this, that
    Third-party references do not count either direction.

RATIO CALCULATION:

  reader_focus_ratio = reader_pronouns / (reader_pronouns + sender_pronouns)

  SCORING:
    >= 60% reader-focused: PASS (no issue)
    40-59% reader-focused: P3 (minor -- email leans sender-focused)
    < 40% reader-focused: P2 (major -- email reads as company newsletter)

  EXCEPTION:
    Story-type emails (ST body type) may have lower reader-focus ratio
    because the story is told in first person. For ST emails:
      >= 40% reader-focused: PASS
      25-39% reader-focused: P3
      < 25% reader-focused: P2

    The story MUST still connect back to the reader before the bridge.
    A story that never becomes "and here's why this matters for YOU" fails
    regardless of ratio.
```

### Step 2: Benefit-Forward Language Check

For EVERY email, check whether the content is framed in terms of reader benefit:

```
BENEFIT-FORWARD PATTERNS (good):
  "This means you can..."
  "What this gives you is..."
  "The result? You [benefit]..."
  "Here's what changes for you..."
  "You'll notice [specific improvement]..."

FEATURE-FORWARD PATTERNS (flag):
  "Our product has..."
  "We developed..."
  "The system includes..."
  "It contains 14 modules..."
  "We're proud to offer..."

SCORING:
  FOR each email:
    Count benefit-forward statements vs. feature-forward statements.
    benefit_ratio = benefit_statements / (benefit_statements + feature_statements)

    >= 70% benefit-forward: PASS
    50-69% benefit-forward: P4 (cosmetic -- could be more reader-centric)
    < 50% benefit-forward: P3 (minor -- email leads with features over benefits)

  NOTE: Pitch sections naturally have more feature language.
        Only flag if the CONTENT section is feature-heavy.
        Pitch sections are expected to name features -- that's their job.
```

### Step 3: Spam Trigger Word Scan

Scan every email (subject line AND body) for deliverability risk words:

```
HIGH-RISK SPAM TRIGGERS (flag each occurrence):

  FINANCIAL:
    free, guarantee, no obligation, no cost, no fees,
    earn money, cash, million, billion, income,
    act now, limited time, don't miss, urgent

  HEALTH:
    cure, miracle, weight loss, lose weight,
    no side effects, clinically proven

  PRESSURE:
    buy now, order now, click here, apply now,
    call now, don't delete, not spam

  DECEPTIVE:
    congratulations, you've been selected,
    you're a winner, claim your, as seen on

RISK SCORING:

  Per email:
    0 triggers: CLEAN
    1-2 triggers: LOW RISK (P4 -- cosmetic, swap if easy)
    3-5 triggers: MODERATE RISK (P3 -- minor, should address)
    6+ triggers: HIGH RISK (P2 -- major, deliverability threatened)

  Per subject line specifically:
    0 triggers: CLEAN
    1 trigger: P3 (subject lines are more heavily filtered)
    2+ triggers: P2 (subject line spam triggers are high-impact)

  NOTE: Context matters. "Free" in "feel free to reply" is not a spam trigger.
        "Free" in "get your FREE bonus" IS a spam trigger.
        Use judgment, not blind pattern matching.
```

### Step 4: Formatting Deliverability Risks

Check for formatting patterns that trigger spam filters:

```
FORMATTING RISK PATTERNS:

  ALL CAPS:
    1-2 ALL CAPS words per email: ACCEPTABLE (emphasis)
    3-5 ALL CAPS words: P4 (cosmetic risk)
    6+ ALL CAPS words: P3 (minor -- reduce caps)
    Entire sentences in ALL CAPS: P2 (major -- spam filter trigger)

  EXCESSIVE PUNCTUATION:
    Multiple exclamation marks (!!!): P3 per occurrence
    Multiple question marks (???): P4 per occurrence
    Exclamation + question combo (!?!): P3 per occurrence
    More than 3 exclamation marks in one email: P2

  LINK DENSITY:
    1-2 links per email: ACCEPTABLE
    3-4 links: P4 (cosmetic -- consolidate if possible)
    5+ links: P3 (minor -- high link density triggers filters)
    Same link repeated 3+ times: P3

  COLOR/HTML FORMATTING (if applicable):
    Red text: P3 (spam association)
    Large font size changes: P4
    Multiple font colors: P3

  IMAGE-TO-TEXT RATIO (if applicable):
    Image-heavy emails with little text: P3
    All-image emails: P2 (many filters block or flag)
```

### Step 5: Reading Level Check

Verify the email's reading level is appropriate for the target audience:

```
READING LEVEL ASSESSMENT:

  Measure average:
    - Sentence length (words per sentence)
    - Word complexity (syllables per word)
    - Paragraph length (sentences per paragraph)

  TARGET RANGES (direct response email):
    Sentence length: 8-18 words average (conversational)
    Word complexity: 1.2-1.8 syllables average (plain language)
    Paragraph length: 1-3 sentences (scannable)

  SCORING:
    All metrics in range: PASS
    1 metric out of range: P4 (cosmetic -- minor adjustment)
    2 metrics out of range: P3 (minor -- reading level needs work)
    All metrics out of range: P2 (major -- email is too complex for DR audience)

  COMMON FAILURES:
    Academic/formal language in a casual email sequence
    Run-on sentences (30+ words)
    Multi-clause sentences requiring re-reading
    Jargon without explanation (unless audience expects it)
    Wall-of-text paragraphs (4+ sentences)
```

## Failure Modes

| Failure | Remediation |
|---------|-------------|
| Email body too short for meaningful ratio (<30 words) | Log as "insufficient text for ratio analysis." Score conservatively. Do not flag. |
| Story-type email has low reader-focus ratio | Apply ST exception threshold (40% PASS). Only flag if below 25%. |
| Spam trigger word used in legitimate context ("feel free") | Use context judgment. Do not blind-flag. Note the context in the report. |
| Email is intentionally ALL CAPS (stylistic choice) | Flag but note as "intentional stylistic choice -- verify with human." |
| Reading level is high because audience is sophisticated (e.g., financial professionals) | Note audience context. Adjust thresholds if campaign brief specifies sophisticated audience. |
| Subject line is a single word or emoji | Cannot reliably scan for spam triggers. Note as "minimal SL -- no trigger scan possible." |
