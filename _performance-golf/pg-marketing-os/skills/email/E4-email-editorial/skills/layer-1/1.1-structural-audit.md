# Layer 1.1 -- Structural Audit

## Purpose

Check each email's compliance with its assigned body type structural template. Every body type (CT, QO, TM, QA, LB, ST, NR) has a specific structural skeleton with required elements in a specific order. This audit verifies that each email follows its assigned template -- not loosely, but structurally.

This is NOT about quality (that was Layer 0 scoring). This is about STRUCTURE: are the required elements present? Are they in the right order? Is anything missing or misplaced?

## Execution Protocol

### Step 1: Load Body Type Structural Templates

For each of the 7 body types, load the structural skeleton:

```
CONTRARIAN (CT):
  Required elements (in order):
    1. Bold contrarian claim (first 1-3 lines)
    2. Argument stacking (supporting points, evidence, reasoning)
    3. Dismissal of conventional wisdom (specific, not vague)
    4. Bridge/transition to pitch
    5. CTA + sign-off

  STRUCTURAL VIOLATIONS:
    - No bold claim in opening → P1 (fundamentally not a Contrarian email)
    - Arguments but no dismissals → P3 (minor structural gap)
    - Missing bridge → P2 (major -- pitch appears abruptly)

QUOTE-OPENER (QO):
  Required elements (in order):
    1. Formatted quote block (attribution + quote text)
    2. Riff on the quote (personal commentary, expansion)
    3. Lesson extraction (what does this mean for the reader)
    4. Bridge/transition to pitch
    5. CTA + sign-off

  STRUCTURAL VIOLATIONS:
    - No quote in opening → P1 (fundamentally not a Quote-Opener)
    - Quote but no riff → P3 (missing commentary layer)
    - No lesson extracted → P3 (quote without application)

TESTIMONIAL (TM):
  Required elements (in order):
    1. Attribution (who is this testimonial from)
    2. Testimonial block (the actual testimonial, clearly demarcated)
    3. Brief reaction/commentary (writer's take on the testimonial)
    4. CTA (often directly after reaction, minimal bridge needed)

  STRUCTURAL VIOLATIONS:
    - No attribution → P3 (anonymous testimonial is weaker)
    - Testimonial not clearly blocked → P3 (visual clarity issue)
    - Extended reaction (>30% of email) → P3 (too much commentary)

Q&A / READER RESPONSE (QA):
  Required elements (in order):
    1. Attribution (who asked the question)
    2. Question displayed (the actual question, clearly formatted)
    3. Substantive answer (not a brush-off -- real value)
    4. Bridge/transition to pitch
    5. CTA + sign-off

  STRUCTURAL VIOLATIONS:
    - No question displayed → P1 (fundamentally not a Q&A email)
    - Answer is 1-2 sentences only → P2 (not substantive)
    - No attribution → P4 (cosmetic -- who asked matters less)

LIST-BASED (LB):
  Required elements (in order):
    1. Setup/context (why this list matters)
    2. Numbered or bulleted list (minimum 3 items)
    3. Wrap-up/summary (brief synthesis)
    4. CTA + sign-off

  STRUCTURAL VIOLATIONS:
    - List has fewer than 3 items → P3 (too thin for a list email)
    - No setup → P3 (list without context)
    - List items are one word each → P3 (not substantive)

STORY (ST):
  Required elements (in order):
    1. Story opener/hook (drops reader into a scene or moment)
    2. Narrative development (the story unfolds with tension)
    3. Lesson extraction (what the story teaches)
    4. Bridge/transition to pitch
    5. CTA + sign-off

  STRUCTURAL VIOLATIONS:
    - No story in a Story email → P1 (fundamentally wrong body type)
    - Story but no lesson → P2 (entertainment without purpose)
    - Story opener is abstract, not specific → P3 (weak opening)

NEGATIVE RESPONSE (NR):
  Required elements (in order):
    1. Troll/critic display (the negative comment, clearly shown)
    2. Demolition (entertaining takedown or reframe)
    3. Lesson extraction (what this teaches the reader)
    4. Bridge/transition to pitch
    5. CTA + sign-off

  STRUCTURAL VIOLATIONS:
    - No troll/critic displayed → P1 (fundamentally not an NR email)
    - Demolition is mean-spirited (not entertaining) → P2 (tone problem)
    - No lesson extracted → P3 (demolition without purpose)
```

### Step 2: Audit Each Email

For EVERY email in the sequence:

```
FOR email IN loaded_sequence.emails:

  1. IDENTIFY assigned body type
  2. LOAD structural template for that body type
  3. CHECK each required element:
     - Present? [Y/N]
     - In correct position? [Y/N]
     - Meets minimum quality? [Y/N]
  4. IDENTIFY any structural violations
  5. CLASSIFY violation severity (P1/P2/P3/P4)
  6. RECORD findings
```

### Step 3: Check for Body Type Misassignment

Beyond template compliance, check whether the email's ACTUAL structure matches a DIFFERENT body type better:

```
IF email assigned as CT but reads like a ST:
  → P1 issue: Body type misassignment
  → Note: "Email [N] is assigned Contrarian but is structurally a Story email."

IF email assigned as QO but has no quote:
  → P1 issue: Body type fundamentally wrong
  → Note: "Email [N] is assigned Quote-Opener but contains no quote."
```

### Step 4: Compile Structural Audit Results

Aggregate all findings into a structured report.

## Failure Modes

| Failure | Remediation |
|---------|-------------|
| Structural analysis reference file not found | Use inline template definitions from this microskill. Log warning. |
| Body type code not recognized | Log as P1 issue. Email has invalid type assignment. |
| Email too short to have all required elements | Audit what exists. Missing elements still count as violations. |
| Ambiguous structure (could be 2 types) | Check against assigned type first. If assigned type fails, note better-fit type. |
