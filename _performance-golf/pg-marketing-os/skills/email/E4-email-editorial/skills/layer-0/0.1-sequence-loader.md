# Layer 0.1 -- Sequence Loader

## Purpose

Load the complete assembled email sequence from E3 (Email Sequence Assembler) output. Parse every email into a working structure with all metadata (position, body type, function, subject line, body text, word count). Validate that the sequence is complete and parseable.

---

## Input Requirements

```yaml
from_E3:
  assembled_sequence_path: "[project]/E3-email-sequence-assembler/assembled-sequence.yaml"
  # OR revised path if E3 produced revised version:
  alternate_path: "[project]/E3-email-sequence-assembler/revised-sequence.yaml"

required_shared_files:
  quality_rubric: "./skills/email/shared/email-quality-rubric.md"
  niche_guide: "./skills/email/shared/niche-adaptation-guide.md"
```

---

## Execution Protocol

### Step 1: Locate E3 Output

Search for the assembled sequence file in the project directory. Check both standard and alternate paths.

```
SEARCH ORDER:
  1. [project]/E3-email-sequence-assembler/assembled-sequence.yaml
  2. [project]/E3-email-sequence-assembler/revised-sequence.yaml
  3. [project]/assembled-sequence.yaml (fallback)

IF no sequence file found:
  HALT -- Cannot proceed without E3 assembled sequence.
  Log: "E3 output not found at any expected path."
```

### Step 2: Parse Sequence Metadata

Extract campaign-level metadata:
- `campaign_name` -- the campaign identifier
- `campaign_type` -- launch_sequence, autoresponder, daily_relationship, etc.
- `total_emails` -- count of emails in the sequence
- `duration_days` -- campaign duration
- `target_audience` -- audience description
- `emotional_arc` -- planned emotional progression

### Step 3: Parse Individual Emails

For each email in the sequence, extract:
- `position` -- ordinal position in the sequence (1, 2, 3...)
- `subject_line` -- the subject line (from E2)
- `body` -- full email body text
- `body_type` -- assigned body type code (CT, QO, TM, QA, LB, ST, NR)
- `function` -- campaign function code (DR, DU, PL, AL, BP, CP, AR)
- `word_count` -- count words in the body
- `target_word_count` -- planned word count from E0 blueprint
- `urgency_level` -- 0-5 urgency rating
- `emotional_target` -- planned emotional target
- `cta_pattern` -- CTA pattern code (A-E)

### Step 4: Validate Completeness

```
VALIDATION CHECKS:
  [ ] Total emails parsed matches expected count
  [ ] Every email has a subject line (not empty)
  [ ] Every email has body text (not empty)
  [ ] Every email has a body type assignment
  [ ] Every email has a function assignment
  [ ] Word counts calculated for all emails
  [ ] No duplicate position numbers

IF any validation fails:
  Log specific failure with email position and field.
  IF > 20% of emails have issues: HALT -- sequence is incomplete.
  IF <= 20%: Log warnings, proceed with valid emails, flag incomplete ones.
```

### Step 5: Load Shared Reference Files

Confirm availability of shared reference files needed for scoring:
- `email-quality-rubric.md` -- required for Layer 0.2 scoring
- `niche-adaptation-guide.md` -- required for niche appropriateness checks

```
IF email-quality-rubric.md not found:
  HALT -- Cannot score without rubric.

IF niche-adaptation-guide.md not found:
  WARN -- Niche scoring will be limited. Proceed with flag.
```

### Step 6: Build Working Sequence Object

Assemble all parsed data into a structured working object that downstream microskills will consume.

## Failure Modes

| Failure | Remediation |
|---------|-------------|
| E3 assembled sequence file not found | HALT -- cannot proceed. Request E3 execution first. |
| Sequence file is malformed YAML | HALT -- cannot parse. Request E3 re-export. |
| More than 20% of emails incomplete | HALT -- sequence too incomplete for editorial. |
| Subject lines missing from emails | Log which emails lack SLs. Proceed -- SL-body alignment check will flag these as P1. |
| Body text empty for an email | Log as critical. This email will score 0 baseline. Proceed -- will be flagged as P1. |
| Rubric file not found | HALT -- cannot score without rubric. |
