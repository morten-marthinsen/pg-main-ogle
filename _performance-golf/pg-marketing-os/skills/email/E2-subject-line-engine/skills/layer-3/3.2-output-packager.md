# 3.2-output-packager.md

> **Layer:** 3 (Selection & Output)
> **Skill:** E2-subject-line-engine
> **Purpose:** Package top 5-10 ranked SLs into subject-line-package.yaml and SUBJECT-LINE-SUMMARY.md
> **Outputs:** subject-line-package.yaml (PRIMARY), SUBJECT-LINE-SUMMARY.md (human handoff), execution-log.md

---

## FUNCTION

Package the final ranked subject lines from 3.1 into structured output files for downstream consumption by E3 (sequence assembly) and E4 (editorial). Produce three outputs: the primary YAML package with all structured data, the human-readable summary with rationale, and the execution log confirming all steps completed.

---

## OUTPUT PACKAGING PROTOCOL

```
EXECUTE 3.2-output-packager:
  INPUT: sl_ranking (from 3.1), email_context, execution history
  OUTPUT: subject-line-package.yaml, SUBJECT-LINE-SUMMARY.md, execution-log.md

  1. VERIFY all required data is available:
     a. sl_ranking.ranked_sls has 5-10 entries
     b. All entries have weighted_score >= 8.0
     c. All entries have sl_body_alignment >= 7.0
     d. All entries have word_count <= 10
     e. Human selection was received (HUMAN_SELECTION.yaml exists)

  2. BUILD subject-line-package.yaml:
     a. Header with email ID, timestamp, skill version
     b. Selected SLs with full scoring data
     c. Formula metadata per SL
     d. Alignment evidence per SL
     e. Batch context (if batch mode)
     f. Downstream handoff data for E3 and E4

  3. BUILD SUBJECT-LINE-SUMMARY.md:
     a. Executive summary (email identified, top SL, key scores)
     b. Full ranked list with rationale
     c. Formula category distribution
     d. Niche calibration notes
     e. Recommendations for human (which SL and why)
     f. Arena highlights (what worked, what was cut)

  4. BUILD execution-log.md:
     a. Complete microskill execution record
     b. Spec files read: Y/N per microskill
     c. Output files created: Y/N per microskill
     d. File sizes per output
     e. Gate results per layer
     f. Arena round summaries
     g. Timestamp per major action

  5. VERIFY all three outputs:
     a. YAML is valid and parseable
     b. Summary contains all required sections
     c. Execution log covers all microskills
     d. All file sizes meet minimums

  6. CREATE LAYER_3_COMPLETE.yaml checkpoint
```
