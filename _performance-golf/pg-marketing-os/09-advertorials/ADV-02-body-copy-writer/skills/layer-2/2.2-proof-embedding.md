# 2.2-proof-embedding.md

> **Layer:** 2 (Generation)
> **Skill:** ADV-02-body-copy-writer
> **Purpose:** Weave proof elements into narrative at planned positions (max 1 per paragraph)
> **Outputs:** `proof_embedded_body` with complete body including proof

---

## FUNCTION

Take the generated body sections (with [PROOF_SLOT] markers) and embed the planned proof elements using editorial framing. Each proof element is woven into the narrative so it feels like part of the article rather than a sales claim. Enforce the density rule: maximum 1 proof element per paragraph, maximum 2 consecutive proof paragraphs.

---

## EMBEDDING PROTOCOL

### Step 1: Retrieve Proof Plan

```
FROM proof_weaving_plan:
  FOR each section with proof_slot:
    IDENTIFY:
      - Which proof element goes here (proof_id)
      - Editorial frame to use
      - Weaving technique (inline, supporting, closing)
```

### Step 2: Embed Proof Elements

```
FOR each [PROOF_SLOT] in body_sections:
  RETRIEVE proof element from plan
  APPLY editorial framing:

  TECHNIQUE: inline_attribution
    "According to a study published in [Journal], [finding]."
    Integration: Proof becomes part of the sentence flow

  TECHNIQUE: supporting_evidence
    "[Narrative claim]. This aligns with research from [source],
    which found [evidence]."
    Integration: Proof supports a narrative claim

  TECHNIQUE: closing_anchor
    "[Narrative discussion]. As [Expert] at [Institution] put it,
    '[quote].'"
    Integration: Proof anchors the end of a section

  EMBED naturally:
    - Proof should feel like part of the article, not an interruption
    - Attribution must be specific (named source, institution, journal)
    - Transition into proof must be smooth
    - Transition out of proof must return to narrative
```

### Step 3: Density Enforcement

```
AFTER all proof embedded:
  COUNT proof elements per paragraph:
    IF any paragraph has 2+ proof elements:
      FAIL: "Proof density violation -- paragraph [N] has [count] elements"
      ACTION: Move excess proof to different paragraph or remove

  COUNT consecutive proof paragraphs:
    IF 3+ consecutive paragraphs contain proof:
      FAIL: "Proof stacking detected -- [count] consecutive proof paragraphs"
      ACTION: Insert narrative paragraph between proof sections

  VERIFY rhythm:
    IDEAL: N-P-N-P-N (narrative-proof alternation)
    ACCEPTABLE: N-P-P-N-P-N (max 2 consecutive)
    FORBIDDEN: N-P-P-P-N (3+ consecutive)
```

### Step 4: Post-Embedding Quality Check

```
FOR each embedded proof element:
  CHECK:
    [ ] Reads naturally in context (not forced)
    [ ] Attribution is specific (not vague)
    [ ] Editorial framing maintained (not promotional)
    [ ] Transition in/out smooth
    [ ] No stacking with adjacent proofs
    [ ] Proof content accurate to source
```

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| Density violation | 2+ proof per paragraph | Move excess to different paragraph |
| Stacking violation | 3+ consecutive proof paragraphs | Insert narrative break |
| Forced embedding | Proof doesn't fit section naturally | Adjust framing or move to better section |
| Promotional framing | Proof framed as ad claim | Rewrite with editorial attribution |
| Missing attribution | Proof lacks specific source | Add specific source or remove proof |
