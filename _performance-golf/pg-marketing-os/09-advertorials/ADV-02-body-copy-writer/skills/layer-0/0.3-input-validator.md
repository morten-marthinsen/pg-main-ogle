# 0.3-input-validator.md

> **Layer:** 0 (Foundation)
> **Skill:** ADV-02-body-copy-writer
> **Purpose:** Validate type-specific inputs present for body copy generation
> **Outputs:** `validation_result` with PASS/FAIL

---

## FUNCTION

Validate that all type-specific inputs required for body copy generation are present. Different advertorial types require different inputs -- a listicle needs list item data, a native article needs journalistic context, a product review needs feature comparison data.

---

## VALIDATION PROTOCOL

### Step 1: Universal Validations

```
REQUIRED (all types):
  [ ] strategy.type.selected -- valid type
  [ ] strategy.proof_inventory -- at least 1 element
  [ ] lead_handoff.voice_register -- defined
  [ ] lead_handoff.narrative_thread -- defined
  [ ] mechanism.name -- defined
  [ ] specimens loaded -- at least 3
```

### Step 2: Type-Specific Validations

```
IF type == listicle:
  [ ] List item count defined (3-10)
  [ ] Each item has a proof element assigned

IF type == native_article:
  [ ] Journalistic context available (source, institution, research)
  [ ] Expert quotes or citations available

IF type == blog_style:
  [ ] First-person narrative elements available
  [ ] Personal experience framing defined

IF type == product_review:
  [ ] Feature list for comparison
  [ ] Alternative products for context

IF type == PAS:
  [ ] Problem clearly defined with emotional weight
  [ ] Agitation elements available
  [ ] Solution framing mapped to mechanism

IF type == sponsored_content:
  [ ] Brand voice guidelines available
  [ ] Content value proposition defined

IF type == editorial_feature:
  [ ] Expert sources identified
  [ ] Investigation narrative available
```

---

## OUTPUT STRUCTURE

```json
{
  "validation_result": {
    "status": "PASS|FAIL",
    "universal_checks": {"all_passed": true},
    "type_specific_checks": {"type": "native_article", "all_passed": true},
    "warnings": [],
    "proceed_to_layer_1": true
  }
}
```

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Strategy or lead missing | HALT -- Execute upstream |
| CRITICAL | Zero proof elements | HALT -- Cannot write proof-woven body |
| DEGRADED | Type-specific inputs thin | Proceed with available data, flag |
