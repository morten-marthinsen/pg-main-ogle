# 0.1-upstream-loader.md

> **Layer:** 0 (Foundation)
> **Skill:** ADV-02-body-copy-writer
> **Purpose:** Load strategy, lead draft, and campaign brief for body copy generation
> **Outputs:** `upstream_context` object with all body-relevant data

---

## FUNCTION

Load all upstream outputs required for body copy generation. The body depends on three upstream sources: the strategy (for type, structure, proof inventory), the lead draft (for voice register, narrative thread, mechanism status), and the campaign brief (for Big Idea, mechanism, promise).

---

## REQUIRED UPSTREAM PACKAGES

| Package | Source | Critical Fields | Required |
|---------|--------|-----------------|----------|
| `advertorial-strategy.yaml` | ADV-00 | type, proof_inventory, downstream_instructions.adv_02 | YES |
| `advertorial-lead-draft.md` | ADV-01 | body_handoff, voice_register, mechanism_status | YES |
| `campaign-brief-package.json` | 01-core-message | big_idea, mechanism, promise, proof | YES |

---

## LOADING PROTOCOL

```
1. Load advertorial-strategy.yaml
   EXTRACT: type, type_constraints, proof_inventory, five_laws, adv_02_instructions
2. Load advertorial-lead-draft.md
   EXTRACT: body_handoff (narrative_thread, voice, mechanism_status, reader_expectation)
3. Load campaign-brief-package.json
   EXTRACT: mechanism details, promise, proof pairings
4. VALIDATE all critical fields present
5. ASSEMBLE upstream_context
```
