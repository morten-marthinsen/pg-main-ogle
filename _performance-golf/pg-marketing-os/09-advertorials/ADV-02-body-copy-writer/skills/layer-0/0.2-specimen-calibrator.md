# 0.2-specimen-calibrator.md

> **Layer:** 0 (Foundation)
> **Skill:** ADV-02-body-copy-writer
> **Purpose:** Load specimens matching selected advertorial type for body calibration
> **Outputs:** `body_specimen_patterns` object with structure and proof-weaving patterns

---

## FUNCTION

Load advertorial specimens that match the selected type and decompose their body sections. Extract structure patterns (how sections are organized), proof-weaving techniques (how proof is embedded in narrative), mechanism introduction patterns, and editorial voice markers specific to body copy.

---

## CALIBRATION PROTOCOL

### Step 1: Load Type-Matched Specimens

```
FROM strategy.advertorial_type:
  LOAD specimens matching type
  MINIMUM: 3 body sections from specimens
  FOCUS: Body section only (skip hooks/leads/bridges)
```

### Step 2: Decompose Body Structures

```
FOR each specimen body:
  EXTRACT:
    - section_count: Number of body sections
    - section_types: [context, mechanism, proof, narrative, transition]
    - section_order: How sections are arranged
    - word_count_per_section: Average length per section
    - total_word_count: Total body word count
    - proof_placement: Where proof elements appear
    - proof_density: Proof per paragraph ratio
    - mechanism_introduction_point: Which section introduces mechanism
    - product_name_first_mention: Where product first named
    - transition_technique: How body transitions to bridge
```

### Step 3: Extract Proof-Weaving Patterns

```
FOR each specimen:
  ANALYZE proof integration:
    - weaving_technique: How proof is embedded (inline, supporting, closing)
    - framing_language: "According to...", "A study published in...", etc.
    - proof_types_used: Which proof types in which positions
    - narrative_recovery: How narrative resumes after proof paragraph
    - density_pattern: How many paragraphs between proof elements
```

### Step 4: Build Body Calibration Reference

```
ASSEMBLE:
  - Typical section count for type
  - Typical proof placement pattern
  - Typical mechanism introduction point
  - Editorial framing language for proof
  - Quality benchmarks for body copy
```

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Zero specimens | HALT -- Cannot calibrate |
| DEGRADED | < 3 specimens | Proceed with limited calibration, flag |
| DEGRADED | No type match | Use closest type, flag mismatch |
