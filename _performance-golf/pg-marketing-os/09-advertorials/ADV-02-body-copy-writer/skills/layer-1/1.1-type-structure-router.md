# 1.1-type-structure-router.md

> **Layer:** 1 (Architecture)
> **Skill:** ADV-02-body-copy-writer
> **Purpose:** Route to type-specific body structure template
> **Outputs:** `body_structure_template` with section plan for selected type

---

## FUNCTION

Select and configure the body structure template that matches the selected advertorial type. Each type has a distinct section order, section count, pacing rhythm, and mechanism introduction pattern. This microskill ensures body generation follows the correct structural conventions.

---

## TYPE STRUCTURE TEMPLATES

### Listicle Body Structure
```
Section 1: Context setter (why this list matters)
Section 2-N: List items (each: claim + proof + narrative)
  - Item structure: editorial statement -> supporting evidence -> implication
  - Mechanism woven into 1-2 items (not all)
  - Proof distributed: 1 element per item max
Final Section: Synthesis (what the list means collectively)
Word target: 500-800 for body
```

### Native Article Body Structure
```
Section 1: Background context (the research/trend/event)
Section 2: The finding (what was discovered -- mechanism tease)
Section 3: Expert perspective (quote or interpretation)
Section 4: Mechanism explanation (editorial framing)
Section 5: Implications (what this means for reader)
Section 6: Current status (what's available now -- bridge setup)
Word target: 600-1000 for body
```

### Blog-Style Body Structure
```
Section 1: Personal context ("My experience with [problem]...")
Section 2: What I tried (conventional approaches that failed)
Section 3: The discovery (how I found the mechanism)
Section 4: My experience with it (personal results)
Section 5: What I learned (broader implications)
Word target: 500-800 for body
```

### Product Review Body Structure
```
Section 1: Review context (why reviewing this)
Section 2: What it is (mechanism explanation, editorial framing)
Section 3: How it works (mechanism detail)
Section 4: Pros (with proof support)
Section 5: Cons (authentic balance)
Section 6: Verdict setup (leads into bridge)
Word target: 600-900 for body
```

### PAS Body Structure
```
Section 1: Problem deep-dive (expand on pain, add data)
Section 2: Agitation (consequences of inaction, emotional stakes)
Section 3: Solution introduction (mechanism as answer)
Section 4: Solution evidence (proof integration)
Section 5: Solution implications (what changes)
Word target: 500-800 for body
```

### Sponsored Content Body Structure
```
Section 1: Trend or topic context (broader editorial frame)
Section 2: Why this matters now (timeliness)
Section 3: The approach (mechanism as part of larger trend)
Section 4: Evidence and perspectives (proof + expert views)
Section 5: What readers should know (practical takeaway)
Word target: 700-1200 for body
```

### Editorial Feature Body Structure
```
Section 1: The investigation premise (what prompted the piece)
Section 2: Background research (historical/scientific context)
Section 3: Key finding (mechanism as central discovery)
Section 4: Expert interviews (multiple perspectives)
Section 5: Counter-arguments or limitations (editorial balance)
Section 6: Conclusions and implications (setup for bridge)
Word target: 800-1400 for body
```

---

## ROUTING PROTOCOL

```
FROM strategy.advertorial_type:
  1. SELECT matching template
  2. CONFIGURE section count for content volume
  3. MAP mechanism introduction to appropriate section
  4. SET word count targets per section
  5. DEFINE pacing rhythm (narrative-proof alternation)
  6. IDENTIFY bridge transition section

IF type not in templates:
  FALLBACK to native_article (most versatile)
  FLAG: "Using fallback template -- non-standard type"
```

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| Type not in templates | Custom or unknown type | Fallback to native_article |
| Section count mismatch | Content volume doesn't fit template | Adjust section count within type |
| Word target conflict | Platform limit vs type requirement | Prioritize platform limit |
