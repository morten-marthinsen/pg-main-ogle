# 1.2-proof-weaving-planner.md

> **Layer:** 1 (Architecture)
> **Skill:** ADV-02-body-copy-writer
> **Purpose:** Plan where proof elements embed in narrative with density enforcement
> **Outputs:** `proof_weaving_plan` with placement map and density rules

---

## FUNCTION

Create a precise plan for where and how proof elements are embedded in the body narrative. The plan must enforce density rules (max 1 per paragraph, max 2 consecutive proof paragraphs), select which proof elements to use from inventory, and define editorial framing for each embedded proof.

---

## PLANNING PROTOCOL

### Step 1: Inventory Assessment

```
FROM strategy.proof_inventory:
  LIST all available proof elements:
    - clinical_studies (highest editorial authority)
    - expert_quotes (strong editorial voice)
    - statistics (concrete and specific)
    - testimonials (peer credibility)
    - case_studies (narrative-compatible)
    - mechanism_evidence (explains how it works)

  RANK by:
    1. strength (high/medium/low)
    2. editorial_compatibility (how naturally it fits in article)
    3. embedding_priority (from strategy)

  SELECT top 3-5 elements for body embedding
  RESERVE others for bridge or omission
```

### Step 2: Placement Mapping

```
FROM body_structure_template:
  MAP proof to sections:

  RULE 1: Maximum 1 proof element per paragraph
  RULE 2: Maximum 2 consecutive proof-containing paragraphs
  RULE 3: After 2 consecutive proof paragraphs, MUST have narrative paragraph
  RULE 4: Proof rhythm: narrative -> proof -> narrative -> proof (ideal)

  EXAMPLE MAPPING (native_article, 6 sections):
    Section 1 (context): NO proof -- pure editorial setup
    Section 2 (finding): PROOF #1 -- study or statistic supporting the finding
    Section 3 (expert): NO proof -- narrative/perspective paragraph
    Section 4 (mechanism): PROOF #2 -- mechanism evidence
    Section 5 (implications): PROOF #3 -- testimonial or case study
    Section 6 (transition): NO proof -- bridge setup

  VERIFY:
    - No section has 2+ proof elements
    - No 3+ consecutive sections contain proof
    - Total proof count <= inventory available
```

### Step 3: Define Editorial Framing

```
FOR each placed proof element:
  DEFINE editorial_frame:
    clinical_study: "In a study published in [Journal], researchers found..."
    expert_quote: "[Name], a [title] at [institution], explains: '...'"
    statistic: "According to data from [source], [X]% of..."
    testimonial: "[Name], a [age]-year-old [occupation] from [location]..."
    case_study: "When [Name] began [approach], the results..."
    mechanism_evidence: "The compound works by [mechanism], which [study] demonstrated..."

  FORBIDDEN framing:
    - "Our research shows..." (promotional)
    - "Customers love..." (promotional)
    - "As seen in our clinical trials..." (promotional)
    - "9 out of 10 users..." (infomercial)
```

### Step 4: Density Validation

```
VALIDATE plan against density rules:
  [ ] Max 1 proof per paragraph: CHECKED
  [ ] Max 2 consecutive proof paragraphs: CHECKED
  [ ] No 3+ proof sequence: CHECKED
  [ ] Narrative recovery after proof: CHECKED
  [ ] Total proof count reasonable (3-5 for typical body): CHECKED

IF any rule violated in plan:
  REVISE: Redistribute proof to comply
  LOG: "Density violation fixed in planning phase"
```

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| Proof stacking in plan | 3+ consecutive proof sections | Redistribute with narrative breaks |
| Insufficient proof | < 2 proof elements available | Flag thin-proof body, use narrative strength |
| No editorial framing | Proof elements lack attribution sources | Create editorial frames from available context |
| Over-proving | Trying to use all proof | Select top 3-5, reserve rest |
