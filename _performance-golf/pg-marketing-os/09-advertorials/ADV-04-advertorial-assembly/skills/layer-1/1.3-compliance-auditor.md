# 1.3-compliance-auditor.md

> **Layer:** 1 (Assembly)
> **Skill:** ADV-04-advertorial-assembly
> **Purpose:** Run platform compliance check on assembled advertorial
> **Outputs:** `compliance_result` with PASS/FAIL and violation report

---

## FUNCTION

Audit the assembled advertorial against the target platform's compliance requirements. Check word count, disclosure placement, headline character limits, claim substantiation, CTA count, and content policy compliance.

---

## COMPLIANCE AUDIT PROTOCOL

### Step 1: Word Count Check

```
FROM strategy_reference.word_count_bounds:
  CHECK: assembled_word_count within [min, max]

  IF below minimum:
    FLAG: "Word count [N] below minimum [min]"
    SEVERITY: WARNING (may need content expansion in ADV-05)

  IF above maximum:
    FLAG: "Word count [N] above maximum [max]"
    SEVERITY: WARNING (may need trimming in ADV-05)
```

### Step 2: Disclosure Check

```
IF strategy_reference.disclosure_required:
  CHECK:
    [ ] Disclosure text present in assembled document
    [ ] Disclosure in correct position (top/bottom/both)
    [ ] Disclosure text matches platform requirements
    [ ] Disclosure visible (not hidden or obscured)

  IF any check fails:
    FLAG: "Disclosure violation: [specific issue]"
    SEVERITY: CRITICAL (platform rejection risk)
```

### Step 3: Headline Check

```
FROM strategy_reference.platform.headline_rules:
  CHECK hook/headline:
    [ ] Within character limit
    [ ] No forbidden patterns used
    [ ] No ALL CAPS words (except acronyms)
    [ ] No excessive punctuation
```

### Step 4: CTA Compliance Check

```
CHECK CTA in assembled piece:
  [ ] Exactly 1 CTA present
  [ ] CTA is text link format
  [ ] CTA language appropriate (no "Buy Now")
  [ ] CTA destination URL valid
  [ ] No secondary/hidden CTAs in body
```

### Step 5: Content Policy Check

```
CHECK against platform content policies:
  [ ] No before/after claims without disclaimer
  [ ] No unsubstantiated health claims
  [ ] No personal attribute assertions (Meta)
  [ ] No misleading imagery references
  [ ] All claims attributed to sources
  [ ] Testimonials disclosed if present
```

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| Disclosure missing | Not placed during assembly | Add disclosure immediately |
| Word count violation | Outside bounds | Flag for ADV-05 revision |
| Headline too long | Exceeds char limit | Trim headline |
| Multiple CTAs | Stray links found | Remove non-intentional CTAs |
| Content policy violation | Unsubstantiated claims | Add attribution or remove claim |
