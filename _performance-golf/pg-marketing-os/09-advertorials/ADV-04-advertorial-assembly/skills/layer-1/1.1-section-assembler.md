# 1.1-section-assembler.md

> **Layer:** 1 (Assembly)
> **Skill:** ADV-04-advertorial-assembly
> **Purpose:** Assemble hook, lead, body, and bridge into single document
> **Outputs:** `assembled_document` with complete advertorial text

---

## FUNCTION

Concatenate all section drafts in the correct order with proper formatting. Add disclosure per platform requirements. Smooth seams between sections with minimal transitional elements (max 1 connecting sentence per seam). Do NOT modify section content -- assembly scope only.

---

## ASSEMBLY PROTOCOL

### Step 1: Add Disclosure

```
IF platform_compliance.disclosure_required:
  PLACE disclosure per rules:
    IF placement == "top":
      INSERT disclosure_text at very beginning of document
    IF placement == "bottom":
      INSERT disclosure_text at very end
    IF placement == "both":
      INSERT at both beginning and end

  DISCLOSURE FORMAT:
    "[Sponsored Content]" or "[Paid Advertisement]"
    Styled as small text, not prominent
```

### Step 2: Assemble Sections

```
BUILD assembled document in order:

  [DISCLOSURE if top]

  ## [Hook]
  [Hook text from lead draft]

  [Lead paragraphs from lead draft]

  [Body sections from body draft]

  [Bridge paragraphs from bridge draft]

  [CTA from bridge draft]

  [DISCLOSURE if bottom]
```

### Step 3: Smooth Seams

```
FOR each seam (4 total):
  READ last sentence of preceding section
  READ first sentence of following section
  EVALUATE: Do they flow naturally?

  IF natural flow: No modification needed
  IF minor gap: Add 1 transitional sentence (max)
  IF major gap: FLAG for voice-consistency-checker

  SCOPE CONSTRAINT:
    - Transitional sentence must match voice register
    - Must not introduce new information
    - Must not add proof or sales language
    - Maximum 1 sentence per seam
```

### Step 4: Format Final Document

```
FORMAT:
  - Clean paragraph breaks between sections
  - No section headers visible (reads as continuous article)
  - CTA formatted as inline text link
  - Disclosure formatted per platform convention
  - Remove any metadata/YAML frontmatter from section drafts
```

---

## OUTPUT STRUCTURE

```json
{
  "assembled_document": {
    "full_text": "Complete assembled advertorial text",
    "word_count": 1088,
    "section_order": ["disclosure", "hook", "lead", "body", "bridge", "cta"],
    "transitional_sentences_added": 1,
    "seams_checked": 4,
    "disclosure_placed": true,
    "disclosure_position": "top",
    "formatting_clean": true
  }
}
```

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| Section order wrong | Sections assembled out of sequence | Reorder per protocol |
| Seam too rough | Major gap between sections | Flag for voice checker |
| Disclosure missing | Forgot to add required disclosure | Add per platform rules |
| Metadata leaked | YAML frontmatter in final output | Strip metadata |
| Content modified | Scope creep during assembly | Revert to original section text |
