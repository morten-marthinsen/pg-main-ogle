# 0.2-strategy-loader.md

> **Layer:** 0 (Foundation)
> **Skill:** ADV-04-advertorial-assembly
> **Purpose:** Load advertorial strategy for validation reference during assembly
> **Outputs:** `strategy_reference` object with compliance and type constraints

---

## FUNCTION

Load the advertorial strategy to use as validation reference during assembly. The strategy provides type-specific word count bounds, platform compliance requirements, disclosure placement rules, and voice register specifications.

---

## LOADING PROTOCOL

```
Load advertorial-strategy.yaml (from ADV-00):
  EXTRACT for assembly validation:
    - type_constraints.word_count_range: [min, max]
    - platform_compliance: headline rules, disclosure, content rules
    - editorial_angle.voice_register: expected voice
    - five_laws_compliance: validation reference
    - downstream_instructions.adv_04: assembly-specific instructions

VERIFY: strategy exists and contains above fields
```

---

## OUTPUT STRUCTURE

```json
{
  "strategy_reference": {
    "word_count_bounds": [800, 1500],
    "platform": "taboola",
    "disclosure_required": true,
    "disclosure_placement": "top",
    "disclosure_text": "Sponsored Content",
    "voice_register": "journalistic",
    "five_laws": {},
    "assembly_instructions": "specific instructions"
  }
}
```

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Strategy missing | HALT -- Execute ADV-00 |
| DEGRADED | Compliance section incomplete | Use most restrictive defaults |
