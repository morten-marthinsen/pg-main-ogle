# 0.1-upstream-loader.md

> **Layer:** 0 (Foundation)
> **Skill:** ADV-04-advertorial-assembly
> **Purpose:** Load lead, body, and bridge drafts for assembly
> **Outputs:** `section_drafts` object with all three section texts

---

## FUNCTION

Load all three section drafts produced by upstream skills. Verify each section is present, non-empty, and contains its expected metadata. Extract section text and handoff data for assembly.

---

## LOADING PROTOCOL

```
1. Load advertorial-lead-draft.md (from ADV-01)
   EXTRACT: hook text, lead paragraphs, body handoff
   VERIFY: non-empty, hook present, lead present

2. Load advertorial-body-draft.md (from ADV-02)
   EXTRACT: body sections, proof density report, bridge handoff
   VERIFY: non-empty, sections present, proof compliant

3. Load advertorial-bridge-draft.md (from ADV-03)
   EXTRACT: bridge paragraphs, CTA (text + URL), assembly handoff
   VERIFY: non-empty, bridge present, CTA present and singular

IF any section missing: FAIL -- "Cannot assemble without all sections"
```

---

## OUTPUT STRUCTURE

```json
{
  "section_drafts": {
    "lead": {
      "hook": "Hook text",
      "lead": "Lead paragraphs",
      "word_count": 213,
      "voice_register": "journalistic"
    },
    "body": {
      "sections": [],
      "word_count": 720,
      "proof_density_compliant": true
    },
    "bridge": {
      "paragraphs": [],
      "cta": {"text": "...", "url": "...", "format": "text_link"},
      "word_count": 140
    },
    "total_word_count": 1073,
    "all_loaded": true
  }
}
```

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Lead draft missing | HALT -- Execute ADV-01 |
| CRITICAL | Body draft missing | HALT -- Execute ADV-02 |
| CRITICAL | Bridge draft missing | HALT -- Execute ADV-03 |
| CRITICAL | CTA missing from bridge | HALT -- Re-execute ADV-03 |
