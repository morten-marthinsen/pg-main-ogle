# 2.1-narrative-thread-validator.md

> **Layer:** 2 (Verification)
> **Skill:** ADV-04-advertorial-assembly
> **Purpose:** Verify unbroken narrative thread from hook through bridge
> **Outputs:** `narrative_validation` with thread analysis and continuity score

---

## FUNCTION

Read the complete assembled advertorial as a reader would -- from first word to last -- and verify that the narrative thread is continuous, logical, and engaging throughout. Identify any points where the reader might lose the thread, encounter a logical gap, or feel the piece is disjointed.

---

## NARRATIVE VALIDATION PROTOCOL

### Step 1: Thread Identification

```
READ complete assembled advertorial and identify:
  - Primary thread: The main narrative or argument running through the piece
  - Secondary threads: Supporting narratives or evidence chains
  - Mechanism thread: How mechanism is teased, introduced, and explained
  - Emotional thread: The emotional journey from hook to CTA

DOCUMENT each thread:
  thread_name: [name]
  start_point: [where thread begins]
  end_point: [where thread resolves or hands off]
  continuity: [continuous/broken/intermittent]
```

### Step 2: Gap Analysis

```
FOR each identified thread:
  SCAN for gaps:
    - Information gaps: Does reader need information not provided?
    - Logical gaps: Does a conclusion not follow from premises?
    - Temporal gaps: Does timeline skip unexplained?
    - Emotional gaps: Does emotional tone shift without reason?
    - Reference gaps: Does text reference something not introduced?

  FOR each gap found:
    LOG:
      - gap_type: information|logical|temporal|emotional|reference
      - location: paragraph and sentence number
      - severity: minor (reader might not notice) | major (reader confused)
      - remediation: specific fix recommendation
```

### Step 3: Reader Experience Simulation

```
SIMULATE reader journey:
  AT hook: "What does reader expect next?"
  AT lead end: "Is reader pulled into body?"
  AT body midpoint: "Is reader still engaged?"
  AT body end: "Is reader ready for bridge?"
  AT bridge: "Does recommendation feel natural?"
  AT CTA: "Would reader click?"

  SCORE each transition point (1-10)
  MINIMUM per point: 7
```

### Step 4: Callback Verification

```
CHECK for narrative callbacks:
  - Does body reference hook elements?
  - Does bridge reference body insights?
  - Are mechanism references consistent throughout?
  - Do proof elements connect to earlier claims?

  Callbacks found: [list]
  Callbacks expected but missing: [list]
```

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| Major gap found | Logical or information gap | Add connecting information |
| Thread breaks | Narrative thread discontinuous | Bridge with transitional element |
| Reader disengagement point | Journey score < 7 at any point | Strengthen weak point |
| Missing callbacks | Expected callbacks absent | Add callback references |
