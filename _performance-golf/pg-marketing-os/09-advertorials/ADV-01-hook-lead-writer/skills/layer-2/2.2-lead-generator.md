# 2.2-lead-generator.md

> **Layer:** 2 (Generation)
> **Skill:** ADV-01-hook-lead-writer
> **Purpose:** Generate lead paragraphs establishing editorial voice for each hook
> **Outputs:** `lead_candidates` array paired with corresponding hooks

---

## FUNCTION

Generate 3-5 lead paragraphs for each passing hook candidate. The lead must sustain the hook's momentum, establish the editorial voice register that carries through the entire advertorial, and begin teasing the mechanism without naming the product. Each lead is paired with its hook to form a complete hook/lead combination.

---

## GENERATION PROTOCOL

### Step 1: Generate Leads Per Hook

```
FOR each hook in hook_candidates (passing):
  USING lead_structure plan from 1.2:
    GENERATE 3-5 lead paragraphs:

    Paragraph 1: CONTEXT EXPANSION
      - Expand on hook's premise
      - Add depth without repeating hook
      - Maintain same voice register as hook
      - Target: 2-3 sentences

    Paragraph 2: READER RELEVANCE
      - Connect hook context to reader's life/problem
      - Use audience pain_language from brief
      - Build emotional engagement
      - Target: 2-3 sentences

    Paragraph 3: SCOPE/STAKES
      - Expand scope (how many affected, how long problem persisted)
      - Raise stakes (what happens if nothing changes)
      - First mechanism tease (Level 1 maximum)
      - Target: 3-4 sentences

    Paragraph 4: FORWARD PULL
      - Tease the mechanism (Level 2 maximum)
      - Create specific curiosity about what comes next
      - Final sentence MUST pull reader into body copy
      - Target: 2-3 sentences

    CRITICAL CONSTRAINTS:
      - Product name: NEVER in any lead paragraph
      - Voice register: Must match hook's register exactly
      - Promotional language: Zero tolerance
      - Word count: 100-250 words total for lead
```

### Step 2: Apply Voice Markers

```
FOR each lead:
  INJECT voice markers from lead_structure.voice_plan:

  IF journalistic:
    - Third-person attribution in paragraph 1 or 2
    - At least one institutional reference
    - Hedging language for unproven claims
    - Source diversity (2+ attribution types)

  IF conversational:
    - First-person experience marker
    - Reader direct address
    - At least one colloquial phrase
    - Personal credibility marker

  IF expert:
    - Technical-accessible language balance
    - Process explanation
    - Evidence-first ordering

  IF peer:
    - Shared experience reference
    - Discovery language
    - Authentic uncertainty marker
```

### Step 3: Validate Each Lead

```
FOR each generated lead:
  VALIDATE:
    [ ] Voice register consistent with hook (no drift)
    [ ] Product name absent from all paragraphs
    [ ] Mechanism tease does not exceed Level 2
    [ ] No promotional language detected
    [ ] Word count within 100-250 range
    [ ] Last sentence creates forward pull
    [ ] Editorial smell test: remove product context, still reads as article

  IF any validation fails:
    REVISE specific paragraph
    RE-VALIDATE
    LOG revision reason
```

### Step 4: Pair Hooks and Leads

```
CREATE hook_lead_combinations:
  FOR each hook:
    PAIR with its best-fitting lead
    VERIFY combination reads as unified piece (no seam between hook and lead)
    SCORE combination coherence (1-10)

  MINIMUM: 5 hook/lead combinations must pass to Arena
```

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| Voice drift between hook and lead | Register shifts between sections | Rewrite lead matching hook register |
| Product name in lead | Automatic rejection | Replace with mechanism tease language |
| Lead too promotional | Selling language detected | Rewrite through editorial lens |
| Weak forward pull | Last paragraph doesn't create momentum | Strengthen curiosity in final sentence |
| Fewer than 5 combinations | Too many filtered out | Generate additional hooks and/or leads |
