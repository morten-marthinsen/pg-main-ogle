# 0.2-specimen-calibrator.md

> **Layer:** 0 (Foundation)
> **Skill:** ADV-05-advertorial-editorial
> **Purpose:** Load specimens for editorial quality comparison
> **Outputs:** `editorial_benchmarks` object with quality reference points

---

## FUNCTION

Load advertorial specimens for comparative editorial review. The editorial review compares the assembled advertorial against known-good specimens to calibrate quality judgment. Extract editorial quality markers, voice consistency patterns, and overall publication readiness indicators.

---

## CALIBRATION PROTOCOL

### Step 1: Load Comparison Specimens

```
FROM specimen collection:
  LOAD 3-5 specimens matching advertorial type
  FOCUS: Overall editorial quality characteristics
  EXTRACT per specimen:
    - Editorial quality markers (what makes it read as genuine editorial)
    - Voice consistency indicators (how voice stays locked throughout)
    - Proof integration technique (how proof is woven without ad-smell)
    - Bridge quality (how recommendation feels natural)
    - Hook effectiveness (what creates scroll-stop in context)
```

### Step 2: Build Quality Benchmarks

```
FROM specimens:
  ESTABLISH benchmarks:
    - editorial_authenticity_benchmark: What "genuine editorial" looks like
    - voice_unity_benchmark: What "one voice throughout" looks like
    - proof_integration_benchmark: What "invisible proof" looks like
    - bridge_naturalness_benchmark: What "natural recommendation" looks like
    - overall_quality_benchmark: What "publication-ready" looks like
```

---

## OUTPUT STRUCTURE

```json
{
  "editorial_benchmarks": {

    "benchmarks": {

    ]
  }

```

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| DEGRADED | < 3 specimens | Proceed with formula-based review |
| DEGRADED | No type match | Use closest type specimens |
