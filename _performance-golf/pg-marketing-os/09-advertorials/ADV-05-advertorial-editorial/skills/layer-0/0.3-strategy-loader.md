# 0.3-strategy-loader.md

> **Layer:** 0 (Foundation)
> **Skill:** ADV-05-advertorial-editorial
> **Purpose:** Load original strategy for constraint checking during review
> **Outputs:** `strategy_constraints` object with review-relevant constraints

---

## FUNCTION

Load the original advertorial strategy (from ADV-00) to use as constraint reference during editorial review. The strategy defines what the advertorial should achieve, what constraints it must satisfy, and what the 5 Laws require.

---

## LOADING PROTOCOL

```
Load advertorial-strategy.yaml (from ADV-00):
  EXTRACT for editorial review:
    - type + type_constraints (word count, structure, proof density)
    - editorial_angle (what the advertorial should express)
    - platform_compliance (what platform requires)
    - five_laws_compliance (how each law should be satisfied)
    - proof_inventory (what proof was available)
    - voice_register (what voice should be used)

VERIFY: strategy exists and is parseable
```

---

## OUTPUT STRUCTURE

```json
{
  "strategy_constraints": {
    "type": "native_article",
    "word_count_bounds": [1000, 1500],
    "voice_register": "journalistic",
    "angle": "discovery_framed",
    "platform": "taboola",
    "five_laws": {},
    "proof_density_rules": {
      "max_per_paragraph": 1,
      "max_consecutive": 2
    },
    "compliance_rules": {}
  }
}
```

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Strategy missing | HALT -- Cannot review without constraints |
| DEGRADED | Partial strategy | Review with available constraints, flag gaps |
