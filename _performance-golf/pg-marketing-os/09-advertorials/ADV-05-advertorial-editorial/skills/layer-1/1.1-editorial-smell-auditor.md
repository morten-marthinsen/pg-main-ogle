# 1.1-editorial-smell-auditor.md

> **Layer:** 1 (Auditing)
> **Skill:** ADV-05-advertorial-editorial
> **Purpose:** Full editorial smell test on complete assembled advertorial
> **Outputs:** `smell_audit` object with findings and issue locations

---

## FUNCTION

Execute a comprehensive editorial smell test on the COMPLETE assembled advertorial. This is a FRESH smell test -- not recycled from ADV-02. The full-piece context may reveal issues invisible in isolated sections. Identify every point where the piece reads as ad rather than article.

---

## SMELL AUDIT PROTOCOL

### Step 1: Full Redaction Test

```
CREATE fully redacted version:
  REMOVE: product name, brand name, URLs, pricing, branded terms
  KEEP: mechanism name (if generic), category descriptions, sources

READ redacted version as a complete piece
EVALUATE: "Would this be publishable in a real publication?"
```

### Step 2: Section-by-Section Smell Analysis

```
FOR each section of assembled piece:

  HOOK:
    [ ] Reads as editorial headline? (not ad headline)
    [ ] No product name? (not "Introducing [Product]")
    [ ] Creates genuine curiosity? (not clickbait)

  LEAD:
    [ ] Reads as article opening? (not sales letter opening)
    [ ] Provides information? (not just setup for sell)
    [ ] Voice is editorial? (not promotional)

  BODY:
    [ ] Each paragraph reads as content? (not feature list)
    [ ] Proof woven naturally? (not evidence dump)
    [ ] Mechanism explained editorially? (not product pitch)
    [ ] Value independent of product? (useful information)

  BRIDGE:
    [ ] Transition gradual? (not abrupt pivot)
    [ ] Recommendation framing? (not sales close)
    [ ] Single CTA as text link? (not button/multi-CTA)

  FOR each check that FAILS:
    LOG:
      - section: [hook|lead|body|bridge]
      - paragraph: [number]
      - issue: [specific description]
      - severity: [minor|moderate|severe]
      - suggestion: [specific fix]
```

### Step 3: Ad-Smell Trigger Scan

```
SCAN complete piece for ad-smell triggers:
  - Product name appearing too frequently
  - Branded language density (how often brand referenced)
  - Feature-benefit language patterns
  - Superlative claims without attribution
  - Direct-response patterns ("Act now", "Limited time")
  - Testimonial stacking
  - Before/after implications
  - Price anchoring language

COUNT triggers per section
MAP trigger density across piece
IDENTIFY hotspots (sections with concentrated triggers)
```

### Step 4: Publication Readiness Assessment

```
ASSESS overall:
  - Would a magazine editor publish this? [Y/N]
  - Would a blog editor publish this? [Y/N]
  - Would a reader feel deceived when they realize it's sponsored? [Y/N]
  - Does the piece deliver genuine value regardless of product? [Y/N]

OVERALL SMELL TEST: PASS | CONDITIONAL | FAIL
  PASS: Publishable quality, minimal ad signals
  CONDITIONAL: Publishable with revisions, specific issues identified
  FAIL: Too promotional, significant rewrite needed
```

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| Overall FAIL | Too many ad signals | Comprehensive revision needed |
| Hotspot detected | Concentrated triggers in one section | Target revision at hotspot |
| Redaction test fail | Piece meaningless without product | Add editorial value content |
| Publication fail | No editor would publish | Major editorial revision |
