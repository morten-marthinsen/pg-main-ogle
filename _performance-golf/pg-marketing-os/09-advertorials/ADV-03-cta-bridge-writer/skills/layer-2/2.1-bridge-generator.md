# 2.1-bridge-generator.md

> **Layer:** 2 (Generation)
> **Skill:** ADV-03-cta-bridge-writer
> **Purpose:** Generate bridge + CTA candidates using selected frame and transition plan
> **Outputs:** `bridge_candidates` array with complete bridge/CTA combinations

---

## FUNCTION

Generate 5+ complete bridge/CTA combinations using the selected bridge frame and transition plan. Each combination includes 3-4 bridge paragraphs plus exactly ONE CTA as a text link. Combinations must vary in approach while all maintaining editorial recommendation voice.

---

## GENERATION PROTOCOL

### Step 1: Generate Per Transition Plan

```
FOR each candidate (generate 5-7):
  FOLLOW transition_plan structure:

  Paragraph 1 (Editorial Extension):
    - Continue body's editorial thread
    - Zero sales language
    - Maintain voice register from body
    - Vary: different editorial extensions per candidate

  Paragraph 2 (Soft Pivot):
    - Shift to practical application
    - Keep editorial voice dominant
    - Introduce "for those interested" framing
    - Vary: different pivot angles per candidate

  Paragraph 3 (Recommendation):
    - Apply selected bridge frame
    - Use frame-specific recommendation language
    - Name product for first time (if not named in body)
    - Keep recommendation natural, not pushy
    - Vary: different recommendation phrasings per candidate

  Paragraph 4 (CTA):
    - EXACTLY ONE text link
    - Integrated into natural sentence
    - No button formatting
    - No urgency language
    - Traffic destination URL embedded
    - Vary: different CTA sentences per candidate
```

### Step 2: Apply Frame-Specific Language

```
FOR each candidate:
  IF frame == discovery:
    "After looking into this further, I found [product/site] offers more information about [mechanism]."

  IF frame == recommendation:
    "Based on the evidence reviewed in this article, [product/approach] appears to address [problem] through [mechanism]. You can learn more at [link]."

  IF frame == expert:
    "[Expert name]'s work on [mechanism] is available in more detail at [link], including [specific additional info]."

  IF frame == community:
    "Many who have tried this approach share their experiences at [link], where you can also learn more about [mechanism]."
```

### Step 3: Validate Each Candidate

```
FOR each bridge/CTA candidate:
  VALIDATE:
    [ ] CTA count == 1 (exactly one)
    [ ] CTA format == text_link (not button)
    [ ] Zero "Buy Now" or equivalent language
    [ ] Zero urgency language
    [ ] Voice register consistent with body
    [ ] Transition gradient >= 7
    [ ] Recommendation framing present (not sales framing)
    [ ] Product name used appropriately (not repeated excessively)

  IF any validation fails:
    REVISE specific issue
    RE-VALIDATE
    LOG revision reason
```

### Step 4: Score Candidates

```
FOR each candidate:
  QUICK SCORE (1-10):
    - recommendation_authenticity: Does it feel genuine?
    - transition_smoothness: Is the gradient natural?
    - cta_restraint: Is the CTA subtle and appropriate?
    - voice_consistency: Does it match the body's voice?

  FILTER: Remove any below 6.0 on any criterion
  RANK: Order by average score
  PASS: Top 5+ to Arena
```

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| Multiple CTAs | > 1 link/CTA per candidate | Remove extras, keep strongest |
| Sales language | "Buy now" or equivalent detected | Rewrite in recommendation frame |
| Urgency detected | "Limited time" etc. | Remove urgency completely |
| Button CTA | CTA formatted as button | Convert to inline text link |
| Transition too abrupt | Gradient < 7 | Add intermediary paragraph |
| Voice drift | Bridge voice differs from body | Align to body's register |
