# 0.3-input-validator.md

> **Layer:** 0 (Foundation)
> **Skill:** ADV-03-cta-bridge-writer
> **Purpose:** Validate body draft and traffic destination present
> **Outputs:** `validation_result` with PASS/FAIL

---

## FUNCTION

Validate that all required inputs for bridge/CTA generation are present: body draft with bridge handoff data, traffic destination URL, product name, and strategy bridge constraints.

---

## VALIDATION PROTOCOL

```
REQUIRED (FAIL if missing):
  [ ] body_draft exists and is non-empty
  [ ] body_handoff.reader_state defined
  [ ] body_handoff.mechanism_status defined
  [ ] body_handoff.voice_register defined
  [ ] traffic_destination.url defined and valid
  [ ] traffic_destination.product_name defined
  [ ] strategy.law_4_constraints defined
  [ ] strategy.law_5_constraints defined

QUALITY (WARN if missing):
  [ ] body_handoff.recommended_bridge_frame defined
  [ ] specimens loaded >= 3
  [ ] strategy.adv_03_instructions defined
```

---

## OUTPUT STRUCTURE

```json
{
  "validation_result": {
    "status": "PASS|FAIL",
    "body_draft_valid": true,
    "traffic_destination_valid": true,
    "law_constraints_loaded": true,
    "proceed_to_layer_1": true,
    "warnings": []
  }
}
```

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Body draft missing | HALT -- Execute ADV-02 |
| CRITICAL | Traffic destination missing | Request from human |
| CRITICAL | Product name undefined | Request from campaign brief |
| DEGRADED | Bridge frame not recommended | Default to recommendation |
