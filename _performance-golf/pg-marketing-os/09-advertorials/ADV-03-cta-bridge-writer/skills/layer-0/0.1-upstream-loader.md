# 0.1-upstream-loader.md

> **Layer:** 0 (Foundation)
> **Skill:** ADV-03-cta-bridge-writer
> **Purpose:** Load strategy, body draft, and traffic destination for bridge/CTA generation
> **Outputs:** `upstream_context` object

---

## FUNCTION

Load all upstream outputs required for bridge and CTA generation. The bridge depends on: the strategy (for bridge frame guidance, Law 4/5 constraints), the body draft (for voice register, narrative thread endpoint, mechanism status), and the traffic destination (for CTA linking).

---

## REQUIRED UPSTREAM PACKAGES

| Package | Source | Critical Fields | Required |
|---------|--------|-----------------|----------|
| `advertorial-strategy.yaml` | ADV-00 | five_laws, downstream_instructions.adv_03, bridge_frame | YES |
| `advertorial-body-draft.md` | ADV-02 | bridge_handoff, voice_register, last_section | YES |
| `campaign-brief-package.json` | 01-core-message | traffic_destination, product_name | YES |

---

## LOADING PROTOCOL

```
1. Load strategy -> EXTRACT: Law 4/5 constraints, ADV-03 instructions, bridge frame
2. Load body draft -> EXTRACT: bridge_handoff (reader_state, transition_point, mechanism_status)
3. Load campaign brief -> EXTRACT: traffic_destination URL, product_name
4. VALIDATE: body draft present, traffic destination defined
5. ASSEMBLE upstream_context
```

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Body draft missing | HALT -- Execute ADV-02 |
| CRITICAL | Traffic destination undefined | Request from human |
| DEGRADED | Strategy bridge guidance missing | Default to recommendation frame |
