# 1.1-bridge-frame-selector.md

> **Layer:** 1 (Architecture)
> **Skill:** ADV-03-cta-bridge-writer
> **Purpose:** Select bridge frame (discovery/recommendation/expert/community)
> **Outputs:** `bridge_frame` object with selected frame and application guidance

---

## FUNCTION

Select the bridge frame that creates the most natural transition from editorial body to product recommendation. The frame determines the "voice" of the recommendation -- who is recommending and from what authority.

---

## BRIDGE FRAME OPTIONS

| Frame | Voice | Pattern | Best For |
|-------|-------|---------|----------|
| **Discovery** | "I found..." | Personal discovery narrative | Blog-style, peer voice, personal experience |
| **Recommendation** | "After researching..." | Informed recommendation based on evidence | Native article, journalistic voice |
| **Expert** | "Doctors are now saying..." | Expert/authority endorsement | Editorial feature, expert voice, health niches |
| **Community** | "Thousands have already..." | Social proof and collective action | Listicle, review, community-oriented |

---

## SELECTION PROTOCOL

### Step 1: Score Frames Against Context

```
FOR each bridge_frame:
  SCORE against:
    - voice_register_fit (0-10): Does frame match established voice?
    - type_compatibility (0-10): Does frame work for this advertorial type?
    - mechanism_status_fit (0-10): Does frame work given how mechanism was introduced?
    - reader_state_fit (0-10): Does frame match reader's current mindset?
    - specimen_precedent (0-10): Do specimens support this frame?

  CALCULATE: weighted average
```

### Step 2: Apply Selection Logic

```
IF journalistic voice -> PRIORITIZE recommendation or expert
IF conversational voice -> PRIORITIZE discovery
IF expert voice -> PRIORITIZE expert or recommendation
IF peer voice -> PRIORITIZE discovery or community

IF type == blog_style -> discovery preferred
IF type == native_article -> recommendation preferred
IF type == editorial_feature -> expert preferred
IF type == listicle -> community preferred
```

### Step 3: Define Frame Application

```
FOR selected_frame:
  DOCUMENT:
    - opening_sentence_pattern: How bridge opens
    - recommendation_language: Specific phrases for the frame
    - transition_technique: How to transition from body
    - cta_integration: How CTA fits within the frame
    - forbidden_in_frame: What breaks this frame
```

---

## OUTPUT STRUCTURE

```json
{
  "bridge_frame": {
    "selected": "recommendation",
    "rationale": "Journalistic voice established in body naturally leads to informed recommendation after research",
    "confidence": "HIGH",
    "application": {
      "opening_pattern": "Based on the research reviewed in this article...",
      "recommendation_language": ["After looking into this...", "The evidence suggests..."],
      "transition_technique": "Extend editorial analysis into personal recommendation",
      "cta_integration": "CTA as natural extension of recommendation sentence",
      "forbidden": ["Buy now", "Don't miss", "You need this"]
    },
    "alternatives": [
      {"frame": "expert", "score": 7.2, "gap": 1.3}
    ]
  }
}
```

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| No frame fits | All frames score < 6.0 | Default to recommendation (most versatile) |
| Voice mismatch | Frame doesn't match established voice | Select voice-compatible frame |
| Frames tied | Top 2 score equally | Select frame with more specimen support |
