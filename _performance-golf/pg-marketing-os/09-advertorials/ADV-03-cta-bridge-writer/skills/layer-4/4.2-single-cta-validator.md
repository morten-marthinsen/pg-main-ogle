# 4.2-single-cta-validator.md

> **Layer:** 4 (Validation)
> **Skill:** ADV-03-cta-bridge-writer
> **Purpose:** Verify exactly ONE CTA, text link not button
> **Outputs:** `cta_validation_result` with PASS/FAIL

---

## FUNCTION

Definitive CTA validation: count all call-to-action elements in the bridge, verify exactly one exists, verify it is formatted as a text link (not a button), and verify its placement is appropriate. This enforces Law 4: one CTA only, text link not button.

---

## CTA VALIDATION PROTOCOL

### Step 1: CTA Count

```
SCAN bridge text for CTA elements:
  COUNT:
    - Hyperlinks (markdown links, HTML links)
    - Action phrases directing to URLs
    - "Click here", "Visit", "Go to" + URL patterns
    - Button-style elements
    - Any element that directs reader to take purchase/signup action

  cta_count = total count

  IF cta_count == 0:
    FAIL: "No CTA found -- exactly 1 required"
  IF cta_count == 1:
    PASS (count check)
  IF cta_count > 1:
    FAIL: "Multiple CTAs detected ([count]) -- exactly 1 required"
    LOG each CTA location and text
```

### Step 2: Format Validation

```
FOR the single CTA:
  CHECK format:
    [ ] Is it a text link? (inline markdown/HTML link)
    [ ] NOT a button? (no button HTML, no button styling)
    [ ] NOT all caps? ("CLICK HERE" fails)
    [ ] NOT styled as standalone element? (not centered, not boxed)
    [ ] Integrated into natural sentence? (reads as part of paragraph)

  IF any format check fails:
    FAIL: "CTA format violation: [specific issue]"
    ACTION: Convert to inline text link
```

### Step 3: Placement Validation

```
CHECK CTA placement:
  [ ] In bridge section (not in body or hook/lead)
  [ ] In last 2 paragraphs of bridge (not too early)
  [ ] Not the first element reader encounters in bridge
  [ ] Preceded by recommendation context

  IF placement inappropriate:
    WARN: "CTA placement suboptimal -- [specific issue]"
```

### Step 4: Language Validation

```
CHECK CTA text:
  APPROVED patterns:
    - "learn more", "find out more", "read more"
    - "more information", "full details", "explore"
    - Descriptive anchor text (not "click here")

  REJECTED patterns:
    - "buy now", "order today", "add to cart"
    - "click here" (vague), "CLICK HERE" (aggressive)
    - "sign up", "subscribe", "join" (in purchase context)
    - "get yours", "claim your", "grab"

  IF rejected pattern used:
    FAIL: "CTA language inappropriate: [text]"
```

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| Zero CTAs | No link found | Add single CTA in recommendation context |
| Multiple CTAs | > 1 link detected | Remove extras, keep best-placed |
| Button format | CTA styled as button | Convert to inline text link |
| Sales language | "Buy now" etc. | Replace with "learn more" pattern |
| Premature placement | CTA before recommendation paragraph | Move to end of bridge |
