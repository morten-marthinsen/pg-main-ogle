# 2.1-strategy-assembler.md

> **Layer:** 2 (Assembly)
> **Skill:** ADV-00-advertorial-strategist
> **Purpose:** Assemble full advertorial strategy from all Layer 0 and Layer 1 outputs
> **Outputs:** `assembled_strategy` object -- complete strategy document

---

## FUNCTION

Assemble all strategic components -- type selection, editorial angle, platform compliance, proof inventory, story elements, and audience profile -- into a unified strategy document. Ensure all 5 Laws of Advertorials are explicitly addressed. Generate specific downstream instructions for each ADV skill.

---

## ASSEMBLY PROTOCOL

### Step 1: Gather All Inputs

```
COLLECT from Layer 0:
  - parsed_brief.mechanism
  - parsed_brief.story_elements
  - parsed_brief.proof_inventory
  - parsed_brief.audience_profile

COLLECT from Layer 1:
  - type_selection (type, rationale, constraints)
  - editorial_angle (angle, hook direction, voice)
  - platform_compliance (all platform rules)

VERIFY all inputs present before assembly
IF any input missing: HALT -- return to incomplete layer
```

### Step 2: Assemble Core Strategy

```
BUILD strategy document:

  SECTION 1: Strategic Overview
    - Campaign objective (from brief)
    - Selected type + rationale
    - Editorial angle + hook direction
    - Target platform + compliance summary
    - Word count target (from type constraints)

  SECTION 2: Narrative Architecture
    - Story arc: [protagonist] faces [conflict] -> discovers [mechanism] -> experiences [transformation]
    - Protagonist selection (from story_elements.protagonist_options)
    - Narrative voice (from editorial_angle.voice_register)
    - Pacing guidance (for type-specific structure)

  SECTION 3: Proof Integration Plan
    - Proof elements ranked by embedding priority
    - Placement map: which proof goes where in narrative
    - Density enforcement: max 1 per paragraph, max 2 consecutive
    - Editorial framing for each proof element

  SECTION 4: Platform Constraints
    - Headline constraints (char limits, forbidden patterns)
    - Content constraints (word count, disclosure)
    - Image constraints (if applicable)
    - Claim constraints (what can/cannot be claimed)
```

### Step 3: Map 5 Laws Compliance

```
FOR each of the 5 Laws:

  LAW 1 -- Looks Like Content:
    DOCUMENT how the strategy ensures content appearance:
    - Layout mimics publication format
    - No product imagery in hero position
    - Typography matches editorial style
    - No price mentions in content body

  LAW 2 -- Reads Like Article:
    DOCUMENT how the strategy ensures article feel:
    - Voice register is editorial, not promotional
    - Narrative structure follows journalistic/blog conventions
    - Source attribution for claims
    - No direct-response language in body

  LAW 3 -- Editorial Smell Test:
    DOCUMENT how the strategy passes smell test:
    - Remove product name -> content still reads as article
    - No branded language in first 60% of content
    - Mechanism described without brand attribution initially
    - Value delivered independent of product mention

  LAW 4 -- One CTA, Text Link:
    DOCUMENT CTA constraints:
    - Exactly ONE call-to-action in entire piece
    - CTA is TEXT LINK, not button
    - CTA appears in bridge section only
    - CTA language is recommendation, not command

  LAW 5 -- Bridge = Recommendation:
    DOCUMENT bridge framing:
    - Bridge frames product as discovery/recommendation
    - No sales close language
    - Frame options: discovery/recommendation/expert/community
    - Transition from editorial to offer is gradual, not abrupt
```

### Step 4: Generate Downstream Instructions

```
FOR each downstream skill:

  ADV-01 (Hook & Lead):
    - Hook must align with: [angle_type, hook_direction]
    - Voice register: [voice_register]
    - Headline char limit: [from platform]
    - Lead must establish: [editorial context]
    - Forbidden in hook: [product name, price, CTA language]

  ADV-02 (Body Copy):
    - Structure follows: [type-specific template]
    - Proof placement: [from proof integration plan]
    - Proof density: max 1 per paragraph
    - Mechanism introduction point: [where in narrative]
    - Word count target: [from type constraints]

  ADV-03 (CTA & Bridge):
    - Bridge frame: [selected frame]
    - CTA format: text link only, one CTA
    - Transition approach: [gradual, recommendation-framed]
    - Traffic destination: [from brief]
    - Forbidden: sales close language, multiple CTAs, buttons

  ADV-04 (Assembly):
    - Section order: [from type template]
    - Voice consistency check: [voice register]
    - Disclosure placement: [from platform compliance]
    - Word count bounds: [min, max]

  ADV-05 (Editorial):
    - Smell test focus: [specific areas to check]
    - Compliance checklist: [from platform rules]
    - Quality criteria: [type-specific]
    - Word count validation: [from type constraints]
```

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| Incomplete inputs | Layer 0 or 1 output missing | Return to incomplete layer |
| 5 Laws gap | One or more laws not addressable with current strategy | Revise type/angle to satisfy all laws |
| Downstream instructions vague | Instructions lack specificity for execution | Expand with type-specific detail |
| Proof plan violates density | Proof stacking detected in plan | Redistribute proof elements |
