# 0.3-input-validator.md

> **Layer:** 0 (Foundation)
> **Skill:** ADV-00-advertorial-strategist
> **Purpose:** Validate minimum fields present for advertorial strategy creation
> **Outputs:** `validation_result` object with PASS/FAIL status

---

## FUNCTION

Validate that all required inputs have been loaded and parsed successfully. Check for minimum field completeness across mechanism, story elements, proof inventory, and audience profile. Gate Layer 1 execution behind this validation.

---

## VALIDATION PROTOCOL

### Step 1: Critical Field Validation

```
REQUIRED FIELDS (FAIL if missing):
  [ ] mechanism.name -- non-empty string
  [ ] mechanism.editorial_frame -- non-empty string
  [ ] story_elements.conflict -- non-empty string
  [ ] story_elements.discovery_moment -- non-empty string
  [ ] audience_profile.awareness_level -- valid enum value
  [ ] campaign_brief loaded -- boolean true

IF any REQUIRED field missing:
  STATUS = FAIL
  LOG: "Missing required field: [field_name]"
  HALT: Cannot proceed to Layer 1
```

### Step 2: Quality Field Validation

```
QUALITY FIELDS (WARN if missing, proceed with flags):
  [ ] proof_inventory.elements -- at least 1 element
  [ ] story_elements.protagonist_options -- at least 1 option
  [ ] story_elements.transformation -- non-empty string
  [ ] audience_profile.pain_language -- at least 1 phrase
  [ ] audience_profile.desire_language -- at least 1 phrase
  [ ] mechanism.forbidden_frames -- at least 1 frame

IF any QUALITY field missing:
  STATUS = PASS with WARNINGS
  LOG: "Warning: [field_name] missing -- strategy quality may be reduced"
  FLAG: degraded_inputs = true
```

### Step 3: Consistency Validation

```
CONSISTENCY CHECKS:
  [ ] mechanism.name matches across all references
  [ ] audience awareness_level consistent with recommended voice_match
  [ ] proof elements reference actual mechanism (not unrelated claims)
  [ ] story conflict relates to audience pain_language

IF inconsistency found:
  STATUS = PASS with WARNINGS
  LOG: "Inconsistency: [description]"
  FLAG: consistency_issues = true
```

```json
      "missing": [],

    },
    "consistency": {
      "all_consistent": true,
      "issues": []
    },
    "gate_0_status": "PASS",

  }
}
```

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| CRITICAL | Any required field missing | HALT -- Return to 0.1 or 0.2, identify gap |
| WARNING | Quality fields missing | Proceed with degraded flag, note in strategy |
| WARNING | Consistency issues found | Proceed with flag, note for human review |
