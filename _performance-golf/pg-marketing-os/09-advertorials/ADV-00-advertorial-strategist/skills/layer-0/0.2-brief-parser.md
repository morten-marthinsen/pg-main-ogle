# 0.2-brief-parser.md

> **Layer:** 0 (Foundation)
> **Skill:** ADV-00-advertorial-strategist
> **Purpose:** Extract mechanism, story elements, proof inventory, and audience from campaign brief
> **Outputs:** `parsed_brief` object with advertorial-specific extractions

---

## FUNCTION

Transform raw campaign brief data into advertorial-specific components. Extract and organize the mechanism for editorial narrative, catalogue story elements for narrative threading, build proof inventory with density metadata, and profile the audience for voice calibration.

---

## PARSING PROTOCOL

### Step 1: Mechanism Extraction

Extract the mechanism in editorial-friendly form:

```
FROM campaign_brief.mechanism:
  EXTRACT:
    - mechanism_name: The named mechanism (exact)
    - mechanism_description: What it does in plain language
    - editorial_frame: How to describe mechanism as discovery/finding/approach
    - forbidden_frames: ["product feature", "proprietary formula", "patented blend"]

  REFRAME for editorial voice:
    PRODUCT LANGUAGE -> EDITORIAL LANGUAGE
    "Our patented formula" -> "A compound researchers identified"
    "Clinically proven ingredient" -> "An ingredient studied at [university]"
    "Revolutionary breakthrough" -> "A finding published in [journal]"
```

### Step 2: Story Element Extraction

Extract narrative components for advertorial storytelling:

```
FROM campaign_brief (big_idea, mechanism, avatar):
  EXTRACT:
    - protagonist_options:
        option_1: "The researcher/doctor who discovered [mechanism]"
        option_2: "A person like the reader who struggled with [pain]"
        option_3: "The reader themselves (second person)"
    - conflict: The problem/pain that creates narrative tension
    - discovery_moment: When/how the mechanism was found
    - transformation: What changed after applying the mechanism
    - stakes: What happens if the reader does nothing
    - narrative_arc: conflict -> discovery -> proof -> transformation
```

### Step 3: Proof Inventory Cataloguing

Build complete proof inventory with embedding metadata:

```
FROM campaign_brief.proof_inventory:
  FOR each proof element:
    CATALOGUE:
      - type: clinical_study | testimonial | expert_quote | statistic | case_study | mechanism_evidence
      - content: The actual proof content
      - strength: high (published study, named expert) | medium (general research) | low (anecdotal)
      - editorial_format: How to present in editorial voice
      - embedding_priority: 1 (must use) through 5 (optional)
      - paragraph_placement: standalone | woven_into_narrative | supporting_detail

  BUILD density rules:
    - max_per_paragraph: 1
    - max_consecutive_proof_paragraphs: 2
    - stacking_threshold: 3 (auto-fail)
    - preferred_rhythm: narrative -> proof -> narrative -> proof
```

### Step 4: Audience Profiling

Profile audience for voice calibration:

```
FROM campaign_brief.avatar:
  EXTRACT:
    - awareness_level: unaware | problem_aware | solution_aware | product_aware
    - sophistication: low (tabloid reader) | medium (magazine reader) | high (journal reader)
    - reading_context: social_scroll | search_result | email_click | native_feed
    - voice_match:
        IF awareness = unaware: curiosity-driven, pattern-interrupt
        IF awareness = problem_aware: empathy-first, story-driven
        IF awareness = solution_aware: discovery-framed, mechanism-forward
        IF awareness = product_aware: review/comparison, proof-forward
    - pain_language: Exact words the audience uses for their problem
    - desire_language: Exact words the audience uses for their desired state
```

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| No mechanism found | Big Idea present but mechanism absent | Extract implicit mechanism from Big Idea |
| Zero proof elements | Proof inventory empty | Flag narrative-only strategy, proceed |
| Audience undefined | Avatar section missing | Use defaults based on niche, flag risk |
| Story elements thin | Brief lacks narrative components | Construct from mechanism + avatar, flag |
