# 1.1-advertorial-type-selector.md

> **Layer:** 1 (Architecture)
> **Skill:** ADV-00-advertorial-strategist
> **Purpose:** Select optimal advertorial type based on campaign brief analysis
> **Outputs:** `type_selection` object with selected type and rationale

---

## FUNCTION

Analyze the parsed campaign brief to select the optimal advertorial type. Match campaign objectives, audience awareness level, proof inventory strength, and platform requirements to the best-fitting advertorial format. Provide documented rationale for selection.

---

## ADVERTORIAL TYPE TAXONOMY

| Type | Best For | Structure | Word Range | Proof Style |
|------|----------|-----------|------------|-------------|
| **Listicle** | Problem-aware audiences, multiple benefits | N reasons/ways/tips + mechanism weave | 800-1200 | Distributed across items |
| **Native Article** | Unaware audiences, news-style platforms | Journalistic structure, inverted pyramid | 1000-1500 | Embedded in reporting |
| **Blog-Style** | Solution-aware audiences, casual platforms | Personal voice, first-person experience | 800-1200 | Testimonial-driven |
| **Product Review** | Product-aware audiences, comparison shoppers | Review format with pros/cons/verdict | 1000-1400 | Feature-by-feature evidence |
| **PAS Format** | Problem-aware, high-urgency niches | Problem -> Agitate -> Solve structure | 800-1200 | Concentrated in Solution section |
| **Sponsored Content** | Brand-aware platforms, premium publishers | Magazine-quality editorial | 1200-1800 | Woven into narrative |
| **Editorial Feature** | Unaware audiences, authority positioning | Expert interview or investigation format | 1200-2000 | Expert citation heavy |

---

## SELECTION PROTOCOL

### Step 1: Score Each Type Against Campaign Factors

```
FOR each advertorial_type:
  SCORE against:
    - audience_awareness_fit (0-10):
        How well does this type match the audience awareness level?
    - proof_inventory_fit (0-10):
        Does the proof inventory support this type's proof style?
    - mechanism_presentation_fit (0-10):
        Can the mechanism be presented naturally in this format?
    - platform_compatibility (0-10):
        Does this type work on the target platform?
    - narrative_strength_fit (0-10):
        Do the story elements support this type's narrative needs?

  CALCULATE weighted_score:
    awareness_fit * 0.25 +
    proof_fit * 0.20 +
    mechanism_fit * 0.20 +
    platform_fit * 0.20 +
    narrative_fit * 0.15
```

### Step 2: Apply Selection Rules

```
IF top_score > 7.5 AND gap_to_second > 1.0:
  SELECT top scorer -- clear winner
  confidence = HIGH

IF top_score > 7.0 AND gap_to_second < 1.0:
  PRESENT top 2 options to human
  confidence = MODERATE

IF top_score < 7.0:
  PRESENT top 3 options with analysis
  confidence = LOW
  REQUEST human direction
```

### Step 3: Document Type Constraints

```
FOR selected_type:
  DOCUMENT:
    - word_count_range: [min, max]
    - required_sections: [list]
    - proof_density_max: "1 per paragraph"
    - cta_placement: [where CTA goes]
    - voice_register: [appropriate voice for this type]
    - structural_template: [section order]
    - 5_laws_alignment: [how this type naturally satisfies each law]
```

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| All types score < 7.0 | Campaign brief misaligned with advertorial format | Present analysis to human, request direction |
| Multiple types tied | Scoring factors balanced across types | Present top options with trade-off analysis |
| Platform incompatible | Selected type doesn't work on target platform | Constrain to platform-compatible types, re-score |
| Proof too thin for type | Selected type needs more proof than available | Downgrade to narrative-heavy type or flag risk |
