# 1.3-platform-compliance-mapper.md

> **Layer:** 1 (Architecture)
> **Skill:** ADV-00-advertorial-strategist
> **Purpose:** Map platform-specific compliance requirements for the advertorial
> **Outputs:** `platform_compliance` object with rules and constraints

---

## FUNCTION

Map the specific compliance requirements for the target advertising platform. Each platform (Taboola, Outbrain, Meta, organic) has different rules for headlines, images, content length, disclosures, and prohibited claims. These constraints shape every downstream decision.

---

## PLATFORM COMPLIANCE DATABASE

### Taboola

```yaml
taboola:
  headlines:
    max_chars: 60
    forbidden_patterns:
      - ALL CAPS words (except acronyms)
      - Excessive punctuation (!! or ??)
      - Misleading health claims
      - Before/after implications without disclaimer
    required: thumbnail image
  content:
    word_count_range: [500, 2000]
    disclosure: "Sponsored" or "Paid Content" label required
    disclosure_placement: top of article
    landing_page: must match headline promise
  images:
    forbidden:
      - Before/after photos without disclaimer
      - Misleading medical imagery
      - Clickbait shock imagery
    recommended: editorial-style photography
  claims:
    health_claims: must be substantiated
    income_claims: forbidden or heavily restricted
    celebrity_endorsement: must be genuine and disclosed
```

### Outbrain

```yaml
outbrain:
  headlines:
    max_chars: 65
    forbidden_patterns:
      - Clickbait triggers ("You won't believe")
      - Excessive capitalization
      - False urgency ("Act now")
    required: high-quality thumbnail
  content:
    word_count_range: [500, 2500]
    disclosure: clear sponsorship disclosure
    disclosure_placement: top and bottom
    quality_score: content must pass editorial quality review
  images:
    min_resolution: 400x300
    forbidden:
      - Misleading imagery
      - Graphic content
      - Text-heavy overlays
    recommended: authentic, non-stock photography
  claims:
    substantiation: all claims must be supportable
    testimonials: must be genuine with disclosure
```

### Meta (Facebook/Instagram)

```yaml
meta:
  headlines:
    max_chars: 40 (primary), 25 (headline field)
    forbidden_patterns:
      - Personal attributes ("Are you overweight?")
      - Before/after claims
      - Sensationalized language
    required: compliant ad creative
  content:
    landing_page: must comply with Meta advertising policies
    text_in_image: 20% maximum text overlay
    disclosure: "Sponsored" label automatic
  images:
    text_overlay_max: 20%
    forbidden:
      - Before/after imagery
      - Misleading transformations
      - Personal health condition imagery
    recommended: lifestyle imagery, no text overlay
  claims:
    health_claims: heavily restricted
    income_claims: forbidden
    personal_attributes: cannot assert or imply
```

### Organic (SEO/Social)

```yaml
organic:
  headlines:
    max_chars: 60 (SEO title), no limit (social)
    seo_requirements:
      - Primary keyword in first 60 chars
      - Meta description 150-160 chars
    social_requirements:
      - Platform-native formatting
      - Hashtag strategy if applicable
  content:
    word_count_range: [1000, 3000]
    disclosure: none required for organic
    seo: heading hierarchy, keyword density, internal linking
  images:
    seo: alt text, proper sizing, compression
    social: platform-optimal dimensions
  claims:
    FTC_compliance: still applies for endorsements
    testimonials: FTC disclosure required
```

## MAPPING PROTOCOL

### Step 1: Identify Target Platform

```
FROM campaign_brief.creative_direction OR strategy input:
  IDENTIFY primary_platform:
    IF specified: Use specified platform
    IF not specified: Default to taboola (most restrictive)
    IF multi-platform: Map ALL platforms, use most restrictive as primary

  LOG platform selection and rationale
```

### Step 2: Extract Applicable Rules

```
FOR primary_platform:
  EXTRACT all rules from compliance database above
  MAP rules to advertorial components:
    - headline_rules -> ADV-01 constraints
    - content_rules -> ADV-02, ADV-03 constraints
    - image_rules -> visual strategy constraints
    - claim_rules -> proof embedding constraints
```

### Step 3: Document Platform-Specific Constraints

```
FOR each downstream skill:
  DOCUMENT specific constraints:
    ADV-01 (Hook/Lead): headline char limits, forbidden patterns
    ADV-02 (Body): word count limits, claim substantiation rules
    ADV-03 (CTA/Bridge): disclosure requirements, CTA formatting rules
    ADV-04 (Assembly): disclosure placement, overall compliance check
    ADV-05 (Editorial): final compliance validation points
```

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| Platform unknown | No platform specified in brief | Default to Taboola (most restrictive) |
| Multi-platform conflict | Rules conflict across platforms | Use most restrictive rule set |
| Claim restriction blocks strategy | Platform forbids core claim type | Reframe claims to comply, flag to human |
| Word count mismatch | Type requires more words than platform allows | Adjust type or constrain scope |
