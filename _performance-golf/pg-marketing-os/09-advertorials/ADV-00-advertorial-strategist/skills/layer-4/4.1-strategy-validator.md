# 4.1-strategy-validator.md

> **Layer:** 4 (Validation)
> **Skill:** ADV-00-advertorial-strategist
> **Purpose:** Validate advertorial strategy completeness against required checklist
> **Outputs:** `validation_result` object with PASS/FAIL and gap report

---

## FUNCTION

Run the assembled strategy through a comprehensive validation checklist. Verify all required sections are populated, all 5 Laws are addressed, downstream instructions are specific enough for execution, and platform compliance is complete. Gate packaging behind this validation.

---

## VALIDATION PROTOCOL

### Step 1: Section Completeness Check

```
REQUIRED SECTIONS (FAIL if missing or empty):
  [ ] strategic_overview.campaign_objective -- non-empty
  [ ] strategic_overview.selected_type -- valid type enum
  [ ] strategic_overview.type_rationale -- non-empty, > 50 chars
  [ ] strategic_overview.editorial_angle -- non-empty
  [ ] strategic_overview.target_platform -- valid platform enum
  [ ] narrative_architecture.story_arc -- non-empty
  [ ] narrative_architecture.voice -- valid voice register
  [ ] proof_integration.density_rules -- populated
  [ ] platform_constraints -- all subsections populated
  [ ] five_laws_compliance -- all 5 laws addressed
  [ ] downstream_instructions -- all 5 skills addressed
```

### Step 2: 5 Laws Validation

```
FOR each of the 5 Laws:
  VERIFY:
    - Law is explicitly addressed (not just mentioned)
    - Strategy element cited is actionable
    - Downstream instruction references the law
    - No contradiction between law and strategy elements

  IF any law NOT addressed:
    STATUS = FAIL
    GAP: "Law [N] not addressed: [description]"
```

### Step 3: Downstream Instruction Quality

```
FOR each downstream skill instruction:
  VERIFY:
    - Instructions are specific to selected type (not generic)
    - Voice register specified
    - Constraints listed with specific values
    - Forbidden elements listed
    - Word count or structural guidance provided

  IF any instruction is generic/vague:
    STATUS = FAIL
    GAP: "ADV-[N] instructions too vague: [specific issue]"
```

### Step 4: Proof Density Validation

```
CHECK proof integration plan:
  [ ] max_per_paragraph <= 1
  [ ] No planned sequence of 3+ proof paragraphs
  [ ] Proof rhythm follows narrative-proof-narrative pattern
  [ ] Each proof element has editorial framing

  IF density rules violated:
    STATUS = FAIL
    GAP: "Proof density violation: [specific issue]"
```

### Step 5: Platform Compliance Validation

```
CHECK platform constraints:
  [ ] Headline char limit specified
  [ ] Word count range specified
  [ ] Disclosure requirement documented
  [ ] Claim restrictions documented
  [ ] Image rules documented (if applicable)

  IF any platform rule missing:
    STATUS = FAIL
    GAP: "Platform compliance gap: [specific missing rule]"
```

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| Section missing | Required section empty | Return to Layer 2 assembly |
| 5 Laws gap | One or more laws not addressed | Return to 2.1 with specific law gaps |
| Vague instructions | Downstream instructions generic | Expand instructions in 2.1 |
| Proof density violation | Stacking planned | Revise proof integration plan |
| Platform gaps | Compliance rules incomplete | Return to 1.3 for completion |
