# 4.2-output-packager.md

> **Layer:** 4 (Validation)
> **Skill:** ADV-00-advertorial-strategist
> **Purpose:** Package validated strategy as advertorial-strategy.yaml for downstream consumption
> **Outputs:** `advertorial-strategy.yaml` file

---

## FUNCTION

Package the validated, assembled strategy into the canonical output file (`advertorial-strategy.yaml`) that all downstream ADV skills consume. This file is the single source of truth for the advertorial project. It must be complete, schema-valid, and self-contained.

---

## PACKAGING PROTOCOL

### Step 1: Verify Validation Passed

```
CHECK validation_result.status:
  IF status == "PASS":
    PROCEED to packaging
  IF status == "FAIL":
    HALT -- cannot package failed strategy
    LOG: "Packaging blocked -- validation failed with [N] gaps"
```

### Step 2: Assemble YAML Structure

```
BUILD advertorial-strategy.yaml:

  metadata:
    version: "1.0"
    generated_at: "[ISO timestamp]"
    skill_id: "ADV-00-advertorial-strategist"
    project_id: "[from campaign brief]"
    validation_status: "PASS"

  type:
    selected: "[type]"
    rationale: "[rationale]"
    constraints: [type-specific constraints]

  angle:
    type: "[angle type]"
    statement: "[angle statement]"
    hook_direction: "[hook direction]"
    voice_register: "[voice]"

  compliance:
    platform: "[platform]"
    rules: [platform-specific rules]

  proof:
    inventory: [catalogued elements]
    density_rules: [rules]
    placement_map: [map]

  story:
    arc: "[narrative arc]"
    protagonist: "[selected protagonist]"
    elements: [story elements]

  five_laws: [compliance mapping]

  downstream: [per-skill instructions]
```

### Step 3: Schema Validation

```
VALIDATE output file:
  [ ] YAML is valid (parseable)
  [ ] All required top-level keys present
  [ ] No empty values where required
  [ ] No placeholder text ("[TODO]", "[INSERT]")
  [ ] Timestamps are ISO format
  [ ] Enum values are valid

IF validation fails:
  DO NOT write file
  FIX specific issues
  RE-VALIDATE
```

### Step 4: Write File

```
WRITE advertorial-strategy.yaml to:
  [project]/09-advertorials/outputs/advertorial-strategy.yaml

VERIFY:
  [ ] File exists at expected path
  [ ] File is non-empty
  [ ] File is valid YAML
  [ ] File size > 2KB (minimum for complete strategy)

LOG: "advertorial-strategy.yaml written successfully at [timestamp]"
```

---

## OUTPUT STRUCTURE

The output IS the `advertorial-strategy.yaml` file itself.

Additionally, create a verification object:

```json
{
  "packaging_result": {
    "status": "complete",
    "file_written": "advertorial-strategy.yaml",
    "file_path": "[project]/09-advertorials/outputs/advertorial-strategy.yaml",
    "schema_version": "1.0",
    "validation_passed": true,
    "file_size_kb": 4.2,
    "written_at": "ISO timestamp"
  }
}
```

---

## FAILURE MODES

| Mode | Trigger | Remediation |
|------|---------|-------------|
| Validation not passed | Packaging attempted before validation | HALT -- run 4.1 first |
| YAML invalid | Malformed YAML in output | Fix syntax, re-validate |
| Missing fields | Required fields absent in assembled strategy | Return to Layer 2 |
| File write failure | Filesystem error | Check path permissions, retry |
| File too small | Output < 2KB indicates incomplete | Return to assembly |
