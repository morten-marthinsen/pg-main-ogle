## Validation Checks

### Check 1: Platform Coverage

For each platform identified in 1.1-A:
- Is there at least ONE Tier 1 or Tier 2 source?
- If not → FLAG as COVERAGE_GAP

```
Expected platforms: [YouTube, Reddit, GolfWRX, Instagram, Amazon, FB Ads Library]
Covered platforms: [YouTube ✓, Reddit ✓, GolfWRX ✓, Instagram ⚠, Amazon ✓, FB Ads ✓]
Gap: Instagram has only Tier 3 sources
```

### Check 2: Platform Type Coverage

Ensure coverage across source TYPES:
- [ ] Video platform (at least 1 Tier 1/2)
- [ ] Forum/community (at least 1 Tier 1/2)
- [ ] Social platform (at least 1 Tier 1/2)
- [ ] Review source (at least 1 Tier 1/2)
- [ ] Competitive intel (at least 1 source)

If any type is missing → FLAG as TYPE_GAP

### Check 3: Tier Distribution

Healthy distribution:
- Tier 1: 15-25% of sources
- Tier 2: 30-40% of sources
- Tier 3: 20-30% of sources
- Exclude: Remaining

Concerning patterns:
- All Tier 1 → Possible inflation, too narrow
- No Tier 1 → Category may lack good sources (flag for human)
- Too many Exclude → Queries may have been off-target

### Check 4: Score Anomalies

Flag sources where:
- Very high composite (9+) → Verify not inflated
- Very low composite (<3) on expected platform → Verify accurate
- Large variance between dimensions → May indicate data issue

### Check 5: Missing Expected Sources

For common categories, check if obvious sources are present:

**Golf:**
- YouTube: Rick Shiels, TXG, MyGolfSpy (expected)
- Reddit: r/golf (expected)
- Forum: GolfWRX (expected)

If obviously expected source is missing → FLAG as MISSING_EXPECTED

### Check 6: Budget Estimation

Based on approved sources and tiers:
- Tier 1 sources: Estimate full depth scraping cost
- Tier 2 sources: Estimate standard scraping cost
- Tier 3 sources: Estimate light scraping cost

Total estimated cost vs. budget:
- Under budget → Proceed
- Near budget (80-100%) → Warn
- Over budget → FLAG, recommend tier reduction

---

## Validation Process

### Step 1: Run All Checks
Execute each validation check, recording results

### Step 2: Compile Flags
Create list of all flags with severity:
- **HIGH:** Must be addressed before proceeding
- **MEDIUM:** Should be reviewed by human
- **LOW:** Note for awareness

### Step 3: Generate Coverage Report
Create summary of what will be scraped

### Step 4: Prepare Human Review Items
For borderline cases, prepare specific questions for human reviewer

### Step 5: Determine Validation Result
- **APPROVED:** All checks pass, no high-severity flags
- **APPROVED_WITH_FLAGS:** Minor issues, can proceed with awareness
- **REQUIRES_REVISION:** High-severity flags need addressing

## Checkpoint 1 Protocol

After this skill completes:

1. **Present to human:**
   - Validation result
   - Coverage report summary
   - All flags (sorted by severity)
   - Human review items as questions

2. **Human can:**
   - Approve as-is
   - Add sources manually
   - Remove sources
   - Request re-scan of specific platforms
   - Adjust tier assignments

3. **Proceed only after:**
   - Human explicitly approves
   - OR human makes modifications and approves

4. **Do NOT proceed if:**
   - Human requests revision
   - High-severity flags unaddressed
   - Human hasn't responded

---

## Quality Criteria

- [ ] All validation checks executed
- [ ] Flags include severity and recommendations
- [ ] Coverage report is complete
- [ ] Human review items are specific, actionable questions
- [ ] Estimated cost calculated

---

## Failure Modes

- **Rubber stamping:** Approving without actual validation
- **Over-flagging:** Too many minor flags obscures important issues
- **Under-flagging:** Missing obvious coverage gaps
- **Vague questions:** Human review items too general to be useful

---

## Notes

- This skill is a CRITIC, not a cheerleader
- Its job is to find problems before money is spent
- Human approval is mandatory - no automatic proceeding
- After approval, scraping begins (1.4 skills)
