## MANDATORY: Confidence Calculation (PRD Rule 5)

**Confidence is CALCULATED, not estimated.** The formula prevents subjective assessment.

### Confidence Calculation Formula

```yaml
confidence_formula:
  # BLOCKERS (if ANY true, confidence = 0)
  blockers:
    - total_quotes < 500  # BLOCKED - insufficient data
    - any_bucket < minimum  # BLOCKED - incomplete coverage
    - pair_extraction_failed  # BLOCKED - pairs not extracted

  # If no blockers, calculate base confidence
  base_calculation:
    volume_score:
      formula: min((total_quotes / 1500) * 30, 30)  # Max 30 points
      threshold: 1000 quotes = 20 points, 1500+ = 30 points

    coverage_score:
      formula: (buckets_above_minimum / 6) * 25  # Max 25 points
      notes: 6 buckets total (PAIN, HOPE, RC, ST, CM, VILLAIN)

    quality_score:
      formula: (gold_quotes * 3 + high_quotes * 1) / total_quotes * 25  # Max 25 points
      notes: Higher priority quotes increase quality score

    source_balance_score:
      formula: 20 - (forum_percentage_over_50 * 0.4)  # Max 20 points
      notes: Penalty if >50% from forums (per forum awareness protocol)

  final_confidence:
    formula: volume_score + coverage_score + quality_score + source_balance_score
    max: 100
    interpretation:
      90-100: "EXCEPTIONAL - research extremely robust"
      80-89: "HIGH - research very reliable"
      70-79: "SOLID - research dependable"
      60-69: "ADEQUATE - research usable with caveats"
      50-59: "MARGINAL - proceed with documented limitations"
      <50: "INSUFFICIENT - consider expansion before Layer 2"
```

## Review Criteria

### 1. Saturation Check

**What is saturation?**
When scraping continues to return the same patterns, language, and sentiments we've already captured.

**Saturation Indicators:**
- Last 20% of quotes repeat themes from first 80%
- Same specific phrases appearing 3+ times
- Same pain points/desires being expressed
- No new audience segments emerging

**Saturation Assessment:**
```
IF new_pattern_rate < 10% in last batch:
  saturation = ACHIEVED
ELSE:
  saturation = NOT_ACHIEVED
  Flag: "Consider additional scraping if budget allows"
```

### 2. Silent Failure Check

**What is a silent failure?**
When a scraper reports success but actually captured minimal or low-quality data.

**Silent Failure Indicators:**
- Platform in approved_sources but 0 quotes from that platform
- Scraper reported "complete" but quote count is <10
- Large gap between expected sources and actual quotes

**Silent Failure Detection:**
```
FOR EACH platform in approved_sources:
  expected_sources = count(approved_sources[platform])
  actual_quotes = count(tagged_quotes WHERE platform)

  IF actual_quotes < (expected_sources × 5):
    FLAG: "Silent failure suspected for {platform}"
    DETAIL: "Expected ~{expected × 10-50} quotes, got {actual}"
```

### 3. Platform Coverage Check

**What is adequate coverage?**
Sufficient data from diverse platforms to ensure findings aren't platform-biased.

**Coverage Requirements:**
- At least 3 platforms with quotes
- No single platform >60% of total quotes
- At least one community platform (Reddit/forum)
- At least one review platform (Amazon/reviews)

**Coverage Assessment:**
```
platforms_with_quotes = unique platforms in tagged_quotes
coverage_distribution = quotes per platform / total quotes

IF platforms_with_quotes < 3:
  FLAG: "Insufficient platform diversity"

IF max(coverage_distribution) > 0.6:
  FLAG: "Platform bias detected - {platform} has {%}%"
```

### 4. Quote Quality Check

**What is sufficient quality?**
Enough high-SQS quotes to support meaningful Layer 2 analysis.

**Quality Thresholds:**
- Minimum 100 total quotes (for standard research)
- Minimum 30% with SQS ≥ 6 (medium-high quality)
- Minimum 10% with SQS ≥ 8 (high quality)
- Average SQS ≥ 5.0

**Quality Assessment:**
```
total_quotes = count(tagged_quotes)
high_quality = count(tagged_quotes WHERE sqs >= 6)
exceptional = count(tagged_quotes WHERE sqs >= 8)
average_sqs = mean(tagged_quotes.sqs)

IF total_quotes < 100:
  FLAG: "Insufficient quote volume"
  SEVERITY: CRITICAL if <50, WARNING if 50-100

IF high_quality / total_quotes < 0.30:
  FLAG: "Insufficient quote quality"

IF average_sqs < 5.0:
  FLAG: "Low average quality score"
```

### 5. Sentiment Distribution Check

**What is healthy sentiment distribution?**
A mix of positive, negative, and mixed sentiment indicating balanced data.

**Distribution Requirements:**
- Negative sentiment: 20-60%
- Positive sentiment: 15-50%
- No single sentiment >70%

**Distribution Assessment:**
```
sentiment_dist = tagged_quotes grouped by sentiment

IF any sentiment > 70%:
  FLAG: "Sentiment imbalance - potential source bias"
  NOTE: "{sentiment} at {%}%"
```

---

## Pass/Fail Criteria

### PASS Conditions (All must be true)
- Total quotes ≥ 100
- High-quality quotes (SQS ≥ 6) ≥ 30%
- Platform diversity ≥ 3
- No CRITICAL flags

### WARNING Conditions (Pass with notes)
- Total quotes 50-100
- Any non-critical flags raised
- Saturation not achieved
- Mild coverage imbalance

### FAIL Conditions (Any triggers failure)
- Total quotes < 50
- No high-quality quotes (SQS ≥ 6) found
- Silent failure detected on >50% of platforms
- Platform diversity < 2
- All quotes from single sentiment (>90%)

## Failure Protocol

```
IF status == "failed":
  1. HALT pipeline
  2. Generate detailed failure report
  3. Identify specific failures
  4. Recommend remediation:
     - If quote volume low: Expand sources or scraping depth
     - If quality low: Review source selection criteria
     - If coverage imbalanced: Target underrepresented platforms
     - If silent failure: Debug specific scraper
  5. DO NOT proceed to Layer 2
  6. Present to human for decision
```

## Edge Cases

### Minimum Viable Dataset
If budget constraints force smaller sample:
- Absolute minimum: 50 quotes
- Must have: At least 15 high-SQS quotes
- Lower flag to WARNING, allow proceed with documented risk

### Platform Unavailability
If approved platform couldn't be scraped:
- Document in issues
- Adjust expectations for analysis
- Proceed if remaining platforms sufficient

### High Saturation, Low Volume
If same patterns appear quickly but total quotes low:
- Saturation may indicate niche market
- Document as finding, not failure
- Proceed with caution
