**Layer:** 1 - Infrastructure
**Purpose:** Scrape Facebook Ads Library for competitor advertising intelligence
**Dependency:** 1.3-A Source Validator (Checkpoint 1 passed)

## CRITICAL: Use Tool Chains, Not Direct Tool Calls

**OLD (Single Point of Failure):**
```python
# DO NOT DO THIS
result = mcp__firecrawl__firecrawl_scrape(url=ads_library_url)
```

**NEW (With Fallback Chain):**
```python
# REQUIRED APPROACH
result = execute_chain(
    chain_id="ADS_LIBRARY_SCRAPING",
    params={"advertiser": advertiser_name, "category": product_category}
)
```

All scraping MUST go through the tool chain system defined in [[0.2-tool-resilience]].

## Scraping Protocol

### Step 1: Advertiser Discovery
For known competitors:
1. Search Ads Library by page name
2. Confirm correct page found
3. Add to scrape queue

For discovery:
1. Search by category keywords
2. Filter for active advertisers
3. Add relevant advertisers

### Step 2: Ad Collection
For each advertiser:
1. Navigate to all ads
2. Filter for active ads
3. Capture all ad creatives
4. Extract ad copy

### Step 3: Landing Page Extraction
For each ad:
1. Extract destination URL
2. Clean tracking parameters (optional, keep raw)
3. Compile landing page list for 1.4-G

### Step 4: Duration Analysis
For active ads:
1. Note start date
2. Calculate run duration
3. Flag long-running ads (>30 days = likely winners)

## Tool Configuration

### Tool Chain: ADS_LIBRARY_SCRAPING (from 0.2-tool-resilience.md)

```yaml
chain_id: ADS_LIBRARY_SCRAPING
fallback_sequence:
  1. Firecrawl scrape (primary - good for dynamic content)
  2. Apify facebook-ads-library-scraper (fallback)
  3. Manual intervention

notes:
  - Ads Library is publicly accessible but can be flaky
  - Dynamic content loading requires wait times
  - May need to handle pagination
```

### Error Handling (Via Tool Chain)

```
ON tool_chain_success:
  - Continue processing
  - Log which tool succeeded (for monitoring)
  - Compile landing_page_urls for 1.4-G

ON tool_chain_fallback_used:
  - Log: primary tool failed, fallback succeeded
  - Continue processing
  - No human intervention needed

ON tool_chain_complete_failure:
  - Ads Library is important but not critical
  - Log full failure report with all attempted tools
  - Add advertiser to manual_review_queue.json
  - Continue with other advertisers
  - If >50% advertisers fail: HALT and report to human

ON dynamic_content_timeout:
  - Increase wait time and retry
  - If persistent: Try different tool in chain
```

### Raw Data Preservation (For Authenticity Validation)

**CRITICAL:** All scraped content must be saved to raw-data/ for authenticity verification.

```yaml
on_successful_scrape:
  1. Save full response to: raw-data/ads_{advertiser}_{timestamp}.json
  2. Generate SHA-256 hash of file
  3. Save hash to: raw-data/ads_{advertiser}_{timestamp}_hash.json
  4. Log in state manager: {file, hash, items_count, timestamp}

NEVER modify raw-data/ files after creation - they are the source of truth.
```

Use tool chain (Firecrawl primary, Apify fallback) depending on effectiveness.

## Ad Copy Analysis Markers

### What to Capture (for Layer 2 analysis)

**Headlines:**
- Main claims
- Benefit statements
- Hook language

**Body Copy:**
- Value propositions
- Problem/solution framing
- Social proof claims
- Objection handling

**CTAs:**
- Action language
- Urgency indicators
- Offer framing

### Duration-Based Signals

```
0-7 days: Testing phase - could be anything
7-30 days: Showing promise - worth noting
30-90 days: Proven winner - pay attention
90+ days: Evergreen - core positioning
```

Long-running ads indicate:
- Claims that work
- Offers that convert
- Positioning that resonates

## Failure Protocol

```
IF Ads Library blocked/changed:
  - Report specific issue
  - Try alternative access method
  - If blocked, HALT
  - Note critical gap

IF advertiser not found:
  - Report which advertiser
  - Try alternative search terms
  - Continue with found advertisers
  - Note gap

IF ad copy truncated:
  - Note truncation
  - Capture what's visible
  - Flag for potential manual review
```
