## What is Saturation?

**Saturation is reached when:**
1. Same patterns appear 5+ times from different sources
2. New scraping produces quotes that repeat existing patterns (not new ones)
3. All research topics have adequate quote coverage (50+ per major topic)
4. No ambiguous clusters remain unresolved

**Saturation is NOT reached when:**
1. PRD minimums are met but patterns are still emerging
2. Clusters of similar pain points cannot be differentiated
3. Research topics have < 50 quotes
4. Competitor mechanisms are not fully mapped

---

## Saturation Analysis Process

### Step 1: Pattern Repetition Analysis

**Definition:** A "pattern" is a recurring theme, complaint, desire, or belief.

```yaml
pattern_analysis:
  method:
    1. Identify unique patterns across all quotes
    2. Count occurrences of each pattern
    3. Track source diversity for each pattern
    4. Flag patterns appearing < 5 times (may need expansion)

  saturation_indicators:
    high: "Pattern appears 5+ times from 3+ different sources"
    medium: "Pattern appears 3-4 times"
    low: "Pattern appears 1-2 times"

  saturation_threshold:
    - 90%+ patterns at HIGH → SATURATED
    - 70-89% patterns at HIGH → PARTIAL (may need expansion)
    - < 70% patterns at HIGH → EXPANDING_REQUIRED
```

### Step 2: Cluster Detection

**Definition:** A "cluster" is a group of related pain points/hopes that may be the same phenomenon with different names, OR may be distinct issues that need differentiation.

**Cluster Detection Algorithm:**

```yaml
cluster_detection:
  method:
    1. Identify groups of subcategories with high semantic similarity
    2. Calculate quote overlap (quotes assigned to multiple related subcategories)
    3. Analyze language patterns for differentiation signals

  cluster_types:
    validated_single:
      description: "Different names for same phenomenon"
      action: "Merge subcategories, document as single phenomenon"
      example: "Thinning, skulling, blading = same miss with different names"

    validated_distinct:
      description: "Related but clearly different phenomena"
      action: "Keep separate, document differentiation"
      example: "Chunking (club digs) vs Fat shot (bottom swing) = different causes"

    ambiguous:
      description: "Cannot determine if same or different"
      action: "EXPAND search to gather differentiating quotes"
      example: "Chunking, fat shots, digging - unclear if same or different"
```

### Step 3: Cluster Resolution Protocol

**For each ambiguous cluster:**

```yaml
resolution_protocol:
  step_1_analyze:
    - Identify all quotes in cluster
    - Look for differentiation language ("unlike chunking, digging is...")
    - Look for conflation language ("I chunk it, you know, dig into the ground")

  step_2_decide:
    if differentiation_evidence_strong:
      → VALIDATE_AS_DISTINCT
      → Keep subcategories separate
      → Document differentiation criteria

    if conflation_evidence_strong:
      → VALIDATE_AS_SINGLE
      → Merge subcategories
      → Use customer's most common term

    if evidence_unclear:
      → EXPAND_SEARCH
      → Generate targeted queries
      → Return to scraping
      → Re-analyze after expansion

  step_3_expand_queries:
    - "difference between [term A] and [term B]"
    - "[term A] vs [term B]"
    - "is [term A] the same as [term B]"
    - "[term A] causes" + "[term B] causes" (compare)
```

### Step 4: Topic Coverage Analysis

**Check each research topic from context expansion:**

```yaml
topic_coverage_analysis:
  minimum_per_topic: 50 quotes

  analysis:
    for each topic in context_expansion.research_topics.primary:
      - Count quotes tagged to this topic
      - If < 50: Flag for expansion
      - Generate topic-specific expansion queries

  expansion_triggers:
    - Topic has < 50 quotes
    - Topic has no emotional angle quotes
    - Topic has no competitive context
```

### Step 5: Competitor Coverage Check

```yaml
competitor_coverage:
  minimum_competitors: 5
  minimum_mechanisms: 15 unique

  check:
    - Each competitor has root cause identified?
    - Each competitor has mechanism mapped?
    - Each competitor has weakness identified?

  expansion_if:
    - < 5 competitors fully analyzed
    - < 15 mechanisms mapped
    - Any competitor missing root cause/mechanism/weakness
```

## Expansion Decision Logic

### Decision Tree

```
START
  │
  ├─> Check PRD Minimums
  │     │
  │     ├─> NOT MET → EXPAND (target underperforming buckets)
  │     │
  │     └─> MET → Continue
  │
  ├─> Check Pattern Saturation
  │     │
  │     ├─> < 70% HIGH → EXPAND (general expansion)
  │     │
  │     └─> >= 70% HIGH → Continue
  │
  ├─> Check Cluster Ambiguity
  │     │
  │     ├─> Ambiguous clusters exist → EXPAND (targeted cluster resolution)
  │     │
  │     └─> All clusters resolved → Continue
  │
  ├─> Check Topic Coverage
  │     │
  │     ├─> Topics with < 50 quotes → EXPAND (target specific topics)
  │     │
  │     └─> All topics >= 50 → Continue
  │
  ├─> Check Competitor Coverage
  │     │
  │     ├─> < 5 competitors or < 15 mechanisms → EXPAND (competitor research)
  │     │
  │     └─> Coverage adequate → Continue
  │
  └─> All checks pass → SATURATION_ACHIEVED
```

### Expansion Types

```yaml
expansion_types:
  GENERAL:
    trigger: "PRD minimums not met, overall quote shortage"
    action: "Add more sources across all topics"

  TARGETED_BUCKET:
    trigger: "Specific bucket below minimum"
    action: "Add sources likely to yield that bucket type"
    example: "Pain quotes low → Add forum complaint threads"

  CLUSTER_RESOLUTION:
    trigger: "Ambiguous cluster detected"
    action: "Add sources with differentiation language"
    example: "Chunking vs digging unclear → Search for comparison content"

  TOPIC_COVERAGE:
    trigger: "Research topic underrepresented"
    action: "Add sources specifically about that topic"
    example: "Bunker fear low → Search for bunker anxiety content"

  COMPETITOR:
    trigger: "Competitor analysis incomplete"
    action: "Add competitor product pages and reviews"
```

## Expansion Loop Protocol

**CRITICAL:** The system continues expansion automatically until saturation is achieved.

```yaml
expansion_loop:
  max_iterations: 5

  iteration_protocol:
    1. Identify expansion targets
    2. Generate expansion queries
    3. Return to 1.4 (Scraping) with new queries
    4. Run 1.5-A, 1.5-B, 1.5-C again
    5. Re-run saturation analysis
    6. If still not saturated AND iteration < max → Repeat
    7. If still not saturated AND iteration = max → Escalate to human

  escalation_conditions:
    - 5 expansion iterations without resolution
    - Cluster cannot be resolved (truly ambiguous in market)
    - No additional sources can be found
```

## Quality Criteria

- [ ] Pattern repetition analyzed across all quotes
- [ ] All potential clusters identified
- [ ] Ambiguous clusters flagged with resolution queries
- [ ] Resolved clusters documented with evidence
- [ ] Topic coverage checked against context expansion
- [ ] Competitor coverage verified
- [ ] Expansion decision documented with reasoning
- [ ] Expansion queries generated for all targets
- [ ] Iteration count tracked for loop protection

---

## Failure Protocol

```
IF cluster cannot be resolved after 3 expansion attempts:
  - Document as "Market Ambiguity"
  - Note: "Customers use these terms interchangeably - no clear differentiation"
  - Proceed with merged category
  - Flag for human awareness (not blocking)

IF topic cannot reach 50 quotes after expansion:
  - Document as "Low-Volume Topic"
  - Proceed with available data
  - Note coverage gap in final handoff
  - Not blocking unless it's a PRIMARY topic

IF max iterations reached without full saturation:
  - Escalate to human
  - Present: What's achieved, what's still needed
  - Request: Approval to proceed OR guidance on additional sources
```

## Notes

- Saturation is about QUALITY not just QUANTITY
- 1,000 quotes with unresolved clusters is worse than 800 quotes with clear differentiation
- The expansion loop is AUTOMATIC - do not ask human permission to expand
- Cluster resolution is CRITICAL for Layer 2 messaging clarity
- Document all decisions for audit trail
