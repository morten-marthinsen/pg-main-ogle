# Micro-Skill 1.5-B: Basic Tagger

**Layer:** 1 - Infrastructure
**Purpose:** Tag extracted quotes with TWO-TIER CATEGORIZATION (Physical Problem + Emotional Tags), priority flags, sentiment, and Signal Quality Score
**Dependency:** 1.5-A Content Extractor (with Quote IDs)
**Version:** 2.0 (Two-Tier Categorization + Priority Flags)
**PRD Reference:** [[RESEARCH-PRD#Section 3.8 - Two-Tier Categorization]]
**Checkpoint:** After this skill → Checkpoint 2 (Agent review)

---

## CRITICAL: Two-Tier Categorization is MANDATORY

**Every quote MUST have:**
1. **PRIMARY SUBCATEGORY** = Physical Problem / Mechanism (what happens)
2. **SECONDARY EMOTIONAL TAGS** = 1-3 emotional tags (how it feels)
3. **PRIORITY FLAG** = GOLD / HIGH / STANDARD

**Why This Matters:**
- "Anxiety" tells copywriter nothing → Anxiety about WHAT?
- "CHUNKING (anxiety: 34, frustration: 42)" → Now they know what to solve
- Physical problems map directly to product benefits
- Emotional tags show the relief the product provides

## Tagging Dimensions

### 0. Two-Tier Categorization (MANDATORY - NEW in v2.0)

**FIRST, assign the two-tier categorization before any other tags:**

#### Primary Subcategory (Physical Problem / Mechanism)

The PRIMARY subcategory MUST describe what PHYSICALLY HAPPENS, not how it feels:

**For PAIN Bucket:**
```yaml
physical_problem_subcategories:
  - CHUNKING / FAT SHOTS: Club digs, ball goes nowhere
  - THINNING / SKULLING / BLADING: Ball rockets low
  - YIPS / INVOLUNTARY MOVEMENTS: Flinch, decel, freeze
  - DIGGING / LEADING EDGE: Club stops in turf
  - DISTANCE CONTROL: Can't judge 30-80 yards
  - TRAJECTORY ISSUES: Ball flight problems
  - BUNKER STRUGGLES: Sand play failures
  - INCONSISTENCY: Variable contact quality
```

**For HOPE Bucket:**
```yaml
product_outcome_subcategories:
  - ANTI-DIG / GLIDE: Club accelerates through
  - CONSISTENT CONTACT: Clean strikes regularly
  - FORGIVENESS: Mishits still work
  - CONFIDENCE RESTORED: No longer afraid
  - SCORE IMPROVEMENT: Strokes saved
  - ENJOYMENT RETURNED: Fun again
```

**For ROOT_CAUSE Bucket:**
```yaml
belief_subcategories:
  - TECHNIQUE BLAME: Swing/mechanics issue
  - EQUIPMENT BLAME: Club design issue
  - MENTAL GAME BLAME: Psychology issue
  - PHYSICAL LIMITATION: Body/age issue
  - TEACHING FAILURE: Bad instruction
```

**For SOLUTIONS_TRIED Bucket:**
```yaml
solution_type_subcategories:
  - LESSONS/INSTRUCTION: Worked with teacher
  - EQUIPMENT PURCHASE: Bought clubs/balls
  - VIDEO LEARNING: YouTube, online courses
  - DRILLS/PRACTICE: Range work
  - MENTAL TECHNIQUES: Visualization, routines
  - WORKAROUNDS: Changed technique to avoid problem
```

#### Secondary Emotional Tags (1-3 per quote)

```yaml
emotional_tags:
  pain_emotions:
    - anxiety
    - fear
    - frustration
    - embarrassment
    - despair
    - hopelessness
    - anger
    - shame
    - score_impact
    - quitting_consideration

  hope_emotions:
    - relief
    - confidence
    - freedom
    - trust
    - predictability
    - permission
    - ease
    - enjoyment
    - validation
    - progress
```

#### Priority Flag Assignment

```yaml
priority_flags:
  GOLD:
    criteria:
      - Academic/research source (PhD studies, scientific papers)
      - Actual product testimonial (review of THE product being researched)
      - Direct validation of specific product mechanism
      - Expert authority source
    action: "MUST appear in Top 25 for subcategory"

  HIGH:
    criteria:
      - Highly specific language with details (names, numbers, money)
      - Strong emotional intensity with clear mechanism
      - Competitor product review showing unmet need
      - Before/after transformation story
    action: "Prioritize in Top 25 selection"

  STANDARD:
    criteria:
      - General user discussion
      - Relevant but not highly quotable
      - Supporting evidence
    action: "Include in counts, lower priority for Top 25"
```

#### Product Benefit Validation

When assigning priority, check if quote validates a specific product benefit:

```yaml
benefit_validation:
  check_for:
    - Does quote describe problem that product solves?
    - Does quote express desire that product fulfills?
    - Does quote confirm mechanism product uses?

  if_validates:
    - Set priority_flag to GOLD or HIGH
    - Set validates_product_benefit to "[specific benefit name]"

  examples:
    - Quote: "Fear of club digging into ground"
      validates_product_benefit: "Anti-dig sole design"
      priority_flag: "GOLD"

    - Quote: "I just want the club to glide through"
      validates_product_benefit: "Glide technology"
      priority_flag: "HIGH"
```

### 1. Sentiment

| Tag | Criteria | Examples |
|-----|----------|----------|
| **positive** | Expresses satisfaction, enthusiasm, love | "Best ball I've ever played", "Finally found what works" |
| **negative** | Expresses frustration, disappointment, complaint | "Waste of money", "Doesn't perform as advertised" |
| **neutral** | Factual, descriptive, no emotional valence | "The compression is 90", "I switched last month" |
| **mixed** | Contains both positive and negative elements | "Great feel but too expensive", "Works but scratches easily" |

### 2. Length Tier

| Tier | Word Count | Value |
|------|------------|-------|
| **short** | <25 words | Lower - less context |
| **medium** | 25-100 words | Standard - adequate detail |
| **long** | >100 words | Higher - rich detail |

### 3. Source Type

| Type | Platforms |
|------|-----------|
| **forum** | Reddit, GolfWRX, specialty forums |
| **video** | YouTube (comments and transcripts) |
| **social** | TikTok, Instagram |
| **review** | Amazon, specialty retailers |
| **competitive** | Ads Library, competitor funnels |

### 4. Signal Quality Score (SQS)

1-10 scale measuring potential research value:

**SQS 9-10: Exceptional**
- Specific product experience with details
- Emotional language + specifics
- Personal story with before/after
- Decision-making reasoning explained

**SQS 7-8: High**
- Clear opinion with some reasoning
- Specific product mentions
- Emotional language OR specifics
- Comparison between options

**SQS 5-6: Medium**
- General opinion stated
- Some relevance but vague
- Generic benefit/complaint
- No specific details

**SQS 3-4: Low**
- Tangential relevance
- Very brief
- No emotional signal
- No specifics

**SQS 1-2: Minimal**
- Barely relevant
- Generic statement
- No usable signal
- Consider excluding

## SQS Calculation

### Scoring Components

| Component | Points | Criteria |
|-----------|--------|----------|
| Specificity | 0-3 | Names, numbers, dollar amounts |
| Emotional intensity | 0-3 | Strong feeling words, personal stakes |
| Detail depth | 0-2 | Story, context, reasoning |
| Engagement | 0-2 | High upvotes/likes/helpful votes |

**SQS = Sum of components (cap at 10)**

### Scoring Examples

```
"Pro V1 is good" → SQS 3
- Specificity: 1 (product name)
- Emotion: 0 (neutral)
- Detail: 0 (no context)
- Engagement: unknown

"I've played Pro V1 for 5 years and finally switched to Kirkland. Can't tell the difference. Saving $30/dozen." → SQS 8
- Specificity: 3 (products, time, money)
- Emotion: 2 (surprise, satisfaction)
- Detail: 2 (personal story)
- Engagement: 1 (if high score)

"After losing 4 balls on the front nine (at $4 each, that's $16!), I was so frustrated I started using found balls from the woods. My ego couldn't take playing 'real' balls anymore when I lose that many." → SQS 10
- Specificity: 3 (numbers, dollar amounts)
- Emotion: 3 (frustration, shame, identity)
- Detail: 2 (full story, reasoning)
- Engagement: 2 (if resonant)
```

## Tagging Process

### Step 1: Sentiment Classification
For each quote:
1. Scan for sentiment indicators
2. Check for mixed signals
3. Assign sentiment tag

### Step 2: Length Tiering
For each quote:
1. Count words
2. Assign tier

### Step 3: Source Type
From source metadata:
1. Map platform to source type
2. Assign tag

### Step 4: SQS Calculation
For each quote:
1. Score each component
2. Sum and cap at 10
3. Record component breakdown

### Step 5: Distribution Analysis
After all quotes tagged:
1. Calculate sentiment distribution
2. Calculate length distribution
3. Calculate SQS distribution
4. Flag anomalies

## Distribution Flags

Flag for Checkpoint 2 review if:

| Condition | Flag |
|-----------|------|
| >80% one sentiment | Possible bias in sources |
| <10% high SQS (7+) | Quality concern |
| >50% low SQS (<4) | Relevance concern |
| <20% long quotes | Depth concern |

---

## Quality Criteria

- [ ] All quotes tagged on all dimensions
- [ ] SQS calculated with breakdown
- [ ] Distributions calculated
- [ ] Anomalies flagged
- [ ] No untagged quotes

---

## Failure Protocol

```
IF tagging unclear:
  - Default to middle values
  - Note uncertainty
  - Continue

IF SQS calculation ambiguous:
  - Score conservatively
  - Note in breakdown
```
