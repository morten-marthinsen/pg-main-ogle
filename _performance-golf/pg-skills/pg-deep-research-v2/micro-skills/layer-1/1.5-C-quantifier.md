# Micro-Skill 1.5-C: Quantifier

**Layer:** 1 - Infrastructure
**Purpose:** Tally quotes into quantified buckets with TWO-TIER subcategorization, generate totals, extract top 25 verbatim per subcategory, perform PAIR EXTRACTION, and track SOURCE DISTRIBUTION
**Dependency:** 1.5-A Content Extractor, 1.5-B Basic Tagger complete
**Version:** 5.0 (Pair Extraction + Forum Awareness Protocol)
**PRD Reference:** [[RESEARCH-PRD#Section 3.2]], [[RESEARCH-PRD#Section 3.6 - Pair Extraction]], [[RESEARCH-PRD#Rule 2 - Source Quality]]
**Core Infrastructure:** [[0.5-chunking-manager]] - Required for processing 1,000+ quotes

---

## CRITICAL: TWO-TIER CATEGORIZATION IS MANDATORY

### The Core Problem This Solves

Previous versions categorized by EMOTIONAL STATE (anxiety, frustration, fear). This made outputs less actionable because:
- "Fear and dread of WHAT?" → The emotion doesn't tell you what product benefit to emphasize
- Copywriters need to know the PHYSICAL PROBLEM the product solves

**THE FIX: Two-Tier System**
- **Primary Tier (MANDATORY):** PHYSICAL PROBLEM / MECHANISM (what happens)
- **Secondary Tier (MANDATORY):** EMOTIONAL TAG (how it makes them feel)

Every quote must be:
1. Assigned to a PRIMARY bucket (Pain, Hope, Root Cause, Solutions Tried, Competitor)
2. Assigned to a PRIMARY SUBCATEGORY = **PHYSICAL PROBLEM** (chunking, thinning, digging, etc.)
3. Assigned SECONDARY EMOTIONAL TAGS (anxiety, frustration, embarrassment, etc.)
4. Flagged for PRIORITY if it validates a specific product benefit
5. Counted and tallied
6. If subcategory has 25+ quotes, top 25 verbatim extracted

**This is NOT optional. This is the foundation for all Layer 2 analysis.**

## MANDATORY: Pair Extraction (PRD Section 3.6)

**Pairs are the copywriting GOLD MINE.** They connect problems to desires in the prospect's own words.

### Required Pair Types

```yaml
pair_extraction:
  pain_hope_pairs:
    description: "Connect what hurts to what they want"
    minimum_required: 25
    format:
      pain_quote_id: "P-XXX"
      hope_quote_id: "H-XXX"
      connection_strength: "STRONG | MEDIUM | WEAK"
      copy_angle: "[How this pair could be used in marketing]"

    example:
      pain: "P-047: I chunked so many pitches I wanted to throw my wedge in the lake"
      hope: "H-023: I just want to be able to chip without fear"
      connection: "STRONG - Same author, same thread, direct problem→desire arc"
      copy_angle: "From lake-throwing frustration to fear-free chipping"

  root_cause_mechanism_pairs:
    description: "Connect their beliefs about WHY to HOW the product works"
    minimum_required: 25
    format:
      root_cause_quote_id: "RC-XXX"
      mechanism_quote_id: "CM-XXX"
      alignment: "ALIGNED | CHALLENGED"
      copy_angle: "[How to use this belief→mechanism connection]"

    example:
      root_cause: "RC-012: I think it's because the leading edge digs in too much"
      mechanism: "CM-034: The wide sole design prevents the club from digging"
      alignment: "ALIGNED - Belief matches mechanism directly"
      copy_angle: "They already believe it's the edge - we just confirm and solve"
```

### Pair Identification Process

```yaml
pair_identification:
  step_1_same_source_pairs:
    - Search for quotes from same author
    - Search for quotes from same thread
    - These are HIGHEST confidence pairs
    - Mark as "SAME_SOURCE"

  step_2_thematic_pairs:
    - Match pain physical problem to hope outcome
    - Match root cause belief to mechanism that addresses it
    - These require semantic matching
    - Mark as "THEMATIC_MATCH"

  step_3_contrast_pairs:
    - Find quotes that show before/after contrast
    - "I used to... now I..."
    - "Before I had X, after I have Y"
    - Mark as "CONTRAST_PAIR"

  quality_scoring:
    same_source: +3 points
    same_thread: +2 points
    same_topic: +1 point
    strong_semantic_match: +2 points
    contrast_language: +2 points
```

```yaml

    - Analyze forum sources separately
    - Analyze non-forum sources separately

    - Identify patterns ONLY in forums (advanced prospect specific)
```

## Primary Bucket Definitions

### Bucket 1: PAIN QUOTES
**Definition:** Physical problems, failures, frustrations, and negative experiences

**What belongs here:**
- Descriptions of bad shots (chunks, thins, shanks)
- Descriptions of failures (missed up-and-downs, poor scores)
- Expressions of frustration about performance
- Fear-based statements about specific shots/situations

**What does NOT belong here:**
- Beliefs about WHY problems happen (that's Root Cause)
- What they've tried to fix it (that's Solutions Tried)
- What they wish would happen (that's Hope)

### Bucket 2: HOPE QUOTES
**Definition:** Desires, aspirations, what they want to experience

**What belongs here:**
- "I wish I could..."
- "I want to..."
- "My goal is to..."
- Descriptions of desired outcomes
- Expressions of what success looks like

**What does NOT belong here:**
- Current problems (that's Pain)
- How to achieve the desire (that's Solutions Tried)

### Bucket 3: ROOT CAUSE BELIEFS (Why It Hurts)
**Definition:** What customers BELIEVE causes their problem

**What belongs here:**
- "I think the problem is..."
- "The reason I struggle is..."
- "It's because of my..."
- Technique beliefs
- Equipment beliefs
- Mental/psychological beliefs
- Physical limitation beliefs

**What does NOT belong here:**
- Actual problems (that's Pain)
- Solutions they've tried (that's Solutions Tried)

### Bucket 4: SOLUTIONS TRIED (How It Works)
**Definition:** What customers have ALREADY attempted to solve their problem

**What belongs here:**
- "I tried..."
- "I switched to..."
- "I've been working on..."
- Technique changes attempted
- Equipment changes made
- Lessons/instruction received
- Mental approaches tried

**What does NOT belong here:**
- Whether it worked or not (capture as separate data point)
- Future plans (that might be Hope)

### Bucket 5: COMPETITOR MECHANISM
**Definition:** How competitors claim their solution works

**What belongs here:**
- Technology names and descriptions
- Marketing claims about how product works
- Feature-to-benefit explanations
- Proprietary terminology

**What does NOT belong here:**
- User opinions about competitors (that's Pain or Hope depending on sentiment)

## TWO-TIER SUBCATEGORY SYSTEM

### Why Two Tiers?

**The Problem with Single-Tier Emotional Categories:**
- "Anxiety/Fear/Dread: 67 mentions" → Anxiety about WHAT?
- Copywriters can't use this → Which product benefit addresses "anxiety"?
- Emotional categories are symptoms, not mechanisms

**The Solution: Physical Problem Primary, Emotion Secondary**
- "CHUNKING: 112 mentions (anxiety: 34, frustration: 42, embarrassment: 28)"
- Now copywriters know: Product solves CHUNKING → causes relief from anxiety
- The mechanism maps directly to product benefits

### Primary Tier: PHYSICAL PROBLEM / MECHANISM

**For PAIN Bucket:**
The primary subcategory MUST describe what PHYSICALLY HAPPENS:

| Primary Subcategory | Definition | Product Benefit It Maps To |
|---------------------|------------|---------------------------|
| CHUNKING / FAT SHOTS | Club digs, ball goes nowhere | Anti-dig sole, glide technology |
| THINNING / SKULLING / BLADING | Ball rockets low across green | Lower CG, consistent launch |
| YIPS / INVOLUNTARY MOVEMENTS | Flinch, decel, hands stop | Confidence-building design |
| DIGGING / LEADING EDGE | Club stops in turf | Wide sole, bounce design |
| DISTANCE CONTROL | Can't judge 30-80 yards | Consistent launch, predictable flight |

**For HOPE Bucket:**
The primary subcategory MUST describe what PRODUCT OUTCOME they want:

| Primary Subcategory | Definition | Emotional Tags |
|---------------------|------------|----------------|
| ANTI-DIG / GLIDE | Club accelerates through | relief, confidence |
| CONSISTENT CONTACT | Clean strikes regularly | trust, predictability |
| FORGIVENESS | Mishits still work | permission, ease |
| CONFIDENCE RESTORED | No longer afraid | freedom, enjoyment |
| SCORE IMPROVEMENT | Strokes saved | validation, progress |

### Secondary Tier: EMOTIONAL TAGS

Every quote gets 1-3 emotional tags from this list:

```yaml
pain_emotions:
  - anxiety
  - fear
  - frustration
  - embarrassment
  - despair
  - hopelessness
  - score_impact
  - quitting
  - equipment_blame
  - fear_of_digging

hope_emotions:
  - relief
  - confidence
  - freedom
  - trust
  - predictability
  - permission
  - ease
  - enjoyment
  - validation
  - progress
```

### Priority Flagging

**GOLD Priority (Must-Use Quotes):**
- Academic/research validation (PhD studies, scientific sources)
- Actual product testimonials (reviews of THE product being researched)
- Quotes that directly validate a specific product mechanism
- Example: "Fear of club digging into ground" (PGA PhD Research)

**HIGH Priority:**
- Highly specific user language with details
- Strong emotional intensity
- Competitor product reviews showing unmet needs
- Example: "I went from blading across the green... to getting a few tap-ins"

**STANDARD Priority:**
- General user discussions
- Forum comments without specific details
- Supportive but not quotable verbatim

### Three Questions for Every Quote

When categorizing, ask IN THIS ORDER:

1. **"What PHYSICAL PROBLEM does this quote describe?"** → Primary subcategory
   - Bad answer: "Anxiety" (that's an emotion, not a mechanism)
   - Good answer: "Chunking" (that's what physically happens)

2. **"What EMOTIONAL IMPACT does this cause?"** → Secondary tag(s)
   - Answer: anxiety, frustration, embarrassment

3. **"Does this quote validate a specific PRODUCT BENEFIT?"** → Priority flag
   - If YES (academic source, product review, mechanism validation) → GOLD or HIGH
   - If NO → STANDARD

## Subcategory Generation

### Dynamic Subcategory Identification

Subcategories are identified DYNAMICALLY from the content, not pre-defined. However, the PRIMARY tier must ALWAYS be a PHYSICAL PROBLEM or MECHANISM, never an emotion.

**Process:**
1. Cluster similar quotes within each bucket
2. Identify natural groupings (3+ quotes about same specific PHYSICAL PROBLEM)
3. Name the subcategory using PHYSICAL/MECHANICAL language
4. Assign emotional tags as secondary data
5. Flag priority quotes

**For Pain Bucket (Example - Wedges):**
```yaml
primary_subcategories_physical:
  - CHUNKING / FAT SHOTS
  - THINNING / SKULLING / BLADING
  - YIPS / INVOLUNTARY MOVEMENTS
  - DIGGING / LEADING EDGE
  - DISTANCE CONTROL PROBLEMS
  - TRAJECTORY ISSUES
  - BUNKER FAILURES

secondary_tags_emotional:
  - anxiety, fear, frustration, embarrassment, despair, etc.
```

**For Pain Bucket (Example - Irons):**
```yaml
primary_subcategories_physical:
  - FAT / CHUNKED SHOTS
  - THIN / TOPPED SHOTS
  - SHANKS
  - DISTANCE INCONSISTENCY
  - TRAJECTORY PROBLEMS
  - MISHIT PUNISHMENT
  - DISPERSION ISSUES

secondary_tags_emotional:
  - anxiety, fear, frustration, embarrassment, etc.
```

### Subcategory Naming Rules

1. PRIMARY tier uses MECHANICAL/PHYSICAL language (what happens)
2. SECONDARY tier uses EMOTIONAL language (how it feels)
3. NEVER name a primary subcategory with only emotional terms
4. Use CUSTOMER LANGUAGE where possible
5. Be specific enough to differentiate
6. Be broad enough to contain 15+ quotes
7. If a subcategory has < 15 quotes, consider merging with related subcategory

## Top 25 Verbatim Selection

### Selection Criteria

For each subcategory with 25+ quotes, select the TOP 25 based on:

**1. Specificity Score (40% weight)**
- Contains dollar amounts
- Contains time periods
- Contains product names
- Contains quantities
- Contains personal details (handicap, years playing)

**2. Emotional Intensity (30% weight)**
- Strong emotional language
- Vivid descriptions
- Personal frustration/joy expressed

**3. Usability for Copy (20% weight)**
- Clear, quotable language
- Complete thought
- Resonant phrasing

**4. Source Authority (10% weight)**
- High engagement (upvotes, replies)
- Experienced golfer indicators
- Verified/authentic source

### Selection Process

```python
# Pseudo-code for top 25 selection
for each subcategory:
    if quote_count >= 25:
        score_all_quotes(specificity, emotion, usability, authority)
        rank_by_total_score()
        select_top_25()
        include_selection_reason_for_each()
```

## Quantification Process

### Step 1: Bucket Assignment Verification

For each tagged quote:
1. Verify bucket assignment from 1.5-B is correct
2. Re-assign if necessary
3. Flag any quotes that could belong to multiple buckets

### Step 2: Subcategory Identification

For each bucket:
1. Read all quotes
2. Identify natural clusters
3. Name subcategories
4. Assign quotes to subcategories
5. Handle edge cases (quotes that fit multiple subcategories → assign to primary)

### Step 3: Tallying

For each subcategory:
1. Count total quotes
2. Generate "Total Mentions: [X]" format
3. Aggregate to bucket level
4. Aggregate to total level

### Step 4: Top 25 Selection

For each subcategory with 25+ quotes:
1. Score all quotes
2. Rank by score
3. Select top 25
4. Document selection reason for each

### Step 5: PRD Validation

Check against [[RESEARCH-PRD#Section 3.1]]:
```yaml
validation:
  total_quotes: >= 1000
  pain_quotes: >= 300
  hope_quotes: >= 250
  root_cause_quotes: >= 200
  solutions_tried_quotes: >= 150
```

**IF ANY MINIMUM NOT MET:**
- Identify which bucket(s) are deficient
- Identify which subcategories are underrepresented
- Return: `expansion_needed: true` with `expansion_targets`
- This triggers return to Step 1.4 for additional scraping

## Failure Protocol

```
IF bucket assignment unclear:
  - Assign to most likely bucket
  - Flag for review
  - Continue processing

IF subcategory too small (<15 quotes):
  - Consider merging with related subcategory
  - OR keep separate if genuinely distinct topic

IF PRD minimum not met:
  - Set expansion_needed: true
  - Identify specific expansion_targets
  - Return to Master Agent for expansion loop
  - DO NOT proceed to saturation analysis
```

## Notes

- This skill is the BRIDGE between extraction and analysis
- Quality of quantification directly impacts Layer 2 intelligence
- Top 25 selection is for COPYWRITING USE - choose quotable, usable quotes
- Subcategories should reflect CUSTOMER LANGUAGE, not marketing jargon
- PRD validation is BLOCKING - cannot proceed if minimums not met

### WHY TWO-TIER CATEGORIZATION MATTERS (v4.0)

**The Core Insight:**
"Fear and dread of WHAT? Of chunking!"

Emotions are SYMPTOMS. Physical problems are MECHANISMS. Copywriters need to know:
- What physical problem does the product solve? → Primary subcategory
- What emotion does solving that problem relieve? → Secondary tag
- Can I trust this quote for marketing? → Priority flag

**Before v4.0:**
- "Anxiety/Fear/Dread: 67 mentions" → Useless for copy
- No way to map quotes to product benefits
- Emotional categories didn't answer "anxiety about WHAT?"

**After v4.0:**
- "CHUNKING: 112 mentions (anxiety: 34, frustration: 42)" → Actionable
- Direct mapping: CHUNKING → Anti-dig sole benefit
- Copywriter knows: "Solve chunking → relieve anxiety"

**This ensures ANY product researched will have mechanically-relevant categorization.**
