# Micro-Skill 1.4-D: Deep Scraper - Social (TikTok/Instagram)

**Layer:** 1 - Infrastructure
**Purpose:** Scrape social media posts and comments from approved sources
**Dependency:** 1.3-A Source Validator (Checkpoint 1 passed)
**Parallel:** Can run simultaneously with other 1.4 skills
**Tool Requirement:** APIFY (specialized actors for each platform)
**Version:** 2.0 (Updated with Tool Resilience)
**Core Dependency:** 0.2-Tool-Resilience (Fallback Chains)

---

## CRITICAL: Use Tool Chains, Not Direct Tool Calls

**OLD (Single Point of Failure):**
```python
# DO NOT DO THIS
result = mcp__apify__call_actor(actor="clockworks/tiktok-scraper", input={...})
```

**NEW (With Fallback Chain):**
```python
# REQUIRED APPROACH
result = execute_chain(
    chain_id="SOCIAL_SCRAPING",
    params={"hashtags": hashtags, "profiles": profiles, "platform": "tiktok"}
)
```

All scraping MUST go through the tool chain system defined in [[0.2-tool-resilience]].

## Scraping Process

### Step 1: Target Identification
For each approved social source:
1. Identify specific accounts to scrape
2. Identify relevant hashtags
3. Build search queries

### Step 2: Hashtag Scraping
For relevant hashtags:
1. Scrape recent posts
2. Scrape top/popular posts
3. Capture all comments

### Step 3: Account Scraping
For influencer/brand accounts:
1. Scrape recent posts
2. Filter for product-relevant content
3. Capture all comments

### Step 4: Comment Extraction
For each post:
1. Load all comments (paginate if needed)
2. Capture reply threads where available
3. Extract engagement metrics

## Tool Configuration

### Tool Chain: SOCIAL_SCRAPING (from 0.2-tool-resilience.md)

```yaml
chain_id: SOCIAL_SCRAPING
fallback_sequence:
  tiktok:
    1. Apify clockworks/tiktok-scraper (primary)
    2. Apify apify/tiktok-scraper (secondary)
    3. Manual intervention

  instagram:
    1. Apify apify/instagram-scraper (primary)
    2. Apify shu8hern/instagram-scraper (secondary)
    3. Manual intervention
```

### Error Handling (Via Tool Chain)

```
ON tool_chain_success:
  - Continue processing
  - Log which tool succeeded (for monitoring)

ON tool_chain_fallback_used:
  - Log: primary actor failed, fallback succeeded
  - Continue processing
  - No human intervention needed

ON tool_chain_complete_failure:
  - Log failure for specific platform
  - Continue with OTHER platform (TikTok fails → still scrape Instagram)
  - Only HALT if BOTH platforms completely fail
  - Add failed URLs to manual_review_queue.json

ON rate_limit_detected:
  - Social platforms are aggressive with rate limits
  - Tool chain handles automatic backoff
  - If persistent: Pause that platform for 15 minutes, continue other platform
```

### Raw Data Preservation (For Authenticity Validation)

**CRITICAL:** All scraped content must be saved to raw-data/ for authenticity verification.

```yaml
on_successful_scrape:
  1. Save full response to: raw-data/social_{platform}_{source_id}_{timestamp}.json
  2. Generate SHA-256 hash of file
  3. Save hash to: raw-data/social_{platform}_{source_id}_{timestamp}_hash.json
  4. Log in state manager: {file, hash, items_count, timestamp}

NEVER modify raw-data/ files after creation - they are the source of truth.
```

## Social Media Value

Social platforms provide:
- Younger demographic language (different from forums)
- Visual context (product usage scenarios)
- Viral/trending topics
- Influencer positioning
- Quick emotional reactions

### High-Signal Content Types

**TikTok:**
- "Golf hack" videos
- Product comparison content
- "POV: when you..." format
- Duets/responses (debates)

**Instagram:**
- Brand content + comments
- User-generated product photos
- Reel reactions
- Story Q&As (if accessible)

## Failure Protocol

```
IF TikTok blocked:
  - Report specific error
  - Continue with Instagram
  - Note gap in output

IF Instagram blocked:
  - Report specific error
  - Continue with TikTok
  - Note gap in output

IF both blocked:
  - HALT
  - Report critical failure
  - Wait for human decision

IF rate limited:
  - Slow down requests
  - Continue with delays
  - Report slowdown
```

## Language Differences

Social media language tends to be:
- More casual/slang-heavy
- More emoji-laden
- More hyperbolic
- Shorter form
- More trend-influenced

Capture this language AS-IS for later analysis. These patterns matter for reaching different audiences.
