## Scraping Protocol

### Depth Levels

**Full (Tier 1 sources):**
- ALL reviews for each product (paginate completely)
- ALL Q&A items
- Top 20 products in category
- Capture 1-star through 5-star

**Standard (Tier 2 sources):**
- Top 200 reviews per product (sorted by helpful)
- Top 50 Q&A items
- Top 10 products in category

**Light (Tier 3 sources):**
- Top 50 reviews per product
- Top 20 Q&A items
- Top 5 products in category

### Review Selection Priority

1. Verified purchase reviews
2. Reviews with images/video
3. High-helpful-vote reviews
4. Recent reviews (last 12 months)
5. Extreme ratings (1-star and 5-star)

### Rating Distribution

Capture across the full spectrum:
- 5-star reviews (what people love)
- 1-star reviews (what people hate)
- 3-star reviews (often most nuanced)

Don't over-index on positive. Negative reviews are gold.

## Scraping Process

### Step 1: Product Discovery
For each approved review source:
1. Navigate to category pages
2. Identify best sellers
3. Identify product variants (colors, sizes, models)
4. Build product queue

### Step 2: Product Page Scraping
For each product:
1. Capture product metadata
2. Navigate to reviews section
3. Scrape all review pages
4. Navigate to Q&A section
5. Scrape all Q&A

### Step 3: Review Extraction
For each review:
1. Capture full review text
2. Extract metadata (rating, date, verified, helpful)
3. Note if review has images/video

### Step 4: Q&A Extraction
For each Q&A:
1. Capture question text
2. Capture all answers
3. Extract vote counts

## Tool Configuration

### Tool Chain: REVIEW_SCRAPING (from 0.2-tool-resilience.md)

```yaml
chain_id: REVIEW_SCRAPING
fallback_sequence:
  amazon:
    1. Apify junglee/amazon-reviews-scraper (primary - most reliable)
    2. Firecrawl scrape (fallback)
    3. Apify apify/amazon-scraper (secondary Apify)
    4. Manual intervention

  specialty_retail:
    1. Firecrawl scrape (primary)
    2. Apify web-scraper (fallback)
    3. Manual intervention
```

### Error Handling (Via Tool Chain)

```
ON tool_chain_success:
  - Continue processing
  - Log which tool succeeded (for monitoring)

ON tool_chain_fallback_used:
  - Log: primary tool failed, fallback succeeded
  - Continue processing
  - No human intervention needed

ON tool_chain_complete_failure:
  - Amazon is CRITICAL source
  - Log full failure report with all attempted tools
  - Add product URLs to manual_review_queue.json
  - HALT and report to human if Amazon completely fails
  - Continue with specialty retailers

ON rate_limit_detected:
  - Amazon is aggressive with rate limits
  - Tool chain handles automatic backoff
  - If persistent: Switch to Apify (often has better proxy handling)
```

### Raw Data Preservation (For Authenticity Validation)

**CRITICAL:** All scraped content must be saved to raw-data/ for authenticity verification.

```yaml
on_successful_scrape:
  1. Save full response to: raw-data/reviews_{source}_{product_id}_{timestamp}.json
  2. Generate SHA-256 hash of file
  3. Save hash to: raw-data/reviews_{source}_{product_id}_{timestamp}_hash.json
  4. Log in state manager: {file, hash, items_count, timestamp}

NEVER modify raw-data/ files after creation - they are the source of truth.
```

## High-Signal Review Patterns

### Positive Reviews to Flag
- "Finally found..." (solved long search)
- "Wish I switched sooner" (barrier overcome)
- "Worth every penny" (justified premium)
- "Game changer" (transformation language)

### Negative Reviews to Flag
- "Don't believe the hype" (expectation vs reality)
- "Returning this" (failed promise)
- "No different from cheaper option" (value challenge)
- "Quality has declined" (trend signal)

### Q&A Gold
- "Does this work for..." (audience signals)
- "How does this compare to..." (competitive context)
- "Is it worth upgrading from..." (decision triggers)

## Failure Protocol

```
IF Amazon blocked:
  - Report specific error
  - Try alternative approach (Apify actor)
  - If still blocked, HALT
  - Note critical gap (Amazon is primary)

IF specialty retailer blocked:
  - Report which retailer
  - Continue with other sources
  - Note gap in output

IF pagination fails:
  - Note how many pages captured
  - Report where it stopped
  - Continue with captured data
  - Note incomplete in output
```
