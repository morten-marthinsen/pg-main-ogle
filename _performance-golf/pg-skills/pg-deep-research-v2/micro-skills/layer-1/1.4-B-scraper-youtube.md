## Scraping Protocol

### Depth Levels

**Full (Tier 1 sources):**
- 30+ videos per channel
- ALL comments (paginate to get all)
- Full transcripts
- Include video descriptions
- Target: 10K+ comments minimum

**Standard (Tier 2 sources):**
- 15-30 videos per channel
- Top 500 comments per video
- Full transcripts
- Target: 5K+ comments minimum

**Light (Tier 3 sources):**
- 5-15 videos per channel
- Top 100 comments per video
- Full transcripts
- Target: 1K+ comments minimum

### Video Selection Priority

1. Videos directly about the product category
2. Comparison/review videos
3. High-engagement videos (views + comments)
4. Recent videos (last 12 months)

## Scraping Process

### Step 1: Channel Analysis
For each approved YouTube source:
1. Access channel page
2. Identify relevant playlists/sections
3. Build video queue prioritized by relevance

### Step 2: Video Discovery
```
Method 1: Channel uploads (recent)
Method 2: Search within channel
Method 3: Relevant playlists
```

Filter for videos mentioning:
- Product category keywords
- Comparison/review language
- Problem/solution language

### Step 3: Transcript Extraction
For each relevant video:
1. Extract transcript (auto-generated OK if manual unavailable)
2. Preserve timestamps if useful
3. Clean formatting (remove [Music], timestamps if not needed)

### Step 4: Comment Scraping
For each video:
1. Load all comments (paginate)
2. Capture full comment thread (replies included)
3. Extract engagement metrics
4. Continue until all comments captured OR limit reached

### Step 5: Saturation Check
Monitor for:
- Repeated themes in comments
- Similar questions appearing
- Same debates/arguments

Note saturation but continue to depth target.

## Tool Configuration

### Tool Chain: VIDEO_SCRAPING (from 0.2-tool-resilience.md)

```yaml
chain_id: VIDEO_SCRAPING
fallback_sequence:
  1. Apify youtube-scraper (primary for comments + metadata)
  2. Firecrawl scrape (fallback for transcripts/page content)
  3. Manual intervention (if all automated tools fail)
```

### Apify for Comments (Primary)

```yaml
actor: "youtube-comment-scraper"
maxComments: 10000
includeReplies: true
```

### Firecrawl for Transcripts (Fallback)

```yaml
mode: "scrape"
target: video pages
extract: transcript widget
```

### YouTube API (if available)

```yaml
endpoints:
  - commentThreads
  - captions
  - videos (for metadata)
```

### Error Handling (Via Tool Chain)

```
ON tool_chain_success:
  - Continue processing
  - Log which tool succeeded (for monitoring)

ON tool_chain_fallback_used:
  - Log: primary tool failed, fallback succeeded
  - Continue processing
  - No human intervention needed

ON tool_chain_complete_failure:
  - All tools in chain failed for this URL
  - Log full failure report with all attempted tools
  - Add URL to manual_review_queue.json
  - Continue with other sources (do NOT halt entire scrape)
  - Report at end: "X URLs could not be scraped after all fallbacks"
```

### Raw Data Preservation (For Authenticity Validation)

**CRITICAL:** All scraped content must be saved to raw-data/ for authenticity verification.

```yaml
on_successful_scrape:
  1. Save full response to: raw-data/youtube_{source_id}_{timestamp}.json
  2. Generate SHA-256 hash of file
  3. Save hash to: raw-data/youtube_{source_id}_{timestamp}_hash.json
  4. Log in state manager: {file, hash, items_count, timestamp}

NEVER modify raw-data/ files after creation - they are the source of truth.
```

## Failure Protocol

```
IF transcript unavailable:
  - Log which video
  - Continue with comments only
  - Note gap in output

IF comments blocked/limited:
  - HALT
  - Report exact limitation
  - Wait for human decision

IF channel inaccessible:
  - Report which channel
  - HALT for that source
  - Continue with other sources
  - Note gap in output
```

## Comment Mining Value

YouTube comments are particularly valuable for:
- Unfiltered opinions (less curated than forum posts)
- Emotional reactions
- Questions about products
- Social proof language ("I switched to X and...")
- Debates between viewers

### Comment Patterns to Watch For

- "I've been using X for Y years..."
- "Why does everyone recommend X when..."
- "I tried X and it was [positive/negative]..."
- "What about [alternative]?"
- Debates in replies
