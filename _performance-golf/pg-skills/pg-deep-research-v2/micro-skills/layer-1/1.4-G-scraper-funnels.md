**Layer:** 1 - Infrastructure
**Purpose:** Scrape competitor landing pages and funnels from Ads Library URLs
**Dependency:** 1.4-F Ads Library Scraper (must complete first)

## CRITICAL: Use Tool Chains, Not Direct Tool Calls

**OLD (Single Point of Failure):**

## Scraping Protocol

### Step 1: URL Deduplication
From 1.4-F output:
1. Extract unique landing page URLs
2. Group by advertiser
3. Prioritize long-running ad destinations

### Step 2: Landing Page Scraping
For each URL:
1. Load full page
2. Wait for dynamic content
3. Extract all text content
4. Extract pricing information
5. Note page structure

### Step 3: Funnel Following (Limited)
From landing page:
1. Identify internal links (product pages, category pages)
2. Follow links one level deep
3. Do NOT follow to external sites
4. Do NOT enter checkout (no form submission)

### Step 4: Offer Extraction
Parse for:
1. Price points
2. Bundle options
3. Subscription options
4. Guarantee language
5. Shipping information

## Tool Configuration

### Tool Chain: FUNNEL_SCRAPING (from 0.2-tool-resilience.md)

```yaml
chain_id: FUNNEL_SCRAPING
fallback_sequence:
  1. Firecrawl scrape (primary - best for dynamic landing pages)
  2. Apify web-scraper (fallback)
  3. Manual intervention

notes:
  - Landing pages often have dynamic content
  - Need to wait for JavaScript loading
  - Scroll to capture below-fold content
  - Do NOT enter checkout or submit forms
```

### Error Handling (Via Tool Chain)

```
ON tool_chain_success:
  - Continue processing
  - Log which tool succeeded (for monitoring)

ON tool_chain_fallback_used:
  - Log: primary tool failed, fallback succeeded
  - Continue processing
  - No human intervention needed

ON tool_chain_complete_failure:
  - Funnels are important but not critical
  - Log full failure report with all attempted tools
  - Add URL to manual_review_queue.json
  - Continue with other landing pages
  - Note gap in competitive intelligence

ON captcha_detected:
  - Log URL and captcha type
  - Skip URL and continue with others
  - Do NOT attempt to solve captcha
  - Add to manual_review_queue.json

ON redirect_detected:
  - Follow redirect (same domain only)
  - Log original URL and final destination
  - Capture content at destination
```

### Raw Data Preservation (For Authenticity Validation)

**CRITICAL:** All scraped content must be saved to raw-data/ for authenticity verification.

```yaml
on_successful_scrape:
  1. Save full response to: raw-data/funnel_{advertiser}_{url_hash}_{timestamp}.json
  2. Generate SHA-256 hash of file
  3. Save hash to: raw-data/funnel_{advertiser}_{url_hash}_{timestamp}_hash.json
  4. Log in state manager: {file, hash, items_count, timestamp}

NEVER modify raw-data/ files after creation - they are the source of truth.
```

### Firecrawl Settings (Primary)

```yaml

wait_for: 3000  # ms - allow dynamic loading

  - text

  - structured_data

```

## Failure Protocol

```
IF page blocked/captcha:
  - Report which URL
  - Skip and continue with others
  - Note gap in output

IF dynamic content not loading:
  - Increase wait time
  - Try scroll-to-load
  - Capture what's available
  - Note incomplete

IF redirect to different page:
  - Note original URL and redirect
  - Capture destination content
  - Flag for review
```
