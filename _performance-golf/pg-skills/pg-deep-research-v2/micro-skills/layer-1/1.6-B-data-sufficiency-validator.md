# Micro-Skill 1.6-B: Data Sufficiency Validator

**Layer:** 1 - Infrastructure
**Purpose:** BLOCKING validation that extraction produced sufficient AUTHENTIC data before Layer 2 proceeds
**Dependency:** 1.5-A Content Extractor complete, 0.3-Authenticity-Validator complete
**Version:** 2.0 (Upgraded with Authenticity Integration)
**Type:** BLOCKING GATE - Cannot be bypassed

---

## CRITICAL: THIS IS A BLOCKING GATE

This skill exists because of a critical failure mode: **extraction can appear successful but produce insufficient or FABRICATED data**. This validator MUST:

1. **Actually count extracted quotes** (not trust reported numbers)
2. **Verify quotes contain real content** (not placeholders)
3. **VERIFY AUTHENTICITY** via 0.3-Authenticity-Validator (NEW)
4. **BLOCK Layer 2 if thresholds not met**
5. **Trigger re-scraping workflow if data insufficient**

```yaml
gate_type: BLOCKING
bypass_allowed: false
human_override: required_for_bypass
failure_action: HALT_AND_REMEDIATE
prerequisite: 0.3-Authenticity-Validator MUST complete first
```

## Validation Steps

### Step 1: File Existence Check

```
VERIFY extracted_quotes.json EXISTS
VERIFY file is not empty
VERIFY file is valid JSON
VERIFY file contains "quotes" array

IF any check fails:
  status = CRITICAL_FAIL
  HALT immediately
  Report: "Extraction output file missing or invalid"
```

### Step 2: Actual Quote Count

**DO NOT trust reported numbers. COUNT THE QUOTES.**

```python
# Pseudocode
actual_count = len(extracted_quotes["quotes"])
claimed_count = extraction_stats["totals"]["extracted_quotes"]

IF actual_count != claimed_count:
  FLAG: "Count mismatch - claimed {claimed}, found {actual}"
  SEVERITY: WARNING if within 10%, CRITICAL if >10% discrepancy

IF actual_count < 75:
  status = FAIL
  BLOCK Layer 2
  Trigger re-scraping workflow
```

### Step 3: Content Verification

**Verify quotes contain REAL content, not placeholders**

```python
# Pseudocode
verified_quotes = 0
empty_quotes = 0
placeholder_patterns = [
  "Lorem ipsum",
  "[placeholder]",
  "Sample quote",
  "Example text",
  "TODO",
  "..."
]

FOR each quote in extracted_quotes:
  IF quote.text is empty OR len(quote.text) < 10:
    empty_quotes += 1
    FLAG quote as "empty_or_too_short"
  ELSE IF any(pattern in quote.text for pattern in placeholder_patterns):
    empty_quotes += 1
    FLAG quote as "placeholder_detected"
  ELSE:
    verified_quotes += 1

real_data_rate = verified_quotes / total_quotes

IF real_data_rate < 0.90:
  status = FAIL
  Report: "Only {rate}% of quotes contain real content"
```

### Step 4: Random Sample Validation

**Actually read 10 random quotes and verify they're real**

```python
# Pseudocode
sample_size = min(10, total_quotes)
random_sample = random.sample(extracted_quotes, sample_size)

FOR each quote in random_sample:
  # Verify each has:
  CHECK: text field exists and has content
  CHECK: source.platform exists
  CHECK: source.url exists (can be empty for some platforms)
  CHECK: text appears to be natural language, not code/JSON

  LOG: "Sample {n}: '{quote.text[:50]}...' from {quote.source.platform}"

IF any sample fails basic checks:
  FLAG: "Sample validation failed"
  DETAIL: Show failed samples
```

### Step 5: Platform Distribution Check

```python
# Pseudocode
platform_counts = count_by_platform(extracted_quotes)

FOR platform, expected_yield in expected_yields.items():
  actual = platform_counts.get(platform, 0)

  IF actual == 0 AND expected_yield.min_items > 0:
    FLAG: "{platform} has 0 quotes but expected data"
    SEVERITY: WARNING

  IF actual < expected_yield.min_quotes_expected * 0.5:
    FLAG: "{platform} significantly underperformed"
    DETAIL: "Expected ~{expected}, got {actual}"

# Check diversity
platforms_with_data = len([p for p in platform_counts if platform_counts[p] > 0])

IF platforms_with_data < 3:
  status = FAIL
  Report: "Insufficient platform diversity ({platforms_with_data} platforms)"
```

### Step 6: Content Type Balance

```python
# Pseudocode
user_voice_count = count(quotes WHERE content_type == "user_voice")
competitive_intel_count = count(quotes WHERE content_type == "competitive_intel")

IF user_voice_count < 50:
  status = FAIL
  Report: "Insufficient user voice data ({user_voice_count} quotes)"
  Note: "Layer 2 pattern analysis requires minimum 50 user voices"
```

## Threshold Definitions

### BLOCKING Thresholds (Cannot proceed if not met)

| Metric | Minimum | Rationale |
|--------|---------|-----------|
| Total Verified Quotes | 75 | Statistical significance for patterns |
| User Voice Quotes | 50 | Core requirement for pattern analysis |
| Platforms with Data | 3 | Prevents platform bias |
| Real Data Rate | 90% | Ensures data integrity |
| High-Value Quotes (specifics) | 20 | Ensures copy ammunition exists |

### WARNING Thresholds (Proceed with caution)

| Metric | Warning Level | Note |
|--------|---------------|------|
| Total Quotes | 75-100 | Sufficient but not ideal |
| User Voice | 50-75 | Minimal for patterns |
| Any platform at 0 | Any | Indicates potential scraping failure |
| Real Data Rate | 90-95% | Some data quality issues |

---

## Pass/Fail Criteria

### PASS (All conditions met)
- Total verified quotes >= 75
- Real data rate >= 90%
- User voice quotes >= 50
- Platforms with data >= 3
- High-value quotes >= 20
- No CRITICAL flags

### WARNING PASS (Proceed with documented risks)
- Meets minimums but with warnings
- One platform at 0 but others compensate
- Real data rate 85-90%
- Document risks for Layer 2

### FAIL (BLOCKING - Cannot proceed)
- Total verified quotes < 75
- Real data rate < 85%
- User voice quotes < 50
- Platforms with data < 3
- Any CRITICAL flag raised

### CRITICAL FAIL (Immediate halt)
- Extraction file missing or invalid
- Count mismatch > 50%
- Real data rate < 50%
- Signs of fabricated data

## Failure Response Protocol

```yaml
on_fail:
  step_1: HALT pipeline immediately
  step_2: Generate detailed failure report
  step_3: Identify specific failure points
  step_4: Determine remediation path:

    if_volume_insufficient:
      - Review extraction criteria (too strict?)
      - Check raw data availability
      - Trigger re-scraping if raw data inadequate
      - Expand source list if needed

    if_quality_insufficient:
      - Check extraction implementation
      - Review platform-specific rules
      - Verify raw data was actually read

    if_platform_gaps:
      - Debug specific platform scraper
      - Check if Apify data was retrieved
      - Try alternative scraping approach

  step_5: DO NOT proceed to Layer 2
  step_6: Present failure report to human for decision

on_critical_fail:
  step_1: IMMEDIATE HALT
  step_2: Flag as data integrity issue
  step_3: Review extraction process for errors
  step_4: Require human review before any retry
```

## Integration with Pipeline

```

                                       ↓
                              BLOCKS if fails
                                       ↓
                              Re-scrape workflow
```

**Key Integration Points:**

1. **Reads from:** extracted_quotes.json (output of 1.5-A)
2. **Outputs to:** data_sufficiency_validation.json
3. **Gates:** Layer 1 Checkpoint cannot run until this PASSES
4. **Triggers:** Re-scraping workflow if FAILS

## Why This Skill Exists

**Root Cause:** Previous executions created fake output files without actually processing raw data. This led to:
- 91% data loss undetected until final handoff
- Invalid Layer 2 analysis based on non-existent data
- Wasted time and resources
- Critical failure in research quality

**This validator ensures:**
1. Extraction actually happened
2. Real quotes were extracted
3. Sufficient volume for analysis
4. No fake/placeholder data
5. Pipeline cannot proceed with bad data
