# Micro-Skill 1.5-A: Content Extractor

**Layer:** 1 - Infrastructure
**Purpose:** Extract clean, structured quotes from raw scraped content into PRD-defined buckets with mandatory ID assignment
**Dependency:** All 1.4 scrapers complete, Context Expansion (1.0-A), Authenticity Validator (0.3)
**Version:** 4.0 (Mandatory Quote ID System + Chunking Integration)
**PRD Reference:** [[RESEARCH-PRD#Section 3 - Quote Requirements]], [[RESEARCH-PRD#Section 3.5 - Numbered Quote System]]
**Core Infrastructure:** [[0.5-chunking-manager]] - Required for processing 1,000+ quotes

---

## CRITICAL: PRD-Aligned Bucket Extraction

This skill extracts quotes into the 5 PRD-defined buckets. Each quote must be categorized upon extraction and assigned a **mandatory unique ID**.

**Minimum Total Requirement:** 1,000+ quotes (PRD Minimum - NO CEILING)
**Bucket Minimums:** 150 quotes per bucket minimum

---

## MANDATORY: Quote ID System (PRD Section 3.5)

**Every quote MUST have a unique identifier following this format:**

```yaml
quote_id_format:
  PAIN: "P-001", "P-002", "P-003"...
  HOPE: "H-001", "H-002", "H-003"...
  ROOT_CAUSE: "RC-001", "RC-002", "RC-003"...
  SOLUTIONS_TRIED: "ST-001", "ST-002", "ST-003"...
  COMPETITOR_MECHANISMS: "CM-001", "CM-002", "CM-003"...
  VILLAIN: "V-001", "V-002", "V-003"...

id_requirements:
  - Sequential within bucket (P-001, P-002, not P-001, P-047)
  - Zero-padded to 3 digits minimum (P-001 not P-1)
  - Never reuse IDs, even if quote deleted
  - ID persists through entire pipeline to FINAL_HANDOFF

id_tracking:
  manifest_file: "layer-1-outputs/quote_id_manifest.json"
  tracks:
    - Last assigned ID per bucket
    - Total count per bucket
    - ID-to-source mapping for traceability
```

**Why IDs Matter:**
- Enables traceability from final handoff back to raw source
- Prevents quote duplication across documents
- Allows specific quote references in copywriting
- Required for authenticity validation (0.3)

## CRITICAL: Chunking Integration

**For datasets exceeding 100 quotes, this skill MUST use the chunking manager.**

```yaml
chunking_protocol:
  threshold: 100 quotes OR 50KB raw data

  when_triggered:
    1. Call 0.5-chunking-manager with operation="quote_extraction"
    2. Process quotes in batches of 50
    3. Assign IDs with offset per batch:
       - Batch 1: P-001 to P-050
       - Batch 2: P-051 to P-100
       - etc.
    4. Write batch outputs to layer-1-outputs/batches/
    5. Aggregate after all batches complete

  id_continuity:
    - Batch manifest tracks last_id_assigned per bucket
    - Next batch starts from last_id + 1
    - No ID gaps across batches
```

## Extraction Principles

### 1. Verbatim Preservation
- Extract EXACTLY as written
- Preserve typos, slang, capitalization
- Preserve emojis and formatting
- Do NOT clean, correct, or paraphrase

### 2. Bucket Assignment at Extraction
- Every quote MUST have a bucket
- Use classification signals above
- When ambiguous, assign PRIMARY bucket
- Add secondary_bucket field if applicable

### 3. Context Preservation
- Include source URL
- Include timestamp
- Include author info
- Include parent content (for replies)
- Include thread/post title

### 4. Meaningful Units
- Extract complete thoughts
- Don't truncate mid-sentence
- Include enough context to understand
- Don't merge separate comments

## Platform-Specific Extraction Rules

### Reddit (Posts & Comments)
```yaml
data_structure:
  post: { title, body, username, upVotes, numberOfComments }
  comment: { body, username, upVotes, parent_id }

extraction_criteria:
  posts: body > 50 chars AND relevance to context
  comments: body > 30 chars AND substantive content
  minimum_engagement: upVotes >= 1 OR meaningful reply

expected_yield_rate: 50-70%

what_to_extract:
  - Full post body
  - Full comment body
  - Thread title as context
  - Author, date, upvotes

skip_if:
  - Single word response
  - Just a link with no commentary
  - "[deleted]" or "[removed]"
  - Off-topic banter
```

### YouTube (Comments)
```yaml
data_structure:
  comment: { comment, author, voteCount, replyCount, videoId, title }

extraction_criteria:
  minimum_length: 20 characters
  relevance: Must reference product context
  engagement_bonus: voteCount >= 3 OR replyCount >= 1

expected_yield_rate: 40-60%

what_to_extract:
  - Full comment text
  - Video title as context
  - Vote count as engagement signal

skip_if:
  - Just emojis
  - "First" or "Early" comments
  - Spam/promotion
  - Off-topic
  - Generic praise only
```

### Forum Content (GolfWRX, etc.)
```yaml
data_structure:
  post: { title, body, author, date, thread_url }

extraction_criteria:
  minimum_length: 40 characters
  relevance: Technical discussion or experience sharing

expected_yield_rate: 70-90%

what_to_extract:
  - Full post content
  - Thread context
  - Author credentials if shown
  - Technical specifics

skip_if:
  - Just "+1" or "agree"
  - Off-topic tangents
  - Administrative posts
```

### Competitor Pages & Ads
```yaml
data_structure:
  page: { product_name, description, features, technology_claims }
  ad: { headline, body, cta, brand }

extraction_criteria:
  all_claims: Extract ALL mechanism and benefit claims
  technology_details: Extract HOW and WHY explanations

expected_yield_rate: 90-95%

what_to_extract:
  - Technology names and descriptions
  - How-it-works explanations
  - Benefit claims
  - Positioning statements
  - Verbatim marketing copy

skip_if:
  - Generic navigation text
  - Legal disclaimers (unless claim-relevant)
  - Duplicate content
```

### Amazon Reviews
```yaml
data_structure:
  review: { product, title, description, rating, date, verified }

extraction_criteria:
  minimum_length: 40 characters in description
  priority: Verified reviews > Non-verified
  include_all_ratings: Negative reviews often most insightful

expected_yield_rate: 60-80%

what_to_extract:
  - Full review description
  - Review title
  - Rating
  - Verified status
  - Product name

skip_if:
  - "Gift for someone" with no opinion
  - "Five stars" with no explanation
  - Shipping/packaging complaints only
```

## Dynamic Expansion Trigger

Per PRD requirements, if bucket minimums not met:

```yaml
expansion_trigger:
  condition: ANY bucket < 150 quotes
  action:
    1. Identify underperforming bucket
    2. Generate additional queries for that bucket's topics
    3. Scrape additional sources
    4. Re-extract and merge
    5. Continue until minimum met

  max_expansion_iterations: 3

  if_still_under_minimum:
    - Flag bucket with detailed gap analysis
    - Document what sources were exhausted
    - Proceed with documented limitation
```

## Extraction Process

### Step 1: Data Audit (MANDATORY)
Before extraction:
1. List all data files
2. Read sample from each file
3. Document actual data structure
4. Create platform-specific extraction plan

### Step 2: Content Parsing
For each raw content item:
1. Parse into individual units
2. Identify boundaries between units
3. Preserve hierarchy (reply-to relationships)

### Step 3: Bucket Classification
For each content unit:
1. Apply classification signals
2. Assign primary bucket
3. Assign subcategory within bucket
4. Note secondary bucket if applicable

### Step 4: Platform-Specific Filtering
Apply platform-specific rules:
1. Check minimum length
2. Check relevance criteria
3. Check engagement thresholds
4. Apply skip rules

### Step 5: Quote Extraction
For relevant units:
1. Extract full text verbatim
2. Attach all metadata
3. Assign unique quote_id
4. Note context

### Step 6: Specificity Flagging
Flag quotes containing:
- Dollar amounts
- Product names
- Quantities
- Time periods
- Personal stories
- High emotional intensity

### Step 7: Deduplication
Remove duplicates:
```
IF quote.text == existing_quote.text:
  DISCARD (keep first)

IF similarity > 0.85:
  KEEP higher engagement version
```

### Step 8: Bucket Tally & Validation
For each bucket:
1. Count total quotes
2. Compare to minimum (150)
3. If under minimum, trigger expansion
4. If over minimum, continue

### Step 9: Platform Yield Report
Calculate for each platform:
- Raw items processed
- Quotes extracted
- Yield rate
- Bucket distribution

## Failure Protocol

```
IF total quotes < 1,000:
  - DO NOT PROCEED to Layer 2
  - Identify source gaps
  - Trigger expansion scraping
  - Re-extract from new sources
  - Continue until minimum met

IF any bucket < 150:
  - Identify which bucket
  - Generate targeted queries
  - Scrape bucket-specific sources
  - Re-extract and merge
  - Continue until bucket minimum met

IF parsing fails:
  - Log which content item
  - Note error type
  - Skip and continue
  - Report gaps

IF bucket classification unclear:
  - Assign best-fit bucket
  - Add "classification_confidence: low"
  - Continue extraction
```
