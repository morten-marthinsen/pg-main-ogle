## Scraping Protocol

### Depth Levels

**Full (Tier 1 sources):**
- Scrape 100+ threads per forum
- Include ALL replies (not just top replies)
- Go back 12 months minimum
- Continue until saturation (same patterns 3+ times)

**Standard (Tier 2 sources):**
- Scrape 50-100 threads per forum
- Include all replies
- Go back 6 months minimum

**Light (Tier 3 sources):**
- Scrape 20-50 threads per forum
- Include top-voted replies
- Go back 3 months minimum

### Saturation Definition

Saturation reached when:
- Same complaint/desire pattern appears 3+ times from different users
- Same question asked repeatedly with similar answers
- Same debate/argument pattern emerges repeatedly

**Do NOT stop at arbitrary number if saturation not reached.**

## Scraping Process

### Step 1: Source Preparation
For each approved forum source:
1. Identify target subforums/sections
2. Build scrape queue (threads to scrape)
3. Prioritize by relevance and recency

### Step 2: Thread Discovery
```
Method 1: Browse recent threads
Method 2: Search for product-related terms
Method 3: Follow links from high-engagement threads
```

### Step 3: Thread Scraping
For each thread:
1. Scrape full thread content
2. Capture all replies (paginate if needed)
3. Extract metadata
4. Store with source attribution

### Step 4: Saturation Monitoring
During scraping:
- Track emerging patterns
- Note when patterns repeat
- Continue until saturation OR depth limit reached

### Step 5: Error Handling (Via Tool Chain)

**The tool chain system (0.2-tool-resilience) handles most errors automatically.**

```
ON tool_chain_success:
  - Continue processing
  - Log which tool succeeded (for monitoring)

ON tool_chain_fallback_used:
  - Log: primary tool failed, fallback succeeded
  - Continue processing
  - No human intervention needed

ON tool_chain_complete_failure:
  - All tools in chain failed for this URL
  - Log full failure report with all attempted tools
  - Add URL to manual_review_queue.json
  - Continue with other sources (do NOT halt entire scrape)
  - Report at end: "X URLs could not be scraped after all fallbacks"

ON rate_limit_detected:
  - Tool chain handles automatic backoff
  - If persistent (>3 rate limits in 5 minutes): Pause scraping for 5 minutes
  - Report slowdown in progress update

ON budget_threshold_reached:
  - Pause scraping
  - Report: "Estimated cost approaching budget limit"
  - Await human decision: Continue, stop, or adjust
```

## Failure Protocol

```
IF scraping fails for any source:
  1. Log exact error
  2. Report:
     - Which source failed
     - How many items collected before failure
     - What the error was
  3. HALT - do not continue automatically
  4. Wait for human decision:
     - Skip this source
     - Retry with different approach
     - Abort scraping phase
```
