# 2.5-D Transformation Grid Generator

**Version:** 1.0
**Created:** January 20, 2026
**Layer:** 2.5 - Synthesis
**Purpose:** Generate Now→After Grid mapping prospect transformation across 4 dimensions

---

## Purpose

This skill creates the **Now→After Grid** - a 4-dimension transformation matrix showing the prospect's journey from current state to desired state. This grid is critical for:
1. Positioning framework
2. Before/after messaging
3. Transformation promise articulation
4. Copy proof structure

## The 4 Dimensions

### HAVE (Tangible)
What they physically possess or measurably experience:
- **NOW:** What tangible outcomes they currently suffer
- **AFTER:** What tangible outcomes they'll gain

Examples:
- NOW: "33% fairways" → AFTER: "80% fairways"
- NOW: "Wasted strokes" → AFTER: "Saved strokes"

### EXPERIENCE (Emotional Process)
What they go through during the activity:
- **NOW:** The emotional process during driver shots
- **AFTER:** The transformed emotional experience

Examples:
- NOW: "Dreading" → AFTER: "Enjoying"
- NOW: "Tensing up" → AFTER: "Swinging freely"

### STATUS (External Perception)
How others perceive them:
- **NOW:** Their current reputation/label
- **AFTER:** Their transformed reputation/label

Examples:
- NOW: "Military golf guy" → AFTER: "Fairway finder"
- NOW: "Holds up the group" → AFTER: "Helps others find their ball"

### FEELING (Internal State)
Their internal emotional state:
- **NOW:** How they feel about themselves
- **AFTER:** How they'll feel after transformation

Examples:
- NOW: "Embarrassed" → AFTER: "Proud"
- NOW: "Defeated" → AFTER: "In control"

---

## Validation Rules

### Structural Validation
- [ ] All 4 dimensions present (HAVE, EXPERIENCE, STATUS, FEELING)
- [ ] Each dimension has NOW statement
- [ ] Each dimension has AFTER statement
- [ ] Each dimension has evidence (2+ quotes)

### Content Validation
- [ ] NOW statements reflect actual PAIN quotes
- [ ] AFTER statements reflect actual HOPE quotes
- [ ] Evidence quotes support the statements
- [ ] Transformation is believable and product-connected

### Output Validation
- [ ] one_line_transformation present
- [ ] short_transformation present
- [ ] copy_ready_versions present

## Failure Handling

If any dimension incomplete:
1. Log: "Now→After Grid incomplete: {dimension} missing {NOW/AFTER}"
2. Review source quotes for missing dimension
3. Pull from transformation_pairs.json if needed
4. DO NOT proceed to Gate 2.5 until all dimensions complete
