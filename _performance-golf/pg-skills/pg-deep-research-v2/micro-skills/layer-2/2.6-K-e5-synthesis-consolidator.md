## CRITICAL: E5 Methodology Synthesis

The E5 methodology (Todd Brown) has three examination pillars:
1. **Prospect Examination** - Who they are, what they want, where they are
2. **Competitor Examination** - What exists, what to avoid, what's hated
3. **Product Examination** - How to position benefits, handle objections

This skill **CONSOLIDATES** all E5 component outputs into a unified strategic framework.

## Synthesis Process

### Step 1: Validate All E5 Component Inputs

```yaml
input_validation:
  prospect_examination_components:
    - CHECK: awareness_pyramid exists and has distribution
    - CHECK: web_analysis has all 3 sections (W, E, B)
    - CHECK: now_after_grid has all 4 categories
    - CHECK: ideal_client_outcome has all 3 sections
    - CHECK: magic_wand has blue_sky + calibration + boundaries
    - CHECK: promise_exposure has stage diagnosis

  competitor_examination_components:
    - CHECK: mechanism_mapping exists with exclusion flags
    - CHECK: offer_analysis has price tiers + guarantees
    - CHECK: villain_inventory has all 3 hate categories

  product_examination_components:
    - CHECK: benefit_patterns available for dimensionalization
    - CHECK: objection_patterns available for CPT matrix

  on_missing_component:
    - LOG which component is missing
    - FLAG as "E5 incomplete - missing [component]"
    - ATTEMPT to derive from available data
    - If cannot derive: HALT and report gap
```

### Step 2: Synthesize Prospect Examination

```yaml
prospect_synthesis:
  step_1_awareness:
    - Diagnose primary awareness level from quote analysis
    - Calculate distribution percentages from source types
    - Determine strategic implications (lead strategy, copy length, differentiation)

  step_2_web:
    - Consolidate WANTS from hope bucket + magic wand
    - Consolidate EMOTIONS from pain bucket + sentiment analysis
    - Consolidate BELIEFS from root cause bucket + belief excavator

  step_3_transformation:
    - Map NOW state from pain patterns
    - Map AFTER state from hope patterns
    - Ensure all 4 dimensions covered (HAVE, EXPERIENCE, STATUS, FEELING)

  step_4_outcome:
    - Extract primary results from highest-frequency hope patterns
    - Extract secondary transformations from identity/status shifts
    - Build testimonial framework from before/after quotes

  step_5_magic_wand:
    - Identify blue-sky desires from extreme hope quotes
    - Calibrate against reality (what's achievable)
    - Set promise boundaries (what we should/shouldn't claim)

  step_6_promise_exposure:
    - Diagnose market sophistication stage from promise fatigue signals
    - Determine appropriate lead strategy
    - Document evidence for stage diagnosis
```

### Step 3: Synthesize Competitor Examination

```yaml
competitor_synthesis:
  step_1_mechanism_registry:
    - Pull all mechanisms from 2.4-C
    - Flag each for exclusion (avoid using their names/articulations)
    - Document white space for new mechanism creation

  step_2_offer_landscape:
    - Map price tiers from competitor offer analysis
    - Identify guarantee gaps
    - Document SIN opportunities (Superior Incontestable Narrative)

  step_3_villain_inventory:
    - Consolidate hated features from villain identifier
    - Consolidate hated products
    - Consolidate hated messaging patterns
    - Calculate contrast opportunities for each
```

### Step 4: Synthesize Product Examination

```yaml
product_synthesis:
  step_1_benefit_dimensionalization:
    FOR EACH product feature:
      - State functional benefit (what it does)
      - Dimensionalize (specific scenario/application)
      - Identify emotional benefit (how it makes them feel)
      - Link to evidence quotes

  step_2_objection_matrix:
    FOR EACH strategic opportunity:
      - Identify likely objection
      - Build CPT response:
        - Claim (what we assert)
        - Proof (evidence backing claim)
        - Transition (bridge to benefit)
      - Link to evidence quotes
```

### Step 5: Generate Strategic Synthesis

```yaml
strategic_synthesis:
  cross_component_analysis:
    - Identify consistencies across all 3 examinations
    - Flag any contradictions for human review
    - Extract dominant patterns

  synthesis_elements:
    - PRIMARY TARGET: From awareness pyramid primary segment
    - MARKET STAGE: From promise exposure analysis
    - POSITIONING OPPORTUNITY: From mechanism exclusion + white space
    - VILLAIN CONTRAST: From villain inventory + contrast opportunities
    - TRANSFORMATION PROMISE: From now-after grid
    - PROOF STRATEGY: From awareness level + belief patterns
```

## Quality Standards

```yaml
output_requirements:
  completeness:
    - ALL 6 prospect examination sections present
    - ALL 3 competitor examination sections present
    - ALL 2 product examination sections present
    - Strategic synthesis with all 6 elements

  traceability:
    - Every insight links to quote IDs
    - Every percentage cites calculation source
    - Every claim has evidence reference

  confidence_requirements:
    - Overall confidence calculated per PRD Rule 5
    - Each examination pillar has sub-confidence
    - Any section <50% confidence flagged for review

  consistency:
    - No contradictions between prospect and competitor insights
    - Awareness level aligns with promise exposure stage
    - Villain inventory aligns with pain patterns
```

## Error Handling

```yaml
error_handling:
  missing_input:
    condition: Required E5 component not found
    action:
      - Log missing component
      - Attempt to derive from related data
      - If cannot derive: Create placeholder with "MISSING" flag
      - Flag for human review
    flag: "E5 incomplete - [component] missing or derived"

  conflicting_insights:
    condition: Component A says X, Component B says not-X
    action:
      - Log both insights with sources
      - Weight by evidence count
      - Flag for human review
    flag: "E5 conflict - [insight A] vs [insight B]"

  low_evidence:
    condition: Section has <10 supporting quotes
    action:
      - Flag section as "low confidence"
      - Note evidence gap
      - Continue with caveat
    flag: "Low evidence base for [section]"
```
