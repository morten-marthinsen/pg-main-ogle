# Micro-Skill 2.6-D: Transformation Mapper

**Layer:** 2 - Intelligence
**Purpose:** Map before/after states and transformation arcs
**Dependency:** Layer 1 desire/pain data
**Parallel:** Can run simultaneously with 2.6-A through 2.6-C, 2.6-E, 2.6-F

---


## What We're Mapping

### Before State
How people describe their current situation:
- Current pain/frustration
- Current identity
- Current limitations
- Current feelings

### After State
What people want to become/experience:
- Desired outcome
- Aspirational identity
- Removed limitations
- Target feelings

### Transformation Arc
The journey from before to after:
- What changes?
- How does it feel?
- What's the story?

---

## Transformation Types

### Functional Transformation
Changes in capability/outcome:
- "Can't hit consistent distances" → "Know exactly how far I hit"
- "Lose lots of balls" → "Play confidently"

### Emotional Transformation
Changes in feeling:
- "Anxious over shots" → "Confident and free"
- "Frustrated by wasting money" → "Satisfied with smart choice"

### Identity Transformation
Changes in who they are:
- "Sucker paying premium" → "Smart golfer"
- "Not good enough" → "Worthy of good equipment"

### Social Transformation
Changes in perception:
- "Embarrassed about ball choice" → "Respected for smart choice"
- "Out of place with premium players" → "Confident in any group"

---

## Detection Process

### Step 1: Before State Mining
Search for current state language:
- "I currently..."
- "Right now I..."
- "My problem is..."
- "I struggle with..."
- "I hate that..."

### Step 2: After State Mining
Search for desired state language:
- "I want to..."
- "I wish I could..."
- "If only..."
- "I'd love to..."
- "Imagine if..."

### Step 3: Transformation Story Mining
Search for completed transformations:
- "I used to... now I..."
- "Since I switched..."
- "After I changed..."
- "Finally..."
- "Now I can..."

### Step 4: Arc Construction
For each before/after pair:
1. What's the transformation?
2. What enables it?
3. What's the emotional journey?

### Step 5: Product Connection
How does the product enable transformation?
- Direct enabler
- Catalyst
- Symbol of change

## Story Arc Template

```
Before: [Current painful state]
    ↓
Catalyst: [What triggered change]
    ↓
Transformation: [What shifted]
    ↓
After: [New desired state]
    ↓
Evidence: [Proof of transformation]
```

- Real stories from data are more powerful than invented ones
- Multiple transformation types can be layered
- Primary transformation should drive main messaging
- Output feeds Layer 3 opportunity synthesis
