# Core Skill 0.1: State Manager

**Layer:** 0 - Core Infrastructure
**Purpose:** Persistent state management, checkpointing, and session recovery
**Dependency:** None - This is the foundation
**Version:** 1.0
**Type:** ALWAYS ACTIVE - Runs throughout entire pipeline

---

## CRITICAL: State Management Prevents Data Loss

This skill exists because:
1. Long research runs (hours) will experience interruptions
2. Token exhaustion mid-process should not lose progress
3. Tool failures should allow graceful recovery
4. Human review points need resumable state

**This skill runs AUTOMATICALLY at every skill transition.**

## Checkpoint Protocol

### When to Create Checkpoints

```yaml
checkpoint_triggers:
  automatic:
    - After every skill completion
    - Every 15 minutes during long-running skills
    - Before any external API call batch
    - Before expansion loop iteration

  explicit:
    - Before human review points
    - Before destructive operations
    - On user request ("/checkpoint")
```

### Checkpoint Creation Process

```
CHECKPOINT_CREATE(trigger_reason):
  1. Generate checkpoint_id: "cp_{incremental_number}"

  2. Capture current state:
     - Copy current_state.json
     - Record all output file hashes
     - Record partial progress of in-flight skills

  3. Write checkpoint file:
     - checkpoints/checkpoint_{id}.json

  4. Update checkpoint_history.log:
     - Append: {id, timestamp, trigger_reason, state_summary}

  5. Prune old checkpoints:
     - Keep last 10 checkpoints
     - Always keep: first, last human-approved, last successful

  6. Update current_state.json:
     - last_checkpoint_id = new id
     - last_checkpoint_at = now

  RETURN checkpoint_id
```

## Session Recovery Protocol

### On Session Start

```
SESSION_START():
  1. Check for existing project state:
     IF projects/{project}/checkpoints/current_state.json EXISTS:

       state = READ(current_state.json)

       IF state.execution.status == "completed":
         PROMPT: "Previous research completed. Start new research or review results?"

       ELSE IF state.execution.status IN ["running", "paused", "failed"]:
         PROMPT: "Found interrupted session from {last_activity}. Options:
                  1. Resume from last checkpoint (cp_{last_checkpoint_id})
                  2. Resume from earlier checkpoint (show list)
                  3. Restart from beginning (WARNING: loses progress)
                  4. Review current state before deciding"

         AWAIT human_choice

         IF choice == "resume_last":
           EXECUTE recovery_protocol(last_checkpoint_id)
         ELSE IF choice == "resume_earlier":
           SHOW checkpoint_list
           AWAIT checkpoint_selection
           EXECUTE recovery_protocol(selected_checkpoint)
         ELSE IF choice == "restart":
           CONFIRM: "This will delete all progress. Type 'CONFIRM' to proceed"
           IF confirmed: EXECUTE fresh_start()
         ELSE:
           SHOW current_state_summary()
           AWAIT decision

     ELSE:
       EXECUTE fresh_start()
```

### Recovery Protocol

```
RECOVERY_PROTOCOL(checkpoint_id):
  1. Load checkpoint:
     checkpoint = READ(checkpoints/checkpoint_{id}.json)

  2. Verify file integrity:
     FOR each file in checkpoint.file_manifest:
       IF file.exists:
         current_hash = HASH(file.path)
         IF current_hash != file.hash:
           FLAG: "File modified since checkpoint: {file.path}"
           PROMPT: "Use current file or restore from backup?"

  3. Restore state:
     WRITE(current_state.json, checkpoint.state_snapshot)
     UPDATE current_state.execution.session_id = NEW_SESSION_ID
     UPDATE current_state.execution.status = "running"

  4. Identify resume point:
     resume_skill = checkpoint.recovery_instructions.resume_from_skill
     required_inputs = checkpoint.recovery_instructions.required_inputs

  5. Validate inputs available:
     FOR each input in required_inputs:
       IF NOT EXISTS(input):
         ERROR: "Cannot resume - missing required input: {input}"
         SUGGEST: "Resume from earlier checkpoint or restart"

  6. Log recovery:
     APPEND execution_log: {
       event: "session_recovered",
       from_checkpoint: checkpoint_id,
       resume_skill: resume_skill,
       timestamp: NOW
     }

  7. Resume execution:
     NOTIFY: "Resuming from skill {resume_skill}"
     RETURN resume_skill
```

## State Update Protocol

### On Skill Start

```
ON_SKILL_START(skill_id):
  1. Validate dependencies:
     FOR each dep in dependency_graph[skill_id]:
       IF outputs[dep].status != "complete":
         ERROR: "Cannot start {skill_id} - dependency {dep} not complete"
       IF NOT FILE_EXISTS(outputs[dep].primary_output):
         ERROR: "Cannot start {skill_id} - output file missing for {dep}"

  2. Update state:
     current_state.execution.current_skill = skill_id
     current_state.progress.skills_in_progress.ADD({
       skill_id: skill_id,
       started_at: NOW,
       progress: "0%",
       partial_output: null
     })

  3. Log:
     APPEND execution_log: {event: "skill_started", skill_id, inputs}

  4. Save state:
     WRITE current_state.json
```

### On Skill Complete

```
ON_SKILL_COMPLETE(skill_id, output_file):
  1. Validate output:
     IF NOT FILE_EXISTS(output_file):
       ERROR: "Skill reported complete but output file missing"
     output_hash = HASH(output_file)

  2. Update state:
     REMOVE skill_id FROM skills_in_progress
     ADD to skills_completed: {
       skill_id,
       completed_at: NOW,
       output_file,
       output_hash,
       validation_status: "pending"
     }
     current_state.execution.current_skill = null

  3. Update manifest:
     outputs[skill_id] = {
       status: "complete",
       primary_output: output_file,
       output_hash: output_hash,
       created_at: NOW
     }

  4. Create checkpoint:
     CHECKPOINT_CREATE("skill_completion:" + skill_id)

  5. Log:
     APPEND execution_log: {event: "skill_completed", skill_id, output_file}

  6. Identify next skill:
     next_skills = SKILLS_WITH_DEPENDENCIES_MET()
     RETURN next_skills
```

### On Skill Failure

```
ON_SKILL_FAILURE(skill_id, error):
  1. Log error:
     APPEND error_log: {
       skill_id,
       error_type: error.type,
       error_message: error.message,
       timestamp: NOW,
       context: CURRENT_CONTEXT
     }

  2. Attempt recovery:
     IF error.recoverable:
       recovery_result = ATTEMPT_RECOVERY(error)
       IF recovery_result.success:
         LOG: "Recovered from error, continuing"
         RETURN "continue"

  3. Update state:
     current_state.execution.status = "failed"
     current_state.metrics.errors_encountered += 1

  4. Create checkpoint:
     CHECKPOINT_CREATE("skill_failure:" + skill_id)

  5. Determine action:
     IF error.type == "tool_failure":
       RETURN "try_fallback"
     ELSE IF error.type == "data_validation":
       RETURN "human_review"
     ELSE:
       RETURN "halt"
```

## Progress Tracking

### Real-Time Progress Updates

For long-running skills, update progress periodically:

```
UPDATE_PROGRESS(skill_id, progress_info):
  1. Update in-progress entry:
     skills_in_progress[skill_id].progress = progress_info.message
     skills_in_progress[skill_id].partial_output = progress_info.partial_file

  2. Update metrics:
     IF progress_info.quotes_added:
       metrics.total_quotes_collected += progress_info.quotes_added
     IF progress_info.sources_scraped:
       metrics.sources_scraped += progress_info.sources_scraped

  3. Check for checkpoint trigger:
     IF TIME_SINCE_LAST_CHECKPOINT > 15 minutes:
       CHECKPOINT_CREATE("periodic:15min")

  4. Write state:
     WRITE current_state.json
```

## Failure Protocol

```
IF state file corrupted:
  - Attempt to load most recent checkpoint
  - If all checkpoints corrupted, require human decision
  - Log corruption event for debugging

IF checkpoint restore fails:
  - Try earlier checkpoint
  - If all fail, offer fresh start with data preservation option

IF dependency validation fails:
  - Identify missing dependency
  - Check if dependency output file exists but manifest outdated
  - If file exists, update manifest and continue
  - If file missing, re-run dependency skill
```
