# Core Skill 0.2: Tool Resilience & Fallback Chains

**Layer:** 0 - Core Infrastructure
**Purpose:** Provide tool fallback chains to prevent single-point failures
**Dependency:** None - This is infrastructure
**Version:** 1.0
**Type:** ALWAYS ACTIVE - Referenced by all skills that use external tools

---

## CRITICAL: No Single Point of Failure

External tools (Firecrawl, Apify, Exa) will fail. This is not a possibility - it's a certainty.

**Failure Modes:**
- Rate limiting (429 errors)
- Service outages
- IP blocks
- Timeout exhaustion
- Quota exceeded
- Authentication failures

**This skill ensures:** When primary tool fails, system automatically tries fallbacks.

### Chain: FORUM_SCRAPING

```yaml
chain_id: FORUM_SCRAPING
description: "Scraping forum threads and replies"
use_case: "GolfWRX, TheSandTrap, generic forums"

  primary:
    tool: mcp__firecrawl-mcp__firecrawl_scrape
    config:
      formats: ["markdown"]
      onlyMainContent: false
      waitFor: 2000
    timeout_ms: 30000
    max_retries: 2

  fallback_1:
    tool: mcp__apify__call-actor
    actor: "apify/web-scraper"
    trigger_on: ["rate_limit", "timeout", "blocked"]
    config:
      startUrls: ["{url}"]
      maxPagesPerCrawl: 50
      pageFunction: "extract forum content"
    timeout_ms: 60000

  fallback_2:
    tool: mcp__apify__apify-slash-rag-web-browser
    trigger_on: ["actor_failed", "no_results"]
    config:
      query: "{url}"
      maxResults: 1
      outputFormats: ["markdown"]
    timeout_ms: 30000

  fallback_3:
    tool: WebFetch
    trigger_on: ["all_above_failed"]
    config:
      url: "{url}"
      prompt: "Extract all forum posts and replies from this page"
    timeout_ms: 15000

  final_fallback:
    action: "log_and_skip"
    message: "All tools failed for {url}, logging for manual review"
```

## Fallback Execution Protocol

### Execute Chain

```python
def execute_chain(chain_id: str, params: dict) -> Result:
    """
    Execute a tool chain with automatic fallbacks.

    Args:
        chain_id: Which chain to use (e.g., "REDDIT_SCRAPING")
        params: Parameters like url, search_terms, etc.

    Returns:
        Result with data or final failure
    """
    chain = CHAINS[chain_id]
    tools_tried = []
    last_error = None

    # Try primary tool
    result = try_tool(
        chain.primary,
        params,
        log_as="primary"
    )

    if result.success:
        return result

    tools_tried.append({
        "tool": chain.primary.tool,
        "error": result.error,
        "timestamp": now()
    })
    last_error = result.error

    # Try fallbacks in order
    for i, fallback in enumerate(chain.fallbacks):
        # Check if this fallback's trigger matches our error
        if not error_matches_trigger(last_error, fallback.trigger_on):
            continue

        log(f"Primary failed with {last_error.type}, trying fallback_{i+1}")

        result = try_tool(
            fallback,
            params,
            log_as=f"fallback_{i+1}"
        )

        if result.success:
            # Log that we used a fallback (for monitoring)
            log_fallback_used(chain_id, fallback.tool, i+1)
            return result

        tools_tried.append({
            "tool": fallback.tool,
            "error": result.error,
            "timestamp": now()
        })
        last_error = result.error

    # All tools failed - execute final fallback action
    return execute_final_fallback(
        chain.final_fallback,
        tools_tried,
        params
    )
```

### Try Tool

```python
def try_tool(tool_config: dict, params: dict, log_as: str) -> Result:
    """
    Attempt to use a single tool with timeout and retry.
    """
    for attempt in range(tool_config.get('max_retries', 1) + 1):
        try:
            # Substitute parameters into config
            config = substitute_params(tool_config.config, params)

            # Execute with timeout
            result = execute_with_timeout(
                tool_config.tool,
                config,
                timeout_ms=tool_config.timeout_ms
            )

            # Validate result has data
            if result and has_meaningful_data(result):
                return Result(success=True, data=result)
            else:
                return Result(
                    success=False,
                    error=Error(type="no_data", message="Tool returned empty/invalid data")
                )

        except RateLimitError as e:
            if attempt < tool_config.max_retries:
                wait_time = e.retry_after or (30 * (attempt + 1))
                log(f"Rate limited, waiting {wait_time}s before retry")
                sleep(wait_time)
                continue
            return Result(success=False, error=Error(type="rate_limit", message=str(e)))

        except TimeoutError as e:
            return Result(success=False, error=Error(type="timeout", message=str(e)))

        except BlockedError as e:
            return Result(success=False, error=Error(type="blocked", message=str(e)))

        except Exception as e:
            return Result(success=False, error=Error(type="unknown", message=str(e)))

    return Result(success=False, error=Error(type="retries_exhausted", message="Max retries reached"))
```

### Error Matching

```python
def error_matches_trigger(error: Error, triggers: list) -> bool:
    """
    Check if error type matches any of the fallback triggers.
    """
    if "all_above_failed" in triggers:
        return True

    trigger_mapping = {
        "rate_limit": ["rate_limit", "429", "too_many_requests"],
        "timeout": ["timeout", "deadline_exceeded"],
        "blocked": ["blocked", "403", "access_denied", "captcha"],
        "quota_exceeded": ["quota", "limit_exceeded", "billing"],
        "actor_failed": ["actor_failed", "apify_error", "run_failed"],
        "no_results": ["no_data", "empty_response", "no_results"],
    }

    for trigger in triggers:
        if trigger in trigger_mapping:
            if error.type in trigger_mapping[trigger]:
                return True
        elif error.type == trigger:
            return True

    return False
```

## Platform-Specific Tool Selection

### Automatic Chain Selection

```python
def select_chain_for_url(url: str) -> str:
    """
    Automatically select the appropriate chain based on URL.
    """
    domain = extract_domain(url)

    chain_mapping = {
        # Reddit
        "reddit.com": "REDDIT_SCRAPING",
        "old.reddit.com": "REDDIT_SCRAPING",

        # YouTube
        "youtube.com": "YOUTUBE_SCRAPING",
        "youtu.be": "YOUTUBE_SCRAPING",

        # Amazon
        "amazon.com": "AMAZON_REVIEWS",
        "amazon.co.uk": "AMAZON_REVIEWS",

        # Social Media
        "instagram.com": "SOCIAL_MEDIA",
        "tiktok.com": "SOCIAL_MEDIA",
        "facebook.com": "SOCIAL_MEDIA",

        # Forums (known)
        "golfwrx.com": "FORUM_SCRAPING",
        "thesandtrap.com": "FORUM_SCRAPING",
        "mygolfspy.com": "FORUM_SCRAPING",

        # Competitor sites
        "clevelandgolf.com": "COMPETITOR_PAGES",
        "callawaygolf.com": "COMPETITOR_PAGES",
        "titleist.com": "COMPETITOR_PAGES",
        "taylormadegolf.com": "COMPETITOR_PAGES",
    }

    # Check for exact match
    if domain in chain_mapping:
        return chain_mapping[domain]

    # Check for partial match
    for pattern, chain in chain_mapping.items():
        if pattern in domain:
            return chain

    # Default to forum scraping (most versatile)
    return "FORUM_SCRAPING"
```

## Integration with Scrapers

### How 1.4-* Skills Use This

Each scraper skill should call the chain executor instead of tools directly:

```python
# OLD (single point of failure)
result = mcp__firecrawl-mcp__firecrawl_scrape(url=url, formats=["markdown"])

# NEW (with fallback chain)
result = execute_chain(
    chain_id=select_chain_for_url(url),
    params={
        "url": url,
        "search_terms": context.product_name
    }
)
```

### Required Updates to Scrapers

All 1.4-* skills need this update:

```yaml
before:
  tool: Firecrawl
  # No fallback

after:
  tool_chain: "{appropriate_chain}"
  chain_params:
    url: "{url}"
    search_terms: "{from context}"
  on_chain_failure:
    - Log full failure report
    - Add to manual_review_queue
    - Continue with other sources
```
