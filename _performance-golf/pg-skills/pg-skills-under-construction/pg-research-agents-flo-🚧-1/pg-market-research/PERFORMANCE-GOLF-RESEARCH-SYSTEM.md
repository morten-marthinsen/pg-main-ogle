# ⚠️ CRITICAL: ULTRA RICH IS NOW EMBEDDED

**This system now includes the Ultra Rich anti-mediocrity protocol at EVERY stage.**

The difference between B-list and A-list research output is where all the value lives. Ultra Rich is no longer a "final audit" — it's woven throughout the entire pipeline.

## What This Means for Execution

1. **Before Agent 1**: Run Pre-Task Interrogation
2. **At Every Agent**: Apply Satisficing Check before declaring complete
3. **At Every Gate**: Verify outputs LAND (not just exist)
4. **Agent 8**: Full Ultra Rich Post-Task Verification required
5. **After Gate 8**: Run Impact Audit on entire package

## The Core Ultra Rich Question

> "Am I producing something that will make the user think 'Claude really delivered' or just 'Claude did what I asked'?"

## Key Concepts

| Concept | Definition | Action |
|---------|------------|--------|
| **Hollow Technique** | Element that is PRESENT but doesn't LAND | Identify and fix before proceeding |
| **A-list Output** | Specific, surprising, integrated, impactful | The standard |
| **B-list Output** | Competent but unremarkable, technically complete but experientially empty | Not acceptable |
| **Satisficing** | Accepting "good enough" when "excellent" is possible | The enemy |
