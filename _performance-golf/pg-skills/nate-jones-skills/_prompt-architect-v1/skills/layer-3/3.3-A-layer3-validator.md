# 3.3-A: Layer 3 Validator
*Self-audit the rewritten skill to verify improvement*

---

## PURPOSE

Run Layer 1 diagnostics on the REWRITTEN skill to verify that optimization actually improved quality. This prevents the rare but catastrophic case where prescriptions make a skill worse. The self-audit must show measurable improvement or flag regression.

**Scope:** Re-run all Layer 1 checks on the new version. Compare scores. Verify all critical dimensions pass. Flag any regressions.
**Non-Goals:** Do NOT prescribe additional changes. Do NOT enter another optimization loop. Validate and report.

---

## INPUTS

- Rewritten skill file from 3.1-A
- Original Layer 1 scores (from 1.4-A Diagnostic Summary)
- Prescriptions that were applied (for regression tracing)

---

## INSTRUCTIONS

### Step 1: Re-Run All Layer 1 Checks

On the REWRITTEN file, execute:
1. Structure Audit (Four-Block scores)
2. Constraint Ratio calculation
3. Specificity scoring
4. Guardrail pattern check
5. Anti-slop scan
6. Production readiness check
7. Failure mode scan (if applicable)

**Use the same methodology as the original Layer 1 run.** Same rubrics, same thresholds, same counting methods.

### Step 2: Compare Before/After Scores

For each dimension, calculate:
```
delta = new_score - original_score
direction = improved | unchanged | regressed
```

### Step 3: Verify Critical Dimensions

ALL critical dimensions must now PASS:
- Four-Block Compliance ≥16 ✓ or ✗
- Constraint Ratio ≥0.60 ✓ or ✗
- Specificity Score ≥80% ✓ or ✗
- Guardrail Coverage ≥type threshold ✓ or ✗
- Slop Density ≤2.0 ✓ or ✗

If ANY critical dimension still fails after optimization: **FLAG AS REGRESSION OR INCOMPLETE FIX.**

### Step 4: Check for Regressions

A regression means a dimension scored LOWER after optimization. This should never happen, but check:
- Did constraint rewrites accidentally reduce specificity? (made things too rigid)
- Did guardrail additions bloat the file and reduce coherence?
- Did exemplar content introduce slop?
- Did restructuring break section flow?

If regression detected:
- Identify which prescription caused it
- Assess severity (minor regression in non-critical vs critical regression)
- Determine if it's acceptable trade-off (e.g., constraint ratio up but specificity slightly down)

### Step 5: Calculate Net Improvement

```
aggregate_before = sum(all dimension scores as percentages of max)
aggregate_after = sum(all dimension scores as percentages of max)
net_improvement = aggregate_after - aggregate_before
```

### Step 6: Gate Decision

```
IF all critical dimensions PASS AND no regressions:
    → APPROVE: Optimization successful. Package for delivery.

IF all critical dimensions PASS AND minor regressions in non-critical:
    → APPROVE WITH NOTES: Package for delivery, note trade-offs.

IF any critical dimension still FAILS:
    → REJECT: Identify the specific prescription that should have fixed this.
    → If possible: Apply targeted fix and re-validate (max 1 iteration)
    → If not fixable: Deliver original with AUDIT-REPORT only (don't deliver a failed optimization)

IF net improvement is negative (overall worse):
    → REJECT: Discard rewritten version. Deliver original with AUDIT-REPORT.
    → Note: "Optimization did not improve this skill. Original is superior."
```

---

## CONSTRAINTS

- NEVER approve a rewrite where critical dimensions regressed
- NEVER approve a rewrite where net improvement is negative
- NEVER enter an infinite optimization loop (max 1 retry if REJECT)
- MUST use identical methodology for before/after comparison (same rubrics, same thresholds)
- MUST be honest about regressions — don't hide trade-offs
- If the optimization genuinely failed: The correct answer is to deliver the ORIGINAL with the audit report (diagnosis is still valuable even if optimization didn't work)
- A regression in a non-critical dimension is acceptable IF critical dimensions all improved AND net improvement is positive
- The self-audit is the FINAL quality gate — nothing ships without passing this check

---

## FALLBACK

- If self-audit can't be completed (system error): Deliver rewritten version with "UNVALIDATED" flag and recommend human review
- If scores are identical before/after: Report as "UNCHANGED — prescriptions had no measurable impact" and recommend reviewing prescription quality
