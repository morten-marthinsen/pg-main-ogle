# 1.1-A: Structure Auditor
*Evaluate Four-Block Blueprint compliance*

---

## PURPOSE

Score the target skill against the Four-Block Blueprint (Purpose, Instructions, Reference, Output). Each block is scored 0-5 using the rubric in QUALITY-STANDARDS.md §2.1. This determines whether the skill has the foundational architecture for reliable execution.

**Scope:** Score four blocks. Identify structural gaps. Produce per-block assessment.
**Non-Goals:** Do NOT rewrite content. Do NOT prescribe fixes. Diagnose only.

---

## INPUTS

- Consumption receipt from 1.0-A (required)
- The target skill file content (already in context from 1.0-A)

---

## INSTRUCTIONS

### Step 1: Evaluate Block 1 — PURPOSE

Score 0-5 based on:
- Is there an explicit purpose/objective statement?
- Does it identify WHO this skill serves (which agent, which user)?
- Does it identify WHAT PROBLEM it solves?
- Does it define SCOPE (what's in bounds)?
- Does it define NON-GOALS (what's explicitly out)?
- Does it define SUCCESS CRITERIA?

**Scoring:**
| 0 | No purpose statement |
| 1 | Vague purpose ("helps with things") |
| 2 | General purpose, missing specifics |
| 3 | Clear purpose, missing audience OR problem |
| 4 | Clear purpose with audience AND problem |
| 5 | Precise purpose + scope boundaries + non-goals + success criteria |

### Step 2: Evaluate Block 2 — INSTRUCTIONS

Score 0-5 based on:
- Are execution steps explicit and sequenced?
- Are there decision rules (IF conditions)?
- Are dependencies between steps stated?
- Are conditional paths handled?
- Are edge cases addressed?
- Is there fallback logic for failures?

**Scoring:**
| 0 | No instructions (just description) |
| 1 | Vague directions ("do the analysis") |
| 2 | General steps without decision rules |
| 3 | Clear steps with some decision logic |
| 4 | Explicit steps + decision rules + sequence + dependencies |
| 5 | Complete protocol + conditional logic + edge cases + fallbacks |

### Step 3: Evaluate Block 3 — REFERENCE

Score 0-5 based on:
- Is context/background provided for the agent?
- Are there exemplars of good output?
- Are there anti-exemplars of bad output?
- Is domain knowledge embedded (not just referenced)?
- Are style/voice guides included (if applicable)?

**Scoring:**
| 0 | No reference material or exemplars |
| 1 | Mentions "refer to X" without content |
| 2 | Minimal context, no exemplars |
| 3 | Context provided, no output exemplars |
| 4 | Context + 1-2 output exemplars |
| 5 | Full context + 2-3 exemplars + anti-exemplars |

### Step 4: Evaluate Block 4 — OUTPUT

Score 0-5 based on:
- Is the output format specified?
- Is the structure defined (sections, hierarchy)?
- Are length constraints stated?
- Are quality criteria defined?
- Are validation checks specified (how agent confirms "done")?

**Scoring:**
| 0 | No output specification |
| 1 | "Produce a report" (undefined) |
| 2 | General format mentioned |
| 3 | Format specified with sections |
| 4 | Precise format + structure + length constraints |
| 5 | Complete: format + structure + length + quality criteria + validation |

### Step 5: Calculate Aggregate

- Sum all four block scores (0-20)
- Compare to threshold (≥16 passes)
- Identify which block(s) are weakest

---

## OUTPUT SPECIFICATION

```yaml
structure_audit:
  target: [filename]
  total_score: [0-20]
  threshold: 16
  status: [PASS | FAIL]

  blocks:
    purpose:
      score: [0-5]
      evidence: "[quote or description of what was found]"
      gap: "[what's missing, if anything]"

    instructions:
      score: [0-5]
      evidence: "[quote or description]"
      gap: "[what's missing]"

    reference:
      score: [0-5]
      evidence: "[quote or description]"
      gap: "[what's missing]"

    output:
      score: [0-5]
      evidence: "[quote or description]"
      gap: "[what's missing]"

  weakest_block: [which block scored lowest]
  critical_finding: [primary structural issue, if any]
```

---

## CONSTRAINTS

- NEVER score based on content quality — only structural presence/completeness
- NEVER give a 5 out of generosity — 5 requires ALL criteria met
- NEVER give a 0 if ANY relevant content exists — 0 means literally nothing
- MUST quote or specifically reference the evidence for each score
- MUST identify gaps, not just scores — "what's missing" is the diagnostic value
- If a block is split across multiple sections, still evaluate as one unit
- If content is implicit but not explicit, score as present but note "implicit — should be explicit"

---

## FALLBACK

- If consumption receipt is missing: HALT. Cannot audit without ingestion.
- If target has non-standard structure (not Markdown): Adapt evaluation to whatever structure exists, note non-standard format.
