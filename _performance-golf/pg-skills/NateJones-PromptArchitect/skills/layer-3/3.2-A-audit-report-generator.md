# 3.2-A: Audit Report Generator
*Produce the final AUDIT-REPORT.md deliverable*

---

## PURPOSE

Compile all diagnostic findings, prescriptions applied, and quality improvements into a single comprehensive audit report. This is the primary human-readable deliverable — it tells the skill owner exactly what was found, what was fixed, and what remains.

**Scope:** Synthesize all Layer 1-3 outputs into one coherent report. Include before/after scores. Provide actionable next steps.
**Non-Goals:** Do NOT perform new analysis. Only compile and present what was already produced.

---

## INPUTS

- Diagnostic Summary from 1.4-A
- All Layer 2 prescription outputs
- Rewrite metadata from 3.1-A
- Before/after quality scores

---

## INSTRUCTIONS

### Step 1: Write Executive Summary

One paragraph covering:
- What was audited (target name, type, size)
- Overall health rating (before → after)
- Number of findings and prescriptions
- Most significant improvement made
- Remaining risks (if any)

### Step 2: Compile Score Comparison

Side-by-side before/after for every scored dimension:

| Dimension | Before | After | Change | Status |
|-----------|--------|-------|--------|--------|
| [dim] | [score] | [score] | [+/-] | [improved/unchanged/regressed] |

### Step 3: Detail Findings by Priority

For each finding from Layer 1, in priority order:
- What was found (specific, with evidence)
- Why it matters (risk if unaddressed)
- What was prescribed (Layer 2 reference)
- What was implemented (Layer 3 action taken)
- Result (did the score improve?)

### Step 4: Document All Changes Made

Complete list of modifications to the skill file:
- Constraint rewrites (count, examples)
- Guardrails added (which patterns)
- Output spec changes (what was tightened)
- Exemplars added (count, types)
- Structural changes (context layers, positional)
- Chain-of-Refinement added (Y/N, dimensions)
- Wake words extracted (count, names)

### Step 5: Identify Remaining Risks

After optimization, what's still imperfect:
- Dimensions still below optimal (passing but not excellent)
- Issues deferred (P3-P4 findings not addressed)
- Structural concerns (skill may need deeper rebuild in future)
- Dependencies on external factors (parent system, tools, etc.)

### Step 6: Provide Recommendations

Actionable next steps for the skill owner:
- Immediate actions (if any critical items remain)
- Future improvements (for next audit cycle)
- System-level recommendations (if audit revealed parent system issues)

## CONSTRAINTS

- NEVER include findings without evidence (every finding must have a specific reference)
- NEVER present scores without context (threshold, what the number means)
- MUST include both before AND after scores (the improvement is the value demonstration)
- MUST be honest about remaining risks (don't oversell the optimization)
- MUST be concise — the report should be 100-200 lines (not a novel)
- All recommendations must be ACTIONABLE (not "consider improving quality")
- If no changes were made (PASS recommendation): Produce a brief "Clean Audit" report confirming all standards met
- The report is for a HUMAN reader — write clearly, not in jargon-heavy technical style

---

## FALLBACK

- If audit was AUDIT mode (no optimization performed): Report shows diagnostics only, with "Recommended Actions" instead of "Changes Applied"
- If some Layer 2/3 skills failed: Note gaps in report, provide scores for what was completed
