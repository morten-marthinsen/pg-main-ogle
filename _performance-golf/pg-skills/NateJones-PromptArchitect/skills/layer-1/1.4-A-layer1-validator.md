# 1.4-A: Layer 1 Validator
*Aggregate all diagnostic scores and determine recommendation*

---

## PURPOSE

Collect all Layer 1 diagnostic outputs, aggregate into a unified Diagnostic Summary, calculate overall health rating, and determine the recommendation (PASS / OPTIMIZE / REWRITE / REBUILD). This is the gate that decides whether to stop (AUDIT mode) or continue to Layer 2 (OPTIMIZE mode).

**Scope:** Aggregate scores. Apply health rating logic. Produce Diagnostic Summary. Make gate decision.
**Non-Goals:** Do NOT prescribe fixes. Do NOT begin rewriting. Only aggregate and decide.

---

## INPUTS

All Layer 1 skill outputs:
- 1.0-A: Consumption receipt (structure map, line counts)
- 1.1-A: Structure audit (four-block scores)
- 1.1-B: Constraint ratio (ratio value)
- 1.1-C: Specificity score (goldilocks percentage)
- 1.2-A: Guardrail audit (pattern count, missing patterns)
- 1.2-B: Anti-slop scan (density score, instances)
- 1.3-A: Production readiness (principle scores)
- 1.3-B: Failure mode scan (modes detected)

---

## INSTRUCTIONS

### Step 1: Collect All Scores

Assemble the complete scoring picture:

```yaml
scores:
  structural:
    four_block: [0-20]
    constraint_ratio: [0.0-1.0]
    specificity: [0-100%]

  production:
    guardrails: [0-7]
    slop_density: [X per 100 lines]
    production_principles: [0-12]
    failure_modes: [0-7 detected]
```

### Step 2: Identify Critical Dimension Status

Check each CRITICAL dimension against threshold:

| Dimension | Threshold | Status |
|-----------|-----------|--------|
| Four-Block Compliance | ≥16 | PASS/FAIL |
| Constraint Ratio | ≥0.60 | PASS/FAIL |
| Specificity Score | ≥80% | PASS/FAIL |
| Guardrail Coverage | ≥type threshold | PASS/FAIL |
| Slop Density | ≤2.0 | PASS/FAIL |

Count: How many critical dimensions are FAILING?

### Step 3: Apply Health Rating

```
EXCELLENT: All critical PASS + all non-critical at threshold
GOOD:      All critical PASS + ≥75% non-critical at threshold
FAIR:      All critical PASS + <75% non-critical at threshold
POOR:      1-2 critical dimensions FAILING
CRITICAL:  3+ critical dimensions FAILING
```

### Step 4: Determine Recommendation

| Health | Recommendation | Meaning |
|--------|---------------|---------|
| EXCELLENT | PASS | No changes needed. Skill meets all standards. |
| GOOD | PASS (with notes) | Meets standards. Optional improvements listed. |
| FAIR | OPTIMIZE | Passes critical checks but needs non-critical improvements. Layer 2 prescriptions. |
| POOR | REWRITE | 1-2 critical failures. Full Layer 2 + Layer 3 pipeline needed. |
| CRITICAL | REBUILD | Fundamental problems. Start from SKILL-TEMPLATE. |

### Step 5: Prioritize Findings

Rank all findings by impact:

**Priority 1 (Critical):**
- Any critical dimension failing
- Guardrail patterns missing for production skill
- Failure modes detected (severity HIGH)

**Priority 2 (High):**
- Constraint ratio below 0.50 (instruction-heavy)
- Specificity below 60% (mostly vague)
- No output validation present
- Anti-slop density above 4.0

**Priority 3 (Medium):**
- Non-critical dimensions below threshold
- Missing production principles
- Partial guardrail coverage
- Moderate slop density (2.0-4.0)

**Priority 4 (Low):**
- Minor specificity issues
- Single missing guardrail pattern
- Positional reinforcement absent
- Low-severity failure modes

### Step 6: Generate Diagnostic Summary

Produce the final summary document.

## CONSTRAINTS

- NEVER upgrade a health rating if critical dimensions are failing (POOR is POOR regardless of non-critical scores)
- NEVER recommend PASS if any critical dimension fails
- NEVER recommend REBUILD unless 3+ critical dimensions fail (that's the threshold)
- MUST include specific evidence references for every finding (not just "structure is weak")
- MUST prioritize findings — not everything is equally important
- The Diagnostic Summary is the PRIMARY deliverable in AUDIT mode — it must be comprehensive and actionable
- If mode is AUDIT: Stop here. Produce only the Diagnostic Summary.
- If mode is OPTIMIZE or BUILD: Pass the summary to Layer 2 and continue.

---

## GATE LOGIC

```
IF execution_mode == AUDIT:
    → Save Diagnostic Summary to projects/[target]/AUDIT-REPORT.md
    → STOP. Audit complete.

IF execution_mode == OPTIMIZE:
    → Save Diagnostic Summary
    → Pass all findings to Layer 2
    → CONTINUE to Phase 2

IF execution_mode == BUILD:
    → This validator runs AFTER the build (as self-audit)
    → If PASS: Deliver the built skill
    → If not PASS: Loop back to fix issues before delivery
```

---

## FALLBACK

- If any Layer 1 skill produced an error instead of output: Note the gap, score that dimension as UNKNOWN (not 0), and flag for manual review
- If multiple skills report N/A: Reduce the critical dimension count accordingly (don't fail a skill for not needing a feature)
