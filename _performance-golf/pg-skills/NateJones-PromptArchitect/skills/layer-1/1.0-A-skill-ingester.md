# 1.0-A: Skill Ingester
*Read, parse, and produce consumption receipt for target skill file(s)*

---

## PURPOSE

Ingest one or more skill files completely, parse their structure, and produce a verified consumption receipt. This is the foundational step — nothing downstream works if the target isn't fully consumed and understood.

**Scope:** Read target file(s). Produce structured consumption receipt. Identify immediate structural elements.
**Non-Goals:** Do NOT score, judge, or prescribe at this stage. Only consume and document.

---

## INSTRUCTIONS

### Step 1: Read Target Completely

- Read the entire file using the Read tool
- NEVER skim, skip sections, or assume based on file name
- If file exceeds single-read capacity, use chunked reading (offset/limit parameters)
- Track total line count

### Step 2: Parse Structure

Identify and catalog:
- All H1/H2/H3 headings (document skeleton)
- All code blocks (templates, formats, examples)
- All tables (structured data)
- All bullet/numbered lists (instructions, constraints, rules)
- All bold/emphasis text (key terms, warnings)
- Presence or absence of: purpose statement, instructions, reference/exemplars, output spec

### Step 3: Classify Content Types

For each major section, classify as:
- **CONSTRAINT:** Boundary-setting content (NEVER, ALWAYS, MUST NOT)
- **INSTRUCTION:** Action-directing content (do this, create that)
- **CONTEXT:** Background/reference information
- **EXEMPLAR:** Example inputs or outputs
- **OUTPUT_SPEC:** Format/structure definitions for deliverables
- **META:** Document info, version history, non-operational content

### Step 4: Produce Consumption Receipt

---

---

## CONSTRAINTS

- NEVER produce a receipt without reading the complete file
- NEVER skip sections because they seem familiar or repetitive
- NEVER begin Layer 1 diagnostics before this receipt is complete
- MUST read in sequential order (don't jump around)
- MUST count actual lines, not estimate
- MUST identify all four blocks explicitly, even if marking as "Absent"
- If file is >2000 lines, produce a chunked consumption log instead of single receipt

---

## FALLBACK

- If file cannot be read (permission error, missing): Log the error, report to orchestrator, do NOT continue with assumptions
- If file is empty: Report as empty, recommend REBUILD mode
- If file is not a skill file (wrong format entirely): Report as non-skill, recommend manual review
