# 2.3-B: Context Layer Separator
*Split mixed context into Layer 1 (Deterministic) and Layer 2 (Probabilistic)*

---

## PURPOSE

Reorganize the target skill's content into the Two-Layer Context Architecture (ARCHITECTURE-PRD §4.3). Most skills mix immutable rules with context-dependent guidance, reducing clarity and enforcement. Separating them ensures critical rules are never overridden by adaptive logic.

**Scope:** Classify every operational statement as Layer 1 or Layer 2. Propose restructured organization. Verify no information lost.
**Non-Goals:** Do NOT rewrite content. Only REORGANIZE into two clearly labeled layers.

---

## INPUTS

- Target skill file content
- Constraint Ratio output from 1.1-B (constraints already identified)
- Guardrail Auditor output from 1.2-A (binary rules already identified)

---

## INSTRUCTIONS

### Step 1: Define Layer Classification

**Layer 1 — Deterministic (Hard Rules):**
Content that NEVER changes regardless of input, context, or situation:
- Identity statements (who the agent is/isn't)
- Binary constraints (ALWAYS/NEVER)
- Format requirements (structural invariants)
- Safety boundaries (what's forbidden)
- Quality floors (minimum acceptable standards)
- Scope limitations (what's in/out of bounds)

**Layer 2 — Probabilistic (Soft Guidance):**
Content that ADAPTS based on context, input, or situation:
- Conditional logic (IF situation X, THEN approach Y)
- Quality preferences (aim for Z when possible)
- Style guidance (tone shifts based on audience)
- Escalation rules (simple → complex based on input)
- Heuristics (general rules with known exceptions)
- Trade-off navigation (when two principles conflict, prefer X)

### Step 2: Classify Every Operational Statement

Go through the skill line by line. For each operational statement (not headers, not metadata):
- Tag as L1 (deterministic) or L2 (probabilistic)
- If unclear: ask "Would violating this EVER be acceptable in some context?"
  - NO → Layer 1
  - YES (in some contexts) → Layer 2

### Step 3: Check for Misplaced Content

Common misplacements:
- **L1 content buried in middle sections** → Should be first
- **L2 content in the constraints section** → Should be separated (it's guidance, not constraint)
- **L1 and L2 mixed in same paragraph** → Should be split
- **Critical rules only appearing in examples** → Should be explicit L1 statements

### Step 4: Propose Restructured Layout

```markdown
## LAYER 1: INVARIANTS (deterministic — never override)

[All L1 content, organized by type:]
- Identity rules
- Binary constraints (ALWAYS/NEVER)
- Format requirements
- Scope boundaries
- Quality floors

---

## LAYER 2: ADAPTIVE (probabilistic — context-dependent)

[All L2 content, organized by type:]
- Conditional logic (IF/THEN)
- Style/tone guidance
- Escalation rules
- Preference heuristics
- Trade-off navigation
```

### Step 5: Verify Completeness

Check that:
- Every operational statement from the original is present in one layer
- No content was lost in reorganization
- No content was duplicated (unless intentional positional reinforcement)
- Layer 1 content is positioned FIRST (positional reinforcement pattern)

### Step 6: Assess Current State

Rate the current mixing level:
- **CLEAN:** Already well-separated (minimal work needed)
- **MIXED:** Some L1/L2 interleaving but sections are mostly coherent
- **TANGLED:** L1 and L2 thoroughly mixed throughout, significant reorganization needed

## CONSTRAINTS

- NEVER delete content during reorganization — only move
- NEVER reclassify a clear ALWAYS/NEVER statement as Layer 2 (if it says NEVER, it's Layer 1)
- NEVER reclassify conditional logic (IF/THEN) as Layer 1 (conditions = probabilistic by definition)
- MUST maintain the skill's logical flow — don't just alphabetize rules
- MUST group related Layer 1 rules together (identity cluster, format cluster, scope cluster)
- MUST group related Layer 2 rules together (audience-dependent cluster, complexity-dependent cluster)
- If separation would break the skill's readability: Propose a compromise (e.g., L1 section at top, L2 integrated into instructions)
- Layer 1 MUST come before Layer 2 in document order (positional reinforcement)

---

## FALLBACK

- If skill is already cleanly separated: Report "Already well-structured — no separation needed"
- If skill is too short for meaningful separation (<20 operational statements): Propose inline markers (prefix L1 rules with "INVARIANT:" ) instead of full restructure
