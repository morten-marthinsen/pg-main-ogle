# Microskill 4.2: Multi-Pass Revision

**Layer:** 4 (Fix Application)
**Purpose:** Execute multiple focused revision passes for systematic improvement

---

## MULTI-PASS STRATEGY

Instead of trying to fix everything at once, use focused passes:

```
Pass 1: FLOW PASS - Ensure momentum and transitions
Pass 2: PROOF PASS - Strengthen believability
Pass 3: EMOTION PASS - Amplify desire and urgency
Pass 4: CLARITY PASS - Eliminate confusion
Pass 5: ECONOMY PASS - Tighten and trim
```

---

## PASS 1: FLOW PASS

**Focus:** Momentum, transitions, reader engagement

### Flow Pass Checklist

```yaml
flow_pass:
  objective: "Ensure copy pulls reader forward without stopping"

  review_areas:
    opening_velocity:
      current_state: "[assessment]"
      fixes_applied:
        - "[fix 1]"
        - "[fix 2]"
      post_fix_state: "[improved | same | needs more work]"

    sentence_to_sentence_pull:
      weak_spots_identified: "[count]"
      fixes_applied:
        - location: "[where]"
          before: "[original]"
          after: "[revised]"
      post_fix_state: "[assessment]"

    paragraph_momentum:
      energy_dead_spots: "[count]"
      fixes_applied:
        - location: "[where]"
          fix: "[what was done]"
      post_fix_state: "[assessment]"

    section_transitions:
      jarring_transitions: "[count]"
      fixes_applied:
        - transition: "[from A to B]"
          before: "[original transition]"
          after: "[revised transition]"
      post_fix_state: "[assessment]"

    engagement_hooks:
      hooks_added: "[count]"
      locations: ["[where hooks added]"]

  flow_pass_result:
    improvements_made: "[count]"
    remaining_issues: "[count]"
    flow_score_before: "[1-10]"
    flow_score_after: "[1-10]"
```
