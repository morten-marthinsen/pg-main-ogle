# Quality Checkpoints
## Section-by-Section Evaluation Criteria

---

## HOOK Quality Checkpoint

| Criterion | Pass | Fail |
|-----------|------|------|
| Self-identification in first 2 sentences | Reader can say "that's me" immediately | Reader unsure who this is for |
| Parenthetical objection handler present | Primary objection neutralized | No objection handling |
| Specific number included | At least one metric (strokes, yards, time) | Only vague promises |
| Dual promise structure | Immediate result + permanence | Single-dimensional promise |
| Length | 3-5 sentences | Too long or too short |
| Momentum | Compels continued reading | Could stop here |

---

## LEAD Quality Checkpoint

| Criterion | Pass | Fail |
|-----------|------|------|
| "This is NOT" sequence | 3-5 category negations | Fewer than 3 or none |
| Exclusivity signals | At least 2 present | No exclusivity framing |
| Promise amplification | Hook promise expanded | Mere repetition |
| Audience expansion | Inclusive language present | Narrow framing |
| Transition to proof | Clear bridge statement | Abrupt jump |

---

## PROOF STACK Quality Checkpoint

| Criterion | Pass | Fail |
|-----------|------|------|
| Testimonial count | 3-5 testimonials | Fewer than 3 |
| Specificity | Each has numbers/metrics | Vague claims |
| Objection crusher | At least one present | No objection handling |
| Expectation-exceeding frame | Used at least once | Never used |
| Sequence logic | Dramatic → Emotional flow | Random order |
| Transition | Clear bridge to problem | Abrupt jump |

---

## PROBLEM/PAIN Quality Checkpoint

| Criterion | Pass | Fail |
|-----------|------|------|
| Root cause named | Specific external cause identified | Vague or blames reader |
| External blame shift | "Not your fault" language | Implies reader failure |
| Pain point count | 4-6 specific pains | Fewer than 4 |
| Pain types | Both functional and emotional | Single dimension |
| Recognition language | "You know that feeling..." | No shared experience |
| Hope transition | Clear bridge to story | Stays in agitation |

---

## GURU ORIGIN STORY Quality Checkpoint

| Criterion | Pass | Fail |
|-----------|------|------|
| Humble opening | Relatable beginnings | Starts with success |
| Struggle sequence | Obstacles/sacrifice shown | Easy path to success |
| Turning point | Specific moment identified | Vague "over time" |
| Credibility markers | At least 3 woven in | Bare assertion |
| Mission articulation | Purpose beyond profit | Just selling |
| Offer connection | Story explains THIS offer | Disconnected |
| Length | 12-18% of total | Too long or too short |

---

## MECHANISM Quality Checkpoint

| Criterion | Pass | Fail |
|-----------|------|------|
| Proprietary name | Named, ownable system | Generic description |
| Step structure | 4-6 numbered steps | Unstructured |
| "So you can" benefits | Each step has benefit | Features only |
| Technology anchor | Credible tech/method | Unsupported claims |
| "Why it works" logic | Clear explanation | No reasoning |
| Experience elements | Demonstration/sensory | Abstract only |
| Length | 15-20% of total | Too long or too short |

---

## OFFER REVEAL Quality Checkpoint

| Criterion | Pass | Fail |
|-----------|------|------|
| Ceremonial intro | Framed reveal | Just lists |
| Component structure | Each has dimensionalized benefit | Features only |
| Required components | All 5 types present | Missing elements |
| Bonus separation | Distinct from core | Buried |
| Value anchors | 2-3 price comparisons | No anchoring |
| Price hidden | Not revealed | Price shown |
| Transition | Clear to qualification | Abrupt |

---

## QUALIFICATION Quality Checkpoint

| Criterion | Pass | Fail |
|-----------|------|------|
| Selectivity statement | "Not for everyone" | Open to all |
| Requirement count | 3-4 specific | Too few or too many |
| Legitimate reason | Explained why selective | Arbitrary |
| Scarcity signal | Limited availability | Unlimited |
| Tone | Respectful challenge | Insulting or begging |
| Transition | Conditional welcome | Weak bridge |

---

## CLOSE Quality Checkpoint

| Criterion | Pass | Fail |
|-----------|------|------|
| CTA friction | Low (10-min call) | High (purchase now) |
| "Even if no" value | Call has standalone value | All-or-nothing |
| Urgency layers | At least 3 types | Single urgency |
| Future-pacing | Vivid transformation scene | No visualization |
| Guarantee (if used) | Named, confident | Weak/generic |
| Polarizing question | Binary choice at end | Soft ending |
| CTA clarity | Specific button instruction | Vague next step |

## Global Quality Checklist

### Voice Consistency
- [ ] Reading level: 7th-9th grade throughout
- [ ] "You/your" density appropriate per section
- [ ] No forbidden words present
- [ ] Confidence without arrogance maintained
- [ ] Conversational but professional tone

### Structural Integrity
- [ ] All 9 beats present in correct order
- [ ] Section proportions within guidelines
- [ ] Clear transitions between all sections
- [ ] No beat skipped or abbreviated

### Persuasion Completeness
- [ ] All Tier 1 elements present
- [ ] Majority of Tier 2 elements present
- [ ] Tier 3 elements included where contextually appropriate
- [ ] No unsupported claims

### Technical Accuracy
- [ ] Golf terminology used correctly
- [ ] Numbers internally consistent
- [ ] No contradictions between sections
- [ ] All testimonials plausible

---

## Final "Ready to Publish" Checklist

Before submitting any script, verify:

1. [ ] **Hook** qualifies reader within first sentence
2. [ ] **Lead** includes 3+ "This is NOT" statements
3. [ ] **Proof** includes 3+ specific testimonials
4. [ ] **Problem** includes external blame shift
5. [ ] **Story** includes humble origin + turning point
6. [ ] **Mechanism** has named system + numbered steps
7. [ ] **Offer** has value anchors but no price reveal
8. [ ] **Qualification** reverses sales dynamic
9. [ ] **Close** stacks 3+ urgency types
10. [ ] **Voice** passes "19th hole test" (sounds like expert advice to a friend)
